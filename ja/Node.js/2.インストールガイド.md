# Node.js インストールガイド

## 公式インストール方法

Node.jsは、さまざまなオペレーティングシステム向けの公式インストーラーを提供しています。すべての公式方法は、Node.jsプロジェクトから直接入手できる、改ざんされていないバイナリを保証します。

### 公式ウェブサイトからのダウンロード

Node.jsをインストールする最も簡単な方法は、[公式Node.jsウェブサイト](https://nodejs.org/ja/download)から適切なインストーラーをダウンロードすることです。

**利用可能なバージョン：**
- **LTS（長期サポート）**：ほとんどのユーザーに推奨されます。安定性と長期サポートを提供します。
- **Current**：最新機能を含み、サポート期間は短めです。開発やテストに適しています。

### パッケージマネージャー

#### macOS（Homebrew）
```bash
# Homebrewがまだインストールされていない場合
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# LTSバージョンをインストール
brew install node@20

# または最新のCurrentバージョンをインストール
brew install node
```

#### Linux（各種ディストリビューション）

**Ubuntu/Debian：**
```bash
# NodeSourceリポジトリを使用（推奨）
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora：**
```bash
# NodeSourceリポジトリを使用
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# または新しいバージョンの場合：sudo dnf install nodejs
```

#### Windows
- 公式ウェブサイトからWindowsインストーラー（.msi）をダウンロード
- インストーラーを実行し、セットアップウィザードに従う
- Node.jsとnpmが自動的にPATHに追加されます

### Nodeバージョンマネージャー

複数のNode.jsバージョンを扱う必要がある開発者には、バージョンマネージャーが優れたソリューションを提供します。

#### nvm（Node Version Manager）- macOS/Linux
```bash
# nvmをインストール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# LTSバージョンをインストールして使用
nvm install --lts
nvm use --lts

# 特定のバージョンをインストール
nvm install 20.14.0
nvm use 20.14.0

# 利用可能なバージョンを一覧表示
nvm list-remote
```

#### nvm-windows - Windows
- [nvm-windows GitHubリリース](https://github.com/coreybutler/nvm-windows/releases)からダウンロード
- インストール手順に従う
- Unixベースのnvmと同様のコマンドを使用

## 検証

インストール後、Node.jsとnpmが正しくインストールされていることを確認します：

```bash
# Node.jsのバージョンを確認
node --version

# npmのバージョンを確認
npm --version

# 基本的な機能をテスト
node -e "console.log('Node.js is working!')"
```

## インストール後の設定

### グローバルnpmパッケージの場所
デフォルトでは、グローバルにインストールされたnpmパッケージはシステムディレクトリに配置されます。権限の問題を回避するために、npmを別のディレクトリを使用するように設定することを検討してください：

```bash
# グローバルパッケージ用のディレクトリを作成
mkdir ~/.npm-global

# npmを新しいディレクトリを使用するように設定
npm config set prefix '~/.npm-global'

# ディレクトリをPATHに追加（~/.bashrcまたは~/.zshrcに追加）
export PATH=~/.npm-global/bin:$PATH
```

### 環境変数
Node.js開発のための一般的な環境変数：

- `NODE_ENV`：本番環境では`production`に設定
- `NODE_OPTIONS`：Node.jsプロセスにコマンドラインオプションを渡す
- `NPM_CONFIG_*`：環境変数を通じてnpmの動作を設定

## リリーススケジュールの理解

Node.jsは予測可能なリリーススケジュールに従っています：
- **Currentリリース**：6か月ごとに新しいメジャーバージョン
- **LTSリリース**：偶数バージョンは6か月後にLTSになります
- **Active LTS**：18か月のアクティブサポート
- **Maintenance LTS**：12か月のメンテナンスサポート（合計30か月）

本番アプリケーションにはLTSバージョンを選択して、安定性と長期サポートを確保してください。

## セキュリティに関する考慮事項

- 常に公式ソース（nodejs.orgまたは公式パッケージリポジトリ）からダウンロードする
- 可能であれば、提供されたSHASUMSを使用してダウンロードしたバイナリを検証する
- セキュリティパッチを受けるためにNode.jsを最新の状態に保つ
- 本番環境ではLTSバージョンを使用して、セキュリティ更新を保証する
