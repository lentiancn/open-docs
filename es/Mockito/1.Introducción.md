# Mockito Introducción

## Qué es Mockito

Mockito es un potente framework de mocking para pruebas unitarias de Java, disenado para el Desarrollo Guiado por Pruebas (TDD) y el Desarrollo Guiado por Comportamiento (BDD). Permite a los desarrolladores crear objetos simulados que emulan el comportamiento de objetos reales de manera controlada, logrando pruebas unitarias independientes y confiables.

## Por qué elegir Mockito

### API limpia y simple

Mockito ofrece una API intuitiva y fácil de usar, con una curva de aprendizaje suave. Los desarrolladores pueden comenzar rápidamente y escribir casos de prueba claros con código simplificado. En comparacion con otros frameworks de mocking complejos, la sintaxis de Mockito es mas natural, y leer código de prueba es como leer código de negocio normal.

### Funciones potentes

Mockito soporta varias características avanzadas, incluyendo verificación de llamadas a métodos, configuración de valores de retorno, simulacion de excepciones, coincidencia de argumentos, verificación de orden de llamadas, y mas. Estas capacidades satisfacen varios escenarios de prueba complejos y ayudan a los desarrolladores a cubrir completamente la lógica del código.

### Excelente legibilidad

El código de prueba escrito con Mockito tiene una excelente legibilidad. Cuando una prueba falla, Mockito puede proporcionar mensajes de error de verificación claros, ayudando a los desarrolladores a localizar rápidamente el problema. Esta presentación de errores amigable mejora enormemente la eficiencia de depuración.

### Comunidad activa

Mockito es el framework de mocking mas popular en el ecosistema Java, con una gran base de usuarios y una comunidad activa. El framework se mantiene y actualiza continuamente, con documentacion completa y rica. Cuando surgen problemas, es muy fácil encontrar soluciones. En Stack Overflow, Mockito ha recibido la mayor cantidad de votos, siendo reconocido como el mejor framework de mocking para Java.

## Conceptos basicos

### Objetos Mock

Un objeto mock es un sustituto del objeto real durante la prueba. Al crear objetos mock, se pueden aislar las dependencias externas, asegurando que las pruebas se enfoquen unicamente en la lógica de la unidad bajo prueba, sin verse afectadas por sistemas externos.

### Stubbing

El stubbing se usa para configurar el comportamiento de un objeto mock cuando se llaman métodos especificos. Con la sintaxis when().thenReturn(), se puede controlar con precision las respuestas de los métodos simulados, permitiendo que las pruebas cubran diversos escenarios.

### Verificacion

La verificación se usa para confirmar que la interaccion entre el código bajo prueba y los objetos mock ocurre segun lo esperado. El metodo verify() puede verificar si un metodo fue llamado, cuantas veces fue llamado, con que argumentos, asegurando que el código bajo prueba utilize correctamente sus dependencias.

### Spy

Spy permite el mocking parcial de objetos reales. A diferencia de los objetos completamente simulados, los Spy mantienen la mayor parte del comportamiento del objeto real, simulando solo métodos especificos. Esto es util cuando se necesita probar la mayor parte de la lógica real mientras se simulan solo algunos métodos.

## Casos de uso tipicos

### Aislamiento de dependencias externas

Cuando el código bajo prueba depende de sistemas externos como bases de datos, servicios web o sistemas de archivos, Mockito puede simular estas dependencias externas, permitiendo que las pruebas se ejecuten sin necesidad de sistemas externos reales. Esto no solo acelera la ejecucion de pruebas, sino que juga elimina el riesgo de fallas de prueba causadas por sistemas externos inestables.

### Simulacion de modulos no terminados

En proyectos grandes, diferentes modulos pueden estar en diferentes etapas de desarrollo. Mockito puede simular modulos que aun no estan terminados, permitiendo que los modulos ya completados se prueben con anticipacion, evitando bloqueos en el desarrollo.

### Simulacion de objetos complejos

 Algunos objetos de negocio son muy complejos de construir, con multiples niveles de dependencias. Mockito fácilita la creacion de versiones simuladas de estos objetos complejos, enfocandose en probar la lógica del negocio en si.

## Integracion con JUnit

Mockito generalmente se usa junto con el framework de pruebas JUnit. Usando @RunWith(MockitoJUnitRunner.class) o @ExtendWith(MockitoExtension.class) de JUnit, los campos anotados con @Mock se inicializan automaticamente, simplificando la escritura del código de prueba. Esta combinacion representa la mejor practica para pruebas unitarias en Java.

## Compatibilidad de versiones

Mockito 5.x requiere JDK 11 o superior, soportando características nuevas de Java 8 como expresiones lambda. La serie Mockito 4.x todavia soporta JDK 8. Los desarrolladores pueden elegir la version apropiada segun los requisitos del proyecto.

## Resumen

Mockito es una herramienta de prueba esencial para desarrolladores de Java. Resuelve elegantemente el problema del aislamiento de dependencias en pruebas unitarias, permitiendo a los desarrolladores escribir código de prueba confiable y mantenible. Dominar Mockito mejorara significativamente la calidad del código y la eficiencia del desarrollo, siendo una habilidad que todo ingeniero Java debe poseer.
