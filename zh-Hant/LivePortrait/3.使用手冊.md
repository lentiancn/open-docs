# LivePortrait 使用手冊

本文檔詳細介紹LivePortrait的各種使用方法，包括基礎推理、Gradio介面、動物模式、影片編輯等功能。

## 基礎推理

### 圖像驅動

最基本的用法是使用靜態圖像作為來源，驅動影片作為動畫參考：

```bash
python inference.py
```

預設情況下，腳本會使用assets/examples/source目錄下的範例圖像和assets/examples/driving目錄下的範例驅動影片。執行成功後，結果將儲存在animations目錄中，輸出檔案命名為「來源檔案--驅動檔案_concat.mp4」。

輸出影片包含三個部分：驅動影片、來源圖像/影片、以及生成的動畫結果，按順序拼接在一起。

### 自訂輸入

您可以透過命令列參數指定自己的來源檔案和驅動檔案：

```bash
# 來源檔案為圖像
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# 來源檔案為影片
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

其中-s參數指定來源圖像或來源影片路徑，-d參數指定驅動影片路徑。

### 查看更多選項

查看所有可用的命令列參數：

```bash
python inference.py -h
```

參數包括輸出路徑、圖像尺寸、動畫速度、拼接方式等詳細配置。

## 使用Gradio Web介面

對於不熟悉命令列的使用者，LivePortrait提供了基於Gradio的圖形化介面。

### 啟動人類模式介面

```bash
python app.py
```

啟動後，開啟瀏覽器存取顯示的URL（通常是http://localhost:7860）。在介面中上傳來源圖像或影片，然後選擇驅動影片，點擊生成按鈕即可獲得動畫結果。

可以透過--server_port參數指定連接埠，--share參數產生公開連結，--server_name參數指定伺服器位址。

### 啟動動物模式介面

在Linux系統上使用NVIDIA GPU，可以啟動動物模式介面：

```bash
python app_animals.py
```

## 驅動影片規範

為了獲得最佳效果，驅動影片需要滿足以下要求：

**寬高比**：建議裁剪為1:1正方形（如512x512或256x256像素）。也可以啟用自動裁剪功能（--flag_crop_driving_video）。

**內容要求**：驅動影片應聚焦於頭部區域，類似於範例影片。應盡量減少肩部動作。

**首幀要求**：驅動影片的第一幀應為正面面部表情自然的中性面孔。這是確保動畫效果準確的關鍵。

### 自動裁剪

如果驅動影片不滿足上述要求，可以使用自動裁剪功能：

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

如果自動裁剪效果不理想，可以調整--scale_crop_driving_video和--vy_ratio_crop_driving_video參數來微調縮放和偏移。

## 使用運動範本

為了加速推理並保護隱私，LivePortrait支援使用預生成的運動範本檔案（.pkl格式）。這些範本檔案記錄了驅動影片中的運動資訊，可以重複使用而無需每次都處理原始影片。

### 使用方法

```肖像動畫
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# 肖像影片編輯
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

範本檔案不僅加速推理，還可以保護隱私——使用範本時無需上傳原始驅動影片。

## 肖像影片編輯（Video-to-Video）

LivePortrait支援對現有影片中的人物表情和頭部動作進行編輯修改。

### 基本用法

```bash
# 來源為圖像，驅動為影片
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### 調整動畫參數

**表情強度**：透過--driving_multiplier參數調整表情變化的強度。值大於1會增強表情變化，小於1會減弱。

**拼接控制**：使用--flag_stitching參數控制是否啟用拼接功能。關閉拼接可能會改變面部區域的呈現方式。

## 推理加速

### Torch Compile加速

LivePortrait支援使用PyTorch的torch.compile功能加速推理。首次推理時會觸發優化程序（約一分鐘），之後的推理速度將提升20%-30%。加速效果可能因CUDA版本不同而有所差異。

```bash
python app.py --flag_do_torch_compile
```

注意：torch.compile不支援Windows和macOS系統。

### 速度測試

專案提供了推理速度測試腳本：

```bash
python speed.py
```

測試結果會顯示各個模組的處理耗時，幫助您了解效能瓶頸。

## 動物模式

動物模式支援對貓和狗等寵物的肖像進行動畫處理。

### 使用條件

動物模式僅在以下環境中經過測試：Linux系統搭配NVIDIA GPU、Windows系統搭配NVIDIA GPU。

### 安裝需求

在使用動物模式之前，需要按照安裝指南中的說明安裝X-Pose依賴，並下載動物模型權重。

### 執行動物模式推理

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### 動物模式介面

```bash
python app_animals.py
```

## 區域控制

LivePortrait支援對圖像的特定區域進行動畫控制，實現更精細的效果調整。

### 啟用區域控制

在Gradio介面中，您可以找到區域控制選項，選擇要驅動的面部區域（如眼睛、嘴巴、頭部姿態等）。

## 肖像編輯

在Gradio介面中，使用者可以精確編輯來源肖像的各個屬性，包括表情、眼神方向、頭部角度等。

## 隱私保護

LivePortrait提供了多種隱私保護功能：

**運動範本**：使用.pkl範本檔案代替原始驅動影片，避免上傳敏感內容

**自動裁剪**：驅動影片自動裁剪聚焦頭部，減少背景資訊

**影片保護**：提供範本製作功能，將影片轉換為運動範本後再使用

## 輸出格式

預設輸出格式為MP4影片檔案，包含三個部分：驅動影片、來源內容、生成結果。可以使用--no_flag_concat參數只輸出生成的動畫部分。

## 社區工具整合

LivePortrait被多個社區專案整合使用：

**ComfyUI**：多個ComfyUI節點支援LivePortrait，提供即時預覽功能

**Stable Diffusion WebUI**：sd-webui-live-portrait擴充將LivePortrait整合到SD WebUI中

**FaceFusion**：3.0版本整合了LivePortrait作為表情恢復和面部編輯處理器

**HuggingFace Spaces**：線上體驗LivePortrait，無需本地安裝

## 最佳實踐

為了獲得最佳動畫效果，建議遵循以下最佳實踐：

選擇高品質的來源圖像：清晰的正面肖像效果最好

使用專業的驅動影片：正面、表情自然的影片驅動效果更佳

控制驅動影片長度：過長的驅動影片會增加處理時間並可能導致效果衰減

合理設定參數：根據實際效果調整表情強度和其他參數

注意光照一致：來源圖像和驅動影片的光照條件盡量保持一致
