# Nginx 安装指南

## 环境要求

Nginx 支持多种操作系统，包括：

- Linux（Debian、Ubuntu、CentOS、RHEL、Alpine 等）
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## 安装方式

### 方式一：使用包管理器安装（推荐）

#### Debian/Ubuntu

```bash
# 更新软件源
sudo apt update

# 安装 Nginx
sudo apt install nginx

# 启动 Nginx
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# 安装 EPEL 源
sudo yum install epel-release

# 安装 Nginx
sudo yum install nginx

# 启动 Nginx
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# 使用 Homebrew 安装
brew install nginx

# 启动 Nginx
brew services start nginx
```

### 方式二：从源码编译安装

#### 安装编译依赖

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### 下载源码

```bash
# 下载 Nginx 最新稳定版
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# 解压
tar -zxf nginx-1.26.0.tar.gz

# 进入目录
cd nginx-1.26.0
```

#### 编译配置

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### 编译并安装

```bash
# 编译
make

# 安装
sudo make install
```

### 方式三：使用 Docker 安装

```bash
# 拉取官方镜像
docker pull nginx

# 运行 Nginx 容器
docker run -d -p 80:80 --name mynginx nginx

# 映射配置文件和日志
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### 方式四：Windows 安装

1. 从 https://nginx.org/en/download.html 下载 Windows 版本的 Nginx
2. 解压到指定目录
3. 双击 nginx.exe 启动
4. 或者使用命令行启动：

```cmd
cd C:\nginx
start nginx
```

## 安装后验证

### 检查安装

```bash
# 检查 Nginx 版本
nginx -v

# 查看详细版本信息
nginx -V
```

### 检查运行状态

```bash
# 检查进程是否运行
ps aux | grep nginx

# 使用 systemctl 检查状态
systemctl status nginx
```

### 测试访问

```bash
# 使用 curl 测试
curl http://localhost

# 或者打开浏览器访问
http://localhost
```

## 目录结构

### Linux 默认目录

| 路径 | 说明 |
|------|------|
| /etc/nginx/ | 配置目录 |
| /etc/nginx/nginx.conf | 主配置文件 |
| /etc/nginx/conf.d/ | 子配置文件目录 |
| /var/log/nginx/ | 日志目录 |
| /var/www/html/ | 默认网站根目录 |
| /usr/sbin/nginx | 可执行文件 |
| /usr/share/nginx/html/ | 默认静态文件目录 |

### 编译安装目录

| 路径 | 说明 |
|------|------|
| /usr/local/nginx/ | 安装根目录 |
| /usr/local/nginx/conf/ | 配置目录 |
| /usr/local/nginx/html/ | 默认网站根目录 |
| /usr/local/nginx/logs/ | 日志目录 |
| /usr/local/nginx/sbin/ | 可执行文件目录 |

## 卸载 Nginx

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### 编译安装

```bash
# 停止 Nginx
sudo nginx -s stop

# 删除安装目录
sudo rm -rf /usr/local/nginx
```

## 常见问题

### 端口冲突

如果启动失败，提示端口被占用：

```bash
# 查看端口占用
netstat -tulpn | grep :80

# 修改 Nginx 监听端口
# 编辑配置文件 /etc/nginx/nginx.conf
# 将 listen 80; 改为 listen 8080;
```

### 权限问题

如果遇到权限错误：

```bash
# 检查 Nginx 用户
ps aux | grep nginx

# 修改文件权限
sudo chown -R www-data:www-data /var/www/html/
```

### 配置文件语法检查

```bash
# 测试配置文件语法
nginx -t

# 指定配置文件测试
nginx -t -c /path/to/nginx.conf
```
