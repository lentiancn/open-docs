# OpenCode FAQ

Common questions and answers.

## Installation

### Q: Installation failed?

**Solutions**:
1. Check system requirements (needs modern terminal emulator)
2. Confirm network connection
3. Try install script: `curl -fsSL https://opencode.ai/install | bash`
4. Check error message and retry

### Q: Cannot install on Windows?

**Solutions**:
1. Try Chocolatey: `choco install opencode`
2. Or use Scoop: `scoop install opencode`
3. Or use NPM: `npm install -g opencode-ai`

### Q: Verify installation failed?

**Solutions**:
1. Confirm opencode is added to PATH
2. Restart terminal
3. Check if installation succeeded

## Usage

### Q: Cannot connect to LLM?

**Solutions**:
1. Run `/connect` command
2. Visit opencode.ai/auth to log in
3. Add billing info and get API key
4. Paste API key in terminal

### Q: Response is slow?

**Solutions**:
1. Check network connection
2. Try switching to another LLM provider
3. Reduce conversation context length

### Q: How to switch models?

Use `/connect` command to select different LLM provider.

### Q: Which models are supported?

Supports 75+ LLM providers:
- OpenAI (GPT-4, Claude, etc.)
- Anthropic (Claude)
- Google (Gemini)
- Local models
- See [Models.dev](https://models.dev)

## Features

### Q: What is AGENTS.md?

AGENTS.md is a file created when OpenCode initializes, containing project structure and coding conventions to help OpenCode understand the project.

### Q: Difference between Plan and Build mode?

- **Plan mode**: Only generates plans, doesn't modify code
- **Build mode**: Directly executes changes

Press **Tab** to switch modes.

### Q: How to share sessions?

Use `/share` command to create share link.

## Privacy

### Q: Does OpenCode save my code?

No. OpenCode doesn't store any code or context data, safe to use in sensitive environments.

### Q: How to protect privacy?

- Don't enter sensitive info in conversations
- Use local models
- Regularly clear sessions
