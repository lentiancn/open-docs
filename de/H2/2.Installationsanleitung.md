# Installationsanleitung für H2

## Umgebungsanforderungen

- Java 11 oder höher
- Jedes Java-unterstützte Betriebssystem (Windows, Linux, macOS usw.)

## Installationsmethoden

### Methode 1: ZIP-Paket herunterladen

1. Besuchen Sie die Download-Seite der offiziellen Website: https://www.h2database.com/html/download.html
2. Laden Sie die "Platform-Independent Zip"-Version herunter
3. Entpacken Sie die heruntergeladene ZIP-Datei in ein beliebiges Verzeichnis, z.B.: C:\h2 oder /opt/h2

### Methode 2: Maven-Abhängigkeit

Wenn Sie ein Java-Projekt mit Maven oder Gradle verwenden, können Sie die Abhängigkeit direkt hinzufügen:

**Maven:**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle:**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### Methode 3: Windows-Installer

1. Laden Sie die Version "Windows Installer" herunter
2. Führen Sie den Installer aus und folgen Sie dem Assistenten zur Fertigstellung der Installation

## Umgebungsvariablen konfigurieren (optional)

Wenn Sie das ZIP-Paket heruntergeladen haben und H2 direkt über die Befehlszeile verwenden möchten:

**Windows:**
1. Rechtsklick auf "Dieser PC" → "Eigenschaften"
2. Klicken Sie auf "Erweiterte Systemeinstellungen"
3. Klicken Sie auf "Umgebungsvariablen"
4. Suchen Sie unter Systemvariablen den Pfad und klicken Sie auf "Bearbeiten"
5. Fügen Sie den H2-bin-Ordnerpfad hinzu

**Linux/macOS:**
```bash
export PATH=$PATH:/opt/h2/bin
```

## Installation überprüfen

### Verwenden der Befehlszeile

Nach dem Entpacken führen Sie in der Befehlszeile aus:

```bash
java -jar h2.jar
```

Dadurch wird die H2-Webkonsole gestartet.

### JDBC-Verbindung verwenden

So verbinden Sie sich mit der H2-Datenbank im Java-Code:

```java
// Eingebetteter Modus
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// Dateidatenbank
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### H2-Konsole verwenden

1. Führen Sie den Befehl aus: `java -jar h2.jar`
2. Öffnen Sie einen Browser und rufen Sie auf: http://localhost:8082
3. Standard-Benutzername: sa, Passwort leer

## Verbindungs-URL-Formate

H2 unterstützt verschiedene Verbindungs-URL-Formate:

| Modus | URL-Beispiel | Beschreibung |
|-------|-------------|---------------|
| Speichermodus | jdbc:h2:mem:testdb | Daten im Speicher |
| Eingebettete Datei | jdbc:h2:file:./data/testdb | Daten in Datei |
| TCP-Server | jdbc:h2:tcp://localhost:9092/testdb | Client-Server-Modus |
| TCP-verschlüsselt | jdbc:h2:ssl://localhost:9092/testdb | SSL-verschlüsselte Verbindung |
| PostgreSQL-Kompatibilität | jdbc:h2:postgres://localhost:5432/testdb | PostgreSQL-Protokollkompatibilitätsmodus |

## Häufige Probleme

### Port belegt
H2 verwendet standardmäßig Port 9092 (TCP-Modus) und 8082 (Webkonsole). Wenn der Port belegt ist, können Sie einen anderen Port in der Verbindungs-URL angeben.

### Java-Versionsproblem
Stellen Sie sicher, dass Java 11 oder höher installiert ist. Sie können es mit folgendem Befehl überprüfen:

```bash
java -version
```

### Treiber nicht gefunden
Stellen Sie sicher, dass die H2-JAR-Datei im Classpath ist. Bei Maven-Projekten stellen Sie sicher, dass die Abhängigkeit korrekt hinzugefügt wurde.
