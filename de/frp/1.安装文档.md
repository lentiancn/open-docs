## Installationsmethoden

### Methode 1: Linux/macOS

#### 1. Herunterladen

```bash
# Neueste Version herunterladen
wget https://github.com/fatedier/frp/releases/download/v0.51.0/frp_0.51.0_linux_amd64.tar.gz

# Entpacken
tar -xzf frp_0.51.0_linux_amd64.tar.gz
cd frp_0.51.0_linux_amd64
```

#### 2. Verzeichnisstruktur

```
frp_0.51.0_linux_amd64/
├── frpc          # Client
├── frpc.ini      # Client-Konfiguration
├── frps          # Server
├── frps.ini      # Server-Konfiguration
└── systemd/      # systemd Service-Dateien
```

### Methode 2: Windows

#### 1. Herunterladen

1. Besuchen Sie https://github.com/fatedier/frp/releases
2. Windows-Version herunterladen
3. In gewünschtes Verzeichnis entpacken

#### 2. Ausführen

```powershell
# Server
.\frps.exe -c frps.ini

# Client
.\frpc.exe -c frpc.ini
```

### Methode 3: Docker

```bash
# Server
docker run --network host -d -v ./frps.ini:/etc/frp/frps.ini --name frps fatedier/frps:v0.51.0

# Client
docker run -d -v ./frpc.ini:/etc/frp/frpc.ini --name frpc fatedier/frpc:v0.51.0
```

### Methode 4: Quellcode-Kompilierung

```bash
# Repository klonen
git clone https://github.com/fatedier/frp.git

# Kompilieren
cd frp
make

# Ausgabe im bin/ Verzeichnis
```

## Server-Konfiguration

### Grundkonfiguration (frps.ini)

```ini
[common]
bind_port = 7000
dashboard_port = 7500
token = your_token_here
dashboard_user = admin
dashboard_pwd = admin

# Protokollierung
log_file = ./frps.log
log_level = info
```

### Erweiterte Konfiguration

```ini
[common]
# Ports
bind_port = 7000
vhost_http_port = 80
vhost_https_port = 443

# Authentifizierung
token = your_token_here

# Verbindungs-Pool
max_pool_count = 5
max_ports_per_client = 0

# Heartbeat
heartbeat_timeout = 90

# Erlaubte Client-Ports
allow_ports = 2000-3000,3001,3003,4000-50000
```

### systemd Service

```ini
# /etc/systemd/system/frps.service
[Unit]
Description=frp server
After=network.target

[Service]
Type=simple
User=root
ExecStart=/path/to/frps -c /path/to/frps.ini

[Install]
WantedBy=multi-user.target
```

```bash
# Service starten
sudo systemctl start frps
sudo systemctl enable frps
```

## Client-Konfiguration

### Grundkonfiguration (frpc.ini)

```ini
[common]
server_addr = your_server_ip
server_port = 7000
token = your_token_here

[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 6000
```

### HTTP-Dienst

```ini
[common]
server_addr = your_server_ip
server_port = 7000

[web]
type = http
local_port = 8080
custom_domains = yourdomain.com
```

### TCP-Port-Mapping

```ini
[common]
server_addr = your_server_ip
server_port = 7000

[mysql]
type = tcp
local_ip = 127.0.0.1
local_port = 3306
remote_port = 13306
```

### STCP (Sicheres TCP)

```ini
[common]
server_addr = your_server_ip
server_port = 7000

[secret_ssh]
type = stcp
sk = your_secret_key
local_ip = 127.0.0.1
local_port = 22
```

## Häufig gestellte Fragen

### Verbindungsfehler?

1. Prüfen Sie, ob der Server läuft
2. Firewall-Ports prüfen
3. Token-Übereinstimmung verifizieren

### Port belegt?

Ändern Sie remote_port oder verwenden Sie Port-Bereiche.

### Leistungsprobleme?

Passen Sie die Verbindungs-Pool-Größe und Timeout-Einstellungen an.
