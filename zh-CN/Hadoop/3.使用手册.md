# Hadoop 使用手册

## HDFS 操作指南

### 基本文件系统操作

HDFS 提供了类似于传统文件系统的命令行接口，用户可以通过 hdfs dfs 命令进行各种文件操作。这些命令与 Linux 的 shell 命令非常相似，熟悉 Linux 系统的用户可以快速上手。理解这些基本操作是使用 Hadoop 的基础，下面详细介绍常用的 HDFS 操作命令。

创建目录使用 mkdir 命令，可以创建单层或多层目录：

```bash
# 创建单层目录
hdfs dfs -mkdir /user/data

# 创建多层目录
hdfs dfs -mkdir -p /user/data/input

# 查看目录列表
hdfs dfs -ls /user/data

# 递归查看目录列表
hdfs dfs -ls -R /user/data
```

文件上传和下载是常用的操作：

```bash
# 上传本地文件到 HDFS
hdfs dfs -put localfile.txt /user/data/

# 上传并重命名
hdfs dfs -put localfile.txt /user/data/newname.txt

# 下载 HDFS 文件到本地
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# 复制 HDFS 内部文件
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# 移动 HDFS 内部文件
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

文件查看和删除：

```bash
# 查看文件内容
hdfs dfs -cat /user/data/file.txt

# 查看文件最后几行
hdfs dfs -tail /user/data/file.txt

# 查看文件大小
hdfs dfs -du /user/data/file.txt

# 删除文件
hdfs dfs -rm /user/data/file.txt

# 递归删除目录
hdfs dfs -rm -r /user/data/dir
```

### 权限管理

HDFS 的权限模型与 Unix/Linux 类似，基于用户、用户组和其他人的三层权限体系。每个文件和目录都有读（r）、写（w）、执行（x）三种权限，分别对应所有者、用户组和其他用户。理解权限管理对于保护数据安全至关重要。

```bash
# 修改文件所有者
hdfs dfs -chown user:group /user/data/file.txt

# 修改文件权限
hdfs dfs -chmod 755 /user/data/file.txt

# 修改所属组
hdfs dfs -chgrp groupname /user/data/file.txt
```

### 数据复制与均衡

HDFS 默认会将每个数据块复制三份，存储在不同的 DataNode 上。可以通过 dfs.replication 属性修改默认副本数：

```bash
# 设置特定文件的副本数
hdfs dfs -setrep -w 2 /user/data/file.txt

# 查看文件的块位置信息
hdfs fsck /user/data/file.txt -files -blocks -locations
```

当集群新增节点或节点下线时，数据可能分布不均，此时可以使用 rebalance 命令进行数据均衡：

```bash
start-balancer.sh -threshold 10
```

threshold 参数表示节点磁盘使用率与集群平均水平的最大差异百分比。均衡过程会占用一定的网络带宽，建议在业务低峰期执行。

### HDFS Web UI

HDFS 提供了功能丰富的 Web 界面，可以通过浏览器访问 NameNode 的 9870 端口。在 Web UI 上可以浏览文件系统目录、查看文件内容、监控 DataNode 状态、查看数据块信息等。Web UI 是运维人员日常监控和排查问题的重要工具。

Web UI 的主要功能包括：浏览 HDFS 目录树，查看文件列表和文件属性；查看 DataNode 列表和每个 DataNode 的存储使用情况；查看数据块的分布和复制状态；查看 NameNode 的运行时信息和日志；进行简单的文件操作如下载和删除。

## MapReduce 编程

### WordCount 示例

WordCount 是 MapReduce 编程的入门示例，统计文本中每个单词出现的次数。理解这个示例有助于掌握 MapReduce 编程模型。程序分为 Map 阶段和 Reduce 阶段，Map 阶段将文本拆分成单词，Reduce 阶段对相同单词的计数进行汇总。

Map 阶段代码示例：

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduce 阶段代码示例：

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### 提交作业

将 MapReduce 打包成 JAR 文件后，使用 hadoop jar 命令提交作业：

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

作业提交后，可以在 YARN 的 Web UI（端口 8088）上查看作业进度和结果。作业完成时，会输出作业的统计信息，包括作业ID、任务数量、运行时间等。

### 作业配置

可以通过 Configuration 对象配置作业的各种参数：

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

常用的配置参数包括：mapreduce.job.reduces 设置 Reduce 任务数量；mapreduce.map.memory.mb 和 mapreduce.reduce.memory.mb 设置容器内存；mapreduce.map.cpu.vcores 和 mapreduce.reduce.cpu.vcores 设置容器 CPU 核心数。

## YARN 操作

### 查看集群资源

YARN 提供了丰富的命令行工具来管理集群资源：

```bash
# 查看所有节点状态
yarn node -list -all

# 查看应用程序列表
yarn application -list

# 查看特定应用程序状态
yarn application -status application_1234567890123_0001

# 查看应用程序日志
yarn logs -applicationId application_1234567890123_0001
```

### 队列管理

YARN 使用队列来管理资源分配。可以通过 Capacity Scheduler 或 Fair Scheduler 配置队列：

```bash
# 查看队列信息
yarn queue -status default

# 查看队列容量
yarn resourcemanager -refreshQueues
```

### 资源调度配置

Capacity Scheduler 是默认的调度器，支持多租户资源共享。可以在 yarn-site.xml 中进行配置：

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</value>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## 性能优化

### MapReduce 调优

MapReduce 作业的性能调优涉及多个方面。首先是合理设置 Map 和 Reduce 任务数量。Map 任务数量通常由输入数据的分片数量决定，可以通过 mapreduce.input.fileinputformat.split.maxsize 参数调整。Reduce 任务数量建议设置为集群 Reduce 槽位总数的 0.9 到 1.2 倍。

```bash
# 设置 Map 任务数量
-D mapreduce.job.maps=10

# 设置 Reduce 任务数量
-D mapreduce.job.reduces=5
```

Shuffle 阶段的优化对整体性能影响很大。可以调整以下参数：

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFS 调优

HDFS 的性能优化主要关注数据读写效率。对于大文件，可以调整块大小来平衡数据本地性和故障恢复：

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

启用短路读取可以让客户端直接读取本地 DataNode 的数据，减少网络传输：

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARN 调优

YARN 的资源分配对集群利用率有重要影响。合理设置 NodeManager 的资源容量：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

设置虚拟 CPU 核心数以更好地利用多核处理器：

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## 数据导入导出

### 从数据库导入数据

使用 Sqoop 可以方便地在关系型数据库和 HDFS 之间迁移数据：

```bash
# 导入整个表
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# 导入并创建 Hive 表
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### 导出数据到数据库

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### 数据格式选择

不同的数据格式对性能有不同的影响。常用的格式包括：TextFile（通用性好，但性能一般）；SequenceFile（适合 MapReduce，性能好）；Parquet（列式存储，适合分析查询）；ORC（Hive 优化列式存储）。对于分析型工作负载，建议使用列式存储格式如 Parquet 或 ORC。
