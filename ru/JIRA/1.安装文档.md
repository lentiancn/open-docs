# Руководство по установке JIRA

JIRA — это инструмент для отслеживания проблем и управления проектами, разработанный компанией Atlassian. В этом руководстве описаны методы установки в различных средах.

## Информация о версиях

| Версия | Статус | Описание |
|--------|--------|----------|
| JIRA 9.x | ✅ Текущая | Последняя версия |
| JIRA 8.x | ✅ Поддерживаемая | Стабильная версия |

## Системные требования

### Аппаратные требования

| Требование | Минимум | Рекомендуется |
|------------|---------|---------------|
| CPU | 4 ядра | 8+ ядер |
| Оперативная память | 4 ГБ | 16+ ГБ |
| Диск | 10 ГБ | 50+ ГБ |

### Программные требования

| Программное обеспечение | Версия |
|------------------------|--------|
| Java | JDK 11 или 17 |
| Операционная система | Linux/Windows/macOS |
| База данных | PostgreSQL/MySQL/Oracle |

## Способы установки

### Способ 1: Установка в Windows

#### 1. Загрузка установщика

1. Посетите https://www.atlassian.com/software/jira/download
2. Загрузите JIRA Software
3. Запустите установщик

#### 2. Запуск установки

Для завершения установки следуйте мастеру установки:
- Выберите тип установки
- Настройте порт
- Настройте базу данных
- Создайте учётную запись администратора

### Способ 2: Установка в Linux

#### 1. Загрузка установочного пакета

```bash
# Загрузить JIRA
wget https://product-downloads.atlassian.com/software/jira/downloads/atlassian-jira-software-X.X.X-x64.bin
```

#### 2. Установка

```bash
# Добавить права на выполнение
chmod +x atlassian-jira-software-X.X.X-x64.bin

# Запустить установку
sudo ./atlassian-jira-software-X.X.X-x64.bin
```

#### 3. Настройка

```bash
# Запустить JIRA
sudo /opt/atlassian/jira/bin/start-jira.sh

# Остановить JIRA
sudo /opt/atlassian/jira/bin/stop-jira.sh
```

### Способ 3: Установка через Docker

```bash
# Загрузить образ
docker pull atlassian/jira-software

# Запустить контейнер
docker run -v jira-data:/var/atlassian/application-data/jira --name="jira" -d -p 8080:8080 atlassian/jira-software
```

### Способ 4: Использование AWS

1. Найдите JIRA в AWS Marketplace
2. Выберите тип экземпляра
3. Настройте базу данных
4. Запустите экземпляр

## Настройка

### Настройка базы данных

```properties
# dbconfig.xml
<?xml version="1.0" encoding="UTF-8"?>
<jira-database-config>
  <name>defaultDS</name>
  <connection-type>default</connection-type>
  <url>jdbc:postgresql://localhost:5432/jira</url>
  <username>jira</username>
  <password>password</password>
  <pool-size>20</pool-size>
</jira-database-config>
```

### Настройка обратного прокси

#### Nginx

```nginx
server {
    listen 80;
    server_name jira.example.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Часто задаваемые вопросы

### Ошибка запуска?

Проверьте версию Java:

```bash
java -version
```

### Ошибка подключения к базе данных?

Проверьте настройки базы данных и информацию о подключении.

### Недостаточно памяти?

Измените настройки памяти:

```bash
# setenv.sh
export JVM_MINIMUM_MEMORY="2048m"
export JVM_MAXIMUM_MEMORY="4096m"
```
