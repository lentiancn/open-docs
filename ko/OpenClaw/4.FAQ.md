# OpenClaw FAQ

> OpenClaw에 대한 자주 묻는 질문

---

## 일반

### OpenClaw란?

OpenClaw는 고급 에이전트 기술을 활용하여 작업을 자동화하는 AI 어시스턴트 프레임워크입니다. 파일 관리, 브라우저 제어 등을 위한 다양한 도구를 결합합니다.

### OpenClaw는 누구를 위한 것인가?

OpenClaw는 개발자, 파워 유저, AI 기반 자동화를 통해 생산성을 높이고 싶은 모든 사람에게 적합합니다.

## 설치

### 시스템 요구사항은 무엇인가?

| 요구사항 | 최소 |
|---------|-----|
| 운영 체제 | Windows 10+, macOS, Linux |
| Node.js | 버전 18+ |
| RAM | 4GB |
| 저장공간 | 500MB |

### 설치가 실패하나요?

**해결책**:
1. Node.js 버전 확인: `node --version`
2. npm 캐시 지우기: `npm cache clean --force`
3. 관리자/루트 권한으로 재시도
4. 오류 메시지 기록

### Gateway가 시작되지 않음?

**해결책**:
1. 포트 확인: `netstat -ano | findstr 3000`
2. 다른 포트 사용: `openclaw gateway start --port 3001`
3. 로그 검토: `openclaw gateway logs`

## 사용법

### 대화를 시작하려면?

```bash
openclaw chat
```

### 단일 명령을 실행하려면?

```bash
openclaw exec "내 명령"
```

### OpenClaw를 헤드리스로 사용할 수 있나요?

네, OpenClaw는 헤드리스 모드로 작동할 수 있습니다. Gateway 서비스를 시작하고 API 또는 CLI를 사용합니다.

## 파일 관리

### 파일을 읽으려면?

read 도구 사용:
```
read /path/to/file.txt
```

### 새 파일을 생성하려면?

write 도구 사용:
```
write --content "content" --path /new/file.txt
```

### 파일을 편집하려면?

정확한 변경에는 edit 도구 사용:
```
edit --path /file.txt --oldText "old" --newText "new"
```

## 브라우저 제어

### 지원되는 브라우저는?

- Chrome (확장 프로그램経由)
- 격리된 OpenClaw 브라우저
- 브라우저 자동화를 통한 기타

### 브라우저를 제어하려면?

```bash
browser start --profile openclaw
browser navigate --url https://example.com
browser screenshot --path /output.png
```

### 브라우저가 시작되지 않음?

**해결책**:
1. Chrome 확장 프로그램 설치
2. 방화벽 설정 확인
3. 브라우저 캐시 지우기

## 하위 에이전트

### 하위 에이전트란?

하위 에이전트는 특정 작업을 병렬 또는 순차적으로 실행할 수 있는 독립적인 인스턴스입니다.

### 하위 에이전트를 생성하려면?

```bash
subagents create --task "description"
```

### 하위 에이전트를 종료하려면?

```bash
subagents kill --id agent-id
```

## 보안

### OpenClaw가 안전한가요?

OpenClaw는 여러 보안 레이어를 구현합니다:
- 파일 권한
- 외부 작업에 대한 확인
- 샌드박스 옵션

### 민감한 데이터를 보호하려면?

1. 암호화된 구성 사용
2. 대화에서 비밀번호 공유 금지
3. 비밀에는 환경 변수 사용

## 문제 해결

### "명령을 찾을 수 없음"?

확인:
1. OpenClaw가 올바르게 설치됨
2. PATH 변수가 올바르게 설정됨
3. Gateway가 실행 중

### 파일 작업이 실패?

1. 경로 확인
2. 권한 검증
3 special 문자가 있는지 파일 이름 확인

### 응답 시간이 느림?

1. Gateway 상태 확인
2. 네트워크 연결 검증
3. 로그 분석
