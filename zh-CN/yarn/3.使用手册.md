# Yarn 使用手册

## 基本命令

### 通用命令

- `yarn add` - 向项目添加依赖
- `yarn install` - 安装项目依赖
- `yarn remove` - 从项目中移除依赖
- `yarn run` - 运行 package.json 中定义的脚本
- `yarn init` - 创建新包
- `yarn info` - 查看包相关信息
- `yarn why` - 显示为什么需要某个包

### 高级命令

- `yarn dlx` - 在临时环境中运行包
- `yarn dedupe` - 去重具有重叠范围的依赖
- `yarn up` - 跨项目升级依赖
- `yarn pack` - 从活动工作区生成 tarball
- `yarn rebuild` - 重新构建项目的原生包

### NPM 相关命令

- `yarn npm audit` - 对已安装的包执行漏洞审计
- `yarn npm publish` - 将活动工作区发布到 npm 注册表
- `yarn npm login` - 存储新的登录信息以访问 npm 注册表
- `yarn npm logout` - 从 npm 注册表登出
- `yarn npm whoami` - 显示已认证用户的名称

### 插件管理

- `yarn plugin list` - 列出可用的官方插件
- `yarn plugin import` - 下载插件
- `yarn plugin remove` - 移除插件
- `yarn plugin runtime` - 列出活动插件

## 工作区 (Workspaces)

Yarn 是第一个专门为工作区构建的包管理器，允许您将项目拆分为子组件。

### 工作区配置

在 `package.json` 中配置工作区：

```json
{
  "workspaces": [
    "packages/*"
  ]
}
```

### 工作区命令

- `yarn workspaces list` - 列出所有工作区
- `yarn workspace <name> <command>` - 在特定工作区中运行命令

## 配置文件

### .yarnrc.yml

Yarn v2+ 使用 `.yarnrc.yml` 文件进行配置：

```yaml
nodeLinker: pnp
yarnPath: .yarn/releases/yarn-4.0.0.cjs
```

### .yarnrc (Yarn v1)

Yarn v1 使用 `.yarnrc` 文件：

```
--registry https://registry.npmjs.org
--strict-semver true
```

## 版本差异说明

### Yarn v4+ vs Yarn v1

| 功能 | Yarn v4+ | Yarn v1 |
|------|----------|---------|
| 架构 | 插件化 (Berry) | 传统 |
| 安装速度 | 更快 (并行安装) | 标准速度 |
| 磁盘空间 | 更节省 (PnP) | 标准 node_modules |
| 工作区 | 原生支持 | 需要额外配置 |
| 插件系统 | 内置支持 | 不支持 |
| 配置文件 | .yarnrc.yml | .yarnrc |

### 迁移注意事项

- Yarn v4+ 默认使用 PnP，可能与某些工具不兼容
- 需要更新 CI/CD 配置以适应新的安装流程
- 插件系统提供了更多自定义选项
- 工作区功能更加强大和灵活
