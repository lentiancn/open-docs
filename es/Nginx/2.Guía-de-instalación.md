# Guía de instalación de Nginx

## Requisitos del entorno

Nginx es compatible con múltiples sistemas operativos:

- Linux (Debian, Ubuntu, CentOS, RHEL, Alpine, etc.)
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## Métodos de instalación

### Método 1: Usando el gestor de paquetes (Recomendado)

#### Ubuntu/Debian

```bash
# Actualizar índice de paquetes
sudo apt update

# Instalar Nginx
sudo apt install nginx

# Iniciar Nginx
sudo systemctl start nginx

# Habilitar al inicio
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# Instalar repositorio EPEL
sudo yum install epel-release

# Instalar Nginx
sudo yum install nginx

# Iniciar Nginx
sudo systemctl start nginx

# Habilitar al inicio
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# Instalar usando Homebrew
brew install nginx

# Iniciar Nginx
brew services start nginx
```

### Método 2: Instalar desde código fuente

#### Instalar dependencias de compilación

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### Descargar código fuente

```bash
# Descargar última versión estable
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# Extraer
tar -zxf nginx-1.26.0.tar.gz

# Entrar al directorio
cd nginx-1.26.0
```

#### Configurar compilación

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### Compilar e instalar

```bash
# Compilar
make

# Instalar
sudo make install
```

### Método 3: Usando Docker

```bash
# Descargar imagen oficial
docker pull nginx

# Ejecutar contenedor Nginx
docker run -d -p 80:80 --name mynginx nginx

# Montar configuración y logs
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### Método 4: Instalación en Windows

1. Descargar versión para Windows desde https://nginx.org/en/download.html
2. Extraer al directorio deseado
3. Hacer doble clic en nginx.exe para iniciar
4. O iniciar desde línea de comandos:

```cmd
cd C:\nginx
start nginx
```

## Verificación post-instalación

### Verificar instalación

```bash
# Ver versión de Nginx
nginx -v

# Ver información detallada de versión
nginx -V
```

### Verificar estado de ejecución

```bash
# Verificar si el proceso está ejecutándose
ps aux | grep nginx

# Verificar estado usando systemctl
systemctl status nginx
```

### Probar acceso

```bash
# Usar curl para probar
curl http://localhost

# O abrir navegador
http://localhost
```

## Estructura de directorios

### Directorios predeterminados en Linux

| Ruta | Descripción |
|------|-------------|
| /etc/nginx/ | Directorio de configuración |
| /etc/nginx/nginx.conf | Archivo de configuración principal |
| /etc/nginx/conf.d/ | Directorio de subconfiguración |
| /var/log/nginx/ | Directorio de logs |
| /var/www/html/ | Raíz de documentos predeterminada |
| /usr/sbin/nginx | Ejecutable |
| /usr/share/nginx/html/ | Directorio de archivos estáticos predeterminados |

### Directorios de instalación compilada

| Ruta | Descripción |
|------|-------------|
| /usr/local/nginx/ | Raíz de instalación |
| /usr/local/nginx/conf/ | Directorio de configuración |
| /usr/local/nginx/html/ | Raíz de documentos predeterminada |
| /usr/local/nginx/logs/ | Directorio de logs |
| /usr/local/nginx/sbin/ | Directorio de ejecutables |

## Desinstalar Nginx

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### Instalación compilada

```bash
# Detener Nginx
sudo nginx -s stop

# Eliminar directorio de instalación
sudo rm -rf /usr/local/nginx
```

## Problemas comunes

### Conflicto de puertos

Si el inicio falla debido a conflicto de puertos:

```bash
# Ver uso de puertos
netstat -tulpn | grep :80

# Modificar puerto de escucha de Nginx
# Editar /etc/nginx/nginx.conf
# Cambiar listen 80; a listen 8080;
```

### Problemas de permisos

Si encuentra errores de permisos:

```bash
# Ver usuario de Nginx
ps aux | nginx

# Corregir permisos de archivos
sudo chown -R www-data:www-data /var/www/html/
```

### Verificar sintaxis de configuración

```bash
# Probar sintaxis de configuración
nginx -t

# Probar con archivo de configuración específico
nginx -t -c /path/to/nginx.conf
```
