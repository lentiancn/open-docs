## System Requirements

### Hardware Requirements
- **Processor**: Dual-core processor or better
- **RAM**: Minimum 4GB (8GB recommended)
- **Disk Space**: At least 3GB free space

### Software Requirements

#### For Windows
- Windows 10 or later
- Node.js 18.19 or later (LTS version recommended)
- npm 9.x or later (comes with Node.js)
- Git for version control

#### For macOS
- macOS 10.15 (Catalina) or later
- Node.js 18.19 or later (LTS version recommended)
- npm 9.x or later
- Git (comes with Xcode Command Line Tools)

#### For Linux
- Ubuntu 20.04 LTS or later / Debian-based distributions
- Fedora 36 or later / RHEL-based distributions
- Node.js 18.19 or later (LTS version recommended)
- npm 9.x or later
- Git

## Prerequisites

### 1. Install Node.js

Node.js is the runtime environment required to run Angular.

#### Windows Installation

**Method A: Using Node.js Installer**
1. Visit https://nodejs.org/
2. Download the **LTS** version (recommended for most users)
3. Run the installer (.msi file)
4. Follow the installation wizard:
   - Accept the license agreement
   - Choose installation location (default is fine)
   - Select features to install (default is fine)
5. Verify installation by opening Command Prompt and typing:
   ```
   node --version
   npm --version
   ```

**Method B: Using Chocolatey**
```powershell
# Install Chocolatey first (run as Administrator)
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# Install Node.js
choco install nodejs-lts -y
```

**Method C: Using nvm-windows (Recommended for developers)**
```powershell
# Install nvm-windows
# Download from https://github.com/coreybutler/nvm-windows/releases

# Install and use Node.js LTS
nvm install 18.19.0
nvm use 18.19.0
```

#### macOS Installation

**Method A: Using Homebrew (Recommended)**
```bash
# Install Homebrew if not installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js LTS
brew install node@18

# Add to PATH (if needed)
echo 'export PATH="/usr/local/opt/node@18/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Method B: Using nvm**
```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js LTS
nvm install 18.19.0
nvm use 18.19.0
```

**Method C: Using Node.js Installer**
1. Visit https://nodejs.org/
2. Download the macOS Installer (.pkg)
3. Follow the installation wizard

#### Linux Installation

**Ubuntu/Debian**
```bash
# Update package list
sudo apt update

# Install curl and build-essential
sudo apt install curl build-essential

# Add Node.js repository
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -

# Install Node.js
sudo apt install nodejs

# Verify installation
node --version
npm --version
```

**Fedora/RHEL**
```bash
# Add Node.js repository
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -

# Install Node.js
sudo dnf install nodejs

# Verify installation
node --version
npm --version
```

**Using nvm (Recommended)**
```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Add to .bashrc or .zshrc
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# Install Node.js LTS
nvm install 18.19.0
nvm use 18.19.0
```

### 2. Install Git

Git is required for Angular CLI's version control features.

#### Windows
- Download from https://git-scm.com/
- During installation, select "Use Git from the Windows Command Prompt"
- Recommended: Use Git Bash that comes with the installer

#### macOS
```bash
# Using Homebrew
brew install git

# Or install Xcode Command Line Tools
xcode-select --install
```

#### Linux
```bash
# Ubuntu/Debian
sudo apt install git

# Fedora
sudo dnf install git
```

## Installation on Windows

### Step-by-Step Guide

1. **Install Node.js**
   - Download Node.js LTS from https://nodejs.org/
   - Run the installer
   - Restart your terminal

2. **Install Git** (if not already installed)
   - Download from https://git-scm.com/
   - Use default settings

3. **Install Angular CLI**
   ```bash
   npm install -g @angular/cli
   ```

4. **Verify Installation**
   ```bash
   ng version
   ```

5. **Create Your First Project**
   ```bash
   ng new my-first-app
   cd my-first-app
   ng serve
   ```

### Using Visual Studio Code (Recommended)

1. Download VS Code from https://code.visualstudio.com/
2. Install Angular extensions:
   - Angular Language Service
   - Angular Snippets
   - TypeScript Hero

## Installation on Linux

### Ubuntu/Debian

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install prerequisites
sudo apt install curl build-essential git

# Add Node.js repository
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -

# Install Node.js
sudo apt install nodejs

# Install Angular CLI
sudo npm install -g @angular/cli

# Verify
ng version
```

### Fedora

```bash
# Install prerequisites
sudo dnf install curl git

# Add Node.js repository
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -

# Install Node.js
sudo dnf install nodejs

# Install Angular CLI
sudo npm install -g @angular/cli

# Verify
ng version
```

### Arch Linux

```bash
# Install Node.js
sudo pacman -S nodejs npm

# Install Angular CLI
npm install -g @angular/cli

# Verify
ng version
```

## Creating Your First Angular Project

### 1. Create New Project
```bash
ng new my-angular-app
```

Options:
- Would you like to add Angular routing? (Yes/No)
- Which stylesheet format would you like to use? (CSS/SCSS/SSAS/Less)

### 2. Navigate to Project
```bash
cd my-angular-app
```

### 3. Start Development Server
```bash
ng serve
```

### 4. Open in Browser
Navigate to: http://localhost:4200

### 5. Generate Components
```bash
ng generate component my-component
# or
ng g c my-component
```

## Troubleshooting

### Common Issues and Solutions

#### 1. npm permission errors (EACCES)

**Solution:**
```bash
# Option A: Use nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Option B: Fix npm permissions
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

#### 2. Node.js version mismatch

**Solution:**
```bash
# Check required version in Angular documentation
# Use nvm to switch versions
nvm install 18.19.0
nvm use 18.19.0
```

#### 3. Angular CLI not found

**Solution:**
```bash
# Reinstall Angular CLI
npm uninstall -g @angular/cli
npm install -g @angular/cli

# Add npm global bin to PATH
export PATH="$PATH:$(npm config get prefix)/bin"
```

#### 4. Port 4200 already in use

**Solution:**
```bash
# Use different port
ng serve --port 4201

# Or kill process on port 4200
# Windows
netstat -ano | findstr :4200
taskkill /PID <PID> /F

# macOS/Linux
lsof -i :4200
kill -9 <PID>
```

#### 5. Angular version compatibility

| Angular Version | Node.js Version | TypeScript Version |
|-----------------|-----------------|-------------------|
| Angular 17      | 18.x, 20.x      | ~5.2.0            |
| Angular 16      | 16.x, 18.x      | ~5.0.0            |
| Angular 15      | 16.x, 18.x      | ~4.9.0            |
| Angular 14      | 14.x, 16.x      | ~4.7.0            |

#### 6. SSL certificate errors

**Solution:**
```bash
# Disable strict SSL (not recommended for production)
npm config set strict-ssl false

# Or update ca certificates
# macOS
brew install ca-certificates

# Ubuntu
sudo apt-get install ca-certificates
```

#### 7. Clear npm cache

```bash
npm cache clean --force
npm cache verify
```

#### 8. Update Angular CLI

```bash
# Check current version
ng version

# Update to latest
npm install -g @angular/cli@latest

# Or update within project
ng update @angular/cli
```
