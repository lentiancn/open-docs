# OpenClaw Benutzerhandbuch

Dieses Handbuch bietet eine detaillierte Anleitung zu OpenClaws Funktionen und Nutzung.

---

## Grundkonzepte

### Gateway

Das Gateway ist die Kernkomponente von OpenClaw. Es läuft auf Ihrem lokalen Rechner als Kontrollplane für alle Kanalverbindungen, KI-Agenten und Clients.

### Kanäle

Kanäle sind die Chat-Plattformen, die Sie zur Interaktion mit dem KI-Assistenten verwenden: WhatsApp, Telegram, Discord, iMessage und mehr.

### Agenten

Agenten sind Instanzen des KI-Assistenten. Jeder Agent hat seinen eigenen Workspace, Sessions und Konfiguration.

### Sessions

Sessions verwalten den Gesprächskontext. OpenClaw unterstützt mehrere Session-Modi:
- **main**: Geteilte Session
- **per-peer**: Separate Session für jeden Kontakt
- **per-channel-peer**: Separate Session für jede Kanal-Kontakt-Kombination
- **per-account-channel-peer**: Getrennte Session für jede Konto-Kanal-Kombination

## Tägliche Nutzung

### Gateway starten und stoppen

**Starten:**

```bash
# Im Vordergrund (zum Debuggen)
openclaw gateway --port 18789

# Im Hintergrund
openclaw gateway start
```

**Stoppen:**

```bash
openclaw gateway stop
```

**Status prüfen:**

```bash
openclaw gateway status
```

### Kontrolloberfläche verwenden

Besuchen Sie http://127.0.0.1:18789/ in Ihrem Browser.

### CLI-Befehle

**Nachricht senden:**

```bash
openclaw message send --target +15555550123 --message "Hallo, OpenClaw"
```

**Mit KI chatten:**

```bash
openclaw agent --message "Schreibe ein Hello World Programm" --thinking high
```

## Kanalverwaltung

### WhatsApp konfigurieren

```bash
openclaw channels login
```

### Telegram konfigurieren

```json5
{
  channels: {
    telegram: {
      botToken: "IhrBotToken",
    },
  },
}
```

### Discord konfigurieren

```json5
{
  channels: {
    discord: {
      token: "IhrBotToken",
    },
  },
}
```

## Erweiterte Funktionen

### Multi-Agenten-Routing

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" },
    ],
  },
}
```

### Cron-Jobs

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2,
  },
}
```

### Sandbox-Modus

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent",
      },
    },
  },
}
```

## Häufige Befehle

| Befehl | Beschreibung |
|--------|--------------|
| `openclaw --version` | Version anzeigen |
| `openclaw status` | Status anzeigen |
| `openclaw gateway start` | Gateway starten |
| `openclaw gateway stop` | Gateway stoppen |
| `openclaw dashboard` | Kontrolloberfläche öffnen |
| `openclaw channels login` | Bei Kanälen anmelden |
| `openclaw config get` | Konfiguration abrufen |
| `openclaw config set` | Konfiguration setzen |
| `openclaw doctor` | Probleme diagnostizieren |

---

## Chat-Befehle

- `/status` - Session-Status anzeigen
- `/new` oder `/reset` - Session zurücksetzen
- `/compact` - Session-Kontext komprimieren
- `/think` - Denklevel setzen
- `/verbose on|off` - Ausführliche Ausgabe umschalten

## Mobile Nodes

OpenClaw unterstützt das Pairen von iOS- und Android-Geräten als Nodes.

### iOS Node

1. OpenClaw aus dem App Store herunterladen
2. Mit Gateway pairen
3. Berechtigungen erteilen (Kamera, Mikrofon, Standort)

### Android Node

Ähnlich wie iOS.

## Backup und Wiederherstellung

### Konfiguration sichern

```bash
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup
```

### Konfiguration wiederherstellen

```bash
cp ~/.openclaw/openclaw.json.backup ~/.openclaw/openclaw.json
```
