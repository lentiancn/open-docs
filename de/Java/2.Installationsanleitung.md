# Java Installationsanleitung

## Java Development Kit herunterladen

Oracle bietet kostenlose Downloads des Java Development Kit (JDK). Die aktuellen Haupt-LTS-Versionen umfassen JDK 25, JDK 21, JDK 17, JDK 11 und JDK 8.

Offizielle Download-Seite: https://www.oracle.com/java/technologies/downloads/

## Installation unter Windows

### Verwendung des Installers

1. Besuchen Sie die offizielle Oracle-Download-Seite und wählen Sie die Windows-Version des JDK
2. Laden Sie den x64 Installer (exe-Datei) oder MSI Installer herunter
3. Führen Sie den heruntergeladenen Installer aus
4. Folgen Sie den Anweisungen des Installationsassistenten, um die Installation abzuschließen. Standardpfad ist C:\Program Files\Java\jdk-version
5. Überprüfen Sie nach der Installation, ob sie erfolgreich war

### Verwendung des komprimierten Archivs

1. Laden Sie das x64 Compressed Archive (zip-Datei) herunter
2. Extrahieren Sie es in ein angegebenes Verzeichnis, z.B. C:\Java\jdk-25
3. Konfigurieren Sie Umgebungsvariablen

### Umgebungsvariablen konfigurieren

1. Klicken Sie mit der rechten Maustaste auf "Dieser PC" und wählen Sie "Eigenschaften"
2. Klicken Sie auf "Erweiterte Systemeinstellungen"
3. Klicken Sie auf "Umgebungsvariablen"
4. Klicken Sie unter Systemvariablen auf "Neu"
   - Variablenname: JAVA_HOME
   - Variablenwert: JDK-Installationspfad (z.B. C:\Program Files\Java\jdk-25)
5. Suchen Sie die Path-Variable und klicken Sie auf "Bearbeiten"
6. Fügen Sie eine neue Variable hinzu: %JAVA_HOME%\bin
7. Klicken Sie auf OK zum Speichern

### Installation überprüfen

Öffnen Sie die Eingabeaufforderung und geben Sie ein:

```cmd
java -version
javac -version
```

Wenn Versionsinformationen angezeigt werden, war die Installation erfolgreich.

## Installation unter Linux

### Installation aus komprimiertem Archiv

1. Laden Sie die Linux-Version des JDK (tar.gz-Format) herunter
2. Öffnen Sie das Terminal und extrahieren Sie es in das angegebene Verzeichnis:

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. Konfigurieren Sie Umgebungsvariablen durch Bearbeiten von ~/.bashrc oder /etc/profile:

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. Wenden Sie die Konfiguration an:

```bash
source ~/.bashrc
```

5. Überprüfen Sie die Installation:

```bash
java -version
```

### Installation mit Paketmanager

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## Installation unter macOS

### Verwendung des DMG-Installers

1. Laden Sie die macOS-Version des JDK (dmg-Datei) herunter
2. Öffnen Sie die heruntergeladene dmg-Datei
3. Ziehen Sie JDK in den Applications-Ordner
4. Öffnen Sie das Terminal und konfigurieren Sie Umgebungsvariablen:

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. Fügen Sie das Obige zu ~/.zshrc oder ~/.bash_profile hinzu
6. Überprüfen Sie die Installation:

```bash
java -version
```

### Verwendung von Homebrew

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## Verwaltung mehrerer JDK-Versionen

### Verwendung von update-alternatives unter Linux

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Versionen unter Windows wechseln

Ändern Sie die Umgebungsvariable JAVA_HOME, um auf verschiedene JDK-Verzeichnisse zu verweisen.

### Verwendung von SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Java-Umgebung überprüfen

### Java-Version überprüfen

```bash
java -version
```

Beispielausgabe:

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### Compiler-Version überprüfen

```bash
javac -version
```

Beispielausgabe:

```
javac 25.0.1
```

### JAVA_HOME überprüfen

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## Häufige Installationsprobleme

### Problem 1: java-Befehl nicht gefunden

Lösung: Überprüfen Sie, ob die Path-Umgebungsvariable das JDK-bin-Verzeichnis enthält.

### Problem 2: Unzureichende Berechtigungen

Lösung: Verwenden Sie sudo für Höherstufung unter Linux/macOS oder überprüfen Sie Dateiberechtigungen.

### Problem 3: Port bereits belegt

Lösung: Java-Anwendungen verwenden standardmäßig Port 8080. Stellen Sie sicher, dass der Port nicht von anderen Programmen verwendet wird.

### Problem 4: Unterschied zwischen JDK und JRE

- JDK (Java Development Kit): Enthält Entwicklungstools und JRE
- JRE (Java Runtime Environment): Enthält nur Komponenten, die zum Ausführen von Java-Programmen erforderlich sind

JDK ist erforderlich für die Entwicklung von Java-Programmen. JRE reicht aus zum Ausführen kompilierter Java-Programme.

## Empfehlungen nach der Installation

1. Konfigurieren Sie MAVEN_HOME oder GRADLE_HOME, wenn Sie Maven oder Gradle verwenden
2. Konfigurieren Sie M2_HOME für den Maven-lokalen Repository-Pfad
3. Konfigurieren Sie HTTP-Proxy bei Bedarf in Unternehmensumgebungen
4. Stellen Sie die Zeitzone ein, um die korrekte Zeit für Java-Anwendungen sicherzustellen

## Java deinstallieren

### Windows Systemsteuerung

1. Öffnen Sie die Systemsteuerung
2. Wählen Sie "Programme und Funktionen"
3. Suchen Sie Java-bezogene Programme
4. Klicken Sie mit der rechten Maustaste und wählen Sie "Deinstallieren"

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
