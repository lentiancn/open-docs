# JDK Introducción

## ¿Qué es JDK?

JDK (Java Development Kit) es el kit de desarrollo de software para el lenguaje de programación Java y constituye el núcleo de todo el desarrollo en Java. Incluye el entorno de ejecución de Java (JRE), el compilador de Java (javac), la máquina virtual de Java (JVM) y extensas bibliotecas de clases y herramientas de Java. JDK es esencial para el desarrollo de aplicaciones Java, ya sea que estés desarrollando aplicaciones de escritorio, aplicaciones web, aplicaciones móviles o sistemas de nivel empresarial.

JDK es desarrollado y mantenido por Oracle Corporation (y su predecesora Sun Microsystems), sirviendo como la base de la plataforma Java. Las versiones LTS (soporte a largo plazo) más recientes son JDK 25 y JDK 21, con JDK 17 también siendo una versión LTS ampliamente utilizada. Oracle lanza nuevas versiones de JDK periódicamente, cada una trae mejoras de rendimiento, nuevas características y correcciones de vulnerabilidades de seguridad.

JDK es más que un simple entorno de ejecución; proporciona todos los componentes necesarios para el desarrollo completo de Java. El componente más fundamental es el compilador de Java (javac), que compila el código fuente Java en bytecode (archivos .class). Además, está la máquina virtual de Java (JVM), que ejecuta el bytecode, junto con varias herramientas como jar (herramienta de archivos), javadoc (generador de documentación) y javap (desensamblador). Estas herramientas juntas constituyen un entorno de desarrollo Java completo.

## Versiones principales de JDK

### JDK 25 (Nueva versión)

JDK 25 es la versión más reciente de JDK, trae muchas características y mejoras nuevas. Esta es una versión de soporte a corto plazo con un ciclo de soporte de 6 meses. Las características principales incluyen: optimizaciones de rendimiento mejoradas, recolectores de basura mejorados, nuevas características de lenguaje y soporte para el hardware más reciente. JDK 25 es adecuada para desarrolladores que desean usar las últimas características de Java, pero deben tener en cuenta su ciclo de soporte más corto.

### JDK 21 (LTS)

JDK 21 es la versión de soporte a largo plazo (LTS) más reciente y recibirá al menos 8 años de soporte. Esta es la última versión LTS con uso gratuito de Oracle JDK bajo los Términos y Condiciones sin tarifa de Oracle. JDK 21 introduce características importantes como Hilos Virtuales (Virtual Threads), Coincidencia de Patrones (Pattern Matching) y Registros (Records), que simplifican enormemente la programación en Java y mejoran la eficiencia del desarrollo.

Los Hilos Virtuales son una de las características más importantes en JDK 21, haciendo que escribir aplicaciones altamente concurrentes sea mucho más simple. En comparación con los hilos tradicionales, los hilos virtuales tienen costos de creación extremadamente bajos y pueden crear fácilmente millones de hilos virtuales para manejar tareas concurrentes. Esto es significativo para construir sistemas distribuidos a gran escala, arquitecturas de microservicios y aplicaciones de servidor de alto rendimiento.

### JDK 17 (LTS)

JDK 17 es una de las versiones de soporte a largo plazo más maduras disponibles actualmente y tiene una adopción generalizada. Es la última versión con uso gratuito de Oracle JDK (según los términos de licencia de Oracle), por lo que muchas empresas todavía eligen usar JDK 17. JDK 17 incluye muchas características de OpenJDK, como Clases Selladas (Sealed Classes), Bloques de Texto (Text Blocks) y expresiones switch.

El rendimiento de JDK 17 ha mejorado significativamente en comparación con JDK 11, especialmente en la recolección de basura y la compilación JIT. Para aplicaciones empresariales que requieren soporte estable a largo plazo, JDK 17 sigue siendo una excelente elección. Logra un buen equilibrio entre rendimiento, estabilidad y características.

### JDK 11 (LTS)

JDK 11 es otra versión importante de soporte a largo plazo, que marca la línea divisoria de las características pagadas de Java. A partir de JDK 11, algunas características avanzadas de Oracle JDK requieren una suscripción paga. JDK 11 introduce el recolector de basura ZGC (experimental), la nueva API de cliente HTTP y la inferencia de tipo de variable local (palabra clave var). Aunque ya no es la última versión LTS, JDK 11 todavía se usa ampliamente en muchos proyectos existentes.

## Componentes principales de JDK

### Compilador Java (javac)

El compilador Java (javac) es una de las herramientas más esenciales de JDK, responsable de compilar código fuente Java (archivos .java) en bytecode Java (archivos .class). El bytecode es una forma intermedia de código que no depende de plataformas hardware específicas y solo puede ser ejecutado por la Máquina Virtual Java. Este diseño da a los programas Java la característica de "Escribe una vez, ejecuta en cualquier lugar".

El compilador javac soporta numerosas opciones de compilación, permitiendo a los desarrolladores controlar varios comportamientos del proceso de compilación a través de parámetros de línea de comandos. Por ejemplo, puede usar la opción -d para especificar el directorio de salida de compilación, -classpath para especificar la ruta de búsqueda de clases, y -source y -target para especificar las versiones de Java del código fuente y el bytecode objetivo. Dominar estas opciones de compilación es crucial para construir proyectos grandes.

### Máquina Virtual Java (JVM)

La Máquina Virtual Java (JVM) es el entorno central para ejecutar programas Java. Es responsable de cargar bytecode e interpretarlo o compilarlo para ejecución nativa. La JVM es clave para la independencia de plataforma de Java, protegiendo las diferencias entre sistemas operativos y hardware subyacentes, permitiendo que el mismo programa Java se ejecute en diferentes plataformas.

Los componentes principales de la JVM incluyen: ClassLoader responsable de cargar archivos de clase, Execution Engine responsable de ejecutar bytecode, y Áreas de Datos en Tiempo de Ejecución que incluyen el área de métodos, heap, pila de máquinas virtuales, pila de métodos nativos y contador de programas. Entender cómo funciona la JVM es esencial para escribir aplicaciones Java de alto rendimiento.

La JVM también proporciona una potente función de Recolección de Basura (GC), gestionando automáticamente la asignación y recuperación de memoria. Diferentes recolectores de basura como Serial, Parallel, CMS, G1, ZGC y Shenandoah son adecuados para diferentes escenarios de aplicación. Los desarrolladores pueden elegir el recolector de basura apropiado según las características de la aplicación para lograr un rendimiento óptimo.

### Entorno de Ejecución Java (JRE)

El Entorno de Ejecución Java (JRE) es el entorno requerido para ejecutar programas Java. Incluye la JVM, las bibliotecas de clases centrales de Java y otros archivos de soporte. El JRE es un subconjunto de JDK; JDK incluye JRE más herramientas de desarrollo. Si solo necesita ejecutar programas Java sin desarrollo, instalar JRE es suficiente.

El núcleo del JRE son las bibliotecas de clases centrales de Java, que proporcionan la funcionalidad básica del lenguaje Java como java.lang, java.util, java.io, java.net. Estas bibliotecas de clases han sido desarrolladas y optimizadas durante muchos años y son muy maduras y estables, cubriendo todo, desde estructuras de datos básicas hasta programación de red.

### Conjunto de herramientas Java

JDK proporciona un rico conjunto de herramientas de línea de comandos para el desarrollo, depuración y diagnóstico de programas Java. Las herramientas principales incluyen:

jar: Se usa para crear y administrar archivos de almacenamiento Java (archivos .jar), que pueden empacar múltiples archivos de clase en un archivo conveniente para distribución y despliegue.

javadoc: Se usa para generar documentación de API en formato HTML a partir de comentarios en el código fuente Java, la herramienta estándar para crear documentación de código.

javap: Se usa para desensamblar bytecode Java, puede ver estructuras de archivos de clase y firmas de métodos, muy útil para entender el proceso de compilación de Java y la depuración.

jdb: Depurador de Java, proporciona funciones de depuración de interfaz de línea de comandos, puede establecer puntos de interrupción, ejecutar paso a paso, ver valores de variables, etc.

jps: Muestra información sobre todos los procesos Java en el sistema actual, incluyendo ID de proceso y nombre de clase principal.

jstat: Muestra estadísticas de JVM como información de recolección de basura, información de carga de clases, etc.

jstack: Imprime información de seguimiento de pila de hilos Java, usado para diagnosticar problemas de bloqueo de hilos.

jmap: Muestra información de memoria heap de Java, puede generar archivos de volcado de heap para análisis.

Estas herramientas son muy útiles en el desarrollo diario y el diagnóstico de problemas en producción, dominarlas puede mejorar enormemente la eficiencia del desarrollo y las capacidades de diagnóstico.

## Áreas de aplicación de JDK

### Desarrollo de aplicaciones empresariales

JDK es la base del desarrollo de aplicaciones empresariales. Various Java EE frameworks como Spring, Hibernate y Struts requieren ejecutarse en JDK. Estos frameworks proporcionan soluciones completas para aplicaciones empresariales, incluyendo inyección de dependencias, gestión de transacciones, persistencia de datos y servicios web.

Las aplicaciones empresariales grandes típicamente tienen requisitos extremadamente altos de estabilidad y rendimiento, por lo que elegir la versión de JDK apropiada es muy importante. Muchas empresas usan versiones LTS maduras como JDK 17 combinadas con servicios de soporte comercial para garantizar la operación estable a largo plazo del sistema.

### Desarrollo de Android

Aunque las aplicaciones de Android se escriben principalmente en Java o Kotlin, el desarrollo de aplicaciones de Android no puede proceder sin JDK. Las herramientas del SDK de Android necesitan JDK para compilar proyectos de Android y generar archivos APK que se pueden instalar en dispositivos Android. Android Studio también incluye JDK incorporado, y los desarrolladores necesitan especificar la ruta de JDK al configurar el entorno de desarrollo.

### Procesamiento de Big Data

En el campo de big data, Java y JDK desempeñan un papel central. Los principales frameworks de big data como Hadoop, Spark y Flink están todos desarrollados en Java y requieren ejecutarse en JDK. Estos frameworks típicamente necesitan procesar cantidades masivas de datos y tienen requisitos muy altos de rendimiento y estabilidad.

### Desarrollo de aplicaciones web

El desarrollo de aplicaciones web Java es una fortaleza tradicional de JDK. Tecnologías como Servlet, JSP, Filter y Listener forman la base del desarrollo web Java. Los frameworks web principales como Spring MVC, Struts y JSF todos se ejecutan en JDK.

### Computación en la nube y microservicios

Con la popularidad de la computación en la nube y la arquitectura de microservicios, JDK también juega un papel importante en esta área. Los frameworks de microservicios como Spring Cloud y Dubbo proporcionan soluciones completas para construir sistemas distribuidos. Todos estos frameworks requieren ejecutarse en JDK.

## JDK y otras implementaciones de Java

### OpenJDK

OpenJDK es la implementación de código abierto de Java SE, desarrollada y mantenida conjuntamente por Oracle y la comunidad. OpenJDK es la base de la mayoría de las otras distribuciones de JDK. Muchos proveedores como Eclipse (Eclipse Temurin), Amazon (Amazon Corretto) y Microsoft (Microsoft Build of OpenJDK) construyen sus propias distribuciones de JDK basadas en OpenJDK.

OpenJDK y Oracle JDK son casi idénticos a nivel de código, con las principales diferencias siendo los acuerdos de licencia y los servicios de soporte. Oracle JDK proporciona soporte comercial y características adicionales de nivel empresarial, mientras que OpenJDK es completamente de código abierto y gratuito. Para la mayoría de los usuarios, usar OpenJDK es perfectamente adecuado.

### Otras distribuciones de JDK

Además de Oracle JDK y OpenJDK, hay muchas otras distribuciones de JDK:

Eclipse Temurin (anteriormente AdoptOpenJDK): Distribución de OpenJDK mantenida por la Eclipse Foundation, que proporciona construcciones de alta calidad y soporte a largo plazo.

Amazon Corretto: Distribución de JDK gratuita y multiplataforma proporcionada por Amazon, con soporte a largo plazo y optimización de rendimiento.

Microsoft Build of OpenJDK: Construcción de OpenJDK proporcionada por Microsoft, optimizada para Windows y Azure.

Azul Zulu: Distribución de JDK proporcionada por Azul Systems, conocida por alto rendimiento y estabilidad.

GraalVM: Máquina virtual políglota proporcionada por Oracle, que soporta no solo Java sino también JavaScript, Python, Ruby y otros idiomas.

## Importancia de JDK

Como la base del desarrollo Java, la importancia de JDK es evidente. Sin importar qué frameworks o herramientas de Java se utilicen, el soporte de JDK es esencial. JDK no es solo un conjunto de herramientas para compilar y ejecutar programas Java; también representa el núcleo del ecosistema de Java.

Elegir una versión de JDK apropiada y configurarla correctamente es crucial para el éxito del proyecto. Los factores a considerar incluyen: estabilidad de la versión, frecuencia de actualizaciones de seguridad, nuevas características, optimización de rendimiento y soporte de la comunidad. Para aplicaciones de nivel empresarial, se recomienda usar versiones LTS para obtener soporte a largo plazo.

Con el desarrollo continuo del lenguaje Java, JDK continúa actualizándose. Las nuevas versiones de JDK traen mejor rendimiento, más características y mayor seguridad. Mantener JDK actualizado puede proporcionar mejor experiencia de desarrollo y eficiencia operativa.

## Diferencias entre versiones

Las diferentes versiones principales de JDK tienen diferencias significativas en instalación, uso y características:

**JDK 8**: La última versión de Oracle JDK completamente gratuita para uso comercial. Introdujo expresiones lambda, API de streams y nueva API de fecha/hora. Todavía ampliamente utilizado en sistemas heredados.

**JDK 11**: Marcó el comienzo de las características pagadas de Java. Introdujo inferencia de tipo de variable local (var), nuevo cliente HTTP y ZGC (experimental). La última versión que soporta sistemas de 32 bits.

**JDK 17**: La última versión de Oracle JDK completamente gratuita bajo NFTC. Introdujo clases selladas, bloques de texto y expresiones switch. Logra un excelente equilibrio entre rendimiento, estabilidad y características.

**JDK 21**: La versión LTS más reciente, con Hilos Virtuales, coincidencia de patrones para switch y registros como características estables. Recomendada para nuevos proyectos.

**JDK 25**: La versión más reciente con las últimas características de lenguaje y mejoras de rendimiento. Adecuada para desarrolladores que desean las últimas características pero con un ciclo de soporte más corto.
