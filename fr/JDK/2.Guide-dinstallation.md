# JDK Guide d'installation

## Installation de JDK sous Windows

### Configuration requise

Avant d'installer JDK, assurez-vous que votre ordinateur répond aux exigences minimales suivantes : Processeur, Intel Core i3 ou supérieur recommandé ; Mémoire, 4 Go de RAM minimum recommandés, 8 Go ou plus préférés ; Espace disque, au moins 2 Go d'espace libre pour l'installation de JDK, espace supplémentaire pour les projets de développement ; Système d'exploitation, Windows 10 ou Windows 11 64 bits.

Bien que Windows 7 et 8.1 puissent fonctionner, Oracle ne prend plus en charge officiellement ces systèmes d'exploitation. L'exécution sur des systèmes plus anciens peut causer des problèmes de compatibilité.

### Téléchargement de JDK

Tout d'abord, visitez le site officiel d'Oracle à https://www.oracle.com/java/technologies/downloads/ pour télécharger JDK. Sur la page de téléchargement, vous pouvez voir les différentes versions de JDK : JDK 25 est la dernière version, JDK 21 est la dernière version LTS, JDK 17 est également une version LTS mature. Choisissez la version appropriée selon les exigences de votre projet.

Pour les systèmes Windows, choisissez l'installeur x64 (.exe) ou l'installeur MSI x64 (.msi). L'.exe est l'assistant d'installation traditionnel, le .msi convient au déploiement par lots en environnement d'entreprise. Si vous avez seulement besoin d'exécuter des programmes Java sans installer, choisissez l'archive compressée x64 (.zip), qui peut être extraite et utilisée directement.

Faites attention aux termes de licence lors du téléchargement. Oracle JDK est gratuit sous certaines versions et utilisations, mais a des restrictions pour l'utilisation commerciale et la redistribution. Si vous avez besoin d'une utilisation entièrement gratuite, envisagez d'utiliser OpenJDK ou d'autres distributions basées sur OpenJDK comme Eclipse Temurin.

### Étapes d'installation

Trouvez le fichier d'installation JDK téléchargé (généralement jdk-xx_windows-x64_bin.exe) et double-cliquez pour exécuter. Dans l'assistant d'installation, cliquez sur « Suivant » pour continuer. Vous verrez d'abord l'écran de sélection du dossier cible, le chemin d'installation par défaut est C:\Program Files\Java\jdk-xx. Si vous devez modifier l'emplacement d'installation, cliquez sur « Modifier » pour choisir un autre répertoire. Il est recommandé d'utiliser le chemin par défaut pour éviter les problèmes de configuration ultérieurs.

Vient ensuite l'écran de sélection des fonctionnalités, par défaut toutes les fonctionnalités sont sélectionnées, y compris les outils de développement, le code source et le JRE public. Il est recommandé de conserver l'installation complète par défaut. Cliquez sur « Suivant » pour commencer l'installation. Le processus d'installation peut prendre quelques minutes, veuillez patienter.

Après l'installation, vous verrez un message. Cliquez sur « Fermer » pour terminer. À ce stade, JDK est installé dans le répertoire spécifié, mais vous devez configurer les variables d'environnement pour utiliser java, javac et d'autres commandes depuis la ligne de commande.

### Configuration des variables d'environnement

Le but de configurer les variables d'environnement est de permettre au système de trouver l'emplacement d'installation de JDK afin que vous puissiez utiliser java, javac et d'autres commandes depuis n'importe quel répertoire. Faites un clic droit sur « Ce PC » ou « Mon PC » et sélectionnez « Propriétés ». Dans le menu de gauche, cliquez sur « Paramètres système avancés ».

Dans la fenêtre Propriétés système ouverte, cliquez sur le bouton « Variables d'environnement ». Dans la zone « Variables système », cliquez sur « Nouvelle ». Entrez JAVA_HOME pour le nom de variable et le chemin d'installation de JDK (comme C:\Program Files\Java\jdk-21) pour la valeur de variable, puis cliquez sur OK.

Trouvez la variable Path dans les variables système et cliquez sur « Modifier ». Cliquez sur « Nouvelle » et entrez %JAVA_HOME%\bin. Assurez-vous que cette ligne est en haut pour que le JDK nouvellement installé soit prioritaire. Cliquez sur OK pour enregistrer tous les paramètres.

Ouvrez une nouvelle fenêtre de ligne de commande (cmd ou PowerShell) et entrez java -version et javac -version. Si vous voyez les informations de version, la configuration a réussi.

### Vérification de l'installation

Ouvrez l'invite de commandes (Win+R, entrez cmd), et exécutez les commandes suivantes pour vérifier que JDK est correctement installé :

Exécutez java -version, qui doit afficher les informations de version de l'environnement d'exécution Java. Exécutez javac -version, qui doit afficher les informations de version du compilateur Java. Exécutez echo %JAVA_HOME%, qui doit afficher le chemin JAVA_HOME défini précédemment.

Si une commande affiche une erreur, vérifiez si les variables d'environnement sont correctement configurées ou si JDK est correctement installé.

## Installation de JDK sur macOS

### Utilisation du package d'installation DMG

Les utilisateurs macOS peuvent installer JDK rapidement en utilisant le package d'installation DMG. Visitez la page de téléchargement officielle Oracle et téléchargez la version macOS de JDK. Pour Apple Silicon (puces M1/M2/M3), Choisissez ARM64 DMG Installer ; pour les Mac à puces Intel, Choisissez x64 DMG Installer.

Après le téléchargement, ouvrez le fichier DMG, vous verrez un fichier d'installation pkg. Double-cliquez pour exécuter l'assistant d'installation et suivez les invites pour terminer l'installation. L'installeur placera JDK dans le répertoire /Library/Java/JavaVirtualMachines/.

Après l'installation, ouvrez Terminal et tapez java -version pour vérifier. Si le système a plusieurs versions de JDK, vous pouvez utiliser /usr/libexec/java_home -V pour voir toutes les versions installées.

### Installation avec Homebrew

Si Homebrew est déjà installé, vous pouvez l'utiliser pour installer JDK, ce qui est plus pratique. Ouvrez Terminal et exécutez : brew update. Puis installez JDK 21 : brew install openjdk@21.

Après l'installation, vous devez ajouter JDK au PATH. Exécutez la commande suivante pour lier JDK au système : sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.

Ensuite, ajoutez des variables d'environnement dans ~/.zshrc ou ~/.bash_profile : export PATH="/usr/local/opt/openjdk@21/bin:$PATH". Enregistrez et exécutez source ~/.zshrc pour appliquer la configuration.

### Configuration de JAVA_HOME

Sur macOS, configurer la variable d'environnement JAVA_HOME n'est pas obligatoire car le système trouve automatiquement les JDK installés. Cependant, certains outils (comme Maven, Gradle) peuvent nécessiter une configuration manuelle. Dans Terminal, éditez le fichier ~/.zshrc et ajoutez : export JAVA_HOME=$(/usr/libexec/java_home). Enregistrez et exécutez source ~/.zshrc pour appliquer la configuration.

## Installation de JDK sur Linux

### Installation avec APT (Debian/Ubuntu)

Sur les systèmes Debian et Ubuntu, vous pouvez installer OpenJDK en utilisant le gestionnaire de packages APT. Tout d'abord, mettez à jour la liste des packages : sudo apt update. Puis installez OpenJDK 21 version de développement : sudo apt install openjdk-21-jdk. Si vous avez seulement besoin d'exécuter des programmes Java sans le compilateur, installez JRE : sudo apt install openjdk-21-jre.

Pendant l'installation, il vous sera demandé si vous souhaitez créer le répertoire JAVA_HOME. Sélectionnez « Oui » pour la configuration automatique. Après l'installation, utilisez java -version et javac -version pour vérifier l'installation.

Si vous devez configurer JAVA_HOME manuellement, éditez le fichier ~/.bashrc ou ~/.profile et ajoutez : export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64. Puis exécutez source ~/.bashrc pour appliquer la configuration.

### Installation avec YUM (Red Hat/CentOS)

Sur les systèmes utilisant YUM comme Red Hat, CentOS et Fedora, vous pouvez installer JDK directement. Exécutez la commande suivante pour installer OpenJDK 21 : sudo yum install java-21-openjdk java-21-openjdk-devel. Le package -devel inclut le compilateur et d'autres outils de développement.

Après l'installation, utilisez la commande alternatives pour définir la version Java par défaut : sudo alternatives --config java. Suivez les invites pour sélectionner la version Java à définir par défaut.

Éditez le fichier ~/.bashrc et ajoutez : export JAVA_HOME=/usr/lib/jvm/java-21-openjdk. Exécutez source ~/.bashrc pour appliquer la configuration.

### Installation manuelle de fichiers binaires

Pour les situations nécessitant l'installation d'une version spécifique de JDK, vous pouvez télécharger et extraire manuellement des fichiers binaires. Visitez la page de téléchargement Oracle ou OpenJDK et téléchargez le fichier .tar.gz de la version souhaitée.

Créez le répertoire d'installation de JDK : sudo mkdir -p /opt/jdk. Extrayez le fichier téléchargé : sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk. Entrez dans le répertoire extrait et notez le chemin complet.

Éditez le fichier ~/.bashrc et ajoutez : export JAVA_HOME=/opt/jdk/jdk-21 et export PATH=$JAVA_HOME/bin:$PATH. Exécutez source ~/.bashrc pour appliquer la configuration.

## Gestion de plusieurs versions de JDK

### Gestion multi-version Windows

Après avoir installé plusieurs versions de JDK sur Windows, vous devez changer la version par défaut via les variables d'environnement. La méthode la plus simple est de modifier l'ordre des répertoires bin de JDK dans la variable d'environnement PATH, en plaçant la version que vous souhaitez utiliser en premier.

Une autre méthode est d'utiliser la variable d'environnement JAVA_HOME, en la pointant vers différents répertoires d'installation de JDK. Cette méthode fonctionne pour la plupart des outils, mais certains outils peuvent utiliser la commande java dans PATH au lieu de JAVA_HOME.

Certains outils tiers peuvent aider à gérer plusieurs versions de JDK, comme la version Windows de SDKMAN! ou JetBrains Toolbox. Ces outils facilitent la commutation entre versions.

### Gestion multi-version macOS

Sur macOS, vous pouvez utiliser l'outil jenv pour gérer plusieurs versions de JDK. Tout d'abord, installez jenv : brew install jenv. Ajoutez ensuite l'initialisation de jenv au fichier de configuration du shell.

Ajoutez le JDK installé à jenv : jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home. Ensuite, vous pouvez utiliser jenv versions pour voir toutes les versions gérées et jenv global xx pour définir la version globale par défaut.

Une autre méthode est d'utiliser /usr/libexec/java_home -v pour spécifier une version particulière pour exécuter des programmes. Par exemple : /usr/libexec/java_home -v 21 --exec java -version.

### Gestion multi-version Linux

Sur Linux, utilisez l'outil update-alternatives pour gérer plusieurs versions de JDK. Créez d'abord des entrées alternatives pour chaque version de JDK : sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1.

Utilisez ensuite update-alternatives --config java pour sélectionner la version par défaut. jenv est également disponible sur Linux, avec des méthodes d'installation et de configuration similaires à macOS.

## Problèmes d'installation courants

### Commande java introuvable après l'installation

Si vous ne trouvez pas la commande java après avoir installé JDK, vérifiez d'abord si JDK est correctement installé. Vérifiez si la variable d'environnement JAVA_HOME est correctement définie et si PATH inclut %JAVA_HOME%\bin (Windows) ou $JAVA_HOME/bin (Linux/macOS).

Sur Windows, ouvrez une nouvelle fenêtre de ligne de commande car les modifications des variables d'environnement peuvent ne pas prendre effet immédiatement. Sur Linux/macOS, exécutez source ~/.bashrc ou rouvrez le terminal.

Si le problème persiste, vérifiez si le répertoire d'installation de JDK est correct et si le fichier bin/java existe dans ce répertoire.

### Problèmes de permissions

Lors de l'installation de JDK sur Linux et macOS, vous pouvez rencontrer des problèmes de permissions. L'utilisation du gestionnaire de packages nécessite généralement des privilèges administrateur (sudo). Lors de l'installation manuelle, assurez-vous que le répertoire d'installation a les permissions de lecture et d'écriture appropriées.

Si les utilisateurs réguliers ne peuvent pas utiliser la commande java, vérifiez les permissions du répertoire JAVA_HOME et du répertoire bin de JDK. Vous pouvez utiliser ls -la /usr/lib/jvm pour vérifier les permissions du répertoire.

### Configuration JAVA_HOME non fonctionnelle

Si définir JAVA_HOME ne fonctionne pas, vérifiez si la syntaxe de la variable d'environnement est correcte. Sous Windows, la valeur de variable ne doit pas contenir de guillemets. Sous Linux/macOS, assurez-vous d'utiliser la bonne commande export.

Vérifiez également si d'autres scripts ou fichiers de configuration écrasent le paramètre JAVA_HOME. Sous Linux, la configuration au niveau du système peut être dans /etc/environment ou /etc/profile.d/.

### Désinstallation des anciennes versions

Après la mise à niveau de JDK, vous devrez peut-être désinstaller les anciennes versions pour économiser de l'espace disque. Sous Windows, recherchez l'ancienne version JDK dans le Panneau de configuration sous « Programmes et fonctionnalités » et cliquez sur Désinstaller. Sur macOS, supprimez l'ancien dossier de version sous /Library/Java/JavairtualMachines/. Sur Linux, utilisez le gestionnaire de packages pour désinstaller : sudo apt remove openjdk-xx-jdk (APT) ou sudo yum remove java-xx-openjdk (YUM).
