# MongoDB

## Обзор

MongoDB — это документоориентированная база данных, созданная для того, чтобы помочь разработчикам быстрее создавать современные приложения.

## Основные характеристики

- **Хранение на основе документов**: хранение данных в формате BSON с гибкой структурой
- **Высокая производительность**: поддержка нескольких типов индексов для эффективных запросов
- **Высокая доступность**: Replica Sets обеспечивают автоматическое переключение при отказе
- **Горизонтальное масштабирование**: поддержка больших данных через шардинг
- **Богатые запросы**: поддержка точных запросов, диапазонных запросов, агрегационных конвейеров
- **Поддержка транзакций**: мультидокументные ACID транзакции
- **Безопасность**: полная аутентификация пользователей и контроль доступа на основе ролей

## Ключевые понятия

### Документ

Документ — это основная единица данных в MongoDB:

```json
{
  "_id": ObjectId("..."),
  "name": "Иван",
  "age": 30,
  "email": "ivan@example.ru"
}
```

### Коллекция

Коллекция — это группа документов.

### Replica Set

Replica Set — это группа экземпляров MongoDB, поддерживающих одни и те же данные.

### Шардинг

Шардинг распределяет данные на несколько серверов.

## Быстрый старт

### Подключение к базе данных

```bash
mongosh
```

### Вставка данных

```bash
db.users.insertOne({ name: "Иван", age: 30, city: "Москва" })
```

### Запрос данных

```bash
db.users.find()
db.users.find({ age: { $gte: 25 } })
```

### Обновление данных

```bash
db.users.updateOne({ name: "Иван" }, { $set: { age: 31 } })
```

### Удаление данных

```bash
db.users.deleteOne({ name: "Иван" })
```
