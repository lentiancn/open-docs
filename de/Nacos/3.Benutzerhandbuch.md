# Nacos Benutzerhandbuch

## Service-Discovery

### Service registrieren

```java
NamingService namingService = NacosFactory.createNamingService("127.0.0.1:8848");
namingService.registerInstance("mein-service", "192.168.1.10", 8080);
```

### Service entdecken

```java
List<Instance> instances = namingService.selectInstances("mein-service", true);
```

## Konfigurationsmanagement

### Konfiguration erstellen

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v1/cs/configs' \
  -d 'dataId=app-config' \
  -d 'group=DEFAULT_GROUP' \
  -d 'content=schlüssel=wert'
```

### Konfiguration abrufen

```bash
curl 'http://127.0.0.1:8848/nacos/v1/cs/configs?dataId=app-config&group=DEFAULT_GROUP'
```

## Namespace

### Namespace erstellen

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v1/console/namespaces' \
  -d 'customNamespaceId=dev' \
  -d 'namespaceName=Entwicklung'
```

## Best Practices

1. Verwenden Sie aussagekräftige Servicenamen
2. Organisieren Sie Konfigurationen mit Gruppen
3. Aktivieren Sie Authentifizierung in der Produktion
4. Überwachen Sie die Cluster-Gesundheit
