# Node.js 常見問題

## 安裝相關問題

### Q: 如何檢查 Node.js 是否正確安裝？
**A:** 在終端機中執行以下指令：
```bash
node --version
npm --version
```
如果顯示版本號，說明安裝成功。

### Q: 安裝後指令未找到怎麼辦？
**A:** 這通常是因為 PATH 環境變數未正確設定。解決方案：
- **Windows**: 重新執行安裝程式，確保勾選 "Add to PATH" 選項
- **macOS/Linux**: 如果使用套件管理器安裝，通常會自動設定。手動新增：
  ```bash
  export PATH=$PATH:/usr/local/bin
  ```

### Q: 如何在不同專案中使用不同版本的 Node.js？
**A:** 推薦使用 nvm (Node Version Manager)：
```bash
# 安裝 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安裝特定版本
nvm install 18.17.0

# 切換版本
nvm use 18.17.0

# 設定預設版本
nvm alias default 18.17.0
```

## npm 相關問題

### Q: npm install 太慢怎麼辦？
**A:** 可以設定使用國內鏡像來源：
```bash
# 臨時使用淘寶鏡像
npm install --registry https://registry.npmmirror.com

# 永久設定淘寶鏡像
npm config set registry https://registry.npmmirror.com

# 驗證設定
npm config get registry
```

### Q: 全域安裝套件時出現權限錯誤？
**A:** 解決方案：
1. **推薦**: 使用 nvm 管理 Node.js
2. **設定 npm 全域目錄**:
   ```bash
   mkdir ~/.npm-global
   npm config set prefix '~/.npm-global'
   export PATH=~/.npm-global/bin:$PATH
   ```
3. **不推薦**: 使用 sudo（存在安全風險）

### Q: 如何更新全域安裝的套件？
**A:** 
```bash
# 查看過期的全域套件
npm outdated -g

# 更新特定套件
npm update -g package-name

# 批次更新所有全域套件（需要額外套件）
npm install -g npm-check-updates
ncu -g
```

## 開發相關問題

### Q: 如何處理非同步操作中的錯誤？
**A:** 始終使用 try-catch 包裝 async/await 程式碼：
```javascript
async function fetchData() {
  try {
    const data = await api.getData();
    return data;
  } catch (error) {
    console.error('取得資料失敗:', error);
    throw error; // 或回傳預設值
  }
}
```

對於 Promise 鏈，使用 `.catch()`：
```javascript
api.getData()
  .then(data => process(data))
  .catch(error => console.error('Error:', error));
```

### Q: 如何避免回呼地獄（Callback Hell）？
**A:** 使用現代 JavaScript 特性：
1. **Promise 鏈**:
   ```javascript
   step1()
     .then(result1 => step2(result1))
     .then(result2 => step3(result2))
     .catch(error => handleError(error));
   ```

2. **async/await** (推薦):
   ```javascript
   async function processSteps() {
     try {
       const result1 = await step1();
       const result2 = await step2(result1);
       const result3 = await step3(result2);
       return result3;
     } catch (error) {
       handleError(error);
     }
   }
   ```

### Q: 如何處理記憶體洩漏？
**A:** 常見原因和解決方案：
1. **全域變數累積**: 避免不必要的全域變數
2. **事件監聽器未移除**: 使用 `removeListener()` 或 `once()`
3. **定時器未清除**: 使用 `clearTimeout()` 和 `clearInterval()`
4. **快取未清理**: 實作 LRU 快取或定期清理

使用記憶體分析工具：
```bash
# 產生堆疊快照
node --inspect app.js
```

## 效能相關問題

### Q: 如何提高 Node.js 應用效能？
**A:** 
1. **使用叢集模式**: 利用多核心 CPU
   ```javascript
   const cluster = require('cluster');
   const numCPUs = require('os').cpus().length;
   ```

2. **避免阻塞操作**: 始終使用非同步 I/O

3. **使用串流處理大檔案**:
   ```javascript
   const fs = require('fs');
   const readStream = fs.createReadStream('large-file.txt');
   const writeStream = fs.createWriteStream('output.txt');
   readStream.pipe(writeStream);
   ```

4. **最佳化資料庫查詢**: 使用索引，避免 N+1 查詢

5. **使用快取**: Redis 或記憶體快取減少重複計算

### Q: 如何監控 Node.js 應用？
**A:** 內建監控工具：
```bash
# 啟用效能監控
node --prof app.js

# 產生效能報告
node --prof-process isolate-*.log > profile.txt
```

第三方監控工具：
- **PM2**: 行程管理和監控
- **New Relic**: APM 監控
- **Datadog**: 基礎設施監控

## 安全相關問題

### Q: 如何防止相依性漏洞？
**A:** 
1. **定期稽核相依性**:
   ```bash
   npm audit
   npm audit fix
   ```

2. **使用鎖定檔案**: 提交 `package-lock.json` 確保相依性一致性

3. **限制相依性範圍**: 只安裝必要的相依性

4. **使用 Snyk 或 Dependabot**: 自動偵測和修復漏洞

### Q: 如何處理環境變數？
**A:** 使用 `.env` 檔案（不要提交到版本控制）：
```bash
# .env
DATABASE_URL=your-database-url
API_KEY=your-api-key
```

在程式碼中使用：
```javascript
require('dotenv').config();
const dbUrl = process.env.DATABASE_URL;
```

## 偵錯相關問題

### Q: 如何偵錯 Node.js 應用？
**A:** 
1. **內建偵錯器**:
   ```bash
   node --inspect-brk app.js
   ```

2. **VS Code 偵錯**: 建立 `.vscode/launch.json` 設定

3. **日誌記錄**: 使用 `winston` 或 `bunyan` 等日誌函式庫

4. **堆疊追蹤**: 使用 `console.trace()` 查看呼叫堆疊

### Q: 如何處理未捕獲的例外？
**A:** 新增全域例外處理器：
```javascript
// 未捕獲的例外
process.on('uncaughtException', (error) => {
  console.error('未捕獲的例外:', error);
  process.exit(1);
});

// 未處理的 Promise 拒絕
process.on('unhandledRejection', (reason, promise) => {
  console.error('未處理的 Promise 拒絕:', reason);
  process.exit(1);
});
```

## 部署相關問題

### Q: 如何部署 Node.js 應用？
**A:** 推薦使用行程管理器：
1. **PM2**:
   ```bash
   npm install -g pm2
   pm2 start app.js
   pm2 startup
   pm2 save
   ```

2. **Docker**:
   ```dockerfile
   FROM node:18-alpine
   WORKDIR /app
   COPY package*.json ./
   RUN npm ci --only=production
   COPY . .
   CMD ["node", "app.js"]
   ```

3. **Systemd** (Linux): 建立服務檔案管理應用

### Q: 如何處理環境差異？
**A:** 使用環境變數區分不同環境：
```javascript
const isProduction = process.env.NODE_ENV === 'production';
const port = process.env.PORT || 3000;

if (isProduction) {
  // 生產環境設定
} else {
  // 開發環境設定
}
```
