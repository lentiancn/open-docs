# Flutter 常见问题

## 安装问题

### Flutter doctor 显示错误怎么办？

flutter doctor 命令会检查 Flutter 开发环境的状态。常见错误及解决方法：

- Android SDK 未找到：需要安装 Android Studio 或手动配置 ANDROID_SDK_ROOT 环境变量
- Xcode 版本过旧：更新 Xcode 到最新版本
- 许可证未接受：运行 flutter doctor --android-licenses 接受许可证

### 下载 Flutter 失败怎么办？

如果下载速度慢或失败，可以尝试使用镜像源。中国用户可以设置 PUB_HOSTED_URL 和 FLUTTER_STORAGE_BASE_URL 环境变量使用国内镜像。

### Git 克隆失败怎么办？

确保网络可以访问 GitHub。可以尝试使用代理或使用国内镜像。也可以从 Flutter 官网下载预编译的压缩包手动安装。

## 开发问题

### 热重载不生效怎么办？

热重载失败时，尝试以下方法：点击 Stop 然后重新运行；执行 flutter clean 清理构建缓存；检查代码中是否有语法错误。

### 应用卡顿怎么办？

检查是否在 build 方法中执行了耗时操作。复杂计算应该使用 Isolate 或异步处理。避免不必要的 Widget 重建，使用 const 构造函数和适当的 key。

### 状态丢失怎么办？

热重载会保留 StatefulWidget 的状态，但重建应用或执行完全热重启后会重置状态。如果需要持久化状态，使用 SharedPreferences、SQLite 或其他存储方案。

### 包依赖冲突怎么办？

运行 flutter pub get 查看依赖树。使用 flutter pub deps 检查包依赖。尝试更新包版本或使用 dependency_overrides 解决冲突。

## 平台问题

### Android 构建失败怎么办？

常见原因包括：SDK 版本不匹配、Gradle 配置问题、签名配置错误等。尝试运行 flutter clean 后重新构建。检查 android/build.gradle 和 android/app/build.gradle 配置。

### iOS 模拟器无法启动怎么办？

确保 Xcode 已安装并正确配置。在终端运行 xcrun simctl list devices 查看可用的模拟器。尝试重置模拟器或创建新模拟器。

### Web 构建报错怎么办？

检查 pubspec.yaml 中的 web 依赖是否正确。使用 flutter clean 清理后重新构建。确保 Chrome 浏览器已安装。

## 性能问题

### 应用体积太大怎么办？

使用 flutter build apk --split-per-abi 生成针对不同 CPU 架构的 APK。检查并移除未使用的图片和资源。使用混淆和压缩。

### 启动时间太长怎么办？

减少 main 函数中的初始化操作。延迟加载非必要的组件。使用 Splash Screen 改善用户体验。

### 内存占用过高怎么办？

检查是否有内存泄漏。使用 Profile 工具分析内存使用。及时释放不再使用的资源和大对象。

## 学习问题

### 如何选择状态管理方案？

简单应用使用 setState 即可。中等复杂度应用推荐 Provider。复杂应用可以使用 Riverpod、GetX 或 BLoC。

### 从其他框架迁移怎么办？

Flutter 官网有针对 Android、iOS、React Native 等开发者的迁移指南。核心概念是相通的，重点学习 Widget 和布局系统。

### 如何提高开发效率？

熟练使用 VS Code 或 Android Studio 的 Flutter 插件。掌握快捷键和代码模板。使用热重载和调试工具。参考官方文档和社区资源。
