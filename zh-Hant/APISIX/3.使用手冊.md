# 使用手册

本手册详细介绍 Apache APISIX 的核心功能和使用方法，包括路由配置、上游管理、负载均衡、限速、身份验证等关键功能。

## 基本概念

### Route（路由）

Route（路由）是访问上游目标的路径。在 APISIX 中，路由首先通过预定的规则来匹配客户端请求，然后加载和执行相应的插件，最后将请求转发至特定的上游服务。

一个最简单的路由仅由匹配路径和上游地址两个信息组成。

### Upstream（上游）

Upstream（上游）是一组具备相同功能的节点集合，是对虚拟主机的抽象。Upstream 可以通过预先配置的规则对多个服务节点进行负载均衡。

### Consumer（消费者）

Consumer（消费者）是指使用 API 的应用或开发人员。在 APISIX 中，消费者需要一个全局唯一的名称，并绑定一个身份验证插件。

### Plugin（插件）

APISIX 提供了丰富的插件系统，用于扩展网关功能。插件可以在路由、消费者或全局级别进行路由

### 创建配置。

## 配置路由

通过管理 API 可以轻松创建路由。以下示例创建一个将请求转发到 httpbin.org 的路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "getting-started-ip",
  "uri": "/ip",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

创建成功将返回 `HTTP/1.1 201 Created`。

### 验证路由

```bash
curl "http://127.0.0.1:9080/ip"
```

正常情况下会返回类似以下内容：

```json
{
  "origin": "183.94.122.205"
}
```

### 路由匹配规则

APISIX 支持多种路由匹配方式：

**URI 匹配：**

```json
{
  "uri": "/api/v1/*",
  "uri": ["/api/v1/*", "/api/v2/*"]
}
```

**Host 匹配：**

```json
{
  "host": "example.com",
  "host": ["example.com", "*.example.com"]
}
```

**请求方法匹配：**

```json
{
  "methods": ["GET", "POST"]
}
```

**Header 匹配：**

```json
{
  "headers": {
    "Content-Type": "application/json",
    "X-API-Key": "*"
  }
}
```

## 负载均衡

APISIX 支持多种负载均衡算法，包括轮询、加权轮询、最小连接数等。

### 配置加权负载均衡

创建一个具有多个上游服务的路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "load-balance-route",
  "uri": "/headers",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:443": 1,
      "mock.api7.ai:443": 1
    },
    "pass_host": "node",
    "scheme": "https"
  }
}'
```

**参数说明：**

- `type`：负载均衡算法类型，支持 `roundrobin`（轮询）、`weighted_roundrobin`（加权轮询）、`least_conn`（最小连接数）
- `nodes`：上游服务节点，格式为 `{"host:port": weight}`
- `pass_host`：设置为 `node` 时，将请求头传递给上游
- `scheme`：上游协议，支持 `http` 和 `https`

### 验证负载均衡

生成 100 个请求测试均衡效果：

```bash
hc=$(seq 100 | xargs -I {} curl "http://127.0.0.1:9080/headers" -sL | grep "httpbin" | wc -l); echo httpbin.org: $hc, mock.api7.ai: $((100 - $hc))
```

请求会几乎平均分配给两个上游服务。

## 限速

APISIX 提供灵活的限速功能，通过 `limit-count` 插件实现。

### 启用限速

为已有路由添加限速插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "count": 2,
      "time_window": 10,
      "rejected_code": 503
    }
  }
}'
```

**参数说明：**

- `count`：时间窗口内允许的最大请求数
- `time_window`：时间窗口大小（秒）
- `rejected_code`：超过限制时返回的 HTTP 状态码

上述配置将传入流量限制为每 10 秒最多 2 个请求。

### 验证限速

生成 100 个请求测试限速效果：

```bash
count=$(seq 100 | xargs -I {} curl "http://127.0.0.1:9080/ip" -I -sL | grep "503" | wc -l); echo "200": $((100 - count)), "503": $count
```

结果：2 个请求成功（200），98 个请求被拒绝（503）。

### 禁用限速

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "_meta": {
        "disable": true
      }
    }
  }
}'
```

### 高级限速配置

**使用变量：**

```json
{
  "plugins": {
    "limit-count": {
      "count": 100,
      "time_window": 60,
      "key": "remote_addr"
    }
  }
}
```

**使用 Redis 集群：**

```json
{
  "plugins": {
    "limit-count": {
      "count": 100,
      "time_window": 60,
      "key": "remote_addr",
      "policy": "redis",
      "redis_host": "127.0.0.1",
      "redis_port": 6379,
      "redis_password": "",
      "redis_database": 1
    }
  }
}
```

## 身份验证

APISIX 支持多种身份验证方式，包括 Key Auth、Basic Auth、JWT、OAuth2 等。

### Key Authentication（密钥验证）

#### 创建消费者

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/consumers" -X PUT -d '
{
  "username": "tom",
  "plugins": {
    "key-auth": {
      "key": "secret-key"
    }
  }
}'
```

> ⚠️ 生产环境请使用复杂的密钥。

#### 为路由启用密钥验证

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "key-auth": {}
  }
}'
```

#### 验证身份验证

**不带密钥的请求：**

```bash
curl -i "http://127.0.0.1:9080/ip"
```

返回：`HTTP/1.1 401 Unauthorized`

**携带错误密钥：**

```bash
curl -i "http://127.0.0.1:9080/ip" -H 'apikey: wrong-key'
```

返回：`HTTP/1.1 401 Unauthorized`

**携带正确密钥：**

```bash
curl -i "http://127.0.0.1:9080/ip" -H 'apikey: secret-key'
```

返回：`HTTP/1.1 200 OK`

### 其他身份验证方式

**JWT Authentication：**

```json
{
  "plugins": {
    "jwt-auth": {
      "key": "user-key",
      "secret": "my-secret-key"
    }
  }
}
```

**Basic Authentication：**

```json
{
  "plugins": {
    "basic-auth": {
      "hide_credentials": false
    }
  }
}
```

**HMAC Authentication：**

```json
{
  "plugins": {
    "hmac-auth": {
      "secret_key": "my-secret-key"
    }
  }
}
```

## 服务治理

### 灰度发布

APISIX 支持基于权重的灰度发布和金丝雀发布：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "canary-route",
  "uri": "/api/*",
  "upstream": {
    "type": "weighted_roundrobin",
    "nodes": [
      {"host": "v1.example.com", "port": 80, "weight": 90},
      {"host": "v2.example.com", "port": 80, "weight": 10}
    ]
  }
}'
```

### 请求改写

使用 `proxy-rewrite` 插件改写请求：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "rewrite-route",
  "uri": "/old-path/*",
  "plugins": {
    "proxy-rewrite": {
      "uri": "/new-path/$uri",
      "headers": {
        "X-New-Header": "value"
      }
    }
  },
  "upstream": {
    "nodes": {
      "backend.example.com:80": 1
    }
  }
}'
```

### 响应缓存

使用 `proxy-cache` 插件缓存上游响应：

```json
{
  "plugins": {
    "proxy-cache": {
      "cache_key": ["uri", "query_string"],
      "cache_http_status": [200, 301, 404],
      "cache_ttl": 60
    }
  }
}
```

## 插件配置

### 插件执行顺序

可以通过 `plugins` 配置中的 `phase` 指定插件执行阶段：

- `rewrite`：请求改写阶段
- `access`：访问控制阶段
- `before_proxy`：代理前阶段
- `header_filter`：响应头过滤阶段
- `body_filter`：响应体过滤阶段
- `log`：日志记录阶段

### 禁用插件

将插件的 `_meta.disable` 设置为 `true`：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/route-id" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "_meta": {
        "disable": true
      }
    }
  }
}'
```

## 监控与可观测性

### 集成 Prometheus

```json
{
  "plugins": {
    "prometheus": {
      "prefer_name": true
    }
  }
}
```

访问指标：

```bash
curl "http://127.0.0.1:9091/prometheus"
```

### 日志记录

APISIX 支持多种日志输出：

**HTTP 日志：**

```json
{
  "plugins": {
    "http-logger": {
      "uri": "http://log-server:8080/logs"
    }
  }
}
```

**错误日志：**

```json
{
  "plugins": {
    "error-log-logger": {
      "level": "ERROR",
      "uri": "http://log-server:8080/logs"
    }
  }
}
```

## 最佳实践

1. **使用有意义的 ID**：为路由、上游等资源使用描述性 ID
2. **合理设置超时**：根据上游响应时间设置合理的超时参数
3. **启用健康检查**：为关键服务配置主动和被动健康检查
4. **配置适当的限速**：根据业务需求和服务器能力设置限速规则
5. **启用身份验证**：生产环境必须启用身份验证
6. **使用 TLS**：对外暴露的接口应使用 HTTPS
7. **监控关键指标**：配置 Prometheus 监控并设置告警
