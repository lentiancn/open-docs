# インストールガイド

このドキュメントでは、Apache APISIX の複数のインストール方法について説明し、APIゲートウェイ環境をすばやく構築できるようサポートします。

## 動作環境

APISIX をインストールする前に、以下の動作環境を満たしていることを確認してください：

- **OS**：Linux（Ubuntu 20.04+、CentOS 7+、Debian 11+ を推奨）
- **Docker**（Dockerデプロイの場合）：Docker と Docker Compose がインストール済み
- **etcd**：APISIX は etcd を設定ストレージとして使用し、etcd 3.4.0+ が必要
- **Perl**：一部の変更Dependencies で Perl 5.5.32+ が必要
- **curl**：インストール確認用

## Docker クイックスタート（推奨）

これが最も速いインストール方法で、開発やテスト環境に適しています。

### ステップ1：Quick Start ディレクトリを作成

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### ステップ2：Quick Start スクリプトを実行

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

このコマンドは apisix-quickstart と etcd の2つのコンテナを自動的に起動します。APISIX は etcd を使用して設定を保存と同期を行い、Docker の host ネットワークモードを使用するため、ローカルから直接アクセスできます。

### ステップ3：インストール確認

インストールに成功すると、ターミナルに以下が表示されます：

```
✔ APISIX is ready!
```

以下のコマンドで APISIX の実行状態を確認できます：

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

正常に動作している場合、以下が返されます：

```
Server: APISIX/Version
```

Version にはインストールされた APISIX のバージョン（例：APISIX/3.15.0）が入ります。

### ステップ4：Dashboard にアクセス

APISIX には組み込みの Dashboard UI が付属しており、以下のアドレスでアクセスできます：

```
http://127.0.0.1:9180/ui
```

> ⚠️ 注意：より良い体験を提供するため、管理APIはデフォルトで認証が無効になっています。本番環境では必ず認証を有効にしてください。

## Docker Compose インストール

カスタム設定が必要な場合は、Docker Compose でデプロイできます。

### ステップ1：設定ファイルを作成

docker-compose.yaml ファイルを作成します：

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### ステップ2：サービスを起動

```bash
docker-compose up -d
```

### ステップ3：サービス確認

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Kubernetes インストール

APISIX は公式の Ingress Controller を提供しており、Kubernetes に直接デプロイできます。

### Helm Chart でインストール

```bash
# APISIX Helm リポジトリを追加
helm repo add apisix https://charts.apiseven.com
helm repo update

# APISIX をインストール
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### kubectl でインストール

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## ソースコードからビルド

カスタムビルドが必要な本番環境に適しています。

### ステップ1：依存関係をインストール

**Ubuntu/Debian：**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL：**

```bash
sudo yum install -y perl gcc git curl
```

### ステップ2：OpenResty をインストール

```bash
# OpenResty リポジトリを追加
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# OpenResty をインストール
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### ステップ3：APISIX をインストール

```bash
# ソースをクローン
git clone https://github.com/apache/apisix.git
cd apisix

# 依存関係をインストール
make deps

# 設定を初期化
make init

# APISIX を起動
make run
```

## インストール確認

どのインストール方法を選択しても、以下で確認できます：

1. **プロセス状態を確認**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **テストルートを作成**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **ルートをテスト**

```bash
curl "http://127.0.0.1:9080/test"
```

## アンインストール

**Docker の場合：**

```bash
# コンテナを停止
docker-compose down

# データボリュームを削除（必要な場合）
docker-compose down -v
```

**Kubernetes の場合：**

```bash
helm uninstall apisix
```
