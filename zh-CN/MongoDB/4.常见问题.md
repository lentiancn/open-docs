# MongoDB 常见问题

## 基础问题

### MongoDB 是关系型数据库吗？

不是。MongoDB 是文档型 NoSQL 数据库，以 BSON 格式存储数据，结构灵活，不像关系型数据库那样需要预先定义表结构。

### MongoDB 中的 "Collection" 是什么？

Collection（集合）相当于关系型数据库中的表，是存储文档的容器。但与表不同的是，集合中的文档可以拥有不同的结构。

### MongoDB 中的 "Document" 是什么？

Document（文档）是 MongoDB 中的基本数据单元，类似于 JSON 对象，使用 BSON 格式存储。每个文档有一个唯一的 `_id` 字段作为主键。

### 为什么要使用 MongoDB？

MongoDB 适合以下场景：数据结构频繁变化、需要存储嵌套对象、追求开发速度、需要水平扩展能力、大量非结构化或半结构化数据。

### MongoDB 和 MySQL 有什么区别？

MySQL 是关系型数据库，需要预先定义表结构，使用 SQL 查询；MongoDB 是文档型数据库，结构灵活，使用 JavaScript 查询语法。MongoDB 适合快速迭代，MySQL 适合事务性要求高的场景。

## 安装问题

### MongoDB 安装失败怎么办？

常见解决方法：检查系统兼容性、以管理员权限运行安装程序、关闭杀毒软件后重试、确认磁盘空间充足、查看安装日志定位具体错误。

### 如何修改 MongoDB 数据存储路径？

编辑配置文件 `mongod.conf`，修改 `storage.dbPath` 为目标路径，然后重启 MongoDB 服务。确保目标目录有正确的读写权限。

### MongoDB 默认端口是多少？

默认端口是 27017。可以在配置文件中通过 `net.port` 修改。

### 如何以服务方式运行 MongoDB？

Windows：安装时勾选 "Install as a Windows Service" 选项。Linux：使用 systemctl 命令启用开机自启。

### 如何查看 MongoDB 日志？

默认日志路径：Linux/macOS 在 `/var/log/mongodb/mongod.log`，Windows 在数据目录下的 `log` 文件夹。也可以在配置文件中指定自定义路径。

## 使用问题

### 如何备份 MongoDB 数据？

使用 mongodump 命令：

```bash
mongodump --db myDatabase --out ./backup
```

恢复使用 mongorestore：

```bash
mongorestore --db myDatabase ./backup/myDatabase
```

### MongoDB 支持事务吗？

支持。从 MongoDB 4.0 开始支持多文档事务，4.2+ 支持分片集群事务。事务可以确保 ACID 特性，但会带来性能开销，谨慎使用。

### MongoDB 如何实现分页查询？

使用 skip 和 limit：

```javascript
db.collection.find().skip((page - 1) * pageSize).limit(pageSize)
```

大规模数据分页时，建议使用基于游标的分页方式，避免 skip 跳过大量数据带来的性能问题。

### 如何优化 MongoDB 查询性能？

创建适当的索引、使用投影只返回必要字段、避免全表扫描、利用聚合管道进行数据处理、使用覆盖索引。

### MongoDB 如何处理大量数据写入？

使用批量插入（bulk write）、合理设置写入concern级别、考虑使用副本集分担压力、在非高峰时段进行大批量操作。

### 如何查看慢查询？

```javascript
// 查看最近的慢查询
db.getProfilingStatus()

// 设置慢查询阈值（毫秒）
db.setProfilingLevel(1, 100)

// 查看日志
db.system.profile.find().sort({ millis: -1 }).limit(10)
```

### MongoDB 支持 SQL 查询吗？

MongoDB 自己有一套查询语言，不直接支持 SQL。但可以使用 MongoDB Connector for BI 等工具将 SQL 转换为 MongoDB 查询。

## 性能问题

### MongoDB 占用内存过高怎么办？

MongoDB 默认会尽量占用可用内存用于数据缓存，这是正常行为。如果内存占用异常，检查：是否创建了不必要的索引、是否有大集合未分片、是否开启了内存映射。

### 查询很慢怎么办？

检查是否需要创建索引、使用 explain() 分析查询计划、避免在查询中使用函数、考虑使用覆盖索引、适当使用投影减少数据传输量。

### 如何监控 MongoDB 性能？

可以使用：MongoDB Atlas 免费监控、mongostat 和 mongotop 命令、db.serverStatus() 查看状态、第三方监控工具如 Prometheus + Grafana。

### 副本集和分片有什么区别？

副本集提供数据冗余和高可用，主节点故障时自动切换；分片将数据分散到多个服务器，实现水平扩展。两者可以同时使用。

### 何时需要使用分片？

当单机存储容量不足、写入压力过大、查询性能下降时，可以考虑分片。分片会增加系统复杂度，建议在确实需要时才使用。

## 安全问题

### 如何启用 MongoDB 身份验证？

1. 创建管理员用户：

```javascript
use admin
db.createUser({
  user: "admin",
  pwd: "密码",
  roles: [{ role: "root", db: "admin" }]
})
```

2. 修改配置文件启用认证：

```yaml
security:
  authorization: enabled
```

3. 重启 MongoDB 服务。

### MongoDB 默认不需要密码吗？

是的，默认安装不启用认证。生产环境必须启用身份验证，并配置防火墙限制访问。

### 如何加密 MongoDB 数据？

MongoDB 支持静态数据加密（Encryption at Rest），需要企业版或使用文件级加密（如 Linux dm-crypt）。传输层加密使用 TLS/SSL。

## 常见错误

### "Connection refused" 错误？

检查 MongoDB 服务是否启动、防火墙是否放行 27017 端口、bindIp 配置是否正确。

### "Duplicate key error" 错误？

尝试插入已存在的 `_id` 字段或有唯一索引的字段。解决方法：检查数据、使用 update 代替 insert、更新唯一键值。

### "Out of memory" 错误？

内存不足。解决方法：增加服务器内存、优化查询减少内存使用、分片大数据集、使用聚合管道时注意内存限制。

### "Lock timeout" 错误？

写锁等待超时。常见原因：长时间运行的更新操作、大批量写入、磁盘 I/O 瓶颈。优化方法：拆分大批量操作、使用索引减少锁竞争时间。

## 版本与升级

### 如何升级 MongoDB 版本？

1. 备份所有数据
2. 查阅官方升级指南
3. 按步骤逐版本升级（不要跨大版本）
4. 验证数据和功能正常

### MongoDB 社区版和企业版有什么区别？

企业版提供：高级安全特性（LDAP 集成、审计、加密）、管理工具、技术支持。社区版功能足够大多数使用场景。
