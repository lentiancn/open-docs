# Mockito

## Обзор

Mockito - самый популярный фреймворк для mockирования юнит-тестов в экосистеме Java, разработанный для тестирования, управляемого тестами (TDD) и разработки, управляемой поведением (BDD). Он позволяет разработчикам создавать макетные объекты, которые эмулируют поведение реальных объектов контролируемым образом, обеспечивая независимое и надежное юнит-тестирование.

## Основные характеристики

Чистый и простой API упрощает изучение и использование. Мощные функции поддерживают проверку вызовов методов, настройку возвращаемых значений, симуляцию исключений, сопоставление аргументов и многое другое. Отличная читаемость делает тестовый код интуитивно понятным, с четкими сообщениями об ошибках при сбое проверки. Активное сообщество обеспечивает непрерывную поддержку фреймворка и полную документацию.

## Быстрый старт

### Создание Mock-объекта

```java
List<String> mockedList = mock(List.class);
```

### Определение поведения

```java
when(mockedList.get(0)).thenReturn("first");
```

### Проверка взаимодействий

```java
verify(mockedList).get(0);
```

## Требования

Mockito 5.x требует JDK 11 или выше. Mockito 4.x поддерживает JDK 8. Проекты могут использовать Maven или Gradle для управления зависимостями.
