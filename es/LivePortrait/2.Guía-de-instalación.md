# Guía de Instalación de LivePortrait

Este documento proporciona instrucciones de instalación detalladas para LivePortrait en diferentes sistemas operativos y entornos.

## Requisitos

Antes de la instalación, asegúrese de que su sistema cumpla con los siguientes requisitos básicos:

**Sistema Operativo**: Linux (recomendado), Windows 10/11, macOS (requiere Apple Silicon)

**Versión de Python**: Python 3.10

**Requisitos de Hardware**: NVIDIA GPU (RTX 4090 u otra tarjeta compatible con CUDA recomendada), mínimo 8GB de VRAM

**Espacio de Almacenamiento**: Mínimo 20GB recomendados para pesos de modelos y entorno de ejecución

## Configuración del Entorno Básico

### Verificar la Versión de CUDA

Primero, confirme la versión de CUDA de su sistema para instalar la versión apropiada de PyTorch:

```bash
nvcc -V
```

Las versiones comunes de CUDA incluyen 11.1, 11.8 y 12.1. Según la versión mostrada, seleccione el comando de instalación de PyTorch correspondiente.

### Instalar PyTorch

Elija el comando de instalación de PyTorch apropiado según su versión de CUDA:

**Para CUDA 11.1**:

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**Para CUDA 11.8**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**Para CUDA 12.1**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

Si solo necesita ejecutar el modo Humanos, puede omitir la verificación de versión de CUDA e instalar la última versión de PyTorch directamente.

## Descargar el Código del Proyecto

Clone el repositorio de LivePortrait usando Git:

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Crear un Entorno Python

Se recomienda crear un entorno Python aislado usando conda:

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

O usando venv:

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## Instalar Dependencias

Ejecute lo siguiente en el directorio raíz del proyecto:

```bash
pip install -r requirements.txt
```

## Descargar Modelos Preentrenados

Los modelos preentrenados se pueden descargar de HuggingFace. Si su red tiene acceso a HuggingFace:

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Si no puede acceder a HuggingFace, use un sitio espejo:

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Alternativamente, descargue los archivos completos del modelo desde Google Drive o Baidu Netdisk, extráigalos y colóquelos en el directorio ./pretrained_weights.

## Verificar la Instalación

Después de la instalación, ejecute el script de prueba para verificar que el entorno esté configurado correctamente:

```bash
python inference.py
```

Si todo funciona correctamente, verá el archivo de salida generado en el directorio animations.

## Pasos Adicionales de Instalación para Modelos de Animales

El modo de animales solo es compatible con sistemas Linux y Windows con GPU NVIDIA.

### Instalar Dependencia X-Pose

El modo de animales requiere el proyecto X-Pose como marco de detección de puntos clave. Primero, navegue al directorio X-Pose:

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### Descargar Modelos de Animales desde HuggingFace

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Notas Especiales para Windows

En sistemas Windows, ciertas versiones más altas de CUDA (como 12.4, 12.6) pueden causar problemas de compatibilidad. Se recomienda downgradear CUDA a la versión 11.8 para una mejor estabilidad.

Los usuarios de Windows también pueden elegir descargar el instalador de un clic y ejecutar haciendo doble clic en run_windows.bat después de extraer:

```bash
# Descargar el instalador de un clic para Windows desde HuggingFace
# Dirección: https://huggingface.co/cleardusk/LivePortrait-Windows
```

## Notas Especiales para macOS

### macOS con Apple Silicon

Para usuarios de macOS con Apple Silicon, use el archivo de dependencias especial:

```bash
pip install -r requirements_macOS.txt
```

Al ejecutar inferencia, establezca la variable de entorno:

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

Nota: Ejecutar inferencia en macOS con Apple Silicon puede ser aproximadamente 20 veces más lento que en RTX 4090.

### Mac con Intel

Los Macs basados en Intel no han sido probados oficialmente y pueden no funcionar correctamente.

### Limitación del Modo de Animales

Como la dependencia X-Pose no es compatible con macOS, el modo de animales no está disponible en macOS. Solo se puede usar el modo humano.

## Verificar la Estructura del Directorio

Después de la instalación, asegúrese de que la estructura del directorio cumpla con lo siguiente:

```
LivePortrait/
├── pretrained_weights/     # Directorio de pesos de modelos preentrenados
├── assets/                 # Directorio de materiales de ejemplo
│   ├── examples/
│   │   ├── source/        # Imagen/video fuente
│   │   └── driving/       # Video de control
├── src/                   # Directorio de código fuente
├── inference.py          # Script de inferencia
├── app.py                # Interfaz Gradio
└── requirements.txt     # Lista de dependencias
```

## Problemas Comunes

Si encuentra problemas durante la instalación, verifique lo siguiente:

**CUDA no disponible**: Confirme que los controladores NVIDIA estén correctamente instalados; ejecute nvidia-smi para verificar el estado del controlador

**VRAM insuficiente**: Intente usar imágenes de entrada más pequeñas o reducir el tamaño del lote

**Fallo de descarga del modelo**: Verifique la conexión de red o use sitios espejo nacionales

**Conflictos de dependencias**: Se recomienda usar un entorno Python nuevo para evitar conflictos con otros proyectos
