# Yarn 使用手冊

## 基本命令

### 通用命令

- `yarn add` - 向專案新增相依性
- `yarn install` - 安裝專案相依性
- `yarn remove` - 從專案中移除相依性
- `yarn run` - 執行 package.json 中定義的指令稿
- `yarn init` - 建立新套件
- `yarn info` - 查看套件相關資訊
- `yarn why` - 顯示為什麼需要某個套件

### 進階命令

- `yarn dlx` - 在臨時環境中執行套件
- `yarn dedupe` - 去重具有重疊範圍的相依性
- `yarn up` - 跨專案升級相依性
- `yarn pack` - 從活動工作區產生 tarball
- `yarn rebuild` - 重新建置專案的原生套件

### NPM 相關命令

- `yarn npm audit` - 對已安裝的套件執行漏洞稽核
- `yarn npm publish` - 將活動工作區發布到 npm 註冊表
- `yarn npm login` - 儲存新的登入資訊以存取 npm 註冊表
- `yarn npm logout` - 從 npm 註冊表登出
- `yarn npm whoami` - 顯示已驗證使用者的名稱

### 外掛管理

- `yarn plugin list` - 列出可用的官方外掛
- `yarn plugin import` - 下載外掛
- `yarn plugin remove` - 移除外掛
- `yarn plugin runtime` - 列出活動外掛

## 工作區 (Workspaces)

Yarn 是第一個專門為工作區建置的套件管理器，允許您將專案拆分為子元件。

### 工作區配置

在 `package.json` 中配置工作區：

```json
{
  "workspaces": [
    "packages/*"
  ]
}
```

### 工作區命令

- `yarn workspaces list` - 列出所有工作區
- `yarn workspace <name> <command>` - 在特定工作區中執行命令

## 設定檔

### .yarnrc.yml

Yarn v2+ 使用 `.yarnrc.yml` 檔案進行設定：

```yaml
nodeLinker: pnp
yarnPath: .yarn/releases/yarn-4.0.0.cjs
```

### .yarnrc (Yarn v1)

Yarn v1 使用 `.yarnrc` 檔案：

```
--registry https://registry.npmjs.org
--strict-semver true
```

## 版本差異說明

### Yarn v4+ vs Yarn v1

| 功能 | Yarn v4+ | Yarn v1 |
|------|----------|---------|
| 架構 | 外掛式 (Berry) | 傳統 |
| 安裝速度 | 更快 (平行安裝) | 標準速度 |
| 磁碟空間 | 更節省 (PnP) | 標準 node_modules |
| 工作區 | 原生支援 | 需要額外配置 |
| 外掛系統 | 內建支援 | 不支援 |
| 設定檔 | .yarnrc.yml | .yarnrc |

### 遷移注意事項

- Yarn v4+ 預設使用 PnP，可能與某些工具不相容
- 需要更新 CI/CD 配置以適應新的安裝流程
- 外掛系統提供了更多自訂選項
- 工作區功能更加強大和靈活
