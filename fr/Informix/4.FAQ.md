# Foire Aux Questions d'IBM Informix

## Questions Générales

### Q : Quelle est la différence entre IBM Informix et HCL Informix ?

R : IBM Informix est vendu et commercialisé par IBM, tandis que HCL Informix est développé et supporté par HCLSoftware. Les deux partagent la même base de code et sont fonctionnellement équivalents. IBM gère les ventes et les licences, tandis que HCL gère le développement et le support technique.

### Q : Quelle édition dois-je choisir pour le développement ?

R : Utilisez la **Developer Edition** (gratuit) ou **Innovator-C Edition** (gratuit) pour le développement et les tests. Les deux ont des limites de ressources mais incluent toutes les fonctionnalités :
- Developer Edition : 1 CPU VP/Core, 1Go RAM, 8Go stockage
- Innovator-C Edition : 2 CPU Cores, 8Go RAM, 50Go stockage

### Q : Informix est-il adapté aux applications IoT ?

R : Oui, Informix est excellent pour l'IoT grâce à sa petite empreinte (aussi petite que 100 Mo), ses faibles exigences d'administration, son support intégré des séries temporelles et sa capacité à fonctionner sur des appareils edge.

### Q : Quels langages de programmation sont supportés ?

R : Informix prend en charge plusieurs interfaces de programmation :
- SQL (natif)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- Compatibilité MongoDB (JSON/BSON)

## Problèmes d'Installation

### Q : L'installation échoue avec une erreur "Pas de permission"

R : Assurez-vous d'avoir les privilèges root/sudo. Créez l'utilisateur et le groupe informix avant l'installation :

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### Q : Le serveur ne démarre pas après l'installation

R : Vérifiez les points suivants :
- Les variables d'environnement sont correctement définies (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- Les fichiers `onconfig` et `sqlhosts` sont corrects
- Consultez les journaux d'erreurs dans `$INFORMIXDIR/tmp/`
- Assurez-vous d'avoir suffisamment d'espace disque

### Q : Quels ports utilise Informix ?

R : Le port par défaut est 9088. Vous pouvez le vérifier dans le fichier `sqlhosts`. Assurez-vous que le pare-feu permet ce port.

## Problèmes Opérationnels Courants

### Q : Comment sauvegarder et restaurer Informix ?

R : Utilisez les utilitaires `ontape` ou `onbar` :

```bash
# Sauvegarde complète avec ontape
ontape -s -L 0

# Restauration avec ontape
ontape -r
```

### Q : Comment vérifier le statut du serveur ?

R : Utilisez la commande `onstat` :

```bash
onstat -        # Statut général
onstat -d       # Utilisation de l'espace disque
onstat -l       # Journaux logiques
onstat -u       # Utilisateurs actifs
onstat -k       # Informations sur les verrous
```

### Q : Comment se connecter à Informix depuis des applications externes ?

R : Configurez les pilotes ODBC, JDBC ou clients natifs. Mettez à jour le fichier `sqlhosts`. Exemple de chaîne de connexion JDBC :

```
jdbc:informix-sqli://hostname:9088/dbname:INFORMIXSERVER=nomduserveur
```

### Q : Qu'est-ce qui cause l'erreur "Logical Log Full" ?

R : Cela se produit lorsque tous les journaux logiques sont plein et n'ont pas été sauvegardés. Solutions :
- Sauvegarder les journaux logiques : `ontape -s -L 0`
- Ajouter plus de fichiers de journaux logiques : `onparams -a -d dbspace -s taille`

### Q : Comment gérer les conflits de verrous ?

R : Utilisez `onstat -k` pour voir les verrous et `onstat -u` pour identifier les sessions bloquantes. Solutions :
- Augmenter le timeout de verrou
- Utiliser un niveau d'isolation plus bas (dirty read)
- Terminer la session bloquante : `onmode -z id_session`

## Performance et Optimisation

### Q : Comment améliorer les performances des requêtes ?

R :
- Créez des index appropriés pour les colonnes fréquemment interrogées
- Utilisez `EXPLAIN` pour analyser les plans de requête
- Configurez la mémoire partagée de manière appropriée
- Mettez régulièrement à jour les statistiques : `UPDATE STATISTICS FOR TABLE table`

### Q : Quels sont les paramètres clés de configuration mémoire ?

R :
- `SHMVIRTSIZE` - Taille de mémoire partagée virtuelle
- `SHMADD` - Segments de mémoire partagée supplémentaires
- `BUFFERPOOL` - Configuration du cache de données
