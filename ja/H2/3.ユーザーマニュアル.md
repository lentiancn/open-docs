# H2 データベースユーザーマニュアル

## 基本的な操作

### データベースの作成

組み込みモード（メモリデータベース）：
```java
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");
```

ファイルデータベース：
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### テーブルの作成

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### データの挿入

```sql
INSERT INTO users (name, email) VALUES ('田中', 'tanaka@example.com');
INSERT INTO users (name, email) VALUES ('鈴木', 'suzuki@example.com');
```

### データのクエリ

```sql
SELECT * FROM users;
SELECT * FROM users WHERE name = '田中';
SELECT COUNT(*) FROM users;
```

### データの更新

```sql
UPDATE users SET email = 'newemail@example.com' WHERE id = 1;
```

### データの削除

```sql
DELETE FROM users WHERE id = 2;
```

## Javaコード例

### 完全なJDBC例

```java
import java.sql.*;

public class H2Example {
    public static void main(String[] args) throws Exception {
        // ドライバーのロード
        Class.forName("org.h2.Driver");
        
        // データベースに接続
        String url = "jdbc:h2:mem:testdb";
        Connection conn = DriverManager.getConnection(url, "sa", "");
        
        // テーブルの作成
        Statement stmt = conn.createStatement();
        stmt.execute("CREATE TABLE users (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100))");
        
        // データの挿入
        stmt.execute("INSERT INTO users (name) VALUES ('田中')");
        stmt.execute("INSERT INTO users (name) VALUES ('鈴木')");
        
        // データのクエリ
        ResultSet rs = stmt.executeQuery("SELECT * FROM users");
        while (rs.next()) {
            System.out.println(rs.getInt("id") + ": " + rs.getString("name"));
        }
        
        // 接続を閉じる
        rs.close();
        stmt.close();
        conn.close();
    }
}
```

### PreparedStatementの使用

```java
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name, email) VALUES (?, ?)");
pstmt.setString(1, "佐藤");
pstmt.setString(2, "sato@example.com");
pstmt.executeUpdate();
pstmt.close();
```

### トランザクションの使用

```sql
conn.setAutoCommit(false);
try {
    Statement stmt = conn.createStatement();
    stmt.execute("INSERT INTO users (name) VALUES ('高橋')");
    stmt.execute("INSERT INTO users (name) VALUES ('伊藤')");
    conn.commit();
} catch (Exception e) {
    conn.rollback();
} finally {
    conn.setAutoCommit(true);
}
```

## 上級機能

### データベースモードの切り替え

H2は他のデータベースのSQL構文をシミュレートできます：

```java
// MySQLモード
String url = "jdbc:h2:mem:testdb;MODE=MySQL";

// PostgreSQLモード
String url = "jdbc:h2:mem:testdb;MODE=PostgreSQL";

// Oracleモード
String url = "jdbc:h2:mem:testdb;MODE=Oracle";
```

### データベース暗号化

```java
// 暗号化データベースの使用
String url = "jdbc:h2:file:./data/encrypted;CIPHER=AES";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### トレース機能

```java
// SQLトレースを有効化
String url = "jdbc:h2:mem:testdb;TRACE_LEVEL_FILE=3";
```

### TCPサーバーの起動

```java
import org.h2.tools.Server;

Server server = Server.createTcpServer().start();
System.out.println("H2 TCP Server started: " + server.getURL());
```

### H2コンソールの使用

H2 Webコンソールの起動：

```bash
java -jar h2.jar -web
```

次にブラウザで開く：http://localhost:8082

## 一般的なSQL文

### データ型

H2は複数のデータ型をサポートします：
- 整数型：INT、INTEGER、BIGINT、SMALLINT、TINYINT
- 浮動小数点型：FLOAT、REAL、DOUBLE、DECIMAL
- 文字列型：VARCHAR、CHAR、CLOB
- 日時型：DATE、TIME、TIMESTAMP
- ブール型：BOOLEAN
- バイナリ型：BLOB、BINARY

### インデックス操作

インデックスの作成：
```sql
CREATE INDEX idx_name ON users(name);
```

インデックスの削除：
```sql
DROP INDEX idx_name;
```

### 制約操作

主キーの追加：
```sql
ALTER TABLE users ADD PRIMARY KEY (id);
```

外部キーの追加：
```sql
ALTER TABLE orders ADD FOREIGN KEY (user_id) REFERENCES users(id);
```

### テーブル構造の表示

```sql
DESCRIBE users;
-- または
SHOW COLUMNS FROM users;
```

### すべてのテーブルの表示

```sql
SHOW TABLES;
```

## パフォーマンス最適化

### バッチ操作

バッチ操作を使用してパフォーマンスを向上：

```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (int i = 0; i < 1000; i++) {
    pstmt.setString(1, "User" + i);
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### 接続プール

本番環境では、HikariCPなどの接続プールを使用することをお勧めします：

```java
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:h2:file:./data/testdb");
config.setUsername("sa");
config.setPassword("");
config.setMaximumPoolSize(10);

HikariDataSource ds = new HikariDataSource(config);
```

## バックアップと復元

### データベースのバックアップ

```sql
SCRIPT TO 'backup.sql';
```

### データベースの復元

```sql
RUNSCRIPT FROM 'backup.sql';
```

## よくあるエラー処理

### データベースファイルがロックされている
他の接続がデータベースファイルを使用していないことを確認してください。H2は同時読み取りをサポートしていますが、書き込みは独占アクセスが必要です。

### メモリデータベースのデータが失われる
メモリデータベース（jdbc:h2:mem:）はすべての接続が閉じるとデータが失われます。永続化が必要な場合は、ファイルデータベースモードを使用してください。

### クラスが見つからない
org.h2.Driverクラスがclasspathにあることを確認してください。
