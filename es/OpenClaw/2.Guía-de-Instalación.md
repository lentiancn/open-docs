# OpenClaw Guía de Instalación

> Guía completa para configurar e instalar OpenClaw

---

## Requisitos del Sistema

| Requisito | Especificación |
|-----------|----------------|
| Sistema Operativo | Windows 10+, macOS, Linux |
| Node.js | Versión 18+ |
| Memoria | Al menos 4 GB RAM |
| Almacenamiento | ~500 MB |

---

## Instalación

### Prerrequisitos

Asegúrate de que Node.js esté instalado:

```bash
node --version
npm --version
```

### Instalación Paso a Paso

#### 1. Instalar vía npm

```bash
npm install -g openclaw
```

#### 2. Iniciar el Servicio Gateway

El servicio Gateway es el núcleo de OpenClaw:

```bash
# Verificar estado
openclaw gateway status

# Iniciar Gateway
openclaw gateway start

# Reiniciar Gateway
openclaw gateway restart

# Detener Gateway
openclaw gateway stop
```

#### 3. Configuración Inicial

Después de la instalación, configura los archivos:

```bash
# Crear directorio de configuración
mkdir -p ~/.openclaw

# Copiar configuración predeterminada
cp /ruta/a/openclaw/config/* ~/.openclaw/
```

---

## Configuración

### Archivos de Configuración Principales

| Archivo | Descripción |
|---------|-------------|
| AGENTS.md | Configuración del espacio de trabajo |
| SOUL.md | Configuración de personalidad |
| TOOLS.md | Configuración de herramientas |
| USER.md | Preferencias del usuario |

### Variables de Entorno

```bash
# Opcional: Directorio de trabajo personalizado
OPENCLAW_WORKSPACE=/ruta/al/workspace

# Opcional: Puerto del Gateway
OPENCLAW_PORT=3000
```

---

## Verificación

### Probar Instalación

```bash
# Mostrar información de versión
openclaw --version

# Verificar estado del Gateway
openclaw gateway status

# Mostrar comandos disponibles
openclaw --help
```

### Inicio Rápido

```bash
# Iniciar sesión interactiva
openclaw chat

# Ejecutar una tarea individual
openclaw exec "decir hola"
```

---

## Solución de Problemas

### ¿El Gateway no inicia?

**Solución**:
1. Verifica que Node.js esté correctamente instalado
2. Revisa los permisos
3. Consulta los logs: `openclaw gateway logs`

### ¿Puerto ya está en uso?

**Solución**:
```bash
# Cambiar puerto
openclaw gateway start --port 3001
```

### ¿Módulo no encontrado?

**Solución**:
```bash
# Reinstalar
npm uninstall -g openclaw
npm install -g openclaw
```

---

## Desinstalación

```bash
npm uninstall -g openclaw

# Eliminar servicio Gateway
openclaw gateway stop
rm -rf ~/.openclaw
```
