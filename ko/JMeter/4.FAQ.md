# JMeter FAQ

## 일반적인 질문

### Apache JMeter란?
Apache JMeter는 부하 테스트 및 성능 측정을 위해 설계된 오픈소스 소프트웨어입니다. 기능 동작을 테스트하고 성능을 측정할 수 있는 100% 순수 Java 애플리케이션입니다.

### JMeter는 어떤 프로토콜을 지원하나요?
JMeter는 여러 프로토콜을 지원합니다:
- HTTP/HTTPS
- SOAP/REST 웹 서비스
- FTP
- JDBC(데이터베이스)
- LDAP
- JMS(메시지 지향 미들웨어)
- SMTP/POP3/IMAP(메일)
- TCP
- Java 객체

### JMeter는 브라우저인가요?
아니요, JMeter는 브라우저가 아닙니다. 프로토콜 수준에서 작동하며 실제 브라우저처럼 JavaScript를 실행하거나 HTML 페이지를 렌더링하지 않습니다.

## 설치 질문

### 시스템 요구 사항은 무엇인가요?
- Java 8 이상
- 최소 4GB RAM 권장
- 호환 OS: Windows, Linux, macOS

### JMeter를 설치하려면 어떻게 하나요?
1. https://jmeter.apache.org/에서 다운로드
2. 아카이브 추출
3. jmeter.sh(Linux/Mac) 또는 jmeter.bat(Windows) 실행

### JMeter가 시작되지 않는 이유는 무엇인가요?
- Java가 설치되어 있는지 확인: `java -version`
- JAVA_HOME이 설정되어 있는지 확인
- 포트 충돌 확인

### JMeter의 메모리를 늘리려면 어떻게 하나요?
jmeter 시작 스크립트를 편집하고 HEAP 변수 수정:
```bash
HEAP="-Xms1g -Xmx4g"
```

## 사용 질문

### 기본 테스트 플랜을 만들려면 어떻게 하나요?
1. 스레드 그룹 추가
2. HTTP 요청 샘플러 추가
3. 리스너 추가(결과 트리, 요약 보고서)
4. 테스트 실행

### 스레드 그룹이란?
스레드 그룹은 동일한 테스트를 실행할 가상 사용자 그룹을 정의합니다. 다음을 제어:
- 스레드 수(사용자)
- 램프업 기간
- 루프 카운트

### 응답에서 데이터를 추출하려면 어떻게 하나요?
추출기 사용:
- JSON 응답에는 JSON 추출기
- XML/HTML에는 XPath 추출기
- 모든 텍스트에는 정규식 추출기

### 테스트를 매개변수화하려면 어떻게 하나요?
변수 및 함수 사용:
- `${variable}` 변수 참조
- `${__Random(1,100)}` 랜덤 값
- `${__time()}`

### JMeter가 인증을 처리할 수 있나요?
네, JMeter는 다음을 지원:
- 기본 인증
- NTLM 인증
- 쿠키 관리자
- 권한 관리자

## 성능 질문

### JMeter는 몇 명의 사용자를 시뮬레이션할 수 있나요?
수는 하드웨어에 따라 다릅니다. 적절한 튜닝 사용:
- GUI 모드: 100-300명
- CLI 모드: 1000명 이상
- 분산 테스트: 10,000명 이상

### 분산 테스트란?
분산 테스트는 더 많은 부하를 생성하기 위해 여러 시스템을 사용합니다. 마스터가 여러 슬레이브 시스템을 조정합니다.

### JMeter를 헤드리스 모드로 실행하려면?
```bash
jmeter -n -t testplan.jmx -l results.jtl
```

## 문제 해결

### 테스트 중 OutOfMemoryError
- 힙 크기 증가
- 스레드 수 감소
- GUI 대신 CLI 모드 사용
- 불필요한 리스너 비활성화

### 결과가 표시되지 않음
- 테스트가 실제로 실행 중인지 확인
- 리스너가 올바른 요소에 연결되어 있는지 확인
- 로그 파일에서 오류 확인

### 응답 시간 높음
- 네트워크 확인
- 서버 성능 확인
- 서버가 압도당하면 부하 감소

### 테스트 플랜을 디버그하려면?
1. 결과 트리 리스너 사용
2. 디버그 로깅 활성화
3. 단일 스레드로 시작
4. 어설션 확인

## 모범 사례

1. 부하 테스트에는 항상 CLI 모드 사용
2. 낮게 시작하여 점진적으로 증가
3. 응답을 검증하기 위해 어설션 사용
4. 테스트 후 정리(tarDown)
5. 테스트 플랜을 버전 관리
6. 시스템 리소스 모니터링
