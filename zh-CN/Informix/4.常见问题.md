# Informix 常见问题解答

## 逻辑日志满

**故障现象**：
- 数据库停止所有操作
- `onstat -l` 显示所有逻辑日志处于 U----- 标志（已使用未备份）

**故障分析**：
- 逻辑日志未及时备份
- 逻辑日志空间分配过小
- 逻辑日志包含活动事务
- 逻辑日志包含检查点信息

**故障处理**：
1. 检查备份设备是否正常（磁带满、磁带机故障等）
2. 检查是否有活动事务：`onstat -x`
3. 检查点之后的日志即使备份也不能立即使用
4. 动态增加逻辑日志：
```bash
onspaces -c -d llogdbs -p /home/informix/chunk/llogchk -s 20000 -o 0
onparams -a -d llogdbs -s 2000
```

## 锁冲突

**故障现象**：
- 出现 -243、-244 错误码
- `Could not position within a table`
- `Could not do a physical-order read to fetch next row`

**故障分析**：
- 其他用户占用了锁资源

**故障处理**：
1. 查看表锁占用情况：
```bash
# 获取表的 partnum
oncheck -pt database:tablename

# 查看锁信息
onstat -k | grep <partnum_hex>

# 查看具体会话
onstat -u | grep <address>
```
2. 解决方案：
   - 使用 `dirty read` 隔离级别
   - 将页级锁改为行级锁
   - 设置锁等待时间
   - 终止占用锁的会话：`onmode -z <sid>`

## 长事务

**故障现象**：
- 日志中出现长事务提示
- 事务处于回滚状态
- 严重时导致整个数据库实例停止

**故障分析**：
- 活动事务占用逻辑日志比例达到 `LTXHWM` 阈值
- 比例达到 `LTXEHWM` 时会停止其他会话

**故障处理**：
1. 查看日志定位具体事务
2. 将大事务拆分为小事务
3. 避免长时间无操作的事务
4. 增加逻辑日志空间

## Chunk I/O 错误

**故障现象**：
- 日志中出现 chunk IO 错误
- `onstat -d` 显示 chunk 状态为 down

**故障分析**：
- 磁盘设备故障
- chunk 路径不存在
- 设备权限错误

**故障处理**：
1. 检查存储设备：
```bash
dd if=/home/informix/940/dsk/data_chk1 of=/dev/null bs=2048k
```
2. 恢复 chunk：
```bash
onspaces -s <dbspace> -r
```
3. 如不能恢复，联系 IBM 技术支持

## 不能插入数据

**故障现象**：
- 报 -131、-136、-271 错误
- `ISAM error: no free disk space`
- `ISAM error: no more extents`

**故障分析**：
- dbspace 没有足够连续空间
- 达到表的最大 extent 数量
- 达到表的记录数上限

**故障处理**：
1. 定位问题表
2. 增加 chunk 空间
3. 重建表或进行分片
4. 应急处理：小批量删除数据，避免大事务

## 连接问题

**故障现象**：
- 客户端无法连接到 Informix 服务器
- 报 -25596 或 -951 错误

**故障处理**：
1. 检查 Informix 服务是否运行：`onstat`
2. 检查 sqlhosts 配置是否正确
3. 检查防火墙是否开放对应端口
4. 验证环境变量 `INFORMIXSERVER` 和 `INFORMIXDIR` 是否正确设置

## 性能问题

**故障现象**：
- 查询响应变慢
- CPU 或内存使用率过高

**故障处理**：
1. 使用 `onstat -g <option>` 查看各类统计信息
2. 检查索引是否缺失或失效
3. 分析查询计划：`SET EXPLAIN ON`
4. 调整 `onconfig` 中的缓冲池大小
5. 定期执行 `UPDATE STATISTICS` 更新统计信息

## 数据备份与恢复

**问题**：如何备份和恢复 Informix 数据库？

**解决方案**：
- 使用 `ontape` 进行逻辑日志备份和全量备份
- 使用 `onbar` 进行备份（需要配置存储管理软件）
- 定期执行 `ontape -s -L 0` 进行全量备份
- 使用 `ontape -r` 进行恢复

## 表空间管理

**问题**：如何查看和扩展表空间？

**解决方案**：
- 查看表空间：`onstat -d`
- 创建新的 chunk：`onspaces -c -d <dbspace> -p <path> -s <size>`
- 添加 chunk 到现有 dbspace：`onspaces -a <dbspace> -p <path> -s <size>`
