# Claude Code Manual del Usuario

Este manual te ayudará a comenzar rápidamente con Claude Code y dominar las operaciones comunes en el desarrollo diario.

## Preparación

Antes de comenzar, asegúrate de tener:

- Una terminal o símbolo del sistema abierto
- Un proyecto de código con el que trabajar
- Una suscripción de Claude (Pro, Max, Teams o Enterprise), cuenta de Claude Console, o acceso a través de un proveedor de nube soportado

## Iniciar una Sesión

Abre una terminal en el directorio de tu proyecto e inicia Claude Code:

```bash
cd /path/to/your/project
claude
```

En el primer uso, verás una pantalla de bienvenida con información de la sesión, conversaciones recientes y las últimas actualizaciones. Escribe `/help` para ver comandos disponibles o `/resume` para continuar una conversación anterior.

## Operaciones Básicas

### Entender la Base de Código

Pide a Claude que analice tu proyecto:

```
¿Qué hace este proyecto?
```

```
¿Qué tecnologías usa este proyecto?
```

```
¿Dónde está el punto de entrada principal?
```

```
Explica la estructura de carpetas
```

También puedes preguntarle a Claude sobre sus capacidades:

```
¿Qué puede hacer Claude Code?
```

```
¿Cómo creo habilidades personalizadas en Claude Code?
```

**Nota: Claude Code lee los archivos del proyecto según sea necesario. No tienes que agregar contexto manualmente.**

### Tu Primer Cambio de Código

Haz que Claude Code haga un trabajo de codificación real. Prueba una tarea simple:

```
Agrega una función hello world al archivo principal
```

Claude Code:
1. Encontrará el archivo apropiado
2. Te mostrará los cambios propuestos
3. Pedirá tu aprobación
4. Realizará la edición

**Nota: Claude Code siempre pide permiso antes de modificar archivos. Puedes aprobar cambios individuales o habilitar el modo "Aceptar todo" para una sesión.**

### Operaciones Git

Claude Code hace que las operaciones Git sean conversacionales:

```
¿Qué archivos he cambiado?
```

```
Haz commit de mis cambios con un mensaje descriptivo
```

```
Crea una nueva rama llamada feature/quickstart
```

```
Muéstrame los últimos 5 commits
```

```
Ayúdame a resolver conflictos de fusión
```

### Corregir Errores y Agregar Funciones

Claude es experto en depuración e implementación de funciones.

Describe lo que quieres en lenguaje natural:

```
Agrega validación de entrada al formulario de registro de usuarios
```

O corrige problemas existentes:

```
Hay un error donde los usuarios pueden enviar formularios vacíos - corrígelo
```

Claude Code:
- Localizará el código relevante
- Entenderá el contexto
- Implementará una solución
- Ejecutará pruebas si están disponibles

### Flujos de Trabajo Comunes

**Refactorizar código**

```
Refactoriza el módulo de autenticación para usar async/await en lugar de callbacks
```

**Escribir pruebas**

```
Escribe pruebas unitarias para las funciones de calculadora
```

**Actualizar documentación**

```
Actualiza el README con instrucciones de instalación
```

**Revisión de código**

```
Revisa mis cambios y sugiere mejoras
```

## Comandos Esenciales

Aquí están los comandos más importantes para el uso diario:

| Comando | Función | Ejemplo |
|---------|---------|---------|
| `claude` | Iniciar modo interactivo | `claude` |
| `claude "tarea"` | Ejecutar tarea única | `claude "fix the build error"` |
| `claude -p "consulta"` | Ejecutar consulta única y salir | `claude -p "explain this function"` |
| `claude -c` | Continuar la conversación más reciente en el directorio actual | `claude -c` |
| `claude -r` | Reanudar una conversación anterior | `claude -r` |
| `claude commit` | Crear un commit de Git | `claude commit` |
| `/clear` | Limpiar historial de conversación | `/clear` |
| `/help` | Mostrar comandos disponibles | `/help` |
| `exit` o Ctrl+C | Salir de Claude Code | `exit` |

## Consejos Avanzados

### Usando CLAUDE.md

Crea un archivo `CLAUDE.md` en la raíz de tu proyecto para proporcionar contexto persistente del proyecto a Claude Code. Este archivo se carga automáticamente al inicio de cada sesión.

CLAUDE.md puede contener:
- Estándares y especificaciones de codificación
- Decisiones de arquitectura
- Bibliotecas preferidas
- Listas de verificación de revisión
- Comandos y scripts específicos del proyecto

### Habilidades Personalizadas

Crea comandos personalizados para encapsular flujos de trabajo repetibles, como `/review-pr` o `/deploy-staging`.

### Usando Hooks

Los Hooks te permiten ejecutar comandos shell antes o después de las acciones de Claude Code, como:
- Auto-formatear después de cada edición de archivo
- Ejecutar lint antes de un commit

### Servidores MCP

Conecta Claude Code a herramientas y fuentes de datos externas a través del Model Context Protocol:
- Leer documentos de diseño desde Google Drive
- Actualizar tickets de Jira
- Obtener datos de Slack
- Usar herramientas personalizadas

### Colaboración Multi-Agente

Para tareas complejas, puedes activar múltiples agentes de Claude Code para trabajar en paralelo. Un agente líder coordina el trabajo, asigna subtareas y fusiona resultados.

### Control Remoto

El trabajo no está ligado a un solo dispositivo:
- Usa Control Remoto para continuar una sesión local desde tu teléfono u otro dispositivo
- Inicia una tarea de larga duración en la Web o aplicación iOS, luego tráela de vuelta a tu terminal con `/teleport`
- Usa `/desktop` para cambiar una sesión de terminal a la aplicación de escritorio para revisión visual de diffs

## Mejores Prácticas

### Sé Específico con tus Peticiones

❌ No digas: "corrige el error"

✅ Mejor di: "corrige el error de login donde los usuarios ven una pantalla en blanco después de ingresar credenciales incorrectas"

### Usa Instrucciones Paso a Paso

Divide tareas complejas en pasos:

```
1. Crea una nueva tabla de base de datos para perfiles de usuario
2. Crea un endpoint de API para obtener y actualizar perfiles de usuario
3. Construye una página web que permita a los usuarios ver y editar su información
```

### Deja que Claude Explore Primero

Antes de hacer cambios, deja que Claude entienda tu código:

```
Analiza el esquema de la base de datos
```

```
Construye un panel mostrando productos más frecuentemente devueltos por nuestros clientes del Reino Unido
```

### Ahorra Tiempo con Atajos

- Presiona `?` para ver todos los atajos de teclado
- Usa Tab para autocompletar comandos
- Presiona `↑` para historial de comandos
- Escribe `/` para ver todos los comandos y habilidades

## Preguntas Frecuentes

**¿Claude Code lee mi código?**

Sí, Claude Code lee los archivos del proyecto para entender tu base de código. No enviará tu código a servidores externos (a menos que uses un proveedor de API de terceros).

**¿Claude Code modifica automáticamente mis archivos?**

No. Claude Code siempre pide permiso antes de modificar archivos. Puedes aprobar cambios individuales o habilitar el modo "Aceptar todo" para toda una sesión.

**¿Mi código es seguro?**

Claude Code está diseñado pensando en la seguridad. Tu código no se envía a Anthropic para entrenamiento de modelos (a menos que eligas hacerlo voluntariamente). Obtén más información en la [Política de Privacidad de Anthropic](https://www.anthropic.com/privacy).
