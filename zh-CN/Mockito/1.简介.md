# Mockito 简介

## 什么是 Mockito

Mockito 是一个功能强大的 Java 单元测试模拟（Mocking）框架，专为测试驱动开发（TDD）和行为驱动开发（BDD）而设计。它允许开发者创建模拟对象（Mock 对象），以受控的方式模拟真实对象的行为，从而实现独立的单元测试。

## 为什么选择 Mockito

### 简洁的 API 设计

Mockito 提供了直观易用的 API，学习曲线平缓。开发者可以快速上手，用简洁的代码编写清晰的测试用例。与其他复杂的模拟框架相比，Mockito 的语法更加自然，阅读测试代码就像阅读普通业务代码一样简单。

### 强大的功能支持

Mockito 支持多种高级特性，包括方法调用验证、返回值设定、异常抛出模拟、参数匹配、调用顺序验证等。这些功能能够满足各种复杂的测试场景需求，帮助开发者全面覆盖代码逻辑。

### 优秀的可读性

使用 Mockito 编写的测试代码具有良好的可读性。当测试失败时，Mockito 能够提供清晰的验证错误信息，帮助开发者快速定位问题所在。这种友好的错误提示大大提升了调试效率。

### 活跃的社区支持

Mockito 是 Java 生态中最流行的 Mock 框架，拥有庞大的用户群体和活跃的社区。框架持续更新维护，文档完善丰富，遇到问题能够很容易找到解决方案。Stack Overflow 上 Mockito 获得了最高的投票数，被公认为 Java 最佳 Mock 框架。

## Mockito 核心概念

### Mock 对象

Mock 对象是真实对象的替身，在测试期间用来替代真实对象。通过创建 Mock 对象，可以隔离外部依赖，确保测试只关注被测试单元本身的逻辑，而不受外部系统的影响。

### Stubbing（打桩）

Stubbing 用于设定 Mock 对象在特定方法调用时的返回值或异常行为。通过 when().thenReturn() 语法，可以精确控制模拟对象的方法响应，使测试能够覆盖各种场景。

### Verification（验证）

Verification 用于确认测试代码与 Mock 对象之间的交互是否按预期发生。verify() 方法可以检查方法是否被调用、调用次数、调用参数等，确保被测试代码正确地使用了依赖对象。

### Spy（监视）

Spy 允许对真实对象进行部分模拟。与完全模拟的 Mock 对象不同，Spy 会保留真实对象的大部分行为，只对特定方法进行模拟。这在需要测试大部分真实逻辑同时又要模拟少数方法时非常有用。

## 典型应用场景

### 隔离外部依赖

当被测试代码依赖数据库、网络服务、文件系统等外部系统时，使用 Mockito 可以将这些外部依赖模拟掉，使测试能够在没有真实外部系统的情况下运行。这不仅加快了测试速度，还消除了外部系统不稳定带来的测试失败风险。

### 模拟尚未开发完成的模块

在大型项目中，不同模块的开发进度可能不同。使用 Mockito 可以模拟尚未完成的模块，使已完成的模块能够提前进行测试，避免开发阻塞。

### 模拟复杂对象

某些业务对象的构造非常复杂，包含多个层级的依赖。通过 Mockito，可以轻松创建这些复杂对象的模拟版本，专注于测试业务逻辑本身。

## 与 JUnit 的集成

Mockito 通常与 JUnit 测试框架配合使用。通过 JUnit 的 @RunWith(MockitoJUnitRunner.class) 或 @ExtendWith(MockitoExtension.class)，可以自动初始化 @Mock 注解标记的字段，简化测试代码的编写。这种组合是 Java 单元测试的最佳实践。

## 版本兼容性

Mockito 5.x 版本需要 JDK 11 或更高版本，支持 Java 8 引入了 lambda 表达式等新特性。Mockito 4.x 系列仍然支持 JDK 8，开发者可以根据项目实际情况选择合适的版本。

## 总结

Mockito 是 Java 开发者必备的测试工具。它以简洁优雅的方式解决了单元测试中的依赖隔离问题，让开发者能够编写出可靠、可维护的测试代码。掌握 Mockito 将显著提升代码质量和开发效率，是每个 Java 工程师都应该具备的技能。
