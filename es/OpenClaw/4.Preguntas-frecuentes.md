# Preguntas Frecuentes de OpenClaw

Esta página responde preguntas comunes sobre OpenClaw.

## Conceptos Básicos

### ¿Qué es OpenClaw?

OpenClaw es una puerta de enlace autohospedada que conecta tus aplicaciones de chat (WhatsApp, Telegram, Discord, etc.) con asistentes de codificación de IA. Te permite comunicarte con un asistente de IA desde cualquier lugar mientras mantienes el control total sobre tus datos.

### ¿Es OpenClaw gratuito?

Sí, OpenClaw es un proyecto de código abierto bajo licencia MIT. Sin embargo, necesitarás hacerte cargo de:
- El servidor o computadora para ejecutarlo
- Los costos de la clave API para servicios de IA

### ¿Necesito una suscripción a un servicio de IA?

OpenClaw supports multiple model providers, including:
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- Any other OpenAI API-compatible service

Puedes usar claves API de pago por uso o autenticación de suscripción.

### ¿Qué plataformas soporta OpenClaw?

- **Escritorio**: macOS, Linux, Windows
- **Móvil**: iOS, Android (como nodos)
- **Apps de Chat**: WhatsApp, Telegram, Discord, iMessage, Signal, Slack, and 20+ more platforms

## Instalación

### ¿Qué hago si la instalación falla?

1. Verifica la versión de Node.js: `node --version` (se requiere 22 o superior)
2. Ejecuta diagnósticos: `openclaw doctor`
3. Revisa los registros: `openclaw logs`

### ¿Puedo ejecutarlo en una Raspberry Pi?

Sí, OpenClaw funciona en Raspberry Pi. Recomendamos Raspberry Pi 4 o más reciente, con Raspberry Pi OS o Ubuntu.

### ¿Necesito una Mac?

No, OpenClaw funciona en cualquier sistema que soporte Node.js. Solo el canal de iMessage requiere una Mac (debido a las restricciones de Apple).

## Configuración

### ¿Dónde está el archivo de configuración?

Ubicación predeterminada: `~/.openclaw/openclaw.json`

### ¿Cómo controlo quién puede enviar mensajes?

Configura a través de `dmPolicy`:

- `pairing`: los remitentes desconocidos necesitan un código de emparejamiento
- `allowlist`: solo usuarios en la lista de permitidos
- `open`: permitir todos los mensajes directos
- `disabled`: deshabilitar mensajes directos

### ¿Cómo configuro grupos de chat?

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## Uso

### ¿Cómo agrego un nuevo canal de chat?

```bash
openclaw channels login whatsapp
```

### ¿Puedo usar múltiples canales a la vez?

Sí, un Gateway puede conectarse a múltiples canales simultáneamente.

### ¿Cómo hago que la IA recuerde conversaciones anteriores?

OpenClaw tiene memoria incorporada que guarda automáticamente el contexto de la sesión. Puedes ajustar las políticas de retención de sesiones a través de la configuración.

### ¿Puedo ejecutar múltiples agentes?

Sí, configura múltiples agentes con reglas de enrutamiento:

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## Seguridad

### ¿Mis datos están seguros?

Sí. OpenClaw es autohospedado: todos los datos permanecen en tu dispositivo y no se envían a servidores de terceros (excepto a las APIs de proveedores de IA).

### ¿Cómo protejo mi Gateway?

1. Configura `allowFrom` para restringir fuentes de acceso
2. Usa contraseñas fuertes y claves API
3. Mantén OpenClaw actualizado
4. Usa firewall para restringir el acceso a puertos

## Solución de Problemas

### El Gateway no inicia

1. Verifica si el puerto está en uso: `openclaw gateway status`
2. Revisa los registros de error: `openclaw logs`
3. Ejecuta diagnósticos: `openclaw doctor`

### La IA no responde

1. Verifica que la clave API esté configurada correctamente
2. Confirma que tu cuenta de proveedor de modelo tenga saldo suficiente
3. Busca mensajes de error en los registros

### El envío de mensajes falla

1. Verifica si el canal ha iniciado sesión
2. Confirma que el usuario o grupo objetivo exista
3. Revisa los registros de error detallados
