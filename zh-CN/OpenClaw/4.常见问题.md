# OpenClaw 常见问题

本页面汇总了使用 OpenClaw 过程中的常见问题及解决方案。

## 安装相关

### OpenClaw 需要什么环境？

- Node.js 22.x 或更高版本
- macOS、Linux 或 Windows (WSL2)
- 推荐至少 2GB 内存

### 安装失败怎么办？

1. 确认 Node.js 版本：`node --version`（需 >= 22）
2. 检查网络连接
3. 尝试使用不同的包管理器：npm、pnpm 或 bun
4. 运行 `openclaw doctor` 诊断问题

### 如何更新 OpenClaw？

```bash
# 切换到稳定版
openclaw update --channel stable

# 然后重启 Gateway
openclaw gateway restart
```

## Gateway 相关

### Gateway 无法启动

- 检查端口 18789 是否被占用
- 查看日志：`openclaw gateway --verbose`
- 尝试杀死可能存在的进程

### 如何让 Gateway 始终运行？

使用 `--install-daemon` 安装为系统服务：

```bash
openclaw onboard --install-daemon
```

### 如何远程访问 Gateway？

1. **Tailscale**：使用 `gateway.tailscale.mode: "serve"` 或 `"funnel"`
2. **SSH 隧道**：建立 SSH 隧道连接

## 渠道相关

### WhatsApp 登录失败

- 确认已使用 `openclaw channels login` 配对设备
- 检查网络是否可以访问 WhatsApp 服务器

### Telegram Bot 无法使用

- 确认 Bot Token 正确
- 检查是否已与 Bot 开始对话
- 验证 `allowFrom` 配置

### Discord 消息没有反应

- 检查 Bot 是否已加入服务器
- 确认已配置正确的权限
- 查看频道是否在允许列表中

## 安全相关

### 陌生人可以给我发消息？

默认情况下，OpenClaw 会对未知发送者进行配对验证。使用 `dmPolicy: "open"` 可改为开放模式，但不推荐。

### 如何管理配对？

```bash
# 查看配对列表
openclaw pairing list

# 批准配对
openclaw pairing approve

# 撤销配对
openclaw pairing revoke
```

### 运行 `openclaw doctor` 显示警告？

按照指示修复配置。常见问题包括：
- DM 政策过于开放
- 缺少必要的认证配置
- 端口暴露风险

## 技能相关

### 如何安装新技能？

```bash
# 通过 ClawHub 安装
openclaw skill install 技能名

# 或手动创建技能目录和 SKILL.md
```

### 技能不工作怎么办？

- 检查技能配置是否正确
- 查看 Gateway 日志
- 确认技能依赖已安装

## 语音相关

### Voice Wake 不工作？

- macOS：检查系统偏好设置中的辅助功能权限
- iOS：确认麦克风权限已开启
- 检查唤醒词是否正确设置

### 语音响应太慢？

- 检查网络延迟
- 考虑使用本地 TTS
- 调整语音模型设置

## 性能相关

### OpenClaw 占用内存太多？

- 定期清理会话历史
- 调整模型设置
- 使用会话压缩功能

### 消息响应慢？

- 检查网络连接
- 尝试切换到更快的模型
- 查看 Gateway 负载

## 数据相关

### 数据存储在哪里？

- 配置：`~/.openclaw/`
- 工作区：`~/.openclaw/workspace`
- 凭证：`~/.openclaw/credentials`

### 如何备份数据？

```bash
# 备份整个配置目录
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### 如何重置所有设置？

1. 停止 Gateway
2. 删除 `~/.openclaw` 目录
3. 重新运行 `openclaw onboard`

## 其他问题

### 如何查看日志？

```bash
# 查看 Gateway 日志
openclaw gateway --verbose

# 或查看特定日志文件
openclaw logs
```

### 如何获取帮助？

- 访问文档：https://docs.openclaw.ai
- 加入 Discord 社区
- 在 GitHub 上提交 Issue

### OpenClaw 与其他 AI 助手有什么区别？

- **完全本地**：您的数据存储在本地，不上传云端
- **开源透明**：代码完全开放，可自行审查
- **高度可定制**：可以修改和扩展任何功能
- **多渠道统一**：一个助手管理所有通讯渠道

### 我可以同时运行多个 OpenClaw 实例吗？

可以，每个实例需要：
- 不同的端口
- 独立的工作区
- 不同的配置
