# Installationsanleitung für Node.js

## Offizielle Installationsmethoden

Node.js bietet mehrere offizielle Installationsmethoden, um verschiedenen Betriebssystemen und Benutzerpräferenzen gerecht zu werden. Alle offiziellen Methoden stellen sicher, dass Sie die authentischen, unveränderten Binärdateien direkt vom Node.js-Projekt erhalten.

### Herunterladen von der offiziellen Website

Der einfachste Weg, Node.js zu installieren, ist das Herunterladen des entsprechenden Installers von der [offiziellen Node.js-Website](https://nodejs.org/en/download).

**Verfügbare Versionen:**
- **LTS (Langfristiger Support)**: Empfohlen für die meisten Benutzer. Bietet Stabilität und langfristigen Support.
- **Current**: Neueste Funktionen mit kürzerem Support-Zyklus. Geeignet für Entwicklung und Tests.

### Paketmanager

#### macOS (Homebrew)
```bash
# Homebrew installieren, falls noch nicht vorhanden
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# LTS-Version installieren
brew install node@lts

# Oder die neueste Current-Version installieren
brew install node
```

#### Linux (Verschiedene Distributionen)

**Ubuntu/Debian:**
```bash
# Verwendung des NodeSource-Repositorys (empfohlen)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora:**
```bash
# Verwendung des NodeSource-Repositorys
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# oder für neuere Versionen: sudo dnf install nodejs
```

#### Windows
- Laden Sie den Windows-Installer (.msi) von der offiziellen Website herunter
- Führen Sie den Installer aus und folgen Sie dem Einrichtungsassistenten
- Node.js und npm werden automatisch Ihrem PATH hinzugefügt

### Überprüfung

Nach der Installation überprüfen Sie, ob Node.js und npm korrekt installiert sind:

```bash
# Node.js-Version überprüfen
node --version

# npm-Version überprüfen  
npm --version

# Grundlegende Funktionalität testen
node -e "console.log('Hallo Node.js!')"
```

### Verwaltung mehrerer Node.js-Versionen

Für Entwickler, die in verschiedenen Projekten unterschiedliche Node.js-Versionen verwenden müssen, bieten Versionsmanager eine hervorragende Lösung.

#### nvm (Node Version Manager) - macOS/Linux
```bash
# nvm installieren
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# LTS-Version installieren und verwenden
nvm install --lts
nvm use --lts

# Bestimmte Version installieren
nvm install 20.14.0
nvm use 20.14.0

# Verfügbare Versionen auflisten
nvm list-remote
```

#### nvm-windows - Windows
- Laden Sie von [nvm-windows GitHub Releases](https://github.com/coreybutler/nvm-windows/releases) herunter
- Folgen Sie den Installationsanweisungen
- Verwenden Sie ähnliche Befehle wie Unix-basiertes nvm

### Umgebungsvariablen

Stellen Sie sicher, dass die ausführbaren Dateien von Node.js und npm zum System-PATH-Umgebungsvariablen hinzugefügt wurden.

- **Windows**: Wird normalerweise automatisch zum PATH hinzugefügt
- **macOS/Linux**: Wenn Sie einen Paketmanager verwenden, wird dies normalerweise automatisch konfiguriert

Wenn der Befehl nicht gefunden wird, fügen Sie ihn manuell zum PATH hinzu:
```bash
# Normalerweise unter /usr/local/bin oder ~/.nvm/versions/node/vXX.XX.XX/bin
export PATH=$PATH:/usr/local/bin
```

### Fehlerbehebung

#### Berechtigungsprobleme
Unter Linux/macOS, wenn Sie bei der globalen Installation von npm-Paketen auf Berechtigungsprobleme stoßen, empfehlen wir:
- Verwenden Sie nvm zur Verwaltung von Node.js
- Oder konfigurieren Sie das globale Verzeichnis von npm:
  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  export PATH=~/.npm-global/bin:$PATH
  ```

#### Netzwerkprobleme
Wenn die Download-Geschwindigkeit langsam ist oder keine Verbindung möglich ist, können Sie npm so konfigurieren, dass ein lokaler Mirror verwendet wird:
```bash
# Taobao-Mirror einstellen (China)
npm config set registry https://registry.npmmirror.com

# Oder temporär verwenden
npm install --registry https://registry.npmmirror.com
```
