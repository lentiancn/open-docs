# JDK Introduction

## Qu'est-ce que JDK

JDK (Java Development Kit) est le kit de développement logiciel pour le langage de programmation Java et constitue le cœur de tout développement Java. Il comprend l'environnement d'exécution Java (JRE), le compilateur Java (javac), la machine virtuelle Java (JVM) ainsi que de vastes bibliothèques de classes et outils Java. JDK est essentiel pour le développement d'applications Java, qu'il s'agisse d'applications de bureau, d'applications Web, d'applications mobiles ou de systèmes d'entreprise.

JDK est développé et maintenu par Oracle Corporation (et son prédécesseur Sun Microsystems), servant de base à la plateforme Java. Les dernières versions LTS (support à long terme) sont JDK 25 et JDK 21, JDK 17 étant également une version LTS largement utilisée. Oracle publie régulièrement de nouvelles versions de JDK, chacune apportant des améliorations de performances, de nouvelles fonctionnalités et des corrections de vulnérabilités de sécurité.

JDK est plus qu'un simple environnement d'exécution ; il fournit tous les composants nécessaires au développement Java complet. Le composant le plus fondamental est le compilateur Java (javac), qui compile le code source Java en bytecode (fichiers .class). Ensuite, il y a la machine virtuelle Java (JVM), qui exécute le bytecode, ainsi que divers outils comme jar (outil d'archivage), javadoc (générateur de documentation) et javap (désassembleur). Ensemble, ces outils constituent un environnement de développement Java complet.

## Versions principales de JDK

### JDK 25 (Nouvelle version)

JDK 25 est la dernière version de JDK, apportant de nombreuses nouvelles fonctionnalités et améliorations. Il s'agit d'une version à support à court terme avec un cycle de support de 6 mois. Les fonctionnalités principales incluent : optimisations de performances améliorées, collecteuses d'ordures améliorées, nouvelles fonctionnalités de langage et support du matériel le plus récent. JDK 25 convient aux développeurs qui souhaitent utiliser les dernières fonctionnalités Java, mais attention à son cycle de support plus court.

### JDK 21 (LTS)

JDK 21 est la dernière version de support à long terme (LTS) et recevra au moins 8 ans de support. Il s'agit de la dernière version LTS avec utilisation gratuite d'Oracle JDK en vertu des Conditions d'utilisation gratuites d'Oracle. JDK 21 introduit des fonctionnalités importantes comme les threads virtuels (Virtual Threads), la correspondance de motifs (Pattern Matching) et les records (Records), qui simplifient considérablement la programmation Java et améliorent l'efficacité du développement.

Les threads virtuels sont l'une des fonctionnalités les plus importantes de JDK 21, rendant la rédaction d'applications hautement concurrentes beaucoup plus simple. Par rapport aux threads traditionnels, les threads virtuels ont des coûts de création extrêmement bas et peuvent créer facilement des millions de threads virtuels pour gérer les tâches concurrentes. Ceci est significatif pour la construction de systèmes distribués à grande échelle, d'architectures de microservices et d'applications serveur haute performance.

### JDK 17 (LTS)

JDK 17 est l'une des versions LTS les plus matures actuellement disponibles et a une adoption généralisée. C'est la dernière version avec utilisation gratuite d'Oracle JDK (selon les termes de licence d'Oracle), donc de nombreuses entreprises utilisent encore JDK 17. JDK 17 contient de nombreuses fonctionnalités d'OpenJDK, comme les classes scellées (Sealed Classes), les blocs de texte (Text Blocks) et les expressions switch.

Les performances de JDK 17 se sont considérablement améliorées par rapport à JDK 11, particulièrement dans la collecte d'ordures et la compilation JIT. Pour les applications d'entreprise nécessitant un support stable à long terme, JDK 17 reste un excellent choix. Il atteint un bon équilibre entre performances, stabilité et fonctionnalités.

### JDK 11 (LTS)

JDK 11 est une autre version LTS importante, marquant la ligne de démarcation des fonctionnalités payantes de Java. À partir de JDK 11, certaines fonctionnalités avancées d'Oracle JDK nécessitent un abonnement payant. JDK 11 introduit le nettoyeur de mémoire ZGC (expérimental), la nouvelle API client HTTP et l'inférence de type de variable locale (mot-clé var). Bien que ce ne soit plus la dernière version LTS, JDK 11 est encore largement utilisé dans de nombreux projets existants.

## Composants principaux de JDK

### Compilateur Java (javac)

Le compilateur Java (javac) est l'un des outils les plus essentiels de JDK, responsable de la compilation du code source Java (fichiers .java) en bytecode Java (fichiers .class). Le bytecode est une forme intermédiaire de code qui ne dépend pas de plates-formes matérielles spécifiques et ne peut être exécuté que par la machine virtuelle Java. Cette conception donne aux programmes Java la caractéristique « Écrire une fois, exécuter partout ».

Le compilateur javac prend en charge de nombreuses options de compilation, permettant aux développeurs de contrôler divers comportements du processus de compilation via des paramètres de ligne de commande. Par exemple, vous pouvez utiliser l'option -d pour spécifier le répertoire de sortie de compilation, -classpath pour spécifier le chemin de recherche de classes, et -source et -target pour spécifier les versions Java du code source et du bytecode cible. La maîtrise de ces options de compilation est cruciale pour la construction de grands projets.

### Machine virtuelle Java (JVM)

La machine virtuelle Java (JVM) est l'environnement central d'exécution des programmes Java. Elle est responsable du chargement du bytecode et de son interprétation ou compilation pour l'exécution native. La JVM est la clé de l'indépendance de la plateforme Java, masquant les différences entre les systèmes d'exploitation et le matériel sous-jacents, permettant au même programme Java de s'exécuter sur différentes plates-formes.

Les composants principaux de la JVM incluent : le chargeur de classes responsable du chargement des fichiers de classe, le moteur d'exécution responsable de l'exécution du bytecode, et les zones de données d'exécution incluant la zone de méthodes, le tas, la pile de machines virtuelles, la pile de méthodes natives et le compteur de programmes. Comprendre le fonctionnement de la JVM est essentiel pour écrire des applications Java haute performance.

La JVM fournit également une puissante fonctionnalité de collecte automatique d'ordures (GC), gérant automatiquement l'allocation et la récupération de mémoire. Différentes collecteuses d'ordures comme Serial, Parallel, CMS, G1, ZGC et Shenandoah conviennent à différents scénarios d'application. Les développeurs peuvent choisir la collecteuse d'ordures appropriée selon les caractéristiques de l'application pour obtenir des performances optimales.

### Environnement d'exécution Java (JRE)

L'environnement d'exécution Java (JRE) est l'environnement requis pour exécuter des programmes Java. Il comprend la JVM, les bibliothèques de classes centrales Java et d'autres fichiers de support. Le JRE est un sous-ensemble de JDK ; JDK comprend JRE plus les outils de développement. Si vous avez seulement besoin d'exécuter des programmes Java sans développement, l'installation du JRE suffit.

Le cœur du JRE est les bibliothèques de classes centrales Java, qui fournissent la fonctionnalité de base du langage Java comme java.lang, java.util, java.io, java.net. Ces bibliothèques de classes ont été développées et optimisées au fil des ans et sont très matures et stables, couvrant tout, des structures de données de base à la programmation réseau.

### Ensemble d'outils Java

JDK fournit un riche ensemble d'outils en ligne de commande pour le développement, le débogage et le diagnostic de programmes Java. Les outils principaux incluent :

jar : Utilisé pour créer et gérer des fichiers d'archivage Java (fichiers .jar), qui peuvent emballer plusieurs fichiers de classe dans un fichier pratique pour la distribution et le déploiement.

javadoc : Utilisé pour générer de la documentation d'API au format HTML à partir de commentaires dans le code source Java, l'outil standard pour créer de la documentation de code.

javap : Utilisé pour désassembler le bytecode Java, peut voir les structures de fichiers de classe et les signatures de méthodes, très utile pour comprendre le processus de compilation Java et le débogage.

jdb : Débogueur Java, fournit des fonctionnalités de débogage en interface de ligne de commande, peut définir des points d'arrêt, exécuter pas à pas, voir les valeurs de variables, etc.

jps : Affiche des informations sur tous les processus Java dans le système actuel, y compris l'ID de processus et le nom de la classe principale.

jstat : Affiche les statistiques de la JVM comme les informations de collecte d'ordures, les informations de chargement de classes, etc.

jstack : Imprime les informations de trace de pile des threads Java, utilisé pour diagnostiquer les problèmes de blocage de threads.

jmap : Affiche les informations de mémoire du tas Java, peut générer des fichiers de vidage de tas pour analyse.

Ces outils sont très utiles dans le développement quotidien et le diagnostic de problèmes en production, les maîtriser peut améliorer considérablement l'efficacité du développement et les capacités de diagnostic.

## Domaines d'application de JDK

### Développement d'applications d'entreprise

JDK est la base du développement d'applications d'entreprise. Divers frameworks Java EE (maintenant renommés Jakarta EE) comme Spring, Hibernate et Struts nécessitent de s'exécuter sur JDK. Ces frameworks fournissent des solutions complètes pour les applications d'entreprise, incluant l'injection de dépendances, la gestion des transactions, la persistance des données et les services Web.

Les grandes applications d'entreprise ont généralement des exigences très élevées en matière de stabilité et de performances, donc choisir la version de JDK appropriée est très important. De nombreuses entreprises utilisent des versions LTS matures comme JDK 17 combinées avec des services de support commercial pour assurer le fonctionnement stable à long terme du système.

### Développement Android

Bien que les applications Android soient principalement écrites en Java ou Kotlin, le développement d'applications Android ne peut pas se faire sans JDK. Les outils du SDK Android ont besoin de JDK pour compiler des projets Android et générer des fichiers APK pouvant être installés sur les appareils Android. Android Studio inclut également JDK intégré, et les développeurs doivent spécifier le chemin JDK lors de la configuration de l'environnement de développement.

### Traitement Big Data

Dans le domaine du big data, Java et JDK jouent un rôle central. Les frameworks big data majeurs comme Hadoop, Spark et Flink sont tous développés en Java et nécessitent de s'exécuter sur JDK. Ces frameworks doivent généralement traiter des quantités massives de données et ont des exigences très élevées en matière de performances et de stabilité.

### Développement d'applications Web

Le développement d'applications Web Java est une force traditionnelle de JDK. Les technologies comme Servlet, JSP, Filter et Listener constituent la base du développement Web Java. Les frameworks Web majeurs comme Spring MVC, Struts et JSF s'exécutent tous sur JDK.

### Cloud computing et microservices

Avec la popularité du cloud computing et de l'architecture microservices, JDK joue également un rôle important dans ce domaine. Les frameworks microservices comme Spring Cloud et Dubbo fournissent des solutions complètes pour construire des systèmes distribués. Tous ces frameworks doivent s'exécuter sur JDK.

## JDK et autres implémentations Java

### OpenJDK

OpenJDK est l'implémentation open source de Java SE, développée et maintenue conjointement par Oracle et la communauté. OpenJDK est la base de la plupart des autres distributions JDK. De nombreux fournisseurs comme Eclipse (Eclipse Temurin), Amazon (Amazon Corretto) et Microsoft (Microsoft Build of OpenJDK) construisent leurs propres distributions JDK basées sur OpenJDK.

OpenJDK et Oracle JDK sont presque identiques au niveau du code, les principales différences étant les accords de licence et les services de support. Oracle JDK fournit un support commercial et des fonctionnalités de niveau entreprise supplémentaires, tandis qu'OpenJDK est entièrement open source et gratuit. Pour la plupart des utilisateurs, utiliser OpenJDK est parfaitement adéquat.

### Autres distributions JDK

En plus d'Oracle JDK et OpenJDK, il existe de nombreuses autres distributions JDK :

Eclipse Temurin (anciennement AdoptOpenJDK) : Distribution OpenJDK maintenue par la Eclipse Foundation, fournissant des builds de haute qualité et un support à long terme.

Amazon Corretto : Distribution JDK gratuite et multiplateforme fournie par Amazon, avec support à long terme et optimisation des performances.

Microsoft Build of OpenJDK : Build OpenJDK fourni par Microsoft, optimisé pour Windows et Azure.

Azul Zulu : Distribution JDK fournie par Azul Systems, connue pour ses performances élevées et sa stabilité.

GraalVM : Machine virtuelle multi-langages fournie par Oracle, supportant non seulement Java mais aussi JavaScript, Python, Ruby et d'autres langages.

## Importance de JDK

En tant que base du développement Java, l'importance de JDK va de soi. Quel que soit le framework ou les outils Java utilisés, le support de JDK est essentiel. JDK n'est pas seulement un ensemble d'outils pour compiler et exécuter des programmes Java ; il représente également le cœur de l'écosystème Java.

Choisir une version de JDK appropriée et la configurer correctement est crucial pour le succès du projet. Les facteurs à considérer incluent : la stabilité de la version, la fréquence des mises à jour de sécurité, les nouvelles fonctionnalités, l'optimisation des performances et le support communautaire. Pour les applications d'entreprise, il est recommandé d'utiliser des versions LTS pour obtenir un support à long terme.

Avec le développement et l'évolution continus du langage Java, JDK continue d'être mis à jour. Les nouvelles versions de JDK apportent de meilleures performances, plus de fonctionnalités et une sécurité accrue. Garder JDK à jour peut offrir une meilleure expérience de développement et une efficacité opérationnelle.

## Différences entre versions

Les différentes versions principales de JDK ont des différences significatives en matière d'installation, d'utilisation et de fonctionnalités :

**JDK 8** : Dernière version d'Oracle JDK entièrement gratuite pour utilisation commerciale. A introduit les expressions lambda, l'API streams et la nouvelle API date/heure. Encore largement utilisé dans les systèmes hérités.

**JDK 11** : A marqué le début des fonctionnalités payantes de Java. A introduit l'inférence de type de variable locale (var), le nouveau client HTTP et ZGC (expérimental). Dernière version supportant les systèmes 32 bits.

**JDK 17** : Dernière version Oracle JDK gratuite sous NFTC. A introduit les classes scellées, les blocs de texte et les expressions switch. Atteint un excellent équilibre entre performances, stabilité et fonctionnalités.

**JDK 21** : Dernière version LTS, avec les threads virtuels, la correspondance de motifs pour switch et les records comme fonctionnalités stables. Recommandée pour les nouveaux projets.

**JDK 25** : Dernière version avec les dernières fonctionnalités de langage et les améliorations de performances. Conviens aux développeurs voulant les dernières fonctionnalités mais avec un cycle de support plus court.
