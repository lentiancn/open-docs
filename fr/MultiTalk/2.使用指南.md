# Guide d'utilisation de MultiTalk

Ce guide explique comment utiliser MultiTalk pour générer des contenus audio multi-locuteurs.

## Démarrage rapide

### Utilisation de base

```bash
python inference.py \
  --text "Bonjour à tous, bienvenue sur MultiTalk." \
  --speakers speaker1,speaker2 \
  --output result.wav
```

## Exigences d'entrée

### Exigences du texte

- Langue : Support multilingue
- Format : Texte brut
- Longueur : Pas de limite stricte

### Configuration des locuteurs

Définir les locuteurs dans `speakers.yaml`:

```yaml
speakers:
  speaker1:
    voice_id: "voice_001"
    language: "fr"
  speaker2:
    voice_id: "voice_002"
    language: "fr"
```

## Utilisation avancée

### IDs vocaux personnalisés

```bash
python inference.py \
  --text "Ceci est une démonstration vocale personnalisée." \
  --voice_ids voice001,voice002 \
  --output output.wav
```

### Ajuster la vitesse

```bash
python inference.py \
  --text "Exemple de discours lent." \
  --speakers speaker1 \
  --speed 0.8 \
  --output slow.wav
```

### Traitement par lots

```bash
python batch_inference.py \
  --text_file texts.txt \
  --output_dir output/
```

## Configuration

### Fichier de configuration

Modifier `config.yaml`:

```yaml
inference:
  sample_rate: 22050
  speed: 1.0
  
model:
  checkpoint: "checkpoints/multitalk.pth"
```

## Bonnes pratiques

### 1. Utiliser des IDs vocaux appropriés

Choisissez des IDs vocaux correspondant aux caractéristiques du locuteur.

### 2. Correspondance des langues

Assurez-vous que la langue du texte correspond à la langue vocale.

### 3. Optimiser la vitesse

Ajustez la vitesse pour une sortie naturelle.

## Dépannage

### Mauvaise qualité audio ?

Essayez différents IDs vocaux.

### Problèmes de vitesse ?

Ajustez progressivement le paramètre de vitesse.
