# Nacos FAQ

## Was ist Nacos?

Nacos (Dynamic Naming and Configuration Service) ist eine Open-Source-Plattform von Alibaba für Service-Discovery und dynamisches Konfigurationsmanagement.

## Wie installiere ich Nacos?

**Standalone-Modus:**
```bash
wget https://github.com/alibaba/nacos/releases/download/v2.3.0/nacos-server-2.3.0.tar.gz
tar -xvf nacos-server-2.3.0.tar.gz
cd nacos/bin
./startup.sh -m standalone
```

**Docker-Modus:**
```bash
docker run --name nacos -d -p 8848:8848 nacos/nacos-server:v2.3.0
```

## Wie greife ich auf die Nacos-Konsole zu?

- URL: http://localhost:8848/nacos
- Standard-Benutzername: nacos
- Standard-Passwort: nacos

## Wie registriere ich einen Service?

```java
NamingService namingService = NacosFactory.createNamingService("127.0.0.1:8848");
namingService.registerInstance("mein-service", "192.168.1.10", 8080);
```

## Wie erstelle ich eine Konfiguration?

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v1/cs/configs' \
  -d 'dataId=app-config' \
  -d 'group=DEFAULT_GROUP' \
  -d 'content=schlüssel=wert'
```

## Wie richte ich einen Cluster ein?

1. Bereiten Sie mehrere Nacos-Server vor
2. Konfigurieren Sie `cluster.conf` mit allen Knotenadressen
3. Verwenden Sie MySQL als persistenten Speicher
4. Starten Sie alle Knoten im Cluster-Modus
