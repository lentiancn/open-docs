# FAQ OpenClaw

Cette page répond aux questions fréquentes sur OpenClaw.

## Bases

### Qu'est-ce qu'OpenClaw?

OpenClaw est une passerelle auto-hébergée qui connecte vos applications de chat (WhatsApp, Telegram, Discord, etc.) à des assistants de codage IA. Il vous permet de communiquer avec un assistant IA de n'importe où tout en gardant le contrôle total sur vos données.

### OpenClaw est-il gratuit?

Oui, OpenClaw est un projet open source sous licence MIT. Cependant, vous devrez assumer:
- Le serveur ou ordinateur pour l'exécuter
- Les coûts de clé API pour les services IA

### Ai-je besoin d'un abonnement à un service IA?

OpenClaw prend en charge plusieurs fournisseurs de modèles, y compris:
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- Tout autre service compatible API OpenAI

Vous pouvez utiliser des clés API payantes ou l'authentification par abonnement.

### Quelles plateformes OpenClaw prend-il en charge?

- **Bureau**: macOS, Linux, Windows
- **Mobile**: iOS, Android (en tant que nœuds)
- **Apps de Chat**: WhatsApp, Telegram, Discord, iMessage, Signal, Slack et plus de 20 plateformes

## Installation

### Que faire si l'installation échoue?

1. Vérifiez la version Node.js: `node --version` (22 ou supérieur requis)
2. Exécutez les diagnostics: `openclaw doctor`
3. Vérifiez les journaux: `openclaw logs`

### Puis-je l'exécuter sur un Raspberry Pi?

Oui, OpenClaw fonctionne sur Raspberry Pi. Nous recommandons Raspberry Pi 4 ou plus récent, avec Raspberry Pi OS ou Ubuntu.

### Ai-je besoin d'un Mac?

Non, OpenClaw s'exécute sur tout système supportant Node.js. Seul le canal iMessage nécessite un Mac (en raison des restrictions Apple).

## Configuration

### Où est le fichier de configuration?

Emplacement par défaut: `~/.openclaw/openclaw.json`

### Comment contrôler qui peut envoyer des messages?

Configurez via `dmPolicy`:

- `pairing`: les expéditeurs inconnus ont besoin d'un code de couplage
- `allowlist`: seuls les utilisateurs de la liste autorisée
- `open`: autoriser tous les messages directs
- `disabled`: désactiver les messages directs

### Comment configurer les groupes de chat?

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## Utilisation

### Comment ajouter un nouveau canal de chat?

```bash
openclaw channels login whatsapp
```

### Puis-je utiliser plusieurs canaux à la fois?

Oui, un Gateway peut se connecter à plusieurs canaux simultanément.

### Comment faire en sorte que l'IA se souvienne des conversations précédentes?

OpenClaw a une mémoire intégrée qui sauvegarde automatiquement le contexte de session. Vous pouvez ajuster les politiques de conservation des sessions via la configuration.

### Puis-je exécuter plusieurs agents?

Oui, configurez plusieurs agents avec des règles de routage:

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## Sécurité

### Mes données sont-elles sécurisées?

Oui. OpenClaw est auto-hébergé — toutes les données restent sur votre appareil et ne sont pas envoyées à des serveurs tiers (sauf aux API des fournisseurs IA).

### Comment protéger mon Gateway?

1. Définissez `allowFrom` pour restreindre les sources d'accès
2. Utilisez des mots de passe forts et des clés API
3. Gardez OpenClaw à jour
4. Utilisez un pare-feu pour restreindre l'accès aux ports

## Dépannage

### Le Gateway ne démarre pas

1. Vérifiez si le port est utilisé: `openclaw gateway status`
2. Vérifiez les journaux d'erreurs: `openclaw logs`
3. Exécutez les diagnostics: `openclaw doctor`

### L'IA ne répond pas

1. Vérifiez que la clé API est correctement configurée
2. Confirmez que votre compte fournisseur de modèle a un solde suffisant
3. Cherchez les messages d'erreur dans les journaux

### L'envoi de messages échoue

1. Vérifiez si le canal est connecté
2. Confirmez que l'utilisateur ou le groupe cible existe
3. Vérifiez les journaux d'erreurs détaillés
