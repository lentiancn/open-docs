# Preguntas Frecuentes

## Preguntas Básicas

### ¿Qué es Jenkins?

Jenkins es un servidor de automatización de código abierto utilizado principalmente para Integración Continua (CI) y Entrega Continua (CD). Ayuda a los equipos de desarrollo a automatizar el proceso de construcción, prueba y despliegue de software, mejorando la eficiencia y calidad del desarrollo.

### ¿Es Jenkins gratuito?

Sí, Jenkins es completamente gratuito y de código abierto. Utiliza la licencia MIT, permitiendo que cualquiera lo use, modifique y distribuya libremente.

### ¿Qué entorno de ejecución necesita Jenkins?

Jenkins requiere Java 8 o superior. Puede ejecutarse en múltiples sistemas operativos incluyendo Windows, Linux y macOS. Se recomienda un mínimo de 1GB de memoria y 10GB de espacio en disco.

### ¿Cuál es la diferencia entre Jenkins y Hudson?

Jenkins es un fork de Hudson. En 2011, después de una disputa de marca con Oracle, el equipo principal de desarrollo de Hudson hizo un fork y creó Jenkins. Ahora Jenkins es el proyecto activo mientras que Hudson ha sido descontinuado.

## Preguntas de Instalación

### No puedo acceder después del primer inicio. ¿Qué debo hacer?

1. Verificar si Jenkins comenzó correctamente viendo los registros
2. Confirmar que el puerto no esté en uso (puerto predeterminado 8080)
3. Revisar la configuración del firewall para asegurar acceso al puerto
4. Ver mensajes de error en los registros de inicio

### ¿Dónde está la contraseña de desbloqueo?

En la primera instalación, la contraseña de desbloqueo se muestra en los registros de la consola de Jenkins. Si ejecuta vía Docker, use `docker logs jenkins-blueocean` para ver. El archivo de contraseña está generalmente en `/var/jenkins_home/secrets/initialAdminPassword`.

### La instalación de plugins falló. ¿Qué debo hacer?

1. Verificar conexión de red: Jenkins necesita acceso al centro de actualizaciones
2. Instalar manualmente: Intentar subir archivo hpi del plugin desde la pestaña Avanzado
3. Usar espejo de Jenkins: Configurar fuente de actualizaciones nacional
4. Ver registros: Entender la razón específica del fallo

### ¿Cómo cambiar el puerto de Jenkins?

- Método Docker: Modificar parámetro -p
- Método WAR: Usar `java -jar jenkins.war --httpPort=9090`
- Paquete Linux: Modificar HTTP_PORT en /etc/default/jenkins

## Preguntas de Credenciales

### Las credenciales no funcionan. ¿Qué debo hacer?

1. Confirmar que el ID de credencial sea correcto
2. Verificar que el tipo de credencial coincida
3. Validar que las credenciales no hayan expirado
4. Revisar permisos de usuario

### ¿Cómo almacenar contraseñas de forma segura?

Usar la función de almacenamiento de credenciales de Jenkins:

1. Ir a Manage Credentials
2. Agregar credenciales, elegir tipo
3. Jenkins cifra el almacenamiento, no muestra texto plano

### La autenticación con clave SSH falló. ¿Qué debo hacer?

1. Confirmar que la clave pública fue agregada al servidor objetivo
2. Revisar permisos de clave: el archivo de clave privada debe ser 600
3. Validar que el formato de clave sea correcto
4. Intentar regenerar el par de claves

## Preguntas de Pipeline

### ¿Cómo solucionar errores de sintaxis de Jenkinsfile?

1. Usar validador de Pipeline Declarativo
2. Revisar mensajes de error en la consola de Jenkins
3. Usar simulador de Jenkins localmente
4. Referirse a la documentación oficial de sintaxis

### La ejecución del pipeline falló. ¿Qué debo hacer?

1. Ver registros de construcción para información detallada de errores
2. Revisar estado de retorno de cada etapa
3. Confirmar que las herramientas requeridas estén disponibles
4. Validar variables de entorno y credenciales

### ¿Cómo implementar ejecución paralela?

Pipeline Declarativo soporta sintaxis paralela:

```groovy
pipeline {
    agent any
    stages {
        stage('Construcción Paralela') {
            parallel {
                stage('Módulo A') {
                    steps {
                        sh './build-a.sh'
                    }
                }
                stage('Módulo B') {
                    steps {
                        sh './build-b.sh'
                    }
                }
            }
        }
    }
}
```

### ¿Cómo esperar confirmación manual?

Usar paso input:

```groovy
stage('Desplegar a Producción') {
    steps {
        input message: 'Confirmar despliegue a producción?', ok: 'Confirmar'
    }
}
```

## Preguntas de Construcción

### La construcción es muy lenta. ¿Qué debo hacer?

1. Usar construcciones incrementales
2. Configurar caché de construcción
3. Usar etapas paralelas
4. Agregar más nodos de construcción
5. Optimizar descargas de dependencias

### Las descargas de dependencias de Maven son lentas. ¿Qué debo hacer?

1. Configurar espejo de Maven
2. Usar administrador de repositorio privado
3. Configurar y persistir ruta de repositorio local

### ¿Cómo configurar tiempo de espera de construcción?

```groovy
pipeline {
    agent any
    stages {
        stage('Prueba de Tiempo') {
            steps {
                timeout(time: 30, unit: 'MINUTES') {
                    // pasos de construcción
                }
            }
        }
    }
}
```

### ¿Cómo mantener artefactos de construcción?

Usar paso archive:

```groovy
post {
    always {
        archiveArtifacts artifacts: 'target/*.jar', fingerprint: true
    }
}
```

## Preguntas de Construcción Distribuida

### La conexión del nodo falló. ¿Qué debo hacer?

1. Verificar conectividad de red
2. Confirmar compatibilidad de versión de Java
3. Validar configuración de clave del nodo
4. Ver registros del nodo

### ¿Cómo configurar nodos Docker?

1. Instalar plugin de Docker
2. Configurar nube Docker
3. Especificar agente docker en pipeline

### ¿Cómo configurar balance de carga?

Configurar estrategia de selección de agente en pipeline:

```groovy
pipeline {
    agent none
    stages {
        stage('Build') {
            agent { label 'docker' }
            steps {
                // construir
            }
        }
    }
}
```

## Backup y Recuperación

### ¿Cómo hacer backup de Jenkins?

Necesita hacer backup de:

1. Directorio JENKINS_HOME (contiene todas las configuraciones y registros de construcción)
2. Directorio de plugins
3. Archivos de configuración de entorno

Se recomienda usar el plugin Configuration as Code para gestionar configuraciones.

### ¿Cómo migrar Jenkins?

1. Detener Jenkins origen
2. Copiar JENKINS_HOME al nuevo servidor
3. Instalar la misma versión de Jenkins
4. Asegurar compatibilidad de versión de Java
5. Iniciar nuevo Jenkins

## Preguntas de Seguridad

### ¿Cómo habilitar registro de usuarios?

1. Ir a Configure Global Security
2. Habilitar seguridad (base de datos de usuarios propia de Jenkins)
3. Configurar estrategia de autorización (usuarios logueados pueden hacer cualquier cosa)

### ¿Cómo restringir acceso a proyectos?

Usar estrategia de autorización matricial:

1. Configurar seguridad global
2. Elegir Matrix Authorization Strategy
3. Configurar diferentes permisos para diferentes usuarios

### ¿Cómo manejar vulnerabilidades de seguridad?

1. Actualizar versión del núcleo de Jenkins oportunamente
2. Actualizar plugins regularmente
3. Prestar atención a avisos de seguridad de Jenkins
4. Limitar acceso anónimo

## Preguntas de Rendimiento

### ¿Qué hacer cuando hay poca memoria?

1. Ajustar parámetros JVM: agregar `-Xmx` en script de inicio
2. Reducir número de construcciones concurrentes
3. Limpiar registros de construcción antiguos regularmente
4. Aumentar memoria del sistema

### ¿Los registros de construcción ocupan mucho espacio?

1. Configurar política de retención de construcción
2. Limpiar construcciones antiguas regularmente
3. Usar archivado en lugar de mantener registros completos de construcción

## Preguntas de Integración

### ¿La integración con GitHub falló?

1. Verificar permisos de API de GitHub
2. Validar configuración de webhook
3. Confirmar que las credenciales sean correctas

### ¿La integración con Docker falló?

1. Verificar configuración de API de Docker
2. Confirmar permisos de socket de Docker
3. Validar compatibilidad de versión de Docker

### ¿La integración con Kubernetes falló?

1. Verificar configuración de kubeconfig
2. Validar permisos de API de Kubernetes
3. Confirmar que el contenedor de Jenkins tenga suficientes permisos

## Preguntas de Actualización

### ¿Cómo actualizar Jenkins?

- Método Docker: Actualizar versión de imagen
- Método WAR: Reemplazar archivo WAR
- Método paquete: Usar administrador de paquetes

### ¿Plugins incompatibles después de actualizar?

1. Verificar lista de compatibilidad
2. Actualizar plugins a versiones compatibles
3. Revertir versión de Jenkins si es necesario
