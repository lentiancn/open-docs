# LivePortrait ユーザーマニュアル

このドキュメントでは、基本的な推論、Gradioインターフェース、動物モード、映像編集などのLivePortraitのさまざまな使用方法について詳しく説明します。

## 基本的な推論

### 画像駆動

基本的な使用方法は、靜止画像をソースとして、駆動動画をアニメーションリファレンスとして使用することです：

```bash
python inference.py
```

デフォルトでは、スクリプトはassets/examples/sourceディレクトリのサンプル画像とassets/examples/drivingディレクトリのサンプル駆動動画を使用します。正常に実行されると、結果はanimationsディレクトリに保存され、出力ファイルは「ソースファイル--駆動ファイル_concat.mp4」という名前になります。

出力動画には3つの部分が含まれています：駆動動画、ソース画像/動画、そして生成されたアニメーション結果が順に連結されています。

### カスタム入力

コマンドラインパラメータで独自のソースファイルと駆動ファイルを指定できます：

```bash
# ソースファイルが画像の場合
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# ソースファイルが動画の場合
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

-sパラメータはソース画像またはソース動画のパスを指定し、-dパラメータは駆動動画のパスを指定します。

### その他のオプションの表示

使用可能なすべてのコマンドラインパラメータを表示します：

```bash
python inference.py -h
```

パラメータには、出力パス、画像サイズ、アニメーション速度、連結方法などの詳細な設定が含まれます。

## Gradio Webインターフェースの使い方

コマンドラインに不慣れなユーザーには、LivePortraitはGradioベースのグラフィカルインターフェースを提供しています。

### 人文モードインターフェースの起動

```bash
python app.py
```

起動後、ブラウザで表示されたURLを開きます（通常はhttp://localhost:7860）。インターフェースでソース画像または動画をアップロードし、駆動動画を選択して生成ボタンをクリックするとアニメーション結果が得られます。

--server_portパラメータでポートを指定でき、--shareパラメータで公開リンクを生成でき、--server_nameパラメータでサーバーアドレスを指定できます。

### 動物モードインターフェースの起動

LinuxシステムでNVIDIA GPUを使用する場合、動物モードインターフェースを起動できます：

```bash
python app_animals.py
```

## 駆動動画の仕様

最良の結果を得るために、駆動動画は以下の要件を満たす必要があります：

**アスペクト比**：1:1正方形（512x512または256x256ピクセルなど）にクロップすることをお勧めします。自動クロップ機能（--flag_crop_driving_video）を有効にすることもできます。

**コンテンツ要件**：駆動動画示例動画と同様に頭に焦点を当てるべきです。肩の動きは最小限に抑えるべきです。

**最初のフレーム要件**：駆動動画の最初のフレームは、正面からの顔と中立的な表情であるべきです。これはアニメーション効果が正確であることを確認するための重要です。

### 自動クロップ

駆動動画が上記の要件を満たしていない場合は、自动クロップ功能を使用できます：

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

自动クロップの結果が満足できない場合は、--scale_crop_driving_videoと--vy_ratio_crop_driving_videoパラメータを調整して倍率とオフセットを微調整できます。

## モーション模板の使用

推論を高速化しプライバシーを保護するために、LivePortraitは事前生成されたモーション模板ファイル（.pkl形式）の使用をサポートしています。これらの模板ファイルには驱动動画からの运动情報が記録されており、毎回原始動画を処理せずに再利用できます。

### 使用方法

```bash
# ポートレートアニメーション
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# ポートレート動画編集
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

模板ファイルは推論を高速化するだけでなく、プライバシーを保護します。模板を使用すると、原始驱动動画をアップロードする必要がなくなります。

## ポートレート動画編集（Video-to-Video）

LivePortraitは、既存の動画内の人物的表情と頭部动作を編集・修正することをサポートしています。

### 基本的な使用方法

```bash
# ソースが画像、駆動が動画
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### アニメーションパラメータの調整

**表情強度**：--driving_multiplierパラメータを使用して表情变化の強度を調整します。値が1より大きいと表情变化が強くなり、値が1より小さいと弱くなります。

**連結制御**：--flag_stitchingパラメータを使用して、連結を有効にするかどうかを制御します。連結をオフにすると、顔の領域の提示方法が改变される可能性があります。

## 推論の高速化

### Torch Compile高速化

LivePortraitはPyTorchのtorch.compile機能を使用した推論高速化をサポートしています。最初の推論は最適化プロセスをトリガーし（約1分）、その後の推論速度は20-30%向上します。高速化効果はCUDAバージョンによって異なる場合があります。

```bash
python app.py --flag_do_torch_compile
```

注意：torch.compileはWindowsとmacOSではサポートされていません。

### 速度テスト

プロジェクトは推論速度テストスクリプトを提供します：

```bash
python speed.py
```

テスト結果には各モジュールの処理時間が表示され、ボトルネックを特定するのに役立ちます。

## 動物モード

動物モードは、猫や犬などのペットのポートレートをアニメーション化することをサポートします。

### 使用条件

動物モードは次の環境でテストされています：NVIDIA GPU搭载のLinuxシステム、NVIDIA GPU搭载のWindowsシステム。

### インストール要件

动物モードを使用する前に、インストールガイドの説明に従ってX-Pose依存関係をインストールし、動物モデルをダウンロードする必要があります。

### 動物モード推論の実行

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### 動物モードインターフェース

```bash
python app_animals.py
```

## 領域制御

LivePortraitは画像の特定の領域のアニメーション制御をサポートしており、より精细な效果调整を可能にします。

### 領域制御の有効化

Gradioインターフェースでは、領域制御オプションを見つけることができ、驱动する顔の領域（目、口、頭部ポーズなど）を選択できます。

## ポートレート編集

Gradioインターフェースでは、ユーザーはソースポートレートの各种属性を精密に編集できます。

## プライバシー保護

LivePortraitは多个のプライバシー保護機能を提供します：

**モーション模板**：原始驱动動画の代わりに.pkl模板ファイルを使用して、敏感なコンテンツをアップロードするリスクを回避します

**自动クロップ**：驱动動画を自動的にクロップして頭に焦点を当て、背景情報を削減します

**動画保護**：模板作成機能により、视频を運動模板に変換してから使用できます

## 出力形式

デフォルトの出力形式はMP4動画ファイルで、3つの部分（驱动動画、ソースコンテンツ、生成結果）を含んでいます。--no_flag_concatパラメータを使用すると、生成されたアニメーション部分のみを出力できます。

## コミュニティツールの統合

LivePortraitは複数のコミュニティプロジェクトに統合されています：

**ComfyUI**：複数のComfyUIノードがリアルタイムプレビュー機能を備えたLivePortraitをサポートしています

**Stable Diffusion WebUI**：sd-webui-live-portrait拡張機能がLivePortraitをSD WebUIに統合しています

**FaceFusion**：バージョン3.0はLivePortraitを表情恢复と顔エディタプロセッサとして統合しています

**HuggingFace Spaces**：ローカルインストールなしでオンラインでLivePortraitを体験できます

## ベストプラクティス

最良のアニメーション効果を得るには、以下のベストプラクティスに従ってください：

高品質のソース画像を選択してください：明確な正面ポートレートが最も効果的です

プロフェッショナルな驱动動画を使用してください：正面で表情が自然な動画が効果的です

驱动视频的长さを控制してください：长すぎる驱动视频は処理时间を增加させ、效果が低下する可能性があります

パラメータを合理的に設定してください：実際の效果に基づいて表情强度和其他パラメータを調整してください

照明条件を一致させてください：ソース画像と驱动動画の照明条件を尽量一致させてください
