# Kotlin 자주 묻는 질문

## 기본 질문

### Kotlin이란?

Kotlin은 JetBrains에서 개발하고 유지 관리하는 현대적인 정적 타입 프로그래밍 언어입니다. 프로젝트는 2010년에 시작되었으며 2016년 2월 첫 번째 안정 버전 1.0이 출시되었습니다. Kotlin은 JVM, Android, JavaScript, WebAssembly 및 네이티브 플랫폼에서 실행됩니다. Apache 2.0 라이선스에 따라 공개된 오픈소스 언어입니다.

### Kotlin은 무료인가요?

네, Kotlin은 완전히 무료입니다. 과거에도 무료였고, 지금도 무료이며, 앞으로도 무료로 유지될 것입니다. Kotlin은 Apache 2.0 라이선스에 따라 개발되며 소스 코드는 GitHub에서 확인할 수 있습니다.

### Kotlin은 객체 지향인가요, 함수형인가요?

Kotlin은 객체 지향 프로그래밍과 함수형 프로그래밍 스타일을 모두 지원합니다. 필요에 따라 어느 하나를 사용하거나 둘 다의 요소를 조합할 수 있습니다. 고차 함수, 함수 타입, 람다 등의 최고 수준의 지원을 통해 Kotlin은 함수형 프로그래밍에 적합한 선택입니다.

### Kotlin과 Java의 차이점은?

Java와 비교할 때 Kotlin은 더 간결합니다. 동일한 기능에 대해 약 40% 적은 코드 줄 수가 소요됩니다. Kotlin은 또한 NullPointerException을 줄이는 nullable 타입 지원 등으로 더 타입 안전합니다. 스마트 캐스트, 고차 함수, 확장 함수, 람다 등의 기타 기능은 표현력 있는 코드를 작성하고 DSL 생성을 용이하게 합니다.

### Kotlin은 Java와 호환되나요?

네, Kotlin은 Java와 100% 상호 운용 가능합니다. Java에서 Kotlin 코드를 쉽게 호출하거나 Kotlin에서 Java 코드를 호출할 수 있어 채택이 훨씬 쉽고 위험이 적습니다. 마이그레이션을 단순화하는 IDE에 내장된 자동 Java에서 Kotlin으로 변환기도 있습니다.

## 설치 및 구성

### Kotlin을 어떻게 설치하나요?

Kotlin은 모든 IntelliJ IDEA 및 Android Studio 릴리스에 포함되어 있습니다. Kotlin 사용을 시작하려면 이 IDE 중 하나를 다운로드하여 설치하세요. 독립형 Kotlin 명령줄 컴파일러를 다운로드하거나 SDKMAN!, Homebrew, Snap 등의 패키지 관리자를 통해 설치할 수도 있습니다.

### 어떤 IDE가 Kotlin을 지원하나요?

Kotlin은 JetBrains에서 개발한 공식 Kotlin 플러그인과 함께 IntelliJ IDEA와 Android Studio에서 완전히 즉시 지원됩니다. 다른 IDE와 코드 편집기에는 커뮤니티에서 지원되는 Kotlin 플러그인만 있습니다. 브라우저에서 Kotlin 코드를 작성, 실행 및 공유하려면 Kotlin Playground도 사용해 볼 수 있습니다.

### 어떤 빌드 도구가 Kotlin을 지원하나요?

JVM에서 주요 빌드 도구는 Gradle과 Maven입니다. 클라이언트 측 JavaScript를 대상으로 하는 빌드 도구도 있습니다.

### Kotlin은 무엇으로 컴파일되나요?

JVM을 대상으로 할 때 Kotlin은 Java 호환 바이트코드를 생성합니다. JavaScript를 대상으로 할 때 Kotlin은 ES5.1로 트랜스파일하고 AMD 및 CommonJS를 포함한 모듈 시스템과 호환되는 코드를 생성합니다. 네이티브를 대상으로 할 때 Kotlin은 LLVM을 통해 플랫폼별 코드를 생성합니다.

### Kotlin이 대상하는 JVM 버전은?

Kotlin을 사용하면 실행을 위한 JVM 버전을 선택할 수 있습니다. 기본적으로 Kotlin/JVM 컴파일러는 Java 8 호환 바이트코드를 생성합니다. 최신 Java 버전의 최적화를 사용하려면 9~25까지 대상 Java 버전을 명시적으로 지정할 수 있습니다. 결과 바이트코드가 이전 버전에서 실행되지 않을 수 있습니다. Kotlin 1.5부터는 Java 8 미만의 버전과 호환되는 바이트코드 생성이 지원되지 않습니다.

## 적용 분야

### Kotlin으로 무엇을 할 수 있나요?

Kotlin은 서버 측, 클라이언트 측 웹, Android 또는 다중 플랫폼 등 어떤 종류의 개발에도 사용할 수 있습니다. Kotlin/Native로 임베디드 시스템, macOS, iOS 등 다른 플랫폼 지원도 가능합니다.

### Android 개발에 Kotlin을 사용할 수 있나요?

네, Kotlin은 Android에서 일급 언어로 지원됩니다. Basecamp, Pinterest 등 Android용 Kotlin을 사용하는 애플리케이션이 수백 개 있습니다. 자세한 내용은 Android 개발에 대한 Google 리소스를 확인하세요.

### 서버 측 개발에 Kotlin을 사용할 수 있나요?

네, Kotlin은 JVM과 100% 호환되므로 Spring Boot, Vert.x, JSF 등 기존 프레임워크를 사용할 수 있습니다. 또한 Ktor 등 Kotlin으로 작성된 프레임워크도 있습니다.

### 웹 개발에 Kotlin을 사용할 수 있나요?

네, 백엔드 웹 개발에 Kotlin은 Ktor, Spring 등 프레임워크와 함께 잘 작동하여 효율적인 서버 측 애플리케이션 구축이 가능합니다. 또한 클라이언트 측 웹 개발에 Kotlin/Wasm을 사용할 수 있습니다.

### 데스크톱 개발에 Kotlin을 사용할 수 있나요?

네, JavaFX, Swing 등 모든 Java UI 프레임워크를 사용할 수 있습니다. 또한 TornadoFX 등 Kotlin 전용 프레임워크도 있습니다.

### 네이티브 개발에 Kotlin을 사용할 수 있나요?

네, Kotlin/Native은 Kotlin의 일부로 제공됩니다. VM 없이 실행될 수 있는 네이티브 코드로 Kotlin을 컴파일합니다. 인기 있는 데스크톱 및 모바일 플랫폼, 일부 IoT 기기에서 시도해 볼 수 있습니다.

## 학습 질문

### Kotlin 배우기가 어렵나요?

Kotlin은 Java, C#, JavaScript, Scala, Groovy 등 기존 언어에서 영감을 받았습니다. 사람들이 며칠 만에 Kotlin 읽고 쓰기를 쉽게 시작할 수 있도록 Kotlin이 배우기 쉽게 만들었습니다. 관용적 Kotlin을 배우고 일부 고급 기능을 사용하려면 시간이 더 걸릴 수 있지만, 전반적으로 복잡한 언어는 아닙니다.

### Kotlin을 어떻게 배우나요?

시작하기에 가장 좋은 곳은 Kotlin 웹사이트입니다. Kotlin을 시작하려면 공식 IDE 중 하나를 설치하거나 온라인에서 시도해 보세요. 공식 Tour은 이론과 연습을 통해 초급에서 중급으로 안내하는 대화형 튜토리얼을 제공합니다.

### Kotlin 관련 책이 있나요?

Kotlin에 대한 책이 많이 있습니다. 검토하고 추천할 수 있는 책은 Books 페이지에 나열되어 있습니다. 더 많은 책은 kotlin.link의 커뮤니티 유지 목록을 참조하세요.

### 온라인 Kotlin 코스가 있나요?

JetBrains Academy의 Kotlin Core 트랙으로 작동하는 애플리케이션을 만들면서 모든 Kotlin 필수 사항을 배울 수 있습니다. Pluralsight, O'Reilly, Udemy 등의 다른 코스도 있습니다. 공식 YouTube 채널도 튜토리얼과 콘텐츠를 제공합니다.

### Kotlin 커뮤니티가 활발한가요?

네, Kotlin은 매우 활발한 커뮤니티를 가지고 있습니다. Kotlin 개발자들은 약 30,000명의 회원이 있는 Kotlin 포럼, Stack Overflow, 특히 Kotlin Slack에서 활동합니다.

### Kotlin 이벤트가 있나요?

네, Kotlin에만 전념하는 많은 사용자 그룹과 모임이 있습니다. 웹사이트에서 목록을 찾을 수 있습니다. 또한 전 세계에서 커뮤니티가 개최하는 Kotlin Nights 이벤트가 있습니다.

### Kotlin 컨퍼런스가 있나요?

네, KotlinConf는 JetBrains가 주관하는 연례 컨퍼런스로 전 세계 개발자를 모습니다. 또한 전 세계 여러 컨퍼런스에서 Kotlin이 다루어집니다.

## 개발 질문

### Kotlin의 현재 버전은?

현재 출시된 버전은 2026년 2월 5일에 출시된 2.3.10입니다. 자세한 내용은 GitHub에서 확인하세요.

### 어떤 기업이 Kotlin을 사용하나요?

Kotlin을 사용하는 기업이 너무 많아 나열할 수 없지만, Square, Pinterest, Basecamp, 맥도날드, Adobe, Atlassian, AWS, Philips, Forbes 등의 더 알려진 기업이 있습니다.

### Kotlin을 누가 개발하나요?

Kotlin은 100명 이상의 JetBrains 엔지니어 팀이 개발합니다. 수석 언어 설계자는 Michail Zarečenskij입니다. 핵심 팀 외에도 GitHub에 250명 이상의 외부 기여자가 있습니다.

### Kotlin 재단이란?

Kotlin 재단은 Kotlin 생태계 개발에서 커뮤니티 노력을 적극적으로 지원합니다.

### Kotlin 브랜드 자산은 무엇이 있나요?

로고 및 기타 브랜드 자산은 Kotlin 웹사이트에서 다운로드할 수 있습니다. 로고 사용 시 가이드라인의 간단한 규칙과 Kotlin 브랜드 사용 가이드라인을 따르세요.

## 기술 질문

### Kotlin이 멀티스레딩을 지원하나요?

Kotlin 코루틴은 단일 스레드에서 여러 코루틴 실행을 허용하는 강력한 동시성 지원을 제공합니다. 코루틴은 스레드보다 훨씬 가벼워 시스템 리소스를 고갈시키지 않고 수만 개의 코루틴을 생성할 수 있습니다. Kotlin은 비동기 데이터 흐름 처리를 위한 Flow도 제공합니다.

### Kotlin에 가비지 컬렉션이 있나요?

Kotlin이 JVM에서 실행되면 JVM 가비지 컬렉터에 의해 관리됩니다. Kotlin/Native은 다른 메모리 관리 전략을 사용합니다.

### Kotlin의 성능은?

Kotlin은 성능이 유사한 Java와 유사한 바이트코드로 컴파일됩니다. Kotlin의 기능과 구문 설탕은 일반적으로 중요한 성능 오버헤드를 도입하지 않습니다. 경우에 따라 인라인 함수 같은 Kotlin 기능이 오히려 성능 이점을 제공할 수도 있습니다.

### Kotlin 코드를 디버그하려면?

IntelliJ IDEA와 Android Studio에서는 Java와 동일한 디버깅 도구를 사용할 수 있습니다. 중단점 설정, 단계별 실행, 변수 값 보기 등 표준 디버깅 기능이 모두 지원됩니다.

### Kotlin이 제네릭스를 지원하나요?

네, Kotlin의 제네릭 시스템은 Java와 유사하지만 타입 투영과 선언 위치 공변성 등의 고급 기능을 추가하여 제네릭스를 더 안전하고 유연하게 만듭니다.

### Kotlin에 표준 라이브러리가 있나요?

네, Kotlin 표준 라이브러리는 컬렉션 작업, 문자열 처리, I/O 작업, 수학 함수 등을 포함한 광범위한 기능을 제공합니다. 표준 라이브러리는 API 일관성과 사용 용이성을 고려하여 설계되었습니다.

## 마이그레이션 질문

### Java 코드를 Kotlin으로 어떻게 변환하나요?

IntelliJ IDEA와 Android Studio에는 Java에서 Kotlin으로의 내장 변환기가 있습니다. 변환할 Java 파일을 선택하고 코드 > Java 파일을 Kotlin 파일로 변환을 선택하면 자동 변환됩니다.

### 프로젝트에서 Java와 Kotlin을 함께 사용할 수 있나요?

네, Kotlin과 Java는 완전히 호환되며 프로젝트에서 혼합할 수 있습니다. 일반적인 마이그레이션 전략은 전체 프로젝트를 한 번에 마이그레이션하는 대신 핵심 클래스를 점진적으로 Kotlin으로 변환하는 것입니다.

### Kotlin으로 마이그레이션하는 데 얼마나 걸리나요?

프로젝트 크기에 따라 다릅니다. 소규모 프로젝트의 경우 몇 시간만 걸릴 수 있습니다. 대규모 프로젝트의 경우 일반적으로 점진적 마이그레이션 전략이 사용되며, 정상적인 프로젝트 개발 속도를 유지하면서 주~개월 내에 완료될 수 있습니다.

### Kotlin 모범 사례는?

관용적 Kotlin을 따르는 것이 중요합니다. Kotlin의 공식 코딩 표준을 사용하고, var보다 val을 선호하며, 데이터 클래스, sealed 클래스, 확장 함수 등 Kotlin 언어 기능을 충분히 활용하세요. Kotlin은 상세한 코딩 스타일 가이드를 제공합니다.

## 생태계 및 도구 질문

### 일반적인 Kotlin 라이브러리는?

Kotlin 생태계가 풍부합니다. 일반적인 라이브러리에는 비동기 프로그래밍용 Kotlin Coroutines, 의존성 주입용 Koin, 웹 프레임워크용 Ktor, 네트워크 요청용 Retrofit, 데이터베이스용 Room, SQL 프레임워크용 Exposed 등이 있습니다.

### Kotlin 라이브러리를 어떻게 게시하나요?

Gradle 또는 Maven을 통해 Maven Central 또는 다른 저장소에 라이브러리를 게시할 수 있습니다. Kotlin 라이브러리는 일반적으로 JVM 바이트코드로 게시되지만, Kotlin/Wasm 또는 Kotlin/Native 버전도 게시할 수 있습니다.

### Kotlin에는 어떤 테스트 프레임워크가 있나요?

Kotlin은 JUnit, TestNG 등 모든 Java 테스트 프레임워크를 사용할 수 있습니다. Kotlin은 더 Kotlin 스타일의 테스트 API를 제공하는 Kotlin Test 라이브러리도 제공합니다. Kotlin Multiplatform 프로젝트의 경우 Kotlin Common Test를 사용할 수 있습니다.

## 미래 및 버전 질문

### Kotlin의 향후 방향은?

Kotlin 팀은 지속적으로 언어 성능과 개발 경험을 개선하고 있습니다. Kotlin Multiplatform은 다중 플랫폼 개발을 더 원활하게 만드는 주요 개발 방향입니다. Kotlin/Wasm도 빠르게 발전하여 브라우저에서 실행하기 위한 더 나은 지원을 제공합니다. 또한 Kotlin은 AI 통합을 탐구하고 있습니다.

### Kotlin이 하위 호환성을 유지하나요?

Kotlin은 하위 호환성을 매우 중요하게 생각합니다. Kotlin 1.0 시대에 작성된 대부분의 코드는 Kotlin 2.x 버전에서 정상적으로 실행될 수 있습니다. Kotlin 팀은-breaking 변경을 신중하게 처리하며 필요시 마이그레이션 도구를 제공합니다.

### 최신 Kotlin 소식을 얻으려면?

Kotlin YouTube 채널을 구독하고 Kotlin을 Twitter 또는 Bluesky에서 팔로우하세요. Kotlin 블로그는 정기적으로 버전 정보와 튜토리얼을 게시합니다.
