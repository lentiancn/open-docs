# Node.js 安裝指南

## 下載 Node.js

Node.js 提供了適用於各種作業系統的官方安裝套件。造訪 [Node.js 官方網站](https://nodejs.org/zh-tw/download) 下載最新版本。

### 版本選擇

- **LTS (長期支援) 版本**: 推薦用於生產環境，提供穩定的 API 和長期支援
- **Current (目前) 版本**: 包含最新功能，適合開發和測試環境

## Windows 安裝

### 方法一：使用官方安裝套件

1. 造訪 [Node.js 下載頁面](https://nodejs.org/zh-tw/download)
2. 下載 Windows 安裝程式 (.msi)
3. 執行安裝程式並依照精靈完成安裝
4. 驗證安裝：
   ```bash
   node --version
   npm --version
   ```

### 方法二：使用 Windows Package Manager (winget)

```bash
# 安裝 LTS 版本
winget install OpenJS.NodeJS.LTS

# 安裝最新版本
winget install OpenJS.NodeJS
```

### 方法三：使用 Chocolatey

```bash
# 安裝 LTS 版本
choco install nodejs-lts

# 安裝最新版本
choco install nodejs
```

## macOS 安裝

### 方法一：使用官方安裝套件

1. 下載 macOS 安裝套件 (.pkg)
2. 按兩下執行安裝程式
3. 依照精靈完成安裝

### 方法二：使用 Homebrew

```bash
# 安裝 LTS 版本
brew install node@lts

# 或安裝最新版本
brew install node

# 如果需要將命令新增到 PATH
echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 方法三：使用 MacPorts

```bash
# 安裝 LTS 版本
sudo port install nodejs-lts

# 安裝最新版本
sudo port install nodejs
```

## Linux 安裝

### Ubuntu/Debian

#### 方法一：使用套件管理員（可能不是最新版本）

```bash
sudo apt update
sudo apt install nodejs npm
```

#### 方法二：使用 NodeSource 儲存庫取得最新版本

```bash
# 對於 LTS 版本
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# 對於最新版本
curl -fsSL https://deb.nodesource.com/setup_current.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### CentOS/RHEL/Fedora

#### 方法一：使用套件管理員

```bash
# Fedora
sudo dnf install nodejs npm

# CentOS/RHEL 8+
sudo dnf module enable nodejs:18
sudo dnf install nodejs
```

#### 方法二：使用 NodeSource 儲存庫

```bash
# 對於 LTS 版本
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs

# 對於最新版本
curl -fsSL https://rpm.nodesource.com/setup_current.x | sudo bash -
sudo yum install nodejs
```

### 其他 Linux 發行版

大多數 Linux 發行版都可以透過各自的套件管理員安裝 Node.js：

- **Arch Linux**: `sudo pacman -S nodejs npm`
- **openSUSE**: `sudo zypper install nodejs npm`
- **Alpine Linux**: `apk add nodejs npm`

## 驗證安裝

安裝完成後，驗證 Node.js 和 npm 是否正確安裝：

```bash
# 檢查 Node.js 版本
node --version

# 檢查 npm 版本  
npm --version

# 執行簡單的測試
node -e "console.log('Hello Node.js!')"

# 檢查全域安裝目錄
npm config get prefix
```

## 管理多個 Node.js 版本

在開發中可能需要在不同專案中使用不同的 Node.js 版本。推薦使用以下工具：

### nvm (Node Version Manager) - macOS/Linux

```bash
# 安裝 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 或使用 wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新載入 shell 設定
source ~/.bashrc  # 或 ~/.zshrc
```

常用命令：
```bash
# 列出所有可用版本
nvm list-remote

# 列出已安裝版本
nvm list

# 安裝特定版本
nvm install 18.17.0

# 安裝 LTS 版本
nvm install --lts

# 切換版本
nvm use 18.17.0

# 設定預設版本
nvm alias default 18.17.0

# 卸載版本
nvm uninstall 16.20.0
```

### nvm-windows - Windows

1. 下載 [nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. 執行安裝程式
3. 使用命令列管理版本：

```cmd
:: 列出可用版本
nvm list available

:: 安裝版本
nvm install 18.17.0

:: 切換版本
nvm use 18.17.0

:: 查看已安裝版本
nvm list
```

### fnm (Fast Node Manager) - 跨平台

```bash
# 安裝 fnm
curl -fsSL https://fnm.vercel.app/install | bash

# 常用命令
fnm install 18.17.0
fnm use 18.17.0
fnm default 18.17.0
```

## 環境變數設定

確保 Node.js 和 npm 的可執行檔路徑已新增到系統 PATH 環境變數中。

### Windows

- 通常自動新增到 PATH
- 如果未找到命令，檢查系統環境變數中的 PATH

### macOS/Linux

如果使用套件管理員安裝，通常會自動設定。如果手動安裝或使用版本管理員：

```bash
# 對於 nvm 使用者，通常自動設定
# 對於手動安裝，新增到 ~/.bashrc 或 ~/.zshrc
export PATH=$PATH:/usr/local/bin
export PATH=$PATH:$HOME/.local/bin
```

## 故障排除

### 權限問題

在 Linux/macOS 上，如果遇到 npm 全域安裝的權限問題，建議：

#### 解決方案 1：使用版本管理員
- 使用 nvm 或 fnm 管理 Node.js（推薦）

#### 解決方案 2：設定 npm 全域目錄
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

#### 解決方案 3：使用 sudo（不推薦）
```bash
sudo npm install -g package-name
```

### 網路問題

如果下載速度慢或無法連線，可以設定 npm 使用國內鏡像：

```bash
# 設定淘寶鏡像（推薦）
npm config set registry https://registry.npmmirror.com

# 或臨時使用
npm install --registry https://registry.npmmirror.com

# 驗證設定
npm config get registry

# 恢復官方來源
npm config delete registry
```

### 版本衝突

如果系統中存在多個 Node.js 版本導致衝突：

1. **解除安裝所有版本**：
   - Windows: 透過控制台解除安裝
   - macOS: `brew uninstall node` 或刪除安裝目錄
   - Linux: `sudo apt remove nodejs npm`

2. **使用版本管理員重新安裝**：
   - 安裝 nvm/fnm
   - 透過版本管理員安裝所需版本

### 常見錯誤訊息

#### "command not found: node"
- **原因**: PATH 未正確設定
- **解決**: 重新安裝或手動新增到 PATH

#### "npm ERR! code EACCES"
- **原因**: 權限不足
- **解決**: 使用版本管理員或設定 npm 全域目錄

#### "Error: Cannot find module"
- **原因**: 相依性未安裝或路徑錯誤
- **解決**: 執行 `npm install` 或檢查模組路徑

透過以上步驟，您應該能夠成功安裝和設定 Node.js 環境。如果遇到其他問題，請參考官方文件或社群支援資源。
