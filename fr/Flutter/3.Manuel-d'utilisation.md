# Manuel Utilisateur Flutter

## Concepts de Base

### Introduction aux Widgets

Widget est le bloc de construction de base de l'interface utilisateur Flutter. Dans Flutter, tout est un widget, y compris la mise en page, le padding, l'alignement, etc. Les widgets décrivent à quoi leur vue devrait ressembler étant donné la configuration et l'état actuels.

Les widgets Flutter sont immuables. Lorsque vous devez modifier l'interface utilisateur, vous devez recréer le widget ou utiliser StatefulWidget avec la gestion d'état. Les widgets peuvent être imbriqués pour former un arbre de widgets, et Flutter compare efficacement l'ancien et le nouvel arbre de widgets, ne rendant que les parties réellement modifiées.

Les widgets courants incluent : Text (texte), Image (image), Icon (icône), Container, Row/Column (mise en page horizontale/verticale), Stack (mise en page empilée), ListView (liste), etc.

### StatelessWidget et StatefulWidget

StatelessWidget est un widget sans état, adapté aux affichages d'interface utilisateur statiques comme le texte et les images. Définir un StatelessWidget ne nécessite que d'implémenter la méthode build, renvoyant l'arbre de widgets à rendre.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget est un widget avec état, adapté aux scénarios nécessitant des mises à jour dynamiques de l'interface utilisateur comme les compteurs et les formulaires. StatefulWidget doit être utilisé avec State, en appelant la méthode setState pour notifier Flutter de se rerendre lorsque l'état change.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## Bases de la Mise en Page

### Mise en Page Linéaire

Row et Column sont les widgets de mise en page linéaire les plus utilisés. Row organise les widgets enfants horizontalement, Column verticalement. Utilisez MainAxisAlignment et CrossAxisAlignment pour contrôler l'alignement.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### Mise en Page Empilée

Stack est utilisé pour créer des mises en page empilées, plaçant des widgets au-dessus d'un autre widget. Positioned est utilisé pour positionner les enfants dans un Stack.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### Mise en Page Liste

ListView est le widget de liste le plus utilisé, adapté aux longues listes. ListView.builder est adapté aux grands ensembles de données avec chargement paresseux. ListView.separated peut ajouter des séparateurs.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## Gestion d'État

### setState

setState est la méthode de gestion d'état la plus basique, adaptée à l'état local simple. Dans StatefulWidget, appeler setState déclenche le rebuild du widget.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider est l'une des solutions de gestion d'état recommandées par Flutter. Grâce à Provider, vous pouvez partager l'état dans l'arbre des widgets pour éviter de le transmettre couche par couche.

```dart
// Définir ChangeNotifier
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Utiliser Provider
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## Requêtes Réseau

### Package HTTP

Utilisez le package http pour faire des requêtes HTTP simples :

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

## Stockage Local

### SharedPreferences

Utilisé pour stocker des données clé-valeur simples :

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

## Navigation et Routage

Flutter utilise Navigator pour la navigation entre pages. Les opérations de base incluent Navigator.push (nouvelle page), Navigator.pop (retour), Navigator.pushReplacement (remplacer la page actuelle), etc.

Le routage nommé est la méthode de gestion de routage recommandée :

```dart
// Définir les routes
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// Naviguer
Navigator.pushNamed(context, '/detail');
```

## Construction et Publication

### Publication Android

Utilisez keytool pour générer une clé de signature :

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

Configurez la signature dans android/app/build.gradle, puis exécutez à la racine du projet :

```bash
flutter build apk --release
```

### Publication iOS

Configurez le certificat de signature et le profil de provisionnement dans Xcode, puis exécutez :

```bash
flutter build ios --release
```

Notez que la publication iOS nécessite un compte développeur Apple et les certificats correspondants.
