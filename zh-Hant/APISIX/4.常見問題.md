# 常見問題

本文檔彙總了 Apache APISIX 使用過程中常見的技術問題和解答。

## 基礎問題

### 什麼是 Apache APISIX？

Apache APISIX 是 Apache 軟體基金會下的頂級項目，是一個具有動態、實時、高效能等特點的雲端原生 API 閘道器。它提供了動態路由、動態上游、動態憑證、A/B 測試、灰度發布、限速、防攻擊、監控警報等功能。

### APISIX 和 Nginx 有什麼區別？

Nginx 是傳統的 Web 伺服器和反向代理，而 APISIX 是基於 Nginx 構建的 API 閘道器。APISIX 在 Nginx 基礎上增加了完全動態的配置管理、豐富的插件系統、精細的流量控制和內建的可觀測性功能。

### APISIX 的效能如何？

APISIX 繼承了 Nginx 的高效能特性，同時通過優化和插件機制保持了高效的處理能力。具體效能取決於硬體配置和業務複雜度，一般情況下可以處理每秒數萬甚至數十萬請求。

## 安裝問題

### Docker 啟動失敗怎麼辦？

1. 檢查 Docker 是否正常運行：`docker ps`
2. 查看容器日誌：`docker-compose logs apisix`
3. 常見問題：
   - 連接阜衝突：檢查 9080、9180、9443 連接阜是否被占用
   - etcd 連線失敗：確認 etcd 容器是否正常啟動
   - 權限問題：確保有足夠的權限運行 Docker

### 如何查看 APISIX 版本？

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

返回格式：`Server: APISIX/Version`

### 安裝後無法存取 Dashboard？

1. 確認 Dashboard 連接阜是否正確映射（預設 9180）
2. 檢查防火牆設定
3. 確認容器網路配置是否正確

## 配置問題

### 如何修改管理 API 連接阜？

編輯 `apisix.yml` 配置檔案：

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### 如何啟用 HTTPS？

1. 配置 SSL 憑證：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "憑證內容",
  "key": "私鑰內容",
  "snis": ["example.com"]
}'
```

2. 在路由中啟用 HTTPS：

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### 路由配置不生效？

1. 檢查路由配置是否正確
2. 確認 etcd 配置是否正確同步
3. 檢查插件是否有語法錯誤
4. 查看 APISIX 錯誤日誌

### 如何實現跨域請求？

使用 `cors` 插件：

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## 效能問題

### 如何提高吞吐量？

1. **增加工作進程數：**`apisix.worker_processes: auto`
2. **啟用 HTTP/2：**在 listen 配置中設置 `http2: true`
3. **優化上游連線池：**配置 `keepalive_pool.size`
4. **使用快取插件：**啟用 `proxy-cache` 插件

### 記憶體占用過高怎麼辦？

1. 減少不必要的插件載入
2. 限制上游連線數
3. 調整日誌緩衝區大小
4. 使用串流響應處理大檔案

## 功能問題

### 如何實現 A/B 測試？

使用 `traffic-split` 插件：

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### 如何實現請求限流？

使用 `limit-req` 插件：

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## 整合問題

### 如何與 Kubernetes 整合？

使用 APISIX Ingress Controller：

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### 如何與 Prometheus 整合？

1. 啟用 Prometheus 插件
2. 配置 Prometheus 抓取配置

## 安全問題

### 如何防止 APISIX 被濫用？

1. **啟用身分驗證**：在所有路由上啟用 key-auth 或其他認證插件
2. **限制管理 API 存取**：只允許受信任的 IP 存取管理 API
3. **使用 HTTPS**：啟用 TLS 加密
4. **配置限速**：防止惡意請求
5. **啟用存取日誌**：記錄所有請求以便審計

### 管理 API 如何開啟授權？

```yaml
deployment:
  admin:
    admin_key_required: true
```

## 運維問題

### 如何升級 APISIX？

**Docker 方式：**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes 方式：**

```bash
helm upgrade apisix apisix/apisix
```

### 常見錯誤代碼

- **401 Unauthorized**：身分驗證失敗
- **403 Forbidden**：沒有存取權限
- **404 Not Found**：路由不存在
- **429 Too Many Requests**：請求超過限速
- **500 Internal Server Error**：上游服務錯誤
- **502 Bad Gateway**：上游服務不可用
- **503 Service Unavailable**：服務不可用
