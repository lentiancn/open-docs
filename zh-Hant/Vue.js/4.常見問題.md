# Vue.js 常見問題

## Vue 2 和 Vue 3 的區別

Vue 3 是當前最新主版本，包含 Vue 2 中沒有的新特性（如 Teleport、Suspense、多根元素模板）以及一些非兼容性變更。

**主要優勢：**
- 更小的包體積
- 更好的性能  
- 更好的可擴展性
- 更好的 TypeScript/IDE 支持

**選擇建議：**
- **新項目推薦使用 Vue 3**
- **需要支持 IE11 時選擇 Vue 2**（Vue 3 不支持 IE11）

**遷移信息：**
- 組合式 API 已兼容至 Vue 2.7
- 詳細遷移指南：[Vue 3 遷移指南](https://v3-migration.vuejs.org/zh/)

## Vue 2 維護狀態

- Vue 2 在 2022 年 6 月發布最後版本 2.7
- **2023 年 12 月 31 日已結束官方維護**
- 需要延長支持的團隊可參考 [Vue 2 延長 LTS](https://v2.vuejs.org/lts/)

## 瀏覽器支持

- **Vue 3.x**：僅支持原生支持 ES2016 的瀏覽器（不包括 IE11）
- **Vue 2.x**：支持 IE11 及更廣泛的瀏覽器

## 性能表現

- Vue 3 是性能最強的主流前端框架之一
- 在 [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html) 中表現優異
- 幾乎不需要手動優化即可處理大多數應用場景
- 文檔站點在 Moto G4 + 低速 4G 網絡下仍獲得 100 分性能得分

## 包體積

- **最小應用**（構建工具 + tree-shaking）：約 16kb（brotli 壓縮）
- **完整功能應用**：約 27kb
- **無構建工具**（包含編譯器）：約 41kb
- **漸進式增強場景**：可考慮 [petite-vue](https://github.com/vuejs/petite-vue)（僅 6kb）

## 大規模應用支持

Vue 完全能夠勝任大規模應用場景：

- **單文件組件**：提供模塊化開發模型
- **組合式 API**：一流的 TypeScript 集成，便於組織複雜邏輯
- **全面工具鏈**：支持應用持續增長
- **低入門門檻**：降低團隊培訓成本

## 開源協議

- 被眾多知名組織使用：Wikimedia、NASA、Apple、Google、Microsoft、GitLab、Zoom、騰訊、微博、嗶哩嗶哩、快手等

## API 風格選擇

### 選項式 API vs 組合式 API

**選項式 API 適合：**
- 初學者或面向對象背景的開發者
- 無需構建工具的簡單場景
- 漸進式增強應用

**組合式 API 適合：**
- 完整單頁應用開發
- 需要更好的邏輯組織和重用
- TypeScript 項目

## JavaScript vs TypeScript

- Vue 本身用 TypeScript 實現，提供一流 TS 支持
- **不強制要求使用 TypeScript**
- 即使使用 JavaScript，也能受益於 TS 友好的 API 設計
- 選擇取決於團隊背景和項目規模需求

## Vue 與 Web Components

- Vue 設計受 Web Components 啟發（如插槽）
- Web Components 更底層（以自定義元素為中心）
- Vue 解決更多上層問題（渲染、狀態管理、路由、SSR 等）
- Vue 完全支持在組件中使用原生自定義元素
- 也支持將 Vue 組件導出為原生自定義元素
