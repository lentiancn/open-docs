# GitLab Guía de Instalación

## Requisitos

Antes de instalar GitLab, asegúrese de que su servidor cumpla con los siguientes requisitos de hardware y software. Para equipos pequeños y medianos, se recomienda un mínimo de 4 núcleos de CPU y 8GB de RAM, que puede soportar cientos de repositorios y decenas de usuarios concurrentes. Los equipos más grandes o escenarios de alta carga requieren más recursos, preferiblemente 8+ núcleos de CPU y 16GB+ de RAM.

En cuanto a sistemas operativos, GitLab soporta varias distribuciones de Linux incluyendo Ubuntu, Debian, CentOS, Red Hat Enterprise Linux y Fedora. Ubuntu o Debian se recomiendan oficialmente porque su gestión de paquetes es la más conveniente. GitLab requiere al menos 20GB de espacio en disco para la instalación, con 50GB+ recomendados para configuraciones básicas, y a menudo cientos de GB para entornos de producción.

Las dependencias de software incluyen OpenSSH Server para soporte de protocolo SSH, base de datos PostgreSQL (GitLab incluye una versión incorporada), y servidor SMTP para notificaciones de correo. También necesita configurar la zona horaria y el nombre de host correctos, y asegurar que el servidor tenga acceso normal a Internet para descargar paquetes de dependencias.

## Pasos de Instalación

### Instalación en Ubuntu/Debian

En sistemas Ubuntu o Debian, se recomienda instalar GitLab a través del repositorio apt oficial, ya que esto asegura obtener las últimas versiones estables y actualizaciones de seguridad automáticas. Primero, instale los paquetes de dependencia necesarios:

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

Agregue el repositorio APT de GitLab e instale GitLab CE (Edición Comunitaria):

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

Reemplace EXTERNAL_URL con el dominio o dirección IP real para acceder a GitLab. El proceso de instalación configurará automáticamente PostgreSQL, Redis y el propio GitLab.

### Instalación en CentOS/RHEL

En CentOS o Red Hat Enterprise Linux, puede instalar GitLab usando el administrador de paquetes yum o dnf. Primero, instale las dependencias necesarias:

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

Agregue el repositorio YUM de GitLab e instale:

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## Configuración Inicial

Después de la instalación, necesita configurar los parámetros básicos de GitLab. Edite el archivo /etc/gitlab/gitlab.rb, que es el archivo de configuración principal de GitLab. El elemento de configuración más importante es external_url, que es la dirección de acceso externo para el servicio GitLab:

```ruby
external_url 'https://gitlab.example.com'
```

Si necesita usar un puerto SSH personalizado, configure el parámetro gitlab_rails['gitlab_shell_ssh_port']. La configuración del servidor de correo también es importante, usada para enviar notificaciones y correos de restablecimiento de contraseña:

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

Después de guardar la configuración, ejecute el siguiente comando para aplicar los cambios:

```bash
sudo gitlab-ctl reconfigure
```

Este comando reconfigurará todos los componentes de GitLab, incluyendo migraciones de base de datos y creación de usuario inicial. Todo el proceso puede tomar varios minutos.

## Verificando la Instalación

Después de que la configuración esté completa, puede acceder a la dirección especificada por external_url a través de su navegador. Si todo es correcto, debería ver la página de inicio de sesión de GitLab. En el primer acceso, verá la página de inicio de sesión de la cuenta de administrador. El nombre de usuario de administrador predeterminado es root, y necesitará establecer una contraseña en el primer inicio de sesión.

GitLab proporciona la herramienta de línea de comandos gitlab-ctl para gestionar el estado del servicio:

```bash
# Verificar estado del servicio
sudo gitlab-ctl status

# Ver registros
sudo gitlab-ctl tail

# Reiniciar servicio
sudo gitlab-ctl restart
```

También puede usar el comando gitlab-rake para verificaciones de salud:

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Instalación con Docker

Para pruebas rápidas o despliegues pequeños, puede ejecutar GitLab usando Docker. Primero, instale Docker Engine, luego ejecute el siguiente comando en el host:

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

La instalación con Docker monta tres volúmenes para persistir configuración, registros y datos. El primer inicio del contenedor Docker toma tiempo para inicializarse. Después de esperar pacientemente, puede acceder a través de su navegador.

## Actualización y Mantenimiento

Actualizar GitLab regularmente a la última versión proporciona nuevas funciones y correcciones de seguridad. Actualizando en Ubuntu/Debian:

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

Se recomienda hacer respaldo de datos antes de actualizar. GitLab proporciona una herramienta de respaldo incorporada:

```bash
sudo gitlab-backup create
```

Los archivos de respaldo se almacenan por defecto en /var/opt/gitlab/backups, y puede configurar políticas de retención de respaldos y ubicaciones de almacenamiento remoto.
