# Go インストールガイド

## 環境要件

GoはLinux、macOS、Windowsを含む複数のオペレーティングシステムで実行できます。異なるオペレーティングシステムには異なるインストール方法がありますが、コア要件は同じです。Go言語開発環境を正常にインストールして使用するために、システムが次の基本要件を満たしていることを確認してください。

オペレーティングシステムに関して、Goは幅広いプラットフォームをサポートしています。Linuxでは、Ubuntu、Debian、Fedora、CentOS、RHELなどの主流なディストリビューションでGoを実行できます。macOSはmacOS 10.13（High Sierra）以降が必要です。WindowsはWindows 10以降が必要ですが、Windows 7と8.1もサポートします（追加設定が必要な場合があります）。

ハードウェアに関して、Goの要件は比較的低いです。Goのインストールには約100MBのディスク容量が必要ですが、実際のプロジェクト開発ではプロジェクトのサイズに応じて更多の容量が必要な場合があります。メモリに関しては、日常の開発には2GB RAMで十分ですが、大規模なコンパイルや大きなテストの実行には更多のメモリが必要になる場合があります。

Goコンパイラ自体には特別なCPU要件はなく、x86-64やARM64などの现代のアーキテクチャーはスムーズに動作します。Goはクロスコンパイルをサポートしており、あるプラットフォーム에서他のプラットフォーム用のバイナリをコンパイルできます。

## インストール手順

### Linuxへのインストール

Linuxシステムでは、公式提供のtarballを使用してインストールすることが推奨されます。これが最もクリーンで柔軟な方法です。まず、Goの公式ウェブサイトから最新バージョンのtarballをダウンロードします：

```bash
# Goをダウンロード（最新バージョンに置き換えてください）
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# 古いGoインストールを削除（存在する場合）
sudo rm -rf /usr/local/go

# /usr/localに展開
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

次に、GoのバイナリディレクトリをPATH環境変数に追加します。~/.bashrcまたは~/.profileファイルを編集し、次の行を追加します：

```bash
export PATH=$PATH:/usr/local/go/bin
```

保存後、以下のコマンドを実行して設定を適用します：

```bash
source ~/.bashrc
```

またはmacOSや一部のLinuxディストリビューションでは：

```bash
source ~/.profile
```

### macOSへのインストール

macOSシステムには、2つの主要なインストール方法があります。1つはHomebrewパッケージマネージャーを使用する方法で、Homebrewをすでに設定しているユーザーに適しています：

```bash
# Homebrewでインストール
brew install go

# インストールを確認
go version
```

2番目の方法は、公式のmacOSインストーラーパッケージをダウンロードすることです。Goの公式ウェブサイトにアクセスして.pkgインストーラーをダウンロードし、ダブルクリックして実行し、インストールウィザードに従ってインストールを完了します。インストーラーは自動的に環境変数を設定します。

インストール後、ターミナルを開いてインストールを確認します：

```bash
go version
```

"go1.21.5 darwin/amd64"のような出力が表示されるはずです。

### Windowsへのインストール

Windowsシステムでは、MSIインストーラーを使用することが推奨されます。Goの公式ウェブサイトにアクセスしてWindows用MSIインストーラーをダウンロードし、ダブルクリックして実行し、インストールウィザードに従ってインストールを完了します。インストーラーはPATHを含む環境変数を自動的に設定します。

インストール後、コマンドプロンプトまたはPowerShellを開いてインストールを確認します：

```bash
go version
```

WindowsユーザーはChocolateyパッケージマネージャーを使用することもできます：

```bash
choco install golang
```

### バージョンマネージャーの使用（推奨）

複数のGoバージョンを同時に管理する必要がある開発者には、バージョンマネージャーの使用が推奨されます。一般的なバージョンマネージャーには、gvm（Go Version Manager）とasdf-goがあります：

gvmのインストールと使用：

```bash
# gvmをインストール
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# Goバージョンをインストール
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdfはGoをサポートする多言語バージョンマネージャーです：

```bash
# asdfプラグインをインストール
asdf plugin add golang

# Goバージョンをインストール
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## インストールの確認

インストール後、Goが正しくインストールされていることを確認する必要があります。ターミナルまたはコマンドプロンプトを開いて、以下のコマンドを実行します：

```bash
# Goバージョンを確認
go version

# Go環境情報を確認
go env
```

go envコマンドは、GOPATH、GOROOT、GOOS、GOARCHなどの複数の環境変数を出力します。この情報は、Goの動作方法と環境の理解に重要です。

シンプルなプログラムを実行して、開発環境全体が正常に動作することを確認します：

```bash
# 作業ディレクトリを作成
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# モジュールを初期化（Go 1.16以降ではgo modを推奨）
go mod init hello

# テストプログラムを作成
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# プログラムを実行
go run main.go
```

"Hello, Go!"が出力されれば、Go開発環境は正しく設定されています。

## 環境設定

### GOPATHとモジュール

Go 1.11より前、GoはGOPATHモードを使用して依存関係を管理していました。Go 1.11以降、Goはモジュールサポートを導入し、go modでプロジェクト依存関係を管理することが推奨されています。モジュールを使用する場合、GOPATHを設定する必要はなく、任意のディレクトリでプロジェクトを作成できます。

企業イントラネット環境やプライベートリポジトリを使用する場合などは、GOPRIVATE環境変数を設定する必要がある場合があります：

```bash
export GOPRIVATE=github.com/company/*
```

### プロキシ設定

中国大陆などの地域では、Go公式モジュールリポジトリへの直接アクセスが遅い、またはアクセスできない場合があります。Goモジュールプロキシを設定できます：

```bash
# アリババクラウドプロキシを使用
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct

# または七牛雲プロキシを使用
go env -w GOPROXY=https://goproxy.cn,direct
```

### IDE設定

Go開発には通常、VS Code、GoLand、Sublime Textなどのエディターが使用されます。VS Codeでは、コード補完、デバッグ、フォーマットなどの良好な開発体験を得るために、Go拡張機能をインストールする必要があります。

GoLandはJetBrains開発の 전문 Go IDEであり、高度なデバッグ、リファクタリング、テストを含む最も完全なGo開発サポートを提供します。

## アップグレードとアンインストール

### Goバージョンのアップグレード

Goのアップグレードは非常に簡単です。新規バージョンのインストーラーをダウンロードし、上記のインストール手順に従って再インストールするだけです。新しいインストーラーは古いバージョンを上書きします。

バージョンマネージャーを使用すると、バージョン間を 쉽게切り替えることができます：

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### Goのアンインストール

LinuxまたはmacOSでは、Goのアンインストール只需要削除インストールディレクトリ：

```bash
# Goインストールディレクトリを削除
sudo rm -rf /usr/local/go

# PATHから削除
# ~/.bashrcまたは~/.profileを編集し、相关行を削除
```

Windowsでは、コントロールパネルの「プログラムと機能」からアンインストールするか、Chocolateyを使用します：

```bash
choco uninstall golang
```
