# Nacos 설치 가이드

## 환경 요구 사항

Nacos는 Java 런타임 환경을 필요로 합니다. 다음 버전을 사용하고 있는지 확인하세요:

- 64비트 OS: Linux/Unix/Mac/Windows 지원, Linux/Unix/Mac 권장
- 64비트 JDK 17+ (JDK 17 이상 권장)
- JDK 다운로드: https://www.oracle.com/java/technologies/downloads/#java17
- JDK 환경 설정: https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## 권장 구성

Nacos는 최소 다음 시스템 구성으로 실행하는 것이 권장됩니다:

- CPU: 2코어
- 메모리: 4GB
- 디스크: 60GB

## 설치 방법

Nacos는 여러 설치 방법을 지원합니다. 요구 사항에 맞는 적절한 방법을 선택할 수 있습니다.

### 방법 1: nacos-setup 원클릭 배포 사용 (권장)

nacos-setup은 다운로드, 인증 구성, 포트 감지 및 Java 환경 검증을 자동으로 처리하는 자동화된 원클릭 배포를 제공합니다.

#### nacos-setup 설치

Linux/macOS:

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows (PowerShell):

```powershell
# PowerShell로 실행
```

#### Nacos 서비스 시작

로컬에 최신 버전의 단일 Nacos를 원클릭으로 배포:

```bash
nacos-setup
```

로컬에 특정 버전의 단일 Nacos 배포:

```bash
nacos-setup -v 3.1.1
```

로컬에 최신 버전의 클러스터 Nacos 배포:

```bash
nacos-setup -c cluster
```

외부 저장소를 전역적으로 구성하고, 데이터베이스 주소, 포트, 사용자 이름 및 비밀번호를 입력한 후 nacos-setup 다시 실행:

```bash
nacos-setup --datasource-conf
```

사용법에 대한 자세한 내용은 nacos-setup 사용자 매뉴얼을 참조하세요.

### 방법 2: 수동 다운로드 및 배포

#### 설치 패키지 다운로드

Nacos 배포 패키지는 두 가지 방법으로 얻을 수 있습니다:

**방법 1: Nacos 공식 웹사이트에서 다운로드**

Nacos 공식 웹사이트 버전 다운로드 페이지로 이동하여 안정 버전을 선택한 후 바이너리 패키지 열의 zip 파일을 클릭하여 다운로드합니다.

**방법 2: GitHub에서 다운로드**

Nacos GitHub 릴리스 페이지로 이동하여 해당 버전의 배포 패키지를 다운로드합니다.

참고: 많은 사용자가 동시에 다운로드할 때 다운로드 속도 제한 실패가 발생할 수 있습니다.这种现象发生时，请稍后重试或从GitHub下载.

#### Nacos 배포 패키지 압축 풀기

Linux/Unix/Mac:

```bash
unzip nacos-server-$version.zip
# 또는
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows:

압축 해제 도구를 사용하여 nacos-server-$version.zip을 직접 압축 풉니다.

#### 서버 시작

Linux/Unix/Mac 시작 명령 (standalone은 비클러스터 모드를 의미):

```bash
sh startup.sh -m standalone
```

Ubuntu 시스템을 사용하거나 기호를 찾을 수 없다는 오류 메시지가 표시되면 다음을 시도하세요:

```bash
bash startup.sh -m standalone
```

시작 프로그램은 세 가지 인증 관련 구성을 입력하라는 메시지를 표시합니다 (Nacos 3.0.0부터 콘솔 인증이 기본적으로 활성화되어 있으므로 다음 세 가지 인증 구성을 입력해야 합니다):

```
nacos.core.auth.plugin.nacos.token.secret.key이(가) 누락되었습니다. Base64 문자열로 설정하세요: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key이(가) 누락되었습니다. 설정하세요: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value이(가) 누락되었습니다. 설정하세요: ${your_input_server_identity_key}
```

Windows 시작 명령:

```bash
cmd startup.cmd
```

또는 startup.cmd를 더블 클릭하여 실행합니다.

### 방법 3: Docker를 사용한 설치

#### Nacos 이미지 가져오기

```bash
docker pull nacos/nacos-server
```

#### 단일 모드로 실행

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### 클러스터 모드로 실행

Docker Compose 또는 Kubernetes를 사용한 클러스터 배포가 필요합니다.

### 방법 4: Kubernetes에 설치

Nacos는 Kubernetes 환경에서의 배포를 지원하며 Helm Chart 또는 Operator를 사용하여 설치할 수 있습니다. 구체적인 단계는 Nacos Kubernetes 배포 문서를 참조하세요.

## Nacos 서비스 시작 확인

Nacos 로그 디렉토리로 이동하여 `tail -f start.out`을 사용하여 로그를 확인합니다. 다음 메시지가 표시되면 서비스가 성공적으로 시작된 것입니다:

```
Nacos started successfully in stand alone mode. use embedded storage
```

## 빠른 확인

### Nacos 콘솔

모든 브라우저를 열고 주소를 입력하세요: http://127.0.0.1:8080/index.html - Nacos 콘솔에 접근합니다.

참고: 처음 열면 사용자 nacos의 관리자 비밀번호를 초기화해야 합니다. 기본 사용자 이름/비밀번호: nacos/nacos@demo.

### 서비스 등록

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### 서비스 검색

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### 구성 게시

먼저 로그인하여 액세스 토큰을 가져옵니다:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

액세스 토큰을 사용하여 구성 만들기:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### 구성 가져오기

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## 서버 종료

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

또는 shutdown.cmd를 더블 클릭하여 실행합니다.

## 버전 선택

현재 프로덕션에 권장되는 안정 버전은 3.1.1입니다. 각 버전에서 지원되는 기능에 대해 자세히 알아볼 수 있는 곳:

- Nacos 버전 다운로드 페이지
- Nacos 릴리스 노트
- Nacos 릴리스 공지

## 참고 사항

1. Nacos 3.0.0부터 콘솔 인증이 기본적으로 활성화되어 있습니다. 첫 번째 시작 시 인증 관련 매개변수를 구성해야 합니다.
2. 클러스터 모드 배포에는 데이터 저장을 위해 외부 데이터베이스(MySQL 등)가 필요합니다.
3. 프로덕션 환경에서는 고가용성을 보장하기 위해 클러스터 모드를 사용해야 합니다.
4. 방화벽에서 필수 포트가 열려 있는지 확인하세요(기본값 8848, 9848).
5. 최상의 성능을 위해 JDK 17 이상을 권장합니다.
