# Руководство по установке

В этом документе описаны разные способы установки Apache APISIX, чтобы вы могли быстро развернуть API-шлюз.

## Требования к окружению

Перед установкой убедитесь, что у вас есть:

- **Операционная система**: Linux (рекомендуется Ubuntu 20.04+, CentOS 7+, Debian 11+)
- **Docker** (для Docker-деплоя): Docker и Docker Compose уже установлены
- **etcd**: APISIX использует etcd для хранения конфигурации, версия 3.4.0 или выше
- **Perl**: некоторые зависимости требуют Perl 5.5.32+
- **curl**: для проверки установки

## Быстрая установка через Docker (рекомендуется)

Самый простой способ — идеально подходит для разработки и тестирования.

### Шаг 1: Создаём директорию Quick Start

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### Шаг 2: Запускаем скрипт Quick Start

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

Эта команда автоматически запустит два контейнера: apisix-quickstart и etcd. APISIX использует etcd для хранения и синхронизации конфигурации, контейнеры работают в режиме Docker host network, так что доступ к ним возможен напрямую с локальной машины.

### Шаг 3: Проверяем установку

После успешной установки в терминале появится:

```
✔ APISIX is ready!
```

Проверить статус работы APISIX можно так:

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Должен вернуться ответ:

```
Server: APISIX/Version
```

Где Version — установленная версия APISIX, например APISIX/3.15.0.

### Шаг 4: Заходим в Dashboard

В APISIX есть встроенный веб-интерфейс Dashboard. Откройте:

```
http://127.0.0.1:9180/ui
```

> ⚠️ Важно: для удобства управляющий API по умолчанию работает без авторизации. В продакшене обязательно включите авторизацию.

## Установка через Docker Compose

Если нужна кастомная конфигурация, используйте Docker Compose.

### Шаг 1: Создаём файл конфигурации

Создайте файл docker-compose.yaml:

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### Шаг 2: Запускаем сервисы

```bash
docker-compose up -d
```

### Шаг 3: Проверяем работу

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Установка в Kubernetes

У APISIX есть официальный Ingress Controller — можно развернуть прямо в Kubernetes.

### Установка через Helm Chart

```bash
# Добавляем репозиторий Helm
helm repo add apisix https://charts.apiseven.com
helm repo update

# Устанавливаем APISIX
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### Установка через kubectl

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## Установка из исходного кода

Подходит для кастомной сборки в продакшене.

### Шаг 1: Устанавливаем зависимости

**Ubuntu/Debian:**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL:**

```bash
sudo yum install -y perl gcc git curl
```

### Шаг 2: Устанавливаем OpenResty

```bash
# Добавляем репозиторий OpenResty
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# Устанавливаем OpenResty
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### Шаг 3: Устанавливаем APISIX

```bash
# Клонируем исходники
git clone https://github.com/apache/apisix.git
cd apisix

# Устанавливаем зависимости
make deps

# Инициализируем конфигурацию
make init

# Запускаем APISIX
make run
```

## Проверка установки

Независимо от способа установки, проверить работу можно так:

1. **Проверяем статус процессов**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **Создаём тестовый маршрут**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **Тестируем маршрут**

```bash
curl "http://127.0.0.1:9080/test"
```

## Удаление

**Docker:**

```bash
# Останавливаем контейнеры
docker-compose down

# Удаляем тома (если нужно)
docker-compose down -v
```

**Kubernetes:**

```bash
helm uninstall apisix
```
