# Kubernetes ユーザーマニュアル

## 基本コンセプト

### Pod
PodはKubernetesで展開可能な最小単位であり、クラスターで実行されている1つ以上のコンテナを表します。通常、Podは1つのコンテナを含みますが、密に結合された複数のコンテナを含むこともできます。

### Deployment
Deploymentはステートレスアプリケーションを管理するために使用され、宣言的な更新機能を提供し、アプリケーションの展開、スケーリング、更新を容易にします。

### Service
ServiceはPodのセットに安定したネットワークエンドポイントを提供し、PodのIPアドレスが変更されても安定性を維持します。

### Namespace
Namespaceはクラスターリソースを複数の論理パーティションに分割し、複数のチームやプロジェクトが同じクラスターを共有する場合に適しています。

## 一般的なコマンド

### クラスター管理

```bash
# クラスター情報を表示
kubectl cluster-info

# ノードを表示
kubectl get nodes

# すべてのPodを表示
kubectl get pods --all-namespaces

# クラスターのリソース使用状況を表示
kubectl top nodes
kubectl top pods
```

### アプリケーションの展開

```bash
# Deploymentを作成
kubectl create deployment nginx --image=nginx

# Deploymentを表示
kubectl get deployments

# スケーリング
kubectl scale deployment nginx --replicas=3

# イメージをアップデート
kubectl set image deployment/nginx nginx=nginx:1.21

# ロールバック
kubectl rollout undo deployment/nginx

# ロールアウト状況を表示
kubectl rollout status deployment/nginx
```

### サービス管理

```bash
# サービスを公開
kubectl expose deployment nginx --port=80 --type=LoadBalancer

# サービスを表示
kubectl get services

# サービスを削除
kubectl delete service <service-name>
```

### デバッグ

```bash
# Podのログを表示
kubectl logs <pod-name>

# コンテナに入る
kubectl exec -it <pod-name> -- /bin/bash

# リソースの詳細情報を表示
kubectl describe pod <pod-name>

# ポート転送（デバッグ用）
kubectl port-forward <pod-name> 8080:80
```

## リソースマニフェスト

KubernetesはYAMLファイルを使用した宣言的なリソース管理をサポートしています。一般的なリソース定義の例：

### Deploymentの例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
```

### Serviceの例

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: ClusterIP
```

### ConfigMapの例

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  database_url: "postgres://db:5432/myapp"
  cache_enabled: "true"
```

### Secretの例

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: changeme
```

## アプリケーションライフサイクル管理

### アプリケーションの展開

```bash
# YAMLファイルを使用して展開
kubectl apply -f deployment.yaml

# 展開状況を表示
kubectl get pods -l app=my-app
```

### アプリケーションの更新

```bash
# イメージバージョンを更新
kubectl set image deployment/my-app my-app=my-app:v2

# 更新の進捗を表示
kubectl rollout status deployment/my-app
```

### ロールバック

```bash
# 前のバージョンにロールバック
kubectl rollout undo deployment/my-app

# 特定のリビジョンにロールバック
kubectl rollout undo deployment/my-app --to-revision=2

# 履歴を表示
kubectl rollout history deployment/my-app
```

### スケーリング

```bash
# 手動スケーリング
kubectl scale deployment my-app --replicas=5

# 自動スケーリング（HPA）
kubectl autoscale deployment my-app --min=2 --max=10 --cpu-percent=80
```

## ストレージ管理

### PersistentVolumeの作成

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  storageClassName: standard
  hostPath:
    path: /data/my-app
```

### PersistentVolumeClaimの作成

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

### PodでPVCを使用

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: my-app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

## 構成管理

### ConfigMapの使用

```bash
# ファイルからConfigMapを作成
kubectl create configmap my-config --from-file=config.properties

# 環境変数ファイルから作成
kubectl create configmap my-config --from-env-file=env.properties

# Podで使用
kubectl apply -f pod-with-configmap.yaml
```

### Secretの使用

```bash
# ファイルからSecretを作成
kubectl create secret generic my-secret --from-file=credentials.json

# Podで使用
kubectl apply -f pod-with-secret.yaml
```

## リソースクォータと制限

### ResourceQuota

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: my-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    pods: "20"
```

### LimitRange

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: my-limits
spec:
  limits:
  - max:
      cpu: "2"
      memory: 1Gi
    min:
      cpu: 100m
      memory: 128Mi
    default:
      cpu: 500m
      memory: 512Mi
    defaultRequest:
      cpu: 200m
      memory: 256Mi
    type: Container
```

## モニタリングとログ

### モニタリング

```bash
# kubectl topでリソース使用状況を表示
kubectl top nodes
kubectl top pods

# kubectl describeで詳細を表示
kubectl describe node <node-name>
kubectl describe pod <pod-name>
```

### ログ

```bash
# Podのログを表示
kubectl logs my-app-pod

# リアルタイムでログを表示
kubectl logs -f my-app-pod

# 以前のコンテナインスタンスのログを表示
kubectl logs --previous my-app-pod
```

6. ヘルスチェックプローブを設定
7. RBACを使用して権限を制御
