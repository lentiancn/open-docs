# MongoDB Benutzerhandbuch

## Grundkonzepte

### Datenbank

Eine Datenbank ist der oberste Container für Daten in MongoDB. Jede Datenbank hat ihre eigenen Collections und Berechtigungseinstellungen.

### Collection

Eine Collection ist eine Gruppe von MongoDB-Dokumenten, ähnlich einer Tabelle in einer relationalen Datenbank. Dokumente in einer Collection können unterschiedliche Felder haben.

### Dokument

Ein Dokument ist die Grundeinheit von Daten in MongoDB, bestehend aus Feld-Wert-Paaren. Dokumente ähneln JSON-Objekten und werden im BSON-Format gespeichert.

## Mit MongoDB verbinden

### Mit lokaler Datenbank verbinden

```bash
mongosh
```

### Mit Remote-Datenbank verbinden

```bash
mongosh "mongodb://hostname:port"
```

## Datenbankoperationen

### Datenbanken anzeigen

```bash
show dbs
```

### Datenbank wechseln/erstellen

```bash
use meinedatenbank
```

### Datenbank löschen

```bash
db.dropDatabase()
```

## Collection-Operationen

### Collections anzeigen

```bash
show collections
```

### Collection erstellen

```bash
db.createCollection("meinecollection")
```

### Collection löschen

```bash
db.meinecollection.drop()
```

## Dokumente einfügen

### Ein Dokument einfügen

```bash
db.benutzer.insertOne({
  name: "Max",
  alter: 30,
  email: "max@beispiel.de"
})
```

### Mehrere Dokumente einfügen

```bash
db.benutzer.insertMany([
  { name: "Anna", alter: 25 },
  { name: "Peter", alter: 35 }
])
```

## Dokumente abfragen

### Alle Dokumente abfragen

```bash
db.benutzer.find()
```

### Mit Bedingungen abfragen

```bash
db.benutzer.find({ alter: 30 })
db.benutzer.find({ alter: { $gte: 25 } })
```

### Ergebnisse sortieren

```bash
db.benutzer.find().sort({ alter: -1 })
```

### Ergebnisse begrenzen

```bash
db.benutzer.find().limit(5)
```

## Dokumente aktualisieren

### Ein Dokument aktualisieren

```bash
db.benutzer.updateOne(
  { name: "Max" },
  { $set: { alter: 31 } }
)
```

### Mehrere Dokumente aktualisieren

```bash
db.benutzer.updateMany(
  { alter: { $lt: 30 } },
  { $set: { status: "jung" } }
)
```

## Dokumente löschen

### Ein Dokument löschen

```bash
db.benutzer.deleteOne({ name: "Max" })
```

### Mehrere Dokumente löschen

```bash
db.benutzer.deleteMany({ alter: { $lt: 20 } })
```

## Aggregation

### Aggregations-Pipeline

```bash
db.benutzer.aggregate([
  { $match: { alter: { $gte: 25 } } },
  { $group: { _id: "$stadt", gesamt: { $sum: 1 } } },
  { $sort: { gesamt: -1 } }
])
```

## Indizes

### Index erstellen

```bash
db.benutzer.createIndex({ name: 1 })
db.benutzer.createIndex({ alter: 1, name: 1 })
db.benutzer.createIndex({ email: 1 }, { unique: true })
```

### Indizes anzeigen

```bash
db.benutzer.getIndexes()
```

### Index löschen

```bash
db.benutzer.dropIndex("name_1")
```

## Transaktionen

MongoDB 4.0+ unterstützt Multi-Dokument-Transaktionen.

```bash
sitzung = db.getMongo().startSession()
sitzung.startTransaction()
try {
  sitzung.getDatabase("meineDB").konten.updateOne(
    { kontoId: 1 },
    { $inc: { kontostand: -100 } }
  )
  sitzung.commitTransaction()
} catch (fehler) {
  sitzung.abortTransaction()
} finally {
  sitzung.endSession()
}
```

## Change Streams

Change Streams ermöglichen es Anwendungen, Datenänderungen in Echtzeit zu überwachen.

```bash
var changeStream = db.benutzer.watch()
while (changeStream.hasNext()) {
  printjson(changeStream.next())
}
```

## GridFS

GridFS wird verwendet, um große Dateien zu speichern, die das 16-MB-Limit überschreiten.

```bash
mongofiles put meinedatei.txt
mongofiles list
mongofiles get meinedatei.txt
mongofiles delete meinedatei.txt
```
