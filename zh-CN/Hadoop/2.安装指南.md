# Hadoop 安装指南

## 环境要求

在安装 Hadoop 之前，需要确保系统满足基本的硬件和软件要求。Hadoop 集群可以由多台计算机组成，也可以是单机伪分布式模式。不同的部署模式对硬件有不同的要求，但核心的软件依赖是相似的。了解这些要求有助于规划 Hadoop 集群的规模和配置，避免在安装过程中遇到不必要的麻烦。

硬件方面，对于小型测试集群，建议每台机器至少配备 4 核 CPU、8GB 内存和 500GB 硬盘。对于生产环境，建议每台机器配备 8 核或更多 CPU、16GB 或更多内存，以及 1TB 或更大的硬盘。Hadoop 的数据节点会存储大量数据，因此硬盘容量尤为重要。Hadoop 建议使用 RAID 配置或 JBOD（Just a Bunch Of Disks）配置，后者允许 Hadoop 更灵活地管理磁盘空间。

操作系统方面，Hadoop 支持 Linux（Ubuntu、CentOS、Debian、Red Hat 等）和 Windows（通过 Cygwin 或 Windows Subsystem for Linux）。生产环境强烈建议使用 Linux 系统，以获得最佳的性能和稳定性。Hadoop 与各种云平台也有良好的集成，可以在 AWS、Azure、Google Cloud 等云服务上部署。

Java 是 Hadoop 唯一的运行时依赖。Hadoop 3.x 需要 Java 8 或更高版本。建议使用 Oracle JDK 或 OpenJDK。在安装 Hadoop 之前，需要确保 Java 已正确安装，并且 JAVA_HOME 环境变量已设置。可以通过以下命令验证 Java 安装：

```bash
java -version
echo $JAVA_HOME
```

网络是 Hadoop 集群正常工作的基础。集群中的所有节点应该能够通过网络相互通信，建议使用高速网络（如千兆以太网）。节点之间需要配置 SSH 无密码登录，以便 Hadoop 可以在节点之间分发任务和传输数据。此外，需要在 /etc/hosts 文件中配置所有节点的主机名和 IP 地址映射。

## 安装步骤

### 单机模式安装

单机模式是 Hadoop 最简单的安装方式，所有组件都运行在同一个 JVM 进程中。这种模式主要用于开发和测试，不涉及分布式存储或计算。首先下载 Hadoop 的二进制发行版，可以从 Apache Hadoop 官方网站获取：

```bash
# 下载 Hadoop（替换为最新版本）
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# 解压到指定目录
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# 重命名目录（可选）
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

接下来配置环境变量。编辑 ~/.bashrc 或 /etc/profile 文件，添加以下内容：

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # 根据实际 Java 路径调整
```

使配置生效：

```bash
source ~/.bashrc
```

验证安装：

```bash
hadoop version
```

### 伪分布式模式安装

伪分布式模式在单机上模拟分布式环境，NameNode、DataNode、ResourceManager、NodeManager 等组件都运行在同一台机器上。这种模式适合开发和测试分布式功能。

首先配置 Hadoop 的核心文件。进入 Hadoop 配置目录：

```bash
cd $HADOOP_HOME/etc/hadoop
```

配置 core-site.xml，指定 HDFS 的默认文件系统：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

配置 hdfs-site.xml，指定 HDFS 的副本数量（伪分布式模式下为 1）：

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

配置 mapred-site.xml，指定 MapReduce 使用的资源调度框架：

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

配置 yarn-site.xml，配置 YARN 相关的属性：

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### SSH 无密码登录配置

Hadoop 集群中的节点需要配置 SSH 无密码登录，以便 master 节点可以免密码访问 slave 节点：

```bash
# 生成 SSH 密钥对
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# 将公钥添加到authorized_keys
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# 设置正确的权限
chmod 0600 ~/.ssh/authorized_keys

# 测试 SSH 登录
ssh localhost
```

如果使用多节点集群，需要将公钥复制到所有节点：

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### 格式化与启动 HDFS

首次使用 HDFS 之前，需要格式化 NameNode：

```bash
hdfs namenode -format
```

启动 HDFS 和 YARN：

```bash
start-dfs.sh
start-yarn.sh
```

或者使用 start-all.sh 一次性启动所有服务：

```bash
start-all.sh
```

验证服务启动状态：

```bash
# 查看 Java 进程
jps

# 预期的进程包括：
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

可以通过 Web UI 访问 Hadoop 服务：NameNode 的 Web UI 地址是 http://localhost:9870，ResourceManager 的 Web UI 地址是 http://localhost:8088。

## 完全分布式集群安装

### 集群规划

完全分布式模式需要多台物理机或虚拟机组成集群。在安装之前，需要进行合理的集群规划。确定集群的规模（包含多少个节点），分配各节点的角色（NameNode、Secondary NameNode、DataNode、ResourceManager、NodeManager）。对于小型集群，可以将 NameNode 和 ResourceManager 部署在同一台机器上；对于大型集群，建议分开部署以提高性能。

网络规划也很重要。建议所有节点在同一内网中，延迟低、带宽高。在 /etc/hosts 文件中配置所有节点的主机名和 IP 地址：

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### 配置文件修改

在完全分布式模式下，需要修改更多配置文件。首先配置 workers 文件（早期版本使用 slaves 文件），列出所有 DataNode 和 NodeManager 节点：

```
slave1
slave2
slave3
```

core-site.xml 和 hdfs-site.xml 的配置与伪分布式类似，但需要添加更多属性。对于 NameNode 高可用（HA）配置，需要配置两个 NameNode：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

hdfs-site.xml 配置 NameNode HA：

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

YARN 的配置类似，可以在 yarn-site.xml 中配置 ResourceManager HA。

### 同步配置文件

将配置好的 Hadoop 安装目录同步到所有节点。可以使用 scp 命令逐个复制：

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

或者使用分布式复制工具如 rsync：

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### 启动集群

在 NameNode 节点上格式化 NameNode（首次启动时）：

```bash
hdfs namenode -format
```

启动 HDFS：

```bash
start-dfs.sh
```

启动 YARN：

```bash
start-yarn.sh
```

检查所有节点的进程状态，确保服务正常运行：

```bash
# 在 NameNode 上
jps

# 在 DataNode 上
jps
```

## 验证与测试

### HDFS 测试

创建目录并上传文件测试 HDFS：

```bash
# 创建目录
hdfs dfs -mkdir -p /user/test

# 上传文件
hdfs dfs -put localfile.txt /user/test/

# 查看文件
hdfs dfs -ls /user/test

# 读取文件
hdfs dfs -cat /user/test/localfile.txt

# 删除文件
hdfs dfs -rm /user/test/localfile.txt
```

### MapReduce 测试

运行 MapReduce 示例程序验证集群功能：

```bash
# 运行 WordCount 示例
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# 查看结果
hdfs dfs -cat /output/part-r-00000
```

### 集群健康检查

使用 Hadoop 提供的命令检查集群状态：

```bash
# 查看 HDFS 状态
hdfs dfsadmin -report

# 查看 YARN 节点状态
yarn node -list -all

# 查看应用程序状态
yarn application -list
```

## 常见问题

### Java 版本不兼容

如果遇到 Java 版本相关的错误，确保安装的 Java 版本与 Hadoop 版本兼容。Hadoop 3.x 需要 Java 8 或更高版本。可以通过以下命令切换 Java 版本：

```bash
update-alternatives --config java
```

### 磁盘空间不足

DataNode 可能会因为磁盘空间不足而无法正常工作。定期监控磁盘使用情况，确保 DataNode 有足够的可用空间。可以在 hdfs-site.xml 中配置 dfs.datanode.fsdataset.volume.choosing.policy 为 AvailableSpaceVolumeChoosingPolicy，让 DataNode 优先使用可用空间更多的磁盘。

### 内存不足

如果集群节点内存不足，YARN 可能无法启动容器。可以通过 yarn-site.xml 调整内存配置：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
