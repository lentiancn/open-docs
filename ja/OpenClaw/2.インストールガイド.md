# OpenClaw インストールガイド

このガイドでは、異なるオペレーティングシステムへのOpenClawのインストール手順を詳しく説明します。

## 必要環境

- **Node.js**：バージョン22以上
- **オペレーティングシステム**：macOS、Linux、Windows
- **オプション**：Docker（サンドボックス機能が必要な場合）

## インストール方法

### macOS / Linux（推奨）

インストールスクリプトを使用するのが最も簡単です：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

このスクリプトはシステム環境を自動検出し、必要な依存関係をインストールしてOpenClawを設定します。

### Windows（PowerShell）

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### npmで直接インストール

Node.jsがすでにインストールされている場合は、npmを使用できます：

```bash
npm install -g openclaw@latest
```

### ソースからインストール

最新の開発バージョンを使用する場合や、カスタムインストールが必要な場合：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## インストール後の設定

### オンボーディングウィザードを実行

インストールが完了したら、ウィザードを実行して認証、Gateway設定、オプションのチャンネルを設定します：

```bash
openclaw onboard --install-daemon
```

ウィザードでは以下を 안내します：
- APIキー設定
- Gatewayポート設定
- チャンネル接続（WhatsApp、Telegram、Discordなど）
- セキュリティと権限設定

### Gatewayの状態を確認

上記のようにサービスをインストールした場合、Gatewayはすでに実行中です：

```bash
openclaw gateway status
```

### Gatewayをフォアグラウンドで起動

テストやトラブルシューティング用にGatewayをフォアグラウンドで実行する場合：

```bash
openclaw gateway --port 18789
```

### コントロールUIを開く

```bash
openclaw dashboard
```

またはブラウザで直接開く：`http://127.0.0.1:18789/`

## 環境変数

設定やステートディレクトリの場所你需要自定义的话，可以使用以下环境变量：

- `OPENCLAW_HOME`：内部パス解決に使用されるホームディレクトリを設定
- `OPENCLAW_STATE_DIR`：ステートディレクトリをオーバーライド
- `OPENCLAW_CONFIG_PATH`：設定ファイルパスをオーバーライド

## インストール確認

インストール後は、以下を実行して確認できます：

1. `openclaw --version` でバージョンを確認
2. `openclaw doctor` でシステム健康状態をチェック
3. コントロールUIを開いてテストメッセージを送信

## よくあるインストール問題

### Node.jsバージョンの確認

```bash
node --version
```

バージョンが22以上であることを確認してください。違場合は、Node.jsをアップグレードしてください。

### WindowsでGitが見つからない

Gitがインストールされ、システムPATHに追加されていることを確認してください。[git-scm.com](https://git-scm.com)からダウンロードできます。

### 権限エラー

インストール中に権限エラーが発生した場合は、以下を試してください：

```bash
sudo npm install -g openclaw@latest
```
