# Vite Guide d'Utilisation

Ce guide fournit des informations détaillées sur l'utilisation de Vite.

## Concepts de Base

### Modules ES Natifs

Vite utilise des modules ES natifs dans le navigateur sans bundling pendant le développement:
- Pas d'attente de bundling au démarrage
- Mises à jour HMR rapides
- Compilation à la demande

### Pre-bundling des Dépendances

Vite utilise esbuild pour pre-bundler les dépendances:
- Convertit CommonJS en ESM
- Réduit le nombre de requêtes
- Accélère le démarrage

## Gestion des Fichiers

### Point d'Entrée HTML

Vite utilise `index.html` comme point d'entrée:

```html
<script type="module" src="/src/main.js"></script>
```

### Import JavaScript

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import './style.css'

createApp(App).mount('#app')
```

### Import CSS

```javascript
import './style.css'
```

## Développement

### Hot Module Replacement

Vite offre un HMR instantané:
- Maintient l'état de l'application pendant les mises à jour
- Met à jour uniquement les modules modifiés

### Serveur de Développement

```bash
npm run dev
npm run dev -- --port 3000
npm run dev -- --open false
```

## Production

### Optimisation du Build

Vite optimise automatiquement:
- Tree Shaking
- Code Splitting
- Minification

### Stratégie de Chunking Personnalisée

```javascript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom']
        }
      }
    }
  }
})
```

## Meilleures Pratiques

1. Utiliser des imports relatifs pour un meilleur caching
2. Configurer les aliases de résolution
3. Utiliser le lazy loading pour les routes
4. Utiliser les variables d'environnement pour la configuration
