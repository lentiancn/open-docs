# Руководство пользователя Hadoop

## Руководство по операциям HDFS

### Основные операции файловой системы

HDFS предоставляет интерфейс командной строки, аналогичный традиционным файловым системам, позволяя пользователям выполнять различные файловые операции с помощью команды hdfs dfs. Эти команды очень похожи на команды оболочки Linux, поэтому пользователи, знакомые с системами Linux, могут быстро освоиться. Понимание этих основных операций фундаментально для использования Hadoop; ниже подробно описаны наиболее часто используемые команды HDFS.

Создание каталогов выполняется командой mkdir, которая может создавать одноуровневые или многоуровневые каталоги:

```bash
# Создать одноуровневый каталог
hdfs dfs -mkdir /user/data

# Создать многоуровневый каталог
hdfs dfs -mkdir -p /user/data/input

# Список содержимого каталога
hdfs dfs -ls /user/data

# Рекурсивный список содержимого каталога
hdfs dfs -ls -R /user/data
```

Загрузка и выгрузка файлов — распространённые операции:

```bash
# Загрузить локальный файл в HDFS
hdfs dfs -put localfile.txt /user/data/

# Загрузить и переименовать
hdfs dfs -put localfile.txt /user/data/newname.txt

# Скачать файл HDFS локально
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# Копировать внутренние файлы HDFS
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# Переместить внутренние файлы HDFS
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

Просмотр и удаление файлов:

```bash
# Просмотреть содержимое файла
hdfs dfs -cat /user/data/file.txt

# Просмотреть последние строки файла
hdfs dfs -tail /user/data/file.txt

# Просмотреть размер файла
hdfs dfs -du /user/data/file.txt

# Удалить файл
hdfs dfs -rm /user/data/file.txt

# Рекурсивно удалить каталог
hdfs dfs -rm -r /user/data/dir
```

### Управление разрешениями

Модель разрешений HDFS аналогична Unix/Linux, основана на трёхуровневой системе: пользователь, группа и остальные. Каждый файл и каталог имеют три разрешения: чтение (r), запись (w), выполнение (x), соответствующие владельцу, группе и остальным. Понимание управления разрешениями критически важно для защиты безопасности данных.

```bash
# Изменить владельца файла
hdfs dfs -chown user:group /user/data/file.txt

# Изменить разрешения файла
hdfs dfs -chmod 755 /user/data/file.txt

# Изменить группу владельца
hdfs dfs -chgrp groupname /user/data/file.txt
```

### Репликация и балансировка данных

HDFS по умолчанию реплицирует каждый блок данных три раза, сохраняя их на разных DataNode. Коэффициент репликации по умолчанию можно изменить через свойство dfs.replication:

```bash
# Установить коэффициент репликации для определённого файла
hdfs dfs -setrep -w 2 /user/data/file.txt

# Просмотреть информацию о расположении блоков файла
hdfs fsck /user/data/file.txt -files -blocks -locations
```

При добавлении или удалении узлов из кластера данные могут распределяться неравномерно. Используйте команду rebalance для балансировки данных:

```bash
start-balancer.sh -threshold 10
```

Параметр threshold представляет максимальный процент разницы между использованием диска узла и средним показателем кластера. Процесс балансировки потребляет сетевую пропускную способность; рекомендуется выполнять в периоды низкой нагрузки.

### Веб-интерфейс HDFS

HDFS предоставляет богатый веб-интерфейс, доступный через браузер на порту 9870 NameNode. Веб-интерфейс позволяет просматривать каталоги файловой системы, просматривать содержимое файлов, отслеживать состояние DataNode, просматривать информацию о блоках и многое другое. Веб-интерфейс — важный инструмент для персонала по эксплуатации при ежедневном мониторинге и устранении неполадок.

Основные функции веб-интерфейса включают: просмотр дерева каталогов HDFS, просмотр списков файлов и свойств файлов; просмотр списка DataNode и использования хранилища каждым DataNode; просмотр распределения блоков и состояния репликации; просмотр информации времени выполнения и журналов NameNode; выполнение простых файловых операций, таких как загрузка и удаление.

## Программирование MapReduce

### Пример WordCount

WordCount — это вводный пример программирования MapReduce, подсчитывающий количество появлений каждого слова в тексте. Понимание этого примера помогает освоить модель программирования MapReduce. Программа делится на фазы Map и Reduce: на фазе Map текст разбивается на слова, а на фазе Reduce подсчёты одинаковых слов суммируются.

Пример кода фазы Map:

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Пример кода фазы Reduce:

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### Отправка задания

После упаковки MapReduce в JAR-файл отправьте задание с помощью команды hadoop jar:

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

После отправки задания можно просматривать ход выполнения и результаты в веб-интерфейсе YARN (порт 8088). При завершении задания выводятся статистические данные, включая идентификатор задания, количество задач, время выполнения и т.д.

### Настройка задания

Различные параметры задания можно настроить через объект Configuration:

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

Общие параметры конфигурации включают: mapreduce.job.reduces устанавливает количество задач Reduce; mapreduce.map.memory.mb и mapreduce.reduce.memory.mb устанавливают память контейнера; mapreduce.map.cpu.vcores и mapreduce.reduce.cpu.vcores устанавливают количество ядер ЦП контейнера.

## Операции YARN

### Просмотр ресурсов кластера

YARN предоставляет богатые инструменты командной строки для управления ресурсами кластера:

```bash
# Просмотр состояния всех узлов
yarn node -list -all

# Просмотр списка приложений
yarn application -list

# Просмотр состояния конкретного приложения
yarn application -status application_1234567890123_0001

# Просмотр журналов приложения
yarn logs -applicationId application_1234567890123_0001
```

### Управление очередями

YARN использует очереди для управления распределением ресурсов. Очереди можно настроить через планировщик ёмкости или справедливый планировщик:

```bash
# Просмотр информации об очереди
yarn queue -status default

# Просмотр ёмкости очереди
yarn resourcemanager -refreshQueues
```

### Настройка планировщика ресурсов

Планировщик ёмкости — планировщик по умолчанию, поддерживающий разделение ресурсов между несколькими арендаторами. Настройте в yarn-site.xml:

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## Оптимизация производительности

### Настройка MapReduce

Оптимизация производительности заданий MapReduce включает несколько аспектов. Сначала разумно установите количество задач Map и Reduce. Количество задач Map обычно определяется количеством фрагментов входных данных, которое можно настроить через параметр mapreduce.input.fileinputformat.split.maxsize. Количество задач Reduce рекомендуется устанавливать в 0,9–1,2 раза от общего количества слотов Reduce кластера.

```bash
# Установить количество задач Map
-D mapreduce.job.maps=10

# Установить количество задач Reduce
-D mapreduce.job.reduces=5
```

Оптимизация фазы перемешивания значительно влияет на общую производительность. Можно настроить следующие параметры:

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### Настройка HDFS

Оптимизация производительности HDFS в основном фокусируется на эффективности чтения/записи данных. Для больших файлов можно настроить размер блока для баланса между локальностью данных и восстановлением после сбоев:

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

Включите короткое замыкание чтения, чтобы клиенты могли читать данные напрямую из локальных DataNode, сокращая сетевую передачу:

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### Настройка YARN

Распределение ресурсов YARN значительно влияет на использование кластера. Разумно установите ёмкость ресурсов NodeManager:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

Установите количество виртуальных ядер ЦП для лучшего использования многоядерных процессоров:

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## Импорт и экспорт данных

### Импорт данных из баз данных

Используйте Sqoop для удобной миграции данных между реляционными базами данных и HDFS:

```bash
# Импортировать всю таблицу
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Импортировать и создать таблицу Hive
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### Экспорт данных в базы данных

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### Выбор формата данных

Разные форматы данных по-разному влияют на производительность. commonly используемые форматы включают: TextFile (универсальность, но средняя производительность); SequenceFile (хорошо для MapReduce); Parquet (колоночное хранилище, хорошо для аналитических запросов); ORC (колоночное хранилище, оптимизированное для Hive). Для аналитических рабочих нагрузок рекомендуется использовать колоночные форматы хранения, такие как Parquet или ORC.
