# ユーザーマニュアル

このマニュアルでは、Apifox の主な機能と使い方について詳しく説明します。プロジェクト管理、接口設計、デバッグ、Mock、自動化テストなどについてカバーしています。

## プロジェクト管理

### プロジェクト作成

Apifox では、プロジェクトが API をまとめる一番大きな単位になります。

```bash
# 新規プロジェクト作成の手順
1. 左側の「プロジェクト」リスト横の「+」をクリック
2. プロジェクト名を入力
3. プロジェクトタイプを選択
4. 基礎テンプレートを選択（任意）
5. 「作成」をクリック
```

### プロジェクト設定

各プロジェクトには独自の設定オプションがあります：

- **基本情報**：プロジェクト名、説明アイコン
- **環境変数**：開発、テスト、本番など、不同的な環境の変数を定義
- **認証**：グローバルな認証方式を設定
- **エクスポート/インポート**：プロジェクトのデータをインポート・エクスポート

### 環境管理

Apifox は複数環境の設定に対応しています：

```bash
# 環境変数の使い方
リクエストでは {{変数名}} で環境変数を参照
例：{{baseUrl}}/api/users

環境を切り替える：上部にある環境セレクターで選択
```

## 接口設計

### 接口作成

接口は API の具体的な定義です：

```bash
# 接口作成の手順
1. プロジェクト内で「+」→「新規接口作成」をクリック
2. 基本情報を入力：
   - 接口の名前：ユーザーリスト
   - リクエストパス：/api/users
   - リクエストメソッド：GET
3. リクエストパラメータを定義
4. レスポンス構造を定義
5. 保存
```

### リクエストパラメータ

以下の类型のリクエストパラメータに対応しています：

- **パスパラメータ**：例：/api/users/:id の id
- **クエリパラメータ**：URL の ?name=test
- **リクエストヘッダー**：例：Authorization
- **リクエストボディ**：JSON、フォーム、XML などの形式

### レスポンス定義

各接口に成功と失敗のレスポンスを定義します：

```json
{
  "成功レスポンス 200": {
    "id": 1,
    "name": "田中",
    "email": "tanaka@example.com"
  },
  "エラーレスポンス 404": {
    "code": 404,
    "message": "ユーザーが見つかりません"
  }
}
```

### 接口リスト管理

接口をグループで管理できます。モジュールや機能ごとに分類可能：

- ドラッグで順序を替えられる
- 一括操作（一括削除、一括エクスポートなど）
- 検索とフィルター

## 接口デバッグ

### リクエスト送信

接口の詳細ページで、「送信」ボタンをクリックすればリクエストが送れます：

```bash
# 基本的なデバッグ手順
1. 接口を開く
2. 环境を選択
3. パラメータ値を設定
4. 「送信」をクリック
5. レスポンスを確認
```

### 認証設定

いくつかの認証方式に対応しています：

- **None**：認証なし
- **Bearer Token**：JWT 認証
- **Basic Auth**：基本認証
- **OAuth 2.0**：OAuth2 認証
- **カスタム認証**：必要な認証ヘッダーを独自に定義

### リクエスト前処理

リクエスト送信前にスクリプトを実行できます：

```javascript
// 例：タイムスタンプを追加
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// 例：ランダムIDを生成
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### レスポンス後処理

レスポンス受信後の処理スクリプトも設定できます：

```javascript
// 例：レスポンスデータを抽出
const response = pm.response.json();
console.log('レスポンスデータ:', response);

// 例：環境変数を設定
pm.environment.set('access_token', response.data.token);
```

## API Mock

### スマート Mock

Apifox 最も強力な機能の一つです。データ構造を定義するだけで、システムが自動的に Mock データを生成します：

```bash
# スマート Mock をオンにする
1. 接口を開く
2. 「Mock」タブをクリック
3. 「スマート Mock」をオン
4. 保存
```

### Mock ルール

カスタム Mock ルールも対応しています：

- `@string`：ランダムな文字列を生成
- `@integer`：ランダムな整数を生成
- `@float`：ランダムな浮動小数点を生成
- `@boolean`：ランダムな真偽値を生成
- `@date`：ランダムな日付を生成
- `@email`：ランダムなメールアドレスを生成
- `@url`：ランダムな URL を生成
- `@guid`：UUID を生成

### 例

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

生成される Mock データ：

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock サーバー

Apifox には内置 Mock サーバーが付いています：

```bash
# Mock URL を取得
接口の詳細ページで、Mock URL をコピー
例：https://mock.apifox.com/api/users
```

## 自動化テスト

### テストケース作成

```bash
# テスト作成の手順
1. 接口を開く
2. 「テスト」タブをクリック
3. 「テスト作成」をクリック
4. テストケース名を設定
5. アサーションを追加
6. 保存
```

### アサーションの種類

いくつかのアサーションに対応しています：

- **ステータスコードアサーション**：レスポンスのステータスコードをチェック
- **JSON アサーション**：レスポンスの JSON データをチェック
- **レスポンス時間アサーション**：レスポンス時間をチェック
- **レスポンスヘッダーアサーション**：レスポンスヘッダーをチェック

### 例

```javascript
// ステータスコードアサーション
pm.test("ステータスコードが 200", function() {
    pm.response.to.have.status(200);
});

// JSON アサーション
pm.test("ユーザーリストが返ってくる", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// レスポンス時間アサーション
pm.test("レスポンス時間が 500ms 未満", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### テストスイート

複数の接口のテストケースをテストスイートにまとめられます：

1. テストスイートを作成
2. テストケースを追加
3. 実行順序を設定
4. スイートを実行

### テストレポート生成

テスト実行後、詳細なテストレポートを生成できます：

- 成功率の統計
- 失敗ケースの分析
- レスポンス時間の統計

## チームコラボレーション

### メンバー招待

```bash
# チームメンバーを招待
1. プロジェクト設定に入る
2. 「メンバー管理」をクリック
3. 「メンバー招待」をクリック
4. メンバーのメールアドレスを入力
5. 役割と権限を設定
6. 招待を送信
```

### 役割と権限

- **管理者**：プロジェクトの全権限
- **編集者**：接口の編集とテスト実行が可能
- **閲覧者**：閲覧のみ、編集不可

### リアルタイム同期

チームメンバーがログインすると、プロジェクトデータは自動的にリアルタイム同期されます：

- 接口の変更は自動同期
- テスト結果も自動同期
- チームメンバーは常に最新内容を確認できる

### コメント機能

接口にコメントを追加できます：

1. 接口を開く
2. 「コメント」をクリック
3. コメント内容を入力
4. チームメンバーを @ でメンション

## ショートカットキー

よく使うキーボードショートカット：

- `Ctrl/Cmd + S`：保存
- `Ctrl/Cmd + Enter`：リクエスト送信
- `Ctrl/Cmd + B`：JSON を整形
- `Ctrl/Cmd + /`：コメント/コメント解除

## ベストプラクティス

1. **命名規則を守る**：接口の名前は機能を明確に説明すること
2. **コメントをが充実**：全てのパラメータに説明を追加
3. **適切にグループ化**：模块ごとに接口を整理
4. **環境を活用**：開発、テスト、本番環境を 구분
5. **テストを書く**：重要な接口には自動化テストを作成
6. **定期的に同期**：ローカルとクラウドのデータを合わせる
