# Preguntas Frecuentes de OpenClaw

Esta página lista preguntas comunes y soluciones al usar OpenClaw.

## Instalación

### ¿Cuáles son los requisitos de OpenClaw?

- Node.js 22.x o superior
- macOS, Linux o Windows (WSL2)
- Al menos 2GB de RAM recomendados

### La instalación falla. ¿Qué debo hacer?

1. Verificar versión de Node.js: `node --version` (debe ser >= 22)
2. Verificar conexión de red
3. Intentar diferentes gestores de paquetes: npm, pnpm o bun
4. Ejecutar `openclaw doctor` para diagnosticar

### ¿Cómo actualizo OpenClaw?

```bash
# Cambiar a canal estable
openclaw update --channel stable

# Luego reiniciar Gateway
openclaw gateway restart
```

## Gateway

### Gateway no inicia

- Verificar si el puerto 18789 está ocupado
- Revisar logs: `openclaw gateway --verbose`
- Intentar matar procesos existentes

### ¿Cómo mantener Gateway funcionando constantemente?

Usar `--install-daemon` para instalar como servicio del sistema:

```bash
openclaw onboard --install-daemon
```

### ¿Cómo accedo a Gateway remotamente?

1. **Tailscale**: Usar `gateway.tailscale.mode: "serve"` o `"funnel"`
2. **Túnel SSH**: Establecer conexión de túnel SSH

## Canales

### Login de WhatsApp falla

- Asegúrate de haber emparejado tu dispositivo con `openclaw channels login`
- Verificar si la red puede alcanzar los servidores de WhatsApp

### Bot de Telegram no funciona

- Verificar que el Token del Bot sea correcto
- Verificar que hayas iniciado un chat con el Bot
- Verificar configuración de `allowFrom`

### Mensajes de Discord no responden

- Verificar si el Bot se ha unido al servidor
- Confirmar que los permisos correctos están configurados
- Verificar si el canal está en la lista de permitidos

## Seguridad

### ¿Pueden extraños enviarme mensajes?

Por defecto, OpenClaw requiere verificación de emparejamiento para remitentes desconocidos. Usar `dmPolicy: "open"` para cambiar a modo abierto, pero no recomendado.

### ¿Cómo gestionar emparejamiento?

```bash
# Ver lista de emparejamientos
openclaw pairing list

# Aprobar emparejamiento
openclaw pairing approve

# Revocar emparejamiento
openclaw pairing revoke
```

### Ejecutar `openclaw doctor` muestra advertencias?

Seguir las instrucciones para arreglar. Problemas comunes:
- Política de DM muy abierta
- Configuración de autenticación requerida faltante
- Riesgos de exposición de puertos

## Habilidades

### ¿Cómo instalar nuevas habilidades?

```bash
# Instalar vía ClawHub
openclaw skill install nombre-de-habilidad

# O crear manualmente directorio de habilidad y SKILL.md
```

### ¿Habilidades no funcionan?

- Verificar que la configuración de habilidad sea correcta
- Revisar logs de Gateway
- Confirmar que las dependencias de la habilidad estén instaladas

## Voz

### Voice Wake no funciona?

- macOS: Verificar permisos de Accesibilidad en Preferencias del Sistema
- iOS: Confirmar que el permiso de micrófono esté habilitado
- Verificar si la palabra de activación está configurada correctamente

### Respuesta de voz muy lenta?

- Verificar latencia de red
- Considerar usar TTS local
- Ajustar configuración del modelo de voz

## Rendimiento

### OpenClaw usa mucha memoria?

- Limpiar historial de sesiones regularmente
- Ajustar configuración del modelo
- Usar compresión de sesión

### Mensajes responden lentamente?

- Verificar conexión de red
- Intentar cambiar a un modelo más rápido
- Verificar carga de Gateway

## Datos

### ¿Dónde se almacenan los datos?

- Configuración: `~/.openclaw/`
- Espacio de trabajo: `~/.openclaw/workspace`
- Credenciales: `~/.openclaw/credentials`

### ¿Cómo respaldar datos?

```bash
# Respaldar directorio de configuración completo
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### ¿Cómo reiniciar todas las configuraciones?

1. Detener Gateway
2. Eliminar directorio `~/.openclaw`
3. Ejecutar `openclaw onboard` nuevamente

## Otros

### ¿Cómo ver logs?

```bash
# Ver logs de Gateway
openclaw gateway --verbose

# O revisar archivos de logs específicos
openclaw logs
```

### ¿Cómo obtener ayuda?

- Documentación: https://docs.openclaw.ai
- Unirse a la comunidad Discord
- Enviar Issue en GitHub

### ¿Cuál es la diferencia entre OpenClaw y otros asistentes de IA?

- **Completamente local**: Tus datos permanecen locales, no se suben a la nube
- **Código abierto**: El código es completamente abierto, puedes revisarlo tú mismo
- **Altamente personalizable**: Puedes modificar y extender cualquier función
- **Multi-canal unificado**: Un asistente gestiona todos los canales de comunicación

### ¿Puedo ejecutar múltiples instancias de OpenClaw simultáneamente?

Sí, cada instancia necesita:
- Puerto diferente
- Espacio de trabajo independiente
- Configuración diferente
