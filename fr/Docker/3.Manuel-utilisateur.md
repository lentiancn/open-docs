# Manuel Utilisateur de Docker

Ce manuel vous aidera à commencer rapidement avec Docker et à maîtriser les opérations courantes pour une utilisation quotidienne.

## Prérequis

Avant de commencer, assurez-vous d'avoir :

- Docker installé (Docker Desktop ou Docker Engine)
- Accès au terminal de ligne de commande
- Connaissances de base des opérations en ligne de commande

## Concepts de Base

### Images et Conteneurs

- **Image** : Un modèle en lecture seule utilisé pour créer des conteneurs
- **Conteneur** : Une instance en cours d'exécution d'une image

### Commandes Courantes

```bash
# Exécuter un conteneur
docker run hello-world

# Lister tous les conteneurs
docker ps -a

# Lister toutes les images
docker images

# Arrêter un conteneur
docker stop id_conteneur

# Supprimer un conteneur
docker rm id_conteneur

# Supprimer une image
docker rmi id_image
```

## Travailler avec des Conteneurs

### Exécuter un Conteneur

La commande de base pour exécuter un conteneur est :

```bash
docker run [options] nom_image [commande]
```

Exemples :

```bash
# Démarrer un conteneur interactif
docker run -it ubuntu bash

# Exécuter un conteneur en arrière-plan
docker run -d nginx

# Exécuter un conteneur avec redirection de ports
docker run -p 8080:80 nginx
```

### Gérer les Conteneurs

```bash
# Afficher tous les conteneurs en cours d'exécution
docker ps

# Afficher tous les conteneurs (y compris les arrêtés)
docker ps -a

# Démarrer un conteneur
docker start nom_conteneur

# Arrêter un conteneur
docker stop nom_conteneur

# Redémarrer un conteneur
docker restart nom_conteneur

# Supprimer un conteneur
docker rm nom_conteneur
```

### Afficher les Logs des Conteneurs

```bash
# Afficher les logs d'un conteneur
docker logs nom_conteneur

# Suivre les logs en temps réel
docker logs -f nom_conteneur

# Afficher les 100 dernières lignes
docker logs --tail 100 nom_conteneur
```

## Travailler avec des Images

### Rechercher et Télécharger des Images

```bash
# Rechercher des images sur Docker Hub
docker search nginx

# Télécharger une image
docker pull nginx

# Télécharger une image avec un tag
docker pull nginx:latest
```

### Créer des Images

Vous pouvez créer vos propres images avec un Dockerfile :

```dockerfile
FROM ubuntu:20.04
RUN apt-get update
RUN apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Puis créez l'image :

```bash
docker build -t mon-image-nginx .
```

### Gérer les Images

```bash
# Lister toutes les images
docker images

# Supprimer une image
docker rmi nom_image

# Supprimer les images inutilisées
docker image prune
```

## Utiliser des Volumes

Les volumes sont utilisés pour stocker des données en dehors des conteneurs.

```bash
# Créer un volume
docker volume create mon_volume

# Monter un volume
docker run -v mon_volume:/donnees nginx

# Lister tous les volumes
docker volume ls

# Supprimer un volume
docker volume rm mon_volume
```

## Réseaux

Docker offre des fonctionnalités de réseau pour la communication entre conteneurs.

```bash
# Créer un réseau
docker network create mon_reseau

# Connecter un conteneur à un réseau
docker run --network mon_reseau nginx

# Lister tous les réseaux
docker network ls
```

## Docker Compose

Docker Compose permet de définir et d'exécuter des applications multi-conteneurs.

### Exemple docker-compose.yml

```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: motdepasse
```

### Commandes

```bash
# Démarrer tous les conteneurs
docker-compose up

# Démarrer en arrière-plan
docker-compose up -d

# Arrêter tous les conteneurs
docker-compose down
```

## Conseils et Meilleures Pratiques

### Sécurité

- Utilisez des images officielles quand possible
- N'exécutez pas les conteneurs en tant que root
- Gardez les images à jour
- Analysez les images pour les vulnérabilités

### Performance

- Utilisez des fichiers .dockerignore
- Utilisez des constructions multi-étapes
- Minimisez le nombre de couches dans les images
- Utilisez des tags d'image spécifiques

### Maintenabilité

- Utilisez des noms de conteneurs significatifs
- Documentez vos Dockerfiles
- Utilisez des volumes pour les données persistentes
- Enregistrez l'activité des conteneurs
