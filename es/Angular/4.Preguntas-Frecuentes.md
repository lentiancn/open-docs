# Angular Preguntas Frecuentes

Preguntas y respuestas comunes.

## Instalación

### Q: ¿La instalación de Angular CLI falló?

**Soluciones:**
1. Verificar versión de Node.js
```bash
node --version  # Requiere 18+ o 20+
```
2. Limpiar caché de npm
```bash
npm cache clean --force
```
3. Usar administrador (Windows)
4. Verificar conexión de red

### Q: ¿Comando ng no encontrado?

**Soluciones:**
1. Confirmar que Angular CLI está instalado
```bash
npm list -g @angular/cli
```
2. Verificar variable de entorno PATH
3. Reinstalar
```bash
npm uninstall -g @angular/cli
npm install -g @angular/cli@latest
```

### Q: ¿La creación del proyecto falló?

**Soluciones:**
1. Verificar espacio en disco
2. Verificar permisos de npm
3. Usar --skip-install
```bash
ng new my-app --skip-install
cd my-app
npm install
```

## Desarrollo

### Q: ¿Componente no se muestra?

**Soluciones:**
1. Confirmar que el componente está importado correctamente
2. Confirmar que el selector se usa correctamente
3. Verificar sintaxis de plantilla
4. Verificar errores en consola

### Q: ¿Datos no se actualizan?

**Soluciones:**
1. Para Signals, usar métodos set/update
```typescript
count.set(1);           // Establecer nuevo valor
count.update(c => c + 1); // Actualizar basado en valor anterior
```
2. Para RxJS, asegurar suscripción correcta
3. Usar ChangeDetectorRef para detección manual
```typescript
constructor(private cdr: ChangeDetectorRef) {}

refresh() {
  this.cdr.detectChanges();
}
```

### Q: ¿Enrutamiento no funciona?

**Soluciones:**
1. Confirmar que la configuración de rutas es correcta
2. Confirmar que RouterOutlet está colocado
3. Verificar orden de rutas
```typescript
// Rutas específicas antes de comodín
{ path: 'user/:id', ... },
{ path: '**', component: NotFoundComponent }
```

### Q: ¿Validación de formulario no funciona?

**Soluciones:**
1. Importar ReactiveFormsModule o FormsModule
2. Confirmar que los nombres de FormControl son correctos
3. Usar validadores correctamente
```typescript
this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', [Validators.required, Validators.minLength(6)]]
});
```

## Rendimiento

### Q: ¿Carga inicial lenta?

**Soluciones:**
1. Usar lazy loading de rutas
```typescript
{
  path: 'admin',
  loadComponent: () => import('./admin/admin.component')
    .then(m => m.AdminComponent)
}
```
2. Habilitar modo producción
```bash
ng build --configuration=production
```
3. La compilación AOT está habilitada por defecto

### Q: ¿Fugas de memoria?

**Soluciones:**
1. Cancelar suscripción correctamente
```typescript
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

@Component({...})
export class MyComponent implements OnDestroy {
  private destroy$ = new Subject<void>();

  ngOnInit() {
    this.http.get('/api').pipe(
      takeUntil(this.destroy$)
    ).subscribe();
  }

  ngOnDestroy() {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
```
2. Usar pipe async para suscripción/cancelación automática

### Q: ¿Bundle muy grande?

**Soluciones:**
1. Verificar tamaños de dependencias
```bash
npm run build -- --stats-json
npx webpack-bundle-analyzer dist/stats.json
```
2. Usar lazy loading
3. Eliminar imports no utilizados

## Pruebas

### Q: ¿Cómo escribir pruebas unitarias?

**Soluciones:**
```typescript
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { UserCardComponent } from './user-card.component';

describe('UserCardComponent', () => {
  let component: UserCardComponent;
  let fixture: ComponentFixture<UserCardComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [UserCardComponent]
    }).compileComponents();

    fixture = TestBed.createComponent(UserCardComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display user name', () => {
    component.user = { name: 'Juan', email: 'test@example.com' };
    fixture.detectChanges();
    const compiled = fixture.nativeElement as HTMLElement;
    expect(compiled.querySelector('h3')?.textContent).toContain('Juan');
  });
});
```

**Soluciones:**

    CommonModule,

## Build

### Q: ¿Build falló?

**Soluciones:**
1. Verificar errores de TypeScript
```bash
npx tsc --noEmit
```
2. Limpiar caché
```bash
rm -rf node_modules
npm install
```
3. Actualizar versión de Angular
```bash
ng update @angular/core @angular/cli
```

### Q: ¿Variables de entorno no funcionan?

**Soluciones:**
1. Confirmar que nombres de archivos de entorno son correctos
2. Configurar entorno en angular.json
```json
"configurations": {
  "production": {
    "fileReplacements": [
      {
        "replace": "src/environments/environment.ts",
        "with": "src/environments/environment.prod.ts"
      }
    ]
  }
}
```

## Seguridad

### Q: ¿Cómo prevenir ataques XSS?

**Soluciones:**
1. Angular escapa automáticamente por defecto
2. Tener cuidado con innerHTML
```typescript
import { DomSanitizer } from '@angular/platform-browser';

constructor(private sanitizer: DomSanitizer) {}

getHtml() {
  // Solo para contenido confiable
  return this.sanitizer.bypassSecurityTrustHtml(this.trustedContent);
}
```

### Q: ¿Protección CSRF?

**Soluciones:**
```typescript
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
    HttpClientXsrfModule.withOptions({
      cookieName: 'XSRF-TOKEN',
      headerName: 'X-XSRF-TOKEN'
    })
  ]
})
export class AppModule {}
```

## Actualización

### Q: ¿Cómo actualizar versión de Angular?

**Soluciones:**
1. Usar ng update
```bash
ng update @angular/core@17 @angular/cli@17
```
2. Verificar guía de migración
3. Actualizar versiones de dependencias
4. Ejecutar pruebas para asegurar compatibilidad

### Q: ¿Código incompatible después de actualización?

**Soluciones:**
1. Verificar guía oficial de actualización
2. Usar ng update para migración automática
3. Actualizar incrementalmente, no saltar versiones
