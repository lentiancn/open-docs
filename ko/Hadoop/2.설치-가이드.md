# Hadoop 설치 가이드

## 요구 사항

Hadoop을 설치하기 전에 시스템이 기본 하드웨어 및 소프트웨어 요구 사항을 충족하는지 확인하세요. Hadoop 클러스터는 여러 컴퓨터로 구성하거나 단일 모드 또는 pseudo-distributed 모드에서 실행할 수 있습니다. 서로 다른 배포 모드는 서로 다른 하드웨어 요구 사항이 있지만 핵심 소프트웨어 의존성은 유사합니다. 이러한 요구 사항을 이해하면 Hadoop 클러스터의 규모와 구성을 계획하는 데 도움이 되며 설치 과정에서 불필요한 문제를 피할 수 있습니다.

하드웨어의 경우 소규모 테스트 클러스터는 각 컴퓨터에 최소 4코어 CPU, 8GB RAM, 500GB 하드디스크를 갖추어야 합니다. 프로덕션 환경에서는 각 컴퓨터에 8코어 이상 CPU, 16GB 이상 RAM, 1TB 이상 하드디스크를 갖추어야 합니다. Hadoop의 DataNode는 대량 데이터를 저장하므로 디스크 용량이 특히 중요합니다. Hadoop은 RAID 구성 또는 JBOD(Just a Bunch Of Disks) 구성을 권장합니다. 후자는 Hadoop이 디스크 공간을 더 유연하게 관리할 수 있게 합니다.

운영 체제의 경우 Hadoop은 Linux(Ubuntu, CentOS, Debian, Red Hat 등)와 Windows(Cygwin 또는 Windows Subsystem for Linux를 통해)를 지원합니다. 프로덕션 환경에서는 최고 성능과 안정성을 위해 Linux 시스템 사용을 강력히 권장합니다. Hadoop은 다양한 클라우드 플랫폼과 잘 통합되어 있으며 AWS, Azure, Google Cloud 등의 클라우드 서비스에 배포할 수 있습니다.

Java는 Hadoop의 유일한 런타임 의존성입니다. Hadoop 3.x는 Java 8 이상이 필요합니다. Oracle JDK 또는 OpenJDK를 권장합니다. Hadoop을 설치하기 전에 Java가 올바르게 설치되어 있고 JAVA_HOME 환경 변수가 설정되어 있는지 확인하세요. 다음 명령으로 Java 설치를 확인하세요:

```bash
java -version
echo $JAVA_HOME
```

네트워크는 Hadoop 클러스터의 기반입니다. 클러스터의 모든 노드는 네트워크를 통해 서로 통신할 수 있어야 하며, Gigabit Ethernet과 같은 고속 네트워크를 사용하는 것이 좋습니다. Hadoop이 노드 간에 작업을 분배하고 데이터를 전송할 수 있도록 노드에 SSH 비밀번호 없는 로그인이 구성되어 있어야 합니다. 또한 /etc/hosts 파일에서 모든 노드의 호스트 이름과 IP 주소 매핑을 구성하세요.

## 설치 단계

### 독립형 모드 설치

독립형 모드는 Hadoop의 가장 간단한 설치 방식으로, 모든 구성 요소가 동일한 JVM 프로세스에서 실행됩니다. 이 모드는 주로 개발 및 테스트에 사용되며 분산 스토리지나 컴퓨팅을 포함하지 않습니다. 먼저 Apache Hadoop 공식 웹사이트에서 Hadoop 바이너리 배포를 다운로드하세요:

```bash
# Hadoop 다운로드 (최신 버전으로 교체)
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# 지정된 디렉토리에 압축 해제
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# 디렉토리 이름 변경 (선택 사항)
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

다음으로 환경 변수를 구성하세요. ~/.bashrc 또는 /etc/profile 파일을 편집하고 다음을 추가하세요:

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # 실제 Java 경로에 따라 조정
```

구성을 적용하세요:

```bash
source ~/.bashrc
```

설치를 확인하세요:

```bash
hadoop version
```

### 의사 분산 모드 설치

의사 분산 모드는 단일 시스템에서 분산 환경을 시뮬레이션하며 NameNode, DataNode, ResourceManager, NodeManager가 모두 같은 시스템에서 실행됩니다. 이 모드는 분산 기능 개발 및 테스트에 적합합니다.

먼저 Hadoop 핵심 파일을 구성하세요. Hadoop 구성 디렉토리로 이동하세요:

```bash
cd $HADOOP_HOME/etc/hadoop
```

기본 파일 시스템으로 HDFS를 지정하여 core-site.xml을 구성하세요:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

HDFS 복제 계수(의사 분산 모드에서는 1)를 지정하여 hdfs-site.xml을 구성하세요:

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

MapReduce에 대한 리소스 스케줄링 프레임워크를 지정하여 mapred-site.xml을 구성하세요:

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

YARN 관련 속성을 구성하여 yarn-site.xml을 구성하세요:

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### SSH 비밀번호 없는 로그인 구성

Hadoop 클러스터의 노드에는 마스터 노드가 슬레이브 노드에 비밀번호 없이 액세스할 수 있도록 SSH 비밀번호 없는 로그인이 구성되어 있어야 합니다:

```bash
# SSH 키 쌍 생성
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# 공개 키를 authorized_keys에 추가
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# 올바른 권한 설정
chmod 0600 ~/.ssh/authorized_keys

# SSH 로그인 테스트
ssh localhost
```

멀티노드 클러스터의 경우 공개 키를 모든 노드에 복사하세요:

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### HDFS 포맷 및 시작

처음으로 HDFS를 사용하기 전에 NameNode를 포맷하세요:

```bash
hdfs namenode -format
```

HDFS 및 YARN을 시작하세요:

```bash
start-dfs.sh
start-yarn.sh
```

또는 start-all.sh를 사용하여 모든 서비스를 한 번에 시작하세요:

```bash
start-all.sh
```

서비스 시작 상태를 확인하세요:

```bash
# Java 프로세스 보기
jps

# 예상 프로세스:
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Web UI를 통해 Hadoop 서비스에 액세스할 수 있습니다: NameNode는 http://localhost:9870, ResourceManager는 http://localhost:8088입니다.

## 완전 분산 클러스터 설치

### 클러스터 계획

완전 분산 모드에는 클러스터를 형성하기 위해 여러 물리적 또는 가상 머신이 필요합니다. 설치 전에 적절한 클러스터 계획을 수행하세요. 클러스터 규모(노드 수)를 결정하고 각 노드에 역할을 할당하세요(NameNode, Secondary NameNode, DataNode, ResourceManager, NodeManager). 소규모 클러스터에서는 NameNode와 ResourceManager를 동일한 시스템에 배포할 수 있습니다. 대규모 클러스터에서는 더 나은 성능을 위해 별도로 배포하는 것이 좋습니다.

네트워크 계획도 중요합니다. 모든 노드가 낮은 지연 시간과 높은 대역폭을 가진 내부 네트워크에 있어야 합니다. /etc/hosts 파일에서 모든 노드의 호스트 이름과 IP 주소를 구성하세요:

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### 구성 파일 수정

완전 분산 모드에서는 더 많은 구성 파일을 수정해야 합니다. 먼저 workers 파일(이전 버전에서는 slaves 파일)을 구성하여 모든 DataNode와 NodeManager 노드를 나열하세요:

```
slave1
slave2
slave3
```

의사 분산 모드와 유사한 core-site.xml 및 hdfs-site.xml 구성이지만 더 많은 속성을 추가해야 합니다. NameNode HA(고가용성) 구성의 경우 두 개의 NameNode를 구성하세요:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

NameNode HA에 대한 hdfs-site.xml을 구성하세요:

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

YARN 구성도 유사합니다. yarn-site.xml에서 ResourceManager HA를 구성할 수 있습니다.

### 구성 파일 동기화

구성이 완료된 Hadoop 설치 디렉토리를 모든 노드에 동기화하세요. scp를 사용하여 개별적으로 복사하세요:

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

또는 rsync를 사용하여 분산 복제를 수행하세요:

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

## 확인 및 테스트

### HDFS 테스트

HDFS를 테스트하기 위해 디렉토리를 만들고 파일을 업로드하세요:

```bash
# 디렉토리 생성
hdfs dfs -mkdir -p /user/test

# 로컬 파일을 HDFS에 업로드
hdfs dfs -put localfile.txt /user/test/

# 파일 나열
hdfs dfs -ls /user/test

# 파일 읽기
hdfs dfs -cat /user/test/localfile.txt

# 파일 삭제
hdfs dfs -rm /user/test/localfile.txt
```

### MapReduce 테스트

MapReduce 샘플 프로그램을 실행하여 클러스터 기능을 확인하세요:

```bash
# WordCount 샘플 실행
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# 결과 보기
hdfs dfs -cat /output/part-r-00000
```

### 클러스터 상태 확인

Hadoop의 내장 명령을 사용하여 클러스터 상태를 확인하세요:

```bash
# HDFS 상태 보기
hdfs dfsadmin -report

# YARN 노드 상태 보기
yarn node -list -all

# 애플리케이션 상태 보기
yarn application -list
```

## 일반적인 문제

### Java 버전 호환성 문제

Java 버전 관련 오류가 발생하면 설치된 Java 버전이 Hadoop 버전과 호환되는지 확인하세요. Hadoop 3.x는 Java 8 이상이 필요합니다. 다음 명령으로 Java 버전을 전환하세요:

```bash
update-alternatives --config java
```

### 디스크 공간 부족

DataNode는 디스크 공간 부족으로 인해하지 않을 수 있습니다. 정기적으로 디스크 사용량을 모니터링하여 DataNode에 충분한 여유 공간이 있는지 확인하세요. hdfs-site.xml에서 dfs.datanode.fsdataset.volume.choosing.policy를 AvailableSpaceVolumeChoosingPolicy로 구성하여 DataNode가 여유 공간이 더 많은 디스크를 우선 사용하도록 할 수 있습니다.

### 메모리 부족

클러스터 노드의 메모리가 부족하면 YARN이 컨테이너를 시작하지 못할 수 있습니다. yarn-site.xml에서 메모리 구성을 조정하세요:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
