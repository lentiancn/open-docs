# OpenFeign 소개

## OpenFeign이란?

OpenFeign은 웹 서비스 클라이언트 작성을 더 쉽게 만들어주는 선언형 웹 서비스 클라이언트입니다. Retrofit, JAX-RS-2.0, WebSocket에서 영감을 받아 HTTP API에 Denominator를 균일하게 바인딩하는 복잡성을 줄이는 것이 초기 목표였습니다.

### 핵심 설계 철학

OpenFeign의 핵심 설계 철학은 **선언형 프로그래밍**입니다. 개발자는 인터페이스와 애노테이션만 정의하면 되며 구체적인 HTTP 요청 코드를 작성할 필요가 없습니다. 이 설계 패턴은 마이크로서비스 간 호출을 크게 단순화하여 코드를 더욱 간결하고 읽기 쉽고 유지보수하기 쉽게 만듭니다.

## 주요 기능

### 1. 선언형 인터페이스

간단한 인터페이스 정의와 애노테이션을 통해 기능이 완전한 HTTP 클라이언트를 생성할 수 있습니다:

```java
@FeignClient(name = "user-service")
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
    
    @PostMapping("/users")
    User createUser(@RequestBody User user);
}
```

### 2. Spring Cloud 통합

Spring Cloud OpenFeign은 네이티브 Feign 기능을 확장하여 Spring 생태계와 깊이 통합합니다:

- **Spring MVC 애노테이션 지원**: `@RequestMapping`, `@GetMapping`, `@PostMapping` 등 지원
- **HttpMessageConverters**: Spring Web의 기본 HttpMessageConverters 사용
- **서비스 디스커버리 통합**: Eureka, Consul 등의 서비스 디스커버리 컴포넌트와 원활한 통합
- **로드 밸런싱**: 내장된 Spring Cloud LoadBalancer 지원
- **서킷 브레이커**: Spring Cloud CircuitBreaker와의 통합으로 장애 허용 능력 제공

### 3. 플러그인 가능한 아키텍처

OpenFeign은 플러그인 가능한 아키텍처 설계를 채택하여 개발자가 다양한 컴포넌트를 사용자 정의할 수 있도록 합니다:

- **인코더 (Encoder)**: 요청 본문의 직렬화 제어
- **디코더 (Decoder)**: 응답 본문의 역직렬화 제어
- **계약 (Contract)**: 애노테이션 처리 규칙 정의
- **인터셉터 (Interceptor)**: 요청 전후에 사용자 정의 로직 실행
- **클라이언트 (Client)**: 하위 HTTP 클라이언트 구현 사용자 정의

### 4. 다양한 데이터 형식 지원

OpenFeign은 다양한 데이터 형식의 인코딩 및 디코딩을 지원합니다:

- **JSON**: Jackson, Gson, Fastjson2 등을 통해 지원
- **XML**: JAXB, SAX 등을 통해 지원
- **폼 데이터**: `application/x-www-form-urlencoded` 및 `multipart/form-data` 지원
- **SOAP**: 전용 SOAP 인코더/디코더를 통해 지원

### 5. 고급 기능

- **요청/응답 압축**: 네트워크 전송을 줄이기 위한 GZIP 압축 지원
- **로깅**: 구성 가능한 로그 레벨을 갖는 내장 로깅 지원
- **OAuth2 지원**: 내장된 OAuth2 인증 지원
- **キャッシング 지원**: Spring Cache와의 통합으로 메서드 레벨 캐싱 지원
- **메트릭 모니터링**: Micrometer와의 통합으로 상세한 성능 메트릭 제공

## 사용 사례

### 마이크로서비스 아키텍처

OpenFeign은 마이크로서비스 아키텍처에 이상적이며 특히 다음에 적합합니다:

- **서비스 간 호출**: 마이크로서비스 간 HTTP 호출 단순화
- **API 게이트웨이**: API 게이트웨이의 백엔드 서비스 호출 클라이언트 역할
- **분산 시스템**: 분산 시스템에서 서비스 호출 통합 관리

### 기존 애플리케이션 통합

- **타사 API 호출**: 외부 RESTful API 또는 SOAP 서비스 호출
- **레거시 시스템 통합**: 기존 웹 서비스와의 통합
- **배치 데이터 처리**: 배치 처리 작업에서 원격 서비스 호출

## 버전 진화

### Feign 9.x
- JDK 6+ 지원
- 기본 기능 안정적

### Feign 10.x+
- Java 8+ 필요
- Java 9, 10, 11+ 지원
- 비동기 지원 (`CompletableFuture`) 도입
- 개선된 플러그인 가능한 아키텍처

### Spring Cloud OpenFeign
- Spring Cloud 생태계와의 깊은 통합
- 엔터프라이즈급 기능(로드 밸런싱, 서킷 브레이커 등) 제공
- Spring Boot 자동 설정과의 원활한 통합
