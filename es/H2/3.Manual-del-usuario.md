# Manual de usuario de H2

## Operaciones básicas

### Crear base de datos

Modo embebido (base de datos en memoria):
```java
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");
```

Base de datos de archivo:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Crear tabla

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Insertar datos

```sql
INSERT INTO users (name, email) VALUES ('Juan', 'juan@ejemplo.com');
INSERT INTO users (name, email) VALUES ('María', 'maria@ejemplo.com');
```

### Consultar datos

```sql
SELECT * FROM users;
SELECT * FROM users WHERE name = 'Juan';
SELECT COUNT(*) FROM users;
```

### Actualizar datos

```sql
UPDATE users SET email = 'nuevoemail@ejemplo.com' WHERE id = 1;
```

### Eliminar datos

```sql
DELETE FROM users WHERE id = 2;
```

## Ejemplos de código Java

### Ejemplo completo de JDBC

```java
import java.sql.*;

public class H2Example {
    public static void main(String[] args) throws Exception {
        // Cargar driver
        Class.forName("org.h2.Driver");
        
        // Conectar a la base de datos
        String url = "jdbc:h2:mem:testdb";
        Connection conn = DriverManager.getConnection(url, "sa", "");
        
        // Crear tabla
        Statement stmt = conn.createStatement();
        stmt.execute("CREATE TABLE users (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100))");
        
        // Insertar datos
        stmt.execute("INSERT INTO users (name) VALUES ('Juan')");
        stmt.execute("INSERT INTO users (name) VALUES ('María')");
        
        // Consultar datos
        ResultSet rs = stmt.executeQuery("SELECT * FROM users");
        while (rs.next()) {
            System.out.println(rs.getInt("id") + ": " + rs.getString("name"));
        }
        
        // Cerrar conexión
        rs.close();
        stmt.close();
        conn.close();
    }
}
```

### Usando PreparedStatement

```java
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name, email) VALUES (?, ?)");
pstmt.setString(1, "Pedro");
pstmt.setString(2, "pedro@ejemplo.com");
pstmt.executeUpdate();
pstmt.close();
```

### Usando transacciones

```java
        conn.setAutoCommit(false);
try {
    Statement stmt = conn.createStatement();
    stmt.execute("INSERT INTO users (name) VALUES ('Luis')");
    stmt.execute("INSERT INTO users (name) VALUES ('Ana')");
    conn.commit();
} catch (Exception e) {
    conn.rollback();
} finally {
    conn.setAutoCommit(true);
}
```

## Características avanzadas

### Cambio de modo de base de datos

H2 puede simular la sintaxis SQL de otras bases de datos:

```java
// Modo MySQL
String url = "jdbc:h2:mem:testdb;MODE=MySQL";

// Modo PostgreSQL
String url = "jdbc:h2:mem:testdb;MODE=PostgreSQL";

// Modo Oracle
String url = "jdbc:h2:mem:testdb;MODE=Oracle";
```

### Cifrado de base de datos

```java
// Usar base de datos cifrada
String url = "jdbc:h2:file:./data/encrypted;CIPHER=AES";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Funcionalidad de seguimiento

```java
// Habilitar seguimiento SQL
String url = "jdbc:h2:mem:testdb;TRACE_LEVEL_FILE=3";
```

### Iniciar servidor TCP

```java
import org.h2.tools.Server;

Server server = Server.createTcpServer().start();
System.out.println("H2 TCP Server started: " + server.getURL());
```

### Usando la consola H2

Iniciar la consola web de H2:

```bash
java -jar h2.jar -web
```

Luego abre en el navegador: http://localhost:8082

## Sentencias SQL comunes

### Tipos de datos

H2 soporta múltiples tipos de datos, incluyendo:
- Tipos enteros: INT, INTEGER, BIGINT, SMALLINT, TINYINT
- Tipos de punto flotante: FLOAT, REAL, DOUBLE, DECIMAL
- Tipos de cadena: VARCHAR, CHAR, CLOB
- Tipos de fecha y hora: DATE, TIME, TIMESTAMP
- Tipos booleanos: BOOLEAN
- Tipos binarios: BLOB, BINARY

### Operaciones de índice

Crear índice:
```sql
CREATE INDEX idx_name ON users(name);
```

Eliminar índice:
```sql
DROP INDEX idx_name;
```

### Operaciones de restricción

Agregar clave primaria:
```sql
ALTER TABLE users ADD PRIMARY KEY (id);
```

Agregar clave foránea:
```sql
ALTER TABLE orders ADD FOREIGN KEY (user_id) REFERENCES users(id);
```

### Ver estructura de tabla

```sql
DESCRIBE users;
-- o
SHOW COLUMNS FROM users;
```

### Ver todas las tablas

```sql
SHOW TABLES;
```

## Optimización del rendimiento

### Operaciones por lotes

Usar operaciones por lotes para mejorar el rendimiento:

```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (int i = 0; i < 1000; i++) {
    pstmt.setString(1, "Usuario" + i);
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Pool de conexiones

En entornos de producción, se recomienda usar un pool de conexiones como HikariCP:

```java
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:h2:file:./data/testdb");
config.setUsername("sa");
config.setPassword("");
config.setMaximumPoolSize(10);

HikariDataSource ds = new HikariDataSource(config);
```

## Respaldo y restauración

### Respaldar base de datos

```sql
SCRIPT TO 'backup.sql';
```

### Restaurar base de datos

```sql
RUNSCRIPT FROM 'backup.sql';
```

## Manejo de errores comunes

### Archivo de base de datos bloqueado
Asegúrate de que no haya otras conexiones usando el archivo de base de datos. H2 soporta lecturas concurrentes, pero la escritura requiere acceso exclusivo.

### Pérdida de datos en memoria
La base de datos en memoria pierde los datos después de cerrar la conexión. Para persistencia, usa el modo de archivo.

### Clase no encontrada
Asegúrate de que la clase org.h2.Driver esté en el classpath.
