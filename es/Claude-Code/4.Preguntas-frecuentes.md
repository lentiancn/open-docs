# Claude Code Preguntas Frecuentes

Esta página enumera los problemas más comunes encontrados durante la instalación y uso de Claude Code, junto con soluciones.

## Problemas de Instalación

### Comando No Encontrado

Después de la instalación, encuentras errores `command not found: claude` o `'claude' is not recognized`, lo que indica que el directorio de instalación no está en tu PATH del sistema.

**Solución:** Verifica y configura tu variable de entorno PATH.

Agregar a .zshrc o .bashrc en macOS/Linux:
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Windows PowerShell:
```powershell
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
```

Luego reinicia tu terminal y verifica:
```bash
claude --version
```

### El Script de Instalación Devuelve HTML

Al ejecutar el comando de instalación, ves errores como:
- `bash: line 1: syntax error near unexpected token '<'`
- `bash: line 1: '<!DOCTYPE html>'`
- En PowerShell: `Invoke-Expression: Missing argument in parameter list`

Esto significa que la URL de instalación devolvió una página HTML en lugar de un script de instalación.

**Solución:**

1. Usa un método de instalación alternativo:
   - macOS/Linux: Usa Homebrew
     ```bash
     brew install --cask claude-code
     ```
   - Windows: Usa WinGet
     ```powershell
     winget install Anthropic.ClaudeCode
     ```

2. Si la página HTML muestra "App unavailable in region", Claude Code no está disponible en tu país/región. Consulta la [lista de países y regiones soportados](https://www.anthropic.com/supported-countries).

3. Espera unos minutos y retry—el problema suele ser temporal.

### Error de Descarga curl

Mensaje de error: `curl: (56) Failure writing output to destination`

Esto significa que la conexión se interrumpió antes de que terminara de descargarse el script.

**Solución:**

1. Verifica la estabilidad de la red:
   ```bash
   curl -fsSL https://storage.googleapis.com -o /dev/null
   ```
   Si el comando se completa exitosamente, tu conexión está bien; solo reintenta el comando de instalación.

2. Usa un método de instalación alternativo (ver arriba).

### Errores de Conexión TLS/SSL

Mensajes de error: `curl: (35) TLS connect error`, `schannel: next InitializeSecurityContext failed`, o `Could not establish trust relationship for the SSL/TLS secure channel`

**Solución:**

1. Actualiza los certificados CA de tu sistema:
   - Ubuntu/Debian:
     ```bash
     sudo apt-get update && sudo apt-get install ca-certificates
     ```
   - macOS (vía Homebrew):
     ```bash
     brew install ca-certificates
     ```

2. Habilita TLS 1.2 en Windows PowerShell:
   ```powershell
   [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
   irm https://claude.ai/install.ps1 | iex
   ```

3. Verifica la interferencia de proxy o firewall; configura la variable de entorno `NODE_EXTRA_CA_CERTS`.

### Error al Obtener Versión

Mensaje de error: `Failed to fetch version from storage.googleapis.com`

Esto significa que `storage.googleapis.com` está bloqueado en tu red.

**Solución:**

1. Prueba conectividad directa:
   ```bash
   curl -sI https://storage.googleapis.com
   ```

2. Si estás detrás de un proxy, establece la variable de entorno `HTTPS_PROXY`.

### Windows Requiere Git Bash

Mensaje de error: `Claude Code on Windows requires git-bash`

**Solución:** Instala [Git for Windows](https://git-scm.com/downloads/win), luego vuelve a ejecutar el comando de instalación.

Si Git ya está instalado pero el error persiste, configura la ruta de Git Bash en tu archivo de configuración:
```json
{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}
```

### Instrucción Ilegal en Linux

Mensaje de error: `Illegal instruction`

Esto generalmente es causado por incompatibilidad de arquitectura. Por favor confirma que la arquitectura de tu CPU es compatible con el binario instalado.

### El Binario No Carga en macOS

Mensajes de error: `dyld: cannot load` o `Abort trap`

Esto significa que el binario es incompatible con tu sistema. Por favor confirma que descargaste la versión correcta de Claude Code (Intel o Apple Silicon).

### Instalación Terminada en Servidores de Baja Memoria

Mensaje de error: Muestra `Killed` durante la instalación en servidores Linux

**Solución:** Agrega espacio swap para servidores de baja memoria.

## Problemas de Autenticación

### Error de OAuth o 403 Forbidden

**Solución:**

1. Verifica si tu cuenta tiene acceso a Claude Code (Pro, Max, Teams, Enterprise o Console)
2. Limpia las credenciales en caché e inicia sesión nuevamente:
   ```bash
   rm -rf ~/.claude
   claude
   ```
3. Verifica si tu región está en la [lista de soporte](https://www.anthropic.com/supported-countries)

### Región No Disponible

Mensaje de error: `App unavailable in region`

Claude Code no está disponible en tu país/región. Consulta [países y regiones soportados por Anthropic](https://www.anthropic.com/supported-countries).

## Configuración de Red

### Verificar Conectividad de Red

El instalador descarga desde `storage.googleapis.com`. Verifica que puedas acceder:

```bash
curl -sI https://storage.googleapis.com
```

Si esto falla, las causas posibles incluyen:
- Firewalls corporativos o proxies bloqueando Google Cloud Storage
- Restricciones de red regionales: Intenta una VPN o red alternativa
- Problemas TLS/SSL: Actualiza certificados CA del sistema, o verifica si `HTTPS_PROXY` está configurado

### Configuración de Proxy

Si estás detrás de un proxy corporativo, establece variables de entorno de proxy antes de instalar:

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

## Verificar Instalaciones en Conflicto

Múltiples instalaciones de Claude Code pueden causar incompatibilidades de versión o comportamiento inesperado.

Verifica qué está instalado:

**macOS/Linux:**
```bash
which -a claude
ls -la ~/.local/bin/claude
npm -g ls @anthropic-ai/claude-code 2>/dev/null
```

**Windows:**
```powershell
where.exe claude
Test-Path "$env:LOCALAPPDATA\Claude Code\claude.exe"
```

Si encuentras múltiples instalaciones, mantén solo una (la instalación nativa en `~/.local/bin/claude` es recomendada).

Remueve instalaciones extra:
- Instalación npm global: `npm uninstall -g @anthropic-ai/claude-code`
- Instalación Homebrew: `brew uninstall --cask claude-code`

## Verificar Permisos de Directorio

El instalador necesita acceso de escritura a `~/.local/bin/` y `~/.claude/`. Verifica si estos directorios son escribibles:

```bash
test -w ~/.local/bin && echo "writable" || echo "not writable"
test -w ~/.claude && echo "writable" || echo "not writable"
```

Si un directorio no es escribible:

```bash
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```

## Verificar Binario

Si `claude` está instalado pero se cuelga o bloquea al iniciar:

Confirma que el binario existe y es ejecutable:
```bash
ls -la $(which claude)
```

En Linux, verifica librerías compartidas faltantes:
```bash
ldd $(which claude) | grep "not found"
```

Ejecuta verificaciones básicas:
```bash
claude --version
```

## Herramientas de Diagnóstico

Usa `claude doctor` para información más detallada de instalación y configuración:

```bash
claude doctor
```
