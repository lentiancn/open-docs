# JIRA 사용 가이드

이 가이드에서는 JIRA를 사용하여 프로젝트 관리와 문제 추적 방법을 설명합니다.

## 빠른 시작

### 프로젝트 만들기

1. "프로젝트 만들기" 클릭
2. 프로젝트 템플릿 선택
3. 프로젝트 이름과 키 입력
4. "만들기" 클릭

## 프로젝트 유형

### 애자일 프로젝트

```markdown
- Scrum 프로젝트
- Kanban 프로젝트
- 스케일 애자일
```

### 클래식 프로젝트

```markdown
- 비즈니스 프로젝트
- 프로젝트 관리
- 작업 추적
```

## 문제 유형

### 문제 만들기

```markdown
1. "문제 만들기" 클릭
2. 프로젝트 선택
3. 문제 유형 선택
4. 제목과 설명 입력
5. 우선순위와 컴포넌트 설정
6. 사용자에게 할당
```

### 문제 유형

- **Bug** - 소프트웨어 결함
- **Story** - 사용자 스토리
- **Task** - 작업
- **Epic** - 에픽
- **Sub-task** - 하위 작업

## 워크플로

### 기본 워크플로

```
만들기 → 할당 → 해결 → 닫기
```

### 애자일 워크플로

```
백로그 → 진행 중 → 완료 → 검토
```

## 애자일 기능

### Kanban

1. Kanban 만들기
2. 열 추가
3. 문제 카드 드래그
4. WIP 제한 설정

### Scrum

```markdown
- 스프린트 계획
- 데일리 스탠드업
- 스프린트 리뷰
- 스프린트 회고
```

## 필드

### 사용자 정의 필드

1. 관리 → 필드로 이동
2. 사용자 정의 필드 추가
3. 필드 옵션 구성

### 필드 구성

```markdown
- 필수 필드
- 기본값
- 필드 권한
```

## 권한

### 프로젝트 권한

- 프로젝트 탐색
- 문제 만들기
- 문제 편집
- 문제 삭제
- 문제 할당

### 문제 보안 수준

- 공개
- 프로젝트 멤버
- 특정 사용자
- 특정 그룹

## 자동화

### 규칙 만들기

1. 프로젝트 설정 → 자동화로 이동
2. 규칙 만들기
3. 트리거 구성
4. 작업 설정

### 자동화 예시

```
트리거: 문제 생성
조건: 유형 = Bug
작업: 할당 = 담당자
```
