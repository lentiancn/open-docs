# JIRA ユーザーガイド

このガイドでは、JIRAを使用してプロジェクト管理と問題追跡を行う方法について説明します。

## クイックスタート

### プロジェクトの作成

1. 「プロジェクトを作成」をクリック
2. プロジェクトテンプレートを選択
3. プロジェクト名とキーを入力
4. 「作成」をクリック

## プロジェクトタイプ

### アジャイルプロジェクト

```markdown
- Scrumプロジェクト
- カンバンプロジェクト
- スケールド・アジャイル
```

### クラシックプロジェクト

```markdown
- ビジネスプロジェクト
- プロジェクト管理
- タスク追跡
```

## イシュータイプ

### イシューの作成

```markdown
1. 「イシューを作成」をクリック
2. プロジェクトを選択
3. イシュータイプを選択
4. タイトルと説明を入力
5. 優先度とコンポーネントを設定
6. ユーザーに割り当て
```

### イシュータイプ

- **Bug** - ソフトウェア欠陥
- **Story** - ユーザーストーリー
- **Task** - タスク
- **Epic** - エピック
- **Sub-task** - サブタスク

## ワークフロー

### デフォルトワークフロー

```
作成 → 割り当て → 解決 → 完了
```

### アジャイルワークフロー

```
バックログ → 進行中 → 完了 → レビュー
```

## アジャイル機能

### カンバン

1. カンバンを作成
2. 列を追加
3. イシューカードをドラッグ
4. WIP制限を設定

### Scrum

```markdown
- スプリントプランニング
- デイリースタンドアップ
- スプリントレビュー
- スプリントレトロスペクティブ
```

## フィールド

### カスタムフィールド

1. 管理 → フィールドに移動
2. カスタムフィールドを追加
3. フィールドオプションを設定

### フィールド設定

```markdown
- 必須フィールド
- デフォルト値
- フィールド権限
```

## 権限

### プロジェクト権限

- プロジェクトの閲覧
- イシューの作成
- イシューの編集
- イシューの削除
- イシューの割り当て

### イシューセキュリティレベル

- 公開
- プロジェクトメンバー
- 特定のユーザー
- 特定のグループ

## 自動化

### ルールを作成

1. プロジェクト設定 → 自動化に移動
2. ルールを作成
3. トリガーを設定
4. アクションを設定

### 自動化例

```
トリガー: イシュー作成
条件: タイプ = Bug
アクション: 割り当て先 = 責任者
```

## レポート

### アジャイルレポート

- バーンダウングラフ
- ベロシティグラフ
- 累積フロー図
- カンバン統計

### プロジェクトレポート

- イシュー統計
- 作業量分布
- トレンド分析

## 統合

### Git 統合

1. Gitプラグインをインストール
2. リポジトリを設定
3. コミットとイシューを関連付け

### CI/CD 統合

- Jenkins
- Bamboo
- GitHub Actions

## ベストプラクティス

### 1. ワークフローを適切に設定

```
作成 → レビュー → 開発 → テスト → 完了
```

### 2. バージョンを使用

- バージョン計画
- マイルストーン管理
- リリースノート

### 3. 定期的にクリーンアップ

- 古いイシューをアーカイブ
- 無効なイシューを清理
