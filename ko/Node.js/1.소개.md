# Node.js 소개

Node.js®는 개발자가 서버, 웹 애플리케이션, 명령줄 도구 및 스크립트를 생성할 수 있는 무료 오픈 소스 크로스 플랫폼 JavaScript 런타임 환경입니다.

## 핵심 기능

비동기 이벤트 기반 JavaScript 런타임으로서 Node.js는 확장 가능한 네트워크 애플리케이션을 구축하도록 설계되었습니다. 클래식 "hello world" 예제에서는 많은 연결을 동시에 처리할 수 있습니다. 각 연결 시 콜백이 실행되지만 작업할 내용이 없으면 Node.js는 절전 모드로 전환됩니다.

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

## 설계 철학

이는 오늘날 더 일반적인 동시성 모델인 운영체제 스레드를 사용하는 모델과 대조됩니다. 스레드 기반 네트워킹은 상대적으로 비효율적이며 사용하기 매우 어렵습니다. 또한 Node.js 사용자는 프로세스 데드락에 대한 걱정 없이 안전합니다. 왜냐하면 잠금(lock)이 없기 때문입니다.

Node.js에서는 거의 모든 함수가 직접 I/O를 수행하지 않으므로 Node.js 표준 라이브러리의 동기 메서드를 사용하지 않는 한 프로세스가 차단되지 않습니다. 아무것도 차단되지 않기 때문에 Node.js에서 확장 가능한 시스템을 개발하는 것이 매우 합리적입니다.

## 이벤트 기반 아키텍처

Node.js는 Ruby의 Event Machine 및 Python의 Twisted와 같은 시스템과 유사한 설계를 가지고 있으며 이들로부터 영향을 받았습니다. Node.js는 이벤트 모델을 한 단계 더 발전시켰습니다. 라이브러리가 아닌 런타임 구성으로 이벤트 루프를 제공합니다.

다른 시스템에서는 항상 이벤트 루프를 시작하는 차단 호출이 있습니다. 일반적으로 스크립트 시작 부분에서 콜백을 통해 동작이 정의되고, 스크립트 끝에서 `EventMachine::run()`과 같은 차단 호출을 통해 서버가 시작됩니다.

Node.js에는 이러한 이벤트 루프 시작 호출이 없습니다. Node.js는 입력 스크립트를 실행한 후 단순히 이벤트 루프에 진입합니다. 더 이상 실행할 콜백이 없을 때 Node.js는 이벤트 루프를 종료합니다. 이 동작은 브라우저 JavaScript와 유사합니다. 즉, 이벤트 루프는 사용자에게 숨겨져 있습니다.

## HTTP 지원

HTTP는 Node.js에서 일급 객체(first-class citizen)이며 스트리밍과 낮은 지연 시간을 염두에 두고 설계되었습니다. 이로 인해 Node.js는 웹 라이브러리나 프레임워크의 기반으로 매우 적합합니다.

## 멀티코어 프로세서 지원

Node.js가 스레드 없이 설계되었다고 해서 환경의 여러 코어를 활용할 수 없는 것은 아닙니다. `child_process.fork()` API를 사용하여 자식 프로세스를 생성할 수 있으며, 이들은 통신이 용이하도록 설계되어 있습니다.

동일한 인터페이스 위에 구축된 `cluster` 모듈을 사용하면 프로세스 간에 소켓을 공유하여 코어 간 부하 분산을 활성화할 수 있습니다.
