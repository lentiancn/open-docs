# Guide d'installation de Jenkins

Jenkins est un serveur d'automatisation open source pour implémenter l'intégration continue et le déploiement continu (CI/CD). Ce guide couvre les méthodes d'installation dans divers environnements.

## Informations de version

| Version | Statut | Description |
|---------|--------|-------------|
| Jenkins 2.x | ✅ Actuelle | Dernière version stable |
| Jenkins 1.x | ❌ Obsolète | Maintenance abandonnée |

## Configuration requise

### Matériel

| Exigence | Minimum | Recommandé |
|----------|---------|------------|
| CPU | 1 cœur | 2+ cœurs |
| Mémoire | 1 Go | 4+ Go |
| Disque | 10 Go | 50+ Go |

### Logiciel

| Logiciel | Version |
|----------|---------|
| Java | JDK 8 ou supérieur |
| Système d'exploitation | Windows/macOS/Linux |

## Méthodes d'installation

### Méthode 1 : Installation sur Windows

#### 1. Utiliser l'installateur

1. Télécharger Jenkins : https://www.jenkins.io/download/
2. Exécuter l'installateur jenkins.msi
3. Suivre l'assistant d'installation

#### 2. Utiliser Chocolatey

```powershell
choco install jenkins
```

#### 3. Installation manuelle

```powershell
# Télécharger le fichier WAR
# Extraire dans le répertoire spécifié
java -jar jenkins.war
```

### Méthode 2 : Installation sur macOS

#### 1. Utiliser Homebrew

```bash
# Installer Jenkins
brew install jenkins

# Démarrer le service
brew services start jenkins

# Arrêter le service
brew services stop jenkins
```

#### 2. Installation manuelle

```bash
# Télécharger le fichier WAR
wget https://get.jenkins.io/jenkins.war

# Exécuter
java -jar jenkins.war
```

### Méthode 3 : Installation sur Linux

#### Ubuntu/Debian

```bash
# Ajouter la clé
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -

# Ajouter le dépôt
sudo sh -c 'echo deb https://pkg.jenkins.io/debian binary/ > /etc/apt/sources.list.d/jenkins.list'

# Mettre à jour et installer
sudo apt-get update
sudo apt-get install jenkins

# Démarrer le service
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

#### CentOS/RHEL

```bash
# Ajouter le dépôt
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat/jenkins.repo

# Importer la clé
sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

# Installer
sudo yum install jenkins

# Démarrer le service
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

### Méthode 4 : Installation avec Docker

```bash
# Exécuter le conteneur Jenkins
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts
```

#### Docker Compose

```yaml
version: '3'
services:
  jenkins:
    image: jenkins/jenkins:lts
    ports:
      - "8080:8080"
    volumes:
      - jenkins_home:/var/jenkins_home
```

### Méthode 5 : Déploiement du fichier WAR

```bash
# Télécharger le fichier WAR
wget https://get.jenkins.io/war-stable/latest/jenkins.war

# Exécuter
java -jar jenkins.war --httpPort=8080
```

## Configuration post-installation

### Premier accès

1. Ouvrir le navigateur à l'adresse http://localhost:8080
2. Obtenir le mot de passe initial de l'administrateur :
   ```bash
   # Linux/macOS
   sudo cat /var/lib/jenkins/secrets/initialAdminPassword
   
   # Docker
   docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
   ```
3. Installer les plugins recommandés
4. Créer un utilisateur administrateur

### Configurer l'interface en français

1. Aller dans Manage Jenkins → Manage Plugins
2. Installer "Localization: French"
3. Redémarrer Jenkins

## Questions fréquentes

### Échec du démarrage ?

Vérifier si Java est installé :

```bash
java -version
```

### Port occupé ?

Modifier le port :

```bash
java -jar jenkins.war --httpPort=8090
```

### Échec de l'installation des plugins ?

Vérifier la connexion réseau ou utiliser un proxy.
