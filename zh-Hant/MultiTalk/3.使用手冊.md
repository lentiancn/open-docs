# MultiTalk 使用手冊

本手冊詳細介紹 MultiTalk 的使用方法，包括基本操作、參數配置和進階技巧。

## 基礎使用方法

### 準備輸入檔案

MultiTalk 需要準備特定的輸入格式。專案使用 JSON 檔案定義生成參數，包括音訊路徑、圖像路徑和文字提示等資訊。

單人物生成的輸入 JSON 格式如下：需要指定音訊檔案路徑、參考圖像路徑、影片儲存名稱等參數。範例檔案位於 examples 目錄下的 single_example_1.json。

多人對話生成的輸入 JSON 需要包含多個音訊流和對應的人物區域定義。範例檔案位於 examples 目錄下的 multitalk_example_2.json。

### 執行基本生成

安裝配置完成後，進入專案目錄，執行以下命令進行影片生成：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### 參數說明

--ckpt_dir 指定模型權重目錄路徑。--wav2vec_dir 指定音訊編碼器目錄路徑。--input_json 指定輸入參數 JSON 檔案路徑。--sample_steps 指定採樣步數，值越大生成品質越高但速度越慢。--mode 指定生成模式，streaming 為長影片模式，clip 為短影片模式。--use_teacache 是否啟用 TeaCache 加速。--save_file 指定輸出檔案儲存名稱。

## 進階使用技巧

### 調整生成品質

採樣步數是影響生成品質的關鍵參數。預設 40 步可獲得良好效果。如需快速生成，可將步數降至 10 步，這對口型同步影響較小，但會影響動作流暢度和視覺品質。增加步數可提升細節表現。

口型同步強度透過音訊引導係數控制。Audio CFG 值設定在 3 到 5 之間效果最佳。增加該值可獲得更好的口型同步效果。

### 長影片生成

MultiTalk 支援生成長達 15 秒的影片。使用 streaming 模式可以生成較長的影片片段：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

生成超長影片時，不同片段之間可能存在色調差異。將 Audio CFG 設定為 3 可以緩解這種色調變化問題。

### 低記憶體執行

如果記憶體有限，可以使用以下參數降低記憶體佔用：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

新增 --num_persistent_param_in_dit 0 參數可以在單張 RTX 4090 上生成 480P 影片。

### 啟用加速

專案支援多種加速方式來提升生成速度。

TeaCache 加速可在保持品質的同時顯著提升速度。推薦將 --teacache_thresh 設定在 0.2 到 0.5 之間：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### 使用 LoRA 加速

使用 LoRA 可以大幅減少採樣步數。使用 FusioniX LoRA 只需 8 步，使用 lightx2v LoRA 只需 4 步：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## 使用 Gradio 介面

專案提供了 Gradio 網頁介面，方便互動式使用。進入專案目錄，執行：

```bash
python app.py
```

或者使用低記憶體版本：

```bash
python app.py --num_persistent_param_in_dit 0
```

## 整合 TTS 語音合成

MultiTalk 支援直接輸入文字，透過整合 TTS 技術生成語音後再生成影片：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## 多 GPU 配置

生成 720P 高解析度影片需要多 GPU 支援。使用以下命令配置多 GPU：

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## 使用 ComfyUI

專案還支援整合到 ComfyUI 工作流中使用。需要安裝 ComfyUI-WanVideoWrapper 擴充套件，然後載入 MultiTalk 模型進行工作流編排。
