# OpenClaw 사용자 매뉴얼

> OpenClaw 사용에 대한 종합 가이드

---

## 시작하기

### 기본 사용법

OpenClaw는 여러 가지 방법으로 사용할 수 있습니다:

#### 대화형 세션

```bash
# 대화형 대화 시작
openclaw chat
```

#### 명령 실행

```bash
# 단일 명령 실행
openclaw exec "파일 생성 도와주기"

# 더 복잡한 작업 실행
openclaw exec --task "최신 뉴스 조사"
```

### Gateway 관리

대부분의 작업에는 Gateway 서비스가 활성화되어 있어야 합니다:

```bash
# Gateway 시작
openclaw gateway start

# 상태 확인
openclaw gateway status

# Gateway 중지
openclaw gateway stop
```

## 파일 관리

### 파일 읽기

```bash
# read 도구 사용
read /path/to/file.txt
```

### 파일 쓰기

```bash
# 새 파일 생성 또는 덮어쓰기
write --content "파일 내용" --path /path/to/file.txt
```

### 파일 편집

```bash
# 파일 내 정확한 텍스트 대체
edit --path /path/to/file.txt --oldText "이전 텍스트" --newText "새 텍스트"
```

## 브라우저 제어

OpenClaw는 웹 브라우저를 자동화할 수 있습니다:

### 브라우저 시작

```bash
# 격리된 브라우저 시작
browser start --profile openclaw

# Chrome 확장 프로그램 사용
browser start --profile chrome
```

### 브라우저 작업

```bash
# URL로 이동
browser navigate --url https://example.com

# 스크린샷 촬영
browser screenshot --path /output/screenshot.png

# 요소 클릭
browser click --selector "#button-id"
```

## 하위 에이전트

### 하위 에이전트 생성


## 구성 및 개인화

### 성격 사용자 지정

어시스턴트의 성격을 사용자 지정하려면 `SOUL.md`를 편집합니다:

```markdown
# SOUL.md - 당신이是谁

## 핵심 진실

- 진정으로 도움이 되기
- 의견 갖기
- 창의적으로 하기
```

### 도구 구성

특정 도구 설정은 `TOOLS.md`를 편집합니다:

```markdown
# TOOLS.md - 로컬 노트

## SSH 연결
- server1 -> 192.168.1.100
```

## 모범 사례

1. **작업 공간**: 정의된 작업 공간에 파일 유지
2. **보안**: 외부 작업 주의
3. **메모리**: 중요한 정보에는 메모리 파일 사용
4. **Gateway**: 빠른 응답 시간을 위해 Gateway 서비스 활성화 유지

## 고급 기능

### 자동화

반복 작업을 자동화하는 스크립트 만들기:

```bash
# 자동화된 데이터 수집
openclaw exec --task "주제 X에 대한 정보 수집"
```

### Cron 작업

주기적 작업 설정:

```bash
# 예약 작업 생성
openclaw cron --schedule "0 9 * * *" --task "아침 브리핑"
```
