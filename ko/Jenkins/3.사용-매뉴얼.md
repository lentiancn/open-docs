# Jenkins 사용자 매뉴얼

## 파이프라인 기초

파이프라인은 소프트웨어 개발 프로세스인 빌드, 테스트, 배포 전체를 자동화하는 Jenkins의 핵심 기능입니다.

## 파이프라인이란?

파이프라인은 Jenkins에 지속적인 배포 파이프라인을 구현하고 통합할 수 있도록 하는 플러그인 모음입니다. 간단히 말하면, 파이프라인은 소프트웨어를 버전 관리에서 사용자까지 전체 과정을 자동화합니다.

파이프라인 정의는 프로젝트의 소스 코드 저장소에 커밋할 수 있는 텍스트 파일(Jenkinsfile)에 작성됩니다. 이를 통해 "파이프라인 애즈 코드" 철학을 구현합니다.

### 파이프라인의 장점

- **코드 기반**: 파이프라인은 코드처럼 버전 관리되어 검토 및 반복 가능
- **지속성**: 파이프라인은 계획되거나 계획되지 않은 Jenkins 재시작에서 생존
- **일시 중지 가능**: 파이프라인은 사람의 입력이나 승인을 기다리면서 중지할 수 있음
- **다용도**: 병렬 실행 및 루프를 포함한 복잡한 실제 CD 요구 사항 지원
- **확장성**: 플러그인은 파이프라인 DSL을 확장하고 다른 도구와 통합

### 파이프라인 개념

#### 노드

노드는 Jenkins 환경의 일부이며 파이프라인을 실행할 수 있는 시스템입니다. 노드는 마스터 또는 에이전트 시스템일 수 있습니다.

#### 스테이지

스테이지는 "빌드", "테스트", "배포" 스테이지와 같이 전체 파이프라인에서 개념적으로 다른 작업 하위 집합을 정의합니다. 스테이지는 파이프라인 실행 상태 시각화에 사용됩니다.

#### 스텝

스텝은 특정 시간에 Jenkins에게 무엇을 해야 하는지 알려주는 Jenkins 파이프라인의 기본 작업입니다. 예를 들어, 셸 명령을 실행하려면 `sh` 스텝을 사용합니다.

### 파이프라인 문법

#### 선언적 파이프라인

선언적 파이프라인은 더 풍부한 기능을 가진 최신 문법으로 작성하고 읽기가 더 쉽습니다:

```groovy
pipeline {
    agent any
    
    stages {
        stage('빌드') {
            steps {
                echo '애플리케이션 빌드 중...'
            }
        }
        stage('테스트') {
            steps {
                echo '테스트 실행 중...'
            }
        }
        stage('배포') {
            steps {
                echo '애플리케이션 배포 중...'
            }
        }
    }
}
```

#### 스크립트형 파이프라인

스크립트형 파이프라인은 Groovy 문법을 사용하며 더 유연하지만 더 많은 프로그래밍 경험이 필요합니다:

```groovy
node {
    stage('빌드') {
        echo '애플리케이션 빌드 중...'
    }
    stage('테스트') {
        echo '테스트 실행 중...'
    }
    stage('배포') {
        echo '애플리케이션 배포 중...'
    }
}
```

## 첫 번째 파이프라인 만들기

### Blue Ocean 사용

Blue Ocean은 파이프라인 생성을 더 직관적으로 만드는 현대적인 Jenkins 인터페이스입니다:

1. Blue Ocean 플러그인 설치(미설치된 경우)
2. "Blue Ocean 열기" 클릭
3. "파이프라인 만들기" 클릭
4. 코드 저장소 위치 선택(GitHub, GitLab 등)
5. 저장소 및 브랜치 선택
6. Blue Ocean이 Jenkinsfile을 자동으로 감지하거나 생성 도움

### Jenkinsfile 사용

프로젝트에 Jenkinsfile을 만들면 Jenkins가 자동으로 인식합니다:

1. 프로젝트 루트에 Jenkinsfile 만들기
2. 파이프라인 스테이지 정의
3. 버전 관리에 커밋
4. Jenkins에서 파이프라인 작업 만들고 저장소와 연결

## 자격 증명 관리

Jenkins는 코드 저장소, 배포 대상 등 다양한 외부 리소스에 액세스해야 하므로 자격 증명 관리가 필요합니다.

### 자격 증명 추가

1. "Manage Jenkins" > "Manage Credentials"로 이동
2. "Add Credentials" 클릭
3. 자격 증명 유형 선택(사용자 이름/비밀번호, SSH 개인 키, 시크릿 파일 등)
4. 자격 증명 정보 입력
5. 자격 증명의 ID 및 설명 설정

### 파이프라인에서 자격 증명 사용

withCredentials 스텝 사용:

```groovy
pipeline {
    agent any
    stages {
        stage('자격 증명 사용') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "사용자 이름: $USER"
                }
            }
        }
    }
}
```

## 빌드 트리거

자동화된 빌드에는 트리거 구성이 필요합니다. Jenkins는 여러 트리거 방법을 지원합니다.

### SCM 폴링

소스 코드가 변경되었는지 정기적으로 확인:

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // 5분마다 확인
    }
    stages {
        stage('빌드') {
            steps {
                echo '코드 변경 감지, 빌드 시작'
            }
        }
    }
}
```

### 웹훅 트리거

저장소 웹훅을 통해 빌드를 트리거하여 더 효율적:

1. 해당 플러그인 설치(예: GitHub Hook Trigger)
2. 저장소 웹훅이 Jenkins를 가리키도록 구성
3. 코드 커밋 시 자동으로 빌드 트리거

### 예약된 빌드

일정에 따라 자동으로 실행:

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // 매일 오전 2시 실행
    }
    stages {
        stage('예약된 빌드') {
            steps {
                echo '예약된 빌드 실행 중'
            }
        }
    }
}
```

## 에이전트 및 컨테이너 사용

### 에이전트 구성

Jenkins는 분산 빌드를 지원하여 여러 시스템에 빌드 작업을 분산할 수 있습니다:

1. "Manage Jenkins" > "Manage Nodes"로 이동
2. 에이전트 노드 추가 및 연결 정보 구성
3. 파이프라인에서 에이전트 지정

### 파이프라인에서 Docker 사용

선언적 파이프라인은 Docker 컨테이너에서 빌드를 실행할 수 있습니다:

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('빌드') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## 테스트 보고서

Jenkins는 다양한 테스트 프레임워크와 통합하여 시각적 테스트 보고서를 생성할 수 있습니다.

### JUnit 테스트 보고서

```groovy
pipeline {
    agent any
    stages {
        stage('테스트') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### 기타 테스트 보고서

Jenkins는 다양한 테스트 보고서 플러그인을 지원합니다:

- TestNG
- Cobertura(코드 커버리지)
- JaCoCo(코드 커버리지)
- ESLint(프론트엔드 코드 분석)

## 배포 실습

### 다양한 환경에 배포

전형적인 다단계 배포 파이프라인:

```groovy
pipeline {
    agent any
    stages {
        stage('빌드') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('테스트 환경 배포') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('프로덕션 배포') {
            when {
                branch 'main'
            }
            steps {
                input message: '프로덕션 배포 확인?', ok: '배포'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### 블루-그린 배포

가동 중지 시간 없이 업데이트를 위해 Jenkins로 블루-그린 배포 구현:

1. 두 개의 동일한 환경 유지(블루 및 그린)
2. 트래픽을 새 버전으로 전환
3. 문제가 있으면 빠르게 롤백

## 권장 플러그인

### 필수 플러그인

- Git: 코드 저장소 통합
- Pipeline: 파이프라인 지원
- Blue Ocean: 현대적인 인터페이스
- Credentials: 자격 증명 관리
- Configuration as Code: 코드로 구성

### 일반 플러그인

- Docker: Docker 통합
- Kubernetes: Kubernetes 통합
- Maven Integration: Maven 프로젝트 지원
- Gradle: Gradle 프로젝트 지원
- Slack Notification: Slack 알림
- Email Extension: 이메일 알림

## 모범 사례

### Jenkinsfile 모범 사례

- Jenkinsfile을 버전 관리에 유지
- 선언적 파이프라인 문법 사용
- 파이프라인을 단순하고 명확하게 유지
- 스테이지를 합리적으로 분할
- 필요한 시간 초과 및 재시도 메커니즘 추가

### 보안 권장 사항

- Jenkins 및 플러그인을 정기적으로 업데이트
- 익명 사용자 권한 제한
- 민감한 정보에 자격 증명 사용
- 적절한 액세스 제어 구성
- 보안 감사 로깅 활성화

### 성능 최적화

- 빌드 노드 적절히 구성
- 증분 빌드로 시간 절약
- 빌드 캐시 구성
- 오래된 빌드 기록 정기적으로 정리
- 병렬 스테이지로 빌드 가속화

## 문제 해결

### 빌드 로그 보기

빌드가 실패하면 먼저 빌드 로그를 확인합니다:

1. 작업 페이지로 이동
2. 실패한 빌드 번호 클릭
3. 콘솔 출력 확인

### 일반적인 문제 해결

- 빌드 멈춤: 보류 중인 입력 또는 리소스 잠금 확인
- 자격 증명: 자격 증명이 만료되거나 취소되었는지 확인
- 플러그인 충돌: 최근 설치한 플러그인 비활성화 시도
- 메모리 부족: JVM 힙 매개변수 조정

## 요약

Jenkins 파이프라인은 CI/CD를 구현하는 핵심 도구입니다. 이 매뉴얼을 통해 다음을 수행할 수 있어야 합니다:

- 핵심 파이프라인 개념 이해
- 파이프라인 작업 생성 및 구성
- 자격 증명 및 트리거 관리
- 테스트 및 배포 도구 통합
- 워크플로 최적화를 위한 모범 사례 적용

지속적인 학습과 연습은 Jenkins를 마스터하는 열쇠입니다. 실제 프로젝트에서 계속 탐색하고 최적화하세요.
