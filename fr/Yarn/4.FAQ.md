# Questions fréquemment posées sur Yarn

## Questions liées à l'installation

### Q: Comment choisir entre les versions de Yarn ?
**R**: 
- **Nouveaux projets**: Il est recommandé d'utiliser Yarn v4+ pour profiter des fonctionnalités modernes et de meilleures performances
- **Projets existants**: Si vous utilisez déjà Yarn v1, vous pouvez continuer à l'utiliser ou envisager de migrer vers la version moderne
- **Environnements d'entreprise**: Yarn v1 fournit un support à long terme plus stable, tandis que v4+ offre des fonctionnalités plus innovantes

### Q: Qu'est-ce que Corepack et pourquoi en ai-je besoin ?
**R**: Corepack est un outil intermédiaire inclus avec Node.js qui vous permet de configurer la version de votre gestionnaire de paquets par projet. Il garantit que les membres de l'équipe utilisent la même version de Yarn, évitant ainsi les problèmes d'incohérence de version.

### Q: Que faire en cas d'erreurs de permission lors de l'installation de Yarn ?
**R**: 
- Ajoutez le drapeau `--unsafe-perm` lors de l'utilisation de `npm install --global yarn`
- Utilisez nvm pour gérer les versions de Node.js
- Dans les environnements CI/CD, l'installation globale n'est généralement pas nécessaire

## Questions liées à l'utilisation

### Q: Quelles sont les différences entre Yarn et npm ?
**R**: 
- **Vitesse**: Yarn utilise des installations parallèles et une mise en cache hors ligne, généralement plus rapide
- **Déterminisme**: Yarn garantit des installations cohérentes (yarn.lock)
- **Fonctionnalités**: Yarn fournit des workspaces, PnP, un système de plugins et d'autres fonctionnalités avancées
- **Commandes**: Yarn réimplémente toutes les commandes pour une meilleure expérience développeur

### Q: Qu'est-ce que PnP (Plug'n'Play) ?
**R**: PnP est une fonctionnalité de Yarn v2+ qui élimine le répertoire node_modules en résolvant les dépendances directement depuis le cache. Cela économise de l'espace disque et améliore la vitesse d'installation, mais peut être incompatible avec certains outils qui ne supportent pas PnP.

### Q: Comment utiliser les workspaces dans mon projet ?
**R**: 
1. Ajoutez un champ `"workspaces"` à votre `package.json` racine
2. Créez des répertoires de sous-packages (par exemple, `packages/`)
3. Créez des fichiers `package.json` individuels dans les sous-packages
4. Utilisez `yarn workspace <name> <command>` pour exécuter des commandes dans des workspaces spécifiques

## Questions sur la migration des versions

### Q: Comment migrer de Yarn v1 à v4+ ?
**R**: 
1. Assurez-vous que votre projet est sous contrôle de version Git
2. Exécutez `yarn set version stable` (pour v4+)
3. Suivez le guide de migration pour mettre à jour les configurations étape par étape
4. Testez toutes les fonctionnalités pour assurer la compatibilité

## Dépannage

### Q: Que faire si `yarn install` échoue ?
**R**: 
- Effacer le cache : `yarn cache clean`
- Supprimer `yarn.lock` et réinstaller
- Vérifier la connexion réseau et la configuration du registre
- Pour Yarn v4+, vérifier la configuration de `.yarnrc.yml`

### Q: Comment vérifier la version de Yarn ?
**R**: 
- Version globale : `yarn --version`
- Version du projet : Vérifier le répertoire `.yarn/releases/` ou le fichier `.yarnrc.yml`

### Q: Que faire si les commandes Yarn sont très lentes ?
**R**: 
- Assurez-vous que la mise en cache hors ligne est activée
- Vérifiez la connexion réseau
- Pour les grands projets, envisagez d'utiliser des workspaces pour diviser la base de code
- Mettez à jour vers la dernière version pour des améliorations de performance
