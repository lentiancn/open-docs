# JDK Installationsanleitung

## JDK unter Windows installieren

### Systemanforderungen

Bevor Sie JDK installieren, stellen Sie sicher, dass Ihr Computer die folgenden Mindestanforderungen erfüllt: Prozessor, Intel Core i3 oder besser wird empfohlen; Speicher, mindestens 4GB RAM werden empfohlen, besser 8GB oder mehr; Festplattenplatz, mindestens 2GB freier Platz für die JDK-Installation, zusätzlicher Platz für Entwicklungsprojekte; Betriebssystem, Windows 10 oder Windows 11 64-Bit-Versionen.

Obwohl Windows 7 und 8.1 funktionieren könnten, werden diese Betriebssysteme von Oracle offiziell nicht mehr unterstützt. Bei Ausführung auf älteren Systemen können Kompatibilitätsprobleme auftreten.

### JDK herunterladen

Besuchen Sie zunächst die Oracle-Website unter https://www.oracle.com/java/technologies/downloads/ um JDK herunterzuladen. Auf der Download-Seite sehen Sie verschiedene JDK-Versionen: JDK 25 ist die neueste Version, JDK 21 ist die neueste LTS-Version, JDK 17 ist auch eine reife LTS-Version. Wählen Sie die passende Version entsprechend Ihren Projektanforderungen.

Für Windows-Systeme wählen Sie den x64 Installer (.exe) oder den x64 MSI Installer (.msi). Der .exe ist der klassische Installationsassistent, der .msi eignet sich für Bereitstellungen in Unternehmen. Wenn Sie nur Java-Programme ausführen möchten, ohne zu installieren, wählen Sie das komprimierte Archiv x64 (.zip), das Sie entpacken und direkt verwenden können.

Beachten Sie die Lizenzbedingungen beim Herunterladen. Oracle JDK ist unter bestimmten Versionen und Nutzungsarten kostenlos, hat jedoch Einschränkungen für kommerzielle Nutzung und Weiterverteilung. Wenn Sie vollständig kostenlose Nutzung benötigen, sollten Sie OpenJDK oder andere auf OpenJDK basierende Distributionen wie Eclipse Temurin in Betracht ziehen.

### Installationsschritte

Suchen Sie die heruntergeladene JDK-Installationsdatei (normalerweise jdk-xx_windows-x64_bin.exe) und doppelklicken Sie zum Ausführen. Klicken Sie im Installationsassistenten auf "Weiter". Zuerst sehen Sie die Zielordner-Auswahl, der Standard-Installationspfad ist C:\Programme\Java\jdk-xx. Wenn Sie den Installationsort ändern müssen, klicken Sie auf "Ändern", um ein anderes Verzeichnis zu wählen. Es wird empfohlen, den Standardpfad zu verwenden, um spätere Konfigurationsprobleme zu vermeiden.

Als nächstes sehen Sie die Funktionsauswahl, standardmäßig sind alle Funktionen ausgewählt, einschließlich Entwicklungstools, Quellcode und öffentliches JRE. Es wird empfohlen, die vollständige Standardinstallation beizubehalten. Klicken Sie auf "Weiter", um die Installation zu starten. Der Installationsvorgang kann einige Minuten dauern.

Nach Abschluss der Installation sehen Sie eine Meldung. Klicken Sie auf "Schließen", um die Installation abzuschließen. JDK ist jetzt im angegebenen Verzeichnis installiert, aber Sie müssen Umgebungsvariablen konfigurieren, um java, javac und andere Befehle in der Befehlszeile verwenden zu können.

### Umgebungsvariablen konfigurieren

Der Zweck der Konfiguration von Umgebungsvariablen ist, dass das System den JDK-Installationsort finden kann, damit Sie java, javac und andere Befehle aus einem beliebigen Verzeichnis verwenden können. Klicken Sie mit der rechten Maustaste auf "Dieser PC" oder "Mein PC" und wählen Sie "Eigenschaften". Klicken Sie im linken Menü auf "Erweiterte Systemeinstellungen".

Klicken Sie im geöffneten Systemeigenschaften-Fenster auf die Schaltfläche "Umgebungsvariablen". Klicken Sie im Bereich "Systemvariablen" auf "Neu". Geben Sie JAVA_HOME als Variablennamen und den JDK-Installationspfad (z.B. C:\Programme\Java\jdk-21) als Variablenwert ein und klicken Sie auf OK.

Suchen Sie die Path-Variable unter den Systemvariablen und klicken Sie auf "Bearbeiten". Klicken Sie auf "Neu" und geben Sie %JAVA_HOME%\bin ein. Stellen Sie sicher, dass diese Zeile oben steht, damit das neu installierte JDK priorisiert wird. Klicken Sie auf OK, um alle Einstellungen zu speichern.

Öffnen Sie ein neues Befehlszeilenfenster (cmd oder PowerShell) und geben Sie java -version und javac -version ein. Wenn Versionsinformationen angezeigt werden, war die Konfiguration erfolgreich.

### Installation überprüfen

Öffnen Sie die Eingabeaufforderung (Win+R, cmd eingeben) und führen Sie die folgenden Befehle aus, um zu überprüfen, ob JDK korrekt installiert ist:

Geben Sie java -version ein, es sollten die Versionsinformationen der Java-Laufzeitumgebung angezeigt werden. Geben Sie javac -version ein, es sollten die Versionsinformationen des Java-Compilers angezeigt werden. Geben Sie echo %JAVA_HOME% ein, es sollte der zuvor eingestellte JAVA_HOME-Pfad angezeigt werden.

Wenn einer der Befehle Fehler anzeigt, überprüfen Sie, ob die Umgebungsvariablen korrekt konfiguriert oder ob JDK korrekt installiert ist.

## JDK unter macOS installieren

### DMG-Installationspaket verwenden

macOS-Benutzer können JDK schnell mit dem DMG-Installationspaket installieren. Besuchen Sie die offizielle Oracle-Download-Seite und laden Sie die macOS-Version von JDK herunter. Für Apple Silicon (M1/M2/M3-Chips) wählen Sie ARM64 DMG Installer; für Intel-Chip-Macs wählen Sie x64 DMG Installer.

Nach dem Download öffnen Sie die DMG-Datei, Sie sehen eine pkg-Installationsdatei. Doppelklicken Sie zum Ausführen des Installationsassistenten und folgen Sie den Anweisungen, um die Installation abzuschließen. Der Installer platziert JDK im Verzeichnis /Library/Java/JavaVirtualMachines/.

Nach Abschluss der Installation öffnen Sie Terminal und geben Sie java -version ein, um die Installation zu überprüfen. Wenn mehrere JDK-Versionen auf dem System vorhanden sind, können Sie /usr/libexec/java_home -V verwenden, um alle installierten Versionen anzuzeigen.

### Installation mit Homebrew

Wenn Homebrew bereits installiert ist, können Sie es verwenden, um JDK zu installieren, was bequemer ist. Öffnen Sie Terminal und führen Sie aus: brew update. Dann installieren Sie JDK 21: brew install openjdk@21.

Nach der Installation müssen Sie JDK zum PATH hinzufügen. Führen Sie den folgenden Befehl aus, um JDK mit dem System zu verknüpfen: sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.

Fügen Sie dann Umgebungsvariablen in ~/.zshrc oder ~/.bash_profile hinzu: export PATH="/usr/local/opt/openjdk@21/bin:$PATH". Speichern und führen Sie source ~/.zshrc aus, um die Konfiguration anzuwenden.

### JAVA_HOME konfigurieren

Unter macOS ist das Konfigurieren der Umgebungsvariable JAVA_HOME nicht zwingend erforderlich, da das System automatisch installierte JDKs findet. Einige Tools (wie Maven, Gradle) erfordern möglicherweise eine manuelle Konfiguration. Bearbeiten Sie in Terminal die Datei ~/.zshrc und fügen Sie hinzu: export JAVA_HOME=$(/usr/libexec/java_home). Speichern und führen Sie source ~/.zshrc aus, um die Konfiguration anzuwenden.

## JDK unter Linux installieren

### Installation mit APT (Debian/Ubuntu)

Auf Debian- und Ubuntu-Systemen können Sie OpenJDK mit dem APT-Paketmanager installieren. Aktualisieren Sie zuerst die Paketliste: sudo apt update. Installieren Sie dann OpenJDK 21 Entwicklerversion: sudo apt install openjdk-21-jdk. Wenn Sie nur Java-Programme ausführen möchten, ohne den Compiler zu benötigen, installieren Sie JRE: sudo apt install openjdk-21-jre.

Während der Installation werden Sie gefragt, ob Sie das JAVA_HOME-Verzeichnis erstellen möchten. Wählen Sie "Ja" für automatische Konfiguration. Nach Abschluss der Installation verwenden Sie java -version und javac -version, um die Installation zu überprüfen.

Wenn Sie JAVA_HOME manuell konfigurieren müssen, bearbeiten Sie die Datei ~/.bashrc oder ~/.profile und fügen Sie hinzu: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64. Führen Sie dann source ~/.bashrc aus, um die Konfiguration anzuwenden.

### Installation mit YUM (Red Hat/CentOS)

Auf Systemen mit YUM wie Red Hat, CentOS und Fedora können Sie JDK direkt installieren. Führen Sie den folgenden Befehl aus, um OpenJDK 21 zu installieren: sudo yum install java-21-openjdk java-21-openjdk-devel. Das -devel-Paket enthält den Compiler und andere Entwicklungstools.

Nach Abschluss der Installation verwenden Sie den alternatives-Befehl, um die Standard-Java-Version einzustellen: sudo alternatives --config java. Folgen Sie den Anweisungen, um die als Standard festzulegende Java-Version auszuwählen.

Bearbeiten Sie die Datei ~/.bashrc und fügen Sie hinzu: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk. Führen Sie source ~/.bashrc aus, um die Konfiguration anzuwenden.

### Binärdateien manuell installieren

Für Situationen, die die Installation einer bestimmten JDK-Version erfordern, können Sie Binärarchivdateien manuell herunterladen und entpacken. Besuchen Sie die Oracle- oder OpenJDK-Download-Seite und laden Sie die .tar.gz-Datei der gewünschten Version herunter.

Erstellen Sie das JDK-Installationsverzeichnis: sudo mkdir -p /opt/jdk. Entpacken Sie die heruntergeladene Datei: sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk. Gehen Sie in das entpackte Verzeichnis und notieren Sie sich den vollständigen Pfad.

Bearbeiten Sie die Datei ~/.bashrc und fügen Sie hinzu: export JAVA_HOME=/opt/jdk/jdk-21 und export PATH=$JAVA_HOME/bin:$PATH. Führen Sie source ~/.bashrc aus, um die Konfiguration anzuwenden.

## Mehrere JDK-Versionen verwalten

### Windows Multi-Version-Verwaltung

Nach der Installation mehrerer JDK-Versionen unter Windows müssen Sie die Standardversion über Umgebungsvariablen wechseln. Die einfachste Methode ist, die Reihenfolge der JDK-bin-Verzeichnisse in der PATH-Umgebungsvariable zu ändern und die gewünschte Version nach vorne zu stellen.

Eine andere Methode ist die Verwendung der JAVA_HOME-Umgebungsvariable, die auf verschiedene JDK-Installationsverzeichnisse zeigt. Diese Methode funktioniert für die meisten Tools, aber einige Tools verwenden möglicherweise den java-Befehl in PATH anstelle von JAVA_HOME.

Einige Drittanbieter-Tools können bei der Verwaltung mehrerer JDK-Versionen helfen, wie die Windows-Version von SDKMAN! oder JetBrains Toolbox. Diese Tools erleichtern das Wechseln zwischen Versionen.

### macOS Multi-Version-Verwaltung

Unter macOS können Sie das jenv-Tool verwenden, um mehrere JDK-Versionen zu verwalten. Installieren Sie zuerst jenv: brew install jenv. Fügen Sie dann die jenv-Initialisierung zur Shell-Konfigurationsdatei hinzu.

Fügen Sie das installierte JDK zu jenv hinzu: jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home. Verwenden Sie dann jenv versions, um alle verwalteten Versionen anzuzeigen, und jenv global xx, um die globale Standardversion festzulegen.

Eine andere Methode ist die Verwendung von /usr/libexec/java_home -v, um eine bestimmte Version zum Ausführen von Programmen anzugeben. Zum Beispiel: /usr/libexec/java_home -v 21 --exec java -version.

### Linux Multi-Version-Verwaltung

Unter Linux können Sie das update-alternatives-Tool verwenden, um mehrere JDK-Versionen zu verwalten. Erstellen Sie zunächst Alternativen-Einträge für jede JDK-Version: sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1.

Verwenden Sie dann update-alternatives --config java, um die Standardversion auszuwählen. jenv ist auch unter Linux verfügbar, mit ähnlichen Installations- und Konfigurationsmethoden wie unter macOS.

## Häufige Installationsprobleme

### java-Befehl nach Installation nicht gefunden

Wenn Sie den java-Befehl nach der JDK-Installation nicht finden können, überprüfen Sie zunächst, ob JDK korrekt installiert ist. Überprüfen Sie, ob die JAVA_HOME-Umgebungsvariable korrekt eingestellt ist und ob PATH %JAVA_HOME%\bin (Windows) oder $JAVA_HOME/bin (Linux/macOS) enthält.

Unter Windows öffnen Sie ein neues Befehlszeilenfenster, da Änderungen an Umgebungsvariablen möglicherweise nicht sofort wirksam werden. Unter Linux/macOS führen Sie source ~/.bashrc aus oder öffnen Sie das Terminal erneut.

Wenn das Problem weiterhin besteht, überprüfen Sie, ob das JDK-Installationsverzeichnis korrekt ist und ob die bin/java-Datei in diesem Verzeichnis vorhanden ist.

### Berechtigungsprobleme

Bei der Installation von JDK unter Linux und macOS können Berechtigungsprobleme auftreten. Bei Verwendung des Paketmanagers sind normalerweise Administratorrechte (sudo) erforderlich. Bei manueller Installation stellen Sie sicher, dass das Installationsverzeichnis über entsprechende Lese- und Schreibrechte verfügt.

Wenn normale Benutzer den java-Befehl nicht verwenden können, überprüfen Sie die Berechtigungen des JAVA_HOME-Verzeichnisses und des JDK-bin-Verzeichnisses. Sie können ls -la /usr/lib/jvm verwenden, um Verzeichnisberechtigungen zu überprüfen.

### JAVA_HOME-Einstellung funktioniert nicht

Wenn das Einstellen von JAVA_HOME nicht funktioniert, überprüfen Sie, ob die Syntax der Umgebungsvariable korrekt ist. Unter Windows darf der Variablenwert keine Anführungszeichen enthalten. Unter Linux/macOS verwenden Sie den richtigen export-Befehl.

Überprüfen Sie auch, ob andere Skripte oder Konfigurationsdateien die JAVA_HOME-Einstellung überschreiben. Unter Linux kann die systemweite Konfiguration in /etc/environment oder /etc/profile.d/ sein.

### Alte Versionen deinstallieren

Nach dem Upgrade von JDK müssen Sie möglicherweise alte Versionen deinstallieren, um Speicherplatz zu sparen. Unter Windows suchen Sie die alte JDK-Version in der Systemsteuerung unter "Programme und Funktionen" und klicken Sie auf Deinstallieren. Unter macOS löschen Sie den alten Versionsordner unter /Library/Java/JavaVirtualMachines/. Unter Linux deinstallieren Sie mit dem Paketmanager: sudo apt remove openjdk-xx-jdk (APT) oder sudo yum remove java-xx-openjdk (YUM).
