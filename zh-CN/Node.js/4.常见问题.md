# Node.js 常见问题

## 安装相关问题

### Q: 如何检查 Node.js 是否正确安装？
**A:** 在终端中运行以下命令：
```bash
node --version
npm --version
```
如果显示版本号，说明安装成功。

### Q: 安装后命令未找到怎么办？
**A:** 这通常是因为 PATH 环境变量未正确配置。解决方案：
- **Windows**: 重新运行安装程序，确保勾选 "Add to PATH" 选项
- **macOS/Linux**: 如果使用包管理器安装，通常会自动配置。手动添加：
  ```bash
  export PATH=$PATH:/usr/local/bin
  ```

### Q: 如何在不同项目中使用不同版本的 Node.js？
**A:** 推荐使用 nvm (Node Version Manager)：
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装特定版本
nvm install 18.17.0

# 切换版本
nvm use 18.17.0

# 设置默认版本
nvm alias default 18.17.0
```

## npm 相关问题

### Q: npm install 太慢怎么办？
**A:** 可以配置使用国内镜像源：
```bash
# 临时使用淘宝镜像
npm install --registry https://registry.npmmirror.com

# 永久设置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 验证设置
npm config get registry
```

### Q: 全局安装包时出现权限错误？
**A:** 解决方案：
1. **推荐**: 使用 nvm 管理 Node.js
2. **配置 npm 全局目录**:
   ```bash
   mkdir ~/.npm-global
   npm config set prefix '~/.npm-global'
   export PATH=~/.npm-global/bin:$PATH
   ```
3. **不推荐**: 使用 sudo（存在安全风险）

### Q: 如何更新全局安装的包？
**A:** 
```bash
# 查看过期的全局包
npm outdated -g

# 更新特定包
npm update -g package-name

# 批量更新所有全局包（需要额外工具）
npm install -g npm-check-updates
ncu -g
```

## 开发相关问题

### Q: 如何处理异步操作中的错误？
**A:** 始终使用 try-catch 包装 async/await 代码：
```javascript
async function fetchData() {
  try {
    const data = await api.getData();
    return data;
  } catch (error) {
    console.error('获取数据失败:', error);
    throw error; // 或返回默认值
  }
}
```

对于 Promise 链，使用 `.catch()`：
```javascript
api.getData()
  .then(data => process(data))
  .catch(error => console.error('Error:', error));
```

### Q: 如何避免回调地狱（Callback Hell）？
**A:** 使用现代 JavaScript 特性：
1. **Promise 链**:
   ```javascript
   step1()
     .then(result1 => step2(result1))
     .then(result2 => step3(result2))
     .catch(error => handleError(error));
   ```

2. **async/await** (推荐):
   ```javascript
   async function processSteps() {
     try {
       const result1 = await step1();
       const result2 = await step2(result1);
       const result3 = await step3(result2);
       return result3;
     } catch (error) {
       handleError(error);
     }
   }
   ```

### Q: 如何处理内存泄漏？
**A:** 常见原因和解决方案：
1. **全局变量积累**: 避免不必要的全局变量
2. **事件监听器未移除**: 使用 `removeListener()` 或 `once()`
3. **定时器未清除**: 使用 `clearTimeout()` 和 `clearInterval()`
4. **缓存未清理**: 实现 LRU 缓存或定期清理

使用内存分析工具：
```bash
# 生成堆快照
node --inspect app.js
```

## 性能相关问题

### Q: 如何提高 Node.js 应用性能？
**A:** 
1. **使用集群模式**: 利用多核 CPU
   ```javascript
   const cluster = require('cluster');
   const numCPUs = require('os').cpus().length;
   ```

2. **避免阻塞操作**: 始终使用异步 I/O

3. **使用流处理大文件**:
   ```javascript
   const fs = require('fs');
   const readStream = fs.createReadStream('large-file.txt');
   const writeStream = fs.createWriteStream('output.txt');
   readStream.pipe(writeStream);
   ```

4. **优化数据库查询**: 使用索引，避免 N+1 查询

5. **使用缓存**: Redis 或内存缓存减少重复计算

### Q: 如何监控 Node.js 应用？
**A:** 内置监控工具：
```bash
# 启用性能监控
node --prof app.js

# 生成性能报告
node --prof-process isolate-*.log > profile.txt
```

第三方监控工具：
- **PM2**: 进程管理和监控
- **New Relic**: APM 监控
- **Datadog**: 基础设施监控

## 安全相关问题

### Q: 如何防止依赖漏洞？
**A:** 
1. **定期审计依赖**:
   ```bash
   npm audit
   npm audit fix
   ```

2. **使用锁定文件**: 提交 `package-lock.json` 确保依赖一致性

3. **限制依赖范围**: 只安装必要的依赖

4. **使用 Snyk 或 Dependabot**: 自动检测和修复漏洞

### Q: 如何处理环境变量？
**A:** 使用 `.env` 文件（不要提交到版本控制）：
```bash
# .env
DATABASE_URL=your-database-url
API_KEY=your-api-key
```

在代码中使用：
```javascript
require('dotenv').config();
const dbUrl = process.env.DATABASE_URL;
```

## 调试相关问题

### Q: 如何调试 Node.js 应用？
**A:** 
1. **内置调试器**:
   ```bash
   node --inspect-brk app.js
   ```

2. **VS Code 调试**: 创建 `.vscode/launch.json` 配置

3. **日志记录**: 使用 `winston` 或 `bunyan` 等日志库

4. **堆栈跟踪**: 使用 `console.trace()` 查看调用栈

### Q: 如何处理未捕获的异常？
**A:** 添加全局异常处理器：
```javascript
// 未捕获的异常
process.on('uncaughtException', (error) => {
  console.error('未捕获的异常:', error);
  process.exit(1);
});

// 未处理的 Promise 拒绝
process.on('unhandledRejection', (reason, promise) => {
  console.error('未处理的 Promise 拒绝:', reason);
  process.exit(1);
});
```

## 部署相关问题

### Q: 如何部署 Node.js 应用？
**A:** 推荐使用进程管理器：
1. **PM2**:
   ```bash
   npm install -g pm2
   pm2 start app.js
   pm2 startup
   pm2 save
   ```

2. **Docker**:
   ```dockerfile
   FROM node:18-alpine
   WORKDIR /app
   COPY package*.json ./
   RUN npm ci --only=production
   COPY . .
   CMD ["node", "app.js"]
   ```

3. **Systemd** (Linux): 创建服务文件管理应用

### Q: 如何处理环境差异？
**A:** 使用环境变量区分不同环境：
```javascript
const isProduction = process.env.NODE_ENV === 'production';
const port = process.env.PORT || 3000;

if (isProduction) {
  // 生产环境配置
} else {
  // 开发环境配置
}
```
