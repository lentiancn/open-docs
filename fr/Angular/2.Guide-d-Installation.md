# Angular Guide d'Installation

Configuration complète de l'environnement de développement Angular.

## Exigences 环境

### Exigences matérielles

- CPU: 2 cœurs ou plus
- Mémoire: 4 Go ou plus
- Disque: 10 Go ou plus

### Exigences logicielles

- Système d'exploitation: Windows/macOS/Linux
- Node.js: 18.10.0+ ou 20.0.0+
- npm: 9.0.0+ (inclus avec Node.js)
- Git: N'importe quelle version

## Étapes d'installation

### 1. Installer Node.js

**Windows/macOS:**

Télécharger depuis: https://nodejs.org

Ou utiliser nvm:

```bash
# Installer nvm (macOS/Linux)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Installer Node.js LTS
nvm install --lts
nvm use --lts
```

**Vérifier l'installation:**

```bash
node --version  # Doit afficher v18.x.x ou v20.x.x
npm --version    # Doit afficher 9.x.x ou supérieur
```

### 2. Installer Angular CLI

```bash
# Installer globalement via npm
npm install -g @angular/cli

# Vérifier l'installation
ng version
```

### 3. Créer un nouveau projet

```bash
# Créer une nouvelle application Angular
ng new my-app

# Naviguer vers le projet
cd my-app

# Démarrer le serveur de développement
ng serve
```

### 4. Accéder à l'application

Ouvrir le navigateur: http://localhost:4200

## Commandes courantes

### Commandes de projet

```bash
# Créer un nouveau projet avec routage et SCSS
ng new my-app --routing --style=scss

# Créer une application standalone (Angular 14+)
ng new my-app --standalone

# Démarrer le serveur de développement
ng serve

# Compiler pour la production
ng build

# Exécuter les tests unitaires
ng test

# Exécuter les tests E2E
ng e2e
```

### Commandes de génération

```bash
# Générer un composant
ng generate component my-component
ng g c my-component

# Générer un service
ng generate service my-service
ng g s my-service

# Générer une directive
ng generate directive my-directive
ng g d my-directive

# Générer un pipe
ng generate pipe my-pipe
ng g p my-pipe

# Générer un module
ng generate module my-module
ng g m my-module

# Générer une classe
ng generate class my-class
ng g cl my-class

# Générer une interface
ng generate interface my-interface
ng g i my-interface

# Générer un guard
ng generate guard my-guard
ng g g my-guard

# Générer un interceptor
ng generate interceptor my-interceptor
ng g in my-interceptor
```

## Configuration d'environnement

### Environnement de développement

```typescript
// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: 'http://localhost:3000/api'
};
```

### Environnement de production

```typescript
// src/environments/environment.prod.ts
export const environment = {
  production: true,
  apiUrl: 'https://api.example.com'
};
```

### Utiliser l'environnement

```typescript
import { Component } from '@angular/core';
import { environment } from '../environments/environment';

@Component({
  selector: 'app-root',
  template: `<p>API: {{ apiUrl }}</p>`
})
export class AppComponent {
  apiUrl = environment.apiUrl;
}
```

## Problèmes courants

### npm lent?

```bash
# Utiliser le miroir Taobao
npm config set registry https://registry.npmmirror.com

# Ou utiliser nrm pour gérer les miroirs
npm install -g nrm
nrm use taobao
```

### Version Node non correspondante?

```bash
# Utiliser nvm pour changer de version
nvm install 20
nvm use 20
```

### Problèmes de permissions (Linux/macOS)?

```bash
# Corriger les permissions npm
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## Désinstaller

```bash
# Désinstaller Angular CLI
npm uninstall -g @angular/cli

# Supprimer le projet
rm -rf my-app
```
