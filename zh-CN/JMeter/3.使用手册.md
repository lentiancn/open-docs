# JMeter 使用手册

本手册将详细介绍如何使用 Apache JMeter 创建和执行负载测试。

## 测试计划基本结构

JMeter 测试计划由以下核心组件构成：

### 1. 测试计划（Test Plan）

测试计划是所有测试的根节点，包含整个测试的配置信息。在测试计划中可以设置：
- 用户定义的变量
- 全局属性设置
- 线程组配置

### 2. 线程组（Thread Group）

线程组是测试的执行单元，定义了：
- **线程数量**：模拟的用户数
- **启动时间**：所有线程启动所需时间
- **循环次数**：每个线程执行测试的次数
- **调度器**：设置测试的开始和持续时间

### 3. 取样器（Samplers）

取样器是向服务器发送请求的组件，常见的取样器包括：

- **HTTP 请求**：发送 HTTP/HTTPS 请求
- **FTP 请求**：测试 FTP 服务器
- **JDBC 请求**：测试数据库连接
- **LDAP 请求**：测试 LDAP 目录
- **JMS 消息**：测试消息队列
- **TCP 取样器**：测试 TCP 服务

### 4. 逻辑控制器（Logic Controllers）

逻辑控制器用于控制请求的执行顺序：

- **简单控制器**：对请求进行分组
- **循环控制器**：重复执行其下的请求
- **仅一次控制器**：在测试期间只执行一次
- **交替控制器**：轮流执行其下的请求
- **随机控制器**：随机选择执行请求
- **吞吐量控制器**：控制请求的执行频率

### 5. 监听器（Listeners）

监听器用于查看和分析测试结果：

- **查看结果树**：查看每个请求的详细响应
- **聚合报告**：查看汇总的性能统计
- **图形结果**：以图形方式展示测试数据
- **响应时间图表**：展示响应时间变化趋势

### 6. 配置元件（Config Elements）

配置元件用于设置默认值和共享数据：

- **HTTP 请求默认值**：设置 HTTP 请求的默认参数
- **用户定义的变量**：定义可复用的变量
- **CSV 数据文件配置**：从 CSV 文件读取测试数据
- **HTTP Cookie 管理器**：管理 Cookies
- **HTTP 头管理器**：设置请求头信息

### 7. 定时器（Timers）

定时器用于设置请求之间的延迟：

- **固定定时器**：设置固定延迟
- **高斯随机定时器**：设置随机延迟
- **同步定时器**：模拟并发请求
- **吞吐量定时器**：控制请求的吞吐量

### 8. 前置处理器和后置处理器

- **前置处理器**：在请求发送前执行操作
- **后置处理器**：在请求响应后提取数据

## 创建第一个测试计划

### 步骤一：创建测试计划

1. 启动 JMeter
2. 选择"文件" -> "新建"创建新测试计划
3. 为测试计划命名

### 步骤二：添加线程组

1. 右键点击测试计划
2. 选择"添加" -> "线程（用户）" -> "线程组"
3. 设置线程数（如 10）、启动时间（如 5 秒）、循环次数（如 1）

### 步骤三：添加 HTTP 请求

1. 右键点击线程组
2. 选择"添加" -> "取样器" -> "HTTP 请求"
3. 设置服务器名称或 IP、端口号、请求路径等

### 步骤四：添加监听器

1. 右键点击线程组或 HTTP 请求
2. 选择"添加" -> "监听器" -> "查看结果树"
3. 可以添加"聚合报告"查看性能统计

### 步骤五：运行测试

1. 点击工具栏上的"启动"按钮
2. 观察监听器中显示的结果数据

## 参数化和数据驱动测试

### 使用 CSV 数据文件

1. 添加"CSV 数据文件配置"元件
2. 设置文件名、变量名称
3. 在请求中使用 ${变量名} 引用数据

### 使用用户定义的变量

1. 添加"用户定义的变量"元件
2. 定义变量名称和值
3. 在请求中使用 ${变量名} 引用

## 关联处理

### 使用正则表达式提取器

当请求需要动态数据（如 Session ID）时：

1. 添加"正则表达式提取器"
2. 设置正则表达式匹配模式
3. 在后续请求中使用提取的变量

### 使用 JSON 提取器

对于 JSON 响应：

1. 添加"JSON 提取器"
2. 设置 JSON Path 表达式
3. 提取需要的 JSON 字段值

## 分布式测试

当单台机器无法产生足够负载时，可以使用分布式测试：

### 主节点配置

1. 在主节点 bin 目录下编辑 jmeter.properties
2. 设置 remote_hosts 指向从节点地址
3. 启动 JMeter

### 从节点配置

1. 在每台从节点上运行 jmeter-server
2. 确保防火墙允许指定端口通信

### 启动分布式测试

1. 在主节点选择"远程启动"
2. 选择要启动的从节点或全部启动

## 性能优化建议

- 使用命令行模式运行测试，减少 GUI 资源消耗
- 合理设置线程数和启动时间，避免瞬时高负载
- 关闭不必要的监听器，仅在需要时查看结果
- 使用断言验证响应内容，确保测试准确性
- 及时清理不再使用的测试计划
- 对于大规模测试，考虑使用分布式测试
