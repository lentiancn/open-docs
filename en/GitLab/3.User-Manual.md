# GitLab User Manual

> Comprehensive guide to using GitLab for development and operations

---

## Getting Started

### Creating an Account

1. Navigate to your GitLab instance
2. Click "Register" to create an account
3. Verify your email address
4. Complete your user profile

### Creating a Project

1. Click "New Project" button
2. Choose project template or blank project
3. Set project name and description
4. Configure visibility level (private, internal, public)
5. Click "Create Project"

---

## Git Operations

### Clone a Repository

```bash
# Clone using HTTPS
git clone https://gitlab.example.com/username/project.git

# Clone using SSH
git clone git@gitlab.example.com:username/project.git
```

### Basic Git Workflow

```bash
# Create a new branch
git checkout -b feature/new-feature

# Make changes and stage them
git add .
git commit -m "Add new feature"

# Push to remote
git push origin feature/new-feature

# Create merge request from branch
```

### Managing Branches

```bash
# List branches
git branch -a

# Delete a local branch
git branch -d feature/old-feature

# Delete a remote branch
git push origin --delete feature/old-feature
```

## Issue Tracking

### Creating Issues

1. Navigate to your project
2. Go to Issues > New Issue
3. Add title and description
4. Assign labels, milestones, and assignees
5. Set weight and due date
6. Submit issue

### Using Issue Boards

1. Go to Issues > Boards
2. Create new board or use default
3. Add lists for each stage (e.g., To Do, In Progress, Done)
4. Drag and drop issues between lists

### Labels and Milestones

- Create descriptive labels for categorization
- Use milestones to track progress toward goals
- Close issues with commit messages

---

## Merge Requests

### Creating a Merge Request

1. Push your feature branch to GitLab
2. Click "Create Merge Request" button
3. Fill in title and description
4. Select source and target branches
5. Add labels and assign reviewers
6. Submit merge request

### Code Review Process

- Reviewers can add comments on specific lines
- Use thread discussions for conversations
- Approve or request changes
- Resolve all threads before merging

### Squash and Merge

- Squash commits to keep history clean
- Choose to delete source branch after merge
- Edit commit message before merging

## CI/CD Pipelines

### Basic .gitlab-ci.yml

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/

test:
  stage: test
  script:
    - npm test
  coverage: '/Coverage: \d+\.\d+%/'

deploy:
  stage: deploy
  script:
    - npm run deploy
  only:
    - main
```

### Running Pipelines

- Pipelines run automatically on push
- Trigger manually from UI
- View pipeline status in merge requests

### Pipeline Variables

- Define variables in project settings
- Use CI/CD variables for sensitive data
- Masked and protected variables available
