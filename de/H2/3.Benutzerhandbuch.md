# H2 Datenbank - Benutzerhandbuch

## Grundlegende Operationen

### Datenbank erstellen

Eingebetteter Modus (In-Memory-Datenbank):
```java
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");
```

Dateidatenbank:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Tabelle erstellen

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Daten einfügen

```sql
INSERT INTO users (name, email) VALUES ('Max', 'max@beispiel.de');
INSERT INTO users (name, email) VALUES ('Anna', 'anna@beispiel.de');
```

### Daten abfragen

```sql
SELECT * FROM users;
SELECT * FROM users WHERE name = 'Max';
SELECT COUNT(*) FROM users;
```

### Daten aktualisieren

```sql
UPDATE users SET email = 'neueemail@beispiel.de' WHERE id = 1;
```

### Daten löschen

```sql
DELETE FROM users WHERE id = 2;
```

## Java-Code-Beispiele

### Vollständiges JDBC-Beispiel

```java
import java.sql.*;

public class H2Example {
    public static void main(String[] args) throws Exception {
        // Treiber laden
        Class.forName("org.h2.Driver");
        
        // Datenbank verbinden
        String url = "jdbc:h2:mem:testdb";
        Connection conn = DriverManager.getConnection(url, "sa", "");
        
        // Tabelle erstellen
        Statement stmt = conn.createStatement();
        stmt.execute("CREATE TABLE users (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100))");
        
        // Daten einfügen
        stmt.execute("INSERT INTO users (name) VALUES ('Max')");
        stmt.execute("INSERT INTO users (name) VALUES ('Anna')");
        
        // Daten abfragen
        ResultSet rs = stmt.executeQuery("SELECT * FROM users");
        while (rs.next()) {
            System.out.println(rs.getInt("id") + ": " + rs.getString("name"));
        }
        
        // Verbindung schließen
        rs.close();
        stmt.close();
        conn.close();
    }
}
```

### PreparedStatement verwenden

```java
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name, email) VALUES (?, ?)");
pstmt.setString(1, "Peter");
pstmt.setString(2, "peter@beispiel.de");
pstmt.executeUpdate();
pstmt.close();
```

### Transaktionen verwenden

```java
conn.setAutoCommit(false);
try {
    Statement stmt = conn.createStatement();
    stmt.execute("INSERT INTO users (name) VALUES ('Klaus')");
    stmt.execute("INSERT INTO users (name) VALUES ('Maria')");
    conn.commit();
} catch (Exception e) {
    conn.rollback();
} finally {
    conn.setAutoCommit(true);
}
```

## Erweiterte Funktionen

### Datenbankmodus-Wechsel

H2 kann die SQL-Syntax anderer Datenbanken simulieren:

```java
// MySQL-Modus
String url = "jdbc:h2:mem:testdb;MODE=MySQL";

// PostgreSQL-Modus
String url = "jdbc:h2:mem:testdb;MODE=PostgreSQL";

// Oracle-Modus
String url = "jdbc:h2:mem:testdb;MODE=Oracle";
```

### Datenbankverschlüsselung

```java
// Verschlüsselte Datenbank verwenden
String url = "jdbc:h2:file:./data/encrypted;CIPHER=AES";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Trace-Funktion

```java
// SQL-Trace aktivieren
String url = "jdbc:h2:mem:testdb;TRACE_LEVEL_FILE=3";
```

### TCP-Server starten

```java
import org.h2.tools.Server;

Server server = Server.createTcpServer().start();
System.out.println("H2 TCP Server started: " + server.getURL());
```

### H2-Konsole verwenden

H2-Webkonsole starten:

```bash
java -jar h2.jar -web
```

Dann im Browser öffnen: http://localhost:8082

## Häufige SQL-Anweisungen

### Datentypen

H2 unterstützt verschiedene Datentypen, einschließlich:
- Ganzzahltypen: INT, INTEGER, BIGINT, SMALLINT, TINYINT
- Gleitkommatypen: FLOAT, REAL, DOUBLE, DECIMAL
- Zeichenkettentypen: VARCHAR, CHAR, CLOB
- Datum/Uhrzeit-Typen: DATE, TIME, TIMESTAMP
- Boolean-Typen: BOOLEAN
- Binäre Typen: BLOB, BINARY

### Indexoperationen

Index erstellen:
```sql
CREATE INDEX idx_name ON users(name);
```

Index löschen:
```sql
DROP INDEX idx_name;
```

### Constraint-Operationen

Primärschlüssel hinzufügen:
```sql
ALTER TABLE users ADD PRIMARY KEY (id);
```

Fremdschlüssel hinzufügen:
```sql
ALTER TABLE orders ADD FOREIGN KEY (user_id) REFERENCES users(id);
```

### Tabellenstruktur anzeigen

```sql
DESCRIBE users;
-- oder
SHOW COLUMNS FROM users;
```

### Alle Tabellen anzeigen

```sql
SHOW TABLES;
```

## Leistungsoptimierung

### Stapelverarbeitung

Stapelverarbeitung zur Leistungsverbesserung verwenden:

```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (int i = 0; i < 1000; i++) {
    pstmt.setString(1, "Benutzer" + i);
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Verbindungspool

In Produktionsumgebungen wird die Verwendung eines Verbindungspools wie HikariCP empfohlen:

```java
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:h2:file:./data/testdb");
config.setUsername("sa");
config.setPassword("");
config.setMaximumPoolSize(10);

HikariDataSource ds = new HikariDataSource(config);
```

## Sicherung und Wiederherstellung

### Datenbank sichern

```sql
SCRIPT TO 'backup.sql';
```

### Datenbank wiederherstellen

```sql
RUNSCRIPT FROM 'backup.sql';
```

## Umgang mit häufigen Fehlern

### Datenbankdatei gesperrt
Stellen Sie sicher, dass keine anderen Verbindungen die Datenbankdatei verwenden. H2 unterstützt gleichzeitiges Lesen, aber Schreiben erfordert exklusiven Zugriff.

### In-Memory-Daten gehen verloren
In-Memory-Datenbanken verlieren Daten nach dem Schließen der Verbindung. Für Persistenz den Dateidatenbankmodus verwenden.

### Klasse nicht gefunden
Stellen Sie sicher, dass die Klasse org.h2.Driver im Classpath ist.
