# Spring Boot Preguntas Frecuentes

## Preguntas Generales

### ¿Qué es Spring Boot?
Spring Boot es un framework Java de código abierto que simplifica el desarrollo de aplicaciones empresariales proporcionando auto-configuración, servidores embebidos y características listas para producción.

### ¿Por qué debería usar Spring Boot?
Spring Boot reduce el código boilerplate, simplifica la configuración y acelera el desarrollo. Te permite enfocarte en la lógica de negocio en lugar de la infraestructura.

### ¿Cuál es la diferencia entre Spring y Spring Boot?
| Aspecto | Spring | Spring Boot |
|---------|--------|-------------|
| Configuración | XML/Annotation-based | Auto-configuración |
| Servidor | Externo (Tomcat, etc.) | Embebido |
| Dependencias | Gestión manual | Starter POMs |
| Desarrollo | Verboso | Conciso |

## Preguntas de Instalación

### ¿Cómo creo un proyecto Spring Boot?
1. Usa Spring Initializr (https://start.spring.io)
2. Usa IDE (IntelliJ, Eclipse) con plugin Spring Boot
3. Usa Spring Boot CLI: `spring init --name myapp`

### ¿Qué versión de Java requiere Spring Boot?
- Spring Boot 2.x: Java
- Spring Boot 8-17 3.x: Java 17+

### ¿Cómo agrego dependencias?
Agrega starters a tu `pom.xml`:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

## Preguntas de Uso

### ¿Cómo creo un REST controller?
```java
@RestController
@RequestMapping("/api")
public class HolaController {
    
    @GetMapping("/hola")
    public String hola() {
        return "¡Hola, Mundo!";
    }
}
```

### ¿Cómo me conecto a una base de datos?
```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: secret
```

```java
@Repository
public interface UsuarioRepository extends JpaRepository<Usuario, Long> {
}
```

### ¿Cómo habilito seguridad?
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

## Preguntas de Configuración

### ¿Cuál es la diferencia entre @SpringBootApplication y @EnableAutoConfiguration?
- @SpringBootApplication: Combina @Configuration, @EnableAutoConfiguration y @ComponentScan
- @EnableAutoConfiguration: Habilita el mecanismo de auto-configuración de Spring Boot

### ¿Cómo uso perfiles?
```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/devdb
```

Activar con: `spring.profiles.active=dev`

### ¿Cómo creo auto-configuración personalizada?
```java
@Configuration
@ConditionalOnClass(RedisTemplate.class)
public class RedisAutoConfiguration {
    @Bean
    public RedisTemplate<String, Object> redisTemplate() {
        // configuración
    }
}
```

## Preguntas de Despliegue

### ¿Cómo creo un JAR ejecutable?
```bash
./mvnw clean package
java -jar target/myapp-1.0.0.jar
```

### ¿Cómo despliego en Docker?
```dockerfile
FROM eclipse-temurin:17-jre
COPY target/myapp.jar app.jar
ENTRYPOINT ["java", "-jar", "app.jar"]
```

```bash
docker build -t myapp .
docker run -p 8080:8080 myapp
```

## Solución de Problemas

### La aplicación falla al iniciar
- Verificar dependencias faltantes
- Revisar trace de pila para errores de configuración
- Verificar conexiones a base de datos
- Verificar disponibilidad de puerto

### Error de Bean no encontrado
- Asegurar que component scanning esté habilitado
- Verificar que el bean esté en el paquete correcto
- Verificar anotación correcta

### Errores OutOfMemory
- Aumentar tamaño del heap JVM: `-Xmx2g`
- Optimizar consultas a base de datos
- Verificar fugas de memoria

## Mejores Prácticas

1. Usa application.yml sobre application.properties
2. Mantén configuración externalizada
3. Usa inyección de constructor sobre inyección de campos
4. Escribe pruebas unitarias e integración
5. Usa perfiles para configuraciones específicas del entorno
6. Habilita características listas para producción
