# Руководство пользователя

В этом разделе описаны основные способы использования MultiTalk для генерации видео.

## Основные режимы работы

MultiTalk поддерживает несколько режимов генерации видео:

### Режим стриминга (streaming)

Используется для генерации длинных видео:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_long_exp
```

### Режим клипа (clip)

Используется для генерации коротких видео:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode clip \
 --use_teacache \
 --save_file single_clip_exp
```

## Параметры генерации

### Основные параметры

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|----------------------|
| --ckpt_dir | Путь к директории с моделью | weights/Wan2.1-I2V-14B-480P |
| --wav2vec_dir | Путь к аудио энкодеру | weights/chinese-wav2vec2-base |
| --input_json | Путь к JSON файлу с данными | examples/*.json |
| --sample_steps | Количество шагов семплирования | 40 |
| --mode | Режим работы | streaming или clip |
| --save_file | Имя выходного файла | любое |

### Параметры разрешения

```bash
--size multitalk-480  # Генерация 480P видео
--size multitalk-720  # Генерация 720P видео
```

### Параметры ускорения

```bash
--use_teacache           # Использовать TeaCache ускорение
--teacache_thresh 0.3    # Коэффициент ускорения (0.2-0.5)
```

### Параметры качества

```bash
--sample_text_guide_scale 5   # Масштаб текстовой инструкции (без LoRA: 5, с LoRA: 1)
--sample_audio_guide_scale 4  # Масштаб аудио инструкции (без LoRA: 4, с LoRA: 2)
--sample_shift 2              # Сдвиг семплирования
```

## Использование с LoRA

MultiTalk поддерживает использование LoRA для ускорения генерации и улучшения качества:

### С FusioniX LoRA (8 шагов)

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --save_file fusionx_exp \
 --sample_shift 2
```

### С lightx2v LoRA (4 шага)

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan21_T2V_14B_lightx2v_cfg_step_distill_lora_rank32.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 4 \
 --mode streaming \
 --save_file lightx2v_exp
```

## Генерация с низким потреблением памяти

```bash
python generate_multitalk.py \

 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file low_vram_exp
```

## Многопользовательская генерация (Multi-GPU)

Для использования нескольких GPU:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## Квантование модели

Для уменьшения потребления памяти можно использовать INT8 квантование:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir 'weights/chinese-wav2vec2-base' \
 --input_json examples/multitalk_example_2.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --quant int8 \
 --quant_dir weights/MeiGen-MultiTalk \
 --num_persistent_param_in_dit 0 \
 --save_file quant_exp
```

## Советы по оптимизации

### Точность синхронизации губ

- Значение Audio CFG оптимально в диапазоне 3-5
- Увеличьте значение Audio CFG для лучшей синхронизации

### Длина видеоклипа

- Модель обучена на видео 81 кадр при 25 FPS
- Для оптимального следования инструкциям генерируйте клипы по 81 кадру
- Максимально возможно до 201 кадра

### Ускорение генерации

- Уменьшите количество шагов семплирования до 10 для быстрой генерации
- Это не сильно влияет на точность синхронизации губ, но может ухудшить качество движения

### TeaCache ускорение

- Оптимальный диапазон для --teacache_thresh: 0.2-0.5
- Увеличение значения может улучшить ускорение, но может снизить качество
