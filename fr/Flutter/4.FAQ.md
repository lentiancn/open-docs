# Flutter - FAQ

Questions fréquentes et réponses sur le développement Flutter.

## Problèmes d'Installation

### Q: Échec du téléchargement de Flutter ?

**Solution :**
1. Utiliser un miroir :
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
export PUB_CACHE=/chemin/vers/cache
```
2. Utiliser un proxy
3. Télécharger manuellement l'archive SDK

### Q: Erreur flutter doctor ?

**Solution :**
1. Vérifier la configuration PATH
2. S'assurer que Git est installé
3. Exécuter `flutter doctor -v` pour des informations détaillées

### Q: Android SDK non trouvé ?

**Solution :**
```bash
flutter config --android-sdk /chemin/vers/android-sdk

# Ou définir une variable d'environnement
export ANDROID_SDK_ROOT=/chemin/vers/android-sdk
```

### Q: Outils de ligne de commande Xcode non installés sur macOS ?

**Solution :**
```bash
xcode-select --install
```

## Problèmes de Développement

### Q: Hot reload ne fonctionne pas ?

**Solution :**
1. Sauvegarder le fichier et attendre le hot reload
2. Vérifier la console pour les erreurs
3. Essayer de redémarrer complètement l'application

### Q: Échec du build APK ?

**Solution :**
1. Exécuter `flutter clean`
2. Mettre à jour Gradle
3. Vérifier la connexion réseau

### Q: Échec du démarrage de l'émulateur ?

**Solution :**
1. Activer la virtualisation CPU
2. Vérifier l'installation HAXM
3. Utiliser un appareil physique pour le débogage

### Q: Widget ne s'affiche pas ?

**Solution :**
1. Vérifier la valeur de retour de la méthode build()
2. Confirmer ajouté à MaterialApp
3. Vérifier les contraintes de layout

### Q: État ne se met pas à jour ?

**Solution :**
1. S'assurer que setState() est appelé
2. Vérifier que StatefulWidget est correct
3. Confirmer l'utilisation du bon Context

### Q: Fuite de mémoire ?

**Solution :**
1. Disposer correctement les contrôleurs
2. Nettoyer Provider correctement
3. Éviter de créer des objets dans build

## Problèmes de Performance

### Q: Application lente ?

**Solution :**
1. Utiliser les constructeur const
2. Éviter les setState inutiles
3. Utiliser ListView.builder
4. Optimiser le chargement des images

### Q: APK trop volumineux ?

**Solution :**
1. Utiliser la minification
2. Supprimer les symboles de debug
3. Utiliser split-per-abi
4. Nettoyer les ressources non utilisées

### Q: Démarrage lent ?

**Solution :**
1. Réduire les opérations init()
2. Charger les composants de manière paresseuse
3. Utiliser Deferred Components

## Problèmes de Plateforme

### Q: Échec du build iOS ?

**Solution :**
1. Vérifier la version Xcode
2. Mettre à jour CocoaPods
3. Vérifier les certificats de signature

### Q: Échec du build Web ?

**Solution :**
1. Confirmer le support Flutter Web
2. Vérifier la compatibilité du navigateur
3. Mettre à jour Chrome

### Q: Canal de plateforme ne fonctionne pas ?

**Solution :**
1. Vérifier la correspondance des noms de méthode
2. Confirmer les types de retour
3. Vérifier les erreurs de console
