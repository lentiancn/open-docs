# Preguntas Frecuentes de Apache APISIX

## Preguntas Básicas

### 1. ¿Qué es Apache APISIX?

Apache APISIX es una puerta de enlace API nativa para la nube gestionada por Apache Software Foundation. Proporciona gestión de tráfico API dinámica, en tiempo real y de alto rendimiento.

### 2. Diferencia entre APISIX y Nginx

Nginx es un servidor web tradicional, mientras APISIX es una puerta de enlace API construida sobre Nginx. La principal diferencia es que APISIX soporta actualizaciones sin reinicio.

### 3. ¿Qué dependencias necesita APISIX?

- etcd: Para almacenamiento de configuración
- Lua: Para lógica de plugins
- Nginx: Para procesamiento de solicitudes HTTP

### 4. Rendimiento de APISIX

APISIX puede alcanzar 18,000 QPS por núcleo con latencia promedio menor a 0.2ms.

### 5. Plataformas Soportadas

Linux, macOS, Windows (vía WSL/Docker), Kubernetes

## Instalación

### 6. Error en Script de Inicio Rápido

- Verificar que Docker esté ejecutándose
- Verificar que los puertos no estén en uso
- Verificar conectividad de red

### 7. Cambiar Puertos

Modificar docker-compose.yml o config.yaml

### 8. Actualizar APISIX

```bash
docker pull apache/apisix:nueva-versión
```

## Configuración

### 9. Cambiar Clave Admin API

Editar config.yaml y reiniciar APISIX

### 10. Habilitar HTTPS

Usar plugin acme o configurar certificados manualmente

### 11. Múltiples Dominios

Usar campo "hosts" al crear rutas

## Funciones

### 12. Implementar Release Canario

Usar plugin traffic-split con pesos

### 13. Implementar Cortacircuito

Configurar health checks con opciones pasivas

### 14. Habilitar CORS

Usar plugin cors

### 15. Limitación de Tasa

Usar plugins limit-count, limit-req o limit-concurrency

### 16. Autenticación con Consumers

Crear consumer y habilitar plugin en ruta

## Monitoreo

### 17. Integrar Prometheus

Habilitar plugin prometheus

### 18. Logs Externos

Usar plugins http-logger, kafka-logger, etc.

## Seguridad

### 19. Recomendaciones de Seguridad

- Cambiar claves API por defecto
- Restringuir acceso a Admin API
- Usar HTTPS
- Actualizar regularmente

### 20. Prevenir Acceso Malicioso

- Usar lista negra de IP
- Habilitar validación de solicitudes
- Usar protección CSRF

## Resolución de Problemas

### 21. Solicitud Retorna 404

- Verificar que la ruta exista
- Verificar coincidencia de URI
- Verificar que upstream esté alcanzable

### 22. Solicitud Retorna 503

- Verificar que nodos upstream estén funcionando
- Verificar conectividad de red
- Revisar configuración de health checks

### 23. Ver Logs

```bash
docker logs apisix
```
