# Vue.js Manuel d'Utilisation

## Instance d'Application

Chaque application Vue commence par la création d'une nouvelle instance d'application avec la fonction `createApp` :

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* options du composant racine */
})
```

### Composant Racine
L'objet que nous passons à `createApp` est en fait un composant. Chaque application nécessite un "composant racine" qui peut contenir d'autres composants comme ses enfants.

### Montage de l'Application
Une instance d'application ne rend rien tant que sa méthode `.mount()` n'est pas appelée :

```javascript
app.mount('#app')
```

### Configurations de l'Application
L'instance d'application expose un objet `.config` qui permet de configurer des options au niveau de l'application :

```javascript
// Gestionnaire d'erreurs au niveau de l'application
app.config.errorHandler = (err) => {
  /* gérer l'erreur */
}

// Enregistrer un composant global
app.component('TodoDeleteButton', TodoDeleteButton)
```

### Multiples Instances d'Application
Vous pouvez créer plusieurs applications Vue sur la même page :

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## Syntaxe des Templates

Vue utilise une syntaxe de template basée sur HTML qui vous permet de lier de manière déclarative le DOM rendu aux données de l'instance de composant sous-jacente.

### Interpolation de Texte
La forme la plus basique de liaison de données est l'interpolation de texte utilisant la syntaxe "Mustache" (accolades doubles) :

```html
<span>Message: {{ msg }}</span>
```

### HTML Brut
Utilisez la directive `v-html` pour afficher du HTML réel :

```html
<p v-html="rawHtml"></p>
```

**Avertissement de Sécurité** : Le rendu dynamique de HTML arbitraire sur votre site web peut être très dangereux car il peut facilement conduire à des vulnérabilités XSS. Utilisez `v-html` uniquement sur du contenu de confiance et jamais sur du contenu fourni par l'utilisateur.

### Liaisons d'Attributs
Utilisez la directive `v-bind` pour lier les attributs HTML :

```html
<!-- Syntaxe complète -->
<div v-bind:id="dynamicId"></div>

<!-- Raccourci -->
<div :id="dynamicId"></div>

<!-- Raccourci même nom (Vue 3.4+) -->
<div :id></div>
```

### Attributs Booléens
Les attributs booléens sont inclus lorsque la valeur liée est vraie :

```html
<button :disabled="isButtonDisabled">Button</button>
```

### Liaison Dynamique de Multiples Attributs
Lie les multiples attributs depuis un objet en utilisant `v-bind` sans argument :

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## Expressions JavaScript

Vue prend en charge toute la puissance des expressions JavaScript dans toutes les liaisons de données :

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### Appel de Fonctions
Il est possible d'appeler une méthode exposée par le composant à l'intérieur d'une expression de liaison :

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### Accès Restreint aux Globales
Les expressions de template sont sandboxées et n'ont accès qu'à une [liste restreinte de globales](https://github.com/vuejs/core/blob/main/packages/shared/src/globalsAllowList.ts#L3). La liste expose les globales intégrées couramment utilisées telles que Math et Date.

Les globales non explicitement incluses dans la liste, par exemple les propriétés attachées par l'utilisateur sur window, ne seront pas accessibles dans les expressions de template. Cependant, vous pouvez définir explicitement des globales supplémentaires pour toutes les expressions Vue en les ajoutant à [app.config.globalProperties](/api/application#app-config-globalproperties).

## Directives

Les directives sont des attributs spéciaux avec le préfixe `v-`.

### Arguments
Certaines directives peuvent prendre un "argument", indiqué par un deux-points après le nom de la directive. Par exemple, la directive `v-bind` est utilisée pour mettre à jour de manière réactive un attribut HTML :

```html
<!-- Argument v-bind -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- Argument v-on -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### Arguments Dynamiques
Il est également possible d'utiliser une expression JavaScript dans un argument de directive en l'entourant de crochets :

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**Contraintes** :
- Les arguments dynamiques doivent s'évaluer à une chaîne de caractères ou null
- Évitez les expressions complexes ; utilisez plutôt des propriétés calculées
- Dans les templates DOM, évitez les majuscules dans les clés dynamiques

### Modificateurs
Les modificateurs sont des suffixes spéciaux indiqués par un point, qui indiquent qu'une directive doit être liée d'une manière spéciale. Par exemple, le modificateur `.prevent` indique à la directive `v-on` d'appeler `event.preventDefault()` sur l'événement déclenché :

```html
<form @submit.prevent="onSubmit">...</form>
```

## Fondamentaux de la Réactivité

### Déclaration d'un État Réactif
Créez des références réactives avec `ref()` :

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### Objets Réactifs
Créez des objets réactifs avec `reactive()` :

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### Propriétés Calculées
Créez des propriétés calculées avec `computed()` :

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### Observateurs
Observez les changements de données réactives avec `watch()` :

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// Observer une seule référence
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## Bases des Composants

### Définition des Composants
```javascript
// Définir le composant
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Passage de Props
```javascript
// Composant enfant
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// Composant parent
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### Émission d'Événements
```javascript
// Composant enfant
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### Slots
```html
<!-- Composant enfant -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- Composant parent -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
