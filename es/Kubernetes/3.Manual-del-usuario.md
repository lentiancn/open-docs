# Manual del Usuario de Kubernetes

## Conceptos Básicos

### Pod
Pod es la unidad más pequeña desplegable en Kubernetes, representando uno o más contenedores ejecutándose en el clúster. Típicamente un Pod contiene un solo contenedor, pero también puede contener múltiples contenedores fuertemente acoplados.

### Deployment
Los Deployments se usan para gestionar aplicaciones sin estado, proporcionando capacidades de actualización declarativa para fácil despliegue, escalado y actualizaciones.

### Service
Los Services proporcionan puntos finales de red estables para un conjunto de Pods, manteniendo la estabilidad incluso cuando las direcciones IP de los Pods cambian.

### Namespace
Los Namespaces dividen los recursos del clúster en múltiples particiones lógicas, adecuadas para múltiples equipos o proyectos que comparten el mismo clúster.

## Comandos Comunes

### Gestión del Clúster

```bash
# Ver información del clúster
kubectl cluster-info

# Ver nodos
kubectl get nodes

# Ver todos los Pods
kubectl get pods --all-namespaces

# Ver uso de recursos del clúster
kubectl top nodes
kubectl top pods
```

### Despliegue de Aplicaciones

```bash
# Crear un Deployment
kubectl create deployment nginx --image=nginx

# Ver Deployments
kubectl get deployments

# Escalar
kubectl scale deployment nginx --replicas=3

# Actualizar imagen
kubectl set image deployment/nginx nginx=nginx:1.21

# Retrotracción
kubectl rollout undo deployment/nginx

# Ver estado del rollout
kubectl rollout status deployment/nginx
```

### Gestión de Servicios

```bash
# Exponer un Servicio
kubectl expose deployment nginx --port=80 --type=LoadBalancer

# Ver Servicios
kubectl get services

# Eliminar Servicio
kubectl delete service <service-name>
```

### Depuración

```bash
# Ver logs del Pod
kubectl logs <pod-name>

# Ejecutar comando en contenedor
kubectl exec -it <pod-name> -- /bin/bash

# Describir detalles del recurso
kubectl describe pod <pod-name>

# Reenvío de puerto para depuración
kubectl port-forward <pod-name> 8080:80
```

## Manifiestos de Recursos

Kubernetes soporta gestión declarativa de recursos usando archivos YAML. Ejemplos de definiciones de recursos comunes:

### Ejemplo de Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
```

### Ejemplo de Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: ClusterIP
```

### Ejemplo de ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  database_url: "postgres://db:5432/myapp"
  cache_enabled: "true"
```

### Ejemplo de Secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: changeme
```

## Gestión del Ciclo de Vida de Aplicaciones

### Desplegar Aplicaciones

```bash
# Desplegar usando archivo YAML
kubectl apply -f deployment.yaml

# Ver estado del despliegue
kubectl get pods -l app=my-app
```

### Actualizar Aplicaciones

```bash
# Actualizar versión de imagen
kubectl set image deployment/my-app my-app=my-app:v2

# Ver progreso de actualización
kubectl rollout status deployment/my-app
```

### Retrotracción

```bash
# Retrotracción a versión anterior
kubectl rollout undo deployment/my-app

# Retrotracción a revisión específica
kubectl rollout undo deployment/my-app --to-revision=2

# Ver historial de revisiones
kubectl rollout history deployment/my-app
```

### Escalado

```bash
# Escalado manual
kubectl scale deployment my-app --replicas=5

# Auto-escalado (HPA)
kubectl autoscale deployment my-app --min=2 --max=10 --cpu-percent=80
```

## Gestión de Almacenamiento

### Crear PersistentVolume

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  storageClassName: standard
  hostPath:
    path: /data/my-app
```

### Crear PersistentVolumeClaim

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

### Usar PVC en Pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: my-app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

## Gestión de Configuración

### Usar ConfigMap

```bash
# Crear ConfigMap desde archivo
kubectl create configmap my-config --from-file=config.properties

# Crear desde archivo de variables de entorno
kubectl create configmap my-config --from-env-file=env.properties

# Usar en Pod
kubectl apply -f pod-with-configmap.yaml
```

### Usar Secret

```bash
# Crear Secret desde archivo
kubectl create secret generic my-secret --from-file=credentials.json

# Usar en Pod
kubectl apply -f pod-with-secret.yaml
```

## Cuotas y Límites de Recursos

### ResourceQuota

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: my-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    pods: "20"
```

### LimitRange

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: my-limits
spec:
  limits:
  - max:
      cpu: "2"
      memory: 1Gi
    min:
      cpu: 100m
      memory: 128Mi
    default:
      cpu: 500m
      memory: 512Mi
    defaultRequest:
      cpu: 200m
      memory: 256Mi
    type: Container
```

## Monitoreo y Registros

### Monitoreo

```bash
# Ver uso de recursos con kubectl top
kubectl top nodes
kubectl top pods

# Ver estado detallado
kubectl describe node <node-name>
kubectl describe pod <pod-name>
```

### Registros

```bash
# Ver logs del Pod
kubectl logs my-app-pod

# Ver logs en tiempo real
kubectl logs -f my-app-pod

# Ver logs de instancia de contenedor anterior
kubectl logs --previous my-app-pod
```

## Mejores Prácticas

1. Usar configuración declarativa para gestionar infraestructura
2. Establecer límites y solicitudes de recursos apropiados
3. Organizar recursos usando etiquetas
4. Limpiar regularmente recursos no utilizados
5. Usar namespaces para aislar diferentes entornos
6. Configurar sondeos de verificación de salud
7. Usar RBAC para controlar permisos
8. Respaldar regularmente datos de etcd
