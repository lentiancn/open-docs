# Manuel Utilisateur Hadoop

## Guide des Opérations HDFS

### Opérations de Base du Système de Fichiers

HDFS fournit une interface de ligne de commande similaire aux systèmes de fichiers traditionnels, permettant aux utilisateurs d'effectuer diverses opérations de fichiers via la commande hdfs dfs. Ces commandes sont très similaires aux commandes shell Linux, donc les utilisateurs familiarisés avec les systèmes Linux peuvent s'adapter rapidement. Comprendre ces opérations de base est fondamental pour utiliser Hadoop ; nous détaillons ci-dessous les commandes HDFS les plus utilisées.

La création de répertoires utilise la commande mkdir, qui peut créer des répertoires à un ou plusieurs niveaux :

```bash
# Créer un répertoire à un niveau
hdfs dfs -mkdir /user/data

# Créer un répertoire à plusieurs niveaux
hdfs dfs -mkdir -p /user/data/input

# Lister le contenu du répertoire
hdfs dfs -ls /user/data

# Lister le contenu du répertoire récursivement
hdfs dfs -ls -R /user/data
```

Le téléversement et le téléchargement de fichiers sont des opérations courantes :

```bash
# Téléverser un fichier local vers HDFS
hdfs dfs -put localfile.txt /user/data/

# Téléverser et renommer
hdfs dfs -put localfile.txt /user/data/newname.txt

# Télécharger un fichier HDFS en local
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# Copier des fichiers internes HDFS
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# Déplacer des fichiers internes HDFS
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

Visualisation et suppression de fichiers :

```bash
# Voir le contenu du fichier
hdfs dfs -cat /user/data/file.txt

# Voir les dernières lignes du fichier
hdfs dfs -tail /user/data/file.txt

# Voir la taille du fichier
hdfs dfs -du /user/data/file.txt

# Supprimer un fichier
hdfs dfs -rm /user/data/file.txt

# Supprimer un répertoire récursivement
hdfs dfs -rm -r /user/data/dir
```

### Gestion des Permissions

Le modèle de permissions HDFS est similaire à Unix/Linux, basé sur un système à trois niveaux : utilisateur, groupe et autres. Chaque fichier et répertoire a trois permissions : lecture (r), écriture (w) et exécution (x), correspondant au propriétaire, au groupe et aux autres. Comprendre la gestion des permissions est crucial pour protéger la sécurité des données.

```bash
# Changer le propriétaire du fichier
hdfs dfs -chown user:group /user/data/file.txt

# Changer les permissions du fichier
hdfs dfs -chmod 755 /user/data/file.txt

# Changer le groupe propriétaire
hdfs dfs -chgrp groupname /user/data/file.txt
```

### Réplication et Équilibrage des Données

HDFS réplique chaque bloc de données trois fois par défaut, les stockant sur différents DataNodes. Le facteur de réplication par défaut peut être modifié via la propriété dfs.replication :

```bash
# Définir le facteur de réplication pour un fichier spécifique
hdfs dfs -setrep -w 2 /user/data/file.txt

# Voir les informations d'emplacement des blocs du fichier
hdfs fsck /user/data/file.txt -files -blocks -locations
```

Lorsque des nœuds sont ajoutés ou supprimés du cluster, les données peuvent devenir déséquilibrées. Utilisez la commande rebalance pour équilibrer les données :

```bash
start-balancer.sh -threshold 10
```

Le paramètre threshold représente le pourcentage maximal de différence entre l'utilisation du disque d'un nœud et la moyenne du cluster. Le processus d'équilibrage consomme de la bande passante réseau ; il est recommandé de l'exécuter pendant les périodes de faible trafic.

### Interface Web HDFS

HDFS fournit une interface Web riche en fonctionnalités, accessible via le navigateur sur le port 9870 du NameNode. L'interface Web peut parcourir les répertoires du système de fichiers, voir le contenu des fichiers, surveiller l'état des DataNodes, voir les informations sur les blocs, etc. L'interface Web est un outil important pour le personnel opérationnel dans la surveillance quotidienne et le dépannage.

Les principales fonctionnalités de l'interface Web incluent : parcourir l'arborescence des répertoires HDFS, voir les listes de fichiers et les propriétés des fichiers ; voir la liste des DataNodes et l'utilisation du stockage de chaque DataNode ; voir la distribution des blocs et l'état de réplication ; voir les informations d'exécution et les journaux du NameNode ; effectuer des opérations simples sur les fichiers comme le téléchargement et la suppression.

## Programmation MapReduce

### Exemple WordCount

WordCount est l'exemple d'introduction à la programmation MapReduce, comptant les occurrences de chaque mot dans un texte. Comprendre cet exemple aide à maîtriser le modèle de programmation MapReduce. Le programme est divisé en phases Map et Reduce : la phase Map divise le texte en mots, et la phase Reduce agrège les comptages des mêmes mots.

Exemple de code de la phase Map :

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Exemple de code de la phase Reduce :

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### Soumission de Travail

Après avoir empaqueté MapReduce dans un fichier JAR, soumettez le travail en utilisant la commande hadoop jar :

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

Après la soumission du travail, vous pouvez voir la progression et les résultats du travail sur l'interface Web YARN (port 8088). Lorsque le travail se termine, il génère des statistiques sur le travail, y compris l'ID du travail, le nombre de tâches, le temps d'exécution, etc.

### Configuration du Travail

Divers paramètres du travail peuvent être configurés via l'objet Configuration :

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

Les paramètres de configuration courants incluent : mapreduce.job.reduces définit le nombre de tâches Reduce ; mapreduce.map.memory.mb et mapreduce.reduce.memory.mb définissent la mémoire du conteneur ; mapreduce.map.cpu.vcores et mapreduce.reduce.cpu.vcores définissent les cœurs CPU du conteneur.

## Opérations YARN

### Visualisation des Ressources du Cluster

YARN fournit des outils de ligne de commande riches pour gérer les ressources du cluster :

```bash
# Voir l'état de tous les nœuds
yarn node -list -all

# Voir la liste des applications
yarn application -list

# Voir l'état d'une application spécifique
yarn application -status application_1234567890123_0001

# Voir les journaux d'une application
yarn logs -applicationId application_1234567890123_0001
```

### Gestion des Files d'Attente

YARN utilise des files d'attente pour gérer l'allocation des ressources. Les files d'attente peuvent être configurées via l'Ordonnanceur de Capacité ou l'Ordonnanceur Équitable :

```bash
# Voir les informations de file d'attente
yarn queue -status default

# Voir la capacité de la file d'attente
yarn resourcemanager -refreshQueues
```

### Configuration de l'Ordonnanceur de Ressources

L'Ordonnanceur de Capacité est l'ordonnanceur par défaut, supportant le partage de ressources multi-locataire. Configurez dans yarn-site.xml :

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## Optimisation des Performances

### Ajustement MapReduce

L'optimisation des performances des travaux MapReduce implique plusieurs aspects. Tout d'abord, définissez raisonner le nombre de tâches Map et Reduce. Le nombre de tâches Map est généralement déterminé par le nombre de divisions des données d'entrée, qui peut être ajusté via le paramètre mapreduce.input.fileinputformat.split.maxsize. Le nombre de tâches Reduce est recommandé d'être défini entre 0,9 et 1,2 fois le nombre total d'emplacements Reduce du cluster.

```bash
# Définir le nombre de tâches Map
-D mapreduce.job.maps=10

# Définir le nombre de tâches Reduce
-D mapreduce.job.reduces=5
```

L'optimisation de la phase de shuffle a un impact significatif sur les performances globales. Ajustez ces paramètres :

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### Ajustement HDFS

L'optimisation des performances HDFS se concentre principalement sur l'efficacité de lecture/écriture des données. Pour les gros fichiers, ajustez la taille des blocs pour équilibrer la localité des données et la récupération après panne :

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

Activez les lectures en direct pour permettre aux clients de lire directement les données des DataNodes locaux, réduisant le transfert réseau :

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### Ajustement YARN

L'allocation des ressources YARN a un impact significatif sur l'utilisation du cluster. Définissez raisonnablement la capacité des ressources du NodeManager :

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

Définissez les cœurs CPU virtuels pour mieux utiliser les processeurs multi-cœurs :

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## Importation et Exportation de Données

### Importation de Données depuis des Bases de Données

Utilisez Sqoop pour migrer commodément les données entre les bases de données relationnelles et HDFS :

```bash
# Importer une table entière
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Importer et créer une table Hive
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### Exportation de Données vers des Bases de Données

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### Sélection du Format de Données

Différents formats de données ont des impacts différents sur les performances. Les formats couramment utilisés incluent : TextFile (bonne généricité, mais performances moyennes) ; SequenceFile (bon pour MapReduce) ; Parquet (stockage en colonnes, bon pour les requêtes analytiques) ; ORC (stockage en colonnes optimisé pour Hive). Pour les charges de travail analytiques, utilisez des formats de stockage en colonnes comme Parquet ou ORC.
