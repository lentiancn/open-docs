# Flutter - FAQ

Häufige Fragen und Antworten zur Flutter-Entwicklung.

## Installationsprobleme

### Q: Flutter-Download fehlgeschlagen?

**Lösung:**
1. Spiegelquelle verwenden:
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
export PUB_CACHE=/path/to/cache
```

2. Proxy verwenden
3. SDK-Zip-Datei manuell herunterladen

### Q: flutter doctor-Fehler?

**Lösung:**
1. PATH-Konfiguration prüfen
2. Git ist installiert
3. `flutter doctor -v` für detaillierte Informationen ausführen

### Q: Android SDK nicht gefunden?

**Lösung:**
```bash
# Android SDK-Pfad konfigurieren
flutter config --android-sdk /path/to/android-sdk

# Oder Umgebungsvariable setzen
export ANDROID_SDK_ROOT=/path/to/android-sdk
```

### Q: macOS Xcode Command Line Tools nicht installiert?

**Lösung:**
```bash
xcode-select --install
```

## Entwicklungsprobleme

### Q: Hot Reload funktioniert nicht?

**Lösung:**
1. Datei speichern und auf Hot Reload warten
2. Konsole auf Fehler prüfen
3. Anwendung vollständig neu starten versuchen

### Q: APK-Build fehlgeschlagen?

**Lösung:**
 clean` ausf1. `flutterühren
2. Gradle aktualisieren
3. Netzwerkverbindung prüfen

### Q: Emulator-Start fehlgeschlagen?

**Lösung:**
1. CPU-Virtualisierung aktivieren
2. HAXM-Installation prüfen
3. Echtgerät zum Debuggen verwenden

### Q: Widget wird nicht angezeigt?

**Lösung:**
1. build()-Methodenrückgabe prüfen
2. Zu MaterialApp hinzugefügt
3. Layout-Einschränkungen prüfen

### Q: Zustand wird nicht aktualisiert?

**Lösung:**
1. setState() wird verwendet
2. StatefulWidget korrekt
3. Korrekter Context verwendet

### Q: Speicherleck?

**Lösung:**
1. Controller korrekt entsorgen
2. Bei Provider korrekt aufräumen
3. Objekterstellung in build vermeiden

## Leistungsprobleme

### Q: Anwendung ruckelt?

**Lösung:**
1. const-Konstruktoren verwenden
2. Unnötiges setState vermeiden
3. ListView.builder verwenden
4. Bildladung optimieren

### Q: APK zu groß?

**Lösung:**
1. Minifizierung verwenden
2. Debug-Symbole entfernen
3. split-per-abi verwenden
4. Ungenutzte Ressourcen bereinigen

### Q: Startzeit zu lang?

**Lösung:**
1. init()-Operationen reduzieren
2. Komponenten verzögert laden
3. Deferred Components verwenden

## Plattformprobleme

### Q: iOS-Build fehlgeschlagen?

**Lösung:**
1. Xcode-Version prüfen
2. CocoaPods aktualisieren
3. Signaturzertifikate prüfen

### Q: Web-Build fehlgeschlagen?

**Lösung:**
1. Flutter Web-Unterstützung bestätigen
2. Browser-Kompatibilität prüfen
3. Chrome aktualisieren

### Q: Plattformkanal funktioniert nicht?

**Lösung:**
1. Methodennamenübereinstimmung prüfen
2. Rückgabetypen bestätigen
3. Konsolenfehler anzeigen
