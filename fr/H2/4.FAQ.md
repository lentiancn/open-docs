# Foire aux questions sur H2

## Problèmes d'installation

### Q : Comment télécharger la base de données H2 ?

R : Visitez https://www.h2database.com/html/download.html pour télécharger la version adaptée à votre plateforme. Il y a deux options : installateur Windows et package ZIP multiplateforme.

### Q : Quelle version de Java est nécessaire pour H2 ?

R : H2 nécessite Java 11 ou supérieur.

### Q : Comment utiliser H2 dans un projet Maven ?

R : Ajoutez la dépendance dans pom.xml :
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### Q : Comment exécuter H2 après avoir décompressé le ZIP ?

R : Exécutez dans la ligne de commande :
```bash
java -jar h2.jar
```
Cela démarrera la console web H2.

## Problèmes de connexion

### Q : Comment se connecter à la base de données H2 ?

R : Utilisez la connexion JDBC :
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### Q : Quel est le nom d'utilisateur et mot de passe par défaut de H2 ?

R : Le nom d'utilisateur par défaut est "sa", avec un mot de passe vide.

### Q : Comment se connecter à une base de données fichier ?

R :
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Q : Comment démarrer le serveur TCP de H2 ?

R :
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### Q : Que faire si le port est occupé ?

R : Vous pouvez spécifier un autre port dans l'URL de connexion :
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## Problèmes de données

### Q : Les données en mémoire sont-elles perdues ?

R : Oui, la base de données en mémoire (jdbc:h2:mem:) perd les données après la fermeture de toutes les connexions. Pour la persistance, utilisez le mode fichier.

### Q : Comment voir l'emplacement du fichier de base de données ?

R : Spécifiez le chemin dans l'URL de connexion :
```java
jdbc:h2:file:/chemin/vers/base de données
```

### Q : Que faire si le fichier de base de données est corrompu ?

R : Essayez le mode de récupération :
```java
jdbc:h2:file:./testdb;LOG=0
```
Ou restaurez à partir d'une sauvegarde.

### Q : Comment chiffrer une base de données ?

R :
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## Problèmes de performances

### Q : Comment améliorer les performances d'insertion par lots ?

R : Utilisez les opérations par lots et désactivez l'auto-commit :
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Q : Que faire si les requêtes sont lentes ?

R : 
1. Créez des index pour les conditions de requête courantes
2. Utilisez EXPLAIN pour analyser le plan de requête
3. Évitez SELECT *, ne Requêtez que les colonnes nécessaires

### Q : Que faire si l'utilisation de la mémoire est trop élevée ?

R : Ajustez la taille du tas JVM et utilisez les pools de connexions de manière raisonnable.

## Problèmes de compatibilité SQL

### Q : Quels modes de base de données H2 prend-il en charge ?

R : H2 peut simuler plusieurs bases de données :
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

Utilisation :
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### Q : Comment gérer les IDs auto-incrémentés ?

R : H2 prend en charge plusieurs méthodes :
```sql
-- Méthode 1 : IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- Méthode 2 : SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## Problèmes de transactions

### Q : Que se passe-t-il si une transaction n'est pas validée ?

R : Les transactions non validées sont automatiquement annulées à la fermeture de la connexion.

### Q : Comment gérer les deadlocks ?

R : H2 détecte automatiquement les deadlocks et annule l'une des transactions. Il est recommandé de raccourcir la portée des transactions pour éviter les verrouillages prolongés.

## Problèmes de sécurité

### Q : La base de données H2 est-elle sécurisée ?

R : H2 convient aux environnements de développement et de test. Pour les environnements de production, il est recommandé de :
1. Utiliser une base de données chiffrée
2. Utiliser des mots de passe forts
3. Limiter l'accès réseau
4. Utiliser des connexions SSL

### Q : Comment prévenir l'injection SQL ?

R : Utilisez PreparedStatement au lieu de la concaténation de chaînes :
```java
// Correct
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// Incorrect
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## Problèmes d'outils

### Q : Comment utiliser la console H2 ?

R : Exécutez `java -jar h2.jar -web` puis accédez à http://localhost:8082

### Q : Comment utiliser le shell H2 ?

R : Exécutez `java -jar h2.jar -shell` ou `java -jar h2.jar -tool`

### Q : Comment voir la version de H2 ?

R :
```bash
java -jar h2.jar -version
```
Ou en SQL : `SELECT H2VERSION();`

## Problèmes de migration

### Q : Comment migrer de MySQL vers H2 ?

R :
1. Configurez le mode de compatibilité H2 : `jdbc:h2:file:db;MODE=MySQL`
2. Exportez les données MySQL en script SQL
3. Exécutez le script dans H2

### Q : Que faut-il注意 lors de la mise à jour de la version de H2 ?

R : La mise à niveau de H2 1.x vers 2.x nécessite des considérations spéciales, veuillez consulter le guide de migration officiel :
https://www.h2database.com/html/migration-to-v2.html
