# Guía de Instalación de Kotlin

## Requisitos Previos

Antes de instalar Kotlin, asegúrate de que tu computadora cumpla con los requisitos básicos del sistema. Kotlin puede ejecutarse en múltiples sistemas operativos incluyendo Windows, macOS, Linux y otros sistemas similares a Unix. Independientemente del sistema operativo que uses, asegúrate de tener suficiente espacio en disco para instalar herramientas de desarrollo y ejecutar programas compilados.

Para usuarios de Windows, se recomienda Windows 10 o superior para la mejor experiencia. Los usuarios de macOS necesitan macOS 10.14 o superior. Los usuarios de Linux pueden usar cualquier distribución principal como Ubuntu, Fedora o Debian.

## Instalación mediante IDE

### IntelliJ IDEA

IntelliJ IDEA es el entorno de desarrollo integrado insignia de JetBrains y proporciona el soporte más completo para Kotlin. Si tu objetivo principal es el desarrollo con Kotlin, IntelliJ IDEA es la mejor opción.

IntelliJ IDEA viene en dos versiones: Community (gratuita y de código abierto) y Ultimate (para usuarios empresariales con características avanzadas). La versión Community es gratuita y proporciona suficientes características para el desarrollo con Kotlin. La versión Ultimate ofrece características empresariales adicionales como herramientas avanzadas de base de datos y funciones de seguridad empresarial.

Para instalar, visita el sitio web oficial de JetBrains (https://www.jetbrains.com/idea/download/) y descarga el instalador apropiado para tu sistema operativo. Después de descargar, ejecuta el instalador y sigue el asistente para completar la instalación. Durante la instalación, puedes elegir la ruta de instalación y crear accesos directos en el escritorio.

Cuando lancés IntelliJ IDEA por primera vez, el IDE detectará automáticamente y te pedirá que instales el plugin de Kotlin. Si no detecta automáticamente, puedes buscar manualmente Kotlin en el mercado de plugins e instalarlo. Después de la instalación, reinicia el IDE para comenzar el desarrollo con Kotlin.

### Android Studio

Android Studio es el IDE oficial de desarrollo de Android de Google con soporte integrado para Kotlin. Si desarrollas principalmente para Android, Android Studio es la opción ideal.

Visita el sitio web de desarrolladores de Android (https://developer.android.com/studio) para descargar Android Studio. Después de descargar, ejecuta el instalador para completar la instalación. Android Studio incluye el plugin de Kotlin por defecto, por lo que no se necesita configuración adicional para comenzar a desarrollar.

Android Studio lanza múltiples versiones de actualización cada año, cada una conteniendo el último soporte de Kotlin y herramientas de desarrollo de Android. Se recomienda mantener tu versión actualizada para obtener la mejor experiencia de desarrollo y las últimas características del lenguaje.

## Instalación mediante Línea de Comandos

Además de usar un IDE, también puedes instalar el compilador de línea de comandos de Kotlin. Este enfoque es adecuado para desarrolladores que prefieren un entorno de desarrollo ligero o necesitan Kotlin para desarrollo del lado del servidor.

### Instalación Manual

Primero, visita la página de Lanzamientos de Kotlin en GitHub (https://github.com/JetBrains/kotlin/releases) para descargar la última versión del compilador. La descarga es un archivo zip; extráelo a un directorio deseado.

Después de extraer, agrega el directorio kotlinc/bin a la variable de entorno PATH de tu sistema para que puedas usar el comando kotlinc desde la línea de comandos. En Windows, agrega la ruta a través de Propiedades del Sistema > Variables de Entorno. En macOS y Linux, modifica tu archivo ~/.bashrc o ~/.zshrc y agrega una línea como:

export PATH="$PATH:/path/to/kotlinc/bin"

Después de agregar la ruta, abre una nueva ventana de terminal e ingresa kotlinc -version. Si ves información de la versión, la instalación fue exitosa.

### Usando SDKMAN!

SDKMAN! es una herramienta para gestionar versiones de herramientas de desarrollo de Java y funciona convenientemente en sistemas similares a Unix. Si usas macOS, Linux, Cygwin, FreeBSD o Solaris, ¡SDKMAN! es la forma recomendada de instalar Kotlin.

Primero, instala SDKMAN! ejecutando este comando en tu terminal:

curl -s "https://get.sdkman.io" | bash

Después de la instalación, recarga la configuración del entorno:

source ~/.sdkman/bin/sdkman-init.sh

Ahora puedes usar el comando sdk para instalar Kotlin:

sdk install kotlin

Durante la instalación, se te pedirá que confirmes la versión. Después de confirmar, la instalación está completa. Puedes verificar la instalación con kotlinc -version.

### Usando Homebrew

En macOS, también puedes usar el administrador de paquetes Homebrew para instalar Kotlin. Primero, asegúrate de que Homebrew esté instalado, luego ejecuta:

brew update
brew install kotlin

Homebrew descargará e instalará automáticamente el último compilador de Kotlin. Verifica la instalación con kotlinc -version.

### Usando Snap

En Ubuntu 16.04 y posteriores, puedes usar Snap para instalar Kotlin:

sudo snap install --classic kotlin

La instalación mediante Snap instala Kotlin como una aplicación accesible desde cualquier lugar a través de la línea de comandos.

## Creando Tu Primer Programa Kotlin

Independientemente del método de instalación que elijas, puedes crear y ejecutar tu primer programa Kotlin con estos pasos.

Primero, crea un archivo llamado hello.kt con el siguiente contenido:

fun main() {
    println("Hello, World!")
}

Este es el programa Kotlin más simple, definiendo una función main que es el punto de entrada de un programa Kotlin. La función println produce texto en la consola.

Si usas el compilador de línea de comandos, compilar y ejecutar este programa requiere dos pasos. Primero, compila:

kotlinc hello.kt -include-runtime -d hello.jar

El parámetro -include-runtime empaqueta la biblioteca de tiempo de ejecución de Kotlin en el archivo jar generado, haciéndolo autocontenido. El parámetro -d especifica la ruta del archivo de salida.

Después de la compilación, ejecuta el programa:

java -jar hello.jar

Deberías ver "Hello, World!" producido en la consola.

Si no quieres generar un archivo jar independiente, también puedes usar la función de ejecución interactiva de kotlinc:

kotlinc -script hello.kt

O usa el comando kotlin para ejecutar archivos class compilados directamente:

kotlin -classpath hello.jar HelloKt

Aquí, HelloKt es el nombre de la clase principal generado por el compilador Kotlin para el archivo llamado hello.kt.

## Usando REPL

Kotlin proporciona un entorno REPL (Read-Eval-Print Loop) interactivo donde puedes ingresar código Kotlin y ver resultados inmediatamente. Esto es útil para aprender características del lenguaje o probar rápidamente fragmentos de código.

Para iniciar REPL, simplemente ejecuta kotlinc sin ningún parámetro:

kotlinc

Esto entra en modo REPL. Puedes ingresar cualquier código Kotlin válido y presionar Enter para ver el resultado:

>>> val x = 10
>>> val y = 20
>>> x + y
30

Ingresa :quit o presiona Ctrl+C para salir del REPL.

## Scripts de Kotlin

Kotlin también puede usarse como lenguaje de scripting. Los archivos de script de Kotlin tienen la extensión .kts, y el código de nivel superior en el archivo se trata como código ejecutable.

Por ejemplo, crea un script llamado list_folders.kts:

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

Ejecuta el script:

kotlinc -script list_folders.kts -- -d /path/to/folder

## Configurando Herramientas de Construcción

En proyectos reales, Gradle o Maven se usan típicamente como herramientas de construcción. Ambos soportan proyectos Kotlin de forma nativa.

### Gradle

Gradle es la herramienta de construcción más popular para el desarrollo con Kotlin. El equipo de Kotlin también proporciona Kotlin DSL, permitiendo que los scripts de construcción se escriban en sintaxis Kotlin.

Para crear un proyecto básico Gradle con Kotlin, agrega el plugin de Kotlin a tu proyecto. Tu archivo build.gradle necesita estos contenidos:

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

O usando Kotlin DSL:

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

Luego crea la estructura del directorio de código fuente, típicamente src/main/kotlin. Agrega archivos fuente Kotlin y usa gradle build para compilar el proyecto.

### Maven

Maven es otra herramienta de construcción popular para proyectos Java que también soporta proyectos Kotlin. Agrega las dependencias de Kotlin a tu archivo pom.xml.

## Verificando la Instalación

Después de la instalación, puedes verificar que todo funcione correctamente.

Primero, verifica el compilador de Kotlin:

kotlinc -version

Deberías ver algo como "Kotlin compiler version 2.3.10".

Luego verifica la biblioteca estándar de Kotlin:

kotlin -e 'println("Hello Kotlin!")'

Si todo funciona, la instalación fue exitosa y puedes comenzar tu viaje con Kotlin.

## Próximos Pasos

Después de instalar Kotlin, se recomienda continuar el aprendizaje leyendo el Tour oficial de Kotlin para entender las características básicas del lenguaje; luego, eligiendo una dirección de desarrollo que te interese como desarrollo Android, desarrollo del lado del servidor o desarrollo multiplataforma; finalmente, practicando con proyectos reales para aplicar lo que has aprendido.

Kotlin proporciona recursos de aprendizaje abundantes incluyendo documentación oficial, tutoriales en línea y cursos de video. Puedes encontrar enlaces a estos recursos en el sitio web de Kotlin.
