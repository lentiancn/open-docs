# Docker ユーザーマニュアル

このマニュアルは，Docker に素早く inúmer，毎日の使用のための一般的な操作をマスターするのに役立ちます。

## 前提条件

始める前に，必ず以下を確認してください：

- Docker がインストールされている（Docker Desktop または Docker Engine）
- コマンドラインターミナルへのアクセス
- コマンドライン操作の基本的な知識

## 基本概念

### イメージとコンテナ

- **イメージ**: コンテナを作成するための読み取り専用テンプレート
- **コンテナ**: イメージの実行中のインスタンス

### 一般的なコマンド

```bash
# コンテナを実行
docker run hello-world

# すべてのコンテナを一覧表示
docker ps -a

# すべてのイメージを一覧表示
docker images

# コンテナを停止
docker stop コンテナID

# コンテナを削除
docker rm コンテナID

# イメージを削除
docker rmi イメージID
```

## コンテナの操作

### コンテナを実行する

コンテナを実行するための基本コマンドは次のとおりです：

```bash
docker run [オプション] イメージ名 [コマンド]
```

例：

```bash
# インタラクティブなコンテナを起動
docker run -it ubuntu bash

# バックグラウンドでコンテナを実行
docker run -d nginx

# ポートマッピングでコンテナを実行
docker run -p 8080:80 nginx
```

### コンテナの 管理

```bash
# 実行中のすべてのコンテナを表示
docker ps

# すべてのコンテナ（停止中の含む）を表示
docker ps -a

# コンテナを起動
docker start コンテナ名

# コンテナを停止
docker stop コンテナ名

# コンテナを再起動
docker restart コンテナ名

# コンテナを削除
docker rm コンテナ名
```

### コンテナログの表示

```bash
# コンテナのログを表示
docker logs コンテナ名

# リアルタイムでログをフォロー
docker logs -f コンテナ名

# 最後の100行を表示
docker logs --tail 100 コンテナ名
```

## イメージの操作

### イメージの検索とダウンロード

```bash
# Docker Hub でイメージをearch
docker search nginx

# イメージををダウンロード
docker pull nginx

# タグ付きイメージををダウンロード
docker pull nginx:latest
```

### イメージの作成

Dockerfile で独自のイメージを作成できます：

```dockerfile
FROM ubuntu:20.04
RUN apt-get update
RUN apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

次に，イメージを構築します：

```bash
docker build -t マイnginxイメージ .
```

### イメージの管理

```bash
# すべてのイメージを一覧表示
docker images

# イメージを削除
docker rmi イメージ名

# 未使用のイメージを削除
docker image prune
```

## ボリウムの使用

ボリウムは，コンテナの外にデータを存储するために使用されます。

```bash
# ボリウムを作成
docker volume create マイボリウム

# ボリウムをマウント
docker run -v マイボリウム:/データ nginx

# すべてのボリウムを一覧表示
docker volume ls

# ボリウムを削除
docker volume rm マイボリウム
```

## ネットワーク

Docker は，コンテナ間通信のためのネットワーク機能を提供します。

```bash
# ネットワークを作成
docker network create マイネットワーク

# コンテナをネットワークに接続
docker run --network マイネットワーク nginx

# すべてのネットワークを一覧表示
docker network ls
```

## Docker Compose

Docker Compose を使用すると，マルチコンテナアプリケーションを定義して実行できます。

### docker-compose.yml の例

```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: パスワード
```

### コマンド

```bash
# すべてのコンテナを起動
docker-compose up

# バックグラウンドで起動
docker-compose up -d

# すべてのコンテナを停止
docker-compose down
```

## ヒントとベストプラクティス

### セキュリティ

- 可能な限り公式イメージを使用
- コンテナを root として実行しない
- イメージを最新の状態に保つ
- イメージの脆弱性をスキャン

### パフォーマンス

- .dockerignore ファイルを使用
- マルチステージビルドを使用
- イメージのレイヤー数を最小限に
- 特定のイメージタグを使用

### 保守性

- 意味のあるコンテナ名を使用
- Dockerfile をドキュメント化
- 永続データにはボリウムを使用
- コンテナアクティビティを記録
