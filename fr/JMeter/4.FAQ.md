# FAQ JMeter

## Questions générales

### Qu'est-ce qu'Apache JMeter ?
Apache JMeter est un logiciel open source conçu pour les tests de charge et la mesure de performance. C'est une application 100% pure Java qui peut tester le comportement fonctionnel et mesurer les performances.

### Quels protocoles JMeter supporte-t-il ?
JMeter supporte plusieurs protocoles :
- HTTP/HTTPS
- Services Web SOAP/REST
- FTP
- JDBC (Base de données)
- LDAP
- JMS (Middleware orienté messages)
- SMTP/POP3/IMAP (Mail)
- TCP
- Objets Java

### JMeter est-il un navigateur ?
Non, JMeter n'est pas un navigateur. Il fonctionne au niveau du protocole et n'exécute pas le JavaScript ou ne rend pas les pages HTML comme un vrai navigateur.

## Questions d'installation

### Quels sont les exigences système ?
- Java 8 ou supérieur
- Au moins 4 Go de RAM recommandés
- OS compatible : Windows, Linux, macOS

### Comment installer JMeter ?
1. Télécharger depuis https://jmeter.apache.org/
2. Extraire l'archive
3. Exécuter jmeter.sh (Linux/Mac) ou jmeter.bat (Windows)

### Pourquoi JMeter ne démarre pas ?
- Vérifier que Java est installé : `java -version`
- Vérifier que JAVA_HOME est défini
- Vérifier les conflits de ports

### Comment augmenter la mémoire pour JMeter ?
Modifier le script de démarrage de jmeter et modifier la variable HEAP :
```bash
HEAP="-Xms1g -Xmx4g"
```

## Questions d'utilisation

### Comment créer un plan de test basique ?
1. Ajouter un Groupe de Threads
2. Ajouter un Sampler de Requête HTTP
3. Ajouter des Listeners (Arbre de Résultats, Rapport Résumé)
4. Exécuter le test

### Qu'est-ce qu'un Groupe de Threads ?
Groupe de Threads définit un groupe d'utilisateurs virtuels qui exécuteront le même test. Il contrôle :
- Nombre de threads (utilisateurs)
- Période de rampe
- Nombre de boucles

### Comment extraire des données des réponses ?
Utiliser des extracteurs :
- Extracteur JSON pour les réponses JSON
- Extracteur XPath pour XML/HTML
- Extracteur d'Expression Régulière pour tout texte

### Comment paramétrer les tests ?
Utiliser des variables et fonctions :
- `${variable}` pour les références de variables
- `${__Random(1,100)}` pour les- `${__time valeurs aléatoires
()}`

### JMeter peut-il gérer l'authentification ?
Oui, JMeter supporte :
- Authentification Basique
- Authentification NTLM
- Gestionnaire de Cookies
- Gestionnaire d'Autorisation

## Questions de performance

### Combien d'utilisateurs JMeter peut-il simuler ?
Le nombre dépend de votre hardware. Avec une configuration appropriée :
- Mode GUI : 100-300 utilisateurs
- Mode CLI : 1000+ utilisateurs
- Tests distribués : 10,000+ utilisateurs

### Que sont les tests distribués ?
Les tests distribués utilisent plusieurs machines pour générer plus de charge. Un maître coordonne plusieurs machines esclaves.

### Comment exécuter JMeter en mode headless ?
```bash
jmeter -n -t testplan.jmx -l results.jtl
```

## Dépannage

### OutOfMemoryError pendant le test
- Augmenter la taille du heap
- Réduire le nombre de threads
- Utiliser le mode CLI au lieu du GUI
- Désactiver les listeners inutiles

### Résultats non affichés
- Vérifier si le test est en cours d'exécution
- Vérifier que le listener est attaché à l'élément correct
- Vérifier les fichiers de log pour les erreurs

### Temps de réponse élevés
- Vérifier votre réseau
- Vérifier les performances du serveur
- Réduire la charge si le serveur est submergé

### Comment déboguer mon plan de test ?
1. Utiliser le listener Arbre de Résultats
2. Activer le logging de débogage
3. Commencer avec un seul thread
4. Vérifier les assertions

## Bonnes pratiques

1. Toujours utiliser le mode CLI pour les tests de charge
2. Commencer avec une charge faible et augmenter progressivement
3. Utiliser des assertions pour valider les réponses
4. Nettoyer après les tests (tearDown)
5. Garder les plans de test sous contrôle de version
6. Surveiller les ressources système
