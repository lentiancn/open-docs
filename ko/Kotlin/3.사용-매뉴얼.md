# Kotlin 사용자 매뉴얼

## 변수와 기본 타입

Kotlin은 변수 선언에 var과 val 두 가지 키워드를 사용합니다. var은 변경 가능한 변수를 선언하며 Java의 일반 변수와 유사합니다. val은 변경 불가능한 변수를 선언하며 Java의 final 변수와 유사합니다. Kotlin에서 val을 선호하는 것은 코드를 더 안전하고 이해하기 쉽게 하는 모범 사례입니다.

변수를 선언할 때 타입을 명시적으로 지정하거나 컴파일러가 타입을 추론하도록 할 수 있습니다:

val name: String = "Kotlin"
val age = 10 // 컴파일러가 Int 타입을 추론
var count = 0 // 변경 가능한 변수

Kotlin의 기본 타입에는 정수(Int, Long, Short, Byte), 부동 소수점(Double, Float), 문자(Char), 부울(Boolean), 문자열(String)이 포함됩니다. Java와 달리 Kotlin의 모든 기본 타입은 해당 멤버 메서드를 가진 객체입니다.

Kotlin의 정수는 기본적으로 Int 타입입니다. 더 큰 숫자를 나타내려면 Long 접미사를 사용하세요:

val longNumber = 100L // Long 타입

Kotlin의 문자열은 강력하며 문자열 템플릿을 지원하여 문자열에 변수나 표현식을 직접 포함할 수 있습니다:

val name = "World"
println("Hello, $name!") // 출력: Hello, World!
println("Length: ${name.length}") // 출력: Length: 5

## 컬렉션

Kotlin 표준 라이브러리는 List, Set, Map을 포함한 풍부한 컬렉션 타입을 제공합니다. 각 컬렉션 타입에는 불변 버전과 가변 버전이 있습니다.

불변 컬렉션은 생성 후 추가, 삭제 또는 수정할 수 없습니다:

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

가변 컬렉션은 요소 수정을 허용합니다:

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Kotlin 컬렉션은 map, filter, reduce 등 풍부한 함수형 연산을 제공합니다:

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## 흐름 제어

Kotlin의 흐름 제어에는 조건식, 루프, 점프 표현식이 포함됩니다.

Kotlin의 if 표현식은 값을 반환하며 변수에 직접 할당할 수 있습니다:

val max = if (a > b) a else b

when 표현식은 다양한 조건과 일치시킬 수 있는 Kotlin의 강력한 다중 분기 매칭 도구입니다:

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Kotlin의 루프에는 for 루프와 while 루프가 있습니다. for 루프는 범위, 배열, 컬렉션을 반복할 수 있습니다:

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## 함수

Kotlin은 함수 선언에 fun 키워드를 사용합니다. 함수는 매개변수 타입과 반환 타입을 지정할 수 있습니다:

fun greet(name: String): String {
    return "Hello, $name!"
}

함수 본문에 표현식이 하나만 있으면 단일 표현식 구문을 사용할 수 있습니다:

fun greet(name: String) = "Hello, $name!"

Kotlin은 기본 인자와 이름付き인자를 지원하여 함수 오버로드의 필요성을 크게 줄입니다:

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin은 vararg 키워드를 사용하여 가변 인자도 지원합니다:

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## 클래스와 객체

Kotlin 클래스 선언은 Java보다 더 간결합니다. 기본적으로 Kotlin 클래스는 final이며 상속할 수 없습니다. 클래스를 상속 가능하게 하려면 open 키워드를 사용하세요:

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Kotlin의 생성자에는 주 생성자와 보조 생성자가 있습니다. 주 생성자는 클래스 이름 바로 뒤에 선언합니다:

class Person(val name: String, val age: Int)

생성자에서 로직을 실행해야 하는 경우 init 블록을 사용할 수 있습니다:

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Kotlin의 데이터 클래스(data class)는 equals, hashCode, toString, copy 메서드를 자동으로 생성하여 데이터를 저장하는 클래스에 이상적입니다:

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Null 안전

Null 안전은 Kotlin의 가장 중요한 기능 중 하나입니다. Kotlin에서 타입 시스템은 nullable 타입과 non-nullable 타입을 구분하여 NullPointerException을 근본적으로 줄입니다.

기본적으로 타입은 nullable이 아닙니다:

val name: String = "Kotlin" // null이 될 수 없음
val name: String = null // 컴파일 오류

변수가 null이 될 수 있도록 하려면 nullable 타입(?)을 사용하세요:

val name: String? = null // null이 될 수 있음

nullable 타입의 속성이나 메서드에 접근할 때는 안전한 호출 연산자(?.) 또는 null 주장(!!)을 사용하세요:

val length = name?.length // name이 null이면 null 반환
val length = name!!.length // name이 null이면 예외 발생

Elvis 연산자(?:)는 기본값을 처리하는 간결한 방법을 제공합니다:

val length = name?.length ?: 0 // name이 null이면 0 반환

## 확장 함수와 확장 속성

Kotlin을 사용하면 기존 클래스를 상속하거나 수정하지 않고도 새 함수와 속성을 추가할 수 있습니다. 이를 확장 함수와 확장 속성이라고 합니다.

확장 함수는 확장할 클래스 이름을 함수 이름 앞에 붙여서 선언합니다:

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

확장 속성을 사용하여 기존 클래스에 새 속성을 추가할 수도 있습니다:

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## 고차 함수와 람다

고차 함수는 함수를 매개변수로 받거나 함수를 반환하는 함수입니다. 람다 표현식은 이 기능을 구현하는 주요 방법입니다.

람다 표현식 구문은 중괄호를 사용합니다:

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

람다가 함수의 마지막 매개변수인 경우 괄호 밖으로 이동할 수 있습니다:

list.filter { it > 0 }

람다 표현식에서 it은 단일 매개변수의 암시적 이름입니다. 매개변수가 하나만 있으면 매개변수 선언을 생략할 수 있습니다.

## 코루틴

코루틴은 Kotlin 비동기 프로그래밍의 핵심 기능입니다. 스레드에 비해 코루틴은 훨씬 가벼우며 단일 스레드에서 여러 코루틴을 실행할 수 있습니다.

launch는 백그라운드에서 새 코루틴을 시작하는 일반적인 방법입니다:

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

suspend 함수는 코루틴 내에서만 호출할 수 있습니다:

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        // 시간이 걸리는 작업 수행
    }
}

Kotlin 코루틴은 코루틴이 실행되는 스레드를 제어하기 위한 여러 디스패처를 제공합니다: Dispatchers.Main은 UI 작업용, Dispatchers.IO는 I/O 작업용, Dispatchers.Default는 CPU 집약적 작업용입니다.

## sealed 클래스와 열거형

sealed 클래스는 상속할 수 있는 하위 클래스 수를 제한하는 제한된 클래스 계층 구조입니다. 모든 하위 클래스는 동일한 파일에서 선언해야 합니다. 이는 유한 상태나 결과 타입을 나타낼 때 매우 유용합니다:

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

when 표현식은 sealed 클래스의 기능을 최대한 활용할 수 있으며, 컴파일러는 가능한 모든 케이스가 처리되었는지 확인합니다:

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

열거형은 유한한 상수 값 집합을 나타냅니다:

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

열거형은 속성과 메서드를 가질 수 있습니다:

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## 인터페이스와 추상 클래스

Kotlin의 인터페이스는 기본 구현을 포함할 수 있어 더 유연합니다:

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
)

클래스는 여러 인터페이스를 구현할 수 있습니다:

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## 객체 표현식과 객체 선언

객체 표현식은 Java의 익명 내부 클래스와 유사하게 익명 클래스의 인스턴스를 생성합니다:

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // 클릭 이벤트 처리
    }
})

객체 선언은 싱글톤을 생성하는 데 사용됩니다:

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## 컴패니언 객체

각 클래스는 Java의 정적 멤버와 유사한 컴패니언 객체를 가질 수 있습니다:

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

컴패니언 객체의 멤버는 클래스 이름을 통해 직접 접근할 수 있습니다:

MyClass.TAG
MyClass.create()

## 타입 확인과 변환

is 연산자는 Java의 instanceof와 유사하게 타입 확인에 사용됩니다:

if (obj is String) {
    // 이 분기에서 obj는 자동으로 String으로 변환됨
    println(obj.length)
)

스마트 캐스트는 확인된 타입을 자동으로 변환합니다. 컴파일러가 특정 분기에서 변수의 타입을 유추할 수 있으면 자동으로 변환을 수행합니다.

스마트 캐스트가 적용되지 않으면 명시적 변환을 사용할 수 있습니다:

val str = obj as String

안전한 타입 변환은 as?를 사용하며 변환 실패 시 null을 반환합니다:

val str = obj as? String // 변환 실패 시 null 반환
