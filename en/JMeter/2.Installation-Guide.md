# JMeter Installation Guide

This guide provides detailed instructions for installing and configuring Apache JMeter on different operating systems.

## Requirements

- **Java Runtime Environment**: JMeter requires Java 8 or higher
- **Operating System**: Supports any Java compatible OS including Linux, Windows, and Mac OSX
- **Memory**: At least 4GB available memory recommended (more may be needed depending on test scale)
- **Disk Space**: At least 1GB available space recommended

## Downloading JMeter

1. Visit the official JMeter download page: https://jmeter.apache.org/download_jmeter.cgi
2. Download the latest binary distribution (Binaries)
3. It is recommended to also download PGP or SHA-512 signature files to verify download integrity

## Windows Installation

### Extraction

1. Locate the downloaded Apache JMeter archive (apache-jmeter-x.x.zip)
2. Choose an appropriate installation directory (avoid paths with spaces and special characters)
3. Extract the archive to the selected directory
4. JMeter is now installed

### Verification

1. Navigate to the bin folder in the JMeter installation directory
2. Double-click jmeter.bat (Windows batch file)
3. If successful, the JMeter GUI will launch

## Linux and Mac OSX Installation

### Extraction

1. Locate the downloaded Apache JMeter archive (apache-jmeter-x.x.tar.gz)
2. Extract using the following command:
   ```bash
   tar -xf apache-jmeter-x.x.tar.gz
   ```
3. JMeter is now installed

### Environment Variables (Optional)

1. Open your terminal configuration file (~/.bashrc or ~/.profile)
2. Add the following environment variables:
   ```bash
   export JMETER_HOME=/path/to/apache-jmeter-x.x
   export PATH=$PATH:$JMETER_HOME/bin:$JMETER_HOME/lib
   ```
3. Save and refresh the configuration:
   ```bash
   source ~/.bashrc
   ```

### Verification

1. Run the following in terminal:
   ```bash
   jmeter --version
   ```
2. If version information is displayed, installation was successful

## Starting JMeter

### GUI Mode

- **Windows**: Double-click jmeter.bat in the bin directory
- **Linux/Mac**: Run jmeter command in terminal

### Command-line Mode

For load testing, command-line mode is recommended to reduce resource usage:

```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/output
```

Parameters:
- -n: Run in non-GUI mode
- -t: Specify test plan file
- -l: Specify results file
- -e: Generate report after test ends
- -o: Specify report output directory

## Directory Structure

- **bin**: Contains executable files and startup scripts
- **lib**: Contains JMeter dependency libraries
- **docs**: Contains official JMeter documentation
- **extras**: Contains additional auxiliary tools
- **printable_docs**: Contains printable user manual

## Dependency Management

JMeter's lib directory contains core dependencies. To add extra plugins or libraries:

1. Copy JAR files to the lib directory
2. Some plugins need to be placed in the lib/ext directory
3. Restart JMeter for new plugins to take effect

## Common Installation Issues

- **Java Not Found**: Ensure Java is properly installed and JAVA_HOME environment variable is set
- **Insufficient Memory**: Modify Heap size parameters in bin/jmeter.sh or jmeter.bat
- **Startup Failure**: Check if correct Java version (Java 8 or higher) is installed
