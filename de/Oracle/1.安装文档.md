# Oracle Datenbank Installationsanleitung

Oracle Datenbank ist das führende relationale Datenbankmanagementsystem der Branche, entwickelt von Oracle. Diese Anleitung deckt die Installationsmethoden in verschiedenen Umgebungen ab.

## Versionsinformationen

| Version | Status | Beschreibung |
|------|------|------|
| Oracle 21c | ✅ Aktuell | Neueste Version |
| Oracle 19c | ✅ LTS | Langzeitunterstützungsversion |
| Oracle 18c | ❌ Veraltet | Support eingestellt |

> **Wichtig**: Diese Anleitung richtet sich hauptsächlich an **Oracle 19c/21c**.

## Systemanforderungen

### Hardwareanforderungen

| Anforderung | Mindestkonfiguration | Empfohlene Konfiguration |
|------|----------|----------|
| CPU | 2 Kerne | 4+ Kerne |
| Speicher | 2 GB | 8+ GB |
| Festplatte | 40 GB | 100+ GB |

### Unterstützte Betriebssysteme

- **Linux**: Oracle Linux, RHEL, CentOS, SLES, Ubuntu
- **Windows**: Windows Server 2016+

## Installationsmethoden

### Methode 1: Linux-Installation

#### 1. System vorbereiten

```bash
# Benutzergruppen erstellen
groupadd oinstall
groupadd dba
groupadd oper
groupadd backupdba
groupadd dgdba
groupadd kmdba
groupadd racdba
groupadd asmdba
groupadd asmoper
groupadd asmadmin

# Oracle-Benutzer erstellen
useradd -g oinstall -G dba,oper,backupdba,dgdba,kmdba,racdba,asmdba,asmoper,asmadmin oracle

# Passwort festlegen
passwd oracle

# Verzeichnisse erstellen
mkdir -p /u01/app/oracle/product/19.3.0/dbhome_1
mkdir -p /u01/app/oraInventory
mkdir -p /u01/app/oracle

# Berechtigungen festlegen
chown -R oracle:oinstall /u01
chmod -R 775 /u01
```

#### 2. Kernel-Parameter konfigurieren

```bash
# /etc/sysctl.conf
cat >> /etc/sysctl.conf << EOF
kernel.shmmax = 68719476736
kernel.shmmni = 4096
kernel.shmall = 2097152
kernel.sem = 250 32000 100 128
fs.file-max = 6815744
net.ipv4.ip_local_port_range = 9000 65500
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 1048576
fs.aio-max-nr = 1048576
EOF

# Anwenden
sysctl -p

# /etc/security/limits.conf
cat >> /etc/security/limits.conf << EOF
oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536
oracle soft stack 10240
oracle hard stack 32768
EOF
```

#### 3. Abhängigkeiten installieren

```bash
# Oracle Linux
yum install -y libaio libaio-devel libXi libXtst libXext

# Ubuntu
apt-get install -y libaio1 libaio-dev libxi6 libxtst6 libxext6
```

#### 4. Installer ausführen

```bash
# Installationsdateien entpacken
unzip -q LINUX.X64_193000_db_home.zip -d /u01/app/oracle/product/19.3.0/dbhome_1/

# Als Oracle-Benutzer ausführen
cd /u01/app/oracle/product/19.3.0/dbhome_1
./runInstaller
```

### Methode 2: Windows-Installation

1. Oracle-Installer herunterladen
2. setup.exe ausführen
3. Installationsassistent folgen:
   - "Erstellen und Konfigurieren einer Single-Instance-Datenbank" auswählen
   - "Desktop-Klasse" oder "Server-Klasse" auswählen
   - sys- und system-Passwörter festlegen
   - Installation abschließen

### Methode 3: Docker verwenden

```bash
# Image herunterladen
docker pull container-registry.oracle.com/database/express:21.3.0-xe

# Container ausführen
docker run -d \
  --name oracle \
  -p 1521:1521 \
  -e ORACLE_PWD=MyPassword123 \
  -v oracle_data:/opt/oracle/oradata \
  container-registry.oracle.com/database/express:21.3.0-xe
```

## Konfiguration nach der Installation

### Umgebungsvariablen

```bash
# ~/.bash_profile
export ORACLE_HOME=/u01/app/oracle/product/19.3.0/dbhome_1
export ORACLE_SID=orcl
export PATH=$ORACLE_HOME/bin:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:$LD_LIBRARY_PATH
```

### Datenbank starten

```bash
# Listener starten
lsnrctl start

# Datenbank starten
sqlplus / as sysdba
STARTUP;
```

## Häufige Probleme

### Nicht genügend Speicher?

```bash
# Speicher überprüfen
grep MemTotal /proc/meminfo

# Swap anpassen
```

### Installation fehlgeschlagen?

Logs überprüfen:
```bash
cat /u01/app/oraInventory/logs/installActions*.log
```
