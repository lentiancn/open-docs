# Claude Code Manuel d'Utilisation

Ce manuel vous guide pour commencer avec Claude Code et maîtriser les opérations quotidiennes.

## Préparation

Assurez-vous d'avoir:
- Un terminal ouvert
- Un projet de code
- Un abonnement Claude (Pro, Max, Teams ou Enterprise)

## Démarrer une Session

Ouvrez un terminal dans votre projet et lancez Claude Code:

```bash
cd /path/to/your/project
claude
```

## Opérations de Base

### Comprendre la Base de Code

Demandez à Claude d'analyser votre projet:
```
Que fait ce projet?
```

```
Quelles technologies utilise ce projet?
```

### Première Modification

Essayez une tâche simple:
```
Ajoutez une fonction hello world au fichier principal
```

Claude Code va:
1. Trouver le fichier approprié
2. Montrer les modifications proposées
3. Demander votre approbation
4. Effectuer l'édition

### Opérations Git

```
Quels fichiers ai-je modifiés?
```

```
Committez mes changements avec un message descriptif
```

```
Créez une nouvelle branche appelée feature/quickstart
```

### Corrections et Fonctionnalités

Décrivez ce que vous voulez en langage naturel:
```
Ajoutez une validation au formulaire d'inscription
```

## Commandes Essentielles

| Commande | Fonction |
|----------|----------|
| `claude` | Mode interactif |
| `claude "tâche"` | Exécuter une tâche unique |
| `claude -p "requête"` | Requête unique, puis quitter |
| `claude commit` | Créer un commit Git |
| `/help` | Commandes disponibles |

## Conseils Avancés

### CLAUDE.md
Créez un fichier CLAUDE.md à la racine du projet pour le contexte persistant.

### Commandes Personnalisées
Créez des commandes comme `/review-pr` ou `/deploy-staging`.

### Collaboration Multi-Agents
Pour les tâches complexes, lancez plusieurs agents en parallèle.

## Meilleures Pratiques

### Soyez Spécifique
❌ Ne dites pas: "corrigez le bug"
✅ Dites: "corrigez le bug de login où les utilisateurs voient un écran vide"

### Laissez Claude Explorer
Avant de modifier, laissez Claude comprendre votre code:
```
Analysez le schéma de base de données
```
