# Yarn 설치 가이드

## Yarn v4+ (모던 버전) 설치

### 권장 방법: Corepack 사용

Corepack은 프로젝트별로 패키지 매니저 버전을 구성할 수 있게 해주는 중개 도구입니다.

```bash
# 1. Corepack 설치
npm install -g corepack

# 2. 새 프로젝트 초기화
yarn init -2
```

### Yarn 업데이트

언제든지 Yarn을 최신 버전으로 업데이트하려면 다음 명령어를 실행하세요:

```bash
yarn set version stable
yarn install
```

Yarn은 프로젝트가 최신 안정 바이너리를 사용하도록 구성합니다.

### 캐나리 빌드 사용

Yarn은 자주 릴리스 후보 빌드도 제공합니다. 아직 안정 채널에 출시되지 않은 기능이 필요한 경우 다음을 사용하세요:

```bash
yarn set version canary
```

이러한 빌드는 매우 안정적이며, 일반 채널과의 유일한 차이점은 새로운 중단 변경사항을 구현할 때 주요 버전 간에 더 점진적인 마이그레이션을 수행한다는 것입니다.

### master에서 최신 빌드 설치

아직 릴리스 후보에서도 출시되지 않았거나 병합되지 않은 매우 최신의 Yarn 버전을 테스트하고 싶을 수도 있습니다. 다음 명령어는 저장소에서 직접 Yarn을 클론, 빌드 및 설치합니다:

```bash
yarn set version from sources
```

특정 PR을 테스트하는 데 사용할 수 있는 `--branch` 플래그를 허용합니다:

```bash
yarn set version from sources --branch 1211
```

> 경고: 안정 및 캐나리 채널과 달리, `yarn set version from sources` 명령어는 Corepack을 활용할 수 없으며 Yarn 바이너리를 `.yarn/releases` 폴더 내에 저장하고 프로젝트의 `.yarnrc.yml` 파일에서 참조해야 합니다.

## Yarn v1 (Classic) 설치

### npm을 통한 설치 (권장)

시스템에 [Node.js](https://nodejs.org/)를 설치할 때 함께 제공되는 [npm 패키지 매니저](http://npmjs.org/)를 통해 Yarn을 설치하는 것이 좋습니다.

npm이 설치되어 있으면 다음 명령어를 실행하여 Yarn을 설치 및 업그레이드할 수 있습니다:

```bash
npm install --global yarn
```

### 설치 확인

다음 명령어를 실행하여 Yarn이 설치되었는지 확인하세요:

```bash
yarn --version
```

## 버전 선택 권장사항

- **신규 프로젝트**: 모던 기능과 더 나은 성능을 위해 Yarn v4+ 사용 권장
- **기존 프로젝트**: 이미 Yarn v1을 사용 중이라면 계속 사용하거나 모던 버전으로 마이그레이션 고려
- **기업 환경**: Yarn v1은 더 안정적인 장기 지원을 제공하며, v4+는 더 혁신적인 기능을 제공
