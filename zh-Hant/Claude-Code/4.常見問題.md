# Claude Code 常見問題

本頁面列出 Claude Code 安裝和使用過程中最常見的問題及解決方案。

## 安裝問題

### 命令找不到

安裝完成後遇到 `command not found: claude` 或 `'claude' is not recognized` 錯誤，說明安裝目錄不在系統 PATH 中。

**解決方案：** 檢查並配置 PATH 環境變數。

macOS/Linux 添加到 .zshrc 或 .bashrc：
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Windows PowerShell：
```powershell
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
```

然後重啟終端並驗證：
```bash
claude --version
```

### 安裝腳本返回 HTML

執行安裝命令時看到以下錯誤：
- `bash: line 1: syntax error near unexpected token '<'`
- `bash: line 1: '<!DOCTYPE html>'`
- PowerShell 上顯示：`Invoke-Expression: Missing argument in parameter list`

這表示安裝 URL 返回了 HTML 頁面而不是安裝腳本。

**解決方案：**

1. 使用替代安裝方法：
   - macOS/Linux：使用 Homebrew
     ```bash
     brew install --cask claude-code
     ```
   - Windows：使用 WinGet
     ```powershell
     winget install Anthropic.ClaudeCode
     ```

2. 如果 HTML 頁面顯示 "App unavailable in region"，表示 Claude Code 在你所在的地區不可用。請參閱 [支援的地區列表](https://www.anthropic.com/supported-countries)。

3. 等待幾分鐘後重試，問題通常是暫時的。

### curl 下載失敗

錯誤資訊：`curl: (56) Failure writing output to destination`

這表示連接在腳本下載完成前中斷了。

**解決方案：**

1. 檢查網路穩定性：
   ```bash
   curl -fsSL https://storage.googleapis.com -o /dev/null
   ```
   如果命令成功完成，說明連接正常，重試安裝命令即可。

2. 使用替代安裝方法（見上文）。

### TLS/SSL 連接錯誤

錯誤資訊：`curl: (35) TLS connect error`、`schannel: next InitializeSecurityContext failed` 或 `Could not establish trust relationship for the SSL/TLS secure channel`

**解決方案：**

1. 更新系統 CA 憑證：
   - Ubuntu/Debian：
     ```bash
     sudo apt-get update && sudo apt-get install ca-certificates
     ```
   - macOS（透過 Homebrew）：
     ```bash
     brew install ca-certificates
     ```

2. 在 Windows PowerShell 中啟用 TLS 1.2：
   ```powershell
   [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
   irm https://claude.ai/install.ps1 | iex
   ```

3. 檢查代理或防火牆干擾，配置 `NODE_EXTRA_CA_CERTS` 環境變數。

### 無法取得版本資訊

錯誤資訊：`Failed to fetch version from storage.googleapis.com`

這表示 `storage.googleapis.com` 在你的網路被阻止。

**解決方案：**

1. 測試直接連接：
   ```bash
   curl -sI https://storage.googleapis.com
   ```

2. 如果在代理後面，設定 `HTTPS_PROXY` 環境變數。

### Windows 需要 Git Bash

錯誤資訊：`Claude Code on Windows requires git-bash`

**解決方案：** 安裝 [Git for Windows](https://git-scm.com/downloads/win)，然後重新執行安裝命令。

如果已安裝 Git 但仍然報錯，在設定檔案中配置 Git Bash 路徑：
```json
{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}
```

### Linux 上非法指令

錯誤資訊：`Illegal instruction`

這通常是架構不匹配導致的。請確認你的 CPU 架構與安裝的二進制相容。

### macOS 上二進制無法載入

錯誤資訊：`dyld: cannot load` 或 `Abort trap`

這表示二進制與你的系統不相容。請確認你下載了正確版本的 Claude Code（Intel 或 Apple Silicon）。

### 低記憶體伺服器安裝被殺

錯誤資訊：在 Linux 伺服器上安裝時顯示 `Killed`

**解決方案：** 為低記憶體伺服器添加 swap 空間。

## 認證問題

### OAuth 錯誤或 403 Forbidden

**解決方案：**

1. 檢查你的帳戶是否有 Claude Code 存取權限（Pro、Max、Teams、Enterprise 或 Console）
2. 清除快取的憑證並重新登入：
   ```bash
   rm -rf ~/.claude
   claude
   ```
3. 檢查你的地區是否在 [支援列表](https://www.anthropic.com/supported-countries) 中

### 地區不可用

錯誤資訊：`App unavailable in region`

Claude Code 在你的國家/地區不可用。請參閱 [Anthropic 支援的地區](https://www.anthropic.com/supported-countries)。

## 網路配置

### 檢查網路連接

安裝程式從 `storage.googleapis.com` 下載檔案。驗證你能否訪問：

```bash
curl -sI https://storage.googleapis.com
```

如果失敗，可能是以下原因：
- 公司防火牆或代理阻止了 Google Cloud Storage
- 地區網路限制：嘗試使用 VPN 或其他網路
- TLS/SSL 問題：更新系統 CA 憑證，或檢查是否配置了 `HTTPS_PROXY`

### 代理配置

如果在公司代理後面，在安裝前設定代理環境變數：

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

## 檢查衝突安裝

多個 Claude Code 安裝可能導致版本不匹配或意外行為。

檢查已安裝的內容：

**macOS/Linux：**
```bash
which -a claude
ls -la ~/.local/bin/claude
npm -g ls @anthropic-ai/claude-code 2>/dev/null
```

**Windows：**
```powershell
where.exe claude
Test-Path "$env:LOCALAPPDATA\Claude Code\claude.exe"
```

如果發現多個安裝，只保留一個（推薦原生安裝 `~/.local/bin/claude`）。

移除額外的安裝：
- npm 全域安裝：`npm uninstall -g @anthropic-ai/claude-code`
- Homebrew 安裝：`brew uninstall --cask claude-code`

## 檢查目錄權限

安裝程式需要寫入 `~/.local/bin/` 和 `~/.claude/` 的權限。檢查這些目錄是否可寫：

```bash
test -w ~/.local/bin && echo "writable" || echo "not writable"
test -w ~/.claude && echo "writable" || echo "not writable"
```

如果目錄不可寫：

```bash
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```

## 驗證二進制

如果 `claude` 已安裝但啟動時崩潰或掛起：

確認二進制存在且可執行：
```bash
ls -la $(which claude)
```

在 Linux 上檢查缺失的共用庫：
```bash
ldd $(which claude) | grep "not found"
```

執行基本檢查：
```bash
claude --version
```

## 診斷工具

使用 `claude doctor` 取得更詳細的安裝和配置資訊：

```bash
claude doctor
```
