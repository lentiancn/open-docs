# Docker 자주 묻는 질문

이 페이지는 Docker 설치 및 사용에 대한 일반적인 문제와 솔루션을 나열합니다.

## 설치 문제

### Docker Desktop이 시작되지 않음

**증상**: Docker Desktop이 시작되지 않거나 시작 후 즉시 충돌함

**솔루션**:

1. 시스템이 하드웨어 요구 사항을 충족하는지 확인:
   - Windows: Hyper-V 활성화 또는 WSL 2 설치
   - Mac: 오래된 Mac 모델을 사용하지 않는지 확인

2. Docker Desktop 초기화:
   - Windows: Docker 아이콘 우클릭 → 문제 해결 → 공장 설정으로 재설정
   - Mac: Docker → 문제 해결 → 공장 설정으로 재설정

3. 시스템 로그 확인:
   - Windows: 이벤트 뷰어 → Windows 로그 → 애플리케이션
   - Mac: 콘솔을 열고 Docker 오류 검색

### Docker 명령에 대한 권한 없음

**증상**: "permission denied while trying to connect to the Docker daemon socket"

**솔루션**:

1. 사용자를 Docker 그룹에 추가:
   ```bash
   sudo usermod -aG docker $USER
   ```

2. 로그아웃하고 다시 로그인

3. 또는 Docker 명령 앞에 sudo 사용

### Windows에서 WSL 2 설치 오류

**증상**: "WSL 2 installation is incomplete"

**솔루션**:

1. Linux용 Windows 하위 시스템 활성화:
   ```bash
   wsl --install
   ```

2. 컴퓨터 재시작

3. WSL을 최신 버전으로 업데이트:
   ```bash
   wsl --update
   ```

## 사용 문제

### 컨테이너가 시작되지 않음

**증상**: 컨테이너가 "Exited" 또는 "Created" 상태로 유지됨

**솔루션**:

1. 컨테이너 로그 확인:
   ```bash
   docker logs 컨테이너_이름
   ```

2. 필요한 모든 포트가 사용 가능한지 확인

3. 이미지 무결성 확인

### 포트가 이미 사용 중임

**증상**: "port is already allocated"

**솔루션**:

1. 사용 중인 포트 찾기:
   ```bash
   netstat -ano | findstr :8080
   ```

2. 다른 서비스 중지 또는 다른 포트 사용

3. 포트 매핑 변경:
   ```bash
   docker run -p 8081:80 nginx
```

### 디스크 공간 문제

**증상**: "no space left on device"

**솔루션**:

1. 미사용 컨테이너 제거:
   ```bash
   docker container prune
   ```

2. 미사용 이미지 제거:
   ```bash
   docker image prune -a
   ```

3. 미사용 볼륨 제거:
   ```bash
   docker volume prune
   ```

### 네트워킹 문제

**증상**: 컨테이너가 외부에 연결할 수 없음

**솔루션**:

1. DNS 설정 확인:
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

2. Docker 네트워크 초기화:
   ```bash
   docker network prune
   ```

3. 방화벽 설정 확인

## 성능 문제

### 컨테이너 성능이 느림

**솔루션**:

1. 리소스 제한 확인:
   ```bash
   docker stats
   ```

2. 더 많은 RAM 할당 (Docker Desktop → 설정 → 리소스)

3. Docker에 SSD 대신 HDD 사용

### 디스크 사용량 높음

**솔루션**:

1. 정기적으로 빌드 캐시 정리:
   ```bash
   docker builder prune
   ```

2. 이전 버전 이미지 제거

3. 로그 크기 제한

## 보안 문제

### 컨테이너를 root로 실행

**권장 사항**:

비特权 사용자 사용:

```dockerfile
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser
USER appuser
```

### 이미지의 취약점

**솔루션**:

1. 공식 및 최신 이미지 사용

2. 정기적으로 이미지 스캔:
   ```bash
   docker scan 이미지_이름
   ```

3. distroless 이미지 사용 고려
