# Manuel d'utilisation d'OpenClaw

Ce manuel fournit un guide détaillé des fonctionnalités et de l'utilisation d'OpenClaw, vous aidant à tirer le meilleur parti de cette puissante passerelle d'assistant IA.

---

## Concepts de base

### Gateway

Le Gateway est le composant central d'OpenClaw. Il s'exécute sur votre machine locale comme plan de contrôle pour toutes les connexions de canal, agents IA et clients.

### Canaux

Les canaux désignent les plateformes de chat que vous utilisez pour interagir avec l'assistant IA, y compris WhatsApp, Telegram, Discord, iMessage et plus.

### Agents

Les agents sont des instances de l'assistant IA. Chaque agent a son propre espace de travail, sessions et configuration.

### Sessions

Les sessions gèrent le contexte de conversation. OpenClaw prend en charge plusieurs modes de session :
- **main** : Session partagée
- **per-peer** : Session séparée pour chaque contact
- **per-channel-peer** : Session séparée pour chaque combinaison canal-contact
- **per-account-channel-peer** : La plus granulaire

## Utilisation quotidienne

### Démarrer et arrêter le Gateway

**Démarrer :**

```bash
# Exécuter au premier plan (pour le débogage)
openclaw gateway --port 18789

# Exécuter en arrière-plan
openclaw gateway start
```

**Arrêter :**

```bash
openclaw gateway stop
```

**Vérifier le statut :**

```bash
openclaw gateway status
```

### Utiliser l'interface de contrôle

Visitez http://127.0.0.1:18789/ dans votre navigateur.

### Commandes CLI

**Envoyer un message :**

```bash
openclaw message send --target +15555550123 --message "Bonjour, OpenClaw"
```

**Discuter avec l'IA :**

```bash
openclaw agent --message "Aide-moi à écrire un programme Hello World" --thinking high

## Gestion des canaux

### Configurer WhatsApp

```bash
openclaw channels login
```

### Configurer Telegram

```json5
{
  channels: {
    telegram: {
      botToken: "VotreTokenBot",
    },
  },
}
```

### Configurer Discord

```json5
{
  channels: {
    discord: {
      token: "VotreTokenBot",
    },
  },
}
```

## Fonctionnalités avancées

### Routage multi-agents

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" },
    ],
  },
}
```

### Tâches Cron

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2,
  },
}
```

### Mode sandbox

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent",
      },
    },
  },
}
```

## Commandes courantes

| Commande | Description |
|----------|-------------|
| `openclaw --version` | Voir la version |
| `openclaw status` | Voir le statut |
| `openclaw gateway start` | Démarrer le Gateway |
| `openclaw gateway stop` | Arrêter le Gateway |
| `openclaw dashboard` | Ouvrir l'interface de contrôle |
| `openclaw channels login` | Se connecter aux canaux |
| `openclaw config get` | Obtenir la configuration |
| `openclaw config set` | Définir la configuration |
| `openclaw doctor` | Diagnostiquer les problèmes |

---

## Commandes de chat

- `/status` - Voir le statut de la session
- `/new` ou `/reset` - Réinitialiser la session
- `/compact` - Compacter le contexte de session
- `/think` - Définir le niveau de réflexion
- `/verbose on|off` - Basculer la sortie détaillée
