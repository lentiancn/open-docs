# JDK 安裝指南

## 在 Windows 上安裝 JDK

### 系統需求

在安裝 JDK 之前，需要確保電腦滿足以下最低需求：處理器方面，建議使用 Intel Core i3 或同等以上的處理器；記憶體方面，建議至少 4GB RAM，推薦 8GB 或以上；磁碟空間方面，需要至少 2GB 的可用空間用於安裝 JDK，額外的空間用於開發專案；作業系統方面，Windows 10 或 Windows 11 的 64 位元版本。

Windows 7 和 8.1 雖然可能可以運行，但 Oracle 官方不再支援這些作業系統。如果在較舊的系統上運行，可能會遇到相容性問題。

### 下載 JDK

首先，訪問 Oracle 官方網站 https://www.oracle.com/java/technologies/downloads/ 下載 JDK。在下載頁面，可以看到不同版本的 JDK：JDK 25 是最新版本，JDK 21 是最新的長期支援版本，JDK 17 也是一個成熟的長期支援版本。根據專案需求選擇合適的版本。

對於 Windows 系統，選擇 x64 Installer（.exe）或 x64 MSI Installer（.msi）格式的安裝套件。.exe 是傳統的安裝精靈，.msi 適合企業環境下的批量部署。如果只需要運行 Java 程式而不需要安裝，可以選擇 x64 Compressed Archive（.zip）格式，解壓後即可使用。

下載時需要注意許可條款。Oracle JDK 在某些版本和用途下是免費的，但在商業使用和再分發方面有限制。如果需要完全免費的使用，可以考慮使用 OpenJDK 或其他基於 OpenJDK 的發行版，如 Eclipse Temurin。

### 安裝步驟

找到下載的 JDK 安裝檔案（通常是 jdk-xx_windows-x64_bin.exe），按兩下執行。在安裝精靈中，按一下「下一步」繼續。首先會看到目標資料夾的選擇介面，預設安裝路徑是 C:\Program Files\Java\jdk-xx。如果需要更改安裝位置，按一下「變更」按鈕選擇其他目錄。建議使用預設路徑以避免後續的設定問題。

接下來是功能選擇介面，預設會勾選所有功能，包括開發工具、原始碼和公用 JRE。建議保持預設選項完整安裝。按一下「下一步」開始安裝。安裝過程可能需要幾分鐘時間，請耐心等待。

安裝完成後，會看到提示資訊。按一下「關閉」完成安裝。此時 JDK 已經安裝到指定的目錄中，但還需要設定環境變數才能在命令列中使用。

### 設定環境變數

設定環境變數的目的是讓系統能夠找到 JDK 的安裝位置，這樣就可以在任何目錄下使用 java、javac 等命令。右鍵點選「這台電腦」或「我的電腦」，選擇「內容」。在左側功能表中點選「進階系統設定」。

在開啟的系統內容視窗中，按一下「環境變數」按鈕。在「系統變數」區域，按一下「新增」。變數名稱輸入 JAVA_HOME，變數值輸入 JDK 的安裝路徑（如 C:\Program Files\Java\jdk-21），按一下確定。

找到系統變數中的 Path 變數，按一下「編輯」。按一下「新增」，輸入 %JAVA_HOME%\bin。確保這一行在最上面，這樣會優先使用新安裝的 JDK。按一下確定儲存所有設定。

開啟一個新的命令列視窗（cmd 或 PowerShell），輸入 java -version 和 javac -version，如果能看到版本資訊，說明設定成功。

### 驗證安裝

開啟命令提示字元（Win+R，輸入 cmd），依序輸入以下命令驗證 JDK 是否正確安裝：

輸入 java -version，應該顯示 Java 執行環境的版本資訊。輸入 javac -version，應該顯示 Java 編譯器的版本資訊。輸入 echo %JAVA_HOME%，應該顯示之前設定的 JAVA_HOME 路徑。

如果任何一個命令出現錯誤，檢查環境變數設定是否正確，或者 JDK 是否正確安裝。

## 在 macOS 上安裝 JDK

### 使用 DMG 安裝套件

macOS 用戶可以通過 DMG 安裝套件快速安裝 JDK。訪問 Oracle 官方下載頁面，下載 macOS 版本的 JDK。對於 Apple Silicon（M1/M2/M3 晶片)，選擇 ARM64 DMG Installer；對於 Intel 晶片的 Mac，選擇 x64 DMG Installer。

下載完成後，開啟 DMG 檔案，會看到一個 pkg 安裝檔案。按兩下執行安裝精靈，按照提示完成安裝。安裝程式會將 JDK 安裝到 /Library/Java/JavaVirtualMachines/ 目錄下。

安裝完成後，開啟終端機，輸入 java -version 驗證安裝。如果系統有多個 JDK 版本，可以使用 /usr/libexec/java_home -V 查看所有已安裝的版本。

### 使用 Homebrew 安裝

如果已經安裝了 Homebrew 包管理器，可以使用它來安裝 JDK，這更加簡單和方便。開啟終端機，執行以下命令更新 Homebrew：brew update。然後安裝 JDK 21：brew install openjdk@21。

安裝完成後，需要將 JDK 新增到 PATH。執行以下命令將 JDK 連結到系統：sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21。

之後在 ~/.zshrc 或 ~/.bash_profile 中新增環境變數：export PATH="/usr/local/opt/openjdk@21/bin:$PATH"。儲存後執行 source ~/.zshrc 使設定生效。

### 設定 JAVA_HOME

在 macOS 上，JAVA_HOME 環境變數的設定不是必須的，因為系統會自動找到已安裝的 JDK。但在某些工具（如 Maven、Gradle）中可能需要手動設定。在終端機中編輯 ~/.zshrc 檔案，新增：export JAVA_HOME=$(/usr/libexec/java_home)。儲存後執行 source ~/.zshrc 使設定生效。

## 在 Linux 上安裝 JDK

### 使用 APT 安裝（Debian/Ubuntu）

在 Debian 和 Ubuntu 系統上，可以使用 APT 包管理器安裝 OpenJDK。首先更新軟體包列表：sudo apt update。然後安裝 OpenJDK 21 開發版：sudo apt install openjdk-21-jdk。如果只需要運行 Java 程式而不需要編譯器，可以安裝 JRE：sudo apt install openjdk-21-jre。

安裝過程中會提示建立 JAVA_HOME 目錄，選擇「是」自動設定。安裝完成後，使用 java -version 和 javac -version 驗證安裝。

如果需要手動設定 JAVA_HOME，編輯 ~/.bashrc 或 ~/.profile 檔案，新增：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64。然後執行 source ~/.bashrc 使設定生效。

### 使用 YUM 安裝（Red Hat/CentOS）

在 Red Hat、CentOS、Fedora 等使用 YUM 的系統上，可以直接安裝 JDK。執行以下命令安裝 OpenJDK 21：sudo yum install java-21-openjdk java-21-openjdk-devel。-devel 包包含編譯器和其他開發工具。

安裝完成後，使用 alternatives 命令設定預設的 Java 版本：sudo alternatives --config java。按照提示選擇要設定為預設的 Java 版本。

編輯 ~/.bashrc 檔案，新增：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk。執行 source ~/.bashrc 使設定生效。

### 手動安裝二進制檔案

對於需要安裝特定版本 JDK 的情況，可以手動下載和解壓二進制歸檔檔案。訪問 Oracle 或 OpenJDK 下載頁面，下載對應版本的 .tar.gz 檔案。

建立 JDK 安裝目錄：sudo mkdir -p /opt/jdk。下載的檔案解壓：sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk。進入解壓後的目錄，記錄完整路徑。

編輯 ~/.bashrc 檔案，新增：export JAVA_HOME=/opt/jdk/jdk-21 和 export PATH=$JAVA_HOME/bin:$PATH。執行 source ~/.bashrc 使設定生效。

## 安裝多個 JDK 版本

### Windows 多版本管理

在 Windows 上安裝多個 JDK 版本後，需要通過環境變數切換預設版本。最簡單的方法是修改 PATH 環境變數中 JDK bin 目錄的順序，將要使用的版本放在前面。

另一個方法是使用 JAVA_HOME 環境變數，修改其指向不同的 JDK 安裝目錄。這種方法對大多數工具都有效，但某些工具可能直接使用 PATH 中的 java 命令而不是 JAVA_HOME。

還有一些第三方工具可以幫助管理多個 JDK 版本，如 SDKMAN! 的 Windows 版本或者 JetBrains 的 Toolbox。這些工具可以更方便地在不同版本之間切換。

### macOS 多版本管理

在 macOS 上，可以使用 jenv 工具來管理多個 JDK 版本。首先安裝 jenv：brew install jenv。然後將 jenv 初始化新增到 shell 設定檔案中。

使用 jenv 新增已安裝的 JDK：jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home。然後可以使用 jenv versions 查看所有管理的版本，使用 jenv global xx 設定全域預設版本。

另一個方法是使用 /usr/libexec/java_home -v 來指定特定版本執行。例如：/usr/libexec/java_home -v 21 --exec java -version。

### Linux 多版本管理

在 Linux 上，可以使用 update-alternatives 工具管理多個 JDK 版本。首先為每個 JDK 版本建立 alternatives 項目：sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1。

然後使用 update-alternatives --config java 選擇預設版本。jenv 在 Linux 上同樣可用，安裝和設定方法與 macOS 類似。

## 常見安裝問題

### 安裝後找不到 java 命令

如果在安裝 JDK 後找不到 java 命令，首先確認 JDK 是否正確安裝。檢查 JAVA_HOME 環境變數是否設定正確，檢查 PATH 中是否包含 %JAVA_HOME%\bin（Windows）或 $JAVA_HOME/bin（Linux/macOS）。

在 Windows 上，開啟新的命令列視窗嘗試，因為環境變數的修改可能不會立即生效。在 Linux/macOS 上，執行 source ~/.bashrc 或重新開啟終端機。

如果問題仍然存在，檢查 JDK 安裝目錄是否正確，目錄中是否包含 bin/java 檔案。

### 權限問題

在 Linux 和 macOS 上安裝 JDK 時，可能遇到權限問題。使用包管理器安裝時，通常需要管理員權限（sudo）。手動安裝時，確保安裝目錄有適當的讀寫權限。

如果一般用戶無法使用 java 命令，檢查 JAVA_HOME 目錄和 JDK bin 目錄的權限設定。可以使用 ls -la /usr/lib/jvm 檢查目錄權限。

### JAVA_HOME 設定無效

如果設定 JAVA_HOME 後無效，檢查環境變數設定的語法是否正確。在 Windows 上，變數值不應包含引號。在 Linux/macOS 上，確保使用正確的 export 命令。

同時檢查是否有其他腳本或設定檔覆蓋了 JAVA_HOME 的設定。在 Linux 上，系統層級的設定可能在 /etc/environment 或 /etc/profile.d/ 中。

### 舊版本解除安裝

在升級 JDK 後，可能需要解除安裝舊版本以節省磁碟空間。在 Windows 上，透過控制台的「程式和功能」找到舊版本 JDK，按一下解除安裝。在 macOS 上，刪除 /Library/Java/JavaVirtualMachines/ 目錄下的舊版本資料夾。在 Linux 上，使用包管理器解除安裝：sudo apt remove openjdk-xx-jdk（APT）或 sudo yum remove java-xx-openjdk（YUM）。
