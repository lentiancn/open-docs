# MongoDB 자주 묻는 질문

## 기본 개념

### MongoDB란?

MongoDB는 개발자가 최신 애플리케이션을 더 빨리 구축할 수 있도록 설계된 문서 지향 데이터베이스입니다.

### MongoDB는 관계형 데이터베이스인가요?

아닙니다. MongoDB는 BSON 형식을 사용하는 문서 지향 NoSQL 데이터베이스입니다.

### MongoDB의 문서란?

문서는 필드-값 쌍으로 구성된 MongoDB의 기본 데이터 단위입니다.

### MongoDB와 MySQL의 차이점은?

| 기능 | MongoDB | MySQL |
|------|---------|-------|
| 데이터 모델 | 문서 | 관계형 |
| 스키마 | 유연 | 고정 |
| 확장 | 수평 | 수직 |

## 설치 문제

### MongoDB는 얼마나 많은 메모리가 필요한가요?

MongoDB는 사용 가능한 메모리를 캐시로 사용합니다. 개발 환경에는 최소 4GB를 권장합니다.

### MongoDB는 32비트 시스템을 지원하나요?

아닙니다. 64비트 운영 체제가 필요합니다.

## 사용 문제

### MongoDB 데이터는 어떻게 백업하나요?

```bash
mongodump --db=mydatabase --out=/backup
```

### MongoDB는 어떻게 고가용성을 달성하나요?

Replica Set을 통해 고가용성을 달성합니다.

### Sharding이란?

Sharding은 데이터를 여러 서버에 분산시키는 MongoDB의 수평 확장 기술입니다.

### MongoDB는 트랜잭션을 지원하나요?

네. MongoDB 4.0+는 다중 문서 트랜잭션을 지원합니다.

### 쿼리 성능을 최적화하는 방법은?

1. 적절한 인덱스 생성
2. 프로젝션 사용
3. 전체 컬렉션 스캔 피하기

### MongoDB에서 데이터를 보호하는 방법은?

1. 인증 활성화
2. 역할 기반 액세스 제어 (RBAC) 사용
3. TLS/SSL 활성화
4. 정기적인 백업 수행

### Community와 Enterprise의 차이점은?

Enterprise Edition에는 다음이 포함됩니다:
- LDAP 인증
- 감사
- 암호화된 스토리지 엔진
- 기술 지원

## 개발

### MongoDB는 어떤 프로그래밍 언어를 지원하나요?

MongoDB는 주요 언어용 드라이버를 제공합니다:
- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP

### 애플리케이션에서 MongoDB를 사용하는 방법은?

1. 언어용 드라이버 설치
2. 연결 설정
3. CRUD 작업 실행

Node.js 예제:

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('myDB');
  const collection = database.collection('users');
  
  await collection.insertOne({ name: '홍길동', age: 30 });
  const user = await collection.findOne({ name: '홍길동' });
  
  await client.close();
}
```

### MongoDB Atlas란?

MongoDB Atlas는 MongoDB에서 제공하는 완전히 관리되는 클라우드 데이터베이스 서비스입니다.
