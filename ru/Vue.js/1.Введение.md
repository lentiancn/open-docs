# Vue.js Введение

Vue (произносится как /vjuː/, похоже на "view") — это JavaScript-фреймворк для создания пользовательских интерфейсов. Он основан на стандартных HTML, CSS и JavaScript и предоставляет декларативную, компонентную модель программирования, которая помогает эффективно разрабатывать пользовательские интерфейсы любой сложности.

## Основные функции

### Декларативный рендеринг
Vue расширяет стандартный HTML с помощью синтаксиса шаблонов, который позволяет нам декларативно описывать HTML-вывод на основе состояния JavaScript.

### Реактивность
Vue автоматически отслеживает изменения состояния JavaScript и эффективно обновляет DOM при возникновении изменений.

## Прогрессивный фреймворк

Vue — это фреймворк и экосистема, которые охватывают большинство общих функций, необходимых для фронтенд-разработки. Веб-пространство чрезвычайно разнообразно — вещи, которые мы создаем в интернете, могут сильно различаться по форме и масштабу. С учетом этого Vue спроектирован так, чтобы быть гибким и постепенно внедряемым. В зависимости от вашего случая использования, Vue можно использовать по-разному:

- Улучшение статического HTML без этапа сборки
- Встраивание как Web Components на любой странице
- Одностраничное приложение (SPA)
- Fullstack / Серверный рендеринг (SSR)
- Jamstack / Генерация статических сайтов (SSG)
- Разработка для десктопа, мобильных устройств, WebGL и даже терминала

## Компоненты в одном файле

В большинстве проектов Vue, использующих инструменты сборки, мы создаем компоненты Vue с использованием формата файла, похожего на HTML, называемого Single-File Component (также известного как файлы *.vue, сокращенно SFC). Vue SFC, как следует из названия, инкапсулирует логику компонента (JavaScript), шаблон (HTML) и стили (CSS) в одном файле.

## Стили API

Компоненты Vue могут быть созданы в двух разных стилях API: Options API и Composition API.

### Options API
С Options API мы определяем логику компонента с помощью объекта опций, таких как data, methods и mounted. Свойства, определенные опциями, доступны через this внутри функций, что указывает на экземпляр компонента.

### Composition API
С Composition API мы определяем логику компонента с помощью импортированных функций API. В SFC Composition API обычно используется с `<script setup>`. Атрибут setup является подсказкой, которая заставляет Vue выполнять преобразования во время компиляции, позволяя нам использовать Composition API с меньшим количеством шаблонного кода.
