# Manual del Usuario

## Fundamentos de Pipeline

Pipeline es una función principal de Jenkins que automatiza todo el proceso de desarrollo de software, desde la construcción hasta las pruebas y el despliegue.

## ¿Qué es Pipeline?

Pipeline es un conjunto de plugins de Jenkins que permite implementar e integrar pipelines de entrega continua en Jenkins. En pocas palabras, Pipeline automatiza todo el proceso de llevar el software desde el control de versiones hasta los usuarios.

Las definiciones de Pipeline se escriben en un archivo de texto (Jenkinsfile) que puede comprometerse con el repositorio de control de código fuente del proyecto. Esto implementa la filosofía de "Pipeline como Código".

### Beneficios de Pipeline

- **Basado en código**: Los pipelines se versionan como código, permitiendo revisión e iteración
- **Duradero**: Los pipelines sobreviven tanto a reinicios planeados como no planeados de Jenkins
- **Pausable**: Los pipelines pueden detenerse y esperar entrada o aprobación humana
- **Versátil**: Soporta requisitos complejos de CD del mundo real, incluyendo ejecución paralela y bucles
- **Extensible**: Los plugins pueden extender el DSL de Pipeline e integrar con otras herramientas

### Conceptos de Pipeline

#### Nodo

Un nodo es una máquina que forma parte del entorno de Jenkins y es capaz de ejecutar un Pipeline. Los nodos pueden ser máquinas maestras o agentes.

#### Etapa

Una etapa define un subconjunto conceptualmente diferente de tareas realizadas a través de todo el Pipeline, como las etapas "Build", "Test" y "Deploy". Las etapas se usan para visualizar el estado de ejecución del Pipeline.

#### Paso

Un paso es una acción fundamental en Jenkins Pipeline que le dice a Jenkins qué hacer en un momento particular. Por ejemplo, ejecutar un comando shell usando el paso `sh`.

### Sintaxis de Pipeline

#### Pipeline Declarativo

Pipeline Declarativo es una sintaxis más nueva con funciones más ricas, más fácil de escribir y leer:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                echo 'Construyendo aplicación...'
            }
        }
        stage('Test') {
            steps {
                echo 'Ejecutando pruebas...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Desplegando aplicación...'
            }
        }
    }
}
```

#### Pipeline Scripted

Pipeline Scripted usa sintaxis Groovy, más flexible pero requiere más experiencia en programación:

```groovy
node {
    stage('Build') {
        echo 'Construyendo aplicación...'
    }
    stage('Test') {
        echo 'Ejecutando pruebas...'
    }
    stage('Deploy') {
        echo 'Desplegando aplicación...'
    }
}
```

## Creando su Primer Pipeline

### Usando Blue Ocean

Blue Ocean es la interfaz moderna de Jenkins que hace que crear pipelines sea más intuitivo:

1. Instale el plugin de Blue Ocean (si no está preinstalado)
2. Haga clic en "Abrir Blue Ocean"
3. Haga clic en "Crear Pipeline"
4. Seleccione la ubicación del repositorio de código (GitHub, GitLab, etc.)
5. Seleccione repositorio y rama
6. Blue Ocean detectará automáticamente el Jenkinsfile o ayudará a crear uno

### Usando Jenkinsfile

Cree un Jenkinsfile en su proyecto, y Jenkins lo reconocerá automáticamente:

1. Cree Jenkinsfile en la raíz del proyecto
2. Defina las etapas del pipeline
3. Comprometa con el control de versiones
4. Cree un trabajo de pipeline en Jenkins y vincule al repositorio

## Gestión de Credenciales

Jenkins necesita acceder a varios recursos externos como repositorios de código y objetivos de despliegue, lo cual requiere gestión de credenciales.

### Agregar Credenciales

1. Vaya a "Manage Jenkins" > "Manage Credentials"
2. Haga clic en "Add Credentials"
3. Elija el tipo de credencial (Usuario y Contraseña, SSH con Clave Privada, Archivo Secreto, etc.)
4. Complete la información de credenciales
5. Establezca ID y descripción para la credencial

### Usando Credenciales en Pipeline

Use el paso withCredentials:

```groovy
pipeline {
    agent any
    stages {
        stage('Usar Credenciales') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "Usuario: $USER"
                }
            }
        }
    }
}
```

## Disparadores de Construcción

Las construcciones automatizadas requieren disparadores. Jenkins soporta múltiples métodos de disparador.

### Sondeo de SCM

Verificar regularmente si el código fuente ha cambiado:

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // Verificar cada 5 minutos
    }
    stages {
        stage('Build') {
            steps {
                echo 'Cambio de código detectado, iniciando construcción'
            }
        }
    }
}
```

### Disparador por Webhook

Dispare construcciones vía webhooks del repositorio para mayor eficiencia:

1. Instale el plugin correspondiente (ej. GitHub Hook Trigger)
2. Configure el webhook del repositorio apuntando a Jenkins
3. La construcción se activa automáticamente al comprometer código

### Construcciones Programadas

Ejecutar automáticamente según un calendario:

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // Ejecutar a las 2 AM diariamente
    }
    stages {
        stage('Construcción Programada') {
            steps {
                echo 'Construcción programada en ejecución'
            }
        }
    }
}
```

## Usando Agentes y Contenedores

### Configurando Agentes

Jenkins soporta construcciones distribuidas, distribuyendo tareas de construcción entre múltiples máquinas:

1. Vaya a "Manage Jenkins" > "Manage Nodes"
2. Agregue nodo agente y configure información de conexión
3. Especifique el agente en el pipeline

### Usando Docker en Pipeline

Pipeline Declarativo puede ejecutar construcciones en contenedores Docker:

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## Informes de Pruebas

Jenkins puede integrarse con varios marcos de pruebas para generar informes visuales de pruebas.

### Informes de Pruebas JUnit

```groovy
pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### Otros Informes de Pruebas

Jenkins soporta varios plugins de informes de pruebas:

- TestNG
- Cobertura (cobertura de código)
- JaCoCo (cobertura de código)
- ESLint (análisis de código frontend)

## Prácticas de Despliegue

### Desplegando a Diferentes Entornos

Pipeline típico de despliegue multi-etapa:

```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('Deploy to Test') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('Deploy to Production') {
            when {
                branch 'main'
            }
            steps {
                input message: 'Confirmar despliegue a producción?', ok: 'Desplegar'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### Despliegue Blue-Green

Implemente despliegue blue-green con Jenkins para actualizaciones sin tiempo de inactividad:

1. Mantenga dos entornos idénticos (blue y green)
2. Cambie el tráfico a la nueva versión
3. Revierta rápidamente si surgen problemas

## Mejores Prácticas

### Mejores Prácticas de Jenkinsfile

- Mantener Jenkinsfile en control de versiones
- Usar sintaxis de pipeline declarativo
- Mantener pipelines simples y claros
- Dividir etapas razonablemente
- Agregar mecanismos necesarios de tiempo de espera y reintentos

### Recomendaciones de Seguridad

- Actualizar regularmente Jenkins y plugins
- Limitar permisos de usuarios anónimos
- Usar credenciales para información sensible
- Configurar control de acceso apropiado
- Habilitar registro de auditoría de seguridad

### Optimización de Rendimiento

- Configurar nodos de construcción apropiadamente
- usar construcciones incrementales para reducir tiempo
- Configurar caché de construcción
- Limpiar regularmente registros de construcción antiguos
- Usar etapas paralelas para acelerar construcciones

## Solución de Problemas

### Ver Registros de Construcción

Cuando una construcción falla, primero revise los registros:

1. Vaya a la página del trabajo
2. Haga clic en el número de construcción que falló
3. Revise la salida de la consola

### Solución de Problemas Comunes

- Construcción atascada: Verificar si hay entrada pendiente o bloqueos de recursos
- Credenciales inválidas: Verificar si las credenciales expiraron o fueron revocadas
- Conflictos de plugins: Intentar deshabilitar plugins instalados recientemente
- Sin memoria: Ajustar parámetros de heap de JVM

- Integrar herramientas de prueba y despliegue
- Aplicar mejores prácticas para optimizar flujos de trabajo

El aprendizaje y práctica continuos son clave para dominar Jenkins. Continúe explorando y optimizando en proyectos reales.
