# Java 자주 묻는 질문

## 기본 문제

### 문제 1: java 및 javac 명령을 찾을 수 없음

**설명**: 명령줄에 java -version 또는 javac -version을 입력하면 명령을 찾을 수 없다고 표시됩니다.

**해결책**:
1. JDK가 올바르게 설치되었는지 확인하세요
2. JAVA_HOME 환경 변수가 구성되어 있는지 확인하세요
3. Path 환경 변수에 %JAVA_HOME%\bin(Windows) 또는 $JAVA_HOME/bin(Linux/macOS)이 포함되어 있는지 확인하세요
4. 환경 변수 변경을 적용하려면 터미널을 다시 여세요

### 문제 2: 기본 클래스를 찾거나 로드할 수 없음

**설명**: java 명령을 실행하면 "Could not find or load main class"가 표시됩니다.

**해결책**:
1. 올바른 디렉터리에서 명령을 실행하고 있는지 확인하세요
2. 클래스 이름이 파일 이름과 정확히 일치하는지 확인하세요
3. 클래스가 패키지에 있는 경우 실행할 때 전체 패키지 이름을 사용하세요
4. classpath가 올바르게 설정되어 있는지 확인하세요

### 문제 3: 인코딩 문제로 인한 中文乱码

**설명**: 프로그램의 출력이 깨져 보입니다.

**해결책**:
1. 컴파일 시 인코딩 지정: javac -encoding UTF-8 MyClass.java
2. 실행 시 인코딩 지정: java -Dfile.encoding=UTF-8 MyClass
3. 소스 파일이 UTF-8로 저장되어 있는지 확인하세요
4. IDE에서 올바른 파일 인코딩을 설정하세요

## 환경 구성 문제

### 문제 4: 여러 JDK 버전 충돌

**설명**: 시스템에 여러 JDK 버전이 있고 java 명령이 잘못된 버전을 사용합니다.

**해결책**:
1. JAVA_HOME이 어떤 JDK 버전을 가리키는지 확인하세요
2. Windows에서는 where java를 사용하여 모든 java 명령 위치를 확인하세요
3. Linux/macOS에서는 which java를 사용하여 경로를 확인하세요
4. Path 환경 변수의 JDK bin 디렉터리 순서를 조정하세요
5. update-alternatives(Linux)를 사용하여 여러 버전을 관리하세요

### 문제 5: 포트가 이미 사용 중

**설명**: Java 웹 애플리케이션을 시작하면 포트가 이미 사용 중이라고 표시됩니다.

**해결책**:
1. Windows: netstat -ano | findstr 8080으로 포트를 사용하는 프로세스를 찾으세요
2. Linux: lsof -i:8080 또는 netstat -tulpn | grep 8080
3. 포트를 사용하는 프로세스를 종료하거나 애플리케이션의 포트 구성을 변경하세요

### 문제 6: 메모리 부족 오류

**설명**: 프로그램 실행 중 OutOfMemoryError가 발생합니다.

**해결책**:
1. JVM 힙 크기 증가: java -Xmx2048m MyClass
2. 코드에 메모리 누수가 있는지 확인하세요
3. 메모리 분석 도구(VisualVM, JProfiler 등)를 사용하여 문제를 파악하세요
4. 데이터 구조를 최적화하여 메모리 사용량을 줄이세요

## 일반적인 개발 문제

### 문제 7: NullPointerException

**설명**: NullPointerException은 가장 일반적인 런타임 예외입니다.

**해결책**:
1. 객체를 사용하기 전에 null인지 확인하세요
2. Optional 클래스를 사용하여 null 포인터를 피하세요
3. Objects.requireNonNull()을 사용하여 매개변수를 검증하세요
4. 필요한 모든 객체를 초기화하세요

### 문제 8: ConcurrentModificationException

**설명**: 컬렉션을 반복하는 동안 컬렉션을 수정하면 예외가 발생합니다.

**해결책**:
1. 반복자의 remove() 메서드를 사용하여 요소를 삭제하세요
2. ArrayList 대신 CopyOnWriteArrayList를 사용하세요
3. 먼저 삭제할 요소를 기록한 후 반복 후 삭제하세요
4. Stream API의 필터를 사용하여 요소를 필터링하세요

### 문제 9: ClassCastException

**설명**: ClassCastException이 발생합니다.

**해결책**:
1. instanceof를 사용하여 객체 유형을 확인하세요
2. 구체적인 구현보다 인터페이스 유형을 선호하세요
3. 변환 전에 유형 검사를 수행하세요

### 문제 10: 숫자 정밀도 문제

**설명**: 부동소수점 계산이 정확하지 않습니다.

**해결책**:
1. 정확한 계산을 위해 double 대신 BigDecimal을 사용하세요
2. 부동소수점 정밀도 제한을 이해하세요
3. 부동소수점 비교를 피하고 BigDecimal의 compareTo를 사용하세요

## 성능 문제

### 문제 11: 프로그램 실행缓慢

**설명**: Java 프로그램이 느리게 실행됩니다.

**해결책**:
1. 성능 프로파일링 도구를 사용하여 병목 지점을 파악하세요
2. 반복문에서 반복 계산을 피하세요
3. 문자열 연결 대신 StringBuilder를 사용하세요
4. 적절한 컬렉션 유형을 선택하세요
5. JIT 컴파일러 최적화를 활성화하세요

### 문제 12: 데드락

**설명**: 프로그램이 데드락되어 실행을 계속할 수 없습니다.

**해결책**:
1. 중첩 잠금을 피하세요
2. 잠금 순서를 통일하세요
3. tryLock() 시간 초과 메커니즘을 사용하세요
4. 동기화 블록 범위를 줄이세요

## 빌드 도구 문제

### 문제 13: Maven/Gradle 종속성 충돌

**설명**: 빌드 중 종속성 버전 충돌이 발생합니다.

**해결책**:
1. mvn dependency:tree를 사용하여 종속성 트리를 확인하세요
2. pom/gradle에서 종속성 버전을 명시적으로 지정하세요
3. 전달 종속성을 제외하세요
4. Maven Central의 Bom을 사용하여 버전을 관리하세요

### 문제 14: 빌드 실패

**설명**: Maven 또는 Gradle 빌드가 실패합니다.

**해결책**:
1. 오류 메시지를 보고 문제를 파악하세요
2. 캐시 지우기: mvn clean 또는 gradle clean
3. 네트워크 연결 확인
4. JDK 버전이 프로젝트 요구 사항을 충족하는지 확인
5. 구문 오류 확인

## 프레임워크 문제

### 문제 15: Spring Boot 시작 실패

**설명**: Spring Boot 애플리케이션을 시작할 수 없습니다.

**해결책**:
1. 시작 로그에서 오류 메시지를 확인하세요
2. application.properties 또는 application.yml 구성을 확인하세요
3. 데이터베이스 연결 구성이 올바른지 확인하세요
4. 종속성이 완전한지 확인하세요
5. 포트가 사용 중인지 확인하세요

### 문제 16: Hibernate 지연 로드 예외

**설명**: 세션이 닫힌 후 지연 로드된 객체에 접근합니다.

**해결책**:
1. @Fetch(FetchMode.JOIN)을 사용하여 즉시 로드하세요
2. 세션이 닫히기 전에 필요한 객체를 초기화하세요
3. Hibernate.initialize()를 사용하여 수동 로드하세요
4. spring.jpa.open-in-view=true를 구성하세요

## 버전 선택 문제

### 문제 17: 어떤 Java 버전을 학습해야 할까요

**권장 사항**:
- 초보자: Java 21 또는 Java 17 권장, 최신 문법, 자료 풍부
- 기업 프로젝트: 프로젝트 종속성에 따라, 일반적으로 Java 8, 11 또는 17
- 새 프로젝트: Java 21 이상 강력 권장

### 문제 18: JDK와 JRE의 차이점

**답변**:
- JDK (Java Development Kit): 개발 키트, JRE 및 컴파일/디버그 도구 포함
- JRE (Java Runtime Environment): 실행 환경, Java 프로그램 실행에 필요한 클래스 라이브러리와 JVM만 포함

Java 프로그램을 개발하려면 JDK를 설치하세요. 컴파일된 Java 프로그램만 실행하려면 JRE로 충분합니다.

## 기타 일반적인 문제

### 문제 19: Java를 어떻게 학습해야 할까요

**권장 사항**:
1. Java 기본 문법과 객체 지향 개념을 마스터하세요
2. Java 컬렉션 프레임워크와 일반적인 유틸리티 클래스에 익숙해지세요
3. JDBC 데이터베이스 작업을 학습하세요
4. 최소 하나의 웹 프레임워크(Spring/Spring Boot)를 마스터하세요
5. 디자인 패턴과 코드 리팩터링을 이해하세요
6. 더 많이 연습하고 프로젝트를 작성하세요

### 문제 20: Java가 얼마나 오래 계속热门할까요

**분석**: 기업 개발의 주요 언어로 Java는 성숙한 생태계와庞大的 사용자 베이스를 가지고 있습니다. TIOBE 순위에 따르면 Java는 Consistently 상위 3위에 랭크됩니다. Java는 계속 업데이트되며, 6개월마다 새 버전이, 2년마다 LTS 버전이 출시됩니다. 따라서 예측 가능한 미래에 Java는 여전히 가치 있는 기술이 될 것입니다.
