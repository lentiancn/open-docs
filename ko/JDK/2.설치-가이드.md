# JDK 설치 가이드

## Windows에서 JDK 설치

### 시스템 요구 사항

JDK를 설치하기 전에 컴퓨터가 다음 최소 요구 사항을 충족하는지 확인하세요: 프로세서, Intel Core i3 이상 권장; 메모리, 최소 4GB RAM 권장, 8GB 이상 선호; 디스크 공간, JDK 설치를 위해 최소 2GB 여유 공간, 개발 프로젝트용 추가 공간; 운영 체제, Windows 10 또는 Windows 11 64비트.

Windows 7 및 8.1도 작동할 수 있지만, Oracle은 더 이상 이러한 운영 체제를 공식적으로 지원하지 않습니다. 더 오래된 시스템에서 실행하면 호환성 문제가 발생할 수 있습니다.

### JDK 다운로드

먼저 https://www.oracle.com/java/technologies/downloads/ 의 Oracle 공식 웹사이트를 방문하여 JDK를 다운로드하세요. 다운로드 페이지에서 다양한 JDK 버전을 볼 수 있습니다: JDK 25는 최신 버전, JDK 21은 최신 LTS 버전, JDK 17도 성숙한 LTS 버전입니다. 프로젝트 요구 사항에 따라 적절한 버전을 선택하세요.

Windows 시스템의 경우 x64 설치 프로그램(.exe) 또는 x64 MSI 설치 프로그램을 선택하세요. .exe는 기존 설치 마법사이고, .msi는 기업 환경에서 일괄 배포에 적합합니다. 설치 없이 Java 프로그램만 실행해야 하는 경우 압축된 x64(.zip) 아카이브를 선택하면 되며, 압축을 풀고 직접 사용할 수 있습니다.

다운로드 시 라이선스 약관에 주의하세요. Oracle JDK는 특정 버전과 사용에 대해 무료이지만 상업적 사용 및 재분배에 제한이 있습니다. 완전히 무료로 사용해야 하는 경우 Eclipse Temurin과 같은 OpenJDK 또는 OpenJDK 기반 다른 배포판 사용을 고려하세요.

### 설치 단계

다운로드한 JDK 설치 파일(일반적으로 jdk-xx_windows-x64_bin.exe)을 찾고 두 번 클릭하여 실행하세요. 설치 마법사에서는 "다음"을 클릭하여 계속하세요. 먼저 대상 폴더 선택 화면이 표시되며, 기본 설치 경로는 C:\Program Files\Java\jdk-xx입니다. 설치 위치를 변경하려면 "찾아보기"를 클릭하여 다른 디렉토리를 선택하세요. 나중에 구성 문제를 피하기 위해 기본 경로를 사용하는 것이 좋습니다.

그 다음에는 기능 선택 화면이 나타나며, 기본적으로 개발 도구, 소스 코드 및 공용 JRE를 포함한 모든 기능이 선택되어 있습니다. 기본값으로 전체 설치를 유지하는 것이 좋습니다. "다음"을 클릭하여 설치를 시작하세요. 설치 프로세스에 몇 분이 걸릴 수 있으므로 잠시 기다려주세요.

설치가 완료되면 메시지가 표시됩니다. "닫기"를 클릭하여 완료하세요. 이 시점에서 JDK가 지정된 디렉토리에 설치되었지만, 명령줄에서 java, javac 및 다른 명령어를 사용하려면 환경 변수를 구성해야 합니다.

### 환경 변수 구성

환경 변수 구성의目的是 시스템이 JDK 설치 위치를 찾을 수 있도록 하여 어디서나 java, javac 및 다른 명령어를 사용할 수 있도록 하는 것입니다. "이 컴퓨터" 또는 "내 컴퓨터"를 마우스 오른쪽 버튼으로 클릭하고 "속성"을 선택하세요. 왼쪽 메뉴에서 "고급 시스템 설정"을 클릭하세요.

열린 시스템 속성 창에서 "환경 변수" 버튼을 클릭하세요. 시스템 변수 영역에서 "새로 만들기"를 클릭하세요. 변수 이름에 JAVA_HOME을, 변수 값에 JDK 설치 경로(예: C:\Program Files\Java\jdk-21)를 입력한 다음 확인을 클릭하세요.

시스템 변수에서 Path 변수를 찾고 "편집"을 클릭하세요. "새로 만들기"를 클릭하고 %JAVA_HOME%\bin을 입력하세요. 이 줄이 맨 위에 있도록 하여 새로 설치된 JDK가 우선순위를 갖도록 하세요. 모든 설정을 저장하려면 확인을 클릭하세요.

새 명령줄 창(cmd 또는 PowerShell)을 열고 java -version과 javac -version을 입력하세요. 버전 정보가 표시되면 구성이 성공적인 것입니다.

### 설치 확인

명령 프롬프트(Win+R, cmd 입력)를 열고 다음 명령을 실행하여 JDK가 올바르게 설치되었는지 확인하세요:

java -version을 실행하면 Java 런타임 환경 정보가 표시되어야 합니다. javac -version을 실행하면 Java 컴파일러 정보가 표시되어야 합니다. echo %JAVA_HOME%를 실행하면 이전에 설정한 JAVA_HOME 경로가 표시되어야 합니다.

명령이 오류를 표시하면 환경 변수가 올바르게 구성되었는지, JDK가 올바르게 설치되었는지 확인하세요.

## macOS에서 JDK 설치

### DMG 설치 프로그램 사용

macOS 사용자는 DMG 설치 프로그램을 사용하여 JDK를 빠르게 설치할 수 있습니다. Oracle 공식 다운로드 페이지를 방문하여 macOS용 JDK 버전을 다운로드하세요. Apple Silicon( M1/M2/M3 칩)의 경우 ARM64 DMG Installer를, Intel 칩 기반 Mac의 경우 x64 DMG Installer를 선택하세요.

다운로드 후 DMG 파일을 열면 pkg 설치 파일이 표시됩니다. 두 번 클릭하여 설치 마법사를 실행하고 지침에 따라 설치를 완료하세요. 설치 프로그램은 JDK를 /Library/Java/JavaVirtualMachines/ 디렉토리에 배치합니다.

설치 후 터미널을 열고 java -version을 입력하여 확인하세요. 시스템에 여러 버전의 JDK가 있는 경우 /usr/libexec/java_home -V를 사용하여 설치된 모든 버전을 볼 수 있습니다.

### Homebrew로 설치

Homebrew가 이미 설치되어 있다면 사용하여 JDK를 설치하는 것이 더 편리합니다. 터미널을 열고 실행하세요: brew update. 그런 다음 JDK 21을 설치하세요: brew install openjdk@21.

설치 후 JDK를 PATH에 추가해야 합니다. 다음 명령을 실행하여 JDK를 시스템에 연결하세요: sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.

그런 다음 ~/.zshrc 또는 ~/.bash_profile에 환경 변수를 추가하세요: export PATH="/usr/local/opt/openjdk@21/bin:$PATH". 저장하고 source ~/.zshrc를 실행하여 구성을 적용하세요.

### JAVA_HOME 구성

macOS에서 JAVA_HOME 환경 변수 구성은 필수적이지 않습니다. 시스템이 자동으로 설치된 JDK를 찾기 때문입니다. 그러나 일부 도구(Maven, Gradle 등)는 수동 구성이 필요할 수 있습니다. 터미널에서 ~/.zshrc 파일을 편집하고 추가하세요: export JAVA_HOME=$(/usr/libexec/java_home). 저장하고 source ~/.zshrc를 실행하여 구성을 적용하세요.

## Linux에서 JDK 설치

### APT로 설치 (Debian/Ubuntu)

Debian 및 Ubuntu 시스템에서는 APT 패키지 관리자를 사용하여 OpenJDK를 설치할 수 있습니다. 먼저 패키지 목록을 업데이트하세요: sudo apt update. 그런 다음 OpenJDK 21 개발 버전을 설치하세요: sudo apt install openjdk-21-jdk. 컴파일러 없이 Java 프로그램만 실행해야 하는 경우 JRE를 설치하세요: sudo apt install openjdk-21-jre.

설치하는 동안 JAVA_HOME 디렉토리를 생성할 것인지 묻는 메시지가 표시됩니다. 자동 구성을 위해 "예"를 선택하세요. 설치 후 java -version과 javac -version을 사용하여 설치를 확인하세요.

JAVA_HOME을 수동으로 구성해야 하는 경우 ~/.bashrc 또는 ~/.profile 파일을 편집하고 추가하세요: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64. 그런 다음 source ~/.bashrc를 실행하여 구성을 적용하세요.

### YUM으로 설치 (Red Hat/CentOS)

YUM을 사용하는 시스템(Red Hat, CentOS, Fedora)에서는 JDK를 직접 설치할 수 있습니다. 다음 명령을 실행하여 OpenJDK 21을 설치하세요: sudo yum install java-21-openjdk java-21-openjdk-devel. -devel 패키지에는 컴파일러 및 기타 개발 도구가 포함되어 있습니다.

설치 후 alternatives 명령을 사용하여 기본 Java 버전을 정의하세요: sudo alternatives --config java. 지침에 따라 기본 Java 버전을 선택하세요.

~/.bashrc 파일을 편집하고 추가하세요: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk. source ~/.bashrc를 실행하여 구성을 적용하세요.

### 바이너리 파일 수동 설치

특정 JDK 버전을 설치해야 하는 상황에서는 바이너리 파일을 수동으로 다운로드하고 추출할 수 있습니다. Oracle 또는 OpenJDK 다운로드 페이지를 방문하여 원하는 버전의 .tar.gz 파일을 다운로드하세요.

JDK 설치 디렉토리를 생성하세요: sudo mkdir -p /opt/jdk. 다운로드한 파일을 추출하세요: sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk. 추출된 디렉토리로 이동하여 전체 경로를 기억하세요.

~/.bashrc 파일을 편집하고 추가하세요: export JAVA_HOME=/opt/jdk/jdk-21 및 export PATH=$JAVA_HOME/bin:$PATH. source ~/.bashrc를 실행하여 구성을 적용하세요.

## 여러 JDK 버전 관리

### Windows에서 다중 버전 관리

Windows에 여러 버전의 JDK를 설치한 후 환경 변수를 통해 기본 버전을 변경할 수 있습니다. 가장 간단한 방법은 PATH 환경 변수에서 JDK bin 디렉토리 순서를 변경하여 사용하려는 버전을 맨 앞에 배치하는 것입니다.

또 다른 방법은 JAVA_HOME 환경 변수를 사용하여 다른 JDK 설치 디렉토리를 가리키게 하는 것입니다. 이 방법은 대부분의 도구에 적용되지만 일부 도구는 JAVA_HOME 대신 PATH의 java 명령을 사용할 수 있습니다.

일부 타사 도구는 JDK 다중 버전 관리에 도움이 될 수 있습니다(예: Windows용 SDKMAN! 버전 또는 JetBrains Toolbox). 이러한 도구는 버전 간 전환을更容易하게 합니다.

### macOS에서 다중 버전 관리

macOS에서는 jenv 도구를 사용하여 여러 JDK 버전을 관리할 수 있습니다. 먼저 jenv를 설치하세요: brew install jenv. 그런 다음 셸 구성 파일에 jenv 초기화를 추가하세요.

jenv에 설치된 JDK를 추가하세요: jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home. 그런 다음 jenv versions를 사용하여 관리되는 모든 버전을 보고 jenv global xx로 기본 전역 버전을 설정할 수 있습니다.

또 다른 방법은 /usr/libexec/java_home -v를 사용하여 특정 버전을 지정하여 프로그램을 실행하는 것입니다. 예: /usr/libexec/java_home -v 21 --exec java -version.

### Linux에서 다중 버전 관리

Linux에서는 update-alternatives 도구를 사용하여 여러 JDK 버전을 관리합니다. 먼저 각 JDK 버전에 대한 대안 항목을 만드세요: sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1.

그런 다음 update-alternatives --config java를 사용하여 기본 버전을 선택하세요. jenv도 Linux에서 사용할 수 있으며, macOS와 유사한 설치 및 구성 방법이 있습니다.

## 일반적인 설치 문제

### 설치 후 java 명령을 찾을 수 없음

JDK를 설치한 후 java 명령을 찾을 수 없는 경우 먼저 JDK가 올바르게 설치되었는지 확인하세요. JAVA_HOME 환경 변수가 올바르게 설정되어 있는지, PATH에 %JAVA_HOME%\bin(Windows) 또는 $JAVA_HOME/bin(Linux/macOS)이 포함되어 있는지 확인하세요.

Windows에서는 환경 변수 변경이 즉시 적용되지 않을 수 있으므로 새 명령줄 창을 여세요. Linux/macOS에서는 source ~/.bashrc를 실행하거나 터미널을 다시 여세요.

문제가 지속되면 JDK 설치 디렉토리가 올바른지, 해당 디렉토리에 bin/java 파일이 있는지 확인하세요.

### 권한 문제

Linux 및 macOS에서 JDK를 설치할 때 권한 문제가 발생할 수 있습니다. 패키지 관리자 사용에는 일반적으로 관리자 권한(sudo)이 필요합니다. 수동 설치에서는 설치 디렉토리에 적절한 읽기/쓰기 권한이 있는지 확인하세요.

일반 사용자가 java 명령을 사용할 수 없는 경우 JAVA_HOME 디렉토리 및 JDK bin 디렉토리의 권한을 확인하세요. ls -la /usr/lib/jvm을 사용하여 디렉토리 권한을 확인할 수 있습니다.

### JAVA_HOME 구성이 작동하지 않음

JAVA_HOME 설정이 작동하지 않으면 환경 변수 구문이 올바른지 확인하세요. Windows에서는 변수 값에 따옴표가 없어야 합니다. Linux/macOS에서는 올바른 export 명령을 사용하고 있는지 확인하세요.

다른 스크립트나 구성 파일이 JAVA_HOME 설정을 덮어쓰지 않는지도 확인하세요. Linux에서는 시스템 수준 구성이 /etc/environment 또는 /etc/profile.d/에 있을 수 있습니다.

### 이전 버전 제거

JDK를 업그레이드한 후 디스크 공간을 절약하려면 이전 버전을 제거해야 할 수 있습니다. Windows에서 "제어판"의 "프로그램 및 기능"에서 이전 JDK 버전을 찾아 "제어판"을 클릭하여 제거하세요. macOS에서 /Library/Java/JavaVirtualMachines/ 아래의 이전 버전 폴더를 삭제하세요. Linux에서 패키지 관리자를 사용하여 제거하세요: sudo apt remove openjdk-xx-jdk(APT) 또는 sudo yum remove java-xx-openjdk(YUM).
