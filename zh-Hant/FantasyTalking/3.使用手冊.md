# FantasyTalking 使用手冊

## 基本功能

FantasyTalking 提供了完善的語�合成和對話功能，本手冊將詳細介紹各項功能的使用方法。

### 語音合成

語音合成是 FantasyTalking 的核心功能，將文字轉換為自然流暢的語音。以下是基本的使用方法：

```javascript
// 簡單文字轉語音
const audio = await client.speak('你好，歡迎使用 FantasyTalking');

// 指定語音風格
const audio = await client.speak('今天天氣真好', {
  voice: {
    style: 'happy',
    language: 'zh-TW'
  }
});

// 指定語速和音調
const audio = await client.speak('請稍等片刻', {
  voice: {
    speed: 1.2,
    pitch: 0.9
  }
});
```

### SSML 標記語言

FantasyTalking 支援 SSML 標記語言，可以對語音輸出進行更精細的控制：

```xml
<speak>
  <p>
    <s>第一句話。</s>
    <s>第二句話。</s>
  </p>
  <break time="500ms"/>
  <p>
    <emphasis level="moderate">這是重點內容</emphasis>
  </p>
</speak>
```

對應的 JavaScript 代碼：

```javascript
const ssml = `
  <speak>
    <p>
      <s>第一句話。</s>
      <s>第二句話。</s>
    </p>
    <break time="500ms"/>
    <p>
      <emphasis level="moderate">這是重點內容</emphasis>
    </p>
  </speak>
`;

const audio = await client.speak(ssml, { ssml: true });
```

### 語音事件

在語音播放過程中，可以監聽各種事件來實現互動：

```javascript
// 監聽語音開始
client.on('speech-start', (event) => {
  console.log('語音開始播放');
});

// 監聽單詞邊界（用於字幕同步）
client.on('word-boundary', (event) => {
  console.log(`當前單詞: ${event.word}`);
  console.log(`開始時間: ${event.start}ms`);
  console.log(`結束時間: ${event.end}ms`);
});

// 監聽語音結束
client.on('speech-end', (event) => {
  console.log('語音播放完成');
});

// 監聽錯誤
client.on('error', (event) => {
  console.error('錯誤:', event.error);
});
```

### 批量合成

需要合成大量文字時，可以使用批量合成功能來提高效率：

```javascript
const texts = [
  '這是第一條語音',
  '這是第二條語音',
  '這是第三條語音'
];

// 平行合成
const results = await Promise.all(
  texts.map(text => client.speak(text))
);

// 串行合成（保留順序）
const results = [];
for (const text of texts) {
  const audio = await client.speak(text);
  results.push(audio);
}
```

## 對話功能

### 基礎對話

FantasyTalking 提供了強大的對話管理功能，支援多輪對話和上下文記憶：

```javascript
// 建立對話會話
const session = client.createSession({
  // 可選：設定系統提示
  systemPrompt: '你是一個友善的助手，擅長回答問題',
  // 可選：設定語言
  language: 'zh-TW'
});

// 第一輪對話
const response1 = await session.chat('今天天氣怎麼樣？');
console.log(response1.text);  # AI 回覆的文字
console.log(response1.audio);  # AI 回覆的語音

// 第二輪對話（自動包含上下文）
const response2 = await session.chat('明天呢？');
console.log(response2.text);
```

### 對話配置

可以針對不同的對話場景進行配置：

```javascript
const session = client.createSession({
  // 對話配置
  dialogue: {
    // 溫度參數，控制回覆的創造性
    temperature: 0.7,
    // 最大回覆 token 數
    maxTokens: 500,
    // 話題標籤，用於區分不同場景
    tags: ['客服', '售前諮詢']
  },
  
  // 語音配置
  voice: {
    // AI 回覆時的語音風格
    style: 'professional',
    // 語速
    speed: 1.0
  }
});
```

### 對話歷史管理

```javascript
// 取得對話歷史
const history = session.getHistory();
console.log(history);

// 保存對話歷史
const savedHistory = session.save();
fs.writeFileSync('conversation.json', JSON.stringify(savedHistory));

// 載入對話歷史
const loadedSession = client.loadSession(savedHistory);
```

## 進階功能

### 串流語音合成

對於需要即時回饋的應用場景，可以使用串流語音合成：

```javascript
// 啟用串流輸出
const stream = await client.speakStream('這是一段很長的文字');

// 監聽資料區塊
stream.on('data', (chunk) => {
  // 播放音訊區塊
  audioPlayer.play(chunk);
});

stream.on('end', () => {
  console.log('語音合成完成');
});

stream.on('error', (error) => {
  console.error('錯誤:', error);
});
```

### 自定義聲音模型

如果需要使用品牌專屬的聲音，可以訓練自定義模型：

```javascript
// 準備訓練資料
const trainingData = {
  name: '品牌聲音',
  description: '用於企業品牌的聲音',
  samples: [
    { text: '樣本文本1', audio: 'base64編碼的音訊1' },
    { text: '樣本文本2', audio: 'base64編碼的音訊2' },
    // 更多樣本...
  ]
};

// 提交訓練任務
const job = await client.trainVoice(trainingData);

// 輪詢訓練狀態
while (job.status !== 'completed') {
  await sleep(5000);
  const status = await client.getJobStatus(job.id);
  console.log(`訓練進度: ${status.progress}%`);
}

// 訓練完成，使用自定義聲音
const audio = await client.speak('使用自定義聲音', {
  voice: {
    modelId: job.modelId
  }
});
```

### 音訊後處理

FantasyTalking 提供了多種音訊後處理功能：

```javascript
const audio = await client.speak('原始語音');

// 音量調整
const normalized = client.process(audio, {
  normalize: true,
  targetLevel: -20  // dB
});

// 降噪
const denoised = client.process(audio, {
  denoise: true,
  noiseThreshold: 0.5
});

// 語速調整
const spedUp = client.process(audio, {
  speed: 1.5  // 1.5倍速
});
```

## 最佳實踐

### 效能優化

在實際應用中，需要注意以下幾點來優化效能：

1. **連線複用**：避免頻繁建立客戶端實例，複用連線可以顯著降低延遲
2. **批量處理**：需要處理大量文字時，使用批量合成功能
3. **快取機制**：對於相同的文字，可以使用快取避免重複合成
4. **串流輸出**：長文字使用串流輸出可以提前開始播放，改善用戶體驗

### 錯誤處理

建議實現完善的錯誤處理機制：

```javascript
try {
  const audio = await client.speak('測試文字');
} catch (error) {
  if (error.code === 'RATE_LIMIT') {
    // 處理速率限制
    console.log('請求過於頻繁，請稍後重試');
  } else if (error.code === 'INVALID_API_KEY') {
    // 處理認證錯誤
    console.log('API 密鑰無效');
  } else {
    // 其他錯誤
    console.error('未知錯誤:', error);
  }
}
```

### 用戶體驗

為了提供更好的用戶體驗，建議：

1. **預先載入**：在用戶可能需要語音前提前合成
2. **回饋提示**：語音播放時提供視覺回饋
3. **中斷處理**：支援用戶中斷語音播放
4. **離線支援**：實現基礎功能的離線支援
