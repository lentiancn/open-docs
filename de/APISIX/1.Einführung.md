# Einführung

Apache APISIX ist ein Top-Level-Projekt der Apache Software Foundation und wurde von API7.ai entwickelt und gespendet. Es handelt sich um einen Cloud-nativen API-Gateway mit dynamischen, Echtzeit- und Hochleistungseigenschaften.

APISIX fungiert als Eingangstor für den gesamten Datenverkehr und bietet eine Fülle von Funktionen: dynamisches Routing, dynamische Upstreams, dynamische Zertifikate, A/B-Tests, Canary-Releases, Blue-Green-Deployments, Ratenbegrenzung, Angriffsschutz, Metrikerfassung, Monitoring-Warnungen, Observability und Service-Governance.

## Kernfunktionen

### Hohe Leistung

APISIX basiert auf Nginx und erbt dessen hervorragende Leistung. Durch das Plugin-System werden Geschäftslogik und Kernfunktionen entkoppelt, was flexible Erweiterungen ermöglicht und gleichzeitig eine hohe Verarbeitungsgeschwindigkeit bewahrt.

### Vollständig dynamisch

Im Gegensatz zu herkömmlichen Nginx, das eine Neukonfiguration erfordert, unterstützt APISIX eine vollständig dynamische Konfiguration. Alle Routen, Upstreams und Plugins können über die Admin-API in Echtzeit aktualisiert werden – ein Neustart ist nicht erforderlich.

### Cloud-natives Design

APISIX wurde von Grund auf für Cloud-native Szenarien entwickelt und unterstützt verschiedene Deployments, darunter Docker, Kubernetes und Helm Chart. Der integrierte Ingress Controller kann direkt als Eingangsgateway für Kubernetes verwendet werden.

### Plugin-Ökosystem

APISIX verfügt über ein umfangreiches Plugin-System, das Authentifizierung, Ratenbegrenzung, Request-Umschreibung, Response-Verarbeitung, Monitoring und Logging abdeckt. Das Plugin-System ermöglicht eine einfache Erweiterung der Gateway-Funktionalität.

### Multi-Protokoll-Unterstützung

Neben den traditionellen HTTP/HTTPS-Protokollen unterstützt APISIX auch gRPC, Dubbo, WebSocket und MQTT – ideal für verschiedene Szenarien in Microservice-Architekturen.

## Architekturüberblick

Die APISIX-Architektur besteht aus zwei Teilen: Data Plane und Control Plane.

Die **Data Plane** ist für die eigentliche Request-Weiterleitung zuständig. Das auf Nginx basierende APISIX Core ermöglicht eine hochleistungsfähige Request-Verarbeitung.

Die **Control Plane** verwaltet alle Konfigurationen über eine RESTful API und unterstützt verschiedene Persistenzmethoden wie etcd oder Consul. APISIX bietet auch ein Dashboard für die visuelle Konfiguration über eine Web-Oberfläche.

## Anwendungsfälle

- Zentraler Eingang für Microservice-APIs
- Kubernetes Ingress Controller
- API-Verkehrsmanagement und Monitoring
- Multi-Tenant-API-Verwaltung
- Service Mesh Sidecar-Proxy
- AI-Anwendungs-Gateway (LLM und AI-Workloads)

Als Top-Level-Projekt der Apache Software Foundation verfügt APISIX über eine aktive Community und kontinuierliche Versionen – eine zuverlässige Wahl für Enterprise-Anwendungen.
