# PlantUML 사용 가이드

이 가이드에서는 PlantUML을 사용하여 다양한 UML 다이어그램을 생성하는 방법을 설명합니다.

## 빠른 시작

### 기본 문법

```plantuml
@startuml
Alice -> Bob: Hello
@enduml
```

## 시퀀스 다이어그램

```plantuml
@startuml
actor 사용자
participant "프론트엔드" as Front
participant "백엔드" as Back
database "데이터베이스" as DB

사용자 -> Front: 요청 제출
Front -> Back: API 호출
Back -> DB: 데이터 조회
DB --> Back: 결과 반환
Back --> Front: 응답
Front --> 사용자: 결과 표시
@enduml
```

## 클래스 다이어그램

```plantuml
@startuml
class User {
  +id: int
  +username: string
  +email: string
  +login()
  +logout()
}

class Admin extends User {
  +manageUsers()
  +viewReports()
}

class Order {
  +id: int
  +total: decimal
  +createOrder()
  +cancelOrder()
}

User "1" -- "*" Order: places
@enduml
```

## 유스케이스 다이어그램

```plantuml
@startuml
left to right direction
actor 사용자
rectangle 시스템 {
  사용자 -> (로그인)
  사용자 -> (상품 浏览)
  사용자 -> (장바구니 추가)
  사용자 -> (주문)
  (로그인) .> (상품 浏览): extends
}
@enduml
```

## 상태 다이어그램

```plantuml
@startuml
[*] --> 결제 대기
결제 대기 --> 결제 완료: 결제 성공
결제 완료 --> 배송 중: 배송
배송 중 --> 배송 완료: 배송 도착
배송 완료 --> [*]
결제 대기 --> 취소: 사용자 취소
취소 --> [*]
@enduml
```

## 액티비티 다이어그램

```plantuml
@startuml
start
:쇼핑 시작;
:상품 浏览;
if (상품을 찾았나요?) then (예)
  :장바구니 추가;
else (아니오)
  :상품 검색;
endif
:결제;
if (결제 성공?) then (예)
  :주문 완료;
else (아니오)
  :결제 실패;
endif
stop
@enduml
```

## 컴포넌트 다이어그램

```plantuml
@startuml
package "프론트엔드" {
  [Web App] as WA
  [Mobile App] as MA
}
package "백엔드" {
  [API Gateway] as API
  [Auth Service] as Auth
  [Order Service] as Order
}
package "데이터" {
  database "MySQL" as DB
  database "Redis" as Cache
}

WA --> API
MA --> API
API --> Auth
API --> Order
Order --> DB
Order --> Cache
@enduml
```

## 배포 다이어그램

```plantuml
@startuml
node "사용자 디바이스" as Client {
  [브라우저]
}
node "로드밸런서" as LB {
  [Nginx]
}
node "애플리케이션 서버" as App {
  [Tomcat 1]
  [Tomcat 2]
}
node "데이터베이스 서버" as DB {
  [MySQL]
  [Redis]
}

Client --> LB
LB --> App
App --> DB
@enduml
```

## 타이밍 다이어그램

```plantuml
@startuml
clock "클록" as C with period 1000
binary "동기화" as SY
binary "데이터" as DT

C is idle
SY is low
DT is low

@1000
SY is high
@1500
SY is low
@2000
DT is high
@3000
DT is low
@enduml
```

```plantuml
[테스트] starts at 2024-01-16
[배포] starts at 2024-01-21
@enduml
```

## 일반적인 명령어

### 요소 숨기기

```plantuml
@startuml
hide footbox
actor A
participant B
A -> B: 메시지
@enduml
```

### 스타일 설정

```plantuml
@startuml
skinparam backgroundColor #EEEBDC
skinparam handwritten true
skinparam actorStyle awesome

actor User
User -> (로그인)
@enduml
```
