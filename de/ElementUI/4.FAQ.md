# Element UI FAQ

Häufig gestellte Fragen und Antworten.

## Installationsprobleme

### Q: Installation fehlgeschlagen?

**Lösungen:**
1. Node.js-Version überprüfen
```bash
node --version  # Erfordert 12.0.0+
```
2. npm-Cache leeren
```bash
npm cache clean --force
```
3. Taobao-Spiegel verwenden
```bash
npm config set registry https://registry.npmmirror.com
```
4. node_modules löschen und neu installieren
```bash
rm -rf node_modules
npm install
```

### Q: Modul nicht gefunden?

**Lösungen:**
1. Bestätigen, dass element-ui installiert ist
```bash
npm list element-ui
```
2. package.json auf Abhängigkeiten prüfen

### Q: Vue-Version inkompatibel?

**Lösungen:**
Element UI 2.x erfordert Vue 2.6.0+:
```bash
npm install vue@2
```

## Verwendungsprobleme

### Q: Komponenten werden nicht angezeigt?

**Lösungen:**
1. Bestätigen, dass Komponenten korrekt importiert wurden
```javascript
import { Button } from 'element-ui'
Vue.use(Button)
```
2. Konsole auf Fehler prüfen
3. Komponentennamen überprüfen

### Q: Styles werden nicht angewendet?

**Lösungen:**
1. Bestätigen, dass Style-Datei importiert wurde
```javascript
import 'element-ui/lib/theme-chalk/index.css'
```
2. Auf Style-Überschreibungen prüfen
3. Entwicklertools verwenden, um Element-Styles zu prüfen

### Q: Tabelle zeigt keine Daten?

**Lösungen:**
1. Datenformat prüfen
2. Spaltendefinition überprüfen
```html
<el-table-column prop="name" label="Name"></el-table-column>
```
3. Datenreaktivität prüfen

### Q: Formularvalidierung funktioniert nicht?

**Lösungen:**
1. Form und FormItem importieren
2. prop-Attribut setzen
```html
<el-form-item label="Name" prop="name">
  <el-input v-model="form.name"></el-input>
</el-form-item>
```
3. Regeln in data definieren

### Q: Dropdown-Optionen werden nicht angezeigt?

**Lösungen:**
1. options-Daten prüfen
2. value und label korrekt zuordnen
```html
<el-option
  :value="item.value"
  :label="item.label">
</el-option>
```

### Q: Dialog-Daten werden nach dem Schließen nicht zurückgesetzt?

**Lösungen:**
1. Dialog mit v-if steuern
```html
<el-dialog :visible.sync="dialogVisible" v-if="dialogVisible">
```
2. Oder Formular beim Schließen manuell zurücksetzen
```javascript
this.$refs.form.resetFields()
```

## Style-Probleme

### Q: Style-Konflikte?

**Lösungen:**
1. Spezifischere Selektoren verwenden
2. z-index-Ebenen prüfen
3. !important zum Überschreiben verwenden (nicht empfohlen)

### Q: Icons werden nicht angezeigt?

**Lösungen:**
1. Bestätigen, dass Icon-Bibliothek installiert ist
```bash
npm install element-ui-icons
```
2. Korrekt importieren
```javascript
import 'element-ui-icons'
```

### Q: Theme-Farbe nicht korrekt?

**Lösungen:**
1. Auf globale Style-Überschreibungen prüfen
2. SCSS-Variablen-Importreihenfolge prüfen
3. Browser-Cache leeren

## Internationalisierungsprobleme

### Q: Sprachwechsel funktioniert nicht?

**Lösungen:**
1. i18n korrekt konfigurieren
```javascript
Vue.use(ElementUI, {
  i18n: (key, value) => i18n.t(key, value)
})
```
2. Sprachpaket importieren

### Q: Einige Texte werden nicht übersetzt?

**Lösungen:**
1. Vollständiges Sprachpaket prüfen
2. ElementUI locales-Konfiguration verwenden

## Leistungsprobleme

### Q: Seite lädt langsam?

**Lösungen:**
1. On-Demand-Import verwenden
2. CDN-Einbindung verwenden
3. Gzip-Komprimierung aktivieren

### Q: Große Datenmengen in Tabellen causing Lag?

**Lösungen:**
1. Virtuelles Scrollen verwenden
2. Daten seitenweise laden
3. Unnötige Features deaktivieren

## Build-Probleme

### Q: Nach Build gehen Styles verloren?

**Lösungen:**
1. vue.config.js-Konfiguration prüfen
2. Style-Importposition überprüfen

### Q: Komponenten in Produktion werden nicht angezeigt?

**Lösungen:**
1. Vue.use()-Aufruf prüfen
2. On-Demand-Import vollständig prüfen

## Upgrade-Probleme

### Q: Wie von 1.x auf 2.x upgraden?

**Lösungen:**
1. Offiziellen Migrationsleitfaden lesen
2. Inkonsistente APIs schrittweise ersetzen
3. Tests ausführen

### Q: Vue 3-Projekt - welche Option?

Element UI entspricht Vue 2. Für Vue 3projekte Element Plus verwenden:
```bash
npm install element-plus
```
