# Docker FAQ

Häufig gestellte Fragen.

## Installation

### Q: Docker Desktop startet nicht unter Windows?

**Lösungen**isierung im BIOS aktiv:
1. Virtualieren
2. WSL 2 installieren/aktualisieren
3. Computer neu starten
4. Auf konfliktierende Software prüfen (VirtualBox, VMware)

### Q: "docker: permission denied" Fehler?

**Lösung**:
```
sudo usermod -aG docker $USER
# Abmelden und wieder anmelden
```

### Q: Welche Docker Version habe ich?

```
docker --version
docker-compose --version
```

## Images

### Q: Wie verkleinere ich Docker Image?

**Best Practices**:
1. Alpine-basierte Images verwenden
2. Mehrstufige Builds verwenden
3. Schichten minimieren
4. In derselben Schicht aufräumen

### Q: Wie baue ich Image ohne Cache?

```
docker build --no-cache -t my-image .
```

### Q: Wie lösche ich unbenutzte Images?

```
docker image prune -a
```

## Container

### Q: Wie halte ich Container am Laufen?

```
docker run -d nginx tail -f /dev/null
# oder
docker run -d nginx sleep infinity
```

### Q: Container beendet sofort nach Start?

**Gründe**:
1. Kein Vordergrundprozess
2. Prozess beendet

**Lösungen**:
```
docker run -it image_name /bin/bash
docker run -d image_name tail -f /dev/null
```

### Q: Wie steige ich in laufenden Container ein?

```
docker exec -it container_id /bin/bash
```

### Q: Wie kopiere ich Dateien zu/von Container?

```
# Von Container zu Host kopieren
docker cp container_id:/path/file.txt ./

# Von Host zu Container kopieren
docker cp ./file.txt container_id:/path/
```

## Netzwerk

### Q: Wie greife ich auf Container vom Host zu?

Port-Mapping verwenden:
```
docker run -p 8080:80 nginx
```

### Q: Wie verbinde ich Container?

Netzwerk erstellen:
```
docker network create my-network
docker run --network my-network -d nginx
```

## Volumes

### Q: Wie persistentiere ich Daten?

Volumes verwenden:
```
docker volume create my-data
docker run -v my-data:/data nginx
```

## Docker Compose

### Q: Wie baue ich nach Codeänderungen neu?

```
docker-compose up -d --build
```

### Q: Wie sehe ich Logs?

```
docker-compose logs -f
```

### Q: Wie skaliere ich Services?

```
docker-compose up -d --scale web=3
```

## Performance

### Q: Docker ist langsam / belegt zu viel Speicher?

**Lösungen**:
```
docker system prune -a
docker system df
docker volume prune
```

### Q: Wie begrenze ich Container-Ressourcen?

```
docker run -m 512m nginx
docker run --cpus=0.5 nginx
```

## Fehlersuche

### Q: Wie debugge ich fehlgeschlagenen Container?

```
docker logs container_id
docker inspect container_id
```

### Q: Container-Netzwerk funktioniert nicht?

```
sudo systemctl restart docker
docker network ls
```

## Sicherheit

### Q: Wie scanne ich auf Schwachstellen?

```
docker scan my-image
```

### Q: Sollte ich als root laufen?

Nein, erstelle einen Nicht-Root-Benutzer:

```
FROM node:18-alpine
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -D appuser
USER appuser
```

## Docker vs VM

### Q: Was ist der Unterschied zwischen Docker und VM?

| Aspekt | Docker | VM |
|--------|--------|-----|
| OS | Teilt Host-Kernel | Eigenes OS |
| Größe | MBs | GBs |
| Start | Sekunden | Minuten |
| Isolation | Container | Volle Isolation |
