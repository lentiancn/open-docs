# Mockito インストールガイド

## 動作環境

Mockitoをインストールする前に、開発環境が以下の要件を満たしていることを確認してください。

Mockito 5.xにはJDK 11以上が必要です。プロジェクトがJDK 8をサポートする必要がある場合は、Mockito 4.xシリーズを使用できます。最新バージョンのMockitoを使用して、最新の機能とセキュリティ修正を取得することをお勧めします。

## Maven インストール

### 依存関係の追加

Mavenプロジェクトのpom.xmlファイルにmockito-core依存関係を追加します。Mockitoバージョンを定義するためにプロパティを使用することをお勧めします。

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Mockitoのインライン機能を使用してfinalクラスやメソッドをモックする必要がある場合は、mockito-inline依存関係も追加する必要があります。

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### スナップショットバージョン

Mockitoは每次のコード変更をSonatypeリポジトリにスナップショットバージョンとして公開します。最新開発中の機能を使用する必要がある場合は、pom.xmlにスナップショットリポジトリを追加できます。

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Gradle インストール

### 依存関係の追加

Gradleプロジェクトのbuild.gradleファイルに依存関係宣言を追加します。

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

GradleのKotlin DSLファイル（build.gradle.kts）を使用する場合は構文が少し異なります。

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### BOM を使用したバージョン管理

Spring Dependency ManagementプラグインまたはGradleのBOM機能を使用すると、バージョン管理を簡素化できます。

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## 手動ダウンロード

プロジェクトがMavenまたはGradleを使用していない場合は、Maven CentralリポジトリからJARファイルを直接ダウンロードできます。Maven Centralにアクセスしてorg.mockitoを検索すると、mockito-coreとmockito-all（廃止済み）のすべてのバージョンが見つかります。

ダウンロード後、JARファイルをプロジェクトのクラスパスに追加する必要があります。Byte BuddyやObjenesisなどの依存ライブラリもダウンロードする必要があります。

## Spring Boot との統合

Spring Bootのspring-boot-starter-testにはすでにMockito依存関係が含まれています。プロジェクトがSpring Bootを使用している場合、一般的にはMockito依存関係を別途追加する必要はありません。

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Spring環境でより便利にMockを使用する必要がある場合は、@MockBeanアノテーションを使用できます。これはモックオブジェクトをSpringコンテナに自動的に注入します。

## JUnit 5 との統合

### MockitoExtension の使用

JUnit 5は拡張モデルを導入し、MockitoはJUnit 5をサポートするためにMockitoExtensionを提供します。追加の依存関係設定は不要で、mockito-coreにはすでに必要なサポートが含まれています。

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### MockitoJUnitRunner の使用

JUnit 4の場合は、MockitoJUnitRunnerを使用してMockオブジェクトを自動的に初期化できます。

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## インストールの確認

インストール後、以下の方法でMockitoが正しく構成されていることを確認できます。シンプルなテストクラスを作成し、テストが通ればMockitoが正しくインストールされています。

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockitoのインストール成功！");
    }
}
```

## バージョン選択の推奨

新規プロジェクトの場合は、最新Mockito 5.xバージョンを使用することをお勧めします。プロジェクトがJDK 8互換性が必要な場合は、Mockito 4.xシリーズが利用可能です。常に最新の安定バージョンを使用して、より良いパフォーマンスとセキュリティを確保してください。
