# JMeter 자주 묻는 질문

이 섹션에서는 Apache JMeter 사용 시 일반적인 질문과 솔루션을 모았습니다.

## 설치 관련 문제

### Q1: JMeter 실행 시 "java"가 내부 또는 외부 명령으로 인식되지 않습니다

**A**: Java가 올바르게 설치되어 있고 구성되어 있는지 확인하세요. 명령줄에서 확인:
```bash
java -version
```
설치되어 있지 않으면 Java JDK 8 이상을 다운로드하여 설치하세요. Windows의 경우 JAVA_HOME 환경 변수도 설정해야 합니다.

### Q2: JMeter 인터페이스가 깨진 문자를 표시합니다

**A**: 글꼴 인코딩 문제로 인한 것입니다. 해결 방법:
1. bin/jmeter.properties 파일 열기
2. 추가 또는 수정:
   ```
   sampleresult.default.encoding=UTF-8
   ```
3. JMeter 재시작

### Q3: JMeter 메모리가 부족하여 충돌합니다

**A**: 기본 힙 크기가 부족할 수 있습니다. 조정:
- Windows: bin/jmeter.bat 편집, set HEAP 매개변수를 찾아 증가
- Linux/Mac: bin/jmeter 편집, HEAP 변수 조정
2GB 이상 권장:
```bash
HEAP="-Xms2g -Xmx2g"
```

## 테스트 실행 문제

### Q4: 여러 다른 사용자 로그인을 시뮬레이션하려면 어떻게 해야 하나요?

**A**: 매개변수화를 사용하세요:
1. 사용자 이름과 비밀번호가 포함된 CSV 파일 만들기
2. "CSV 데이터 세트 구성" 요소 추가
3. HTTP 요청에서 ${username} 및 ${password} 변수 사용

### Q5: JMeter가 Session 또는 Cookie를 추출할 수 없습니다

**A**: 적절한 요소가 추가되었는지 확인:
1. 스레드 그룹에 "HTTP 쿠키 관리자" 추가
2. 로그인이 필요한 테스트의 경우 "반복 시 쿠키 지우기"가 올바르게 구성되어 있는지 확인
3. 요청이 올바르게 Cookie를携带하고 있는지 확인

### Q6: 응답 데이터의 한자가 깨져서 표시됩니다

**A**: 해결 방법:
1. HTTP 요청에서 "콘텐츠 인코딩"을 UTF-8로 추가 또는 수정
2. "HTTP 요청 기본값"에서 기본 인코딩 설정
3. 후처리기에서 "정규식 추출기"를 사용하고 인코딩 지정

### Q7: HTTPS 인증서 문제를 처리하려면 어떻게 해야 하나요?

**A**: 테스트 환경의 경우:
1. 신뢰할 수 있는 인증서를 Java cacerts 저장소로 가져오기
2. 또는 HTTP 요청에서 "구현"을 Java로 설정
3. 자체 서명 인증서의 경우 JMeter truststore를 가져와야 할 수 있음

### Q8: 장기간 테스트 실행 중 메모리가 계속 증가합니다

**A**: 권장 조치:
1. 리스너가 수집하는 데이터 양 줄이기
2. 명령줄 모드에서 테스트 실행
3. jmeter.properties에서 가비지 컬렉션 빈도 증가:
   ```
   mode=Stripped
   ```
4. 결과 파일 정기적으로 정리

## 성능 문제

### Q9: JMeter가 충분한 부하를 생성할 수 없습니다

**A**: 다음 방법을 시도해 보세요:
1. 분산 테스트 사용(원격 테스트)
2. 스레드 수 및 반복 횟수 조정
3. 불필요한 리스너 줄이기
4. GUI 모드 대신 명령줄 모드 사용
5. 네트워크 대역폭이 병목인지 확인

### Q10: 테스트 결과에서 응답 시간이 크게 변동합니다

**A**: 가능한 원인 및 해결 방법:
1. 서버 측 GC 또는 캐싱 문제
2. 불안정한 네트워크
3. 테스트 데이터에 종속성이 있음
4. 동시 요청을 보장하기 위해 동기화 타이머 사용
5. 통계 수집 전에 시스템을 안정화하기 위해 예열 시간 증가

### Q11: 합리적인 동시성을 어떻게 설정하나요?

**A**: 권장 접근 방식:
1. 낮은 동시성으로 시작(10-50)
2. 동시성을 점진적으로 증가시키고 시스템 응답 관찰
3. 성능 전환점 찾기(응답 시간이明显히 증가하기 시작)
4. 최대 안전 동시성 결정
5. 시스템 한계를 검증하기 위해 스트레스 테스트 수행

## 보고서 및 결과 분석

### Q12: HTML 보고서를 생성하려면 어떻게 해야 하나요?

**A**: 명령줄 사용:
```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/report
```
또는 기존 결과 파일에서 생성:
```bash
jmeter -g results.jtl -o /path/to/report
```

### Q13: 집계 보고서의 지표 의미는?

- **샘플**: 요청 총 수
- **평균**: 밀리초 단위 평균 응답 시간
- **중앙값**: 중앙 응답 시간
- **90%/95%/99% 백분위수**: 90%/95%/99%의 요청이 이 값보다 빠르게 응답
- **최소/최대**: 최소 및 최대 응답 시간
- **오류 %**: 실패한 요청의 백분율
- **처리량**: 초당 처리된 요청 수

### Q14: 결과 파일이 너무 큽니다

**A**: 해결 방법:
1. 리스너에서 CSV 형식으로 저장
2 필요한 필드만 저장하도록 구성
3. "분산 테스트"를 사용하여 저장 분산
4. 결과 파일 정기적으로 순환

## 스크립팅 문제

### Q15: 동적으로 변경되는 매개변수를 처리하려면 어떻게 해야 하나요?

**A**: 일반적인 방법:
1. 응답에서 추출하기 위해 "정규식 추출기" 사용
2. JSON 데이터를 추출하기 위해 "JSON 추출기" 사용
3. HTML 요소를 추출하기 위해 "CSS/JQuery 추출기" 사용
4. XML 데이터를 추출하기 위해 "XPath 추출기" 사용

### Q16: BeanShell 스크립트를 실행할 수 없습니다

**A**: 다음을 확인:
1. JDK가 올바르게 설치됨(JRE 아님)
2. BeanShell 라이브러리 파일이 lib 디렉토리에 있음
3. 스크립트 구문이 올바른지 확인

### Q17: 복잡한 비즈니스 로직을 구현하려면 어떻게 해야 하나요?

**A**: 다음을 사용할 수 있음:
1. 논리 컨트롤러 조합
2. BeanShell 또는 JSR223 스크립트
3. 전/후처리기
4. 조건부 실행을 위한 처리량 컨트롤러

## 기타 문제

### Q18: JMeter와 브라우저의 차이점은?

JMeter는 프로토콜 수준에서 작동하며 브라우저가 아닙니다:
- JavaScript를 실행하지 않음
- HTML 페이지를 렌더링하지 않음
- CSS를 처리하지 않음
- 브라우저의 모든 동작을 시뮬레이션할 수 없음

### Q19: JMeter를 배우려면 어떻게 해야 하나요?

권장 학습 경로:
1. JMeter 공식 사용자 매뉴얼 읽기
2. 간단한 HTTP 요청으로 시작
3. 매개변수화 및 상관관계 배우기
4. 분산 테스트 마스터
5. 사용자 정의 스크립트 개발 배우기

### Q20: JMeter는 어떤 운영 체제를 지원하나요?

JMeter는 순수 Java 애플리케이션이며 Java가 설치된 모든 운영 체제를 지원합니다. Windows, Linux, Mac OSX, Solaris 등을 포함합니다.
