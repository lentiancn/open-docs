# Guide d'installation d'Oracle Database

Oracle Database est le système de gestion de base de données relationnelle leader du marché, développé par Oracle. Ce guide couvre les méthodes d'installation dans divers environnements.

## Informations de version

| Version | Statut | Description |
|------|------|------|
| Oracle 21c | ✅ Actuelle | Dernière version |
| Oracle 19c | ✅ LTS | Version de support à long terme |
| Oracle 18c | ❌ Obsolète | Support terminé |

> **Important**: Ce guide concerne principalement **Oracle 19c/21c**.

## Configuration requise

### Configuration matérielle

| Exigence | Configuration minimale | Configuration recommandée |
|------|----------|----------|
| CPU | 2 cœurs | 4+ cœurs |
| Mémoire | 2 Go | 8+ Go |
| Disque | 40 Go | 100+ Go |

### Systèmes d'exploitation pris en charge

- **Linux**: Oracle Linux, RHEL, CentOS, SLES, Ubuntu
- **Windows**: Windows Server 2016+

## Méthodes d'installation

### Méthode 1 : Installation Linux

#### 1. Préparer le système

```bash
# Créer des groupes d'utilisateurs
groupadd oinstall
groupadd dba
groupadd oper
groupadd backupdba
groupadd dgdba
groupadd kmdba
groupadd racdba
groupadd asmdba
groupadd asmoper
groupadd asmadmin

# Créer l'utilisateur Oracle
useradd -g oinstall -G dba,oper,backupdba,dgdba,kmdba,racdba,asmdba,asmoper,asmadmin oracle

# Définir le mot de passe
passwd oracle

# Créer les répertoires
mkdir -p /u01/app/oracle/product/19.3.0/dbhome_1
mkdir -p /u01/app/oraInventory
mkdir -p /u01/app/oracle

# Définir les permissions
chown -R oracle:oinstall /u01
chmod -R 775 /u01
```

#### 2. Configurer les paramètres du noyau

```bash
# /etc/sysctl.conf
cat >> /etc/sysctl.conf << EOF
kernel.shmmax = 68719476736
kernel.shmmni = 4096
kernel.shmall = 2097152
kernel.sem = 250 32000 100 128
fs.file-max = 6815744
net.ipv4.ip_local_port_range = 9000 65500
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 1048576
fs.aio-max-nr = 1048576
EOF

# Appliquer
sysctl -p

# /etc/security/limits.conf
cat >> /etc/security/limits.conf << EOF
oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536
oracle soft stack 10240
oracle hard stack 32768
EOF
```

#### 3. Installer les dépendances

```bash
# Oracle Linux
yum install -y libaio libaio-devel libXi libXtst libXext

# Ubuntu
apt-get install -y libaio1 libaio-dev libxi6 libxtst6 libxext6
```

#### 4. Exécuter l'installateur

```bash
# Extraire les fichiers d'installation
unzip -q LINUX.X64_193000_db_home.zip -d /u01/app/oracle/product/19.3.0/dbhome_1/

# Exécuter en tant qu'utilisateur oracle
cd /u01/app/oracle/product/19.3.0/dbhome_1
./runInstaller
```

### Méthode 2 : Installation Windows

1. Télécharger l'installateur Oracle
2. Exécuter setup.exe
3. Suivre l'assistant d'installation :
   - Sélectionner « Créer et configurer une base de données à instance unique »
   - Sélectionner « Classe bureau » ou « Classe serveur »
   - Définir les mots de passe sys et system
   - Terminer l'installation

### Méthode 3 : Utiliser Docker

```bash
# Télécharger l'image
docker pull container-registry.oracle.com/database/express:21.3.0-xe

# Exécuter le conteneur
docker run -d \
  --name oracle \
  -p 1521:1521 \
  -e ORACLE_PWD=MyPassword123 \
  -v oracle_data:/opt/oracle/oradata \
  container-registry.oracle.com/database/express:21.3.0-xe
```

## Configuration post-installation

### Variables d'environnement

```bash
# ~/.bash_profile
export ORACLE_HOME=/u01/app/oracle/product/19.3.0/dbhome_1
export ORACLE_SID=orcl
export PATH=$ORACLE_HOME/bin:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:$LD_LIBRARY_PATH
```

### Démarrer la base de données

```bash
# Démarrer le listener
lsnrctl start

# Démarrer la base de données
sqlplus / as sysdba
STARTUP;
```

## Problèmes courants

### Mémoire insuffisante ?

```bash
# Vérifier la mémoire
grep MemTotal /proc/meminfo

# Ajuster le swap
```

### Échec de l'installation ?

Vérifier les journaux :
```bash
cat /u01/app/oraInventory/logs/installActions*.log
```
