# JUnit 사용 매뉴얼

이 매뉴얼은 JUnit 4와 JUnit 5의 사용 방법, 테스트 작성, 단언, 테스트 스위트 등 핵심 기능을 다룹니다.

## JUnit 5 기본 사용법

### 테스트 클래스 작성

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void testAddition() {
        int result = 2 + 3;
        assertEquals(5, result);
    }
}
```

### 일반적인 단언

```java
import static org.junit.jupiter.api.Assertions.*;

// 동등성 단언
assertEquals(expected, actual);
assertEquals(expected, actual, "사용자 정의 오류 메시지");

// 부울 단언
assertTrue(condition);
assertFalse(condition);

// Null 단언
assertNull(object);
assertNotNull(object);

// 배열 단언
assertArrayEquals(expectedArray, actualArray);

// 예외 단언
assertThrows(Exception.class, () -> {
    // 예외를 발생시킬 수 있는 코드
});

// 조합 단언
assertAll("Person 속성",
    () -> assertEquals("John", person.getFirstName()),
    () -> assertEquals("Doe", person.getLastName())
);
```

### Before 및 After 작업

```java
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;

class MyTest {
    
    @BeforeEach
    void setUp() {
        // 각 테스트 메서드 실행 전에 실행
    }
    
    @AfterEach
    void tearDown() {
        // 각 테스트 메서드 실행 후에 실행
    }
}
```

### 중첩 테스트

```java
import org.junit.jupiter.api.Nested;

@Nested
class NestedTests {
    
    @Test
    void testInNestedClass() {
        // 중첩 테스트 클래스의 테스트
    }
}
```

### 매개변수화 테스트

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void testWithMultipleValues(int number) {
    assertTrue(number > 0);
}
```

### 동적 테스트

```java
import org.junit.jupiter.api.TestFactory;
import org.junit.jupiter.api.DynamicTest;
import java.util.Arrays;

@TestFactory
Collection<DynamicTest> dynamicTestsFromIterator() {
    return Arrays.asList(
        DynamicTest.dynamicTest("첫 번째 동적 테스트", () -> assertTrue(true)),
        DynamicTest.dynamicTest("두 번째 동적 테스트", () -> assertEquals(4, 2 + 2))
    );
}
```

## JUnit 4 기본 사용법

### 테스트 클래스 작성

```java
import org.junit.Test;
import static org.junit.Assert.*;

public class CalculatorTest {
    
    @Test
    public void testAddition() {
        assertEquals(5, 2 + 3);
    }
}
```

### 일반적인 단언

```java
// 동등성 단언
assertEquals(expected, actual);
assertEquals("메시지", expected, actual);

// 부울 단언
assertTrue(condition);
assertFalse(condition);

// Null 단언
assertNull(object);
assertNotNull(object);

// 예외 단언
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}
```

### Before 및 After

```java
import org.junit.Before;
import org.junit.After;

public class MyTest {
    
    @Before
    public void setUp() {
        // 각 테스트 메서드 실행 전에 실행
    }
    
    @After
    public void tearDown() {
        // 각 테스트 메서드 실행 후에 실행
    }
    
    @BeforeClass
    public static void setUpClass() {
        // 클래스 내 모든 테스트 전에 한 번 실행
    }
    
    @AfterClass
    public static void tearDownClass() {
        // 클래스 내 모든 테스트 후에 한 번 실행
    }
}
```

### 테스트 스위트

```java
import org.junit.runner.RunWith;
import org.junit.runners.Suite;
import org.junit.runners.Suite.SuiteClasses;

@RunWith(Suite.class)
@SuiteClasses({Test1.class, Test2.class, Test3.class})
public class AllTests {
    // Test1, Test2, Test3의 모든 테스트 실행
}
```

## JUnit 4에서 JUnit 5로 마이그레이션

### 애노테이션 변경

| JUnit 4 | JUnit 5 |
|---------|---------|
| @Test | @Test |
| @Before | @BeforeEach |
| @After | @AfterEach |
| @BeforeClass | @BeforeAll |
| @AfterClass | @AfterAll |
| @Ignore | @Disabled |
| @SuiteClasses | @SelectClasses |

### 단언 변경

```java
// JUnit 4
Assert.assertEquals(expected, actual);

// JUnit 5
Assertions.assertEquals(expected, actual);
```

### 예외 테스트 변경

```java
// JUnit 4
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}

// JUnit 5
@Test
public void testException() {
    assertThrows(NullPointerException.class, () -> {
        Object obj = null;
        obj.toString();
    });
}
```

## 테스트 실행

### 명령줄

```bash
# Maven
mvn test

# Gradle
gradle test

# 독립형 콘솔
java -jar junit-platform-console-standalone-5.14.3.jar -c MyTestClass
```

### IDE에서 실행

- IntelliJ IDEA: 테스트 클래스 또는 메서드를 마우스 오른쪽 버튼으로 클릭하고 Run 선택
- Eclipse: 테스트 클래스 또는 메서드를 마우스 오른쪽 버튼으로 클릭하고 Run As → JUnit Test 선택
- VS Code: JUnit 확장 프로그램을 사용하여 테스트 실행
