# Element Plus 安装指南

> Element Plus 是由 Element 团队打造的 Vue 3 桌面端组件库
> 
> 更新日期：2026年2月
> 
> 当前版本：2.x（稳定版）

---

## 一、项目简介

Element Plus 是一个基于 Vue 3 的桌面端 UI 组件库，由 Element 团队开发和维护。作为 Element UI 的 Vue 3 升级版本，Element Plus 完全遵循 Vue 3 的 Composition API 风格编写，并全面采用 TypeScript 重构，提供了类型安全的组件开发体验。

Element Plus 首次正式稳定版于 2022 年 2 月 7 日发布，经过多年迭代，API 已经非常稳定。目前支持简体中文、英文等多种语言，并且拥有活跃的社区生态。

## 二、环境要求

### 2.1 开发环境要求

| 环境 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Node.js | 16.x | 18.x 或 20.x |
| npm | 8.x | 9.x 或 10.x |
| Vue | 3.2.0 | 3.4.x |
| Vite | 4.x | 5.x |

### 2.2 浏览器兼容性

Element Plus 支持所有现代浏览器，包括：

- Chrome >= 80
- Firefox >= 78
- Safari >= 14
- Edge >= 80

不支持 Internet Explorer 11。

### 2.3 技术栈要求

| 技术 | 说明 |
|------|------|
| Vue 3 | 必须使用 Vue 3.2.0 及以上版本 |
| TypeScript | 推荐使用 4.5 及以上版本 |
| 构建工具 | Vite（推荐）或 Webpack |

## 三、安装步骤

### 3.1 使用 npm 安装（推荐）

npm 是 Node.js 的包管理工具，也是安装 Element Plus 的官方推荐方式。

```bash
# 进入你的 Vue 3 项目目录
cd your-vue-project

# 安装 Element Plus
npm install element-plus
```

### 3.2 使用 yarn 安装

```bash
# 安装 Element Plus
yarn add element-plus
```

### 3.3 使用 pnpm 安装

```bash
# 安装 Element Plus
pnpm add element-plus
```

## 四、在 Vue 3 项目中引入

### 4.1 完整引入（适合小型项目）

完整引入会打包所有组件，配置简单，但打包体积较大。

```typescript
// main.ts 或 main.js
import { createApp } from 'vue'
import App from './App.vue'

// 导入 Element Plus 和它的样式
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// 使用 Element Plus
app.use(ElementPlus)

app.mount('#app')
```

### 4.2 按需引入（推荐，适合生产环境）

按需引入可以显著减少打包体积，只引入实际使用的组件。

#### 4.2.1 安装自动导入插件

首先安装 unplugin-vue-components 和 unplugin-auto-import：

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

#### 4.2.2 配置 Vite

在 vite.config.ts 中添加配置：

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

这样配置后，组件可以自动导入，无需手动 import。例如直接使用：

```vue
<template>
  <el-button type="primary">你好</el-button>
</template>
```

#### 4.2.3 手动按需引入（备选方案）

如果不使用自动导入插件，也可以手动引入单个组件：

```typescript
// main.ts
import { createApp } from 'vue'
import App from './App.vue'

// 手动引入需要的组件
import { ElButton, ElInput, ElForm } from 'element-plus'
import 'element-plus/es/components/button/style'
import 'element-plus/es/components/input/style'
import 'element-plus/es/components/form/style'

const app = createApp(App)
app.use(ElButton)
app.use(ElInput)
app.use(ElForm)

app.mount('#app')
```

## 五、使用 CDN 引入（无需构建工具）

如果你不想使用 npm 或构建工具，可以通过 CDN 方式引入 Element Plus。

### 5.1 通过 unpkg CDN 引入

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Element Plus CDN 示例</title>
  <!-- 引入 Vue 3 -->
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <!-- 引入 Element Plus -->
  <link rel="stylesheet" href="https://unpkg.com/element-plus/dist/index.css">
  <script src="https://unpkg.com/element-plus"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary">你好 Element Plus</el-button>
  </div>

  <script>
    const { createApp, ref } = Vue
    
    const App = {
      setup() {
        const visible = ref(false)
        return { visible }
      },
      template: `
        <el-button @click="visible = true">打开对话框</el-button>
        <el-dialog v-model="visible" title="你好">这是 Element Plus</el-dialog>
      `
    }
    
    const app = createApp(App)
    app.use(ElementPlus)
    app.mount('#app')
  </script>
</body>
</html>
```

### 5.2 通过 jsDelivr CDN 引入

```html
<!-- 替换上面的 unpkg 为 jsDelivr -->
<script src="https://cdn.jsdelivr.net/npm/vue@3/dist/vue.global.js"></script>
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/element-plus/dist/index.css">
<script src="https://cdn.jsdelivr.net/npm/element-plus"></script>
```

## 六、国际化配置

Element Plus 默认使用英文，要使用中文需要配置国际化。

### 6.1 全局中文配置

```typescript
// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import zhCn from 'element-plus/es/locale/lang/zh-cn'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus, {
  locale: zhCn,
})

app.mount('#app')
```

### 6.2 动态切换语言

```typescript
// i18n.ts
import { createI18n } from 'vue-i18n'
import zhCn from 'element-plus/es/locale/lang/zh-cn'
import en from 'element-plus/es/locale/lang/en'

const i18n = createI18n({
  legacy: false,
  locale: 'zh-cn',
  messages: {
    'zh-cn': {
      ...zhCn,
    },
    en: {
      ...en,
    },
  },
})

export default i18n
```

然后在 main.ts 中使用：

```typescript
import { createApp } from 'vue'
import App from './App.vue'
import ElementPlus from 'element-plus'
import i18n from './i18n'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus)
app.use(i18n)

app.mount('#app')
```

## 七、主题定制

### 7.1 使用 SCSS 变量覆盖

Element Plus 使用 SCSS 编写，可以通过覆盖变量来定制主题。

```scss
// styles/element-variables.scss
// 主题色
$--color-primary: #409EFF;

// 覆盖组件变量
@use "element-plus/theme-chalk/src/common/var.scss" as *;

// 方法一：使用 scss 变量覆盖（需要 sass）
$el-button-primary-bg-color: #409EFF;
```

### 7.2 在线主题定制

访问 Element Plus 官方主题定制页面：https://element-plus.org/zh-CN/guide/theming.html

可以在线选择主题色、边框颜色、圆角等参数，下载定制后的 CSS 文件。

### 7.3 暗黑模式

Element Plus 内置支持暗黑模式：

```typescript
// 开启暗黑模式
import 'element-plus/theme-chalk/dark/css-vars.css'
```

在 HTML 标签上添加 class：

```html
<html class="dark">
  <!-- 你的应用 -->
</html>
```

## 八、常见问题

### 8.1 安装失败

**问题**：npm install 安装失败

**解决方案**：

1. 清除 npm 缓存：`npm cache clean --force`
2. 检查网络，可以使用国内镜像源：

```bash
# 设置 npm 镜像
npm config set registry https://registry.npmmirror.com

# 或者使用 yarn
yarn config set registry https://registry.npmmirror.com
```

### 8.2 样式不生效

**问题**：组件样式显示异常

**解决方案**：

1. 确保已引入样式文件：`import 'element-plus/dist/index.css'`
2. 检查构建配置是否正确处理了 CSS
3. 清除浏览器缓存并重新构建

### 8.3 TypeScript 类型错误

**问题**：引入组件时提示类型错误

**解决方案**：

1. 确保安装了最新版本的 TypeScript：`npm install -D typescript`
2. 在 tsconfig.json 中添加配置：

```json
{
  "compilerOptions": {
    "moduleResolution": "node",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true
  }
}
```

### 8.4 组件不显示

**问题**：组件引入后不显示

**解决方案**：

1. 检查是否正确使用了 app.use(ElementPlus)
2. 确认 Vue 版本是否在 3.2.0 以上
3. 检查浏览器控制台是否有错误信息

## 九、版本迁移（从 Element UI 迁移）

如果项目之前使用 Element UI，需要迁移到 Element Plus，可以参考以下步骤：

### 9.1 主要变化

| 变化项 | Element UI | Element Plus |
|--------|------------|--------------|
| Vue 版本 | Vue 2 | Vue 3 |
| 组件前缀 | el- | el-（保持不变） |
| 导入方式 | 全量/按需 | 全量/按需 |
| 事件绑定 | v-model | v-model（Vue 3 语法） |

### 9.2 使用迁移工具

Element Plus 官方提供了迁移工具 gogocode：

```bash
# 全局安装 gogocode
npm install -g gogocode

# 执行迁移
gogocode -s your-project-path -t element-plus -o
```
