# APISIX 常見問題

## 安裝問題

### Q1: Docker 容器啟動失敗

**問題**：執行 `docker run` 後容器立即退出。

**解決**：
1. 檢查 Docker 日誌：`docker logs apisix`
2. 確認 etcd 是否正常執行
3. 檢查連接埠是否被佔用：`netstat -tlnp | grep 9080`
4. 嘗試使用 docker-compose 方式啟動

### Q2: Admin API 無法存取

**問題**：存取 `http://127.0.0.1:9180/apisix/admin` 返回 404 或 403。

**解決**：
1. 確認 `enable_admin: true` 在 `config.yaml` 中已設定
2. 檢查 Admin API 金鑰設定是否正確
3. 驗證防火牆允許 9180 連接埠存取

### Q3: etcd 連線失敗

**問題**：啟動時報錯 "etcd connection failed"。

**解決**：
1. 確認 etcd 服務正在執行
2. 檢查 etcd 連接埠（預設 2379）是否可存取
3. 驗證 etcd 設定中的 endpoints 位址
4. 如果使用 Docker，確認網路設定正確

## 設定問題

### Q4: 路由匹配失敗

**問題**：請求返回 404，但路由設定看起來正確。

**解決**：
1. 檢查路由的 `uri` 設定是否正確
2. 確認上游節點是否可用
3. 查看 APISIX 錯誤日誌
4. 使用 `curl -v` 查看詳細請求資訊

### Q5: 插件不生效

**問題**：設定了插件但沒有效果。

**解決**：
1. 確認插件名稱正確
2. 檢查插件是否在正確的作用域（route/plugin）設定
3. 驗證插件設定格式正確
4. 檢查插件依賴是否安裝

### Q6: 負載均衡不生效

**問題**：請求沒有按預期分配到不同節點。

**解決**：
1. 確認上游節點設定正確
2. 檢查負載均衡類型設定
3. 如果使用雜湊，確認 key 欄位正確

## 效能問題

### Q7: 效能不達預期

**問題**：QPS 遠低於預期。

**解決**：
1. 調整 worker 行程數：`nginx.worker_processes`
2. 增加 worker 連線數：`nginx.worker_connections`
3. 啟用 HTTP/2：`proxy_http_version: "2"`
4. 優化插件設定，停用不必要的插件
5. 使用高效能儲存

### Q8: 記憶體佔用過高

**問題**：APISIX 記憶體使用持續增長。

**解決**：
1. 檢查是否有記憶體洩漏的插件
2. 限制快取大小
3. 定期清理日誌
4. 監控並調整緩衝區大小

## 安全問題

### Q9: 如何啟用 Admin API 鑑權

**解決**：

在 `config.yaml` 中設定 Admin API 金鑰：

```yaml
apisix:
  admin_key:
    - name: admin
      key: your-secret-key-here
      role: admin
```

使用金鑰存取 Admin API：

```bash
curl -H "X-API-Key: your-secret-key-here" \
  http://127.0.0.1:9180/apisix/admin/routes
```

### Q10: 如何設定 HTTPS

**解決**：

1. 取得 SSL 憑證
2. 設定 SSL 路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "https-route",
  "uri": "/api/*",
  "plugins": {
    "ssl": {
      "cert": "/path/to/cert.pem",
      "key": "/path/to/key.pem"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 整合問題

### Q11: 如何與 Kubernetes 整合

**解決**：

使用 APISIX Ingress Controller：

```bash
helm install apisix-ingress apisix/apisix-ingress \
  --set ingressController.enabled=true
```

### Q12: 如何與 Prometheus 整合

**解決**：

啟用 Prometheus 插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/plugins" -X PUT -d '
{
  "prometheus": {
    "prefer_name": true
  }
}'
```

Prometheus 指標端點：`http://127.0.0.1:9091/apisix/prometheus/metrics`

### Q13: 如何設定日誌

**解決**：

設定存取日誌：

```yaml
nginx:
  access_log: /var/log/apisix/access.log
  error_log: /var/log/apisix/error.log
  error_log_level: warn
```

使用日誌插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "log-route",
  "uri": "/api/*",
  "plugins": {
    "log-rotate": {
      "max_size": 100,
      "max_backups": 50,
      "max_age": 30,
      "compress": true
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 常見錯誤碼

| 錯誤碼 | 說明 | 解決方法 |
|--------|------|----------|
| 400 | 請求參數錯誤 | 檢查請求格式和參數 |
| 401 | 未授權 | 設定並使用 API 金鑰 |
| 403 | 禁止存取 | 檢查權限設定 |
| 404 | 路由不存在 | 確認路由設定 |
| 429 | 超過限流 | 調整限流設定 |
| 500 | 伺服器內部錯誤 | 檢查日誌 |
| 502 | 閘道錯誤 | 檢查上游服務 |
| 503 | 服務不可用 | 檢查上游健康狀態 |
