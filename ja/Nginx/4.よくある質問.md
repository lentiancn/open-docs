# Nginx よくある質問

## インストール問題

### Nginx のインストールに失敗した場合は？

1. システム依存関係が完全か確認
2. エラーログで具体的な失敗理由を確認
3. ディスク容量が十分か確認
4. 権限問題があるか確認

### Nginx バージョンを確認方法は？

```bash
nginx -v
nginx -V  # コンパイルパラメータを含む詳細情報
```

### Nginx はどのオペレーティングシステムをサポート？

Nginx は Linux、FreeBSD、Solaris、AIX、HP-UX、macOS、Windows をサポートしています。

## 設定問題

### Nginx 設定が正しいか確認する方法は？

```bash
nginx -t
```

### 設定ファイルはどこ？

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- ビルドインストール: `/usr/local/nginx/conf/nginx.conf`

### 設定変更後に再起動が必要？

再起動は不要です。`nginx -s reload` で設定を再読み込みできます。

## パフォーマンス問題

### Nginx パフォーマンスを最適化するには？

1. **ワーカープロセス数を調整**

```nginx
worker_processes auto;
```

2. **ワーカー接続数を増加**

```nginx
events {
    worker_connections 4096;
}
```

3. **gzip 圧縮を有効化**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **ブラウザキャッシュを有効化**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **接続プールを使用**

```nginx
upstream backend {
    server backend1.example.com;
    keepalive 32;
}
```

### Nginx のメモリ使用量が多すぎる場合は？

1. worker_connections 数を減らす
2. リクエストボディサイズを制限
3. キャッシュ設定を最適化
4. メモリリークがないか確認

### 高同時接続を処理するには？

1. ロードバランシングを使用
2. キャッシュを有効化
3. カーネルパラメータを調整
4. CDN を使用

## セキュリティ問題

### DDoS 攻撃を防ぐには？

1. **接続を制限**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **リクエストレートを制限**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **低速接続を閉じる**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### SSL 証明書を設定するには？

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### HTTP を強制的に HTTPS にリダイレクトするには？

```nginx
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

## プロキシ問題

### リ버스プロキシが失敗する場合は？

1. バックエンドサービスが実行中か確認
2. ファイアウォール設定を確認
3. proxy_pass 設定が正しいか検証
4. エラーログを確認

### WebSocket をプロキシするには？

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### プロキシ後に реальный IP を取得するには？

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## ログ問題

### ログファイルが大きすぎる場合は？

logrotate を使用：

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### アクセスログを無効化するには？

```nginx
access_log off;
```

### ログフォーマットを設定するには？

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## ロードバランシング問題

### ロードバランシングはどのようにバックエンドを選択？

Nginx は複数のロードバランシングアルゴリズムをサポート：

- **ラウンドロビン（デフォルト）**：順序대로リクエストを分配
- **重み付けラウンドロビン**：重み付けてリクエストを分配
- **IP ハッシュ**：同じ IP のリクエストを同じバックエンドに
- **最小接続**：接続数が最も少ないバックエンドに

### バックエンドサーバーが停止した場合は？

Nginx は自動的に障害ノードをスキップし、リクエストを正常なノードに転送します。

## キャッシュ問題

### プロキシキャッシュを設定するには？

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### キャッシュが効かない場合は？

1. キャッシュパス権限を確認
2. レスポンスヘッダーがキャッシュを許可するか確認
3. キャッシュキー設定が正しいか検証

### キャッシュをクリアするには？

```bash
rm -rf /var/cache/nginx/*
```

## ポート問題

### ポートが既に使用中の場合は？

```bash
# ポート使用状況を確認
netstat -tulpn | grep :80

# 別のポートを使用
listen 8080;
```

### 複数のポートをリスンするには？

```nginx
server {
    listen 80;
    listen 8080;
    server_name example.com;
}
```

## 一般的なエラーコード

### 400 Bad Request

- リクエストヘッダーが大きすぎる：`client_header_buffer_size` を増加
- URL が長すぎる：リクエストパスを確認

### 403 Forbidden

- インデックスファイルが存在しない
- 権限不足：ファイルとディレクトリ権限を確認

### 404 Not Found

- ファイルパスエラー
- SELinux または AppArmor がブロック

### 502 Bad Gateway

- バックエンドサービスが実行されていない
- FastCGI/PHP-FPM 設定エラー

### 504 Gateway Timeout

- バックエンド応答タイムアウト
- `proxy_read_timeout` を増加

## デバッグ技巧

### 詳細エラー情報を表示するには？

1. デバッグログを有効化：

```nginx
error_log /var/log/nginx/error.log debug;
```

2. エラーログを確認：

```bash
tail -f /var/log/nginx/error.log
```

### 設定が反映されたかテストするには？

```bash
# 設定テスト
nginx -t

# 完全な設定を表示
nginx -T
```

### 正規表現をデバッグするには？

`location` を使用：

```nginx
location /test {
    access_log /var/log/nginx/test.log;
}
```

## アップグレード問題

### Nginx をアップグレードするには？

1. 設定ファイルをバックアップ
2. Nginx を停止
3. 新バージョンをインストール
4. Nginx を開始
5. 機能をテスト

### スムーズアップグレードを実行するには？

```bash
# 新バージョンをコンパイル
make

# USR2 シグナルを送信
kill -s USR2 $(cat /var/run/nginx.pid)

# 古いワーカープロセスを優雅に終了
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
