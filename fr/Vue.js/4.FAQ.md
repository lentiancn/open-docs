# Vue.js FAQ

## Différences entre Vue 2 et Vue 3

Vue 3 est la version majeure actuelle et la plus récente de Vue. Elle contient de nouvelles fonctionnalités qui ne sont pas présentes dans Vue 2, telles que Teleport, Suspense et plusieurs éléments racine par template. Elle contient également des changements cassants qui la rendent incompatible avec Vue 2.

**Principaux Avantages :**
- Tailles de bundle plus petites
- Meilleures performances
- Meilleure évolutivité
- Meilleur support TypeScript / IDE

**Recommandation :**
- **Les nouveaux projets doivent utiliser Vue 3**
- **Considérez Vue 2 uniquement si vous avez besoin du support IE11** (Vue 3 ne supporte pas IE11)

**Informations de Migration :**
- L'API Composition était à l'origine une fonctionnalité exclusive à Vue 3, mais est maintenant disponible dans Vue 2.7
- Guide de migration détaillé : [Guide de Migration Vue 3](https://v3-migration.vuejs.org/)

## Vue 2 est-il encore supporté ?

- Vue 2.7, publié en juillet 2022, est la dernière version mineure de la gamme Vue 2
- **Vue 2 a atteint la fin de vie le 31 décembre 2023**
- Support étendu disponible pour les équipes ayant des exigences de conformité : [LTS Étendu Vue 2](https://v2.vuejs.org/lts/)

## Quels navigateurs Vue supporte-t-il ?

- **Vue 3.x** : Supporte uniquement les navigateurs avec support natif ES2016 (exclut IE11)
- **Vue 2.x** : Supporte IE11 et une compatibilité plus large avec les navigateurs

## Vue est-il fiable ?

- Vue est un framework mature et éprouvé
- Plus de **1,5 million d'utilisateurs dans le monde**
- Presque **10 millions de téléchargements mensuels** sur npm
- Utilisé en production par des organisations renommées : Wikimedia Foundation, NASA, Apple, Google, Microsoft, GitLab, Zoom, Tencent, Weibo, Bilibili, Kuaishou, et bien d'autres

## Vue est-il rapide ?

- Vue 3 est l'un des frameworks frontend les plus performants
- Surpasse React et Angular dans le [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html)
- Le site web de documentation officiel obtient un score de 100 en performance sur un Moto G4 émulé avec throttling CPU 4x sur des réseaux 4G lents
- Gère la plupart des cas d'utilisation d'applications web sans optimisation manuelle

## Vue est-il léger ?

- **Application minimale** (avec outil de build + tree-shaking) : ~16ko (compression brotli)
- **Application complète** : ~27ko
- **Sans outil de build** (inclut le compilateur) : ~41ko
- **Amélioration progressive** : Considérez [petite-vue](https://github.com/vuejs/petite-vue) (~6ko)

## Vue évolue-t-il ?

Oui. Malgré la croyance commune que Vue n'est adapté qu'à des cas d'utilisation simples, Vue est parfaitement capable de gérer des applications à grande échelle :

- **Composants à Fichier Unique** : Fournissent un modèle de développement modularisé
- **API Composition** : Intégration TypeScript de première classe pour organiser la logique complexe
- **Support d'outils complet** : Garantit une expérience de développement fluide à mesure que l'application grandit
- **Barrière d'entrée faible** : Réduit les coûts d'intégration et de formation pour les nouveaux développeurs

## Licence Open Source

## Choix du Style d'API

### API Options vs API Composition

**L'API Options convient pour :**
- Les débutants ou développeurs issus de la POO
- Scénarios simples sans outils de build
- Applications d'amélioration progressive

**L'API Composition convient pour :**
- Développement complet d'applications avec Vue
- Meilleure organisation et réutilisation de la logique
- Projets TypeScript

## JavaScript vs TypeScript

- Vue lui-même est implémenté en TypeScript avec un support TS de première classe
- **N'impose pas l'utilisation de TypeScript**
- Les APIs conçues avec TypeScript en tête bénéficient à tous les utilisateurs (meilleur support IDE)
- Le choix dépend du contexte de l'équipe et de l'échelle du projet

## Vue vs Web Components

- Vue a été créé avant que les Web Components soient disponibles
- Les Web Components sont de bas niveau (centrés sur les éléments personnalisés)
- Vue aborde des préoccupations de plus haut niveau (rendu, gestion d'état, routage, SSR)
- Vue supporte complètement la consommation et l'exportation d'éléments personnalisés natifs
- Comparaison détaillée : [Guide Vue et Web Components](/guide/extras/web-components)
