# Installationsanleitung

Dieses Dokument zeigt verschiedene Installationsmethoden für Apache APISIX, damit Sie schnell ein API-Gateway-Umfeld einrichten können.

## Anforderungen an die Umgebung

Bevor Sie APISIX installieren, stellen Sie sicher, dass folgende Voraussetzungen erfüllt sind:

- **Betriebssystem**: Linux (empfohlen: Ubuntu 20.04+, CentOS 7+, Debian 11+)
- **Docker** (für Docker-Deployment): Docker und Docker Compose müssen installiert sein
- **etcd**: APISIX verwendet etcd als Konfigurationsspeicher, Version 3.4.0 oder höher erforderlich
- **Perl**: Einige Abhängigkeiten benötigen Perl 5.5.32+
- **curl**: Zur Überprüfung der Installation

## Docker-Schnellinstallation (empfohlen)

Dies ist der schnellste Weg zur Installation und eignet sich für Entwicklung und Testumgebungen.

### Schritt 1: Quick-Start-Verzeichnis erstellen

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### Schritt 2: Quick-Start-Skript ausführen

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

Dieser Befehl startet automatisch zwei Container: apisix-quickstart und etcd. APISIX nutzt etcd zum Speichern und Synchronisieren der Konfiguration. Die Container verwenden den Docker-Host-Netzwerkmodus, sodass Sie lokal darauf zugreifen können.

### Schritt 3: Installation überprüfen

Bei erfolgreicher Installation zeigt das Terminal folgende Meldung:

```
✔ APISIX is ready!
```

Sie können den APISIX-Status mit folgendem Befehl prüfen:

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Normalerweise erhalten Sie:

```
Server: APISIX/Version
```

Dabei ist „Version" die installierte APISIX-Version, z. B. APISIX/3.15.0.

### Schritt 4: Dashboard aufrufen

APISIX enthält ein eingebautes Dashboard, das Sie unter folgender Adresse erreichen:

```
http://127.0.0.1:9180/ui
```

> ⚠️ Hinweis: Die Admin-API ist standardmäßig ohne Authentifizierung erreichbar. In Produktionsumgebungen sollten Sie die Autorisierung aktivieren.

## Docker Compose Installation

Für angepasste Konfigurationen können Sie Docker Compose verwenden.

### Schritt 1: Konfigurationsdatei erstellen

Erstellen Sie eine docker-compose.yaml-Datei:

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### Schritt 2: Dienste starten

```bash
docker-compose up -d
```

### Schritt 3: Dienste überprüfen

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Kubernetes-Installation

APISIX bietet einen offiziellen Ingress Controller, der direkt in Kubernetes deployed werden kann.

### Installation mit Helm Chart

```bash
# APISIX Helm-Repository hinzufügen
helm repo add apisix https://charts.apiseven.com
helm repo update

# APISIX installieren
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### Installation mit kubectl

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## Installation aus dem Quellcode

Geeignet für Produktionsumgebungen mit angepassten Builds.

### Schritt 1: Abhängigkeiten installieren

**Ubuntu/Debian:**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL:**

```bash
sudo yum install -y perl gcc git curl
```

### Schritt 2: OpenResty installieren

```bash
# OpenResty-Repository hinzufügen
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# OpenResty installieren
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### Schritt 3: APISIX installieren

```bash
# Quellcode klonen
git clone https://github.com/apache/apisix.git
cd apisix

# Abhängigkeiten installieren
make deps

# Konfiguration initialisieren
make init

# APISIX starten
make run
```

## Installation überprüfen

Unabhängig von der gewählten Installationsmethode können Sie die Installation wie folgt überprüfen:

1. **Prozessstatus prüfen**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **Testroute erstellen**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **Route testen**

```bash
curl "http://127.0.0.1:9080/test"
```

## Deinstallation

**Docker:**

```bash
# Container stoppen
docker-compose down

# Volumes löschen (falls gewünscht)
docker-compose down -v
```

**Kubernetes:**

```bash
helm uninstall apisix
```
