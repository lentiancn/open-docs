# Jenkins 常見問題

## 基礎問題

### 什麼是 Jenkins？

Jenkins 是一個開源的自動化伺服器，主要用於持續整合（CI）和持續交付（CD）。它可以幫助開發團隊自動化建置、測試和部署軟體過程，提高開發效率和品質。

### Jenkins 是免費的嗎？

是的，Jenkins 是完全開源免費的專案。它採用 MIT 授權，任何人都可以自由使用、修改和分發。

### Jenkins 需要什麼執行環境？

Jenkins 需要 Java 8 或更高版本。可以在多種作業系統上執行，包括 Windows、Linux、macOS 等。推薦至少 1GB 記憶體和 10GB 磁碟空間。

### Jenkins 和 Hudson 有什麼區別？

Jenkins 是 Hudson 的 fork。2011 年，由於與 Oracle 的商標爭議，Hudson 專案的主要開發團隊 fork 了 Hudson 並建立了 Jenkins。現在 Jenkins 是活躍的專案，而 Hudson 已停止維護。

## 安裝問題

### 首次啟動無法存取怎麼辦？

1. 檢查 Jenkins 是否正常啟動：查看日誌
2. 確認連接埠未被佔用：預設連接埠是 8080
3. 檢查防火牆設定：確保允許存取連接埠
4. 查看啟動日誌中的錯誤資訊

### 解鎖密碼在哪裡？

首次安裝時，解鎖密碼在 Jenkins 控制台日誌中顯示。如果是 Docker 方式執行，使用 `docker logs jenkins-blueocean` 查看。密碼檔案位置通常在 `/var/jenkins_home/secrets/initialAdminPassword`。

### 外掛程式安裝失敗怎麼辦？

1. 檢查網路連線：Jenkins 需要存取更新中心
2. 手動安裝：嘗試從進階選項卡上傳外掛程式 hpi 檔案
3. 設定 Jenkins 镜像：設定國內更新源
4. 查看日誌：了解具體失敗原因

### 如何修改 Jenkins 連接埠？

- Docker 方式：修改 -p 參數
- WAR 方式：使用 `java -jar jenkins.war --httpPort=9090`
- Linux 系統套件：修改 /etc/default/jenkins 中的 HTTP_PORT

## 憑證問題

### 憑證無法使用怎麼辦？

1. 確認憑證 ID 正確
2. 檢查憑證類型是否匹配
3. 驗證憑證是否過期
4. 檢查使用者權限

### 如何安全地儲存密碼？

使用 Jenkins 的憑證儲存功能：

1. 進入 Manage Credentials
2. 新增憑證，選擇類型
3. Jenkins 會加密儲存，不會明文顯示

### SSH 金鑰認證失敗怎麼辦？

1. 確認公鑰已新增到目標伺服器
2. 檢查金鑰權限：私鑰檔案權限應為 600
3. 驗證金鑰格式是否正確
4. 嘗試重新產生金鑰對

## 流水線問題

### Jenkinsfile 語法錯誤怎麼排查？

1. 使用聲明式流水線驗證器
2. 查看 Jenkins 控制台錯誤提示
3. 在本地使用 Jenkins 模擬器驗證
4. 參考官方語法文件

### 流水線執行失敗怎麼辦？

1. 查看建置日誌取得詳細錯誤資訊
2. 檢查各階段的返回狀態
3. 確認依賴的工具是否可用
4. 驗證環境變數和憑證

### 如何實現平行執行？

聲明式流水線支援 parallel 語法：

```groovy
pipeline {
    agent any
    stages {
        stage('平行建置') {
            parallel {
                stage('模組A') {
                    steps {
                        sh './build-a.sh'
                    }
                }
                stage('模組B') {
                    steps {
                        sh './build-b.sh'
                    }
                }
            }
        }
    }
}
```

### 如何等待人工確認？

使用 input 步驟：

```groovy
stage('部署生產') {
    steps {
        input message: '確認部署到生產環境?', ok: '確認部署'
    }
}
```

## 建置問題

### 建置速度很慢怎麼辦？

1. 使用增量建置
2. 設定建置快取
3. 使用平行階段
4. 增加建置節點
5. 優化依賴下載

### Maven 建置下載依賴慢怎麼辦？

1. 設定 Maven 鏡像倉庫
2. 使用私有倉庫管理員
3. 設定本地倉庫路徑並持久化

### 如何設定建置超時？

```groovy
pipeline {
    agent any
    stages {
        stage('超時測試') {
            options {
                timeout(time: 30, unit: 'MINUTES')
            }
            steps {
                // 建置步驟
            }
        }
    }
}
```

### 建置產物如何保留？

使用封存步驟：

```groovy
post {
    always {
        archiveArtifacts artifacts: 'target/*.jar', fingerprint: true
    }
}
```

## 分散式建置問題

### 節點連線失敗怎麼辦？

1. 檢查網路連通性
2. 確認 Java 版本相容性
3. 驗證節點金鑰設定
4. 查看節點日誌

### 如何設定 Docker 節點？

1. 安裝 Docker 外掛程式
2. 設定 Docker 雲
3. 在流水線中指定 docker 代理程式

### 負載平衡如何設定？

在流水線中設定代理程式選擇策略：

```groovy
pipeline {
    agent none
    stages {
        stage('建置') {
            agent { label 'docker' }
            steps {
                // 建置
            }
        }
    }
}
```

### 如何設定郵件通知？

1. 安裝 Email Extension 外掛程式
2. 設定 SMTP 伺服器
3. 在流水線中新增郵件步驟

### Slack 通知如何設定？

1. 安裝 Slack 外掛程式
2. 在 Slack 建立 Jenkins 應用
3. 設定 Jenkins 中的 Slack 憑證
4. 在流水線中新增通知步驟

## 備份和復原

### 如何備份 Jenkins？

需要備份以下內容：

1. JENKINS_HOME 目錄（包含所有設定和建置記錄）
2. 外掛程式目錄
3. 環境設定檔

推薦使用 Configuration as Code 外掛程式管理設定。

### 如何移轉 Jenkins？

1. 停止來源 Jenkins
2. 複製 JENKINS_HOME 到新伺服器
3. 安裝相同版本的 Jenkins
4. 確保 Java 版本相容
5. 啟動新 Jenkins

## 安全問題

### 如何開啟使用者註冊？

1. 進入 Configure Global Security
2. 啟用安全網域（Jenkins 自己的使用者資料庫）
3. 設定授權策略（Logged-in users can do anything）

### 如何限制專案存取？

使用矩陣授權策略：

1. 設定全域安全
2. 選擇 Matrix Authorization Strategy
3. 為不同使用者設定不同權限

### 如何處理安全漏洞？

1. 及時更新 Jenkins 核心版本
2. 定期更新外掛程式
3. 關注 Jenkins 安全公告
4. 限制匿名存取權限

## 效能問題

### 記憶體不足怎麼辦？

1. 調整 JVM 參數：在啟動腳本中新增 `-Xmx`
2. 減少並發建置數
3. 定期清理舊建置記錄
4. 增加系統記憶體

### 建置記錄佔用太多空間？

1. 設定建置保留策略
2. 定期清理舊建置
3. 使用封存而非保留完整建置記錄

## 整合問題

### 與 GitHub 整合失敗？

1. 檢查 GitHub API 權限
2. 驗證 Webhook 設定
3. 確認憑證正確

### 與 Docker 整合失敗？

1. 檢查 Docker API 設定
2. 確認 Docker socket 權限
3. 驗證 Docker 版本相容性

### 與 Kubernetes 整合失敗？

1. 檢查 kubeconfig 設定
2. 驗證 Kubernetes API 權限
3. 確認 Jenkins 容器有足夠權限

## 升級問題

### 如何升級 Jenkins？

- Docker 方式：更新映像檔版本
- WAR 方式：替換 WAR 檔案
- 系統套件：使用套件管理員更新

### 升級後外掛程式不相容？

1. 查看相容性清單
2. 更新外掛程式到相容版本
3. 必要時回滾 Jenkins 版本
