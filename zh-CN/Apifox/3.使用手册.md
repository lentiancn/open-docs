# Apifox 使用手册

## 界面介绍

初次打开 Apifox，你会看到一个清晰的界面布局。让我带你熟悉一下各个区域的功能：

顶部是菜单栏和工具栏，包含项目切换、快捷操作按钮（新建、导入、导出等）、搜索框等。中间左侧是侧边栏，展示项目结构和接口列表。中间是主要工作区，用于接口设计、调试和测试。右侧是详情面板，展示当前选中项的详细信息。

侧边栏支持折叠和展开，你可以根据需要调整宽度。接口列表支持搜索和筛选，可以快速定位到需要的接口。工作区支持多标签页，你可以同时打开多个接口进行编辑。

## 项目管理

### 创建项目

在 Apifox 中，项目是组织接口的最小单位。点击侧边栏顶部的「+」按钮，选择「新建项目」，填写项目名称和描述即可创建新项目。

建议按照实际项目来创建项目，比如「用户中心 API」「订单系统 API」等。一个项目中可以包含多个模块，方便接口的分类管理。

### 项目设置

每个项目都有一些可以自定义的设置。点击项目名称右侧的「...」按钮，选择「项目设置」，可以配置以下内容：

**基本设置**：修改项目名称、描述，设置项目图标。

**环境配置**：配置不同环境的变量，比如开发环境、测试环境、生产环境的接口地址可能不同，通过环境变量可以方便地进行切换。

**Auth 认证**：配置全局的认证信息，比如 OAuth 2.0、Bearer Token、Basic Auth 等。配置完成后，所有接口都会自动使用这个认证信息。

### 导入导出

Apifox 支持多种格式的导入导出，方便与其他工具进行数据交换：

**支持导入的格式**：Postman 集合、Swagger/OpenAPI、Har、JSON、Dart、Java、TypeScript、Go、Python 等。导入时可以选择导入到现有项目或创建新项目。

**支持导出的格式**：Postman 集合、Swagger/OpenAPI、Markdown 文档、HTML 文档、JSON Schema 等。

## 接口设计与调试

### 创建接口

点击侧边栏的「+」按钮，选择「新建接口」，或者直接使用快捷键 Ctrl+N（macOS 为 Cmd+N）。在弹出的对话框中填写接口的基本信息：

**接口名称**：给接口起一个清晰的名字，比如「获取用户信息」「创建订单」等。

**请求方式**：选择 HTTP 方法，常用的有 GET（获取数据）、POST（创建数据）、PUT（更新数据）、DELETE（删除数据）。

**请求路径**：填写接口的 URL 路径，比如 /api/users/:id，其中 :id 表示路径参数。

### 请求参数设置

创建接口后，在工作区可以看到请求参数的配置区域。这里可以设置以下内容：

**Path 参数**：URL 路径中的参数，比如 /api/users/:id 中的 id。需要在「路径参数」区域填写具体的值来测试。

**Query 参数**：URL 问号后面的参数，比如 /api/users?page=1&size=10。在「请求参数」区域添加参数，填写参数名和值。

**Headers**：HTTP 请求头，比如 Content-Type、Authorization 等。根据需要添加必要的请求头。

**Body**：请求体，用于 POST、PUT 等请求。Apifox 支持多种 Body 类型：

- None：不带请求体
- form-data：表单数据，支持文件上传
- x-www-form-urlencoded：URL 编码的表单
- raw：原始数据，支持 JSON、XML、Text、HTML 等格式
- binary：二进制数据
- GraphQL：GraphQL 查询

**前置脚本**：在请求发送前执行的 JavaScript 代码。可以用来动态设置参数值、生成签名等。

### 发送请求与查看响应

配置好请求参数后，点击工作区上方的「发送」按钮，或者使用快捷键 Ctrl+Enter，即可发送请求。

响应区域会显示以下信息：

**响应状态**：HTTP 状态码，如 200 表示成功，404 表示未找到，500 表示服务器错误。

**响应时间**：请求花费的时间，方便评估接口性能。

**响应大小**：响应数据的大小。

**响应体**：服务器返回的数据。Apifox 会自动格式化 JSON、XML 等格式，方便阅读。你也可以切换到「原始」标签查看原始数据。

**响应头**：服务器返回的响应头信息。

**Cookies**：响应中设置的 cookies。

### 断言与后置脚本

Apifox 支持在请求响应后进行断言验证和后置脚本处理：

**断言**：用于验证响应是否符合预期。比如验证状态码是否为 200、响应体中某个字段的值是否正确等。点击「断言」标签，添加断言规则。常见的断言类型包括：状态码断言、响应体内容断言、响应时间断言等。

**后置脚本**：在请求完成后执行的 JavaScript 代码。可以用来提取响应数据、设置变量、进行数据处理等。比如将登录接口返回的 token 保存到环境变量，供后续接口使用。

## Mock 功能

### 什么是 Mock

Mock 即模拟数据的意思。在前后端分离的开发模式中，前端开发往往需要等后端接口开发完成后才能进行联调。如果后端接口还未完成，前端就无法正常开发。

Mock 功能就是来解决这个问题的。你可以提前定义好接口的数据结构，Apifox 会根据定义自动生成模拟的响应数据。这样前端就可以像调用真实接口一样调用 Mock 接口，获取模拟数据进行开发。

### 创建 Mock 数据

Apifox 提供了两种创建 Mock 数据的方式：

**自动 Mock**：在接口的响应示例中填写示例数据时，Apifox 会自动尝试生成 Mock 数据。比如你填写 `"name": "张三"`，Apifox 会识别出这是姓名类型，自动生成中文名字。

**自定义 Mock 规则**：对于更复杂的场景，你可以自定义 Mock 规则。比如指定某个字段固定返回一个值、随机返回一个范围内的数字、或者按照特定规则生成字符串。点击响应字段右侧的「Mock」图标，可以设置字段的自定义规则。

### Mock 服务器

创建好 Mock 数据后，你需要启动 Mock 服务器才能使用。

点击右侧面板中的「Mock」标签，找到「服务器地址」。默认情况下，Apifox 会在本地启动一个 Mock 服务器（地址类似 http://127.0.0.1:8080）。

如果你使用的是本地 Mock 服务器，需要确保 Mock 服务正在运行。点击「启动 Mock 服务」按钮即可启动。

### 使用 Mock 接口

启动 Mock 服务后，你可以在项目中找到所有设置了 Mock 数据的接口。接口名称旁边会有一个 Mock 图标标识。

Mock 接口的使用方式与真实接口一样，只需要将请求地址替换为 Mock 服务器地址即可。Apifox 支持一键切换，可以选择「使用 Mock URL」或「使用实际 URL」。

### 智能 Mock

Apifox 的智能 Mock 功能非常强大。它会根据字段名和字段类型自动推断并生成合适的数据：

- 包含「name」「user」等关键词的字符串字段 → 自动生成中文姓名
- 包含「email」的字段 → 自动生成邮箱地址
- 包含「phone」「mobile」的字段 → 自动生成手机号
- 包含「avatar」「image」「photo」的字段 → 自动生成图片 URL
- 日期类型字段 → 自动生成日期
- 数字类型字段 → 自动生成随机数字
- 布尔类型字段 → 自动生成 true/false

这种智能推断可以大大减少手动配置 Mock 规则的工作量。

## 接口测试

### 测试用例

Apifox 允许为接口创建测试用例，对接口进行自动化测试。在接口详情页面，点击「测试」标签，然后点击「新建测试用例」。

测试用例中可以添加多个断言，验证接口的各个方面：

- 响应状态码是否符合预期
- 响应体中特定字段的值是否正确
- 响应时间是否在可接受范围内
- 响应体是否为有效的 JSON 格式

保存测试用例后，可以在接口列表中看到每个接口的测试用例数量。

### 测试集合

将多个相关的测试用例组织在一起，就形成了测试集合。点击侧边栏的「+」按钮，选择「新建测试集合」，可以创建测试集合。

在测试集合中可以添加接口或测试用例，设置执行顺序。测试集合支持批量运行，可以一次性执行集合中的所有测试用例。

### 运行测试

创建好测试用例或测试集合后，可以点击「运行」按钮执行测试。

运行结果会显示每个测试用例的执行状态（通过/失败）、响应时间、错误信息等。如果有测试失败，会详细说明失败的原因，方便定位问题。

Apifox 还支持定时自动运行测试，你可以设置定时任务，定期执行测试集合，实现接口的持续监控。

## 接口文档

### 生成文档

Apifox 可以自动为项目生成接口文档。在项目设置中开启「自动生成文档」后，每次保存接口时，文档都会自动更新。

接口文档的内容包括：接口名称、请求方式、请求路径、请求参数说明、响应参数说明、请求示例、响应示例等。这些信息都来自于接口的设计和调试过程，无需额外填写。

### 查看与分享

生成的文档可以通过以下方式查看：

**内嵌查看**：在 Apifox 中直接查看文档。点击侧边栏的「文档」按钮，或者在接口列表中点击接口名称旁边的「文档」图标。

**在线分享**：将文档发布到 Apifox 提供的在线文档服务，获得一个可分享的链接。团队成员或外部人员可以通过链接查看文档，无需安装 Apifox。

**导出文档**：支持导出为 Markdown、HTML、PDF 等格式，方便离线阅读或集成到其他文档系统。

### 文档设置

在项目设置的「文档」选项卡中，可以配置文档的展示样式：

- 文档标题和描述
- 导航栏颜色和 Logo
- 是否显示调试功能
- 是否允许导出

## 团队协作

### 邀请成员

Apifox 支持团队协作功能。在项目设置中点击「添加成员」，输入成员的账号（邮箱或手机号）即可邀请成员加入项目。

被邀请的成员接受邀请后，会获得相应的项目权限。权限分为管理员和成员两种：管理员可以管理项目设置、删除接口等操作；成员可以查看和编辑接口。

### 同步与协作

登录账号后，项目数据会自动同步到云端。团队成员可以实时看到彼此的修改，保持数据一致。

如果多人同时编辑同一个接口，Apifox 会提示冲突，让你选择保留哪个版本。建议团队成员在编辑前先拉取最新数据，编辑完成后及时保存推送。

### 分支管理

在团队协作中，分支功能非常有用。你可以创建「开发分支」和「主分支」，在开发分支上进行接口设计和调试，确认无误后再合并到主分支。

这种工作方式可以避免正在开发中的接口影响已经稳定的接口，也方便进行功能对比和回滚。
