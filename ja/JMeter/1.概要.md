# Apache JMeter 概要

Apache JMeter はApache Software Foundation旗下的オープンソースソフトウェアであり、100%純粋なJavaで作成されたアプリケーションです。負荷テスト機能とパフォーマンス測定のために設計されています。JMeterは元々はWebアプリケーションのテストのために設計されていましたが、その後他のテスト機能にも拡張されています。

## JMeterできること

JMeterは静的リソースと動的リソースのパフォーマンステストに使用でき、Webアプリケーションを含みます。サーバー、サーバーグループ、ネットワーク、またはオブジェクトに対する高負荷をシミュレートして、その強度を確認したり、さまざまな負荷タイプでの全体的なパフォーマンスを分析したりできます。

主な機能的特性：

- **マルチプロトコルサポート**：HTTP、HTTPS、SOAP/REST Webサービス、FTP、JDBC、LDAP、JMSメッセージミドルウェア、メールプロトコル（SMTP/POP3/IMAP）、TCP、Javaオブジェクトなど、さまざまなアプリケーション、サーバー、プロトコルタイプをサポート
- **完全なテストIDE**：ブラウザまたはネイティブアプリケーションからテスト計画を素早く記録し、ビルド、デバッグできる完全な機能を提供
- **コマンドラインモード**：CLIモード（コマンドラインモード/ヘッドレスモード）をサポートし，任何Java対応OS（Linux、Windows、Mac OSXなど）で負荷テストを実行可能
- **動的HTMLレポート**：完全でプレゼンテーションしやすい動的HTMLレポート生成機能を内置
- **データ抽出機能**：HTML、JSON、XMLまたは任意のテキスト形式など、一般的なレスポンス形式からデータを抽出をサポート
- **純粋なJava**：完全にポータブル、100%純粋なJava実装
- **マルチスレッドフレームワーク**：完全な并发サンプリング能力をサポートし、複数のスレッドによる同時サンプリングと異なる機能グループの個別テストを実現
- **高い拡張性**：プラガブルサンプラー、计时器、JSR223対応スクリプト言語（GroovyやBeanShellなど）をサポート
- **継続的インテグレーション**：Maven、Gradle、Jenkinsとの統合に関する第三方开源ライブラリを提供

## JMeterはブラウザではない

JMeterはプロトコルレベルで動作するブラウザではありません。Webサービスとリモートサービスに関しては、JMeterはブラウザのように見えます（むしろ複数のブラウザ）が、JMeterはブラウザがサポートするすべてのアクションを実行するわけではありません。特に、JMeterはHTMLページにあるJavaScriptを実行せず、ブラウザのようにHTMLページをレンダリングしません。

## JMeterのユースケース

- 負荷テスト：大量并发ユーザーをシミュレートし、高負荷下でのシステムパフォーマンスをテスト
- パフォーマンステスト：さまざまな負荷条件下でのシステムの応答時間とスループットを測定
- 機能テスト：アプリケーションの機能動作を検証
- ストレステスト：高圧下でのシステムの安定性とフォールトトレランスをテスト
- 分散テスト：複数のマシンでの協調負荷テストをサポート
