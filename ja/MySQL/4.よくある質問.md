# MySQL よくある質問

## インストールに関する問題

### Q: MySQL サービスが起動しない場合は？

A: 以下を確認してください：
1. MySQL が正しくインストールされていることを確認
2. エラーログを確認：`mysqld --console`
3. ポートが使用されていないか確認：`netstat -an | grep 3306`
4. データディレクトリの権限が正しいことを確認
5. データディレクトリの再初期化を試みる

### Q: root パスワードを忘れた場合は？

A: 
1. MySQL サービスを停止
2. 権限チェックをスキップして起動：`mysqld --skip-grant-tables`
3. MySQL にログイン
4. パスワードをリセット：`UPDATE mysql.user SET Password=PASSWORD('新しいパスワード') WHERE User='root';`
5. 権限をフラッシュ：`FLUSH PRIVILEGES;`
6. MySQL サービスを再起動

### Q: MySQL のポート番号を変更するには？

A: my.cnf または my.ini 設定ファイルを変更：
```ini
[mysqld]
port=3307
```
その後、MySQL サービスを再起動してください。

## 接続に関する問題

### Q: リモート接続が拒否される場合は？

A: 
1. MySQL がリモート接続を許可していることを確認（ユーザーテーブルでユーザーの host を確認）
2. ファイアウォールがポートを許可していることを確認
3. MySQL 設定で bind-address が 127.0.0.1 でないことを確認
4. SELinux または AppArmor の設定を確認

### Q: 接続タイムアウトした場合は？

A: 
1. ネットワーク接続を確認
2. wait_timeout 設定を増やす
3. 永続接続または接続プールを使用
4. サーバーの負荷を確認

## データベース操作の問題

### Q: 文字化けが発生するのですが？

A: 
1. データベース作成時に文字セットを指定：`CREATE DATABASE db CHARACTER SET utf8mb4;`
2. 接続時に文字セットを指定：`SET NAMES utf8mb4;`
3. 設定ファイルでデフォルト文字セットを設定：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: インポート・エクスポート時に文字化けする場合は？

A: 
1. エクスポート時に文字セットを指定：`mysqldump --default-character-set=utf8mb4`
2. インポート時に文字セットを指定：`mysql --default-character-set=utf8mb4`

### Q: 自動increament 主キーが連続しないのですが？

A: 正常です。自動Increament ID が連続しない理由は次のとおりです：
- 挿入に失敗しても ID は消費される
- トランザクションのロールバックでは使用された ID が回収されない
- バッチ挿入では飛び飛びになる可能性がある
- レコードを削除しても自動 Increament カウンターには影響しない

## パフォーマンスの問題

### Q: クエリが遅い場合は？

A: 
1. EXPLAIN を使用してクエリの実行計画を分析
2. WHERE、JOIN、ORDER BY 涉及的列にインデックスを作成
3. フルテーブルスキャンを避ける
4. クエリを最適化し、SELECT * を避ける
5. クエリキャッシュの使用を検討（MySQL 8.0 では削除済み）
6. テーブル分割またはデータベース分割で負荷を分散

### Q: スロークエリを確認するには？

A: 
1. スロークエリログを有効化：
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. スロークエリログの場所を確認：
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: メモリ使用量が多い場合の最適化は？

A: 
1. innodb_buffer_pool_size を調整（利用可能なメモリの 70% を推奨）
2. クエリを最適化して不必要なスキャンを削減
3. 接続数を制限
4. 不要なキャッシュをクリア

## バックアップとリストアの問題

### Q: バックアップファイルが大きすぎる場合は？

A: 
1. 圧縮バックアップを使用：`mysqldump | gzip > backup.sql.gz`
2. データベース/テーブルごとにバックアップ
3. xtrabackup を使用して物理バックアップを実施
4. 定期的に過去のデータをクリーンアップ

### Q: 増分バックアップを実装するには？

A: 
1. バイナリログを有効化：`log-bin=mysql-bin`
2. バイナリログを定期的にバックアップ
3. mysqlbinlog ツールを使用して増分データをリストア

## レプリケーションの問題

### Q: マスター-スレーブ レプリケーションの遅延が高い場合は？

A: 
1. ネットワーク遅延を確認
2. slave_parallel_workers を調整して並列レプリケーションが増加
3. スレーブのハードウェアパフォーマンスが十分であることを確認
4. マスターの書き込み速度を最適化
5. レプリケーションフィルタルールを確認

### Q: マスター-スレーブ データ不一致の場合は？

A: 
1. スレーブを再同期
2. pt-table-checksum を使用して不一致を確認
3. pt-table-sync を使用してデータを修復
4. マスター-スレーブ レプリケーションを再設定

## セキュリティの問題

### Q: MySQL のセキュリティを強化するには？

A: 
1. 強力なパスワードポリシーを設定
2. root ユーザーのリモートログインを制限
3. 匿名アカウントを削除
4. テストデータベースを削除
5. MySQL バージョンを定期的に更新
6. SSL/TLS 暗号化接続を有効化
7. ファイアウォールルールを設定
8. 最小権限の原則に従う

### Q: SQL インジェクションを防ぐには？

A: 
1. プリペアードステートメントを使用
2. ユーザー入力を検証およびフィルタリング
3. データベースユーザーの権限を制限
4. 動的 SQL 連結を避ける
5. ストアドプロシージャを使用
