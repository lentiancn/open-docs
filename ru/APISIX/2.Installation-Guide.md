# APISIX Installation Guide

This document describes various installation methods for Apache APISIX.

## Docker Installation (Recommended)

Deploying APISIX with Docker is the simplest way:

```bash
# Pull APISIX image
docker pull apache/apisix:latest

# Start APISIX
docker run -d -p 9080:9080 -p 9180:9180 -p 9443:9443 \
  --name apisix \
  apache/apisix:latest
```

### Install with Docker Compose

Create a `docker-compose.yml` file:

```yaml
version: '3'

services:
  apisix:
    image: apache/apisix:latest
    ports:
      - "9080:9080"
      - "9443:9443"
      - "9180:9180"
    environment:
      - APISIX_DASHBOARD_URL=http://apisix:9180
    volumes:
      - ./apisix_log:/var/log/apisix
    network_mode: host

  etcd:
    image: apache/apisix:etcd:latest
    ports:
      - "2379:2379"
    environment:
      - ETCD_HOST=127.0.0.1
    network_mode: host
```

Start the services:

```bash
docker-compose up -d
```

## Quickstart Script Installation

APISIX provides a quick installation script that requires Docker and curl:

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

This script will automatically:
1. Start the etcd container
2. Start the APISIX container
3. Configure network connections

After installation, you will see:
```
✔ APISIX is ready!
```

Verify the installation:
```bash
curl "http://127.0.0.1:9080" --head | grep Server
# Output: Server: APISIX/Version
```

## Helm Chart Installation (Kubernetes)

Install APISIX using Helm in a Kubernetes environment:

```bash
# Add APISIX Helm repository
helm repo add apisix https://apache.github.io/apisix-helm-chart
helm repo update

# Install APISIX
helm install apisix apisix/apisix
```

## Source Compilation Installation

### Requirements

- Nginx ≥ 1.21.4
- OpenResty ≥ 1.21.4
- Lua ≥ 5.1
- etcd ≥ 3.4.0

### Build Steps

```bash
# Clone the source code
git clone https://github.com/apache/apisix.git
cd apisix

# Install dependencies
make deps

# Start APISIX
make run
```

## Verify Installation

After installation, you can verify it in the following ways:

### 1. Check Service Status

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

### 2. Access Dashboard

Dashboard default address: http://127.0.0.1:9180/ui

### 3. Use Admin API

```bash
# Get route list
curl -s http://127.0.0.1:9180/apisix/admin/routes

# Create a sample route
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

## Configuration File说明

Main configuration file `config.yaml`:

```yaml
apisix:
  node_listen: 9080
  enable_admin: true
  admin_key:
    - name: admin
      key: your-admin-key
      role: admin

etcd:
  host:
    - "http://127.0.0.1:2379"
  prefix: /apisix
```

## Common Issues

### 1. Port Conflicts

If ports 9080 or 9443 are occupied, modify the port configuration in `config.yaml`.

### 2. etcd Connection Failed

Confirm that etcd is running and check if the `etcd.host` configuration is correct.

### 3. Performance Issues

- Adjust worker processes: `nginx.worker_processes`
- Increase connections: `nginx.worker_connections`
