# Guía de Instalación de Hadoop

## Requisitos

Antes de instalar Hadoop, asegúrese de que su sistema cumpla con los requisitos básicos de hardware y software. Los clústeres de Hadoop pueden constar de múltiples computadoras o ejecutarse en modo pseudo-distribuido. Diferentes modos de implementación tienen diferentes requisitos de hardware, pero las dependencias de software centrales son similares. Comprender estos requisitos ayuda a planificar la escala y configuración del clúster de Hadoop, evitando problemas innecesarios durante la instalación.

En cuanto al hardware, los clústeres de prueba pequeños deben tener al menos 4 núcleos de CPU, 8GB de RAM y 500GB de disco duro por máquina. Para entornos de producción, cada máquina debe tener 8 o más núcleos de CPU, 16GB o más de RAM, y 1TB o más de disco duro. Los DataNodes de Hadoop almacenan grandes cantidades de datos, por lo que la capacidad del disco es especialmente importante. Hadoop recomienda configuración RAID o configuración JBOD (Just a Bunch Of Disks): esta última permite a Hadoop administrar el espacio en disco de manera más flexible.

En cuanto a los sistemas operativos, Hadoop soporta Linux (Ubuntu, CentOS, Debian, Red Hat, etc.) y Windows (a través de Cygwin o Windows Subsystem for Linux). Los entornos de producción recomiendan fuertemente Linux para el mejor rendimiento y estabilidad. Hadoop también se integra bien con varias plataformas de nube y puede implementarse en AWS, Azure, Google Cloud y otros servicios en la nube.

Java es la única dependencia de tiempo de ejecución de Hadoop. Hadoop 3.x requiere Java 8 o superior. Se recomienda Oracle JDK u OpenJDK. Asegúrese de que Java esté instalado correctamente y que la variable de entorno JAVA_HOME esté configurada antes de instalar Hadoop. Verifique la instalación de Java con:

```bash
java -version
echo $JAVA_HOME
```

La red es fundamental para el funcionamiento del clúster de Hadoop. Todos los nodos del clúster deben poder comunicarse entre sí a través de la red, preferiblemente usando redes de alta velocidad (como Gigabit Ethernet). Los nodos necesitan SSH sin contraseña configurado para que Hadoop pueda distribuir tareas y transferir datos entre nodos. Además, configure mapeos de nombre de host y dirección IP para todos los nodos en /etc/hosts.

## Pasos de Instalación

### Instalación en Modo Autónomo

El modo autónomo es el método de instalación más simple de Hadoop, con todos los componentes ejecutándose en el mismo proceso JVM. Este modo se usa principalmente para desarrollo y prueba y no implica almacenamiento o computación distribuida. Primero, descargue la distribución binaria de Hadoop del sitio web oficial de Apache Hadoop:

```bash
# Descargar Hadoop (reemplazar con la última versión)
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# Extraer al directorio especificado
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# Renombrar directorio (opcional)
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

A continuación, configure las variables de entorno. Edite el archivo ~/.bashrc o /etc/profile y agregue lo siguiente:

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # Ajustar según la ruta real de Java
```

Aplique la configuración:

```bash
source ~/.bashrc
```

Verifique la instalación:

```bash
hadoop version
```

### Instalación en Modo Pseudo-Distribuido

El modo pseudo-distribuido simula un entorno distribuido en una sola máquina, con NameNode, DataNode, ResourceManager y NodeManager ejecutándose todos en la misma máquina. Este modo es adecuado para desarrollar y probar funcionalidad distribuida.

Primero, configure los archivos centrales de Hadoop. Navegue al directorio de configuración de Hadoop:

```bash
cd $HADOOP_HOME/etc/hadoop
```

Configure core-site.xml, especificando HDFS como el sistema de archivos predeterminado:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Configure hdfs-site.xml, especificando el factor de replicación de HDFS (1 para pseudo-distribuido):

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

Configure mapred-site.xml, especificando el marco de programación de recursos para MapReduce:

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

Configure yarn-site.properties, configurando propiedades relacionadas con YARN:

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### Configuración de SSH sin Contraseña

Los nodos en el clúster de Hadoop necesitan SSH sin contraseña configurado para que el nodo maestro pueda acceder a los nodos esclavos sin contraseñas:

```bash
# Generar par de claves SSH
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# Agregar clave pública a authorized_keys
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# Establecer permisos correctos
chmod 0600 ~/.ssh/authorized_keys

# Probar inicio de sesión SSH
ssh localhost
```

Para clústeres de múltiples nodos, copie la clave pública a todos los nodos:

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### Formatear e Iniciar HDFS

Antes de usar HDFS por primera vez, formatee el NameNode:

```bash
hdfs namenode -format
```

Inicie HDFS y YARN:

```bash
start-dfs.sh
start-yarn.sh
```

O use start-all.sh para iniciar todos los servicios a la vez:

```bash
start-all.sh
```

Verifique el estado de inicio del servicio:

```bash
# Ver procesos Java
jps

# Procesos esperados:
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Acceda a los servicios de Hadoop a través de la interfaz web: NameNode en http://localhost:9870, ResourceManager en http://localhost:8088.

## Instalación de Clúster Completamente Distribuido

### Planificación del Clúster

El modo completamente distribuido requiere múltiples máquinas físicas o virtuales para formar un clúster. Antes de la instalación, realice una planificación razonable del clúster. Determine la escala del clúster (número de nodos) y asigne roles a cada nodo (NameNode, Secondary NameNode, DataNode, ResourceManager, NodeManager). Para clústeres pequeños, NameNode y ResourceManager pueden implementarse en la misma máquina; para clústeres grandes, se recomienda implementarlos por separado para un mejor rendimiento.

La planificación de red también es importante. Todos los nodos deben estar en la misma red interna con baja latencia y alto ancho de banda. Configure nombres de host y direcciones IP para todos los nodos en /etc/hosts:

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### Modificaciones de Archivos de Configuración

En el modo completamente distribuido, se necesitan modificar más archivos de configuración. Primero configure el archivo workers (archivo slaves en versiones anteriores), listando todos los nodos DataNode y NodeManager:

```
slave1
slave2
slave3
```

La configuración de core-site.xml y hdfs-site.xml es similar al modo pseudo-distribuido, pero se necesitan agregar más propiedades. Para la configuración de Alta Disponibilidad de NameNode, configure dos NameNodes:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Configure hdfs-site.xml para NameNode HA:

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

La configuración de YARN es similar; configure ResourceManager HA en yarn-site.xml.

### Sincronizar Archivos de Configuración

Sincronice el directorio de instalación de Hadoop configurado a todos los nodos. Copie individualmente usando scp:

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

O use rsync para replicación distribuida:

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### Iniciar el Clúster

Formatee el NameNode en el nodo NameNode (primer inicio):

```bash
hdfs namenode -format
```

Inicie HDFS:

```bash
start-dfs.sh
```

Inicie YARN:

```bash
start-yarn.sh
```

Verifique el estado de los procesos en todos los nodos para asegurar que los servicios se estén ejecutando correctamente:

```bash
# En NameNode
jps

# En DataNode
jps
```

## Verificación y Pruebas

### Prueba de HDFS

Cree directorios y suba archivos para probar HDFS:

```bash
# Crear directorio
hdfs dfs -mkdir -p /user/test

# Subir archivo local a HDFS
hdfs dfs -put localfile.txt /user/test/

# Listar archivos
hdfs dfs -ls /user/test

# Leer archivo
hdfs dfs -cat /user/test/localfile.txt

# Eliminar archivo
hdfs dfs -rm /user/test/localfile.txt
```

### Prueba de MapReduce

Ejecute programas de ejemplo de MapReduce para verificar la funcionalidad del clúster:

```bash
# Ejecutar ejemplo WordCount
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# Ver resultados
hdfs dfs -cat /output/part-r-00000
```

### Verificación de Salud del Clúster

Use los comandos integrados de Hadoop para verificar el estado del clúster:

```bash
# Ver estado de HDFS
hdfs dfsadmin -report

# Ver estado de nodos YARN
yarn node -list -all

# Ver estado de aplicaciones
yarn application -list
```

## Problemas Comunes

### Incompatibilidad de Versión de Java

Si encuentra errores relacionados con la versión de Java, asegúrese de que la versión de Java instalada sea compatible con la versión de Hadoop. Hadoop 3.x requiere Java 8 o superior. Cambie la versión de Java con:

```bash
update-alternatives --config java
```

### Espacio en Disco Insuficiente

Los DataNodes pueden no funcionar correctamente debido a espacio en disco insuficiente. Monitoree regularmente el uso del disco para asegurar que los DataNodes tengan suficiente espacio disponible. Configure dfs.datanode.fsdataset.volume.choosing.policy en hdfs-site.xml como AvailableSpaceVolumeChoosingPolicy para permitir que los DataNodes prefieran discos con más espacio disponible.

### Memoria Insuficiente

Si los nodos del clúster tienen memoria insuficiente, YARN puede no iniciar contenedores. Ajuste la configuración de memoria en yarn-site.xml:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
