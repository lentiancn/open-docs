# Node.js 简介

Node.js® 是一个免费、开源、跨平台的 JavaScript 运行时环境，它让开发人员能够创建服务器、Web 应用、命令行工具和脚本。

## 核心概念

### 什么是 Node.js？

Node.js 是一个基于 Chrome V8 JavaScript 引擎构建的 JavaScript 运行时环境。它使用事件驱动、非阻塞 I/O 模型，使其轻量且高效。Node.js 的包生态系统 npm 是世界上最大的开源库生态系统。

### 设计理念

Node.js 的核心设计理念是**非阻塞 I/O**和**事件驱动架构**。与传统的多线程并发模型不同，Node.js 使用单线程事件循环来处理并发请求，这使得它在处理大量并发连接时非常高效。

## 核心特性

### 1. 事件驱动架构

Node.js 将事件循环作为运行时构造而不是库。这意味着：

- 执行完输入脚本后，Node.js 自动进入事件循环
- 当没有更多的回调要执行时，Node.js 退出事件循环
- 这种行为类似于浏览器中的 JavaScript

### 2. 非阻塞 I/O

Node.js 几乎没有任何函数直接执行阻塞 I/O 操作（除了同步方法）。这意味着：

- 进程永远不会因为等待 I/O 操作而阻塞
- 可以同时处理数千个并发连接
- 避免了传统多线程模型中的死锁问题

### 3. 单线程模型

虽然 Node.js 使用单线程处理请求，但它可以通过以下方式利用多核处理器：

- **child_process.fork()**: 创建子进程
- **cluster 模块**: 在多个核心上负载均衡

### 4. HTTP 支持

HTTP 在 Node.js 中是第一类公民，设计时考虑了流式传输和低延迟。这使得 Node.js 非常适合作为 Web 库或框架的基础。

## 基本示例

### Hello World Web 服务器

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

### 特点说明

- **createServer()**: 创建新的 HTTP 服务器
- **request 对象**: 包含请求详细信息（headers、data 等）
- **response 对象**: 用于向客户端返回数据
- **statusCode**: 设置 HTTP 状态码（200 表示成功）
- **setHeader()**: 设置响应头
- **end()**: 结束响应并发送内容

## 适用场景

### 适合的场景

- **实时应用**: 聊天应用、实时通知、在线游戏
- **API 服务器**: RESTful API、微服务
- **数据流应用**: 视频流、音频流、文件上传/下载
- **单页应用 (SPA)**: 前后端分离的现代 Web 应用
- **工具和脚本**: 命令行工具、自动化脚本

### 不适合的场景

- **CPU 密集型任务**: 大量计算、图像处理、视频编码
- **关系型数据库密集型应用**: 复杂的 SQL 查询和事务

## 生态系统

### npm (Node Package Manager)

- 世界上最大的软件注册表
- 超过 200 万个可用包
- 简单的依赖管理和版本控制

### 主流框架和库

- **Express.js**: 最流行的 Web 应用框架
- **Koa**: 由 Express 团队开发的下一代框架
- **Fastify**: 高性能 Web 框架
- **NestJS**: 基于 TypeScript 的渐进式框架
- **Socket.IO**: 实时双向通信
