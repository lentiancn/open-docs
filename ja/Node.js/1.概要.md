# Node.js 概要

Node.js®は、クロスプラットフォームに対応したフリーでオープンソースのJavaScript実行環境です。開発者にサーバー、ウェブアプリ、コマンドラインツール、スクリプトなどを開発する環境を提供します。

## コア機能

非同期イベント駆動型のJavaScriptランタイムとして、Node.jsはスケーラブルなネットワークアプリケーションを構築するために設計されています。古典的な「hello world」の例では、多数の接続を同時に処理できます。各接続時にコールバックが起動されますが、処理すべき作業がない場合、Node.jsはスリープ状態になります。

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

## 設計哲学

これは、今日より一般的なOSスレッドを採用した並行モデルとは対照的です。スレッドベースのネットワーキングは比較的非効率的で、非常に使いにくいものです。さらに、Node.jsのユーザーはプロセスのデッドロックを心配する必要がありません。なぜなら、ロックが存在しないからです。

Node.jsでは、Node.js標準ライブラリの同期メソッドを使用してI/Oを実行しない限り、I/Oを直接実行する関数はほとんどありません。そのため、プロセスはブロッキングしません。何もブロッキングしないため、Node.jsでスケーラブルなシステムを開発するのは非常に合理的です。

## イベント駆動アーキテクチャ

Node.jsは、RubyのEvent MachineやPythonのTwistedなどのシステムと似た設計を持ち、それらに影響を受けています。Node.jsはイベントモデルをさらに一歩進め、イベントループをライブラリではなくランタイム構成として提示します。

他のシステムでは、常にイベントループを開始するためのブロッキング呼び出しがあります。通常、スクリプトの先頭でコールバックを通じて動作が定義され、最後に`EventMachine::run()`のようなブロッキング呼び出しでサーバーが開始されます。

Node.jsには、そのようなイベントループを開始する呼び出しがありません。Node.jsは入力スクリプトを実行した後、単純にイベントループに入ります。実行すべきコールバックがなくなると、Node.jsはイベントループを終了します。この動作はブラウザのJavaScriptに似ており、イベントループはユーザーから隠されています。

## HTTPサポート

HTTPはNode.jsにおいてファーストクラスの市民であり、ストリーミングと低遅延を念頭に置いて設計されています。これにより、Node.jsはWebライブラリやフレームワークの基盤として非常に適しています。

## マルチコアプロセッササポート

スレッドを使用せずに設計されているNode.jsですが、環境内の複数のコアを活用できないわけではありません。`child_process.fork()` APIを使用して子プロセスを生成でき、これらは通信が簡単になるように設計されています。

その同じインターフェース上に構築された`cluster`モジュールを使用すると、プロセス間でソケットを共有し、コア間で負荷分散を実現できます。
