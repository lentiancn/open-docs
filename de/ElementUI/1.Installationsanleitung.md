# Element UI Installationsanleitung

Element UI ist eine UI-Komponentenbibliothek für Vue 2.x Anwendungen.

## Versionsübersicht

| Version | Vue-Version | Status | Veröffentlichungsdatum |
|---------|-------------|--------|----------------------|
| Element UI 2.x | Vue 2.6+ | Aktuell | 2020 |
| Element UI 1.x | Vue 2.x | Veraltet | 2016 |

**Hinweis:** Element UI ist nur für Vue 2. Für Vue 3 verwenden Sie Element Plus.

### Element UI vs Element Plus

| Funktion | Element UI (Vue 2) | Element Plus (Vue 3) |
|----------|-------------------|---------------------|
| Vue-Version | 2.x | 3.x |
| TypeScript | Begrenzt | Vollständige Unterstützung |
| Status | Wartungsmodus | Aktive Entwicklung |

## Installationsmethoden

### Methode 1: npm (Empfohlen)

```bash
# Element UI installieren
npm install element-ui --save
```

### Methode 2: Vue CLI verwenden

```bash
# Vue CLI installieren
npm install -g @vue/cli

# Projekt mit Element UI-Vorlage erstellen
vue init webpack my-element-app
```

### Methode 3: CDN

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Element UI CDN Demo</title>
  <!-- Vue 2 importieren -->
  <script src="https://unpkg.com/vue@2.6.14/dist/vue.js"></script>
  <!-- Element UI-Stile importieren -->
  <link rel="stylesheet" href="https://unpkg.com/element-ui/lib/theme-chalk/index.css">
  <!-- Element UI importieren -->
  <script src="https://unpkg.com/element-ui/lib/index.js"></script>
</head>
<body>
  <div id="app">
    <el-button @click="visible = true">Dialog öffnen</el-button>
    <el-dialog :visible.sync="visible" title="Hello Element UI">
      <p>Willkommen bei Element UI!</p>
    </el-dialog>
  </div>

  <script>
    new Vue({
      el: '#app',
      data: {
        visible: false
      }
    })
  </script>
</body>
</html>
```

## Projekteinrichtung

### Schritt 1: Vue-Projekt erstellen

```bash
# Vue CLI verwenden
npm install -g @vue/cli
vue create my-element-app
cd my-element-app
```

### Schritt 2: Element UI installieren

```bash
npm install element-ui --save
```

### Schritt 3: In main.js importieren

#### Vollständiger Import

```javascript
import Vue from 'vue'
import App from './App.vue'

// Element UI importieren
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'

// Element UI verwenden
Vue.use(ElementUI)

Vue.config.productionTip = false

new Vue({
  render: h => h(App)
}).$mount('#app')
```

#### Bedarfsweiser Import

Zuerst das Plugin installieren:

```bash
npm install babel-plugin-component -D
```

babel.config.js konfigurieren:

```javascript
module.exports = {
  presets: ['@vue/cli-plugin-babel/preset'],
  plugins: [
    [
      'component',
      {
        libraryName: 'element-ui',
        styleLibraryName: 'theme-chalk'
      }
    ]
  ]
}
```

Dann Komponenten importieren:

```javascript
import { Button, Dialog, Input } from 'element-ui'

Vue.component(Button.name, Button)
Vue.component(Dialog.name, Dialog)
Vue.component(Input.name, Input)

// Oder lokal in Komponenten verwenden
export default {
  components: {
    [Button.name]: Button
  }
}
```

## Fehlerbehebung

### Komponente nicht gefunden

**Problem:** Komponenten werden nicht gerendert

**Lösungen:**
1. Sicherstellen, dass es installiert ist: `npm list element-ui`
2. Import in main.js überprüfen
3. Überprüfen, ob Vue.use() aufgerufen wurde

### Stile werden nicht angewendet

**Problem:** Stile fehlen

**Lösungen:**
1. CSS importieren: `import 'element-ui/lib/theme-chalk/index.css'`
2. Webpack CSS-Loader-Konfiguration überprüfen

### IE10-Kompatibilität

```bash
# Polyfills installieren
npm install babel-polyfill --save
```

```javascript
// main.js
import 'babel-polyfill'
import Vue from 'vue'
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'

Vue.use(ElementUI)
```

## Migration zu Element Plus (Vue 3)

Wenn Sie zu Vue 3 migrieren, verwenden Sie stattdessen Element Plus:

```bash
npm install element-plus --save
```

Wichtige Änderungen:
1. Paketname: `element-ui` → `element-plus`
2. Importpfad: `element-ui/lib/index` → `element-plus`
3. Einige Komponentenprops haben sich möglicherweise geändert

## Best Practices

1. **Bedarfsweisen Import verwenden** - Reduziert die Bundle-Größe
2. **Thema anpassen** - SCSS-Variablen verwenden
3. **Vue 2.6+ verwenden** - Bessere Kompatibilität
4. **Babel konfigurieren** - Für Produktions-Builds
