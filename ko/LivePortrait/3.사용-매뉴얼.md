# LivePortrait 사용 매뉴얼

이 문서에서는 기본 추론, Gradio 인터페이스, 동물 모드, 비디오 편집 등을 포함한 LivePortrait의 다양한 사용 방법에 대한상세 지침을 제공합니다.

## 기본 추론

### 이미지 구동

가장 기본적인 사용법은 정적 이미지를 소스로, 구동 비디오를 애니메이션 참조로 사용하는 것입니다:

```bash
python inference.py
```

기본적으로 스크립트는 assets/examples/source 디렉터리의 샘플 이미지와 assets/examples/driving 디렉터리의 샘플 구동 비디오를 사용합니다. 성공적으로 실행되면 결과는 "소스파일--구동파일_concat.mp4"라는 이름으로 animations 디렉터리에 저장됩니다.

출력 비디오에는 세 가지 부분이 순서대로 연결되어 있습니다: 구동 비디오, 소스 이미지/비디오, 생성된 애니메이션 결과.

### 사용자 정의 입력

명령줄 매개변수를 사용하여 자체 소스 및 구동 파일을 지정할 수 있습니다:

```bash
# 소스 파일이 이미지인 경우
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# 소스 파일이 비디오인 경우
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

-s 매개변수는 소스 이미지 또는 비디오 경로를 지정하고, -d 매개변수는 구동 비디오 경로를 지정합니다.

### 추가 옵션 보기

사용 가능한 모든 명령줄 매개변수 보기:

```bash
python inference.py -h
```

매개변수에는 출력 경로, 이미지 크기, 애니메이션 속도, 연결 방법 등의 상세 구성이 포함됩니다.

## Gradio 웹 인터페이스 사용

명령줄 인터페이스에 익숙하지 않은 사용자를 위해 LivePortrait는 Gradio 기반 그래픽 인터페이스를 제공합니다.

### 휴먼 모드 인터페이스 시작

```bash
python app.py
```

시작 후 표시된 URL(일반적으로 http://localhost:7860)을 브라우저에서 엽니다. 인터페이스에서 소스 이미지 또는 비디오를 업로드한 다음 구동 비디오를 선택하고 생성 버튼을 클릭하면 애니메이션 결과가 표시됩니다.

--server_port 매개변수를 사용하여 포트를 지정하고, --share 매개변수를 사용하여 공개 링크를 생성하고, --server_name 매개변수를 사용하여 서버 주소를 지정할 수 있습니다.

### 동물 모드 인터페이스 시작

NVIDIA GPU가 있는 Linux 시스템에서 동물 모드 인터페이스를 시작할 수 있습니다:

```bash
python app_animals.py
```

## 구동 비디오 사양

최상의 결과를 얻으려면 구동 비디오가 다음 요구 사항을 충족해야 합니다:

**화면 비율**: 1:1 정사각형(예: 512x512 또는 256x256 픽셀)으로 자르는 것이 좋습니다. 자동 자르기 기능(--flag_crop_driving_video)을 활성화할 수도 있습니다.

**콘텐츠 요구 사항**: 구동 비디오는 샘플 비디오와 유사하게 머리 영역에 집중해야 합니다. 어깨 움직임은 최소화해야 합니다.

**첫 번째 프레임 요구 사항**: 구동 비디오의 첫 번째 프레임은 중립적인 표정의 정면 얼굴이어야 합니다. 이는 정확한 애니메이션 효과를 보장하는 데 중요합니다.

### 자동 자르기

구동 비디오가 위의 요구 사항을 충족하지 않으면 자동 자르기 기능을 사용할 수 있습니다:

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

자동 자르기 결과가 만족스럽지 않으면 --scale_crop_driving_video 및 --vy_ratio_crop_driving_video 매개변수를 조정하여 배율과 오프셋을 미세 조정할 수 있습니다.

## 모션 템플릿 사용

추론을 가속화하고 개인정보를 보호하기 위해 LivePortrait는 사전 생성된 모션 템플릿 파일(.pkl 형식)의 사용을 지원합니다. 이러한 템플릿 파일은 구동 비디오의 모션 정보를 기록하며 매번 원본 비디오를 처리하지 않고 재사용할 수 있습니다.

### 사용법

```bash
# 초상화 애니메이션
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# 초상화 비디오 편집
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

템플릿 파일은 추론을 가속화할 뿐만 아니라 개인정보도 보호합니다—템플릿을 사용하면 원본 구동 비디오를 업로드할 필요가 없습니다.

## 초상화 비디오 편집 (Video-to-Video)

LivePortrait는 기존 비디오에서 인물의 표정과 머리 움직임을 편집하고 수정하는 것을 지원합니다.

### 기본 사용법

```bash
# 소스는 이미지, 구동은 비디오
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### 애니메이션 매개변수 조정

**표정 강도**: --driving_multiplier 매개변수를 사용하여 표정 변화의 강도를 조정합니다. 1보다 큰 값은 표정 변화를 강화하고 1보다 작은 값은 약화합니다.

**연결 제어**: --flag_stitching 매개변수를 사용하여 연결이 활성화되었는지를 제어합니다. 연결을 비활성화하면 얼굴 영역 표시 방식이 변경될 수 있습니다.

## 추론 가속화

### Torch Compile 가속

LivePortrait는 추론을 가속화하기 위해 PyTorch의 torch.compile 기능 사용을 지원합니다. 첫 번째 추론은 최적화 프로세스(약 1분)를 트리거하며 그 후 추론 속도가 20-30% 향상됩니다. 가속 효과는 CUDA 버전에 따라 다를 수 있습니다.

```bash
python app.py --flag_do_torch_compile
```

참고: torch.compile은 Windows 및 macOS에서 지원되지 않습니다.

### 속도 테스트

프로젝트는 추론 속도 테스트 스크립트를 제공합니다:

```bash
python speed.py
```

테스트 결과는 각 모듈의 처리 시간을 보여주어 성능 병목 지점을 파악하는 데 도움이 됩니다.

## 동물 모드

동물 모드는 고양이와 강아지 애완동물 초상화의 애니메이션을 지원합니다.

### 사용 조건

동물 모드는 다음 환경에서만 테스트되었습니다: NVIDIA GPU가 있는 Linux 시스템, NVIDIA GPU가 있는 Windows 시스템.

### 설치 요구 사항

동물 모드를 사용하기 전에 설치 가이드에 설명된 대로 X-Pose 종속성을 설치하고 동물 모델 가중치를 다운로드하세요.

### 동물 모드 추론 실행

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### 동물 모드 인터페이스

```bash
python app_animals.py
```

## 지역 제어

LivePortrait는 이미지의 특정 영역에 대한 애니메이션 제어를 지원하여 더 세밀한 효과 조정을 가능.

### 지역 제어 활성화

Gradio 인터페이스에서 지역 제어 옵션을 찾아 애니메이션할 얼굴 영역(예: 눈,, 머리 자세 등)을 선택할 수 있습니다.

## 초상화 편집

Gradio 인터페이스에서 사용자는 표정, 시선 방향, 머리 각도 등을 포함한 소스 초상화의 다양한 속성을 정밀하게 편집할 수 있습니다.

## 개인정보 보호

LivePortrait는 여러 가지 개인정보 보호 기능을 제공합니다:

**모션 템플릿**: 민감한 콘텐츠 업로드를 피하기 위해 원본 구동 비디오 대신 .pkl 템플릿 파일 사용

**자동 자르기**: 구동 비디오를 자동으로 잘라 머리에 집중시킴으로써 배경 정보 감소

**비디오 보호**: 템플릿 생성 기능은 사용 전에 비디오를 모션 템플릿으로 변환합니다

## 출력 형식

기본 출력 형식은 세 부분(구동 비디오, 소스 콘텐츠, 생성 결과)을 포함하는 MP4 비디오 파일입니다. --no_flag_concat 매개변수를 사용하면 생성된 애니메이션 부분만 출력할 수 있습니다.

## 커뮤니티 도구 통합

LivePortrait는 여러 커뮤니티 프로젝트에 통합되어 있습니다:

**ComfyUI**: 실시간 미리보기 기능이 있는 여러 ComfyUI 노드가 LivePortrait를 지원합니다

**Stable Diffusion WebUI**: sd-webui-live-portrait 확장이 LivePortrait를 SD WebUI에 통합합니다

**FaceFusion**: 버전 3.0은 LivePortrait를 표정 복원 및 얼굴 편집 프로세서로 통합합니다

**HuggingFace Spaces**: 로컬 설치 없이 온라인에서 LivePortrait를 경험할 수 있습니다

## 모범 사례

최상의 애니메이션 효과를 얻으려면 다음 모범 사례를 따르세요:

고품질 소스 이미지 선택: 명확한 정면 초상화가 가장 좋은 결과를 산출합니다

전문적인 구동 비디오 사용: 정면이고 자연스러운 표정의 비디오가 더 나은 효과를 산출합니다

구동 비디오 길이 조절: 지나치게 긴 구동 비디오는 처리 시간을 증가시키고 효과가 저하될 수 있습니다

매개변수를 합리적으로 설정: 실제 결과에 따라 표정 강도 및 기타 매개변수를 조정하세요

일관된 조명 유지: 소스 이미지와 구동 비디오 간의 조명 조건을 일관되게 유지하려고 노력하세요
