# MongoDB ユーザーマニュアル

## 基本概念

### データベース

データベースは MongoDB で最も高いレベルのデータコンテナです。各データベースには独自のコレクションと権限設定があります。MongoDB で複数のデータベースを作成し、それぞれ独立させることができます。

### コレクション

コレクションは MongoDB ドキュメントのグループで、関係データベースのテーブルに相当します。コレクション内のドキュメントは異なるフィールドを持つことができ、コレクションは非常に柔軟です。関係データベースとは異なり、コレクションは厳密に事前定義されたスキーマを強制しません。

### ドキュメント

ドキュメントは MongoDB の基本的なデータユニットで、フィールドと値のペアで構成されます。ドキュメントは JSON オブジェクトに類似し、BSON（バイナリ JSON）形式で保存されます。

BSON は JSON ドキュメントのバイナリ表現で、追加のデータ型をサポートします。ドキュメントサイズの最大치는 16 MB입니다.

## ドキュメントの挿入

MongoDB はドキュメントをコレクションに挿入するための次の方法を提供します。

### 単一ドキュメントを挿入する

```bash
db.users.insertOne({
  name: "田中",
  age: 30,
  email: "tanaka@example.com",
  address: {
    city: "東京",
    street: "新宿"
  },
  hobbies: ["サッカー", "水泳"]
})
```

### 複数ドキュメントを挿入する

```bash
db.users.insertMany([
  { name: "山田", age: 25, city: "大阪" },
  { name: "鈴木", age: 35, city: "名古屋" },
  { name: "佐藤", age: 28, city: "福岡" }
])
```

### ドキュメントを挿入する（旧式だが使用可能）

```bash
db.users.insert({
  name: "高橋",
  age: 32
})
```

### 挿入結果を表示する

```bash
# 挿入されたドキュメント ID を表示
db.users.insertOne({ name: "テスト" })
# 返り値: { acknowledged: true, insertedId: ObjectId("...") }
```

## ドキュメントのクエリ

### すべてのドキュメントをクエリする

```bash
db.users.find()
```

SQL の `SELECT * FROM users` に相当します。

### 単一ドキュメントをクエリする

```bash
db.users.findOne()
```

### 条件付きクエリ

```bash
# 等価クエリ
db.users.find({ age: 30 })

# 不等価クエリ
db.users.find({ age: { $ne: 30 } })

# より大きいクエリ
db.users.find({ age: { $gt: 25 } })

# 以上クエリ
db.users.find({ age: { $gte: 25 } })

# 未満クエリ
db.users.find({ age: { $lt: 30 } })

# 以下クエリ
db.users.find({ age: { $lte: 30 } })

# 範囲クエリ
db.users.find({ age: { $gte: 20, $lte: 30 } })
```

### 論理演算子

```bash
# AND（デフォルト）
db.users.find({ age: { $gte: 25 }, name: "山田" })

# OR
db.users.find({ $or: [{ age: { $gte: 30 } }, { name: "鈴木" }] })

# NOR
db.users.find({ $nor: [{ age: { $lt: 20 } }, { city: "東京" }] })

# NOT
db.users.find({ age: { $not: { $gte: 25 } } })
```

### 配列クエリ

```bash
# 配列が要素を含む
db.users.find({ hobbies: "サッカー" })

# 配列がすべての要素を含む
db.users.find({ hobbies: { $all: ["サッカー", "水泳"] } })

# 配列の長さ
db.users.find({ hobbies: { $size: 2 } })

# 配列の特定位置的要素
db.users.find({ "hobbies.0": "サッカー" })

# 配列要素が条件を満たす
db.users.find({ hobbies: { $elemMatch: { $eq: "サッカー" } } })
```

### ネストドキュメントのクエリ

```bash
# ネストドキュメントフィールドのクエリ
db.users.find({ "address.city": "東京" })

# ネストドキュメントの完全一致
db.users.find({ address: { city: "東京", street: "新宿" } })
```

### プロジェクション（指定したフィールドのみを返す）

```bash
# name と email フィールドのみを返す
db.users.find({ age: 30 }, { name: 1, email: 1 })

# フィールドを除外
db.users.find({}, { age: 0 })

# _id 以外のすべてのフィールドを返す
db.users.find({}, { _id: 0 })
```

### ソート

```bash
# 年齢順に昇ソート
db.users.find().sort({ age: 1 })

# 年齢順に降ソート
db.users.find().sort({ age: -1 })

# 複数フィールドでソート
db.users.find().sort({ age: -1, name: 1 })
```

### リミットとスキップ

```bash
# 先頭5件を返す
db.users.find().limit(5)

# 先頭10件をスキップして次の5件を返す（ページネーション）
db.users.find().skip(10).limit(5)

# ドキュメント数をカウント
db.users.countDocuments({ age: { $gte: 25 } })

# コレクションのドキュメント数を推定（速いが正確でない場合あり）
db.users.estimatedDocumentCount()
```

### カーソル操作

```bash
# カーソルを使用
var cursor = db.users.find()
while (cursor.hasNext()) {
  printjson(cursor.next())
}

# forEach で反復
db.users.find().forEach(function(doc) {
  print("Name: " + doc.name)
})
```

## ドキュメントの更新

### 単一ドキュメントを更新する

```bash
db.users.updateOne(
  { name: "田中" },
  { $set: { age: 31 } }
)
```

### 複数ドキュメントを更新する

```bash
db.users.updateMany(
  { age: { $lt: 30 } },
  { $set: { status: "young" } }
)
```

### ドキュメントを置換する

```bash
db.users.replaceOne(
  { name: "田中" },
  { name: "田中", age: 32, email: "new@example.com", city: "札幌" }
)
```

### 更新演算子

```bash
# $set - フィールド値を設定
db.users.updateOne({ name: "山田" }, { $set: { age: 26 } })

# $unset - フィールドを削除
db.users.updateOne({ name: "山田" }, { $unset: { email: "" } })

# $inc - 増減
db.users.updateOne({ name: "山田" }, { $inc: { age: 1 } })
db.users.updateOne({ name: "山田" }, { $inc: { age: -1 } })

# $rename - フィールド名を変更
db.users.updateOne({ name: "山田" }, { $rename: { "name": "fullname" } })

# $mul - 乗算
db.users.updateOne({ name: "山田" }, { $mul: { age: 2 } })

# $min - 最小値を設定
db.users.updateOne({ name: "山田" }, { $min: { age: 20 } })

# $max - 最大値を設定
db.users.updateOne({ name: "山田" }, { $max: { age: 40 } })

# $push - 配列に要素を追加
db.users.updateOne({ name: "山田" }, { $push: { hobbies: "ランニング" } })

# $pull - 配列から一致する要素を削除
db.users.updateOne({ name: "山田" }, { $pull: { hobbies: "水泳" } })

# $addToSet - 配列に要素を追加（重複なし）
db.users.updateOne({ name: "山田" }, { $addToSet: { hobbies: "サッカー" } })

# $pop - 配列の最初または最後の要素を削除
db.users.updateOne({ name: "山田" }, { $pop: { hobbies: 1 } })
```

### upsert 操作

```bash
# 存在しない場合は挿入、存在する場合は更新
db.users.updateOne(
  { name: "新規ユーザー" },
  { $set: { age: 25, city: "東京" } },
  { upsert: true }
)
```

## 集約パイプライン

集約パイプラインは、MongoDB の最も強力な機能の1つで、データフローを処理し、複雑なデータ変換和分析を実行します。

### 基本構文

```bash
db.collection.aggregate([
  { $stage1: { ... } },
  { $stage2: { ... } },
  ...
])
```

### 一般的な集約ステージ

#### $match - ドキュメントをフィルタ

```bash
db.users.aggregate([
  { $match: { age: { $gte: 25 } } }
])
```

#### $project - プロジェクション

```bash
db.users.aggregate([
  { $project: { _id: 0, name: 1, age: 1 } }
])
```

#### $group - グループ化

```bash
# 年齢별ユーザー数をカウント
db.users.aggregate([
  { $group: { _id: "$age", count: { $sum: 1 } } }
])

# 平均年齢を計算
db.users.aggregate([
  { $group: { _id: null, avgAge: { $avg: "$age" } } }
])

# グループ化して名前を収集
db.users.aggregate([
  { $group: { _id: "$city", users: { $push: "$name" } } }
])
```

#### $sort - ソート

```bash
db.users.aggregate([
  { $sort: { age: -1, name: 1 } }
])
```

#### $limit - 件数制限

```bash
db.users.aggregate([
  { $limit: 5 }
])
```

#### $skip - ドキュメントをスキップ

```bash
db.users.aggregate([
  { $skip: 10 },
  { $limit: 5 }
])
```

#### $unwind - 配列を展開

```bash
db.users.aggregate([
  { $unwind: "$hobbies" }
])
```

#### $lookup - コレクション結合

```bash
# 2つのコレクションがあると仮定: orders と products
db.orders.aggregate([
  {
    $lookup: {
      from: "products",
      localField: "productId",
      foreignField: "_id",
      as: "productDetails"
    }
  }
])
```

### 集約演算子

#### 算術演算子

```bash
db.sales.aggregate([
  {
    $project: {
      total: { $add: ["$price", "$tax"] },
      discount: { $multiply: ["$price", 0.1] }
    }
  }
])
```

#### 文字列演算子

```bash
db.users.aggregate([
  {
    $project: {
      fullName: { $concat: ["$firstName", " ", "$lastName"] },
      upperName: { $toUpper: "$name" }
    }
  }
])
```

#### 日付演算子

```bash
db.users.aggregate([
  {
    $project: {
      year: { $year: "$birthdate" },
      month: { $month: "$birthdate" },
      day: { $dayOfMonth: "$birthdate" }
    }
  }
])
```

### 完全な例

```bash
# PDX 空港から出発する直行便が最も多い上位3つの航空会社を見つける
db.routes.aggregate([
  {
    $match: {
      "src_airport": "PDX",
      "stops": 0
    }
  },
  {
    $group: {
      _id: "$airline.name",
      count: { $sum: 1 }
    }
  },
  {
    $sort: { count: -1 }
  },
  {
    $limit: 3
  }
])
```

## インデックス

インデックスは MongoDB でクエリを効率的に実行をサポートします。インデックスがない場合、MongoDB はクエリ結果を返すためにコレクション内のすべてのドキュメントをスキャンする必要があります。

### インデックスを作成する

```bash
# 単一フィールドインデックス
db.users.createIndex({ name: 1 })

# 降順インデックス
db.users.createIndex({ age: -1 })

# 複合インデックス
db.users.createIndex({ age: 1, name: 1 })

# 一意インデックス
db.users.createIndex({ email: 1 }, { unique: true })

# テキストインデックス
db.users.createIndex({ description: "text" })

# マルチキーインデックス（配列フィールド）
db.users.createIndex({ hobbies: 1 })

# 地理空間インデックス
db.locations.createIndex({ location: "2dsphere" })

# ハッシュインデックス
db.users.createIndex({ _id: "hashed" })
```

### インデックスを表示する

```bash
# コレクションのすべてのインデックスを表示
db.users.getIndexes()

# インデックスのサイズを表示
db.users.totalIndexSize()
```

### インデックスを削除する

```bash
# 指定したインデックスを削除
db.users.dropIndex("name_1")

# すべてのインデックスを削除（_id インデックスは保持）
db.users.dropIndexes()

# テキストインデックスを削除
db.users.dropIndex("description_text")
```

### インデックスのプロパティ

```bash
# スパースインデックス（null 値のみインデックス）
db.users.createIndex({ email: 1 }, { sparse: true })

# パーシャルインデックス
db.users.createIndex(
  { email: 1 },
  { partialFilterExpression: { email: { $exists: true } } }
)

# TTL インデックス（自動期限切れ）
db.sessions.createIndex(
  { createdAt: 1 },
  { expireAfterSeconds: 3600 }
)
```

### インデックス戦略の推奨事項

- クエリ条件で頻繁に使用されるフィールドにインデックスを作成します
- 複合インデックスのフィールド順序は重要です：完全一致フィールドを前に、範囲クエリフィールドを後に配置します
- 書き込み頻度の高いコレクションに過多的インデックスを作成しないでください
- explain() を使用してクエリパフォーマンスを分析します

### クエリ分析

```bash
# クエリを分析
db.users.find({ age: 25 }).explain()

# 実行プランを分析
db.users.find({ age: { $gte: 25 } }).sort({ name: 1 }).explain("executionStats")
```

## データのエクスポートとインポート

### mongodump と mongorestore を使用する

#### データをエクスポート

```bash
# データベース全体をエクスポート
mongodump --db=mydatabase --out=/backup

# 指定コレクションをエクスポート
mongodump --db=mydatabase --collection=users --out=/backup

# 圧縮してエクスポート
mongodump --db=mydatabase --gzip --out=/backup
```

#### データをインポート

```bash
# データベース全体をリストア
mongorestore --db=mydatabase /backup/mydatabase

# 指定コレクションをリストア
mongorestore --db=mydatabase --collection=users /backup/mydatabase/users.bson

# 圧縮をリストア
mongorestore --db=mydatabase --gzip /backup/mongodb
```

### mongoexport と mongoimport を使用する

#### JSON としてエクスポート

```bash
# JSON としてエクスポート
mongoexport --db=mydatabase --collection=users --out=users.json

# JSON 配列としてエクスポート
mongoexport --db=mydatabase --collection=users --out=users.json --jsonArray

# CSV としてエクスポート
mongoexport --db=mydatabase --collection=users --out=users.csv --fields=name,age,email
```

#### JSON からインポート

```bash
# JSON からインポート
mongoimport --db=mydatabase --collection=users --file=users.json

# JSON 配列からインポート
mongoimport --db=mydatabase --collection=users --file=users.json --jsonArray

# CSV からインポート
mongoimport --db=mydatabase --collection=users --file=users.csv --headerline
```

## トランザクション操作

MongoDB 4.0+ はマルチドキュメントトランザクションをサポートします。

### 基本的なトランザクション

```bash
session = db.getMongo().startSession()
session.startTransaction({ readConcern: { level: "snapshot" }, writeConcern: { w: "majority" } })

try {
  session.getDatabase("mydb").accounts.updateOne(
    { accountId: 1 },
    { $inc: { balance: -100 } }
  )
  session.getDatabase("mydb").accounts.updateOne(
    { accountId: 2 },
    { $inc: { balance: 100 } }
  )
  session.commitTransaction()
} catch (error) {
  session.abortTransaction()
  throw error
} finally {
  session.endSession()
}
```

## GridFS

GridFS は 16MB 制限を超える大きなファイルを保存するために使用されます。

```bash
# ファイルをアップロード
mongofiles put myfile.txt

# ファイルを一覧表示
mongofiles list

# ファイルをダウンロード
mongofiles get myfile.txt

# ファイルを削除
mongofiles delete myfile.txt
```
