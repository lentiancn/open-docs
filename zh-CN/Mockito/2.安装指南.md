# Mockito 安装指南

## 环境要求

在安装 Mockito 之前，需要确保开发环境满足以下要求。

Mockito 5.x 版本需要 JDK 11 或更高版本。如果项目需要支持 JDK 8，可以选择 Mockito 4.x 系列。推荐使用最新版本的 Mockito 以获得最新的功能和安全修复。

## Maven 安装方式

### 添加依赖

在 Maven 项目的 pom.xml 文件中添加 mockito-core 依赖。建议使用占位符定义 Mockito 版本，便于版本统一管理。

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

如果需要使用 Mockito 的内联（inline）功能来模拟 final 类和方法，还需要添加 mockito-inline 依赖。

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### 快照版本

Mockito 每次代码变更都会发布快照版本到 Sonatype 仓库。如果需要使用最新开发中的功能，可以在 pom.xml 中添加快照仓库地址。

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Gradle 安装方式

### 添加依赖

在 Gradle 项目的 build.gradle 文件中添加依赖声明。

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

如果使用 Gradle 的 Kotlin DSL 文件（build.gradle.kts），语法稍有不同。

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### 使用 BOM 管理版本

通过 Spring Dependency Management 插件或 Gradle 的 BOM 功能，可以简化版本管理工作。

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## 手动下载安装

如果项目不使用 Maven 或 Gradle，可以直接从 Maven Central 仓库下载 JAR 文件。访问 Maven Central 搜索 org.mockito，可以找到所有版本的 mockito-core 和 mockito-all（已废弃）分发包。

下载后，将 JAR 文件添加到项目的类路径中即可。需要同时下载 Byte Buddy 和 Objenesis 等依赖库。

## 与 Spring Boot 集成

Spring Boot 的 spring-boot-starter-test 已经包含了 Mockito 依赖。如果项目使用 Spring Boot，基本不需要额外添加 Mockito 依赖。

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

如果需要在 Spring 环境中更方便地使用 Mock，可以使用 @MockBean 注解，它会自动将模拟对象注入到 Spring 容器中。

## 与 JUnit 5 集成

### 使用 MockitoExtension

JUnit 5 引入了扩展模型，Mockito 提供了 MockitoExtension 来支持 JUnit 5。无需额外的依赖配置，mockito-core 已经包含了必要的支持。

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### 使用 MockitoJUnitRunner

对于 JUnit 4，可以使用 MockitoJUnitRunner 来自动初始化 Mock 对象。

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## 验证安装

安装完成后，可以通过以下方式验证 Mockito 是否正确配置。创建一个简单的测试类，运行后如果测试通过，说明 Mockito 已正确安装。

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito 安装成功！");
    }
}
```

## 版本选择建议

对于新项目，推荐使用最新的 Mockito 5.x 版本。如果项目需要兼容 JDK 8，可以选择 Mockito 4.x 系列。建议始终使用最新的稳定版本，以获得更好的性能和安全性。
