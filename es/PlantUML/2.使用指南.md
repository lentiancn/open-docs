# Guía de Uso de PlantUML

Esta guía introduce cómo usar PlantUML para crear varios diagramas UML.

## Inicio Rápido

### Sintaxis Básica

```plantuml
@startuml
Alice -> Bob: Hello
@enduml
```

## Diagrama de Secuencia

```plantuml
@startuml
actor Usuario
participant "Frontend" as Front
participant "Backend" as Back
database "Base de Datos" as DB

Usuario -> Front: Enviar Solicitud
Front -> Back: Llamada API
Back -> DB: Consultar Datos
DB --> Back: Retornar Resultado
Back --> Front: Respuesta
Front --> Usuario: Mostrar Resultado
@enduml
```

## Diagrama de Clases

```plantuml
@startuml
class User {
  +id: int
  +username: string
  +email: string
  +login()
  +logout()
}

class Admin extends User {
  +manageUsers()
  +viewReports()
}

class Order {
  +id: int
  +total: decimal
  +createOrder()
  +cancelOrder()
}

User "1" -- "*" Order: places
@enduml
```

## Diagrama de Casos de Uso

```plantuml
@startuml
left to right direction
actor Usuario
rectangle Sistema {
  Usuario -> (Iniciar Sesión)
  Usuario -> (Explorar Productos)
  Usuario -> (Agregar al Carrito)
  Usuario -> (Realizar Pedido)
  (Iniciar Sesión) .> (Explorar Productos): extends
}
@enduml
```

## Diagrama de Estados

```plantuml
@startuml
[*] --> Pendiente de Pago
Pendiente de Pago --> Pagado: Pago Exitoso
Pagado --> Enviado: Envío
Enviado --> Entregado: Entrega
Entregado --> [*]
Pendiente de Pago --> Cancelado: Usuario Cancela
Cancelado --> [*]
@enduml
```

## Diagrama de Actividades

```plantuml
@startuml
start
:Comenzar Compra;
:Explorar Productos;
if (¿Producto Encontrado?) then (Sí)
  :Agregar al Carrito;
else (No)
  :Buscar Producto;
endif
:Finalizar Compra;
if (¿Pago Exitoso?) then (Sí)
  :Pedido Exitoso;
else (No)
  :Pago Fallido;
endif
stop
@enduml
```

## Diagrama de Componentes

```plantuml
@startuml
package "Frontend" {
  [Web App] as WA
  [Mobile App] as MA
}
package "Backend" {
  [API Gateway] as API
  [Auth Service] as Auth
  [Order Service] as Order
}
package "Datos" {
  database "MySQL" as DB
  database "Redis" as Cache
}

WA --> API
MA --> API
API --> Auth
API --> Order
Order --> DB
Order --> Cache
@enduml
```

## Diagrama de Despliegue

```plantuml
@startuml
node "Dispositivo del Usuario" as Client {
  [Navegador]
}
node "Balanceador de Carga" as LB {
  [Nginx]
}
node "Servidor de Aplicaciones" as App {
  [Tomcat 1]
  [Tomcat 2]
}
node "Servidor de Base de Datos" as DB {
  [MySQL]
  [Redis]
}

Client --> LB
LB --> App
App --> DB
@enduml
```

## Diagrama de Timing

```plantuml
@startuml
clock "Reloj" as C with period 1000
binary "Sincronización" as SY
binary "Datos" as DT

C is idle
SY is low
DT is low

@1000
SY is high
@1500
SY is low
@2000
DT is high
@3000
DT is low
@enduml
```

## Diagrama de Gantt

```plantuml
@startuml
[Diseño] lasts 5 days
[Desarrollo] lasts 10 days
[Pruebas] lasts 5 days
[Lanzamiento] lasts 3 days

[Diseño] starts at 2024-01-01
[Desarrollo] starts at 2024-01-06
[Pruebas] starts at 2024-01-16
[Lanzamiento] starts at 2024-01-21
@enduml
```

## Comandos Comunes

### Ocultar Elementos

```plantuml
@startuml
hide footbox
actor A
participant B
A -> B: Mensaje
@enduml
```

### Configuración de Estilos

```plantuml
@startuml
skinparam backgroundColor #EEEBDC
skinparam handwritten true
skinparam actorStyle awesome

actor User
User -> (Iniciar Sesión)
@enduml
```
