# JMeter FAQ

## Allgemeine Fragen

### Was ist Apache JMeter?
Apache JMeter ist eine Open-Source-Software für Lasttests und Leistungsmessung. Es ist eine zu 100% reine Java-Anwendung, die funktionales Verhalten testen und die Leistung messen kann.

### Welche Protokolle unterstützt JMeter?
JMeter unterstützt mehrere Protokolle:
- HTTP/HTTPS
- SOAP/REST Web-Services
- FTP
- JDBC (Datenbank)
- LDAP
- JMS (Message-orientierte Middleware)
- SMTP/POP3/IMAP (Mail)
- TCP
- Java-Objekte

### Ist JMeter ein Browser?
Nein, JMeter ist kein Browser. Es arbeitet auf Protokollebene und führt kein JavaScript aus oder rendert HTML-Seiten wie ein echter Browser.

## Installationsfragen

### Was sind die Systemanforderungen?
- Java 8 oder höher
- Mindestens 4GB RAM empfohlen
- Kompatibles OS: Windows, Linux, macOS

### Wie installiere ich JMeter?
1. Herunterladen von https://jmeter.apache.org/
2. Archiv entpacken
3. jmeter.sh (Linux/Mac) oder jmeter.bat (Windows) ausführen

### Warum startet JMeter nicht?
- Prüfen Sie ob Java installiert ist: `java -version`
- JAVA_HOME überprüfen
- Auf Port-Konflikte prüfen

### Wie erhöhe ich den Speicher für JMeter?
Bearbeiten Sie das JMeter-Startskript und ändern Sie die HEAP-Variable:
```bash
HEAP="-Xms1g -Xmx4g"
```

## Nutzungsfragen

### Wie erstelle ich einen grundlegenden Testplan?
1. Thread-Gruppe hinzufügen
2. HTTP-Request-Sampler hinzufügen
3. Listener hinzufügen (Ergebnisbaum, Zusammenfassungsbericht)
4. Test ausführen

### Was ist eine Thread-Gruppe?
Thread-Gruppe definiert eine Gruppe von virtuellen Benutzern, die denselben Test ausführen. Sie kontrolliert:
- Anzahl der Threads (Benutzer)
- Anlaufzeit
- Schleifenanzahl

### Wie extrahiere ich Daten aus Antworten?
Verwenden Sie Extraktoren:
- JSON-Extraktor für JSON-Antworten
- XPath-Extraktor für XML/HTML
- Regulärer Ausdruck Extraktor für jeden Text

### Wie parametrisiere ich Tests?
Verwenden Sie Variablen und Funktionen:
- `${variable}` für Variablenreferenzen
- `${__Random(1,100)}` für Zufallswerte
- `${__time()}`

### Kann JMeter Authentifizierung handhaben?
Ja, JMeter unterstützt:
- Basic Authentication
- NTLM Authentication
- Cookie Manager
- Authorization Manager

## Leistungsfragen

### Wie viele Benutzer kann JMeter simulieren?
Die Anzahl hängt von Ihrer Hardware ab. Mit richtiger Optimierung:
- GUI-Modus: 100-300 Benutzer
- CLI-Modus: 1000+ Benutzer
- Verteilte Tests: 10.000+ Benutzer

### Was sind verteilte Tests?
Verteilte Tests verwenden mehrere Maschinen, um mehr Last zu erzeugen. Ein Master koordiniert mehrere Slave-Maschinen.

### Wie führe ich JMeter im Headless-Modus aus?
```bash
jmeter -n -t testplan.jmx -l results.jtl
```

## Fehlerbehebung

### OutOfMemoryError während des Tests
- Heap-Größe erhöhen
- Anzahl der Threads reduzieren
- CLI-Modus statt GUI-Modus verwenden
- Unnötige Listener deaktivieren

### Ergebnisse werden nicht angezeigt
- Prüfen ob Test wirklich läuft
- Überprüfen ob Listener an korrektes Element angehängt
- Log-Dateien auf Fehler prüfen

### Hohe Antwortzeiten
- Netzwerk prüfen
- Server-Leistung verifizieren
- Last reduzieren wenn Server überlastet ist

### Wie debugge ich meinen Testplan?
1. Ergebnisbaum-Listener verwenden
2. Debug-Logging aktivieren
3. Mit einzelnem Thread beginnen
4. Assertions überprüfen

## Best Practices

1. Immer CLI-Modus für Lasttests verwenden
2. Mit niedriger Last beginnen und allmählich erhöhen
3. Assertions zur Validierung von Antworten verwenden
4. Nach Tests aufräumen (tearDown)
5. Testpläne in Versionskontrolle halten
6. Systemressourcen überwachen
