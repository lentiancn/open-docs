# Jenkins FAQ

## Allgemeine Fragen

### Was ist Jenkins?
Jenkins ist ein Open-Source-Automatisierungsserver, der hilft, Softwareentwicklungsprozesse wie Build, Test und Deployment zu automatisieren.

### Was ist CI/CD?
CI (Continuous Integration) führt Codeänderungen automatisch zusammen. CD (Continuous Delivery/Deployment) automatisiert den Release-Prozess.

### Ist Jenkins kostenlos?
Ja, Jenkins ist Open Source und kostenlos nutzbar.

## Installationsfragen

### Wie installiere ich Jenkins?
1. Herunterladen von https://www.jenkins.io/download/
2. Installer ausführen oder WAR-Datei entpacken
3. Auf http://localhost:8080 zugreifen
4. Einrichtungsassistent folgen

### Systemanforderungen?
- Java 8 oder höher
- Mindestens 256MB RAM (2GB+ empfohlen)
- 1GB+ Festplattenspeicher

## Verwendungsfragen

### Wie erstelle ich einen Jenkins-Job?
1. "New Item" auf dem Dashboard klicken
2. Jobnamen eingeben und Typ auswählen
3. Quellcodeverwaltung konfigurieren
4. Build-Schritte hinzufügen
5. Speichern und ausführen

### Was ist eine Jenkins-Pipeline?
Eine Pipeline ist eine Sammlung von Plugins zur Implementierung von Continuous Delivery Pipelines.

```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
    }
}
```
