# Claude Code 常见问题

本章节汇总了使用 Claude Code 时最常见的问题和解决方案。如果你遇到了问题，先看看这里是否能找到答案。

## 基础问题

### Q1: Claude Code 是什么？和 Claude有什么区别？

Claude Code 是 Claude 的编程专用版本，专为开发者设计。它能够在终端中运行，执行代码编辑、文件操作、命令执行等编程任务。而普通 Claude 更偏向于一般性的对话和问答。

### Q2: Claude Code 是免费的吗？

Claude Code 需要使用 Anthropic 的 API，会产生费用。具体价格请查看 [Anthropic 定价页面](https://www.anthropic.com/pricing)。新用户通常会有一定的免费额度。

### Q3: 支持哪些编程语言？

Claude Code 支持主流编程语言，包括但不限于：
- JavaScript / TypeScript
- Python
- Go
- Java
- C / C++
- Rust
- Ruby
- PHP
- 以及许多其他语言

对于不常见的语言，支持程度可能会有所不同。

## 安装问题

### Q4: 安装时报错 "command not found"

**解决方法：**

1. 确认安装是否成功
   ```bash
   npm list -g @anthropic-ai/claude-code
   ```

2. 检查 npm 全局 bin 路径是否在 PATH 中
   ```bash
   npm bin -g
   ```
   如果不在，添加到 PATH：
   ```bash
   export PATH="$(npm bin -g):$PATH"
   ```

3. 尝试重新安装
   ```bash
   npm uninstall -g @anthropic-ai/claude-code
   npm install -g @anthropic-ai/claude-code
   ```

### Q5: API Key 无效或过期

**解决方法：**

1. 登录 [Anthropic Console](https://console.anthropic.com/)
2. 进入 API Keys 页面
3. 创建新的 API Key
4. 更新环境变量
   ```bash
   export ANTHROPIC_API_KEY="新Key"
   ```

### Q6: 连接超时或网络错误

**可能原因：**
- 网络不稳定
- 防火墙或代理阻止
- 地区网络限制

**解决方法：**

1. 检查网络连接
2. 配置代理（如果需要）
   ```bash
   export HTTP_PROXY="http://127.0.0.1:7890"
   export HTTPS_PROXY="http://127.0.0.1:7890"
   ```
3. 尝试使用 VPN
4. 稍后重试

### Q7: Node.js 版本不满足要求

**解决方法：**

使用 nvm（Node Version Manager）升级 Node.js：

```bash
# 安装 nvm（如果没有）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装最新 LTS 版本
nvm install --lts

# 切换到 LTS 版本
nvm use --lts
```

## 使用问题

### Q8: Claude Code 不理解我的需求

**建议：**

1. **描述更具体**
   - ❌ "帮我修 bug"
   - ✅ "登录接口返回 401 错误，请帮我排查"

2. **提供更多上下文**
   - 说明项目使用的技术栈
   - 描述期望的行为
   - 提供相关代码片段

3. **分步描述**
   把复杂需求拆分成多个简单需求

### Q9: 修改了错误的文件

**解决方法：**

1. 使用 Git 撤销更改
   ```bash
   # 撤销未提交的更改
   git checkout -- .
   
   # 撤销特定文件
   git checkout -- filename
   
   # 如果已经 commit
   git revert HEAD
   ```

2. 使用 Claude Code 的 undo 命令
   在对话中输入 `undo` 尝试撤销上一步操作

### Q10: 执行命令时报错

**常见错误：**

1. **权限不足**
   ```bash
   # 添加执行权限
   chmod +x script.sh
   ```

2. **命令不存在**
   ```bash
   # 安装缺失的命令
   # Ubuntu/Debian
   sudo apt install package-name
   
   # macOS
   brew install package-name
   ```

3. **依赖缺失**
   ```bash
   # 安装项目依赖
   npm install
   # 或
   pip install -r requirements.txt
   ```

### Q11: 响应速度慢

**可能原因：**
- 网络延迟
- 服务器负载高
- 请求内容太大

**优化方法：**
1. 减少单次请求的代码量
2. 使用更精确的搜索条件
3. 避开高峰期使用

### Q12: IDE 扩展不工作

**VS Code 排查步骤：**

1. 检查扩展是否安装成功
2. 重启 VS Code
3. 查看扩展日志（Help → Toggle Developer Tools → Console）
4. 重新安装扩展

**JetBrains 排查步骤：**

1. 检查插件是否启用（Settings → Plugins）
2. 重启 IDE
3. 检查 IDE 日志
4. 重新安装插件

## 安全问题

### Q16: Claude Code 会保存我的代码吗？

Claude Code 会在会话期间读取和分析你的代码来完成请求，但不会主动保存你的代码到外部。所有修改都在你的本地环境中进行。

### Q17: 如何保护敏感信息？

1. **使用 .gitignore**：排除敏感文件
   ```
   .env
   *.pem
   credentials.json
   ```

2. **配置排除路径**：在 `.claude.json` 中配置
   ```json
   {
     "blockedPaths": ["/path/to/sensitive"]
   }
   ```

3. **不要在对话中暴露**：避免在请求中包含真实密钥、密码等

### Q18: 可以离线使用吗？

目前不可以。Claude Code 需要连接 Anthropic 的 API 来处理请求。离线状态下只能查看本地代码，无法使用 AI 功能。

### Q21: Claude Code 和 GitHub Copilot 哪个更好？

两者定位不同：
- **Copilot**：专注于代码补全，在编写代码时提供即时建议
- **Claude Code**：专注于任务级别的编程代理，能够理解和执行更复杂的编程任务

建议根据具体需求选择，也可以同时使用两者。

### Q22: 如何提升 Claude Code 的使用效率？

1. **熟悉快捷命令**：提高操作效率
2. **创建项目指令文件**：让 AI 更了解项目规范
3. **保持清晰的对话**：一次只处理一个主题
4. **利用上下文**：在同一个会话中完成相关任务
