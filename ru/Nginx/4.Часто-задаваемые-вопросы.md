# Часто задаваемые вопросы о Nginx

## Проблемы установки

### Что делать, если установка Nginx не удалась?

1. Проверить, установлены ли все системные зависимости
2. Проверить логи ошибок, чтобы понять конкретную причину сбоя
3. Убедиться, что достаточно свободного места на диске
4. Проверить наличие проблем с правами доступа

### Как проверить версию Nginx?

```bash
nginx -v
nginx -V  # Подробная информация, включая параметры компиляции
```

### Какие операционные системы поддерживает Nginx?

Nginx поддерживает Linux, FreeBSD, Solaris, AIX, HP-UX, macOS и Windows.

## Проблемы конфигурации

### Как проверить, правильная ли конфигурация Nginx?

```bash
nginx -t
```

### Где находится конфигурационный файл?

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- Компилированная установка: `/usr/local/nginx/conf/nginx.conf`

### Нужно ли перезапускать после изменения конфигурации?

Нет. Используйте `nginx -s reload` для перезагрузки конфигурации.

## Проблемы производительности

### Как оптимизировать производительность Nginx?

1. **Настроить количество рабочих процессов**

```nginx
worker_processes auto;
```

2. **Увеличить количество соединений на worker**

```nginx
events {
    worker_connections 4096;
}
```

3. **Включить сжатие gzip**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **Включить кэширование в браузере**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **Использовать пул соединений**

```nginx
upstream backend {
    server backend.example.ru;
    keepalive 32;
}
```

### Что делать, если Nginx потребляет слишком много памяти?

1. Уменьшить количество worker_connections
2. Ограничить размер тела запроса
3. Оптимизировать настройки кэша
4. Проверить наличие утечек памяти

### Как обрабатывать высокую параллельность?

1. Использовать балансировку нагрузки
2. Включить кэширование
3. Настроить параметры ядра
4. Использовать CDN

## Проблемы безопасности

### Как предотвратить DDoS-атаки?

1. **Ограничить соединения**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **Ограничить частоту запросов**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **Закрыть медленные соединения**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### Как настроить SSL-сертификат?

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### Как принудительно использовать HTTPS?

```nginx
server {
    listen 80;
    server_name example.ru;
    return 301 https://$server_name$request_uri;
}
```

## Проблемы прокси

### Что делать, если обратный прокси не работает?

1. Проверить, запущен ли backend-сервис
2. Проверить настройки брандмауэра
3. Убедиться, что конфигурация proxy_pass правильная
4. Проверить логи ошибок

### Как проксировать WebSocket?

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### Как получить реальный IP после проксирования?

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## Проблемы с логами

### Что делать, если файлы логов слишком большие?

Использовать logrotate:

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### Как отключить логирование доступа?

```nginx
access_log off;
```

### Как настроить формат лога?

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## Проблемы балансировки нагрузки

### Как балансировщик выбирает backend?

Nginx поддерживает несколько алгоритмов балансировки:

- **Round-Robin (по умолчанию)**: Распределяет запросы последовательно
- **Weighted Round-Robin**: Распределяет на основе веса
- **IP Hash**: Направляет запросы с одного IP на тот же backend
- **Least Connections**: Направляет на backend с наименьшим количеством соединений

### Что происходит, если backend-сервер выходит из строя?

Nginx автоматически пропускает вышедшие из строя узлы и перенаправляет запросы на рабочие узлы.

## Проблемы кэширования

### Как настроить прокси-кэширование?

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### Что делать, если кэширование не работает?

1. Проверить права на директорию кэша
2. Убедиться, что заголовки ответа разрешают кэширование
3. Проверить правильность настройки ключа кэша

### Как очистить кэш?

```bash
rm -rf /var/cache/nginx/*
```

## Проблемы с портами

### Что делать, если порт уже занят?

```bash
# Проверить использование порта
netstat -tulpn | grep :80

# Использовать другой порт
listen 8080;
```

### Как слушать на нескольких портах?

```nginx
server {
    listen 80;
    listen 8080;
    server_name example.ru;
}
```

## Распространённые коды ошибок

### 400 Bad Request

- Слишком большой заголовок запроса: Увеличить `client_header_buffer_size`
- Слишком длинный URL: Проверить путь запроса

### 403 Forbidden

- Индексный файл не существует
- Недостаточные права доступа: Проверить права на файлы и директории

### 404 Not Found

- Ошибка пути к файлу
- Заблокировано SELinux или AppArmor

### 502 Bad Gateway

- Backend-сервис не запущен
- Ошибка конфигурации FastCGI/PHP-FPM

### 504 Gateway Timeout

- Таймаут ответа backend
- Увеличить `proxy_read_timeout`

## Советы по отладке

### Как получить подробную информацию об ошибках?

1. Включить отладочное логирование:

```nginx
error_log /var/log/nginx/error.log debug;
```

2. Проверить логи ошибок:

```bash
tail -f /var/log/nginx/error.log
```

### Как проверить, что конфигурация применяется?

```bash
# Тестировать конфигурацию
nginx -t

# Показать полную конфигурацию
nginx -T
```

### Как отлаживать регулярные выражения?

Использовать тестирование location:

```nginx
location /test {
    access_log /var/log/nginx/test.log;
}
```

## Проблемы обновления

### Как обновить Nginx?

1. Сделать резервную копию конфигурационных файлов
2. Остановить Nginx
3. Установить новую версию
4. Запустить Nginx
5. Проверить функциональность

### Как выполнить обновление без прерывания?

```bash
# Скомпилировать новую версию
make

# Отправить сигнал USR2 для обновления
kill -s USR2 $(cat /var/run/nginx.pid)

# Изящно завершить старые рабочие процессы
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
