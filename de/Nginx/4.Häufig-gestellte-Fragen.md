# Nginx FAQ

## Installationsprobleme

### Was tun, wenn die Nginx-Installation fehlschlägt?

1. Überprüfen, ob Systemabhängigkeiten vollständig sind
2. Fehlerlogs prüfen, um die spezifische Fehlerursache zu verstehen
3. Sicherstellen, dass ausreichend Speicherplatz vorhanden ist
4. Auf Berechtigungsprobleme prüfen

### Wie überprüfe ich die Nginx-Version?

```bash
nginx -v
nginx -V  # Detaillierte Informationen einschließlich Kompilierungsparameter
```

### Welche Betriebssysteme unterstützt Nginx?

Nginx unterstützt Linux, FreeBSD, Solaris, AIX, HP-UX, macOS und Windows.

## Konfigurationsprobleme

### Wie überprüfe ich, ob die Nginx-Konfiguration korrekt ist?

```bash
nginx -t
```

### Wo ist die Konfigurationsdatei?

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- Kompilierte Installation: `/usr/local/nginx/conf/nginx.conf`

### Muss ich nach Änderung der Konfiguration neu starten?

Nein. Verwenden Sie `nginx -s reload` zum Neuladen der Konfiguration.

## Leistungsprobleme

### Wie optimiere ich die Nginx-Leistung?

1. **Anzahl der Worker-Prozesse anpassen**

```nginx
worker_processes auto;
```

2. **Worker-Verbindungen erhöhen**

```nginx
events {
    worker_connections 4096;
}
```

3. **Gzip-Komprimierung aktivieren**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **Browser-Caching aktivieren**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **Verbindungspooling verwenden**

```nginx
upstream backend {
    server backend.beispiel.de;
    keepalive 32;
}
```

### Was tun, wenn Nginx zu viel Speicher verbraucht?

1. worker_connections-Anzahl reduzieren
2. Anfragegröße begrenzen
3. Cache-Einstellungen optimieren
4. Auf Speicherlecks prüfen

### Wie handle ich hohe Gleichzeitigkeit?

1. Load Balancing verwenden
2. Caching aktivieren
3. Kernel-Parameter anpassen
4. CDN verwenden

## Sicherheitsprobleme

### Wie verhindere ich DDoS-Angriffe?

1. **Verbindungen begrenzen**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **Anfragerate begrenzen**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **Langsame Verbindungen schließen**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### Wie konfiguriere ich ein SSL-Zertifikat?

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### Wie erzwinge ich HTTPS?

```nginx
server {
    listen 80;
    server_name beispiel.de;
    return 301 https://$server_name$request_uri;
}
```

## Proxy-Probleme

### Was tun, wenn Reverse-Proxy fehlschlägt?

1. Prüfen, ob Backend-Dienst läuft
2. Firewall-Einstellungen prüfen
3. proxy_pass-Konfiguration verifizieren
4. Fehlerlogs prüfen

### Wie proxye ich WebSocket?

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### Wie erhalte ich die echte IP nach Proxy?

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## Log-Probleme

### Was tun, wenn Log-Dateien zu groß sind?

Logrotate verwenden:

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### Wie deaktiviere ich Access-Logging?

```nginx
access_log off;
```

### Wie konfiguriere ich das Log-Format?

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## Load-Balancing-Probleme

### Wie wählt Load-Balancing den Backend aus?

Nginx unterstützt mehrere Load-Balancing-Algorithmen:

- **Round-Robin (Standard)**: Verteilt Anfragen sequenziell
- **Gewichtetes Round-Robin**: Verteilt basierend auf Gewichtung
- **IP-Hash**: Leitet Anfragen derselben IP zum selben Backend
- **Least Connections**: Leitet zum Backend mit den wenigsten Verbindungen

### Was passiert, wenn ein Backend-Server ausfällt?

Nginx überspringt automatisch ausgefallene Knoten und leitet Anfragen an gesunde Knoten weiter.

## Cache-Probleme

### Wie konfiguriere ich Proxy-Caching?

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### Was tun, wenn Caching nicht funktioniert?

1. Cache-Pfad-Berechtigungen prüfen
2. Bestätigen, dass Antwort-Header Caching erlauben
3. Cache-Key-Konfiguration verifizieren

### Wie lösche ich den Cache?

```bash
rm -rf /var/cache/nginx/*
```

## Port-Probleme

### Was, wenn Port bereits belegt ist?

```bash
# Portbelegung prüfen
netstat -tulpn | grep :80

# Anderen Port verwenden
listen 8080;
```

### Wie höre ich auf mehreren Ports?

```nginx
server {
    listen 80;
    listen 8080;
    server_name beispiel.de;
}
```

## Häufige Fehlercodes

### 400 Bad Request

- Anfrage-Header zu groß: `client_header_buffer_size` erhöhen
- URL zu lang: Anfragepfad prüfen

### 403 Forbidden

- Index-Datei existiert nicht
- Unzureichende Berechtigungen: Datei- und Verzeichnisberechtigungen prüfen

### 404 Not Found

- Dateipfadfehler
- Blockiert von SELinux oder AppArmor

### 502 Bad Gateway

- Backend-Dienst läuft nicht
- FastCGI/PHP-FPM-Konfigurationsfehler

### 504 Gateway Timeout

- Backend-Antwort-Timeout
- `proxy_read_timeout` erhöhen

## Debugging-Tipps

### Wie erhalte ich detaillierte Fehlerinformationen?

1. Debug-Logging aktivieren:

```nginx
error_log /var/log/nginx/error.log debug;
```

2. Fehlerlogs prüfen:

```bash
tail -f /var/log/nginx/error.log
```

### Wie teste ich, ob Konfiguration wirksam wird?

```bash
# Konfiguration testen
nginx -t

# Vollständige Konfiguration anzeigen
nginx -T
```

### Wie debugge ich reguläre Ausdrücke?

`location`-Test verwenden:

```nginx
location /test {
    access_log /var/log/nginx/test.log;
}
```

## Upgrade-Probleme

### Wie upgrade ich Nginx?

1. Konfigurationsdateien sichern
2. Nginx stoppen
3. Neue Version installieren
4. Nginx starten
5. Funktionalität testen

### Wie führe ich ein nahtloses Upgrade durch?

```bash
# Neue Version kompilieren
make

# USR2-Signal zum Upgrade senden
kill -s USR2 $(cat /var/run/nginx.pid)

# Alte Worker-Prozesse anmutig beenden
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
