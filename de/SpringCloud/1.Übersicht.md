# Spring Cloud Übersicht

Spring Cloud ist ein umfassendes Framework für die Erstellung von Cloud-nativen Anwendungen auf der Spring-Plattform. Es bietet Tools und Bibliotheken für die Erstellung verteilter Systeme mit Mustern wie Service-Discovery, Konfigurationsmanagement, Circuit Breakers und Load Balancing.

## Was ist Spring Cloud?

Spring Cloud abstrahiert die Komplexität verteilter Systeme und bietet ein einheitliches Programmiermodell für die Erstellung von Cloud-nativen Anwendungen.

## Kernkomponenten

### Service-Discovery
- **Eureka**: Netflix Service-Registry
- **Consul**: HashiCorp Service-Discovery
- **Zookeeper**: Apache Koordinationsservice

### Konfigurationsmanagement
- **Spring Cloud Config**: Zentralisierte Konfiguration
- **Consul Config**: Consul-basierte Konfiguration
- **Vault**: Geheimnisverwaltung

### API-Gateway
- **Spring Cloud Gateway**: Modernes reaktives Gateway
- **Zuul**: Netflix Edge Service

### Circuit Breaker
- **Resilience4j**: Moderner Circuit Breaker
- **Hystrix**: Netflix Latenz- und Fehlertoleranz

## Hauptmerkmale

### Service-Registrierung und Discovery
Dienste können sich dynamisch registrieren und andere Dienste entdecken.

### Verteilte Konfiguration
Zentralisiertes, versioniertes Konfigurationsmanagement.

### Client-seitiges Load Balancing
Intelligentes Load Balancing mit verschiedenen Strategien.

### Circuit Breaker Muster
Verhindert Kaskadenausfälle durch automatische Fallback-Mechanismen.

## Warum Spring Cloud?

1. **Konsistenz**: Einheitliches Spring Programmiermodell
2. **Flexibilität**: Mehrere Implementierungsoptionen
3. **Resilienz**: Integrierte Fehlertoleranz
4. **Beobachtbarkeit**: Umfassende Überwachung
5. **Cloud-nativ**: Erstklassige Kubernetes-Unterstützung
