# Kotlin ユーザーマニュアル

## 変数と基本型

Kotlin は変数の宣言に var と val の2つのキーワードを使用します。var は変更可能な変数を宣言し、Java の通常の変数に類似しています。val は変更不可能な変数を宣言し、Java の final 変数に類似しています。Kotlin では、val を使用することがベストプラクティスであり、コードをより安全で理解しやすくします。

変数を宣言する場合、型を明示的に指定するか、コンパイラの推論に委ねることができます：

val name: String = "Kotlin"
val age = 10 // コンパイラが Int 型と推論
var count = 0 // 変更可能な変数

Kotlin の基本型には、整数型（Int、Long、Short、Byte）、浮動小数点型（Double、Float）、文字型（Char）、ブール型（Boolean）、文字列型（String）が含まれます。Java とは異なり、Kotlin のすべての基本型はオブジェクトであり、対応するメンバーメソッドを持っています。

Kotlin の整数型はデフォルトで Int 型です。より大きな数値を表現するには、Long サフィックスを使用します：

val longNumber = 100L // Long 型

Kotlin の文字列は強力で、文字列テンプレート機能をサポートしており、文字列内に直接変数や式を埋め込むことができます：

val name = "World"
println("Hello, $name!") // 出力: Hello, World!
println("Length: ${name.length}") // 出力: Length: 5

## コレクション

Kotlin 標準ライブラリは、List（リスト）、Set（セット）、Map（マップ）を含む丰富的なコレクション型を提供します。各コレクション型には、不変バージョンと可変バージョンの両方があります。

不変コレクションは作成後に要素を追加、削除、変更できません：

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

可変コレクションは要素の変更を許可します：

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Kotlin のコレクションは、map、filter、reduce などの丰富的な関数型操作を提供します：

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## 制御フロー

Kotlin の制御フローには、条件式、ループ、ジャンプ式が含まれます。

Kotlin の if 式は値を返すため、変数に直接代入できます：

val max = if (a > b) a else b

when 式は Kotlin の強力な複数ブランチマッチングツールで、さまざまな条件をマッチできます：

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Kotlin のループには for ループと while ループがあります。for ループは範囲、配列、コレクションを反復処理できます：

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## 関数

Kotlin は関数の宣言に fun キーワードを使用します。関数はパラメータ型と戻り値型を指定できます：

fun greet(name: String): String {
    return "Hello, $name!"
}

関数の本体が単一の式だけの場合、単一式構文を使用できます：

fun greet(name: String) = "Hello, $name!"

Kotlin はデフォルト引数と名前付き引数をサポートしており、関数のオーバーロードの必要性を大幅に削減します：

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin は可変長引数もサポートしており、vararg キーワードを使用します：

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## クラスとオブジェクト

Kotlin のクラス宣言は Java よりも簡潔です。デフォルトでは、Kotlin のクラスは final であり、継承できません。クラスを継承可能にするには、open キーワードを使用します：

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Kotlin のコンストラクタには、主コンストラクタと副コンストラクタがあります。主コンストラクタはクラス名の後に 直接宣言します：

class Person(val name: String, val age: Int)

コンストラクタでロジックを実行する必要がある場合、init ブロックを使用できます：

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Kotlin のデータクラス（data class）は equals、hashCode、toString、copy メソッドを自動生成し、データ存储用のクラスに最適です：

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Null 安全

Null 安全は Kotlin の最も重要な機能の一つです。Kotlin では、タイプシステムは Null 許容型と非 Null 許容型を区別し、これは NullPointerException を根本的に削減します。

デフォルトでは、型は Null 許容ではありません：

val name: String = "Kotlin" // Null にできません
val name: String = null // コンパイルエラー

変数を Null にすることを許可するには、Null 許容型（?）を使用します：

val name: String? = null // Null にできます

Null 許容型のプロパティやメソッドにアクセスするには、安全な呼び出し演算子（?.）または Null アサーション（!!）を使用します：

val length = name?.length // name が Null の場合、Null を返す
val length = name!!.length // name が Null の場合、例外をスロー

Elvis 演算子（?:）はデフォルト値を処理する簡潔な方法を提供します：

val length = name?.length ?: 0 // name が Null の場合、0 を返す

## 拡張関数と拡張プロパティ

Kotlin は、継承や元のクラスの変更なしで、既存のクラスに新しい関数とプロパティを追加できます。これらは拡張関数と拡張プロパティ」と呼ばれます。

拡張関数の宣言は、拡張するクラス名を関数名の前に付けて行います：

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

拡張プロパティは既存のクラスに新しいプロパティを追加することもできます：

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## 高階関数とラムダ

高階関数は、パラメータとして関数を受け取るか関数を返す関数です。ラムダ式はこの機能を実装するための主要な方法です。

ラムダ式の構文では中括弧を使用します：

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

ラムダが関数の最後のパラメータの場合、ラムダを括弧の外に移動できます：

list.filter { it > 0 }

ラムダ式では、it は単一パラメータの暗黙的な名前を表します。パラメータが1つだけの場合、パラメータ宣言を省略できます。

## コルーチン

コルーチンは Kotlin 非同期プログラミングの中核機能です。スレッドと比較して、コルーチンはるかに軽量であり、単一のスレッドで複数のコルーチンを実行できます。

launch はコルーチンを起動する一般的な方法で、バックグラウンドで新しいコルーチンを起動します：

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

suspend 関数はコルrustin でのみ呼び出すことができます：

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        //  時間のかかる操作を実行
    }
}

Kotlin コルーチンは、コルーチンが実行されるスレッドを制御するための複数のディスパッチャを提供します：Dispatchers.Main は UI 操作用、Dispatchers.IO は I/O 操作用、Dispatchers.Default は CPU 集中的なタスク用です。

## 密封クラスと列挙型

密封クラスは制限されたクラス階層であり、継承できるサブクラスの数を制限します。すべてのサブクラスは同じファイルで宣言する必要があります。これは、有限の状態や結果型を表すときに非常に便利です：

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

when 式は密封クラスの機能を十分に活用でき、コンパイラはすべての可能性のあるケースが処理されたかどうかをチェックします：

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

列挙型は有限数の定数値を表すために使用されます：

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

列挙型にはプロパティとメソッドを持たせることができます：

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## インターフェースと抽象クラス

Kotlin のインターフェースにはデフォルト実装を含めることができ、これによりインターフェースはより柔軟になります：

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
}

クラスは複数のインターフェースを実装できます：

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## オブジェクト式とオブジェクト宣言

オブジェクト式は匿名クラスのインスタンスを作成し、Java の匿名内部クラスに類似しています：

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // クリックイベントを処理
    }
})

オブジェクト宣言はシングルトンを作成するために使用されます：

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## コンパニオンオブジェクト

各クラスには、Java の静的なメンバーに類似したコンパニオンオブジェクトを含めることができます：

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

コンパニオンオブジェクトのメンバーはクラス名を介して直接アクセスできます：

MyClass.TAG
MyClass.create()

## タイプチェックと変換

is 演算子はタイプチェックに使用され、Java の instanceof に類似しています：

if (obj is String) {
    // このブランチでは、obj は自動的に String に変換されます
    println(obj.length)
}

スマートキャストはチェックされたタイプを自動的に変換します。コンパイラが特定のブランチで変数のタイプを推断できる場合、自動的に変換を行います。

スマートキャストが適用できない場合、明示的な変換を使用できます：

val str = obj as String

安全なタイプ変換は as? を使用하며、変換に失敗すると Null を返します：

val str = obj as? String // 変換に失敗した場合、Null を返す
