# Руководство по установке Nacos

## Требования к среде

Nacos требует среду выполнения Java. Убедитесь, что используете следующие версии:

- 64-битная операционная система: поддерживаются Linux/Unix/Mac/Windows, рекомендуется Linux/Unix/Mac
- 64-битный JDK 17+ (рекомендуется JDK 17 или выше)
- Загрузка JDK: https://www.oracle.com/java/technologies/downloads/#java17
- Настройка переменной JAVA_HOME: https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## Рекомендуемая конфигурация

Рекомендуется запускать Nacos со следующей конфигурацией:

- ЦП: 2 ядра
- Оперативная память: 4 ГБ
- Диск: 60 ГБ

## Способы установки

Nacos поддерживает несколько способов установки. Вы можете выбрать подходящий способ в соответствии с вашими потребностями.

### Способ 1: Использование автоматического развертывания nacos-setup (рекомендуется)

nacos-setup обеспечивает автоматическое развертывание одним щелчком, автоматически обрабатывая загрузку, настройку аутентификации, обнаружение портов и проверку среды Java.

#### Установка nacos-setup

Linux/macOS:

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows (PowerShell):

```powershell
# Запуск с помощью PowerShell
```

#### Запуск службы Nacos

Развернуть последнюю версию автономного Nacos локально одним щелчком:

```bash
nacos-setup
```

Развернуть определенную версию автономного Nacos локально:

```bash
nacos-setup -v 3.1.1
```

Развернуть последнюю версию кластера Nacos локально:

```bash
nacos-setup -c cluster
```

Настроить внешнее хранилище глобально, ввести адрес базы данных, порт, имя пользователя и пароль, затем повторно запустить nacos-setup:

```bash
nacos-setup --datasource-conf
```

Дополнительные сведения см. в руководстве пользователя nacos-setup.

### Способ 2: Ручная загрузка и развертывание

#### Загрузка установочного пакета

Вы можете получить дистрибутив Nacos двумя способами:

**Способ 1: Загрузка с официального сайта Nacos**

Посетите страницу загрузки версий на официальном сайте Nacos, выберите стабильную версию, затем щелкните, чтобы загрузить ZIP-файл из столбца бинарных пакетов.

**Способ 2: Загрузка с GitHub**

Посетите страницу релизов Nacos на GitHub и загрузите дистрибутив соответствующей версии.

Примечание: При одновременной загрузке большим количеством пользователей могут возникнуть ошибки ограничения скорости загрузки. В этом случае подождите и повторите попытку или загрузите с GitHub.

#### Распаковка дистрибутива Nacos

Linux/Unix/Mac:

```bash
unzip nacos-server-$version.zip
# или
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows:

Используйте инструменты извлечения для распаковки nacos-server-$version.zip напрямую.

#### Запуск сервера

Команда запуска Linux/Unix/Mac (standalone означает некластерный режим):

```bash
sh startup.sh -m standalone
```

Если вы используете Ubuntu или получаете сообщения об ошибках о не найденных символах, попробуйте:

```bash
bash startup.sh -m standalone
```

При запуске вам будет предложено ввести три параметра аутентификации (начиная с Nacos 3.0.0, аутентификация консоли включена по умолчанию, поэтому следующие три параметра аутентификации необходимо заполнить):

```
nacos.core.auth.plugin.nacos.token.secret.key отсутствует, укажите в виде строки Base64: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key отсутствует, укажите: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value отсутствует, укажите: ${your_input_server_identity_key}
```

Команда запуска Windows:

```bash
cmd startup.cmd
```

Или дважды щелкните startup.cmd для запуска.

### Способ 3: Установка с Docker

#### Загрузка образа Nacos

```bash
docker pull nacos/nacos-server
```

#### Запуск в автономном режиме

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### Запуск в кластерном режиме

Требуется кластерное развертывание с Docker Compose или Kubernetes.

### Способ 4: Установка на Kubernetes

Nacos поддерживает развертывание в средах Kubernetes и может быть установлен с помощью Helm Chart или Operator. Конкретные шаги см. в документации по развертыванию Nacos на Kubernetes.

## Проверка запуска службы Nacos

Перейдите в каталог журналов Nacos и используйте `tail -f start.out` для просмотра журналов. Если вы видите следующее сообщение, служба успешно запущена:

```
Nacos started successfully in stand alone mode. use embedded storage
```

## Быстрая проверка

### Консоль Nacos

Откройте любой браузер и введите: http://127.0.0.1:8080/index.html для доступа к консоли Nacos.

Примечание: При первом открытии вам нужно будет инициализировать пароль администратора для пользователя nacos. Имя пользователя/пароль по умолчанию: nacos/nacos@demo.

### Регистрация сервиса

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### Обнаружение сервиса

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### Публикация конфигурации

Сначала войдите для получения токена доступа:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

Используйте токен доступа для создания конфигурации:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### Получение конфигурации

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## Остановка сервера

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

Или дважды щелкните shutdown.cmd для запуска.

## Примечания

1. Начиная с Nacos 3.0.0, аутентификация консоли включена по умолчанию. Параметры аутентификации необходимо настроить при первом запуске.
2. Развертывание в кластерном режиме требует внешней базы данных (например, MySQL) для хранения данных.
3. В производственных средах следует использовать кластерный режим для обеспечения высокой доступности.
4. Убедитесь, что необходимые порты открыты в брандмауэре (по умолчанию 8848, 9848).
5. Для оптимальной производительности рекомендуется JDK 17 или выше.
