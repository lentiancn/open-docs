# OpenFeign Frequently Asked Questions

## Installation Related Questions

### Q: How to verify OpenFeign is properly installed?
**A:** After adding dependencies, verify by creating a basic application:

```java
@SpringBootApplication
@EnableFeignClients
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

Ensure your Spring Boot application starts successfully and Feign clients can be injected properly.

### Q: How to handle dependency version conflicts?
**A:** Use Spring Cloud BOM for version management:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>2022.0.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

This prevents version conflicts effectively.

### Q: Why do I need to explicitly add spring-cloud-starter-loadbalancer?
**A:** Starting from Spring Cloud 2020.0.0, Ribbon was removed and Spring Cloud LoadBalancer became the default load balancer. `spring-cloud-starter-openfeign` includes it as an optional dependency, so you need to add it explicitly:

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

## Configuration Related Questions

### Q: How to configure timeout settings?
**A:** You can configure timeouts via configuration file or Java configuration:

**Configuration file approach:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 5000
            read-timeout: 5000
          user-service:
            connect-timeout: 3000
            read-timeout: 10000
```

**Java configuration approach:**
```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Request.Options requestOptions() {
        return new Request.Options(3000, 10000);
    }
}
```

### Q: How to enable GZIP compression?
**A:** Enable compression in configuration file:

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

Note: When using OkHttpClient, compression is disabled if `content-encoding` or `accept-encoding` headers are present.

### Q: How to configure log levels?
**A:** Two configuration approaches:

**Configuration file:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
            
logging:
  level:
    com.example.clients.UserService: DEBUG
```

**Java configuration:**
```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Supported log levels:
- **NONE**: No logging (default)
- **BASIC**: Log only request method, URL, response status code, and execution time
- **HEADERS**: Log basic information along with request and response headers
- **FULL**: Log headers, body, and metadata for both requests and responses

## Usage Related Questions

### Q: FeignClient interface cannot be injected, what should I do?
**A:** Common causes and solutions:

**1. Package scan path issue:**
```java
// Ensure FeignClient is in the scanned package path
@SpringBootApplication
@EnableFeignClients(basePackages = "com.example.clients")
public class Application {
    // ...
}
```

**2. Circular dependency issue:**
If FeignClient is referenced by multiple Services, circular dependency may occur. Solutions:
- Use `@Lazy` annotation for lazy injection
- Refactor code structure to avoid circular dependencies

**3. Bean name conflicts:**
If you have multiple FeignClients with the same name, use `contextId` to differentiate:
```java
@FeignClient(contextId = "userClient1", name = "user-service")
public interface UserClient1 { }

@FeignClient(contextId = "userClient2", name = "user-service")  
public interface UserClient2 { }
```

### Q: How to handle complex request parameters?
**A:** Several approaches for complex parameters:

**1. Using @SpringQueryMap:**
```java
public class SearchCriteria {
    private String name;
    private String status;
    private Integer page;
}

@FeignClient(name = "search-service")
public interface SearchService {
    @GetMapping("/search")
    SearchResult search(@SpringQueryMap SearchCriteria criteria);
}
```

**2. Using Map:**
```java
@FeignClient(name = "api-service")
public interface ApiService {
    @GetMapping("/data")
    Data getData(@RequestParam Map<String, Object> params);
}
```

**3. Custom QueryMapEncoder:**
```java
@Bean
public QueryMapEncoder queryMapEncoder() {
    return new CustomQueryMapEncoder();
}
```

### Q: How to handle file uploads?
**A:** File upload requires special configuration:

```java
@FeignClient(name = "file-service")
public interface FileService {
    
    @PostMapping(value = "/upload", 
                consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    UploadResponse uploadFile(@RequestPart("file") MultipartFile file,
                             @RequestPart("metadata") FileMetadata metadata);
}
```

Also configure encoder:
```java
@Bean
public Encoder feignFormEncoder() {
    return new FeignFormEncoder(new SpringEncoder(messageConverters()));
}
```

## Error Handling Questions

### Q: How to customize error handling?
**A:** Implement ErrorDecoder interface:

```java
@Component
public class CustomErrorDecoder implements ErrorDecoder {
    
    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() == 404) {
            return new ResourceNotFoundException("Resource not found");
        } else if (response.status() >= 500) {
            return new ServerException("Server error");
        }
        return new Default().decode(methodKey, response);
    }
}
```

### Q: Circuit breaker not working, what's wrong?
**A:** Check the following configurations:

**1. Enable circuit breaker:**
```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

**2. Add required dependency:**
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

**3. Configure circuit breaker rules:**
```yaml
resilience4j:
  circuitbreaker:
    instances:
      UserServicegetUser:
        failure-rate-threshold: 50
        minimum-number-of-calls: 5
        wait-duration-in-open-state: 5000
```

### Q: Fallback not triggering, what should I check?
**A:** Common issues:

**1. Ensure fallback class is a Spring Bean:**
```java
@Component  // This annotation is required
public class UserServiceFallback implements UserService {
    // ...
}
```

**2. Check exception types:**
Circuit breaker only triggers fallback for specific exceptions, ensure the thrown exception type is expected.

**3. Configure correct fallback factory:**
```java
@FeignClient(name = "service", fallbackFactory = MyFallbackFactory.class)
public interface MyService { }
```

## Performance Related Questions

### Q: Feign client performance is slow, how to optimize?
**A:** Performance optimization recommendations:

**1. Configure appropriate timeouts:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
```

**2. Enable connection pooling:**
```yaml
spring:
  cloud:
    openfeign:
      httpclient:
        hc5:
          enabled: true
          max-connections: 200
          max-connections-per-route: 20
```

**3. Enable compression:**
```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
        response:
          enabled: true
```

**4. Use caching:**
```java
@FeignClient(name = "config-service")
public interface ConfigService {
    @GetMapping("/config/{key}")
    @Cacheable(cacheNames = "config-cache", key = "#key")
    ConfigValue getConfig(@PathVariable String key);
}
```

### Q: How to resolve memory leak issues?
**A:** Memory leaks are typically caused by:

**1. Connections not closed:**
Ensure HTTP clients properly manage connection pools.

**2. Cache not cleaned up:**
Regularly clean up caches or set appropriate expiration times.

**3. Monitor memory usage:**
```java
@RestController
public class MetricsController {
    
    @GetMapping("/memory")
    public MemoryInfo getMemoryInfo() {
        Runtime runtime = Runtime.getRuntime();
        return new MemoryInfo(
            runtime.totalMemory(),
            runtime.freeMemory(),
            runtime.maxMemory()
        );
    }
}
```

## Security Related Questions

### Q: How to add authentication tokens to requests?
**A:** Use RequestInterceptor:

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor authRequestInterceptor() {
        return requestTemplate -> {
            String token = getToken(); // Logic to get token
            requestTemplate.header("Authorization", "Bearer " + token);
        };
    }
    
    private String getToken() {
        // Implement token retrieval logic
        return "your-token";
    }
}
```

### Q: How to handle OAuth2 authentication?
**A:** Enable OAuth2 support:

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
        
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: client_credentials
            scope: read,write
```

### Q: How to filter sensitive information?
**A:** Custom log filtering:

```java
public class SecureLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        Map<String, Collection<String>> safeHeaders = new HashMap<>(request.headers());
        safeHeaders.remove("Authorization");
        safeHeaders.remove("X-API-Key");
        
        super.logRequest(configKey, logLevel, 
            Request.create(request.httpMethod(), request.url(), safeHeaders,
                request.body(), request.charset(), request.requestTemplate()));
    }
}
```

## Testing Related Questions

### Q: How to test Feign clients?
**A:** Recommended to use WireMock:

```java
@SpringBootTest
@AutoConfigureWireMock(port = 0)
class FeignClientTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    void shouldGetUserSuccessfully() {
        // Given
        stubFor(get(urlEqualTo("/users/1"))
            .willReturn(aResponse()
                .withStatus(200)
                .withHeader("Content-Type", "application/json")
                .withBody("{\"id\":1,\"name\":\"John\"}")));
        
        // When
        User user = userService.getUser(1L);
        
        // Then
        assertThat(user.getName()).isEqualTo("John");
    }
}
```

### Q: How to simulate error scenarios?
**A:** Use WireMock to simulate various errors:

```java
@Test
void shouldHandleServerError() {
    // Given
    stubFor(get(urlEqualTo("/users/999"))
        .willReturn(aResponse().withStatus(500)));
    
    // When & Then
    assertThatThrownBy(() -> userService.getUser(999L))
        .isInstanceOf(ServerException.class);
}
```

## Deployment Related Questions

### Q: How to configure for production environment?
**A:** Production environment recommended configuration:

```yaml
# application-prod.yml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
            logger-level: basic
      compression:
        request:
          enabled: true
        response:
          enabled: true
      circuitbreaker:
        enabled: true
      cache:
        enabled: true
        
logging:
  level:
    com.example.clients: WARN
```

### Q: How to monitor Feign clients?
**A:** Integrate with Micrometer:

```yaml
spring:
  cloud:
    openfeign:
      micrometer:
        enabled: true
```

Then view metrics through Actuator endpoints:
```bash
GET /actuator/metrics
GET /actuator/metrics/feign.client.requests
```

By understanding and applying these solutions, you'll be able to develop, debug, and maintain OpenFeign-based applications more effectively.
