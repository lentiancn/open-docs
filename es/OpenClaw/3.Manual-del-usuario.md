# Manual del Usuario de OpenClaw

Este manual te ayudará a entender cómo usar OpenClaw de manera efectiva.

## Uso Básico

### Hablar con OpenClaw

Después de la instalación, puedes interactuar con OpenClaw de varias maneras:

1. **Control UI**: Visita `http://127.0.0.1:18789/` en tu navegador
2. **Comando CLI**: `openclaw agent --message "Tu mensaje"`
3. **Canales configurados**: Envía mensajes vía WhatsApp, Telegram, Discord, etc.

### Enviar Mensajes

Envía mensajes usando CLI:

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## Conceptos Clave

### Gateway

Gateway es el plano de control principal de OpenClaw, gestionando todas las sesiones, canales, herramientas y eventos. Escucha por defecto: `ws://127.0.0.1:18789`.

### Sesión

OpenClaw usa un modelo de sesión para gestionar conversaciones:

- **Sesión principal**: Chat directo
- **Aislamiento de grupos**: Mensajes de grupo manejados por separado
- **Modo de activación**: Controla cuándo responder
- **Estrategia de respuesta**: Decide cómo responder

### Espacio de Trabajo

El espacio de trabajo es donde OpenClaw guarda configuración y datos:

- Ubicación por defecto: `~/.openclaw/workspace`
- Contiene: AGENTS.md, SOUL.md, TOOLS.md
- Directorio de habilidades: `~/.openclaw/workspace/skills/`

## Configuración de Canales

### Canales Soportados

OpenClaw soporta estas plataformas de mensajería:

- Mensajería instantánea: WhatsApp, Telegram, Signal, iMessage
- Plataformas sociales: Discord, Slack
- Herramientas empresariales: Microsoft Teams, Google Chat, Feishu
- Otras: IRC, Matrix, LINE, Mattermost, Nostr, etc.

### Ejemplos de Configuración

**Telegram**:

```json
{
  "channels": {
    "telegram": {
      "botToken": "TU_BOT_TOKEN"
    }
  }
}
```

**Discord**:

```json
{
  "channels": {
    "discord": {
      "token": "TU_BOT_TOKEN"
    }
  }
}
```

**WhatsApp**:

```bash
openclaw channels login
```

## Sistema de Habilidades

Las habilidades son la forma en que OpenClaw extiende funcionalidad. Puedes:

### Instalar Habilidades

Instala habilidades de la comunidad desde ClawHub, o crea las tuyas propias.

### Estructura de Habilidades

Cada habilidad es un directorio que contiene un archivo SKILL.md:

```
skills/
  └── mi-habilidad/
      └── SKILL.md
```

### ClawHub

ClawHub es un registro de habilidades. Cuando está habilitado, OpenClaw puede buscar e instalar automáticamente las habilidades necesarias.

## Automatización

### Tareas Programadas

Configura tareas programadas usando Cron:

```bash
openclaw cron add "0 9 * * *" "Resumen matutino"
```

### Webhooks

Configura webhooks para activar automatización:

```bash
openclaw webhook add --url https://tu-endpoint.com/hook
```

### Integración con Gmail

Configura Gmail Pub/Sub para escuchar nuevos correos.

## Funciones Avanzadas

### Control del Navegador

OpenClaw puede controlar un navegador dedicado:

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Visualización Canvas

Canvas es el espacio de trabajo visual de OpenClaw, soportando push/reset A2UI, evaluación y snapshots.

### Funciones de Voz

- **Voice Wake**: Activar vía palabras de activación en macOS/iOS
- **Talk Mode**: Conversación de voz continua (soporte en Android)

### Nodos de Dispositivos

OpenClaw puede conectar varios dispositivos:

- **Nodo macOS**: Ejecución del sistema, notificaciones, Canvas, cámara
- **Nodo iOS**: Canvas, voz, cámara, grabación de pantalla
- **Nodo Android**: Conectar, chat, voz, Canvas, cámara, ubicación, etc.

## Configuración de Seguridad

### Emparejamiento de DM

Por defecto, los remitentes desconocidos necesitan verificación de emparejamiento. Cambia al modo abierto con `dmPolicy: "open"` (no recomendado).

### Gestionar Emparejamiento:

```bash
# Ver lista de emparejamientos
openclaw pairing list

# Aprobar emparejamiento
openclaw pairing approve

# Revocar emparejamiento
openclaw pairing revoke
```

### Ejecutar Diagnósticos

Ejecuta diagnósticos regularmente para seguridad:

```bash
openclaw doctor
```

## Referencia de Comandos

| Comando | Descripción |
|---------|-------------|
| `/status` | Ver estado de la sesión |
| `/new` o `/reset` | Reiniciar sesión |
| `/compact` | Compactar contexto de sesión |
| `/think` | Establecer nivel de pensamiento |
| `/verbose` | Alternar salida detallada |
| `/usage` | Ver estadísticas de uso |
| `/restart` | Reiniciar Gateway |

## Mejores Prácticas

1. **Ejecutar doctor regularmente**: Mantener el sistema saludable
2. **Tener cuidado con DMs**: Revisar configuración de seguridad
3. **Usar modelos fuertes**: Obtener la mejor experiencia
4. **Respaldar regularmente**: Guardar configuraciones importantes

## Solución de Problemas

Problemas comunes:

- **Gateway no inicia**: Verificar puerto 18789
- **Conexión de canal falla**: Verificar credenciales
- **Envío de mensajes falla**: Revisar logs
