# 자주 묻는 질문

이 문서는 Angular 작업 시 발생하는 일반적인 기술적인 질문과 답변을 모았습니다.

## 시작하기

### Angular란 무엇인가요?

Angular는 Google에서 개발하고 관리하는 최신 웹 애플리케이션 구축을 위한 오픈소스 웹 애플리케이션 프레임워크입니다. TypeScript를 주요 언어로 사용하며 컴포넌트 기반 아키텍처, 양방향 데이터 바인딩, 의존성 주입 같은 기능을 제공합니다.

### Angular와 AngularJS의 차이점은 무엇인가요?

AngularJS는 Angular 1.x의 이름입니다—원래 버전입니다. Angular는 TypeScript를 사용하는 완전히 다른 아키텍처로 완전히 재작성되었습니다. 서로 호환되지 않습니다.

### Angular 배우기가 어렵나요?

Angular는 학습 곡선이 있습니다, 특히 의존성 주입과 RxJS 같은 개념은 이해하는 데 시간이 걸립니다. 하지만 문서가 훌륭하고 커뮤니티가 활발하며, 일단 익숙해지면 개발이 매우 효율적입니다.

## 설치 문제

### npm install이 계속 실패합니다. 어떻게 해야 하나요?

1. npm 캐시 지우기: `npm cache clean --force`
2. node_modules와 package-lock.json 삭제
3. 중국 미러 사용
4. Node.js 버전이 호환되는지 확인

### Angular CLI 명령을 찾을 수 없어요?

전역으로 설치되어 있는지 확인:
```bash
npm install -g @angular/cli
```

안 되면 npx 사용:
```bash
npx @angular/cli new my-app
```

### Node.js 버전이 호환되지 않나요?

nvm으로 Node.js 버전 관리:
```bash
nvm install 18
nvm use 18
```

## 개발 문제

### 컴포넌트 스타일이 작동하지 않아요?

1. 스타일 파일 경로가 정확한지 확인
2. styles.scss에 문법 오류가 없는지 확인
3. ViewEncapsulation 설정 확인

### 라우트 네비게이션이 실패해요?

1. 라우트가 올바르게 구성되어 있는지 확인
2. RouterModule이 제대로 가져와졌는지 확인
3. 라우트 가드가 네비게이션을 차단하고 있는지 확인

### HTTP 요청이 실패해요?

1. 백엔드 서비스가 실행 중인지 확인
2. CORS가 구성되어 있는지 확인
3. HttpInterceptor를 사용하여 오류 처리 추가

### 데이터가 업데이트되지 않아요?

1. 변경 감지 전략 확인
2. OnPush 사용 시 수동으로 감지 트리거 필요
3. 데이터 참조가 변경되었는지 확인

## 성능 문제

### 초기 로드가 느려요?

1. 라우트 지연 로딩 활성화
2. 프리로딩 전략 사용
3 이미지 리소스 최적화
4. 프로덕션 모드로 빌드

### 메모리 누수가 있나요?

1. ngOnDestroy에서 Observable 구독 취소
2. takeUntil 또는 async 파이프 사용
3. 타이머와 이벤트 리스너 정리

### 변경 감지가 너무 많이 실행돼요?

1. OnPush 전략 사용
2. 가변 속성과 불변 속성 분리
3. *ngFor 최적화를 위해 trackBy 사용

## 테스트 문제

### 단위 테스트가 실패해요?

1. TestBed 설정 확인
2. 비동기 작업이 제대로 처리되는지 확인
3. 비동기 코드 테스트를 위해 fakeAsync와 tick 사용

### 모듈을 찾을 수 없어요?

tsconfig.json에서 경로가 올바르게 구성되어 있는지 확인하세요.

## 빌드 문제

### 빌드가 실패해요?

1. 실제 오류 메시지 확인
2. TypeScript 컴파일 옵션 검토
3. 캐시 지우고 다시 빌드

### 빌드 파일이 너무 커요?

1. 프로덕션 모드 활성화
2. 지연 로딩 사용
3. 사용하지 않는 의존성 확인

## 일반적인 오류

### ExpressionChangedAfterItHasBeenCheckedError

변경 감지 중에 변경되어서는 안 되는 값을 수정했습니다. ngOnInit에서 뷰 관련 코드를 직접 수정하고 있는지 확인하세요.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

위와 동일—데이터 바인딩 타이밍을 확인하세요.

### Cannot read property of undefined

데이터가 초기화되기 전에 접근하고 있습니다. 옵션 체이닝(?.) 또는 *ngIf를 사용하여 방지하세요.

### CORS 오류

백엔드에서 CORS를 구성하지 않았거나 프론트엔드 요청 헤더에 문제가 있습니다.

### 401/403 오류

인증 실패입니다. 토큰과 권한 구성을 확인하세요.

## 버전 업그레이드

### Angular를 업그레이드하려면 어떻게 하나요?

1. CLI 사용: `ng update @angular/cli`
2. 공식 업그레이드 가이드 확인
3. 의존성을 하나씩 업그레이드
4.一切都 작동하는지 테스트

### 업그레이드가 실패했어요?

1. 코드 백업
2. package.json에서 버전 수동 업데이트
3. node_modules 삭제하고 다시 설치
