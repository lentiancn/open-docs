# Guide d'installation de H2

## Configuration requise

- Java 11 ou supérieur
- Tout système d'exploitation supportant Java (Windows, Linux, macOS, etc.)

## Méthodes d'installation

### Méthode 1 : Télécharger le package ZIP

1. Visitez la page de téléchargement du site officiel : https://www.h2database.com/html/download.html
2. Téléchargez la version "Platform-Independent Zip"
3. Décompressez le fichier ZIP téléchargé dans n'importe quel répertoire, par exemple : C:\h2 ou /opt/h2

### Méthode 2 : Dépendance Maven

Si vous utilisez un projet Java avec Maven ou Gradle, vous pouvez ajouter la dépendance directement :

**Maven :**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle :**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### Méthode 3 : Installateur Windows

1. Téléchargez la version "Windows Installer"
2. Exécutez l'installeur et suivez l'assistant pour terminer l'installation

## Configurer les variables d'environnement (optionnel)

Si vous avez téléchargé le package ZIP et souhaitez utiliser H2 directement depuis la ligne de commande :

**Windows :**
1. Faites un clic droit sur "Ce PC" → "Propriétés"
2. Cliquez sur "Paramètres système avancés"
3. Cliquez sur "Variables d'environnement"
4. Dans les variables système, recherchez Path et cliquez sur "Modifier"
5. Ajoutez le chemin du répertoire bin de H2

**Linux/macOS :**
```bash
export PATH=$PATH:/opt/h2/bin
```

## Vérifier l'installation

### Utiliser la ligne de commande

Après décompression, exécutez dans la ligne de commande :

```bash
java -jar h2.jar
```

Cela démarrera la console web H2.

### Utiliser la connexion JDBC

Pour connecter à la base de données H2 dans le code Java :

```java
// Mode intégré
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// Base de données fichier
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Utiliser la console H2

1. Exécutez la commande : `java -jar h2.jar`
2. Ouvrez un navigateur et accédez à : http://localhost:8082
3. Nom d'utilisateur par défaut : sa, mot de passe vide

## Formats d'URL de connexion

H2 prend en charge plusieurs formats d'URL de connexion :

| Mode | URL exemple | Description |
|------|-------------|-------------|
| Mode mémoire | jdbc:h2:mem:testdb | Données en mémoire |
| Fichier intégré | jdbc:h2:file:./data/testdb | Données dans un fichier |
| Serveur TCP | jdbc:h2:tcp://localhost:9092/testdb | Mode client-serveur |
| TCP chiffré | jdbc:h2:ssl://localhost:9092/testdb | Connexion SSL |
| Compatibilité PostgreSQL | jdbc:h2:postgres://localhost:5432/testdb | Mode compatibilité protocole PostgreSQL |

## Problèmes courants

### Port occupé
H2 utilise le port 9092 par défaut (mode TCP) et 8082 (console web). Si le port est occupé, vous pouvez spécifier un autre port dans l'URL de connexion.

### Problème de version Java
Assurez-vous d'avoir Java 11 ou supérieur installé. Vous pouvez le vérifier avec la commande suivante :

```bash
java -version
```

### Pilote non trouvé
Assurez-vous que le fichier JAR de H2 est dans le classpath. Pour les projets Maven, assurez-vous que la dépendance a été correctement ajoutée.
