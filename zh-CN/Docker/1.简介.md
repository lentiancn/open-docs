# Docker - 简介

Docker 是一个开源的容器化平台，用于开发、部署和运行应用程序。通过容器化技术，Docker 可以将应用程序及其依赖打包成轻量级、可移植的容器，确保应用在任何环境中都能一致运行。

## 什么是容器？

容器是一种轻量级的虚拟化技术，它共享宿主机的操作系统内核，但为应用程序提供独立的运行环境。与传统虚拟机相比，容器更加轻量、启动更快、资源占用更少。

### 容器 vs 虚拟机

| 特性 | 容器 | 虚拟机 |
|------|------|--------|
| 启动时间 | 秒级 | 分钟级 |
| 资源占用 | MB 级 | GB 级 |
| 操作系统 | 共享内核 | 独立操作系统 |
| 隔离性 | 进程级 | 硬件级 |
| 可移植性 | 极高 | 较高 |

## Docker 的核心概念

### 1. 镜像（Image）

镜像是一个只读的模板，用于创建容器。你可以把它想象成面向对象编程中的"类"。镜像可以包含操作系统、应用程序及其依赖。

**镜像特点：**
- 分层存储
- 不可变性
- 可复用性
- 版本控制

### 2. 容器（Container）

容器是镜像的运行实例。你可以启动、停止、删除容器。每个容器都是相互隔离的，拥有自己的文件系统、网络和进程空间。

**容器特点：**
- 轻量级
- 快速启动
- 相互隔离
- 可随时销毁和创建

### 3. 仓库（Repository）

仓库用于存储镜像。Docker Hub 是默认的公共仓库，你也可以搭建私有仓库。

**常见仓库：**
- Docker Hub：https://hub.docker.com
- 阿里云容器镜像服务：https://cr.console.aliyun.com
- GitHub Container Registry：ghcr.io

## Docker 的优势

### 1. 环境一致性

开发、测试、生产环境使用相同的容器镜像，消除"在我机器上能运行"的问题。

### 2. 快速部署

容器可以在秒级启动，快速部署和扩展应用。

### 3. 资源隔离

多个容器相互隔离，一个容器的问题不会影响其他容器。

### 4. 弹性伸缩

可以快速创建和销毁容器，实现应用的弹性伸缩。

### 5. 持续集成/持续部署

完美支持 CI/CD 流程，实现自动化构建和部署。

## Docker 的应用场景

### 1. 微服务架构

每个微服务运行在独立的容器中，便于开发、部署和扩展。

### 2. 持续集成/持续部署

自动化构建、测试和部署流程。

### 3. 快速环境搭建

快速搭建开发、测试环境，无需手动配置。

### 4. 多版本并存

同一台机器上运行不同版本的应用，互不影响。

### 5. 迁移和扩展

容器可以在不同平台间轻松迁移，支持水平扩展。

## 为什么选择 Docker？

1. **快速部署**：秒级启动容器
2. **资源高效**：共享内核，资源占用低
3. **环境一致**：开发生产环境统一
4. **社区活跃**：庞大的社区和生态
5. **云原生**：云原生技术的基础
6. **标准统一**：容器已成为行业标准

## Docker 与其他容器技术

### Docker vs Podman

| 特性 | Docker | Podman |
|------|--------|--------|
| 守护进程 | 需要 | 不需要 |
| root 权限 | 需要 | 可选 |
| 兼容性 | 完全兼容 | 兼容 Docker |
| 安全性 | 较高 | 更高 |

### Docker vs containerd

| 特性 | Docker | containerd |
|------|--------|------------|
| 定位 | 完整平台 | 容器运行时 |
| 功能 | 更全面 | 核心功能 |
| 使用场景 | 开发/生产 | 嵌入式 |
