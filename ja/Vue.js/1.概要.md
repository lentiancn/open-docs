# Vue.js 概要

Vue（発音は /vjuː/、ビューに似ています）はユーザインターフェースを構築するための JavaScript フレームワークです。標準的な HTML、CSS、JavaScript をベースに構築されており、宣言的でコンポーネントベースのプログラミングモデルを提供し、あらゆる複雑さのユーザインターフェースを効率的に開発するのに役立ちます。

## コア機能

### 宣言的レンダリング
Vue は標準的な HTML をテンプレート構文で拡張し、JavaScript の状態に基づいて HTML 出力を宣言的に記述できるようにします。

### リアクティビティ
Vue は自動的に JavaScript の状態変化を追跡し、変化が発生した際に DOM を効率的に更新します。

## プログレッシブフレームワーク

Vue はフロントエンド開発に必要な一般的な機能をカバーするフレームワークであり、エコシステムでもあります。Web は非常に多様であり、私たちが Web 上で構築するものも形や規模が大きく異なる可能性があります。このことを念頭に置いて、Vue は柔軟性と段階的な採用が可能なように設計されています。ユースケースに応じて、Vue はさまざまな方法で使用できます：

- ビルドステップなしで静的 HTML を強化
- 任意のページに Web Components として埋め込み
- シングルページアプリケーション (SPA)
- フルスタック / サーバーサイドレンダリング (SSR)
- Jamstack / 静的サイト生成 (SSG)
- デスクトップ、モバイル、WebGL、さらにはターミナル向けの開発

## シングルファイルコンポーネント

ビルドツールが有効になっているほとんどの Vue プロジェクトでは、HTML に似たファイル形式を使用して Vue コンポーネントを作成します。これはシングルファイルコンポーネント（*.vue ファイル、SFC と略される）と呼ばれます。名前が示す通り、Vue の SFC はコンポーネントのロジック (JavaScript)、テンプレート (HTML)、スタイル (CSS) を単一のファイルにカプセル化します。

## API スタイル

Vue コンポーネントは、オプション API とコンポジション API の2つの異なる API スタイルで作成できます。

### オプション API
オプション API では、data、methods、mounted などのオプションを持つオブジェクトを使用してコンポーネントのロジックを定義します。オプションで定義されたプロパティは、関数内の this に公開され、これはコンポーネントインスタンスを指します。

### コンポジション API
コンポジション API では、インポートされた API 関数を使用してコンポーネントのロジックを定義します。SFC では、コンポジション API は通常 `<script setup>` と一緒に使用されます。setup 属性は、Vue がコンパイル時に変換を実行して、より少ない定型コードでコンポジション API を使用できるようにするためのヒントです。
