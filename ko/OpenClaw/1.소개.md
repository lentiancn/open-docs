# OpenClaw 🦞

## OpenClaw란?

OpenClaw는 즐겨 사용하는 채팅 앱인 WhatsApp, Telegram, Discord, iMessage 등을 AI 코딩 에이전트(예: Pi)에 연결하는 **자체 호스팅 게이트웨이**입니다. 자체 머신(또는 서버)에서 단일 Gateway 프로세스를 실행하면 메시징 앱과 항상 사용 가능한 AI 어시스턴트 사이의 다리가 됩니다.

### 핵심 기능

- **자체 호스팅**: 하드웨어에서 실행, 자신의 규칙
- **멀티 채널**: 하나의 Gateway가 WhatsApp, Telegram, Discord 등을 동시에 지원
- **에이전트 기본**: 도구 사용, 세션, 메모리 및 멀티 에이전트 라우팅을 갖춘 코딩 에이전트용으로 구축
- **오픈 소스**: MIT 라이선스, 커뮤니티 중심

### 대상 사용자

데이터 제어를 포기하거나 호스팅 서비스에 의존하지 않고 어디서나 메시지를 보낼 수 있는 개인 AI 어시스턴트를 원하는 개발자 및 고급 사용자입니다.

### 시스템 요구사항

- Node.js 22 이상
- API 키 (Anthropic 권장)
- 5분 정도의 시간

## 주요 기능

### 멀티 채널 게이트웨이
단일 Gateway 프로세스가 WhatsApp, Telegram, Discord, iMessage 및 기타 주요 채팅 플랫폼을 지원합니다. Mattermost와 같은 확장 플러그인을 통해 더 많은 채널을 추가할 수도 있습니다.

### 멀티 에이전트 라우팅
각각 자체 작업 공간과 세션을 가진 다양한 채널, 계정 또는 사용자에 대해 격리된 에이전트를 실행할 수 있습니다.

### 미디어 지원
이미지, 오디오 및 문서를 보내고 받을 수 있습니다. 미디어 파일은 전사 및 처리될 수 있습니다.

### 웹 제어 UI
브라우저 기반 대시보드를 통해 채팅, 설정 구성, 세션 관리 및 노드 제어가 가능합니다.

### 모바일 노드
카메라 캡처, 화면 녹화, 위치 액세스 등을 활성화하기 위해 iOS 및 Android 장치를 노드로 페어링할 수 있습니다.

### 음성 모드
Voice Wake 및 Talk Mode를 통해 어디서나 음성 상호작용을 활성화하며 macOS, iOS, Android를 지원합니다.

### 자동화
Cron 작업 예약, Webhooks 설정 및 Gmail Pub/Sub 통합으로 복잡한 워크플로 자동화를 구현합니다.

---

## 작동 원리

```
채팅 앱 → Gateway → AI 에이전트
                  ↓
        CLI / 웹 UI / macOS 앱 / 모바일 노드
```

Gateway는 세션, 라우팅 및 채널 연결의 단일 정보 소스입니다.

## 빠른 시작

### 1단계: OpenClaw 설치

**macOS / Linux:**
```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows (PowerShell):**
```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

**npm을 통해:**
```bash
npm install -g openclaw@latest
```

### 2단계: 온보딩 마법사 실행

```bash
openclaw onboard --install-daemon
```

### 3단계: Gateway 상태 확인

```bash
openclaw gateway status
```

### 4단계: 제어 UI 열기

```bash
openclaw dashboard
```

또는 브라우저에서 http://127.0.0.1:18789/를 방문하세요.

## 채널 설정

### WhatsApp

```bash
openclaw channels login
openclaw gateway --port 18789
```

화이트리스트를 설정하려면 `channels.whatsapp.allowFrom`을 사용하세요.

### Telegram

구성 파일에 추가:
```json5
{
  channels: {
    telegram: {
      botToken: "당신의Bot토큰",
    },
  },
}
```

### Discord

```json5
{
  channels: {
    discord: {
      token: "당신의Bot토큰",
    },
  },
}
```

## 구성

OpenClaw는 `~/.openclaw/openclaw.json`에 JSON5 구성 파일을 사용합니다.

### 최소 구성 예

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
    },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
    },
  },
}
```

### 일반적인 구성 옵션

- **agents.defaults.model**: 기본 및 폴백 모델 설정
- **channels**: 각 채널의 연결 및 정책 구성
- **session**: 세션 범위 및 재설정 정책 제어
- **cron**: 예약된 작업 구성
- **hooks**: Webhook 엔드포인트 구성
- **browser**: 브라우저 제어 구성

## 보안 권장사항

- 수신 DM을 신뢰할 수 없는 입력으로 처리
- 페어링 전략 사용 - 알 수 없는 발신자는 승인이 필요
- 그룹의 경우 멘션 모드를 활성화하는 것이 좋음
- 샌드박스 모드를 사용하여 비주 세션 격리
- `openclaw doctor`를 정기적으로 실행하여 구성 위험 확인
