# Guide d'installation de Kotlin

## Prérequis

Avant d'installer Kotlin, asegurez-vous que votre ordinateur répond aux exigences système de base. Kotlin peut fonctionner sur plusieurs systèmes d'exploitation, notamment Windows, macOS, Linux et autres systèmes de type Unix. Quel que soit le système d'exploitation que vous utilisez, assurez-vous d'avoir suffisamment d'espace disque pour installer les outils de développement et exécuter les programmes compilés.

Pour les utilisateurs Windows, Windows 10 ou supérieur est recommandé pour une meilleure expérience. Les utilisateurs de macOS ont besoin de macOS 10.14 ou supérieur. Les utilisateurs Linux peuvent utiliser n'importe quelle distribution majeure comme Ubuntu, Fedora ou Debian.

## Installation via IDE

### IntelliJ IDEA

IntelliJ IDEA est l'environnement de développement intégré phare de JetBrains et fournit le support le plus complet pour Kotlin. Si votre objectif principal est le développement Kotlin, IntelliJ IDEA est le meilleur choix.

IntelliJ IDEA est disponible en deux versions : Community (gratuite et open source) et Ultimate (pour les utilisateurs Enterprise avec des fonctionnalités avancées). La version Community est gratuite et fournit suffisamment de fonctionnalités pour le développement Kotlin. La version Ultimate offre des fonctionnalités entreprise supplémentaires comme des outils de base de données avancés et des fonctionnalités de sécurité d'entreprise.

Pour installer, visitez le site officiel de JetBrains (https://www.jetbrains.com/idea/download/) et téléchargez l'installateur approprié pour votre système d'exploitation. Après le téléchargement, exécutez l'installateur et suivez l'assistant pour terminer l'installation. Pendant l'installation, vous pouvez choisir le chemin d'installation et créer des raccourcis bureau.

Lorsque vous lancez IntelliJ IDEA pour la première fois, l'IDE détectera automatiquement et vous demandera d'installer le plugin Kotlin. Si la détection automatique ne fonctionne pas, vous pouvez rechercher manuellement Kotlin dans le marketplace des plugins et l'installer. Après l'installation, redémarrez l'IDE pour commencer le développement Kotlin.

### Android Studio

Android Studio est l'IDE de développement Android officiel de Google avec un support intégré pour Kotlin. Si vous développez principalement pour Android, Android Studio est le choix idéal.

Visitez le site des développeurs Android (https://developer.android.com/studio) pour télécharger Android Studio. Après le téléchargement, exécutez l'installateur pour terminer l'installation. Android Studio inclut le plugin Kotlin par défaut, aucune configuration supplémentaire n'est nécessaire pour commencer à développer.

Android Studio publie plusieurs versions de mise à jour chaque année, chacune contenant le dernier support Kotlin et les outils de développement Android. Il est recommandé de maintenir votre version à jour pour obtenir la meilleure expérience de développement et les dernières fonctionnalités du langage.

## Installation via ligne de commande

En plus d'utiliser un IDE, vous pouvez également installer le compilateur Kotlin en ligne de commande. Cette approche convient aux développeurs qui préfèrent un environnement de développement léger ou qui ont besoin de Kotlin pour le développement côté serveur.

### Installation manuelle

Premièrement, visitez la page des versions Kotlin sur GitHub (https://github.com/JetBrains/kotlin/releases) pour télécharger la dernière version du compilateur. Le téléchargement est une archive zip ; extrayez-la dans un répertoire de votre choix.

Après l'extraction, ajoutez le répertoire kotlinc/bin à la variable d'environnement PATH de votre système afin que vous puissiez utiliser la commande kotlinc depuis la ligne de commande. Sous Windows, ajoutez le chemin via Propriétés système > Variables d'environnement. Sous macOS et Linux, modifiez votre fichier ~/.bashrc ou ~/.zshrc et ajoutez une ligne comme :

export PATH="$PATH:/path/to/kotlinc/bin"

Après l'ajout du chemin, ouvrez une nouvelle fenêtre de terminal et tapez kotlinc -version. Si vous voyez des informations de version, l'installation a réussi.

### Utilisation de SDKMAN !

SDKMAN ! est un outil pour gérer les versions des outils de développement Java et fonctionne commodément sur les systèmes de type Unix. Si vous utilisez macOS, Linux, Cygwin, FreeBSD ou Solaris, SDKMAN ! est la méthode recommandée pour installer Kotlin.

Premièrement, installez SDKMAN ! en exécutant cette commande dans votre terminal :

curl -s "https://get.sdkman.io" | bash

Après l'installation, rechargez la configuration de l'environnement :

source ~/.sdkman/bin/sdkman-init.sh

Maintenant, vous pouvez utiliser la commande sdk pour installer Kotlin :

sdk install kotlin

Pendant l'installation, il vous sera demandé de confirmer la version. Après confirmation, l'installation est terminée. Vous pouvez vérifier l'installation avec kotlinc -version.

### Utilisation de Homebrew

Sur macOS, vous pouvez également utiliser le gestionnaire de packages Homebrew pour installer Kotlin. Assurez-vous d'abord que Homebrew est installé, puis exécutez :

brew update
brew install kotlin

Homebrew téléchargera et installera automatiquement le dernier compilateur Kotlin. Vérifiez l'installation avec kotlinc -version.

### Utilisation de Snap

Sur Ubuntu 16.04 et versions ultérieures, vous pouvez utiliser Snap pour installer Kotlin :

sudo snap install --classic kotlin

L'installation via Snap installe Kotlin comme une application accessible depuis n'importe où via la ligne de commande.

## Créer votre premier programme Kotlin

Quelle que soit la méthode d'installation choisie, vous pouvez créer et exécuter votre premier programme Kotlin avec ces étapes.

Premièrement, créez un fichier nommé hello.kt avec le contenu suivant :

fun main() {
    println("Hello, World!")
}

C'est le programme Kotlin le plus simple, définissant une fonction main qui est le point d'entrée d'un programme Kotlin. La fonction println produit du texte sur la console.

Si vous utilisez le compilateur en ligne de commande, compiler et exécuter ce programme nécessite deux étapes. Premièrement, compilez :

kotlinc hello.kt -include-runtime -d hello.jar

Le paramètre -include-runtime emballe la bibliothèque d'exécution Kotlin dans le fichier jar généré, le rendant autonome. Le paramètre -d spécifie le chemin du fichier de sortie.

Après la compilation, exécutez le programme :

java -jar hello.jar

Vous devriez voir "Hello, World !" s'afficher dans la console.

Si vous ne voulez pas générer un fichier jar autonome, vous pouvez également utiliser la fonction d'exécution interactive de kotlinc :

kotlinc -script hello.kt

Ou utilisez la commande kotlin pour exécuter les fichiers class compilés directement :

kotlin -classpath hello.jar HelloKt

 Ici, HelloKt est le nom de la classe principale généré par le compilateur Kotlin pour le fichier nommé hello.kt.

## Utilisation du REPL

Kotlin fournit un environnement REPL (Read-Eval-Print Loop) interactif où vous pouvez entrer du code Kotlin et voir immédiatement les résultats. Ceci est utile pour apprendre les fonctionnalités du langage ou tester rapidement des extraits de code.

Pour démarrer le REPL, exécutez simplement kotlinc sans paramètre :

kotlinc

Cela entre en mode REPL. Vous pouvez entrer n'importe quel code Kotlin valide et appuyez sur Entrée pour voir le résultat :

>>> val x = 10
>>> val y = 20
>>> x + y
30

Entrez :quit ou appuyez sur Ctrl+C pour quitter le REPL.

## Scripts Kotlin

Kotlin peut également être utilisé comme langage de script. Les fichiers de script Kotlin ont l'extension .kts, et le code de niveau supérieur dans le fichier est traité comme du code exécutable.

Par exemple, créez un script nommé list_folders.kts :

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

Exécutez le script :

kotlinc -script list_folders.kts -- -d /path/to/folder

## Configuration des outils de construction

Dans les projets réels, Gradle ou Maven sont généralement utilisés comme outils de construction. Les deux supportent natiquement les projets Kotlin.

### Gradle

Gradle est l'outil de construction le plus populaire pour le développement Kotlin. L'équipe Kotlin fournit également Kotlin DSL, permettant d'écrire les scripts de construction en syntaxe Kotlin.

Pour créer un projet Gradle Kotlin basique, ajoutez le plugin Kotlin à votre projet. Votre fichier build.gradle doit contenir ces éléments :

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

Ou en utilisant Kotlin DSL :

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

Créez ensuite la structure de répertoire du code source, généralement src/main/kotlin. Ajoutez des fichiers source Kotlin et utilisez gradle build pour compiler le projet.

### Maven

Maven est un autre outil de construction populaire pour les projets Java qui supporte également les projets Kotlin. Ajoutez les dépendances Kotlin à votre fichier pom.xml.

## Vérification de l'installation

Après l'installation, vous pouvez vérifier que tout fonctionne correctement.

Premièrement, vérifiez le compilateur Kotlin :

kotlinc -version

Vous devriez voir quelque chose comme "Kotlin compiler version 2.3.10".

Puis vérifiez la bibliothèque standard Kotlin :

kotlin -e 'println("Hello Kotlin!")'

Si tout fonctionne, l'installation est réussie et vous pouvez commencer votre voyage avec Kotlin.

## Étapes suivantes

Après avoir installé Kotlin, il est recommandé de continuer à apprendre en lisant d'abord le Tour officiel de Kotlin pour comprendre les fonctionnalités de base du langage ; puis en choisissant une direction de développement qui vous intéresse comme le développement Android, le développement côté serveur ou le développement multiplateforme ; enfin en pratiquant avec de vrais projets pour appliquer ce que vous avez appris.

Kotlin fournit de nombreuses ressources d'apprentissage, notamment la documentation officielle, des tutoriels en ligne et des cours vidéo. Vous pouvez trouver des liens vers ces ressources sur le site Web de Kotlin.
