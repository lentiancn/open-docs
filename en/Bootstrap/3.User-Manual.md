# User Manual

This guide walks you through Bootstrap 5's core features — the grid system, components, CSS utilities, and more.

## The Grid System

The grid system is Bootstrap's backbone, giving you a 12-column layout.

### Basic Usage

```html
<div class="container">
    <div class="row">
        <div class="col">Column 1</div>
        <div class="col">Column 2</div>
        <div class="col">Column 3</div>
    </div>
</div>
```

### Responsive Breakpoints

| Class | Screen Width | Description |
|-------|--------------|-------------|
| `.col-xs` | < 576px | Extra small |
| `.col-sm` | ≥ 576px | Small |
| `.col-md` | ≥ 768px | Medium |
| `.col-lg` | ≥ 992px | Large |
| `.col-xl` | ≥ 1200px | Extra large |
| `.col-xxl` | ≥ 1400px | Extra extra large |

### Column Offsets

```html
<div class="row">
    <div class="col-md-4">.col-md-4</div>
    <div class="col-md-4 offset-md-4">.col-md-4 .offset-md-4</div>
</div>
```

### Responsive Columns

```html
<div class="row">
    <div class="col-12 col-md-6 col-lg-4">
        Full width on mobile, half on tablet, one-third on large screens
    </div>
</div>
```

## Modals

```html
<!-- Trigger Button -->
<button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#exampleModal">
    Open Modal
</button>

<!-- Modal -->
<div class="modal fade" id="exampleModal" tabindex="-1">
    <div class="modal-dialog">
        <div class="modal-content">
            <div class="modal-header">
                <h5 class="modal-title">Modal Title</h5>
                <button type="button" class="btn-close" data-bs-dismiss="modal"></button>
            </div>
            <div class="modal-body">
                Content goes here...
            </div>
            <div class="modal-footer">
                <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
                <button type="button" class="btn btn-primary">Save</button>
            </div>
        </div>
    </div>
</div>
```

## Best Practices

1. **Use semantic HTML** — make use of header, nav, main, footer, and similar tags
2. **Keep it simple** — avoid over-nesting
3. **Start mobile-first** — design for mobile first, then scale up
4. **Leverage utility classes** — tweak styles quickly
5. **Customize with Sass variables** — keep things consistent across your project
