# Руководство пользователя Webpack

В этом руководстве показано, как использовать Webpack для создания современных JavaScript-приложений.

## Быстрый старт

### Создание проекта

```bash
mkdir my-project
cd my-project
npm init -y
npm install webpack webpack-cli --save-dev
```

### Создание файлов входа

```javascript
// src/index.js
import './style.css';
import { add } from './math.js';

console.log('Hello Webpack!');
console.log(add(2, 3));
```

```javascript
// src/math.js
export function add(a, b) {
  return a + b;
}
```

```css
/* src/style.css */
body {
  background: #f0f0f0;
  font-family: sans-serif;
}
```

```html
<!-- src/index.html -->
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Webpack App</title>
</head>
<body>
  <h1>Hello Webpack!</h1>
  <script src="index.js"></script>
</body>
</html>
```

### Создание файла конфигурации

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
```

### Запуск сборки

```bash
npx webpack
```

## Основные концепции

### Точка входа (Entry)

```javascript
module.exports = {
  // Одна точка входа
  entry: './src/index.js',
  
  // Несколько точек входа
  entry: {
    home: './src/home.js',
    about: './src/about.js',
  },
};
```

### Вывод (Output)

```javascript
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    clean: true,
    publicPath: '/',
  },
};
```

### Загрузчики (Loaders)

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader',
      },
    ],
  },
};
```

### Плагины (Plugins)

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
    new MiniCssExtractPlugin({
      filename: '[name].css',
    }),
  ],
};
```

### Режим (Mode)

```javascript
// Режим разработки
module.exports = {
  mode: 'development',
  devtool: 'source-map',
};

// Режим production
module.exports = {
  mode: 'production',
  devtool: false,
};
```

## Часто используемые загрузчики

### Обработка CSS

```bash
npm install --save-dev style-loader css-loader
```

```javascript
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader'],
}
```

### Обработка Sass

```bash
npm install --save-dev sass sass-loader
```

```javascript
{
  test: /\.scss$/,
  use: ['style-loader', 'css-loader', 'sass-loader'],
}
```

### Обработка изображений

```javascript
{
  test: /\.(png|jpg|gif)$/,
  type: 'asset/resource',
}
```

### Обработка шрифтов

```javascript
{
  test: /\.(woff|woff2|eot|ttf|otf)$/,
  type: 'asset/resource',
}
```

## Сервер разработки

### Установка

```bash
npm install --save-dev webpack-dev-server
```

### Настройка

```javascript
module.exports = {
  devServer: {
    static: './dist',
    hot: true,
    port: 3000,
    open: true,
    compress: true,
  },
};
```

### Запуск

```bash
webpack serve --open
```

Или добавить скрипты в package.json:

```json
{
  "scripts": {
    "start": "webpack serve --mode development",
    "build": "webpack --mode production"
  }
}
```

## Разделение кода

### Динамический импорт

```javascript
// Ленивая загрузка маршрутов
const Home = () => import('./Home.vue');
const About = () => import('./About.vue');

// Условная загрузка
if (condition) {
  import('./module').then(module => {
    module.doSomething();
  });
}
```

### SplitChunks

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};
```

## Конфигурация для разных сред

### Среда разработки

```javascript
module.exports = {
  mode: 'development',
  devtool: 'inline-source-map',
  devServer: {
    static: './dist',
    hot: true,
  },
};
```

### Production-среда

```javascript
module.exports = {
  mode: 'production',
  devtool: 'source-map',
  optimization: {
    minimize: true,
  },
};
```

### Мультиконфигурация

```javascript
const devConfig = require('./webpack.dev');
const prodConfig = require('./webpack.prod');

module.exports = (env, argv) => {
  return argv.mode === 'production' ? prodConfig : devConfig;
};
```

## Оптимизация производительности

### 1. Использовать production-режим

```javascript
module.exports = {
  mode: 'production',
};
```

### 2. Включить Tree Shaking

```javascript
// package.json
{
  "sideEffects": false
}
```

### 3. Разделение кода

```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
  },
}
```

### 4. Оптимизация кэширования

```javascript
output: {
  filename: '[name].[contenthash].js',
  chunkFilename: '[name].[contenthash].chunk.js',
}
```

### 5. Минификация

```javascript
const TerserPlugin = require('terser-webpack-plugin');

optimization: {
  minimizer: [new TerserPlugin()],
}
```

## Частые проблемы

### Ошибка разрешения модуля?

Проверьте конфигурацию resolve:

```javascript
resolve: {
  extensions: ['.js', '.json'],
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
}
```

### Сборка слишком медленная?

- Используйте кэш
- Исключите node_modules
- Используйте thread-loader

```javascript
{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: 'thread-loader',
    options: {
      workers: 2,
    },
  },
}
```
