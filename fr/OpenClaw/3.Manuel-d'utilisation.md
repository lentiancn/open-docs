# Manuel Utilisateur OpenClaw

Ce manuel couvre l'utilisation quotidienne et les fonctionnalités principales d'OpenClaw.

## Concepts de Base

### Gateway

Le Gateway est le composant central d'OpenClaw, responsable de la gestion des sessions, du routage des messages et des connexions de canal. Vous n'avez besoin d'exécuter qu'un seul processus Gateway, et il gère automatiquement toutes les demandes de connexion de canal.

### Canaux

Les canaux sont les applications de chat que vous utilisez pour communiquer avec votre assistant IA. OpenClaw prend en charge la connexion simultanée de plusieurs canaux, y compris WhatsApp, Telegram, Discord, iMessage et plus encore.

### Agents

Les agents sont des instances d'assistants IA. Chaque agent a son propre espace de travail et ses propres sessions. Vous pouvez configurer plusieurs agents pour gérer différentes tâches indépendamment.

## Opérations Quotidiennes

### Démarrer et Arrêter le Gateway

```bash
# Démarrer le Gateway (service en arrière-plan)
openclaw gateway start

# Arrêter le Gateway
openclaw gateway stop

# Redémarrer le Gateway
openclaw gateway restart

# Vérifier le statut
openclaw gateway status

# Exécuter au premier plan (pour le débogage)
openclaw gateway --port 18789
```

### Envoyer des Messages

```bash
# Envoyer un message direct
openclaw message send --target +15555550123 --message "Bonjour"

# Envoyer à un groupe
openclaw message send --target "Nom du Groupe" --message "Bonjour à tous"
```

### Gérer la Configuration

```bash
# Voir la configuration actuelle
openclaw config get

# Définir une valeur de configuration
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# Supprimer une valeur de configuration
openclaw config unset channels.whatsapp.enabled
```

## Gestion de la Configuration

### Emplacement du Fichier de Configuration

Le fichier de configuration est situé à `~/.openclaw/openclaw.json`.

### Rechargement à Chaud

OpenClaw prend en charge le rechargement à chaud de la configuration — vous pouvez modifier le fichier de configuration sans redémarrer manuellement le Gateway. La plupart des modifications de configuration sont appliquées automatiquement.

### Exemples de Configuration Courants

#### Définir les Sources de Messages Autorisées

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### Définir les Règles de Mention dans les Groupes

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### Configurer Plusieurs Agents

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## Gestion des Canaux

### Ajouter de Nouveaux Canaux

```bash
# Se connecter à WhatsApp
openclaw channels login whatsapp

# Se connecter à Telegram
openclaw channels login telegram
```

### Voir les Canaux Connectés

```bash
openclaw channels list
```

## Fonctionnalités Avancées

### Tâches Cron

Configurer des tâches planifiées:

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Heartbeat

Configurer des vérifications périodiques:

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Mode Sandbox

Exécuter les agents dans des conteneurs Docker isolés:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

Configurer des points de terminaison webhook HTTP:

```json5
{
  hooks: {
    enabled: true,
    token: "votre-token-secret",
    path: "/hooks"
  }
}
```

## Dépannage

### Voir les Journaux

```bash
# Voir les journaux du Gateway
openclaw logs

# Suivre les journaux en temps réel
openclaw logs -f
```

### Vérification de Santé

```bash
openclaw health
```

### Diagnostiquer les Problèmes

```bash
openclaw doctor
```

## Interface de Contrôle

OpenClaw fournit une interface de contrôle Web où vous pouvez:

- Envoyer et recevoir des messages
- Gérer la configuration
- Voir l'historique des sessions
- Gérer les nœuds mobiles
- Surveiller l'état du Gateway

Accédez à: `http://127.0.0.1:18789/`
