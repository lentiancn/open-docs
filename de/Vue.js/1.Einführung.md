# Vue.js Einführung

Vue (ausgesprochen /vjuː/, ähnlich wie "view") ist ein JavaScript-Framework zum Erstellen von Benutzeroberflächen. Es baut auf standardmäßigem HTML, CSS und JavaScript auf und bietet ein deklaratives, komponentenbasiertes Programmiermodell, das Ihnen hilft, Benutzeroberflächen jeder Komplexität effizient zu entwickeln.

## Kernfunktionen

### Deklaratives Rendering
Vue erweitert standardmäßiges HTML um eine Templatesyntax, die es uns ermöglicht, die HTML-Ausgabe deklarativ basierend auf dem JavaScript-Zustand zu beschreiben.

### Reaktivität
Vue verfolgt automatisch Änderungen am JavaScript-Zustand und aktualisiert das DOM effizient, wenn Änderungen auftreten.

## Das Progressive Framework

Vue ist ein Framework und Ökosystem, das die meisten gängigen Funktionen abdeckt, die für die Frontend-Entwicklung benötigt werden. Das Web ist extrem vielfältig – die Dinge, die wir im Web erstellen, können sich in Form und Umfang stark unterscheiden. Mit diesem Gedanken wurde Vue so konzipiert, dass es flexibel und schrittweise einsetzbar ist. Je nach Anwendungsfall kann Vue auf verschiedene Weise verwendet werden:

- Verbesserung statischen HTML ohne Build-Schritt
- Einbettung als Web Components auf jeder Seite
- Single-Page-Anwendung (SPA)
- Fullstack / Server-Side Rendering (SSR)
- Jamstack / Statische Site-Generierung (SSG)
- Zielgerichtet auf Desktop, Mobile, WebGL und sogar das Terminal

## Single-File-Komponenten

In den meisten Vue-Projekten, die Build-Tools verwenden, erstellen wir Vue-Komponenten mit einem HTML-ähnlichen Dateiformat namens Single-File Component (auch bekannt als *.vue-Dateien, abgekürzt als SFC). Eine Vue-SFC kapselt, wie der Name schon sagt, die Komponentenlogik (JavaScript), das Template (HTML) und die Stile (CSS) in einer einzigen Datei.

## API-Stile

Vue-Komponenten können in zwei verschiedenen API-Stilen erstellt werden: Options-API und Composition-API.

### Options-API
Mit der Options-API definieren wir die Logik einer Komponente mithilfe eines Objekts mit Optionen wie data, methods und mounted. Von Optionen definierte Eigenschaften werden innerhalb von Funktionen auf this verfügbar gemacht, was auf die Komponenteninstanz verweist.

### Composition-API
Mit der Composition-API definieren wir die Logik einer Komponente mithilfe importierter API-Funktionen. In SFCs wird die Composition-API typischerweise mit `<script setup>` verwendet. Das setup-Attribut ist ein Hinweis, der Vue veranlasst, Kompilierzeit-Transformationen durchzuführen, die es uns ermöglichen, die Composition-API mit weniger Boilerplate-Code zu verwenden.
