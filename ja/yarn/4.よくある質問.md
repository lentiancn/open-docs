# Yarn よくある質問

## インストールに関する質問

### Q: Yarn のバージョンをどのように選べばよいですか？
**A**: 
- **新規プロジェクト**: モダンな機能とより良いパフォーマンスを享受するために Yarn v4+ を使用することを推奨
- **既存プロジェクト**: Yarn v1 をすでに使用している場合は、引き続き使用するか、モダンバージョンへの移行を検討
- **エンタープライズ環境**: Yarn v1 はより安定した長期サポートを提供し、v4+ はより革新的な機能を提供

### Q: Corepack とは何ですか？なぜ必要なのですか？
**A**: Corepack は Node.js にバンドルされている中間ツールで、プロジェクトごとにパッケージマネージャーのバージョンを設定できます。チームメンバーが同じ Yarn バージョンを使用することを保証し、バージョンの不整合問題を回避します。

### Q: Yarn のインストール時に権限エラーが発生した場合はどうすればよいですか？
**A**: 
- `npm install --global yarn` を使用する際に `--unsafe-perm` フラグを追加
- nvm を使用して Node.js のバージョンを管理
- CI/CD 環境では、通常グローバルインストールは不要

## 使用に関する質問

### Q: Yarn と npm の違いは何ですか？
**A**: 
- **速度**: Yarn は並列インストールとオフラインキャッシュを使用し、通常より高速
- **決定性**: Yarn は一貫性のあるインストールを保証 (yarn.lock)
- **機能**: Yarn はワークスペース、PnP、プラグインシステムなどの高度な機能を提供
- **コマンド**: Yarn はすべてのコマンドを再実装し、より良い開発者エクスペリエンスを提供

### Q: PnP (Plug'n'Play) とは何ですか？
**A**: PnP は Yarn v2+ の機能で、node_modules ディレクトリを排除し、依存関係を直接キャッシュから解決します。これによりディスク容量を節約し、インストール速度を向上させますが、PnP をサポートしていないツールと互換性がない可能性があります。

### Q: プロジェクトでワークスペースを使用するにはどうすればよいですか？
**A**: 
1. ルートの `package.json` に `"workspaces"` フィールドを追加
2. サブパッケージディレクトリを作成 (例: `packages/`)
3. サブパッケージ内に個別の `package.json` を作成
4. `yarn workspace <name> <command>` を使用して特定のワークスペースでコマンドを実行

## バージョン移行に関する質問

### Q: Yarn v1 から v4+ に移行するにはどうすればよいですか？
**A**: 
1. プロジェクトが Git バージョン管理下にあることを確認
2. `yarn set version stable` を実行 (v4+ の場合)
3. 移行ガイドに従って設定を段階的に更新
4. すべての機能をテストして互換性を確認

### Q: 移行後に互換性の問題が発生した場合はどうすればよいですか？
**A**: 
- PnP をサポートしていないツールがあるかどうかを確認し、PnP を無効化可能: `yarn config set nodeLinker node-modules`
- 依存関係を互換性のあるバージョンに更新
- 公式移行ドキュメントを参照して具体的な解決策を確認

## トラブルシューティング

### Q: `yarn install` が失敗した場合はどうすればよいですか？
**A**: 
- キャッシュをクリア: `yarn cache clean`
- `yarn.lock` を削除して再インストール
- ネットワーク接続とレジストリ設定を確認
- Yarn v4+ の場合、`.yarnrc.yml` 設定を確認

### Q: Yarn のバージョンを確認するにはどうすればよいですか？
**A**: 
- グローバルバージョン: `yarn --version`
- プロジェクトバージョン: `.yarn/releases/` ディレクトリまたは `.yarnrc.yml` ファイルを確認

### Q: Yarn コマンドの実行が非常に遅い場合はどうすればよいですか？
**A**: 
- オフラインキャッシュが有効になっていることを確認
- ネットワーク接続を確認
- 大規模プロジェクトの場合、ワークスペースを使用してコードベースを分割することを検討
- パフォーマンスの改善のために最新バージョンに更新
