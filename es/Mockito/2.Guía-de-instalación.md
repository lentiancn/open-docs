# Guia de instalacion de Mockito

## Requisitos del entorno

Antes de instalar Mockito, asegurese de que su entorno de desarrollo cumpla con los siguientes requisitos.

Mockito 5.x requiere JDK 11 o superior. Si su proyecto necesita soportar JDK 8, puede usar la serie Mockito 4.x. Se recomienda usar la ultima version de Mockito para obtener las ultimas funciones y correcciones de seguridad.

## Instalacion con Maven

### Agregar dependencias

Agregue la dependencia mockito-core al archivo pom.xml de su proyecto Maven. Se recomienda usar una propiedad para definir la version de Mockito para una mejor gestion de versiones.

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Si necesita usar la funcionalidad inline de Mockito para simular clases y métodos finales, tambien necesita agregar la dependencia mockito-inline.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### Versiones snapshot

Mockito publica versiones snapshot en el repositorio Sonatype despues de cada cambio de código. Si necesita usar las ultimas funciones en desarrollo, puede agregar la direccion del repositorio snapshot a su pom.xml.

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Instalacion con Gradle

### Agregar dependencias

Agregue la declaracion de dependencia al archivo build.gradle de su proyecto Gradle.

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

Si usa el archivo Kotlin DSL de Gradle (build.gradle.kts), la sintaxis es ligeramente diferente.

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### Usar BOM para gestion de versiones

Usar el plugin Spring Dependency Management o la funcion BOM de Gradle puede simplificar la gestion de versiones.

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## Descarga manual

Si su proyecto no usa Maven o Gradle, puede descargar los archivos JAR directamente del repositorio Maven Central. Visite Maven Central y busque org.mockito para encontrar todas las versiones de mockito-core y mockito-all (descontinuado).

Despues de descargar, agregue los archivos JAR al classpath de su proyecto. Tenga en cuenta que tambien necesita descargar bibliotecas dependientes como Byte Buddy y Objenesis.

## Integracion con Spring Boot

spring-boot-starter-test de Spring Boot ya incluye la dependencia de Mockito. Si su proyecto usa Spring Boot, generalmente no necesita agregar la dependencia de Mockito por separado.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Si necesita usar Mocks mas convenientemente en un entorno Spring, puede usar la anotacion @MockBean, que inyecta automaticamente objetos simulados en el contenedor Spring.

## Integracion con JUnit 5

### Usar MockitoExtension

JUnit 5 introdujo un modelo de extension, y Mockito proporciona MockitoExtension para soportar JUnit 5. No se necesita configuración adicional de dependencia, ya que mockito-core ya contiene el soporte necesario.

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### Usar MockitoJUnitRunner

Para JUnit 4, puede usar MockitoJUnitRunner para inicializar automaticamente objetos Mock.

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## Verificacion de instalacion

Despues de la instalacion, puede verificar que Mockito este correctamente configurado creando una clase de prueba simple. Si la prueba pasa despues de ejecutarla, Mockito esta instalado correctamente.

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito instalado correctamente!");
    }
}
```

## Recomendaciones de seleccion de version

Para nuevos proyectos, se recomienda usar la ultima version de Mockito 5.x. Si su proyecto necesita compatibilidad con JDK 8, esta disponible la serie Mockito 4.x. Siempre use la ultima version estable para mejor rendimiento y seguridad.
