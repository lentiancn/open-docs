# Flutter 常見問題

常見問題解答。

## 安裝問題

### Q: Flutter 下載失敗？

**解決方案：**
1. 使用鏡像：
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
export PUB_CACHE=/path/to/cache
```
2. 使用代理
3. 手動下載 SDK

### Q: flutter doctor 報錯？

**解決方案：**
1. 檢查 PATH 配置
2. 確認 Git 已安裝
3. 運行 `flutter doctor -v` 查看詳細信息

### Q: Android SDK 找不到？

**解決方案：**
```bash
flutter config --android-sdk /path/to/android-sdk
export ANDROID_SDK_ROOT=/path/to/android-sdk
```

### Q: macOS Xcode 命令行工具未安裝？

**解決方案：**
```bash
xcode-select --install
```

## 開發問題

### Q: 熱重載不生效？

**解決方案：**
1. 保存文件後等待熱重載
2. 檢查控制台錯誤
3. 嘗試完全重啟應用

### Q: 構建 APK 失敗？

**解決方案：**
1. 運行 `flutter clean`
2. 更新 Gradle
3. 檢查網絡連接

### Q: 模擬器啟動失敗？

**解決方案：**
1. 啟用 CPU 虛擬化
2. 檢查 HAXM 安裝
3. 使用真機調試

### Q: 組件不顯示？

**解決方案：**
1. 檢查 build() 方法返回值
2. 確認添加到了 MaterialApp
3. 檢查佈局約束

### Q: 狀態不更新？

**解決方案：**
1. 確認使用了 setState()
2. 檢查 StatefulWidget 是否正確
3. 確認使用了正確的 Context

### Q: 內存洩漏？

**解決方案：**
1. 正確 dispose 控制器
2. 使用 Provider 時正確清理
3. 避免在 build 中創建對象

## 性能問題

### Q: 應用卡頓？

**解決方案：**
1. 使用 const 構造函數
2. 避免不必要的 setState
3. 使用 ListView.builder
4. 優化圖片加載

### Q: APK 體積太大？

**解決方案：**
1. 使用混淆
2. 移除調試符號
3. 使用 split-per-abi
4. 清理未使用的資源

### Q: 啟動時間長？

**解決方案：**
1. 減少 init() 操作
2. 延遲加載組件
3. 使用 Deferred Components

## 平台問題

### Q: iOS 構建失敗？

**解決方案：**
1. 確認 Xcode 版本
2. 更新 CocoaPods
3. 檢查簽名證書

### Q: Web 構建失敗？

**解決方案：**
1. 確認 Flutter Web 支持
2. 檢查瀏覽器兼容性
3. 更新 Chrome

### Q: 平台通道不工作？

**解決方案：**
1. 檢查方法名匹配
2. 確認返回值類型
3. 查看控制台錯誤
