# Mockito 자주 묻는 질문

## 기본 질문

### Mockito와 PowerMock의 차이점은 무엇인가요?

Mockito는 주로 일반 객체를 모킹하는 데 사용되는 반면, PowerMock은 정적 메서드, 생성자, private 메서드 등을 모킹할 수 있습니다. 그러나 PowerMock은 바이트코드 조작 기술을 사용하여 테스트 격리를 깨뜨려 테스트가 단위 테스트보다 통합 테스트에 더 가깝습니다. 공식 권장 사항은 먼저 Mockito를 선호하고, Mockito가 정적 메서드 모킹과 같은 특정 시나리오를 정말 처리할 수 없는 경우에만 PowerMock을 고려하는 것입니다. Mockito 2.x부터는 mockito-inline을 통해 정적 메서드 모킹도 지원됩니다.

### Spy와 Mock을 언제 사용해야 하나요?

대부분의 실제 행동이 필요하지만 몇 가지 메서드만 모킹하려면 Spy를 사용하세요. Spy는 대부분의 실제 객체 기능을 유지하면서 특정 메서드만 스텁할 수 있습니다. Mock은 실제 객체를 완전히 대체하므로 실제 메서드 로직이 실행되지 않습니다. 객체 동작을 완전히 제어해야 할 때 Mock을 사용하세요. 실제로 Spy는 상대적으로 거의 사용되지 않습니다. 대부분의 경우 실제 테스트 객체를 사용하거나 완전한 모킹으로 요구 사항을 충족하기 때문입니다.

### Mockito를 최종 클래스 및 메서드에 사용할 수 있나요?

Mockito 2.1.0부터 최종 클래스 및 메서드를 모킹하도록 Mockito를 구성할 수 있습니다. mockito-core 대신 mockito-inline 의존성을 추가해야 합니다. mockito-inline은 최종 클래스 및 메서드를 처리하기 위한 동적 서브클래싱 메커니즘을 활성화합니다. 이 기능은 Mockito의 적용성을 크게 향상시켜 거의 모든 Java 클래스를 모킹 가능하게 합니다.

### 모ック 객체 메서드가 null을 반환하는 이유는 무엇인가요?

기본적으로 모ック 객체의 모든 메서드는 해당 타입의 기본값인 null 또는 기본값을 반환합니다(int는 0을 반환, boolean은 false를 반환, 컬렉션 타입은 빈 컬렉션 반환 등). 이것이 Mockito의 기본 Answer 동작입니다. 특정 반환값이 필요한 경우 when().thenReturn()을 사용하여 명시적으로 설정해야 합니다.

## 사용 질문

### @Mock 어노테이션이 작동하지 않습니다. 어떻게 해야 하나요?

@Mock 어노테이션이 작동하려면 Mockito 프레임워크를 먼저 초기화해야 합니다. 테스트에서 MockitoAnnotations.initMocks(this), @RunWith(MockitoJUnitRunner.class) 또는 @ExtendWith(MockitoExtension.class)(JUnit 5용) 중 하나를 사용해야 합니다. 올바르게 초기화되면 @Mock이 주석된 필드는 테스트 메서드가 실행되기 전에 자동으로 모ック 객체를 생성합니다.

### void 메서드를 모킹하려면 어떻게 하나요?

반환 타입이 void인 메서드의 경우 동작을 정의하기 위해 doThrow(), doAnswer() 또는 doNothing()을 사용하세요.

```java
doThrow(new RuntimeException("Error")).when(mockedObject).voidMethod();
doAnswer(invocation -> null).when(mockedObject).voidMethod();
doNothing().when(mockedObject).voidMethod();
```

### 메서드 호출 인자를 검증하려면 어떻게 하나요?

검증에 인자 매칭자를 사용하거나 ArgumentCaptor를 사용하여 인자를 캡처하여 자세한 어설션을 수행할 수 있습니다.

```java
verify(mockedList).add(anyString());

ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
verify(mockedList).add(captor.capture());
assertTrue(captor.getValue().length() > 0);
```

### 반복자를 모킹하려면 어떻게 하나요?

반복자를 모킹할 때는 hasNext()와 next() 메서드 간의 조정에 주의해야 합니다.

```java
Iterator<String> mockIterator = mock(Iterator.class);
when(mockIterator.hasNext()).thenReturn(true, true, false);
when(mockIterator.next()).thenReturn("first", "second");

assertTrue(mockIterator.hasNext());
assertEquals("first", mockIterator.next());
assertTrue(mockIterator.hasNext());
assertEquals("second", mockIterator.next());
assertFalse(mockIterator.hasNext());
```

### 메서드 오버로딩을 처리하려면 어떻게 하나요?

클래스에 오버로드된 메서드가 있을 때 Mockito는 매개변수 타입과 수를 기반으로 다른 메서드를 구분합니다. when()의 매개변수 타입이 모킹하려는 메서드 시그니처와 정확히 일치하는지 확인하세요.

```java
when(mockedList.get(0)).thenReturn("string");      // get(int index)
when(mockedList.get(anyInt())).thenReturn("any"); // 오버로드된 메서드
```

## 고급 질문

### when().thenReturn()이 작동하지 않는 이유는 무엇인가요?

일반적인 원인: 동일한 메서드에 대한 동작을 여러 번 설정하는 경우(마지막에만 적용됨); Spy를 사용하고 실제 메서드를 호출하는 경우; 메서드가 void를 반환하는 경우(doReturn().when()을 사용해야 함); 인자 매칭자를 잘못 사용하여 매칭 실패가 발생하는 경우. 동작 설정 구문이 올바른지 확인하세요.

### 추상 클래스를 테스트하려면 어떻게 하나요?

추상 클래스를 테스트할 때 Mockito를 사용하여 추상 클래스의 구체 메서드를 모킹하거나 일부 실제 행동을 유지하기 위해 Spy를 사용할 수 있습니다.另一种方法是创建具体测试子类来运行测试.

```java
@Test
public void testAbstractClass() {
    AbstractClass mock = mock(AbstractClass.class, Mockito.CALLS_REAL_METHODS);
    when(mock.concreteMethod()).thenReturn("mocked");
}
```

### 제네릭 메서드를 모킹하려면 어떻게 하나요?

제네릭 메서드를 모킹할 때 타입 소거로 인한 제한을 인식하세요. any() 또는 anyObject()를 사용하여 제네릭 매개변수를 매칭하세요.

```java
when(mockedObject.genericMethod(any())).thenReturn("result");
```

### 호출 순서를 검증하려면 어떻게 하나요?

InOrder를 사용하여 메서드 호출이 특정 순서로 실행되었는지 확인하세요.

```java
InOrder inOrder = inOrder(mockedList);
inOrder.verify(mockedList).add("first");
inOrder.verify(mockedList).add("second");
```

### 객체를 부분적으로 모킹하려면 어떻게 하나요?

부분적 모킹에는 Spy를 사용하거나 withSettings()를 사용하여 Mock 동작을 사용자 정의하세요.

```java
MyClass spy = spy(new MyClass());
doReturn("mocked").when(spy).method();
```

## 성능 질문

### 테스트가 느리게 실행됩니다. 어떻게 해야 하나요?

불필요한 실제 메서드 호출을 사용하지 않도록 확인하세요. 모든 테스트에서 새 Mock 객체를 생성하고 있는지 확인하세요. Mockito를 많이 사용하는 프로젝트의 경우 로그 출력을 줄이기 위해 @RunWith(MockitoJUnitRunner.Silent.class) 사용을 고려하세요.

### Mock 객체의 메모리 사용량을 줄으려면 어떻게 하나요?

Mockito의 모ック 객체 자체는 많은 메모리를 소비하지 않습니다. 그러나 많은 수의 모ック 객체를 생성하면 여전히 성능에 영향을 미치는 경우 @Mock 어노테이션과 @Scope를 함께 사용하여 모ック 객체를 재사용하거나 정적 모ック 팩토리 메서드를 사용하는 것을 고려하세요.

## 디버깅 질문

### 모크 객체 호출 로그를 보려면 어떻게 하나요?

Mockito의 로깅 기능을 사용하거나 리스너를 추가하여 호출을 추적할 수 있습니다.

```java
Mockito.framework().addListener(new VerificationListener());
```

또는 Maven/Gradle 테스트 출력을 사용하여 자세한 모ック 호출 정보를 볼 수 있습니다.

### 테스트가 실패할 때 디버깅하려면 어떻게 하나요?

Mockito는 상세한 오류 메시지를 제공합니다. 검증 실패에 표시된 예상 호출과 실제 호출의 차이를 확인하세요. Print Mockito Logs 기능을 사용하여 모크 객체 동작을 이해하세요. 문제를 파악하기 위해 테스트 케이스를 단계별로 단순화하세요.
