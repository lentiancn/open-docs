# Введение

## Что такое Mockito

Mockito — это мощный фреймворк для мокирования при юнит-тестировании на Java, разработанный для тестирования, управляемого тестами (TDD), и разработки, управляемой поведением (BDD). Он позволяет разработчикам создавать макетные объекты, которые имитируют поведение реальных объектов контролируемым образом, обеспечивая изолированное и независимое юнит-тестирование.

## Почему выбрать Mockito

### Чистый и простой API

Mockito предоставляет интуитивно понятный и простой в использовании API, что делает обучение и применение straightforward. Фреймворк позволяет разработчикам писать красивые тесты с чистым, читаемым кодом. Синтаксис ощущается естественным, что делает тестовый код таким же легким для чтения, как и продакшн-код.

### Мощные функции

Mockito поддерживает широкий спектр продвинутых функций, включая проверку вызовов методов, заглушки возвращаемых значений, генерацию исключений, сопоставление аргументов и проверку порядка вызовов. Эти возможности охватывают различные сложные тестовые сценарии и помогают разработчикам достичь всестороннего покрытия кода.

### Отличная читаемость

Тесты, написанные с Mockito, имеют отличную читаемость. Когда тест падает, Mockito produces clear and informative verification error messages that help developers quickly identify the problem. Эта поддержка отладки значительно повышает эффективность разработки.

### Активная поддержка сообщества

Mockito — самый популярный фреймворк для мокирования в экосистеме Java с большой базой пользователей и активным сообществом. Фреймворк continuously поддерживается и обновляется с полной документацией. Когда возникают проблемы, решения легко найти. На Stack Overflow Mockito получил наибольшее количество голосов, что делает его признанным лучшим фреймворком для мокирования на Java.

## Основные концепции

### Макетные объекты

Макетный объект служит заменой реального объекта во время тестирования. Создавая макетные объекты, вы можете изолировать внешние зависимости, ensuring that tests focus solely on the unit under test without being affected by external systems.

### Заглушки

Заглушки определяют, как макетный объект должен вести себя при вызове конкретных методов. Используя синтаксис when().thenReturn(), вы можете точно контролировать ответы макетных методов, позволяя тестам охватывать различные сценарии.

### Проверка

Проверка подтверждает, что взаимодействия между тестируемым кодом и макетными объектами происходили ожидаемым образом. Метод verify() проверяет, были ли вызваны методы, сколько раз они были вызваны и какие аргументы были переданы, обеспечивая правильное использование кода его зависимостей.

### Шпион

Шпион позволяет выполнять частичное мокирование реальных объектов. В отличие от полностью замокированных объектов, шпионы сохраняют大部分 поведения реального объекта, позволяя при этом заглушать конкретные методы. Это особенно полезно, когда вам нужно протестировать большую часть реальной логики, замокировав лишь несколько методов.

## Типичные случаи использования

### Изоляция внешних зависимостей

Когда тестируемый код зависит от внешних систем, таких как базы данных, сетевые сервисы или файловые системы, Mockito может мокировать эти внешние зависимости. Это позволяет тестам выполняться без необходимости реальных внешних систем, что не только ускоряет выполнение тестов, но и устраняет сбои тестов из-за нестабильных внешних систем.

### Мокирование незавершённых модулей

В больших проектах разные модули могут находиться на разных этапах разработки. Mockito может мокировать модули, которые ещё не завершены, позволяя уже готовые модули тестировать заранее и предотвращая узкие места в разработке.

### Мокирование сложных объектов

Некоторые бизнес-объекты сложно построить, поскольку они содержат несколько уровней зависимостей. Mockito упрощает создание замокированных версий этих сложных объектов, фокусируясь на тестировании самой бизнес-логики.

## Интеграция с JUnit

Mockito обычно используется совместно с тестовым фреймворком JUnit. Используя @RunWith(MockitoJUnitRunner.class) или @ExtendWith(MockitoExtension.class) из JUnit, поля с аннотацией @Mock автоматически инициализируются, что упрощает тестовый код. Эта комбинация представляет лучшую практику юнит-тестирования на Java.

## Совместимость версий

Mockito 5.x требует JDK 11 или выше и поддерживает функции Java 8, такие как лямбда-выражения. Серия Mockito 4.x по-прежнему поддерживает JDK 8. Разработчики могут выбрать подходящую версию в зависимости от требований проекта.

## Резюме

Mockito — незаменимый инструмент тестирования для Java-разработчиков. Он элегантно решает проблему изоляции зависимостей в юнит-тестировании, позволяя разработчикам писать надежный и поддерживаемый тестовый код. Освоение Mockito значительно повышает качество кода и эффективность разработки, что делает его обязательным навыком для каждого Java-инженера.
