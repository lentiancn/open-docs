# JDK インストールガイド

## WindowsへのJDKのインストール

### システム要件

JDKをインストールする前に、システムが以下の最小要件を満たしていることを確認してください：プロセッサはIntel Core i3以上を推奨します。メモリは少なくとも4GB RAM、推奨は8GB以上です。ディスク容量はJDKインストール用に少なくとも2GBの空き容量が必要です。オペレーティングシステムはWindows 10またはWindows 11の64ビットバージョンです。

Windows 7と8.1は動作する可能性がありますが、Oracleはこれらのオペレーティングシステムを公式にサポートしなくなりました。古いシステムで実行する場合、互換性问题が発生する可能性があります。

### JDKのダウンロード

まず、Oracleの公式ウェブサイトhttps://www.oracle.com/java/technologies/downloads/にアクセスしてJDKをダウンロードします。ダウンロードページでは、異なるJDKバージョンを見つけることができます：JDK 25は最新バージョン、JDK 21は最新の長期サポートバージョン、JDK 17も成熟したLTSバージョンです。プロジェクトの要件に基づいて適切なバージョンを選択してください。

Windowsシステムの場合、x64 Installer（.exe）またはx64 MSI Installer（.msi）形式を選択してください。.exeは従来のインストールウィザードであり、.msiは企業環境での一括展開に適しています。Javaプログラムを実行するだけでインストールが不要な場合は、x64 Compressed Archive（.zip）を選択し、解凍して使用できます。

ダウンロード時にはライセンス条項に注意してください。Oracle JDKは特定のバージョンと使用状況で無料ですが、商用使用と再配布には制限があります。完全に無料のを使用する必要がある場合は、Eclipse TemurinなどのOpenJDKまたはOpenJDKベースの他のディストリビューションの使用を検討してください。

### インストール手順

ダウンロードしたJDKインストールファイル（通常はjdk-xx_windows-x64_bin.exe）をダブルクリックして実行します。インストールウィザードで、「次へ」をクリックして続行します。最初のターゲットフォルダ選択画面には、C:\Program Files\Java\jdk-xxがデフォルトのインストールパスとして表示されます。インストール場所を変更する必要がある場合は、「変更」をクリックして別のディレクトリを選択してください。デフォルトパスを使用することをお勧めします。

次に機能選択画面が表示され、デフォルトでは開発ツール、ソースコード、公衆JREを含むすべての機能が選択されています。デフォルトのオプションを完全インストールとして保持することをお勧めします。「次へ」をクリックしてインストールを開始します。インストールプロセスには数分かかる場合があります。

インストールが完了すると、プロンプトが表示されます。「閉じる」をクリックしてインストールを完了します。この時点でJDKは指定されたディレクトリにインストールされていますが、環境変数を設定しないとコマンドラインでjavaやjavacなどのコマンドを使用できません。

### 環境変数の設定

環境変数を設定する目的は、システムがJDKのインストール場所を 찾아、的任何ディレクトリからjava、javacなどのコマンドを使用できるようにすることです。「このPC」または「マイコンピュータ」を右クリックし、「プロパティ」を選択します。左側のメニューで「システムの詳細設定」をクリックします。

開いたシステムプロパティウィンドウで、「環境変数」ボタンをクリックします。「システム環境変数」領域で、「新規」をクリックします。変数名にJAVA_HOMEを入力し、変数値にJDKのインストールパス（例：C:\Program Files\Java\jdk-21）を入力してOKをクリックします。

システム環境変数でPath変数を見つけ、「編集」をクリックします。「新規」をクリックし、%JAVA_HOME%\binを入力します。この行が最も上にあることを確認し、新しくインストールしたJDKが優先されるようにします。OKをクリックしてすべての設定を保存します。

新しいコマンドプロンプトウィンドウ（cmdまたはPowerShell）を開いて、java -versionとjavac -versionを実行し、バージョン情報が表示されれば設定成功です。

### インストールの確認

コマンドプロンプト（Win+R、cmdを入力）を開き、次のコマンドを実行してJDKが正しくインストールされていることを確認します：

java -versionを実行すると、Java実行環境のバージョン情報が表示されます。javac -versionを実行すると、Javaコンパイラのバージョン情報が表示されます。echo %JAVA_HOME%を実行すると、前に設定したJAVA_HOMEパスが表示されます。

いずれかのコマンドがエラーを表示した場合は、環境変数が正しく設定されているか、JDKが正しくインストールされているかを確認してください。

## macOSへのJDKのインストール

### DMGインストールパッケージの使用

macOSユーザーはDMGインストールパッケージを使用してJDKを素早くインストールできます。Oracle公式ダウンロードページにアクセスして、macOS용JDKをダウンロードします。Apple Silicon（M1/M2/M3チップ）の場合はARM64 DMG Installerを、IntelチップのMacの場合はx64 DMG Installerをダウンロードします。

ダウンロード完成后、DMGファイルを開くとpkgインストールファイルが表示されます。ダブルクリックしてインストールウィザードを実行し、指示に従ってインストールを完了します。インストーラーはJDKを/Library/Java/JavaVirtualMachines/ディレクトリにインストールします。

インストールが完了したら、ターミナルを開いてjava -versionと入力してインストールを確認します。システムに複数のJDKバージョンがある場合は、/usr/libexec/java_home -Vを使用してインストールされたすべてのバージョンを表示できます。

### Homebrewを使用したインストール

Homebrewパッケージマネージャーがすでにインストールされている場合は、それを使用してJDKをインストールできます会更加便利です。ターミナルを開いてを実行：brew update。然后安装JDK 21：brew install openjdk@21。

インストールが完了したら、JDKをPATHに追加する必要があります。以下のコマンドを実行してJDKをシステムにリンクします：sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21。

次に~/.zshrcまたは~/.bash_profileに环境変数を追加します：export PATH="/usr/local/opt/openjdk@21/bin:$PATH"。保存してsource ~/.zshrcを実行して設定を適用します。

### JAVA_HOMEの設定

macOSでは、JAVA_HOME环境変数の设定は必須ではありません。システムは自動的にインストールされたJDKを見つけます。ただし、一部のツール（MavenやGradleなど）では手動设定が必要な場合があります。ターミナルで~/.zshrcファイルを編集し、次を追加します：export JAVA_HOME=$(/usr/libexec/java_home)。保存してsource ~/.zshrcを実行して設定を適用します。

## LinuxへのJDKのインストール

### APTを使用したインストール（Debian/Ubuntu）

DebianとUbuntuシステムでは、APTパッケージマネージャーを使用してOpenJDKをインストールできます。まずソフトウェアパッケージリストを更新します：sudo apt update。次にOpenJDK 21開発版をインストールします：sudo apt install openjdk-21-jdk。Javaプログラムを実行するだけでコンパイラが不要な場合は、JREをインストールします：sudo apt install openjdk-21-jre。

インストール中に、JAVA_HOMEディレクトリを作成するかどうかを尋ねられます。「はい」を選択して自動設定します。インストールが完了したら、java -versionとjavac -versionを使用してインストールを確認します。

手動でJAVA_HOMEを設定する必要がある場合は、~/.bashrcまたは~/.profileファイルを編集し、次を追加します：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64。然后実行source ~/.bashrcして設定を適用します。

### YUMを使用したインストール（Red Hat/CentOS）

Red Hat、CentOS、FedoraなどのYUMを使用するシステムでは、JDKを直接インストールできます。以下のコマンドを実行してOpenJDK 21をインストールします：sudo yum install java-21-openjdk java-21-openjdk-devel。-develパッケージにはコンパイラ和其他开发工具が含まれています。

インストールが完了したら、alternativesコマンドを使用してデフォルトのJavaバージョンを設定します：sudo alternatives --config java。指示に従ってデフォルトのJavaバージョンとして選択する版本を設定します。

~/.bashrcファイルを編集し、次を追加します：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk。実行source ~/.bashrcして設定を適用します。

### バイナリファイルの手動インストール

特定のJDKバージョンをインストールする必要がある場合は、バイナリアーカイブファイルを手動でダウンロードして展開できます。OracleまたはOpenJDKダウンロードページにアクセスして、適切なバージョンの.tar.gzファイルをダウンロードします。

JDKインストールディレクトリを作成します：sudo mkdir -p /opt/jdk。ダウンロードしたファイルを展開します：sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk。展開したディレクトリに入って、完全なパスをメモします。

~/.bashrcファイルを編集し、次を追加します：export JAVA_HOME=/opt/jdk/jdk-21とexport PATH=$JAVA_HOME/bin:$PATH。実行source ~/.bashrcして設定を適用します。

## 複数のJDKバージョンの管理

### Windows複数バージョン管理

Windowsに複数のJDKバージョンをインストールした後、環境変数を介してデフォルトバージョンを切り替える必要があります。最も簡単な方法は、PATH環境変数でJDK binディレクトリの順序を変更し、使用するバージョンを最初に配置することです。

別の方法は、JAVA_HOME環境変数を使用して、異なるJDKインストールディレクトリを指すようにすることです。この方法はほとんどのツールに效しますが、一部のツールはJAVA_HOMEではなくPATH内のjavaコマンドを直接使用する場合があります。

複数のJDKバージョンの管理を支援する第三方ツールもあります，例如SDKMAN!のWindowsバージョンやJetBrainsのToolbox。これらのツールを使用すると、バージョン間の切り替えがより便利になります。

### macOS複数バージョン管理

macOSでは、jenvツールを使用して複数のJDKバージョンを管理できます。まずjenvをインストールします：brew install jenv。然后shell設定ファイルにjenvの初期化を добавьте。

インストールされたJDKをjenvに追加します：jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home。然后jenvを使用してすべての管理されたバージョンを表示できますversionsを使用してグローバルデフォルトバージョンを設定します。

別の方法は、/usr/libexec/java_home -vを使用して特定のバージョンを指定してプログラムを実行することです，例如：/usr/libexec/java_home -v 21 --exec java -version。

### Linux複数バージョン管理

Linuxでは、update-alternativesツールを使用して複数のJDKバージョンを管理できます。まず各JDKバージョンに対してalternativesエントリを作成します：sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1。

次にupdate-alternatives --config javaを使用してデフォルトバージョンを選択します。jenvはLinuxでも利用可能で、安装と设定方法はmacOSと同様です。

## 一般的なインストール問題

### インストール後javaコマンドが見つからない

JDKをインストールした後javaコマンドが見つからない場合は、まずJDKが正しくインストールされているか确认してください。JAVA_HOME環境変数が正しく設定されているか、PATHに%JAVA_HOME%\bin（Windows）または$JAVA_HOME/bin（Linux/macOS）が含まれているかを確認してください。

Windowsでは、新しいコマンドプロンプトウィンドウを開いてください。環境変数の変更はすぐに有効にならない場合があるためです。Linux/macOSでは、source ~/.bashrcを実行するか、ターミナルを再起動してください。

問題が解決しない場合は、JDKインストールディレクトリが正しいか、そのディレクトリにbin/javaファイルが含まれているかを確認してください。

### 権限の問題

LinuxとmacOSにJDKをインストールする場合、権限の問題が発生することがあります。パッケージマネージャーを使用してインストールする場合、通常は管理者権限（sudo）が必要です。手動でインストールする場合、インストールディレクトリに適切な読み取り/書き込み権限があることを確認してください。

 일반 사용자가 java 명령을 사용할 수 없는 경우 JAVA_HOME 디렉토리와 JDK bin 디렉토리의 권한을 확인하세요. ls -la /usr/lib/jvm을 사용하여 디렉토리 권한을 확인할 수 있습니다.

### JAVA_HOME 설정이 작동しない

JAVA_HOME을 설정한 후 작동하지 않으면 환경 변수 구문이 정확한지 확인하세요. Windows에서는 변수 값에 따옴표가 없어야 합니다. Linux/macOS에서는 올바른 export 명령을 사용하고 있는지 확인하세요.

다른 스크립트나 구성 파일이 JAVA_HOME 설정을 재정의하고 있는지 확인하세요. Linux에서 시스템 수준 구성은 /etc/environment 또는 /etc/profile.d/에 있을 수 있습니다.

### 이전 버전 제거

JDK를 업그레이드한 후 이전 버전을 제거하여 디스크 공간을 절약해야 할 수 있습니다. Windows에서는 제어판의 프로그램 추가/제거에서 이전 JDK 버전을 찾아 제거를 클릭합니다. macOS에서는 /Library/Java/JavaVirtualMachines/ 디렉토리에서 이전 버전 폴더를 삭제합니다. Linux에서는 패키지 관리자를 사용하여 제거합니다: sudo apt remove openjdk-xx-jdk (APT) 또는 sudo yum remove java-xx-openjdk (YUM).
