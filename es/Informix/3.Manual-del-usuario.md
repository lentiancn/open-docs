# Manual de Usuario de IBM Informix

## Comandos Básicos

### Gestión del Servidor

| Comando | Descripción |
|---------|-------------|
| `oninit` | Inicializar y iniciar el servidor Informix |
| `onmode` | Cambiar el modo del servidor (quiescente, online, etc.) |
| `onstat` | Mostrar estado y estadísticas del servidor |
| `ontape` | Utilidad de respaldo y restauración |
| `onbar` | Respaldo y restauración usando IBM Storage Manager |
| `onspaces` | Gestionar espacios de almacenamiento |
| `onparams` | Modificar parámetros del servidor |

### Comandos Comunes del Servidor

```bash
# Inicializar e iniciar el servidor
oninit

# Iniciar en modo producción completo
onmode -vy

# Apagar el servidor
onstat -

# Ver espacio en disco
onstat -d

# Ver logs lógicos
onstat -l

# Ver usuarios activos
onstat -u

# Ver bloqueos
onstat -k

# Ver transacciones
onstat -x
```

## Conexión a la Base de Datos

### Usando dbaccess

```bash
# Modo interactivo
dbaccess

# Ejecutar script SQL
dbaccess nombre_base_datos < script_sql.sql

# Conectar con usuario específico
dbaccess -user miusuario midb
```

### Parámetros de Conexión

```
Host: localhost (o IP del servidor)
Puerto: 9088 (predeterminado)
Base de Datos: su_base_de_datos
Nombre del Servidor: su_servidor_informix
```

## Sentencias SQL

### Crear una Base de Datos

```sql
-- Crear una base de datos simple
CREATE DATABASE midb;

-- Crear con registro buffering
CREATE DATABASE midb WITH BUFFERED LOG;

-- Crear con registro
CREATE DATABASE midb WITH LOG;

-- Crear base de datos compatible ANSI
CREATE DATABASE midb WITH LOG MODE ANSI;
```

### Crear Tablas

```sql
-- Creación básica de tabla
CREATE TABLE clientes (
    cliente_id SERIAL PRIMARY KEY,
    nombre CHAR(50),
    correo CHAR(100),
    fecha_creacion DATE
);

-- Tabla con restricciones
CREATE TABLE pedidos (
    pedido_id SERIAL PRIMARY KEY,
    cliente_id INTEGER REFERENCES clientes(cliente_id),
    fecha_pedido DATE,
    monto_total DECIMAL(10,2)
);

-- Tabla temporal
CREATE TEMP TABLE datos_sesion (
    id INTEGER,
    datos VARCHAR(255)
);
```

### Insertar Datos

```sql
-- Insertar una fila
INSERT INTO clientes (nombre, correo, fecha_creacion)
VALUES ('Juan Pérez', 'juan@ejemplo.com', TODAY);

-- Insertar múltiples filas
INSERT INTO clientes (nombre, correo, fecha_creacion)
VALUES 
    ('Ana García', 'ana@ejemplo.com', TODAY),
    ('Pedro López', 'pedro@ejemplo.com TODAY);
```

### Consultar Datos

```sql
-- Seleccionar todas las columnas
SELECT * FROM clientes;

-- Seleccionar columnas específicas
SELECT nombre, correo FROM clientes;

-- Cláusula WHERE
SELECT * FROM clientes WHERE cliente_id > 100;

-- Ordenar resultados
SELECT nombre, correo FROM clientes ORDER BY nombre;

-- Agrupar con funciones agregadas
SELECT cliente_id, COUNT(*) as numero_pedidos
FROM pedidos GROUP BY cliente_id;

-- Unir tablas
SELECT p.pedido_id, c.nombre, p.fecha_pedido
FROM pedidos p, clientes c
WHERE p.cliente_id = c.cliente_id;
```

### Actualizar Datos

```sql
-- Actualizar una fila
UPDATE clientes 
SET correo = 'nuevo@ejemplo.com' 
WHERE cliente_id = 101;
```

### Eliminar Datos

```sql
-- Eliminar una fila
DELETE FROM clientes WHERE cliente_id = 101;

-- Eliminar basado en condición
DELETE FROM pedidos WHERE fecha_pedido < '2024-01-01';
```

## Archivos de Configuración

### Archivos de Configuración Clave

| Archivo | Propósito |
|---------|-----------|
| `$INFORMIXDIR/etc/onconfig` | Configuración principal del servidor |
| `$INFORMIXDIR/etc/sqlhosts` | Configuración de red |
| `$INFORMIXDIR/etc/$ONCONFIG` | Configuración de root dbspace |

### Parámetros de Ejemplo de onconfig

```
ROOTNAME        rootdbs
ROOTPATH        /opt/IBM/informix/online.root
ROOTOFFSET      0
ROOTSIZE        200000
DBSERVERNAME    ol_informix
```

## Respaldo y Restauración

### Usando ontape

```bash
# Respaldo completo
ontape -s -L 0

# Respaldo nivel 1
ontape -s -L 1

# Restaurar desde respaldo
ontape -r

# Restaurar a un momento específico
ontape -r -t 2025-03-17
```

### Usando onbar

```bash
# Respaldo completo
onbar -b -L 0

# Restaurar
onbar -r
```

## Ajuste de Rendimiento

### Parámetros Clave de Memoria

- `SHMVIRTSIZE` - Tamaño de memoria compartida virtual
- `SHMADD` - Segmentos adicionales de memoria compartida
- `BUFFERPOOL` - Configuración de caché de datos

### Índices

```sql
-- Crear índice
CREATE INDEX idx_clientes_correo ON clientes(correo);

-- Crear índice único
CREATE INDEX idx_pedidos_id ON pedidos(pedido_id);
```

### Estadísticas

```sql
-- Actualizar estadísticas de tabla
UPDATE STATISTICS FOR TABLE clientes;

-- Actualizar estadísticas con alto detalle
UPDATE STATISTICS HIGH FOR TABLE clientes;
```
