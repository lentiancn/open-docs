# Vite Frequently Asked Questions

## General Questions

### What is Vite?
Vite is a next-generation frontend build tool created by Evan You. It provides extremely fast HMR (Hot Module Replacement) and optimized production builds using native ES modules.

### Is Vite only for Vue?
No, Vite is framework-agnostic and works with Vue, React, Svelte, Preact, and vanilla JavaScript projects.

### Do I need to use a specific package manager?
Vite works with npm, yarn, and pnpm. Use whatever you prefer.

## Installation Questions

### How do I create a new Vite project?
```bash
# Using npm
npm create vite@latest my-project -- --template vue

# Using yarn
yarn create vite my-project --template vue

# Using pnpm
pnpm create vite my-project --template vue
```

### How do I install Vite in an existing project?
```bash
npm install -D vite
```

### What are the dependencies needed?
For Vue projects:
```bash
npm install vue
npm install -D @vitejs/plugin-vue
```

For React projects:
```bash
npm install react react-dom
npm install -D @vitejs/plugin-react
```

## Usage Questions

### How do I start the development server?
```bash
npm run dev
```

### How do I build for production?
```bash
npm run build
```

### How do I preview the production build?
```bash
npm run preview
```

### How do I configure Vite?
Create `vite.config.js`:

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8000'
    }
  },
  build: {
    outDir: 'dist',
    sourcemap: true
  }
})
```

## Troubleshooting

### Module not found errors
- Check if all dependencies are installed: `npm install`
- Verify import paths are correct
- Ensure package.json has correct type: "module"

### CSS not being applied
- Import CSS in your main JavaScript file
- Check if PostCSS plugins are needed
- Verify CSS file path is correct

### Port already in use
Change port in vite.config.js:
```javascript
server: {
  port: 3001
}
```

### Slow initial load
- Check network tab for large dependencies
- Optimize images and assets
- Enable lazy loading for routes

## Production Questions

### How do I optimize bundle size?
- Use dynamic imports for code splitting
- Configure build options:
```javascript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        vendor: ['react', 'react-dom'],
        utils: ['lodash']
      }
    }
  }
}
```

### How do I deploy to GitHub Pages?
```javascript
export default defineConfig({
  base: '/your-repo-name/'
})
```

Then run `npm run build` and deploy the `dist` folder.

### How do I set environment variables?
Create `.env` files:
```
VITE_API_URL=http://localhost:8000
```

Access in code:
```javascript
const apiUrl = import.meta.env.VITE_API_URL
```

## Advanced Questions

### How does Vite's HMR work?
Vite uses native ES modules and establishes a WebSocket connection for HMR. When a file changes, Vite only updates that specific module without reloading the page.

### Can I use Webpack loaders with Vite?
Use `vite-plugin-webpack` or convert loaders to Vite plugins. Some loaders have Vite equivalents.

### How do I add custom plugins?
```javascript
import { defineConfig } from 'vite'

function myPlugin() {
  return {
    name: 'my-plugin',
    transform(code, id) {
      // Transform code
      return code
    }
  }
}

export default defineConfig({
  plugins: [myPlugin()]
})
```

## Best Practices

4. Enable production source maps carefully
5. Use lazy loading for routes
6. Optimize images before building
