# 소개

Apache APISIX는 Apache Software Foundation의 상위 프로젝트로, API7.ai에서 개발하고 기부했습니다. 동적, 실시간, 고성능 특성을 갖춘 클라우드 네이티브 API 게이트웨이입니다.

APISIX는 모든 비즈니스 트래픽의 진입점 역할을 하며, 동적 라우팅, 동적 업스트림, 동적 인증서, A/B 테스트, 배포(카나리 배포), 블루-그린 배포, 속도 제한, 공격 방어, 메트릭 수집, 모니터링 알림, 관측 가능성, 서비스 거버넌스 등 다양한 기능을 제공합니다.

## 주요 기능

### 고성능

Nginx 기반으로 구축되어 APISIX는 Nginx의 우수한 성능을합니다. 플러그인 메커니즘을 통해 비즈니스 로직이 핵심 기능과 분리되어 있어 확장성이 더 유연하면서도 효율적인 처리 능력을 유지합니다.

### 완전한 동적 제어

 traditionnel Nginx와 달리 설정 reload가 필요한 반면, APISIX는 완전한 동적 설정 관리를 지원합니다. 모든 라우트, 업스트림, 플러그인은 서비스 재시작 없이 Admin API를 통해 실시간으로 업데이트할 수 있습니다.

### 클라우드 네이티브 설계

APISIX는 처음부터 클라우드 네이티브 시나리오를 고려하여 설계되었으며, Docker, Kubernetes, Helm Chart를 포함한 다양한 배포 방법을 지원합니다. Kubernetes 인그레스 게이트웨이로 직접 사용할 수 있는 Ingress Controller도 제공합니다.

### 플러그인 생태계

APISIX는 신원 인증, 속도 제한, 요청 재작성, 응답 처리, 모니터링, 로깅을 포함하는 풍부한 플러그인 시스템을 제공합니다. 사용자는 플러그인 메커니즘을 통해 게이트웨이 기능을 쉽게 확장할 수 있습니다.

### 다중 프로토콜 지원

전통적인 HTTP/HTTPS 프로토콜 외에도, APISIX는 gRPC, Dubbo, WebSocket, MQTT 및 기타 프로토콜을 지원하여 마이크로서비스 아키텍처의 다양한 시나리오 요구를 충족합니다.

## 아키텍처 개요

APISIX의 아키텍처는 데이터 플레인과 컨트롤 플레인 두 부분으로 나뉩니다.

**데이터 플레인**은 실제 요청 전달을 처리합니다. 수정된 Nginx 기반의 APISIX 코어는 고성능 요청 처리 파이프라인을 구현합니다.

**컨트롤 플레인**은 RESTful API를 통해 모든 설정을 관리하며, etcd와 Consul을 포함한 다양한 설정 지속성 방법을 지원합니다. APISIX는 시각적 설정 관리를 위한 대시보드도 제공합니다.

## 사용 사례

- 마이크로서비스 API의 통합 진입점
- Kubernetes Ingress Controller
- API 트래픽 관리 및 모니터링
- 멀티테넌시 API 관리
- 서비스 메시 사이드카 프록시
- AI 애플리케이션 게이트웨이 (LLM 및 AI 워크로드)

Apache Software Foundation의 상위 프로젝트로서, APISIX는 활발한 커뮤니티와 지속적인 버전 업데이트를 가지고 있어 기업 애플리케이션에 대한 신뢰할 수 있는 선택입니다.
