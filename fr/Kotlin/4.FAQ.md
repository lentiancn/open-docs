# Foire aux questions sur Kotlin

## Questions fondamentales

### Qu'est-ce que Kotlin ?

Kotlin est un langage de programmation moderne à typage statique développé et maintenu par JetBrains. Le projet a commencé en 2010, et la première version stable 1.0 a été publiée en février 2016. Kotlin s'exécute sur JVM, Android, JavaScript, WebAssembly et les plateformes natives. C'est un langage open source publié sous licence Apache 2.0.

### Kotlin est-il gratuit ?

Oui, Kotlin est complètement gratuit. Il était gratuit, il est gratuit et le restera. Kotlin est développé sous licence Apache 2.0, et le code source est disponible sur GitHub.

### Kotlin est-il orienté objet ou fonctionnel ?

Kotlin supporte à la fois la programmation orientée objet et les styles de programmation fonctionnelle. Vous pouvez utiliser l'un ou l'autre selon vos besoins, ou combiner des éléments des deux. Avec un support de première classe pour des fonctionnalités comme les fonctions d'ordre supérieur, les types de fonctions et les lambdas, Kotlin est un excellent choix pour la programmation fonctionnelle.

### Quelles sont les différences entre Kotlin et Java ?

Par rapport à Java, Kotlin est plus concis. Des estimations approximatives indiquent environ 40% moins de lignes de code pour la même fonctionnalité. Kotlin est également plus sûr en termes de types, par exemple avec le support des types nullables qui réduisent les NullPointerExceptions. D'autres fonctionnalités comme les casts intelligents, les fonctions d'ordre supérieur, les fonctions d'extension et les lambdas permettent d'écrire du code expressif et facilitent la création de DSL.

### Kotlin est-il compatible avec Java ?

Oui, Kotlin est interopérable à 100% avec Java. Vous pouvez facilement appeler du code Kotlin depuis Java et du code Java depuis Kotlin, ce qui rend l'adoption beaucoup plus facile et moins risquée. Il existe également un convertisseur automatique Java vers Kotlin intégré à l'IDE qui simplifie la migration.

## Installation et configuration

### Comment installer Kotlin ?

Kotlin est inclus dans chaque version d'IntelliJ IDEA et Android Studio. Téléchargez et installez simplement l'un de ces IDEs pour commencer à utiliser Kotlin. Vous pouvez également télécharger le compilateur Kotlin en ligne de commande ou l'installer via des gestionnaires de packages comme SDKMAN !, Homebrew ou Snap.

### Quels IDEs supportent Kotlin ?

Kotlin a un support complet prêt à l'emploi dans IntelliJ IDEA et Android Studio avec un plugin Kotlin officiel développé par JetBrains. Les autres IDEs et éditeurs de code n'ont que des plugins Kotlin supportés par la communauté. Vous pouvez également essayer Kotlin Playground pour écrire, exécuter et partager du code Kotlin dans votre navigateur.

### Quels outils de construction supportent Kotlin ?

Sur la JVM, les principaux outils de construction sont Gradle et Maven. Il existe également des outils de construction pour JavaScript côté client.

### En quoi Kotlin compile-t-il ?

Lors du ciblage de la JVM, Kotlin produit du bytecode compatible Java. Lors du ciblage de JavaScript, Kotlin se transpile en ES5.1 et produit du code compatible avec les systèmes de modules y compris AMD et CommonJS. Lors du ciblage natif, Kotlin produit du code spécifique à la plateforme via LLVM.

### Quelles versions de JVM Kotlin cible-t-il ?

Kotlin vous permet de choisir la version JVM pour l'exécution. Par défaut, le compilateur Kotlin/JVM produit du bytecode compatible Java 8. Si vous voulez utiliser les optimisations disponibles dans les versions plus récentes de Java, vous pouvez spécifier explicitement la version Java cible de 9 à 25. Notez que le bytecode résultant peut ne pas fonctionner sur les versions inférieures. À partir de Kotlin 1.5, le compilateur ne supporte pas le bytecode compatible avec les versions Java inférieures à 8.

## Domaines d'application

### Pour quoi puis-je utiliser Kotlin ?

Kotlin peut être utilisé pour tout type de développement : côté serveur, web côté client, Android ou multiplateforme. Avec Kotlin/Native, le support d'autres plateformes comme les systèmes intégrés, macOS et iOS est disponible.

### Puis-je utiliser Kotlin pour le développement Android ?

Oui, Kotlin est supporté comme langage de première classe sur Android. Il y a des centaines d'applications utilisant Kotlin pour Android, comme Basecamp et Pinterest. Pour plus d'informations, consultez les ressources de Google sur le développement Android.

### Puis-je utiliser Kotlin pour le développement côté serveur ?

Oui, Kotlin est compatible à 100% avec la JVM, donc vous pouvez utiliser n'importe quel framework existant comme Spring Boot, Vert.x ou JSF. De plus, il existe des frameworks écrits spécifiquement en Kotlin, comme Ktor.

### Puis-je utiliser Kotlin pour le développement web ?

Oui, pour le développement web backend, Kotlin fonctionne bien avec des frameworks comme Ktor et Spring, permettant de construire efficacement des applications côté serveur. De plus, vous pouvez utiliser Kotlin/Wasm pour le développement web côté client.

### Puis-je utiliser Kotlin pour le développement bureau ?

Oui, vous pouvez utiliser n'importe quel framework UI Java comme JavaFX, Swing ou autres. De plus, il existe des frameworks spécifiques à Kotlin comme TornadoFX.

### Puis-je utiliser Kotlin pour le développement natif ?

Oui, Kotlin/Native est disponible dans Kotlin. Il compile Kotlin en code natif qui peut s'exécuter sans VM. Vous pouvez l'essayer sur les plateformes de bureau et mobiles populaires et même sur certains appareils IoT.

## Questions techniques

### Kotlin supporte-t-il le multithreading ?

Les coroutines Kotlin fournissent un support puissant pour la concurrence, permettant à plusieurs coroutines de s'exécuter dans un seul thread. Les coroutines sont beaucoup plus légères que les threads : vous pouvez créer des dizaines de milliers de coroutines sans épuiser les ressources du système. Kotlin fournit également Flow pour gérer les flux de données asynchrones.

### Kotlin a-t-il un ramasse-miettes ?

Lorsque Kotlin s'exécute sur la JVM, il est géré par le ramasse-miettes de la JVM. Kotlin/Native utilise différentes stratégies de gestion de mémoire.

### Quelles sont les performances de Kotlin ?

Kotlin compile en bytecode similaire à Java, avec des performances comparables. Les fonctionnalités et le sucre syntaxique de Kotlin n'introduisent généralement pas de surcharge de performance significative. Dans certains cas, les fonctionnalités de Kotlin comme les fonctions inline peuvent même offrir des avantages de performance.

### Comment déboguer le code Kotlin ?

Dans IntelliJ IDEA et Android Studio, vous pouvez utiliser les mêmes outils de débogage qu'avec Java. Les fonctionnalités de débogage standard comme définir des points d'arrêt, exécuter pas à pas et afficher les valeurs des variables sont toutes prises en charge.

### Kotlin supporte-t-il les génériques ?

Oui, le système de génériques de Kotlin est similaire à Java mais ajoute des fonctionnalités avancées comme les projections de types et la covariance au site de déclaration, rendant les génériques plus sûrs et flexibles.

### Kotlin a-t-il une bibliothèque standard ?

Oui, la bibliothèque standard Kotlin fournit des fonctionnalités étendues incluant les opérations de collections, la manipulation de chaînes, les opérations I/O, les fonctions mathématiques et plus encore. La bibliothèque standard est conçue en tenant compte de la cohérence de l'API et de la facilité d'utilisation.

## Questions de migration

### Comment convertir du code Java en Kotlin ?

IntelliJ IDEA et Android Studio ont un convertisseur Java vers Kotlin intégré. Sélectionnez le fichier Java à convertir et choisissez Code > Convertir le fichier Java en fichier Kotlin pour une conversion automatique.

### Puis-je utiliser Java et Kotlin dans le même projet ?

Oui, Kotlin et Java sont complètement compatibles et peuvent être mélangés dans le même projet. La stratégie de migration typique consiste à convertir les classes clés en Kotlin progressivement plutôt que de migrer tout le projet d'un coup.

### Combien de temps prend la migration vers Kotlin ?

Cela dépend de la taille du projet. Pour les petits projets, cela ne peut prendre que quelques heures. Pour les grands projets, une stratégie de migration progressive est généralement utilisée, qui peut être complétée en semaines à mois tout en maintenant les progrès normaux de développement du projet.

### Quelles sont les meilleures pratiques Kotlin ?

Suivre le Kotlin idiomatique est important. Utilisez les standards de codage officiels de Kotlin, préférez val à var, et utilisez pleinement les fonctionnalités du langage Kotlin comme les classes de données, les classes scellées et les fonctions d'extension. Kotlin fournit des directives de style de codage détaillées.

## Questions sur l'écosystème et les outils

### Quelles sont les bibliothèques Kotlin courantes ?

L'écosystème Kotlin est riche. Les bibliothèques courantes incluent : Kotlin Coroutines pour la programmation asynchrone, Koin pour l'injection de dépendances, Ktor pour les frameworks web, Retrofit pour les requêtes réseau, Room pour les bases de données et Exposed pour les frameworks SQL.

### Comment publier une bibliothèque Kotlin ?

Vous pouvez publier des bibliothèques sur Maven Central ou d'autres référentiels via Gradle ou Maven. Les bibliothèques Kotlin sont généralement publiées en bytecode JVM, mais vous pouvez également publier des versions Kotlin/Wasm ou Kotlin/Native.

### Quels frameworks de test Kotlin a-t-il ?

Kotlin peut utiliser n'importe quel framework de test Java comme JUnit ou TestNG. Kotlin fournit également la bibliothèque Kotlin Test avec des API de test plus idiomatiques. Pour les projets Kotlin Multiplatform, vous pouvez utiliser Kotlin Common Test.
