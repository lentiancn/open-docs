# Руководство пользователя IBM Informix

## Основные команды

### Управление сервером

| Команда | Описание |
|---------|----------|
| `oninit` | Инициализировать и запустить сервер Informix |
| `onmode` | Изменить режим сервера (quiescent, online и т.д.) |
| `onstat` | Показать статус и статистику сервера |
| `ontape` | Утилита резервного копирования и восстановления |
| `onbar` | Резервное копирование с IBM Storage Manager |
| `onspaces` | Управление хранилищами |
| `onparams` | Изменение параметров сервера |

### Типичные команды сервера

```bash
# Инициализация и запуск сервера
oninit

# Запуск в полном производственном режиме
onmode -vy

# Остановка сервера
onstat -

# Просмотр места на диске
onstat -d

# Просмотр логических журналов
onstat -l

# Просмотр активных пользователей
onstat -u

# Просмотр блокировок
onstat -k

# Просмотр транзакций
onstat -x
```

## Подключение к базе данных

### Использование dbaccess

```bash
# Интерактивный режим
dbaccess

# Выполнение SQL-скрипта
dbaccess имя_базы_данных < скрипт.sql

# Подключение с конкретным пользователем
dbaccess -user мой_пользователь моя_база
```

### Параметры подключения

```
Хост: localhost (или IP-сервера)
Порт: 9088 (по умолчанию)
База данных: ваша_база_данных
Имя сервера: ваш_informix_сервер
```

## SQL-операторы

### Создание базы данных

```sql
-- Создать простую базу данных
CREATE DATABASE моябаза;

-- Создать с буферизованным журналом
CREATE DATABASE моябаза WITH BUFFERED LOG;

-- Создать с журналированием
CREATE DATABASE моябаза WITH LOG;

-- Создать базу данных в режиме ANSI
CREATE DATABASE моябаза WITH LOG MODE ANSI;
```

### Создание таблиц

```sql
-- Базовое создание таблицы
CREATE TABLE клиенты (
    id_клиента SERIAL PRIMARY KEY,
    имя CHAR(50),
    email CHAR(100),
    дата_создания DATE
);

-- Таблица с ограничениями
CREATE TABLE заказы (
    id_заказа SERIAL PRIMARY KEY,
    id_клиента INTEGER REFERENCES клиенты(id_клиента),
    дата_заказа DATE,
    сумма DECIMAL(10,2)
);

-- Временная таблица
CREATE TEMP TABLE данные_сессии (
    id INTEGER,
    данные VARCHAR(255)
);
```

### Вставка данных

```sql
-- Вставить одну строку
INSERT INTO клиенты (имя, email, дата_создания)
VALUES ('Иван Иванов', 'ivan@пример.рф', TODAY);

-- Вставить несколько строк
INSERT INTO клиенты (имя, email, дата_создания)
VALUES 
    ('Пётр Петров', 'petr@пример.рф', TODAY),
    ('Мария Сидорова', 'maria@пример.рф', TODAY);
```

### Запрос данных

```sql
-- Выбрать все столбцы
SELECT * FROM клиенты;

-- Выбрать конкретные столбцы
SELECT имя, email FROM клиенты;

-- Условие WHERE
SELECT * FROM клиенты WHERE id_клиента > 100;

-- Сортировка
SELECT имя, email FROM клиенты ORDER BY имя;

-- Группировка с агрегатными функциями
SELECT id_клиента, COUNT(*) AS количество_заказов
FROM заказы GROUP BY id_клиента;

-- Объединение таблиц
SELECT з.id_заказа, к.имя, з.дата_заказа
FROM заказы з, клиенты к
WHERE з.id_клиента = к.id_клиента;
```

### Обновление данных

```sql
-- Обновить строку
UPDATE клиенты 
SET email = 'новый@пример.рф' 
WHERE id_клиента = 101;
```

### Удаление данных

```sql
-- Удалить строку
DELETE FROM клиенты WHERE id_клиента = 101;

-- Удалить по условию
DELETE FROM заказы WHERE дата_заказа < '2024-01-01';
```

## Файлы конфигурации

### Ключевые файлы конфигурации

| Файл | Назначение |
|------|-----------|
| `$INFORMIXDIR/etc/onconfig` | Основная конфигурация сервера |
| `$INFORMIXDIR/etc/sqlhosts` | Сетевая конфигурация |
| `$INFORMIXDIR/etc/$ONCONFIG` | Конфигурация корневого хранилища |

### Пример параметров onconfig

```
ROOTNAME        rootdbs
ROOTPATH        /opt/IBM/informix/online.root
ROOTOFFSET      0
ROOTSIZE        200000
DBSERVERNAME    ol_informix
```

## Резервное копирование и восстановление

### Использование ontape

```bash
# Полное резервное копирование
ontape -s -L 0

# Восстановление
ontape -r
```

### Использование onbar

```bash
# Резервное копирование
onbar -b -L 0

# Восстановление
onbar -r
```

## Оптимизация производительности

### Ключевые параметры памяти

- `SHMVIRTSIZE` - Размер виртуальной разделяемой памяти
- `SHMADD` - Дополнительные сегменты разделяемой памяти
- `BUFFERPOOL` - Конфигурация кэша данных

### Статистика

```sql
-- Обновить статистику таблицы
UPDATE STATISTICS FOR TABLE клиенты;

-- Обновить статистику с высокой детализацией
UPDATE STATISTICS HIGH FOR TABLE клиенты;
```
