# JUnit 소개

JUnit는 Java 플랫폼 가장 인기 있는 단위 테스트 프레임워크로, 반복 가능한 테스트를 작성하는 데 사용됩니다. 고품질 Java 코드 개발의 기반이며 전 세계 수백만 개발자가 널리 사용하고 있습니다.

## JUnit 4와 JUnit 5의 차이점

### 버전 개요

- **JUnit 4**: 2010년 출시, 오랫동안 Java 커뮤니티에서 사용된 테스트 프레임워크의 마지막 주요 버전
- **JUnit 5**: 2017년 출시, 세 개의 하위 프로젝트로 구성된 JUnit 프레임워크의 완전한 개편: JUnit Platform, JUnit Jupiter, JUnit Vintage

### 아키텍처 차이점

JUnit 4는 단일 라이브러리에 모든 기능이 통합된 모놀리식 아키텍처를 사용합니다. JUnit 5는 세 개의 개별 프로젝트로 구성된 모듈식 아키텍처를 사용합니다:

- **JUnit Platform**: 테스트 프레임워크 실행을 위한 기반, TestEngine API 및 콘솔 런처 제공
- **JUnit Jupiter**: 테스트 및 확장을 작성하기 위한 새로운 프로그래밍 모델 및 확장 모델
- **JUnit Vintage**: JUnit 5 플랫폼에서 JUnit 3 및 JUnit 4 테스트를 실행할 수 있는 호환 엔진

### 주요 차이점

| 기능 | JUnit 4 | JUnit 5 |
|------|---------|---------|
| 최소 Java 버전 | Java 5+ | Java 8+ |
| 애노테이션 패키지 | org.junit | org.junit.jupiter.api |
| 테스트 클래스 상속 불필요 | 아니오 | 예 |
| 단언 스타일 | Assert 정적 메서드 | Assertions 정적 메서드 |
| 매개변수화 테스트 | @Parameters | @ParameterizedTest |
| 중첩 테스트 | 미지원 | 지원 |
| 동적 테스트 | 미지원 | 지원 |
| 병렬 실행 | 제한적 지원 | 네이티브 지원 |
| 확장 모델 | @RunWith | @ExtendWith |

### 버전 선택 권장

- **새 프로젝트**: JUnit 5 권장 - 최신 기능과 우수한 확장성
- **기존 프로젝트**: JUnit 4 계속 사용하거나 JUnit Vintage로 점진적 마이그레이션
- **장기 유지보수**: JUnit 5로 점진적으로 마이그레이션하여 더 나은 테스트 경험 얻기
