# Jenkins 使用手册

## 流水线基础

流水线是 Jenkins 的核心功能，它将软件开发过程中的构建、测试、部署等环节自动化地串联起来。

### 什么是流水线

流水线（Pipeline）是 Jenkins 的一套插件，用于实现和集成持续交付流程。简单来说，流水线就是将软件从版本控制到用户手中的整个过程自动化表达。

流水线定义在文本文件（Jenkinsfile）中，可以提交到项目的源代码仓库进行版本管理。这实现了"流水线即代码"的理念。

### 流水线的优势

- **代码化**：流水线作为代码版本化，便于审查和迭代
- **持久性**：流水线可以从 Jenkins 主节点的计划内和计划外重启中存活
- **可暂停**：流水线可以选择性停止，等待人工输入或批准后再继续
- **多功能**：支持复杂的现实世界 CD 需求，包括并行执行、循环等
- **可扩展**：通过插件可以扩展流水线 DSL 和集成其他工具

### 流水线概念

#### 节点

节点是 Jenkins 环境的一部分，是能够执行流水线的机器。节点可以是主节点，也可以是从节点。

#### 阶段

阶段（Stage）定义了流水线中概念上不同的子集，比如"构建"、"测试"、"部署"阶段。阶段用于可视化流水线的执行状态。

#### 步骤

步骤（Step）是 Jenkins 流水线中的基本动作，告诉 Jenkins 在特定时间点要做什么。比如执行 shell 命令使用 `sh` 步骤。

### 流水线语法

#### 声明式流水线

声明式流水线是较新的语法，提供更丰富的特性，更容易编写和阅读：

```groovy
pipeline {
    agent any
    
    stages {
        stage('构建') {
            steps {
                echo '正在构建应用程序...'
            }
        }
        stage('测试') {
            steps {
                echo '正在运行测试...'
            }
        }
        stage('部署') {
            steps {
                echo '正在部署应用程序...'
            }
        }
    }
}
```

#### 脚本化流水线

脚本化流水线使用 Groovy 语法编写，更灵活但需要更多的编程经验：

```groovy
node {
    stage('构建') {
        echo '正在构建应用程序...'
    }
    stage('测试') {
        echo '正在运行测试...'
    }
    stage('部署') {
        echo '正在部署应用程序...'
    }
}
```

## 创建第一个流水线

### 使用 Blue Ocean 创建

Blue Ocean 是 Jenkins 的现代化界面，让创建流水线更加直观：

1. 安装 Blue Ocean 插件（如未预装）
2. 点击"打开 Blue Ocean"
3. 点击"创建流水线"
4. 选择代码存储位置（GitHub、GitLab 等）
5. 选择仓库和分支
6. Blue Ocean 会自动检测 Jenkinsfile 或帮助创建

### 使用 Jenkinsfile 创建

在项目中创建 Jenkinsfile，然后 Jenkins 会自动识别：

1. 在项目根目录创建 Jenkinsfile 文件
2. 定义流水线的各个阶段
3. 将代码提交到版本控制系统
4. 在 Jenkins 中创建流水线任务并关联仓库

## 凭证管理

Jenkins 需要访问各种外部资源，如代码仓库、部署目标等，这些都需要凭证管理。

### 添加凭证

1. 进入"Manage Jenkins" -> "Manage Credentials"
2. 点击"Add Credentials"
3. 选择凭证类型（用户名密码、SSH 密钥、Secret 文件等）
4. 填写凭证信息
5. 为凭证设置 ID 和描述

### 在流水线中使用凭证

使用凭证步骤调用：

```groovy
pipeline {
    agent any
    stages {
        stage('使用凭证') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "用户名: $USER"
                }
            }
        }
    }
}
```

## 构建触发器

自动化构建需要配置触发器，Jenkins 支持多种触发方式。

### 轮询代码仓库

定期检查代码仓库是否有变更：

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // 每5分钟检查一次
    }
    stages {
        stage('构建') {
            steps {
                echo '检测到代码变更，开始构建'
            }
        }
    }
}
```

### Webhook 触发

通过代码仓库的 Webhook 触发构建，效率更高：

1. 在 Jenkins 中安装对应的插件（如 GitHub Hook Trigger）
2. 配置代码仓库的 Webhook 指向 Jenkins
3. 代码提交时自动触发构建

### 定时构建

按照设定的时间表自动执行：

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // 每天凌晨2点执行
    }
    stages {
        stage('定时构建') {
            steps {
                echo '定时构建任务执行中'
            }
        }
    }
}
```

## 使用代理和容器

### 配置代理

Jenkins 支持分布式构建，可以在多台机器上分配构建任务：

1. 进入"Manage Jenkins" -> "Manage Nodes"
2. 添加代理节点，配置连接信息
3. 在流水线中指定代理

### 在流水线中使用 Docker

声明式流水线可以在 Docker 容器中执行构建：

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('构建') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## 测试报告

Jenkins 可以集成各类测试框架，生成可视化的测试报告。

### JUnit 测试报告

```groovy
pipeline {
    agent any
    stages {
        stage('测试') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### 其他测试报告

Jenkins 还支持多种测试报告插件：

- TestNG
- Cobertura（代码覆盖率）
- JaCoCo（代码覆盖率）
- ESLint（前端代码检查）

## 部署实践

### 部署到不同环境

典型的多阶段部署流程：

```groovy
pipeline {
    agent any
    stages {
        stage('构建') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('测试环境部署') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('生产环境部署') {
            when {
                branch 'main'
            }
            steps {
                input message: '确认部署到生产环境?', ok: '部署'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### 蓝绿部署

通过 Jenkins 可以实现蓝绿部署，零停机更新：

1. 维护两套相同的环境（蓝环境和绿环境）
2. 流量切换到新版本
3. 如有问题快速回滚

## 最佳实践

### Jenkinsfile 最佳实践

- 将 Jenkinsfile 纳入版本控制
- 使用声明式流水线语法
- 保持流水线简洁清晰
- 合理划分阶段
- 添加必要的超时和重试机制

### 安全建议

- 定期更新 Jenkins 和插件
- 限制匿名用户权限
- 使用凭证存储敏感信息
- 配置适当的访问控制
- 启用安全日志审计

### 性能优化

- 合理配置构建节点
- 使用增量构建减少耗时
- 配置构建缓存
- 定期清理旧的构建记录
- 使用并行阶段加速构建

## 故障排查

### 查看构建日志

构建失败时，首先查看构建日志：

1. 进入构建任务页面
2. 点击失败的构建编号
3. 查看控制台输出

### 常见问题处理

- 构建卡住：检查是否有挂起的输入等待，是否有资源死锁
- 凭证失效：检查凭证是否过期或被撤销
- 插件冲突：尝试禁用最近安装的插件
- 内存不足：调整 JVM 堆内存参数

- 创建和配置流水线任务
- 管理凭证和触发器
- 集成测试和部署工具
- 应用最佳实践优化工作流程
