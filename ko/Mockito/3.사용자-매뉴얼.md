# Mockito 사용자 매뉴얼

## 모ック 객체 생성

모ック 객체 생성은 Mockito를 사용하여 테스트하는 첫 번째 단계입니다. 모ック 객체를 생성하는 여러 가지 방법이 있으며, 개발자는 필요에 따라 적절한 방법을 선택할 수 있습니다.

### mock() 메서드 사용

가장 기본적인 방법은 Mockito 클래스의 정적 메서드 mock()를 사용하여 모ック 객체를 생성하는 것입니다. 이 메서드는 Class 매개변수를 받아 해당 클래스의 모ック 인스턴스를 반환합니다.

```java
import static org.mockito.Mockito.*;

List<String> mockedList = mock(List.class);
Map<String, Integer> mockedMap = mock(Map.class);
```

Mockito 4.10.0부터는 Class 매개변수를 생략할 수 있으며 컴파일러가 자동으로 타입을 추론합니다.

```java
List<String> mockedList = mock();
```

### @Mock 어노테이션 사용

@Mock 어노테이션은 더 깔끔한 선언 방식을 제공하며, 특히 테스트 클래스에서 여러 모ック 객체를 생성해야 할 때 유용합니다. 어노테이션을 사용하려면 먼저 Mockito 프레임워크를 초기화해야 합니다.

```java
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
    
    @Mock
    private EmailService emailService;
    
    @Before
    public void setup() {
        MockitoAnnotations.initMocks(this);
    }
}
```

MockitoJUnitRunner를 사용하면 초기화를 자동으로 처리할 수 있습니다.

```java
@RunWith(MockitoJUnitRunner.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
}
```

JUnit 5의 @ExtendWith 어노테이션을 사용해도 자동 초기화를 달성할 수 있습니다.

```java
@ExtendWith(MockitoExtension.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
}
```

## 메서드 동작 정의

메서드 동작 정의는 스텁bing이라고도 하며, 모ック 객체가 호출될 때 반환되어야 할 값이나 발생해야 할 예외를 지정하는 데 사용됩니다.

### 기본 반환값 스텁

when().thenReturn() 체인 구문을 사용하여 반환값을 정의합니다.

```java
when(mockedList.size()).thenReturn(100);
when(mockedList.get(0)).thenReturn("first");
when(mockedList.get(1)).thenReturn("second");
```

연속적인 여러 반환값을 정의할 수 있으며, 각 후속 호출은 다른 값을 반환합니다.

```java
when(mockedList.get(anyInt()))
    .thenReturn("first")
    .thenReturn("second")
    .thenThrow(new RuntimeException("Error"));

System.out.println(mockedList.get(0)); // 출력: first
System.out.println(mockedList.get(0)); // 출력: second
System.out.println(mockedList.get(0)); // 예외 발생
```

### 예외 발생 정의

thenThrow()를 사용하여 메서드가 호출될 때 발생해야 할 예외를 정의합니다.

```java
when(mockedList.get(0)).thenThrow(new IndexOutOfBoundsException("Index 0"));
```

반환 타입이 void인 메서드의 경우 doThrow()를 사용합니다.

```java
doThrow(new RuntimeException("Error")).when(mockedList).clear();
```

### Answer를 사용한 사용자 정의 응답

사전 정의된 반환값이 충분하지 않은 경우 Answer 인터페이스를 구현하여 반환값을 동적으로 계산할 수 있습니다.

```java
when(mockedList.get(anyInt())).thenAnswer(invocation -> {
    int index = invocation.getArgument(0);
    return "Value at index " + index;
});
```

## 상호작용 검증

검증은 테스트 중인 코드가 모ック 객체와 예상대로 올바르게 상호작용하는지 확인하는 데 사용됩니다.

### 기본 검증

verify()를 사용하여 메서드가 호출되었는지 확인합니다.

```java
mockedList.add("one");
verify(mockedList).add("one");
```

### 호출 횟수 검증

메서드가 몇 번 호출되었는지 확인할 수 있습니다.

```java
mockedList.add("one");
mockedList.add("one");
mockedList.add("one");

verify(mockedList, times(3)).add("one");       // 정확히 3번
verify(mockedList, atLeast(2)).add("one");    // 최소 2번
verify(mockedList, atMost(3)).add("one");     // 최대 3번
verify(mockedList, never()).add("two");       // 호출되지 않음
verify(mockedList, atLeastOnce()).add("one"); // 최소 한 번
```

### 호출 순서 검증

InOrder 검증자는 메서드 호출이 특정 순서로 실행되었는지 확인할 수 있습니다.

```java
InOrder inOrder = inOrder(mockedList);
inOrder.verify(mockedList).add("first");
inOrder.verify(mockedList).add("second");
inOrder.verify(mockedList).add("third");
```

### 호출 인자 검증

인자 매칭자를 사용하여 호출 인자를 유연하게 검증할 수 있습니다.

```java
verify(mockedList).add(anyString());
verify(mockedList).get(anyInt());
verify(mockedList).contains(argThat(s -> s.length() > 5));
```

## 인자 매칭자

Mockito는 메서드 매개변수의 유연한 매칭을 위한 다양한 인자 매칭자를 제공합니다.

### 기본 매칭자

```java
anyInt()          // 모든 정수
anyString()       // 모든 문자열
any()             // 모든 객체
anyList()         // 모든 리스트
anyMap()          // 모든 맵
isNull()         // null 값
isNotNull()      // null이 아닌 값
eq(value)        // 정확히 일치
startsWith()     // 지정된 문자열로 시작
endsWith()       // 지정된 문자열로 끝
contains()       // 지정된 문자열 포함
```

### 사용자 정의 매칭자

argThat() 메서드를 사용하여 사용자 정의 인자 매칭자를 생성할 수 있습니다.

```java
when(mockedList.add(argThat(s -> s != null && s.length() > 3)))
    .thenReturn(true);
```

## 스파이 사용

스파이는 모ック와 달리 대부분의 실제 객체 행동을 유지하면서 특정 메서드만 스텁할 수 있습니다.

### 스파이 생성

```java
List<String> realList = new ArrayList<>();
List<String> spyList = spy(realList);
```

또는 @Spy 어노테이션을 사용합니다.

```java
@Spy
private List<String> list = new ArrayList<>();
```

### 스파이 동작 정의

모ック와 달리 스파이에서 when()을 사용할 때는 실제 메서드 호출에 영향을 줄 수 있어 주의가 필요합니다. doReturn().when() 구문을 사용하는 것이 좋습니다.

```java
doReturn(100).when(spyList).size();
```

스파이는 명시적으로 스텁하지 않는 한 실제 메서드를 호출합니다.

```java
spyList.add("element");  // 실제 메서드가 호출됨
verify(spyList).add("element");  // 검증 통과
```

## @InjectMocks 사용

@InjectMocks 어노테이션은 테스트할 객체를 자동으로 생성하고 @Mock 또는 @Spy로 주석된 필드를 주입하는 데 사용됩니다.

```java
@RunWith(MockitoJUnitRunner.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
    
    @Mock
    private EmailService emailService;
    
    @InjectMocks
    private UserService userService;
    
    @Test
    public void testCreateUser() {
        User user = new User("testUser");
        when(userRepository.save(any(User.class))).thenReturn(user);
        
        User result = userService.createUser("testUser");
        
        assertEquals("testUser", result.getUsername());
        verify(userRepository).save(any(User.class));
    }
}
```

## ArgumentCaptor 사용

ArgumentCaptor는 메서드 호출 중에 전달된 인자를 캡처하여 더詳細な 어설션을 수행하는 데 사용됩니다.

```java
@Mock
private List<String> mockedList;

@Test
public void testCaptureArgument() {
    mockedList.add("test");
    
    ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
    verify(mockedList).add(captor.capture());
    
    assertEquals("test", captor.getValue());
}
```

메서드가 여러 번 호출될 때 모든 인자를 캡처할 수 있습니다.

```java
mockedList.add("first");
mockedList.add("second");
mockedList.add("third");

ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
verify(mockedList, times(3)).add(captor.capture());

assertEquals(3, captor.getAllValues().size());
assertEquals("second", captor.getAllValues().get(1));
```

## BDD 스타일 테스트

Mockito는 given().willReturn() 구문을 사용하여 동작驱动 개발(BDD) 스타일 테스트를 지원합니다.

```java
import static org.mockito.BDDMockito.*;

@Test
public void testBddStyle() {
    // Given
    given(mockedList.get(0)).willReturn("first");
    
    // When
    String result = mockedList.get(0);
    
    // Then
    assertEquals("first", result);
}
```

## 일반적인 사용 패턴

### void 메서드 테스트

```java
@Test
public void testVoidMethod() {
    doNothing().when(mockedList).clear();
    mockedList.clear();
    verify(mockedList).clear();
}
```

### 메서드 체인 모킹

```java
@Test
public void testMethodChaining() {
    when(mockedList.iterator()).thenReturn(Collections.emptyIterator());
    
    Iterator<String> iterator = mockedList.iterator();
    assertFalse(iterator.hasNext());
}
```

### 예외 처리 모킹

```java
@Test(expected = RuntimeException.class)
public void testException() {
    when(mockedList.get(0)).thenThrow(new RuntimeException("Error"));
    mockedList.get(0);
}
```

## 모범 사례

Mockito를 사용할 때 테스트 코드의 품질과 유지보수를 보장하기 위해 모범 사례를 따르세요. 소유한 타입이나 값 객체를 모킹하지 마세요,，因为这会破坏测试的真实性. 테스트의 진실성을 유지하기 위해 필요한 의존성만 모킹하세요. 각 테스트 메서드는 하나의 기능 테스트에 집중해야 합니다. @Before을 사용하여 공통 설정을 하고 테스트 코드를 적절히 구성하세요. 검증은适中해야 하며, 모든 메서드 호출을 검증하기보다는 중요한 상호작용에 집중하세요.
