# JDK Guía de Instalación

## Instalación de JDK en Windows

### Requisitos del sistema

Antes de instalar JDK, asegúrese de que su computadora cumpla con los siguientes requisitos mínimos: Procesador, se recomienda Intel Core i3 o superior; Memoria, se recomiendan al menos 4GB de RAM, preferiblemente 8GB o más; Espacio en disco, se necesitan al menos 2GB de espacio libre para instalar JDK, más espacio adicional para proyectos de desarrollo; Sistema operativo, Windows 10 o Windows 11 de 64 bits.

Aunque Windows 7 y 8.1 pueden funcionar, Oracle ya no officially supports estos sistemas operativos. Si ejecuta en sistemas más antiguos, pueden ocurrir problemas de compatibilidad.

### Descarga de JDK

Primero, visite el sitio web oficial de Oracle en https://www.oracle.com/java/technologies/downloads/ para descargar JDK. En la página de descarga, puede ver las diferentes versiones de JDK: JDK 25 es la versión más reciente, JDK 21 es la última versión de soporte a largo plazo, JDK 17 también es una versión LTS madura. Elija la versión apropiada según los requisitos de su proyecto.

Para sistemas Windows, elija el instalador x64 ( .exe) o el instalador MSI x64 ( .msi). El .exe es el asistente de instalación tradicional, el .msi es adecuado para implementación por lotes en entornos empresariales. Si solo necesita ejecutar programas Java sin instalar, elija el archivo comprimido x64 ( .zip), que puede extraer y usar directamente.

Preste atención a los términos de licencia durante la descarga. Oracle JDK es gratuito bajo ciertas versiones y usos, pero tiene restricciones para uso comercial y redistribución. Si necesita uso completamente gratuito, considere usar OpenJDK u otras distribuciones basadas en OpenJDK como Eclipse Temurin.

### Pasos de instalación

Encuentre el archivo de instalación de JDK descargado (generalmente jdk-xx_windows-x64_bin.exe) y haga doble clic para ejecutar. En el asistente de instalación, haga clic en "Siguiente" para continuar. Primero verá la pantalla de selección de carpeta de destino, la ruta de instalación predeterminada es C:\Program Files\Java\jdk-xx. Si necesita cambiar la ubicación de instalación, haga clic en "Cambiar" para elegir otro directorio. Se recomienda usar la ruta predeterminada para evitar problemas de configuración posteriores.

A continuación está la pantalla de selección de funciones, por defecto todas las funciones están seleccionadas, incluyendo herramientas de desarrollo, código fuente y JRE público. Se recomienda mantener la instalación completa predeterminada. Haga clic en "Siguiente" para comenzar la instalación. El proceso de instalación puede tomar unos minutos, por favor tenga paciencia.

Después de que se complete la instalación, verá un mensaje. Haga clic en "Cerrar" para terminar. En este punto, JDK está instalado en el directorio especificado, pero necesita configurar las variables de entorno para usar java, javac y otros comandos desde la línea de comandos.

### Configuración de variables de entorno

El propósito de configurar las variables de entorno es permitir que el sistema encuentre la ubicación de instalación de JDK para que pueda usar java, javac y otros comandos desde cualquier directorio. Haga clic derecho en "Esta PC" o "Mi PC" y seleccione "Propiedades". En el menú de la izquierda, haga clic en "Configuración avanzada del sistema".

En la ventana de Propiedades del sistema abierta, haga clic en el botón "Variables de entorno". En el área "Variables del sistema", haga clic en "Nueva". Ingrese JAVA_HOME para el nombre de variable y la ruta de instalación de JDK (como C:\Program Files\Java\jdk-21) para el valor de variable, luego haga clic en Aceptar.

Encuentre la variable Path en las variables del sistema y haga clic en "Editar". Haga clic en "Nueva" e ingrese %JAVA_HOME%\bin. Asegúrese de que esta línea esté arriba para que se priorice el JDK recién instalado. Haga clic en Aceptar para guardar toda la configuración.

Abra una nueva ventana de línea de comandos (cmd o PowerShell) e ingrese java -version y javac -version. Si puede ver información de versión, la configuración fue exitosa.

### Verificación de la instalación

Abra el símbolo del sistema (Win+R, ingrese cmd), y ejecute los siguientes comandos para verificar que JDK esté instalado correctamente:

Ejecute java -version, que debe mostrar la información de la versión del entorno de ejecución de Java. Ejecute javac -version, que debe mostrar la información de la versión del compilador de Java. Ejecute echo %JAVA_HOME%, que debe mostrar la ruta de JAVA_HOME configurada anteriormente.

Si algún comando muestra un error, verifique si las variables de entorno están configuradas correctamente o si JDK está instalado correctamente.

## Instalación de JDK en macOS

### Uso del paquete de instalación DMG

Los usuarios de macOS pueden instalar JDK rápidamente usando el paquete de instalación DMG. Visite la página de descarga oficial de Oracle y descargue la versión de JDK para macOS. Para Apple Silicon (chips M1/M2/M3), elija ARM64 DMG Installer; para Mac con chips Intel, elija x64 DMG Installer.

Después de descargar, abra el archivo DMG y verá un archivo de instalación pkg. Haga doble clic para ejecutar el asistente de instalación y siga las indicaciones para completar la instalación. El instalador colocará JDK en el directorio /Library/Java/JavaVirtualMachines/.

Después de que se complete la instalación, abra Terminal y escriba java -version para verificar. Si hay varias versiones de JDK en el sistema, puede usar /usr/libexec/java_home -V para ver todas las versiones instaladas.

### Instalación usando Homebrew

Si Homebrew ya está instalado, puede usarlo para instalar JDK, lo cual es más conveniente. Abra Terminal y ejecute: brew update. Luego instale JDK 21: brew install openjdk@21.

Después de que se complete la instalación, necesita agregar JDK a PATH. Ejecute el siguiente comando para vincular JDK al sistema: sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.

Luego agregue variables de entorno en ~/.zshrc o ~/.bash_profile: export PATH="/usr/local/opt/openjdk@21/bin:$PATH". Guarde y ejecute source ~/.zshrc para aplicar la configuración.

### Configuración de JAVA_HOME

En macOS, configurar la variable de entorno JAVA_HOME no es obligatorio porque el sistema encuentra automáticamente los JDK instalados. Sin embargo, algunas herramientas (como Maven, Gradle) pueden requerir configuración manual. En Terminal, edite el archivo ~/.zshrc y agregue: export JAVA_HOME=$(/usr/libexec/java_home). Guarde y ejecute source ~/.zshrc para aplicar la configuración.

## Instalación de JDK en Linux

### Instalación usando APT (Debian/Ubuntu)

En sistemas Debian y Ubuntu, puede instalar OpenJDK usando el gestor de paquetes APT. Primero actualice la lista de paquetes: sudo apt update. Luego instale OpenJDK 21 versión de desarrollo: sudo apt install openjdk-21-jdk. Si solo necesita ejecutar programas Java sin el compilador, instale JRE: sudo apt install openjdk-21-jre.

Durante la instalación, se le preguntará si desea crear el directorio JAVA_HOME. Seleccione "Sí" para configuración automática. Después de que se complete la instalación, use java -version y javac -version para verificar la instalación.

Si necesita configurar JAVA_HOME manualmente, edite el archivo ~/.bashrc o ~/.profile y agregue: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64. Luego ejecute source ~/.bashrc para aplicar la configuración.

### Instalación usando YUM (Red Hat/CentOS)

En sistemas que usan YUM como Red Hat, CentOS y Fedora, puede instalar JDK directamente. Ejecute el siguiente comando para instalar OpenJDK 21: sudo yum install java-21-openjdk java-21-openjdk-devel. El paquete -devel incluye el compilador y otras herramientas de desarrollo.

Después de que se complete la instalación, use el comando alternatives para establecer la versión predeterminada de Java: sudo alternatives --config java. Siga las indicaciones para seleccionar la versión de Java para establecer como predeterminada.

Edite el archivo ~/.bashrc y agregue: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk. Ejecute source ~/.bashrc para aplicar la configuración.

### Instalación manual de archivos binarios

Para situaciones que requieren instalar una versión específica de JDK, puede descargar y extraer manualmente archivos binarios. Visite la página de descarga de Oracle o OpenJDK y descargue el archivo .tar.gz de la versión deseada.

Cree el directorio de instalación de JDK: sudo mkdir -p /opt/jdk. Extraiga el archivo descargado: sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk. Ingrese al directorio extraído y anote la ruta completa.

Edite el archivo ~/.bashrc y agregue: export JAVA_HOME=/opt/jdk/jdk-21 y export PATH=$JAVA_HOME/bin:$PATH. Ejecute source ~/.bashrc para aplicar la configuración.

## Gestión de múltiples versiones de JDK

### Gestión de múltiples versiones en Windows

Después de instalar múltiples versiones de JDK en Windows, cambie la versión predeterminada a través de variables de entorno. El método más simple es modificar el orden de los directorios bin de JDK en la variable de entorno PATH, colocando la versión que desea usar al frente.

Otro método es usar la variable de entorno JAVA_HOME, apuntándola a diferentes directorios de instalación de JDK. Este método funciona para la mayoría de las herramientas, pero algunas herramientas pueden usar el comando java en PATH en lugar de JAVA_HOME.

Algunas herramientas de terceros pueden ayudar a administrar múltiples versiones de JDK, como la versión de Windows de SDKMAN! o JetBrains Toolbox. Estas herramientas hacen que cambiar entre versiones sea más conveniente.

### Gestión de múltiples versiones en macOS

En macOS, puede usar la herramienta jenv para administrar múltiples versiones de JDK. Primero instale jenv: brew install jenv. Luego agregue la inicialización de jenv al archivo de configuración de shell.

Agregue el JDK instalado a jenv: jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home. Luego puede usar jenv versions para ver todas las versiones administradas y jenv global xx para establecer la versión predeterminada global.

Otro método es usar /usr/libexec/java_home -v para especificar una versión particular para ejecutar programas. Por ejemplo: /usr/libexec/java_home -v 21 --exec java -version.

### Gestión de múltiples versiones en Linux

En Linux, use la herramienta update-alternatives para administrar múltiples versiones de JDK. Primero cree entradas de alternatives para cada versión de JDK: sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1.

Luego use update-alternatives --config java para seleccionar la versión predeterminada. jenv también está disponible en Linux, con métodos de instalación y configuración similares a macOS.

## Problemas comunes de instalación

### No se puede encontrar el comando java después de la instalación

Si no puede encontrar el comando java después de instalar JDK, primero confirme si JDK está instalado correctamente. Verifique si la variable de entorno JAVA_HOME está configurada correctamente y si PATH incluye %JAVA_HOME%\bin (Windows) o $JAVA_HOME/bin (Linux/macOS).

En Windows, abra una nueva ventana de línea de comandos porque los cambios de variables de entorno pueden no tener efecto inmediato. En Linux/macOS, ejecute source ~/.bashrc o vuelva a abrir la terminal.

Si el problema persiste, verifique si el directorio de instalación de JDK es correcto y si el archivo bin/java existe en ese directorio.

### Problemas de permisos

Al instalar JDK en Linux y macOS, puede encontrar problemas de permisos. Al usar el administrador de paquetes, generalmente se requieren privilegios de administrador (sudo). Al instalar manualmente, asegúrese de que el directorio de instalación tenga los permisos de lectura y escritura apropiados.

Si los usuarios regulares no pueden usar el comando java, verifique los permisos del directorio JAVA_HOME y del directorio bin de JDK. Puede usar ls -la /usr/lib/jvm para verificar los permisos del directorio.

### Configuración de JAVA_HOME no funciona

Si establecer JAVA_HOME no funciona, verifique si la sintaxis de la variable de entorno es correcta. En Windows, el valor de la variable no debe contener comillas. En Linux/macOS, asegúrese de usar el comando export correcto.

También verifique si otros scripts o archivos de configuración están sobrescribiendo la configuración de JAVA_HOME. En Linux, la configuración a nivel de sistema puede estar en /etc/environment o /etc/profile.d/.

### Desinstalación de versiones anteriores

Después de actualizar JDK, puede necesitar desinstalar versiones anteriores para ahorrar espacio en disco. En Windows,找到控制面板的"程序和功能"中的旧JDK版本，点击卸载。在macOS上，删除/Library/Java/JavaVirtualMachines/目录下对应的旧版本文件夹。在Linux上，使用包管理器卸载：sudo apt remove openjdk-xx-jdk（APT）或sudo yum remove java-xx-openjdk（YUM）。
