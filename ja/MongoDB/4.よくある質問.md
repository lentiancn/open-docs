# MongoDB よくある質問

## 基本概念

### MongoDB とは？

MongoDB は、開発者が最新のアプリケーションをより速く構築できるように設計されたドキュメントデータベースです。データを柔軟な JSON 様のドキュメントに保存することで、アプリケーションコードがデータを使用するのと同じ方法でデータをモデル化できます。MongoDB は強力なクエリエンジン、水平スケーリング、組み込みの高可用性を提供します。

### MongoDB は関係データベースですか？

いいえ。MongoDB は NoSQL ドキュメントデータベースで、JSON 様の BSON 形式を使用してデータを保存します。従来の関係データベースとは本質的に異なり、表、行、列の概念を使用せず、コレクションとドキュメントを使用します。

### MongoDB のドキュメントとは？

ドキュメントは MongoDB の基本的なデータユニットで、フィールドと値のペアで構成されます。ドキュメントには他のドキュメント、配列、ネストされたオブジェクトを含めることができ、非常に柔軟な構造を持っています。各ドキュメントには主キーとして固有の _id フィールドがあります。

### MongoDB と MySQL の違いは？

| 機能 | MongoDB | MySQL |
|------|---------|-------|
| データモデル | ドキュメント（JSON類似） | 関係（行と列） |
| スキーマ | 柔軟（動的スキーマ） | 固定（静的スキーマ） |
| スケーリング | 水平（シャーディング） | 垂直 |
| クエリ言語 | MongoDB クエリ言語 | SQL |
| トランザクション | マルチドキュメント ACID | マルチテーブル ACID |
| 関係 | 埋め込みドキュメント/配列 | JOIN操作 |
| インデックス | 多种インデックスタイプ | インデックスをサポート |

### BSON とは？

BSON は Binary JSON の略で、JSON ドキュメントのバイナリ表現で、追加のデータ型（ObjectId、Date、Binary など）をサポートします。

### MongoDB の最大ドキュメントサイズは？

BSON ドキュメントサイズの最大치는 16 MB です。この制限を超えるファイルには、MongoDB が GridFS API を提供します。

### MongoDB は哪些データ型をサポートします？

MongoDB は多くの BSON データ型をサポートします：Double、String、Object、Array、Binary Data、ObjectId、Boolean、Date、Null、Regular Expression、JavaScript、JavaScript with Scope、32-bit Integer、Timestamp、64-bit Integer、Decimal128、Min Key、Max Key など。

## インストール問題

### MongoDB はどのくらいのメモリが必要ですか？

MongoDB は利用可能なメモリをキャッシュとして使用しようとします。開発環境では少なくとも 4GB のメモリを推奨します。本番環境では、データ量と負荷に基づいて評価してください。

### MongoDB は 32 ビットシステムをサポートしていますか？

サポートしていません。MongoDB はあるバージョンから 32 ビットプラットフォームのサポートを停止了。64 ビットオペレーティングシステムを使用してください。

### MongoDB のデフォルトポートを変更するには？

MongoDB のデフォルトポートは 27017 です。変更方法は：

1. 設定ファイルの `net.port` を変更する
2. 起動時に `--port` パラメータを指定する

### 同じマシンで複数の MongoDB インスタンスを実行できますか？

できますが、それぞれ異なる設定が必要です：

- ポート番号
- データディレクトリ（`--dbpath`）
- ログファイルパス
- 設定ファイル

### MongoDB はどこにデータを保存しますか？

デフォルトでは：

- Linux/macOS: `/data/db`
- Windows: `C:\data\db`

設定ファイルの `storage.dbPath` で別のパスを指定できます。

### MongoDB 8.0 と 7.0 のどちらをインストールすべきですか？

- 新規プロジェクト: 最新の安定バージョン MongoDB 8.2 を使用
- 本番環境: 最新の安定バージョンまたは長期サポートバージョン MongoDB 7.0 を選択

MongoDB 6.0 は2025年7月31日にサポートが終了しました。

## 使用問題

### MongoDB データのバックアップ方法は？

mongodump ツールを使用:

```bash
# データベース全体をエクスポート
mongodump --db=mydatabase --out=/backup

# 指定コレクションをエクスポート
mongodump --db=mydatabase --collection=users --out=/backup
```

mongorestore でリストア:

```bash
mongorestore --db=mydatabase /backup/mydatabase
```

### MongoDB はどのように高可用性を実現しますか？

レプリカセット（Replica Set）を通じて高可用性を實現します。レプリカセットはプライマリノードと複数のセカンダリで構成されます：

- プライマリノードはすべての書き込み操作を処理
- セカンダリはプライマリのデータをレプリケーション
- プライマリ障害時に自動的に新しいプライマリを選举

### シャーディングとは？

シャーディングは MongoDB の水平スケーリング方法で、データを複数のサーバーに分散させます。各シャードはデータの一部を保持し、組み合わせて完全なデータセットを形成します。

### レプリカセットとシャードクラスターの違いは？

- **レプリカセット**: 複数の MongoDB インスタンスが同じデータコピーを維持し、高可用性と読み取りスケーリングを提供
- **シャードクラスター**: データが複数のサーバーに分散し、水平スケーリングをサポートし、大規模データセットをサポート

### クエリパフォーマンスを最適化する方法は？

1. 適切なインデックスを作成: 一般的なクエリフィールドに `createIndex()` を使用
2. プロジェクションを使用: 必要なフィールドのみを返してネットワーク転送を削減
3. フルコレクションスキャンを避ける: クエリがインデックスを使用することを確認
4. 集約パイプラインを使用: 複雑なデータ処理のため
5. カバードインデックスを適切に使用: インデックスがすべてのクエリフィールドを含む
6. 結果を制限: `limit()` と `skip()` でページネーション

### MongoDB はトランザクションをサポートしていますか？

サポートしています。MongoDB 4.0+ はレプリカセットとシャードクラスター間で動作するマルチドキュメントトランザクションを導入しました。トランザクションは ACID セマンティクスを遵循します：

- 原子性: トランザクション内のすべての操作は完全に成功するか完全に失敗します
- 一貫性: トランザクションの前後でデータベース状態は一貫しています
- 分離: 同時実行トランザクションは相互に分離されています
- 持続性: コミット後、データは永続的に保存されます

### 変更ストリームとは？

変更ストリームは MongoDB の機能で、アプリケーションがコレクション、データベース、またはクラスター全体のデータ変更イベントをリアルタイムでリッスンできます。アプリケーションは変更イベントをサブスクライブして、リアルタイム同期、監査ログなどの機能を実装できます。

### MongoDB はどのようにデータセキュリティを保証しますか？

1. **認証**: ユーザー名/パスワード認証または LDAP/Kerberos（エンタープライズ版）を有効化
2. **アクセス制御**: ロールベースのアクセス制御（RBAC）を使用
3. **暗号化通信**: TLS/SSL 暗号化を有効化
4. **保存暗号化**: MongoDB Enterprise の暗号化保存エンジン使用
5. **フィールドレベル暗号化**: 機密フィールドを暗号化
6. **監査ログ**: データベース操作を記録（エンタープライズ版）
7. **定期的なバックアップ**: データを定期的にバックアップ

### MongoDB エンタープライズ版とコミュニティ版の違いは？

エンタープライズ版には更多の高機能機能が含まれています：

- LDAP 認証
- Kerberos 認証
- 監査
- 暗号化保存エンジン
- フラット分析
- 技術サポート

コミュニティ版の機能はほとんどのユースケースに十分です。

### 適切なバージョンを選択する方法は？

- 新規プロジェクト: 最新の安定バージョン（MongoDB 8.2）を使用
- 本番環境: 最新の安定バージョンまたは長期サポートバージョン（MongoDB 7.0）を選択
- 特定の要件: アプリケーションの互換性と機能要件に基づいて選択

### GridFS とは？

GridFS は MongoDB で大きなファイルを保存するための仕様です。大きなファイルを複数のチャンク（デフォルト 255KB）に分割し、`fs.chunks` コレクションに保存し、ファイルメタデータを `fs.files` コレクションに保存します。

### MongoDB はどのようなデータの保存に適していますか？

- コンテンツ管理システムデータ
- ユーザー設定とプロファイルデータ
- ログとイベントデータ
- IoT センサーデータ
- リアルタイム分析データ
- モバイルアプリケーションバックエンドデータ
- 电子商务製品カタログ
- チャットメッセージとリアルタイム通信データ

### レプリカセット oplog とは？

oplog（操作ログ）はレプリカセットの特別なコレクションで、プライマリノード上のすべての書き込み操作を記録します。セカンダリは oplog から操作を読み取って適用することで、プライマリとのデータ同期を維持します。

### MongoDB パフォーマンスを監視する方法は？

1. `db.serverStatus()` でサーバーステータスを確認
2. `db.stats()` でデータベースステータスを確認
3. プロファイラーでスロークエリを分析: `db.setProfilingLevel(1, 100)`
4. `db.currentOp()` で現在の操作を確認
5. Prometheus と Grafana を統合して視覚的監視
6. MongoDB Atlas の組み込み監視機能を使用（クラウドサービスを使用する場合）

### レプリカセットの複製遅延とは何ですか？

複製遅延は、プライマリノード上の操作とその操作がセカンダリに適用されるまでの遅延です。少量の遅延は許容できますが、複製遅延が増加すると重大な問題が発生します。以下の方法で監視:

```bash
rs.status()
```

`optimes.lastCommittedOpTime` とプライマリの操作時間の差を確認します。

### 読み取り設定（Read Preference）とは？

読み取り設定はクライアントがレプリカセットのどのノードから読み取るかを決定します：

- **primary**: プライマリから読み取り（デフォルト）
- **primaryPreferred**: 優先してプライマリから、必要に応じてセカンダリから
- **secondary**: セカンダリからのみ読み取り
- **secondaryPreferred**: 優先してセカンダリから、必要に応じてプライマリから
- **nearest**: ネットワーク遅延が最も低いノードから読み取り

### 書き込み設定（Write Concern）とは？

書き込み設定は書き込み操作の確認レベルを指定します：

- **w: 1**: プライマリの確認を待機（デフォルト）
- **w: majority**: ほとんどのレプリカセットメンバーの確認を待機
- **w: "all"**: すべてのレプリカセットメンバーの確認を待機
- **j: true**: ディスクジャーナルへの書き込みを待機
- **wtimeout**: タイムアウト値

### capped collection とは？

capped collection は固定サイズのコレクションで、循環キューに類似しています。サイズ制限に達すると、古いドキュメントは自動的に新しいドキュメントに置き換えられます。ログ、キャッシュなどのシナリオによく使用されます。

### MongoDB は哪些インデックスタイプをサポートしていますか？

- 単一フィールドインデックス
- 複合インデックス
- マルチキーインデックス（配列フィールド）
- テキストインデックス
- 地理空間インデックス（2dsphere、2d）
- ハッシュインデックス
- 一意インデックス
- パーシャルインデックス
- スパースインデックス
- TTL インデックス

### TTL インデックスとは？

TTL（Time-To-Live）インデックスにより、ドキュメントは指定された時間後に自動的に期限切れになり削除されます。セッション、キャッシュ、一時データなどのシナリオによく使用されます。

## トラブルシューティング

### 接続が拒否された場合は？

1. MongoDB サービスが実行されているか確認: `sudo systemctl status mongod`
2. ポートが正しいか確認: デフォルトは 27017
3. ファイアウォール設定を確認
4. バインド IP 設定を確認

### ディスク容量が不足した場合は？

1. 古いデータをクリーンアップ
2. 不要なコレクションを削除
3. `db.repairDatabase()` で容量を回復
4. ディスク容量を拡張

### クエリが遅い場合は？

1. `explain()` でクエリプランを分析
2. 適切なインデックスを作成
3. スロークエリがあるか確認: `db.getProfilingLevel()`
4. プロジェクションを使用して必要なフィールドのみを返すことを検討

### レプリカセットの选举が失敗した場合は？

1. ネットワーク接続を確認
2. ノード設定を確認
3. ほとんどのノードが利用可能であることを確認
4. ログを確認: `rs.status()`

### データが不整合の場合は？

1. セカンダリを同期
2. レプリカセットを再初期化
3. バックアップからデータをリストア

## 開発関連

### MongoDB は哪些プログラミング言語をサポートしていますか？

MongoDB はほとんどすべての主要なプログラミング言語のドライバーを提供します：

- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP
- Scala
- Kotlin
- Swift

### アプリケーションで MongoDB を使用するには？

1. 対応する言語のドライバーをインストール
2. 接続を確立
3. CRUD 操作を実行

Node.js の例:

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('mydb');
  const collection = database.collection('users');
  
  // ドキュメントを挿入
  await collection.insertOne({ name: '田中', age: 30 });
  
  // ドキュメントをクエリ
  const user = await collection.findOne({ name: '田中' });
  
  await client.close();
}
```

### MongoDB Atlas とは？

MongoDB Atlas は MongoDB が提供するクラウドデータベースサービスで、完全管理型の MongoDB 展開を提供：

- 自動バックアップとリストア
- グローバル分散
- リアルタイムパフォーマンス監視
- 運用管理不要
- 免费層が試用可能

### データモデリングを行う方法は？

MongoDB データモデリングの原則：

1. 正規化よりもドキュメント構造を優先
2. 埋め込みドキュメントで「包含」関係を表現
3. 参照で「多対多」関係を表現
4. クエリパターンに基づいてコレクションとインデックスを設計
5. データ成長とスケーラビリティを考慮
