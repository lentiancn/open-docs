# OpenClaw Installation Guide

OpenClaw supports multiple installation methods. Choose the one that best fits your needs.

## Requirements

- **Node.js**: 22.x or higher
- **Operating System**: macOS, Linux, Windows (via WSL2)
- **Optional**: Tailscale (for remote access)

## Quick Install

### macOS / Linux

The easiest way is using the install script:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Or using npm/pnpm:

```bash
npm install -g openclaw@latest
# or
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## Initial Setup

After installation, run the onboarding wizard:

```bash
openclaw onboard --install-daemon
```

The wizard will guide you through:

1. **Auth configuration**: Set up API keys or OAuth
2. **Gateway settings**: Configure gateway port and service
3. **Channel connection**: Connect messaging platforms
4. **Skills installation**: Select and install needed skills

## Install as System Service

Use `--install-daemon` to install OpenClaw as a background service:

```bash
openclaw onboard --install-daemon
```

This creates a user-level service (launchd or systemd) that ensures OpenClaw always runs.

## Run Gateway Manually

If you prefer not to install as a service:

```bash
openclaw gateway --port 18789 --verbose
```

## Docker Installation

If you prefer Docker, refer to the official Docker deployment documentation.

## Build from Source

If you want to contribute or customize OpenClaw:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

Note: pnpm is used for builds. TypeScript can run directly via tsx.

## Verify Installation

After installation, you can:

1. **Check Gateway status**:
   ```bash
   openclaw gateway status
   ```

2. **Open Control UI**:
   ```bash
   openclaw dashboard
   ```
   Or visit `http://127.0.0.1:18789/` in your browser

3. **Send a test message** (requires channel setup):
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## Update OpenClaw

OpenClaw has three release channels:

- **stable**: Official releases (recommended)
- **beta**: Pre-release versions
- **dev**: Development builds

Switch channels:

```bash
openclaw update --channel stable|beta|dev
```

After updating, run:

```bash
openclaw doctor
```

## Common Commands

| Command | Description |
|---------|-------------|
| `openclaw onboard` | Run initial setup wizard |
| `openclaw gateway start` | Start Gateway |
| `openclaw gateway status` | Check Gateway status |
| `openclaw dashboard` | Open Control UI |
| `openclaw agent --message "..."` | Send message to assistant |
| `openclaw message send --to xxx --message "xxx"` | Send a message |
| `openclaw doctor` | Diagnose and fix issues |
| `openclaw update` | Update OpenClaw |

## Environment Variables

For custom configuration locations:

- `OPENCLAW_HOME`: Sets the home directory
- `OPENCLAW_STATE_DIR`: Overrides state directory
- `OPENCLAW_CONFIG_PATH`: Overrides config file path

## Troubleshooting

**Installation fails?**

- Ensure Node.js version >= 22
- Check network connection
- Run `openclaw doctor` for diagnosis

**How to uninstall?**

- Stop the Gateway service
- Remove the installation directory
- Clean up config files
