# Guide d'installation d'OpenClaw

OpenClaw prend en charge plusieurs méthodes d'installation, que vous utilisiez macOS, Linux ou Windows. Ce guide vous accompagnera à travers chaque option d'installation et leurs prérequis.

---

## Configuration requise

Avant d'installer, assurez-vous que votre système répond à ces exigences :

- **Node.js** : 22.0.0 ou supérieur
- **Système d'exploitation** :
  - macOS 10.15 (Catalina) ou supérieur
  - Linux (Ubuntu 20.04+, Debian 11+, Fedora 36+, etc.)
  - Windows 10/11 (WSL2 fortement recommandé)
- **Réseau** : Accès au registre npm et aux services OpenClaw
- **Stockage** : Au moins 500 Mo d'espace libre

Vérifiez votre version de Node.js :

```bash
node --version
```

Si la version est inférieure à 22.0.0,Veuillez mettre à niveau Node.js. Nous recommandons d'utiliser nvm (Node Version Manager) pour gérer plusieurs versions de Node.js.

## Méthodes d'installation

### Méthode 1 : Script d'installation officiel (Recommandé)

C'est la méthode la plus simple et la plus rapide. Le script d'installation officiel détecte automatiquement votre système d'exploitation et termine toute la configuration.

**macOS / Linux :**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows (PowerShell) :**

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Méthode 2 : Installation globale npm

Si vous avez déjà Node.js et npm installés :

```bash
npm install -g openclaw@latest
```

Vous pouvez également spécifier une version particulière :

```bash
npm install -g openclaw@v2026.2.28
```

Balises de version disponibles :
- `latest` : Dernière version stable
- `beta` : Version bêta
- `dev` : Version de développement

### Méthode 3 : Installation pnpm

OpenClaw recommande d'utiliser pnpm :

```bash
pnpm add -g openclaw@latest
```

### Méthode 4 : Installation Bun

Si vous utilisez Bun comme runtime JavaScript :

```bash
bun add -g openclaw@latest
```

### Méthode 5 : Compiler depuis les sources

```bash
# Cloner le dépôt
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Installer les dépendances
pnpm install

# Compiler le projet
pnpm ui:build
pnpm build

# Initialiser et installer le service
pnpm openclaw onboard --install-daemon

# Mode développement (rechargement automatique)
pnpm gateway:watch
```

## Configuration post-installation

### Assistant d'intégration

Après l'installation, exécutez l'assistant :

```bash
openclaw onboard --install-daemon
```

### Vérifier l'installation

```bash
# Vérifier la version
openclaw --version

# Vérifier le statut du Gateway
openclaw gateway status

# Ouvrir l'interface de contrôle
openclaw dashboard
```

## Problèmes courants d'installation

### Problème 1 : Erreurs de permission npm

Utilisez nvm :

```bash
# Installer nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Utiliser nvm pour installer Node.js
nvm install 22
nvm use 22

# Ensuite installer OpenClaw
npm install -g openclaw@latest
```

### Problème 2 : Le Gateway ne démarre pas

Vérifiez si le port est déjà utilisé :

```bash
lsof -i :18789
```

### Problème 3 : Version obsolète

Mettez à jour :

```bash
npm update -g openclaw@latest
```

---

## Mettre à jour OpenClaw

```bash
npm update -g openclaw@latest
openclaw doctor
```

---

## Désinstaller OpenClaw

```bash
# Arrêter le service
openclaw gateway stop

# Désinstaller le package global
npm uninstall -g openclaw

# Supprimer le répertoire de configuration (optionnel)
rm -rf ~/.openclaw
```
