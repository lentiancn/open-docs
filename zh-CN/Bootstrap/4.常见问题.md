# Bootstrap 常见问题

常见问题解答。

## 安装问题

### Q: CDN 无法加载？

**解决方案**：
1. 检查网络连接
2. 确认 URL 正确
3. 使用备用 CDN（如 bootcdn、staticfile）

### Q: npm 安装失败？

**解决方案**：
1. 清除 npm 缓存：`npm cache clean --force`
2. 检查 Node.js 版本
3. 使用管理员权限

### Q: jQuery 是否还需要？

**解决方案**：
不需要。Bootstrap 5 已移除 jQuery 依赖，使用原生 JavaScript。

## 使用问题

### Q: 样式不生效？

**解决方案**：
1. 确认 CSS 文件正确引入
2. 检查 CSS 引入顺序
3. 清理浏览器缓存

### Q: 响应式不工作？

**解决方案**：
1. 添加 viewport meta 标签：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
2. 检查栅格类使用是否正确

### Q: 组件不工作？

**解决方案**：
1. 确认引入 Bootstrap JavaScript 文件
2. 检查 Popper.js 是否正确引入
3. 查看控制台错误信息

### Q: 如何自定义 Bootstrap？

**解决方案**：
1. 使用 SASS 变量覆盖
2. 使用 CSS 自定义属性
3. 使用 Bootstrap 自定义工具

## 布局问题

### Q: 容器宽度不对？

**解决方案**：
- `.container`：固定宽度
- `.container-fluid`：100% 宽度
- `.container-{breakpoint}`：响应式容器

### Q: 列之间的间距怎么调？

**解决方案**：
1. 使用 gutter 类：`g-4`, `g-md-5`
2. 自定义 `$grid-gutter-width` SASS 变量

## 兼容性问题

### Q: 不支持 IE 浏览器？

**解决方案**：
- Bootstrap 5 不再支持 IE 10 以下
- 如需兼容，使用 Bootstrap 4

### Q: 旧版浏览器支持？

**解决方案**：
使用 Bootstrap 4 可获得更好的旧浏览器支持。

## 性能问题

### Q: 文件太大？

**解决方案**：
1. 使用 SASS 按需引入
2. 移除未使用的 CSS
3. 使用生产版本（.min 文件）

### Q: 如何减少 JavaScript 大小？

**解决方案**：
按需引入单个组件。
