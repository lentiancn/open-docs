# Introducción

Apache APISIX es un proyecto de nivel superior de la Apache Software Foundation, desarrollado y donado por API7.ai. Se trata de una puerta de enlace (gateway) de API nativa para la nube, conocida por ser dinámica, en tiempo real y de alto rendimiento.

APISIX actúa como el punto de entrada para todo el tráfico de tu negocio y ofrece un conjunto completo de funcionalidades: rutas dinámicas, upstream dinámico, certificados dinámicos, pruebas A/B, despliegues progresivos (canary), despliegues blue-green, limitación de velocidad, protección contra ataques, recolección de métricas, alertas de monitoreo, observabilidad y gobierno de servicios.

## Características Principales

### Alto Rendimiento

APISIX está construido sobre Nginx, heredando su excelente rendimiento. Gracias a su sistema de plugins, la lógica de negocio está desacoplada de las funciones centrales, lo que hace que las extensiones sean más flexibles sin sacrificar la eficiencia en el procesamiento de solicitudes.

### Totalmente Dinámico

A diferencia de Nginx tradicional, que requiere recargar la configuración, APISIX permite una gestión completamente dinámica. Todas las rutas, upstream, plugins y demás configuraciones pueden actualizarse en tiempo real a través de la API de administración, sin necesidad de reiniciar el servicio.

### Diseño Nativo para la Nube

APISIX fue diseñado desde cero para escenarios cloud-native, soportando múltiples formas de despliegue: Docker, Kubernetes, Helm Chart, entre otros. También proporciona un Ingress Controller que puede usarse directamente como puerta de entrada en Kubernetes.

### Ecosistema de Plugins

APISIX cuenta con un extenso sistema de plugins que cubre autenticación, limitación de velocidad, reescritura de solicitudes, procesamiento de respuestas, logs de monitoreo y más. Los usuarios pueden ampliar las funcionalidades del gateway fácilmente a través de este mecanismo.

### Soporte Multi-Protocolo

Además de los protocolos tradicionales HTTP/HTTPS, APISIX también soporta gRPC, Dubbo, WebSocket, MQTT y muchos otros protocolos más,satisfaciendo las necesidades de diferentes escenarios en arquitecturas de microservicios.

## Arquitectura

La arquitectura de APISIX se divide en dos partes: el plano de datos y el plano de control.

**Plano de datos** se encarga de procesar el reenvío real de solicitudes. El núcleo de APISIX, basado en Nginx, implementa un pipeline de alto rendimiento para el manejo de peticiones.

**Plano de control** gestiona todas las configuraciones a través de una API RESTful, soportando múltiples opciones de persistencia como etcd, Consul, etc. APISIX también ofrece un Dashboard que permite la configuración visual a través de una interfaz web.

## Cuándo Usar APISIX

- Como punto de entrada unificado para APIs de microservicios
- Como Ingress Controller en Kubernetes
- Para gestión y monitoreo del tráfico de APIs
- En entornos multi-tenant
- Como sidecar proxy en service mesh
- Como gateway para aplicaciones de IA (LLM y cargas de trabajo de IA)

Al ser un proyecto de nivel superior de la Apache Software Foundation, APISIX cuenta con una comunidad activa y actualizaciones continuas, lo que lo convierte en una opción confiable para aplicaciones empresariales.
