# Nginx インストールガイド

## 動作環境

Nginx は複数のオペレーティングシステムをサポートしています：

- Linux（Debian、Ubuntu、CentOS、RHEL、Alpine など）
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## インストール方法

### 方法1：パッケージマネージャーを使用（推奨）

#### Ubuntu/Debian

```bash
# パッケージインデックスを更新
sudo apt update

# Nginx をインストール
sudo apt install nginx

# Nginx を起動
sudo systemctl start nginx

# ブート時に開始
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# EPEL リポジトリをインストール
sudo yum install epel-release

# Nginx をインストール
sudo yum install nginx

# Nginx を起動
sudo systemctl start nginx

# ブート時に開始
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# Homebrew でインストール
brew install nginx

# Nginx を起動
brew services start nginx
```

### 方法2：ソースからビルド

#### ビルド依存関係をインストール

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### ソースコードをダウンロード

```bash
# 最新安定版をダウンロード
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# 解凍
tar -zxf nginx-1.26.0.tar.gz

# ディレクトリに入る
cd nginx-1.26.0
```

#### ビルドを設定

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### ビルドしてインストール

```bash
# ビルド
make

# インストール
sudo make install
```

### 方法3：Docker を使用

```bash
# 公式イメージをプル
docker pull nginx

# Nginx コンテナを実行
docker run -d -p 80:80 --name mynginx nginx

# 設定とログをマウント
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### 方法4：Windows インストール

1. https://nginx.org/en/download.html から Windows 版をダウンロード
2.  원하는 디렉토리에 압축 해제
3. nginx.exe をダブルクリックして開始
4. またはコマンドプロンプトから開始:

```cmd
cd C:\nginx
start nginx
```

## インストール後の確認

### インストールを確認

```bash
# Nginx バージョンを確認
nginx -v

# 詳細バージョン情報
nginx -V
```

### 実行状態を確認

```bash
# プロセスが実行中か確認
ps aux | grep nginx

# systemctl で状態を確認
systemctl status nginx
```

### アクセスをテスト

```bash
# curl でテスト
curl http://localhost

# またはブラウザで開く
http://localhost
```

## ディレクトリ構造

### Linux デフォルトディレクトリ

| パス | 説明 |
|------|------|
| /etc/nginx/ | 設定ディレクトリ |
| /etc/nginx/nginx.conf | メイン設定ファイル |
| /etc/nginx/conf.d/ | サブ設定ディレクトリ |
| /var/log/nginx/ | ログディレクトリ |
| /var/www/html/ | デフォルトドキュメントルート |
| /usr/sbin/nginx | 実行ファイル |

### ビルドインストールディレクトリ

| パス | 説明 |
|------|------|
| /usr/local/nginx/ | インストールルート |
| /usr/local/nginx/conf/ | 設定ディレクトリ |
| /usr/local/nginx/html/ | デフォルトドキュメントルート |
| /usr/local/nginx/logs/ | ログディレクトリ |
| /usr/local/nginx/sbin/ | 実行ファイルディレクトリ |

## Nginx のアンインストール

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### ビルドインストール

```bash
# Nginx を停止
sudo nginx -s stop

# インストールディレクトリを削除
sudo rm -rf /usr/local/nginx
```

## 一般的な問題

### ポート競合

起動が失敗した場合：

```bash
# ポート使用状況を確認
netstat -tulpn | grep :80

# Nginx ポートを変更
# /etc/nginx/nginx.conf を編集
# listen 80; を listen 8080; に変更
```

### 権限問題

権限エラーが発生した場合：

```bash
# Nginx ユーザーを確認
ps aux | grep nginx

# ファイルの権限を修正
sudo chown -R www-data:www-data /var/www/html/
```

### 設定ファイル構文チェック

```bash
# 設定ファイルの構文をテスト
nginx -t

# 特定の設定ファイルでテスト
nginx -t -c /path/to/nginx.conf
```
