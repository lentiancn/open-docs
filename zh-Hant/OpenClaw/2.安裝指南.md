# OpenClaw 安裝指南

本文檔詳細介紹在不同作業系統上安裝 OpenClaw 的步驟。

## 環境需求

- Node.js：22 或更高版本
- 作業系統：macOS、Linux、Windows
- 可選：Docker（如果你需要使用沙箱隔離功能）

## 安裝方法

### macOS 和 Linux（推薦）

最簡便的方式是使用官方安裝腳本：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

該腳本會自動檢測你的系統環境，完成依賴安裝和基礎配置。

### Windows 系統

在 PowerShell 中執行：

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 使用 npm 直接安裝

如果你已經配置好了 Node.js 環境：

```bash
npm install -g openclaw@latest
```

### 從源碼安裝

如果你想使用最新的開發版本：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 安裝後的初始配置

### 執行初始化精靈

安裝完成後，建議先執行初始化精靈來完成基本配置：

```bash
openclaw onboard --install-daemon
```

精靈會引導你完成以下設置：
- API 密鑰配置
- 服務連接埠設定
- 聊天渠道的登入（如 WhatsApp、Telegram 等）
- 安全相關的訪問控制配置

### 驗證服務狀態

如果是按照上面的步驟安裝的，服務應該已經在後台運行了：

```bash
openclaw gateway status
```

### 前台運行模式

如果你想在終端視窗中直接看到運行日誌：

```bash
openclaw gateway --port 18789
```

### 開啟網頁管理介面

```bash
openclaw dashboard
```

或者直接在瀏覽器中訪問：`http://127.0.0.1:18789/`

## 環境變數說明

- `OPENCLAW_HOME`：指定 OpenClaw 的主目錄
- `OPENCLAW_STATE_DIR`：指定狀態資料的存儲目錄
- `OPENCLAW_CONFIG_PATH`：指定配置文件的自定義路徑

## 驗證安裝結果

安裝完成後，你可以：

1. 執行 `openclaw --version` 查看版本號確認安裝成功
2. 執行 `openclaw doctor` 檢查系統健康狀態
3. 開啟網頁管理介面發送測試訊息

## 常見問題

### Node.js 版本檢查

```bash
node --version
```

確保顯示的版本號在 22 以上。

### 權限報錯

如果在安裝過程中遇到權限問題：

```bash
sudo npm install -g openclaw@latest
```
