# APISIX 安装指南

本文档介绍 Apache APISIX 的多种安装方式。

## Docker 安装（推荐）

使用 Docker 快速部署 APISIX 是最简单的方式：

```bash
# 拉取 APISIX 镜像
docker pull apache/apisix:latest

# 启动 APISIX
docker run -d -p 9080:9080 -p 9180:9180 -p 9443:9443 \
  --name apisix \
  apache/apisix:latest
```

### 使用 Docker Compose 安装

创建 `docker-compose.yml` 文件：

```yaml
version: '3'

services:
  apisix:
    image: apache/apisix:latest
    ports:
      - "9080:9080"
      - "9443:9443"
      - "9180:9180"
    environment:
      - APISIX_DASHBOARD_URL=http://apisix:9180
    volumes:
      - ./apisix_log:/var/log/apisix
    network_mode: host

  etcd:
    image: apache/apisix:etcd:latest
    ports:
      - "2379:2379"
    environment:
      - ETCD_HOST=127.0.0.1
    network_mode: host
```

启动服务：

```bash
docker-compose up -d
```

## 快速入门脚本安装

APISIX 官方提供了快速安装脚本，依赖 Docker 和 curl：

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

该脚本会自动：
1. 启动 etcd 容器
2. 启动 APISIX 容器
3. 配置网络连接

安装完成后输出：
```
✔ APISIX is ready!
```

验证安装：
```bash
curl "http://127.0.0.1:9080" --head | grep Server
# 输出: Server: APISIX/Version
```

## Helm Chart 安装（Kubernetes）

在 Kubernetes 环境中使用 Helm 安装：

```bash
# 添加 APISIX Helm 仓库
helm repo add apisix https://apache.github.io/apisix-helm-chart
helm repo update

# 安装 APISIX
helm install apisix apisix/apisix
```

### Ingress Controller 部署

```bash
helm install apisix-ingress apisix/apisix-ingress \
  --set ingressController.enabled=true
```

## 源码编译安装

### 环境要求

- Nginx ≥ 1.21.4
- OpenResty ≥ 1.21.4
- Lua ≥ 5.1
- etcd ≥ 3.4.0
- LuaRocks

### 编译步骤

```bash
# 克隆源码
git clone https://github.com/apache/apisix.git
cd apisix

# 安装依赖
make deps

# 启动 APISIX
make run
```

### 解决 LuaRocks 安装慢的问题

如果安装依赖时遇到超时，可以使用国内源：

```bash
# 使用 LuaRocks 国内源
make deps ENV_LUAROCKS_SERVER=https://luarocks.cn

# 或指定服务器
luarocks config rocks_servers https://luarocks.cn
```

## 验证安装

安装完成后，可以通过以下方式验证：

### 1. 检查服务状态

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

### 2. 访问 Dashboard

Dashboard 默认地址：http://127.0.0.1:9180/ui

默认用户名/密码：admin/admin

### 3. 使用 Admin API

```bash
# 获取路由列表
curl -s http://127.0.0.1:9180/apisix/admin/routes

# 创建示例路由
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

## 配置文件说明

APISIX 主要配置文件 `config.yaml`：

```yaml
apisix:
  node_listen: 9080
  enable_admin: true
  admin_key:
    - name: admin
      key: your-admin-key
      role: admin

etcd:
  host:
    - "http://127.0.0.1:2379"
  prefix: /apisix

deployment:
  admin:
    admin_key:
      - name: admin
        key: secret-key
        role: admin
```

## 多端口监听配置

默认 APISIX 仅监听 9080 端口处理 HTTP 请求。配置多端口：

```yaml
apisix:
  node_listen:
    - 9080
    - 9081
    - 9082
```

HTTPS 多端口：

```yaml
apisix:
  ssl:
    enable: true
    listen:
      - port: 9443
      - port: 9444
      - port: 9445
```

## 常见问题

### 1. 端口冲突

如果 9080 或 9443 端口被占用，修改 `config.yaml` 中的端口配置。

### 2. etcd 连接失败

确认 etcd 服务正常运行，检查 `etcd.host` 配置是否正确。

### 3. 性能问题

- 调整 worker 进程数：`nginx.worker_processes`
- 增加连接数：`nginx.worker_connections`

### 4. LuaRocks 安装超时

使用国内源：`make deps ENV_LUAROCKS_SERVER=https://luarocks.cn`
