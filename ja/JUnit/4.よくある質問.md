# JUnit よくある質問

## JUnit 4とJUnit 5のどちらを使用すべきですか？

新規プロジェクトにはJUnit 5を強く推奨します。ネストされたテスト、パラメータ化テスト、動的テストなど、より豊富な機能と優れた拡張性があります。既存のJUnit 4プロジェクトがあるユーザーは、JUnit Vintageを使用してJUnit 5プラットフォームでテストを実行し、段階的に移行できます。

## JUnit 5でJUnit 4テストを実行するには？

JUnit Vintageエンジン依存関係を追加します：

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## なぜテストが実行されないのですか？

一般的な原因：テストクラスに@Testアノテーションがない、テストメソッドがpublicでない、maven-surefire-pluginがない、またはuseJUnitPlatform()が設定されていない、Javaバージョンが要件を満たしていない。

## JUnit 5の最低Javaバージョンは？

JUnit 5はJava 8以上が必要です。JUnit 4はJava 5以上が必要です。

## 特定のテストをスキップするには？

@Disabledアノテーションを使用して、単一のテストまたはテストクラスをスキップできます：

```java
@Disabled("一時的に無効")
@Test
void testSkipped() {
    // 実行されない
}
```

## プライベートメソッドをテストするには？

ユニットテストはpublicメソッドのみをテストする必要があります。プライベートメソッドをテストする必要がある場合は、リファクタリングしてテストしたいロジックを別のpublicメソッドに移動するか、リフレクションを使用します（非推奨）。

## アサーション失敗時に詳細情報を表示するには？

3番目のパラメータでカスタムエラーメッセージを提供します：

```java
assertEquals(expected, actual, "計算結果不正");
```

## 例外をテストするには？

assertThrowsメソッドを使用します：

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("エラーメッセージ");
    });
}
```

## テストを並列実行するには？

JUnit 5で並列実行を設定：

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // テストは並列で実行される
}
```

## @Beforeと@BeforeEachの違いは？

@BeforeはJUnit 4のアノテーション、@BeforeEachはJUnit 5のアノテーションです。両者は同じ機能を持ち、各テストメソッドの前に実行されます。JUnit 5は@BeforeAllと@AfterAllもサポートしており、テストクラス全体で1回だけ実行される前処理と後処理に使用できます。

## パラメータ化テストを作成するには？

@ParameterizedTestアノテーションを@ValueSourceと組み合わせて使用：

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## テストがタイムアウトした場合は？

@Timeoutアノテーションでタイムアウトを設定：

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // 100ミリ秒を超えると失敗
}
```

## テストを繰り返すには？

@RepeatedTestアノテーションを使用：

```java
@RepeatedTest(10)
void testRepeated() {
    // 10回繰り返される
}
```

## JUnit 5の拡張モデルとは？

JUnit 5は@ExtendWithアノテーションを使用して拡張メカニズムを提供し、より柔軟なテストカスタマイズを実現：

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // カスタム拡張を使用
}
```
