# Hadoop ユーザーマニュアル

## HDFS 操作ガイド

### 基本的なファイルシステム操作

HDFSは従来のファイルシステムに類似したコマンドラインインターフェースを提供しており、ユーザーはhdfs dfsコマンドを通じてさまざまなファイル操作を行うことができます。これらのコマンドはLinuxのshellコマンドと非常に似ており、Linuxシステムに精通しているユーザーはすぐに始めることができます。これらの基本的な操作を理解することはHadoopを使用するための基礎です。以下では一般的なHDFS操作コマンドについて詳しく説明します。

ディレクトリ作成はmkdirコマンドを使用し、単層または複数層のディレクトリを作成できます：

```bash
# 単層ディレクトリを作成
hdfs dfs -mkdir /user/data

# 複数層ディレクトリを作成
hdfs dfs -mkdir -p /user/data/input

# ディレクトリリストを表示
hdfs dfs -ls /user/data

# ディレクトリリストを再帰的に表示
hdfs dfs -ls -R /user/data
```

ファイルのアップロードとダウンロードは一般的な操作です：

```bash
# ローカルファイルをHDFSにアップロード
hdfs dfs -put localfile.txt /user/data/

# アップロードして名前を変更
hdfs dfs -put localfile.txt /user/data/newname.txt

# HDFSファイルをローカルにダウンロード
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# HDFS内部ファイルをコピー
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# HDFS内部ファイルを移動
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

ファイルの表示と削除：

```bash
# ファイル内容を表示
hdfs dfs -cat /user/data/file.txt

# ファイルの最後の数行を表示
hdfs dfs -tail /user/data/file.txt

# ファイルサイズを表示
hdfs dfs -du /user/data/file.txt

# ファイルを削除
hdfs dfs -rm /user/data/file.txt

# ディレクトリを再帰的に削除
hdfs dfs -rm -r /user/data/dir
```

### 権限管理

HDFSの権限モデルはUnix/Linuxに類似しており、ユーザー、ユーザーが所属するグループ、および他のユーザーの3層システムに基づいています。各ファイルとディレクトリには所有者、所属グループ、他のユーザーに対応する読み取り（r）、書き込み（w）、実行（x）の3つの権限があります。権限管理を理解することはデータセキュリティを保護するために重要です。

```bash
# ファイルの所有者を変更
hdfs dfs -chown user:group /user/data/file.txt

# ファイルの権限を変更
hdfs dfs -chmod 755 /user/data/file.txt

# 所属グループを変更
hdfs dfs -chgrp groupname /user/data/file.txt
```

### データ複製とバランシング

HDFSはデフォルトで各データブロックを3回複製し、異なるDataNodeに保存します。dfs.replicationプロパティでデフォルトのレプリカ数を変更できます：

```bash
# 特定のファイルのレプリカ数を設定
hdfs dfs -setrep -w 2 /user/data/file.txt

# ファイルのブロック位置情報を表示
hdfs fsck /user/data/file.txt -files -blocks -locations
```

クラスターにノードが追加または削除されたとき、データが不均一になる可能性があります。この場合、rebalanceコマンドを使用してデータをバランシングできます：

```bash
start-balancer.sh -threshold 10
```

thresholdパラメータはノードのディスク使用率とクラスター平均の最大差分パーセンテージを表します。バランシングプロセスはネットワーク帯域幅を消費するため、低負荷時に実行することを推奨します。

### HDFS Web UI

HDFSは機能が豊富なWebインターフェースを提供しており、ブラウザからNameNodeの9870ポートでアクセスできます。Web UIではファイルシステムディレクトリの閲覧、ファイル内容の表示、DataNode状態の監視、ブロック情報の表示などができます。Web UIは運用担当者の日常的な監視とトラブルシューティングの重要なツールです。

Web UIの主な機能には以下が含まれます：HDFSディレクトリツリーの閲覧、ファイルリストとファイルプロパティの表示、DataNodeリストと各DataNodeのストレージ使用状況の表示、ブロックの分散とレプリケーション状態の表示、NameNodeの実行時情報とログの表示、ダウンロードや削除などのシンプルなファイル操作。

## MapReduce プログラミング

### WordCountの例

WordCountはMapReduceプログラミングの入門例で、テキスト内の各単語の出現回数をカウントします。この例を理解することはMapReduceプログラミングモデルをマスターするのに役立ちます。プログラムはMapフェーズとReduceフェーズに分かれており、Mapフェーズではテキストを単語に分割し、Reduceフェーズでは同じ単語のカウントを集約します。

Mapフェーズのコード例：

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduceフェーズのコード例：

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### ジョブの提交

MapReduceをJARファイルにパッケージ化した後、hadoop jarコマンドでジョブを提交します：

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

ジョブを提交すると、YARNのWeb UI（ポート8088）でジョブの進捗と結果を表示できます。ジョブが完了すると、ジョブID、タスク数、実行時間などのジョブ統計情報が出力されます。

### ジョブ構成

Configurationオブジェクトを通じてジョブ的各种パラメータを構成できます：

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

一般的に使用される構成パラメータには以下が含まれます：mapreduce.job.reducesでReduceタスク数を設定、mapreduce.map.memory.mbとmapreduce.reduce.memory.mbでコンテナメモリを設定、mapreduce.map.cpu.vcoresとmapreduce.reduce.cpu.vcoresでコンテナCPUコア数を設定。

## YARN 操作

### クラスターリソースの表示

YARNはクラスターリソースを管理するための豊富なコマンドラインツールを提供しています：

```bash
# 全ノード状態を表示
yarn node -list -all

# アプリケーションリストを表示
yarn application -list

# 特定のアプリケーション状態を表示
yarn application -status application_1234567890123_0001

# アプリケーションログを表示
yarn logs -applicationId application_1234567890123_0001
```

### キュー管理

YARNはキューを使用してリソース分配を管理します。Capacity SchedulerまたはFair Schedulerで構成できます：

```bash
# キュー情報を表示
yarn queue -status default

# キュー容量を表示
yarn resourcemanager -refreshQueues
```

### リソーススケジューラー構成

Capacity Schedulerはデフォルトのスケジューラーで、マルチテナントリソース共有をサポートします。yarn-site.xmlで構成できます：

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## パフォーマンス最適化

### MapReduceチューニング

MapReduceジョブのパフォーマンスチューニングにはいくつかの側面があります。まず、MapとReduceタスク数を合理的に設定します。Mapタスク数は通常入力データのスプリット数によって決定され、mapreduce.input.fileinputformat.split.maxsizeパラメータで調整できます。Reduceタスク数はクラスターのReduceスロット总数的0.9〜1.2倍にすることをお勧めします。

```bash
# Mapタスク数を設定
-D mapreduce.job.maps=10

# Reduceタスク数を設定
-D mapreduce.job.reduces=5
```

シャッフルフェーズの最適化は全体のパフォーマンスに大きな影響を与えます。以下のパラメータを調整できます：

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFSチューニング

HDFSのパフォーマンス最適化は主にデータ読み取り/書き込み効率に焦点を当てています。大きなファイルの場合、ブロックサイズを調整してデータローカリティと障害回復のバランスを取ることができます：

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

ショートサーキット読み取りを有効にすると、クライアントがローカルDataNodeから直接データを読み取り、ネットワーク転送を削減できます：

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARNチューニング

YARNのリソース分配はクラスターの利用率に大きな影響を与えます。NodeManagerのリソース容量を合理的に設定します：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

仮想CPUコア数を設定してマルチコアプロセッサをより効果的に活用します：

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## データのインポートとエクスポート

### データベースからのデータインポート

Sqoopを使用すると、リレーショナルデータベースとHDFS間のデータ移行が簡単になります：

```bash
# テーブル全体をインポート
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Hiveテーブルを作成してインポート
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### データベースへのデータエクスポート

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### データフォーマットの選択

異なるデータフォーマットはパフォーマンスに異なる影響を与えます。一般的に使用されるフォーマットには以下が含まれます：TextFile（汎用性が高いが、パフォーマンスは普通）、SequenceFile（MapReduceに適している、パフォーマンスが良い）、Parquet（列指向ストレージ、分析クエリに適している）、ORC（Hive最適化列指向ストレージ）。分析ワークロードには、ParquetやORCなどの列指向ストレージフォーマットを使用することをお勧めします。
