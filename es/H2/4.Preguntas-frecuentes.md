# Preguntas frecuentes sobre H2

## Problemas de instalación

### P: ¿Cómo descargar la base de datos H2?

R: Visita https://www.h2database.com/html/download.html para descargar la versión adecuada para tu plataforma. Hay dos opciones: instalador de Windows y paquete ZIP multiplataforma.

### P: ¿Qué versión de Java necesita H2?

R: H2 necesita Java 11 o superior.

### P: ¿Cómo usar H2 en un proyecto Maven?

R: Agrega la dependencia en pom.xml:
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### P: ¿Cómo ejecutar H2 después de descomprimir el ZIP?

R: Ejecuta en la línea de comandos:
```bash
java -jar h2.jar
```
Esto iniciaría la consola web de H2.

## Problemas de conexión

### P: ¿Cómo conectar a la base de datos H2?

R: Usa conexión JDBC:
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### P: ¿Cuál es el nombre de usuario y contraseña predeterminados de H2?

R: El nombre de usuario predeterminado es "sa", con contraseña vacía.

### P: ¿Cómo conectar a una base de datos de archivo?

R:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### P: ¿Cómo iniciar el servidor TCP de H2?

R:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### P: ¿Qué hacer si el puerto está ocupado?

R: Puedes especificar el puerto en la URL de conexión:
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## Problemas de datos

### P: ¿Los datos de la base de datos en memoria se pierden?

R: Sí, la base de datos en memoria (jdbc:h2:mem:) pierde los datos después de cerrar todas las conexiones. Para persistencia, usa el modo de base de datos de archivo.

### P: ¿Cómo ver la ubicación del archivo de base de datos?

R: Especifica la ruta en la URL de conexión:
```java
jdbc:h2:file:/ruta/a/base de datos
```

### P: ¿Qué hacer si el archivo de base de datos está dañado?

R: Intenta usar el modo de recuperación:
```java
jdbc:h2:file:./testdb;LOG=0
```
O restaura desde una copia de seguridad.

### P: ¿Cómo cifrar una base de datos?

R:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## Problemas de rendimiento

### P: ¿Cómo mejorar el rendimiento de inserción por lotes?

R: Usa operaciones por lotes y desactiva el auto-commit:
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### P: ¿La consulta es lenta怎么办?

R: 
1. Crea índices para condiciones de consulta comunes
2. Usa EXPLAIN para analizar el plan de consultas
3. Evita SELECT *, solo consulta las columnas necesarias

### P: ¿El uso de memoria es muy alto?

R: Ajusta el tamaño del heap de JVM y usa pools de conexión de manera razonable.

## Problemas de compatibilidad SQL

### P: ¿Qué modos de base de datos soporta H2?

R: H2 puede simular múltiples bases de datos:
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

Uso:
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### P: ¿Cómo manejar IDs de autoincremento?

R: H2 soporta múltiples formas:
```sql
-- Forma 1: IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- Forma 2: SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## Problemas de transacciones

### P: ¿Qué pasa si una transacción no se confirma?

R: Las transacciones no confirmadas se revierten automáticamente al cerrar la conexión.

### P: ¿Cómo manejar deadlocks?

R: H2 detecta automáticamente los deadlocks y revierte una de las transacciones. Se recomienda reducir el alcance de las transacciones para evitar bloqueos prolongados.

## Problemas de seguridad

### P: ¿Es segura la base de datos H2?

R: H2 es adecuada para entornos de desarrollo y pruebas. Para entornos de producción, se recomienda:
1. Usar base de datos cifrada
2. Usar contraseñas seguras
3. Restringir acceso a la red
4. Usar conexiones SSL

### P: ¿Cómo prevenir inyección SQL?

R: Usa PreparedStatement en lugar de concatenación de cadenas:
```java
// Correcto
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// Incorrecto
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## Problemas de herramientas

### P: ¿Cómo usar la consola H2?

R: Ejecuta `java -jar h2.jar -web` y luego accede a http://localhost:8082

### P: ¿Cómo usar H2 Shell?

R: Ejecuta `java -jar h2.jar -shell` o `java -jar h2.jar -tool`

### P: ¿Cómo ver la versión de H2?

R:
```bash
java -jar h2.jar -version
```
O en SQL: `SELECT H2VERSION();`

## Problemas de migración

### P: ¿Cómo migrar de MySQL a H2?

R:
1. Configura el modo de compatibilidad H2: `jdbc:h2:file:db;MODE=MySQL`
2. Exporta los datos de MySQL como script SQL
3. Ejecuta el script en H2

### P: ¿Qué tener en cuenta al actualizar la versión de H2?

R: Actualizar de H2 1.x a 2.x requiere consideraciones especiales, consulta la guía de migración oficial:
https://www.h2database.com/html/migration-to-v2.html
