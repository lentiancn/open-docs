# Часто задаваемые вопросы Element Plus

На этой странице перечислены распространенные проблемы и их решения при использовании Element Plus.

## Проблемы с установкой

### Ошибка установки

Симптомы: npm install или yarn add не удается

Решения:

1. Проверьте версию Node.js, она должна быть 14.0.0 или выше
2. Очистите кэш npm:
   npm cache clean --force
3. Удалите node_modules и package-lock.json, затем переустановите

### Ошибки типов TypeScript

Симптомы: Ошибки типов при импорте компонентов

Решения:

1. Убедитесь, что плагин Volar установлен
2. Добавьте в tsconfig.json:
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }

## Проблемы использования

### Стили компонентов не применяются

Симптомы: Стили компонентов не соответствуют ожиданиям

Решения:

1. Убедитесь, что CSS правильно импортирован:
   import 'element-plus/dist/index.css'
2. Проверьте наличие глобальных переопределений стилей

### Иконки не отображаются

Симптомы: Компоненты иконок отображаются пустыми или квадратами

Решения:

1. Установите библиотеку иконок:
   npm install @element-plus/icons-vue

2. Зарегистрируйте иконки глобально:
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }

### Импорт по требованию не работает

Симптомы: Размер пакета остается большим

Решения:

1. Проверьте правильность конфигурации плагина
2. Убедитесь, что используется правильный резолвер

### Интернационализация не работает

Симптомы: Текст компонента не изменяется после смены языка

Решения:

1. Правильно импортируйте языковой пакет:
   import ru from 'element-plus/es/locale/lang/ru'
2. Настройте локаль:
   app.use(ElementPlus, { locale: ru })

## Проблемы с компонентами

### Проблемы с таблицей

#### Сортировка таблицы не работает

Убедитесь, что установлен атрибут prop:

<el-table-column prop="date" sortable />

#### Фиксированные столбцы таблицы смещены

Убедитесь, что таблица имеет фиксированную высоту:

<el-table :data="data" height="300" />

### Проблемы с формой

#### Валидация формы не срабатывает

Убедитесь, что el-form-item имеет атрибут prop:

<el-form-item label="Имя" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>

### Проблемы с диалогом

#### Проблемы с прокруткой диалога

Когда содержимое диалога слишком длинное, убедитесь, что добавлена прокрутка:

<el-dialog>
  <div style="max-height: 60vh; overflow-y: auto;">
    Содержимое
  </div>
</el-dialog>

## Проблемы со стилями

### Конфликты стилей

Симптомы: Пользовательские стили переопределяются

Решения:

1. Используйте более конкретные селекторы
2. Используйте ::v-deep или :deep()

### Цвет темы не применяется

Проверьте, правильно ли переопределены CSS переменные:

:root {
  --el-color-primary: #409eff;
}

Убедитесь, что он загружен после стилей element-plus.

## Проблемы с производительностью

### Задержка страницы

При больших объемах данных:

1. Используйте el-table-v2 (виртуальная таблица)
2. Используйте пагинацию для загрузки данных
3. Избегайте сложных компонентов в таблицах

### Слишком большой размер пакета

1. Используйте импорт по требованию
2. Удалите неиспользуемые плагины

## Проблемы с версией Vue

### Несовместимость с Vue 2

Element Plus поддерживает только Vue 3. Если вам нужна Vue 2, используйте Element UI.

### Composition API не поддерживается

Убедитесь, что используется Vue 3.2 или выше для лучшей поддержки Composition API.
