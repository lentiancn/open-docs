# Руководство по установке Java

## Получение комплекта разработки Java

Oracle предоставляет бесплатные загрузки Java Development Kit (JDK). Основные текущие версии LTS включают JDK 25, JDK 21, JDK 17, JDK 11 и JDK 8.

Официальная страница загрузки: https://www.oracle.com/java/technologies/downloads/

## Установка в Windows

### Использование установщика

1. Посетите официальную страницу загрузки Oracle и выберите версию JDK для Windows
2. Загрузите установщик x64 (файл exe) или установщик MSI
3. Запустите загруженный установщик
4. Следуйте инструкциям мастера установки для завершения установки. Путь по умолчанию: C:\Program Files\Java\jdk-version
5. После установки проверьте её успешность

### Использование сжатого архива

1. Загрузите сжатый архив x64 (файл zip)
2. Распакуйте в указанный каталог, например C:\Java\jdk-25
3. Настройте переменные среды

### Настройка переменных среды

1. Щёлкните правой кнопкой мыши на «Этот компьютер» и выберите «Свойства»
2. Нажмите «Дополнительные параметры системы»
3. Нажмите «Переменные среды»
4. В системных переменных нажмите «Создать»
   - Имя переменной: JAVA_HOME
   - Значение переменной: Путь установки JDK (например, C:\Program Files\Java\jdk-25)
5. Найдите переменную Path и нажмите «Изменить»
6. Добавьте новую переменную: %JAVA_HOME%\bin
7. Нажмите ОК для сохранения

### Проверка установки

Откройте командную строку и введите:

```cmd
java -version
javac -version
```

Если отображается информация о версии, установка прошла успешно.

## Установка в Linux

### Установка из сжатого архива

1. Загрузите версию JDK для Linux (формат tar.gz)
2. Откройте терминал и распакуйте в указанный каталог:

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. Настройте переменные среды, отредактировав ~/.bashrc или /etc/profile:

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. Примените конфигурацию:

```bash
source ~/.bashrc
```

5. Проверьте установку:

```bash
java -version
```

### Установка с помощью менеджера пакетов

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## Установка в macOS

### Использование установщика DMG

1. Загрузите версию JDK для macOS (файл dmg)
2. Откройте загруженный файл dmg
3. Перетащите JDK в папку Applications
4. Откройте терминал и настройте переменные среды:

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. Добавьте вышеуказанное в ~/.zshrc или ~/.bash_profile
6. Проверьте установку:

```bash
java -version
```

### Использование Homebrew

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## Управление несколькими версиями JDK

### Использование update-alternatives в Linux

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Переключение версий в Windows

Измените переменную среды JAVA_HOME, чтобы она указывала на разные каталоги JDK.

### Использование SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Проверка среды Java

### Проверка версии Java

```bash
java -version
```

Пример вывода:

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### Проверка версии компилятора

```bash
javac -version
```

Пример вывода:

```
javac 25.0.1
```

### Проверка JAVA_HOME

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## Распространённые проблемы установки

### Проблема 1: команда java не найдена

Решение: Проверьте, содержит ли переменная среды Path каталог bin JDK.

### Проблема 2: недостаточно прав

Решение: Используйте sudo для повышения прав в Linux/macOS или проверьте права на файлы.

### Проблема 3: порт занят

Решение: Приложения Java используют порт 8080 по умолчанию. Убедитесь, что порт не используется другими программами.

### Проблема 4: разница между JDK и JRE

- JDK (Java Development Kit): Комплект разработки, включает JRE и инструменты разработки
- JRE (Java Runtime Environment): Среда выполнения, содержит только компоненты, необходимые для запуска программ Java

JDK требуется для разработки программ Java. JRE достаточно для запуска скомпилированных программ Java.

## Рекомендации после установки

1. Настройте MAVEN_HOME или GRADLE_HOME, если используете Maven или Gradle
2. Настройте M2_HOME для пути к локальному репозиторию Maven
3. Настройте HTTP-прокси при необходимости в корпоративных средах
4. Установите часовой пояс для правильного времени в приложениях Java

## Удаление Java

### Панель управления Windows

1. Откройте панель управления
2. Выберите «Программы и компоненты»
3. Найдите программы, связанные с Java
4. Щёлкните правой кнопкой мыши и выберите «Удалить»

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
