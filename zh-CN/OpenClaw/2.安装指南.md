# OpenClaw 安装指南

本指南详细介绍 OpenClaw 在不同操作系统上的安装步骤。

## 环境要求

- **Node.js**：22 或更高版本
- **操作系统**：macOS、Linux、Windows
- **可选**：Docker（如果需要沙箱功能）

## 安装方法

### macOS / Linux（推荐）

使用安装脚本是最简单的方式：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

该脚本会自动检测你的系统环境，安装必要的依赖，并配置好 OpenClaw。

### Windows（PowerShell）

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 使用 npm 直接安装

如果你已经安装了 Node.js，也可以直接使用 npm：

```bash
npm install -g openclaw@latest
```

### 从源码安装

如果你想使用最新开发版本或者想要自定义安装：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 安装后配置

### 运行入门向导

安装完成后，运行向导来配置认证、Gateway 设置和可选的通道：

```bash
openclaw onboard --install-daemon
```

向导会引导你完成：
- API 密钥配置
- Gateway 端口设置
- 通道连接（WhatsApp、Telegram、Discord 等）
- 安全和权限设置

### 检查 Gateway 状态

如果你是按照上面的命令安装了服务，Gateway 应该已经在运行：

```bash
openclaw gateway status
```

### 启动 Gateway（前台运行）

如果你想在终端前台运行 Gateway（适合测试或排查问题）：

```bash
openclaw gateway --port 18789
```

### 打开控制界面

```bash
openclaw dashboard
```

或者直接在浏览器中打开：`http://127.0.0.1:18789/`

## 环境变量

如果你需要自定义配置或状态目录位置，可以使用以下环境变量：

- `OPENCLAW_HOME`：设置用于内部路径解析的主目录
- `OPENCLAW_STATE_DIR`：覆盖状态目录
- `OPENCLAW_CONFIG_PATH`：覆盖配置文件路径

## 验证安装

安装完成后，你可以：

1. 运行 `openclaw --version` 确认版本
2. 运行 `openclaw doctor` 检查系统健康状况
3. 打开控制界面发送测试消息

## 常见安装问题

### Node.js 版本检查

```bash
node --version
```

确保版本号是 22 或更高。如果不是，请先升级 Node.js。

### Windows 上 Git 未找到

确保 Git 已安装并添加到系统 PATH 环境变量中。你可以从 [git-scm.com](https://git-scm.com) 下载安装。

### 权限问题

如果在安装过程中遇到权限错误，尝试：

```bash
sudo npm install -g openclaw@latest
```
