# Benutzerhandbuch für Node.js

## Erste Schritte

### Grundlegende Konzepte

Node.js basiert auf der V8 JavaScript-Engine von Chrome und bietet ein ereignisgesteuertes, nicht-blockierendes I/O-Modell, das es leicht und effizient für datenintensive Echtzeitanwendungen macht.

#### Ereignisschleife
Die Node.js-Ereignisschleife ist der Kern des Nebenläufigkeitsmodells. Im Gegensatz zu traditionellen threadbasierten Modellen verwendet Node.js eine einzelne Thread-Ereignisschleife, die asynchrone Operationen über Callbacks, Promises oder async/await behandelt.

#### Nicht-blockierende E/A
Die meisten Node.js-Operationen sind nicht-blockierend, was bedeutet, dass sie die Programmausführung nicht unterbrechen, während sie auf den Abschluss von E/A-Operationen warten. Stattdessen verwenden sie Callbacks, um Ergebnisse zu verarbeiten, wenn diese bereit sind.

### Hallo-Welt-Anwendung

Erstellen Sie Ihre erste Node.js-Anwendung mit diesem einfachen HTTP-Server:

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Führen Sie es mit aus: `node server.js`

## Kernmodule

Node.js wird mit einer umfangreichen Sammlung integrierter Module geliefert, die wesentliche Funktionalität ohne externe Abhängigkeiten bereitstellen.

### Dateisystem (fs)
Behandeln Sie Dateioperationen synchron oder asynchron:

```javascript
const fs = require('fs');

// Asynchrones Lesen
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Synchrones Lesen (blockiert die Ausführung)
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Datei schreiben
fs.writeFile('/path/to/output.txt', 'Hallo Node.js!', (err) => {
  if (err) throw err;
  console.log('Datei erfolgreich geschrieben');
});
```

### HTTP-Server
Erstellen Sie Webserver und führen Sie HTTP-Anfragen durch:

```javascript
const http = require('http');
const https = require('https');

// HTTP-Server erstellen
const server = http.createServer((req, res) => {
  // Anfrage behandeln
});

// HTTP-Anfrage durchführen
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });
});
req.end();
```

## Paketverwaltung mit npm

### Grundlegende Befehle

```bash

npm init

npm install express           # Zu Abhängigkeiten hinzufügen

npm run build

npm outdated
```

## Fehlerbehandlung

Eine ordnungsgemäße Fehlerbehandlung ist aufgrund der asynchronen Natur von Node.js-Anwendungen entscheidend.

### Callback-basierte Fehlerbehandlung
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Fehler beim Lesen der Datei:', err);
    return;
  }
  console.log('Dateiinhalt:', data);
});
```

### Promise-basierte Fehlerbehandlung
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('Dateiinhalt:', data);
  })
  .catch(err => {
    console.error('Fehler beim Lesen der Datei:', err);
  });
```

### Async/Await-Fehlerbehandlung
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('Dateiinhalt:', data);
  } catch (err) {
    console.error('Fehler beim Lesen der Datei:', err);
  }
}
```

## Leistungsoptimierung

### Clustering
Nutzen Sie mehrere CPU-Kerne mit dem Cluster-Modul:

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Master ${process.pid} läuft`);
  
  // Worker-Prozesse erstellen
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} ist beendet`);
  });
} else {
  // Worker teilen sich TCP-Verbindung
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`Worker ${process.pid} gestartet`);
}
```

### Caching
Implementieren Sie Caching-Strategien zur Verbesserung der Leistung:

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```

```javascript
if (!validator.isEmail(userInput)) {
```

```bash

```

### Ratenbegrenzung
Implementieren Sie Ratenbegrenzung, um Missbrauch zu verhindern:

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 Minuten
  max: 100 // Begrenzt jede IP auf 100 Anfragen pro windowMs
});

app.use(limiter);
```
