# OpenClaw Benutzerhandbuch

Dieses Handbuch deckt den täglichen Betrieb und die Funktionen von OpenClaw ab.

## Grundkonzepte

### Gateway

Gateway ist das Herzstück von OpenClaw. Das gesamte Nachrichtenrouting, die KI-Anfragen und die Kanalverwaltung laufen durch diesen einen Prozess. Du musst nur ein Gateway ausführen und es automatisch die Verbindungen zu all deinen Chat-Plattformen verwalten.

### Kanäle

Kanäle sind die Messaging-Apps, die du nutzt um mit deinem KI-Assistenten zu kommunizieren. OpenClaw unterstützt das gleichzeitige Verbinden mehrerer Kanäle, einschließlich WhatsApp, Telegram, Discord, iMessage und mehr.

### Agenten

Agenten sind unabhängige KI-Assistenten-Instanzen. Jeder Agent hat seinen eigenen Arbeitsbereich und Gesprächsverlauf. Du kannst mehrere Agenten für verschiedene Zwecke betreiben.

## Tägliche Operationen

### Starten und Stoppen

```bash
# Starten (läuft im Hintergrund)
openclaw gateway start

# Stoppen
openclaw gateway stop

# Neustarten
openclaw gateway restart

# Status prüfen
openclaw gateway status

# Im Vordergrund ausführen (zum Debuggen)
openclaw gateway --port 18789
```

### Nachrichten senden

```bash
# Direktnachricht senden
openclaw message send --target +15555550123 --message "Testnachricht"

# An Gruppe senden
openclaw message send --target "Gruppenname" --message "Hallo zusammen"
```

### Konfiguration verwalten

OpenClaw nutzt JSON5-Konfigurationsdateien unter `~/.openclaw/openclaw.json`.

Möglichkeiten zur Änderung:

1. **Assistenten-Modus**:
   ```bash
   openclaw onboard
   openclaw configure
   ```

2. **Befehlszeile**:
   ```bash
   openclaw config get agents.defaults.workspace
   openclaw config set agents.defaults.heartbeat.every "2h"
   openclaw config unset tools.web.search.apiKey
   ```

3. **Web-Interface**:
   Öffne http://127.0.0.1:18789/, gehe zum Reiter Config

4. **Direkte Bearbeitung**:
   ```bash
   nano ~/.openclaw/openclaw.json
   ```

### Heiße Neuladung

Die meisten Konfigurationsänderungen werden automatisch ohne Neustart wirksam.

## Häufige Konfigurationsbeispiele

### Zugriff einschränken

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

### Gruppen-Chat-Einstellungen

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

### Mehrere Agenten

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

### Geplante Aufgaben

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Periodische Check-ins

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Sandbox-Isolierung

Um KI-Operationen in isolierten Containern auszuführen:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

```json5
{
  hooks: {
    enabled: true,
    token: "dein-geheimer-token",
    path: "/hooks"
  }
}
```

## Kanalverwaltung

### Kanäle hinzufügen

```bash
# Bei WhatsApp anmelden
openclaw channels login whatsapp

# Bei Telegram anmelden
openclaw channels login telegram
```

### Verbundene Kanäle auflisten

```bash
openclaw channels list
```

## Fehlerbehebung

### Logs anzeigen

```bash
# Service-Logs
openclaw logs

# Logs in Echtzeit verfolgen
openclaw logs -f
```

### Gesundheitsprüfung

```bash
openclaw health
```

### Automatische Diagnose

```bash
openclaw doctor
```

## Web-Konsole

Du kannst OpenClaw auch über das Web-Interface verwalten:

- Nachrichten senden und empfangen
- Einstellungen ändern
- Gesprächsverlauf anzeigen
- Mobile Knoten verwalten
- Service-Status überwachen

Besuche: http://127.0.0.1:18789/
