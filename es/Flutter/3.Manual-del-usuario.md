# Manual de Usuario de Flutter

## Conceptos Básicos

### Introducción a Widgets

Widget es el bloque de construcción básico de la interfaz de usuario de Flutter. En Flutter, todo es un widget, incluyendo diseño, padding, align, etc. Los widgets describen cómo debe verse su vista dada la configuración y estado actual.

Los widgets de Flutter son inmutables. Cuando necesitas cambiar la interfaz de usuario, necesitas recrear el widget o usar StatefulWidget con gestión de estado. Los widgets se pueden anidar para formar un árbol de widgets, y Flutter compara eficientemente el árbol de widgets antiguo y nuevo, renderizando solo las partes que realmente cambiaron.

Los widgets comunes incluyen: Text (texto), Image (imagen), Icon (icono), Container, Row/Column (diseño horizontal/vertical), Stack (diseño apilado), ListView (lista), etc.

### StatelessWidget y StatefulWidget

StatelessWidget es un widget sin estado, adecuado para exhibiciones estáticas de interfaz de usuario como texto e imágenes. Definir un StatelessWidget solo requiere implementar el método build, devolviendo el árbol de widgets a renderizar.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget es un widget con estado, adecuado para escenarios que requieren actualizaciones dinámicas de interfaz de usuario como contadores y formularios. StatefulWidget necesita usarse con State, llamando al método setState para notificar a Flutter que vuelva a renderizar cuando el estado cambie.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## Fundamentos de Diseño

### Diseño Lineal

Row y Column son los widgets de diseño lineal más utilizados. Row organiza los widgets hijos horizontalmente, Column los organiza verticalmente. Usa MainAxisAlignment y CrossAxisAlignment para controlar la alineación.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### Diseño de Pila

Stack se usa para crear diseños apilados, colocando widgets encima de otro widget. Positioned se usa para posicionar hijos en un Stack.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### Diseño de Lista

ListView es el widget de lista más utilizado, adecuado para listas largas. ListView.builder es adecuado para grandes conjuntos de datos con carga diferida. ListView.separated puede agregar separadores.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## Gestión de Estado

### setState

setState es el método más básico de gestión de estado, adecuado para estado local simple. En StatefulWidget, llamar a setState dispara la reconstrucción del widget.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider es una de las soluciones de gestión de estado recomendadas por Flutter. A través de Provider, puedes compartir estado en el árbol de widgets para evitar pasarlo capa por capa.

```dart
// Definir ChangeNotifier
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Usar Provider
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## Solicitudes de Red

### Paquete HTTP

Usa el paquete http para hacer solicitudes HTTP simples:

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

## Almacenamiento Local

### SharedPreferences

Usado para almacenar datos simples de clave-valor:

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

## Navegación y Enrutamiento

Flutter usa Navigator para la navegación de páginas. Las operaciones básicas incluyen Navigator.push (empujar nueva página), Navigator.pop (regresar), Navigator.pushReplacement (reemplazar página actual), etc.

El enrutamiento por nombre es el método de gestión de enrutamiento recomendado:

```dart
// Definir rutas
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// Navegar
Navigator.pushNamed(context, '/detail');
```

## Construcción y Publicación

### Publicación Android

Usa keytool para generar clave de firma:

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

Configura la firma en android/app/build.gradle, luego ejecuta en la raíz del proyecto:

```bash
flutter build apk --release
```

### Publicación iOS

Configura certificado de firma y perfil de aprovisionamiento en Xcode, luego ejecuta:

```bash
flutter build ios --release
```

Ten en cuenta que la publicación de iOS requiere una cuenta de desarrollador de Apple y certificados correspondientes.
