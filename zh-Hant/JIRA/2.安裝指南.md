# 安裝指南

Jira 提供多種安裝方式，以滿足不同規模和技術能力團隊的需求。本指南將詳細介紹雲端版本和 Data Center 本地版本的安裝步驟。

## 雲端版本安裝

雲端版本是開始使用 Jira 最快捷的方式，無需管理任何基礎設施。

### 註冊帳號

首先訪問 Jira 官方網站，點擊免費取得按鈕。在註冊頁面填寫工作電子郵件地址，建立帳戶。Atlassian 會自動識別團隊成員並促進協作。

### 選擇方案

雲端版本提供四個定價層次：

**Free 方案** 永久免費，最多支援10名使用者。包含無限目標和專案、待辦事項列表和看板視圖、每月100次自動化規則、2GB儲存空間。

**Standard 方案** 每使用者每月7.91美元。包含使用者角色和權限、外部協作、多區域資料駐留、每月1700次自動化規則、250GB儲存空間。

**Premium 方案** 每使用者每月14.54美元。包含跨團隊規劃和依賴關係管理、可自訂審批流程、無限儲存、全天候支援、99.9%正常運行時間 SLA。

**Enterprise 方案** 面向大型企業。包含高階分析和企業級安全性、多個站點支援、無限自動化規則、99.95%正常運行時間 SLA。

### 建立站點

完成註冊後，系統會引導您建立第一個 Jira 站點。您可以為單位或個人建立站點，並選擇適合您團隊的專案模板。

### 新增使用者

建立站點後，可以透過管理員面板新增團隊成員。使用者可以是免費使用者或付費使用者，具體取決於您選擇的方案。

## Data Center 本地部署

對於需要嚴格資料控制或擁有特殊安全要求的企業，可以選擇 Data Center 本地部署版本。

### 系統需求

Data Center 版本需要滿足以下基本要求：

作業系統支援 Linux（Red Hat Enterprise Linux、Ubuntu Server、CentOS 等）、Windows Server 等主流伺服器作業系統。

資料庫支援 PostgreSQL、Oracle、Microsoft SQL Server 等企業級資料庫。

記憶體建議至少 8GB RAM，生產環境建議 16GB 或更多。

磁碟空間取決於資料量，建議至少 100GB 可用空間。

### 安裝步驟

首先從 Atlassian 官方下載 Data Center 安裝包。執行安裝程式，按照精靈選擇安裝目錄和資料庫連線設定。完成安裝後，透過瀏覽器存取管理介面進行初始設定。

### 資料遷移

如果您從舊版本遷移，需要使用 Jira 提供的匯入匯出工具進行資料遷移。建議在遷移前備份所有資料，並在測試環境中驗證遷移結果的完整性。

## 行動端應用

Jira 提供 iOS 和 Android 行動應用，讓您可以隨時隨地查看和更新任務。行動應用支援任務查看、評論建立、狀態更新等功能，適合需要頻繁行動辦公的團隊成員使用。

## 初始設定

無論選擇哪種部署方式，初始設定都包括以下步驟：

設定專案結構，建立適合團隊的專案和面板。設定工作流，定義任務狀態和轉換規則。新增團隊成員並設定權限。整合其他工具，如 Slack、GitHub 等。建立自動化規則，減少手動操作。
