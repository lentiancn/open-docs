# Preguntas Frecuentes sobre Nacos

## Preguntas de Inicio Rápido

### ¿Qué sistemas operativos soporta Nacos?

Nacos soporta sistemas operativos Linux, Unix, Mac y Windows. Se recomienda Linux/Unix/Mac para entornos de producción.

### ¿Qué versión de Java requiere Nacos?

Nacos requiere JDK 17 o superior. Asegúrate de que JAVA_HOME esté correctamente configurado.

### ¿Qué debo hacer si Nacos no inicia?

1. Verificar entorno Java: Ejecutar `java -version` para confirmar la versión correcta del JDK
2. Verificar uso de puertos: Confirmar que los puertos 8848 y 9848 no estén ocupados
3. Verificar logs: Revisar mensajes de error en logs/start.out o logs/nacos.log
4. Verificar configuración de autenticación: Nacos 3.0+ requiere parámetros de autenticación

### ¿Cómo cambiar el puerto predeterminado de Nacos?

Modificar la siguiente configuración en conf/application.properties:

```properties
server.port=8848
```

### ¿Cuáles son las credenciales predeterminadas de la consola Nacos?

- Usuario por defecto: nacos
- Contraseña por defecto: nacos@demo

Por favor cambia la contraseña predeterminada en entornos de producción.

## Preguntas de Descubrimiento de Servicios

### Después del registro exitoso del servicio, ¿por qué el cliente no puede descubrir el servicio?

1. Verificar estado de salud del servicio: Asegurar que el estado de la instancia sea saludable
2. Verificar conectividad de red: Asegurar que el cliente pueda acceder al servidor Nacos
3. Verificar nombre del servicio: Asegurar que el nombre del servicio consultado coincida con el registrado (incluyendo espacio de nombres y grupo)
4. Verificar configuración del cliente: Confirmar que la configuración de nacos.server-addr sea correcta

### ¿Cómo implementar balanceo de carga?

Nacos proporciona múltiples estrategias de balanceo de carga:

1. Balanceo de carga del lado del cliente: Usar el método selectOneHealthyInstance del SDK de Nacos
2. Balanceo de carga ponderado: Implementar configurando pesos de instancia
3. Balanceo de carga de clúster: Implementado automáticamente en entornos de clúster

### Después de que una instancia de servicio se desconecta, ¿por qué el cliente todavía llama a la instancia desconectada?

1. Verificar configuración de verificación de salud: Asegurar que el mecanismo de verificación de salud funcione correctamente
2. Verificar caché del cliente: Los clientes almacenan en caché listas de instancias de servicio con algo de retraso
3. Ajustar intervalo de sincronización: Modificar el intervalo de actualización del cliente

## Preguntas de Gestión de Configuración

### Después de publicar la configuración, ¿por qué el cliente no recibe notificaciones de actualización?

1. Verificar listener: Asegurar que el cliente haya agregado correctamente un listener de configuración
2. Verificar red: Asegurar que la conectividad de red sea normal
3. Verificar logs: Revisar mensajes de error en logs del cliente
4. Confirmar dataId y group: Asegurar que coincidan exactamente

### ¿Cómo lograr gestión de configuración multi-entorno?

1. Usar Namespace: Diferentes entornos usan diferentes espacios de nombres para aislamiento completo
2. Usar Group: Usar diferentes grupos bajo el mismo espacio de nombres para gestionar configuraciones
3. Usar DataId: Usar diferentes dataIds para distinguir configuraciones de diferentes entornos

### ¿Cómo ver versiones históricas de configuración?

La consola Nacos proporciona historial de cambios de configuración, permitiendo ver:

- Registros de modificación de configuración
- Comparación de contenido de cada modificación
- Reversión con un clic a versiones históricas

### ¿Cómo implementar lanzamiento canario de configuración?

Nacos soporta lanzamiento canario de configuración:

1. Crear configuración canaria en la consola
2. Especificar alcance del cliente canario
3. Verificar efecto de configuración canaria
4. Publicación completa o reversión

## Preguntas de Implementación en Clúster

### ¿Cuántos nodos se requieren para implementación en clúster de Nacos?

Se recomiendan al menos 3 nodos para entornos de producción para garantizar alta disponibilidad.

### ¿Cómo elegir almacenamiento de datos en modo clúster?

- Entorno de Prueba/Desarrollo: Puede usar base de datos incorporada Derby
- Entorno de Producción: Debe usar base de datos externa (MySQL) para garantizar confiabilidad de datos

### ¿Cómo garantiza el clúster la consistencia de datos?

Nacos usa el protocolo Distro desarrollado internamente y el protocolo Raft para garantizar consistencia de datos del clúster:

- Protocolo Distro: Responsable de sincronización de datos
- Protocolo Raft: Responsable de elección de líder y replicación de logs

### ¿Qué sucede cuando falla un nodo del clúster?

El clúster Nacos tiene capacidades de conmutación automática por error:

1. Las verificaciones de salud detectan automáticamente nodos fallidos
2. El tráfico se cambia automáticamente a nodos saludables
3. Los nodos fallidos sincronizan datos automáticamente después de la recuperación

## Preguntas de Autenticación y Seguridad

### ¿Cómo configurar autenticación en Nacos 3.0?

A partir de Nacos 3.0, la autenticación está habilitada por defecto. Los siguientes parámetros deben configurarse al iniciar:

```bash
# Clave secreta del token
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# Clave de identidad del servicio
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### ¿Cómo deshabilitar la autenticación de Nacos?

Modificar conf/application.properties:

```properties
nacos.core.auth.enabled=false
```

Nota: No se recomienda deshabilitar la autenticación en entornos de producción.

### ¿Cómo asignar permisos a diferentes usuarios?

Nacos soporta control de acceso basado en roles:

1. Crear usuarios en la consola
2. Asignar roles a usuarios
3. Configurar permisos de rol

## Preguntas sobre MCP

### ¿Qué debo hacer si ocurren problemas de compatibilidad al iniciar el Servidor MCP?

Asegurar que los datos del Servidor MCP registrado automáticamente sean compatibles con la versión correspondiente en Nacos:

- ¿El protocolo del Servidor MCP (stdio, sse, streamable) es consistente?
- ¿La referencia del servicio es consistente?
- ¿Los datos de Tools son consistentes (nombre de Tools, cantidad de Tools, tipos de parámetros, parámetros opcionales)?

Nota: Las descripciones de Tools y las descripciones de parámetros no necesitan ser consistentes.

### ¿Por qué no se puede encontrar el Servidor MCP después del registro?

1. Verificar estado del Servidor MCP: Asegurar que el servicio esté correctamente registrado
2. Verificar espacio de nombres: Asegurar que el cliente y el servicio estén en el mismo espacio de nombres
3. Verificar información de autenticación: Asegurar que usuario y contraseña sean correctos

### ¿Cómo modificar dinámicamente la configuración de herramientas del Servidor MCP?

Después de que el Servidor MCP se registra automáticamente en Nacos, puedes cambiar dinámicamente las descripciones y cambios de Tools del Servidor MCP en la consola. Las modificaciones relacionadas se aplicarán dinámicamente al Servidor MCP en tiempo de ejecución sin reiniciar el servicio.

## Preguntas de Rendimiento y Optimización

### ¿Cómo maneja Nacos el registro de servicios a gran escala?

Nacos nació de la escala de millones de instancias de Alibaba y tiene las siguientes capacidades:

- Registro y descubrimiento de servicios de alto rendimiento
- Respuesta de actualización de configuración de baja latencia
- Gestión de servicios y configuración a escala masiva

### ¿Cómo optimizar el rendimiento de Nacos?

1. Establecer intervalos de verificación de salud apropiados
2. Usar configuración de pool de conexión de base de datos apropiada
3. Habilitar almacenamiento en caché del lado del cliente para reducir frecuencia de solicitudes
4. Planificar espacios de nombres y grupos razonablemente

### ¿Recomendaciones de configuración de memoria para Nacos?

Basado en escala de servicios:

- Escala pequeña (servicios < 1000): 2C4G
- Escala media (servicios 1000-5000): 4C8G
- Escala grande (servicios > 5000): 8C16G+

## Preguntas de Integración del Ecosistema

### ¿Cómo integrar Nacos con Spring Cloud?

Agregar dependencias en pom.xml:

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### ¿Cómo integrar Nacos con Kubernetes?

Nacos soporta implementación en entornos Kubernetes usando:

- Implementación con Helm Chart
- Implementación con Operator
- Implementación con YAML personalizado

### ¿Para qué idiomas Nacos soporta SDKs?

Nacos proporciona SDKs para los siguientes idiomas:

- Java (oficial)
- Python
- Go
- Node.js
- C/C++
- Ruby
- PHP

## Solución de Problemas

### ¿Cómo solucionar fallas de registro de servicios?

1. Verificar si Nacos Server está funcionando correctamente
2. Verificar conectividad de red
3. Revisar logs del cliente
4. Confirmar que los parámetros de registro sean correctos

### ¿Cómo solucionar problemas de recuperación de configuración?

1. Confirmar que la configuración se haya publicado exitosamente
2. Verificar si dataId y group coinciden
3. Verificar si los listeners están correctamente agregados
4. Revisar logs del cliente para errores específicos

### ¿Dónde están los logs de Nacos?

Los archivos de logs de Nacos están ubicados en:

- Log principal: logs/nacos.log
- Log de inicio: logs/start.out
- Log de acceso: logs/access.log

## Actualización y Migración

### ¿Cómo actualizar desde Nacos 1.x a 2.x o 3.x?

1. Respaldar datos: Respaldar base de datos o archivos de instantáneas
2. Leer guía de actualización: Consultar documentación de actualización de la versión correspondiente
3. Actualización gradual: Probar en entorno de preparación primero
4. Migrar configuraciones: Prestar atención a cambios de autenticación y otras configuraciones

### ¿Cómo migrar datos de Nacos?

1. Exportar datos de configuración del entorno original
2. Importar a base de datos del nuevo entorno
3. Verificar integridad de datos
4. Cambiar tráfico al nuevo entorno
