# JUnit Часто задаваемые вопросы

## Какой JUnit использовать — 4 или 5?

Для новых проектов настоятельно рекомендуется JUnit 5. Он предоставляет более богатые функции, включая вложенные тесты, параметризованные тесты, динамические тесты и лучшую расширяемость. Для существующих проектов с JUnit 4 можно использовать JUnit Vintage для запуска тестов на платформе JUnit 5 с постепенной миграцией.

## Как запустить тесты JUnit 4 в JUnit 5?

Добавьте зависимость движка JUnit Vintage:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## Почему мои тесты не запускаются?

Распространённые причины: у тестового класса нет аннотации @Test, тестовые методы не являются public, отсутствует maven-surefire-plugin или не настроен useJUnitPlatform(), версия Java не соответствует требованиям.

## Какая минимальная версия Java для JUnit 5?

JUnit 5 требует Java 8 или выше. JUnit 4 требует Java 5 или выше.

## Как пропустить определённые тесты?

Используйте аннотацию @Disabled для пропуска отдельных тестов или целых классов:

```java
@Disabled("Временно отключён")
@Test
void testSkipped() {
    // Не будет выполнен
}
```

## Как тестировать приватные методы?

Модульные тесты должны тестировать только публичные методы. Если нужно протестировать приватные методы, рефакторите код, переместив логику в отдельные публичные методы, или используйте рефлексию (не рекомендуется).

## Как получить подробную информацию при ошибке проверки?

Используйте третий параметр для пользовательских сообщений об ошибках:

```java
assertEquals(expected, actual, "Результат вычисления неверен");
```

## Как тестировать исключения?

Используйте метод assertThrows:

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("сообщение об ошибке");
    });
}
```

## Как запускать тесты параллельно?

Настройте параллельное выполнение в JUnit 5:

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // Тесты будут выполняться параллельно
}
```

## В чём разница между @Before и @BeforeEach?

@Before — аннотация JUnit 4, @BeforeEach — аннотация JUnit 5. Обе выполняют одну и ту же функцию: выполняются перед каждым тестовым методом. JUnit 5 также поддерживает @BeforeAll и @AfterAll для операций настройки/завершения, которые выполняются один раз для всего тестового класса.

## Как создавать параметризованные тесты?

Используйте аннотацию @ParameterizedTest с @ValueSource:

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## Что делать, если тесты превышают время ожидания?

Используйте аннотацию @Timeout для установки тайм-аута:

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // Ошибка, если превысит 100 миллисекунд
}
```

## Как повторять тесты?

Используйте аннотацию @RepeatedTest:

```java
@RepeatedTest(10)
void testRepeated() {
    // Повторится 10 раз
}
```

## Что такое модель расширения JUnit 5?

JUnit 5 использует аннотацию @ExtendWith для предоставления механизма расширения, позволяющего более гибко настраивать тесты:

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // Использует пользовательское расширение
}
```
