# Spring Boot Häufig Gestellte Fragen

## Allgemeine Fragen

### Was ist Spring Boot?
Spring Boot ist ein Open-Source-Java-Framework, das die Enterprise-Anwendungsentwicklung durch Auto-Konfiguration, eingebettete Server und produktionsreife Funktionen vereinfacht.

### Warum sollte ich Spring Boot verwenden?
Spring Boot reduziert Boilerplate-Code, vereinfacht die Konfiguration und beschleunigt die Entwicklung. Es ermöglicht Ihnen, sich auf Geschäftslogik zu konzentrieren.

### Was ist der Unterschied zwischen Spring und Spring Boot?
| Aspekt | Spring | Spring Boot |
|--------|--------|--------------|
| Konfiguration | XML/Annotation-basiert | Auto-Konfiguration |
| Server | Extern (Tomcat, etc.) | Eingebettet |
| Abhängigkeiten | Manuelle Verwaltung | Starter POMs |

## Installationsfragen

### Wie erstelle ich ein Spring Boot Projekt?
1. Verwenden Sie Spring Initializr (https://start.spring.io)
2. Verwenden Sie eine IDE (IntelliJ, Eclipse) mit Spring Boot Plugin
3. Verwenden Sie Spring Boot CLI: `spring init --name myapp`

### Welche Java-Version erfordert Spring Boot?
- Spring Boot 2.x: Java 8-17
- Spring Boot 3.x: Java 17+

### Wie füge ich Abhängigkeiten hinzu?
Fügen Sie Starter zu Ihrer `pom.xml` hinzu:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

## Nutzungsfragen

### Wie erstelle ich einen REST-Controller?
```java
@RestController
@RequestMapping("/api")
public class HelloController {
    
    @GetMapping("/hello")
    public String hello() {
        return "Hello, World!";
    }
}
```

### Wie verbinde ich mich mit einer Datenbank?
```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: secret
```

### Wie aktiviere ich Sicherheit?
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

## Fehlerbehebung

### Anwendung startet nicht
- Überprüfen Sie fehlende Abhängigkeiten
- Überprüfen Sie Datenbankverbindungen
- Verifizieren Sie Port-Verfügbarkeit

### Bean nicht gefunden
- Stellen Sie sicher, dass Component Scanning aktiviert ist
- Überprüfen Sie, ob Bean im richtigen Paket ist
