# JUnit Preguntas frecuentes

## ¿Debo usar JUnit 4 o JUnit 5?

Para proyectos nuevos, se recomienda强烈mente JUnit 5. Proporciona funciones más ricas, incluyendo pruebas anidadas, pruebas parametrizadas, pruebas dinámicas y mejor extensibilidad. Para proyectos existentes con JUnit 4, puedes usar JUnit Vintage para ejecutar pruebas en la plataforma JUnit 5 y migrar gradualmente.

## ¿Cómo ejecutar pruebas JUnit 4 en JUnit 5?

Agrega la dependencia del motor JUnit Vintage:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## ¿Por qué mis pruebas no se ejecutan?

Razones comunes: clase de prueba sin anotación @Test, métodos de prueba no públicos, falta de maven-surefire-plugin o no configurado useJUnitPlatform(), versión de Java no cumple requisitos.

## ¿Cuál es la versión mínima de Java para JUnit 5?

JUnit 5 requiere Java 8 o superior. JUnit 4 requiere Java 5 o superior.

## ¿Cómo omitir ciertas pruebas?

Usa la anotación @Disabled para omitir pruebas individuales o clases enteras:

```java
@Disabled("Temporalmente deshabilitada")
@Test
void testSkipped() {
    // No se ejecutará
}
```

## ¿Cómo probar métodos privados?

Las pruebas unitarias solo deben probar métodos públicos. Si necesitas probar métodos privados, refactoriza el código para mover la lógica a probar a métodos públicos separados, o usa reflexión (no recomendado).

## ¿Cómo obtener información detallada cuando falla una aserción?

Proporciona mensajes de error personalizados usando el tercer parámetro:

```java
assertEquals(expected, actual, "El resultado del cálculo es incorrecto");
```

## ¿Cómo probar excepciones?

Usa el método assertThrows:

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("mensaje de error");
    });
}
```

## ¿Cómo ejecutar pruebas en paralelo?

Configura la ejecución paralela en JUnit 5:

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // Las pruebas se ejecutarán en paralelo
}
```

## ¿Cuál es la diferencia entre @Before y @BeforeEach?

@Before es una anotación de JUnit 4, @BeforeEach es de JUnit 5. Ambas cumplen el mismo propósito: ejecutarse antes de cada método de prueba. JUnit 5 también soporta @BeforeAll y @AfterAll para operaciones de configuración/limpieza que se ejecutan una vez para toda la clase de prueba.

## ¿Cómo crear pruebas parametrizadas?

Usa la anotación @ParameterizedTest con @ValueSource:

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## ¿Qué pasa si las pruebas expiran?

Usa la anotación @Timeout para establecer tiempo de espera:

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // Fallará si excede 100 milisegundos
}
```

## ¿Cómo repetir pruebas?

Usa la anotación @RepeatedTest:

```java
@RepeatedTest(10)
void testRepeated() {
    // Se repetirá 10 veces
}
```

## ¿Qué es el modelo de extensión de JUnit 5?

JUnit 5 usa la anotación @ExtendWith para proporcionar un mecanismo de extensión que permite una personalización más flexible de las pruebas:

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // Usa extensión personalizada
}
```
