# IBM Informix 자주 묻는 질문

## 일반 질문

### Q: IBM Informix와 HCL Informix의 차이점은 무엇인가요?

A: IBM Informix는 IBM에서 판매하고 마케팅하는 반면, HCL Informix는 HCLSoftware에서 개발하고 지원합니다. 둘 다 동일한 코드베이스를 공유하며 온프레미스 배포에 대해 기능적으로 동일합니다. IBM는 판매 및 라이선스를 담당하고 HCL은 개발 및 기술 지원을 담당합니다.

### Q: 개발에 어떤 에디션을 선택해야 하나요?

A: 개발 및 테스트에는 **Developer Edition** (무료) 또는 **Innovator-C Edition** (무료)을 사용하세요. 둘 다 리소스 제한이 있지만 전체 기능을 포함합니다:
- Developer Edition: 1 CPU VP/Core, 1GB RAM, 8GB 저장용량
- Innovator-C Edition: 2 CPU 코어, 8GB RAM, 50GB 저장용량

### Q: Informix는 IoT 애플리케이션에 적합한가요?

A: 네, Informix는 작은 공간(100MB까지), 낮은 관리 요구사항, 기본 제공 시계열 지원, Raspberry Pi를 포함한 엣지 장치에서 실행 능력으로 IoT에 적합합니다.

### Q: 어떤 프로그래밍 언어가 지원되나요?

A: Informix는 여러 프로그래밍 인터페이스를 지원합니다:
- SQL (기본)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- MongoDB 호환성 (JSON/BSON)

## 설치 문제

### Q: "권한 없음" 오류로 설치가 실패합니다

A: root/sudo 권한이 있는지 확인하세요. 설치 전에 informix 사용자와 그룹을 생성하세요:

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### Q: 설치 후 서버가 시작되지 않습니다

A: 다음을 확인하세요:
- 환경 변수가 올바르게 설정되었는지 (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- `onconfig` 및 `sqlhosts` 파일이 올바른지
- `$INFORMIXDIR/tmp/`의 오류 로그 검토
- 충분한 디스크 공간 및 권한 확인

### Q: Informix는 어떤 포트를 사용하나요?

A: 기본 포트는 9088입니다. `sqlhosts` 파일에서 확인할 수 있습니다. 원격 연결을 위해 방화벽이 이 포트를 허용하는지 확인하세요.

## 일반 운영 문제

### Q: Informix를 백업하고 복원하는 방법은?

A: `ontape` 또는 `onbar` 유틸리티를 사용하세요:

```bash
# ontape로 전체 백업
ontape -s -L 0

# ontape로 복원
ontape -r
```

### Q: 서버 상태를 확인하는 방법은?

A: `onstat` 명령을 사용하세요:

```bash
onstat -        # 일반 상태
onstat -d       # 디스크 공간 사용량
onstat -l       # 논리적 로그
onstat -u       # 활성 사용자
onstat -k       # 잠금 정보
```

### Q: 외부 애플리케이션에서 Informix에 연결하는 방법은?

A: ODBC, JDBC 또는 기본 클라이언트 드라이버를 구성하세요. 네트워크 연결을 위해 `sqlhosts` 파일을 업데이트하세요. JDBC 연결 문자열 예시:

```
jdbc:informix-sqli://호스트이름:9088/데이터베이스이름:INFORMIXSERVER=서버이름
```

### Q: "Logical Log Full" 오류의 원인은?

A: 모든 논리적 로그가 가득 차 백업되지 않았을 때 발생합니다. 해결 방법:
- 논리적 로그 백업: `ontape -s -L 0`
- 더 많은 논리적 로그 파일 추가: `onparams -a -d dbspace -s 크기`

### Q: 잠금 충돌을 처리하는 방법은?

A: `onstat -k`로 잠금을 확인하고 `onstat -u`로 차단 세션을 식별하세요. 해결 방법:
- 잠금 시간 초과 증가
- 더 낮은 격리 수준 사용 (dirty read)
- 차단 세션 종료: `onmode -z 세션ID`

## 성능 및 튜닝

### Q: 쿼리 성능을 향상시키는 방법은?

A:
- 자주 쿼리하는 열에 적절한 인덱스 생성
- `EXPLAIN`으로 쿼리 계획 분석
- 적절하게 공유 메모리 구성 (`SHMVIRTSIZE`, `BUFFERPOOL`)
- 정기적으로 통계 업데이트: `UPDATE STATISTICS FOR TABLE 테이블이름`

### Q: 주요 메모리 구성 매개변수는 무엇인가요?

A:
- `SHMVIRTSIZE` - 가상 공유 메모리 크기 (초기 할당)
- `SHMADD` - 추가 공유 메모리 세그먼트
- `BUFFERPOOL` - 다양한 페이지 크기에 대한 데이터 캐시 구성
