# GitLab 使用手冊

## 帳戶管理

### 註冊與登入

首次使用 GitLab，需要在登入頁面點擊「註冊」建立帳戶。註冊時需要提供使用者名稱、電子郵箱和密碼。使用者名稱的選擇需謹慎，建立後無法更改。完成註冊後，會收到一封驗證郵件，點擊郵件中的連結完成郵箱驗證。

登入後可以在個人設定中完善個人資料，包括頭像、姓名、全名和 bio 等資訊。建議填寫完整，因為這些資訊會在專案成員清單和提交記錄中顯示。還可以綁定 SSH 金鑰，用於透過命令列推送程式碼。

### SSH 金鑰設定

SSH 金鑰允許安全地與 GitLab 倉庫進行身份驗證，無需每次輸入使用者名稱和密碼。首先需要產生 SSH 金鑰對：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

產生的公鑰位於 ~/.ssh/id_ed25519.pub 檔案中。登入 GitLab，進入使用者設定頁面，點擊「SSH 金鑰」索引標籤，將公鑰內容貼上到文字方塊中並儲存。新增金鑰後，可以使用 SSH 方式複製倉庫：

```bash
git clone git@gitlab.example.com:username/project.git
```

## 專案操作

### 建立專案

登入後，點擊頂部導航列的「新建專案」按鈕建立新專案。可以選擇建立空白專案、從範本建立或匯入已有專案。建立時需要設定專案名稱、描述、可見性層級（公開、內部、私有）以及預設分支名稱。

專案名稱應簡潔明瞭，使用英文和短橫線命名。私有專案僅專案成員可以存取，內部專案對企業內所有登入使用者可見，公開專案對所有人可見。建立後可以進入專案設定頁面修改這些選項。

### 分支管理

GitLab 預設使用 main 作為主分支（早期版本為 master）。在專案頁面點擊「分支」可以查看和管理所有分支。建立新分支可以透過以下方式：在分支頁面點擊「新建分支」，或者使用 Git 命令：

```bash
git checkout -b feature/new-feature
```

保護分支是團隊協作的重要功能。進入專案設定 → 倉庫 → 分支保護，可以設定哪些使用者可以推送或合併到特定分支。建議保護 main 分支，只允許特定角色或維護者合併程式碼。

### Merge Request

Merge Request（合併請求）是 GitLab 程式碼協作的核心功能。完成功能開發並推送分支後，在專案頁面點擊「合併請求」→「新建合併請求」，選擇來源分支和目標分支，填寫標題和描述後建立。

合併請求頁面顯示了來源分支與目標分支的差異，包括檔案變更和提交歷史。團隊成員可以在這裡進行程式碼審查，逐行新增評論。審查完成後，如果滿足合併條件（如通過 CI 測試、獲得審批），則可以合併分支。

## CI/CD 流水線

### .gitlab-ci.yml 設定

GitLab CI/CD 使用專案根目錄下的 .gitlab-ci.yml 檔案定義流水線。這是一個 YAML 格式的設定檔，描述了各個建置階段和作業。以下是一個簡單的範例：

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/

test:
  stage: test
  script:
    - npm run test
  coverage: '/Coverage: \d+\.\d+%/'

deploy:
  stage: deploy
  script:
    - ./deploy.sh
  only:
    - main
```

這個設定定義了三個階段：build 建置專案，test 執行測試，deploy 部署應用。artifacts 用於在作業之間傳遞建置產物，only 指定只在 main 分支上執行部署作業。

### GitLab Runner

GitLab Runner 是執行 CI/CD 任務的代理程式。GitLab 官方提供共享 Runner，也可以自行註冊專有 Runner。在伺服器上安裝 GitLab Runner：

```bash
# 安裝
curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh | sudo bash
sudo apt-get install gitlab-runner

# 註冊 Runner
sudo gitlab-runner register
```

註冊時需要提供 GitLab 執行個體 URL 和註冊權杖，這些資訊可以在專案設定 → CI/CD → Runners 中找到。Runner 可以設定為平行執行作業，提高建置速度。

### 流水線監控

每次推送程式碼後，流水線會自動觸發。在專案的 CI/CD → 流水線頁面可以查看所有流水線執行狀態。點擊具體的流水線可以看到各個作業的執行情況，包括日誌輸出、建置時間和失敗原因。

GitLab 還提供流水線編輯器，採用視覺化方式建立和編輯 CI/CD 設定，無需手動編寫 YAML 檔案。編輯器提供語法檢查和即時預覽功能，降低了設定錯誤的機率。

## 問題追蹤

### 建立和管理問題

問題系統是 GitLab 專案管理的基礎。在專案頁面點擊「問題」→「新建問題」建立新問題。需要填寫標題和描述，可以選擇指派給特定成員、設定標籤、里程碑和截止日期。

問題狀態透過標籤或里程碑來管理，常用標籤包括 bug、feature、enhancement 等。可以透過看板視圖視覺化問題狀態，將問題按列分組（如待辦、處理中、已完成）。

### 問題與程式碼關聯

在提交資訊或合併請求描述中使用關鍵詞加問題編號，可以自動關聯程式碼與問題。例如：

- Fixes #123 - 修復問題 123，合併後會自動關閉問題
- Closes #456 - 關閉問題 456
- See #789 - 參考問題 789

這種關聯機制幫助團隊追蹤工作進度，在程式碼審查時了解問題的背景和目的。

## 維基與檔案

每個 GitLab 專案都有內建的維基功能，用於建立專案檔案。維基頁面使用 Markdown 格式編寫，支援程式碼醒目提示、表格和圖片。維基內容與程式碼倉庫一樣支援版本控制，可以查看歷史版本和回滾。

建議為專案維護以下檔案：README 專案簡介、CONTRIBUTING 貢獻指南、部署指南和 API 檔案。這些檔案可以顯著降低專案上手難度，促進社群貢獻。
