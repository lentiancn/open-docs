# MultiTalk ユーザーマニュアル

この手册では、MultiTalkの使用方法について詳しく説明します。基本操作、パラメータ設定、高度なテクニックが含まれています。

## 基本的な使用方法

### 入力ファイルの準備

MultiTalkは特定の入力形式を準備する必要があります。プロジェクトはJSONファイルを使用して生成パラメータを定義します。これにはオーディオパス、画像パス、テキストプロンプトが含まれます。

単一人物生成の入力JSON形式では、オーディオファイルパス、参照画像パス、ビデオ保存名などのパラメータを指定する必要があります。サンプルファイルはexamplesディレクトリのsingle_example_1.jsonにあります。

多人対話生成の入力JSONには、複数のオーディオストリームと対応する人物領域定義を含める必要があります。サンプルファイルはexamplesディレクトリのmultitalk_example_2.jsonにあります。

### 基本的な生成の実行

インストールと設定が完了したら、プロジェクトディレクトリに入り、以下のコマンドを実行してビデオ生成を行います：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### パラメータの説明

--ckpt_dirはモデルウェイトディレクトリパスを指定します。--wav2vec_dirはオーディオエンコーダーディレクトリパスを指定します。--input_jsonは入力パラメータJSONファイルパスを指定します。--sample_stepsはサンプリングステップ数を指定します。値が大きいほど生成品質は向上しますが、速度は遅くなります。--modeは生成モードを指定し、streamingは長動画モード、clipは短い動画モードです。--use_teacacheはTeaCache加速を有効にするかどうかを指定します。--save_fileは出力ファイルの保存名を指定します。

## 高度な使用テクニック

### 生成品質の調整

サンプリングステップは生成品質に影響を与える重要なパラメータです。デフォルトの40ステップで良好な結果が得られます。より速く生成するには、ステップ数を10にまで減らすことができます。これは口唇同期への影響は小さいですが、動きの滑らかさと視覚品質に影響します。ステップ数を増やすとディテールが向上します。

口唇同期の強度はオーディオガイダンス係数によって制御されます。Audio CFG値は3〜5の間で最適に動作します。この値を増やすとより良い口唇同期効果が得られます。

### 長尺動画生成

MultiTalkは最大15秒の動画を生成するをサポートしています。streamingモードを使用して長い動画クリップを生成します：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

非常に長い動画を生成する場合、異なるセグメント間に色調のDifferenceがある場合があります。Audio CFGを3に設定することでこの色調の問題を軽減できます。

### 低VRAM動作

VRAMが限られている場合は、以下のパラメータを使用してメモリ使用量を削減できます：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

--num_persistent_param_in_dit 0パラメータを追加すると、単一のRTX 4090で480P動画を生成できます。

### 加速の有効化

プロジェクトは生成速度を向上させるための複数の加速方法をサポートしています。

TeaCache加速は、品質を維持しながら速度を大幅に向上させることができます。--teacache_threshの推奨値は0.2〜0.5の間です：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### LoRA加速の使用

LoRAを使用すると、サンプリングステップを大幅に削減できます。FusioniX LoRAは8ステップ、lightx2v LoRAは4ステップのみ必要です：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Gradioインターフェースの使い方

プロジェクトにはインタラクティブな使用のためのGradio Webインターフェースを提供しています。プロジェクトディレクトリに入り、以下を実行します：

```bash
python app.py
```

または低VRAMバージョンを使用します：

```bash
python app.py --num_persistent_param_in_dit 0
```

## TTS音声合成の統合

MultiTalkは直接テキスト入力をサポートしています。統合されたTTSテクノロジーを通じて、テキストから音声を生成し、その後動画を生成します：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## マルチGPU設定

720P高解像度動画の生成には複数のGPUが必要です。以下のコマンドを使用してマルチGPUを設定します：

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## ComfyUIの使用

プロジェクトはComfyUIワークフローへの統合もサポートしています。ComfyUI-WanVideoWrapper拡張機能をインストールし、MultiTalkモデルを読み込んでワークフローオーケストレーショを行います。
