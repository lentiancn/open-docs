# Manuel d'utilisation

Ce manuel fournit des instructions detaillees sur l'utilisation de MultiTalk, y compris les operations de base, la configuration des parametres et les techniques avancees.

## Utilisation de base

### Preparation des fichiers d'entree

MultiTalk necessite la preparation de formats d'entree specifiques. Le projet utilise des fichiers JSON pour definir les parametres de generation, y compris les chemins audio, les chemins d'images et les invites de texte.

Pour la generation d'une seule personne, le format JSON d'entree necessite de specifier le chemin du fichier audio, le chemin de l'image de reference, le nom de sauvegarde de la video et d'autres parametres. Des fichiers d'exemple se trouvent dans le repertoire examples, comme single_example_1.json.

Pour la generation conversationnelle multipersonnes, le JSON d'entree doit contenir plusieurs flux audio et les definitions de region de personne correspondantes. Des fichiers d'exemple se trouvent dans le repertoire examples, comme multitalk_example_2.json.

### Execution de la generation de base

Apres l'installation et la configuration, entrez dans le repertoire du projet et executez la commande suivante pour la generation video:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### Description des parametres

--ckpt_dir specifie le chemin du repertoire des poids du modele. --wav2vec_dir specifie le chemin du repertoire de l'encodeur audio. --input_json specifie le chemin du fichier JSON des parametres d'entree. --specifie le nombre d'etapes d'echantillonnage; des valeurs plus elevees produisent une meilleure qualite mais une vitesse plus lente. --mode specifie le mode de generation, streaming pour les longues videos, clip pour les courtes videos. --use_teacache active l'acceleration TeaCache. --save_file specifie le nom de sauvegarde du fichier de sortie.

## Techniques d'utilisation avancees

### Ajustement de la qualite de generation

Les etapes d'echantillonnage sont des parametres cles affectant la qualite de generation. Les 40 etapes par defaut offrent de bons resultats. Pour une generation plus rapide, les etapes peuvent etre reduites a 10, ce qui a un impact minimal sur la synchronisation des levres mais affecte la fluidite du mouvement et la qualite visuelle. Augmenter les etapes ameliore la representation des details.

La force de synchronisation des levres est controlee par le coefficient de guidage audio. Les valeurs Audio CFG entre 3 et 5 fonctionnent le mieux. Augmenter cette valeur permet d'obtenir une meilleure synchronisation des levres.

### Generation de longues videos

MultiTalk supporte la generation de videos jusqu'a 15 secondes. Utilisez le mode streaming pour generer des clips video plus longs:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

Lors de la generation de tres longues videos, il peut y avoir des differences de tonalité entre les segments. Definir Audio CFG a 3 aide a maintenir la consistance de la tonalité.

### Fonctionnement a faible VRAM

Si le VRAM est limite, vous pouvez utiliser les parametres suivants pour reduire l'utilisation de la memoire:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

L'ajout de --num_persistent_param_in_dit 0 permet la generation de videos 480P sur un seul RTX 4090.

### Activation de l'acceleration

Le projet supporte plusieurs methodes d'acceleration pour ameliorer la vitesse de generation.

L'acceleration TeaCache peut ameliorer significativement la vitesse tout en maintenant la qualite. Les valeurs recommandees pour --teacache_thresh sont entre 0.2 et 0.5:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### Utilisation de l'acceleration LoRA

L'utilisation de LoRA peut reduire significativement les etapes d'echantillonnage. FusioniX LoRA ne necessite que 8 etapes, tandis que lightx2v LoRA ne necessite que 4 etapes:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Utilisation de l'interface Gradio

Le projet fournit une interface web Gradio pour une utilisation interactive. Entrez dans le repertoire du projet et executez:

```bash
python app.py
```

Ou utilisez la version a faible VRAM:

```bash
python app.py --num_persistent_param_in_dit 0
```

## Integration de la synthese vocale TTS

MultiTalk supporte l'entree de texte directe. Grace a la technologie TTS integree, il genere de la parole a partir du texte puis cree des videos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## Configuration multi-GPU

La generation de videos haute resolution 720P necessite plusieurs GPU. Utilisez la commande suivante pour configurer plusieurs GPU:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## Utilisation de ComfyUI

Le projet supporte egalement l'integration dans les flux de travail ComfyUI. Vous devez installer l'extension ComfyUI-WanVideoWrapper, puis charger le modele MultiTalk pour l'orchestration du flux de travail.
