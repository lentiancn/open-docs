# 安裝指南

本文檔介紹 Apache APISIX 的多種安裝方式，幫助您快速搭建 API 閘道器環境。

## 環境要求

在安裝 APISIX 之前，請確保滿足以下環境要求：

- **作業系統**：Linux（推薦 Ubuntu 20.04+、CentOS 7+、Debian 11+）
- **Docker**（用於 Docker 部署）：已安裝 Docker 和 Docker Compose
- **etcd**：APISIX 使用 etcd 作為配置存儲，需要 etcd 3.4.0+
- **Perl**：部分依賴需要 Perl 5.5.32+
- **curl**：用於驗證安裝

## Docker 快速安裝（推薦）

這是最快捷的安裝方式，適合開發和測試環境。

### 步驟一：建立 Quick Start 目錄

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### 步驟二：執行 Quick Start 腳本

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

該命令會自動啟動 apisix-quickstart 和 etcd 兩個容器。APISIX 使用 etcd 保存和同步配置，容器之間使用 Docker 的 host 網路模式，因此可以直接從本地存取。

### 步驟三：驗證安裝

安裝成功後，終端會輸出以下資訊：

```
✔ APISIX is ready!
```

可以通過以下命令驗證 APISIX 運行狀態：

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

正常情況下會返回：

```
Server: APISIX/Version
```

其中 Version 是已安裝的 APISIX 版本，例如 APISIX/3.15.0。

### 步驟四：存取 Dashboard

APISIX 提供內建的 Dashboard UI，可通過以下地址存取：

```
http://127.0.0.1:9180/ui
```

> ⚠️ 注意：為了提供更好的體驗，管理 API 預設無需授權，請在生產環境中打開授權開關。

## Docker Compose 安裝

如果需要自訂配置，可以使用 Docker Compose 方式部署。

### 步驟一：建立配置檔案

建立 docker-compose.yaml 檔案：

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### 步驟二：啟動服務

```bash
docker-compose up -d
```

### 步驟三：驗證服務

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Kubernetes 安裝

APISIX 提供了官方 Ingress Controller，可以直接在 Kubernetes 中部署。

### 使用 Helm Chart 安裝

```bash
# 添加 APISIX Helm 倉庫
helm repo add apisix https://charts.apiseven.com
helm repo update

# 安裝 APISIX
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### 使用 kubectl 安裝

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## 原始碼編譯安裝

適用於需要自訂建置的生產環境。

### 步驟一：安裝依賴

**Ubuntu/Debian：**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL：**

```bash
sudo yum install -y perl gcc git curl
```

### 步驟二：安裝 OpenResty

```bash
# 添加 OpenResty 倉庫
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# 安裝 OpenResty
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### 步驟三：安裝 APISIX

```bash
# 克隆原始碼
git clone https://github.com/apache/apisix.git
cd apisix

# 安裝依賴
make deps

# 初始化配置
make init

# 啟動 APISIX
make run
```

## 驗證安裝

無論採用哪種安裝方式，都可以通過以下方式驗證：

1. **檢查進程狀態**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **建立測試路由**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **測試路由**

```bash
curl "http://127.0.0.1:9080/test"
```

## 解除安裝

**Docker 方式：**

```bash
# 停止容器
docker-compose down

# 刪除資料卷（如需要）
docker-compose down -v
```

**Kubernetes 方式：**

```bash
helm uninstall apisix
```
