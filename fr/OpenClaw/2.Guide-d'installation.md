# Guide d'Installation d'OpenClaw

Ce guide fournit les étapes d'installation détaillées pour OpenClaw sur différents systèmes d'exploitation.

## Exigences

- **Node.js**: version 22 ou supérieure
- **Système d'exploitation**: macOS, Linux ou Windows
- **Optionnel**: Docker (pour la fonctionnalité sandbox)

## Méthodes d'Installation

### macOS / Linux (Recommandé)

Le moyen le plus simple d'installer OpenClaw est d'utiliser le script d'installation:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Ce script détecte automatiquement votre environnement système, installe les dépendances nécessaires et configure OpenClaw.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Installation Directe npm

Si vous avez déjà Node.js installé, vous pouvez également utiliser npm directement:

```bash
npm install -g openclaw@latest
```

### Installation depuis le Source

Si vous voulez la dernière version de développement ou avez besoin d'une installation personnalisée:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Configuration Post-Installation

### Exécuter l'Assistant d'Intégration

Après l'installation, exécutez l'assistant pour configurer l'authentification, les paramètres Gateway et les canaux optionnels:

```bash
openclaw onboard --install-daemon
```

L'assistant vous guidera à travers:
- Configuration de la clé API
- Paramètres du port Gateway
- Connexions de canaux (WhatsApp, Telegram, Discord, etc.)
- Paramètres de sécurité et permissions

### Vérifier l'État du Gateway

Si vous avez installé le service comme indiqué ci-dessus, le Gateway devrait déjà être en cours d'exécution:

```bash
openclaw gateway status
```

### Démarrer le Gateway au Premier Plan

Si vous voulez exécuter le Gateway au premier plan (utile pour les tests ou le dépannage):

```bash
openclaw gateway --port 18789
```

### Ouvrir l'Interface de Contrôle

```bash
openclaw dashboard
```

Ou simplement ouvrir dans votre navigateur: `http://127.0.0.1:18789/`

## Variables d'Environnement

Si vous devez personnaliser l'emplacement du config ou du répertoire d'état, vous pouvez utiliser ces variables d'environnement:

- `OPENCLAW_HOME`: définit le répertoire home utilisé pour la résolution de chemin interne
- `OPENCLAW_STATE_DIR`: écrase le répertoire d'état
- `OPENCLAW_CONFIG_PATH`: écrase le chemin du fichier de configuration

## Vérifier l'Installation

Après l'installation, vous pouvez:

1. Exécuter `openclaw --version` pour confirmer la version
2. Exécuter `openclaw doctor` pour vérifier la santé du système
3. Ouvrir l'Interface de Contrôle pour envoyer un message de test

## Problèmes Courants d'Installation

### Vérifier la Version Node.js

```bash
node --version
```

Assurez-vous que la version est 22 ou supérieure. Si ce n'est pas le cas, mettez d'abord à niveau Node.js.

### Git Non Trouvé sur Windows

Assurez-vous que Git est installé et ajouté à votre PATH système. Vous pouvez le télécharger depuis [git-scm.com](https://git-scm.com).

### Erreurs de Permissions

Si vous rencontrez des erreurs de permission pendant l'installation, essayez:

```bash
sudo npm install -g openclaw@latest
```
