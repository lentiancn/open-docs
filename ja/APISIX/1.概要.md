# 概要

Apache APISIX は、Apache Software Foundation のトッププロジェクトであり、API7.ai によって開発され寄贈されました。動的・リアルタイム・高性能を特徴とするクラウドネイティブAPIゲートウェイです。

APISIX はすべてのビジネスのトラフィック入口として機能し、動的ルート、動的アップストリーム、動的証明書、A/Bテスト、グレーディングリリース、カナリアリリース、青緑デプロイ、レート制限、セキュリティ保護、メトリクス収集、モニタリングアラート、可観測性、サービスガバナンスなど豊富な機能を提供します。

## コア機能

### 高性能

APISIX は Nginx をベースしており、Nginx の優れたパフォーマンスを継承しています。プラグイン機構により、ビジネスロジックとコア機能が分離されているため、拡張性が向上的同时に効率的な処理能力を維持しています。

### 完全動的

従来の Nginx は設定の再読み込みが必要ですが、APISIX は完全動的な設定管理をサポートしています。ルート、アップストリーム、プラグインなどすべての設定が管理APIでリアルタイムに更新でき、サービスを再起動せずに反映されます。

### クラウドネイティブ設計

APISIX は設計段階からクラウドネイティブシナリオを考慮しており、Docker、Kubernetes、Helm Chart など複数のデプロイメント方式をサポートしています。Ingress Controller も提供しており、Kubernetes のエントリゲートウェイとして直接使用できます。

### プラグインエコシステム

APISIX は豊富なプラグインシステムを備えています。認証、レート制限、リクエスト書き出し、レスポンス処理、モニタリングログなど、さまざまな機能をカバーしています。プラグイン機構により、ゲートウェイ機能を容易に拡張できます。

### マルチプロトコルサポート

従来の HTTP/HTTPS プロトコルに加えて、APISIX は gRPC、Dubbo、WebSocket、MQTT などの多様なプロトコルをサポートしており、マイクロサービスアーキテクチャのさまざまなシナリオニーズに応えることができます。

## アーキテクチャ概要

APISIX のアーキテクチャはデータプレーンとコントロールプレーンの2つに分かれています：

**データプレーン**は実際のリクエスト転送を担当し、Nginx をベースに開発された APISIX Core が高性能なリクエスト処理パイプラインを実現しています。

**コントロールプレーン**は RESTful API で設定を管理し、etcd、Consul など複数の設定永続化方式をサポートしています。APISIX は Dashboard も提供しており、Web UI で視覚的に設定を管理できます。

## ユースケース

- マイクロサービス API の統一エントリポイント
- Kubernetes Ingress Controller
- API トラフィック管理とモニタリング
- マルチテナント API 管理
- サービスメッシュサイドカープロキシ
- AI アプリケーションゲートウェイ（LLM と AI ワークロード）

Apache Software Foundation のトッププロジェクトとして、APISIX は活発なコミュニティと継続的なバージョン更新を維持しており、企業向けアプリケーションにとって信頼できる選択肢となっています。
