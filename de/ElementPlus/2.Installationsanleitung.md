# Element Plus Installationsanleitung

Diese Anleitung beschreibt, wie Sie Element Plus in verschiedenen Projekten installieren und verwenden.

## Umgebungsanforderungen

Element Plus erfordert:

- Vue 3.0.0 oder höher
- Node.js 14.0.0 oder höher
- Moderne Browser wie Chrome, Firefox, Safari und Edge

## Installation

### Paketmanager

Installieren Sie mit npm, yarn oder pnpm:

Mit npm:
npm install element-plus

Mit yarn:
yarn add element-plus

Mit pnpm:
pnpm add element-plus

## Vollständiger Import

Wenn Sie sich keine Sorgen um die Paketgröße machen, ist der vollständige Import bequemer.

In main.ts:

import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'

const app = createApp(App)

app.use(ElementPlus)
app.mount('#app')

### Volar-Unterstützung

Wenn Sie Volar verwenden, fügen Sie die globale Komponententypdefinition in compilerOptions.types in tsconfig.json hinzu:

{
  "compilerOptions": {
    "types": ["element-plus/global"]
  }
}

## Import bei Bedarf

Der Import bei Bedarf kann die Projektgröße erheblich reduzieren. Sie müssen zusätzliche Plugins verwenden, um die benötigten Komponenten zu importieren.

### Auto-Import (Empfohlen)

Zuerst die Plugins installieren:

Mit npm:
npm install -D unplugin-vue-components unplugin-auto-import

Mit yarn:
yarn add -D unplugin-vue-components unplugin-auto-import

Mit pnpm:
pnpm add -D unplugin-vue-components unplugin-auto-import

#### Vite-Konfiguration

In vite.config.ts:

import { defineConfig } from 'vite'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    AutoImport({ resolvers: [ElementPlusResolver()] }),
    Components({ resolvers: [ElementPlusResolver()] }),
  ],
})

#### Webpack-Konfiguration

In webpack.config.js:

const AutoImport = require('unplugin-auto-import/webpack')
const Components = require('unplugin-vue-components/webpack')
const { ElementPlusResolver } = require('unplugin-vue-components/resolvers')

module.exports = {
  plugins: [
    AutoImport({ resolvers: [ElementPlusResolver()] }),
    Components({ resolvers: [ElementPlusResolver()] }),
  ],
}

### Verwendung mit Nuxt

Für Nuxt-Benutzer müssen Sie nur @element-plus/nuxt installieren:

npm install -D @element-plus/nuxt

Dann die Konfiguration in nuxt.config.ts hinzufügen:

export default defineNuxtConfig({
  modules: ['@element-plus/nuxt'],
})

### Manueller Import

Element Plus bietet sofort einsatzbereite Tree-Shaking-Funktionen basierend auf ES Module.

Sie müssen unplugin-element-plus installieren, um Stile zu importieren:

npm install -D unplugin-element-plus

In Komponenten:

<template>
  <el-button>Schaltfläche</el-button>
</template>

<script>
import { ElButton } from 'element-plus'

export default {
  components: { ElButton },
}
</script>

Vite-Konfiguration:

import { defineConfig } from 'vite'
import ElementPlus from 'unplugin-element-plus/vite'

export default defineConfig({
  plugins: [ElementPlus()],
})

## Globale Konfiguration

Bei der Registrierung von Element Plus können Sie ein globales Konfigurationsobjekt mit size und zIndex übergeben:

import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import App from './App.vue'

const app = createApp(App)
app.use(ElementPlus, { size: 'small', zIndex: 3000 })

Konfigurationsoptionen:

- size: Standardgröße für Formularkomponenten festlegen. Mögliche Werte sind large, default und small
- zIndex: z-index für Popup-Komponenten festlegen. Der Standardwert ist 2000

### Konfiguration für Import bei Bedarf

Verwenden Sie die el-config-provider-Komponente:

<template>
  <el-config-provider :size="size" :z-index="zIndex">
    <app />
  </el-config-provider>
</template>

<script>
import { defineComponent } from 'vue'
import { ElConfigProvider } from 'element-plus'

export default defineComponent({
  components: { ElConfigProvider },
  setup() {
    return { zIndex: 3000, size: 'small' }
  },
})
</script>
