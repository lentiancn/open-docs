# Java 설치 가이드

## Java 개발 키트 받기

Oracle는 Java Development Kit(JDK)을 무료로 다운로드할 수 있게 제공합니다. 현재 주요 LTS 버전에는 JDK 25, JDK 21, JDK 17, JDK 11, JDK 8이 포함됩니다.

공식 다운로드 페이지: https://www.oracle.com/java/technologies/downloads/

## Windows에 설치

### 인스톨러 사용

1. Oracle 공식 다운로드 페이지를 방문하고 JDK의 Windows 버전 선택
2. x64 인스톨러(exe 파일) 또는 MSI 인스톨러 다운로드
3. 다운로드한 인스톨러 실행
4. 설치 마법사를 따라 설치 완료. 기본 경로는 C:\Program Files\Java\jdk-version
5. 설치 후 설치 성공 여부 확인

### 압축 아카이브 사용

1. x64 압축 아카이브(zip 파일) 다운로드
2. C:\Java\jdk-25과 같은 지정된 디렉토리에 압축 해제
3. 환경 변수 구성

### 환경 변수 구성

1. "내 PC"를 마우스 오른쪽 버튼으로 클릭하고 "속성" 선택
2. "고급 시스템 설정" 클릭
3. "환경 변수" 클릭
4. 시스템 변수 아래 "새로 만들기" 클릭
   - 변수 이름: JAVA_HOME
   - 변수 값: JDK 설치 경로(예: C:\Program Files\Java\jdk-25)
5. Path 변수를 찾아 "편집" 클릭
6. 새 변수 추가: %JAVA_HOME%\bin
7. 저장하려면 확인 클릭

### 설치 확인

명령 프롬프트 열기 입력:

```cmd
java -version
javac -version
```

버전 정보가 표시되면 설치 성공입니다.

## Linux에 설치

### 압축 아카이브에서 설치

1. Linux 버전 JDK(tar.gz 형식) 다운로드
2. 터미널을 열고 지정된 디렉토리에 압축 해제:

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. ~/.bashrc 또는 /etc/profile을 편집하여 환경 변수 구성:

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. 구성 적용:

```bash
source ~/.bashrc
```

5. 설치 확인:

```bash
java -version
```

### 패키지 관리자 사용

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## macOS에 설치

### DMG 인스톨러 사용

1. macOS 버전 JDK(dmg 파일) 다운로드
2. 다운로드한 dmg 파일 열기
3. JDK를 Applications 폴더로 드래그
4. 터미널을 열고 환경 변수 구성:

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. 위 내용을 ~/.zshrc 또는 ~/.bash_profile에 추가
6. 설치 확인:

```bash
java -version
```

### Homebrew 사용

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## 여러 JDK 버전 관리

### Linux에서 update-alternatives 사용

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Windows에서 버전 전환

JAVA_HOME 환경 변수를 다른 JDK 디렉토리를 가리키도록 수정합니다.

### SDKMAN 사용

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Java 환경 확인

### Java 버전 확인

```bash
java -version
```

예제 출력:

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### 컴파일러 버전 확인

```bash
javac -version
```

예제 출력:

```
javac 25.0.1
```

### JAVA_HOME 확인

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## 일반적인 설치 문제

### 문제 1: java 명령을 찾을 수 없음

해결 방법: Path 환경 변수에 JDK의 bin 디렉토리가 포함되어 있는지 확인합니다.

### 문제 2: 권한 거부

해결 방법: Linux/macOS에서 sudo로 권한 상승하거나 파일 권한을 확인합니다.

### 문제 3: 포트가 이미 사용 중

해결 방법: Java 애플리케이션은 기본적으로 포트 8080을 사용합니다. 다른 프로그램이 포트를 사용하지 않는지 확인합니다.

### 문제 4: JDK와 JRE의 차이

- JDK(Java Development Kit): 개발 도구 및 JRE 포함
- JRE(Java Runtime Environment): Java 프로그램 실행에 필요한 구성요소만 포함

Java 프로그램 개발에는 JDK가 필요합니다. 컴파일된 Java 프로그램 실행에는 JRE로 충분합니다.

## 설치 후 구성 권장사항

1. Maven 또는 Gradle을 사용하는 경우 MAVEN_HOME 또는 GRADLE_HOME 구성
2. Maven 로컬 저장소 경로에 M2_HOME 구성
3. 기업 환경에서 필요한 경우 HTTP 프록시 구성
4. Java 애플리케이션의 정확한 시간을 위해 시간대 설정

## Java 제거

### Windows 제어판

1. 제어판 열기
2. "프로그램 및 기능" 선택
3. Java 관련 프로그램 찾기
4. 마우스 오른쪽 버튼으로 클릭하고 "제거" 선택

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
