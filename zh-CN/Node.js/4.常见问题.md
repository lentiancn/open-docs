# Node.js 常见问题

## 安装相关问题

### Q: 如何检查 Node.js 是否正确安装？
**A:** 在终端中运行以下命令：
```bash
node --version
npm --version
```
如果显示版本号（如 v18.17.0），说明安装成功。还可以运行简单的测试：
```bash
node -e "console.log('Hello Node.js!')"
```

### Q: 安装后命令未找到怎么办？
**A:** 这通常是因为 PATH 环境变量未正确配置。

**Windows 解决方案:**
- 重新运行安装程序，确保勾选 "Add to PATH" 选项
- 或手动添加到系统环境变量：`C:\Program Files\nodejs\`

**macOS/Linux 解决方案:**
- 如果使用包管理器安装，通常会自动配置
- 手动添加到 shell 配置文件（~/.bashrc、~/.zshrc 等）：
  ```bash
  export PATH=$PATH:/usr/local/bin
  # 或者对于 Homebrew (Apple Silicon)
  export PATH=$PATH:/opt/homebrew/bin
  ```
- 重新加载配置：`source ~/.bashrc`

### Q: 如何在不同项目中使用不同版本的 Node.js？
**A:** 推荐使用版本管理工具：

**nvm (Node Version Manager) - macOS/Linux:**
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 安装特定版本
nvm install 18.17.0

# 切换版本
nvm use 18.17.0

# 设置默认版本
nvm alias default 18.17.0

# 在项目目录中创建 .nvmrc 文件
echo "18.17.0" > .nvmrc
nvm use  # 自动使用 .nvmrc 中指定的版本
```

**nvm-windows - Windows:**
```cmd
:: 安装特定版本
nvm install 18.17.0

:: 切换版本
nvm use 18.17.0

:: 查看已安装版本
nvm list
```

## npm 相关问题

### Q: npm install 太慢怎么办？
**A:** 可以配置使用国内镜像源：

```bash
# 临时使用淘宝镜像
npm install --registry https://registry.npmmirror.com

# 永久设置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 验证设置
npm config get registry

# 恢复官方源
npm config delete registry
```

**其他镜像源:**
- 官方源: `https://registry.npmjs.org`
- 淘宝镜像: `https://registry.npmmirror.com`
- 华为云: `https://mirrors.huaweicloud.com/repository/npm/`

### Q: 全局安装包时出现权限错误？
**A:** 这是常见的权限问题，有几种解决方案：

**推荐方案: 使用版本管理器**
- 使用 nvm 或 fnm 管理 Node.js，这样全局包会安装在用户目录下

**方案二: 配置 npm 全局目录**
```bash
# 创建全局目录
mkdir ~/.npm-global

# 配置 npm 使用新目录
npm config set prefix '~/.npm-global'

# 添加到 PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

**方案三: 使用 sudo（不推荐）**
```bash
sudo npm install -g package-name
```
注意：这可能存在安全风险，且可能导致权限混乱。

### Q: 如何更新全局安装的包？
**A:** 
```bash
# 查看过期的全局包
npm outdated -g

# 更新特定包
npm update -g package-name

# 批量更新所有全局包（需要额外工具）
npm install -g npm-check-updates
ncu -g
ncu -g -u  # 更新 package.json
```

### Q: package-lock.json 和 yarn.lock 有什么区别？
**A:** 
- **package-lock.json**: npm 生成的锁定文件，确保依赖版本一致性
- **yarn.lock**: Yarn 生成的锁定文件，功能类似但格式不同
- **重要**: 应该将锁定文件提交到版本控制，确保团队成员使用相同的依赖版本

## 开发相关问题

### Q: 如何处理异步操作中的错误？
**A:** 始终使用适当的错误处理机制：

**async/await 方式（推荐）:**
```javascript
async function fetchData() {
  try {
    const data = await api.getData();
    return data;
  } catch (error) {
    console.error('获取数据失败:', error);
    // 可以抛出错误或返回默认值
    throw error;
    // 或 return null;
  }
}
```

**Promise 方式:**
```javascript
api.getData()
  .then(data => process(data))
  .catch(error => console.error('Error:', error));
```

**回调函数方式:**
```javascript
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('读取文件失败:', err);
    return;
  }
  console.log(data);
});
```

### Q: 如何避免回调地狱（Callback Hell）？
**A:** 使用现代 JavaScript 特性替代嵌套回调：

**Promise 链:**
```javascript
step1()
  .then(result1 => step2(result1))
  .then(result2 => step3(result2))
  .catch(error => handleError(error));
```

**async/await（推荐）:**
```javascript
async function processSteps() {
  try {
    const result1 = await step1();
    const result2 = await step2(result1);
    const result3 = await step3(result2);
    return result3;
  } catch (error) {
    handleError(error);
  }
}
```

**并行执行:**
```javascript
// 并行执行多个独立操作
const [result1, result2, result3] = await Promise.all([
  operation1(),
  operation2(),
  operation3()
]);
```

### Q: 如何处理内存泄漏？
**A:** 常见原因和解决方案：

**1. 全局变量积累:**
```javascript
// ❌ 避免不必要的全局变量
let globalCache = {};

// ✅ 使用局部变量或适当清理
function processData() {
  const cache = {};
  // ... 处理逻辑
  return result;
}
```

**2. 事件监听器未移除:**
```javascript
// ❌ 忘记移除监听器
emitter.on('event', handler);

// ✅ 使用 once() 或手动移除
emitter.once('event', handler);
// 或
emitter.on('event', handler);
// ... 在适当时机
emitter.removeListener('event', handler);
```

**3. 定时器未清除:**
```javascript
// ❌ 忘记清除定时器
const timer = setInterval(() => {}, 1000);

// ✅ 保存引用并清除
const timer = setInterval(() => {}, 1000);
// ... 在适当时机
clearInterval(timer);
```

**4. 缓存未清理:**
```javascript
// ✅ 实现 LRU 缓存或定期清理
const LRU = require('lru-cache');
const cache = new LRU({ max: 100 });
```

**内存分析工具:**
```bash
# 生成堆快照
node --inspect app.js

# 使用 Chrome DevTools 分析内存
```

## 性能相关问题

### Q: 如何提高 Node.js 应用性能？
**A:** 

**1. 使用集群模式（多核 CPU）:**
```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isPrimary) {
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
} else {
  // 启动 HTTP 服务器
  require('./server');
}
```

**2. 避免阻塞操作:**
- 始终使用异步 I/O 方法
- 对于 CPU 密集型任务，使用 worker_threads 或 child_process

**3. 使用流处理大文件:**
```javascript
const fs = require('fs');
const readStream = fs.createReadStream('large-file.txt');
const writeStream = fs.createWriteStream('output.txt');
readStream.pipe(writeStream);
```

**4. 优化数据库查询:**
- 使用索引
- 避免 N+1 查询问题
- 使用连接池

**5. 使用缓存:**
- 内存缓存：Map、LRU cache
- Redis：分布式缓存
- HTTP 缓存：ETag、Cache-Control

**6. 代码分割和懒加载:**
- 只在需要时加载模块
- 使用动态 import()

### Q: 如何监控 Node.js 应用？
**A:** 

**内置监控工具:**
```bash
# 启用性能监控
node --prof app.js

# 生成性能报告
node --prof-process isolate-*.log > profile.txt
```

**性能钩子 API:**
```javascript
const { PerformanceObserver, performance } = require('perf_hooks');

const obs = new PerformanceObserver((items) => {
  console.log(items.getEntries());
});
obs.observe({ entryTypes: ['measure'] });

performance.mark('start');
// ... 代码
performance.mark('end');
performance.measure('operation', 'start', 'end');
```

**第三方监控工具:**
- **PM2**: 进程管理和监控
- **New Relic**: APM 监控
- **Datadog**: 基础设施监控
- **Prometheus + Grafana**: 指标收集和可视化

## 安全相关问题

### Q: 如何防止依赖漏洞？
**A:** 

**1. 定期审计依赖:**
```bash
npm audit
npm audit fix
npm audit fix --force  # 强制修复（可能破坏兼容性）
```

**2. 使用锁定文件:**
- 提交 `package-lock.json` 确保依赖一致性
- 定期更新依赖

**3. 限制依赖范围:**
- 只安装必要的依赖
- 定期审查依赖树：`npm ls`

**4. 使用自动化工具:**
- **Snyk**: 自动检测和修复漏洞
- **Dependabot**: GitHub 集成的自动依赖更新
- **npm outdated**: 检查过期依赖

### Q: 如何安全地处理环境变量？
**A:** 

**1. 使用 .env 文件（不要提交到版本控制）:**
```bash
# .env
DATABASE_URL=your-database-url
API_KEY=your-api-key
JWT_SECRET=your-secret-key
```

**2. 在 .gitignore 中忽略 .env:**
```gitignore
.env
.env.local
.env.development
.env.production
```

**3. 在代码中安全使用:**
```javascript
require('dotenv').config();

// 验证必需的环境变量
const requiredEnvVars = ['DATABASE_URL', 'JWT_SECRET'];
requiredEnvVars.forEach(envVar => {
  if (!process.env[envVar]) {
    throw new Error(`Missing required environment variable: ${envVar}`);
  }
});

const dbUrl = process.env.DATABASE_URL;
```

**4. 不同环境使用不同配置:**
```javascript
// config/index.js
const env = process.env.NODE_ENV || 'development';

const configs = {
  development: require('./development'),
  production: require('./production'),
  test: require('./test')
};

module.exports = configs[env];
```

### Q: 如何处理未捕获的异常？
**A:** 添加全局异常处理器，但要注意这不能完全防止应用崩溃：

```javascript
// 未捕获的异常（同步错误）
process.on('uncaughtException', (error) => {
  console.error('未捕获的异常:', error);
  // 执行清理操作
  gracefullyShutdown();
  // 退出进程
  process.exit(1);
});

// 未处理的 Promise 拒绝
process.on('unhandledRejection', (reason, promise) => {
  console.error('未处理的 Promise 拒绝:', reason);
  // 注意：从 Node.js 15 开始，这会导致进程退出
});

// 优雅关闭
function gracefullyShutdown() {
  // 关闭数据库连接
  // 关闭 HTTP 服务器
  // 清理临时文件
}
```

**最佳实践:**
- 尽量在代码层面处理所有可能的错误
- 使用 try-catch 包装 async/await
- 不要完全依赖全局异常处理器

## 部署相关问题

### Q: 如何部署 Node.js 应用？
**A:** 推荐使用进程管理器确保应用稳定运行：

**PM2（推荐）:**
```bash
# 安装 PM2
npm install -g pm2

# 启动应用
pm2 start app.js --name "my-app"

# 查看应用状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save

# 日志查看
pm2 logs my-app
```

**Docker:**
```dockerfile
FROM node:18-alpine

WORKDIR /app

# 复制 package 文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production

# 复制应用代码
COPY . .

# 创建非 root 用户
RUN chown -R node:node /app
USER node

EXPOSE 3000

CMD ["node", "app.js"]
```

**Systemd（Linux）:**
```ini
# /etc/systemd/system/my-app.service
[Unit]
Description=My Node.js App
After=network.target

[Service]
Type=simple
User=node
WorkingDirectory=/var/www/my-app
ExecStart=/usr/bin/node app.js
Restart=always

[Install]
WantedBy=multi-user.target
```

### Q: 如何处理环境差异？
**A:** 使用环境变量区分不同环境：

```javascript
const isProduction = process.env.NODE_ENV === 'production';
const port = process.env.PORT || 3000;
const dbUrl = process.env.DATABASE_URL;

const config = {
  port,
  database: {
    url: dbUrl,
    poolSize: isProduction ? 20 : 5
  },
  logging: !isProduction,
  cors: {
    origin: isProduction ? 'https://myapp.com' : '*'
  }
};

module.exports = config;
```

**环境文件:**
```
# .env.development
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://localhost:5432/myapp_dev

# .env.production  
NODE_ENV=production
PORT=8080
DATABASE_URL=postgresql://prod-server:5432/myapp_prod
```
