# JDK 소개

## JDK란 무엇인가

JDK(Java Development Kit)는 Java 프로그래밍 언어를 위한 소프트웨어 개발 키트이며 모든 Java 개발의 핵심입니다. JDK에는 Java 런타임 환경(JRE), Java 컴파일러(javac), Java 가상 머신(JVM) 및 광범위한 Java 클래스 라이브러리와 도구가 포함되어 있습니다. JDK는 데스크톱 애플리케이션, 웹 애플리케이션, 모바일 애플리케이션 또는 기업 시스템이든 관계없이 Java 애플리케이션 개발에 필수적입니다.

JDK는 Oracle Corporation(과 그 전신인 Sun Microsystems)에서 개발 및 유지 관리하며 Java 플랫폼의 기반이 됩니다. 최신 LTS(장기 지원) 버전은 JDK 25와 JDK 21이며, JDK 17도 널리 사용되는 LTS 버전입니다. Oracle은 정기적으로 새로운 JDK 버전을 출시하며, 각 버전은 성능 개선, 새로운 기능 및 보안 취약점 수정 사항을带来합니다.

JDK는 단순한 런타임 환경이 아닙니다;它是 Java 개발에 필요한 모든 구성 요소를 제공합니다. 가장 기본적인 구성 요소는 Java 소스 코드를 바이트코드(.class 파일)로 컴파일하는 Java 컴파일러(javac)입니다. 그 다음에는 바이트코드를 실행하는 Java 가상 머신(JVM), 그리고 jar(아카이브 도구), javadoc(문서 생성기), javap(역어셈블러) 등의 다양한 도구가 있습니다. 이러한 도구들이 함께 완전한 Java 개발 환경을 구성합니다.

## JDK 주요 버전

### JDK 25 (최신 버전)

JDK 25는 많은 새로운 기능과 개선 사항을带来的 최신 JDK 버전입니다. 지원 기간이 6개월인 단기 지원 버전입니다. 주요 기능에는 향상된 성능 최적화, 개선된 가비지 컬렉터, 새로운 언어 기능 및 최신 하드웨어 지원이 포함됩니다. JDK 25는 최신 Java 기능을 사용하고자 하는 개발자에게 적합하지만, 더 짧은 지원 주기를 고려해야 합니다.

### JDK 21 (LTS)

JDK 21은 장기 지원(LTS) 버전의 최신 버전으로 최소 8년간 지원을 받을 것입니다. Oracle 무료 사용 약관에 따라 Oracle JDK를 무료로 사용할 수 있는 최신 LTS 버전입니다. JDK 21은 가상 스레드(Virtual Threads), 패턴 매칭(Pattern Matching) 및 레코드(Records)와 같은 중요한 기능을 도입하여 Java 프로그래밍을 크게 단순화하고 개발 효율성을 향상시킵니다.

가상 스레드는 JDK 21의 가장 중요한 기능 중 하나로, 고도로 병렬적인 애플리케이션 작성이 훨씬 간단해집니다. 기존 스레드에 비해 가상 스레드는 생성 비용이 극도로 낮으며 병렬 작업을 처리하기 위해 수백만 개의 가상 스레드를 쉽게 생성할 수 있습니다. 이는 대규모 분산 시스템, 마이크로서비스 아키텍처 및 고성능 서버 애플리케이션 구축에 중요합니다.

### JDK 17 (LTS)

JDK 17은 현재 사용 가능한 가장 성숙한 LTS 버전 중 하나이며 널리 채택되어 있습니다. Oracle 라이선스 계약에 따라 Oracle JDK를 무료로 사용할 수 있는 마지막 버전이므로 많은 기업이 아직 JDK 17을 사용하고 있습니다. JDK 17에는 sealed 클래스, 텍스트 블록, switch 표현식과 같은 많은 OpenJDK 기능이 포함되어 있습니다.

JDK 17의 성능은 특히 가비지 컬렉션 및 JIT 컴파일에서 JDK 11보다 크게 향상되었습니다. 안정적인 장기 지원이 필요한 기업 애플리케이션의 경우 JDK 17은 여전히 탁월한 선택입니다. 성능, 안정성 및 기능 측면에서 좋은 균형을 달성했습니다.

### JDK 11 (LTS)

JDK 11은 Java의 유료 기능 분할을 나타내는 또 다른 중요한 LTS 버전입니다. JDK 11부터는 일부 Oracle JDK 고급 기능에 유료 구독이 필요합니다. JDK 11은 실험적 기능인 ZGC 가비지 컬렉터, 새로운 HTTP 클라이언트 API 및 로컬 변수 타입 추론(var 키워드)을 도입했습니다. 이것이 최신 LTS 버전은 아니지만, JDK 11은 여전히 많은 기존 프로젝트에서 널리 사용되고 있습니다.

## JDK 주요 구성 요소

### Java 컴파일러 (javac)

Java 컴파일러(javac)는 JDK의 가장 필수적인 도구 중 하나로, Java 소스 코드(.java 파일)를 Java 바이트코드(.class 파일)로 컴파일합니다. 바이트코드는 특정 하드웨어 플랫폼에 의존하지 않는 중간 형태의 코드로, Java 가상 머신에서만 실행될 수 있습니다. 이 설계는 Java 프로그램에 "한 번 작성하면 어디서나 실행"이라는 특성을 부여합니다.

javac 컴파일러는 다양한 컴파일 옵션을 지원하여 개발자가 명령줄 매개변수를 통해 컴파일 과정의 다양한 동작을 제어할 수 있습니다. 예를 들어, -d 옵션을 사용하여 컴파일 출력 디렉토리를 지정하고, -classpath를 사용하여 클래스 검색 경로를 지정하며, -source와 -target을 사용하여 소스 코드와 대상 바이트코드의 Java 버전을 지정할 수 있습니다. 이러한 컴파일 옵션 마스터링은 대규모 프로젝트 구축에 중요합니다.

### Java 가상 머신 (JVM)

Java 가상 머신(JVM)은 Java 프로그램 실행의 핵심 환경입니다. JVM은 바이트코드를 로드하고 네이티브 코드 실행을 위해 해석하거나 컴파일하는 역할을 합니다. JVM은 Java의 플랫폼 독립성의 열쇠로, 기본 운영 체제와 하드웨어의 차이를 숨겨 동일한 Java 프로그램이 다양한 플랫폼에서 실행될 수 있게 합니다.

JVM의 주요 구성 요소에는 파일 클래스 로딩을 담당하는 클래스 로더, 바이트코드 실행을 담당하는 실행 엔진, 메서드 영역, 힙, 가상 머신 스택, 네이티브 메서드 스택 및 프로그램 카운터를 포함한 런타임 데이터 영역이 포함됩니다. JVM의 작동 방식을 이해하는 것은高性能 Java 애플리케이션 작성에 필수적입니다.

JVM은 또한 자동 가비지 컬렉션(GC)의 강력한 기능을 제공하여 메모리 할당과 회수를 자동으로 관리합니다. Serial, Parallel, CMS, G1, ZGC, Shenandoah와 같은 다양한 가비지 컬렉터는 다양한 애플리케이션 시나리오에 적합합니다. 개발자는 애플리케이션의 특성에 따라 적절한 가비지 컬렉터를 선택하여 최적의 성능을 달성할 수 있습니다.

### Java 런타임 환경 (JRE)

Java 런타임 환경(JRE)은 Java 프로그램을 실행하는 데 필요한 환경입니다. JVM, Java 핵심 클래스 라이브러리 및 기타 지원 파일을 포함합니다. JRE는 JDK의 하위 집합입니다; JDK에는 JRE plus 개발 도구가 포함되어 있습니다. 개발 없이 Java 프로그램만 실행해야 하는 경우 JRE 설치로 충분합니다.

JRE의 핵심은 java.lang, java.util, java.io, java.net과 같은 Java 핵심 클래스 라이브러리입니다. 이러한 클래스 라이브러리는 수년에 걸쳐 개발 및 최적화되었으며 매우 성숙하고 안정적입니다. 기본 데이터 구조부터 네트워킹 프로그래밍까지 모든 것을涵盖합니다.

### Java 도구 세트

JDK는 Java 프로그램 개발, 디버깅 및 진단을 위한 다양한 명령줄 도구를 제공합니다. 주요 도구는 다음과 같습니다:

jar: 여러 Java 클래스 파일을 배포 및 배포하기 위한 단일 편리한 파일로 패키징하는 데 사용되는 Java 아카이브(.jar 파일) 생성 및 관리 도구입니다.

javadoc: Java 소스 코드의 주석에서 HTML 형식의 API 문서를 생성하는 데 사용되며, 코드 문서를 생성하는 표준 도구입니다.

javap: Java 바이트코드를 역어셈블하는 데 사용되어 파일 클래스 구조와 메서드 시그니처를 볼 수 있으며, Java 컴파일 과정 이해와 디버깅에 매우 유용합니다.

jdb: 기본 디버깅 기능을 제공하는 Java 명령줄 디버거로, 중단점 설정, 단계별 실행, 변수 값 보기 등을 할 수 있습니다.

jps: 현재 시스템의 모든 Java 프로세스 정보를 표시하며, 프로세스 ID와 기본 클래스 이름을 포함합니다.

jstat: 가비지 컬렉션 정보, 클래스 로딩 정보 등 JVM 통계를 표시합니다.

jstack: Java 스레드의 스택 추적 정보를 인쇄하며, 스레드 정지 문제 진단에 사용됩니다.

jmap: Java 힙 메모리 정보를 표시하며, 분석을 위한 힙 덤프 파일을 생성할 수 있습니다.

이러한 도구들은日常 개발과 프로덕션 문제 진단에 매우 유용하며, 마스터하면 개발 효율성과 진단 능력을 크게 향상시킬 수 있습니다.

## JDK 적용 분야

### 기업 애플리케이션 개발

JDK는 기업 애플리케이션 개발의 기반입니다. Spring, Hibernate, Struts와 같은 다양한 Java EE 프레임워크(지금 이름은 Jakarta EE)가 JDK에서 실행되어야 합니다. 이러한 프레임워크는 의존성 주입, 트랜잭션 관리, 데이터 지속성 및 웹 서비스를 포함한 기업 애플리케이션에 대한 종합적인 솔루션을 제공합니다.

대기업 애플리케이션은 일반적으로 안정성과 성능에 대한 요구 사항이 매우 높으므로 적절한 JDK 버전을 선택하는 것이 매우 중요합니다. 많은 기업들은 JDK 17과 같은 성숙한 LTS 버전을 상업적 지원 서비스와 결합하여 시스템의 장기적인 안정적인 운영을 보장합니다.

### Android 개발

Android 앱은 주로 Java 또는 Kotlin으로 작성되지만 Android 앱 개발은 JDK 없이는 불가능합니다. Android SDK 도구는 Android 프로젝트를 컴파일하고 Android 기기에 설치할 수 있는 APK 파일을 생성하는 데 JDK가 필요합니다. Android Studio에는 JDK가 내장되어 있으며, 개발자는 개발 환경 설정 시 JDK 경로를 지정해야 합니다.

### 빅 데이터 처리

빅 데이터 분야에서 Java와 JDK는 핵심적인 역할을 합니다. Hadoop, Spark, Flink와 같은 주요 빅 데이터 프레임워크는 모두 Java로 개발되었으며 JDK에서 실행되어야 합니다. 이러한 프레임워크는 일반적으로 막대한 양의 데이터를 처리해야 하며 성능과 안정성에 대한 요구 사항이 매우 높습니다.

### 웹 애플리케이션 개발

Java 웹 애플리케이션 개발은 JDK의 전통적인 강점입니다. Servlet, JSP, Filter, Listener와 같은 기술은 Java 웹 개발의 기반을 형성합니다. Spring MVC, Struts, JSF와 같은 주요 웹 프레임워크가 모두 JDK에서 실행됩니다.

### 클라우드 컴퓨팅 및 마이크로서비스

클라우드 컴퓨팅과 마이크로서비스 아키텍처의 인기에 따라 JDK도 이러한 분야에서 중요한 역할을 합니다. Spring Cloud 및 Dubbo와 같은 마이크로서비스 프레임워크는 분산 시스템 구축을 위한 종합적인 솔루션을 제공합니다. 이러한 모든 프레임워크는 JDK에서 실행되어야 합니다.

## JDK 및 기타 Java 구현체

### OpenJDK

OpenJDK는 Oracle과 커뮤니티가 공동으로 개발하고 유지 관리하는 Java SE의 오픈소스 구현체입니다. OpenJDK는 대부분의 다른 JDK 배포판의 기반입니다. Eclipse(Eclipse Temurin), Amazon(Amazon Corretto), Microsoft(Microsoft Build of OpenJDK)와 같은 많은 공급업체가 OpenJDK를 기반으로 자체 JDK 배포판을 구축하고 있습니다.

OpenJDK와 Oracle JDK는 코드 수준에서 거의 동일하며, 주요 차이점은 라이선스 계약과 지원 서비스에 있습니다. Oracle JDK는 상업적 지원 및エンタープ라이즈 수준의 추가 기능을 제공하지만, 일부 버전은 상업적 사용에 대해 비용이 발생할 수 있습니다. OpenJDK는 완전히 오픈소스이며 무료로 커뮤니티에서 유지 관리됩니다.

대부분의 사용자에게 OpenJDK를 사용하는 것은 전혀 충분합니다. 많은 조직들은 라이선스 문제를 피하면서 Oracle JDK와 거의 동일한 성능과 기능을 얻기 위해 OpenJDK 사용을 선택합니다.

### 기타 JDK 배포판

Oracle JDK와 OpenJDK 외에도 많은 다른 JDK 배포판이 있습니다:

Eclipse Temurin(이전 AdoptOpenJDK): Eclipse Foundation에서 유지 관리하는 OpenJDK 배포판으로, 고품질 빌드와 장기 지원을 제공합니다.

Amazon Corretto: Amazon에서 제공하는 무료 크로스플랫폼 JDK 배포판으로, 장기 지원과 성능 최적화가 제공됩니다.

Microsoft Build of OpenJDK: Microsoft에서 제공하는 OpenJDK 빌드로, Windows 및 Azure에 최적화되어 있습니다.

Azul Zulu: Azul Systems에서 제공하는 JDK 배포판으로,高性能과 안정성으로 유명합니다.

GraalVM: Oracle에서 제공하는 다중 언어 가상 머신으로, Java뿐만 아니라 JavaScript, Python, Ruby 및 다른 언어도 지원합니다.

## JDK의 중요성

Java 개발의 기반으로서 JDK의 중요성은 말할 필요도 없습니다. 사용되는 프레임워크나 도구에 관계없이 JDK 지원이 필수적입니다. JDK는 단순히 Java 프로그램을 컴파일하고 실행하기 위한 도구 세트가 아닙니다; Java 생태계의 핵심을代表합니다.

적절한 JDK 버전을 선택하고 올바르게 구성하는 것은 프로젝트의 성공에至关重要합니다. 고려해야 할 요소에는 버전의 안정성, 보안 업데이트 빈도, 새로운 기능, 성능 최적화 및 커뮤니티 지원이 포함됩니다. 기업 애플리케이션의 경우 장기 지원을 얻기 위해 LTS 버전을 사용하는 것이 좋습니다.

Java 프로그래밍 언어의 지속적인 발전과 함께 JDK도 계속 업데이트되고 있습니다. 새로운 JDK 버전은 더 나은 성능, 더 많은 기능 및 향상된 보안을 제공합니다. JDK를 최신 상태로 유지하면 더 나은 개발 경험과 운영 효율성을 얻을 수 있습니다.

## 버전별 차이점

JDK의 서로 다른 주요 버전은 설치, 사용 및 기능에서 중요한 차이가 있습니다:

**JDK 8**: 상업적 사용을 위한 마지막 무료 Oracle JDK 버전입니다. 람다表达式, 스트림 API 및 새로운 날짜/시간 API를 도입했습니다. 레거시 시스템에서 여전히 널리 사용됩니다.

**JDK 11**: Java 유료 기능의 시작을 나타냅니다. 로컬 변수 타입 추론(var), 새로운 HTTP 클라이언트 및 ZGC(실험적)를 도입했습니다. 32비트 시스템 지원의 마지막 버전입니다.

**JDK 17**: NFTC에 따른 마지막 무료 Oracle JDK 버전입니다. sealed 클래스, 텍스트 블록 및 switch 표현식을 도입했습니다. 성능, 안정성 및 기능 사이의 훌륭한 균형을 달성했습니다.

**JDK 21**: 가상 스레드, switch용 패턴 매칭 및 레코드가 안정적인 기능으로 포함된 최신 LTS 버전입니다. 새 프로젝트에 권장됩니다.

**JDK 25**: 최신 언어 기능과 성능 개선이 포함된 최신 버전입니다. 최신 기능을 원하는 개발자에게 적합하지만 더 짧은 지원 주기가 있습니다.
