# MongoDB 常見問題

## 基礎概念

### 什麼是 MongoDB？

MongoDB 是一個文件資料庫，旨在幫助開發者更快地建構現代應用程式。它將資料儲存在類似 JSON 的靈活文件中，從而可以像應用程式碼使用資料一樣輕鬆地對資料進行建模。MongoDB 提供強大的查詢引擎、水平擴展和內建高可用性。

### MongoDB 是關聯式資料庫嗎？

不是。MongoDB 是面向文件的 NoSQL 資料庫，使用類似 JSON 的 BSON 格式儲存資料。它與傳統關聯式資料庫有本質區別，不使用表、行和列的概念，而是使用集合和文件。

### MongoDB 中的文件是什麼？

文件是 MongoDB 中的基本資料單元，由欄位和值對組成。文件可以包含其他文件、陣列和巢狀物件，結構靈活多樣。每個文件都有一個唯一的 _id 欄位作為主鍵。

### MongoDB 和 MySQL 有什麼區別？

| 特性 | MongoDB | MySQL |
|------|---------|-------|
| 資料模型 | 文件型（類似 JSON） | 關聯型（行列） |
| 模式 | 靈活（動態模式） | 固定（靜態模式） |
| 擴展方式 | 水平擴展（分片） | 垂直擴展 |
| 查詢語言 | MongoDB 查詢語言 | SQL |
| 交易 | 多文件 ACID 交易 | 多表 ACID 交易 |
| 關聯 | 嵌入式文件/陣列 | JOIN 操作 |
| 索引 | 支援多種索引類型 | 支援索引 |

### 什麼是 BSON？

BSON 是 Binary JSON 的縮寫，是 JSON 文件的二進位表示形式，有額外的資料類型支援（如 ObjectId、Date、Binary 等）。

### MongoDB 的最大文件大小是多少？

BSON 文件大小的上限為 16 MB。對於超過此限制的檔案，MongoDB 提供 GridFS API。

### MongoDB 支援哪些資料類型？

MongoDB 支援多種 BSON 資料類型，包括：Double、String、Object、Array、Binary Data、ObjectId、Boolean、Date、Null、Regular Expression、JavaScript、JavaScript with Scope、32-bit Integer、Timestamp、64-bit Integer、Decimal128、Min Key、Max Key 等。

## 安裝問題

### MongoDB 需要多少記憶體？

MongoDB 會盡量使用可用記憶體作為快取，官方建議至少 4GB 記憶體用於開發環境，生產環境根據資料量和負載具體評估。對於大型生產環境，通常建議更多記憶體以獲得最佳效能。

### MongoDB 支援 32 位元系統嗎？

不支援。MongoDB 從某個版本開始停止支援 32 位元平台，請使用 64 位元作業系統。

### 如何更改 MongoDB 預設連接埠？

MongoDB 預設連接埠是 27017。可以透過以下方式更改：

1. 修改設定檔中的 `net.port`
2. 在啟動時指定 `--port` 參數

### 多個 MongoDB 執行個體可以在同一台機器上執行嗎？

可以，但需要設定不同的：

- 連接埠號碼
- 資料目錄（`--dbpath`）
- 日誌檔案路徑
- 設定檔

### MongoDB 資料儲存在哪裡？

預設情況下：

- Linux/macOS：`/data/db`
- Windows：`C:\data\db`

可以透過設定檔中的 `storage.dbPath` 指定其他路徑。

### 安裝 MongoDB 8.0 還是 7.0？

- 新專案：推薦使用最新的穩定版本 MongoDB 8.2
- 生產環境：可以選擇最新的穩定版本或長期支援版本 MongoDB 7.0

MongoDB 6.0 已於 2025年7月31日停止支援，不再接收安全更新。

## 使用問題

### 如何備份 MongoDB 資料？

使用 mongodump 工具：

```bash
# 匯出整個資料庫
mongodump --db=mydatabase --out=/backup

# 匯出指定集合
mongodump --db=mydatabase --collection=users --out=/backup
```

復原使用 mongorestore：

```bash
mongorestore --db=mydatabase /backup/mydatabase
```

### MongoDB 如何實現高可用？

透過副本集（Replica Set）實現高可用。副本集由主節點和多個從節點組成：

- 主節點處理所有寫入操作
- 從節點複寫主節點的資料
- 主節點故障時自動選舉新的主節點

### 什麼是分片？

分片是 MongoDB 水平擴展的方式，將資料分佈到多個伺服器上。每個分片包含資料的一部分，共同組成完整的資料集。分片允許您建構非常大的資料集和高吞吐量的操作。

### 副本集和分片叢集有什麼區別？

- **副本集**：多個 MongoDB 執行個體維護相同的資料副本，提供高可用性和讀取擴展
- **分片叢集**：資料分佈在多個伺服器上，提供水平擴展能力，支援大規模資料集

### 如何優化查詢效能？

1. 建立適當的索引：使用 `createIndex()` 為常用查詢欄位建立索引
2. 使用投影：只回傳必要欄位，減少網路傳輸
3. 避免全集合掃描：確保查詢使用索引
4. 使用聚合管道：進行複雜的資料處理
5. 適當使用覆蓋索引：索引包含所有查詢欄位
6. 限制回傳結果：使用 `limit()` 和 `skip()` 進行分頁

### MongoDB 支援交易嗎？

支援。MongoDB 4.0+ 引入了多文件交易支援，跨副本集和分片叢集都可以使用交易。交易遵循 ACID 語意：

- 原子性：交易中的所有操作要么全部成功，要么全部失敗
- 一致性：交易前後資料庫狀態保持一致
- 隔離性：並發交易相互隔離
- 持久性：交易提交後，資料持久保存

### 什麼是變更串流？

變更串流是 MongoDB 的一種功能，允許應用程式即時監聽集合、資料庫或整個叢集的資料變更事件。應用程式可以訂閱變更事件，實現即時同步、稽核日誌等功能。

### MongoDB 如何保證資料安全？

1. **身份驗證**：啟用使用者名稱密碼驗證或 LDAP/Kerberos（企業版）
2. **存取控制**：使用基於角色的存取控制（RBAC）
3. **加密通訊**：啟用 TLS/SSL 加密客戶端和伺服器之間的通訊
4. **加密儲存**：使用 MongoDB Enterprise 的加密儲存引擎
5. **欄位級加密**：對敏感欄位進行加密
6. **稽核日誌**：記錄資料庫操作（企業版）
7. **定期備份**：定期備份資料

### MongoDB 企業版和社群版有什麼區別？

企業版包含更多進階功能：

- LDAP 身份驗證
- Kerberos 身份驗證
- 稽核
- 加密儲存引擎
- 平面分析
- 技術支援

社群版功能對於大多數使用場景已經足夠。

### 如何選擇合適的版本？

- 新專案：推薦使用最新的穩定版本（MongoDB 8.2）
- 生產環境：可以選擇最新的穩定版本或長期支援版本（MongoDB 7.0）
- 特定需求：根據應用相容性和功能需求選擇

### 什麼是 GridFS？

GridFS 是 MongoDB 用於儲存大型檔案的規範。它將大型檔案分割成多個區塊（chunk），每個區塊預設 255KB，儲存在 `fs.chunks` 集合中，檔案中繼資料儲存在 `fs.files` 集合中。

### MongoDB 適合儲存什麼類型的資料？

- 內容管理系統資料
- 使用者設定和画像資料
- 日誌和事件資料
- 物聯網感測器資料
- 即時分析資料
- 行動應用後端資料
- 電子商務產品目錄
- 聊天訊息和即時通訊資料

### 什麼是副本集 oplog？

oplog（操作日誌）是副本集中的一個特殊集合，記錄主節點上的所有寫入操作。從節點透過讀取和應用 oplog 中的操作來保持與主節點的資料同步。

### 如何監控 MongoDB 效能？

1. 使用 `db.serverStatus()` 查看伺服器狀態
2. 使用 `db.stats()` 查看資料庫狀態
3. 使用 Profiler 分析慢查詢：`db.setProfilingLevel(1, 100)`
4. 使用 `db.currentOp()` 查看目前操作
5. 整合 Prometheus 和 Grafana 進行視覺化監控
6. 使用 MongoDB Atlas 的內建監控功能（如果使用雲端服務）

### MongoDB 的複寫延遲是什麼？

複寫延遲是指主節點上的操作與將該操作應用到從節點之間的延遲。少量延遲是可以接受的，但隨著複寫延遲增加，會出現嚴重問題。可以透過以下方式監控：

```bash
rs.status()
```

查看 `optimes.lastCommittedOpTime` 和主節點操作時間之間的差異。

### 什麼是讀取偏好（Read Preference）？

讀取偏好決定客戶端從副本集的哪個節點讀取資料：

- **primary**：從主節點讀取（預設）
- **primaryPreferred**：優先從主節點，必要時從從節點
- **secondary**：只從從節點讀取
- **secondaryPreferred**：優先從從節點，必要時從主節點
- **nearest**：從網路延遲最低的節點讀取

### 什麼是寫入關注（Write Concern）？

寫入關注指定寫入操作的確認級別：

- **w: 1**：等待主節點確認（預設）
- **w: majority**：等待大多數副本集成員確認
- **w: "all"**：等待所有副本集成員確認
- **j: true**：等待寫入到磁碟日誌
- **wtimeout**：逾時時間

### 什麼是 capped collection？

capped collection 是固定大小的集合，類似於循環佇列，當達到大小限制時，舊文件會自動被新文件取代。常用於儲存日誌、快取等場景。

### MongoDB 支援哪些索引類型？

- 單欄位索引
- 複合索引
- 多鍵索引（陣列欄位）
- 文字索引
- 地理空間索引（2dsphere、2d）
- 雜湊索引
- 唯一索引
- 部分索引
- 稀疏索引
- TTL 索引

### 什麼是 TTL 索引？

TTL（Time-To-Live）索引允許文件在指定時間後自動過期刪除。常用于工作階段儲存、快取、臨時資料等場景。

## 故障排除

### 連線被拒絕怎麼辦？

1. 檢查 MongoDB 服務是否執行：`sudo systemctl status mongod`
2. 檢查連接埠是否正確：預設 27017
3. 檢查防火牆設定
4. 檢查綁定 IP 設定

### 磁碟空間不足怎麼辦？

1. 清理舊資料
2. 刪除不需要的集合
3. 使用 `db.repairDatabase()` 回收空間
4. 擴展磁碟空間

### 查詢很慢怎麼辦？

1. 使用 `explain()` 分析查詢計畫
2. 建立適當的索引
3. 檢查是否有慢查詢：`db.getProfilingLevel()`
4. 考慮使用投影只回傳必要欄位

### 副本集選舉失敗怎麼辦？

1. 檢查網路連線
2. 檢查節點設定
3. 確保大多數節點可用
4. 查看日誌：`rs.status()`

### 資料不一致怎麼辦？

1. 同步從節點
2. 重新初始化副本集
3. 從備份復原資料

## 開發相關

### MongoDB 支援哪些程式語言？

MongoDB 提供幾乎所有主流程式語言的驅動程式，包括：

- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP
- Scala
- Kotlin
- Swift

### 如何在應用程式中使用 MongoDB？

1. 安裝對應語言的驅動程式
2. 建立連線
3. 執行 CRUD 操作

例如 Node.js：

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('mydb');
  const collection = database.collection('users');
  
  // 插入文件
  await collection.insertOne({ name: '張三', age: 30 });
  
  // 查詢文件
  const user = await collection.findOne({ name: '張三' });
  
  await client.close();
}
```

### 什麼是 MongoDB Atlas？

MongoDB Atlas 是 MongoDB 提供的雲端資料庫服務，提供完全受管的 MongoDB 部署，包括：

- 自動備份和復原
- 全球分布
- 即時效能監控
- 免運維管理
- 免費層級可供試用

### 如何進行資料建模？

MongoDB 資料建模原則：

1. 優先考慮文件結構而不是正規化
2. 使用嵌入式文件表示「包含」關係
3. 使用參考表示「多對多」關係
4. 根據查詢模式設計集合和索引
5. 考慮資料成長和擴展性
