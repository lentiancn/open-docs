# Flutter 사용 매뉴얼

## 기본 개념

### 위젯 소개

위젯은 Flutter UI의 기본 구성 요소입니다. Flutter에서는 everything이 위젯이며, 레이아웃, 패딩, 정렬 등을 포함합니다. 위젯은 현재 구성과 상태가 주어졌을 때 뷰가 어떻게 보여야 하는지를 설명합니다.

Flutter 위젯은 불변입니다. UI를 변경해야 할 때 위젯을 다시 만들거나 상태 관리가 있는 StatefulWidget을 사용해야 합니다. 위젯을 중첩하여 위젯 트리를 형성할 수 있으며, Flutter는 이전과 새로운 위젯 트리를 효율적으로 비교하여 실제로 변경된 부분만 렌더링합니다.

일반적인 위젯: Text, Image, Icon, Container, Row/Column, Stack, ListView 등

### StatelessWidget과 StatefulWidget

StatelessWidget은 상태가 없는 위젯으로, 텍스트와 이미지 같은 정적 UI 표시에 적합합니다. StatelessWidget 정의는 빌드 메서드를 구현하고 렌더링할 위젯 트리를 반환하면 됩니다.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('안녕하세요 Flutter');
  }
}
```

StatefulWidget은 상태가 있는 위젯으로, 카운터와 폼 같은 동적 UI 업데이트가 필요한 시나리오에 적합합니다. StatefulWidget은 State와 함께 사용해야 하며, 상태가 변경되면 setState를 호출하여 Flutter에 다시 빌드하도록 알립니다.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('카운트: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('증가'),
        ),
      ],
    );
  }
}
```

## 레이아웃 기초

### 선형 레이아웃

Row와 Column은 가장 일반적으로 사용되는 선형 레이아웃 위젯입니다. Row는 자식 위젯을 수평으로 배치하고, Column은 수직으로 배치합니다. MainAxisAlignment와 CrossAxisAlignment를 사용하여 정렬을 제어합니다.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('홈'),
    Icon(Icons.settings),
  ],
)
```

### 스택 레이아웃

Stack은 스택 레이아웃을 만드는 데 사용되며, 위젯을 다른 위젯 위에 배치합니다. Positioned는 Stack에서 자식의 위치를 지정하는 데 사용됩니다.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('오버레이'),
    ),
  ],
)
```

### 리스트 레이아웃

ListView는 가장 일반적으로 사용되는 리스트 위젯으로, 긴 목록에 적합합니다. ListView.builder는 지연 로딩이 있는 큰 데이터셋에 적합합니다. ListView.separated는 구분자를 추가할 수 있습니다.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## 상태 관리

### setState

setState는 가장 기본적인 상태 관리 방법이며, 간단한 로컬 상태에 적합합니다. StatefulWidget에서 setState를 호출하면 위젯 재빌드가 트리거됩니다.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider는 Flutter의 권장 상태 관리 솔루션 중 하나입니다. Provider를 통해 위젯 트리에서 상태를 공유하여 레이어별로 전달하는 것을 피할 수 있습니다.

```dart
// ChangeNotifier 정의
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Provider 사용
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

### 기타 상태 관리 솔루션

복잡한 애플리케이션의 경우 Riverpod, GetX, BLoC도 있습니다. Riverpod는 Provider의 업그레이드로, 더 나은 성능과 타입 안전성을 제공합니다. GetX는 상태 관리, 라우팅, 의존성 주입을 통합합니다. BLoC는 이벤트 스트림을 사용하여 상태를 관리하며, 크고 복잡한 애플리케이션에 적합합니다.

## 네트워크 요청

### HTTP 패키지

http 패키지를 사용하여 간단한 HTTP 요청을 합니다:

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

### Dio

Dio는 강력한 HTTP 클라이언트로, 인터셉터, 요청 취소, 파일 업로드 등을 지원합니다:

```dart
import 'package:dio/dio.dart';

final dio = Dio();
final response = await dio.get('https://api.example.com/data');
```

## 로컬 저장소

### SharedPreferences

간단한 키-값 데이터 저장에 사용됩니다:

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

### SQLite

구조화된 데이터의 경우 sqflite 패키지를 사용합니다:

```dart
import 'package:sqflite/sqflite.dart';

final db = await openDatabase('my.db');
await db.insert('users', {'name': 'Flutter'});
```

## 네비게이션과 라우팅

Flutter는 페이지 네비게이션에 Navigator를 사용합니다. 기본 작업에는 Navigator.push(새 페이지 푸시), Navigator.pop(반환), Navigator.pushReplacement(현재 페이지 교체) 등이 있습니다.

명명된 라우팅은 권장되는 라우팅 관리 방법입니다:

```dart
// 라우트 정의
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// 네비게이션
Navigator.pushNamed(context, '/detail');
```

## 빌드와 배포

### Android 릴리스

keytool을 사용하여 서명 키를 생성합니다:

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

android/app/build.gradle에서 서명을 구성한 다음, 프로젝트 루트에서 실행합니다:

```bash
flutter build apk --release
```

### iOS 릴리스

Xcode에서 서명 인증서와 provisioning 프로파일을 구성한 다음, 실행합니다:

```bash
flutter build ios --release
```

iOS 릴리스는 Apple Developer 계정과 해당 인증서가 필요합니다.
