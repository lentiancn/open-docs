# Guía de Instalación

## Requisitos del Sistema

Antes de instalar Jenkins, asegúrese de que su sistema cumpla con los siguientes requisitos mínimos:

### Configuración Mínima Recomendada

- 256MB de memoria disponible
- 1GB de espacio en disco disponible (10GB recomendados al ejecutar como contenedor Docker)

### Configuración Recomendada para Equipos Pequeños

- 1GB+ de memoria disponible
- 50GB+ de espacio en disco disponible

### Requisitos de Software

- Java 8 o superior (ya sea JRE o JDK)
- Nota: Si ejecuta Jenkins como contenedor Docker, no es necesario instalar Java en el host

## Plataformas de Instalación

Jenkins soporta múltiples métodos de instalación. Esta guía cubre los enfoques más comunes.

## Instalación con Docker

Docker es la forma recomendada de instalar Jenkins ya que proporciona un entorno aislado y simplifica la configuración.

### Instalación de Docker

Primero, instale Docker en su sistema. Visite el sitio web oficial de Docker y elija la versión apropiada de Docker Community Edition para su sistema operativo.

### Descargar y Ejecutar Jenkins

En macOS y Linux, abra una terminal y ejecute:

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

En Windows, abra un símbolo del sistema y ejecute un comando similar:

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

Explicación de parámetros:

- `-u root`: Ejecutar contenedor como usuario root
- `--rm`: Eliminar automáticamente el contenedor al detenerlo
- `-d`: Ejecutar en segundo plano (modo separado)
- `-p 8080:8080`: Mapear puerto del contenedor 8080 al puerto del host 8080
- `-p 50000:50000`: Mapear puerto del contenedor 50000 al puerto del host 50000 (para comunicación de agentes JNLP)
- `-v jenkins-data:/var/jenkins_home`: Persistir datos de Jenkins
- `-v /var/run/docker.sock:/var/run/docker.sock`: Permitir a Jenkins ejecutar comandos Docker en pipelines
- `jenkinsci/blueocean`: Imagen oficial de Jenkins con interfaz Blue Ocean

### Acceder al Contenedor de Jenkins

Para acceder al interior del contenedor de Jenkins, use:

```bash
docker exec -it jenkins-blueocean bash
```

### Ver Registros de Jenkins

Vea los registros de la consola de Jenkins con:

```bash
docker logs jenkins-blueocean
```

## Instalación con Archivo WAR

La versión del archivo WAR de Jenkins puede instalarse en cualquier sistema operativo que soporte Java.

### Pasos de Instalación

1. Descargue el paquete WAR estable más reciente de Jenkins desde el sitio web oficial
2. Abra una terminal en el directorio de descarga
3. Ejecute: `java -jar jenkins.war`
4. Navegue a http://localhost:8080 y espere a que aparezca la página de Desbloqueo de Jenkins

### Notas

- La instalación con archivo WAR no instala automáticamente Blue Ocean; debe instalarse por separado a través de la gestión de plugins
- Puede cambiar el puerto usando `--httpPort`, por ejemplo: `java -jar jenkins.war --httpPort=9090`

## Instalación en macOS

### Usando el Paquete

1. Descargue el paquete de Jenkins para macOS
2. Haga doble clic para abrir y siga el asistente de instalación

### Usando Homebrew

Instale la última versión:

```bash
brew install jenkins
```

Instale la versión LTS:

```bash
brew install jenkins-lts
```

## Instalación en Linux (Debian/Ubuntu)

En distribuciones basadas en Debian, instale Jenkins a través de apt:

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

Después de la instalación, Jenkins:

- Se configurará para iniciar como un daemon al arrancar
- Creará un usuario jenkins para ejecutar el servicio
- Enviará registros a /var/log/jenkins/jenkins.log
- Escuchará en el puerto 8080 por defecto

## Asistente de Configuración Post-Instalación

Al iniciar Jenkins por primera vez, comienza el asistente de configuración.

### Desbloquear Jenkins

1. Navegue a http://localhost:8080
2. Cuando aparezca la página de Desbloquear Jenkins, copie la contraseña generada automáticamente de los registros de la consola
3. Pegue la contraseña en el campo de contraseña de administrador y haga clic en Continuar

### Personalizar Plugins de Jenkins

En la página de instalación de plugins, elija:

- Instalar plugins sugeridos: Instala un conjunto recomendado de plugins basado en casos de uso comunes
- Seleccionar plugins para instalar: Elegir manualmente qué plugins instalar

### Crear el Primer Usuario Administrador

Finalmente, cree una cuenta de administrador proporcionando:

- Nombre de usuario
- Contraseña
- Confirmar contraseña
- Nombre completo
- Dirección de correo electrónico

Haga clic en Guardar y Finalizar, luego haga clic en Comenzar a usar Jenkins para acceder a la interfaz principal.

## Verificar la Instalación

Verifique la instalación:

1. Navegue a http://localhost:8080
2. Inicie sesión con la cuenta de administrador creada
3. Verifique que el estado del sistema sea normal

## Problemas Comunes

### Puerto Ya en Uso

Si el puerto 8080 está siendo usado por otro programa:

- Método Docker: Modificar el parámetro -p
- Método WAR: Usar parámetro --httpPort

### Memoria Insuficiente

Si la memoria del sistema es limitada:

- Usar Docker con límites de recursos del contenedor
- Ajustar el tamaño del heap de JVM en los parámetros de inicio de Jenkins

### Primer Inicio Lento

El primer inicio descarga e instala plugins, lo que puede tomar tiempo. Por favor tenga paciencia.

## Próximos Pasos

Después de la instalación, aprenda a:

- Crear su primer trabajo de pipeline
- Configurar código fuente (como Git)
- Configurar disparadores de construcción
- Integrar herramientas de prueba y despliegue
