# Claude Code よくある質問

このガイドでは、Claude Codeのインストールと使用中に発生する一般的な問題とその解決策を説明します。

## インストール問題

### コマンドが見つからない

インストール後、`command not found: claude`または`'claude' is not recognized`エラーが発生した場合は、インストールディレクトリがシステムPATHに含まれていません。

**解決策：** PATH環境変数を確認して構成します。

macOS/Linuxで.zshrcまたは.bashrcに追加：
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Windows PowerShell：
```powershell
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
```

ターミナルを再起動して確認：
```bash
claude --version
```

### インストールスクリプトがHTMLを返す

インストールコマンドを実行すると次のエラーが表示されます：
- `bash: line 1: syntax error near unexpected token '<'`
- `bash: line 1: '<!DOCTYPE html>'`
- PowerShellの場合：`Invoke-Expression: Missing argument in parameter list`

これはインストールURLがインストールスクリプトではなくHTMLページを返したことを意味します。

**解決策：**

1. 代替インストール方法を使用：
   - macOS/Linux：Homebrewを使用
     ```bash
     brew install --cask claude-code
     ```
   - Windows：WinGetを使用
     ```powershell
     winget install Anthropic.ClaudeCode
     ```

2. HTMLページに「App unavailable in region」と表示されている場合は、Claude Codeが居住地域で利用できません。[サポートされている国と地域のリスト](https://www.anthropic.com/supported-countries)を参照してください。

3. 数分待って再試行してください。問題は通常一時的なものです。

### curlダウンロードの失敗

エラーメッセージ：`curl: (56) Failure writing output to destination`

これはスクリプトのダウンロードが完了する前に接続が切断されたことを意味します。

**解決策：**

1. ネットワークの安定性を確認：
   ```bash
   curl -fsSL https://storage.googleapis.com -o /dev/null
   ```
   コマンドが正常に完了した場合は接続に問題はありません。インストールコマンドを再試行してください。

2. 代替インストール方法を使用（上記参照）。

### TLS/SSL接続エラー

エラーメッセージ：`curl: (35) TLS connect error`、`schannel: next InitializeSecurityContext failed`、または`Could not establish trust relationship for the SSL/TLS secure channel`

**解決策：**

1. システムのCA証明書を更新：
   - Ubuntu/Debian：
     ```bash
     sudo apt-get update && sudo apt-get install ca-certificates
     ```
   - macOS（Homebrewを使用）：
     ```bash
     brew install ca-certificates
     ```

2. Windows PowerShellでTLS 1.2を有効化：
   ```powershell
   [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
   irm https://claude.ai/install.ps1 | iex
   ```

3. プロキシまたはファイアフォールの干渉を確認し、`NODE_EXTRA_CA_CERTS`環境変数を構成します。

### バージョンの取得に失敗

エラーメッセージ：`Failed to fetch version from storage.googleapis.com`

これは`storage.googleapis.com`がネットワークでブロックされていることを意味します。

**解決策：**

1. 直接接続をテスト：
   ```bash
   curl -sI https://storage.googleapis.com
   ```

2. プロキシの後ろにある場合は、`HTTPS_PROXY`環境変数を設定します。

### WindowsでGit Bashが必要

エラーメッセージ：`Claude Code on Windows requires git-bash`

**解決策：**[Git for Windows](https://git-scm.com/downloads/win)をインストールしてからインストールコマンドを再実行します。

Gitがインストールされているのにエラーが続く場合は、設定ファイルでGit Bashのパスを構成：
```json
{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}
```

### Linuxで不正な命令

エラーメッセージ：`Illegal instruction`

これは通常、アーキテクチャの不一致によって発生します。インストールされたバイナリとお使いのCPUアーキテクチャが互換性があることを確認してください。

### macOSでバイナリをロードできない

エラーメッセージ：`dyld: cannot load`または`Abort trap`

これはバイナリとお使いのシステムが互換性がないことを意味します。正しいバージョンのClaude Code（IntelまたはApple Silicon）をダウンロードしたことを確認してください。

### 低メモリサーバーでのインストール強制終了

エラーメッセージ：Linuxサーバーでインストール中に`Killed`と表示

**解決策：** 低メモリサーバーにswap領域を追加します。

## 認証問題

### OAuthエラーまたは403 Forbidden

**解決策：**

1. アカウントにClaude Codeアクセス権があるか確認（Pro、Max、Teams、Enterprise、またはConsole）
2. キャッシュされた認証情報をクリアして再ログイン：
   ```bash
   rm -rf ~/.claude
   claude
   ```
3. 居住地域が[サポートリスト](https://www.anthropic.com/supported-countries)にあるか確認

### 地域で利用不可

エラーメッセージ：`App unavailable in region`

Claude Codeはお使いの国/地域では利用できません。[Anthropicがサポートする国と地域](https://www.anthropic.com/supported-countries)を参照してください。

## ネットワーク構成

### ネットワーク接続の確認

インストーラーは`storage.googleapis.com`からダウンロードします。アクセスできるか確認：

```bash
curl -sI https://storage.googleapis.com
```

失敗した場合の原因：
- Google Cloud Storageをブロックしている企業のファイアウォールまたはプロキシ
- 地域のネットワーク制限：VPNまたは代替ネットワークを試行
- TLS/SSLの問題：システムのCA証明書を更新、または`HTTPS_PROXY`が構成されているか確認

### プロキシ構成

企業のプロキシの後ろにいる場合は、インストール前にプロキシ環境変数を設定：

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

## 競合するインストールの確認

複数のClaude Codeインストールは、バージョンの不一致や予期しない動作を引き起こす可能性があります。

インストール内容を確認：

**macOS/Linux：**
```bash
which -a claude
ls -la ~/.local/bin/claude
npm -g ls @anthropic-ai/claude-code 2>/dev/null
```

**Windows：**
```powershell
where.exe claude
Test-Path "$env:LOCALAPPDATA\Claude Code\claude.exe"
```

複数のインストールが見つかった場合は、1つだけ保持します（ネイティブインストール`~/.local/bin/claude`を推奨）。

追加のインストールを削除：
- npmグローバルインストール：`npm uninstall -g @anthropic-ai/claude-code`
- Homebrewインストール：`brew uninstall --cask claude-code`

## ディレクトリ権限の確認

インストーラーには`~/.local/bin/`と`~/.claude/`への書き込み権限が必要です。 эти каталоги записываемы:

```bash
test -w ~/.local/bin && echo "writable" || echo "not writable"
test -w ~/.claude && echo "writable" || echo "not writable"
```

書き込み不可の場合：

```bash
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```

## バイナリの確認

`claude`がインストールされているのに起動時にクラッシュまたはハングする場合：

バイナリが存在し実行可能か確認：
```bash
ls -la $(which claude)
```

Linuxで欠落している共有ライブラリを確認：
```bash
ldd $(which claude) | grep "not found"
```

基本チェックを実行：
```bash
claude --version
```

## 診断ツール

より詳細なインストールと構成情報を得るには`claude doctor`を使用：

```bash
claude doctor
```

## その他の問題

上記の解決策で問題が解決しない場合：

1. Claude Codeで`/help`を入力するか「どのように...」と質問
2. 完全なドキュメントを参照
3. コミュニティサポートのために[Discord](https://www.anthropic.com/discord)に参加
