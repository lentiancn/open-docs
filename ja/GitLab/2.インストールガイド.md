# GitLab インストールガイド

## 必要環境

GitLabをインストールする前に、サーバーが次のハードウェアとソフトウェア要件を満たしていることを確認してください。小規模から中規模のチームには、最低4コアCPUと8GBのRAMを推奨しており、これは数百のレポジトリと数十人の同時ユーザーをサポートできます。大きなチームや高負荷シナリオでは、8コア以上のCPUと16GB以上のRAMを使用する必要があります。

オペレーティングシステムについては、GitLabはUbuntu、Debian、CentOS、Red Hat Enterprise Linux、Fedoraなど、さまざまなLinuxディストリビューションをサポートしています。UbuntuまたはDebianを推奨しているのは、これらのシステムのパッケージ管理が最も便利だからです。GitLabのインストールには最低20GBのディスク容量が必要で、基本的な設定には50GB以上、本番環境では数百GBが必要な場合があります。

ソフトウェアの依存関係には、SSHプロトコルサポート用のOpenSSH Server、データベース（GitLabには組み込みバージョンがあります）、メール通知用のSMTPサーバーが必要です。正しいタイムゾーンとホスト名を設定し、サーバーがインターネットにアクセスして依存パッケージをダウンロードできることを確認してください。

## インストール手順

### Ubuntu/Debian へのインストール

UbuntuまたはDebianシステムでは、公式のaptリポジトリからGitLabをインストールすることを推奨しています。これにより、最新の安定したバージョンと自動セキュリティアップデートを確保できます。まず、必要な依存パッケージをインストールします：

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

GitLab APTリポジトリを追加し、GitLab CE（コミュニティエディション）をインストールします：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

EXTERNAL_URLはGitLabにアクセスするための実際のドメインまたはIPアドレスに置き換えてください。インストールプロセスでは、PostgreSQL、Redis、GitLab自体が自動的に設定されます。

### CentOS/RHEL へのインストール

CentOSまたはRed Hat Enterprise Linuxでは、yumまたはdnfパッケージマネージャーを使用してGitLabをインストールできます。まず、必要な依存関係をインストールします：

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

GitLab YUMリポジトリを追加し、インストールします：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## 初期設定

インストール後、GitLabの基本パラメータを設定する必要があります。/etc/gitlab/gitlab.rbファイルを編集します。これがGitLabのメイン設定ファイルです。最も重要な設定項目はexternal_urlで、GitLabサービスへの外部アクセスアドレスです：

```ruby
external_url 'https://gitlab.example.com'
```

カスタムSSHポートを使用する必要がある場合は、gitlab_rails['gitlab_shell_ssh_port']パラメータを設定します。メールサーバー設定も重要で、通知とパスワードリセットメールの送信に使用されます：

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

設定を保存した後、以下のコマンドを実行して設定を適用します：

```bash
sudo gitlab-ctl reconfigure
```

このコマンドは、データベースマイグレーションや初期ユーザー作成など、すべてのGitLabコンポーネントを再設定します。全体プロセスには数分かかる場合があります。

## インストール確認

設定が完了したら、ブラウザでexternal_urlで指定したアドレスにアクセスできます。すべて正常であれば、GitLabのログインページが表示されます。最初のアクセス時には管理者アカウントのログインページが表示され、デフォルトの管理ユーザー名はrootで、初回ログイン時にパスワードを設定する必要があります。

GitLabはgitlab-ctlコマンドラインツールを提供し、サービス状態の 管理に使用できます：

```bash
# サービス状態の確認
sudo gitlab-ctl status

# ログの表示
sudo gitlab-ctl tail

# サービスの再起動
sudo gitlab-ctl restart
```

gitlab-rakeコマンドを使用してヘルスチェックを行うこともできます：

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Docker でのインストール

クイックテストや小規模デプロイには、Dockerを使用してGitLabを実行できます。まずDocker Engineをインストールし、ホストで以下のコマンドを実行します：

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

Dockerインストールでは、設定、永続化のための3つのボリュームをマウントします。最初の起動には初期化に時間がかかります。辛抱強く待った後、ブラウザでアクセスできます。

## アップグレードとメンテナンス

GitLabを定期的に最新バージョンにアップグレードすると、新機能とセキュリティ修正を取得できます。Ubuntu/Debianでのアップグレード：

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

アップグレード前にデータのバックアップを取ることを推奨しています。GitLabには組み込みバックアップツールがあります：

```bash
sudo gitlab-backup create
```

バックアップファイルはデフォルトで/var/opt/gitlab/backupsディレクトリに保存され、バックアップ保持ポリシーとリモート保存場所を設定できます。
