# GitLab FAQ

> GitLab에 대한 자주 묻는 질문

---

## 일반

### GitLab이란?

GitLab은 Git 저장소 관리, 이슈 추적, CI/CD 파이프라인 등을 제공하는 완전한 DevOps 플랫폼입니다. SaaS(gitlab.com)로 또는 자체 호스팅으로 사용할 수 있습니다.

---

## 설치

### 최소 요구사항은?

| 리소스 | 최소 |
|--------|-----|
| CPU | 2코어 |
| RAM | 4GB |
| 저장소 | 10GB |

### 설치 실패?

**해결책**:
1. 시스템 패키지 업데이트
2. 누락된 의존성 수동 설치
3. 충돌하는 패키지 확인

---

## 사용

### 병합 요청 만들기

1. 기능 브랜치를 GitLab에 푸시
2. "병합 요청 만들기" 클릭
3. 제목 및 설명 작성
4. 소스 및 대상 브랜치 선택
5. 레이블 및 검토자 추가
6. 병합 요청 제출

---

## CI/CD

### 파이프라인 트리거 안됨?

**해결책**:
1. .gitlab-ci.yml 구문 확인
2. GitLab Runner 등록 확인
3. 프로젝트 설정에서 Runner 상태 확인

---

## 문제 해결

### 502 Bad Gateway 에러?

**해결책**:
1. 유니콘/worker 프로세스 확인
2. Worker 메모리 증가
3. 로그 검토: `sudo gitlab-ctl tail`
