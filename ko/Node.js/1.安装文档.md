# Node.js 설치 가이드

Node.js는 Chrome V8 엔진 기반의 JavaScript 런타임입니다. 이 가이드에서는 모든 환경에서의 설치 방법을 설명합니다.

## 버전 정보

| 버전 | 상태 | 설명 |
|------|------|------|
| Node.js 22.x | ✅ 최신 | 최신 LTS 버전 |
| Node.js 20.x | ✅ LTS | 장기 지원 버전 |
| Node.js 18.x | ✅ LTS | 장기 지원 버전 |
| Node.js 16.x | ⚠️ 유지보수 | 보안 업데이트만 |
| Node.js 14.x | ❌ 폐기됨 | 지원 종료 |

> **중요**: 이 가이드는 primarily **Node.js 20.x LTS**를 대상으로 합니다. 새 프로젝트에는 LTS 버전을 사용하는 것이 좋습니다.

## 설치 방법

### 방법 1: Windows 설치

#### 1. 설치 프로그램 사용 (권장)

1. Node.js 설치 프로그램 다운로드: https://nodejs.org/
2. .msi 설치 프로그램 실행
3. 설치 마법사 따라 완료

#### 2. Chocolatey 사용

```powershell
# Node.js 설치
choco install nodejs

# 특정 버전 설치
choco install nodejs --version=20.10.0
```

#### 3. winget 사용

```powershell
# Node.js LTS 설치
winget install OpenJS.NodeJS.LTS

# 특정 버전 설치
winget install OpenJS.NodeJS --version=20.10.0
```

### 방법 2: macOS 설치

#### 1. Homebrew 사용 (권장)

```bash
# Node.js LTS 설치
brew install node@20

# 서비스 시작
brew services start node@20
```

#### 2. nvm 사용

```bash
# nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Node.js 설치
nvm install 20
nvm use 20
nvm alias default 20
```

#### 3. 설치 프로그램 사용

macOS 설치 프로그램 다운로드: https://nodejs.org/

### 방법 3: Linux 설치

#### 1. 패키지 관리자 사용

**Ubuntu/Debian:**

```bash
# Node.js 저장소 추가
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Node.js 설치
sudo apt-get install -y nodejs
```

**CentOS/RHEL:**

```bash
# Node.js 저장소 추가
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -

# Node.js 설치
sudo yum install -y nodejs
```

**Fedora:**

```bash
# Node.js 저장소 추가
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -

# Node.js 설치
sudo dnf install -y nodejs
```

#### 2. nvm 사용 (권장)

```bash
# nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Node.js 설치
nvm install 20
nvm use 20
```

### 방법 4: fnm 사용 (빠른 노드 버전 관리자)

```bash
# fnm 설치
curl -fsSL https://fnm.vercel.app/install | bash

# Node.js 설치
fnm install 20
fnm use 20
fnm default 20
```

### 방법 5: Docker

```bash
# Node.js 공식 이미지 사용
docker pull node:20

# 컨테이너 실행
docker run -it node:20 node --version
```

#### Dockerfile 예시

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 3000
CMD ["node", "server.js"]
```

## 설치 확인

### 버전 확인

```bash
# Node.js 버전 확인
node --version

# npm 버전 확인
npm --version
```

### npm 업데이트

```bash
# npm을 최신 버전으로 업데이트
npm install -g npm
```

## 패키지 관리

### npm 일반 명령어

```bash
# 프로젝트 초기화
npm init

# 의존성 설치
npm install

# 프로덕션 의존성 설치
npm install <패키지>

# 개발 의존성 설치
npm install <패키지> --save-dev

# 전역 설치
npm install -g <패키지>

# 의존성 업데이트
npm update

# 의존성 제거
npm uninstall <패키지>
```

### yarn 사용

```bash
# yarn 설치
npm install -g yarn

# 의존성 설치
yarn install

# 의존성 추가
yarn add <패키지>

# 전역 설치
yarn global add <패키지>
```

### pnpm 사용

```bash
# pnpm 설치
npm install -g pnpm

# 의존성 설치
pnpm install

# 의존성 추가
pnpm add <패키지>
```

## 환경 설정

### nvm으로 여러 버전 관리

```bash
# 설치된 버전 목록
nvm list

# 새 버전 설치
nvm install 21

# 버전 전환
nvm use 21

# 기본 버전 설정
nvm alias default 20
```

### 미러 설정

```bash
# npm 미러 설정
npm config set registry https://registry.npmmirror.com

# yarn 미러 설정
yarn config set registry https://registry.npmmirror.com
```

## 일반적인 문제

### Node.js와 npm 버전이 안 맞습니다?

nvm으로 버전 통일 관리:

```bash
nvm use 20
```

### 설치 실패 (권한 문제)?

```bash
# Linux/macOS는 sudo 사용
sudo npm install -g <패키지>

# 또는 npm 권한 수정
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### 빌드 실패?

캐시 비우기:

```bash
npm cache clean --force
```
