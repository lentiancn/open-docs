# Guide d'installation JIRA

JIRA est un outil de suivi des problèmes et de gestion de projet développé par Atlassian. Ce guide couvre les méthodes d'installation dans divers environnements.

## Informations de version

| Version | Statut | Description |
|---------|--------|-------------|
| JIRA 9.x | ✅ Actuelle | Dernière version |
| JIRA 8.x | ✅ Supportée | Version stable |

## Configuration requise

### Configuration matérielle

| Exigence | Minimum | Recommandé |
|----------|---------|-------------|
| CPU | 4 cœurs | 8+ cœurs |
| Mémoire | 4 Go | 16+ Go |
| Disque | 10 Go | 50+ Go |

### Configuration logicielle

| Logiciel | Version |
|----------|---------|
| Java | JDK 11 ou 17 |
| Système d'exploitation | Linux/Windows/macOS |
| Base de données | PostgreSQL/MySQL/Oracle |

## Méthodes d'installation

### Méthode 1 : Installation Windows

#### 1. Télécharger le programme d'installation

1. Visitez https://www.atlassian.com/software/jira/download
2. Téléchargez JIRA Software
3. Exécutez le programme d'installation

#### 2. Exécuter l'installation

Suivez l'assistant d'installation :
- Sélectionnez le type d'installation
- Configurez le port
- Configurez la base de données
- Créez le compte administrateur

### Méthode 2 : Installation Linux

#### 1. Télécharger le package d'installation

```bash
# Télécharger JIRA
wget https://product-downloads.atlassian.com/software/jira/downloads/atlassian-jira-software-X.X.X-x64.bin
```

#### 2. Installer

```bash
# Ajouter les permissions d'exécution
chmod +x atlassian-jira-software-X.X.X-x64.bin

# Exécuter l'installation
sudo ./atlassian-jira-software-X.X.X-x64.bin
```

#### 3. Configurer

```bash
# Démarrer JIRA
sudo /opt/atlassian/jira/bin/start-jira.sh

# Arrêter JIRA
sudo /opt/atlassian/jira/bin/stop-jira.sh
```

### Méthode 3 : Installation Docker

```bash
# Tirer l'image
docker pull atlassian/jira-software

# Exécuter le conteneur
docker run -v jira-data:/var/atlassian/application-data/jira --name="jira" -d -p 8080:8080 atlassian/jira-software
```

### Méthode 4 : Utiliser AWS

1. Recherchez JIRA sur AWS Marketplace
2. Sélectionnez le type d'instance
3. Configurez la base de données
4. Lancez l'instance

## Configuration

### Configuration de la base de données

```properties
# dbconfig.xml
<?xml version="1.0" encoding="UTF-8"?>
<jira-database-config>
  <name>defaultDS</name>
  <connection-type>default</connection-type>
  <url>jdbc:postgresql://localhost:5432/jira</url>
  <username>jira</username>
  <password>password</password>
  <pool-size>20</pool-size>
</jira-database-config>
```

### Configuration du proxy inverse

#### Nginx

```nginx
server {
    listen 80;
    server_name jira.example.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Foire aux questions

### Échec du démarrage ?

Vérifiez la version Java :

```bash
java -version
```

### Échec de connexion à la base de données ?

Vérifiez la configuration de la base de données et les informations de connexion.

### Mémoire insuffisante ?

Modifiez les paramètres de mémoire :

```bash
# setenv.sh
export JVM_MINIMUM_MEMORY="2048m"
export JVM_MAXIMUM_MEMORY="4096m"
```
