# Руководство по установке H2

## Требования к среде

- Java 11 или выше
- Любая операционная система с поддержкой Java (Windows, Linux, macOS и т.д.)

## Способы установки

### Способ 1: Скачать ZIP-пакет

1. Посетите страницу загрузки на официальном сайте: https://www.h2database.com/html/download.html
2. Скачайте версию "Platform-Independent Zip"
3. Распакуйте скачанный ZIP-файл в любой каталог, например: C:\h2 или /opt/h2

### Способ 2: Maven зависимость

Если вы используете Java-проект с Maven или Gradle, вы можете добавить зависимость напрямую:

**Maven:**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle:**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### Способ 3: Установщик Windows

1. Скачайте версию "Windows Installer"
2. Запустите установщик и следуйте мастеру для завершения установки

## Настройка переменных среды (необязательно)

Если вы скачали ZIP-пакет и хотите использовать H2 напрямую из командной строки:

**Windows:**
1. Щелкните правой кнопкой мыши "Этот компьютер" → "Свойства"
2. Нажмите "Дополнительные параметры системы"
3. Нажмите "Переменные среды"
4. В системных переменных найдите Path и нажмите "Изменить"
5. Добавьте путь к каталогу bin H2

**Linux/macOS:**
```bash
export PATH=$PATH:/opt/h2/bin
```

## Проверка установки

### Использование командной строки

После распаковки выполните в командной строке:

```bash
java -jar h2.jar
```

Это запустит веб-консоль H2.

### Использование JDBC-подключения

Для подключения к базе данных H2 в Java-коде:

```java
// Встроенный режим
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// Файловая база данных
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Использование консоли H2

1. Выполните команду: `java -jar h2.jar`
2. Откройте браузер и перейдите по адресу: http://localhost:8082
3. Имя пользователя по умолчанию: sa, пароль пустой

## Форматы URL подключения

H2 поддерживает несколько форматов URL подключения:

| Режим | Пример URL | Описание |
|-------|-----------|----------|
| Режим памяти | jdbc:h2:mem:testdb | Данные в памяти |
| Встроенный файл | jdbc:h2:file:./data/testdb | Данные в файле |
| TCP-сервер | jdbc:h2:tcp://localhost:9092/testdb | Режим клиент-сервер |
| TCP шифрованный | jdbc:h2:ssl://localhost:9092/testdb | SSL-шифрованное подключение |
| Совместимость PostgreSQL | jdbc:h2:postgres://localhost:5432/testdb | Режим совместимости с протоколом PostgreSQL |

## Распространенные проблемы

### Порт занят
H2 использует порт 9092 по умолчанию (режим TCP) и 8082 (веб-консоль). Если порт занят, вы можете указать другой порт в URL подключения.

### Проблема с версией Java
Убедитесь, что установлена Java 11 или выше. Проверить можно следующей командой:

```bash
java -version
```

### Драйвер не найден
Убедитесь, что JAR-файл H2 находится в classpath. Для Maven-проектов убедитесь, что зависимость добавлена правильно.
