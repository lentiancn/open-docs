# Claude Code 使用手册

本手册将详细介绍 Claude Code 的各种使用方式，帮助你充分发挥这个 AI 编程助手的潜力。

## 启动方式

### 终端启动

在终端中输入以下命令启动 Claude Code：

```bash
claude-code
```

或者直接指定项目目录：

```bash
claude-code /path/to/your/project
```

启动后，Claude Code 会分析当前目录的代码结构，然后你就可以开始对话了。

### IDE 内启动

在 VS Code 或 JetBrains IDE 中：
1. 点击侧边栏的 Claude Code 图标
2. 或者使用快捷键（Ctrl+Shift+C / Cmd+Shift+C）
3. 在弹出的输入框中描述你的需求

## 基本交互

### 发起请求

你可以用自然语言向 Claude Code 描述你想要完成的任务：

```
帮我添加一个用户登录功能
```

```
解释一下这个函数的作用
```

```
把这段代码重构为箭头函数
```

### 查看建议

Claude Code 会分析你的请求，然后：
1. 解释它理解你的需求
2. 列出将要执行的操作
3. 等待你确认后执行

### 确认操作

每个操作都需要你的确认：
- **是 (y)**：确认执行当前建议
- **否 (n)**：拒绝并重新描述需求
- **详情 (d)**：查看详细的修改内容
- **退出 (q)**：终止当前会话

## 核心功能

### 1. 代码阅读

Claude Code 能够理解整个代码库：

```bash
# 分析代码结构
"这个项目是怎么组织的？"

# 查找特定功能
"找到处理用户认证的代码"

# 解释复杂逻辑
"解释这段代码做了什么"
```

### 2. 代码编辑

#### 修改现有代码

```bash
# 修改单个文件
"把 userService.ts 中的 getUser 函数改为异步"

# 批量修改
"把所有 console.log 替换为 logger.info"
```

#### 创建新文件

```bash
# 创建新组件
"帮我创建一个 React 按钮组件"

# 创建测试文件
"为 auth.js 创建单元测试"
```

### 3. 终端命令

Claude Code 可以在终端中执行命令：

```bash
# 安装依赖
"运行 npm install"

# 执行脚本
"启动开发服务器"

# Git 操作
"提交当前的修改"
```

### 4. 搜索功能

```bash
# 搜索代码
"找到所有使用 fetch 的地方"

# 搜索文件
"找到配置文件"
```

## 高级用法

### 会话上下文

Claude Code 会记住当前会话的上下文，你可以：

1. **追问**：在得到答案后继续深入提问
2. **修改需求**：说"不对，我想要的是..."来调整需求
3. **多步骤任务**：分步骤完成复杂任务

### 快捷命令

以下是一些常用的快捷命令：

| 命令 | 说明 |
|------|------|
| `?` 或 `help` | 显示帮助信息 |
| `ls` | 列出当前目录文件 |
| `cd <目录>` | 切换目录 |
| `git status` | 查看 Git 状态 |
| `undo` | 撤销上一步操作 |
| `quit` 或 `exit` | 退出 Claude Code |

### 配置文件

你可以在项目根目录创建 `.claude.json` 来配置行为：

```json
{
  "allowedCommands": ["npm", "git", "docker"],
  "blockedPaths": ["/path/to/sensitive"],
  "autoApprove": false,
  "maxTokens": 4000
}
```

### 项目级指令

在项目根目录创建 `CLAUDE.md` 文件，可以给 Claude Code 持久的项目级指令：

```markdown
# 项目规范

- 使用 TypeScript 严格模式
- 组件放在 components 目录
- 样式使用 SCSS
- 所有 API 调用需要错误处理
```

## 工作流程示例

### 场景一：实现新功能

1. **描述需求**
   ```
   添加用户注册功能，包括表单验证、密码加密、数据库存储
   ```

2. **查看方案**
   Claude Code 会展示将要创建/修改的文件列表

3. **确认执行**
   确认后它会逐个创建文件

4. **代码审查**
   创建完成后，你可以让它解释代码或提出改进建议

### 场景二：修复 Bug

1. **描述问题**
   ```
   用户登录后返回 401 错误
   ```

2. **自动排查**
   Claude Code 会搜索相关代码，分析可能的原因

3. **修复问题**
   确认后进行修复

4. **验证修复**
   可以让它运行测试确认问题已解决

### 场景三：代码重构

1. **说明目标**
   ```
   把所有类组件改为函数组件
   ```

2. **评估影响**
   Claude Code 会显示受影响的所有文件

3. **逐步执行**
   确认后逐个文件进行重构

## 最佳实践

### 1. 清晰描述需求

好的描述：
```
在用户个人资料页面添加头像上传功能，图片需要压缩后存储到 S3
```

模糊的描述：
```
帮我做一个上传功能"
```

### 2. 分步处理复杂任务

复杂任务建议分步骤：
1. 先创建基础结构
2. 逐步添加功能
3. 最后处理边界情况

### 3. 利用上下文

充分利用会话上下文：
- 可以让 Claude Code "继续刚才的任务"
- 可以说"像刚才那样处理这个文件"
- 可以在一个会话中完成多个相关任务

### 4. 审查代码

不要完全依赖 AI：
- 仔细审查生成的代码
- 运行测试确认功能正常
- 检查是否符合项目规范

## 限制和注意事项

### 安全限制

- 不会执行未经确认的删除操作
- 不会自动推送代码到远程仓库
- 不会访问你明确排除的目录

### 能力限制

- 对于非常小众的技术栈可能理解不准确
- 某些特定框架的最佳实践可能不完整
- 需要你提供足够的上下文信息

### 使用建议

- 重要操作前确认修改内容
- 定期备份重要代码
- 使用版本控制跟踪变更
- 遇到问题时主动提问

通过掌握以上使用技巧，你将能够更高效地使用 Claude Code 进行开发工作。
