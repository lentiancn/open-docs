# MySQL 常見問題

## 安裝問題

### Q: MySQL 服務無法啟動怎麼辦？

A: 檢查以下內容：
1. 確認 MySQL 已正確安裝
2. 查看錯誤日誌：`mysqld --console`
3. 檢查連接埠是否被佔用：`netstat -an | grep 3306`
4. 確認資料目錄權限正確
5. 嘗試重新初始化資料目錄

### Q: 忘記 root 密碼如何重設？

A: 
1. 停止 MySQL 服務
2. 以跳過權限檢查模式啟動：`mysqld --skip-grant-tables`
3. 登入 MySQL
4. 重設密碼：`UPDATE mysql.user SET Password=PASSWORD('new_password') WHERE User='root';`
5. 重新載入權限：`FLUSH PRIVILEGES;`
6. 重新啟動 MySQL 服務

### Q: 如何更改 MySQL 連接埠號？

A: 修改 my.cnf 或 my.ini 設定檔：
```ini
[mysqld]
port=3307
```
然後重新啟動 MySQL 服務。

## 連線問題

### Q: 遠端連線被拒絕怎麼辦？

A: 
1. 確認 MySQL 允許遠端連線（檢查 user 表中使用者的 host）
2. 檢查防火牆是否允許連接埠
3. 確認 MySQL 設定中 bind-address 不是 127.0.0.1
4. 檢查 SELinux 或 AppArmor 設定

### Q: 連線逾時如何處理？

A: 
1. 檢查網路連線是否正常
2. 增加 wait_timeout 設定
3. 使用長連線或連線池
4. 檢查伺服器負載

## 資料庫操作問題

### Q: 如何處理字符集亂碼？

A: 
1. 建立資料庫時指定字符集：`CREATE DATABASE db CHARACTER SET utf8mb4;`
2. 連線時指定字符集：`SET NAMES utf8mb4;`
3. 設定檔設定預設字符集：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: 如何處理資料匯入匯出亂碼？

A: 
1. 匯出時指定字符集：`mysqldump --default-character-set=utf8mb4`
2. 匯入時指定字符集：`mysql --default-character-set=utf8mb4`

### Q: 主鍵自動編號不連續正常嗎？

A: 正常。自動編號 ID 不連續的原因包括：
- 插入失敗會消耗 ID
- 交易復原不會回收已使用的 ID
- 批次插入多筆資料時可能跳躍
- 刪除記錄不影響自動編號計數器

## 效能問題

### Q: 查詢很慢怎麼辦？

A: 
1. 使用 EXPLAIN 分析查詢執行計畫
2. 為 WHERE、JOIN、ORDER BY 涉及的資料行建立索引
3. 避免全資料表掃描
4. 最佳化查詢語句，避免 SELECT *
5. 考慮使用查詢快取（MySQL 8.0 已移除）
6. 分表分庫分散壓力

### Q: 如何查看慢查詢？

A: 
1. 開啟慢查詢日誌：
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. 查看慢查詢日誌檔案位置：
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: 記憶體使用過高如何最佳化？

A: 
1. 調整 innodb_buffer_pool_size（建議設定為可用記憶體的 70%）
2. 最佳化查詢減少不必要的掃描
3. 限制連線數
4. 清理不必要的快取

## 備份復原問題

### Q: 備份檔案太大怎麼辦？

A: 
1. 使用壓縮備份：`mysqldump | gzip > backup.sql.gz`
2. 分庫分表備份
3. 使用 xtrabackup 進行實體備份
4. 定期清理歷史資料

### Q: 如何實現增量備份？

A: 
1. 開啟二進制日誌：`log-bin=mysql-bin`
2. 定期備份二進制日誌
3. 使用 mysqlbinlog 工具復原增量資料

## 複製問題

### Q: 主從複製延遲過高怎麼辦？

A: 
1. 檢查網路延遲
2. 調整 slave_parallel_workers 增加並行複製
3. 確認從庫硬體效能足夠
4. 最佳化主庫寫入速度
5. 檢查複製篩選規則

### Q: 如何處理主從資料不一致？

A: 
1. 重新同步從庫
2. 使用 pt-table-checksum 檢查不一致
3. 使用 pt-table-sync 修復資料
4. 重新設定主從複製

## 安全問題

### Q: 如何加強 MySQL 安全？

A: 
1. 設定強密碼原則
2. 限制 root 使用者遠端登入
3. 刪除匿名帳戶
4. 刪除測試資料庫
5. 定期更新 MySQL 版本
6. 啟用 SSL/TLS 加密連線
7. 設定防火牆規則
8. 最小化授權原則

### Q: 如何防止 SQL 注入？

A: 
1. 使用預處理語句（Prepared Statements）
2. 驗證和過濾使用者輸入
3. 限制資料庫使用者權限
4. 避免動態拼接 SQL
5. 使用預存程序
