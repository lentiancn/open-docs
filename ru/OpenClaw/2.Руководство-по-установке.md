# Руководство по установке OpenClaw

Это руководство содержит подробные инструкции по установке OpenClaw на различных операционных системах.

## Требования

- **Node.js**: версия 22 или выше
- **Операционная система**: macOS, Linux или Windows
- **Необязательно**: Docker (для функции песочницы)

## Способы установки

### macOS / Linux (Рекомендуется)

Самый простой способ установить OpenClaw — использовать скрипт установки:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Этот скрипт автоматически определяет вашу системную среду, устанавливает необходимые зависимости и настраивает OpenClaw.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Прямая установка через npm

Если Node.js уже установлен, можно использовать npm напрямую:

```bash
npm install -g openclaw@latest
```

### Установка из исходного кода

Если вам нужна последняя версия для разработки или требуется пользовательская установка:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Настройка после установки

### Запуск мастера настройки

После установки запустите мастер для настройки аутентификации, параметров Gateway и дополнительных каналов:

```bash
openclaw onboard --install-daemon
```

Мастер проведёт вас через:
- Настройку API-ключа
- Параметры порта Gateway
- Подключение каналов (WhatsApp, Telegram, Discord и т.д.)
- Настройки безопасности и разрешений

### Проверка состояния Gateway

Если вы установили сервис, как показано выше, Gateway уже должен работать:

```bash
openclaw gateway status
```

### Запуск Gateway в foreground

Если вы хотите запустить Gateway в foreground (полезно для тестирования и устранения неполадок):

```bash
openclaw gateway --port 18789
```

### Открытие интерфейса управления

```bash
openclaw dashboard
```

Или просто откройте в браузере: `http://127.0.0.1:18789/`

## Переменные окружения

Если вам нужно настроить расположение конфигурации или каталога состояния, можно использовать следующие переменные окружения:

- `OPENCLAW_HOME`: устанавливает домашний каталог для внутреннего разрешения путей
- `OPENCLAW_STATE_DIR`: переопределяет каталог состояния
- `OPENCLAW_CONFIG_PATH`: переопределяет путь к файлу конфигурации

## Проверка установки

После установки вы можете:

1. Запустить `openclaw --version` для подтверждения версии
2. Запустить `openclaw doctor` для проверки состояния системы
3. Открыть интерфейс управления для отправки тестового сообщения

## Распространённые проблемы при установке

### Проверка версии Node.js

```bash
node --version
```

Убедитесь, что версия 22 или выше. Если нет, сначала обновите Node.js.

### Git не найден в Windows

Убедитесь, что Git установлен и добавлен в системный PATH. Скачать можно с [git-scm.com](https://git-scm.com).

### Ошибки разрешений

При возникновении ошибок разрешений попробуйте:

```bash
sudo npm install -g openclaw@latest
```
