# OpenClaw 安装指南

OpenClaw 支持多种安装方式，您可以根据自己的需求选择最合适的方法。

## 环境要求

- **Node.js**：22.x 或更高版本
- **操作系统**：macOS、Linux、Windows（通过 WSL2）
- **可选**：Tailscale（用于远程访问）

## 快速安装

### macOS / Linux

使用安装脚本是最简单的方式：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

或者使用 npm/pnpm：

```bash
npm install -g openclaw@latest
# 或
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## 初始化配置

安装完成后，运行引导向导进行初始配置：

```bash
openclaw onboard --install-daemon
```

向导会引导您完成以下设置：

1. **认证配置**：设置 API 密钥或 OAuth
2. **Gateway 设置**：配置网关端口和服务
3. **渠道连接**：连接您想要使用的通讯平台
4. **技能安装**：选择和安装需要的技能

## 安装为系统服务

使用 `--install-daemon` 参数可以将 OpenClaw 安装为后台服务：

```bash
openclaw onboard --install-daemon
```

这会在您的系统上创建一个用户级服务（launchd 或 systemd），确保 OpenClaw 始终运行。

## 手动运行 Gateway

如果您不想安装服务，也可以手动启动 Gateway：

```bash
openclaw gateway --port 18789 --verbose
```

## Docker 安装

如果您更倾向于使用 Docker，可以参考官方 Docker 部署文档。

## 从源码构建

如果您想参与开发或自定义 OpenClaw：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

注意：pnpm 用于构建，TypeScript 可以直接通过 tsx 运行。

## 验证安装

安装完成后，您可以：

1. **检查 Gateway 状态**：
   ```bash
   openclaw gateway status
   ```

2. **打开 Control UI**：
   ```bash
   openclaw dashboard
   ```
   或者在浏览器中访问 `http://127.0.0.1:18789/`

3. **发送测试消息**（需要先配置渠道）：
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## 更新 OpenClaw

OpenClaw 有三个发布渠道：

- **stable**：正式版发布（推荐）
- **beta**：测试版
- **dev**：开发版

切换渠道：

```bash
openclaw update --channel stable|beta|dev
```

更新后建议运行：

```bash
openclaw doctor
```

## 常用命令

| 命令 | 说明 |
|------|------|
| `openclaw onboard` | 运行初始配置向导 |
| `openclaw gateway start` | 启动 Gateway |
| `openclaw gateway status` | 查看 Gateway 状态 |
| `openclaw dashboard` | 打开 Control UI |
| `openclaw agent --message "..."` | 直接发送消息给助手 |
| `openclaw message send --to xxx --message "xxx"` | 发送消息 |
| `openclaw doctor` | 诊断和修复问题 |
| `openclaw update` | 更新 OpenClaw |

## 环境变量

如果需要自定义配置位置：

- `OPENCLAW_HOME`：设置主目录
- `OPENCLAW_STATE_DIR`：覆盖状态目录
- `OPENCLAW_CONFIG_PATH`：覆盖配置文件路径

## 常见问题

**安装失败怎么办？**

- 确保 Node.js 版本 >= 22
- 检查网络连接
- 运行 `openclaw doctor` 进行诊断

**如何卸载？**

- 停止 Gateway 服务
- 删除安装目录
- 清理配置文件
