# Руководство пользователя

## Основы конвейера

Конвейер — это основная функция Jenkins, которая автоматизирует весь процесс разработки программного обеспечения от сборки до тестирования и развертывания.

## Что такое конвейер

Конвейер — это набор плагинов Jenkins для реализации и интеграции конвейеров непрерывной доставки в Jenkins. Проще говоря, конвейер автоматизирует весь процесс доставки программного обеспечения от системы контроля версий к пользователю.

Определения конвейера записываются в текстовый файл (Jenkinsfile), который может быть помещен в репозиторий исходного кода проекта. Это реализует философию «конвейера как кода».

### Преимущества конвейера

- **На основе кода**: конвейеры версионируются как код, что позволяет проверять и повторять
- **Устойчивость**: конвейеры переживают как запланированные, так и незапланированные перезапуски Jenkins
- **Приостанавливаемость**: конвейеры могут останавливаться и ожидать ввода или утверждения человека
- **Универсальность**: поддерживает сложные требования CD реального мира, включая параллельное выполнение и циклы
- **Расширяемость**: плагины могут расширять DSL конвейера и интегрировать другие инструменты

### Концепции конвейера

#### Узел

Узел — это машина, которая является частью среды Jenkins и способна выполнять конвейер. Узлы могут быть главными или агентскими машинами.

#### Этап

Этап определяет концептуально отдельное подмножество задач во всем конвейере, например этапы «Сборка», «Тестирование» и «Развертывание». Этапы используются для визуализации состояния выполнения конвейера.

#### Шаг

Шаг — это фундаментальное действие в конвейере Jenkins, которое указывает Jenkins, что делать в определенный момент времени. Например, для выполнения команды shell используется шаг `sh`.

### Синтаксис конвейера

#### Декларативный конвейер

Декларативный конвейер — это более новый синтаксис с богатыми функциями, который легче писать и читать:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Сборка') {
            steps {
                echo 'Сборка приложения...'
            }
        }
        stage('Тестирование') {
            steps {
                echo 'Запуск тестов...'
            }
        }
        stage('Развертывание') {
            steps {
                echo 'Развертывание приложения...'
            }
        }
    }
}
```

#### Скриптовый конвейер

Скриптовый конвейер использует синтаксис Groovy, более гибкий, но требующий большего опыта программирования:

```groovy
node {
    stage('Сборка') {
        echo 'Сборка приложения...'
    }
    stage('Тестирование') {
        echo 'Запуск тестов...'
    }
    stage('Развертывание') {
        echo 'Развертывание приложения...'
    }
}
```

## Создание первого конвейера

### Использование Blue Ocean

Blue Ocean — современный интерфейс Jenkins, который делает создание конвейеров более интуитивным:

1. Установите плагин Blue Ocean (если не установлен заранее)
2. Нажмите «Открыть Blue Ocean»
3. Нажмите «Создать конвейер»
4. Выберите расположение репозитория кода (GitHub, GitLab и т.д.)
5. Выберите репозиторий и ветку
6. Blue Ocean автоматически определит Jenkinsfile или поможет создать его

### Использование Jenkinsfile

Создайте Jenkinsfile в своем проекте, и Jenkins автоматически его распознает:

1. Создайте Jenkinsfile в корне проекта
2. Определите этапы конвейера
3. Поместите в систему контроля версий
4. Создайте задачу конвейера в Jenkins и свяжите с репозиторием

## Управление учетными данными

Jenkins требуется доступ к различным внешним ресурсам, таким как репозитории кода и цели развертывания, что требует управления учетными данными.

### Добавление учетных данных

1. Перейдите в «Manage Jenkins» > «Manage Credentials»
2. Нажмите «Add Credentials»
3. Выберите тип учетных данных (имя пользователя и пароль, SSH с закрытым ключом, секретный файл и т.д.)
4. Заполните информацию об учетных данных
5. Установите ID и описание для учетных данных

### Использование учетных данных в конвейере

Используйте шаг withCredentials:

```groovy
pipeline {
    agent any
    stages {
        stage('Использовать учетные данные') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "Имя пользователя: $USER"
                }
            }
        }
    }
}
```

## Триггеры сборки

Автоматические сборки требуют настройки триггеров. Jenkins поддерживает несколько методов триггеров.

### Опрос репозитория

Регулярная проверка наличия изменений в исходном коде:

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // Проверять каждые 5 минут
    }
    stages {
        stage('Сборка') {
            steps {
                echo 'Обнаружено изменение кода, запуск сборки'
            }
        }
    }
}
```

### Webhook-триггер

Запуск сборок через webhook репозитория для большей эффективности:

1. Установите соответствующий плагин (например, GitHub Hook Trigger)
2. Настройте webhook репозитория для указывания на Jenkins
3. Сборка автоматически запускается при помещении кода

### Запланированные сборки

Автоматическое выполнение по расписанию:

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // Выполнять в 2 часа ночи ежедневно
    }
    stages {
        stage('Запланированная сборка') {
            steps {
                echo 'Запланированная сборка выполняется'
            }
        }
    }
}
```

## Использование агентов и контейнеров

### Настройка агентов

Jenkins поддерживает распределенные сборки, распределяя задачи сборки между несколькими машинами:

1. Перейдите в «Manage Jenkins» > «Manage Nodes»
2. Добавьте агентский узел и настройте информацию о подключении
3. Укажите агента в конвейере

### Использование Docker в конвейере

Декларативный конвейер может выполнять сборки в Docker-контейнерах:

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Сборка') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## Отчеты о тестировании

Jenkins может интегрироваться с различными тестовыми фреймворками для создания визуальных отчетов о тестировании.

### Отчеты JUnit

```groovy
pipeline {
    agent any
    stages {
        stage('Тестирование') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### Другие отчеты о тестировании

Jenkins поддерживает различные плагины отчетов о тестировании:

- TestNG
- Cobertura (покрытие кода)
- JaCoCo (покрытие кода)
- ESLint (анализ фронтенд-кода)

## Практика развертывания

### Развертывание в различные среды

Типичный конвейер многоэтапного развертывания:

```groovy
pipeline {
    agent any
    stages {
        stage('Сборка') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('Развертывание в тестовую среду') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('Развертывание в производственную среду') {
            when {
                branch 'main'
            }
            steps {
                input message: 'Подтвердить развертывание в производство?', ok: 'Развернуть'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### Сине-зеленое развертывание

Реализуйте сине-зеленое развертывание с Jenkins для обновлений без простоя:

1. Поддерживайте две идентичные среды (синюю и зеленую)
2. Переключите трафик на новую версию
3. Быстро откатитесь при возникновении проблем

## Рекомендуемые плагины

### Необходимые плагины

- Git: интеграция с репозиторием кода
- Pipeline: поддержка конвейера
- Blue Ocean: современный интерфейс
- Credentials: управление учетными данными
- Configuration as Code: конфигурация как код

### Часто используемые плагины

- Docker: интеграция с Docker
- Kubernetes: интеграция с Kubernetes

## Рекомендации

### Рекомендации по Jenkinsfile

- Хранить Jenkinsfile в системе контроля версий
- Использовать синтаксис декларативного конвейера
- Делать конвейеры простыми и понятными
- Разумно разделять этапы
- Добавлять необходимые механизмы тайм-аута и повтора

### Рекомендации по безопасности

- Регулярно обновлять Jenkins и плагины
- Ограничивать права анонимных пользователей
- Использовать учетные данные для конфиденциальной информации
- Настраивать соответствующий контроль доступа
- Включать аудит безопасности

### Оптимизация производительности

- Правильно настраивать узлы сборки
- Использовать инкрементные сборки для экономии времени
- Настраивать кэш сборки
- Регулярно очищать старые записи сборки
- Использовать параллельные этапы для ускорения сборки

## Устранение неполадок

### Просмотр журналов сборки

При сбое сборки сначала проверьте журналы сборки:

1. Перейдите на страницу задачи
2. Нажмите на номер завершившейся сборки
3. Проверьте вывод консоли

### Решение распространенных проблем

- Сборка зависла: проверьте наличие ожидающего ввода или блокировки ресурсов
- Учетные данные недействительны: проверьте, не истекли ли учетные данные или не были ли отозваны
- Конфликты плагинов: попробуйте отключить недавно установленные плагины
- Недостаточно памяти: настройте параметры кучи JVM

## Резюме

Конвейер Jenkins — основной инструмент для реализации CI/CD. Используя это руководство, вы должны уметь:

- Понимать основные концепции конвейера
- Создавать и настраивать задачи конвейера
- Управлять учетными данными и триггерами
- Интегрировать инструменты тестирования и развертывания
- Применять рекомендации для оптимизации рабочих процессов

Непрерывное обучение и практика — ключ к освоению Jenkins. Продолжайте изучать и оптимизировать в реальных проектах.
