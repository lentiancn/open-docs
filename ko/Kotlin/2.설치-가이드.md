# Kotlin 설치 가이드

## 전제 조건

Kotlin을 설치하기 전에 컴퓨터가 기본 시스템 요구 사항을 충족하는지 확인하세요. Kotlin은 Windows, macOS, Linux 및 기타 유사 Unix 시스템 등 여러 운영 체제에서 실행될 수 있습니다. 사용하는 운영 체제에 관계없이 개발 도구를 설치하고 컴파일된 프로그램을 실행할 수 있는 충분한 디스크 공간이 있는지 확인하세요.

Windows 사용자의 경우 최상의 환경을 위해 Windows 10 이상을 권장합니다. macOS 사용자는 macOS 10.14 이상이 필요합니다. Linux 사용자는 Ubuntu, Fedora, Debian 등 주요 배포판을 사용할 수 있습니다.

## IDE를 통한 설치

### IntelliJ IDEA

IntelliJ IDEA는 JetBrains의 플래그십 통합 개발 환경으로 Kotlin에 가장 완벽한 지원을 제공합니다. 주요 목표가 Kotlin 개발인 경우 IntelliJ IDEA가 최상의 선택입니다.

IntelliJ IDEA는 Community(무료 및 오픈소스)와 Ultimate(고급 기능이 있는 기업 사용자용)의 두 가지 버전이 있습니다. Community 버전은 무료이며 Kotlin 개발에 충분한 기능을 제공합니다. Ultimate 버전은 고급 데이터베이스 도구 및 기업 보안 기능 등 추가 기업 기능을 제공합니다.

설치하려면 JetBrains 공식 웹사이트(https://www.jetbrains.com/idea/download/)를 방문하여 운영 체제에 맞는 설치 프로그램을 다운로드하세요. 다운로드 후 설치 마법사를 실행하여 설치를 완료하세요. 설치 중에 설치 경로를 선택하고 데스크톱 바로 가기를 생성할 수 있습니다.

IntelliJ IDEA를 처음 시작하면 IDE가 자동으로 감지하여 Kotlin 플러그인 설치를 요청합니다. 자동 감지가되지 않으면 플러그인 마켓플레이스에서 Kotlin을 수동으로 검색하여 설치할 수 있습니다. 설치 후 IDE를 다시 시작하면 Kotlin 개발을 시작할 수 있습니다.

### Android Studio

Android Studio는 Kotlin 지원이 내장된 Google의 공식 Android 개발 IDE입니다. 주로 Android 개발을 하는 경우 Android Studio가 이상적인 선택입니다.

Android Studio를 다운로드하려면 Android 개발자 웹사이트(https://developer.android.com/studio)를 방문하세요. 다운로드 후 설치 마법사를 실행하여 설치를 완료하세요. Android Studio는 기본적으로 Kotlin 플러그인이 포함되어 있어 추가 구성 없이 개발을 시작할 수 있습니다.

Android Studio는 매년 여러 업데이트 버전을 출시하며 각 버전에는 최신 Kotlin 지원과 Android 개발 도구가 포함되어 있습니다. 최상의 개발 환경과 최신 언어 기능을 얻으려면 버전을 최신 상태로 유지하는 것이 좋습니다.

## 명령줄을 통한 설치

IDE를 사용하는 외에도 Kotlin 명령줄 컴파일러를 설치할 수 있습니다. 이 접근 방식은 가벼운 개발 환경을 선호하거나 서버 측 개발에 Kotlin이 필요한 개발자에게 적합합니다.

### 수동 설치

먼저 Kotlin GitHub 릴리스 페이지(https://github.com/JetBrains/kotlin/releases)를 방문하여 최신 버전의 컴파일러를 다운로드하세요. 다운로드 파일은 zip 압축 파일이므로 원하는 디렉터리에 푸세요.

압축을 푼 후 시스템의 PATH 환경 변수에 kotlinc/bin 디렉터리를 추가하면 명령줄에서 kotlinc 명령을 사용할 수 있습니다. Windows에서는 시스템 속성 > 환경 변수를 통해 경로를 추가합니다. macOS 및 Linux에서는 ~/.bashrc 또는 ~/.zshrc 파일을 편집하고 다음 줄을 추가하세요:

export PATH="$PATH:/path/to/kotlinc/bin"

경로를 추가한 후 새 터미널 창을 열고 kotlinc -version 명령을 입력하세요. 버전 정보가 표시되면 설치가 성공한 것입니다.

### SDKMAN! 사용

SDKMAN!은 Java 개발 도구 버전을 관리하는 도구로 유닉스 유사 시스템에서 편리하게 사용할 수 있습니다. macOS, Linux, Cygwin, FreeBSD 또는 Solaris를 사용하는 경우 SDKMAN!이 Kotlin을 설치하는 권장 방법입니다.

먼저 터미널에서 다음 명령을 실행하여 SDKMAN!을 설치하세요:

curl -s "https://get.sdkman.io" | bash

설치 후 환경 구성을 다시 로드하세요:

source ~/.sdkman/bin/sdkman-init.sh

이제 sdk 명령을 사용하여 Kotlin을 설치할 수 있습니다:

sdk install kotlin

설치。 설치가 완료됩니다. kotlinc -version으로 설치를 확인할 수 있습니다.

### Homebrew 사용

macOS에서는 Homebrew 패키지 관리자를 사용하여 Kotlin을 설치할 수도 있습니다. 먼저 Homebrew가 설치되어 있는지 확인한 후 다음을 실행하세요:

brew update
brew install kotlin

Homebrew가 자동으로 최신 Kotlin 컴파일러를 다운로드하여 설치합니다. kotlinc -version으로 설치를 확인하세요.

### Snap 사용

Ubuntu 16.04 이상에서는 Snap을 사용하여 Kotlin을 설치할 수 있습니다:

sudo snap install --classic kotlin

Snap 설치 방식은 Kotlin을 시스템 전체에서 명령줄로 호출할 수 있는 애플리케이션으로 설치합니다.

## 첫 번째 Kotlin 프로그램 만들기

설치 방법에 관계없이 다음 단계로 첫 번째 Kotlin 프로그램을 만들고 실행할 수 있습니다.

먼저 다음 내용으로 hello.kt라는 파일을 만드세요:

fun main() {
    println("Hello, World!")
}

가장 간단한 Kotlin 프로그램으로, Kotlin 프로그램의 진입점인 main 함수를 정의합니다. println 함수는 콘솔에 텍스트를 출력합니다.

명령줄 컴파일러를 사용하는 경우 이 프로그램을 컴파일하고 실행하려면 두 단계가 필요합니다. 먼저 컴파일합니다:

kotlinc hello.kt -include-runtime -d hello.jar

-Include-runtime 매개변수는 생성된 jar 파일에 Kotlin 런타임 라이브러리를 패키징하여 독립 실행형으로 만듭니다. -d 매개변수는 출력 파일 경로를 지정합니다.

컴파일 후 프로그램을 실행합니다:

java -jar hello.jar

콘솔에 "Hello, World!"가 출력되어야 합니다.

독립 실행형 jar 파일을 생성하고 싶지 않은 경우 kotlinc의 대화형 실행 기능을 사용할 수도 있습니다:

kotlinc -script hello.kt

또는 kotlin 명령을 사용하여 컴파일된 클래스 파일을 직접 실행할 수도 있습니다:

kotlin -classpath hello.jar HelloKt

여기서 HelloKt는 hello.kt 파일에 대해 Kotlin 컴파일러가 생성한 기본 클래스 이름입니다.

## REPL 사용

Kotlin은 Kotlin 코드를 입력하고 즉시 결과를 볼 수 있는 대화형 REPL(Read-Eval-Print Loop) 환경을 제공합니다. 이는 언어 기능을 배우거나 코드 스니펫을 빠르게 테스트하는 데 유용합니다.

REPL을 시작하려면 kotlinc를 매개변수 없이 실행하세요:

kotlinc

이렇게 하면 REPL 모드로 들어갑니다. 유효한 Kotlin 코드를 입력하고 Enter를 치면 결과를 볼 수 있습니다:

>>> val x = 10
>>> val y = 20
>>> x + y
30

:quit를 입력하거나 Ctrl+C를 눌러 REPL을 종료하세요.

## Kotlin 스크립트

Kotlin은 스크립트 언어로도 사용할 수 있습니다. Kotlin 스크립트 파일은 .kts 확장자를 가지며 파일의 최상위 코드는 실행 가능한 코드로 처리됩니다.

예를 들어 list_folders.kts라는 스크립트를 만드세요:

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

스크립트를 실행합니다:

kotlinc -script list_folders.kts -- -d /path/to/folder

## 빌드 도구 구성

실제 프로젝트에서는 일반적으로 Gradle 또는 Maven을 빌드 도구로 사용합니다. 둘 다 Kotlin 프로젝트를 기본적으로 지원합니다.

### Gradle

Gradle은 Kotlin 개발에서 가장 인기 있는 빌드 도구입니다. Kotlin 팀은 빌드 스크립트를 Kotlin 구문으로 작성할 수 있는 Kotlin DSL도 제공합니다.

기본 Kotlin Gradle 프로젝트를 만들려면 프로젝트에 Kotlin 플러그인을 추가하세요. build.gradle 파일에 다음 내용이 포함되어야 합니다:

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

또는 Kotlin DSL 사용:

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

그런 다음 일반적으로 src/main/kotlin인 소스 코드 디렉터리 구조를 만드세요. Kotlin 소스 파일을 추가하고 gradle build 명령을 사용하여 프로젝트를 컴파일하세요.

### Maven

Maven은 Kotlin 프로젝트도 지원하는 또 다른 인기 있는 Java 프로젝트 빌드 도구입니다. pom.xml 파일에 Kotlin 종속성을 추가하세요.

## 설치 확인

설치 후 모든 것이 정상적으로 작동하는지 확인할 수 있습니다.

먼저 Kotlin 컴파일러를 확인합니다:

kotlinc -version

"Kotlin compiler version 2.3.10"과 같은 출력이 표시되어야 합니다.

그런 다음 Kotlin 표준 라이브러리를 확인합니다:

kotlin -e 'println("Hello Kotlin!")'

모두 작동하면 설치가 성공적이고 Kotlin 여정을 시작할 수 있습니다.

## 다음 단계

Kotlin을 설치한 후 학습을 계속하는 것이 좋습니다: 먼저 공식 Kotlin 투어를 읽어 기본 언어 기능을 이해하세요. 그런 다음 Android 개발, 서버 측 개발 또는 다중 플랫폼 개발 등 관심 있는 개발 방향을 선택하세요. 마지막으로 실제 프로젝트를 연습하여 배운 것을 적용하세요.

Kotlin은 공식 문서, 온라인 튜토리얼, 비디오 과정 등 풍부한 학습 리소스를 제공합니다. 이러한 리소스에 대한 링크는 Kotlin 웹사이트에서 찾을 수 있습니다.
