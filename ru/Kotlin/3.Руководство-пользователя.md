# Руководство пользователя Kotlin

## Переменные и базовые типы

Kotlin использует два ключевых слова для объявления переменных: var и val. var объявляет изменяемую переменную, аналогичную обычной переменной в Java. val объявляет неизменяемую переменную, аналогичную переменной final в Java. В Kotlin предпочтительное использование val — это лучшая практика, которая делает код более безопасным и понятным.

При объявлении переменных можно явно указать тип или позволить компилятору вывести тип:

val name: String = "Kotlin"
val age = 10 // Компилятор выводит тип Int
var count = 0 // Изменяемая переменная

Базовые типы Kotlin включают: целые числа (Int, Long, Short, Byte), числа с плавающей точкой (Double, Float), символы (Char), логические значения (Boolean) и строки (String). В отличие от Java, все базовые типы в Kotlin являются объектами с соответствующими методами-членами.

Целые числа в Kotlin по умолчанию имеют тип Int. Для представления больших чисел используйте суффикс Long:

val longNumber = 100L // Тип Long

Строки в Kotlin мощные и поддерживают строковые шаблоны, позволяющие встраивать переменные или выражения прямо в строки:

val name = "World"
println("Hello, $name!") // Выводит: Hello, World!
println("Length: ${name.length}") // Выводит: Length: 5

## Коллекции

Стандартная библиотека Kotlin предоставляет богатые типы коллекций: List (список), Set (множество) и Map (словарь). Каждый тип коллекции имеет неизменяемые и изменяемые версии.

Неизменяемые коллекции нельзя добавлять, удалять или изменять после создания:

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

Изменяемые коллекции допускают изменение элементов:

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Коллекции Kotlin предоставляют богатые функциональные операции, такие как map, filter и reduce:

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## Управление потоком

Управление потоком в Kotlin включает условные выражения, циклы и выражения перехода.

Выражение if в Kotlin возвращает значение и может быть напрямую присвоено переменной:

val max = if (a > b) a else b

Выражение when — мощный инструмент Kotlin для множественного сопоставления с образцом, который может соответствовать различным условиям:

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Циклы в Kotlin включают циклы for и while. Циклы for могут итерировать по диапазонам, массивам и коллекциям:

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## Функции

Kotlin использует ключевое слово fun для объявления функций. Функции могут указывать типы параметров и типы возврата:

fun greet(name: String): String {
    return "Hello, $name!"
}

Если тело функции содержит только одно выражение, можно использовать синтаксис одноexpressionного тела:

fun greet(name: String) = "Hello, $name!"

Kotlin поддерживает аргументы по умолчанию и именованные аргументы, что значительно сокращает необходимость перегрузки функций:

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin также поддерживает переменное количество аргументов с помощью ключевого слова vararg:

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## Классы и объекты

Объявления классов в Kotlin более лаконичны, чем в Java. По умолчанию классы в Kotlin являются финальными и не могут наследоваться. Чтобы сделать класс наследуемым, используйте ключевое слово open:

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Конструкторы Kotlin включают основные и вторичные конструкторы. Основной конструктор объявляется сразу после имени класса:

class Person(val name: String, val age: Int)

Если нужно выполнить логику в конструкторе, используйте блок init:

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Data-классы (data class) в Kotlin автоматически генерируют методы equals, hashCode, toString и copy, что делает их идеальными для классов, хранящих данные:

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Защита от null

Защита от null — одна из важнейших функций Kotlin. В Kotlin система типов различает nullable и non-nullable типы, что фундаментально уменьшает количество NullPointerException.

По умолчанию типы не допускают null:

val name: String = "Kotlin" // Не может быть null
val name: String = null // Ошибка компиляции

Чтобы разрешить переменной быть null, используйте nullable типы (?):

val name: String? = null // Может быть null

При доступе к свойствам или методам nullable типов используйте безопасный оператор вызова (?.) или утверждение non-null (!!):

val length = name?.length // Возвращает null, если name = null
val length = name!!.length // Выбрасывает исключение, если name = null

Оператор Элвис (?:) предоставляет лаконичный способ обработки значений по умолчанию:

val length = name?.length ?: 0 // Возвращает 0, если name = null

## Функции расширения и свойства расширения

Kotlin позволяет добавлять новые функции и свойства к существующим классам без наследования или изменения оригинального класса. Это называется функциями расширения и свойствами расширения.

Функции расширения объявляются путём добавления имени класса перед именем функции:

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

Свойства расширения также могут добавлять новые свойства к существующим классам:

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## Функции высшего порядка и лямбды

Функции высшего порядка — это функции, которые принимают функции в качестве параметров или возвращают функции. Лямбда-выражения — основной способ реализации этой функции.

Синтаксис лямбда-выражений использует фигурные скобки:

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

Если лямбда является последним параметром функции, её можно вынести за скобки:

list.filter { it > 0 }

В лямбда-выражениях it представляет неявное имя для единственного параметра. Если параметр только один, можно опустить объявление параметра.

## Корутины

Корутины — это центральная функция асинхронного программирования в Kotlin. По сравнению с потоками, корутины намного легче и могут выполнять несколько корутин в одном потоке.

launch — распространённый способ запуска корутин, который запускает новую корутину в фоновом режиме:

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

Функции suspend могут вызываться только внутри корутин:

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        // Выполнение длительной операции
    }
}

Корутины Kotlin предоставляют несколько диспетчеров для управления тем, в каком потоке выполняются корутины: Dispatchers.Main для UI-операций, Dispatchers.IO для I/O-операций, Dispatchers.Default для CPU-интенсивных задач.

## Запечатлённые классы и перечисления

Запечатлённые классы — это ограниченные иерархии классов, которые ограничивают количество подклассов, которые могут наследоваться. Все подклассы должны быть объявлены в том же файле. Это очень полезно для представления конечных состояний или типов результатов:

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

Выражение when может полностью использовать функции запечатлённых классов, при этом компилятор проверяет, обработаны ли все возможные случаи:

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

Перечисления представляют конечный набор постоянных значений:

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

Перечисления могут иметь свойства и методы:

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## Интерфейсы и абстрактные классы

Интерфейсы в Kotlin могут содержать реализации по умолчанию, что делает их более гибкими:

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
}

Классы могут реализовывать несколько интерфейсов:

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## Выражения объектов и объявления объектов

Выражения объектов создают экземпляры анонимных классов, аналогично анонимным внутренним классам в Java:

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // Обработка события клика
    }
})

Объявления объектов создают одиночки:

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## Сопровождающие объекты

Каждый класс может содержать сопровождающий объект, аналогичный статическим членам в Java:

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

К сопровождающему объекту можно обращаться напрямую через имя класса:

MyClass.TAG
MyClass.create()

## Проверки и преобразования типов

Оператор is используется для проверки типов, аналогично instanceof в Java:

if (obj is String) {
    // В этой ветке obj автоматически преобразуется в String
    println(obj.length)
}

Умные приведения автоматически преобразуют типы, которые были проверены. Если компилятор может определить тип переменной в определённой ветке, он автоматически выполняет преобразование.

Если умные приведения неприменимы, можно использовать явное преобразование:

val str = obj as String

Безопасное преобразование типов использует as?, которое возвращает null при неудачном преобразовании:

val str = obj as? String // Возвращает null при неудачном преобразовании
