# OpenClaw 사용 매뉴얼

이 매뉴얼에서는 OpenClaw의 일상적인 사용법과 주요 기능을 설명합니다.

## 기본 개념

### Gateway

Gateway는 세션 관리, 메시지 라우팅 및 채널 연결을 담당하는 OpenClaw의 핵심 구성 요소입니다. 단일 Gateway 프로세스만 실행하면 모든 채널 연결 요청을 자동으로 처리합니다.

### 채널

채널은 AI 도우미와 통신하는 데 사용하는 채팅 앱입니다. OpenClaw는 WhatsApp, Telegram, Discord, iMessage 등 여러 채널을 동시에 연결할 수 있습니다.

### 에이전트

에이전트는 AI 도우미의 인스턴스입니다. 각 에이전트에는 자체 작업 공간과 세션이 있습니다. 다양한 작업을 독립적으로 처리하도록 여러 에이전트를 구성할 수 있습니다.

## 일상적인 작업

### Gateway 시작 및 중지

```bash
# Gateway 시작(백그라운드 서비스)
openclaw gateway start

# Gateway 중지
openclaw gateway stop

# Gateway 재시작
openclaw gateway restart

# 상태 확인
openclaw gateway status

# 전경에서 실행(디버깅용)
openclaw gateway --port 18789
```

### 메시지 전송

```bash
# 직접 메시지 전송
openclaw message send --target +15555550123 --message "안녕하세요"

# 그룹으로 전송
openclaw message send --target "그룹 이름" --message "모든 분 안녕하세요"
```

### 구성 관리

```bash
# 현재 구성 보기
openclaw config get

# 구성 값 설정
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# 구성 값 제거
openclaw config unset channels.whatsapp.enabled
```

## 구성 관리

### 구성 파일 위치

구성 파일은 `~/.openclaw/openclaw.json`에 있습니다.

### 핫 리로드

OpenClaw는 구성 핫 리로드를 지원합니다 — Gateway를 수동으로 다시 시작하지 않고 구성 파일을 수정할 수 있습니다. 대부분의 구성 변경 사항은 자동으로 적용됩니다.

### 일반적인 구성 예

#### 허용된 메시지 소스 설정

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### 그룹 채팅 멘션 규칙 설정

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### 여러 에이전트 구성

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## 채널 관리

### 새 채널 추가

```bash
# WhatsApp 로그인
openclaw channels login whatsapp

# Telegram 로그인
openclaw channels login telegram
```

### 연결된 채널 보기

```bash
openclaw channels list
```

## 고급 기능

### Cron 작업

예약된 작업 구성:

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Heartbeat

주기적인 체크인 설정:

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### 샌드박스 모드

격리된 Docker 컨테이너에서 에이전트 실행:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

HTTP webhook 엔드포인트 설정:

```json5
{
  hooks: {
    enabled: true,
    token: "비밀 토큰",
    path: "/hooks"
  }
}
```

## 문제 해결

### 로그 보기

```bash
# Gateway 로그 보기
openclaw logs

# 실시간 로그 추적
openclaw logs -f
```

### 상태 확인

```bash
openclaw health
```

### 문제 진단

```bash
openclaw doctor
```

## 제어 UI

OpenClaw는 다음을 수행할 수 있는 웹 기반 제어 UI를 제공합니다:

- 메시지 전송 및 수신
- 구성 관리
- 세션 기록 보기
- 모바일 노드 관리
- Gateway 상태 모니터링

접근 주소: `http://127.0.0.1:18789/`
