# Guide d'Installation de Go

## Exigences

Go peut s'exécuter sur plusieurs systèmes d'exploitation, y compris Linux, macOS et Windows. Différents systèmes d'exploitation ont différentes méthodes d'installation, mais les exigences principales sont similaires. Assurez-vous que votre système répond aux exigences suivantes pour installer et utiliser correctement l'environnement de développement du langage Go.

En ce qui concerne les systèmes d'exploitation, Go prend en charge une large gamme de plateformes. Pour Linux, Go peut s'exécuter sur des distributions principales comme Ubuntu, Debian, Fedora, CentOS et RHEL. macOS nécessite macOS 10.13 (High Sierra) ou ultérieur. Windows nécessite Windows 10 ou ultérieur, tout en supportant également Windows 7 et 8.1 (bien qu'une configuration supplémentaire puisse être nécessaire).

En ce qui concerne le matériel, les exigences de Go sont relativement faibles. L'installation de Go ne nécessite qu'environ 100 Mo d'espace disque, bien que le développement réel de projets puisse nécessiter plus d'espace selon la taille du projet. Pour la mémoire, 2 Go de RAM suffisent pour le développement quotidien, bien que plus puisse être nécessaire pour une compilation à grande échelle ou l'exécution de tests volumineux.

Le compilateur Go n'a pas d'exigences particulières en matière de CPU, et les architectures modernes comme x86-64 et ARM64 fonctionnent bien. Go prend en charge la compilation croisée, permettant de compiler des binaires pour d'autres plateformes depuis une plateforme.

## Étapes d'Installation

### Installation sur Linux

Sur les systèmes Linux, l'utilisation du tarball officiel fourni est recommandée, car cette méthode est la plus propre et flexible. Tout d'abord, téléchargez le tarball de la dernière version depuis le site officiel de Go:

```bash
# Télécharger Go (remplacer par la dernière version)
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# Supprimer l'ancienne installation de Go (si elle existe)
sudo rm -rf /usr/local/go

# Extraire dans /usr/local
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

Ensuite, ajoutez le répertoire binaire de Go à la variable d'environnement PATH. Modifiez le fichier ~/.bashrc ou ~/.profile et ajoutez la ligne suivante:

```bash
export PATH=$PATH:/usr/local/go/bin
```

Après la sauvegarde, exécutez la commande suivante pour appliquer les modifications:

```bash
source ~/.bashrc
```

Ou sur macOS ou certaines distributions Linux:

```bash
source ~/.profile
```

### Installation sur macOS

Sur les systèmes macOS, il y a deux méthodes d'installation principales. La première consiste à utiliser le gestionnaire de paquets Homebrew, adapté aux utilisateurs ayant déjà Homebrew configuré:

```bash
# Installer avec Homebrew
brew install go

# Vérifier l'installation
go version
```

La deuxième méthode consiste à télécharger le programme d'installation officiel macOS. Visitez le site officiel de Go pour télécharger le programme d'installation .pkg, double-cliquez pour exécuter et suivez l'assistant d'installation. Le programme d'installation configure automatiquement les variables d'environnement.

Après l'installation, ouvrez Terminal et vérifiez:

```bash
go version
```

Vous devriez voir une sortie comme "go1.21.5 darwin/amd64".

### Installation sur Windows

Sur les systèmes Windows, l'utilisation du programme d'installation MSI est recommandée. Visitez le site officiel de Go pour télécharger le programme d'installation MSI pour Windows, double-cliquez pour exécuter et suivez l'assistant d'installation. Le programme d'installation configure automatiquement les variables d'environnement, y compris PATH.

Après l'installation, ouvrez l'invite de commandes ou PowerShell et vérifiez:

```bash
go version
```

Les utilisateurs Windows peuvent également utiliser le gestionnaire de paquets Chocolatey:

```bash
choco install golang
```

### Utilisation de Gestionnaires de Version (Recommandé)

Pour les développeurs ayant besoin de gérer plusieurs versions de Go simultanément, l'utilisation d'un gestionnaire de version est recommandée. Les gestionnaires de version courants incluent gvm (Go Version Manager) et asdf-go:

Installation et utilisation de gvm:

```bash
# Installer gvm
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# Installer une version de Go
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdf est un gestionnaire de versions multi-langages qui supporte Go:

```bash
# Installer le plugin asdf
asdf plugin add golang

# Installer une version de Go
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## Vérification de l'Installation

Après l'installation, vérifiez que Go est correctement installé. Ouvrez Terminal ou l'invite de commandes et exécutez:

```bash
# Vérifier la version de Go
go version

# Voir les informations d'environnement Go
go env
```

La commande go env produit plusieurs variables d'environnement, y compris GOPATH, GOROOT, GOOS, GOARCH, etc. Ces informations sont importantes pour comprendre le fonctionnement et la configuration de l'environnement Go.

Exécutez un programme simple pour vérifier que tout l'environnement de développement fonctionne correctement:

```bash
# Créer le répertoire de travail
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# Initialiser le module (Go 1.16+ recommande d'utiliser go mod)
go mod init hello

# Créer un programme de test
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# Exécuter le programme
go run main.go
```

Si cela affiche "Hello, Go!", l'environnement de développement Go est correctement configuré.

## Configuration de l'Environnement

### GOPATH et Modules

Avant Go 1.11, Go utilisait le mode GOPATH pour gérer les dépendances. Depuis Go 1.11, Go a introduit le support des modules, et l'utilisation de go mod pour gérer les dépendances du projet est recommandée. Lors de l'utilisation des modules, il n'est pas nécessaire de définir GOPATH, et les projets peuvent être créés dans n'importe quel répertoire.

Pour les environnements intranet d'entreprise ou lors de l'utilisation de référentiels privés, il peut être nécessaire de configurer la variable d'environnement GOPRIVATE:

```bash
export GOPRIVATE=github.com/company/*
```

### Configuration du Proxy

Dans des régions comme la Chine continentale, l'accès direct au référentiel de modules Go officiel peut être lent ou indisponible. Vous pouvez configurer un proxy de modules Go:

```bash
# Utiliser le proxy Alibaba Cloud
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct

# Ou utiliser le proxy Qiniu Cloud
go env -w GOPROXY=https://goproxy.cn,direct
```

### Configuration de l'IDE

Le développement Go utilise généralement des éditeurs comme VS Code, GoLand ou Sublime Text. VS Code nécessite l'installation de l'extension Go pour une bonne expérience de développement, y compris la complétion de code, le débogage, le formatage et plus.

GoLand est un IDE Go professionnel de JetBrains qui offre le support de développement Go le plus complet, y compris le débogage avancé, le refactoring et les tests.

## Mise à Niveau et Désinstallation

### Mise à Niveau de la Version Go

La mise à niveau de Go est simple. Téléchargez le programme d'installation de la nouvelle version et réinstallez en suivant les étapes d'installation ci-dessus. Le nouveau programme d'installation écrasera l'ancienne version.

Les utilisateurs de gestionnaires de versions peuvent facilement basculer entre différentes versions:

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### Désinstallation de Go

Sur Linux ou macOS, désinstaller Go nécessite simplement de supprimer le répertoire d'installation:

```bash
# Supprimer le répertoire d'installation de Go
sudo rm -rf /usr/local/go

# Retirer de PATH
# Modifiez ~/.bashrc ou ~/.profile et supprimez les lignes associées
```

Sur Windows, désinstallez via le Panneau de configuration Programmes et fonctionnalités, ou utilisez Chocolatey:

```bash
choco uninstall golang
```
