# Nacos 자주 묻는 질문

## 빠른 시작 질문

### Nacos는 어떤 운영 체제를 지원합니까?

Nacos는 Linux, Unix, Mac 및 Windows 운영 체제를 지원합니다. 프로덕션 환경에는 Linux/Unix/Mac이 권장됩니다.

### Nacos는 어떤 Java 버전이 필요합니까?

Nacos는 JDK 17 이상이 필요합니다. JAVA_HOME이 올바르게 구성되어 있는지 확인하세요.

### Nacos가 시작되지 않으면 어떻게 해야 합니까?

1. Java 환경 확인: 올바른 JDK 버전을 확인하려면 `java -version`을 실행하세요
2. 포트 사용 확인: 포트 8848와 9848가 사용 중인지 확인하세요
3. 로그 확인: logs/start.out 또는 logs/nacos.log의 오류 메시지를 확인하세요
4. 인증 구성 확인: Nacos 3.0+는 인증 매개변수가 필요합니다

### Nacos 기본 포트를 변경하려면 어떻게 해야 합니까?

conf/application.properties의 다음 구성을 수정하세요:

```properties
server.port=8848
```

### Nacos 콘솔 기본 자격 증명은 무엇입니까?

- 기본 사용자 이름: nacos
- 기본 비밀번호: nacos@demo

프로덕션 환경에서 기본 비밀번호를 변경하세요.

## 서비스 검색 질문

### 서비스 등록이 성공한 후 클라이언트가 서비스를 검색할 수 없는 이유는 무엇입니까?

1. 서비스 상태 확인: 인스턴스 상태가 정상인지 확인
2. 네트워크 연결 확인: 클라이언트가 Nacos 서버에 접근할 수 있는지 확인
3. 서비스 이름 확인: 검색한 서비스 이름이 등록된 이름과 일치하는지 확인(네임스페이스 및 그룹 포함)
4. 클라이언트 구성 확인: nacos.server-addr 구성이 올바른지 확인

### 로드 밸런싱을 구현하려면 어떻게 해야 합니까?

Nacos는 여러 로드 밸런싱 전략을 제공합니다:

1. 클라이언트 측 로드 밸런싱: Nacos SDK의 selectOneHealthyInstance 메서드 사용
2. 가중치 로드 밸런싱: 인스턴스 가중치 설정으로 구현
3. 클러스터 로드 밸런싱: 클러스터 환경에서 자동으로 구현

### 서비스 인스턴스가 오프라인 상태가 된 후에도 클라이언트가 여전히 오프라인 인스턴스를 호출하는 이유는 무엇입니까?

1. 상태 확인 구성 확인: 상태 확인 메커니즘이 제대로 작동하는지 확인
2. 클라이언트 캐시 확인: 클라이언트는 서비스 인스턴스 목록을 캐시하므로 약간의 지연이 있습니다
3. 동기화 간격 조정: 클라이언트의 새로 고침 간격 수정

## 구성 관리 질문

### 구성을 게시한 후 클라이언트가 업데이트 알림을 받지 않는 이유는 무엇입니까?

1. 리스너 확인: 클라이언트가 구성 리스너를 올바르게 추가했는지 확인
2. 네트워크 확인: 네트워크 연결이 정상인지 확인
3. 로그 확인: 클라이언트 로그의 오류 메시지 확인
4. dataId 및 group 확인: 완전히 일치하는지 확인

### 멀티 환경 구성 관리를 구현하려면 어떻게 해야 합니까?

1. 네임스페이스 사용: 서로 다른 환경은 완전한 분리를 위해 서로 다른 네임스페이스 사용
2. 그룹 사용: 동일한 네임스페이스에서 다른 그룹을 사용하여 구성 관리
3. DataId 사용: 서로 다른 환경의 구성을 구분하기 위해 다른 dataId 사용

### 구성 과거 버전을 보려면 어떻게 해야 합니까?

Nacos 콘솔은 구성 변경 기록 기능을 제공하여 다음을 볼 수 있습니다:

- 구성 수정 기록
- 각 수정에 대한 콘텐츠 비교
- 원클릭으로 과거 버전으로 롤백

### 구성 카나리 배포를 구현하려면 어떻게 해야 합니까?

Nacos는 구성 카나리 배포를 지원합니다:

1. 콘솔에서 카나리 구성 만들기
2. 카나리 클라이언트 범위 지정
3. 카나리 구성 효과 확인
4. 전체 배포 또는 롤백

## 클러스터 배포 질문

### Nacos 클러스터 배포에는 몇 개의 노드가 필요합니까?

고가용성을 보장하기 위해 프로덕션 환경에서는 최소 3개의 노드 배포가 권장됩니다.

### 클러스터 모드에서 데이터 저장소를 선택하려면 어떻게 해야 합니까?

- 테스트/개발 환경: 기본 제공 Derby 데이터베이스 사용 가능
- 프로덕션 환경: 데이터 신뢰성을 보장하기 위해 외부 데이터베이스(MySQL) 사용 필요

### 클러스터는 데이터 일관성을 어떻게 보장합니까?

Nacos는 자체 개발한 Distro 프로토콜과 Raft 프로토콜을 사용하여 클러스터 데이터 일관성을 보장합니다:

- Distro 프로토콜: 데이터 동기화 담당
- Raft 프로토콜: 리더 선출 및 로그 복제 담당

### 클러스터 노드 장애 시 어떻게 됩니까?

Nacos 클러스터에는 자동 장애 전환 기능이 있습니다:

1. 상태 확인이 실패한 노드를 자동으로 감지
2. 트래픽이 자동으로 정상 노드로 전환
3. 실패한 노드가 복구 후 자동으로 데이터 동기화

## 인증 및 보안 질문

### Nacos 3.0에서 인증을 구성하려면 어떻게 해야 합니까?

Nacos 3.0부터 인증이 기본적으로 활성화되어 있습니다. 시작 시 다음 매개변수를 구성해야 합니다:

```bash
# 토큰 비밀 키
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# 서비스 ID 키
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### Nacos 인증을 비활성화하려면 어떻게 해야 합니까?

conf/application.properties 수정:

```properties
nacos.core.auth.enabled=false
```

참고: 프로덕션 환경에서 인증 비활성화는 권장되지 않습니다.

### 다른 사용자에게 권한을 할당하려면 어떻게 해야 합니까?

Nacos는 역할 기반 액세스 제어를 지원합니다:

1. 콘솔에서 사용자 만들기
2. 사용자에게 역할 할당
3. 역할 권한 구성

## MCP 질문

### MCP 서버 시작 시 호환성 문제가 발생하면 어떻게 해야 합니까?

자동 등록된 MCP 서버 데이터가 Nacos의 해당 버전 데이터와 호환되는지 확인:

- MCP 서버 프로토콜(stdio, sse, streamable)이 일관적인지
- 서비스 참조가 일관적인지
- 도구 데이터가 일관적인지(도구 이름, 도구 수, 매개변수 유형, 선택적 매개변수)

참고: 도구 설명 및 매개변수 설명은 일관적일 필요 없습니다.

### 등록된 MCP 서버를 검색할 수 없는 이유는 무엇입니까?

1. MCP 서버 상태 확인: 서비스가 올바르게 등록되었는지 확인
2. 네임스페이스 확인: 클라이언트와 서비스가 동일한 네임스페이스에 있는지 확인
3. 인증 정보 확인: 사용자 이름과 비밀번호가 올바른지 확인

### MCP 서버 도구 구성을 동적으로 수정하려면 어떻게 해야 합니까?

MCP 서버가 Nacos에 자동 등록된 후 콘솔에서 MCP 서버 도구 설명 및 스위치를 동적으로 변경할 수 있습니다. 관련 수정 사항은 서비스를 다시 시작하지 않고 런타임에 MCP 서버에 동적으로 적용됩니다.

## 성능 및 최적화 질문

### Nacos는 대규모 서비스 등록을 어떻게 처리합니까?

Nacos는 Alibaba의 수백만 인스턴스 규모에서 탄생하여 다음 기능을 갖추고 있습니다:

-高性能なサービス登録と発見
-低延迟の構成更新応答
-大量的サービスと構成管理

### Nacos 성능을 최적화하려면 어떻게 해야 합니까?

1. 상태 확인 간격 적절하게 설정
2. 적절한 데이터베이스 연결 풀 구성 사용
3. 클라이언트 캐시를 활성화하여 요청 빈도 줄이기
4. 네임스페이스와 그룹 적절하게 계획

### Nacos 메모리 구성 권장 사항은?

서비스 규모에 따라:

- 소규모(서비스 < 1000): 2C4G
- 중규모(서비스 1000-5000): 4C8G
- 대규모(서비스 > 5000): 8C16G 이상

## 생태계 통합 질문

### Nacos를 Spring Cloud와 통합하려면 어떻게 해야 합니까?

pom.xml에 의존성 추가:

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### Nacos를 Kubernetes와 통합하려면 어떻게 해야 합니까?

Nacos는 Kubernetes 환경에서의 배포를 지원하며 다음을 사용하여 배포할 수 있습니다:

- Helm Chart 배포
- Operator 배포
- 사용자 정의 YAML 배포

### Nacos는 어떤 언어의 SDK를 지원합니까?

Nacos는 다음 언어의 SDK를 제공합니다:

- Java (공식)
- Python
- Go
- Node.js
- C/C++
- Ruby
- PHP

## 문제 해결

### 서비스 등록 실패를 해결하려면 어떻게 해야 합니까?

1. Nacos 서버가 제대로 실행되고 있는지 확인
2. 네트워크 연결 확인
3. 클라이언트 로그 검토
4. 등록 매개변수가 올바른지 확인

### 구성 검색 문제를 해결하려면 어떻게 해야 합니까?

1. 구성이 성공적으로 게시되었는지 확인
2. dataId와 group이 일치하는지 확인
3. 리스너가 올바르게 추가되었는지 확인
4. 구체적인 오류에 대해 클라이언트 로그 검토

### Nacos 로그는 어디에 있습니까?

Nacos 로그 파일은 다음에 위치합니다:

- 메인 로그: logs/nacos.log
- 시작 로그: logs/start.out
- 액세스 로그: logs/access.log

## 업그레이드 및 마이그레이션

### Nacos 1.x에서 2.x 또는 3.x로 업그레이드하려면 어떻게 해야 합니까?

1. 데이터 백업: 데이터베이스 또는 스냅샷 파일 백업
2. 업그레이드 가이드 읽기: 해당 버전의 업그레이드 문서 확인
3. 단계적 업그레이드: 먼저 스테이징 환경에서 테스트
4. 구성 마이그레이션: 인증 및 기타 구성 변경에 주의

### Nacos 데이터를 마이그레이션하려면 어떻게 해야 합니까?

1. 원본 환경 구성 데이터 내보내기
2. 새 환경 데이터베이스로 가져오기
3. 데이터 무결성 확인
4. 트래픽을 새 환경으로 전환
