# LivePortrait インストールガイド

このドキュメントでは、LivePortraitの異なるオペレーティングシステムと環境でのインストール手順について詳しく説明します。

## 動作環境

インストールを開始する前に、システムが次の基本要件を満たしていることを確認してください：

**オペレーティングシステム**：Linux（推奨）、Windows 10/11、macOS（Apple Siliconが必要）

**Pythonバージョン**：Python 3.10

**ハードウェア要件**：NVIDIA GPU（RTX 4090またはその他のCUDA対応GPUを推奨）、8GB以上のVRAM

**ストレージ容量**：モデルと実行環境に20GB以上の空き容量を推奨

## 基本的な環境設定

### CUDAバージョンの確認

まず、システムにインストールされているCUDAバージョンを確認して、適切なバージョンのPyTorchをインストールします：

```bash
nvcc -V
```

一般的なCUDAバージョンには11.1、11.8、12.1があります。表示されたバージョン番号に基づいて、対応するPyTorchインストールコマンドを選択してください。

### PyTorchのインストール

CUDAバージョンに応じて、適切なPyTorchインストールコマンドを選択してください：

**CUDA 11.1の場合**：

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**CUDA 11.8の場合**：

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**CUDA 12.1の場合**：

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

人文モード（Humans mode）のみ必要な場合は、CUDAバージョンの確認をスキップして、最新バージョンのPyTorchを直接インストールできます。

## プロジェクトコードのダウンロード

Gitを使用してLivePortraitリポジトリをクローンします：

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Python環境の作成

condaを使用して独立したPython環境を作成することを推奨します：

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

またはvenvを使用：

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## 依存関係のインストール

プロジェクトルートで以下を実行します：

```bash
pip install -r requirements.txt
```

## 事前学習モデルのダウンロード

事前学習モデルはHuggingFaceからダウンロードできます。ネットワークがHuggingFaceにアクセスできる場合：

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

HuggingFaceにアクセスできない場合は、ミラーサイトを使用します：

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

またはGoogle DriveまたはBaidu Netdiskから完全なモデルファイルをダウンロードし、./pretrained_weightsディレクトリに配置します。

## インストール確認

インストール後、テストスクリプトを実行して環境が正しく設定されていることを確認します：

```bash
python inference.py
```

すべてが正常に機能する場合、animationsディレクトリに生成された出力ファイルが表示されます。

## 動物モデル用の追加インストール手順

動物モード（Animals mode）はLinuxおよびWindowsシステムでのみサポートされ、NVIDIA GPUが必要です。

### X-Pose依存関係のインストール

動物モードには、キーポイント検出フレームワークとしてX-Poseプロジェクトが必要です。まず、X-Poseディレクトリに移動します：

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### HuggingFaceから動物モデルをダウンロード

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Windowsに関する特別な注意

Windowsシステムでは、特定のバージョンのCUDA（12.4、12.6など）で互換性问题が発生する可能性があります。安定性を 위해CUDAをバージョン11.8にダウングレートすることをお勧めします。

Windowsユーザーは、ワンクリックインストーラーをダウンロードして、展開後にrun_windows.batをダブルクリックして実行することもできます：

```bash
# HuggingFaceからWindowsワンクリックインストーラーをダウンロード
# アドレス：https://huggingface.co/cleardusk/LivePortrait-Windows
```

## macOSに関する特別な注意

### Apple Silicon macOS

Apple Silicon搭载のmacOSユーザーは特別な依存関係ファイルを使用する必要があります：

```bash
pip install -r requirements_macOS.txt
```

推論を実行するには、環境変数を設定します：

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

注意：Apple Silicon macOSでの推論はRTX 4090より約20倍遅い場合があります。

### Intel Mac

IntelベースのMacは公式にテストされておらず、正しく動作しない可能性があります。

### 動物モードの制限

X-Pose依存関係がmacOSをサポートしていないため、动物モードはmacOSでは利用できません。人文モードのみ使用できます。

## ディレクトリ構造の確認

インストール後、ディレクトリ構造が以下を満たしていることを確認します：

```
LivePortrait/
├── pretrained_weights/     # 事前学習モデルディレクトリ
├── assets/                 # サンプルのマテリアルディレクトリ
│   ├── examples/
│   │   ├── source/        # ソース画像/動画
│   │   └── driving/       # 駆動動画
├── src/                   # ソースコードディレクトリ
├── inference.py          # 推論スクリプト
├── app.py                # Gradioインターフェース
└── requirements.txt      # 依存関係リスト
```

## 一般的な問題

インストール中に 문제가 발생した場合は、以下を確認してください：

**CUDAが利用できない**：NVIDIAドライバーが正しくインストールされていることを確認し、nvidia-smiを実行して駆動プログラムの状態を確認します

**VRAM不足**：より小さい入力画像を使用するか、バッチサイズを小さくしてみてください

**モデルダウンロード失敗**：ネットワーク接続を確認するか、国内のミラーサイトを使用します

**依存関係冲突**：他のプロジェクトとの冲突を避けるために、新しいPython環境を使用することをお勧めします
