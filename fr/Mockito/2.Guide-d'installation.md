# Guide d'installation de Mockito

## Configuration requise

Avant d'installer Mockito, assurez-vous que votre environnement de développement repond aux exigences suivantes.

Mockito 5.x necessite JDK 11 ou superieur. Si votre projet doit supporter JDK 8, vous pouvez utiliser la serie Mockito 4.x. Il est recommande d'utiliser la derniere version de Mockito pour obtenir les dernieres fonctionnalites et corrections de securite.

## Installation Maven

### Ajout des dependances

Ajoutez la dependance mockito-core au fichier pom.xml de votre projet Maven. Il est recommande d'utiliser une propriete pour definir la version de Mockito pour une meilleure gestion des versions.

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Si vous avez besoin d'utiliser la fonction inline de Mockito pour simuler des classes et methodes finales, vous devez egalement ajouter la dependance mockito-inline.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### Versions snapshot

Mockito publie des versions snapshot dans le repository Sonatype apres chaque modification de code. Si vous devez utiliser les dernieres fonctionnalites en cours de développement, vous pouvez ajouter l'adresse du repository snapshot a votre pom.xml.

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Installation Gradle

### Ajout des dependances

Ajoutez la declaration de dependance au fichier build.gradle de votre projet Gradle.

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

Si vous utilisez le fichier Kotlin DSL de Gradle (build.gradle.kts), la syntaxe est legerement differente.

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### Utilisation du BOM pour la gestion des versions

L'utilisation du plugin Spring Dependency Management ou de la fonction BOM de Gradle peut simplifiér la gestion des versions.

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## Telechargement manuel

Si votre projet n'utilise pas Maven ou Gradle, vous pouvez telecharger les fichiers JAR directement depuis le repository Maven Central. Visitez Maven Central et recherchez org.mockito pour trouver toutes les versions de mockito-core et mockito-all (obsolete).

Apres le telechangement, ajoutez les fichiers JAR au classpath de votre projet. Notez que vous devez egalement telecharger les bibliotheques dependantes comme Byte Buddy et Objenesis.

## Integration avec Spring Boot

spring-boot-starter-test de Spring Boot inclut deja la dependance Mockito. Si votre projet utilise Spring Boot, vous n'avez generalement pas besoin d'ajouter la dependance Mockito separement.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Si vous avez besoin d'utiliser des Mocks plus facilement dans un environnement Spring, vous pouvez utiliser l'annotation @MockBean, qui injecte automatiquement les objets simulés dans le conteneur Spring.

## Integration avec JUnit 5

### Utilisation de MockitoExtension

JUnit 5 a introduit un model d'extension, et Mockito fournit MockitoExtension pour supporter JUnit 5. Aucune configuration de dependance supplementaire n'est necessaire car mockito-core contient deja le support necessaire.

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### Utilisation de MockitoJUnitRunner

Pour JUnit 4, vous pouvez utiliser MockitoJUnitRunner pour initialiser automatiquement les objets Mock.

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## Verification de l'installation

Apres l'installation, vous pouvez verifier que Mockito est correctement configure en creant une simple classe de test. Si le test passe apres execution, Mockito est correctement installe.

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito installe avec succes!");
    }
}
```

## Recommandations de choix de version

Pour les nouveaux projets, il est recommande d'utiliser la derniere version de Mockito 5.x. Si votre projet necessite la compatibilite avec JDK 8, la serie Mockito 4.x est disponible. Utilisez toujours la derniere version stable pour une meilleure performance et securite.
