# Spring Boot - 简介

Spring Boot 是 Spring 框架的子项目，由 Pivotal 团队于 2014 年发布。Spring Boot 旨在简化 Spring 应用的创建和开发过程，通过"约定优于配置"的理念，让开发者能够快速搭建生产级别的 Spring 应用。

## Spring Boot 的核心特性

### 1. 快速启动

Spring Boot 提供了大量的自动配置，减少了样板代码：

```java
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

只需一个主类和一个注解，即可启动一个 Web 应用。

### 2. 自动配置

Spring Boot 会根据类路径中的依赖自动配置 Bean：

```java
// 如果 classpath 中有 spring-boot-starter-web
// 自动配置 Tomcat、Spring MVC

// 如果 classpath 中有 spring-boot-starter-data-jpa
// 自动配置 EntityManager、TransactionManager
```

### 3. 嵌入式服务器

Spring Boot 内嵌了 Tomcat、Jetty 或 Undertow，无需部署 WAR 文件：

```java
// 打包后直接运行
java -jar myapp.jar
```

### 4. starter 依赖

Spring Boot 提供了一系列 starter 依赖，简化依赖管理：

| Starter | 说明 |
|---------|------|
| spring-boot-starter-web | Web 开发 |
| spring-boot-starter-data-jpa | JPA 数据访问 |
| spring-boot-starter-security | 安全认证 |
| spring-boot-starter-test | 测试 |
| spring-boot-starter-validation | 参数校验 |

### 5. 健康检查

Spring Boot Actuator 提供了应用健康检查端点：

```properties
management.endpoints.web.exposure.include=health,info
```

### 6. 外部化配置

支持多种配置方式：

```properties
# application.properties
server.port=8080
spring.application.name=myapp
```

```yaml
# application.yml
server:
  port: 8080
spring:
  application:
    name: myapp
```

## Spring Boot 的应用场景

### 1. RESTful API

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping
    public List<User> getUsers() {
        return userService.findAll();
    }
    
    @PostMapping
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
}
```

### 2. 微服务

Spring Cloud 基于 Spring Boot，构建微服务架构。

### 3. 企业应用

处理复杂业务逻辑的后端服务。

## Spring Boot 与 Spring

| 特性 | Spring | Spring Boot |
|------|--------|-------------|
| 配置 | XML/Java 配置 | 自动配置 |
| 依赖管理 | 手动 | Starter |
| 服务器 | 外部部署 | 内嵌 |
| 健康检查 | 手动实现 | 内置 |

## 为什么选择 Spring Boot？

1. **简化开发**：自动配置减少样板代码
2. **快速启动**：嵌入式服务器，即刻运行
3. **微服务友好**：云原生支持
4. **生产就绪**：监控、健康检查
5. **社区强大**：丰富的文档和资源
6. **生态完整**：集成 Spring 家族所有项目
