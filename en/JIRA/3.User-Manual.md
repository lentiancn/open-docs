# User Manual

This manual provides detailed instructions on Jira's core features and usage, helping teams get started and manage projects efficiently.

## Project Management Basics

### Creating a Project

In Jira, a project is the primary container for organizing work. Click the "Create project" button in the top navigation and select a template suitable for your team's needs. Jira offers various templates including software development, marketing, human resources, and sales scenarios. You can also start from a blank project and customize it.

### Managing Issues

Issues are the basic work units in Jira. When creating an issue, you need to fill in the title, description, assignee, due date, and other information. Issues can be set to different priority levels: highest, high, medium, low, and lowest.

Issue status reflects work progress. Typical workflows include To Do, In Progress, In Review, and Done. You can customize workflows to match your actual business processes.

### Working with Boards

Boards are one of the most commonly used views in Jira. On a Kanban board, issues are displayed as cards in different columns, with each column representing a workflow stage. You can drag issue cards between columns to quickly update status.

Scrum boards are suitable for teams using iterative development. You can create sprints, pull issues from the backlog into the sprint, and track sprint progress. After a sprint ends, you can view burndown charts and other metrics to evaluate team performance.

## Agile Development Support

### Scrum Framework

Jira's Scrum board helps teams unite around a goal and achieve iterative, incremental delivery. Each sprint typically lasts 2 to 4 weeks. Teams plan work at the beginning of a sprint, determine what can be completed during the sprint, and deliver usable software increments at the end.

Sprint planning is one of the core Scrum activities. Team members discuss the backlog and identify tasks for the sprint. After planning, tasks are added to the sprint backlog.

Daily standups help teams stay synchronized. Team members briefly report what they completed yesterday, what they plan to do today, and any blockers. Jira supports comments on issues, enabling asynchronous communication among team members.

### Kanban Method

Kanban is another popular agile method that emphasizes visualizing workflow and continuous improvement. In the Kanban view, you can see the entire flow of issues from creation to completion. By limiting work in progress, teams can complete tasks faster and improve delivery efficiency.

## Team Collaboration

### Assignments and Permissions

Each issue can be assigned to specific team members. Jira supports fine-grained permission control, allowing administrators to set different access levels for different user groups to ensure information security and clear responsibilities.

### Comments and Mentions

Issue comments are important for team collaboration. Members can discuss technical details, share progress, or ask questions on issues. Using the @ symbol mentions team members, who will receive notifications.

### Attachments and Documents

Jira allows adding attachments to issues, supporting file uploads and screenshots. You can view and manage these files directly in issues without using additional document management tools.

## Automation and Integrations

### Automation Rules

Jira's automation features help teams reduce manual operations and improve efficiency. Common automation rules include: automatically assigning newly created issues to specific members, sending notifications when issue status changes, automatically updating fields when conditions are met.

Different pricing plans offer different automation limits: Free plan has 100 rules per month, Standard plan has 1,700 rules per month, Premium plan has 1,000 rules per user per month, Enterprise plan has unlimited rules.

### Third-Party Integrations

Jira supports integration with thousands of third-party tools covering development tools, communication platforms, and design tools. Common integrations include: code repository tools like GitHub, GitLab, and Bitbucket, allowing direct viewing of code commit records in issues; communication tools like Slack for receiving issue notifications; design tools like Figma for viewing design mockups in issues.

## Reporting and Analytics

Jira provides rich reporting features to help teams understand project progress and team performance. Dashboards can display key metrics such as task completion speed, backlog size, and team workload distribution. Scrum teams can use burndown charts to track sprint progress and velocity charts to evaluate team delivery capability.

Through these data, teams can continuously improve workflows and increase delivery efficiency. Jira also supports integration with Atlassian Analytics for deeper data analysis.

## AI Assistant

Rovo AI is Jira's artificial intelligence assistant that helps teams work more efficiently. Rovo can help create project plans, break down big ideas, optimize issue descriptions, and quickly link relevant resources. It can also extract data from various company resources, helping teams understand context faster and make better decisions.
