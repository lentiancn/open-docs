# Guide d'Installation Nacos

## Exigences Environnementales

Nacos nécessite un environnement d'exécution Java. Veuillez vous assurer d'utiliser les versions suivantes :

- Système d'exploitation 64 bits : Linux/Unix/Mac/Windows pris en charge, Linux/Unix/Mac recommandé
- JDK 64 bits 17+ (JDK 17 ou supérieur recommandé)
- Téléchargement JDK : https://www.oracle.com/java/technologies/downloads/#java17
- Configuration environnement JDK : https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## Configuration Recommandée

Il est recommandé d'exécuter Nacos avec au moins la configuration machine suivante :

- CPU : 2 cœurs
- Mémoire : 4 Go
- Disque : 60 Go

## Méthodes d'Installation

Nacos prend en charge plusieurs méthodes d'installation. Vous pouvez choisir la méthode appropriée selon vos besoins.

### Méthode 1 : Utilisation du Déploiement en Un Clic nacos-setup (Recommandé)

nacos-setup fournit un déploiement automatisé en un clic, gérant automatiquement le téléchargement, la configuration d'authentification, la détection de port et la vérification de l'environnement Java.

#### Installer nacos-setup

Linux/macOS :

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows (PowerShell) :

```powershell
# Exécuter avec PowerShell
```

#### Démarrer le Service Nacos

Déployer localement la dernière version de Nacos autonome en un clic :

```bash
nacos-setup
```

Déployer localement une version spécifique de Nacos autonome :

```bash
nacos-setup -v 3.1.1
```

Déployer localement la dernière version de Nacos en cluster :

```bash
nacos-setup -c cluster
```

Configurer le stockage externe globalement, entrer l'adresse de la base de données, le port, le nom d'utilisateur et le mot de passe, puis réexécuter nacos-setup :

```bash
nacos-setup --datasource-conf
```

Pour plus d'utilisation, consultez le manuel utilisateur de nacos-setup.

### Méthode 2 : Téléchargement Manuel et Déploiement

#### Télécharger le Paquet d'Installation

Vous pouvez obtenir le paquet de distribution Nacos de deux manières :

**Méthode 1 : Télécharger depuis le Site Web Officiel Nacos**

Visitez la page de téléchargement des versions du site web officiel Nacos, sélectionnez une version stable, puis cliquez pour télécharger le fichier zip dans la colonne du paquet binaire.

**Méthode 2 : Télécharger depuis GitHub**

Visitez la page des Releases Nacos sur GitHub et téléchargez le paquet de distribution pour la version correspondante.

Note : Lorsque de nombreux utilisateurs téléchargent simultanément, vous pouvez rencontrer des échecs de limitation de téléchargement. Si cela se produit, attendez et réessayez, ou téléchargez depuis GitHub.

#### Extraire le Paquet de Distribution Nacos

Linux/Unix/Mac :

```bash
unzip nacos-server-$version.zip
# ou
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows :

Utilisez des outils d'extraction pour décompresser directement nacos-server-$version.zip.

#### Démarrer le Serveur

Commande de démarrage Linux/Unix/Mac (standalone signifie mode non cluster) :

```bash
sh startup.sh -m standalone
```

Si vous utilisez Ubuntu, ou rencontrez des messages d'erreur concernant des symboles non trouvés, essayez :

```bash
bash startup.sh -m standalone
```

Le programme de démarrage vous demandera de saisir trois configurations liées à l'authentification (à partir de Nacos 3.0.0, l'authentification console est activée par défaut, donc les trois configurations d'authentification suivantes doivent être remplies) :

```
nacos.core.auth.plugin.nacos.token.secret.key manquant, veuillez définir avec une chaîne Base64 : ${your_input_token_secret_key}
nacos.core.auth.server.identity.key manquant, veuillez définir : ${your_input_server_identity_key}
nacos.core.auth.server.identity.value manquant, veuillez définir : ${your_input_server_identity_key}
```

Commande de démarrage Windows :

```bash
cmd startup.cmd
```

Ou double-cliquez sur startup.cmd pour exécuter.

### Méthode 3 : Installation avec Docker

#### Tirer l'Image Nacos

```bash
docker pull nacos/nacos-server
```

#### Exécuter en Mode Autonome

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### Exécuter en Mode Cluster

Nécessite un déploiement en cluster avec Docker Compose ou Kubernetes.

### Méthode 4 : Installation sur Kubernetes

Nacos prend en charge le déploiement dans les environnements Kubernetes et peut être installé en utilisant Helm Chart ou Operator. Pour les étapes spécifiques, consultez la documentation de déploiement Nacos Kubernetes.

## Vérifier le Démarrage du Service Nacos

Naviguez vers le répertoire des logs Nacos et utilisez `tail -f start.out` pour voir les logs. Si vous voyez le message suivant, le service a démarré avec succès :

```
Nacos started successfully in stand alone mode. use embedded storage
```

## Vérification Rapide

### Console Nacos

Ouvrez n'importe quel navigateur et entrez : http://127.0.0.1:8080/index.html pour accéder à la console Nacos.

Note : La première fois que vous l'ouvrez, vous devrez initialiser le mot de passe administrateur pour l'utilisateur nacos. Nom d'utilisateur/mot de passe par défaut : nacos/nacos@demo.

### Enregistrement de Service

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### Découverte de Service

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### Publier la Configuration

Tout d'abord, connectez-vous pour obtenir le jeton d'accès :

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

Utilisez le jeton d'accès pour créer une configuration :

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### Obtenir la Configuration

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## Arrêter le Serveur

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

Ou double-cliquez sur shutdown.cmd pour exécuter.

## Sélection de Version

La version stable actuellement recommandée pour la production est 3.1.1. Vous pouvez conocer les fonctionnalités prises en charge par chaque version via :

- Page de téléchargement des versions Nacos
- Notes de version Nacos
- Annonces de version Nacos

## Notes

1. À partir de Nacos 3.0.0, l'authentification console est activée par défaut. Les paramètres liés à l'authentification doivent être configurés lors du premier démarrage.
2. Le déploiement en mode cluster nécessite une base de données externe (comme MySQL) pour le stockage des données.
3. Les environnements de production doivent utiliser le mode cluster pour garantir la haute disponibilité.
4. Assurez-vous que les ports nécessaires sont ouverts dans le pare-feu (par défaut 8848, 9848).
5. JDK 17 ou supérieur est recommandé pour des performances optimales.
