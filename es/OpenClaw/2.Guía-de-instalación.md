# Guía de Instalación de OpenClaw

OpenClaw soporta múltiples métodos de instalación. Puedes elegir el que mejor se adapte a tus necesidades.

## Requisitos

- **Node.js**: 22.x o superior
- **Sistema Operativo**: macOS, Linux, Windows (vía WSL2)
- **Opcional**: Tailscale (para acceso remoto)

## Instalación Rápida

### macOS / Linux

La forma más fácil es usar el script de instalación:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

O usando npm/pnpm:

```bash
npm install -g openclaw@latest
# o
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## Configuración Inicial

Después de la instalación, ejecuta el asistente de incorporación:

```bash
openclaw onboard --install-daemon
```

El asistente te guiará a través de:

1. **Configuración de autenticación**: Configurar claves API o OAuth
2. **Configuración de Gateway**: Configurar puerto y servicio del gateway
3. **Conexión de canales**: Conectar plataformas de mensajería
4. **Instalación de habilidades**: Seleccionar e instalar las habilidades necesarias

## Instalar como Servicio del Sistema

Usa `--install-daemon` para instalar OpenClaw como servicio en segundo plano:

```bash
openclaw onboard --install-daemon
```

Esto crea un servicio a nivel de usuario (launchd o systemd) que asegura que OpenClaw siempre se ejecute.

## Ejecutar Gateway Manualmente

Si prefieres no instalar como servicio:

```bash
openclaw gateway --port 18789 --verbose
```

## Instalación con Docker

Si prefieres Docker, consulta la documentación oficial de despliegue con Docker.

## Compilar desde el Código Fuente

Si quieres contribuir o personalizar OpenClaw:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

Nota: pnpm se usa para compilar. TypeScript puede ejecutarse directamente vía tsx.

## Verificar la Instalación

Después de la instalación, puedes:

1. **Verificar estado del Gateway**:
   ```bash
   openclaw gateway status
   ```

2. **Abrir Control UI**:
   ```bash
   openclaw dashboard
   ```
   O visita `http://127.0.0.1:18789/` en tu navegador

3. **Enviar un mensaje de prueba** (requiere configuración de canal):
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## Actualizar OpenClaw

OpenClaw tiene tres canales de lanzamiento:

- **stable**: Versiones oficiales (recomendado)
- **beta**: Versiones preliminaires
- **dev**: Compilaciones de desarrollo

Cambiar canales:

```bash
openclaw update --channel stable|beta|dev
```

Después de actualizar, ejecuta:

```bash
openclaw doctor
```

## Comandos Comunes

| Comando | Descripción |
|---------|-------------|
| `openclaw onboard` | Ejecutar asistente de configuración inicial |
| `openclaw gateway start` | Iniciar Gateway |
| `openclaw gateway status` | Ver estado del Gateway |
| `openclaw dashboard` | Abrir Control UI |
| `openclaw agent --message "..."` | Enviar mensaje al asistente |
| `openclaw message send --to xxx --message "xxx"` | Enviar un mensaje |
| `openclaw doctor` | Diagnosticar y arreglar problemas |
| `openclaw update` | Actualizar OpenClaw |

## Variables de Entorno

Para ubicaciones de configuración personalizadas:

- `OPENCLAW_HOME`: Establece el directorio principal
- `OPENCLAW_STATE_DIR`: Sobreescribe el directorio de estado
- `OPENCLAW_CONFIG_PATH`: Sobreescribe la ruta del archivo de configuración

## Solución de Problemas

**¿La instalación falla?**

- Asegúrate de que Node.js sea >= 22
- Verifica la conexión de red
- Ejecuta `openclaw doctor` para diagnosticar

**¿Cómo desinstalar?**

- Detén el servicio Gateway
- Elimina el directorio de instalación
- Limpia los archivos de configuración
