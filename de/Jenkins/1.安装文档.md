# Jenkins Installationsanleitung

Jenkins ist ein Open-Source-Automatisierungsserver für die Implementierung von Continuous Integration und Continuous Deployment (CI/CD). Diese Anleitung behandelt Installationsmethoden in verschiedenen Umgebungen.

## Versionsinformationen

| Version | Status | Beschreibung |
|---------|--------|--------------|
| Jenkins 2.x | ✅ Aktuell | Neueste stabile Version |
| Jenkins 1.x | ❌ Veraltet | Wartung eingestellt |

## Systemanforderungen

### Hardwareanforderungen

| Anforderung | Minimum | Empfohlen |
|-------------|---------|-----------|
| CPU | 1 Kern | 2+ Kerne |
| Speicher | 1 GB | 4+ GB |
| Festplatte | 10 GB | 50+ GB |

### Softwareanforderungen

| Software | Version |
|----------|---------|
| Java | JDK 8 oder höher |
| Betriebssystem | Windows/macOS/Linux |

## Installationsmethoden

### Methode 1: Windows-Installation

#### 1. Installer verwenden

1. Jenkins herunterladen: https://www.jenkins.io/download/
2. jenkins.msi Installer ausführen
3. Installationsassistenten folgen

#### 2. Chocolatey verwenden

```powershell
choco install jenkins
```

#### 3. Manuelle Installation

```powershell
# WAR-Datei herunterladen
# In das angegebene Verzeichnis entpacken
java -jar jenkins.war
```

### Methode 2: macOS-Installation

#### 1. Homebrew verwenden

```bash
# Jenkins installieren
brew install jenkins

# Dienst starten
brew services start jenkins

# Dienst stoppen
brew services stop jenkins
```

#### 2. Manuelle Installation

```bash
# WAR-Datei herunterladen
wget https://get.jenkins.io/jenkins.war

# Ausführen
java -jar jenkins.war
```

### Methode 3: Linux-Installation

#### Ubuntu/Debian

```bash
# Schlüssel hinzufügen
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -

# Repository hinzufügen
sudo sh -c 'echo deb https://pkg.jenkins.io/debian binary/ > /etc/apt/sources.list.d/jenkins.list'

# Aktualisieren und installieren
sudo apt-get update
sudo apt-get install jenkins

# Dienst starten
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

#### CentOS/RHEL

```bash
# Repository hinzufügen
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat/jenkins.repo

# Schlüssel importieren
sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

# Installieren
sudo yum install jenkins

# Dienst starten
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

### Methode 4: Docker-Installation

```bash
# Jenkins-Container ausführen
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts
```

#### Docker Compose

```yaml
version: '3'
services:
  jenkins:
    image: jenkins/jenkins:lts
    ports:
      - "8080:8080"
    volumes:
      - jenkins_home:/var/jenkins_home
```

### Methode 5: WAR-Datei bereitstellen

```bash
# WAR-Datei herunterladen
wget https://get.jenkins.io/war-stable/latest/jenkins.war

# Ausführen
java -jar jenkins.war --httpPort=8080
```

## Konfiguration nach der Installation

### Erster Zugriff

1. Browser unter http://localhost:8080 öffnen
2. Erstes Admin-Passwort abrufen:
   ```bash
   # Linux/macOS
   sudo cat /var/lib/jenkins/secrets/initialAdminPassword
   
   # Docker
   docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
   ```
3. Empfohlene Plugins installieren
4. Admin-Benutzer erstellen

### Deutsche Oberfläche einrichten

1. Zu Manage Jenkins → Manage Plugins gehen
2. "Localization: German" installieren
3. Jenkins neu starten

## Häufig gestellte Fragen

### Start schlägt fehl?

Prüfen, ob Java installiert ist:

```bash
java -version
```

### Port belegt?

Port ändern:

```bash
java -jar jenkins.war --httpPort=8090
```

### Plugin-Installation fehlgeschlagen?

Netzwerkverbindung prüfen oder Proxy verwenden.
