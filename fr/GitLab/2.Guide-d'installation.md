# Guide d'Installation de GitLab

## Exigences

Avant d'installer GitLab, assurez-vous que votre serveur répond aux exigences matérielles et logicielles suivantes. Pour les petites et moyennes équipes, un minimum de 4 cœurs CPU et 8 Go de RAM est recommandé, ce qui peut prendre en charge des centaines de dépôts et des dizaines d'utilisateurs simultanés. Les équipes plus grandes ou les scénarios à charge élevée nécessitent plus de ressources, de préférence 8+ cœurs CPU et 16 Go+ de RAM.

En ce qui concerne les systèmes d'exploitation, GitLab prend en charge plusieurs distributions Linux, notamment Ubuntu, Debian, CentOS, Red Hat Enterprise Linux et Fedora. Ubuntu ou Debian est officiellement recommandé car leur gestion de paquets est la plus pratique. GitLab nécessite au moins 20 Go d'espace disque pour l'installation, 50 Go+ recommandés pour les configurations de base, et souvent des centaines de Go pour les environnements de production.

Les dépendances logicielles incluent OpenSSH Server pour le support du protocole SSH, la base de données PostgreSQL (GitLab inclut une version intégrée) et le serveur SMTP pour les notifications par e-mail. Vous devez également configurer le bon fuseau horaire et nom d'hôte, et assurer que le serveur a un accès normal à Internet pour télécharger les paquets de dépendances.

## Étapes d'Installation

### Installation sur Ubuntu/Debian

Sur les systèmes Ubuntu ou Debian, l'installation via le dépôt apt officiel est recommandée, car cela garantit l'obtention des dernières versions stables et des mises à jour de sécurité automatiques. Tout d'abord, installez les paquets de dépendances nécessaires :

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

Ajoutez le dépôt APT GitLab et installez GitLab CE (Community Edition) :

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

Remplacez EXTERNAL_URL par le domaine ou l'adresse IP réel pour accéder à GitLab. Le processus d'installation configurera automatiquement PostgreSQL, Redis et GitLab lui-même.

### Installation sur CentOS/RHEL

Sur CentOS ou Red Hat Enterprise Linux, vous pouvez installer GitLab en utilisant le gestionnaire de paquets yum ou dnf. Tout d'abord, installez les dépendances nécessaires :

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

Ajoutez le dépôt YUM GitLab et installez :

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## Configuration Initiale

Après l'installation, vous devez configurer les paramètres de base de GitLab. Modifiez le fichier /etc/gitlab/gitlab.rb, qui est le fichier de configuration principal de GitLab. L'élément de configuration le plus important est external_url, qui est l'adresse d'accès externe pour le service GitLab :

```ruby
external_url 'https://gitlab.example.com'
```

Si vous devez utiliser un port SSH personnalisé, configurez le paramètre gitlab_rails['gitlab_shell_ssh_port']. La configuration du serveur de messagerie est également importante, utilisée pour envoyer des notifications et des e-mails de réinitialisation de mot de passe :

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

Après avoir enregistré la configuration, exécutez la commande suivante pour appliquer les modifications :

```bash
sudo gitlab-ctl reconfigure
```

Cette commande reconfigurera tous les composants de GitLab, y compris les migrations de base de données et la création de l'utilisateur initial. L'ensemble du processus peut prendre plusieurs minutes.

## Vérification de l'Installation

Une fois la configuration terminée, vous pouvez accéder à l'adresse spécifiée par external_url via votre navigateur. Si tout est correct, vous devriez voir la page de connexion GitLab. Au premier accès, vous verrez la page de connexion du compte administrateur. Le nom d'utilisateur administrateur par défaut est root, et vous devrez définir un mot de passe lors de la première connexion.

GitLab fournit l'outil de ligne de commande gitlab-ctl pour gérer l'état du service :

```bash
# Vérifier l'état du service
sudo gitlab-ctl status

# Voir les journaux
sudo gitlab-ctl tail

# Redémarrer le service
sudo gitlab-ctl restart
```

Vous pouvez également utiliser la commande gitlab-rake pour les vérifications de santé :

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Installation Docker

Pour des tests rapides ou des déploiements de petite taille, vous pouvez exécuter GitLab en utilisant Docker. Tout d'abord, installez Docker Engine, puis exécutez la commande suivante sur l'hôte :

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

L'installation Docker monte trois volumes pour persister la configuration, les journaux et les données. Le premier démarrage du conteneur Docker prend du temps pour s'initialiser. Après avoir attendu patiemment, vous pouvez accéder via votre navigateur.

## Mise à Niveau et Maintenance

La mise à niveau régulière de GitLab vers la dernière version offre de nouvelles fonctionnalités et des correctifs de sécurité. Mise à niveau sur Ubuntu/Debian :

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

Il est recommandé de sauvegarder les données avant la mise à niveau. GitLab fournit un outil de sauvegarde intégré :

```bash
sudo gitlab-backup create
```

Les fichiers de sauvegarde sont stockés par défaut dans le répertoire /var/opt/gitlab/backups, et vous pouvez configurer les politiques de conservation des sauvegardes et les emplacements de stockage distant.
