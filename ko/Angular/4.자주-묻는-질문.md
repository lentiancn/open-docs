# 자주 묻는 질문

이 문서에서는 Angular 사용 과정에서 자주 발생하는 기술적인 질문과 답변을 모았습니다.

## 기초적인 질문

### Angular란 무엇인가요?

Angular는 Google에서 개발하고 유지관리하는 오픈소스 웹 애플리케이션 프레임워크로, 최신 웹 애플리케이션을 구축하는 데 사용됩니다. TypeScript를 주요 개발 언어로 채택하고, 컴포넌트 기반 아키텍처, 양방향 데이터 바인딩, 의존성 주입 등의 특성을 제공합니다.

### Angular와 AngularJS의 차이점은 무엇인가요?

AngularJS는 Angular 1.x의 이름으로, Angular의 전신입니다. Angular는 완전히 새로 작성된 버전으로, TypeScript를 사용하고 아키텍처가 완전히 다릅니다. AngularJS와는 호환되지 않습니다.

### Angular는 배우기 어렵나요?

Angular는 어느 정도 학습 곡선이 있습니다. 특히 의존성 주입, RxJS 같은 개념을 이해하는 데 시간이 걸립니다. 하지만 문서가 잘 되어 있고 커뮤니티가活跃하여, 한번 익히면 개발 효율이 매우 좋습니다.

## 설치 관련 질문

### npm install이 실패하면 어떻게 하나요?

1. npm 캐시 지우기: `npm cache clean --force`
2. node_modules와 package-lock.json 삭제
3. 국내 미러 사용
4. Node.js 버전이 호환되는지 확인

### Angular CLI 명령을 찾을 수 없어요?

전역으로 설치되어 있는지 확인:
```bash
npm install -g @angular/cli```

그래도 안 되면, npx 사용:
```bash
npx @angular/cli new my-app
```

### Node.js 버전이 호환되지 않아요?

nvm으로 Node.js 버전 관리:
```bash
nvm install 18
nvm use 18
```

## 개발 관련 질문

### 컴포넌트 스타일이 적용되지 않아요?

1. 스타일 파일 경로가 올바른지 확인
2. styles.scss에 문법 오류가 없는지 확인
3. ViewEncapsulation 설정 확인

### 라우터 이동이 실패해요?

1. 라우터 설정이 올바른지 확인
2. RouterModule이 제대로 임포트되어 있는지 확인
3. 라우트 가드가 이동阻碍하고 있는지 확인

### HTTP 요청이 실패해요?

1. 백엔드 서비스가 실행 중인지 확인
2. CORS 설정 확인
3. HttpInterceptor로 오류 처리 추가

### 데이터가 업데이트되지 않아요?

1. 변경 감지 전략 확인
2. OnPush를 사용하는 경우, 수동으로 감지 트리거
3. 데이터 참조가 변경되었는지 확인

## 성능 관련 질문

### 초기 화면 로딩이 느려요?

1. 라우터 지연 로딩 활성화
2. 프리로드 전략 사용
3. 이미지 리소스 최적화
4. 프로덕션 모드로 빌드

### 메모리 누수가 발생해요?

1. ngOnDestroy에서 Observable 구독 취소
2. takeUntil 또는 async 파이프 사용
3. 타이머와 이벤트 리스너 정리

### 변경 감지가 너무 자주 발생해요?

1. OnPush 전략 사용
2. 변하는 속성과 변하지 않는 속성 분리
3. *ngFor에 trackBy 사용

## 테스트 관련 질문

### 단위 테스트가 실패해요?

1. TestBed 설정 확인
2. 비동기 작업이 제대로 처리되었는지 확인
3. fakeAsync와 tick으로 비동기 코드 테스트

### 모듈을 찾을 수 없어요?

tsconfig.json에서 paths가 올바르게 설정되어 있는지 확인하세요.

## 빌드 관련 질문

### 빌드가 실패해요?

1. 구체적인 오류 메시지 확인
2. TypeScript 컴파일 옵션 확인
3. 캐시 삭제 후 다시 빌드

### 빌드 파일이 너무 커요?

1. 프로덕션 모드 활성화
2. 지연 로딩 사용
3. 사용하지 않는 의존성 확인

## 흔한 오류

### ExpressionChangedAfterItHasBeenCheckedError

변경 감지 중에 변경되지 않아야 할 값을 템플릿에서 수정했습니다. ngOnInit에 뷰 관련 코드를 직접 수정하는 것이 있는지 확인하세요.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

위와 같은 오류입니다. 데이터 바인딩 타이밍을 확인하세요.

### Cannot read property of undefined

데이터가 초기화되지 않은 상태에서 접근했습니다. 옵션 체이닝(?.) 또는 *ngIf로 보호하세요.

### CORS 오류

백엔드에서 CORS가 설정되지 않았거나, 프론트엔드 요청 헤더에 문제가 있습니다.

### 401/403 오류

인증 실패입니다. 토큰과 권한 설정을 확인하세요.

## 버전 업그레이드

### Angular를 업그레이드하려면 어떻게 하나요?

1. CLI로 업그레이드: `ng update @angular/core`
2. 공식 업그레이드 가이드 참조
3. 의존성 하나씩 업그레이드
4. 테스트를 실행하여 정상 작동 확인

### 업그레이드가 실패하면 어떻게 하나요?

1. 코드 백업
2. package.json의 버전を手動更新
3. node_modules 삭제 후 다시 설치
