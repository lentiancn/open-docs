# Kubernetes 簡介

## 什麼是 Kubernetes

Kubernetes（簡稱 K8s）是一個開源的容器編排平台，用於自動化部署、擴展和管理容器化應用程式。它將組成應用程式的容器分組為邏輯單元，便於管理和服務發現。Kubernetes 源自 Google 15 年執行生產環境工作負載的經驗，同時融合了社群的最佳構想和實踐。

## 核心功能

### 自動化部署與回滾
Kubernetes 會逐步推出對應用程式或其配置的變更，同時監控應用程式健康狀況，確保不會同時終止所有執行個體。如果發生問題，Kubernetes 會為你回滾變更。你可以充分利用不斷增長的部署解決方案生態系統。

### 服務發現與負載均衡
無需修改應用程式即可使用不熟悉的服務發現機制。Kubernetes 為每個 Pod 提供自己的 IP 位址，並為一組 Pod 提供單一 DNS 名稱，並可在它們之間進行負載均衡。

### 儲存編排
自動掛載所選的儲存系統，無論是本地儲存、公有雲供應商，還是網路儲存系統（如 iSCSI 或 NFS）。

### Secret 與配置管理
部署和更新 Secret 及應用程式配置，無需重建映像檔，也不會在堆疊配置中暴露 Secret。

### 自動組箱
根據資源需求和其他限制自動放置容器，同時不犧牲可用性。混合放置關鍵性和盡力而為的工作負載，以提高資源利用率並節省更多資源。

### 批次執行
除了服務之外，Kubernetes 還可以管理批次和 CI 工作負載，並在需要時替換失敗的容器。

### 自我修復
Kubernetes 會自動重新啟動崩潰的容器，必要時替換整個 Pod，回應更大範圍的故障時重新附加儲存，並可與節點自動擴展器整合，實現節點級別的自我修復能力。

### 水平擴展
使用簡單的指令、UI 或根據 CPU 使用情況自動對應用程式進行擴展或縮減。

### 垂直擴展
根據實際使用模式自動調整資源請求和限制。

### 可擴展性設計
無需更改上游原始碼即可擴展 Kubernetes 叢集。

## 架構概述

### 控制平面
控制平面負責維護整個叢集的狀態，包括：
- API Server：叢集的入口点，提供 RESTful API
- etcd：分散式鍵值儲存，儲存叢集資料
- Scheduler：負責將 Pod 調度到合適的節點
- Controller Manager：執行各種控制器以維護叢集狀態

### 工作節點
工作節點是執行容器的地方，每個節點包含：
- Kubelet：與控制平面通訊，管理節點上的容器
- Container Runtime：負責執行容器（Docker、containerd 等）
- Kube Proxy：負責網路代理和負載均衡

## 應用場景

### 雲端運算
Kubernetes 是開源的，讓你可以自由地利用本地、私有雲、混合雲或公有雲基礎設施，讓你能輕鬆地將工作負載移動到需要的地方。

### 微型服務架構
Kubernetes 是執行微型服務的理想平台，能夠有效管理大量獨立的服務執行個體。

### 機器學習
Kubernetes 是機器學習的絕佳平台，因為它提供了機器學習工作負載所需的所有排程功能。

### 持續部署
支援快速迭代，能夠每天多次部署應用程式的更新版本。

## 優勢

### 行星級規模
採用與 Google 每週執行數十億個容器的相同原則設計，Kubernetes 可以在不增加營運團隊的情況下進行擴展。

### 永不过时
無論是本地測試還是執行全球企業，Kubernetes 的靈活性都會隨著你的成長而成長，幫助你始終如一且輕鬆地交付應用程式。

### 強大社群支援
Kubernetes 擁有龐大且快速增長的生態系統，服務、支援和工具的應用範圍非常廣泛。
