# Nginx 설치 가이드

## 환경 요구 사항

Nginx는 다양한 운영 체제를 지원합니다:

- Linux (Debian, Ubuntu, CentOS, RHEL, Alpine 등)
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## 설치 방법

### 방법 1: 패키지 관리자 사용 (권장)

#### Ubuntu/Debian

```bash
# 패키지 인덱스 업데이트
sudo apt update

# Nginx 설치
sudo apt install nginx

# Nginx 시작
sudo systemctl start nginx

# 부팅 시 시작
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# EPEL 저장소 설치
sudo yum install epel-release

# Nginx 설치
sudo yum install nginx

# Nginx 시작
sudo systemctl start nginx

# 부팅 시 시작
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# Homebrew로 설치
brew install nginx

# Nginx 시작
brew services start nginx
```

### 방법 2: 소스에서 컴파일

#### 빌드 종속성 설치

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### 소스 코드 다운로드

```bash
# 최신 안정 버전 다운로드
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# 압축 해제
tar -zxf nginx-1.26.0.tar.gz

# 디렉토리 이동
cd nginx-1.26.0
```

#### 빌드 구성

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### 컴파일 및 설치

```bash
# 컴파일
make

# 설치
sudo make install
```

### 방법 3: Docker 사용

```bash
# 공식 이미지 다운로드
docker pull nginx

# Nginx 컨테이너 실행
docker run -d -p 80:80 --name mynginx nginx

# 구성 및 로그 마운트
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### 방법 4: Windows

1. https://nginx.org/en/download.html에서 Windows 버전 다운로드
2. 원하는 디렉토리에 압축 해제
3. nginx.exe를 더블 클릭하여 시작
4. 또는 명령 프롬프트에서 시작:

```cmd
cd C:\nginx
start nginx
```

## 설치 후 확인

### 설치 확인

```bash
# Nginx 버전 확인
nginx -v

# 상세 버전 정보
nginx -V
```

### 실행 상태 확인

```bash
# 프로세스 확인
ps aux | grep nginx

# systemctl로 상태 확인
systemctl status nginx
```

### 접근 테스트

```bash
# curl로 테스트
curl http://localhost

# 또는 브라우저에서 열기
http://localhost
```

## 디렉토리 구조

### Linux 기본 디렉토리

| 경로 | 설명 |
|------|------|
| /etc/nginx/ | 구성 디렉토리 |
| /etc/nginx/nginx.conf | 주요 구성 파일 |
| /etc/nginx/conf.d/ | 하위 구성 디렉토리 |
| /var/log/nginx/ | 로그 디렉토리 |
| /var/www/html/ | 기본 문서 루트 |
| /usr/sbin/nginx | 실행 파일 |
| /usr/share/nginx/html/ | 기본 정적 파일 디렉토리 |

### 컴파일 설치 디렉토리

| 경로 | 설명 |
|------|------|
| /usr/local/nginx/ | 설치 루트 디렉토리 |
| /usr/local/nginx/conf/ | 구성 디렉토리 |
| /usr/local/nginx/html/ | 문서 루트 |
| /usr/local/nginx/logs/ | 로그 디렉토리 |
| /usr/local/nginx/sbin/ | 실행 파일 디렉토리 |

## Nginx 제거

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### 컴파일 설치

```bash
# Nginx 중지
sudo nginx -s stop

# 설치 디렉토리 삭제
sudo rm -rf /usr/local/nginx
```

## 일반적인 문제

### 포트 충돌

포트 충돌로 시작 실패 시:

```bash
# 포트 사용량 확인
netstat -tulpn | grep :80

# Nginx 포트 변경
# /etc/nginx/nginx.conf 편집
# listen 80;을 listen 8080;으로 변경
```

### 권한 문제

권한 오류 발생 시:

```bash
# Nginx 사용자 확인
ps aux | grep nginx

# 파일 권한 수정
sudo chown -R www-data:www-data /var/www/html/
```

### 구성 구문 확인

```bash
# 구성 구문 테스트
nginx -t

# 지정된 구성으로 테스트
nginx -t -c /path/to/nginx.conf
```
