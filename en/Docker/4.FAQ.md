## 1. Installation Issues

### Q1: Docker Desktop fails to start?

**Solutions**:
1. Ensure virtualization is enabled in BIOS
2. Ensure WSL 2 is properly installed (Windows)
3. Restart computer and try again

---

### Q2: docker command not found?

Ensure Docker is properly installed and added to PATH.

```bash
# Reload shell
source ~/.bashrc
```

---

### Q3: Permission denied?

```bash
# Add user to docker group
sudo usermod -aG docker $USER

# Log out and log back in
```

## 2. Image Issues

### Q4: Image download too slow?

Configure domestic registry mirrors:

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn"
  ]
}
```

---

### Q5: How to remove unused images?

```bash
# Remove dangling images
docker image prune

# Remove all unused images
docker image prune -a
```

---

## 3. Container Issues

### Q6: Container won't start?

```bash
# View container logs
docker logs container_id

# View details
docker inspect container_id
```

---

### Q7: Container exits automatically?

Check if startup command is correct:

```dockerfile
CMD ["tail", "-f", "/dev/null"]
```

---

### Q8: How to enter running container?

```bash
docker exec -it container_id /bin/bash
```

---

## 4. Network Issues

### Q9: Container can't access external network?

```bash
# Check DNS
docker exec container_id cat /etc/resolv.conf

# Restart Docker
sudo systemctl restart docker
```

---

### Q10: Port conflict?

```bash
# Check port usage
netstat -tulpn | grep PORT

# Use different port
docker run -p 8081:80 nginx
```

---

## 5. Storage Issues

### Q11: Docker taking too much space?

```bash
# View space usage
docker system df

# Clean all unused resources
docker system prune -a
```

---

### Q12: How to persist data?

Use volumes:

```bash
docker run -v /host/path:/container/path my-app
```

---

## 6. Performance Issues

### Q13: Container running slow?

1. Increase resource limits:
```bash
docker run -m 2g --cpus=2 my-app
```

2. Use lightweight images:
```dockerfile
FROM node:18-alpine
```

---

### Q14: Build too slow?

Use build cache:

```dockerfile
# Copy dependencies first
COPY package*.json ./
RUN npm install
# Then copy code
COPY . .
```
