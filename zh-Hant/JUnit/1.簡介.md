# JUnit 簡介

JUnit 是 Java 平台上最流行的單元測試框架，用於編寫和運行可重複的測試。它是開發高品質 Java 程式碼的基礎工具，被全球數百萬開發者廣泛使用。

## JUnit 4 與 JUnit 5 的區別

### 版本概述

- **JUnit 4**：2010 年發布，是 Java 社群長期使用的測試框架的最後一個大版本
- **JUnit 5**：2017 年發布，是 JUnit 框架的全新升級版，包含三個子項目：JUnit Platform、JUnit Jupiter、JUnit Vintage

### 架構差異

JUnit 4 採用單體架構，所有功能整合在單個函式庫中。JUnit 5 採用模組化架構，由三個獨立專案組成：

- **JUnit Platform**：測試框架的基礎，提供測試引擎 API、命令列啟動器
- **JUnit Jupiter**：新的程式設計模型和擴展模型，用於編寫測試和擴展
- **JUnit Vintage**：相容引擎，允許在 JUnit 5 平台上執行 JUnit 3 和 JUnit 4 測試

### 主要區別對比

| 特性 | JUnit 4 | JUnit 5 |
|------|---------|---------|
| 最低 Java 版本 | Java 5+ | Java 8+ |
| 注解套件 | org.junit | org.junit.jupiter.api |
| 測試類無需繼承 | 否 | 是 |
| 斷言風格 | Assert 靜態方法 | Assertions 靜態方法 |
| 參數化測試 | @Parameters | @ParameterizedTest |
| 巢狀測試 | 不支援 | 支援 |
| 動態測試 | 不支援 | 支援 |
| 平行執行 | 有限支援 | 原生支援 |
| 擴展模型 | @RunWith | @ExtendWith |

### 選擇建議

- **新專案**：推薦使用 JUnit 5，利用其現代特性和更好的擴展性
- **既有專案**：可繼續使用 JUnit 4，或使用 JUnit Vintage 進行漸進式遷移
- **長期維護**：建議逐步遷移到 JUnit 5，享受更好的測試體驗
