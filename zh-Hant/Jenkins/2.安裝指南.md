# Jenkins 安裝指南

## 系統需求

在安裝 Jenkins 之前，需要確保系統滿足以下最低配置需求：

### 最低推薦配置

- 256MB 可用記憶體
- 1GB 可用磁碟空間（作為 Docker 容器執行則推薦 10GB）

### 小團隊推薦配置

- 1GB+ 可用記憶體
- 50GB+ 可用磁碟空間

### 軟體需求

- Java 8 或更高版本（無論是 JRE 還是 JDK 都可以）
- 注意：如果將 Jenkins 作為 Docker 容器執行，則不需要在主機上安裝 Java

## 安裝平台

Jenkins 支援多種安裝方式，本文詳細介紹幾種常用的安裝方法。

## Docker 安裝

Docker 是最推薦的 Jenkins 安裝方式，因為它提供了隔離的執行環境，簡化了設定過程。

### 安裝 Docker

首先需要在系統上安裝 Docker。存取 Docker 官方網站，根據您的作業系統選擇合適的 Docker Community Edition 版本進行安裝。

### 下載並執行 Jenkins

在 macOS 和 Linux 上，開啟終端機視窗，執行以下命令：

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

在 Windows 上，開啟命令提示字元視窗，執行類似命令（使用 ^ 換行）：

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

參數說明：

- `-u root`：以 root 使用者執行容器
- `--rm`：容器停止時自動刪除
- `-d`：在背景執行（分離模式）
- `-p 8080:8080`：將容器連接埠 8080 對應到主機連接埠 8080
- `-p 50000:50000`：將容器連接埠 50000 對應到主機連接埠 50000（用於 JNLP 代理程式通訊）
- `-v jenkins-data:/var/jenkins_home`：持久化 Jenkins 資料
- `-v /var/run/docker.sock:/var/run/docker.sock`：允許 Jenkins 在流水線中執行 Docker 命令
- `jenkinsci/blueocean`：Jenkins 官方映像檔，包含 Blue Ocean 介面

### 存取 Jenkins 容器

如果需要進入 Jenkins 容器內部進行操作，可以使用以下命令：

```bash
docker exec -it jenkins-blueocean bash
```

### 查看 Jenkins 日誌

可以透過以下命令查看 Jenkins 控制台日誌：

```bash
docker logs jenkins-blueocean
```

## WAR 檔案安裝

Jenkins 的 WAR 檔案版本可以安裝在任何支援 Java 的作業系統上。

### 安裝步驟

1. 從官方網站下載最新的穩定 Jenkins WAR 套件
2. 在下載目錄中開啟終端機或命令提示字元
3. 執行命令：`java -jar jenkins.war`
4. 瀏覽器存取 http://localhost:8080，等待解鎖頁面出現

### 注意事項

- WAR 檔案方式不會自動安裝 Blue Ocean 功能，需要後續透過外掛程式管理手動安裝
- 可以透過 `--httpPort` 參數更改連接埠，例如：`java -jar jenkins.war --httpPort=9090`

## macOS 安裝

### 使用套件安裝

1. 下載 macOS 版本的 Jenkins 套件
2. 雙擊開啟並按照安裝精靈操作

### 使用 Homebrew 安裝

安裝最新版本：

```bash
brew install jenkins
```

安裝 LTS 版本：

```bash
brew install jenkins-lts
```

## Linux 安裝（Debian/Ubuntu）

在基於 Debian 的發行版上，可以透過 apt 套件管理員安裝 Jenkins：

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

安裝完成後，Jenkins 會：

- 設定為開機自動啟動的守護行程
- 建立 jenkins 使用者執行服務
- 輸出日誌到 /var/log/jenkins/jenkins.log
- 預設監聽 8080 連接埠

## 安裝後設定精靈

首次啟動 Jenkins 時，會進入安裝精靈。

### 解鎖 Jenkins

1. 瀏覽器存取 http://localhost:8080
2. 解鎖 Jenkins 頁面出現後，從控制台日誌中複製自動產生的密碼
3. 將密碼貼上到管理員密碼欄位，點擊繼續

### 自訂 Jenkins 外掛程式

在外掛程式安裝頁面，可以選擇：

- 安裝建議的外掛程式：安裝推薦的一組外掛程式，基於最常見用例
- 選擇要安裝的外掛程式：手動選擇需要安裝的外掛程式集

### 建立第一個管理員使用者

最後一步是建立管理員使用者，需要填寫：

- 使用者名稱
- 密碼
- 確認密碼
- 全名
- 電子郵件地址

完成後，點擊「儲存並完成」，然後點擊「開始使用 Jenkins」進入主介面。

## 驗證安裝

安裝完成後，可以透過以下方式驗證：

1. 瀏覽器存取 http://localhost:8080
2. 使用建立的管理員帳戶登入
3. 檢查系統狀態是否正常

## 常見問題

### 連接埠被佔用

如果 8080 連接埠被其他程式佔用，可以在啟動時指定其他連接埠：

- Docker 方式：修改 -p 參數
- WAR 方式：使用 --httpPort 參數

### 記憶體不足

如果系統記憶體較小，可以考慮：

- 使用 Docker 方式，透過容器資源限制
- 調整 Jenkins 啟動參數中的 JVM 堆記憶體大小

### 首次啟動慢

首次啟動時需要下載和安裝外掛程式，可能需要較長時間，請耐心等待。

## 下一步

安裝完成後，建議學習：

- 建立第一個流水線任務
- 設定程式碼來源（如 Git）
- 設定建置觸發器
- 整合測試和部署工具
