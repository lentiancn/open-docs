# Element Plus Preguntas frecuentes

Preguntas y respuestas comunes.

## Problemas de instalación

### Q: ¿La instalación falla?

**Soluciones:**
1. Verificar versión de Node.js
```bash
node --version  # Requiere 18.0+
```
2. Limpiar caché de npm
```bash
npm cache clean --force
```
3. Eliminar node_modules y package-lock.json, reinstalar
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: ¿Módulo no encontrado?

**Soluciones:**
1. Confirmar que el paquete está instalado
```bash
npm list element-plus
```
2. Verificar configuración de npm/yarn/pnpm
3. Intentar eliminar archivo lock y reinstalar

### Q: ¿Errores de tipo TypeScript?

**Soluciones:**
1. Instalar definiciones de tipos
```bash
npm install -D @types/node
```
2. Agregar en tsconfig.json
```json
{
  "compilerOptions": {
    "types": ["element-plus/global"]
  }
}
```

## Problemas de uso

### Q: ¿El componente no se muestra?

**Soluciones:**
1. Confirmar importación correcta
2. Verificar mensajes de error en consola
3. Confirmar versión de Vue (3.2+ requerida)

### Q: ¿Los estilos no se aplican?

**Soluciones:**
1. Confirmar que CSS está importado
```typescript
import 'element-plus/dist/index.css'
```
2. Verificar prioridad CSS, usar `:deep()` o `!important`
3. Verificar sobrescrituras de estilos por otras reglas

### Q: ¿Los iconos no se muestran?

**Soluciones:**
1. Instalar paquete de iconos
```bash
npm install @element-plus/icons-vue
```
2. Registrar globalmente
```typescript
import * as icons from '@element-plus/icons-vue'
for (const [key, component] of Object.entries(icons)) {
  app.component(key, component)
}
```
3. O importar bajo demanda

### Q: ¿La tabla no se renderiza?

**Soluciones:**
1. Confirmar que la propiedad data está correctamente vinculada
2. Verificar que los props son correctos
3. Confirmar que la tabla está fuera de el-table-column

### Q: ¿La validación del formulario no funciona?

**Soluciones:**
1. Asegurarse de que el prop corresponde a v-model
2. Asegurarse de que hay referencia ref
3. Llamar correctamente el método validate
```typescript
const formRef = ref()
await formRef.value.validate()
```

### Q: ¿El diálogo no se muestra?

**Soluciones:**
1. Verificar vinculación v-model
2. Confirmar que visible es true
3. Verificar conflictos de z-index

## Problemas de tema

### Q: ¿Cómo cambiar el color del tema?

**Soluciones:**
Usar variables CSS:
```css
:root {
  --el-color-primary: #409eff;
}
```

### Q: ¿Cómo usar tema oscuro?

**Soluciones:**
Importar CSS oscuro:
```typescript
import 'element-plus/theme-chalk/dark/css-vars.css'
```

### Q: ¿Cómo sobrescribir estilos de componentes?

**Soluciones:**
Usar selectores CSS:
```css
.el-button--primary {
  background-color: #tu-color;
}
```

## Problemas de internacionalización

### Q: ¿Cómo cambiar de idioma?

**Soluciones:**
```typescript
import es from 'element-plus/es/locale/lang/es'
import en from 'element-plus/es/locale/lang/en'

app.use(ElementPlus, { locale: es })
```

### Q: ¿Personalizar textos?

**Soluciones:**
```typescript
import {ElMessage} from 'element-plus'
ElMessage('Mensaje personalizado')
```

## Problemas de rendimiento

### Q: ¿La carga inicial es lenta?

**Soluciones:**
1. Usar importación bajo demanda
2. Usar plugin de importación automática
3. Configurar pre-build de Vite

### Q: ¿El tamaño del bundle es muy grande?

**Soluciones:**
1. Asegurarse de que se usa importación bajo demanda
2. Usar tree shaking
3. Verificar si se importa el paquete completo

## Problemas de compatibilidad

### Q: ¿El navegador IE no es compatible?

Element Plus no soporta el navegador IE. Usar:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### Q: ¿Conflictos con otras bibliotecas?

**Soluciones:**
1. Verificar niveles de z-index
2. Usar CSS Modules para evitar conflictos de estilos
3. Verificar compatibilidad de versión de Vue
