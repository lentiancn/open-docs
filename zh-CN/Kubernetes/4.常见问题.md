# Kubernetes 常见问题

## 基础概念

### 什么是 Kubernetes？

Kubernetes（也称为 K8s）是一个开源的容器编排平台，用于自动化部署、扩缩和管理容器化应用程序。它将组成应用程序的容器组合成逻辑单元，便于管理和服务发现。

### Pod 和容器有什么区别？

Pod 是 Kubernetes 中的最小部署单元，一个 Pod 可以包含一个或多个容器。通常建议每个 Pod 只运行一个容器。同一 Pod 内的容器共享网络和存储资源。

### Deployment 和 StatefulSet 有什么区别？

Deployment 用于管理无状态应用，支持声明式更新、扩缩和回滚。StatefulSet 用于管理有状态应用，提供稳定的网络标识和持久化存储，适合数据库等需要持久化的应用。

### 什么是 Namespace？

Namespace 用于将集群资源划分为多个逻辑分区，帮助在不同团队或项目之间隔离资源。常见的默认命名空间包括 default、kube-system 和 kube-public。

## 安装和配置

### Minikube 和 Kind 有什么区别？

Minikube 创建一个完整的单节点 Kubernetes 集群，适合学习和开发。Kind 使用 Docker 容器模拟 Kubernetes 节点，更轻量，适合快速测试。

### 如何选择容器运行时？

常见的容器运行时包括 Docker、containerd 和 CRI-O。containerd 是 Docker 的核心容器运行时，目前被广泛推荐使用。

### kubeadm 初始化失败怎么办？

常见原因包括：
- 端口被占用：检查 6443、10250、10255 等端口
- Swap 未关闭：执行 `swapoff -a`
- 防火墙规则：确保节点间网络通信正常
- SELinux/AppArmor：可以设置为宽容模式或关闭

### 如何配置多主节点集群？

使用 kubeadm 可以创建高可用集群：
1. 准备多个服务器节点
2. 使用 kubeadm init 创建第一个控制平面节点
3. 生成证书并复制到其他节点
4. 使用 kubeadm join 添加额外的控制平面节点

## 网络和存储

### Pod 无法互相通信怎么办？

常见原因包括：
- 网络插件未正确安装
- 防火墙阻止了 Pod 网络 CIDR 的通信
- CNI 插件配置错误
- 检查 kube-proxy 是否正常运行

### 如何为 Pod 配置持久化存储？

需要创建 PersistentVolume（PV）和 PersistentVolumeClaim（PVC）。PV 可以是本地存储、网络存储（NFS、iSCSI）或云存储（AWS EBS、GCE PD、Azure Disk）。

### Service 类型有哪些？

- ClusterIP：集群内部可访问的虚拟 IP
- NodePort：在每个节点上开放一个端口
- LoadBalancer：使用云提供商的负载均衡器
- ExternalName：将服务映射到外部域名

## 资源管理

### 如何设置容器资源限制？

在 Pod 或 Deployment 的 spec 中添加 resources 字段：
```yaml
resources:
  limits:
    memory: "512Mi"
    cpu: "500m"
  requests:
    memory: "256Mi"
    cpu: "250m"
```

### 什么是资源配额（ResourceQuota）？

ResourceQuota 用于限制命名空间中的资源总量，防止单个团队或项目消耗过多资源。

### 什么是 LimitRange？

LimitRange 用于为命名空间中的 Pod 和容器设置默认的资源限制和请求值。

## 部署和运维

### 如何实现应用的滚动更新？

使用 Deployment 时，Kubernetes 自动进行滚动更新：
```bash
kubectl set image deployment/my-app my-app=my-app:v2
kubectl rollout status deployment/my-app
```

### 如何回滚部署？

```bash
# 回滚到上一个版本
kubectl rollout undo deployment/my-app

# 回滚到指定版本
kubectl rollout undo deployment/my-app --to-revision=2
```

### 什么是 HPA？

Horizontal Pod Autoscaler（HPA）根据 CPU、内存等指标自动调整 Pod 副本数。需要配合 Metrics Server 使用。

### 如何配置健康检查？

使用探针（Probes）配置存活性和就绪检查：
```yaml
livenessProbe:
  httpGet:
    path: /healthz
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

## 安全

### 什么是 RBAC？

基于角色的访问控制（RBAC）用于管理 Kubernetes 集群中的权限。可以创建 Role 和 ClusterRole，并绑定到用户或组。

### 如何管理敏感信息？

使用 Secret 存储敏感信息，如密码、密钥和证书。可以使用 kubectl 命令行创建或通过 YAML 文件定义。

### 如何保护集群安全？

- 启用 RBAC 授权
- 使用网络策略限制 Pod 间的通信
- 定期更新 Kubernetes 版本
- 加密 etcd 数据
- 使用 TLS 加密通信
- 限制 kubelet 的访问权限

## 故障排查

### Pod 一直处于 Pending 状态？

可能是以下原因：
- 资源不足（CPU 或内存）
- 节点选择器或污点不匹配
- PVC 未正确绑定
- 使用 kubectl describe pod <pod-name> 查看详细信息

### Pod 一直处于 ImagePullBackOff？

常见原因：
- 镜像名称错误
- 镜像标签不存在
- 私有仓库认证问题
- 网络问题导致无法拉取镜像

### Pod 一直处于 CrashLoopBackOff？

- 检查容器日志：kubectl logs --previous <pod-name>
- 检查应用程序是否正确启动
- 验证资源配置是否正确
- 检查依赖服务是否可用

### 如何调试网络问题？

- 检查 kube-proxy 是否正常运行
- 检查 CNI 插件状态
- 使用 kubectl exec 进入容器测试网络连通性
- 查看 iptables 规则
- 使用网络诊断工具（如 curl、ping、traceroute）

## 性能优化

### 如何提高集群性能？

- 合理配置资源请求和限制
- 使用反亲和性分布 Pod
- 启用缓存减少 API 请求
- 使用本地存储减少网络延迟
- 优化镜像大小减少拉取时间

### 如何处理大规模集群？

- 使用联邦（Federation）管理多个集群
- 配置合适的 etcd 硬件
- 启用 API 缓存
- 使用负载均衡分发请求
