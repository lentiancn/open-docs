# Apache JMeter 소개

Apache JMeter는 Apache Software Foundation의 오픈소스 소프트웨어로, 로드 테스트 및 성능 측정을 위해 설계된 100% 순수 Java 애플리케이션입니다. JMeter는 원래 웹 애플리케이션 테스트를 위해 설계되었지만 이후 다른 테스트 기능으로 확장되었습니다.

## JMeter로 무엇을 할 수 있나요?

JMeter는 웹 애플리케이션을 포함한 정적 및 동적 리소스의 성능 테스트에 사용할 수 있습니다. 서버, 서버 그룹, 네트워크 또는 객체에 대한 높은 부하를 시뮬레이션하여 강도를 테스트하거나 다양한 부하 유형에서 전체 성능을 분석할 수 있습니다.

주요 기능 특성:

- **다중 프로토콜 지원**: 다양한 애플리케이션, 서버, 프로토콜 유형에 대한 로드 및 성능 테스트 기능:
  - 웹 - HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET 등)
  - SOAP / REST 웹서비스
  - FTP
  - JDBC를 통한 데이터베이스
  - LDAP
  - JMS를 통한 메시지 지향 미들웨어 (MOM)
  - 메일 - SMTP(S), POP3(S) 및 IMAP(S)
  - 네이티브 명령어 또는 셸 스크립트
  - TCP
  - Java 객체

- **전체 기능 테스트 IDE**: 브라우저 또는 네이티브 애플리케이션에서 빠른 테스트 계획 녹화, 빌드 및 디버깅 지원

- **CLI 모드**: 명령줄 모드(이전에는 Non GUI 또는 헤드리스 모드)는 Java 호환 OS(Linux, Windows, Mac OSX 등)에서 로드 테스트를 수행

- **동적 HTML 보고서**: 완전히 준비된 동적 HTML 보고서 생성 기능

- **데이터 추출**: HTML, JSON, XML 또는 모든 텍스트 형식에서 데이터를 추출하는 기능

- **100% Java 이식성**: 완전히 이식 가능하며 100% 순수 Java 구현

- **다중 스레드 프레임워크**: 많은 스레드동시 샘플링 및 별도 스레드 그룹다양한 기능 동시 샘플링

- **높은 확장성**: 플러그형 샘플러, 스크립트 가능 샘플러(JSR223 호환 언어 like Groovy 및 BeanShell)

- **지속적 통합**: Maven, Gradle 및 Jenkins 통합 지원

## JMeter는 브라우저가 아닙니다

JMeter는 프로토콜 수준에서 작동합니다. 웹 서비스 및 원격 서비스의 관점에서 JMeter는 브라우저(또는 여러 브라우저처럼) 보입니다. 그러나 JMeter는 브라우저가 지원하는 모든 작업을 수행하지 않습니다. 특히 JMeter는 HTML 페이지에서 발견된 JavaScript를 실행하지 않으며 브라우저처럼 HTML 페이지를 렌더링하지 않습니다.

## JMeter 적용 시나리오

- **로드 테스트**: 높은 부하에서 시스템 성능을 테스트하기 위해 많은 동시 사용자 시뮬레이션
- **성능 테스트**: 다양한 부하 조건에서 시스템 응답 시간 및 처리량 측정
- **기능 테스트**: 애플리케이션의 기능 동작 확인
- **스트레스 테스트**: 높은 압력에서 시스템 안정성 및 내결함성 테스트
- **분산 테스트**: 여러 시스템에서 조정된 로드 테스트 지원
