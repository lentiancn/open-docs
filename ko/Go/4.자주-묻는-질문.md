## 설치

### Q: 설치 후 명령어를 찾을 수 없음?

**A**: 
- Go가 PATH 환경 변수에 추가되었는지 확인
- Windows는 기본적으로 C:\Go에 설치되며 PATH에 추가 필요
- Linux/macOS는 보통 ~/.bashrc 또는 ~/.zshrc에 수동으로 추가

### Q: Go가 설치되어 있는지 확인하는 방법은?

**A**: 
```bash
go version
go env GOPATH
```

### Q: Go 버전을 업그레이드하는 방법은?

**A**: 
- Windows: 새 버전 다운로드하여 재설치
- macOS: `brew upgrade go`
- Linux: 새 버전 다운로드하여 교체하거나 패키지 매니저 사용

## 문법

### Q: Go에 예외 메커니즘이 있나요?

**A**: Go에는 전통적인 의미의 예외가 없습니다. 오류는 반환값을 통해 전달됩니다:
```go
result, err := doSomething()
if err != nil {
    // 오류 처리
}
```

### Q: :=와 var를 혼합할 수 없는 이유는?

**A**: 
- `:=`는 짧은 변수 선언으로, 함수 내에서 사용
- `var`는 명시적 선언으로, 함수 외부에서도 사용 가능
- 같은 변수에 대해 동시에 사용할 수 없습니다

### Q: Go가 제네릭을 지원하나요?

**A**: Go 1.18+에서 제네릭 지원:
```go
func Print[T any](s []T) {
    for _, v := range s {
        fmt.Println(v)
    }
}
```

## 동시성

### Q: 고루틴과 스레드의 차이는?

**A**: 
- 고루틴은 Go 런타임에서 관리되어 스레드보다 가벼움
- 수천 개의 고루틴을 생성해도 리소스를 고갈시키지 않음
- 채널을 통한 통신

### Q: 채널을 닫아야 하나요?

**A**: 
- 보통 전송자가 채널을 닫음
- 수신자는 `v, ok := <-ch`로 채널이 닫혔는지 확인 가능
- 닫힌 채널로 전송하지 않음

### Q: 고루틴 종료를优雅하게 기다리는 방법은?

**A**: sync.WaitGroup 사용:
```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(i int) {
        defer wg.Done()
        fmt.Println(i)
    }(i)
}

wg.Wait()
```

## 패키지 관리

### Q: GOPATH와 Go Modules의 차이는?

**A**: 
- GOPATH:古い의존성 관리 방법, 의존성을 $GOPATH/src에 저장
- Go Module (go.mod): 현대적인 접근 방식, 더 나은 버전 관리

### Q: 의존성 다운로드가 느림?

**A**: 
```bash
# 중국 미러 사용
go env -w GOPROXY=https://goproxy.cn,direct

# 설정 가능
go env -w GOPROXY=https://proxy.golang.org,direct
```

### Q: 의존성 관계 확인 방법은?

**A**: 
```bash
go mod graph
go list -m all
```

## 성능

### Q: 성능 분석 방법은?

**A**: 
```bash
# CPU 프로파일링
go test -cpuprofile=cpu.out

# 메모리 프로파일링
go test -memprofile=mem.out

# pprof로 분석
go tool pprof cpu.out
```

### Q: Go 프로그램을 최적화하는 방법은?

**A**: 
- 문자열 연결에는 strings.Builder 사용
- 불필요한 메모리 할당 피하기
- 객체 재사용에는 풀 (sync.Pool) 사용
- 채널을 효율적으로 사용

### Q: go fmt와 go vet의 차이는?

**A**: 
- go fmt: 코드 스타일 포맷
- go vet: 정적 분석, 잠재적 오류 검사

```bash
go doc -http=:6060
```

## 오류

### Q: 일반적인 컴파일 오류는?

**A**: 
- `undefined: xxx` - 변수가 정의되지 않음
- `mismatched types` - 타입이 일치하지 않음
- `invalid character` - 문자 인코딩 문제
- `package xxx is not in GOROOT` - 패키지 경로 오류

### Q: 런타임 패닉이 발생하면 어떻게 해야 하나요?

**A**: 
- defer와 recover 사용:
```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("복구됨:", r)
        }
    }()
    // 패닉이 발생할 수 있는 코드
}
```
