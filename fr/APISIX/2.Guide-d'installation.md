# Guide d'installation

Ce document présente les différentes méthodes d'installation d'Apache APISIX pour vous permettre de mettre en place rapidement votre environnement de passerelle API.

## Prérequis

Avant d'installer APISIX, assurez-vous de disposer des éléments suivants :

- **Système d'exploitation** : Linux (Ubuntu 20.04+, CentOS 7+, Debian 11+ recommandés)
- **Docker** (pour le déploiement Docker) : Docker et Docker Compose installés
- **etcd** : APISIX utilise etcd comme stockage de configuration, nécessite etcd 3.4.0+
- **Perl** : Certaines dépendances nécessitent Perl 5.5.32+
- **curl** : Pour vérifier l'installation

## Installation rapide avec Docker (recommandée)

C'est la méthode la plus rapide, idéale pour les environnements de développement et de test.

### Étape 1 : Créer le répertoire Quick Start

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### Étape 2 : Exécuter le script Quick Start

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

Cette commande lance automatiquement deux conteneurs : apisix-quickstart et etcd. APISIX utilise etcd pour sauvegarder et synchroniser les configurations. Les conteneurs utilisent le mode réseau host de Docker, donc vous pouvez y accéder directement depuis votre machine locale.

### Étape 3 : Vérifier l'installation

En cas de succès, le terminal affichera :

```
✔ APISIX is ready!
```

Vous pouvez vérifier l'état d'APISIX avec cette commande :

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Elle devrait retourner :

```
Server: APISIX/Version
```

Où Version correspond à la version installée, par exemple APISIX/3.15.0.

### Étape 4 : Accéder au Dashboard

APISIX est livré avec un Dashboard UI intégré, accessible à l'adresse :

```
http://127.0.0.1:9180/ui
```

> ⚠️ Note : Pour une meilleure expérience, l'API d'administration est ouverte par défaut sans authentication. Veuillez activer l'authentification en production.

## Installation avec Docker Compose

Pour une configuration personnalisée, vous pouvez utiliser Docker Compose.

### Étape 1 : Créer le fichier de configuration

Créez un fichier docker-compose.yaml :

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### Étape 2 : Lancer les services

```bash
docker-compose up -d
```

### Étape 3 : Vérifier les services

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Installation sur Kubernetes

APISIX propose un Ingress Controller officiel qui peut être déployé directement sur Kubernetes.

### Installation avec Helm Chart

```bash
# Ajouter le dépôt Helm APISIX
helm repo add apisix https://charts.apiseven.com
helm repo update

# Installer APISIX
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### Installation avec kubectl

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## Installation à partir des sources

Adaptée aux environnements de production nécessitant une construction personnalisée.

### Étape 1 : Installer les dépendances

**Ubuntu/Debian :**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL :**

```bash
sudo yum install -y perl gcc git curl
```

### Étape 2 : Installer OpenResty

```bash
# Ajouter le dépôt OpenResty
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# Installer OpenResty
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### Étape 3 : Installer APISIX

```bash
# Cloner les sources
git clone https://github.com/apache/apisix.git
cd apisix

# Installer les dépendances
make deps

# Initialiser la configuration
make init

# Lancer APISIX
make run
```

## Vérifier l'installation

Quelle que soit la méthode d'installation utilisée, vous pouvez vérifier le bon fonctionnement :

1. **Vérifier l'état des processus**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **Créer une route de test**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **Tester la route**

```bash
curl "http://127.0.0.1:9080/test"
```

## Désinstallation

**Mode Docker :**

```bash
# Arrêter les conteneurs
docker-compose down

# Supprimer les volumes (si nécessaire)
docker-compose down -v
```

**Mode Kubernetes :**

```bash
helm uninstall apisix
```
