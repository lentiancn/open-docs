# Jenkins インストールガイド

## システム要件

Jenkinsをインストールする前に、システムが次の最低要件を満たしていることを確認してください：

### 最低推奨構成

- 256MBの空きメモリ
- 1GBの空きディスク容量（Dockerコンテナとして実行する場合は10GBを推奨）

### 小規模チーム推奨構成

- 1GB以上の空きメモリ
- 50GB以上の空きディスク容量

### ソフトウェア要件

- Java 8以上（JREまたはJDKのいずれか）
- 注意：DockerコンテナとしてJenkinsを実行する場合は、ホストにJavaをインストールする必要はありません

## インストールプラットフォーム

Jenkinsは複数のインストール方法をサポートしています。このガイドでは、一般的なインストール方法を説明します。

## Dockerインストール

Dockerは、Jenkinsをインストールする推奨方法です。分離された実行環境を提供し、設定プロセスを簡素化します。

### Dockerのインストール

最初に、システムにDockerをインストールする必要があります。Docker公式ウェブサイトにアクセスして、オペレーティングシステムに対応する適切なDocker Community Editionバージョンを選択してインストールします。

### Jenkinsのダウンロードと実行

macOSとLinuxでは、ターミナルウィンドウを開いて次のコマンドを実行します：

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

Windowsでは、コマンドプロンプトを開いて類似のコマンドを実行します（^を使用）：

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

パラメータの説明：

- `-u root`：rootユーザーとしてコンテナを実行
- `--rm`：コンテナが停止したときに自動的に削除
- `-d`：バックグラウンドで実行（デタッチモード）
- `-p 8080:8080`：コンテナポート8080をホストポート8080にマッピング
- `-p 50000:50000`：コンテナポート50000をホストポート50000にマッピング（JNLPエージェント通信用）
- `-v jenkins-data:/var/jenkins_home`：Jenkinsデータを永続化
- `-v /var/run/docker.sock:/var/run/docker.sock`：パイプラインでDockerコマンドを実行することを許可
- `jenkinsci/blueocean`：Blue Oceanインターフェースを含む公式Jenkinsイメージ

### Jenkinsコンテナへのアクセス

Jenkinsコンテナの内側にアクセスする必要がある場合は、次のコマンドを使用します：

```bash
docker exec -it jenkins-blueocean bash
```

### Jenkinsログの表示

Jenkinsコンソールログを表示するには：

```bash
docker logs jenkins-blueocean
```

## WARファイルインストール

JenkinsのWARファイルバージョンは、Javaをサポートする任意のオペレーティングシステムにインストールできます。

### インストール手順

1. 公式ウェブサイトから最新の安定版Jenkins WARパッケージをダウンロード
2. ダウンロードディレクトリでターミナルまたはコマンドプロンプトを開く
3. コマンドを実行：`java -jar jenkins.war`
4. ブラウザでhttp://localhost:8080にアクセスし、「Jenkinsのロック解除」ページが表示されるのを待つ

### 注意事项

- WARファイル方式ではBlue Oceanは自動インストールされません。後からプラグイン管理でインストールする必要があります
- `--httpPort`パラメータを使用してポートを変更できます（例：`java -jar jenkins.war --httpPort=9090`）

## macOSインストール

### パッケージを使用

1. macOS용Jenkinsパッケージをダウンロード
2. ダブルクリックしてインストールウィザードに従う

### Homebrewを使用

最新バージョンをインストール：

```bash
brew install jenkins
```

LTSバージョンをインストール：

```bash
brew install jenkins-lts
```

## Linuxインストール（Debian/Ubuntu）

Debianベースのディストリビューションでは、aptパッケージマネージャーでJenkinsをインストールできます：

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

インストール後、Jenkinsは以下を行います：

- ブート時にデーモンとして起動するように設定
- jenkinsユーザーを作成してサービスを実行
- ログを/var/log/jenkins/jenkins.logに出力
- デフォルトでポート8080をリッスン

## インストール後セットアップウィザード

初めてJenkinsを起動すると、セットアップウィザードが開始されます。

### Jenkinsのロック解除

1. ブラウザでhttp://localhost:8080にアクセス
2. 「Jenkinsのロック解除」ページが表示されたら、コンソールログから自動生成されたパスワードをコピー
3. パスワードを管理者パスワードフィールドに貼り付けて「続ける」をクリック

### Jenkinsプラグインのカスタマイズ

プラグインインストールページでは以下を選択できます：

- 推奨プラグインをインストール：一般的なユースケースに基づいて推奨されるプラグインのセットをインストール
- インストールするプラグインを選択：インストールするプラグインを手動で選択

### 最初の管理者ユーザーの作成

最後に管理者アカウントを作成します：

- ユーザー名
- パスワード
- パスワードの確認
- 完全名
- メールアドレス

「保存して完了」をクリックし、「Jenkinsの使用を開始」をクリックしてメインインターフェースに入ります。

## インストールの確認

インストールは以下で確認できます：

1. ブラウザでhttp://localhost:8080にアクセス
2. 作成した管理者アカウントでログイン
3. システムステータスが正常か確認

## 一般的な問題

### ポートが使用中の場合

ポート8080が他のプログラムで使用されている場合：

- Docker方式：-pパラメータを変更
- WAR方式：--httpPortパラメータを使用

### メモリ不足の場合

システムメモリが少ない場合：

- Dockerでコンテナリソース制限を使用
- Jenkins起動パラメータでJVMヒープサイズを調整

### 最初の起動が遅い場合

最初の起動ではプラグインのダウンロードとインストールが行われるため、時間がかかることがあります。

## 次のステップ

インストール後は以下を学ぶことをお勧めします：

- 最初のパイプラインジョブの作成
- ソースコード（Gitなど）の設定
- ビルドトリガーの設定
- テストとデプロイツールの統合
