# Go 常見問題

## 安裝問題

### Go 安裝失敗怎麼辦？

安裝失敗通常由幾個常見原因導致。首先檢查系統是否滿足最低要求，Linux 需要較新的核心版本，Windows 需要 64 位元系統。如果使用壓縮包安裝，確保下載了正確的版本（amd64、arm64 等）。

如果遇到權限錯誤，在 Linux 或 macOS 上使用 sudo 提升權限。確保安裝目標目錄存在且可寫：

```bash
# 建立安裝目錄
sudo mkdir -p /usr/local
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

如果問題仍然存在，嘗試重新下載安裝包，可能下載過程中檔案損壞。驗證 SHA256 校驗和：

```bash
sha256sum go1.21.5.linux-amd64.tar.gz
```

### 如何驗證 Go 是否正確安裝？

執行以下命令檢查 Go 是否正確安裝：

```bash
go version
go env GOROOT
go env GOPATH
```

如果命令輸出類似 "go1.21.5 linux/amd64"，說明安裝成功。如果提示命令未找到，檢查 PATH 環境變數是否包含 Go 的 bin 目錄。

## 開發問題

### 如何組織專案結構？

Go 1.11 引入的模組（module）是推薦的依賴管理方式。建立新專案：

```bash
mkdir myproject
cd myproject
go mod init github.com/username/myproject
```

推薦的專案結構：

```
myproject/
├── go.mod
├── go.sum
├── main.go
├── internal/
│   ├── handler/
│   └── service/
└── pkg/
    └── utils/
```

- main.go：程式入口
- internal：私有程式碼，不能被外部導入
- pkg：可供外部使用的公共庫

### GOPATH 和 Go Modules 有什麼區別？

GOPATH 是 Go 1.11 之前的依賴管理方式，需要將程式碼放在 $GOPATH/src 目錄下。Go Modules 是從 Go 1.11 引入的現代方案，可以在任意位置建立專案，透過 go.mod 檔案管理依賴。

推薦使用 Go Modules，它解決了 GOPATH 的多個限制，包括版本管理和離線開發。Go 1.16 起，Go Modules 成為預設模式。

### 如何處理循環導入？

Go 不允許包之間循環導入，這會導致編譯錯誤。解決方法是重構程式碼，消除循環依賴：

- 將共享的類型移到一個獨立的包中
- 使用介面解耦依賴
- 將程式碼合併到一個包中

檢查循環導入：

```bash
go mod tidy
```

### Goroutine 洩漏如何排查？

Goroutine 洩漏通常發生在 channel 未正確關閉或發送後無人的情況下接收。使用 pprof 工具排查：

```bash
# 啟用 pprof
import _ "net/http/pprof"

# 匯出分析資料
go tool pprof http://localhost:6060/debug/pprof/goroutine
```

或者使用 runtime.NumGoroutine() 定期列印 goroutine 數量，定位洩漏發生的位置。

### 如何進行偵錯？

VS Code 使用者可以安裝 Go 擴充功能，使用 F5 啟動偵錯。GoLand 內建了強大的偵錯支援。命令列偵錯使用 delve：

```bash
# 安裝 delve
go install github.com/go-delve/delve/cmd/dlv@latest

# 啟動偵錯
dlv debug main.go
```

## 效能問題

### 如何進行效能分析？

Go 內建了 pprof 效能分析工具。首先在程式碼中啟用效能分析端點：

```go
import (
    "net/http"
    _ "net/http/pprof"
)

func main() {
    go func() {
        http.ListenAndServe("localhost:6060", nil)
    }()
    // 主程式邏輯
}
```

然後使用 pprof 分析：

```bash
go tool pprof http://localhost:6060/debug/pprof/profile  # CPU 分析
go tool pprof http://localhost:6060/debug/pprof/heap     # 記憶體分析
```

### 如何優化記憶體使用？

使用 sync.Pool 減少記憶體配置：

```go
var bufferPool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

func process() {
    buf := bufferPool.Get().([]byte)
    defer bufferPool.Put(buf)
    // 使用 buf
}
```

使用 -ldflags 減小二進制檔案大小：

```bash
go build -ldflags="-s -w" main.go
```

### 如何編寫高效能程式碼？

- 避免在迴圈中建立物件，盡量復用
- 使用 defer 釋放資源，但注意 defer 有效能開銷
- 使用 strconv 而不是 fmt 進行類型轉換
- 使用 strings.Builder 進行字串拼接
- 使用切片預分配容量：make([]int, 0, 100)

## 部署問題

### 如何編譯不同平台的程式？

使用 GOOS 和 GOARCH 環境變數進行交叉編譯：

```bash
# Linux
GOOS=linux GOARCH=amd64 go build -o myapp-linux

# macOS
GOOS=darwin GOARCH=amd64 go build -o myapp-macos

# Windows
GOOS=windows GOARCH=amd64 go build -o myapp.exe
```

### 如何減小二進制檔案大小？

靜態編譯可以減少依賴：

```bash
CGO_ENABLED=0 go build -ldflags="-s -w" main.go
```

還可以使用 upx 壓縮：

```bash
upx -9 myapp
```

### Docker 多階段建構

使用多階段建構減小映像大小：

```dockerfile
# 建構階段
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o main .

# 執行階段
FROM alpine
WORKDIR /app
COPY --from=builder /app/main .
CMD ["./main"]
```

## 學習問題

### 如何學習 Go 語言？

推薦的學習路徑：

1. 閱讀 Go 官方文件：https://go.dev/doc/
2. 完成 A Tour of Go 互動教程
3. 閱讀《Go 語言聖經》（The Go Programming Language）
4. 實踐：嘗試用 Go 寫一個小專案
5. 閱讀優秀的開源專案原始碼

### Go 適合開發什麼類型的應用？

Go 特別適合開發：微服務、命令列工具、網路伺服器、分散式系統、雲端運算平台、容器編排工具（如 Kubernetes、Docker）、區塊鏈應用等。

Go 不適合開發需要複雜物件導向特性的應用，或者對啟動速度要求極高的場景（可以考慮 Rust）。

### 推薦的 Go 專案有哪些？

- Kubernetes：容器編排平台
- Docker：容器執行時
- Prometheus：監控系統
- etcd：分散式鍵值儲存
- Hugo：靜態網站生成器
- Gin：Web 框架
- Cobra：命令列工具框架
