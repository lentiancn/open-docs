# Guide d'installation de Kubernetes

## Prérequis

Avant d'installer Kubernetes, comprenez les exigences de base :

### Exigences système
- Linux est le système d'exploitation recommandé pour exécuter le plan de contrôle Kubernetes
- Dans votre cluster, vous pouvez exécuter des applications sur Linux ou d'autres systèmes d'exploitation, y compris Windows
- Un système d'exploitation supportant le runtime de conteneurs est requis

### Choix du type d'installation
Lors de l'installation de Kubernetes, Choisissez un type d'installation basé sur :
- Facilité de maintenance
- Sécurité
- Contrôle
- Ressources disponibles
- Expertise requise pour operates et gérer un cluster

## Méthodes d'Installation

### Environnement d'apprentissage (Recommandé pour les débutants)

Si vous apprenez Kubernetes, utilisez les outils soutenus par la communauté Kubernetes ou les outils de l'écosystème pour configurer un cluster sur votre machine locale :

#### Minikube
Minikube est une implémentation Kubernetes légère qui crée un cluster à nœud unique sur votre machine locale. Idéal pour l'apprentissage et les tests à petite échelle.

Étapes d'installation :
1. Installer Minikube
2. Installer kubectl (outil en ligne de commande Kubernetes)
3. Démarrer le cluster Minikube

#### Kind
Kind (Kubernetes in Docker) est un autre outil pour créer des clusters Kubernetes locaux, utilisant des conteneurs Docker pour simuler des nœuds.

### Environnement de production

Pour les environnements de production, considérez ces options :

#### kubeadm
L'outil officiellement soutenu pour déployer Kubernetes. Crée des clusters sécurisés en suivant les meilleures pratiques.

Étapes de base :
1. Préparer les serveurs (installer l'OS, configurer le réseau)
2. Installer le runtime de conteneurs (containerd ou Docker)
3. Installer kubeadm, kubelet et kubectl
4. Initialiser le cluster avec kubeadm
5. Configurer le plugin réseau (Calico, Cilium, etc.)
6. Joindre les nœuds workers

#### Services managés
Si vous ne souhaitez pas gérer un cluster Kubernetes vous-même, considérez les services managés :
- Amazon EKS (Elastic Kubernetes Service)
- Google GKE (Google Kubernetes Engine)
- Azure AKS (Azure Kubernetes Service)
- Alibaba Cloud ACK
- Tencent Cloud TKE

#### Autres solutions
Il existe également des solutions standardisées et personnalisées dans un large éventail d'environnements cloud et bare metal.

## Installation des Outils

### Installer kubectl

kubectl est l'outil en ligne de commande pour interagir avec les clusters Kubernetes.

Installation Linux :
```bash
# Télécharger la dernière version
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# Ajouter les permissions d'exécution
chmod +x kubectl

# Déplacer vers le chemin système
sudo mv kubectl /usr/local/bin/
```

### Installer kubeadm

```bash
# Mettre à jour l'index des paquets apt
sudo apt-get update

# Installer les dépendances
sudo apt-get install -y apt-transport-https ca-certificates curl

# Ajouter le dépôt APT Kubernetes
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Mettre à jour et installer
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## Initialisation du Cluster

### Créer un Cluster avec kubeadm

1. Initialiser le nœud du plan de contrôle :
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. Configurer kubectl :
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. Installer le plugin réseau (Flannel en exemple) :
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. Joindre les nœuds workers :
```bash
# Exécuter sur chaque nœud worker la commande de sortie d'init
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## Vérifier l'Installation

Vérifier l'état du cluster :
```bash
# Voir les nœuds
kubectl get nodes

# Voir les pods
kubectl get pods --all-namespaces
```

Si tous les composants sont à l'état Running, l'installation a réussi.

## Désinstallation

Si vous avez besoin de désinstaller Kubernetes :

```bash
# Arrêter tous les services Kubernetes
sudo kubeadm reset

# Supprimer les fichiers de configuration
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# Désinstaller les paquets
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## Notes Importantes

1. Choix du runtime de conteneurs : Choisissez un runtime approprié pour votre nouveau cluster
2. Planification des ressources : Allouez les ressources CPU et mémoire de manière raisonnable selon les charges de travail
3. Planification du réseau : Planifiez les CIDR réseau Pod et Service à l'avance
4. Durcissement de la sécurité : Configurez l'authentification, l'autorisation et les politiques réseau pour la production
5. Haute disponibilité : Configurez des clusters multi-plans de contrôle pour les environnements de production
