# Yarn 常见问题

## 安装相关问题

### Q: 如何选择 Yarn 版本？
**A**: 
- **新项目**: 推荐使用 Yarn v4+，享受现代特性和更好的性能
- **现有项目**: 如果已在使用 Yarn v1，可以继续使用或考虑迁移到现代版本
- **企业环境**: Yarn v1 提供更稳定的长期支持，而 v4+ 提供更多创新功能

### Q: Corepack 是什么？为什么需要它？
**A**: Corepack 是 Node.js 附带的中介工具，允许您基于每个项目配置包管理器版本。它确保团队成员使用相同的 Yarn 版本，避免版本不一致问题。

### Q: 安装 Yarn 时遇到权限错误怎么办？
**A**: 
- 使用 `npm install --global yarn` 时添加 `--unsafe-perm` 标志
- 或者使用 nvm 管理 Node.js 版本
- 在 CI/CD 环境中，通常不需要全局安装

## 使用相关问题

### Q: Yarn 和 npm 有什么区别？
**A**: 
- **速度**: Yarn 使用并行安装和离线缓存，通常更快
- **确定性**: Yarn 保证安装的一致性（yarn.lock）
- **功能**: Yarn 提供工作区、PnP、插件系统等高级功能
- **命令**: Yarn 重新实现了所有命令，提供更好的开发者体验

### Q: 什么是 PnP (Plug'n'Play)？
**A**: PnP 是 Yarn v2+ 的一项功能，它消除了 node_modules 目录，直接从缓存中解析依赖。这节省了磁盘空间并提高了安装速度，但可能与某些不支持 PnP 的工具不兼容。

### Q: 如何在项目中使用工作区？
**A**: 
1. 在根目录的 `package.json` 中添加 `"workspaces"` 字段
2. 创建子包目录（如 `packages/`）
3. 在子包中创建各自的 `package.json`
4. 使用 `yarn workspace <name> <command>` 在特定工作区中运行命令

## 版本迁移问题

### Q: 如何从 Yarn v1 迁移到 v4+？
**A**: 
1. 确保项目使用 Git 进行版本控制
2. 运行 `yarn set version berry`（对于 v2/v3）或 `yarn set version stable`（对于 v4+）
3. 按照迁移指南逐步更新配置
4. 测试所有功能确保兼容性

### Q: 迁移后遇到兼容性问题怎么办？
**A**: 
- 检查是否有工具不支持 PnP，可以禁用 PnP：`yarn config set nodeLinker node-modules`
- 更新依赖到兼容版本
- 查看官方迁移文档获取具体解决方案

## 故障排除

### Q: `yarn install` 失败怎么办？
**A**: 
- 清除缓存：`yarn cache clean`
- 删除 `yarn.lock` 并重新安装
- 检查网络连接和注册表配置
- 对于 Yarn v4+，检查 `.yarnrc.yml` 配置

### Q: 如何查看 Yarn 版本？
**A**: 
- 全局版本：`yarn --version`
- 项目版本：查看 `.yarn/releases/` 目录或 `.yarnrc.yml` 文件

### Q: Yarn 命令执行很慢怎么办？
**A**: 
- 确保启用了离线缓存
- 检查网络连接
- 对于大型项目，考虑使用工作区分割
- 更新到最新版本以获得性能改进
