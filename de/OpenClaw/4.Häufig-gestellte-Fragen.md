# OpenClaw Häufig Gestellte Fragen

Diese Seite listet häufige Fragen und Lösungen bei der Verwendung von OpenClaw.

## Installation

### Was sind die Anforderungen für OpenClaw?

- Node.js 22.x oder höher
- macOS, Linux oder Windows (WSL2)
- Mindestens 2GB RAM empfohlen

### Installation schlägt fehl. Was soll ich tun?

1. Node.js-Version prüfen: `node --version` (muss >= 22 sein)
2. Netzwerkverbindung prüfen
3. Verschiedene Paketmanager versuchen: npm, pnpm oder bun
4. `openclaw doctor` zur Diagnose ausführen

### Wie aktualisiere ich OpenClaw?

```bash
# Auf stabilen Kanal wechseln
openclaw update --channel stable

# Dann Gateway neu starten
openclaw gateway restart
```

## Gateway

### Gateway startet nicht

- Prüfen, ob Port 18789 belegt ist
- Logs prüfen: `openclaw gateway --verbose`
- Versuchen, bestehende Prozesse zu beenden

### Wie halte ich Gateway ständig am Laufen?

Mit `--install-daemon` als Systemdienst installieren:

```bash
openclaw onboard --install-daemon
```

### Wie greife ich remote auf Gateway zu?

1. **Tailscale**: `gateway.tailscale.mode: "serve"` oder `"funnel"` verwenden
2. **SSH-Tunnel**: SSH-Tunnelverbindung herstellen

## Kanäle

### WhatsApp-Login fehlgeschlagen

- Stellen Sie sicher, dass Sie Ihr Gerät mit `openclaw channels login` gepairt haben
- Prüfen, ob das Netzwerk WhatsApp-Server erreichen kann

### Telegram-Bot funktioniert nicht

- Prüfen, ob Bot-Token korrekt ist
- Verifizieren, dass Sie einen Chat mit dem Bot gestartet haben
- `allowFrom`-Konfiguration verifizieren

### Discord-Nachrichten reagieren nicht

- Prüfen, ob Bot dem Server beigetreten ist
- Bestätigen, dass korrekte Berechtigungen konfiguriert sind
- Prüfen, ob Kanal auf der Erlaubnisliste steht

## Sicherheit

### Können Fremde mir Nachrichten schicken?

Standardmäßig erfordert OpenClaw eine Pairing-Verifikation für unbekannte Absender. Verwenden Sie `dmPolicy: "open"` für den offenen Modus, aber nicht empfohlen.

### Wie verwalte ich Pairing?

```bash
# Pairing-Liste anzeigen
openclaw pairing list

# Pairing genehmigen
openclaw pairing approve

# Pairing widerrufen
openclaw pairing revoke
```

### `openclaw doctor` zeigt Warnungen?

Folgen Sie den Anweisungen zur Behebung. Häufige Probleme:
- DM-Richtlinie zu offen
- Erforderliche Auth-Konfiguration fehlt
- Port-Exposing-Risiken

## Fähigkeiten

### Wie installiere ich neue Fähigkeiten?

```bash
# Über ClawHub installieren
openclaw skill install fähigkeits-name

# Oder manuell Fähigkeitsverzeichnis und SKILL.md erstellen
```

### Fähigkeiten funktionieren nicht?

- Prüfen, ob Fähigkeitskonfiguration korrekt ist
- Gateway-Logs prüfen
- Bestätigen, dass Fähigkeitsabhängigkeiten installiert sind

## Sprache

### Voice Wake funktioniert nicht?

- macOS: Systemeinstellungen auf Barrierefreiheits-Berechtigungen prüfen
- iOS: Bestätigen, dass Mikrofonberechtigung aktiviert ist
- Prüfen, ob Aktivierungswort korrekt eingestellt ist

### Sprachantwort zu langsam?

- Netzwerklatenz prüfen
- Lokale TTS in Betracht ziehen
- Sprachmodell-Einstellungen anpassen

## Leistung

### OpenClaw verbraucht zu viel Speicher?

- Sitzungsverlauf regelmäßig bereinigen
- Modelleinstellungen anpassen
- Sitzungskompression verwenden

### Nachrichten antworten langsam?

- Netzwerkverbindung prüfen
- Versuchen, zu einem schnelleren Modell zu wechseln
- Gateway-Last prüfen

## Daten

### Wo werden Daten gespeichert?

- Konfiguration: `~/.openclaw/`
- Arbeitsbereich: `~/.openclaw/workspace`
- Anmeldedaten: `~/.openclaw/credentials`

### Wie sichere ich Daten?

```bash
# Gesamtes Konfigurationsverzeichnis sichern
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### Wie setze ich alle Einstellungen zurück?

1. Gateway stoppen
2. Verzeichnis `~/.openclaw` löschen
3. `openclaw onboard` erneut ausführen

## Sonstiges

### Wie sehe ich Logs?

```bash
# Gateway-Logs anzeigen
openclaw gateway --verbose

# Oder bestimmte Logdateien prüfen
openclaw logs
```

### Wie erhalte ich Hilfe?

- Dokumentation: https://docs.openclaw.ai
- Discord-Community beitreten
- Issue auf GitHub erstellen

### Was ist der Unterschied zwischen OpenClaw und anderen KI-Assistenten?

- **Völlig lokal**: Ihre Daten bleiben lokal, nicht in die Cloud hochgeladen
- **Open Source**: Code ist vollständig offen, Sie können ihn selbst prüfen
- **Hochgradig anpassbar**: Sie können jede Funktion ändern und erweitern
- **Multi-Kanal vereinigt**: Ein Assistent verwaltet alle Kommunikationskanäle

### Kann ich mehrere OpenClaw-Instanzen gleichzeitig ausführen?

Ja, jede Instanz braucht:
- Unterschiedlichen Port
- Unabhängigen Arbeitsbereich
- Unterschiedliche Konfiguration
