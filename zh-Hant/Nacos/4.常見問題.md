# Nacos 常見問題

## 快速入門問題

### Nacos 支援哪些作業系統？

Nacos 支援 Linux、Unix、Mac 和 Windows 作業系統，推薦在 Linux/Unix/Mac 環境下運行生產環境。

### Nacos 需要什麼 Java 版本？

Nacos 需要 JDK 17 或更高版本。請確保正確設定 JAVA_HOME 環境變數。

### Nacos 啟動失敗怎麼辦？

1. 檢查 Java 環境：運行 java -version 確認 JDK 版本正確
2. 檢查埠占用：確認 8848、9848 埠未被占用
3. 查看日誌：檢查 logs/start.out 或 logs/nacos.log 中的錯誤資訊
4. 檢查鑒權配置：Nacos 3.0+ 需要配置鑒權相關參數

### 如何修改 Nacos 預設埠？

修改 conf/application.properties 檔案中的以下配置：

```properties
server.port=8848
```

### Nacos 控制台預設使用者名稱密碼是什麼？

- 預設使用者名稱：nacos
- 預設密碼：nacos@demo

生產環境請及時修改預設密碼。

## 服務發現相關問題

### 服務註冊成功後，為什麼用戶端無法發現服務？

1. 檢查服務健康狀態：確保實例狀態為健康
2. 檢查網路連通性：確保用戶端能存取 Nacos 伺服器
3. 檢查服務名稱：確保查詢的服務名與註冊時一致（包括命名空間和分組）
4. 檢查用戶端配置：確認 nacos.server-addr 配置正確

### 如何實現負載均衡？

Nacos 提供了多種負載均衡策略：

1. 用戶端負載均衡：透過 Nacos SDK 的 selectOneHealthyInstance 方法
2. 權重負載均衡：透過設定實例權重實現
3. 叢集負載均衡：在叢集環境中自動實現

### 服務實例下線後，為什麼用戶端仍然調用到已下線的實例？

1. 檢查健康檢查配置：確保健康檢查機制正常運作
2. 檢查用戶端快取：用戶端會快取服務實例列表，有一定延遲
3. 調整同步間隔：修改用戶端的刷新間隔

## 配置管理相關問題

### 配置發布後，用戶端沒有收到更新通知？

1. 檢查監聽器：確保用戶端正確添加了配置監聽器
2. 檢查網路：確保網路連通性正常
3. 查看日誌：檢查用戶端日誌中的錯誤資訊
4. 確認 dataId 和 group：確保完全匹配

### 如何實現配置的多環境管理？

1. 使用命名空間（Namespace）：不同環境使用不同命名空間實現完全隔離
2. 使用分組（Group）：同一命名空間下使用不同分組管理配置
3. 使用 DataId：透過不同的 dataId 區分不同環境配置

### 配置歷史版本如何查看？

Nacos 控制台提供配置變更歷史功能，可以查看：

- 配置的修改記錄
- 每次修改的內容對比
- 一鍵回滾到歷史版本

### 如何實現配置的灰度發布？

Nacos 支援配置灰度發布功能：

1. 在控制台建立灰度配置
2. 指定灰度用戶端範圍
3. 驗證灰度配置效果
4. 全量發布或回滾

## 叢集部署問題

### Nacos 叢集部署需要多少節點？

生產環境推薦部署至少 3 個節點，以保證高可用性。

### 叢集模式下，如何選擇資料存儲？

- 測試/開發環境：可以使用內嵌 Derby 資料庫
- 生產環境：必須使用外置資料庫（MySQL），保證資料可靠性

### 叢集節點之間如何保證資料一致性？

Nacos 使用自研的 Distro 協定和 Raft 協定保證叢集資料一致性：

- Distro 協定：負責資料同步
- Raft 協定：負責選主和日誌複製

### 叢集節點故障怎麼辦？

Nacos 叢集具有自動故障轉移能力：

1. 健康檢查會自動檢測故障節點
2. 流量會自動切換到健康節點
3. 故障節點恢復後自動同步資料

## 鑒權與安全

### Nacos 3.0 鑒權如何配置？

Nacos 3.0+ 預設啟用鑒權，啟動時需要配置以下參數：

```bash
# Token 密鑰
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# 服務身份密鑰
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### 如何關閉 Nacos 鑒權？

修改 conf/application.properties：

```properties
nacos.core.auth.enabled=false
```

注意：生產環境不建議關閉鑒權。

### 如何為不同使用者分配權限？

Nacos 支援基於角色的存取控制：

1. 在控制台建立使用者
2. 為使用者分配角色
3. 配置角色權限

## MCP 相關問題

### 啟動 MCP Server 時出現相容性問題怎麼辦？

確保自動註冊的 MCP Server 資料和 Nacos 上對應版本的資料相容性：

- MCP Server 協定（stdio、sse、streamable）是否一致
- 服務引用是否一致
- Tools 資料是否一致（Tools 名稱、Tools 數量、參數類型、參數是否可選）

注意：Tools 描述和參數描述不要求一致。

### MCP Server 註冊後無法被發現？

1. 檢查 MCP Server 狀態：確保服務已正確註冊
2. 檢查命名空間：確保用戶端和服務在同一命名空間
3. 檢查認證資訊：確保使用者名稱密碼正確

### 如何動態修改 MCP Server 工具配置？

MCP Server 自動註冊到 Nacos 之後，可以在控制台上動態變更 MCP Server Tools 的描述和開關，相關修改將會在運行時中動態應用到 MCP Server 中，無需重啟服務。

## 效能與優化

### Nacos 如何處理大規模服務註冊？

Nacos 誕生自阿里的百萬實例規模，具備以下能力：

- 高效能服務註冊和發現
- 低延遲配置更新回應
- 海量服務和配置管理

### 如何優化 Nacos 效能？

1. 合理設定健康檢查間隔
2. 使用合適的資料庫連接池配置
3. 啟用用戶端快取減少請求次數
4. 合理規劃命名空間和分組

### Nacos 記憶體配置建議？

根據服務規模建議：

- 小規模（服務數 < 1000）：2C4G
- 中規模（服務數 1000-5000）：4C8G
- 大規模（服務數 > 5000）：8C16G 以上

## 生態整合問題

### Nacos 與 Spring Cloud 如何整合？

在 pom.xml 中添加以下依賴：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### Nacos 與 Kubernetes 如何整合？

Nacos 支援在 Kubernetes 環境中部署，可以使用：

- Helm Chart 部署
- Operator 部署
- 自訂 YAML 部署

### Nacos 支援哪些語言的 SDK？

Nacos 提供以下語言的 SDK：

- Java（官方）
- Python
- Go
- Node.js
- C/C++
- Ruby
- PHP

## 故障排查

### 如何排查服務註冊失敗？

1. 檢查 Nacos Server 是否正常運行
2. 檢查網路連通性
3. 查看用戶端日誌
4. 確認註冊參數是否正確

### 如何排查配置無法獲取？

1. 確認配置已發布成功
2. 檢查 dataId 和 group 是否匹配
3. 檢查監聽器是否正確添加
4. 查看用戶端日誌排查具體錯誤

### Nacos 日誌在哪裡？

Nacos 日誌檔案位於：

- 主日誌：logs/nacos.log
- 啟動日誌：logs/start.out
- 存取日誌：logs/access.log

## 升級與遷移

### 如何從 Nacos 1.x 升級到 2.x 或 3.x？

1. 備份資料：備份資料庫或快照檔案
2. 閱讀升級指南：查看對應版本的升級文檔
3. 逐步升級：建議先在測試環境驗證
4. 遷移配置：注意鑒權等配置的變化

### 如何遷移 Nacos 資料？

1. 匯出原環境配置資料
2. 匯入到新環境資料庫
3. 驗證資料完整性
4. 切換流量到新環境
