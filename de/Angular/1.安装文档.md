# Angular Installationsanleitung

Angular ist ein modernes Web-Anwendungsframework, das von Google gepflegt wird. Diese Anleitung deckt die Installationsmethoden in allen Umgebungen ab.

## Versionsinformationen

| Version | Status | Beschreibung |
|---------|--------|--------------|
| Angular 17+ | ✅ Aktuell | Neueste Version mit allen neuen Funktionen |
| Angular 16 | ✅ Unterstützt | Stabile Version |
| Angular 14-15 | ⚠️ Wartung | Sicherheitsupdates |
| Angular 2-13 | ❌ Veraltet | Nicht für neue Projekte empfohlen |

> **Wichtig**: Diese Anleitung richtet sich hauptsächlich an **Angular 17+**. Für neue Projekte wird empfohlen, die neueste Version zu verwenden, um die beste Leistung und die neuesten Funktionen zu erhalten.

## Installationsmethoden

### Methode 1: Angular CLI verwenden (Empfohlen)

Angular CLI ist der offiziell empfohlene Weg, um Angular-Projekte zu erstellen und zu verwalten.

```bash
# Angular CLI installieren
npm install -g @angular/cli

# Neues Projekt erstellen
ng new my-app

# In das Projektverzeichnis wechseln
cd my-app

# Entwicklungsserver starten
ng serve
```

#### Projekt-Erstellungsoptionen

```bash
# Projekt mit Routing erstellen
ng new my-app --routing

# Projekt mit SCSS erstellen
ng new my-app --style=scss

# Projekt mit SSR erstellen
ng new my-app --ssr

# Abhängigkeitsinstallation überspringen
ng new my-app --skip-install
```

### Methode 2: Projekt manuell erstellen

```bash
# npm-Projekt initialisieren
npm init -y

# Angular-Kernabhängigkeiten installieren
npm install @angular/core@17 @angular/common@17 @angular/compiler@17

# Angular-Plattform installieren
npm install @angular/platform-browser@17 @angular/platform-browser-dynamic@17

# Angular-Compiler installieren
npm install @angular/compiler-cli@17 typescript@~5.2.0 zone.js@~0.14.0
```

### Methode 3: Vite verwenden

```bash
# Angular-Projekt mit Vite erstellen
npm create vite@latest my-angular-app -- --template angular

# In das Projektverzeichnis wechseln
cd my-angular-app

# Abhängigkeiten installieren
npm install
```

### Methode 4: Nx Workspace verwenden

Nx ist ein leistungsstarkes Monorepo-Tool, das sich für große Unternehmensprojekte eignet.

```bash
# Nx installieren
npm install -g nx

# Angular-Anwendung erstellen
npx create-nx-workspace my-org --preset=angular

# Angular-Anwendung hinzufügen
nx g @nx/angular:app my-app
```

## Entwicklungsbefehle

### Allgemeine Befehle

```bash
# Entwicklungsserver starten
ng serve

# Produktions-Build erstellen
ng build

# Tests ausführen
ng test

# End-to-End-Tests ausführen
ng e2e

# Komponente generieren
ng g c my-component

# Service generieren
ng g s my-service

# Modul generieren
ng g m my-module

# Routing generieren
ng g m my-module --routing
```

### Entwicklungsserver-Optionen

```bash
# Port angeben
ng serve --port 4200

# Browser öffnen
ng serve --open

# Hot Module Replacement
ng serve --hmr

# Produktions-Build
ng serve --configuration production
```

## Abhängigkeitsverwaltung

### Abhängigkeiten installieren

```bash
# Laufzeitabhängigkeiten installieren
npm install @angular/animations @angular/common @angular/compiler @angular/core @angular/forms @angular/platform-browser @angular/platform-browser-dynamic @angular/router

# Entwicklungsabhängigkeiten installieren
npm install --save-dev @angular-devkit/build-angular @angular/cli @angular/compiler-cli @types/jasmine @types/node typescript jasmine-core jasmine-spec-reporter karma karma-chrome-launcher karma-coverage karma-jasmine karma-jasmine-html-reporter zone.js
```

### yarn verwenden

```bash
# Mit yarn installieren
yarn install

# Abhängigkeit hinzufügen
yarn add @angular/core@17

# Entwicklungsabhängigkeit hinzufügen
yarn add --dev @angular/cli@17
```

## Häufig gestellte Fragen

### Angular CLI-Befehle funktionieren nicht?

Stellen Sie sicher, dass Angular CLI global installiert ist:

```bash
npm install -g @angular/cli
```

### Node.js-Version stimmt nicht?

Verwenden Sie nvm zur Verwaltung der Node.js-Versionen:

```bash
# nvm installieren (Windows)
winget install CoreyButler.NVMforWindows

# Bestimmte Version verwenden
nvm install 18
nvm use 18
```

### Build fehlgeschlagen?

Cache leeren und neu installieren:

```bash
rm -rf node_modules dist
npm install
```
