# FantasyTalking 설치 가이드

## 요구사항

FantasyTalking을 설치하기 전에 개발 환경이 다음 요구사항을 충족하는지 확인하세요. 운영 체제의 경우 이 프로젝트는 Windows, macOS, Linux를 지원합니다. 최적의 성능을 위해 64비트 시스템 사용을 권장합니다. Node.js 버전 14.0 이상이 필요하며, Python 버전 3.7 이상이 필요합니다. Python 백엔드를 사용하는 경우 FFmpeg 오디오 처리 도제도 설치해야 합니다.

하드웨어의 경우 원활한 음성 합성 경험을 위해 최소 4GB RAM과 듀얼 코어 프로세서가 있는 시스템을 권장합니다. 실시간 음성 합성이 필요한 애플리케이션의 경우 I/O 지연 시간을 줄이기 위해 SSD 저장소 사용을 권장합니다. 모델 훈련이나 사용자 정의 음성 개발을 계획하는 경우 CUDA 지원 그래픽 카드가 필요합니다.

## 설치 방법

FantasyTalking은 다양한 설치 방법을 지원합니다. 개발자는 필요와 기술 스택에 따라 가장 적합한 방법을 선택할 수 있습니다.

### npm 사용 (Node.js)

```bash
# 핵심 라이브러리 설치
npm install fantasy-talking

# 선택적 구성요소 설치
npm install fantasy-talking-ui  # 웹 UI 컴포넌트
npm install fantasy-talking-node  # Node.js 백엔드 지원
```

### pip 사용 (Python)

```bash
# 핵심 라이브러리 설치
pip install fantasy-talking

# 선택적 구성요소 설치
pip install fantasy-talking[ui]  # 웹 UI 컴포넌트
pip install fantasy-talking[all]  # 모든 선택적 구성요소
```

### 소스에서 빌드

```bash
# 저장소 복제
git clone https://github.com/fantasytalking/fantasy-talking.git
cd fantasy-talking

# Node.js 종속성 설치
npm install

# Python 종속성 설치 (선택사항)
pip install -r requirements.txt

# 프로젝트 빌드
npm run build
```

## 빠른 시작

### JavaScript/TypeScript 예시

```javascript
const FantasyTalking = require('fantasy-talking');

// 클라이언트 초기화
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  // 선택사항: 음성 스타일 지정
  voice: {
    style: 'neutral',
    language: 'en-US',
    speed: 1.0,
    pitch: 1.0
  }
});

// 음성 합성
async function synthesize() {
  const audio = await client.speak('Hello, this is a test voice from FantasyTalking');
  console.log('음성 합성 성공');
  
  // 파일로 저장
  const fs = require('fs');
  fs.writeFileSync('output.wav', audio);
  
  // 또는 직접 재생
  client.play(audio);
}

synthesize();
```

### Python 예시

```python
from fantasy_talking import FantasyTalking

# 클라이언트 초기화
client = FantasyTalking(
    api_key='your-api-key',
    voice={
        'style': 'neutral',
        'language': 'en-US',
        'speed': 1.0,
        'pitch': 1.0
    }
)

# 음성 합성
audio = client.speak('Hello, this is a test voice from FantasyTalking')

# 파일로 저장
with open('output.wav', 'wb') as f:
    f.write(audio)

print('음성 합성 성공')
```

## 구성 옵션

FantasyTalking은 특정 요구에 따라 조정할 수 있는구성 옵션을 제공합니다.

### 기본 구성

| 매개변수 | 유형 | 기본값 | 설명 |
|---------|------|--------|------|
| apiKey | 문자열 | - | API 키, 필수 |
| apiUrl | 문자열 | 기본값 | API 서버 주소 |
| timeout | 숫자 | 30000 | 요청 시간 초과 (ms) |
| retries | 숫자 | 3 | 요청 재시도 횟수 |

### 음성 구성

| 매개변수 | 유형 | 기본값 | 설명 |
|---------|------|--------|------|
| voice.style | 문자열 | neutral | 음성 스타일: neutral, happy, sad, angry, excited |
| voice.language | 문자열 | en-US | 언어 코드 |
| voice.speed | 숫자 | 1.0 | 음성 속도, 범위 0.5-2.0 |
| voice.pitch | 숫자 | 1.0 | 음높이, 범위 0.5-2.0 |
| voice.volume | 숫자 | 1.0 | 볼륨, 범위 0-1.0 |

### 고급 구성

고급 사용자의 경우 더 세밀한 제어를 위해 더 많은 매개변수를 구성할 수 있습니다:

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  
  // 오디오 출력 형식
  audio: {
    format: 'wav',  // wav, mp3, ogg
    sampleRate: 16000,  // 8000, 16000, 24000, 48000
    bitDepth: 16  // 8, 16, 24
  },
  
  // 대화 구성
  dialogue: {
    enable: true,
    contextWindow: 5,  // 유지할 대화 턴 수
    systemPrompt: '당신은 친절한 음성 어시스턴트입니다'
  },
  
  // 스트리밍 출력
  stream: {
    enable: true,
    chunkSize: 1024  // 스트리밍 출력용 청크 크기
  }
});
```

## 서비스 배포

### Docker 배포

프로젝트는 손쉬운 배포를 위한 Docker 이미지를 제공합니다:

```bash
# 이미지 풀
docker pull fantasytalking/server:latest

# 컨테이너 실행
docker run -d \
  --name fantasy-talking \
  -p 8080:8080 \
  -e API_KEY=your-api-key \
  -e PORT=8080 \
  fantasytalking/server:latest
```

### 자체 호스팅 서비스

완전한 제어를 위해 자체 백엔드 서비스를 배포할 수 있습니다:

```bash
# 백엔드 코드 복제
git clone https://github.com/fantasytalking/server.git
cd server

# 환경 변수 구성
cp .env.example .env
# .env 파일 편집, API 키 및 기타 구성 입력

# 서비스 시작
npm run start
```

서비스가 시작된 후 RESTful API를 통해 음성 합성 요청을 보낼 수 있습니다. 자세한 API 문서는 공식 문서를 참조하세요.

## 설치 확인

설치 후 FantasyTalking이 올바르게 작동하는지 확인할 수 있습니다:

```bash
# 명령줄 도구로 테스트
fantasy-talking --text "Test voice" --output test.wav

# 또는 예시 실행
npm run example
```

모든 것이 작동하면 음성 파일이 성공적으로 생성된 것을 볼 수 있습니다. 일반적인 설치 문제와 해결책은 FAQ 페이지를 참조하세요.
