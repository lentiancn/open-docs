# Angular 常见问题

常见问题解答。

## 安装问题

### Q: Angular CLI 安装失败？

**解决方案**：
1. 检查 Node.js 版本
```bash
node --version  # 需要 18+ 或 20+
```
2. 清理 npm 缓存
```bash
npm cache clean --force
```
3. 使用管理员权限（Windows）
4. 检查网络连接

### Q: ng 命令找不到？

**解决方案**：
1. 确认 Angular CLI 已安装
```bash
npm list -g @angular/cli
```
2. 检查环境变量
3. 重新安装
```bash
npm uninstall -g @angular/cli
npm install -g @angular/cli@latest
```

### Q: 项目创建失败？

**解决方案**：
1. 检查磁盘空间
2. 检查 npm 权限
3. 使用 --skip-install 跳过安装
```bash
ng new my-app --skip-install
cd my-app
npm install
```

## 开发问题

### Q: 组件不显示？

**解决方案**：
1. 确认组件已正确导入
2. 确认选择器正确使用
3. 检查模板语法
4. 检查控制台错误

### Q: 数据不更新？

**解决方案**：
1. 对于 Signal，更新使用 set/update 方法
```typescript
count.set(1);           // 设置新值
count.update(c => c + 1); // 基于旧值更新
```
2. 对于 RxJS，确保正确订阅
3. 使用 ChangeDetectorRef 手动检测

### Q: 路由不生效？

**解决方案**：
1. 确认路由配置正确
2. 确认 RouterOutlet 已放置
3. 检查路由顺序（特定路由在前，通配符在后）

### Q: 表单验证不生效？

**解决方案**：
1. 导入 ReactiveFormsModule 或 FormsModule
2. 确认 FormControl 名称正确
3. 正确使用验证器

## 性能问题

### Q: 首屏加载慢？

**解决方案**：
1. 使用懒加载路由
2. 启用生产模式
```bash
ng build --configuration=production
```
3. 使用 AOT 编译（默认开启）

### Q: 内存泄漏？

**解决方案**：
1. 及时取消订阅
2. 使用 async 管道自动订阅/取消
3. 在 ngOnDestroy 中清理

### Q: 打包体积太大？

**解决方案**：
1. 检查依赖大小
2. 使用懒加载
3. 移除未使用的导入

## 安全问题

### Q: XSS 攻击如何防范？

**解决方案**：
1. Angular 默认会自动转义
2. 谨慎使用 innerHTML

### Q: CSRF 防护？

**解决方案**：
使用 HttpClientXsrfModule 配置 CSRF 防护。

## 升级问题

### Q: 如何升级 Angular 版本？

**解决方案**：
1. 使用 ng update
```bash
ng update @angular/core@19 @angular/cli@19
```
2. 检查迁移指南
3. 更新依赖版本
4. 运行测试确保兼容性
