# Руководство пользователя Node.js

## Первые шаги

### Основные концепции

Node.js построен на движке JavaScript V8 от Chrome и предоставляет модель ввода-вывода, ориентированную на события и неблокирующую, что делает его легким и эффективным для приложений реального времени, интенсивно использующих данные.

#### Цикл событий
Цикл событий Node.js является ядром его модели конкурентности. В отличие от традиционных моделей, основанных на потоках, Node.js использует однопоточный цикл событий, который обрабатывает асинхронные операции через обратные вызовы, промисы или async/await.

#### Неблокирующий ввод-вывод
Большинство операций Node.js являются неблокирующими, что означает, что они не останавливают выполнение программы во время ожидания завершения операций ввода-вывода. Вместо этого они используют обратные вызовы для обработки результатов, когда они готовы.

### Приложение Hello World

Создайте свое первое приложение Node.js с этим простым HTTP-сервером:

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Запустите его с помощью: `node server.js`

## Основные модули

Node.js поставляется с богатым набором встроенных модулей, которые предоставляют основные функции без необходимости внешних зависимостей.

### Файловая система (fs)
Обрабатывайте операции с файлами синхронно или асинхронно:

```javascript
const fs = require('fs');

// Асинхронное чтение
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Синхронное чтение (блокирует выполнение)
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Запись файла
fs.writeFile('/path/to/output.txt', 'Привет, Node.js!', (err) => {
  if (err) throw err;
  console.log('Файл успешно записан');
});
```

### HTTP-сервер
Создавайте веб-серверы и выполняйте HTTP-запросы:

```javascript
const http = require('http');
const https = require('https');

// Создать HTTP-сервер
const server = http.createServer((req, res) => {
  // Обработать запрос
});

// Выполнить HTTP-запрос
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });

```

## Обработка ошибок

Правильная обработка ошибок имеет решающее значение в приложениях Node.js из-за их асинхронной природы.

### Обработка ошибок на основе обратных вызовов
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Ошибка при чтении файла:', err);
    return;
  }
  console.log('Содержимое файла:', data);
});
```

### Обработка ошибок на основе промисов
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('Содержимое файла:', data);
  })
  .catch(err => {
    console.error('Ошибка при чтении файла:', err);
  });
```

### Обработка ошибок с Async/Await
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('Содержимое файла:', data);
  } catch (err) {
    console.error('Ошибка при чтении файла:', err);
  }
}
```

## Оптимизация производительности

### Кластеризация
Используйте несколько ядер CPU с помощью модуля cluster:

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Мастер ${process.pid} запущен`);
  
  // Создать рабочие процессы
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Рабочий процесс ${worker.process.pid} завершился`);
  });
} else {
  // Рабочие процессы делят TCP-соединение
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`Рабочий процесс ${process.pid} запущен`);
}
```

- Используйте потоки для обработки больших файлов вместо загрузки всего в память
- Отслеживайте использование памяти с помощью `process.memoryUsage()`

### Кэширование
Реализуйте стратегии кэширования для повышения производительности:

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```
