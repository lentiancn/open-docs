# Guide d'installation

## Configuration requise

Avant d'installer Jenkins, assurez-vous que votre système répond aux exigences minimales suivantes :

### Configuration minimale recommandée

- 256 Mo de mémoire disponible
- 1 Go d'espace disque disponible (10 Go recommandés en ejecución en tant que conteneur Docker)

### Configuration recommandée pour les petites équipes

- 1 Go+ de mémoire disponible
- 50 Go+ d'espace disque disponible

### Logiciels requis

- Java 8 ou supérieur (soit JRE soit JDK)
- Remarque : Si Jenkins fonctionne en tant que conteneur Docker, il n'est pas nécessaire d'installer Java sur l'hôte

## Plateformes d'installation

Jenkins supporte plusieurs méthodes d'installation. Ce guide couvre les approches les plus courantes.

## Installation Docker

Docker est la méthode recommandée pour installer Jenkins car il fournit un environnement isolé et simplifie le processus de configuration.

### Installation de Docker

Tout d'abord, vous devez installer Docker sur votre système. Visitez le site web officiel de Docker et choisissez la version Docker Community Edition appropriée pour votre système d'exploitation.

### Télécharger et exécuter Jenkins

Sur macOS et Linux, ouvrez un terminal et exécutez :

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

Sur Windows, ouvrez une invite de commande et exécutez une commande similaire :

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

Explication des paramètres :

- `-u root` : Exécuter le conteneur en tant qu'utilisateur root
- `--rm` : Supprimer automatiquement le conteneur à l'arrêt
- `-d` : Exécuter en arrière-plan (mode détaché)
- `-p 8080:8080` : Mapper le port 8080 du conteneur au port 8080 de l'hôte
- `-p 50000:50000` : Mapper le port 50000 du conteneur au port 50000 de l'hôte (pour la communication des agents JNLP)
- `-v jenkins-data:/var/jenkins_home` : Persister les données Jenkins
- `-v /var/run/docker.sock:/var/run/docker.sock` : Permettre à Jenkins d'exécuter des commandes Docker dans les pipelines
- `jenkinsci/blueocean` : Image officielle Jenkins avec l'interface Blue Ocean

### Accéder au conteneur Jenkins

Pour accéder à l'intérieur du conteneur Jenkins, utilisez :

```bash
docker exec -it jenkins-blueocean bash
```

### Voir les journaux Jenkins

Affichez les journaux de la console Jenkins avec :

```bash
docker logs jenkins-blueocean
```

## Installation par fichier WAR

La version du fichier WAR de Jenkins peut être installée sur tout système d'exploitation supportant Java.

### Étapes d'installation

1. Téléchargez le dernier package WAR stable Jenkins depuis le site officiel
2. Ouvrez un terminal dans le répertoire de téléchargement
3. Exécutez la commande : `java -jar jenkins.war`
4. Accédez à http://localhost:8080 et attendez que la page Déverrouiller Jenkins apparaisse

### Notes

- L'installation par fichier WAR n'installe pas automatiquement Blue Ocean ; il doit être installé séparément via la gestion des plugins
- Vous pouvez changer le port en utilisant `--httpPort`, par exemple : `java -jar jenkins.war --httpPort=9090`

## Installation sur macOS

### Utiliser le package

1. Téléchargez le package Jenkins pour macOS
2. Double-cliquez pour ouvrir et suivez l'assistant d'installation

### Utiliser Homebrew

Installer la dernière version :

```bash
brew install jenkins
```

Installer la version LTS :

```bash
brew install jenkins-lts
```

## Installation sur Linux (Debian/Ubuntu)

Sur les distributions basées sur Debian, installez Jenkins via apt :

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

Après l'installation, Jenkins va :

- Être configuré pour démarrer en tant que démon au démarrage
- Créer un utilisateur jenkins pour exécuter le service
- Envoyer les journaux vers /var/log/jenkins/jenkins.log
- Écouter sur le port 8080 par défaut

## Assistant de configuration post-installation

Au premier démarrage de Jenkins, l'assistant de configuration commence.

### Déverrouiller Jenkins

1. Accédez à http://localhost:8080
2. Lorsque la page Déverrouiller Jenkins apparaît, copiez le mot de passe généré automatiquement à partir des journaux de la console
3. Collez le mot de passe dans le champ mot de passe administrateur et cliquez sur Continuer

### Personnaliser les plugins Jenkins

Sur la page d'installation des plugins, vous pouvez choisir :

- Installer les plugins suggérés : Installe un ensemble de plugins recommandés basé sur les cas d'utilisation courants
- Sélectionner les plugins à installer : Choisir manuellement les plugins à installer

### Créer le premier utilisateur administrateur

Enfin, créez un compte administrateur en indiquant :

- Nom d'utilisateur
- Mot de passe
- Confirmer le mot de passe
- Nom complet
- Adresse e-mail

Cliquez sur Sauvegarder et Terminer, puis sur Commencer à utiliser Jenkins pour accéder à l'interface principale.

## Vérifier l'installation

Vérifiez l'installation en :

1. Accédant à http://localhost:8080
2. Connectez-vous avec le compte administrateur créé
3. Vérifiant que le statut du système est normal

## Problèmes courants

### Port déjà utilisé

Si le port 8080 est utilisé par un autre programme :

- Méthode Docker : Modifier le paramètre -p
- Méthode WAR : Utiliser le paramètre --httpPort

### Mémoire insuffisante

Si la mémoire système est limitée :

- Utiliser Docker avec les limites de ressources du conteneur
- Ajuster la taille du heap JVM dans les paramètres de démarrage de Jenkins

### Premier démarrage lent

Le premier démarrage télécharge et installe les plugins, ce qui peut prendre du temps. Veuillez patienter.

## Prochaines étapes

Après l'installation, il est recommandé d'apprendre à :

- Créer votre première tâche pipeline
- Configurer le code source (comme Git)
- Configurer les déclencheurs de build
- Intégrer les outils de test et déploiement
