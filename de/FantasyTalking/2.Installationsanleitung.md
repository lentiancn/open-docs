# FantasyTalking Installationsanleitung

## Anforderungen

Bevor Sie FantasyTalking installieren, stellen Sie sicher, dass Ihre Entwicklungsumgebung die folgenden Anforderungen erfüllt. Bei Betriebssystemen unterstützt das Projekt Windows, macOS und Linux. Die Verwendung eines 64-Bit-Systems wird für optimale Leistung empfohlen. Node.js Version 14.0 oder höher ist erforderlich, Python Version 3.7 oder höher. Wenn Sie das Python-Backend verwenden, muss auch das FFmpeg-Audioverarbeitungstool installiert werden.

Für Hardware wird empfohlen, dass das System mindestens 4 GB RAM und einen Dual-Core-Prozessor für ein reibungsloses Sprachsynthese-Erlebnis hat. Für Anwendungen, die Echtzeit-Sprachsynthese erfordern, wird die Verwendung von SSD-Speicher zur Reduzierung der E/A-Latenz empfohlen. Wenn Sie Modell-Training oder die Entwicklung benutzerdefinierter Stimmen planen, wird eine CUDA-fähige Grafikkarte benötigt.

## Installationsmethoden

FantasyTalking unterstützt mehrere Installationsmethoden. Entwickler können die für ihre Bedürfnisse und ihren Technologie-Stack am besten geeignete Methode wählen.

### Installation mit npm (Node.js)

```bash
# Kernbibliothek installieren
npm install fantasy-talking

# Optionale Komponenten installieren
npm install fantasy-talking-ui  # Web-UI-Komponenten
npm install fantasy-talking-node  # Node.js Backend-Unterstützung
```

### Installation mit pip (Python)

```bash
# Kernbibliothek installieren
pip install fantasy-talking

# Optionale Komponenten installieren
pip install fantasy-talking[ui]  # Web-UI-Komponenten
pip install fantasy-talking[all]  # Alle optionalen Komponenten
```

### Kompilierung aus dem Quellcode

```bash
# Repository klonen
git clone https://github.com/fantasytalking/fantasy-talking.git
cd fantasy-talking

# Node.js-Abhängigkeiten installieren
npm install

# Python-Abhängigkeiten installieren (optional)
pip install -r requirements.txt

# Projekt kompilieren
npm run build
```

## Schnellstart

### JavaScript/TypeScript Beispiel

```javascript
const FantasyTalking = require('fantasy-talking');

// Client initialisieren
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  // Optional: Sprachstil angeben
  voice: {
    style: 'neutral',
    language: 'de-DE',
    speed: 1.0,
    pitch: 1.0
  }
});

// Sprache synthetisieren
async function synthesize() {
  const audio = await client.speak('Hallo, das ist ein Test von FantasyTalking');
  console.log('Sprachsynthese erfolgreich');
  
  // In Datei speichern
  const fs = require('fs');
  fs.writeFileSync('output.wav', audio);
  
  // Oder direkt abspielen
  client.play(audio);
}

synthesize();
```

### Python Beispiel

```python
from fantasy_talking import FantasyTalking

# Client initialisieren
client = FantasyTalking(
    api_key='your-api-key',
    voice={
        'style': 'neutral',
        'language': 'de-DE',
        'speed': 1.0,
        'pitch': 1.0
    }
)

# Sprache synthetisieren
audio = client.speak('Hallo, das ist ein Test von FantasyTalking')

# In Datei speichern
with open('output.wav', 'wb') as f:
    f.write(audio)

print('Sprachsynthese erfolgreich')
```

## Konfigurationsoptionen

FantasyTalking bietet reichhaltige Konfigurationsoptionen, die an spezifische Bedürfnisse angepasst werden können.

### Basiskonfiguration

| Parameter | Typ | Standard | Beschreibung |
|-----------|------|---------|--------------|
| apiKey | string | - | API-Schlüssel, erforderlich |
| apiUrl | string | Standard | API-Serveradresse |
| timeout | number | 30000 | Anfrage-Timeout (ms) |
| retries | number | 3 | Anzahl der Wiederholungen |

### Sprachkonfiguration

| Parameter | Typ | Standard | Beschreibung |
|-----------|------|---------|--------------|
| voice.style | string | neutral | Sprachstil: neutral, happy, sad, angry, excited |
| voice.language | string | de-DE | Sprachcode |
| voice.speed | number | 1.0 | Sprechgeschwindigkeit, Bereich 0.5-2.0 |
| voice.pitch | number | 1.0 | Tonhöhe, Bereich 0.5-2.0 |
| voice.volume | number | 1.0 | Lautstärke, Bereich 0-1.0 |

## Service-Bereitstellung

### Docker-Bereitstellung

Das Projekt bietet Docker-Images für einfache Bereitstellung:

```bash
# Image pullen
docker pull fantasytalking/server:latest

# Container ausführen
docker run -d \
  --name fantasy-talking \
  -p 8080:8080 \
  -e API_KEY=your-api-key \
  -e PORT=8080 \
  fantasytalking/server:latest
```

### Selbst gehosteter Service

Für vollständige Kontrolle können Sie Ihren eigenen Backend-Service bereitstellen:

```bash
# Backend-Code klonen
git clone https://github.com/fantasytalking/server.git
cd server

# Umgebungsvariablen konfigurieren
cp .env.example .env
# .env-Datei bearbeiten, API-Schlüssel und andere Konfigurationen eingeben

# Service starten
npm run start
```

Nach dem Service-Start können Sie Sprachsynthese-Anfragen über die RESTful API stellen.

## Installation überprüfen

Nach der Installation können Sie überprüfen, ob FantasyTalking korrekt funktioniert:

```bash
# Mit Kommandozeilentest testen
fantasy-talking --text "Testtext" --output test.wav

# Oder Beispiel ausführen
npm run example
```

Wenn alles funktioniert, sollten Sie die erfolgreiche Generierung der Audiodatei sehen.
