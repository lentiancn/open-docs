# OpenCode Benutzerhandbuch

Grundlegende Nutzung und Techniken für OpenCode.

## Schnellstart

### Fragen stellen

Du kannst OpenCode bitten, den Codebase zu erklären:

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### Funktionen hinzufügen

Bevor du Funktionen hinzufügst, wird empfohlen, dass OpenCode zuerst einen Plan erstellt:

1. Drücke **Tab** um in den Plan-Modus zu wechseln
2. Beschreibe die Funktion, die du möchtest
3. Nach dem Plan, gib Feedback oder füge Details hinzu
4. Wenn du zufrieden bist, drücke **Tab** um zurück in den Build-Modus zu wechseln

### Direkte Änderungen

Für einfachere Änderungen kannst du OpenCode direkt fragen:

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### Änderungen rückgängig

Wenn du mit den Änderungen nicht zufrieden bist, verwende den Rückgängig-Befehl:

```bash
/undo
```

OpenCode wird die Änderungen rückgängig machen und deine ursprüngliche Nachricht erneut anzeigen. Du kannst deine Eingabe anpassen und erneut fragen.

### Wiederholen

Du kannst `/redo` verwenden um Änderungen zu wiederholen.

## Sitzungen teilen

```bash
/share
```

Dies erstellt einen Link zur aktuellen Sitzung und kopiert ihn in die Zwischenablage.

## Häufige Befehle

| Befehl | Funktion |
|--------|----------|
| /init | Projekt initialisieren |
| /connect | LLM-Anbieter verbinden |
| /undo | Änderungen rückgängig |
| /redo | Änderungen wiederholen |
| /share | Sitzung teilen |

## Best Practices

1. Sprich mit OpenCode wie mit einem Junior-Entwickler
2. Gib ausreichend Kontext und Details
3. Lass OpenCode zuerst einen Plan erstellen, dann ausführen
4. Verwende /undo wenn du nicht zufrieden bist
5. Überprüfe und aktualisiere AGENTS.md regelmäßig
