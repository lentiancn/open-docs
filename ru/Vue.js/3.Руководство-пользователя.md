# Vue.js Руководство пользователя

## Экземпляр приложения

Каждое приложение Vue начинается с создания нового экземпляра приложения с помощью функции `createApp`:

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* опции корневого компонента */
})
```

### Корневой компонент
Объект, который мы передаем в `createApp`, на самом деле является компонентом. Каждое приложение требует "корневого компонента", который может содержать другие компоненты как своих детей.

### Монтирование приложения
Экземпляр приложения ничего не отображает, пока не будет вызван его метод `.mount()`:

```javascript
app.mount('#app')
```

### Конфигурации приложения
Экземпляр приложения предоставляет объект `.config`, который позволяет настраивать параметры на уровне приложения:

```javascript
// Обработчик ошибок на уровне приложения
app.config.errorHandler = (err) => {
  /* обработка ошибки */
}

// Регистрация глобального компонента
app.component('TodoDeleteButton', TodoDeleteButton)
```

### Несколько экземпляров приложения
Вы можете создавать несколько приложений Vue на одной странице:

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## Синтаксис шаблонов

Vue использует синтаксис шаблонов на основе HTML, который позволяет вам декларативно привязывать отображаемый DOM к данным базового экземпляра компонента.

### Интерполяция текста
Самая основная форма привязки данных — это интерполяция текста с использованием синтаксиса "Mustache" (двойные фигурные скобки):

```html
<span>Message: {{ msg }}</span>
```

### Необработанный HTML
Используйте директиву `v-html` для вывода реального HTML:

```html
<p v-html="rawHtml"></p>
```

**Предупреждение о безопасности**: Динамический рендеринг произвольного HTML на вашем сайте может быть очень опасным, так как это легко может привести к уязвимостям XSS. Используйте `v-html` только для доверенного контента и никогда не используйте его для контента, предоставленного пользователем.

### Привязка атрибутов
Используйте директиву `v-bind` для привязки атрибутов HTML:

```html
<!-- Полный синтаксис -->
<div v-bind:id="dynamicId"></div>

<!-- Сокращенный синтаксис -->
<div :id="dynamicId"></div>

<!-- Сокращение с одинаковым именем (Vue 3.4+) -->
<div :id></div>
```

### Булевы атрибуты
Булевы атрибуты включаются, когда привязанное значение истинно:

```html
<button :disabled="isButtonDisabled">Button</button>
```

### Динамическая привязка нескольких атрибутов
Привяжите несколько атрибутов из объекта, используя `v-bind` без аргумента:

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## Выражения JavaScript

Vue поддерживает полную мощь выражений JavaScript во всех привязках данных:

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### Вызов функций
Можно вызывать метод, предоставленный компонентом, внутри выражения привязки:

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### Ограниченный доступ к глобальным переменным
Выражения шаблонов изолированы и имеют доступ только к [ограниченному списку глобальных переменных](https://github.com/vuejs/core/blob/main/packages/shared/src/globalsAllowList.ts#L3). Список предоставляет часто используемые встроенные глобальные объекты, такие как Math и Date.

Глобальные переменные, явно не включенные в список, например, свойства, прикрепленные пользователем к window, не будут доступны в выражениях шаблонов. Однако вы можете явно определить дополнительные глобальные переменные для всех выражений Vue, добавив их в [app.config.globalProperties](/api/application#app-config-globalproperties).

## Директивы

Директивы — это специальные атрибуты с префиксом `v-`.

### Аргументы
Некоторые директивы могут принимать "аргумент", обозначаемый двоеточием после имени директивы. Например, директива `v-bind` используется для реактивного обновления атрибута HTML:

```html
<!-- Аргумент v-bind -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- Аргумент v-on -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### Динамические аргументы
Также можно использовать выражение JavaScript в аргументе директивы, заключив его в квадратные скобки:

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**Ограничения**:
- Динамические аргументы должны вычисляться в строку или null
- Избегайте сложных выражений; вместо этого используйте вычисляемые свойства
- В шаблонах DOM избегайте заглавных букв в динамических ключах

### Модификаторы
Модификаторы — это специальные суффиксы, обозначаемые точкой, которые указывают, что директива должна быть привязана особым образом. Например, модификатор `.prevent` указывает директиве `v-on` вызывать `event.preventDefault()` для срабатываемого события:

```html
<form @submit.prevent="onSubmit">...</form>
```

## Основы реактивности

### Объявление реактивного состояния
Создайте реактивные ссылки с помощью `ref()`:

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### Реактивные объекты
Создайте реактивные объекты с помощью `reactive()`:

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### Вычисляемые свойства
Создайте вычисляемые свойства с помощью `computed()`:

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### Наблюдатели
Наблюдайте за изменениями реактивных данных с помощью `watch()`:

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// Наблюдение за одной ссылкой
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## Основы компонентов

### Определение компонентов
```javascript
// Определение компонента
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Передача пропсов
```javascript
// Дочерний компонент
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// Родительский компонент
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### Генерация событий
```javascript
// Дочерний компонент
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### Слоты
```html
<!-- Дочерний компонент -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- Родительский компонент -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
