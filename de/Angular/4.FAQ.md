# Angular FAQ

Häufig gestellte Fragen.

## Installation

### Q: Angular CLI Installation fehlgeschlagen?

**Lösungen**:
1. npm-Cache leeren: `npm cache clean --force`
2. Admin-/Root-Rechte verwenden
3. Node.js Version prüfen

### Q: ng Befehl nicht gefunden?

**Lösungen**:
1. Angular CLI ist global installiert?
2. PATH-Umgebungsvariable prüfen
3. Neuinstallation versuchen

### Q: Node.js Version inkompatibel?

**Lösungen**:
1. nvm zur Verwaltung von Node.js-Versionen verwenden
2. Node.js auf empfohlene Version aktualisieren

## Nutzung

### Q: Komponenten-Stile funktionieren nicht?

**Lösungen**:
1. styleUrls-Pfad ist korrekt?
2. CSS-Selektoren sind korrekt?
3. ViewEncapsulation-Einstellungen prüfen

### Q: Routing funktioniert nicht?

**Lösungen**:
1. Routenkonfiguration ist korrekt?
2. RouterModule importiert?
3. routerLink-Syntax prüfen

### Q: HTTP-Anfragen fehlgeschlagen?

**Lösungen**:
1. HttpClientModule importiert?
2. CORS-Konfiguration prüfen
3. API-URL ist korrekt?

### Q: Formularvalidierung funktioniert nicht?

**Lösungen**:
1. ReactiveFormsModule importiert?
2. Validators-Konfiguration prüfen
3. formControlName korrekt verwenden

## Leistung

### Q: Anwendung lädt langsam?

**Lösungen**:
1. Lazy Loading aktivieren
2. OnPush-Änderungserkennung verwenden
3. Bilder und Ressourcen komprimieren
4. Im Produktionsmodus bauen

### Q: Speicherlecks?

**Lösungen**:
1. Subscriptions kündigen
2. Intervalle löschen
3. Event-Listener zerstören

## Version

### Q: Wie Angular aktualisieren?

```bash
ng update @angular/core @angular/cli
```

### Q: Von AngularJS zu Angular migrieren?

angular-upgrade Tool für schrittweise Migration verwenden.
