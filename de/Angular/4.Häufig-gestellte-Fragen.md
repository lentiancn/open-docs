# Häufig gestellte Fragen

Dieses Dokument fasst häufige technische Fragen und Antworten während der Angular-Nutzung zusammen.

## Grundlagen

### Was ist Angular?

Angular ist ein Open-Source-Webframework, das von Google entwickelt und gepflegt wird und zur Erstellung moderner Webanwendungen dient. Es verwendet TypeScript als hauptsächliche Programmiersprache und bietet Funktionen wie komponentenbasierte Architektur, Two-Way-Databinding und Dependency Injection.

### Was ist der Unterschied zwischen Angular und AngularJS?

AngularJS ist der Name für Angular 1.x und der Vorgänger von Angular. Angular ist eine komplette Neuentwicklung, die TypeScript verwendet und eine völlig andere Architektur hat – es ist nicht mit AngularJS kompatibel.

### Ist Angular schwer zu lernen?

Angular hat eine gewisse Lernkurve, insbesondere Konzepte wie Dependency Injection und RxJS brauchen Zeit zum Verstehen. Die Dokumentation ist jedoch umfassend, die Community ist aktiv und nach der Einarbeitung ist die Entwicklung sehr effizient.

## Installationsprobleme

### npm install schlägt fehl – was tun?

1. npm-Cache leeren: `npm cache clean --force`
2. node_modules und package-lock.json löschen
3. Chinesischen Spiegelserver verwenden
4. Prüfen, ob die Node.js-Version kompatibel ist

### Angular-CLI-Befehl nicht gefunden?

Stelle sicher, dass die globale Installation erfolgreich war:

```bash
npm install -g @angular/cli
```

Falls es immer noch nicht funktioniert, versuche npx:

```bash
npx @angular/cli new my-app
```

### Node.js-Version nicht kompatibel?

Verwende nvm zur Verwaltung der Node.js-Version:

```bash
nvm install 18
nvm use 18
```

## Entwicklungsprobleme

### Komponenten-Styles werden nicht angewendet?

1. Prüfe, ob der Stylesheet-Pfad korrekt ist
2. Stelle sicher, dass in styles.scss keine Syntaxfehler vorliegen
3. Überprüfe die ViewEncapsulation-Einstellung

### Routing funktioniert nicht?

1. Prüfe die Routing-Konfiguration
2. Stelle sicher, dass RouterModule korrekt importiert wurde
3. Prüfe, ob Route-Guards die Navigation blockieren

### HTTP-Anfragen schlagen fehl?

1. Prüfe, ob der Backend-Dienst läuft
2. CORS-Konfiguration überprüfen
3. HttpInterceptor für Fehlerbehandlung verwenden

### Daten werden nicht aktualisiert?

1. Change-Detection-Strategie prüfen
2. Bei Verwendung von OnPush muss die Erkennung manuell ausgelöst werden
3. Sicherstellen, dass sich die Datenreferenz geändert hat

## Performance-Probleme

### Erster Bildschirm lädt langsam?

1. Routing Lazy Loading aktivieren
2. Preloading-Strategie verwenden
3. Bilderressourcen optimieren
4. Produktionsmodus-Build aktivieren

### Memory Leaks?

1. Observables in ngOnDestroy unsubscribeen
2. takeUntil oder async-Pipe verwenden
3. Timer und Event-Listener aufräumen

### Häufige Change Detection?

1. OnPush-Strategie verwenden
2. Veränderliche und unveränderliche Eigenschaften trennen
3. trackBy für *ngFor optimieren

## Testprobleme

### Unit-Tests schlagen fehl?

1. TestBed-Konfiguration prüfen
2. Asynchrone Operationen korrekt behandeln
3. fakeAsync und tick zum Testen von asynchronem Code verwenden

### Modul nicht gefunden?

Stelle sicher, dass paths in tsconfig.json korrekt konfiguriert sind.

## Build-Probleme

### Build schlägt fehl?

1. Konkrete Fehlermeldung prüfen
2. TypeScript-Compileroptionen überprüfen
3. Cache leeren und neu bauen

### Build-Dateien zu groß?

1. Produktionsmodus aktivieren
2. Lazy Loading verwenden
3. Prüfen, ob ungenutzte Abhängigkeiten vorhanden sind

## Häufige Fehler

### ExpressionChangedAfterItHasBeenCheckedError

Werte wurden während der Überprüfung geändert, die sich nicht ändern sollten. Prüfe, ob in ngOnInit Code ausgeführt wird, der die Ansicht direkt beeinflusst.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

Derselbe Fehler wie oben – prüfe das Timing der Datenbindung.

### Cannot read property of undefined

Auf Daten zugegriffen, bevor sie initialisiert wurden. Verwende Optional Chaining (?.) oder *ngIf als Schutz.

### CORS-Fehler

CORS wurde auf dem Backend nicht konfiguriert oder es gibt Probleme mit den Request-Headern.

### 401/403-Fehler

Authentifizierung fehlgeschlagen – Token und Berechtigungskonfiguration prüfen.

## Versions-Upgrade

### Wie aktualisiere ich Angular?

1. CLI zum Upgrade verwenden: `ng update @angular/core`
2. Offizielles Upgrade-Handbuch konsultieren
3. Abhängigkeiten einzeln aktualisieren
4. Tests ausführen, um sicherzustellen, dass alles funktioniert

### Upgrade schlägt fehl – was tun?

1. Code sichern
2. package.json-Versionen manuell aktualisieren
3. node_modules löschen und neu installieren
