# Guide d'installation

Ce guide explique comment installer Angular et configurer votre environnement de développement pour démarrer rapidement.

## Prérequis

Avant d'installer Angular, assurez-vous de respecter les exigences suivantes :

- **Node.js** : version 18.10.0 ou supérieure recommandée
- **npm** : inclus avec Node.js, version 6.x ou supérieure
- **Système d'exploitation** : Windows, macOS ou Linux

## Installation d'Angular CLI

Angular CLI est l'outil en ligne de commande officiel d'Angular, utilisé pour créer et gérer les projets Angular.

### Installation globale

```bash
npm install -g @angular/cli
```

### Vérifier l'installation

```bash
ng version
```

Si l'installation a réussi, la version d'Angular CLI s'affichera.

### Installer une version spécifique

Si vous avez besoin d'une version particulière :

```bash
npm install -g @angular/cli@17
```

## Créer un nouveau projet

### Utiliser CLI pour créer

```bash
ng new my-app
```

Cette commande crée un nouveau projet nommé my-app et installe automatiquement les dépendances.

### Options de création

```bash
# Créer sans installer
ng new my-app --skip-install

# Utiliser SCSS
ng new my-app --style=scss

# Activer le routage
ng new my-app --routing

# Créer des composants autonomes (Angular 14+)
ng new my-app --standalone
```

## Configuration de l'environnement de développement

### Utiliser un miroir chinois

En raison des restrictions réseau, il est recommandé de configurer un miroir chinois :

```bash
# Configurer le miroir npm
npm config set registry https://registry.npmmirror.com

# Ou utiliser l'outil nrm pour gérer les miroirs
npm install -g nrm
nrm use taobao
```

### Installer Node.js

**Installation avec nvm (recommandé) :**

```bash
# Installer nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Installer la version LTS de Node.js
nvm install --lts
nvm use --lts
```

**Windows : utiliser nvm-windows :**

Téléchargez le package d'installation depuis https://github.com/coreybutler/nvm-windows.

## Lancer le serveur de développement

### Entrer dans le répertoire du projet

```bash
cd my-app
```

### Lancer le serveur

```bash
ng serve
```

Adresse par défaut : http://localhost:4200

### Rechargement automatique

Angular CLI supporte le rechargement automatique. Les modifications de code rafraichissent automatiquement le navigateur.

## Construire pour la production

### Commande de build

```bash
ng build
```

Les fichiers construits sont générés dans le répertoire dist.

### Optimisation pour la production

```bash
ng build --configuration=production
```

Cette commande active la minification du code, le tree-shaking et d'autres optimisations.

## Commandes CLI courantes

| Commande | Description |
|----------|-------------|
| ng generate component xxx | Générer un composant |
| ng generate service xxx | Générer un service |
| ng generate module xxx | Générer un module |
| ng generate directive xxx | Générer une directive |
| ng generate pipe xxx | Générer un pipe |
| ng generate guard xxx | Générer un gardien de route |

## Outils de développement

### Extensions VS Code

- **Angular Language Service** : vérification de types dans les modèles et autocomplétion
- **Angular Snippets** : extraits de code couramment utilisés
- **Prettier** : formatage du code

### Débogage

Angular propose l'extension de navigateur Angular DevTools pour le débogage et l'analyse des performances.

## Désinstaller Angular CLI

Si vous avez besoin de désinstaller :

```bash
npm uninstall -g @angular/cli
```
