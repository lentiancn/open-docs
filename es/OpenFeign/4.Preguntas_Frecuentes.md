# OpenFeign Preguntas Frecuentes

## Preguntas Relacionadas con la Instalación

### P: ¿Cómo verificar si OpenFeign está instalado correctamente?
**R:** Después de agregar las dependencias, verifique creando una aplicación básica:

```java
@SpringBootApplication
@EnableFeignClients
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

Asegúrese de que su aplicación Spring Boot se inicie correctamente y que los clientes Feign puedan inyectarse adecuadamente.

### P: ¿Cómo manejar conflictos de versión de dependencias?
**R:** Utilice Spring Cloud BOM para la gestión de versiones:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>2022.0.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Esto previene eficazmente los conflictos de versión.

### P: ¿Por qué necesito agregar explícitamente spring-cloud-starter-loadbalancer?
**R:** A partir de Spring Cloud 2020.0.0, Ribbon fue eliminado y Spring Cloud LoadBalancer se convirtió en el balanceador de carga predeterminado. `spring-cloud-starter-openfeign` lo incluye como una dependencia opcional, por lo que necesita agregarlo explícitamente:

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

## Preguntas Relacionadas con la Configuración

### P: ¿Cómo configurar tiempos de espera?
**R:** Puede configurar tiempos de espera mediante archivo de configuración o configuración Java:

**Enfoque de archivo de configuración:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 5000
            read-timeout: 5000
          user-service:
            connect-timeout: 3000
            read-timeout: 10000
```

**Enfoque de configuración Java:**
```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Request.Options requestOptions() {
        return new Request.Options(3000, 10000);
    }
}
```

### P: ¿Cómo habilitar la compresión GZIP?
**R:** Habilite la compresión en el archivo de configuración:

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

Nota: Cuando se utiliza OkHttpClient, la compresión se deshabilita si los encabezados `content-encoding` o `accept-encoding` están presentes.

### P: ¿Cómo configurar niveles de registro?
**R:** Dos enfoques de configuración:

**Archivo de configuración:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
            
logging:
  level:
    com.example.clients.UserService: DEBUG
```

**Configuración Java:**
```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Niveles de registro soportados:
- **NONE**: Sin registro (predeterminado)
- **BASIC**: Registra solo método de solicitud, URL, código de estado de respuesta y tiempo de ejecución
- **HEADERS**: Registra información básica junto con encabezados de solicitud y respuesta
- **FULL**: Registra encabezados, cuerpo y metadatos para solicitudes y respuestas

## Preguntas Relacionadas con el Uso

### P: La interfaz FeignClient no se puede inyectar, ¿qué debo hacer?
**R:** Causas comunes y soluciones:

**1. Problema de ruta de paquete de escaneo:**
```java
// Asegúrese de que FeignClient esté en la ruta de paquete escaneada
@SpringBootApplication
@EnableFeignClients(basePackages = "com.example.clients")
public class Application {
    // ...
}
```

**2. Problema de dependencia circular:**
Si FeignClient es referenciado por múltiples Servicios, puede ocurrir una dependencia circular. Soluciones:
- Utilice la anotación `@Lazy` para inyección diferida
- Refactorice la estructura del código para evitar dependencias circulares

**3. Conflictos de nombre de Bean:**
Si tiene múltiples FeignClients con el mismo nombre, utilice `contextId` para diferenciar:
```java
@FeignClient(contextId = "userClient1", name = "user-service")
public interface UserClient1 { }

@FeignClient(contextId = "userClient2", name = "user-service")  
public interface UserClient2 { }
```

### P: ¿Cómo manejar parámetros de solicitud complejos?
**R:** Varios enfoques para parámetros complejos:

**1. Usando @SpringQueryMap:**
```java
public class SearchCriteria {
    private String name;
    private String status;
    private Integer page;
}

@FeignClient(name = "search-service")
public interface SearchService {
    @GetMapping("/search")
    SearchResult search(@SpringQueryMap SearchCriteria criteria);
}
```

**2. Usando Map:**
```java
@FeignClient(name = "api-service")
public interface ApiService {
    @GetMapping("/data")
    Data getData(@RequestParam Map<String, Object> params);
}
```

**3. QueryMapEncoder personalizado:**
```java
@Bean
public QueryMapEncoder queryMapEncoder() {
    return new CustomQueryMapEncoder();
}
```

### P: ¿Cómo manejar la carga de archivos?
**R:** La carga de archivos requiere configuración especial:

```java
@FeignClient(name = "file-service")
public interface FileService {
    
    @PostMapping(value = "/upload", 
                consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    UploadResponse uploadFile(@RequestPart("file") MultipartFile file,
                             @RequestPart("metadata") FileMetadata metadata);
}
```

También configure el codificador:
```java
@Bean
public Encoder feignFormEncoder() {
    return new FeignFormEncoder(new SpringEncoder(messageConverters()));
}
```

## Preguntas sobre Manejo de Errores

### P: ¿Cómo personalizar el manejo de errores?
**R:** Implemente la interfaz ErrorDecoder:

```java
@Component
public class CustomErrorDecoder implements ErrorDecoder {
    
    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() == 404) {
            return new ResourceNotFoundException("Resource not found");
        } else if (response.status() >= 500) {
            return new ServerException("Server error");
        }
        return new Default().decode(methodKey, response);
    }
}
```

### P: El cortacircuitos no funciona, ¿qué está mal?
**R:** Verifique las siguientes configuraciones:

**1. Habilitar cortacircuitos:**
```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

**2. Agregar dependencia requerida:**
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

**3. Configurar reglas de cortacircuitos:**
```yaml
resilience4j:
  circuitbreaker:
    instances:
      UserServicegetUser:
        failure-rate-threshold: 50
        minimum-number-of-calls: 5
        wait-duration-in-open-state: 5000
```

### P: El fallback no se activa, ¿qué debo verificar?
**R:** Problemas comunes:

**1. Asegúrese de que la clase de fallback sea un Bean de Spring:**
```java
@Component  // Esta anotación es requerida
public class UserServiceFallback implements UserService {
    // ...
}
```

**2. Verificar tipos de excepción:**
El cortacircuitos solo activa el fallback para excepciones específicas, asegúrese de que la excepción lanzada sea del tipo esperado.

**3. Configurar fábrica de fallback correcta:**
```java
@FeignClient(name = "service", fallbackFactory = MyFallbackFactory.class)
public interface MyService { }
```

## Preguntas Relacionadas con el Rendimiento

### P: El rendimiento del cliente Feign es lento, ¿cómo optimizarlo?
**R:** Recomendaciones de optimización de rendimiento:

**1. Configurar tiempos de espera apropiados:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
```

**2. Habilitar agrupación de conexiones:**
```yaml
spring:
  cloud:
    openfeign:
      httpclient:
        hc5:
          enabled: true
          max-connections: 200
          max-connections-per-route: 20
```

**3. Habilitar compresión:**
```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
        response:
          enabled: true
```

**4. Usar caché:**
```java
@FeignClient(name = "config-service")
public interface ConfigService {
    @GetMapping("/config/{key}")
    @Cacheable(cacheNames = "config-cache", key = "#key")
    ConfigValue getConfig(@PathVariable String key);
}
```

### P: ¿Cómo resolver problemas de fuga de memoria?
**R:** Las fugas de memoria son típicamente causadas por:

**1. Conexiones no cerradas:**
Asegúrese de que los clientes HTTP gestionen correctamente los grupos de conexiones.

**2. Caché no limpiado:**
Limpie regularmente el caché o establezca tiempos de expiración apropiados.

**3. Monitorear uso de memoria:**
```java
@RestController
public class MetricsController {
    
    @GetMapping("/memory")
    public MemoryInfo getMemoryInfo() {
        Runtime runtime = Runtime.getRuntime();
        return new MemoryInfo(
            runtime.totalMemory(),
            runtime.freeMemory(),
            runtime.maxMemory()
        );
    }
}
```

## Preguntas Relacionadas con la Seguridad

### P: ¿Cómo agregar tokens de autenticación a las solicitudes?
**R:** Utilice RequestInterceptor:

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor authRequestInterceptor() {
        return requestTemplate -> {
            String token = getToken(); // Lógica para obtener token
            requestTemplate.header("Authorization", "Bearer " + token);
        };
    }
    
    private String getToken() {
        // Implementar lógica de recuperación de token
        return "your-token";
    }
}
```

### P: ¿Cómo manejar la autenticación OAuth2?
**R:** Habilitar soporte OAuth2:

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
        
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: client_credentials
            scope: read,write
```

### P: ¿Cómo filtrar información sensible?
**R:** Filtrado de registro personalizado:

```java
public class SecureLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        Map<String, Collection<String>> safeHeaders = new HashMap<>(request.headers());
        safeHeaders.remove("Authorization");
        safeHeaders.remove("X-API-Key");
        
        super.logRequest(configKey, logLevel, 
            Request.create(request.httpMethod(), request.url(), safeHeaders,
                request.body(), request.charset(), request.requestTemplate()));
    }
}
```

## Preguntas Relacionadas con las Pruebas

### P: ¿Cómo probar clientes Feign?
**R:** Se recomienda usar WireMock:

```java
@SpringBootTest
@AutoConfigureWireMock(port = 0)
class FeignClientTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    void shouldGetUserSuccessfully() {
        // Given
        stubFor(get(urlEqualTo("/users/1"))
            .willReturn(aResponse()
                .withStatus(200)
                .withHeader("Content-Type", "application/json")
                .withBody("{\"id\":1,\"name\":\"John\"}")));
        
        // When
        User user = userService.getUser(1L);
        
        // Then
        assertThat(user.getName()).isEqualTo("John");
    }
}
```

### P: ¿Cómo simular escenarios de error?
**R:** Use WireMock para simular varios errores:

```java
@Test
void shouldHandleServerError() {
    // Given
    stubFor(get(urlEqualTo("/users/999"))
        .willReturn(aResponse().withStatus(500)));
    
    // When & Then
    assertThatThrownBy(() -> userService.getUser(999L))
        .isInstanceOf(ServerException.class);
}
```

## Preguntas Relacionadas con la Implementación

### P: ¿Cómo configurar para entorno de producción?
**R:** Configuración recomendada para entorno de producción:

```yaml
# application-prod.yml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
            logger-level: basic
      compression:
        request:
          enabled: true
        response:
          enabled: true
      circuitbreaker:
        enabled: true
      cache:
        enabled: true
        
logging:
  level:
    com.example.clients: WARN
```

### P: ¿Cómo monitorear clientes Feign?
**R:** Integrar con Micrometer:

```yaml
spring:
  cloud:
    openfeign:
      micrometer:
        enabled: true
```

Luego ver métricas a través de endpoints de Actuator:
```bash
GET /actuator/metrics
GET /actuator/metrics/feign.client.requests
```

Al entender y aplicar estas soluciones, podrá desarrollar, depurar y mantener aplicaciones basadas en OpenFeign de manera más efectiva.
