# Benutzerhandbuch

## Pipeline-Grundlagen

Pipeline ist eine Kernfunktion von Jenkins, die den gesamten Softwareentwicklungsprozess vom Erstellen über Testen bis zum Bereitstellen automatisiert.

## Was ist Pipeline?

Pipeline ist eine Suite von Jenkins-Plugins, die die Implementierung und Integration von Continuous Delivery-Pipelines in Jenkins ermöglicht. Einfach ausgedrückt automatisiert Pipeline den gesamten Prozess, Software von der Versionskontrolle zum Benutzer zu bringen.

Pipeline-Definitionen werden in einer Textdatei (Jenkinsfile) geschrieben, die in das Quellcode-Repository des Projekts eingecheckt werden kann. Dies implementiert die Philosophie von "Pipeline as Code".

### Vorteile von Pipeline

- **Code-basiert**: Pipelines werden wie Code versioniert, ermöglicht Überprüfung und Iteration
- **Langlebig**: Pipelines überstehen sowohl geplante als auch ungeplante Neustarts von Jenkins
- **Pausierbar**: Pipelines können anhalten und auf menschliche Eingabe oder Genehmigung warten
- **Vielseitig**: Unterstützt komplexe reale CD-Anforderungen einschließlich paralleler Ausführung und Schleifen
- **Erweiterbar**: Plugins können das Pipeline-DSL erweitern und andere Tools integrieren

### Pipeline-Konzepte

#### Knoten

Ein Knoten ist eine Maschine, die Teil der Jenkins-Umgebung ist und eine Pipeline ausführen kann. Knoten können Master- oder Agent-Maschinen sein.

#### Stufe

Eine Stufe definiert einen konzeptuell unterschiedlichen Aufgabensatz der gesamten Pipeline, wie "Build", "Test" und "Deploy"-Stufen. Stufen werden zur Visualisierung des Pipeline-Ausführungsstatus verwendet.

#### Schritt

Ein Schritt ist eine grundlegende Aktion in der Jenkins-Pipeline, die Jenkins mitteilt, was zu einem bestimmten Zeitpunkt zu tun ist. Um beispielsweise einen Shell-Befehl auszuführen, verwenden Sie den `sh`-Schritt.

### Pipeline-Syntax

#### Deklarative Pipeline

Deklarative Pipeline ist neuere Syntax mit reicheren Funktionen, einfacher zu schreiben und zu lesen:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                echo 'Anwendung wird erstellt...'
            }
        }
        stage('Test') {
            steps {
                echo 'Tests werden ausgeführt...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Anwendung wird bereitgestellt...'
            }
        }
    }
}
```

#### Scripted Pipeline

Scripted Pipeline verwendet Groovy-Syntax, flexibler erfordert aber mehr Programmiererfahrung:

```groovy
node {
    stage('Build') {
        echo 'Anwendung wird erstellt...'
    }
    stage('Test') {
        echo 'Tests werden ausgeführt...'
    }
    stage('Deploy') {
        echo 'Anwendung wird bereitgestellt...'
    }
}
```

## Erste Pipeline erstellen

### Blue Ocean verwenden

Blue Ocean ist die moderne Jenkins-Oberfläche, die das Erstellen von Pipelines intuitiver macht:

1. Blue Ocean-Plugin installieren (falls nicht vorinstalliert)
2. Klicken Sie auf "Blue Ocean öffnen"
3. Klicken Sie auf "Pipeline erstellen"
4. Wählen Sie den Speicherort des Code-Repositories (GitHub, GitLab usw.)
5. Wählen Sie Repository und Branch
6. Blue Ocean erkennt automatisch das Jenkinsfile oder hilft beim Erstellen

### Jenkinsfile verwenden

Erstellen Sie ein Jenkinsfile in Ihrem Projekt, und Jenkins erkennt es automatisch:

1. Erstellen Sie Jenkinsfile im Projektstammverzeichnis
2. Definieren Sie Pipeline-Stufen
3. Committen Sie in die Versionskontrolle
4. Erstellen Sie eine Pipeline-Aufgabe in Jenkins und verknüpfen Sie sie mit dem Repository

## Anmeldedatenverwaltung

Jenkins muss auf verschiedene externe Ressourcen wie Code-Repositories und Bereitstellungsziele zugreifen, was eine Anmeldedatenverwaltung erfordert.

### Anmeldedaten hinzufügen

1. Gehen Sie zu "Manage Jenkins" > "Manage Credentials"
2. Klicken Sie auf "Add Credentials"
3. Wählen Sie den Anmeldedatentyp (Benutzername mit Passwort, SSH mit privatem Schlüssel, Secret-Datei usw.)
4. Füllen Sie die Anmeldedateninformationen aus
5. Legen Sie ID und Beschreibung für die Anmeldedaten fest

### Anmeldedaten in Pipeline verwenden

Verwenden Sie den withCredentials-Schritt:

```groovy
pipeline {
    agent any
    stages {
        stage('Anmeldedaten verwenden') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "Benutzername: $USER"
                }
            }
        }
    }
}
```

## Build-Trigger

Automatisierte Builds erfordern Trigger. Jenkins unterstützt mehrere Trigger-Methoden.

### SCM-Abfrage

Regelmäßig prüfen, ob sich der Quellcode geändert hat:

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // Alle 5 Minuten prüfen
    }
    stages {
        stage('Build') {
            steps {
                echo 'Code-Änderung erkannt, Build wird gestartet'
            }
        }
    }
}
```

### Webhook-Trigger

Builds über Repository-Webhooks auslösen für bessere Effizienz:

1. Entsprechendes Plugin installieren (z.B. GitHub Hook Trigger)
2. Repository-Webhook so konfigurieren, dass es auf Jenkins zeigt
3. Build wird automatisch bei Code-Commit ausgelöst

### Geplante Builds

Automatisch nach Zeitplan ausführen:

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // Täglich um 2 Uhr ausführen
    }
    stages {
        stage('Geplanter Build') {
            steps {
                echo 'Geplanter Build läuft'
            }
        }
    }
}
```

## Agenten und Container verwenden

### Agenten konfigurieren

Jenkins unterstützt verteilte Builds und verteilt Build-Aufgaben auf mehrere Maschinen:

1. Gehen Sie zu "Manage Jenkins" > "Manage Nodes"
2. Fügen Sie einen Agent-Knoten hinzu und konfigurieren Sie die Verbindungsinformationen
3. Geben Sie den Agent in der Pipeline an

### Docker in Pipeline verwenden

Deklarative Pipeline kann Builds in Docker-Containern ausführen:

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## Testberichte

Jenkins kann sich in verschiedene Test-Frameworks integrieren und visuelle Testberichte erstellen.

### JUnit-Testberichte

```groovy
pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### Andere Testberichte

Jenkins unterstützt verschiedene Testbericht-Plugins:

- TestNG
- Cobertura (Code-Abdeckung)
- JaCoCo (Code-Abdeckung)
- ESLint (Frontend-Code-Analyse)

## Bereitstellungspraktiken

### In verschiedene Umgebungen bereitstellen

Typische Multi-Stage-Bereitstellungs-Pipeline:

```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('In Testumgebung bereitstellen') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('In Produktion bereitstellen') {
            when {
                branch 'main'
            }
            steps {
                input message: 'Bereitstellung in Produktion bestätigen?', ok: 'Bereitstellen'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### Blue-Green-Deployment

Implementieren Sie Blue-Green-Deployment mit Jenkins für Updates ohne Ausfallzeiten:

1. Zwei identische Umgebungen pflegen (blau und grün)
2. Verkehr auf neue Version umschalten
3. Bei Problemen schnell zurücksetzen

## Best Practices

### Best Practices für Jenkinsfile

- Jenkinsfile in Versionskontrolle halten
- Deklarative Pipeline-Syntax verwenden
- Pipelines einfach und klar halten
- Stufen vernünftig aufteilen
- Notwendige Timeout- und Wiederholungsmechanismen hinzufügen

### Sicherheitsempfehlungen

- Jenkins und Plugins regelmäßig aktualisieren
- Berechtigungen für anonyme Benutzer einschränken
- Anmeldedaten für sensible Informationen verwenden
- Angemessene Zugriffskontrolle konfigurieren
- Sicherheits-Audit-Protokollierung aktivieren

### Leistungsoptimierung

- Build-Knoten richtig konfigurieren
- Inkrementelle Builds verwenden, um Zeit zu sparen
- Build-Cache konfigurieren
- Alte Build-Aufzeichnungen regelmäßig bereinigen
- Parallele Stufen verwenden, um Builds zu beschleunigen

## Fehlerbehebung

### Build-Protokolle anzeigen

Wenn ein Build fehlschlägt, überprüfen Sie zuerst die Build-Protokolle:

1. Gehen Sie zur Aufgabenseite
2. Klicken Sie auf die fehlgeschlagene Build-Nummer
3. Überprüfen Sie die Konsolenausgabe

### Häufige Probleme lösen

- Build hängt: Auf ausstehende Eingabe oder Ressourcensperren prüfen
- Anmeldedaten ungültig: Prüfen, ob Anmeldedaten abgelaufen oder widerrufen wurden
- Plugin-Konflikte: Versuchen Sie, kürzlich installierte Plugins zu deaktivieren
- Nicht genügend Speicher: JVM-Heap-Parameter anpassen

## Zusammenfassung

Jenkins Pipeline ist das Kernwerkzeug für die Implementierung von CI/CD. Durch dieses Handbuch sollten Sie in der Lage sein:

- Kern-Pipeline-Konzepte zu verstehen
- Pipeline-Aufgaben zu erstellen und zu konfigurieren
- Anmeldedaten und Trigger zu verwalten
- Test- und Bereitstellungstools zu integrieren
- Best Practices anzuwenden, um Workflows zu optimieren

Kontinuierliches Lernen und Üben ist der Schlüssel zum Beherrschen von Jenkins. Entdecken und optimieren Sie weiterhin in echten Projekten.
