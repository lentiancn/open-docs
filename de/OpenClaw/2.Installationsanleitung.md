# OpenClaw Installationsanleitung

> Vollständige Anleitung zur Einrichtung und Installation von OpenClaw

---

## Systemanforderungen

| Anforderung | Spezifikation |
|-------------|---------------|
| Betriebssystem | Windows 10+, macOS, Linux |
| Node.js | Version 18+ |
| Arbeitsspeicher | Mindestens 4 GB RAM |
| Speicherplatz | ~500 MB |

---

## Installation

### Voraussetzungen

Stellen Sie sicher, dass Node.js installiert ist:

```bash
node --version
npm --version
```

### Schritt-für-Schritt-Installation

#### 1. Installation über npm

```bash
npm install -g openclaw
```

#### 2. Gateway-Dienst starten

Der Gateway-Dienst ist das Herzstück von OpenClaw:

```bash
# pr Statusüfen
openclaw gateway status

# Gateway starten
openclaw gateway start

# Gateway neu starten
openclaw gateway restart

# Gateway stoppen
openclaw gateway stop
```

#### 3. Erstkonfiguration

Nach der Installation müssen Sie die Konfigurationsdateien einrichten:

```bash
# Konfigurationsverzeichnis erstellen
mkdir -p ~/.openclaw

# Standardkonfiguration kopieren
cp /pfad/zu/openclaw/config/* ~/.openclaw/
```

## Konfiguration

### Hauptkonfigurationsdateien

| Datei | Beschreibung |
|-------|--------------|
| AGENTS.md | Arbeitsbereich-Konfiguration |
| SOUL.md | Persönlichkeits-Einstellungen |
| TOOLS.md | Werkzeugkonfiguration |
| USER.md | Benutzereinstellungen |

### Umgebungsvariablen

```bash
# Optional: Benutzerdefiniertes Arbeitsverzeichnis
OPENCLAW_WORKSPACE=/pfad/zum/workspace

# Optional: Gateway-Port
OPENCLAW_PORT=3000
```

## Verifizierung

### Installation testen

```bash
# Versionsinformationen anzeigen
openclaw --version

# Gateway-Status prüfen
openclaw gateway status

# Verfügbare Befehle anzeigen
openclaw --help
```

### Schnellstart

```bash
# Interaktive Sitzung starten
openclaw chat

# Einzelne Aufgabe ausführen
openclaw exec "sage eine Nachricht"
```

---

## Fehlerbehebung

### Gateway startet nicht?

**Lösung**:
1. Prüfen Sie, ob Node.js korrekt installiert ist
2. Überprüfen Sie die Berechtigungen
3. Logs prüfen: `openclaw gateway logs`

### Port bereits belegt?

**Lösung**:
```bash
# Port ändern
openclaw gateway start --port 3001
```

### Modul nicht gefunden?

**Lösung**:
```bash
# Neu installieren
npm uninstall -g openclaw
npm install -g openclaw
```

---

## Deinstallation

```bash
npm uninstall -g openclaw

# Gateway-Dienst entfernen
openclaw gateway stop
rm -rf ~/.openclaw
```
