# Spring Cloud FAQ

## Questions Générales

### Qu'est-ce que Spring Cloud?
Spring Cloud est un framework pour construire des systèmes distribués cloud-native. Il fournit des outils pour service discovery, gestion de configuration, circuit breakers et API gateways.

### Quelle est la différence entre Spring Boot et Spring Cloud?
Spring Boot sert à construire des applications individuelles. Spring Cloud coordonne plusieurs services pour créer un système distribué.

## Questions d'Installation

### Comment ajouter Spring Cloud à mon projet?
Ajoutez le BOM et les dépendances:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>${spring-cloud.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

### Quelle version de Spring Cloud utiliser?
- Spring Boot 2.7.x → Spring Cloud 2022.0.x

## Questions Service Discovery

### Comment fonctionne Eureka?
1. Les services s'enregistrent avec Eureka au démarrage
2. Les clients interrogent Eureka pour trouver les services

### Comment enregistrer un service?
```java
@SpringBootApplication
@EnableEurekaClient
public class MonServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(MonServiceApplication.class, args);
    }
}
```

## Questions Circuit Breaker

### Comment utiliser Resilience4j?
```java
@Service
public class MonService {
    
    @CircuitBreaker(name = "monService", fallbackMethod = "fallback")
    public String appelerService() {
        // appel service externe
    }
    
    public String fallback(Exception e) {
        return "Réponse fallback";
    }
}
```
