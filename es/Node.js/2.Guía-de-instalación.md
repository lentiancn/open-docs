# Guía de Instalación de Node.js

## Métodos de Instalación Oficiales

Node.js proporciona varios métodos oficiales de instalación para adaptarse a diferentes sistemas operativos y preferencias de usuario. Todos los métodos oficiales garantizan que obtengas los binarios auténticos y sin modificar directamente del proyecto Node.js.

### Descargar desde el Sitio Web Oficial

La forma más sencilla de instalar Node.js es descargando el instalador apropiado desde el [sitio web oficial de Node.js](https://nodejs.org/es/download).

**Versiones disponibles:**
- **LTS (Soporte a Largo Plazo)**: Recomendado para la mayoría de los usuarios. Proporciona estabilidad y soporte a largo plazo.
- **Current**: Últimas características con ciclo de soporte más corto. Adecuado para desarrollo y pruebas.

### Gestores de Paquetes

#### macOS (Homebrew)
```bash
# Instalar Homebrew si aún no está instalado
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Instalar versión LTS
brew install node@lts

# O instalar la última versión actual
brew install node
```

#### Linux (Varias Distribuciones)

**Ubuntu/Debian:**
```bash
# Usando repositorio de NodeSource (recomendado)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora:**
```bash
# Usando repositorio de NodeSource
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# o para versiones más nuevas: sudo dnf install nodejs
```

#### Windows
- Descargar el instalador de Windows (.msi) desde el sitio web oficial
- Ejecutar el instalador y seguir el asistente de configuración
- Node.js y npm se agregarán automáticamente a tu PATH

### Verificación

Después de la instalación, verifica que Node.js y npm estén correctamente instalados:

```bash
# Verificar versión de Node.js
node --version

# Verificar versión de npm  
npm --version

# Probar funcionalidad básica
node -e "console.log('¡Hola Node.js!')"
```

### Gestión de Múltiples Versiones de Node.js

En desarrollo, es posible que necesites usar diferentes versiones de Node.js en diferentes proyectos. Se recomienda utilizar las siguientes herramientas:

#### nvm (Node Version Manager)
- **macOS/Linux**: 
  ```bash
  curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
  ```
- **Windows**: Usar [nvm-windows](https://github.com/coreybutler/nvm-windows)

Comandos comunes:
```bash
# Listar versiones disponibles
nvm list available

# Instalar versión específica
nvm install 18.17.0

# Cambiar versión
nvm use 18.17.0

# Establecer versión predeterminada
nvm alias default 18.17.0
```

### Configuración de Variables de Entorno

Asegúrate de que las rutas de los archivos ejecutables de Node.js y npm se hayan agregado a la variable de entorno PATH del sistema.

- **Windows**: Normalmente se agrega automáticamente a PATH
- **macOS/Linux**: Si usas un gestor de paquetes, normalmente se configura automáticamente

Si no se encuentra el comando, agrégalo manualmente a PATH:
```bash
# Normalmente ubicado en /usr/local/bin o ~/.nvm/versions/node/vXX.XX.XX/bin
export PATH=$PATH:/usr/local/bin
```

### Solución de Problemas

#### Problemas de Permisos
En Linux/macOS, si encuentras problemas de permisos al instalar paquetes globales de npm, se recomienda:
- Usar nvm para gestionar Node.js
- O configurar el directorio global de npm:
  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  export PATH=~/.npm-global/bin:$PATH
  ```

#### Problemas de Red
Si la velocidad de descarga es lenta o no puedes conectarte, puedes configurar npm para usar un mirror local:
```bash
# Establecer mirror de Taobao (China)
npm config set registry https://registry.npmmirror.com

# O usar temporalmente
npm install --registry https://registry.npmmirror.com
```
