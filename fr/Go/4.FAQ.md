# FAQ Go

> Réponses aux questions fréquentes sur Go

## Installation

### Q: Commande non trouvée après installation?
**A**: Vérifier que Go est ajouté à la variable PATH

### Q: Comment vérifier l'installation?
```bash
go version
go env GOPATH
```

### Q: Comment mettre à jour Go?
- Windows: Réinstaller nouvelle version
- macOS: `brew upgrade go`

---

## Syntaxe

### Q: Go a-t-il des exceptions?
**A**: Non, les erreurs sont gérées via返回值:
```go
result, err := doSomething()
if err != nil {
    // handle error
}
```

### Q: Go supporte-t-il les génériques?
**A**: Go 1.18+ supporte les génériques:
```go
func Print[T any](s []T) {
    for _, v := range s {
        fmt.Println(v)
    }
}
```

## Concurrence

### Q: Différence entre Goroutine et thread?
- Goroutines sont gérées par le runtime Go
- Plus légères que les threads

### Q: Comment attendre la fin des goroutines?
```go
var wg sync.WaitGroup
wg.Add(1)
go func() {
    defer wg.Done()
}()
wg.Wait()
```

## Performance

### Q: Comment profiler?
```bash
go test -cpuprofile=cpu.out
go tool pprof cpu.out
```
