# Manuel d'utilisation de Yarn

## Commandes de base

### Commandes générales

- `yarn add` - Ajouter des dépendances au projet
- `yarn install` - Installer les dépendances du projet
- `yarn remove` - Supprimer des dépendances du projet
- `yarn run` - Exécuter un script défini dans package.json
- `yarn init` - Créer un nouveau package
- `yarn info` - Voir les informations relatives aux packages
- `yarn why` - Afficher la raison pour laquelle un package est nécessaire

### Commandes avancées

- `yarn dlx` - Exécuter un package dans un environnement temporaire
- `yarn dedupe` - Dédupliquer les dépendances avec des plages superposées
- `yarn up` - Mettre à jour les dépendances dans tout le projet
- `yarn pack` - Générer un tarball à partir du workspace actif
- `yarn rebuild` - Reconstruire les packages natifs du projet

### Commandes liées à NPM

- `yarn npm audit` - Effectuer un audit de vulnérabilité sur les packages installés
- `yarn npm publish` - Publier le workspace actif sur le registre npm
- `yarn npm login` - Stocker de nouvelles informations de connexion pour accéder au registre npm
- `yarn npm logout` - Se déconnecter du registre npm
- `yarn npm whoami` - Afficher le nom de l'utilisateur authentifié

### Gestion des plugins

- `yarn plugin list` - Lister les plugins officiels disponibles
- `yarn plugin import` - Télécharger un plugin
- `yarn plugin remove` - Supprimer un plugin
- `yarn plugin runtime` - Lister les plugins actifs

## Workspaces

Yarn a été le premier gestionnaire de paquets construit spécifiquement autour des workspaces, vous permettant de diviser votre projet en sous-composants.

### Configuration des Workspaces

Configurez les workspaces dans votre `package.json` racine :

```json
{
  "workspaces": [
    "packages/*"
  ]
}
```

### Commandes Workspace

- `yarn workspaces list` - Lister tous les workspaces
- `yarn workspace <name> <command>` - Exécuter une commande dans un workspace spécifique

## Fichiers de configuration

### .yarnrc.yml

Yarn v2+ utilise le fichier `.yarnrc.yml` pour la configuration :

```yaml
nodeLinker: pnp
yarnPath: .yarn/releases/yarn-4.0.0.cjs
```

### .yarnrc (Yarn v1)

Yarn v1 utilise le fichier `.yarnrc` :

```
--registry https://registry.npmjs.org
--strict-semver true
```

## Différences entre versions

### Yarn v4+ vs Yarn v1

| Fonctionnalité | Yarn v4+ | Yarn v1 |
|----------------|----------|---------|
| Architecture | Basée sur les plugins (Berry) | Traditionnelle |
| Vitesse d'installation | Plus rapide (installations parallèles) | Vitesse standard |
| Espace disque | Plus efficace (PnP) | node_modules standard |
| Workspaces | Support natif | Nécessite une configuration supplémentaire |
| Système de plugins | Support intégré | Non supporté |
| Configuration | .yarnrc.yml | .yarnrc |
