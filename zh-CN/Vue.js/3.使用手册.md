# Vue.js 使用手册

## 应用实例

每个 Vue 应用都是通过 `createApp` 函数创建一个新的应用实例：

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* 根组件选项 */
})
```

### 根组件
传入 `createApp` 的对象实际上是一个组件，每个应用都需要一个"根组件"，其他组件将作为其子组件。

### 挂载应用
应用实例必须在调用了 `.mount()` 方法后才会渲染出来：

```javascript
app.mount('#app')
```

### 应用配置
应用实例会暴露一个 `.config` 对象允许配置应用级选项：

```javascript
// 应用级错误处理器
app.config.errorHandler = (err) => {
  /* 处理错误 */
}

// 注册全局组件
app.component('TodoDeleteButton', TodoDeleteButton)
```

## 模板语法

Vue 使用基于 HTML 的模板语法，使我们能够声明式地将组件实例的数据绑定到呈现的 DOM 上。

### 文本插值
最基本的数据绑定形式是文本插值，使用"Mustache"语法（双大括号）：

```html
<span>Message: {{ msg }}</span>
```

### 原始 HTML
使用 `v-html` 指令插入 HTML：

```html
<p v-html="rawHtml"></p>
```

**安全警告**：动态渲染任意 HTML 非常危险，容易造成 XSS 漏洞。仅在内容安全可信时使用 `v-html`。

### Attribute 绑定
使用 `v-bind` 指令响应式地绑定 attribute：

```html
<!-- 完整语法 -->
<div v-bind:id="dynamicId"></div>

<!-- 简写语法 -->
<div :id="dynamicId"></div>

<!-- 同名简写（Vue 3.4+） -->
<div :id></div>
```

### 布尔型 Attribute
布尔型 attribute 依据 true/false 值决定是否存在于元素上：

```html
<button :disabled="isButtonDisabled">Button</button>
```

### 动态绑定多个值
通过不带参数的 `v-bind` 绑定对象中的多个 attribute：

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript 表达式

Vue 在所有数据绑定中都支持完整的 JavaScript 表达式：

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### 调用函数
可以在绑定表达式中使用组件暴露的方法：

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### 受限的全局访问
模板表达式仅能访问有限的全局对象列表（如 Math、Date），其他全局对象需通过 `app.config.globalProperties` 显式添加。

## 指令 (Directives)

指令是带有 `v-` 前缀的特殊 attribute。

### 参数 (Arguments)
某些指令需要参数，通过冒号隔开：

```html
<!-- v-bind 参数 -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on 参数 -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### 动态参数
在指令参数上使用 JavaScript 表达式，包含在方括号内：

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

### 修饰符 (Modifiers)
以点开头的特殊后缀，表明指令需要特殊处理：

```html
<form @submit.prevent="onSubmit">...</form>
```

## 响应式基础

### 声明响应式状态
使用 `ref()` 创建响应式引用：

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### 响应式对象
使用 `reactive()` 创建响应式对象：

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### 计算属性
使用 `computed()` 创建计算属性：

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### 侦听器
使用 `watch()` 侦听响应式数据的变化：

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// 侦听单个 ref
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## 组件基础

### 定义组件
```javascript
// 定义组件
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### 传递 Props
```javascript
// 子组件
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// 父组件
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### 发送事件
```javascript
// 子组件
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### 插槽 (Slots)
```html
<!-- 子组件 -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- 父组件 -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
