# JDK 安装指南

## 在 Windows 上安装 JDK

### 系统要求

在安装 JDK 之前，需要确保计算机满足以下最低要求：处理器方面，建议使用 Intel Core i3 或同等以上的处理器；内存方面，建议至少 4GB RAM，推荐 8GB 或以上；磁盘空间方面，需要至少 2GB 的可用空间用于安装 JDK，额外的空间用于开发项目；操作系统方面，Windows 10 或 Windows 11 的 64 位版本。

Windows 7 和 8.1 虽然可能可以运行，但 Oracle 官方不再支持这些操作系统。如果在较旧的系统上运行，可能会遇到兼容性问题。

### 下载 JDK

首先，访问 Oracle 官方网站 https://www.oracle.com/java/technologies/downloads/ 下载 JDK。在下载页面，可以看到不同版本的 JDK：JDK 25 是最新版本，JDK 21 是最新的长期支持版本，JDK 17 也是一个成熟的长期支持版本。根据项目需求选择合适的版本。

对于 Windows 系统，选择 x64 Installer（.exe）或 x64 MSI Installer（.msi）格式的安装包。.exe 是传统的安装向导，.msi 适合企业环境下的批量部署。如果只需要运行 Java 程序而不需要安装，可以选择 x64 Compressed Archive（.zip）格式，解压后即可使用。

下载时需要注意许可条款。Oracle JDK 在某些版本和用途下是免费的，但在商业使用和再分发方面有限制。如果需要完全免费的使用，可以考虑使用 OpenJDK 或其他基于 OpenJDK 的发行版，如 Eclipse Temurin。

### 安装步骤

找到下载的 JDK 安装文件（通常是 jdk-xx_windows-x64_bin.exe），双击运行。在安装向导中，点击“下一步”继续。首先会看到目标文件夹的选择界面，默认安装路径是 C:\Program Files\Java\jdk-xx。如果需要更改安装位置，点击“更改”按钮选择其他目录。建议使用默认路径以避免后续的配置问题。

接下来是功能选择界面，默认会选中所有功能，包括开发工具、源代码和公共 JRE。建议保持默认选项完整安装。点击“下一步”开始安装。安装过程可能需要几分钟时间，请耐心等待。

安装完成后，会看到提示信息。点击“关闭”完成安装。此时 JDK 已经安装到指定的目录中，但还需要配置环境变量才能在命令行中使用。

### 配置环境变量

配置环境变量的目的是让系统能够找到 JDK 的安装位置，这样就可以在任何目录下使用 java、javac 等命令。右键点击“此电脑”或“我的电脑”，选择“属性”。在左侧菜单中点击“高级系统设置”。

在打开的系统属性窗口中，点击“环境变量”按钮。在“系统变量”区域，点击“新建”。变量名输入 JAVA_HOME，变量值输入 JDK 的安装路径（如 C:\Program Files\Java\jdk-21），点击确定。

找到系统变量中的 Path 变量，点击“编辑”。点击“新建”，输入 %JAVA_HOME%\bin。确保这一行在最上面，这样会优先使用新安装的 JDK。点击确定保存所有设置。

打开一个新的命令行窗口（cmd 或 PowerShell），输入 java -version 和 javac -version，如果能看到版本信息，说明配置成功。

### 验证安装

打开命令提示符（Win+R，输入 cmd），依次输入以下命令验证 JDK 是否正确安装：

输入 java -version，应该显示 Java 运行时环境的版本信息。输入 javac -version，应该显示 Java 编译器的版本信息。输入 echo %JAVA_HOME%，应该显示之前设置的 JAVA_HOME 路径。

如果任何一个命令出现错误，检查环境变量配置是否正确，或者 JDK 是否正确安装。

## 在 macOS 上安装 JDK

### 使用 DMG 安装包

macOS 用户可以通过 DMG 安装包快速安装 JDK。访问 Oracle 官方下载页面，下载 macOS 版本的 JDK。对于 Apple Silicon（M1/M2/M3 芯片），选择 ARM64 DMG Installer；对于 Intel 芯片的 Mac，选择 x64 DMG Installer。

下载完成后，打开 DMG 文件，会看到一个 pkg 安装文件。双击运行安装向导，按照提示完成安装。安装程序会将 JDK 安装到 /Library/Java/JavaVirtualMachines/ 目录下。

安装完成后，打开终端，输入 java -version 验证安装。如果系统有多个 JDK 版本，可以使用 /usr/libexec/java_home -V 查看所有已安装的版本。

### 使用 Homebrew 安装

如果已经安装了 Homebrew 包管理器，可以使用它来安装 JDK，这更加简单和方便。打开终端，执行以下命令更新 Homebrew：brew update。然后安装 JDK 21：brew install openjdk@21。

安装完成后，需要将 JDK 添加到 PATH。执行以下命令将 JDK 链接到系统：sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21。

之后在 ~/.zshrc 或 ~/.bash_profile 中添加环境变量：export PATH="/usr/local/opt/openjdk@21/bin:$PATH"。保存后执行 source ~/.zshrc 使配置生效。

### 配置 JAVA_HOME

在 macOS 上，JAVA_HOME 环境变量的配置不是必须的，因为系统会自动找到已安装的 JDK。但在某些工具（如 Maven、Gradle）中可能需要手动设置。在终端中编辑 ~/.zshrc 文件，添加：export JAVA_HOME=$(/usr/libexec/java_home)。保存后执行 source ~/.zshrc 使配置生效。

## 在 Linux 上安装 JDK

### 使用 APT 安装（Debian/Ubuntu）

在 Debian 和 Ubuntu 系统上，可以使用 APT 包管理器安装 OpenJDK。首先更新软件包列表：sudo apt update。然后安装 OpenJDK 21 开发版：sudo apt install openjdk-21-jdk。如果只需要运行 Java 程序而不需要编译器，可以安装 JRE：sudo apt install openjdk-21-jre。

安装过程中会提示创建 JAVA_HOME 目录，选择“是”自动配置。安装完成后，使用 java -version 和 javac -version 验证安装。

如果需要手动配置 JAVA_HOME，编辑 ~/.bashrc 或 ~/.profile 文件，添加：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64。然后执行 source ~/.bashrc 使配置生效。

### 使用 YUM 安装（Red Hat/CentOS）

在 Red Hat、CentOS、Fedora 等使用 YUM 的系统上，可以直接安装 JDK。执行以下命令安装 OpenJDK 21：sudo yum install java-21-openjdk java-21-openjdk-devel。-devel 包包含编译器和其他开发工具。

安装完成后，使用alternatives 命令设置默认的 Java 版本：sudo alternatives --config java。按照提示选择要设置为默认的 Java 版本。

编辑 ~/.bashrc 文件，添加：export JAVA_HOME=/usr/lib/jvm/java-21-openjdk。执行 source ~/.bashrc 使配置生效。

### 手动安装二进制文件

对于需要安装特定版本 JDK 的情况，可以手动下载和解压二进制归档文件。访问 Oracle 或 OpenJDK 下载页面，下载对应版本的 .tar.gz 文件。

创建 JDK 安装目录：sudo mkdir -p /opt/jdk。解压下载的文件：sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk。进入解压后的目录，记录完整路径。

编辑 ~/.bashrc 文件，添加：export JAVA_HOME=/opt/jdk/jdk-21 和 export PATH=$JAVA_HOME/bin:$PATH。执行 source ~/.bashrc 使配置生效。

## 安装多个 JDK 版本

### Windows 多版本管理

在 Windows 上安装多个 JDK 版本后，需要通过环境变量切换默认版本。最简单的方法是修改 PATH 环境变量中 JDK bin 目录的顺序，将要使用的版本放在前面。

另一个方法是使用 JAVA_HOME 环境变量，修改其指向不同的 JDK 安装目录。这种方法对大多数工具都有效，但某些工具可能直接使用 PATH 中的 java 命令而不是 JAVA_HOME。

还有一些第三方工具可以帮助管理多个 JDK 版本，如 SDKMAN! 的 Windows 版本或者 JetBrains 的 Toolbox。这些工具可以更方便地在不同版本之间切换。

### macOS 多版本管理

在 macOS 上，可以使用 jenv 工具来管理多个 JDK 版本。首先安装 jenv：brew install jenv。然后将 jenv 初始化添加到 shell 配置文件中。

使用 jenv 添加已安装的 JDK：jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home。然后可以使用 jenv versions 查看所有管理的版本，使用 jenv global xx 设置全局默认版本。

另一个方法是使用 /usr/libexec/java_home -v 来指定特定版本运行程序。例如：/usr/libexec/java_home -v 21 --exec java -version。

### Linux 多版本管理

在 Linux 上，可以使用 update-alternatives 工具管理多个 JDK 版本。首先为每个 JDK 版本创建 alternatives 条目：sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1。

然后使用 update-alternatives --config java 选择默认版本。jenv 在 Linux 上同样可用，安装和配置方法与 macOS 类似。

## 常见安装问题

### 安装后找不到 java 命令

如果在安装 JDK 后找不到 java 命令，首先确认 JDK 是否正确安装。检查 JAVA_HOME 环境变量是否设置正确，检查 PATH 中是否包含 %JAVA_HOME%\bin（Windows）或 $JAVA_HOME/bin（Linux/macOS）。

在 Windows 上，打开新的命令行窗口尝试，因为环境变量的修改可能不会立即生效。在 Linux/macOS 上，执行 source ~/.bashrc 或重新打开终端。

如果问题仍然存在，检查 JDK 安装目录是否正确，目录中是否包含 bin/java 文件。

### 权限问题

在 Linux 和 macOS 上安装 JDK 时，可能遇到权限问题。使用包管理器安装时，通常需要管理员权限（sudo）。手动安装时，确保安装目录有适当的读写权限。

如果普通用户无法使用 java 命令，检查 JAVA_HOME 目录和 JDK bin 目录的权限设置。可以使用 ls -la /usr/lib/jvm 检查目录权限。

### JAVA_HOME 设置无效

如果设置 JAVA_HOME 后无效，检查环境变量设置的语法是否正确。在 Windows 上，变量值不应包含引号。在 Linux/macOS 上，确保使用正确的 export 命令。

同时检查是否有其他脚本或配置文件覆盖了 JAVA_HOME 的设置。在 Linux 上，系统级别的配置可能在 /etc/environment 或 /etc/profile.d/ 中。

### 旧版本卸载

在升级 JDK 后，可能需要卸载旧版本以节省磁盘空间。在 Windows 上，通过控制面板的“程序和功能”找到旧版本 JDK，点击卸载。在 macOS 上，删除 /Library/Java/JavaVirtualMachines/ 目录下的旧版本文件夹。在 Linux 上，使用包管理器卸载：sudo apt remove openjdk-xx-jdk（APT）或 sudo yum remove java-xx-openjdk（YUM）。
