# Mockito

## 概要

Mockitoは、テスト駆動開発（TDD）とビヘイビア駆動開発（BDD）のために設計されたJavaエコシステムで最も人気のある单元テストモックフレームワークです。開発者がモックオブジェクトを作成し、控制された方法で実オブジェクトの动作をシミュレートすることで、独立的で信頼性の高い单元テストを実現できます。

## 主要機能

Mockitoは、学習と使用が簡単なクリーンでシンプルなAPI設計が特徴です。強力な機能は、メソッド呼び出しの検証、戻り値のスタブ設定、例外の抛出、引数マッチングなどをサポートします。優れた可読性によりテストコードは直感的で理解しやすく、検証失敗時には明確なエラーメッセージが表示されます。活発なコミュニティはフレームワークの継続的なメンテナンスと包括的なドキュメントを保証します。

## クイックスタート

### モックオブジェクトの作成

```java
List<String> mockedList = mock(List.class);
```

### 动作の定義

```java
when(mockedList.get(0)).thenReturn("first");
```

### インタラクションの検証

```java
verify(mockedList).get(0);
```

## リソース

公式ウェブサイトは完全なドキュメントとサンプルコードを提供します。GitHubリポジトリにはソースコードとリリース情報が含まれます。オンラインドキュメントはAPIリファレンスと使用ガイドを提供します。

## 要件

Mockito 5.xにはJDK 11以上が必要です。Mockito 4.xはJDK 8をサポートします。プロジェクトはMavenまたはGradleを使用して依存関係管理を行えます。
