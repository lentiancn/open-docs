# Docker Häufig gestellte Fragen

Diese Seite listet häufige Probleme und Lösungen für Docker-Installation und -Nutzung.

## Installationsprobleme

### Docker Desktop startet nicht

**Symptome**: Docker Desktop startet nicht oder stürzt unmittelbar nach dem Start ab

**Lösungen**:

1. Stellen Sie sicher, dass das System die Hardware-Anforderungen erfüllt:
   - Windows: Aktivieren Sie Hyper-V oder installieren Sie WSL 2
   - Mac: Stellen Sie sicher, dass Sie keine veralteten Mac-Modelle verwenden

2. Docker Desktop zurücksetzen:
   - Windows: Rechtsklick auf Docker-Symbol → Fehlerbehebung → Auf Werkseinstellungen zurücksetzen
   - Mac: Docker → Fehlerbehebung → Auf Werkseinstellungen zurücksetzen

3. Überprüfen Sie die Systemprotokolle:
   - Windows: Ereignisanzeige → Windows-Protokolle → Anwendung
   - Mac: Öffnen Sie die Konsole und suchen Sie nach Docker-Fehlern

### Keine Berechtigung für Docker-Befehle

**Symptome**: "permission denied while trying to connect to the Docker daemon socket"

**Lösungen**:

1. Fügen Sie Ihren Benutzer zur Docker-Gruppe hinzu:
   ```bash
   sudo usermod -aG docker $USER
   ```

2. Melden Sie sich ab und wieder an

3. Oder verwenden Sie sudo vor Docker-Befehlen

### WSL 2-Installationsfehler unter Windows

**Symptome**: "WSL 2 installation is incomplete"

**Lösungen**:

1. Aktivieren Sie das Windows-Subsystem für Linux:
   ```bash
   wsl --install
   ```

2. Starten Sie Ihren Computer neu

3. Aktualisieren Sie WSL auf die neueste Version:
   ```bash
   wsl --update
   ```

## Nutzungsprobleme

### Container startet nicht

**Symptome**: Container bleibt im Status "Exited" oder "Created"

**Lösungen**:

1. Überprüfen Sie die Container-Logs:
   ```bash
   docker logs container_name
   ```

2. Stellen Sie sicher, dass alle erforderlichen Ports verfügbar sind

3. Überprüfen Sie die Image-Integrität

### Port bereits belegt

**Symptome**: "port is already allocated"

**Lösungen**:

1. Finden Sie den belegten Port:
   ```bash
   netstat -ano | findstr :8080
   ```

2. Stoppen Sie den anderen Dienst oder verwenden Sie einen anderen Port

3. Ändern Sie die Portweiterleitung:
   ```bash
   docker run -p 8081:80 nginx
   ```

### Speicherplatzprobleme

**Symptome**: "no space left on device"

**Lösungen**:

1. Unbenutzte Container entfernen:
   ```bash
   docker container prune
   ```

2. Unbenutzte Images entfernen:
   ```bash
   docker image prune -a
   ```

3. Unbenutzte Volumes entfernen:
   ```bash
   docker volume prune
   ```

### Netzwerkprobleme

**Symptome**: Container kann keine Verbindung nach außen herstellen

**Lösungen**:

1. DNS-Einstellungen überprüfen:
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

2. Docker-Netzwerk zurücksetzen:
   ```bash
   docker network prune
   ```

3. Firewall-Einstellungen überprüfen

## Performance-Probleme

### Langsame Container-Performance

**Lösungen**:

1. Ressourcenlimits überprüfen:
   ```bash
   docker stats
   ```

2. Mehr RAM zuweisen (Docker Desktop → Einstellungen → Ressourcen)

3. SSD statt HDD für Docker verwenden

### Hohe Festplattennutzung

**Lösungen**:

1. Build-Cache regelmäßig bereinigen:
   ```bash
   docker builder prune
   ```

2. Alte Versionen von Images entfernen

3. Log-Größe begrenzen

## Sicherheitsfragen

### Container als Root ausführen

**Empfehlung**:

Verwenden Sie einen nicht-privilegierten Benutzer:

```dockerfile
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser
USER appuser
```

### Sicherheitslücken in Images

**Lösungen**:

1. Verwenden Sie offizielle, aktuelle Images

2. Scannen Sie Images regelmäßig:
   ```bash
   docker scan image_name
   ```

3. Erwägen Sie die Verwendung von distroless-Images

## Fortgeschrittene Fragen

### Docker im CI/CD verwenden

**Empfehlungen**:

1. Nutzen Sie Build-Kits für parallele Builds
2. Verwenden Sie Multi-Stage-Builds für kleinere Images
3. Cachen Sie Abhängigkeiten effektiv
4. Verwenden Sie Docker Buildx für plattformübergreifende Builds

### Datenpersistenz

**Empfehlungen**:

1. Verwenden Sie benannte Volumes für kritische Daten
2. Sichern Sie Volumes regelmäßig
3. Nutzen Sie Bind-Mounts für Entwicklungszwecke

### Mehrere Umgebungen

**Empfehlungen**:

1. Verwenden Sie Docker Compose für Umgebungsvariablen
2. Nutzen Sie .env-Dateien
3. Erwägen Sie Docker Swarm oder Kubernetes für Produktion
