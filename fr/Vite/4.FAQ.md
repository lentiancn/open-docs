# Vite FAQ

## Questions Générales

### Qu'est-ce que Vite?
Vite est un outil de construction frontend de nouvelle génération créé par Evan You. Il offre un HMR extrêmement rapide et des builds optimisés en utilisant des modules ES natifs.

### Vite est-il uniquement pour Vue?
Non, Vite est agnostique et fonctionne avec Vue, React, Svelte, Preact et JavaScript vanilla.

### Ai-je besoin d'un package manager spécifique?
Vite fonctionne avec npm, yarn et pnpm.

## Questions d'Installation

### Comment créer un nouveau projet Vite?
```bash
npm create vite@latest my-project -- --template vue
```

### Comment installer Vite dans un projet existant?
```bash
npm install -D vite
```

## Questions d'Utilisation

### Comment démarrer le serveur de développement?
```bash
npm run dev
```

### Comment faire un build de production?
```bash
npm run build
```

### Comment prévisualiser le build de production?
```bash
npm run preview
```

### Comment configurer Vite?
Créez `vite.config.js`:

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: { port: 3000 }
})
```

## Dépannage

### Module non trouvé
- Vérifier que toutes les dépendances sont installées
- Vérifier les chemins d'import

### Port déjà utilisé
Modifier le port dans vite.config.js:
```javascript
server: { port: 3001 }
```

## Questions Avancées

### Comment fonctionne le HMR de Vite?
Vite utilise des modules ES natifs et établit une connexion WebSocket pour HMR. Quand un fichier change, Vite met à jour ce module sans recharger la page.

### Comment ajouter des plugins personnalisés?
```javascript
function myPlugin() {
  return {
    name: 'my-plugin',
    transform(code, id) { return code }
  }
}
export default defineConfig({ plugins: [myPlugin()] })
```
