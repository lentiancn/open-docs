# Vue Guía de Instalación

Vue es un framework progresivo de JavaScript para construir interfaces de usuario.

## Versiones

Vue tiene dos versiones principales en uso activo:

| Versión | Fecha de lanzamiento | Estado |
|---------|---------------------|--------|
| Vue 3 | Septiembre 2020 | Actual - Recomendado |
| Vue 2 | Octubre 2016 | Legado (Fin de vida en 2023) |

**Recomendación:** Use Vue 3 para todos los proyectos nuevos. Vue 2 terminó su soporte en 2023.

### Diferencias principales: Vue 2 vs Vue 3

| Característica | Vue 2 | Vue 3 |
|----------------|-------|-------|
| Arquitectura | Options API | Composition API + Options API |
| Reactividad | Object.defineProperty | Proxy |
| TypeScript | Soporte parcial | Soporte completo |
| Rendimiento | Bueno | Significativamente mejor |
| Tamaño | ~20KB gzip | ~10KB gzip |
| Ciclo de vida | created, mounted | setup + onMounted |
| Teleport | No | Sí |
| Fragments | No | Sí |
| Múltiples raíces | No | Sí |

## Requisitos previos

### Requisitos del sistema

| Requisito | Vue 2 | Vue 3 |
|-----------|-------|-------|
| Node.js | 8.9+ | 16+ (18+ recomendado) |
| npm | 5.6+ | 8+ |
| Navegador | IE9+ | IE11 no soportado |

### Software requerido

- **Node.js** - Entorno de ejecución JavaScript
- **npm** - Gestor de paquetes (incluido con Node.js)
- **Git** - Control de versiones (opcional pero recomendado)

## Métodos de instalación

### Método 1: Usando npm (Recomendado)

#### Vue 3 (Recomendado)

```bash
# Crear un nuevo proyecto Vue
npm create vue@latest

# O usar Vue CLI
npm install -g @vue/cli

# Crear proyecto
vue create my-vue-app
```

#### Vue 2 (Legado)

```bash
# Instalar Vue CLI
npm install -g @vue/cli

# Crear proyecto Vue 2
vue create my-vue2-app --default
```

### Método 2: Usando Vite (Recomendado para Vue 3)

Vite es una herramienta de construcción moderna que proporciona un servidor de desarrollo extremadamente rápido.

```bash
# Crear proyecto Vue 3 con Vite
npm create vite@latest my-vue-app -- --template vue

# Navegar al proyecto
cd my-vue-app

# Instalar dependencias
npm install

# Iniciar servidor de desarrollo
npm run dev
```

### Método 3: Usando CDN

#### Vue 3 CDN

```html
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
```

#### Vue 2 CDN

```html
<script src="https://unpkg.com/vue@2/dist/vue.js"></script>
```

### Método 4: Usando Vue CLI

```bash
# Instalar Vue CLI globalmente
npm install -g @vue/cli

# Verificar instalación
vue --version

# Crear proyecto
vue create my-project

# O usar GUI
vue ui
```

### Método 5: Usando Nuxt.js

Nuxt es un framework para crear aplicaciones Vue con mejor estructura.

```bash
# Crear proyecto Nuxt 3
npx nuxi@latest init my-nuxt-app

# Navegar al proyecto
cd my-nuxt-app

# Instalar dependencias
npm install

# Iniciar servidor de desarrollo
npm run dev
```

## Configuración del entorno

### Instalar Node.js

#### Windows

1. Descargue Node.js desde https://nodejs.org
2. Ejecute el instalador (versión LTS recomendada)
3. Verificar: `node --version`

#### macOS

```bash
# Usando Homebrew
brew install node
```

#### Linux (Ubuntu/Debian)

```bash
# Usando NodeSource
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# O usando nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
```

### Configurar npm

```bash
# Ver versión de npm
npm --version

# Configurar registry de npm (opcional, para China)
npm config set registry https://registry.npmmirror.com
```

## Configuración de Vue Router

### Instalación

```bash
npm install vue-router@4
```

### Configuración básica (Vue 3)

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

## Pinia (Gestión de estado)

### Instalación

```bash
npm install pinia
```

### Configuración básica (Vue 3)

```javascript
// src/stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  getters: {
    doubleCount: (state) => state.count * 2
  },
  actions: {
    increment() {
      this.count++
    }
  }
})
```

## Vue 2 Options API (Legado)

```javascript
new Vue({
  el: '#app',
  data() {
    return {
      message: 'Hello Vue 2!',
      items: []
    }
  },
  computed: {
    reversedMessage() {
      return this.message.split('').reverse().join('')
    }
  },
  methods: {
    handleClick() {
      console.log('Clicked!')
    }
  },
  created() {
    console.log('Component created')
  }
})
```

## Vue 3 Composition API

```javascript
import { ref, computed, onMounted } from 'vue'

export default {
  setup() {
    // Estado reactivo
    const message = ref('Hello Vue 3!')
    const count = ref(0)
    
    // Propiedades calculadas
    const reversedMessage = computed(() => {
      return message.value.split('').reverse().join('')
    })
    
    // Métodos
    const increment = () => {
      count.value++
    }
    
    // Ciclo de vida
    onMounted(() => {
      console.log('Component mounted')
    })
    
    return {
      message,
      count,
      reversedMessage,
      increment
    }
  }
}
```

## Solución de problemas

### Problemas comunes

#### npm install falla

```bash
# Limpiar caché de npm
npm cache clean --force

# Eliminar node_modules y reinstallar
rm -rf node_modules
npm install
```

#### Puerto en uso

```bash
# Usar puerto diferente
npm run dev -- --port 3000
```

#### Problemas de versión de Node

```bash
# Ver versión de Node
node --version

# Usar nvm para cambiar versión
nvm use 18
```

## Mejores prácticas

1. **Usar Vue 3** - A menos que tenga requisitos específicos de Vue 2
2. **Usar Vite** - Desarrollo y construcción más rápidos
3. **Usar Composition API** - Mejor soporte de TypeScript y organización de código
4. **Usar TypeScript** - Mejor experiencia de desarrollador
5. **Usar Pinia** - Para gestión de estado (no Vuex en proyectos nuevos)
