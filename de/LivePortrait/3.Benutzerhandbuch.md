# LivePortrait Benutzerhandbuch

Dieses Dokument bietet detaillierte Gebrauchsanweisungen für LivePortrait, einschließlich grundlegender Inferenz, Gradio-Oberfläche, Tiermodus, Video-Bearbeitung und anderer Funktionen.

## Grundlegende Inferenz

### Bildsteuerung

Die grundlegendste Verwendung beinhaltet die Verwendung eines statischen Bildes als Quelle und eines Steuerungsvideos als Animationsreferenz:

```bash
python inference.py
```

Standardmäßig verwendet das Skript Beispielbilder aus dem Verzeichnis assets/examples/source und Beispiel-Steuerungsvideos aus dem Verzeichnis assets/examples/driving. Bei erfolgreicher Ausführung werden Ergebnisse im animations-Verzeichnis gespeichert, mit dem Dateinamen "Quelldatei--Steuerungsdatei_concat.mp4".

Das Ausgabevideo enthält drei Teile: das Steuerungsvideo, das Quellbild/Video und das generierte Animationsergebnis, sequentiell zusammengefügt.

### Benutzerdefinierte Eingaben

Sie können Ihre eigenen Quell- und Steuerungsdateien mit Befehlszeilenparametern angeben:

```bash
# Quelldatei ist ein Bild
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# Quelldatei ist ein Video
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

Der Parameter -s gibt den Pfad zum Quellbild oder Quellvideo an, während der Parameter -d den Pfad zum Steuerungsvideo angibt.

### Weitere Optionen anzeigen

Alle verfügbaren Befehlszeilenparameter anzeigen:

```bash
python inference.py -h
```

Parameter umfassen Ausgabepfad, Bildgröße, Animationsgeschwindigkeit, Zusammenführungsmethode und andere detaillierte Konfigurationen.

## Verwenden der Gradio-Weboberfläche

Für Benutzer, die mit Befehlszeilen nicht vertraut sind, bietet LivePortrait eine grafische Oberfläche basierend auf Gradio.

### Mensch-Modus-Oberfläche starten

```bash
python app.py
```

Nach dem Start öffnen Sie Ihren Browser zur angezeigten URL (normalerweise http://localhost:7860). Laden Sie in der Oberfläche ein Quellbild oder Video hoch, wählen Sie dann ein Steuerungsvideo aus und klicken Sie auf die Generierungsschaltfläche, um das Animationsergebnis zu erhalten.

Sie können den Port mit dem Parameter --server_port angeben, einen öffentlichen Link mit dem Parameter --share generieren und die Serveradresse mit dem Parameter --server_name festlegen.

### Tiermodus-Oberfläche starten

Auf Linux-Systemen mit NVIDIA-GPU können Sie die Tiermodus-Oberfläche starten:

```bash
python app_animals.py
```

## Spezifikationen für Steuerungsvideos

Für optimale Ergebnisse sollten Steuerungsvideos die folgenden Anforderungen erfüllen:

**Seitenverhältnis**: Es wird empfohlen, auf 1:1-Quadrat zuzuschneiden (z.B. 512x512 oder 256x256 Pixel). Sie können auch die automatische Zuschneidefunktion aktivieren (--flag_crop_driving_video).

**Inhaltsanforderungen**: Das Steuerungsvideo sollte sich auf den Kopfbereich konzentrieren, ähnlich wie in den Beispielvideos. Schulterbewegungen sollten minimiert werden.

**Anforderung an das erste Bild**: Das erste Bild des Steuerungsvideos sollte ein.frontales Gesicht mit neutralem Ausdruck sein. Dies ist entscheidend für die Gewährleistung genauer Animationseffekte.

### Automatisches Zuschneiden

Wenn Ihr Steuerungsvideo die oben genannten Anforderungen nicht erfüllt, können Sie die automatische Zuschneidefunktion verwenden:

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

Wenn die Ergebnisse des automatischen Zuschneidens nicht zufriedenstellend sind, können Sie die Parameter --scale_crop_driving_video und --vy_ratio_crop_driving_video anpassen, um Skalierung und Versatz feinabzustimmen.

## Verwenden von Bewegungsvorlagen

Um die Inferenz zu beschleunigen und die Privatsphäre zu schützen, unterstützt LivePortrait die Verwendung von vorgenerierten Bewegungsvorlagendateien (.pkl-Format). Diese Vorlagendateien zeichnen Bewegungsinformationen aus Steuerungsvideos auf und können wiederverwendet werden, ohne das Originalvideo jedes Mal zu verarbeiten.

### Verwendung

```bash
# Porträt-Animation
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# Porträt-Video-Bearbeitung
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

Vorlagendateien beschleunigen nicht nur die Inferenz, sondern schützen auch die Privatsphäre—die Verwendung von Vorlagen macht das Hochladen ursprünglicher Steuerungsvideos überflüssig.

## Porträt-Video-Bearbeitung (Video-zu-Video)

LivePortrait unterstützt das Bearbeiten und Ändern von Ausdrücken und Kopfbewegungen von Personen in vorhandenen Videos.

### Grundlegende Verwendung

```bash
# Quelle ist Bild, Steuerung ist Video
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### Animationsparameter anpassen

**Ausdrucksstärke**: Passen Sie die Stärke der Ausdrucksänderungen mit dem Parameter --driving_multiplier an. Werte größer als 1 verstärken Ausdrucksänderungen, während Werte kleiner als 1 sie reduzieren.

**Zusammenführungskontrolle**: Verwenden Sie den Parameter --flag_stitching, um zu steuern, ob das Zusammenfügen aktiviert ist. Das Deaktivieren des Zusammenführens kann die Darstellung von Gesichtsregionen verändern.

## Inferenzbeschleunigung

### Torch Compile-Beschleunigung

LivePortrait unterstützt die Verwendung der torch.compile-Funktion von PyTorch zur Beschleunigung der Inferenz. Die erste Inferenz löst einen Optimierungsprozess aus (ca. eine Minute), danach verbessert sich die Inferenzgeschwindigkeit um 20-30%. Die Beschleunigungseffekte können je nach CUDA-Version variieren.

```bash
python app.py --flag_do_torch_compile
```

Hinweis: torch.compile wird auf Windows und macOS nicht unterstützt.

### Geschwindigkeitstest

Das Projekt bietet ein Inferenz-Geschwindigkeitstest-Skript:

```bash
python speed.py
```

Die Testergebnisse zeigen die Verarbeitungszeit für jedes Modul und helfen Ihnen, Leistungsengpässe zu identifizieren.

## Tiermodus

Der Tiermodus unterstützt das Animieren von Porträts von Haustieren wie Katzen und Hunden.

### Nutzungsbedingungen

Der Tiermodus wurde nur in folgenden Umgebungen getestet: Linux-System mit NVIDIA-GPU, Windows-System mit NVIDIA-GPU.

### Installationsanforderungen

Bevor Sie den Tiermodus verwenden, installieren Sie die X-Pose-Abhängigkeit wie in der Installationsanleitung beschrieben und laden Sie die Tiermodellgewichte herunter.

### Tiermodus-Inferenz ausführen

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### Tiermodus-Oberfläche

```bash
python app_animals.py
```

## Regionssteuerung

LivePortrait unterstützt die Animationssteuerung für bestimmte Bildbereiche und ermöglicht feinere Effektanpassungen.

### Regionssteuerung aktivieren

In der Gradio-Oberfläche finden Sie Optionsregionssteuerung, um auszuwählen, welche Gesichtsregionen animiert werden sollen (z.B. Augen, Mund, Kopfhaltung usw.).

## Porträt-Bearbeitung

In der Gradio-Oberfläche können Benutzer verschiedene Attribute des Quellporträts präzise bearbeiten, einschließlich Ausdruck, Blickrichtung, Kopfwinkel und mehr.

## Datenschutz

LivePortrait bietet mehrere Datenschutzfunktionen:

**Bewegungsvorlagen**: Verwenden Sie .pkl-Vorlagendateien anstelle von Original-Steuerungsvideos, um das Hochladen sensibler Inhalte zu vermeiden

**Automatisches Zuschneiden**: Steuerungsvideos werden automatisch zugeschnitten und auf den Kopf fokussiert, wodurch Hintergrundinformationen reduziert werden

**Videoschutz**: Vorlagenerstellungsfunktion konvertiert Videos vor der Verwendung in Bewegungsvorlagen

## Ausgabeformat

Das Standardausgabeformat ist eine MP4-Videodatei, die drei Teile enthält: Steuerungsvideo, Quellinhalt und generiertes Ergebnis. Verwenden Sie den Parameter --no_flag_concat, um nur den generierten Animationsbereich auszugeben.

## Integration von Community-Tools

LivePortrait wird von mehreren Community-Projekten integriert:

**ComfyUI**: Mehrere ComfyUI-Knoten unterstützen LivePortrait mit Echtzeit-Vorschaufunktionen

**Stable Diffusion WebUI**: Die sd-webui-live-portrait-Erweiterung integriert LivePortrait in SD WebUI

**FaceFusion**: Version 3.0 integriert LivePortrait als Ausdruckswiederhersteller und Gesichtseditor-Prozessor

**HuggingFace Spaces**: LivePortrait online erleben ohne lokale Installation

## Best Practices

Für optimale Animationseffekte befolgen Sie diese Best Practices:

Wählen Sie hochwertige Quellbilder: Klare Frontalporträts erzielen die besten Ergebnisse

Verwenden Sie professionelle Steuerungsvideos: Videos mit frontaler, natürlicher Expression führen zu besseren Effekten

Steuern Sie die Länge des Steuerungsvideos: Übermäßig lange Steuerungsvideos erhöhen die Verarbeitungszeit und können zu Effektverlust führen

Legen Sie Parameter vernünftig fest: Passen Sie die Ausdrucksstärke und andere Parameter basierend auf tatsächlichen Ergebnissen an

Halten Sie die Beleuchtung konsistent: Versuchen Sie, die Beleuchtungsbedingungen zwischen Quellbildern und Steuerungsvideos konsistent zu halten
