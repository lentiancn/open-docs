# GitLab Manuel Utilisateur

> Guide complet pour utiliser GitLab en développement et opérations

---

## Pour Commencer

### Créer un Compte

1. Accédez à votre instance GitLab
2. Cliquez sur "S'inscrire" pour créer un compte
3. Vérifiez votre adresse email
4. Complétez votre profil utilisateur

### Créer un Projet

1. Cliquez sur "Nouveau Projet"
2. Choisissez un modèle de projet ou un projet vide
3. Définissez le nom et la description du projet
4. Configurez le niveau de visibilité (privé, interne, public)
5. Cliquez sur "Créer le Projet"

## Opérations Git

### Cloner un Dépôt

```bash
# Cloner avec HTTPS
git clone https://gitlab.example.com/utilisateur/projet.git

# Cloner avec SSH
git clone git@gitlab.example.com:utilisateur/projet.git
```

### Flux de Travail Git de Base

```bash
# Créer une nouvelle branche
git checkout -b feature/nouvelle-fonction

# Faire des modifications et les indexer
git add .
git commit -m "Ajouter nouvelle fonction"

# Pusher vers le remote
git push origin feature/nouvelle-fonction

# Créer une merge request depuis la branche
```

### Gérer les Branches

```bash
# Lister les branches
git branch -a

# Supprimer une branche locale
git branch -d feature/ancienne-fonction

# Supprimer une branche distante
git push origin --delete feature/ancienne-fonction
```

## Suivi des Problèmes

### Créer des Problèmes

1. Accédez à votre projet
2. Allez dans Problèmes > Nouveau Problème
3. Ajoutez un titre et une description
4. Assignez des étiquettes, jalons et responsables
5. Définissez le poids et la date d'échéance
6. Soumettez le problème

### Utiliser les Tableaux de Problèmes

1. Allez dans Problèmes > Tableaux
2. Créez un nouveau tableau ou utilisez celui par défaut
3. Ajoutez des listes pour chaque étape (À faire, En cours, Terminé)
4. Glissez-déposez les problèmes entre les listes

### Étiquettes et Jalons

- Créer des étiquettes descriptives pour la catégorisation
- Utiliser les jalons pour suivre les progrès
- Fermer les problèmes avec des messages de commit

## Merge Requests

### Créer une Merge Request

1. Poussez votre branche de fonctionnalité vers GitLab
2. Cliquez sur "Créer une Merge Request"
3. Remplissez le titre et la description
4. Sélectionnez les branches source et cible
5. Ajoutez des étiquettes et des réviseurs
6. Soumettez la merge request

### Processus de Révision de Code

- Les réviseurs peuvent ajouter des commentaires sur des lignes spécifiques
- Utiliser les discussions en fil pour les conversations
- Approuver ou demander des modifications
- Résoudre tous les fils avant de fusionner

### Squash et Merge

- Combiner les commits pour garder un historique propre
- Choisir de supprimer la branche source après fusion
- Modifier le message de commit avant de fusionner

## Pipelines CI/CD

### .gitlab-ci.yml de Base

```yaml
stades:
  - build
  - test
  - deploy

build:
  stade: build
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/

test:
  stade: test
  script:
    - npm test
  coverage: '/Coverage: \d+\.\d+%/'

deploy:
  stade: deploy
  script:
    - npm run deploy
  only:
    - main
```

### Exécuter des Pipelines

- Les pipelines s'exécutent automatiquement lors du push
- Déclencher manuellement depuis l'UI
- Voir le statut du pipeline dans les merge requests

### Variables de Pipeline

- Définir des variables dans les paramètres du projet
- Utiliser les variables CI/CD pour les données sensibles
- Variables masquées et protégées disponibles

## Wiki

### Créer des Pages Wiki

1. Allez dans Projet > Wiki
2. Cliquez sur "Nouvelle Page"
3. Écrivez le contenu en Markdown
4. Ajoutez la navigation de la barre latérale
5. Enregistrez la page

### Fonctionnalités Wiki

- Support Markdown
- Pièces jointes
- Structure de pages imbriquées
- Historique des versions
