# Bootstrap FAQ

Common questions and answers.

## Installation

### Q: CDN not loading?

**Solutions**:
1. Check network connection
2. Confirm URL is correct
3. Use alternative CDN (bootcdn, staticfile)

### Q: npm installation failed?

**Solutions**:
1. Clear npm cache: `npm cache clean --force`
2. Check Node.js version
3. Use admin privileges

### Q: Do I still need jQuery?

**Solutions**:
No. Bootstrap 5 removed jQuery dependency, uses vanilla JavaScript.

## Usage

### Q: Styles not working?

**Solutions**:
1. Confirm CSS file is properly imported
2. Check CSS import order
3. Clear browser cache

### Q: Responsive not working?

**Solutions**:
1. Add viewport meta tag:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
2. Check grid class usage

### Q: Components not working?

**Solutions**:
1. Confirm Bootstrap JavaScript is imported
2. Check if Popper.js is properly imported
3. Check console for errors

### Q: How to customize Bootstrap?

**Solutions**:
1. Use SASS variable overrides
2. Use CSS custom properties
3. Use Bootstrap customizer

## Layout

### Q: Container width incorrect?

**Solutions**:
- `.container`: Fixed width
- `.container-fluid`: 100% width
- `.container-{breakpoint}`: Responsive container

### Q: How to adjust column spacing?

**Solutions**:
1. Use gutter classes: `g-4`, `g-md-5`
2. Customize `$grid-gutter-width` SASS variable

## Compatibility

### Q: IE browser not supported?

**Solutions**:
- Bootstrap 5 no longer supports IE 10 and below
- Use Bootstrap 4 for compatibility

### Q: Older browser support?

**Solutions**:
Use Bootstrap 4 for better older browser support.

## Performance

### Q: Files too large?

**Solutions**:
1. Use SASS to import only needed components
2. Remove unused CSS
3. Use production versions (.min files)

### Q: How to reduce JavaScript size?

**Solutions**:
Import individual components as needed.
