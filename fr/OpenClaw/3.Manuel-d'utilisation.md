# Manuel utilisateur OpenClaw

Ce manuel couvre les opérations quotidiennes et les fonctionnalités d'OpenClaw.

## Concepts de base

### Gateway

Gateway est le cœur d'OpenClaw. Tout le routage des messages, les requêtes IA et la gestion des canaux passent par ce seul processus. Vous n'avez besoin d'exécuter qu'un seul Gateway et il gère automatiquement les connexions à toutes vos plateformes de chat.

### Canaux

Les canaux sont les applications de messagerie que vous utilisez pour communiquer avec votre assistant IA. OpenClaw permet de connecter plusieurs canaux simultanément, y compris WhatsApp, Telegram, Discord, iMessage et plus encore.

### Agents

Les agents sont des instances indépendantes d'assistants IA. Chaque agent a son propre espace de travail et historique de conversation. Vous pouvez exécuter plusieurs agents pour différents usages.

## Opérations quotidiennes

### Démarrer et arrêter

```bash
# Démarrer (s'exécute en arrière-plan)
openclaw gateway start

# Arrêter
openclaw gateway stop

# Redémarrer
openclaw gateway restart

# Vérifier le statut
openclaw gateway status

# Exécuter au premier plan (pour le débogage)
openclaw gateway --port 18789
```

### Envoyer des messages

```bash
# Envoyer un message direct
openclaw message send --target +15555550123 --message "Message de test"

# Envoyer à un groupe
openclaw message send --target "Nom du Groupe" --message "Bonjour à tous"
```

### Gérer la configuration

OpenClaw utilise des fichiers de configuration au format JSON5, situés dans `~/.openclaw/openclaw.json`.

Façons de modifier la configuration:

1. **Mode assistant**:
   ```bash
   openclaw onboard
   openclaw configure
   ```

2. **Ligne de commande**:
   ```bash
   openclaw config get agents.defaults.workspace
   openclaw config set agents.defaults.heartbeat.every "2h"
   openclaw config unset tools.web.search.apiKey
   ```

3. **Interface web**:
   Ouvrez http://127.0.0.1:18789/, allez dans l'onglet Config

4. **Édition directe**:
   ```bash
   nano ~/.openclaw/openclaw.json
   ```

### Rechargement à chaud

La plupart des modifications de configuration sont appliquées automatiquement sans redémarrage.

## Exemples de configuration courants

### Restreindre l'accès

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

### Configuration des groupes

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

### Multiples agents

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

### Tâches planifiées

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Vérifications périodiques

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Isolation sandbox

Pour exécuter des opérations IA dans des conteneurs isolés:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

```json5
{
  hooks: {
    enabled: true,
    token: "votre-jeton-secret",
    path: "/hooks"
  }
}
```

## Gestion des canaux

### Ajouter des canaux

```bash
# Se connecter à WhatsApp
openclaw channels login whatsapp

# Se connecter à Telegram
openclaw channels login telegram
```

### Lister les canaux connectés

```bash
openclaw channels list
```

## Dépannage

### Voir les logs

```bash
# Logs du service
openclaw logs

# Suivre les logs en temps réel
openclaw logs -f
```

### Vérification de santé

```bash
openclaw health
```

### Diagnostic automatique

```bash
openclaw doctor
```

## Console web

Vous pouvez également gérer OpenClaw via l'interface web:

- Envoyer et recevoir des messages
- Modifier les paramètres
- Voir l'historique des conversations
- Gérer les nœuds mobiles
- Surveiller l'état du service

Visitez: http://127.0.0.1:18789/
