# Руководство по установке Node.js

## Официальные методы установки

Node.js предоставляет несколько официальных методов установки для различных операционных систем и предпочтений пользователей. Все официальные методы гарантируют, что вы получите аутентичные, неизмененные двоичные файлы непосредственно от проекта Node.js.

### Скачивание с официального сайта

Самый простой способ установить Node.js — скачать соответствующий установщик с [официального сайта Node.js](https://nodejs.org/en/download).

**Доступные версии:**
- **LTS (Поддержка долгосрочная)**: Рекомендуется для большинства пользователей. Обеспечивает стабильность и долгосрочную поддержку.
- **Current**: Последние функции с более коротким циклом поддержки. Подходит для разработки и тестирования.

### Менеджеры пакетов

#### macOS (Homebrew)
```bash
# Установить Homebrew, если еще не установлен
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установить LTS-версию
brew install node@lts

# Или установить последнюю версию Current
brew install node
```

#### Linux (Различные дистрибутивы)

**Ubuntu/Debian:**
```bash
# Использование репозитория NodeSource (рекомендуется)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora:**
```bash
# Использование репозитория NodeSource
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# или для более новых версий: sudo dnf install nodejs
```

#### Windows
- Скачать установщик Windows (.msi) с официального сайта
- Запустить установщик и следовать мастеру настройки
- Node.js и npm автоматически добавятся в ваш PATH

### Проверка

После установки проверьте, что Node.js и npm правильно установлены:

```bash
# Проверить версию Node.js
node --version

# Проверить версию npm  
npm --version

# Протестировать базовую функциональность
node -e "console.log('Привет, Node.js!')"
```

### Управление несколькими версиями Node.js

Для разработчиков, которым нужно работать с разными версиями Node.js в разных проектах, менеджеры версий предоставляют отличное решение.

#### nvm (Node Version Manager) - macOS/Linux
```bash
# Установить nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Установить и использовать LTS-версию
nvm install --lts
nvm use --lts

# Установить конкретную версию
nvm install 20.14.0
nvm use 20.14.0

# Список доступных версий
nvm list-remote
```

#### nvm-windows - Windows
- Скачать с [nvm-windows GitHub Releases](https://github.com/coreybutler/nvm-windows/releases)
- Следовать инструкциям по установке
- Использовать аналогичные команды как Unix-базированный nvm

### Переменные окружения

Убедитесь, что исполняемые файлы Node.js и npm были добавлены в системную переменную окружения PATH.

- **Windows**: Обычно автоматически добавляется в PATH
- **macOS/Linux**: Если вы используете менеджер пакетов, обычно автоматически настраивается

Если команда не найдена, добавьте ее вручную в PATH:
```bash
# Обычно расположено в /usr/local/bin или ~/.nvm/versions/node/vXX.XX.XX/bin
export PATH=$PATH:/usr/local/bin
```

### Устранение неполадок

#### Проблемы с правами
В Linux/macOS, если вы сталкиваетесь с проблемами прав при глобальной установке пакетов npm, рекомендуем:
- Использовать nvm для управления Node.js
- Или настроить глобальный каталог npm:
  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  export PATH=~/.npm-global/bin:$PATH
  ```

#### Проблемы с сетью
Если скорость загрузки медленная или невозможно подключиться, вы можете настроить npm для использования локального зеркала:
```bash
# Настроить зеркало Taobao (Китай)
npm config set registry https://registry.npmmirror.com

# Или использовать временно
npm install --registry https://registry.npmmirror.com
```
