# Введение в Node.js

Node.js® — это бесплатная, с открытым исходным кодом, кроссплатформенная среда выполнения JavaScript, которая позволяет разработчикам создавать серверы, веб-приложения, инструменты командной строки и скрипты.

## Основные возможности

Как асинхронная среда выполнения JavaScript, ориентированная на события, Node.js предназначена для создания масштабируемых сетевых приложений. В классическом примере "hello world" можно одновременно обрабатывать множество подключений. При каждом подключении вызывается обратный вызов (callback), но если нет работы для выполнения, Node.js переходит в спящий режим.

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

## Философия проектирования

Это контрастирует с более распространенной сегодня моделью конкурентности, в которой используются потоки операционной системы. Сетевое программирование на основе потоков относительно неэффективно и очень сложно в использовании. Кроме того, пользователи Node.js избавлены от забот о взаимоблокировках процесса, поскольку блокировок нет.

Почти ни одна функция в Node.js напрямую не выполняет операции ввода-вывода, поэтому процесс никогда не блокируется, за исключением случаев использования синхронных методов стандартной библиотеки Node.js. Поскольку ничто не блокируется, разработка масштабируемых систем в Node.js является вполне разумной.

## Архитектура, ориентированная на события

Node.js похож по конструкции на такие системы, как Event Machine Ruby и Twisted Python, и был под влиянием них. Node.js продвинул событийную модель еще дальше. Он представляет цикл событий как конструкцию времени выполнения, а не как библиотеку.

В других системах всегда есть блокирующий вызов для запуска цикла событий. Обычно поведение определяется через обратные вызовы в начале скрипта, а в конце сервер запускается через блокирующий вызов, такой как `EventMachine::run()`.

В Node.js нет такого вызова для запуска цикла событий. Node.js просто входит в цикл событий после выполнения входного скрипта. Node.js выходит из цикла событий, когда больше нет обратных вызовов для выполнения. Это поведение похоже на JavaScript в браузере — цикл событий скрыт от пользователя.

## Поддержка HTTP

HTTP является первоклассным гражданином в Node.js, разработан с учетом потоковой передачи и низкой задержки. Это делает Node.js отлично подходящим в качестве основы для веб-библиотеки или фреймворка.

## Поддержка многоядерных процессоров

То, что Node.js разработан без потоков, не означает, что вы не можете использовать преимущества нескольких ядер в вашей среде. Дочерние процессы могут быть созданы с помощью нашего API `child_process.fork()`, и они предназначены для легкого взаимодействия.

На той же самой интерфейсе построен модуль `cluster`, который позволяет вам делить сокеты между процессами для обеспечения балансировки нагрузки на ваши ядра.
