# Руководство по установке JMeter

## Требования

- **Java Development Kit (JDK)**: JMeter требует Java 8 или выше
- **Операционная система**: Windows, Linux или macOS
- **Оперативная память**: Не менее 4 ГБ ОЗУ рекомендуется для нагрузочного тестирования

## Шаги установки

### 1. Скачайте JMeter

Посетите официальный веб-сайт Apache JMeter: https://jmeter.apache.org/

Скачайте последнюю версию (обычно файл .zip или .tar.gz).

### 2. Извлеките архив

Извлеките скачанный файл в нужное место:

```bash
# Linux/Mac
tar -xvf apache-jmeter-x.x.x.tgz

# Windows
# Используйте 7-Zip или WinRAR для извлечения
```

### 3. Настройте переменные среды (дополнительно)

Для удобства можно добавить JMeter в системный PATH:

```bash
# Linux/Mac (добавить в ~/.bashrc или ~/.zshrc)
export JMETER_HOME=/path/to/apache-jmeter-x.x.x
export PATH=$PATH:$JMETER_HOME/bin

# Windows
# Добавить в Свойства системы -> Переменные среды -> PATH
```

### 4. Проверьте установку

Выполните следующую команду, чтобы убедиться, что JMeter установлен правильно:

```bash
# Linux/Mac
./jmeter.sh --version

# Windows
jmeter.bat --version
```

### 5. Запустите GUI JMeter

```bash
# Linux/Mac
./jmeter.sh

# Windows
jmeter.bat
```

## Структура каталогов

```
apache-jmeter/
├── bin/          # Исполняемые скрипты и исполняемый файл JMeter
├── lib/          # Библиотеки JMeter
├── lib/ext/      # Плагины и расширения
├── docs/         # Документация
├── printable_docs/  # Руководство пользователя
└── LICENSE       # Файл лицензии
```

## Запуск в разных режимах

### Режим GUI (интерактивный)
```bash
jmeter.sh          # Linux/Mac
jmeter.bat         # Windows
```

### Режим CLI (без графического интерфейса)
```bash
jmeter -n -t testplan.jmx -l results.jtl -e -o html-report
```

Параметры:
- `-n`: Запуск в режиме без GUI
- `-t`: Файл тестового плана
- `-l`: Файл результатов
- `-e`: Генерация HTML-отчёта после теста
- `-o`: Каталог вывода для HTML-отчёта

## Устранение проблем установки

### Java не найдена
Убедитесь, что JAVA_HOME настроен правильно:
```bash
export JAVA_HOME=/path/to/jdk
```

### Проблемы с памятью
Отредактируйте скрипт запуска jmeter, чтобы увеличить размер кучи:
```bash
HEAP="-Xms1g -Xmx4g"
```
