# Foire Aux Questions

Ce document recense les questions techniques fréquentes et leurs réponses lors de l'utilisation d'Angular.

## Questions de base

### Qu'est-ce qu'Angular ?

Angular est un framework open-source développé et maintenu par Google, conçu pour créer des applications web modernes. Il utilise TypeScript comme langage principal et offre des fonctionnalités comme l'architecture par composants, la liaison bidirectionnelle des données et l'injection de dépendances.

### Quelle est la différence entre Angular et AngularJS ?

AngularJS est le nom d'Angular 1.x, le précurseur d'Angular. Angular est une version complètement réécrite, utilisant TypeScript et une architecture totalement différente. Elle n'est pas compatible avec AngularJS.

### Angular est-il difficile à apprendre ?

Angular possède une courbe d'apprentissage prononcée, notamment pour des concepts comme l'injection de dépendances et RxJS qui nécessitent du temps pour être maîtrisés. Cependant, sa documentation complète et sa communauté active rendent le développement très efficace une fois familiarisé avec le framework.

## Questions d'installation

### npm install échoue, que faire ?

1. Vider le cache npm : `npm cache clean --force`
2. Supprimer node_modules et package-lock.json
3. Utiliser un miroir chinois
4. Vérifier que la version de Node.js est compatible

### La commande Angular CLI n'est pas trouvée ?

Assurez-vous d'avoir installé le package global :
```bash
npm install -g @angular/cli
```

Si cela ne fonctionne toujours pas, essayez avec npx :
```bash
npx @angular/cli new my-app
```

### Version de Node.js incompatible ?

Utilisez nvm pour gérer les versions de Node.js :
```bash
nvm install 18
nvm use 18
```

## Questions de développement

### Les styles du composant ne s'appliquent pas ?

1. Vérifiez que le chemin du fichier de styles est correct
2. Assurez-vous qu'il n'y a pas d'erreurs de syntaxe dans styles.scss
3. Vérifiez la configuration ViewEncapsulation

### La navigation échoue ?

1. Vérifiez que la configuration des routes est correcte
2. Confirmez que RouterModule est bien importé
3. Vérifiez si un gardien de route bloque la navigation

### Les requêtes HTTP échouent ?

1. Vérifiez que le service backend est démarré
2. Confirmez la configuration CORS
3. Utilisez un HttpInterceptor pour ajouter la gestion des erreurs

### Les données ne se mettent pas à jour ?

1. Vérifiez la stratégie de détection des changements
2. Si vous utilisez OnPush, déclenchez la détection manuellement
3. Assurez-vous que les références des données changent

## Questions de performances

### Le chargement initial est lent ?

1. Activez le chargement paresseux des routes
2. Utilisez une stratégie de préchargement
3. Optimisez les ressources images
4. Activez le mode production pour le build

### Fuite de mémoire ?

1. Désabonnez-vous des Observables dans ngOnDestroy
2. Utilisez takeUntil ou le pipe async
3. Nettoyez les intervalles et les écouteurs d'événements

### Détection des changements trop fréquente ?

1. Utilisez la stratégie OnPush
2. Séparez les propriétés qui changent de celles qui ne changent pas
3. Utilisez trackBy pour optimiser *ngFor

## Questions de tests

### Les tests unitaires échouent ?

1. Vérifiez la configuration TestBed
2. Confirmez que les opérations asynchrones sont correctement gérées
3. Utilisez fakeAsync et tick pour tester le code asynchrone

### Module introuvable ?

Assurez-vous que les paths sont correctement configurés dans tsconfig.json.

## Questions de build

### Le build échoue ?

1. Consultez le message d'erreur spécifique
2. Vérifiez les options de compilation TypeScript
3. Videz le cache et reconstruisez

### Les fichiers de build sont trop volumineux ?

1. Activez le mode production
2. Utilisez le chargement parlessein
3. Vérifiez les dépendances non utilisées

## Erreurs courantes

### ExpressionChangedAfterItHasBeenCheckedError

Une valeur qui ne devrait pas changer pendant la détection a été modifiée. Vérifiez si vous modifiez directement dans ngOnInit des données liées à la vue.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

Même erreur que ci-dessus, vérifiez le séquençage des liaisons de données.

### Cannot read property of undefined

Accès à des données non initialisées. Utilisez l'opérateur optionnel (?.) ou *ngIf pour protéger.

### Erreur CORS

Le backend n'a pas configuré CORS, ou les en-têtes de requête frontend ont un problème.

### Erreur 401/403

Échec de l'authentification, vérifiez le token et la configuration des permissions.

## Mise à jour de version

### Comment mettre à jour Angular ?

1. Utilisez CLI pour mettre à jour : `ng update @angular/core`
2. Consultez le guide de mise à niveau officiel
3. Mettez à jour les dépendances une par une
4. Exécutez les tests pour vérifier que tout fonctionne

### La mise à jour échoue, que faire ?

1. Sauvegardez votre code
2. Mettez à jour manuellement les versions dans package.json
3. Supprimez node_modules et réinstallez
