# Mockito Preguntas Frecuentes

Preguntas frecuentes sobre Mockito.

## Preguntas Generales

### ¿Qué es Mockito?

Mockito es un framework de mocking de código abierto para Java. Permite crear y configurar objetos mock para pruebas unitarias.

### ¿Por qué usar Mockito?

- Pruebas aisladas sin dependencias externas
- Ejecución más rápida de pruebas
- API simple e intuitiva
- Bien integrado con JUnit

### ¿Cuál es la diferencia entre Mock y Spy?

- **Mock** - Un objeto completamente artificial que usted controla completamente
- **Spy** - Un objeto parcialmente real donde solo anula métodos específicos

## Preguntas Técnicas

### ¿Cómo hacer mock de una clase final?

Use `mockito-inline`:

```groovy
testImplementation 'org.mockito:mockito-inline:5.8.0'
```

### ¿Cómo hacer mock de métodos estáticos?

Con Mockito 3.7+, puede hacer mock de métodos estáticos:

```java
try (MockedStatic<Utilities> mocked = Mockito.mockStatic(Utilities.class)) {
    mocked.when(Utilities::getName).thenReturn("mocked");
    // Su código de prueba
}
```

## Solución de Problemas

### Error: "Cannot mock/spy class"

**Causa:** La clase es final o tiene métodos finales.

**Solución:**
- Use `mockito-inline`
- Haga mock de una interfaz en lugar de una clase

### Advertencia: "Unnecessary stubbing"

**Causa:** No se usó un stub.

**Solución:**
- Elimine el stub no usado
- O use `lenient()`:

```java
lenient().when(mock.method()).thenReturn("value");
```

## Integración

### Con JUnit 4

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private List mockList;
}
```

### Con JUnit 5

```java
@ExtendWith(MockitoExtension.class)
public class MyTest {
    @Mock
    private List mockList;
}
```
