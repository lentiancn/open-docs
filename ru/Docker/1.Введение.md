# Введение в Docker

Docker — это открытая платформа для разработки, доставки и запуска приложений. Docker позволяет отделить ваши приложения от инфраструктуры, чтобы вы могли быстро поставлять программное обеспечение. С Docker вы можете управлять инфраструктурой так же, как управляете приложениями.

## Что такое Docker

Docker — это платформа контейнеризации с открытым исходным кодом, которая помогает разработчикам упаковывать приложения и их зависимости в лёгкие переносимые контейнеры. Эти контейнеры могут работать в любой среде, поддерживающей Docker, обеспечивая принцип «создай один раз, запускай везде».

Docker решает проблему «работает на моей машине». Команды разработчиков могут использовать Docker для устранения различий в средах, обеспечивая согласованную работу приложений в различных средах разработки, тестирования и production.

## Основные концепции Docker

### Контейнеры

Контейнеры — одна из основных концепций Docker. Это лёгкие исполняемые пакеты, которые содержат всё необходимое для запуска приложения: код, среду выполнения, системные инструменты, системные библиотеки и настройки. Контейнеры легче виртуальных машин, поскольку они используют ядро операционной системы хоста без дополнительных затрат на отдельные операционные системы.

### Образы

Образы — это шаблоны для контейнеров. Это шаблоны только для чтения, используемые для создания контейнеров. Образы могут содержать операционные системы, приложения и все зависимости. Образы Docker используют многоуровневую структуру хранения, что упрощает создание, совместное использование и контроль версий образов.

### Dockerfile

Dockerfile — это текстовый файл с инструкциями для сборки образов. Через Dockerfile вы можете определить содержимое, конфигурацию и команды запуска образа. Dockerfile очень читаемы, что делает процесс создания образа прозрачным и облегчает командное сотрудничество и автоматические сборки.

### Docker Hub

Docker Hub — это официальный репозиторий образов Docker. Здесь вы можете найти десятки тысяч готовых образов, включая различные операционные системы, среды выполнения языков программирования, базы данных и инструменты разработки. Вы также можете отправлять свои образы в Docker Hub для совместного использования с другими.

## Основные применения Docker

### Контейнеризация приложений

Наиболее распространённое применение Docker — контейнеризация приложений. Упаковывая приложения и все их зависимости в контейнеры, вы можете обеспечить согласованную работу приложений в любой среде, будь то машина разработчика, тестовый сервер или production-среда.

### Архитектура микросервисов

Docker идеально подходит для построения архитектур микросервисов. Каждый микросервис может быть упакован в независимый контейнер для независимого запуска и масштабирования, что делает обслуживание, обновление и масштабирование системы гораздо более гибкими.

### Непрерывная интеграция и непрерывное развёртывание (CI/CD)

Docker играет важную роль в конвейерах CI/CD. Контейнеры можно быстро создавать и уничтожать в конвейерах, обеспечивая чистые и согласованные среды для тестирования. Docker делает весь процесс от коммита кода до развёртывания в production более автоматизированным и надёжным.

### Стандартизация сред разработки

Команды могут использовать Docker для стандартизации сред разработки. Новые разработчики могут получить полностью согласованную среду разработки, выполнив всего несколько команд, без ручной установки различных зависимостей и настроек.

## Экосистема Docker

Docker — это не только среда выполнения контейнеров, но и полная экосистема:

- **Docker Engine**: основная среда выполнения контейнеров
- **Docker Desktop**: приложение для рабочего стола для Mac, Windows и Linux
- **Docker Compose**: инструмент для определения и запуска многоконтейнерных приложений
- **Docker Swarm**: инструмент оркестрации контейнеров
- **Docker Hub**: сервис репозитория образов
- **Docker Build**: инструмент сборки образов

## Обзор Docker Desktop

Docker Desktop — это приложение с установкой в один клик для Mac, Linux или Windows, которое позволяет создавать, распространять и запускать контейнеризированные приложения и микросервисы. Оно предоставляет простой графический интерфейс для управления контейнерами, приложениями и образами непосредственно с вашего компьютера. Docker Desktop сокращает время на сложные настройки, позволяя вам сосредоточиться на написании кода.

Ключевые функции Docker Desktop:

- Возможность контейнеризации и распространения любого приложения на любой облачной платформе на нескольких языках и фреймворках
- Быстрая установка и настройка полной среды разработки Docker
- Включает последнюю версию Kubernetes
- В Windows возможность переключения между Linux и Windows контейнерами для сборки приложений
- Быстрая и надёжная производительность с нативной виртуализацией Windows Hyper-V
- Возможность нативной работы на Linux через WSL 2 на компьютерах Windows
- Монтирование томов для кода и данных, включая уведомления об изменениях файлов и лёгкий доступ к запущенным контейнерам в сети localhost

## Преимущества использования Docker

### Согласованность сред

Контейнеры Docker обеспечивают согласованную работу приложений в различных средах. Среды разработки, тестирования и production могут использовать один и тот же образ, устраняя проблему «работает на моей машине».

### Изоляция ресурсов

Каждый контейнер работает изолированно. Проблемы в одном контейнере не влияют на другие контейнеры. Эта изоляция повышает безопасность и стабильность приложений.

### Быстрое развёртывание

Контейнеры Docker могут запускаться за секунды, что намного быстрее традиционных виртуальных машин, на запуск которых уходят минуты. Эта возможность быстрого запуска значительно повышает эффективность разработки и развёртывания.

### Эластичное масштабирование

Контейнеры Docker можно быстро масштабировать в зависимости от нагрузки. В сочетании с инструментами оркестрации, такими как Kubernetes, приложения могут масштабироваться автоматически.

### Контроль версий

Образы Docker поддерживают контроль версий. Вы можете легко откатиться к предыдущим версиям. Многоуровневая структура образов также делает хранение различий версий более эффективным.

## Лицензирование Docker

Коммерческое использование Docker Desktop в крупных компаниях (более 250 сотрудников ИЛИ годовой доход более 10 миллионов долларов США) требует платной подписки. Docker Engine имеет открытый исходный код и использует лицензию Apache 2.0.
