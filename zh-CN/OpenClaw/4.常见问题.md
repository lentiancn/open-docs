# OpenClaw 常见问题

本页面解答关于 OpenClaw 的常见问题。

## 基础问题

### 什么是 OpenClaw？

OpenClaw 是一个自托管网关，将你的聊天应用（WhatsApp、Telegram、Discord 等）连接到 AI 编码助手。它让你可以从任何地方通过消息应用与 AI 助手交流，同时完全掌控自己的数据。

### OpenClaw 免费吗？

是的，OpenClaw 是开源项目，采用 MIT 许可证。但你需要自行承担：
- 运行所需的服务器或电脑
- AI 服务的 API 密钥费用

### 我需要 AI 服务商订阅吗？

OpenClaw 支持多种模型提供商，包括：
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- 以及其他兼容 OpenAI API 的服务商

你可以选择按量付费的 API 密钥，或者使用订阅认证。

### OpenClaw 支持哪些平台？

- **桌面**：macOS、Linux、Windows
- **移动端**：iOS、Android（作为节点）
- **聊天应用**：WhatsApp、Telegram、Discord、iMessage、Signal、Slack 等 20+ 个平台

## 安装问题

### 安装失败怎么办？

1. 检查 Node.js 版本：`node --version`（需要 22 或更高）
2. 运行诊断：`openclaw doctor`
3. 查看日志：`openclaw logs`

### 可以在树莓派上运行吗？

可以，OpenClaw 可以在树莓派上运行。建议使用树莓派 4 或更新型号，并使用 Raspberry Pi OS 或 Ubuntu。

### 需要买 Mac 电脑吗？

不需要。OpenClaw 可以在任何支持 Node.js 的系统上运行。只有 iMessage 通道需要 Mac 电脑（因为 Apple 的限制）。

## 配置问题

### 配置文件在哪里？

默认位置：`~/.openclaw/openclaw.json`

### 如何限制谁可以发消息？

通过 `dmPolicy` 配置：

- `pairing`：未知发送者需要配对码
- `allowlist`：仅允许列表中的用户
- `open`：允许所有私信
- `disabled`：禁用私信

### 如何设置群组聊天？

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## 使用问题

### 如何添加新的聊天通道？

```bash
openclaw channels login whatsapp
```

### 可以同时使用多个通道吗？

可以，一个 Gateway 可以同时连接多个通道。

### 如何让 AI 记住之前的对话？

OpenClaw 内置记忆功能，会自动保存会话上下文。你可以通过配置调整会话保留策略。

### 可以运行多个代理吗？

可以，配置多个代理并设置路由规则：

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## 安全问题

### 我的数据安全吗？

是的。OpenClaw 是自托管的，所有数据都存储在你的设备上，不会发送到第三方服务器（除了 AI 服务商的 API）。

### 如何保护我的 Gateway？

1. 设置 `allowFrom` 限制访问来源
2. 使用强密码和 API 密钥
3. 定期更新 OpenClaw 版本
4. 使用防火墙限制端口访问

## 故障排查

### Gateway 启动失败

1. 检查端口是否被占用：`openclaw gateway status`
2. 查看错误日志：`openclaw logs`
3. 运行诊断：`openclaw doctor`

### AI 不回复消息

1. 检查 API 密钥是否正确配置
2. 确认模型服务商账户有足够余额
3. 查看日志中的错误信息

### 消息发送失败

1. 检查通道是否已登录
2. 确认目标用户或群组是否存在
3. 查看详细的错误日志
