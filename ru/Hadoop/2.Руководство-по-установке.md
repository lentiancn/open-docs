# Руководство по установке Hadoop

## Требования

Перед установкой Hadoop убедитесь, что ваша система соответствует базовым требованиям к аппаратному и программному обеспечению. Кластеры Hadoop могут состоять из нескольких компьютеров или работать в псевдораспределённом режиме. Разные режимы развёртывания имеют разные требования к оборудованию, но основные программные зависимости схожи. Понимание этих требований помогает планировать масштаб и конфигурацию кластера Hadoop, избегая ненужных проблем во время установки.

Для оборудования небольшие тестовые кластеры должны иметь как минимум 4 ядра ЦП, 8 ГБ ОЗУ и 500 ГБ жёсткого диска на машину. Для производственных сред каждая машина должна иметь 8 или более ядер ЦП, 16 ГБ или более ОЗУ и жёсткий диск 1 ТБ или больше. DataNode Hadoop хранят большие объёмы данных, поэтому ёмкость диска особенно важна. Hadoop рекомендует конфигурацию RAID или JBOD (Just a Bunch Of Disks) — последняя позволяет Hadoop более гибко управлять дисковым пространством.

Для операционных систем Hadoop поддерживает Linux (Ubuntu, CentOS, Debian, Red Hat и т.д.) и Windows (через Cygwin или Windows Subsystem for Linux). Для производственных сред настоятельно рекомендуется использовать Linux для достижения наилушей производительности и стабильности. Hadoop также хорошо интегрируется с различными облачными платформами и может быть развёрнут на AWS, Azure, Google Cloud и других облачных сервисах.

Java — единственная зависимость Hadoop во время выполнения. Hadoop 3.x требует Java 8 или выше. Рекомендуется Oracle JDK или OpenJDK. Перед установкой Hadoop убедитесь, что Java установлена правильно и переменная среды JAVA_HOME настроена. Проверьте установку Java:

```bash
java -version
echo $JAVA_HOME
```

Сеть — основа нормальной работы кластера Hadoop. Все узлы в кластере должны иметь возможность связываться друг с другом по сети, предпочтительно с использованием высокоскоростных сетей (например, гигабитного Ethernet). На узлах необходимо настроить SSH без пароля, чтобы Hadoop мог распределять задачи и передавать данные между узлами. Также настройте сопоставления имён хостов и IP-адресов для всех узлов в /etc/hosts.

## Шаги установки

### Установка в автономном режиме

Автономный режим — самый простой способ установки Hadoop, при котором все компоненты работают в одном JVM-процессе. Этот режим используется в основном для разработки и тестирования и не предполагает распределённого хранения или вычислений. Сначала загрузите бинарный дистрибутив Hadoop с официального сайта Apache Hadoop:

```bash
# Загрузить Hadoop (замените на последнюю версию)
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# Распаковать в указанный каталог
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# Переименовать каталог (необязательно)
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

Затем настройте переменные среды. Отредактируйте файл ~/.bashrc или /etc/profile и добавьте следующее:

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # Настройте в соответствии с реальным путём Java
```

Примените конфигурацию:

```bash
source ~/.bashrc
```

Проверьте установку:

```bash
hadoop version
```

### Установка в псевдораспределённом режиме

Псевдораспределённый режим имитирует распределённую среду на одной машине, где NameNode, DataNode, ResourceManager и NodeManager работают на одном компьютере. Этот режим подходит для разработки и тестирования распределённых функций.

Сначала настройте основные файлы Hadoop. Перейдите в каталог конфигурации Hadoop:

```bash
cd $HADOOP_HOME/etc/hadoop
```

Настройте core-site.xml, указав HDFS в качестве файловой системы по умолчанию:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Настройте hdfs-site.xml, указав коэффициент репликации HDFS (1 для псевдораспределённого режима):

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

Настройте mapred-site.xml, указав фреймворк планирования ресурсов для MapReduce:

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

Настройте yarn-site.xml, настроив свойства, связанные с YARN:

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### Настройка SSH без пароля

Узлы в кластере Hadoop должны иметь настроенный SSH без пароля, чтобы главный узел мог обращаться к подчинённым узлам без пароля:

```bash
# Сгенерировать пару ключей SSH
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# Добавить открытый ключ в authorized_keys
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# Установить правильные разрешения
chmod 0600 ~/.ssh/authorized_keys

# Проверить SSH-вход
ssh localhost
```

Для многузловых кластеров скопируйте открытый ключ на все узлы:

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### Форматирование и запуск HDFS

Перед первым использованием HDFS отформатируйте NameNode:

```bash
hdfs namenode -format
```

Запустите HDFS и YARN:

```bash
start-dfs.sh
start-yarn.sh
```

Или используйте start-all.sh для одновременного запуска всех сервисов:

```bash
start-all.sh
```

Проверьте состояние запуска сервисов:

```bash
# Просмотр Java-процессов
jps

# Ожидаемые процессы:
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Вы можете получить доступ к сервисам Hadoop через веб-интерфейс: NameNode по адресу http://localhost:9870, ResourceManager по адресу http://localhost:8088.

## Установка полностью распределённого кластера

### Планирование кластера

Полностью распределённый режим требует нескольких физических или виртуальных машин для формирования кластера. Перед установкой проведите разумное планирование кластера. Определите масштаб кластера (количество узлов) и назначьте роли каждому узлу (NameNode, Secondary NameNode, DataNode, ResourceManager, NodeManager). Для небольших кластеров NameNode и ResourceManager могут быть развёрнуты на одной машине; для больших кластеров рекомендуется раздельное развёртывание для лучшей производительности.

Планирование сети также важно. Все узлы должны находиться в одной внутренней сети с низкой задержкой и высокой пропускной способностью. Настройте имена хостов и IP-адреса для всех узлов в /etc/hosts:

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### Изменение конфигурационных файлов

В полностью распределённом режиме нужно изменить больше конфигурационных файлов. Сначала настройте файл workers (файл slaves в более ранних версиях), перечислив все узлы DataNode и NodeManager:

```
slave1
slave2
slave3
```

Конфигурация core-site.xml и hdfs-site.xml аналогична псевдораспределённому режиму, но нужно добавить больше свойств. Для конфигурации высокой доступности NameNode настройте два NameNode:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Настройте hdfs-site.xml для высокой доступности NameNode:

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

Конфигурация YARN аналогична; можно настроить высокую доступность ResourceManager в yarn-site.xml.

### Синхронизация конфигурационных файлов

Синхронизируйте настроенный каталог установки Hadoop со всеми узлами. Копируйте индивидуально с помощью scp:

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

Или используйте rsync для распределённой репликации:

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### Запуск кластера

Отформатируйте NameNode на узле NameNode (при первом запуске):

```bash
hdfs namenode -format
```

Запустите HDFS:

```bash
start-dfs.sh
```

Запустите YARN:

```bash
start-yarn.sh
```

Проверьте состояние процессов на всех узлах, чтобы убедиться, что сервисы работают правильно:

```bash
# На NameNode
jps

# На DataNode
jps
```

## Проверка и тестирование

### Тестирование HDFS

Создайте каталоги и загрузите файлы для тестирования HDFS:

```bash
# Создать каталог
hdfs dfs -mkdir -p /user/test

# Загрузить локальный файл в HDFS
hdfs dfs -put localfile.txt /user/test/

# Список файлов
hdfs dfs -ls /user/test

# Прочитать файл
hdfs dfs -cat /user/test/localfile.txt

# Удалить файл
hdfs dfs -rm /user/test/localfile.txt
```

### Тестирование MapReduce

Запустите примеры программ MapReduce для проверки функциональности кластера:

```bash
# Запустить пример WordCount
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# Просмотреть результаты
hdfs dfs -cat /output/part-r-00000
```

### Проверка работоспособности кластера

Используйте встроенные команды Hadoop для проверки состояния кластера:

```bash
# Просмотр состояния HDFS
hdfs dfsadmin -report

# Просмотр состояния узлов YARN
yarn node -list -all

# Просмотр состояния приложений
yarn application -list
```

## Распространённые проблемы

### Несовместимость версии Java

При возникновении ошибок, связанных с версией Java, убедитесь, что установленная версия Java совместима с версией Hadoop. Hadoop 3.x требует Java 8 или выше. Измените версию Java:

```bash
update-alternatives --config java
```

### Недостаточно дискового пространства

DataNode могут работать неправильно из-за недостаточного дискового пространства. Регулярно контролируйте использование диска, чтобы убедиться, что у DataNode достаточно свободного пространства. Настройте dfs.datanode.fsdataset.volume.choosing.policy в hdfs-site.xml как AvailableSpaceVolumeChoosingPolicy, чтобы DataNode предпочитали диски с большим свободным пространством.

### Недостаточно памяти

Если узлы кластера имеют недостаточно памяти, YARN может не запускать контейнеры. Настройте конфигурацию памяти в yarn-site.xml:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
