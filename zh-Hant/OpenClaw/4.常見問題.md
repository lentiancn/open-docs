# OpenClaw 常見問題

本頁面彙總了使用 OpenClaw 過程中的常見問題及解決方案。

## 安裝相關

### OpenClaw 需要什麼環境？

- Node.js 22.x 或更高版本
- macOS、Linux 或 Windows (WSL2)
- 建議至少 2GB 記憶體

### 安裝失敗怎麼辦？

1. 確認 Node.js 版本：`node --version`（需 >= 22）
2. 檢查網路連線
3. 嘗試使用不同的套件管理工具：npm、pnpm 或 bun
4. 執行 `openclaw doctor` 診斷問題

### 如何更新 OpenClaw？

```bash
# 切換到穩定版
openclaw update --channel stable

# 然後重啟 Gateway
openclaw gateway restart
```

## Gateway 相關

### Gateway 無法啟動

- 檢查連接埠 18789 是否被佔用
- 查看日誌：`openclaw gateway --verbose`
- 嘗試終止可能存在的程序

### 如何讓 Gateway 始終運行？

使用 `--install-daemon` 安裝為系統服務：

```bash
openclaw onboard --install-daemon
```

### 如何遠端存取 Gateway？

1. **Tailscale**：使用 `gateway.tailscale.mode: "serve"` 或 `"funnel"`
2. **SSH 隧道**：建立 SSH 隧道連線

## 渠道相關

### WhatsApp 登入失敗

- 確認已使用 `openclaw channels login` 配對裝置
- 檢查網路是否可以存取 WhatsApp 伺服器

### Telegram Bot 無法使用

- 確認 Bot Token 正確
- 檢查是否已與 Bot 開始對話
- 驗證 `allowFrom` 配置

### Discord 訊息沒有反應

- 檢查 Bot 是否已加入伺服器
- 確認已配置正確的權限
- 查看頻道是否在允許清單中

## 安全相關

### 陌生人可以給我發訊息？

預設情況下，OpenClaw 會對未知傳送者進行配對驗證。使用 `dmPolicy: "open"` 可改為開放模式，但不推薦。

### 如何管理配對？

```bash
# 查看配對清單
openclaw pairing list

# 批准配對
openclaw pairing approve

# 撤銷配對
openclaw pairing revoke
```

### 執行 `openclaw doctor` 顯示警告？

按照指示修復配置。常見問題包括：
- DM 政策過於開放
- 缺少必要的驗證配置
- 連接埠暴露風險

## 技能相關

### 如何安裝新技能？

```bash
# 透過 ClawHub 安裝
openclaw skill install 技能名稱

# 或手動建立技能目錄和 SKILL.md
```

### 技能不工作怎麼辦？

- 檢查技能配置是否正確
- 查看 Gateway 日誌
- 確認技能依賴已安裝

## 語音相關

### Voice Wake 不工作？

- macOS：檢查系統偏好設定中的輔助功能權限
- iOS：確認麥克風權限已開啟
- 檢查喚醒詞是否正確設定

### 語音回應太慢？

- 檢查網路延遲
- 考慮使用本地 TTS
- 調整語音模型設定

## 效能相關

### OpenClaw 佔用太多記憶體？

- 定期清理會話歷史
- 調整模型設定
- 使用會話壓縮功能

### 訊息回應慢？

- 檢查網路連線
- 嘗試切換到更快的模型
- 查看 Gateway 負載

## 資料相關

### 資料儲存在哪裡？

- 配置：`~/.openclaw/`
- 工作區：`~/.openclaw/workspace`
- 憑證：`~/.openclaw/credentials`

### 如何備份資料？

```bash
# 備份整個配置目錄
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### 如何重設所有設定？

1. 停止 Gateway
2. 刪除 `~/.openclaw` 目錄
3. 重新執行 `openclaw onboard`

## 其他問題

### 如何查看日誌？

```bash
# 查看 Gateway 日誌
openclaw gateway --verbose

# 或查看特定日誌檔案
openclaw logs
```

### 如何取得協助？

- 存取文件：https://docs.openclaw.ai
- 加入 Discord 社群
- 在 GitHub 上提交 Issue

### OpenClaw 與其他 AI 助手有什麼區別？

- **完全本地**：您的資料儲存在本地，不上傳雲端
- **開源透明**：代碼完全開放，可自行審查
- **高度可自訂**：可以修改和擴展任何功能
- **多渠道統一**：一個助手管理所有通訊渠道

### 我可以同時執行多個 OpenClaw 執行個體嗎？

可以，每個執行個體需要：
- 不同的連接埠
- 獨立的工作區
- 不同的配置
