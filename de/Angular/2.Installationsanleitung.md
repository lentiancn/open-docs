# Angular Installationsanleitung

Vollständige Einrichtung der Angular-Entwicklungsumgebung.

## Umgebungsanforderungen

### Hardware-Anforderungen

- CPU: 2 Kerne oder mehr
- Speicher: 4GB oder mehr
- Festplatte: 10GB oder mehr

### Software-Anforderungen

- Betriebssystem: Windows/macOS/Linux
- Node.js: 18.10.0+ oder 20.0.0+
- npm: 9.0.0+ (in Node.js enthalten)
- Git: Beliebige Version

## Installationsschritte

### 1. Node.js installieren

**Windows/macOS:**

Herunterladen von: https://nodejs.org

Oder nvm verwenden:

```bash
# nvm installieren (macOS/Linux)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Node.js LTS installieren
nvm install --lts
nvm use --lts
```

**Installation überprüfen:**

```bash
node --version  # Sollte v18.x.x oder v20.x.x anzeigen
npm --version   # Sollte 9.x.x oder höher anzeigen
```

### 2. Angular CLI installieren

```bash
# Global über npm installieren
npm install -g @angular/cli

# Installation überprüfen
ng version
```

### 3. Neues Projekt erstellen

```bash
# Neue Angular-Anwendung erstellen
ng new my-app

# Zum Projekt navigieren
cd my-app

# Entwicklungsserver starten
ng serve
```

### 4. Anwendung aufrufen

Browser öffnen: http://localhost:4200

## Häufige Befehle

### Projektbefehle

```bash
# Neues Projekt mit Routing und SCSS erstellen
ng new my-app --routing --style=scss

# Standalone-App erstellen (Angular 14+)
ng new my-app --standalone

# Entwicklungsserver starten
ng serve

# Für Produktion bauen
ng build

# Unit-Tests ausführen
ng test

# E2E-Tests ausführen
ng e2e
```

### Generierungsbefehle

```bash
# Komponente generieren
ng generate component my-component
ng g c my-component

# Service generieren
ng generate service my-service
ng g s my-service

# Direktive generieren
ng generate directive my-directive
ng g d my-directive

# Pipe generieren
ng generate pipe my-pipe
ng g p my-pipe

# Modul generieren
ng generate module my-module
ng g m my-module

# Klasse generieren
ng generate class my-class
ng g cl my-class

# Interface generieren
ng generate interface my-interface
ng g i my-interface

# Guard generieren
ng generate guard my-guard
ng g g my-guard

# Interceptor generieren
ng generate interceptor my-interceptor
ng g in my-interceptor
```

## Umgebungskonfiguration

### Entwicklungsumgebung

```typescript
// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: 'http://localhost:3000/api'
};
```

### Produktionsumgebung

```typescript
// src/environments/environment.prod.ts
export const environment = {
  production: true,
  apiUrl: 'https://api.example.com'
};
```

### Umgebung verwenden

```typescript
import { Component } from '@angular/core';
import { environment } from '../environments/environment';

@Component({
  selector: 'app-root',
  template: `<p>API: {{ apiUrl }}</p>`
})
export class AppComponent {
  apiUrl = environment.apiUrl;
}
```

## Häufige Probleme

### npm langsam?

```bash
# Taobao-Spiegel verwenden
npm config set registry https://registry.npmmirror.com

# Oder nrm zur Spiegelverwaltung verwenden
npm install -g nrm
nrm use taobao
```

### Node-Version stimmt nicht?

```bash
# nvm zum Versionswechsel verwenden
nvm install 20
nvm use 20
```

### Berechtigungsprobleme (Linux/macOS)

```bash
# npm-Berechtigungen korrigieren
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## Deinstallieren

```bash
# Angular CLI deinstallieren
npm uninstall -g @angular/cli

# Projekt löschen
rm -rf my-app
```
