# JUnit 常见问题

## JUnit 4 和 JUnit 5 我应该用哪个？

对于新项目，强烈建议使用 JUnit 5。它提供了更丰富的功能，包括嵌套测试、参数化测试、动态测试和更好的扩展模型。对于已有 JUnit 4 测试的项目，可以使用 JUnit Vintage 在 JUnit 5 平台上运行，逐步迁移。

## 如何在 JUnit 5 中运行 JUnit 4 测试？

添加 JUnit Vintage 引擎依赖即可：

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## 为什么我的测试没有运行？

常见原因包括：测试类没有加 @Test 注解、测试方法不是 public、没有添加 maven-surefire-plugin 或未配置 useJUnitPlatform()、Java 版本不满足要求。

## JUnit 5 最低 Java 版本是多少？

JUnit 5 需要 Java 8 或更高版本。JUnit 4 需要 Java 5 或更高版本。

## 如何跳过某些测试？

使用 @Disabled 注解可以跳过单个测试或整个测试类：

```java
@Disabled("暂时禁用此测试")
@Test
void testSkipped() {
    // 不会执行
}
```

## 如何测试私有方法？

单元测试应该只测试 public 方法。如果需要测试私有方法，建议重构代码，将需要测试的逻辑移到单独的 public 方法中，或者使用反射（不推荐）。

## 断言失败时如何查看详细信息？

使用第三个参数提供自定义错误消息：

```java
assertEquals(expected, actual, "计算结果不正确");
```

## 如何测试异常？

使用 assertThrows 方法：

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("错误消息");
    });
}
```

## 如何并行运行测试？

在 JUnit 5 中配置并行执行：

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // 测试将并行执行
}
```

## @Before 和 @BeforeEach 有什么区别？

@Before 是 JUnit 4 的注解，@BeforeEach 是 JUnit 5 的注解。两者功能相同，都是在每个测试方法执行前运行。JUnit 5 还支持 @BeforeAll 和 @AfterAll，用于整个测试类只运行一次的前置和后置操作。

## 如何创建参数化测试？

使用 @ParameterizedTest 注解配合 @ValueSource：

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## 测试超时怎么办？

使用 @Timeout 注解设置超时时间：

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // 超过 100 毫秒将失败
}
```

## 如何重复执行测试？

使用 @RepeatedTest 注解：

```java
@RepeatedTest(10)
void testRepeated() {
    // 将重复执行 10 次
}
```

## JUnit 5 的扩展模型是什么？

JUnit 5 使用 @ExtendWith 注解提供扩展机制，可以实现更灵活的测试定制：

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // 使用自定义扩展
}
```
