# Nacos 常见问题

## 快速入门问题

### Nacos 支持哪些操作系统？

Nacos 支持 Linux、Unix、Mac 和 Windows 操作系统，推荐在 Linux/Unix/Mac 环境下运行生产环境。

### Nacos 需要什么 Java 版本？

Nacos 需要 JDK 17 或更高版本。请确保正确配置 JAVA_HOME 环境变量。

### Nacos 启动失败怎么办？

1. 检查 Java 环境：运行 java -version 确认 JDK 版本正确
2. 检查端口占用：确认 8848、9848 端口未被占用
3. 查看日志：检查 logs/start.out 或 logs/nacos.log 中的错误信息
4. 检查鉴权配置：Nacos 3.0+ 需要配置鉴权相关参数

### 如何修改 Nacos 默认端口？

修改 conf/application.properties 文件中的以下配置：

```properties
server.port=8848
```

### Nacos 控制台默认用户名密码是什么？

- 默认用户名：nacos
- 默认密码：nacos@demo

生产环境请及时修改默认密码。

## 服务发现相关问题

### 服务注册成功后，为什么客户端无法发现服务？

1. 检查服务健康状态：确保实例状态为健康
2. 检查网络连通性：确保客户端能访问 Nacos 服务器
3. 检查服务名称：确保查询的服务名与注册时一致（包括命名空间和分组）
4. 检查客户端配置：确认 nacos.server-addr 配置正确

### 如何实现负载均衡？

Nacos 提供了多种负载均衡策略：

1. 客户端负载均衡：通过 Nacos SDK 的 selectOneHealthyInstance 方法
2. 权重负载均衡：通过设置实例权重实现
3. 集群负载均衡：在集群环境中自动实现

### 服务实例下线后，为什么客户端仍然调用到已下线的实例？

1. 检查健康检查配置：确保健康检查机制正常工作
2. 检查客户端缓存：客户端会缓存服务实例列表，有一定延迟
3. 调整同步间隔：修改客户端的刷新间隔

## 配置管理相关问题

### 配置发布后，客户端没有收到更新通知？

1. 检查监听器：确保客户端正确添加了配置监听器
2. 检查网络：确保网络连通性正常
3. 查看日志：检查客户端日志中的错误信息
4. 确认 dataId 和 group：确保完全匹配

### 如何实现配置的多环境管理？

1. 使用命名空间（Namespace）：不同环境使用不同命名空间实现完全隔离
2. 使用分组（Group）：同一命名空间下使用不同分组管理配置
3. 使用 DataId：通过不同的 dataId 区分不同环境配置

### 配置历史版本如何查看？

Nacos 控制台提供配置变更历史功能，可以查看：

- 配置的修改记录
- 每次修改的内容对比
- 一键回滚到历史版本

### 如何实现配置的灰度发布？

Nacos 支持配置灰度发布功能：

1. 在控制台创建灰度配置
2. 指定灰度客户端范围
3. 验证灰度配置效果
4. 全量发布或回滚

## 集群部署问题

### Nacos 集群部署需要多少节点？

生产环境推荐部署至少 3 个节点，以保证高可用性。

### 集群模式下，如何选择数据存储？

- 测试/开发环境：可以使用内置 Derby 数据库
- 生产环境：必须使用外置数据库（MySQL），保证数据可靠性

### 集群节点之间如何保证数据一致性？

Nacos 使用自研的 Distro 协议和 Raft 协议保证集群数据一致性：

- Distro 协议：负责数据同步
- Raft 协议：负责选主和日志复制

### 集群节点故障怎么办？

Nacos 集群具有自动故障转移能力：

1. 健康检查会自动检测故障节点
2. 流量会自动切换到健康节点
3. 故障节点恢复后自动同步数据

## 鉴权与安全

### Nacos 3.0 鉴权如何配置？

Nacos 3.0+ 默认启用鉴权，启动时需要配置以下参数：

```bash
# Token 密钥
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# 服务身份密钥
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### 如何关闭 Nacos 鉴权？

修改 conf/application.properties：

```properties
nacos.core.auth.enabled=false
```

注意：生产环境不建议关闭鉴权。

### 如何为不同用户分配权限？

Nacos 支持基于角色的访问控制：

1. 在控制台创建用户
2. 为用户分配角色
3. 配置角色权限

## MCP 相关问题

### 启动 MCP Server 时出现兼容性问题怎么办？

确保自动注册的 MCP Server 数据和 Nacos 上对应版本的数据兼容性：

- MCP Server 协议（stdio、sse、streamable）是否一致
- 服务引用是否一致
- Tools 数据是否一致（Tools 名称、Tools 数量、参数类型、参数是否可选）

注意：Tools 描述和参数描述不要求一致。

### MCP Server 注册后无法被发现？

1. 检查 MCP Server 状态：确保服务已正确注册
2. 检查命名空间：确保客户端和服务在同一命名空间
3. 检查认证信息：确保用户名密码正确

### 如何动态修改 MCP Server 工具配置？

MCP Server 自动注册到 Nacos 之后，可以在控制台上动态变更 MCP Server Tools 的描述和开关，相关修改将会在运行时中动态应用到 MCP Server 中，无需重启服务。

## 性能与优化

### Nacos 如何处理大规模服务注册？

Nacos 诞生自阿里的百万实例规模，具备以下能力：

- 高性能服务注册和发现
- 低延迟配置更新响应
- 海量服务和配置管理

### 如何优化 Nacos 性能？

1. 合理设置健康检查间隔
2. 使用合适的数据库连接池配置
3. 启用客户端缓存减少请求次数
4. 合理规划命名空间和分组

### Nacos 内存配置建议？

根据服务规模建议：

- 小规模（服务数 < 1000）：2C4G
- 中规模（服务数 1000-5000）：4C8G
- 大规模（服务数 > 5000）：8C16G 以上

## 生态集成问题

### Nacos 与 Spring Cloud 如何集成？

在 pom.xml 中添加以下依赖：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### Nacos 与 Kubernetes 如何集成？

Nacos 支持在 Kubernetes 环境中部署，可以使用：

- Helm Chart 部署
- Operator 部署
- 自定义 YAML 部署

### Nacos 支持哪些语言的 SDK？

Nacos 提供以下语言的 SDK：

- Java（官方）
- Python
- Go
- Node.js
- C/C++
- Ruby
- PHP

## 故障排查

### 如何排查服务注册失败？

1. 检查 Nacos Server 是否正常运行
2. 检查网络连通性
3. 查看客户端日志
4. 确认注册参数是否正确

### 如何排查配置无法获取？

1. 确认配置已发布成功
2. 检查 dataId 和 group 是否匹配
3. 检查监听器是否正确添加
4. 查看客户端日志排查具体错误

### Nacos 日志在哪里？

Nacos 日志文件位于：

- 主日志：logs/nacos.log
- 启动日志：logs/start.out
- 访问日志：logs/access.log

## 升级与迁移

### 如何从 Nacos 1.x 升级到 2.x 或 3.x？

1. 备份数据：备份数据库或快照文件
2. 阅读升级指南：查看对应版本的升级文档
3. 逐步升级：建议先在测试环境验证
4. 迁移配置：注意鉴权等配置的变化

### 如何迁移 Nacos 数据？

1. 导出原环境配置数据
2. 导入到新环境数据库
3. 验证数据完整性
4. 切换流量到新环境
