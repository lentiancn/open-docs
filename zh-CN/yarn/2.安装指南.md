# Yarn 安装指南

## Yarn v4+ (现代版本) 安装

### 推荐方式：使用 Corepack

Corepack 是一个中介工具，可以让您基于每个项目配置包管理器版本。

```bash
# 1. 安装 Corepack
npm install -g corepack

# 2. 初始化新项目
yarn init -2
```

### 更新 Yarn

任何时候想要将 Yarn 更新到最新版本，只需运行：

```bash
yarn set version stable
yarn install
```

Yarn 将配置您的项目使用最新的稳定二进制文件。

### 使用 Canary 版本

Yarn 还经常发布 Release Candidate 构建。如果您需要尚未在稳定频道发布的功能，请使用：

```bash
yarn set version canary
```

这些构建非常稳定，与常规频道的唯一区别是在实现新的破坏性更改时采用更渐进的迁移方式。

### 从源码安装最新版本

如果您想测试尚未发布甚至未合并的 Yarn 版本，以下命令将直接从仓库克隆、构建并安装 Yarn 到您的项目中：

```bash
yarn set version from sources
```

它接受 `--branch` 标志，您可以用来测试特定的 PR：

```bash
yarn set version from sources --branch 1211
```

> 注意：与稳定和 canary 频道不同，`yarn set version from sources` 命令无法利用 Corepack，需要将 Yarn 二进制文件存储在 `.yarn/releases` 文件夹中，并从项目的 `.yarnrc.yml` 文件中引用它。

## Yarn v1 (Classic) 安装

### 通过 npm 安装（推荐）

建议通过 [npm 包管理器](http://npmjs.org/) 安装 Yarn，该管理器在您在系统上安装 [Node.js](https://nodejs.org/) 时会捆绑在一起。

安装 npm 后，您可以运行以下命令来安装和升级 Yarn：

```bash
npm install --global yarn
```

### 验证安装

通过运行以下命令检查 Yarn 是否已安装：

```bash
yarn --version
```

## 版本选择建议

- **新项目**: 推荐使用 Yarn v4+，享受现代特性和更好的性能
- **现有项目**: 如果已在使用 Yarn v1，可以继续使用或考虑迁移到现代版本
- **企业环境**: Yarn v1 提供更稳定的长期支持，而 v4+ 提供更多创新功能
