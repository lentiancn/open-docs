# Flutter 常見問題

## 安裝問題

### Flutter doctor 顯示錯誤怎麼辦？

flutter doctor 命令會檢查 Flutter 開發環境的狀態。常見錯誤及解決方法：

- Android SDK 未找到：需要安裝 Android Studio 或手動配置 ANDROID_SDK_ROOT 環境變數
- Xcode 版本過舊：更新 Xcode 到最新版本
- 授權未接受：執行 flutter doctor --android-licenses 接受授權

### 下載 Flutter 失敗怎麼辦？

如果下載速度慢或失敗，可以嘗試使用鏡像源。中國用戶可以設定 PUB_HOSTED_URL 和 FLUTTER_STORAGE_BASE_URL 環境變數使用國內鏡像。

### Git 克隆失敗怎麼辦？

確保網路可以存取 GitHub。可以嘗試使用代理或使用國內鏡像。也可以從 Flutter 官網下載預編譯的壓縮包手動安裝。

## 開發問題

### 熱重載不生效怎麼辦？

熱重載失敗時，嘗試以下方法：點擊 Stop 然後重新執行；執行 flutter clean 清理構建緩衝；檢查程式碼中是否有語法錯誤。

### 應用卡頓怎麼辦？

檢查是否在 build 方法中執行了耗時操作。複雜計算應該使用 Isolate 或非同步處理。避免不必要的 Widget 重建，使用 const 建構函數和適當的 key。

### 狀態丟失怎麼辦？

熱重載會保留 StatefulWidget 的狀態，但重建應用或執行完全熱重啟後會重置狀態。如果需要持久化狀態，使用 SharedPreferences、SQLite 或其他儲存方案。

### 封包依賴衝突怎麼辦？

執行 flutter pub get 查看依賴樹。使用 flutter pub deps 檢查封包依賴。嘗試更新封包版本或使用 dependency_overrides 解決衝突。

## 平台問題

### Android 構建失敗怎麼辦？

常見原因包括：SDK 版本不匹配、Gradle 配置錯誤、簽章配置錯誤等。嘗試執行 flutter clean 後重新構建。檢查 android/build.gradle 和 android/app/build.gradle 配置。

### iOS 模擬器無法啟動怎麼辦？

確保 Xcode 已安裝並正確配置。在終端執行 xcrun simctl list devices 查看可用的模擬器。嘗試重置模擬器或建立新模擬器。

### Web 構建報錯怎麼辦？

檢查 pubspec.yaml 中的 web 依賴是否正確。使用 flutter clean 清理後重新構建。確保 Chrome 瀏覽器已安裝。

## 效能問題

### 應用體積太大怎麼辦？

使用 flutter build apk --split-per-abi 產生針對不同 CPU 架構的 APK。檢查並移除未使用的圖片和資源。使用混淆和壓縮。

### 啟動時間太長怎麼辦？

減少 main 函數中的初始化操作。延遲載入非必要的元件。使用 Splash Screen 改善使用者體驗。

### 記憶體佔用過高怎麼辦？

檢查是否有記憶體洩漏。使用 Profile 工具分析記憶體使用。及時釋放不再使用的資源和大物件。

## 學習問題

### 如何選擇狀態管理方案？

簡單應用使用 setState 即可。中等複雜度應用推薦 Provider。複雜應用可以使用 Riverpod、GetX 或 BLoC。

### 從其他框架遷移怎麼辦？

Flutter 官網有針對 Android、iOS、React Native 等開發者的遷移指南。核心概念是相通的，重點學習 Widget 和佈局系統。

### 如何提高開發效率？

熟練使用 VS Code 或 Android Studio 的 Flutter 外掛。掌握快捷鍵和程式碼模板。使用熱重載和調試工具。參考官方文件和社區資源。
