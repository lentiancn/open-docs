# Hadoop Benutzerhandbuch

## HDFS-Betriebsanleitung

### Grundlegende Dateisystemoperationen

HDFS bietet eine Befehlszeilenschnittstelle, die herkömmlichen Dateisystemen ähnelt, sodass Benutzer über den Befehl hdfs dfs verschiedene Dateioperationen durchführen können. Diese Befehle ähneln stark Linux-Shell-Befehlen, sodass Benutzer, die mit Linux-Systemen vertraut sind, schnell damit beginnen können. Das Verständnis dieser grundlegenden Operationen ist fundamental für die Nutzung von Hadoop; im Folgenden werden die am häufigsten verwendeten HDFS-Befehle erläutert.

Das Erstellen von Verzeichnissen erfolgt mit dem mkdir-Befehl, der einstufige oder mehrstufige Verzeichnisse erstellen kann:

```bash
# Einstufiges Verzeichnis erstellen
hdfs dfs -mkdir /user/data

# Mehrstufiges Verzeichnis erstellen
hdfs dfs -mkdir -p /user/data/input

# Verzeichnisinhalt auflisten
hdfs dfs -ls /user/data

# Verzeichnisinhalt rekursiv auflisten
hdfs dfs -ls -R /user/data
```

Das Hochladen und Herunterladen von Dateien sind häufig verwendete Operationen:

```bash
# Lokale Datei nach HDFS hochladen
hdfs dfs -put localfile.txt /user/data/

# Hochladen und umbenennen
hdfs dfs -put localfile.txt /user/data/newname.txt

# HDFS-Datei lokal herunterladen
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# Interne HDFS-Dateien kopieren
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# Interne HDFS-Dateien verschieben
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

Datei anzeigen und löschen:

```bash
# Dateiinhalt anzeigen
hdfs dfs -cat /user/data/file.txt

# Letzte Zeilen der Datei anzeigen
hdfs dfs -tail /user/data/file.txt

# Dateigröße anzeigen
hdfs dfs -du /user/data/file.txt

# Datei löschen
hdfs dfs -rm /user/data/file.txt

# Verzeichnis rekursiv löschen
hdfs dfs -rm -r /user/data/dir
```

### Berechtigungsverwaltung

Das Berechtigungsmodell von HDFS ähnelt Unix/Linux und basiert auf einem dreistufigen System aus Benutzer, Gruppe und Sonstige. Jede Datei und jedes Verzeichnis hat drei Berechtigungen: Lesen (r), Schreiben (w), Ausführen (x), die jeweils Eigentümer, Gruppe und Sonstige betreffen. Das Verständnis der Berechtigungsverwaltung ist entscheidend für den Schutz der Datensicherheit.

```bash
# Dateieigentümer ändern
hdfs dfs -chown user:group /user/data/file.txt

# Dateiberechtigungen ändern
hdfs dfs -chmod 755 /user/data/file.txt

# Gruppenzugehörigkeit ändern
hdfs dfs -chgrp groupname /user/data/file.txt
```

### Datenreplikation und -balancing

HDFS repliziert standardmäßig jeden Datenblock dreimal und speichert ihn auf verschiedenen DataNodes. Der Standard-Replikationsfaktor kann über die Eigenschaft dfs.replication geändert werden:

```bash
# Replikationsfaktor für bestimmte Datei festlegen
hdfs dfs -setrep -w 2 /user/data/file.txt

# Blockstandortinformationen für Datei anzeigen
hdfs fsck /user/data/file.txt -files -blocks -locations
```

Wenn Knoten zum Cluster hinzugefügt oder daraus entfernt werden, können die Daten ungleichmäßig verteilt sein. Verwenden Sie in diesem Fall den rebalance-Befehl zum Ausbalancieren der Daten:

```bash
start-balancer.sh -threshold 10
```

Der threshold-Parameter stellt die maximale prozentuale Abweichung zwischen Knotenfestplattennutzung und Clusterdurchschnitt dar. Der Balance-Prozess verbraucht Netzwerkbandbreite; es wird empfohlen, ihn in Zeiten geringer Auslastung auszuführen.

### HDFS-Web-Benutzeroberfläche

HDFS bietet eine funktionsreiche Web-Oberfläche, die über den Browser am NameNode-Port 9870 zugänglich ist. Über die Web-Oberfläche können Sie Dateisystemverzeichnisse durchsuchen, Dateiinhalte anzeigen, DataNode-Status überwachen, Blockinformationen anzeigen usw. Die Web-Oberfläche ist ein wichtiges Werkzeug für das Betriebspersonal bei der täglichen Überwachung und Fehlerbehebung.

Die Hauptfunktionen der Web-Oberfläche umfassen: Durchsuchen des HDFS-Verzeichnisbaums, Anzeigen von Dateilisten und Dateieigenschaften; Anzeigen der DataNode-Liste und der Speichernutzung jedes DataNode; Anzeigen der Blockverteilung und Replikationsstatus; Anzeigen der NameNode-Laufzeitinformationen und -Protokolle; Ausführen einfacher Dateioperationen wie Herunterladen und Löschen.

## MapReduce-Programmierung

### WordCount-Beispiel

WordCount ist das Einstiegsbeispiel für die MapReduce-Programmierung und zählt das Vorkommen jedes Wortes in einem Text. Das Verständnis dieses Beispiels hilft beim Erlernen des MapReduce-Programmiermodells. Das Programm ist in Map- und Reduce-Phasen unterteilt: In der Map-Phase wird der Text in Wörter zerlegt, und in der Reduce-Phase werden die Anzahlungen derselben Wörter zusammengefasst.

Map-Phasen-Code-Beispiel:

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduce-Phasen-Code-Beispiel:

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### Job-Übermittlung

Nach dem Verpacken von MapReduce in eine JAR-Datei übermitteln Sie den Job mit dem Befehl hadoop jar:

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

Nach der Job-Übermittlung können Sie den Job-Fortschritt und die Ergebnisse in der YARN-Web-Oberfläche (Port 8088) anzeigen. Nach Abschluss des Jobs werden Job-Statistiken ausgegeben, einschließlich Job-ID, Anzahl der Aufgaben, Ausführungszeit usw.

### Job-Konfiguration

Verschiedene Job-Parameter können über das Configuration-Objekt konfiguriert werden:

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

Häufig verwendete Konfigurationsparameter umfassen: mapreduce.job.reduces legt die Anzahl der Reduce-Aufgaben fest; mapreduce.map.memory.mb und mapreduce.reduce.memory.mb legen den Container-Speicher fest; mapreduce.map.cpu.vcores und mapreduce.reduce.cpu.vcores legen die Container-CPU-Kerne fest.

## YARN-Betrieb

### Cluster-Ressourcen anzeigen

YARN bietet umfangreiche Befehlszeilentools zur Verwaltung von Cluster-Ressourcen:

```bash
# Alle Knotenstatus anzeigen
yarn node -list -all

# Anwendungsliste anzeigen
yarn application -list

# Bestimmten Anwendungsstatus anzeigen
yarn application -status application_1234567890123_0001

# Anwendungsprotokolle anzeigen
yarn logs -applicationId application_1234567890123_0001
```

### Warteschlangenverwaltung

YARN verwendet Warteschlangen zur Verwaltung der Ressourcenzuweisung. Warteschlangen können über den Capacity Scheduler oder Fair Scheduler konfiguriert werden:

```bash
# Warteschlangeninformationen anzeigen
yarn queue -status default

# Warteschlangenkapazität anzeigen
yarn resourcemanager -refreshQueues
```

### Ressourcenplaner-Konfiguration

Der Capacity Scheduler ist der Standard-Planer und unterstützt die gemeinsame Nutzung von Ressourcen durch mehrere Mandanten. Konfigurieren Sie in yarn-site.xml:

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## Leistungsoptimierung

### MapReduce-Abstimmung

Die Leistungsoptimierung von MapReduce-Jobs umfasst mehrere Aspekte. Zunächst legen Sie die Anzahl der Map- und Reduce-Aufgaben vernünftig fest. Die Anzahl der Map-Aufgaben wird normalerweise durch die Anzahl der Eingabedaten-Splits bestimmt, die über den Parameter mapreduce.input.fileinputformat.split.maxsize angepasst werden kann. Die Anzahl der Reduce-Aufgaben wird empfohlen, auf das 0,9- bis 1,2-fache der Gesamtzahl der Cluster-Reduce-Slots festzulegen.

```bash
# Anzahl der Map-Aufgaben festlegen
-D mapreduce.job.maps=10

# Anzahl der Reduce-Aufgaben festlegen
-D mapreduce.job.reduces=5
```

Die Optimierung der Shuffle-Phase hat erheblichen Einfluss auf die Gesamtleistung. Diese Parameter können angepasst werden:

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFS-Abstimmung

Die HDFS-Leistungsoptimierung konzentriert sich hauptsächlich auf die Datenlesen/-schreiben-Effizienz. Für große Dateien kann die Blockgröße angepasst werden, um Datenlokalität und Fehlerwiederherstellung auszubalancieren:

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

Aktivieren Sie Short-Circuit-Reads, damit Clients Daten direkt von lokalen DataNodes lesen können, was die Netzwerkübertragung reduziert:

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARN-Abstimmung

Die Ressourcenzuweisung von YARN hat erheblichen Einfluss auf die Cluster-Auslastung. Legen Sie die NodeManager-Ressourcenkapazität vernünftig fest:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

Legen Sie virtuelle CPU-Kerne fest, um Multi-Core-Prozessoren besser zu nutzen:

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## Datenimport und -export

### Daten aus Datenbanken importieren

Verwenden Sie Sqoop, um Daten bequem zwischen relationalen Datenbanken und HDFS zu migrieren:

```bash
# Ganze Tabelle importieren
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Importieren und Hive-Tabelle erstellen
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### Daten in Datenbanken exportieren

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### Auswahl des Datenformats

Verschiedene Datenformate wirken sich unterschiedlich auf die Leistung aus. Häufig verwendete Formate umfassen: TextFile (gute Allgemeinheit, aber durchschnittliche Leistung); SequenceFile (gut für MapReduce); Parquet (spaltenorientierter Speicher, gut für analytische Abfragen); ORC (Hive-optimierter spaltenorientierter Speicher). Für analytische Workloads wird empfohlen, spaltenorientierte Speicherformate wie Parquet oder ORC zu verwenden.
