# Nacos 安装指南

## 环境要求

Nacos 依赖 Java 环境来运行，请确保在以下版本环境中安装使用：

- 64 位操作系统，支持 Linux/Unix/Mac/Windows，推荐选用 Linux/Unix/Mac
- 64 位 JDK 17+（推荐使用 JDK 17 或更高版本）
- JDK 下载地址：https://www.oracle.com/java/technologies/downloads/#java17
- JDK 环境配置说明：https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## 推荐配置

Nacos 的运行建议至少在以下机器配置下运行：

- CPU：2 核
- 内存：4 GB
- 磁盘：60 GB

## 安装方式

Nacos 支持多种安装方式，您可以根据实际需求选择适合的方式。

### 方式一：使用 nacos-setup 一键部署（推荐）

nacos-setup 提供一键自动化部署，自动处理下载、鉴权配置、端口检测和 Java 环境验证等操作。

#### 安装 nacos-setup

Linux/macOS：

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows（PowerShell）：

```powershell
# 使用 PowerShell 执行
```

#### 启动 Nacos 服务

本地一键部署一个最新版本的单机 Nacos：

```bash
nacos-setup
```

本地一键部署指定版本的单机 Nacos：

```bash
nacos-setup -v 3.1.1
```

本地一键部署一个最新版本的集群 Nacos：

```bash
nacos-setup -c cluster
```

全局配置使用外置存储，输入数据库地址端口账号密码，配置完成后重新运行 nacos-setup：

```bash
nacos-setup --datasource-conf
```

更多用法请参考 nacos-setup 使用手册。

### 方式二：手动下载发行包部署

#### 下载安装包

您可以通过以下两种方式获取 Nacos 发行包：

**方式一：通过 Nacos 官网下载**

进入 Nacos 官网版本下载页面，选择稳定版本，然后点击二进制包下载列中的 zip 文件进行下载。

**方式二：通过 GitHub 下载**

进入 Nacos GitHub Releases 页面，下载对应版本的发行包。

注意：有时大量用户同时进行下载时，可能会遇到下载限流失败的情况，若出现下载限流失败，请稍等后重试，或采用从 GitHub 下载方式。

#### 解压缩 Nacos 发行包

Linux/Unix/Mac：

```bash
unzip nacos-server-$version.zip
# 或者
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows：

直接使用解压工具解压 nacos-server-$version.zip

#### 启动服务器

Linux/Unix/Mac 启动命令（standalone 代表着单机模式运行，非集群模式）：

```bash
sh startup.sh -m standalone
```

如果您使用的是 Ubuntu 系统，或者运行脚本报错提示符号找不到，可尝试如下运行：

```bash
bash startup.sh -m standalone
```

启动程序会提示您输入三个鉴权相关配置（Nacos 从 3.0.0 版本开始默认启用控制台鉴权功能，因此如下三个鉴权相关配置必须填写）：

```
nacos.core.auth.plugin.nacos.token.secret.key is missing, please set with Base64 string: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key is missing, please set: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value is missing, please set: ${your_input_server_identity_key}
```

Windows 启动命令：

```bash
cmd startup.cmd
```

或者双击 startup.cmd 运行文件。

### 方式三：使用 Docker 安装

#### 拉取 Nacos 镜像

```bash
docker pull nacos/nacos-server
```

#### 运行单机模式

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### 运行集群模式

需要配合 Docker Compose 或 Kubernetes 进行集群部署。

### 方式四：使用 Kubernetes 安装

Nacos 支持在 Kubernetes 环境中部署，可以使用 Helm Chart 或 Operator 进行安装。具体步骤请参考 Nacos Kubernetes 部署文档。

## 验证 Nacos 服务是否启动成功

进入 Nacos 日志目录下，使用 tail -f start.out 查看日志，如果看到如下日志，说明服务启动成功：

```
Nacos started successfully in stand alone mode. use embedded storage
```

## 快速验证功能

### Nacos 控制台页面

打开任意浏览器，输入地址：http://127.0.0.1:8080/index.html，即可进入 Nacos 控制台页面。

注意：首次打开会要求初始化管理员用户 nacos 的密码。默认用户名密码为 nacos/nacos@demo。

### 服务注册

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### 服务发现

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### 发布配置

首先登录获取 access token：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

使用 access token 创建配置：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### 获取配置

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## 关闭服务器

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

或者双击 shutdown.cmd 运行文件。

## 版本选择

生产环境当前推荐的稳定版本为 3.1.1。您可以通过以下途径了解每个版本支持的功能：

- Nacos 版本下载页面
- Nacos Release Notes
- Nacos 发布声明

## 注意事项

1. Nacos 3.0.0 版本开始默认启用控制台鉴权功能，首次启动需要配置鉴权相关参数
2. 集群模式部署需要使用外置数据库（如 MySQL）进行数据存储
3. 生产环境建议使用集群模式以保证高可用性
4. 确保防火墙开放必要端口（默认 8848、9848）
5. 建议使用 JDK 17 或更高版本以获得最佳性能
