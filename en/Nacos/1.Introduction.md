# Introduction

## What is Nacos

Nacos /nɑ:kəʊs/ is the acronym for Dynamic Naming and Configuration Service, an easy-to-use dynamic service discovery, configuration management, and service management platform for building cloud native applications.

Nacos is committed to helping you discover, configure, and manage microservices and AI agent applications. It provides a set of simple and useful features enabling you to realize dynamic service discovery, service configuration, service metadata, and traffic management.

Nacos makes it easier and faster to construct, deliver, and manage your microservices platform. It is the infrastructure that supports a service-centered modern application architecture with a microservices or cloud-native approach.

Nacos supports discovering, configuring, and managing almost all types of services:

- Kubernetes Service
- gRPC
- Dubbo RPC Service
- Spring Cloud RESTful Service
- Model Context Protocol

## Core Features

### Service Discovery and Health Check

Nacos supports both DNS-based and RPC-based service discovery. After a service provider registers a service using native SDK, OpenAPI, or a dedicated agent, consumers can discover the service using either DNS or HTTP API.

Nacos provides real-time health checks to prevent services from sending requests to unhealthy hosts or service instances. Nacos supports both transport layer (PING or TCP) health checks and application layer (such as HTTP, MySQL, user-defined protocols) health checks. For health checks in complex cloud environments and network topologies (such as VPC, Edge Services), Nacos provides both agent reporting mode and server-side active detection. Nacos also provides a unified health check dashboard to help you manage service availability and traffic based on health status.

### Dynamic Configuration Service

Dynamic configuration service allows you to manage the configuration of all applications and services in a centralized, externalized, and dynamic manner across all environments. Dynamic configuration eliminates the need to redeploy applications and services when configurations are updated, making configuration management more efficient and agile.

Centralized management of configuration makes it easier to achieve stateless services and elastic expansion of service instances on-demand. Nacos provides an easy-to-use UI to help you manage all your application and service configurations. Nacos also provides out-of-box features including configuration version tracking, canary release, one-click configuration rollback, and client configuration update status tracking to ensure safety and control risks in production environments.

### Dynamic DNS Service

Dynamic DNS service supports weighted routing, making it easier to implement mid-tier load balancing, flexible routing policies, traffic control, and simple DNS resolution services within your data center. Dynamic DNS service also makes it easier to implement DNS-based service discovery to help eliminate coupling to vendor-specific private service discovery APIs.

Nacos provides simple DNS APIs to help you manage service-associated domain names and available IP:PORT lists.

### Service and Metadata Management

Nacos allows you to manage all services and metadata from a microservices platform builder's perspective. This includes managing service description, lifecycle, static dependency analysis, health status, traffic management, routing and security policies, SLA, and key metrics.

### MCP Service Management and Transformation

Nacos enables you to quickly transform existing microservice APIs into MCP services through simple declarations with zero code changes. Additionally, using Spring AI Alibaba, nacos-mcp-wrapper-python, and Nacos-MCP-Router, MCP services can be automatically registered and discovered, allowing MCP services to be used just like microservices. From the MCP service platform perspective, Nacos also manages MCP service descriptions, tools, prompts, and version management.

### Easy to Use

Nacos has been refined through feedback from tens of thousands of users, providing unified service discovery and configuration management functions. Through intuitive web interface and concise APIs, it brings convenient service registration, discovery, and configuration update operations for developers and operations personnel in cloud-native environments. As a one-stop solution for dynamic configuration management and service discovery for AI and cloud-native applications, it offers over twenty ready-to-use AI service-centric architecture features with a lightweight and easy-to-use control interface for rapidly building AI applications.

Nacos provides a comprehensive set of features including service discovery, configuration management, dynamic DNS service, service metadata management, traffic management, service monitoring, and service governance. It helps you build, deliver, and manage microservices more easily in the cloud-native era. It supports AI Registry scenarios including Prompt, MCP, and A2A. With direct support or simple extensions, it can support a large number of useful internet and AI application scenarios for traffic scheduling and service governance, including rate limiting, promotional planning, and multi-region active-active deployment.

### Extreme Performance

Nacos, honed through Alibaba's Double 11 shopping festival extreme scaling scenarios, provides high-performance service registration and discovery capabilities along with low-latency configuration update responses, ensuring efficient and stable operation in large-scale distributed systems.

### Massive Capacity

Born from Alibaba's millions of instances scale, Nacos supports massive-scale service and configuration management, meeting the needs of large distributed systems for high concurrency and high availability.

### Stable and Reliable

Derived from Alibaba's internally proven product with over ten years of production validation, it supports large-scale scenarios with millions of services and is an enterprise-level SLA open-source product used by tens of millions of users. Nacos uses self-developed synchronization protocols combined with the widely-used Raft protocol in the ecosystem to ensure high availability of services and data stability, guaranteeing Alibaba's Double 11 system's high availability and stable operation.

### Open Ecosystem

Nacos has an active open-source community, extensive ecosystem integration, and continuous innovation. It is not only highly compatible with popular open-source frameworks like Spring Cloud and Dubbo but also provides rich plugin capabilities to help users customize their unique cloud-native microservice systems in the cloud-native era. It seamlessly supports Kubernetes, Spring Cloud, Spring AI, and Dify, making deployment and operation easier on mainstream public clouds (such as Alibaba Cloud, Huawei Cloud, Tencent Cloud, Azure, and AWS) with multi-tenant and multi-environment support.

## Design Philosophy

Nacos aims to help users in the AI era, in private cloud, hybrid cloud, or public cloud environments, to better build, deliver, and manage their AI management platforms, faster reuse and combine business services, and faster deliver commercial innovation value. Nacos's design philosophy is positioned as:

### Easy to Use

Easy to use is a core philosophy of Nacos. It simplifies service registration, discovery, and configuration management through a user-friendly web interface and concise APIs. Developers can easily integrate Nacos into their applications without investing significant time in complex setup and learning.

### Standards-Oriented

Nacos follows a standards-oriented design philosophy, adhering to best practices and standard protocols for cloud-native application development to ensure its service discovery and configuration management functions seamlessly integrate with a wide range of technology stacks and platforms.

### High Availability

To meet enterprise application requirements for high availability, Nacos implements cluster mode, ensuring that service discovery and configuration management functions are not affected when node failures occur. Cluster mode also means Nacos can scale horizontally by adding nodes, improving overall system performance and carrying capacity.

### Easy to Extend

Nacos also focuses on easy extension, using modular design so that each component can be independently extended or replaced. This also provides convenience for community contributors to develop new features or improve existing ones for specific requirements, further promoting Nacos's ecosystem development.

Through the implementation of these design philosophies, Nacos provides users with a powerful and flexible platform to support ever-changing business needs, accelerate business innovation and digital transformation, ultimately helping users gain a competitive advantage in the market.

## Deployment Modes

### Standalone Mode

Standalone mode, also known as single-instance mode, possesses all Nacos features and characteristics with extremely easy deployment and quick startup. However, it cannot form a cluster with other nodes and cannot provide high availability when node or network failures occur. Standalone mode can use either built-in Derby database (default) or external database for storage. Standalone mode is mainly suitable for engineers building Nacos environments locally or in testing environments for development, debugging, and testing. It can also handle some business scenarios with low stability and availability requirements.

### Cluster Mode

Cluster mode constructs a high-availability Nacos cluster by combining multiple Nacos nodes through the self-developed Distro protocol and Raft protocol. Data is synchronized across all nodes in the cluster to ensure data consistency. Cluster mode has advantages of high availability, high scalability, and high concurrency, ensuring business operations are not affected when failures occur. Cluster mode uses external database for storage by default but can also use built-in database. This mode is mainly suitable for production environments and is the most recommended deployment method by the community.
