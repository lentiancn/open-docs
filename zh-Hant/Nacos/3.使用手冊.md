# Nacos 使用手冊

## 概述

Nacos 提供兩大核心功能：服務發現和配置管理。本手冊將詳細介紹如何使用這些功能。

## 服務發現

### 服務註冊

服務提供者可以透過以下方式將服務註冊到 Nacos：

#### 使用原生 SDK 註冊

透過 Nacos 提供的 Java SDK 進行服務註冊：

```java
import com.alibaba.nacos.api.naming.NamingService;
import com.alibaba.nacos.api.naming.pojo.Instance;

NamingService namingService = NacosFactory.createNamingService("127.0.0.1:8848");

Instance instance = new Instance();
instance.setIp("192.168.1.100");
instance.setPort(8080);
instance.setEphemeral(true);

Map<String, String> metadata = new HashMap<>();
metadata.put("version", "1.0");
instance.setMetadata(metadata);

namingService.registerInstance("service-provider", instance);
```

#### 使用 OpenAPI 註冊

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance' \
  -d 'serviceName=service-provider' \
  -d 'ip=192.168.1.100' \
  -d 'port=8080'
```

### 服務發現

服務消費者可以透過以下方式發現服務：

#### 使用原生 SDK 發現服務

```java
NamingService namingService = NacosFactory.createNamingService("127.0.0.1:8848");

// 獲取所有健康實例
List<Instance> instances = namingService.selectInstances("service-provider", true);

// 使用負載均衡選擇實例
Instance instance = namingService.selectOneHealthyInstance("service-provider");
```

#### 使用 OpenAPI 發現服務

```bash
# 獲取服務列表
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=service-provider'
```

### 服務元資料管理

Nacos 支援為服務添加和管理元資料：

```java
Map<String, String> metadata = new HashMap<>();
metadata.put("version", "1.0");
metadata.put("env", "production");

namingService.registerInstance("service-name", instance);
```

## 配置管理

### 發布配置

#### 使用 OpenAPI 發布配置

```bash
# 首先登入獲取 access token
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' \
  -d 'username=nacos' \
  -d 'password=${your_password}'

# 使用 access token 創建配置
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=example.config&groupName=DEFAULT_GROUP&content=HelloWorld' \
  -H "accessToken:${your_access_token}"
```

#### 使用 SDK 發布配置

```java
ConfigService configService = NacosFactory.createConfigService("127.0.0.1:8848");

boolean isPublished = configService.publishConfig(
    "example.config",
    "DEFAULT_GROUP",
    "Hello Nacos"
);
```

### 獲取配置

#### 使用 OpenAPI 獲取配置

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=example.config&groupName=DEFAULT_GROUP'
```

#### 使用 SDK 獲取配置

```java
String config = configService.getConfig(
    "example.config",
    "DEFAULT_GROUP",
    5000
);
```

### 配置監聽

#### 使用 SDK 監聽配置變化

```java
configService.addListener(
    "example.config",
    "DEFAULT_GROUP",
    new Listener() {
        @Override
        public void receiveConfigInfo(String configInfo) {
            System.out.println("配置變更:" + configInfo);
        }

        @Override
        public Executor getExecutor() {
            return Executors.newSingleThreadExecutor();
        }
    }
);
```

## Nacos 與 Spring Cloud 整合

### 前提條件

確保已安裝並啟動 Nacos Server。

### 引入依賴

在 Spring Cloud 專案的 pom.xml 中添加以下依賴：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>

<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### 配置管理

在 application.properties 中配置 Nacos server 位址：

```properties
spring.application.name=springclouddemo
spring.config.import[0]=nacos:springclouddemo.properties?group=DEFAULT_GROUP
spring.cloud.nacos.config.server-addr=127.0.0.1:8848
spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848
```

### 使用註解

#### @NacosConfig 註解

將 Nacos 中的配置值直接注入到 Spring Bean 的屬性中：

```java
@RestController
public class ConfigController {
    @NacosConfig(dataId = "routeconfig", group = "config", key = "rate")
    String rate;

    @RequestMapping("/rate")
    public String rate() {
        return rate;
    }
}
```

#### @NacosConfigListener 註解

接收 Nacos 中配置的變更事件：

```java
@Service
public class MyRateConfigService {
    @NacosConfigListener(dataId = "routeconfig", group = "config")
    public void rate(String rateConfig) {
        System.out.println("收到配置變更:" + rateConfig);
    }
}
```

### 服務註冊發現

在 Spring Boot 應用中啟用服務註冊發現：

```java
@SpringBootApplication
@EnableDiscoveryClient
public class NacosProviderApplication {
    public static void main(String[] args) {
        SpringApplication.run(NacosProviderApplication.class, args);
    }

    @RestController
    class EchoController {
        @RequestMapping(value = "/echo/{string}", method = RequestMethod.GET)
        public String echo(@PathVariable String string) {
            return "Hello Nacos Discovery " + string;
        }
    }
}
```

在 application.properties 中配置：

```properties
server.port=8070
spring.application.name=service-provider
spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848
```

服務消費者配置：

```java
@SpringBootApplication
@EnableDiscoveryClient
public class NacosConsumerApplication {
    @LoadBalanced
    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @RestController
    public class TestController {
        private final RestTemplate restTemplate;

        @Autowired
        public TestController(RestTemplate restTemplate) {
            this.restTemplate = restTemplate;
        }

        @RequestMapping(value = "/echo/{str}", method = RequestMethod.GET)
        public String echo(@PathVariable String str) {
            return restTemplate.getForObject("http://service-provider/echo/" + str, String.class);
        }
    }
}
```

## MCP 服務管理

Nacos 3.0 支援 MCP（Model Context Protocol）服務的註冊、發現和管理。

### MCP Server 自動註冊

#### 使用 Spring AI Alibaba Nacos MCP 框架

1. 引入依賴：

```xml
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-mcp-registry</artifactId>
    <version>1.0.0.3及以上版本</version>
</dependency>
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-mcp-server-webmvc</artifactId>
    <version>1.0.0及以上版本</version>
</dependency>
```

2. 配置自動註冊：

```yaml
spring:
  application:
    name: mcp-nacos-registry-example
  ai:
    mcp:
      server:
        name: webmvc-mcp-server
        version: 1.0.0
        type: SYNC
        instructions: "This mcp server provides time information tools and resources"
  alibaba:
    mcp:
      nacos:
        server-addr: 127.0.0.1:8848
        namespace: public
        username: nacos
        password: nacos
        register:
          enabled: true
```

#### 使用 Nacos MCP Wrapper Python

```python
pip install nacos-mcp-wrapper-python

from nacos_mcp_wrapper.server.nacos_mcp import NacosMCP
from nacos_mcp_wrapper.server.nacos_settings import NacosSettings

nacos_settings = NacosSettings()
nacos_settings.SERVER_ADDR = "127.0.0.1:8848"
nacos_settings.NAMESPACE = "public"
nacos_settings.USERNAME = "nacos"
nacos_settings.PASSWORD = "nacos"

mcp = NacosMCP("nacos-mcp-python", nacos_settings=nacos_settings, version="1.0.1", port=18001)

@mcp.tool()
def add(a: int, b: int) -> int:
    """Add two integers together"""
    return a + b

if __name__ == "__main__":
    mcp.run(transport="sse")
```

## 控制台使用

Nacos 提供 Web 控制台進行可視化管理：

- 控制台位址：http://127.0.0.1:8080/index.html
- 預設使用者名稱：nacos
- 預設密碼：nacos@demo（生產環境請及時修改）

控制台主要功能：

- 服務管理：查看、編輯、刪除已註冊的服務
- 配置管理：建立、修改、刪除、查詢配置
- 命名空間管理：建立隔離的命名空間
- 使用者管理：管理存取使用者和權限
- 叢集管理：查看叢集節點狀態
