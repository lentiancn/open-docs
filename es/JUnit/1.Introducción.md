# JUnit Introducción

JUnit es el framework de pruebas unitarias más popular para Java, utilizado para escribir y ejecutar pruebas repetibles. Es la base para desarrollar código Java de alta calidad y es ampliamente utilizado por millones de desarrolladores en todo el mundo.

## JUnit 4 vs JUnit 5

### Descripción de versiones

- **JUnit 4**: Lanzado en 2010, la última versión principal del framework de pruebas utilizado durante mucho tiempo por la comunidad Java
- **JUnit 5**: Lanzado en 2017, una renovación completa del framework JUnit, que consiste en tres subproyectos: JUnit Platform, JUnit Jupiter y JUnit Vintage

### Diferencias arquitectónicas

JUnit 4 utiliza una arquitectura monolítica con todas las funciones en una única biblioteca. JUnit 5 utiliza una arquitectura modular con tres proyectos separados:

- **JUnit Platform**: La base para lanzar frameworks de pruebas, proporcionando la API TestEngine y el lanzador de consola
- **JUnit Jupiter**: El nuevo modelo de programación y modelo de extensión para escribir pruebas y extensiones
- **JUnit Vintage**: El motor de compatibilidad que permite ejecutar pruebas JUnit 3 y JUnit 4 en la plataforma JUnit 5

### Diferencias principales

| Característica | JUnit 4 | JUnit 5 |
|-----------------|---------|---------|
| Versión mínima de Java | Java 5+ | Java 8+ |
| Paquete de anotaciones | org.junit | org.junit.jupiter.api |
| Clase de prueba sin herencia | No | Sí |
| Estilo de aserciones | Métodos estáticos Assert | Métodos estáticos Assertions |
| Pruebas parametrizadas | @Parameters | @ParameterizedTest |
| Pruebas anidadas | No soportado | Soportado |
| Pruebas dinámicas | No soportado | Soportado |
| Ejecución en paralelo | Soporte limitado | Soporte nativo |
| Modelo de extensión | @RunWith | @ExtendWith |

### Recomendaciones de versión

- **Proyectos nuevos**: Usar JUnit 5 para características modernas y mejor extensibilidad
- **Proyectos heredados**: Continuar usando JUnit 4, o usar JUnit Vintage para migración gradual
- **Mantenimiento a largo plazo**: Migrar gradualmente a JUnit 5 para mejor experiencia de pruebas
