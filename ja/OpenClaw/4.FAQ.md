# OpenClaw FAQ

> OpenClawに関するよくある質問

---

## 一般

### OpenClawとは何ですか？

OpenClawは、高度なエージェント技術を活用してタスクを自動化するAIアシスタントフレームワークです。ファイル管理、ブラウザ制御などのさまざまなツールを組み合わせています。

### OpenClawは誰に向いていますか？

OpenClawは、開発者、パワーユーザー、AIパワード自動化を通じて生産性を向上させたいすべての人々に適しています。

## インストール

### システム要件は何ですか？

| 要件 | 最小 |
|-----|-----|
| OS | Windows 10+、macOS、Linux |
| Node.js | バージョン18+ |
| RAM | 4GB |
| ストレージ | 500MB |

### インストールが失敗しますか？

**解決策**：
1. Node.jsバージョンを確認：`node --version`
2. npmキャッシュをクリア：`npm cache clean --force`
3. 管理者/ルート権限で再試行
4. エラーメッセージを記録

### Gatewayが起動しない？

**解決策**：
1. ポートを確認：`netstat -ano | findstr 3000`
2. 別のポートを使用：`openclaw gateway start --port 3001`
3. ログを確認：`openclaw gateway logs`

## 使用方法

### 会話を開始するには？

```bash
openclaw chat
```

### 単一のコマンドを実行するには？

```bash
openclaw exec "私のコマンド"
```

### OpenClawをヘッドレスで使用できますか？

はい、OpenClawはヘッドレスモードで動作できます。Gatewayサービスを起動し、APIまたはCLIを使用します。

## ファイル管理

### ファイルを読むには？

readツールを使用：
```
read /path/to/file.txt
```

### 新規ファイルを作成するには？

writeツールを使用：
```
write --content "content" --path /new/file.txt
```

### ファイルを編集するには？

正確な変更にはeditツールを使用：
```
edit --path /file.txt --oldText "old" --newText "new"
```

## ブラウザ制御

### サポートされているブラウザは？

- Chrome（拡張機能経由）
- 分離OpenClawブラウザ
- ブラウザ自動化によるその他

### ブラウザを制御するには？

```bash
browser start --profile openclaw
browser navigate --url https://example.com
browser screenshot --path /output.png
```

### ブラウザが起動しない？

**解決策**：
1. Chrome拡張機能をインストール
2. ファイアウォール設定を確認
3. ブラウザキャッシュをクリア

## サブエージェント

### サブエージェントとは何ですか？

サブエージェントは、特定のタスクを並行してまたは順次実行できる独立したインスタンスです。

### サブエージェントを作成するには？

```bash
subagents create --task "description"
```

### サブエージェントを終了するには？

```bash
subagents kill --id agent-id
```

## セキュリティ

### OpenClawは安全ですか？

OpenClawは複数のセキュリティレイヤーを実装しています：
- ファイル権限
- 外部アクションの確認
- サンドボックスオプション

### 機密データを保護するには？

1. 暗号化された設定を使用
2. 会話でパスワードを共有しない
3. シークレットには環境変数を使用

## トラブルシューティング

### "コマンドが見つかりません"？

確認：
1. OpenClawが正しくインストールされている
2. PATH変数が正しく設定されている
3. Gatewayが実行中

### ファイル操作が失敗する？

1. パスを確認
2. 権限を検証
3. ファイル名に特殊文字がないか確認

### 応答時間が遅い？

1. Gatewayステータスを確認
2. ネットワーク接続を検証
3. ログを分析
