# Руководство по установке LivePortrait

В этом документе представлены подробные инструкции по установке LivePortrait для различных операционных систем и сред.

## Требования

Перед установкой убедитесь, что ваша система соответствует следующим базовым требованиям:

**Операционная система**: Linux (рекомендуется), Windows 10/11, macOS (требуется Apple Silicon)

**Версия Python**: Python 3.10

**Аппаратные требования**: NVIDIA GPU (RTX 4090 или другая совместимая с CUDA видеокарта рекомендуется), минимум 8 ГБ видеопамяти

**Место на диске**: Минимум 20 ГБ рекомендуется для весов моделей и среды выполнения

## Базовая настройка среды

### Проверка версии CUDA

Сначала подтвердите версию CUDA вашей системы, чтобы установить соответствующую версию PyTorch:

```bash
nvcc -V
```

Распространенные версии CUDA включают 11.1, 11.8 и 12.1. В зависимости от отображаемого номера версии выберите соответствующую команду установки PyTorch.

### Установка PyTorch

Выберите соответствующую команду установки PyTorch в зависимости от версии CUDA:

**Для CUDA 11.1**:

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**Для CUDA 11.8**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**Для CUDA 12.1**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

Если вам нужно только запустить режим человека, вы можете пропустить проверку версии CUDA и установить последнюю версию PyTorch напрямую.

## Загрузка кода проекта

Клонируйте репозиторий LivePortrait с помощью Git:

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Создание среды Python

Рекомендуется создать изолированную среду Python с помощью conda:

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

Или используя venv:

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## Установка зависимостей

Выполните следующее в корневом каталоге проекта:

```bash
pip install -r requirements.txt
```

## Загрузка предварительно обученных моделей

Предварительно обученные модели можно загрузить из HuggingFace. Если ваша сеть имеет доступ к HuggingFace:

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Если вы не можете получить доступ к HuggingFace, используйте зеркальный сайт:

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Альтернативно, вы можете загрузить все файлы моделей из Google Drive или Baidu Netdisk, распаковать и поместить их в каталог ./pretrained_weights.

## Проверка установки

После установки запустите тестовый скрипт, чтобы убедиться, что среда настроена правильно:

```bash
python inference.py
```

Если все работает правильно, вы увидите сгенерированный файл вывода в каталоге animations.

## Дополнительные шаги установки для моделей животных

Режим животных поддерживается только в системах Linux и Windows с GPU NVIDIA.

### Установка зависимости X-Pose

Режим животных требует проект X-Pose как фреймворк для определения ключевых точек. Сначала перейдите в каталог X-Pose:

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### Загрузка моделей животных из HuggingFace

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Особые примечания для Windows

В системах Windows некоторые более высокие версии CUDA (например, 12.4, 12.6) могут вызывать проблемы совместимости. Рекомендуется понизить версию CUDA до версии 11.8 для лучшей стабильности.

Пользователи Windows также могут загрузить установщик в один клик и запустить его, дважды щелкнув run_windows.bat после распаковки:

```bash
# Загрузить установщик Windows в один клик из HuggingFace
# Адрес: https://huggingface.co/cleardusk/LivePortrait-Windows
```

## Особые примечания для macOS

### macOS с Apple Silicon

Для пользователей macOS с Apple Silicon используйте специальный файл зависимостей:

```bash
pip install -r requirements_macOS.txt
```

При запуске вывода установите переменную окружения:

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

Примечание: Запуск вывода на macOS с Apple Silicon может быть примерно в 20 раз медленнее, чем на RTX 4090.

### Mac на базе Intel

Mac на базе Intel официально не тестировались и могут работать неправильно.

### Ограничение режима животных

Поскольку зависимость X-Pose не поддерживает macOS, режим животных недоступен на macOS. Можно использовать только человеческий режим.

## Проверка структуры каталогов

После установки убедитесь, что структура каталогов соответствует следующему:

```
LivePortrait/
├── pretrained_weights/     # Каталог весов предварительно обученных моделей
├── assets/                # Каталог примеров материалов
│   ├── examples/
│   │   ├── source/        # Исходное изображение/видео
│   │   └── driving/       # Управляющее видео
├── src/                   # Каталог исходного кода
├── inference.py          # Скрипт вывода
├── app.py                # Интерфейс Gradio
└── requirements.txt      # Список зависимостей
```

## Распространенные проблемы

При возникновении проблем во время установки проверьте следующее:

**CUDA недоступна**: Подтвердите, что драйверы NVIDIA правильно установлены; запустите nvidia-smi, чтобы проверить состояние драйвера

**Недостаточно видеопамяти**: Попробуйте использовать меньшие входные изображения или уменьшить размер пакета

**Ошибка загрузки модели**: Проверьте подключение к сети или используйте отечественные зеркальные сайты

**Конфликты зависимостей**: Рекомендуется использовать новую среду Python, чтобы избежать конфликтов с другими проектами
