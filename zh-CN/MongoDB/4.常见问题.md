# MongoDB 常见问题

## 基础概念

### 什么是 MongoDB？

MongoDB 是一个文档数据库，旨在帮助开发者更快地构建现代应用程序。它将数据存储在类似 JSON 的灵活文档中，从而可以像应用程序代码使用数据一样轻松地对数据进行建模。MongoDB 提供强大的查询引擎、水平扩展和内置高可用性。

### MongoDB 是关系数据库吗？

不是。MongoDB 是面向文档的 NoSQL 数据库，使用类似 JSON 的 BSON 格式存储数据。它与传统关系数据库有本质区别，不使用表、行和列的概念，而是使用集合和文档。

### MongoDB 中的文档是什么？

文档是 MongoDB 中的基本数据单元，由字段和值对组成。文档可以包含其他文档、数组和嵌套对象，结构灵活多样。每个文档都有一个唯一的 _id 字段作为主键。

### MongoDB 和 MySQL 有什么区别？

| 特性 | MongoDB | MySQL |
|------|---------|-------|
| 数据模型 | 文档型（类似 JSON） | 关系型（行列） |
| 模式 | 灵活（动态模式） | 固定（静态模式） |
| 扩展方式 | 水平扩展（分片） | 垂直扩展 |
| 查询语言 | MongoDB 查询语言 | SQL |
| 事务 | 多文档 ACID 事务 | 多表 ACID 事务 |
| 关联 | 嵌入式文档/数组 | JOIN 操作 |
| 索引 | 支持多种索引类型 | 支持索引 |

### 什么是 BSON？

BSON 是 Binary JSON 的缩写，是 JSON 文档的二进制表示形式，有额外的数据类型支持（如 ObjectId、Date、Binary 等）。

### MongoDB 的最大文档大小是多少？

BSON 文档大小的上限为 16 MB。对于超过此限制的文件，MongoDB 提供 GridFS API。

### MongoDB 支持哪些数据类型？

MongoDB 支持多种 BSON 数据类型，包括：Double、String、Object、Array、Binary Data、ObjectId、Boolean、Date、Null、Regular Expression、JavaScript、JavaScript with Scope、32-bit Integer、Timestamp、64-bit Integer、Decimal128、Min Key、Max Key 等。

## 安装问题

### MongoDB 需要多少内存？

MongoDB 会尽量使用可用内存作为缓存，官方建议至少 4GB 内存用于开发环境，生产环境根据数据量和负载具体评估。对于大型生产环境，通常建议更多内存以获得最佳性能。

### MongoDB 支持 32 位系统吗？

不支持。MongoDB 从某个版本开始停止支持 32 位平台，请使用 64 位操作系统。

### 如何更改 MongoDB 默认端口？

MongoDB 默认端口是 27017。可以通过以下方式更改：

1. 修改配置文件中的 `net.port`
2. 在启动时指定 `--port` 参数

### 多个 MongoDB 实例可以运行在同一台机器上吗？

可以，但需要配置不同的：

- 端口号
- 数据目录（`--dbpath`）
- 日志文件路径
- 配置文件

### MongoDB 数据存储在哪里？

默认情况下：

- Linux/macOS：`/data/db`
- Windows：`C:\data\db`

可以通过配置文件中的 `storage.dbPath` 指定其他路径。

### 安装 MongoDB 8.0 还是 7.0？

- 新项目：推荐使用最新的稳定版本 MongoDB 8.2
- 生产环境：可以选择最新的稳定版本或长期支持版本 MongoDB 7.0

MongoDB 6.0 已于 2025年7月31日停止支持，不再接收安全更新。

## 使用问题

### 如何备份 MongoDB 数据？

使用 mongodump 工具：

```bash
# 导出整个数据库
mongodump --db=mydatabase --out=/backup

# 导出指定集合
mongodump --db=mydatabase --collection=users --out=/backup
```

恢复使用 mongorestore：

```bash
mongorestore --db=mydatabase /backup/mydatabase
```

### MongoDB 如何实现高可用？

通过副本集（Replica Set）实现高可用。副本集由主节点和多个从节点组成：

- 主节点处理所有写入操作
- 从节点复制主节点的数据
- 主节点故障时自动选举新的主节点

### 什么是分片？

分片是 MongoDB 水平扩展的方式，将数据分布到多个服务器上。每个分片包含数据的一部分，共同组成完整的数据集。分片允许您构建非常大的数据集和高吞吐量操作。

### 副本集和分片集群有什么区别？

- **副本集**：多个 MongoDB 实例维护相同的数据副本，提供高可用性和读取扩展
- **分片集群**：数据分布在多个服务器上，提供水平扩展能力，支持大规模数据集

### 如何优化查询性能？

1. 创建适当的索引：使用 `createIndex()` 为常用查询字段创建索引
2. 使用投影：只返回必要字段，减少网络传输
3. 避免全集合扫描：确保查询使用索引
4. 使用聚合管道：进行复杂的数据处理
5. 适当使用覆盖索引：索引包含所有查询字段
6. 限制返回结果：使用 `limit()` 和 `skip()` 进行分页

### MongoDB 支持事务吗？

支持。MongoDB 4.0+ 引入了多文档事务支持，跨副本集和分片集群都可以使用事务。事务遵循 ACID 语义：

- 原子性：事务中的所有操作要么全部成功，要么全部失败
- 一致性：事务前后数据库状态保持一致
- 隔离性：并发事务相互隔离
- 持久性：事务提交后，数据持久保存

### 什么是变更流？

变更流是 MongoDB 的一种功能，允许应用程序实时监听集合、数据库或整个集群的数据变更事件。应用程序可以订阅变更事件，实现实时同步、审计日志等功能。

### MongoDB 如何保证数据安全？

1. **身份认证**：启用用户名密码认证或 LDAP/Kerberos（企业版）
2. **访问控制**：使用基于角色的访问控制（RBAC）
3. **加密通信**：启用 TLS/SSL 加密客户端和服务器之间的通信
4. **加密存储**：使用 MongoDB Enterprise 的加密存储引擎
5. **字段级加密**：对敏感字段进行加密
6. **审计日志**：记录数据库操作（企业版）
7. **定期备份**：定期备份数据

### MongoDB 企业版和社区版有什么区别？

企业版包含更多高级功能：

- LDAP 身份验证
- Kerberos 身份验证
- 审计
- 加密存储引擎
- 平面分析
- 技术支持

社区版功能对于大多数使用场景已经足够。

### 如何选择合适的版本？

- 新项目：推荐使用最新的稳定版本（MongoDB 8.2）
- 生产环境：可以选择最新的稳定版本或长期支持版本（MongoDB 7.0）
- 特定需求：根据应用兼容性和功能需求选择

### 什么是 GridFS？

GridFS 是 MongoDB 用于存储大文件的规范。它将大文件分割成多个块（chunk），每个块默认 255KB，存储在 `fs.chunks` 集合中，文件元数据存储在 `fs.files` 集合中。

### MongoDB 适合存储什么类型的数据？

- 内容管理系统数据（文章、评论、用户生成内容）
- 用户配置和画像数据
- 日志和事件数据
- 物联网传感器数据
- 实时分析数据
- 移动应用后端数据
- 电子商务产品目录
- 聊天消息和实时通信数据

### 什么是副本集 oplog？

oplog（操作日志）是副本集中的一个特殊集合，记录主节点上的所有写操作。从节点通过读取和应用 olog 中的操作来保持与主节点的数据同步。

### 如何监控 MongoDB 性能？

1. 使用 `db.serverStatus()` 查看服务器状态
2. 使用 `db.stats()` 查看数据库状态
3. 使用 Profiler 分析慢查询：`db.setProfilingLevel(1, 100)`
4. 使用 `db.currentOp()` 查看当前操作
5. 集成 Prometheus 和 Grafana 进行可视化监控
6. 使用 MongoDB Atlas 的内置监控功能（如果使用云服务）

### MongoDB 的复制延迟是什么？

复制延迟是指主节点上的操作与将该操作应用到从节点之间的延迟。少量延迟是可以接受的，但随着复制延迟增加，会出现严重问题。可以通过以下方式监控：

```bash
rs.status()
```

查看 `optimes.lastCommittedOpTime` 和主节点操作时间之间的差异。

### 什么是读偏好（Read Preference）？

读偏好决定了客户端从副本集的哪个节点读取数据：

- **primary**：从主节点读取（默认）
- **primaryPreferred**：优先从主节点，必要时从从节点
- **secondary**：只从从节点读取
- **secondaryPreferred**：优先从从节点，必要时从主节点
- **nearest**：从网络延迟最低的节点读取

### 什么是写关注（Write Concern）？

写关注指定写入操作的确认级别：

- **w: 1**：等待主节点确认（默认）
- **w: majority**：等待大多数副本集成员确认
- **w: "all"**：等待所有副本集成员确认
- **j: true**：等待写入到磁盘日志
- **wtimeout**：超时时间

### 什么是 capped collection？

capped collection 是固定大小的集合，类似于循环队列，当达到大小限制时，旧文档会自动被新文档替换。常用于存储日志、缓存等场景。

### MongoDB 支持哪些索引类型？

- 单字段索引
- 复合索引
- 多键索引（数组字段）
- 文本索引
- 地理空间索引（2dsphere、2d）
- 哈希索引
- 唯一索引
- 部分索引
- 稀疏索引
- TTL 索引

### 什么是 TTL 索引？

TTL（Time-To-Live）索引允许文档在指定时间后自动过期删除。常用于会话存储、缓存、临时数据等场景。

## 故障排除

### 连接被拒绝怎么办？

1. 检查 MongoDB 服务是否运行：`sudo systemctl status mongod`
2. 检查端口是否正确：默认 27017
3. 检查防火墙设置
4. 检查绑定 IP 配置

### 磁盘空间不足怎么办？

1. 清理旧数据
2. 删除不需要的集合
3. 使用 `db.repairDatabase()` 回收空间
4. 扩展磁盘空间

### 查询很慢怎么办？

1. 使用 `explain()` 分析查询计划
2. 创建适当的索引
3. 检查是否有慢查询：`db.getProfilingLevel()`
4. 考虑使用投影只返回必要字段

### 副本集选举失败怎么办？

1. 检查网络连接
2. 检查节点配置
3. 确保大多数节点可用
4. 查看日志：`rs.status()`

### 数据不一致怎么办？

1. 同步从节点
2. 重新初始化副本集
3. 从备份恢复数据

## 开发相关

### MongoDB 支持哪些编程语言？

MongoDB 提供几乎所有主流编程语言的驱动程序，包括：

- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP
- Scala
- Kotlin
- Swift

### 如何在应用中使用 MongoDB？

1. 安装对应语言的驱动程序
2. 建立连接
3. 执行 CRUD 操作

例如 Node.js：

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('mydb');
  const collection = database.collection('users');
  
  // 插入文档
  await collection.insertOne({ name: '张三', age: 30 });
  
  // 查询文档
  const user = await collection.findOne({ name: '张三' });
  
  await client.close();
}
```

### 什么是 MongoDB Atlas？

MongoDB Atlas 是 MongoDB 提供的云数据库服务，提供完全托管的 MongoDB 部署，包括：

- 自动备份和恢复
- 全球分布
- 实时性能监控
- 免运维管理
- 免费层级可供试用

### 如何进行数据建模？

MongoDB 数据建模原则：

1. 优先考虑文档结构而不是规范化
2. 使用嵌入式文档表示"包含"关系
3. 使用引用表示"多对多"关系
4. 根据查询模式设计集合和索引
5. 考虑数据增长和扩展性
