# Vue.js Introduction

Vue (prononcé /vjuː/, comme "view") est un framework JavaScript pour construire des interfaces utilisateur. Il s'appuie sur HTML, CSS et JavaScript standards et fournit un modèle de programmation déclaratif et basé sur les composants qui vous aide à développer efficacement des interfaces utilisateur de toute complexité.

## Fonctionnalités Principales

### Rendu Déclaratif
Vue étend HTML standard avec une syntaxe de template qui nous permet de décrire de manière déclarative la sortie HTML basée sur l'état JavaScript.

### Réactivité
Vue suit automatiquement les changements d'état JavaScript et met à jour efficacement le DOM lorsque des changements se produisent.

## Le Framework Progressif

Vue est un framework et un écosystème qui couvre la plupart des fonctionnalités courantes nécessaires au développement frontend. Le web est extrêmement diversifié - les choses que nous construisons sur le web peuvent varier considérablement en forme et en échelle. Avec cela à l'esprit, Vue est conçu pour être flexible et adoptable de manière incrémentielle. Selon votre cas d'utilisation, Vue peut être utilisé de différentes manières :

- Amélioration du HTML statique sans étape de compilation
- Intégration en tant que Web Components sur n'importe quelle page
- Application à Page Unique (SPA)
- Rendu côté serveur (SSR) / Fullstack
- Génération de sites statiques (SSG) / Jamstack
- Ciblage desktop, mobile, WebGL, et même le terminal

## Composants à Fichier Unique

Dans la plupart des projets Vue utilisant des outils de compilation, nous créons des composants Vue en utilisant un format de fichier similaire à HTML appelé Composant à Fichier Unique (également connu sous le nom de fichiers *.vue, abrégé en SFC). Un SFC Vue, comme son nom l'indique, encapsule la logique du composant (JavaScript), le template (HTML) et les styles (CSS) dans un seul fichier.

## Styles d'API

Les composants Vue peuvent être créés dans deux styles d'API différents : API Options et API Composition.

### API Options
Avec l'API Options, nous définissons la logique d'un composant en utilisant un objet d'options telles que data, methods et mounted. Les propriétés définies par les options sont exposées sur this à l'intérieur des fonctions, qui pointe vers l'instance du composant.

### API Composition
Avec l'API Composition, nous définissons la logique d'un composant en utilisant des fonctions d'API importées. Dans les SFC, l'API Composition est généralement utilisée avec `<script setup>`. L'attribut setup est un indicateur qui fait en sorte que Vue effectue des transformations au moment de la compilation qui nous permettent d'utiliser l'API Composition avec moins de code répétitif.
