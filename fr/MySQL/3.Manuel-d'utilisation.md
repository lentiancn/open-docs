# Manuel d'utilisation de MySQL

## Opérations de base de données

### Connexion à MySQL

Connexion locale :
```bash
mysql -u root -p
```

Connexion distante :
```bash
mysql -h hostname -u username -p database_name
```

### Créer une base de données

```sql
CREATE DATABASE nom_base_de_donnees;
```

Créer avec un jeu de caractères spécifique :
```sql
CREATE DATABASE nom_base_de_donnees CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### Afficher les bases de données

```sql
SHOW DATABASES;
```

### Sélectionner une base de données

```sql
USE nom_base_de_donnees;
```

### Supprimer une base de données

```sql
DROP DATABASE nom_base_de_donnees;
```

## Opérations sur les tables

### Créer une table

```sql
CREATE TABLE nom_table (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    cree_le TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    mis_a_jour_le TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Afficher la structure d'une table

```sql
DESCRIBE nom_table;
-- ou
SHOW COLUMNS FROM nom_table;
```

### Afficher toutes les tables

```sql
SHOW TABLES;
```

### Modifier la structure d'une table

Ajouter une colonne :
```sql
ALTER TABLE nom_table ADD COLUMN nouvelle_colonne VARCHAR(50);
```

Modifier une colonne :
```sql
ALTER TABLE nom_table MODIFY nom_colonne VARCHAR(200);
```

Supprimer une colonne :
```sql
ALTER TABLE nom_table DROP COLUMN nom_colonne;
```

### Supprimer une table

```sql
DROP TABLE nom_table;
```

## Opérations sur les données

### Insérer des données

Insertion simple :
```sql
INSERT INTO nom_table (colonne1, colonne2) VALUES ('valeur1', 'valeur2');
```

Insertions multiples :
```sql
INSERT INTO nom_table (colonne1, colonne2) VALUES 
('valeur1', 'valeur2'),
('valeur3', 'valeur4');
```

### Interroger les données

Requête de base :
```sql
SELECT * FROM nom_table;
```

Requête conditionnelle :
```sql
SELECT colonne1, colonne2 FROM nom_table WHERE condition;
```

Tri :
```sql
SELECT * FROM nom_table ORDER BY nom_colonne DESC;
```

Pagination :
```sql
SELECT * FROM nom_table LIMIT 10 OFFSET 20;
```

### Mettre à jour les données

```sql
UPDATE nom_table SET colonne1 = 'nouvelle_valeur' WHERE condition;
```

### Supprimer des données

```sql
DELETE FROM nom_table WHERE condition;
```

Vider la table :
```sql
TRUNCATE TABLE nom_table;
```

## Opérations sur les index

### Créer des index

Index normal :
```sql
CREATE INDEX nom_index ON nom_table (nom_colonne);
```

Index unique :
```sql
CREATE UNIQUE INDEX nom_index ON nom_table (nom_colonne);
```

Index composite :
```sql
CREATE INDEX nom_index ON nom_table (colonne1, colonne2);
```

### Afficher les index

```sql
SHOW INDEX FROM nom_table;
```

### Supprimer les index

```sql
DROP INDEX nom_index ON nom_table;
```

## Gestion des utilisateurs et des privilèges

### Créer un utilisateur

```sql
CREATE USER 'nom_utilisateur'@'localhost' IDENTIFIED BY 'mot_de_passe';
```

### Accorder des privilèges

```sql
GRANT SELECT, INSERT, UPDATE ON nom_base_de_donnees.* TO 'nom_utilisateur'@'localhost';
```

Accorder tous les privilèges :
```sql
GRANT ALL PRIVILEGES ON nom_base_de_donnees.* TO 'nom_utilisateur'@'localhost';
```

### Actualiser les privilèges

```sql
FLUSH PRIVILEGES;
```

### Afficher les privilèges d'un utilisateur

```sql
SHOW GRANTS FOR 'nom_utilisateur'@'localhost';
```

### Supprimer un utilisateur

```sql
DROP USER 'nom_utilisateur'@'localhost';
```

## Contrôle des transactions

### Démarrer une transaction

```sql
START TRANSACTION;
-- ou
BEGIN;
```

### Valider une transaction

```sql
COMMIT;
```

### Annuler une transaction

```sql
ROLLBACK;
```

### Définir des points de sauvegarde

```sql
SAVEPOINT nom_point_de_sauvegarde;
ROLLBACK TO SAVEPOINT nom_point_de_sauvegarde;
```

## Fonctions courantes

### Fonctions de chaîne

```sql
SELECT CONCAT('Bonjour', ' ', 'Monde');
SELECT UPPER('bonjour');
SELECT LOWER('BONJOUR');
SELECT SUBSTRING('Bonjour Monde', 1, 5);
SELECT LENGTH('Bonjour');
SELECT TRIM('  Bonjour  ');
```

### Fonctions numériques

```sql
SELECT ABS(-5);
SELECT CEIL(3.14);
SELECT FLOOR(3.14);
SELECT ROUND(3.14159, 2);
SELECT MOD(10, 3);
```

### Fonctions de date

```sql
SELECT NOW();
SELECT CURDATE();
SELECT CURTIME();
SELECT DATE('2024-01-01 10:30:00');
SELECT YEAR('2024-01-01');
SELECT MONTH('2024-01-01');
SELECT DAY('2024-01-01');
SELECT DATE_ADD('2024-01-01', INTERVAL 1 DAY);
SELECT DATEDIFF('2024-01-01', '2023-12-01');
```

### Fonctions d'agrégation

```sql
SELECT COUNT(*) FROM nom_table;
SELECT SUM(nom_colonne) FROM nom_table;
SELECT AVG(nom_colonne) FROM nom_table;
SELECT MAX(nom_colonne) FROM nom_table;
SELECT MIN(nom_colonne) FROM nom_table;
```

## Sauvegarde et restauration

### Sauvegarder une base de données

```bash
mysqldump -u root -p nom_base_de_donnees > backup.sql
```

Sauvegarder toutes les bases de données :
```bash
mysqldump -u root -p --all-databases > toutes_les_bases.sql
```

### Restaurer une base de données

```bash
mysql -u root -p nom_base_de_donnees < backup.sql
```

### Exporter les résultats d'une requête

```bash
mysql -u root -p -e "SELECT * FROM nom_table" nom_base_de_donnees > sortie.tsv
```

## Optimisation des performances

### Analyser les requêtes avec EXPLAIN

```sql
EXPLAIN SELECT * FROM nom_table WHERE condition;
```

### Conseils d'optimisation

1. Créer des index pour les conditions de requête courantes
2. Éviter SELECT *, ne requêter que les colonnes nécessaires
3. Utiliser LIMIT pour restreindre les ensembles de résultats
4. Éviter d'utiliser des fonctions sur les colonnes indexées
5. Utiliser correctement les JOIN, éviter les produits cartésiens
6. Analyser et optimiser régulièrement les tables

### Afficher le journal des requêtes lentes

```sql
SHOW VARIABLES LIKE 'slow_query_log%';
```

## Gestion des erreurs courantes

### Erreurs de connexion
Vérifiez que le service MySQL fonctionne, que le port est correct et que les privilèges de l'utilisateur sont correctement configurés.

### Erreurs de jeu de caractères
Assurez-vous que la base de données, la table et la connexion utilisent le même jeu de caractères (utf8mb4 recommandé).

### Erreurs de privilèges
Confirmez que l'utilisateur dispose des privilèges appropriés pour les opérations sur la base de données et la table.
