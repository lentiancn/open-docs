# Nginx FAQ

## 설치 문제

### Nginx 설치가 실패하면 어떻게 해야 합니까?

1. 시스템 종속성이 완전히 설치되어 있는지 확인
2. 오류 로그를 확인하여 구체적인 실패 원인 파악
3. 충분한 디스크 공간이 있는지 확인
4. 권한 문제 확인

### Nginx 버전을 확인하는 방법은?

```bash
nginx -v
nginx -V  # 컴파일 매개변수를 포함한 상세 정보
```

### Nginx는 어떤 운영 체제를 지원합니까?

Nginx는 Linux, FreeBSD, Solaris, AIX, HP-UX, macOS 및 Windows를 지원합니다.

## 구성 문제

### Nginx 구성이 올바른지 확인하는 방법은?

```bash
nginx -t
```

### 구성 파일은 어디에 있습니까?

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- 컴파일 설치: `/usr/local/nginx/conf/nginx.conf`

### 구성을 변경한 후 재시작이 필요합니까?

아니요. `nginx -s reload`를 사용하여 구성을 다시 로드하세요.

## 성능 문제

### Nginx 성능을 최적화하는 방법은?

1. **워커 프로세스 수 조정**

```nginx
worker_processes auto;
```

2. **워커 연결 수 증가**

```nginx
events {
    worker_connections 4096;
}
```

3. **gzip 압축 활성화**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **브라우저 캐싱 활성화**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **연결 풀링 사용**

```nginx
upstream backend {
    server backend.example.co.kr;
    keepalive 32;
}
```

### Nginx가 너무 많은 메모리를 사용하면 어떻게 해야 합니까?

1. worker_connections 수 감소
2. 요청 본문 크기 제한
3. 캐시 설정 최적화
4. 메모리 누수 확인

### 높은 동시성을 처리하는 방법은?

1. 로드 밸런싱 사용
2. 캐싱 활성화
3. 커널 매개변수 조정
4. CDN 사용

##安全问题

### DDoS 공격을 방지하는 방법은?

1. **연결 제한**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **요청 속도 제한**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **느린 연결 닫기**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### SSL 인증서를 구성하는 방법은?

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### HTTPS를 강제하는 방법은?

```nginx
server {
    listen 80;
    server_name example.co.kr;
    return 301 https://$server_name$request_uri;
}
```

## 프록시 문제

### 리버스 프록시가 실패하면 어떻게 해야 합니까?

1. 백엔드 서비스가 실행 중인지 확인
2. 방화벽 설정 확인
3. proxy_pass 구성이 올바른지 확인
4. 오류 로그 확인

### WebSocket을 프록시하는 방법은?

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### 프록시 후 실제 IP를 얻는 방법은?

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## 로그 문제

### 로그 파일이 너무 크면 어떻게 해야 합니까?

logrotate 사용:

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### 액세스 로깅을 비활성화하는 방법은?

```nginx
access_log off;
```

### 로그 형식을 구성하는 방법은?

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## 로드 밸런싱 문제

### 로드 밸런서가 백엔드를 선택하는 방법은?

Nginx는 여러 로드 밸런싱 알고리즘을 지원합니다:

- **Round-Robin (기본)**: 요청을 순차적으로 분배
- **가중 Round-Robin**: 가중치에 따라 분배
- **IP 해시**: 동일한 IP의 요청을 같은 백엔드로 라우팅
- **Least Connections**: 최소 연결 수의 백엔드로 라우팅

### 백엔드 서버가 실패하면 어떻게 됩니까?

Nginx는 자동으로 실패한 노드를 건너뛰고 정상적인 노드로 요청을 라우팅합니다.

## 캐싱 문제

### 프록시 캐싱을 구성하는 방법은?

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### 캐싱이 작동하지 않으면 어떻게 해야 합니까?

1. 캐시 경로 권한 확인
2. 응답 헤더가 캐싱을 허용하는지 확인
3. 캐시 키 구성 확인

### 캐시를 지우는 방법은?

```bash
rm -rf /var/cache/nginx/*
```

## 포트 문제

### 포트가 이미 사용 중이면 어떻게 해야 합니까?

```bash
# 포트 사용량 확인
netstat -tulpn | grep :80

# 다른 포트 사용
listen 8080;
```

### 여러 포트에서侦听하는 방법은?

```nginx

    listen 80;
    listen 8080;
    server_name example.co.kr;
}
```

## 일반적인 오류 코드

### 400 Bad Request

- 요청 헤더가 너무 큼: `client_header_buffer_size` 증가
- URL이 너무 김: 요청 경로 확인

### 403 Forbidden

- 인덱스 파일이 존재하지 않음
- 권한 부족: 파일 및 디렉토리 권한 확인

### 404 Not Found

- 파일 경로 오류
- SELinux 또는 AppArmor에 의해 차단됨

### 502 Bad Gateway

- 백엔드 서비스가 실행 중이지 않음
- FastCGI/PHP-FPM 구성 오류

### 504 Gateway Timeout

- 백엔드 응답 타임아웃
- `proxy_read_timeout` 증가

## 디버깅 팁

### 상세 오류 정보를 얻는 방법은?

1. 디버그 로깅 활성화:

```nginx
error_log /var/log/nginx/error.log debug;
```

2. 오류 로그 확인:

```bash
tail -f /var/log/nginx/error.log
```

### 구성이 적용되는지 테스트하는 방법은?

```bash
# 구성 테스트
nginx -t

# 전체 구성 표시
nginx -T
```

### 정규식을 디버그하는 방법은?

location 테스트 사용:

```nginx
location /test {
    access_log /var/log/nginx/test.log;
}
```

## 업그레이드 문제

### Nginx를 업그레이드하는 방법은?

1. 구성 파일 백업
2. Nginx 중지
3. 새 버전 설치
4. Nginx 시작
5. 기능 테스트

### 중단 없이 업그레이드하는 방법은?

```bash
# 새 버전 컴파일
make

# 업그레이드를 위해 USR2 신호 보내기
kill -s USR2 $(cat /var/run/nginx.pid)

# 이전 워커 프로세스优雅하게 종료
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
