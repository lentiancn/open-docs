# 使用手册

本手册详细介绍 Apifox 的核心功能和使用方法，包括项目管理、接口设计、调试、Mock、自动化测试等。

## 项目管理

### 创建项目

在 Apifox 中，项目是组织 API 的最大单位。

```bash
# 创建新项目步骤
1. 点击左侧"项目"列表旁边的"+"
2. 输入项目名称
3. 选择项目类型
4. 选择基础模板（可选）
5. 点击"创建"
```

### 项目设置

每个项目都有独立的设置选项：

- **基本信息**：项目名称、描述、图标
- **环境变量**：定义开发、测试、生产等不同环境的变量
- **认证**：设置全局认证方式
- **导出/导入**：项目数据的导入导出

### 环境管理

Apifox 支持多环境配置：

```bash
# 环境变量使用
在请求中使用 {{变量名}} 引用环境变量
例如：{{baseUrl}}/api/users

切换环境：点击顶部环境选择器选择不同环境
```

## 接口设计

### 创建接口

接口是 API 的具体定义：

```bash
# 创建接口步骤
1. 在项目中点击"+"选择"新建接口"
2. 填写基本信息：
   - 接口名称：用户列表
   - 请求路径：/api/users
   - 请求方法：GET
3. 定义请求参数
4. 定义响应结构
5. 保存
```

### 请求参数

支持以下类型的请求参数：

- **路径参数**：如 /api/users/:id 中的 id
- **查询参数**：URL 中的 ?name=test
- **请求头**：如 Authorization
- **请求体**：JSON、表单、XML 等格式

### 响应定义

为每个接口定义成功和失败的响应：

```json
{
  "成功响应 200": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com"
  },
  "错误响应 404": {
    "code": 404,
    "message": "用户不存在"
  }
}
```

### 接口列表管理

支持分组管理接口，可以按模块或功能进行分类：

- 拖拽调整顺序
- 批量操作（批量删除、批量导出等）
- 搜索和筛选

## 接口调试

### 发送请求

在接口详情页面，点击"发送"按钮即可发送请求：

```bash
# 基本调试步骤
1. 打开接口
2. 选择环境
3. 设置参数值
4. 点击"发送"
5. 查看响应
```

### 认证设置

支持多种认证方式：

- **None**：无认证
- **Bearer Token**：JWT 认证
- **Basic Auth**：基本认证
- **OAuth 2.0**：OAuth2 认证
- **自定义认证**：根据需要自定义认证头

### 请求前置处理

可以设置请求发送前的脚本：

```javascript
// 示例：添加时间戳
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// 示例：生成随机 ID
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### 响应后置处理

可以设置收到响应后的处理脚本：

```javascript
// 示例：提取响应数据
const response = pm.response.json();
console.log('响应数据:', response);

// 示例：设置环境变量
pm.environment.set('access_token', response.data.token);
```

## API Mock

### 智能 Mock

Apifox 最强大的功能之一，只需定义数据结构，系统自动生成 Mock 数据：

```bash
# 开启智能 Mock
1. 打开接口
2. 点击"Mock"标签
3. 开启"智能 Mock"
4. 保存
```

### Mock 规则

支持自定义 Mock 规则：

- `@string`：生成随机字符串
- `@integer`：生成随机整数
- `@float`：生成随机浮点数
- `@boolean`：生成随机布尔值
- `@date`：生成随机日期
- `@email`：生成随机邮箱
- `@url`：生成随机 URL
- `@guid`：生成 UUID

### 示例

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

生成的 Mock 数据：

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock 服务器

Apifox 提供内置 Mock 服务器：

```bash
# 获取 Mock 地址
在接口详情页面，复制 Mock URL
例如：https://mock.apifox.com/api/users
```

## 自动化测试

### 创建测试用例

```bash
# 创建测试步骤
1. 打开接口
2. 点击"测试"标签
3. 点击"创建测试"
4. 设置测试用例名称
5. 添加断言
6. 保存
```

### 断言类型

支持多种断言：

- **状态码断言**：检查响应状态码
- **JSON 断言**：检查响应 JSON 数据
- **响应时间断言**：检查响应时间
- **响应头断言**：检查响应头

### 示例

```javascript
// 状态码断言
pm.test("状态码为 200", function() {
    pm.response.to.have.status(200);
});

// JSON 断言
pm.test("返回用户列表", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// 响应时间断言
pm.test("响应时间小于 500ms", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### 测试套件

可以将多个接口的测试用例组织成测试套件：

1. 创建测试套件
2. 添加测试用例
3. 设置执行顺序
4. 运行套件

### 生成测试报告

执行测试后，可以生成详细的测试报告：

- 通过率统计
- 失败用例分析
- 响应时间统计

## 团队协作

### 邀请成员

```bash
# 邀请团队成员
1. 进入项目设置
2. 点击"成员管理"
3. 点击"邀请成员"
4. 输入成员邮箱
5. 设置角色权限
6. 发送邀请
```

### 角色权限

- **管理员**：项目全部权限
- **编辑者**：可以编辑接口和运行测试
- **查看者**：只能查看，不能编辑

### 实时同步

团队成员登录后，项目数据会自动实时同步：

- 接口变更自动同步
- 测试结果自动同步
- 团队成员可以看到最新内容

### 评论功能

可以在接口上添加评论：

1. 打开接口
2. 点击"评论"
3. 输入评论内容
4. @提及团队成员

**导出：**

## 快捷键

常用快捷键：

- `Ctrl/Cmd + S`：保存
- `Ctrl/Cmd + Enter`：发送请求
- `Ctrl/Cmd + B`：格式化 JSON
- `Ctrl/Cmd + /`：注释/取消注释

## 最佳实践

1. **规范命名**：接口名称清晰描述功能
2. **完整注释**：为每个参数添加说明
3. **合理分组**：按模块组织接口
4. **使用环境**：区分开发、测试、生产环境
5. **编写测试**：为重要接口编写自动化测试
6. **定期同步**：保持本地和云端数据一致
