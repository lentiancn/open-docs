# Часто задаваемые вопросы о OpenClaw

Эта страница содержит распространённые вопросы и решения при использовании OpenClaw.

## Установка

### Какие требования для OpenClaw?

- Node.js 22.x или выше
- macOS, Linux или Windows (WSL2)
- Рекомендуется минимум 2 ГБ оперативной памяти

### Установка не удалась. Что делать?

1. Проверьте версию Node.js: `node --version` (должна быть >= 22)
2. Проверьте подключение к сети
3. Попробуйте другие пакетные менеджеры: npm, pnpm или bun
4. Запустите `openclaw doctor` для диагностики

### Как обновить OpenClaw?

```bash
# Переключиться на стабильный канал
openclaw update --channel stable

# Затем перезапустить Gateway
openclaw gateway restart
```

## Gateway

### Gateway не запускается

- Проверьте, не занят ли порт 18789
- Проверьте логи: `openclaw gateway --verbose`
- Попробуйте завершить существующие процессы

### Как запускать Gateway постоянно?

Используйте `--install-daemon` для установки как системная служба:

```bash
openclaw onboard --install-daemon
```

### Как получить удалённый доступ к Gateway?

1. **Tailscale**: Используйте `gateway.tailscale.mode: "serve"` или `"funnel"`
2. **SSH-туннель**: Установите SSH-туннель

## Каналы

### Ошибка входа в WhatsApp

- Убедитесь, что выполнили спаривание устройства с `openclaw channels login`
- Проверьте, может ли сетьreach WhatsApp-серверы

### Telegram-бот не работает

- Проверьте правильность токена бота
- Проверьте, начали ли вы диалог с ботом
- Проверьте настройку `allowFrom`

### Сообщения Discord не отвечают

- Проверьте, присоединился ли бот к серверу
- Подтвердите правильность настроенных разрешений
- Проверьте, находится ли канал в списке разрешённых

## Безопасность

### Могут ли незнакомцы отправлять мне сообщения?

По умолчанию OpenClaw требует проверку спаривания для неизвестных отправителей. Используйте `dmPolicy: "open"` для открытого режима, но не рекомендуется.

### Как управлять спариванием?

```bash
# Просмотр списка спариваний
openclaw pairing list

# Одобрение спаривания
openclaw pairing approve

# Отзыв спаривания
openclaw pairing revoke
```

### Запуск `openclaw doctor` показывает предупреждения?

Следуйте инструкциям по исправлению. Распространённые проблемы:
- Слишком открытая политика DM
- Отсутствует необходимая настройка аутентификации
- Риски открытия портов

## Навыки

### Как установить новые навыки?

```bash
# Установить через ClawHub
openclaw skill install название-навыка

# Или вручную создать каталог навыков и SKILL.md
```

### Навыки не работают?

- Проверьте правильность настройки навыка
- Проверьте логи Gateway
- Подтвердите установку зависимостей навыка

## Голос

### Voice Wake не работает?

- macOS: Проверьте разрешения специальных возможностей в Системных настройках
- iOS: Подтвердите включение разрешения микрофона
- Проверьте правильность настройки кодового слова

### Голосовой ответ слишком медленный?

- Проверьте задержку сети
- Рассмотрите использование локального TTS
- Настройте параметры голосовой модели

## Производительность

### OpenClaw использует слишком много памяти?

- Регулярно очищайте историю сеансов
- Настройте параметры модели
- Используйте сжатие сеансов

### Сообщения отвечают медленно?

- Проверьте подключение к сети
- Попробуйте переключиться на более быструю модель
- Проверьте нагрузку на Gateway

## Данные

### Где хранятся данные?

- Конфигурация: `~/.openclaw/`
- Рабочее пространство: `~/.openclaw/workspace`
- Учётные данные: `~/.openclaw/credentials`

### Как создать резервную копию данных?

```bash
# Резервная копия всего каталога конфигурации
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### Как сбросить все настройки?

1. Остановите Gateway
2. Удалите каталог `~/.openclaw`
3. Снова выполните `openclaw onboard`

## Другое

### Как посмотреть логи?

```bash
# Просмотр логов Gateway
openclaw gateway --verbose

# Или проверьте конкретные файлы логов
openclaw logs
```

### Как получить помощь?

- Документация: https://docs.openclaw.ai
- Присоединитесь к сообществу Discord
- Создайте Issue на GitHub

### В чём разница между OpenClaw и другими ИИ-ассистентами?

- **Полностью локально**: Ваши данные остаются локальными, не загружаются в облако
- **Открытый исходный код**: Код полностью открыт, вы можете проверить его сами
- **Высокая настраиваемость**: Вы можете изменять и расширять любую функцию
- **Мультиканальное объединение**: Один ассистент управляет всеми каналами связи

### Можно ли запустить несколько экземпляров OpenClaw одновременно?

Да, для каждого экземпляра нужно:
- Разный порт
- Независимое рабочее пространство
- Разная конфигурация
