# OpenClaw 설치 가이드

이 가이드에서는 다양한 운영 체제에 OpenClaw를 설치하는 세부 단계를 제공합니다.

## 요구 사항

- **Node.js**: 버전 22 이상
- **운영 체제**: macOS, Linux 또는 Windows
- **선택 사항**: Docker(샌드박스 기능용)

## 설치 방법

### macOS / Linux (권장)

OpenClaw를 설치하는 가장 쉬운 방법은 설치 스크립트를 사용하는 것입니다:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

이 스크립트는 시스템 환경을 자동으로 감지하고 필요한 종속성을 설치한 후 OpenClaw를 구성합니다.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### npm 직접 설치

Node.js가 이미 설치되어 있는 경우 npm을 직접 사용할 수도 있습니다:

```bash
npm install -g openclaw@latest
```

### 소스에서 설치

최신 개발 버전을 원하거나 사용자 정의 설치가 필요한 경우:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 설치 후 설정

### 온보딩 마법사 실행

설치 후 마법사를 실행하여 인증, Gateway 설정 및 선택적 채널을 구성합니다:

```bash
openclaw onboard --install-daemon
```

마법사가 안내하는 내용:
- API 키 구성
- Gateway 포트 설정
- 채널 연결 (WhatsApp, Telegram, Discord 등)
- 보안 및 권한 설정

### Gateway 상태 확인

위와 같이 서비스를 설치했다면 Gateway가 이미 실행 중이어야 합니다:

```bash
openclaw gateway status
```

### Gateway 전경 실행

테스트 또는 문제 해결에 유용한 전경에서 Gateway를 실행하려면:

```bash
openclaw gateway --port 18789
```

### 제어 UI 열기

```bash
openclaw dashboard
```

또는 브라우저에서 직접 열기: `http://127.0.0.1:18789/`

## 환경 변수

구성 또는 상태 디렉토리 위치를 사용자 정의해야 하는 경우 다음 환경 변수를 사용할 수 있습니다:

- `OPENCLAW_HOME`: 내부 경로 해결에 사용될 홈 디렉토리 설정
- `OPENCLAW_STATE_DIR`: 상태 디렉토리 재정의
- `OPENCLAW_CONFIG_PATH`: 구성 파일 경로 재정의

## 설치 확인

설치 후 다음을 수행할 수 있습니다:

1. `openclaw --version`을 실행하여 버전 확인
2. `openclaw doctor`를 실행하여 시스템 상태 확인
3. 제어 UI를 열어 테스트 메시지 전송

## 일반적인 설치 문제

### Node.js 버전 확인

```bash
node --version
```

버전이 22 이상인지 확인하세요. 그렇지 않으면 먼저 Node.js를 업그레이드하세요.

### Windows에서 Git을 찾을 수 없음

Git이 설치되어 시스템 PATH에 추가되어 있는지 확인하세요. [git-scm.com](https://git-scm.com)에서 다운로드할 수 있습니다.

### 권한 오류

설치 중 권한 오류가 발생하면 다음을 시도하세요:

```bash
sudo npm install -g openclaw@latest
```
