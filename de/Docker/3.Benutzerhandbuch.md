# Docker User Manual

This manual will help you get started quickly with Docker and master common operations for daily use.

## Prerequisites

Before you begin, make sure you have:

- Docker installed (Docker Desktop or Docker Engine)
- Command line terminal access
- Basic command line operation knowledge

## Basic Concepts Review

### Images and Containers

- **Image**: A read-only template used to create containers
- **Container**: A running instance of an image

### Common Commands

| Command | Function |
|---------|----------|
| `docker run` | Create and run a container |
| `docker ps` | View running containers |
| `docker images` | View local images |
| `docker pull` | Pull an image |
| `docker build` | Build an image |
| `docker stop` | Stop a container |
| `docker rm` | Remove a container |
| `docker rmi` | Remove an image |

## Your First Container

### Running Hello World

The simplest test is to run the hello-world image:

```bash
docker run hello-world
```

This command will:
1. Check if hello-world image exists locally
2. If not, pull from Docker Hub
3. Create and run a container
4. Display a welcome message and exit

### Running an Interactive Container

Run an interactive Ubuntu container:

```bash
docker run -it ubuntu bash
```

Parameter explanation:
- `-i`: Interactive mode
- `-t`: Allocate a pseudo-terminal

Inside the container, you can operate like using a normal Linux system. Type `exit` to leave the container.

### Running Containers in Background

Run a container in background mode:

```bash
docker docker run -d nginx
```

Parameter explanation:
- `-d`: Run in background (detached mode)

View running containers:

```bash
docker ps
```

Stop a background container:

```bash
docker stop <container ID or name>
```

## Container Management

### Viewing Containers

View running containers:

```bash
docker ps
```

View all containers (including stopped):

```bash
docker ps -a
```

View container details:

```bash
docker inspect <container name>
```

### Starting and Stopping Containers

Start a stopped container:

```bash
docker start <container name>
```

Stop a running container:

```bash
docker stop <container name>
```

Restart a container:

```bash
docker restart <container name>
```

### Removing Containers

Remove a stopped container:

```bash
docker rm <container name>
```

Force remove a running container:

```bash
docker rm -f <container name>
```

Remove all stopped containers:

```bash
docker container prune
```

Remove a local image:

```bash

```

```bash

```

Remove unused images:

```bash

```

## Port Mapping

Containers are isolated by default. You need to map ports to access services inside containers.

### Mapping Ports

Map host port 8080 to container port 80:

```bash
docker run -d -p 8080:80 nginx
```

Now visiting http://localhost:8080 will show the Nginx welcome page.

### Common Port Mapping Formats

- `-p 8080:80`: Map host 8080 to container 80
- `-p 127.0.0.1:8080:80`: Host-local access only
- `-p 8080-8090:80-90`: Port range mapping

## Volumes

Volumes are used to persist data generated by containers, or share host files with containers.

### Creating a Volume

```bash
docker volume create mydata
```

### Using a Volume

```bash
docker run -d -v mydata:/app/data nginx
```

### Mounting Host Directories

```bash
docker run -d -v /home/user/data:/app/data nginx
```

### Viewing Volumes

```bash
docker volume ls
docker volume inspect mydata
```

### Removing Volumes

```bash
docker volume rm mydata
docker volume prune  # Remove unused volumes
```

## Building Images

### Dockerfile Basics

Create a simple Dockerfile:

```dockerfile
# Specify base image
FROM node:18-alpine

# Set working directory
WORKDIR /app

# Copy files
COPY package*.json ./

# Install dependencies
RUN npm install

# Copy source code
COPY . .

# Expose port
EXPOSE 3000

# Start command
CMD ["node", "index.js"]
```

### Building an Image

```bash
docker build -t myapp:latest .
```

Parameter explanation:
- `-t`: Specify image name and tag

### Using .dockerignore

Create a .dockerignore file to exclude files:

```
node_modules
npm-debug.log
.git
README.md
```

## Docker Compose

Docker Compose is used to define and run multi-container applications.

### docker-compose.yml Example

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: postgres:15
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=secret

volumes:
  db-data:
```

### Common Commands

Start all services:

```bash
docker-compose up
```

Run in background:

```bash
docker-compose up -d
```

Stop all services:

```bash
docker-compose down
```

View logs:

```bash
docker-compose logs -f
```

Rebuild and start:

```bash
docker-compose up --build
```

## Networking

### Viewing Networks

```bash
docker network ls
```

### Creating a Network

```bash
docker docker network create mynetwork
```

### Container Network Connection

Connect container to network:

```bash
docker network connect mynetwork mycontainer
```

### Container Communication

When using Docker Compose, containers can communicate with each other using service names. For example:

```yaml
services:
  web:
    build: .
    depends_on:
      - db
  db:
    image: postgres:15
```

In the web container, you can access the database using the hostname `db`.

## Common Techniques

### Enter a Running Container

```bash
docker exec -it <container name> bash
```

### View Container Logs

```bash
docker logs -f <container name>
```

### Copy Files

From container to host:

```bash
docker cp container:/path/in/container /local/path
```

From host to container:

```bash
docker cp /local/path container:/path/in/container
```

### View Resource Usage

```bash
docker stats
```

### View Container Processes

```bash
docker top <container name>
```

### Cleanup

Clean up stopped containers, unused images, and build cache:

```bash
docker system prune
```

Complete cleanup:

```bash
docker system prune -a
```

## Best Practices

### Image Build Optimization

1. **Reduce image layers**: Combine related commands
2. **Use multi-stage builds**: Reduce final image size
3. **Use .dockerignore properly**: Exclude unnecessary files
4. **Use specific version tags**: Avoid using latest tag

### Security Recommendations

1. **Don't run containers as root**
2. **Use official verified images**
3. **Regularly update base images**
4. **Use Docker secrets for sensitive information** (Swarm mode) or Kubernetes secrets

### Resource Limits

Set resource limits for containers:

```bash
docker run -d --memory="512m" --cpus="1.5" nginx
```

## Common Issues

### Docker Permission Issues

If you encounter permission errors, add current user to docker group:

```bash
sudo usermod -aG docker $USER
```

Then log out and log back in.

### Containers Exit Automatically

Containers will exit if there's no continuously running process. To keep running:

```bash
docker run -d nginx tail -f /dev/null
```

Or use a process manager like supervisor.

### Port Conflicts

If a port is already in use, modify the mapped port:

```bash
docker run -d -p 8081:80 nginx
```
