# Element UI 常見問題

本頁面列出 Element UI 使用過程中最常見的問題及解決方案。

## 安裝問題

### 安裝失敗

症狀：npm install 失敗

解決方案：

1. 檢查 Node.js 版本，確保 >= 8.0.0
2. 清除 npm 緩存：
   npm cache clean --force
3. 刪除 node_modules 和 package-lock.json 後重新安裝

### 按需導入不生效

症狀：打包後體積仍然很大

解決方案：

1. 確保 .babelrc 配置正確
2. 檢查 babel-plugin-component 是否正確安裝

## 使用問題

### 組件樣式不生效

症狀：組件樣式與預期不符

解決方案：

1. 確保已正確引入 CSS：
   import 'element-ui/lib/theme-chalk/index.css'

### 圖標不顯示

症狀：圖標組件顯示為空白或方塊

解決方案：

1. 引入圖標字體：
   import 'element-ui/lib/theme-chalk/fonts/element-icons.ttf'

### 對話框問題

#### 對話框滾動問題

在對話框內容過多時，確保添加滾動：

```vue
<el-dialog>
  <div style="height: 400px; overflow-y: scroll;">
    內容
  </div>
</el-dialog>
```

## Vue 版本問題

### 與 Vue 3 不兼容

Element UI 僅支援 Vue 2。如果需要 Vue 3，請使用 Element Plus。

### 使用 babel6 問題

如果使用 babel6，確保正確配置：

```json
{
  "presets": ["es2015", "stage-2"],
  "plugins": ["transform-runtime"]
}
```

## 樣式問題

### 樣式衝突

症狀：自定義樣式被覆蓋

解決方案：

1. 使用更具體的選擇器
2. 使用 ::deep()

### 主題色不生效

檢查 SCSS 變量是否被正確覆蓋：

```scss
$--color-primary: #409eff;
```

## 效能問題

### 頁面卡頓

對於大數據量：

1. 使用分頁加載數據
2. 避免在表格中使用複雜組件

### 打包體積過大

1. 使用按需導入
2. 移除未使用的組件
