# OpenClaw 자주 묻는 질문

이 페이지는 OpenClaw 사용 시 일반적인 질문과 솔루션을 나열합니다.

## 설치

### OpenClaw의 요구 사항은?

- Node.js 22.x 이상
- macOS, Linux 또는 Windows (WSL2)
- 최소 2GB RAM 권장

### 설치 실패. 어떻게 해야 하나요?

1. Node.js 버전 확인: `node --version` (>= 22 필요)
2. 네트워크 연결 확인
3. 다양한 패키지 관리자 시도: npm, pnpm 또는 bun
4. 진단을 위해 `openclaw doctor` 실행

### OpenClaw를 어떻게 업데이트하나요?

```bash
# 안정 채널로 전환
openclaw update --channel stable

# Gateway 재시작
openclaw gateway restart
```

## Gateway

### Gateway가 시작 안 됨

- 포트 18789가 사용 중인지 확인
- 로그 확인: `openclaw gateway --verbose`
- 기존 프로세스 종료 시도

### Gateway를 계속 실행하게 하려면?

`--install-daemon`을 사용하여 시스템 서비스로 설치:

```bash
openclaw onboard --install-daemon
```

### Gateway에 원격으로 액세스하려면?

1. **Tailscale**: `gateway.tailscale.mode: "serve"` 또는 `"funnel"` 사용
2. **SSH 터널**: SSH 터널 연결 설정

## 채널

### WhatsApp 로그인 실패

- `openclaw channels login`으로 기기 페어링 확인
- 네트워크가 WhatsApp 서버에 연결할 수 있는지 확인

### Telegram Bot이 작동 안 함

- Bot 토큰이 올바른지 확인
- Bot과 대화 시작했는지 확인
- `allowFrom` 구성 확인

### Discord 메시지가 응답 없음

- Bot이 서버에 참가했는지 확인
- 올바른 권한이 구성되었는지 확인
- 채널이 허용 목록에 있는지 확인

## 보안

### 모르는 사람이 메시지를 보낼 수 있나요?

기본적으로 OpenClaw는 모르는 발신자에게 페어링 검증이 필요합니다. `dmPolicy: "open"`으로 공개 모드로 변경 가능 (권장 안 함).

### 페어링을 어떻게 관리하나요?

```bash
# 페어링 목록 보기
openclaw pairing list

# 페어링 승인
openclaw pairing approve

# 페어링 취소
openclaw pairing revoke
```

### `openclaw doctor`가 경고를 표시하면?

지침에 따라 수정하세요. 일반적인 문제:
- DM 정책이 너무 개방적
- 필요한 인증 구성 누락
- 포트 노출 위험

## 스킬

### 새 스킬을 어떻게 설치하나요?

```bash
# ClawHub를 통해 설치
openclaw skill install 스킬이름

# 또는 수동으로 스킬 디렉토리 및 SKILL.md 생성
```

### 스킬이 작동 안 하면?

- 스킬 구성이 올바른지 확인
- Gateway 로그 확인
- 스킬 종속성이 설치되었는지 확인

## 음성

### Voice Wake가 작동 안 하면?

- macOS: 시스템 환경설정에서 접근성 권한 확인
- iOS: 마이크 권한이 활성화되었는지 확인
- 깨우기 단어가 올바르게 설정되었는지 확인

### 음성 응답이 너무 느리면?

- 네트워크 지연 확인
- 로컬 TTS 사용 고려
- 음성 모델 설정 조정

## 성능

### OpenClaw가 너무 많은 메모리를 사용하면?

- 세션 기록 정기적으로 정리
- 모델 설정 조정
- 세션 압축 사용

### 메시지 응답이 느리면?

- 네트워크 연결 확인
- 더 빠른 모델로 전환 시도
- Gateway 부하 확인

## 데이터

### 데이터는 어디에 저장되나요?

- 구성: `~/.openclaw/`
- 워크스페이스: `~/.openclaw/workspace`
- 자격 증명: `~/.openclaw/credentials`

### 데이터를 어떻게 백업하나요?

```bash
# 전체 구성 디렉토리 백업
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### 모든 설정을 어떻게 재설정하나요?

1. Gateway 중지
2. `~/.openclaw` 디렉토리 삭제
3. `openclaw onboard` 다시 실행

## 기타

### OpenClaw와 다른 AI 어시스턴트의 차이점은?

- **완전히 로컬**: 데이터가 로컬에 유지, 클라우드에 업로드되지 않음
- **오픈소스**: 코드가 완전히 열려 있어 직접 검토 가능
- **높은 사용자 정의**: 모든 기능을 수정하고 확장 가능
- **멀티 채널 통합**: 하나의 어시스턴트가 모든 커뮤니케이션 채널 관리

### 여러 OpenClaw 인스턴스를 동시에 실행할 수 있나요?

가능합니다. 각 인스턴스에는 다음이 필요합니다:
- 다른 포트
- 독립적인 워크스페이스
- 다른 구성
