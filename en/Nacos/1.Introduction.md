# Nacos Introduction

Nacos (Dynamic Naming and Configuration Service) is an easy-to-use platform for service discovery and dynamic configuration management. It provides a comprehensive set of features including service discovery, configuration management, and DNS-based service registration.

## What is Nacos?

Nacos is an open-source project developed by Alibaba that helps developers build cloud-native applications with service discovery and configuration management capabilities.

### Key Features

- **Service Discovery** - Register and discover services dynamically
- **Configuration Management** - Centralized configuration with version control
- **Dynamic DNS** - DNS-based service discovery
- **Service Health Checks** - Monitor service health status
- **Service Metadata** - Store and manage service metadata
- **Dynamic Configuration Updates** - Push configuration changes in real-time
- **Multi-environment Support** - Manage configurations across environments

## Architecture

```
└─────────────────────────────────────┘
```

## Core Concepts

### Service

A service is a collection of instances with the same logical function. Each service has a unique identifier (serviceName).

### Instance

An instance is a single running process of a service. Each instance has metadata including IP, port, and health status.

### Cluster

A cluster is a group of instances that provide the same service. Instances in a cluster are typically deployed in the same data center.

### Namespace

Namespaces provide logical isolation for configurations and services. Different environments (dev, test, prod) can use different namespaces.

## Use Cases

### Service Discovery

- Load balancing across service instances

### Configuration Management

- Centralized configuration management
- Configuration versioning and rollback

### DNS-based Service Discovery

- DNS-based service resolution

## Quick Start

### Standalone Mode

```bash
# Download and extract
wget https://github.com/alibaba/nacos/releases/download/v2.3.0/nacos-server-2.3.0.tar.gz
tar -xvf nacos-server-2.3.0.tar.gz

# Start Nacos (standalone mode)
cd nacos/bin
./startup.sh -m standalone

# Access Nacos console
# http://localhost:8848/nacos
# Default username: nacos
# Default password: nacos
```

### Docker Mode

```bash
# Run Nacos in Docker
docker run --name nacos -d -p 8848:8848 -p 9848:9848 nacos/nacos-server:v2.3.0
```
