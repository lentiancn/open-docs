# Element Plus FAQ

This page lists the most common issues and solutions when using Element Plus.

## Installation Issues

### Installation Failed

**Symptoms**: npm install or yarn add fails

**Solutions**:

1. Check Node.js version, ensure >= 14.0.0
2. Clear npm cache:
   ```bash
   npm cache clean --force
   ```
3. Delete node_modules and package-lock.json, then reinstall

### TypeScript Type Errors

**Symptoms**: Type errors when importing components

**Solutions**:

1. Make sure Volar plugin is installed
2. Add to tsconfig.json:
   ```json
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }
   ```

## Usage Issues

### Component Styles Not Applied

**Symptoms**: Component styles don't match expectations

**Solutions**:

1. Make sure CSS is correctly imported:
   ```javascript
   import 'element-plus/dist/index.css'
   ```
2. Check for global style overrides
3. Use developer tools to inspect element styles

### Icons Not Displayed

**Symptoms**: Icon components show blank or squares

**Solutions**:

1. Install icon library:
   ```bash
   npm install @element-plus/icons-vue
   ```

2. Register icons globally:
   ```javascript
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }
   ```

3. Or import on demand:
   ```javascript
   import { Search, Edit } from '@element-plus/icons-vue'
   ```

### On-Demand Import Not Working

**Symptoms**: Bundle size still large after on-demand import

**Solutions**:

1. Check if plugin configuration is correct
2. Make sure correct resolver is used:
   ```javascript
   import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'
   ```
3. Check vite.config.ts or webpack.config.js configuration

### Internationalization Not Working

**Symptoms**: Component text doesn't change after language switch

**Solutions**:

1. Correctly import language pack:
   ```javascript
   import zhCn from 'element-plus/es/locale/lang/zh-cn'
   import en from 'element-plus/es/locale/lang/en'
   ```
2. Configure locale:
   ```javascript
   app.use(ElementPlus, {
     locale: zhCn,
   })
   ```

## Component Issues

### Table Issues

#### Table Column Width Adjustment Failed

Check for fixed column conflicts.

#### Table Sorting Not Working

Make sure prop attribute is set:

```vue
<el-table-column prop="date" label="Date" sortable />
```

#### Table Fixed Columns Misaligned

Make sure table has fixed height:

```vue
<el-table :data="data" height="300" />
```

### Form Issues

#### Form Validation Not Triggering

Make sure el-form-item has prop attribute:

```vue
<el-form-item label="Name" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>
```

#### Custom Validation Rules Not Working

Check validate method call:

```javascript
const validatePass = (rule, value, callback) => {
  if (value === '') {
    callback(new Error('Please enter password'))
  } else {
    callback()
  }
}
```

### Dialog Issues

#### Dialog Scrolling Issues

When dialog content is too long, ensure scrolling is added:

```vue
<el-dialog :close-on-click-modal="false">
  <div style="max-height: 60vh; overflow-y: auto;">
    <!-- Content -->
  </div>
</el-dialog>
```

### Menu Issues

#### Menu Click Not Responding

Make sure index attribute is set:

```vue
<el-menu-item index="1">Menu Item</el-menu-item>
```

## Style Issues

### Style Conflicts

**Symptoms**: Custom styles being overridden

**Solutions**:

1. Use more specific selectors
2. Use ::v-deep or :deep()
3. Use !important (not recommended)

### Theme Color Not Applied

Check if CSS variables are correctly overridden:

```css
:root {
  --el-color-primary: #409eff;
}
```

Make sure it loads after element-plus styles.

## Performance Issues

### Page Lag

For large datasets:

1. Use el-table-v2 (virtual table)
2. Use pagination to load data
3. Avoid using complex components in tables

### Bundle Size Too Large

1. Use on-demand import
2. Remove unused plugins
3. Use production build

## Vue Version Issues

### Not Compatible with Vue 2

Element Plus only supports Vue 3. If you need Vue 2, use Element UI.

### Composition API Not Supported

Make sure using Vue 3.2+ for better Composition API support.

## Other Issues

### Component Events Not Triggering

Check if correct event binding method is used:

```vue
<!-- Correct -->
<el-input @change="handleChange" />

<!-- Or -->
<el-input v-model="value" />
```

### Popup Layer Issues

If popup is blocked, adjust z-index:

```vue
<el-dialog :z-index="2000">
  <!-- Content -->
</el-dialog>
```

### Date Picker Language Not Changed After Internationalization

Need to configure internationalization for date picker:

```javascript
import dayjs from 'dayjs'
import 'dayjs/locale/zh-cn'

dayjs.locale('zh-cn')
```
