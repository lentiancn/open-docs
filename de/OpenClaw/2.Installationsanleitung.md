# OpenClaw Installationsanleitung

OpenClaw unterstützt mehrere Installationsmethoden. Sie können die für Ihre Bedürfnisse am besten geeignete wählen.

## Anforderungen

- **Node.js**: 22.x oder höher
- **Betriebssystem**: macOS, Linux, Windows (über WSL2)
- **Optional**: Tailscale (für Fernzugriff)

## Schnellinstallation

### macOS / Linux

Der einfachste Weg ist die Verwendung des Installationsskripts:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Oder mit npm/pnpm:

```bash
npm install -g openclaw@latest
# oder
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## Ersteinrichtung

Führen Sie nach der Installation den Onboarding-Assistenten aus:

```bash
openclaw onboard --install-daemon
```

Der Assistent führt Sie durch:

1. **Auth-Konfiguration**: API-Schlüssel oder OAuth einrichten
2. **Gateway-Einstellungen**: Gateway-Port und -Dienst konfigurieren
3. **Kanalverbindung**: Messaging-Plattformen verbinden
4. **Fähigkeiten-Installation**: Benötigte Fähigkeiten auswählen und installieren

## Als Systemdienst installieren

Verwenden Sie `--install-daemon`, um OpenClaw als Hintergrunddienst zu installieren:

```bash
openclaw onboard --install-daemon
```

Dies erstellt einen Benutzerdienst (launchd oder systemd), der sicherstellt, dass OpenClaw immer läuft.

## Gateway manuell ausführen

Wenn Sie den Dienst nicht installieren möchten:

```bash
openclaw gateway --port 18789 --verbose
```

## Docker-Installation

Wenn Sie Docker bevorzugen, lesen Sie die offizielle Docker-Deploy-Dokumentation.

## Aus dem Quellcode erstellen

Wenn Sie zur Entwicklung beitragen oder OpenClaw anpassen möchten:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

Hinweis: pnpm wird zum Erstellen verwendet. TypeScript kann direkt über tsx ausgeführt werden.

## Installation überprüfen

Nach der Installation können Sie:

1. **Gateway-Status prüfen**:
   ```bash
   openclaw gateway status
   ```

2. **Control UI öffnen**:
   ```bash
   openclaw dashboard
   ```
   Oder besuchen Sie `http://127.0.0.1:18789/` in Ihrem Browser

3. **Testnachricht senden** (erfordert Kanal-Einrichtung):
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## OpenClaw aktualisieren

OpenClaw hat drei Release-Kanäle:

- **stable**: Offizielle Releases (empfohlen)
- **beta**: Vorabversionen
- **dev**: Entwicklungs-Builds

Kanäle wechseln:

```bash
openclaw update --channel stable|beta|dev
```

Nach dem Aktualisieren empfiehlt es sich, auszuführen:

```bash
openclaw doctor
```

## Häufige Befehle

| Befehl | Beschreibung |
|--------|-------------|
| `openclaw onboard` | Einrichtungsassistenten ausführen |
| `openclaw gateway start` | Gateway starten |
| `openclaw gateway status` | Gateway-Status prüfen |
| `openclaw dashboard` | Control UI öffnen |
| `openclaw agent --message "..."` | Nachricht an Assistenten senden |
| `openclaw message send --to xxx --message "xxx"` | Nachricht senden |
| `openclaw doctor` | Probleme diagnostizieren und beheben |
| `openclaw update` | OpenClaw aktualisieren |

## Umgebungsvariablen

Für benutzerdefinierte Konfigurationsspeicherorte:

- `OPENCLAW_HOME`: Legt das Hauptverzeichnis fest
- `OPENCLAW_STATE_DIR`: Überschreibt das Statusverzeichnis
- `OPENCLAW_CONFIG_PATH`: Überschreibt den Konfigurationsdateipfad

## Fehlerbehebung

**Installation schlägt fehl?**

- Stellen Sie sicher, dass Node.js >= 22 ist
- Netzwerkverbindung prüfen
- `openclaw doctor` zur Diagnose ausführen

**Wie deinstallieren?**

- Gateway-Dienst stoppen
- Installationsverzeichnis entfernen
- Konfigurationsdateien bereinigen
