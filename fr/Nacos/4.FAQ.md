# Questions Fréquentes sur Nacos

## Questions de Démarrage Rapide

### Quels systèmes d'exploitation Nacos prend-il en charge ?

Nacos prend en charge les systèmes d'exploitation Linux, Unix, Mac et Windows. Linux/Unix/Mac est recommandé pour les environnements de production.

### Quelle version de Java Nacos nécessite-t-il ?

Nacos nécessite JDK 17 ou supérieur. Assurez-vous que JAVA_HOME est correctement configuré.

### Que faire si Nacos ne démarre pas ?

1. Vérifier l'environnement Java : Exécuter `java -version` pour confirmer la bonne version JDK
2. Vérifier l'utilisation des ports : Confirmer que les ports 8848 et 9848 ne sont pas occupés
3. Vérifier les logs : Examiner les messages d'erreur dans logs/start.out ou logs/nacos.log
4. Vérifier la configuration d'authentification : Nacos 3.0+ nécessite des paramètres d'authentification

### Comment changer le port par défaut de Nacos ?

Modifier la configuration suivante dans conf/application.properties :

```properties
server.port=8848
```

### Quelles sont les identifiants par défaut de la console Nacos ?

- Nom d'utilisateur par défaut : nacos
- Mot de passe par défaut : nacos@demo

Veuillez modifier le mot de passe par défaut dans les environnements de production.

## Questions sur la Découverte de Services

### Après un enregistrement de service réussi, pourquoi le client ne peut-il pas découvrir le service ?

1. Vérifier l'état de santé du service : S'assurer que l'état de l'instance est sain
2. Vérifier la connectivité réseau : S'assurer que le client peut accéder au serveur Nacos
3. Vérifier le nom du service : S'assurer que le nom de service demandé correspond à celui enregistré (y compris l'espace de noms et le groupe)
4. Vérifier la configuration client : Confirmer que la configuration nacos.server-addr est correcte

### Comment implémenter l'équilibrage de charge ?

Nacos fournit plusieurs stratégies d'équilibrage de charge :

1. Équilibrage de charge côté client : Utiliser la méthode selectOneHealthyInstance du SDK Nacos
2. Équilibrage de charge pondéré : Implémenter en définissant des poids d'instances
3. Équilibrage de charge de cluster : Implémenté automatiquement dans les environnements cluster

### Après qu'une instance de service est hors ligne, pourquoi le client appelle-t-il toujours l'instance hors ligne ?

1. Vérifier la configuration de vérification de santé : S'assurer que le mécanisme de vérification de santé fonctionne correctement
2. Vérifier le cache client : Les clients mettent en cache les listes d'instances de service avec un certain délai
3. Ajuster l'intervalle de synchronisation : Modifier l'intervalle de rafraîchissement du client

## Questions sur la Gestion de Configuration

### Après la publication d'une configuration, pourquoi le client ne reçoit-il pas de notifications de mise à jour ?

1. Vérifier l'écouteur : S'assurer que le client a correctement ajouté un écouteur de configuration
2. Vérifier le réseau : S'assurer que la connectivité réseau est normale
3. Vérifier les logs : Examiner les messages d'erreur dans les logs client
4. Confirmer dataId et group : S'assurer qu'ils correspondent exactement

### Comment réaliser la gestion de configuration multi-environnement ?

1. Utiliser Namespace : Différents environnements utilisent différents espaces de noms pour une isolation complète
2. Utiliser Group : Utiliser différents groupes sous le même espace de noms pour gérer les configurations
3. Utiliser DataId : Utiliser différents dataId pour distinguer les configurations des différents environnements

### Comment afficher les versions historiques de configuration ?

La console Nacos fournit l'historique des modifications de configuration, permettant de voir :

- Les enregistrements de modification de configuration
- La comparaison de contenu pour chaque modification
- Le rollback en un clic vers les versions historiques

### Comment implémenter le déploiement canary de configuration ?

Nacos prend en charge le déploiement canary de configuration :

1. Créer une configuration canary dans la console
2. Spécifier la portée du client canary
3. Vérifier l'effet de configuration canary
4. Publication complète ou rollback

## Questions sur le Déploiement en Cluster

### Combien de nœuds sont nécessaires pour le déploiement en cluster Nacos ?

Au moins 3 nœuds sont recommandés pour les environnements de production afin de garantir la haute disponibilité.

### Comment choisir le stockage de données en mode cluster ?

- Environnement de Test/Développement : Peut utiliser la base de données Derby intégrée
- Environnement de Production : Doit utiliser une base de données externe (MySQL) pour garantir la fiabilité des données

### Comment le cluster garantit-il la cohérence des données ?

Nacos utilise le protocole Distro développé en interne et le protocole Raft pour garantir la cohérence des données du cluster :

- Protocole Distro : Responsable de la synchronisation des données
- Protocole Raft : Responsable de l'élection du leader et de la réplication des logs

### Que se passe-t-il lorsqu'un nœud de cluster tombe en panne ?

Le cluster Nacos a des capacités de basculement automatique :

1. Les vérifications de santé détectent automatiquement les nœuds défaillants
2. Le trafic bascule automatiquement vers les nœuds sains
3. Les nœuds défaillants synchronisent automatiquement les données après la récupération

## Questions sur l'Authentification et la Sécurité

### Comment configurer l'authentification dans Nacos 3.0 ?

À partir de Nacos 3.0, l'authentification est activée par défaut. Les paramètres suivants doivent être configurés au démarrage :

```bash
# Clé secrète du jeton
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# Clé d'identité du service
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### Comment désactiver l'authentification Nacos ?

Modifier conf/application.properties :

```properties
nacos.core.auth.enabled=false
```

Note : Il n'est pas recommandé de désactiver l'authentification dans les environnements de production.

### Comment attribuer des permissions à différents utilisateurs ?

Nacos prend en charge le contrôle d'accès basé sur les rôles :

1. Créer des utilisateurs dans la console
2. Attribuer des rôles aux utilisateurs
3. Configurer les permissions des rôles

## Questions sur MCP

### Que faire si des problèmes de compatibilité se produisent lors du démarrage du Serveur MCP ?

S'assurer que les données du Serveur MCP enregistré automatiquement sont compatibles avec les données de version correspondantes dans Nacos :

- Le protocole du Serveur MCP (stdio, sse, streamable) est-il cohérent ?
- La référence du service est-elle cohérente ?
- Les données Tools sont-elles cohérentes (nom Tools, nombre Tools, types de paramètres, paramètres optionnels) ?

Note : Les descriptions Tools et les descriptions de paramètres n'ont pas besoin d'être cohérentes.

### Pourquoi le Serveur MCP enregistré ne peut-il pas être découvert ?

1. Vérifier l'état du Serveur MCP : S'assurer que le service est correctement enregistré
2. Vérifier l'espace de noms : S'assurer que le client et le service sont dans le même espace de noms
3. Vérifier les informations d'authentification : S'assurer que le nom d'utilisateur et le mot de passe sont corrects

### Comment modifier dynamiquement la configuration des outils du Serveur MCP ?

Après que le Serveur MCP est automatiquement enregistré sur Nacos, vous pouvez modifier dynamiquement les descriptions et commutateurs des Tools du Serveur MCP dans la console. Les modifications associées seront appliquées dynamiquement au Serveur MCP au runtime sans redémarrer le service.

## Questions sur la Performance et l'Optimisation

### Comment Nacos gère-t-il l'enregistrement de services à grande échelle ?

Nacos est né de l'échelle de millions d'instances d'Alibaba et a les capacités suivantes :

- Enregistrement et découverte de services haute performance
- Réponse de mise à jour de configuration à faible latence
- Gestion de services et configurations à grande échelle

### Comment optimiser la performance de Nacos ?

1. Définir correctement les intervalles de vérification de santé
2. Utiliser une configuration de pool de connexions de base de données appropriée
3. Activer la mise en cache côté client pour réduire la fréquence des requêtes
4. Planifier correctement les espaces de noms et les groupes

### Recommandations de configuration mémoire pour Nacos ?

Basé sur l'échelle des services :

- Petite échelle (services < 1000) : 2C4G
- Moyenne échelle (services 1000-5000) : 4C8G
- Grande échelle (services > 5000) : 8C16G+

## Questions sur l'Intégration de l'Écosystème

### Comment intégrer Nacos avec Spring Cloud ?

Ajouter des dépendances dans pom.xml :

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### Comment intégrer Nacos avec Kubernetes ?

Nacos prend en charge le déploiement dans les environnements Kubernetes en utilisant :

- Déploiement avec Helm Chart
- Déploiement avec Operator
- Déploiement avec YAML personnalisé

Nacos fournit des SDK pour les langages suivants :

- PHP

## Dépannage

### Comment dépanner les échecs d'enregistrement de services ?

1. Vérifier si le serveur Nacos fonctionne correctement
2. Vérifier la connectivité réseau
3. Examiner les logs client
4. Confirmer que les paramètres d'enregistrement sont corrects

### Comment dépanner les problèmes de récupération de configuration ?

1. Confirmer que la configuration a été publiée avec succès
2. Vérifier si dataId et group correspondent
3. Vérifier si les écouteurs sont correctement ajoutés
4. Examiner les logs client pour les erreurs spécifiques

### Où sont les logs Nacos ?

Les fichiers de logs Nacos sont situés dans :

- Log principal : logs/nacos.log
- Log de démarrage : logs/start.out
- Log d'accès : logs/access.log

## Mise à Niveau et Migration

### Comment mettre à niveau depuis Nacos 1.x vers 2.x ou 3.x ?

1. Sauvegarder les données : Sauvegarder la base de données ou les fichiers de instantanés
2. Lire le guide de mise à niveau : Consulter la documentation de mise à niveau de la version correspondante
3. Mise à niveau progressive : Tester d'abord dans l'environnement de préproduction
4. Migrer les configurations : Faire attention aux changements d'authentification et autres configurations

### Comment migrer les données Nacos ?

1. Exporter les données de configuration de l'environnement original
2. Importer dans la base de données du nouvel environnement
3. Vérifier l'intégrité des données
4. Basculer le trafic vers le nouvel environnement
