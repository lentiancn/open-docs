# Guía de Instalación

Jira ofrece múltiples opciones de instalación para satisfacer las necesidades de equipos de diferentes tamaños y capacidades técnicas. Esta guía proporciona pasos detallados para la implementación en la nube y Data Center local.

## Instalación de la Versión en la Nube

La versión en la nube es la forma más rápida de comenzar con Jira, sin necesidad de gestión de infraestructura.

### Registro de Cuenta

Primero, visita el sitio web de Jira y haz clic en el botón de registro gratuito. Ingresa tu correo electrónico profesional para crear una cuenta. Atlassian identificará automáticamente a los miembros del equipo y facilitará la colaboración.

### Selección del Plan

La versión en la nube ofrece cuatro niveles de precios:

**Plan Free** es permanentemente gratuito para hasta 10 usuarios. Incluye metas y proyectos ilimitados, listas de tareas y vistas de tablero, 100 reglas de automatización por mes y 2GB de almacenamiento.

**Plan Standard** cuesta $7.50 por usuario al mes. Incluye roles de usuario y permisos, colaboración externa, residencia de datos multiregión, 1,700 reglas de automatización por mes y 250GB de almacenamiento.

**Plan Premium** cuesta $14.50 por usuario al mes. Incluye planificación entre equipos y gestión de dependencias, procesos de aprobación personalizables, almacenamiento ilimitado, soporte 24/7 y SLA de disponibilidad del 99.9%.

**Plan Enterprise** está diseñado para grandes empresas. Incluye análisis avanzados y seguridad de nivel empresarial, soporte para múltiples sitios, reglas de automatización ilimitadas y SLA de disponibilidad del 99.95%.

### Creación del Sitio

Después del registro, el sistema te guiará para crear tu primer sitio de Jira. Puedes crear un sitio para tu organización o uso personal, y seleccionar plantillas de proyecto que se adapten a las necesidades de tu equipo.

### Adición de Usuarios

Después de crear un sitio, puedes agregar miembros del equipo a través del panel de administración. Los usuarios pueden ser gratuitos o pagos dependiendo del plan seleccionado.

## Implementación Data Center Local

Las empresas que requieren control estricto de datos o que tienen requisitos especiales de seguridad pueden elegir la versión Data Center local.

### Requisitos del Sistema

La versión Data Center requiere lo siguiente:

Soporte de sistema operativo incluye Linux (Red Hat Enterprise Linux, Ubuntu Server, CentOS, etc.) y Windows Server.

Soporte de base de datos incluye PostgreSQL, Oracle, Microsoft SQL Server y otras bases de datos empresariales.

Memoria: se recomienda al menos 8GB de RAM, 16GB o más para entornos de producción.

Espacio en disco: depende del volumen de datos, se recomienda al menos 100GB de espacio disponible.

### Pasos de Instalación

Primero, descarga el instalador de Data Center del sitio web de Atlassian. Ejecuta el instalador y sigue el asistente para seleccionar el directorio de instalación y configurar las conexiones de base de datos. Después de la instalación, accede a la interfaz de administración a través de un navegador para la configuración inicial.

### Migración de Datos

Si migras desde una versión anterior, usa las herramientas de importación y exportación de Jira para migrar los datos. Se recomienda hacer copia de seguridad de todos los datos antes de la migración y verificar los resultados en un entorno de prueba.

## Aplicaciones Móviles

Jira ofrece aplicaciones móviles para iOS y Android, permitiéndote ver y actualizar problemas en cualquier lugar. Las aplicaciones móviles soportan visualización de problemas, creación de comentarios, actualizaciones de estado y más, adecuadas para miembros del equipo que necesitan trabajar mientras se mueven.

## Configuración Inicial

Independientemente de la opción de implementación, la configuración inicial incluye:

Configurar la estructura del proyecto, crear proyectos y tableros adecuados para tu equipo. Configurar flujos de trabajo, definir estados de problemas y reglas de transición. Agregar miembros del equipo y establecer permisos. Integrar otras herramientas como Slack y GitHub. Establecer reglas de automatización para reducir operaciones manuales.
