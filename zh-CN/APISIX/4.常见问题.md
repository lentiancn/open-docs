# APISIX 常见问题

## 安装问题

### Q1: Docker 容器启动失败

**问题**：运行 `docker run` 后容器立即退出。

**解决**：
1. 检查 Docker 日志：`docker logs apisix`
2. 确认 etcd 是否正常运行
3. 检查端口是否被占用：`netstat -tlnp | grep 9080`
4. 尝试使用 docker-compose 方式启动

### Q2: Admin API 无法访问

**问题**：访问 `http://127.0.0.1:9180/apisix/admin` 返回 404 或 403。

**解决**：
1. 确认 `enable_admin: true` 在 `config.yaml` 中已设置
2. 检查 Admin API 密钥配置是否正确
3. 验证防火墙允许 9180 端口访问

### Q3: etcd 连接失败

**问题**：启动时报错 "etcd connection failed"。

**解决**：
1. 确认 etcd 服务正在运行
2. 检查 etcd 端口（默认 2379）是否可访问
3. 验证 etcd 配置中的 endpoints 地址
4. 如果使用 Docker，确认网络配置正确

## 配置问题

### Q4: 路由匹配失败

**问题**：请求返回 404，但路由配置看起来正确。

**解决**：
1. 检查路由的 `uri` 配置是否正确
2. 确认上游节点是否可用
3. 查看 APISIX 错误日志
4. 使用 `curl -v` 查看详细请求信息

### Q5: 插件不生效

**问题**：配置了插件但没有效果。

**解决**：
1. 确认插件名称正确
2. 检查插件是否在正确的作用域（route/plugin）配置
3. 验证插件配置格式正确
4. 检查插件依赖是否安装

### Q6: 负载均衡不生效

**问题**：请求没有按预期分配到不同节点。

**解决**：
1. 确认上游节点配置正确
2. 检查负载均衡类型设置
3. 如果使用哈希，确认 key 字段正确

## 性能问题

### Q7: 性能不达预期

**问题**：QPS 远低于预期。

**解决**：
1. 调整 worker 进程数：`nginx.worker_processes`
2. 增加 worker 连接数：`nginx.worker_connections`
3. 启用 HTTP/2：`proxy_http_version: "2"`
4. 优化插件配置，禁用不必要的插件
5. 使用高性能存储

### Q8: 内存占用过高

**问题**：APISIX 内存使用持续增长。

**解决**：
1. 检查是否有内存泄漏的插件
2. 限制缓存大小
3. 定期清理日志
4. 监控并调整缓冲区大小

## 安全问题

### Q9: 如何启用 Admin API 鉴权

**解决**：

在 `config.yaml` 中配置 Admin API 密钥：

```yaml
apisix:
  admin_key:
    - name: admin
      key: your-secret-key-here
      role: admin
```

使用密钥访问 Admin API：

```bash
curl -H "X-API-Key: your-secret-key-here" \
  http://127.0.0.1:9180/apisix/admin/routes
```

### Q10: 如何配置 HTTPS

**解决**：

1. 获取 SSL 证书
2. 配置 SSL 路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "https-route",
  "uri": "/api/*",
  "plugins": {
    "ssl": {
      "cert": "/path/to/cert.pem",
      "key": "/path/to/key.pem"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 集成问题

### Q11: 如何与 Kubernetes 集成

**解决**：

使用 APISIX Ingress Controller：

```bash
helm install apisix-ingress apisix/apisix-ingress \
  --set ingressController.enabled=true
```

### Q12: 如何与 Prometheus 集成

**解决**：

启用 Prometheus 插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/plugins" -X PUT -d '
{
  "prometheus": {
    "prefer_name": true
  }
}'
```

Prometheus 指标端点：`http://127.0.0.1:9091/apisix/prometheus/metrics`

### Q13: 如何配置日志

**解决**：

配置访问日志：

```yaml
nginx:
  access_log: /var/log/apisix/access.log
  error_log: /var/log/apisix/error.log
  error_log_level: warn
```

使用日志插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "log-route",
  "uri": "/api/*",
  "plugins": {
    "log-rotate": {
      "max_size": 100,
      "max_backups": 50,
      "max_age": 30,
      "compress": true
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 常见错误码

| 错误码 | 说明 | 解决方法 |
|--------|------|----------|
| 400 | 请求参数错误 | 检查请求格式和参数 |
| 401 | 未授权 | 配置并使用 API 密钥 |
| 403 | 禁止访问 | 检查权限配置 |
| 404 | 路由不存在 | 确认路由配置 |
| 429 | 超过限流 | 调整限流配置 |
| 500 | 服务器内部错误 | 检查日志 |
| 502 | 网关错误 | 检查上游服务 |
| 503 | 服务不可用 | 检查上游健康状态 |
