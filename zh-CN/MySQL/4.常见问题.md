# MySQL 常见问题

## 安装问题

### Q: MySQL 服务无法启动怎么办？

A: 检查以下内容：
1. 确认 MySQL 已正确安装
2. 查看错误日志：`mysqld --console`
3. 检查端口是否被占用：`netstat -an | grep 3306`
4. 确认数据目录权限正确
5. 尝试重新初始化数据目录

### Q: 忘记 root 密码如何重置？

A: 
1. 停止 MySQL 服务
2. 以跳过权限检查模式启动：`mysqld --skip-grant-tables`
3. 登录 MySQL
4. 重置密码：`UPDATE mysql.user SET Password=PASSWORD('new_password') WHERE User='root';`
5. 刷新权限：`FLUSH PRIVILEGES;`
6. 重启 MySQL 服务

### Q: 如何更改 MySQL 端口号？

A: 修改 my.cnf 或 my.ini 配置文件：
```ini
[mysqld]
port=3307
```
然后重启 MySQL 服务。

## 连接问题

### Q: 远程连接被拒绝怎么办？

A: 
1. 确认 MySQL 允许远程连接（检查 user 表中用户的 host）
2. 检查防火墙是否允许端口
3. 确认 MySQL 配置中 bind-address 不是 127.0.0.1
4. 检查 SELinux 或 AppArmor 设置

### Q: 连接超时如何处理？

A: 
1. 检查网络连接是否正常
2. 增加 wait_timeout 配置
3. 使用长连接或连接池
4. 检查服务器负载

## 数据库操作问题

### Q: 如何处理字符集乱码？

A: 
1. 创建数据库时指定字符集：`CREATE DATABASE db CHARACTER SET utf8mb4;`
2. 连接时指定字符集：`SET NAMES utf8mb4;`
3. 配置文件设置默认字符集：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: 如何处理数据导入导出乱码？

A: 
1. 导出时指定字符集：`mysqldump --default-character-set=utf8mb4`
2. 导入时指定字符集：`mysql --default-character-set=utf8mb4`

### Q: 主键自增不连续正常吗？

A: 正常。自增 ID 不连续的原因包括：
- 插入失败会消耗 ID
- 事务回滚不回收已使用的 ID
- 批量插入多条记录时可能跳跃
- 删除记录不影响自增计数器

## 性能问题

### Q: 查询很慢怎么办？

A: 
1. 使用 EXPLAIN 分析查询执行计划
2. 为 WHERE、JOIN、ORDER BY 涉及的列创建索引
3. 避免全表扫描
4. 优化查询语句，避免 SELECT *
5. 考虑使用查询缓存（MySQL 8.0 已移除）
6. 分表分库分散压力

### Q: 如何查看慢查询？

A: 
1. 开启慢查询日志：
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. 查看慢查询日志文件位置：
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: 内存使用过高如何优化？

A: 
1. 调整 innodb_buffer_pool_size（建议设置为可用内存的 70%）
2. 优化查询减少不必要的扫描
3. 限制连接数
4. 清理不必要的缓存

## 备份恢复问题

### Q: 备份文件太大怎么办？

A: 
1. 使用压缩备份：`mysqldump | gzip > backup.sql.gz`
2. 分库分表备份
3. 使用 xtrabackup 进行物理备份
4. 定期清理历史数据

### Q: 如何实现增量备份？

A: 
1. 开启二进制日志：`log-bin=mysql-bin`
2. 定期备份二进制日志
3. 使用 mysqlbinlog 工具恢复增量数据

## 复制问题

### Q: 主从复制延迟过高怎么办？

A: 
1. 检查网络延迟
2. 调整 slave_parallel_workers 增加并行复制
3. 确认从库硬件性能足够
4. 优化主库写入速度
5. 检查复制过滤规则

### Q: 如何处理主从数据不一致？

A: 
1. 重新同步从库
2. 使用 pt-table-checksum 检查不一致
3. 使用 pt-table-sync 修复数据
4. 重新配置主从复制

## 安全问题

### Q: 如何加强 MySQL 安全？

A: 
1. 设置强密码策略
2. 限制 root 用户远程登录
3. 删除匿名账户
4. 删除测试数据库
5. 定期更新 MySQL 版本
6. 启用 SSL/TLS 加密连接
7. 配置防火墙规则
8. 最小化授权原则

### Q: 如何防止 SQL 注入？

A: 
1. 使用预处理语句（Prepared Statements）
2. 验证和过滤用户输入
3. 限制数据库用户权限
4. 避免动态拼接 SQL
5. 使用存储过程
