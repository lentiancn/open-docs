# Go 安裝指南

## 環境要求

Go 語言可以在多種作業系統上執行，包括 Linux、macOS 和 Windows。不同的作業系統有不同的安裝方式，但核心要求類似。確保你的系統滿足以下基本要求，以便順利安裝和使用 Go 語言開發環境。

在作業系統方面，Go 支援廣泛的平台。Linux 方面，Go 可以在 Ubuntu、Debian、Fedora、CentOS、RHEL 等主流發行版上執行。macOS 需要 macOS 10.13 (High Sierra) 或更高版本。Windows 需要 Windows 10 或更高版本，同時也支援 Windows 7 和 8.1，但可能需要額外的配置。

在硬體方面，Go 的要求相對較低。安裝 Go 只需要約 100MB 的磁碟空間，實際專案開發可能需要更多空間，這取決於專案規模。記憶體方面，2GB RAM 足以進行日常開發，但如果進行大規模編譯或執行大型測試，可能需要更多記憶體。

Go 編譯器本身對 CPU 沒有特殊要求，現代的 x86-64、ARM64 等架構都可以良好執行。Go 支援交叉編譯，可以在一種平台上編譯出其他平台的二進制檔案。

## 安裝步驟

### 在 Linux 上安裝

在 Linux 系統上，推薦使用官方提供的壓縮包進行安裝，這種方式最為乾淨和靈活。首先從 Go 官方網站下載最新版本的壓縮包：

```bash
# 下載 Go 安裝包（替換為最新版本號）
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# 移除舊的 Go 安裝（如果存在）
sudo rm -rf /usr/local/go

# 解壓到 /usr/local
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

接下來需要將 Go 的二進制目錄添加到 PATH 環境變數中。編輯 ~/.bashrc 或 ~/.profile 檔案，新增以下行：

```bash
export PATH=$PATH:/usr/local/go/bin
```

儲存後，執行以下命令使配置生效：

```bash
source ~/.bashrc
```

或者在 macOS 或某些 Linux 發行版上：

```bash
source ~/.profile
```

### 在 macOS 上安裝

在 macOS 系統上，有兩種主要的安裝方式。第一種是透過 Homebrew 套件管理器安裝，這適合已經配置了 Homebrew 的使用者：

```bash
# 使用 Homebrew 安裝
brew install go

# 驗證安裝
go version
```

第二種方式是下載官方提供的 macOS 安裝包。訪問 Go 官方網站下載 .pkg 安裝包，雙擊執行並按照安裝精靈完成安裝。安裝程式會自動配置環境變數。

安裝完成後，開啟終端機並驗證安裝：

```bash
go version
```

應該能夠看到類似 "go1.21.5 darwin/amd64" 的輸出。

### 在 Windows 上安裝

在 Windows 系統上，推薦使用 MSI 安裝包進行安裝。訪問 Go 官方網站下載 Windows 版本的 MSI 安裝包，雙擊執行並按照安裝精靈完成安裝。安裝程式會自動配置環境變數，包括 PATH。

安裝完成後，開啟命令提示字元或 PowerShell，驗證安裝：

```bash
go version
```

Windows 使用者也可以選擇使用 Chocolatey 套件管理器安裝：

```bash
choco install golang
```

### 使用版本管理器（推薦）

對於需要同時管理多個 Go 版本的開發者，推薦使用 Go 版本管理器。常見的版本管理器包括 gvm（Go Version Manager）和 asdf-go：

gvm 的安裝和使用：

```bash
# 安裝 gvm
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# 安裝 Go 版本
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdf 是一個多語言版本管理器，支援 Go：

```bash
# 安裝 asdf 插件
asdf plugin add golang

# 安裝 Go 版本
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## 驗證安裝

安裝完成後，需要驗證 Go 是否正確安裝。開啟終端機或命令提示字元，執行以下命令：

```bash
# 查看 Go 版本
go version

# 查看 Go 環境資訊
go env
```

go env 命令會輸出多個環境變數，包括 GOPATH、GOROOT、GOOS、GOARCH 等。這些資訊對於理解 Go 的工作方式和進行環境配置非常重要。

執行一個簡單的程式來驗證整個開發環境是否正常運作：

```bash
# 建立工作目錄
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# 初始化模組（Go 1.16+ 推薦使用 go mod）
go mod init hello

# 建立測試程式
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# 執行程式
go run main.go
```

如果輸出 "Hello, Go!"，說明 Go 開發環境已經正確配置。

## 環境配置

### GOPATH 和模組

在 Go 1.11 之前，Go 使用 GOPATH 模式管理依賴。從 Go 1.11 開始，Go 引入了模組（modules）支援，推薦使用 go mod 管理專案依賴。使用模組時，不需要設定 GOPATH，可以在任意目錄建立專案。

對於企業內網環境或需要使用私人倉庫的情況，可能需要設定 GOPRIVATE 環境變數：

```bash
export GOPRIVATE=github.com/company/*
```

### 代理配置

在中國大陸等地區，直接存取 Go 官方模組倉庫可能較慢或無法存取。可以配置 Go 模組代理：

```bash
# 使用阿里雲代理
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct

# 或者使用七牛雲代理
go env -w GOPROXY=https://goproxy.cn,direct
```

### IDE 配置

Go 開發通常使用 VS Code、GoLand、Sublime Text 等編輯器。VS Code 需要安裝 Go 擴充功能插件才能獲得良好的開發體驗，包括程式碼補全、偵錯、格式化等功能。

GoLand 是 JetBrains 出品的專業 Go IDE，提供了最完善的 Go 開發支援，包括進階偵錯、重構、測試等功能。

## 升級和解除安裝

### 升級 Go 版本

升級 Go 版本非常簡單。下載新版本的安裝包，按照上述安裝步驟重新安裝即可。新的安裝包會覆蓋舊版本。

使用版本管理器的使用者可以使用版本管理器輕鬆切換不同版本：

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### 解除安裝 Go

在 Linux 或 macOS 上，解除安裝 Go 只需要移除安裝目錄：

```bash
# 移除 Go 安裝目錄
sudo rm -rf /usr/local/go

# 從 PATH 中移除
# 編輯 ~/.bashrc 或 ~/.profile，移除相關行
```

在 Windows 上，可以透過控制台的程式和功能進行解除安裝，或者使用 Chocolatey 解除安裝：

```bash
choco uninstall golang
```
