# GitLab 安裝指南

## 環境要求

在安裝 GitLab 之前，需要確保伺服器滿足以下硬體和軟體需求。對於中小型團隊，推薦配置至少 4 核心 CPU 和 8GB 記憶體，可以支援數百個倉庫和數十個並發使用者。大型團隊或高負載場景需要更多資源，建議使用 8 核心以上 CPU 和 16GB 以上記憶體。

作業系統方面，GitLab 支援多種 Linux 發行版，包括 Ubuntu、Debian、CentOS、Red Hat Enterprise Linux 和 Fedora。官方推薦使用 Ubuntu 或 Debian，因為這些系統的軟體包管理最為便捷。GitLab 至少需要 20GB 的磁碟空間用於安裝，基本配置建議預留 50GB 以上，生產環境通常需要數百 GB。

軟體依賴方面，GitLab 需要安裝 OpenSSH Server 用於 SSH 協定支援，PostgreSQL 資料庫（GitLab 自帶內建版本），以及 SMTP 伺服器用於郵件通知。還需要設定正確的時區和主機名，確保伺服器網路正常，可以存取互聯網以下載依賴包。

## 安裝步驟

### Ubuntu/Debian 安裝

在 Ubuntu 或 Debian 系統上，推薦使用官方 apt 倉庫安裝 GitLab，這樣可以確保獲得最新的穩定版本和自動安全更新。首先需要安裝必要的依賴包：

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

新增 GitLab APT 倉庫並安裝 GitLab CE（社群版）：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

其中 EXTERNAL_URL 需要替換為實際存取 GitLab 的網域或 IP 位址。安裝過程會自動設定 PostgreSQL、Redis 和 GitLab 本身。

### CentOS/RHEL 安裝

在 CentOS 或 Red Hat Enterprise Linux 上，可以使用 yum 或 dnf 軟體包管理器安裝 GitLab。首先安裝必要的依賴：

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

新增 GitLab YUM 倉庫並安裝：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## 初始設定

安裝完成後，需要設定 GitLab 的基本參數。編輯 /etc/gitlab/gitlab.rb 檔案，這是 GitLab 的主設定檔。最重要的設定項目是 external_url，即 GitLab 服務的外部存取位址：

```ruby
external_url 'https://gitlab.example.com'
```

如果需要使用自訂 SSH 埠，需要設定 gitlab_rails['gitlab_shell_ssh_port'] 參數。郵件伺服器設定也很重要，用於傳送通知和密碼重設郵件：

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

儲存設定後，執行以下命令使設定生效：

```bash
sudo gitlab-ctl reconfigure
```

此命令會重新設定 GitLab 的所有元件，包括資料庫遷移、初始使用者建立等。整個過程可能需要幾分鐘時間。

## 驗證安裝

設定完成後，可以透過瀏覽器存取 external_url 指定的位址。如果一切正常，應該能看到 GitLab 的登入頁面。首次存取時，會看到管理員帳戶登入頁面，預設管理員使用者名稱是 root，首次登入需要設定密碼。

GitLab 提供命令列工具 gitlab-ctl 用於管理服務狀態：

```bash
# 檢查服務狀態
sudo gitlab-ctl status

# 查看日誌
sudo gitlab-ctl tail

# 重新啟動服務
sudo gitlab-ctl restart
```

也可以使用 gitlab-rake 命令進行健康檢查：

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Docker 安裝

對於快速測試或小型部署，可以使用 Docker 執行 GitLab。首先需要安裝 Docker Engine，然後在主機上執行以下命令：

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

Docker 安裝方式掛載了三個卷用於持久化設定、日誌和資料。首次啟動 Docker 容器需要較長時間初始化，耐心等待後即可透過瀏覽器存取。

## 升級維護

定期升級 GitLab 到最新版本可以獲得新功能和安全修復。在 Ubuntu/Debian上升級：

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

升級前建議備份資料，GitLab 提供內建備份工具：

```bash
sudo gitlab-backup create
```

備份檔案預設保存在 /var/opt/gitlab/backups 目錄，可以設定備份保留策略和遠端儲存位置。
