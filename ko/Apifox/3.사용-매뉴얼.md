# 사용 매뉴얼

이 매뉴얼에서는 Apifox의 핵심 기능과 사용 방법에 대해 상세히 소개합니다. 프로젝트 관리, 인터페이스 설계, 디버깅, Mock, 자동화 테스트 등을 다룹니다.

## 프로젝트 관리

### 프로젝트 생성

Apifox에서 프로젝트는 API를 조직하는 가장 큰 단위입니다.

```bash
# 새 프로젝트 생성 단계
1. 왼쪽 "프로젝트" 목록 옆의 "+" 클릭
2. 프로젝트 이름 입력
3. 프로젝트 유형 선택
4. 기본 템플릿 선택 (선택 사항)
5. "생성" 클릭
```

### 프로젝트 설정

각 프로젝트는 독립적인 설정 옵션을 가집니다:

- **기본 정보**: 프로젝트 이름, 설명, 아이콘
- **환경 변수**: 개발, 테스트, 프로덕션 등 다양한 환경의 변수 정의
- **인증**: 전역 인증 방식 설정
- **내보내기/가져오기**: 프로젝트 데이터의 가져오기와 내보내기

### 환경 관리

Apifox는 다중 환경 구성을 지원합니다:

```bash
# 환경 변수 사용
요청에서 {{변수명}}으로 환경 변수 참조
예시: {{baseUrl}}/api/users

환경 전환: 상단 환경 선택기를 클릭하여 다른 환경 선택
```

## 인터페이스 설계

### 인터페이스 생성

인터페이스는 API의 구체적인 정의입니다:

```bash
# 인터페이스 생성 단계
1. 프로젝트에서 "+" 클릭 후 "새 인터페이스" 선택
2. 기본 정보 입력:
   - 인터페이스 이름: 사용자 목록
   - 요청 경로: /api/users
   - 요청 메서드: GET
3. 요청 매개변수 정의
4. 응답 구조 정의
5. 저장
```

### 요청 매개변수

다음 유형의 요청 매개변수를 지원합니다:

- **경로 매개변수**: 예: /api/users/:id 中的 id
- **쿼리 매개변수**: URL 中的 ?name=test
- **요청 헤더**: 예: Authorization
- **요청 본문**: JSON, 폼, XML 등 형식

### 응답 정의

각 인터페이스의 성공 및 실패 응답을 정의합니다:

```json
{
  "성공 응답 200": {
    "id": 1,
    "name": "김철수",
    "email": "chulsoo@example.com"
  },
  "오류 응답 404": {
    "code": 404,
    "message": "사용자가 존재하지 않습니다"
  }
}
```

### 인터페이스 목록 관리

그룹별로 인터페이스를 관리할 수 있으며, 모듈이나 기능별로 분류할 수 있습니다:

- 드래그하여 순서 조정
- 일괄 작업 (일괄 삭제, 일괄 내보내기 등)
- 검색 및 필터링

## 인터페이스 디버깅

### 요청 보내기

인터페이스 상세 페이지에서 "보내기" 버튼을 클릭하여 요청을 보냅니다:

```bash
# 기본 디버깅 단계
1. 인터페이스 열기
2. 환경 선택
3. 매개변수 값 설정
4. "보내기" 클릭
5. 응답 확인
```

### 인증 설정

다양한 인증 방식을 지원합니다:

- **None**: 인증 없음
- **Bearer Token**: JWT 인증
- **Basic Auth**: 기본 인증
- **OAuth 2.0**: OAuth2 인증
- **사용자 정의 인증**: 필요에 따라 사용자 정의 인증 헤더

### 요청 전 처리 스크립트

요청을 보내기 전에 실행할 스크립트를 설정할 수 있습니다:

```javascript
// 예시: 타임스탬프 추가
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// 예시: 랜덤 ID 생성
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### 응답 후 처리 스크립트

응답을 받은 후 처리할 스크립트를 설정할 수 있습니다:

```javascript
// 예시: 응답 데이터 추출
const response = pm.response.json();
console.log('응답 데이터:', response);

// 예시: 환경 변수 설정
pm.environment.set('access_token', response.data.token);
```

## API Mock

### 지능형 Mock

Apifox의 가장 강력한 기능 중 하나로, 데이터 구조만 정의하면 시스템이 자동으로 Mock 데이터를 생성합니다:

```bash
# 지능형 Mock 활성화
1. 인터페이스 열기
2. "Mock" 탭 클릭
3. "지능형 Mock" 활성화
4. 저장
```

### Mock 규칙

사용자 정의 Mock 규칙을 지원합니다:

- `@string`: 랜덤 문자열 생성
- `@integer`: 랜덤 정수 생성
- `@float`: 랜덤 부동소수점 생성
- `@boolean`: 랜덤 부울값 생성
- `@date`: 랜덤 날짜 생성
- `@email`: 랜덤 이메일 생성
- `@url`: 랜덤 URL 생성
- `@guid`: UUID 생성

### 예시

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

생성된 Mock 데이터:

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock 서버

Apifox는 내장 Mock 서버를 제공합니다:

```bash
# Mock 주소 가져오기
인터페이스 상세 페이지에서 Mock URL 복사
예시: https://mock.apifox.com/api/users
```

## 자동화 테스트

### 테스트 케이스 생성

```bash
# 테스트 생성 단계
1. 인터페이스 열기
2. "테스트" 탭 클릭
3. "테스트 생성" 클릭
4. 테스트 케이스 이름 설정
5. assertion 추가
6. 저장
```

### assertion 유형

다양한 assertion을 지원합니다:

- **상태 코드 assertion**: 응답 상태 코드 확인
- **JSON assertion**: 응답 JSON 데이터 확인
- **응답 시간 assertion**: 응답 시간 확인
- **응답 헤더 assertion**: 응답 헤더 확인

### 예시

```javascript
// 상태 코드 assertion
pm.test("상태 코드가 200", function() {
    pm.response.to.have.status(200);
});

// JSON assertion
pm.test("사용자 목록 반환", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// 응답 시간 assertion
pm.test("응답 시간이 500ms 미만", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### 테스트 스위트

여러 인터페이스의 테스트 케이스를 테스트 스위트로 구성할 수 있습니다:

1. 테스트 스위트 생성
2. 테스트 케이스 추가
3. 실행 순서 설정
4. 스위트 실행

### 테스트 보고서 생성

테스트 실행 후 상세한 테스트 보고서를 생성할 수 있습니다:

- 통과율 통계
- 실패 케이스 분석
- 응답 시간 통계

## 팀 협업

### 팀원 초대

```bash
# 팀원 초대
1. 프로젝트 설정으로 이동
2. "멤버 관리" 클릭
3. "팀원 초대" 클릭
4. 멤버 이메일 입력
5. 역할 권한 설정
6. 초대 보내기
```

### 역할 권한

- **관리자**: 프로젝트 전체 권한
- **편집자**: 인터페이스 편집 및 테스트 실행 가능
- **뷰어**: 조회만 가능, 편집 불가

### 실시간 동기화

팀원이 로그인하면 프로젝트 데이터가 자동으로 실시간 동기화됩니다:

- 인터페이스 변경 자동 동기화
- 테스트 결과 자동 동기화
- 팀원이 최신 내용을 볼 수 있음

### 댓글 기능

인터페이스에 댓글을 추가할 수 있습니다:

1. 인터페이스 열기
2. "댓글" 클릭
3. 댓글 내용 입력
4. 팀원 @멘션
