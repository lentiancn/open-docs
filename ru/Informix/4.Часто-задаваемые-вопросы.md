# Часто задаваемые вопросы по IBM Informix

## Общие вопросы

### В: В чём разница между IBM Informix и HCL Informix?

О: IBM Informix продаётся и продвигается IBM, в то время как HCL Informix разрабатывается и поддерживается HCLSoftware. Оба имеют одинаковую кодовую базу и функционально эквивалентны. IBM занимается продажами и лицензированием, HCL — разработкой и технической поддержкой.

### В: Какую редакцию выбрать для разработки?

О: Используйте **Developer Edition** (бесплатно) или **Innovator-C Edition** (бесплатно) для разработки и тестирования. Обе имеют ограничения по ресурсам, но включают полную функциональность:
- Developer Edition: 1 CPU VP/Core, 1 ГБ ОЗУ, 8 ГБ хранилища
- Innovator-C Edition: 2 CPU Cores, 8 ГБ ОЗУ, 50 ГБ хранилища

### В: Подходит ли Informix для IoT-приложений?

О: Да, Informix отлично подходит для IoT благодаря компактности (всего 100 МБ), минимальным требованиям к администрированию, встроенной поддержке временных рядов и возможности работать на пограничных устройствах.

### В: Какие языки программирования поддерживаются?

О: Informix поддерживает множество интерфейсов:
- SQL (нативный)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- Совместимость с MongoDB (JSON/BSON)

## Проблемы установки

### В: Установка завершается с ошибкой «Нет разрешения»

О: Убедитесь, что у вас есть права root/sudo. Создайте пользователя и группу informix перед установкой:

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### В: Сервер не запускается после установки

О: Проверьте следующее:
- Переменные окружения настроены правильно (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- Файлы `onconfig` и `sqlhosts` корректны
- Просмотрите журналы ошибок в `$INFORMIXDIR/tmp/`
- Убедитесь в достаточном месте на диске

### В: Какие порты использует Informix?

О: Порт по умолчанию — 9088. Проверить можно в файле `sqlhosts`. Убедитесь, что брандмауэр разрешает этот порт.

## Типичные операционные проблемы

### В: Как создавать резервные копии и восстанавливать Informix?

О: Используйте утилиты `ontape` или `onbar`:

```bash
# Полное резервное копирование с ontape
ontape -s -L 0

# Восстановление с ontape
ontape -r
```

### В: Как проверить статус сервера?

О: Используйте команду `onstat`:

```bash
onstat -        # Общий статус
onstat -d       # Использование диска
onstat -l       # Логические журналы
onstat -u       # Активные пользователи
onstat -k       # Информация о блокировках
```

### В: Как подключиться к Informix из внешних приложений?

О: Настройте драйверы ODBC, JDBC или нативные клиенты. Обновите файл `sqlhosts`. Пример строки подключения JDBC:

```
jdbc:informix-sqli://hostname:9088/dbname:INFORMIXSERVER=имя_сервера
```

### В: Что вызывает ошибку «Logical Log Full»?

О: Это происходит, когда все логические журналы заполнены и не были скопированы. Решения:
- Скопировать логические журналы: `ontape -s -L 0`
- Добавить файлы логических журналов: `onparams -a -d dbspace -s размер`

### В: Как обрабатывать конфликты блокировок?

О: Используйте `onstat -k` для просмотра блокировок и `onstat -u` для идентификации блокирующих сессий. Решения:
- Увеличить тайм-аут блокировки
- Использовать более низкий уровень изоляции (dirty read)
- Завершить блокирующую сессию: `onmode -z id_сессии`

## Производительность и настройка

### В: Как улучшить производительность запросов?

О:
- Создавайте индексы для часто запрашиваемых столбцов
- Используйте `EXPLAIN` для анализа планов запросов
- Настраивайте разделяемую память соответствующим образом
- Регулярно обновляйте статистику: `UPDATE STATISTICS FOR TABLE таблица`

### В: Какие ключевые параметры памяти?

О:
- `SHMVIRTSIZE` - Размер виртуальной разделяемой памяти
- `SHMADD` - Дополнительные сегменты разделяемой памяти
- `BUFFERPOOL` - Конфигурация кэша данных
