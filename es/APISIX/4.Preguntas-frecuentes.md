# Preguntas Frecuentes

Este documento recopila las preguntas técnicas más comunes y sus respuestas sobre el uso de Apache APISIX.

## Preguntas Básicas

### ¿Qué es Apache APISIX?

Apache APISIX es un proyecto de nivel superior de la Apache Software Foundation. Es una puerta de enlace (gateway) de API nativa para la nube, conocida por ser dinámica, en tiempo real y de alto rendimiento. Ofrece funciones como rutas dinámicas, upstream dinámico, certificados dinámicos, pruebas A/B, despliegues progresivos, limitación de velocidad, protección contra ataques, monitoreo y alertas.

### ¿Cuál es la diferencia entre APISIX y Nginx?

Nginx es un servidor web tradicional y proxy inverso, mientras que APISIX es un gateway de API construido sobre Nginx. APISIX añade sobre Nginx: gestión de configuración completamente dinámica, un extenso sistema de plugins, gestión de tráfico refinada y capacidades de observabilidad integradas.

### ¿Cuál es el rendimiento de APISIX?

APISIX hereda las características de alto rendimiento de Nginx, manteniendo una capacidad de procesamiento eficiente gracias a su optimización y sistema de plugins. El rendimiento real depende de la configuración del hardware y la complejidad del negocio; en condiciones normales, puede manejar decenas de miles o incluso cientos de miles de solicitudes por segundo.

## Problemas de Instalación

### ¿Qué hacer si Docker no inicia?

1. Verificar si Docker está funcionando correctamente: `docker ps`
2. Revisar los logs del contenedor: `docker-compose logs apisix`
3. Problemas comunes:
   - Conflicto de puertos: Verificar si los puertos 9080, 9180, 9443 están en uso
   - Error de conexión con etcd: Confirmar que el contenedor de etcd esté arrancado correctamente
   - Permisos: Asegurarse de tener permisos suficientes para ejecutar Docker

### ¿Cómo ver la versión de APISIX?

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Devuelve: `Server: APISIX/Versión`

### ¿No puedo acceder al Dashboard después de instalar?

1. Verificar que el puerto del Dashboard esté mapeado correctamente (por defecto 9180)
2. Revisar la configuración del firewall
3. Confirmar que la configuración de red del contenedor sea correcta

## Problemas de Configuración

### ¿Cómo cambiar el puerto de la API de administración?

Editar el archivo de configuración `apisix.yml`:

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### ¿Cómo activar HTTPS?

1. Configurar el certificado SSL:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "contenido-del-certificado",
  "key": "contenido-de-la-clave-privada",
  "snis": ["example.com"]
}'
```

2. Activar HTTPS en la ruta:

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### ¿La configuración de la ruta no surte efecto?

1. Verificar que la configuración de la ruta sea correcta
2. Confirmar que la configuración de etcd esté sincronizada correctamente
3. Revisar si hay errores de sintaxis en los plugins
4. Consultar los logs de error de APISIX

### ¿Cómo implementar solicitudes CORS?

Usar el plugin `cors`:

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## Problemas de Rendimiento

### ¿Cómo aumentar el throughput?

1. **Aumentar el número de worker processes:** `apisix.worker_processes: auto`
2. **Activar HTTP/2:** Configurar `http2: true` en la configuración de listen
3. **Optimizar el pool de conexiones del upstream:** Configurar `keepalive_pool.size`
4. **Usar plugins de caché:** Activar el plugin `proxy-cache`

### ¿Qué hacer si el uso de memoria es muy alto?

1. Reducir la carga de plugins innecesarios
2. Limitar el número de conexiones al upstream
3. Ajustar el tamaño del buffer de logs
4. Usar respuestas en streaming para archivos grandes

## Problemas de Funcionalidades

### ¿Cómo implementar pruebas A/B?

Usar el plugin `traffic-split`:

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### ¿Cómo implementar limitación de solicitudes?

Usar el plugin `limit-req`:

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## Problemas de Integración

### ¿Cómo integrar con Kubernetes?

Usar el APISIX Ingress Controller:

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: ejemplo-ruta
spec:
  http:
    - name: ejemplo
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: servicio-ejemplo
          servicePort: 8080
```

### ¿Cómo integrar con Prometheus?

1. Activar el plugin de Prometheus
2. Configurar Prometheus para hacer scraping

## Problemas de Seguridad

### ¿Cómo evitar el abuso de APISIX?

1. **Activar autenticación:** Activar key-auth u otros plugins de autenticación en todas las rutas
2. **Limitar el acceso a la API de administración:** Solo permitir IPs de confianza
3. **Usar HTTPS:** Activar cifrado TLS
4. **Configurar limitación de velocidad:** Prevenir solicitudes maliciosas
5. **Activar logs de acceso:** Registrar todas las solicitudes para auditoría

### ¿Cómo activar la autenticación en la API de administración?

```yaml
deployment:
  admin:
    admin_key_required: true
```

## Problemas de Operación y Mantenimiento

### ¿Cómo actualizar APISIX?

**Con Docker:**

```bash
docker-compose pull
docker-compose up -d
```

**Con Kubernetes:**

```bash
helm upgrade apisix apisix/apisix
```

### Códigos de Error Comunes

- **401 Unauthorized**: Fallo de autenticación
- **403 Forbidden**: Sin permisos de acceso
- **404 Not Found**: La ruta no existe
- **429 Too Many Requests**: Solicitudes superan el límite
- **500 Internal Server Error**: Error del servicio upstream
- **502 Bad Gateway**: Servicio upstream no disponible
- **503 Service Unavailable**: Servicio no disponible
