# FantasyTalking 常见问题

## 安装问题

### 安装失败怎么办？

安装过程中可能会遇到各种问题，首先请确认环境满足要求。检查 Node.js 版本是否在 14.0 以上，可以使用 `node --version` 命令查看。如果使用的是 Python 版本，确保 Python 版本在 3.7 以上。

如果安装仍然失败，可以尝试以下方法：清除 npm 缓存 `npm cache clean --force`；使用国内镜像源 `npm config set registry https://registry.npmmirror.com`；或者从源码编译安装。

### FFmpeg 安装问题

在某些系统上，FFmpeg 可能没有正确安装或配置。可以从 FFmpeg 官网下载预编译的二进制文件，或者使用包管理器安装：macOS 上使用 `brew install ffmpeg`，Linux 上使用 `apt-get install ffmpeg` 或 `yum install ffmpeg`，Windows 上可以从官网下载或使用 Chocolatey 安装。

## 使用问题

### 语音合成速度慢怎么办？

语音合成的速度取决于多个因素。首先检查网络连接，如果使用云服务，网络延迟会影响响应时间。其次可以考虑自建服务来减少网络开销。对于大批量合成任务，建议使用批量合成接口，可以显著提高效率。还可以调整音频参数，降低采样率可以加快合成速度。

### 如何选择合适的语音风格？

FantasyTalking 提供了多种语音风格，包括 neutral（中性）、happy（开心）、sad（悲伤）、angry（愤怒）、excited（兴奋）等。选择时需要考虑应用场景：客服场景推荐使用 neutral 或 professional；教育应用可以根据内容选择不同风格；娱乐应用可以选择更具表现力的风格。可以通过 A/B 测试来选择最适合的风格。

### 如何实现英文语音合成？

默认配置下需要指定语言为英文：

```javascript
const audio = await client.speak('Hello, world!', {
  voice: {
    language: 'en-US',  // 或 en-GB
    style: 'neutral'
  }
});
```

支持的语言代码包括 en-US、en-GB、en-AU 等。不同的语言代码会使用对应地区的发音模型。

### SSML 标记不生效怎么办？

确保在调用 speak 函数时传入 `ssml: true` 参数。检查 SSML 标记是否正确，标记名称应该是小写。某些特殊字符需要进行 XML 转义，例如小于号需要改为 `<`。如果问题仍然存在，可能是当前语言不支持某些 SSML 特性。

## 对话功能

### 对话上下文不连贯怎么办？

确保使用的是同一个会话实例，每次调用 `session.chat()` 都会自动包含之前的对话历史。如果需要新的对话，创建新的会话实例。检查是否达到了上下文窗口限制，默认保留 5 轮对话。可以通过修改配置来调整上下文窗口大小。

### 如何保存和恢复对话？

使用 `session.getHistory()` 获取对话历史，返回的是 JSON 格式的数据，可以保存到文件。要恢复对话，使用 `client.loadSession(history)` 方法加载历史记录。需要注意的是，加载的会话会从保存的状态继续对话。

## 音频问题

### 生成的音频有杂音怎么办？

杂音问题通常与音频配置或网络传输有关。检查音频格式和采样率是否匹配目标播放设备。尝试使用降噪功能处理音频 `client.process(audio, { denoise: true })`。如果是网络传输问题，考虑使用本地部署的服务。

### 如何调整语速和音调？

在语音配置中调整相关参数：

```javascript
const audio = await client.speak('文本', {
  voice: {
    speed: 1.5,  // 1.5 倍速，范围 0.5-2.0
    pitch: 0.8  // 降低音调，范围 0.5-2.0
  }
});
```

需要注意的是，过度调整参数可能会影响语音的自然度。

## 账号和计费

### 如何获取 API 密钥？

访问 FantasyTalking 官网注册账号，进入控制台创建 API 密钥。新用户通常会获得一定的免费额度用于测试。API 密钥应该妥善保管，不要在客户端代码中硬编码，建议使用环境变量存储。

### 如何查看使用量？

登录控制台后，可以在仪表盘查看 API 调用次数和消耗的配额。也可以使用 SDK 提供的方法查询：

```javascript
const usage = await client.getUsage();
console.log(`本月使用次数: ${usage.count}`);
console.log(`剩余免费次数: ${usage.remaining}`);
console.log(`当前计费周期: ${usage.period}`);
```

## 部署问题

### Docker 部署失败怎么办？

首先确认 Docker 已正确安装并运行。检查端口 8080 是否被占用，可以更改映射端口。如果使用云服务，确保安全组已开放相应端口。查看容器日志 `docker logs fantasy-talking` 获取详细错误信息。

### 如何实现高可用部署？

对于生产环境，建议使用负载均衡器配合多个 FantasyTalking 实例。会话数据需要使用外部存储（如 Redis）来实现多实例共享。可以使用 Docker Swarm 或 Kubernetes 进行容器编排，实现自动扩缩容和故障恢复。

## 开发和调试

### 如何开启调试模式？

在初始化客户端时设置 debug 选项：

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  debug: true  // 开启调试模式
});
```

调试模式下会输出详细的请求和响应信息，便于排查问题。

### 如何获取详细的错误信息？

所有 SDK 方法都会在失败时抛出包含详细信息的错误对象：

```javascript
try {
  await client.speak('text');
} catch (error) {
  console.log('错误代码:', error.code);
  console.log('错误消息:', error.message);
  console.log('HTTP 状态:', error.status);
  console.log('请求 ID:', error.requestId);  // 用于向技术支持反馈
}
```
