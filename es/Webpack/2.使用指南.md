# Guía de Usuario de Webpack

Esta guía te enseña cómo usar Webpack para construir aplicaciones JavaScript modernas.

## Inicio Rápido

### Crear un Proyecto

```bash
mkdir my-project
cd my-project
npm init -y
npm install webpack webpack-cli --save-dev
```

### Crear Archivos de Entrada

```javascript
// src/index.js
import './style.css';
import { add } from './math.js';

console.log('Hello Webpack!');
console.log(add(2, 3));
```

```javascript
// src/math.js
export function add(a, b) {
  return a + b;
}
```

```css
/* src/style.css */
body {
  background: #f0f0f0;
  font-family: sans-serif;
}
```

```html
<!-- src/index.html -->
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Webpack App</title>
</head>
<body>
  <h1>Hello Webpack!</h1>
  <script src="index.js"></script>
</body>
</html>
```

### Crear Archivo de Configuración

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
```

### Ejecutar Construcción

```bash
npx webpack
```

## Conceptos Fundamentales

### Entrada (Entry)

```javascript
module.exports = {
  // Entrada única
  entry: './src/index.js',
  
  // Múltiples entradas
  entry: {
    home: './src/home.js',
    about: './src/about.js',
  },
};
```

### Salida (Output)

```javascript
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    clean: true,
    publicPath: '/',
  },
};
```

### Cargadores (Loaders)

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader',
      },
    ],
  },
};
```

### Plugins

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
    new MiniCssExtractPlugin({
      filename: '[name].css',
    }),
  ],
};
```

### Modo (Mode)

```javascript
// Modo desarrollo
module.exports = {
  mode: 'development',
  devtool: 'source-map',
};

// Modo producción
module.exports = {
  mode: 'production',
  devtool: false,
};
```

## Cargadores Comunes

### Procesar CSS

```bash
npm install --save-dev style-loader css-loader
```

```javascript
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader'],
}
```

### Procesar Sass

```bash
npm install --save-dev sass sass-loader
```

```javascript
{
  test: /\.scss$/,
  use: ['style-loader', 'css-loader', 'sass-loader'],
}
```

### Procesar Imágenes

```javascript
{
  test: /\.(png|jpg|gif)$/,
  type: 'asset/resource',
}
```

### Procesar Fuentes

```javascript
{
  test: /\.(woff|woff2|eot|ttf|otf)$/,
  type: 'asset/resource',
}
```

## Servidor de Desarrollo

### Instalación

```bash
npm install --save-dev webpack-dev-server
```

### Configuración

```javascript
module.exports = {
  devServer: {
    static: './dist',
    hot: true,
    port: 3000,
    open: true,
    compress: true,
  },
};
```

### Iniciar

```bash
webpack serve --open
```

O agregar scripts a package.json:

```json
{
  "scripts": {
    "start": "webpack serve --mode development",
    "build": "webpack --mode production"
  }
}
```

## División de Código

### Importación Dinámica

```javascript
// Carga diferida de rutas
const Home = () => import('./Home.vue');
const About = () => import('./About.vue');

// Carga condicional
if (condition) {
  import('./module').then(module => {
    module.doSomething();
  });
}
```

### SplitChunks

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};
```

## Configuración de Entorno

### Entorno de Desarrollo

```javascript
module.exports = {
  mode: 'development',
  devtool: 'inline-source-map',
  devServer: {
    static: './dist',
    hot: true,
  },
};
```

### Entorno de Producción

```javascript
module.exports = {
  mode: 'production',
  devtool: 'source-map',
  optimization: {
    minimize: true,
  },
};
```

### Configuración Multi-Entorno

```javascript
const devConfig = require('./webpack.dev');
const prodConfig = require('./webpack.prod');

module.exports = (env, argv) => {
  return argv.mode === 'production' ? prodConfig : devConfig;
};
```

## Optimización de Rendimiento

### 1. Usar Modo Production

```javascript
module.exports = {
  mode: 'production',
};
```

### 2. Habilitar Tree Shaking

```javascript
// package.json
{
  "sideEffects": false
}
```

### 3. División de Código

```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
  },
}
```

### 4. Optimización de Caché

```javascript
output: {
  filename: '[name].[contenthash].js',
  chunkFilename: '[name].[contenthash].chunk.js',
}
```

### 5. Compresión

```javascript
const TerserPlugin = require('terser-webpack-plugin');

optimization: {
  minimizer: [new TerserPlugin()],
}
```

## Problemas Comunes

### ¿Falla la resolución de módulos?

Verifica la configuración de resolve:

```javascript
resolve: {
  extensions: ['.js', '.json'],
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
}
```

### ¿La construcción es lenta?

- Usar cache
- Excluir node_modules
- Usar thread-loader

```javascript
{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: 'thread-loader',
    options: {
      workers: 2,
    },
  },
}
```
