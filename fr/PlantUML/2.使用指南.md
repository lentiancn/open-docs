# Guide d'utilisation PlantUML

Ce guide explique comment utiliser PlantUML pour créer divers diagrammes UML.

## Démarrage rapide

### Syntaxe de base

```plantuml
@startuml
Alice -> Bob: Hello
@enduml
```

## Diagramme de séquence

```plantuml
@startuml
actor Utilisateur
participant "Frontend" as Front
participant "Backend" as Back
database "Base de données" as DB

Utilisateur -> Front: Soumettre la requête
Front -> Back: Appel API
Back -> DB: Interroger les données
DB --> Back: Renvoyer le résultat
Back --> Front: Réponse
Front --> Utilisateur: Afficher le résultat
@enduml
```

## Diagramme de classes

```plantuml
@startuml
class User {
  +id: int
  +username: string
  +email: string
  +login()
  +logout()
}

class Admin extends User {
  +manageUsers()
  +viewReports()
}

class Order {
  +id: int
  +total: decimal
  +createOrder()
  +cancelOrder()
}

User "1" -- "*" Order: places
@enduml
```

## Diagramme de cas d'utilisation

```plantuml
@startuml
left to right direction
actor Utilisateur
rectangle Système {
  Utilisateur -> (Connexion)
  Utilisateur -> (Parcourir les produits)
  Utilisateur -> (Ajouter au panier)
  Utilisateur -> (Passer commande)
  (Connexion) .> (Parcourir les produits): extends
}
@enduml
```

## Diagramme d'état

```plantuml
@startuml
[*] --> En attente de paiement
En attente de paiement --> Payé: Paiement réussi
Payé --> Expédié: Expédition
Expédié --> Livré: Livraison
Livré --> [*]
En attente de paiement --> Annulé: Utilisateur annule
Annulé --> [*]
@enduml
```

## Diagramme d'activité

```plantuml
@startuml
start
:Commencer les achats;
:Parcourir les produits;
if (Produit trouvé ?) then (Oui)
  :Ajouter au panier;
else (Non)
  :Rechercher un produit;
endif
:Passer à la caisse;
if (Paiement réussi ?) then (Oui)
  :Commande réussie;
else (Non)
  :Paiement échoué;
endif
stop
@enduml
```

## Diagramme de composants

```plantuml
@startuml
package "Frontend" {
  [Web App] as WA
  [Mobile App] as MA
}
package "Backend" {
  [API Gateway] as API
  [Auth Service] as Auth
  [Order Service] as Order
}
package "Données" {
  database "MySQL" as DB
  database "Redis" as Cache
}

WA --> API
MA --> API
API --> Auth
API --> Order
Order --> DB
Order --> Cache
@enduml
```

## Diagramme de déploiement

```plantuml
@startuml
node "Équipement utilisateur" as Client {
  [Navigateur]
}
node "Équilibreur de charge" as LB {
  [Nginx]
}
node "Serveur d'application" as App {
  [Tomcat 1]
  [Tomcat 2]
}
node "Serveur de base de données" as DB {
  [MySQL]
  [Redis]
}

Client --> LB
LB --> App
App --> DB
@enduml
```

## Diagramme de timing

```plantuml
@startuml
clock "Horloge" as C with period 1000
binary "Synchronisation" as SY
binary "Données" as DT

C is idle
SY is low
DT is low

@1000
SY is high
@150 low
@2000
DT is0
SY is high
@3000
DT is low
@enduml
```

## Commandes courantes

### Masquer les éléments

```plantuml
@startuml
hide footbox
actor A
participant B
A -> B: Message
@enduml
```

### Configuration des styles

```plantuml
@startuml
skinparam backgroundColor #EEEBDC
skinparam handwritten true
skinparam actorStyle awesome

actor User
User -> (Connexion)
@enduml
```
