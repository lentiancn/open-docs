# よくある質問

このドキュメントでは、Apache APISIX の使用中に、よくある技術的な質問とその回答をまとめています。

## 基本質問

### Apache APISIX とは何ですか？

Apache APISIX は、Apache Software Foundation のトッププロジェクトであり、動的・リアルタイム・高性能を特徴とするクラウドネイティブAPIゲートウェイです。動的ルート、動的アップストリーム、動的証明書、A/Bテスト、グレーディングリリース、レート制限、セキュリティ保護、モニタリングアラートなどの機能を提供します。

### APISIX と Nginx の違いは何ですか？

Nginx は伝統的なWebサーバーとリバースプロキシですが、APISIX は Nginx をベースに構築されたAPIゲートウェイです。APISIX は Nginx の上に、完全動的な設定管理、豊富なプラグイン機構、精细的なトラフィック管理、組み込みの可観測性機能を追加しています。

### APISIX のパフォーマンスはどのくらいですか？

APISIX は Nginx の高性能特性を継承しており、最適化とプラグイン機構を通じて効率的な処理能力を維持しています。具体的なパフォーマンスはハードウェア構成とビジネスの複雑さに応じて異なりますが、一般的には每秒数万件から数十万件のリクエストを処理できます。

## インストール質問

### Docker が起動しません，怎么办？

1. Docker が正常に動作しているか確認：`docker ps`
2. コンテナログを確認：`docker-compose logs apisix`
3. よくある問題：
   - ポート競合：9080、9180、9443 ポートが占有されていないか確認
   - etcd 接続失敗：etcd コンテナが正常に起動しているか確認
   - 権限問題：Docker を実行する十分な権限があるか確認

### APISIX のバージョンを確認する方法は？

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

返答形式：`Server: APISIX/Version`

### インストール後、Dashboard にアクセスできません

1. Dashboard ポートが正しくマッピングされているか確認（デフォルト 9180）
2. ファイアウォール設定を確認
3. コンテナネットワーク設定が正しいか確認

## 設定質問

### 管理APIのポートを変更するには？

`apisix.yml` 設定ファイルを編集します：

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### HTTPS を有効にする 방법은？

1. SSL 証明書を設定：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "証明書内容",
  "key": "秘密鍵内容",
  "snis": ["example.com"]
}'
```

2. ルートで HTTPS を有効化：

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### ルート設定が反映されません

1. ルート設定が正しいか確認
2. etcd 設定が正しく同期しているか確認
3. プラグインに構文エラーがないか確認
4. APISIX エラーログを確認

### クロスドメインリクエストを実現するには？

`cors` プラグインを使用します：

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## パフォーマンス質問

### スループットを向上させる方法は？

1. **ワーカープロセス数を増やす：**`apisix.worker_processes: auto`
2. **HTTP/2 を有効化：**listen 設定で `http2: true` を設定
3. **アップストリーム接続プールを最適化：**`keepalive_pool.size` を設定
4. **キャッシュプラグインを使用：**`proxy-cache` プラグインを有効化

### メモリ使用量が多すぎます

1. 不要なプラグインのロードを減らす
2. アップストリーム接続数を制限する
3. ログバッファサイズを調整する
4. 大きなファイルにはストリーミングレスポンスを使用する

## 機能質問

### A/Bテストを実現するには？

`traffic-split` プラグインを使用します：

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### リクエストレート制限を実現するには？

`limit-req` プラグインを使用します：

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## 統合質問

### Kubernetes との統合方法は？

APISIX Ingress Controller を使用します：

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### Prometheus との統合方法は？

1. Prometheus プラグインを有効化
2. Prometheus スクレイプ設定を設定

## セキュリティ質問

### APISIX の悪用を防ぐ方法は？

1. **認証を有効化：**すべてのルートで key-auth またはその他の認証プラグインを有効化
2. **管理APIアクセスを制限：**管理APIへのアクセスは信頼できるIPのみに制限
3. **HTTPS を使用：**TLS 暗号化を有効化
4. **レート制限を設定：**悪意のあるリクエストを防止
5. **アクセスログを有効化：**監査のためにすべてのリクエストを記録

### 管理APIで認証を有効にする方法は？

```yaml
deployment:
  admin:
    admin_key_required: true
```

## 運用質問

### APISIX をアップグレードする方法は？

**Docker の場合：**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes の場合：**

```bash
helm upgrade apisix apisix/apisix
```

### よくあるエラーコード

- **401 Unauthorized**：認証に失敗しました
- **403 Forbidden**：アクセス権限がありません
- **404 Not Found**：ルートが存在しません
- **429 Too Many Requests**：レート制限を超過しました
- **500 Internal Server Error**：アップストリームサービスエラー
- **502 Bad Gateway**：アップストリームサービスが利用できません
- **503 Service Unavailable**：サービスを利用できません
