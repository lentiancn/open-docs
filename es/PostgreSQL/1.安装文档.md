# Guía de Instalación de PostgreSQL

PostgreSQL es un potente sistema de base de datos objeto-relacional de código abierto. Esta guía cubre los métodos de instalación en todos los entornos.

## Información de Versión

| Versión | Estado | Descripción |
|---------|--------|-------------|
| PostgreSQL 16.x | ✅ Actual | Última versión estable |
| PostgreSQL 15.x | ✅ Soportado | Versión estable |
| PostgreSQL 14.x | ⚠️ Mantenimiento | Actualizaciones de seguridad |
| PostgreSQL 13.x | ⚠️ Mantenimiento | Soporte a largo plazo |
| PostgreSQL 12.x | ❌ Obsoleto | Sin soporte |

> **Importante**: Esta guía está orientada principalmente a **PostgreSQL 16.x**. Se recomienda usar la última versión para proyectos nuevos.

## Requisitos del Sistema

### Requisitos de Hardware

| Requisito | Mínimo | Recomendado |
|-----------|--------|-------------|
| CPU | 1 núcleo | 2+ núcleos |
| Memoria | 1 GB | 4+ GB |
| Disco | 10 GB | 50+ GB |

### Sistemas Operativos Soportados

- **Linux**: Ubuntu, Debian, CentOS, RHEL, Fedora, openSUSE
- **Windows**: Windows Server 2016+, Windows 10+
- **macOS**: 10.14+

## Métodos de Instalación

### Método 1: Instalación en Windows

#### 1. Usando el Instalador (Recomendado)

1. Descarga el instalador de PostgreSQL: https://www.postgresql.org/download/windows/
2. Ejecuta postgresql-16.x-x-windows-x64.exe
3. Sigue el asistente de instalación
4. Establece la contraseña del superusuario postgres

#### 2. Usando Chocolatey

```powershell
# Instalar PostgreSQL
choco install postgresql

# Instalar una versión específica
choco install postgresql --version=16.0
```

#### 3. Usando ZIP

```powershell
# Descargar el paquete ZIP
# Extraer a C:\Program Files\PostgreSQL\16

# Inicializar la base de datos
"C:\Program Files\PostgreSQL\16\bin\initdb" -D "C:\Program Files\PostgreSQL\16\data"

# Iniciar el servicio
"C:\Program Files\PostgreSQL\16\bin\pg_ctl" -D "C:\Program Files\PostgreSQL\16\data" -l logfile start
```

### Método 2: Instalación en macOS

#### 1. Usando Homebrew (Recomendado)

```bash
# Instalar PostgreSQL
brew install postgresql@16

# Iniciar el servicio
brew services start postgresql@16

# Detener el servicio
brew services stop postgresql@16
```

#### 2. Usando PostgreSQL.app

1. Descarga PostgreSQL.app: https://postgresapp.com/
2. Arrastra a la carpeta Applications
3. Haz clic para iniciar

#### 3. Usando MacPorts

```bash
sudo port install postgresql16
sudo port load postgresql16-server
```

### Método 3: Instalación en Linux

#### Ubuntu/Debian

```bash
# Añadir el repositorio oficial de PostgreSQL
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# Importar la clave
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# Actualizar la lista de paquetes
sudo apt-get update

# Instalar PostgreSQL
sudo apt-get install -y postgresql-16

# Iniciar el servicio
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

#### CentOS/RHEL

```bash
# Instalar el repositorio RPM
sudo yum install -y https://download.postgresql.org/pub/repos/yum/16/redhat/rhel-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# Instalar PostgreSQL
sudo yum install -y postgresql16-server

# Inicializar la base de datos
sudo /usr/pgsql-16/bin/postgresql-16-setup initdb

# Iniciar el servicio
sudo systemctl start postgresql-16
sudo systemctl enable postgresql-16
```

#### Fedora

```bash
# Instalar PostgreSQL
sudo dnf install -y postgresql-server postgresql-contrib

# Inicializar la base de datos
sudo postgresql-setup --initdb

# Iniciar el servicio
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### Método 4: Instalación con Docker

```bash
# Descargar la imagen
docker pull postgres:16

# Ejecutar el contenedor
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  -v postgres_data:/var/lib/postgresql/data \
  postgres:16
```

#### Docker Compose

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:16
    container_name: postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: mysecretpassword
      POSTGRES_DB: mydb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

Ejecutar:

```bash
docker-compose up -d
```

## Configuración Post-Instalación

### Conectar a PostgreSQL

```bash
# Usando psql
sudo -u postgres psql

# O
psql -U postgres -h localhost -p 5432
```

### Crear una Base de Datos

```sql
CREATE DATABASE mydb;

-- Crear usuario
CREATE USER myuser WITH PASSWORD 'mypassword';

-- Otorgar permisos
GRANT ALL PRIVILEGES ON DATABASE mydb TO myuser;
```

### Archivos de Configuración

#### postgresql.conf

```properties
# Dirección de escucha
listen_addresses = '*'

# Puerto
port = 5432

# Máximo de conexiones
max_connections = 100

# Buffer compartido
shared_buffers = 128MB

# Memoria de trabajo
work_mem = 4MB
```

#### pg_hba.conf

```properties
# Permitir conexiones locales
host    all             all             127.0.0.1/32            trust
host    all             all             ::1/128                 trust

# Permitir conexiones remotas
host    all             all             0.0.0.0/0              md5
host    all             all             ::/0                    md5
```

### Configuración de Conexión Remota

```sql
-- Permitir conexiones remotas
ALTER SYSTEM SET listen_addresses = '*';

-- Recargar configuración
SELECT pg_reload_conf();
```

## Preguntas Frecuentes

### ¿Conexión rechazada?

Verifica el firewall:

```bash
# Ubuntu
sudo ufw allow 5432/tcp

# CentOS
sudo firewall-cmd --add-port=5432/tcp --permanent
sudo firewall-cmd --reload
```

### ¿Olvidaste la contraseña de postgres?

```bash
# Detener PostgreSQL
sudo systemctl stop postgresql

# Iniciar en modo bypass
sudo -u postgres /usr/lib/postgresql/16/bin/pg_ctl -D /var/lib/postgresql/16/main -m fast stop

# Cambiar contraseña
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'newpassword';"
```

### ¿La base de datos no inicia?

Verifica los logs:

```bash
# Ver logs
tail -f /var/log/postgresql/postgresql-16-main.log
```
