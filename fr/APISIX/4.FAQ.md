# FAQ Apache APISIX

## Bases

### 1. Qu'est-ce qu'APISIX ?

APISIX est une passerelle API cloud-native offrant gestion dynamique du trafic.

### 2. APISIX vs Nginx

APISIX est construit sur Nginx avec support hot-update sans redémarrage.

### 3. Dépendances

- etcd: Stockage configuration
- Lua: Logique des plugins
- Nginx: Traitement HTTP

### 4. Performance

18 000 QPS par cœur, latence <0.2ms

## Installation

### 5. Erreur Quickstart

- Vérifier Docker
- Vérifier les ports
- Vérifier le réseau

### 6. Changer les Ports

Modifier docker-compose.yml ou config.yaml

### 7. Mise à niveau

```bash
docker pull apache/apisix:nouvelle-version
```

## Configuration

### 8. Clé Admin

Modifier dans config.yaml

### 9. HTTPS

Utiliser plugin acme ou certificats manuels

### 10. Multi-domaines

Utiliser champ "hosts"

## Fonctions

### 11. Canary Release

Plugin traffic-split avec pondération

### 12. Circuit Breaker

Health checks passifs

### 13. CORS

Plugin cors

### 14. Rate Limiting

Plugins limit-count, limit-req, limit-conn

### 15. Authentification

Créer consumer + activer plugin

## Monitoring

### 16. Prometheus

Activer plugin prometheus

### 17. Logs

Plugins http-logger, kafka-logger

## Sécurité

### 18. Recommandations

- Changer clés par défaut
- Restreindre accès admin
- Utiliser HTTPS
- Mettre à jour régulièrement

## Dépannage

### 19. Erreur 404

- Route existe ?
- URI correspond ?
- Backend joignable ?

### 20. Erreur 503

- Backend fonctionne ?
- Réseau OK ?
- Health checks trop stricts ?

### 21. Voir les Logs

```bash
docker logs apisix
```
