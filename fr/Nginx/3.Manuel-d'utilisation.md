# Manuel d'utilisation

#### Démarrage, arrêt et rechargement de la configuration

Pour démarrer nginx, exécutez le fichier exécutable :

```bash
nginx
```

Une fois démarré, nginx peut être contrôlé en invoquant l'exécutable avec le paramètre -s :

```bash
nginx -s signal
```

Où signal peut être :

- stop — Arrêt rapide
- quit — Arrêt gracieux
- reload — Rechargement du fichier de configuration
- reopen — Réouverture des fichiers journaux

Pour arrêter nginx en attendant que les processus workers terminent les requêtes en cours :

```bash
nginx -s quit
```

Pour recharger la configuration :

```bash
nginx -s reload
```

#### Structure du fichier de configuration

nginx est composé de modules contrôlés par des directives spécifiées dans le fichier de configuration.

Les directives sont divisées en directives simples et directives de bloc. Une directive simple consiste en un nom et des paramètres séparés par des espaces et se termine par un point-virgule (;).

Une directive de bloc a la même structure mais se termine par un ensemble d'instructions supplémentaires entre accolades ({ et }).

Si une directive de bloc peut contenir d'autres directives, elle est appelée contexte. Les contextes courants sont : events, http, server et location.

Les directives placées en dehors de tout contexte sont considérées comme étant dans le contexte principal. Le caractère # commence un commentaire.

#### Service de contenu statique

Voici un exemple de configuration pour servir des fichiers statiques :

```nginx
http {
    server {
        location / {
            root /data/www;
        }

        location /images/ {
            root /data;
        }
    }
}
```

La directive root spécifie le répertoire de base pour les fichiers demandés. L'URI de la requête est ajouté à ce chemin pour former le chemin complet du fichier.

#### Configuration d'un serveur proxy

L'une des utilisations fréquentes de nginx est de le configurer comme serveur proxy :

```nginx
server {
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

Directives importantes pour le proxy :

- proxy_pass — Spécifie l'URL du serveur proxy
- proxy_set_header — Permet de modifier ou ajouter des en-têtes de requête
- proxy_connect_timeout — Délai de connexion au serveur proxy
- proxy_read_timeout — Délai de lecture de la réponse
- proxy_send_timeout — Délai d'envoi de la requête

#### Équilibrage de charge

nginx prend en charge plusieurs méthodes d'équilibrage de charge :

```nginx
upstream backend {
    server 192.168.1.10:8080 weight=5;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080 backup;
}
```

Méthodes d'équilibrage :

- round-robin (par défaut) — Répartition égale entre les serveurs
- weight — Répartition pondérée selon le poids défini
- ip_hash —同一 IP toujours vers le même serveur
- least_conn — Serveur avec le moins de connexions actives

Paramètres de serveur :

- weight — Pondération du serveur
- max_fails — Nombre d'échecs avant considers le serveur indisponible
- fail_timeout — Durée de indisponibilité après max_fails
- backup — Marque ce serveur comme serveur de backup

#### Configuration SSL/TLS

```nginx
server {
    listen 443 ssl http2;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/certificat.crt;
    ssl_certificate_key /etc/nginx/ssl/clef.privee.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
}
```

#### Mise en cache

Configuration de base du cache :

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=cache_zone:10m 
                 max_size=1g inactive=60m;

location / {
    proxy_pass http://backend;
    proxy_cache cache_zone;
    proxy_cache_key "$scheme$request_method$host$request_uri";
    proxy_cache_valid 200 60m;
    proxy_cache_valid 404 1m;
    add_header X-Cache-Status $upstream_cache_status;
}
```

Directives de cache :

- proxy_cache_path — Définit le chemin et les paramètres du cache
- proxy_cache — Active le cache pour une zone
- proxy_cache_key — Définit la clé de cache
- proxy_cache_valid — Définit la durée de validité par code de réponse

#### Location et correspondance

Les blocs location supportent plusieurs types de correspondance :

- location / — Correspondance de préfixe (le plus long匹配优先)
- location = /exact — Correspondance exacte
- location ^~ /images — Préfixe,arrête les regex
- location ~ /api — Regex (sensible à la casse)
- location ~* \.(jpg|png)$ — Regex (insensible à la casse)

#### Compression gzip

Activation de la compression :

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
gzip_min_length 1024;
gzip_comp_level 6;
```

#### Commandes utiles

```bash
# Tester la configuration
nginx -t

# Voir les processus
ps -ef | grep nginx

# Logs
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log

# Arrêt gracieux
nginx -s quit

# Rechargement
nginx -s reload
```
