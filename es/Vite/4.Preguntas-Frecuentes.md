# Vite Preguntas Frecuentes

## Preguntas Generales

### ¿Qué es Vite?
Vite es una herramienta de construcción de frontend de próxima generación creada por Evan You. Ofrece HMR extremadamente rápido y builds de producción optimizados usando módulos ES nativos.

### ¿Vite solo es para Vue?
No, Vite es agnóstico de framework y funciona con Vue, React, Svelte, Preact y JavaScript vanilla.

### ¿Necesito usar un package manager específico?
Vite funciona con npm, yarn y pnpm. Usa el que prefieras.

## Preguntas de Instalación

### ¿Cómo creo un nuevo proyecto Vite?
```bash
# npm
npm create vite@latest my-project -- --template vue

# yarn
yarn create vite my-project -- --template vue

# pnpm
pnpm create vite my-project -- --template vue
```

### ¿Cómo instalo Vite en un proyecto existente?
```bash
npm install -D vite
```

### ¿Cuáles son las dependencias requeridas?
Para proyectos Vue:
```bash
npm install vue
npm install -D @vitejs/plugin-vue
```

Para proyectos React:
```bash
npm install react react-dom
npm install -D @vitejs/plugin-react
```

## Preguntas de Uso

### ¿Cómo inicio el servidor de desarrollo?
```bash
npm run dev
```

### ¿Cómo hago build para producción?
```bash
npm run build
```

### ¿Cómo previsualizo el build de producción?
```bash
npm run preview
```

### ¿Cómo configuro Vite?
Crea `vite.config.js`:

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8000'
    }
  },
  build: {
    outDir: 'dist',
    sourcemap: true
  }
})
```

## Solución de Problemas

### Módulo no encontrado
- Verificar que todas las dependencias estén instaladas: `npm install`
- Verificar que import paths sean correctos
- Asegurar que package.json tenga type: "module"

### CSS no se aplica
- Importar CSS en tu archivo JavaScript principal
- Verificar si se necesitan plugins PostCSS
- Verificar que path del archivo CSS sea correcto

### Puerto ya en uso
Cambiar puerto en vite.config.js:
```javascript
server: {
  port: 3001
}
```

### Carga inicial lenta
- Verificar network tab para dependencias grandes
- Optimizar imágenes y assets
- Habilitar lazy loading para rutas

## Preguntas de Producción

### ¿Cómo optimizo el tamaño del bundle?
- Usar dynamic imports para code splitting
- Configurar opciones de build:
```javascript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        vendor: ['react', 'react-dom'],
        utils: ['lodash']
      }
    }
  }
}
```

### ¿Cómo deployar a GitHub Pages?
```javascript
export default defineConfig({
  base: '/tu-nombre-repo/'
})
```

### ¿Cómo configuro variables de entorno?
Crea archivos `.env`:
```
VITE_API_URL=http://localhost:8000
```

Acceder en código:
```javascript
const apiUrl = import.meta.env.VITE_API_URL
```

## Preguntas Avanzadas

### ¿Cómo funciona el HMR de Vite?
Vite usa módulos ES nativos y establece una conexión WebSocket para HMR. Cuando un archivo cambia, Vite solo actualiza ese módulo sin recargar la página.

### ¿Puedo usar loaders de Webpack con Vite?
Usa `vite-plugin-webpack` o convierte loaders a plugins de Vite.

### ¿Cómo agrego plugins personalizados?
```javascript
function myPlugin() {
  return {
    name: 'my-plugin',
    transform(code, id) {
      // Transformar código
      return code
    }
  }
}

export default defineConfig({
  plugins: [myPlugin()]
})
```
