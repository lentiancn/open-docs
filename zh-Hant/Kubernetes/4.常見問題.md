# Kubernetes 常見問題

## 基本概念

### 什麼是 Kubernetes？

Kubernetes（也稱為 K8s）是一個開源的容器編排平台，用於自動化部署、擴展和管理容器化應用程式。它將組成應用程式的容器分組為邏輯單元，便於管理和服務發現。

### Pod 和容器有什麼差別？

Pod 是 Kubernetes 中的最小部署單元，一個 Pod 可以包含一個或多個容器。通常建議每個 Pod 只執行一個容器。同一 Pod 內的容器共享網路和儲存資源。

### Deployment 和 StatefulSet 有什麼差別？

Deployment 用於管理無狀態應用程式，支援宣告式更新、擴展和回滾。StatefulSet 用於管理有狀態應用程式，提供穩定的網路識別和持久化儲存，適合資料庫等需要持久化的應用。

### 什麼是 Namespace？

Namespace 用於將叢集資源劃分為多個邏輯分區，幫助在不同團隊或專案之間隔離資源。常見的預設命名空間包括 default、kube-system 和 kube-public。

## 安裝和配置

### Minikube 和 Kind 有什麼差別？

Minikube 建立一個完整的單節點 Kubernetes 叢集，適合學習和開發。Kind 使用 Docker 容器模擬 Kubernetes 節點，更輕量，適合快速測試。

### 如何選擇容器執行時？

常見的容器執行時包括 Docker、containerd 和 CRI-O。containerd 是 Docker 的核心容器執行時，目前被廣泛推薦使用。

### kubeadm 初始化失敗怎麼辦？

常見原因包括：
- 連接埠被佔用：檢查 6443、10250、10255 等連接埠
- Swap 未關閉：執行 `swapoff -a`
- 防火牆規則：確保節點間網路通訊正常
- SELinux/AppArmor：可以設為寬容模式或關閉

### 如何配置多主節點叢集？

使用 kubeadm 可以建立高可用性叢集：
1. 準備多個伺服器節點
2. 使用 kubeadm init 建立第一個控制平面節點
3. 產生憑證並複製到其他節點
4. 使用 kubeadm join 新增額外的控制平面節點

## 網路和儲存

### Pod 無法互相通訊怎麼辦？

常見原因包括：
- 網路插件未正確安裝
- 防火牆阻止了 Pod 網路 CIDR 的通訊
- CNI 插件配置錯誤
- 檢查 kube-proxy 是否正常執行

### 如何為 Pod 配置持久化儲存？

需要建立 PersistentVolume（PV）和 PersistentVolumeClaim（PVC）。PV 可以是本地儲存、網路儲存（NFS、iSCSI）或雲端儲存（AWS EBS、GCE PD、Azure Disk）。

### Service 類型有哪些？

- ClusterIP：叢集內部可存取的虛擬 IP
- NodePort：在每個節點上開放一個連接埠
- LoadBalancer：使用雲端供應商的負載均衡器
- ExternalName：將服務對應到外部網域

## 資源管理

### 如何設定容器資源限制？

在 Pod 或 Deployment 的 spec 中新增 resources 欄位：
```yaml
resources:
  limits:
    memory: "512Mi"
    cpu: "500m"
  requests:
    memory: "256Mi"
    cpu: "250m"
```

### 什麼是 ResourceQuota？

ResourceQuota 用於限制命名空間中的資源總量，防止單個團隊或專案消耗過多資源。

### 什麼是 LimitRange？

LimitRange 用於為命名空間中的 Pod 和容器設定預設的資源限制和請求值。

## 部署和運維

### 如何實現應用程式的滾動更新？

使用 Deployment 時，Kubernetes 會自動進行滾動更新：
```bash
kubectl set image deployment/my-app my-app=my-app:v2
kubectl rollout status deployment/my-app
```

### 如何回滾部署？

```bash
# 回滾到上一個版本
kubectl rollout undo deployment/my-app

# 回滾到指定版本
kubectl rollout undo deployment/my-app --to-revision=2
```

### 什麼是 HPA？

Horizontal Pod Autoscaler（HPA）根據 CPU、記憶體等指標自動調整 Pod 複本數。需要配合 Metrics Server 使用。

### 如何配置健康檢查？

使用探針（Probes）配置存活性和平就緒檢查：
```yaml
livenessProbe:
  httpGet:
    path: /healthz
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

## 安全

### 什麼是 RBAC？

基於角色的存取控制（RBAC）用於管理 Kubernetes 叢集中的權限。可以建立 Role 和 ClusterRole，並綁定到使用者或群組。

### 如何管理敏感資訊？

使用 Secret 儲存敏感資訊，如密碼、金鑰和憑證。可以使用 kubectl 命令列建立或透過 YAML 檔案定義。

### 如何保護叢集安全？

- 啟用 RBAC 授權
- 使用網路原則限制 Pod 間的通訊
- 定期更新 Kubernetes 版本
- 加密 etcd 資料
- 使用 TLS 加密通訊
- 限制 kubelet 的存取權限

## 故障排除

### Pod 一直處於 Pending 狀態？

可能是以下原因：
- 資源不足（CPU 或記憶體）
- 節點選擇器或污點不相符
- PVC 未正確綁定
- 使用 kubectl describe pod <pod-name> 查看詳細資訊

### Pod 一直處於 ImagePullBackOff？

常見原因：
- 映像檔名稱錯誤
- 映像檔標籤不存在
- 私人倉庫認證問題
- 網路問題導致無法拉取映像檔

### Pod 一直處於 CrashLoopBackOff？

- 檢查容器日誌：kubectl logs --previous <pod-name>
- 檢查應用程式是否正確啟動
- 驗證資源配置是否正確
- 檢查相依服務是否可用

### 如何偵錯網路問題？

- 檢查 kube-proxy 是否正常執行
- 檢查 CNI 插件狀態
- 使用 kubectl exec 進入容器測試網路連通性
- 查看 iptables 規則
- 使用網路診斷工具（如 curl、ping、traceroute）

## 效能優化

### 如何提高叢集效能？

- 合理配置資源請求和限制
- 使用反親和性分布 Pod
- 啟用快取減少 API 請求
- 使用本地儲存減少網路延遲
- 優化映像檔大小減少拉取時間

### 如何處理大規模叢集？

- 使用聯邦（Federation）管理多個叢集
- 配置合適的 etcd 硬體
- 啟用 API 快取
- 使用負載均衡分散請求
