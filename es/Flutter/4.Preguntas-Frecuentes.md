# Flutter - Preguntas Frecuentes respuestas comunes sobre el desarrollo con

Preguntas y Flutter.

## Problemas de Instalación

### Q: ¿Descarga de Flutter fallida?

**Solución:**
1. Usar espejo:
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
export PUB_CACHE=/ruta/al/cache
```
2. Usar proxy
3. Descargar manualmente el SDK

### Q: ¿Error de flutter doctor?

**Solución:**
1. Verificar configuración de PATH
2. Asegurarse que Git está instalado
3. Ejecutar `flutter doctor -v` para información detallada

### Q: ¿Android SDK no encontrado?

**Solución:**
```bash
flutter config --android-sdk /ruta/a/android/sdk

# O establecer variable de entorno
export ANDROID_SDK_ROOT=/ruta/a/android-sdk
```

### Q: ¿Herramientas de línea de comandos de Xcode no instaladas en macOS?

**Solución:**
```bash
xcode-select --install
```

## Problemas de Desarrollo

### Q: ¿Hot reload no funciona?

**Solución:**
1. Guardar archivo y esperar hot reload
2. Ver consola por errores
3. Intentar reiniciar completamente la aplicación

### Q: ¿Compilación de APK fallida?

**Solución:**
1. Ejecutar `flutter clean`
2. Actualizar Gradle
3. Verificar conexión de red

### Q: ¿Inicio de emulador fallido?

**Solución:**
1. Habilitar virtualización de CPU
2. Verificar instalación de HAXM
3. Usar dispositivo físico para depuración

### Q: ¿Widget no se muestra?

**Solución:**
1. Verificar retorno del método build()
2. Confirmar agregado a MaterialApp
3. Verificar restricciones de layout

### Q: ¿Estado no se actualiza?

**Solución:**
1. Asegurarse que setState() es llamado
2. Verificar StatefulWidget correcto
3. Confirmar uso de Context correcto

### Q: ¿Fuga de memoria?

**Solución:**
1. Disponer controladores correctamente
2. Limpiar Provider adecuadamente
3. Evitar crear objetos en build

## Problemas de Rendimiento

### Q: ¿Aplicación lenta?

**Solución:**
1. Usar constructores const
2. Evitar setState innecesario
3. Usar ListView.builder
4. Optimizar carga de imágenes

### Q: ¿APK muy grande?

**Solución:**
1. Usar minificación
2. Eliminar símbolos de debug
3. Usar split-per-abi
4. Limpiar recursos no usados

### Q: ¿Inicio lento?

**Solución:**
1. Reducir operaciones init()
2. Cargar componentes lazy
3. Usar Deferred Components

## Problemas de Plataforma

### Q: ¿Compilación de iOS fallida?

**Solución:**
1. Verificar versión de Xcode
2. Actualizar CocoaPods
3. Verificar certificados de firma

### Q: ¿Compilación web fallida?

**Solución:**
1. Confirmar soporte de Flutter Web
2. Verificar compatibilidad de navegador
3. Actualizar Chrome

### Q: ¿Canal de plataforma no funciona?

**Solución:**
1. Verificar coincidencia de nombres de método
2. Confirmar tipos de retorno
3. Ver errores de consola
