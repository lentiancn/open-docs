# JIRA インストールガイド

JIRAは、Atlassian社が開発した問題追跡およびプロジェクト管理ツールです。このガイドでは、さまざまな環境でのインストール方法について説明します。

## バージョン情報

| バージョン | ステータス | 説明 |
|------------|------------|------|
| JIRA 9.x | ✅ 最新 | 現在のバージョン |
| JIRA 8.x | ✅ サポート | 安定バージョン |

## システム要件

### ハードウェア要件

| 要件 | 最小構成 | 推奨構成 |
|------|----------|----------|
| CPU | 4コア | 8+コア |
| メモリ | 4 GB | 16+ GB |
| ディスク | 10 GB | 50+ GB |

### ソフトウェア要件

| ソフトウェア | バージョン |
|--------------|------------|
| Java | JDK 11 または 17 |
| OS | Linux/Windows/macOS |
| データベース | PostgreSQL/MySQL/Oracle |

## インストール方法

### 方法1：Windows インストール

#### 1. インストーラーをダウンロード

1. https://www.atlassian.com/software/jira/download にアクセス
2. JIRA Softwareをダウンロード
3. インストーラーを実行

#### 2. インストールを実行

インストールウィザードに従ってインストールを完了：
- インストールタイプを選択
- ポートを設定
- データベースを設定
- 管理者アカウントを作成

### 方法2：Linux インストール

#### 1. インストールパッケージをダウンロード

```bash
# JIRAをダウンロード
wget https://product-downloads.atlassian.com/software/jira/downloads/atlassian-jira-software-X.X.X-x64.bin
```

#### 2. インストール

```bash
# 実行権限を追加
chmod +x atlassian-jira-software-X.X.X-x64.bin

# インストールを実行
sudo ./atlassian-jira-software-X.X.X-x64.bin
```

#### 3. 設定

```bash
# JIRAを起動
sudo /opt/atlassian/jira/bin/start-jira.sh

# JIRAを停止
sudo /opt/atlassian/jira/bin/stop-jira.sh
```

### 方法3：Docker インストール

```bash
# イメージをプル
docker pull atlassian/jira-software

# コンテナを実行
docker run -v jira-data:/var/atlassian/application-data/jira --name="jira" -d -p 8080:8080 atlassian/jira-software
```

### 方法4：AWSを使用

1. AWS MarketplaceでJIRAを検索
2. インスタンスタイプを選択
3. データベースを設定
4. インスタンスを起動

## 設定

### データベース設定

```properties
# dbconfig.xml
<?xml version="1.0" encoding="UTF-8"?>
<jira-database-config>
  <name>defaultDS</name>
  <connection-type>default</connection-type>
  <url>jdbc:postgresql://localhost:5432/jira</url>
  <username>jira</username>
  <password>password</password>
  <pool-size>20</pool-size>
</jira-database-config>
```

### リバースプロキシ設定

#### Nginx

```nginx
server {
    listen 80;
    server_name jira.example.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## よくある質問

### 起動に失敗する場合

Javaバージョンを確認：

```bash
java -version
```

### データベース接続に失敗する場合

データベース設定と接続情報を確認してください。

### メモリ不足の場合

メモリ設定を変更：

```bash
# setenv.sh
export JVM_MINIMUM_MEMORY="2048m"
export JVM_MAXIMUM_MEMORY="4096m"
```
