# OpenClaw 常見問題

這裡彙總了關於 OpenClaw 的常見問題及解答。

## 基礎問題

### OpenClaw 是什麼？

OpenClaw 是一個可以部署在自有設備上的工具，它能把常用的通訊軟體和 AI 服務連接起來。讓你可以在常用的聊天軟體裡直接用上 AI 助手。

### OpenClaw 需要付費嗎？

基礎使用是免費的，因為它是開源項目。但運行它需要：
- 一台電腦或伺服器
- AI 服務的 API 密鑰（按用量收費）

### 需要購買 AI 服務商的訂閱嗎？

不一定。你可以選擇按調用次數付費的 API 密鑰，也可以使用訂閱式的帳號。OpenClaw 支援 Anthropic、OpenAI、Google Gemini 等多種服務商。

### 可以在哪些平台上運行？

- 電腦系統：macOS、Linux、Windows
- 手機端：iOS、Android（作為配套節點）
- 通訊軟體：WhatsApp、Telegram、Discord、iMessage、Signal、Slack 等 20 多種

## 安裝問題

### 安裝失敗了怎麼辦？

1. 檢查 Node.js 版本：`node --version`（需要 22 或更高）
2. 執行診斷：`openclaw doctor`
3. 查看日誌：`openclaw logs`

### 能在樹莓派上用嗎？

可以。樹莓派 4 或更新型號配合 Raspberry Pi OS 或 Ubuntu 都可以運行 OpenClaw。

### 需要用 Mac 電腦嗎？

不需要。OpenClaw 可以在任何安裝了 Node.js 的系統上運行。只有 iMessage 這個渠道需要 Mac（因為 Apple 的限制）。

## 配置問題

### 配置文件在哪裡？

預設位置是 `~/.openclaw/openclaw.json`。

### 如何設置誰能發訊息給我？

通過 `dmPolicy` 參數來控制：
- `pairing`：陌生人需要配對碼才能對話
- `allowlist`：只有白名單裡的用戶可以發訊息
- `open`：所有人都可以發訊息
- `disabled`：關閉私人訊息功能

### 群聊怎麼設置？

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## 使用問題

### 如何添加新的聊天渠道？

```bash
openclaw channels login whatsapp
```

### 可以同時用好幾個渠道嗎？

可以。一個 Gateway 進程可以同時連接多個聊天渠道。

### AI 怎么能記住之前的對話？

OpenClaw 會自動保存對話上下文。你可以在配置裡調整記憶的保留策略。

### 能同時運行多個 AI 代理嗎？

可以。配置多個代理並設置不同的路由規則。

## 安全相關

### 我的資料安全嗎？

是的。OpenClaw 是部署在本地的，所有資料都保存在你的設備上，不會發送到第三方伺服器。

### 怎麼保護我的服務安全？

1. 通過 `allowFrom` 設置可以訪問的來源
2. 使用強密碼和安全的 API 密鑰
3. 保持 OpenClaw 版本更新
4. 用防火牆限制開放連接埠

## 問題排查

### 服務啟動不了

1. 檢查連接埠是否被佔用：`openclaw gateway status`
2. 查看錯誤日誌：`openclaw logs`
3. 執行診斷：`openclaw doctor`

### AI 不回覆

1. 確認 API 密鑰配置正確
2. 檢查 AI 服務商帳號餘額是否充足

### 訊息發不出去

1. 檢查渠道是否已經登入成功
2. 確認對方帳號或群組是否存在
