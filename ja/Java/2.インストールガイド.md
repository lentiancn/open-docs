# Java インストールガイド

## Java開発キットの入手

Oracle社はJava開発キット（JDK）の無料ダウンロードを提供しています。現在の主要なLTSバージョンには、JDK 25、JDK 21、JDK 17、JDK 11、JDK 8があります。

公式ダウンロードページ：https://www.oracle.com/java/technologies/downloads/

## Windowsシステムへのインストール

### インストーラーを使用する場合

1. Oracle社の公式ダウンロードページにアクセスし、Windows版のJDKを選択してください
2. x64 Installer（exeファイル）またはMSI Installerをダウンロードしてください
3. ダウンロードしたインストーラーを実行してください
4. インストールウィザードの指示に従ってインストールを完了してください。デフォルトのインストールパスはC:\Program Files\Java\jdk-versionです
5. インストールが完了したら、インストールが成功したか確認してください

### Zipアーカイブを使用する場合

1. x64 Compressed Archive（zipファイル）をダウンロードしてください
2. 指定したディレクトリ（例：C:\Java\jdk-25）に展開してください
3. 環境変数を設定してください

### 環境変数の設定

1. 「このPC」を右クリックし、「プロパティ」を選択してください
2. 「システムの詳細設定」をクリックしてください
3. 「環境変数」をクリックしてください
4. システム環境変数で、「新規」をクリックしてください
   - 変数名：JAVA_HOME
   - 変数値：JDKのインストールパス（例：C:\Program Files\Java\jdk-25）
5. Path変数を見つけ、「編集」をクリックしてください
6. 新しい変数を追加してください：%JAVA_HOME%\bin
7. OKをクリックして保存してください

### インストール確認

コマンドプロンプトを開いて、以下のコマンドを入力してください：

```cmd
java -version
javac -version
```

バージョン情報が表示されれば、インストールは成功です。

## Linuxシステムへのインストール

### アーカイブからインストールする場合

1. Linux版のJDK（tar.gz形式）をダウンロードしてください
2. ターミナルを開いて、指定したディレクトリに展開してください：

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. 環境変数を設定してください。~/.bashrcまたは/etc/profileを編集してください：

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. 設定を反映させてください：

```bash
source ~/.bashrc
```

5. インストールを確認してください：

```bash
java -version
```

### パッケージマネージャーを使用する場合

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## macOSシステムへのインストール

### DMGインストーラーを使用する場合

1. macOS版のJDK（dmgファイル）をダウンロードしてください
2. ダウンロードしたdmgファイルを開いてください
3. JDKをApplicationsフォルダにドラッグしてください
4. ターミナルを開いて、環境変数を設定してください：

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. 上記の内容を~/.zshrcまたは~/.bash_profileに追加してください
6. インストールを確認してください：

```bash
java -version
```

### Homebrewを使用する場合

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## 複数のJDKバージョンの共存

### Linuxでupdate-alternativesを使用する場合

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Windowsでバージョンを切り替える場合

JAVA_HOME環境変数を異なるJDKディレクトリに指向させることで、バージョンを切り替えられます。

### SDKMANを使用する場合

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Java環境の確認

### Javaバージョンの確認

```bash
java -version
```

出力例：

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### コンパイラバージョンの確認

```bash
javac -version
```

出力例：

```
javac 25.0.1
```

### JAVA_HOMEの確認

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## 一般的なインストール問題

### 問題1：javaコマンドが利用できない

**解決策**：Path環境変数にJDKのbinディレクトリが含まれているか確認してください。

### 問題2：権限が不十分

**解決策**：Linux/macOSではsudoを使用して権限を昇格させるか、ファイルの権限を確認してください。

### 問題3：ポートが既に使用されている

**解決策**：Javaアプリケーションはデフォルトで8080ポートを使用します。ポートが他のプログラムによって使用されていないか確認してください。

### 問題4：JDKとJREの違い

- **JDK**（Java Development Kit）：開発ツールとJREを含むJava開発キット
- **JRE**（Java Runtime Environment）：Javaプログラムの実行に必要なコンポーネントのみを含むJavaランタイム環境

Javaプログラムの開発にはJDKのインストールが必要です。コンパイル済みのJavaプログラムの実行にはJREだけで十分です。

## インストール後の設定アドバイス

1. MavenまたはGradを使用する場合は、MAVEN_HOMEまたはGRADLE_HOMEを設定してください
2. Mavenローカルリポジトリのパスを設定する場合は、M2_HOMEを設定してください
3. 企業環境では外部ネットワークへのアクセスにプロキシ設定が必要な場合があります
4. Javaアプリケーションが正しいタイムゾーンを使用するように設定してください

## Javaのアンインストール

### Windowsのコントロールパネルからアンインストール

1. コントロールパネルを開いてください
2. 「プログラムと機能」を選択してください
3. Java関連のプログラムを見つけてください
4. 右クリックして「アンインストール」を選択してください

### Linuxでのアンインストール

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOSでのアンインストール

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
