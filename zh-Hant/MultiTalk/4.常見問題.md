# MultiTalk 常見問題

本頁面匯總了使用者在使用 MultiTalk 時經常遇到的疑問及其解答。

## 基礎問題

### MultiTalk 是什麼？

MultiTalk 是由梅津人工智慧團隊開發的音訊驅動多人對話影片生成框架。它能夠根據多流音訊輸入、參考圖像和文字提示，生成包含多人互動的高品質影片內容，支援口型與音訊同步。

### MultiTalk 支援哪些功能？

MultiTalk 支援多種功能：單人物和多人物對話影片生成、卡通角色影片生成、唱歌影片生成、指令跟隨影片生成。支援輸出 480P 和 720P 解析度，最長可生成 15 秒的影片片段。

### MultiTalk 是免費的嗎？

MultiTalk 是開源專案，可以免費使用。專案採用 Apache 2.0 許可證，允許學術和商業使用。

## 安裝問題

### 安裝過程中出現 CUDA 版本不相容怎麼辦？

MultiTalk 需要 CUDA 12.1 或更高版本。請先檢查系統 CUDA 版本：nvcc --version。如果版本過低，需要升級 CUDA 驅動程式。也可以嘗試使用對應的 PyTorch CUDA 版本。

### 記憶體不足如何處理？

如果記憶體不足，可以嘗試以下方法：使用 --num_persistent_param_in_dit 0 參數降低記憶體佔用；使用 TeaCache 加速減少記憶體使用；使用 LoRA 加速減少步數；降低輸出解析度。

### flash_attn 安裝失敗怎麼辦？

flash_attn 安裝需要特定的編譯環境。確保已安裝 ninja 和正確的 CUDA 工具鏈。如果持續失敗，可以嘗試安裝預編譯版本或跳過該依賴。

### 模型下載速度太慢怎麼辦？

可以使用鏡像源或透過其他下載工具預先下載模型檔案，然後移動到指定目錄。也可以嘗試使用國內鏡像源。

## 使用問題

### 生成的影片口型不同步怎麼辦？

可以嘗試增加 Audio CFG 參數值，推薦設定在 3 到 5 之間。也可以增加採樣步數來提升同步效果。

### 影片生成速度太慢怎麼辦？

可以啟用以下加速方式：使用 TeaCache 加速，可提升 2 到 3 倍速度；使用 LoRA 加速，FusioniX 只需 8 步，lightx2v 只需 4 步；使用量化版本減少計算量。

### 如何生成更長的人物對話影片？

使用 streaming 模式可以生成長影片。確保 Audio CFG 設定為 3 以保持色調一致。生成的影片最長可達 15 秒。

### 多人影片中人物綁定錯誤怎麼辦？

這是多人影片生成的常見問題。MultiTalk 透過 L-RoPE 技術解決此問題。確保輸入 JSON 中正確指定了每個人物區域的座標。

### 如何使用 TTS 文字轉語音功能？

準備包含文字的輸入 JSON 檔案，設定 --audio_mode tts 參數即可自動將文字轉為語音並生成影片。

### 可以生成卡通角色影片嗎？

是的，MultiTalk 支援卡通角色生成。在參考圖像中使用卡通人物圖片，配合相應描述文字即可。

## 技術問題

### 支援哪些輸入音訊格式？

支援常見的音訊格式，如 WAV、MP3 等。音訊中應包含清晰的人聲對話。

### 影片解析度如何選擇？

目前版本預設支援 480P 解析度。720P 需要多 GPU 配置。480P 足以滿足大多數使用場景。

### 如何在低記憶體裝置上執行？

使用 --num_persistent_param_in_dit 0 參數可以大幅降低記憶體需求，在單張 RTX 4090 上也能執行。

### 如何整合到其他平台使用？

MultiTalk 支援多種使用方式：可以透過 Replicate 雲端平台直接呼叫；可以整合到 ComfyUI 工作流；也可以使用專案提供的 Gradio 介面。

## 許可證問題

### 生成的影片可以商用嗎？

專案採用 Apache 2.0 許可證，生成的影片內容可用於商業用途。但請確保使用方式符合許可證條款，不違反相關法律法規。

### 使用時需要引用什麼？

如果 MultiTalk 對您的研究有幫助，請引用相關論文。論文資訊可在專案 GitHub 頁面上找到。
