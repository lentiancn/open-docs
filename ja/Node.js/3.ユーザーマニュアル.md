# Node.js ユーザーマニュアル

## はじめに

### 基本概念

Node.jsはChromeのV8 JavaScriptエンジン上に構築されており、イベント駆動型でノンブロッキングI/Oモデルを提供します。これにより、データ集約型のリアルタイムアプリケーションにとって軽量かつ効率的になります。

#### イベントループ
Node.jsのイベントループは、並行性モデルの中核です。従来のスレッドベースのモデルとは異なり、Node.jsは単一スレッドのイベントループを使用し、非同期操作をコールバック、Promise、またはasync/awaitを通じて処理します。

#### ノンブロッキングI/O
Node.jsのほとんどの操作はノンブロッキングであり、I/O操作の完了を待っている間にプログラムの実行を停止しません。代わりに、結果の準備ができたときにコールバックを使用して結果を処理します。

### Hello Worldアプリケーション

このシンプルなHTTPサーバーで最初のNode.jsアプリケーションを作成します：

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

`node server.js`で実行します。

## コアモジュール

Node.jsには、外部依存関係を必要とせずに重要な機能を提供する豊富な組み込みモジュールセットが付属しています。

### ファイルシステム（fs）
同期または非同期でファイル操作を処理します：

```javascript
const fs = require('fs');

// 非同期読み取り
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// 同期読み取り（実行をブロック）
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// ファイル書き込み
fs.writeFile('/path/to/output.txt', 'Hello Node.js!', (err) => {
  if (err) throw err;
  console.log('File written successfully');
});
```

### HTTPサーバー
ウェブサーバーを作成し、HTTPリクエストを行います：

```javascript
const http = require('http');
const https = require('https');

// HTTPサーバーを作成
const server = http.createServer((req, res) => {
  // リクエストを処理
});

// HTTPリクエストを行う
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });
});
req.end();
```

### パスユーティリティ
クロスプラットフォームな方法でファイルパスを処理します：

```javascript
const path = require('path');

// パスを結合
const fullPath = path.join('/home', 'user', 'documents', 'file.txt');

// ディレクトリ名を取得
const dir = path.dirname('/home/user/documents/file.txt');

// ファイル拡張子を取得
const ext = path.extname('file.txt'); // '.txt'

// 絶対パスを解決
const absPath = path.resolve('./relative/path');
```

### イベント
EventEmitterクラスを使用してカスタムイベントを処理します：

```javascript
const EventEmitter = require('events');

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();

myEmitter.on('event', () => {
  console.log('An event occurred!');
});

myEmitter.emit('event');
```

## npmによるパッケージ管理


## エラーハンドリング

非同期性のあるNode.jsアプリケーションでは、適切なエラーハンドリングが重要です。

### コールバックベースのエラーハンドリング
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Error reading file:', err);
    return;
  }
  console.log('File content:', data);
});
```

### Promiseベースのエラーハンドリング
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('File content:', data);
  })
  .catch(err => {
    console.error('Error reading file:', err);
  });
```

### Async/Awaitエラーハンドリング
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('File content:', data);
  } catch (err) {
    console.error('Error reading file:', err);
  }
}
```

## パフォーマンス最適化

### クラスタリング
複数のCPUコアを活用します：

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);
  
  // ワーカーをフォーク
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died`);
  });
} else {
  // ワーカーはTCP接続を共有
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('Hello World\n');
  }).listen(8000);
  
  console.log(`Worker ${process.pid} started`);
}
```

### メモリ管理
- イベントリスナーを適切にクリーンアップしてメモリリークを回避する
- 大きなファイルを処理する場合は、すべてをメモリに読み込む代わりにストリームを使用する
- `process.memoryUsage()`でメモリ使用量を監視する

### キャッシュ
パフォーマンスを向上させるためのキャッシング戦略を実装します：

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```

### ロギング
本番アプリケーションのための適切なロギングを実装します：

```javascript
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));
}
```

### パフォーマンス監視

```javascript
const { PerformanceObserver, performance } = require('perf_hooks');

const obs = new PerformanceObserver((items) => {
  console.log(items.getEntries());
});

obs.observe({ entryTypes: ['measure'] });

performance.mark('start');
// あなたのコードはこちら
performance.mark('end');
performance.measure('My Operation', 'start', 'end');
```

悪用を防ぐためにレート制限を実装します：

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分
  max: 100 // 各IPをwindowMsごとに100リクエストに制限
});

app.use(limiter);
```
