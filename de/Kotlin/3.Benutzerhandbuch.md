# Kotlin Benutzerhandbuch

## Variablen und Grundlegende Typen

Kotlin verwendet zwei Schlüsselwörter zur Variablendeklaration: var und val. var deklariert eine veränderliche Variable, ähnlich einer regulären Variable in Java. val deklariert eine unveränderliche Variable, ähnlich einer final-Variable in Java. In Kotlin ist die Bevorzugung von val eine Best Practice, die den Code sicherer und leichter verständlich macht.

Bei der Deklaration von Variablen können Sie den Typ explizit angeben oder den Compiler den Typ ableiten lassen:

val name: String = "Kotlin"
val age = 10 // Compiler leitet Int-Typ ab
var count = 0 // Veränderliche Variable

Kotlins grundlegende Typen umfassen: Ganzzahlen (Int, Long, Short, Byte), Gleitkommazahlen (Double, Float), Zeichen (Char), Booleans (Boolean) und Strings. Anders als in Java sind alle grundlegenden Typen in Kotlin Objekte mit entsprechenden Membermethoden.

Kotlin-Ganzzahlen sind standardmäßig vom Typ Int. Um größere Zahlen darzustellen, verwenden Sie das Long-Suffix:

val longNumber = 100L // Long-Typ

Kotlin-Strings sind leistungsstark und unterstützen String-Vorlagen, die das Einbetten von Variablen oder Ausdrücken direkt in Strings ermöglichen:

val name = "World"
println("Hello, $name!") // Gibt aus: Hello, World!
println("Length: ${name.length}") // Gibt aus: Length: 5

## Sammlungen

Die Kotlin-Standardbibliothek bietet reichhaltige Sammlungstypen, darunter List (Liste), Set (Menge) und Map (Abbildung). Jeder Sammlungstyp hat sowohl unveränderliche als auch veränderliche Versionen.

Unveränderliche Sammlungen können nach der Erstellung nicht hinzugefügt, entfernt oder geändert werden:

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

Veränderliche Sammlungen erlauben Elementänderungen:

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Kotlin-Sammlungen bieten reichhaltige funktionale Operationen wie map, filter und reduce:

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## Kontrollfluss

Kotlins Kontrollfluss umfasst bedingte Ausdrücke, Schleifen und Sprungausdrücke.

Der if-Ausdruck in Kotlin gibt einen Wert zurück und kann direkt einer Variablen zugewiesen werden:

val max = if (a > b) a else b

Der when-Ausdruck ist Kotlins leistungsstarkes Multi-Branch-Matching-Tool, das verschiedene Bedingungen abgleichen kann:

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Kotlins Schleifen umfassen for-Schleifen und while-Schleifen. for-Schleifen können über Bereiche, Arrays und Sammlungen iterieren:

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## Funktionen

Kotlin verwendet das Schlüsselwort fun zur Funktionsdeklaration. Funktionen können Parametertypen und Rückgabetypen angeben:

fun greet(name: String): String {
    return "Hello, $name!"
}

Wenn der Funktionskörper nur einen Ausdruck enthält, können Sie die Einzeilausdrucks-Syntax verwenden:

fun greet(name: String) = "Hello, $name!"

Kotlin unterstützt Standardargumente und benannte Argumente, was den Bedarf an Funktionsüberladung erheblich reduziert:

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin unterstützt auch variadische Parameter mit dem Schlüsselwort vararg:

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## Klassen und Objekte

Kotlin-Klassendeklarationen sind prägnanter als Java. Standardmäßig sind Kotlin-Klassen final und können nicht vererbt werden. Um eine Klasse vererbbar zu machen, verwenden Sie das Schlüsselwort open:

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Kotlins Konstruktoren umfassen Haupt- und Nebenkonstruktoren. Der Hauptkonstruktor wird direkt nach dem Klassennamen deklariert:

class Person(val name: String, val age: Int)

Wenn Sie Logik im Konstruktor ausführen müssen, können Sie einen init-Block verwenden:

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Kotlins Datenklassen (data class) generieren automatisch equals-, hashCode-, toString- und copy-Methoden, was sie perfekt für Klassen macht, die Daten speichern:

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Nullsicherheit

Nullsicherheit ist eine der wichtigsten Funktionen von Kotlin. In Kotlin unterscheidet das Typsystem zwischen Nullable- und Nicht-Nullable-Typen, was NullPointerExceptions grundlegend reduziert.

Standardmäßig sind Typen nicht nullable:

val name: String = "Kotlin" // Kann nicht null sein
val name: String = null // Kompilierungsfehler

Um eine Variable als null zulassen zu können, verwenden Sie Nullable-Typen (?):

val name: String? = null // Kann null sein

Beim Zugreifen auf Eigenschaften oder Methoden von Nullable-Typen verwenden Sie den sicheren Aufrufoperator (?.) oder die Nicht-Null-Assertion (!!):

val length = name?.length // Gibt null zurück, wenn name null ist
val length = name!!.length // Wirft Ausnahme, wenn name null ist

Der Elvis-Operator (?:) bietet eine prägnante Möglichkeit, Standardwerte zu behandeln:

val length = name?.length ?: 0 // Gibt 0 zurück, wenn name null ist

## Erweiterungsfunktionen und -eigenschaften

Kotlin ermöglicht das Hinzufügen neuer Funktionen und Eigenschaften zu bestehenden Klassen, ohne die Originalklasse zu erben oder zu verändern. Diese werden Erweiterungsfunktionen und Erweiterungseigenschaften genannt.

Erweiterungsfunktionen werden deklariert, indem der Klassenname, der erweitert werden soll, dem Funktionsnamen vorangestellt wird:

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

Erweiterungseigenschaften können auch neue Eigenschaften zu bestehenden Klassen hinzufügen:

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## Funktionen höherer Ordnung und Lambdas

Funktionen höherer Ordnung sind Funktionen, die Funktionen als Parameter nehmen oder Funktionen zurückgeben. Lambda-Ausdrücke sind die Hauptmethode zur Implementierung dieser Funktion.

Lambda-Ausdruckssyntax verwendet geschweifte Klammern:

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

Wenn ein Lambda der letzte Parameter einer Funktion ist, kann es außerhalb der Klammern verschoben werden:

list.filter { it > 0 }

In Lambda-Ausdrücken steht it für den impliziten Namen eines einzelnen Parameters. Wenn nur ein Parameter vorhanden ist, können Sie die Parameterdeklaration weglassen.

## Koroutinen

Koroutinen sind das Kernfeature der asynchronen Programmierung in Kotlin. Im Vergleich zu Threads sind Koroutinen viel leichtgewichtiger und können mehrere Koroutinen in einem einzigen Thread ausführen.

launch ist eine gängige Methode zum Starten von Koroutinen, die eine neue Koroutine im Hintergrund startet:

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

Suspend-Funktionen können nur innerhalb von Koroutinen aufgerufen werden:

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        // Zeitaufwändige Operation ausführen
    }
}

Kotlin-Koroutinen bieten mehrere Dispatcher, um zu steuern, in welchem Thread Koroutinen ausgeführt werden: Dispatchers.Main für UI-Operationen, Dispatchers.IO für I/O-Operationen und Dispatchers.Default für CPU-intensive Aufgaben.

## Versiegelte Klassen und Enums

Versiegelte Klassen sind eingeschränkte Klassenhierarchien, die die Anzahl der Unterklassen begrenzen, die von ihr erben können. Alle Unterklassen müssen in derselben Datei deklariert werden. Dies ist sehr nützlich bei der Darstellung endlicher Zustände oder Ergebnistypen:

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

Der when-Ausdruck kann die Funktionen versiegelter Klassen vollständig nutzen, wobei der Compiler überprüft, ob alle möglichen Fälle behandelt werden:

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

Enums repräsentieren eine endliche Menge konstanter Werte:

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

Enums können Eigenschaften und Methoden haben:

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## Schnittstellen und abstrakte Klassen

Schnittstellen in Kotlin können Standardimplementierungen enthalten, was sie flexibler macht:

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
)

Klassen können mehrere Schnittstellen implementieren:

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## Objektausdrücke und Objektdeklarationen

Objektausdrücke erstellen Instanzen anonymer Klassen, ähnlich wie Java anonyme innere Klassen:

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // Klickereignis behandeln
    }
})

Objektdeklarationen erstellen Singletons:

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## Begleitobjekte

Jede Klasse kann ein Begleitobjekt enthalten, ähnlich wie statische Mitglieder in Java:

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

Auf Mitglieder des Begleitobjekts kann direkt über den Klassennamen zugegriffen werden:

MyClass.TAG
MyClass.create()

## Typüberprüfungen und -konvertierungen

Der Operator is wird zur Typüberprüfung verwendet, ähnlich wie Java instanceof:

if (obj is String) {
    // In diesem Zweig wird obj automatisch in String konvertiert
    println(obj.length)
}

Intelligente Typumwandlungen konvertieren automatisch Typen, die überprüft wurden. Wenn der Compiler den Typ der Variable in einem bestimmten Zweig ableiten kann, führt er automatisch die Konvertierung durch.

Wenn intelligente Umwandlungen nicht anwendbar sind, können Sie explizite Konvertierung verwenden:

val str = obj as String

Sichere Typkonvertierung verwendet as?, das null zurückgibt, wenn die Konvertierung fehlschlägt:

val str = obj as? String // Gibt null zurück, wenn die Konvertierung fehlschlägt
