# Vue.js Benutzerhandbuch

## Anwendungsinstanz

Jede Vue-Anwendung beginnt mit der Erstellung einer neuen Anwendungsinstanz mit der Funktion `createApp`:

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* Optionen der Root-Komponente */
})
```

### Root-Komponente
Das Objekt, das wir an `createApp` übergeben, ist tatsächlich eine Komponente. Jede Anwendung benötigt eine "Root-Komponente", die andere Komponenten als ihre Kinder enthalten kann.

### Bereitstellen der Anwendung
Eine Anwendungsinstanz rendert nichts, bis ihre `.mount()`-Methode aufgerufen wird:

```javascript
app.mount('#app')
```

### Anwendungskonfigurationen
Die Anwendungsinstanz stellt ein `.config`-Objekt bereit, das es uns ermöglicht, Optionen auf Anwendungsebene zu konfigurieren:

```javascript
// Anwendungsweiter Fehlerhandler
app.config.errorHandler = (err) => {
  /* Fehler behandeln */
}

// Globale Komponente registrieren
app.component('TodoDeleteButton', TodoDeleteButton)
```

### Mehrere Anwendungsinstanzen
Sie können mehrere Vue-Anwendungen auf derselben Seite erstellen:

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## Templatesyntax

Vue verwendet eine HTML-basierte Templatesyntax, die es Ihnen ermöglicht, das gerenderte DOM deklarativ an die Daten der zugrunde liegenden Komponenteninstanz zu binden.

### Textinterpolation
Die grundlegendste Form der Datenbindung ist die Textinterpolation unter Verwendung der "Mustache"-Syntax (doppelte geschweifte Klammern):

```html
<span>Message: {{ msg }}</span>
```

### Rohes HTML
Verwenden Sie die `v-html`-Direktive, um echtes HTML auszugeben:

```html
<p v-html="rawHtml"></p>
```

**Sicherheitswarnung**: Das dynamische Rendern beliebigen HTML auf Ihrer Website kann sehr gefährlich sein, da es leicht zu XSS-Schwachstellen führen kann. Verwenden Sie `v-html` nur bei vertrauenswürdigem Inhalt und niemals bei benutzerbereitgestelltem Inhalt.

### Attribut-Bindungen
Verwenden Sie die `v-bind`-Direktive, um HTML-Attribute zu binden:

```html
<!-- Vollständige Syntax -->
<div v-bind:id="dynamicId"></div>

<!-- Kurzschreibweise -->
<div :id="dynamicId"></div>

<!-- Gleichnamige Kurzschreibweise (Vue 3.4+) -->
<div :id></div>
```

### Boolesche Attribute
Boolesche Attribute werden eingeschlossen, wenn der gebundene Wert wahr ist:

```html
<button :disabled="isButtonDisabled">Button</button>
```

### Dynamisches Binden mehrerer Attribute
Binden Sie mehrere Attribute aus einem Objekt, indem Sie `v-bind` ohne Argument verwenden:

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript-Ausdrücke

Vue unterstützt die volle Leistungsfähigkeit von JavaScript-Ausdrücken in allen Datenbindungen:

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### Aufruf von Funktionen
Es ist möglich, eine komponentenexponierte Methode innerhalb eines Bindungsausdrucks aufzurufen:

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### Eingeschränkter globaler Zugriff
Template-Ausdrücke sind sandgeboxt und haben nur Zugriff auf eine [eingeschränkte Liste globaler Objekte](https://github.com/vuejs/core/blob/main/packages/shared/src/globalsAllowList.ts#L3). Die Liste enthält häufig verwendete integrierte globale Objekte wie Math und Date.

Globale Objekte, die nicht explizit in der Liste enthalten sind, z. B. benutzerdefinierte Eigenschaften auf window, sind in Template-Ausdrücken nicht zugänglich. Sie können jedoch zusätzliche globale Objekte für alle Vue-Ausdrücke explizit definieren, indem Sie sie zu [app.config.globalProperties](/api/application#app-config-globalproperties) hinzufügen.

## Direktiven

Direktiven sind spezielle Attribute mit dem Präfix `v-`.

### Argumente
Einige Direktiven können ein "Argument" annehmen, das durch einen Doppelpunkt nach dem Direktivnamen gekennzeichnet ist. Zum Beispiel wird die `v-bind`-Direktive verwendet, um ein HTML-Attribut reaktiv zu aktualisieren:

```html
<!-- v-bind Argument -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on Argument -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### Dynamische Argumente
Es ist auch möglich, einen JavaScript-Ausdruck in einem Direktivargument zu verwenden, indem er mit eckigen Klammern umschlossen wird:

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**Einschränkungen**:
- Dynamische Argumente müssen zu einem String oder null ausgewertet werden
- Vermeiden Sie komplexe Ausdrücke; verwenden Sie stattdessen berechnete Eigenschaften
- Vermeiden Sie bei DOM-Templates Großbuchstaben in dynamischen Schlüsseln

### Modifikatoren
Modifikatoren sind spezielle Nachsilben, die durch einen Punkt gekennzeichnet sind, was anzeigt, dass eine Direktive auf besondere Weise gebunden werden soll. Zum Beispiel teilt der `.prevent`-Modifikator der `v-on`-Direktive mit, `event.preventDefault()` für das ausgelöste Ereignis aufzurufen:

```html
<form @submit.prevent="onSubmit">...</form>
```

## Reaktivitätsgrundlagen

### Deklarieren reaktiver Zustände
Erstellen Sie reaktive Referenzen mit `ref()`:

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### Reaktive Objekte
Erstellen Sie reaktive Objekte mit `reactive()`:

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### Berechnete Eigenschaften
Erstellen Sie berechnete Eigenschaften mit `computed()`:

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### Beobachter
Beobachten Sie Änderungen reaktiver Daten mit `watch()`:

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// Einzelne Referenz beobachten
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## Komponentengrundlagen

### Definieren von Komponenten
```javascript
// Komponente definieren
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Übergeben von Props
```javascript
// Kindkomponente
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// Elternkomponente
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### Senden von Events
```javascript
// Kindkomponente
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### Slots
```html
<!-- Kindkomponente -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- Elternkomponente -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
