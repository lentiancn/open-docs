# 安装指南

本文档介绍 Apache APISIX 的多种安装方式，帮助您快速搭建 API 网关环境。

## 环境要求

在安装 APISIX 之前，请确保满足以下环境要求：

- **操作系统**：Linux（推荐 Ubuntu 20.04+、CentOS 7+、Debian 11+）
- **Docker**（用于 Docker 部署）：已安装 Docker 和 Docker Compose
- **etcd**：APISIX 使用 etcd 作为配置存储，需要 etcd 3.4.0+
- **Perl**：部分依赖需要 Perl 5.5.32+
- **curl**：用于验证安装

## Docker 快速安装（推荐）

这是最快捷的安装方式，适合开发和测试环境。

### 步骤一：创建 Quick Start 目录

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### 步骤二：运行 Quick Start 脚本

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

该命令会自动启动 apisix-quickstart 和 etcd 两个容器。APISIX 使用 etcd 保存和同步配置，容器之间使用 Docker 的 host 网络模式，因此可以直接从本地访问。

### 步骤三：验证安装

安装成功后，终端会输出以下信息：

```
✔ APISIX is ready!
```

可以通过以下命令验证 APISIX 运行状态：

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

正常情况下会返回：

```
Server: APISIX/Version
```

其中 Version 是已安装的 APISIX 版本，例如 APISIX/3.15.0。

### 步骤四：访问 Dashboard

APISIX 提供内置的 Dashboard UI，可通过以下地址访问：

```
http://127.0.0.1:9180/ui
```

> ⚠️ 注意：为了提供更好的体验，管理 API 默认无需授权，请在生产环境中打开授权开关。

## Docker Compose 安装

如果需要自定义配置，可以使用 Docker Compose 方式部署。

### 步骤一：创建配置文件

创建 docker-compose.yaml 文件：

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### 步骤二：启动服务

```bash
docker-compose up -d
```

### 步骤三：验证服务

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Kubernetes 安装

APISIX 提供了官方 Ingress Controller，可以直接在 Kubernetes 中部署。

### 使用 Helm Chart 安装

```bash
# 添加 APISIX Helm 仓库
helm repo add apisix https://charts.apiseven.com
helm repo update

# 安装 APISIX
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### 使用 kubectl 安装

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## 源码编译安装

适用于需要自定义构建的生产环境。

### 步骤一：安装依赖

**Ubuntu/Debian：**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL：**

```bash
sudo yum install -y perl gcc git curl
```

### 步骤二：安装 OpenResty

```bash
# 添加 OpenResty 仓库
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# 安装 OpenResty
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### 步骤三：安装 APISIX

```bash
# 克隆源码
git clone https://github.com/apache/apisix.git
cd apisix

# 安装依赖
make deps

# 初始化配置
make init

# 启动 APISIX
make run
```

## 验证安装

无论采用哪种安装方式，都可以通过以下方式验证：

1. **检查进程状态**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **创建测试路由**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **测试路由**

```bash
curl "http://127.0.0.1:9080/test"
```

## 卸载

**Docker 方式：**

```bash
# 停止容器
docker-compose down

# 删除数据卷（如需要）
docker-compose down -v
```

**Kubernetes 方式：**

```bash
helm uninstall apisix
```
