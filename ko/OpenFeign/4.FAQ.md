# OpenFeign FAQ

## 설치 관련 질문

### Q: OpenFeign이 올바르게 설치되었는지 어떻게 확인하나요?
**A:** 의존성을 추가한 후 기본 애플리케이션을 생성하여 확인하세요:

```java
@SpringBootApplication
@EnableFeignClients
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

Spring Boot 애플리케이션이 성공적으로 시작되고 Feign 클라이언트가 올바르게 주입될 수 있는지 확인하세요.

### Q: 의존성 버전 충돌은 어떻게 처리하나요?
**A:** Spring Cloud BOM을 사용하여 버전 관리:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>2022.0.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

이렇게 하면 버전 충돌을 효과적으로 방지할 수 있습니다.

### Q: 왜 spring-cloud-starter-loadbalancer를 명시적으로 추가해야 하나요?
**A:** Spring Cloud 2020.0.0부터 Ribbon이 제거되고 Spring Cloud LoadBalancer가 기본 로드 밸런서가 되었습니다. `spring-cloud-starter-openfeign`은 이를 선택적 의존성으로 포함하므로 명시적으로 추가해야 합니다:

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

## 구성 관련 질문

### Q: 타임아웃 설정은 어떻게 하나요?
**A:** 구성 파일 또는 Java 구성 두 가지 방법으로 설정할 수 있습니다:

**구성 파일 접근법:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 5000
            read-timeout: 5000
          user-service:
            connect-timeout: 3000
            read-timeout: 10000
```

**Java 구성 접근법:**
```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Request.Options requestOptions() {
        return new Request.Options(3000, 10000);
    }
}
```

### Q: GZIP 압축은 어떻게 활성화하나요?
**A:** 구성 파일에서 압축 활성화:

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

참고: OkHttpClient를 사용할 때 `content-encoding` 또는 `accept-encoding` 헤더가 있으면 압축이 비활성화됩니다.

### Q: 로그 레벨은 어떻게 구성하나요?
**A:** 두 가지 구성 접근법:

**구성 파일:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
            
logging:
  level:
    com.example.clients.UserService: DEBUG
```

**Java 구성:**
```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

지원되는 로그 레벨:
- **NONE**: 로깅 없음(기본값)
- **BASIC**: 요청 메서드, URL, 응답 상태 코드, 실행 시간만 기록
- **HEADERS**: 기본 정보 및 요청/응답 헤더 기록
- **FULL**: 헤더, 본문, 메타데이터(요청 및 응답) 기록

## 사용 관련 질문

### Q: FeignClient 인터페이스가 주입되지 않는데 어떻게 해야 하나요?
**A:** 일반적인 원인과 해결책:

**1. 패키지 스캔 경로 문제:**
```java
// FeignClient가 스캔 경로에 있는지 확인
@SpringBootApplication
@EnableFeignClients(basePackages = "com.example.clients")
public class Application {
    // ...
}
```

**2. 순환 의존성 문제:**
FeignClient가 여러 Service에서 참조되면 순환 의존성이 발생할 수 있습니다. 해결책:
- `@Lazy` 애노테이션을 사용하여 지연 주입
- 순환 의존성을 피하도록 코드 구조 리팩토링

**3. Bean 이름 충돌:**
동일한 이름의 FeignClient가 여러 개 있는 경우 `contextId`를 사용하여 구분:
```java
@FeignClient(contextId = "userClient1", name = "user-service")
public interface UserClient1 { }

@FeignClient(contextId = "userClient2", name = "user-service")  
public interface UserClient2 { }
```

### Q: 복잡한 요청 파라미터는 어떻게 처리하나요?
**A:** 복잡한 파라미터에 대한 여러 접근법:

**1. @SpringQueryMap 사용:**
```java
public class SearchCriteria {
    private String name;
    private String status;
    private Integer page;
}

@FeignClient(name = "search-service")
public interface SearchService {
    @GetMapping("/search")
    SearchResult search(@SpringQueryMap SearchCriteria criteria);
}
```

**2. Map 사용:**
```java
@FeignClient(name = "api-service")
public interface ApiService {
    @GetMapping("/data")
    Data getData(@RequestParam Map<String, Object> params);
}
```

**3. 사용자 정의 QueryMapEncoder:**
```java
@Bean
public QueryMapEncoder queryMapEncoder() {
    return new CustomQueryMapEncoder();
}
```

### Q: 파일 업로드는 어떻게 처리하나요?
**A:** 파일 업로드에는 특별한 구성이 필요합니다:

```java
@FeignClient(name = "file-service")
public interface FileService {
    
    @PostMapping(value = "/upload", 
                consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    UploadResponse uploadFile(@RequestPart("file") MultipartFile file,
                             @RequestPart("metadata") FileMetadata metadata);
}
```

인코더도 구성해야 합니다:
```java
@Bean
public Encoder feignFormEncoder() {
    return new FeignFormEncoder(new SpringEncoder(messageConverters()));
}
```

## 오류 처리 질문

### Q: 오류 처리를 사용자 정의하려면 어떻게 해야 하나요?
**A:** ErrorDecoder 인터페이스 구현:

```java
@Component
public class CustomErrorDecoder implements ErrorDecoder {
    
    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() == 404) {
            return new ResourceNotFoundException("Resource not found");
        } else if (response.status() >= 500) {
            return new ServerException("Server error");
        }
        return new Default().decode(methodKey, response);
    }
}
```

### Q: 서킷 브레이커가 작동하지 않는데 무엇이 문제인가요?
**A:** 다음 구성들을 확인하세요:

**1. 서킷 브레이커 활성화:**
```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

**2. 필요한 의존성 추가:**
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

**3. 서킷 브레이커 규칙 구성:**
```yaml
resilience4j:
  circuitbreaker:
    instances:
      UserServicegetUser:
        failure-rate-threshold: 50
        minimum-number-of-calls: 5
        wait-duration-in-open-state: 5000
```

### Q: 폴백이 트리거되지 않는데 무엇을 확인해야 하나요?
**A:** 일반적인 문제들:

**1. 폴백 클래스가 Spring Bean인지 확인:**
```java
@Component  // 이 애노테이션이 필요합니다
public class UserServiceFallback implements UserService {
    // ...
}
```

**2. 예외 유형 확인:**
서킷 브레이커는 특정 예외에 대해서만 폴백을 트리거하므로 발생하는 예외 유형이 예상된 것인지 확인하세요.

**3. 올바른 폴백 팩토리 구성:**
```java
@FeignClient(name = "service", fallbackFactory = MyFallbackFactory.class)
public interface MyService { }
```

## 성능 관련 질문

### Q: Feign 클라이언트 성능이 느린데 어떻게 최적화하나요?
**A:** 성능 최적화 권장사항:

**1. 적절한 타임아웃 구성:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
```

**2. 연결 풀 활성화:**
```yaml
spring:
  cloud:
    openfeign:
      httpclient:
        hc5:
          enabled: true
          max-connections: 200
          max-connections-per-route: 20
```

**3. 압축 활성화:**
```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
        response:
          enabled: true
```

**4. 캐싱 사용:**
```java
@FeignClient(name = "config-service")
public interface ConfigService {
    @GetMapping("/config/{key}")
    @Cacheable(cacheNames = "config-cache", key = "#key")
    ConfigValue getConfig(@PathVariable String key);
}
```

### Q: 메모리 누수 문제는 어떻게 해결하나요?
**A:** 메모리 누수는 일반적으로 다음 원인으로 발생합니다:

**1. 닫히지 않은 연결:**
HTTP 클라이언트가 연결 풀을 올바르게 관리하는지 확인하세요.

**2. 정리되지 않은 캐시:**
캐시를 정기적으로 정리하거나 적절한 만료 시간을 설정하세요.

**3. 메모리 사용 모니터링:**
```java
@RestController
public class MetricsController {
    
    @GetMapping("/memory")
    public MemoryInfo getMemoryInfo() {
        Runtime runtime = Runtime.getRuntime();
        return new MemoryInfo(
            runtime.totalMemory(),
            runtime.freeMemory(),
            runtime.maxMemory()
        );
    }
}
```

## 보안 관련 질문

### Q: 요청에 인증 토큰을 추가하려면 어떻게 해야 하나요?
**A:** RequestInterceptor 사용:

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor authRequestInterceptor() {
        return requestTemplate -> {
            String token = getToken(); // 토큰을 얻는 로직
            requestTemplate.header("Authorization", "Bearer " + token);
        };
    }
    
    private String getToken() {
        // 토큰 검색 로직 구현
        return "your-token";
    }
}
```

### Q: OAuth2 인증은 어떻게 처리하나요?
**A:** OAuth2 지원 활성화:

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
        
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: client_credentials
            scope: read,write
```

### Q: 민감한 정보를 어떻게 필터링하나요?
**A:** 사용자 정의 로그 필터링:

```java
public class SecureLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        Map<String, Collection<String>> safeHeaders = new HashMap<>(request.headers());
        safeHeaders.remove("Authorization");
        safeHeaders.remove("X-API-Key");
        
        super.logRequest(configKey, logLevel, 
            Request.create(request.httpMethod(), request.url(), safeHeaders,
                request.body(), request.charset(), request.requestTemplate()));
    }
}
```

## 테스트 관련 질문

### Q: Feign 클라이언트는 어떻게 테스트하나요?
**A:** WireMock 사용을 권장합니다:

```java
@SpringBootTest
@AutoConfigureWireMock(port = 0)
class FeignClientTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    void shouldGetUserSuccessfully() {
        // Given
        stubFor(get(urlEqualTo("/users/1"))
            .willReturn(aResponse()
                .withStatus(200)
                .withHeader("Content-Type", "application/json")
                .withBody("{\"id\":1,\"name\":\"John\"}")));
        
        // When
        User user = userService.getUser(1L);
        
        // Then
        assertThat(user.getName()).isEqualTo("John");
    }
}
```

### Q: 오류 시나리오는 어떻게 시뮬레이션하나요?
**A:** WireMock을 사용하여 다양한 오류 시뮬레이션:

```java
@Test
void shouldHandleServerError() {
    // Given
    stubFor(get(urlEqualTo("/users/999"))
        .willReturn(aResponse().withStatus(500)));
    
    // When & Then
    assertThatThrownBy(() -> userService.getUser(999L))
        .isInstanceOf(ServerException.class);
}
```

## 배포 관련 질문

### Q: 프로덕션 환경에서는 어떻게 구성하나요?
**A:** 프로덕션 환경 권장 구성:

```yaml
# application-prod.yml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
            logger-level: basic
      compression:
        request:
          enabled: true
        response:
          enabled: true
      circuitbreaker:
        enabled: true
      cache:
        enabled: true
        
logging:
  level:
    com.example.clients: WARN
```

### Q: Feign 클라이언트는 어떻게 모니터링하나요?
**A:** Micrometer와 통합:

```yaml
spring:
  cloud:
    openfeign:
      micrometer:
        enabled: true
```

그런 다음 Actuator 엔드포인트를 통해 메트릭 확인:
```bash
GET /actuator/metrics
GET /actuator/metrics/feign.client.requests
```

이러한 솔루션들을 이해하고 적용함으로써 OpenFeign 기반 애플리케이션을 더 효과적으로 개발, 디버그 및 유지보수할 수 있습니다.
