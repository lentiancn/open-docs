# Руководство пользователя JMeter

Это руководство содержит подробные инструкции по использованию Apache JMeter для создания и выполнения нагрузочных тестов.

## Базовая структура плана тестирования

План тестирования JMeter состоит из следующих основных компонентов:

### 1. План тестирования

План тестирования — корневой узел всех тестов, содержащий информацию о конфигурации всего теста. Он может включать:
- Определяемые пользователем переменные
- Настройки глобальных свойств
- Конфигурацию группы потоков

### 2. Группа потоков

Группы потоков — единицы выполнения тестов, определяющие:
- **Количество потоков**: Количество имитируемых пользователей
- **Время разгона**: Время, необходимое для запуска всех потоков
- **Количество циклов**: Количество раз, которое каждый поток выполняет тест
- **Планировщик**: Установка времени начала и продолжительности теста

### 3. Сэмплеры

Сэмплеры отправляют запросы на серверы.常见的сэмплеры включают:

- **HTTP-запрос**: Отправка HTTP/HTTPS-запросов
- **FTP-запрос**: Тестирование FTP-серверов
- **JDBC-запрос**: Тестирование подключений к базам данных
- **LDAP-запрос**: Тестирование LDAP-каталогов
- **JMS-сообщение**: Тестирование очередей сообщений
- **TCP-сэмплер**: Тестирование TCP-служб

### 4. Логические контроллеры

Логические контроллеры управляют порядком выполнения запросов:

- **Простой контроллер**: Группировка запросов
- **Контроллер цикла**: Повторное выполнение запросов
- **Контроллер один раз**: Выполнение один раз во время теста
- **Контроллер чередования**: Чередование между запросами
- **Контроллер случайного выбора**: Случайный выбор запросов для выполнения
- **Контроллер пропускной способности**: Управление частотой выполнения запросов

### 5. Слушатели

Слушатели просматривают и анализируют результаты тестов:

- **Дерево результатов**: Просмотр подробного ответа для каждого запроса
- **Агрегированный отчёт**: Просмотр агрегированной статистики производительности
- **Графические результаты**: Графическое отображение данных теста
- **График времени отклика**: Отображение тенденций времени отклика

### 6. Элементы конфигурации

Элементы конфигурации устанавливают значения по умолчанию и общие данные:

- **Значения HTTP-запроса по умолчанию**: Установка параметров по умолчанию для HTTP-запросов
- **Определяемые пользователем переменные**: Определение повторно используемых переменных
- **Конфигурация CSV-набора данных**: Чтение тестовых данных из CSV-файлов
- **Менеджер HTTP-cookie**: Управление файлами cookie
- **Менеджер HTTP-заголовков**: Установка информации заголовков запросов

### 7. Таймеры

Таймеры устанавливают задержки между запросами:

- **Фиксированный таймер**: Установка фиксированной задержки
- **Гауссовский случайный таймер**: Установка случайной задержки
- **Синхронизирующий таймер**: Имитация параллельных запросов
- **Таймер пропускной способности**: Управление пропускной способностью запросов

### 8. Препроцессоры и постпроцессоры

- **Препроцессоры**: Выполнение операций перед отправкой запроса
- **Постпроцессоры**: Извлечение данных после получения ответа

## Создание первого плана тестирования

### Шаг 1: Создание плана тестирования

1. Запустите JMeter
2. Выберите «Файл» -> «Создать» для создания нового плана тестирования
3. Назовите план тестирования

### Шаг 2: Добавление группы потоков

1. Щёлкните правой кнопкой мыши на план тестирования
2. Выберите «Добавить» -> «Потоки (Пользователи)» -> «Группа потоков»
3. Установите количество потоков (например, 10), время разгона (например, 5 секунд), количество циклов (например, 1)

### Шаг 3: Добавление HTTP-запроса

1. Щёлкните правой кнопкой мыши на группу потоков
2. Выберите «Добавить» -> «Сэмплер» -> «HTTP-запрос»
3. Установите имя сервера или IP, номер порта, путь запроса и т.д.

### Шаг 4: Добавление слушателя

1. Щёлкните правой кнопкой мыши на группу потоков или HTTP-запрос
2. Выберите «Добавить» -> «Слушатель» -> «Дерево результатов»
3. Добавьте «Агрегированный отчёт» для просмотра статистики производительности

### Шаг 5: Запуск теста

1. Нажмите кнопку «Пуск» на панели инструментов
2. Наблюдайте за данными результатов в слушателе

## Параметризация и тестирование на основе данных

### Использование CSV-файлов данных

1. Добавьте элемент «Конфигурация CSV-набора данных»
2. Установите имя файла и имена переменных
3. Используйте ${имяПеременной} в запросах для ссылки на данные

### Использование определяемых пользователем переменных

1. Добавьте элемент «Определяемые пользователем переменные»
2. Определите имена и значения переменных
3. Используйте ${имяПеременной} в запросах

## Обработка корреляции

### Использование экстрактора регулярных выражений

Когда запросы требуют динамических данных (например, идентификатора сессии):

1. Добавьте «Экстрактор регулярных выражений»
2. Установите шаблон регулярного выражения для сопоставления
3. Используйте извлечённую переменную в последующих запросах

### Использование JSON-экстрактора

Для JSON-ответов:

1. Добавьте «JSON-экстрактор»
2. Установите выражение JSON Path
3. Извлеките требуемые значения полей JSON

## Распределённое тестирование

Когда одна машина не может генерировать достаточную нагрузку, можно использовать распределённое тестирование:

### Настройка мастера

1. Отредактируйте jmeter.properties в каталоге bin мастера
2. Установите remote_hosts для указания на адреса подчинённых узлов
3. Запустите JMeter

### Настройка подчинённого узла

1. Запустите jmeter-server на каждом подчинённом узле
2. Убедитесь, что брандмауэр разрешает связь на указанных портах

### Запуск распределённого тестирования

1. Выберите «Удалённый запуск» на мастере
2. Выберите, какие подчинённые узлы запустить, или запустите все

## Советы по оптимизации производительности

- Используйте режим командной строки для снижения потребления ресурсов GUI
- Установите соответствующее количество потоков и время разгона, чтобы избежать мгновенной высокой нагрузки
- Отключите ненужные слушатели, просматривайте результаты только при необходимости
- Используйте утверждения для проверки содержимого ответа
- Регулярно очищайте неиспользуемые планы тестирования
- Для крупномасштабного тестирования рассмотрите распределённое тестирование
