# Claude Code User Manual

This manual will quickly get you started with Claude Code and teach you the operations for daily development.

## Prerequisites

Before you begin, make sure you have:

- A terminal or command prompt open
- A code project to work with
- A Claude subscription (Pro, Max, Teams, or Enterprise), Claude Console account, or access through a supported cloud provider

## Start a Session

Open a terminal in your project directory and start Claude Code:

```bash
cd /path/to/your/project
claude
```

On first use, you'll see a welcome screen showing session information, recent conversations, and latest updates. Type `/help` for available commands or `/resume` to continue a previous conversation.

## Basic Operations

### Understand the Codebase

Have Claude analyze your project:

```
What does this project do?
```

```
What technologies does this project use?
```

```
Where is the main entry point?
```

```
Explain the folder structure
```

You can also ask Claude about its capabilities:

```
What can Claude Code do?
```

```
How do I create custom skills in Claude Code?
```

**Note: Claude Code reads project files as needed. You don't have to manually add context.**

### Your First Code Change

Have Claude Code do some actual coding work. Try a simple task:

```
Add a hello world function to the main file
```

Claude Code will:
1. Find the appropriate file
2. Show you the proposed changes
3. Ask for your approval
4. Make the edit

**Note: Claude Code always asks for permission before modifying files. You can approve individual changes or enable "Accept all" mode for a session.**

### Git Operations


### Fix Bugs and Add Features

Claude is proficient at debugging and feature implementation.

Describe what you want in natural language:

```
Add input validation to the user registration form
```

Or fix existing issues:

```
There's a bug where users can submit empty forms - fix it
```

Claude Code will:
- Locate the relevant code
- Understand the context
- Implement a solution
- Run tests if available

## Essential Commands

Here are the most important commands for daily use:

| Command | What it does | Example |
|---------|-------------|---------|
| `claude` | Start interactive mode | `claude` |
| `claude "task"` | Run a one-time task | `claude "fix the build error"` |
| `claude -p "query"` | Run one-off query, then exit | `claude -p "explain this function"` |
| `claude -c` | Continue most recent conversation in current directory | `claude -c` |
| `claude -r` | Resume a previous conversation | `claude -r` |
| `claude commit` | Create a Git commit | `claude commit` |
| `/clear` | Clear conversation history | `/clear` |
| `/help` | Show available commands | `/help` |
| `exit` or Ctrl+C | Exit Claude Code | `exit` |

## Advanced Tips

### Using CLAUDE.md

Create a `CLAUDE.md` file in your project root to provide persistent project context for Claude Code. This file loads automatically at the start of every session.

CLAUDE.md can contain:
- Coding standards and specifications
- Architecture decisions
- Preferred libraries
- Review checklists
- Project-specific commands and scripts

### Custom Skills

Create custom commands to encapsulate repeatable workflows, such as `/review-pr` or `/deploy-staging`.

### Using Hooks

Hooks let you run shell commands before or after Claude Code actions, such as:
- Auto-formatting after every file edit
- Running lint before a commit

### MCP Servers

Connect Claude Code to external tools and data sources through the Model Context Protocol:
- Read design documents from Google Drive
- Update Jira tickets
- Pull data from Slack
- Use custom tools

### Multi-Agent Collaboration

For complex tasks, spawn multiple Claude Code agents to work in parallel. A lead agent coordinates work, assigns subtasks, and merges results.

### Remote Control

Work isn't tied to a single device:
- Use Remote Control to continue a local session from your phone or another device
- Start a long-running task on the Web or iOS app, then pull it back to your terminal with `/teleport`
- Use `/desktop` to switch a terminal session to the desktop app for visual diff review

## Best Practices

### Be Specific with Your Requests

❌ Don't say: "fix the bug"

✅ Do say: "fix the login bug where users see a blank screen after entering wrong credentials"

### Use Step-by-Step Instructions

Break complex tasks into steps:

```
1. Create a new database table for user profiles
2. Create an API endpoint to get and update user profiles
3. Build a webpage that allows users to see and edit their information
```

### Let Claude Explore First

Before making changes, let Claude understand your code:

```
Analyze the database schema
```

```
Build a dashboard showing products that are most frequently returned by our UK customers
```

### Save Time with Shortcuts

- Press `?` to see all available keyboard shortcuts
- Use Tab for command completion
- Press `↑` for command history
- Type `/` to see all commands and skills

## Frequently Asked Questions

**Does Claude Code read my code?**

Yes, Claude Code reads project files to understand your codebase. It won't send your code to external servers (unless you use a third-party API provider).

**Does Claude Code automatically modify my files?**

No. Claude Code always asks for permission before modifying files. You can approve individual changes or enable "Accept all" mode for an entire session.

**Is my code secure?**

Claude Code is designed with security in mind. Your code is not sent to Anthropic for model training (unless you explicitly opt in). Learn more in the [Anthropic Privacy Policy](https://www.anthropic.com/privacy).
