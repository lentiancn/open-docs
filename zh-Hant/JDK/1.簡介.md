# JDK 簡介

## 什麼是 JDK

JDK（Java Development Kit）是 Java 語言的軟體開發工具包，是整個 Java 開發的核心所在。它包含了 Java 運行環境（Java Runtime Environment，JRE）、Java 編譯器（javac）、Java 虛擬機（JVM）以及大量的 Java 類別庫和工具。JDK 是 Java 程式設計師進行 Java 應用開發所必須安裝的軟體包，無論是開發桌面應用、Web 應用、行動應用還是企業級應用，都離不開 JDK 的支援。

JDK 由 Oracle 公司（及其前身 Sun Microsystems）開發和維護，是 Java 平台的基礎。目前最新的 LTS（長期支援）版本是 JDK 25 和 JDK 21，JDK 17 也是一個廣泛使用的長期支援版本。Oracle 公司每隔一段時間就會發布新的 JDK 版本，每個版本都會帶來效能提升、新特性以及安全漏洞的修復。

JDK 不僅僅是一個運行環境，它提供了完整的 Java 開發所需的全部組件。其中最核心的是 Java 編譯器（javac），負責將 Java 原始碼編譯成位元組碼（.class 檔案）；其次是 Java 虛擬機（JVM），負責執行位元組碼；此外還包括各種工具如 jar（歸檔工具）、javadoc（文檔生成工具）、javap（反組譯工具）等。這些工具共同構成了完整的 Java 開發環境。

## JDK 的主要版本

### JDK 25（新版本）

JDK 25 是最新的 JDK 版本，帶來了許多新特性和改進。這是一個短期支援版本，將獲得 6 個月的支援週期。主要特性包括：改進的效能優化、增強的垃圾回收器、新的語言特性以及對最新硬體的支援。JDK 25 適合那些希望使用最新 Java 特性的開發者，但需要注意其較短的支援週期。

### JDK 21（LTS）

JDK 21 是最新的長期支援版本（LTS），將獲得至少 8 年的支援。這是 Oracle JDK 免費使用的最後一個長期支援版本，之後的 LTS 版本將需要商業許可。JDK 21 引入了虛擬線程（Virtual Threads）、模式匹配（Pattern Matching）、記錄類（Records）等重要特性，這些特性極大地簡化了 Java 程式的編寫，提高了開發效率。

虛擬線程是 JDK 21 最重要的特性之一，它使得編寫高並發應用變得更加簡單。與傳統的線程相比，虛擬線程的建立成本極低，可以輕鬆建立數百萬個虛擬線程來處理並發任務。這對於構建大規模分散式系統、微服務架構以及高效能伺服器應用具有重要意義。

### JDK 17（LTS）

JDK 17 是目前最成熟的長期支援版本之一，獲得了廣泛的應用。它是 Oracle JDK 免費使用的最後一個版本（根據 Oracle 的許可條款），因此許多企業仍然選擇使用 JDK 17。JDK 17 包含了來自 OpenJDK 的許多特性，如密封類（Sealed Classes）、文字區塊（Text Blocks）、switch 表達式等。

JDK 17 的效能相較於 JDK 11 有了顯著提升，特別是在垃圾回收和 JIT 編譯方面。對於需要長期穩定支援的企業應用來說，JDK 17 仍然是一個優秀的選擇。它在效能、穩定性和功能之間取得了良好的平衡。

### JDK 11（LTS）

JDK 11 是另一個重要的長期支援版本，它標誌著 Java 付費特性的分水嶺。從 JDK 11 開始，Oracle JDK 的一些高級特性開始需要付費訂閱。JDK 11 引入了新的垃圾回收器 ZGC（實驗性）、新的 HTTP 用戶端 API 以及局部變數類型推斷（var 關鍵字）。雖然已經不再是最新的 LTS 版本，但 JDK 11 仍然在許多現有專案中廣泛使用。

## JDK 的核心組件

### Java 編譯器（javac）

Java 編譯器（javac）是 JDK 最核心的工具之一，負責將 Java 原始碼（.java 檔案）編譯成 Java 位元組碼（.class 檔案）。位元組碼是一種中間形式的代碼，它不依賴於具體的硬體平台，只能由 Java 虛擬機執行。這種設計使得 Java 程式具有「一次編寫，到處運行」的特性。

javac 編譯器支援大量的編譯選項，開發者可以通過命令列參數控制編譯過程的各種行為。例如，可以使用 -d 選項指定編譯輸出目錄，使用 -classpath 選項指定類別搜尋路徑，使用 -source 和 -target 選項指定 Java 原始碼和目標位元組碼的版本等。掌握這些編譯選項對於大型專案的建置非常重要。

### Java 虛擬機（JVM）

Java 虛擬機（JVM）是 Java 程式運行的核心環境，它負責載入位元組碼並將其解釋或編譯為本機代碼執行。JVM 是 Java 語言「平台無關性」的關鍵所在，它屏蔽了底層作業系統和硬體的差異，使得同一個 Java 程式可以在不同的平台上運行。

JVM 的核心組件包括：類別載入器（ClassLoader）負責載入類別檔案；執行引擎（Execution Engine）負責執行位元組碼；執行期資料區（Runtime Data Areas）包括方法區、堆、虛擬機器堆疊、本地方法堆疊和程式計數器等。理解 JVM 的工作原理對於編寫高效能 Java 應用至關重要。

JVM 還提供了強大的垃圾回收（GC）功能，自動管理記憶體配置和回收。不同的垃圾回收器，如 Serial、Parallel、CMS、G1、ZGC 和 Shenandoah，適用於不同的應用場景。開發者可以根據應用的特點選擇合適的垃圾回收器，以獲得最佳的效能。

### Java 執行環境（JRE）

Java 執行環境（JRE）是運行 Java 程式所必需的環境，它包含了 JVM、Java 核心類別庫以及其他支援檔案。JRE 是 JDK 的一個子集，JDK 包含了 JRE 以及開發工具。如果只需要運行 Java 程式而不需要進行開發，只需要安裝 JRE 即可。

JRE 的核心是 Java 核心類別庫，這些類別庫提供了 Java 語言的基礎功能，如 java.lang、java.util、java.io、java.net 等。這些類別庫經過多年的發展和優化，已經非常成熟和穩定，涵蓋了從基礎資料結構到網路編程的各個方面。

### Java 工具集

JDK 提供了豐富的命令列工具，用於 Java 程式的開發、偵錯和診斷。主要工具包括：

jar：用於建立和管理 Java 歸檔檔案（.jar 檔案），可以將多個類別檔案打包成一個方便分發和部署的檔案。

javadoc：用於從 Java 原始碼註解中產生 HTML 格式的 API 文檔，這是建立程式碼文檔的標準工具。

javap：用於反組譯 Java 位元組碼，可以查看類別檔案的結構和方法簽章，對於理解 Java 編譯過程和偵錯非常有幫助。

jdb：Java 偵錯器，提供命令列介面的偵錯功能，可以設定中斷點，單步執行、查看變數值等。

jps：顯示當前系統中所有的 Java 程式資訊，包括程式 ID 和主類別名。

jstat：顯示 JVM 的統計資訊，如垃圾回收情況、類別載入資訊等。

jstack：列印 Java 線程的堆疊追蹤資訊，用於診斷線程死鎖等問題。

jmap：顯示 Java 堆記憶體資訊，可以產生堆轉儲檔案用於分析。

這些工具在日常開發和生產環境的問題排查中都非常有用，熟練掌握它們可以大大提高開發效率和問題診斷能力。

## JDK 的應用領域

### 企業級應用開發

JDK 是企業級應用開發的基礎。Java EE（現更名為 Jakarta EE）的各種框架，如 Spring、Hibernate、Struts 等，都需要運行在 JDK 之上。這些框架為企業應用提供了完整的解決方案，包括依賴注入、交易管理、資料持久化、Web 服務等。

大型企業應用通常對穩定性和效能有極高的要求，因此選擇合適的 JDK 版本非常重要。許多企業會採用 JDK 17 這樣的成熟 LTS 版本，並配合商業支援服務，以確保系統的長期穩定運行。

### Android 開發

雖然 Android 應用主要使用 Java 或 Kotlin 編寫，但 Android 應用的開發離不開 JDK。Android SDK 工具需要 JDK 來編譯 Android 專案，生成可以在 Android 設備上安裝的 APK 檔案。Android Studio 也內建了 JDK，開發者在設定開發環境時需要指定 JDK 的路徑。

值得注意的是，Android 開發不需要完整的 JDK，通常只需要 JRE 即可。但為了使用 Android SDK 的全部功能，建議安裝完整的 JDK。

### 大數據處理

在大數據領域，Java 和 JDK 扮演著核心角色。Hadoop、Spark、Flink 等主流大數據框架都是用 Java 開發的，它們需要運行在 JDK 之上。這些框架通常需要處理海量資料，對效能和穩定性有極高要求。

Hadoop 生態系統中的許多元件，如 HBase、Hive、Pig 等，都需要 JDK 才能運行。在大數據環境部署時，需要選擇合適的 JDK 版本，並進行相應的效能調優。

### Web 應用開發

Java Web 應用開發是 JDK 的傳統強項。Servlet、JSP、Filter、Listener 等技術構成了 Java Web 開發的基礎。主流的 Web 框架如 Spring MVC、Struts、JSF 等都運行在 JDK 之上。

Tomcat、Jetty、WildFly 等 Web 伺服器為 Java Web 應用提供運行容器。這些伺服器需要 JDK 才能啟動和運行。在生產環境中，通常需要選擇效能優秀且穩定的 JDK 版本。

### 雲端運算和微服務

隨著雲端運算和微服務架構的流行，JDK 在這一領域也發揮著重要作用。Spring Cloud、Dubbo 等微服務框架為構建分散式系統提供了完整的解決方案。這些框架都需要運行在 JDK 之上。

雲端原生應用對 JDK 提出了新的要求，如更快的啟動速度、更低的記憶體佔用等。為此，Oracle 和其他廠商提供了針對雲端場景優化的 JDK 發行版，如 Oracle GraalVM Native Image 可以將 Java 應用編譯為原生可執行檔案，大大減少啟動時間和記憶體佔用。

## JDK 與其他 Java 實現

### OpenJDK

OpenJDK 是 Java SE 的開源實現，由 Oracle 和社區共同開發和維護。OpenJDK 是大多數其他 JDK 發行版的基礎，許多廠商，如 Eclipse（Eclipse Temurin）、Amazon（Amazon Corretto）、Microsoft（Microsoft Build of OpenJDK）等，都基於 OpenJDK 建構自己的 JDK 發行版。

OpenJDK 與 Oracle JDK 在代碼層面幾乎是相同的，主要區別在於許可協議和支援服務。Oracle JDK 提供商業支援和額外的企業級功能，而 OpenJDK 則是完全開源免費的。對於大多數用戶來說，使用 OpenJDK 完全可以滿足需求。

### 其他 JDK 發行版

除了 Oracle JDK 和 OpenJDK，還有許多其他的 JDK 發行版：

Eclipse Temurin（前身為 AdoptOpenJDK）：由 Eclipse 基金會維護的 OpenJDK 發行版，提供高品質的建置和長期支援。

Amazon Corretto：Amazon 提供的免費、多平台 JDK 發行版，提供長期支援和效能優化。

Microsoft Build of OpenJDK：Microsoft 提供的 OpenJDK 建置，針對 Windows 和 Azure 進行了優化。

Azul Zulu：Azul Systems 提供的 JDK 發行版，以高效能和穩定性著稱。

GraalVM：Oracle 提供的多語言虛擬機器，不僅支援 Java，還支援 JavaScript、Python、Ruby 等其他語言。

選擇哪個 JDK 發行版取決於具體的需求和環境。大多數情況下，標準的 Oracle JDK 或 OpenJDK 都可以滿足需求。如果需要商業支援，可以選擇 Oracle JDK 或廠商提供的支援服務。

## JDK 的重要性

JDK 作為 Java 開發的基礎，其重要性不言而喻。無論使用哪種 Java 框架或工具，都離不開 JDK 的支援。JDK 不僅僅是編譯和運行 Java 程式的工具集，它還代表了 Java 生態系統的核心。

選擇一個合適的 JDK 版本並正確配置，對於專案的成功至關重要。需要考慮的因素包括：版本的穩定性、安全更新頻率、新特性、效能優化以及社區支援等。對於企業級應用，建議使用 LTS 版本以獲得長期支援。

隨著 Java 語言的不斷發展和演進，JDK 也在持續更新。新的 JDK 版本帶來了更好的效能、更多的特性和更高的安全性。保持 JDK 的更新，可以獲得更好的開發體驗和運行效率。

## 版本差異

不同的 JDK 主要版本在安裝、使用和功能上存在顯著差異：

**JDK 8**：最後一個免費用於商業使用的 Oracle JDK 版本。引入了 Lambda 表達式、串流 API 和新的日期/時間 API。仍然廣泛用於遺留系統。

**JDK 11**：標誌著 Java 付費特性的開始。引入了局部變數類型推斷（var）、新的 HTTP 用戶端 API 和 ZGC（實驗性）。最後一個支援 32 位元系統的版本。

**JDK 17**：最後一個在 NFTC 下完全免費的 Oracle JDK 版本。引入了密封類、文字區塊和 switch 表達式。在效能、穩定性和功能之間取得了絕佳的平衡。

**JDK 21**：最新的 LTS 版本，虛擬線程、switch 的模式匹配和記錄類作為穩定特性。建議用於新專案。

**JDK 25**：最新版本，帶來最新的語言特性和效能改進。適合想要最新特性的開發者，但支援週期較短。
