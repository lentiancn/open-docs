# Kubernetes Installationsanleitung

## Voraussetzungen

Bevor du Kubernetes installierst, solltest du die grundlegenden Anforderungen verstehen:

### Systemanforderungen
- Linux ist das bevorzugte Betriebssystem für die Ausführung der Kubernetes-Kontrollplane
- In deinem Cluster kannst du Anwendungen unter Linux oder anderen Betriebssystemen einschließlich Windows ausführen
- Ein Betriebssystem, das Container-Runtime unterstützt, ist erforderlich

### Auswahl des Installationstyps
Wähle bei der Installation von Kubernetes einen Installationstyp basierend auf:
- Wartungsfreundlichkeit
- Sicherheit
- Kontrolle
- Verfügbare Ressourcen
- Erforderliche Fachkenntnisse für den Betrieb und die Verwaltung eines Clusters

## Installationsmethoden

### Lernumgebung (Empfohlen für Anfänger)

Wenn du Kubernetes lernst, verwende Tools, die von der Kubernetes-Community unterstützt werden, oder Tools im Ökosystem, um ein Cluster auf deinem lokalen Computer einzurichten:

#### Minikube
Minikube ist eine lightweight Kubernetes-Implementierung, die ein Ein-Knoten-Cluster auf deinem lokalen Computer erstellt. Ideal zum Lernen und für Small-Scale-Tests.

Installationsschritte:
1. Minikube installieren
2. kubectl (Kubernetes-Befehlszeilentool) installieren
3. Minikube-Cluster starten

#### Kind
Kind (Kubernetes in Docker) ist ein weiteres Tool zum Erstellen lokaler Kubernetes-Cluster, das Docker-Container verwendet, um Knoten zu simulieren.

### Produktionsumgebung

Für Produktionsumgebungen consider these options:

#### kubeadm
Das offiziell unterstützte Tool zur Bereitstellung von Kubernetes. Erstellt sichere Cluster nach Best Practices.

Grundlegende Schritte:
1. Server vorbereiten (BS installieren, Netzwerk konfigurieren)
2. Container-Runtime installieren (containerd oder Docker)
3. kubeadm, kubelet und kubectl installieren
4. Cluster mit kubeadm initialisieren
5. Netzwerk-Plugin konfigurieren (Calico, Cilium usw.)
6. Workerknoten hinzufügen

#### Verwaltete Dienste
Wenn du kein Kubernetes-Cluster selbst verwalten möchtest, consider verwaltete Dienste:
- Amazon EKS (Elastic Kubernetes Service)
- Google GKE (Google Kubernetes Engine)
- Azure AKS (Azure Kubernetes Service)
- Alibaba Cloud ACK
- Tencent Cloud TKE

#### Andere Lösungen
Es gibt auch standardisierte und benutzerdefinierte Lösungen in einer breiten Palette von Cloud- und Bare-Metal-Umgebungen.

## Tool-Installation

### kubectl installieren

kubectl ist das Befehlszeilentool für die Interaktion mit Kubernetes-Clustern.

Linux-Installation:
```bash
# Neueste Version herunterladen
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# Ausführungsrechte hinzufügen
chmod +x kubectl

# In den Systempfad verschieben
sudo mv kubectl /usr/local/bin/
```

### kubeadm installieren

```bash
# apt-Paketindex aktualisieren
sudo apt-get update

# Abhängigkeiten installieren
sudo apt-get install -y apt-transport-https ca-certificates curl

# Kubernetes APT-Repository hinzufügen
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Aktualisieren und installieren
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## Cluster-Initialisierung

### Cluster mit kubeadm erstellen

1. Kontrollplane-Knoten initialisieren:
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. kubectl konfigurieren:
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. Netzwerk-Plugin installieren (Flannel als Beispiel):
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. Workerknoten hinzufügen:
```bash
# Auf jedem Workerknoten den Befehl aus der Init-Ausgabe ausführen
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## Installation überprüfen

Cluster-Status überprüfen:
```bash
# Knoten anzeigen
kubectl get nodes

# Pods anzeigen
kubectl get pods --all-namespaces
```

Wenn alle Komponenten den Status Running haben, war die Installation erfolgreich.

## Deinstallation

Wenn du Kubernetes deinstallieren musst:

```bash
# Alle Kubernetes-Dienste stoppen
sudo kubeadm reset

# Konfigurationsdateien entfernen
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# Pakete deinstallieren
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## Wichtige Hinweise

1. Container-Runtime-Auswahl: Wähle eine geeignete Container-Runtime für dein neues Cluster
2. Ressourcenplanung: Ordne CPU- und Speicherressourcen basierend auf Workloads angemessen zu
3. Netzwerkplanung: Plane Pod- und Service-Netzwerk-CIDRs im Voraus
4. Sicherheitshärtung: Authentifizierung, Autorisierung und Netzwerkrichtlinien für die Produktion konfigurieren
5. Hochverfügbarkeit: Multi-Kontrollplane-Cluster für Produktionsumgebungen empfohlen
