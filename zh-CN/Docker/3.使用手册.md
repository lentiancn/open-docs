# Docker 使用手册

本手册将带你快速上手 Docker，掌握日常使用中常用的操作。

## 准备工作

开始之前，请确保你具备以下条件：

- 已安装 Docker（Docker Desktop 或 Docker Engine）
- 有命令行终端访问权限
- 了解基本的命令行操作

## 基本概念复习

### 镜像与容器

- **镜像（Image）**：只读的模板，用于创建容器
- **容器（Container）**：镜像的运行实例

### 常用命令

| 命令 | 功能 |
|------|------|
| `docker run` | 创建并运行容器 |
| `docker ps` | 查看运行中的容器 |
| `docker images` | 查看本地镜像 |
| `docker pull` | 拉取镜像 |
| `docker build` | 构建镜像 |
| `docker stop` | 停止容器 |
| `docker rm` | 删除容器 |
| `docker rmi` | 删除镜像 |

## 第一个容器

### 运行 Hello World

最简单的测试方式是运行 hello-world 镜像：

```bash
docker run hello-world
```

这个命令会：
1. 检查本地是否有 hello-world 镜像
2. 如果没有，从 Docker Hub 拉取
3. 创建并运行容器
4. 显示欢迎信息并退出

### 运行一个交互式容器

运行一个交互式的 Ubuntu 容器：

```bash
docker run -it ubuntu bash
```

参数说明：
- `-i`：交互式模式
- `-t`：分配一个伪终端

在容器内，你可以像使用普通 Linux 系统一样操作。输入 `exit` 退出容器。

### 后台运行容器

以后台模式运行容器：

```bash
docker run -d nginx
```

参数说明：
- `-d`：后台运行（detached 模式）

查看运行中的容器：

```bash
docker ps
```

停止后台容器：

```bash
docker stop <容器ID或名称>
```

## 容器管理

### 查看容器

查看运行中的容器：

```bash
docker ps
```

查看所有容器（包括已停止的）：

```bash
docker ps -a
```

查看容器详细信息：

```bash
docker inspect <容器名称>
```

### 启动和停止容器

启动已停止的容器：

```bash
docker start <容器名称>
```

停止运行中的容器：

```bash
docker stop <容器名称>
```

重启容器：

```bash
docker restart <容器名称>
```

### 删除容器

删除已停止的容器：

```bash
docker rm <容器名称>
```

强制删除运行中的容器：

```bash
docker rm -f <容器名称>
```

删除所有已停止的容器：

```bash
docker container prune
```

## 镜像管理

### 查看镜像

列出本地镜像：

```bash
docker images
```

或

```bash
docker image ls
```

查看镜像详细信息：

```bash
docker image inspect <镜像名称>
```

### 拉取镜像

从 Docker Hub 拉取镜像：

```bash
docker pull ubuntu:latest
```

指定版本拉取：

```bash
docker pull nginx:1.25
```

### 删除镜像

删除本地镜像：

```bash
docker rmi <镜像名称>
```

或

```bash
docker image rm <镜像名称>
```

删除未使用的镜像：

```bash
docker image prune
```

## 端口映射

容器默认是隔离的，需要通过端口映射来访问容器内的服务。

### 映射端口

将宿主机的 8080 端口映射到容器的 80 端口：

```bash
docker run -d -p 8080:80 nginx
```

现在访问 `http://localhost:8080` 就能看到 Nginx 欢迎页。

### 常见端口映射格式

- `-p 8080:80`：将宿主机的 8080 映射到容器的 80
- `-p 127.0.0.1:8080:80`：仅本地访问
- `-p 8080-8090:80-90`：端口范围映射

## 数据卷

数据卷用于持久化容器产生的数据，或共享宿主机的文件到容器。

### 创建数据卷

```bash
docker volume create mydata
```

### 使用数据卷

```bash
docker run -d -v mydata:/app/data nginx
```

### 挂载宿主机目录

```bash
docker run -d -v /home/user/data:/app/data nginx
```

### 查看数据卷

```bash
docker volume ls
docker volume inspect mydata
```

### 删除数据卷

```bash
docker volume rm mydata
docker volume prune  # 删除未使用的卷
```

## 构建镜像

### Dockerfile 基础

创建一个简单的 Dockerfile：

```dockerfile
# 指定基础镜像
FROM node:18-alpine

# 设置工作目录
WORKDIR /app

# 复制文件
COPY package*.json ./

# 安装依赖
RUN npm install

# 复制源代码
COPY . .

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["node", "index.js"]
```

### 构建镜像

```bash
docker build -t myapp:latest .
```

参数说明：
- `-t`：指定镜像名称和标签

### 使用 .dockerignore

创建 .dockerignore 文件排除不需要的文件：

```
node_modules
npm-debug.log
.git
README.md
```

## Docker Compose

Docker Compose 用于定义和运行多容器应用。

### docker-compose.yml 示例

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: postgres:15
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=secret

volumes:
  db-data:
```

### 常用命令

启动所有服务：

```bash
docker-compose up
```

后台运行：

```bash
docker-compose up -d
```

停止所有服务：

```bash
docker-compose down
```

查看日志：

```bash
docker-compose logs -f
```

重新构建并启动：

```bash
docker-compose up --build
```

## 网络

### 查看网络

```bash
docker network ls
```

### 创建网络

```bash
docker network create mynetwork
```

### 容器网络连接

将容器连接到网络：

```bash
docker network connect mynetwork mycontainer
```

### 容器间通信

使用 Docker Compose 时，容器可以通过服务名相互访问。例如：

```yaml
services:
  web:
    build: .
    depends_on:
      - db
  db:
    image: postgres:15
```

在 web 容器中可以通过 `db` 主机名访问数据库。

## 常用技巧

### 进入运行中的容器

```bash
docker exec -it <容器名称> bash
```

### 查看容器日志

```bash
docker logs -f <容器名称>
```

### 复制文件

从容器复制到宿主机：

```bash
docker cp container:/path/in/container /local/path
```

从宿主机复制到容器：

```bash
docker cp /local/path container:/path/in/container
```

### 查看资源使用

```bash
docker stats
```

### 查看容器进程

```bash
docker top <容器名称>
```

### 清理

清理停止的容器、未使用的镜像和构建缓存：

```bash
docker system prune
```

完全清理：

```bash
docker system prune -a
```

## 最佳实践

### 镜像构建优化

1. **减少镜像层数**：将相关命令合并
2. **使用多阶段构建**：减小最终镜像大小
3. **合理使用 .dockerignore**：排除不需要的文件
4. **使用特定版本标签**：避免使用 latest 标签

### 安全建议

1. **不要在容器中运行 root 用户**
2. **使用官方认证镜像**
3. **定期更新基础镜像**
4. **敏感信息使用 Docker secrets**（Swarm 模式）或 Kubernetes secrets

### 资源限制

为容器设置资源限制：

```bash
docker run -d --memory="512m" --cpus="1.5" nginx
```

## 常见问题

### Docker 权限问题

如果遇到权限错误，将当前用户添加到 docker 组：

```bash
sudo usermod -aG docker $USER
```

然后重新登录。

### 容器自动退出

容器如果没有持续运行的进程，会自动退出。可以使用以下方式保持运行：

```bash
docker run -d nginx tail -f /dev/null
```

或使用类似 supervisor 的进程管理器。

### 端口冲突

如果端口已被占用，修改映射端口：

```bash
docker run -d -p 8081:80 nginx
```
