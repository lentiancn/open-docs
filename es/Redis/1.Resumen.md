# Redis Resumen

Redis (Remote Dictionary Server) es un almacenamiento de estructura de datos en memoria de código abierto, utilizado como base de datos, caché y broker de mensajes con soporte para varias estructuras de datos como strings, hashes, listas, sets, sets ordenados, bitmaps, hyperloglogs e índices geoespaciales.

## ¿Qué es Redis?

Redis es una base de datos NoSQL que almacena datos en pares clave-valor. Es conocido por su excepcional rendimiento, flexibilidad y rico conjunto de funciones.

## Características Principales

### Almacenamiento en Memoria
Redis almacena todos los datos en memoria (RAM), proporcionando operaciones de lectura y escritura extremadamente rápidas.

### Estructuras de Datos
Redis soporta múltiples tipos de datos:
- **Strings**: Almacenamiento básico clave-valor
- **Lists**: Colecciones ordenadas de strings
- **Sets**: Colecciones no ordenadas de strings únicos
- **Sorted Sets**: Sets con puntuación para rankings
- **Hashes**: Mapas entre campos string y valores

### Persistencia
Redis ofrece persistencia opcional a través de:
- RDB (Redis Database) snapshots
- AOF (Append-Only File) logging

### Replicación
Redis soporta replicación maestro-esclavo para redundancia de datos y escalabilidad de lectura.

## Casos de Uso

1. **Caché**: Almacenar datos de acceso frecuente en memoria
2. **Gestión de Sesiones**: Mantener sesiones de usuario en aplicaciones web
3. **Analíticas en Tiempo Real**: Procesar flujos de datos de alta velocidad
4. **Colas de Mensajes**: Implementar patrones pub/sub
5. **Limitación de Tasa**: Controlar tasas de solicitudes API

## Ventajas

- **Rendimiento**: Latencia sub-milisegundo para operaciones de lectura/escritura
- **Flexibilidad**: Soporte para diversas estructuras de datos
- **Escalabilidad**: Escalado horizontal a través de clustering
- **Comunidad**: Gran comunidad con documentación extensa
