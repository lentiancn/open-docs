# JUnit 常見問題

## JUnit 4 和 JUnit 5 我應該用哪個？

對於新專案，強烈建議使用 JUnit 5。它提供了更豐富的功能，包括巢狀測試、參數化測試、動態測試和更好的擴展模型。對於已有 JUnit 4 測試的專案，可以使用 JUnit Vintage 在 JUnit 5 平台上執行，逐步遷移。

## 如何在 JUnit 5 中執行 JUnit 4 測試？

添加 JUnit Vintage 引擎依賴即可：

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## 為什麼我的測試沒有執行？

常見原因包括：測試類沒有加 @Test 注解、測試方法不是 public、沒有添加 maven-surefire-plugin 或未配置 useJUnitPlatform()、Java 版本不滿足要求。

## JUnit 5 最低 Java 版本是多少？

JUnit 5 需要 Java 8 或更高版本。JUnit 4 需要 Java 5 或更高版本。

## 如何跳過某些測試？

使用 @Disabled 注解可以跳過單個測試或整個測試類：

```java
@Disabled("暫時停用此測試")
@Test
void testSkipped() {
    // 不會執行
}
```

## 如何測試私有方法？

單元測試應該只測試 public 方法。如果需要測試私有方法，建議重構程式碼，將需要測試的邏輯移到單獨的 public 方法中，或者使用反射（不推薦）。

## 斷言失敗時如何查看詳細資訊？

使用第三個參數提供自訂錯誤訊息：

```java
assertEquals(expected, actual, "計算結果不正確");
```

## 如何測試例外？

使用 assertThrows 方法：

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("錯誤訊息");
    });
}
```

## 如何平行執行測試？

在 JUnit 5 中配置平行執行：

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // 測試將平行執行
}
```

## @Before 和 @BeforeEach 有什麼區別？

@Before 是 JUnit 4 的注解，@BeforeEach 是 JUnit 5 的注解。兩者功能相同，都是在每個測試方法執行前運行。JUnit 5 還支援 @BeforeAll 和 @AfterAll，用於整個測試類只執行一次的前置和後置操作。

## 如何建立參數化測試？

使用 @ParameterizedTest 注解配合 @ValueSource：

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## 測試超時怎麼辦？

使用 @Timeout 注解設定超時時間：

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // 超過 100 毫秒將失敗
}
```

## 如何重複執行測試？

使用 @RepeatedTest 注解：

```java
@RepeatedTest(10)
void testRepeated() {
    // 將重複執行 10 次
}
```

## JUnit 5 的擴展模型是什麼？

JUnit 5 使用 @ExtendWith 注解提供擴展機制，可以實現更靈活的測試定制：

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // 使用自訂擴展
}
```
