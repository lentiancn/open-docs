# Manuel d'utilisation

## Installer et exécuter des applications Android

Waydroid peut effectuer plusieurs opérations trouvées en utilisant la commande `waydroid app -h` :

```bash
usage: waydroid app [-h] {install,remove,launch,list} ...

optional arguments:
  -h, --help            afficher ce message d'aide et quitter

subaction:
  {install,remove,launch,list}
    install             pousser un seul paquet vers le conteneur et l'installer
    remove              supprimer un seul paquet d'application du conteneur
    launch              démarrer une seule application
    list                lister les applications installées
```

### Installer des applications Android

Vous pouvez installer des applications Android avec :

```bash
waydroid app install xyz.apk
```

Ensuite, vous pouvez lancer l'application via le menu des applications de votre bureau.

Si vous connaissez le nom du package Android de l'application (`com.xxx.yyy`), vous pouvez également utiliser la ligne de commande avec :

```bash
waydroid app launch com.foo.bar
```

### Notes

Les fichiers apk que vous trouverez parfois sur Internet ont tendance à ne supporter que arm, et ne fonctionneront donc pas sur x86_64.

Vous voudrez peut-être installer [F-Droid](https://f-droid.org) pour obtenir des applications graphiquement. Ou utilisez le Play Store si vous avez installé une image GAPPS.

## Commandes courantes

### Options de ligne de commande Waydroid

Waydroid fournit de nombreuses options de ligne de commande pour contrôler les conteneurs et les sessions :

- `waydroid show-full-ui` : Afficher l'interface utilisateur Android complète
- `waydroid session start` : Démarrer une session Waydroid
- `waydroid session stop` : Arrêter une session Waydroid
- `waydroid container start` : Démarrer un conteneur Waydroid
- `waydroid container stop` : Arrêter un conteneur Waydroid
- `waydroid logcat` : Afficher les journaux Android
- `waydroid shell` : Entrer dans le shell Android
- `waydroid prop set <property> <value>` : Définir les propriétés système
- `waydroid bugreport` : Générer des journaux de rapport de bogue

### Configuration des propriétés système

Diverses propriétés système peuvent être configurées via la commande `waydroid prop` :

- `persist.waydroid.multi_windows` : Activer le mode multi-fenêtres
- Autres propriétés consultez la documentation officielle

## Fonctionnalités avancées

### Utilisation d'images personnalisées

Pour utiliser des images personnalisées, suivez ces étapes :

1. Téléchargez des images depuis [SourceForge](https://sourceforge.net/projects/waydroid/files/images/) ou utilisez des `system.img` et `vendor.img` compilés personnalisés
2. Copiez les images dans le répertoire `/etc/waydroid-extra/images/`
3. Réinitialisez Waydroid : `sudo waydroid init -f`

### Débogage ADB

Waydroid prend en charge le débogage ADB. Par défaut, le mode sécurisé ADB est activé et ne se connectera pas automatiquement au démarrage de la session. Utilisez la commande suivante pour vous connecter :

```bash
waydroid adb connect
```

Ensuite, fournissez l'autorisation côté Android.

### Transfert de notifications

Waydroid prend en charge le transfert des notifications Android vers l'environnement de bureau, nécessitant une mise à jour de l'image système Android vers une version compatible.
