# Element Plus Installationsanleitung

> Element Plus ist eine Vue 3 Desktop-UI-Komponentenbibliothek vom Element-Team
> 
> Aktualisiert: Februar 2026
> 
> Aktuelle Version: 2.x (Stabil)

---

## 1. Projektübersicht

Element Plus ist eine auf Vue 3 basierende Desktop-UI-Komponentenbibliothek, die vom Element-Team entwickelt und gepflegt wird. Als Vue 3-Upgrade-Version von Element Plus ist Element Plus vollständig im Vue 3 Composition API-Stil geschrieben und vollständig in TypeScript umgeschrieben, was eine typsichere Komponentenentwicklungserfahrung bietet.

Die erste stabile Version von Element Plus wurde am 7. Februar 2022 veröffentlicht. Nach Jahren der Iteration ist die API sehr stabil. Derzeit wird Simplified Chinese, Englisch und andere Sprachen unterstützt und es gibt ein aktives Community-Ökosystem.

## 2. Umgebungsanforderungen

### 2.1 Entwicklungsumgebung Anforderungen

| Umgebung | Mindestversion | Empfohlene Version |
|------|----------|----------|
| Node.js | 16.x | 18.x oder 20.x |
| npm | 8.x | 9.x oder 10.x |
| Vue | 3.2.0 | 3.4.x |
| Vite | 4.x | 5.x |

### 2.2 Browser-Kompatibilität

Element Plus unterstützt alle modernen Browser, einschließlich:

- Chrome >= 80
- Firefox >= 78
- Safari >= 14
- Edge >= 80

Internet Explorer 11 wird nicht unterstützt.

### 2.3 Technologie-Stack-Anforderungen

| Technologie | Beschreibung |
|------|------|
| Vue 3 | Muss Vue 3.2.0 oder höher verwenden |
| TypeScript | Version 4.5 oder höher empfohlen |
| Build-Tool | Vite (empfohlen) oder Webpack |

## 3. Installationsschritte

### 3.1 npm verwenden (Empfohlen)

npm ist der Paketmanager von Node.js und ist der offiziell empfohlene Weg, Element Plus zu installieren.

```bash
# Gehe in dein Vue 3 Projektverzeichnis
cd dein-vue-projekt

# Element Plus installieren
npm install element-plus
```

### 3.2 yarn verwenden

```bash
yarn add element-plus
```

### 3.3 pnpm verwenden

```bash
pnpm add element-plus
```

## 4. In Vue 3 Projekt importieren

### 4.1 Vollständiger Import (geeignet für kleine Projekte)

Der vollständige Import bündelt alle Komponenten, die Konfiguration ist einfach, aber das Bundle ist größer.

```typescript
// main.ts oder main.js
import { createApp } from 'vue'
import App from './App.vue'

// Element Plus und seine Stile importieren
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// Element Plus verwenden
app.use(ElementPlus)

app.mount('#app')
```

### 4.2 Bedarfsweiser Import (empfohlen für Produktion)

Bedarfsweiser Import kann die Bundle-Größe erheblich reduzieren, indem nur tatsächlich verwendete Komponenten importiert werden.

#### 4.2.1 Auto-Import-Plugin installieren

Installiere zuerst unplugin-vue-components und unplugin-auto-import:

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

#### 4.2.2 Vite konfigurieren

Füge die Konfiguration in vite.config.ts hinzu:

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

Mit dieser Konfiguration können Komponenten automatisch importiert werden, ohne manuell zu importieren:

```vue
<template>
  <el-button type="primary">Hallo</el-button>
</template>
```

## 5. Import via CDN (kein Build-Tool erforderlich)

Wenn du npm oder Build-Tools nicht verwenden möchtest, kannst du Element Plus über CDN importieren.

### 5.1 Import via unpkg CDN

```html
<!DOCTYPE html>
<html lang="de">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Element Plus CDN Beispiel</title>
  <!-- Vue 3 importieren -->
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <!-- Element Plus importieren -->
  <link rel="stylesheet" href="https://unpkg.com/element-plus/dist/index.css">
  <script src="https://unpkg.com/element-plus"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary">Hallo Element Plus</el-button>
  </div>

  <script>
    const { createApp, ref } = Vue
    
    const App = {
      setup() {
        const visible = ref(false)
        return { visible }
      },
      template: `
        <el-button @click="visible = true">Dialog öffnen</el-button>
        <el-dialog v-model="visible" title="Hallo">Das ist Element Plus</el-dialog>
      `
    }
    
    const app = createApp(App)
    app.use(ElementPlus)
    app.mount('#app')
  </script>
</body>
</html>
```

## 6. Internationalisierungskonfiguration

Element Plus verwendet standardmäßig Englisch. Um Deutsch zu verwenden, musst du die Internationalisierung konfigurieren.

### 6.1 Globale deutsche Konfiguration

```typescript
// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import de from 'element-plus/es/locale/lang/de'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus, {
  locale: de,
})

app.mount('#app')
```

## 7. Theme-Anpassung

### 7.1 SCSS-Variablen überschreiben

Element Plus ist in SCSS geschrieben. Du kannst das Theme anpassen, indem du Variablen überschreibst.

```scss
// styles/element-variables.scss
// Primärfarbe
$--color-primary: #409EFF;

// Komponentenvariablen überschreiben
@use "element-plus/theme-chalk/src/common/var.scss" as *;
```

### 7.2 Dunkelmodus

Element Plus hat eingebaute Unterstützung für den Dunkelmodus:

```typescript
# Dunkelmodus aktivieren
import 'element-plus/theme-chalk/dark/css-vars.css'
```

Klasse zum HTML-Tag hinzufügen:

```html
<html class="dark">
  <!-- Deine App -->
</html>
```

## 8. Häufige Probleme

### 8.1 Installationsfehler

**Problem**: npm install schlägt fehl

**Lösungen**:

1. npm-Cache leeren: `npm cache clean --force`
2. Netzwerk überprüfen, du kannst inländische Spiegel verwenden:

```bash
# npm-Spiegel einstellen
npm config set registry https://registry.npmmirror.com
```

### 8.2 Stile werden nicht angewendet

**Problem**: Komponentenstile werden abnormal angezeigt

**Lösungen**:

1. Stelle sicher, dass du die Stildatei importiert hast: `import 'element-plus/dist/index.css'`
2. Überprüfe, ob die Build-Konfiguration CSS korrekt verarbeitet
3. Browser-Cache leeren und neu bauen
