# Mockito FAQ

Mockitoに関するよくある質問。

## 一般的な質問

### Mockitoとは？

Mockitoは、Java向けのオープンソースモックフレームワークです。ユニットテスト用のモックオブジェクトを作成および設定できます。

### Mockitoを使用する理由？

- 外部依存関係のない分離テスト
- より速いテスト実行
- シンプルで直感的なAPI
- JUnitとの優れた統合

### MockとSpyの違いは？

- **Mock** - 完全に制御できる人工的なオブジェクト
- **Spy** - 特定のメソッドのみをオーバーライドする部分的に実際のオブジェクト

## 技術的な質問

### finalクラスのモック方法は？

`mockito-inline`を使用:

```groovy
testImplementation 'org.mockito:mockito-inline:5.8.0'
```

### 静的メソッドのモック方法は？

Mockito 3.7以降で可能:

```java
try (MockedStatic<Utilities> mocked = Mockito.mockStatic(Utilities.class)) {
    mocked.when(Utilities::getName).thenReturn("mocked");
    // テストコード
}
```

## トラブルシューティング

### エラー: "Cannot mock/spy class"

**原因:** クラスがfinalまたはメソッドがfinalです。

**解決策:**
- `mockito-inline`を使用
- クラスの代わりにインターフェースをモック

### 警告: "Unnecessary stubbing"

**原因:** スタブが使用されませんでした。

**解決策:**
- 未使用のスタブを削除
- または`lenient()`を使用:

```java
lenient().when(mock.method()).thenReturn("value");
```

## 統合

### JUnit 4の場合

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private List mockList;
}
```

### JUnit 5の場合

```java
@ExtendWith(MockitoExtension.class)
public class MyTest {
    @Mock
    private List mockList;
}
```
