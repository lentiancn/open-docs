# Hadoop 사용자 매뉴얼

## HDFS 작업 가이드

### 기본 파일 시스템 작업

HDFS는 기존 파일 시스템과 유사한 명령줄 인터페이스를 제공하여 사용자가 hdfs dfs 명령을 통해 다양한 파일 작업을 수행할 수 있습니다. 이 명령은 Linux 셸 명령과 매우 유사하므로 Linux 시스템에 익숙한 사용자는 빠르게 시작할 수 있습니다. Hadoop 사용의 기본이 되는 이러한 기본 작업을 이해하세요. 아래에서는 가장 많이 사용되는 HDFS 명령을 설명합니다.

디렉토리 생성은 mkdir 명령을 사용하며 단일 수준 또는 다중 수준 디렉토리를 만들 수 있습니다:

```bash
# 단일 수준 디렉토리 생성
hdfs dfs -mkdir /user/data

# 다중 수준 디렉토리 생성
hdfs dfs -mkdir -p /user/data/input

# 디렉토리 내용 나열
hdfs dfs -ls /user/data

# 디렉토리 내용을 재귀적으로 나열
hdfs dfs -ls -R /user/data
```

파일 업로드 및 다운로드는 일반적인 작업입니다:

```bash
# 로컬 파일을 HDFS에 업로드
hdfs dfs -put localfile.txt /user/data/

# 업로드 및 이름 변경
hdfs dfs -put localfile.txt /user/data/newname.txt

# HDFS 파일을 로컬로 다운로드
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# HDFS 내부 파일 복사
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# HDFS 내부 파일 이동
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

파일 보기 및 삭제:

```bash
# 파일 내용 보기
hdfs dfs -cat /user/data/file.txt

# 파일의 마지막 줄 보기
hdfs dfs -tail /user/data/file.txt

# 파일 크기 보기
hdfs dfs -du /user/data/file.txt

# 파일 삭제
hdfs dfs -rm /user/data/file.txt

# 디렉토리 재귀적으로 삭제
hdfs dfs -rm -r /user/data/dir
```

### 권한 관리

HDFS의 권한 모델은 Unix/Linux와 유사하며 사용자, 그룹, 기타 사용자의 3단계 시스템에 기반합니다. 각 파일과 디렉토리에는 소유자, 그룹, 기타 사용자에 해당하는 읽기(r), 쓰기(w), 실행(x) 세 가지 권한이 있습니다. 데이터 보안을 보호하기 위해 권한 관리를 이해하는 것이 중요합니다.

```bash
# 파일 소유자 변경
hdfs dfs -chown user:group /user/data/file.txt

# 파일 권한 변경
hdfs dfs -chmod 755 /user/data/file.txt

# 소유 그룹 변경
hdfs dfs -chgrp groupname /user/data/file.txt
```

### 데이터 복제 및 균형 조정

HDFS는 기본적으로 각 데이터 블록을 3번 복제하여 서로 다른 DataNode에 저장합니다. 기본 복제 계수는 dfs.replication 속성을 통해 수정할 수 있습니다:

```bash
# 특정 파일의 복제 계수 설정
hdfs dfs -setrep -w 2 /user/data/file.txt

# 파일의 블록 위치 정보 보기
hdfs fsck /user/data/file.txt -files -blocks -locations
```

클러스터에 노드가 추가되거나 제거되면 데이터가 고르게 분산되지 않을 수 있습니다. 이 경우 rebalance 명령을 사용하여 데이터를 균형 조정하세요:

```bash
start-balancer.sh -threshold 10
```

threshold 매개변수는 노드 디스크 사용률과 클러스터 평균의 최대 차이 백분율을 나타냅니다. 균형 조정 프로세스는 네트워크 대역폭을 소비합니다. 낮은 트래픽 기간에 실행하는 것이 좋습니다.

### HDFS Web UI

HDFS는 NameNode의 9870 포트를 통해 브라우저에서 액세스할 수 있는 기능이 풍부한 Web UI를 제공합니다. Web UI에서 파일 시스템 디렉토리 검색, 파일 내용 보기, DataNode 상태 모니터링, 블록 정보 보기 등을 할 수 있습니다. Web UI는 운영 담당자의 일상적인 모니터링 및 문제 해결에 중요한 도구입니다.

Web UI의 주요 기능: HDFS 디렉토리 트리 검색, 파일 목록 및 파일 속성 보기; DataNode 목록 및 각 DataNode의 스토리지 사용량 보기; 블록 분배 및 복제 상태 보기; NameNode 런타임 정보 및 로그 보기; 다운로드 및 삭제와 같은 간단한 파일 작업 수행.

## MapReduce 프로그래밍

### WordCount 예제

WordCount는 MapReduce 프로그래밍의 입문 예제로, 텍스트에서 각 단어의 출현 횟수를 계산합니다. 이 예제를 이해하면 MapReduce 프로그래밍 모델을 마스터하는 데 도움이 됩니다. 프로그램은 Map 단계와 Reduce 단계로 나뉩니다: Map 단계에서 텍스트를 단어로 분할하고, Reduce 단계에서 동일한 단어의 카운트를 집계합니다.

Map 단계 코드 예제:

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduce 단계 코드 예제:

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### 작업 제출

MapReduce를 JAR 파일로 패키징한 후 hadoop jar 명령을 사용하여 작업을 제출하세요:

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

작업을 제출하면 YARN의 Web UI(포트 8088)에서 작업 진행 상황과 결과를 볼 수 있습니다. 작업이 완료되면 작업 ID, 작업 수, 실행 시간 등의 작업 통계가 출력됩니다.

### 작업 구성

Configuration 객체를 통해 다양한 작업 매개변수를 구성할 수 있습니다:

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

일반적으로 사용되는 구성 매개변수: mapreduce.job.reduces는 Reduce 작업 수를 설정; mapreduce.map.memory.mb와 mapreduce.reduce.memory.mb는 컨테이너 메모리를 설정; mapreduce.map.cpu.vcores와 mapreduce.reduce.cpu.vcores는 컨테이너 CPU 코어 수를 설정.

## YARN 작업

### 클러스터 리소스 보기

YARN은 클러스터 리소스를 관리하기 위한丰富的命令行 도구를 제공합니다:

```bash
# 모든 노드 상태 보기
yarn node -list -all

# 애플리케이션 목록 보기
yarn application -list

# 특정 애플리케이션 상태 보기
yarn application -status application_1234567890123_0001

# 애플리케이션 로그 보기
yarn apps -logs application_1234567890123_0001
```

### 대기열 관리

YARN은 리소스 할당을 관리하기 위해 대기열을 사용합니다. 용량 스케줄러 또는 공정 스케줄러를 통해 대기열을 구성할 수 있습니다:

```bash
# 대기열 정보 보기
yarn queue -status default

# 대기열 용량 보기
yarn resourcemanager -refreshQueues
```

### 리소스 스케줄러 구성

용량 스케줄러는 기본 스케줄러로, 멀티테넌시 리소스 공유를 지원합니다. yarn-site.xml에서 구성하세요:

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## 성능 최적화

### MapReduce 튜닝

MapReduce 작업 성능 튜닝은 여러 측면을 포함합니다. 먼저 Map 및 Reduce 작업 수를 합리적으로 설정하세요. Map 작업 수는 일반적으로 입력 데이터 분할 수에 의해 결정되며 mapreduce.input.fileinputformat.split.maxsize 매개변수로 조정할 수 있습니다. Reduce 작업 수는 클러스터의 전체 Reduce 슬롯 수의 0.9~1.2배로 설정하는 것이 좋습니다.

```bash
# Map 작업 수 설정
-D mapreduce.job.maps=10

# Reduce 작업 수 설정
-D mapreduce.job.reduces=5
```

셔플 단계 최적화는 전체 성능에 큰 영향을 미칩니다. 다음 매개변수를 조정하세요:

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFS 튜닝

HDFS 성능 최적화는 주로 데이터 읽기/쓰기 효율에 중점을 둡니다. 큰 파일의 경우 데이터 지역성과 결함 복구 사이의 균형을 위해 블록 크기를 조정하세요:

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

단락 읽기를 활성화하면 클라이언트가 로컬 DataNode에서 직접 데이터를 읽어 네트워크 전송을 줄일 수 있습니다:

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARN 튜닝

YARN의 리소스 할당은 클러스터 활용도에 큰 영향을 미칩니다. NodeManager의 리소스 용량을 합리적으로 설정하세요:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

멀티코어 프로세서를 더 잘 활용하려면 가상 CPU 코어 수를 설정하세요:

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## 데이터 가져오기 및 내보내기

### 데이터베이스에서 데이터 가져오기

Sqoop을 사용하면 관계형 데이터베이스와 HDFS 간에 편리하게 데이터를 마이그레이션할 수 있습니다:

```bash
# 전체 테이블 가져오기
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Hive 테이블 생성하여 가져오기
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### 데이터베이스로 데이터 내보내기

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### 데이터 형식 선택

다른 데이터 형식은 성능에 다른 영향을 미칩니다. 일반적으로 사용되는 형식: TextFile(범용성 좋음, 평균 성능); SequenceFile(MapReduce에 좋음); Parquet(열 지향 스토리지, 분석 쿼리에 좋음); ORC(Hive 최적화 열 지향 스토리지). 분석 워크로드에는 Parquet 또는 ORC와 같은 열 지향 스토리지 형식을 사용하는 것이 좋습니다.
