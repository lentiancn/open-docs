# GitLab Installationsanleitung

## Anforderungen

Bevor Sie GitLab installieren, stellen Sie sicher, dass Ihr Server die folgenden Hardware- und Softwareanforderungen erfüllt. Für kleine bis mittlere Teams werden mindestens 4 CPU-Kerne und 8 GB RAM empfohlen, was Hunderte von Repositorys und Dutzende gleichzeitiger Benutzer unterstützen kann. Größere Teams oder Hochlastszenarien erfordern mehr Ressourcen, vorzugsweise 8+ CPU-Kerne und 16 GB+ RAM.

Bei Betriebssystemen unterstützt GitLab verschiedene Linux-Distributionen, darunter Ubuntu, Debian, CentOS, Red Hat Enterprise Linux und Fedora. Ubuntu oder Debian wird offiziell empfohlen, da ihr Paketmanagement am bequemsten ist. GitLab benötigt mindestens 20 GB Festplattenspeicher für die Installation, 50 GB+ werden für Basis-Konfigurationen empfohlen, und oft Hunderte von GB für Produktionsumgebungen.

Software-Abhängigkeiten umfassen OpenSSH Server für SSH-Protokollunterstützung, PostgreSQL-Datenbank (GitLab enthält eine integrierte Version) und SMTP-Server für E-Mail-Benachrichtigungen. Sie müssen auch die richtige Zeitzone und Hostname konfigurieren und sicherstellen, dass der Server normalen Internetzugang hat, um Abhängigkeitspakete herunterzuladen.

## Installationsschritte

### Installation unter Ubuntu/Debian

Unter Ubuntu oder Debian-Systemen wird die Installation über das offizielle apt-Repository empfohlen, da dies sicherstellt, dass Sie die neuesten stabilen Versionen und automatische Sicherheitsupdates erhalten. Installieren Sie zunächst die erforderlichen Abhängigkeitspakete:

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

Fügen Sie das GitLab APT-Repository hinzu und installieren Sie GitLab CE (Community Edition):

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

Ersetzen Sie EXTERNAL_URL durch die tatsächliche Domain oder IP-Adresse für den Zugriff auf GitLab. Der Installationsprozess konfiguriert automatisch PostgreSQL, Redis und GitLab selbst.

### Installation unter CentOS/RHEL

Unter CentOS oder Red Hat Enterprise Linux können Sie GitLab mit dem yum- oder dnf-Paketmanager installieren. Installieren Sie zunächst die erforderlichen Abhängigkeiten:

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

Fügen Sie das GitLab YUM-Repository hinzu und installieren Sie:

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## Erstkonfiguration

Nach der Installation müssen Sie die grundlegenden Parameter von GitLab konfigurieren. Bearbeiten Sie die Datei /etc/gitlab/gitlab.rb, die die Hauptkonfigurationsdatei von GitLab ist. Der wichtigste Konfigurationspunkt ist external_url, die externe Zugriffsadresse für den GitLab-Dienst:

```ruby
external_url 'https://gitlab.example.com'
```

Wenn Sie einen benutzerdefinierten SSH-Port verwenden müssen, konfigurieren Sie den Parameter gitlab_rails['gitlab_shell_ssh_port']. Die E-Mail-Server-Konfiguration ist ebenfalls wichtig,用于发送通知和密码重置邮件：

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

Speichern Sie nach dem Speichern der Konfiguration den folgenden Befehl, um die Änderungen anzuwenden:

```bash
sudo gitlab-ctl reconfigure
```

Dieser Befehl konfiguriert alle GitLab-Komponenten neu, einschließlich Datenbankmigrationen und Erstellung des Erstbenutzers. Der gesamte Prozess kann mehrere Minuten dauern.

## Installation überprüfen

Nachdem die Konfiguration abgeschlossen ist, können Sie über Ihren Browser auf die von external_url angegebene Adresse zugreifen. Wenn alles korrekt ist, sollten Sie die GitLab-Anmeldeseite sehen. Beim ersten Zugriff sehen Sie die Anmeldeseite des Administratorkontos. Der Standard-Benutzername ist root, und Sie müssen beim ersten Anmelden ein Passwort festlegen.

GitLab bietet das Befehlszeilentool gitlab-ctl zur Verwaltung des Dienststatus:

```bash
# Dienststatus prüfen
sudo gitlab-ctl status

# Logs anzeigen
sudo gitlab-ctl tail

# Dienst neu starten
sudo gitlab-ctl restart
```

Sie können auch den Befehl gitlab-rake für Gesundheitsprüfungen verwenden:

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Docker-Installation

Für schnelle Tests oder kleine Bereitstellungen können Sie GitLab mit Docker ausführen. Installieren Sie zunächst Docker Engine und führen Sie dann den folgenden Befehl auf dem Host aus:

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

Die Docker-Installation montiert drei Volumes für die Persistenz von Konfiguration, Protokollen und Daten. Der erste Start des Docker-Containers nimmt Zeit für die Initialisierung in Anspruch. Nach geduldigem Warten können Sie über Ihren Browser darauf zugreifen.

## Upgrade und Wartung

Das regelmäßige Aktualisieren von GitLab auf die neueste Version bietet neue Funktionen und Sicherheitsfixes. Upgrade unter Ubuntu/Debian:

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

Es wird empfohlen, vor dem Upgrade ein Backup zu erstellen. GitLab bietet ein integriertes Backup-Tool:

```bash
sudo gitlab-backup create
```

Backup-Dateien werden standardmäßig im Verzeichnis /var/opt/gitlab/backups gespeichert, und Sie können Backup-Aufbewahrungsrichtlinien und Remote-Speicherorte konfigurieren.
