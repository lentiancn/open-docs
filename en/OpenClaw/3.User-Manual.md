# OpenClaw User Manual

This manual covers daily usage and core features of OpenClaw.

## Basic Concepts

### Gateway

The Gateway is the core component of OpenClaw, responsible for session management, message routing, and channel connections. You only need to run one Gateway process, and it automatically handles all channel connections.

### Channels

Channels are the chat applications you use to communicate with your AI assistant. OpenClaw supports connecting multiple channels simultaneously, including WhatsApp, Telegram, Discord, iMessage, and more.

### Agents

Agents are instances of AI assistants. Each agent has its own workspace and sessions. You can configure multiple agents to handle different tasks independently.

## Daily Operations

### Starting and Stopping Gateway

```bash
# Start Gateway (background service)
openclaw gateway start

# Stop Gateway
openclaw gateway stop

# Restart Gateway
openclaw gateway restart

# Check status
openclaw gateway status

# Run in foreground (for debugging)
openclaw gateway --port 18789
```

### Sending Messages

```bash
# Send a direct message
openclaw message send --target +15555550123 --message "Hello"

# Send to a group
openclaw message send --target "Group Name" --message "Hello everyone"
```

### Managing Configuration

```bash
# View current configuration
openclaw config get

# Set a configuration value
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# Remove a configuration value
openclaw config unset channels.whatsapp.enabled
```

## Configuration Management

### Configuration File Location

The configuration file is located at `~/.openclaw/openclaw.json`.

### Hot Reload

OpenClaw supports configuration hot reload — you can modify the config file without manually restarting the Gateway. Most configuration changes take effect automatically.

### Common Configuration Examples

#### Setting Allowed Message Sources

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### Setting Group Chat Mention Rules

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### Configuring Multiple Agents

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

```bash
openclaw channels list
```

## Advanced Features

### Cron Jobs

Configure scheduled tasks:

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Heartbeat

Set up periodic check-ins:

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Sandbox Mode

Run agents in isolated Docker containers:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

Set up HTTP webhook endpoints:

```json5
{
  hooks: {
    enabled: true,
    token: "your-secret-token",
    path: "/hooks"
  }
}
```

```bash
openclaw logs

openclaw logs -f
```

### Health Check

```bash
openclaw health
```

### Diagnosing Issues

```bash
openclaw doctor
```

## Control UI

OpenClaw provides a web-based Control UI where you can:

- Send and receive messages
- Manage configuration
- View session history
- Manage mobile nodes
- Monitor Gateway status

Access it at: `http://127.0.0.1:18789/`
