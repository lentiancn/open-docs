# GitLab Manual de Usuario

> Guía completa para usar GitLab en desarrollo y operaciones

---

## Primeros Pasos

### Crear una Cuenta

1. Navegue a su instancia de GitLab
2. Haga clic en "Registrarse" para crear una cuenta
3. Verifique su dirección de correo electrónico
4. Complete su perfil de usuario

### Crear un Proyecto

1. Haga clic en "Nuevo Proyecto"
2. Elija plantilla de proyecto o proyecto en blanco
3. Establezca nombre y descripción del proyecto
4. Configure nivel de visibilidad (privado, interno, público)
5. Haga clic en "Crear Proyecto"

---

## Operaciones Git

### Clonar un Repositorio

```bash
# Clonar usando HTTPS
git clone https://gitlab.example.com/usuario/proyecto.git

# Clonar usando SSH
git clone git@gitlab.example.com:usuario/proyecto.git
```

### Flujo de Trabajo Git Básico

```bash nueva rama
git
# Crear checkout -b feature/nueva-caracteristica

# Hacer cambios y prepararlos
git add .
git commit -m "Agregar nueva característica"

# Push al remoto
git push origin feature/nueva-caracteristica

# Crear merge request desde rama
```

### Gestionar Ramas

```bash
# Listar ramas
git branch -a

# Eliminar rama local
git branch -d feature/antigua-caracteristica

# Eliminar rama remota
git push origin --delete feature/antigua-caracteristica
```

## Seguimiento de Problemas

### Crear Problemas

1. Navegue a su proyecto
2. Vaya a Problemas > Nuevo Problema
3. Agregue título y descripción
4. Asigne etiquetas, hitos y responsables
5. Establezca peso y fecha de vencimiento
6. Envíe el problema

### Usar Tableros de Problemas

1. Vaya a Problemas > Tableros
2. Cree nuevo tablero o use el predeterminado
3. Agregue listas para cada etapa (Por hacer, En progreso, Hecho)
4. Arrastre y suelte problemas entre listas

### Etiquetas e Hitos

- Crear etiquetas descriptivas para categorización
- Usar hitos para seguir el progreso
- Cerrar problemas con mensajes de commit

## Merge Requests

### Crear un Merge Request

1. Push su rama de característica a GitLab
2. Haga clic en "Crear Merge Request"
3. Complete título y descripción
4. Seleccione rama fuente y destino
5. Agregue etiquetas y revisores
6. Envíe el merge request

### Proceso de Revisión de Código

- Los revisores pueden agregar comentarios en líneas específicas
- Usar discusiones de hilo para conversaciones
- Aprobar o solicitar cambios
- Resolver todos los hilos antes de fusionar

### Squash y Merge

- Combinar commits para mantener historial limpio
- Elegir eliminar rama fuente después de fusionar
- Editar mensaje de commit antes de fusionar

## Pipelines CI/CD

### .gitlab-ci.yml Básico

```yaml
etapas:
  - build
  - test
  - deploy

build:
  etapa: build
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/

test:
  etapa: test
  script:
    - npm test
  coverage: '/Coverage: \d+\.\d+%/'

deploy:
  etapa: deploy
  script:
    - npm run deploy
  only:
    - main
```

### Ejecutar Pipelines

- Los pipelines se ejecutan automáticamente con push
- Disparar manualmente desde UI
- Ver estado del pipeline en merge requests

### Variables de Pipeline

- Definir variables en configuración de proyecto
- Usar variables CI/CD para datos sensibles
- Variables enmascaradas y protegidas disponibles

## Wiki

### Crear Páginas Wiki

1. Vaya a Proyecto > Wiki
2. Haga clic en "Nueva Página"
3. Escriba contenido en Markdown
4. Agregue navegación de barra lateral
5. Guarde la página

### Características Wiki

- Soporte Markdown
- Archivos adjuntos
- Estructura de páginas anidadas
- Historial de versiones
