# JUnit 5 FAQ

## General Questions

### What is JUnit?
JUnit is an open source unit testing framework for Java and the JVM. It is part of the xUnit family of testing frameworks and is the most widely used testing framework for Java applications.

### What is JUnit 5?
JUnit 5 (Jupiter) is the latest major version of the JUnit testing framework. It introduces a new architecture with three modules: Platform, Jupiter, and Vintage.

### What is the difference between JUnit 4 and JUnit 5?

| Feature | JUnit 4 | JUnit 5 |
|---------|----------|---------|
| Architecture | Monolithic | Modular (Platform, Jupiter, Vintage) |
| Java Version | Java 5+ | Java 8+ |
| Test Naming | Method names only | Display names |
| Nested Tests | Not supported | Supported |
| Parameterized Tests | External library | Built-in |
| Extensions | @RunWith | @ExtendWith |

### Do I need to rewrite all my JUnit 4 tests?
No! JUnit 5 includes JUnit Vintage, which allows running JUnit 3 and 4 tests on the JUnit 5 platform.

## Installation Questions

### How do I add JUnit 5 to my Maven project?
Add the following dependency to your pom.xml:

```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```

### How do I add JUnit 5 to my Gradle project?
Add the following to your build.gradle:

```groovy
test {
    useJUnitPlatform()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}
```

### What are the minimum Java requirements for JUnit 5?
JUnit 5 requires Java 8 or higher at runtime.

## Usage Questions

### How do I write a basic test in JUnit 5?

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class MyFirstTest {
    
    @Test
    void simpleTest() {
        assertEquals(2, 1 + 1);
    }
}
```

### How do I use display names?

```java
@Test
@DisplayName("A simple math test")
void testAddition() {
    assertEquals(4, 2 + 2);
}
```

### How do I create nested tests?

```java
class NestedTests {
    
    @Nested
    class AdditionTests {
        @Test
        void positiveNumbers() {
            assertEquals(5, 2 + 3);
        }
    }
    
    @Nested
    class SubtractionTests {
        @Test
        void positiveResult() {
            assertEquals(1, 3 - 2);
        }
    }
}
```

### How do I write parameterized tests?

```java
@ParameterizedTest
@ValueSource(ints = {1, 2, 3})
void testIsPositive(int number) {
    assertTrue(number > 0);
}
```

### How do I test for exceptions?

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, 
        () -> { throw new IllegalArgumentException("error"); });
}
```

## Advanced Questions

### How do I create custom assertions?

```java
public class CustomAssertions {
    
    public static void assertEven(int number) {
        assertTrue(number % 2 == 0, "Number must be even");
    }
}
```

### How do I use the Extension model?

```java
@ExtendWith(MyExtension.class)
class MyTest {
    // Test methods
}
```

### Can JUnit 5 tests run in parallel?
Yes, JUnit 5 supports parallel execution. Enable it in your build configuration.

## Migration Questions

### How do I migrate from JUnit 4 to JUnit 5?

1. Update your dependencies to JUnit 5
2. Replace `@Test` from `org.junit.Test` to `org.junit.jupiter.api.Test`
3. Replace `assertEquals` from `org.junit.Assert` to `org.junit.jupiter.api.Assertions`
4. Replace `@Before` and `@After` with `@BeforeEach` and `@AfterEach`
5. Replace `@BeforeClass` and `@AfterClass` with `@BeforeAll` and `@AfterAll`
6. Replace `@RunWith` with `@ExtendWith`

### Can I run JUnit 3 and 4 tests with JUnit 5?
Yes, add the JUnit Vintage dependency:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```
