# GitLab 安装指南

## 环境要求

在安装 GitLab 之前，需要确保服务器满足以下硬件和软件要求。对于中小型团队，推荐配置至少 4 核心 CPU 和 8GB 内存，可以支持数百个仓库和数十个并发用户。大型团队或高负载场景需要更多资源，建议使用 8 核心以上 CPU 和 16GB 以上内存。

操作系统方面，GitLab 支持多种 Linux 发行版，包括 Ubuntu、Debian、CentOS、Red Hat Enterprise Linux 和 Fedora。官方推荐使用 Ubuntu 或 Debian，因为这些系统的软件包管理最为便捷。GitLab 需要至少 20GB 的磁盘空间用于安装，基本配置建议预留 50GB 以上，生产环境通常需要数百 GB。

软件依赖方面，GitLab 需要安装 OpenSSH Server 用于 SSH 协议支持，PostgreSQL 数据库（GitLab 自带内置版本），以及 SMTP 服务器用于邮件通知。还需要配置正确的时区和主机名，确保服务器网络正常，可以访问互联网以下载依赖包。

## 安装步骤

### Ubuntu/Debian 安装

在 Ubuntu 或 Debian 系统上，推荐使用官方 apt 仓库安装 GitLab，这样可以确保获得最新的稳定版本和自动安全更新。首先需要安装必要的依赖包：

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

添加 GitLab APT 仓库并安装 GitLab CE（社区版）：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

其中 EXTERNAL_URL 需要替换为实际访问 GitLab 的域名或 IP 地址。安装过程会自动配置 PostgreSQL、Redis 和 GitLab 本身。

### CentOS/RHEL 安装

在 CentOS 或 Red Hat Enterprise Linux 上，可以使用 yum 或 dnf 包管理器安装 GitLab。首先安装必要的依赖：

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

添加 GitLab YUM 仓库并安装：

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## 初始配置

安装完成后，需要配置 GitLab 的基本参数。编辑 /etc/gitlab/gitlab.rb 文件，这是 GitLab 的主配置文件。最重要的配置项是 external_url，即 GitLab 服务的外部访问地址：

```ruby
external_url 'https://gitlab.example.com'
```

如果需要使用自定义 SSH 端口，需要配置 gitlab_rails['gitlab_shell_ssh_port'] 参数。邮件服务器配置也很重要，用于发送通知和密码重置邮件：

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

保存配置后，运行以下命令使配置生效：

```bash
sudo gitlab-ctl reconfigure
```

此命令会重新配置 GitLab 的所有组件，包括数据库迁移、初始用户创建等。整个过程可能需要几分钟时间。

## 验证安装

配置完成后，可以通过浏览器访问 external_url 指定的地址。如果一切正常，应该能看到 GitLab 的登录页面。首次访问时，会看到管理员账户登录页面，默认管理员用户名是 root，首次登录需要设置密码。

GitLab 提供命令行工具 gitlab-ctl 用于管理服务状态：

```bash
# 检查服务状态
sudo gitlab-ctl status

# 查看日志
sudo gitlab-ctl tail

# 重启服务
sudo gitlab-ctl restart
```

也可以使用 gitlab-rake 命令进行健康检查：

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Docker 安装

对于快速测试或小型部署，可以使用 Docker 运行 GitLab。首先需要安装 Docker Engine，然后在主机上运行以下命令：

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

Docker 安装方式挂载了三个卷用于持久化配置、日志和数据。首次启动 Docker 容器需要较长时间初始化，耐心等待后即可通过浏览器访问。

## 升级维护

定期升级 GitLab 到最新版本可以获得新功能和安全修复。在 Ubuntu/Debian 上升级：

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

升级前建议备份数据，GitLab 提供内置备份工具：

```bash
sudo gitlab-backup create
```

备份文件默认保存在 /var/opt/gitlab/backups 目录，可以配置备份保留策略和远程存储位置。
