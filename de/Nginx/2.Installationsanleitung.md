# Nginx Installationsanleitung

## Umgebungsanforderungen

Nginx unterstützt mehrere Betriebssysteme:

- Linux (Debian, Ubuntu, CentOS, RHEL, Alpine usw.)
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## Installationsmethoden

### Methode 1: Paketmanager verwenden (Empfohlen)

#### Ubuntu/Debian

```bash
# Paketindex aktualisieren
sudo apt update

# Nginx installieren
sudo apt install nginx

# Nginx starten
sudo systemctl start nginx

# Beim Boot starten
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# EPEL-Repository installieren
sudo yum install epel-release

# Nginx installieren
sudo yum install nginx

# Nginx starten
sudo systemctl start nginx

# Beim Boot starten
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# Mit Homebrew installieren
brew install nginx

# Nginx starten
brew services start nginx
```

### Methode 2: Aus Quelle kompilieren

#### Build-Abhängigkeiten installieren

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### Quellcode herunterladen

```bash
# Neueste stabile Version herunterladen
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# Entpacken
tar -zxf nginx-1.26.0.tar.gz

# Verzeichnis betreten
cd nginx-1.26.0
```

#### Build konfigurieren

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### Kompilieren und installieren

```bash
# Kompilieren
make

# Installieren
sudo make install
```

### Methode 3: Docker verwenden

```bash
# Offizielles Image herunterladen
docker pull nginx

# Nginx-Container ausführen
docker run -d -p 80:80 --name mynginx nginx

# Konfiguration und Logs einbinden
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### Methode 4: Windows-Installation

1. Windows-Version von https://nginx.org/en/download.html herunterladen
2. In gewünschtes Verzeichnis entpacken
3. nginx.exe doppelklicken zum Starten
4. Oder über Eingabeaufforderung starten:

```cmd
cd C:\nginx
start nginx
```

## Verifizierung nach der Installation

### Installation überprüfen

```bash
# Nginx-Version prüfen
nginx -v

# Detaillierte Versionsinformationen
nginx -V
```

### Ausführungsstatus prüfen

```bash
# Prozess läuft prüfen
ps aux | grep nginx

# Status mit systemctl prüfen
systemctl status nginx
```

### Zugriff testen

```bash
# Mit curl testen
curl http://localhost

# Oder Browser öffnen
http://localhost
```

## Verzeichnisstruktur

### Standardverzeichnisse unter Linux

| Pfad | Beschreibung |
|------|---------------|
| /etc/nginx/ | Konfigurationsverzeichnis |
| /etc/nginx/nginx.conf | Hauptkonfigurationsdatei |
| /etc/nginx/conf.d/ | Unterkonfigurationsverzeichnis |
| /var/log/nginx/ | Log-Verzeichnis |
| /var/www/html/ | Standard-Dokumentenstamm |
| /usr/sbin/nginx | Ausführbare Datei |
| /usr/share/nginx/html/ | Standard-Statische-Dateien-Verzeichnis |

### Kompilierte Installationsverzeichnisse

| Pfad | Beschreibung |
|------|---------------|
| /usr/local/nginx/ | Installations-Stammverzeichnis |
| /usr/local/nginx/conf/ | Konfigurationsverzeichnis |
| /usr/local/nginx/html/ | Standard-Dokumentenstamm |
| /usr/local/nginx/logs/ | Log-Verzeichnis |
| /usr/local/nginx/sbin/ | Ausführbares Verzeichnis |

## Nginx deinstallieren

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### Kompilierte Installation

```bash
# Nginx stoppen
sudo nginx -s stop

# Installationsverzeichnis löschen
sudo rm -rf /usr/local/nginx
```

## Häufige Probleme

### Portkonflikt

Wenn Start wegen Portkonflikt fehlschlägt:

```bash
# Portbelegung prüfen
netstat -tulpn | grep :80

# Nginx-Port ändern
# /etc/nginx/nginx.conf bearbeiten
# listen 80; in listen 8080; ändern
```

### Berechtigungsprobleme

Bei Berechtigungsfehlern:

```bash
# Nginx-Benutzer prüfen
ps aux | grep nginx

# Dateiberechtigungen korrigieren
sudo chown -R www-data:www-data /var/www/html/
```

### Konfigurationssyntax prüfen

```bash
# Konfigurationssyntax testen
nginx -t

# Mit bestimmter Konfiguration testen
nginx -t -c /path/to/nginx.conf
```
