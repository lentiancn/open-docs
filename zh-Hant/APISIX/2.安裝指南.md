# APISIX 安裝指南

本文檔介紹 Apache APISIX 的多種安裝方式。

## Docker 安裝（推薦）

使用 Docker 快速部署 APISIX 是最簡單的方式：

```bash
# 拉取 APISIX 映像檔
docker pull apache/apisix:latest

# 啟動 APISIX
docker run -d -p 9080:9080 -p 9180:9180 -p 9443:9443 \
  --name apisix \
  apache/apisix:latest
```

### 使用 Docker Compose 安裝

建立 `docker-compose.yml` 檔案：

```yaml
version: '3'

services:
  apisix:
    image: apache/apisix:latest
    ports:
      - "9080:9080"
      - "9443:9443"
      - "9180:9180"
    environment:
      - APISIX_DASHBOARD_URL=http://apisix:9180
    volumes:
      - ./apisix_log:/var/log/apisix
    network_mode: host

  etcd:
    image: apache/apisix:etcd:latest
    ports:
      - "2379:2379"
    environment:
      - ETCD_HOST=127.0.0.1
    network_mode: host
```

啟動服務：

```bash
docker-compose up -d
```

## 快速入門腳本安裝

APISIX 官方提供快速安裝腳本，依賴 Docker 和 curl：

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

該腳本會自動：
1. 啟動 etcd 容器
2. 啟動 APISIX 容器
3. 設定網路連線

安裝完成後輸出：
```
✔ APISIX is ready!
```

驗證安裝：
```bash
curl "http://127.0.0.1:9080" --head | grep Server
# 輸出: Server: APISIX/Version
```

## Helm Chart 安裝（Kubernetes）

在 Kubernetes 環境中使用 Helm 安裝：

```bash
# 新增 APISIX Helm 倉庫
helm repo add apisix https://apache.github.io/apisix-helm-chart
helm repo update

# 安裝 APISIX
helm install apisix apisix/apisix
```

## 原始碼編譯安裝

### 環境要求

- Nginx ≥ 1.21.4
- OpenResty ≥ 1.21.4
- Lua ≥ 5.1
- etcd ≥ 3.4.0

### 編譯步驟

```bash
# 複製原始碼
git clone https://github.com/apache/apisix.git
cd apisix

# 安裝依賴
make deps

# 啟動 APISIX
make run
```

## 驗證安裝

安裝完成後，可以透過以下方式驗證：

### 1. 檢查服務狀態

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

### 2. 存取 Dashboard

Dashboard 預設位址：http://127.0.0.1:9180/ui

### 3. 使用 Admin API

```bash
# 取得路由列表
curl -s http://127.0.0.1:9180/apisix/admin/routes

# 建立範例路由
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

## 設定檔說明

APISIX 主要設定檔 `config.yaml`：

```yaml
apisix:
  node_listen: 9080
  enable_admin: true
  admin_key:
    - name: admin
      key: your-admin-key
      role: admin

etcd:
  host:
    - "http://127.0.0.1:2379"
  prefix: /apisix
```

## 常見問題

### 1. 連接埠衝突

如果 9080 或 9443 連接埠被佔用，修改 `config.yaml` 中的連接埠設定。

### 2. etcd 連線失敗

確認 etcd 服務正常執行，檢查 `etcd.host` 設定是否正確。

### 3. 效能問題

- 調整 worker 行程數：`nginx.worker_processes`
- 增加連線數：`nginx.worker_connections`
