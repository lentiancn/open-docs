# Введение в Apache JMeter

Apache JMeter — программное обеспечение с открытым исходным кодом от Apache Software Foundation, приложение на чистом Java, предназначенное для нагрузочного тестирования и измерения производительности. JMeter изначально был разработан для тестирования веб-приложений, но с тех пор расширился и на другие функции тестирования.

## Что может JMeter

JMeter можно использовать для тестирования производительности статических и динамических ресурсов, включая веб-приложения. Он может имитировать высокую нагрузку на сервер, группу серверов, сеть или объект для проверки их прочности или анализа общей производительности при различных типах нагрузки.

Основные характеристики включают:

- **Поддержка множества протоколов**: Возможность нагрузочного тестирования различных типов приложений, серверов и протоколов:
  - Веб — HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET и др.)
  - Веб-службы SOAP / REST
  - FTP
  - Базы данных через JDBC
  - LDAP
  - Ориентированный на сообщения промежуточное ПО (MOM) через JMS
  - Почта — SMTP(S), POP3(S) и IMAP(S)
  - Нативные команды или сценарии оболочки
  - TCP
  - Java-объекты

- **Полнофункциональная среда тестирования IDE**: Позволяет быструю запись планов тестирования (из браузеров или нативных приложений), создание и отладку

- **Режим CLI**: Режим командной строки (ранее называвшийся режимом без GUI или headless) для нагрузочного тестирования с любой OC, совместимой с Java (Linux, Windows, Mac OSX и др.)

- **Динамические HTML-отчёты**: Полная и удобная для представления генерация динамических HTML-отчётов

- **Извлечение данных**: Простая корреляция благодаря возможности извлекать данные из популярных форматов — HTML, JSON, XML или любого текстового формата

- **Полная переносимость на Java**: Полностью портативная и 100% чистая реализация на Java

- **Многопоточная среда**: Параллельная выборка многими потоками и одновременная выборка различных функций отдельными группами потоков

- **Высокая расширяемость**: Подключаемые сэмплеры, сценариемые сэмплеры (языки, совместимые с JSR223, такие как Groovy и BeanShell)

- **Непрерывная интеграция**: Поддержка интеграции с Maven, Gradle и Jenkins

## JMeter — это не браузер

JMeter работает на уровне протокола. Что касается веб-служб и удалённых служб, JMeter выглядит как браузер (скорее, как несколько браузеров); однако JMeter не выполняет все действия, поддерживаемые браузерами. В частности, JMeter не выполняет JavaScript, найденный на HTML-страницах, и не отображает HTML-страницы так, как это делает браузер.

## Области применения JMeter

- **Нагрузочное тестирование**: Имитация большого количества одновременных пользователей для проверки производительности системы при высокой нагрузке
- **Тестирование производительности**: Измерение времени отклика и пропускной способности системы при различных условиях нагрузки
- **Функциональное тестирование**: Проверка функционального поведения приложения
- **Стресс-тестирование**: Проверка стабильности и отказоустойчивости системы при высоком давлении
- **Распределённое тестирование**: Поддержка координированного нагрузочного тестирования на нескольких машинах
