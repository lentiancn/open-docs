# Flutter Benutzerhandbuch

## Grundkonzepte

### Widget-Einführung

Widget ist der grundlegende Baustein der Flutter-Benutzeroberfläche. In Flutter ist alles ein Widget, einschließlich Layout, Padding, Align usw. Widgets beschreiben, wie ihre Ansicht bei gegebener aktueller Konfiguration und Zustand aussehen sollte.

Flutter-Widgets sind unveränderlich. Wenn Sie die Benutzeroberfläche ändern müssen, müssen Sie das Widget neu erstellen oder StatefulWidget mit Zustandsverwaltung verwenden. Widgets können verschachtelt werden, um einen Widget-Baum zu bilden, und Flutter vergleicht effizient den alten und neuen Widget-Baum und rendert nur die tatsächlich geänderten Teile.

Häufige Widgets umfassen: Text, Image, Icon, Container, Row/Column, Stack, ListView usw.

### StatelessWidget und StatefulWidget

StatelessWidget ist ein Widget ohne Zustand, geeignet für statische UI-Anzeigen wie Text und Bilder. Die Definition eines StatelessWidget erfordert nur die Implementierung der build-Methode, die den zu rendernden Widget-Baum zurückgibt.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget ist ein Widget mit Zustand, geeignet für Szenarien, die dynamische UI-Updates erfordern, wie Zähler und Formulare. StatefulWidget muss mit Zustand verwendet werden, wobei setState aufgerufen wird, um Flutter zu benachrichtigen, wenn sich der Zustand ändert.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## Layout-Grundlagen

### Lineares Layout

Row und Column sind die am häufigsten verwendeten linearen Layout-Widgets. Row arrangiert untergeordnete Widgets horizontal, Column vertikal. Sie können MainAxisAlignment und CrossAxisAlignment verwenden, um die Ausrichtung zu steuern.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### Stack-Layout

Stack wird verwendet, um Stack-Layouts zu erstellen, wobei Widgets auf ein anderes Widget gesetzt werden. Positioned wird verwendet, um untergeordnete Widgets in einem Stack zu positionieren.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### Listen-Layout

ListView ist das am häufigsten verwendete Listen-Widget, geeignet für lange Listen. ListView.builder eignet sich für große Datenmengen mit Lazy Loading. ListView.separated kann Trennlinien hinzufügen.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## Zustandsverwaltung

### setState

setState ist die grundlegendste Methode zur Zustandsverwaltung, geeignet für einfachen lokalen Zustand. In StatefulWidget löst das Aufrufen von setState einen Widget-Neuaufbau aus.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider ist eine der von Flutter empfohlenen Lösungen zur Zustandsverwaltung. Durch Provider kann Zustand im Widget-Baum geteilt werden, um层层传递 zu vermeiden.

```dart
// ChangeNotifier definieren
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Provider verwenden
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## Netzwerkanfragen

### HTTP-Paket

Verwenden Sie das http-Paket für einfache HTTP-Anfragen:

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

## Lokale Speicherung

### SharedPreferences

Wird zum Speichern einfacher Schlüssel-Wert-Daten verwendet:

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

## Navigation und Routing

Flutter verwendet Navigator für die Seitenavigation. Grundlegende Operationen umfassen Navigator.push (neue Seite hinzufügen), Navigator.pop (zurück), Navigator.pushReplacement (aktuelle Seite ersetzen) usw.

Benanntes Routing ist die empfohlene Routing-Verwaltungsmethode:

```dart
// Routen definieren
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// Navigieren
Navigator.pushNamed(context, '/detail');
```

## Erstellen und Veröffentlichen

### Android-Veröffentlichung

Verwenden Sie keytool zum Generieren eines Signaturschlüssels:

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

Konfigurieren Sie die Signatur in android/app/build.gradle, dann führen Sie im Projektverzeichnis aus:

```bash
flutter build apk --release
```

### iOS-Veröffentlichung

Konfigurieren Sie das Signaturzertifikat und Bereitstellungsprofil in Xcode, dann führen Sie aus:

```bash
flutter build ios --release
```

Beachten Sie, dass für die iOS-Veröffentlichung ein Apple-Entwicklerkonto und entsprechende Zertifikate erforderlich sind.
