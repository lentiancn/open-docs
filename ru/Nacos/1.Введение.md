# Введение

## Что такое Nacos

Nacos /nɑ:kəʊs/ — это аббревиатура от Dynamic Naming and Configuration Service, простая в использовании платформа для динамического обнаружения сервисов, управления конфигурацией и управления сервисами для создания облачных приложений.

Nacos стремится помочь вам обнаруживать, настраивать и управлять микросервисами и приложениями ИИ-агентов. Он предоставляет набор простых и полезных функций, позволяющих реализовать динамическое обнаружение сервисов, настройку сервисов, метаданные сервисов и управление трафиком.

Nacos упрощает и ускоряет создание, доставку и управление вашей платформой микросервисов. Это инфраструктура, поддерживающая современную архитектуру приложений, ориентированную на сервисы, с использованием микросервисов или облачных технологий.

Nacos поддерживает обнаружение, настройку и управление практически всеми основными типами сервисов:

- Kubernetes Service
- gRPC
- Dubbo RPC Service
- Spring Cloud RESTful Service
- Model Context Protocol

## Основные функции

### Обнаружение сервисов и проверка работоспособности

Nacos поддерживает обнаружение сервисов на основе DNS и RPC. После того как провайдер сервиса регистрирует сервис с помощью нативного SDK, OpenAPI или выделенного агента, потребители могут обнаруживать сервис через DNS или HTTP API.

Nacos обеспечивает проверку работоспособности в реальном времени, предотвращая отправку запросов на неработающие хосты или экземпляры сервисов. Nacos поддерживает проверки работоспособности на транспортном уровне (PING или TCP) и на уровне приложения (например, HTTP, MySQL, пользовательские протоколы). Для проверки работоспособности в сложных облачных средах и сетевых топологиях (например, VPC, пограничные сервисы) Nacos предоставляет два режима проверки: режим сообщения агента и активное обнаружение на стороне сервера. Nacos также предоставляет унифицированную панель мониторинга работоспособности, которая помогает управлять доступностью и трафиком сервисов на основе их состояния.

### Динамическая конфигурация

Динамическая конфигурация позволяет управлять конфигурацией всех приложений и сервисов централизованно, внешне и динамически во всех средах. Динамическая конфигурация устраняет необходимость повторного развертывания приложений и сервисов при обновлении конфигурации, делая управление конфигурацией более эффективным и гибким.

Централизованное управление конфигурацией упрощает реализацию бессервисных приложений и эластичное масштабирование экземпляров сервисов по требованию. Nacos предоставляет удобный пользовательский интерфейс для управления всеми конфигурациями приложений и сервисов. Nacos также предоставляет готовые функции, включая отслеживание версий конфигурации, канареечное развертывание, откат конфигурации одним щелчком и отслеживание статуса обновления конфигурации на стороне клиента, обеспечивая безопасность и контролируя риски изменений конфигурации в производственной среде.

### Динамическая служба DNS

Диynamическая служба DNS поддерживает взвешенную маршрутизацию, упрощая балансировку нагрузки промежуточного уровня, гибкие политики маршрутизации, управление трафиком и простые службы разрешения DNS в вашем центре обработки данных. Динамическая служба DNS также упрощает реализацию обнаружения сервисов на основе DNS, помогая избавиться от привязки к проприетарным API обнаружения сервисов.

Nacos предоставляет простые DNS API для управления доменными именами, связанными с сервисами, и списками доступных IP:PORT.

### Управление сервисами и метаданными

Nacos позволяет управлять всеми сервисами и метаданными с точки зрения создателя платформы микросервисов. Это включает управление описанием сервиса, жизненным циклом, анализом статических зависимостей, состоянием работоспособности, управлением трафиком, политиками маршрутизации и безопасности, SLA и ключевыми метриками.

### Управление и преобразование MCP сервисов

Nacos позволяет быстро преобразовывать существующие микросервисные API в MCP сервисы с помощью простых объявлений без изменения кода. Кроме того, используя Spring AI Alibaba, nacos-mcp-wrapper-python и Nacos-MCP-Router, MCP сервисы могут автоматически регистрироваться и обнаруживаться, позволяя использовать MCP сервисы как микросервисы. С точки зрения платформы MCP сервисов Nacos также управляет описаниями MCP сервисов, инструментами, промптами и управлением версиями.

## Преимущества продукта

### Простота использования

Nacos был усовершенствован на основе отзывов десятков тысяч пользователей, обеспечивая унифицированные функции обнаружения сервисов и управления конфигурацией. Благодаря интуитивному веб-интерфейсу и лаконичным API он предоставляет удобные операции регистрации, обнаружения и обновления конфигурации для разработчиков и эксплуатационного персонала в облачных средах. Как комплексное решение для динамического управления конфигурацией и обнаружения сервисов для приложений ИИ и облачных технологий, оно предлагает более двадцати готовых к использованию функций архитектуры, ориентированных на ИИ-сервисы, с легким и удобным интерфейсом управления для быстрого создания ИИ-приложений.

### Богатая функциональность

Nacos предоставляет полный набор функций, включая обнаружение сервисов, управление конфигурацией, динамическую службу DNS, управление метаданными сервисов, управление трафиком, мониторинг сервисов и управление сервисами. Он помогает вам легче создавать, доставлять и управлять микросервисами в эпоху облачных технологий. Он поддерживает сценарии реестра ИИ, включая Prompt, MCP и A2A. При прямой поддержке или простых расширениях он может поддерживать множество полезных сценариев интернет-приложений и приложений ИИ для планирования трафика и управления сервисами, включая ограничение скорости, планирование промо-акций и многорегиональное развертывание.

### Экстремальная производительность

Nacos, закаленный в условиях экстремального масштабирования во время распродажи Alibaba Double 11, обеспечивает высокопроизводительную регистрацию и обнаружение сервисов, а также низкую задержку при обновлении конфигурации, обеспечивая эффективную и стабильную работу в крупномасштабных распределенных системах.

### Масштаб

Nacos родился в масштабе миллионов экземпляров Alibaba и поддерживает управление огромным количеством сервисов и конфигураций, удовлетворяя потребности крупных распределенных систем в высокой параллельности и доступности.

### Стабильность и надежность

Производный от внутреннего продукта Alibaba с более чем десятилетней проверкой в производстве, он поддерживает крупномасштабные сценарии с миллионами сервисов и является продуктом с открытым исходным кодом корпоративного уровня с SLA, используемым десятками миллионов пользователей. Nacos использует самостоятельно разработанные протоколы синхронизации в сочетании с широко используемым в экосистеме протоколом Raft для обеспечения высокой доступности сервисов и стабильности данных, гарантируя высокую доступность системы Alibaba Double 11.

### Открытая экосистема

Nacos имеет активное сообщество с открытым исходным кодом, широкую интеграцию экосистемы и постоянные инновации. Он не только полностью совместим с популярными платформами с открытым исходным кодом, такими как Spring Cloud и Dubbo, но и предоставляет богатые возможности подключаемых модулей, помогая пользователям настраивать свои уникальные облачные микросервисные системы в эпоху облачных технологий. Он легко интегрируется с Kubernetes, Spring Cloud, Spring AI и Dify, упрощая развертывание и эксплуатацию на основных публичных облаках (таких как Alibaba Cloud, Huawei Cloud, Tencent Cloud, Azure и AWS) с поддержкой нескольких арендаторов и сред.

## Философия проектирования

Nacos стремится помочь пользователям в эпоху ИИ в частных, гибридных или публичных облачных средах лучше создавать, доставлять и управлять своими платформами управления ИИ, быстрее повторно использовать и комбинировать бизнес-услуги и быстрее предоставлять коммерческую инновационную ценность. Философия проектирования Nacos определена следующим образом:

### Простота использования

Простота использования — это основная философия Nacos. Он упрощает регистрацию сервисов, обнаружение и управление конфигурацией через удобный веб-интерфейс и лаконичные API. Разработчики могут легко интегрировать Nacos в свои приложения, не затрачивая много времени на сложные настройки и обучение.

### Ориентация на стандарты

Nacos следует философии проектирования, ориентированной на стандарты, придерживаясь лучших практик и стандартных протоколов разработки облачных приложений, чтобы гарантировать бесшовную интеграцию функций обнаружения сервисов и управления конфигурацией с широким спектром технологических стеков и платформ.

### Высокая доступность

Для удовлетворения требований корпоративных приложений к высокой доступности Nacos реализует кластерный режим, гарантируя, что функции обнаружения сервисов и управления конфигурацией не будут затронуты при сбоях узлов. Кластерный режим также означает, что Nacos может масштабироваться горизонтально путем добавления узлов, улучшая общую производительность системы и пропускную способность.

### Простота расширения

Nacos также уделяет внимание простоте расширения, используя модульную конструкцию, позволяющую каждому компоненту независимо расширяться или заменяться. Это также обеспечивает удобство для участников сообщества, позволяя им разрабатывать новые функции или улучшать существующие для конкретных потребностей, что еще больше способствует развитию экосистемы Nacos.

Благодаря реализации этих принципов проектирования Nacos предоставляет пользователям мощную и гибкую платформу для поддержки постоянно меняющихся бизнес-потребностей, ускорения бизнес-инноваций и цифровой трансформации, в конечном счете помогая пользователям завоевать конкурентное преимущество на рынке.

## Режимы развертывания

### Автономный режим

Автономный режим, также известный как режим одного экземпляра, обладает всеми функциями и характеристиками Nacos с чрезвычайно простым развертыванием и быстрым запуском. Однако он не может формировать кластер с другими узлами и не может обеспечить высокую доступность при сбоях узлов или сети. Автономный режим может использовать встроенную базу данных Derby (по умолчанию) или внешнюю базу данных для хранения. Автономный режим в основном подходит для инженеров, создающих среды Nacos локально или в тестовых средах для разработки, отладки и тестирования. Он также может обрабатывать некоторые бизнес-сценарии с низкими требованиями к стабильности и доступности.

### Кластерный режим

Кластерный режим создает высокодоступный кластер Nacos, объединяя несколько узлов Nacos через самостоятельно разработанный протокол Distro и протокол Raft. Данные синхронизируются между всеми узлами кластера для обеспечения согласованности данных. Кластерный режим обладает преимуществами высокой доступности, высокой масштабируемости и высокой параллельности, гарантируя, что бизнес-операции не будут затронуты при сбоях. Кластерный режим по умолчанию использует внешнюю базу данных для хранения, но также может использовать встроенную базу данных. Этот режим в основном подходит для производственных сред и является наиболее рекомендуемым сообществом способом развертывания.

Nacos подходит для различных сценариев применения:

- Управление конфигурацией в распределенных системах
