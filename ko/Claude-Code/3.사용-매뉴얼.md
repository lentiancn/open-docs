# Claude Code 사용 매뉴얼

이 매뉴얼은 Claude Code를 빠르게 시작하고 일상적인 개발 작업을 위한 조작법을 알려줍니다.

## 사전 준비

시작하기 전에 다음이 있는지 확인하세요:

- 열린 터미널 또는 명령 프롬프트
- 작업할 코드 프로젝트
- Claude 구독 (Pro, Max, Teams 또는 Enterprise), Claude Console 계정, 또는 지원되는 클라우드 제공자를 통한 접근

## 세션 시작

프로젝트 디렉토리에서 터미널을 열고 Claude Code를 시작합니다:

```bash
cd /path/to/your/project
claude
```

첫 사용 시 세션 정보, 최근 대화, 최신 업데이트를 보여주는 시작 화면이 표시됩니다. 사용 가능한 명령은 `/help`를 입력하거나 이전 대화를 계속하려면 `/resume`을 입력하세요.

## 기본 작업

### 코드베이스 이해

Claude에게 프로젝트를 분석하게 합니다:

```
이 프로젝트는 무엇을 하나요?
```

```
이 프로젝트는 어떤 기술을 사용하나요?
```

```
메인 진입점은 어디에 있나요?
```

```
폴더 구조를 설명해줘
```

Claude에게 기능에 대해 물어볼 수도 있습니다:

```
Claude Code는 무엇을 할 수 있나요?
```

```
Claude Code에서 사용자 정의 스킬은 어떻게 만드나요?
```

**참고: Claude Code는 필요할 때 프로젝트 파일을 읽습니다. 수동으로 컨텍스트를 추가할 필요가 없습니다.**

### 첫 번째 코드 변경

Claude Code에게 실제 코딩 작업을 하게 합니다. 간단한 작업을 시도해 보세요:

```
메인 파일에 hello world 함수를 추가해줘
```

Claude Code는:
1. 적절한 파일을 찾음
2. 제안된 변경 사항을 보여줌
3. 승인을 요청함
4. 편집을 수행함

**참고: Claude Code는 항상 파일을 수정하기 전에 권한을 요청합니다. 개별 변경 사항을 승인하거나 세션 전체에 "모두 수락" 모드를 활성화할 수 있습니다.**

### Git 작업

Claude Code는 Git 작업을 대화형으로 수행합니다:

```
내가 변경한 파일은 뭐가 있나요?
```

```
변경 사항을 설명적인 메시지와 함께 커밋해줘
```

```
feature/quickstart라는 새 브랜치를 만들어줘
```

```
마지막 5개의 커밋을 보여줘
```

```
병합 충돌 해결을 도와줘
```

### 버그 수정 및 기능 추가

Claude는 디버깅과 기능 구현에 능합니다.

원하는 내용을 자연어로 설명하세요:

```
사용자 등록 폼에 입력 검증 기능을 추가해줘
```

또는 기존 문제를 수정하세요:

```
사용자가 빈 폼을 제출할 수 있는 버그가 있어 - 수정해줘
```

Claude Code는:
- 관련 코드를 찾음
- 컨텍스트를 이해함
- 솔루션을 구현함
- 사용 가능한 경우 테스트를 실행함

### 일반적인 워크플로

**코드 리팩토링**

```
인증 모듈을 콜백 대신 async/await를 사용하도록 리팩토링해줘
```

**테스트 작성**

```
계산기 함수에 대한 단위 테스트를 작성해줘
```

**문서 업데이트**

```
README에 설치 안내를 추가해줘
```

**코드 리뷰**

```
내 변경 사항을 리뷰하고 개선점을 제안해줘
```

## 필수 명령어

매일 사용되는 가장 중요한 명령어입니다:

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `claude` | 대화형 모드 시작 | `claude` |
| `claude "task"` | 일회성 작업 실행 | `claude "build 오류 수정"` |
| `claude -p "query"` | 일회성 쿼리 실행 후 종료 | `claude -p "이 함수 설명해줘"` |
| `claude -c` | 현재 디렉토리에서 가장 최근 대화를 계속함 | `claude -c` |
| `claude -r` | 이전 대화 다시 시작 | `claude -r` |
| `claude commit` | Git 커밋 생성 | `claude commit` |
| `/clear` | 대화 기록 지우기 | `/clear` |
| `/help` | 사용 가능한 명령어 표시 | `/help` |
| `exit` 또는 Ctrl+C | Claude Code 종료 | `exit` |

## 고급 팁

### CLAUDE.md 사용

프로젝트 루트에 `CLAUDE.md` 파일을 만들어 Claude Code에 지속 가능한 프로젝트 컨텍스트를 제공합니다. 이 파일은 모든 세션 시작 시 자동으로 로드됩니다.

CLAUDE.md에 포함할 수 있는 내용:
- 코딩 표준 및 사양
- 아키텍처 결정
- 선호 라이브러리
- 검토 체크리스트
- 프로젝트별 명령어 및 스크립트

### 사용자 정의 스킬

`/review-pr` 또는 `/deploy-staging`과 같은 반복 가능한 워크플로를 캡슐화하기 위한 사용자 정의 명령을 만듭니다.

### Hooks 사용

Hook을 사용하면 Claude Code 작업 전후에 셸 명령을 실행할 수 있습니다:
- 모든 파일 편집 후 자동 포맷
- 커밋 전 린트 실행

### MCP 서버

Model Context Protocol을 통해 Claude Code를 외부 도구 및 데이터 소스에 연결:
- Google Drive에서 설계 문서 읽기
- Jira 티켓 업데이트
- Slack에서 데이터 가져오기
- 사용자 정의 도구 사용

### 멀티 에이전트 협업

복잡한 작업의 경우 여러 Claude Code 에이전트를 병렬로 작업하게 합니다. 리드 에이전트가 작업을 조정하고, 하위 작업을 할당하고, 결과를 병합합니다.

### 원격 제어

작업이 단일 장치에되지 않습니다:
- 원격 제어를 사용하여 전화기나 다른 장치에서 로컬 세션을 계속합니다
- 웹 또는 iOS에서작업을 시작한 다음 `/teleport`로 터미널로 다시 가져옵니다
- `/desktop`을 사용하여 시각적 diff 검토를 위해 터미널 세션을 데스크톱 앱으로 전환합니다

### 단계별 지침 사용

복잡한 작업을 단계로 나눕니다:

```
1. 사용자 프로필을 위한 새 데이터베이스 테이블 생성
2. 사용자 프로필을 가져오고 업데이트하는 API 엔드포인트 생성
3. 사용자가 자신의 정보를 보고 편집할 수 있는 웹페이지 구축
```

### 먼저 Claude에게 탐색하게 하세요

변경하기 전에 Claude에게 코드를 이해하게 합니다:

```
데이터베이스 스키마 분석해줘
```

```
우리 영국 고객이 가장 자주 반품하는 제품을 보여주는 대시보드 만들어줘
```

### 단축키로 시간 절약

- `?`를 눌러 사용 가능한 모든 키보드 단축키 확인
- 탭으로 명령 자동 완성 사용
- `↑`로 명령 기록 보기
- `/`를 입력하여 모든 명령 및 스킬 확인

## 자주 묻는 질문

**Claude Code가 내 코드를 읽나요?**

네, Claude Code는 코드베이스를 이해하기 위해 프로젝트 파일을 읽습니다. (서드파티 API 제공자를 사용하지 않는 한) 코드를 외부 서버로 보내지 않습니다.

**Claude Code가 자동으로 내 파일을 수정하나요?**

아니요. Claude Code는 항상 파일을 수정하기 전에 권한을 요청합니다. 개별 변경 사항을 승인하거나 전체 세션에 "모두 수락" 모드를 활성화할 수 있습니다.

**내 코드가 안전한가요?**

Claude Code는 보안을 고려하여 설계되었습니다. 코드는 Anthropic의 모델 학습을 위해 전송되지 않습니다 (명시적으로 옵트인하지 않는 한). [Anthropic 개인정보 보호정책](https://www.anthropic.com/privacy)에서 자세히 알아보세요.
