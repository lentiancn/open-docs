# OpenFeign Benutzerhandbuch

## Lastverteilung

### Spring Cloud LoadBalancer-Integration

OpenFeign integriert standardmäßig mit Spring Cloud LoadBalancer und bietet Client-seitige Lastverteilungsfunktionalität.

```java
@FeignClient(name = "user-service")
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}
```

Konfigurieren Sie Service-Instanzen in `application.yml`:

```yaml
spring:
  cloud:
    discovery:
      client:
        simple:
          instances:
            user-service:
              - uri: http://localhost:8081
              - uri: http://localhost:8082
              - uri: http://localhost:8083
```

### Benutzerdefinierte Lastverteilungsstrategie

```java
@Configuration
public class LoadBalancerConfig {
    
    @Bean
    public ServiceInstanceListSupplier serviceInstanceListSupplier(
            ConfigurableApplicationContext context) {
        return ServiceInstanceListSupplier.builder()
                .withDiscoveryClient()
                .withCaching()
                .build(context);
    }
}
```

### Transformieren der lastverteilten HTTP-Anfrage

```java
@Bean
public LoadBalancerFeignRequestTransformer transformer() {
    return new LoadBalancerFeignRequestTransformer() {
        @Override
        public Request transformRequest(Request request, ServiceInstance instance) {
            Map<String, Collection<String>> headers = new HashMap<>(request.headers());
            headers.put("X-ServiceId", Collections.singletonList(instance.getServiceId()));
            headers.put("X-InstanceId", Collections.singletonList(instance.getInstanceId()));
            return Request.create(request.httpMethod(), request.url(), headers, 
                request.body(), request.charset(), request.requestTemplate());
        }
    };
}
```

## Circuit Breaker-Unterstützung

### Spring Cloud Circuit Breaker-Integration

Aktivieren Sie die Circuit Breaker-Unterstützung:

```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

### Circuit Breaker-Fallback

```java
@FeignClient(name = "user-service", fallback = UserServiceFallback.class)
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}

@Component
public class UserServiceFallback implements UserService {
    @Override
    public User getUser(Long id) {
        // Standardbenutzer zurückgeben oder spezifische Ausnahme auslösen
        return new User(id, "Default User");
    }
}
```

### Fallback-Factory mit Ausnahmeinformationen

```java
@FeignClient(name = "user-service", fallbackFactory = UserServiceFallbackFactory.class)
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}

@Component
public class UserServiceFallbackFactory implements FallbackFactory<UserService> {
    @Override
    public UserService create(Throwable cause) {
        return new UserService() {
            @Override
            public User getUser(Long id) {
                // Kann auf die Ausnahme zugreifen, die den Fallback ausgelöst hat
                log.error("UserService fallback triggered", cause);
                return new User(id, "Fallback User");
            }
        };
    }
}
```

### Benutzerdefinierter Circuit Breaker-Name

```java
@Configuration
public class CircuitBreakerConfig {
    @Bean
    public CircuitBreakerNameResolver circuitBreakerNameResolver() {
        return (String feignClientName, Target<?> target, Method method) -> 
            feignClientName + "_" + method.getName();
    }
}
```

## Protokollierung

### Protokollebene konfigurieren

```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Oder über Konfigurationsdatei:

```yaml
logging:
  level:
    com.example.clients.UserService: DEBUG

spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
```

### Benutzerdefiniertes Protokollfiltering

```java
public class CustomLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        // Vertrauliche Informationen filtern
        if (!shouldLogRequestHeader(request)) {
            // Benutzerdefinierte Protokollierungslogik
        }
    }
    
    @Override
    protected Response logAndRebufferResponse(String configKey, Level logLevel, 
            Response response, long elapsedTime) throws IOException {
        // Vertrauliche Antwortinformationen filtern
        if (!shouldLogResponseHeader(response)) {
            // Benutzerdefinierte Protokollierungslogik
        }
        return super.logAndRebufferResponse(configKey, logLevel, response, elapsedTime);
    }
    
    private boolean shouldLogRequestHeader(Request request) {
        // Authorization-Header und andere vertrauliche Informationen filtern
        return !request.headers().containsKey("Authorization");
    }
    
    private boolean shouldLogResponseHeader(Response response) {
        return true; // Oder bei Bedarf filtern
    }
}
```

## Anfrage/Antwort-Komprimierung

### Komprimierung aktivieren

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

### Wichtige Hinweise

- Bei Verwendung von OkHttpClient wird die Komprimierung deaktiviert, wenn `content-encoding`- oder `accept-encoding`-Header vorhanden sind
- Die Komprimierung gilt nur für angegebene MIME-Typen
- Die Anfragengröße muss `min-request-size` überschreiten, um komprimiert zu werden

## Caching-Unterstützung

### Caching aktivieren

```java
@SpringBootApplication
@EnableCaching
@EnableFeignClients
public class Application {
    // ...
}
```

### Caching in Feign-Client verwenden

```java
@FeignClient(name = "data-service")
public interface DataService {
    
    @GetMapping("/data/{key}")
    @Cacheable(cacheNames = "data-cache", key = "#key")
    Data getData(@PathVariable String key);
    
    @PostMapping("/data")
    @CachePut(cacheNames = "data-cache", key = "#data.key")
    Data updateData(@RequestBody Data data);
    
    @DeleteMapping("/data/{key}")
    @CacheEvict(cacheNames = "data-cache", key = "#key")
    void deleteData(@PathVariable String key);
}
```

### Caching deaktivieren

```yaml
spring:
  cloud:
    openfeign:
      cache:
        enabled: false
```

## Asynchrone Unterstützung

### CompletableFuture-Unterstützung

```java
@FeignClient(name = "async-service")
public interface AsyncService {
    
    @GetMapping("/data/{id}")
    CompletableFuture<Data> getDataAsync(@PathVariable String id);
}
```

### Asynchronen Client konfigurieren

```java
@Configuration
public class AsyncFeignConfig {
    
    @Bean
    public AsyncClient asyncClient() {
        return new AsyncClient.Default();
    }
}
```

### Verwendungsbeispiel

```java
@Service
public class AsyncProcessingService {
    
    private final AsyncService asyncService;
    
    public AsyncProcessingService(AsyncService asyncService) {
        this.asyncService = asyncService;
    }
    
    public void processData() {
        CompletableFuture<Data> future1 = asyncService.getDataAsync("1");
        CompletableFuture<Data> future2 = asyncService.getDataAsync("2");
        
        // Parallel verarbeiten
        CompletableFuture.allOf(future1, future2).thenRun(() -> {
            try {
                Data data1 = future1.get();
                Data data2 = future2.get();
                // Daten verarbeiten
            } catch (Exception e) {
                log.error("Error processing async data", e);
            }
        });
    }
}
```

## Vererbungsunterstützung

### Basisschnittstellendefinition

```java
public interface BaseService<T, ID> {
    @GetMapping("/{id}")
    T findById(@PathVariable ID id);
    
    @GetMapping
    List<T> findAll();
    
    @PostMapping
    T save(@RequestBody T entity);
    
    @DeleteMapping("/{id}")
    void deleteById(@PathVariable ID id);
}
```

```java
@FeignClient(name = "user-service")

@FeignClient(name = "order-service")

```

## Refresh-Scope-Unterstützung

### Refresh-Unterstützung aktivieren

```yaml
spring:
  cloud:
    openfeign:
      client:
        refresh-enabled: true
```

### Konfiguration aktualisieren

Aktualisieren Sie über den Actuator-Endpunkt:

```bash
POST /actuator/refresh
```

### Wichtige Hinweise

- Annotieren Sie die `@FeignClient`-Schnittstelle nicht mit `@RefreshScope`
- Die Refresh-Funktionalität ist standardmäßig deaktiviert und muss explizit aktiviert werden
- Unterstützt das Aktualisieren von Verbindungs-Timeout, Lese-Timeout und URL-Konfiguration

## AOT- und Native-Image-Unterstützung

### Konfigurationsanforderungen

```yaml
spring:
  cloud:
    refresh:
      enabled: false
    openfeign:
      client:
        refresh-enabled: false
      lazy-attributes-resolution: false
```

### Native Image erstellen

Stellen Sie während des Builds die korrekte Konfiguration sicher und überschreiben Sie die URL zur Laufzeit über Systemeigenschaften:

```bash
-Dspring.cloud.openfeign.client.config.[clientId].url=[url]
```

Diese erweiterten Funktionen machen OpenFeign nicht nur zu einem einfachen HTTP-Client, sondern zu einer umfassenden Microservice-Kommunikationslösung, die verschiedene Anforderungen an Unternehmensanwendungen erfüllen kann.
