# OpenClaw 使用手册

本手册介绍 OpenClaw 的日常使用方法和核心功能。

## 基本概念

### Gateway

Gateway 是 OpenClaw 的核心组件，负责会话管理、消息路由和通道连接。你只需要运行一个 Gateway 进程，它会自动处理所有通道的接入请求。

### 通道（Channels）

通道是指你用来与 AI 助手交流的聊天应用。OpenClaw 支持同时连接多个通道，包括 WhatsApp、Telegram、Discord、iMessage 等。

### 代理（Agents）

代理是 AI 助手的实例。每个代理有独立的工作区和会话。你可以配置多个代理，分别处理不同的工作任务。

## 日常操作

### 启动和停止 Gateway

```bash
# 启动 Gateway（后台服务）
openclaw gateway start

# 停止 Gateway
openclaw gateway stop

# 重启 Gateway
openclaw gateway restart

# 查看状态
openclaw gateway status

# 前台运行（用于调试）
openclaw gateway --port 18789
```

### 发送消息

```bash
# 发送私信
openclaw message send --target +15555550123 --message "你好"

# 发送群组消息
openclaw message send --target "群组名称" --message "大家好"
```

### 管理配置

```bash
# 查看当前配置
openclaw config get

# 设置配置项
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# 删除配置项
openclaw config unset channels.whatsapp.enabled
```

## 配置管理

### 配置文件位置

配置文件位于 `~/.openclaw/openclaw.json`。

### 热重载

OpenClaw 支持配置热重载，修改配置文件后无需手动重启 Gateway。大多数配置更改会自动生效。

### 常用配置示例

#### 设置允许的消息来源

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### 设置群组聊天提及规则

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### 配置多个代理

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## 通道管理

### 添加新通道

```bash
# 登录 WhatsApp
openclaw channels login whatsapp

# 登录 Telegram
openclaw channels login telegram
```

### 查看已连接的通道

```bash
openclaw channels list
```

## 高级功能

### 定时任务（Cron Jobs）

配置定时执行的任务：

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### 心跳检测（Heartbeat）

设置定期检查：

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### 沙箱模式

在隔离的 Docker 容器中运行代理：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

设置 HTTP Webhook 端点：

```json5
{
  hooks: {
    enabled: true,
    token: "your-secret-token",
    path: "/hooks"
  }
}
```

## 故障排查

### 查看日志

```bash
# 查看 Gateway 日志
openclaw logs

# 查看实时日志
openclaw logs -f
```

### 健康检查

```bash
openclaw health
```

### 诊断问题

```bash
openclaw doctor
```

## 控制界面

OpenClaw 提供 Web 控制界面，你可以：

- 发送和接收消息
- 管理配置
- 查看会话历史
- 管理移动节点
- 监控 Gateway 状态

访问地址：`http://127.0.0.1:18789/`
