# Kotlin インストールガイド

## 前提条件

Kotlin のインストールを開始する前に、コンピュータが基本的なシステム要件を満たしていることを確認してください。Kotlin は、Windows、macOS、Linux 以及 Unix 系オペレーティングシステムを含む複数のオペレーティングシステムで実行できます。使用するオペレーティングシステムに関係なく、開発ツールのインストールとコンパイル済みプログラムの実行十分なディスク容量があることを確認してください。

Windows ユーザーの場合は、最適なエクスペリエンスのために Windows 10 以降を使用することをお勧めします。macOS ユーザーは macOS 10.14 以降が必要です。Linux ユーザーは Ubuntu、Fedora、Debian などの任意のメジャーディストリビューションを使用できます。

## IDE 経由のインストール

### IntelliJ IDEA

IntelliJ IDEA は JetBrains 製のフラッグシップ統合開発環境で、Kotlin 最も完全なサポートを提供しています。主な目標が Kotlin 開発の場合、IntelliJ IDEA が最適な選択です。

IntelliJ IDEA には2つのバージョンがあります：コミュニティ版（Community）とультimas版（Ultimate）。コミュニティ版は無料でオープンソースであり、Kotlin 開発には十分な機能を提供します。Ultimate 版は、高度などの追加機能を持つエンタープライズ向用户提供します。

インストール手順：首先、JetBrains 公式サイト（https://www.jetbrains.com/idea/download/）にアクセスし、オペレーティングシステム对应的なインストーラーをダウンロードします。ダウンロード完成后、インストールウィザードを実行してインストールを完了します。インストール中に、インストールパスとデスクトップショートカットの選択できます。

インストール完成后、IntelliJ IDEA を最初に起動すると、IDE が自動的に検出し、Kotlin プラグインのインストールを求めるプロンプトが表示されます。システムが自動的に検出しない場合は、プラグイン市場で Kotlin を検索して手動でインストールできます。インストール完成后、IDE を再起動すると Kotlin 開発を開始できます。

### Android Studio

Android Studio は、Google 公式の Android 開発 IDE で、組み込みの Kotlin サポートを提供します。主に Android 開発を行う場合、Android Studio が理想的な選択です。

Android 開発者公式サイト（https://developer.android.com/studio）にアクセスして Android Studio をダウンロードします。ダウンロード完成后、インストールウィザードを実行してインストールを完了します。Android Studio にはデフォルトで Kotlin プラグインが含まれているため、追加の構成なしで開発を開始できます。

Android Studio は每年、複数のアップデートバージョンをリリースし、各バージョンには最新の Kotlin サポートと Android 開発ツールが含まれています。最高の開発体験と最新の言語機能を得るために、バージョンを更新し続けることをお勧めします。

## コマンドラインからのインストール

IDE を使用する他还，还有 Kotlin コマンドラインコンパイラをインストールする选项もあります。この方法は、轻量级开发環境を好む人やサーバーで Kotlin 開発が必要な人に适しています。

### 手動インストール

まず、Kotlin GitHub Releases ページ（https://github.com/JetBrains/kotlin/releases）にアクセスして、最新版のコンパイラをダウンロードします。ダウンロード完成后、zip アーカイブを 원하는ディレクトリに解凍します。

解凍後、kotlinc/bin ディレクトリをシステムの PATH 環境変数に追加すると、コマンドラインで kotlinc コマンドを使用できます。Windows の場合は、システムプロパティの環境変数からパスを追加できます。macOS と Linux の場合は、~/.bashrc または ~/.zshrc ファイルを編集して、以下のような行を追加します：

export PATH="$PATH:/path/to/kotlinc/bin"

パス добавление 后、新しいターミナルウィンドウを開いて kotlinc -version コマンドを入力し、バージョン情報が表示されればインストール成功です。

### SDKMAN! を使用したインストール

SDKMAN! は、Java 開発ツールのバージョンを管理するためのツールで、Unix 系システムで非常に便利です。macOS、Linux、Cygwin、FreeBSD、または Solaris を使用している場合、SDKMAN! が Kotlin をインストールする推奨方法です。

まず、SDKMAN! をインストールします。ターミナルで次のコマンドを実行します：

curl -s "https://get.sdkman.io" | bash

インストール完成后、環境構成を再読み込みします：

source ~/.sdkman/bin/sdkman-init.sh

これで sdk コマンドを使用して Kotlin をインストールできます：

sdk install kotlin

インストール过程中で確認を求めるプロンプトが表示されます。确认を入力するとインストールが完了します。インストール完成后、同様に kotlinc -version でインストールを確認できます。

### Homebrew を使用したインストール

macOS では、Homebrew パッケージマネージャーを使用して Kotlin をインストールすることもできます。まず Homebrew がインストールされていることを確認してから、次のコマンドを実行します：

brew update
brew install kotlin

Homebrew は自動的に最新版の Kotlin コンパイラをダウンロードしてインストールします。インストール完成后、同様に kotlinc -version でインストールを確認できます。

### Snap を使用したインストール

Ubuntu 16.04 以降では、 Snap パッケージマネージャーを使用して Kotlin をインストールできます：

sudo snap install --classic kotlin

Snap インストール方式是将 Kotlin 安装为系统级应用，任何场所からコマンドラインで呼び出すことができます。

## 最初の Kotlin プログラムの作成

安装方法が何であっても、以下の步骤で最初の Kotlin プログラムを作成して実行できます。

まず、次のような内容の hello.kt というファイルを作成します：

fun main() {
    println("Hello, World!")
}

これは最も簡単な Kotlin プログラムで、main 関数を定義しています。これは Kotlin プログラムのエントリポイントです。println 関数はコンソールにテキストを出力します。

コマンドラインコンパイラを使用する場合、このプログラムのコンパイルと実行には2つのステップが必要です。最初はコンパイルです：

kotlinc hello.kt -include-runtime -d hello.jar

这里的 -include-runtime パラメータは Kotlin ランタイムライブラリを生成された jar ファイルにパックし、jar ファイルを独立して実行可能にします。-d パラメータは出力ファイルのパスを指定します。

コンパイル完成后、プログラムを実行します：

java -jar hello.jar

コンソールに "Hello, World!" が出力されるはずです。

スタンドアロンの jar ファイルを生成したくない場合は、kotlinc のインタラクティブ実行機能を使用することもできます：

kotlinc -script hello.kt

または kotlin コマンドを使用してコンパイル済みクラスファイルを直接実行することもできます：

kotlin -classpath hello.jar HelloKt

 здесь HelloKt は hello.kt ファイルの Kotlin コンパイラが生成したメインクラス名です。

## REPL の使用

Kotlin はインタラクティブな REPL（Read-Eval-Print Loop）環境を提供しており、Kotlin コードを入力してすぐに結果を見ることができます。これは、言語機能の学習やコードスニペットの素早いテストに便利です。

REPL を開始するには、コマンドラインで単に kotlinc と入力します：

パラメータなしで kotlinc を実行すると REPL モードに入ります。このモードでは、有効な Kotlin コードを入力して Enter を押すと結果を見ることができます。例えば：

>>> val x = 10
>>> val y = 20
>>> x + y
30

:quit を入力するか Ctrl+C を押して REPL を終了します。

## Kotlin スクリプト

Kotlin はスクリプト言語としても使用できます。Kotlin スクリプトファイルは .kts 拡張子を持ち、ファイル内のトップレベルコードは実行可能コードとして扱われます。

例えば、list_folders.kts というスクリプトを作成します：

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

このスクリプトを実行します：

kotlinc -script list_folders.kts -- -d /path/to/folder

## ビルドツールの構成

実際のプロジェクトでは、通常、Gradle または Maven をビルドツールとして使用します。これらのツールは両方とも Kotlin プロジェクトをネイティブにサポートします。

### Gradle

Gradle は Kotlin 開発で最も人気のあるビルドツールです。Kotlin チームも Kotlin DSL を提供しており、ビルドスクリプトを Kotlin 構文で記述できます。

基本的な Kotlin Gradle プロジェクトを作成するには、まずプロジェクトに Kotlin プラグインを追加します。プロジェクトの build.gradle ファイルには以下の内容が含まれている必要があります：

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

または Kotlin DSL を使用：

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

次に、ソースコードディレクトリ構造を作成します，通常は src/main/kotlin です。その中に Kotlin ソースファイルを作成し、gradle build コマンドを使用してプロジェクトをコンパイルします。

### Maven

Maven は Another 人気の Java プロジェクトビルドツールで，同样支持 Kotlin プロジェクト。pom.xml ファイルに Kotlin 相关の依存関係を追加します。

## インストールの確認

インストール完成后、すべてが正常に動作していることを確認するために、いくつか簡単なコマンドを使用できます。

まず、Kotlin コンパイラを確認します：

kotlinc -version

「Kotlin compiler version 2.3.10」类似的出力されるはずです。

次に、Kotlin 標準ライブラリを確認します：

kotlin -e 'println("Hello Kotlin!")'

すべて正常に動作すれば、インストールは成功で、Kotlin 之旅を始めることができます。

## 次のステップ

Kotlin のインストール完成后、以下の学習パスに進むことをお勧めします：首先、Kotlin 公式 Tour を読んで言語の基本特性を学びます。次に、Android 開発、サーバーサイド開発、またはクロスプラットフォーム開発など、感兴趣の開発方向に焦点を当てて深く学びます。最後に、実際のプロジェクトで練習し、学んだ知識を実践に適用します。

Kotlin 公式は、公式ドキュメント、オンラインチュートリアル、ビデオコースなど、丰富的な学習リソースを提供しています。 这些リソースへのリンクは Kotlin ウェブサイトで見つけることができます。
