# Manuel Utilisateur Go

## Syntaxe de Base

### Déclaration de Variables

Go a plusieurs façons de déclarer des variables. La déclaration de base utilise le mot-clé var:

```go
// Déclarer une variable avec le type
var name string = "Go"

// Déclaration avec inférence de type
var age = 10

// Déclaration par lot
var (
    firstName string
    lastName  string
    score     float64
)

// Déclaration courte (dans une fonction)
message := "Hello, World!"
```

Les variables Go doivent être utilisées après déclaration, sinon il y aura une erreur de compilation. Si vous devez déclarer une variable temporairement non utilisée, vous pouvez utiliser le tiret bas _ pour la recevoir:

```go
result, _ := someFunction()  // Ignorer la première valeur de retour
```

### Déclaration de Constantes

Les constantes sont déclarées avec le mot-clé const et peuvent être des caractères, des chaînes, des booléens ou des nombres:

```go
const Pi = 3.14159
const (
    StatusOK = 200
    StatusNotFound = 404
)
```

### Types de Données de Base

Les types de données de base de Go comprennent:

- Booléen: bool (true ou false)
- Entiers: int, int8, int16, int32, int64, et les types non signés correspondants uint, etc.
- Flottants: float32, float64
- Complexes: complex64, complex128
- Chaînes: string

La conversion de type doit être explicite:

```go
var i int = 42
var f float64 = float64(i)
var s string = strconv.Itoa(i)  // Entier vers chaîne
```

## Fonctions

### Définition de Fonctions

Les fonctions Go sont définies avec le mot-clé func:

```go
func greet(name string) string {
    return "Hello, " + name
}

// Valeurs de retour multiples
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// Valeurs de retour nommées
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return  // Pas besoin de spécifier les valeurs de retour
}
```

### Paramètres Variadiques

Les fonctions peuvent accepter un nombre variable de paramètres:

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

// Utilisation
result := sum(1, 2, 3, 4, 5)
```

### Fermetures

Go supporte les fermetures, où les fonctions peuvent capturer et modifier des variables de la portée externe:

```go
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

// Utilisation
c := counter()
fmt.Println(c())  // 1
fmt.Println(c())  // 2
```

## Programmation Concurrente

### Goroutines

Les goroutines sont des unités d'exécution légères dans Go avec un coût de création extrêmement bas:

```go
// Démarrer une goroutine
go func() {
    fmt.Println("Hello from goroutine!")
}()

// Goroutine avec paramètres
go func(msg string) {
    fmt.Println(msg)
}("Hello!")
```

### Canaux

Les canaux sont le mécanisme de communication entre goroutines:

```go
// Créer un canal
ch := make(chan string)

// Envoyer des données
ch <- "message"

// Recevoir des données
msg := <-ch

// Canal avec tampon
bufferedCh := make(chan int, 10)
```

### Select

L'instruction select est utilisée pour attendre plusieurs opérations de canal simultanément:

```go
select {
case msg := <-ch1:
    fmt.Println("Received:", msg)
case msg := <-ch2:
    fmt.Println("Received:", msg)
case <-time.After(time.Second):
    fmt.Println("Timeout")
}
```

### Sécurité de la Concurrence

Utilisez le package sync pour le contrôle de concurrence:

```go
var (
    mu    sync.Mutex
    count int
)

func increment() {
    mu.Lock()
    defer mu.Unlock()
    count++
}
```

## Gestion des Erreurs

### Retour d'Erreurs

Go utilise les valeurs de retour pour la gestion des erreurs, ce qui est la manière idiomatique en Go:

```go
result, err := doSomething()
if err != nil {
    return err
}
```

### Erreurs Personnalisées

Créez des erreurs personnalisées en utilisant le package errors:

```go
import "errors"

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}
```

### Wrapping d'Erreurs

Go 1.13 a introduit le wrapping d'erreurs en utilisant fmt.Errorf et le verbe %w:

```go
if err != nil {
    return fmt.Errorf("failed to do something: %w", err)
}
```

## Interfaces

### Définition d'Interfaces

Les interfaces Go sont implicitement implémentées, sans déclaration explicite:

```go
type Writer interface {
    Write([]byte) (int, error)
}

type Reader interface {
    Read([]byte) (int, error)
}

// Interface composite commune
type ReadWriter interface {
    Reader
    Writer
}
```

### Interface Vide

L'interface vide interface{} peut stocker des valeurs de tout type:

```go
var i interface{} = 42
var s interface{} = "hello"
var m interface{} = map[string]int{"a": 1}
```

Go 1.18 a introduit les génériques, permettant d'utiliser any au lieu de interface{}:

```go
var i any = 42
```

## Paquets et Modules

### Créer un Module

Utilisez go mod init pour créer un nouveau module:

```bash
go mod init github.com/username/project
```

### Gestion des Dépendances

Ajouter des dépendances:

```bash
go get github.com/pkg/errors
```

Mettre à jour les dépendances:

```bash
go get -u all
```

Nettoyer les dépendances inutilisées:

```bash
go mod tidy
```

## Tests

### Tests Unitaires

Go dispose d'un framework de test intégré. Les fichiers de test se terminent par _test.go:

```go
import "testing"

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    if result != 5 {
        t.Errorf("Add(2, 3) = %d; want 5", result)
    }
}
```

Exécuter les tests:

```bash
go test ./...
go test -v ./...
go test -cover ./...
```

### Tests de Performance

Les fonctions de test commencent par Benchmark:

```go
func BenchmarkHello(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Hello()
    }
}
```

Exécuter les tests de performance:

```bash
go test -bench=.
```

## Bibliothèque Standard Courante

### fmt - E/S Formatée

```go
fmt.Println("Hello")
fmt.Printf("Name: %s, Age: %d\n", name, age)
fmt.Sprintf("Result: %d", result)
```

### strings - Opérations sur Chaînes

```go
strings.Contains(s, "sub")
strings.Split(s, ",")
strings.TrimSpace(s)
strings.ToUpper(s)
```

### net/http - Serveur HTTP

```go
http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprint(w, "Hello, HTTP!")
})
http.ListenAndServe(":8080", nil)
```

### json - Traitement JSON

```go
// Encoder
data, _ := json.Marshal(person)

// Décoder
var p Person
json.Unmarshal(data, &p)
```

```bash
gofmt -w main.go  # Formater et écrire dans le fichier
gofmt -d main.go  # Afficher les différences de formatage
```

### go vet - Analyse Statique

```bash
go vet ./...  # Vérifier les problèmes de code
```

### go doc - Visionneur de Documentation

```go
go doc fmt.Println  # Voir la documentation de la fonction
go doc fmt          # Voir la documentation du package
```
