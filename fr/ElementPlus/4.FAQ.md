# FAQ Element Plus

Cette page liste les problèmes courants et leurs solutions lors de l'utilisation de Element Plus.

## Problèmes d'installation

### Échec d'installation

Symptômes: npm install ou yarn add échoue

Solutions:

1. Vérifiez la version de Node.js, elle doit être supérieure ou égale à 14.0.0
2. Videz le cache npm:
   npm cache clean --force
3. Supprimez node_modules et package-lock.json, puis réinstallez

### Erreurs de type TypeScript

Symptômes: Erreurs de type lors de l'importation de composants

Solutions:

1. Assurez-vous que le plugin Volar est installé
2. Ajoutez dans tsconfig.json:
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }

## Problèmes d'utilisation

### Styles de composants non appliqués

Symptômes: Les styles de composants ne correspondent pas aux attentes

Solutions:

1. Assurez-vous que le CSS est correctement importé:
   import 'element-plus/dist/index.css'
2. Vérifiez s'il y a des substitutions de styles globaux

### Icônes non affichées

Symptômes: Les composants d'icône s'affichent en blanc ou comme des carrés

Solutions:

1. Installez la bibliothèque d'icônes:
   npm install @element-plus/icons-vue

2. Enregistrez les icônes globalement:
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }

### Importation à la demande ne fonctionne pas

Symptômes: La taille du bundle reste grande

Solutions:

1. Vérifiez que la configuration du plugin est correcte
2. Assurez-vous d'utiliser le bon resolver

### Internationalisation ne fonctionne pas

Symptômes: Le texte du composant ne change pas après le changement de langue

Solutions:

1. Importez correctement le paquet linguistique:
   import fr from 'element-plus/es/locale/lang/fr'
2. Configurez le locale:
   app.use(ElementPlus, { locale: fr })

## Problèmes de composants

### Problèmes de tableau

#### Tri du tableau ne fonctionne pas

Assurez-vous que l'attribut prop est défini:

<el-table-column prop="date" sortable />

#### Colonnes fixes de tableau mal alignées

Assurez-vous que le tableau a une hauteur fixe:

<el-table :data="data" height="300" />

### Problèmes de formulaire

#### Validation de formulaire non déclenchée

Assurez-vous que el-form-item a l'attribut prop:

<el-form-item label="Nom" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>

### Problèmes de dialogue

#### Problèmes de défilement du dialogue

Lorsque le contenu du dialogue est trop long, assurez-vous d'ajouter le défilement:

<el-dialog>
  <div style="max-height: 60vh; overflow-y: auto;">
    Contenu
  </div>
</el-dialog>

## Problèmes de styles

### Conflits de styles

Symptômes: Les styles personnalisés sont substitués

Solutions:

1. Utilisez des sélecteurs plus spécifiques
2. Utilisez ::v-deep ou :deep()

### Couleur de thème non appliquée

Vérifiez si les variables CSS sont correctement substituées:

:root {
  --el-color-primary: #409eff;
}

Assurez-vous qu'il est chargé après les styles element-plus.

## Problèmes de performances

### Lag de page

Pour grandes quantités de données:

1. Utilisez el-table-v2 (table virtuelle)
2. Utilisez la pagination pour charger les données
3. Évitez les composants complexes dans les tableaux

### Taille du bundle trop grande

1. Utilisez l'importation à la demande
2. Supprimez les plugins non utilisés

## Problèmes de version Vue

### Non compatible avec Vue 2

Element Plus supporte uniquement Vue 3. Si vous avez besoin de Vue 2, utilisez Element UI.

### Composition API non supportée

Assurez-vous d'utiliser Vue 3.2 ou supérieur pour un meilleur support Composition API.
