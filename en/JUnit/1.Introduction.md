# JUnit Introduction

JUnit is the most popular unit testing framework for Java, used to write and run repeatable tests. It is the foundation for developing high-quality Java code and is widely used by millions of developers worldwide.

## JUnit 4 vs JUnit 5

### Version Overview

- **JUnit 4**: Released in 2010, the last major version of the testing framework used by the Java community for years
- **JUnit 5**: Released in 2017, a complete redesign of the JUnit framework, consisting of three sub-projects: JUnit Platform, JUnit Jupiter, and JUnit Vintage

### Architectural Differences

JUnit 4 uses a monolithic architecture with all features in a single library. JUnit 5 uses a modular architecture with three separate projects:

- **JUnit Platform**: The foundation for launching testing frameworks, providing the TestEngine API and console launcher
- **JUnit Jupiter**: The new programming model and extension model for writing tests and extensions
- **JUnit Vintage**: The compatibility engine that allows running JUnit 3 and JUnit 4 tests on the JUnit 5 platform

### Key Differences

| Feature | JUnit 4 | JUnit 5 |
|---------|---------|---------|
| Minimum Java Version | Java 5+ | Java 8+ |
| Annotation Package | org.junit | org.junit.jupiter.api |
| Test Class No Inheritance Required | No | Yes |
| Assertion Style | Assert static methods | Assertions static methods |
| Parameterized Tests | @Parameters | @ParameterizedTest |
| Nested Tests | Not supported | Supported |
| Dynamic Tests | Not supported | Supported |
| Parallel Execution | Limited support | Native support |
| Extension Model | @RunWith | @ExtendWith |

### Choosing a Version

- **New Projects**: Use JUnit 5 for modern features and better extensibility
- **Legacy Projects**: Continue using JUnit 4, or use JUnit Vintage for gradual migration
- **Long-term Maintenance**: Gradually migrate to JUnit 5 for better testing experience
