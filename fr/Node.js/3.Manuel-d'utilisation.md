# Manuel d'utilisation de Node.js

## Premiers pas

### Concepts de base

Node.js est construit sur le moteur JavaScript V8 de Chrome et fournit un modèle d'E/S orienté événements et non bloquant qui le rend léger et efficace pour les applications en temps réel intensives en données.

#### Boucle d'événements
La boucle d'événements de Node.js est le cœur de son modèle de concurrence. Contrairement aux modèles traditionnels basés sur les threads, Node.js utilise une boucle d'événements à thread unique qui gère les opérations asynchrones via des callbacks, des promesses ou async/await.

#### E/S non bloquantes
La plupart des opérations Node.js sont non bloquantes, ce qui signifie qu'elles n'arrêtent pas l'exécution du programme pendant qu'elles attendent la fin des opérations d'E/S. Au lieu de cela, elles utilisent des callbacks pour traiter les résultats lorsqu'ils sont prêts.

### Application Hello World

Créez votre première application Node.js avec ce serveur HTTP simple :

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Exécutez-la avec : `node server.js`

## Modules principaux

Node.js est livré avec un ensemble riche de modules intégrés qui fournissent des fonctionnalités essentielles sans nécessiter de dépendances externes.

### Système de fichiers (fs)
Gérez les opérations de fichiers de manière synchrone ou asynchrone :

```javascript
const fs = require('fs');

// Lecture asynchrone
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Lecture synchrone (bloque l'exécution)
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Écrire un fichier
fs.writeFile('/path/to/output.txt', 'Bonjour Node.js!', (err) => {
  if (err) throw err;
  console.log('Fichier écrit avec succès');
});
```

### Serveur HTTP
Créez des serveurs web et effectuez des requêtes HTTP :

```javascript
const http = require('http');
const https = require('https');

// Créer un serveur HTTP
const server = http.createServer((req, res) => {
  // Gérer la requête
});

// Effectuer une requête HTTP
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });
});
req.end();
```

## Gestion des erreurs

Une gestion appropriée des erreurs est cruciale dans les applications Node.js en raison de leur nature asynchrone.

### Gestion des erreurs basée sur les callbacks
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Erreur lors de la lecture du fichier :', err);
    return;
  }
  console.log('Contenu du fichier :', data);
});
```

### Gestion des erreurs basée sur les promesses
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('Contenu du fichier :', data);
  })
  .catch(err => {
    console.error('Erreur lors de la lecture du fichier :', err);
  });
```

### Gestion des erreurs avec Async/Await
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('Contenu du fichier :', data);
  } catch (err) {
    console.error('Erreur lors de la lecture du fichier :', err);
  }
}
```

## Optimisation des performances

### Clustering
Utilisez plusieurs cœurs CPU avec le module cluster :

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Maître ${process.pid} est en cours d'exécution`);
  
  // Créer des workers
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} est mort`);
  });
} else {
  // Les workers partagent la connexion TCP
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`Worker ${process.pid} démarré`);
}
```

### Gestion de la mémoire
- Évitez les fuites de mémoire en nettoyant correctement les écouteurs d'événements
- Utilisez des flux pour traiter les gros fichiers au lieu de tout charger en mémoire
- Surveillez l'utilisation de la mémoire avec `process.memoryUsage()`

### Mise en cache
Implémentez des stratégies de mise en cache pour améliorer les performances :

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```

```javascript
const logger = winston.createLogger({
```


### Limitation de débit
Implémentez la limitation de débit pour prévenir les abus :

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limite chaque IP à 100 requêtes par windowMs
});

app.use(limiter);
```

```javascript

const config = {
```
