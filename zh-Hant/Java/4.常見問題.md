# Java 常見問題

## 基礎問題

### 問題一：java 和 javac 命令不可用

**問題描述**：在命令列輸入 java -version 或 javac -version 提示命令不存在。

**解決方法**：
1. 確認 JDK 已正確安裝
2. 檢查 JAVA_HOME 環境變數是否設定
3. 檢查 Path 環境變數是否包含 %JAVA_HOME%\bin（Windows）或 $JAVA_HOME/bin（Linux/macOS）
4. 重新開啟終端機使環境變數生效

### 問題二：找不到或無法載入主類別

**問題描述**：執行 java 命令時提示 "Could not find or load main class"。

**解決方法**：
1. 確保在正確的目錄下執行命令
2. 檢查類別名稱是否與檔案名稱完全一致
3. 如果類別在套件中，需要使用完整套件名執行
4. 檢查 classpath 是否正確設定

### 問題三：編碼問題導致中文亂碼

**問題描述**：程式中的中文輸出顯示為亂碼。

**解決方法**：
1. 編譯時指定編碼：javac -encoding UTF-8 MyClass.java
2. 執行指定編碼：java -Dfile.encoding=UTF-8 MyClass
3. 確保原始檔案儲存為 UTF-8 編碼
4. 整合開發環境中設定正確的檔案編碼

## 環境設定問題

### 問題四：多個 JDK 版本衝突

**問題描述**：系統中有多個 JDK 版本，java 命令使用了錯誤的版本。

**解決方法**：
1. 檢查 JAVA_HOME 指向的 JDK 版本
2. Windows 上使用 where java 查看所有 java 命令位置
3. Linux/macOS 上使用 which java 查看路徑
4. 調整 PATH 環境變數中 JDK bin 目錄的順序
5. 使用 update-alternatives（Linux）管理多版本

### 問題五：連接埠被佔用

**問題描述**：啟動 Java Web 應用程式時提示連接埠被佔用。

**解決方法**：
1. Windows：netstat -ano | findstr 8080 查詢佔用連接埠的程式
2. Linux：lsof -i:8080 或 netstat -tulpn | grep 8080
3. 結束佔用連接埠的程式，或修改應用程式的連接埠設定

### 問題六：記憶體溢位

**問題描述**：程式執行時出現 OutOfMemoryError。

**解決方法**：
1. 增加 JVM 堆疊記憶體大小：java -Xmx2048m MyClass
2. 檢查程式碼是否存在記憶體洩漏
3. 使用記憶體分析工具（如 VisualVM、JProfiler）排查問題
4. 優化資料結構，減少記憶體佔用

## 開發常見問題

### 問題七：NullPointerException

**問題描述**：空指標例外是最常見的執行時例外。

**解決方法**：
1. 在使用物件前檢查是否為 null
2. 使用 Optional 類別避免空指標
3. 使用 Objects.requireNonNull() 驗證參數
4. 初始化所有必要的物件

### 問題八：ConcurrentModificationException

**問題描述**：遍歷集合時修改集合內容導致例外。

**解決方法**：
1. 使用迭代器的 remove() 方法刪除元素
2. 使用 CopyOnWriteArrayList 替代 ArrayList
3. 先記錄要刪除的元素，遍歷後再刪除
4. 使用 Stream API 的 filter 過濾元素

### 問題九：類別轉換例外

**問題描述**：出現 ClassCastException。

**解決方法**：
1. 使用 instanceof 檢查物件型態
2. 盡量使用介面型態而非具體實現
3. 在需要轉換前進行型態檢查

### 問題十：數字精度問題

**問題描述**：浮點數計算結果不準確。

**解決方法**：
1. 使用 BigDecimal 替代 double 進行精確計算
2. 了解浮點數的精度限制
3. 避免使用浮點數比較，使用 BigDecimal 的 compareTo

## 效能問題

### 問題十一：程式執行緩慢

**問題描述**：Java 程式執行速度慢。

**解決方法**：
1. 使用效能分析工具定位瓶頸
2. 避免在迴圈中進行重複計算
3. 使用 StringBuilder 替代字串拼接
4. 選擇合適的集合型態
5. 啟用 JIT 編譯器的優化

### 問題十二：死結

**問題描述**：程式出現死結，無法繼續執行。

**解決方法**：
1. 避免巢狀鎖
2. 統一加鎖順序
3. 使用 tryLock() 逾時機制
4. 減少同步區塊的範圍

## 建構工具問題

### 問題十三：Maven/Gradle 依賴衝突

**問題描述**：建構時出現依賴版本衝突。

**解決方法**：
1. 使用 mvn dependency:tree 查看依賴樹
2. 在 pom/gradle 中明確指定依賴版本
3. 排除傳遞性依賴
4. 使用 Maven Central 的 Bom 管理版本

### 問題十四：建構失敗

**問題描述**：Maven 或 Gradle 建構失敗。

**解決方法**：
1. 查看錯誤訊息定位問題
2. 清理快取：mvn clean 或 gradle clean
3. 檢查網路連線
4. 確認 JDK 版本符合專案要求
5. 查看是否有語法錯誤

## 框架問題

### 問題十五：Spring Boot 啟動失敗

**問題描述**：Spring Boot 應用程式無法啟動。

**解決方法**：
1. 查看啟動日誌中的錯誤訊息
2. 檢查 application.properties 或 application.yml 設定
3. 確認資料庫連線設定正確
4. 檢查依賴是否完整
5. 查看連接埠是否被佔用

### 問題十六：Hibernate 延遲載入例外

**問題描述**：Session 關閉後存取延遲載入的物件。

**解決方法**：
1. 使用 @Fetch(FetchMode.JOIN) 立即載入
2. 在 Session 關閉前初始化需要的物件
3. 使用 Hibernate.initialize() 手動載入
4. 設定 spring.jpa.open-in-view=true

## 版本選擇問題

### 問題十七：應該學習哪個 Java 版本

**建議**：
- 初學者：推薦 Java 21 或 Java 17，語法現代，資料豐富
- 企業專案：根據專案依賴選擇，通常是 Java 8、11 或 17
- 新專案：強烈推薦 Java 21 或更高版本

### 問題十八：JDK 和 JRE 的差異

**解答**：
- JDK（Java Development Kit）：開發工具包，包含 JRE 和編譯、偵錯等開發工具
- JRE（Java Runtime Environment）：Java 執行時環境，僅包含執行 Java 程式所需的類別庫和 JVM

如果需要開發 Java 程式，安裝 JDK；如果只需要執行已編譯的 Java 程式，安裝 JRE 即可。

## 其他常見問題

### 問題十九：如何學習 Java

**建議**：
1. 掌握 Java 基礎語法和物件導向概念
2. 熟悉 Java 集合框架和常用工具類別
3. 學習 JDBC 資料庫操作
4. 掌握至少一種 Web 框架（Spring/Spring Boot）
5. 了解設計模式和程式碼重構
6. 多實作，動手撰寫專案

### 問題二十：Java 還能火多久

**分析**：Java 作為企業級開發的主流語言，擁有成熟的生態和龐大的用戶基礎。根據 TIOBE 排行榜，Java 長期排名前三。Java 仍在持續更新，每半年發布一個新版本，每兩年發布一個 LTS 版本。因此，在可預見的未來，Java 仍將是非常有價值的技能。
