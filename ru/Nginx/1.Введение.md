# Введение в Nginx

## Что такое Nginx

Nginx (произносится как "engine x") — это HTTP-веб-сервер, обратный прокси, кэш, балансировщик нагрузки, TCP/UDP-прокси сервер и почтовый прокси-сервер.

Nginx был изначально написан Игорем Сысоевым и распространяется по двухпунктовой BSD-лицензии.

## Особенности Nginx

Nginx известен своей гибкостью и высокой производительностью при низком потреблении ресурсов:

- **Самый популярный веб-сервер в мире**
- **Один из самых популярных Docker-образов**
- **Питает несколько контроллеров Ingress для Kubernetes**

### Основные функции HTTP-сервера

- Обслуживание статических и индексных файлов
- Функция автоматического индексирования
- Кэш дескрипторов открытых файлов
- Ускоренное проксирование с кэшированием
- Балансировка нагрузки и отказоустойчивость
- Ускоренная поддержка с кэшированием для серверов FastCGI, uwsgi, SCGI и memcached
- Модульная архитектура
- Фильтры: сжатие gzip, диапазоны байтов, чанк-ответы, XSLT, SSI и преобразование изображений
- Поддержка SSL и TLS SNI
- Поддержка HTTP/2 с приоритизацией на основе весов и зависимостей
- Поддержка HTTP/3

### Другие функции HTTP-сервера

- Виртуальные серверы на основе имён и IP-адресов
- Поддержка соединений keep-alive и pipelined
- Форматы логов доступа, буферизованная запись логов, быстрая ротация логов и syslog
- Перенаправление кодов ошибок 3xx-5xx
- Модуль перезаписи: изменение URI с использованием регулярных выражений
- Контроль доступа на основе IP-адреса клиента, пароля (HTTP Basic Auth) и результата подзапроса
- Валидация HTTP referer
- Методы PUT, DELETE, MKCOL, COPY и MOVE
- Потоковая передача FLV и MP4
- Ограничение частоты ответов
- Ограничение одновременных соединений или запросов с одного адреса
- Геолокация на основе IP
- A/B-тестирование
- Зеркалирование запросов
- Встроенный Perl
- Язык сценариев njs

### Функции почтового прокси-сервера

- Перенаправление пользователей на IMAP или POP3-сервер с использованием внешнего HTTP-сервера аутентификации
- Аутентификация пользователей с использованием внешнего HTTP-сервера аутентификации и перенаправление подключения на внутренний SMTP-сервер
- Поддерживаемые методы аутентификации: POP3 (USER/PASS, APOP, AUTH LOGIN/PLAIN/CRAM-MD5), IMAP (LOGIN, AUTH LOGIN/PLAIN/CRAM-MD5), SMTP (AUTH LOGIN/PLAIN/CRAM-MD5)
- Поддержка SSL
- Поддержка STARTTLS и STLS

### Функции TCP/UDP-прокси сервера

- Generic-проксирование TCP и UDP
- Поддержка SSL и TLS SNI для TCP
- Балансировка нагрузки и отказоустойчивость
- Контроль доступа на основе адреса клиента
- Ограничение одновременных подключений с одного адреса
- Форматы логов доступа, буферизованная запись логов, быстрая ротация логов и syslog
- Геолокация на основе IP
- A/B-тестирование
- Язык сценариев njs

### Архитектура и масштабируемость

- Один главный процесс и несколько рабочих процессов
- Рабочие процессы запускаются от непривилегированного пользователя
- Гибкая конфигурация
- Переконфигурация и обновление исполняемого файла без прерывания обслуживания клиентов
- Поддержка kqueue (FreeBSD 4.1+), epoll (Linux 2.6+), /dev/poll (Solaris 7 11/99+), event ports (Solaris 10), select и poll
- Поддержка различных возможностей kqueue
- Поддержка различных возможностей epoll
- Поддержка sendfile, sendfile64 и sendfilev
- Поддержка File AIO
- Поддержка DIRECTIO
- Поддержка Accept-filters и TCP_DEFER_ACCEPT
- 10 000 неактивных HTTP keep-alive соединений занимают около 2,5 МБ памяти
- Операции копирования данных сведены к минимуму

## Преимущества Nginx

### Высокая производительность

Nginx использует событийную асинхронную неблокирующую модель, способную обрабатывать высокие параллельные соединения при низком потреблении памяти.

### Низкое потребление ресурсов

По сравнению с другими веб-серверами, Nginx потребляет значительно меньше памяти и CPU при том же уровне параллелизма.

### Высокая стабильность

Стабильность Nginx проверена миллионами веб-сайтов по всему миру, с крайне редкими сбоями.

### Простая конфигурация

Конфигурационные файлы Nginx лаконичны и просты для понимания.

### Широкие возможности расширения

Поддерживает богатые модули, позволяя включать или отключать функции по необходимости.

### Горячее развёртывание

Перезагружайте конфигурацию и обновляйте версии без прерывания обслуживания.

## Области применения

Nginx широко используется в следующих сценариях:

- **Веб-сервер**: Как HTTP-сервер для статических файлов
- **Обратный прокси**: Как обратный прокси для бэкенд-серверов
- **Балансировщик нагрузки**: Как балансировщик нагрузки для нескольких бэкенд-серверов
- **Сервер кэширования**: Как CDN или внутренний кэш-сервер
- **API-шлюз**: Как входной шлюз для микросервисов
- **Почтовый прокси**: Как POP3-, IMAP- и SMTP-прокси
- **TCP/UDP-прокси**: Как прокси для протоколов TCP и UDP
