# Руководство по установке Nginx

## Требования к окружению

Nginx поддерживает множество операционных систем:

- Linux (Debian, Ubuntu, CentOS, RHEL, Alpine и др.)
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## Способы установки

### Способ 1: Использование пакетного менеджера (Рекомендуется)

#### Ubuntu/Debian

```bash
# Обновление списка пакетов
sudo apt update

# Установка Nginx
sudo apt install nginx

# Запуск Nginx
sudo systemctl start nginx

# Автозапуск при загрузке
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# Установка EPEL-репозитория
sudo yum install epel-release

# Установка Nginx
sudo yum install nginx

# Запуск Nginx
sudo systemctl start nginx

# Автозапуск при загрузке
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# Установка через Homebrew
brew install nginx

# Запуск Nginx
brew services start nginx
```

### Способ 2: Компиляция из исходного кода

#### Установка зависимостей для сборки

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### Скачивание исходного кода

```bash
# Скачать последнюю стабильную версию
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# Распаковать
tar -zxf nginx-1.26.0.tar.gz

# Перейти в директорию
cd nginx-1.26.0
```

#### Настройка сборки

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### Компиляция и установка

```bash
# Компиляция
make

# Установка
sudo make install
```

### Способ 3: Использование Docker

```bash
# Скачать официальный образ
docker pull nginx

# Запустить Nginx-контейнер
docker run -d -p 80:80 --name mynginx nginx

# Смонтировать конфигурацию и логи
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### Способ 4: Windows

1. Скачать версию для Windows с https://nginx.org/en/download.html
2. Распаковать в нужную директорию
3. Дважды кликнуть по nginx.exe для запуска
4. Или запустить через командную строку:

```cmd
cd C:\nginx
start nginx
```

## Проверка после установки

### Проверка установки

```bash
# Проверить версию Nginx
nginx -v

# Подробная информация о версии
nginx -V
```

### Проверка статуса выполнения

```bash
# Проверить запущенные процессы
ps aux | grep nginx

# Проверить статус через systemctl
systemctl status nginx
```

### Тестирование доступа

```bash
# Тест через curl
curl http://localhost

# Или открыть в браузере
http://localhost
```

## Структура директорий

### Стандартные директории в Linux

| Путь | Описание |
|------|----------|
| /etc/nginx/ | Директория конфигурации |
| /etc/nginx/nginx.conf | Главный конфигурационный файл |
| /etc/nginx/conf.d/ | Директория подконфигураций |
| /var/log/nginx/ | Директория логов |
| /var/www/html/ | Корневая директория документов по умолчанию |
| /usr/sbin/nginx | Исполняемый файл |
| /usr/share/nginx/html/ | Директория статических файлов по умолчанию |

### Директории при компиляции

| Путь | Описание |
|------|----------|
| /usr/local/nginx/ | Корневая директория установки |
| /usr/local/nginx/conf/ | Директория конфигурации |
| /usr/local/nginx/html/ | Корневая директория документов |
| /usr/local/nginx/logs/ | Директория логов |
| /usr/local/nginx/sbin/ | Директория исполняемых файлов |

## Удаление Nginx

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### Компилированная установка

```bash
# Остановить Nginx
sudo nginx -s stop

# Удалить директорию установки
sudo rm -rf /usr/local/nginx
```

## Частые проблемы

### Конфликт портов

Если запуск не удаётся из-за конфликта портов:

```bash
# Проверить занятость порта
netstat -tulpn | grep :80

# Изменить порт Nginx
# Отредактировать /etc/nginx/nginx.conf
# Изменить listen 80; на listen 8080;
```

### Проблемы с правами

При ошибках прав доступа:

```bash
# Проверить пользователя Nginx
ps aux | grep nginx

# Исправить права на файлы
sudo chown -R www-data:www-data /var/www/html/
```

### Проверка синтаксиса конфигурации

```bash
# Тестирование синтаксиса конфигурации
nginx -t

# Тестирование с указанным конфигом
nginx -t -c /path/to/nginx.conf
```
