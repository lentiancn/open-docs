# Node.js 簡介

Node.js® 是一個免費、開源、跨平台的 JavaScript 執行環境，它讓開發人員能夠建立伺服器、Web 應用程式、命令列工具和指令碼。

## 核心特性

作為一個非同步事件驅動的 JavaScript 執行環境，Node.js 旨在建構可擴展的網路應用程式。在經典的 "hello world" 範例中，可以同時處理許多連線。每次連線時，回呼函數被觸發，但若沒有工作要做，Node.js 將進入睡眠狀態。

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

## 設計理念

這與當今更常見的使用作業系統執行緒併發模型形成對比。基於執行緒的網路編程效率相對低效，而且非常難以使用。此外，Node.js 的使用者不必擔心死結行程，因為沒有鎖定機制。

Node.js 幾乎沒有任何函數直接執行 I/O 操作，除非使用 Node.js 標準函式庫的同步方法執行 I/O，否則行程永遠不會阻塞。由於沒有任何阻塞，在 Node.js 中開發可擴展的系統非常合理。

## 事件驅動架構

Node.js 在設計上受到了諸如 Ruby 的 Event Machine 和 Python 的 Twisted 等系統的影響。Node.js 將事件模型推進了一步，它將事件迴圈呈現為執行時期建構而不是函式庫。

在其他系統中，始終存在一個阻塞呼叫來啟動事件迴圈。通常，行為是透過指令稿開頭的回呼函數定義的，並且在指令稿末尾透過像 `EventMachine::run()` 這樣的阻塞呼叫啟動伺服器。

在 Node.js 中，沒有這樣的啟動事件迴圈呼叫。Node.js 在執行輸入指令稿後簡單地進入事件迴圈。當沒有更多的回呼要執行時，Node.js 退出事件迴圈。這種行為類似於瀏覽器 JavaScript —— 事件迴圈對使用者是隱藏的。

## HTTP 支援

HTTP 在 Node.js 中是一等公民，設計時考慮了串流傳輸和低延遲。這使得 Node.js 非常適合作為 Web 函式庫或框架的基礎。

## 多核心處理器支援

Node.js 的設計雖然沒有使用執行緒，但這並不意味著你無法利用環境中的多核心處理器。可以使用 `child_process.fork()` API 來產生子行程，這些子行程之間的通訊非常簡單。

在此介面的基礎上，還建構了 `cluster` 模組，它允許多個行程之間共用通訊端，從而在多個核心上實現負載平衡。
