# Element UI FAQ

Common questions and answers.

## Installation Issues

### Q: Installation failed?

**Solutions:**
1. Check Node.js version
```bash
node --version  # Requires 12.0.0+
```
2. Clean npm cache
```bash
npm cache clean --force
```
3. Use Taobao mirror
```bash
npm config set registry https://registry.npmmirror.com
```
4. Delete node_modules and reinstall
```bash
rm -rf node_modules
npm install
```

### Q: Module not found?

**Solutions:**
1. Confirm element-ui is installed
```bash
npm list element-ui
```
2. Check package.json for dependencies

### Q: Vue version incompatible?

**Solutions:**
Element UI 2.x requires Vue 2.6.0+:
```bash
npm install vue@2
```

## Usage Issues

### Q: Components not displaying?

**Solutions:**
1. Confirm components are properly imported
```javascript
import { Button } from 'element-ui'
Vue.use(Button)
```
2. Check console for errors
3. Confirm component names are correct

### Q: Styles not applied?

**Solutions:**
1. Confirm style file is imported
```javascript
import 'element-ui/lib/theme-chalk/index.css'
```
2. Check for style overrides
3. Use developer tools to inspect element styles

### Q: Table not showing data?

**Solutions:**
1. Check data format
2. Confirm column definitions are correct
```html
<el-table-column prop="name" label="Name"></el-table-column>
```
3. Check if data is reactive

### Q: Form validation not working?

**Solutions:**
1. Import Form and FormItem
2. Set prop attribute
```html
<el-form-item label="Name" prop="name">
  <el-input v-model="form.name"></el-input>
</el-form-item>
```
3. Define rules in data

### Q: Dropdown options not displaying?

**Solutions:**
1. Check options data
2. Confirm value and label mapping
```html
<el-option
  :value="item.value"
  :label="item.label">
</el-option>
```

### Q: Dialog data not resetting after close?

**Solutions:**
1. Use v-if to control dialog
```html
<el-dialog :visible.sync="dialogVisible" v-if="dialogVisible">
```
2. Manually reset form on close
```javascript
this.$refs.form.resetFields()
```

## Style Issues

### Q: Style conflicts?

**Solutions:**
1. Use more specific selectors
2. Check z-index levels
3. Use !important to override (not recommended)

### Q: Icons not displaying?

**Solutions:**
1. Confirm element-ui-icons is installed
```bash
npm install element-ui-icons
```
2. Import properly
```javascript
import 'element-ui-icons'
```

### Q: Theme color incorrect?

**Solutions:**
1. Check for global style overrides
2. Confirm SCSS variable import order is correct
3. Clear browser cache

## Internationalization Issues

### Q: Language switch not working?

**Solutions:**
1. Configure i18n properly
```javascript
Vue.use(ElementUI, {
  i18n: (key, value) => i18n.t(key, value)
})
```
2. Confirm language pack is imported

### Q: Some text not translated?

**Solutions:**
1. Check complete language pack
2. Use ElementUI locales configuration

## Performance Issues

### Q: Page loading slow?

**Solutions:**
1. Use on-demand import
2. Use CDN import
3. Enable Gzip compression

### Q: Large data table laggy?

**Solutions:**
1. Use virtual scrolling
2. Load data in pages
3. Disable unnecessary features

## Build Issues

### Q: Styles lost after build?

**Solutions:**
1. Check vue.config.js configuration
2. Confirm style import position is correct

### Q: Components not displaying in production?

**Solutions:**
1. Check Vue.use() is called
2. Confirm on-demand import is complete

## Upgrade Issues

### Q: How to upgrade from 1.x to 2.x?

**Solutions:**
1. Read official migration guide
2. Replace incompatible APIs gradually
3. Run tests to ensure functionality

### Q: Which to use for Vue 3 project?

**Solutions:**
Element UI is for Vue 2. For Vue 3 projects, use Element Plus:
```bash
npm install element-plus
```
