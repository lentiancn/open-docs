# Kubernetes 简介

## 什么是 Kubernetes

Kubernetes（简称 K8s）是一个开源的容器编排平台，用于自动化部署、扩缩和管理容器化应用程序。它将组成应用程序的容器组合成逻辑单元，便于管理和服务发现。

Kubernetes 源自 Google 15 年生产环境的运维经验，同时凝聚了社区的最佳创意和实践。

## 核心功能

### 自动化部署与回滚
Kubernetes 会分步骤地将针对应用或其配置的更改上线，同时监视应用程序运行状况以确保不会同时终止所有实例。如果出现问题，Kubernetes 会自动回滚所作更改。

### 服务发现与负载均衡
你无需修改应用来使用陌生的服务发现机制。Kubernetes 为每个 Pod 提供了自己的 IP 地址并为一组 Pod 提供一个 DNS 名称，并且可以在它们之间实现负载均衡。

### 存储编排
自动挂载所选存储系统，包括本地存储、公有云提供商所提供的存储或者诸如 iSCSI 或 NFS 这类网络存储系统。

### Secret 和配置管理
部署和更新 Secret 和应用程序的配置而不必重新构建容器镜像，且不必将软件堆栈配置中的秘密信息暴露出来。

### 自动装箱
根据资源需求和其他限制自动放置容器，同时避免影响可用性。将关键性的和尽力而为性质的工作负载进行混合放置，以提高资源利用率并节省更多资源。

### 批量执行
除了服务之外，Kubernetes 还可以管理批处理和 CI 工作负载，在期望时替换掉失效的容器。

### 自我修复
Kubernetes 会自动重启崩溃的容器，在必要时替换整个 Pod，在发生更大范围的故障时重新挂载存储，并且能够与节点自动扩缩容器集成，实现节点级别的自我修复能力。

### 水平扩缩
使用一个简单的命令、一个 UI 或基于 CPU 使用情况自动对应用程序进行扩缩。

### 可扩展性设计
无需更改上游源码即可扩展 Kubernetes 集群。

## 架构概述

### 控制平面
控制平面负责维护整个集群的状态，包括：
- API Server：集群的入口，提供 RESTful API
- etcd：分布式键值存储，保存集群数据
- Scheduler：负责调度 Pod 到合适的节点
- Controller Manager：运行各种控制器维护集群状态

### 工作节点
工作节点是运行容器的地方，每个节点包含：
- Kubelet：与控制平面通信，管理节点上的容器
- Container Runtime：负责运行容器（Docker、containerd 等）
- Kube Proxy：负责网络代理和负载均衡

## 应用场景

### 云计算
Kubernetes 可以自由地部署在企业内部、私有云、混合云或公有云，让你灵活选择适合的基础设施。

### 微服务架构
Kubernetes 是运行微服务的理想平台，能够高效管理大量独立的服务实例。

### 机器学习
Kubernetes 是机器学习的优秀平台，因为它提供了所有的调度功能。

### 持续部署
支持快速迭代，能够每天多次部署应用的更新版本。

## 优势

### 规模扩展
Google 每周运行数十亿个容器，Kubernetes 基于与之相同的原则来设计，能够在不扩张运维团队的情况下进行规模扩展。

### 灵活性
无论是本地测试，还是跨国公司，Kubernetes 的灵活性都能让你在应对复杂系统时得心应手。

### 社区支持
Kubernetes 拥有庞大且快速增长的生态系统，服务、支持和工具的使用范围广泛。
