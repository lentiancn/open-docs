# JMeter 常見問題

本章節整理了使用 Apache JMeter 過程中常見的問題及解決方案。

## 安裝相關問題

### Q1: 執行 JMeter 時提示 "java" 不是內部或外部命令

**A**: 確保已正確安裝 Java 並設定環境變數。在命令列中執行以下命令驗證：
```bash
java -version
```
如果未安裝，請下載並安裝 Java JDK 8 或更高版本。如果是 Windows 系統，還需要設定 JAVA_HOME 環境變數。

### Q2: JMeter 啟動後介面顯示亂碼

**A**: 這是由於字體編碼問題導致的。可以透過以下方式解決：
1. 開啟 bin/jmeter.properties 檔案
2. 新增或修改以下設定：
   ```
   sampleresult.default.encoding=UTF-8
   ```
3. 重新啟動 JMeter

### Q3: JMeter 記憶體不足導致崩潰

**A**: 預設的堆記憶體可能不夠用，可以調整：
- Windows：編輯 bin/jmeter.bat 檔案，找到 set HEAP 參數並增加大小
- Linux/Mac：編輯 bin/jmeter 檔案，調整 HEAP 變數
建議設定為 2GB 或更高：
```bash
HEAP="-Xms2g -Xmx2g"
```

## 測試執行問題

### Q4: 如何模擬多個不同使用者登入？

**A**: 使用參數化方式：
1. 建立包含使用者名稱密碼的 CSV 檔案
2. 新增"CSV 資料檔案設定"元件
3. 在 HTTP 請求中使用 ${username} 和 ${password} 變數

### Q5: JMeter 無法擷取 Session 或 Cookie

**A**: 確保已新增適當的元件：
1. 新增"HTTP Cookie 管理器"到執行緒組
2. 對於需要登入的測試，確保 Cookie 管理器啟用了"Clear cookies each iteration"選項
3. 檢查請求是否正確攜帶了 Cookie

### Q6: 回應資料中的中文顯示為亂碼

**A**: 可以透過以下方式解決：
1. 在 HTTP 請求中新增或修改"內容編碼"為 UTF-8
2. 在"HTTP 請求預設值"中設定預設編碼
3. 使用後置處理器中的"正規表達式擷取器"並指定編碼

### Q7: 如何處理 HTTPS 憑證問題？

**A**: 對於測試環境：
1. 將受信任的憑證匯入 Java cacerts 存放區
2. 或者在 HTTP 請求中設定"實作"為 Java
3. 對於自我簽署憑證，可能需要匯入 JMeter 的 truststore

### Q8: JMeter 執行時間長了記憶體持續增長

**A**: 建議採取以下措施：
1. 減少監聽器收集的資料量
2. 在命令列模式下執行測試
3. 增加垃圾回收頻率，在 jmeter.properties 中設定：
   ```
   mode=Stripped
   ```
4. 定期清理結果檔案

## 效能相關問題

### Q9: JMeter 無法產生足夠的負載

**A**: 可以嘗試以下方法：
1. 使用分散式測試（遠端測試）
2. 調整執行緒數和迴圈次數
3. 減少不必要的監聽器
4. 使用命令列模式而非 GUI 模式
5. 檢查網路頻寬是否成為瓶頸

### Q10: 測試結果中回應時間波動很大

**A**: 可能的原因和解決方案：
1. 伺服器端存在 GC 或快取問題
2. 網路不穩定
3. 測試資料存在依賴關係
4. 使用同步計時器確保並發請求
5. 增加預熱時間，讓伺服器穩定後再開始統計

### Q11: 如何設定合理的並發數？

**A**: 建議按照以下步驟：
1. 從較小並發開始（如 10-50）
2. 逐步增加並發，觀察系統回應
3. 找到效能拐點（回應時間開始明顯增加）
4. 確定最大安全並發數
5. 進行壓力測試，驗證系統極限

## 報告和結果分析

### Q12: 如何生成 HTML 報告？

**A**: 使用命令列生成：
```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/report
```
或者使用現有的結果檔案生成：
```bash
jmeter -g results.jtl -o /path/to/report
```

### Q13: 聚合報告中的指標含義是什麼？

- **樣本數（Samples）**：請求總數
- **平均值（Average）**：平均回應時間（毫秒）
- **中位數（Median）**：回應時間中間值
- **90%/95%/99% 百分位**：90%/95%/99% 的請求回應時間低於此值
- **最小值/最大值**：最小和最大回應時間
- **錯誤率（Error %）**：失敗請求百分比
- **吞吐量（Throughput）**：每秒處理的請求數

### Q14: 結果檔案過大如何處理？

**A**: 可以：
1. 在監聽器中設定"儲存為 CSV"格式
2. 設定只儲存必要的欄位
3. 使用"分散式測試"分散儲存
4. 定期輪轉結果檔案

## 指令碼相關問題

### Q15: 如何處理動態變化的參數？

**A**: 常用的方法包括：
1. 使用"正規表達式擷取器"從回應中擷取
2. 使用"JSON 擷取器"擷取 JSON 資料
3. 使用"CSS/JQuery 選擇器"擷取 HTML 元素
4. 使用"XPath 擷取器"擷取 XML 資料

### Q16: BeanShell 指令碼無法執行

**A**: 確保：
1. JDK 已正確安裝（不是 JRE）
2. BeanShell 庫檔案在 lib 目錄中
3. 檢查指令碼語法是否正確

### Q17: 如何實現複雜的業務邏輯？

**A**: 可以使用：
1. 邏輯控制器組合
2. BeanShell 或 JSR223 指令碼
3. 前置/後置處理器
4. 吞吐量控制器實現條件執行

## 其他問題

### Q18: JMeter 與瀏覽器的區別是什麼？

JMeter 不是瀏覽器，它工作在協定級別：
- 不執行 JavaScript
- 不渲染 HTML 頁面
- 不處理 CSS
- 無法模擬瀏覽器的所有行為

### Q19: 如何學習 JMeter？

建議的學習路徑：
1. 閱讀 JMeter 官方使用者手冊
2. 從簡單的 HTTP 請求開始練習
3. 學習參數化和關聯處理
4. 掌握分散式測試
5. 學習自訂指令碼開發

### Q20: JMeter 支援哪些作業系統？

JMeter 是純 Java 應用，支援任何安裝 Java 的作業系統，包括 Windows、Linux、Mac OSX、Solaris 等。
