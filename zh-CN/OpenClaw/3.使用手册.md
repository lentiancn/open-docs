# OpenClaw 使用手册

本手册将帮助您了解如何有效地使用 OpenClaw。

## 基本使用

### 与 OpenClaw 对话

安装完成后，您可以通过多种方式与 OpenClaw 交互：

1. **Control UI**：通过浏览器访问 `http://127.0.0.1:18789/`
2. **CLI 命令**：`openclaw agent --message "您的消息"`
3. **已配置的渠道**：通过 WhatsApp、Telegram、Discord 等发送消息

### 发送消息

使用 CLI 发送消息：

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## 核心概念

### Gateway

Gateway 是 OpenClaw 的核心控制平面，管理所有会话、渠道、工具和事件。默认监听 `ws://127.0.0.1:18789`。

### Session（会话）

OpenClaw 使用会话模型来管理对话：

- **主会话**：直接对话
- **群组隔离**：群组消息单独处理
- **激活模式**：控制何时响应
- **回复策略**：决定如何回复

### Workspace

工作区是 OpenClaw 存储配置和数据的目录：

- 默认位置：`~/.openclaw/workspace`
- 包含文件：AGENTS.md、SOUL.md、TOOLS.md
- 技能目录：`~/.openclaw/workspace/skills/`

## 渠道配置

### 支持的渠道

OpenClaw 支持以下通讯平台：

- 即时通讯：WhatsApp、Telegram、Signal、iMessage
- 社交平台：Discord、Slack
- 企业工具：Microsoft Teams、Google Chat、飞书
- 其他：IRC、Matrix、LINE、Mattermost、Nostr 等

### 配置示例

**Telegram**：

```json
{
  "channels": {
    "telegram": {
      "botToken": "您的Bot Token"
    }
  }
}
```

**Discord**：

```json
{
  "channels": {
    "discord": {
      "token": "您的Bot Token"
    }
  }
}
```

**WhatsApp**：

```bash
openclaw channels login
```

## 技能系统

技能是 OpenClaw 扩展功能的方式。您可以：

### 安装技能

从 ClawHub 安装社区技能，或创建自己的技能。

### 技能结构

每个技能是一个包含 SKILL.md 文件的目录：

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHub 是技能注册中心。启用后，OpenClaw 可以自动搜索和安装需要的技能。

## 自动化

### 定时任务

使用 Cron 设置定时执行的任务：

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhook

配置 Webhook 来触发自动化：

```bash
openclaw webhook add --url https://your-endpoint.com/hook
```

### Gmail 集成

设置 Gmail Pub/Sub 来监听新邮件。

## 高级功能

### 浏览器控制

OpenClaw 可以控制专用浏览器：

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvas 可视化

Canvas 是 OpenClaw 的可视化工作空间，支持 A2UI 推送/重置、评估和快照。

### 语音功能

- **Voice Wake**：在 macOS/iOS 上通过唤醒词激活
- **Talk Mode**：连续语音对话（Android 支持）

### 设备节点

OpenClaw 可以连接多种设备：

- **macOS 节点**：系统运行、通知、Canvas、摄像头
- **iOS 节点**：Canvas、语音、摄像头、屏幕录制
- **Android 节点**：连接、聊天、语音、Canvas、摄像头、位置等

## 安全设置

### DM 配对

默认情况下，未知发送者需要配对才能与 OpenClaw 对话：

```bash
openclaw pairing approve
```

### 沙盒模式

为群组/频道设置沙盒隔离：

```json
{
  "agents": {
    "defaults": {
      "sandbox": {
        "mode": "non-main"
      }
    }
  }
}
```

### 运行诊断

定期运行诊断检查安全问题：

```bash
openclaw doctor
```

## 命令参考

| 命令 | 说明 |
|------|------|
| `/status` | 查看会话状态 |
| `/new` 或 `/reset` | 重置会话 |
| `/compact` | 压缩会话上下文 |
| `/think` | 设置思考级别 |
| `/verbose` | 开关详细输出 |
| `/usage` | 查看使用统计 |
| `/restart` | 重启 Gateway |

## 最佳实践

1. **定期运行 doctor**：保持系统健康
2. **谨慎开放 DM**：注意安全设置
3. **使用强模型**：获得最佳体验
4. **定期备份**：保存重要配置

## 故障排除

常见问题解决：

- **Gateway 不启动**：检查端口占用
- **渠道连接失败**：检查凭证配置
- **消息发送失败**：查看日志排查
