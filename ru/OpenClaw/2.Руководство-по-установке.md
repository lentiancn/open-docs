# Руководство по установке OpenClaw

OpenClaw поддерживает несколько способов установки. Выберите наиболее подходящий для ваших нужд.

## Требования

- **Node.js**: 22.x или выше
- **Операционная система**: macOS, Linux, Windows (через WSL2)
- **Опционально**: Tailscale (для удалённого доступа)

## Быстрая установка

### macOS / Linux

Самый простой способ — использовать скрипт установки:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Или с помощью npm/pnpm:

```bash
npm install -g openclaw@latest
# или
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## Начальная настройка

После установки запустите мастер настройки:

```bash
openclaw onboard --install-daemon
```

Мастер проведёт вас через:

1. **Настройка аутентификации**: Настройка ключей API или OAuth
2. **Настройки Gateway**: Настройка порта и службы шлюза
3. **Подключение каналов**: Подключение мессенджеров
4. **Установка навыков**: Выбор и установка необходимых навыков

## Установка как системная служба

Используйте `--install-daemon` для установки OpenClaw как фоновой службы:

```bash
openclaw onboard --install-daemon
```

Это создаст службу уровня пользователя (launchd или systemd), которая обеспечит постоянную работу OpenClaw.

## Запуск Gateway вручную

Если вы не хотите устанавливать как службу:

```bash
openclaw gateway --port 18789 --verbose
```

## Установка через Docker

Если вы предпочитаете Docker, обратитесь к официальной документации по развёртыванию.

## Сборка из исходного кода

Если вы хотите участвовать в разработке или настраивать OpenClaw:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

Примечание: pnpm используется для сборки. TypeScript можно запускать напрямую через tsx.

## Проверка установки

После установки вы можете:

1. **Проверить статус Gateway**:
   ```bash
   openclaw gateway status
   ```

2. **Открыть Control UI**:
   ```bash
   openclaw dashboard
   ```
   Или откройте `http://127.0.0.1:18789/` в браузере

3. **Отправить тестовое сообщение** (требует настройки канала):
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## Обновление OpenClaw

OpenClaw имеет три канала обновлений:

- **stable**: Стабильные выпуски (рекомендуется)
- **beta**: Предварительные версии
- **dev**: Сборки для разработки

Переключение каналов:

```bash
openclaw update --channel stable|beta|dev
```

После обновления рекомендуется выполнить:

```bash
openclaw doctor
```

## Частые команды

| Команда | Описание |
|---------|----------|
| `openclaw onboard` | Запуск мастера настройки |
| `openclaw gateway start` | Запуск Gateway |
| `openclaw gateway status` | Проверка статуса Gateway |
| `openclaw dashboard` | Открытие Control UI |
| `openclaw agent --message "..."` | Отправка сообщения ассистенту |
| `openclaw message send --to xxx --message "xxx"` | Отправка сообщения |
| `openclaw doctor` | Диагностика и исправление проблем |
| `openclaw update` | Обновление OpenClaw |

## Переменные окружения

Для пользовательских расположений настроек:

- `OPENCLAW_HOME`: Устанавливает домашний каталог
- `OPENCLAW_STATE_DIR`: Переопределяет каталог состояния
- `OPENCLAW_CONFIG_PATH`: Переопределяет путь к файлу конфигурации

## Устранение неполадок

**Ошибка установки?**

- Убедитесь, что Node.js >= 22
- Проверьте подключение к сети
- Запустите `openclaw doctor` для диагностики

**Как удалить?**

- Остановите службу Gateway
- Удалите каталог установки
- Очистите файлы конфигурации
