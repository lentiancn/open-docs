# PlantUML Benutzungsanleitung

Diese Anleitung zeigt, wie Sie mit PlantUML verschiedene UML-Diagramme erstellen.

## Schnellstart

### Grundlegende Syntax

```plantuml
@startuml
Alice -> Bob: Hello
@enduml
```

## Sequenzdiagramm

```plantuml
@startuml
actor Benutzer
participant "Frontend" as Front
participant "Backend" as Back
database "Datenbank" as DB

Benutzer -> Front: Anfrage senden
Front -> Back: API-Aufruf
Back -> DB: Daten abfragen
DB --> Back: Ergebnis zurückgeben
Back --> Front: Antwort
Front --> Benutzer: Ergebnis anzeigen
@enduml
```

## Klassendiagramm

```plantuml
@startuml
class User {
  +id: int
  +username: string
  +email: string
  +login()
  +logout()
}

class Admin extends User {
  +manageUsers()
  +viewReports()
}

class Order {
  +id: int
  +total: decimal
  +createOrder()
  +cancelOrder()
}

User "1" -- "*" Order: places
@enduml
```

## Anwendungsfalldiagramm

```plantuml
@startuml
left to right direction
actor Benutzer
rectangle System {
  Benutzer -> (Anmelden)
  Benutzer -> (Produkte durchsuchen)
  Benutzer -> (In den Warenkorb)
  Benutzer -> (Bestellen)
  (Anmelden) .> (Produkte durchsuchen): extends
}
@enduml
```

## Zustandsdiagramm

```plantuml
@startuml
[*] --> Ausstehend
Ausstehend --> Bezahlt: Zahlung erfolgreich
Bezahlt --> Versandt: Versand
Versandt --> Zugestellt: Zustellung
Zugestellt --> [*]
Ausstehend --> Storniert: Benutzer storniert
Storniert --> [*]
@enduml
```

## Aktivitätsdiagramm

```plantuml
@startuml
start
:Einkauf beginnen;
:Produkte durchsuchen;
if (Produkt gefunden?) then (Ja)
  :In den Warenkorb;
else (Nein)
  :Produkt suchen;
endif
:Zur Kasse;
if (Zahlung erfolgreich?) then (Ja)
  :Bestellung erfolgreich;
else (Nein)
  :Zahlung fehlgeschlagen;
endif
stop
@enduml
```

## Komponentendiagramm

```plantuml
@startuml
package "Frontend" {
  [Web App] as WA
  [Mobile App] as MA
}
package "Backend" {
  [API Gateway] as API
  [Auth Service] as Auth
  [Order Service] as Order
}
package "Daten" {
  database "MySQL" as DB
  database "Redis" as Cache
}

WA --> API
MA --> API
API --> Auth
API --> Order
Order --> DB
Order --> Cache
@enduml
```

## Verteilungsdiagramm

```plantuml
@startuml
node "Benutzergerät" as Client {
  [Browser]
}
node "Load Balancer" as LB {
  [Nginx]
}
node "Anwendungsserver" as App {
  [Tomcat 1]
  [Tomcat 2]
}
node "Datenbankserver" as DB {
  [MySQL]
  [Redis]
}

Client --> LB
LB --> App
App --> DB
@enduml
```

## Zeitdiagramm

```plantuml
@startuml
clock "Uhr" as C with period 1000
binary "Synchronisation" as SY
binary "Daten" as DT

C is idle
SY is low
DT is low

@1000
SY is high
@1500
SY is low
@2000
DT is high
@3000
DT is low
@enduml
```

## Gantt-Diagramm

```plantuml
@startuml
[Design] lasts 5 days
[Entwicklung] lasts 10 days
[Tests] lasts 5 days
[Veröffentlichung] lasts 3 days

[Design] starts at 2024-01-01
[Entwicklung] starts at 2024-01-06
[Tests] starts at 2024-01-16
[Veröffentlichung] starts at 2024-01-21
@enduml
```

## Häufige Befehle

### Elemente ausblenden

```plantuml
@startuml
hide footbox
actor A
participant B
A -> B: Nachricht
@enduml
```

### Stil-Einstellungen

```plantuml
@startuml
skinparam backgroundColor #EEEBDC
skinparam handwritten true
skinparam actorStyle awesome

actor User
User -> (Anmelden)
@enduml
```
