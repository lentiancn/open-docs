# MongoDB 簡介

## 什麼是 MongoDB？

MongoDB 是一個文件資料庫，旨在幫助開發者更快地建構現代應用程式。它將資料儲存在類似 JSON 的靈活文件中，讓您可以像應用程式碼使用資料的方式來建立資料模型。靈活的模式可讓您在不停機的情況下發展資料模型、快速迭代並輕鬆處理非統一資料。

MongoDB 提供強大的查詢引擎、水平擴展和內建高可用性，因此您可以支援從快速原型設計到大型關鍵任務工作負載的所有需求。

MongoDB 是一個完全交易性的操作資料庫，支援廣泛的工作負載類型，包括：

- 基於文件的結構化搜尋（OLTP）
- 資料聚合
- 全文搜尋
- 向量搜尋
- 地理空間搜尋
- 時間序列資料

## 文件資料模型

MongoDB 中的記錄就是一個文件，它是由欄位和值對組成的資料結構。MongoDB 文件類似於 JSON 物件。欄位值可以包含其他文件、陣列和文件陣列。

MongoDB 靈活的物件導向資料模型可讓您以反映程式碼庫中物件模型的方式建構資料。這樣就無需在使用關聯式資料庫時進行複雜的物件關聯對應。

以下是一個 MongoDB 文件範例，展示文件如何以反映應用程式物件模型的格式表達複雜的分層資料：

```
{
  _id: ObjectId("507f1f77bcf86cd799439011"),
  name: "Alice",
  birthdate: ISODate("1990-01-01T00:00:00Z"),
  address: {
    street: "123 Main St",
    city: "Springfield",
    state: "IL"
  },
  hobbies: ["reading", "hiking", "coding"]
}
```

文件資料模型的優點包括：

- 文件對應於許多程式語言中的原生資料類型
- 嵌入式文件和陣列減少了對昂貴連線的需求，這些連線可能會降低效能
- 動態模式支援多態性，允許同一集合中的文件具有不同的結構

## MongoDB 核心架構特性

### 文件資料庫

靈活的文件資料模型可讓您將資料對應到應用程式的需求。與傳統具有嚴格行列格式的關聯式資料庫不同，MongoDB 文件可以包含巢狀結構，非常適合處理複雜的資料關係。

### 交易支援

MongoDB 支援多文件交易，讓您可以將多個讀取和寫入操作作為單一全有或全無事件執行。主要 ACID 交易功能包括：

- ACID 保證：交易可跨多個操作提供原子性、一致性、隔離性和持久性
- 多文件操作：您可以執行跨多個文件的複雜操作，同時保持資料一致性
- 分散式交易：您可以在具有相同 ACID 保證的情況下跨分片叢集協調交易

### 高可用性

MongoDB 的內建複寫機制提供自動故障轉移、資料冗餘和更高的讀取容量。透過自動故障轉移，如果主伺服器不可用，叢集會自動選擇新的主伺服器，確保寫入操作保持可用。此外，資料的多個副本會儲存在不同的伺服器上，以提高資料的持久性。

### 水平擴展

MongoDB 透過稱為分片的技術原生支援水平擴展。主要分片功能包括：

- 自動資料分发：MongoDB 會根據分片鍵自動對資料進行分區，並将其分发到機器叢集中
- 區域分片：您可以定義地理區域，根據分片鍵範圍控制文件的放置位置
- 分片鍵優化：隨著應用程式的發展，您可以優化分片鍵以提高效能

## MongoDB 版本

MongoDB 提供兩個主要版本：

- **社群版（Community Edition）**：開源免費，適用於個人開發者和小型專案
- **企業版（Enterprise Edition）**：商業版本，提供進階安全和管理功能

目前最新穩定版本為 MongoDB 8.2，同時也長期支援 7.0 版本。MongoDB 6.0 已於 2025年7月31日停止支援。

## MongoDB 不支援的平台

MongoDB 不支援 32 位元平台，請確保您的作業系統是 64 位元。

## 應用場景

MongoDB 適用於多種應用場景：

- **內容管理系統**：儲存文章、評論、使用者生成內容等非結構化或半結構化資料
- **行動應用**：儲存使用者設定、應用程式狀態等靈活的資料結構
- **即時分析**：高速寫入和即時聚合能力適合日誌分析、即時統計
- **物聯網**：儲存來自感測器的海量時序資料
- **電子商務**：產品目錄、訂單管理、使用者画像等
- **人工智慧應用**：向量搜尋功能支援 AI 應用的語意搜尋需求

## MongoDB 與關聯式資料庫的對比

| 特性 | MongoDB | 關聯式資料庫 |
|------|---------|------------|
| 資料模型 | 文件（類似 JSON） | 行和列 |
| 模式 | 靈活（動態模式） | 固定（靜態模式） |
| 擴展方式 | 水平擴展（分片） | 垂直擴展 |
| 交易 | 多文件 ACID 交易 | 多表 ACID 交易 |
| 關聯 | 嵌入式文件/陣列 | JOIN 操作 |
| 索引 | 支援多種索引類型 | 支援索引 |

## 總結

MongoDB 作為領先的 NoSQL 資料庫，以其靈活的文件模型、強大的查詢能力、高效能和高可用性，成為現代應用程式開發的熱門選擇。無論是新創公司還是大型企業，都可以根據自身需求選擇合適的部署方式。MongoDB 的設計理念是讓開發者能夠更快地建構現代應用程式，透過靈活的文件模型、直觀的查詢語言和強大的水平擴展能力，幫助團隊快速迭代和擴展應用。
