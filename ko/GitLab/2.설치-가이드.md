# GitLab 설치 가이드

## 요구 사항

GitLab을 설치하기 전에 서버가 다음 하드웨어 및 소프트웨어 요구 사항을 충족하는지 확인하세요. 소규모에서 중규모 팀의 경우 최소 4코어 CPU와 8GB RAM을 권장하며, 이는 수백 개의 저장소와 수십 명의 동시 사용자를 지원할 수 있습니다. 더 큰 팀이나 고부하 시나리오에는 8코어 이상 CPU와 16GB 이상 RAM이 더 필요합니다.

운영 체제의 경우 GitLab은 Ubuntu, Debian, CentOS, Red Hat Enterprise Linux 및 Fedora를 포함한 다양한 Linux 배포판을 지원합니다. Ubuntu 또는 Debian은 패키지 관리가 가장 편리하므로 공식적으로 권장됩니다. GitLab은 설치에 최소 20GB의 디스크 공간이 필요하며, 기본 구성에는 50GB 이상이, 프로덕션 환경에는 종종 수백 GB가 필요합니다.

소프트웨어 종속성에는 SSH 프로토콜 지원을 위한 OpenSSH Server, 데이터베이스(PostgreSQL - GitLab에 내장 버전 포함), 이메일 알림을 위한 SMTP 서버가 포함됩니다. 올바른 시간대와 호스트 이름을 구성하고 서버가 종속성 패키지를 다운로드하기 위해 인터넷에 정상적으로 접근할 수 있는지 확인해야 합니다.

## 설치 단계

### Ubuntu/Debian 설치

Ubuntu 또는 Debian 시스템에서는 공식 apt 리포지토리를 통해 GitLab을 설치할 것을 권장합니다. 이렇게 하면 최신 안정 버전과 자동 보안 업데이트를 받을 수 있습니다. 먼저 필요한 종속성 패키지를 설치하세요:

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

GitLab APT 리포지토리를 추가하고 GitLab CE(Community Edition)를 설치합니다:

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

EXTERNAL_URL을 GitLab에 접근하기 위한 실제 도메인 또는 IP 주소로 바꾸세요. 설치 프로세스가 자동으로 PostgreSQL, Redis 및 GitLab 자체를 구성합니다.

### CentOS/RHEL 설치

CentOS 또는 Red Hat Enterprise Linux에서는 yum 또는 dnf 패키지 관리자를 사용하여 GitLab을 설치할 수 있습니다. 먼저 필요한 종속성을 설치하세요:

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

GitLab YUM 리포지토리를 추가하고 설치합니다:

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## 초기 구성

설치 후 GitLab의 기본 매개변수를 구성해야 합니다. GitLab의 주요 구성 파일인 /etc/gitlab/gitlab.rb 파일을 편집하세요. 가장 중요한 구성 항목은 GitLab 서비스에 대한 외부 액세스 주소인 external_url입니다:

```ruby
external_url 'https://gitlab.example.com'
```

사용자 정의 SSH 포트를 사용해야 하는 경우 gitlab_rails['gitlab_shell_ssh_port'] 매개변수를 구성하세요. 이메일 서버 구성도 알림 및 비밀번호 재설정 이메일을 보내는 데 중요합니다:

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

구성을 저장한 후 다음 명령을 실행하여 변경 사항을 적용하세요:

```bash
sudo gitlab-ctl reconfigure
```

이 명령은 데이터베이스 마이그레이션 및 초기 사용자 생성을 포함한 모든 GitLab 구성 요소를 다시 구성합니다. 전체 프로세스는 몇 분이 걸릴 수 있습니다.

## 설치 확인

구성이 완료되면 브라우저에서 external_url로 지정된 주소에 접근할 수 있습니다.、GitLab 로그인 page。，。한 번，。root，。

GitLab은 서비스 상태를 관리하기 위한 gitlab-ctl 명령줄 도구를 제공합니다:

```bash
# 서비스 상태 확인
sudo gitlab-ctl status

# 로그 보기
sudo gitlab-ctl tail

# 서비스 재시작
sudo gitlab-ctl restart
```

또한 상태 검사를 위해 gitlab-rake 명령을 사용할 수 있습니다:

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Docker 설치

빠른 테스트 또는 소규모 배포의 경우 Docker를 사용하여 GitLab을 실행할 수 있습니다. 먼저 Docker Engine을 설치한 다음 호스트에서 다음 명령을 실행하세요:

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

Docker 설치는 구성, 로그 및 데이터를 지속하기 위해 세 개의 볼륨을 마운트합니다. Docker 컨테이너의 첫 시작은 초기화하는 데 시간이 걸립니다.，。

## 업그레이드 및 유지보수

최신 버전으로 GitLab을 정기적으로 업그레이드하면 새 기능과 보안 수정 프로그램을 얻을 수 있습니다. Ubuntu/Debian에서 업그레이드:

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

업그레이드 전에 데이터 백업을 만드는 것이 권장됩니다. GitLab은 내장 백업 도구를 제공합니다:

```bash
sudo gitlab-backup create
```

백업 파일은 기본적으로 /var/opt/gitlab/backups 디렉터리에 저장되며 백업 보존 정책 및 원격 저장소 위치를 구성할 수 있습니다.
