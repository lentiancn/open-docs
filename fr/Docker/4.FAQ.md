# FAQ sur Docker

Cette page liste les problèmes courants et les solutions pour l'installation et l'utilisation de Docker.

## Problèmes d'Installation

### Docker Desktop ne démarre pas

**Symptômes** : Docker Desktop ne démarre pas ou se ferme immédiatement après le démarrage

**Solutions** :

1. Assurez-vous que le système répond aux exigences matérielles :
   - Windows : Activez Hyper-V ou installez WSL 2
   - Mac : Assurez-vous de ne pas utiliser des modèles Mac obsolètes

2. Réinitialiser Docker Desktop :
   - Windows : Clic droit sur l'icône Docker → Dépannage → Réinitialiser aux paramètres d'usine
   - Mac : Docker → Dépannage → Réinitialiser aux paramètres d'usine

3. Vérifiez les journaux système :
   - Windows : Observateur d'événements → Journaux Windows → Application
   - Mac : Ouvrez la Console et recherchez les erreurs Docker

### Pas de permission pour les commandes Docker

**Symptômes** : "permission denied while trying to connect to the Docker daemon socket"

**Solutions** :

1. Ajoutez votre utilisateur au groupe Docker :
   ```bash
   sudo usermod -aG docker $USER
   ```

2. Déconnectez-vous et reconnectez-vous

3. Ou utilisez sudo avant les commandes Docker

### Erreur d'installation WSL 2 sur Windows

**Symptômes** : "WSL 2 installation is incomplete"

**Solutions** :

1. Activez le Sous-système Windows pour Linux :
   ```bash
   wsl --install
   ```

2. Redémarrez votre ordinateur

3. Mettez à jour WSL vers la dernière version :
   ```bash
   wsl --update
   ```

## Problèmes d'Utilisation

### Le conteneur ne démarre pas

**Symptôme** : Le conteneur reste dans l'état "Exited" ou "Created"

**Solutions** :

1. Vérifiez les logs du conteneur :
   ```bash
   docker logs nom_conteneur
   ```

2. Assurez-vous que tous les ports requis sont disponibles

3. Vérifiez l'intégrité de l'image

### Port déjà utilisé

**Symptôme** : "port is already allocated"

**Solutions** :

1. Trouvez le port utilisé :
   ```bash
   netstat -ano | findstr :8080
   ```

2. Arrêtez l'autre service ou utilisez un autre port

3. Changez la redirection de ports :
   ```bash
   docker run -p 8081:80 nginx
```

### Problèmes d'espace disque

**Symptôme** : "no space left on device"

**Solutions** :

1. Supprimez les conteneurs inutilisés :
   ```bash
   docker container prune
   ```

2. Supprimez les images inutilisées :
   ```bash
   docker image prune -a
   ```

3. Supprimez les volumes inutilisés :
   ```bash
   docker volume prune
   ```

### Problèmes de réseau

**Symptôme** : Le conteneur ne peut pas se connecter à l'extérieur

**Solutions** :

1. Vérifiez la configuration DNS :
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

2. Réinitialisez le réseau Docker :
   ```bash
   docker network prune
   ```

3. Vérifiez la configuration du pare-feu

## Problèmes de Performance

### Performance lente du conteneur

**Solutions** :

1. Vérifiez les limites de ressources :
   ```bash
   docker stats
   ```

2. Allouez plus de RAM (Docker Desktop → Paramètres → Ressources)

3. Utilisez SSD au lieu de HDD pour Docker

### Utilisation élevée du disque

**Solutions** :

1. Nettoyez régulièrement le cache de construction :
   ```bash
   docker builder prune
   ```

2. Supprimez les anciennes versions d'images

3. Limitez la taille des logs

## Questions de Sécurité

### Exécuter des conteneurs en tant que root

**Recommandation** :

Utilisez un utilisateur non privilégié :

```dockerfile
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser
USER appuser
```

### Vulnérabilités dans les images

**Solutions** :

1. Utilisez des images officielles et à jour

2. Analysez les images régulièrement :
   ```bash
   docker scan nom_image
   ```

3. Envisagez d'utiliser des images distroless

## Questions Avancées

### Utiliser Docker dans CI/CD

**Recommandations** :

1. Utilisez Build Kits pour les constructions parallèles
2. Utilisez des constructions multi-étapes pour des images plus petites
3. Mettez en cache les dépendances efficacement
4. Utilisez Docker Buildx pour les constructions multiplateformes

### Persistance des données

**Recommandations** :

1. Utilisez des volumes nommés pour les données critiques
2. Sauvegardez les volumes régulièrement
3. Utilisez des montages de liaison pour le développement

### Environnements multiples

**Recommandations** :

1. Utilisez Docker Compose pour les variables d'environnement
2. Utilisez des fichiers .env
3. Envisagez Docker Swarm ou Kubernetes pour la production
