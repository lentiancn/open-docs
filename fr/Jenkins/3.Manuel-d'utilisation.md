# Manuel d'utilisation

## Principes de base du pipeline

Le pipeline est une fonctionnalité principale de Jenkins qui automatise l'ensemble du processus de développement logiciel, de la construction au test en passant par le déploiement.

## Qu'est-ce que le pipeline ?

Le pipeline est une suite de plugins Jenkins qui permet d'implémenter et d'intégrer des pipelines de livraison continue dans Jenkins. En termes simples, le pipeline automatise l'ensemble du processus permettant d'amener le logiciel du contrôle de version jusqu'aux utilisateurs.

Les définitions de pipeline sont écrites dans un fichier texte (Jenkinsfile) qui peut être validé dans le dépôt de code source du projet. Cela implémente la philosophie du "Pipeline as Code".

### Avantages du pipeline

- **Basé sur le code** : Les pipelines sont versionnés comme du code, permettant la révision et l'itération
- **Durable** : Les pipelines survivent aux redémarrages planifiés et non planifiés de Jenkins
- **Interruptible** : Les pipelines peuvent s'arrêter et attendre une entrée ou une approbation humaine
- **Polyvalent** : Supporte les exigences complexes de CD du monde réel, y compris l'exécution parallèle et les boucles
- **Extensible** : Les plugins peuvent étendre le DSL du pipeline et intégrer d'autres outils

### Concepts du pipeline

#### Nœud

Un nœud est une machine qui fait partie de l'environnement Jenkins et est capable d'exécuter un pipeline. Les nœuds peuvent être des machines maîtres ou des agents.

#### Étape

Une étape définit un sous-ensemble conceptuellement distinct de tâches effectuées dans l'ensemble du pipeline, comme les étapes "Build", "Test" et "Deploy". Les étapes sont utilisées pour visualiser l'état d'exécution du pipeline.

#### Pas

Un pas est une action fondamentale dans le pipeline Jenkins qui dit à Jenkins quoi faire à un moment particulier. Par exemple, pour exécuter une commande shell, utilisez le pas `sh`.

### Syntaxe du pipeline

#### Pipeline déclaratif

Le pipeline déclaratif est une syntaxe plus récente avec des fonctionnalités plus riches, plus facile à écrire et à lire :

```groovy
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                echo 'Construction de l\'application...'
            }
        }
        stage('Test') {
            steps {
                echo 'Exécution des tests...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Déploiement de l\'application...'
            }
        }
    }
}
```

#### Pipeline scripté

Le pipeline scripté utilise la syntaxe Groovy, plus flexible mais nécessitant plus d'expérience en programmation :

```groovy
node {
    stage('Build') {
        echo 'Construction de l\'application...'
    }
    stage('Test') {
        echo 'Exécution des tests...'
    }
    stage('Deploy') {
        echo 'Déploiement de l\'application...'
    }
}
```

## Créer votre premier pipeline

### Utiliser Blue Ocean

Blue Ocean est l'interface moderne de Jenkins qui rend la création de pipelines plus intuitive :

1. Installez le plugin Blue Ocean (s'il n'est pas préinstallé)
2. Cliquez sur "Ouvrir Blue Ocean"
3. Cliquez sur "Créer un pipeline"
4. Sélectionnez l'emplacement du dépôt de code (GitHub, GitLab, etc.)
5. Sélectionnez le dépôt et la branche
6. Blue Ocean détectera automatiquement le Jenkinsfile ou aider à en créer un

### Utiliser Jenkinsfile

Créez un Jenkinsfile dans votre projet, et Jenkins le reconnaîtra automatiquement :

1. Créez Jenkinsfile à la racine du projet
2. Définissez les étapes du pipeline
3. Validez dans le contrôle de version
4. Créez une tâche pipeline dans Jenkins et liez-la au dépôt

## Gestion des identifiants

Jenkins doit accéder à diverses ressources externes comme les dépôts de code et les cibles de déploiement, ce qui nécessite une gestion des identifiants.

### Ajouter des identifiants

1. Allez dans "Manage Jenkins" > "Manage Credentials"
2. Cliquez sur "Add Credentials"
3. Choisissez le type d'identifiant (Nom d'utilisateur et mot de passe, SSH avec clé privée, Fichier secret, etc.)
4. Remplissez les informations d'identifiant
5. Définissez l'ID et la description pour l'identifiant

### Utiliser les identifiants dans le pipeline

Utilisez le pas withCredentials :

```groovy
pipeline {
    agent any
    stages {
        stage('Utiliser les identifiants') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "Nom d'utilisateur: $USER"
                }
            }
        }
    }
}
```

## Déclencheurs de build

Les builds automatisés nécessitent des déclencheurs. Jenkins supporte plusieurs méthodes de déclenchement.

### Sondage SCM

Vérifier régulièrement si le code source a changé :

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // Vérifier toutes les 5 minutes
    }
    stages {
        stage('Build') {
            steps {
                echo 'Changement de code détecté, démarrage du build'
            }
        }
    }
}
```

### Déclencheur Webhook

Déclencher les builds via les webhooks du dépôt pour une meilleure efficacité :

1. Installez le plugin correspondant (ex: GitHub Hook Trigger)
2. Configurez le webhook du dépôt pour pointer vers Jenkins
3. Le build se déclenche automatiquement lors de la validation du code

### Builds planifiés

Exécuter automatiquement selon un calendrier :

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // Exécuter à 2h du matin tous les jours
    }
    stages {
        stage('Build planifié') {
            steps {
                echo 'Build planifié en cours'
            }
        }
    }
}
```

## Utilisation des agents et conteneurs

### Configuration des agents

Jenkins supporte les builds distribués, distribuant les tâches de build sur plusieurs machines :

1. Allez dans "Manage Jenkins" > "Manage Nodes"
2. Ajoutez un nœud agent et configurez les informations de connexion
3. Spécifiez l'agent dans le pipeline

### Utiliser Docker dans le pipeline

Le pipeline déclaratif peut exécuter des builds dans des conteneurs Docker :

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## Rapports de tests

Jenkins peut s'intégrer avec divers frameworks de test pour générer des rapports de tests visuels.

### Rapports de tests JUnit

```groovy
pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### Autres rapports de tests

Jenkins supporte plusieurs plugins de rapports de tests :

- TestNG
- Cobertura (couverture de code)
- JaCoCo (couverture de code)
- ESLint (analyse de code frontend)

## Pratiques de déploiement

### Déployer vers différents environnements

Pipeline de déploiement multi-étapes typique :

```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('Déployer en test') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('Déployer en production') {
            when {
                branch 'main'
            }
            steps {
                input message: 'Confirmer le déploiement en production ?', ok: 'Déployer'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### Déploiement blue-green

Implémentez le déploiement blue-green avec Jenkins pour des mises à jour sans temps d'arrêt :

1. Maintenez deux environnements identiques (bleu et vert)
2. Basculez le trafic vers la nouvelle version
3. Revenez en arrière rapidement si des problèmes surviennent

## Bonnes pratiques

### Bonnes pratiques pour Jenkinsfile

- Garder Jenkinsfile dans le contrôle de version
- Utiliser la syntaxe du pipeline déclaratif
- Garder les pipelines simples et clairs
- Diviser les étapes de manière raisonnable
- Ajouter les mécanismes de timeout et de retry nécessaires

### Recommandations de sécurité

- Mettre à jour régulièrement Jenkins et les plugins
- Limiter les permissions des utilisateurs anonymes
- Utiliser les identifiants pour les informations sensibles
- Configurer un contrôle d'accès approprié
- Activer l'audit de sécurité

### Optimisation des performances

- Configurer correctement les nœuds de build
- Utiliser les builds incrémentaux pour réduire le temps
- Configurer le cache de build
- Nettoyer régulièrement les anciens enregistrements de build
- Utiliser les étapes parallèles pour accélérer les builds

## Dépannage

### Voir les journaux de build

Lorsqu'un build échoue, commencez par vérifier les journaux de build :

1. Allez sur la page de la tâche
2. Cliquez sur le numéro de build qui a échoué
3. Vérifiez la sortie console

### Résoudre les problèmes courants

- Build bloqué : Vérifier s'il y a une entrée en attente ou des blocages de ressources
- Identifiants invalides : Vérifier si les identifiants ont expiré ou ont été révoqués
- Conflits de plugins : Essayer de désactiver les plugins récemment installés
- Mémoire insuffisante : Ajuster les paramètres du heap JVM

Le pipeline Jenkins est l'outil principal pour implémenter CI/CD. Grâce à ce manuel, vous devriez être en mesure de :

- Comprendre les concepts fondamentaux du pipeline
- Créer et configurer des tâches pipeline
- Gérer les identifiants et les déclencheurs
- Intégrer les outils de test et de déploiement
- Appliquer les bonnes pratiques pour optimiser les workflows

L'apprentissage et la pratique continus sont la clé pour maîtriser Jenkins. Continuez à explorer et à optimiser dans des projets réels.
