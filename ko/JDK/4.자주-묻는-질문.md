# JDK 자주 묻는 질문

## 기본 질문

### JDK란 무엇인가?

JDK(Java Development Kit)는 Java 프로그래밍 언어를 위한 소프트웨어 개발 키트입니다. Java 애플리케이션 개발에 필요한 소프트웨어 패키지입니다. JDK에는 Java 컴파일러(javac), Java 가상 머신(JVM), Java 핵심 클래스 라이브러리 및 다양한 개발 도구가 포함되어 있습니다. JDK 없이는 Java 프로그램을 개발하거나 실행할 수 없습니다.

JDK와 JRE의 차이: JRE(Java Runtime Environment)는 개발 도구 없이 Java 프로그램을 실행하는 데 필요한 환경만 포함합니다. 다른 사람이 개발한 Java 프로그램만 실행하려면 JRE를 설치하세요; 직접 Java 프로그램을 개발하려면 JDK를 설치해야 합니다.

### JVM이란 무엇인가?

JVM(Java Virtual Machine)은 Java 프로그램 실행의 핵심입니다. JVM은 Java 바이트코드(.class 파일)를 해석하거나 네이티브 코드 실행을 컴파일하는 역할을 합니다. JVM의 존재는 Java 프로그램이 "한 번 작성하면 어디서나 실행"이라는 특성을 달성할 수 있게 합니다.

JVM은 Java 프로그램만 실행하는 것이 아니라 메모리 관리(가비지 컬렉션), 스레드 관리, тоже 담당합니다. JVM의 작동 방식을 이해하는 것은고성능 Java 애플리케이션 작성에 필수적입니다.

### JDK와 JRE의 차이점은?

JDK(Java Development Kit)는 JRE와 개발 도구를 포함한 완전한 개발 패키지입니다. JRE(Java Runtime Environment)는 Java 프로그램을 실행하는 데 필요한 구성 요소만 포함하는 Java 실행 환경입니다.

단어로: 다른 사람이 개발한 Java 프로그램만 실행하려면 JRE 설치로 충분합니다; 직접 Java 프로그램을 개발하려면 JDK를 설치해야 합니다. JDK는 전체 JRE를 포함하므로 JDK를 설치하면 JRE를 별도로 설치할 필요가 없습니다.

### 어떤 JDK 버전이 가장 좋은가?

절대적으로 "가장 좋은" 버전은 없습니다; 선택은 특정 요구 사항에 따라 다릅니다. 새 프로젝트의 경우 JDK 21(최신 LTS 버전) 또는 JDK 25(최신 버전)가 권장됩니다. 장기 유지 관리가 필요한 기업 프로젝트의 경우 JDK 21 또는 JDK 17과 같은 LTS 버전이 권장됩니다.

JDK 21은 최소 8년간 지원을 받을 최신 LTS 버전입니다. JDK 17은 여전히 널리 사용되며 무료입니다. JDK 8은 구식이지만 많은 레거시 시스템에서 여전히 사용됩니다.

## 설치 질문

### JDK가 올바르게 설치되었는지 확인하는 방법은?

명령줄에서 java -version을 실행하세요. 버전 정보가 표시되면 JDK가 올바르게 설치된 것입니다. javac -version을 실행하여 컴파일러가 사용 가능한지 확인하세요. echo %JAVA_HOME%(Windows) 또는 echo $JAVA_HOME(Linux/macOS)을 실행하여 환경 변수가 올바르게 구성되었는지 확인하세요.

이 명령이 명령줄에서 오류를 표시하면 먼저 JDK가 올바르게 설치되었는지 확인한 다음 환경 변수가 올바르게 구성되었는지 확인하세요.

### JAVA_HOME이란?

JAVA_HOME은 JDK 설치 디렉토리를 가리키는 환경 변수입니다. Maven, Gradle과 같은 많은 Java 도구 및 빌드 스크립트가 JDK를 찾기 위해 JAVA_HOME에 의존합니다. JAVA_HOME을 올바르게 설정하면 이러한 도구가 올바르게 작동할 수 있습니다.

Windows에서 JAVA_HOME은 일반적으로 C:\Program Files\Java\jdk-21과 같은 경로로 설정됩니다. Linux/macOS에서는 일반적으로 /usr/lib/jvm/java-21-openjdk와 같은 경로로 설정됩니다.

### Windows에 여러 JDK 버전을 어떻게 설치하나?

같은 컴퓨터에 여러 JDK 버전을 설치할 수 있습니다. 여러 버전을 설치한 후 환경 변수에서 PATH의 JDK bin 디렉토리 순서를 변경하거나 JAVA_HOME 환경 변수를 변경하여 다른 JDK 설치 디렉토리를 가리키게 하여 기본 버전을 변경합니다.

SDKMAN!의 Windows 버전 또는 JetBrains Toolbox와 같은 타사 도구를 사용하여 여러 JDK 버전을 관리할 수 있습니다. 이러한 도구는 버전 간 전환을 쉽게 합니다.

### Oracle JDK와 OpenJDK의 차이점은?

Oracle JDK와 OpenJDK는 코드 수준에서 거의 동일하며, 주요 차이점은 라이선스 계약과 지원 서비스에 있습니다. Oracle JDK는 상업적 지원 및 추가 엔터프라이즈 수준 기능을 제공하지만 일부 버전은 상업적 사용에 대해 비용이 발생할 수 있습니다. OpenJDK는 완전히 오픈소스이며 무료로 커뮤니티에서 유지 관리됩니다.

대부분의 사용 시나리오에서 OpenJDK는 전혀 충분합니다. 많은 조직들은 라이선스 문제를 피하면서 Oracle JDK와 거의 동일한 성능과 기능을 얻기 위해 OpenJDK 사용을 선택합니다.

## 사용 질문

### Classpath란?

Classpath는 JVM이 클래스 파일을 찾는 데 사용하는 경로입니다. java MyClass를 실행할 때 JVM은 Classpath로 지정된 디렉토리와 JAR 파일에서 My.class를 찾습니다.

Java 프로그램을 컴파일하고 실행할 때 -classpath 또는 -cp 옵션을 사용하여 Classpath를 지정할 수 있습니다. 지정되지 않으면 JVM은 현재 디렉토리를 Classpath로 사용합니다.

### 바이트코드란?

바이트코드는 Java 컴파일러가 소스 코드에서 컴파일한 중간 형태의 코드입니다. 바이트코드는 네이티브 코드가 아니며 CPU에서 직접 실행할 수 없습니다; JVM이 해석하거나 JIT 컴파일하여 실행해야 합니다.

바이트코드의 존재는 Java 프로그램에 플랫폼 독립성 기능을 부여합니다. 동일한 바이트코드는 JVM이 설치된 모든 시스템(Windows, Linux 또는 macOS)에서 실행될 수 있습니다.

### JAR 파일이란?

JAR(Java Archive)는 여러 Java 클래스 파일을 단일 파일로 패키징하는 형식입니다. JAR 파일은 기본적으로 컴파일된 .class 파일과 메타데이터를 포함하는 ZIP 형식의 압축 파일입니다.

jar 도구를 사용하여 JAR 파일 생성, 보기 및 추출할 수 있습니다. 실행 가능한 JAR 파일은 기본 클래스를 지정하는 MANIFEST.MF 파일을 포함하며 java -jar로 직접 실행할 수 있습니다.

### 모듈 시스템이란?

Java 모듈 시스템(Project Jigsaw)은 JDK 9에서 도입되어 JDK와 애플리케이션을 모듈로 분할합니다. 모듈은 더 나은 캡슐화를 제공하고 종속성을 명확하게 선언하여 더 강력하고 유지 관리하기 쉬운 애플리케이션 구축에 도움이 됩니다.

모듈은 module-info.java 파일을 통해 선언되며, 모듈 이름, 다른 모듈에 대한 종속성, 내보내기 패키지 등을 정의합니다. 모듈 시스템은 Classpath 충돌 해결에 도움이 되며 더 나은 버전 관리를 제공합니다.

## 성능 질문

### JVM 성능을 어떻게 최적화합니까?

JVM 성능 최적화는 주로 다음을 기반으로 합니다: 힙 메모리 크기 올바르게 설정, 너무 크거나 설정 피하기; 적절한 가비지 컬렉터 선택, G1이 대부분의 경우 좋은 선택; 저지연이 필요한 애플리케이션의 경우 ZGC 또는 Shenandoah 사용 고려; 클래스 데이터 공유 활성화(-Xshare:classes)로 시작 시간 단축.

VisualVM, JConsole과 같은 성능 모니터링 도구를 사용하여 애플리케이션 성능을 분석하고 병목 현상을 찾은 후 목표 지향적 최적화를 수행하세요.

### 가비지 컬렉션이란?

가비지 컬렉션(GC)은 JVM이 자동으로 메모리를 관리하는 메커니즘입니다. 프로그램 실행 중 개체가 생성되어 메모리를 소비합니다. 더 이상 참조되지 않으면 가비지 컬렉터가 자동으로 해당 개체가 점유한 메모리를 식별하고 해제합니다.

Java의 가비지 컬렉션은 메모리을 크게 단순화합니다; 프로그래머는 명시적으로 메모리를 해제할 필요가 없습니다. 그러나 가비지 컬렉션의 원리를 이해하면 메모리 누수와 빈번한 GC를 피하면서 더 효율적인 코드 작성에 도움이 됩니다.

### 메모리 누수란?

메모리 누수는 프로그램 실행 중 회수할 수 없는 메모리를하고, 사용 가능한 메모리가 점진적으로 감소함을 의미합니다. Java에서 메모리 누수는 일반적으로 제때 정리하지 않고 지속적으로 개체를 추가하는 컬렉션(예: 정적 Map, List)으로 인해 발생합니다.

메모리 누수를 조사하려면 jmap을 사용하여 힙 덤프 파일을 생성한 다음 MAT(Memory Analyzer Tool)와 같은 도구를 사용하여 분석하고 원인 개체를 찾을 수 있습니다.

### JIT 컴파일러란?

JIT(Just-In-Time) 컴파일러는 JVM의 일부로, 실행 성능을 개선하기 위해 hot 바이트코드를 네이티브 코드로 컴파일합니다. 런타임에 JVM은 코드 실행을 분석합니다; 자주 실행되는 코드(핫스팟)에 대해 JIT 컴파일러는 네이티브 기계 코드로 컴파일하여 성능을 크게 향상시킵니다.

JIT 컴pililation은 추가 컴pililation 단계 없이 런타임에 발생합니다. C1 컴파일러는 빠르게 컴파일하지만 최적화가 적고, C2 컴파일러는 느리게 컴파일하지만 더 많이 최적화합니다. 다단계 컴pililation은 두 가지의 장점을 결합합니다.

## 버전 질문

### 어떤 Java 버전을 사용해야 합니까?

새 프로젝트의 경우 최신 LTS 버전 JDK 21이 권장됩니다. 이는 좋은 성능, 풍부한 기능 및 장기 지원을 제공합니다. Java 8/11과의 호환성이 필요한 프로젝트의 경우 해당 버전을 사용할 수 있습니다.

지원이 중단된 버전(유료 구독이 없는 JDK 8 등) 사용은 권장되지 않습니다. 새로운 JDK 버전으로 적시에 업그레이드하면 보안 업데이트와 성능 개선을 제공합니다.

### Java 버전 번호에서 LTS는 무엇을 의미합니까?

LTS(Long-Term Support)는 장기 지원 버전을 의미합니다. LTS 버전은 Oracle에서 최소 8년간 기술 지원 및 패치 업데이트를 받습니다. 현재 LTS 버전에는 JDK 8, JDK 11, JDK 17, JDK 21이 포함됩니다.

비 LTS 버전(예: JDK 25)은 더 짧은 지원 주기가 있으며 일반적으로 6개월만 지원합니다. 이러한 버전은 최신 기능을 원하는 개발자에게 적합하지만 프로덕션 환경에는 LTS 버전이 권장됩니다.

### Java 17을 계속 사용할 수 있습니까?

Java 17은 성숙한 LTS 버전이며 정상적으로 계속 사용할 수 있습니다. Oracle은 2024년 10월 JDK 17 라이선스를 변경했습니다. 그러나 Oracle의에 따르면 JDK 17은 여전히 개발, 개인 사용 및 Oracle 라이선스 제품 실행에 무료로 사용할 수 있습니다.

아직 많은 기업들이 JDK 17을 사용하고 있습니다. 이는 매우 성숙하고 안정적인 버전으로 많은 커뮤니티 지원 리소스를 가지고 있기 때문입니다. 프로젝트가 이미 JDK 17을 사용하고 있으면 계속 사용할 수 있습니다; 새 프로젝트에는 JDK 21이 더 권장됩니다.

## 개발 질문

### Java 파일을 어떻게 컴파일합니까?

javac 명령을 사용하여 Java 파일을 컴파일합니다. 기본 사용: javac MyClass.java. 컴pililation 후 My.class 바이트코드 파일이 생성됩니다.

프로젝트의 경우 일반적으로 Maven 또는 Gradle과 같은 빌드 도구를 사용하여 컴파일 프로세스를 관리합니다. 이러한 도구는 종속성과 컴파일 옵션을 자동으로 처리합니다.

### Java 프로그램을 어떻게 실행합니까?

컴파일된 Java 프로그램을 실행하려면 java 명령을 사용합니다. 기본 사용: java MyClass. 여기서 .class 확장자는 필요하지 않습니다.

프로그램이 JAR 파일로 패키징된 경우 java -jar MyApp.jar로 실행합니다.

### Classpath를 어떻게 설정합니까?

실행 시 -classpath 또는 -cp를 사용하여 Classpath 지정: java -cp lib/*:out MyClass. Windows에서는 세미콜론(;)이 구분 기호이고 Linux/macOS에서는 콜론(:)입니다.

CLASSPATH 환경 변수를 통해 구성할 수도 있지만 명령줄 지정이 더 유연합니다.

### Maven과 Gradle이란?

Maven과 Gradle은 Java 프로젝트용 빌드 도구입니다. 종속성을 자동으로 다운로드하고, 코드를 컴파일하고, 테스트를 실행하고, 배포를하여 프로젝트 빌드 프로세스를 크게 단순화합니다.

Maven은 프로젝트 구성을 정의하기 위해 XML(pom.xml)을 사용하는 반면, Gradle은 빌드 로직을 정의하기 위해 Groovy 또는 Kotlin 스크립트(build.gradle)를 사용합니다. 둘 다 Java 프로젝트에서 널리 사용되는 빌드 도구입니다.
