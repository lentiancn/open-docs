# Introducción a Node.js

Node.js® es un entorno de ejecución de JavaScript gratuito, de código abierto y multiplataforma que permite a los desarrolladores crear servidores, aplicaciones web, herramientas de línea de comandos y scripts.

## Características Principales

Como un entorno de ejecución de JavaScript orientado a eventos y asíncrono, Node.js está diseñado para construir aplicaciones de red escalables. En el ejemplo clásico de "hola mundo", se pueden manejar muchas conexiones simultáneamente. Al producirse cada conexión, se activa la función de callback, pero si no hay trabajo que realizar, Node.js entrará en estado de suspensión.

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

## Filosofía de Diseño

Esto contrasta con el modelo de concurrencia más común actualmente, que emplea hilos del sistema operativo. La programación en red basada en hilos es relativamente ineficiente y muy difícil de usar. Además, los usuarios de Node.js están libres de preocuparse por bloquear el proceso, ya que no existen bloqueos.

Casi ninguna función en Node.js realiza operaciones de E/S directamente, por lo que el proceso nunca se bloquea excepto cuando se utilizan métodos síncronos de la biblioteca estándar de Node.js. Debido a que nada se bloquea, es muy razonable desarrollar sistemas escalables en Node.js.

## Arquitectura Orientada a Eventos

Node.js es similar en diseño a sistemas como Event Machine de Ruby y Twisted de Python, y ha sido influenciado por ellos. Node.js lleva el modelo de eventos un paso más allá. Presenta un bucle de eventos como una construcción en tiempo de ejecución en lugar de como una biblioteca.

En otros sistemas, siempre existe una llamada de bloqueo para iniciar el bucle de eventos. Típicamente, el comportamiento se define mediante callbacks al principio de un script, y al final se inicia un servidor mediante una llamada de bloqueo como `EventMachine::run()`.

En Node.js, no existe tal llamada para iniciar el bucle de eventos. Node.js simplemente entra en el bucle de eventos después de ejecutar el script de entrada. Node.js sale del bucle de eventos cuando no hay más callbacks que ejecutar. Este comportamiento es similar al JavaScript del navegador: el bucle de eventos está oculto al usuario.

## Soporte HTTP

HTTP es un ciudadano de primera clase en Node.js, diseñado teniendo en cuenta la transmisión y la baja latencia. Esto hace que Node.js sea muy adecuado como base para una biblioteca o framework web.

## Soporte para Procesadores Multi-Núcleo

El hecho de que Node.js esté diseñado sin hilos no significa que no puedas aprovechar múltiples núcleos en tu entorno. Se pueden generar procesos secundarios utilizando nuestra API `child_process.fork()`, y están diseñados para ser fáciles de comunicar.

Construido sobre esa misma interfaz está el módulo `cluster`, que te permite compartir sockets entre procesos para habilitar el balanceo de carga sobre tus núcleos.
