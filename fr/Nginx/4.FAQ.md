# FAQ

#### Questions générales

**Qu'est-ce que nginx ?**

nginx (prononcé « engine x ») est un serveur web HTTP haute performance, un reverse proxy, un cache et un équilibrateur de charge. Il est connu pour sa flexibilité, ses performances élevées et sa faible consommation de ressources.

**Quelle est la licence de nginx ?**

nginx est distribué sous la licence BSD à 2 clauses. Cela signifie qu'il peut être utilisé librement dans des projets commerciaux et personnels.

**Quelle est la dernière version stable de nginx ?**

La version stable actuelle est nginx 1.26. Les versions stables sont publiées environ toutes les 6 semaines.

#### Installation

**Comment installer nginx sur Linux ?**

La méthode la plus simple est d'utiliser le gestionnaire de packages de votre distribution :

- Ubuntu/Debian : `sudo apt install nginx`
- CentOS/RHEL : `sudo yum install nginx`
- Fedora : `sudo dnf install nginx`

**Comment installer nginx depuis les sources ?**

Téléchargez les sources, installez les dépendances, puis compilez :

```bash
wget https://nginx.org/download/nginx-1.26.0.tar.gz
tar -xzvf nginx-1.26.0.tar.gz
cd nginx-1.26.0
./configure --prefix=/usr/local/nginx
make && sudo make install
```

**Comment utiliser nginx avec Docker ?**

```bash
docker run -d -p 80:80 --name nginx nginx:latest
```

#### Configuration

**Où est le fichier de configuration nginx ?**

Le fichier de configuration par défaut se trouve à :

- /etc/nginx/nginx.conf (Linux)
- /usr/local/nginx/conf/nginx.conf (compilation depuis les sources)

**Comment tester ma configuration ?**

Utilisez la commande `nginx -t` pour tester la syntaxe et la validité de votre configuration.

**Comment recharger la configuration sans arrêter nginx ?**

Utilisez `nginx -s reload` pour recharger la configuration sans interruption de service.

#### Problèmes courants

**Erreur 403 Forbidden**

Cette erreur peut survenir pour plusieurs raisons :

- Permissions incorrectes sur les fichiers ou répertoires
- Index manquant dans le répertoire
- Directive deny explicite dans la configuration

Solution : Vérifiez les permissions et ajoutez un fichier index.

**Erreur 404 Not Found**

Le fichier demandé n'existe pas ou le chemin dans la directive root est incorrect.

Solution : Vérifiez le chemin dans la directive root et l'existence du fichier.

**Erreur 413 Request Entity Too Large**

La taille de la requête dépasse la limite configurée.

Solution : Augmentez la valeur de `client_max_body_size` dans votre configuration.

**Erreur 504 Gateway Timeout**

Le serveur proxy met trop de temps à répondre.

Solution : Augmentez les valeurs de `proxy_connect_timeout`, `proxy_send_timeout` et `proxy_read_timeout`.

**Comment résoudre les problèmes de performance ?**

- Augmentez `worker_connections` dans le bloc events
- Activez `gzip` pour compresser les réponses
- Configurez la mise en cache du navigateur
- Utilisez `worker_processes auto` pour utiliser tous les cœurs CPU

#### Sécurité

**Comment configurer SSL/TLS ?**

Utilisez les directives `ssl_certificate` et `ssl_certificate_key` dans votre bloc server. Il est recommandé d'utiliser TLS 1.2 ou supérieur et des chiffrements sécurisés.

**Comment prévenir les hotlinks ?**

Utilisez la directive `valid_referers` pour vérifier le referer :

```nginx
location ~* \.(jpg|png|gif)$ {
    valid_referers none blocked example.com;
    if ($invalid_referer) {
        return 403;
    }
}
```

**Comment limiter le taux de requêtes ?**

Utilisez le module `limit_req` :

```nginx
limit_req_zone $binary_remote_addr zone=limiter:10m rate=10r/s;
```

#### Avancé

**Qu'est-ce que le équilibrage de charge ?**

L'équilibrage de charge distribue le trafic réseau sur plusieurs serveurs pour optimiser les performances et la disponibilité.

**Comment configurer un cluster nginx ?**

Vous pouvez configurer plusieurs blocs server dans un upstream pour créer un cluster avec équilibrage de charge et haute disponibilité.

**nginx supporte-t-il HTTP/2 et HTTP/3 ?**

Oui, nginx supporte HTTP/2 (à partir de la version 1.9.5) et HTTP/3 (à partir de la version 1.25.0).
