# OpenClaw よくある質問

このFAQでは、OpenClawの使用における一般的な問題と解決策をまとめています。

## インストール

### OpenClawの動作環境は？

- Node.js 22.x 以上
- macOS、Linux、Windows (WSL2)
- 推奨メモリ2GB以上

### インストールに失敗した場合は？

1. Node.jsバージョンを確認：`node --version`（22以上）
2. ネットワーク接続を確認
3. 異なるパッケージマネージャーを試す：npm、pnpm、bun
4. `openclaw doctor`を実行して診断

### OpenClawを更新するには？

```bash
# ステーブルチャンネルに切り替え
openclaw update --channel stable

# Gatewayを再起動
openclaw gateway restart
```

## Gateway

### Gatewayが起動しない

- ポート18789が占有されていないか確認
- ログを確認：`openclaw gateway --verbose`
- 既存のプロセスを終了试着

### Gatewayを常に実行し続けるには？

`--install-daemon`を使用してシステムサービスとしてインストール：

```bash
openclaw onboard --install-daemon
```

### Gatewayにリモートアクセスするには？

1. **Tailscale**：`gateway.tailscale.mode: "serve"` または `"funnel"`を使用
2. **SSHトンネル**：SSHトンネル接続を確立

## チャネル

### WhatsAppログインに失敗

- `openclaw channels login`でデバイスをペアリングしていることを確認
- ネットワークがWhatsAppサーバーにアクセスできるか確認

### Telegram Botが動作しない

- Botトークンが正しいか確認
- Botとの会話を開始しているか確認
- `allowFrom`設定を確認

### Discordメッセージが反応しない

- Botがサーバーに参加しているか確認
- 正しい権限が設定されているか確認
- チャンネルが.allowlistにあるか確認

## セキュリティ

### 見知らぬ人がメッセージを送れますか？

デフォルトでは、OpenClawは未知の送信者にペアリング検証を要求します。`dmPolicy: "open"`でオープンモードに変更できますが、お勧めしません。

### ペアリングの管理方法：

```bash
# ペアリングリストを表示
openclaw pairing list

# ペアリングを承認
openclaw pairing approve

# ペアリングを取り消し
openclaw pairing revoke
```

### `openclaw doctor`が警告を表示した場合は？

指示に従って設定を修正してください。常见の問題：
- DMポリシーが開きすぎている
- 必要な認証設定が欠けている
- ポート露出のリスク

## スキル

### 新しいスキルをインストールするには？

```bash
# ClawHub経由でインストール
openclaw skill install スキル名

# または手動でスキルディレクトリとSKILL.mdを作成
```

### スキルが動作しない場合は？

- スキル設定が正しいか確認
- Gatewayログを確認
- スキルの依存関係がインストールされているか確認

## 音声

### Voice Wakeが動作しない？

- macOS：システム環境設定のアクセシビリティ権限を確認
- iOS：マイク権限が有効になっているか確認
- ウェイクワードが正しく設定されているか確認

### 音声応答が遅い？

- ネットワーク遅延を確認
- ローカルTTSの使用を検討
- 音声モデル設定を調整

## パフォーマンス

### OpenClawがメモリを多量に使用する？

- セッション履歴を定期的にクリーンアップ
- モデル設定を調整
- セッション圧縮を使用

### メッセージの応答が遅い？

- ネットワーク接続を確認
- より速いモデルに切り替えを検討
- Gatewayの負荷を確認

## データ

### データはどこに保存されますか？

- 設定：`~/.openclaw/`
- ワークスペース：`~/.openclaw/workspace`
- 認証情報：`~/.openclaw/credentials`

### データをバックアップするには？

```bash
# 設定ディレクトリ全体をバックアップ
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### すべての設定をリセットするには？

1. Gatewayを停止
2. `~/.openclaw`ディレクトリを削除
3. `openclaw onboard`を再実行

## その他

### ログを表示するには？

```bash
# Gatewayログを表示
openclaw gateway --verbose

# または特定のログファイルを確認
openclaw logs
```

### ヘルプを得るには？

- ドキュメント：https://docs.openclaw.ai
- Discordコミュニティに参加
- GitHubでIssueを送信

### OpenClaw与其他AIアシスタントの違いは？

- **完全にローカル**：データはクラウドにアップロードされず、ローカルに保存
- **オープンソース**：コードは完全にオープンで、自分でレビュー可能
- **高いカスタマイズ性**： любую機能を変更して拡張できる
- **マルチチャンネル統一**：1つのアシスタントがすべてのコミュニケーションチャネルを管理

### 複数のOpenClawインスタンスを同時に実行できますか？

可能ですが、各インスタンスには以下が必要です：
- 異なるポート
- 独立したワークスペース
- 異なる設定
