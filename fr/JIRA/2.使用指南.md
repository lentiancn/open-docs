# Guide utilisateur JIRA

Ce guide explique comment utiliser JIRA pour la gestion de projet et le suivi des problèmes.

## Démarrage rapide

### Créer un projet

1. Cliquez sur "Créer un projet"
2. Sélectionnez un modèle de projet
3. Saisissez le nom et la clé du projet
4. Cliquez sur "Créer"

## Types de projets

### Projets agiles

```markdown
- Projets Scrum
- Projets Kanban
- Agile à l'échelle
```

### Projets classiques

```markdown
- Projets métier
- Gestion de projet
- Suivi des tâches
```

## Types de problèmes

### Créer un problème

```markdown
1. Cliquez sur "Créer un problème"
2. Sélectionnez le projet
3. Sélectionnez le type de problème
4. Remplissez le titre et la description
5. Définissez la priorité et les composants
6. Assignez à un utilisateur
```

### Types de problèmes

- **Bug** - Défaut logiciel
- **Story** - Histoire utilisateur
- **Task** - Tâche
- **Epic** - Épopée
- **Sub-task** - Sous-tâche

## Flux de travail

### Flux de travail par défaut

```
Créer → Assigner → Résoudre → Fermer
```

### Flux de travail agile

```
À faire → En cours → Terminé → Révisé
```

## Fonctionnalités agiles

### Kanban

1. Créez un Kanban
2. Ajoutez des colonnes
3. Glissez les cartes de problèmes
4. Définissez les limites WIP

### Scrum

```markdown
- Planification du sprint
- Daily standup
- Revue de sprint
- Rétrospective de sprint
```

## Champs

### Champs personnalisés

1. Allez dans Administration → Champs
2. Ajoutez un champ personnalisé
3. Configurez les options du champ

### Configuration des champs

```markdown
- Champs obligatoires
- Valeurs par défaut
- Permissions de champ
```

## Permissions

### Permissions du projet

- Parcourir le projet
- Créer des problèmes
- Modifier des problèmes
- Supprimer des problèmes
- Assigner des problèmes

### Niveaux de sécurité des problèmes

- Public
- Membres du projet
- Utilisateurs spécifiques
- Groupes spécifiques

## Automatisation

### Créer une règle

1. Allez dans Paramètres du projet → Automatisation
2. Créez une règle
3. Configurez les déclencheurs
4. Définissez les actions

### Exemple d'automatisation

```
Déclencheur : Création de problème
Condition : Type = Bug
Action : Assigner à = Responsable
```
