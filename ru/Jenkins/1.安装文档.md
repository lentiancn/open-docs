# Руководство по установке Jenkins

Jenkins — это сервер автоматизации с открытым исходным кодом для реализации непрерывной интеграции и непрерывного развёртывания (CI/CD). В этом руководстве описаны методы установки в различных средах.

## Информация о версиях

| Версия | Статус | Описание |
|---------|--------|----------|
| Jenkins 2.x | ✅ Текущая | Последняя стабильная версия |
| Jenkins 1.x | ❌ Устаревшая | Поддержка прекращена |

## Системные требования

### Аппаратные требования

| Требование | Минимум | Рекомендуется |
|------------|---------|---------------|
| CPU | 1 ядро | 2+ ядра |
| Оперативная память | 1 ГБ | 4+ ГБ |
| Диск | 10 ГБ | 50+ ГБ |

### Программные требования

| Программное обеспечение | Версия |
|-------------------------|--------|
| Java | JDK 8 или выше |
| Операционная система | Windows/macOS/Linux |

## Способы установки

### Способ 1: Установка в Windows

#### 1. Использование установщика

1. Скачать Jenkins: https://www.jenkins.io/download/
2. Запустить установщик jenkins.msi
3. Следовать инструкциям мастера установки

#### 2. Использование Chocolatey

```powershell
choco install jenkins
```

#### 3. Ручная установка

```powershell
# Скачать WAR-файл
# Распаковать в указанный каталог
java -jar jenkins.war
```

### Способ 2: Установка в macOS

#### 1. Использование Homebrew

```bash
# Установить Jenkins
brew install jenkins

# Запустить службу
brew services start jenkins

# Остановить службу
brew services stop jenkins
```

#### 2. Ручная установка

```bash
# Скачать WAR-файл
wget https://get.jenkins.io/jenkins.war

# Запустить
java -jar jenkins.war
```

### Способ 3: Установка в Linux

#### Ubuntu/Debian

```bash
# Добавить ключ
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -

# Добавить репозиторий
sudo sh -c 'echo deb https://pkg.jenkins.io/debian binary/ > /etc/apt/sources.list.d/jenkins.list'

# Обновить и установить
sudo apt-get update
sudo apt-get install jenkins

# Запустить службу
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

#### CentOS/RHEL

```bash
# Добавить репозиторий
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat/jenkins.repo

# Импортировать ключ
sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

# Установить
sudo yum install jenkins

# Запустить службу
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

### Способ 4: Установка через Docker

```bash
# Запустить контейнер Jenkins
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts
```

#### Docker Compose

```yaml
version: '3'
services:
  jenkins:
    image: jenkins/jenkins:lts
    ports:
      - "8080:8080"
    volumes:
      - jenkins_home:/var/jenkins_home
```

### Способ 5: Развёртывание WAR-файла

```bash
# Скачать WAR-файл
wget https://get.jenkins.io/war-stable/latest/jenkins.war

# Запустить
java -jar jenkins.war --httpPort=8080
```

## Настройка после установки

### Первый доступ

1. Открыть браузер по адресу http://localhost:8080
2. Получить начальный пароль администратора:
   ```bash
   # Linux/macOS
   sudo cat /var/lib/jenkins/secrets/initialAdminPassword
   
   # Docker
   docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
   ```
3. Установить рекомендуемые плагины
4. Создать учётную запись администратора

### Настройка русского интерфейса

1. Перейти в Manage Jenkins → Manage Plugins
2. Установить "Localization: Russian"
3. Перезапустить Jenkins

## Часто задаваемые вопросы

### Не запускается?

Проверить установку Java:

```bash
java -version
```

### Порт занят?

Изменить порт:

```bash
java -jar jenkins.war --httpPort=8090
```

### Ошибка установки плагинов?

Проверить сетевое подключение или использовать прокси.
