# OpenCode 사용자 매뉴얼

OpenCode 기본 사용법 및 기술.

## 빠른 시작

### 질문하기

OpenCode에 코드베이스를 설명해달라고 요청할 수 있습니다:

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### 기능 추가

기능을 추가하기 전에 먼저 계획을 작성하도록 요청하는 것이 좋습니다:

1. **Tab**을 눌러 Plan 모드로 전환
2. 원하는 기능 설명
3. 계획을 확인한 후 피드백이나 세부 사항 추가
4. 만족하면 **Tab**을 눌러 Build 모드로 돌아와서 실행

### 직접 변경

더 직접적인 변경을 요청할 수 있습니다:

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### 변경 취소

변경에 만족하지 않으면 취소 명령 사용:

```bash
/undo
```

OpenCode가 변경을 되돌리고 원래 메시지를 다시 표시합니다. 프롬프트를 조정하고 다시 요청할 수 있습니다.

### 다시 실행

`/redo`를 사용하여 변경을 다시 실행할 수 있습니다.

## 세션 공유

```bash
/share
```

이렇게 하면 현재 세션에 대한 링크가 생성되어 클립보드에 복사됩니다.

## 일반 명령

| 명령 | 기능 |
|------|------|
| /init | 프로젝트 초기화 |
| /connect | LLM 제공자 연결 |
| /undo | 변경 취소 |
| /redo | 다시 실행 |
| /share | 세션 공유 |
