# Docker よくある質問

このページでは，Docker のインストールと使用に関する一般的な問題と解決策を説明します。

## インストール問題

### Docker Desktop が起動しない

**症状**: Docker Desktop が起動しない，または起動後すぐにクラッシュする

**解決策**:

1. システムがハードウェア要件を満たしていることを確認：
   - Windows: Hyper-V を有効にするか WSL 2 をインストール
   - Mac: 古い Mac モデルを使用していないことを確認

2. Docker Desktop をリセット：
   - Windows: Docker アイコンを右クリック → トラブルシューティング → 工場設定にリセット
   - Mac: Docker → トラブルシューティング → 工場設定にリセット

3. システムログを確認：
   - Windows: イベントビューア → Windows ログ → アプリケーション
   - Mac: コンソールを開いて Docker エラーを検索

### Docker コマンドの権限がない

**症状**: "permission denied while trying to connect to the Docker daemon socket"

**解決策**:

1. ユーザーを Docker グループに追加：
   ```bash
   sudo usermod -aG docker $USER
   ```

2. ログアウトしてログインし直す

3. または Docker コマンドの前に sudo を使用

### Windows で WSL 2 インストールエラー

**症状**: "WSL 2 installation is incomplete"

**解決策**:

1. Linux 用 Windows サブシステムを有効にする：
   ```bash
   wsl --install
   ```

2. コンピュータを再起動

3. WSL を最新バージョンに更新：
   ```bash
   wsl --update
   ```

## 使用問題

### コンテナが起動しない

**症状**: コンテナが「Exited」または「Created」状态的のまま

**解決策**:

1. コンテナログを確認：
   ```bash
   docker logs コンテナ名
   ```

2. 必要なすべてのポートが利用可能であることを確認

3. イメージの整合性を確認

### ポートが既に使用されている

**症状**: "port is already allocated"

**解決策**:

1. 使用中のポートを見つける：
   ```bash
   netstat -ano | findstr :8080
   ```

2. 他のサービスを停止するか，別のポートを使用

3. ポートマッピングを変更：
   ```bash
   docker run -p 8081:80 nginx
```

### ディスク容量の問題

**症状**: "no space left on device"

**解決策**:

1. 未使用のコンテナを削除：
   ```bash
   docker container prune
   ```

2. 未使用のイメージを削除：
   ```bash
   docker image prune -a
   ```

3. 未使用のボリウムを削除：
   ```bash
   docker volume prune
   ```

### ネットワークの問題

**症状**: コンテナが外部に接続できない

**解決策**:

1. DNS 設定を確認：
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

2. Docker ネットワークをリセット：
   ```bash
   docker network prune
   ```

3. ファイアウォール設定を確認

## パフォーマンスの問題

### コンテナの性能が悪い

**解決策**:

1. リソース制限を確認：
   ```bash
   docker stats
   ```

2. より多くの RAM を割り当て（Docker Desktop → 設定 → リソース）

3. Docker に SSD ではなく HDD を使用

### ディスク使用量が多い

**解決策**:

1. ビルドキャッシュを定期的にクリーン：
   ```bash
   docker builder prune
   ```

2. 古いバージョンのイメージを削除

3. ログサイズを制限

## セキュリティの問題

### コンテナを root として実行

**推奨事項**:

非特権ユーザーを使用：

```dockerfile
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser
USER appuser
```

### イメージの脆弱性

**解決策**:

1. 公式で最新のイメージを使用

2. イメージを定期的にスキャン：
   ```bash
   docker scan イメージ名
   ```

3. distroless イメージの使用を検討

## 上級者向け質問

### CI/CD で Docker を使用

**推奨事項**:

1. ビルドキットを使用してパラレルビルド
2. マルチステージビルドでより小さいイメージに
3. 依存関係を効果的にキャッシュ
4. クロスプラットフォームビルドに Docker Buildx を使用

### データ永続化

**推奨事項**:

1. 重要なデータには名前付きボリウムを使用
2. ボリウムを定期的にバックアップ
3. 開発にはバインドマウントを使用

### 複数の環境

**推奨事項**:

1. Docker Compose で環境変数を使用
2. .env ファイルを使用
3. 本番環境には Docker Swarm または Kubernetes を検討
