# 简介

Apache APISIX 是 Apache 软件基金会下的顶级项目，由 API7.ai 开发并捐赠。它是一个具有动态、实时、高性能等特点的云原生 API 网关。

APISIX 作为所有业务的流量入口，提供了丰富的功能特性，包括动态路由、动态上游、动态证书、A/B 测试、灰度发布、金丝雀发布、蓝绿部署、限速、防攻击、收集指标、监控报警、可观测性和服务治理等。

## 核心特性

### 高性能

APISIX 基于 Nginx 构建，继承了 Nginx 出色的性能表现。通过插件机制，业务逻辑与核心功能解耦，使得扩展更加灵活，同时保持了高效的处理能力。

### 完全动态

与传统 Nginx 需要重载配置不同，APISIX 支持完全动态的配置管理。所有路由、上游、插件等都可以通过管理 API 实时更新，无需重启服务即可生效。

### 云原生设计

APISIX 从设计之初就考虑了云原生场景，支持多种部署方式，包括 Docker、Kubernetes、Helm Chart 等。同时提供了 Ingress Controller，可以直接作为 Kubernetes 的入口网关使用。

### 插件生态

APISIX 拥有丰富的插件系统，涵盖了身份认证、限流限速、请求改写、响应处理、监控日志等各个方面。用户可以通过插件机制轻松扩展网关功能。

### 多协议支持

除了传统的 HTTP/HTTPS 协议，APISIX 还支持 gRPC、Dubbo、WebSocket、MQTT 等多种协议，能够满足微服务架构中不同场景的需求。

## 架构概述

APISIX 的架构分为数据面和控制面两部分：

**数据面**负责处理实际的请求转发，基于 Nginx 修改而来的 APISIX Core 实现了高性能的请求处理管道。

**控制面**通过 RESTful API 管理所有配置，支持多种配置持久化方式，包括 etcd、Consul 等。APISIX 还提供了 Dashboard，可通过 Web 界面进行可视化配置管理。

## 适用场景

- 微服务 API 统一入口
- Kubernetes Ingress Controller
- API 流量管理与监控
- 多租户 API 管理
- 服务网格边车代理
- AI 应用网关（LLM 和 AI 工作负载）

作为 Apache 软件基金会的顶级项目，APISIX 拥有活跃的社区和持续的版本更新，为企业级应用提供了可靠的选择。
