# 사용 매뉴얼

## Android 애플리케이션 설치 및 실행

Waydroid는 `waydroid app -h` 명령어를 사용하여 다양한 작업을 수행할 수 있습니다:

```bash
usage: waydroid app [-h] {install,remove,launch,list} ...

optional arguments:
  -h, --help            이 도움말 메시지를 표시하고 종료

subaction:
  {install,remove,launch,list}
    install             단일 패키지를 컨테이너에 푸시하고 설치
    remove              컨테이너에서 단일 앱 패키지 제거
    launch              단일 애플리케이션 시작
    list                설치된 애플리케이션 목록 표시
```

### Android 애플리케이션 설치

다음 명령어로 Android 애플리케이션을 설치할 수 있습니다:

```bash
waydroid app install xyz.apk
```

설치 후 데스크톱 애플리케이션 메뉴를 통해 애플리케이션을 실행할 수 있습니다.

애플리케이션의 Android 패키지 이름(`com.xxx.yyy`)을 알고 있다면 명령줄에서도 사용할 수 있습니다:

```bash
waydroid app launch com.foo.bar
```

### 참고 사항

인터넷에서 가끔 찾을 수 있는 apk 파일은 일반적으로 arm 지원만 제공하므로 x86_64에서는 작동하지 않습니다.

그래픽 방식으로 애플리케이션을 얻으려면 [F-Droid](https://f-droid.org)를 설치하는 것이 좋습니다. 또는 GAPPS 이미지를 설치한 경우 Play 스토어를 사용할 수 있습니다.

## 일반 명령어

### Waydroid 명령줄 옵션

Waydroid는 컨테이너와 세션을 제어하기 위한 다양한 명령줄 옵션을 제공합니다:

- `waydroid show-full-ui`: 전체 Android UI 표시
- `waydroid session start`: Waydroid 세션 시작
- `waydroid session stop`: Waydroid 세션 중지
- `waydroid container start`: Waydroid 컨테이너 시작
- `waydroid container stop`: Waydroid 컨테이너 중지
- `waydroid logcat`: Android 로그 보기
- `waydroid shell`: Android 셸 진입
- `waydroid prop set <property> <value>`: 시스템 속성 설정
- `waydroid bugreport`: 버그 리포트 로그 생성

### 시스템 속성 설정

`waydroid prop` 명령어를 통해 다양한 시스템 속성을 설정할 수 있습니다:

- `persist.waydroid.multi_windows`: 멀티 윈도우 모드 활성화
- 기타 속성은 공식 문서 참조

## 고급 기능

### 사용자 정의 이미지 사용

사용자 정의 이미지를 사용하려면 다음 단계를 따르세요:

1. [SourceForge](https://sourceforge.net/projects/waydroid/files/images/)에서 이미지를 다운로드하거나 사용자 정의 빌드된 `system.img` 및 `vendor.img` 사용
2. 이미지를 `/etc/waydroid-extra/images/` 디렉터리에 복사
3. Waydroid 재초기화: `sudo waydroid init -f`

### ADB 디버깅

Waydroid는 ADB 디버깅을 지원합니다. 기본적으로 ADB 보안 모드가 활성화되어 세션 시작 시 자동으로 연결되지 않습니다. 연결하려면 다음 명령어를 사용하세요:

```bash
waydroid adb connect
```

그런 다음 Android 측에서 인증을 제공하세요.

### 알림 전달

Waydroid는 Android 알림을 데스크톱 환경으로 전달하는 것을 지원하며, 호환되는 버전으로 Android 시스템 이미지를 업데이트해야 합니다.
