# Vue.js User Manual

## Application Instance

Every Vue application starts by creating a new application instance with the `createApp` function:

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* root component options */
})
```

### Root Component
The object passed into `createApp` is a component. Every app requires a "root component" that can contain other components as its children.

### Mounting the App
An application instance won't render anything until its `.mount()` method is called:

```javascript
app.mount('#app')
```

### App Configurations
The application instance exposes a `.config` object for app-level configuration:

```javascript
// App-level error handler
app.config.errorHandler = (err) => {
  /* handle error */
}

// Register global component
app.component('TodoDeleteButton', TodoDeleteButton)
```

### Multiple Application Instances
You can create multiple Vue applications on the same page:

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## Template Syntax

Vue uses an HTML-based template syntax that allows you to declaratively bind the rendered DOM to the underlying component instance's data.

### Text Interpolation
The most basic form of data binding uses "Mustache" syntax (double curly braces):

```html
<span>Message: {{ msg }}</span>
```

### Raw HTML
Use the `v-html` directive to output real HTML:

```html
<p v-html="rawHtml"></p>
```

**Security Warning**: Dynamically rendering arbitrary HTML can lead to XSS vulnerabilities. Only use `v-html` on trusted content and never on user-provided content.

### Attribute Bindings
Use the `v-bind` directive to bind HTML attributes:

```html
<!-- Full syntax -->
<div v-bind:id="dynamicId"></div>

<!-- Shorthand -->
<div :id="dynamicId"></div>

<!-- Same-name shorthand (Vue 3.4+) -->
<div :id></div>
```

### Boolean Attributes
Boolean attributes are included when the bound value is truthy:

```html
<button :disabled="isButtonDisabled">Button</button>
```

### Dynamically Binding Multiple Attributes
Bind multiple attributes from an object using `v-bind` without an argument:

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript Expressions

Vue supports full JavaScript expressions in all data bindings:

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### Calling Functions
Call component-exposed methods in binding expressions:

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### Restricted Globals Access
Template expressions only have access to a restricted list of globals (Math, Date, etc.). Add additional globals via `app.config.globalProperties`.

## Directives

Directives are special attributes with the `v-` prefix.

### Arguments
Some directives take arguments denoted by a colon:

```html
<!-- v-bind argument -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on argument -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### Dynamic Arguments
Use JavaScript expressions in directive arguments with square brackets:

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**Constraints**:
- Dynamic arguments must evaluate to a string or null
- Avoid complex expressions; use computed properties instead
- In DOM templates, avoid uppercase characters in dynamic keys

### Modifiers
Modifiers are special postfixes denoted by a dot:

```html
<form @submit.prevent="onSubmit">...</form>
```

## Reactivity Fundamentals

### Declaring Reactive State
Create reactive references with `ref()`:

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### Reactive Objects
Create reactive objects with `reactive()`:

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### Computed Properties
Create computed properties with `computed()`:

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### Watchers
Watch reactive data changes with `watch()`:

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// Watch single ref
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## Component Basics

### Defining Components
```javascript
// Define component
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Passing Props
```javascript
// Child component
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// Parent component
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### Emitting Events
```javascript
// Child component
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### Slots
```html
<!-- Child component -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- Parent component -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
