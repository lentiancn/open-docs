## 1. Introduction to Docker

### 1.1 What is Docker?

Docker is an open-source platform for developing, deploying, and running applications. Through containerization technology, Docker can package applications and their dependencies into lightweight, portable containers.

### 1.2 Docker Core Concepts

- **Image**: A read-only template used to create containers
- **Container**: A running instance of an image
- **Repository**: A place to store images
- **Dockerfile**: A script that defines how to build an image

## 3. Install on Windows

### 3.1 Method One: Docker Desktop (Recommended)

#### Step 1: Download Docker Desktop

1. Visit Docker website: https://www.docker.com/products/docker-desktop
2. Click "Download for Windows"
3. Run the downloaded installer `Docker Desktop Installer.exe`

#### Step 2: Install Docker Desktop

1. Double-click the installer
2. Check "Use WSL 2 instead of Hyper-V" (recommended)
3. Click "Install"
4. After installation, click "Close"

#### Step 3: Launch Docker Desktop

1. Find "Docker Desktop" in Start menu
2. First launch requires Docker Hub login (optional)
3. Wait for Docker engine to start

#### Step 4: Verify Installation

Open PowerShell or Command Prompt, execute:

```powershell
docker --version
docker-compose --version
docker run hello-world
```

### 3.2 Method Two: Using Winget

```powershell
winget install Docker.DockerDesktop
```

### 3.3 Method Three: Manual Installation (Legacy)

If Docker Desktop is not available, install Docker Engine:

1. Install WSL 2:
```powershell
wsl --install
```

2. Install Docker:
```powershell
# Add Docker GPG key
curl -fsSL https://download.docker.com/linux/wsl/docker-archive-keyring.gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Add Docker repository
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/wsl docker-20.10" | sudo tee /etc/apt/sources.list.d/docker.list

# Install Docker
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io
```

## 4. Install on macOS

### 4.1 Method One: Docker Desktop (Recommended)

#### Step 1: Download Docker Desktop

1. Visit https://www.docker.com/products/docker-desktop
2. Click "Download for Mac"
3. Select chip type:
   - Apple Silicon: "Apple Chip"
   - Intel: "Intel Chip"

#### Step 2: Install Docker Desktop

1. Double-click `Docker.dmg`

3. Launch Docker Desktop

#### Step 3: Verify Installation

Open Terminal, execute:

```bash
docker --version
docker-compose --version
docker run hello-world
```

### 4.2 Method Two: Using Homebrew

```bash
# Install Homebrew (if not present)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Docker
brew install --cask docker
```

### 4.3 Apple Silicon Notes

On M1/M2/M3 Mac:

1. Ensure you install Apple Silicon version of Docker Desktop
2. Most images already have arm64 version
3. To run x86 images, use:
```bash
docker run --platform linux/amd64 image-name
```

## 5. Install on Linux

### 5.1 Ubuntu / Debian

#### Step 1: Uninstall Old Versions

```bash
sudo apt remove docker docker-engine docker.io containerd runc
```

#### Step 2: Install Dependencies

```bash
sudo apt update
sudo apt install ca-certificates curl gnupg lsb-release
```

#### Step 3: Add Docker GPG Key

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
```

#### Step 4: Add Docker Repository

```bash
# Ubuntu
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Debian
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

#### Step 5: Install Docker

```bash
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### 5.2 CentOS / RHEL / Fedora

#### Step 1: Uninstall Old Versions

```bash
sudo yum remove docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine
```

#### Step 2: Install Dependencies

```bash
sudo yum install -y yum-utils
```

#### Step 3: Add Docker Repository

```bash
# CentOS/RHEL 9
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# Fedora
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
```

#### Step 4: Install Docker

```bash
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### 5.3 Arch Linux

```bash
sudo pacman -S docker docker-compose
```

### 5.4 openSUSE

```bash
sudo zypper install docker docker-compose
```

### 5.5 Start Docker

```bash
# Start Docker service
sudo systemctl start docker

# Enable on boot
sudo systemctl enable docker

# Verify installation
sudo docker run hello-world
```

## 7. Docker Desktop vs Docker Engine

### 7.1 Docker Desktop

Suitable for:
- Windows 10/11 Pro
- macOS

Features:
- Graphical interface
- Built-in Kubernetes
- Auto-update
- Free for personal and small business use

### 7.2 Docker Engine (Docker Server)

Suitable for:
- Linux
- Windows Server
- Cloud servers

Features:
- Command-line management
- Lighter weight
- Completely open source and free

### 7.3 WSL 2 Backend (Windows)

- Faster build speeds
- Better resource utilization
- Seamless integration with Linux distributions

## 8. Configure Docker

### 8.1 Configure Docker Registry Mirrors

Users in China should configure domestic mirrors:

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ]
}
```

#### Windows/macOS Configuration

1. Open Docker Desktop settings
2. Find "Docker Engine"
3. Add the above configuration
4. Click "Apply & Restart"

#### Linux Configuration

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 8.2 Configure Resource Limits

#### Docker Desktop Configuration

- Memory: Recommended 4GB+
- CPUs: Recommended 2+
- Disk: Recommended 64GB+

### 8.3 Configure Non-root User to Use Docker

```bash
# Create docker group
sudo groupadd docker

# Add current user to docker group
sudo usermod -aG docker $USER

# Re-login for changes to take effect
```

## 9. Uninstall Docker

### 9.1 Windows Uninstall

1. Open "Settings" > "Apps"
2. Find "Docker Desktop"
3. Click "Uninstall"

Or use command line:
```powershell
winget uninstall Docker.DockerDesktop
```

### 9.2 macOS Uninstall

1. Quit Docker Desktop
2. Delete Docker from Applications
3. Delete related files:
```bash
rm -rf ~/Library/Containers/com.docker.docker
rm -rf ~/Library/Application\ Support/Docker\ Desktop
rm -rf ~/.docker
```

### 9.3 Linux Uninstall

```bash
# Uninstall Docker
sudo apt remove docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Delete images, containers, etc.
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
```

## 10. FAQ

### Q1: Docker Desktop fails to start on Windows?

**Solutions**:
1. Ensure virtualization is enabled in BIOS
2. Ensure WSL 2 is correctly installed
3. Restart computer and try again
4. Check for conflicting VM software (VirtualBox, VMware)

---

### Q2: docker run permission denied?

**Solutions**:
```bash
sudo usermod -aG docker $USER
# Re-login
```

---

### Q3: Image download too slow?

**Solutions**:
1. Configure domestic registry mirrors (see section 8.1 above)
2. Use proxy

---

### Q4: Container running out of memory?

**Solutions**:
1. Increase Docker Desktop memory limit
2. Use `-m` parameter to limit container memory:
```bash
docker run -m 512m image-name
```

---

### Q5: How to view Docker logs?

**Linux**:
```bash
sudo journalctl -u docker -f
```

**Windows/macOS**:
Open Docker Desktop, click "Troubleshoot" > "View Logs"

---

### Q6: What's the difference between Docker and containers?

- **Docker** is the container platform
- **Container** is the instance that Docker runs
- Similar to the relationship between **virtual machine** and **VMware**

---

### Q7: How to clean up unused Docker resources?

```bash
# Clean unused images
docker image prune -a

# Clean unused containers
docker container prune

# Clean unused volumes
docker volume prune

# Clean build cache
docker builder prune

# Clean all unused resources
docker system prune -a
```

---

### Q8: How to run GUI programs in containers?

Use X11 socket forwarding:

```bash
docker run -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix image-name
```

---

### Q9: Docker taking up too much disk space?

```bash
# Check disk usage
docker system df

# Full cleanup
docker system prune -a --volumes
```
