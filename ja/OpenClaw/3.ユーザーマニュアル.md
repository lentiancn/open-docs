# OpenClaw ユーザーマニュアル

> OpenClawの使用に関する詳細なガイド

---

## はじめに

### 基本的使用法

OpenClawはいくつかの方法でを使用できます：

#### インタラクティブセッション

```bash
# インタラクティブな会話を開始
openclaw chat
```

#### コマンド実行

```bash
# 単一のコマンドを実行
openclaw exec "ファイルを作成帮我"

# より複雑なタスクを実行
openclaw exec --task "最新ニュースを調査"
```

### Gateway管理

ほとんどの操作にはGatewayサービスがアクティブである必要があります：

```bash
# Gatewayを起動
openclaw gateway start

# ステータスを確認
openclaw gateway status

# Gatewayを停止
openclaw gateway stop
```

## ファイル管理

### ファイルの読み取り

```bash
# readツールを使用
read /path/to/file.txt
```

### ファイルの書き込み

```bash
# 新規ファイルの作成または上書き
write --content "ファイルの内容" --path /path/to/file.txt
```

### ファイルの編集

```bash
# ファイル内の正確なテキスト置換
edit --path /path/to/file.txt --oldText "古いテキスト" --newText "新しいテキスト"
```

## ブラウザ制御

OpenClawはウェブブラウザを自動化できます：

### ブラウザの起動

```bash
# 分離ブラウザ start --profile openを起動
browserclaw

# Chrome拡張機能を使用
browser start --profile chrome
```

### ブラウザアクション

```bash
# URLに移動
browser navigate --url https://example.com

# スクリーンショットを撮影
browser screenshot --path /output/screenshot.png

# 要素をクリック
browser click --selector "#button-id"
```

## サブエージェント

### サブエージェントの作成

```bash
# 特定のタスク用の新しいサブエージェントを起動
subagents create --task "ドキュメント収集"

# サブエージェントを操縦
subagents steer --id agent-123 --message "次のタスク"

# サブエージェントを終了
subagents kill --id agent-123
```

## 設定とカスタマイズ

### パーソナリティのカスタマイズ

アシスタントのパーソナリティをカスタマイズするには`SOUL.md`を編集します：

```markdown
# SOUL.md - あなた是谁

## コアTruths

- 真に親切である
- 意見を持つ
- 資源が豊かである
```

### ツールの設定

特定のツール設定については`TOOLS.md`を編集します：

```markdown
# TOOLS.md - ローカルノート

## SSH接続
- server1 -> 192.168.1.100
```

## ベストプラクティス

1. **ワークスペース**：定義されたワークスペースにファイルを保管
2. **セキュリティ**：外部アクションに注意
3. **メモリ**：重要な情報にはメモリファイルを使用
4. **Gateway**：素早い応答時間のためにGatewayサービスをアクティブに保つ

## 高度な機能

### 自動化

反復的なタスクを自動化するためのスクリプトを作成：

```bash
# 自動データ収集
openclaw exec --task "トピックXに関する情報を収集"
```

### Cronジョブ

定期的なタスクをセットアップ：

```bash
# スケジュールタスクを作成
openclaw cron --schedule "0 9 * * *" --task "朝のブリーフィング"
```
