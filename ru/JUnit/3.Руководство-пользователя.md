# JUnit Руководство пользователя

Это руководство охватывает использование JUnit 4 и JUnit 5, включая написание тестов, проверки, тестовые наборы и другие основные функции.

## базовое использование JUnit 5

### Написание тестовых классов

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void testAddition() {
        int result = 2 + 3;
        assertEquals(5, result);
    }
}
```

### Часто используемые проверки

```java
import static org.junit.jupiter.api.Assertions.*;

// Проверки на равенство
assertEquals(expected, actual);
assertEquals(expected, actual, "Пользовательское сообщение об ошибке");

// Булевы проверки
assertTrue(condition);
assertFalse(condition);

// Проверки на null
assertNull(object);
assertNotNull(object);

// Проверки массивов
assertArrayEquals(expectedArray, actualArray);

// Проверки исключений
assertThrows(Exception.class, () -> {
    // Код, который может вызвать исключение
});

// Комбинированные проверки
assertAll("Свойства человека",
    () -> assertEquals("John", person.getFirstName()),
    () -> assertEquals("Doe", person.getLastName())
);
```

### Операции Before и After

```java
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;

class MyTest {
    
    @BeforeEach
    void setUp() {
        // Выполняется перед каждым тестовым методом
    }
    
    @AfterEach
    void tearDown() {
        // Выполняется после каждого тестового метода
    }
}
```

### Вложенные тесты

```java
import org.junit.jupiter.api.Nested;

@Nested
class NestedTests {
    
    @Test
    void testInNestedClass() {
        // Тесты во вложенном тестовом классе
    }
}
```

### Параметризованные тесты

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void testWithMultipleValues(int number) {
    assertTrue(number > 0);
}
```

### Динамические тесты

```java
import org.junit.jupiter.api.TestFactory;
import org.junit.jupiter.api.DynamicTest;
import java.util.Arrays;

@TestFactory
Collection<DynamicTest> dynamicTestsFromIterator() {
    return Arrays.asList(
        DynamicTest.dynamicTest("Первый динамический тест", () -> assertTrue(true)),
        DynamicTest.dynamicTest("Второй динамический тест", () -> assertEquals(4, 2 + 2))
    );
}
```

## базовое использование JUnit 4

### Написание тестовых классов

```java
import org.junit.Test;
import static org.junit.Assert.*;

public class CalculatorTest {
    
    @Test
    public void testAddition() {
        assertEquals(5, 2 + 3);
    }
}
```

### Часто используемые проверки

```java
// Проверки на равенство
assertEquals(expected, actual);
assertEquals("сообщение", expected, actual);

// Булевы проверки
assertTrue(condition);
assertFalse(condition);

// Проверки на null
assertNull(object);
assertNotNull(object);

// Проверки исключений
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}
```

### Before и After

```java
import org.junit.Before;
import org.junit.After;

public class MyTest {
    
    @Before
    public void setUp() {
        // Выполняется перед каждым тестовым методом
    }
    
    @After
    public void tearDown() {
        // Выполняется после каждого тестового метода
    }
    
    @BeforeClass
    public static void setUpClass() {
        // Выполняется один раз перед всеми тестами
    }
    
    @AfterClass
    public static void tearDownClass() {
        // Выполняется один раз после всех тестов
    }
}
```

### Тестовые наборы

```java
import org.junit.runner.RunWith;
import org.junit.runners.Suite;
import org.junit.runners.Suite.SuiteClasses;

@RunWith(Suite.class)
@SuiteClasses({Test1.class, Test2.class, Test3.class})
public class AllTests {
    // Запускает все тесты в Test1, Test2 и Test3
}
```

## Миграция с JUnit 4 на JUnit 5

### Изменения в аннотациях

| JUnit 4 | JUnit 5 |
|---------|---------|
| @Test | @Test |
| @Before | @BeforeEach |
| @After | @AfterEach |
| @BeforeClass | @BeforeAll |
| @AfterClass | @AfterAll |
| @Ignore | @Disabled |
| @SuiteClasses | @SelectClasses |

### Изменения в проверках

```java
// JUnit 4
Assert.assertEquals(expected, actual);

// JUnit 5
Assertions.assertEquals(expected, actual);
```

### Изменения в тестировании исключений

```java
// JUnit 4
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}

// JUnit 5
@Test
public void testException() {
    assertThrows(NullPointerException.class, () -> {
        Object obj = null;
        obj.toString();
    });
}
```

## Запуск тестов

### Командная строка

```bash
# Maven
mvn test

# Gradle
gradle test

# Автономная консоль
java -jar junit-platform-console-standalone-5.14.3.jar -c MyTestClass
```

### Запуск в IDE

- IntelliJ IDEA: Щёлкните правой кнопкой на тестовом классе или методе, выберите Run
- Eclipse: Щёлкните правой кнопкой на тестовом классе или методе, выберите Run As → JUnit Test
- VS Code: Используйте расширение JUnit для запуска тестов
