# Часто задаваемые вопросы о H2

## Проблемы с установкой

### В: Как скачать базу данных H2?

О: Посетите https://www.h2database.com/html/download.html, чтобы скачать версию для вашей платформы. Есть два варианта: установщик Windows и кроссплатформенный ZIP-пакет.

### В: Какая версия Java нужна для H2?

О: H2 требуется Java 11 или выше.

### В: Как использовать H2 в Maven-проекте?

О: Добавьте зависимость в pom.xml:
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### В: Как запустить H2 после распаковки ZIP?

О: Выполните в командной строке:
```bash
java -jar h2.jar
```
Это запустит веб-консоль H2.

## Проблемы с подключением

### В: Как подключиться к базе данных H2?

О: Используйте JDBC-подключение:
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### В: Какие имя пользователя и пароль по умолчанию в H2?

О: Имя пользователя по умолчанию — "sa", пароль пустой.

### В: Как подключиться к файловой базе данных?

О:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### В: Как запустить TCP-сервер H2?

О:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### В: Что делать, если порт занят?

О: Можно указать другой порт в URL подключения:
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## Проблемы с данными

### В: Данные в памяти теряются?

О: Да, база данных в памяти (jdbc:h2:mem:) теряет данные после закрытия всех подключений. Для сохранности используйте файловый режим.

### В: Как увидеть расположение файла базы данных?

О: Укажите путь в URL подключения:
```java
jdbc:h2:file:/путь/к/базе данных
```

### В: Что делать, если файл базы данных поврежден?

О: Попробуйте режим восстановления:
```java
jdbc:h2:file:./testdb;LOG=0
```
Или восстановите из резервной копии.

### В: Как зашифровать базу данных?

О:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## Проблемы с производительностью

### В: Как улучшить производительность пакетной вставки?

О: Используйте пакетные операции и отключите авто-коммит:
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### В: Запросы выполняются медленно — что делать?

О: 
1. Создайте индексы для распространенных условий запроса
2. Используйте EXPLAIN для анализа плана запроса
3. Избегайте SELECT *, запрашивайте только нужные столбцы

### В: Слишком высокий расход памяти — что делать?

О: Настройте размер кучи JVM и рационально используйте пулы подключений.

## Проблемы совместимости SQL

### В: Какие режимы баз данных поддерживает H2?

О: H2 может имитировать несколько баз данных:
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

Использование:
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### В: Как обрабатывать автоинкрементные ID?

О: H2 поддерживает несколько способов:
```sql
-- Способ 1: IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- Способ 2: SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## Проблемы с транзакциями

### В: Что происходит, если транзакция не зафиксирована?

О: Незафиксированные транзакции автоматически откатываются при закрытии подключения.

### В: Как обрабатывать взаимные блокировки?

О: H2 автоматически обнаруживает взаимные блокировки и откатывает одну из транзакций. Рекомендуется сократить область транзакций, чтобы избежать длительных блокировок.

## Проблемы безопасности

### В: База данных H2 безопасна?

О: H2 подходит для сред разработки и тестирования. Для производственных сред рекомендуется:
1. Использовать зашифрованную базу данных
2. Использовать надежные пароли
3. Ограничить сетевой доступ
4. Использовать SSL-подключения

### В: Как предотвратить SQL-инъекцию?

О: Используйте PreparedStatement вместо конкатенации строк:
```java
// Правильно
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// Неправильно
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## Проблемы с инструментами

### В: Как использовать консоль H2?

О: Выполните `java -jar h2.jar -web`, затем перейдите по адресу http://localhost:8082

### В: Как использовать оболочку H2?

О: Выполните `java -jar h2.jar -shell` или `java -jar h2.jar -tool`

### В: Как увидеть версию H2?

О:
```bash
java -jar h2.jar -version
```
Или в SQL: `SELECT H2VERSION();`

## Проблемы миграции

### В: Как мигрировать с MySQL на H2?

О:
1. Установите режим совместимости H2: `jdbc:h2:file:db;MODE=MySQL`
2. Экспортируйте данные MySQL в SQL-скрипт
3. Выполните скрипт в H2

### В: На что обратить внимание при обновлении версии H2?

О: Обновление с H2 1.x на 2.x требует особого внимания, см. официальное руководство по миграции:
https://www.h2database.com/html/migration-to-v2.html
