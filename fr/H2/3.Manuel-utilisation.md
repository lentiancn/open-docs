# Manuel d'utilisation de H2

## Opérations de base

### Créer une base de données

Mode intégré (base de données en mémoire) :
```java
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");
```

Base de données fichier :
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Créer une table

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Insérer des données

```sql
INSERT INTO users (name, email) VALUES ('Jean', 'jean@exemple.fr');
INSERT INTO users (name, email) VALUES ('Marie', 'marie@exemple.fr');
```

### Requêter des données

```sql
SELECT * FROM users;
SELECT * FROM users WHERE name = 'Jean';
SELECT COUNT(*) FROM users;
```

### Mettre à jour des données

```sql
UPDATE users SET email = 'nouvelemail@exemple.fr' WHERE id = 1;
```

### Supprimer des données

```sql
DELETE FROM users WHERE id = 2;
```

## Exemples de code Java

### Exemple JDBC complet

```java
import java.sql.*;

public class H2Example {
    public static void main(String[] args) throws Exception {
        // Charger le pilote
        Class.forName("org.h2.Driver");
        
        // Connecter à la base de données
        String url = "jdbc:h2:mem:testdb";
        Connection conn = DriverManager.getConnection(url, "sa", "");
        
        // Créer une table
        Statement stmt = conn.createStatement();
        stmt.execute("CREATE TABLE users (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100))");
        
        // Insérer des données
        stmt.execute("INSERT INTO users (name) VALUES ('Jean')");
        stmt.execute("INSERT INTO users (name) VALUES ('Marie')");
        
        // Requêter des données
        ResultSet rs = stmt.executeQuery("SELECT * FROM users");
        while (rs.next()) {
            System.out.println(rs.getInt("id") + ": " + rs.getString("name"));
        }
        
        // Fermer la connexion
        rs.close();
        stmt.close();
        conn.close();
    }
}
```

### Utiliser PreparedStatement

```java
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name, email) VALUES (?, ?)");
pstmt.setString(1, "Pierre");
pstmt.setString(2, "pierre@exemple.fr");
pstmt.executeUpdate();
pstmt.close();
```

### Utiliser les transactions

```java
conn.setAutoCommit(false);
try {
    Statement stmt = conn.createStatement();
    stmt.execute("INSERT INTO users (name) VALUES ('Paul')");
    stmt.execute("INSERT INTO users (name) VALUES ('Julie')");
    conn.commit();
} catch (Exception e) {
    conn.rollback();
} finally {
    conn.setAutoCommit(true);
}
```

## Fonctionnalités avancées

### Changement de mode de base de données

H2 peut simuler la syntaxe SQL d'autres bases de données :

```java
// Mode MySQL
String url = "jdbc:h2:mem:testdb;MODE=MySQL";

// Mode PostgreSQL
String url = "jdbc:h2:mem:testdb;MODE=PostgreSQL";

// Mode Oracle
String url = "jdbc:h2:mem:testdb;MODE=Oracle";
```

### Chiffrement de la base de données

```java
// Utiliser une base de données chiffrée
String url = "jdbc:h2:file:./data/encrypted;CIPHER=AES";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Fonctionnalité de trace

```java
// Activer la trace SQL
String url = "jdbc:h2:mem:testdb;TRACE_LEVEL_FILE=3";
```

### Démarrer le serveur TCP

```java
import org.h2.tools.Server;

Server server = Server.createTcpServer().start();
System.out.println("H2 TCP Server started: " + server.getURL());
```

### Utiliser la console H2

Démarrer la console web H2 :

```bash
java -jar h2.jar -web
```

Puis ouvrir dans le navigateur : http://localhost:8082

## Instructions SQL courantes

### Types de données

H2 prend en charge plusieurs types de données, notamment :
- Types entiers : INT, INTEGER, BIGINT, SMALLINT, TINYINT
- Types flottants : FLOAT, REAL, DOUBLE, DECIMAL
- Types chaîne : VARCHAR, CHAR, CLOB
- Types date/heure : DATE, TIME, TIMESTAMP
- Types booléens : BOOLEAN
- Types binaires : BLOB, BINARY

### Opérations d'index

Créer un index :
```sql
CREATE INDEX idx_name ON users(name);
```

Supprimer un index :
```sql
DROP INDEX idx_name;
```

### Opérations de contrainte

Ajouter une clé primaire :
```sql
ALTER TABLE users ADD PRIMARY KEY (id);
```

Ajouter une clé étrangère :
```sql
ALTER TABLE orders ADD FOREIGN KEY (user_id) REFERENCES users(id);
```

### Voir la structure d'une table

```sql
DESCRIBE users;
-- ou
SHOW COLUMNS FROM users;
```

### Voir toutes les tables

```sql
SHOW TABLES;
```

## Optimisation des performances

### Opérations par lots

Utiliser les opérations par lots pour améliorer les performances :

```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (int i = 0; i < 1000; i++) {
    pstmt.setString(1, "Utilisateur" + i);
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Pool de connexions

En environnement de production, l'utilisation d'un pool de connexions comme HikariCP est recommandée :

```java
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:h2:file:./data/testdb");
config.setUsername("sa");
config.setPassword("");
config.setMaximumPoolSize(10);

HikariDataSource ds = new HikariDataSource(config);
```

## Sauvegarde et restauration

### Sauvegarder la base de données

```sql
SCRIPT TO 'backup.sql';
```

### Restaurer la base de données

```sql
RUNSCRIPT FROM 'backup.sql';
```

## Gestion des erreurs courantes

### Fichier de base de données verrouillé
Assurez-vous qu'aucune autre connexion n'utilise le fichier de base de données. H2 prend en charge les lectures concurrentes, mais l'écriture nécessite un accès exclusif.

### Données en mémoire perdues
Les bases de données en mémoire perdent les données après la fermeture de la connexion. Pour la persistance, utilisez le mode fichier.

### Classe non trouvée
Assurez-vous que la classe org.h2.Driver est dans le classpath.
