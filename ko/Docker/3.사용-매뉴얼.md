# Docker 사용자 매뉴얼

이 매뉴얼은 Docker를 빠르게 시작하고 일상적인 사용을 위한 일반적인 작업을 마스터하는 데 도움이 됩니다.

## 전제 조건

시작하기 전에 다음을 갖추고 있는지 확인하세요:

- Docker가 설치됨 (Docker Desktop 또는 Docker Engine)
- 명령줄 터미널 액세스
- 명령줄 작업에 대한 기본 지식

## 기본 개념

### 이미지와 컨테이너

- **이미지**: 컨테이너를 만들기 위한 읽기 전용 템플릿
- **컨테이너**: 실행 중인 이미지의 인스턴스

### 일반적인 명령

```bash
# 컨테이너 실행
docker run hello-world

# 모든 컨테이너 나열
docker ps -a

# 모든 이미지 나열
docker images

# 컨테이너 중지
docker stop 컨테이너_ID

# 컨테이너 제거
docker rm 컨테이너_ID

# 이미지 제거
docker rmi 이미지_ID
```

## 컨테이너 작업

### 컨테이너 실행

컨테이너를 실행하는 기본 명령은 다음과 같습니다:

```bash
docker run [옵션] 이미지_이름 [명령]
```

예시:

```bash
# 대화형 컨테이너 시작
docker run -it ubuntu bash

# 백그라운드에서 컨테이너 실행
docker run -d nginx

# 포트 매핑으로 컨테이너 실행
docker run -p 8080:80 nginx
```

### 컨테이너 관리

```bash
# 실행 중인 모든 컨테이너 표시
docker ps

# 모든 컨테이너 (중지된 것 포함) 표시
docker ps -a

# 컨테이너 시작
docker start 컨테이너_이름

# 컨테이너 중지
docker stop 컨테이너_이름

# 컨테이너 재시작
docker restart 컨테이너_이름

# 컨테이너 제거
docker rm 컨테이너_이름
```

### 컨테이너 로그 보기

```bash
# 컨테이너 로그 표시
docker logs 컨테이너_이름

# 실시간으로 로그 팔로우
docker logs -f 컨테이너_이름

# 마지막 100줄 표시
docker logs --tail 100 컨테이너_이름
```

## 이미지 작업

### 이미지 검색 및 다운로드

```bash
# Docker Hub에서 이미지 검색
docker search nginx

# 이미지 다운로드
docker pull nginx

# 태그가 있는 이미지 다운로드
docker pull nginx:latest
```

### 이미지 생성

Dockerfile로 자체 이미지를 생성할 수 있습니다:

```dockerfile
FROM ubuntu:20.04
RUN apt-get update
RUN apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

그런 다음 이미지를 생성합니다:

```bash
docker build -t 내-nginx-이미지 .
```

### 이미지 관리

```bash
# 모든 이미지 나열
docker images

# 이미지 제거
docker rmi 이미지_이름

# 미사용 이미지 제거
docker image prune
```

## 볼륨 사용

볼륨은 컨테이너 외부에 데이터를 저장하는 데 사용됩니다.

```bash
# 볼륨 생성
docker volume create 내_볼륨

# 볼륨 마운트
docker run -v 내_볼륨:/데이터 nginx

# 모든 볼륨 나열
docker volume ls

# 볼륨 제거
docker volume rm 내_볼륨
```

## 네트워킹

Docker는 컨테이너 간 통신을 위한 네트워킹 기능을 제공합니다.

```bash
# 네트워크 생성
docker network create 내_네트워크

# 컨테이너를 네트워크에 연결
docker run --network 내_네트워크 nginx

# 모든 네트워크 나열
docker network ls
```

## Docker Compose

Docker Compose를 사용하면 다중 컨테이너 애플리케이션을 정의하고 실행할 수 있습니다.

### docker-compose.yml 예시

```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: 비밀번호
```

### 명령

```bash
# 모든 컨테이너 시작
docker-compose up

# 백그라운드에서 시작
docker-compose up -d

# 모든 컨테이너 중지
docker-compose down
```

## 팁 및 모범 사례

### 보안

- 가능한 항상 공식 이미지 사용
- 컨테이너를 root로 실행하지 않음
- 이미지를 최신 상태로 유지
- 이미지의 취약점 스캔

### 성능

- .dockerignore 파일 사용
- 멀티스테이지 빌드 사용
- 이미지의 레이어 수 최소화
- 특정 이미지 태그 사용

### 유지 관리성

- 의미 있는 컨테이너 이름 사용
- Dockerfile 문서화
- 지속적 데이터에 볼륨 사용
- 컨테이너 활동 기록
