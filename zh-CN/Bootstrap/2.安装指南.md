# Bootstrap 安装指南

## 概述

Bootstrap 提供了多种安装方式，从简单的 CDN 引用到使用包管理器安装，开发者可以根据项目需求选择最合适的方式。本指南将详细介绍每种安装方法，帮助你快速在项目中集成 Bootstrap。

## 方法一：使用 CDN（推荐入门）

内容分发网络（CDN）是快速上手 Bootstrap 的最简单方式，无需下载任何文件，直接通过链接引用即可。

### 基本模板

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>我的 Bootstrap 项目</title>
    <!-- 引入 Bootstrap CSS -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" crossorigin="anonymous">
</head>
<body>
    <h1>你好，Bootstrap！</h1>

    <!-- 引入 Bootstrap JS（包含 Popper） -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js" integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz" crossorigin="anonymous"></script>
</body>
</html>
```

### CDN 链接说明

| 资源类型 | CDN 链接 | 用途 |
|----------|----------|------|
| Bootstrap CSS | https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css | 核心样式文件 |
| Bootstrap JS | https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js | 包含 Popper 的 JavaScript |
| Bootstrap Icons | https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.css | 图标字体 |

### integrity 属性说明

CDN 链接中的 `integrity` 属性用于验证文件的完整性，防止在传输过程中被篡改。每次版本更新时，这个值也会相应变化。`crossorigin="anonymous"` 属性则用于处理跨域请求，避免 CORS 错误。

## 方法二：使用 npm 安装

如果你使用 npm 或 yarn 作为项目包管理器，可以通过命令行快速安装 Bootstrap。

### 安装 Bootstrap

```bash
# 使用 npm
npm install bootstrap

# 使用 yarn
yarn add bootstrap

# 使用 pnpm
pnpm add bootstrap
```

### 在项目中引入

#### CommonJS 方式
```javascript
// 引入 Bootstrap CSS
import 'bootstrap/dist/css/bootstrap.min.css';

// 引入 Bootstrap JS
import * as bootstrap from 'bootstrap';
```

#### 异步加载方式
```javascript
// 动态加载 Bootstrap JS
import('bootstrap/dist/js/bootstrap.bundle.min.js')
    .then(({ Modal }) => {
        // Bootstrap 已加载
    });
```

### 安装 Bootstrap Icons

```bash
npm install bootstrap-icons
```

在 CSS 中引入图标字体：

```css
@import "bootstrap-icons/font/bootstrap-icons.css";
```

## 方法三：下载源文件

### 下载预编译版本

1. 访问 Bootstrap 官网下载页面：https://getbootstrap.com/docs/5.3/getting-started/download/
2. 下载「Compiled CSS and JS」版本
3. 解压文件，将 css 和 js 文件夹复制到你的项目中

目录结构：
```
project/
├── css/
│   ├── bootstrap.min.css
│   └── bootstrap.min.css.map
├── js/
│   └── bootstrap.bundle.min.js
└── index.html
```

### 下载源代码版本

如果你需要自定义 Bootstrap 的样式或构建流程，可以下载完整的源代码版本：

```bash
# 克隆 GitHub 仓库
git clone https://github.com/twbs/bootstrap.git

# 安装依赖
npm install

# 构建项目
npm run dist
```

## 方法四：使用模板项目

### 通过 Vite 创建

```bash
# 创建新项目
npm create vite@latest my-bootstrap-app -- --template vanilla

# 进入项目目录
cd my-bootstrap-app

# 安装 Bootstrap
npm install bootstrap

# 启动开发服务器
npm run dev
```

### 通过 Webpack 使用

1. 安装依赖：
```bash
npm install bootstrap sass sass-loader css-loader style-loader
```

2. 在入口文件中引入：
```javascript
import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap/dist/js/bootstrap.bundle.min.js';
```

3. 配置 Webpack 的 sass-loader。

### 通过 React 使用

```bash
# 创建 React 项目
npx create-react-app my-bootstrap-app

# 进入项目目录
cd my-bootstrap-app

# 安装 Bootstrap
npm install bootstrap

# 在 index.js 中引入
import 'bootstrap/dist/css/bootstrap.min.css';
```

### 通过 Vue 使用

```bash
# 创建 Vue 项目
npm create vue@latest

# 安装 Bootstrap
npm install bootstrap

# 在 main.js 中引入
import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap/dist/js/bootstrap.bundle.min.js';
```

## 项目配置要点

### 1. 设置 viewport meta 标签

在 HTML 的 `<head>` 中添加以下标签，确保页面在移动设备上正确显示：

```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

这个标签的作用是：
- `width=device-width`：页面宽度等于设备屏幕宽度
- `initial-scale=1.0`：初始缩放比例为 100%

### 2. 使用 HTML5 doctype

Bootstrap 要求使用 HTML5 doctype，否则会看到异常的样式：

```html
<!DOCTYPE html>
<html lang="zh-CN">
    <!-- 页面内容 -->
</html>
```

### 3. 引入 Popper

部分 Bootstrap 组件（如下拉菜单、工具提示、弹出框）依赖于 Popper 进行定位。如果你使用 `bootstrap.bundle.min.js`，Popper 已经包含在内。如果单独引入 Bootstrap JS，需要额外引入 Popper：

```html
<script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.min.js"></script>
```

## 开发环境 vs 生产环境

### 开发环境

在开发阶段，可以使用未压缩的文件，便于调试：

```html
<!-- CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.css" rel="stylesheet">

<!-- JS -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.js"></script>
```

### 生产环境

上线时务必使用压缩版本，以减少文件体积：

```html
<!-- CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- JS -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>
```

## 常见问题

### Q: Bootstrap 5 是否需要 jQuery？

**不需要。** Bootstrap 5 完全使用原生 JavaScript 重写，不再依赖 jQuery。

### Q: 如何选择 Bootstrap 版本？

建议始终使用最新的稳定版本。当前最新版本是 5.3.8。可以在 CDN 链接中修改版本号来使用不同版本。

### Q: bootstrap.bundle.js 和 bootstrap.js 有什么区别？

- `bootstrap.bundle.min.js`：已包含 Popper，无需额外引入
- `bootstrap.min.js`：不包含 Popper，如果需要使用下拉菜单、工具提示等功能，需要单独引入 Popper

### Q: 为什么我的样式不生效？

常见原因：
1. CDN 链接错误或版本号有误
2. CSS 文件路径不正确
3. 缺少 viewport meta 标签
4. 浏览器缓存问题，尝试强制刷新（Ctrl+F5）
