# Manuel d'utilisation

Ce manuel présente en détail les fonctionnalités principales et les méthodes d'utilisation d'Apifox, y compris la gestion de projet, la conception d'interface, le débogage, le mock et les tests automatisés.

## Gestion de projet

### Créer un projet

Dans Apifox, le projet est le niveau le plus élevé pour organiser les API.

```bash
# Étapes pour créer un nouveau projet
1. Cliquez sur le "+" à côté de la liste "Projet" dans le panneau de gauche
2. Entrez le nom du projet
3. Choisissez le type de projet
4. Choisissez un modèle de base (optionnel)
5. Cliquez sur "Créer"
```

### Paramètres du projet

Chaque projet dispose d'options de paramètres indépendantes :

- **Informations de base** : nom, description et icône du projet
- **Variables d'environnement** : définissez des variables pour différents environnements comme le développement, les tests et la production
- **Authentification** : configurez le mode d'authentification global
- **Exportation/Importation** : import et export des données du projet

### Gestion de l'environnement

Apifox prend en charge la configuration de plusieurs environnements :

```bash
# Utilisation des variables d'environnement
Dans les requêtes, utilisez {{nom_variable}} pour référencer les variables d'environnement
Exemple : {{baseUrl}}/api/users

Changer d'environnement : cliquez sur le sélecteur d'environnement en haut pour choisir un environnement différent
```

## Conception d'interface

### Créer une interface

Une interface est une définition concrète d'API :

```bash
# Étapes pour créer une interface
1. Dans le projet, cliquez sur "+" et sélectionnez "Nouvelle interface"
2. Renseignez les informations de base :
   - Nom de l'interface : Liste des utilisateurs
   - Chemin de la requête : /api/users
   - Méthode de requête : GET
3. Définissez les paramètres de requête
4. Définissez la structure de réponse
5. Enregistrez
```

### Paramètres de requête

Les types de paramètres de requête suivants sont pris en charge :

- **Paramètres de chemin** : comme id dans /api/users/:id
- **Paramètres de requête** : dans l'URL comme ?name=test
- **En-têtes de requête** : comme Authorization
- **Corps de requête** : formats JSON, formulaire, XML, etc.

### Définition des réponses

Définissez les réponses réussies et échouées pour chaque interface :

```json
{
  "Réponse réussie 200": {
    "id": 1,
    "name": "Zhang San",
    "email": "zhangsan@example.com"
  },
  "Réponse d'erreur 404": {
    "code": 404,
    "message": "L'utilisateur n'existe pas"
  }
}
```

### Gestion de la liste des interfaces

La gestion groupée des interfaces est prise en charge, permettant une classification par module ou par fonction :

- Glisser-déposer pour réorganiser
- Opérations en lot (suppression en lot, exportation en lot, etc.)
- Recherche et filtrage

## Débogage d'interface

### Envoyer une requête

Sur la page de détails de l'interface, cliquez sur le bouton "Envoyer" pour envoyer la requête :

```bash
# Étapes de débogage de base
1. Ouvrez l'interface
2. Sélectionnez l'environnement
3. Définissez les valeurs des paramètres
4. Cliquez sur "Envoyer"
5. Consultez la réponse
```

### Configuration de l'authentification

Plusieurs modes d'authentification sont pris en charge :

- **Aucun** : pas d'authentification
- **Bearer Token** : authentification JWT
- **Basic Auth** : authentification basique
- **OAuth 2.0** : authentification OAuth2
- **Authentification personnalisée** : définissez des en-têtes d'authentification selon vos besoins

### Prétraitement des requêtes

Vous pouvez définir des scripts à exécuter avant l'envoi de la requête :

```javascript
// Exemple : ajouter un horodatage
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// Exemple : générer un ID aléatoire
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### Post-traitement des réponses

Vous pouvez définir des scripts à exécuter après réception de la réponse :

```javascript
// Exemple : extraire les données de réponse
const response = pm.response.json();
console.log('Données de réponse :', response);

// Exemple : définir une variable d'environnement
pm.environment.set('access_token', response.data.token);
```

## Mock API

### Mock intelligent

L'une des fonctionnalités les plus puissantes d'Apifox : il suffit de définir la structure des données, et le système génère automatiquement les données Mock :

```bash
# Activer le Mock intelligent
1. Ouvrez l'interface
2. Cliquez sur l'onglet "Mock"
3. Activez le "Mock intelligent"
4. Enregistrez
```

### Règles de Mock

Les règles de Mock personnalisées sont prises en charge :

- `@string` : générer une chaîne aléatoire
- `@integer` : générer un entier aléatoire
- `@float` : générer un nombre à virgule flottante aléatoire
- `@boolean` : générer une valeur booléenne aléatoire
- `@date` : générer une date aléatoire
- `@email` : générer un e-mail aléatoire
- `@url` : générer une URL aléatoire
- `@guid` : générer un UUID

### Exemple

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

Données Mock générées :

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Serveur Mock

Apifox fournit un serveur Mock intégré :

```bash
# Obtenir l'adresse Mock
Sur la page de détails de l'interface, copiez l'URL Mock
Exemple : https://mock.apifox.com/api/users
```

## Tests automatisés

### Créer un cas de test

```bash
# Étapes pour créer un test
1. Ouvrez l'interface
2. Cliquez sur l'onglet "Test"
3. Cliquez sur "Créer un test"
4. Définissez le nom du cas de test
5. Ajoutez des assertions
6. Enregistrez
```

### Types d'assertions

Plusieurs types d'assertions sont pris en charge :

- **Assertion de code de statut** : vérifier le code de statut de la réponse
- **Assertion JSON** : vérifier les données JSON de la réponse
- **Assertion de temps de réponse** : vérifier le temps de réponse
- **Assertion d'en-tête de réponse** : vérifier les en-têtes de réponse

### Exemple

```javascript
// Assertion de code de statut
pm.test("Le code de statut est 200", function() {
    pm.response.to.have.status(200);
});

// Assertion JSON
pm.test("Retourne la liste des utilisateurs", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// Assertion de temps de réponse
pm.test("Temps de réponse inférieur à 500ms", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### Suite de tests

Vous pouvez organiser les cas de test de plusieurs interfaces en suites de tests :

1. Créez une suite de tests
2. Ajoutez des cas de test
3. Définissez l'ordre d'exécution
4. Exécutez la suite

### Générer un rapport de test

Après l'exécution des tests, vous pouvez générer un rapport de test détaillé :

- Statistiques de taux de réussite
- Analyse des cas échoués
- Statistiques de temps de réponse

## Collaboration d'équipe

### Inviter des membres

```bash
# Inviter un membre de l'équipe
1. Accédez aux paramètres du projet
2. Cliquez sur "Gestion des membres"
3. Cliquez sur "Inviter un membre"
4. Entrez l'adresse e-mail du membre
5. Définissez les permissions du rôle
6. Envoyz l'invitation
```

### Rôles et permissions

- **Administrateur** : tous les droits sur le projet
- **Éditeur** : peut modifier les interfaces et exécuter des tests
- **Lecteur** : consultation seule, pas de modification

### Synchronisation en temps réel

Une fois connectés, les membres de l'équipe bénéficient d'une synchronisation automatique en temps réel des données du projet :

- Les modifications d'interface se synchronisent automatiquement
- Les résultats de test se synchronisent automatiquement
- Les membres de l'équipe voient toujours le contenu le plus récent

### Fonction de commentaires

Vous pouvez ajouter des commentaires sur les interfaces :

1. Ouvrez l'interface
2. Cliquez sur "Commentaires"
3. Entrez le contenu du commentaire
4. Mentionnez un membre de l'équipe avec @

## Importation et exportation de données

### Formats pris en charge

**Importation :**

- Postman Collection (v2.1)
- Swagger/OpenAPI (2.0, 3.0)
- HAR
- Apiguardian
- YApi

**Exportation :**

- OpenAPI 3.0
- Swagger 2.0
- Markdown
- HTML
- Format Apifox

## Raccourcis clavier

Raccourcis couramment utilisés :

- `Ctrl/Cmd + S` : enregistrer
- `Ctrl/Cmd + Entrée` : envoyer la requête
- `Ctrl/Cmd + B` : formater JSON
- `Ctrl/Cmd + /` : commenter/décommenter

## Bonnes pratiques

1. **Nommage standardisé** : les noms d'interfaces doivent décrire clairement la fonctionnalité
2. **Commentaires complets** : ajoutez une description pour chaque paramètre
3. **Regroupement logique** : organisez les interfaces par module
4. **Utilisation des environnements** : distinguez les environnements de développement, de test et de production
5. **Rédaction de tests** : écrivez des tests automatisés pour les interfaces importantes
6. **Synchronisation régulière** : maintenez la cohérence entre les données locales et cloud
