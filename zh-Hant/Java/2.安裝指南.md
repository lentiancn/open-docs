# Java 安裝指南

## 取得 Java 開發工具包

Oracle 官方提供 Java 開發工具包（JDK）的免費下載。當前主要的 LTS 版本包括 JDK 25、JDK 21、JDK 17、JDK 11 和 JDK 8。

官方下載頁面：https://www.oracle.com/java/technologies/downloads/

## Windows 系統安裝

### 使用安裝程式

1. 造訪 Oracle 官方下載頁面，選擇 Windows 版本的 JDK
2. 下載 x64 Installer（exe 檔案）或 MSI Installer
3. 執行下載的安裝程式
4. 按照安裝精靈的提示完成安裝，預設安裝路徑為 C:\Program Files\Java\jdk-version
5. 安裝完成後，驗證安裝是否成功

### 使用壓縮檔

1. 下載 x64 Compressed Archive（zip 檔案）
2. 解壓到指定目錄，如 C:\Java\jdk-25
3. 設定環境變數

### 設定環境變數

1. 右鍵點擊「此電腦」，選擇「內容」
2. 點擊「進階系統設定」
3. 點擊「環境變數」
4. 在系統變數中，點擊「新增」
   - 變數名稱：JAVA_HOME
   - 變數值：JDK 安裝路徑（如 C:\Program Files\Java\jdk-25）
5. 找到 Path 變數，點擊「編輯」
6. 新增變數：%JAVA_HOME%\bin
7. 點擊確定儲存

### 驗證安裝

開啟命令提示字元，輸入以下命令：

```cmd
java -version
javac -version
```

如果顯示版本資訊，表示安裝成功。

## Linux 系統安裝

### 使用壓縮檔安裝

1. 下載 Linux 版本的 JDK（tar.gz 格式）
2. 開啟終端機，解壓到指定目錄：

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. 設定環境變數，編輯 ~/.bashrc 或 /etc/profile：

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. 使設定生效：

```bash
source ~/.bashrc
```

5. 驗證安裝：

```bash
java -version
```

### 使用套件管理員安裝

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## macOS 系統安裝

### 使用 DMG 安裝程式

1. 下載 macOS 版本的 JDK（dmg 檔案）
2. 開啟下載的 dmg 檔案
3. 將 JDK 拖入 Applications 資料夾
4. 開啟終端機，設定環境變數：

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. 將以上內容添加到 ~/.zshrc 或 ~/.bash_profile
6. 驗證安裝：

```bash
java -version
```

### 使用 Homebrew

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## 多個 JDK 版本共存

### Linux 下使用 update-alternatives

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Windows 下切換版本

修改 JAVA_HOME 環境變數指向不同的 JDK 目錄即可切換版本。

### 使用 SDKMAN 管理多版本

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## 驗證 Java 環境

### 檢查 Java 版本

```bash
java -version
```

輸出範例：

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### 檢查編譯工具版本

```bash
javac -version
```

輸出範例：

```
javac 25.0.1
```

### 檢查 JAVA_HOME

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## 常見安裝問題

### 問題一：java 命令不可用

**解決方法**：檢查 Path 環境變數是否包含 JDK 的 bin 目錄。

### 問題二：權限不足

**解決方法**：Linux/macOS 上使用 sudo 提權，或檢查檔案權限。

### 問題三：連接埠被佔用

**解決方法**：Java 應用程式預設使用 8080 連接埠，確保連接埠未被其他程式佔用。

### 問題四：JDK 和 JRE 的差異

- **JDK**（Java Development Kit）：Java 開發工具包，包含開發工具和 JRE
- **JRE**（Java Runtime Environment）：Java 執行時環境，僅包含執行 Java 程式所需的元件

開發 Java 程式需要安裝 JDK，執行已編譯的 Java 程式只需 JRE。

## 安裝後設定建議

1. 設定 MAVEN_HOME 或 GRADLE_HOME：如果使用 Maven 或 Gradle
2. 設定 M2_HOME：設定 Maven 本地儲存庫路徑
3. 設定 HTTP 代理：如果在企業環境中，需要設定代理才能存取外網
4. 設定時區：確保 Java 應用程式使用正確的時區

## 解除安裝 Java

### Windows 控制台解除安裝

1. 開啟控制台
2. 選擇「程式和功能」
3. 找到 Java 相關程式
4. 右鍵選擇「解除安裝」

### Linux 解除安裝

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS 解除安裝

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
