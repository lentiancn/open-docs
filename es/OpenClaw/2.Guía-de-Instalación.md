# Guía de Instalación de OpenClaw

OpenClaw soporta múltiples métodos de instalación, ya sea que uses macOS, Linux o Windows. Esta guía te llevará a través de cada opción de instalación y sus requisitos previos.

---

## Requisitos del Sistema

Antes de instalar, asegúrate de que tu sistema cumple estos requisitos:

- **Node.js**: 22.0.0 o superior
- **Sistema Operativo**:
  - macOS 10.15 (Catalina) o superior
  - Linux (Ubuntu 20.04+, Debian 11+, Fedora 36+, etc.)
  - Windows 10/11 (se recomienda WSL2 fuertemente)
- **Red**: Acceso al registro de npm y servicios de OpenClaw
- **Almacenamiento**: Al menos 500MB de espacio libre

Verifica tu versión de Node.js:

```bash
node --version
```

Si la versión está por debajo de 22.0.0, por favor actualiza Node.js primero. Recomendamos usar nvm (Node Version Manager) para administrar múltiples versiones de Node.js.

---

## Métodos de Instalación

### Método 1: Script de Instalación Oficial (Recomendado)

Esta es la forma más simple y rápida. El script de instalación oficial detecta automáticamente tu sistema operativo y completa toda la configuración.

**macOS / Linux:**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

El script de instalación realizará:
1. Detectar entorno del sistema
2. Instalar o actualizar el paquete global de OpenClaw
3. Crear directorios de configuración necesarios
4. Establecer permisos apropiados

**Windows (PowerShell):**

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Método 2: Instalación Global con npm

Si ya tienes Node.js y npm instalado, puedes instalar directamente a través de npm:

```bash
npm install -g openclaw@latest
```

También puedes especificar una versión particular:

```bash
npm install -g openclaw@v2026.2.28
```

Etiquetas de versión disponibles:
- `latest`: Último lanzamiento estable
- `beta`: Versión beta
- `dev`: Versión de desarrollo

### Método 3: Instalación con pnpm

OpenClaw recomienda usar pnpm para mejor gestión de dependencias e instalación más rápida:

```bash
pnpm add -g openclaw@latest
```

### Método 4: Instalación con Bun

Si usas Bun como tu runtime de JavaScript:

```bash
bun add -g openclaw@latest
```

### Método 5: Construir desde el Código Fuente

Si quieres usar la última versión de desarrollo o contribuir al proyecto, clona desde GitHub y construye:

```bash
# Clonar el repositorio
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Instalar dependencias
pnpm install

# Construir el proyecto
pnpm ui:build
pnpm build

# Inicializar e instalar servicio
pnpm openclaw onboard --install-daemon

# Modo desarrollo (recarga automática)
pnpm gateway:watch
```

**Nota**: Ejecutar a través de `pnpm openclaw ...` ejecuta TypeScript directamente (vía tsx). Después de construir, puedes ejecutar a través de `node dist/`.

## Configuración Post-Instalación

### Asistente de Incorporación

Después de la instalación, ejecuta el asistente de incorporación para completar la configuración básica:

```bash
openclaw onboard --install-daemon
```

El asistente te guiará a través de:
1. Configurar claves API (se recomienda Anthropic)
2. Configurar directorio de espacio de trabajo
3. Configurar conexiones de canal (WhatsApp, Telegram, Discord, etc.)
4. Instalar el servicio de fondo del Gateway

### Configuración Manual

Si prefieres solo configuración manual:

```bash
openclaw configure
```

Esto inicia un asistente de configuración interactivo.

### Verificar Instalación

Después de la instalación, verifica que todo funcione:

```bash
# Verificar versión
openclaw --version

# Verificar estado del Gateway
openclaw gateway status

# Abrir interfaz de control
openclaw dashboard
```

Si todo está funcionando, deberías ver el Gateway en ejecución y poder abrir la interfaz de control en tu navegador.

## Notas Específicas por Plataforma

### macOS

- Recomendado: Usa Homebrew para instalar Node.js: `brew install node`
- Después de la instalación, puede que necesites recargar el entorno de shell
- Recomendado: Habilita el Acceso a Llaveros para almacenamiento seguro de credenciales

### Linux

- La mayoría de distribuciones pueden instalar Node.js 22+ a través del gestor de paquetes
- Ubuntu/Debian: Usa el repositorio NodeSource para la última versión
- Asegúrate de tener build-essential instalado (para compilar módulos nativos)

### Windows

- **WSL2 es fuertemente recomendado** (Windows Subsystem for Linux)
- Ejecutar directamente en Windows puede tener problemas de compatibilidad
- Usar WSL2 proporciona la mejor experiencia

## Problemas Comunes de Instalación

### Problema 1: Errores de Permiso de npm

Si encuentras errores de permiso `EACCES`, no uses sudo para instalar paquetes npm. En cambio, usa nvm para administrar Node.js:

```bash
# Instalar nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Usar nvm para instalar Node.js
nvm install 22
nvm use 22

# Luego instalar OpenClaw
npm install -g openclaw@latest
```

### Problema 2: Gateway No Inicia

Verifica si el puerto ya está en uso:

```bash
lsof -i :18789
```

Si el puerto está ocupado, cambia el puerto en la configuración o detén el proceso que usa ese puerto.

### Problema 3: Versión Obsoleta

Si la versión instalada es muy antigua, intenta actualizar a la última versión:

```bash
npm update -g openclaw@latest
```

O reinstala:

```bash
npm uninstall -g openclaw
npm install -g openclaw@latest

## Instalación Docker (Opcional)

Si prefieres usar Docker, OpenClaw también ofrece imágenes oficiales de Docker:

```bash
# Descargar imagen
docker pull openclaw/openclaw:latest

# Ejecutar contenedor
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  openclaw/openclaw:latest
```

Docker es adecuado para pruebas rápidas, pero para funcionalidad completa se recomienda instalar directamente en el host.

---

## Actualizando OpenClaw

Para actualizar a la última versión:

```bash
# Verificar versión actual
openclaw --version

# Actualizar
npm update -g openclaw@latest

# O usar script de actualización oficial
openclaw update
```

Después de actualizar, ejecuta diagnósticos:

```bash
openclaw doctor
```

Esto verificará compatibilidad de configuración y problemas potenciales.

---

## Desinstalando OpenClaw

Si necesitas desinstalar OpenClaw:

```bash
# Detener servicio
openclaw gateway stop

# Desinstalar paquete global
npm uninstall -g openclaw

# Eliminar directorio de configuración (opcional)
rm -rf ~/.openclaw
```
