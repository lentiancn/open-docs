# Mockito 常见问题

## 基础问题

### Mockito 和 PowerMock 有什么区别

Mockito 主要用于普通对象的模拟，而 PowerMock 可以模拟静态方法、构造函数、私有方法等。然而，PowerMock 通过字节码操作技术破坏测试的隔离性，使得测试更接近集成测试而非单元测试。官方建议优先使用 Mockito，只有在确实需要模拟静态方法等 Mockito 无法做到的场景时才考虑 PowerMock。从 Mockito 2.x 开始，通过 mockito-inline 也支持模拟静态方法。

### 什么时候应该使用 Spy 而不是 Mock

使用 Spy 的场景是当需要大部分真实行为，只对少数方法进行模拟时。Spy 会保留真实对象的大部分功能，只对明确指定的方法进行模拟。而 Mock 则完全替换了真实对象，所有方法调用都不会执行真实逻辑。如果需要完全控制对象行为，则使用 Mock。在实际开发中，Spy 使用场景相对较少，因为大多数情况下使用真实的测试对象或完全 Mock 就能满足需求。

### Mockito 能否用于 final 类或方法

从 Mockito 2.1.0 版本开始，通过配置可以模拟 final 类和方法。需要在项目中添加 mockito-inline 依赖，而不是使用 mockito-core。mockito-inline 会启用子类的动态代理机制，从而能够处理 final 类和方法。这个功能大大增强了 Mockito 的适用范围，使得几乎所有 Java 类都可以被模拟。

### 为什么 Mock 对象的方法返回 null

默认情况下，Mock 对象的所有方法都会返回 null 或对应类型的默认值（int 返回 0，boolean 返回 false，集合类型返回空集合等）。这是 Mockito 的默认 Answer 行为。如果需要特定的返回值，需要使用 when().thenReturn() 进行明确的设定。

## 使用问题

### @Mock 注解不生效怎么办

@Mock 注解需要先初始化 Mockito 框架才能生效。确保在测试中使用了 MockitoAnnotations.initMocks(this)、@RunWith(MockitoJUnitRunner.class) 或 @ExtendWith(MockitoExtension.class)（JUnit 5）中的一种方式来初始化。如果初始化正确，@Mock 注解的字段会在测试方法执行前自动创建模拟对象。

### 如何模拟 void 方法

对于返回类型为 void 的方法，使用 doThrow()、doAnswer() 或 doNothing() 来设定行为。

```java
doThrow(new RuntimeException("Error")).when(mockedObject).voidMethod();
doAnswer(invocation -> null).when(mockedObject).voidMethod();
doNothing().when(mockedObject).voidMethod();
```

### 如何验证方法的调用参数

可以使用参数匹配器进行验证，或者使用 ArgumentCaptor 捕获参数进行详细断言。

```java
verify(mockedList).add(anyString());

ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
verify(mockedList).add(captor.capture());
assertTrue(captor.getValue().length() > 0);
```

### 如何模拟迭代器

迭代器的模拟需要注意 hasNext() 和 next() 方法的配合。

```java
Iterator<String> mockIterator = mock(Iterator.class);
when(mockIterator.hasNext()).thenReturn(true, true, false);
when(mockIterator.next()).thenReturn("first", "second");

assertTrue(mockIterator.hasNext());
assertEquals("first", mockIterator.next());
assertTrue(mockIterator.hasNext());
assertEquals("second", mockIterator.next());
assertFalse(mockIterator.hasNext());
```

### 如何处理方法重载

如果类中有方法重载，Mockito 会根据参数类型和数量来区分不同的方法。确保 when() 中的参数类型与要模拟的方法签名完全匹配。

```java
when(mockedList.get(0)).thenReturn("string");      // get(int index)
when(mockedList.get(anyInt())).thenReturn("any");  // 重载方法
```

## 高级问题

### 为什么 when().thenReturn() 不工作

常见原因包括：对同一个方法多次设定行为（只有最后一次生效）；使用 Spy 时调用了真实方法；方法返回类型为 void（应该使用 doReturn().when()）；参数匹配器使用不当导致匹配失败。检查代码确保行为设定的语法正确。

### 如何测试抽象类

测试抽象类时，可以使用 Mockito 模拟抽象类的具体方法，或者使用 Spy 来保留部分真实行为。另一种方式是创建一个具体的测试子类来运行测试。

```java
@Test
public void testAbstractClass() {
    AbstractClass mock = mock(AbstractClass.class, Mockito.CALLS_REAL_METHODS);
    when(mock.concreteMethod()).thenReturn("mocked");
}
```

### 如何模拟泛型方法

模拟泛型方法时，需要注意类型擦除带来的限制。使用 any() 或 anyObject() 来匹配泛型参数。

```java
when(mockedObject.genericMethod(any())).thenReturn("result");
```

### 如何验证调用顺序

使用 InOrder 来验证方法调用按照特定顺序执行。

```java
InOrder inOrder = inOrder(mockedList);
inOrder.verify(mockedList).add("first");
inOrder.verify(mockedList).add("second");
```

### 如何部分模拟对象

使用 Spy 进行部分模拟，或者使用 withSettings() 自定义 Mock 的行为。

```java
MyClass spy = spy(new MyClass());
doReturn("mocked").when(spy).method();
```

## 性能问题

### 测试运行很慢怎么办

确保没有使用不必要的 real method calls。检查是否在每个测试中都创建新的 Mock 对象。对于大量使用 Mockito 的项目，可以考虑使用 @RunWith(MockitoJUnitRunner.Silent.class) 来减少日志输出。

### 如何减少 Mock 对象的内存占用

Mockito 的 Mock 对象本身不会占用太多内存。但如果创建大量 Mock 对象仍然影响性能，可以考虑使用 @Mock 注解配合 @Scope 来复用 Mock 对象，或者使用静态的 Mock 工厂方法。

## 调试问题

### 如何查看 Mock 对象的调用日志

可以使用 Mockito 的日志功能或通过添加监听器来追踪调用。

```java
Mockito.framework().addListener(new VerificationListener());
```

或者使用 Maven/Gradle 的测试输出查看详细的 Mock 调用信息。

### 测试失败时如何调试

Mockito 提供了详细的错误信息。检查 verify 失败时显示的预期调用与实际调用的差异。使用 Print Mockito Logs 功能可以帮助了解 Mock 对象的具体行为。逐步简化测试用例，定位问题所在。
