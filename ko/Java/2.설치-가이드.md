# Java 설치 가이드

## Java 개발 키트 받기

Oracle는 Java 개발 키트(JDK)의 무료 다운로드를 제공합니다. 현재 주요 LTS 버전은 JDK 25, JDK 21, JDK 17, JDK 11 및 JDK 8입니다.

공식 다운로드 페이지: https://www.oracle.com/java/technologies/downloads/

## Windows에 설치

### 설치 프로그램 사용

1. Oracle 공식 다운로드 페이지를 방문하여 Windows용 JDK 버전을 선택하세요
2. x64 Installer(exe 파일) 또는 MSI Installer를 다운로드하세요
3. 다운로드한 설치 프로그램을 실행하세요
4. 설치 마법사의 지시에 따라 설치를 완료하세요. 기본 설치 경로는 C:\Program Files\Java\jdk-version입니다
5. 설치 후 성공적으로 설치되었는지 확인하세요

### 압축 파일 사용

1. x64 Compressed Archive(zip 파일)를 다운로드하세요
2. 지정된 디렉터리(예: C:\Java\jdk-25)에 추출하세요
3. 환경 변수 구성

### 환경 변수 구성

1. "이 컴퓨터"를 마우스右键 클릭하고 "속성"을 선택하세요
2. "고급 시스템 설정"을 클릭하세요
3. "환경 변수"를 클릭하세요
4. 시스템 변수에서 "새로 만들기"를 클릭하세요
   - 변수 이름: JAVA_HOME
   - 변수 값: JDK 설치 경로(예: C:\Program Files\Java\jdk-25)
5. Path 변수를 찾아 "편집"을 클릭하세요
6. 새 변수 추가: %JAVA_HOME%\bin
7. 저장을 클릭하세요

### 설치 확인

명령 프롬프트를 열고 다음을 입력하세요:

```cmd
java -version
javac -version
```

버전 정보가 표시되면 설치가 성공한 것입니다.

## Linux에 설치

### 압축 파일에서 설치

1. Linux용 JDK(tar.gz 형식)를 다운로드하세요
2. 터미널을 열고 지정된 디렉터리에 추출하세요:

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. ~/.bashrc 또는 /etc/profile을 편집하여 환경 변수를 구성하세요:

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. 구성을 적용하세요:

```bash
source ~/.bashrc
```

5. 설치를 확인하세요:

```bash
java -version
```

### 패키지 관리자 사용

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## macOS에 설치

### DMG 설치 프로그램 사용

1. macOS용 JDK(dmg 파일)를 다운로드하세요
2. 다운로드한 dmg 파일을 열하세요
3. JDK를 Applications 폴더로 드래그하세요
4. 터미널을 열고 환경 변수를 구성하세요:

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. 위 내용을 ~/.zshrc 또는 ~/.bash_profile에 추가하세요
6. 설치를 확인하세요:

```bash
java -version
```

### Homebrew 사용

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## 여러 JDK 버전 관리

### Linux에서 update-alternatives 사용

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Windows에서 버전 전환

JAVA_HOME 환경 변수를 다른 JDK 디렉터리로 변경하세요.

### SDKMAN 사용

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Java 환경 확인

### Java 버전 확인

```bash
java -version
```

출력 예시:

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### 컴파일러 버전 확인

```bash
javac -version
```

출력 예시:

```
javac 25.0.1
```

### JAVA_HOME 확인

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## 일반적인 설치 문제

### 문제 1: java 명령을 찾을 수 없음

해결책: Path 환경 변수에 JDK의 bin 디렉터리가 포함되어 있는지 확인하세요.

### 문제 2: 권한 부족

해결책: Linux/macOS에서는 sudo를 사용하여 권한을 상승시키거나 파일 권한을 확인하세요.

### 문제 3: 포트가 이미 사용 중

해결책: Java 애플리케이션은 기본적으로 포트 8080을 사용합니다. 포트가 다른 프로그램에서 사용되지 않는지 확인하세요.

### 문제 4: JDK와 JRE의 차이점

- JDK (Java Development Kit): 개발 도구와 JRE를 포함한 개발 키트
- JRE (Java Runtime Environment): Java 프로그램 실행에 필요한 구성 요소만 포함

Java 프로그램 개발에는 JDK가 필요합니다. 컴파일된 Java 프로그램 실행에는 JRE만으로도 충분합니다.

## 설치 후 권장 사항

1. Maven 또는 Gradle을 사용하는 경우 MAVEN_HOME 또는 GRADLE_HOME을 구성하세요
2. Maven 로컬 저장소 경로를 구성하려면 M2_HOME을 설정하세요
3. 기업 환경에서 필요한 경우 HTTP 프록시를 구성하세요
4. Java 애플리케이션이 올바른 시간을 사용하도록 시간대를 설정하세요

## Java 제거

### Windows 제어판

1. 제어판을 여세요
2. "프로그램 및 기능"을 선택하세요
3. Java 관련 프로그램을 찾으세요
4. 마우스右键 클릭하고 "제거"를 선택하세요

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
