# Element UI Installation Guide

> Element UI is a desktop UI component library based on Vue 2
> 
> Updated: February 2026
> 
> Current Version: 2.x (Stable)
> 
> Note: For Vue 3 projects, please use Element Plus

---

## 1. Project Overview

Element UI is a desktop UI component library based on Vue 2.0, developed and maintained by Ele.me team. It provides a rich set of commonly used PC-side components, covering forms, tables, dialogs, navigation menus, buttons, input boxes and other common UI elements.

Element UI has a clean and elegant design style with detailed documentation. It is one of the most popular Vue 2 UI component libraries in China. Although the official maintenance has stopped (recommending migration to Element Plus for Vue 3), Element UI is still highly valuable for learning and reference due to the large number of existing projects still in use.

### 1.1 Version Information

| Version | Vue Version | Status | Description |
|---------|-------------|--------|-------------|
| Element UI 2.x | Vue 2 | Stable | Main documentation version |
| Element UI 1.x | Vue 2 | End of Life | Early version, no longer recommended |

### 1.2 Difference from Element Plus

| Feature | Element UI | Element Plus |
|---------|------------|--------------|
| Vue Version | Vue 2 | Vue 3 |
| TypeScript | Partial Support | Full Support |
| Composition API | Not Supported | Supported |
| Maintenance Status | Stopped | Actively Maintained |
| Release Years | 2016-2021 | 2022-Present |

## 3. Installation Steps

### 3.1 Using npm (Recommended)

```bash
# Enter your Vue 2 project directory
cd your-vue-project

# Install Element UI
npm install element-ui --save
```

### 3.2 Using yarn

```bash
yarn add element-ui
```

### 3.3 Install Specific Version

```bash
# Install latest 2.x version
npm install element-ui@latest

# Install specific version
npm install element-ui@2.15.13
```

## 4. Importing in Vue 2 Project

### 4.1 Full Import (Suitable for Small Projects)

Import Element UI and its styles in the entry file (usually main.js):

```javascript
// main.js
import Vue from 'vue'
import App from './App.vue'

// Import Element UI
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'

Vue.use(ElementUI)

new Vue({
  render: h => h(App)
}).$mount('#app')
```

### 4.2 On-Demand Import (Recommended for Production)

To reduce bundle size, import only the components you need.

#### 4.2.1 Install babel-plugin-component

```bash
npm install babel-plugin-component -D
```

#### 4.2.2 Configure babel

Add configuration in .babelrc or babel.config.js:

```json
{
  "presets": [
    ["env", {
      "modules": false,
      "targets": {
        "browsers": ["> 1%", "last 2 versions", "not ie <= 8"]
      }
    }]
  ],
  "plugins": [
    ["component", {
      "libraryName": "element-ui",
      "styleLibraryName": "theme-chalk"
    }]
  ]
}
```

#### 4.2.3 Import Components On-Demand

```javascript
// main.js
import Vue from 'vue'
import { Button, Form, FormItem, Input, Table, TableColumn } from 'element-ui'
import App from './App.vue'

// Register needed components
Vue.use(Button)
Vue.use(Form)
Vue.use(FormItem)
Vue.use(Input)
Vue.use(Table)
Vue.use(TableColumn)

new Vue({
  render: h => h(App)
}).$mount('#app')
```

### 4.3 Using CDN

If you don't want to use build tools, you can import Element UI via CDN:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Element UI CDN Example</title>
  <!-- Import Vue 2 -->
  <script src="https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.js"></script>
  <!-- Import Element UI -->
  <link rel="stylesheet" href="https://unpkg.com/element-ui/lib/theme-chalk/index.css">
  <script src="https://unpkg.com/element-ui"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary" @click="visible = true">Open Dialog</el-button>
    <el-dialog title="Notice" :visible.sync="visible" width="30%">
      <span>This is a message</span>
      <span slot="footer" class="dialog-footer">
        <el-button @click="visible = false">Cancel</el-button>
        <el-button type="primary" @click="visible = false">OK</el-button>
      </span>
    </el-dialog>
  </div>

  <script>
    new Vue({
      el: '#app',
      data: {
        visible: false
      }
    })
  </script>
</body>
</html>
```

## 5. Quick Start Examples

### 5.1 Basic Button

```html
<template>
  <div>
    <el-button>Default</el-button>
    <el-button type="primary">Primary</el-button>
    <el-button type="success">Success</el-button>
    <el-button type="info">Info</el-button>
    <el-button type="warning">Warning</el-button>
    <el-button type="danger">Danger</el-button>
  </div>
</template>
```

### 5.2 Basic Form

```html
<template>
  <el-form :model="form" :rules="rules" ref="loginForm" label-width="80px">
    <el-form-item label="Username" prop="username">
      <el-input v-model="form.username" placeholder="Enter username"></el-input>
    </el-form-item>
    <el-form-item label="Password" prop="password">
      <el-input v-model="form.password" type="password" placeholder="Enter password"></el-input>
    </el-form-item>
    <el-form-item>
      <el-button type="primary" @click="submitForm">Login</el-button>
      <el-button @click="resetForm">Reset</el-button>
    </el-form-item>
  </el-form>
</template>

<script>
export default {
  data() {
    return {
      form: {
        username: '',
        password: ''
      },
      rules: {
        username: [
          { required: true, message: 'Please enter username', trigger: 'blur' }
        ],
        password: [
          { required: true, message: 'Please enter password', trigger: 'blur' }
        ]
      }
    }
  },
  methods: {
    submitForm() {
      this.$refs.loginForm.validate(valid => {
        if (valid) {
          console.log('Submit login form:', this.form)
        } else {
          this.$message.error('Form validation failed')
        }
      })
    },
    resetForm() {
      this.$refs.loginForm.resetFields()
    }
  }
}
</script>
```

### 5.3 Basic Table

```html
<template>
  <el-table :data="tableData" stripe border>
    <el-table-column prop="date" label="Date" width="180"></el-table-column>
    <el-table-column prop="name" label="Name" width="180"></el-table-column>
    <el-table-column prop="address" label="Address"></el-table-column>
    <el-table-column label="Actions" width="180">
      <template slot-scope="scope">
        <el-button size="mini" @click="handleEdit(scope.$index, scope.row)">Edit</el-button>
        <el-button size="mini" type="danger" @click="handleDelete(scope.$index, scope.row)">Delete</el-button>
      </template>
    </el-table-column>
  </el-table>
</template>

<script>
export default {
  data() {
    return {
      tableData: [{
        date: '2024-01-01',
        name: 'John Doe',
        address: 'New York'
      }, {
        date: '2024-01-02',
        name: 'Jane Smith',
        address: 'Los Angeles'
      }]
    }
  },
  methods: {
    handleEdit(index, row) {
      console.log('Edit:', index, row)
    },
    handleDelete(index, row) {
      console.log('Delete:', index, row)
    }
  }
}
</script>
```

### 5.4 Dialog

```html
<template>
  <el-button type="text" @click="dialogVisible = true">Open Dialog</el-button>

  <el-dialog
    title="Edit Information"
    :visible.sync="dialogVisible"
    width="30%"
    :before-close="handleClose">
    <span>This is dialog content</span>
    <span slot="footer" class="dialog-footer">
      <el-button @click="dialogVisible = false">Cancel</el-button>
      <el-button type="primary" @click="dialogVisible = false">OK</el-button>
    </span>
  </el-dialog>
</template>

<script>
export default {
  data() {
    return {
      dialogVisible: false
    }
  },
  methods: {
    handleClose(done) {
      this.$confirm('Confirm close?')
        .then(_ => {
          done()
        })
        .catch(_ => {})
    }
  }
}
</script>
```

## 6. Internationalization Configuration

Element UI uses English by default. To use other languages, configure internationalization.

### 6.1 Global Chinese Configuration

```javascript
// main.js
import Vue from 'vue'
import ElementUI from 'element-ui'
import locale from 'element-ui/lib/locale/lang/zh-CN'
import 'element-ui/lib/theme-chalk/index.css'

Vue.use(ElementUI, {
  locale
})
```

## 7. Theme Customization

### 7.1 Use Online Theme Tool

Visit Element UI theme customization page: https://elementui.github.io/theme-chalk/

You can select theme color, border color, border radius and other parameters, then download the customized style files.

### 7.2 Manually Override Variables

Create custom style file in your project:

```scss
// styles/element-variables.scss
$--color-primary: #409EFF;
$--color-success: #67C23A;
$--color-warning: #E6A23C;
$--color-danger: #F56C6C;
$--color-info: #909399;

@import 'element-ui/packages/theme-chalk/src/common/var.scss';
```

Then import in main.js:

```javascript
import 'element-ui/lib/theme-chalk/index.css'
import './styles/element-variables.scss'
```

## 8. Common Issues

### 8.1 Installation Failure

**Problem**: npm install fails

**Solutions**:

1. Clear npm cache: `npm cache clean --force`
2. Check network, you can use domestic mirrors

### 8.2 Styles Not Applied

**Problem**: Component styles display abnormally

**Solutions**:

1. Make sure you've imported style file: `import 'element-ui/lib/theme-chalk/index.css'`
2. Check if webpack configuration correctly handles CSS
3. Ensure no CSS conflicts

### 8.3 Components Not Rendering

**Problem**: Components imported but not displaying

**Solutions**:

1. Check if you correctly used Vue.use(ElementUI)
2. Confirm Vue version is 2.6.0 or higher
3. Check browser console for error messages

## 9. Version Migration Suggestions

Element UI maintenance has stopped. Strongly recommend migrating to Element Plus (Vue 3) for new projects. If planning to migrate existing projects, refer to the following steps:

### 9.1 Major Changes

| Change | Element UI | Element Plus |
|--------|------------|--------------|
| Vue Version | Vue 2 | Vue 3 |
| Event Binding | v-model (Vue 2 syntax) | v-model (Vue 3 syntax) |
| Component Registration | Vue.component | app.component |

### 9.2 Migration Tool

Official provides migration tool gogocode:

```bash
# Install globally
npm install -g gogocode

# Run migration
gogocode -s your-project-path -t element-plus -o
```
