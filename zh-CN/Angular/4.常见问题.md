# Angular 常见问题

## 1. 入门问题

### Q: Angular 和 AngularJS 有什么区别？

**A:** AngularJS 是 Angular 1.x 的名称，发布于 2010 年。2016 年，Google 发布了全新的 Angular（从 Angular 2 开始），这是一个完全重写的版本，与 AngularJS 不兼容。主要区别：

- AngularJS 使用 JavaScript，Angular 使用 TypeScript
- Angular 采用组件化架构，而 AngularJS 使用控制器和作用域
- Angular 性能更好，API 更现代

### Q: 学习 Angular 需要什么基础？

**A:** 建议具备以下基础知识：
- HTML 和 CSS
- JavaScript/TypeScript
- 面向对象编程概念

有其他前端框架（如 React、Vue）经验会有帮助，但不是必需的。

### Q: Angular 适合初学者吗？

**A:** Angular 的学习曲线相对较陡，因为它是一个完整的框架，包含很多概念。但对于初学者来说，如果按照官方教程一步步学习，还是可以掌握的。建议先从官方教程 "Tour of Heroes" 开始。

### Q: Angular 最新版本是多少？如何查看？

**A:** 可以通过以下命令查看当前安装的 Angular 版本：

```bash
ng version
```

也可以访问 [Angular 版本说明](https://angular.io/guide/releases) 查看所有版本信息。

## 2. 开发问题

### Q: 组件和模块有什么区别？

**A:** 
- **组件（Component）**：负责渲染 UI 的基本单元，包含模板、样式和逻辑
- **模块（Module）**：用于组织组件、服务和其他模块的容器

在 Angular Standalone API（Angular 14+）中，推荐使用独立组件，不再强制要求使用 NgModule。

### Q: 什么是 Standalone Component？

**A:** Standalone Component（独立组件）是 Angular 14 引入的特性，允许组件不依赖于 NgModule，直接导入其他组件或模块使用。这简化了代码结构，是目前推荐的开发方式。

```typescript
import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { OtherComponent } from './other.component';

@Component({
  selector: 'app-my-component',
  standalone: true,
  imports: [CommonModule, OtherComponent],
  template: `<app-other></app-other>`
})
export class MyComponent {}
```

### Q: 双向数据绑定不起作用怎么办？

**A:** 确保：
1. 已导入 `FormsModule`
2. 正确使用 `[(ngModel)]` 语法

```typescript
import { FormsModule } from '@angular/forms';

@NgModule({
  imports: [BrowserModule, FormsModule]
})
export class AppModule {}
```

### Q: 路由参数如何获取？

**A:** 使用 `ActivatedRoute` 服务：

```typescript
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  this.route.paramMap.subscribe(params => {
    const id = params.get('id');
  });
}
```

### Q: 如何处理 HTTP 错误？

**A:** 使用 RxJS 的 catchError 操作符：

```typescript
import { HttpClient } from '@angular/common/http';
import { catchError, throwError } from 'rxjs';

this.http.get('/api/data').pipe(
  catchError(error => {
    console.error('请求错误', error);
    return throwError(() => error);
  })
).subscribe();
```

### Q: 服务怎么注入？

**A:** 在组件的 constructor 中注入：

```typescript
constructor(private myService: MyService) {}
```

Angular 会自动创建服务的实例。对于服务，需要添加 `@Injectable()` 装饰器：

```typescript
@Injectable({
  providedIn: 'root'
})
export class MyService {}
```

## 3. 性能问题

### Q: 如何优化 Angular 应用性能？

**A:** 

1. **使用懒加载**
   ```typescript
   export const routes: Routes = [
     { path: 'admin', loadComponent: () => import('./admin/admin.component').then(m => m.AdminComponent) }
   ];
   ```

2. **开启生产模式**
   ```bash
   ng build --configuration production
   ```

3. **使用 OnPush 变更检测策略**
   ```typescript
   @Component({
     changeDetection: ChangeDetectionStrategy.OnPush
   })
   export class MyComponent {}
   ```

4. **使用 trackBy 在 *ngFor 中**
   ```html
   <div *ngFor="let item of items; trackBy: trackById">
     {{ item.name }}
   </div>
   ```
   ```typescript
   trackById(index: number, item: any) {
     return item.id;
   }
   ```

5. **使用 Signals**
   Angular 16+ 引入的 Signals 可以提高响应式性能。

### Q: 首屏加载太慢怎么办？

**A:** 
1. 使用 Angular Universal 进行服务器端渲染（SSR）
2. 启用懒加载
3. 优化打包大小
4. 使用 PWA（渐进式 Web 应用）

### Q: 变更检测导致性能问题？

**A:** 可以尝试：
1. 使用 `OnPush` 变更检测策略
2. 使用 `runOutsideAngular` 处理频繁事件
3. 使用 Signals 替代传统的响应式方式

## 4. 构建和部署问题

### Q: 构建失败怎么办？

**A:** 
1. 检查错误信息，通常会指出具体的文件和行号
2. 常见原因：
   - TypeScript 语法错误
   - 缺少依赖
   - 路径配置错误
3. 尝试清理并重新构建：
   ```bash
   rm -rf node_modules dist
   npm install
   ng build
   ```

### Q: 如何部署到生产环境？

**A:** 
1. 构建生产版本：
   ```bash
   ng build --configuration production
   ```

2. 将 `dist/your-project` 目录下的内容部署到 Web 服务器（如 Nginx、Apache）

3. 配置服务器处理 Angular 路由（所有路由都返回 index.html）：

   Nginx 配置示例：
   ```nginx
   location / {
     try_files $uri $uri/ /index.html;
   }
   ```

### Q: 如何处理跨域问题？

**A:** 
1. 开发环境：在 `proxy.conf.json` 中配置代理
   ```json
   {
     "/api": {
       "target": "http://localhost:3000",
       "secure": false
     }
   }
   ```
   然后启动：
   ```bash
   ng serve --proxy-config proxy.conf.json
   ```

2. 生产环境：在服务器端配置 CORS

## 5. 测试问题

### Q: 如何编写单元测试？

**A:** Angular 自带 Jasmine 测试框架。示例：

```typescript
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [MyComponent]
    }).compileComponents();

    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });
});
```

### Q: 测试服务时如何 mock 依赖？

**A:** 使用 TestBed 提供 mock 服务：

```typescript
TestBed.configureTestingModule({
  providers: [
    MyService,
    { provide: HttpClient, useValue: mockHttpClient }
  ]
});
```

**A:** 选择取决于项目需求和团队背景：
