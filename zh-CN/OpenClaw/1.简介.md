# OpenClaw 🦞

OpenClaw 是一个**自托管网关**，它将你喜欢的聊天应用——WhatsApp、Telegram、Discord、iMessage 等——连接到 AI 编码助手（如 Pi）。你可以在自己的电脑或服务器上运行一个 Gateway 进程，它会成为你的消息应用和随时可用的 AI 助手之间的桥梁。

## 目标用户

OpenClaw 面向希望拥有个人 AI 助手的开发者和高级用户。你可以从任何地方通过消息应用与 AI 助手交流，同时无需放弃对数据的控制，也不必依赖托管服务。

## 核心特性

OpenClaw 与众不同的原因在于：

- **自托管**：运行在你的硬件上，你做主
- **多通道**：一个 Gateway 同时服务 WhatsApp、Telegram、Discord 等多个平台
- **原生代理支持**：为编码助手构建，支持工具使用、会话管理、记忆和多代理路由
- **开源**：MIT 许可证，社区驱动

## 系统要求

运行 OpenClaw 你需要：

- Node.js 22 或更高版本
- 一个来自你选择的服务商的 API 密钥
- 5 分钟时间

为了获得最佳质量和安全性，建议使用最新一代最强大的模型。

## 快速开始

```bash
# 安装 OpenClaw
npm install -g openclaw@latest

# 运行入门向导并安装服务
openclaw onboard --install-daemon

# 登录通道并启动 Gateway
openclaw channels login
openclaw gateway --port 18789
```

## 工作原理

Gateway 是会话、路由和通道连接的单一数据源。聊天应用通过它连接到 AI 代理、CLI、Web 控制界面或移动节点。

## 主要功能

- **多通道网关**：一个 Gateway 进程支持 WhatsApp、Telegram、Discord、iMessage
- **插件通道**：通过扩展包添加 Mattermost 等更多平台
- **多代理路由**：每个代理、工作区或发送者都有隔离的会话
- **媒体支持**：支持发送和接收图片、音频和文档
- **Web 控制界面**：浏览器仪表板用于聊天、配置、会话和节点管理
- **移动节点**：配对 iOS 和 Android 节点，实现 Canvas、摄像头/屏幕和语音工作流
