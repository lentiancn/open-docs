# Flutter FAQ

Common questions and answers.

## Installation

### Q: Flutter download failed?

**Solution:**
1. Use mirror:
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
```
2. Use proxy
3. Download manually

### Q: flutter doctor error?

**Solution:**
1. Check PATH
2. Ensure Git is installed
3. Run `flutter doctor -v`

### Q: Android SDK not found?

**Solution:**
```bash
flutter config --android-sdk /path/to/android-sdk
export ANDROID_SDK_ROOT=/path/to/android-sdk
```

## Development

### Q: Hot reload not working?

**Solution:**
1. Save file and wait
2. Check console for errors
3. Restart app completely

### Q: APK build failed?

**Solution:**
1. Run `flutter clean`
2. Update Gradle
3. Check network

### Q: Widget not showing?

**Solution:**
1. Check build() return
2. Add to MaterialApp
3. Check layout constraints

### Q: State not updating?

**Solution:**
1. Ensure setState() is called
2. Check StatefulWidget
3. Use correct Context

### Q: Memory leaks?

**Solution:**
1. Dispose controllers properly
2. Clean up Provider
3. Avoid creating objects in build

## Performance

### Q: App laggy?

**Solution:**
1. Use const constructors
2. Avoid unnecessary setState
3. Use ListView.builder
4. Optimize images

### Q: APK too large?

**Solution:**
1. Use minification
2. Remove debug symbols
3. Use split-per-abi

### Q: Slow startup?

**Solution:**
1. Reduce init operations
2. Lazy load components

## Platform

### Q: iOS build failed?

**Solution:**
1. Check Xcode version
2. Update CocoaPods
3. Check signing

### Q: Web build failed?

**Solution:**
1. Enable Flutter Web
2. Check browser compatibility
3. Update Chrome
