# Nginx 简介

## 什么是 Nginx

Nginx（发音为 "engine x"）是一个 HTTP Web 服务器、反向代理、缓存、负载均衡器、TCP/UDP 代理服务器和邮件代理服务器。

Nginx 最初由 Igor Sysoev 编写，并根据 2 条款 BSD 许可证分发。

## Nginx 特点

Nginx 以其灵活性和高性能、低资源利用率而闻名：

- **全球最受欢迎的 Web 服务器**
- **最受欢迎的 Docker 镜像之一**
- **为 Kubernetes 多个 Ingress Controllers 提供支持**，包括官方的 kubernetes-ingress

### 基本 HTTP 服务器功能

- 提供静态文件和索引文件服务
- 自动索引功能
- 打开文件描述符缓存
- 带缓存的反向代理加速
- 负载均衡和故障转移
- 支持 FastCGI、uwsgi、SCGI 和 memcached 服务器的缓存加速
- 模块化架构
- 过滤器包括：gzip 压缩、字节范围、分块响应、XSLT、SSI 和图像转换
- SSL 和 TLS SNI 支持
- 支持 HTTP/2（基于权重和依赖的优先级）
- 支持 HTTP/3

### 其他 HTTP 服务器功能

- 基于名称和 IP 的虚拟服务器
- Keep-alive 和管道连接支持
- 访问日志格式、缓冲日志写入、快速日志轮换和系统日志
- 3xx-5xx 错误代码重定向
- 重写模块：使用正则表达式更改 URI
- 根据客户端地址执行不同功能
- 基于客户端 IP 地址、密码（HTTP 基本认证）和子请求结果的访问控制
- HTTP referer 验证
- PUT、DELETE、MKCOL、COPY 和 MOVE 方法
- FLV 和 MP4 流媒体
- 响应速率限制
- 限制来自一个地址的并发连接数或请求数
- 基于 IP 的地理位置
- A/B 测试
- 请求镜像
- 内嵌 Perl
- njs 脚本语言

### 邮件代理服务器功能

- 用户重定向到 IMAP 或 POP3 服务器，使用外部 HTTP 认证服务器
- 用户认证使用外部 HTTP 认证服务器，并重定向到内部 SMTP 服务器
- 支持的认证方法：POP3（USER/PASS、APOP、AUTH LOGIN/PLAIN/CRAM-MD5）、IMAP（LOGIN、AUTH LOGIN/PLAIN/CRAM-MD5）、SMTP（AUTH LOGIN/PLAIN/CRAM-MD5）
- SSL 支持
- STARTTLS 和 STLS 支持

### TCP/UDP 代理服务器功能

- TCP 和 UDP 通用代理
- TCP 的 SSL 和 TLS SNI 支持
- 负载均衡和故障转移
- 基于客户端地址的访问控制
- 根据客户端地址执行不同功能
- 限制来自一个地址的并发连接数
- 访问日志格式、缓冲日志写入、快速日志轮换和系统日志
- 基于 IP 的地理位置
- A/B 测试
- njs 脚本语言

### 架构和可扩展性

- 一个主进程和多个工作进程
- 工作进程以非特权用户身份运行
- 灵活的配置
- 重新配置和升级可执行文件而不会中断客户端服务
- 支持 kqueue（FreeBSD 4.1+）、epoll（Linux 2.6+）、/dev/poll（Solaris 7 11/99+）、event ports（Solaris 10）、select 和 poll
- 支持各种 kqueue 特性
- 支持各种 epoll 特性
- 支持 sendfile、sendfile64 和 sendfilev
- 支持 File AIO
- 支持 DIRECTIO
- 支持 Accept-filters 和 TCP_DEFER_ACCEPT
- 10,000 个非活跃 HTTP keep-alive 连接仅占用约 2.5M 内存
- 最小化数据复制操作

## Nginx 优势

### 高性能

Nginx 采用事件驱动的异步非阻塞模型，能够处理高并发连接，同时保持低内存占用。

### 低资源消耗

相比其他 Web 服务器，Nginx 在同等并发下消耗更少的内存和 CPU。

### 高稳定性

Nginx 的稳定性经过全球数百万网站的验证，极少出现崩溃。

### 简单配置

Nginx 的配置文件简洁明了，易于理解和维护。

### 扩展性强

支持丰富的模块，可以根据需要启用或禁用相应功能。

### 热部署

可以在不中断服务的情况下重载配置和升级版本。

## 应用场景

Nginx 广泛应用于以下场景：

- **Web 服务器**：作为静态文件的 HTTP 服务器
- **反向代理**：作为后端服务器的反向代理
- **负载均衡**：作为多台后端服务器的负载均衡器
- **缓存服务器**：作为 CDN 或内部缓存服务器
- **API 网关**：作为微服务的入口网关
- **邮件代理**：作为 POP3、IMAP、SMTP 代理
- **TCP/UDP 代理**：作为 TCP 和 UDP 协议的代理
