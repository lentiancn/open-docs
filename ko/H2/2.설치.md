# H2 설치 가이드

## 환경 요구사항

- Java 11 이상
- Java를 지원하는 모든 운영 체제 (Windows, Linux, macOS 등)

## 설치 방법

### 방법 1: ZIP 패키지 다운로드

1. 공식 웹사이트 다운로드 페이지 방문: https://www.h2database.com/html/download.html
2. "Platform-Independent Zip" 버전 다운로드
3. 다운로드한 ZIP 파일을 모든 디렉토리에 압축 해제, 예: C:\h2 또는 /opt/h2

### 방법 2: Maven 의존성

Maven 또는 Gradle을 사용하는 Java 프로젝트가 있는 경우 의존성을 직접 추가할 수 있습니다:

**Maven:**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle:**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### 방법 3: Windows 인스톨러

1. "Windows Installer" 버전 다운로드
2. 인스톨러를 실행하고 마법사를 따라 설치 완료

## 환경 변수 구성 (선택 사항)

ZIP 패키지를 다운로드하고 명령줄에서 H2를 직접 사용하려면:

**Windows:**
1. "이 컴퓨터"를 우클릭 → "속성"
2. "고급 시스템 설정" 클릭
3. "환경 변수" 클릭
4. 시스템 변수에서 Path를 찾아 "편집" 클릭
5. H2 bin 디렉토리 경로 추가

**Linux/macOS:**
```bash
export PATH=$PATH:/opt/h2/bin
```

## 설치 확인

### 명령줄 사용

압축 해제 후 명령줄에서 실행:

```bash
java -jar h2.jar
```

이렇게 하면 H2 웹 콘솔이 시작됩니다.

### JDBC 연결 사용

Java 코드에서 H2 데이터베이스 연결:

```java
// 임베디드 모드
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// 파일 데이터베이스
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### H2 콘솔 사용

1. 명령 실행: `java -jar h2.jar`
2. 브라우저에서 열기: http://localhost:8082
3. 기본 사용자 이름: sa, 비밀번호 비어 있음

## 연결 URL 형식

H2는 다양한 연결 URL 형식을 지원합니다:

| 모드 | URL 예 | 설명 |
|------|--------|------|
| 메모리 모드 | jdbc:h2:mem:testdb | 데이터가 메모리에 저장 |
| 임베디드 파일 | jdbc:h2:file:./data/testdb | 데이터가 파일에 저장 |
| TCP 서버 | jdbc:h2:tcp://localhost:9092/testdb | 클라이언트-서버 모드 |
| TCP 암호화 | jdbc:h2:ssl://localhost:9092/testdb | SSL 암호화 연결 |
| PostgreSQL 호환 | jdbc:h2:postgres://localhost:5432/testdb | PostgreSQL 프로토콜 호환 모드 |

## 일반적인 문제

### 포트 사용 중
H2는 기본적으로 포트 9092(TCP 모드)와 8082(웹 콘솔)를 사용합니다. 포트가 사용 중이면 연결 URL에서 다른 포트를 지정할 수 있습니다.

### Java 버전 문제
Java 11 이상이 설치되어 있는지 확인합니다. 다음 명령으로 확인할 수 있습니다:

```bash
java -version
```

### 드라이버를 찾을 수 없음
H2 JAR 파일이 classpath에 있는지 확인합니다. Maven 프로젝트의 경우 의존성이 올바르게 추가되었는지 확인합니다.
