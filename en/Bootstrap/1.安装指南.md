## Bootstrap 简介

Bootstrap 是最流行的 HTML、CSS 和 JS 框架，用于开发响应式布局、移动设备优先的网站和 Web 应用程序。

### Bootstrap 的特点

- **响应式设计**：内置响应式栅格系统
- **预定义样式**：丰富的组件和样式类
- **JavaScript 组件**：交互式组件（模态框、轮播等）
- **定制化**：通过 Sass 变量自定义主题
- **浏览器兼容**：支持所有现代浏览器

### Bootstrap 版本历史

| 版本 | 发布时间 | 主要特性 |
|------|----------|----------|
| Bootstrap 5 | 2021年5月 | 移除了 jQuery 依赖、改进的表单组件、新的 Utilities API |
| Bootstrap 4 | 2018年1月 | Flexbox 支持、Sass 替代 Less、新的组件 |
| Bootstrap 3 | 2013年8月 | 移动设备优先设计 |
| Bootstrap 2 | 2012年1月 | 响应式设计改进 |
| Bootstrap 1 | 2011年8月 | 初始版本 |

> **注意**：Bootstrap 5 是当前主要版本，建议新项目使用。Bootstrap 4 仍有维护更新，但新功能开发已停止。

## CDN 安装

### CSS only

将以下代码添加到 HTML 的 `<head>` 部分：

```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" crossorigin="anonymous">
```

### CSS + JavaScript

添加 CSS 后，在 `</body>` 前添加 JS：

```html
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js" integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz" crossorigin="anonymous"></script>
```

> **注意**：Bootstrap 5 使用 `bootstrap.bundle.min.js`（包含 Popper），不再需要单独引入 Popper。

### 使用国内 CDN（推荐国内用户）

```html
<!-- CSS -->
<link href="https://cdn.staticfile.org/twitter-bootstrap/5.3.3/css/bootstrap.min.css" rel="stylesheet">

<!-- JS -->
<script src="https://cdn.staticfile.org/twitter-bootstrap/5.3.3/js/bootstrap.bundle.min.js"></script>
```

## npm 安装

### 安装 Bootstrap

```bash
# 使用 npm 安装
npm install bootstrap

# 安装指定版本
npm install bootstrap@5.3.3

# 安装 Bootstrap 4
npm install bootstrap@4.6.2
```

### 在项目中引用

**CSS**：
```javascript
// webpack 或 bundler
import 'bootstrap/dist/css/bootstrap.min.css';
```

**JavaScript**：
```javascript
import * as bootstrap from 'bootstrap';

// 或只导入需要的模块
import { Modal } from 'bootstrap';
```

## 使用包管理器安装

### yarn

```bash
yarn add bootstrap
```

### pnpm

```bash
pnpm add bootstrap
```

### Ruby on Rails

在 Gemfile 中添加：

```ruby
gem 'bootstrap', '~> 5.3'
```

然后运行 `bundle install`。

### Python/Django

```bash
pip install bootstrap5
```

或在 requirements.txt 中添加：

```
django-bootstrap5
```

## 下载安装

### 下载预编译版本

1. 访问 [Bootstrap 官网](https://getbootstrap.com/)
2. 点击 "Download" 按钮
3. 下载预编译的 CSS 和 JS 文件

### 下载包含源码的版本

下载包含 Sass 源文件的版本：

1. 访问 [Bootstrap 源码](https://github.com/twbs/bootstrap/releases)
2. 下载源码压缩包
3. 解压后包含：
   - `dist/css/` - 编译后的 CSS
   - `dist/js/` - 编译后的 JS
   - `scss/` - Sass 源文件

### 文件结构

```
bootstrap/
├── css/
│   ├── bootstrap.css
│   ├── bootstrap.css.map
│   ├── bootstrap.min.css
│   ├── bootstrap.min.css.map
│   ├── bootstrap.rtl.css
│   └── bootstrap.rtl.min.css
├── js/
│   ├── bootstrap.bundle.js
│   ├── bootstrap.bundle.min.js
│   ├── bootstrap.esm.js
│   └── bootstrap.esm.min.js
└── scss/
    └── ...
```

## 验证安装

### 基本 HTML 模板

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Bootstrap 测试</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
  <div class="container mt-5">
    <h1>Hello, Bootstrap!</h1>
    <button class="btn btn-primary">主要按钮</button>
  </div>
  
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
```

### 验证步骤

1. 保存上述代码为 `.html` 文件
2. 在浏览器中打开
3. 如果看到带有 Bootstrap 样式的页面，说明安装成功

## 快速开始

### 使用栅格系统

```html
<div class="container">
  <div class="row">
    <div class="col-md-4">列 1</div>
    <div class="col-md-4">列 2</div>
    <div class="col-md-4">列 3</div>
  </div>
</div>
```

### 使用组件

**按钮**：
```html
<button class="btn btn-primary">主要</button>
<button class="btn btn-secondary">次要</button>
<button class="btn btn-success">成功</button>
<button class="btn btn-danger">危险</button>
<button class="btn btn-warning">警告</button>
<button class="btn btn-info">信息</button>
```

**卡片**：
```html
<div class="card" style="width: 18rem;">
  <img src="..." class="card-img-top" alt="...">
  <div class="card-body">
    <h5 class="card-title">卡片标题</h5>
    <p class="card-text">卡片内容</p>
    <a href="#" class="btn btn-primary">跳转链接</a>
  </div>
</div>
```

**模态框**：
```html
<!-- 触发按钮 -->
<button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#exampleModal">
  打开模态框
</button>

<!-- 模态框 -->
<div class="modal fade" id="exampleModal" tabindex="-1">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title">模态框标题</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal"></button>
      </div>
      <div class="modal-body">
        模态框内容
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">关闭</button>
        <button type="button" class="btn btn-primary">保存</button>
      </div>
    </div>
  </div>
</div>
```

## 常见问题

### Q1: Bootstrap 5 和 Bootstrap 4 有什么区别？

主要区别：
- Bootstrap 5 移除了 jQuery 依赖
- 使用原生 JavaScript
- 改进的表单组件
- 新的 Utilities API
- 不再支持 IE 浏览器

### Q2: 如何自定义 Bootstrap 主题？

**方式一：使用 Sass**
```scss
// 自定义变量
$primary: #ff6600;
$secondary: #cccccc;

// 导入 Bootstrap
@import "bootstrap";
```

**方式二：使用 Bootstrap 自定义工具**
访问 https://getbootstrap.com/docs/5.3/customize/options/

### Q3: Bootstrap 是否响应式？

是的，Bootstrap 是移动设备优先的设计：
- `col-` 超小屏幕（<576px）
- `col-sm-` 小屏幕（≥576px）
- `col-md-` 中等屏幕（≥768px）
- `col-lg-` 大屏幕（≥992px）
- `col-xl-` 超大屏幕（≥1200px）
- `col-xxl-` 特大屏幕（≥1400px）

### Q4: 如何使用 Bootstrap 图标？

```bash
npm install bootstrap-icons
```

或使用 CDN：

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css">
```

使用：
```html
<i class="bi bi-github"></i>
```

### Q5: Bootstrap 5 是否支持 IE11？

不支持。Bootstrap 5 需要现代浏览器。如果需要 IE11 支持，请使用 Bootstrap 4.6。

### Q6: 如何调试 Bootstrap 样式？

1. 使用浏览器开发者工具（F12）
2. 检查元素的类名
3. 查看 Computed 面板中的样式来源

### Q7: Bootstrap 与其他框架冲突怎么办？

- 使用特定的选择器
- 利用 Bootstrap 的命名空间
- 使用 Bootstrap 提供的 API

### Q8: 如何升级 Bootstrap 版本？

1. 备份当前项目
2. 更新 CDN 链接或 npm 包版本
3. 检查破坏性更改
4. 测试所有功能
