# Mockito Installationsanleitung

## Umgebungsanforderungen

Bevor Sie Mockito installieren, stellen Sie sicher, dass Ihre Entwicklungsumgebung die folgenden Anforderungen erfuellt.

Mockito 5.x erfordert JDK 11 oder höher. Wenn Ihr Projekt JDK 8 unterstützen muss, können Sie die Mockito 4.x-Serie verwenden. Es wird empfohlen, die neueste Version von Mockito zu verwenden, um die neuesten Funktionen und Sicherheitsfixes zu erhalten.

## Maven-Installation

### Abhängigkeiten hinzufuegen

Fuegen Sie die mockito-core-Abhaigkeit zur pom.xml-Datei Ihres Maven-Projekts hinzu. Es wird empfohlen, eine Eigenschaft zu definieren, um die Mockito-Version für ein besseres Versionsmanagement anzugeben.

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Wenn Sie die Inline-Funktion von Mockito zum Mocken von finalen Klassen und Methoden benoetigen, muessen Sie auch die mockito-inline-Abhaigkeit hinzufuegen.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### Snapshot-Versionen

Mockito veröffentlicht nach jeder Codeaenderung Snapshot-Versionen im Sonatype-Repository. Wenn Sie die neuesten Entwicklungsfunktionen verwenden muessen, können Sie die Repository-Adresse zur pom.xml hinzufuegen.

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Gradle-Installation

### Abhängigkeiten hinzufuegen

Fuegen Sie die Abhaengigkeitsdeklaration zur build.gradle-Datei Ihres Gradle-Projekts hinzu.

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

Wenn Sie die Kotlin-DSL-Datei von Gradle (build.gradle.kts) verwenden, ist die Syntax leicht unterschiedlich.

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### BOM zur Versionsverwaltung verwenden

Die Verwendung des Spring Dependency Management-Plugins oder der Gradle-BOM-Funktion kann die Versionsverwaltung vereinfachen.

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## Manuelle Download

Wenn Ihr Projekt Maven oder Gradle nicht verwendet, können Sie die JAR-Dateien direkt aus dem Maven Central Repository herunterladen. Besuchen Sie Maven Central und suchen Sie nach org.mockito, um alle Versionen von mockito-core und mockito-all (veraltet) zu finden.

Nach dem Download fuegen Sie die JAR-Dateien zum Klassenpfad Ihres Projekts hinzu. Beachten Sie, dass Sie auch abhaengige Bibliotheken wie Byte Buddy und Objenesis herunterladen muessen.

## Integration mit Spring Boot

Spring Boot's spring-boot-starter-test enthaelt bereits die Mockito-Abhaigkeit. Wenn Ihr Projekt Spring Boot verwendet, muessen Sie im Allgemeinen keine zusaetzliche Mockito-Abhaigkeit hinzufuegen.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Wenn Sie Mocks bequemer in einer Spring-Umgebung verwenden moechten, können Sie die @MockBean-Annotation verwenden, die Mock-Objekte automatisch in den Spring-Container injiziert.

## Integration mit JUnit 5

### MockitoExtension verwenden

JUnit 5 fuehrte ein Erweiterungsmodell ein, und Mockito bietet MockitoExtension zur Unterstützung von JUnit 5. Es ist keine zusaetzliche Abhaigkeitskonfiguration erforderlich, da mockito-core bereits die erforderliche Unterstützung enthaelt.

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### MockitoJUnitRunner verwenden

Fuer JUnit 4 können Sie MockitoJUnitRunner verwenden, um Mock-Objekte automatisch zu initialisieren.

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## Installation verifizieren

Nach der Installation können Sie verifizieren, dass Mockito korrekt konfiguriert ist, indem Sie eine einfache Testklasse erstellen. Wenn der Test nach dem Ausfuehren besteht, ist Mockito korrekt installiert.

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito erfolgreich installiert!");
    }
}
```

## Empfehlungen zur Versionsauswahl

Fuer neue Projekte wird empfohlen, die neueste Mockito 5.x-Version zu verwenden. Wenn Ihr Projekt JDK 8-Kompatibilitaet benötigt, ist die Mockito 4.x-Serie verfuegbar. Verwenden Sie immer die neueste stabile Version für bessere Leistung und Sicherheit.
