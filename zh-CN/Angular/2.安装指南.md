# Angular 安装指南

## 环境要求

在安装 Angular 之前，你需要确保开发环境满足以下要求：

### 1. Node.js

Angular 需要 Node.js 环境。请访问 [Node.js 官网](https://nodejs.org/) 下载并安装。

**推荐版本：**
- Node.js 18.10.0 或更高版本
- npm 9.0.0 或更高版本

你可以使用以下命令检查当前版本：

```bash
node --version
npm --version
```

### 2. 代码编辑器

推荐使用以下编辑器：
- **Visual Studio Code**（免费，推荐）
- **WebStorm**（付费，功能强大）
- **Sublime Text**（轻量级）

## 安装步骤

### 方法一：使用 npm 安装

这是最推荐的安装方式。

1. **安装 Angular CLI**

Angular CLI 是 Angular 的命令行工具，用于创建项目、生成代码、运行开发服务器等。

```bash
npm install -g @angular/cli
```

> `-g` 参数表示全局安装，这样你可以在任何目录下使用 `ng` 命令。

2. **验证安装**

```bash
ng version
```

如果安装成功，你会看到类似下面的输出：

```
Angular CLI: 19.0.0
Node: 20.10.0
Package Manager: npm 9.0.0
OS: win32 x64
```

### 方法二：使用 Yarn 安装

如果你更喜欢 Yarn：

```bash
yarn global add @angular/cli
```

### 方法三：使用 npx 运行

如果你不想全局安装，可以使用 npx：

```bash
npx @angular/cli new my-app
```

## 创建第一个 Angular 项目

### 1. 创建新项目

```bash
ng new my-app
```

> `my-app` 是你的项目名称，可以自定义。

在创建过程中，CLI 会询问几个问题：

```
? Would you like to add Angular routing? (y/N)
```
是否添加路由功能？输入 `y` 或 `N`，然后按回车。

```
? Which stylesheet format would you like to use?
  - CSS
  - SCSS
  - Sass
  - Less
```
选择样式格式。一般选择 CSS 或 SCSS。

### 2. 进入项目目录

```bash
cd my-app
```

### 3. 启动开发服务器

```bash
ng serve
```

默认情况下，开发服务器运行在 `http://localhost:4200/`。

你也可以指定端口：

```bash
ng serve --port 4201
```

### 4. 在浏览器中查看

打开浏览器，访问 `http://localhost:4200/`，你应该能看到 Angular 的欢迎页面。

## 项目结构

创建项目后，你会看到以下目录结构：

```
my-app/
├── e2e/                  # 端到端测试
├── node_modules/         # 依赖包
├── src/
│   ├── app/              # 应用代码
│   │   ├── app.component.ts    # 根组件
│   │   ├── app.config.ts       # 应用配置
│   │   └── app.routes.ts       # 路由配置
│   ├── assets/           # 静态资源（图片等）
│   ├── index.html        # 入口 HTML
│   ├── main.ts           # 应用入口
│   └── styles.css        # 全局样式
├── angular.json          # Angular 配置文件
├── package.json          # 项目依赖配置
└── tsconfig.json         # TypeScript 配置
```

## 常用命令

### 生成代码

```bash
# 创建组件
ng generate component my-component
# 或简写
ng g c my-component

# 创建服务
ng generate service my-service
ng g s my-service

# 创建模块
ng generate module my-module
ng g m my-module

# 创建类
ng generate class my-class
ng g cl my-class
```

### 构建项目

```bash
# 开发构建
ng build

# 生产构建（优化后的代码）
ng build --configuration production
```

### 运行测试

```bash
# 运行单元测试
ng test

# 运行端到端测试
ng e2e
```

## 更新 Angular

如果需要将 Angular 更新到新版本：

1. **查看当前版本**

```bash
ng version
```

2. **使用 update 命令**

```bash
ng update @angular/cli @angular/core
```

> 注意：大版本更新（如从 17 更新到 18）可能需要额外的迁移步骤，请参考官方升级指南。

## 常见问题

### 1. 安装失败

如果安装过程中遇到错误：

- 清除 npm 缓存：`npm cache clean --force`
- 使用管理员权限运行
- 检查 Node.js 版本是否满足要求

### 2. ng 命令找不到

确保 Angular CLI 已正确全局安装：

```bash
npm list -g @angular/cli
```

如果找不到，尝试重新安装。

### 3. 端口被占用

如果 4200 端口被占用，可以指定其他端口：

```bash
ng serve --port 4201
```
