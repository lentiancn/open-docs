# FantasyTalking - Installationsanleitung

Vollständige Installations- und Konfigurationsanleitung.

## Anforderungen

### Hardware

- **GPU**: NVIDIA-Grafikkarte, 8GB+ VRAM (16GB empfohlen)
- **Speicher**: Mindestens 20GB freier Speicherplatz
- **RAM**: Mindestens 16GB

### Software

- **Betriebssystem**: Linux (Ubuntu 20.04+), Windows 10/11, macOS
- **Python**: 3.8 - 3.10
- **CUDA**: 11.8 oder höher
- **cuDNN**: 8.0 oder höher

## Installation

### Methode 1: Aus dem Quellcode (Empfohlen)

```bash
# 1. Repository klonen
git clone https://github.com/FantasyTalking/FantasyTalking.git
cd FantasyTalking

# 2. Virtuelle Umgebung erstellen (empfohlen)
python -m venv venv
source venv/bin/activate  # Linux/macOS
# oder
venv\Scripts\activate  # Windows

# 3. Abhängigkeiten installieren
pip install -r requirements.txt

# 4. Vortrainierte Modelle herunterladen
python download_models.py
```

### Methode 2: Docker (Empfohlen)

```bash
# 1. Image herunterladen
docker pull fantasytalking/fantasytalking:latest

# 2. Container ausführen
docker run -it --gpus all -v /your/data:/data fantasytalking/fantasytalking:latest
```

### Methode 3: pip

```bash
pip install fantasytalking
```

## Überprüfung

Nach der Installation führen Sie folgenden Befehl aus:

```bash
python -c "import fantasytalking; print(fantasytalking.__version__)"
```

## Konfiguration

### Konfigurationsdatei

Erstellen Sie `config.yaml` im Projektverzeichnis:

```yaml
model:
  checkpoint_path: "./models/checkpoint.pth"
  face_enhancer: "realusf"

output:
  resolution: "512x512"
  fps: 30

audio:
  sample_rate: 16000
```

### Umgebungsvariablen

```bash
export FT_MODEL_PATH="./models"
export FT_OUTPUT_PATH="./output"
```

## Fehlerbehebung

### CUDA nicht verfügbar?

1. Überprüfen Sie, ob der NVIDIA-Treiber installiert ist: `nvidia-smi`
2. Überprüfen Sie die CUDA-Version
3. Installieren Sie die passende PyTorch-Version

### Zu wenig Speicher?

1. Verwenden Sie kleinere Eingabebilder
2. Verringern Sie die Ausgabeauflösung
3. Verwenden Sie den Stapelverarbeitungsmodus

### Modell-Download fehlgeschlagen?

```bash
# Proxy verwenden
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890

# Manuell herunterladen
python download_models.py --mirror china
```

## Deinstallation

```bash
pip uninstall fantasytalking
rm -rf FantasyTalking
```
