# IBM Informix よくある質問

## 導入に関する質問

### Q1: Informix のバージョンを確認するには？

```bash
onstat -V          # サーバーバージョン情報
oninit -V           # 実行ファイルのバージョン
```

または、`sysmaster` データベースの `sysversion` テーブルをクエリしてください。

### Q2: データベースが起動しない場合の対処法は？

1. **エラーログの確認**
   ```bash
   tail -f $INFORMIXDIR/logs/online.log
   ```

2. **環境変数の確認**
   ```bash
   echo $INFORMIXDIR
   echo $INFORMIXSERVER
   echo $ONCONFIG
   ```

3. **設定ファイルの確認**
   - `sqlhosts` ファイルのエントリ是否存在
   - `onconfig` ファイルのパラメータ確認

4. **プロセスの確認**
   ```bash
   ps -ef | grep oninit
   ```

## 接続に関する質問

### Q3: 接続エラーが発生した時の対処法は？

1. **ファイアウォール確認**
   ```bash
   # Linux
   firewall-cmd --list-ports
   # または
   iptables -L
   ```

2. **ポート確認**
   ```bash
   netstat -an | grep 9088
   ```

3. **sqlhosts 確認**
   ```
   ol_informix1210    onipcso    ol_informix1210    on_hostname    9088
   ```

4. **接続テスト**
   ```bash
   telnet localhost 9088
   ```

## バックアップに関する質問

### Q4: バックアップとリストア方法は？

**オンラインバックアップ**

```bash
# レベル0バックアップ
ontape -s -L 0

# レベル1バックアップ
ontape -s -L 1

# レベル2バックアップ
ontape -s -L 2
```

**リストア**

```bash
# フルリストア
ontape -r

# 特定の時点へのリストア
ontape -r -t "2024-01-15 10:00:00"
```

## パフォーマンスに関する質問

### Q5: パフォーマンスを向上させる方法は？

1. **インデックスの作成**
   ```sql
   CREATE INDEX idx_column ON table_name(column_name);
   ```

2. **統計情報の更新**
   ```sql
   UPDATE STATISTICS;
   UPDATE STATISTICS FOR TABLE table_name;
   ```

3. **SQLの最適化**
   - WHERE句にインデックス列を使用
   - SELECT * を避け、必要な列のみ指定
   - サブクエリの代わりにJOINを使用

4. **パラメータ調整**
   - `BUFFERS`、`SHMVIRTSIZE`、`SHMADD` などの設定

5. **onstat コマンドでの監視**
   ```bash
   onstat -g <options>  # 各種監視情報
   ```

## ロックに関する質問

### Q6: ロック競合やデッドロックが発生した時の対処法は？

1. **ロック情報の確認**
   ```bash
   onstat -g lck
   ```

2. **長時間実行SQLの確認**
   ```bash
   onstat -g sql
   ```

3. **ロックタイムアウトの設定**
   ```sql
   SET LOCK MODE TO WAIT 30;  -- 30秒待機
   SET LOCK MODE TO NOT WAIT; -- 待機しない
   ```

4. **問題のセッションを終了**
   ```bash
   onmode -z <session_id>
   ```

## 運用に関する質問

### Q7: Informix を安全に停止するには？

```bash
# セーフ停止（接続中の処理を完了させてから停止）
onmode -sk

# 即時停止
onmode -ky

# メンテナンスモードへ移行
onmode -su
```

**注意**：データベース稼働中の電源断や強制終了は、データ破損の原因になります。必ず上記の停止コマンドを使用してください。

### Q8: JDBC 接続で文字化けが発生する時の対処法は？

1. **JDBC URLに文字コード指定を追加**
   ```java
   String url = "jdbc:informix-sqli://localhost:9088/testdb:INFORMIXSERVER=ol_informix1210;client_locale=ja_JP.UTF-8";
   ```

2. **環境変数の設定**
   ```bash
   export DB_LOCALE=ja_JP.UTF-8
   export CLIENT_LOCALE=ja_JP.UTF-8
   ```

3. **JDKの文字コード確認**
   `-Dfile.encoding=UTF-8` オプションを付与

### Q9: テーブルやデータベースの容量を確認するには？

```bash
# テーブル容量
onstat -t

# データベース容量
onstat -d

# チャンク情報
onstat -d
```

または、`sysmaster` データベースをクエリ：

```sql
SELECT dbsname, sum(chksize) * page_size as size
FROM sysdatabases db, syschunks ck
WHERE db.dbsnum = ck.dbsnum
GROUP BY dbsname;
```

### Q10: Informix のセキュリティ設定方法は？

1. **ユーザー認証**
   ```sql
   CREATE USER informix WITH PASSWORD 'strong_password';
   GRANT DBA TO informix;
   ```

2. **SSL/TLS接続の有効化**
   - `onconfig` の `SSL_KEYFILE`、`SSL_CERTFILE` 設定
   - `sqlhosts` に `ssl` オプション追加

3. **監査ログの設定**
   ```sql
   INSERT INTO sysadmin:task('configure inline audit', 'on');
   ```

4. **ファイアウォール設定**
   必要なポートのみ開放
