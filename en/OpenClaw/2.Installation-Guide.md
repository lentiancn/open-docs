# OpenClaw Installation Guide

This guide provides detailed installation steps for OpenClaw across different operating systems.

## Requirements

- **Node.js**: version 22 or higher
- **Operating System**: macOS, Linux, or Windows
- **Optional**: Docker (for sandbox functionality)

## Installation Methods

### macOS / Linux (Recommended)

The easiest way to install OpenClaw is using the installation script:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

This script automatically detects your system environment, installs necessary dependencies, and configures OpenClaw.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Direct npm Installation

If you already have Node.js installed, you can also use npm directly:

```bash
npm install -g openclaw@latest
```

### Installation from Source

If you want the latest development version or need a custom installation:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Post-Installation Setup

### Run the Onboarding Wizard

After installation, run the wizard to configure authentication, Gateway settings, and optional channels:

```bash
openclaw onboard --install-daemon
```

The wizard will guide you through:
- API key configuration
- Gateway port settings
- Channel connections (WhatsApp, Telegram, Discord, etc.)
- Security and permission settings

### Check Gateway Status

If you installed the service as shown above, the Gateway should already be running:

```bash
openclaw gateway status
```

### Start Gateway (Foreground)

If you want to run the Gateway in the foreground (useful for testing or troubleshooting):

```bash
openclaw gateway --port 18789
```

### Open the Control UI

```bash
openclaw dashboard
```

Or simply open in your browser: `http://127.0.0.1:18789/`

## Environment Variables

If you need to customize config or state directory locations, you can use these environment variables:

- `OPENCLAW_HOME`: sets the home directory for internal path resolution
- `OPENCLAW_STATE_DIR`: overrides the state directory
- `OPENCLAW_CONFIG_PATH`: overrides the config file path

## Verify Installation

After installation, you can:

1. Run `openclaw --version` to confirm the version
2. Run `openclaw doctor` to check system health
3. Open the Control UI to send a test message

## Common Installation Issues

### Checking Node.js Version

```bash
node --version
```

Make sure the version is 22 or higher. If not, upgrade Node.js first.

### Git Not Found on Windows

Make sure Git is installed and added to your system PATH. You can download it from [git-scm.com](https://git-scm.com).

### Permission Errors

If you encounter permission errors during installation, try:

```bash
sudo npm install -g openclaw@latest
```
