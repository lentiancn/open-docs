# Jenkins 使用手冊

## 流水線基礎

流水線是 Jenkins 的核心功能，它將軟體開發過程中的建置、測試、部署等環節自動化地串聯起來。

## 什麼是流水線

流水線（Pipeline）是 Jenkins 的一套外掛程式，用於實現和整合持續交付流程。簡單來說，流水線就是將軟體從版本控制到用戶手中的整個過程自動化表達。

流水線定義在文字檔案（Jenkinsfile）中，可以提交到專案的程式碼倉庫進行版本管理。這實現了「流水線即程式碼」的理念。

### 流水線的優勢

- **程式碼化**：流水線作為程式碼版本化，便於審查和迭代
- **持久性**：流水線可以從 Jenkins 主節點的計畫內和計畫外重啟中存活
- **可暫停**：流水線可以選擇性停止，等待人工輸入或批准後再繼續
- **多功能**：支援複雜的現實世界 CD 需求，包括平行執行、迴圈等
- **可擴展**：透過外掛程式可以擴展流水線 DSL 和整合其他工具

### 流水線概念

#### 節點

節點是 Jenkins 環境的一部分，是能夠執行流水線的機器。節點可以是主節點，也可以是從節點。

#### 階段

階段（Stage）定義了流水線中概念上不同的子集，比如「建置」、「測試」、「部署」階段。階段用於視覺化流水線的執行狀態。

#### 步驟

步驟（Step）是 Jenkins 流水線中的基本動作，告訴 Jenkins 在特定時間點要做什麼。比如執行 shell 命令使用 `sh` 步驟。

### 流水線語法

#### 聲明式流水線

聲明式流水線是較新的語法，提供更豐富的特性，更容易編寫和閱讀：

```groovy
pipeline {
    agent any
    
    stages {
        stage('建置') {
            steps {
                echo '正在建置應用程式...'
            }
        }
        stage('測試') {
            steps {
                echo '正在執行測試...'
            }
        }
        stage('部署') {
            steps {
                echo '正在部署應用程式...'
            }
        }
    }
}
```

#### 腳本化流水線

腳本化流水線使用 Groovy 語法編寫，更靈活但需要更多的程式設計經驗：

```groovy
node {
    stage('建置') {
        echo '正在建置應用程式...'
    }
    stage('測試') {
        echo '正在執行測試...'
    }
    stage('部署') {
        echo '正在部署應用程式...'
    }
}
```

## 建立第一個流水線

### 使用 Blue Ocean 建立

Blue Ocean 是 Jenkins 的現代化介面，讓建立流水線更加直觀：

1. 安裝 Blue Ocean 外掛程式（如未預設安裝）
2. 點擊「開啟 Blue Ocean」
3. 點擊「建立流水線」
4. 選擇程式碼儲存位置（GitHub、GitLab 等）
5. 選擇倉庫和分支
6. Blue Ocean 會自動偵測 Jenkinsfile 或幫助建立

### 使用 Jenkinsfile 建立

在專案中建立 Jenkinsfile，然後 Jenkins 會自動識別：

1. 在專案根目錄建立 Jenkinsfile 檔案
2. 定義流水線的各個階段
3. 將程式碼提交到版本控制系統
4. 在 Jenkins 中建立流水線任務並關聯倉庫

## 憑證管理

Jenkins 需要存取各種外部資源，如程式碼倉庫、部署目標等，這些都需要憑證管理。

### 新增憑證

1. 進入「Manage Jenkins」->「Manage Credentials」
2. 點擊「Add Credentials」
3. 選擇憑證類型（使用者名稱密碼、SSH 金鑰、Secret 檔案等）
4. 填寫憑證資訊
5. 為憑證設定 ID 和描述

### 在流水線中使用憑證

使用憑證步驟調用：

```groovy
pipeline {
    agent any
    stages {
        stage('使用憑證') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "使用者名稱: $USER"
                }
            }
        }
    }
}
```

## 建置觸發器

自動化建置需要設定觸發器，Jenkins 支援多種觸發方式。

### 輪詢程式碼倉庫

定期檢查程式碼倉庫是否有變更：

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // 每5分鐘檢查一次
    }
    stages {
        stage('建置') {
            steps {
                echo '偵測到程式碼變更，開始建置'
            }
        }
    }
}
```

### Webhook 觸發

透過程式碼倉庫的 Webhook 觸發建置，效率更高：

1. 在 Jenkins 中安裝對應的外掛程式（如 GitHub Hook Trigger）
2. 設定程式碼倉庫的 Webhook 指向 Jenkins
3. 程式碼提交時自動觸發建置

### 排程建置

按照設定的時間表自動執行：

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // 每天凌晨2點執行
    }
    stages {
        stage('排程建置') {
            steps {
                echo '排程建置任務執行中'
            }
        }
    }
}
```

## 使用代理程式和容器

### 設定代理程式

Jenkins 支援分散式建置，可以在多台機器上分配建置任務：

1. 進入「Manage Jenkins」->「Manage Nodes」
2. 新增代理程式節點，設定連線資訊
3. 在流水線中指定代理程式

### 在流水線中使用 Docker

聲明式流水線可以在 Docker 容器中執行建置：

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('建置') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## 測試報告

Jenkins 可以整合各類測試框架，產生視覺化的測試報告。

### JUnit 測試報告

```groovy
pipeline {
    agent any
    stages {
        stage('測試') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### 其他測試報告

Jenkins 還支援多種測試報告外掛程式：

- TestNG
- Cobertura（程式碼覆蓋率）
- JaCoCo（程式碼覆蓋率）
- ESLint（前端程式碼檢查）

## 部署實踐

### 部署到不同環境

典型的多階段部署流程：

```groovy
pipeline {
    agent any
    stages {
        stage('建置') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('測試環境部署') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('生產環境部署') {
            when {
                branch 'main'
            }
            steps {
                input message: '確認部署到生產環境?', ok: '部署'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### 藍綠部署

透過 Jenkins 可以實現藍綠部署，零停機更新：

1. 維護兩套相同的環境（藍環境和綠環境）
2. 流量切換到新版本
3. 如有問題快速回滾

## 最佳實踐

### Jenkinsfile 最佳實踐

- 將 Jenkinsfile 納入版本控制
- 使用聲明式流水線語法
- 保持流水線簡潔清晰
- 合理劃分階段

- 設定適當的存取控制
- 啟用安全日誌稽核

### 效能優化

- 合理設定建置節點
- 使用增量建置減少耗時
- 設定建置快取
- 定期清理舊的建置記錄
- 使用平行階段加速建置

## 故障排除

### 查看建置日誌

建置失敗時，首先查看建置日誌：

1. 進入建置任務頁面
2. 點擊失敗的建置編號
3. 查看控制台輸出

### 常見問題處理

- 建置卡住：檢查是否有掛起的輸入等待，是否有資源死鎖
- 憑證失效：檢查憑證是否過期或被撤銷
- 外掛程式衝突：嘗試停用最近安裝的外掛程式
- 記憶體不足：調整 JVM 堆記憶體參數
