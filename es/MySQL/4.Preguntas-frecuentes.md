# Preguntas frecuentes sobre MySQL

## Problemas de instalación

### Q: ¿Qué debo hacer si el servicio MySQL no inicia?

A: Verifique lo siguiente:
1. Confirmar que MySQL esté correctamente instalado
2. Revisar registros de error: `mysqld --console`
3. Verificar que el puerto no esté en uso: `netstat -an | grep 3306`
4. Asegurar que los permisos del directorio de datos sean correctos
5. Intentar reinicializar el directorio de datos

### Q: ¿Cómo restauro la contraseña de root si la olvidé?

A: 
1. Detener el servicio MySQL
2. Iniciar con omisión de tablas de privilegios: `mysqld --skip-grant-tables`
3. Iniciar sesión en MySQL
4. Restablecer contraseña: `UPDATE mysql.user SET Password=PASSWORD('nueva_contraseña') WHERE User='root';`
5. Actualizar privilegios: `FLUSH PRIVILEGES;`
6. Reiniciar servicio MySQL

### Q: ¿Cómo cambio el número de puerto de MySQL?

A: Modificar el archivo de configuración my.cnf o my.ini:
```ini
[mysqld]
port=3307
```
Luego reinicie el servicio MySQL.

## Problemas de conexión

### Q: ¿Qué hacer si la conexión remota es rechazada?

A: 
1. Confirmar que MySQL permita conexiones remotas (verificar host del usuario en tabla de usuarios)
2. Verificar que el firewall permita el puerto
3. Asegurar que bind-address no sea 127.0.0.1 en configuración de MySQL
4. Verificar configuración de SELinux o AppArmor

### Q: ¿Cómo manejar tiempos de espera de conexión?

A: 
1. Verificar conexión de red
2. Aumentar configuración de wait_timeout
3. Usar conexiones persistentes o pool de conexiones
4. Verificar carga del servidor

## Problemas de operaciones de base de datos

### Q: ¿Cómo manejar caracteres codificados incorrectamente?

A: 
1. Especificar charset al crear base de datos: `CREATE DATABASE db CHARACTER SET utf8mb4;`
2. Especificar charset al conectar: `SET NAMES utf8mb4;`
3. Establecer charset predeterminado en configuración:
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: ¿Cómo manejar importación/exportación con caracteres incorrectos?

A: 
1. Especificar charset al exportar: `mysqldump --default-character-set=utf8mb4`
2. Especificar charset al importar: `mysql --default-character-set=utf8mb4`

### Q: ¿Es normal que las claves primarias autoincrementales no sean secuenciales?

A: Sí, esto es normal. Las razones incluyen:
- Las inserciones fallidas consumen IDs
- Los rollbacks de transacciones no recuperan IDs usados
- Las inserciones por lotes pueden causar saltos
- Eliminar registros no afecta el contador autoincremental

## Problemas de rendimiento

### Q: ¿Qué hacer si las consultas son lentas?

A: 
1. Usar EXPLAIN para analizar plan de ejecución de consulta
2. Crear índices para columnas usadas en WHERE, JOIN, ORDER BY
3. Evitar escaneos de tabla completos
4. Optimizar consultas, evitar SELECT *
5. Considerar uso de caché de consultas (eliminado en MySQL 8.0)
6. Dividir tablas o bases de datos para reducir presión

### Q: ¿Cómo ver consultas lentas?

A: 
1. Habilitar registro de consultas lentas:
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. Ver ubicación del registro de consultas lentas:
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: ¿Cómo optimizar alto uso de memoria?

A: 
1. Ajustar innodb_buffer_pool_size (recomendado 70% de memoria disponible)
2. Optimizar consultas para reducir escaneos innecesarios
3. Limitar número de conexiones
4. Limpiar cachés innecesarios

## Problemas de respaldo y recuperación

### Q: ¿Qué hacer si el archivo de respaldo es muy grande?

A: 
1. Usar respaldo comprimido: `mysqldump | gzip > backup.sql.gz`
2. Respaldar bases/tablas por separado
3. Usar xtrabackup para respaldos físicos
4. Limpiar datos históricos regularmente

### Q: ¿Cómo implementar respaldo incremental?

A: 
1. Habilitar registro binario: `log-bin=mysql-bin`
2. Respaldar registros binarios regularmente
3. Usar herramienta mysqlbinlog para restaurar datos incrementales

## Problemas de replicación

### Q: ¿Qué hacer si el retardo de replicación maestro-esclavo es muy alto?

A: 
1. Verificar latencia de red
2. Ajustar slave_parallel_workers para aumentar replicación paralela
3. Asegurar que hardware del esclavo sea suficiente
4. Optimizar velocidad de escritura del maestro
5. Verificar reglas de filtro de replicación

### Q: ¿Cómo manejar inconsistencia de datos maestro-esclavo?

A: 
1. Resincronizar esclavo
2. Usar pt-table-checksum para verificar inconsistencia
3. Usar pt-table-sync para reparar datos
4. Reconfigurar replicación maestro-esclavo

## Problemas de seguridad

### Q: ¿Cómo fortalecer la seguridad de MySQL?

A: 
1. Establecer políticas de contraseña seguras
2. Restringuir inicio de sesión remoto de usuario root
3. Eliminar cuentas anónimas
4. Eliminar base de datos de prueba
5. Actualizar versión de MySQL regularmente
6. Habilitar conexiones cifradas SSL/TLS
7. Configurar reglas de firewall
8. Seguir principio de mínimo privilegio

### Q: ¿Cómo prevenir inyección SQL?

A: 
1. Usar Declaraciones Preparadas (Prepared Statements)
2. Validar y filtrar entrada del usuario
3. Limitar privilegios de usuarios de base de datos
4. Evitar concatenación dinámica de SQL
5. Usar procedimientos almacenados
