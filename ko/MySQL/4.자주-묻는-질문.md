# MySQL 자주 묻는 질문

## 설치 관련 문제

### Q: MySQL 서비스가 시작되지 않으면 어떻게 해야 하나요?

A: 다음을 확인하세요:
1. MySQL이 올바르게 설치되어 있는지 확인
2. 오류 로그 확인: `mysqld --console`
3. 포트가 사용 중인지 확인: `netstat -an | grep 3306`
4. 데이터 디렉터리 권한이 올바른지 확인
5. 데이터 디렉터리 재초기화 시도

### Q: root 비밀번호를 잊어버렸다면 어떻게 재설정하나요?

A: 
1. MySQL 서비스 중지
2. 권한 테이블 건너뛰기 모드로 시작: `mysqld --skip-grant-tables`
3. MySQL에 로그인
4. 비밀번호 재설정: `UPDATE mysql.user SET Password=PASSWORD('새비밀번호') WHERE User='root';`
5. 권한 새로고침: `FLUSH PRIVILEGES;`
6. MySQL 서비스 재시작

### Q: MySQL 포트 번호를 변경하려면 어떻게 하나요?

A: my.cnf 또는 my.ini 구성 파일 수정:
```ini
[mysqld]
port=3307
```
그런 다음 MySQL 서비스를 다시 시작하세요.

## 연결 관련 문제

### Q: 원격 연결이 거부되면 어떻게 하나요?

A: 
1. MySQL이 원격 연결을 허용하는지 확인(사용자 테이블에서 사용자의 호스트 확인)
2. 방화벽이 포트를 허용하는지 확인
3. MySQL 구성에서 bind-address가 127.0.0.1이 아닌지 확인
4. SELinux 또는 AppArmor 설정 확인

### Q: 연결 시간 초과를 처리하려면 어떻게 하나요?

A: 
1. 네트워크 연결 확인
2. wait_timeout 구성 증가
3. 지속 연결 또는 연결 풀 사용
4. 서버 부하 확인

## 데이터베이스 작업 문제

### Q: 문자 집합 문제가 발생하면 어떻게 하나요?

A: 
1. 데이터베이스 생성 시 문자 집합 지정: `CREATE DATABASE db CHARACTER SET utf8mb4;`
2. 연결 시 문자 집합 지정: `SET NAMES utf8mb4;`
3. 구성 파일에서 기본 문자 집합 설정:
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: 가져오기/내보내기 시 문자 집합 문제가 발생하면 어떻게 하나요?

A: 
1. 내보내기 시 문자 집합 지정: `mysqldump --default-character-set=utf8mb4`
2. 가져오기 시 문자 집합 지정: `mysql --default-character-set=utf8mb4`

### Q: 자동 증가 기본 키가 연속적이지 않은 것이 정상인가요?

A: 정상입니다. 이유는 다음과 같습니다:
- 실패한 삽입도 ID를 소비함
- 트랜잭션 롤백이 사용된 ID를 회수하지 않음
- 일괄 삽입으로 인해 건너뛸 수 있음
- 레코드 삭제 시 자동 증가 카운터에 영향 없음

## 성능 문제

### Q: 조회가 느리면 어떻게 하나요?

A: 
1. EXPLAIN을 사용하여 조회 실행 계획 분석
2. WHERE, JOIN, ORDER BY에 사용된 열에 인덱스 생성
3. 전체 테이블 스캔 피하기
4. 조회 최적화, SELECT * 피하기
5. 조회 캐시 사용 고려(MySQL 8.0에서 제거됨)
6. 테이블 또는 데이터베이스 분할하여 부하 감소

### Q: 슬로우 쿼리를 보려면 어떻게 하나요?

A: 
1. 슬로우 쿼리 로깅 활성화:
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. 슬로우 쿼리 로그 위치 확인:
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: 메모리 사용량이 높으면 어떻게 최적화하나요?

A: 
1. innodb_buffer_pool_size 조정(사용 가능한 메모리의 70% 권장)
2. 조회 최적화로 불필요한 스캔 감소
3. 연결 수 제한
4. 불필요한 캐시 정리

## 백업 및 복원 문제

### Q: 백업 파일이 너무 크면 어떻게 하나요?

A: 
1. 압축 백업 사용: `mysqldump | gzip > backup.sql.gz`
2. 데이터베이스/테이블별로 백업
3. xtrabackup을 사용하여 물리적 백업 수행
4. 정기적으로 과거 데이터 정리

### Q:增量 백업을 구현하려면 어떻게 하나요?

A: 
1. 바이너리 로깅 활성화: `log-bin=mysql-bin`
2. 정기적으로 바이너리 로그 백업
3. mysqlbinlog 도구를 사용하여增量 데이터 복원

## 복제 문제

### Q: 마스터-슬레이브 복제 지연이 너무 높으면 어떻게 하나요?

A: 
1. 네트워크 지연 확인
2. slave_parallel_workers 조정하여 병렬 복제 증가
3. 슬레이브 하드웨어 성능이 충분한지 확인
4. 마스터 쓰기 속도 최적화
5. 복제 필터 규칙 확인

### Q: 마스터-슬레이브 데이터 불일치를 처리하려면 어떻게 하나요?

A: 
1. 슬레이브 재동기화
2. pt-table-checksum을 사용하여 불일치 확인
3. pt-table-sync를 사용하여 데이터修復
4. 마스터-슬레이브 복제 재구성

## 보안 문제

### Q: MySQL 보안을 강화하려면 어떻게 하나요?

A: 
1. 강력한 비밀번호 정책 설정
2. root 사용자 원격 로그인 제한
3. 익명 계정 삭제
4. 테스트 데이터베이스 삭제
5. MySQL 버전 정기적으로 업데이트
6. SSL/TLS 암호화된 연결 활성화
7. 방화벽 규칙 구성
8. 최소 권한 원칙 준수

### Q: SQL 주입을 방지하려면 어떻게 하나요?

A: 
1. 준비된 문(Prepared Statements) 사용
2. 사용자 입력 검증 및 필터링
3. 데이터베이스 사용자 권한 제한
4. 동적 SQL 연결 피하기
5. 저장 프로시저 사용
