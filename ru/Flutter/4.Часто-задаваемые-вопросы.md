# FAQ Flutter

## Проблемы установки

### Что делать, если Flutter doctor показывает ошибки?

Команда flutter doctor проверяет состояние среды разработки Flutter. Распространенные ошибки и решения:

- Android SDK не найден: Установите Android Studio или вручную настройте переменную среды ANDROID_SDK_ROOT
- Версия Xcode слишком старая: Обновите Xcode до последней версии
- Лицензии не приняты: Выполните flutter doctor --android-licenses для принятия лицензий

### Что делать, если загрузка Flutter не удается?

Если загрузка медленная или не удается, попробуйте использовать зеркало. Китайские пользователи могут настроить переменные среды PUB_HOSTED_URL и FLUTTER_STORAGE_BASE_URL для использования отечественных зеркал.

### Что делать, если клонирование Git не удается?

Убедитесь, что сеть может получить доступ к GitHub. Попробуйте использовать прокси или отечественное зеркало. Вы также можете загрузить предварительно скомпилированный архив с веб-сайта Flutter и установить его вручную.

## Проблемы разработки

### Горячая перезагрузка не работает?

Когда горячая перезагрузка не удается, попробуйте: Нажать Stop затем снова запустить; Выполнить flutter clean для очистки кэша сборки; Проверить наличие синтаксических ошибок в коде.

### Приложение тормозит?

Проверьте, выполняются ли трудоемкие операции в методе build. Сложные вычисления должны использовать Isolate или асинхронную обработку. Избегайте ненужных пересборок виджетов, используйте конструкторы const и подходящие ключи.

### Состояние теряется?

Горячая перезагрузка сохраняет состояние StatefulWidget, но состояние сбрасывается при пересборке приложения или выполнении полной горячей перезагрузки. Для сохранения состояния используйте SharedPreferences, SQLite или другие решения для хранения.

### Конфликты зависимостей пакетов?

Выполните flutter pub get для просмотра дерева зависимостей. Используйте flutter pub deps для проверки зависимостей пакетов. Попробуйте обновить версии пакетов или использовать dependency_overrides для решения конфликтов.

## Проблемы платформы

### Сборка Android не удается?

Распространенные причины: несовпадающая версия SDK, проблемы с настройкой Gradle, ошибки настройки подписи. Попробуйте выполнить flutter clean затем пересобрать. Проверьте настройки в android/build.gradle и android/app/build.gradle.

### Симулятор iOS не запускается?

Убедитесь, что Xcode установлен и правильно настроен. Выполните xcrun simctl list devices в терминале для просмотра доступных симуляторов. Попробуйте сбросить симулятор или создать новый.

### Ошибка сборки Web?

Проверьте правильность веб-зависимостей в pubspec.yaml. Используйте flutter clean для очистки затем пересборки. Убедитесь, что браузер Chrome установлен.

## Проблемы производительности

### Приложение слишком большое?

Используйте flutter build apk --split-per-abi для создания APK под разные архитектуры CPU. Проверьте и удалите неиспользуемые изображения и ресурсы. Используйте обфускацию и сжатие.

### Время запуска слишком долгое?

Сократите операции инициализации в функции main. Используйте отложенную загрузку для необязательных компонентов. Используйте Splash Screen для улучшения пользовательского опыта.

### Использование памяти слишком высокое?

Проверьте наличие утечек памяти. Используйте инструменты Profile для анализа использования памяти. Своевременно освобождайте неиспользуемые ресурсы и большие объекты.

## Проблемы обучения

### Как выбрать решение для управления состоянием?

Используйте setState для простых приложений. Provider рекомендуется для приложений средней сложности. Riverpod, GetX или BLoC можно использовать для сложных приложений.

### Миграция с других фреймворков?

На веб-сайте Flutter есть руководства по миграции для разработчиков под Android, iOS, React Native. Основные концепции схожи, сосредоточьтесь на изучении виджетов и системы макетов.

### Как повысить эффективность разработки?

Освойте плагины Flutter для VS Code или Android Studio. Изучите горячие клавиши и шаблоны кода. Используйте горячую перезагрузку и инструменты отладки. Обращайтесь к официальной документации и ресурсам сообщества.
