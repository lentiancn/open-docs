# JMeter 使用手冊

本手冊將詳細介紹如何使用 Apache JMeter 建立和執行負載測試。

## 測試計畫基本結構

JMeter 測試計畫由以下核心元件構成：

### 1. 測試計畫（Test Plan）

測試計畫是所有測試的根節點，包含整個測試的設定資訊。在測試計畫中可以設定：
- 使用者定義的變數
- 全域屬性設定
- 執行緒組設定

### 2. 執行緒組（Thread Group）

執行緒組是測試的執行單元，定義了：
- **執行緒數量**：模擬的使用者數
- **啟動時間**：所有執行緒啟動所需時間
- **迴圈次數**：每個執行緒執行測試的次數
- **排程器**：設定測試的開始和持續時間

### 3. 取樣器（Samplers）

取樣器是向伺服器發送請求的元件，常見的取樣器包括：

- **HTTP 請求**：發送 HTTP/HTTPS 請求
- **FTP 請求**：測試 FTP 伺服器
- **JDBC 請求**：測試資料庫連線
- **LDAP 請求**：測試 LDAP 目錄
- **JMS 訊息**：測試訊息佇列
- **TCP 取樣器**：測試 TCP 服務

### 4. 邏輯控制器（Logic Controllers）

邏輯控制器用於控制請求的執行順序：

- **簡單控制器**：對請求進行分組
- **迴圈控制器**：重複執行其下的請求
- **僅一次控制器**：在測試期間只執行一次
- **交替控制器**：輪流執行其下的請求
- **隨機控制器**：隨機選擇執行請求
- **吞吐量控制器**：控制請求的執行頻率

### 5. 監聽器（Listeners）

監聽器用於查看和分析測試結果：

- **查看結果樹**：查看每個請求的詳細回應
- **聚合報告**：查看彙總的效能統計
- **圖形結果**：以圖形方式展示測試資料
- **回應時間圖表**：展示回應時間變化趨勢

### 6. 設定元件（Config Elements）

設定元件用於設定預設值和共用資料：

- **HTTP 請求預設值**：設定 HTTP 請求的預設參數
- **使用者定義的變數**：定義可複用的變數
- **CSV 資料檔案設定**：從 CSV 檔案讀取測試資料
- **HTTP Cookie 管理器**：管理 Cookies
- **HTTP 標頭管理器**：設定請求標頭資訊

### 7. 計時器（Timers）

計時器用於設定請求之間的延遲：

- **固定計時器**：設定固定延遲
- **高斯隨機計時器**：設定隨機延遲
- **同步計時器**：模擬並發請求
- **吞吐量計時器**：控制請求的吞吐量

### 8. 前置處理器和後置處理器

- **前置處理器**：在請求發送前執行操作
- **後置處理器**：在請求回應後擷取資料

## 建立第一個測試計畫

### 步驟一：建立測試計畫

1. 啟動 JMeter
2. 選擇"檔案" -> "新建"建立新測試計畫
3. 為測試計畫命名

### 步驟二：新增執行緒組

1. 點擊右鍵選擇測試計畫
2. 選擇"新增" -> "執行緒（使用者）" -> "執行緒組"
3. 設定執行緒數（如 10）、啟動時間（如 5 秒）、迴圈次數（如 1）

### 步驟三：新增 HTTP 請求

1. 點擊右鍵選擇執行緒組
2. 選擇"新增" -> "取樣器" -> "HTTP 請求"
3. 設定伺服器名稱或 IP、連接埠號、請求路徑等

### 步驟四：新增監聽器

1. 點擊右鍵選擇執行緒組或 HTTP 請求
2. 選擇"新增" -> "監聽器" -> "查看結果樹"
3. 可以新增"聚合報告"查看效能統計

### 步驟五：執行測試

1. 點擊工具列上的"啟動"按鈕
2. 觀察監聽器中顯示的結果資料

## 參數化和資料驅動測試

### 使用 CSV 資料檔案

1. 新增"CSV 資料檔案設定"元件
2. 設定檔案名稱、變數名稱
3. 在請求中使用 ${變數名稱} 參照資料

### 使用使用者定義的變數

1. 新增"使用者定義的變數"元件
2. 定義變數名稱和值
3. 在請求中使用 ${變數名稱} 參照

## 關聯處理

### 使用正規表達式擷取器

當請求需要動態資料（如 Session ID）時：

1. 新增"正規表達式擷取器"
2. 設定正規表達式匹配模式
3. 在後續請求中使用擷取的變數

### 使用 JSON 擷取器

對於 JSON 回應：

1. 新增"JSON 擷取器"
2. 設定 JSON Path 表達式
3. 擷取需要的 JSON 欄位值

## 分散式測試

當單台機器無法產生足夠負載時，可以使用分散式測試：

### 主節點設定

1. 在主節點 bin 目錄下編輯 jmeter.properties
2. 設定 remote_hosts 指向從節點位址
3. 啟動 JMeter

### 從節點設定

1. 在每台從節點上執行 jmeter-server
2. 確保防火牆允許指定連接埠通訊

### 啟動分散式測試

1. 在主節點選擇"遠端啟動"
2. 選擇要啟動的從節點或全部啟動

## 效能優化建議

- 使用命令列模式執行測試，減少 GUI 資源消耗
- 合理設定執行緒數和啟動時間，避免瞬間高負載
- 關閉不必要的監聽器，僅在需要時查看結果
- 使用斷言驗證回應內容，確保測試準確性
- 及時清理不再使用的測試計畫
- 對於大規模測試，考慮使用分散式測試
