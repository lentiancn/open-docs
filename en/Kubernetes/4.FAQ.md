# Kubernetes FAQ

## General Questions

### What is Kubernetes?
Kubernetes (K8s) is an open-source container orchestration platform for automating deployment, scaling, and management of containerized applications.

### What is a Pod?
A Pod is the smallest deployable unit in Kubernetes. It can contain one or more containers that share storage and network.

### What is a Service?
A Service is an abstraction that defines a logical set of Pods and a policy to access them.

### What is a Deployment?
A Deployment provides declarative updates for Pods and ReplicaSets.

## Architecture Questions

### What are the main components of Kubernetes?
- **Master Node**: API Server, Scheduler, Controller Manager, etcd
- **Worker Nodes**: Kubelet, Kube Proxy, Container Runtime
- **etcd**: Distributed key-value store for cluster state

### What is kubectl?
kubectl is the command-line tool for interacting with Kubernetes clusters.

## Usage Questions

### How do I create a deployment?

```bash
kubectl create deployment my-app --image=nginx
```

### How do I scale a deployment?

```bash
kubectl scale deployment my-app --replicas=3
```

### How do I expose a service?

```bash
kubectl expose deployment my-app --port=80 --type=LoadBalancer
```

### How do I view pod logs?

```bash
kubectl logs <pod-name>
```

### How do I access a running container?

```bash
kubectl exec -it <pod-name> -- /bin/bash
```

## Installation Questions

### How do I install Kubernetes?
- **Minikube**: For local development
- **Kind**: Kubernetes in Docker
- **kubeadm**: For production clusters
- **Cloud Providers**: GKE, EKS, AKS

### What is Minikube?
Minikube is a tool that lets you run Kubernetes locally on your machine.

## Troubleshooting Questions

### Pod not starting?
- Check pod status: `kubectl get pods`
- View events: `kubectl describe pod <name>`
- Check logs: `kubectl logs <name>`

### Service not accessible?
- Verify service exists: `kubectl get svc`
- Check endpoints: `kubectl get endpoints`
- Verify pod labels match service selector
