# GitLab FAQ

> Häufig gestellte Fragen zu GitLab

---

## Allgemein

### Was ist GitLab?

GitLab ist eine vollständige DevOps-Plattform mit Git-Repository-Management, Issue-Tracking, CI/CD-Pipelines und mehr. Es kann als SaaS-Dienst (gitlab.com) oder selbstgehostet verwendet werden.

### GitLab vs GitHub?

| Funktion | GitLab | GitHub |
|----------|--------|--------|
| CI/CD | Integriert | GitHub Actions |
| Selbstgehostet | Ja | Nur Enterprise |
| Container-Registry | Integriert | GitHub Packages |
| Wiki | Integriert | Integriert |
| Issue-Tracking | Erweitert | Einfach |

## Installation

### Was sind die Mindestanforderungen?

| Ressource | Minimum |
|-----------|---------|
| CPU | 2 Kerne |
| RAM | 4 GB |
| Speicher | 10 GB |

### Installation schlägt mit Abhängigkeitsfehler fehl?

**Lösung**:
1. Systempakete aktualisieren
2. Fehlende Abhängigkeiten manuell installieren
3. Auf in Konflikt stehende Pakete prüfen

### Wie ändere ich die externe URL nach der Installation?

```bash
# Konfiguration bearbeiten
sudo vim /etc/gitlab/gitlab.rb
external_url 'https://neue-url.com'

# Neu konfigurieren
sudo gitlab-ctl reconfigure
```

## Verwendung

### Wie setze ich das Root-Passwort zurück?

```bash
# Rails-Konsole öffnen
sudo gitlab-rails console -e production

# Passwort finden und zurücksetzen
user = User.where(id: 1).first
user.password = 'neues_passwort'
user.password_confirmation = 'neues_passwort'
user.save!
```

### Wie sichere ich GitLab?

```bash
# Backup erstellen
sudo gitlab-backup create

# Backup-Speicherort: /var/opt/gitlab/backups
```

### Wie stelle ich aus einem Backup wieder her?

```bash
# GitLab stoppen
sudo gitlab-ctl stop

# Backup wiederherstellen
sudo gitlab-backup restore BACKUP=zeitstempel

# GitLab neu starten
sudo gitlab-ctl start
```

## CI/CD

### Pipeline wird nicht ausgelöst?

**Lösung**:
1. .gitlab-ci.yml-Syntax prüfen
2. GitLab-Runner-Registrierung verifizieren
3. Runner-Status in Projekteinstellungen prüfen

### Wie verwende ich Docker in CI?

```yaml
image: docker:latest

services:
  - docker:dind

build:
  script:
    - docker build -t mein-image .
```

### Wie kann ich Abhängigkeiten zwischenspeichern?

```yaml
cache:
  key: ${CI_COMMIT_REF_SLUG}
  pfade:
    - node_modules/
    - .npm/
```

## Sicherheit

### Wie aktiviere ich 2FA?

1. Gehen Sie zu Benutzereinstellungen > Konto
2. Klicken Sie auf "Zwei-Faktor-Authentifizierung aktivieren"
3. QR-Code mit Authenticator-App scannen
4. Backup-Codes speichern

### Wie scanne ich auf Schwachstellen?

Sicherheits-Dashboards in Projekteinstellungen aktivieren oder Vorlagen verwenden:
```yaml
include:
  - template: Security/SAST.gitlab-ci.yml
```

## Leistung

### GitLab ist langsam?

**Lösungen**:
1. Serverressourcen erhöhen
2. Prometheus-Überwachung aktivieren
3. PostgreSQL-Einstellungen optimieren
4. Separate Runner für CI verwenden

### Wie überprüfe ich die GitLab-Gesundheit?

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-ctl status
```

## Fehlerbehebung

### 502 Bad Gateway-Fehler?

**Lösung**:
1. Unicorn/Worker-Prozesse prüfen
2. Worker-Speicher erhöhen
3. Logs prüfen: `sudo gitlab-ctl tail`

### Git-Operationen schlagen fehl?

**Lösung**:
1. SSH-Schlüssel korrekt konfiguriert
2. HTTPS-Zertifikate verifizieren
3. Repository-Berechtigungen prüfen
