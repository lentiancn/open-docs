# OpenClaw 자주 묻는 질문

이 페이지는 OpenClaw 사용에 대한 가장 일반적인 질문과 답변을 모았습니다.

---

## 기본 질문

### OpenClaw란?

OpenClaw은 WhatsApp, Telegram, Discord, iMessage와 같은 여러 채팅 플랫폼을 AI 코딩 어시스턴트(예: Pi)에 연결하는 자체 호스팅 AI 어시스턴트 게이트웨이입니다. 자체 컴퓨터에서 실행하며 데이터에 대한 완전한 제어를 가집니다.

### OpenClaw는 무료인가요?

OpenClaw 자체는 오픈 소스이며 무료입니다(MIT 라이선스). 그러나 AI 모델(예: Anthropic Claude 또는 OpenAI GPT) 사용에는 비용이 발생할 수 있습니다.

### OpenClaw를 실행하려면 어떤 장치가 필요한가요?

최소 요구사항:
- Node.js 22+
- 2GB RAM
- 500MB 디스크 공간

권장:
- 4GB+ RAM
- 안정적인 네트워크 연결
- 항상 켜져 있는 컴퓨터 또는 서버

## 설치 관련 질문

### 설치 실패 — 어떻게 해야 하나요?

1. Node.js 버전 확인:
   ```bash
   node --version
   ```

2. 공식 설치 스크립트 사용:
   ```bash
   curl -fsSL https://openclaw.ai/install.sh | bash
   ```

3. 진단 실행:
   ```bash
   openclaw doctor
   ```

### Windows에서 문제가 있어요?

**WSL2를 강력 권장합니다.** OpenClaw는 Linux 환경에서 가장 잘 작동합니다.

### OpenClaw를 어떻게 업그레이드하나요?

```bash
npm update -g openclaw@latest
openclaw doctor
```

## 구성 관련 질문

### 구성 파일은 어디에 있나요?

기본 위치: `~/.openclaw/openclaw.json`

### 여러 채널을 어떻게 구성하나요?

```json5
{
  channels: {
    whatsapp: { enabled: true },
    telegram: { enabled: true, botToken: "xxx" },
    discord: { enabled: true, token: "xxx" },
  },
}
```

### 구성이 적용되지 않아요?

1. 파일 형식이 올바른지 확인 (JSON5)
2. Gateway 재시작:
   ```bash
   openclaw gateway restart
   ```

## 연결 관련 질문

### Gateway가 시작되지 않아요?

가능한 원인:
- 포트 18789가 이미 사용 중
- 구성 파일 형식 오류
- 권한 부족

해결책:
```bash
lsof -i :18789
openclaw doctor
```

### 채널 연결 실패?

1. 자격 증명이 유효한지 확인
2. 채널 상태 확인:
   ```bash
   openclaw channels status --probe
   ```
3. 로그 확인:
   ```bash
   openclaw logs --follow
   ```

## 사용 관련 질문

### AI가 메시지에 응답하지 않아요?

확인 사항:
1. 채널이 연결되어 있나요?
2. 화이트리스트에 있나요?
3. 그룹에서 @멘션이 필요한가요?
4. 로그의 오류 메시지 확인

### 세션을 어떻게 재설정하나요?

```bash
openclaw session reset
```

### AI 모델을 어떻게 변경하나요?

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",
      },
    },
  },
}
```

## 보안 관련 질문

### OpenClaw가 안전한가요?

OpenClaw는 보안을 우선시합니다:
- 모든 데이터는 로컬에 저장
- 자격 증명은 암호화되어 저장
- 화이트리스트 및 페어링 메커니즘 지원
- 샌드박스 격리 활성화 가능

권장 사항:
- 화이트리스트로 액세스 제어
- 정기적으로 버전 업데이트
- `openclaw doctor`를 실행하여 보안 확인

### 페어링(Pairing)이란?

페어링은 알려지지 않은 사용자가 첫 메시지를 보낼 때 차단되는 보안 메커니즘입니다. AI를 사용하려면 승인이 필요합니다.

사용자 승인:
```bash
openclaw pairing approve --channel whatsapp --sender +15555550123
```
