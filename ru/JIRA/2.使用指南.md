# Руководство пользователя JIRA

В этом руководстве описано, как использовать JIRA для управления проектами и отслеживания проблем.

## Быстрый старт

### Создание проекта

1. Нажмите «Создать проект»
2. Выберите шаблон проекта
3. Введите название и ключ проекта
4. Нажмите «Создать»

## Типы проектов

### Гибкие проекты

```markdown
- Scrum-проекты
- Kanban-проекты
- Масштабируемый Agile
```

### Классические проекты

```markdown
- Бизнес-проекты
- Управление проектами
- Отслеживание задач
```

## Типы проблем

### Создание проблемы

```markdown
1. Нажмите «Создать проблему»
2. Выберите проект
3. Выберите тип проблемы
4. Заполните заголовок и описание
5. Укажите приоритет и компоненты
6. Назначьте пользователю
```

### Типы проблем

- **Bug** — дефект программного обеспечения
- **Story** — пользовательская история
- **Task** — задача
- **Epic** — эпик
- **Sub-task** — подзадача

## Рабочие процессы

### Стандартный рабочий процесс

```
Создать → Назначить → Решить → Закрыть
```

### Гибкий рабочий процесс

```
Бэклог → В работе → Готово → Проверено
```

## Гибкие функции

### Kanban

1. Создайте Kanban
2. Добавьте колонки
3. Перетаскивайте карточки проблем
4. Установите ограничения WIP

### Scrum

```markdown
- Планирование спринта
- Ежедневный стендап
- Обзор спринта
- Ретроспектива спринта
```

## Поля

### Пользовательские поля

1. Перейдите в Администрирование → Поля
2. Добавьте пользовательское поле
3. Настройте параметры поля

### Настройка полей

```markdown
- Обязательные поля
- Значения по умолчанию
- Права на поля
```

## Права

### Права проекта

- Просмотр проекта
- Создание проблем
- Редактирование проблем
- Удаление проблем
- Назначение проблем

### Уровни безопасности проблем

- Публичный
- Участники проекта
- Определённые пользователи
- Определённые группы

## Автоматизация

### Создание правила

1. Перейдите в Настройки проекта → Автоматизация
2. Создайте правило
3. Настройте триггеры
4. Настройте действия

### Пример автоматизации

```
Триггер: Создание проблемы
Условие: Тип = Bug
Действие: Назначить = Ответственный
```

## Отчёты

### Гибкие отчёты

- Диаграмма сгорания
- График скорости
- Кумулятивная диаграмма потока
- Статистика Kanban

### Отчёты проекта

- Статистика проблем

### Интеграция CI/CD

## Рекомендации

### 1. Настройте рабочие процессы правильно

```
Создать → Проверить → Разработать → Тестировать → Завершить
```

### 2. Используйте версии

- Планирование версий
- Управление вехами
- Заметки о выпуске

### 3. Регулярно очищайте

- Архивируйте старые проблемы
- Очищайте недействительные проблемы
