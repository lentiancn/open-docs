# FantasyTalking ユーザーマニュアル

## 基本的な機能

### 音声合成

音声合成はFantasyTalkingの中核機能です:

```javascript
// シンプルテキストから音声へ
const audio = await client.speak('こんにちは、FantasyTalkingへようこそ');

// 声のスタイルを指定
const audio = await client.speak('今日はいい天気ですね', {
  voice: {
    style: 'happy',
    language: 'ja-JP'
  }
});

// 速度とピッチを指定
const audio = await client.speak('しばらくお待ちください', {
  voice: {
    speed: 1.2,
    pitch: 0.9
  }
});
```

### 音声イベント

再生中にさまざまなイベントをリsten:

```javascript
client.on('speech-start', () => console.log('再生開始'));
client.on('speech-end', () => console.log('再生終了'));
client.on('error', (e) => console.error('エラー:', e));
```

### バッチ合成

大量のテキストを合成する場合:

```javascript
const texts = ['最初の音声', '2番目の音声', '3番目の音声'];
const results = await Promise.all(texts.map(t => client.speak(t)));
```

## 対話機能

### 基本的な対話

```javascript
const session = client.createSession({
  systemPrompt: 'あなたはriendlyなアシスタントです',
  language: 'ja-JP'
});

const response = await session.chat('今日の天気は？');
console.log(response.text);
```

## 上級機能

### ストリーミング

リアルタイムフィードバックが必要な場合:

```javascript
const stream = await client.speakStream('長いテキストです');
stream.on('data', chunk => player.play(chunk));
```

## ベストプラクティス

1. **接続の再利用**: クライアントインスタンスを頻繁に作成しない
2. **バッチ処理**: 大量テキストにはバッチ合成を使用
3. **キャッシュ**: 重複合成を避ける
4. **ストリーミング**: 長いテキストにはストリーミング出力を使用

### エラー処理

```javascript
try {
  const audio = await client.speak('テストテキスト');
} catch (error) {
  if (error.code === 'RATE_LIMIT') {
    console.log('リクエストが多すぎます');
  }
}
```
