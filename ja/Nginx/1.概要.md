# Nginx はじめに

## Nginx とは

Nginx（エンジンエックスと読む）は、HTTP ウェブサーバー、リバースプロキシ、キャッシュ、ロードバランサー、TCP/UDP プロキシサーバー、メールプロキシサーバーです。

Nginx は Igor Sysoev によって最初に作成され、2 条項 BSD ライセンスに基づいて配布されました。

## Nginx の特徴

Nginx はその柔軟性と高いパフォーマンス、低リソース消費で知られています：

- **世界で最も人気のあるウェブサーバー**
- **最も人気のある Docker イメージの一つ**
- **Kubernetes の複数の Ingress Controllers をサポート**

### 基本 HTTP サーバー機能

- 静的ファイルとインデックスファイルの提供
- オートインデックス機能
- オープンファイルディスクリプタキャッシュ
- キャッシュ付きのリバースプロキシ加速
- ロードバランシングと障害耐性
- FastCGI、uwsgi、SCGI、memcached サーバーのキャッシュ加速サポート
- モジュール式アーキテクチャ
- フィルター：gzip 圧縮、バイト範囲、チャンク応答、XSLT、SSI、イメージ変換
- SSL と TLS SNI サポート
- HTTP/2 サポート（重み付けと依存ベースの優先順位付け）
- HTTP/3 サポート

### その他の HTTP サーバー機能

- 名前ベースと IP ベースのバーチャルサーバー
- Keep-alive とパイプライン接続サポート
- アクセスログフォーマット、バッファリングログ書き込み、高速ログローテーション、システムログ
- 3xx-5xx エラーコードのリダイレクト
- リライトモジュール：正規表現を使用した URI 変更
- クライアント IP アドレスに基づくアクセス制御
- クライアント IP アドレス、パスワード（HTTP 基本認証）、サブリクエスト結果に基づくアクセス制御
- HTTP referer 検証
- PUT、DELETE、MKCOL、COPY、MOVE メソッド
- FLV と MP4 ストリーミング
- 応答レート制限
- 1 つのアドレスからの同時接続数またはリクエスト数の制限
- IP ベースのジオロケーション
- A/B テスト
- リクエストミラーリング
- 埋め込み Perl
- njs スクリプト言語

### メールプロキシサーバー機能

- 外部 HTTP 認証サーバーを使用した IMAP または POP3 サーバーへのユーザーリダイレクト
- 外部 HTTP 認証サーバーを使用したユーザー認証と内部 SMTP サーバーへの接続リダイレクト
- 認証方法：POP3（USER/PASS、APOP、AUTH LOGIN/PLAIN/CRAM-MD5）、IMAP（LOGIN、AUTH LOGIN/PLAIN/CRAM-MD5）、SMTP（AUTH LOGIN/PLAIN/CRAM-MD5）
- SSL サポート
- STARTTLS と STLS サポート

### TCP/UDP プロキシサーバー機能

- TCP と UDP の汎用プロキシ
- TCP の SSL と TLS SNI サポート
- ロードバランシングと障害耐性
- クライアントアドレスに基づくアクセス制御
- 1 つのアドレスからの同時接続数の制限
- アクセスログフォーマット、バッファリングログ書き込み、高速ログローテーション、システムログ
- IP ベースのジオロケーション
- A/B テスト
- njs スクリプト言語

### アーキテクチャとスケーラビリティ

- 1 つのマスタープロセスと複数のワーカープロセス
- ワーカープロセスは非特権ユーザーとして実行
- 柔軟な設定
- クライアントサービスを中断せずに設定を再構成と実行ファイルのアップグレード
- kqueue（FreeBSD 4.1+）、epoll（Linux 2.6+）、/dev/poll（Solaris 7 11/99+）、event ports（Solaris 10）、select、poll のサポート
- 様々な kqueue 機能のサポート
- 様々な epoll 機能のサポート
- sendfile、sendfile64、sendfilev のサポート
- File AIO サポート
- DIRECTIO サポート
- Accept-filters と TCP_DEFER_ACCEPT サポート
- 10,000 個の非アクティブな HTTP keep-alive 接続は約 2.5M のメモリを消費
- データコピー操作を最小限に抑制

## Nginx の利点

### 高いパフォーマンス

Nginx はイベント駆動の非同期ノンブロッキングモデルを採用し、低メモリ消費を維持しながら高い同時接続を処理できます。

### 低リソース消費

他のウェブサーバーと比較して、Nginx は同じ同時接続レベルでより少ないメモリと CPU を消費します。

### 高い安定性

Nginx の安定性は世界中の何百万ものウェブサイトによって実証されており、極めて少ないクラッシュです。

### シンプルな設定

Nginx の設定ファイルは簡潔で理解しやすく、維持しやすいです。

### 強い拡張性

豊富なモジュールをサポートし、必要に応じて機能を有効または無効にできます。

### ホットデプロイ

サービスを中断せずに設定を再ロードし、バージョンをアップグレードできます。

## ユースケース

Nginx は以下のシナリオに広く使用されています：

- **ウェブサーバー**：静的ファイル用の HTTP サーバー
- **リバースプロキシ**：バックエンドサーバーのリバースプロキシ
- **ロードバランサー**：複数のバックエンドサーバーのロードバランサー
- **キャッシュサーバー**：CDN または内部キャッシュサーバー
- **API ゲートウェイ**：マイクロサービスのエントリゲートウェイ
- **メールプロキシ**：POP3、IMAP、SMTP プロキシ
- **TCP/UDP プロキシ**：TCP と UDP プロトコルのプロキシ
