# Guide d'installation de Java

## Obtenir le Kit de Développement Java

Oracle propose des téléchargements gratuits du Java Development Kit (JDK). Les principales versions LTS actuelles incluent JDK 25, JDK 21, JDK 17, JDK 11 et JDK 8.

Page de téléchargement officielle : https://www.oracle.com/java/technologies/downloads/

## Installation sur Windows

### Utilisation de l'installateur

1. Visitez la page de téléchargement officielle d'Oracle et sélectionnez la version Windows du JDK
2. Téléchargez l'installateur x64 (fichier exe) ou l'installateur MSI
3. Exécutez l'installateur téléchargé
4. Suivez les instructions de l'assistant d'installation pour terminer l'installation. Le chemin par défaut est C:\Program Files\Java\jdk-version
5. Après l'installation, vérifiez qu'elle a réussi

### Utilisation de l'archive compressée

1. Téléchargez l'archive compressée x64 (fichier zip)
2. Extrayez dans un répertoire désigné, comme C:\Java\jdk-25
3. Configurez les variables d'environnement

### Configuration des variables d'environnement

1. Faites un clic droit sur "Ce PC" et sélectionnez "Propriétés"
2. Cliquez sur "Paramètres système avancés"
3. Cliquez sur "Variables d'environnement"
4. Sous Variables système, cliquez sur "Nouveau"
   - Nom de la variable : JAVA_HOME
   - Valeur de la variable : Chemin d'installation du JDK (ex. C:\Program Files\Java\jdk-25)
5. Trouvez la variable Path et cliquez sur "Modifier"
6. Ajoutez une nouvelle variable : %JAVA_HOME%\bin
7. Cliquez sur OK pour enregistrer

### Vérifier l'installation

Ouvrez l'invite de commande et entrez :

```cmd
java -version
javac -version
```

Si les informations de version s'affichent, l'installation a réussi.

## Installation sur Linux

### Installation à partir de l'archive compressée

1. Téléchargez la version Linux du JDK (format tar.gz)
2. Ouvrez le terminal et extrayez dans le répertoire désigné :

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. Configurez les variables d'environnement en modifiant ~/.bashrc ou /etc/profile :

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. Appliquez la configuration :

```bash
source ~/.bashrc
```

5. Vérifiez l'installation :

```bash
java -version
```

### Installation avec le gestionnaire de packages

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## Installation sur macOS

### Utilisation de l'installateur DMG

1. Téléchargez la version macOS du JDK (fichier dmg)
2. Ouvrez le fichier dmg téléchargé
3. Faites glisser JDK vers le dossier Applications
4. Ouvrez le terminal et configurez les variables d'environnement :

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. Ajoutez ce qui précède à ~/.zshrc ou ~/.bash_profile
6. Vérifiez l'installation :

```bash
java -version
```

### Utilisation de Homebrew

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## Gestion de plusieurs versions de JDK

### Utilisation de update-alternatives sur Linux

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Changer de version sur Windows

Modifiez la variable d'environnement JAVA_HOME pour pointer vers différents répertoires JDK.

### Utilisation de SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Vérifier l'environnement Java

### Vérifier la version Java

```bash
java -version
```

Exemple de sortie :

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### Vérifier la version du compilateur

```bash
javac -version
```

Exemple de sortie :

```
javac 25.0.1
```

### Vérifier JAVA_HOME

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## Problèmes courants d'installation

### Problème 1 : Commande java non trouvée

Solution : Vérifiez si la variable d'environnement Path contient le répertoire bin du JDK.

### Problème 2 : Permissions insuffisantes

Solution : Utilisez sudo pour l'élévation sur Linux/macOS, ou vérifiez les permissions des fichiers.

### Problème 3 : Port déjà utilisé

Solution : Les applications Java utilisent le port 8080 par défaut. Assurez-vous que le port n'est pas utilisé par d'autres programmes.

### Problème 4 : Différence entre JDK et JRE

- JDK (Java Development Kit) : Kit de développement, inclut JRE et outils de développement
- JRE (Java Runtime Environment) : Environnement d'exécution, ne contient que les composants nécessaires pour exécuter des programmes Java

Le JDK est requis pour développer des programmes Java. Le JRE suffit pour exécuter des programmes Java compilés.

## Recommandations après l'installation

1. Configurez MAVEN_HOME ou GRADLE_HOME si vous utilisez Maven ou Gradle
2. Configurez M2_HOME pour le chemin du dépôt local Maven
3. Configurez le proxy HTTP si nécessaire dans les environnements d'entreprise
4. Définissez le fuseau horaire pour assurer la bonne heure pour les applications Java

## Désinstaller Java

### Panneau de configuration Windows

1. Ouvrez le Panneau de configuration
2. Sélectionnez "Programmes et fonctionnalités"
3. Trouvez les programmes liés à Java
4. Faites un clic droit et sélectionnez "Désinstaller"

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
