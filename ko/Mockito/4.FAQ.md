# Mockito FAQ

Mockito에 대한 자주 묻는 질문.

## 일반적인 질문

### Mockito란?

Mockito는 Java용 오픈 소스 모킹 프레임워크입니다. 단위 테스트를 위한 모크 객체를 생성하고 구성할 수 있습니다.

### Mockito를 사용하는 이유?

- 외부 종속성 없이 격리된 테스트
- 더 빠른 테스트 실행
- 간단하고 직관적인 API
- JUnit과의 우수한 통합

### Mock과 Spy의 차이점은?

- **Mock** - 완전히 제어할 수 있는 인공적인 객체
- **Spy** - 특정 메서드만 재정의하는 부분적으로 실제 객체

## 기술적인 질문

### final 클래스를 모크하는 방법은?

`mockito-inline` 사용:

```groovy
testImplementation 'org.mockito:mockito-inline:5.8.0'
```

### 정적 메서드를 모크하는 방법은?

Mockito 3.7+에서 가능:

```java
try (MockedStatic<Utilities> mocked = Mockito.mockStatic(Utilities.class)) {
    mocked.when(Utilities::getName).thenReturn("mocked");
    // 테스트 코드
}
```

## 문제 해결

### 오류: "Cannot mock/spy class"

**원인:** 클래스가 final이거나 메서드가 final입니다.

**해결:**
- `mockito-inline` 사용
- 클래스 대신 인터페이스를 모크

### 경고: "Unnecessary stubbing"

**원인:** 스텁이 사용되지 않았습니다.

**해결:**
- 사용하지 않는 스텁 제거
- 또는 `lenient()` 사용:

```java
lenient().when(mock.method()).thenReturn("value");
```

## 통합

### JUnit 4의 경우

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private List mockList;
}
```

### JUnit 5의 경우

```java
@ExtendWith(MockitoExtension.class)
public class MyTest {
    @Mock
    private List mockList;
}
```
