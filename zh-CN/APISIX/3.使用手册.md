# APISIX 使用手册

本文档介绍 APISIX 的核心功能和使用方法。

## 基本概念

### Route（路由）

路由是 APISIX 最核心的概念，用于匹配客户端请求并将请求转发到上游服务。

```bash
# 创建路由
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "my-route",
  "uri": "/api/v1/*",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend1.example.com:8080": 1,
      "backend2.example.com:8080": 1
    }
  }
}'
```

### Upstream（上游）

上游是一组提供相同服务的节点，APISIX 支持多种负载均衡策略：

```bash
# 创建上游
curl -i "http://127.0.0.1:9180/apisix/admin/upstreams" -X PUT -d '
{
  "id": "my-upstream",
  "type": "roundrobin",
  "nodes": {
    "host1:8080": 1,
    "host2:8080": 1
  }
}'
```

### Plugin（插件）

APISIX 提供了丰富的插件系统，支持在请求处理过程中执行各种功能。

## 路由匹配规则

### 1. 精确匹配

```json
{
  "uri": "/api/users"
}
```

### 2. 参数匹配

```json
{
  "uri": "/api/users/:id"
}
```

### 3. 正则匹配

```json
{
  "uri": "/api/*",
  "vars": [["uri", "==", "/api/users"]]
}
```

### 4. 条件匹配

```json
{
  "uri": "/api/*",
  "vars": [
    ["http_x_api_version", "==", "v2"],
    ["arg_page", ">", 1]
  ]
}
```

## 负载均衡策略

### 1. 轮询（Round Robin）

```json
{
  "type": "roundrobin"
}
```

### 2. 哈希（Hash）

```json
{
  "type": "hash",
  "key": "remote_addr"
}
```

### 3. 一致性哈希

```json
{
  "type": "chash",
  "key": "remote_addr"
}
```

## 常用插件

### 1. 限流（limit-req）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "rate-limit-route",
  "uri": "/api/*",
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 2. 密钥认证（key-auth）

### 7. 请求重定向（redirect）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "redirect-route",
```

### 8. 流量分割（traffic-split）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "canary-route",

    "nodes": {
      "new-backend:8080": 1
    }
  }
}'
```

## 服务发现

APISIX 支持多种服务发现机制：

### 1. Consul

```yaml
discovery:
  consul:
    servers:
      - "http://127.0.0.1:8500"
```

### 2. Nacos

```yaml
discovery:
  nacos:
    host:
      - "http://127.0.0.1:8848"
    prefix: /nacos/v1
```

### 3. Eureka

```yaml
discovery:
  eureka:
    host:
      - "http://127.0.0.1:8761"
    port: 8761
```

## 健康检查

APISIX 支持主动和被动健康检查：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/upstreams" -X PUT -d '
{
  "id": "health-check-upstream",
  "type": "roundrobin",
  "nodes": {
    "backend1:8080": 1,
    "backend2:8080": 1
  },
  "checks": {
    "active": {
      "type": "http",
      "http_path": "/health",
      "interval": 5,
      "timeout": 2
    },
    "passive": {
      "type": "http",
      "healthy": {
        "http_statuses": [200, 201, 202, 203, 204],
        "successes": 3
      },
      "unhealthy": {
        "http_statuses": [500, 502, 503, 504],
        "http_failures": 3,
        "timeouts": 3
      }
    }
  }
}'
```

## 自定义插件开发

APISIX 支持使用 Lua、Java、Go、Python 等语言编写自定义插件：

```lua
-- 示例：自定义插件
local plugin_name = "my-custom-plugin"

local schema = {
    type = "object",
    properties = {
        value = {type = "string"}
    },
    required = {"value"},
}

local _M = {
    version = 0.1,
    priority = 1000,
    name = plugin_name,
    schema = schema,
}

function _M.access(conf, ctx)
    -- 在请求处理阶段执行
    ngx.log(ngx.INFO, "Custom plugin executed with value: ", conf.value)
end

return _M
```

## 最佳实践

1. **使用上游**：将上游信息独立配置，便于管理和复用
2. **合理使用插件**：按需启用插件，避免不必要的性能开销
3. **配置健康检查**：保证服务的高可用性
4. **使用服务发现**：在动态环境中自动感知服务变化
5. **监控和日志**：集成 Prometheus 和日志系统，便于问题排查
6. **安全配置**：生产环境务必启用 Admin API 密钥认证
7. **热加载**：充分利用插件热加载功能，零停机更新配置
