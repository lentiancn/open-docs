# Mockito ユーザーマニュアル

## モックオブジェクトの作成

モックオブジェクトの作成は、Mockitoを使用してテストを行う第一步です。モックオブジェクトを作成するには複数の方法があり、開発者はニーズに応じて適切な方法を選択できます。

### mock() メソッドの使用

最も基本的な方法は、Mockitoクラスの静的メソッドmock()を使用してモックオブジェクトを作成することです。このメソッドはClassパラメータを受け取り、そのクラスのモックインスタンスを返します。

```java
import static org.mockito.Mockito.*;

List<String> mockedList = mock(List.class);
Map<String, Integer> mockedMap = mock(Map.class);
```

Mockito 4.10.0以降では、Classパラメータを省略でき、コンパイラが自動的に型を推断します。

```java
List<String> mockedList = mock();
```

### @Mock アノテーションの使用

@Mockアノテーションは、よりクリーンな宣言方式を提供します。特にテストクラスで複数のモックオブジェクトを作成する必要がある場合に便利です。アノテーションを使用するには、まずMockitoフレームワークを初期化する必要があります。

```java
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
    
    @Mock
    private EmailService emailService;
    
    @Before
    public void setup() {
        MockitoAnnotations.initMocks(this);
    }
}
```

MockitoJUnitRunnerを使用すると自動的に初期化が 行われます。

```java
@RunWith(MockitoJUnitRunner.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
}
```

JUnit 5の@ExtendWithアノテーションを使用しても自動初期化を達成できます。

```java
@ExtendWith(MockitoExtension.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
}
```

## メソッド動作の定義

メソッド動作の定義はスタブbingとも呼ばれ、モックオブジェクトが特定のメソッド呼び出し時にどのような値を返すか、またはどの例外を抛出するかを定義するために使用されます。

### 基本的な戻り値の設定

when().thenReturn()チェーン構文を使用して戻り値を定義します。

```java
when(mockedList.size()).thenReturn(100);
when(mockedList.get(0)).thenReturn("first");
when(mockedList.get(1)).thenReturn("second");
```

複数の連続した戻り値を定義でき、後続の呼び出しでは異なる値が返されます。

```java
when(mockedList.get(anyInt()))
    .thenReturn("first")
    .thenReturn("second")
    .thenThrow(new RuntimeException("Error"));

System.out.println(mockedList.get(0)); // 出力: first
System.out.println(mockedList.get(0)); // 出力: second
System.out.println(mockedList.get(0)); // 例外を抛出
```

### 例外の抛出の設定

thenThrow()を使用して、メソッド呼び出し時に抛出される例外を定義できます。

```java
when(mockedList.get(0)).thenThrow(new IndexOutOfBoundsException("Index 0"));
```

void戻り型のメソッドの場合は、doThrow()を使用します。

```java
doThrow(new RuntimeException("Error")).when(mockedList).clear();
```

### Answer を使用したカスタムレスポンス

定義済みの戻り値でが不十分な場合は、Answerインターフェースを実装して動的に戻り値を計算できます。

```java
when(mockedList.get(anyInt())).thenAnswer(invocation -> {
    int index = invocation.getArgument(0);
    return "Value at index " + index;
});
```

## インタラクションの検証

検証は、テストコードがモックオブジェクトと期待通りに正しくインタラクションしたことを確認するために使用されます。

### 基本的な検証

verify()を使用してメソッドが呼び出されたか確認します。

```java
mockedList.add("one");
verify(mockedList).add("one");
```

### 呼び出し回数の検証

メソッドが呼び出された回数を検証できます。

```java
mockedList.add("one");
mockedList.add("one");
mockedList.add("one");

verify(mockedList, times(3)).add("one");       // ちょうど3回
verify(mockedList, atLeast(2)).add("one");     // 少なくとも2回
verify(mockedList, atMost(3)).add("one");      // 最大3回
verify(mockedList, never()).add("two");        // 決して呼び出されない
verify(mockedList, atLeastOnce()).add("one"); // 少なくとも1回
```

### 呼び出し順序の検証

InOrderベリファイアは、メソッド呼び出しが特定の順序で実行されたかを確認できます。

```java
InOrder inOrder = inOrder(mockedList);
inOrder.verify(mockedList).add("first");
inOrder.verify(mockedList).add("second");
inOrder.verify(mockedList).add("third");
```

### 呼び出し引数の検証

引数マッチャーを使用して呼び出し引数を柔軟に検証できます。

```java
verify(mockedList).add(anyString());
verify(mockedList).get(anyInt());
verify(mockedList).contains(argThat(s -> s.length() > 5));
```

## 引数マッチャー

Mockitoは、メソッドパラメータを柔軟にマッチングするための豊富な引数マッチャーを提供します。

### 組み込みマッチャー

```java
anyInt()          // 任意の整数
anyString()       // 任意の文字列
any()             // 任意のオブジェクト
anyList()         // 任意のリスト
anyMap()          // 任意のマップ
isNull()          // null値
isNotNull()       // 非null値
eq(value)         // 完全一致
startsWith()      // 指定文字列で始まる
endsWith()        // 指定文字列で終わる
contains()        // 指定文字列を含む
```

### カスタムマッチャー

argThat()メソッドを使用してカスタム引数マッチャーを作成できます。

```java
when(mockedList.add(argThat(s -> s != null && s.length() > 3)))
    .thenReturn(true);
```

## Spy の使用

SpyはMockとは異なり、実オブジェクトの動作の大部分を保持しながら特定のメソッドのみをモックできます。

### Spy の作成

```java
List<String> realList = new ArrayList<>();
List<String> spyList = spy(realList);
```

または@Spyアノテーションを使用します。

```java
@Spy
private List<String> list = new ArrayList<>();
```

### Spy の動作の定義

Mockとは異なり、Spyでwhen()を使用する際には注意が必要です реальメソッド呼び出しに影響を与える可能性があります。doReturn().when()構文を使用することをお勧めします。

```java
doReturn(100).when(spyList).size();
```

Spyは明示的にスタブされていない限り реальメソッドを呼び出します。

```java
spyList.add("element");  //  реальメソッドが呼び出される
verify(spyList).add("element");  // 検証成功
```

## @InjectMocks の使用

@InjectMocksアノテーションは、テスト対象のオブジェクトを自動的に作成し、@Mockまたは@Spyアノテーションが付けられたフィールドをそこに注入するために使用されます。

```java
@RunWith(MockitoJUnitRunner.class)
public class UserServiceTest {
    @Mock
    private UserRepository userRepository;
    
    @Mock
    private EmailService emailService;
    
    @InjectMocks
    private UserService userService;
    
    @Test
    public void testCreateUser() {
        User user = new User("testUser");
        when(userRepository.save(any(User.class))).thenReturn(user);
        
        User result = userService.createUser("testUser");
        
        assertEquals("testUser", result.getUsername());
        verify(userRepository).save(any(User.class));
    }
}
```

## ArgumentCaptor の使用

ArgumentCaptorは、メソッド呼び出し時に渡された引数をキャプチャして、より詳細なアサーションを行うために使用されます。

```java
@Mock
private List<String> mockedList;

@Test
public void testCaptureArgument() {
    mockedList.add("test");
    
    ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
    verify(mockedList).add(captor.capture());
    
    assertEquals("test", captor.getValue());
}
```

メソッドが複数回呼び出された場合、すべての呼び出しの引数をキャプチャできます。

```java
mockedList.add("first");
mockedList.add("second");
mockedList.add("third");

ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
verify(mockedList, times(3)).add(captor.capture());

assertEquals(3, captor.getAllValues().size());
assertEquals("second", captor.getAllValues().get(1));
```

## BDD スタイルテスト

Mockitoはgiven().willReturn()構文を使用してビヘイビア駆動開発（BDD）スタイルのテストをサポートします。

```java
import static org.mockito.BDDMockito.*;

@Test
public void testBddStyle() {
    // Given
    given(mockedList.get(0)).willReturn("first");
    
    // When
    String result = mockedList.get(0);
    
    // Then
    assertEquals("first", result);
}
```

## 一般的な使用パターン

### void メソッドのテスト

```java
@Test
public void testVoidMethod() {
    doNothing().when(mockedList).clear();
    mockedList.clear();
    verify(mockedList).clear();
}
```

### メソッドチェーンのモック

```java
@Test
public void testMethodChaining() {
    when(mockedList.iterator()).thenReturn(Collections.emptyIterator());
    
    Iterator<String> iterator = mockedList.iterator();
    assertFalse(iterator.hasNext());
}
```

### 例外処理のモック

```java
@Test(expected = RuntimeException.class)
public void testException() {
    when(mockedList.get(0)).thenThrow(new RuntimeException("Error"));
    mockedList.get(0);
}
```

## ベストプラクティス

Mockitoを使用する際は、テストコードの品質と保守性を確保するためにいくつかのベストプラクティスに従ってください。所有するクラスや値オブジェクトをモックしないでください。これによりテストのが真実性が壊れます。必要な依存関係のみをモックし、テストのが真実性を保ってください。各テストメソッドは1つの機能のみをテストする必要があります。@Beforeを使用して共通セットアップを行い、テストコードを適切に整理してください。検証は適度に行い、すべてのメソッド呼び出しを検証するのではなく重要なインタラクションに焦点を当ててください。
