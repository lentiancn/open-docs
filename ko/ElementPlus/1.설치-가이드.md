# Element Plus 설치 가이드

> Element Plus는 Element 팀이 개발한 Vue 3 데스크톱 UI 컴포넌트 라이브러리입니다
> 
> 업데이트 날짜: 2026년 2월
> 
> 현재 버전: 2.x (안정판)

---

## 1. 프로젝트 개요

Element Plus는 Vue 3 기반 데스크톱 UI 컴포넌트 라이브러리로, Element 팀에서 개발하고 유지관리하고 있습니다. Element UI의 Vue 3 업그레이드 버전으로서, Element Plus는 Vue 3의 Composition API 스타일로 완전히 작성되었으며 TypeScript로 완전히 재작성되어 타입 안전성을 제공합니다.

Element Plus의 첫 번째 안정판은 2022년 2월 7일에 출시되었습니다. 수년의 반복을 거쳐 API가 매우 안정적이 되었습니다. 현재 간체 중국어, 영어 및 기타 언어를 지원하며 활성 커뮤니티 생태계를 가지고 있습니다.

## 3. 설치 단계

### 3.1 npm 사용 (권장)

npm은 Node.js 패키지 관리자이며 Element Plus를 설치하는 공식 권장 방법입니다.

```bash
# Vue 3 프로젝트 디렉토리로 이동
cd your-vue-project

# Element Plus 설치
npm install element-plus
```

### 3.2 yarn 사용

```bash
yarn add element-plus
```

### 3.3 pnpm 사용

```bash
pnpm add element-plus
```

## 4. Vue 3 프로젝트에서 가져오기

### 4.1 전체 가져오기 (소규모 프로젝트에 적합)

전체 가져오기는 모든 컴포넌트를 번들링하며, 설정은 간단하지만 번들 크기가 더 큽니다.

```typescript
// main.ts 또는 main.js
import { createApp } from 'vue'
import App from './App.vue'

// Element Plus와 스타일 가져오기
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// Element Plus 사용
app.use(ElementPlus)

app.mount('#app')
```

### 4.2 필요시 가져오기 (프로덕션에 권장)

필요시 가져오기는 실제로 사용되는 컴포넌트만 가져와서 번들 크기를 크게 줄일 수 있습니다.

#### 4.2.1 자동 가져오기 플러그인 설치

먼저 unplugin-vue-components와 unplugin-auto-import를 설치합니다:

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

#### 4.2.2 Vite 구성

vite.config.ts에 구성을 추가합니다:

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

이렇게 구성하면 컴포넌트를 수동으로 가져오지 않고도 자동으로 가져올 수 있습니다. 예:

```vue
<template>
  <el-button type="primary">안녕하세요</el-button>
</template>
```

## 5. CDN으로 가져오기 (빌드 도구 필요 없음)

npm이나 빌드 도구를 사용하지 않으려면 CDN을 통해 Element Plus를 가져올 수 있습니다.

### 5.1 unpkg CDN으로 가져오기

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Element Plus CDN 예제</title>
  <!-- Vue 3 가져오기 -->
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <!-- Element Plus 가져오기 -->
  <link rel="stylesheet" href="https://unpkg.com/element-plus/dist/index.css">
  <script src="https://unpkg.com/element-plus"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary">안녕하세요 Element Plus</el-button>
  </div>

  <script>
    const { createApp, ref } = Vue
    
    const App = {
      setup() {
        const visible = ref(false)
        return { visible }
      },
      template: `
        <el-button @click="visible = true">다이얼로그 열기</el-button>
        <el-dialog v-model="visible" title="안녕하세요">Element Plus입니다</el-dialog>
      `
    }
    
    const app = createApp(App)
    app.use(ElementPlus)
    app.mount('#app')
  </script>
</body>
</html>
```

## 6. 국제화 구성

Element Plus는 기본적으로 영어를 사용합니다. 한국어를 사용하려면 국제화를 구성해야 합니다.

### 6.1 전역 한국어 구성

```typescript
// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import ko from 'element-plus/es/locale/lang/ko'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus, {
  locale: ko,
})

app.mount('#app')
```

## 7. 테마 사용자 지정

### 7.1 SCSS 변수 재정의

Element Plus는 SCSS로 작성되어 있으며, 변수를 재정의하여 테마를 사용자 지정할 수 있습니다.

```scss
// styles/element-variables.scss
// 주요 색상
$--color-primary: #409EFF;

// 컴포넌트 변수 재정의
@use "element-plus/theme-chalk/src/common/var.scss" as *;
```

### 7.2 다크 모드

Element Plus는 다크 모드를 기본 지원합니다:

```typescript
// 다크 모드 활성화
import 'element-plus/theme-chalk/dark/css-vars.css'
```

HTML 태그에 클래스 추가:

```html
<html class="dark">
  <!-- 앱 -->
</html>
```

## 8. 일반적인 문제

### 8.1 설치 실패

**문제**: npm install이 실패합니다

**해결책**:

1. npm 캐시 지우기: `npm cache clean --force`
2. 네트워크 확인, 국내 미러 사용 가능:

```bash
# npm 미러 설정
npm config set registry https://registry.npmmirror.com
```

### 8.2 스타일이 적용되지 않음

**문제**: 컴포넌트 스타일이 비정상적으로 표시됩니다

**해결책**:

1. 스타일 파일을 가져왔는지 확인: `import 'element-plus/dist/index.css'`
2. 빌드 구성이 CSS를 올바르게 처리하는지 확인
3. 브라우저 캐시를 지우고 다시 빌드
