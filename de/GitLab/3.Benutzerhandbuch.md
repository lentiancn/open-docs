# GitLab Benutzerhandbuch

> Umfassende Anleitung zur Verwendung von GitLab für Entwicklung und Betrieb

---

## Erste Schritte

### Konto erstellen

1. Navigieren Sie zu Ihrer GitLab-Instanz
2. Klicken Sie auf "Registrieren", um ein Konto zu erstellen
3. Bestätigen Sie Ihre E-Mail-Adresse
4. Vervollständigen Sie Ihr Benutzerprojekt

### Projekt erstellen

1. Klicken Sie auf "Neues Projekt"
2. Wählen Sie Projekvorlage oder leeres Projekt
3. Projektname und Beschreibung festlegen
4. Sichtbarkeitsstufe konfigurieren (privat, intern, öffentlich)
5. Klicken Sie auf "Projekt erstellen"

---

## Git-Operationen

### Repository klonen

```bash
# Mit HTTPS klonen
git clone https://gitlab.example.com/benutzername/projekt.git

# Mit SSH klonen
git clone git@gitlab.example.com:benutzername/projekt.git
```

### Grundlegender Git-Workflow

```bash
# Neuen Branch erstellen
git checkout -b feature/neue-funktion

# Änderungen vornehmen und bereitstellen
git add .
git commit -m "Neue Funktion hinzufügen"

# Zum Remote pushen
git push origin feature/neue-funktion

# Merge-Request aus Branch erstellen
```

### Branches verwalten

```bash
# Branches auflisten
git branch -a

# Lokalen Branch löschen
git branch -d feature/alte-funktion

# Remote-Branch löschen
git push origin --delete feature/alte-funktion
```

## Issue-Tracking

### Issues erstellen

1. Navigieren Sie zu Ihrem Projekt
2. Gehen Sie zu Issues > Neues Issue
3. Titel und Beschreibung hinzufügen
4. Labels, Meilensteine und Bearbeiter zuweisen
5. Gewichtung und Fälligkeitsdatum festlegen
6. Issue einreichen

### Issue-Boards verwenden

1. Gehen Sie zu Issues > Boards
2. Neues Board erstellen oder Standard verwenden
3. Listen für jede Phase hinzufügen (z.B. Zu erledigen, In Bearbeitung, Erledigt)
4. Issues zwischen Listen ziehen

### Labels und Meilensteine

- Beschreibende Labels für Kategorisierung erstellen
- Meilensteine verwenden, um Fortschritt zu verfolgen
- Issues mit Commit-Nachrichten schließen

---

## Merge-Requests

### Merge-Request erstellen

1. Pushen Sie Ihren Feature-Branch zu GitLab
2. Klicken Sie auf "Merge-Request erstellen"
3. Titel und Beschreibung ausfüllen
4. Quell- und Zielbranch auswählen
5. Labels und Reviewer hinzufügen
6. Merge-Request einreichen

### Code-Review-Prozess

- Reviewer können Kommentare zu bestimmten Zeilen hinzufügen
- Thread-Diskussionen für Gespräche nutzen
- Änderungen genehmigen oder anfordern
- Alle Threads vor dem Merge auflösen

### Squash und Merge

- Commits squaschen, um History sauber zu halten
- Quellbranch nach Merge löschen
- Commit-Nachricht vor dem Merge bearbeiten

## CI/CD-Pipelines

### Grundlegende .gitlab-ci.yml

```yaml
stufen:
  - build
  - test
  - deploy

build:
  stufe: build
  script:
    - npm install
    - npm run build
  artefakte:
    pfade:
      - dist/

test:
  stufe: test
  script:
    - npm test
  coverage: '/Coverage: \d+\.\d+%/'

deploy:
  stufe: deploy
  script:
    - npm run deploy
  nur:
    - main
```

### Pipelines ausführen

- Pipelines werden automatisch bei Push ausgeführt
- Manuell von der UI auslösen
- Pipeline-Status in Merge-Requests anzeigen

### Pipeline-Variablen

- Variablen in Projekteinstellungen definieren
- CI/CD-Variablen für sensible Daten verwenden
- Maskierte und geschützte Variablen verfügbar
