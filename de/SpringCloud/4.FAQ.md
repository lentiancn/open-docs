# Spring Cloud Häufig Gestellte Fragen

## Allgemeine Fragen

### Was ist Spring Cloud?
Spring Cloud ist ein Framework für den Aufbau von Cloud-nativen verteilten Systemen. Es bietet Tools für Service-Discovery, Konfigurationsmanagement, Circuit Breaker und andere Cloud-native Muster.

### Was ist der Unterschied zwischen Spring Boot und Spring Cloud?
Spring Boot dient zum Erstellen einzelner Anwendungen. Spring Cloud koordiniert mehrere Dienste, um ein verteiltes System zu erstellen.

### Ist Spring Cloud nur für Microservices?
Obwohl hauptsächlich für Microservices verwendet, kann Spring Cloud auch verwendet werden für:
- Monolithische Anwendungen mit Cloud-Funktionen
- Serverlose Architekturen
- Ereignisgesteuerte Systeme

## Installationsfragen

### Wie füge ich Spring Cloud zu meinem Projekt hinzu?
Fügen Sie die Spring Cloud BOM und Abhängigkeiten hinzu:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>${spring-cloud.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
    </dependency>
</dependencies>
```

### Welche Spring Cloud Version sollte ich verwenden?
- Spring Boot 2.7.x → Spring Cloud 2022.0.x
- Spring Boot 2.6.x → Spring Cloud 2021.0.x

## Service-Discovery Fragen

### Wie funktioniert Eureka?
1. Dienste registrieren sich bei Eureka beim Start
2. Clients fragen Eureka ab, um Dienststandorte zu finden
3. Eureka verarbeitet Heartbeats, um gesunde Instanzen zu erkennen

### Wie registriere ich einen Dienst?
```java
@SpringBootApplication
@EnableEurekaClient
public class MyServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyServiceApplication.class, args);
    }
}
```

## Circuit Breaker Fragen

### Wie verwende ich Resilience4j?
```java
@Service
public class MyService {
    
    @CircuitBreaker(name = "myService", fallbackMethod = "fallback")
    public String callService() {
        // externen Dienst aufrufen
    }
    
    public String fallback(Exception e) {
        return "Fallback-Antwort";
    }
}
```

## Fehlerbehebung

### Dienste finden sich gegenseitig nicht
- Überprüfen Sie Eureka-Registrierung
- Verifizieren Sie Netzwerkkonnektivität
- Stellen Sie sicher, dass Dienstnamen korrekt sind

### Konfiguration wird nicht geladen
- Verifizieren Sie, dass Config-Server läuft
- Überprüfen Sie Git-Repository-Konfiguration
