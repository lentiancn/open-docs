# JDK 概要

## JDKとは

JDK（Java Development Kit）は、Javaプログラミング言語用のソフトウェア開発キットであり、Java開発の中核を成すものです。Java実行環境（Java Runtime Environment、JRE）、Javaコンパイラ（javac）、Java仮想マシン（JVM）、および大量のJavaクラスライブラリやツールが含まれています。JDKは、Javaアプリケーション開発のために必須のソフトウェアパッケージであり、デスクトップアプリケーション、Webアプリケーション、モバイルアプリケーション、またはエンタープライズアプリケーションの開発に関係なく、JDKのサポートなしにはできません。

JDKはOracle社（とその前身であるSun Microsystems社）によって開発・維持されており、Javaプラットフォームの基礎となっています。現在、最新の長期サポート（LTS）バージョンはJDK 25とJDK 21であり、JDK 17も広く使用されている長期サポートバージョンです。Oracle社は定期的に新しいJDKバージョンをリリースしており、各バージョンはパフォーマンス向上、新機能、セキュリティ脆弱性の修正をもたらします。

JDKは単なる実行環境ではありません。それは完全なJava開発所需的すべてのコンポーネントを提供します。最もコアなコンポーネントはJavaコンパイラ（javac）であり、Javaソースコードをバイトコード（.classファイル）にコンパイルする役割を担います。また、Java仮想マシン（JVM）はバイトコードを実行し、jar（アーカイブツール）、javadoc（ドキュメント生成ツール）、javap（逆アセンブラ）などのさまざまなツールが含まれています。これらのツールが完全なJava開発環境を構成しています。

## JDKのメジャーバージョン

### JDK 25（新バージョン）

JDK 25は最新のJDKバージョンであり、多くの新機能と改良をもたらします。これは6ヶ月のサポートサイクルを持つ短期サポートバージョンです。主要な機能には、パフォーマンス最適化の向上強化されたガベージコレクタ、新しい言語機能、最新のハードウェアサポートが含まれます。JDK 25は最新のJava機能を使用したい開発者に適していますが、サポートサイクルが短いことに注意してください。

### JDK 21（LTS）

JDK 21は最新の長期サポートバージョン（LTS）であり、少なくとも8年間サポートされます。これはOracle No-Fee Terms and Conditionsの下で無料で使用できる最後の長期サポートバージョンです。JDK 21は仮想スレッド（Virtual Threads）、パターンマッチング（Pattern Matching）、レコード（Records）などの重要な機能を導入しており、これらの機能はJavaプログラミング大大簡略化し、開発効率を向上させます。

仮想スレッドはJDK 21で最も重要な機能の一つであり、高并发アプリケーションの作成をよりシンプルにします。従来のスレッドと比較して、仮想スレッドの作成コストは非常に低く、数百万の仮想スレッドを作成して并发タスクを処理できます。これは、大規模分散システム、マイクロサービスアーキテクチャ、高性能サーバーアプリケーションの構築にとって重要です。

### JDK 17（LTS）

JDK 17は現在入手可能な最も成熟した長期サポートバージョンの一つであり、広く採用されています。これはOracleのライセンス条項に基づく無料Oracle JDKの最後のバージョンであるため、多くの企業がまだJDK 17を使用しています。JDK 17には、シールドクラス（Sealed Classes）、テキストブロック（Text Blocks）、switch式などのOpenJDKからの多くの機能が含まれています。

JDK 17のパフォーマンスはJDK 11と比較して大幅に向上しており、特にガベージコレクションとJITコンパイルの面でそうです。長期安定したサポートが必要なエンタープライズアプリケーションにとって、JDK 17はまだ優れた選択です。それはパフォーマンス、安定性、功能の間で良好なバランスを達成しています。

### JDK 11（LTS）

JDK 11はJavaの有料機能の境目を示す別の重要な長期サポートバージョンです。JDK 11以降、Oracle JDKの一部の高度な機能は有料サブスクリプションが必要になります。JDK 11は新しいガベージコレクタZGC（実験的）、新しいHTTPクライアントAPI、ローカル変数型推論（varキーワード）を導入しています。最新のLTSバージョンではなくなった，但仍広く使用されています。

## JDKのコアコンポーネント

### Javaコンパイラ（javac）

Javaコンパイラ（javac）はJDKの最も本質的なツールの一つであり、Javaソースコード（.javaファイル）をJavaバイトコード（.classファイル）にコンパイルする役割を担います。バイトコードは特定のハードウェアプラットフォームに依存しない中間形式のコードであり、Java仮想マシンによってのみ実行できます。この設計により、Javaプログラムは「一度書いたらどこでも実行」の特性を獲得します。

javacコンパイラは多数のコンパイルオプションをサポートしており、開発者はコマンドラインパラメータを通じてコンパイルプロセスのさまざまな動作を制御できます。たとえば、-dオプションでコンパイル出力ディレクトリを指定し、-classpathでクラスパスを指定し、-sourceと-targetでJavaソースとターゲットバイトコードのバージョンを指定できます。大規模プロジェクトのビルドには、これらのコンパイルオプションの習得が重要です。

### Java仮想マシン（JVM）

Java仮想マシン（JVM）はJavaプログラム実行のためのコア環境であり、バイトコードを読み込んで解釈またはネイティブコードにコンパイルして実行する責任を担います。JVMはJavaのプラットフォーム非依存の鍵であり、基盤となるオペレーティングシステムとハードウェアの違いを抽象化し、同じJavaプログラムが異なるプラットフォームで実行できるようになります。

JVMのコアコンポーネントには、クラスファイルを読み込むClassLoader、バイトコードを実行するExecution Engine、メソッドエリア、ヒープ、仮想スタック、ネイティブメソッドスタック、プログラムカウンタを含むRuntime Data Areasが含まれます。JVMの動作を理解することは、高性能Javaアプリケーションの作成に不可欠です。

JVMは強力なガベージコレクション（GC）機能も提供し、メモリの割り当てと解放を自動化管理します。Serial、Parallel、CMS、G1、ZGC、Shenandoahなどのさまざまなガベージコレクタは、さまざまなアプリケーションシナリオに適しています。開発者はアプリケーションの特性に基づいて適切なガベージコレクタを選択し、最適なパフォーマンスを実現できます。

### Java実行環境（JRE）

Java実行環境（JRE）はJavaプログラムを実行するために必要な環境であり、JVM、Javaコアクラスライブラリ、およびその他のサポートファイルを含みます。JREはJDKのサブセットであり、JDKにはJREと開発ツールが含まれています。Javaプログラムを実行するだけで開発が不要な場合は、JREをインストールするだけで十分です。

JREの中核はJavaコアクラスライブラリであり、java.lang、java.util、java.io、java.netなどのJava言語の基本機能を提供します。これらのクラスライブラリは何年もかけて開発・最適化されており、の基礎からネットワークプログラミングまであらゆる側面をカバーしています。

### Javaツールセット

JDKはJavaプログラムの開発、デバッグ、診断のための豊富なコマンドラインツールを提供します。主要なツールには次のものがあります：

jar：複数のclassファイルを удобноな形でパックして配布・展開するためのJavaアーカイブファイル（.jarファイル）の作成・管理に使用します。

javadoc：JavaソースコードのコメントからHTML形式のAPIドキュメントを生成します。標準的なJavaクラスライブラリドキュメント就是这样生成されます。

javap：Javaバイトコードを逆アセンブリし、クラスファイルの構造とメソッドシグネチャを表示します。Javaコンパイルプロセスの理解とデバッグに非常に役立ちます。

jdb：Javaデバッガーで、コマンドラインインターフェースのデバッグ機能を提供し、ブレークポイントの設定、ステップ実行、変数値の表示などができます。

jps：現在のシステム内のすべてのJavaプロセスの情報を表示します。

jstat：ガベージコレクション情報、クラス読み込み情報など、JVM統計情報を表示します。

jstack：スレッドのスタックトレース情報を出力し、デッドロックなどの問題の診断に使用します。

jmap：Javaヒープメモリ情報を表示し、ヒープダンプファイルを生成して分析に使用できます。

## JDKのアプリケーション領域

### エンタープライズアプリケーション開発

JDKはエンタープライズアプリケーション開発の基盤です。Spring、Hibernate、StrutsなどのさまざまなJava EE（現在はJakarta EEと改名）フレームワークはすべてJDK上で動作します。これらのフレームワークは、依存性注入、トランザクション管理、データ永続化、Webサービスを含むエンタープライズアプリケーションの完全なソリューションを提供します。

大規模なエンタープライズアプリケーションは通常、安定性とパフォーマンスに対して非常に高い要件を持つため、適切なJDKバージョンを選択することが重要です。多くの企業では、長期安定したサポートを確保するために、JDK 17などの成熟したLTSバージョンと商用サポートサービスを組み合わせています。

### Android開発

Androidアプリケーションは主にJavaまたはKotlinで記述されていますが、Androidアプリの開発にはJDKが必要です。Android SDKツールはAndroidプロジェクトをコンパイルし、AndroidデバイスにインストールできるAPKファイルを生成するためにJDKを必要とします。Android StudioにはJDKも組み込まれており、開発者は開発環境を構成する際にJDKのパスを指定する必要があります。

### ビッグデータ処理

ビッグデータ分野では、JavaとJDKが中心的な役割を果たします。Hadoop、Spark、Flinkなどの主要なビッグデータフレームワークはすべてJavaで開発されており、JDK上で動作する必要があります。これらのフレームワークは通常、マスターデータを処理する必要があり、非常に高いパフォーマンスと安定性の要件があります。

### Webアプリケーション開発

Java Webアプリケーション開発はJDKの伝統的な強みです。Servlet、JSP、Filter、Listenerなどの技術はJava Web開発の基盤を形成します。Spring MVC、Struts、JSFなどの主要なWebフレームワークはすべてJDK上で動作します。

### クラウドコンピューティングとマイクロサービス

クラウドコンピューティングとマイクロサービスアーキテクチャのPopularityにより、JDKはこの分野でも重要な役割を果たしています。Spring Cloud、Dubboなどのマイクロサービスフレームワークは分散システム構築のための完全なソリューションを提供します。これらのフレームワークはすべてJDK上で動作する必要があります。

## JDKと他のJava実装

### OpenJDK

OpenJDKはJava SEのオープンソース実装であり、Oracleとコミュニティによって共同で開発・維持されています。OpenJDKは他のほとんどのJDKディストリビューションの基礎です。Eclipse（Eclipse Temurin）、Amazon（Amazon Corretto）、Microsoft（Microsoft Build of OpenJDK）など多くのベンダーがOpenJDKに基づいて独自のJDKディストリビューションを構築しています。

OpenJDKとOracle JDKはコードレベルではほぼ同一であり、主な違いはライセンス契約とサポートサービスにあります。Oracle JDKは商用サポートと追加のエンタープライズ機能を提供しますが、一部のバージョンは商用使用に対して料金を要求場合があります。OpenJDKは完全にオープンソースで無料です。ほとんどのユーザーには、OpenJDKで十分です。

### 他のJDKディストリビューション

Oracle JDKとOpenJDKに加えて、多くの他のJDKディストリビューションがあります：

Eclipse Temurin（旧AdoptOpenJDK）：Eclipse Foundationによって維持されるOpenJDKディストリビューションで、高品質のビルドと長期サポートを提供します。

Amazon Corretto：Amazonが提供する無料、マルチプラットフォームのJDKディストリビューションで、長期サポートとパフォーマンス最適化を提供します。

Microsoft Build of OpenJDK：Microsoftが提供するOpenJDKビルドで、WindowsとAzure向けに最適化されています。

Azul Zulu：Azul Systemsが提供するJDKディストリビューションで、高パフォーマンスと安定性で知られています。

GraalVM：Oracleが提供するマルチ言語仮想マシンで、JavaだけでなくJavaScript、Python、Rubyなどの他の言語もサポートします。

## JDKの重要性

Java開発の基礎としてのJDKの重要性は言説を必要としません。使用するJavaフレームワークやツールに関係なく、JDKのサポートは不可欠です。JDKはJavaプログラムのコンパイルと実行のためのツールセット单単にそれはJavaエコシステムのコアを表現しています。

適切なJDKバージョンを選択して正しく設定することは、プロジェクトの成功にとって重要です。考慮すべき要素には、バージョンの安定性、セキュリティ更新頻度、新機能、パフォーマンス最適化、コミュニティサポートが含まれます。エンタープライズレベルのアプリケーションには、長期サポートを得るためにLTSバージョンの使用をお勧めします。

Java言語の継続的な発展と进化に伴い、JDKもまた更新され続けています。新しいJDKバージョンは、より良いパフォーマンス、より多くの機能、より高いセキュリティをもたらします。JDKを最新の状態に保つことで、より良い開発体験と運用効率を実現できます。

## バージョンによる違い

異なるJDKメジャーバージョンには、安装、使用、功能において大きな違いがあります：

**JDK 8**：商用に完全に 무료なOracle JDKの最後のバージョン。ラムダ式、ストリームAPI、新しい日付/時刻APIを導入しました。レガシーシステムでまだ広く使用されています。

**JDK 11**：Java有料機能の始まりをMark。ローカル変数型推論（var）、新しいHTTPクライアントAPI、ZGC（実験的）を導入しました。32ビットシステムをサポートする最後のバージョンです。

**JDK 17**：NFTCの下で完全に 무료なOracle JDKの最後のバージョン。シールドクラス、テキストブロック、switch式を導入しました。パフォーマンス、安定性、功能の間で優れたバランスを達成しています。

**JDK 21**：最新のLTSバージョン。仮想スレッド、パターンマッチング、レコード（安定）が導入されました。新規プロジェクトに推奨されます。

**JDK 25**：最新バージョンで、最新の言語機能とパフォーマンス改善をもたらします。最新機能したい場合は適していますが、サポートサイクルが短いです。
