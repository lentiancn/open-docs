# Claude Code 자주 묻는 질문

이 페이지에서는 Claude Code 설치 및 사용 중 가장 많이 발생하는 문제와 해결책을 나열합니다.

## 설치 관련 문제

### 명령을 찾을 수 없음

설치 후 `command not found: claude` 또는 `'claude' is not recognized` 오류가 발생하면 설치 디렉토리가 시스템 PATH에 없음을 의미합니다.

**해결책:** PATH 환경 변수를 확인하고 구성합니다.

macOS/Linux에서 .zshrc 또는 .bashrc에 추가:
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Windows PowerShell:
```powershell
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
```

그런 다음 터미널을 다시 시작하고 확인합니다:
```bash
claude --version
```

### 설치 스크립트가 HTML을 반환함

설치 명령을 실행할 때 다음과 같은 오류가 표시됩니다:
- `bash: line 1: syntax error near unexpected token '<'`
- `bash: line 1: '<!DOCTYPE html>'`
- PowerShell에서: `Invoke-Expression: Missing argument in parameter list`

이는 설치 URL이 설치 스크립트 대신 HTML 페이지를 반환했음을 의미합니다.

**해결책:**

1. 대체 설치 방법 사용:
   - macOS/Linux: Homebrew 사용
     ```bash
     brew install --cask claude-code
     ```
   - Windows: WinGet 사용
     ```powershell
     winget install Anthropic.ClaudeCode
     ```

2. HTML 페이지에 "App unavailable in region"이 표시되면 Claude Code가 해당 국가/지역에서 사용 불가합니다. [지원되는 국가 및 지역 목록](https://www.anthropic.com/supported-countries)을 참조하세요.

3. 잠시 기다렸다가 다시 시도하세요 — 일반적으로 일시적인 문제입니다.

### curl 다운로드 실패

오류 메시지: `curl: (56) Failure writing output to destination`

이는 스크립트 다운로드가 완료되기 전에 연결이 끊어졌음을 의미합니다.

**해결책:**

1. 네트워크 안정성 확인:
   ```bash
   curl -fsSL https://storage.googleapis.com -o /dev/null
   ```
   명령이 성공적으로 완료되면 연결은 정상입니다; 설치 명령만 다시 시도하세요.

2. 대체 설치 방법 사용 (위 참조).

### TLS/SSL 연결 오류

오류 메시지: `curl: (35) TLS connect error`, `schannel: next InitializeSecurityContext failed`, 또는 `Could not establish trust relationship for the SSL/TLS secure channel`

**해결책:**

1. 시스템 CA 인증서 업데이트:
   - Ubuntu/Debian:
     ```bash
     sudo apt-get update && sudo apt-get install ca-certificates
     ```
   - macOS (Homebrew 사용):
     ```bash
     brew install ca-certificates
     ```

2. Windows PowerShell에서 TLS 1.2 활성화:
   ```powershell
   [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
   irm https://claude.ai/install.ps1 | iex
   ```

3. 프록시 또는 방화벽 간섭 확인; `NODE_EXTRA_CA_CERTS` 환경 변수 구성.

### 버전 가져오기 실패

오류 메시지: `Failed to fetch version from storage.googleapis.com`

이는 네트워크에서 `storage.googleapis.com`이 차단되었음을 의미합니다.

**해결책:**

1. 직접 연결 테스트:
   ```bash
   curl -sI https://storage.googleapis.com
   ```

2. 프록시 뒤인 경우 `HTTPS_PROXY` 환경 변수 설정.

### Windows에서 Git Bash 필요

오류 메시지: `Claude Code on Windows requires git-bash`

**해결책:** [Git for Windows](https://git-scm.com/downloads/win)를 설치한 다음 설치 명령을 다시 실행합니다.

Git이 설치되어 있지만 오류가 지속되는 경우 설정 파일에서 Git Bash 경로 구성:
```json
{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}
```

### Linux에서 불법 명령어

오류 메시지: `Illegal instruction`

이는 일반적으로 아키텍처 불일치로 인한 것입니다. 설치된 바이너리와 호환되는 CPU 아키텍처인지 확인하세요.

### macOS에서 바이너리 로드 안됨

오류 메시지: `dyld: cannot load` 또는 `Abort trap`

이는 바이너리가 시스템과 호환되지 않음을 의미합니다. 올바른 버전의 Claude Code (Intel 또는 Apple Silicon)를 다운로드했는지 확인하세요.

### 저メモ리 서버에서 설치 중단

오류 메시지: Linux 서버에서 설치 중 `Killed`가 표시됨

**해결책:** 저メモ리 서버의 스왑 공간 추가.

## 인증 관련 문제

### OAuth 오류 또는 403 금지

**해결책:**

1. 계정에 Claude Code 접근 권한이 있는지 확인 (Pro, Max, Teams, Enterprise 또는 Console)
2. 캐시된 자격 증명을 지우고 다시 로그인:
   ```bash
   rm -rf ~/.claude
   claude
   ```
3. 지역이 [지원 목록](https://www.anthropic.com/supported-countries)에 있는지 확인

### 지역 사용 불가

오류 메시지: `App unavailable in region`

Claude Code가 해당 국가/지역에서 사용 불가합니다. [Anthropic 지원 국가 및 지역](https://www.anthropic.com/supported-countries)을 참조하세요.

## 네트워크 구성

### 네트워크 연결 확인

설치 프로그램은 `storage.googleapis.com`에서 다운로드합니다. 연결 가능한지 확인:

```bash
curl -sI https://storage.googleapis.com
```

실패하는 경우 가능한 원인:
- 기업 방화벽 또는 프록시가 Google Cloud Storage 차단
- 지역 네트워크 제한: VPN 또는 대안적 네트워크 시도
- TLS/SSL 문제: 시스템 CA 인증서 업데이트 또는 `HTTPS_PROXY` 구성 여부 확인

### 프록시 구성

기업 프록시 뒤인 경우 설치 전에 프록시 환경 변수 설정:

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

## 충돌 설치 확인

여러 Claude Code 설치가 있으면 버전 불일치 또는 예기치 않은 동작이 발생할 수 있습니다.

설치된 항목 확인:

**macOS/Linux:**
```bash
which -a claude
ls -la ~/.local/bin/claude
npm -g ls @anthropic-ai/claude-code 2>/dev/null
```

**Windows:**
```powershell
where.exe claude
Test-Path "$env:LOCALAPPDATA\Claude Code\claude.exe"
```

여러 설치가 발견되면 하나만 유지합니다 (권장: `~/.local/bin/claude`의 네이티브 설치).

추가 설치 제거:
- npm 전역 설치: `npm uninstall -g @anthropic-ai/claude-code`
- Homebrew 설치: `brew uninstall --cask claude-code`

## 디렉토리 권한 확인

설치 프로그램은 `~/.local/bin/`과 `~/.claude/`에 쓰기 권한이 필요합니다. 이러한 디렉토리가 쓰기 가능한지 확인:

```bash
test -w ~/.local/bin && echo "writable" || echo "not writable"
test -w ~/.claude && echo "writable" || echo "not writable"
```

디렉토리가 쓰기 가능하지 않은 경우:

```bash
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```

## 바이너리 확인

`claude`가 설치되어 있지만 시작 시 충돌하거나 정지하는 경우:

바이너리가 존재하고 실행 가능한지 확인:
```bash
ls -la $(which claude)
```

Linux에서 누락된 공유 라이브러리 확인:
```bash
ldd $(which claude) | grep "not found"
```

기본 확인:
```bash
claude --version
```

## 진단 도구

보다詳細な설치 및 구성 정보를 위해 `claude doctor` 사용:

```bash
claude doctor
```
