# Mockito Installation Guide

Guide to installing and configuring Mockito.

## Prerequisites

- Java Development Kit (JDK) 8 or higher
- Maven or Gradle (recommended) or simple JAR file

## Installation

### Maven

Add the following dependency to your `pom.xml`:

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>5.8.0</version>
    <scope>test</scope>
</dependency>
```

### Gradle

Add the dependency to your `build.gradle`:

```groovy
testImplementation 'org.mockito:mockito-core:5.8.0'
```

### JAR File

Download the latest JAR from Maven Central:
https://repo1.maven.org/maven2/org/mockito/mockito-core/

## IntelliJ IDEA Integration

Mockito works out-of-the-box with IntelliJ IDEA:

1. Open your project
2. Add the Maven dependency
3. Mockito will be automatically recognized

## Eclipse Integration

1. Right-click on your project
2. Select "Build Path" > "Configure Build Path"
3. Add the JAR under "Libraries" > "Add External JARs"

## Configuration

### Mockito Extension for JUnit 5

JUnit 5 (Jupiter) has built-in support:

```java
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class)
class MyTest {
    // Your tests here
}
```

### Enabling Mockito Annotations

To use annotations like `@Mock`, `@Spy`:

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @BeforeEach
    void init() {
        MockitoAnnotations.openMocks(this);
    }
}
```

## Troubleshooting

### Problem: NoClassDefFoundError

**Solution:** Make sure all dependencies (ByteBuddy, Objenesis) are included.

### Problem: Mock not working

**Solution:**
- Check if the class is final
- Make sure you're mocking interfaces
- Use `mockito-inline` for final classes

### Problem: Verification fails

**Solution:**
- Check call order
- Use `times(1)` for explicit verification

## Upgrading from Mockito 4.x

1. Update the version in your build file
2. Check for deprecated APIs
3. Run your tests
