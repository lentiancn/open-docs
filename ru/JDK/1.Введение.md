# Введение в JDK

## Что такое JDK

JDK (Java Development Kit) — это комплект разработки программного обеспечения для языка программирования Java и является основой любой Java-разработки. JDK включает среду выполнения Java (JRE), компилятор Java (javac), виртуальную машину Java (JVM) и обширные библиотеки классов и инструменты Java. JDK необходим для разработки приложений Java — будь то настольные приложения, веб-приложения, мобильные приложения или корпоративные системы.

JDK разрабатывается и поддерживается корпорацией Oracle (и её предшественником Sun Microsystems), служа основой платформы Java. Последними версиями с долгосрочной поддержкой (LTS) являются JDK 25 и JDK 21, при этом JDK 17 также является широко используемой версией LTS. Oracle регулярно выпускает новые версии JDK, каждая из которых привносит улучшения производительности, новые функции и исправления уязвимостей безопасности.

JDK — это не просто среда выполнения; он предоставляет все компоненты, необходимые для полноценной Java-разработки. Наиболее фундаментальным компонентом является компилятор Java (javac), который компилирует исходный код Java в байт-код (файлы .class). Далее следует виртуальная машина Java (JVM), которая выполняет байт-код, а также различные инструменты, такие как jar (инструмент архивирования), javadoc (генератор документации) и javap (дизассемблер). Вместе эти инструменты составляют полноценную среду разработки Java.

## Основные версии JDK

### JDK 25 (Новая версия)

JDK 25 — это последняя версия JDK, содержащая множество новых функций и улучшений. Это версия с краткосрочной поддержкой с циклом поддержки в 6 месяцев. Основные функции включают: улучшенную оптимиза производительности, усовершенствованные сборщики мусора, новые языковые функции и поддержку новейшего оборудования. JDK 25 подходит для разработчиков, желающих использовать новейшие функции Java, но следует учитывать её более короткий цикл поддержки.

### JDK 21 (LTS)

JDK 21 — это последняя версия с долгосрочной поддержкой (LTS), которая будет получать поддержку не менее 8 лет. Это последняя версия LTS с бесплатным использованием Oracle JDK согласно условиям бесплатного использования Oracle. JDK 21 представляет важные функции, такие как виртуальные потоки (Virtual Threads), сопоставление с образцом (Pattern Matching) и записи (Records), которые значительно упрощают программирование на Java и повышают эффективность разработки.

Виртуальные потоки — одна из важнейших функций JDK 21, делающая написание высокопараллельных приложений намного проще. По сравнению с традиционными потоками, виртуальные потоки имеют чрезвычайно низкие затраты на создание и позволяют легко создавать миллионы виртуальных потоков для обработки параллельных задач. Это имеет большое значение для построения крупномасштабных распределённых систем, архитектуры микросервисов и высокопроизводительных серверных приложений.

### JDK 17 (LTS)

JDK 17 — одна из наиболее зрелых доступных версий LTS с широким распространением. Это последняя версия с бесплатным использованием Oracle JDK (согласно лицензионному соглашению Oracle), поэтому многие компании продолжают использовать JDK 17. JDK 17 содержит множество функций из OpenJDK, таких как запечатанные классы (Sealed Classes), текстовые блоки (Text Blocks) и выражения switch.

Производительность JDK 17 значительно улучшена по сравнению с JDK 11, особенно в области сборки мусора и JIT-компиляции. Для корпоративных приложений, требующих стабильной долгосрочной поддержки, JDK 17 остаётся отличным выбором. Он достигает хорошего баланса между производительностью, стабильностью и функциональностью.

### JDK 11 (LTS)

JDK 11 — ещё одна важная версия LTS, ознаменовавшая разделение платных функций Java. Начиная с JDK 11, некоторые расширенные функции Oracle JDK требуют платной подписки. JDK 11 представил сборщик мусора ZGC (экспериментальный), новый HTTP-клиент API и вывод типов локальных переменных (ключевое слово var). Хотя это уже не последняя версия LTS, JDK 11 всё ещё широко используется во многих существующих проектах.

## Основные компоненты JDK

### Компилятор Java (javac)

Компилятор Java (javac) — один из наиболее важных инструментов JDK, отвечающий за компиляцию исходного кода Java (файлы .java) в байт-код Java (файлы .class). Байт-код — это промежуточная форма кода, не зависящая от конкретных аппаратных платформ, и может выполняться только виртуальной машиной Java. Эта конструкция даёт Java-программам характеристику «Напиши один раз, запускай везде».

Компилятор javac поддерживает множество опций компиляции, позволяя разработчикам управлять различными поведением процесса компиляции через параметры командной строки. Например, вы можете использовать опцию -d для указания выходного каталога компиляции, -classpath для указания пути поиска классов, а -source и -target для указания версий Java исходного кода и целевого байт-кода. Освоение этих опций компиляции имеет решающее значение для сборки крупных проектов.

### Виртуальная машина Java (JVM)

Виртуальная машина Java (JVM) — это центральная среда выполнения Java-программ. Она отвечает за загрузку байт-кода и его интерпретацию или компиляцию для выполнения нативного кода. JVM является ключом к кроссплатформенности Java, скрывая различия между операционными системами и базовым оборудованием, позволяя одной и той же Java-программе выполняться на разных платформах.

Основные компоненты JVM включают: загрузчик классов, отвечающий за загрузку файлов классов; движок выполнения, отвечающий за выполнение байт-кода; и области данных выполнения, включающие область методов, кучу, стек виртуальных машин, стек нативных методов и программный счётчик. Понимание работы JVM необходимо для написания высокопроизводительных Java-приложений.

JVM также предоставляет мощную функцию автоматической сборки мусора (GC), автоматически управляя выделением и освобождением памяти. Различные сборщики мусора, такие как Serial, Parallel, CMS, G1, ZGC и Shenandoah, подходят для разных сценариев приложений. Разработчики могут выбрать подходящий сборщик мусора в соответствии с характеристиками приложения для достижения оптимальной производительности.

### Среда выполнения Java (JRE)

Среда выполнения Java (JRE) — это среда, необходимая для выполнения Java-программ. Она включает JVM, основные библиотеки классов Java и другие вспомогательные файлы. JRE является подмножеством JDK; JDK включает JRE плюс инструменты разработки. Если вам нужно только запускать Java-программы без разработки, достаточно установки JRE.

Основой JRE являются основные библиотеки классов Java, предоставляющие базовую функциональность языка Java, такие как java.lang, java.util, java.io, java.net. Эти библиотеки классов разрабатывались и оптимизировались на протяжении многих лет и являются очень зрелыми и стабильными, охватывая всё — от базовых структур данных до сетевого программирования.

### Набор инструментов Java

JDK предоставляет богатый набор инструментов командной строки для разработки, отладки и диагностики Java-программ. Основные инструменты включают:

jar: Используется для создания и управления архивными файлами Java (файлы .jar), которые могут упаковать несколько файлов классов в один удобный файл для распространения и развёртывания.

javadoc: Используется для генерации документации API в формате HTML из комментариев в исходном коде Java — стандартный инструмент для создания документации кода.

javap: Используется для дизассемблирования байт-кода Java, позволяет увидеть структуры файлов классов и сигнатуры методов, очень полезен для понимания процесса компиляции Java и отладки.

jdb: Отладчик Java, предоставляющий функции отладки через интерфейс командной строки, может устанавливать точки останова, выполнять пошаговое выполнение, просматривать значения переменных и т.д.

jps: Отображает информацию обо всех Java-процессах в текущей системе, включая идентификатор процесса и имя главного класса.

jstat: Отображает статистику JVM, такую как информация о сборке мусора, загрузке классов и т.д.

jstack: Выводит информацию о трассировке стека Java-потоков, используется для диагностики проблем блокировки потоков.

jmap: Отображает информацию о куче памяти Java, может генерировать файлы дампа кучи для анализа.

Эти инструменты очень полезны в повседневной разработке и диагностике проблем в продакшене, их освоение может значительно повысить эффективность разработки и возможности диагностики.

## Области применения JDK

### Разработка корпоративных приложений

JDK является основой разработки корпоративных приложений. Различные Java EE фреймворки (теперь переименованные в Jakarta EE), такие как Spring, Hibernate и Struts, требуют для работы JDK. Эти фреймворки предоставляют комплексные решения для корпоративных приложений, включая внедрение зависимостей, управление транзакциями, персистентность данных и веб-сервисы.

Крупные корпоративные приложения обычно предъявляют очень высокие требования к стабильности и производительности, поэтому выбор правильной версии JDK очень важен. Многие компании используют зрелые версии LTS, такие как JDK 17, в сочетании с коммерческими услугами поддержки для обеспечения стабильной долгосрочной работы системы.

### Разработка под Android

Хотя приложения Android пишутся в основном на Java или Kotlin, разработка приложений для Android невозможна без JDK. Инструменты Android SDK требуют JDK для компиляции Android-проектов и генерации файлов APK, которые могут быть установлены на устройства Android. Android Studio также включает встроенный JDK, и разработчики должны указать путь к JDK при настройке среды разработки.

### Обработка больших данных

В области больших данных Java и JDK играют центральную роль. Основные фреймворки больших данных, такие как Hadoop, Spark и Flink, все разработаны на Java и требуют для работы JDK. Эти фреймворки обычно должны обрабатывать огромные объёмы данных и предъявляют очень высокие требования к производительности и стабильности.

### Разработка веб-приложений

Разработка веб-приложений на Java — традиционная сильная сторона JDK. Такие технологии, как Servlet, JSP, Filter и Listener, составляют основу веб-разработки на Java. Основные веб-фреймворки, такие как Spring MVC, Struts и JSF, все работают на JDK.

### Облачные вычисления и микросервисы

С ростом популярности облачных вычислений и архитектуры микросервисов JDK также играет важную роль в этих областях. Фреймворки микросервисов, такие как Spring Cloud и Dubbo, предоставляют комплексные решения для построения распределённых систем. Все эти фреймворки должны работать на JDK.

## JDK и другие реализации Java

### OpenJDK

OpenJDK — это реализация Java с открытым исходным кодом, совместно разрабатываемая и поддерживаемая Oracle и сообществом. OpenJDK является основой большинства других дистрибутивов JDK. Многие поставщики, такие как Eclipse (Eclipse Temurin), Amazon (Amazon Corretto) и Microsoft (Microsoft Build of OpenJDK), создают собственные дистрибутивы JDK на основе OpenJDK.

OpenJDK и Oracle JDK почти идентичны на уровне кода, основные различия заключаются в лицензионных соглашениях и услугах поддержки. Oracle JDK предоставляет коммерческую поддержку и дополнительные функции корпоративного уровня, тогда как OpenJDK полностью бесплатен с открытым исходным кодом. Для большинства пользователей использование OpenJDK вполне достаточно.

### Другие дистрибутивы JDK

Помимо Oracle JDK и OpenJDK, существует множество других дистрибутивов JDK:

Eclipse Temurin (ранее AdoptOpenJDK): Дистрибутив OpenJDK, поддерживаемый Eclipse Foundation, предоставляющий высококачественные сборки и долгосрочную поддержку.

Amazon Corretto: Бесплатный кроссплатформенный дистрибутив JDK от Amazon с долгосрочной поддержкой и оптимизацией производительности.

Microsoft Build of OpenJDK: Сборка OpenJDK от Microsoft, оптимизированная для Windows и Azure.

Azul Zulu: Дистрибутив JDK от Azul Systems, известный высокой производительностью и стабильностью.

GraalVM: Многоязыковая виртуальная машина от Oracle, поддерживающая не только Java, но и JavaScript, Python, Ruby и другие языки.

## Важность JDK

Являясь основой разработки на Java, важность JDK говорит сама за себя. Независимо от используемого фреймворка или инструментов Java, поддержка JDK необходима. JDK — это не просто набор инструментов для компиляции и запуска Java-программ; он также представляет собой ядро экосистемы Java.

Выбор подходящей версии JDK и её правильная настройка имеют решающее значение для успеха проекта. Факторы, которые следует учитывать, включают: стабильность версии, частоту обновлений безопасности, новые функции, оптимизацию производительности и поддержку сообщества. Для корпоративных приложений рекомендуется использовать версии LTS для получения долгосрочной поддержки.

С развитием и эволюцией языка программирования Java JDK продолжает обновляться. Новые версии JDK приносят лучшую производительность, больше функций и повышенную безопасность. Поддержание JDK в актуальном состоянии может обеспечить лучший опыт разработки и эксплуатации.

## Различия между версиями

Различные основные версии JDK имеют значительные различия в установке, использовании и функциональности:

**JDK 8**: Последняя полностью бесплатная версия Oracle JDK для коммерческого использования. Ввела лямбда-выражения, API стримов и новый API даты/времени. Широко используется в устаревших системах.

**JDK 11**: Ознаменовала начало платных функций Java. Ввела вывод типов локальных переменных (var), новый HTTP-клиент и ZGC (экспериментальный). Последняя версия с поддержкой 32-битных систем.

**JDK 17**: Последняя бесплатная версия Oracle JDK по NFTC. Ввела запечатанные классы, текстовые блоки и выражения switch. Достигла отличного баланса между производительностью, стабильностью и функциональностью.

**JDK 21**: Последняя версия LTS с виртуальными потоками, сопоставлением с образцом для switch и записями в качестве стабильных функций. Рекомендуется для новых проектов.

**JDK 25**: Последняя версия с новейшими языковыми функциями и улучшениями производительности. Подходит для разработчиков, желающих новейшие функции, но с более коротким циклом поддержки.
