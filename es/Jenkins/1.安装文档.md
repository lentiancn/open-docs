# Guía de Instalación de Jenkins

Jenkins es un servidor de automatización de código abierto para implementar Integración Continua y Despliegue Continuo (CI/CD). Esta guía cubre los métodos de instalación en diversos entornos.

## Información de Versión

| Versión | Estado | Descripción |
|---------|--------|-------------|
| Jenkins 2.x | ✅ Actual | Última versión estable |
| Jenkins 1.x | ❌ Obsoleta | Mantenimiento discontinued |

| Software | Versión |
|----------|---------|

## Métodos de Instalación

### Método 1: Instalación en Windows

#### 1. Usar el Instalador

1. Descargar Jenkins: https://www.jenkins.io/download/
2. Ejecutar el instalador jenkins.msi
3. Seguir el asistente de instalación

#### 2. Usar Chocolatey

```powershell
choco install jenkins
```

#### 3. Instalación Manual

```powershell
# Descargar el archivo WAR
# Extraer al directorio especificado
java -jar jenkins.war
```

### Método 2: Instalación en macOS

#### 1. Usar Homebrew

```bash
# Instalar Jenkins
brew install jenkins

# Iniciar el servicio
brew services start jenkins

# Detener el servicio
brew services stop jenkins
```

#### 2. Instalación Manual

```bash
# Descargar el archivo WAR
wget https://get.jenkins.io/jenkins.war

# Ejecutar
java -jar jenkins.war
```

### Método 3: Instalación en Linux

#### Ubuntu/Debian

```bash
# Añadir la clave
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -

# Añadir el repositorio
sudo sh -c 'echo deb https://pkg.jenkins.io/debian binary/ > /etc/apt/sources.list.d/jenkins.list'

# Actualizar e instalar
sudo apt-get update
sudo apt-get install jenkins

# Iniciar el servicio
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

#### CentOS/RHEL

```bash
# Añadir el repositorio
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat/jenkins.repo

# Importar la clave
sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

# Instalar
sudo yum install jenkins

# Iniciar el servicio
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

### Método 4: Instalación con Docker

```bash
# Ejecutar el contenedor de Jenkins
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts
```

#### Docker Compose

```yaml
version: '3'
services:
  jenkins:
    image: jenkins/jenkins:lts
    ports:
      - "8080:8080"
    volumes:
      - jenkins_home:/var/jenkins_home
```

### Método 5: Despliegue de Archivo WAR

```bash
# Descargar el archivo WAR
wget https://get.jenkins.io/war-stable/latest/jenkins.war

# Ejecutar
java -jar jenkins.war --httpPort=8080
```

## Configuración Post-Instalación

### Primer Acceso

1. Abrir el navegador en http://localhost:8080
2. Obtener la contraseña inicial del administrador:
   ```bash
   # Linux/macOS
   sudo cat /var/lib/jenkins/secrets/initialAdminPassword
   
   # Docker
   docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
   ```
3. Instalar los plugins recomendados
4. Crear un usuario administrador

### Configurar Interfaz en Español

1. Ir a Manage Jenkins → Manage Plugins
2. Instalar "Localization: Spanish"
3. Reiniciar Jenkins

## Preguntas Frecuentes

### ¿No inicia?

Verificar si Java está instalado:

```bash
java -version
```

### ¿Puerto ocupado?

Cambiar el puerto:

```bash
java -jar jenkins.war --httpPort=8090
```

### ¿Error al instalar plugins?

Verificar la conexión de red o usar un proxy.
