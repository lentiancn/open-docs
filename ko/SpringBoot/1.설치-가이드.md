# Spring Boot 설치 가이드

이 가이드에서는 Windows, macOS 및 Linux에서 Spring Boot 개발 환경을 설치하고 설정하는 방법을 설명합니다.

## 시스템 요구사항

### Java 버전 요구사항

| Spring Boot 버전 | 최소 Java 버전 |
|-----------------|---------------|
| 3.x             | Java 17+      |
| 2.7.x           | Java 8+       |
| 2.5.x           | Java 8+       |

**참고:** Spring Boot 3.x에는 Java 17 이상이 필요합니다. 최상의 호환성을 위해 Java 21(LTS)을 사용하는 것이 좋습니다.

## Java 설치

### Windows

**옵션 1: Oracle/OpenJDK에서 설치**
1. [Oracle](https://www.oracle.com/java/technologies/downloads/) 또는 [Adoptium](https://adoptium.net/)에서 JDK를 다운로드합니다
2. 설치 프로그램을 실행합니다
3. 환경 변수를 설정합니다:
   ```
   JAVA_HOME=C:\Program Files\Java\jdk-21
   PATH=%JAVA_HOME%\bin;%PATH%
   ```

**옵션 2: Chocolatey 사용**
```powershell
choco install openjdk21
```

**옵션 3: Winget 사용**
```powershell
winget install Oracle.JDK.21
```

확인:
```cmd
java -version
```

### macOS

**옵션 1: Homebrew 사용**
```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 21)' >> ~/.zshrc
source ~/.zshrc
```

**옵션 2: SDKMAN 사용**
```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.2-tem
```

확인:
```bash
java -version
```

### Linux

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install openjdk-21-jdk
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
```

**RHEL/CentOS/Fedora:**
```bash
sudo dnf install java-21-openjdk-devel
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH=$JAVA_HOME/bin:$PATH
```

**SDKMAN 사용:**
```bash
curl -s "https://get.sdkman.io" | bash
source ~/.sdkman/bin/sdkman-init.sh
sdk install java 21.0.2-tem
```

확인:
```bash
java -version
```

## IDE 설정

### IntelliJ IDEA (권장)

1. [JetBrains](https://www.jetbrains.com/idea/download/)에서 다운로드합니다
2. 설치하고 엽니다
3. JDK 구성: `File` → `Project Structure` → `Project SDK`
4. Spring Boot Assistant 플러그인 설치 (선택사항)

### VS Code

1. [VS Code](https://code.visualstudio.com/)를 설치합니다
2. 확장 설치:
   - Extension Pack for Java
   - Spring Boot Extension Pack
3. Java 구성: `Ctrl+Shift+P` → `Java: Configure Java Runtime`

### Eclipse

1. [Spring Tools 4 (STS)](https://spring.io/tools)를 다운로드합니다
2. 압축을 풀고 실행합니다
3. File → New → Spring Starter Project

## 빠른 시작

### Spring Initializr 사용

1. [start.spring.io](https://start.spring.io/)를 방문합니다
2. 옵션 선택:
   - Project: Maven / Gradle
   - Language: Java / Kotlin / Groovy
   - Spring Boot Version: 3.2.x (최신)
   - Group: com.example
   - Artifact: myapp
   - Packaging: Jar / War
   - Java Version: 17 / 21
3. 의존성 추가:
   - Spring Web
   - Spring Data JPA
   - H2 Database (dev)
   - Spring Boot DevTools
4. "Generate"를 클릭하여 다운로드합니다

### 명령줄 사용

```bash
# Maven 프로젝트 생성
curl https://start.spring.io/starter.zip \
  -d type=maven-project \
  -d language=java \
  -d bootVersion=3.2.5 \
  -d baseDir=myapp \
  -d dependencies=web,jpa,h2,devtools \
  -o myapp.zip

unzip myapp.zip
cd myapp
./mvnw spring-boot:run
```

## 설치 확인

```bash
# Java 확인
java -version

# Maven 확인
mvn -version

# Gradle 확인 (설치된 경우)
gradle -version

# Spring Boot CLI 확인
spring --version

# 데모 프로젝트 실행
curl https://start.spring.io/starter.zip -o demo.zip
unzip demo.zip -d demo
cd demo
./mvnw spring-boot:run
```

## 일반적인 문제

### Java를 찾을 수 없음
- JAVA_HOME이 올바르게 설정되었는지 확인합니다
- 환경 변수를 설정한 후 터미널을 다시 시작합니다
- `echo $JAVA_HOME` (Linux/macOS) 또는 `echo %JAVA_HOME%` (Windows)로 확인합니다

### 포트 8080이 이미 사용 중
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <pid> /F

# Linux/macOS
lsof -i :8080
kill -9 <pid>
```

### Maven 빌드 실패
- Maven 캐시 지우기: `mvn clean`
- 의존성 업데이트: `mvn dependency:resolve`
- 시스템 Maven 대신 `./mvnw` (Maven Wrapper) 사용
