# Часто задаваемые вопросы о MySQL

## Проблемы с установкой

### Q: Что делать, если служба MySQL не запускается?

A: Проверьте следующее:
1. Подтвердите, что MySQL правильно установлен
2. Проверьте журналы ошибок: `mysqld --console`
3. Убедитесь, что порт не занят: `netstat -an | grep 3306`
4. Проверьте правильность разрешений для каталога данных
5. Попробуйте повторно инициализировать каталог данных

### Q: Как сбросить пароль root, если я его забыл?

A: 
1. Остановите службу MySQL
2. Запустите с пропуском таблиц привилегий: `mysqld --skip-grant-tables`
3. Войдите в MySQL
4. Сбросьте пароль: `UPDATE mysql.user SET Password=PASSWORD('новый_пароль') WHERE User='root';`
5. Обновите привилегии: `FLUSH PRIVILEGES;`
6. Перезапустите службу MySQL

### Q: Как изменить номер порта MySQL?

A: Измените файл конфигурации my.cnf или my.ini:
```ini
[mysqld]
port=3307
```
Затем перезапустите службу MySQL.

## Проблемы с подключением

### Q: Что делать, если удалённое подключение отклоняется?

A: 
1. Подтвердите, что MySQL разрешает удалённые подключения (проверьте хост пользователя в таблице пользователей)
2. Проверьте, разрешает ли брандмауэр порт
3. Убедитесь, что bind-address не 127.0.0.1 в конфигурации MySQL
4. Проверьте настройки SELinux или AppArmor

### Q: Как обработать тайм-ауты подключения?

A: 
1. Проверьте сетевое подключение
2. Увеличьте параметр wait_timeout
3. Используйте постоянные подключения или пул подключений
4. Проверьте нагрузку сервера

## Проблемы с операциями базы данных

### Q: как обработать проблемы с кодировкой?

A: 
1. Укажите кодировку при создании базы данных: `CREATE DATABASE db CHARACTER SET utf8mb4;`
2. Укажите кодировку при подключении: `SET NAMES utf8mb4;`
3. Установите кодировку по умолчанию в конфигурации:
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: Как обработать проблемы с кодировкой при импорте/экспорте?

A: 
1. Укажите кодировку при экспорте: `mysqldump --default-character-set=utf8mb4`
2. Укажите кодировку при импорте: `mysql --default-character-set=utf8mb4`

### Q: Нормально ли, что автоинкрементные первичные ключи не последовательны?

A: Да, это нормально. Причины включают:
- Неудавшиеся вставки потребляют ID
- Откаты транзакций не возвращают использованные ID
- Пакетные вставки могут вызывать пропуски
- Удаление записей не влияет на счётчик автоинкремент

## Проблемы с производительностью

### Q: Что делать, если запросы медленные?

A: 
1. Используйте EXPLAIN для анализа плана выполнения запроса
2. Создайте индексы для столбцов в WHERE, JOIN, ORDER BY
3. Избегайте полного сканирования таблиц
4. Оптимизируйте запросы, избегайте SELECT *
5. Рассмотрите использование кэша запросов (удалён в MySQL 8.0)
6. Разделите таблицы или базы данных для снижения нагрузки

### Q: Как просмотреть медленные запросы?

A: 
1. Включите журналирование медленных запросов:
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. Проверьте расположение журнала медленных запросов:
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: Как оптимизировать высокое использование памяти?

A: 
1. Настройте innodb_buffer_pool_size (рекомендуется 70% доступной памяти)
2. Оптимизируйте запросы для устранения ненужных сканирований
3. Ограничьте количество подключений
4. Очистите ненужные кэши

## Проблемы с резервным копированием и восстановлением

### Q: Что делать, если файл резервной копии слишком большой?

A: 
1. Используйте сжатую резервную копию: `mysqldump | gzip > backup.sql.gz`
2. Создавайте резервные копии баз данных/таблиц отдельно
3. Используйте xtrabackup для физического резервного копирования
4. Регулярно очищайте исторические данные

### Q: Как реализовать инкрементное резервное копирование?

A: 
1. Включите бинарное журналирование: `log-bin=mysql-bin`
2. Регулярно создавайте резервные копии бинарных журналов
3. Используйте инструмент mysqlbinlog для восстановления инкрементных данных

## Проблемы с репликацией

### Q: Что делать, если задержка репликации мастер-слэйв слишком высока?

A: 
1. Проверьте задержку сети
2. Настройте slave_parallel_workers для увеличения параллельной репликации
3. Убедитесь, что оборудование слэйва достаточно производительное
4. Оптимизируйте скорость записи мастера
5. Проверьте правила фильтрации репликации

### Q: Как обработать несоответствие данных мастер-слэйв?

A: 
1. Пересинхронизируйте слэйв
2. Используйте pt-table-checksum для проверки несоответствий
3. Используйте pt-table-sync для исправления данных
4. Перенастройте репликацию мастер-слэйв

## Проблемы безопасности

### Q: Как усилить безопасность MySQL?

A: 
1. Установите политики надёжных паролей
2. Ограничьте удалённый вход для пользователя root
3. Удалите анонимные учётные записи
4. Удалите тестовую базу данных
5. Регулярно обновляйте версию MySQL
6. Включите шифрованные подключения SSL/TLS
7. Настройте правила брандмауэра
8. Следуйте принципу минимальных привилегий

### Q: Как предотвратить SQL-инъекцию?

A: 
1. Используйте подготовленные выражения (Prepared Statements)
2. Проверяйте и фильтруйте пользовательский ввод
3. Ограничьте права пользователей базы данных
4. Избегайте динамической конкатенации SQL
5. Используйте хранимые процедуры
