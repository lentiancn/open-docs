# OpenFeign Руководство пользователя

## Балансировка нагрузки

### Интеграция Spring Cloud LoadBalancer

OpenFeign по умолчанию интегрируется с Spring Cloud LoadBalancer, обеспечивая функциональность балансировки нагрузки на стороне клиента.

```java
@FeignClient(name = "user-service")
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}
```

Настройте экземпляры сервисов в `application.yml`:

```yaml
spring:
  cloud:
    discovery:
      client:
        simple:
          instances:
            user-service:
              - uri: http://localhost:8081
              - uri: http://localhost:8082
              - uri: http://localhost:8083
```

### Пользовательская стратегия балансировки нагрузки

```java
@Configuration
public class LoadBalancerConfig {
    
    @Bean
    public ServiceInstanceListSupplier serviceInstanceListSupplier(
            ConfigurableApplicationContext context) {
        return ServiceInstanceListSupplier.builder()
                .withDiscoveryClient()
                .withCaching()
                .build(context);
    }
}
```

### Преобразование HTTP-запроса с балансировкой нагрузки

```java
@Bean
public LoadBalancerFeignRequestTransformer transformer() {
    return new LoadBalancerFeignRequestTransformer() {
        @Override
        public Request transformRequest(Request request, ServiceInstance instance) {
            Map<String, Collection<String>> headers = new HashMap<>(request.headers());
            headers.put("X-ServiceId", Collections.singletonList(instance.getServiceId()));
            headers.put("X-InstanceId", Collections.singletonList(instance.getInstanceId()));
            return Request.create(request.httpMethod(), request.url(), headers, 
                request.body(), request.charset(), request.requestTemplate());
        }
    };
}
```

## Поддержка Circuit Breaker

### Интеграция Spring Cloud CircuitBreaker

Включите поддержку Circuit Breaker:

```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

### Откат Circuit Breaker

```java
@FeignClient(name = "user-service", fallback = UserServiceFallback.class)
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}

@Component
public class UserServiceFallback implements UserService {
    @Override
    public User getUser(Long id) {
        // Возвращает пользователя по умолчанию или выбрасывает конкретное исключение
        return new User(id, "Default User");
    }
}
```

### Фабрика отката с информацией об исключении

```java
@FeignClient(name = "user-service", fallbackFactory = UserServiceFallbackFactory.class)
public interface UserService {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}

@Component
public class UserServiceFallbackFactory implements FallbackFactory<UserService> {
    @Override
    public UserService create(Throwable cause) {
        return new UserService() {
            @Override
            public User getUser(Long id) {
                // Может получить доступ к исключению, вызвавшему откат
                log.error("UserService fallback triggered", cause);
                return new User(id, "Fallback User");
            }
        };
    }
}
```

### Пользовательское имя Circuit Breaker

```java
@Configuration
public class CircuitBreakerConfig {
    @Bean
    public CircuitBreakerNameResolver circuitBreakerNameResolver() {
        return (String feignClientName, Target<?> target, Method method) -> 
            feignClientName + "_" + method.getName();
    }
}
```

## Логирование

### Настройка уровня логирования

```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Или через файл конфигурации:

```yaml
logging:
  level:
    com.example.clients.UserService: DEBUG

spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
```

### Пользовательская фильтрация логов

```java
public class CustomLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        // Фильтрация конфиденциальной информации
        if (!shouldLogRequestHeader(request)) {
            // Пользовательская логика логирования
        }
    }
    
    @Override
    protected Response logAndRebufferResponse(String configKey, Level logLevel, 
            Response response, long elapsedTime) throws IOException {
        // Фильтрация конфиденциальной информации ответа
        if (!shouldLogResponseHeader(response)) {
            // Пользовательская логика логирования
        }
        return super.logAndRebufferResponse(configKey, logLevel, response, elapsedTime);
    }
    
    private boolean shouldLogRequestHeader(Request request) {
        // Фильтрация заголовка Authorization и другой конфиденциальной информации
        return !request.headers().containsKey("Authorization");
    }
    
    private boolean shouldLogResponseHeader(Response response) {
        return true; // Или фильтровать по необходимости
    }
}
```

## Сжатие запросов/ответов

### Включение сжатия

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

### Важные замечания

- При использовании OkHttpClient сжатие отключается, если присутствуют заголовки `content-encoding` или `accept-encoding`
- Сжатие применяется только к указанным MIME-типам
- Размер запроса должен превышать `min-request-size`, чтобы быть сжатым

## Поддержка OAuth2

### Включение OAuth2

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
```

### Конфигурация зависимостей

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>
```

### Конфигурация клиента OAuth2

```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: your-client-id
            client-secret: your-client-secret
            authorization-grant-type: client_credentials
            scope: read,write
        provider:
          my-provider:
            token-uri: https://auth-server.com/oauth/token
```

## Поддержка кэширования

### Включение кэширования

```java
@SpringBootApplication
@EnableCaching
@EnableFeignClients
public class Application {
    // ...
}
```

### Использование кэширования в Feign-клиенте

```java
@FeignClient(name = "data-service")
public interface DataService {
    
    @GetMapping("/data/{key}")
    @Cacheable(cacheNames = "data-cache", key = "#key")
    Data getData(@PathVariable String key);
    
    @PostMapping("/data")
    @CachePut(cacheNames = "data-cache", key = "#data.key")
    Data updateData(@RequestBody Data data);
    
    @DeleteMapping("/data/{key}")
    @CacheEvict(cacheNames = "data-cache", key = "#key")
    void deleteData(@PathVariable String key);
}
```

### Отключение кэширования

```yaml
spring:
  cloud:
    openfeign:
      cache:
        enabled: false
```

## Асинхронная поддержка

### Поддержка CompletableFuture

```java
@FeignClient(name = "async-service")
public interface AsyncService {
    
    @GetMapping("/data/{id}")
    CompletableFuture<Data> getDataAsync(@PathVariable String id);
}
```

### Настройка асинхронного клиента

```java
@Configuration
public class AsyncFeignConfig {
    
    @Bean
    public AsyncClient asyncClient() {
        return new AsyncClient.Default();
    }
}
```

### Пример использования

```java
@Service
public class AsyncProcessingService {
    
    private final AsyncService asyncService;
    
    public AsyncProcessingService(AsyncService asyncService) {
        this.asyncService = asyncService;
    }
    
    public void processData() {
        CompletableFuture<Data> future1 = asyncService.getDataAsync("1");
        CompletableFuture<Data> future2 = asyncService.getDataAsync("2");
        
        // Обработка параллельно
        CompletableFuture.allOf(future1, future2).thenRun(() -> {
            try {
                Data data1 = future1.get();
                Data data2 = future2.get();
                // Обработка данных
            } catch (Exception e) {
                log.error("Error processing async data", e);
            }
        });
    }
}
```

## Поддержка области действия обновления

### Включение поддержки обновления

```yaml
spring:
  cloud:
    openfeign:
      client:
        refresh-enabled: true
```

### Обновление конфигурации

Обновите через конечную точку Actuator:

```bash
POST /actuator/refresh
```

### Важные замечания

- Не аннотируйте интерфейс `@FeignClient` с помощью `@RefreshScope`
- Функциональность обновления по умолчанию отключена, требует явного включения
- Поддерживает обновление времени ожидания подключения, времени ожидания чтения и конфигурации URL

## Поддержка AOT и нативных образов

### Требования к конфигурации

```yaml
spring:
  cloud:
    refresh:
      enabled: false
    openfeign:
      client:
        refresh-enabled: false
      lazy-attributes-resolution: false
```

### Создание нативного образа

Убедитесь, что во время сборки установлена правильная конфигурация, и переопределите URL во время выполнения через системные свойства:

```bash
-Dspring.cloud.openfeign.client.config.[clientId].url=[url]
```

Эти расширенные функции делают OpenFeign не просто простым HTTP-клиентом, а комплексным решением для связи микросервисов, способным удовлетворить различные требования корпоративных приложений.
