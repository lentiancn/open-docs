# JDK 常見問題

## 基礎問題

### 什麼是 JDK

JDK（Java Development Kit）是 Java 語言的軟體開發工具包，它是開發 Java 應用程式所必需的軟體包。JDK 包含了 Java 編譯器（javac）、Java 虛擬機（JVM）、Java 核心類別庫以及各種開發工具。沒有 JDK，就無法開發和運行 Java 程式。

JDK 和 JRE 的區別在於：JRE（Java Runtime Environment）只包含運行 Java 程式所需的環境，不包含開發工具。如果只是要運行別人開發好的 Java 程式，安裝 JRE 就夠了；如果要自己開發 Java 程式，必須安裝 JDK。

### 什麼是 JVM

JVM（Java Virtual Machine）是 Java 虛擬機的縮寫，它是 Java 程式運行的核心。JVM 負責將 Java 位元組碼（.class 檔案）解釋或編譯為本機代碼執行。JVM 的存在使得 Java 程式可以實現「一次編寫，到處運行」的跨平台特性。

JVM 不僅運行 Java 程式，還負責記憶體管理（垃圾回收）、執行緒管理，安全管理工作等。理解 JVM 的工作原理對於編寫高效能 Java 應用非常重要。

### JDK 和 JRE 有什麼區別

JDK（Java Development Kit）是完整的開發工具包，包含了 JRE 和開發工具。JRE（Java Runtime Environment）是 Java 執行環境，只包含運行 Java 程式所需的組件。

簡單來說：如果你只是要運行別人開發好的 Java 程式，安裝 JRE 就夠了；如果你要自己開發 Java 程式，必須安裝 JDK。JDK 中包含了完整的 JRE，所以安裝了 JDK 後不需要再單獨安裝 JRE。

### 哪個 JDK 版本最好

沒有絕對的「最好」版本，選擇取決於具體需求。對於新專案，推薦使用 JDK 21（最新的 LTS 版本）或 JDK 25（最新版本）。對於需要長期維護的企業專案，推薦使用 LTS 版本如 JDK 21 或 JDK 17。

JDK 21 是最新的長期支援版本，獲得至少 8 年的支援。JDK 17 仍然廣泛應用且免費使用。JDK 8 雖然老舊，但仍然在很多遺留專案中使用。

## 安裝問題

### 如何檢查 JDK 是否安裝成功

在命令列中運行 java -version，如果顯示版本資訊則說明 JDK 已正確安裝。運行 javac -version 可以檢查編譯器是否可用。運行 echo %JAVA_HOME%（Windows）或 echo $JAVA_HOME（Linux/macOS）可以檢查環境變數是否設定正確。

如果在命令列中運行這些命令出現錯誤，首先確認 JDK 是否正確安裝，然後檢查環境變數設定是否正確。

### JAVA_HOME 是什麼

JAVA_HOME 是一個環境變數，指向 JDK 的安裝目錄。許多 Java 工具和建置腳本（如 Maven、Gradle）依賴 JAVA_HOME 來定位 JDK。正確設定 JAVA_HOME 可以確保這些工具能夠正常工作。

在 Windows 上，JAVA_HOME 通常設定為如 C:\Program Files\Java\jdk-21 這樣的路徑。在 Linux/macOS 上，通常設定為如 /usr/lib/jvm/java-21-openjdk 這樣的路徑。

### 如何在 Windows 上安裝多個 JDK 版本

可以在同一台機器上安裝多個 JDK 版本。安裝多個版本後，通過修改 PATH 環境變數中 JDK bin 目錄的順序，或者修改 JAVA_HOME 環境變數來切換預設版本。

也可以使用第三方工具來管理多個 JDK 版本，如 SDKMAN! 的 Windows 版本或 JetBrains Toolbox。使用這些工具可以更方便地在不同版本之間切換。

### Oracle JDK 和 OpenJDK 有什麼區別

Oracle JDK 和 OpenJDK 在代碼層面幾乎完全相同，主要區別在於許可協議和技術支援。Oracle JDK 提供商業支援和額外的企業級功能，但某些版本在商業使用時可能需要付費。OpenJDK 是開源免費的，由社區維護。

對於大多數使用場景，OpenJDK 完全可以滿足需求。許多組織選擇使用 OpenJDK 以避免許可問題，同時獲得與 Oracle JDK 幾乎相同的效能和功能。

## 使用問題

### 什麼是 classpath

classpath 是 JVM 用來尋找類別檔案的路徑。當你運行 java MyClass 時，JVM 會在 classpath 指定的路徑和 JAR 檔案中尋找 MyClass.class。

在編譯和運行 Java 程式時，可以通過 -classpath 或 -cp 選項指定 classpath。如果未指定，JVM 預設使用當前目錄作為 classpath。

### 什麼是位元組碼

位元組碼是 Java 編譯器將原始碼編譯成一種中間形式的代碼。位元組碼不是本機代碼，不能直接在 CPU 上運行，只能由 JVM 解釋執行或 JIT 編譯後執行。

位元組碼的存在使得 Java 程式具有跨平台特性。同樣的位元組碼可以在任何安裝了 JVM 的系統上運行，無論是 Windows、Linux 還是 macOS。

### 什麼是 JAR 檔案

JAR（Java Archive）檔案是一種將多個 Java 類別檔案打包成一個檔案的格式。JAR 檔案實際上是 ZIP 格式的壓縮包，裡面包含了編譯後的 .class 檔案和一些中繼資料。

使用 jar 工具可以建立、查看和提取 JAR 檔案。可執行的 JAR 檔案包含 MANIFEST.MF 檔案，其中指定了主類，可以直接用 java -jar 命令運行。

### 什麼是模組系統

Java 模組系統（Project Jigsaw）從 JDK 9 開始引入，將 JDK 和應用程式劃分為模組。模組提供了更好的封裝，明確聲明了依賴關係，有助於建構更健壯和可維護的應用程式。

模組通過 module-info.java 檔案聲明，定義了模組名、依賴的其他模組、匯出的套件等資訊。模組系統幫助解決了 classpath 的類別衝突問題，並提供了更好的版本管理。

## 效能問題

### 如何優化 JVM 效能

JVM 效能優化主要從以下幾個方面入手：合理設定堆記憶體大小，避免設定過大或過小；選擇合適的垃圾回收器，G1 是大多數情況的好選擇；對於追求低延遲的應用，考慮使用 ZGC 或 Shenandoah；啟用類別資料共享（-Xshare:classes）加快啟動速度。

使用效能監控工具（如 VisualVM、JConsole）分析應用效能，找出瓶頸並進行針對性優化。

### 什麼是垃圾回收

垃圾回收（Garbage Collection，GC）是 JVM 自動管理記憶體的機制。程式運行時會產生物件，這些物件佔用記憶體。當物件不再被引用時，垃圾回收器會自動識別並回收這些物件佔用的記憶體。

Java 的垃圾回收大大簡化了記憶體管理工作，程式設計師不需要顯式釋放記憶體。但理解垃圾回收的原理有助於編寫更高效的代碼，避免記憶體洩漏和頻繁的 GC。

### 什麼是記憶體洩漏

記憶體洩漏是指程式在運行過程中申請的記憶體無法被回收，導致可用記憶體逐漸減少。在 Java 中，記憶體洩漏通常是由於持有物件引用的集合（如靜態的 Map、List）不斷添加物件但沒有及時清理造成的。

排查記憶體洩漏可以使用 jmap 產生堆轉儲檔案，然後使用 MAT（Memory Analyzer Tool）等工具分析來找出洩漏的物件。

### 什麼是 JIT 編譯器

JIT（Just-In-Time）編譯器是 JVM 的一部分，負責將熱點位元組碼編譯為本機代碼以提高執行效能。JVM 在運行時會分析代碼的執行情況，對於頻繁執行的代碼（熱點代碼），JIT 編譯器會將其編譯為本地機器碼，從而顯著提升效能。

JIT 編譯是在運行時進行的，不需要額外的編譯步驟。C1 編譯器編譯速度快但優化程度低，C2 編譯器編譯速度慢但優化程度高。分层編譯結合了兩者的優點。

## 版本問題

### 應該使用哪個 Java 版本

對於新專案，推薦使用最新的 LTS 版本 JDK 21。它提供了良好的效能、豐富的特性和長期支援。對於需要兼容 Java 8/11 的專案，可以使用這些版本。

不建议使用已停止支援的版本，如 JDK 8 已經停止公開更新（除非付費訂閱）。及時升級到新的 JDK 版本可以獲得安全更新和效能改進。

### Java 版本號中的 LTS 是什麼意思

LTS（Long-Term Support）是長期支援版本的縮寫。LTS 版本會獲得 Oracle 至少 8 年的技術支援和修補程式更新。目前的 LTS 版本包括 JDK 8、JDK 11、JDK 17 和 JDK 21。

非 LTS 版本（如 JDK 25）的支援週期較短，通常只有 6 個月。這些版本適合想要使用最新特性的開發者，但對於生產環境，推薦使用 LTS 版本。

### Java 17 還能用嗎

Java 17 是一個成熟的 LTS 版本，仍然可以正常使用。Oracle 在 2024 年 10 月更改了 JDK 17 的許可條款，但根據 Oracle 的聲明，JDK 17 仍然可以免費用於開發、個人使用和運行 Oracle 許可的產品。

許多企業仍在使用 JDK 17，因為它是一個非常成熟和穩定的版本，社區也有大量的支援資源。如果專案已經在使用 JDK 17，可以繼續使用；如果開始新專案，更推薦使用 JDK 21。

## 開發問題

### 如何編譯 Java 檔案

使用 javac 命令編譯 Java 檔案。基本用法是：javac MyClass.java。編譯後會產生 MyClass.class 位元組碼檔案。

對於專案，通常使用建置工具如 Maven 或 Gradle 來管理編譯過程，這些工具會自動處理依賴和編譯選項。

### 如何運行 Java 程式

使用 java 命令運行編譯後的 Java 程式。基本用法是：java MyClass。注意這裡不需要 .class 副檔名。

如果程式打包在 JAR 檔案中，使用 java -jar MyApp.jar 運行。

### 如何設定 classpath

在運行時通過 -classpath 或 -cp 選項指定 classpath：java -cp lib/*:out MyClass。在 Windows 上，分隔符是分號（;）；在 Linux/macOS 上，分隔符是冒號（:）。

也可以通過 CLASSPATH 環境變數設定，但這不如命令列指定靈活。

### 什麼是 Maven 和 Gradle

Maven 和 Gradle 是 Java 專案的建置工具。它們可以自動下載依賴、編譯代碼、運行測試、打包發布，大大簡化了專案建置過程。

Maven 使用 XML（pom.xml）定義專案配置，Gradle 使用 Groovy 或 Kotlin 腳本（build.gradle）定義建置邏輯。兩者都是 Java 專案中廣泛使用的建置工具。
