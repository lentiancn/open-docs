# Redis Übersicht

Redis (Remote Dictionary Server) ist ein Open-Source-In-Memory-Datenspeicher, der als Datenbank, Cache und Nachrichtenbroker verwendet wird. Es unterstützt verschiedene Datenstrukturen wie Strings, Hashes, Listen, Sets und sortierte Sets.

## Was ist Redis?

Redis ist eine NoSQL-Datenbank, die Daten als Schlüssel-Wert-Paare speichert. Sie ist bekannt für ihre außergewöhnliche Leistung, Flexibilität und umfangreichen Funktionen.

## Hauptmerkmale

### In-Memory-Speicher
Redis speichert alle Daten im Speicher (RAM) und ermöglicht extrem schnelle Lese- und Schreiboperationen.

### Datenstrukturen
Redis unterstützt mehrere Datentypen:
- **Strings**: Grundlegende Schlüssel-Wert-Speicherung
- **Listen**: Geordnete Sammlungen von Strings
- **Sets**: Ungeordnete Sammlungen eindeutiger Strings
- **Sortierte Sets**: Sets mit Bewertung für Ranglisten
- **Hashes**: Maps zwischen String-Feldern und Werten

### Persistenz
Redis bietet optionale Persistenz durch:
- RDB (Redis Database) Snapshots
- AOF (Append-Only File) Logging

### Replikation
Redis unterstützt Master-Slave-Replikation für Datenredundanz und Lastausgleich.

## Anwendungsfälle

1. **Caching**: Häufig zugegriffene Daten im Speicher speichern
2. **Sitzungsverwaltung**: Benutzersitzungen in Webanwendungen verwalten
3. **Echtzeit-Analysen**: Hochgeschwindigkeits-Datenströme verarbeiten
4. **Nachrichtenwarteschlangen**: Pub/Sub-Muster implementieren
5. **Ratenbegrenzung**: API-Anfrageraten kontrollieren

## Vorteile

- **Leistung**: Sub-Millisekunden-Latenz für Lese-/Schreiboperationen
- **Flexibilität**: Unterstützung verschiedener Datenstrukturen
- **Skalierbarkeit**: Horizontale Skalierung durch Clustering
- **Große Community**: Umfangreiche Dokumentation und aktive Unterstützung
