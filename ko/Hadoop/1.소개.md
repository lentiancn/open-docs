# Hadoop 소개

## Hadoop이란?

Apache Hadoop은 대규모 데이터 세 처리를 위해 설계된 Apache Software Foundation이 개발한 오픈소스 분산 컴퓨팅 프레임워크입니다. Hadoop은 일반 하드웨어로 구성된 클러스터에서 대량 데이터를 처리할 수 있는 신뢰할 수 있고 확장 가능한 분산 데이터 저장 및 컴퓨팅 솔루션을 제공합니다. Hadoop의 핵심 설계 철학은 계산 노드로 많은 양의 데이터를 이동시키는 것이 아니라 데이터를 있는 위치로 계산을 이동시키는 것입니다. 이를 통해 네트워크 전송 오버헤드를 크게 줄이고 처리 효율을 향상시킬 수 있습니다.

Hadoop 프로젝트의 기원은 2003년으로 거슬러 올라갑니다. 그해 Google은 Google File System(GFS)에 관한 논문을 발표했고, 2004년에는 MapReduce에 관한 논문을 발표했습니다. 이러한 논문에 영감을 받아 Yahoo!의 엔지니어 Doug Cutting과 Mike Cafarella가 오픈소스 구현을 개발하기 시작했으며, 이것이 Hadoop이 되었습니다. 2006년 Hadoop은 공식적으로 Apache Software Foundation의 프로젝트가 되었으며, 그 이름은 Doug Cutting의 아들이 가지고 있던 장난감 코끼리에서 유래했습니다.

Hadoop은 Java로 개발되어 있으며 우수한 플랫폼 간 기능을 갖추고 있습니다. 핵심 구성 요소에는 HDFS(Hadoop Distributed File System)와 YARN(Yet Another Resource Negotiator), 그리고 MapReduce 프로그래밍 모델이 포함됩니다. Hadoop의 설계 철학은 "모든 것이 확장 가능"입니다. 시스템은 단순히 더 많은 노드를 추가함으로써 스토리지 용량과 컴퓨팅 파워를 선형적으로 확장할 수 있습니다. 이 간단하지만 강력한 설계는 Hadoop을大数据 처리 분야의 선호 인프라스트럭처로 만들었습니다.

## 핵심 구성 요소

### HDFS - 분산 파일 시스템

HDFS는 Hadoop의 분산 파일 시스템으로, 매우 큰 파일 저장을 위해 설계되었습니다. HDFS는 하나의 NameNode와 여러 DataNode로 구성된 마스터-슬레이브 아키텍처를 사용합니다. NameNode는 파일 시스템 네임스페이스와 메타데이터를 관리하며 파일 시스템의 디렉터리 구조를 유지하고 각 파일의 블록 목록과 각 블록의 DataNode 위치를 기록합니다. DataNode는 실제 데이터 저장을 담당하며 저장된 블록 정보를 정기적으로 NameNode에 보고합니다.

HDFS는 데이터 복제 메커니즘을 통해 높은 내결함성을 갖추고 있습니다. 기본적으로 각 데이터 블록은 3개의 서로 다른 DataNode에 복제되어 노드가 실패해도 다른 복제본에서 데이터를 복구할 수 있습니다. HDFS는 큰 파일의 순차 읽기에 최적화되어 배치 처리 시나리오에 적합하지만 저지연 무작위 읽기/쓰기 작업에는 적합하지 않습니다. HDFS는 또한 랙 인식을 지원하여 다른 랙의 노드에 블록 복제본을 지능적으로 분산하여 데이터 가용성과 시스템 내결함성을 더욱 향상시킵니다.

HDFS 아키텍처는 "한 번 쓰기, 여러 번 읽기"(WORM) 원칙을 따릅니다. 즉, 파일은 쓰여진 후 수정되어서는 안 됩니다. 이 설계는 일관성 모델을 단순화하고 읽기 성능을 향상시키며 데이터 복제와 분산을 더 간단하게 만듭니다. HDFS는 또한 스냅샷 기능을 제공하여 데이터 백업 및 복구를 위한 파일 시스템의 읽기 전용 시점 복사본을 생성할 수 있습니다.

### YARN - 리소스 매니저

YARN은 Hadoop 2.x에서 도입된 리소스 관리 플랫폼으로, 클러스터 리소스의 통합 관리 및 스케줄링을 담당합니다. YARN은 리소스 관리와 애플리케이션 실행을 분리하여 Hadoop이 MapReduce 배치 작업뿐만 아니라 더 많은 유형의 워크로드를 지원할 수 있게 합니다. YARN의 핵심 구성 요소에는 ResourceManager, NodeManager, ApplicationMaster가 있습니다.

ResourceManager는 YARN의 핵심 구성 요소로, 전체 클러스터의 리소스 할당 및 관리를 담당합니다. 애플리케이션의 리소스 요청을 수신하고 클러스터의 사용 가능한 리소스에 따라 스케줄링합니다. ResourceManager는 스케줄러(Scheduler)와 애플리케이션 매니저(ApplicationsManager)로 구성됩니다. 스케줄러는 사전 정의된 정책에 따라 리소스를 할당하고, 애플리케이션 매니저는 애플리케이션 제출 요청을 수락하고 애플리케이션에 대한 ApplicationMaster를 시작합니다.

NodeManager는 각 워커 노드의 에이전트로, 해당 노드의 CPU, 메모리, 디스크, 네트워크 리소스를 포함한 리소스 관리를 담당합니다. NodeManager는 정기적으로 ResourceManager에 리소스 사용情况和 컨테이너 상태를 보고하고 ResourceManager의 지시를 실행합니다. ApplicationMaster는 각 애플리케이션의 관리자로, ResourceManager와 리소스를 협상하고 NodeManager와 협력하여 작업을 실행합니다.

### MapReduce - 프로그래밍 모델

MapReduce는 대규모 데이터의 병렬 처리를 위한 Hadoop의 핵심 컴퓨팅 모델입니다. MapReduce 프로그래밍 모델은 데이터 처리를 Map 단계와 Reduce 단계의 두 가지 주요 단계로 나눕니다. Map 단계에서 시스템은 입력 데이터를 독립적인 청크로 분할하고 각 청크를 병렬로 처리하여 키-값 쌍 형태의 중간 결과를 출력합니다. Reduce 단계에서 시스템은 동일한 키를 가진 중간 결과를 집계和处理하여 최종 계산 결과를 생성합니다.

MapReduce 프레임워크는 데이터 분배, 작업 스케줄링, 부하 분산, 결함 처리 등의 복잡한 세부 사항을 자동으로 처리하여 개발자가 비즈니스 로직 구현에 집중할 수 있게 합니다. 이 추상은 분산 프로그램 개발을 크게 단순화합니다. 개발자는 Map 함수와 Reduce 함수만 작성하면 되고, 프레임워크가 분산 실행의 모든 복잡성을 처리합니다. MapReduce는 또한 데이터 흐름 최적화와 네트워크 전송 감소를 위한 사용자 정의 파티셔너(Partitioner)와 콤바이너(Combiner)를 지원합니다.

MapReduce 작업의 실행 프로세스에는 입력 분할, Map 작업 실행, 중간 결과 셔플, Reduce 작업 실행, 결과 출력 등의 단계가 포함됩니다. 셔플 단계는 MapReduce의 핵심 프로세스로, Map 작업 출력을 키로 파티셔닝, 정렬, 병합한 다음 해당 Reduce 작업으로 전송합니다. 이 프로세스는 컴퓨팅 및 네트워크 집약적이며 작업 전체 성능에 상당한 영향을 미칩니다.

## 기술 아키텍처

### NameNode 및 Secondary NameNode

NameNode는 HDFS의 핵심 구성 요소로, 전체 파일 시스템의 메타데이터를 유지합니다. 파일 이름, 파일 속성, 파일과 데이터 블록 매핑, 블록과 DataNode 매핑을 포함한 파일 시스템의 전체 메타데이터를 메모리에 저장합니다. 메타데이터의 지속성을 보장하기 위해 NameNode는 로컬 파일 시스템의 트랜잭션 로그(EditLog)에도 메타데이터를 씁니다. NameNode가 시작되면 이미지 파일(FsImage)을 읽고 트랜잭션 로그를 재생하여 파일 시스템 상태를 복구합니다.

Secondary NameNode는 NameNode의 백업이 아닙니다. 그 주요 역할은 FsImage와 EditLog를 정기적으로 병합하여 NameNode의 시작 시간과 EditLog 파일 크기를 줄이는 것입니다. Secondary NameNode는 일반적으로 별도의 컴퓨터에서 실행되며 NameNode에서 정기적으로 FsImage와 EditLog를 다운로드하고, 로컬에서 병합 작업을 수행한 다음 새 FsImage를 NameNode에 다시 업로드합니다. 이 프로세스는 체크포인트라고 하며 NameNode의 메모리 압력과 시작 시간을 효과적으로 줄일 수 있습니다.

### ResourceManager 아키텍처

ResourceManager는 이중 레벨 스케줄러 아키텍처를 사용합니다. 첫 번째 레벨은 용량 스케줄러(Capacity Scheduler)와 공정 스케줄러(Fair Scheduler)로, 서로 다른 대기열과 애플리케이션 간 클러스터 리소스 분배를 담당합니다. 두 번째 레벨은 ApplicationMaster로, 각 애플리케이션에는 ResourceManager와 리소스를 협상하고 애플리케이션 실행을 관리하는 자체 ApplicationMaster가 있습니다.

용량 스케줄러는 여러 조직이 클러스터 리소스를 공유할 수 있도록 하여 각 조직이 자체 대기열을 가지며 각 조직이 특정 리소스 할당량을 얻을 것을 보장합니다. 공정 스케줄러는 공정성을 강조하여 실행 중인 모든 애플리케이션 간에 리소스 분배를 자동으로 균형 있게 하여 각 애플리케이션이 상대적으로 공정한 리소스를 얻을 수 있도록 합니다. 두 스케줄러 모두 구성 파일을 통해 유연하게 구성하여 다양한 시나리오 요구 사항을 충족할 수 있습니다.

### 데이터 지역성

데이터 지역성은 네트워크 데이터 전송을 줄이는 것을 목표로 하는 Hadoop의 중요한 최적화 전략입니다. Hadoop은 해당 데이터가 저장된 노드에 계산 작업을 할당하려고 합니다. 이것이 "데이터로 계산 이동"이라는 철학입니다. 데이터 지역성에는 세 가지 레벨이 있습니다: 노드 로컬(계산과 데이터가同一 노드上), 랙 로컬(계산과 데이터가同一 랙上), 랙 간(계산과 데이터가 다른 랙上).

Hadoop은 작업 스케줄링 시 노드 로컬 데이터를 우선 고려하며 해당 노드를 사용할 수 없는 경우 랙 로컬 데이터로 대체됩니다. 이 최적화를 실현하기 위해 NameNode는 각 블록의 위치 정보를 ResourceManager에 보고하며 ResourceManager는 최적의 스케줄링 결정을 내리기 위해 이 정보를 사용합니다. 데이터 지역성은 대규모 데이터 처리에 매우 중요하며 클러스터 전체 성능을 크게 향상시킬 수 있습니다.

## 사용 시나리오

Hadoop은大数据 처리 분야에 널리 사용됩니다. 데이터 분석 분야에서 Hadoop은 대량의 로그 데이터, 사용자 행동 데이터, 비즈니스 트랜잭션 데이터를 처리하고 분석하여 기업이 방대한 데이터 세트에서 가치 있는 정보를 추출하는 데 도움을 줍니다. 데이터웨어하우스 분야에서 Hadoop은 전통적인 데이터웨어하우스를 보완하여 저비용의 스토리지 및 분석 기능으로 구조화 데이터와 비구조화 데이터를 처리할 수 있습니다.

기계 학습 분야에서 Hadoop은 대규모 기능 엔지니어링 및 모델 교육의 인프라스트럭처로 작용하여 분산 기계 학습 알고리즘을 지원합니다. 데이터 레이크 아키텍처에서 HDFS는 다양한 소스의 원시 데이터를 저장하기 위한 저렴하고 신뢰할 수 있는 데이터 스토리지 레이어를 제공합니다. 실시간 처리 측면에서 Hadoop 자체는 실시간 컴퓨팅에 적합하지 않지만 Apache Spark, Apache Flink 등의 스트림 처리 프레임워크와 결합하여 거의 실시간 데이터 처리 및 분석을 달성할 수 있습니다.

Hadoop은 또한 많은 다른大数据 기술의 기반이기도 합니다. Apache Spark, Apache Hive, Apache Pig 등 인기 있는 분산 컴퓨팅 프레임워크의 대부분은 Hadoop 위에 구축되거나 Hadoop과 깊이 통합되어 있습니다. Hadoop의 원리와 실습을 마스터하는 것은 다른大数据 기술を理解하고 사용하는 데 매우 중요합니다.

## 발전 역사

2006년 탄생 이후 Hadoop은 여러 주요 버전的反復을 거쳤습니다. Hadoop 1.0 시대에는 MapReduce가 리소스 스케줄링과 작업 실행을 모두 담당하여 아키텍처는 단순하지만 확장성에 한계가 있었습니다. Hadoop 2.0은 YARN을 도입하여 리소스 관리와 작업 실행을 분리하여 시스템의 확장성과 유연성을 크게 향상시켰으며 더 많은 유형의 워크로드를 지원했습니다.

Hadoop 3.0은 여러 NameNode 지원(HA Federation), 컨테이너화된 실행 환경, GPU 가속을 포함한 중요한 개선을 가져왔습니다. Hadoop 3.x는 또한 스토리지 효율을 최적화하여 데이터 신뢰성을 보장하면서 스토리지 오버헤드를 크게 줄일 수 있는 이레이저 코딩 기반 데이터 보호 메커니즘을 제공합니다. 현재 Hadoop 생태계는 지속적으로 발전하고 개선되고 있으며 새로운 기능과 최적화가 계속 도입되고 있습니다.

Hadoop의 발전은 크고 활발한 생태계도 탄생시켰습니다. Apache Hive는 SQL 친화적 사용자가 쉽게大数据를 처리할 수 있는 SQL 유사 쿼리 언어를 제공합니다. Apache Pig는 복잡한 데이터 처리 작업 개발을 단순화하는 고수준 데이터 흐름 언어를 제공합니다. Apache HBase는 무작위 읽기/쓰기를 지원하는 열 지향 분산 데이터베이스를 제공합니다. Apache Sqoop은 관계형 데이터베이스와 HDFS 간의 데이터 마이그레이션을 단순화합니다. 이러한 프로젝트들은 함께 완전한大数据 처리 솔루션을 구성합니다.
