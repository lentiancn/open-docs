# Guía de Instalación de Kubernetes

## Requisitos Previos

Antes de instalar Kubernetes, comprenda los requisitos básicos:

### Requisitos del sistema
- Linux es el sistema operativo preferido para ejecutar el plano de control de Kubernetes
- Dentro de su clúster puede ejecutar aplicaciones en Linux u otros sistemas operativos, incluyendo Windows
- Se requiere un sistema operativo que admita el runtime de contenedores

### Selección del tipo de instalación
Al instalar Kubernetes, elija un tipo de instalación basado en:
- Facilidad de mantenimiento
- Seguridad
- Control
- Recursos disponibles
- Experiencia requerida para operar y administrar un clúster

## Métodos de Instalación

### Entorno de aprendizaje (Recomendado para principiantes)

Si está aprendiendo Kubernetes, use herramientas soportadas por la comunidad o herramientas del ecosistema para configurar un clúster en su máquina local:

#### Minikube
Minikube es una implementación ligera de Kubernetes que crea un clúster de un solo nodo en su máquina local. Ideal para aprendizaje y pruebas a pequeña escala.

Pasos de instalación:
1. Instalar Minikube
2. Instalar kubectl (herramienta de línea de comandos de Kubernetes)
3. Iniciar el clúster Minikube

#### Kind
Kind (Kubernetes in Docker) es otra herramienta para crear clústeres Kubernetes locales, usando contenedores Docker para simular nodos.

### Entorno de producción

Para entornos de producción, considere estas opciones:

#### kubeadm
La herramienta oficialmente soportada para implementar Kubernetes. Crea clústeres seguros siguiendo las mejores prácticas.

Pasos básicos:
1. Preparar servidores (instalar SO, configurar red)
2. Instalar runtime de contenedores (containerd o Docker)
3. Instalar kubeadm, kubelet y kubectl
4. Inicializar clúster con kubeadm
5. Configurar plugin de red (Calico, Cilium, etc.)
6. Unir nodos workers

#### Servicios gestionados
Si no desea administrar un clúster de Kubernetes, considere servicios gestionados:
- Amazon EKS (Elastic Kubernetes Service)
- Google GKE (Google Kubernetes Engine)
- Azure AKS (Azure Kubernetes Service)
- Alibaba Cloud ACK
- Tencent Cloud TKE

#### Otras soluciones
También existen soluciones estandarizadas y personalizadas en una amplia gama de entornos de nube y metal desnudo.

## Instalación de Herramientas

### Instalar kubectl

kubectl es la herramienta de línea de comandos para interactuar con clústeres Kubernetes.

Instalación en Linux:
```bash
# Descargar última versión
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# Agregar permisos de ejecución
chmod +x kubectl

# Mover al path del sistema
sudo mv kubectl /usr/local/bin/
```

### Instalar kubeadm

```bash
# Actualizar índice de paquetes apt
sudo apt-get update

# Instalar dependencias
sudo apt-get install -y apt-transport-https ca-certificates curl

# Agregar repositorio APT de Kubernetes
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Actualizar e instalar
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## Inicialización del Clúster

### Crear un Clúster con kubeadm

1. Inicializar el nodo del plano de control:
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. Configurar kubectl:
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. Instalar plugin de red (Flannel como ejemplo):
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. Unir nodos workers:
```bash
# Ejecutar en cada nodo worker el comando de la salida de init
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## Verificar Instalación

Verificar estado del clúster:
```bash
# Ver nodos
kubectl get nodes

# Ver Pods
kubectl get pods --all-namespaces
```

Si todos los componentes están en estado Running, la instalación fue exitosa.

## Desinstalación

Si necesita desinstalar Kubernetes:

```bash
# Detener todos los servicios de Kubernetes
sudo kubeadm reset

# Eliminar archivos de configuración
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# Desinstalar paquetes
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## Notas Importantes

1. Selección de runtime de contenedores: Elija un runtime apropiado para su nuevo clúster
2. Planificación de recursos: Asigne recursos de CPU y memoria razonablemente según las cargas de trabajo
3. Planificación de red: Planifique los CIDR de red de Pod y Service con anticipación
4. Endurecimiento de seguridad: Configure autenticación, autorización y políticas de red para producción
5. Alta disponibilidad: Configure clústeres de múltiples planos de control para entornos de producción
