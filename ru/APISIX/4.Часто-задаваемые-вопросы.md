# Часто задаваемые вопросы

Здесь собраны типичные вопросы и ответы по использованию Apache APISIX.

## Базовые вопросы

### Что такое Apache APISIX?

Apache APISIX — это проект верхнего уровня Apache Software Foundation, облачный API-шлюз с динамическим, реальновременным и высокопроизводительным ядром. Он включает динамическую маршрутизацию, динамические upstream-серверы, динамические сертификаты, A/B-тестирование, canary-релизы, ограничение скорости, защиту от атак, мониторинг с оповещениями и многое другое.

### Чем APISIX отличается от Nginx?

Nginx — это традиционный веб-сервер и обратный прокси, а APISIX — это API-шлюз, построенный на базе Nginx. APISIX добавляет к Nginx полностью динамическое управление конфигурацией, богатую систему плагинов, тонкую настройку трафика и встроенную наблюдаемость.

### Какая производительность у APISIX?

APISIX унаследовал высокую производительность Nginx и при этом оптимизирован для эффективной работы благодаря плагинной архитектуре. Конкретные цифры зависят от железа и сложности логики, но в типичных сценариях APISIX обрабатывает десятки и сотни тысяч запросов в секунду.

## Вопросы по установке

### Docker не запускается — что делать?

1. Проверьте, работает ли Docker: `docker ps`
2. Посмотрите логи контейнера: `docker-compose logs apisix`
3. Частые проблемы:
   - Конфликт портов: проверьте, заняты ли порты 9080, 9180, 9443
   - etcd не подключается: убедитесь, что контейнер etcd запущен
   - Проблемы с правами: проверьте, достаточно ли у вас прав для запуска Docker

### Как узнать версию APISIX?

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Ответ: `Server: APISIX/Version`

### После установки не открывается Dashboard?

1. Проверьте, правильно ли проброшен порт Dashboard (по умолчанию 9180)
2. Проверьте настройки файрвола
3. Убедитесь, что сетевые настройки контейнеров корректны

## Вопросы по настройке

### Как изменить порт управляющего API?

Отредактируйте конфигурационный файл `apisix.yml`:

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### Как включить HTTPS?

1. Настройте SSL-сертификат:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "содержимое сертификата",
  "key": "содержание приватного ключа",
  "snis": ["example.com"]
}'
```

2. Включите HTTPS на маршруте:

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### Настройки маршрута не применяются?

1. Проверьте корректность конфигурации маршрута
2. Убедитесь, что конфигурация etcd синхронизирована
3. Проверьте плагины на синтаксические ошибки
4. Посмотрите логи ошибок APISIX

### Как разрешить CORS?

Используйте плагин `cors`:

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## Вопросы по производительности

### Как увеличить пропускную способность?

1. **Увеличить число рабочих процессов:** `apisix.worker_processes: auto`
2. **Включить HTTP/2:** в настройках listen указать `http2: true`
3. **Оптимизировать пул соединений к upstream:** настроить `keepalive_pool.size`
4. **Использовать кэширование:** включить плагин `proxy-cache`

### Слишком много памяти — что делать?

1. Отключите ненужные плагины
2. Ограничьте число соединений к upstream
3. Уменьшите размер буфера логов
4. Используйте потоковую обработку для больших файлов

## Вопросы по функционалу

### Как сделать A/B-тестирование?

Используйте плагин `traffic-split`:

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### Как ограничить частоту запросов?

Используйте плагин `limit-req`:

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## Вопросы по интеграции

### Как интегрировать с Kubernetes?

Используйте APISIX Ingress Controller:

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### Как интегрировать с Prometheus?

1. Включите плагин Prometheus
2. Настройте Prometheus на сбор метрик

## Вопросы безопасности

### Как защитить APISIX от злоупотреблений?

1. **Включите аутентификацию:** на всех маршрутах используйте key-auth или другие плагины
2. **Ограничьте доступ к управляющему API:** разрешите только доверенным IP
3. **Используйте HTTPS:** включите TLS-шифрование
4. **Настройте ограничение скорости:** защита от вредоносных запросов
5. **Ведите логи доступа:** записывайте все запросы для аудита

### Как включить авторизацию для управляющего API?

```yaml
deployment:
  admin:
    admin_key_required: true
```

## Вопросы по эксплуатации

### Как обновить APISIX?

**Docker:**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes:**

```bash
helm upgrade apisix apisix/apisix
```

### Коды ошибок

- **401 Unauthorized** — ошибка аутентификации
- **403 Forbidden** — нет доступа
- **404 Not Found** — маршрут не существует
- **429 Too Many Requests** — превышен лимит запросов
- **500 Internal Server Error** — ошибка upstream-сервиса
- **502 Bad Gateway** — upstream недоступен
- **503 Service Unavailable** — сервис недоступен
