# OpenClaw Benutzerhandbuch

Dieses Handbuch behandelt die tägliche Nutzung und Hauptfunktionen von OpenClaw.

## Grundkonzepte

### Gateway

Das Gateway ist die Kernkomponente von OpenClaw und verantwortlich für Session-Management, Nachrichten-Routing und Kanalverbindungen. Du musst nur einen Gateway-Prozess ausführen, und er automatisch alle Kanalverbindungsanfragen.

### Kanäle

Kanäle sind die Chat-Anwendungen, die du nutzt, um mit deinem KI-Assistenten zu kommunizieren. OpenClaw unterstützt das gleichzeitige Verbinden mehrerer Kanäle, einschließlich WhatsApp, Telegram, Discord, iMessage und mehr.

### Agenten

Agenten sind Instanzen von KI-Assistenten. Jeder Agent hat seinen eigenen Workspace und Sessions. Du kannst mehrere Agenten konfigurieren, um unterschiedliche Aufgaben unabhängig zu bearbeiten.

## Tägliche Operationen

### Gateway starten und stoppen

```bash
# Gateway starten (Hintergrunddienst)
openclaw gateway start

# Gateway stoppen
openclaw gateway stop

# Gateway neu starten
openclaw gateway restart

# Status prüfen
openclaw gateway status

# Im Vordergrund ausführen (zum Debuggen)
openclaw gateway --port 18789
```

### Nachrichten senden

```bash
# Direktnachricht senden
openclaw message send --target +15555550123 --message "Hallo"

# An Gruppe senden
openclaw message send --target "Gruppenname" --message "Hallo zusammen"
```

### Konfiguration verwalten

```bash
# Aktuelle Konfiguration anzeigen
openclaw config get

# Konfigurationswert setzen
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# Konfigurationswert entfernen
openclaw config unset channels.whatsapp.enabled
```

## Konfigurationsmanagement

### Speicherort der Konfigurationsdatei

Die Konfigurationsdatei befindet sich unter `~/.openclaw/openclaw.json`.

### Heiße Neu laden

OpenClaw unterstützt heiße Neuladung der Konfiguration – du kannst die Konfigurationsdatei ändern, ohne das Gateway manuell neu zu starten. Die meisten Konfigurationsänderungen werden automatisch wirksam.

### Häufige Konfigurationsbeispiele

#### Erlaubte Nachrichtenquellen festlegen

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### Gruppenchat-Mentions-Regeln festlegen

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### Mehrere Agenten konfigurieren

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## Kanalverwaltung

### Neue Kanäle hinzufügen

```bash
# Bei WhatsApp anmelden
openclaw channels login whatsapp

# Bei Telegram anmelden
openclaw channels login telegram
```

### Verbundene Kanäle anzeigen

```bash
openclaw channels list
```

## Erweiterte Funktionen

### Cron-Jobs

Geplante Aufgaben konfigurieren:

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Heartbeat

Periodische Check-ins einrichten:

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Sandbox-Modus

Agenten in isolierten Docker-Containern ausführen:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

HTTP-Webhook-Endpunkte einrichten:

```json5
{
  hooks: {
    enabled: true,
    token: "dein-geheimer-token",
    path: "/hooks"
  }
}
```

## Fehlerbehebung

### Logs anzeigen

```bash
# Gateway-Logs anzeigen
openclaw logs

# Logs in Echtzeit verfolgen
openclaw logs -f
```

### Gesundheitsprüfung

```bash
openclaw health
```

### Probleme diagnostizieren

```bash
openclaw doctor
```

## Control UI

OpenClaw bietet ein webbasiertes Control UI, wo du:

- Nachrichten senden und empfangen
- Konfiguration verwalten
- Session-Verlauf anzeigen
- Mobile Knoten verwalten
- Gateway-Status überwachen

Zugang unter: `http://127.0.0.1:18789/`
