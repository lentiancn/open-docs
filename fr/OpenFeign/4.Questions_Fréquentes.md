# OpenFeign Questions Fréquentes

## Questions liées à l'installation

### Q: Comment vérifier qu'OpenFeign est correctement installé ?
**R:** Après avoir ajouté les dépendances, vérifiez en créant une application de base :

```java
@SpringBootApplication
@EnableFeignClients
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

Assurez-vous que votre application Spring Boot démarre correctement et que les clients Feign peuvent être injectés correctement.

### Q: Comment gérer les conflits de version des dépendances ?
**R:** Utilisez Spring Cloud BOM pour la gestion des versions :

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>2022.0.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Cela empêche efficacement les conflits de version.

### Q: Pourquoi dois-je ajouter explicitement spring-cloud-starter-loadbalancer ?
**R:** À partir de Spring Cloud 2020.0.0, Ribbon a été supprimé et Spring Cloud LoadBalancer est devenu l'équilibreur de charge par défaut. `spring-cloud-starter-openfeign` l'inclut comme dépendance optionnelle, vous devez donc l'ajouter explicitement :

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

## Questions liées à la configuration

### Q: Comment configurer les délais d'attente ?
**R:** Vous pouvez configurer les délais d'attente via le fichier de configuration ou la configuration Java :

**Approche du fichier de configuration :**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 5000
            read-timeout: 5000
          user-service:
            connect-timeout: 3000
            read-timeout: 10000
```

**Approche de configuration Java :**
```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Request.Options requestOptions() {
        return new Request.Options(3000, 10000);
    }
}
```

### Q: Comment activer la compression GZIP ?
**R:** Activez la compression dans le fichier de configuration :

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

Remarque : Lors de l'utilisation d'OkHttpClient, la compression est désactivée si les en-têtes `content-encoding` ou `accept-encoding` sont présents.

### Q: Comment configurer les niveaux de journalisation ?
**R:** Deux approches de configuration :

**Fichier de configuration :**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
            
logging:
  level:
    com.example.clients.UserService: DEBUG
```

**Configuration Java :**
```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Niveaux de journalisation pris en charge :
- **NONE**: Aucune journalisation (par défaut)
- **BASIC**: Journalise uniquement la méthode de requête, l'URL, le code de statut de réponse et le temps d'exécution
- **HEADERS**: Journalise les informations de base ainsi que les en-têtes de requête et de réponse
- **FULL**: Journalise les en-têtes, le corps et les métadonnées pour les requêtes et les réponses

## Questions liées à l'utilisation

### Q: L'interface FeignClient ne peut pas être injectée, que dois-je faire ?
**R:** Causes courantes et solutions :

**1. Problème de chemin de package de scan :**
```java
// Assurez-vous que FeignClient est dans le chemin de package scanné
@SpringBootApplication
@EnableFeignClients(basePackages = "com.example.clients")
public class Application {
    // ...
}
```

**2. Problème de dépendance circulaire :**
Si FeignClient est référencé par plusieurs Services, une dépendance circulaire peut se produire. Solutions :
- Utilisez l'annotation `@Lazy` pour une injection différée
- Refactorisez la structure du code pour éviter les dépendances circulaires

**3. Conflits de noms de Bean :**
Si vous avez plusieurs FeignClients avec le même nom, utilisez `contextId` pour les différencier :
```java
@FeignClient(contextId = "userClient1", name = "user-service")
public interface UserClient1 { }

@FeignClient(contextId = "userClient2", name = "user-service")  
public interface UserClient2 { }
```

### Q: Comment gérer les paramètres de requête complexes ?
**R:** Plusieurs approches pour les paramètres complexes :

**1. Utilisation de @SpringQueryMap :**
```java
public class SearchCriteria {
    private String name;
    private String status;
    private Integer page;
}

@FeignClient(name = "search-service")
public interface SearchService {
    @GetMapping("/search")
    SearchResult search(@SpringQueryMap SearchCriteria criteria);
}
```

**2. Utilisation de Map :**
```java
@FeignClient(name = "api-service")
public interface ApiService {
    @GetMapping("/data")
    Data getData(@RequestParam Map<String, Object> params);
}
```

**3. QueryMapEncoder personnalisé :**
```java
@Bean
public QueryMapEncoder queryMapEncoder() {
    return new CustomQueryMapEncoder();
}
```

### Q: Comment gérer les téléchargements de fichiers ?
**R:** Le téléchargement de fichiers nécessite une configuration spéciale :

```java
@FeignClient(name = "file-service")
public interface FileService {
    
    @PostMapping(value = "/upload", 
                consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    UploadResponse uploadFile(@RequestPart("file") MultipartFile file,
                             @RequestPart("metadata") FileMetadata metadata);
}
```

Configurez également l'encodeur :
```java
@Bean
public Encoder feignFormEncoder() {
    return new FeignFormEncoder(new SpringEncoder(messageConverters()));
}
```

## Questions sur la gestion des erreurs

### Q: Comment personnaliser la gestion des erreurs ?
**R:** Implémentez l'interface ErrorDecoder :

```java
@Component
public class CustomErrorDecoder implements ErrorDecoder {
    
    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() == 404) {
            return new ResourceNotFoundException("Resource not found");
        } else if (response.status() >= 500) {
            return new ServerException("Server error");
        }
        return new Default().decode(methodKey, response);
    }
}
```

### Q: Le disjoncteur ne fonctionne pas, qu'est-ce qui ne va pas ?
**R:** Vérifiez les configurations suivantes :

**1. Activer le disjoncteur :**
```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

**2. Ajouter la dépendance requise :**
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

**3. Configurer les règles du disjoncteur :**
```yaml
resilience4j:
  circuitbreaker:
    instances:
      UserServicegetUser:
        failure-rate-threshold: 50
        minimum-number-of-calls: 5
        wait-duration-in-open-state: 5000
```

### Q: Le fallback ne se déclenche pas, que dois-je vérifier ?
**R:** Problèmes courants :

**1. Assurez-vous que la classe de fallback est un Bean Spring :**
```java
@Component  // Cette annotation est requise
public class UserServiceFallback implements UserService {
    // ...
}
```

**2. Vérifier les types d'exception :**
Le disjoncteur ne déclenche le fallback que pour des exceptions spécifiques, assurez-vous que l'exception levée est du type attendu.

**3. Configurer la bonne fabrique de fallback :**
```java
@FeignClient(name = "service", fallbackFactory = MyFallbackFactory.class)
public interface MyService { }
```

## Questions liées aux performances

### Q: La performance du client Feign est lente, comment l'optimiser ?
**R:** Recommandations d'optimisation des performances :

**1. Configurer des délais d'attente appropriés :**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
```

**2. Activer la mise en commun des connexions :**
```yaml
spring:
  cloud:
    openfeign:
      httpclient:
        hc5:
          enabled: true
          max-connections: 200
          max-connections-per-route: 20
```

**3. Activer la compression :**
```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
        response:
          enabled: true
```

**4. Utiliser la mise en cache :**
```java
@FeignClient(name = "config-service")
public interface ConfigService {
    @GetMapping("/config/{key}")
    @Cacheable(cacheNames = "config-cache", key = "#key")
    ConfigValue getConfig(@PathVariable String key);
}
```

### Q: Comment résoudre les problèmes de fuite de mémoire ?
**R:** Les fuites de mémoire sont généralement causées par :

**1. Connexions non fermées :**
Assurez-vous que les clients HTTP gèrent correctement les pools de connexions.

**2. Cache non nettoyé :**
Nettoyez régulièrement le cache ou définissez des temps d'expiration appropriés.

**3. Surveiller l'utilisation de la mémoire :**
```java
@RestController
public class MetricsController {
    
    @GetMapping("/memory")
    public MemoryInfo getMemoryInfo() {
        Runtime runtime = Runtime.getRuntime();
        return new MemoryInfo(
            runtime.totalMemory(),
            runtime.freeMemory(),
            runtime.maxMemory()
        );
    }
}
```

## Questions liées à la sécurité

### Q: Comment ajouter des jetons d'authentification aux requêtes ?
**R:** Utilisez RequestInterceptor :

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor authRequestInterceptor() {
        return requestTemplate -> {
            String token = getToken(); // Logique pour obtenir le jeton
            requestTemplate.header("Authorization", "Bearer " + token);
        };
    }
    
    private String getToken() {
        // Implémenter la logique de récupération du jeton
        return "your-token";
    }
}
```

### Q: Comment gérer l'authentification OAuth2 ?
**R:** Activer la prise en charge OAuth2 :

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
        
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: client_credentials
            scope: read,write
```

### Q: Comment filtrer les informations sensibles ?
**R:** Filtrage de journalisation personnalisé :

```java
public class SecureLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        Map<String, Collection<String>> safeHeaders = new HashMap<>(request.headers());
        safeHeaders.remove("Authorization");
        safeHeaders.remove("X-API-Key");
        
        super.logRequest(configKey, logLevel, 
            Request.create(request.httpMethod(), request.url(), safeHeaders,
                request.body(), request.charset(), request.requestTemplate()));
    }
}
```

## Questions liées aux tests

### Q: Comment tester les clients Feign ?
**R:** Il est recommandé d'utiliser WireMock :

```java
@SpringBootTest
@AutoConfigureWireMock(port = 0)
class FeignClientTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    void shouldGetUserSuccessfully() {
        // Given
        stubFor(get(urlEqualTo("/users/1"))
            .willReturn(aResponse()
                .withStatus(200)
                .withHeader("Content-Type", "application/json")
                .withBody("{\"id\":1,\"name\":\"John\"}")));
        
        // When
        User user = userService.getUser(1L);
        
        // Then
        assertThat(user.getName()).isEqualTo("John");
    }
}
```

### Q: Comment simuler des scénarios d'erreur ?
**R:** Utilisez WireMock pour simuler diverses erreurs :

```java
@Test
void shouldHandleServerError() {
    // Given
    stubFor(get(urlEqualTo("/users/999"))
        .willReturn(aResponse().withStatus(500)));
    
    // When & Then
    assertThatThrownBy(() -> userService.getUser(999L))
        .isInstanceOf(ServerException.class);
}
```

## Questions liées au déploiement

### Q: Comment configurer pour l'environnement de production ?
**R:** Configuration recommandée pour l'environnement de production :

```yaml
# application-prod.yml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
            logger-level: basic
      compression:
        request:
          enabled: true
        response:
          enabled: true
      circuitbreaker:
        enabled: true
      cache:
        enabled: true
        
logging:
  level:
    com.example.clients: WARN
```

### Q: Comment surveiller les clients Feign ?
**R:** Intégration avec Micrometer :

```yaml
spring:
  cloud:
    openfeign:
      micrometer:
        enabled: true
```

Ensuite, visualisez les métriques via les points de terminaison Actuator :
```bash
GET /actuator/metrics
GET /actuator/metrics/feign.client.requests
```

En comprenant et en appliquant ces solutions, vous serez en mesure de développer, déboguer et maintenir plus efficacement les applications basées sur OpenFeign.
