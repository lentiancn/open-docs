# Installation Guide

Jira offers multiple installation options to meet the needs of teams with different sizes and technical capabilities. This guide provides detailed steps for both cloud and Data Center on-premises deployment.

## Cloud Version Installation

The cloud version is the fastest way to get started with Jira, requiring no infrastructure management.

### Account Registration

First, visit the Jira website and click the free signup button. Enter your work email address to create an account. Atlassian will automatically identify team members and facilitate collaboration.

### Plan Selection

The cloud version offers four pricing tiers:

**Free plan** is permanently free for up to 10 users. It includes unlimited goals and projects, to-do lists and board views, 100 automation rules per month, and 2GB storage.

**Standard plan** costs $7.91 per user per month. It includes user roles and permissions, external collaboration, multi-region data residency, 1,700 automation rules per month, and 250GB storage.

**Premium plan** costs $14.54 per user per month. It includes cross-team planning and dependency management, customizable approval processes, unlimited storage, 24/7 support, and 99.9% uptime SLA.

**Enterprise plan** is designed for large enterprises. It includes advanced analytics and enterprise-grade security, multiple site support, unlimited automation rules, and 99.95% uptime SLA.

### Site Creation

After registration, the system guides you to create your first Jira site. You can create a site for your organization or personal use, and select project templates that suit your team's needs.

### User Addition

After creating a site, you can add team members through the admin panel. Users can be free or paid depending on your selected plan.

## Data Center On-Premises Deployment

Enterprises requiring strict data control or having special security requirements can choose the Data Center on-premises version.

### System Requirements

The Data Center version requires the following:

Operating system support includes Linux (Red Hat Enterprise Linux, Ubuntu Server, CentOS, etc.) and Windows Server.

Database support includes PostgreSQL, Oracle, Microsoft SQL Server, and other enterprise databases.

Memory: at least 8GB RAM recommended, 16GB or more for production environments.

Disk space: depends on data volume, at least 100GB available space recommended.

### Installation Steps

First, download the Data Center installer from the Atlassian website. Run the installer and follow the wizard to select the installation directory and configure database connections. After installation, access the admin interface through a browser for initial configuration.

### Data Migration

If migrating from an older version, use Jira's import and export tools to migrate data. It is recommended to back up all data before migration and verify migration results in a test environment.

## Mobile Apps

Jira offers iOS and Android mobile apps, allowing you to view and update issues anywhere. Mobile apps support issue viewing, comment creation, status updates, and more, suitable for team members who need to work on the go.

## Initial Configuration

Regardless of the deployment option, initial configuration includes:

Setting up project structure, creating projects and boards suitable for your team. Configuring workflows, defining issue statuses and transition rules. Adding team members and setting permissions. Integrating other tools like Slack and GitHub. Establishing automation rules to reduce manual operations.
