# 常见问题

本文档汇总了 Angular 使用过程中常见的技术问题和解答。

## 基础问题

### 什么是 Angular？

Angular 是由 Google 开发维护的开源 Web 应用框架，用于构建现代 Web 应用程序。它采用 TypeScript 作为主要开发语言，提供组件化架构、双向数据绑定、依赖注入等特性。

### Angular 和 AngularJS 有什么区别？

AngularJS 是 Angular 1.x 的名称，是 Angular 的前身。Angular 是完全重写的版本，使用 TypeScript，架构完全不同，不兼容 AngularJS。

### Angular 难学吗？

Angular 有一定的学习曲线，特别是依赖注入、RxJS 等概念需要时间理解。但它的文档完善，社区活跃，上手后开发效率很高。

## 安装问题

### npm install 失败怎么办？

1. 清除 npm 缓存：`npm cache clean --force`
2. 删除 node_modules 和 package-lock.json
3. 使用国内镜像
4. 检查 Node.js 版本是否兼容

### Angular CLI 命令找不到？

确保已全局安装：
```bash
npm install -g @angular/cli
```

如果还是不行，尝试使用 npx：
```bash
npx @angular/cli new my-app
```

### Node.js 版本不兼容？

使用 nvm 管理 Node.js 版本：
```bash
nvm install 18
nvm use 18
```

## 开发问题

### 组件样式不生效？

1. 检查样式文件路径是否正确
2. 确认 styles.scss 中没有语法错误
3. 检查 ViewEncapsulation 设置

### 路由跳转失败？

1. 检查路由配置是否正确
2. 确认 RouterModule 已正确导入
3. 检查是否有路由守卫阻止跳转

### HTTP 请求失败？

1. 检查后端服务是否启动
2. 确认 CORS 配置
3. 使用 HttpInterceptor 添加错误处理

### 数据不更新？

1. 检查变更检测策略
2. 如果使用 OnPush，需要手动触发检测
3. 确保数据引用发生变化

## 性能问题

### 首屏加载慢？

1. 启用路由懒加载
2. 使用预加载策略
3. 优化图片资源
4. 启用生产模式构建

### 内存泄漏？

1. 在 ngOnDestroy 中取消订阅 Observable
2. 使用 takeUntil 或 async 管道
3. 清理定时器和事件监听器

### 变更检测频繁？

1. 使用 OnPush 策略
2. 分离变化和不变化的属性
3. 使用 trackBy 优化 *ngFor

## 测试问题

### 单元测试失败？

1. 检查 TestBed 配置
2. 确认异步操作已正确处理
3. 使用 fakeAsync 和 tick 测试异步代码

### 找不到模块？

确保在 tsconfig.json 中正确配置了 paths。

## 构建问题

### 构建失败？

1. 查看具体错误信息
2. 检查 TypeScript 编译选项
3. 清除缓存重新构建

### 构建文件太大？

1. 启用 production 模式
2. 使用懒加载
3. 检查是否有未使用的依赖

## 常见错误

### ExpressionChangedAfterItHasBeenCheckedError

模板中修改了检测期间不应改变的值。检查 ngOnInit 中是否有直接修改视图相关的代码。

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

同上一个错误，检查数据绑定的时序。

### Cannot read property of undefined

数据未初始化就访问。使用可选链（?.）或 *ngIf 保护。

### CORS 错误

后端未配置 CORS，或前端请求头有问题。

### 401/403 错误

身份验证失败，检查 token 和权限配置。

## 版本升级

### 如何升级 Angular？

1. 使用 CLI 升级：`ng update @angular/core`
2. 查阅官方升级指南
3. 逐个升级依赖
4. 运行测试确保正常

### 升级失败怎么办？

1. 备份代码
2. 手动更新 package.json 中的版本
3. 删除 node_modules 重新安装
