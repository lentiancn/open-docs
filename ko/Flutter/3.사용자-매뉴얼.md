# Flutter 사용자 매뉴얼

## 기본 개념

### 위젯 소개

위젯은 Flutter UI의 기본 구성 블록입니다. Flutter에서는 레이아웃, 패딩, 정렬 등 모든 것이 위젯입니다. 위젯은 현재 구성과 상태가 주어졌을 때 어떻게 보여야 하는지를 설명합니다.

Flutter 위젯은 불변입니다. UI를 변경해야 하는 경우 위젯을 다시 만들거나 상태 관리가 있는 StatefulWidget을 사용해야 합니다. 위젯을 중첩하여 위젯 트리를 형성할 수 있으며, Flutter는 효율적으로 이전 위젯 트리와 새 위젯 트리를 비교하여 실제로 변경된 부분만 렌더링합니다.

일반적인 위젯에는 Text(텍스트), Image(이미지), Icon(아이콘), Container, Row/Column(가로/세로 레이아웃), Stack(스택 레이아웃), ListView(목록) 등이 포함됩니다.

### StatelessWidget과 StatefulWidget

StatelessWidget은 상태가 없는 위젯으로 텍스트 및 이미지 같은 정적 UI 표시에 적합합니다. StatelessWidget을 정의하려면 렌더링할 위젯 트리를 반환하는 build 메서드를 구현하기만 하면 됩니다.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget은 카운터 및 양식처럼 동적 UI 업데이트가 필요한 시나리오에 적합한 상태가 있는 위젯입니다. StatefulWidget은 State와 함께 사용해야 하며, 상태가 변경될 때 setState 메서드를 호출하여 Flutter에 다시 렌더링하도록 알립니다.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## 레이아웃 기초

### 선형 레이아웃

Row와 Column은 가장 자주 사용되는 선형 레이아웃 위젯입니다. Row는 자식 위젯을 가로로 배치하고, Column은 세로로 배치합니다. MainAxisAlignment와 CrossAxisAlignment를 사용하여 정렬을 제어할 수 있습니다.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### 스택 레이아웃

Stack은 다른 위젯 위에 위젯을 배치하는 스택 레이아웃을 만드는 데 사용됩니다. Positioned는 Stack에서 자식 위젯을 위치시키는 데 사용됩니다.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### 목록 레이아웃

ListView는 가장 자주 사용되는 목록 위젯으로 긴 목록에 적합합니다. ListView.builder는 지연 로딩이 필요한 대규모 데이터 세트에 적합합니다. ListView.separated는 구분선을 추가할 수 있습니다.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## 상태 관리

### setState

setState는 로컬 상태에 적합한 가장 기본적인 상태 관리 방법입니다. StatefulWidget에서 setState를 호출하면 위젯 재구성이 트리거됩니다.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider는 Flutter에서 권장하는 상태 관리 솔루션 중 하나입니다. Provider를 통해 위젯 트리에서 상태를 공유하여 층층 전달을 피할 수 있습니다.

```dart
// ChangeNotifier 정의
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Provider 사용
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## 네트워크 요청

### HTTP 패키지

http 패키지를 사용하여 간단한 HTTP 요청을 수행할 수 있습니다:

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

## 로컬 저장소

### SharedPreferences

간단한 키-값 데이터 저장에 사용됩니다:

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

## 네비게이션 및 라우팅

Flutter는 페이지 네비게이션에 Navigator를 사용합니다. 기본 작업에는 Navigator.push(새 페이지 푸시), Navigator.pop(뒤로), Navigator.pushReplacement(현재 페이지 교체) 등이 포함됩니다.

명명된 라우팅은 권장되는 라우팅 관리 방법입니다:

```dart
// 라우트 정의
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// 네비게이션
Navigator.pushNamed(context, '/detail');
```

## 빌드 및 게시

### Android 게시

서명 키를 생성하려면 keytool을 사용합니다:

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

android/app/build.gradle에서 서명을 구성한 다음 프로젝트 루트에서 다음을 실행합니다:

```bash
flutter build apk --release
```

### iOS 게시

Xcode에서 서명 인증서와 프로비저닝 프로파일을 구성한 다음 다음을 실행합니다:

```bash
flutter build ios --release
```

iOS 게시에는 해당 인증서가 있는 Apple 개발자 계정이 필요합니다.
