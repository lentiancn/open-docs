# Manual del usuario de OpenClaw

Este manual cubre las operaciones diarias y las funcionalidades de OpenClaw.

## Conceptos básicos

### Gateway

Gateway es el corazón de OpenClaw. Todo el enrutamiento de mensajes, las solicitudes de IA y la gestión de canales pasan por este único proceso. Solo necesitas ejecutar un Gateway y automáticamente maneja las conexiones de todas tus plataformas de chat.

### Canales

Los canales son las aplicaciones de mensajería que usas para comunicarte con tu asistente de IA. OpenClaw permite conectar múltiples canales simultáneamente, incluyendo WhatsApp, Telegram, Discord, iMessage y más.

### Agentes

Los agentes son instancias independientes de asistentes de IA. Cada agente tiene su propio espacio de trabajo e historial de conversación. Puedes ejecutar múltiples agentes para diferentes propósitos.

## Operaciones diarias

### Iniciar y detener

```bash
# Iniciar (se ejecuta en segundo plano)
openclaw gateway start

# Detener
openclaw gateway stop

# Reiniciar
openclaw gateway restart

# Ver estado
openclaw gateway status

# Ejecutar en primer plano (para depuración)
openclaw gateway --port 18789
```

### Enviar mensajes

```bash
# Enviar un mensaje directo
openclaw message send --target +15555550123 --message "Mensaje de prueba"

# Enviar a un grupo
openclaw message send --target "Nombre del Grupo" --message "Hola a todos"
```

### Gestionar configuración

OpenClaw usa archivos de configuración en formato JSON5, ubicados en `~/.openclaw/openclaw.json`.

Formas de modificar la configuración:

1. **Modo asistente**:
   ```bash
   openclaw onboard
   openclaw configure
   ```

2. **Línea de comandos**:
   ```bash
   openclaw config get agents.defaults.workspace
   openclaw config set agents.defaults.heartbeat.every "2h"
   openclaw config unset tools.web.search.apiKey
   ```

3. **Interfaz web**:
   Abre http://127.0.0.1:18789/, ve a la pestaña Config

4. **Edición directa**:
   ```bash
   nano ~/.openclaw/openclaw.json
   ```

### Recarga en caliente

La mayoría de los cambios de configuración se aplican automáticamente sin necesidad de reiniciar.

## Ejemplos de configuración común

### Restringir acceso

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

### Configuración de grupos

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

### Múltiples agentes

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

### Tareas programadas

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### Verificaciones periódicas

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### Aislamiento tipo sandbox

Para ejecutar operaciones de IA en contenedores aislados:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

```json5
{
  hooks: {
    enabled: true,
    token: "tu-token-secreto",
    path: "/hooks"
  }
}
```

## Gestión de canales

### Añadir canales

```bash
# Iniciar sesión en WhatsApp
openclaw channels login whatsapp

# Iniciar sesión en Telegram
openclaw channels login telegram
```

### Listar canales conectados

```bash
openclaw channels list
```

## Solución de problemas

### Ver registros

```bash
# Registros del servicio
openclaw logs

# Ver registros en tiempo real
openclaw logs -f
```

### Verificación de estado

```bash
openclaw health
```

### Diagnóstico automático

```bash
openclaw doctor
```

## Consola web

También puedes gestionar OpenClaw a través de la interfaz web:

- Enviar y recibir mensajes
- Modificar configuraciones
- Ver historial de conversaciones
- Gestionar nodos móviles
- Monitorear estado del servicio

Visita: http://127.0.0.1:18789/
