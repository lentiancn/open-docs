# Mockito Einführung

## Was ist Mockito

Mockito ist ein leistungsstarkes Java-Unit-Test-Mocking-Framework, das für Test-Driven Development (TDD) und Behavior-Driven Development (BDD) entwickelt wurde. Es ermöglicht Entwicklern, Mock-Objekte zu erstellen, die das Verhalten von echten Objekten auf kontrollierte Weise simulieren, unabhängige und zuverlässige Unit-Tests ermöglichen.

## Warum Mockito wählen

### Saubere und einfache API

Mockito bietet eine intuitive und benutzerfreundliche API mit einer flachen Lernkurve. Entwickler können schnell einsteigen und mit minimalem Code klare Testfaelle schreiben. Im Vergleich zu anderen komplexen Mocking-Frameworks ist die Mockito-Syntax natürlicher, und Testcode zu lesen ähnelt dem Lesen normalen Geschäftscode.

### Leistungsstarke Funktionen

Mockito unterstützt verschiedene erweiterte Funktionen einschließlich Methodenaufruf-Verifikation, Rückgabewert-Stubbing, Exception-Simulation, Argument-Matching und Aufrufreihenfolge-Verifikation. Diese Fähigkeiten erfüllen verschiedene komplexe Testszenarien und helfen Entwicklern, den Code umfassend abzudecken.

### Ausgezeichnete Lesbarkeit

Mit Mockito geschriebener Testcode hat eine ausgezeichnete Lesbarkeit. Wenn ein Test fehlschlägt, kann Mockito klare Verifikationsfehlermeldungen liefern, die Entwicklern helfen, das Problem schnell zu lokalisieren. Diese freundliche Fehlerpräsentation verbessert die Debugging-Effizienz erheblich.

### Aktive Community-Unterstützung

Mockito ist das beliebteste Mocking-Framework im Java-Ökosystem mit einer großen Benutzerbasis und einer aktiven Community. Das Framework wird kontinuierlich gewartet und aktualisiert, mit umfassender Dokumentation. Wenn Probleme auftreten, sind Lösungen leicht zu finden. Bei Stack Overflow hat Mockito die höchste Anzahl an Stimmen erhalten und gilt als das beste Mocking-Framework für Java.

## Kernkonzepte

### Mock-Objekte

Ein Mock-Objekt dient als Ersatz für ein echtes Objekt während des Tests. Durch das Erstellen von Mock-Objekten können externe Abhängigkeiten isoliert werden, wodurch sichergestellt wird, dass sich Tests nur auf die Logik der zu testenden Einheit konzentrieren, ohne von externen Systemen beeinflusst zu werden.

### Stubbing

Stubbing wird verwendet, um das Verhalten eines Mock-Objekts zu definieren, wenn spezifische Methoden aufgerufen werden. Mit der when().thenReturn()-Syntax können Sie die Antworten von Mock-Methoden genau steuern, sodass Tests verschiedene Szenarien abdecken können.

### Verifikation

Verifikation wird verwendet, um zu bestaetigen, dass die Interaktion zwischen dem zu testenden Code und den Mock-Objekten wie erwartet erfolgt. Die verify()-Methode kann überpruefen, ob Methoden aufgerufen wurden, wie oft sie aufgerufen wurden und welche Argumente übergeben wurden, um sicherzustellen, dass der zu testende Code seine Abhängigkeiten korrekt verwendet.

### Spy

Ein Spy ermöglicht teilweises Mocking von echten Objekten. Im Gegensatz zu vollstaendig gemockten Objekten behalten Spies das meiste Verhalten des echten Objekts bei, während nur spezifische Methoden gemockt werden. Dies ist besonders nuetzlich, wenn Sie den groessten Teil der echten Logik testen und nur wenige Methoden mocken moechten.

## Typische Anwendungsfaelle

### Isolierung externer Abhängigkeiten

Wenn der zu testende Code von externen Systemen wie Datenbanken, Webdiensten oder Dateisystemen abhaengt, kann Mockito diese externen Abhängigkeiten mocken. Diesermöglicht das Ausfuehren von Tests ohne echte externe Systeme, was nicht nur die Testausfuehrung beschleunigt, sondern auch das Risiko von Testfehlern durch instabile externe Systeme eliminiert.

### Mocking unvollständiger Module

In großen Projekten können verschiedene Module in unterschiedlichen Entwicklungsstadien sein. Mockito kann Module mocken, die noch nicht fertig sind, sodass bereits fertige Module vorab getestet werden können, um Entwicklungsblockaden zu vermeiden.

### Mocking komplexer Objekte

Einige Geschaeftsobjekte sind sehr komplex zu konstruieren, mit mehreren Abhaengigkeitsebenen. Mockito erleichtert das Erstellen von Mock-Versionen dieser komplexen Objekte und konzentriert sich auf das Testen der Geschaeftslogik selbst.

## Integration mit JUnit

Mockito wird normalerweise in Verbindung mit dem JUnit-Testframework verwendet. Durch JUnit's @RunWith(MockitoJUnitRunner.class) oder @ExtendWith(MockitoExtension.class) werden mit @Mock annotierte Felder automatisch initialisiert, was das Schreiben von Testcode vereinfacht. Diese Kombination stellt die beste Praxis für Java-Unit-Tests dar.

## Versionskompatibilitaet

Mockito 5.x erfordert JDK 11 oder höher und unterstützt Java 8-Funktionen wie Lambda-Ausdruecke. Die Mockito 4.x-Serie unterstützt noch JDK 8. Entwickler können die geeignete Version basierend auf den Projektanforderungen wählen.

## Zusammenfassung

Mockito ist ein unverzichtbares Testwerkzeug für Java-Entwickler. Es loest elegant das Problem der Abhaengigkeitsisolierung in Unit-Tests und ermöglicht Entwicklern, zuverlässigen und wartbaren Testcode zu schreiben. Das Beherrschen von Mockito verbessert die Codequalitaet und Entwicklungseffizienz erheblich und ist eine Faehigkeit, die jeder Java-Entwickler besitzen sollte.
