# H2 数据库常见问题

## 安装问题

### Q: 如何下载 H2 数据库？

A: 访问 https://www.h2database.com/html/download.html 下载适合你平台的版本。有 Windows 安装程序和跨平台 ZIP 包两种选择。

### Q: H2 需要什么 Java 版本？

A: H2 需要 Java 11 或更高版本。

### Q: 如何在 Maven 项目中使用 H2？

A: 在 pom.xml 中添加依赖：
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### Q: 解压 ZIP 后如何运行 H2？

A: 在命令行中运行：
```bash
java -jar h2.jar
```
这将启动 H2 Web 控制台。

## 连接问题

### Q: 如何连接 H2 数据库？

A: 使用 JDBC 连接：
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### Q: H2 的默认用户名和密码是什么？

A: 默认用户名是 "sa"，密码为空。

### Q: 如何连接文件数据库？

A:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Q: 如何启动 H2 TCP 服务器？

A:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### Q: 端口被占用怎么办？

A: 可以在连接 URL 中指定端口：
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## 数据问题

### Q: 内存数据库的数据会丢失吗？

A: 是的，内存数据库（jdbc:h2:mem:）在所有连接关闭后数据会丢失。如需持久化，请使用文件数据库模式。

### Q: 如何查看数据库文件位置？

A: 在连接 URL 中指定路径：
```java
jdbc:h2:file:/path/to/database
```

### Q: 数据库文件损坏怎么办？

A: 尝试使用恢复模式：
```java
jdbc:h2:file:./testdb;LOG=0
```
或者从备份恢复。

### Q: 如何加密数据库？

A:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## 性能问题

### Q: 如何提高批量插入性能？

A: 使用批量操作并关闭自动提交：
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT...");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Q: 查询很慢怎么办？

A: 
1. 为常用查询条件创建索引
2. 使用 EXPLAIN 分析查询计划
3. 避免 SELECT *，只查询需要的列

### Q: 内存使用过高怎么办？

A: 调整 JVM 堆内存大小，并合理使用连接池。

## SQL 兼容性问题

### Q: H2 支持哪些数据库模式？

A: H2 可以模拟多种数据库：
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

使用方式：
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### Q: 如何处理自动递增 ID？

A: H2 支持多种方式：
```sql
-- 方式一：IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- 方式二：SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## 事务问题

### Q: 事务没有提交会怎样？

A: 未提交的事务在连接关闭时会自动回滚。

### Q: 如何处理死锁？

A: H2 会自动检测死锁并回滚其中一个事务。建议缩短事务范围，避免长时间持有锁。

## 安全问题

### Q: H2 数据库安全吗？

A: H2 适合开发和测试环境。对于生产环境，建议：
1. 使用加密数据库
2. 使用强密码
3. 限制网络访问
4. 使用 SSL 连接

### Q: 如何防止 SQL 注入？

A: 使用 PreparedStatement 而不是字符串拼接：
```java
// 正确
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// 错误
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## 工具问题

### Q: 如何使用 H2 控制台？

A: 运行 `java -jar h2.jar -web` 然后访问 http://localhost:8082

### Q: 如何使用 H2 Shell？

A: 运行 `java -jar h2.jar -shell` 或 `java -jar h2.jar -tool`

### Q: 如何查看 H2 版本？

A:
```bash
java -jar h2.jar -version
```
或者在 SQL 中：`SELECT H2VERSION();`

## 迁移问题

### Q: 如何从 MySQL 迁移到 H2？

A:
1. 设置 H2 兼容模式：`jdbc:h2:file:db;MODE=MySQL`
2. 导出 MySQL 数据为 SQL 脚本
3. 在 H2 中运行脚本

### Q: H2 版本升级注意什么？

A: 从 H2 1.x 升级到 2.x 需要特别注意事项，请参考官方迁移指南：
https://www.h2database.com/html/migration-to-v2.html
