# Node.js 使用手冊

## 基礎使用

### 建立第一個 Node.js 應用

Node.js 應用程式通常以 `.js` 檔案副檔名儲存。最簡單的 "Hello World" 應用如下：

```javascript
// hello.js
console.log('Hello World!');
```

在終端機中執行：
```bash
node hello.js
```

### 模組系統

Node.js 使用 CommonJS 模組系統。你可以建立自己的模組並匯出功能：

```javascript
// math.js
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, multiply };
```

在其他檔案中使用：
```javascript
// app.js
const math = require('./math');

console.log(math.add(2, 3)); // 輸出: 5
console.log(math.multiply(4, 5)); // 輸出: 20
```

### 內建模組

Node.js 提供了許多內建模組，無需安裝即可使用：

- **fs**: 檔案系統操作
- **http**: HTTP 伺服器和用戶端
- **path**: 路徑操作
- **os**: 作業系統資訊
- **events**: 事件處理

範例 - 建立 HTTP 伺服器：
```javascript
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello from Node.js Server!');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at http://127.0.0.1:3000/');
});
```

## 套件管理 (npm)

### 初始化專案

```bash
npm init
# 或者使用預設設定
npm init -y
```

這會建立 `package.json` 檔案，用於管理專案相依性和詮釋資料。

### 安裝相依性

```bash
# 安裝生產相依性
npm install express

# 安裝開發相依性
npm install --save-dev jest

# 全域安裝
npm install -g nodemon
```

### 執行指令稿

在 `package.json` 中定義指令稿：
```json
{
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  }
}
```

執行指令稿：
```bash
npm run start
npm run dev
npm test
```

## 非同步編程

### 回呼函數

```javascript
const fs = require('fs');

fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});
```

### Promise

```javascript
const fs = require('fs').promises;

async function readFile() {
  try {
    const data = await fs.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

readFile();
```

### async/await

現代 Node.js 開發推薦使用 async/await 語法：

```javascript
async function processData() {
  try {
    const users = await getUsers();
    const processed = await processUsers(users);
    await saveResults(processed);
    console.log('Processing completed!');
  } catch (error) {
    console.error('Error:', error);
  }
}
```

## 效能最佳化

### 避免阻塞操作

始終使用非同步方法處理 I/O 操作：

```javascript
// ❌ 錯誤：阻塞操作
const data = fs.readFileSync('large-file.txt');

// ✅ 正確：非阻塞操作
const data = await fs.promises.readFile('large-file.txt');
```

### 使用叢集模式

利用多核心處理器：

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`主行程 ${process.pid} 正在執行`);
  
  // 衍生工作行程
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker, code, signal) => {
    console.log(`工作行程 ${worker.process.pid} 已結束`);
  });
} else {
  // 工作行程可以共用任何 TCP 連線
  // 在本例中是 HTTP 伺服器
  const http = require('http');
  
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`工作行程 ${process.pid} 已啟動`);
}
```

## 最佳實務

1. **使用嚴格模式**: 在檔案頂部新增 `'use strict';`
2. **錯誤處理**: 始終處理可能的錯誤
3. **環境變數**: 使用 `.env` 檔案管理設定
