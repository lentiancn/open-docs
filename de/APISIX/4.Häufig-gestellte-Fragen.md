# Häufig gestellte Fragen

Dieses Dokument fasst häufige technische Fragen und Antworten zur Verwendung von Apache APISIX zusammen.

## Grundlagen

### Was ist Apache APISIX?

Apache APISIX ist ein Top-Level-Projekt der Apache Software Foundation und ein Cloud-nativer API-Gateway mit dynamischen, Echtzeit- und Hochleistungseigenschaften. Er bietet dynamisches Routing, dynamische Upstreams, dynamische Zertifikate, A/B-Tests, Canary Releases, Ratenbegrenzung, Angriffsschutz, Monitoring-Warnungen und mehr.

### Was ist der Unterschied zwischen APISIX und Nginx?

Nginx ist ein traditioneller Web-Server und Reverse-Proxy, während APISIX ein auf Nginx basierender API-Gateway ist. APISIX erweitert Nginx um vollständig dynamische Konfigurationsverwaltung, ein umfangreiches Plugin-System, granulares Traffic-Management und integrierte Observability-Funktionen.

### Wie ist die Leistung von APISIX?

APISIX erbt die Hochleistungseigenschaften von Nginx und behält durch Optimierungen und das Plugin-System eine hohe Verarbeitungsgeschwindigkeit. Die tatsächliche Leistung hängt von der Hardware und der Komplexität der Geschäftslogik ab – typischerweise können mehrere Zehntausend bis Hunderttausend Anfragen pro Sekunde verarbeitet werden.

## Installationsprobleme

### Docker startet nicht – was tun?

1. Prüfen Sie, ob Docker läuft: `docker ps`
2. Container-Logs anzeigen: `docker-compose logs apisix`
3. Häufige Probleme:
   - Port-Konflikt: Prüfen Sie, ob Ports 9080, 9180 oder 9443 belegt sind
   - etcd-Verbindung fehlgeschlagen: Stellen Sie sicher, dass der etcd-Container läuft
   - Berechtigungsprobleme: Vergewissern Sie sich, dass Sie ausreichende Docker-Rechte haben

### Wie finde ich die APISIX-Version heraus?

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Antwortformat: `Server: APISIX/Version`

### Dashboard nach Installation nicht erreichbar?

1. Prüfen Sie, ob der Dashboard-Port korrekt gemappt ist (Standard: 9180)
2. Überprüfen Sie die Firewall-Einstellungen
3. Stellen Sie sicher, dass die Container-Netzwerkkonfiguration korrekt ist

## Konfigurationsprobleme

### Wie ändere ich den Admin-API-Port?

Bearbeiten Sie die Konfigurationsdatei `apisix.yml`:

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### Wie aktiviere ich HTTPS?

1. SSL-Zertifikat konfigurieren:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "Zertifikatsinhalt",
  "key": "Privater Schlüssel",
  "snis": ["example.com"]
}'
```

2. HTTPS in der Route aktivieren:

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### Route-Konfiguration wird nicht wirksam?

1. Prüfen Sie die Route-Konfiguration auf Fehler
2. Stellen Sie sicher, dass die etcd-Synchronisierung funktioniert
3. Überprüfen Sie Plugins auf Syntaxfehler
4. Schauen Sie in die APISIX-Fehlerlogs

### Wie ermögliche ich Cross-Origin-Anfragen?

Verwenden Sie das `cors`-Plugin:

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## Leistungsprobleme

### Wie erhöhe ich den Durchsatz?

1. **Anzahl der Worker erhöhen:** `apisix.worker_processes: auto`
2. **HTTP/2 aktivieren:** `http2: true` in der Listen-Konfiguration
3. **Upstream-Verbindungspool optimieren:** `keepalive_pool.size` konfigurieren
4. **Caching-Plugins nutzen:** `proxy-cache`-Plugin aktivieren

### Speicherbedarf zu hoch – was tun?

1. Unbenötigte Plugins deaktivieren
2. Upstream-Verbindungen begrenzen
3. Log-Puffergröße anpassen
4. Streaming-Response für große Dateien verwenden

## Funktionsprobleme

### Wie implementiere ich A/B-Tests?

Verwenden Sie das `traffic-split`-Plugin:

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### Wie implementiere ich Request-Ratenbegrenzung?

Verwenden Sie das `limit-req`-Plugin:

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## Integrationsprobleme

### Wie integriere ich APISIX mit Kubernetes?

Verwenden Sie den APISIX Ingress Controller:

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### Wie integriere ich APISIX mit Prometheus?

1. Prometheus-Plugin aktivieren
2. Prometheus-Scraping-Konfiguration einrichten

## Sicherheitsfragen

### Wie verhindere ich Missbrauch von APISIX?

1. **Authentifizierung aktivieren:** Nutzen Sie key-auth oder andere Auth-Plugins auf allen Routen
2. **Admin-API-Zugriff einschränken:** Erlauben Sie nur vertrauenswürdigen IPs den Zugriff auf die Admin-API
3. **HTTPS verwenden:** TLS-Verschlüsselung aktivieren
4. **Ratenbegrenzung konfigurieren:** Schützen Sie sich vor böswilligen Anfragen
5. **Zugriffslogs aktivieren:** Protokollieren Sie alle Anfragen zur Nachverfolgung

### Wie aktiviere ich Autorisierung für die Admin-API?

```yaml
deployment:
  admin:
    admin_key_required: true
```

## Betriebsfragen

### Wie aktualisiere ich APISIX?

**Docker:**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes:**

```bash
helm upgrade apisix apisix/apisix
```

### Häufige Fehlercodes

- **401 Unauthorized**: Authentifizierung fehlgeschlagen
- **403 Forbidden**: Kein Zugriffsrecht
- **404 Not Found**: Route existiert nicht
- **429 Too Many Requests**: Ratenlimit überschritten
- **500 Internal Server Error**: Upstream-Fehler
- **502 Bad Gateway**: Upstream nicht erreichbar
- **503 Service Unavailable**: Dienst nicht verfügbar
