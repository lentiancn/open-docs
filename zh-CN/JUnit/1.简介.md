# JUnit 简介

JUnit 是 Java 平台上最流行的单元测试框架，用于编写和运行可重复的测试。它是开发高质量 Java 代码的基础工具，被全球数百万开发者广泛使用。

## JUnit 4 与 JUnit 5 的区别

### 版本概述

- **JUnit 4**：2010 年发布，是 Java 社区长期使用的测试框架的最后一个大版本
- **JUnit 5**：2017 年发布，是 JUnit 框架的全新升级版，包含三个子项目：JUnit Platform、JUnit Jupiter、JUnit Vintage

### 架构差异

JUnit 4 采用单体架构，所有功能集成在单个库中。JUnit 5 采用模块化架构，由三个独立项目组成：

- **JUnit Platform**：测试框架的基础，提供测试引擎 API、命令行启动器
- **JUnit Jupiter**：新的编程模型和扩展模型，支持编写测试和扩展
- **JUnit Vintage**：兼容引擎，允许在 JUnit 5 平台上运行 JUnit 3 和 JUnit 4 测试

### 主要区别对比

| 特性 | JUnit 4 | JUnit 5 |
|------|---------|---------|
| 最低 Java 版本 | Java 5+ | Java 8+ |
| 注解包 | org.junit | org.junit.jupiter.api |
| 测试类无需继承 | 否 | 是 |
| 断言风格 | Assert 静态方法 | Assertions 静态方法 |
| 参数化测试 | @Parameters | @ParameterizedTest |
| 嵌套测试 | 不支持 | 支持 |
| 动态测试 | 不支持 | 支持 |
| 并行执行 | 有限支持 | 原生支持 |
| 扩展模型 | @RunWith | @ExtendWith |

### 选择建议

- **新项目**：推荐使用 JUnit 5，利用其现代特性和更好的扩展性
- **遗留项目**：可继续使用 JUnit 4，或使用 JUnit Vintage 进行渐进式迁移
- **长期维护**：建议逐步迁移到 JUnit 5，享受更好的测试体验
