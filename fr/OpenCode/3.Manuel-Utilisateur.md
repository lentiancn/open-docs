# OpenCode Manuel utilisateur

Utilisation de base et techniques pour OpenCode.

## Démarrage rapide

### Poser des questions

Vous pouvez demander à OpenCode d'expliquer la base de code :

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### Ajouter des fonctionnalités

Avant d'ajouter des fonctionnalités, il est recommandé de laisser OpenCode créer un plan d'abord :

1. Appuyez sur **Tab** pour passer en mode Plan
2. Décrivez la fonctionnalité que vous voulez
3. Après avoir vu le plan, donnez votre avis ou ajoutez des détails
4. Quando vous êtes satisfait, appuyez sur **Tab** pour repasser en mode Build

### Faire des changements directs

Pour des changements plus directs, vous pouvez demander directement à OpenCode :

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### Annuler les changements

Si vous n'êtes pas satisfait des changements, utilisez la commande annuler :

```bash
/undo
```

OpenCode reverra les changements et affichera à nouveau votre message original. Vous pouvez ajuster votre demande et réessayer.

### Rétablir

Vous pouvez utiliser `/redo` pour rétablir les changements.

## Partager des sessions

```bash
/share
```

Ceci créera un lien vers la session actuelle et le copiera dans le presse-papiers.

## Commandes courantes

| Commande | Fonction |
|----------|----------|
| /init | Initialiser le projet |
| /connect | Connecter le fournisseur LLM |
| /undo | Annuler les changements |
| /redo | Rétablir les changements |
| /share | Partager la session |

## Bonnes pratiques

1. Parlez à OpenCode comme à un développeur junior
2. Fournissez suffisamment de contexte et de détails
3. Demandez à OpenCode de créer un plan d'abord, puis exécutez
4. Utilisez /undo quand vous n'êtes pas satisfait
5. Vérifiez et mettez à jour AGENTS.md régulièrement
