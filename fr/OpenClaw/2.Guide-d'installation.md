# Guide d'installation d'OpenClaw

Ce guide vous accompagne pas à pas dans le processus d'installation sur différents systèmes d'exploitation.

## Exigences

- **Node.js**: version 22 ou supérieure
- **Système d'exploitation**: macOS, Linux ou Windows
- **Optionnel**: Docker (pour les fonctions d'isolation sandbox)

## Méthodes d'installation

### macOS et Linux (Recommandé)

Le moyen le plus simple est d'utiliser le script d'installation officiel:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Ce script détecte automatiquement votre environnement système, installe les dépendances nécessaires et configure les paramètres de base.

### Windows

Exécutez ceci dans PowerShell:

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Installation directe npm

Si vous avez déjà Node.js configuré:

```bash
npm install -g openclaw@latest
```

### À partir du code source

Pour utiliser la dernière version de développement:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Après l'installation

### Exécuter l'assistant de configuration

Après l'installation, il est recommandé d'exécuter l'assistant pour la configuration de base:

```bash
openclaw onboard --install-daemon
```

L'assistant vous guidera à travers:
- Configuration de la clé API
- Paramètres du port de service
- Connexion aux canaux (WhatsApp, Telegram, Discord, etc.)
- Paramètres de sécurité et contrôle d'accès

### Vérifier l'état du service

Si vous avez suivi les étapes d'installation, le service devrait être en cours d'exécution:

```bash
openclaw gateway status
```

### Exécuter au premier plan

Pour voir les logs directement dans votre terminal (utile pour le débogage):

```bash
openclaw gateway --port 18789
```

### Ouvrir l'interface web

```bash
openclaw dashboard
```

Ou simplement ouvrez dans votre navigateur: `http://127.0.0.1:18789/`

## Variables d'environnement

Parfois vous devez personnaliser les emplacements de stockage:

- `OPENCLAW_HOME`: Définit le répertoire principal pour OpenClaw
- `OPENCLAW_STATE_DIR`: Remplace le répertoire des données d'état
- `OPENCLAW_CONFIG_PATH`: Spécifie un chemin personnalisé pour le fichier de configuration

## Vérifier l'installation

Après l'installation, vous pouvez:

1. Exécuter `openclaw --version` pour confirmer que ça fonctionne
2. Exécuter `openclaw doctor` pour vérifier l'état du système
3. Ouvrir l'interface web et envoyer un message de test

## Problèmes courants

### Vérifier la version de Node.js

```bash
node --version
```

Assurez-vous que la version 22 ou supérieure est affichée. Si non, mettez à niveau Node.js d'abord.

### Git non trouvé sur Windows

Assurez-vous que Git est installé et dans votre PATH système. Vous pouvez le télécharger depuis git-scm.com.

### Erreurs de permissions

Si vous rencontrez des problèmes de permissions, essayez:

```bash
sudo npm install -g openclaw@latest
```
