# OpenClaw 安裝指南

本指南詳細介紹 OpenClaw 在不同作業系統上的安裝步驟。

## 環境需求

- **Node.js**：22 或更高版本
- **作業系統**：macOS、Linux、Windows
- **選配**：Docker（如果需要沙箱功能）

## 安裝方法

### macOS / Linux（推薦）

使用安裝腳本是最簡單的方式：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

該腳本會自動檢測你的系統環境，安裝必要的依賴，並配置好 OpenClaw。

### Windows（PowerShell）

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 使用 npm 直接安裝

如果你已經安裝了 Node.js，也可以直接使用 npm：

```bash
npm install -g openclaw@latest
```

### 從源碼安裝

如果你想使用最新開發版本或者想要自定義安裝：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 安裝後配置

### 執行入門精靈

安裝完成後，執行精靈來配置認證、Gateway 設定和可選的通道：

```bash
openclaw onboard --install-daemon
```

精靈會引導你完成：
- API 金鑰配置
- Gateway 連接埠設定
- 通道連接（WhatsApp、Telegram、Discord 等）
- 安全和權限設定

### 檢查 Gateway 狀態

如果你是按照上面的命令安裝了服務，Gateway 應該已經在運行：

```bash
openclaw gateway status
```

### 啟動 Gateway（前景執行）

如果你想在終端前景執行 Gateway（適合測試或排查問題）：

```bash
openclaw gateway --port 18789
```

### 開啟控制介面

```bash
openclaw dashboard
```

或者直接在瀏覽器中開啟：`http://127.0.0.1:18789/`

## 環境變數

如果你需要自定義配置或狀態目錄位置，可以使用以下環境變數：

- `OPENCLAW_HOME`：設定用於內部路徑解析的主目錄
- `OPENCLAW_STATE_DIR`：覆蓋狀態目錄
- `OPENCLAW_CONFIG_PATH`：覆蓋配置檔案路徑

## 驗證安裝

安裝完成後，你可以：

1. 執行 `openclaw --version` 確認版本
2. 執行 `openclaw doctor` 檢查系統健康狀況
3. 開啟控制介面傳送測試訊息

## 常見安裝問題

### Node.js 版本檢查

```bash
node --version
```

確保版本號是 22 或更高。如果不是，請先升級 Node.js。

### Windows 上 Git 未找到

確保 Git 已安裝並添加到系統 PATH 環境變數中。你可以從 [git-scm.com](https://git-scm.com) 下載安裝。

### 權限問題

如果在安裝過程中遇到權限錯誤，嘗試：

```bash
sudo npm install -g openclaw@latest
```
