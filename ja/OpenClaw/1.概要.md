# OpenClaw 🦞

OpenClawは、自己ホスト型のゲートウェイです。WhatsApp、Telegram、Discord、iMessageなどのお気に入りのチャットアプリと、AIコーディングアシスタント（Piなど）を接続します。自身のマシン（またはサーバー）で単一のGatewayプロセスを実行するだけで、メッセージアプリと常に利用可能なAIアシスタントの架け橋になります。

## 対象ユーザー

OpenClawは、どこからでもメッセージでコミュニケーション取れるパーソナルAIアシスタントを必要とする開発者ーパワーユーザー向けに設計されています。データの管理を手放すことなく、ホストサービスに依存しません。

## 主な特徴

OpenClawが他のサービスと異なる点：

- **自己ホスト型**：自分のハードウェアで実行、自分のルール
- **マルチチャンネル**：1つのGatewayがWhatsApp、Telegram、Discordなどを同時にサポート
- **エージェントネイティブ**：ツール使用、セッション、メモリ、マルチエージェントルーティングを備えたコーディングエージェント向け
- **オープンソース**：MITライセンス、社区主導

## 必要なもの

- Node.js 22以上
- 選択したAIプロバイダーからのAPIキー
- 5分

最高の品質とセキュリティのために、最新世代で最も強力なモデルを使用することをお勧めします。

## クイックスタート

```bash
# OpenClawをインストール
npm install -g openclaw@latest

# オンボーディングウィザードを実行してサービスをインストール
openclaw onboard --install-daemon

# チャンネルをログインしてGatewayを起動
openclaw channels login
openclaw gateway --port 18789
```

## 動作原理

Gatewayは、セッション、ルーティング、チャンネル接続の単一の情報源です。チャットアプリを通じてAIエージェント、CLI、WebコントロールUI、またはモバイルノードに接続します。

## コア機能

- **マルチチャンネルゲートウェイ**：単一のGatewayプロセスでWhatsApp、Telegram、Discord、iMessageをサポート
- **プラグインチャンネル**：拡張パッケージでMattermostなどを追加
- **マルチエージェントルーティング**：エージェント、工作区、送信者ごとに分離されたセッション
- **メディアサポート**：画像、音声、ドキュメントの送受信
- **WebコントロールUI**：チャット、設定、セッション、ノード用のブラウザダッシュボード
- **モバイルノード**：Canvas、カメラ/スクリーン、音声対応ワークフローのためのiOS/Androidノードペアリング
