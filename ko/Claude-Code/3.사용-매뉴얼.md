# Claude Code 사용 매뉴얼

 Claude Code를 빠르게 시작하고 일상적인 개발 작업을 마스터하는 가이드입니다.

## 준비

다음이 있는지 확인하세요:
- 열린 터미널
- 작업할 코드 프로젝트
- Claude 구독 (Pro, Max, Teams 또는 Enterprise)

## 세션 시작

프로젝트 디렉토리에서 터미널을 열고 Claude Code를 시작합니다:

```bash
cd /path/to/your/project
claude
```

## 기본 작업

### 코드베이스 이해

 Claude에게 프로젝트를 분석하도록 요청:
```
이 프로젝트는 무엇을 하나요?
```

```
이 프로젝트는 어떤 기술을 사용하나요?
```

### 첫 번째 코드 변경

간단한 작업을 시도:
```
메인 파일에 hello world 함수를 추가하세요
```

 Claude Code는:
1. 적절한 파일을 찾음
2. 제안된 변경 사항을 표시
3. 승인을 요청
4. 편집을 수행

### Git 작업

```
수정한 파일은 무엇인가요?
```

```
설명적인 메시지와 함께 변경 사항을 커밋하세요
```

### 버그 수정 및 기능 추가

자연어로 원하는 것을 설명:
```
등록 폼에 입력 유효성 검사를 추가하세요
```

## 필수 명령어

| 명령어 | 기능 |
|--------|------|
| `claude` | 대화형 모드 |
| `claude "작업"` | 일회성 작업 실행 |
| `claude -p "쿼리"` | 일회성 쿼리 후 종료 |
| `claude commit` | Git 커밋 생성 |
| `/help` | 사용 가능한 명령어 |

## 고급 팁

### CLAUDE.md
프로젝트 루트에 CLAUDE.md 파일을 만들어 지속적인 컨텍스트를 제공하세요.

### 사용자 정의 명령
`/review-pr` 또는 `/deploy-staging` 같은 명령을 만드세요.

### 멀티 에이전트 협업
복잡한 작업의 경우 여러 에이전트를 병렬로 실행하세요.

## 모범 사례

### 구체적으로 요청
❌ "버그를 수정하세요"말할 수 없음
✅ "잘못된 자격 증명 후 빈 화면을 보는 로그인 버그를 수정하세요"

### Claude에게 먼저 탐색하게 하기
변경 전에:
```
데이터베이스 스키마를 분석하세요
```
