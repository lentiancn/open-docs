# Guía de Instalación de OpenClaw

Esta guía proporciona pasos detallados de instalación para OpenClaw en diferentes sistemas operativos.

## Requisitos

- **Node.js**: versión 22 o superior
- **Sistema Operativo**: macOS, Linux o Windows
- **Opcional**: Docker (para funcionalidad de sandbox)

## Métodos de Instalación

### macOS / Linux (Recomendado)

La forma más fácil de instalar OpenClaw es usando el script de instalación:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Este script detecta automáticamente tu entorno de sistema, instala las dependencias necesarias y configura OpenClaw.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Instalación Directa con npm

Si ya tienes Node.js instalado, también puedes usar npm directamente:

```bash
npm install -g openclaw@latest
```

### Instalación desde el Código Fuente

Si quieres la última versión de desarrollo o necesitas una instalación personalizada:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Configuración Post-Instalación

### Ejecutar el Asistente de Incorporación

Después de la instalación, ejecuta el asistente para configurar autenticación, ajustes de Gateway y canales opcionales:

```bash
openclaw onboard --install-daemon
```

El asistente te guiará a través de:
- Configuración de clave API
- Ajustes de puerto del Gateway
- Conexiones de canales (WhatsApp, Telegram, Discord, etc.)
- Configuración de seguridad y permisos

### Verificar el Estado del Gateway

Si instalaste el servicio como se muestra arriba, el Gateway ya debería estar ejecutándose:

```bash
openclaw gateway status
```

### Iniciar Gateway en Primer Plano

Si quieres ejecutar el Gateway en primer plano (útil para pruebas o solución de problemas):

```bash
openclaw gateway --port 18789
```

### Abrir la Interfaz de Control

```bash
openclaw dashboard
```

O simplemente abre en tu navegador: `http://127.0.0.1:18789/`

## Variables de Entorno

Si necesitas personalizar la configuración o las ubicaciones del directorio de estado, puedes usar estas variables de entorno:

- `OPENCLAW_HOME`: establece el directorio home usado para resolución de rutas internas
- `OPENCLAW_STATE_DIR`: anula el directorio de estado
- `OPENCLAW_CONFIG_PATH`: anula la ruta del archivo de configuración

## Verificar la Instalación

Después de la instalación, puedes:

1. Ejecutar `openclaw --version` para confirmar la versión
2. Ejecutar `openclaw doctor` para verificar la salud del sistema
3. Abrir la Interfaz de Control para enviar un mensaje de prueba

## Problemas Comunes de Instalación

### Verificar la Versión de Node.js

```bash
node --version
```

Asegúrate de que la versión sea 22 o superior. Si no, actualiza Node.js primero.

### Git No Encontrado en Windows

Asegúrate de que Git esté instalado y agregado a tu PATH del sistema. Puedes descargarlo de [git-scm.com](https://git-scm.com).

### Errores de Permisos

Si encuentras errores de permisos durante la instalación, intenta:

```bash
sudo npm install -g openclaw@latest
```
