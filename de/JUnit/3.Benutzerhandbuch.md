# JUnit Benutzerhandbuch

Dieses Handbuch behandelt die Verwendung von JUnit 4 und JUnit 5, einschließlich Testschreiben, Assertions, Test-Suites und anderer Kernfunktionen.

## JUnit 5 Grundlegende Verwendung

### Testklassen schreiben

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void testAddition() {
        int result = 2 + 3;
        assertEquals(5, result);
    }
}
```

### Häufige Assertions

```java
import static org.junit.jupiter.api.Assertions.*;

// Gleichheits-Assertions
assertEquals(expected, actual);
assertEquals(expected, actual, "Benutzerdefinierte Fehlermeldung");

// Boolean-Assertions
assertTrue(condition);
assertFalse(condition);

// Null-Assertions
assertNull(object);
assertNotNull(object);

// Array-Assertions
assertArrayEquals(expectedArray, actualArray);

// Exception-Assertions
assertThrows(Exception.class, () -> {
    // Code, der eine Exception auslösen kann
});

// Kombinierte Assertions
assertAll("Personen-Eigenschaften",
    () -> assertEquals("John", person.getFirstName()),
    () -> assertEquals("Doe", person.getLastName())
);
```

### Before- und After-Operationen

```java
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;

class MyTest {
    
    @BeforeEach
    void setUp() {
        // Wird vor jedem Testmethodenaufruf ausgeführt
    }
    
    @AfterEach
    void tearDown() {
        // Wird nach jedem Testmethodenaufruf ausgeführt
    }
}
```

### Verschachtelte Tests

```java
import org.junit.jupiter.api.Nested;

@Nested
class NestedTests {
    
    @Test
    void testInNestedClass() {
        // Tests in verschachtelter Testklasse
    }
}
```

### Parametrisierte Tests

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void testWithMultipleValues(int number) {
    assertTrue(number > 0);
}
```

### Dynamische Tests

```java
import org.junit.jupiter.api.TestFactory;
import org.junit.jupiter.api.DynamicTest;
import java.util.Arrays;

@TestFactory
Collection<DynamicTest> dynamicTestsFromIterator() {
    return Arrays.asList(
        DynamicTest.dynamicTest("Erster dynamischer Test", () -> assertTrue(true)),
        DynamicTest.dynamicTest("Zweiter dynamischer Test", () -> assertEquals(4, 2 + 2))
    );
}
```

## JUnit 4 Grundlegende Verwendung

### Testklassen schreiben

```java
import org.junit.Test;
import static org.junit.Assert.*;

public class CalculatorTest {
    
    @Test
    public void testAddition() {
        assertEquals(5, 2 + 3);
    }
}
```

### Häufige Assertions

```java
// Gleichheits-Assertions
assertEquals(expected, actual);
assertEquals("Nachricht", expected, actual);

// Boolean-Assertions
assertTrue(condition);
assertFalse(condition);

// Null-Assertions
assertNull(object);
assertNotNull(object);

// Exception-Assertions
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}
```

### Before und After

```java
import org.junit.Before;
import org.junit.After;

public class MyTest {
    
    @Before
    public void setUp() {
        // Wird vor jedem Testmethodenaufruf ausgeführt
    }
    
    @After
    public void tearDown() {
        // Wird nach jedem Testmethodenaufruf ausgeführt
    }
    
    @BeforeClass
    public static void setUpClass() {
        // Wird einmal vor allen Tests in der Klasse ausgeführt
    }
    
    @AfterClass
    public static void tearDownClass() {
        // Wird einmal nach allen Tests in der Klasse ausgeführt
    }
}
```

### Test-Suites

```java
import org.junit.runner.RunWith;
import org.junit.runners.Suite;
import org.junit.runners.Suite.SuiteClasses;

@RunWith(Suite.class)
@SuiteClasses({Test1.class, Test2.class, Test3.class})
public class AllTests {
    // Führt alle Tests in Test1, Test2 und Test3 aus
}
```

## Migration von JUnit 4 zu JUnit 5

### Änderungen bei Annotationen

| JUnit 4 | JUnit 5 |
|---------|---------|
| @Test | @Test |
| @Before | @BeforeEach |
| @After | @AfterEach |
| @BeforeClass | @BeforeAll |
| @AfterClass | @AfterAll |
| @Ignore | @Disabled |
| @SuiteClasses | @SelectClasses |

### Änderungen bei Assertions

```java
// JUnit 4
Assert.assertEquals(expected, actual);

// JUnit 5
Assertions.assertEquals(expected, actual);
```

### Änderungen bei Exception-Tests

```java
// JUnit 4
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}

// JUnit 5
@Test
public void testException() {
    assertThrows(NullPointerException.class, () -> {
        Object obj = null;
        obj.toString();
    });
}
```

## Tests ausführen

### Befehlszeile

```bash
# Maven
mvn test

# Gradle
gradle test

# Eigenständige Konsole
java -jar junit-platform-console-standalone-5.14.3.jar -c MyTestClass
```

### IDE-Ausführung

- IntelliJ IDEA: Rechtsklick auf Testklasse oder -methode, Run auswählen
- Eclipse: Rechtsklick auf Testklasse oder -methode, Run As → JUnit Test auswählen
- VS Code: JUnit-Erweiterung zum Ausführen von Tests verwenden
