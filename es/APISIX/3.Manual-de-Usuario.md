# APISIX Manual de Usuario

Características principales y consejos de uso de APISIX.

## Conceptos Básicos

### Terminología

- **Route**: Define reglas de coincidencia de solicitudes
- **Upstream**: Servicios backend reales
- **Service**: Colección de rutas
- **Plugin**: Lógica de procesamiento de solicitudes
- **Consumer**: Consumidor de API

### Flujo de Solicitudes

1. Cliente envía solicitud a APISIX
2. APISIX coincide con reglas de ruta
3. Ejecuta lógica de plugins
4. Reenvía al servicio upstream
5. Devuelve respuesta al cliente

## Gestión de Rutas

### Crear Ruta

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "id": "1",
    "uri": "/api/users/*",
    "name": "user-api",
    "upstream": {
      "type": "roundrobin",
      "nodes": {"backend1.example.com:8080": 1}
    }
  }'
```

### Parámetros de Ruta

| Parámetro | Descripción | Requerido |
|-----------|-------------|------------|
| id | Identificador único | Sí |
| uri | URI a coincidir | Sí |
| name | Nombre de ruta | No |
| upstream | Config upstream | Sí |

## Gestión de Upstream

### Crear Upstream

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/upstreams/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "type": "roundrobin",
    "nodes": {"host1:8080": 1, "host2:8080": 1}
  }'
```

### Estrategias de Balanceo

**Roundrobin:**

```json
{"type": "roundrobin"}
```

**Hash Consistente:**

```json
{"type": "chash", "key": "remote_addr"}
```

## Plugins

### Plugins Comunes

**Control de Tráfico:**
- limit-req: Rate limiting
- limit-conn: Límite de conexiones

**Autenticación:**
- jwt-auth: Autenticación JWT
- key-auth: Autenticación por clave
- basic-auth: Autenticación básica

### Usar Plugin de Rate Limit

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "uri": "/api/*",
    "plugins": {
      "limit-req": {
        "rate": 100,
        "burst": 50,
        "rejected_code": 503
      }
    }
  }'
```

### Usar JWT

**1. Crear Consumer:**

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/consumers \
  -H "X-API-KEY: 123456" \
  -d '{
    "username": "john",
    "plugins": {
      "jwt-auth": {"key": "user-key", "secret": "my-secret"}
    }
  }'
```

**2. Habilitar en Ruta:**

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{"uri":"/api/*","plugins":{"jwt-auth":{}}}'
```
