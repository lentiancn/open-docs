# Angular Руководство по установке

Полная настройка среды разработки Angular.

## Требования к окружению

### Аппаратные требования

- CPU: 2 ядра или больше
- Оперативная память: 4 ГБ или больше
- Диск: 10 ГБ или больше

### Программные требования

- Операционная система: Windows/macOS/Linux
- Node.js: 18.10.0+ или 20.0.0+
- npm: 9.0.0+ (входит в Node.js)
- Git: Любая версия

## Шаги установки

### 1. Установите Node.js

**Windows/macOS:**

Скачать с: https://nodejs.org

Или используйте nvm:

```bash
# Установить nvm (macOS/Linux)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Установить Node.js LTS
nvm install --lts
nvm use --lts
```

**Проверить установку:**

```bash
node --version  # Должно показать v18.x.x или v20.x.x
npm --version    # Должно показать 9.x.x или выше
```

### 2. Установите Angular CLI

```bash
# Установить глобально через npm
npm install -g @angular/cli

# Проверить установку
ng version
```

### 3. Создайте новый проект

```bash
# Создать новое приложение Angular
ng new my-app

# Перейти в проект
cd my-app

# Запустить сервер разработки
ng serve
```

### 4. Откройте приложение

Откройте браузер: http://localhost:4200

## Общие команды

### Команды проекта

```bash
# Создать проект с маршрутизацией и SCSS
ng new my-app --routing --style=scss

# Создать стandalone-приложение (Angular 14+)
ng new my-app --standalone

# Запустить сервер разработки
ng serve

# Собрать для продакшена
ng build

# Запустить юнит-тесты
ng test

# Запустить E2E-тесты
ng e2e
```

### Команды генерации

```bash
# Создать компонент
ng generate component my-component
ng g c my-component

# Создать сервис
ng generate service my s my-service

# Создать директиву
ng generate directive my-directive
ng g d my-service
ng g-directive

# Создать пайп
ng generate pipe my-pipe
ng g p my-pipe

# Создать модуль
ng generate module my-module
ng g m my-module

# Создать класс
ng generate class my-class
ng g cl my-class

# Создать интерфейс
ng generate interface my-interface
ng g i my-interface

# Создать guard
ng generate guard my-guard
ng g g my-guard

# Создать interceptor
ng generate interceptor my-interceptor
ng g in my-interceptor
```

## Конфигурация окружения

### Окружение разработки

```typescript
// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: 'http://localhost:3000/api'
};
```

### Продакшен-окружение

```typescript
// src/environments/environment.prod.ts
export const environment = {
  production: true,
  apiUrl: 'https://api.example.com'
};
```

### Использование окружения

```typescript
import { Component } from '@angular/core';
import { environment } from '../environments/environment';

@Component({
  selector: 'app-root',
  template: `<p>API: {{ apiUrl }}</p>`
})
export class AppComponent {
  apiUrl = environment.apiUrl;
}
```

## Частые проблемы

### npm медленный?

```bash
# Использовать зеркало Taobao
npm config set registry https://registry.npmmirror.com

# Или использовать nrm для управления зеркалами
npm install -g nrm
nrm use taobao
```

### Версия Node не подходит?

```bash
# Использовать nvm для переключения версий
nvm install 20
nvm use 20
```

### Проблемы с правами (Linux/macOS)?

```bash
# Исправить права npm
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## Удаление

```bash
# Удалить Angular CLI
npm uninstall -g @angular/cli

# Удалить проект
rm -rf my-app
```
