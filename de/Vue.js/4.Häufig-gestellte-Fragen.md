# Vue.js Häufig gestellte Fragen

## Unterschiede zwischen Vue 2 und Vue 3

Vue 3 ist die aktuelle, neueste Hauptversion von Vue. Sie enthält neue Funktionen, die in Vue 2 nicht vorhanden sind, wie Teleport, Suspense und mehrere Root-Elemente pro Template. Sie enthält auch Breaking Changes, die sie mit Vue 2 inkompatibel machen.

**Hauptvorteile:**
- Kleinere Bundle-Größen
- Bessere Leistung
- Bessere Skalierbarkeit
- Bessere TypeScript / IDE-Unterstützung

**Empfehlung:**
- **Neue Projekte sollten Vue 3 verwenden**
- **Verwenden Sie Vue 2 nur, wenn IE11-Unterstützung erforderlich ist** (Vue 3 unterstützt IE11 nicht)

**Migrationsinformationen:**
- Die Composition API war ursprünglich eine Vue-3-exklusive Funktion, ist aber jetzt in Vue 2.7 verfügbar
- Detaillierter Migrationsleitfaden: [Vue 3 Migrationsleitfaden](https://v3-migration.vuejs.org/)

## Wird Vue 2 noch unterstützt?

- Vue 2.7, veröffentlicht im Juli 2022, ist die letzte Nebenversion des Vue 2 Versionsbereichs
- **Vue 2 hat am 31. Dezember 2023 das Ende der Lebensdauer erreicht**
- Erweiterter Support verfügbar für Teams mit Compliance-Anforderungen: [Vue 2 Extended LTS](https://v2.vuejs.org/lts/)

## Welche Browser unterstützt Vue?

- **Vue 3.x**: Unterstützt nur Browser mit nativer ES2016-Unterstützung (schließt IE11 aus)
- **Vue 2.x**: Unterstützt IE11 und breitere Browser-Kompatibilität

## Ist Vue zuverlässig?

- Vue ist ein ausgereiftes und bewährtes Framework
- Über **1,5 Millionen Benutzer weltweit**
- Fast **10 Millionen monatliche Downloads** auf npm
- Wird in der Produktion von renommierten Organisationen verwendet: Wikimedia Foundation, NASA, Apple, Google, Microsoft, GitLab, Zoom, Tencent, Weibo, Bilibili, Kuaishou und viele mehr

## Ist Vue schnell?

- Vue 3 ist eines der leistungsstärksten Mainstream-Frontend-Frameworks
- Übertrifft React und Angular im [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html)
- Die offizielle Dokumentationswebsite erzielt 100 Punkte in der Leistung auf einem emulierten Moto G4 mit 4x CPU-Drosselung über langsame 4G-Netzwerke
- Verarbeitet die meisten Webanwendungsfälle ohne manuelle Optimierung

## Ist Vue leichtgewichtig?

- **Minimale App** (mit Build-Tool + Tree-Shaking): ~16kb (brotli-komprimiert)
- **Vollständige App**: ~27kb
- **Ohne Build-Tool** (inkl. Compiler): ~41kb
- **Progressive Enhancement**: Erwägen Sie [petite-vue](https://github.com/vuejs/petite-vue) (~6kb)

## Skaliert Vue?

Ja. Trotz der weit verbreiteten Annahme, dass Vue nur für einfache Anwendungsfälle geeignet ist, ist Vue perfekt in der Lage, groß angelegte Anwendungen zu verarbeiten:

- **Single-File-Komponenten**: Bieten ein modularisiertes Entwicklungsmodell
- **Composition API**: Erstklassige TypeScript-Integration zur Organisation komplexer Logik
- **Umfassende Tooling-Unterstützung**: Sorgt für eine reibungslose Entwicklungserfahrung beim Wachsen der Anwendung
- **Niedrige Einstiegshürde**: Reduziert Onboarding- und Schulungskosten für neue Entwickler

## Open-Source-Lizenz

Vue wird unter der **MIT-Lizenz** veröffentlicht und ist damit kostenlos und Open Source.

## Wahl des API-Stils

### Options API vs Composition API

**Options API ist geeignet für:**
- Anfänger oder Entwickler mit OOP-Hintergrund
- Einfache Szenarien ohne Build-Tools
- Progressive Enhancement-Anwendungen

**Composition API ist geeignet für:**
- Vollständige Anwendungsentwicklung mit Vue
- Bessere Logikorganisation und -wiederverwendung
- TypeScript-Projekte

## JavaScript vs TypeScript

- Vue selbst ist in TypeScript implementiert und bietet erstklassige TS-Unterstützung
- **Erzwingt die Verwendung von TypeScript nicht**
- TypeScript-freundliche APIs kommen allen Benutzern zugute (bessere IDE-Unterstützung)
- Die Wahl hängt vom Teamhintergrund und dem Projektumfang ab

## Vue vs Web Components

- Vue wurde vor der Verfügbarkeit von Web Components erstellt
- Web Components sind Low-Level (zentriert auf benutzerdefinierte Elemente)
- Vue adressiert höherstufige Anliegen (Rendering, Zustandsverwaltung, Routing, SSR)
- Vue unterstützt vollständig das Konsumieren und Exportieren nativer benutzerdefinierter Elemente
- Detaillierter Vergleich: [Vue und Web Components Leitfaden](/guide/extras/web-components)
