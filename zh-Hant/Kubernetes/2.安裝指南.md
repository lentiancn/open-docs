# Kubernetes 安裝指南

## 準備工作

在安裝 Kubernetes 之前，需要了解以下基本要求：

### 系統要求
- Linux 是執行 Kubernetes 控制平面的首選作業系統
- 在叢集中可以在 Linux 或其他作業系統（包括 Windows）上執行應用程式
- 需要支援容器執行的作業系統

### 安裝類型選擇
安裝 Kubernetes 時，請根據以下條件選擇安裝類型：
- 維護便利性
- 安全性
- 可控制性
- 可用資源
- 操作和管理 Kubernetes 叢集所需的專業知識

## 安裝方式

### 學習環境（推薦入門）

如果正在學習 Kubernetes，建議使用 Kubernetes 社群支援或生態系統中的工具在本地電腦上設定叢集：

#### Minikube
Minikube 是一個輕量級的 Kubernetes 實作，可以在本地機器上建立單節點叢集。適合學習和小規模測試。

安裝步驟：
1. 安裝 Minikube
2. 安裝 kubectl（Kubernetes 命令列工具）
3. 啟動 Minikube 叢集

#### Kind
Kind（Kubernetes in Docker）是另一個用於建立本地 Kubernetes 叢集的工具，使用 Docker 容器模擬節點。

### 生產環境

對於生產環境，需要考慮以下選項：

#### kubeadm
官方支援的用於部署 Kubernetes 的工具。可以建立符合最佳實踐的安全叢集。

基本步驟：
1. 準備伺服器（安裝作業系統、配置網路）
2. 安裝容器執行時（如 containerd 或 Docker）
3. 安裝 kubeadm、kubelet 和 kubectl
4. 使用 kubeadm 初始化叢集
5. 配置網路插件（如 Calico、Cilium）
6. 新增工作節點

#### 託管服務
如果不想自己管理 Kubernetes 叢集，可以選擇託管服務：
- Amazon EKS（Elastic Kubernetes Service）
- Google GKE（Google Kubernetes Engine）
- Azure AKS（Azure Kubernetes Service）
- 阿里雲 ACK
- 騰訊雲 TKE

#### 其他方案
在各種雲和裸機環境中，還有其他標準化和自訂的解決方案可供選擇。

## 工具安裝

### 安裝 kubectl

kubectl 是與 Kubernetes 叢集互動的命令列工具。

Linux 安裝：
```bash
# 下載最新版本
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 新增執行權限
chmod +x kubectl

# 移動到系統路徑
sudo mv kubectl /usr/local/bin/
```

### 安裝 kubeadm

```bash
# 更新 apt 套件索引
sudo apt-get update

# 安裝相依套件
sudo apt-get install -y apt-transport-https ca-certificates curl

# 新增 Kubernetes APT 來源
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# 更新並安裝
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## 叢集初始化

### 使用 kubeadm 建立叢集

1. 初始化主節點：
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. 配置 kubectl：
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. 安裝網路插件（以 Flannel 為例）：
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. 加入工作節點：
```bash
# 在每個工作節點上執行初始化輸出中的指令
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## 驗證安裝

檢查叢集狀態：
```bash
# 查看節點
kubectl get nodes

# 查看 Pod
kubectl get pods --all-namespaces
```

如果所有元件都處於 Running 狀態，表示叢集安裝成功。

## 解除安裝

如果需要解除安裝 Kubernetes：

```bash
# 停止所有 Kubernetes 服務
sudo kubeadm reset

# 刪除設定檔
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# 解除安裝套件
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## 注意事項

1. 容器執行時選擇：確保為新叢集選擇合適的容器執行時
2. 資源配置：根據工作負載合理配置 CPU 和記憶體資源
3. 網路規劃：提前規劃 Pod 和 Service 的網路 CIDR
4. 安全強化：生產環境需要配置認證、授權和網路原則
5. 高可用性：生產環境建議配置多主節點叢集
