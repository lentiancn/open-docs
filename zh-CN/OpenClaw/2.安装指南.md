# OpenClaw 安装指南

本文档详细介绍在不同操作系统上安装 OpenClaw 的步骤。

## 环境要求

- Node.js：22 或更高版本
- 操作系统：macOS、Linux、Windows
- 可选：Docker（如果你需要使用沙箱隔离功能）

## 安装方法

### macOS 和 Linux（推荐）

最简便的方式是使用官方安装脚本：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

该脚本会自动检测你的系统环境，完成依赖安装和基础配置。

### Windows 系统

在 PowerShell 中执行：

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 使用 npm 直接安装

如果你已经配置好了 Node.js 环境，也可以直接用 npm 安装：

```bash
npm install -g openclaw@latest
```

### 从源码安装

如果你想使用最新的开发版本，或者需要自定义安装方式：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 安装后的初始配置

### 运行初始化向导

安装完成后，建议先运行初始化向导来完成基本配置：

```bash
openclaw onboard --install-daemon
```

向导会引导你完成以下设置：
- API 密钥配置
- 服务端口设定
- 聊天渠道的登录（如微信、Telegram 等）
- 安全相关的访问控制配置

### 验证服务状态

如果是按照上面的步骤安装的，服务应该已经在后台运行了：

```bash
openclaw gateway status
```

### 前台运行模式

如果你想在终端窗口中直接看到运行日志（适合调试问题）：

```bash
openclaw gateway --port 18789
```

### 打开网页管理界面

```bash
openclaw dashboard
```

或者直接在浏览器中访问：`http://127.0.0.1:18789/`

## 环境变量说明

在某些场景下，你可能需要自定义数据存储位置：

- `OPENCLAW_HOME`：指定 OpenClaw 的主目录
- `OPENCLAW_STATE_DIR`：指定状态数据的存储目录
- `OPENCLAW_CONFIG_PATH`：指定配置文件的自定义路径

## 验证安装结果

安装完成后，你可以：

1. 运行 `openclaw --version` 查看版本号确认安装成功
2. 运行 `openclaw doctor` 检查系统健康状态
3. 打开网页管理界面发送测试消息

## 常见问题

### Node.js 版本检查

```bash
node --version
```

确保显示的版本号在 22 以上。如果版本过低，请先升级 Node.js。

### Windows 上提示找不到 Git

请确认 Git 已经安装并加入了系统环境变量。可以从 git-scm.com 下载安装。

### 权限报错

如果在安装过程中遇到权限问题，尝试加上 sudo：

```bash
sudo npm install -g openclaw@latest
```
