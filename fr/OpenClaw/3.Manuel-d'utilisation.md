# Manuel d'Utilisation d'OpenClaw

Ce manuel vous aidera à comprendre comment utiliser OpenClaw efficacement.

## Utilisation de Base

### Parler à OpenClaw

Après l'installation, vous pouvez interagir avec OpenClaw de plusieurs manières :

1. **Control UI** : Visitez `http://127.0.0.1:18789/` dans votre navigateur
2. **Commande CLI** : `openclaw agent --message "Votre message"`
3. **Canaux configurés** : Envoyez des messages via WhatsApp, Telegram, Discord, etc.

### Envoyer des Messages

Envoyez des messages en utilisant CLI :

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## Concepts Clés

### Gateway

Gateway est le plan de contrôle principal d'OpenClaw, gérant toutes les sessions, canaux, outils et événements. Écoute par défaut : `ws://127.0.0.1:18789`.

### Session

OpenClaw utilise un modèle de session pour gérer les conversations :

- **Session principale** : Chat direct
- **Isolement des groupes** : Messages de groupe traités séparément
- **Mode d'activation** : Contrôle quand répondre
- **Stratégie de réponse** : Décide comment répondre

### Espace de Travail

L'espace de travail est là où OpenClaw stocke la configuration et les données :

- Emplacement par défaut : `~/.openclaw/workspace`
- Contient : AGENTS.md, SOUL.md, TOOLS.md
- Répertoire des compétences : `~/.openclaw/workspace/skills/`

## Configuration des Canaux

### Canaux Supportés

OpenClaw supporte ces plateformes de messagerie :

- Messagerie instantanée : WhatsApp, Telegram, Signal, iMessage
- Plateformes sociales : Discord, Slack
- Outils d'entreprise : Microsoft Teams, Google Chat, Feishu
- Autres : IRC, Matrix, LINE, Mattermost, Nostr, etc.

### Exemples de Configuration

**Telegram** :

```json
{
  "channels": {
    "telegram": {
      "botToken": "VOTRE_BOT_TOKEN"
    }
  }
}
```

**Discord** :

```json
{
  "channels": {
    "discord": {
      "token": "VOTRE_BOT_TOKEN"
    }
  }
}
```

**WhatsApp** :

```bash
openclaw channels login
```

## Système de Compétences

Les compétences sont la façon dont OpenClaw étend les fonctionnalités. Vous pouvez :

### Installer des Compétences

Installez des compétences de la communauté depuis ClawHub, ou créez les vôtres.

### Structure des Compétences

Chaque compétence est un répertoire contenant un fichier SKILL.md :

```
skills/
  └── ma-competence/
      └── SKILL.md
```

### ClawHub

ClawHub est un registre de compétences. Lorsqu'il est activé, OpenClaw peut automatiquement rechercher et installer les compétences nécessaires.

## Automatisation

### Tâches Cron

Configurez des tâches planifiées en utilisant Cron :

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhooks

Configurez des webhooks pour déclencher l'automatisation :

```bash
openclaw webhook add --url https://votre-endpoint.com/hook
```

### Intégration Gmail

Configurez Gmail Pub/Sub pour écouter les nouveaux e-mails.

## Fonctionnalités Avancées

### Contrôle du Navigateur

OpenClaw peut contrôler un navigateur dédié :

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Visualisation Canvas

Canvas est l'espace de travail visuel d'OpenClaw, supportant push/reset A2UI, évaluation et snapshots.

### Fonctionnalités Vocales

- **Voice Wake** : Activer via des mots d'activation sur macOS/iOS
- **Talk Mode** : Conversation vocale continue (support Android)

### Nœuds d'Appareils

OpenClaw peut se connecter à divers appareils :

- **Nœud macOS** : Exécution système, notifications, Canvas, caméra
- **Nœud iOS** : Canvas, voix, caméra, enregistrement d'écran
- **Nœud Android** : Connexion, chat, voix, Canvas, caméra, localisation, etc.

## Paramètres de Sécurité

### Pairing DM

Par défaut, les expéditeurs inconnus ont besoin d'une vérification de pairing. Changez en mode ouvert avec `dmPolicy: "open"` (non recommandé).

### Gérer le Pairing :

```bash
# Voir la liste des pairings
openclaw pairing list

# Approuver le pairing
openclaw pairing approve

# Révoquer le pairing
openclaw pairing revoke
```

### Exécuter les Diagnostiques

Exécutez régulièrement les diagnostiques pour la sécurité :

```bash
openclaw doctor
```

## Référence des Commandes

| Commande | Description |
|----------|-------------|
| `/status` | Voir le statut de la session |
| `/new` ou `/reset` | Réinitialiser la session |
| `/compact` | Compacter le contexte de session |
| `/think` | Définir le niveau de réflexion |
| `/verbose` | Basculer la sortie détaillée |
| `/usage` | Voir les statistiques d'utilisation |
| `/restart` | Redémarrer Gateway |

## Meilleures Pratiques

1. **Exécuter doctor régulièrement** : Garder le système en bonne santé
2. **Faire attention aux DM** : Vérifier les paramètres de sécurité
3. **Utiliser des modèles puissants** : Obtenir la meilleure expérience
4. **Sauvegarder régulièrement** : Sauvegarder les configurations importantes

## Dépannage

Problèmes courants :

- **Gateway ne démarre pas** : Vérifier le port 18789
- **Échec de connexion au canal** : Vérifier les identifiants
- **Échec d'envoi de message** : Vérifier les logs
