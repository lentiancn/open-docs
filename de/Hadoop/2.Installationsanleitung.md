# Hadoop Installationsanleitung

## Anforderungen

Bevor Sie Hadoop installieren, stellen Sie sicher, dass Ihr System die grundlegenden Hardware- und Softwareanforderungen erfüllt. Hadoop-Cluster können aus mehreren Computern bestehen oder im Pseudo-verteilten Modus ausgeführt werden. Verschiedene Bereitstellungsmodi haben unterschiedliche Hardwareanforderungen, aber die Kernsoftwareabhängigkeiten sind ähnlich. Das Verständnis dieser Anforderungen hilft bei der Planung der Größe und Konfiguration des Hadoop-Clusters und vermeidet unnötige Probleme während der Installation.

Für Hardware sollten kleine Testcluster mindestens 4 CPU-Kerne, 8 GB RAM und 500 GB Festplatte pro Maschine haben. Für Produktionsumgebungen sollte jede Maschine über 8 oder mehr CPU-Kerne, 16 GB oder mehr RAM und 1 TB oder größere Festplatte verfügen. Hadoop-DataNodes speichern große Datenmengen, daher ist die Festplattenkapazität besonders wichtig. Hadoop empfiehlt RAID-Konfiguration oder JBOD-Konfiguration (Just a Bunch Of Disks) – letztere ermöglicht es Hadoop, Speicherplatz flexibler zu verwalten.

Bei Betriebssystemen unterstützt Hadoop Linux (Ubuntu, CentOS, Debian, Red Hat usw.) und Windows (über Cygwin oder Windows Subsystem for Linux). Produktionsumgebungen empfehlen dringend Linux für beste Leistung und Stabilität. Hadoop ist auch gut in verschiedene Cloud-Plattformen integriert und kann auf AWS, Azure, Google Cloud und anderen Cloud-Diensten bereitgestellt werden.

Java ist die einzige Laufzeitabhängigkeit von Hadoop. Hadoop 3.x erfordert Java 8 oder höher. Oracle JDK oder OpenJDK wird empfohlen. Stellen Sie vor der Installation von Hadoop sicher, dass Java korrekt installiert ist und die JAVA_HOME-Umgebungsvariable gesetzt ist. Überprüfen Sie die Java-Installation mit:

```bash
java -version
echo $JAVA_HOME
```

Netzwerk ist grundlegend für den Hadoop-Cluster-Betrieb. Alle Knoten im Cluster sollten über das Netzwerk miteinander kommunizieren können, vorzugsweise mit schnellen Netzwerken (wie Gigabit-Ethernet). Knoten müssen SSH ohne Passwort konfiguriert haben, damit Hadoop Aufgaben zwischen Knoten verteilen und Daten übertragen kann. Konfigurieren Sie außerdem Hostname- und IP-Adresszuordnungen für alle Knoten in /etc/hosts.

## Installationsschritte

### Installation im Standalone-Modus

Der Standalone-Modus ist die einfachste Hadoop-Installationsmethode, bei der alle Komponenten im selben JVM-Prozess ausgeführt werden. Dieser Modus wird hauptsächlich für Entwicklung und Tests verwendet und beinhaltet keine verteilte Speicherung oder Berechnung. Laden Sie zunächst die binäre Distribution von Hadoop von der offiziellen Apache Hadoop-Website herunter:

```bash
# Hadoop herunterladen (durch neueste Version ersetzen)
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# In angegebenes Verzeichnis entpacken
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# Verzeichnis umbenennen (optional)
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

Konfigurieren Sie als Nächstes die Umgebungsvariablen. Bearbeiten Sie die Datei ~/.bashrc oder /etc/profile und fügen Sie Folgendes hinzu:

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # An tatsächlichen Java-Pfad anpassen
```

Wenden Sie die Konfiguration an:

```bash
source ~/.bashrc
```

Überprüfen Sie die Installation:

```bash
hadoop version
```

### Installation im Pseudo-verteilten Modus

Der Pseudo-verteilte Modus simuliert eine verteilte Umgebung auf einem einzelnen Computer, wobei NameNode, DataNode, ResourceManager und NodeManager alle auf demselben Computer ausgeführt werden. Dieser Modus eignet sich zum Entwickeln und Testen verteilter Funktionen.

Konfigurieren Sie zunächst die Hadoop-Kerndateien. Navigieren Sie zum Hadoop-Konfigurationsverzeichnis:

```bash
cd $HADOOP_HOME/etc/hadoop
```

Konfigurieren Sie core-site.xml und geben Sie HDFS als Standard-Dateisystem an:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Konfigurieren Sie hdfs-site.xml und geben Sie den HDFS-Replikationsfaktor an (1 für Pseudo-verteilt):

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

Konfigurieren Sie mapred-site.xml und geben Sie das Ressourcenplanungsframework für MapReduce an:

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

Konfigurieren Sie yarn-site.xml und legen Sie YARN-bezogene Eigenschaften fest:

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### SSH-Konfiguration ohne Passwort

Knoten im Hadoop-Cluster müssen SSH ohne Passwort konfiguriert haben, damit der Master-Knoten auf Slave-Knoten zugreifen kann:

```bash
# SSH-Schlüsselpaar generieren
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# Öffentlichen Schlüssel zu authorized_keys hinzufügen
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# Berechtigungen setzen
chmod 0600 ~/.ssh/authorized_keys

# SSH-Anmeldung testen
ssh localhost
```

Bei Multi-Node-Clustern kopieren Sie den öffentlichen Schlüssel auf alle Knoten:

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### HDFS formatieren und starten

Bevor Sie HDFS zum ersten Mal verwenden, formatieren Sie den NameNode:

```bash
hdfs namenode -format
```

Starten Sie HDFS und YARN:

```bash
start-dfs.sh
start-yarn.sh
```

Oder verwenden Sie start-all.sh, um alle Dienste auf einmal zu starten:

```bash
start-all.sh
```

Überprüfen Sie den Service-Startstatus:

```bash
# Java-Prozesse anzeigen
jps

# Erwartete Prozesse:
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Sie können auf Hadoop-Dienste über die Web-Benutzeroberfläche zugreifen: NameNode unter http://localhost:9870, ResourceManager unter http://localhost:8088.

## Vollständig verteilte Cluster-Installation

### Cluster-Planung

Der vollständig verteilte Modus erfordert mehrere physische oder virtuelle Maschinen, um einen Cluster zu bilden. Führen Sie vor der Installation eine vernünftige Cluster-Planung durch. Bestimmen Sie die Cluster-Größe (Anzahl der Knoten) und weisen Sie Rollen zu (NameNode, Secondary NameNode, DataNode, ResourceManager, NodeManager). Bei kleinen Clustern können NameNode und ResourceManager auf derselben Maschine bereitgestellt werden; bei großen Clustern wird eine separate Bereitstellung für bessere Leistung empfohlen.

Die Netzwerkplanung ist ebenfalls wichtig. Alle Knoten sollten in demselben internen Netzwerk mit niedriger Latenz und hoher Bandbreite sein. Konfigurieren Sie Hostnamen und IP-Adressen für alle Knoten in /etc/hosts:

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### Konfigurationsdateien ändern

Im vollständig verteilten Modus müssen mehr Konfigurationsdateien geändert werden. Konfigurieren Sie zuerst die Workers-Datei (Slaves-Datei in früheren Versionen) und listen Sie alle DataNode- und NodeManager-Knoten auf:

```
slave1
slave2
slave3
```

Die Konfiguration von core-site.xml und hdfs-site.xml ähnelt dem Pseudo-verteilten Modus, es müssen jedoch mehr Eigenschaften hinzugefügt werden. Für die NameNode HA-Konfiguration konfigurieren Sie zwei NameNodes:

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Konfigurieren Sie hdfs-site.xml für NameNode HA:

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

Die YARN-Konfiguration ist ähnlich; Sie können ResourceManager HA in yarn-site.xml konfigurieren.

### Konfigurationsdateien synchronisieren

Synchronisieren Sie das konfigurierte Hadoop-Installationsverzeichnis mit allen Knoten. Kopieren Sie einzeln mit scp:

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

Oder verwenden Sie rsync für die verteilte Replikation:

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### Cluster starten

Formatieren Sie den NameNode auf dem NameNode-Knoten (erster Start):

```bash
hdfs namenode -format
```

Starten Sie HDFS:

```bash
start-dfs.sh
```

Starten Sie YARN:

```bash
start-yarn.sh
```

Überprüfen Sie den Prozessstatus auf allen Knoten, um sicherzustellen, dass die Dienste ordnungsgemäß ausgeführt werden:

```bash
# Auf NameNode
jps

# Auf DataNode
jps
```

## Überprüfung und Tests

### HDFS-Test

Erstellen Sie Verzeichnisse und laden Sie Dateien hoch, um HDFS zu testen:

```bash
# Verzeichnis erstellen
hdfs dfs -mkdir -p /user/test

# Lokale Datei nach HDFS hochladen
hdfs dfs -put localfile.txt /user/test/

# Dateien auflisten
hdfs dfs -ls /user/test

# Datei lesen
hdfs dfs -cat /user/test/localfile.txt

# Datei löschen
hdfs dfs -rm /user/test/localfile.txt
```

### MapReduce-Test

Führen Sie MapReduce-Beispielprogramme aus, um die Cluster-Funktionalität zu überprüfen:

```bash
# WordCount-Beispiel ausführen
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# Ergebnisse anzeigen
hdfs dfs -cat /output/part-r-00000
```

### Cluster-Gesundheitsprüfung

Verwenden Sie die integrierten Hadoop-Befehle, um den Cluster-Status zu überprüfen:

```bash
# HDFS-Status anzeigen
hdfs dfsadmin -report

# YARN-Knotenstatus anzeigen
yarn node -list -all

# Anwendungsstatus anzeigen
yarn application -list
```

## Häufige Probleme

### Java-Versionsinkompatibilität

Wenn Sie Java-Versionsfehler feststellen, stellen Sie sicher, dass die installierte Java-Version mit der Hadoop-Version kompatibel ist. Hadoop 3.x erfordert Java 8 oder höher. Wechseln Sie die Java-Version mit:

```bash
update-alternatives --config java
```

### Unzureichender Speicherplatz

DataNodes funktionieren möglicherweise nicht richtig wegen unzureichendem Speicherplatz. Überwachen Sie regelmäßig die Festplattennutzung, um sicherzustellen, dass DataNodes ausreichend freien Speicherplatz haben. Konfigurieren Sie dfs.datanode.fsdataset.volume.choosing.policy in hdfs-site.xml als AvailableSpaceVolumeChoosingPolicy, damit DataNodes Festplatten mit mehr verfügbarem Speicherplatz bevorzugen.

### Unzureichender Speicher

Wenn Cluster-Knoten über unzureichenden Speicher verfügen, kann YARN möglicherweise keine Container starten. Passen Sie die Speicherkonfiguration in yarn-site.xml an:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
