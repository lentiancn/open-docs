# Angular FAQ

Questions fréquemment posées.

## Installation

### Q : L'installation d'Angular CLI a échoué ?

**Solutions** :
1. Vider le cache npm : `npm cache clean --force`
2. Utiliser les privilèges administrateur
3. Vérifier la version de Node.js

### Q : Commande ng non trouvée ?

**Solutions** :
1. Confirmer qu'Angular CLI est installé globalement
2. Vérifier la variable d'environnement PATH
3. Essayer de réinstaller

### Q : Version de Node.js incompatible ?

**Solutions** :
1. Utiliser nvm pour gérer les versions de Node.js
2. Mettre à jour Node.js vers la version recommandée

## Utilisation

### Q : Les styles du composant ne fonctionnent pas ?

**Solutions** :
1. Vérifier que le chemin styleUrls est correct
2. Confirmer que les sélecteurs CSS sont corrects
3. Vérifier les paramètres ViewEncapsulation

### Q : Le routage ne fonctionne pas ?

**Solutions** :
1. Vérifier que la configuration des routes est correcte
2. Confirmer que RouterModule est importé
3. Vérifier la syntaxe routerLink

### Q : Les requêtes HTTP échouent ?

**Solutions** :
1. Confirmer que HttpClientModule est importé
2. Vérifier la configuration CORS
3. Vérifier que l'URL de l'API est correcte

### Q : La validation du formulaire ne fonctionne pas ?

**Solutions** :
1. Confirmer que ReactiveFormsModule est importé
2. Vérifier la configuration des Validators
3. Utiliser correctement formControlName

## Performance

### Q : L'application charge lentement ?

**Solutions** :
1. Activer le chargement paresseux
2. Utiliser la détection de changement OnPush
3. Compresser les images et ressources
4. Builder en mode production

### Q : Fuites de mémoire ?

**Solutions** :
1. Se désabonner des observables
2. Nettoyer les intervalles
3. Détruire les écouteurs d'événements

## Version

### Q : Comment mettre à jour Angular ?

```bash
ng update @angular/core @angular/cli
```

### Q : Migrer d'AngularJS vers Angular ?

Utiliser l'outil angular-upgrade pour une migration progressive.
