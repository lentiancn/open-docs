# JIRA 使用指南

本指南介紹如何使用 JIRA 進行專案管理和問題追蹤。

## 快速開始

### 建立專案

1. 點擊「建立專案」
2. 選擇專案範本
3. 輸入專案名稱和金鑰
4. 點擊「建立」

## 專案類型

### 敏捷專案

```markdown
- Scrum 專案
- 看板專案
- 規模化敏捷
```

### 經典專案

```markdown
- 業務專案
- 專案管理
- 任務追蹤
```

## 問題類型

### 建立問題

```markdown
1. 點擊「建立問題」
2. 選擇專案
3. 選擇問題類型
4. 填寫標題和描述
5. 設定優先順序和元件
6. 指定給使用者
```

### 問題類型

- **Bug** - 軟體缺陷
- **Story** - 使用者故事
- **Task** - 任務
- **Epic** - 史詩
- **Sub-task** - 子任務

## 工作流程

### 預設工作流程

```
建立 → 指派 → 解決 → 關閉
```

### 敏捷工作流程

```
待辦 → 進行中 → 完成 → 審核
```

## 敏捷功能

### 看板

1. 建立看板
2. 新增欄位
3. 拖曳問題卡
4. 設定 WIP 限制

### Scrum

```markdown
- 衝刺規劃
- 每日站會
- 衝刺回顧
- 衝刺評審
```

## 欄位

### 自訂欄位

1. 進入管理 → 欄位
2. 新增自訂欄位
3. 設定欄位選項

### 欄位設定

```markdown
- 必填欄位
- 預設值
- 欄位權限
```

## 權限

### 專案權限

- 瀏覽專案
- 建立問題
- 編輯問題
- 刪除問題
- 指派問題

### 問題安全層級

- 公開
- 專案成員
- 特定使用者
- 特定群組

## 自動化

### 建立規則

1. 進入專案設定 → 自動化
2. 建立規則
3. 設定觸發條件
4. 設定操作

### 自動化範例

```
觸發條件: 問題建立
條件: 類型 = Bug
操作: 指派給 = 負責人
```
