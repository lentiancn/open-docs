# Webpack Benutzeranleitung

Diese Anleitung zeigt dir, wie du Webpack verwendest, um moderne JavaScript-Anwendungen zu erstellen.

## Schnellstart

### Projekt erstellen

```bash
mkdir my-project
cd my-project
npm init -y
npm install webpack webpack-cli --save-dev
```

### Eingabedateien erstellen

```javascript
// src/index.js
import './style.css';
import { add } from './math.js';

console.log('Hello Webpack!');
console.log(add(2, 3));
```

```javascript
// src/math.js
export function add(a, b) {
  return a + b;
}
```

```css
/* src/style.css */
body {
  background: #f0f0f0;
  font-family: sans-serif;
}
```

```html
<!-- src/index.html -->
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Webpack App</title>
</head>
<body>
  <h1>Hello Webpack!</h1>
  <script src="index.js"></script>
</body>
</html>
```

### Konfigurationsdatei erstellen

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
```

### Build```bash
n ausführen

px webpack
```

## Kernkonzepte

### Entry (Eingabe)

```javascript
module.exports = {
  // Einzelner Entry
  entry: './src/index.js',
  
  // Mehrere Entries
  entry: {
    home: './src/home.js',
    about: './src/about.js',
  },
};
```

### Output (Ausgabe)

```javascript
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    clean: true,
    publicPath: '/',
  },
};
```

### Loaders

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader',
      },
    ],
  },
};
```

### Plugins

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
    new MiniCssExtractPlugin({
      filename: '[name].css',
    }),
  ],
};
```

### Mode

```javascript
// Entwicklungsmodus
module.exports = {
  mode: 'development',
  devtool: 'source-map',
};

// Produktionsmodus
module.exports = {
  mode: 'production',
  devtool: false,
};
```

## Häufige Loader

### CSS verarbeiten

```bash
npm install --save-dev style-loader css-loader
```

```javascript
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader'],
}
```

### Sass verarbeiten

```bash
npm install --save-dev sass sass-loader
```

```javascript
{
  test: /\.scss$/,
  use: ['style-loader', 'css-loader', 'sass-loader'],
}
```

### Bilder verarbeiten

```javascript
{
  test: /\.(png|jpg|gif)$/,
  type: 'asset/resource',
}
```

### Schriften verarbeiten

```javascript
{
  test: /\.(woff|woff2|eot|ttf|otf)$/,
  type: 'asset/resource',
}
```

## Entwicklungsserver

### Installation

```bash
npm install --save-dev webpack-dev-server
```

### Konfiguration

```javascript
module.exports = {
  devServer: {
    static: './dist',
    hot: true,
    port: 3000,
    open: true,
    compress: true,
  },
};
```

### Starten

```bash
webpack serve --open
```

Oder Skripte zu package.json hinzufügen:

```json
{
  "scripts": {
    "start": "webpack serve --mode development",
    "build": "webpack --mode production"
  }
}
```

## Code-Splitting

### Dynamische Importe

```javascript
// Route Lazy Loading
const Home = () => import('./Home.vue');
const About = () => import('./About.vue');

// Bedingtes Laden
if (condition) {
  import('./module').then(module => {
    module.doSomething();
  });
}
```

### SplitChunks

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};
```

## Umgebungskonfiguration

### Entwicklungsumgebung

```javascript
module.exports = {
  mode: 'development',
  devtool: 'inline-source-map',
  devServer: {
    static: './dist',
    hot: true,
  },
};
```

### Produktionsumgebung

```javascript
module.exports = {
  mode: 'production',
  devtool: 'source-map',
  optimization: {
    minimize: true,
  },
};
```

### Multi-Umgebung Konfiguration

```javascript
const devConfig = require('./webpack.dev');
const prodConfig = require('./webpack.prod');

module.exports = (env, argv) => {
  return argv.mode === 'production' ? prodConfig : devConfig;
};
```

## Leistungsoptimierung

### 1. Production-Modus verwenden

```javascript
module.exports = {
  mode: 'production',
};
```

### 2. Tree Shaking aktivieren

```javascript
// package.json
{
  "sideEffects": false
}
```

### 3. Code-Splitting

```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
  },
}
```

### 4. Cache-Optimierung

```javascript
output: {
  filename: '[name].[contenthash].js',
  chunkFilename: '[name].[contenthash].chunk.js',
}
```

### 5. Komprimierung

```javascript
const TerserPlugin = require('terser-webpack-plugin');

optimization: {
  minimizer: [new TerserPlugin()],
}
```

## Häufige Probleme

### Modulauflösung fehlgeschlagen?

Überprüfe die resolve-Konfiguration:

```javascript
resolve: {
  extensions: ['.js', '.json'],
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
}
```

### Build zu langsam?

- Cache verwenden
- node_modules ausschließen
- thread-loader verwenden

```javascript
{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: 'thread-loader',
    options: {
      workers: 2,
    },
  },
}
```
