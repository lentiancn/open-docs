# OpenClaw 使用手冊

本手冊將幫助您了解如何有效地使用 OpenClaw。

## 基本使用

### 與 OpenClaw 對話

安裝完成後，您可以透過多種方式與 OpenClaw 互動：

1. **Control UI**：透過瀏覽器存取 `http://127.0.0.1:18789/`
2. **CLI 命令**：`openclaw agent --message "您的訊息"`
3. **已配置的渠道**：透過 WhatsApp、Telegram、Discord 等發送訊息

### 發送訊息

使用 CLI 發送訊息：

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## 核心概念

### Gateway

Gateway 是 OpenClaw 的核心控制平面，管理所有會話、渠道、工具和事件。預設監聽 `ws://127.0.0.1:18789`。

### Session（會話）

OpenClaw 使用會話模型來管理對話：

- **主會話**：直接對話
- **群組隔離**：群組訊息單獨處理
- **啟動模式**：控制何時回應
- **回覆策略**：決定如何回覆

### Workspace

工作區是 OpenClaw 儲存配置和資料的目錄：

- 預設位置：`~/.openclaw/workspace`
- 包含檔案：AGENTS.md、SOUL.md、TOOLS.md
- 技能目錄：`~/.openclaw/workspace/skills/`

## 渠道配置

### 支援的渠道

OpenClaw 支援以下通訊平台：

- 即時通訊：WhatsApp、Telegram、Signal、iMessage
- 社交平台：Discord、Slack
- 企業工具：Microsoft Teams、Google Chat、飛書
- 其他：IRC、Matrix、飛書 LINE、Mattermost、Nostr 等

### 配置範例

**Telegram**：

```json
{
  "channels": {
    "telegram": {
      "botToken": "您的Bot Token"
    }
  }
}
```

**Discord**：

```json
{
  "channels": {
    "discord": {
      "token": "您的Bot Token"
    }
  }
}
```

**WhatsApp**：

```bash
openclaw channels login
```

## 技能系統

技能是 OpenClaw 擴展功能的方式。您可以：

### 安裝技能

從 ClawHub 安裝社群技能，或建立自己的技能。

### 技能結構

每個技能是一個包含 SKILL.md 檔案的目錄：

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHub 是技能註冊中心。啟用後，OpenClaw 可以自動搜尋和安裝需要的技能。

## 自動化

### 定時任務

使用 Cron 設定定時執行的任務：

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhook

設定 Webhook 來觸發自動化：

```bash
openclaw webhook add --url https://your-endpoint.com/hook
```

### Gmail 整合

設定 Gmail Pub/Sub 來監聽新郵件。

## 進階功能

### 瀏覽器控制

OpenClaw 可以控制專用瀏覽器：

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvas 視覺化

Canvas 是 OpenClaw 的視覺化工作空間，支援 A2UI 推送/重置、評估和快照。

### 語音功能

- **Voice Wake**：在 macOS/iOS 上透過喚醒詞激活
- **Talk Mode**：連續語音對話（Android 支援）

### 裝置節點

OpenClaw 可以連接多種裝置：

- **macOS 節點**：系統執行、通知、Canvas、攝影機
- **iOS 節點**：Canvas、語音、攝影機、螢幕錄製
- **Android 節點**：連線、聊天、語音、Canvas、攝影機、位置等

## 安全設定

### DM 配對

預設情況下，未知傳送者需要配對驗證才能與 OpenClaw 對話。使用 `dmPolicy: "open"` 可改為開放模式，但不推薦。

### 管理配對

```bash
# 查看配對清單
openclaw pairing list

# 批准配對
openclaw pairing approve

# 撤銷配對
openclaw pairing revoke
```

### 執行診斷

定期執行診斷檢查安全問題：

```bash
openclaw doctor
```

## 命令參考

| 命令 | 說明 |
|------|------|
| `/status` | 查看會話狀態 |
| `/new` 或 `/reset` | 重置會話 |
| `/compact` | 壓縮會話上下文 |
| `/think` | 設定思考級別 |
| `/verbose` | 開關詳細輸出 |
| `/usage` | 查看使用統計 |
| `/restart` | 重新啟動 Gateway |

## 最佳實踐

1. **定期執行 doctor**：保持系統健康
2. **謹慎開放 DM**：注意安全設定
3. **使用強模型**：獲得最佳體驗
4. **定期備份**：儲存重要配置

## 疑難排解

常見問題解決：

- **Gateway 不啟動**：檢查連接埠佔用
- **渠道連線失敗**：檢查憑證配置
- **訊息發送失敗**：查看日誌排查
