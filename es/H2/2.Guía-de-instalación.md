# Guía de instalación de H2

## Requisitos del entorno

- Java 11 o superior
- Cualquier sistema operativo que soporte Java (Windows, Linux, macOS, etc.)

## Métodos de instalación

### Método 1: Descargar paquete ZIP

1. Visita la página de descarga del sitio web oficial: https://www.h2database.com/html/download.html
2. Descarga la versión "Platform-Independent Zip"
3. Descomprime el archivo ZIP descargado en cualquier directorio, por ejemplo: C:\h2 o /opt/h2

### Método 2: Dependencia Maven

Si tienes un proyecto Java usando Maven o Gradle, puedes agregar la dependencia directamente:

**Maven:**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle:**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### Método 3: Instalador de Windows

1. Descarga la versión "Windows Installer"
2. Ejecuta el instalador y sigue el asistente para completar la instalación

## Configurar variables de entorno (opcional)

Si descargaste el paquete ZIP y deseas usar H2 directamente desde la línea de comandos:

**Windows:**
1. Haz clic derecho en "Este equipo" → "Propiedades"
2. Haz clic en "Configuración avanzada del sistema"
3. Haz clic en "Variables de entorno"
4. En las variables del sistema, encuentra Path y haz clic en "Editar"
5. Agrega la ruta del directorio bin de H2

**Linux/macOS:**
```bash
export PATH=$PATH:/opt/h2/bin
```

## Verificar instalación

### Usando línea de comandos

Después de extraer, ejecuta en la línea de comandos:

```bash
java -jar h2.jar
```

Esto arrancará la interfaz de consola web de H2.

### Usando conexión JDBC

Conectar a la base de datos H2 en código Java:

```java
// Modo embebido
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// Base de datos de archivo
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Usando la consola H2

1. Ejecuta el comando: `java -jar h2.jar`
2. Abre el navegador y accede a: http://localhost:8082
3. Nombre de usuario predeterminado: sa, contraseña vacía

## Formatos de URL de conexión

H2 soporta múltiples formatos de URL de conexión:

| Modo | URL de ejemplo | Descripción |
|------|---------------|-------------|
| Modo memoria | jdbc:h2:mem:testdb | Datos almacenados en memoria |
| Archivo embebido | jdbc:h2:file:./data/testdb | Datos almacenados en archivo |
| Servidor TCP | jdbc:h2:tcp://localhost:9092/testdb | Modo cliente-servidor |
| Cifrado TCP | jdbc:h2:ssl://localhost:9092/testdb | Conexión cifrada SSL |
| Compatibilidad PostgreSQL | jdbc:h2:postgres://localhost:5432/testdb | Modo compatibilidad protocolo PostgreSQL |

## Problemas comunes

### Puerto ocupado
H2 usa el puerto 9092 por defecto (modo TCP) y 8082 (consola web). Si el puerto está ocupado, puedes especificar otro puerto en la URL de conexión.

### Problema de versión Java
Asegúrate de tener Java 11 o superior instalado. Puedes verificar con el siguiente comando:

```bash
java -version
```

### Controlador no encontrado
Asegúrate de que el archivo JAR de H2 esté en el classpath. Si es un proyecto Maven, verifica que la dependencia se haya agregado correctamente.
