# Nacos インストールガイド

## 環境要件

NacosはJavaランタイム環境で動作します。以下のバージョンで使用してください。

- 64ビットOS：Linux/Unix/Mac/Windows対応、Linux/Unix/Mac推奨
- 64ビットJDK 17+（JDK 17以上を推奨）
- JDKダウンロード：https://www.oracle.com/java/technologies/downloads/#java17
- JDK環境設定：https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## 推奨構成

Nacosの実行には、以下のマシン構成以上を推奨します。

- CPU：2コア
- メモリ：4GB
- ディスク：60GB

## インストール方法

Nacosは複数のインストール方法をサポートしています。ニーズに応じた方法を選択してください。

### 方法1：nacos-setupを使用したワンクリック展開（推奨）

nacos-setupは、ダウンロード、認証設定、ポート検出、Java環境検証を自動的に処理するワンクリック自動化展開を提供します。

#### nacos-setupのインストール

Linux/macOS：

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows（PowerShell）：

```powershell
# PowerShellで実行
```

#### Nacosサービスの起動

的最新バージョンのスタンドアロンNacosをローカルにワンクリック展開：

```bash
nacos-setup
```

特定のバージョンのスタンドアロンNacosをローカルに展開：

```bash
nacos-setup -v 3.1.1
```

最新バージョンのクラスターNacosをローカルに展開：

```bash
nacos-setup -c cluster
```

外部ストレージをグローバルに設定し、データベースアドレス、ポート、ユーザー名、パスワードを入力してからnacos-setupを再実行：

```bash
nacos-setup --datasource-conf
```

その他の使用方法は、nacos-setupユーザーマニュアルを参照してください。

### 方法2：發行パッケージの手動ダウンロードと展開

#### インストールパッケージのダウンロード

Nacos發行パッケージを入手方法は2つあります。

**方法1：Nacos公式サイトからダウンロード**

Nacos公式サイト versionsダウンロードページにアクセスし、稳定版を選択して、バイナリパッケージ列のzipファイルをダウンロードします。

**方法2：GitHubからダウンロード**

Nacos GitHub Releasesページにアクセスし、対応するバージョンの發行パッケージをダウンロードします。

注意：多くのユーザーが同時にダウンロードすると、ダウンロード流量制限が発生する場合があります。その場合はしばらくしてから再試行するか、GitHubからダウンロードしてください。

#### Nacos發行パッケージの展開

Linux/Unix/Mac：

```bash
unzip nacos-server-$version.zip
# または
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows：

展開ツールを使用してnacos-server-$version.zipを直接展開します。

#### サーバーの起動

Linux/Unix/Mac起動コマンド（standaloneは非クラスターモードを意味します）：

```bash
sh startup.sh -m standalone
```

Ubuntuシステムを使用している場合、またはシンボルが見つからないというエラーが発生した場合は、次を試してください：

```bash
bash startup.sh -m standalone
\]

起動プログラムは、3つの認証関連設定の入力を求めます（Nacos 3.0.0以降ではコンソール認証がデフォルトで有効になっているため、以下の3つの認証設定を入力する必要があります）：

```
nacos.core.auth.plugin.nacos.token.secret.keyが欠けているので、Base64文字列で設定してください：${your_input_token_secret_key}
nacos.core.auth.server.identity.keyが欠けているので、設定してください：${your_input_server_identity_key}
nacos.core.auth.server.identity.valueが欠けているので、設定してください：${your_input_server_identity_key}
```

Windows起動コマンド：

```bash
cmd startup.cmd
```

またはstartup.cmdをダブルクリックして実行します。

### 方法3：Dockerを使用したインストール

#### Nacosイメージの取得

```bash
docker pull nacos/nacos-server
```

#### スタンドアロンモードの実行

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### クラスターモードの実行

Docker ComposeまたはKubernetesを使用したクラスター展開が必要です。

### 方法4：Kubernetesへのインストール

NacosはKubernetes環境への展開をサポートしており、Helm ChartまたはOperatorを使用してインストールできます。具体的な手順については、Nacos Kubernetes展開ドキュメントを参照してください。

## Nacosサービスの起動確認

Nacosログディレクトリに移動し、tail -f start.outを使用してログを表示します。以下のメッセージが表示されれば、サービスは正常に起動しています：

```
Nacos started successfully in stand alone mode. use embedded storage
```

## クイック検証

### Nacosコンソール

任意のブラウザを開き、アドレス：http://127.0.0.1:8080/index.htmlにアクセスすると、Nacosコンソールに入ります。

注意：初めて開くと、ユーザーnacosの管理者パスワードの初期化が必要になります。デフォルトのユーザー名/パスワード：nacos/nacos@demo。

### サービスの登録

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### サービスの発見

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### 構成の公開

最初にログインしてaccess tokenを取得：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

access tokenを使用して構成を作成：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### 構成の取得

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## サーバーの停止

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

またはshutdown.cmdをダブルクリックして実行します。

## 注意事項

1. Nacos 3.0.0以降ではコンソール認証がデフォルトで有効になっているため、初回の起動時に認証関連のパラメータを設定する必要があります。
2. クラスターモードの展開には、外部データベース（MySQLなど）を使用したデータストレージが必要です。
3. 本番環境では高可用性を確保するためにクラスターモードを使用することをお勧めします。
4. ファイアウォールで必要なポート（デフォルト8848、9848）が開いていることを確認してください。
5. 最適なパフォーマンスを得るために、JDK 17以上を使用することをお勧めします。
