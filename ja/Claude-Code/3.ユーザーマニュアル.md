# Claude Code ユーザーマニュアル

このマニュアルでは、Claude Codeのクイックスタートと日常的な操作について説明します。

## 準備

始める前に、次の条件を満たしていることを確認してください：

- ターミナルまたはコマンドプロンプトを開いている
- 作業するコードプロジェクトがある
- Claudeサブスクリプション（Pro、Max、Teams、またはEnterprise）、Claude Consoleアカウント、またはサポートされているクラウドプロバイダーへのアクセスがある

## セッションの開始

プロジェクトのディレクトリでターミナルを開き、Claude Codeを起動します：

```bash
cd /path/to/your/project
claude
```

初めて使用すると、セッション情報、最近の会話、最新の更新が表示されたウェルカム画面が表示されます。利用可能なコマンドを表示するには`/help`を入力し、以前の会話を続けるには`/resume`を入力します。

## 基本的な操作

### コードベースを理解する

Claudeにプロジェクトを分析させます：

```
このプロジェクトは何をしますか？
```

```
このプロジェクトでは 어떤 기술을 사용합니까?
```

```
メインのエントリーポイントはどこですか？
```

```
フォルダ構造を説明してください
```

Claudeにその能力について聞くこともできます：

```
Claude Codeは何ができますか？
```

```
Claude Codeでカスタムスキルを作成するにはどうすればいいですか？
```

**注意：Claude Codeは必要なときにプロジェクトのファイルを読み取ります。手動でコンテキストを追加する必要はありません。**

### 最初のコード変更

Claude Codeに実際のコーディング 작업을してもらいましょう。簡単なタスクを試してみます：

```
メインファイルにhello world関数を追加してください
```

Claude Codeは：
1. 適切なファイルを見つける
2. 提案された変更を表示する
3. 承認を求める
4. 編集を実行する

**注意：Claude Codeは常にファイルを修正する前に許可を求めます。单个の変更を承認するか、セッションで「すべて接受」モードを有効にすることができます。**

### Git操作

Claude CodeでGit操作が簡単になります：

```
哪些ファイルを変更しましたか？
```

```
説明的なメッセージで変更をコミットしてください
```

```
feature/quickstartという名前の新しいブランチを作成
```

```
最近の5件のコミットを表示
```

```
マージコンフリクトの解決を手伝ってください
```

### バグ修正と機能追加

Claudeはデバッグと機能実装が得意です。

自然言語で 원하는 것을 설명します：

```
ユーザー登録フォームに入力のバリデーションを追加
```

または既存の 문제를 수정합니다：

```
ユーザーが空のフォームを送信できるバグがあります - 修正してください
```

Claude Codeは：
- 相关コードを見つける
- コンテキストを理解する
- ソリューションを実装する
- テストを実行する（ある場合）

### 一般的なワークフロー

**コードのリファクタリング**

```
認証モジュールをコールバックの代わりにasync/awaitを使用するようにリファクタリング
```

**テストの作成**

```
計算機関数のユニットテストを作成
```

**ドキュメントの更新**

```
インストール手順でREADMEを更新
```

**コードレビュー**

```
変更をレビューして改善を提案
```

## 重要なコマンド

日常的に使用する重要なコマンド：

| コマンド | 機能 | 例 |
|----------|------|------|
| `claude` | インタラクティブモードを開始 | `claude` |
| `claude "タスク"` | ワ一次性タスクを実行 | `claude "fix the build error"` |
| `claude -p "クエリ"` | 一次性クエリを実行して終了 | `claude -p "explain this function"` |
| `claude -c` | 現在のディレクトリで最近の会話を続ける | `claude -c` |
| `claude -r` | 以前の会話を再開 | `claude -r` |
| `claude commit` | Gitコミットを作成 | `claude commit` |
| `/clear` | 会話履歴をクリア | `/clear` |
| `/help` | 利用可能なコマンドを表示 | `/help` |
| `exit`またはCtrl+C | Claude Codeを終了 | `exit` |

## 上級ヒント

### CLAUDE.mdの使用

プロジェクトのルートに`CLAUDE.md`ファイルを作成して、Claude Codeに永続的なプロジェクトコンテキストを提供します。このファイルはすべてのセッションの最初に自動的に読み込まれます。

CLAUDE.mdには以下を含めることができます：
- コーディング標準と仕様
- アーキテクチャ決定事項
- 推奨ライブラリ
- レビューチェックリスト
- プロジェクト固有のコマンドとスクリプト

### カスタムスキル

反復可能なワークフローをパッケージ化するカスタムコマンドを作成します（例：`/review-pr`、`/deploy-staging`）。

### Hooksの使用

Hooksを使用すると、Claude Codeのアクションの前後にシェルコマンドを実行できます：
- ファイル編集後の自動フォーマット
- コミット前のlint実行

### MCPサーバー

Model Context Protocolを使用してClaude Codeを外部ツールとデータソースに接続：
- Google Driveの設計ドキュメントを読み取り
- Jiraのチケットを更新
- Slackからデータを取得
- カスタムツールを使用

### マルチエージェントコラボレーション

複雑なタスクには、複数のClaude Codeエージェントを同時に起動して並列作業させることができます。リードエージェントが作業を調整し、サブタスクを割り当て、結果をマージします。

### リモートコントロール

作業は1つのデバイスに限定されません：
- リモートコントロールを使用して電話番号や他のデバイスからローカルセッションを継続
- WebまたはiOSアプリケーションで長時間実行されるタスクを開始し、`/teleport`でターミナルに引き戻す
- `/desktop`を使用してターミナルセッションをデスクトップアプリケーションに切り替え、視覚的なdiffレビューを実行

## ベストプラクティス

### 要求は具体的に

❌ 言わない：「バグを修正」

✅ 言う：「誤った認証情報を入力した後、ユーザーが空白の画面を見るログインのバグを修正」

### ステップバイステップの指示を使用

複雑なタスクをステップに分割：

```
1. ユーザープロファイルの新しいデータベーステーブルを作成
2. ユーザープロファイルの取得と更新を行うAPIエンドポイントを作成
3. ユーザーが情報を表示・編集できる网页を構築
```

### Claudeに先に探検させる

変更を加える前に、Claudeにコードを理解させます：

```
データベーススキーマを分析
```

```
英国顧客によって最も頻繁に返される製品を示すダッシュボードを構築
```

### ショートカットで時間を節約

- `?`を押してすべてのキーボードショートカットを表示
- Tabでコマンド補完を使用
- `↑`でコマンド履歴を表示
- `/`を入力してすべてのコマンドとスキルを表示

## よくある質問

**Claude Codeは私のコードを読みますか？**

はい、Claude Codeはコードベースを理解するためにプロジェクトのファイルを読み取ります。（サードパーティAPIプロバイダーを使用しない限り）外部サーバーにコードを送信ことはありません。

**Claude Codeは自動的にファイルを修正しますか？**

いいえ。Claude Codeは常にファイルを修正する前に入手许否を求めます。单个の変更を承認するか、セッション全体に対して「すべて接受」モードを有効にすることができます。

**私のコードは安全ですか？**

Claude Codeはセキュリティを念頭に置いて設計されています。モデルはトレーニングのためにAnthropicにコードを送信ことはありません（明示的にオプトインしない限り）。[Anthropicプライバシーポリシー](https://www.anthropic.com/privacy)で詳細をご覧ください。
