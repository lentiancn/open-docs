# Часто задаваемые вопросы

В этом документе собраны распространённые технические вопросы и ответы по использованию Angular.

## Базовые вопросы

### Что такое Angular?

Angular — это открытый фреймворк для веб-приложений, разработанный и поддерживаемый компанией Google. Он используется для создания современных веб-приложений и применяет TypeScript как основной язык разработки. Angular предоставляет компонентную архитектуру, двустороннее связывание данных, внедрение зависимостей и другие возможности.

### В чём разница между Angular и AngularJS?

AngularJS — это название Angular версии 1.x, то есть предшествующая версия фреймворка. Angular — это полностью переписанная версия, использующая TypeScript и имеющая совершенно другую архитектуру. Angular несовместим с AngularJS.

### Сложно ли изучить Angular?

У Angular есть определённая кривая обучения, особенно такие концепции, как внедрение зависимостей и RxJS, требуют времени для понимания. Однако документация фреймворка очень подробная, сообщество активно помогает, а после освоения базовых понятий разработка становится очень эффективной.

## Вопросы по установке

### Что делать, если npm install не удаётся?

1. Очистите кэш npm: `npm cache clean --force`
2. Удалите node_modules и package-lock.json
3. Используйте зеркало для России
4. Проверьте совместимость версии Node.js

### Команда Angular CLI не находится?

Убедитесь, что установлен глобально:
```bash
npm install -g @angular/cli
```

Если не помогает, попробуйте использовать npx:
```bash
npx @angular/cli new my-app
```

### Несовместимая версия Node.js?

Используйте nvm для управления версиями Node.js:
```bash
nvm install 18
nvm use 18
```

## Вопросы по разработке

### Стили компонента не применяются?

1. Проверьте правильность пути к файлу стилей
2. Убедитесь, что в styles.scss нет синтаксических ошибок
3. Проверьте настройку ViewEncapsulation

### Маршрутизация не работает?

1. Проверьте правильность настройки маршрутов
2. Убедитесь, что RouterModule импортирован корректно
3. Проверьте, не блокирует ли охранный маршрут переход

### HTTP-запрос не проходит?

1. Проверьте, запущен ли сервер
2. Убедитесь в правильной настройке CORS
3. Используйте HttpInterceptor для обработки ошибок

### Данные не обновляются?

1. Проверьте стратегию обнаружения изменений
2. При использовании OnPush нужно вызывать обнаружение вручную
3. Убедитесь, что ссылка на данные изменилась

## Вопросы по производительности

### Медленная первоначальная загрузка?

1. Включите ленивую загрузку маршрутов
2. Используйте стратегию предзагрузки
3. Оптимизируйте изображения
4. Включите режим production при сборке

### Утечки памяти?

1. Отписывайтесь от Observable в ngOnDestroy
2. Используйте takeUntil или асинхронный канал
3. Очищайте таймеры и обработчики событий

### Частое обнаружение изменений?

1. Используйте стратегию OnPush
2. Разделяйте изменяемые и неизменяемые свойства
3. Применяйте trackBy для оптимизации *ngFor

## Вопросы по тестированию

### Модульные тесты падают?

1. Проверьте настройку TestBed
2. Убедитесь, что асинхронные операции обработаны правильно
3. Используйте fakeAsync и tick для тестирования асинхронного кода

### Модуль не находится?

Убедитесь, что в tsconfig.json правильно настроены paths.

## Вопросы по сборке

### Сборка не проходит?

1. Прочитайте конкретное сообщение об ошибке
2. Проверьте настройки компилятора TypeScript
3. Очистите кэш и пересоберите проект

### Файлы сборки слишком большие?

1. Включите режим production
2. Используйте ленивую загрузку
3. Проверьте наличие неиспользуемых зависимостей

## Распространённые ошибки

### ExpressionChangedAfterItHasBeenCheckedError

Во время проверки изменилось значение, которое не должно было меняться. Проверьте, нет ли в ngOnInit кода, который напрямую изменяет связанные с представлением данные.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

Та же ошибка, что описана выше. Проверьте тайминг привязки данных.

### Cannot read property of undefined

Производится доступ к данным, которые ещё не инициализированы. Используйте опциональную цепочку (?.) или *ngIf для защиты.

### Ошибка CORS

На сервере не настроен CORS или есть проблемы с заголовками запроса.

### Ошибки 401/403

Ошибка аутентификации. Проверьте токен и настройки прав доступа.

## Обновление версии

### Как обновить Angular?

1. Используйте CLI для обновления: `ng update @angular/core`
2. Прочитайте официальное руководство по миграции
3. Обновите зависимости по очереди
4. Запустите тесты, чтобы убедиться в работоспособности

### Обновление не удалось?

1. Сделайте резервную копию кода
2. Вручную обновите версии в package.json
3. Удалите node_modules и установите заново
