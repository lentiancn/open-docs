# Vue.js 사용 매뉴얼

## 애플리케이션 인스턴스

모든 Vue 애플리케이션은 `createApp` 함수를 사용하여 새로운 애플리케이션 인스턴스를 생성하는 것으로 시작합니다:

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* 루트 컴포넌트 옵션 */
})
```

### 루트 컴포넌트
`createApp`에 전달하는 객체는 실제로 컴포넌트입니다. 모든 앱에는 다른 컴포넌트를 자식으로 포함할 수 있는 "루트 컴포넌트"가 필요합니다.

### 앱 마운팅
애플리케이션 인스턴스는 `.mount()` 메서드가 호출될 때까지 아무것도 렌더링하지 않습니다:

```javascript
app.mount('#app')
```

### 앱 구성
애플리케이션 인스턴스는 앱 수준에서 몇 가지 옵션을 구성할 수 있는 `.config` 객체를 노출합니다:

```javascript
// 앱 수준 오류 처리기
app.config.errorHandler = (err) => {
  /* 오류 처리 */
}

// 글로벌 컴포넌트 등록
app.component('TodoDeleteButton', TodoDeleteButton)
```

### 여러 애플리케이션 인스턴스
동일한 페이지에서 여러 Vue 애플리케이션을 공존시킬 수 있습니다:

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## 템플릿 문법

Vue는 렌더링된 DOM을 기본 컴포넌트 인스턴스의 데이터에 선언적으로 바인딩할 수 있는 HTML 기반 템플릿 문법을 사용합니다.

### 텍스트 보간
데이터 바인딩의 가장 기본적인 형태는 "Mustache" 문법(이중 중괄호)을 사용한 텍스트 보간입니다:

```html
<span>Message: {{ msg }}</span>
```

### 원시 HTML
실제 HTML을 출력하려면 `v-html` 디렉티브를 사용해야 합니다:

```html
<p v-html="rawHtml"></p>
```

**보안 경고**: 웹사이트에서 임의의 HTML을 동적으로 렌더링하는 것은 매우 위험할 수 있으며 XSS 취약점으로 쉽게 이어질 수 있습니다. `v-html`은 신뢰할 수 있는 콘텐츠에만 사용하고 절대 사용자 제공 콘텐츠에는 사용하지 마세요.

### 속성 바인딩
HTML 속성 내에서는 Mustache를 사용할 수 없습니다. 대신 `v-bind` 디렉티브를 사용해야 합니다:

```html
<!-- 전체 문법 -->
<div v-bind:id="dynamicId"></div>

<!-- 축약 문법 -->
<div :id="dynamicId"></div>

<!-- 동일 이름 축약 (Vue 3.4+) -->
<div :id></div>
```

### 불리언 속성
불리언 속성은 바인딩된 값이 참일 경우 요소에 포함됩니다:

```html
<button :disabled="isButtonDisabled">Button</button>
```

### 여러 속성 동적 바인딩
인수 없이 `v-bind`를 사용하여 단일 요소에 여러 속성을 바인딩할 수 있습니다:

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript 표현식

Vue는 모든 데이터 바인딩에서 완전한 JavaScript 표현식을 지원합니다:

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### 함수 호출
바인딩 표현식 내에서 컴포넌트가 노출한 메서드를 호출할 수 있습니다:

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### 제한된 전역 접근
템플릿 표현식은 샌드박스화되어 [제한된 전역 객체 목록](https://github.com/vuejs/core/blob/main/packages/shared/src/globalsAllowList.ts#L3)에만 접근할 수 있습니다. 이 목록에는 Math 및 Date와 같은 일반적으로 사용되는 내장 전역 객체가 노출됩니다.

목록에 명시적으로 포함되지 않은 전역 객체(예: window에 사용자가 연결한 속성)는 템플릿 표현식에서 접근할 수 없습니다. 그러나 [app.config.globalProperties](/api/application#app-config-globalproperties)에 추가하여 모든 Vue 표현식에서 사용할 수 있도록 명시적으로 추가할 수 있습니다.

## 디렉티브

디렉티브는 `v-` 접두사를 가진 특수 속성입니다.

### 인수
일부 디렉티브는 디렉티브 이름 뒤에 콜론으로 구분된 "인수"를 가질 수 있습니다. 예를 들어, `v-bind` 디렉티브는 HTML 속성을 반응적으로 업데이트하는 데 사용됩니다:

```html
<!-- v-bind 인수 -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on 인수 -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### 동적 인수
디렉티브 인수에서 JavaScript 표현식을 사용하려면 대괄호로 감싸야 합니다:

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**제약사항**:
- 동적 인수는 문자열 또는 null로 평가되어야 합니다
- 복잡한 표현식은 피하고 계산된 속성을 사용하세요
- DOM 내 템플릿에서는 동적 키에 대문자를 사용하지 마세요

### 수식어
수식어는 디렉티브가 특별한 방식으로 바인딩되어야 함을 나타내는 점으로 표시되는 특수 후위입니다. 예를 들어, `.prevent` 수식어는 트리거된 이벤트에 대해 `event.preventDefault()`를 호출하도록 `v-on` 디렉티브에 지시합니다:

```html
<form @submit.prevent="onSubmit">...</form>
```

## 반응성 기초

### 반응성 상태 선언
`ref()`를 사용하여 반응성 참조를 생성합니다:

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### 반응성 객체
`reactive()`를 사용하여 반응성 객체를 생성합니다:

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### 계산된 속성
`computed()`를 사용하여 계산된 속성을 생성합니다:

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### 감시자
`watch()`를 사용하여 반응성 데이터 변경을 감시합니다:

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// 단일 ref 감시
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## 컴포넌트 기초

### 컴포넌트 정의
```javascript
// 컴포넌트 정의
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Props 전달
```javascript
// 자식 컴포넌트
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// 부모 컴포넌트
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### 이벤트 발생
```javascript
// 자식 컴포넌트
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### 슬롯
```html
<!-- 자식 컴포넌트 -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- 부모 컴포넌트 -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
