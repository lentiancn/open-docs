# Jenkins 자주 묻는 질문

## 기본 질문

### Jenkins란?

Jenkins는 주로 지속적인 통합(CI) 및 지속적인 배포(CD)에 사용되는 오픈소스 자동화 서버입니다. 소프트웨어 빌드, 테스트 및 배포 프로세스를 자동화하여 개발 효율성과 품질을 향상시키는 데 도움이 됩니다.

### Jenkins는 무료인가요?

네, Jenkins는 완전히 무료한 오픈소스 프로젝트입니다. MIT 라이선스를 채택하여 누구나 자유롭게 사용, 수정 및 배포할 수 있습니다.

### Jenkins에 필요한 실행 환경은?

Jenkins에는 Java 8 이상이 필요합니다. Windows, Linux, macOS를 포함한 여러 운영 체제에서 실행될 수 있습니다. 최소 1GB 메모리 및 10GB 디스크 공간이 권장됩니다.

### Jenkins와 Hudson의 차이점은?

Jenkins는 Hudson의 포크입니다. 2011년 Oracle과의 상표 분쟁 이후 Hudson의 주요 개발 팀이 Hudson을 포크하고 Jenkins를 만들었습니다. 현재 Jenkins가 활성 프로젝트이고 Hudson은 중단되었습니다.

## 설치 관련 질문

### 첫 번째 시작 후 액세스할 수 없습니다. 어떻게 해야 하나요?

1. Jenkins가 제대로 시작되었는지 확인: 로그 확인
2. 포트가 사용 중이 아닌지 확인: 기본 포트는 8080
3. 방화벽 설정 확인: 포트 액세스 허용
4. 시작 로그의 오류 메시지 확인

### 잠금 해제 비밀번호는 어디에 있나요?

첫 설치 시 잠금 해제 비밀번호는 Jenkins 콘솔 로그에 표시됩니다. Docker로 실행하는 경우 `docker logs jenkins-blueocean`을 사용하여 확인합니다. 비밀번호 파일은 일반적으로 `/var/jenkins_home/secrets/initialAdminPassword`에 있습니다.

### 플러그인 설치가 실패했습니다. 어떻게 해야 하나요?

1. 네트워크 연결 확인: Jenkins가 업데이트 센터에 액세스해야 함
2. 수동 설치: 고급 탭에서 플러그인 hpi 파일 업로드 시도
3. Jenkins 미러 사용: 국내 업데이트 소스 구성
4. 로그 확인: 구체적인 실패 원인 파악

### Jenkins 포트를 변경하려면?

- Docker 방법: -p 매개변수 수정
- WAR 방법: `--httpPort` 사용
- Linux 패키지: /etc/default/jenkins의 HTTP_PORT 수정

## 자격 증명 관련 질문

### 자격 증명이 작동하지 않습니다. 어떻게 해야 하나요?

1. 자격 증명 ID가 올바른지 확인
2. 자격 증명 유형이 일치하는지 확인
3. 자격 증명이 만료되지 않았는지 확인
4. 사용자 권한 확인

### 비밀번호를 안전하게 저장하려면?

Jenkins 자격 증명 저장 기능 사용:

1. Manage Credentials로 이동
2. 자격 증명 추가, 유형 선택
3. Jenkins가 암호화된 저장소, 일반 텍스트로 표시 안 함

### SSH 키 인증이 실패했습니다. 어떻게 해야 하나요?

1. 공개 키가 대상 서버에 추가되었는지 확인
2. 키 권한 확인: 개인 키 파일 권한은 600이어야 함
3. 키 형식이 올바른지 확인
4. 키 쌍 다시 생성 시도

## 파이프라인 관련 질문

### Jenkinsfile 문법 오류는 어떻게 해결하나요?

1. 선언적 파이프라인 검증기 사용
2. Jenkins 콘솔 오류 메시지 확인
3. 로컬에서 Jenkins 시뮬레이터 사용
4. 공식 문법 문서 참조

### 파이프라인 실행이 실패했습니다. 어떻게 해야 하나요?

1. 자세한 오류 정보를 위해 빌드 로그 확인
2. 각 스테이지의 반환 상태 확인
3. 필요한 도구가 사용 가능한지 확인
4. 환경 변수 및 자격 증명 검증

### 병렬 실행을 구현하려면?

선언적 파이프라인은 병렬 문법을 지원합니다:

```groovy
pipeline {
    agent any
    stages {
        stage('병렬 빌드') {
            parallel {
                stage('모듈 A') {
                    steps {
                        sh './build-a.sh'
                    }
                }
                stage('모듈 B') {
                    steps {
                        sh './build-b.sh'
                    }
                }
            }
        }
    }
}
```

### 수동 확인을 기다리려면?

input 스텝 사용:

```groovy
stage('프로덕션에 배포') {
    steps {
        input message: '프로덕션 배포 확인?', ok: '배포'
    }
}
```

### Maven 빌드의 종속성 다운로드가 느립니다. 어떻게 해야 하나요?

1. Maven 미러 저장소 구성
2. 개인 저장소 관리자 사용
3. 로컬 저장소 경로 구성 및 영구 저장

### 빌드 시간 초과를 구성하려면?

```groovy
pipeline {
    agent any
    stages {
        stage('시간 초과 테스트') {
            options {
                timeout(time: 30, unit: 'MINUTES')
            }
            steps {
                // 빌드 스텝
            }
        }
    }
}
```

### 빌드 산출물을 유지하려면?

archive 스텝 사용:

```groovy
post {
    always {
        archiveArtifacts artifacts: 'target/*.jar', fingerprint: true
    }
}
```

### 부하 분산을 구성하려면?

```groovy

            agent { label 'docker' }
            steps {
                // 빌드
            }
        }
    }
}
```

## 백업 및 복원

### Jenkins를 백업하려면?

다음 항목을 백업해야 합니다:

1. JENKINS_HOME 디렉토리(모든 구성 및 빌드 기록 포함)
2. 플러그인 디렉토리
3. 환경 구성 파일

구성을 관리하기 위해 Configuration as Code 플러그인 사용을 권장합니다.

### Jenkins를 마이그레이션하려면?

1. 원본 Jenkins 중지
2. JENKINS_HOME을 새 서버로 복사
3. 동일한 Jenkins 버전 설치
4. Java 버전 호환성 확인
5. 새 Jenkins 시작
