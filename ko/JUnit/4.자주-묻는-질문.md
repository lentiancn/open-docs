# JUnit 자주 묻는 질문

## JUnit 4와 JUnit 5 중 어느 것을 사용해야 합니까?

새 프로젝트에는 JUnit 5를 적극 권장합니다. 중첩 테스트, 매개변수화 테스트, 동적 테스트 등 더 풍부한 기능과 우수한 확장성을 제공합니다. 기존 JUnit 4 프로젝트가 있는 경우 JUnit Vintage를 사용하여 JUnit 5 플랫폼에서 테스트를 실행하고 점진적으로 마이그레이션할 수 있습니다.

## JUnit 5에서 JUnit 4 테스트를 실행하려면 어떻게 합니까?

JUnit Vintage 엔진 의존성을 추가합니다:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## 왜 내 테스트가 실행되지 않습니까?

일반적인 원인: 테스트 클래스에 @Test 애노테이션이 없음, 테스트 메서드가 public이 아님, maven-surefire-plugin이 없거나 useJUnitPlatform()이 구성되지 않음, Java 버전이 요구사항을 충족하지 않음.

## JUnit 5의 최소 Java 버전은 무엇입니까?

JUnit 5는 Java 8 이상이 필요합니다. JUnit 4는 Java 5 이상이 필요합니다.

## 특정 테스트를 건너뛰려면 어떻게 합니까?

@Disabled 애노테이션을 사용하여 개별 테스트 또는 전체 테스트 클래스를 건너뛸 수 있습니다:

```java
@Disabled("일시적으로 비활성화")
@Test
void testSkipped() {
    // 실행되지 않음
}
```

## 비공개 메서드를 테스트하려면 어떻게 합니까?

단위 테스트는 public 메서드만 테스트해야 합니다. 비공개 메서드를 테스트해야 하는 경우 코드를 리팩토링하여 테스트할 로직을 별도의 public 메서드로 이동하거나 리플렉션을 사용합니다(권장하지 않음).

## 단언 실패 시 자세한 정보를 얻으려면 어떻게 합니까?

세 번째 매개변수를 사용하여 사용자 정의 오류 메시지를 제공합니다:

```java
assertEquals(expected, actual, "계산 결과가 올바르지 않음");
```

## 예외를 테스트하려면 어떻게 합니까?

assertThrows 메서드를 사용합니다:

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("오류 메시지");
    });
}
```

## 테스트를 병렬로 실행하려면 어떻게 합니까?

JUnit 5에서 병렬 실행을 구성합니다:

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // 테스트가 병렬로 실행됩니다
}
```

## @Before와 @BeforeEach의 차이점은 무엇입니까?

@Before는 JUnit 4 애노테이션이고, @BeforeEach는 JUnit 5 애노테이션입니다. 둘 다 동일한 목적, 즉 각 테스트 메서드 실행 전에 실행됩니다. JUnit 5는 @BeforeAll과 @AfterAll도 지원하여 전체 테스트 클래스에 대해 한 번만 실행되는 설정/정리 작업에 사용됩니다.

## 매개변수화 테스트를 생성하려면 어떻게 합니까?

@ParameterizedTest 애노테이션을 @ValueSource와 함께 사용합니다:

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## 테스트가 시간 초과되면 어떻게 합니까?

@Timeout 애노테이션을 사용하여 시간 초과를 설정합니다:

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // 100밀리초를 초과하면 실패
}
```

## 테스트를 반복하려면 어떻게 합니까?

@RepeatedTest 애노테이션을 사용합니다:

```java
@RepeatedTest(10)
void testRepeated() {
    // 10번 반복됩니다
}
```

## JUnit 5 확장 모델이란?

JUnit 5는 @ExtendWith 애노테이션을 사용하여 더 유연한 테스트 사용자 지정을 허용하는 확장 메커니즘을 제공합니다:

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // 사용자 정의 확장 사용
}
```
