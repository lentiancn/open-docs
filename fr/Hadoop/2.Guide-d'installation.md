# Guide d'Installation d'Hadoop

## Configuration Requise

Avant d'installer Hadoop, assurez-vous que votre système répond aux exigences de base en matière de matériel et de logiciel. Les clusters Hadoop peuvent être composés de plusieurs ordinateurs ou fonctionner en mode pseudo-distribué. Différents modes de déploiement ont des exigences matérielles différentes, mais les dépendances logicielles de base sont similaires. Comprendre ces exigences aide à planifier l'échelle et la configuration du cluster Hadoop, évitant les problèmes inutiles lors de l'installation.

En termes de matériel, les petits clusters de test doivent avoir au moins 4 cœurs de CPU, 8 Go de RAM et 500 Go de disque dur par machine. Pour les environnements de production, chaque machine doit avoir 8 cœurs de CPU ou plus, 16 Go de RAM ou plus, et 1 To ou plus de disque dur. Les DataNodes d'Hadoop stockent de grandes quantités de données, donc la capacité du disque est particulièrement importante. Hadoop recommande une configuration RAID ou une configuration JBOD (Just a Bunch Of Disks) : cette dernière permet à Hadoop de gérer l'espace disque de manière plus flexible.

En termes de systèmes d'exploitation, Hadoop supporte Linux (Ubuntu, CentOS, Debian, Red Hat, etc.) et Windows (via Cygwin ou Windows Subsystem for Linux). Les environnements de production recommandent fortement Linux pour des performances et une stabilité optimales. Hadoop s'intègre bien avec diverses plates-formes cloud et peut être déployé sur AWS, Azure, Google Cloud et autres services cloud.

Java est la seule dépendance d'exécution d'Hadoop. Hadoop 3.x nécessite Java 8 ou supérieur. Oracle JDK ou OpenJDK est recommandé. Assurez-vous que Java est correctement installé et que la variable d'environnement JAVA_HOME est définie avant d'installer Hadoop. Vérifiez l'installation de Java avec :

```bash
java -version
echo $JAVA_HOME
```

Le réseau est fondamental pour le fonctionnement du cluster Hadoop. Tous les nœuds du cluster doivent pouvoir communiquer entre eux via le réseau, de préférence en utilisant des réseaux haute vitesse (comme Gigabit Ethernet). Les nœuds ont besoin de SSH sans mot de passe configuré pour qu'Hadoop puisse distribuer les tâches et transférer des données entre les nœuds. De plus, configurez les mappages de noms d'hôtes et d'adresses IP pour tous les nœuds dans /etc/hosts.

## Étapes d'Installation

### Installation en Mode Autonome

Le mode autonome est la méthode d'installation la plus simple d'Hadoop, avec tous les composants s'exécutant dans le même processus JVM. Ce mode est principalement utilisé pour le développement et les tests et n'implique pas de stockage ou de calcul distribué. Tout d'abord, téléchargez la distribution binaire d'Hadoop depuis le site Web officiel d'Apache Hadoop :

```bash
# Télécharger Hadoop (remplacer par la dernière version)
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# Extraire dans le répertoire spécifié
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# Renommer le répertoire (optionnel)
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

Ensuite, configurez les variables d'environnement. Modifiez le fichier ~/.bashrc ou /etc/profile et ajoutez ce qui suit :

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # Ajuster selon le chemin Java réel
```

Appliquez la configuration :

```bash
source ~/.bashrc
```

Vérifiez l'installation :

```bash
hadoop version
```

### Installation en Mode Pseudo-Distribué

Le mode pseudo-distribué simule un environnement distribué sur une seule machine, avec NameNode, DataNode, ResourceManager et NodeManager s'exécutant tous sur la même machine. Ce mode convient au développement et aux tests de fonctionnalités distribuées.

Tout d'abord, configurez les fichiers principaux d'Hadoop. Accédez au répertoire de configuration d'Hadoop :

```bash
cd $HADOOP_HOME/etc/hadoop
```

Configurez core-site.xml, en spécifiant HDFS comme système de fichiers par défaut :

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Configurez hdfs-site.xml, en spécifiant le facteur de réplication HDFS (1 pour pseudo-distribué) :

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

Configurez mapred-site.xml, en spécifiant le framework d'ordonnancement des ressources pour MapReduce :

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

Configurez yarn-site.xml, en configurant les propriétés liées à YARN :

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### Configuration SSH Sans Mot de Passe

Les nœuds du cluster Hadoop ont besoin de SSH sans mot de passe configuré pour que le nœud maître puisse accéder aux nœuds esclaves :

```bash
# Générer une paire de clés SSH
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# Ajouter la clé publique à authorized_keys
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# Définir les permissions correctes
chmod 0600 ~/.ssh/authorized_keys

# Tester la connexion SSH
ssh localhost
```

Pour les clusters multi-nœuds, copiez la clé publique sur tous les nœuds :

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### Formater et Démarrer HDFS

Avant d'utiliser HDFS pour la première fois, formatez le NameNode :

```bash
hdfs namenode -format
```

Démarrez HDFS et YARN :

```bash
start-dfs.sh
start-yarn.sh
```

Ou utilisez start-all.sh pour démarrer tous les services à la fois :

```bash
start-all.sh
```

Vérifiez l'état du démarrage du service :

```bash
# Voir les processus Java
jps

# Processus attendus :
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Accédez aux services Hadoop via l'interface Web : NameNode à http://localhost:9870, ResourceManager à http://localhost:8088.

## Installation en Cluster Totalement Distribué

### Planification du Cluster

Le mode totalement distribué nécessite plusieurs machines physiques ou virtuelles pour former un cluster. Avant l'installation, effectuez une planification raisonnable du cluster. Déterminez l'échelle du cluster (nombre de nœuds) et attribuez les rôles à chaque nœud (NameNode, Secondary NameNode, DataNode, ResourceManager, NodeManager). Pour les petits clusters, NameNode et ResourceManager peuvent être déployés sur la même machine ; pour les grands clusters, un déploiement séparé est recommandé pour de meilleures performances.

La planification réseau est également importante. Tous les nœuds doivent être sur le même réseau interne avec une faible latence et une bande passante élevée. Configurez les noms d'hôtes et les adresses IP pour tous les nœuds dans /etc/hosts :

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### Modification des Fichiers de Configuration

En mode totalement distribué, plus de fichiers de configuration doivent être modifiés. Configurez d'abord le fichier workers (fichier slaves dans les versions antérieures), listant tous les nœuds DataNode et NodeManager :

```
slave1
slave2
slave3
```

La configuration de core-site.xml et hdfs-site.xml est similaire au mode pseudo-distribué, mais plus de propriétés doivent être ajoutées. Pour la configuration de Haute Disponibilité NameNode, configurez deux NameNodes :

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

Configurez hdfs-site.xml pour NameNode HA :

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

La configuration YARN est similaire ; configurez ResourceManager HA dans yarn-site.xml.

### Synchronisation des Fichiers de Configuration

Synchronisez le répertoire d'installation Hadoop configuré sur tous les nœuds. Copiez individuellement en utilisant scp :

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

Ou utilisez rsync pour la réplication distribuée :

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### Démarrage du Cluster

Formatez le NameNode sur le nœud NameNode (premier démarrage) :

```bash
hdfs namenode -format
```

Démarrez HDFS :

```bash
start-dfs.sh
```

Démarrez YARN :

```bash
start-yarn.sh
```

Vérifiez l'état des processus sur tous les nœuds pour vous assurer que les services s'exécutent correctement :

```bash
# Sur NameNode
jps

# Sur DataNode
jps
```

## Vérification et Tests

### Test HDFS

Créez des répertoires et téléchargez des fichiers pour tester HDFS :

```bash
# Créer un répertoire
hdfs dfs -mkdir -p /user/test

# Téléverser un fichier local vers HDFS
hdfs dfs -put localfile.txt /user/test/

# Lister les fichiers
hdfs dfs -ls /user/test

# Lire un fichier
hdfs dfs -cat /user/test/localfile.txt

# Supprimer un fichier
hdfs dfs -rm /user/test/localfile.txt
```

### Test MapReduce

Exécutez des programmes d'exemple MapReduce pour vérifier la fonctionnalité du cluster :

```bash
# Exécuter l'exemple WordCount
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# Voir les résultats
hdfs dfs -cat /output/part-r-00000
```

### Vérification de Santé du Cluster

Utilisez les commandes intégrées d'Hadoop pour vérifier l'état du cluster :

```bash
# Voir l'état HDFS
hdfs dfsadmin -report

# Voir l'état des nœuds YARN
yarn node -list -all

# Voir l'état des applications
yarn application -list
```

## Problèmes Courants

### Incompatibilité de Version Java

Si vous rencontrez des erreurs liées à la version Java, assurezvous que la version Java installée est compatible avec la version Hadoop. Hadoop 3.x nécessite Java 8 ou supérieur. Changez de version Java avec :

```bash
update-alternatives --config java
```

### Espace Disque Insuffisant

Les DataNodes peuvent ne pas fonctionner correctement en raison d'un espace disque insuffisant. Surveillez régulièrement l'utilisation du disque pour vous assurer que les DataNodes ont suffisamment d'espace disponible. Configurez dfs.datanode.fsdataset.volume.choosing.policy dans hdfs-site.xml comme AvailableSpaceVolumeChoosingPolicy pour permettre aux DataNodes de préférer les disques avec plus d'espace disponible.

### Mémoire Insuffisante

Si les nœuds du cluster ont une mémoire insuffisante, YARN peut ne pas démarrer de conteneurs. Ajustez la configuration de la mémoire dans yarn-site.xml :

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
