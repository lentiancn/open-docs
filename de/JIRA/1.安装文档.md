# JIRA Installationsanleitung

JIRA ist ein von Atlassian entwickeltes Problemverfolgungs- und Projektmanagement-Tool. Diese Anleitung behandelt die Installation in verschiedenen Umgebungen.

## Versionsinformationen

| Version | Status | Beschreibung |
|---------|--------|--------------|
| JIRA 9.x | ✅ Aktuell | Neueste Version |
| JIRA 8.x | ✅ Unterstützt | Stabile Version |

## Systemanforderungen

### Hardware-Anforderungen

| Anforderung | Minimum | Empfohlen |
|-------------|---------|-----------|
| CPU | 4 Kerne | 8+ Kerne |
| Speicher | 4 GB | 16+ GB |
| Festplatte | 10 GB | 50+ GB |

### Software-Anforderungen

| Software | Version |
|----------|---------|
| Java | JDK 11 oder 17 |
| Betriebssystem | Linux/Windows/macOS |
| Datenbank | PostgreSQL/MySQL/Oracle |

## Installationsmethoden

### Methode 1: Windows-Installation

#### 1. Installer herunterladen

1. Besuchen Sie https://www.atlassian.com/software/jira/download
2. Laden Sie JIRA Software herunter
3. Führen Sie den Installer aus

#### 2. Installation durchführen

Folgen Sie dem Installationsassistenten:
- Installationsart auswählen
- Port einrichten
- Datenbank konfigurieren
- Administratorkonto erstellen

### Methode 2: Linux-Installation

#### 1. Installationspaket herunterladen

```bash
# JIRA herunterladen
wget https://product-downloads.atlassian.com/software/jira/downloads/atlassian-jira-software-X.X.X-x64.bin
```

#### 2. Installieren

```bash
# Ausführungsberechtigung hinzufügen
chmod +x atlassian-jira-software-X.X.X-x64.bin

# Installation ausführen
sudo ./atlassian-jira-software-X.X.X-x64.bin
```

#### 3. Konfigurieren

```bash
# JIRA starten
sudo /opt/atlassian/jira/bin/start-jira.sh

# JIRA stoppen
sudo /opt/atlassian/jira/bin/stop-jira.sh
```

### Methode 3: Docker-Installation

```bash
# Image herunterladen
docker pull atlassian/jira-software

# Container ausführen
docker run -v jira-data:/var/atlassian/application-data/jira --name="jira" -d -p 8080:8080 atlassian/jira-software
```

### Methode 4: AWS verwenden

1. JIRA in AWS Marketplace suchen
2. Instanztyp auswählen
3. Datenbank konfigurieren
4. Instanz starten

## Konfiguration

### Datenbankkonfiguration

```properties
# dbconfig.xml
<?xml version="1.0" encoding="UTF-8"?>
<jira-database-config>
  <name>defaultDS</name>
  <connection-type>default</connection-type>
  <url>jdbc:postgresql://localhost:5432/jira</url>
  <username>jira</username>
  <password>password</password>
  <pool-size>20</pool-size>
</jira-database-config>
```

### Reverse-Proxy-Konfiguration

#### Nginx

```nginx
server {
    listen 80;
    server_name jira.example.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Häufig gestellte Fragen

### Start fehlgeschlagen?

Java-Version überprüfen:

```bash
java -version
```

### Datenbankverbindung fehlgeschlagen?

Datenbankkonfiguration und Verbindungsinformationen überprüfen.

### Nicht genügend Speicher?

Speichereinstellungen ändern:

```bash
# setenv.sh
export JVM_MINIMUM_MEMORY="2048m"
export JVM_MAXIMUM_MEMORY="4096m"
```
