# Node.js 常見問題

## 安裝相關問題

### Q: 如何檢查 Node.js 是否正確安裝？
**A:** 在終端機中執行以下命令：
```bash
node --version
npm --version
```
如果顯示版本號（如 v18.17.0），說明安裝成功。還可以執行簡單的測試：
```bash
node -e "console.log('Hello Node.js!')"
```

### Q: 安裝後命令未找到怎麼辦？
**A:** 這通常是因為 PATH 環境變數未正確設定。

**Windows 解決方案:**
- 重新執行安裝程式，確保勾選 "Add to PATH" 選項
- 或手動新增到系統環境變數：`C:\Program Files\nodejs\`

**macOS/Linux 解決方案:**
- 如果使用套件管理員安裝，通常會自動設定
- 手動新增到 shell 設定檔（~/.bashrc、~/.zshrc 等）：
  ```bash
  export PATH=$PATH:/usr/local/bin
  # 或者對於 Homebrew (Apple Silicon)
  export PATH=$PATH:/opt/homebrew/bin
  ```
- 重新載入設定：`source ~/.bashrc`

### Q: 如何在不同專案中使用不同版本的 Node.js？
**A:** 推薦使用版本管理工具：

**nvm (Node Version Manager) - macOS/Linux:**
```bash
# 安裝 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 安裝特定版本
nvm install 18.17.0

# 切換版本
nvm use 18.17.0

# 設定預設版本
nvm alias default 18.17.0

# 在專案目錄中建立 .nvmrc 檔案
echo "18.17.0" > .nvmrc
nvm use  # 自動使用 .nvmrc 中指定的版本
```

**nvm-windows - Windows:**
```cmd
:: 安裝特定版本
nvm install 18.17.0

:: 切換版本
nvm use 18.17.0

:: 查看已安裝版本
nvm list
```

## npm 相關問題

### Q: npm install 太慢怎麼辦？
**A:** 可以設定使用國內鏡像來源：

```bash
# 臨時使用淘寶鏡像
npm install --registry https://registry.npmmirror.com

# 永久設定淘寶鏡像
npm config set registry https://registry.npmmirror.com

# 驗證設定
npm config get registry

# 恢復官方來源
npm config delete registry
```

**其他鏡像來源:**
- 官方來源: `https://registry.npmjs.org`
- 淘寶鏡像: `https://registry.npmmirror.com`
- 華為雲: `https://mirrors.huaweicloud.com/repository/npm/`

### Q: 全域安裝套件時出現權限錯誤？
**A:** 這是常見的權限問題，有幾種解決方案：

**推薦方案: 使用版本管理員**
- 使用 nvm 或 fnm 管理 Node.js，這樣全域套件會安裝在使用者目錄下

**方案二: 設定 npm 全域目錄**
```bash
# 建立全域目錄
mkdir ~/.npm-global

# 設定 npm 使用新目錄
npm config set prefix '~/.npm-global'

# 新增到 PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

**方案三: 使用 sudo（不推薦）**
```bash
sudo npm install -g package-name
```
注意：這可能存在安全風險，且可能導致權限混亂。

### Q: 如何更新全域安裝的套件？
**A:** 
```bash
# 查看過期的全域套件
npm outdated -g

# 更新特定套件
npm update -g package-name

# 批次更新所有全域套件（需要額外套件）
npm install -g npm-check-updates
ncu -g
ncu -g -u  # 更新 package.json
```

### Q: package-lock.json 和 yarn.lock 有什麼區別？
**A:** 
- **package-lock.json**: npm 產生的鎖定檔案，確保相依性版本一致性
- **yarn.lock**: Yarn 產生的鎖定檔案，功能類似但格式不同
- **重要**: 應該將鎖定檔案提交到版本控制，確保團隊成員使用相同的相依性版本

## 開發相關問題

### Q: 如何處理非同步操作中的錯誤？
**A:** 始終使用適當的錯誤處理機制：

**async/await 方式（推薦）:**
```javascript
async function fetchData() {
  try {
    const data = await api.getData();
    return data;
  } catch (error) {
    console.error('獲取資料失敗:', error);
    // 可以拋出錯誤或返回預設值
    throw error;
    // 或 return null;
  }
}
```

**Promise 方式:**
```javascript
api.getData()
  .then(data => process(data))
  .catch(error => console.error('Error:', error));
```

**回呼函式方式:**
```javascript
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('讀取檔案失敗:', err);
    return;
  }
  console.log(data);
});
```

### Q: 如何避免回呼地獄（Callback Hell）？
**A:** 使用現代 JavaScript 特性替代巢狀回呼：

**Promise 連鎖:**
```javascript
step1()
  .then(result1 => step2(result1))
  .then(result2 => step3(result2))
  .catch(error => handleError(error));
```

**async/await（推薦）:**
```javascript
async function processSteps() {
  try {
    const result1 = await step1();
    const result2 = await step2(result1);
    const result3 = await step3(result2);
    return result3;
  } catch (error) {
    handleError(error);
  }
}
```

**平行執行:**
```javascript
// 平行執行多個獨立操作
const [result1, result2, result3] = await Promise.all([
  operation1(),
  operation2(),
  operation3()
]);
```

### Q: 如何處理記憶體洩漏？
**A:** 常見原因和解決方案：

**1. 全域變數累積:**
```javascript
// ❌ 避免不必要的全域變數
let globalCache = {};

// ✅ 使用區域變數或適當清理
function processData() {
  const cache = {};
  // ... 處理邏輯
  return result;
}
```

**2. 事件監聽器未移除:**
```javascript
// ❌ 忘記移除監聽器
emitter.on('event', handler);

// ✅ 使用 once() 或手動移除
emitter.once('event', handler);
// 或
emitter.on(' event', handler);
// ... 在適當時機
emitter.removeListener('event', handler);
```

**3. 計時器未清除:**
```javascript
// ❌ 忘記清除計時器
const timer = setInterval(() => {}, 1000);

// ✅ 儲存參考並清除
const timer = setInterval(() => {}, 1000);
// ... 在適當時機
clearInterval(timer);
```

**4. 快取未清理:**
```javascript
// ✅ 實作 LRU 快取或定期清理
const LRU = require('lru-cache');
const cache = new LRU({ max: 100 });
```

**記憶體分析工具:**
```bash
# 產生堆疊快照
node --inspect app.js

# 使用 Chrome DevTools 分析記憶體
```

## 效能相關問題

### Q: 如何提高 Node.js 應用效能？
**A:** 

**1. 使用叢集模式（多核心 CPU）:**
```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isPrimary) {
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
} else {
  // 啟動 HTTP 伺服器
  require('./server');
}
```

**2. 避免阻塞操作:**
- 始終使用非同步 I/O 方法
- 對於 CPU 密集型任務，使用 worker_threads 或 child_process

**3. 使用串流處理大檔案:**
```javascript
const fs = require('fs');
const readStream = fs.createReadStream('large-file.txt');
const writeStream = fs.createWriteStream('output.txt');
readStream.pipe(writeStream);
```

**4. 優化資料庫查詢:**
- 使用索引
- 避免 N+1 查詢問題
- 使用連線池

**5. 使用快取:**
- 記憶體快取：Map、LRU cache
- Redis：分散式快取
- HTTP 快取：ETag、Cache-Control

**6. 程式碼分割和延遲載入:**
- 只在需要時載入模組
- 使用動態 import()

### Q: 如何監控 Node.js 應用？
**A:** 

**內建監控工具:**
```bash
# 啟用效能監控
node --prof app.js

# 產生效能報告
node --prof-process isolate-*.log > profile.txt
```

**效能鉤子 API:**
```javascript
const { PerformanceObserver, performance } = require('perf_hooks');

const obs = new PerformanceObserver((items) => {
  console.log(items.getEntries());
});
obs.observe({ entryTypes: ['measure'] });

performance.mark('start');
// ... 程式碼
performance.mark('end');
performance.measure('operation', 'start', 'end');
```

**第三方監控工具:**
- **PM2**: 處理程序管理和監控
- **New Relic**: APM 監控
- **Datadog**: 基礎設施監控
- **Prometheus + Grafana**: 指標收集和視覺化

## 安全相關問題

### Q: 如何防止相依性漏洞？
**A:** 

**1. 定期稽核相依性:**
```bash
npm audit
npm audit fix
npm audit fix --force  # 強制修復（可能破壞相容性）
```

**2. 使用鎖定檔案:**
- 提交 `package-lock.json` 確保相依性一致性
- 定期更新相依性

**3. 限制相依性範圍:**
- 只安裝必要的相依性
- 定期審查相依性樹：`npm ls`

**4. 使用自動化工具:**
- **Snyk**: 自動檢測和修復漏洞
- **Dependabot**: GitHub 整合的自動相依性更新
- **npm outdated**: 檢查過期相依性

### Q: 如何安全地處理環境變數？
**A:** 

**1. 使用 .env 檔案（不要提交到版本控制）:**
```bash
# .env
DATABASE_URL=your-database-url
API_KEY=your-api-key
JWT_SECRET=your-secret-key
```

**2. 在 .gitignore 中忽略 .env:**
```gitignore
.env
.env.local
.env.development
.env.production
```

**3. 在程式碼中安全使用:**
```javascript
require('dotenv').config();

// 驗證必需的環境變數
const requiredEnvVars = ['DATABASE_URL', 'JWT_SECRET'];
requiredEnvVars.forEach(envVar => {
  if (!process.env[envVar]) {
    throw new Error(`Missing required environment variable: ${envVar}`);
  }
});

const dbUrl = process.env.DATABASE_URL;
```

**4. 不同環境使用不同設定:**
```javascript
// config/index.js
const env = process.env.NODE_ENV || 'development';

const configs = {
  development: require('./development'),
  production: require('./production'),
  test: require('./test')
};

module.exports = configs[env];
```

console.trace('Here I am!');

### Q: 如何處理未捕捉的例外？
**A:** 新增全域例外處理器，但要注意這不能完全防止應用崩潰：

```javascript
// 未捕捉的例外（同步錯誤）
process.on('uncaughtException', (error) => {
  console.error('未捕捉的例外:', error);
  // 執行清理操作
  gracefullyShutdown();
  // 退出處理程序
  process.exit(1);
});

// 未處理的 Promise 拒絕
process.on('unhandledRejection', (reason, promise) => {
  console.error('未處理的 Promise 拒絕:', reason);
  // 注意：從 Node.js 15 開始，這會導致處理程序退出
});

// 優雅關閉
function gracefullyShutdown() {
  // 關閉資料庫連線
  // 關閉 HTTP 伺服器
  // 清理臨時檔案
}
```

**最佳實務:**
- 盡量在程式碼層面處理所有可能的錯誤
- 使用 try-catch 包裝 async/await
- 不要完全依賴全域例外處理器

## 部署相關問題

### Q: 如何部署 Node.js 應用？
**A:** 推薦使用處理程序管理員確保應用穩定執行：

**PM2（推薦）:**
```bash
# 安裝 PM2
npm install -g pm2

# 啟動應用
pm2 start app.js --name "my-app"

# 查看應用狀態
pm2 status

# 設定開機自啟
pm2 startup
pm2 save

# 日誌查看
pm2 logs my-app
```

**Docker:**
```dockerfile
FROM node:18-alpine

WORKDIR /app

# 複製 package 檔案
COPY package*.json ./

# 安裝相依性
RUN npm ci --only=production

# 複製應用程式碼
COPY . .

# 建立非 root 使用者
RUN chown -R node:node /app
USER node

EXPOSE 3000

CMD ["node", "app.js"]
```

**Systemd（Linux）:**
```ini
# /etc/systemd/system/my-app.service
[Unit]
Description=My Node.js App
After=network.target

[Service]
Type=simple
User=node
WorkingDirectory=/var/www/my-app
ExecStart=/usr/bin/node app.js
Restart=always

[Install]
WantedBy=multi-user.target
```

### Q: 如何處理環境差異？
**A:** 使用環境變數區分不同環境：

```javascript
const isProduction = process.env.NODE_ENV === 'production';
const port = process.env.PORT || 3000;
const dbUrl = process.env.DATABASE_URL;

const config = {
  port,
  database: {
    url: dbUrl,
    poolSize: isProduction ? 20 : 5
  },
  logging: !isProduction,
  cors: {
    origin: isProduction ? 'https://myapp.com' : '*'
  }
};

module.exports = config;
```

**環境檔案:**
```
# .env.development
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://localhost:5432/myapp_dev

# .env.production  
NODE_ENV=production
PORT=8080
DATABASE_URL=postgresql://prod-server:5432/myapp_prod
```

透過理解和應用這些解決方案，您將能夠更有效地開發、除錯和維護 Node.js 應用程式。
