# JUnit 5 FAQ

## Allgemeine Fragen

### Was ist JUnit?
JUnit ist ein Open-Source-Framework für Unit-Tests für Java und die JVM. Es ist Teil der xUnit-Familie von Testframeworks und ist der am weitesten verbreitete Testframework für Java-Anwendungen.

### Was ist JUnit 5?
JUnit 5 (Jupiter) ist die neueste Hauptversion des JUnit-Testframeworks. Es führt eine neue Architektur mit drei Modulen ein: Platform, Jupiter und Vintage.

### Was ist der Unterschied zwischen JUnit 4 und JUnit 5?

| Funktion | JUnit 4 | JUnit 5 |
|---------|---------|---------|
| Architektur | Monolithisch | Modular (Platform, Jupiter, Vintage) |
| Java-Version | Java 5+ | Java 8+ |
| Testnamen | Nur Methodennamen | Anzeigenamen |
| Verschachtelte Tests | Nicht unterstützt | Unterstützt |
| Parametrisierte Tests | Externe Bibliothek | Integriert |

### Muss ich alle meine JUnit 4 Tests neu schreiben?
Nein! JUnit 5 enthält JUnit Vintage, mit dem Sie JUnit 3 und 4 Tests auf der JUnit 5 Plattform ausführen können.

## Installationsfragen

### Wie füge ich JUnit 5 zu meinem Maven-Projekt hinzu?
Fügen Sie die folgende Abhängigkeit zu Ihrer pom.xml hinzu:

```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```

### Wie füge ich JUnit 5 zu meinem Gradle-Projekt hinzu?
Fügen Sie folgendes zu build.gradle hinzu:

```groovy
test {
    useJUnitPlatform()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}
```

### Was sind die Mindestanforderungen für Java für JUnit 5?
JUnit 5 erfordert Java 8 oder höher zur Laufzeit.

## Nutzungsfragen

### Wie schreibe ich einen grundlegenden Test in JUnit 5?

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class MyFirstTest {
    
    @Test
    void simpleTest() {
        assertEquals(2, 1 + 1);
    }
}
```

### Wie verwende ich Anzeigenamen?

```java
@Test
@DisplayName("Ein einfacher Mathe-Test")
void testAddition() {
    assertEquals(4, 2 + 2);
}
```

### Wie erstelle ich verschachtelte Tests?

```java
class NestedTests {
    
    @Nested
    class AdditionTests {
        @Test
        void positiveZahlen() {
            assertEquals(5, 2 + 3);
        }
    }
}
```

### Wie schreibe ich parametrisierte Tests?

```java
@ParameterizedTest
@ValueSource(ints = {1, 2, 3})
void testIstPositiv(int Zahl) {
    assertTrue(Zahl > 0);
}
```

### Wie teste ich auf Exceptions?

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, 
        () -> { throw new IllegalArgumentException("Fehler"); });
}
```

## Erweiterte Fragen

### Wie erstelle ich benutzerdefinierte Assertions?

```java
public class CustomAssertions {
    
    public static void assertEven(int number) {
        assertTrue(number % 2 == 0, "Zahl muss gerade sein");
    }
}
```

### Wie verwende ich das Erweiterungsmodell?

```java
@ExtendWith(MyExtension.class)
class MyTest {
    // Testmethoden
}
```

### Können JUnit 5 Tests parallel ausgeführt werden?
Ja, JUnit 5 unterstützt parallele Ausführung. Aktivieren Sie es in Ihrer Build-Konfiguration.

## Migrationsfragen

### Wie migriere ich von JUnit 4 zu JUnit 5?

1. Aktualisieren Sie Ihre Abhängigkeiten auf JUnit 5
2. Ersetzen Sie @Test von org.junit.Test durch org.junit.jupiter.api.Test
3. Ersetzen Sie assertEquals von org.junit.Assert durch org.junit.jupiter.api.Assertions
4. Ersetzen Sie @Before und @After durch @BeforeEach und @AfterEach
5. Ersetzen Sie @BeforeClass und @AfterClass durch @BeforeAll und @AfterAll
6. Ersetzen Sie @RunWith durch @ExtendWith

### Kann ich JUnit 3 und 4 Tests mit JUnit 5 ausführen?
Ja, fügen Sie die JUnit Vintage-Abhängigkeit hinzu:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```
