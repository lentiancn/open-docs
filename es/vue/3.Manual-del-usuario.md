# Vue.js Manual del Usuario

## Instancia de Aplicación

Cada aplicación Vue comienza creando una nueva instancia de aplicación con la función `createApp`:

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* opciones del componente raíz */
})
```

### Componente Raíz
El objeto que pasamos a `createApp` es en realidad un componente. Cada aplicación requiere un "componente raíz" que puede contener otros componentes como sus hijos.

### Montando la Aplicación
Una instancia de aplicación no renderizará nada hasta que se llame a su método `.mount()`:

```javascript
app.mount('#app')
```

### Configuraciones de la Aplicación
La instancia de aplicación expone un objeto `.config` que permite configurar opciones a nivel de aplicación:

```javascript
// Manejador de errores a nivel de aplicación
app.config.errorHandler = (err) => {
  /* manejar error */
}

// Registrar componente global
app.component('TodoDeleteButton', TodoDeleteButton)
```

### Múltiples Instancias de Aplicación
Puedes crear múltiples aplicaciones Vue en la misma página:

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## Sintaxis de Plantillas

Vue utiliza una sintaxis de plantillas basada en HTML que te permite enlazar declarativamente el DOM renderizado con los datos de la instancia del componente subyacente.

### Interpolación de Texto
La forma más básica de enlace de datos es la interpolación de texto usando la sintaxis "Mustache" (llaves dobles):

```html
<span>Message: {{ msg }}</span>
```

### HTML Crudo
Usa la directiva `v-html` para mostrar HTML real:

```html
<p v-html="rawHtml"></p>
```

**Advertencia de Seguridad**: Renderizar HTML arbitrario dinámicamente puede conducir a vulnerabilidades XSS. Usa `v-html` solo en contenido de confianza y nunca en contenido proporcionado por usuarios.

### Enlaces de Atributos
Usa la directiva `v-bind` para enlazar atributos HTML:

```html
<!-- Sintaxis completa -->
<div v-bind:id="dynamicId"></div>

<!-- Abreviatura -->
<div :id="dynamicId"></div>

<!-- Abreviatura de mismo nombre (Vue 3.4+) -->
<div :id></div>
```

### Atributos Booleanos
Los atributos booleanos se incluyen cuando el valor enlazado es verdadero:

```html
<button :disabled="isButtonDisabled">Button</button>
```

### Enlazando Múltiples Atributos Dinámicamente
Enlaza múltiples atributos desde un objeto usando `v-bind` sin argumento:

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## Expresiones JavaScript

Vue admite expresiones JavaScript completas en todos los enlaces de datos:

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### Llamando Funciones
Llama métodos expuestos por el componente en expresiones de enlace:

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### Acceso a Globales Restringido
Las expresiones de plantilla solo tienen acceso a una lista restringida de globales (Math, Date, etc.). Agrega globales adicionales mediante `app.config.globalProperties`.

## Directivas

Las directivas son atributos especiales con el prefijo `v-`.

### Argumentos
Algunas directivas toman "argumentos" denotados por dos puntos después del nombre de la directiva:

```html
<!-- Argumento de v-bind -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- Argumento de v-on -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### Argumentos Dinámicos
Usa expresiones JavaScript en argumentos de directivas envolviéndolas con corchetes:

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**Restricciones**:
- Los argumentos dinámicos deben evaluarse a una cadena o null
- Evita expresiones complejas; usa propiedades computadas en su lugar
- En plantillas DOM, evita caracteres en mayúsculas en claves dinámicas

### Modificadores
Los modificadores son sufijos especiales denotados por un punto:

```html
<form @submit.prevent="onSubmit">...</form>
```

## Fundamentos de Reactividad

### Declarando Estado Reactivo
Crea referencias reactivas con `ref()`:

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### Objetos Reactivos
Crea objetos reactivos con `reactive()`:

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### Propiedades Computadas
Crea propiedades computadas con `computed()`:

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### Observadores
Observa cambios en datos reactivos con `watch()`:

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// Observar una sola referencia
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## Conceptos Básicos de Componentes

### Definiendo Componentes
```javascript
// Definir componente
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Pasando Props
```javascript
// Componente hijo
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// Componente padre
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### Emitiendo Eventos
```javascript
// Componente hijo
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### Slots
```html
<!-- Componente hijo -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- Componente padre -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
