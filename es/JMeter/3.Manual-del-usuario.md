# Manual de usuario de JMeter

Este manual proporciona instrucciones detalladas sobre cómo usar Apache JMeter para crear y ejecutar pruebas de carga.

## Estructura básica del plan de prueba

Un plan de prueba de JMeter consta de los siguientes componentes principales:

### 1. Plan de prueba (Test Plan)

El Plan de prueba es el nodo raíz de todas las pruebas, que contiene información de configuración para toda la prueba. Puede incluir:
- Variables definidas por el usuario
- Configuración de propiedades globales
- Configuración de grupo de hilos

### 2. Grupo de hilos (Thread Group)

Los grupos de hilos son las unidades de ejecución de las pruebas, que definen:
- **Número de hilos**: Número de usuarios simulados
- **Tiempo de calentamiento (Ramp-up)**: Tiempo requerido para iniciar todos los hilos
- **Conteo de bucles**: Número de veces que cada hilo ejecuta la prueba
- **Programador**: Establecer hora de inicio y duración de la prueba

### 3. Samplers

Los samplers envían solicitudes a los servidores. Los samplers comunes incluyen:

- **Solicitud HTTP**: Enviar solicitudes HTTP/HTTPS
- **Solicitud FTP**: Probar servidores FTP
- **Solicitud JDBC**: Probar conexiones de base de datos
- **Solicitud LDAP**: Probar directorios LDAP
- **Mensaje JMS**: Probar colas de mensajes
- **Sampler TCP**: Probar servicios TCP

### 4. Controladores lógicos (Logic Controllers)

Los controladores lógicos controlan el orden de ejecución de las solicitudes:

- **Controlador simple**: Agrupar solicitudes
- **Controlador de bucle**: Repetir ejecución de solicitudes
- **Controlador una vez sola**: Ejecutar solo una vez durante la prueba
- **Controlador alternado**: Alternar entre solicitudes
- **Controlador aleatorio**: Seleccionar aleatoriamente solicitudes para ejecutar
- **Controlador de rendimiento**: Controlar la frecuencia de ejecución de solicitudes

### 5. Listeners

Los listeners ven y analizan los resultados de las pruebas:

- **Ver árbol de resultados**: Ver respuesta detallada de cada solicitud
- **Informe agregado**: Ver estadísticas de rendimiento agregadas
- **Resultados gráficos**: Mostrar datos de prueba gráficamente
- **Gráfico de tiempo de respuesta**: Mostrar tendencias de tiempo de respuesta

### 6. Elementos de configuración (Config Elements)

Los elementos de configuración establecen valores predeterminados y datos compartidos:

- **Valores predeterminados de solicitud HTTP**: Establecer parámetros predeterminados para solicitudes HTTP
- **Variables definidas por el usuario**: Definir variables reutilizables
- **Conjunto de datos CSV**: Leer datos de prueba de archivos CSV
- **Gestor de cookies HTTP**: Gestionar cookies
- **Gestor de encabezados HTTP**: Establecer información de encabezado de solicitud

### 7. Temporizadores (Timers)

Los temporizadores establecen retrasos entre solicitudes:

- **Temporizador fijo**: Establecer retraso fijo
- **Temporizador aleatorio gaussiano**: Establecer retraso aleatorio
- **Temporizador de sincronización**: Simular solicitudes concurrentes
- **Temporizador de rendimiento**: Controlar el rendimiento de solicitudes

### 8. Pre-procesadores y Post-procesadores

- **Pre-procesadores**: Ejecutar operaciones antes de enviar la solicitud
- **Post-procesadores**: Extraer datos después de recibir la respuesta

## Creando su primer plan de prueba

### Paso 1: Crear plan de prueba

1. Inicie JMeter
2. Seleccione "Archivo" -> "Nuevo" para crear un nuevo plan de prueba
3. Nombre su plan de prueba

### Paso 2: Agregar grupo de hilos

1. Haga clic derecho en el plan de prueba
2. Seleccione "Agregar" -> "Hilos (Usuarios)" -> "Grupo de hilos"
3. Establezca número de hilos (ej. 10), tiempo de calentamiento (ej. 5 segundos), conteo de bucles (ej. 1)

### Paso 3: Agregar solicitud HTTP

1. Haga clic derecho en el grupo de hilos
2. Seleccione "Agregar" -> "Sampler" -> "Solicitud HTTP"
3. Establezca nombre o IP del servidor, número de puerto, ruta de solicitud, etc.

### Paso 4: Agregar listener

1. Haga clic derecho en el grupo de hilos o solicitud HTTP
2. Seleccione "Agregar" -> "Listener" -> "Ver árbol de resultados"
3. Agregue "Informe agregado" para ver estadísticas de rendimiento

### Paso 5: Ejecutar prueba

1. Haga clic en el botón "Inicio" en la barra de herramientas
2. Observe los datos de resultados mostrados en el listener

## Parametrización y pruebas controladas por datos

### Usando archivos de datos CSV

1. Agregue elemento "Conjunto de datos CSV"
2. Establezca nombre de archivo y nombres de variables
3. Use ${nombreVariable} en las solicitudes para hacer referencia a los datos

### Usando variables definidas por el usuario

1. Agregue elemento "Variables definidas por el usuario"
2. Defina nombres y valores de variables
3. Use ${nombreVariable} en las solicitudes

## Manejo de correlación

### Usando extractor de expresiones regulares

Cuando las solicitudes requieren datos dinámicos (como ID de sesión):

1. Agregue "Extractor de expresiones regulares"
2. Establezca patrón de expresión regular para coincidir
3. Use la variable extraída en solicitudes posteriores

### Usando extractor JSON

Para respuestas JSON:

1. Agregue "Extractor JSON"
2. Establezca expresión JSON Path
3. Extraiga los valores de campo JSON requeridos

## Pruebas distribuidas

Cuando una sola máquina no puede generar suficiente carga, se pueden usar pruebas distribuidas:

### Configuración del maestro

1. Edite jmeter.properties en el directorio bin del maestro
2. Establezca remote_hosts para apuntar a las direcciones de los esclavos
3. Inicie JMeter

### Configuración del esclavo

1. Ejecute jmeter-server en cada máquina esclava
2. Asegúrese de que el firewall permita la comunicación en los puertos especificados

### Iniciando prueba distribuida

1. Seleccione "Inicio remoto" en el maestro
2. Elija qué esclavos iniciar o iniciar todos

## Consejos de optimización de rendimiento

- Use el modo de línea de comandos para reducir el consumo de recursos de GUI
- Establezca conteo de hilos y tiempo de calentamiento apropiados para evitar carga alta instantánea
- Desactive listeners innecesarios, solo vea resultados cuando sea necesario
- Use aserciones para verificar el contenido de la respuesta
- Limpie los planes de prueba no utilizados regularmente
- Para pruebas a gran escala, considere pruebas distribuidas
