# Apache JMeter 简介

Apache JMeter 是 Apache 软件基金会旗下的开源软件，是一个 100% 纯 Java 编写的应用程序，专为负载测试功能设计和性能测量而开发。JMeter 最初是为测试 Web 应用程序而设计的，后来已扩展到其他测试功能领域。

## JMeter 能做什么

JMeter 可用于对静态资源和动态资源进行性能测试，包括 Web 应用程序。它可以模拟服务器、服务器组、网络或对象的高负载情况，测试其强度或在不同负载类型下分析整体性能。

JMeter 的主要功能特性包括：

- **多协议支持**：支持测试多种应用程序、服务器和协议类型，包括 Web（HTTP/HTTPS）、SOAP/REST Web 服务、FTP、数据库（JDBC）、LDAP、JMS 消息中间件、邮件协议（SMTP/POP3/IMAP）、TCP、Java 对象等
- **完整的测试 IDE**：提供功能完整的测试 IDE，支持从浏览器或原生应用程序快速录制测试计划、构建和调试
- **命令行模式**：支持 CLI 模式（命令行模式/无头模式），可在任何 Java 兼容操作系统（Linux、Windows、Mac OSX 等）上进行负载测试
- **动态 HTML 报告**：内置完整且易于展示的动态 HTML 报告生成功能
- **数据提取能力**：支持从多种常见响应格式中提取数据，包括 HTML、JSON、XML 或任何文本格式
- **纯 Java 便携性**：完全可移植，100% 纯 Java 实现
- **多线程框架**：完整的并发采样能力，支持多线程同时采样和不同功能组独立测试
- **高度可扩展**：支持可插拔的采样器、计时器，以及 JSR223 兼容脚本语言（如 Groovy 和 BeanShell）
- **持续集成支持**：提供与 Maven、Gradle 和 Jenkins 集成的第三方开源库

## JMeter 不是浏览器

JMeter 工作在协议级别，它不是浏览器。对于 Web 服务和远程服务，JMeter 表现得像一个浏览器（或多个浏览器）；但 JMeter 不执行浏览器支持的所有操作，特别是不会执行 HTML 页面中的 JavaScript，也不会像浏览器那样渲染 HTML 页面。

## JMeter 的应用场景

- 负载测试：模拟大量并发用户，测试系统在高负载下的表现
- 性能测试：测量系统在不同负载条件下的响应时间和吞吐量
- 功能测试：验证应用程序的功能行为
- 压力测试：测试系统在高压力下的稳定性和容错能力
- 分布式测试：支持多台机器协同进行负载测试
