# Mockito 安裝指南

## 環境要求

在安裝 Mockito 之前，需要確保開發環境滿足以下要求。

Mockito 5.x 版本需要 JDK 11 或更高版本。如果專案需要支援 JDK 8，可以選擇 Mockito 4.x 系列。建議使用最新版本的 Mockito 以獲得最新的功能和安全性修復。

## Maven 安裝方式

### 新增依賴

在 Maven 專案的 pom.xml 檔案中新增 mockito-core 依賴。建議使用佔位符定義 Mockito 版本，便於版本統一管理。

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

如果需要使用 Mockito 的內聯（inline）功能來模擬 final 類和方法，還需要新增 mockito-inline 依賴。

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### 快照版本

Mockito 每次程式碼變更都會發布快照版本到 Sonatype 倉庫。如果需要使用最新開發中的功能，可以在 pom.xml 中新增快照倉庫地址。

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Gradle 安裝方式

### 新增依賴

在 Gradle 專案的 build.gradle 檔案中新增依賴宣告。

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

如果使用 Gradle 的 Kotlin DSL 檔案（build.gradle.kts），語法稍有不同。

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### 使用 BOM 管理版本

透過 Spring Dependency Management 插件或 Gradle 的 BOM 功能，可以簡化版本管理工作。

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## 手動下載安裝

如果專案不使用 Maven 或 Gradle，可以直接從 Maven Central 倉庫下載 JAR 檔案。訪問 Maven Central 搜尋 org.mockito，可以找到所有版本的 mockito-core 和 mockito-all（已廢棄）分发包。

下載後，將 JAR 檔案新增到專案的類別路徑中即可。需要同時下載 Byte Buddy 和 Objenesis 等依賴庫。

## 與 Spring Boot 整合

Spring Boot 的 spring-boot-starter-test 已經包含了 Mockito 依賴。如果專案使用 Spring Boot，基本不需要額外新增 Mockito 依賴。

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

如果需要在 Spring 環境中更方便地使用 Mock，可以使用 @MockBean 注解，它會自動將模擬物件注入到 Spring 容器中。

## 與 JUnit 5 整合

### 使用 MockitoExtension

JUnit 5 引入了擴展模型，Mockito 提供了 MockitoExtension 來支援 JUnit 5。無需額外的依賴設定，mockito-core 已經包含了必要的支援。

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### 使用 MockitoJUnitRunner

對於 JUnit 4，可以使用 MockitoJUnitRunner 來自動初始化 Mock 物件。

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## 驗證安裝

安裝完成後，可以透過以下方式驗證 Mockito 是否正確設定。建立一個簡單的測試類，執行後如果測試通過，說明 Mockito 已正確安裝。

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito 安裝成功！");
    }
}
```

## 版本選擇建議

對於新專案，推薦使用最新的 Mockito 5.x 版本。如果專案需要相容 JDK 8，可以選擇 Mockito 4.x 系列。建議始終使用最新的穩定版本，以獲得更好的效能和安全性。
