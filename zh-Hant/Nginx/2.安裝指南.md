# Nginx 安裝指南

## 環境需求

Nginx 支援多種作業系統，包括：

- Linux（Debian、Ubuntu、CentOS、RHEL、Alpine 等）
- FreeBSD
- Solaris
- AIX
- HP-UX
- macOS
- Windows

## 安裝方式

### 方式一：使用套件管理員安裝（推薦）

#### Ubuntu/Debian

```bash
# 更新套件索引
sudo apt update

# 安裝 Nginx
sudo apt install nginx

# 啟動 Nginx
sudo systemctl start nginx

# 設定開機自動啟動
sudo systemctl enable nginx
```

#### CentOS/RHEL

```bash
# 安裝 EPEL 套件庫
sudo yum install epel-release

# 安裝 Nginx
sudo yum install nginx

# 啟動 Nginx
sudo systemctl start nginx

# 設定開機自動啟動
sudo systemctl enable nginx
```

#### Alpine

```bash
apk add nginx
```

#### macOS

```bash
# 使用 Homebrew 安裝
brew install nginx

# 啟動 Nginx
brew services start nginx
```

### 方式二：從原始碼編譯安裝

#### 安裝編譯依賴

**Debian/Ubuntu:**

```bash
sudo apt install build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev
```

**CentOS/RHEL:**

```bash
sudo yum install gcc pcre-devel zlib-devel openssl-devel
```

#### 下載原始碼

```bash
# 下載最新穩定版
wget https://nginx.org/download/nginx-1.26.0.tar.gz

# 解壓縮
tar -zxf nginx-1.26.0.tar.gz

# 進入目錄
cd nginx-1.26.0
```

#### 編譯設定

```bash
./configure \
    --prefix=/usr/local/nginx \
    --conf-path=/usr/local/nginx/conf/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/lock/nginx.lock \
    --user=nginx \
    --group=nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_stub_status_module \
    --with-pcre
```

#### 編譯並安裝

```bash
# 編譯
make

# 安裝
sudo make install
```

### 方式三：使用 Docker 安裝

```bash
# 拉取官方映像檔
docker pull nginx

# 執行 Nginx 容器
docker run -d -p 80:80 --name mynginx nginx

# 對應設定檔和日誌
docker run -d -p 80:80 \
    -v /host/path/nginx.conf:/etc/nginx/nginx.conf:ro \
    -v /host/logs:/var/log/nginx \
    --name mynginx nginx
```

### 方式四：Windows 安裝

1. 從 https://nginx.org/en/download.html 下載 Windows 版本的 Nginx
2. 解壓縮到指定目錄
3. 雙擊 nginx.exe 啟動
4. 或者使用命令列啟動：

```cmd
cd C:\nginx
start nginx
```

## 安裝後驗證

### 檢查安裝

```bash
# 檢查 Nginx 版本
nginx -v

# 查看詳細版本資訊
nginx -V
```

### 檢查執行狀態

```bash
# 檢查程序是否執行中
ps aux | grep nginx

# 使用 systemctl 檢查狀態
systemctl status nginx
```

### 測試存取

```bash
# 使用 curl 測試
curl http://localhost

# 或者打開瀏覽器存取
http://localhost
```

## 目錄結構

### Linux 預設目錄

| 路徑 | 說明 |
|------|------|
| /etc/nginx/ | 設定目錄 |
| /etc/nginx/nginx.conf | 主設定檔 |
| /etc/nginx/conf.d/ | 子設定檔目錄 |
| /var/log/nginx/ | 日誌目錄 |
| /var/www/html/ | 預設網站根目錄 |
| /usr/sbin/nginx | 可執行檔 |
| /usr/share/nginx/html/ | 預設靜態檔案目錄 |

### 編譯安裝目錄

| 路徑 | 說明 |
|------|------|
| /usr/local/nginx/ | 安裝根目錄 |
| /usr/local/nginx/conf/ | 設定目錄 |
| /usr/local/nginx/html/ | 預設網站根目錄 |
| /usr/local/nginx/logs/ | 日誌目錄 |
| /usr/local/nginx/sbin/ | 可執行檔目錄 |

## 解除安裝 Nginx

### Debian/Ubuntu

```bash
sudo apt remove nginx nginx-common nginx-full
```

### CentOS/RHEL

```bash
sudo yum remove nginx
```

### 編譯安裝

```bash
# 停止 Nginx
sudo nginx -s stop

# 刪除安裝目錄
sudo rm -rf /usr/local/nginx
```

## 常見問題

### 連接埠衝突

如果啟動失敗，提示連接埠被佔用：

```bash
# 檢查連接埠佔用
netstat -tulpn | grep :80

# 修改 Nginx 監聽連接埠
# 編輯設定檔 /etc/nginx/nginx.conf
# 將 listen 80; 改為 listen 8080;
```

### 權限問題

如果遇到權限錯誤：

```bash
# 檢查 Nginx 用戶
ps aux | grep nginx

# 修改檔案權限
sudo chown -R www-data:www-data /var/www/html/
```

### 設定檔語法檢查

```bash
# 測試設定檔語法
nginx -t

# 指定設定檔測試
nginx -t -c /path/to/nginx.conf
```
