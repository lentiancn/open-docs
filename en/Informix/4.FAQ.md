# IBM Informix FAQ

## General Questions

### Q: What is the difference between IBM Informix and HCL Informix?

A: IBM Informix is sold and marketed by IBM, while HCL Informix is developed and supported by HCLSoftware. Both share the same codebase and are functionally equivalent for on-premises deployments. IBM handles sales and licensing, while HCL handles development and technical support.

### Q: Which edition should I choose for development?

A: Use the **Developer Edition** (free) or **Innovator-C Edition** (free) for development and testing. Both have limitations on resources but include full functionality:
- Developer Edition: 1 CPU VP/Core, 1GB RAM, 8GB storage
- Innovator-C Edition: 2 CPU Cores, 8GB RAM, 50GB storage

### Q: Is Informix suitable for IoT applications?

A: Yes, Informix is excellent for IoT due to its small footprint (as small as 100 MB), low administration requirements, built-in TimeSeries support, and ability to run on edge devices including Raspberry Pi.

### Q: What programming languages are supported?

A: Informix supports multiple programming interfaces:
- SQL (native)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- MongoDB compatibility (JSON/BSON)

## Installation Issues

### Q: Installation fails with "No permission" error

A: Ensure you have root/sudo privileges. Create the informix user and group before installation:

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### Q: Server won't start after installation

A: Check the following:
- Verify environment variables are set correctly (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- Check `onconfig` and `sqlhosts` files for correctness
- Review error logs in `$INFORMIXDIR/tmp/`
- Ensure sufficient disk space and permissions on all directories

### Q: How to resolve license key issues?

A: For evaluation versions, ensure you downloaded from IBM Passport Advantage with correct entitlements. For production, apply the license key using IBM License Key Center.

### Q: What ports does Informix use?

A: Default port is 9088. You can verify in the `sqlhosts` file. Ensure firewall allows this port for remote connections.

## Common Operational Issues

### Q: How to backup and restore Informix?

A: Use `ontape` or `onbar` utilities:

```bash
# Full backup with ontape
ontape -s -L 0

# Restore with ontape
ontape -r

# Backup with onbar
onbar -b -L 0

# Restore with onbar
onbar -r
```

### Q: How to check server status?

A: Use `onstat` command:

```bash
onstat -        # General status
onstat -d       # Disk space usage
onstat -l       # Logical logs
onstat -u       # Active users
onstat -k       # Lock information
onstat -g <cmd> # Specific group (memory, locks, etc.)
```

### Q: How to connect to Informix from external applications?

A: Configure ODBC, JDBC, or native client drivers. Update `sqlhosts` file for network connectivity. Example JDBC connection string:

```
jdbc:informix-sqli://hostname:9088/dbname:INFORMIXSERVER=servername
```

### Q: What causes "Logical Log Full" error?

A: This occurs when all logical logs are full and not backed up. Solutions:
- Backup logical logs: `ontape -s -L 0`
- Add more logical log files: `onparams -a -d dbspace -s size`
- Check for long-running transactions

### Q: How to handle lock conflicts?

A: Use `onstat -k` to view locks and `onstat -u` to identify blocking sessions. Solutions:
- Increase lock timeout
- Use lower isolation level (dirty read)
- Terminate blocking session: `onmode -z session_id`

## Performance and Tuning

### Q: How to improve query performance?

A:
- Create appropriate indexes for frequently queried columns
- Use `EXPLAIN` to analyze query plans
- Configure shared memory appropriately (`SHMVIRTSIZE`, `BUFFERPOOL`)
- Update statistics regularly: `UPDATE STATISTICS FOR TABLE tablename`
- Consider using Informix Warehouse Accelerator for BI queries

### Q: What are the key memory configuration parameters?

A:
- `SHMVIRTSIZE` - Virtual shared memory size (initial allocation)
- `SHMADD` - Additional shared memory segments
- `BUFFERPOOL` - Data cache configuration for different page sizes

### Q: How to monitor performance?

A:
- Use `onstat -g` commands for various statistics
- Monitor with IBM Informix Performance Manager (included in advanced editions)
- Check query plans with `SET EXPLAIN ON`
