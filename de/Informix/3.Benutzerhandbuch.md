# Benutzerhandbuch für IBM Informix

## Grundlegende Befehle

### Serververwaltung

| Befehl | Beschreibung |
|--------|-------------|
| `oninit` | Informix-Server initialisieren und starten |
| `onmode` | Servermodus ändern (quiescent, online, etc.) |
| `onstat` | Serverstatus und -statistiken anzeigen |
| `ontape` | Backup- und Restore-Dienstprogramm |
| `onbar` | Backup und Restore mit IBM Storage Manager |
| `onspaces` | Speicherbereiche verwalten |
| `onparams` | Serverparameter ändern |

### Häufige Serverbefehle

```bash
# Server initialisieren und starten
oninit

# Im vollständigen Produktionsmodus starten
onmode -vy

# Server herunterfahren
onstat -

# Speicherplatz anzeigen
onstat -d

# Logische Protokolle anzeigen
onstat -l

# Aktive Benutzer anzeigen
onstat -u

# Sperren anzeigen
onstat -k

# Transaktionen anzeigen
onstat -x
```

## Datenbankverbindung

### dbaccess verwenden

```bash
# Interaktiver Modus
dbaccess

# SQL-Skript ausführen
dbaccess datenbank_name < sql_skript.sql

# Mit spezifischem Benutzer verbinden
dbaccess -user meinbenutzer meinedb
```

### Verbindungsparameter

```
Host: localhost (oder Server-IP)
Port: 9088 (Standard)
Datenbank: Ihre_Datenbank
Servername: Ihr_informix_server
```

## SQL-Anweisungen

### Datenbank erstellen

```sql
-- Einfache Datenbank erstellen
CREATE DATABASE meinedb;

-- Datenbank mit gepuffertem Protokoll erstellen
CREATE DATABASE meinedb WITH BUFFERED LOG;

-- Datenbank mit Protokoll erstellen
CREATE DATABASE meinedb WITH LOG;

-- ANSI-konforme Datenbank erstellen
CREATE DATABASE meinedb WITH LOG MODE ANSI;
```

### Tabellen erstellen

```sql
-- Grundlegende Tabellenerstellung
CREATE TABLE kunden (
    kunden_id SERIAL PRIMARY KEY,
    name CHAR(50),
    email CHAR(100),
    erstellungsdatum DATE
);

-- Tabelle mit Einschränkungen
CREATE TABLE bestellungen (
    bestellung_id SERIAL PRIMARY KEY,
    kunden_id INTEGER REFERENCES kunden(kunden_id),
    bestelldatum DATE,
    gesamtbetrag DECIMAL(10,2)
);

-- Temporäre Tabelle
CREATE TEMP TABLE sitzungsdaten (
    id INTEGER,
    daten VARCHAR(255)
);
```

### Daten einfügen

```sql
-- Einzelne Zeile einfügen
INSERT INTO kunden (name, email, erstellungsdatum)
VALUES ('Max Mustermann', 'max@beispiel.de', TODAY);

-- Mehrere Zeilen einfügen
INSERT INTO kunden (name, email, erstellungsdatum)
VALUES 
    ('Anna Schmidt', 'anna@beispiel.de', TODAY),
    ('Peter Weber', 'peter@beispiel.de', TODAY);
```

### Daten abfragen

```sql
-- Alle Spalten auswählen
SELECT * FROM kunden;

-- Spezifische Spalten auswählen
SELECT name, email FROM kunden;

-- WHERE-Klausel
SELECT * FROM kunden WHERE kunden_id > 100;

-- Sortieren
SELECT name, email FROM kunden ORDER BY name;

-- Gruppieren mit Aggregatfunktionen
SELECT kunden_id, COUNT(*) as anzahl_bestellungen
FROM bestellungen GROUP BY kunden_id;

-- Tabellen verbinden
SELECT b.bestellung_id, k.name, b.bestelldatum
FROM bestellungen b, kunden k
WHERE b.kunden_id = k.kunden_id;
```

### Daten aktualisieren

```sql
-- Einzelne Zeile aktualisieren
UPDATE kunden 
SET email = 'neue@email.de' 
WHERE kunden_id = 101;
```

### Daten löschen

```sql
-- Einzelne Zeile löschen
DELETE FROM kunden WHERE kunden_id = 101;

-- Basierend auf Bedingung löschen
DELETE FROM bestellungen WHERE bestelldatum < '2024-01-01';
```

## Konfigurationsdateien

### Wichtige Konfigurationsdateien

| Datei | Zweck |
|-------|-------|
| `$INFORMIXDIR/etc/onconfig` | Haupt-Serverkonfiguration |
| `$INFORMIXDIR/etc/sqlhosts` | Netzwerkkonfiguration |
| `$INFORMIXDIR/etc/$ONCONFIG` | Root-Dbspace-Konfiguration |

### Beispiel-onconfig-Parameter

```
ROOTNAME        rootdbs
ROOTPATH        /opt/IBM/informix/online.root
ROOTOFFSET      0
ROOTSIZE        200000
DBSERVERNAME    ol_informix
```

## Backup und Restore

### ontape verwenden

```bash
# Vollständiges Backup
ontape -s -L 0

# Level 1 Backup
ontape -s -L 1

# Restore
ontape -r
```

### onbar verwenden

```bash
# Backup
onbar -b -L 0

# Restore
onbar -r
```

## Leistungsoptimierung

### Wichtige Speicherparameter

- `SHMVIRTSIZE` - Virtuelle Shared-Memory-Größe
- `SHMADD` - Zusätzliche Shared-Memory-Segmente
- `BUFFERPOOL` - Datencache-Konfiguration

### Indizes

```sql
-- Index erstellen
CREATE INDEX idx_kunden_email ON kunden(email);

-- Zusammengesetzten Index erstellen
CREATE INDEX idx_kunden_name ON kunden(name);
```

### Statistiken

```sql
-- Tabellostatistiken aktualisieren
UPDATE STATISTICS FOR TABLE kunden;

-- Statistiken mit hohem Detailgrad aktualisieren
UPDATE STATISTICS HIGH FOR TABLE kunden;
```
