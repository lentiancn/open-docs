# Guía de Instalación de JMeter

## Requisitos previos

- **Java Development Kit (JDK)**: JMeter requiere Java 8 o superior
- **Sistema operativo**: Windows, Linux o macOS
- **Memoria**: Se recomienda al menos 4GB de RAM para pruebas de carga

## Pasos de instalación

### 1. Descargar JMeter

Visite el sitio web oficial de Apache JMeter: https://jmeter.apache.org/

Descargue la última versión (generalmente un archivo .zip o .tar.gz).

### 2. Extraer el archivo

Extraiga el archivo descargado a la ubicación deseada:

```bash
# Linux/Mac
tar -xvf apache-jmeter-x.x.x.tgz

# Windows
# Use 7-Zip o WinRAR para extraer
```

### 3. Configurar variables de entorno (Opcional)

Para mayor comodidad, puede agregar JMeter a su PATH del sistema:

```bash
# Linux/Mac (agregar a ~/.bashrc o ~/.zshrc)
export JMETER_HOME=/path/to/apache-jmeter-x.x.x
export PATH=$PATH:$JMETER_HOME/bin

# Windows
# Agregar a Propiedades del sistema -> Variables de entorno -> PATH
```

### 4. Verificar instalación

Ejecute el siguiente comando para verificar que JMeter está instalado correctamente:

```bash
# Linux/Mac
./jmeter.sh --version

# Windows
jmeter.bat --version
```

### 5. Iniciar GUI de JMeter

```bash
# Linux/Mac
./jmeter.sh

# Windows
jmeter.bat
```

## Estructura de directorios

```
apache-jmeter/
├── bin/          # Scripts ejecutables y ejecutable de JMeter
├── lib/          # Librerías de JMeter
├── lib/ext/      # Plugins y extensiones
├── docs/         # Documentación
├── printable_docs/  # Manual de usuario
└── LICENSE       # Archivo de licencia
```

## Ejecutando en diferentes modos

### Modo GUI (Interactivo)
```bash
jmeter.sh          # Linux/Mac
jmeter.bat         # Windows
```

### Modo CLI (Sin cabeza)
```bash
jmeter -n -t testplan.jmx -l results.jtl -e -o html-report
```

Parámetros:
- `-n`: Ejecutar en modo no-GUI
- `-t`: Archivo de plan de prueba
- `-l`: Archivo de resultados
- `-e`: Generar informe HTML después de la prueba
- `-o`: Directorio de salida para informe HTML

## Solución de problemas de instalación

### Java no encontrado
Asegúrese de que JAVA_HOME esté configurado correctamente:
```bash
export JAVA_HOME=/path/to/jdk
```

### Problemas de memoria
Edite el script de inicio de jmeter para aumentar el tamaño del heap:
```bash
HEAP="-Xms1g -Xmx4g"
```
