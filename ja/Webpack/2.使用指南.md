# Webpack ユーザーガイド

このガイドでは、Webpackを使用してモダンなJavaScriptアプリケーションを構築する方法を説明します。

## クイックスタート

### プロジェクトの作成

```bash
mkdir my-project
cd my-project
npm init -y
npm install webpack webpack-cli --save-dev
```

### エントリーファイルの作成

```javascript
// src/index.js
import './style.css';
import { add } from './math.js';

console.log('Hello Webpack!');
console.log(add(2, 3));
```

```javascript
// src/math.js
export function add(a, b) {
  return a + b;
}
```

```css
/* src/style.css */
body {
  background: #f0f0f0;
  font-family: sans-serif;
}
```

```html
<!-- src/index.html -->
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Webpack App</title>
</head>
<body>
  <h1>Hello Webpack!</h1>
  <script src="index.js"></script>
</body>
</html>
```

### 設定ファイルの作成

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
```

### ビルドの実行

```bash
npx webpack
```

## コアコンセプト

### エントリーポイント（Entry）

```javascript
module.exports = {
  // 単一エントリ
  entry: './src/index.js',
  
  // 複数エントリ
  entry: {
    home: './src/home.js',
    about: './src/about.js',
  },
};
```

### 出力（Output）

```javascript
module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
    clean: true,
    publicPath: '/',
  },
};
```

### ローダー（Loaders）

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader',
      },
    ],
  },
};
```

### プラグイン（Plugins）

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
    new MiniCssExtractPlugin({
      filename: '[name].css',
    }),
  ],
};
```

### モード（Mode）

```javascript
// 開発モード
module.exports = {
  mode: 'development',
  devtool: 'source-map',
};

// 本番モード
module.exports = {
  mode: 'production',
  devtool: false,
};
```

## 一般的なローダー

### CSSの処理

```bash
npm install --save-dev style-loader css-loader
```

```javascript
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader'],
}
```

### Sassの処理

```bash
npm install --save-dev sass sass-loader
```

```javascript
{
  test: /\.scss$/,
  use: ['style-loader', 'css-loader', 'sass-loader'],
}
```

### 画像の処理

```javascript
{
  test: /\.(png|jpg|gif)$/,
  type: 'asset/resource',
}
```

### フォントの処理

```javascript
{
  test: /\.(woff|woff2|eot|ttf|otf)$/,
  type: 'asset/resource',
}
```

## 開発サーバー

### インストール

```bash
npm install --save-dev webpack-dev-server
```

### 設定

```javascript
module.exports = {
  devServer: {
    static: './dist',
    hot: true,
    port: 3000,
    open: true,
    compress: true,
  },
};
```

### 起動

```bash
webpack serve --open
```

またはpackage.jsonにスクリプトを追加：

```json
{
  "scripts": {
    "start": "webpack serve --mode development",
    "build": "webpack --mode production"
  }
}
```

## コード分割

### 動的インポート

```javascript
// ルート遅延読み込み
const Home = () => import('./Home.vue');
const About = () => import('./About.vue');

// 条件付き読み込み
if (condition) {
  import('./module').then(module => {
    module.doSomething();
  });
}
```

### SplitChunks

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',

      },
    },
  },
};
```

## 環境設定

### 複数環境設定

```javascript

const prodConfig = require('./webpack.prod');

module.exports = (env, argv) => {
  return argv.mode === 'production' ? prodConfig : devConfig;
};
```

## パフォーマンス最適化

### 1. productionモードを使用

```javascript
module.exports = {
  mode: 'production',
};
```

### 2. Tree Shakingを有効化

```javascript
// package.json
{
  "sideEffects": false
}
```

### 3. コード分割

```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
  },
}
```

### 4. キャッシュ最適化

```javascript
output: {
  filename: '[name].[contenthash].js',
  chunkFilename: '[name].[contenthash].chunk.js',
}
```

### 5. 圧縮

```javascript
const TerserPlugin = require('terser-webpack-plugin');

optimization: {
  minimizer: [new TerserPlugin()],
}
```

## よくある問題

### モジュール解決に失敗する場合

resolve設定を確認：

```javascript
resolve: {
  extensions: ['.js', '.json'],
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
}
```

### ビルドが遅い場合

- cacheを使用
- node_modulesを除外
- thread-loaderを使用

```javascript
{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: 'thread-loader',
    options: {
      workers: 2,
    },
  },
}
```
