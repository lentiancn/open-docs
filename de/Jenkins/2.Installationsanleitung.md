# Installationsanleitung

## Systemanforderungen

Bevor Sie Jenkins installieren, stellen Sie sicher, dass Ihr System die folgenden Mindestanforderungen erfüllt:

### Minimale empfohlene Konfiguration

- 256MB freier Arbeitsspeicher
- 1GB freier Festplattenspeicher (10GB bei Ausführung als Docker-Container empfohlen)

### Empfohlene Konfiguration für kleine Teams

- 1GB+ freier Arbeitsspeicher
- 50GB+ freier Festplattenspeicher

### Softwareanforderungen

- Java 8 oder höher (entweder JRE oder JDK)
- Hinweis: Bei Ausführung von Jenkins als Docker-Container muss Java nicht auf dem Host installiert werden

## Installationsplattformen

Jenkins unterstützt mehrere Installationsmethoden. Dieser Leitfaden behandelt die gängigsten Ansätze.

## Docker-Installation

Docker ist der empfohlene Weg, Jenkins zu installieren, da es eine isolierte Umgebung bietet und den Konfigurationsprozess vereinfacht.

### Docker installieren

Zunächst müssen Sie Docker auf Ihrem System installieren. Besuchen Sie die offizielle Docker-Website und wählen Sie die entsprechende Docker Community Edition-Version für Ihr Betriebssystem.

### Jenkins herunterladen und ausführen

Unter macOS und Linux öffnen Sie ein Terminal und führen Sie folgenden Befehl aus:

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

Unter Windows öffnen Sie eine Eingabeaufforderung und führen Sie einen ähnlichen Befehl aus:

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

Parametererklärung:

- `-u root`: Container als Root-Benutzer ausführen
- `--rm`: Container beim Stoppen automatisch löschen
- `-d`: Im Hintergrund ausführen (Getrennter Modus)
- `-p 8080:8080`: Container-Port 8080 auf Host-Port 8080 mappen
- `-p 50000:50000`: Container-Port 50000 auf Host-Port 50000 mappen (für JNLP-Agent-Kommunikation)
- `-v jenkins-data:/var/jenkins_home`: Jenkins-Daten persistent speichern
- `-v /var/run/docker.sock:/var/run/docker.sock`: Jenkins erlauben, Docker-Befehle in Pipelines auszuführen
- `jenkinsci/blueocean`: Offizielles Jenkins-Image mit Blue Ocean-Oberfläche

### Auf Jenkins-Container zugreifen

Um auf das Innere des Jenkins-Containers zuzugreifen, verwenden Sie:

```bash
docker exec -it jenkins-blueocean bash
```

### Jenkins-Protokolle anzeigen

Jenkins-Konsolenprotokolle anzeigen mit:

```bash
docker logs jenkins-blueocean
```

## WAR-Datei-Installation

Die Jenkins-WAR-Dateiversion kann auf jedem Java-fähigen Betriebssystem installiert werden.

### Installationsschritte

1. Laden Sie das neueste stabile Jenkins-WAR-Paket von der offiziellen Website herunter
2. Öffnen Sie ein Terminal im Download-Verzeichnis
3. Führen Sie den Befehl aus: `java -jar jenkins.war`
4. Navigieren Sie zu http://localhost:8080 und warten Sie auf die Seite "Jenkins entsperren"

### Hinweise

- Die WAR-Datei-Installation installiert Blue Ocean nicht automatisch; es muss separat über die Plugin-Verwaltung installiert werden
- Sie können den Port mit `--httpPort` ändern, z.B.: `java -jar jenkins.war --httpPort=9090`

## macOS-Installation

### Paket verwenden

1. Laden Sie das macOS-Jenkins-Paket herunter
2. Doppelklicken Sie zum Öffnen und folgen Sie dem Installationsassistenten

### Homebrew verwenden

Neueste Version installieren:

```bash
brew install jenkins
```

LTS-Version installieren:

```bash
brew install jenkins-lts
```

## Linux-Installation (Debian/Ubuntu)

Auf Debian-basierten Distributionen kann Jenkins über apt installiert werden:

```wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins

Nach der Installation wird Jenkins:

- Als Daemon beim Booten starten
- Einen Jenkins-Benutzer zum Ausführen des Dienstes erstellen
- Protokolle nach /var/log/jenkins/jenkins.log ausgeben
- Standardmäßig auf Port 8080 lauschen

## Einrichtungsassistent nach der Installation

Beim ersten Start von Jenkins beginnt der Einrichtungsassistent.

### Jenkins entsperren

1. Navigieren Sie zu http://localhost:8080
2. Wenn die Seite "Jenkins entsperren" erscheint, kopieren Sie das automatisch generierte Passwort aus den Konsolenprotokollen
3. Fügen Sie das Passwort in das Administrator-Passwort-Feld ein und klicken Sie auf "Weiter"

### Jenkins-Plugins anpassen

Auf der Plugin-Installationsseite können Sie wählen:

- Empfohlene Plugins installieren: Installiert einen empfohlenen Satz von Plugins basierend auf häufigen Anwendungsfällen
- Plugins zum Installieren auswählen: Wählen Sie manuell aus, welche Plugins installiert werden sollen

### Ersten Administrator-Benutzer erstellen

Erstellen Sie abschließend ein Administratorkonto durch Angabe von:

- Benutzername
- Passwort
- Passwort bestätigen
- Vollständiger Name
- E-Mail-Adresse

Klicken Sie auf "Speichern und Fertig", dann auf "Jenkins verwenden", um auf die Hauptoberfläche zuzugreifen.

## Installation überprüfen

Überprüfen Sie die Installation durch:

1. Navigieren Sie zu http://localhost:8080
2. Melden Sie sich mit dem erstellten Admin-Konto an
3. Überprüfen Sie, ob der Systemstatus normal ist

## Häufige Probleme

### Port bereits belegt

Wenn Port 8080 von einem anderen Programm verwendet wird:

- Docker-Methode: -p Parameter ändern
- WAR-Methode: --httpPort Parameter verwenden

### Unzureichender Speicher

Wenn der Systemspeicher begrenzt ist:

- Docker mit Container-Ressourcenbeschränkungen verwenden
- JVM-Heap-Größe in Jenkins-Startparametern anpassen

### Langer erster Start

Der erste Start lädt und installiert Plugins, was Zeit dauern kann. Bitte haben Sie Geduld.

## Nächste Schritte

Nach der Installation wird empfohlen zu lernen:

- Erste Pipeline-Aufgabe erstellen
- Quellcode (wie Git) konfigurieren
- Build-Trigger einrichten
- Test- und Bereitstellungstools integrieren
