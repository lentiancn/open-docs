# JUnit 5 - Einführung

## Überblick

JUnit 5 ist die nächste Generation von JUnit, ein programmierfreundliches Testframework für Java und die JVM. Es bietet eine robuste Testgrundlage zum Schreiben von Tests in Java.

## Was ist JUnit?

JUnit ist ein Open-Source-Framework zum Schreiben und Ausführen von Unit-Tests. Es ist Teil der xUnit-Familie von Testframeworks und ist der De-facto-Standard für das Testen von Java-Anwendungen.

## Hauptmerkmale von JUnit 5

### Modulare Architektur
JUnit 5 besteht aus drei verschiedenen Modulen:
- **JUnit Platform**: Grundlage zum Starten von Testframeworks auf der JVM
- **JUnit Jupiter**: Neues Programmiermodell zum Schreiben von Tests (JUnit 5)
- **JUnit Vintage**: Engine zum Ausführen von JUnit 3 und 4 Tests auf der Platform

### Moderne Funktionen
- **Beschreibende Testnamen**: Unterstützung für Anzeigenamen mit Leerzeichen und Sonderzeichen
- **Verschachtelte Tests**: Unterstützung für verschachtelte innere Testklassen
- **Parametrisierte Tests**: Integrierte Unterstützung für parametrisierte Tests
- **Wiederholte Tests**: Tests mehrfach ausführen
- **Dynamische Tests**: Zur Laufzeit generierte Tests
- **Tagging und Filterung**: Tests mit Tags organisieren

### Erweiterungsmodell
- Ersetzt das alte Runner-Modell durch eine flexiblere Extension API
- Dependency Injection für Testkonstruktoren und -methoden
- Bedingte Testausführung

### Verbesserte Assertions
- Gruppierte Assertions für bessere Fehlerberichterstattung
- Exception-Assertions mit Lambdas
- Timeout-Assertions
- Unterstützung für benutzerdefinierte Assertions-Meldungen

## Gründe für ein Upgrade auf JUnit 5

1. **Moderne Java-Funktionen**: Nutzt Java 8+ Funktionen wie Lambdas und Streams
2. **Bessere Testorganisation**: Verschachtelte Klassen und Tagging verbessern die Teststruktur
3. **Flexible Erweiterungen**: Leistungsfähigeres Erweiterungsmodell als JUnit 4
4. **Parallele Ausführung**: Bessere Unterstützung für parallele Testausführung
5. **Klarere Fehler**: Verbesserte Assertions-Meldungen erleichtern das Debugging

## Erste Schritte

Um JUnit 5 zu verwenden, müssen Sie die JUnit Jupiter-Abhängigkeit zu Ihrem Projekt hinzufügen. Es kann mit Maven, Gradle oder jedem Build-Tool verwendet werden.
