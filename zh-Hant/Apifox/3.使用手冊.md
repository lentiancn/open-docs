# 使用手冊

本手冊詳細說明 Apifox 的核心功能和使用方法，包括專案管理、介面設計、調試、Mock、自動化測試等。

## 專案管理

### 建立專案

在 Apifox 中，專案是組織 API 的最大單位。

```bash
# 建立新專案步驟
1. 點擊左側「專案」列表旁邊的「+」
2. 輸入專案名稱
3. 選擇專案類型
4. 選擇基礎範本（可選）
5. 點擊「建立」
```

### 專案設定

每個專案都有獨立的設定選項：

- **基本資訊**：專案名稱、描述、圖示
- **環境變數**：定義開發、測試、生產等不同環境的變數
- **認證**：設定全域認證方式
- **匯入/匯出**：專案資料的匯入匯出

### 環境管理

Apifox 支援多環境設定：

```bash
# 環境變數使用
在請求中使用 {{變數名}} 引用環境變數
例如：{{baseUrl}}/api/users

切換環境：點擊頂部環境選擇器選擇不同環境
```

## 介面設計

### 建立介面

介面是 API 的具體定義：

```bash
# 建立介面步驟
1. 在專案中點擊「+」選擇「新建介面」
2. 填寫基本資訊：
   - 介面名稱：使用者列表
   - 請求路徑：/api/users
   - 請求方法：GET
3. 定義請求參數
4. 定義回應結構
5. 儲存
```

### 請求參數

支援以下類型的請求參數：

- **路徑參數**：如 /api/users/:id 中的 id
- **查詢參數**：URL 中的 ?name=test
- **請求頭**：如 Authorization
- **請求主體**：JSON、表單、XML 等格式

### 回應定義

為每個介面定義成功和失敗的回應：

```json
{
  "成功回應 200": {
    "id": 1,
    "name": "張三",
    "email": "zhangsan@example.com"
  },
  "錯誤回應 404": {
    "code": 404,
    "message": "使用者不存在"
  }
}
```

### 介面列表管理

支援分組管理介面，可以按模組或功能進行分類：

- 拖曳調整順序
- 批次操作（批次刪除、批次匯出等）
- 搜尋和篩選

## 介面調試

### 傳送請求

在介面詳情頁面，點擊「傳送」按鈕即可傳送請求：

```bash
# 基本調試步驟
1. 開啟介面
2. 選擇環境
3. 設定參數值
4. 點擊「傳送」
5. 查看回應
```

### 認證設定

支援多種認證方式：

- **None**：無認證
- **Bearer Token**：JWT 認證
- **Basic Auth**：基本認證
- **OAuth 2.0**：OAuth2 認證
- **自訂認證**：根據需要自訂認證頭

### 請求前置處理

可以設定請求傳送前的腳本：

```javascript
// 範例：加入時間戳記
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// 範例：產生隨機 ID
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### 回應後置處理

可以設定收到回應後的處理腳本：

```javascript
// 範例：擷取回應資料
const response = pm.response.json();
console.log('回應資料:', response);

// 範例：設定環境變數
pm.environment.set('access_token', response.data.token);
```

## API Mock

### 智慧 Mock

Apifox 最強大的功能之一，只要定義資料結構，系統自動產生 Mock 資料：

```bash
# 開啟智慧 Mock
1. 開啟介面
2. 點擊「Mock」標籤
3. 開啟「智慧 Mock」
4. 儲存
```

### Mock 規則

支援自訂 Mock 規則：

- `@string`：產生隨機字串
- `@integer`：產生隨機整數
- `@float`：產生隨機浮點數
- `@boolean`：產生隨機布林值
- `@date`：產生隨機日期
- `@email`：產生隨機電子郵件
- `@url`：產生隨機 URL
- `@guid`：產生 UUID

### 範例

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

產生的 Mock 資料：

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock 伺服器

Apifox 提供內建 Mock 伺服器：

```bash
# 取得 Mock 位址
在介面詳情頁面，複製 Mock URL
例如：https://mock.apifox.com/api/users
```

## 自動化測試

### 建立測試案例

```bash
# 建立測試步驟
1. 開啟介面
2. 點擊「測試」標籤
3. 點擊「建立測試」
4. 設定測試案例名稱
5. 加入判斷條件
6. 儲存
```

### 判斷類型

支援多種判斷條件：

- **狀態碼判斷**：檢查回應狀態碼
- **JSON 判斷**：檢查回應 JSON 資料
- **回應時間判斷**：檢查回應時間
- **回應頭判斷**：檢查回應頭

### 範例

```javascript
// 狀態碼判斷
pm.test("狀態碼為 200", function() {
    pm.response.to.have.status(200);
});

// JSON 判斷
pm.test("回傳使用者列表", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// 回應時間判斷
pm.test("回應時間小於 500ms", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### 測試套件

可以將多個介面的測試案例組織成測試套件：

1. 建立測試套件
2. 加入測試案例
3. 設定執行順序
4. 執行套件

### 產生測試報告

執行測試後，可以產生詳細的測試報告：

- 通過率統計
- 失敗案例分析
- 回應時間統計

## 團隊協作

### 邀請成員

```bash
# 邀請團隊成員
1. 進入專案設定
2. 點擊「成員管理」
3. 點擊「邀請成員」
4. 輸入成員電子郵件
5. 設定角色權限
6. 傳送邀請
```

### 角色權限

- **管理員**：專案全部權限
- **編輯者**：可以編輯介面和執行測試
- **檢視者**：只能查看，不能編輯

### 即時同步

團隊成員登入後，專案資料會自動即時同步：

- 介面變更自動同步
- 測試結果自動同步
- 團隊成員可以看到最新內容

### 評論功能

可以在介面上加入評論：

1. 開啟介面
2. 點擊「評論」
3. 輸入評論內容
4. @提及團隊成員

## 資料匯入匯出

### 支援格式

**匯入：**

- Postman Collection (v2.1)
- Swagger/OpenAPI (2.0, 3.0)
- HAR
- Apiguardian
- YApi

**匯出：**

- OpenAPI 3.0
- Swagger 2.0
- Markdown
- HTML
- Apifox 格式

## 快捷鍵

常用快捷鍵：

- `Ctrl/Cmd + S`：儲存
- `Ctrl/Cmd + Enter`：傳送請求
- `Ctrl/Cmd + B`：格式化 JSON
- `Ctrl/Cmd + /`：註解/取消註解

## 最佳實踐

1. **命名規範**：介面名稱清楚描述功能
2. **完整註解**：為每個參數加入說明
3. **合理分組**：按模組組織介面
4. **使用環境**：區分開發、測試、生產環境
5. **撰寫測試**：為重要介面撰寫自動化測試
6. **定期同步**：保持本地和雲端資料一致
