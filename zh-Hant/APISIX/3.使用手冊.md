# 使用手冊

本手冊詳細介紹 Apache APISIX 的核心功能和使用方法，包括路由配置、上游管理、負載均衡、限速、身分驗證等關鍵功能。

## 基本概念

### Route（路由）

Route（路由）是存取上游目標的路徑。在 APISIX 中，路由首先通過預定的規則來匹配客戶端請求，然後載入和執行相應的插件，最後將請求轉發至特定的上游服務。

一個最簡單的路由僅由匹配路徑和上游位址兩個資訊組成。

### Upstream（上游）

Upstream（上游）是一組具備相同功能的節點集合，是對虛擬主機的抽象。Upstream 可以通過預先配置的規則對多個服務節點進行負載均衡。

### Consumer（消費者）

Consumer（消費者）是指使用 API 的應用或開發人員。在 APISIX 中，消費者需要一個全域唯一的名稱，並綁定一個身分驗證插件。

### Plugin（插件）

APISIX 提供了豐富的插件系統，用於擴展閘道器功能。插件可以在路由、消費者或全域級別進行配置。

## 配置路由

通過管理 API 可以輕鬆建立路由。以下示例建立一個將請求轉發到 httpbin.org 的路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "getting-started-ip",
  "uri": "/ip",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

建立成功將返回 `HTTP/1.1 201 Created`。

### 驗證路由

```bash
curl "http://127.0.0.1:9080/ip"
```

正常情況下會返回類似以下內容：

```json
{
  "origin": "183.94.122.205"
}
```

### 路由匹配規則

APISIX 支援多種路由匹配方式：

**URI 匹配：**

```json
{
  "uri": "/api/v1/*",
  "uri": ["/api/v1/*", "/api/v2/*"]
}
```

**Host 匹配：**

```json
{
  "host": "example.com",
  "host": ["example.com", "*.example.com"]
}
```

**請求方法匹配：**

```json
{
  "methods": ["GET", "POST"]
}
```

**Header 匹配：**

```json
{
  "headers": {
    "Content-Type": "application/json",
    "X-API-Key": "*"
  }
}
```

## 負載均衡

APISIX 支援多種負載均衡算法，包括輪詢、加權輪詢、最小連線數等。

### 配置加權負載均衡

建立一個具有多個上游服務的路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "load-balance-route",
  "uri": "/headers",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:443": 1,
      "mock.api7.ai:443": 1
    },
    "pass_host": "node",
    "scheme": "https"
  }
}'
```

**參數說明：**

- `type`：負載均衡算法類型，支援 `roundrobin`（輪詢）、`weighted_roundrobin`（加權輪詢）、`least_conn`（最小連線數）
- `nodes`：上游服務節點，格式為 `{"host:port": weight}`
- `pass_host`：設置為 `node` 時，將請求頭傳遞給上游
- `scheme`：上游協定，支援 `http` 和 `https`

### 驗證負載均衡

產生 100 個請求測試均衡效果：

```bash
hc=$(seq 100 | xargs -I {} curl "http://127.0.0.1:9080/headers" -sL | grep "httpbin" | wc -l); echo httpbin.org: $hc, mock.api7.ai: $((100 - $hc))
```

請求會幾乎平均分配給兩個上游服務。

## 限速

APISIX 提供靈活的限速功能，通過 `limit-count` 插件實現。

### 啟用限速

為已有路由添加限速插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "count": 2,
      "time_window": 10,
      "rejected_code": 503
    }
  }
}'
```

**參數說明：**

- `count`：時間視窗內允許的最大請求數
- `time_window`：時間視窗大小（秒）
- `rejected_code`：超過限制時返回的 HTTP 狀態碼

上述配置將傳入流量限制為每 10 秒最多 2 個請求。

### 驗證限速

產生 100 個請求測試限速效果：

```bash
count=$(seq 100 | xargs -I {} curl "http://127.0.0.1:9080/ip" -I -sL | grep "503" | wc -l); echo "200": $((100 - count)), "503": $count
```

結果：2 個請求成功（200），98 個請求被拒絕（503）。

### 停用限速

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "_meta": {
        "disable": true
      }
    }
  }
}'
```

### 進階限速配置

**使用變數：**

```json
{
  "plugins": {
    "limit-count": {
      "count": 100,
      "time_window": 60,
      "key": "remote_addr"
    }
  }
}
```

**使用 Redis 叢集：**

```json
{
  "plugins": {
    "limit-count": {
      "count": 100,
      "time_window": 60,
      "key": "remote_addr",
      "policy": "redis",
      "redis_host": "127.0.0.1",
      "redis_port": 6379,
      "redis_password": "",
      "redis_database": 1
    }
  }
}
```

## 身分驗證

APISIX 支援多種身分驗證方式，包括 Key Auth、Basic Auth、JWT、OAuth2 等。

### Key Authentication（密鑰驗證）

#### 建立消費者

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/consumers" -X PUT -d '
{
  "username": "tom",
  "plugins": {
    "key-auth": {
      "key": "secret-key"
    }
  }
}'
```

> ⚠️ 生產環境請使用複雜的密鑰。

#### 為路由啟用密鑰驗證

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "key-auth": {}
  }
}'
```

#### 驗證身分驗證

**不帶密鑰的請求：**

```bash
curl -i "http://127.0.0.1:9080/ip"
```

返回：`HTTP/1.1 401 Unauthorized`

**攜帶錯誤密鑰：**

```bash
curl -i "http://127.0.0.1:9080/ip" -H 'apikey: wrong-key'
```

返回：`HTTP/1.1 401 Unauthorized`

**攜帶正確密鑰：**

```bash
curl -i "http://127.0.0.1:9080/ip" -H 'apikey: secret-key'
```

返回：`HTTP/1.1 200 OK`

### 其他身分驗證方式

**JWT Authentication：**

```json
{
  "plugins": {
    "jwt-auth": {
      "key": "user-key",
      "secret": "my-secret-key"
    }
  }
}
```

**Basic Authentication：**

```json
{
  "plugins": {
    "basic-auth": {
      "hide_credentials": false
    }
  }
}
```

**HMAC Authentication：**

```json
{
  "plugins": {
    "hmac-auth": {
      "secret_key": "my-secret-key"
    }
  }
}
```

## 服務治理

### 灰度發布

APISIX 支援基於權重的灰度發布和金絲雀發布：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "canary-route",
  "uri": "/api/*",
  "upstream": {
    "type": "weighted_roundrobin",
    "nodes": [
      {"host": "v1.example.com", "port": 80, "weight": 90},
      {"host": "v2.example.com", "port": 80, "weight": 10}
    ]
  }
}'
```

### 請求改寫

使用 `proxy-rewrite` 插件改寫請求：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "rewrite-route",
  "uri": "/old-path/*",
  "plugins": {
    "proxy-rewrite": {
      "uri": "/new-path/$uri",
      "headers": {
        "X-New-Header": "value"
      }
    }
  },
  "upstream": {
    "nodes": {
      "backend.example.com:80": 1
    }
  }
}'
```

### 響應快取

使用 `proxy-cache` 插件快取上游響應：

```json
{
  "plugins": {
    "proxy-cache": {
      "cache_key": ["uri", "query_string"],
      "cache_http_status": [200, 301, 404],
      "cache_ttl": 60
    }
  }
}
```

## 插件配置

### 插件執行順序

可以通過 `plugins` 配置中的 `phase` 指定插件執行階段：

- `rewrite`：請求改寫階段
- `access`：存取控制階段
- `before_proxy`：代理前階段
- `header_filter`：響應頭過濾階段
- `body_filter`：響應體過濾階段
- `log`：日誌記錄階段

### 停用插件

將插件的 `_meta.disable` 設置為 `true`：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/route-id" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "_meta": {
        "disable": true
      }
    }
  }
}'
```

## 監控與可觀測性

### 整合 Prometheus

```json
{
  "plugins": {
    "prometheus": {
      "prefer_name": true
    }
  }
}
```

存取指標：

```bash
curl "http://127.0.0.1:9091/prometheus"
```

### 日誌記錄

APISIX 支援多種日誌輸出：

**HTTP 日誌：**

```json
{
  "plugins": {
    "http-logger": {
      "uri": "http://log-server:8080/logs"
    }
  }
}
```

**錯誤日誌：**

```json
{
  "plugins": {
    "error-log-logger": {
      "level": "ERROR",
      "uri": "http://log-server:8080/logs"
    }
  }
}
```

## 最佳實踐

1. **使用有意義的 ID**：為路由、上游等資源使用描述性 ID
2. **合理設置超時**：根據上游回應時間設置合理的超時參數
3. **啟用健康檢查**：為關鍵服務配置主動和被動健康檢查
4. **配置適當的限速**：根據業務需求和伺服器能力設置限速規則
5. **啟用身分驗證**：生產環境必須啟用身分驗證
6. **使用 TLS**：對外暴露的介面應使用 HTTPS
7. **監控關鍵指標**：配置 Prometheus 監控並設置警報
