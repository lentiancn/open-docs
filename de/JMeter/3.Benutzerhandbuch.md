# JMeter Benutzerhandbuch

Dieses Handbuch bietet detaillierte Anweisungen zur Verwendung von Apache JMeter zum Erstellen und Ausführen von Lasttests.

## Grundstruktur eines Testplans

Ein JMeter-Testplan besteht aus den folgenden Hauptkomponenten:

### 1. Testplan

Der Testplan ist der Stammknoten aller Tests und enthält Konfigurationsinformationen für den gesamten Test. Er kann enthalten:
- Benutzerdefinierte Variablen
- Globale Eigenschaftseinstellungen
- Thread-Gruppen-Konfiguration

### 2. Thread-Gruppe

Thread-Gruppen sind die Ausführungseinheiten der Tests und definieren:
- **Anzahl der Threads**: Anzahl der simulierten Benutzer
- **Anlaufzeit**: Zeit, die zum Starten aller Threads benötigt wird
- **Schleifenanzahl**: Wie oft jeder Thread den Test ausführt
- **Scheduler**: Startzeit und Dauer des Tests festlegen

### 3. Sampler

Sampler senden Anfragen an Server. Häufige Sampler umfassen:

- **HTTP-Anfrage**: HTTP/HTTPS-Anfragen senden
- **FTP-Anfrage**: FTP-Server testen
- **JDBC-Anfrage**: Datenbankverbindungen testen
- **LDAP-Anfrage**: LDAP-Verzeichnisse testen
- **JMS-Nachricht**: Nachrichtenwarteschlangen testen
- **TCP-Sampler**: TCP-Dienste testen

### 4. Logik-Controller

Logik-Controller steuern die Ausführungsreihenfolge der Anfragen:

- **Einfacher Controller**: Anfragen gruppieren
- **Schleifen-Controller**: Wiederholte Ausführung von Anfragen
- **Nur-einmal-Controller**: Nur einmal während des Tests ausführen
- **Wechsel-Controller**: Zwischen Anfragen abwechseln
- **Zufalls-Controller**: Zufällig Anfragen zur Ausführung auswählen
- **Durchsatz-Controller**: Ausführungshäufigkeit von Anfragen steuern

### 5. Listener

Listener zeigen und analysieren Testergebnisse:

- **Ergebnisbaum anzeigen**: Detaillierte Antwort für jede Anfrage anzeigen
- **Aggregatbericht**: Aggregierte Leistungsstatistiken anzeigen
- **Graphische Ergebnisse**: Testdaten graphisch anzeigen
- **Antwortzeit-Graph**: Antwortzeittrends anzeigen

### 6. Konfigurationselemente

Konfigurationselemente legen Standardwerte und gemeinsame Daten fest:

- **HTTP-Anfrage-Standardwerte**: Standardparameter für HTTP-Anfragen festlegen
- **Benutzerdefinierte Variablen**: Wiederverwendbare Variablen definieren
- **CSV-Datensatzkonfiguration**: Testdaten aus CSV-Dateien lesen
- **HTTP-Cookie-Manager**: Cookies verwalten
- **HTTP-Header-Manager**: Anfrageheader-Informationen festlegen

### 7. Timer

Timer legen Verzögerungen zwischen Anfragen fest:

- **Fester Timer**: Feste Verzögerung festlegen
- **Gaußscher Zufalls-Timer**: Zufällige Verzögerung festlegen
- **Synchronisations-Timer**: Gleichzeitige Anfragen simulieren
- **Durchsatz-Timer**: Anfragedurchsatz steuern

### 8. Vor- und Nachverarbeitung

- **Vorverarbeitung**: Operationen ausführen, bevor die Anfrage gesendet wird
- **Nachverarbeitung**: Daten extrahieren, nachdem die Antwort empfangen wurde

## Ihren ersten Testplan erstellen

### Schritt 1: Testplan erstellen

1. Starten Sie JMeter
2. Wählen Sie "Datei" -> "Neu" um einen neuen Testplan zu erstellen
3. Benennen Sie Ihren Testplan

### Schritt 2: Thread-Gruppe hinzufügen

1. Klicken Sie mit der rechten Maustaste auf den Testplan
2. Wählen Sie "Hinzufügen" -> "Threads (Benutzer)" -> "Thread-Gruppe"
3. Legen Sie Thread-Anzahl (z.B. 10), Anlaufzeit (z.B. 5 Sekunden), Schleifenanzahl (z.B. 1) fest

### Schritt 3: HTTP-Anfrage hinzufügen

1. Klicken Sie mit der rechten Maustaste auf die Thread-Gruppe
2. Wählen Sie "Hinzufügen" -> "Sampler" -> "HTTP-Anfrage"
3. Servername oder IP, Portnummer, Anfragepfad usw. festlegen

### Schritt 4: Listener hinzufügen

1. Klicken Sie mit der rechten Maustaste auf Thread-Gruppe oder HTTP-Anfrage
2. Wählen Sie "Hinzufügen" -> "Listener" -> "Ergebnisbaum anzeigen"
3. Fügen Sie "Aggregatbericht" hinzu, um Leistungsstatistiken anzuzeigen

### Schritt 5: Test ausführen

1. Klicken Sie auf die "Start"-Schaltfläche in der Symbolleiste
2. Beobachten Sie die in den Listenern angezeigten Ergebnisdaten

## Parametrisierung und datengesteuertes Testen

### CSV-Datendateien verwenden

1. "CSV-Datensatzkonfiguration"-Element hinzufügen
2. Dateinamen und Variablennamen festlegen
3. ${Variablenname} in Anfragen verwenden, um auf Daten zu verweisen

### Benutzerdefinierte Variablen verwenden

1. "Benutzerdefinierte Variablen"-Element hinzufügen
2. Variablennamen und -werte definieren
3. ${Variablenname} in Anfragen verwenden

## Korrelationshandhabung

### Regulären Ausdruck-Extraktor verwenden

Wenn Anfragen dynamische Daten erfordern (wie Session-ID):

1. "Regulären Ausdruck-Extraktor" hinzufügen
2. Regex-Muster zum Abgleichen festlegen
3. Die extrahierte Variable in nachfolgenden Anfragen verwenden

### JSON-Extraktor verwenden

Für JSON-Antworten:

1. "JSON-Extraktor" hinzufügen
2. JSON Path-Ausdruck festlegen
3. Erforderliche JSON-Feldwerte extrahieren

## Verteilte Tests

Wenn ein einzelner Computer nicht genügend Last erzeugen kann, können verteilte Tests verwendet werden:

### Master-Konfiguration

1. Bearbeiten Sie jmeter.properties im bin-Verzeichnis des Masters
2. Legen Sie remote_hosts fest, um auf Slave-Adressen zu zeigen
3. Starten Sie JMeter

### Slave-Konfiguration

1. Führen Sie jmeter-server auf jedem Slave-Computer aus
2. Stellen Sie sicher, dass die Firewall die Kommunikation über die angegebenen Ports erlaubt

### Verteilten Test starten

1. Wählen Sie "Remote-Start" auf dem Master
2. Wählen Sie welche Slaves gestartet werden sollen oder starten Sie alle

## Leistungsoptimierungstipps

- Verwenden Sie den Befehlszeilenmodus, um GUI-Ressourcenverbrauch zu reduzieren
- Legen Sie geeignete Thread-Anzahl und Anlaufzeit fest, um sofortige hohe Last zu vermeiden
- Deaktivieren Sie unnötige Listener, zeigen Sie nur bei Bedarf Ergebnisse
- Verwenden Sie Zusicherungen, um Antwortinhalt zu überprüfen
- Bereinigen Sie regelmäßig ungenutzte Testpläne
- Für großangelegte Tests sollten Sie verteilte Tests in Betracht ziehen
