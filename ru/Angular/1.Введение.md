# Введение

Angular — это открытый фреймворк для веб-приложений, разработанный и поддерживаемый компанией Google. Он используется для создания современных веб-приложений и является преемником AngularJS (Angular 1.x), полностью переписанным с нуля. TypeScript стал основным языком разработки.

Angular предлагает полноценное решение для фронтенд-разработки, включающее компонентную архитектуру, двустороннее связывание данных, внедрение зависимостей, маршрутизацию, обработку форм, HTTP-клиент и многое другое. Благодаря этим возможностям разработчики могут быстро создавать производительные и удобные в поддержке одностраничные приложения (SPA).

## Основные концепции

### Компонент (Component)

Компонент — это базовый строительный блок приложения Angular. Каждый компонент состоит из трёх частей:

- **Шаблон (Template)** — определяет структуру представления
- **Стили (Styles)** — определяют внешний вид представления
- **Логика (Logic)** — определяет поведение компонента

### Модуль (Module)

Модули используются для организации кода, объединяя связанные компоненты, сервисы и директивы. Каждое приложение Angular содержит как минимум один корневой модуль (AppModule).

### Сервис (Service)

Сервисы инкапсулируют переиспользуемую бизнес-логику и предоставляются компонентам через механизм внедрения зависимостей.

### Директива (Directive)

Директивы расширяют возможности HTML-элементов. Они бывают структурными (например, *ngIf, *ngFor) и атрибутивными (например, ngClass, ngStyle).

## Основные возможности

### Компонентная архитектура

Angular использует компонентный подход: каждый компонент представляет собой независимую функциональную единицу, которую можно переиспользовать и комбинировать.

### TypeScript

Angular использует TypeScript как основной язык разработки, что обеспечивает строгую типизацию, объектно-ориентированное программирование и современные возможности JavaScript.

### Внедрение зависимостей

Встроенная система внедрения зависимостей делает код более модульным, тестируемым и удобным в поддержке.

### Двустороннее связывание данных

Данные между шаблоном и компонентом синхронизируются автоматически, что избавляет от необходимости вручную манипулировать DOM.

### Маршрутизация

Мощная система маршрутизации с поддержкой ленивой загрузки, охранных маршрутов, передачи параметров и других возможностей.

### HTTP-клиент

Встроенный HTTP-клиент с поддержкой перехватчиков, преобразователей запросов и ответов и других продвинутых функций.

### Обработка форм

Доступны два подхода: реактивные формы и формы на основе шаблонов. Оба позволяют реализовать сложную валидацию.

### Реактивное программирование

Полная поддержка RxJS для работы с асинхронными операциями и потоками данных.

## Области применения

- Корпоративные веб-приложения
- Одностраничные приложения (SPA)
- Прогрессивные веб-приложения (PWA)
- Гибридные мобильные приложения (в связке с Ionic)
- Настольные приложения (в связке с Electron)

## История версий

- AngularJS (1.x): выпущен в 2010 году
- Angular 2: выпущен в 2016 году, полностью переписан
- Angular 17: последняя стабильная версия, выпущена в 2023 году

Angular имеет активное сообщество и отличную документацию. Это один из популярных выборов для создания современных веб-приложений.
