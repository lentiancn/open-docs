# Preguntas Frecuentes sobre Kubernetes

## Conceptos Básicos

### ¿Qué es Kubernetes?

Kubernetes (K8s) es una plataforma de código abierto para automatizar el despliegue, escalado y administración de aplicaciones en contenedores. Agrupa los contenedores que conforman una aplicación en unidades lógicas para facilitar su administración y descubrimiento.

### ¿Cuál es la diferencia entre Pod y Contenedor?

Pod es la unidad más pequeña desplegable en Kubernetes. Un Pod puede contener uno o más contenedores. Generalmente se recomienda ejecutar solo un contenedor por Pod. Los contenedores dentro del mismo Pod comparten recursos de red y almacenamiento.

### ¿Cuál es la diferencia entre Deployment y StatefulSet?

Deployment se usa para gestionar aplicaciones sin estado, soportando actualizaciones declarativas, escalado y retrotracciones. StatefulSet se usa para gestionar aplicaciones con estado, proporcionando identidades de red estables y almacenamiento persistente, adecuado para bases de datos y otras aplicaciones que requieren persistencia.

### ¿Qué es un Namespace?

Namespace divide los recursos del clúster en múltiples particiones lógicas, ayudando a aislar recursos entre diferentes equipos o proyectos. Los namespaces predeterminados comunes incluyen default, kube-system y kube-public.

## Instalación y Configuración

### ¿Cuál es la diferencia entre Minikube y Kind?

Minikube crea un clúster Kubernetes completo de un solo nodo, adecuado para aprendizaje y desarrollo. Kind (Kubernetes in Docker) usa contenedores Docker para simular nodos Kubernetes, más ligero, adecuado para pruebas rápidas.

### ¿Cómo elegir un runtime de contenedores?

Los runtimes de contenedores comunes incluyen Docker, containerd y CRI-O. containerd es el runtime de contenedores central de Docker y actualmente es ampliamente recomendado.

### ¿Qué hacer si kubeadm init falla?

Razones comunes incluyen:
- Puertos ya en uso: verificar puertos 6443, 10250, 10255
- Swap no desactivado: ejecutar `swapoff -a`
- Reglas de firewall: asegurar comunicación de red entre nodos
- SELinux/AppArmor: puede establecerse en modo permisivo o desactivarse

### ¿Cómo configurar un clúster de múltiples planos de control?

Usando kubeadm para crear clústeres de alta disponibilidad:
1. Preparar múltiples nodos de servidor
2. Usar kubeadm init para crear el primer nodo del plano de control
3. Generar certificados y copiar a otros nodos
4. Usar kubeadm join para agregar planos de control adicionales

## Red y Almacenamiento

### ¿Qué hacer si los Pods no pueden comunicarse entre sí?

Razones comunes incluyen:
- Plugin de red no instalado correctamente
- Firewall bloqueando comunicación del CIDR de red de Pod
- Errores de configuración del plugin CNI
- Verificar si kube-proxy se está ejecutando correctamente

### ¿Cómo configurar almacenamiento persistente para Pods?

Necesita crear PersistentVolume (PV) y PersistentVolumeClaim (PVC). PV puede ser almacenamiento local, almacenamiento de red (NFS, iSCSI) o almacenamiento en nube (AWS EBS, GCE PD, Azure Disk).

### ¿Cuáles son los tipos de Service?

- ClusterIP: IP virtual accesible dentro del clúster
- NodePort: Abre un puerto en cada nodo
- LoadBalancer: Usa el balanceador de carga del proveedor de nube
- ExternalName: Mapea servicio a nombre de dominio externo

## Gestión de Recursos

### ¿Cómo establecer límites de recursos de contenedores?

Agregar campo resources en el spec de Pod o Deployment:
```yaml
resources:
  limits:
    memory: "512Mi"
    cpu: "500m"
  requests:
    memory: "256Mi"
    cpu: "250m"
```

### ¿Qué es ResourceQuota?

ResourceQuota se usa para limitar el total de recursos en un namespace, evitando que un solo equipo o proyecto consuma demasiados recursos.

### ¿Qué es LimitRange?

LimitRange se usa para establecer límites de recursos predeterminados y valores de solicitud para Pods y contenedores en un namespace.

## Despliegue y Operaciones

### ¿Cómo lograr actualizaciones continuas de aplicaciones?

Al usar Deployment, Kubernetes realiza automáticamente actualizaciones continuas:
```bash
kubectl set image deployment/my-app my-app=my-app:v2
kubectl rollout status deployment/my-app
```

### ¿Cómo retrotracción un despliegue?

```bash
# Retrotracción a versión anterior
kubectl rollout undo deployment/my-app

# Retrotracción a revisión específica
kubectl rollout undo deployment/my-app --to-revision=2
```

### ¿Qué es HPA?

Horizontal Pod Autoscaler (HPA) ajusta automáticamente el número de réplicas de Pod basándose en métricas como CPU y memoria. Requiere Metrics Server para funcionar.

### ¿Cómo configurar verificaciones de salud?

Configurar sondas de preparación y viabilidad:
```yaml
livenessProbe:
  httpGet:
    path: /healthz
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

## Seguridad

### ¿Qué es RBAC?

El Control de Acceso Basado en Roles (RBAC) se usa para gestionar permisos en clústeres Kubernetes. Puede crear Role y ClusterRole, y vincularlos a usuarios o grupos.

### ¿Cómo gestionar información sensible?

Use Secret para almacenar información sensible como contraseñas, claves y certificados. Puede crearse usando la línea de comandos kubectl o definirse a través de archivos YAML.

### ¿Cómo proteger su clúster?

- Habilitar autorización RBAC
- Usar políticas de red para restringir comunicación entre Pods
- Actualizar regularmente versiones de Kubernetes
- Cifrar datos de etcd
- Usar TLS para comunicación
- Restringir permisos de acceso de kubelet

## Solución de Problemas

### ¿Pod permanece en estado Pending?

Posibles razones:
- Recursos insuficientes (CPU o memoria)
- Selector de nodo o taint no coinciden
- PVC no bound correctamente
- Usar kubectl describe pod <pod-name> para detalles

### ¿Pod permanece en estado ImagePullBackOff?

Razones comunes:
- Nombre de imagen incorrecto
- Etiqueta de imagen no existe
- Problemas de autenticación de registro privado
- Problemas de red que impiden descarga de imagen

### ¿Pod permanece en estado CrashLoopBackOff?

- Verificar logs del contenedor: kubectl logs --previous <pod-name>
- Verificar que la aplicación inicie correctamente
- Verificar que la configuración de recursos sea correcta
- Verificar que servicios dependientes estén disponibles

### ¿Cómo depurar problemas de red?

- Verificar que kube-proxy se ejecute correctamente
- Verificar estado del plugin CNI
- Usar kubectl exec para entrar al contenedor y probar conectividad de red
- Ver reglas de iptables
- Usar herramientas de diagnóstico de red (curl, ping, traceroute)

## Optimización de Rendimiento

### ¿Cómo mejorar el rendimiento del clúster?

- Configurar solicitudes y límites de recursos apropiados
- Usar anti-afinidad para distribuir Pods
- Habilitar caché de API para reducir solicitudes
- Usar almacenamiento local para reducir latencia de red
- Optimizar tamaños de imagen para reducir tiempo de descarga

### ¿Cómo manejar clústeres a gran escala?

- Usar Federación para gestionar múltiples clústeres
- Configurar hardware apropiado para etcd
- Habilitar caché de API
- Usar balanceadores de carga para distribuir solicitudes
