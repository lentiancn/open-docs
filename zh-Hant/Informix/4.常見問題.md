# Informix 常見問題解答

## 邏輯日誌滿

**故障現象**：
- 資料庫停止所有操作
- `onstat -l` 顯示所有邏輯日誌處於 U----- 標誌（已使用未備份）

**故障分析**：
- 邏輯日誌未及時備份
- 邏輯日誌空間分配過小
- 邏輯日誌包含活動交易
- 邏輯日誌包含檢查點資訊

**故障處理**：
1. 檢查備份設備是否正常（磁帶滿、磁帶機故障等）
2. 檢查是否有活動交易：`onstat -x`
3. 檢查點之後的日誌即使備份也不能立即使用
4. 動態增加邏輯日誌：
```bash
onspaces -c -d llogdbs -p /home/informix/chunk/llogchk -s 20000 -o 0
onparams -a -d llogdbs -s 2000
```

## 鎖衝突

**故障現象**：
- 出現 -243、-244 錯誤碼
- `Could not position within a table`
- `Could not do a physical-order read to fetch next row`

**故障分析**：
- 其他使用者佔用了鎖資源

**故障處理**：
1. 查看表格鎖佔用情況：
```bash
# 取得表格的 partnum
oncheck -pt database:tablename

# 查看鎖定資訊
onstat -k | grep <partnum_hex>

# 查看具體工作階段
onstat -u | grep <address>
```
2. 解決方案：
   - 使用 `dirty read` 隔離等級
   - 將頁級鎖改為列級鎖
   - 設定鎖等待時間
   - 終止佔用鎖的工作階段：`onmode -z <sid>`

## 長交易

**故障現象**：
- 日誌中出現長交易提示
- 交易處於回滾狀態
- 嚴重時導致整個資料庫執行個體停止

**故障分析**：
- 活動交易佔用邏輯日誌比例達到 `LTXHWM` 閾值
- 比例達到 `LTXEHWM` 時會停止其他工作階段

**故障處理**：
1. 查看日誌定位具體交易
2. 將大交易拆分為小交易
3. 避免長時間無操作的交易
4. 增加邏輯日誌空間

## Chunk I/O 錯誤

**故障現象**：
- 日誌中出現 chunk IO 錯誤
- `onstat -d` 顯示 chunk 狀態為 down

**故障分析**：
- 磁碟設備故障
- chunk 路徑不存在
- 設備權限錯誤

**故障處理**：
1. 檢查儲存設備：
```bash
dd if=/home/informix/940/dsk/data_chk1 of=/dev/null bs=2048k
```
2. 復原 chunk：
```bash
onspaces -s <dbspace> -r
```
3. 如不能復原，聯繫 IBM 技術支援

## 不能插入資料

**故障現象**：
- 報 -131、-136、-271 錯誤
- `ISAM error: no free disk space`
- `ISAM error: no more extents`

**故障分析**：
- dbspace 沒有足夠連續空間
- 達到表格的最大 extent 數量
- 達到表格的記錄數上限

**故障處理**：
1. 定位問題表格
2. 增加 chunk 空間
3. 重建表格或進行分片
4. 應急處理：小批量刪除資料，避免大交易

## 連線問題

**故障現象**：
- 用戶端無法連線到 Informix 伺服器
- 報 -25596 或 -951 錯誤

**故障處理**：
1. 檢查 Informix 服務是否執行：`onstat`
2. 檢查 sqlhosts 設定是否正確
3. 檢查防火牆是否開放對應連接埠
4. 驗證環境變數 `INFORMIXSERVER` 和 `INFORMIXDIR` 是否正確設定

## 效能問題

**故障現象**：
- 查詢回應變慢
- CPU 或記憶體使用率過高

**故障處理**：
1. 使用 `onstat -g <option>` 查看各類統計資訊
2. 檢查索引是否缺失或失效
3. 分析查詢計劃：`SET EXPLAIN ON`
4. 調整 `onconfig` 中的緩衝池大小
5. 定期執行 `UPDATE STATISTICS` 更新統計資訊

## 資料備份與復原

**問題**：如何備份和復原 Informix 資料庫？

**解決方案**：
- 使用 `ontape` 進行邏輯日誌備份和全量備份
- 使用 `onbar` 進行備份（需要設定儲存管理軟體）
- 定期執行 `ontape -s -L 0` 進行全量備份
- 使用 `ontape -r` 進行復原

## 表格空間管理

**問題**：如何查看和擴展表格空間？

**解決方案**：
- 查看表格空間：`onstat -d`
- 建立新的 chunk：`onspaces -c -d <dbspace> -p <path> -s <size>`
- 新增 chunk 到現有 dbspace：`onspaces -a <dbspace> -p <path> -s <size>`
