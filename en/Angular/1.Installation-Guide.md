## 1. Prerequisites

### 1.1 Node.js Requirements

Angular requires Node.js. Different versions of Angular have different Node.js requirements:

| Angular Version | Minimum Node.js | Recommended Node.js |
|-----------------|-----------------|---------------------|
| Angular 17+ | Node.js 18.13.0+ | Node.js 20 LTS |
| Angular 16 | Node.js 16.14.0+ | Node.js 18 LTS |
| Angular 15 | Node.js 14.20.0+ | Node.js 16 LTS |
| Angular 14 | Node.js 14.20.0+ | Node.js 16 LTS |

> ðŸ’¡ **Tip**: It is recommended to use Node.js LTS (Long Term Support) for best stability.

### 1.2 npm Requirements

Angular CLI is distributed via npm (Node Package Manager). npm is installed together with Node.js.

- Minimum requirement: npm 6.14.0+
- Recommended: npm 8+ (comes with Node.js 20)

### 1.3 Optional: yarn or pnpm

If you prefer using yarn or pnpm as your package manager:

```bash
# Install yarn (optional)
npm install -g yarn

# Install pnpm (optional)
npm install -g pnpm
```

## 2. Installing on Windows

### 2.1 Method One: Using npm (Recommended)

This is the simplest and most common installation method.

#### Step 1: Install Node.js

1. Visit Node.js official website: https://nodejs.org/
2. Download the **LTS (Long Term Support)** version (Node.js 20 LTS recommended)
3. Run the installer
4. In the installation wizard:
   - Check "Add to PATH" (add to environment variable)
   - Keep other options as default

#### Step 2: Verify Node.js and npm

Open **Command Prompt** (cmd) or **PowerShell** and run:

```cmd
node --version
npm --version
```

If version numbers appear, the installation was successful.

#### Step 3: Install Angular CLI

In Command Prompt or PowerShell, run:

```cmd
npm install -g @angular/cli
```

> âš ï¸ **Note**: The `-g` flag means global installation, so you can use the `ng` command from any directory.

#### Step 4: Verify Angular CLI

```cmd
ng version
```

If successful, it will display the Angular CLI version information.

---

### 2.2 Method Two: Using Administrator Privileges

If you encounter permission errors, run as administrator:

1. Right-click "Command Prompt" or "PowerShell"
2. Select "Run as administrator"
3. Then run the installation command:

```cmd
npm install -g @angular/cli
```

---

### 2.3 Method Three: Using Windows Package Manager (winget)

If you use Windows 10/11, you can use winget:

```cmd
winget install OpenJS.NodeJS.LTS
winget install Angular.AngularCLI
```

---

### 2.4 Method Four: Using Chocolatey

If you have Chocolatey package manager installed:

```cmd
choco install nodejs-lts
choco install angular-cli
```

## 3. Installing on macOS

### 3.1 Method One: Using npm (Recommended)

#### Step 1: Install Node.js

**Method A: Through Homebrew (Recommended)**

```bash
# Install Homebrew (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js LTS
brew install node@20
```

**Method B: Direct Download**

1. Visit https://nodejs.org/
2. Download macOS Installer (.pkg)
3. Run the installer

#### Step 2: Verify Installation

```bash
node --version
npm --version
```

#### Step 3: Install Angular CLI

```bash
sudo npm install -g @angular/cli
```

> ðŸ’¡ **Tip**: macOS may require administrator privileges (sudo).

#### Step 4: Verify Installation

```bash
ng version
```

---

### 3.2 Method Two: Using nvm for Multiple Versions

If you need to use multiple Node.js versions simultaneously, use nvm:

```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js using nvm
nvm install 20

# Then install Angular CLI
npm install -g @angular/cli
```

### 3.3 Method Three: Using yarn

```bash
# Install yarn
brew install yarn

# Install Angular CLI using yarn
yarn global add @angular/cli
```

## 6. Creating Your First Project

After installation, you can create your first Angular project:

```bash

```

This will:
- Create a directory named `my-first-app`
- Install project dependencies
- Configure Angular environment

## 8. Frequently Asked Questions

### Q1: What to do if I encounter permission errors during installation?

**Windows:**
- Run Command Prompt or PowerShell as administrator
- Check permissions of Node.js installation directory

**macOS/Linux:**
- Use `sudo npm install -g @angular/cli`
- Or modify npm global directory permissions

---

### Q2: How to update Angular CLI to the latest version?

```bash
npm install -g @angular/cli@latest
```

---

### Q3: How to check the current Angular version?

Run in project directory:

```bash
ng version
```

Or check in `package.json`:

```json
{
  "dependencies": {
    "@angular/core": "^17.0.0"
  }
}
```

---

### Q4: What to do if "'ng' is not recognized as an internal or external command"?

**Windows:**
1. Check if Node.js is properly installed: `node --version`
2. Check if npm global module path is in PATH
3. Manually add path to system environment variables

**macOS/Linux:**
1. Check npm global module path
2. May need to add npm global bin directory to PATH:
   ```bash
   export PATH="$PATH:$(npm global bin)"
   ```

---

### Q5: What if installation is very slow?

You can use domestic mirror source:

```bash
# Set npm mirror
npm config set registry https://registry.npmmirror.com

# Install Angular CLI
npm install -g @angular/cli
```

---

### Q6: How to use different versions of Angular CLI in different projects?

You can use nvm to manage multiple Node.js versions, or install Angular CLI locally in each project:

```bash
# In project directory
npm install @angular/cli --save-dev
```

Then use:

```bash
npx ng version
```

---

### Q7: What if Node.js version is incompatible?

Use nvm (Node Version Manager) to switch Node.js versions:

```bash
# View available versions
nvm list

# Install specific version
nvm install 18

# Use specific version
nvm use 18
```

---

### Q8: macOS M1/M2 Chip Installation Issues

If using Apple Silicon (M1/M2/M3) chips:

```bash
# Ensure using arm64 version of Node.js
arch -arm64 node --version

# If issues arise, may need to install x64 compatible version
nvm install 20 --arm64
```

---

### Q9: How to view Angular CLI installation path?

```bash
npm list -g @angular/cli
```

---

### Q10: What if enterprise network requires proxy?

```bash
# Set proxy
npm config set proxy http://proxy.company.com:8080
npm config set https-proxy http://proxy.company.com:8080

# Remove proxy
npm config delete proxy
npm config delete https-proxy
```
