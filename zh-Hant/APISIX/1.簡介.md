# 簡介

Apache APISIX 是 Apache 軟體基金會下的頂級項目，由 API7.ai 開發並捐贈。它是一個具有動態、實時、高效能等特點的雲端原生 API 閘道器。

APISIX 作為所有業務的流量入口，提供了豐富的功能特性，包括動態路由、動態上游、動態憑證、A/B 測試、灰度發布、金絲雀發布、藍綠部署、限速、防攻擊、收集指標、監控警報、可觀測性和服務治理等。

## 核心特性

### 高效能

APISIX 基於 Nginx 構建，繼承了 Nginx 出色的效能表現。通過插件機制，業務邏輯與核心功能解耦，使得擴展更加靈活，同時保持了高效的處理能力。

### 完全動態

與傳統 Nginx 需要重新載入配置不同，APISIX 支援完全動態的配置管理。所有路由、上游、插件等都 可以通過管理 API 實時更新，無需重啟服務即可生效。

### 雲端原生設計

APISIX 從設計之初就考慮了雲端原生場景，支援多種部署方式，包括 Docker、Kubernetes、Helm Chart 等。同時提供了 Ingress Controller，可以直接作為 Kubernetes 的入口閘道器使用。

### 插件生態

APISIX 擁有豐富的插件系統，涵蓋了身份認證、限流限速、請求改寫、響應處理、監控日誌等各個方面。用戶可以通過插件機制輕鬆擴展閘道器功能。

### 多協定支援

除了傳統的 HTTP/HTTPS 協定，APISIX 還支援 gRPC、Dubbo、WebSocket、MQTT 等多種協定，能夠滿足微服務架構中不同場景的需求。

## 架構概述

APISIX 的架構分為資料面和控制面兩部分：

**資料面**負責處理實際的請求轉發，基於 Nginx 修改而來的 APISIX Core 實現了高效的請求處理管道。

**控制面**通過 RESTful API 管理所有配置，支援多種配置持久化方式，包括 etcd、Consul 等。APISIX 還提供了 Dashboard，可通過 Web 介面進行可視化配置管理。

## 適用場景

- 微服務 API 統一入口
- Kubernetes Ingress Controller
- API 流量管理與監控
- 多租戶 API 管理
- 服務網格邊車代理
- AI 應用閘道器（LLM 和 AI 工作負載）

作為 Apache 軟體基金會的頂級項目，APISIX 擁有活躍的社區和持續的版本更新，為企業級應用提供了可靠的選擇。
