# APISIX FAQ

자주 묻는 질문과 답변.

## 설치

### Q: Docker 컨테이너가 시작되지 않음?

**해결책:**
1. Docker가 설치되고 실행 중인지 확인
```bash
docker --version
docker ps
```
2. 포트가 사용 중인지 확인
```bash
netstat -tlnp | grep 9080
```
3. 컨테이너 로그 확인
```bash
docker logs apisix
docker logs etcd
```

### Q: etcd 연결 오류?

**해결책:**
1. etcd가 실행 중인지 확인
```bash
docker ps | grep etcd
```
2. etcd가 올바른 포트에서 수신하는지 확인

### Q: 포트가 사용 중?

**해결책:**
1. 포트를 사용하는 프로세스 식별
```bash
lsof -i:9080
```
2. 프로세스 종료 또는 APISIX 포트 변경
```yaml
apisix:
  node_listen: 9081
```

## 사용

### Q: 라우트가 작동하지 않음?

**해결책:**
1. 라우트 구성 형식 확인
2. 라우트가 올바르게 생성되었는지 확인
3. 오류 로그 확인
```bash
tail -f logs/error.log
```

### Q: 플러그인이 작동하지 않음?

**해결책:**
1. 플러그인이 활성화되어 있는지 확인
2. 플러그인 구성 형식 확인

### Q: 업스트림 서비스에 접근할 수 없음?

**해결책:**
1. 업스트림 서비스가 실행 중인지 확인
2. 네트워크 연결 확인
3. 방화벽 규칙 확인

## 구성

### Q: SSL/TLS 인증서 구성 방법?

**해결책:**
1. SSL 인증서 업로드
```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/ssls/1 \
  -H "X-API-KEY: 123456" \
  -d '{"cert":"인증서내용","key":"개인키","sni":"example.com"}'
```

### Q: CORS 구성 방법?

**해결책:**
cors 플러그인 사용:

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{"uri":"/api/*","plugins":{"cors":{}}}'
```
