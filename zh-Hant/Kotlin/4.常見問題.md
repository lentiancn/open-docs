# Kotlin 常見問題

## 基礎問題

### 什麼是 Kotlin？

Kotlin 是一種現代的靜態類型程式語言，由 JetBrains 公司開發並維護。專案始於 2010 年，並於 2016 年 2 月發布 1.0 正式版。Kotlin 可以執行在 JVM、Android、JavaScript、WebAssembly 和原生平台上。它是一種開源語言，採用 Apache 2.0 許可證。

### Kotlin 是免費的嗎？

是的，Kotlin 完全免費。過去免費、現在免費、未來也會繼續免費。Kotlin 採用 Apache 2.0 許可證開源，原始碼托管在 GitHub 上。

### Kotlin 是面向物件還是函數式編程語言？

Kotlin 同時支援面向物件編程和函數式編程風格。您可以根據需要選擇使用其中一種風格，或者將兩者結合使用。Kotlin 對函數式編程提供了原生支援，包括高階函數、函數類型、Lambda 表達式等特性。

### Kotlin 與 Java 有什麼區別？

相比 Java，Kotlin 更加簡潔。粗略估計，相同功能的程式碼量可以減少約 40%。Kotlin 也更加類型安全，例如支援可空類型，可以減少空指標異常的發生。此外，Kotlin 還提供了智慧類型轉換、高階函數、擴展函數等特性，讓程式碼更加富有表現力，便於建立領域特定語言（DSL）。

### Kotlin 與 Java 相容嗎？

完全相容。Kotlin 與 Java 可以無縫互操作，您可以輕鬆地在 Kotlin 程式碼中呼叫 Java 程式碼，反之亦然。這種相容性使得遷移現有 Java 專案到 Kotlin 變得非常簡單。IDE 還內建了 Java 到 Kotlin 的自動轉換工具。

## 安裝與配置問題

### 如何安裝 Kotlin？

Kotlin 包含在 IntelliJ IDEA 和 Android Studio 每個版本中。只要下載並安裝這些 IDE 之一即可開始使用 Kotlin。此外，您也可以下載獨立的 Kotlin 命令列編譯器，或者透過 SDKMAN!、Homebrew、Snip 等套件管理員安裝。

### 哪些 IDE 支援 Kotlin？

IntelliJ IDEA 和 Android Studio 對 Kotlin 提供了開箱即用的完整支援，由 JetBrains 官方開發維護。其他 IDE 和編輯器只有社區開發的插件支援。您也可以使用 Kotlin Playground 在瀏覽器中編寫和執行 Kotlin 程式碼。

### Kotlin 支援哪些建置工具？

在 JVM 方面，主要的建置工具是 Gradle 和 Maven。此外，還有一些針對用戶端 JavaScript 的建置工具。

### Kotlin 編譯後會生成什麼？

當目標平台是 JVM 時，Kotlin 編譯生成與 Java 相容的位元組碼。當目標平台是 JavaScript 時，Kotlin 轉譯為 ES5.1 程式碼，相容 AMD 和 CommonJS 等模組系統。當目標平台是原生平台時，Kotlin 生成特定於平台的程式碼（透過 LLVM）。

### Kotlin 支援哪些版本的 JVM？

Kotlin 允許選擇目標 JVM 版本。預設情況下，Kotlin/JVM 編譯器生成 Java 8 相容的位元組碼。如果想利用新版 Java 的優化特性，可以顯式指定目標 Java 版本從 9 到 25。需要注意的是，生成的位元組碼可能無法在較低版本上執行。從 Kotlin 1.5 開始，編譯器不再支援生成低於 Java 8 相容性的位元組碼。

## 應用場景問題

### Kotlin 能用於哪些開發？

Kotlin 可以用於任何類型的開發，包括伺服器端開發、用戶端 Web 開發、Android 開發或多平台函式庫開發。隨著 Kotlin/Native 的發展，還支援嵌入式系統、macOS 和 iOS 等平台。

### Kotlin 能用於 Android 開發嗎？

可以。Kotlin 是 Android 官方推薦的一等語言。已有數百個應用使用 Kotlin 進行 Android 開發，包括 Basecamp、Pinterest 等知名應用。

### Kotlin 能用於伺服器端開發嗎？

可以。Kotlin 100% 相容 JVM，因此可以使用任何現有框架，如 Spring Boot、Vert.x 或 JSF。此外，還有專門為 Kotlin 編寫的框架，如 Ktor 等。

### Kotlin 能用於 Web 開發嗎？

可以。對於後端 Web 開發，Kotlin 與 Ktor 和 Spring 等框架配合良好，可以高效建構服務端應用。此外，還可以使用 Kotlin/Wasm 進行用戶端 Web 開發。

### Kotlin 能用於桌面開發嗎？

可以使用任何 Java UI 框架，如 JavaFX、Swing 等。此外，還有一些 Kotlin 專用的框架，如 TornadoFX。

### Kotlin 能用於原生開發嗎？

可以。Kotlin/Native 是 Kotlin 的一部分，可以將 Kotlin 編譯為無需 VM 即可執行的原生程式碼。可以在流行的桌面和行動平台上嘗試使用，甚至可以用於一些物聯網設備。

## 學習問題

### Kotlin 難學嗎？

Kotlin 借鑒了 Java、C#、JavaScript、Scala 和 Groovy 等現有語言。我們致力於讓 Kotlin 易於學習，人們可以在幾天內輕鬆掌握 Kotlin 的閱讀和編寫。學習和使用更多高級特性可能需要更長的時間，但總體上 Kotlin 並不複雜。

### 如何學習 Kotlin？

最好的起點是 Kotlin 官網。可以安裝官方 IDE 或線上嘗試 Kotlin。官方還提供了互動式教學 Tour，透過理論學習與練習相結合的方式幫助您從初學者晉升到中級水平。

### 有哪些 Kotlin 書籍推薦？

Kotlin 官方書籍頁面上列出了一些推薦的入門書籍。您可以在網路上找到更多 Kotlin 書籍的社區維護列表。

### 有哪些線上 Kotlin 課程？

JetBrains Academy 的 Kotlin Core 軌道可以在建立實際應用的同時學習所有 Kotlin 基礎知識。還有 Pluralsight、O'Reilly、Udemy 等平台提供的 Kotlin 課程。官方 YouTube 頻道也提供了大量教學和內容。

### Kotlin 社區活躍嗎？

非常活躍。Kotlin 開發者活躍在 Kotlin 論壇、Stack Overflow，尤其是 Kotlin Slack 上（截至 2020 年 4 月已有近 30000 名成員）。

### 有哪些 Kotlin 活動？

有很多專注於 Kotlin 的使用者群組和聚會。您可以在官網上找到列表。此外，全球還有社區組織的 Kotlin Nights 活動。

### 有 Kotlin 會議嗎？

有。KotlinConf 是由 JetBrains 主辦的年度會議，匯聚來自世界各地的開發者分享 Kotlin 知識和經驗。此外，全球各種技術會議也會涵蓋 Kotlin 內容。

## 技術問題

### Kotlin 支援多執行緒嗎？

Kotlin 協程提供了強大的並發支援，可以在單個執行緒中執行多個協程。協程比執行緒更輕量級，建立數萬個協程而不會耗盡系統資源。Kotlin 還提供了 Flow 用於處理非同步資料流。

### Kotlin 有垃圾回收嗎？

Kotlin 執行在 JVM 上時，受 JVM 垃圾回收器管理。Kotlin/Native 使用不同的記憶體管理策略。

### Kotlin 的效能如何？

Kotlin 編譯後的位元組碼與 Java 類似，效能相當。Kotlin 的特性和語法糖通常不會引入顯著的效能開銷。在某些場景下，Kotlin 的內聯函數等特性甚至可以帶來效能優勢。

### 如何調試 Kotlin 程式碼？

在 IntelliJ IDEA 和 Android Studio 中，可以使用與 Java 相同的調試工具。支援設定中斷點、單步執行、檢視變數值等標準調試功能。

### Kotlin 支援泛型嗎？

支援。Kotlin 的泛型系統與 Java 類似，但增加了類型投影和宣告處型變等高級特性，使泛型更加安全和靈活。

### Kotlin 有標準庫嗎？

是的，Kotlin 標準庫提供了豐富的功能，包括集合操作、字串處理、I/O 操作、數學函數等。標準庫的設計注重 API 的一致性和易用性。

## 專案遷移問題

### 如何將 Java 程式碼轉換為 Kotlin？

IntelliJ IDEA 和 Android Studio 內建了 Java 到 Kotlin 的轉換工具。選取要轉換的 Java 檔案，選擇 Code -> Convert Java File to Kotlin File 即可自動轉換。

### 可以在同一個專案中使用 Java 和 Kotlin 嗎？

可以。Kotlin 和 Java 完全相容，可以在同一個專案中混合使用。通常的遷移策略是逐步將關鍵類別轉換為 Kotlin，而非一次性遷移整個專案。

### 遷移到 Kotlin 需要多長時間？

這取決於專案規模。對於小型專案，可能只需要幾個小時。對於大型專案，通常採用逐步遷移策略，可以在幾週到幾個月內完成，同時保持專案的正常開發進度。

### Kotlin 程式碼有什麼最佳實踐？

遵循 Kotlin 慣用寫法很重要。使用 Kotlin 官方提供的編碼規範，優先使用 val 而非 var，充分利用 Kotlin 的語言特性如 data 類別、密封類別、擴展函數等。Kotlin 官方提供了詳細的編碼風格指南。

## 生態與工具問題

### Kotlin 有哪些常用函式庫？

Kotlin 生態非常豐富。常用函式庫包括：Kotlin Coroutines（非同步編程）、Koin（依賴注入）、Ktor（Web 框架）、Retrofit（網路請求）、Room（資料庫）、Exposed（SQL 框架）等。

### 如何發布 Kotlin 函式庫？

可以透過 Gradle 或 Maven 將函式庫發布到 Maven Central 或其他倉庫。Kotlin 函式庫通常以 JVM 位元組碼形式發布，也可以發布 Kotlin/Wasm 或 Kotlin/Native 版本。

### Kotlin 的測試框架有哪些？

Kotlin 可以使用任何 Java 測試框架，如 JUnit、TestNG 等。Kotlin 還專門提供了 Kotlin Test 函式庫，提供了更 Kotlin 風格的測試 API。對於 Kotlin Multiplatform 專案，可以使用 Kotlin Common Test。

## 未來與版本問題

### Kotlin 的未來發展方向是什麼？

Kotlin 團隊持續改進語言效能和開發體驗。Kotlin Multiplatform 是重點發展方向，旨在讓跨平台開發更加無縫。Kotlin/Wasm 也在快速發展，為瀏覽器端提供更好的支援。此外，Kotlin 還在 AI 整合方面進行探索。

### Kotlin 會保持向後相容嗎？

Kotlin 非常重視向後相容性。Kotlin 1.0 時代編寫的程式碼大多可以在 Kotlin 2.x 版本中正常執行。Kotlin 團隊會謹慎處理破壞性變更，並在必要時提供遷移工具。

### 如何取得 Kotlin 的最新資訊？

可以訂閱 Kotlin YouTube 頻道，關注 Kotlin Twitter 或 Bluesky。Kotlin 部落格也會定期發布新版本資訊和教學。
