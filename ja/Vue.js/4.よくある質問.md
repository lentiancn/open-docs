# Vue.js よくある質問

## Vue 2 と Vue 3 の違い

Vue 3 は現在の最新メジャーバージョンで、Vue 2 にはない新機能（Teleport、Suspense、複数ルート要素）と非互換の変更を含んでいます。

**主な利点：**
- より小さいバンドルサイズ
- より良いパフォーマンス
- より良いスケーラビリティ
- より良い TypeScript / IDE サポート

**推奨事項：**
- **新規プロジェクトは Vue 3 を使用してください**
- **IE11 サポートが必要な場合のみ Vue 2 を検討してください**（Vue 3 は IE11 をサポートしていません）

**移行情報：**
- コンポジション API は Vue 2.7 で利用可能になりました
- 詳細な移行ガイド：[Vue 3 移行ガイド](https://v3-migration.vuejs.org/)

## Vue 2 のサポート状況

- Vue 2.7（2022年7月リリース）が最終マイナーリリース
- **Vue 2 は 2023年12月31日にサポート終了**
- コンプライアンス要件があるチーム向けに延長サポートあり：[Vue 2 拡張 LTS](https://v2.vuejs.org/lts/)

## ブラウザサポート

- **Vue 3.x**: ネイティブ ES2016 サポートのあるブラウザのみ（IE11 を除外）
- **Vue 2.x**: IE11 およびより広範なブラウザサポート

## パフォーマンス

- Vue 3 は最もパフォーマンスの高い主流フロントエンドフレームワークの一つ
- [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html) で React や Angular を上回る
- 公式ドキュメントサイトは Moto G4 + 4x CPU 制限 + 低速 4G ネットワークでパフォーマンススコア 100
- ほとんどの Web アプリケーションを手動最適化なしで処理可能

## バンドルサイズ

- **最小アプリ**（ビルドツール + tree-shaking）: 約 16kb（brotli 圧縮）
- **フル機能アプリ**: 約 27kb
- **ビルドツールなし**（コンパイラ含む）: 約 41kb
- **プログレッシブエンハンスメント**: [petite-vue](https://github.com/vuejs/petite-vue)（約 6kb）を検討

## 大規模アプリケーション

Vue は大規模アプリケーションにも完全に対応可能です：

- **シングルファイルコンポーネント**: モジュール化された開発モデルを提供
- **コンポジション API**: TypeScript 統合により複雑なロジックを整理
- **包括的なツールサポート**: アプリケーション成長時のスムーズな開発体験
- **低い学習曲線**: 新規開発者のオンボーディングコストを削減

## オープンソースライセンス

## API スタイルの選択

### オプション API vs コンポジション API

**オプション API が適している場合：**
- 初心者または OOP 背景の開発者
- ビルドツール不要のシンプルなシナリオ
- プログレッシブエンハンスメントアプリケーション

**コンポジション API が適している場合：**
- Vue での完全なアプリケーション開発
- より良いロジックの整理と再利用
- TypeScript プロジェクト

## JavaScript vs TypeScript

- Vue は TypeScript で実装され、一流の TS サポートを提供
- **TypeScript の使用を強制しません**
- TypeScript 対応の API はすべてのユーザーに恩恵（より良い IDE サポート）
- 選択はチームの背景とプロジェクト規模に依存

## Vue と Web Components の比較

- Vue は Web Components より前に作成されました
- Web Components は低レベル（カスタム要素中心）
- Vue は高レベルの懸念事項に対応（レンダリング、状態管理、ルーティング、SSR）
- Vue はネイティブカスタム要素の消費とエクスポートを完全にサポート
- 詳細な比較：[Vue と Web Components ガイド](/guide/extras/web-components)
