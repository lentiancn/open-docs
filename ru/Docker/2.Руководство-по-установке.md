# Руководство по установке Docker

Это руководство содержит подробные инструкции по установке Docker на различных операционных системах.

## Обзор способов установки

Вы можете установить Docker несколькими способами:

1. **Docker Desktop**: включает Docker Engine в приложении для рабочего стола, рекомендуется для большинства пользователей
2. **Только Docker Engine**: установите только основные компоненты Docker, подходит для Linux-серверов
3. **Docker Engine через apt-репозиторий**: установите из официального apt-репозитория Docker
4. **Ручная установка**: загрузите пакеты deb и установите вручную

## Установка Docker Desktop

Docker Desktop — самый простой и быстрый способ начать работу. Он включает полную среду разработки Docker: Docker Engine, Docker CLI, Docker Compose, Docker Build и Kubernetes.

### Системные требования

**macOS:**
- macOS 10.15 (Catalina) или новее
- 4 ГБ оперативной памяти или больше
- Поддержка виртуализации (VT-x или AMD-V)

**Windows:**
- Windows 10 64-бит Professional, Enterprise или Education
- 4 ГБ оперативной памяти или больше
- WSL 2 (рекомендуется) или Hyper-V

**Linux:**
- Ubuntu 22.04 или новее
- Debian 11 или новее
- Fedora 39 или новее

### Примечания по установке на Windows

Перед установкой Docker Desktop на Windows учитывайте следующие ограничения:

- Если вы используете ufw или firewalld для управления настройками брандмауэра, имейте в виду, что при открытии портов контейнеров с помощью Docker эти порты обходят правила брандмауэра
- Docker совместим только с iptables-nft и iptables-legacy. Правила брандмауэра, созданные с помощью nft, не поддерживаются на системах с установленным Docker

### Установка на macOS

**Способ один: Загрузка установщика**

1. Посетите [страницу загрузки Docker Desktop](https://www.docker.com/products/docker-desktop/)
2. Скачайте установщик для Mac (Intel или Apple Silicon)
3. Дважды щёлкните файл .dmg
4. Перетащите Docker.app в папку Applications
5. Запустите Docker Desktop

**Способ два: Homebrew**

```bash
brew install --cask docker
```

### Установка на Linux (на примере Ubuntu)

**Способ один: Установка через apt-репозиторий**

1. Настройте apt-репозиторий Docker:

```bash
# Добавьте официальный GPG-ключ Docker
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Добавьте репозиторий в источники Apt
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
```

2. Установите пакеты Docker:

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

3. Проверьте установку:

```bash
sudo docker run hello-world
```

**Способ два: Установка через скрипт**

Docker предоставляет скрипты для неинтерактивной установки:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

**Примечание**: Скрипты установки рекомендуются только для тестовых и сред разработки, не для production-использования.

### Удаление старых версий

Перед установкой Docker удалите конфликтующие пакеты:

```bash
sudo apt remove docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc
```

## Установка Docker Engine (без Desktop)

Если вам не нужен Docker Desktop, вы можете установить только Docker Engine.

### Установка на Ubuntu

**Предварительные требования**

- Ubuntu 24.04 (Noble) или новее
- Поддерживаемые архитектуры: x86_64, armhf, arm64, s390x, ppc64le

**Шаги установки**

1. Удалите старые версии:

```bash
sudo apt remove docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc
```

2. Настройте apt-репозиторий (см. выше)

3. Установите Docker:

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

4. Запустите службу Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

5. Проверьте установку:

```bash
sudo docker run hello-world
```

### Установка на CentOS/RHEL

1. Установите зависимости:

```bash
sudo yum install -y yum-utils
```

2. Добавьте репозиторий Docker:

```bash
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
```

3. Установите Docker:

```bash
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

4. Запустите Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Установка на Fedora

1. Установите зависимости:

```bash
sudo dnf -y install dnf-plugins-core
```

2. Добавьте репозиторий Docker:

```bash
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
```

3. Установите Docker:

```bash
sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### Установка на Debian

1. Установите зависимости:

```bash
sudo apt update
sudo apt install ca-certificates curl gnupg
```

2. Добавьте GPG-ключ и репозиторий:

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

3. Установите Docker:

```bash
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## Запуск Docker от имени непривилегированного пользователя

По умолчанию команды Docker требуют sudo. Чтобы разрешить обычным пользователям запускать Docker:

```bash
sudo usermod -aG docker $USER
```

Затем выйдите из системы и войдите снова или выполните:

```bash
newgrp docker
```

## Обновление Docker

### Ubuntu/Debian

```bash
sudo apt update
sudo apt upgrade docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### CentOS/RHEL/Fedora

```bash
sudo yum update docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
# или
sudo dnf update docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## Удаление Docker

### Удаление Docker Desktop

- **macOS**: Перетащите Docker.app из папки Applications
- **Windows**: Удалите через Settings → Apps

### Удаление Docker Engine

**Ubuntu/Debian:**

```bash
sudo apt purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
sudo rm /etc/apt/sources.list.d/docker.list
sudo rm /etc/apt/keyrings/docker.asc
```

**CentOS/RHEL/Fedora:**

```bash
sudo yum remove docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
```

## Информация о лицензировании

Docker Desktop требует платной подписки в следующих случаях:
- Более 250 сотрудников
- Годовой доход более 10 миллионов долларов США

Docker Engine имеет открытый исходный код и использует лицензию Apache 2.0.

## Проверка установки

После установки выполните следующие команды для проверки:

```bash
docker --version
docker-compose --version
docker run hello-world
```

Если всё работает нормально, установка Docker прошла успешно.
