# MongoDB Foire aux questions

## Concepts de base

### Qu'est-ce que MongoDB ?

MongoDB est une base de données orientée documents conçue pour aider les développeurs à construire plus rapidement des applications modernes.

### MongoDB est-il une base de données relationnelle ?

Non. MongoDB est une base de données NoSQL orientée documents qui utilise le format BSON.

### Qu'est-ce qu'un document dans MongoDB ?

Un document est l'unité de données de base dans MongoDB, consisting de paires champ-valeur.

### Quelle est la différence entre MongoDB et MySQL ?

| Fonctionnalité | MongoDB | MySQL |
|----------------|---------|-------|
| Modèle de données | Document | Relationnel |
| Schéma | Flexible | Fixe |
| Mise à l'échelle | Horizontale | Verticale |
| Langage de requête | MongoDB Query Language | SQL |

### Qu'est-ce que BSON ?

BSON signifie Binary JSON, la représentation binaire des documents JSON.

### Quelle est la taille maximale d'un document MongoDB ?

16 Mo. Pour les fichiers plus volumineux, MongoDB fournit l'API GridFS.

## Problèmes d'installation

### Quelle quantité de mémoire MongoDB nécessite-t-il ?

MongoDB utilise la mémoire disponible comme cache. 4 Go minimum sont recommandés pour les environnements de développement.

### MongoDB prend-il en charge les systèmes 32 bits ?

Non. MongoDB nécessite un système d'exploitation 64 bits.

## Questions d'utilisation

### Comment sauvegarder les données MongoDB ?

```bash
mongodump --db=mabasededonnees --out=/sauvegarde
```

### Comment MongoDB assure-t-il la haute disponibilité ?

Via les Replica Sets.

### Qu'est-ce que le sharding ?

Le sharding est la technique de MongoDB pour la mise à l'échelle horizontale.

### MongoDB prend-il en charge les transactions ?

Oui. MongoDB 4.0+ prend en charge les transactions multi-documents.

### Comment optimiser les performances des requêtes ?

1. Créer des index appropriés
2. Utiliser des projections
3. Éviter les scans de collections complets
4. Utiliser les pipelines d'agrégation

### Comment sécuriser les données dans MongoDB ?

1. Activer l'authentification
2. Utiliser le contrôle d'accès basé sur les rôles (RBAC)
3. Activer TLS/SSL
4. Utiliser le chiffrement de stockage
5. Effectuer des sauvegardes régulières

### Quelle est la différence entre les éditions Communauté et Entreprise ?

L'édition Entreprise comprend :
- Authentification LDAP
- Authentification Kerberos
- Audit
- Moteur de stockage chiffré
- Support technique

### Pour quels types de données MongoDB est-il adapté ?

- Systèmes de gestion de contenu
- Données IoT
- Analyses en temps réel
- Applications mobiles
- Catalogues de commerce électronique

## Développement

### Quels langages de programmation MongoDB prend-il en charge ?

MongoDB fournit des pilotes pour presque tous les langages principaux :
- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP

### Comment utiliser MongoDB dans les applications ?

1. Installer le pilote pour votre langage
2. Établir une connexion
3. Exécuter des opérations CRUD

Exemple en Node.js :

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('maDB');
  const collection = database.collection('utilisateurs');
  
  await collection.insertOne({ nom: 'Jean', age: 30 });
  const utilisateur = await collection.findOne({ nom: 'Jean' });
  
  await client.close();
}
```

### Qu'est-ce que MongoDB Atlas ?

MongoDB Atlas est un service de base de données cloud fourni par MongoDB.
