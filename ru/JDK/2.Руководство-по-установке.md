# Руководство по установке JDK

## Установка JDK в Windows

### Системные требования

Перед установкой JDK убедитесь, что ваш компьютер соответствует следующим минимальным требованиям: Процессор, рекомендуется Intel Core i3 или выше; Оперативная память, минимум 4 ГБ рекомендуется, 8 ГБ или более предпочтительно; Место на диске, минимум 2 ГБ свободного места для установки JDK, дополнительное место для проектов разработки; Операционная система, Windows 10 или Windows 11 64-бит.

Хотя Windows 7 и 8.1 могут работать, Oracle больше не поддерживает официально эти операционные системы. Запуск на более старых системах может вызвать проблемы совместимости.

### Загрузка JDK

Сначала посетите официальный сайт Oracle по адресу https://www.oracle.com/java/technologies/downloads/ для загрузки JDK. На странице загрузки вы можете увидеть различные версии JDK: JDK 25 — последняя версия, JDK 21 — последняя версия LTS, JDK 17 — также зрелая версия LTS. Выберите подходящую версию в соответствии с требованиями вашего проекта.

Для систем Windows выберите установщик x64 (.exe) или установщик MSI x64 (.msi). Традиционный мастер установки — .exe, а .msi подходит для пакетного развёртывания в корпоративной среде. Если вам нужно только запускать Java-программы без установки, выберите сжатый архив x64 (.zip), который можно распаковать и использовать напрямую.

Обратите внимание на условия лицензии при загрузке. Oracle JDK бесплатен при определённых версиях и использовании, но имеет ограничения для коммерческого использования и перераспределения. Если вам нужно полностью бесплатное использование, рассмотрите возможность использования OpenJDK или других дистрибутивов на основе OpenJDK, таких как Eclipse Temurin.

### Шаги установки

Найдите загруженный файл установки JDK (обычно jdk-xx_windows-x64_bin.exe) и дважды щёлкните, чтобы запустить. В мастере установки нажмите «Далее» для продолжения. Сначала вы увидите экран выбора целевой папки, путь установки по умолчанию — C:\Program Files\Java\jdk-xx. Если вам нужно изменить место установки, нажмите «Изменить», чтобы выбрать другой каталог. Рекомендуется использовать путь по умолчанию, чтобы избежать проблем с настройкой в дальнейшем.

Затем появится экран выбора компонентов, по умолчанию выбраны все компоненты, включая инструменты разработки, исходный код и открытый JRE. Рекомендуется сохранить полную установку по умолчанию. Нажмите «Далее», чтобы начать установку. Процесс установки может занять несколько минут, пожалуйста, подождите.

После установки вы увидите сообщение. Нажмите «Закрыть» для завершения. На этом этапе JDK установлен в указанный каталог, но вам нужно настроить переменные среды, чтобы использовать команды java, javac и другие из командной строки.

### Настройка переменных среды

Цель настройки переменных среды — позволить системе найти место установки JDK, чтобы вы могли использовать команды java, javac и другие из любого каталога. Щёлкните правой кнопкой мыши «Этот ПК» или «Мой компьютер» и выберите «Свойства». В левом меню нажмите «Дополнительные параметры системы».

В открывшемся окне «Свойства системы» нажмите кнопку «Переменные среды». В разделе «Системные переменные» нажмите «Создать». Введите JAVA_HOME для имени переменной и путь установки JDK (например, C:\Program Files\Java\jdk-21) для значения переменной, затем нажмите ОК.

Найдите переменную Path в системных переменных и нажмите «Изменить». Нажмите «Создать» и введите %JAVA_HOME%\bin. Убедитесь, что эта строка вверху, чтобы недавно установленный JDK был приоритетным. Нажмите ОК, чтобы сохранить все настройки.

Откройте новое окно командной строки (cmd или PowerShell) и введите java -version и javac -version. Если вы видите информацию о версии, настройка прошла успешно.

### Проверка установки

Откройте командную строку (Win+R, введите cmd) и выполните следующие команды для проверки правильности установки JDK:

Выполните java -version, должны отобразиться информация о версии среды выполнения Java. Выполните javac -version, должны отобразиться информация о версии компилятора Java. Выполните echo %JAVA_HOME%, должен отобразиться ранее настроенный путь JAVA_HOME.

Если какая-либо команда показывает ошибку, проверьте правильность настройки переменных среды или правильность установки JDK.

## Установка JDK на macOS

### Использование установщика DMG

Пользователи macOS могут быстро установить JDK с помощью установщика DMG. Посетите официальную страницу загрузки Oracle и загрузите версию JDK для macOS. Для Apple Silicon (процессоры M1/M2/M3) выберите ARM64 DMG Installer; для Mac на базе процессоров Intel выберите x64 DMG Installer.

После загрузки откройте файл DMG, вы увидите файл установщика pkg. Дважды щёлкните, чтобы запустить мастер установки, и следуйте инструкциям для завершения установки. Установщик поместит JDK в каталог /Library/Java/JavaVirtualMachines/.

После установки откройте Терминал и введите java -version для проверки. Если в системе несколько версий JDK, вы можете использовать /usr/libexec/java_home -V, чтобы увидеть все установленные версии.

### Установка с помощью Homebrew

Если Homebrew уже установлен, вы можете использовать его для установки JDK, что более удобно. Откройте Терминал и выполните: brew update. Затем установите JDK 21: brew install openjdk@21.

После установки вам нужно добавить JDK в PATH. Выполните следующую команду для связи JDK с системой: sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.

Затем добавьте переменные среды в ~/.zshrc или ~/.bash_profile: export PATH="/usr/local/opt/openjdk@21/bin:$PATH". Сохраните и выполните source ~/.zshrc для применения настроек.

### Настройка JAVA_HOME

На macOS настройка переменной среды JAVA_HOME не обязательна, так как система автоматически находит установленные JDK. Однако некоторые инструменты (например, Maven, Gradle) могут потребовать ручной настройки. В Терминале отредактируйте файл ~/.zshrc и добавьте: export JAVA_HOME=$(/usr/libexec/java_home). Сохраните и выполните source ~/.zshrc для применения настроек.

## Установка JDK на Linux

### Установка с помощью APT (Debian/Ubuntu)

В системах Debian и Ubuntu вы можете установить OpenJDK с помощью менеджера пакетов APT. Сначала обновите список пакетов: sudo apt update. Затем установите версию разработки OpenJDK 21: sudo apt install openjdk-21-jdk. Если вам нужно только запускать Java-программы без компилятора, установите JRE: sudo apt install openjdk-21-jre.

Во время установки вам будет предложено создать каталог JAVA_HOME. Выберите «Да» для автоматической настройки. После установки используйте java -version и javac -version для проверки установки.

Если вам нужно вручную настроить JAVA_HOME, отредактируйте файл ~/.bashrc или ~/.profile и добавьте: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64. Затем выполните source ~/.bashrc для применения настроек.

### Установка с помощью YUM (Red Hat/CentOS)

В системах, использующих YUM, таких как Red Hat, CentOS и Fedora, вы можете установить JDK напрямую. Выполните следующую команду для установки OpenJDK 21: sudo yum install java-21-openjdk java-21-openjdk-devel. Пакет -devel включает компилятор и другие инструменты разработки.

После установки используйте команду alternatives для определения версии Java по умолчанию: sudo alternatives --config java. Следуйте инструкциям, чтобы выбрать версию Java по умолчанию.

Отредактируйте файл ~/.bashrc и добавьте: export JAVA_HOME=/usr/lib/jvm/java-21-openjdk. Выполните source ~/.bashrc для применения настроек.

### Ручная установка бинарных файлов

Для ситуаций, требующих установки определённой версии JDK, вы можете загрузить и извлечь бинарные файлы вручную. Посетите страницу загрузки Oracle или OpenJDK и загрузите файл .tar.gz нужной версии.

Создайте каталог установки JDK: sudo mkdir -p /opt/jdk. Извлеките загруженный файл: sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /opt/jdk. Перейдите в извлечённый каталог и запомните полный путь.

Отредактируйте файл ~/.bashrc и добавьте: export JAVA_HOME=/opt/jdk/jdk-21 и export PATH=$JAVA_HOME/bin:$PATH. Выполните source ~/.bashrc для применения настроек.

## Управление несколькими версиями JDK

### Управление несколькими версиями в Windows

После установки нескольких версий JDK в Windows вы можете изменить версию по умолчанию через переменные среды. Самый простой способ — изменить порядок каталогов bin JDK в переменной среды PATH, поместив версию, которую вы хотите использовать, первой.

Другой способ — использовать переменную среды JAVA_HOME, указывающую на разные каталоги установки JDK. Этот метод работает для большинства инструментов, но некоторые инструменты могут использовать команду java в PATH вместо JAVA_HOME.

Некоторые сторонние инструменты могут помочь управлять несколькими версиями JDK, например, версия SDKMAN! для Windows или JetBrains Toolbox. Эти инструменты упрощают переключение между версиями.

### Управление несколькими версиями на macOS

На macOS вы можете использовать инструмент jenv для управления несколькими версиями JDK. Сначала установите jenv: brew install jenv. Затем добавьте инициализацию jenv в файл конфигурации оболочки.

Добавьте установленный JDK в jenv: jenv add /Library/Java/JavaVirtualMachines/jdk-21/Contents/Home. Затем вы можете использовать jenv versions, чтобы увидеть все управляемые версии, и jenv global xx, чтобы установить глобальную версию по умолчанию.

Другой способ — использовать /usr/libexec/java_home -v для указания конкретной версии для запуска программ. Например: /usr/libexec/java_home -v 21 --exec java -version.

### Управление несколькими версиями в Linux

В Linux используйте инструмент update-alternatives для управления несколькими версиями JDK. Сначала создайте альтернативные записи для каждой версии JDK: sudo update-alternatives --install /usr/bin/java java /opt/jdk/jdk-21/bin/java 1.

Затем используйте update-alternatives --config java, чтобы выбрать версию по умолчанию. jenv также доступен в Linux, с аналогичными методами установки и настройки как на macOS.

## Распространённые проблемы установки

### Команда java не найдена после установки

Если вы не можете найти команду java после установки JDK, сначала проверьте, правильно ли установлен JDK. Проверьте, правильно ли установлена переменная среды JAVA_HOME и включает ли PATH %JAVA_HOME%\bin (Windows) или $JAVA_HOME/bin (Linux/macOS).

В Windows откройте новое окно командной строки, так как изменения переменных среды могут не применяться немедленно. На Linux/macOS выполните source ~/.bashrc или снова откройте терминал.

Если проблема сохраняется, проверьте правильность каталога установки JDK и наличие файла bin/java в этом каталоге.

### Проблемы с правами доступа

При установке JDK в Linux и macOS вы можете столкнуться с проблемами прав доступа. Использование менеджера пакетов обычно требует права администратора (sudo). При ручной установке убедитесь, что каталог установки имеет соответствующие права на чтение и запись.

Если обычные пользователи не могут использовать команду java, проверьте права каталога JAVA_HOME и каталога bin JDK. Вы можете использовать ls -la /usr/lib/jvm для проверки прав каталога.

### Настройка JAVA_HOME не работает

Если настройка JAVA_HOME не работает, проверьте правильность синтаксиса переменной среды. В Windows значение переменной не должно содержать кавычек. На Linux/macOS убедитесь, что используете правильную команду export.

Также проверьте, не переопределяют ли другие скрипты или файлы конфигурации параметр JAVA_HOME. В Linux системная настройка может быть в /etc/environment или /etc/profile.d/.

### Удаление старых версий

После обновления JDK вам может потребоваться удалить старые версии для экономии места на диске. В Windows найдите старую версию JDK в Панели управления в разделе «Программы и компоненты» и нажмите «Удалить». На macOS удалите старую папку версии в /Library/Java/JavaVirtualMachines/. В Linux используйте менеджер пакетов для удаления: sudo apt remove openjdk-xx-jdk (APT) или sudo yum remove java-xx-openjdk (YUM).
