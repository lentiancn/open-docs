# OpenClaw 자주 묻는 질문

이 페이지는 OpenClaw에 대한 일반적인 질문에 답합니다.

## 기본

### OpenClaw란?

OpenClaw는 채팅 앱(WhatsApp, Telegram, Discord 등)을 AI 코딩 도우미에 연결하는 자체 호스팅 게이트웨이입니다. 데이터를 완전히 제어하면서 어디서든 메시징 앱을 통해 AI 도우미와 통신할 수 있습니다.

### OpenClaw가 무료인가요?

네, OpenClaw는 MIT 라이선스下的开源项目입니다. 그러나 다음을 직접 부담해야 합니다:
- 실행할 서버 또는 컴퓨터
- AI 서비스의 API 키 비용

### AI 서비스 구독이 필요한가요?

OpenClaw는 다양한 모델 제공자를 지원합니다:
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- 기타 OpenAI API 호환 서비스

종량제 API 키 또는 구독 인증을 사용할 수 있습니다.

### OpenClaw는 어떤 플랫폼을 지원합니까?

- **데스크톱**: macOS, Linux, Windows
- **모바일**: iOS, Android(노드로)
- **채팅 앱**: WhatsApp, Telegram, Discord, iMessage, Signal, Slack 등 20개 이상 플랫폼

## 설치

### 설치가 실패하면 어떻게 해야 하나요?

1. Node.js 버전 확인: `node --version` (22 이상 필요)
2. 진단 실행: `openclaw doctor`
3. 로그 확인: `openclaw logs`

### Raspberry Pi에서 실행할 수 있나요?

네, OpenClaw는 Raspberry Pi에서 작동합니다. Raspberry Pi 4 이상과 Raspberry Pi OS 또는 Ubuntu를 권장합니다.

### Mac이 필요한가요?

아니요, OpenClaw는 Node.js를 지원하는 모든 시스템에서 실행됩니다. iMessage 채널만 Apple 제한으로 Mac이 필요합니다.

## 구성

### 구성 파일은 어디에 있나요?

기본 위치: `~/.openclaw/openclaw.json`

### 메시지를 보낼 수 있는 사람을 통제하려면 어떻게 해야 하나요?

`dmPolicy`로 구성:

- `pairing`: 알려지지 않은 발신자는 페어링 코드가 필요함
- `allowlist`: 허용 목록의 사용자만
- `open`: 모든 직접 메시지 허용
- `disabled`: 직접 메시지 비활성화

### 그룹 채팅을 설정하려면 어떻게 해야 하나요?

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## 사용

### 새 채팅 채널을 추가하려면 어떻게 해야 하나요?

```bash
openclaw channels login whatsapp
```

### 여러 채널을 동시에 사용할 수 있나요?

네, Gateway는 여러 채널에 동시에 연결할 수 있습니다.

### AI가 이전 대화를 기억하게 하려면 어떻게 해야 하나요?

OpenClaw는 세션 컨텍스트를 자동으로 저장하는 내장 메모리가 있습니다. 구성을 통해 세션 보존 정책을 조정할 수 있습니다.

### 여러 에이전트를 실행할 수 있나요?

네, 라우팅 규칙으로 여러 에이전트를 구성할 수 있습니다:

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## 보안

### 제 데이터가 안전한가요?

네. OpenClaw는 자체 호스팅이므로 모든 데이터가 기기에 남아 있고 타사 서버로 전송되지 않습니다(AI 제공자 API 제외).

### Gateway를 보호하려면 어떻게 해야 하나요?

1. `allowFrom`을 설정하여 액세스 소스 제한
2. 강력한 비밀번호와 API 키 사용
3. OpenClaw를 최신 상태로 유지
4. 방화벽을 사용하여 포트 액세스 제한

## 문제 해결

### Gateway가 시작되지 않음

1. 포트가 사용 중인지 확인: `openclaw gateway status`
2. 오류 로그 확인: `openclaw logs`
3. 진단 실행: `openclaw doctor`

### AI가 응답하지 않음

1. API 키가 올바르게 구성되어 있는지 확인
2. 모델 제공자 계정에 충분한 잔액이 있는지 확인
3. 로그에서 오류 메시지 찾기

### 메시지 전송 실패

1. 채널이 로그인되어 있는지 확인
2. 대상 사용자 또는 그룹이 존재하는지 확인
3. 자세한 오류 로그 확인
