# Benutzerhandbuch

## Android-Anwendungen installieren und ausführen

Waydroid kann verschiedene Operationen durchführen, die mit dem Befehl `waydroid app -h` gefunden werden können:

```bash
usage: waydroid app [-h] {install,remove,launch,list} ...

optional arguments:
  -h, --help            diese Hilfemeldung anzeigen und beenden

subaction:
  {install,remove,launch,list}
    install             ein einzelnes Paket in den Container pushen und installieren
    remove              ein einzelnes App-Paket aus dem Container entfernen
    launch              eine einzelne Anwendung starten
    list                installierte Anwendungen auflisten
```

### Android-Anwendungen installieren

Sie können Android-Anwendungen mit folgendem Befehl installieren:

```bash
waydroid app install xyz.apk
```

Danach können Sie die Anwendung über das Desktop-Anwendungsmenü starten.

Wenn Sie den Android-Paketnamen der App (`com.xxx.yyy`) kennen, können Sie auch die Kommandozeile verwenden:

```bash
waydroid app launch com.foo.bar
```

### Hinweise

Die APK-Dateien, die Sie manchmal im Internet finden, unterstützen oft nur ARM und funktionieren daher nicht auf x86_64.

Sie möchten vielleicht [F-Droid](https://f-droid.org) installieren, um Anwendungen grafisch zu erhalten. Oder verwenden Sie den Play Store, wenn Sie ein GAPPS-Image installiert haben.

## Häufige Befehle

### Waydroid-Kommandozeilenoptionen

Waydroid bietet umfangreiche Kommandozeilenoptionen zur Steuerung von Containern und Sitzungen:

- `waydroid show-full-ui`: Vollständige Android-Benutzeroberfläche anzeigen
- `waydroid session start`: Waydroid-Sitzung starten
- `waydroid session stop`: Waydroid-Sitzung stoppen
- `waydroid container start`: Waydroid-Container starten
- `waydroid container stop`: Waydroid-Container stoppen
- `waydroid logcat`: Android-Logs anzeigen
- `waydroid shell`: Android-Shell betreten
- `waydroid prop set <property> <value>`: Systemeigenschaften festlegen
- `waydroid bugreport`: Fehlerbericht-Logs generieren

### Systemeigenschaften-Konfiguration

Verschiedene Systemeigenschaften können über den `waydroid prop`-Befehl konfiguriert werden:

- `persist.waydroid.multi_windows`: Multi-Fenster-Modus aktivieren
- Weitere Eigenschaften finden Sie in der offiziellen Dokumentation

## Erweiterte Funktionen

### Verwendung benutzerdefinierter Images

Um benutzerdefinierte Images zu verwenden, folgen Sie diesen Schritten:

1. Laden Sie Images von [SourceForge](https://sourceforge.net/projects/waydroid/files/images/) herunter oder verwenden Sie benutzerdefinierte gebaute `system.img` und `vendor.img`
2. Kopieren Sie die Images in das Verzeichnis `/etc/waydroid-extra/images/`
3. Reinitialisieren Sie Waydroid: `sudo waydroid init -f`

### ADB-Debugging

Waydroid unterstützt ADB-Debugging. Standardmäßig ist der ADB-Sicherheitsmodus aktiviert und verbindet sich nicht automatisch beim Start der Sitzung. Verwenden Sie den folgenden Befehl zum Verbinden:

```bash
waydroid adb connect
```

Geben Sie dann die Autorisierung auf der Android-Seite an.

### Benachrichtigungsweiterleitung

Waydroid unterstützt die Weiterleitung von Android-Benachrichtigungen an die Desktop-Umgebung, was ein aktualisiertes Android-Systemimage auf eine kompatible Version erfordert.
