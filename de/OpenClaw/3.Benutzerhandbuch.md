# OpenClaw Benutzerhandbuch

Dieses Handbuch hilft Ihnen zu verstehen, wie Sie OpenClaw effektiv nutzen.

## Grundlegende Nutzung

### Mit OpenClaw sprechen

Nach der Installation können Sie auf verschiedene Weise mit OpenClaw interagieren:

1. **Control UI**: Besuchen Sie `http://127.0.0.1:18789/` in Ihrem Browser
2. **CLI-Befehl**: `openclaw agent --message "Ihre Nachricht"`
3. **Konfigurierte Kanäle**: Senden Sie Nachrichten über WhatsApp, Telegram, Discord usw.

### Nachrichten senden

Senden Sie Nachrichten über die CLI:

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## Kernkonzepte

### Gateway

Gateway ist die Hauptsteuerungsebene von OpenClaw und verwaltet alle Sitzungen, Kanäle, Tools und Ereignisse. Standardmäßig hört es auf `ws://127.0.0.1:18789`.

### Sitzung

OpenClaw verwendet ein Sitzungsmodell zur Verwaltung von Gesprächen:

- **Hauptsitzung**: Direkter Chat
- **Gruppenisolation**: Gruppennachrichten werden separat behandelt
- **Aktivierungsmodus**: Steuert, wann geantwortet wird
- **Antwortstrategie**: Legt fest, wie geantwortet wird

### Arbeitsbereich

Der Arbeitsbereich ist, wo OpenClaw Konfiguration und Daten speichert:

- Standardort: `~/.openclaw/workspace`
- Enthält: AGENTS.md, SOUL.md, TOOLS.md
- Fähigkeiten-Verzeichnis: `~/.openclaw/workspace/skills/`

## Kanal-Konfiguration

### Unterstützte Kanäle

OpenClaw unterstützt diese Messaging-Plattformen:

- Instant Messaging: WhatsApp, Telegram, Signal, iMessage
- Soziale Plattformen: Discord, Slack
- Enterprise-Tools: Microsoft Teams, Google Chat, Feishu
- Andere: IRC, Matrix, LINE, Mattermost, Nostr usw.

### Konfigurationsbeispiele

**Telegram**:

```json
{
  "channels": {
    "telegram": {
      "botToken": "IHR_BOT_TOKEN"
    }
  }
}
```

**Discord**:

```json
{
  "channels": {
    "discord": {
      "token": "IHR_BOT_TOKEN"
    }
  }
}
```

**WhatsApp**:

```bash
openclaw channels login
```

## Fähigkeiten-System

Fähigkeiten sind die Art und Weise, wie OpenClaw Funktionen erweitert. Sie können:

### Fähigkeiten installieren

Installieren Sie Community-Fähigkeiten von ClawHub oder erstellen Sie Ihre eigenen.

### Fähigkeiten-Struktur

Jede Fähigkeit ist ein Verzeichnis mit einer SKILL.md-Datei:

```
skills/
  └── meine-faehigkeit/
      └── SKILL.md
```

### ClawHub

ClawHub ist ein Fähigkeiten-Register. Wenn aktiviert, kann OpenClaw automatisch nach benötigten Fähigkeiten suchen und diese installieren.

## Automatisierung

### Cron-Jobs

Richten Sie geplante Aufgaben mit Cron ein:

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhooks

Konfigurieren Sie Webhooks, um Automatisierung auszulösen:

```bash
openclaw webhook add --url https://ihr-endpunkt.com/hook
```

### Gmail-Integration

Richten Sie Gmail Pub/Sub ein, um auf neue E-Mails zu hören.

## Erweiterte Funktionen

### Browser-Steuerung

OpenClaw kann einen dedizierten Browser steuern:

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvas-Visualisierung

Canvas ist der visuelle Arbeitsbereich von OpenClaw und unterstützt A2UI-Push/Reset, Auswertung und Snapshots.

### Sprachfunktionen

- **Voice Wake**: Aktivierung über Aktivierungswörter auf macOS/iOS
- **Talk Mode**: Kontinuierliche Sprachkonversation (Android-Unterstützung)

### Geräteknoten

OpenClaw kann verschiedene Geräte verbinden:

- **macOS-Knoten**: Systemausführung, Benachrichtigungen, Canvas, Kamera
- **iOS-Knoten**: Canvas, Sprache, Kamera, Bildschirmaufnahme
- **Android-Knoten**: Verbinden, Chat, Sprache, Canvas, Kamera, Standort usw.

## Sicherheitseinstellungen

### DM-Pairing

Standardmäßig brauchen unbekannte Absender eine Pairing-Verifikation. Ändern Sie mit `dmPolicy: "open"` in den offenen Modus (nicht empfohlen).

### Pairing verwalten:

```bash
# Pairing-Liste anzeigen
openclaw pairing list

# Pairing genehmigen
openclaw pairing approve

# Pairing widerrufen
openclaw pairing revoke
```

### Diagnose ausführen

Führen Sie regelmäßig Diagnosen zur Sicherheit aus:

```bash
openclaw doctor
```

## Befehlsreferenz

| Befehl | Beschreibung |
|--------|-------------|
| `/status` | Sitzungsstatus anzeigen |
| `/new` oder `/reset` | Sitzung zurücksetzen |
| `/compact` | Sitzungskontext komprimieren |
| `/think` | Denkniveau einstellen |
| `/verbose` | Ausführliche Ausgabe umschalten |
| `/usage` | Nutzungsstatistiken anzeigen |
| `/restart` | Gateway neu starten |

## Best Practices

1. **doctor regelmäßig ausführen**: System gesund halten
2. **Bei DMs vorsichtig sein**: Sicherheitseinstellungen prüfen
3. **Starke Modelle verwenden**: Beste Erfahrung erhalten
4. **Regelmäßig sichern**: Wichtige Konfigurationen speichern |

## Fehlerbehebung

Häufige Probleme:

- **Gateway startet nicht**: Port 18789 prüfen
- **Kanalverbindung fehlgeschlagen**: Anmeldedaten prüfen
- **Nachricht senden fehlgeschlagen**: Logs prüfen
