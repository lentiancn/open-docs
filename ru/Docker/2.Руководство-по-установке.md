# Docker Installation Guide

This guide provides detailed instructions for installing Docker on various operating systems.

## Installation Methods Overview

You can install Docker in several ways:

1. **Docker Desktop**: Includes Docker Engine in a desktop application, recommended for most users
2. **Docker Engine Only**: Install only the core Docker components, suitable for Linux servers
3. **Docker Engine via apt Repository**: Install from Docker's official apt repository
4. **Manual Installation**: Download deb packages and install manually

## Docker Desktop Installation

Docker Desktop is the easiest and fastest way to get started. It includes a complete Docker development environment, including Docker Engine, Docker CLI, Docker Compose, Docker Build, and Kubernetes.

### System Requirements

**macOS:**
- macOS 10.15 (Catalina) or later
- 4 GB or more RAM
- Virtualization support (VT-x or AMD-V)

**Windows:**
- Windows 10 64-bit Professional, Enterprise, or Education
- 4 GB or more RAM
- WSL 2 (recommended) or Hyper-V

**Linux:**
- Ubuntu 22.04 or later
- Debian 11 or later
- Fedora 39 or later

### Windows Installation Notes

Before installing Docker Desktop on Windows, note the following limitations:

- If you use ufw or firewalld to manage firewall settings, be aware that when you expose container ports using Docker, these ports bypass your firewall rules
- Docker is only compatible with iptables-nft and iptables-legacy. Firewall rules created with nft are not supported on systems with Docker installed

### macOS Installation

**Method One: Download Installer**

1. Visit the [Docker Desktop download page](https://www.docker.com/products/docker-desktop/)
2. Download the installer for Mac (Intel or Apple Silicon)
3. Double-click the .dmg file
4. Drag Docker.app to the Applications folder
5. Launch Docker Desktop

**Method Two: Homebrew**

```bash
brew install --cask docker
```

### Linux Installation (Ubuntu Example)

**Method One: apt Repository Installation**

1. Set up Docker's apt repository:

```bash
# Add Docker's official GPG key
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
```

2. Install Docker packages:

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

3. Verify installation:

```bash
sudo docker run hello-world
```

**Method Two: Convenience Script Installation**

Docker provides convenience scripts for non-interactive installation:

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

**Note**: Convenience scripts are only recommended for testing and development environments, not for production use.

### Uninstalling Old Versions

Before installing Docker, uninstall any conflicting packages:

```bash
sudo apt remove docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc
```

## Docker Engine Installation (Without Desktop)

If you don't need Docker Desktop, you can install Docker Engine only.

### Ubuntu Installation

**Prerequisites**

- Ubuntu 24.04 (Noble) or later
- Supported architectures: x86_64, armhf, arm64, s390x, ppc64le

**Installation Steps**

1. Uninstall old versions:

```bash
sudo apt remove docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc
```

2. Set up apt repository (see above)

3. Install Docker:

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

4. Start Docker service:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

5. Verify installation:

```bash
sudo docker run hello-world
```

### CentOS/RHEL Installation

1. Install dependencies:

```bash
sudo yum install -y yum-utils
```

2. Add Docker repository:

```bash
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
```

3. Install Docker:

```bash
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

4. Start Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Fedora Installation

1. Install dependencies:

```bash
sudo dnf -y install dnf-plugins-core
```

2. Add Docker repository:

```bash
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
```

3. Install Docker:

```bash
sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### Debian Installation

1. Install dependencies:

```bash
sudo apt update
sudo apt install ca-certificates curl gnupg
```

2. Add GPG key and repository:

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

3. Install Docker:

```bash
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## Running Docker as Non-Root User

By default, Docker commands require sudo. To allow non-privileged users to run Docker:

```bash
sudo usermod -aG docker $USER
```

Then log out and log back in, or run:

```bash
newgrp docker
```

## Upgrading Docker

### Ubuntu/Debian

```bash
sudo apt update
sudo apt upgrade docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### CentOS/RHEL/Fedora

```bash
sudo yum update docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
# or
sudo dnf update docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## Uninstalling Docker

### Uninstall Docker Desktop

- **macOS**: Drag Docker.app out of the Applications folder
- **Windows**: Uninstall through Settings → Apps

### Uninstall Docker Engine

**Ubuntu/Debian:**

```bash
sudo apt purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
sudo rm /etc/apt/sources.list.d/docker.list
sudo rm /etc/apt/keyrings/docker.asc
```

**CentOS/RHEL/Fedora:**

```bash
sudo yum remove docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
```

## Verifying Installation

After installation, run the following to verify:

```bash
docker --version
docker-compose --version
docker run hello-world
```

If everything runs normally, Docker installation is successful.
