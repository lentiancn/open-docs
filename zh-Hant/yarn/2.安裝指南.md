# Yarn 安裝指南

## Yarn v4+ (現代版本) 安裝

### 推薦方式：使用 Corepack

Corepack 是一個中介工具，可以讓您基於每個專案配置套件管理器版本。

```bash
# 1. 安裝 Corepack
npm install -g corepack

# 2. 初始化新專案
yarn init -2
```

### 更新 Yarn

任何時候想要將 Yarn 更新到最新版本，只需執行：

```bash
yarn set version stable
yarn install
```

Yarn 將配置您的專案使用最新的穩定二進位檔。

### 使用 Canary 版本

Yarn 還經常發布 Release Candidate 建置。如果您需要尚未在穩定頻道發布的功能，請使用：

```bash
yarn set version canary
```

這些建置非常穩定，與常規頻道的唯一區別是在實現新的破壞性更改時採用更漸進的遷移方式。

### 從原始碼安裝最新版本

如果您想測試尚未發布甚至未合併的 Yarn 版本，以下命令將直接從儲存庫複製、建置並安裝 Yarn 到您的專案中：

```bash
yarn set version from sources
```

它接受 `--branch` 標誌，您可以用来測試特定的 PR：

```bash
yarn set version from sources --branch 1211
```

> 注意：與穩定和 canary 頻道不同，`yarn set version from sources` 命令無法利用 Corepack，需要將 Yarn 二進位檔儲存在 `.yarn/releases` 資料夾中，並從專案的 `.yarnrc.yml` 檔案中引用它。

## Yarn v1 (Classic) 安裝

### 通過 npm 安裝（推薦）

建議通過 [npm 套件管理器](http://npmjs.org/) 安裝 Yarn，該管理器在您在系統上安裝 [Node.js](https://nodejs.org/) 時會捆綁在一起。

安裝 npm 後，您可以執行以下命令來安裝和升級 Yarn：

```bash
npm install --global yarn
```

### 驗證安裝

通過執行以下命令檢查 Yarn 是否已安裝：

```bash
yarn --version
```

## 版本選擇建議

- **新專案**: 推薦使用 Yarn v4+，享受現代特性和更好的效能
- **現有專案**: 如果已在使用 Yarn v1，可以繼續使用或考慮遷移到現代版本
- **企業環境**: Yarn v1 提供更穩定的長期支援，而 v4+ 提供更多創新功能
