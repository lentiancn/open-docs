# Guía de Instalación

Este documento describe cómo instalar y configurar Angular para que puedas comenzar rápidamente con tu entorno de desarrollo.

## Requisitos del Sistema

Antes de instalar Angular, asegúrate de cumplir con los siguientes requisitos:

- **Node.js**: Se recomienda la versión 18.10.0 o superior
- **npm**: Se incluye con Node.js, versión 6.x o superior
- **Sistema operativo**: Windows, macOS o Linux

## Instalar Angular CLI

Angular CLI es la herramienta de línea de comandos oficial de Angular, utilizada para crear y gestionar proyectos Angular.

### Instalación Global

```bash
npm install -g @angular/cli
```

### Verificar la Instalación

```bash
ng version
```

Si la instalación fue exitosa, verás la información de la versión de Angular CLI.

### Instalar una Versión Específica

Si necesitas instalar una versión particular:

```bash
npm install -g @angular/cli@17
```

## Crear un Nuevo Proyecto

### Usando CLI

```bash
ng new my-app
```

Esto creará un nuevo proyecto llamado my-app e instalará automáticamente las dependencias.

### Crear con Opciones Personalizadas

```bash
# Crear sin instalar dependencias
ng new my-app --skip-install

# Usar SCSS para los estilos
ng new my-app --style=scss

# Habilitar enrutamiento
ng new my-app --routing

# Crear componentes independientes (Angular 14+)
ng new my-app --standalone
```

## Configuración del Entorno de Desarrollo

### Usar Repositorios Mirror de China

 debido a problemas de red, se recomienda configurar un repositorio mirror:

```bash
# Configurar el repositorio de npm
npm config set registry https://registry.npmmirror.com

# O usar la herramienta nrm para gestionar repositorios
npm install -g nrm
nrm use taobao
```

### Instalar Node.js

**Instalar con nvm (recomendado):**

```bash
# Instalar nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Instalar la versión LTS de Node.js
nvm install --lts
nvm use --lts
```

**Para Windows, usar nvm-windows:**

Descarga el instalador desde https://github.com/coreybutler/nvm-windows.

## Iniciar el Servidor de Desarrollo

### Entrar al Directorio del Proyecto

```bash
cd my-app
```

### Iniciar el Servidor de Desarrollo

```bash
ng serve
```

Dirección por defecto: http://localhost:4200

### Recarga en Caliente

Angular CLI soporta recarga en caliente, por lo que el navegador se actualiza automáticamente cuando modificas el código.

## Construir para Producción

### Comando de Construcción

```bash
ng build
```

Los archivos generados se guardarán en el directorio dist.

### Optimizaciones para Producción

```bash
ng build --configuration=production
```

Esto habilita la minificación de código, tree-shaking y otras optimizaciones.

## Comandos Útiles de CLI

| Comando | Descripción |
|---------|-------------|
| ng generate component xxx | Generar un componente |
| ng generate service xxx | Generar un servicio |
| ng generate module xxx | Generar un módulo |
| ng generate directive xxx | Generar una directiva |
| ng generate pipe xxx | Generar un pipe |
| ng generate guard xxx | Generar un guard de ruta |

## Herramientas de Desarrollo

### Extensiones para VS Code

- **Angular Language Service**: Proporciona verificación de tipos en plantillas y autocompletado
- **Angular Snippets**: Proporciona fragmentos de código comunes
- **Prettier**: Formateo de código

### Depuración

Angular ofrece la extensión Angular DevTools para depuración y análisis de rendimiento en el navegador.

## Desinstalar Angular CLI

Si necesitas desinstalar:

```bash
npm uninstall -g @angular/cli
```
