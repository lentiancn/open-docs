# JMeter 자주 묻는 질문

이 섹션은 Apache JMeter 사용 중 발생하는 일반적인 질문과 솔루션을 모은 것입니다.

## 설치 문제

### Q1: JMeter 실행 시 "java"가 내부 또는 외부 명령으로 인식되지 않습니다

**A**: Java가 제대로 설치되고 구성되어 있는지 확인합니다. 명령줄에서 확인:
```bash
java -version
```
설치되지 않았으면 Java JDK 8 이상을 다운로드하여 설치합니다. Windows의 경우에도 JAVA_HOME 환경 변수를 설정하세요.

### Q2: JMeter 인터페이스가 깨진 문자를 표시합니다

**A**: 글꼴 인코딩 문제로 인한 것입니다. 해결 방법:
1. bin/jmeter.properties 파일 열기
2. 다음을 추가 또는 수정:
   ```
   sampleresult.default.encoding=UTF-8
   ```
3. JMeter 재시작

### Q3: JMeter가 메모리 부족으로 충돌합니다

**A**: 기본 힙 크기가 부족할 수 있습니다. 조정:
- Windows: bin/jmeter.bat 편집, HEAP 매개변수를 찾아 증가
- Linux/Mac: bin/jmeter에서 HEAP 변수 조정
권장 2GB 이상:
```bash
HEAP="-Xms2g -Xmx2g"
```

## 테스트 실행 문제

### Q4: 여러 다른 사용자 로그인을 시뮬레이션하려면 어떻게 해야 하나요?

**A**: 매개변수화 사용:
1. 사용자 이름과 비밀번호가 있는 CSV 파일 생성
2. "CSV 데이터 세트 구성" 요소 추가
3. HTTP 요청에서 ${username} 및 ${password} 변수 사용

### Q5: JMeter가 Session 또는 Cookie를 추출할 수 없습니다

**A**: 적절한 요소가 추가되었는지 확인:
1. 스레드 그룹에 "HTTP 쿠키 관리자" 추가
2. 로그인이 필요한 테스트의 경우 "반복마다 쿠키 지우기"가 제대로 구성되었는지 확인
3. 요청이 쿠키를 올바르게 전달하는지 확인

### Q6: 응답 데이터의 한글이 깨져서 표시됩니다

**A**: 해결 방법:
1. HTTP 요청에서 "Content Encoding"을 UTF-8로 추가 또는 수정
2. "HTTP 요청 기본값"에서 기본 인코딩 설정
3. 후처리기에서 "정규식 추출기"를 사용하고 인코딩 지정

### Q7: HTTPS 인증서 문제는 어떻게 처리하나요?

**A**: 테스트 환경의 경우:
1. 신뢰할 수 있는 인증서를 Java cacerts 저장소로 가져오기
2. 또는 HTTP 요청에서 "Implementation"을 Java로 설정
3. 자체 서명 인증서의 경우 JMeter의 truststore를 가져와야 할 수 있음

### Q8:长时间测试期间内存持续增长

**A**: 권장 조치:
1. 리스너가 수집하는 데이터 줄이기
2. 명령줄 모드에서 테스트 실행
3. jmeter.properties에서 가비지 수집 빈도 증가:
   ```
   mode=Stripped
   ```
4. 결과 파일 정기적으로 정리

## 성능 문제

### Q9: JMeter가 충분한 부하를 생성할 수 없습니다

**A**: 다음 방법 시도:
1. 분산 테스트(원격 테스트) 사용
2. 스레드 수 및 반복 횟수 조정
3. 불필요한 리스너 줄이기
4. GUI 모드 대신 명령줄 모드 사용
5. 네트워크 대역폭이 병목인지 확인

### Q10: 테스트 결과에서 응답 시간이 크게 변동합니다

**A**: 가능한 원인 및 해결 방법:
1. 서버 측 GC 또는 캐싱 문제
2. 불안정한 네트워크
3. 테스트 데이터에 의존성 있음
4. 동시 요청을 보장하기 위해 동기화 타이머 사용
5. 워밍업 시간 증가, 통계 수집 전에 서버가 안정되도록

### Q11: 합리적인 동시성은 어떻게 설정하나요?

**A**: 권장 접근 방식:
1. 낮은 동시성(10-50)으로 시작
2. 동시성을 점차 늘리고 시스템 응답 관찰
3. 성능 전환점 찾기(응답 시간이 증가하기 시작하는 지점)
4. 최대 안전 동시성 결정
5. 시스템 한계를 확인하기 위해 스트레스 테스트 수행

## 보고 및 결과 분석

### Q12: HTML 보고서는 어떻게 생성하나요?

**A**: 명령줄 사용:
```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/report
```
또는 기존 결과 파일에서 생성:
```bash
jmeter -g results.jtl -o /path/to/report
```

### Q13: 집계 보고서의 지표는 무엇을 의미하나요?

- **Samples**: 총 요청 수
- **Average**: 밀리초 단위 평균 응답 시간
- **Median**: 중앙값 응답 시간
- **90%/95%/99% Percentile**: 이 값보다 빠르게 응답하는 요청의 90%/95%/99%
- **Min/Max**: 최소 및 최대 응답 시간
- **Error %**: 실패한 요청의百分比
- **Throughput**: 초당 처리된 요청 수

### Q14: 결과 파일이 너무 큽니다

**A**: 해결 방법:
1. 리스너에서 CSV 형식으로 저장
2. 필요한 필드만 저장하도록 구성
3. "분산 테스트"를 사용하여 저장소 분산
4. 결과 파일 정기적으로ローテーション

## 스크립팅 문제

### Q15: 동적으로 변경되는 매개변수는 어떻게 처리하나요?

**A**: 일반적인 방법:
1. "정규식 추출기"를 사용하여 응답에서 추출
2. "JSON 추출기"를 사용하여 JSON 데이터 추출
3. "CSS/JQuery 추출기"를 사용하여 HTML 요소 추출
4. "XPath 추출기"를 사용하여 XML 데이터 추출

### Q16: BeanShell 스크립트가 실행되지 않습니다

**A**: 다음 확인:
1. JDK가 제대로 설치됨(JRE가 아닌)
2. BeanShell 라이브러리 파일이 lib 디렉토리에 있음
3. 스크립트 구문 올바른지 확인

### Q17: 복잡한 비즈니스 로직은 어떻게 구현하나요?

**A**: 다음可以使用:
1. 로직 컨트롤러 조합
2. BeanShell 또는 JSR223 스크립트
3. 전/후처리기
4. 조건 실행을 위한 스루풋 컨트롤러

## 기타 문제

### Q18: JMeter와 브라우저의 차이점은 무엇인가요?

JMeter는 프로토콜 수준에서 작동하며 브라우저가 아닙니다:
- JavaScript 실행 안 함
- HTML 페이지 렌더링 안 함
- CSS 처리 안 함
- 모든 브라우저 동작을 시뮬레이션할 수 없음

### Q19: JMeter는 어떻게 학습하나요?

권장 학습 경로:
1. JMeter의 공식 사용자 매뉴얼 읽기
2. 간단한 HTTP 요청으로 시작
3. 매개변수화 및 상관관계 학습
4. 분산 테스트 마스터
5. 사용자 정의 스크립트 개발 학습

### Q20: JMeter는 어떤 운영 체제를 지원하나요?

JMeter는 순수 Java 애플리케이션으로, Windows, Linux, Mac OSX, Solaris 등을 포함한 Java가 설치된 모든 OS를 지원합니다.
