# MongoDB Guide d'installation

## Sélection de version et exigences de plateforme

### Sélection de version

MongoDB dispose actuellement de deux séries de versions principales :

- **MongoDB 8.2** : Dernière version stable, recommandée pour les nouveaux projets
- **MongoDB 7.0** : Version de support à long terme, adaptée aux environnements de production nécessitant de la stabilité

À partir de MongoDB 8.2, les versions mineures sont disponibles pour des cas d'utilisation spécifiques dans les déploiements sur site.

### Exigences de plateforme

- MongoDB ne prend plus en charge les plateformes 32 bits. Assurez-vous que votre système d'exploitation est 64 bits
- MongoDB 6.0 a atteint sa fin de vie le 31 juillet 2025

### Plateformes prises en charge

L'édition communauté MongoDB prend en charge les plateformes et architectures suivantes :

**Ubuntu**

- Ubuntu 24.04 (x86_64, arm64)
- Ubuntu 22.04 (x86_64, arm64)
- Ubuntu 20.04 (x86_64, arm64)

- Debian 12 (x86_64)
- Debian 11 (x86_64)

**RHEL/CentOS/Rocky Linux**

- RHEL/Rocky/Alma/Oracle Linux 9.0+ (x86_64, arm64)
- RHEL/Rocky/Alma/Oracle Linux 8.0+ (x86_64, arm64, ppc64le, s390x)
- RHEL/Oracle Linux 7.0+ (x86_64)

**macOS**

- macOS 14 (x86_64, arm64)
- macOS 13 (x86_64, arm64)

**Windows**

**Autres Linux**

## Installer MongoDB Édition Communauté sur Ubuntu

### Étape 1 : Importer la clé publique

```bash
curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor
```

### Étape 2 : Créer le fichier de liste de référentiels

**Ubuntu 24.04 (Noble)** :

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] http://repo.mongodb.org/apt/debian noble/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
```

**Ubuntu 22.04 (Jammy)** :

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] http://repo.mongodb.org/apt/debian jammy/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
```

**Ubuntu 20.04 (Focal)** :

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] http://repo.mongodb.org/apt/debian focal/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
```

### Étape 3 : Mettre à jour la liste des paquets

```bash
sudo apt update
```

### Étape 4 : Installer MongoDB

```bash
sudo apt install -y mongodb-org
```

### Étape 5 : Démarrer MongoDB

```bash
sudo systemctl start mongod
sudo systemctl enable mongod
```

## Installer sur RHEL/CentOS/Rocky Linux

### Utiliser YUM/DNF

#### Étape 1 : Créer le fichier de référentiel

```bash
[mongodb-org-8.0]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/yum/redhat/$releasever/mongodb-org/8.0/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-8.0.asc
```

#### Étape 2 : Installer MongoDB

```bash
sudo dnf install -y mongodb-org
```

#### Étape 3 : Démarrer MongoDB

```bash
sudo systemctl start mongod
sudo systemctl enable mongod
```

## Installer sur Windows

### Utiliser le programme d'installation MSI

1. Téléchargez le programme d'installation MSI depuis le centre de téléchargement MongoDB
2. Exécutez le programme d'installation et suivez les étapes de l'assistant
3. Sélectionnez « Complete » pour installer tous les composants

### Utiliser le package ZIP

1. Téléchargez le package ZIP Windows
2. Extrayez les fichiers vers `C:\Program Files\MongoDB`
3. Créez le répertoire de données : `mkdir C:\data\db`
4. Démarrez MongoDB : `"C:\Program Files\MongoDB\Server\8.0\bin\mongod.exe" --dbpath C:\data\db`

## Installer avec Docker

### Étape 1 : Tirer l'image

```bash
docker pull mongodb/mongodb-community-server
```

### Étape 2 : Exécuter le conteneur

```bash
docker run --name mongodb -d -p 27017:27017 mongodb/mongodb-community-server
```

### Persistance des données

```bash
docker run --name mongodb -d -p 27017:27017 -v mongodb_data:/data/db mongodb/mongodb-community-server
```

## Configuration post-installation

### Créer un utilisateur administrateur

```bash
use admin
db.createUser({
  user: "admin",
  pwd: "votre_mot_de_passe",
  roles: [
    { role: "userAdminAnyDatabase", db: "admin" },
    { role: "readWriteAnyDatabase", db: "admin" }
  ]
})
```

### Activer l'authentification

1. Arrêter le service MongoDB
2. Modifier `/etc/mongod.conf` :

```yaml
security:
  authorization: enabled
```

3. Redémarrer le service :

```bash
sudo systemctl restart mongod
```

## Désinstaller MongoDB

### Ubuntu/Debian

```bash
sudo apt remove --purge mongodb-org
sudo rm -rf /var/lib/mongodb
sudo rm -rf /var/log/mongodb
```

### macOS

```bash
brew uninstall mongodb-community
brew services stop mongodb-community
```

## Questions fréquentes

### Quel est le port par défaut de MongoDB ?

Le port par défaut est 27017. Vous pouvez spécifier un autre port en utilisant le paramètre `--port`.

### Où MongoDB stocke-t-il les données ?

Par défaut :
- Linux/macOS : `/data/db`
- Windows : `C:\data\db`

### Comment vérifier le statut de MongoDB ?

```bash
sudo systemctl status mongod
```
