# OpenClaw 🦞

OpenClaw 是一個**自託管閘道器**，將你喜愛的聊天應用——WhatsApp、Telegram、Discord、iMessage 等——連接到 AI 編碼助手（如 Pi）。你可以在自己的電腦或伺服器上運行一個 Gateway 程式，它會成為你的訊息應用和隨時可用的 AI 助手之間的橋樑。

## 目標用戶

OpenClaw 面向希望擁有個人 AI 助手的開發者和進階用戶。你可以從任何地方透過訊息應用與 AI 助手交流，同時無需放棄對資料的控制，也不必依賴託管服務。

## 核心特性

OpenClaw 與眾不同的原因：

- **自託管**：運行在你的硬體上，你做主
- **多通道**：一個 Gateway 同時服務 WhatsApp、Telegram、Discord 等多個平台
- **原生代理支援**：為編碼助手構建，支援工具使用、對話管理、記憶和多代理路由
- **開源**：MIT 許可證，社群驅動

## 系統需求

運行 OpenClaw 你需要：

- Node.js 22 或更高版本
- 一個來自你選擇的服務商的 API 金鑰
- 5 分鐘時間

為了獲得最佳品質和安全性，建議使用最新一代最強大的模型。

## 快速開始

```bash
# 安裝 OpenClaw
npm install -g openclaw@latest

# 執行入門精靈並安裝服務
openclaw onboard --install-daemon

# 登入通道並啟動 Gateway
openclaw channels login
openclaw gateway --port 18789
```

## 工作原理

Gateway 是對話、路由和通道連接的單一資料來源。聊天應用透過它連接到 AI 代理、CLI、網頁控制介面或行動節點。

## 主要功能

- **多通道閘道器**：一個 Gateway 程式支援 WhatsApp、Telegram、Discord、iMessage
- **外掛通道**：透過擴充套件添加 Mattermost 等更多平台
- **多代理路由**：每個代理、工作區或傳送者都有隔離的對話
- **媒體支援**：支援傳送和接收圖片、音訊和文件
- **網頁控制介面**：瀏覽器儀表板用於聊天、配置、對話和節點管理
- **行動節點**：配對 iOS 和 Android 節點，實現 Canvas、鏡頭/螢幕和語音工作流
