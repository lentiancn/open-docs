# Руководство по установке

В этом руководстве описана установка и настройка Angular, которая поможет вам быстро подготовить рабочее окружение.

## Требования к окружению

Перед установкой Angular убедитесь, что ваше окружение соответствует следующим требованиям:

- **Node.js**: рекомендуется версия 18.10.0 или выше
- **npm**: поставляется вместе с Node.js, версия 6.x или выше
- **Операционная система**: Windows, macOS или Linux

## Установка Angular CLI

Angular CLI — это официальный инструмент командной строки для создания и управления проектами Angular.

### Глобальная установка

```bash
npm install -g @angular/cli
```

### Проверка установки

```bash
ng version
```

После успешной установки отобразится информация о версии Angular CLI.

### Установка конкретной версии

Если вам нужна определённая версия:

```bash
npm install -g @angular/cli@17
```

## Создание нового проекта

### Создание через CLI

```bash
ng new my-app
```

Эта команда создаёт новый проект с именем my-app и автоматически устанавливает зависимости.

### Дополнительные параметры при создании

```bash
# Создать без установки зависимостей
ng new my-app --skip-install

# Использовать SCSS для стилей
ng new my-app --style=scss

# Включить маршрутизацию
ng new my-app --routing

# Создать независимые компоненты (Angular 14+)
ng new my-app --standalone
```

## Настройка окружения разработки

### Использование зеркал для России

Из-за проблем с сетью рекомендуется настроить российские зеркала:

```bash
# Настроить npm-зеркало
npm config set registry https://registry.npmmirror.com

# Или использовать nrm для управления зеркалами
npm install -g nrm
nrm use taobao
```

### Установка Node.js

**Установка через nvm (рекомендуется):**

```bash
# Установить nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Установить LTS-версию Node.js
nvm install --lts
nvm use --lts
```

**Для Windows используйте nvm-windows:**

Скачайте установщик с https://github.com/coreybutler/nvm-windows

## Запуск сервера разработки

### Переход в директорию проекта

```bash
cd my-app
```

### Запуск сервера разработки

```bash
ng serve
```

По умолчанию приложение доступно по адресу: http://localhost:4200

### Горячая перезагрузка

Angular CLI поддерживает горячую перезагрузку — браузер автоматически обновляется при изменении кода.

## Сборка для продакшена

### Команда сборки

```bash
ng build
```

Результаты сборки будут в директории dist.

### Оптимизация для продакшена

```bash
ng build --configuration=production
```

Эта команда включает сжатие кода, tree-shaking и другие оптимизации.

## Часто используемые команды CLI

| Команда | Описание |
|---------|----------|
| ng generate component xxx | Создать компонент |
| ng generate service xxx | Создать сервис |
| ng generate module xxx | Создать модуль |
| ng generate directive xxx | Создать директиву |
| ng generate pipe xxx | Создать канал |
| ng generate guard xxx | Создать охранный маршрут |

## Инструменты разработки

### Расширения для VS Code

- Angular Language Service — проверка типов в шаблонах и автодополнение
- Angular Snippets — готовые сниппеты кода
- Prettier — форматирование кода

### Отладка

Для отладки и профилирования производительности используйте браузерное расширение Angular DevTools.

## Удаление Angular CLI

Если нужно удалить:

```bash
npm uninstall -g @angular/cli
```
