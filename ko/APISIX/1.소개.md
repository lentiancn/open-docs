# 소개

Apache APISIX는 Apache Software Foundation의 탑 프로젝트이며, API7.ai에서 개발하고 기증했습니다. 동적이고, 실시간이며, 고성능등의 특징을 갖춘 클라우드 네이티브 API 게이트웨이입니다.

APISIX는 모든 비즈니스 트래픽의 진입점으로서 동적 라우팅, 동적 업스트림, 동적 인증서, A/B 테스트, 그레이드 배포(카나리 배포), 블루그린 배포, 속도 제한, 공격 방지, 지표 수집, 모니터링 경고, 가시성 및 서비스 거버넌스 등의 풍부한 기능을 제공합니다.

## 핵심 기능

### 고성능

APISIX는 Nginx 기반으로 구축되어 Nginx의 우수한 성능을 계승합니다. 플러그인 메커니즘을 통해 비즈니스 로직이 핵심 기능과 분리되어 확장이 더 유연하면서도 효율적인 처리 능력을 유지합니다.

### 완전한 동적

기존 Nginx가 구성을 다시 로드해야 하는 것과 달리, APISIX는 완전한 동적 구성을 지원합니다. 모든 라우터, 업스트림, 플러그인 등은 관리 API를 통해 실시간으로 업데이트할 수 있으며, 서비스를 다시 시작할 필요 없이 즉시 적용됩니다.

### 클라우드 네이티브 설계

APISIX는 처음부터 클라우드 네이티브 시나리오를 고려하여 Docker, Kubernetes, Helm Chart 등 다양한 배포 방식을 지원합니다. 또한 Ingress Controller를 제공하여 Kubernetes 인그레스 게이트웨이로 직접 사용할 수 있습니다.

### 플러그인 생태계

APISIX는 신원 인증, 속도 제한, 요청 재작성, 응답 처리, 모니터링 로깅 등을 포괄하는 다양한 플러그인 시스템을 제공합니다. 사용자는 플러그인 메커니즘을 통해 게이트웨이 기능을 쉽게 확장할 수 있습니다.

### 다중 프로토콜 지원

전통적인 HTTP/HTTPS 프로토콜 외에도 APISIX는 gRPC, Dubbo, WebSocket, MQTT 등 다양한 프로토콜을 지원하여 마이크로서비스 아키텍처의 다양한 시나리오 요구를 충족할 수 있습니다.

## 아키텍처 개요

APISIX의 아키텍처는 데이터 플레인과 제어 플레인 두 가지로 나뉩니다:

**데이터 플레인**은 실제 요청전달을 담당하며, Nginx를 개조한 APISIX Core가고성능의 요청 처리 파이프라인을 구현합니다.

**제어 플레인**은 RESTful API를 통해 모든 구성을 관리하며, etcd, Consul 등 다양한 구성 지속화 방식을 지원합니다. APISIX는 또한 Dashboard를 제공하여 Web 인터페이스를 통한 시각적 구성 관리가 가능합니다.

## 적용 시나리오

- 마이크로서비스 API 통합 진입점
- Kubernetes Ingress Controller
- API 트래픽 관리 및 모니터링
- 멀티테넌시 API 관리
- 서비스 메시 사이드카 프록시
- AI 애플리케이션 게이트웨이(LLM 및 AI 워크로드)

Apache Software Foundation의 탑 프로젝트로서, APISIX는활발한 커뮤니티와 지속적인 버전 업데이트를 보유하며 기업 애플리케이션에 대한 신뢰할 수 있는 선택을 제공합니다.
