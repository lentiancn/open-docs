# Preguntas Frecuentes de Yarn

## Preguntas Relacionadas con la Instalación

### P: ¿Cómo elegir entre versiones de Yarn?
**R**: 
- **Nuevos Proyectos**: Se recomienda usar Yarn v4+ para disfrutar de funciones modernas y mejor rendimiento
- **Proyectos Existentes**: Si ya estás usando Yarn v1, puedes continuar usándolo o considerar migrar a la versión moderna
- **Entornos Empresariales**: Yarn v1 proporciona soporte a largo plazo más estable, mientras que v4+ ofrece funciones más innovadoras

### P: ¿Qué es Corepack y por qué lo necesito?
**R**: Corepack es una herramienta intermedia incluida con Node.js que te permite configurar la versión de tu gestor de paquetes por proyecto. Asegura que los miembros del equipo usen la misma versión de Yarn, evitando problemas de inconsistencia de versiones.

### P: ¿Qué hacer si encuentro errores de permisos durante la instalación de Yarn?
**R**: 
- Agrega la bandera `--unsafe-perm` al usar `npm install --global yarn`
- Usa nvm para gestionar versiones de Node.js
- En entornos CI/CD, normalmente no se requiere instalación global

## Preguntas Relacionadas con el Uso

### P: ¿Cuáles son las diferencias entre Yarn y npm?
**R**: 
- **Velocidad**: Yarn usa instalaciones paralelas y caché offline, típicamente más rápido
- **Determinismo**: Yarn garantiza instalaciones consistentes (yarn.lock)
- **Funciones**: Yarn proporciona workspaces, PnP, sistema de plugins y otras funciones avanzadas
- **Comandos**: Yarn reimplementa todos los comandos para una mejor experiencia del desarrollador

### P: ¿Qué es PnP (Plug'n'Play)?
**R**: PnP es una función de Yarn v2+ que elimina el directorio node_modules resolviendo dependencias directamente desde la caché. Esto ahorra espacio en disco y mejora la velocidad de instalación, pero puede ser incompatible con algunas herramientas que no soportan PnP.

### P: ¿Cómo usar workspaces en mi proyecto?
**R**: 
1. Agrega un campo `"workspaces"` a tu `package.json` raíz
2. Crea directorios de subpaquetes (por ejemplo, `packages/`)
3. Crea archivos `package.json` individuales en subpaquetes
4. Usa `yarn workspace <name> <command>` para ejecutar comandos en workspaces específicos

## Preguntas sobre Migración de Versiones

### P: ¿Cómo migrar de Yarn v1 a v4+?
**R**: 
1. Asegúrate de que tu proyecto esté bajo control de versiones Git
2. Ejecuta `yarn set version stable` (para v4+)
3. Sigue la guía de migración para actualizar configuraciones paso a paso
4. Prueba toda la funcionalidad para asegurar compatibilidad

### P: ¿Qué hacer si encuentro problemas de compatibilidad después de la migración?
**R**: 
- Verifica si hay herramientas que no soportan PnP, puedes desactivar PnP: `yarn config set nodeLinker node-modules`
- Actualiza dependencias a versiones compatibles
- Consulta la documentación oficial de migración para soluciones específicas

## Solución de Problemas

### P: ¿Qué hacer si `yarn install` falla?
**R**: 
- Limpiar caché: `yarn cache clean`
- Eliminar `yarn.lock` y reinstalar
- Verificar conexión de red y configuración de registro
- Para Yarn v4+, verificar configuración de `.yarnrc.yml`

### P: ¿Cómo verificar la versión de Yarn?
**R**: 
- Versión global: `yarn --version`
- Versión del proyecto: Verificar directorio `.yarn/releases/` o archivo `.yarnrc.yml`

### P: ¿Qué hacer si los comandos de Yarn son muy lentos?
**R**: 
- Asegurarse de que el caché offline esté habilitado
- Verificar conexión de red
- Para proyectos grandes, considerar usar workspaces para dividir la base de código
- Actualizar a la última versión para mejoras de rendimiento
