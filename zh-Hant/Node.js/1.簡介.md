# Node.js 簡介

Node.js® 是一個免費、開源、跨平台的 JavaScript 執行環境，它讓開發人員能夠建立伺服器、Web 應用、命令列工具和指令碼。

## 核心概念

### 什麼是 Node.js？

Node.js 是一個基於 Chrome V8 JavaScript 引擎建構的 JavaScript 執行環境。它使用事件驅動、非阻塞 I/O 模型，使其輕量且高效能。Node.js 的套件生態系統 npm 是世界上最大的開源函式庫生態系統。

### 設計理念

Node.js 的核心設計理念是**非阻塞 I/O**和**事件驅動架構**。與傳統的多執行緒併發模型不同，Node.js 使用單一執行緒事件迴圈來處理併發請求，這使得它在處理大量併發連線時非常高效能。

## 核心特性

### 1. 事件驅動架構

Node.js 將事件迴圈作為執行時期建構而不是函式庫。這意味著：

- 執行完輸入指令稿後，Node.js 自動進入事件迴圈
- 當沒有更多的回呼要執行時，Node.js 退出事件迴圈
- 這種行為類似於瀏覽器中的 JavaScript

### 2. 非阻塞 I/O

Node.js 幾乎沒有任何函式直接執行阻塞 I/O 操作（除了同步方法）。這意味著：

- 處理程序永遠不會因為等待 I/O 操作而阻塞
- 可以同時處理數千個併發連線
- 避免了傳統多執行緒模型中的死結問題

### 3. 單一執行緒模型

雖然 Node.js 使用單一執行緒處理請求，但它可以透過以下方式利用多核心處理器：

- **child_process.fork()**: 建立子處理程序
- **cluster 模組**: 在多個核心上負載平衡

### 4. HTTP 支援

HTTP 在 Node.js 中是一等公民，設計時考慮了串流傳輸和低延遲。這使得 Node.js 非常適合作為 Web 函式庫或框架的基礎。

## 基本範例

### Hello World Web 伺服器

```javascript
const { createServer } = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

### 特點說明

- **createServer()**: 建立新的 HTTP 伺服器
- **request 物件**: 包含請求詳細資訊（headers、data 等）
- **response 物件**: 用於向用戶端返回資料
- **statusCode**: 設定 HTTP 狀態碼（200 表示成功）
- **setHeader()**: 設定回應標頭
- **end()**: 結束回應並傳送內容

## 適用場景

### 適合的場景

- **即時應用**: 聊天應用、即時通知、線上遊戲
- **API 伺服器**: RESTful API、微服務
- **資料串流應用**: 影片串流、音訊串流、檔案上傳/下載
- **單頁應用 (SPA)**: 前後端分離的現代 Web 應用
- **工具和指令稿**: 命令列工具、自動化指令稿

### 不適合的場景

- **CPU 密集型任務**: 大量計算、影像處理、影片編碼
- **關聯式資料庫密集型應用**: 複雜的 SQL 查詢和交易

## 生態系統

### npm (Node Package Manager)

- 世界上最大的軟體註冊表
- 超過 200 萬個可用套件
- 簡單的相依性和版本控制

### 主流框架和函式庫

- **Express.js**: 最流行的 Web 應用框架
- **Koa**: 由 Express 團隊開發的下一代框架
- **Fastify**: 高效能 Web 框架
- **NestJS**: 基於 TypeScript 的漸進式框架
- **Socket.IO**: 即時雙向通訊
