# Benutzerhandbuch

Dieses Handbuch beschreibt die Kernfunktionen und使用方法 von Apifox im Detail, einschließlich Projektverwaltung, Schnittstellendesign, Debugging, Mocking und automatisierte Tests.

## Projektverwaltung

### Projekt erstellen

In Apifox ist ein Projekt die höchste Organisationseinheit für APIs.

```bash
# Schritte zum Erstellen eines neuen Projekts
1. Klicken Sie auf das "+" neben der Projektliste auf der linken Seite
2. Geben Sie einen Projektnamen ein
3. Wählen Sie den Projekttyp
4. Wählen Sie eine Grundvorlage (optional)
5. Klicken Sie auf "Erstellen"
```

### Projekteinstellungen

Jedes Projekt verfügt über eigene Einstellungsoptionen:

- **Grundinformationen:** Projektname, Beschreibung, Symbol
- **Umgebungsvariablen:** Variablen für verschiedene Umgebungen wie Entwicklung, Test und Produktion
- **Authentifizierung:** Globale Authentifizierungsmethode festlegen
- **Export/Import:** Import und Export von Projektdaten

### Umgebungsverwaltung

Apifox unterstützt die Konfiguration mehrerer Umgebungen:

```bash
# Verwendung von Umgebungsvariablen
Verwenden Sie {{Variablenname}} in Anfragen, um Umgebungsvariablen zu referenzieren
Beispiel: {{baseUrl}}/api/users

Umgebung wechseln: Wählen Sie oben im Umgebungsauswahlfeld eine andere Umgebung
```

## Schnittstellendesign

### Schnittstelle erstellen

Eine Schnittstelle ist die konkrete Definition einer API:

```bash
# Schritte zum Erstellen einer Schnittstelle
1. Klicken Sie im Projekt auf "+" und wählen Sie "Neue Schnittstelle"
2. Füllen Sie die grundlegenden Informationen aus:
   - Schnittstellenname: Benutzerliste
   - Anfragepfad: /api/users
   - Anfragemethode: GET
3. Definieren Sie Anfrageparameter
4. Definieren Sie die Antwortstruktur
5. Speichern Sie
```

### Anfrageparameter

Folgende Arten von Anfrageparametern werden unterstützt:

- **Pfadparameter:** z.B. id in /api/users/:id
- **Abfrageparameter:** ?name=test in der URL
- **Anfrageheader:** z.B. Authorization
- **Anfragekörper:** JSON, Formulardaten, XML und andere Formate

### Antwortdefinition

Definieren Sie erfolgreiche und fehlgeschlagene Antworten für jede Schnittstelle:

```json
{
  "Erfolgreiche Antwort 200": {
    "id": 1,
    "name": "Max Mustermann",
    "email": "max@example.com"
  },
  "Fehlerantwort 404": {
    "code": 404,
    "message": "Benutzer nicht gefunden"
  }
}
```

### Schnittstellenlistenverwaltung

Unterstützt die gruppierte Verwaltung von Schnittstellen, die nach Modulen oder Funktionen kategorisiert werden können:

- Ziehen zum Anpassen der Reihenfolge
- Stapeloperationen (Stapel löschen, Stapel exportieren usw.)
- Suchen und Filtern

## Schnittstellen-Debugging

### Anfrage senden

Klicken Sie auf der Schnittstellendetailseite auf die Schaltfläche "Senden", um eine Anfrage zu senden:

```bash
# Grundlegende Debugging-Schritte
1. Öffnen Sie die Schnittstelle
2. Wählen Sie die Umgebung
3. Parameterwerte festlegen
4. Klicken Sie auf "Senden"
5. Sehen Sie sich die Antwort an
```

### Authentifizierungseinstellungen

Verschiedene Authentifizierungsmethoden werden unterstützt:

- **Keine:** Keine Authentifizierung
- **Bearer Token:** JWT-Authentifizierung
- **Basic Auth:** Basic-Authentifizierung
- **OAuth 2.0:** OAuth2-Authentifizierung
- **Benutzerdefinierte Authentifizierung:** Authentifizierungsheader nach Bedarf anpassen

### Vor-Anfrage-Skripte

Sie können Skripte festlegen, die vor dem Senden der Anfrage ausgeführt werden:

```javascript
// Beispiel: Zeitstempel hinzufügen
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// Beispiel: Zufällige ID generieren
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### Nach-Antwort-Skripte

Sie können Skripte festlegen, die nach dem Empfang der Antwort ausgeführt werden:

```javascript
// Beispiel: Antwortdaten extrahieren
const response = pm.response.json();
console.log('Antwortdaten:', response);

// Beispiel: Umgebungsvariable setzen
pm.environment.set('access_token', response.data.token);
```

## API-Mocking

### Intelligentes Mocking

Eine der mächtigsten Funktionen von Apifox: Sie definieren nur die Datenstruktur, und das System generiert automatisch Mock-Daten:

```bash
# Intelligentes Mocking aktivieren
1. Öffnen Sie die Schnittstelle
2. Klicken Sie auf die Registerkarte "Mock"
3. Aktivieren Sie "Intelligentes Mocking"
4. Speichern Sie
```

### Mock-Regeln

Benutzerdefinierte Mock-Regeln werden unterstützt:

- `@string`: Zufällige Zeichenkette generieren
- `@integer`: Zufällige Ganzzahl generieren
- `@float`: Zufällige Fließkommazahl generieren
- `@boolean`: Zufälligen Boolean generieren
- `@date`: Zufälliges Datum generieren
- `@email`: Zufällige E-Mail generieren
- `@url`: Zufällige URL generieren
- `@guid`: UUID generieren

### Beispiel

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

Generierte Mock-Daten:

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock-Server

Apifox bietet einen integrierten Mock-Server:

```bash
# Mock-Adresse abrufen
Kopieren Sie die Mock-URL auf der Schnittstellendetailseite
Beispiel: https://mock.apifox.com/api/users
```

## Automatisierte Tests

### Testfall erstellen

```bash
# Schritte zum Erstellen eines Tests
1. Öffnen Sie die Schnittstelle
2. Klicken Sie auf die Registerkarte "Test"
3. Klicken Sie auf "Test erstellen"
4. Geben Sie einen Testfallnamen ein
5. Fügen Sie Assertions hinzu
6. Speichern Sie
```

### Assertion-Typen

Verschiedene Assertions werden unterstützt:

- **Statuscode-Assertion:** Antwortstatuscode prüfen
- **JSON-Assertion:** Antwort-JSON-Daten prüfen
- **Antwortzeit-Assertion:** Antwortzeit prüfen
- **Antwortheader-Assertion:** Antwortheader prüfen

### Beispiele

```javascript
// Statuscode-Assertion
pm.test("Statuscode ist 200", function() {
    pm.response.to.have.status(200);
});

// JSON-Assertion
pm.test("Benutzerliste zurückgegeben", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// Antwortzeit-Assertion
pm.test("Antwortzeit unter 500ms", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### Test-Suite

Testfälle mehrerer Schnittstellen können zu einer Test-Suite zusammengefasst werden:

1. Test-Suite erstellen
2. Testfälle hinzufügen
3. Ausführungsreihenfolge festlegen
4. Suite ausführen

### Testbericht erstellen

Nach der Testausführung können detaillierte Testberichte erstellt werden:

- Erfolgsquote-Statistik
- Fehlgeschlagene Testfälle-Analyse
- Antwortzeit-Statistik

## Teamzusammenarbeit

### Mitglieder einladen

```bash
# Teammitglieder einladen
1. Gehen Sie zu den Projekteinstellungen
2. Klicken Sie auf "Mitgliederverwaltung"
3. Klicken Sie auf "Mitglied einladen"
4. Geben Sie die E-Mail-Adresse des Mitglieds ein
5. Rollenberechtigungen festlegen
6. Einladung senden
```

### Rollen und Berechtigungen

- **Administrator:** Volle Projektberechtigungen
- **Bearbeiter:** Schnittstellen bearbeiten und Tests ausführen
- **Betrachter:** Nur Ansicht, keine Bearbeitung

### Echtzeit-Synchronisierung

Nach der Anmeldung werden Projektdaten automatisch in Echtzeit synchronisiert:

- Schnittstellenänderungen werden automatisch synchronisiert
- Testergebnisse werden automatisch synchronisiert
- Teammitglieder sehen stets die neuesten Inhalte

### Kommentarfunktion

Kommentare können zu Schnittstellen hinzugefügt werden:

1. Öffnen Sie die Schnittstelle
2. Klicken Sie auf "Kommentar"
3. Geben Sie den Kommentar ein
4. @Erwähnen Sie Teammitglieder

Häufig verwendete Tastenkombinationen:

## Best Practices

1. **Klare Benennung:** Schnittstellennamen beschreiben klar die Funktion
2. **Vollständige Kommentare:** Fügen Sie Erklärungen für jeden Parameter hinzu
3. **Vernünftige Gruppierung:** Organisieren Sie Schnittstellen nach Modulen
4. **Umgebungen verwenden:** Unterscheiden Sie zwischen Entwicklungs-, Test- und Produktionsumgebung
5. **Tests schreiben:** Schreiben Sie automatisierte Tests für wichtige Schnittstellen
6. **Regelmäßig synchronisieren:** Halten Sie lokale und Cloud-Daten konsistent
