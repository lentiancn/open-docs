# Foire Aux Questions

Ce document recense les questions techniques fréquentes posées lors de l'utilisation d'Apache APISIX.

## Questions générales

### Qu'est-ce qu'Apache APISIX ?

Apache APISIX est un projet de premier plan de la Apache Software Foundation. C'est une passerelle API native pour le cloud, dynamique, en temps réel et haute performance. Elle offre des fonctionnalités telles que le routage dynamique, l'upstream dynamique, les certificats dynamiques, les tests A/B, le déploiement progressif, la limitation de débit, la protection contre les attaques, la surveillance et les alertes.

### Quelle est la différence entre APISIX et Nginx ?

Nginx est un serveur web traditionnel et un proxy inverse, tandis qu'APISIX est une passerelle API construite sur Nginx. APISIX ajoute à Nginx une gestion de configuration entièrement dynamique, un système de plugins riche, une gestion fine du trafic et des fonctionnalités d'observabilité intégrées.

### Quelles sont les performances d'APISIX ?

APISIX hérite des caractéristiques haute performance de Nginx tout en maintenant une capacité de traitement efficace grâce aux optimisations et au système de plugins. Les performances réelles dépendent de la configuration matérielle et de la complexité métier. Dans des conditions normales, il peut traiter plusieurs dizaines de milliers, voire des centaines de milliers de requêtes par seconde.

## Questions d'installation

### Docker ne démarre pas, que faire ?

1. Vérifiez que Docker fonctionne normalement : `docker ps`
2. Consultez les logs du conteneur : `docker-compose logs apisix`
3. Problèmes courants :
   - Conflit de ports : Vérifiez que les ports 9080, 9180 et 9443 ne sont pas utilisés
   - Échec de connexion etcd : Vérifiez que le conteneur etcd est démarré correctement
   - Problèmes de permissions : Assurez-vous d'avoir les droits nécessaires pour exécuter Docker

### Comment connaître la version d'APISIX ?

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Le format de réponse est : `Server: APISIX/Version`

### Impossible d'accéder au Dashboard après l'installation ?

1. Vérifiez que le port du Dashboard est correctementMapper (9180 par défaut)
2. Vérifiez les paramètres du pare-feu
3. Confirrez que la configuration réseau du conteneur est correcte

## Questions de configuration

### Comment modifier le port de l'API d'administration ?

Modifiez le fichier de configuration `apisix.yml` :

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### Comment activer HTTPS ?

1. Configurez le certificat SSL :

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "contenu du certificat",
  "key": "clé privée",
  "snis": ["example.com"]
}'
```

2. Activez HTTPS dans la route :

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### La configuration de la route ne prend pas effet ?

1. Vérifiez que la configuration de la route est correcte
2. Confirmez que la synchronisation etcd fonctionne
3. Vérifiez que les plugins n'ont pas d'erreurs de syntaxe
4. Consultez les logs d'erreur d'APISIX

### Comment gérer les requêtes cross-origin (CORS) ?

Utilisez le plugin `cors` :

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## Questions de performance

### Comment améliorer le débit ?

1. **Augmenter le nombre de workers** : `apisix.worker_processes: auto`
2. **Activer HTTP/2** : Configurez `http2: true` dans listen
3. **Optimiser le pool de connexions upstream** : Configurez `keepalive_pool.size`
4. **Utiliser les plugins de cache** : Activez le plugin `proxy-cache`

### La mémoire est trop élevée, que faire ?

1. Réduire le chargement des plugins inutiles
2. Limiter le nombre de connexions upstream
3. Ajuster la taille du buffer de logs
4. Utiliser le traitement en streaming pour les gros fichiers

## Questions fonctionnelles

### Comment faire des tests A/B ?

Utilisez le plugin `traffic-split` :

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### Comment limiter le taux de requêtes ?

Utilisez le plugin `limit-req` :

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## Questions d'intégration

### Comment intégrer avec Kubernetes ?

Utilisez l'Ingress Controller APISIX :

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### Comment intégrer Prometheus ?

1. Activez le plugin Prometheus
2. Configurez Prometheus pour récupérer les métriques

## Questions de sécurité

### Comment protéger APISIX contre les abus ?

1. **Activer l'authentification** : Activez key-auth ou d'autres plugins d'auth sur toutes les routes
2. **Limiter l'accès à l'API d'administration** : Autorisez uniquement les IP de confiance
3. **Utiliser HTTPS** : Activez le chiffrement TLS
4. **Configurer la limitation de débit** : Prévenez les requêtes malveillantes
5. **Activer les logs d'accès** : Enregistrez toutes les requêtes pour audit

### Comment activer l'authentification pour l'API d'administration ?

```yaml
deployment:
  admin:
    admin_key_required: true
```

## Questions d'exploitation

### Comment mettre à jour APISIX ?

**Mode Docker :**

```bash
docker-compose pull
docker-compose up -d
```

**Mode Kubernetes :**

```bash
helm upgrade apisix apisix/apisix
```

### Codes d'erreur courants

- **401 Unauthorized** : Échec de l'authentification
- **403 Forbidden** : Pas d'autorisation d'accès
- **404 Not Found** : Route inexistante
- **429 Too Many Requests** : Requêtes dépassant la limite
- **500 Internal Server Error** : Erreur du service upstream
- **502 Bad Gateway** : Service upstream indisponible
- **503 Service Unavailable** : Service indisponible
