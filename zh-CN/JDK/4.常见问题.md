# JDK 常见问题

## 基础问题

### 什么是 JDK

JDK（Java Development Kit）是 Java 语言的软件开发工具包，它是开发 Java 应用程序所必需的软件包。JDK 包含了 Java 编译器（javac）、Java 虚拟机（JVM）、Java 核心类库以及各种开发工具。没有 JDK，就无法开发和运行 Java 程序。

JDK 和 JRE 的区别在于：JRE（Java Runtime Environment）只包含运行 Java 程序所需的环境，不包含开发工具。如果只需要运行 Java 程序，安装 JRE 即可；如果需要进行 Java 开发，必须安装 JDK。

### 什么是 JVM

JVM（Java Virtual Machine）是 Java 虚拟机的缩写，它是 Java 程序运行的核心。JVM 负责将 Java 字节码（.class 文件）解释或编译为本机代码执行。JVM 的存在使得 Java 程序可以实现“一次编写，到处运行”的跨平台特性。

JVM 不仅运行 Java 程序，还负责内存管理（垃圾回收）、线程管理、安全管理等工作。理解 JVM 的工作原理对于编写高性能 Java 应用非常重要。

### JDK 和 JRE 有什么区别

JDK（Java Development Kit）是完整的开发工具包，包含 JRE 和开发工具。JRE（Java Runtime Environment）是 Java 运行时环境，只包含运行 Java 程序所需的组件。

简单来说：如果你只是要运行别人开发好的 Java 程序，安装 JRE 就够了；如果你要自己开发 Java 程序，必须安装 JDK。JDK 中包含了完整的 JRE，所以安装了 JDK 后不需要再单独安装 JRE。

### 哪个 JDK 版本最好

没有绝对的“最好”版本，选择取决于具体需求。对于新项目，推荐使用 JDK 21（最新的 LTS 版本）或 JDK 25（最新版本）。对于需要长期维护的企业项目，推荐使用 LTS 版本如 JDK 21 或 JDK 17。

JDK 21 是最新的长期支持版本，获得至少 8 年的支持。JDK 17 仍然广泛应用且免费使用。JDK 8 虽然老旧，但仍然在很多遗留项目中使用。

## 安装问题

### 如何检查 JDK 是否安装成功

在命令行中运行 java -version，如果显示版本信息则说明 JDK 已正确安装。运行 javac -version 可以检查编译器是否可用。运行 echo %JAVA_HOME%（Windows）或 echo $JAVA_HOME（Linux/macOS）可以检查环境变量是否设置正确。

如果在命令行中运行这些命令出现错误，首先确认 JDK 是否正确安装，然后检查环境变量配置是否正确。

### JAVA_HOME 是什么

JAVA_HOME 是一个环境变量，指向 JDK 的安装目录。许多 Java 工具和构建脚本（如 Maven、Gradle）依赖 JAVA_HOME 来定位 JDK。正确设置 JAVA_HOME 可以确保这些工具能够正常工作。

在 Windows 上，JAVA_HOME 通常设置为如 C:\Program Files\Java\jdk-21 这样的路径。在 Linux/macOS 上，通常设置为如 /usr/lib/jvm/java-21-openjdk 这样的路径。

### 如何在 Windows 上安装多个 JDK 版本

可以在同一台机器上安装多个 JDK 版本。安装多个版本后，通过修改 PATH 环境变量中 JDK bin 目录的顺序，或者修改 JAVA_HOME 环境变量来切换默认版本。

也可以使用第三方工具来管理多个 JDK 版本，如 SDKMAN! 的 Windows 版本或 JetBrains Toolbox。使用这些工具可以更方便地在不同版本之间切换。

### Oracle JDK 和 OpenJDK 有什么区别

Oracle JDK 和 OpenJDK 在代码层面几乎完全相同，主要区别在于许可协议和技术支持。Oracle JDK 提供商业支持和额外的企业级功能，但某些版本在商业使用时可能需要付费。OpenJDK 是开源免费的，由社区维护。

对于大多数使用场景，OpenJDK完全可以满足需求。许多组织选择使用 OpenJDK 以避免许可问题，同时获得与 Oracle JDK 几乎相同的性能和功能。

## 使用问题

### 什么是 classpath

classpath 是 JVM 用来查找类文件的路径。当你运行 java MyClass 时，JVM 会在 classpath 指定的目录和 JAR 文件中查找 MyClass.class。

在编译和运行Java程序时，可以通过 -classpath 或 -cp 选项指定 classpath。如果未指定，JVM 默认使用当前目录作为 classpath。

### 什么是字节码

字节码是 Java 编译器将源代码编译成的一种中间形式的代码。字节码不是本机代码，不能直接在 CPU 上运行，只能由 JVM 解释执行或 JIT 编译后执行。

字节码的存在使得 Java 程序具有跨平台特性。同样的字节码可以在任何安装了 JVM 的系统上运行，无论是 Windows、Linux 还是 macOS。

### 什么是 JAR 文件

JAR（Java Archive）文件是一种将多个 Java 类文件打包成一个文件的格式。JAR 文件实际上是 ZIP 格式的压缩包，里面包含了编译后的 .class 文件和一些元数据。

使用 jar 工具可以创建、查看和提取 JAR 文件。可执行的 JAR 文件包含 MANIFEST.MF 文件，其中指定了主类，可以直接用 java -jar 命令运行。

### 什么是模块系统

Java 模块系统（Project Jigsaw）从 JDK 9 开始引入，将 JDK 和应用程序划分为模块。模块提供了更好的封装，明确声明了依赖关系，有助于构建更健壮和可维护的应用程序。

模块通过 module-info.java 文件声明，定义了模块名、依赖的其他模块、导出的包等信息。模块系统帮助解决了类路径的类冲突问题，并提供了更好的版本管理。

## 性能问题

### 如何优化 JVM 性能

JVM 性能优化主要从以下几个方面入手：合理设置堆内存大小，避免设置过大或过小；选择合适的垃圾回收器，G1 是大多数情况的好选择；对于追求低延迟的应用，考虑使用 ZGC 或 Shenandoah；启用类数据共享（-Xshare:classes）加快启动速度。

使用性能监控工具（如 VisualVM、JConsole）分析应用性能，找出瓶颈并进行针对性优化。

### 什么是垃圾回收

垃圾回收（Garbage Collection，GC）是 JVM 自动管理内存的机制。程序运行时会产生对象，这些对象占用内存。当对象不再被引用时，垃圾回收器会自动识别并回收这些对象占用的内存。

Java 的垃圾回收大大简化了内存管理工作，程序员不需要显式释放内存。但理解垃圾回收的原理有助于编写更高效的代码，避免内存泄漏和频繁的 GC。

### 什么是内存泄漏

内存泄漏是指程序在运行过程中申请的内存无法被回收，导致可用内存逐渐减少。在 Java 中，内存泄漏通常是由于持有对象引用的集合（如静态的 Map、List）不断添加对象但没有及时清理造成的。

排查内存泄漏可以使用 jmap 生成堆转储文件，然后使用 MAT（Memory Analyzer Tool）等工具分析来找出泄漏的对象。

### 什么是 JIT 编译器

JIT（Just-In-Time）编译器是 JVM 的一部分，负责将热点字节码编译为本机代码以提高执行性能。JVM 在运行时会分析代码的执行情况，对于频繁执行的代码（热点代码），JIT 编译器会将其编译为本地机器码，从而显著提升性能。

JIT 编译是在运行时进行的，不需要额外的编译步骤。C1 编译器编译速度快但优化程度低，C2 编译器编译速度慢但优化程度高。分层编译结合了两者的优点。

## 版本问题

### 应该使用哪个 Java 版本

对于新项目，推荐使用最新的 LTS 版本 JDK 21。它提供了良好的性能、丰富的特性和长期支持。对于需要兼容 Java 8/11 的项目，可以使用这些版本。

不建议使用已停止支持的版本，如 JDK 8 已经停止公开更新（除非付费订阅）。及时升级到新的 JDK 版本可以获得安全更新和性能改进。

### Java 版本号中的 LTS 是什么意思

LTS（Long-Term Support）是长期支持版本的缩写。LTS 版本会获得 Oracle 至少 8 年的技术支持 和补丁更新。目前的 LTS 版本包括 JDK 8、JDK 11、JDK 17 和 JDK 21。

非 LTS 版本（如 JDK 25）的支持周期较短，通常只有 6 个月。这些版本适合想要使用最新特性的开发者，但对于生产环境，推荐使用 LTS 版本。

### Java 17 还能用吗

Java 17 是一个成熟的 LTS 版本，仍然可以正常使用。Oracle 在 2024 年 10 月更改了 JDK 17 的许可条款，但根据 Oracle 的声明，JDK 17 仍然可以免费用于开发、个人使用和运行 Oracle 许可的产品。

许多企业仍在使用 JDK 17，因为它是一个非常成熟和稳定的版本，社区也有大量的支持资源。如果项目已经在使用 JDK 17，可以继续使用；如果开始新项目，更推荐使用 JDK 21。

## 开发问题

### 如何编译 Java 文件

使用 javac 命令编译 Java 文件。基本用法是：javac MyClass.java。编译后会生成 MyClass.class 字节码文件。

对于项目，通常使用构建工具如 Maven 或 Gradle 来管理编译过程，这些工具会自动处理依赖和编译选项。

### 如何运行 Java 程序

使用 java 命令运行编译后的 Java 程序。基本用法是：java MyClass。注意这里不需要 .class 扩展名。

如果程序打包在 JAR 文件中，使用 java -jar MyApp.jar 运行。

### 如何设置 classpath

在运行时通过 -classpath 或 -cp 选项指定 classpath：java -cp lib/*:out MyClass。在 Windows 上，分隔符是分号（;）；在 Linux/macOS 上，分隔符是冒号（:）。

也可以通过 CLASSPATH 环境变量设置，但这不如命令行指定灵活。

### 什么是 Maven 和 Gradle

Maven 和 Gradle 是 Java 项目的构建工具。它们可以自动下载依赖、编译代码、运行测试、打包发布，大大简化了项目构建过程。

Maven 使用 XML（pom.xml）定义项目配置，Gradle 使用 Groovy 或 Kotlin 脚本（build.gradle）定义构建逻辑。两者都是 Java 项目中广泛使用的构建工具。
