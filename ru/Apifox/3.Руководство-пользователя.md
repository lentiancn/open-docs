# Руководство пользователя

В этом руководстве подробно описаны основные функции и способы работы с Apifox: управление проектами, проектирование интерфейсов, отладка, Mock-данные, автоматическое тестирование и другое.

## Управление проектами

### Создание проекта

В Apifox проект — это основная единица для организации API.

```bash
# Шаги по созданию проекта
1. Нажмите «+» рядом со списком «Проекты» в левой панели
2. Введите название проекта
3. Выберите тип проекта
4. Выберите базовый шаблон (по желанию)
5. Нажмите «Создать»
```

### Настройки проекта

Каждый проект имеет собственные настройки:

- **Основная информация**: название проекта, описание, иконка
- **Переменные окружения**: определение переменных для разных сред (разработка, тестирование, продакшен)
- **Авторизация**: настройка глобального способа авторизации
- **Экспорт/Импорт**: импорт и экспорт данных проекта

### Управление окружениями

Apifox поддерживает работу с несколькими окружениями:

```bash
# Использование переменных окружения
В запросах используйте {{имя_переменной}} для обращения к переменным окружения
Например: {{baseUrl}}/api/users

Переключение окружения: выберите нужное окружение в выпадающем списке в верхней части
```

## Проектирование интерфейсов

### Создание интерфейса

Интерфейс — это конкретное определение API:

```bash
# Шаги по созданию интерфейса
1. В проекте нажмите «+» и выберите «Новый интерфейс»
2. Заполните основную информацию:
   - Название интерфейса: Список пользователей
   - Путь запроса: /api/users
   - Метод запроса: GET
3. Определите параметры запроса
4. Определите структуру ответа
5. Сохраните
```

### Параметры запроса

Поддерживаются следующие типы параметров:

- **Параметры пути**: например, id в /api/users/:id
- **Query-параметры**: параметры в URL вида ?name=test
- **Заголовки**: например, Authorization
- **Тело запроса**: в форматах JSON, формы, XML и других

### Определение ответа

Для каждого интерфейса определяются успешные и ошибочные ответы:

```json
{
  "Успешный ответ 200": {
    "id": 1,
    "name": "Иван Иванов",
    "email": "ivan@example.com"
  },
  "Ошибочный ответ 404": {
    "code": 404,
    "message": "Пользователь не найден"
  }
}
```

### Управление списком интерфейсов

Поддерживается группировка интерфейсов по модулям или функциональности:

- Перетаскивание для изменения порядка
- Пакетные операции (удаление, экспорт и другие)
- Поиск и фильтрация

## Отправка запросов

### Отправка запроса

На странице с деталями интерфейса нажмите кнопку «Отправить»:

```bash
# Базовые шаги отладки
1. Откройте интерфейс
2. Выберите окружение
3. Задайте значения параметров
4. Нажмите «Отправить»
5. Просмотрите ответ
```

### Настройки авторизации

Доступны различные способы авторизации:

- **None**: без авторизации
- **Bearer Token**: JWT-авторизация
- **Basic Auth**: базовая авторизация
- **OAuth 2.0**: авторизация по OAuth2
- **Своя авторизация**: можно настроить собственные заголовки авторизации

### Предварительная обработка запроса

Можно настроить скрипты, которые выполняются перед отправкой запроса:

```javascript
// Пример: добавление временной метки
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// Пример: генерация случайного ID
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### Последующая обработка ответа

Можно настроить скрипты, которые выполняются после получения ответа:

```javascript
// Пример: извлечение данных из ответа
const response = pm.response.json();
console.log('Данные ответа:', response);

// Пример: установка переменной окружения
pm.environment.set('access_token', response.data.token);
```

## API Mock

### Умный Mock

Одна из самых мощных функций Apifox: достаточно определить структуру данных, и система сама сгенерирует Mock-данные:

```bash
# Включение умного Mock
1. Откройте интерфейс
2. Перейдите на вкладку «Mock»
3. Включите «Умный Mock»
4. Сохраните
```

### Правила Mock

Поддерживаются собственные правила генерации Mock-данных:

- `@string`: случайная строка
- `@integer`: случайное целое число
- `@float`: случайное дробное число
- `@boolean`: случайное булево значение
- `@date`: случайная дата
- `@email`: случайный email
- `@url`: случайный URL
- `@guid`: UUID

### Пример

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

Сгенерированные Mock-данные:

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock-сервер

Apifox предоставляет встроенный Mock-сервер:

```bash
# Получение Mock-адреса
На странице с деталями интерфейса скопируйте Mock URL
Например: https://mock.apifox.com/api/users
```

## Автоматическое тестирование

### Создание тестового сценария

```bash
# Шаги по созданию теста
1. Откройте интерфейс
2. Перейдите на вкладку «Тест»
3. Нажмите «Создать тест»
4. Задайте название тестового сценария
5. Добавьте проверки (assertions)
6. Сохраните
```

### Типы проверок

Доступны различные типы проверок:

- **Проверка статус-кода**: проверка кода ответа
- **Проверка JSON**: проверка данных в JSON-ответе
- **Проверка времени ответа**: проверка времени выполнения
- **Проверка заголовков**: проверка заголовков ответа

### Примеры

```javascript
// Проверка статус-кода
pm.test("Код ответа 200", function() {
    pm.response.to.have.status(200);
});

// Проверка JSON
pm.test("Возвращается список пользователей", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// Проверка времени ответа
pm.test("Время ответа меньше 500мс", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### Тестовые наборы

Можно объединить тесты нескольких интерфейсов в тестовые наборы:

1. Создайте тестовый набор
2. Добавьте тестовые сценарии
3. Настройте порядок выполнения
4. Запустите набор

### Генерация отчетов о тестировании

После выполнения тестов можно сгенерировать подробный отчет:

- Статистика прохождения
- Анализ проваленных тестов
- Статистика времени ответа

## Командная работа

### Приглашение участников

```bash
# Приглашение участника в команду
1. Откройте настройки проекта
2. Нажмите «Управление участниками»
3. Нажмите «Пригласить участника»
4. Введите email участника
5. Настройте роль и права
6. Отправьте приглашение
```

### Роли и права

- **Администратор**: полные права на проект
- **Редактор**: может редактировать интерфейсы и запускать тесты
- **Наблюдатель**: только просмотр, без возможности редактирования

### Синхронизация в реальном времени

После входа участников данные проекта синхронизируются в реальном времени:

- Изменения в интерфейсах синхронизируются автоматически
- Результаты тестов синхронизируются автоматически
- Участники команды видят актуальную информацию

### Комментарии

К интерфейсам можно добавлять комментарии:

1. Откройте интерфейс
2. Нажмите «Комментарий»
3. Введите текст комментария
4. Упомяните участника команды через @

## Горячие клавиши

Полезные сочетания клавиш:

- `Ctrl/Cmd + S`: сохранить
- `Ctrl/Cmd + Enter`: отправить запрос
- `Ctrl/Cmd + B`: отформатировать JSON
- `Ctrl/Cmd + /`: закомментировать/раскомментировать

## Рекомендации

1. **Понятные названия**: называйте интерфейсы так, чтобы было ясно, что они делают
2. **Подробные описания**: добавляйте описание для каждого параметра
3. **Логическая группировка**: группируйте интерфейсы по модулям
4. **Используйте окружения**: разделяйте разработку, тестирование и продакшен
5. **Пишите тесты**: создавайте автоматические тесты для важных интерфейсов
6. **Регулярная синхронизация**: поддерживайте данные в актуальном состоянии между локальной версией и облаком
