# 자주 묻는 질문

이 문서에서는 Apache APISIX 사용 중 흔히 발생하는 기술 질문과 답변을 모았습니다.

## 기초 질문

### Apache APISIX란?

Apache APISIX는 Apache Software Foundation의 탑 프로젝트입니다. 동적 라우팅, 동적 업스트림, 동적 인증서, A/B 테스트, 그레이드 배포, 속도 제한, 공격 방지, 모니터링 및 경고 등의 기능을 제공하는 동적이고 실시간이며 고성능의 클라우드 네이티브 API 게이트웨이입니다.

### APISIX와 Nginx의 차이점은?

Nginx는 전통적인 웹 서버 및 역방향 프록시이고, APISIX는 Nginx를 기반으로 구축된 API 게이트웨이입니다. APISIX는 Nginx 위에 완전한 동적 구성 관리, 다양한 플러그인 시스템, 세분화된 트래픽 제어 및 내장된 가시성 기능을 추가합니다.

### APISIX의 성능은?

APISIX는 Nginx의 고성능 특성을 계승하면서 최적화 및 플러그인 메커니즘을 통해 효율적인 처리 능력을 유지합니다. 실제 성능은 하드웨어 구성 및 비즈니스 복잡도에 따라 다르며, 일반적으로 초당수만에서수십만 요청을 처리할 수 있습니다.

## 설치 문제

### Docker 시작 실패 시어떻게 하는지?

1. Docker가 실행 중인지 확인: `docker ps`
2. 컨테이너 로그 확인: `docker-compose logs apisix`
3. 일반적인 문제:
   - 포트 충돌: 9080, 9180, 9443 포트가 사용 중인지 확인
   - etcd 연결 실패: etcd 컨테이너가 실행 중인지 확인
   - 권한 문제: Docker를 실행할 충분한 권한이 있는지 확인

### APISIX 버전 확인 방법은?

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

반환: `Server: APISIX/Version`

## 구성 문제

### 관리 API 포트를 수정하는 방법은?

`apisix.yml` 구성 파일을 편집합니다:

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### HTTPS를 활성화하는 방법은?

1. SSL 인증서 구성:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "인증서 내용",
  "key": "개인 키 내용",
  "snis": ["example.com"]
}'
```

2. 라우터에서 HTTPS 활성화:

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### 라우터 구성이 적용되지 않는다면?

1. 라우터 구성이 올바른지 확인
2. etcd 동기화 확인
3. 플러그인 구문 오류 확인
4. APISIX 오류 로그 확인

## 성능 문제

### 처리량을 향상시키는 방법은?

1. 워커 프로세스 수 증가: `apisix.worker_processes: auto`
2. HTTP/2 활성화: listen 구성에서 `http2: true` 설정
3. 업스트림 연결 풀 최적화: `keepalive_pool.size` 구성
4. 캐시 플러그인 사용: `proxy-cache` 플러그인 활성화

### 메모리 사용량이 높다면?

1. 불필요한 플러그인 로딩 줄이기
2. 업스트림 연결 수 제한
3. 로그 버퍼 크기 조정
4. 대용량 파일 처리에 스트리밍 사용

## 기능 문제

### A/B 테스트를 구현하는 방법은?

`traffic-split` 플러그인을 사용합니다:

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

## 통합 문제

### Kubernetes와 통합하는 방법은?

APISIX Ingress Controller를 사용합니다:

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

## 보안 문제

### APISIX 남용을 방지하는 방법은?

1. **인증 활성화**: 모든 라우터에서 key-auth 또는 기타 인증 플러그인 활성화
2. **관리 API 접근 제한**: 신뢰할 수 있는 IP만 관리 API에 접근 허용
3. **HTTPS 사용**: TLS 암호화 활성화
4. **속도 제한 구성**: 악의적인 요청 방지
5. **접근 로깅 활성화**: 감사을 위해 모든 요청 기록

## 운영 문제

### APISIX를 업그레이드하는 방법은?

**Docker:**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes:**

```bash
helm upgrade apisix apisix/apisix
```

## 일반적인 오류 코드

- **401 Unauthorized**: 인증 실패
- **403 Forbidden**: 접근 권한 없음
- **404 Not Found**: 라우터가 존재하지 않음
- **429 Too Many Requests**: 속도 제한 초과
- **500 Internal Server Error**: 업스트림 서비스 오류
- **502 Bad Gateway**: 업스트림 서비스를 사용할 수 없음
- **503 Service Unavailable**: 서비스를 사용할 수 없음
