# H2 資料庫常見問題

## 安裝問題

### Q: 如何下載 H2 資料庫？

A: 訪問 https://www.h2database.com/html/download.html 下載適合你平台的版本。有 Windows 安裝程式和跨平台 ZIP 包兩種選擇。

### Q: H2 需要什麼 Java 版本？

A: H2 需要 Java 11 或更高版本。

### Q: 如何在 Maven 專案中使用 H2？

A: 在 pom.xml 中添加依賴：
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### Q: 解壓 ZIP 後如何執行 H2？

A: 在命令列中執行：
```bash
java -jar h2.jar
```
這將啟動 H2 Web 控制台。

## 連線問題

### Q: 如何連線 H2 資料庫？

A: 使用 JDBC 連線：
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### Q: H2 的預設使用者名稱和密碼是什麼？

A: 預設使用者名稱是「sa」，密碼為空。

### Q: 如何連線檔案資料庫？

A:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Q: 如何啟動 H2 TCP 伺服器？

A:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### Q: 連接埠被佔用怎麼辦？

A: 可以在連線 URL 中指定連接埠：
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## 資料問題

### Q: 記憶體資料庫的資料會丟失嗎？

A: 是的，記憶體資料庫（jdbc:h2:mem:）在所有連線關閉後資料會丟失。如需持久化，請使用檔案資料庫模式。

### Q: 如何查看資料庫檔案位置？

A: 在連線 URL 中指定路徑：
```java
jdbc:h2:file:/path/to/database
```

### Q: 資料庫檔案損壞怎麼辦？

A: 嘗試使用復原模式：
```java
jdbc:h2:file:./testdb;LOG=0
```
或者從備份復原。

### Q: 如何加密資料庫？

A:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## 效能問題

### Q: 如何提高批量插入效能？

A: 使用批量操作並關閉自動提交：
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Q: 查詢很慢怎麼辦？

A: 
1. 為常用查詢條件建立索引
2. 使用 EXPLAIN 分析查詢計劃
3. 避免 SELECT *，只查詢需要的欄位

### Q: 記憶體使用過高怎麼辦？

A: 調整 JVM 堆記憶體大小，並合理使用連線池。

## SQL 相容性問題

### Q: H2 支援哪些資料庫模式？

A: H2 可以模擬多種資料庫：
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

使用方式：
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### Q: 如何處理自動遞增 ID？

A: H2 支援多種方式：
```sql
-- 方式一：IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- 方式二：SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## 交易問題

### Q: 交易沒有提交會怎樣？

A: 未提交的的交易在連線關閉時會自動回滾。

### Q: 如何處理死結？

A: H2 會自動檢測死結並回滾其中一個交易。建議縮短交易範圍，避免長時間持有鎖。

## 安全問題

### Q: H2 資料庫安全嗎？

A: H2 適合開發和測試環境。對於生產環境，建議：
1. 使用加密資料庫
2. 使用強密碼
3. 限制網路訪問
4. 使用 SSL 連線

### Q: 如何防止 SQL 注入？

A: 使用 PreparedStatement 而不是字串拼接：
```java
// 正確
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// 錯誤
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## 工具問題

### Q: 如何使用 H2 控制台？

A: 執行 `java -jar h2.jar -web` 然後訪問 http://localhost:8082

### Q: 如何使用 H2 Shell？

A: 執行 `java -jar h2.jar -shell` 或 `java -jar h2.jar -tool`

### Q: 如何查看 H2 版本？

A:
```bash
java -jar h2.jar -version
```
或者在 SQL 中：`SELECT H2VERSION();`

## 遷移問題

### Q: 如何從 MySQL 遷移到 H2？

A:
1. 設定 H2 相容模式：`jdbc:h2:file:db;MODE=MySQL`
2. 匯出 MySQL 資料為 SQL 腳本
3. 在 H2 中執行腳本

### Q: H2 版本升級注意什麼？

A: 從 H2 1.x 升級到 2.x 需要特別注意，請參考官方遷移指南：
https://www.h2database.com/html/migration-to-v2.html
