# Häufig gestellte Fragen zu MySQL

## Installationsprobleme

### Q: Was tun, wenn der MySQL-Dienst nicht startet?

A: Überprüfen Sie Folgendes:
1. Bestätigen Sie, dass MySQL korrekt installiert ist
2. Überprüfen Sie Fehlerprotokolle: `mysqld --console`
3. Verifizieren Sie, dass der Port nicht belegt ist: `netstets -an | grep 3306`
4. Stellen Sie sicher, dass die Berechtigungen des Datenverzeichnisses korrekt sind
5. Versuchen Sie, das Datenverzeichnis neu zu initialisieren

### Q: Wie setze ich das root-Passwort zurück, wenn ich es vergessen habe?

A: 
1. MySQL-Dienst stoppen
2. Mit Überspringen der Berechtigungstabellen starten: `mysqld --skip-grant-tables`
3. Bei MySQL anmelden
4. Passwort zurücksetzen: `UPDATE mysql.user SET Password=PASSWORD('neues_passwort') WHERE User='root';`
5. Berechtigungen aktualisieren: `FLUSH PRIVILEGES;`
6. MySQL-Dienst neu starten

### Q: Wie ändere ich die MySQL-Portnummer?

A: Ändern Sie die Konfigurationsdatei my.cnf oder my.ini:
```ini
[mysqld]
port=3307
```
Starten Sie dann den MySQL-Dienst neu.

## Verbindungsprobleme

### Q: Was tun, wenn die Remote-Verbindung abgelehnt wird?

A: 
1. Bestätigen Sie, dass MySQL Remote-Verbindungen erlaubt (überprüfen Sie den Host des Benutzers in der Benutzertabelle)
2. Überprüfen Sie, ob die Firewall den Port erlaubt
3. Stellen Sie sicher, dass bind-address nicht 127.0.0.1 in der MySQL-Konfiguration ist
4. Überprüfen Sie SELinux- oder AppArmor-Einstellungen

### Q: Wie handle ich Verbindungs-Timeouts?

A: 
1. Netzwerkverbindung überprüfen
2. wait_timeout-Konfiguration erhöhen
3. Persistente Verbindungen oder Verbindungspools verwenden
4. Serverlast überprüfen

## Datenbankoperationsprobleme

### Q: Wie behandle ich Zeichensatz-Probleme?

A: 
1. Zeichensatz bei Datenbankerstellung angeben: `CREATE DATABASE db CHARACTER SET utf8mb4;`
2. Zeichensatz bei Verbindung angeben: `SET NAMES utf8mb4;`
3. Standard-Zeichensatz in Konfiguration setzen:
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q: Wie behandle ich Import/Export-Zeichensatz-Probleme?

A: 
1. Zeichensatz beim Export angeben: `mysqldump --default-character-set=utf8mb4`
2. Zeichensatz beim Import angeben: `mysql --default-character-set=utf8mb4`

### Q: Ist es normal, dass automatisch incrementierte Primärschlüssel nicht sequentiell sind?

A: Ja, das ist normal. Gründe sind:
- Fehlgeschlagene Einfügungen verbrauchen IDs
- Transaktions-Rollbacks geben verwendete IDs nicht zurück
- Stapeleinfügungen können Sprünge verursachen
- Löschen von Datensätzen beeinflusst den Auto-Inkrement-Zähler nicht

## Leistungsprobleme

### Q: Was tun, wenn Abfragen langsam sind?

A: 
1. EXPLAIN verwenden, um Abfrageausführungspläne zu analysieren
2. Indizes für Spalten in WHERE, JOIN, ORDER BY erstellen
3. Vollständige Tabellenscans vermeiden
4. Abfragen optimieren, SELECT * vermeiden
5. Abfrage-Cache in Betracht ziehen (in MySQL 8.0 entfernt)
6. Tabellen oder Datenbanken aufteilen, um Druck zu reduzieren

### Q: Wie sehe ich langsame Abfragen?

A: 
1. Langsame Abfrage-Protokollierung aktivieren:
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. Speicherort des langsamen Abfrage-Protokolls anzeigen:
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q: Wie optimiere ich hohe Speichernutzung?

A: 
1. innodb_buffer_pool_size anpassen (70% des verfügbaren Speichers empfohlen)
2. Abfragen optimieren, um unnötige Scans zu reduzieren
3. Anzahl der Verbindungen begrenzen
4. Unnötige Caches bereinigen

## Sicherungs- und Wiederherstellungsprobleme

### Q: Was tun, wenn die Sicherungsdatei zu groß ist?

A: 
1. Komprimierte Sicherung verwenden: `mysqldump | gzip > backup.sql.gz`
2. Datenbanken/Tabellen separat sichern
3. xtrabackup für physische Sicherungen verwenden
4. Regelmäßig historische Daten bereinigen

### Q: Wie implementiere ich inkrementelle Sicherung?

A: 
1. Binärprotokoll aktivieren: `log-bin=mysql-bin`
2. Binärprotokolle regelmäßig sichern
3. mysqlbinlog-Tool zum Wiederherstellen inkrementeller Daten verwenden

## Replikationsprobleme

### Q: Was tun, wenn die Master-Slave-Replikationsverzögerung zu hoch ist?

A: 
1. Netzwerklatenz überprüfen
2. slave_parallel_workers anpassen, um parallele Replikation zu erhöhen
3. Sicherstellen, dass Slave-Hardwareleistung ausreichend ist
4. Master-Schreibgeschwindigkeit optimieren
5. Replikationsfilterregeln überprüfen

### Q: Wie behandle ich Master-Slave-Dateninkonsistenz?

A: 
1. Slave neu synchronisieren
2. pt-table-checksum verwenden, um Inkonsistenz zu überprüfen
3. pt-table-sync verwenden, um Daten zu reparieren
4. Master-Slave-Replikation neu konfigurieren

## Sicherheitsprobleme

### Q: Wie verstärke ich die MySQL-Sicherheit?

A: 
1. Starke Passwortrichtlinien festlegen
2. Remote-Anmeldung für root-Benutzer einschränken
3. Anonyme Konten löschen
4. Testdatenbank löschen
5. MySQL-Version regelmäßig aktualisieren
6. SSL/TLS-verschlüsselte Verbindungen aktivieren
7. Firewall-Regeln konfigurieren
8. Prinzip der geringsten Privilegien befolgen

### Q: Wie verhindere ich SQL-Injection?

A: 
1. Vorbereitete Anweisungen (Prepared Statements) verwenden
2. Benutzereingaben validieren und filtern
3. Datenbankbenutzerberechtigungen einschränken
4. Dynamische SQL-Verkettung vermeiden
5. Gespeicherte Prozeduren verwenden
