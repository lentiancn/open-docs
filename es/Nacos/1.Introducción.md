# Introducción a Nacos

## ¿Qué es Nacos?

Nacos /nɑ:kəʊs/ es el acrónimo de Dynamic Naming and Configuration Service, una plataforma de fácil uso para el descubrimiento dinámico de servicios, gestión de configuración y gestión de servicios para construir aplicaciones nativas de la nube.

Nacos se compromete a ayudarte a descubrir, configurar y administrar microservicios y aplicaciones de agente de IA. Proporciona un conjunto de funciones simples y útiles que te permiten realizar el descubrimiento dinámico de servicios, configuración de servicios, metadatos de servicios y gestión de tráfico.

Nacos facilita y acelera la construcción, entrega y gestión de tu plataforma de microservicios. Es la infraestructura que soporta una arquitectura de aplicación moderna centrada en servicios con un enfoque de microservicios o nativo de la nube.

Nacos supports discovering, configuring, and managing casi todos los tipos de servicios:

- Kubernetes Service
- gRPC
- Dubbo RPC Service
- Spring Cloud RESTful Service
- Model Context Protocol

## Funciones Principales

### Descubrimiento de Servicios y Verificación de Salud

Nacos soporta el descubrimiento de servicios basado en DNS y RPC. Después de que un proveedor de servicios registra un servicio usando SDK nativo, OpenAPI o un agente dedicado, los consumidores pueden descubrir el servicio usando DNS o HTTP API.

Nacos proporciona verificaciones de salud en tiempo real para evitar que los servicios envíen solicitudes a hosts o instancias de servicio no saludables. Nacos soporta verificaciones de salud en la capa de transporte (PING o TCP) y en la capa de aplicación (como HTTP, MySQL, protocolos personalizados). Para verificaciones de salud en entornos de nube complejos y topologías de red (como VPC, Edge Services), Nacos proporciona tanto el modo de informe de agente como la detección activa del servidor. Nacos también proporciona un panel de verificación de salud unificado para ayudarte a gestionar la disponibilidad y el tráfico de los servicios según su estado de salud.

### Servicio de Configuración Dinámica

El servicio de configuración dinámica te permite gestionar la configuración de todas las aplicaciones y servicios de manera centralizada, externalizada y dinámica en todos los entornos. La configuración dinámica elimina la necesidad de reimplementar aplicaciones y servicios cuando se actualiza la configuración, haciendo que la gestión de configuración sea más eficiente y ágil.

La gestión centralizada de la configuración facilita lograr servicios sin estado y la expansión elástica de instancias de servicio bajo demanda. Nacos proporciona una interfaz de usuario fácil de usar para ayudarte a gestionar todas las configuraciones de tus aplicaciones y servicios. Nacos también incluye funciones listas para usar como seguimiento de versiones de configuración, lanzamiento canario, reversión con un clic y seguimiento del estado de actualización de configuración del cliente, asegurando la seguridad y controlando los riesgos de cambios de configuración en producción.

### Servicio DNS Dinámico

El servicio DNS dinámico soporta enrutamiento ponderado, facilitando la implementación de balanceo de carga de nivel medio, políticas de enrutamiento flexibles, control de tráfico y servicios simples de resolución DNS dentro de tu centro de datos. El servicio DNS dinámico también facilita la implementación del descubrimiento de servicios basado en DNS para ayudarte a eliminar el acoplamiento a APIs privadas de descubrimiento de servicios de proveedores.

Nacos proporciona APIs DNS simples para ayudarte a gestionar los nombres de dominio asociados a los servicios y las listas de IP:PORT disponibles.

### Gestión de Servicios y Metadatos

Nacos te permite gestionar todos los servicios y metadatos desde la perspectiva de un constructor de plataformas de microservicios. Esto incluye gestionar la descripción del servicio, ciclo de vida, análisis de dependencias estáticas, estado de salud, gestión de tráfico, políticas de enrutamiento y seguridad, SLA y métricas clave.

### Gestión y Transformación de Servicios MCP

Nacos te permite transformar rápidamente APIs de microservicios existentes en servicios MCP mediante declaraciones simples con cero cambios de código. Además, usando Spring AI Alibaba, nacos-mcp-wrapper-python y Nacos-MCP-Router, los servicios MCP pueden registrarse y descubrirse automáticamente, permitiendo usar servicios MCP como microservicios. Desde la plataforma de servicios MCP, Nacos también gestiona descripciones de servicios MCP, herramientas, prompts y gestión de versiones.

## Ventajas del Producto

### Fácil de Usar

Nacos ha sido refinado con retroalimentación de decenas de miles de usuarios, proporcionando funciones unificadas de descubrimiento de servicios y gestión de configuración. A través de una interfaz web intuitiva y APIs concisas, aporta operaciones convenientes de registro, descubrimiento y actualización de configuración para desarrolladores y personal de operaciones en entornos nativos de la nube. Como solución integral para la gestión dinámica de configuración y descubrimiento de servicios para aplicaciones de IA y nativas de la nube, ofrece más de veinte características listas para usar centradas en servicios de IA con una interfaz de control ligera y fácil de usar para construir aplicaciones de IA rápidamente.

### Rico en Funciones

Nacos proporciona un conjunto completo de funciones que incluyen descubrimiento de servicios, gestión de configuración, servicio DNS dinámico, gestión de metadatos de servicios, gestión de tráfico, monitoreo de servicios y gobernanza de servicios. Te ayuda a construir, entregar y gestionar microservicios más fácilmente en la era nativa de la nube. Soporta escenarios de AI Registry incluyendo Prompt, MCP y A2A. Con soporte directo o extensiones simples, puede soportar una gran cantidad de escenarios útiles de internet y aplicaciones de IA para programación de tráfico y gobernanza de servicios, incluyendo limitación de velocidad, planificación de promociones y despliegue multi-región activo-activo.

### Rendimiento Extremo

Nacos, pulido a través de los escenarios de escalamiento extremo del evento de compras Double 11 de Alibaba, proporciona capacidades de alto rendimiento de registro y descubrimiento de servicios junto con respuestas de actualización de configuración de baja latencia, asegurando una operación eficiente y estable en sistemas distribuidos a gran escala.

### Capacidad Masiva

Nacido de la escala de millones de instancias de Alibaba, Nacos soporta la gestión masiva de servicios y configuración, satisfaciendo las necesidades de sistemas distribuidos grandes para alta concurrencia y alta disponibilidad.

### Estable y Confiable

Derivado de un producto interno de Alibaba con más de diez años de validación en producción, soporta escenarios a gran escala con millones de servicios y es un producto de código abierto con SLA de nivel empresarial utilizado por decenas de millones de usuarios. Nacos utiliza protocolos de sincronización desarrollados internamente combinados con el protocolo Raft ampliamente utilizado en el ecosistema para garantizar la alta disponibilidad de los servicios y la estabilidad de los datos, garantizando la operación estable de alta disponibilidad del sistema Double 11 de Alibaba.

### Ecosistema Abierto

Nacos tiene una comunidad de código abierto activa, integración extensa del ecosistema e innovación continua. Es altamente compatible con marcos de código abierto populares como Spring Cloud y Dubbo, y también proporciona capacidades de plugins ricos para ayudar a los usuarios a personalizar sus sistemas de microservicios nativos de la nube únicos en la era nativa de la nube. Soporta sin problemas Kubernetes, Spring Cloud, Spring AI y Dify, haciendo el despliegue y operación más fáciles en nubes públicas principales (como Alibaba Cloud, Huawei Cloud, Tencent Cloud, Azure y AWS) con soporte multi-inquilino y multi-entorno.

## Filosofía de Diseño

Nacos aims to help users in la era de la IA, en privado cloud, nube híbrida o público cloud entornos, para mejor construir, entregar y gestionar sus plataformas de gestión de IA, reutilizar y combinar servicios de negocio más rápido, y entregar valor de innovación comercial más rápido. La filosofía de diseño de Nacos se posiciona como:

### Fácil de Usar

Fácil de usar es una filosofía central de Nacos. Simplifica el registro de servicios, descubrimiento y gestión de configuración a través de una interfaz de usuario amigable y APIs concisas. Los desarrolladores pueden integrar fácilmente Nacos en sus aplicaciones sin invertir tiempo significativo en configuración y aprendizaje complejos.

### Orientado a Estándares

Nacos sigue una filosofía de diseño orientada a estándares, adheriéndose a las mejores prácticas y protocolos estándar de desarrollo de aplicaciones nativas de la nube para asegurar que sus funciones de descubrimiento de servicios y gestión de configuración se integren perfectamente con una amplia gama de pilas tecnológicas y plataformas.

### Alta Disponibilidad

Para satisfacer los requisitos de aplicaciones empresariales de alta disponibilidad, Nacos implementa el modo clúster, asegurando que las funciones de descubrimiento de servicios y gestión de configuración no se vean afectadas cuando ocurran fallos de nodos. El modo clúster también significa que Nacos puede escalar horizontalmente agregando nodos, mejorando el rendimiento general del sistema y la capacidad de carga.

### Fácil de Extender

Nacos también se enfoca en fácil extensión, usando diseño modular para que cada componente pueda extenderse o reemplazarse independientemente. Esto también proporciona conveniencia para los contribuyentes de la comunidad para desarrollar nuevas funciones o mejorar las existentes para requisitos específicos, promoviendo aún más el ecosistema de Nacos.

A través de la implementación de estas filosofías de diseño, Nacos proporciona a los usuarios una plataforma poderosa y flexible para soportar necesidades de negocio siempre cambiantes, acelerar la innovación de negocio y la transformación digital, ayudando finalmente a los usuarios a ganar una ventaja competitiva en el mercado.

## Modos de Despliegue

### Modo Autónomo

El modo autónomo, también conocido como modo de instancia única, posee todas las características y funciones de Nacos con implementación extremadamente fácil y inicio rápido. Sin embargo, no puede formar un clúster con otros nodos y no puede proporcionar alta disponibilidad cuando ocurren fallos de nodos o red. El modo autónomo puede usar una base de datos incorporada Derby (por defecto) o base de datos externa para almacenamiento. El modo autónomo es principalmente adecuado para ingenieros que construyen entornos Nacos localmente o en entornos de prueba para desarrollo, depuración y prueba. También puede manejar algunos escenarios de negocio con bajos requisitos de estabilidad y disponibilidad.

### Modo Clúster

El modo clúster construye un clúster Nacos de alta disponibilidad combinando múltiples nodos Nacos a través del protocolo Distro desarrollado internamente y el protocolo Raft. Los datos se sincronizan entre todos los nodos del clúster para garantizar la consistencia de los datos. El modo clúster tiene ventajas de alta disponibilidad, alta escalabilidad y alta concurrencia, asegurando que las operaciones de negocio no se vean afectadas cuando ocurren fallos. El modo clúster usa base de datos externa para almacenamiento por defecto pero también puede usar base de datos incorporada. Este modo es principalmente adecuado para entornos de producción y es el método de despliegue más recomendado por la comunidad.

## Casos de Uso

Nacos es adecuado para varios escenarios de aplicación:

- Registro y descubrimiento de servicios en arquitecturas de microservicios
- Gestión de configuración en sistemas distribuidos
- Gestión de configuración multi-entorno (desarrollo, prueba, producción)
- Gestión y enrutamiento de tráfico de servicios
- Infraestructura para aplicaciones nativas de la nube
- Gestión de servicios de modelo para aplicaciones de IA (MCP)
