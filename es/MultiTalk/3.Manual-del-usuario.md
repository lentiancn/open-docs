# Manual del usuario

Este manual proporciona instrucciones detalladas sobre el uso de MultiTalk, incluyendo operaciones basicas, configuracion de parámetros y técnicas avanzadas.

## Uso basico

### Preparacion de archivos de entrada

MultiTalk requiere preparar formatos de entrada especificos. El proyecto usa archivos JSON para definir parámetros de generación, incluyendo rutas de audio, rutas de imágenes y prompts de texto.

Para la generación de una sola persona, el formato JSON de entrada requiere especificar la ruta del archivo de audio, la ruta de la imagen de referencia, el nombre de guardado del video y otros parámetros. Los archivos de ejemplo se encuentran en el directorio examples, como single_example_1.json.

Para la generación conversacional multipersonal, el JSON de entrada necesita contener múltiples flujos de audio y definiciones de region de persona correspondientes. Los archivos de ejemplo se encuentran en el directorio examples, como multitalk_example_2.json.

### Ejecucion de generación basica

Despues de completar la instalacion y configuracion, ingrese al directorio del proyecto y ejecute el siguiente comando para la generación de video:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### Descripcion de parámetros

--ckpt_dir especifica la ruta del directorio de pesos del modelo. --wav2vec_dir especifica la ruta del directorio del codificador de audio. --input_json especifica la ruta del archivo JSON de parámetros de entrada. --sample_steps especifica el numero de pasos de muestreo; valores mas altos producen mejor calidad pero velocidad mas lenta. --mode especifica el modo de generación, streaming para videos largos, clip para videos cortos. --use_teacache habilita la aceleracion TeaCache. --save_file especifica el nombre de guardado del archivo de salida.

## Tecnicas de uso avanzado

### Ajuste de calidad de generación

Los pasos de muestreo son parámetros clave que afectan la calidad de generación. Los 40 pasos predeterminados proporcionan buenos resultados. Para una generación mas rapida, los pasos se pueden reducir a 10, lo cual tiene un impacto minimo en la sincronización de labios pero afecta la fluidez del movimiento y la calidad visual. Aumentar los pasos mejora la representacion de detalles.

La fuerza de sincronización de labios se controla mediante el coeficiente de guia de audio. Los valores de Audio CFG entrenamiento 3 y 5 funcionan mejor. Aumentar este valor logra mejor sincronización de labios.

### Generacion de videos largos

MultiTalk soporta la generación de videos de hasta 15 segundos. Use el modo streaming para generar clips de video mas largos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

Al generar videos muy largos, puede haber diferencias de tono de color entrenamiento segmentos. Establecer Audio CFG en 3 ayuda a mantener la consistencia del color.

### Operacion con VRAM bajo

Si el VRAM es limitado, puede usar los siguientes parámetros para reducir el uso de memoria:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

Agregar --num_persistent_param_in_dit 0 permite la generación de video 480P en una sola RTX 4090.

### Habilitacion de aceleracion

El proyecto soporta múltiples métodos de aceleracion para mejorar la velocidad de generación.

La aceleracion TeaCache puede mejorar significativamente la velocidad mientras mantiene la calidad. Los valores recomendados de --teacache_thresh estan entrenamiento 0.2 y 0.5:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### Uso de aceleracion LoRA

Usar LoRA puede reducir significativamente los pasos de muestreo. FusioniX LoRA requiere solo 8 pasos, mientras que lightx2v LoRA requiere solo 4 pasos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Uso de la interfaz Gradio

El proyecto proporciona una interfaz web Gradio para uso interactivo. Ingrese al directorio del proyecto y ejecute:

```bash
python app.py
```

O use la version de bajo VRAM:

```bash
python app.py --num_persistent_param_in_dit 0
```

## Integracion de sintesis de voz TTS

MultiTalk soporta entrada de texto directa. A traves de tecnologia TTS integrada, genera voz a partir de texto y luego crea videos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## Configuracion de múltiples GPUs

La generación de videos de alta resolucion 720P requiere múltiples GPUs. Use el siguiente comando para configurar múltiples GPUs:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## Uso de ComfyUI

El proyecto tambien soporta integracion en flujos de trabajo de ComfyUI. Necesita instalar la extension ComfyUI-WanVideoWrapper, luego cargar el modelo MultiTalk para la orquestacion del flujo de trabajo.
