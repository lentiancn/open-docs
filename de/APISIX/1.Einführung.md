# Einführung zu Apache APISIX

## Was ist Apache APISIX?

Apache APISIX ist ein Cloud-nativer API-Gateway und AI-Gateway, entwickelt von API7.ai und der Apache Software Foundation gespendet. Es ist ein Top-Level-Projekt der Apache Software Foundation.

APISIX fungiert als "Verkehrsmanager" zwischen Clients und Backend-Diensten. Es empfängt alle Anfragen von Clients und entscheidet basierend auf vordefinierten Regeln, wie diese bearbeitet werden sollen.

## Hauptmerkmale

### 1. Volldynamische Fähigkeiten

APISIX' größtes Merkmal ist das **dynamische Aktualisieren von Konfigurationen ohne Neustart**. Routen, Upstreams und Plugins können jederzeit hinzugefügt, geändert oder gelöscht werden.

- Dynamisches Routing
- Dynamische Upstreams
- Dynamische Zertifikate
- Hot-Pluggable Plugins

### 2. Verkehrsmanagement

- Lastausgleich (Round-Robin, Gewichtet, Consistent Hashing)
- Canary Releases
- A/B-Tests
- Verkehrsteilung
- Circuit Breaker

### 3. Sicherheit

- API-Schlüssel-Authentifizierung
- JWT-Authentifizierung
- OAuth 2.0
- IP-Whitelist/Blacklist
- Ratenbegrenzung

### 4. Observabilität

- Prometheus-Metriken
- Verteilte Traces (Zipkin, SkyWalking)
- Log-Export (Kafka, Elasticsearch)

### 5. Multi-Protokoll

- HTTP/HTTPS
- gRPC
- Dubbo
- MQTT
- WebSocket

## Architektur

- Datenebene: Nginx + Lua
- Kontrollebene: etcd
- Verwaltung: REST Admin API + Dashboard

## Anwendungsfälle

- API-Gateway für Microservices
- Kubernetes Ingress
- API-Schutz
- AI-Workloads
