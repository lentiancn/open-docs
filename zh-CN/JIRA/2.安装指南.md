# 安装指南

Jira 提供多种安装方式，以满足不同规模和技术能力团队的需求。本指南将详细介绍云端版本和 Data Center 本地版本的安装步骤。

## 云端版本安装

云端版本是开始使用 Jira 最快捷的方式，无需管理任何基础设施。

### 注册账号

首先访问 Jira 官方网站，点击免费获取按钮。在注册页面填写工作电子邮件地址，创建账户。Atlassian 会自动识别团队成员并促进协作。

### 选择计划

云端版本提供四个定价层次：

**Free 计划** 永久免费，最多支持10名用户。包含无限目标和项目、待办事项列表和看板视图、每月100次自动化规则、2GB存储空间。

**Standard 计划** 每用户每月7.91美元。包含用户角色和权限、外部协作、多区域数据驻留、每月1700次自动化规则、250GB存储空间。

**Premium 计划** 每用户每月14.54美元。包含跨团队规划和依赖关系管理、可自定义批准流程、无限存储、全天候支持、99.9%正常运行时间 SLA。

**Enterprise 计划** 面向大型企业。包含高级分析和企业级安全性、多个站点支持、无限自动化规则、99.95%正常运行时间 SLA。

### 创建站点

完成注册后，系统会引导您创建第一个 Jira 站点。您可以为单位或个人创建站点，并选择适合您团队的项目模板。

### 添加用户

创建站点后，可以通过管理员面板添加团队成员。用户可以是免费用户或付费用户，具体取决于您选择的计划。

## Data Center 本地部署

对于需要严格数据控制或拥有特殊安全要求的企业，可以选择 Data Center 本地部署版本。

### 系统要求

Data Center 版本需要满足以下基本要求：

操作系统支持 Linux（Red Hat Enterprise Linux、Ubuntu Server、CentOS 等）、Windows Server 等主流服务器操作系统。

数据库支持 PostgreSQL、Oracle、Microsoft SQL Server 等企业级数据库。

内存建议至少 8GB RAM，生产环境建议 16GB 或更多。

磁盘空间取决于数据量，建议至少 100GB 可用空间。

### 安装步骤

首先从 Atlassian 官方下载 Data Center 安装包。运行安装程序，按照向导选择安装目录和数据库连接配置。完成安装后，通过浏览器访问管理界面进行初始配置。

### 数据迁移

如果您从旧版本迁移，需要使用 Jira 提供的导入导出工具进行数据迁移。建议在迁移前备份所有数据，并在测试环境中验证迁移结果的完整性。

## 移动端应用

Jira 提供 iOS 和 Android 移动应用，让您可以随时随地查看和更新任务。移动应用支持任务查看、评论创建、状态更新等功能，适合需要频繁移动办公的团队成员使用。

## 初始配置

无论选择哪种部署方式，初始配置都包括以下步骤：

设置项目结构，创建适合团队的项目和面板。配置工作流，定义任务状态和转换规则。添加团队成员并设置权限。集成其他工具，如 Slack、GitHub 等。建立自动化规则，减少手动操作。
