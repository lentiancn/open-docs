# Mockito 簡介

## 什麼是 Mockito

Mockito 是一個功能強大的 Java 單元測試模擬（Mocking）框架，專為測試驅動開發（TDD）和行為驅動開發（BDD）而設計。它允許開發者建立模擬物件，以受控的方式模擬真實物件的行為，從而實現獨立的單元測試。

## 為什麼選擇 Mockito

### 簡潔的 API 設計

Mockito 提供了直觀易用的 API，學習曲線平緩。開發者可以快速上手，用簡潔的程式碼編寫清晰的測試用例。與其他複雜的模擬框架相比，Mockito 的語法更加自然，閱讀測試程式碼就像閱讀普通業務程式碼一樣簡單。

### 強大的功能支援

Mockito 支援多種高級特性，包括方法呼叫驗證、返回值設定、異常拋出模擬、參數匹配、呼叫順序驗證等。這些功能能夠滿足各種複雜的測試場景需求，幫助開發者全面覆蓋程式碼邏輯。

### 優秀的可讀性

使用 Mockito 編寫的測試程式碼具有良好的可讀性。當測試失敗時，Mockito 能夠提供清晰的驗證錯誤訊息，幫助開發者快速定位問題所在。這種友善的錯誤提示大大提升了除錯效率。

### 活躍的社群支援

Mockito 是 Java 生態中最流行的 Mock 框架，擁有龐大的用戶群體和活躍的社群。框架持續更新維護，文件完善豐富，遇到問題能夠很容易找到解決方案。Stack Overflow 上 Mockito 獲得了最高的投票數，被公認為 Java 最佳 Mock 框架。

## 核心概念

### Mock 物件

Mock 物件是真實物件的替身，在測試期間用來替代真實物件。透過建立 Mock 物件，可以隔離外部依賴，確保測試只關注被測試單元本身的邏輯，而不受外部系統的影響。

### Stubbing（打樁）

Stubbing 用於設定 Mock 物件在特定方法呼叫時的回傳值或異常行為。透過 when().thenReturn() 語法，可以精確控制模擬物件的方法回應，使測試能夠覆蓋各種場景。

### Verification（驗證）

Verification 用於確認測試程式碼與 Mock 物件之間的互動是否按預期發生。verify() 方法可以檢查方法是否被呼叫、呼叫次數、呼叫參數等，確保被測試程式碼正確地使用了依賴物件。

### Spy（監視）

Spy 允許對真實物件進行部分模擬。與完全模擬的 Mock 物件不同，Spy 會保留真實物件的大部分行為，只對特定方法進行模擬。這在需要測試大部分真實邏輯同時又要模擬少數方法時非常有用。

## 典型應用場景

### 隔離外部依賴

當被測試程式碼依賴資料庫、網路服務、檔案系統等外部系統時，使用 Mockito 可以將這些外部依賴模擬掉，使測試能夠在沒有真實外部系統的情況下執行。這不僅加快了測試速度，還消除了外部系統不穩定帶來的測試失敗風險。

### 模擬尚未開發完成的模組

在大型專案中，不同模組的開發進度可能不同。使用 Mockito 可以模擬尚未完成的模組，使已完成的模組能夠提前進行測試，避免開發阻塞。

### 模擬複雜物件

某些業務物件的建構非常複雜，包含多個層級的依賴。Mockito 可以輕鬆建立這些複雜物件的模擬版本，專注於測試業務邏輯本身。

## 與 JUnit 的整合

Mockito 通常與 JUnit 測試框架配合使用。透過 JUnit 的 @RunWith(MockitoJUnitRunner.class) 或 @ExtendWith(MockitoExtension.class)，可以自動初始化 @Mock 注解標記的欄位，簡化測試程式碼的編寫。這種組合是 Java 單元測試的最佳實踐。

## 版本相容性

Mockito 5.x 版本需要 JDK 11 或更高版本，支援 Java 8 引入的 lambda 表達式等新特性。Mockito 4.x 系列仍然支援 JDK 8，開發者可以根據專案實際情況選擇合適的版本。

## 總結

Mockito 是 Java 開發者必備的測試工具。它以簡潔優雅的方式解決了單元測試中的依賴隔離問題，讓開發者能夠編寫可靠、可維護的測試程式碼。掌握 Mockito 將顯著提升程式碼品質和開發效率，是每個 Java 工程師都應該具備的技能。
