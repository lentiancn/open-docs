# Nginx 소개

## Nginx란?

Nginx("engine x"로发音)는 HTTP 웹 서버, 리버스 프록시, 캐시, 로드 밸런서, TCP/UDP 프록시 서버 및 메일 프록시 서버입니다.

Nginx는 Igor Sysoev가 처음 작성했으며 2-clause BSD 라이선스로 배포됩니다.

## Nginx의 특징

Nginx는 낮은 리소스 소비로 유연성과 높은 성능으로 유명합니다:

- **세계에서 가장 인기 있는 웹 서버**
- **가장 인기 있는 Docker 이미지 중 하나**
- **Kubernetes용 여러 Ingress 컨트롤러 구동**

### 기본 HTTP 서버 기능

- 정적 및 인덱스 파일 제공
- 자동 인덱싱 기능
- 열린 파일 디스크립터 캐시
- 캐싱을 통한 가속화된 리버스 프록시
- 로드 밸런싱 및 내결함성
- FastCGI, uwsgi, SCGI 및 memcached 서버를 위한 캐싱 지원
- 모듈식 아키텍처
- 필터: gzip 압축, 바이트 범위, 청크 응답, XSLT, SSI 및 이미지 변환
- SSL 및 TLS SNI 지원
- 가중 및 종속성 기반 우선순위를 지원하는 HTTP/2 지원
- HTTP/3 지원

### 기타 HTTP 서버 기능

- 이름 및 IP 기반 가상 서버
- keep-alive 및 파이프라인 연결 지원
- 액세스 로그 형식, 버퍼링된 로그 기록, 빠른 로그 순환 및 syslog
- 3xx-5xx 오류 리디렉션
- 정규식을 사용한 URI 변경을 위한 재작성 모듈
- 클라이언트 IP 주소, 비밀번호(HTTP 기본 인증) 및 하위 요청 결과에 따른 액세스 제어
- HTTP referer 검증
- PUT, DELETE, MKCOL, COPY 및 MOVE 메서드
- FLV 및 MP4 스트리밍
- 응답 속도 제한
- 한 주소에서의 동시 연결 또는 요청 제한
- IP 기반 지리적 위치
- A/B 테스트
- 요청 미러링
- 내장 Perl
- njs 스크립트 언어

### 메일 프록시 서버 기능

- 외부 HTTP 인증 서버를 사용하여 사용자를 IMAP 또는 POP3 서버로 리디렉션
- 외부 HTTP 인증 서버를 사용한 사용자 인증 및 내부 SMTP 서버로 연결 전달
- 지원 인증 방법: POP3 (USER/PASS, APOP, AUTH LOGIN/PLAIN/CRAM-MD5), IMAP (LOGIN, AUTH LOGIN/PLAIN/CRAM-MD5), SMTP (AUTH LOGIN/PLAIN/CRAM-MD5)
- SSL 지원
- STARTTLS 및 STLS 지원

### TCP/UDP 프록시 서버 기능

- 일반 TCP 및 UDP 프록시
- TCP용 SSL 및 TLS SNI 지원
- 로드 밸런싱 및 내결함성
- 클라이언트 주소에 따른 액세스 제어
- 한 주소에서의 동시 연결 제한
- 액세스 로그 형식, 버퍼링된 로그 기록, 빠른 로그 순환 및 syslog
- IP 기반 지리적 위치
- A/B 테스트
- njs 스크립트 언어

### 아키텍처 및 확장성

- 하나의 마스터 프로세스와 여러 워커 프로세스
- 워커 프로세스는 비특권 사용자 권한으로 실행
- 유연한 구성
- 클라이언트 서비스 중단 없이 실행 파일 재구성 및 업그레이드
- kqueue (FreeBSD 4.1+), epoll (Linux 2.6+), /dev/poll (Solaris 7 11/99+), 이벤트 포트 (Solaris 10), select 및 poll 지원
- 다양한 kqueue 기능 지원
- 다양한 epoll 기능 지원
- sendfile, sendfile64 및 sendfilev 지원
- File AIO 지원
- DIRECTIO 지원
- Accept 필터 및 TCP_DEFER_ACCEPT 지원
- 10,000개의 비활성 HTTP keep-alive 연결은 약 2.5MB 메모리 사용
- 데이터 복사 작업 최소화

## Nginx의 장점

### 높은 성능

Nginx는 이벤트 기반 비동기 비차단 모델을 사용하여 낮은 메모리 사용량으로 높은 동시 연결을 처리할 수 있습니다.

### 낮은 리소스 소비

다른 웹 서버와 비교할 때 Nginx는 동일한 동시성 수준에서 훨씬 적은 메모리와 CPU를 소비합니다.

### 높은 안정성

Nginx의 안정성은 전 세계 수백만 개의 웹 사이트에서 입증되었으며 충돌이 매우 드뭅니다.

### 간단한 구성

Nginx 구성 파일은 간결하고 이해하기 쉽습니다.

### 강력한 확장성

풍부한 모듈을 지원하여 필요에 따라 기능을 활성화하거나 비활성화할 수 있습니다.

### 핫 디플로이먼트

서비스 중단 없이 구성을 다시 로드하고 버전을 업그레이드하세요.

## 사용 사례

Nginx는 다음과 같은 시나리오에 널리 사용됩니다:

- **웹 서버**: 정적 파일용 HTTP 서버로
- **리버스 프록시**: 백엔드 서버용 리버스 프록시로
- **로드 밸런서**: 여러 백엔드 서버용 로드 밸런서로
- **캐시 서버**: CDN 또는 내부 캐시 서버로
- **API 게이트웨이**: 마이크로서비스용 입력 게이트웨이로
- **메일 프록시**: POP3, IMAP 및 SMTP 프록시로
- **TCP/UDP 프록시**: TCP 및 UDP 프로토콜용 프록시로
