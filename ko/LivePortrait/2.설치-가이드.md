# LivePortrait 설치 가이드

이 문서에서는 다양한 운영 체제 및 환경에서 LivePortrait를 설치하는 방법에 대한詳細な 지침을 제공합니다.

## 요구 사항

설치하기 전에 시스템이 다음 기본 요구 사항을 충족하는지 확인하세요:

**운영 체제**: Linux (권장), Windows 10/11, macOS (Apple Silicon 필요)

**Python 버전**: Python 3.10

**하드웨어 요구 사항**: NVIDIA GPU (RTX 4090 또는 기타 CUDA 호환 GPU 권장), 최소 8GB VRAM

**저장 공간**: 모델 가중치 및 런타임 환경에 최소 20GB 권장

## 기본 환경 설정

### CUDA 버전 확인

시스템의 CUDA 버전을 확인하여 적절한 PyTorch 버전을 설치하세요:

```bash
nvcc -V
```

일반적인 CUDA 버전에는 11.1, 11.8, 12.1이 있습니다. 표시된 버전 번호에 따라 해당 PyTorch 설치 명령을 선택하세요.

### PyTorch 설치

CUDA 버전에 따라 적절한 PyTorch 설치 명령을 선택하세요:

**CUDA 11.1의 경우**:

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**CUDA 11.8의 경우**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**CUDA 12.1의 경우**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

휴먼 모드만 실행하면 되면 CUDA 버전 검사를 건너뛰고 최신 버전의 PyTorch를 직접 설치할 수 있습니다.

## 프로젝트 코드 다운로드

Git을 사용하여 LivePortrait 저장소를 클론합니다:

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Python 환경 생성

conda를 사용하여 격리된 Python 환경을 생성하는 것이 좋습니다:

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

또는 venv 사용:

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## 종속성 설치

프로젝트 루트 디렉터리에서 다음을 실행하세요:

```bash
pip install -r requirements.txt
```

## 사전 학습된 모델 다운로드

사전 학습된 모델은 HuggingFace에서 다운로드할 수 있습니다. 네트워크가 HuggingFace에 액세스할 수 있는 경우:

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

HuggingFace에 액세스할 수 없는 경우 미러 사이트를 사용하세요:

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

또는 Google Drive 또는 Baidu Netdisk에서 전체 모델 파일을 다운로드하여 ./pretrained_weights 디렉터리에 넣으세요.

## 설치 확인

설치 후 테스트 스크립트를 실행하여 환경이 올바르게 구성되었는지 확인하세요:

```bash
python inference.py
```

모두 제대로 작동하면 animations 디렉터리에 생성된 출력 파일이 표시됩니다.

## 동물 모델 추가 설치 단계

동물 모드는 Linux 및 Windows 시스템에서만 NVIDIA GPU로 지원됩니다.

### X-Pose 종속성 설치

동물 모드에는 키포인트 감지 프레임워크로 X-Pose 프로젝트가 필요합니다. 먼저 X-Pose 디렉터리로 이동합니다:

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### HuggingFace에서 동물 모델 다운로드

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Windows 관련 참고 사항

Windows 시스템에서 일부 높은 버전의 CUDA(예: 12.4, 12.6)는 호환성 문제를 일으킬 수 있습니다. 더 나은 안정성을 위해 CUDA를 버전 11.8로 다운그레이드하는 것이 좋습니다.

Windows 사용자는 원클릭 인스톨러를 다운로드하여 압축을 푼 후 run_windows.bat을 더블 클릭하여 실행할 수도 있습니다:

```bash
# HuggingFace에서 Windows 원클릭 인스톨러 다운로드
# 주소: https://huggingface.co/cleardusk/LivePortrait-Windows
```

## macOS 관련 참고 사항

### Apple Silicon macOS

Apple Silicon macOS 사용자는 special 종속성 파일을 사용하세요:

```bash
pip install -r requirements_macOS.txt
```

추론을 실행할 때 환경 변수를 설정하세요:

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

참고: Apple Silicon macOS에서 추론을 실행하면 RTX 4090보다 약 20배 느릴 수 있습니다.

### Intel Mac

Intel 기반 Mac은 공식적으로 테스트되지 않았으며 제대로 작동하지 않을 수 있습니다.

### 동물 모드 제한

X-Pose 종속성이 macOS를 지원하지 않으므로 동물 모드는 macOS에서 사용할 수 없습니다. 휴먼 모드만 사용할 수 있습니다.

## 디렉터리 구조 확인

설치 후 디렉터리 구조가 다음을 충족하는지 확인하세요:

```
LivePortrait/
├── pretrained_weights/     # 사전 학습된 모델 가중치 디렉터리
├── assets/                # 샘플 자료 디렉터리
│   ├── examples/
│   │   ├── source/        # 소스 이미지/비디오
│   │   └── driving/       # 구동 비디오
├── src/                   # 소스 코드 디렉터리
├── inference.py          # 추론 스크립트
├── app.py                # Gradio 인터페이스
└── requirements.txt      # 종속성 목록
```

## 일반적인 문제

설치 중에 문제가 발생하면 다음을 확인하세요:

**CUDA 사용 불가**: NVIDIA 드라이버가 올바르게 설치되었는지 확인; nvidia-smi를 실행하여 드라이버 상태 확인

**VRAM 부족**: 더 작은 입력 이미지를 사용하거나 배치 크기를 줄여 보세요

**모델 다운로드 실패**: 네트워크 연결을 확인하거나 국내 미러 사이트를 사용하세요

**종속성 충돌**: 다른 프로젝트와의 충돌을 피하기 위해 새 Python 환경을 사용하는 것이 좋습니다
