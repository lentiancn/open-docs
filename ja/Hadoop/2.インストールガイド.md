# Hadoop インストールガイド

## 環境要件

Hadoopをインストールする前に、システムが基本的なハードウェアとソフトウェア要件を満たしていることを確認してください。Hadoopクラスターは複数のコンピュータで構成，也可以在伪分布式模式下运行单机。異なる展開モードには異なるハードウェア要件がありますが、コアソフトウェアの依存関係は似ています。これらの要件を理解することは、Hadoopクラスターの規模と構成を計画するのに役だち、安装过程中的不必要な問題を避けることができます。

ハードウェア面では、小規模なテストクラスターでは、各マシンに最低4コアCPU、8GBメモリ、500GBハードディスクすることを推奨します。本番環境では、各マシンに8コア以上のCPU、16GB以上のメモリ、1TB以上のハードディスクことを推奨します。HadoopのDataNodeは大量データを保存するため、ハードディスク容量は特に重要です。HadoopはRAID構成またはJBOD（Just a Bunch Of Disks）構成を推奨しています。後者はHadoopがディスクスペースをより柔軟に管理することを可能にします。

オペレーティングシステム面では、HadoopはLinux（Ubuntu、CentOS、Debian、Red Hatなど）とWindows（CygwinまたはWindows Subsystem for Linux経由）をサポートしています。本番環境では、最高のパフォーマンスと安定性のためにLinuxシステムを使用することを強く推奨します。Hadoopはさまざまなクラウドプラットフォームとも很好地統合されており、AWS、Azure、Google Cloudなどのクラウドサービスに展開できます。

JavaはHadoop唯一のランタイム依存関係です。Hadoop 3.xはJava 8以上が必要です。Oracle JDKまたはOpenJDKを推奨しています。Hadoopをインストールする前に、Javaが正しくインストールされ、JAVA_HOME環境変数が設定されていることを確認してください。以下のコマンドでJavaのインストールを確認できます：

```bash
java -version
echo $JAVA_HOME
```

ネットワークはHadoopクラスターが正常に動作するための基盤です。クラスター内のすべてのノードはネットワーク経由で相互に通信できる必要があります（ギガビットイーサネットなどの高速ネットワークを使用することを推奨）。ノード間のSSHパスワードなしログインを設定する必要があります，以便Hadoop可以在节点之间分发任务和传输数据。此外，需要在/etc/hosts文件中配置所有节点的主机名和IP地址映射。

## インストール手順

### スタンドアロンモードのインストール

スタンドアロンモードはHadoop最もシンプルなインストール方法で、すべてのコンポーネントが同じJVMプロセスで動作します。このモードは主に開発とテストに使用され、分散ストレージやコンピューティングは関係ありません。まず、HadoopのバイナリディストリビューションをApache Hadoop公式サイトからダウンロードします：

```bash
# Hadoopをダウンロード的最新版本）
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# 指定されたディレクトリに展開
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# ディレクトリ名を変更（オプション）
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

次に、環境変数を構成します。~/.bashrcまたは/etc/profileファイルを編集し、次を追加します：

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # 実際のJavaパスに応じて調整
```

設定を適用します：

```bash
source ~/.bashrc
```

インストールを確認します：

```bash
hadoop version
```

### 擬似分散モードのインストール

擬似分散モードは、単一マシン上で分散環境をシミュレートし、NameNode、DataNode、ResourceManager、NodeManagerがすべて同じマシンで動作します。このモードは分散機能の開発とテストに適しています。

まず、Hadoopのコアファイルを構成します。Hadoop構成ディレクトリに移動します：

```bash
cd $HADOOP_HOME/etc/hadoop
```

core-site.xmlを構成し、HDFSデフォルトファイルシステムを指定します：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

hdfs-site.xmlを構成し、HDFSレプリケーション係数（擬似分散モードでは1）を指定します：

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

mapred-site.xmlを構成し、MapReduceのリソーススケジューリングフレームワークを指定します：

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapprocess.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

yarn-site.xmlを構成し、YARN関連プロパティを設定します：

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### SSHパスワードなしログインの構成

Hadoopクラスター内のノードは、マスターノードがスレーブノードにパスワードなしでアクセスできるように、SSHパスワードなしログインを構成する必要があります：

```bash
# SSH鍵ペアを生成
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# 公開鍵をauthorized_keysに追加
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# 正しい権限を設定
chmod 0600 ~/.ssh/authorized_keys

# SSHログインをテスト
ssh localhost
```

マルチノードクラスターを使用する場合は、公開鍵をすべてのノードにコピーします：

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### HDFSのフォーマットと起動

初めてHDFSを使用する前に、NameNodeをフォーマットします：

```bash
hdfs namenode -format
```

HDFSとYARNを起動します：

```bash
start-dfs.sh
start-yarn.sh
```

または、start-all.shを使用してすべてのサービスを一度に起動します：

```bash
start-all.sh
```

サービスの起動状態を確認します：

```bash
# Javaプロセスを確認
jps

# 予想されるプロセス：
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

Web UI経由でHadoopサービスにアクセスできます：NameNodeのWeb UIアドレスはhttp://localhost:9870、ResourceManagerのWeb UIアドレスはhttp://localhost:8088です。

## 完全分散クラスターのインストール

### クラスター計画

完全分散モードでは、クラスターを形成するために複数の物理マシンまたは仮想マシンが必要です。インストール前に、合理的なクラスター計画を行います。クラスターの規模（ノード数）を決定し、各ノードの役割を割り当てます（NameNode、Secondary NameNode、DataNode、ResourceManager、NodeManager）。小規模クラスターでは、NameNodeとResourceManagerを同じマシンに展開できます。大規模クラスターでは、パフォーマンス向上のために別々に展開することを推奨します。

ネットワーク計画も重要です。すべてのノードが同一の内部ネットワーク上にあり、低遅延で高帯域幅であることを確認します。/etc/hostsファイルですべてのノードのホスト名とIPアドレスマッピングを構成します：

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### 構成ファイルの変更

完全分散モードでは、より多くの構成ファイルを変更する必要があります。まず、workersファイル（早期バージョンではslavesファイル）を構成し、すべてのDataNodeとNodeManagerノードをリストします：

```
slave1
slave2
slave3
```

core-site.xmlとhdfs-site.xmlの構成は擬似分散モードに似ていますが，更多プロパティを追加する必要があります。NameNode HA（高可用性）構成では、2つのNameNodeを構成します：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

hdfs-site.xmlでNameNode HAを構成します：

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

YARNの構成も似ており、yarn-site.xmlでResourceManager HAを構成できます。

### 構成ファイルの同期

構成済みのHadoopインストールディレクトリをすべてのノードに同期します。scpを使用して個別にコピーできます：

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

または、rsyncなどの分散コピー工具を使用します：

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### クラスターの起動

NameNodeノードでNameNodeをフォーマットします（初回起動時）：

```bash
hdfs namenode -format
```

HDFSを起動します：

```bash
start-dfs.sh
```

YARNを起動します：

```bash
start-yarn.sh
```

すべてのノードでプロセス状態を確認し、服务正常运行を確認します：

```bash
# NameNodeで
jps

# DataNodeで
jps
```

## 確認とテスト

### HDFSのテスト

ディレクトリを作成し、ファイルをアップロードしてHDFSをテストします：

```bash
# ディレクトリを作成
hdfs dfs -mkdir -p /user/test

# ローカルファイルをHDFSにアップロード
hdfs dfs -put localfile.txt /user/test/

# ファイルを確認
hdfs dfs -ls /user/test

# ファイルを読み取り
hdfs dfs -cat /user/test/localfile.txt

# ファイルを削除
hdfs dfs -rm /user/test/localfile.txt
```

### MapReduceのテスト

MapReduceサンプルプログラムを実行してクラスター機能を検証します：

```bash
# WordCountサンプルを実行
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# 結果を確認
hdfs dfs -cat /output/part-r-00000
```

### クラスターの健康診断

Hadoop組み込みのコマンドを使用してクラスター状態を確認します：

```bash
# HDFS状態を確認
hdfs dfsadmin -report

# YARNノード状態を確認
yarn node -list -all

# アプリケーション状態を確認
yarn application -list
```

## よくある問題

### Javaバージョンの互換性问题

Javaバージョン関連のエラーが発生した場合は、インストールされたJavaバージョンがHadoopバージョンと互換性があることを確認してください。Hadoop 3.xはJava 8以上が必要です。以下のコマンドでJavaバージョンを切り替えできます：

```bash
update-alternatives --config java
```

### ディスク容量不足

DataNodeはディスク容量不足により正常に動作しない可能性があります。ディスク使用状況を定期的に監視し、DataNodeに十分な空き容量があることを確認します。hdfs-site.xmlでdfs.datanode.fsdataset.volume.choosing.policyをAvailableSpaceVolumeChoosingPolicyとして構成し、DataNodeが空き容量の多いディスクを優先して使用するようにできます。

### メモリ不足

クラスターノードのメモリが不足している場合、YARNはコンテナを起動できない可能性があります。yarn-site.xmlでメモリ構成を調整します：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
