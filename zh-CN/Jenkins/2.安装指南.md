# Jenkins 安装指南

## 系统要求

在安装 Jenkins 之前，需要确保系统满足以下最低配置要求：

### 最低推荐配置

- 256MB 可用内存
- 1GB 可用磁盘空间（作为 Docker 容器运行则推荐 10GB）

### 小团队推荐配置

- 1GB+ 可用内存
- 50GB+ 可用磁盘空间

### 软件要求

- Java 8 或更高版本（无论是 JRE 还是 JDK 都可以）
- 注意：如果将 Jenkins 作为 Docker 容器运行，则不需要在主机上安装 Java

## 安装平台

Jenkins 支持多种安装方式，本文详细介绍几种常用的安装方法。

## Docker 安装

Docker 是最推荐的 Jenkins 安装方式，因为它提供了隔离的运行环境，简化了配置过程。

### 安装 Docker

首先需要在系统上安装 Docker。访问 Docker 官方网站，根据您的操作系统选择合适的 Docker Community Edition 版本进行安装。

### 下载并运行 Jenkins

在 macOS 和 Linux 上，打开终端窗口，执行以下命令：

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

在 Windows 上，打开命令提示符窗口，执行类似命令（使用 ^ 换行）：

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

参数说明：

- `-u root`：以 root 用户运行容器
- `--rm`：容器停止时自动删除
- `-d`：在后台运行（分离模式）
- `-p 8080:8080`：将容器端口 8080 映射到主机端口 8080
- `-p 50000:50000`：将容器端口 50000 映射到主机端口 50000（用于 JNLP 代理通信）
- `-v jenkins-data:/var/jenkins_home`：持久化 Jenkins 数据
- `-v /var/run/docker.sock:/var/run/docker.sock`：允许 Jenkins 在流水线中运行 Docker 命令
- `jenkinsci/blueocean`：Jenkins 官方镜像，包含 Blue Ocean 界面

### 访问 Jenkins 容器

如果需要进入 Jenkins 容器内部进行操作，可以使用以下命令：

```bash
docker exec -it jenkins-blueocean bash
```

### 查看 Jenkins 日志

可以通过以下命令查看 Jenkins 控制台日志：

```bash
docker logs jenkins-blueocean
```

## WAR 文件安装

Jenkins 的 WAR 文件版本可以安装在任何支持 Java 的操作系统上。

### 安装步骤

1. 从官方网站下载最新的稳定 Jenkins WAR 包
2. 在下载目录中打开终端或命令提示符
3. 运行命令：`java -jar jenkins.war`
4. 浏览器访问 http://localhost:8080，等待解锁页面出现

### 注意事项

- WAR 文件方式不会自动安装 Blue Ocean 功能，需要后续通过插件管理手动安装
- 可以通过 `--httpPort` 参数更改端口，例如：`java -jar jenkins.war --httpPort=9090`

## macOS 安装

### 使用包安装

1. 下载 macOS 版本的 Jenkins 包
2. 双击打开并按照安装向导操作

### 使用 Homebrew 安装

安装最新版本：

```bash
brew install jenkins
```

安装 LTS 版本：

```bash
brew install jenkins-lts
```

## Linux 安装（Debian/Ubuntu）

在基于 Debian 的发行版上，可以通过 apt 包管理器安装 Jenkins：

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

安装完成后，Jenkins 会：

- 设置为开机自启动的守护进程
- 创建 jenkins 用户运行服务
- 输出日志到 /var/log/jenkins/jenkins.log
- 默认监听 8080 端口

## 安装后设置向导

首次启动 Jenkins 时，会进入安装向导。

### 解锁 Jenkins

1. 浏览器访问 http://localhost:8080
2. 解锁 Jenkins 页面出现后，从控制台日志中复制自动生成的密码
3. 将密码粘贴到管理员密码字段，点击继续

### 自定义 Jenkins 插件

在插件安装页面，可以选择：

- 安装建议的插件：安装推荐的一组插件，基于最常见用例
- 选择要安装的插件：手动选择需要安装的插件集

### 创建第一个管理员用户

最后一步是创建管理员用户，需要填写：

- 用户名
- 密码
- 确认密码
- 全名
- 电子邮件地址

完成后，点击"保存并完成"，然后点击"开始使用 Jenkins"进入主界面。

## 验证安装

安装完成后，可以通过以下方式验证：

1. 浏览器访问 http://localhost:8080
2. 使用创建的管理员账户登录
3. 检查系统状态是否正常

## 常见问题

### 端口被占用

如果 8080 端口被其他程序占用，可以在启动时指定其他端口：

- Docker 方式：修改 -p 参数
- WAR 方式：使用 --httpPort 参数

### 内存不足

如果系统内存较小，可以考虑：

- 使用 Docker 方式，通过容器资源限制
- 调整 Jenkins 启动参数中的 JVM 堆内存大小

### 首次启动慢

首次启动时需要下载和安装插件，可能需要较长时间，请耐心等待。
