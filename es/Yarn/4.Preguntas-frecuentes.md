# Yarn FAQ

Common questions and answers about Yarn.

---

## Basic Questions

### What is Yarn?

Yarn is a JavaScript package manager that helps you install, update, configure, and remove package dependencies.

### Yarn vs npm?

- **Speed**: Yarn is faster with parallel installs
- **Determinism**: Yarn's lockfile ensures consistent installs
- **Offline Support**: Yarn has excellent offline caching
- **Workspaces**: Yarn has native workspace support

### Is Yarn free?

Yes, Yarn is completely free and open source (MIT license).

## Installation

### Installation failed?

1. Check Node.js version: `node --version`
2. Clear npm cache: `npm cache clean --force`
3. Use corepack: `corepack enable`

### How to upgrade Yarn?

```bash
corepack prepare yarn@stable --activate
```

---

## Usage

### How to add dependencies?

```bash
yarn add lodash
yarn add -D typescript
```

### How to update?

```bash
yarn up lodash
yarn up --all
```

### Dependencies failed to install?

```bash
yarn cache clean
rm -rf node_modules yarn.lock
yarn install
```

## Workspaces

### What are workspaces?

Workspaces allow you to manage multiple packages in a single repository.

### How to configure?

```json
{
  "workspaces": ["packages/*"]
}
```

## Other

### Should I commit yarn.lock?

**Yes**, always commit yarn.lock to ensure consistent installs.
