# Руководство по установке Mockito

## Требования к окружению

Перед установкой Mockito убедитесь, что ваша среда разработки соответствует следующим требованиям.

Mockito 5.x требует JDK 11 или выше. Если ваш проект должен поддерживать JDK 8, можно использовать серию Mockito 4.x. Рекомендуется использовать последнюю версию Mockito для получения новейших функций и исправлений безопасности.

## Установка через Maven

### Добавление зависимостей

Добавьте зависимость mockito-core в файл pom.xml вашего Maven-проекта. Рекомендуется использовать свойство для определения версии Mockito для лучшего управления версиями.

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Если вам нужно использовать встроенную функцию Mockito для mockирования финальных классов и методов, также нужно добавить зависимость mockito-inline.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### Версии-снимки

Mockito публикует версии-снимки в репозитории Sonatype после каждого изменения кода. Если вам нужно использовать последние функции в разработке, можно добавить адрес репозитория снимков в ваш pom.xml.

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Установка через Gradle

### Добавление зависимостей

Добавьте объявление зависимости в файл build.gradle вашего Gradle-проекта.

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

При использовании файла Kotlin DSL Gradle (build.gradle.kts) синтаксис немного отличается.

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### Использование BOM для управления версиями

Использование плагина Spring Dependency Management или функции BOM в Gradle может упростить управление версиями.

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## Ручная загрузка

Если ваш проект не использует Maven или Gradle, можно загрузить JAR-файлы непосредственно из репозитория Maven Central. Посетите Maven Central и найдите org.mockito, чтобы найти все версии mockito-core и mockito-all (устаревшие).

После загрузки добавьте JAR-файлы в classpath вашего проекта. Обратите внимание, что также необходимо загрузить зависимые библиотеки, такие как Byte Buddy и Objenesis.

## Интеграция со Spring Boot

Spring Boot spring-boot-starter-test уже включает зависимость Mockito. Если ваш проект использует Spring Boot, обычно не нужно добавлять отдельную зависимость Mockito.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Если вам нужно удобнее использовать Mock-объекты в среде Spring, можно использовать аннотацию @MockBean, которая автоматически внедряет макетные объекты в контейнер Spring.

## Интеграция с JUnit 5

### Использование MockitoExtension

JUnit 5 представил модель расширений, и Mockito предоставляет MockitoExtension для поддержки JUnit 5. Дополнительная настройка зависимостей не требуется, так как mockito-core уже содержит необходимую поддержку.

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### Использование MockitoJUnitRunner

Для JUnit 4 можно использовать MockitoJUnitRunner для автоматической инициализации Mock-объектов.

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## Проверка установки

После установки можно проверить, что Mockito правильно настроен, создав простой тестовый класс. Если тест пройдет после выполнения, Mockito установлен правильно.

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito успешно установлен!");
    }
}
```

## Рекомендации по выбору версии

Для новых проектов рекомендуется использовать последнюю версию Mockito 5.x. Если вашему проекту нужна совместимость с JDK 8, доступна серия Mockito 4.x. Всегда используйте последнюю стабильную версию для лучшей производительности и безопасности.
