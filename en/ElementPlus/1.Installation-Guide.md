# Element Plus Installation Guide

> Element Plus is a Vue 3 desktop UI component library made by the Element team
> 
> Updated: February 2026
> 
> Current Version: 2.x (Stable)

---

## 1. Project Overview

Element Plus is a desktop UI component library based on Vue 3, developed and maintained by the Element team. As the Vue 3 upgrade version of Element UI, Element Plus is completely written in Vue 3's Composition API style and fully rewritten in TypeScript, providing a type-safe component development experience.

The first stable release of Element Plus was published on February 7, 2022. After years of iteration, the API has become very stable. Currently, it supports multiple languages including Simplified Chinese and English, and has an active community ecosystem.

## 3. Installation Steps

### 3.1 Using npm (Recommended)

npm is Node.js's package manager and the officially recommended way to install Element Plus.

```bash
# Enter your Vue 3 project directory
cd your-vue-project

# Install Element Plus
npm install element-plus
```

### 3.2 Using yarn

```bash
# Install Element Plus
yarn add element-plus
```

### 3.3 Using pnpm

```bash
# Install Element Plus
pnpm add element-plus
```

## 4. Importing in Vue 3 Project

### 4.1 Full Import (Suitable for Small Projects)

Full import will bundle all components. It's simple to configure but results in a larger bundle size.

```typescript
// main.ts or main.js
import { createApp } from 'vue'
import App from './App.vue'

// Import Element Plus and its styles
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// Use Element Plus
app.use(ElementPlus)

app.mount('#app')
```

### 4.2 On-Demand Import (Recommended for Production)

On-demand import can significantly reduce bundle size by only importing components that are actually used.

#### 4.2.1 Install Auto-Import Plugin

First, install unplugin-vue-components and unplugin-auto-import:

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

#### 4.2.2 Configure Vite

Add configuration in vite.config.ts:

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

With this configuration, components can be auto-imported without manual imports. For example:

```vue
<template>
  <el-button type="primary">Hello</el-button>
</template>
```

#### 4.2.3 Manual On-Demand Import (Alternative)

If you don't want to use the auto-import plugin, you can manually import individual components:

```typescript
// main.ts
import { createApp } from 'vue'
import App from './App.vue'

// Manually import needed components
import { ElButton, ElInput, ElForm } from 'element-plus'
import 'element-plus/es/components/button/style'
import 'element-plus/es/components/input/style'
import 'element-plus/es/components/form/style'

const app = createApp(App)
app.use(ElButton)
app.use(ElInput)
app.use(ElForm)

app.mount('#app')
```

## 5. Import via CDN (No Build Tool Required)

If you don't want to use npm or build tools, you can import Element Plus via CDN.

### 5.1 Import via unpkg CDN

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Element Plus CDN Example</title>
  <!-- Import Vue 3 -->
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <!-- Import Element Plus -->
  <link rel="stylesheet" href="https://unpkg.com/element-plus/dist/index.css">
  <script src="https://unpkg.com/element-plus"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary">Hello Element Plus</el-button>
  </div>

  <script>
    const { createApp, ref } = Vue
    
    const App = {
      setup() {
        const visible = ref(false)
        return { visible }
      },
      template: `
        <el-button @click="visible = true">Open Dialog</el-button>
        <el-dialog v-model="visible" title="Hello">This is Element Plus</el-dialog>
      `
    }
    
    const app = createApp(App)
    app.use(ElementPlus)
    app.mount('#app')
  </script>
</body>
</html>
```

### 5.2 Import via jsDelivr CDN

```html
<!-- Replace unpkg with jsDelivr -->
<script src="https://cdn.jsdelivr.net/npm/vue@3/dist/vue.global.js"></script>
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/element-plus/dist/index.css">
<script src="https://cdn.jsdelivr.net/npm/element-plus"></script>
```

## 6. Internationalization Configuration

Element Plus uses English by default. To use Chinese, you need to configure internationalization.

### 6.1 Global Chinese Configuration

```typescript
// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import zhCn from 'element-plus/es/locale/lang/zh-cn'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus, {
  locale: zhCn,
})

app.mount('#app')
```

### 6.2 Dynamic Language Switching

```typescript
// i18n.ts
import { createI18n } from 'vue-i18n'
import zhCn from 'element-plus/es/locale/lang/zh-cn'
import en from 'element-plus/es/locale/lang/en'

const i18n = createI18n({
  legacy: false,
  locale: 'zh-cn',
  messages: {
    'zh-cn': {
      ...zhCn,
    },
    en: {
      ...en,
    },
  },
})

export default i18n
```

Then use it in main.ts:

```typescript
import { createApp } from 'vue'
import App from './App.vue'
import ElementPlus from 'element-plus'
import i18n from './i18n'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus)
app.use(i18n)

app.mount('#app')
```

## 7. Theme Customization

### 7.1 Using SCSS Variable Override

Element Plus is written in SCSS. You can customize the theme by overriding variables.

```scss
// styles/element-variables.scss
// Primary color
$--color-primary: #409EFF;

// Override component variables
@use "element-plus/theme-chalk/src/common/var.scss" as *;

// Method 1: Using scss variable override (requires sass)
$el-button-primary-bg-color: #409EFF;
```

### 7.2 Online Theme Customization

Visit the Element Plus official theming page: https://element-plus.org/en-US/guide/theming.html

You can choose primary color, border color, border radius and other parameters online, then download the customized CSS file.

### 7.3 Dark Mode

Element Plus has built-in support for dark mode:

```typescript
// Enable dark mode
import 'element-plus/theme-chalk/dark/css-vars.css'
```

Add class to the HTML tag:

```html
<html class="dark">
  <!-- Your app -->
</html>
```

## 8. Common Issues

### 8.1 Installation Failure

**Issue**: npm install fails

**Solutions**:

1. Clear npm cache: `npm cache clean --force`
2. Check your network, you can use domestic mirror:

```bash
# Set npm mirror
npm config set registry https://registry.npmmirror.com

# Or use yarn
yarn config set registry https://registry.npmmirror.com
```

### 8.2 Styles Not Applied

**Issue**: Component styles display abnormally

**Solutions**:

1. Make sure you've imported the style file: `import 'element-plus/dist/index.css'`
2. Check if build configuration correctly handles CSS
3. Clear browser cache and rebuild

### 8.3 TypeScript Type Errors

**Issue**: Type errors when importing components

**Solutions**:

1. Make sure you have the latest TypeScript: `npm install -D typescript`
2. Add configuration in tsconfig.json:

```json
{
  "compilerOptions": {
    "moduleResolution": "node",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true
  }
}
```

### 8.4 Components Not Rendering

**Issue**: Components imported but not displaying

**Solutions**:

1. Check if you correctly used app.use(ElementPlus)
2. Confirm Vue version is 3.2.0 or higher
3. Check browser console for error messages

## 9. Version Migration (from Element UI)

If your project previously used Element UI and needs to migrate to Element Plus, refer to the following steps:

### 9.1 Major Changes

| Change | Element UI | Element Plus |
|--------|------------|--------------|
| Vue Version | Vue 2 | Vue 3 |
| Component Prefix | el- | el- (unchanged) |
| Import Method | Full/On-demand | Full/On-demand |
| Event Binding | v-model | v-model (Vue 3 syntax) |

### 9.2 Using Migration Tool

Element Plus provides an official migration tool called gogocode:

```bash
# Install gogocode globally
npm install -g gogocode

# Run migration
gogocode -s your-project-path -t element-plus -o
```
