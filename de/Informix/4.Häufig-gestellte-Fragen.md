# Häufig gestellte Fragen zu IBM Informix

## Allgemeine Fragen

### F: Was ist der Unterschied zwischen IBM Informix und HCL Informix?

A: IBM Informix wird von IBM verkauft und vermarktet, während HCL Informix von HCLSoftware entwickelt und unterstützt wird. Beide teilen dieselbe Codebasis und sind für lokale Installationen funktional gleichwertig. IBM handhabt Verkauf und Lizenzierung, während HCL Entwicklung und technischen Support übernimmt.

### F: Welche Edition sollte ich für die Entwicklung verwenden?

A: Verwenden Sie die **Developer Edition** (kostenlos) oder **Innovator-C Edition** (kostenlos) für Entwicklung und Tests. Beide haben Ressourcenbeschränkungen, enthalten aber die vollständige Funktionalität:
- Developer Edition: 1 CPU VP/Core, 1GB RAM, 8GB Speicher
- Innovator-C Edition: 2 CPU Cores, 8GB RAM, 50GB Speicher

### F: Ist Informix für IoT-Anwendungen geeignet?

A: Ja, Informix ist hervorragend für IoT geeignet aufgrund seiner kleinen Stellfläche (so klein wie 100 MB), geringen Administrationsanforderungen, integrierter Zeitreihenunterstützung und der Fähigkeit, auf Edge-Geräten einschließlich Raspberry Pi zu laufen.

### F: Welche Programmiersprachen werden unterstützt?

A: Informix unterstützt mehrere Programmierschnittstellen:
- SQL (nativ)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- MongoDB-Kompatibilität (JSON/BSON)

## Installationsprobleme

### F: Installation schlägt mit "Keine Berechtigung" Fehler fehl

A: Stellen Sie sicher, dass Sie Root/Sudo-Rechte haben. Erstellen Sie den informix-Benutzer und die Gruppe vor der Installation:

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### F: Server startet nicht nach der Installation

A: Überprüfen Sie Folgendes:
- Umgebungsvariablen sind korrekt gesetzt (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- `onconfig` und `sqlhosts` Dateien sind korrekt
- Überprüfen Sie Fehlerprotokolle in `$INFORMIXDIR/tmp/`
- Stellen Sie ausreichend Speicherplatz und Berechtigungen sicher

### F: Wie Lizenzschlüsselprobleme lösen?

A: Für Evaluierungsversionen stellen Sie sicher, dass Sie von IBM Passport Advantage mit den richtigen Berechtigungen heruntergeladen haben. Für die Produktion wenden Sie den Lizenzschlüssel mit dem IBM License Key Center an.

### F: Welche Ports verwendet Informix?

A: Der Standardport ist 9088. Sie können dies in der `sqlhosts`-Datei überprüfen. Stellen Sie sicher, dass die Firewall diesen Port für Remote-Verbindungen erlaubt.

## Häufige Betriebsprobleme

### F: Wie Informix sichern und wiederherstellen?

A: Verwenden Sie die `ontape` oder `onbar` Dienstprogramme:

```bash
# Vollständiges Backup mit ontape
ontape -s -L 0

# Restore mit ontape
ontape -r

# Backup mit onbar
onbar -b -L 0

# Restore mit onbar
onbar -r
```

### F: Wie Serverstatus prüfen?

A: Verwenden Sie den `onstat` Befehl:

```bash
onstat -        # Allgemeiner Status
onstat -d       # Speicherplatznutzung
onstat -l       # Logische Protokolle
onstat -u       # Aktive Benutzer
onstat -k       # Sperrinformationen
onstat -g <cmd> # Spezifische Gruppe (Speicher, Sperren, etc.)
```

### F: Wie von externen Anwendungen mit Informix verbinden?

A: Konfigurieren Sie ODBC-, JDBC- oder native Client-Treiber. Aktualisieren Sie die `sqlhosts`-Datei für Netzwerkkonnektivität. Beispiel-JDBC-Verbindungszeichenfolge:

```
jdbc:informix-sqli://hostname:9088/dbname:INFORMIXSERVER=servername
```

### F: Was verursacht den Fehler "Logical Log Full"?

A: Dies tritt auf, wenn alle logischen Protokolle voll sind und nicht gesichert wurden. Lösungen:
- Logische Protokolle sichern: `ontape -s -L 0`
- Mehr logische Protokolldateien hinzufügen: `onparams -a -d dbspace -s größe`
- Langlaufende Transaktionen überprüfen

### F: Wie Sperrkonflikte behandeln?

A: Verwenden Sie `onstat -k` um Sperren anzuzeigen und `onstat -u` um blockierende Sitzungen zu identifizieren. Lösungen:
- Sperr-Timeout erhöhen
- Niedrigere Isolationsstufe verwenden (dirty read)
- Blockierende Sitzung beenden: `onmode -z sitzungs_id`

## Leistung und Tuning

### F: Wie Abfrageleistung verbessern?

A:
- Erstellen Sie geeignete Indizes für häufig abgefragte Spalten
- Verwenden Sie `EXPLAIN` um Abfragepläne zu analysieren
- Konfigurieren Sie Shared Memory angemessen (`SHMVIRTSIZE`, `BUFFERPOOL`)
- Aktualisieren Sie regelmäßig Statistiken: `UPDATE STATISTICS FOR TABLE tabelle`
- Erwägen Sie die Verwendung von Informix Warehouse Accelerator für BI-Abfragen

### F: Was sind die wichtigen Speicherkonfigurationsparameter?

A:
- `SHMVIRTSIZE` - Virtuelle Shared-Memory-Größe (Erstzuweisung)
- `SHMADD` - Zusätzliche Shared-Memory-Segmente
- `BUFFERPOOL` - Datencache-Konfiguration für verschiedene Seitengrößen

### F: Wie Leistung überwachen?

A:
- Verwenden Sie `onstat -g` Befehle für verschiedene Statistiken
- Überwachen Sie mit IBM Informix Performance Manager (in fortgeschrittenen Editionen enthalten)
- Überprüfen Sie Abfragepläne mit `SET EXPLAIN ON`
