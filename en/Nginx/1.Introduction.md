# Nginx Introduction

Nginx (engine x) is a high-performance HTTP server and reverse proxy server. It is known for its high performance, stability, rich feature set, and low resource consumption.

## What is Nginx?

Nginx is an open-source web server that can also be used as a reverse proxy, load balancer, and HTTP cache. It powers many high-traffic websites.

### Key Features

- **High Performance** - Handles thousands of concurrent connections
- **Reverse Proxy** - Load balance across multiple servers
- **Static Content** - Efficiently serves static files
- **SSL/TLS Termination** - Handle HTTPS traffic
- **URL Rewriting** - Powerful rewrite capabilities
- **Caching** - Built-in caching functionality
- **Load Balancing** - Multiple load balancing methods

## Architecture

Nginx uses an event-driven, asynchronous architecture:

```
┌─────────────────────────────────────┐
│           Worker Processes          │
│   (handle requests asynchronously) │
├─────────────────────────────────────┤
│         Event Loop                  │
│   (non-blocking I/O)               │
├─────────────────────────────────────┤
│        Operating System             │
│   (epoll, kqueue, etc.)            │
└─────────────────────────────────────┘
```

## Use Cases

### Web Server

Nginx can serve static and dynamic content directly.

### Reverse Proxy

Nginx can proxy requests to backend application servers.

### Load Balancer

Nginx can distribute load across multiple backend servers.

### API Gateway

Nginx can act as an API gateway with routing and authentication.

## Quick Start

### Installation

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx

# macOS
brew install nginx

# Docker
docker run -d -p 80:80 nginx
```

### Start Nginx

```bash
# Start service
sudo systemctl start nginx

# Enable on boot
sudo systemctl enable nginx
```

### Verify Installation

```bash
# Check version
nginx -v

# Test configuration
nginx -t

# Access default page
curl http://localhost
```
