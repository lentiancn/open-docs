# OpenFeign Häufig gestellte Fragen

## Installationsbezogene Fragen

### F: Wie überprüfe ich, ob OpenFeign korrekt installiert ist?
**A:** Nach dem Hinzufügen der Abhängigkeiten überprüfen Sie dies durch Erstellen einer einfachen Anwendung:

```java
@SpringBootApplication
@EnableFeignClients
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

Stellen Sie sicher, dass Ihre Spring Boot-Anwendung erfolgreich startet und Feign-Clients korrekt injiziert werden können.

### F: Wie gehe ich mit Versionskonflikten bei Abhängigkeiten um?
**A:** Verwenden Sie Spring Cloud BOM zur Versionsverwaltung:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>2022.0.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Dies verhindert effektiv Versionskonflikte.

### F: Warum muss ich spring-cloud-starter-loadbalancer explizit hinzufügen?
**A:** Ab Spring Cloud 2020.0.0 wurde Ribbon entfernt und Spring Cloud LoadBalancer zum Standard-Load-Balancer. `spring-cloud-starter-openfeign` enthält es als optionale Abhängigkeit, daher müssen Sie es explizit hinzufügen:

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

## Konfigurationsbezogene Fragen

### F: Wie konfiguriere ich Timeouts?
**A:** Sie können Timeouts über Konfigurationsdatei oder Java-Konfiguration einstellen:

**Konfigurationsdatei-Ansatz:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 5000
            read-timeout: 5000
          user-service:
            connect-timeout: 3000
            read-timeout: 10000
```

**Java-Konfigurationsansatz:**
```java
@Configuration
public class FeignConfig {
    
    @Bean
    public Request.Options requestOptions() {
        return new Request.Options(3000, 10000);
    }
}
```

### F: Wie aktiviere ich GZIP-Komprimierung?
**A:** Aktivieren Sie die Komprimierung in der Konfigurationsdatei:

```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
          mime-types: text/xml,application/xml,application/json
          min-request-size: 2048
        response:
          enabled: true
```

Hinweis: Bei Verwendung von OkHttpClient wird die Komprimierung deaktiviert, wenn `content-encoding`- oder `accept-encoding`-Header vorhanden sind.

### F: Wie konfiguriere ich Protokollebenen?
**A:** Zwei Konfigurationsansätze:

**Konfigurationsdatei:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          user-service:
            logger-level: full
            
logging:
  level:
    com.example.clients.UserService: DEBUG
```

**Java-Konfiguration:**
```java
@Configuration
public class FeignLoggingConfig {
    
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}
```

Unterstützte Protokollebenen:
- **NONE**: Keine Protokollierung (Standard)
- **BASIC**: Protokolliert nur Anfragemethode, URL, Antwortstatuscode und Ausführungszeit
- **HEADERS**: Protokolliert grundlegende Informationen zusammen mit Anfrage- und Antwortheadern
- **FULL**: Protokolliert Header, Textkörper und Metadaten für Anfragen und Antworten

## Verwendungsbezogene Fragen

### F: Die FeignClient-Schnittstelle kann nicht injiziert werden, was soll ich tun?
**A:** Häufige Ursachen und Lösungen:

**1. Problem mit dem Scan-Paketpfad:**
```java
// Stellen Sie sicher, dass FeignClient im gescannten Paketpfad liegt
@SpringBootApplication
@EnableFeignClients(basePackages = "com.example.clients")
public class Application {
    // ...
}
```

**2. Problem mit zirkulären Abhängigkeiten:**
Wenn FeignClient von mehreren Services referenziert wird, kann eine zirkuläre Abhängigkeit auftreten. Lösungen:
- Verwenden Sie die `@Lazy`-Annotation für verzögerte Injektion
- Refaktorisieren Sie die Code-Struktur, um zirkuläre Abhängigkeiten zu vermeiden

**3. Bean-Namenkonflikte:**
Wenn Sie mehrere FeignClients mit dem gleichen Namen haben, verwenden Sie `contextId` zur Unterscheidung:
```java
@FeignClient(contextId = "userClient1", name = "user-service")
public interface UserClient1 { }

@FeignClient(contextId = "userClient2", name = "user-service")  
public interface UserClient2 { }
```

### F: Wie handle ich komplexe Anfrageparameter?
**A:** Mehrere Ansätze für komplexe Parameter:

**1. Verwendung von @SpringQueryMap:**
```java
public class SearchCriteria {
    private String name;
    private String status;
    private Integer page;
}

@FeignClient(name = "search-service")
public interface SearchService {
    @GetMapping("/search")
    SearchResult search(@SpringQueryMap SearchCriteria criteria);
}
```

**2. Verwendung von Map:**
```java
@FeignClient(name = "api-service")
public interface ApiService {
    @GetMapping("/data")
    Data getData(@RequestParam Map<String, Object> params);
}
```

**3. Benutzerdefinierter QueryMapEncoder:**
```java
@Bean
public QueryMapEncoder queryMapEncoder() {
    return new CustomQueryMapEncoder();
}
```

### F: Wie handle ich Dateiuploads?
**A:** Dateiuploads erfordern spezielle Konfiguration:

```java
@FeignClient(name = "file-service")
public interface FileService {
    
    @PostMapping(value = "/upload", 
                consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    UploadResponse uploadFile(@RequestPart("file") MultipartFile file,
                             @RequestPart("metadata") FileMetadata metadata);
}
```

Konfigurieren Sie auch den Encoder:
```java
@Bean
public Encoder feignFormEncoder() {
    return new FeignFormEncoder(new SpringEncoder(messageConverters()));
}
```

## Fehlerbehandlungsfragen

### F: Wie passe ich die Fehlerbehandlung an?
**A:** Implementieren Sie die ErrorDecoder-Schnittstelle:

```java
@Component
public class CustomErrorDecoder implements ErrorDecoder {
    
    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() == 404) {
            return new ResourceNotFoundException("Resource not found");
        } else if (response.status() >= 500) {
            return new ServerException("Server error");
        }
        return new Default().decode(methodKey, response);
    }
}
```

### F: Circuit Breaker funktioniert nicht, was stimmt nicht?
**A:** Überprüfen Sie die folgenden Konfigurationen:

**1. Circuit Breaker aktivieren:**
```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

**2. Erforderliche Abhängigkeit hinzufügen:**
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

**3. Circuit Breaker-Regeln konfigurieren:**
```yaml
resilience4j:
  circuitbreaker:
    instances:
      UserServicegetUser:
        failure-rate-threshold: 50
        minimum-number-of-calls: 5
        wait-duration-in-open-state: 5000
```

### F: Fallback wird nicht ausgelöst, was soll ich überprüfen?
**A:** Häufige Probleme:

**1. Stellen Sie sicher, dass die Fallback-Klasse ein Spring-Bean ist:**
```java
@Component  // Diese Annotation ist erforderlich
public class UserServiceFallback implements UserService {
    // ...
}
```

**2. Überprüfen Sie Ausnahmetypen:**
Der Circuit Breaker löst Fallback nur für bestimmte Ausnahmen aus. Stellen Sie sicher, dass die ausgelöste Ausnahme vom erwarteten Typ ist.

**3. Konfigurieren Sie die richtige Fallback-Factory:**
```java
@FeignClient(name = "service", fallbackFactory = MyFallbackFactory.class)
public interface MyService { }
```

## Leistungsbezogene Fragen

### F: Die Leistung des Feign-Clients ist langsam, wie optimiere ich sie?
**A:** Empfehlungen zur Leistungsoptimierung:

**1. Passende Timeouts konfigurieren:**
```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connect-timeout: 1000
            read-timeout: 5000
```

**2. Verbindungspooling aktivieren:**
```yaml
spring:
  cloud:
    openfeign:
      httpclient:
        hc5:
          enabled: true
          max-connections: 200
          max-connections-per-route: 20
```

**3. Komprimierung aktivieren:**
```yaml
spring:
  cloud:
    openfeign:
      compression:
        request:
          enabled: true
        response:
          enabled: true
```

**4. Caching verwenden:**
```java
@FeignClient(name = "config-service")
public interface ConfigService {
    @GetMapping("/config/{key}")
    @Cacheable(cacheNames = "config-cache", key = "#key")
    ConfigValue getConfig(@PathVariable String key);
}
```

### F: Wie behebe ich Speicherleckprobleme?
**A:** Speicherlecks werden typischerweise verursacht durch:

**1. Nicht geschlossene Verbindungen:**
Stellen Sie sicher, dass HTTP-Clients Verbindungspools korrekt verwalten.

**2. Nicht bereinigter Cache:**
Bereinigen Sie den Cache regelmäßig oder legen Sie angemessene Ablaufzeiten fest.

**3. Speicherverbrauch überwachen:**
```java
@RestController
public class MetricsController {
    
    @GetMapping("/memory")
    public MemoryInfo getMemoryInfo() {
        Runtime runtime = Runtime.getRuntime();
        return new MemoryInfo(
            runtime.totalMemory(),
            runtime.freeMemory(),
            runtime.maxMemory()
        );
    }
}
```

## Sicherheitsbezogene Fragen

### F: Wie füge ich Authentifizierungstoken zu Anfragen hinzu?
**A:** Verwenden Sie RequestInterceptor:

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor authRequestInterceptor() {
        return requestTemplate -> {
            String token = getToken(); // Logik zum Abrufen des Tokens
            requestTemplate.header("Authorization", "Bearer " + token);
        };
    }
    
    private String getToken() {
        // Implementieren Sie die Token-Abruflogik
        return "your-token";
    }
}
```

### F: Wie handle ich OAuth2-Authentifizierung?
**A:** OAuth2-Unterstützung aktivieren:

```yaml
spring:
  cloud:
    openfeign:
      oauth2:
        enabled: true
        client-registration-id: my-client
        
  security:
    oauth2:
      client:
        registration:
          my-client:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: client_credentials
            scope: read,write
```

### F: Wie filtere ich sensible Informationen?
**A:** Benutzerdefiniertes Protokollfiltering:

```java
public class SecureLogger extends Logger {
    
    @Override
    protected void logRequest(String configKey, Level logLevel, Request request) {
        Map<String, Collection<String>> safeHeaders = new HashMap<>(request.headers());
        safeHeaders.remove("Authorization");
        safeHeaders.remove("X-API-Key");
        
        super.logRequest(configKey, logLevel, 
            Request.create(request.httpMethod(), request.url(), safeHeaders,
                request.body(), request.charset(), request.requestTemplate()));
    }
}
```

## Testbezogene Fragen

### F: Wie teste ich Feign-Clients?
**A:** Es wird empfohlen, WireMock zu verwenden:

```java
@SpringBootTest
@AutoConfigureWireMock(port = 0)
class FeignClientTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    void shouldGetUserSuccessfully() {
        // Given
        stubFor(get(urlEqualTo("/users/1"))
            .willReturn(aResponse()
                .withStatus(200)
                .withHeader("Content-Type", "application/json")
                .withBody("{\"id\":1,\"name\":\"John\"}")));
        
        // When
        User user = userService.getUser(1L);
        
        // Then
        assertThat(user.getName()).isEqualTo("John");
    }
}
```

### F: Wie simuliere ich Fehlerszenarien?
**A:** Verwenden Sie WireMock, um verschiedene Fehler zu simulieren:

```java
@Test
void shouldHandleServerError() {
    // Given
    stubFor(get(urlEqualTo("/users/999"))
        .willReturn(aResponse().withStatus(500)));
    
    // When & Then
    assertThatThrownBy(() -> userService.getUser(999L))
        .isInstanceOf(ServerException.class);
}
```

```yaml
          enabled: true

        enabled: true
      cache:
        enabled: true
        
logging:
  level:
    com.example.clients: WARN
```

### F: Wie überwache ich Feign-Clients?
**A:** Integration mit Micrometer:

```yaml
spring:
  cloud:
    openfeign:
      micrometer:
        enabled: true
```

Dann Metriken über Actuator-Endpunkte anzeigen:
```bash
GET /actuator/metrics
GET /actuator/metrics/feign.client.requests
```

Durch das Verständnis und die Anwendung dieser Lösungen können Sie OpenFeign-basierte Anwendungen effektiver entwickeln, debuggen und warten.
