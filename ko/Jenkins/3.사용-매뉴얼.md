# 사용자 매뉴얼

## 파이프라인 기초

파이프라인은 빌드, 테스트부터 배포까지 전체 소프트웨어 개발 프로세스를 자동화하는 Jenkins의 핵심 기능입니다.

### 파이프라인이란

파이프라인은 Jenkins에 지속적인 제공 파이프라인을 구현하고 통합할 수 있도록 하는 Jenkins 플러그인 모음입니다. 간단히 말해, 파이프라인은 소프트웨어가 버전 관리에서 사용자에게 도달하는 전체 프로세스를 자동화합니다.

파이프라인 정의는 프로젝트의 소스 제어 저장소에 커밋할 수 있는 텍스트 파일(Jenkinsfile)로 작성됩니다. 이를 통해 "파이프라인 as 코드" 철학을 구현합니다.

### 파이프라인의 이점

- **코드 기반**: 파이프라인은 코드처럼 버전 관리되어 검토 및 반복 가능
- **지속성**: 계획되거나 계획되지 않은 Jenkins 재시작 모두에서 파이프라인이 유지
- **일시 중지 가능**: 파이프라인은 사람의 입력 또는 승인을 기다리며 중지될 수 있음
- **다용도**: 병렬 실행 및 루프를 포함한 복잡한 실제 CD 요구사항 지원
- **확장성**: 플러그인이 Pipeline DSL을 확장하고 다른 도구와 통합 가능

### 파이프라인 개념

#### 노드

노드는 파이프라인을 실행할 수 있는 Jenkins 환경의 일부인 컴퓨터입니다. 노드는 마스터 또는 에이전트 컴퓨터일 수 있습니다.

#### 스테이지

스테이지는 "Build", "Test", "Deploy" 스테이지와 같이 전체 파이프라인에서 수행되는 개념적으로 구분된 작업 하위 집합을 정의합니다. 스테이지는 파이프라인 실행 상태를 시각화하는 데 사용됩니다.

#### 스텝

스텝은 특정 시점에서 Jenkins에게 무엇을 할지 지시하는 Jenkins 파이프라인의 기본 작업입니다. 예를 들어, `sh` 스텝을 사용하여 셸 명령을 실행합니다.

### 파이프라인 문법

#### 선언적 파이프라인

선언적 파이프라인은 더 풍부한 기능을 가진 새로운 문법으로 작성 및 읽기가 더 쉽습니다:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                echo 'Building application...'
            }
        }
        stage('Test') {
            steps {
                echo 'Running tests...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying application...'
            }
        }
    }
}
```

#### 스크립트 파이프라인

스크립트 파이프라인은 Groovy 문법을 사용하며 더 유연하지만 더 많은 프로그래밍 경험이 필요합니다:

```groovy
node {
    stage('Build') {
        echo 'Building application...'
    }
    stage('Test') {
        echo 'Running tests...'
    }
    stage('Deploy') {
        echo 'Deploying application...'
    }
}
```

## 첫 번째 파이프라인 만들기

### Blue Ocean 사용

Blue Ocean은 파이프라인 생성을 더 직관적으로 만드는 Jenkins의 현대적인 인터페이스입니다:

1. Blue Ocean 플러그인 설치(미설치된 경우)
2. "Open Blue Ocean" 클릭
3. "Create Pipeline" 클릭
4. 코드 저장소 위치 선택(GitHub, GitLab 등)
5. 저장소 및 브랜치 선택
6. Blue Ocean이 Jenkinsfile을 자동 감지하거나 생성 지원

### Jenkinsfile 사용

프로젝트에 Jenkinsfile을 생성하면 Jenkins가 자동으로 인식합니다:

1. 프로젝트 루트에 Jenkinsfile 생성
2. 파이프라인 스테이지 정의
3. 버전 관리에 커밋
4. Jenkins에서 파이프라인 작업 생성 및 저장소 연결

## 자격 증명 관리

Jenkins는 코드 저장소 및 배포 대상과 같은 다양한 외부 리소스에 접근해야 하며, 자격 증명 관리가 필요합니다.

### 자격 증명 추가

1. "Manage Jenkins" > "Manage Credentials"로 이동
2. "Add Credentials" 클릭
3. 자격 증명 유형 선택(비밀번호가 있는 사용자 이름, 개인 키가 있는 SSH 사용자 이름, 비밀 파일 등)
4. 자격 증명 정보 입력
5. 자격 증명의 ID 및 설명 설정

### 파이프라인에서 자격 증명 사용

withCredentials 스텝과 함께 자격 증명 사용:

```groovy
pipeline {
    agent any
    stages {
        stage('Use Credentials') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'my-cred', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    echo "Username: $USER"
                }
            }
        }
    }
}
```

## 빌드 트리거

자동화된 빌드에는 트리거가 필요합니다. Jenkins는 여러 트리거 방법을 지원합니다.

### 소스 코드 관리 폴링

소스 코드가 변경되었는지 정기적으로 확인:

```groovy
pipeline {
    agent any
    triggers {
        pollSCM('H/5 * * * *')  // 5분마다 확인
    }
    stages {
        stage('Build') {
            steps {
                echo 'Code change detected, starting build'
            }
        }
    }
}
```

### 웹훅 트리거

더 나은 효율성을 위해 저장소 웹훅을 통해 빌드 트리거:

1. 해당 플러그인 설치(예: GitHub Hook Trigger)
2. Jenkins를 가리키는 저장소 웹훅 구성
3. 코드 커밋 시 자동 빌드 트리거

### 예약된 빌드

예약에 따라 자동으로 실행:

```groovy
pipeline {
    agent any
    triggers {
        cron('H 2 * * *')  // 매일 오전 2시 실행
    }
    stages {
        stage('Scheduled Build') {
            steps {
                echo 'Scheduled build running'
            }
        }
    }
}
```

## 에이전트 및 컨테이너 사용

### 에이전트 구성

Jenkins는 분산 빌드를 지원하여 빌드 작업을 여러 컴퓨터에 분산:

1. "Manage Jenkins" > "Manage Nodes"로 이동
2. 에이전트 노드 추가 및 연결 정보 구성
3. 파이프라인에서 에이전트 지정

### 파이프라인에서 Docker 사용

선언적 파이프라인은 Docker 컨테이너에서 빌드를 실행할 수 있습니다:

```groovy
pipeline {
    agent {
        docker {
            image 'maven:3.8-openjdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'mvn --version'
            }
        }
    }
}
```

## 테스트 보고서

Jenkins는 다양한 테스트 프레임워크와 통합하여 시각적인 테스트 보고서를 생성할 수 있습니다.

### JUnit 테스트 보고서

```groovy
pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}
```

### 기타 테스트 보고서

Jenkins는 다양한 테스트 보고서 플러그인을 지원합니다:

- TestNG
- Cobertura(코드 커버리지)
- JaCoCo(코드 커버리지)
- ESLint(프론트엔드 코드 분석)

## 배포 관행

### 다양한 환경에 배포

일반적인 다단계 배포 파이프라인:

```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('Deploy to Test') {
            when {
                branch 'develop'
            }
            steps {
                sh './deploy.sh test'
            }
        }
        stage('Deploy to Production') {
            when {
                branch 'main'
            }
            steps {
                input message: 'Confirm deployment to production?', ok: 'Deploy'
                sh './deploy.sh prod'
            }
        }
    }
}
```

### Blue-Green 배포

Jenkins로 Blue-Green 배포를 구현하여 가동 중지 시간 없이 업데이트:

1. 두 개의 동일한 환경(blue 및 green) 유지
2. 트래픽을 새 버전으로 전환
3. 문제가 발생하면 빠르게 롤백

## 권장 플러그인

### 필수 플러그인

- Git: 코드 저장소 통합
- Pipeline: 파이프라인 지원
- Blue Ocean: 현대적인 인터페이스
- Credentials: 자격 증명 관리
- Configuration as Code: 코드로 구성

### 일반적인 플러그인

- Docker: Docker 통합
- Kubernetes: Kubernetes 통합
- Maven Integration: Maven 프로젝트 지원
- Gradle: Gradle 프로젝트 지원
- Slack Notification: Slack 알림
- Email Extension: 이메일 알림

## 모범 사례

### Jenkinsfile 모범 사례

- Jenkinsfile을 버전 관리에 유지
- 선언적 파이프라인 문법 사용
- 파이프라인을 간단하고 명확하게 유지
- 스테이지를 합리적으로 구분
- 필요한 타임아웃 및 재시도 메커니즘 추가

### 보안 권장사항

- Jenkins 및 플러그인을 정기적으로 업데이트
- 익명 사용자 권한 제한
- 민감한 정보에 자격 증명 사용
- 적절한 접근 제어 구성
- 보안 감사 로깅 활성화

### 성능 최적화

- 빌드 노드 적절히 구성
- 증분 빌드를 사용하여 시간 단축
- 빌드 캐시 구성
- 오래된 빌드 기록 정기적으로 정리
- 병렬 스테이지 사용하여 빌드 속도 향상

## 문제 해결

### 빌드 로그 보기

빌드가 실패하면 먼저 빌드 로그를 확인:

1. 작업 페이지로 이동
2. 실패한 빌드 번호 클릭
3. 콘솔 출력 확인

### 일반적인 문제 해결

- 빌드 막힘: 보류 중인 입력 또는 리소스 데드락 확인
- 자격 증명 무효: 자격 증명이 만료되거나 취소되었는지 확인
- 플러그인 충돌: 최근 설치한 플러그인 비활성화 시도
- 메모리 부족: JVM 힙 매개변수 조정

## 요약

Jenkins 파이프라인은 CI/CD를 구현하는 핵심 도구입니다. 이 매뉴얼을 통해 다음을 수행할 수 있어야 합니다:

- 핵심 파이프라인 개념 이해
- 파이프라인 작업 생성 및 구성
- 자격 증명 및 트리거 관리
- 테스트 및 배포 도구 통합
- 모범 사례를 적용하여 워크플로 최적화

Jenkins 마스터에는 지속적인 학습과 실습이 핵심입니다. 실제 프로젝트에서 계속 탐색하고 최적화하세요.
