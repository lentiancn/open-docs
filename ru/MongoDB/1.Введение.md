# MongoDB Введение

## Что такое MongoDB?

MongoDB — это документоориентированная база данных, созданная для того, чтобы помочь разработчикам быстрее создавать современные приложения. Она хранит данные в гибких JSON-подобных документах, что упрощает моделирование данных так же, как их использует код приложения. Гибкая схема позволяет развивать модель данных без простоя, быстро выполнять итерации и легко обрабатывать неоднородные данные.

MongoDB предоставляет мощный механизм запросов, горизонтальное масштабирование и встроенную высокую доступность для поддержки всего — от быстрого прототипирования до критически важных рабочих нагрузок.

MongoDB — это полностью транзакционная операционная база данных, поддерживающая широкий спектр типов рабочих нагрузок:

- Структурированный поиск на основе документов (OLTP)
- Агрегация данных
- Полнотекстовый поиск
- Векторный поиск
- Геопространственный поиск
- Временные ряды

## Модель данных документов

Запись в MongoDB — это документ, структура данных, состоящая из пар поле-значение. Документы MongoDB похожи на объекты JSON. Значения полей могут содержать другие документы, массивы и массивы документов.

Гибкая объектно-ориентированная модель данных MongoDB позволяет структурировать данные способом, отражающим объектные модели в вашем коде. Это устраняет необходимость в сложном объектно-реляционном отображении, требуемом при использовании реляционных баз данных.

Вот пример документа MongoDB, показывающий, как документы могут выражать сложные иерархические данные в формате, отражающем объектную модель вашего приложения:

```
{
  _id: ObjectId("507f1f77bcf86cd799439011"),
  name: "Alice",
  birthdate: ISODate("1990-01-01T00:00:00Z"),
  address: {
    street: "123 Main St",
    city: "Springfield",
    state: "IL"
  },
  hobbies: ["reading", "hiking", "coding"]
}
```

## Основные компоненты архитектуры MongoDB

### Документная база данных

Гибкая модель данных документов позволяет адаптировать данные к потребностям вашего приложения. В отличие от традиционных реляционных баз данных со строгими форматами строк и столбцов, документы MongoDB могут содержать вложенные структуры.

### Транзакции

MongoDB поддерживает мультидокументные транзакции, позволяющие выполнять несколько операций чтения и записи как единое событие.

### Высокая доступность

Встроенный механизм репликации MongoDB обеспечивает автоматическое переключение при отказе, избыточность данных и повышенную емкость чтения.

### Горизонтальное масштабирование

MongoDB изначально поддерживает горизонтальное масштабирование с помощью методики, называемой шардингом.

## Редакции MongoDB

MongoDB доступен в двух основных редакциях:

- **Community Edition**: бесплатная версия с открытым исходным кодом
- **Enterprise Edition**: коммерческая версия с расширенными функциями безопасности и управления

Текущая последняя стабильная версия — MongoDB 8.2, версия с долгосрочной поддержкой — MongoDB 7.0.

## Поддерживаемые платформы

MongoDB поддерживает различные платформы и архитектуры.

## Случаи использования

MongoDB подходит для различных сценариев применения:

- Системы управления контентом
- Мобильные приложения
- Аналитика в реальном времени
- Интернет вещей
- Электронная коммерция
- Приложения искусственного интеллекта

## MongoDB и реляционные базы данных

| Характеристика | MongoDB | Реляционные БД |
|----------------|---------|----------------|
| Модель данных | Документы | Строки и столбцы |
| Схема | Гибкая | Фиксированная |
| Масштабирование | Горизонтальное | Вертикальное |
| Транзакции | Мультидокументные ACID | Мультитабличные ACID |
