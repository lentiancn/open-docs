# Yarn Frequently Asked Questions

## Installation Related Questions

### Q: How to choose between Yarn versions?
**A**: 
- **New Projects**: Recommended to use Yarn v4+ to enjoy modern features and better performance
- **Existing Projects**: If already using Yarn v1, you can continue using it or consider migrating to the modern version
- **Enterprise Environments**: Yarn v1 provides more stable long-term support, while v4+ offers more innovative features

### Q: What is Corepack and why do I need it?
**A**: Corepack is an intermediary tool bundled with Node.js that allows you to configure your package manager version on a per-project basis. It ensures team members use the same Yarn version, avoiding version inconsistency issues.

### Q: What to do if I encounter permission errors during Yarn installation?
**A**: 
- Add the `--unsafe-perm` flag when using `npm install --global yarn`
- Use nvm to manage Node.js versions
- In CI/CD environments, global installation is typically not required

## Usage Related Questions

### Q: What are the differences between Yarn and npm?
**A**: 
- **Speed**: Yarn uses parallel installs and offline caching, typically faster
- **Determinism**: Yarn guarantees consistent installations (yarn.lock)
- **Features**: Yarn provides workspaces, PnP, plugin system, and other advanced features
- **Commands**: Yarn reimplements all commands for better developer experience

### Q: What is PnP (Plug'n'Play)?
**A**: PnP is a Yarn v2+ feature that eliminates the node_modules directory by resolving dependencies directly from the cache. This saves disk space and improves installation speed, but may be incompatible with some tools that don't support PnP.

### Q: How to use workspaces in my project?
**A**: 
1. Add a `"workspaces"` field to your root `package.json`
2. Create sub-package directories (e.g., `packages/`)
3. Create individual `package.json` files in sub-packages
4. Use `yarn workspace <name> <command>` to run commands in specific workspaces

## Version Migration Questions

### Q: How to migrate from Yarn v1 to v4+?
**A**: 
1. Ensure your project is under Git version control
2. Run `yarn set version stable` (for v4+) 
3. Follow the migration guide to update configurations step by step
4. Test all functionality to ensure compatibility

### Q: What to do if I encounter compatibility issues after migration?
**A**: 
- Check if any tools don't support PnP, you can disable PnP: `yarn config set nodeLinker node-modules`
- Update dependencies to compatible versions
- Refer to the official migration documentation for specific solutions

## Troubleshooting

### Q: What to do if `yarn install` fails?
**A**: 
- Clear cache: `yarn cache clean`
- Delete `yarn.lock` and reinstall
- Check network connection and registry configuration
- For Yarn v4+, check `.yarnrc.yml` configuration

### Q: How to check Yarn version?
**A**: 
- Global version: `yarn --version`
- Project version: Check `.yarn/releases/` directory or `.yarnrc.yml` file

### Q: What to do if Yarn commands are very slow?
**A**: 
- Ensure offline caching is enabled
- Check network connection
- For large projects, consider using workspaces to split the codebase
- Update to the latest version for performance improvements
