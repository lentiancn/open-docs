# Mockito 소개

## Mockito란 무엇인가

Mockito는 테스트구동 개발(TDD)과 동작구동 개발(BDD)을 위해 설계된 강력한 Java 단위 테스트 모킹 프레임워크입니다. 개발자가 제어된 방식으로 실제 객체의 행동을 시뮬레이션하는 모 객체를 생성할 수 있게 해주어, 격리되고 독립적인 단위 테스트를 가능하게 합니다.

## Mockito를 선택하는 이유

### 깔끔하고 간단한 API

Mockito는 학습과 사용을 직관적으로 만드는 직관적이고 간단한 API를 제공합니다. 이 프레임워크는 깔끔하고 가독성이 좋은 코드로 아름다운 테스트를 작성할 수 있게 해줍니다. 구문이 자연스럽， 테스트 코드가 프로덕션 코드처럼 읽기 쉽습니다.

### 강력한 기능

Mockito는 메서드 호출 검증, 반환값 스텁, 예외 발생, 인자 매칭, 호출 순서 검증 등 다양한 고급 기능을 지원합니다. 이러한 기능은 다양한 복잡한 테스트 시나리오를 다루며 개발자가 포괄적인 코드 커버리지를 달성하는 데 도움이 됩니다.

### 우수한 가독성

Mockito로 작성된 테스트는 우수한 가독성을 가지고 있습니다. 테스트가 실패할 때 Mockito는 문제를 빠르게 식별할 수 있도록 명확하고 정보적인 검증 오류 메시지를 제공합니다. 이 디버깅 지원은 개발 효율성을 크게 향상시킵니다.

### 활발한 커뮤니티 지원

Mockito는 Java 생태계에서 가장 인기 있는 모킹 프레임워크로, 큰 사용자 기반과 활발한 커뮤니티를 가지고 있습니다. 프레임워크는 지속적으로 유지보수되고 업데이트되며 종합적인 문서를 제공합니다. 문제가 발생하면 해결책을 쉽게 찾을 수 있습니다. Stack Overflow에서 Mockito는 가장 많은 표를 받아 Java의 인정된 최고의 모킹 프레임워크입니다.

## 핵심 개념

### 모 객체

모 객체는 테스트 중 실제 객체의 대용물로 사용됩니다. 모 객체를 생성함으로써 외부 의존성을 격리할 수 있으며, 테스트가 외부 시스템의 영향을 받지 않고 테스트 중인 단위에만 집중할 수 있도록 합니다.

### 스텁bing

스텁bing은 특정 메서드가 호출될 때 모 객체가 어떻게 행동해야 하는지를 정의합니다. when().thenReturn() 구문을 사용하면 모 메서드의 응답을 정확하게 제어할 수 있어, 테스트가 다양한 시나리오를 다룰 수 있습니다.

### 검증

검증은 테스트 중인 코드와 모 객체 간의 상호작용이 예상대로 발생했는지 확인합니다. verify() 메서드는 메서드가 호출되었는지, 몇 번 호출되었는지, 어떤 인자가 전달되었는지 확인하여 코드가 의존성을 올바르게 사용하고 있는지 보장합니다.

### 스파이

스파이는 실제 객체의 부분 모킹을 허용합니다. 완전히 모킹된 객체와 달리, 스파이는 대부분의 실제 객체 행동을 유지하면서 특정 메서드만 스텁할 수 있습니다. 이는 대부분의 실제 로직을 테스트하면서 몇 가지 메서드만 모킹해야 할 때 특히 유용합니다.

## 일반적인 사용 사례

### 외부 의존성 격리

테스트 중인 코드가 데이터베이스, 네트워크 서비스, 파일 시스템과 같은 외부 시스템에 의존할 때, Mockito는 이러한 외부 의존성을 모킹할 수 있습니다. 이를 통해 실제 외부 시스템 없이 테스트를 실행할 수 있어 테스트 실행 속도가 빨라질 뿐만 아니라 불안정한 외부 시스템으로 인한 테스트 실패도 제거됩니다.

### 미완료 모듈 모킹

대규모 프로젝트에서 서로 다른 모듈은 서로 다른 개발 단계를 가질 수 있습니다. Mockito는 아직 완료되지 않은 모듈을 모킹하여 이미 완료된 모듈을 미리 테스트하고 개발 병목 현상을 방지할 수 있습니다.

### 복잡한 객체 모킹

일부 비즈니스 객체는 여러 수준의 의존성을 포함하여 구성하기 복잡합니다. Mockito를 사용하면 이러한 복잡한 객체의 모 버전을 쉽게 생성하여 비즈니스 로직 자체를 테스트하는 데 집중할 수 있습니다.

## JUnit과의 통합

Mockito는 일반적으로 JUnit 테스트 프레임워크와 함께 사용됩니다. JUnit의 @RunWith(MockitoJUnitRunner.class) 또는 @ExtendWith(MockitoExtension.class)를 사용하면 @Mock이 주석된 필드가 자동으로 초기화되어 테스트 코드가 간소화됩니다. 이 조합은 Java 단위 테스트의 모범 사례를 나타냅니다.

## 버전 호환성

Mockito 5.x는 JDK 11 이상이 필요하며 람다 표현식과 같은 Java 8 기능을 지원합니다. Mockito 4.x 시리즈는 여전히 JDK 8을 지원합니다. 개발자는 프로젝트 요구 사항에 따라 적절한 버전을 선택할 수 있습니다.

## 요약

Mockito는 Java 개발자에게 필수적인 테스트 도구입니다. 단위 테스트에서 의존성 격리 문제를 우아하게 해결하여 개발자가 신뢰할 수 있고 유지보수 가능한 테스트 코드를 작성할 수 있게 합니다. Mockito를 마스터하면 코드 품질과 개발 효율성이 크게 향상되어 모든 Java 엔지니어에게 필수적인 기술이 됩니다.
