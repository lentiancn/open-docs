# Flutter - FAQ

Flutter 개발에 대한 일반적인 질문과 답변.

## 설치 문제

### Q: Flutter 다운로드 실패?

**해결:**
1. 미러 사용:
```bash
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
export PUB_CACHE=/path/to/cache
```
2. 프록시 사용
3. 수동으로 SDK 아카이브 다운로드

### Q: flutter doctor 오류?

**해결:**
1. PATH 구성 확인
2. Git이 설치되어 있는지 확인
3. 상세 정보를 위해 `flutter doctor -v` 실행

### Q: Android SDK를 찾을 수 없음?

**해결:**
```bash
flutter config --android-sdk /path/to/android-sdk

# 또는 환경 변수 설정
export ANDROID_SDK_ROOT=/path/to/android-sdk
```

### Q: macOS에서 Xcode 명령줄 도구가 설치되지 않음?

**해결:**
```bash
xcode-select --install
```

## 개발 문제

### Q: 핫 리로드가 작동하지 않음?

**해결:**
1. 파일을 저장하고 핫 리로드 대기
2. 콘솔에서 오류 확인
3. 앱을 완전히 재시작 시도

### Q: APK 빌드 실패?

**해결:**
1. `flutter clean` 실행
2. Gradle 업데이트
3. 네트워크 연결 확인

### Q: 에뮬레이터 시작 실패?

**해결:**
1. CPU 가상화 활성화
2. HAXM 설치 확인
3. 디버깅에 물리적 디바이스 사용

### Q: 위젯이 표시되지 않음?

**해결:**
1. build() 메서드 반환 값 확인
2. MaterialApp에 추가되었는지 확인
3. 레이아웃 제약 조건 확인

### Q: 상태가 업데이트되지 않음?

**해결:**
1. setState()가 호출되었는지 확인
2. StatefulWidget이 올바른지 확인
3. 올바른 Context를 사용하고 있는지 확인

### Q: 메모리 누수?

**해결:**
1. 컨트롤러를 적절히 dispose()
2. Provider를 적절히 정리
3. build에서 객체 생성 피하기

## 성능 문제

### Q: 앱이 버벅임?

**해결:**
1. const 생성자 사용
2. 불필요한 setState 피하기
3. ListView.builder 사용
4. 이미지 로딩 최적화

### Q: APK가 너무 큼?

**해결:**
1. 축소 사용
2. 디버그 기호 제거
3. split-per-abi 사용
4. 사용하지 않는 리소스 정리

### Q: 시작이 느림?

**해결:**
1. init() 작업 줄이기
2. 컴포넌트 지연 로딩
3. Deferred Components 사용

## 플랫폼 문제

### Q: iOS 빌드 실패?

1. Xcode 버전 확인

1. Flutter 웹 지원 확인
2. 브라우저 호환성 확인

### Q: 플랫폼 채널이 작동하지 않음?

**해결:**
1. 메서드 이름 일치 확인
2. 반환 값 유형 확인
3. 콘솔 오류 확인
