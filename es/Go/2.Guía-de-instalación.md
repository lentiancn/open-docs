# Guía de Instalación de Go

## Requisitos

Go puede ejecutarse en múltiples sistemas operativos incluyendo Linux, macOS y Windows. Los diferentes sistemas operativos tienen diferentes métodos de instalación, pero los requisitos centrales son similares. Asegúrese de que su sistema cumpla los siguientes requisitos básicos para instalar y usar correctamente el entorno de desarrollo del lenguaje Go.

En cuanto a sistemas operativos, Go soporta una amplia gama de plataformas. Para Linux, Go puede ejecutarse en distribuciones principales como Ubuntu, Debian, Fedora, CentOS y RHEL. macOS requiere macOS 10.13 (High Sierra) o posterior. Windows requiere Windows 10 o posterior, mientras también soporta Windows 7 y 8.1, aunque puede requerir configuración adicional.

En cuanto a hardware, los requisitos de Go son relativamente bajos. Instalar Go solo requiere aproximadamente 100MB de espacio en disco, aunque el desarrollo real de proyectos puede requerir más espacio dependiendo del tamaño del proyecto. Para memoria, 2GB de RAM son suficientes para desarrollo diario, aunque puede ser necesario más para compilación a gran escala o ejecutar pruebas extensas.

El compilador de Go no tiene requisitos especiales de CPU, y arquitecturas modernas como x86-64 y ARM64 funcionan bien. Go soporta compilación cruzada, permitiendo compilar binarios para otras plataformas desde una plataforma.

## Pasos de Instalación

### Instalación en Linux

En sistemas Linux, se recomienda usar el tarball oficial proporcionado, ya que este método es el más limpio y flexible. Primero, descargue el tarball de la última versión desde el sitio web oficial de Go:

```bash
# Descargar Go (reemplazar con la última versión)
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# Eliminar instalación anterior de Go (si existe)
sudo rm -rf /usr/local/go

# Extraer en /usr/local
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

A continuación, agregue el directorio binario de Go a la variable de entorno PATH. Edite el archivo ~/.bashrc o ~/.profile y agregue la siguiente línea:

```bash
export PATH=$PATH:/usr/local/go/bin
```

Después de guardar, ejecute el siguiente comando para aplicar los cambios:

```bash
source ~/.bashrc
```

O en macOS o algunas distribuciones Linux:

```bash
source ~/.profile
```

### Instalación en macOS

En sistemas macOS, hay dos métodos principales de instalación. El primero es usar el administrador de paquetes Homebrew, adecuado para usuarios que ya tienen Homebrew configurado:

```bash
# Instalar usando Homebrew
brew install go

# Verificar instalación
go version
```

El segundo método es descargar el instalador oficial de macOS. Visite el sitio web oficial de Go para descargar el instalador .pkg, haga doble clic para ejecutar y siga el asistente de instalación. El instalador configura automáticamente las variables de entorno.

Después de la instalación, abra Terminal y verifique:

```bash
go version
```

Debería ver una salida como "go1.21.5 darwin/amd64".

### Instalación en Windows

En sistemas Windows, se recomienda usar el instalador MSI. Visite el sitio web oficial de Go para descargar el instalador MSI para Windows, haga doble clic para ejecutar y siga el asistente de instalación. El instalador configura automáticamente las variables de entorno incluyendo PATH.

Después de la instalación, abra Símbolo del sistema o PowerShell y verifique:

```bash
go version
```

Los usuarios de Windows también pueden usar el administrador de paquetes Chocolatey:

```bash
choco install golang
```

### Usando Gestores de Versiones (Recomendado)

Para desarrolladores que necesitan gestionar múltiples versiones de Go simultáneamente, se recomienda usar un gestor de versiones. Los gestores de versiones comunes incluyen gvm (Go Version Manager) y asdf-go:

Instalación y uso de gvm:

```bash
# Instalar gvm
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# Instalar versión de Go
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdf es un gestor de versiones multilingüe que soporta Go:

```bash
# Instalar plugin de asdf
asdf plugin add golang

# Instalar versión de Go
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## Verificando la Instalación

Después de la instalación, verifique que Go esté correctamente instalado. Abra Terminal o Símbolo del sistema y ejecute:

```bash
# Ver versión de Go
go version

# Ver información del entorno de Go
go env
```

El comando go env produce múltiples variables de entorno incluyendo GOPATH, GOROOT, GOOS, GOARCH, etc. Esta información es importante para entender el funcionamiento de Go y la configuración del entorno.

Ejecute un programa simple para verificar que todo el entorno de desarrollo funcione correctamente:

```bash
# Crear directorio de trabajo
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# Inicializar módulo (Go 1.16+ recomendado usar go mod)
go mod init hello

# Crear programa de prueba
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# Ejecutar programa
go run main.go
```

Si produce "Hello, Go!", el entorno de desarrollo de Go está correctamente configurado.

## Configuración del Entorno

### GOPATH y Módulos

Antes de Go 1.11, Go usaba el modo GOPATH para gestionar dependencias. Desde Go 1.11, Go introdujo soporte para módulos, y usar go mod para gestionar dependencias de proyectos es recomendado. Cuando se usan módulos, no hay necesidad de establecer GOPATH, y los proyectos pueden crearse en cualquier directorio.

Para entornos de intranet empresarial o cuando se usan repositorios privados, puede ser necesario configurar la variable de entorno GOPRIVATE:

```bash
export GOPRIVATE=github.com/company/*
```

### Configuración de Proxy

En regiones como China continental, el acceso directo al repositorio oficial de módulos de Go puede ser lento o no disponible. Puede configurar el proxy de módulos de Go:

```bash
# Usar proxy de Alibaba Cloud
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct

# O usar proxy de Qiniu Cloud
go env -w GOPROXY=https://goproxy.cn,direct
```

### Configuración de IDE

El desarrollo de Go típicamente usa editores como VS Code, GoLand o Sublime Text. VS Code requiere instalar la extensión de Go para obtener una buena experiencia de desarrollo, incluyendo completado de código, depuración, formateo y más.

GoLand es un IDE profesional de Go de JetBrains que proporciona el soporte de desarrollo de Go más completo, incluyendo depuración avanzada, refactorización y pruebas.

## Actualización y Desinstalación

### Actualizando la Versión de Go

Actualizar Go es simple. Descargue el instalador de la nueva versión y reinstale siguiendo los pasos de instalación anteriores. El nuevo instalador sobrescribirá la versión anterior.

Los usuarios de gestores de versiones pueden cambiar fácilmente entre diferentes versiones:

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### Desinstalando Go

En Linux o macOS, desinstalar Go solo requiere eliminar el directorio de instalación:

```bash
# Eliminar directorio de instalación de Go
sudo rm -rf /usr/local/go

# Eliminar de PATH
# Edite ~/.bashrc o ~/.profile y elimine las líneas relacionadas
```

En Windows, desinstale a través de Programas y Características del Panel de Control, o use Chocolatey:

```bash
choco uninstall golang
```
