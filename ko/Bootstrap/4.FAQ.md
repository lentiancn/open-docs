# Bootstrap FAQ

> 자주 묻는 질문

---

## 설치 문제

### CDN이 로드되지 않음?

**해결책**:

1. 네트워크 연결 확인
2. 대안적 CDN 사용:
```html
<link href="https://cdnjs.cloudflare.com/ajax/libs/bootstrap/5.3.0/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdnjs.cloudflare.com/ajax/libs/bootstrap/5.3.0/js/bootstrap.bundle.min.js"></script>
```

### npm 설치 후 스타일이 작동하지 않음?

올바르게 가져오는지 확인:

```javascript
import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap';
```

## 스타일 문제

### 컴포넌트 스타일이 작동하지 않음?

**일반적인 원인**:

1. Bootstrap CSS를 가져오지 않음
2. JavaScript가 올바르게 로드되지 않음 (일부 컴포넌트에 필요)
3. CSS 우선순위가 재정의됨

### 반응형이 작동하지 않음?

viewport 메타 태그가 추가되었는지 확인:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

---

## 컴포넌트 문제

### 드롭다운이 작동하지 않음?

**체크리스트**:

1. Bootstrap JS를 가져옴 (CSS만 아닌)
2. 올바른 HTML 구조 사용
3. JavaScript 오류 확인

### 모달이 열리지 않음?

확인:
1. Bootstrap JS가 로드됨
2. data-bs-toggle 및 data-bs-target 속성이 올바름
3. 페이지 로드 시 모달이 존재함

## 사용자 정의 문제

### 테마 색상을 사용자 정의하는 방법?

**방법 1: Sass 사용**

```scss
$primary: #ff6600;
@import "bootstrap";
```

**방법 2: CSS 변수 사용**

```css
:root {
  --bs-primary: #ff6600;
}
```

## 호환성 문제

### IE 11 지원 안 함?

Bootstrap 5는 더 이상 IE 11을 지원하지 않습니다. IE 11 지원이 필요하면 Bootstrap 4를 사용하세요.

---

## 성능 문제

### 파일 크기를 줄이는 방법?

1. 필요한 컴포넌트만 가져옴 (Sass)
2. 프로덕션 버전 사용 (.min.css/.min.js)
3. gzip 압축 활성화

---

## 다른 프레임워크와의 충돌

### Vue/React와 함께 사용?

Bootstrap은 모든 JavaScript 프레임워크와 함께 사용할 수 있지만, 주의:

1. 일부 컴포넜트 jQuery에 의존 (Bootstrap 5는 더 이상 의존 안 함)
2. 프레임워크별 Bootstrap 래퍼 라이브러리 사용 (예: react-bootstrap)
