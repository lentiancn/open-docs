# OpenClaw よくある質問

このサーバーでは、OpenClawについての一般的な質問に回答します。

## 基本的な質問

### OpenClawとは何ですか？

OpenClawは、チャットアプリ（WhatsApp、Telegram、Discordなど）をAIコーディングアシスタントに接続する自己ホスト型ゲートウェイです。データを完全に管理しながら、どこからでもメッセージングアプリ経由でAIアシスタントとコミュニケーションできます。

### OpenClawは有料ですか？

いいえ、OpenClawはMITライセンスの下で公開されているオープンソースプロジェクトです。ただし、以下の費用は自分で負担する必要があります：
- 実行所需的サーバーまたはコンピュータ
- AIサービスのAPIキー費用

### AIサービスのサブスクリプションは必要ですか？

OpenClawは複数のモデルプロバイダーをサポートしています：
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- その他OpenAI API互換サービス

従量制のAPIキーまたはサブスクリプション認証を使用できます。

### OpenClawはどのプラットフォームをサポートしていますか？

- **デスクトップ**：macOS、Linux、Windows
- **モバイル**：iOS、Android（ノードとして）
- **チャットアプリ**：WhatsApp、Telegram、Discord、iMessage、Signal、Slackなど20以上

## インストール

### インストールに失敗した場合は？

1. Node.jsバージョンを確認：`node --version`（22以上が必要）
2. 診断を実行：`openclaw doctor`
3. ログを確認：`openclaw logs`

### Raspberry Piで実行できますか？

はい、OpenClawはRaspberry Piで実行できます。Raspberry Pi 4以降とRaspberry Pi OSまたはUbuntuをお勧めします。

### Macが必要ですか？

いいえ、OpenClawはNode.jsをサポートする任意のシステムで実行できます。iMessageチャンネルのみ、Appleの制限によりMacが必要です。

## 設定

### 設定ファイルはどこにありますか？

デフォルトの場所：`~/.openclaw/openclaw.json`

### メッセージを送信できる人を制限するには？

`dmPolicy`で設定：

- `pairing`：不明な送信者はペアリングコードが必要
- `allowlist`：許可リスト上のユーザーのみ
- `open`：すべてのDMを許可
- `disabled`：DMを無効

### グループチャットを設定するには？

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## 使用

### 新しいチャットチャンネルを追加するには？

```bash
openclaw channels login whatsapp
```

### 複数のチャンネルを同時に使用できますか？

はい、1つのGatewayは複数のチャンネルに同時に接続できます。

### AIに以前の会話を記憶させるには？

OpenClawには、セッションコンテキストを自動的に保存する組み込みメモリがあります。設定でセッション保持ポリシーを調整できます。

### 複数のエージェントを実行できますか？

はい、ルーティングルールで複数のエージェントを設定できます：

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## セキュリティ

### データは安全ですか？

はい。OpenClawは自己ホスト型で、すべてのデータはあなたのデバイスに保存され、サードパーティのサーバーに送信されません（AIプロバイダーのAPIを除く）。

### Gatewayを保護するには？

1. `allowFrom`を設定してアクセス元を制限
2. 強力なパスワードとAPIキーを使用
3. OpenClawを定期的に更新
4. ファイアウォールでポートアクセスを制限

## トラブルシューティング

### Gatewayが起動しない

1. ポートが使用されていないか確認：`openclaw gateway status`
2. エラーログを確認：`openclaw logs`
3. 診断を実行：`openclaw doctor`

### AIが返信しない

1. APIキーが正しく設定されているか確認
2. モデルプロバイダーアカウントに十分な残高があるか確認
3. ログのエラーメッセージを確認

### メッセージ送信に失敗する

1. チャンネルがログインしているか確認
2. 対象のユーザーまたはグループが存在するか確認
3. 詳細なエラーログを確認
