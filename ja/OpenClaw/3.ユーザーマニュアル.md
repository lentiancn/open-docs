# OpenClaw ユーザーマニュアル

このマニュアルでは、OpenClawの日常的な使用方法と主要機能を説明します。

## 基本概念

### Gateway

GatewayはOpenClawのコアコンポーネントで、セッション管理、メッセージルーティング、チャンネル接続を担当します。単一のGatewayプロセスを実行するだけで、すべてのチャンネルの接続要求が自動的に処理されます。

### チャンネル（Channels）

チャンネルは、AIアシスタントとコミュニケーションを取るために使用するチャットアプリです。WhatsApp、Telegram、Discord、iMessageなど、複数のチャンネルを同時に接続できます。

### エージェント（Agents）

エージェントはAIアシスタントのインスタンスです。各エージェントには、独立したワークスペースとセッションがあります。複数のエージェントを設定して、それぞれ異なるタスクを処理できます。

## 日常操作

### Gatewayの起動と停止

```bash
# Gatewayを起動（バックグラウンドサービス）
openclaw gateway start

# Gatewayを停止
openclaw gateway stop

# Gatewayを再起動
openclaw gateway restart

# 状態を確認
openclaw gateway status

# フォアグラウンドで実行（デバッグ用）
openclaw gateway --port 18789
```

### メッセージの送信

```bash
# ダイレクトメッセージを送信
openclaw message send --target +15555550123 --message "こんにちは"

# グループに送信
openclaw message send --target "グループ名" --message "大家好"
```

### 設定の管理

```bash
# 現在の設定を表示
openclaw config get

# 設定値を設定
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# 設定値を削除
openclaw config unset channels.whatsapp.enabled
```

## 設定管理

### 設定ファイルの場所

設定ファイルは `~/.openclaw/openclaw.json` にあります。

### ホットリロード

OpenClawは設定のホットリロードをサポートしています。設定ファイルを変更してもGatewayを再起動する必要はありません。ほとんどの設定変更は自動的に有効になります。

### 一般的な設定例

#### 許可されたメッセージソースの設定

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

#### グループチャットのメンションルール設定

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

#### 複数のエージェントの設定

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## チャンネル管理

### 新しいチャンネルの追加

```bash
# WhatsAppにログイン
openclaw channels login whatsapp

# Telegramにログイン
openclaw channels login telegram
```

### 接続されたチャンネルの表示

```bash
openclaw channels list
```

## 高度な機能

### Cronジョブ

スケジュールされたタスクを設定：

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

###  Heartbeat

定期チェックを設定：

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### サンドボックスモード

分離されたDockerコンテナでエージェントを実行：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

HTTP webhookエンドポイントを設定：

```json5
{
  hooks: {
    enabled: true,
    token: "your-secret-token",
    path: "/hooks"
  }
}
```

```bash
openclaw logs -f
```

### ヘルスチェック

```bash
openclaw health
```

### 問題の診断

```bash
openclaw doctor
```

## コントロールUI

OpenClawはWebベースのコントロールUIを提供しており、以下が可能です：

- メッセージの送受信
- 設定の管理
- セッション履歴の表示
- モバイルノードの管理
- Gatewayの状態監視

アクセス先：`http://127.0.0.1:18789/`
