# Preguntas Frecuentes sobre Docker

Esta página lista problemas comunes y soluciones para la instalación y uso de Docker.

## Problemas de Instalación

### Docker Desktop no inicia

**Síntomas**: Docker Desktop no inicia o se cierra inmediatamente después de iniciar

**Soluciones**:

1. Asegúrate de que el sistema cumpla con los requisitos de hardware:
   - Windows: Activa Hyper-V o instala WSL 2
   - Mac: Asegúrate de no usar modelos de Mac obsoletos

2. Restablecer Docker Desktop:
   - Windows: Clic derecho en el icono de Docker → Solucionar problemas → Restablecer a valores de fábrica
   - Mac: Docker → Solucionar problemas → Restablecer a valores de fábrica

3. Verifica los registros del sistema:
   - Windows: Visor de eventos → Registros de Windows → Aplicación
   - Mac: Abre la consola y busca errores de Docker

### Sin permiso para comandos Docker

**Síntomas**: "permission denied while trying to connect to the Docker daemon socket"

**Soluciones**:

1. Añade tu usuario al grupo Docker:
   ```bash
   sudo usermod -aG docker $USER
   ```

2. Cierra sesión y vuelve a iniciar

3. O usa sudo antes de los comandos Docker

### Error de instalación de WSL 2 en Windows

**Síntomas**: "WSL 2 installation is incomplete"

**Soluciones**:

1. Activa el Subsistema de Windows para Linux:
   ```bash
   wsl --install
   ```

2. Reinicia tu computadora

3. Actualiza WSL a la última versión:
   ```bash
   wsl --update
   ```

## Problemas de Uso

### El contenedor no inicia

**Síntoma**: El contenedor permanece en estado "Exited" o "Created"

**Soluciones**:

1. Verifica los logs del contenedor:
   ```bash
   docker logs nombre_contenedor
   ```

2. Asegúrate de que todos los puertos requeridos estén disponibles

3. Verifica la integridad de la imagen

### Puerto ya en uso

**Síntoma**: "port is already allocated"

**Soluciones**:

1. Encuentra el puerto en uso:
   ```bash
   netstat -ano | findstr :8080
   ```

2. Detén el otro servicio o usa otro puerto

3. Cambia la redirección de puertos:
   ```bash
   docker run -p 8081:80 nginx
   ```

### Problemas de espacio en disco

**Síntoma**: "no space left on device"

**Soluciones**:

1. Elimina contenedores sin usar:
   ```bash
   docker container prune
   ```

2. Elimina imágenes sin usar:
   ```bash
   docker image prune -a
   ```

3. Elimina volúmenes sin usar:
   ```bash
   docker volume prune
   ```

### Problemas de red

**Síntoma**: El contenedor no puede conectarse al exterior

**Soluciones**:

1. Verifica la configuración DNS:
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

2. Restablece la red Docker:
   ```bash
   docker network prune
   ```

3. Verifica la configuración del firewall

## Problemas de Rendimiento

### Rendimiento lento del contenedor

**Soluciones**:

1. Verifica los límites de recursos:
   ```bash
   docker stats
   ```

2. Asigna más RAM (Docker Desktop → Configuración → Recursos)

3. Usa SSD en lugar de HDD para Docker

### Alto uso de disco

**Soluciones**:

1. Limpia la caché de construcción regularmente:
   ```bash
   docker builder prune
   ```

2. Elimina versiones antiguas de imágenes

3. Limita el tamaño de los logs

## Preguntas de Seguridad

### Ejecutar contenedores como root

**Recomendación**:

Usa un usuario no privilegiado:

```dockerfile
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser
USER appuser
```

### Vulnerabilidades en imágenes

**Soluciones**:

1. Usa imágenes oficiales y actualizadas

2. Escanea imágenes regularmente:
   ```bash
   docker scan nombre_imagen
   ```

3. Considera usar imágenes distroless

## Preguntas Avanzadas

### Usar Docker en CI/CD

**Recomendaciones**:

1. Usa Build Kits para compilaciones paralelas
2. Usa compilaciones de múltiples etapas para imágenes más pequeñas
3. Caché de dependencias efectivamente
4. Usa Docker Buildx para compilaciones multiplataforma

### Persistencia de datos

**Recomendaciones**:

1. Usa volúmenes nombrados para datos críticos
2. Haz copia de seguridad de los volúmenes regularmente
3. Usa montajes de enlace para desarrollo

### Múltiples entornos

**Recomendaciones**:

1. Usa Docker Compose para variables de entorno
2. Utiliza archivos .env
3. Considera Docker Swarm o Kubernetes para producción
