# 설치 가이드

이 문서에서는 Apache APISIX의 다양한 설치 방법을 설명하며, 빠르게 API 게이트웨이 환경을 설정할 수 있도록 도와줍니다.

## 사전 요구사항

APISIX를 설치하기 전에 다음 요구사항을 충족하는지 확인하세요:

- **운영 체제**: Linux (권장: Ubuntu 20.04+, CentOS 7+, Debian 11+)
- **Docker** (Docker 배포의 경우): Docker 및 Docker Compose 설치됨
- **etcd**: APISIX는 설정 저장을 위해 etcd를 사용하며, etcd 3.4.0+가 필요합니다
- **Perl**: 일부 종속성에는 Perl 5.5.32+가 필요합니다
- **curl**: 설치 확인용

## Docker 빠른 설치 (권장)

가장 빠른 설치 방법으로, 개발 및 테스트 환경에 적합합니다.

### 1단계: 빠른 시작 디렉토리 생성

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### 2단계: 빠른 시작 스크립트 실행

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

이 명령은 apisix-quickstart 및 etcd 컨테이너를 자동으로 시작합니다. APISIX는 etcd를 사용하여 설정을 저장하고 동기화합니다. 컨테이너는 Docker 호스트 네트워크 모드를 사용하므로 localhost에서 직접 접근할 수 있습니다.

### 3단계: 설치 확인

설치가 성공하면 터미널에 다음이 출력됩니다:

```
✔ APISIX가 준비되었습니다!
```

다음 명령으로 APISIX가 실행 중인지 확인할 수 있습니다:

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

다음과 같이 반환됩니다:

```
Server: APISIX/버전
```

여기서 Version은 설치된 APISIX 버전입니다 (예: APISIX/3.15.0).

### 4단계: 대시보드 접근

APISIX는 내장 대시보드 UI를 제공하며, 다음에서 접근할 수 있습니다:

```
http://127.0.0.1:9180/ui
```

> ⚠️ 참고: 더 나은 경험을 위해, 관리 API는 기본적으로 권한이 없습니다. 프로덕션 환경에서는 권한을 활성화하세요.

## Docker Compose 설치

사용자 정의 구성의 경우 Docker Compose 배포를 사용합니다.

### 1단계: 구성 파일 생성

docker-compose.yaml 파일을 생성합니다:

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=your-admin-key
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### 2단계: 서비스 시작

```bash
docker-compose up -d
```

### 3단계: 서비스 확인

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Kubernetes 설치

APISIX는 Kubernetes에서 직접 배포할 수 있는 공식 Ingress Controller를 제공합니다.

### Helm Chart를 사용하여 설치

```bash
# APISIX Helm 저장소 추가
helm repo add apisix https://charts.apiseven.com
helm repo update

# APISIX 설치
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### kubectl을 사용하여 설치

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## 소스 컴파일 설치

사용자 정의 빌드가 필요한 프로덕션 환경용입니다.

### 1단계: 종속성 설치

**Ubuntu/Debian:**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL:**

```bash
sudo yum install -y perl gcc git curl
```

### 2단계: OpenResty 설치

```bash
# OpenResty 저장소 추가
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# OpenResty 설치
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### 3단계: APISIX 설치

```bash
# 소스 코드 복제
git clone https://github.com/apache/apisix.git
cd apisix

# 종속성 설치
make deps

# 설정 초기화
make init

# APISIX 시작
make run
```

## 설치 확인

설치 방법에 관계없이 다음으로 확인합니다:

1. **프로세스 상태 확인**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: your-admin-key"
```

2. **테스트 라우트 생성**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "test-route",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **라우트 테스트**

```bash
curl "http://127.0.0.1:9080/test"
```

## 제거

**Docker:**

```bash
# 컨테이너 중지
docker-compose down

# 볼륨 삭제 (필요한 경우)
docker-compose down -v
```

**Kubernetes:**

```bash
helm uninstall apisix
```
