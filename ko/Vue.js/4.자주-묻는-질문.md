# Vue.js 자주 묻는 질문

## Vue 2와 Vue 3의 차이점

Vue 3는 현재 최신 주요 버전입니다. Vue 2에는 없는 새로운 기능(예: Teleport, Suspense, 템플릿당 여러 루트 요소)을 포함하며, Vue 2와 호환되지 않는 변경사항도 포함합니다.

**주요 장점:**
- 더 작은 번들 크기
- 더 나은 성능
- 더 나은 확장성
- 더 나은 TypeScript / IDE 지원

**권장 사항:**
- **새 프로젝트는 Vue 3를 사용해야 합니다**
- **IE11 지원이 필요한 경우에만 Vue 2를 고려하세요** (Vue 3는 IE11을 지원하지 않음)

**마이그레이션 정보:**
- 컴포지션 API는 원래 Vue 3 전용 기능이었지만, 이제 Vue 2.7에서 사용 가능합니다
- 자세한 마이그레이션 가이드: [Vue 3 마이그레이션 가이드](https://v3-migration.vuejs.org/)

## Vue 2는 여전히 지원되나요?

- Vue 2.7(2022년 7월 출시)는 Vue 2 버전 범위의 마지막 마이너 릴리스입니다
- **Vue 2는 2023년 12월 31일에 수명 종료되었습니다**
- 규정 준수 요구사항이 있는 팀을 위한 확장 지원 제공: [Vue 2 확장 LTS](https://v2.vuejs.org/lts/)

## Vue는 어떤 브라우저를 지원하나요?

- **Vue 3.x**: 네이티브 ES2016 지원 브라우저만 지원 (IE11 제외)
- **Vue 2.x**: IE11 및 더 넓은 브라우저 호환성 지원

## Vue는 신뢰할 수 있나요?

- Vue는 성숙하고 검증된 프레임워크입니다
- 전 세계적으로 **150만 명 이상의 사용자**
- npm에서 월간 **1,000만 회 이상 다운로드**
- 전 세계 유명 기관에서 프로덕션 사용: Wikimedia Foundation, NASA, Apple, Google, Microsoft, GitLab, Zoom, Tencent, Weibo, Bilibili, Kuaishou 등

## Vue는 빠른가요?

- Vue 3는 가장 성능이 뛰어난 주류 프론트엔드 프레임워크 중 하나입니다
- 대부분의 웹 애플리케이션 사용 사례를 수동 최적화 없이 쉽게 처리합니다
- [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html)에서 React 및 Angular보다 우수한 성능
- 공식 문서 사이트는 느린 4G 네트워크에서 4배 CPU 제한이 있는 Moto G4에서도 성능 점수 100을 달성합니다

## Vue는 가벼운가요?

- **최소 앱** (빌드 도구 + tree-shaking): brotli 압축 기준 약 16kb
- **완전한 기능 앱**: 약 27kb
- **빌드 도구 없음** (컴파일러 포함): 약 41kb
- **점진적 향상 시나리오**: [petite-vue](https://github.com/vuejs/petite-vue) (약 6kb) 고려

## Vue는 대규모 시나리오에 적합한가요?

네. Vue는 단순한 사용 사례에만 적합하다는 일반적인 오해와 달리, 대규모 애플리케이션을 완벽하게 처리할 수 있습니다:

- **싱글 파일 컴포넌트**: 모듈화된 개발 모델 제공
- **컴포지션 API**: 복잡한 로직을 구성, 추출 및 재사용하기 위한 깨끗한 패턴 제공
- **포괄적인 도구 지원**: 애플리케이션이 성장함에 따라 원활한 개발 경험 보장
- **낮은 진입 장벽**: 새로운 개발자의 온보딩 및 교육 비용 감소

## 오픈 소스 라이선스

Vue는 **MIT 라이선스**로 배포되며, 완전히 무료이고 오픈 소스입니다.

## API 스타일 선택

### 옵션 API vs 컴포지션 API

**옵션 API가 적합한 경우:**
- 초보자 또는 OOP 언어 배경을 가진 개발자
- 빌드 도구를 사용하지 않거나 점진적 향상과 같은 저복잡도 시나리오
- 간단한 프로젝트

**컴포지션 API가 적합한 경우:**
- Vue로 전체 애플리케이션 구축 계획
- 더 나은 논리 구성 및 재사용 필요
- TypeScript 프로젝트

## JavaScript vs TypeScript

- Vue 자체는 TypeScript로 구현되어 있으며 일류 TypeScript 지원 제공
- **TypeScript 사용을 강제하지 않음**
- TypeScript를 염두에 두고 설계된 API는 IDE 및 정적 분석 도구가 이해하기 쉬움
- 선택은 팀 배경 및 프로젝트 규모에 따라 달라짐

## Vue와 Web Components 비교

- Vue는 Web Components가 네이티브로 사용 가능하기 전에 생성됨
- Web Components 사양은 상대적으로 저수준 (사용자 정의 요소 중심)
- Vue는 효율적인 DOM 렌더링, 반응성 상태 관리, 도구 체인, 클라이언트 라우팅, 서버 사이드 렌더링 등 더 많은 상위 문제를 해결
- Vue는 네이티브 사용자 정의 요소를 Vue 컴포넌트 내에서 사용하는 것을 완전히 지원
- Vue 컴포넌트를 네이티브 사용자 정의 요소로 내보내는 것도 지원
