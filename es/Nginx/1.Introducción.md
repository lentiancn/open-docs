# Introducción a Nginx

## Qué es Nginx

Nginx (pronunciado "engine x") es un servidor web HTTP, proxy inverso, caché, balanceador de carga, servidor proxy TCP/UDP y proxy de correo.

Nginx fue escrito originalmente por Igor Sysoev y distribuido bajo la licencia BSD de 2 cláusulas.

## Características de Nginx

Nginx es conocido por su flexibilidad y alto rendimiento con bajo consumo de recursos:

- **El servidor web más popular del mundo**
- **Una de las imágenes Docker más populares**
- **Potencia múltiples Controladores de Ingress para Kubernetes**, incluyendo el Ingress Controller oficial de Kubernetes

### Funciones básicas del servidor HTTP

- Servicio de archivos estáticos e índices
- Capacidad de autoindexación
- Caché de descriptores de archivos abiertos
- Proxy inverso acelerado con caché
- Balanceo de carga y tolerancia a fallos
- Soporte acelerado con caché para servidores FastCGI, uwsgi, SCGI y memcached
- Arquitectura modular
- Filtros: compresión gzip, rangos de bytes, respuestas fragmentadas, XSLT, SSI y transformación de imágenes
- Soporte SSL y TLS SNI
- Soporte para HTTP/2 con priorización basada en peso y dependencias
- Soporte para HTTP/3

### Otras funciones del servidor HTTP

- Servidores virtuales basados en nombres y direcciones IP
- Soporte para conexiones keep-alive y pipelined
- Formatos de log de acceso, escritura de logs bufferizada, rotación rápida de logs y logging syslog
- Redirección de códigos de error 3xx-5xx
- Módulo de reescritura: cambio de URI usando expresiones regulares
- Control de acceso basado en dirección IP del cliente, contraseña (autenticación HTTP básica) y resultado de subrequest
- Validación de referer HTTP
- Métodos PUT, DELETE, MKCOL, COPY y MOVE
- Streaming FLV y MP4
- Limitación de tasa de respuesta
- Limitación de conexiones o solicitudes simultáneas desde una dirección
- Geolocalización basada en IP
- Pruebas A/B
- Espejo de solicitudes
- Perl embebido
- Lenguaje de scripting njs

### Funciones del servidor proxy de correo

- Redirección de usuarios a servidores IMAP o POP3 usando servidor de autenticación HTTP externo
- Autenticación de usuarios usando servidor de autenticación HTTP externo y redirección a servidor SMTP interno
- Métodos de autenticación: POP3 (USER/PASS, APOP, AUTH LOGIN/PLAIN/CRAM-MD5), IMAP (LOGIN, AUTH LOGIN/PLAIN/CRAM-MD5), SMTP (AUTH LOGIN/PLAIN/CRAM-MD5)
- Soporte SSL
- Soporte STARTTLS y STLS

### Funciones del servidor proxy TCP/UDP

- Proxy genérico de TCP y UDP
- Soporte SSL y TLS SNI para TCP
- Balanceo de carga y tolerancia a fallos
- Control de acceso basado en dirección del cliente
- Limitación de conexiones simultáneas desde una dirección
- Formatos de log de acceso, escritura de logs bufferizada, rotación rápida de logs y logging syslog
- Geolocalización basada en IP
- Pruebas A/B
- Lenguaje de scripting njs

### Arquitectura y escalabilidad

- Un proceso maestro y varios procesos worker
- Los procesos worker se ejecutan bajo un usuario sin privilegios
- Configuración flexible
- Reconfiguración y actualización del ejecutable sin interrumpir el servicio al cliente
- Soporte para kqueue (FreeBSD 4.1+), epoll (Linux 2.6+), /dev/poll (Solaris 7 11/99+), event ports (Solaris 10), select y poll
- Soporte para diversas características de kqueue
- Soporte para diversas características de epoll
- Soporte para sendfile, sendfile64 y sendfilev
- Soporte para File AIO
- Soporte para DIRECTIO
- Soporte para Accept-filters y TCP_DEFER_ACCEPT
- 10,000 conexiones HTTP keep-alive inactivas ocupan aproximadamente 2.5M de memoria
- Las operaciones de copia de datos se mantienen al mínimo

## Ventajas de Nginx

### Alto rendimiento

Nginx utiliza un modelo asíncrono basado en eventos, capaz de manejar conexiones de alta simultaneidad mientras mantiene un bajo consumo de memoria.

### Bajo consumo de recursos

En comparación con otros servidores web, Nginx consume significativamente menos memoria y CPU bajo los mismos niveles de simultaneidad.

### Alta estabilidad

La estabilidad de Nginx ha sido demostrada por millones de sitios web en todo el mundo, con muy pocos bloqueos.

### Configuración simple

Los archivos de configuración de Nginx son concisos y fáciles de entender.

### Gran extensibilidad

Soporta módulos ricos, permitiendo habilitar o deshabilitar funciones según sea necesario.

### Despliegue en caliente

Recargue la configuración y actualice versiones sin interrumpir el servicio.

## Casos de uso

Nginx se usa ampliamente en los siguientes escenarios:

- **Servidor web**: Como servidor HTTP para archivos estáticos
- **Proxy inverso**: Como proxy inverso para servidores backend
- **Balanceador de carga**: Como balanceador de carga para múltiples servidores backend
- **Servidor de caché**: Como CDN o caché interno
- **API Gateway**: Como gateway de entrada para microservicios
- **Proxy de correo**: Como proxy POP3, IMAP y SMTP
- **Proxy TCP/UDP**: Como proxy para protocolos TCP y UDP
