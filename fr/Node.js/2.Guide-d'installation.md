# Guide d'installation de Node.js

## Méthodes d'installation officielles

Node.js propose plusieurs méthodes d'installation officielles pour s'adapter à différents systèmes d'exploitation et préférences utilisateur. Toutes les méthodes officielles garantissent que vous obtenez les binaires authentiques et non modifiés directement depuis le projet Node.js.

### Télécharger depuis le site web officiel

Le moyen le plus simple d'installer Node.js est de télécharger l'installeur approprié depuis le [site web officiel de Node.js](https://nodejs.org/fr/download).

**Versions disponibles :**
- **LTS (Support à Long Terme)** : Recommandé pour la plupart des utilisateurs. Offre stabilité et support à long terme.
- **Current** : Dernières fonctionnalités avec un cycle de support plus court. Convient au développement et aux tests.

### Gestionnaires de paquets

#### macOS (Homebrew)
```bash
# Installer Homebrew si ce n'est pas déjà fait
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Installer la version LTS
brew install node@lts

# Ou installer la dernière version Current
brew install node
```

#### Linux (Diverses distributions)

**Ubuntu/Debian :**
```bash
# Utiliser le dépôt NodeSource (recommandé)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora :**
```bash
# Utiliser le dépôt NodeSource
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# ou pour les versions plus récentes : sudo dnf install nodejs
```

#### Windows
- Télécharger l'installeur Windows (.msi) depuis le site web officiel
- Exécuter l'installeur et suivre l'assistant de configuration
- Node.js et npm seront automatiquement ajoutés à votre PATH

### Vérification

Après l'installation, vérifiez que Node.js et npm sont correctement installés :

```bash
# Vérifier la version de Node.js
node --version

# Vérifier la version de npm  
npm --version

# Tester la fonctionnalité de base
node -e "console.log('Bonjour Node.js!')"
```

### Gestion de plusieurs versions de Node.js

Pour les développeurs qui doivent travailler avec différentes versions de Node.js dans différents projets, les gestionnaires de versions offrent une excellente solution.

#### nvm (Node Version Manager) - macOS/Linux
```bash
# Installer nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Installer et utiliser la version LTS
nvm install --lts
nvm use --lts

# Installer une version spécifique
nvm install 20.14.0
nvm use 20.14.0

# Lister les versions disponibles
nvm list-remote
```

#### nvm-windows - Windows
- Télécharger depuis [nvm-windows GitHub Releases](https://github.com/coreybutler/nvm-windows/releases)
- Suivre les instructions d'installation
- Utiliser des commandes similaires à nvm Unix

### Variables d'environnement

Assurez-vous que les fichiers exécutables de Node.js et npm ont été ajoutés à la variable d'environnement PATH du système.

- **Windows** : Généralement ajouté automatiquement au PATH
- **macOS/Linux** : Si vous utilisez un gestionnaire de paquets, cela est généralement configuré automatiquement

Si la commande n'est pas trouvée, ajoutez-la manuellement au PATH :
```bash
# Généralement situé dans /usr/local/bin ou ~/.nvm/versions/node/vXX.XX.XX/bin
export PATH=$PATH:/usr/local/bin
```

### Dépannage

#### Problèmes de permissions
Sous Linux/macOS, si vous rencontrez des problèmes de permissions lors de l'installation globale de packages npm, nous recommandons :
- Utiliser nvm pour gérer Node.js
- Ou configurer le répertoire global de npm :
  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  export PATH=~/.npm-global/bin:$PATH
  ```

#### Problèmes réseau
Si la vitesse de téléchargement est lente ou si aucune connexion n'est possible, vous pouvez configurer npm pour utiliser un miroir local :
```bash
# Configurer le miroir Taobao (Chine)
npm config set registry https://registry.npmmirror.com

# Ou utiliser temporairement
npm install --registry https://registry.npmmirror.com
```
