# Manual de Usuario de Hadoop

## Guía de Operaciones HDFS

### Operaciones Básicas del Sistema de Archivos

HDFS proporciona una interfaz de línea de comandos similar a los sistemas de archivos tradicionales, permitiendo a los usuarios realizar varias operaciones de archivos a través del comando hdfs dfs. Estos comandos son muy similares a los comandos de shell de Linux, por lo que los usuarios familiarizados con sistemas Linux pueden adaptarse rápidamente. Comprender estas operaciones básicas es fundamental para usar Hadoop; a continuación detallamos los comandos HDFS más utilizados.

Crear directorios usa el comando mkdir, que puede crear directorios de un solo nivel o múltiples niveles:

```bash
# Crear directorio de un nivel
hdfs dfs -mkdir /user/data

# Crear directorio de múltiples niveles
hdfs dfs -mkdir -p /user/data/input

# Listar contenido del directorio
hdfs dfs -ls /user/data

# Listar contenido del directorio recursivamente
hdfs dfs -ls -R /user/data
```

La carga y descarga de archivos son operaciones comunes:

```bash
# Subir archivo local a HDFS
hdfs dfs -put localfile.txt /user/data/

# Subir y renombrar
hdfs dfs -put localfile.txt /user/data/newname.txt

# Descargar archivo HDFS a local
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# Copiar archivos internos de HDFS
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# Mover archivos internos de HDFS
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

Visualización y eliminación de archivos:

```bash
# Ver contenido del archivo
hdfs dfs -cat /user/data/file.txt

# Ver las últimas líneas del archivo
hdfs dfs -tail /user/data/file.txt

# Ver tamaño del archivo
hdfs dfs -du /user/data/file.txt

# Eliminar archivo
hdfs dfs -rm /user/data/file.txt

# Eliminar directorio recursivamente
hdfs dfs -rm -r /user/data/dir
```

### Gestión de Permisos

El modelo de permisos de HDFS es similar a Unix/Linux, basado en un sistema de tres niveles: usuario, grupo y otros. Cada archivo y directorio tiene tres permisos: lectura (r), escritura (w) y ejecución (x), correspondientes a propietario, grupo y otros. Comprender la gestión de permisos es crucial para proteger la seguridad de los datos.

```bash
# Cambiar propietario del archivo
hdfs dfs -chown user:group /user/data/file.txt

# Cambiar permisos del archivo
hdfs dfs -chmod 755 /user/data/file.txt

# Cambiar grupo propietario
hdfs dfs -chgrp groupname /user/data/file.txt
```

### Replicación y Balanceo de Datos

HDFS replica cada bloque de datos tres veces por defecto, almacenándolos en diferentes DataNodes. El factor de replicación predeterminado se puede modificar a través de la propiedad dfs.replication:

```bash
# Establecer factor de replicación para archivo específico
hdfs dfs -setrep -w 2 /user/data/file.txt

# Ver información de ubicación de bloques del archivo
hdfs fsck /user/data/file.txt -files -blocks -locations
```

Cuando se agregan o eliminan nodos del clúster, los datos pueden distribuirse de manera desigual. Use el comando rebalance para balancear los datos:

```bash
start-balancer.sh -threshold 10
```

El parámetro threshold representa el porcentaje máximo de diferencia entre el uso del disco del nodo y el promedio del clúster. El proceso de balanceo consume ancho de banda de red; se recomienda ejecutar durante períodos de bajo tráfico.

### Interfaz Web de HDFS

HDFS proporciona una interfaz web con múltiples funciones, accesible a través del navegador en el puerto 9870 del NameNode. La interfaz web puede explorar directorios del sistema de archivos, ver contenido de archivos, monitorear el estado de DataNodes, ver información de bloques y más. La interfaz web es una herramienta importante para el personal de operaciones en el monitoreo diario y resolución de problemas.

Las principales características de la interfaz web incluyen: explorar el árbol de directorios de HDFS, ver listas de archivos y propiedades de archivos; ver la lista de DataNodes y uso de almacenamiento de cada DataNode; ver distribución de bloques y estado de replicación; ver información de tiempo de ejecución y registros del NameNode; realizar operaciones simples de archivos como descarga y eliminación.

## Programación MapReduce

### Ejemplo de WordCount

WordCount es el ejemplo introductorio para la programación MapReduce, que cuenta las apariciones de cada palabra en un texto. Comprender este ejemplo ayuda a dominar el modelo de programación MapReduce. El programa se divide en fases Map y Reduce: la fase Map divide el texto en palabras, y la fase Reduce agrega los recuentos de las mismas palabras.

Código de ejemplo de la fase Map:

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Código de ejemplo de la fase Reduce:

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### Envío de Trabajos

Después de empaquetar MapReduce en un archivo JAR, envíe el trabajo usando el comando hadoop jar:

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

Después del envío del trabajo, puede ver el progreso y resultados del trabajo en la interfaz web de YARN (puerto 8088). Cuando el trabajo se completa, emite estadísticas del trabajo incluyendo ID del trabajo, número de tareas, tiempo de ejecución, etc.

### Configuración del Trabajo

Se pueden configurar varios parámetros del trabajo a través del objeto Configuration:

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

Los parámetros de configuración comunes incluyen: mapreduce.job.reduces establece el número de tareas Reduce; mapreduce.map.memory.mb y mapreduce.reduce.memory.mb establecen la memoria del contenedor; mapreduce.map.cpu.vcores y mapreduce.reduce.cpu.vcores establecen los núcleos de CPU del contenedor.

## Operaciones YARN

### Ver Recursos del Clúster

YARN proporciona ricas herramientas de línea de comandos para administrar recursos del clúster:

```bash
# Ver estado de todos los nodos
yarn node -list -all

# Ver lista de aplicaciones
yarn application -list

# Ver estado de aplicación específica
yarn application -status application_1234567890123_0001

# Ver registros de aplicación
yarn logs -applicationId application_1234567890123_0001
```

### Gestión de Colas

YARN usa colas para administrar la asignación de recursos. Las colas se pueden configurar a través del Programador de Capacidad o el Programador Justo:

```bash
# Ver información de cola
yarn queue -status default

# Ver capacidad de cola
yarn resourcemanager -refreshQueues
```

### Configuración del Programador de Recursos

El Programador de Capacidad es el programador predeterminado, que soporta uso compartido de recursos multiinquilino. Configure en yarn-site.xml:

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## Optimización del Rendimiento

### Ajuste de MapReduce

El ajuste del rendimiento de trabajos MapReduce involucra múltiples aspectos. Primero, establezca razonablemente el número de tareas Map y Reduce. El número de tareas Map generalmente está determinado por el número de divisiones de datos de entrada, que se puede ajustar a través del parámetro mapreduce.input.fileinputformat.split.maxsize. El número de tareas Reduce se recomienda establecer entre 0.9 y 1.2 veces el número total de ranuras Reduce del clúster.

```bash
# Establecer número de tareas Map
-D mapreduce.job.maps=10

# Establecer número de tareas Reduce
-D mapreduce.job.reduces=5
```

La optimización de la fase de barajado tiene un impacto significativo en el rendimiento general. Ajuste estos parámetros:

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### Ajuste de HDFS

La optimización del rendimiento de HDFS se centra principalmente en la eficiencia de lectura/escritura de datos. Para archivos grandes, ajuste el tamaño de bloque para equilibrar la localidad de datos y la recuperación de fallos:

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

Habilite lecturas de acceso directo para permitir que los clientes lean datos directamente de DataNodes locales, reduciendo la transferencia de red:

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### Ajuste de YARN

La asignación de recursos de YARN tiene un impacto significativo en la utilización del clúster. Establezca razonablemente la capacidad de recursos de NodeManager:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

Establezca núcleos de CPU virtuales para aprovechar mejor los procesadores multinúcleo:

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## Importación y Exportación de Datos

### Importar Datos desde Bases de Datos

Use Sqoop para migrar convenientemente datos entre bases de datos relacionales y HDFS:

```bash
# Importar tabla completa
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Importar y crear tabla Hive
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### Exportar Datos a Bases de Datos

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### Selección de Formato de Datos

Diferentes formatos de datos tienen diferentes impactos en el rendimiento. Los formatos comúnmente usados incluyen: TextFile (buena generalidad, pero rendimiento promedio); SequenceFile (bueno para MapReduce); Parquet (almacenamiento columnar, bueno para consultas analíticas); ORC (almacenamiento columnar optimizado para Hive). Para cargas de trabajo analíticas, use formatos de almacenamiento columnar como Parquet u ORC.
