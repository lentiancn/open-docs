# 쿠버네티스 설치 가이드

## 전제 조건

쿠버네티스를 설치하기 전에 기본 요구 사항을 이해해야 한다:

### 시스템 요구 사항
- Linux는 쿠버네티스 컨트롤 플레인을 실행하기 위한 권장 운영 체제이다
- 클러스터 내에서 Linux 또는 Windows를 포함한 다른 운영 체제에서 애플리케이션을 실행할 수 있다
- 컨테이너 런타임을 지원하는 운영 체제가 필요하다

### 설치 유형 선택
쿠버네티스를 설치할 때 다음을 기반으로 설치 유형을 선택한다:
- 유지 관리 용이성
- 보안
- 제어
- 사용 가능한 리소스
- 클러스터 운영 및 관리에 필요한 전문 지식

## 설치 방법

### 학습 환경 (초보자 권장)

쿠버네티스를 학습 중이라면 Kubernetes 커뮤니티에서 지원하거나 생태계 도구를 사용하여 로컬 컴퓨터에 클러스터를 설정한다:

#### Minikube
Minikube는 로컬 컴퓨터에 단일 노드 클러스터를 만드는 경량 Kubernetes 구현이다. 학습 및 소규모 테스트에 이상적이다.

설치 단계:
1. Minikube 설치
2. kubectl (쿠버네티스 명령줄 도구) 설치
3. Minikube 클러스터 시작

#### Kind
Kind (Kubernetes in Docker)는 Docker 컨테이너를 사용하여 노드를 시뮬레이션하는 로컬 Kubernetes 클러스터를 생성하는 또 다른 도구이다.

### 프로덕션 환경

프로덕션 환경의 경우 다음 옵션을 고려한다:

#### kubeadm
쿠버네티스를 배포하기 위한 공식 지원 도구이다. 모범 사례를 따라 안전한 클러스터를 생성한다.

기본 단계:
1. 서버 준비 (OS 설치, 네트워크 구성)
2. 컨테이너 런타임 설치 (containerd 또는 Docker)
3. kubeadm, kubelet, kubectl 설치
4. kubeadm으로 클러스터 초기화
5. 네트워크 플러그인 구성 (Calico, Cilium 등)
6. 워커 노드 조인

#### 관리형 서비스
직접 Kubernetes 클러스터를 관리하고 싶지 않다면 관리형 서비스 고려:
- Amazon EKS (Elastic Kubernetes Service)
- Google GKE (Google Kubernetes Engine)
- Azure AKS (Azure Kubernetes Service)
- Alibaba Cloud ACK
- Tencent Cloud TKE

#### 기타 솔루션
클라우드 및 베어메탈 환경에서 표준화되고 사용자 정의된 솔루션도 다양하게 있다.

## 도구 설치

### kubectl 설치

kubectl은 Kubernetes 클러스터와 상호 작용하는 명령줄 도구이다.

Linux 설치:
```bash
# 최신 버전 다운로드
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 실행 권한 추가
chmod +x kubectl

# 시스템 경로로 이동
sudo mv kubectl /usr/local/bin/
```

### kubeadm 설치

```bash
# apt 패키지 인덱스 업데이트
sudo apt-get update

# 의존성 설치
sudo apt-get install -y apt-transport-https ca-certificates curl

# Kubernetes APT 저장소 추가
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# 업데이트 및 설치
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## 클러스터 초기화

### kubeadm으로 클러스터 생성

1. 컨트롤 플레인 노드 초기화:
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. kubectl 구성:
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. 네트워크 플러그인 설치 (Flannel 예시):
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. 워커 노드 조인:
```bash
# 각 워커 노드에서 초기화 출력의 명령어 실행
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## 설치 확인

클러스터 상태 확인:
```bash
# 노드 보기
kubectl get nodes

# 파드 보기
kubectl get pods --all-namespaces
```

모든 컴포넌트가 Running 상태이면 설치가 성공한 것이다.

## 제거

쿠버네티스를 제거해야 하는 경우:

```bash
# 모든 Kubernetes 서비스 중지
sudo kubeadm reset

# 구성 파일 제거
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# 패키지 제거
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## 중요 참고 사항

1. 컨테이너 런타임 선택: 새 클러스터에 적합한 컨테이너 런타임 선택
2. 리소스 계획: 워크로드에 따라 CPU 및 메모리 리소스를 합리적으로 할당
3. 네트워크 계획: 파드 및 서비스 네트워크 CIDR을 미리 계획
4. 보안 강화: 프로덕션을 위해 인증, 권한 부여 및 네트워크 정책 구성
5. 고가용성: 프로덕션 환경에 대해 다중 컨트롤 플레인 클러스터 구성 권장
