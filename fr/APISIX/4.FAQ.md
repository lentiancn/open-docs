# APISIX FAQ

## Installation Issues

### Q1: Docker Container Fails to Start

**Problem**: Container exits immediately after running `docker run`.

**Solution**:
1. Check Docker logs: `docker logs apisix`
2. Confirm etcd is running
3. Check if ports are occupied: `netstat -tlnp | grep 9080`
4. Try using docker-compose to start

### Q2: Admin API Not Accessible

**Problem**: Accessing `http://127.0.0.1:9180/apisix/admin` returns 404 or 403.

**Solution**:
1. Confirm `enable_admin: true` is set in `config.yaml`
2. Check Admin API key configuration
3. Verify firewall allows port 9180

### Q3: etcd Connection Failed

**Problem**: Error "etcd connection failed" on startup.

**Solution**:
1. Confirm etcd service is running
2. Check if etcd port (default 2379) is accessible
3. Verify etcd endpoints in configuration
4. If using Docker, confirm network configuration is correct

## Configuration Issues

### Q4: Route Matching Failed

**Problem**: Requests return 404, but route configuration looks correct.

**Solution**:
1. Check if route `uri` configuration is correct
2. Confirm upstream nodes are available
3. Check APISIX error logs
4. Use `curl -v` for detailed request information

### Q5: Plugin Not Taking Effect

**Problem**: Configured plugin has no effect.

**Solution**:
1. Confirm plugin name is correct
2. Check if plugin is configured in the correct scope (route/plugin)
3. Verify plugin configuration format
4. Check if plugin dependencies are installed

### Q6: Load Balancing Not Working

**Problem**: Requests are not distributed to different nodes as expected.

**Solution**:
1. Confirm upstream node configuration is correct
2. Check load balancing type setting
3. If using hash, confirm key field is correct

## Performance Issues

### Q7: Performance Below Expectations

**Problem**: QPS is much lower than expected.

**Solution**:
1. Adjust worker processes: `nginx.worker_processes`
2. Increase worker connections: `nginx.worker_connections`
3. Enable HTTP/2: `proxy_http_version: "2"`
4. Optimize plugin configuration, disable unnecessary plugins
5. Use high-performance storage

### Q8: High Memory Usage

**Problem**: APISIX memory usage keeps growing.

**Solution**:
1. Check for memory leaks in plugins
2. Limit cache size
3. Clean logs regularly
4. Monitor and adjust buffer sizes

## Security Issues

### Q9: How to Enable Admin API Authentication

**Solution**:

Configure Admin API key in `config.yaml`:

```yaml
apisix:
  admin_key:
    - name: admin
      key: your-secret-key-here
      role: admin
```

Access Admin API with the key:

```bash
curl -H "X-API-Key: your-secret-key-here" \
  http://127.0.0.1:9180/apisix/admin/routes
```

### Q10: How to Configure HTTPS

**Solution**:

1. Obtain SSL certificate
2. Configure SSL route:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "https-route",
  "uri": "/api/*",
  "plugins": {
    "ssl": {
      "cert": "/path/to/cert.pem",
      "key": "/path/to/key.pem"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## Integration Issues

### Q11: How to Integrate with Kubernetes

**Solution**:

Use APISIX Ingress Controller:

```bash
helm install apisix-ingress apisix/apisix-ingress \
  --set ingressController.enabled=true
```

### Q12: How to Integrate with Prometheus

**Solution**:

Enable Prometheus plugin:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/plugins" -X PUT -d '
{
  "prometheus": {
    "prefer_name": true
  }
}'
```

Prometheus metrics endpoint: `http://127.0.0.1:9091/apisix/prometheus/metrics`

### Q13: How to Configure Logging

**Solution**:

Configure access log:

```yaml
nginx:
  access_log: /var/log/apisix/access.log
  error_log: /var/log/apisix/error.log
  error_log_level: warn
```

Use logging plugin:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "log-route",
  "uri": "/api/*",
  "plugins": {
    "log-rotate": {
      "max_size": 100,
      "max_backups": 50,
      "max_age": 30,
      "compress": true
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## Common Error Codes

| Error Code | Description | Solution |
|------------|-------------|----------|
| 400 | Invalid request parameters | Check request format and parameters |
| 401 | Unauthorized | Configure and use API key |
| 403 | Forbidden | Check permission configuration |
| 404 | Route not found | Confirm route configuration |
| 429 | Rate limit exceeded | Adjust rate limit configuration |
| 500 | Internal server error | Check logs |
| 502 | Bad gateway | Check upstream service |
| 503 | Service unavailable | Check upstream health status |
