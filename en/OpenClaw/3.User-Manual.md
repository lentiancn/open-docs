# OpenClaw User Manual

This manual will help you understand how to use OpenClaw effectively.

## Basic Usage

### Talking to OpenClaw

After installation, you can interact with OpenClaw in several ways:

1. **Control UI**: Visit `http://127.0.0.1:18789/` in your browser
2. **CLI command**: `openclaw agent --message "Your message"`
3. **Configured channels**: Send messages via WhatsApp, Telegram, Discord, etc.

### Sending Messages

Send messages using CLI:

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## Core Concepts

### Gateway

Gateway is the core control plane of OpenClaw, managing all sessions, channels, tools, and events. Default listening: `ws://127.0.0.1:18789`.

### Session

OpenClaw uses a session model for managing conversations:

- **Main session**: Direct chat
- **Group isolation**: Group messages handled separately
- **Activation mode**: Controls when to respond
- **Reply strategy**: Decides how to reply

### Workspace

Workspace is where OpenClaw stores configuration and data:

- Default location: `~/.openclaw/workspace`
- Contains: AGENTS.md, SOUL.md, TOOLS.md
- Skills directory: `~/.openclaw/workspace/skills/`

## Channel Configuration

### Supported Channels

OpenClaw supports these messaging platforms:

- Instant messaging: WhatsApp, Telegram, Signal, iMessage
- Social platforms: Discord, Slack
- Enterprise tools: Microsoft Teams, Google Chat, Feishu
- Others: IRC, Matrix, LINE, Mattermost, Nostr, etc.

### Configuration Examples

**Telegram**:

```json
{
  "channels": {
    "telegram": {
      "botToken": "YOUR_BOT_TOKEN"
    }
  }
}
```

**Discord**:

```json
{
  "channels": {
    "discord": {
      "token": "YOUR_BOT_TOKEN"
    }
  }
}
```

**WhatsApp**:

```bash
openclaw channels login
```

## Skills System

Skills are how OpenClaw extends functionality. You can:

### Install Skills

Install community skills from ClawHub, or create your own.

### Skill Structure

Each skill is a directory containing a SKILL.md file:

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHub is a skill registry. When enabled, OpenClaw can automatically search for and install needed skills.

## Automation

### Cron Jobs

Set up scheduled tasks using Cron:

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhooks

Configure webhooks to trigger automation:

```bash
openclaw webhook add --url https://your-endpoint.com/hook
```

### Gmail Integration

Set up Gmail Pub/Sub to listen for new emails.

## Advanced Features

### Browser Control

OpenClaw can control a dedicated browser:

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvas Visualization

Canvas is OpenClaw's visual workspace, supporting A2UI push/reset, eval, and snapshots.

### Voice Features

- **Voice Wake**: Activate via wake words on macOS/iOS
- **Talk Mode**: Continuous voice conversation (Android support)

### Device Nodes

OpenClaw can connect to various devices:

- **macOS node**: System run, notifications, Canvas, camera
- **iOS node**: Canvas, voice, camera, screen recording
- **Android node**: Connect, chat, voice, Canvas, camera, location, etc.

## Security Settings

### DM Pairing

By default, unknown senders need pairing verification. Change to open mode with `dmPolicy: "open"` (not recommended).

### Manage Pairing:

```bash
# View pairing list
openclaw pairing list

# Approve pairing
openclaw pairing approve

# Revoke pairing
openclaw pairing revoke
```

### Run Diagnostics

Regularly run diagnostics for security:

```bash
openclaw doctor
```

## Command Reference

| Command | Description |
|---------|-------------|
| `/status` | View session status |
| `/new` or `/reset` | Reset session |
| `/compact` | Compact session context |
| `/think` | Set thinking level |
| `/verbose` | Toggle verbose output |
| `/usage` | View usage statistics |
| `/restart` | Restart Gateway |

## Best Practices

1. **Run doctor regularly**: Keep system healthy
2. **Be careful with DMs**: Check security settings
3. **Use strong models**: Get the best experience
4. **Back up regularly**: Save important configs

## Troubleshooting

Common issues:

- **Gateway won't start**: Check port 18789
- **Channel connection fails**: Check credentials
- **Message sending fails**: Check logs
