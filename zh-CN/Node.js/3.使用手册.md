# Node.js 使用手册

## 基础使用

### 创建第一个 Node.js 应用

Node.js 应用程序通常以 `.js` 文件扩展名保存。最简单的 "Hello World" 应用如下：

```javascript
// hello.js
console.log('Hello World!');
```

在终端中运行：
```bash
node hello.js
```

### 模块系统

Node.js 使用 CommonJS 模块系统。你可以创建自己的模块并导出功能：

```javascript
// math.js
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, multiply };
```

在其他文件中使用：
```javascript
// app.js
const math = require('./math');

console.log(math.add(2, 3)); // 输出: 5
console.log(math.multiply(4, 5)); // 输出: 20
```

### 内置模块

Node.js 提供了许多内置模块，无需安装即可使用：

- **fs**: 文件系统操作
- **http**: HTTP 服务器和客户端
- **path**: 路径操作
- **os**: 操作系统信息
- **events**: 事件处理

示例 - 创建 HTTP 服务器：
```javascript
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello from Node.js Server!');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at http://127.0.0.1:3000/');
});
```

## 包管理 (npm)

### 初始化项目

```bash
npm init
# 或者使用默认设置
npm init -y
```

这会创建 `package.json` 文件，用于管理项目依赖和元数据。

### 安装依赖

```bash
# 安装生产依赖
npm install express

# 安装开发依赖
npm install --save-dev jest

# 全局安装
npm install -g nodemon
```

### 运行脚本

在 `package.json` 中定义脚本：
```json
{
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  }
}
```

运行脚本：
```bash
npm run start
npm run dev
npm test
```

## 异步编程

### 回调函数

```javascript
const fs = require('fs');

fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});
```

### Promise

```javascript
const fs = require('fs').promises;

async function readFile() {
  try {
    const data = await fs.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

readFile();
```

### async/await

现代 Node.js 开发推荐使用 async/await 语法：

```javascript
async function processData() {
  try {
    const users = await getUsers();
    const processed = await processUsers(users);
    await saveResults(processed);
    console.log('Processing completed!');
  } catch (error) {
    console.error('Error:', error);
  }
}
```

## 性能优化

### 避免阻塞操作

始终使用异步方法处理 I/O 操作：

```javascript
// ❌ 错误：阻塞操作
const data = fs.readFileSync('large-file.txt');

// ✅ 正确：非阻塞操作
const data = await fs.promises.readFile('large-file.txt');
```

### 使用集群模式

利用多核处理器：

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`主进程 ${process.pid} 正在运行`);
  
  // 衍生工作进程
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker, code, signal) => {
    console.log(`工作进程 ${worker.process.pid} 已退出`);
  });
} else {
  // 工作进程可以共享任何 TCP 连接
  // 在本例中是 HTTP 服务器
  const http = require('http');
  
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`工作进程 ${process.pid} 已启动`);
}
```

## 最佳实践

1. **使用严格模式**: 在文件顶部添加 `'use strict';`
2. **错误处理**: 始终处理可能的错误
3. **环境变量**: 使用 `.env` 文件管理配置
4. **代码格式化**: 使用 ESLint 和 Prettier
5. **测试**: 编写单元测试和集成测试
6. **安全性**: 定期更新依赖，避免安全漏洞
