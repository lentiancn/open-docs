# Hadoop 安裝指南

## 環境要求

在安裝 Hadoop 之前，需要確保系統滿足基本的硬體和軟體要求。Hadoop 叢集可以由多台電腦組成，也可以是單機偽分散式模式。不同的部署模式對硬體有不同的要求，但核心的軟體依賴是相似的。了解這些要求有助於規劃 Hadoop 叢集的規模和配置，避免在安裝過程中遇到不必要的麻煩。

硬體方面，對於小型測試叢集，建議每台機器至少配備 4 核 CPU、8GB 記憶體和 500GB 硬碟。對於生產環境，建議每台機器配備 8 核或更多 CPU、16GB 或更多記憶體，以及 1TB 或更大的硬碟。Hadoop 的資料節點會儲存大量資料，因此硬碟容量尤為重要。Hadoop 建議使用 RAID 配置或 JBOD（Just a Bunch Of Disks）配置，後者允許 Hadoop 更靈活地管理磁碟空間。

作業系統方面，Hadoop 支援 Linux（Ubuntu、CentOS、Debian、Red Hat 等）和 Windows（透過 Cygwin 或 Windows Subsystem for Linux）。生產環境強烈建議使用 Linux 系統，以獲得最佳的性能和穩定性。Hadoop 與各種雲平台也有良好的整合，可以在 AWS、Azure、Google Cloud 等雲服務上部署。

Java 是 Hadoop 唯一的執行時依賴。Hadoop 3.x 需要 Java 8 或更高版本。建議使用 Oracle JDK 或 OpenJDK。在安裝 Hadoop 之前，需要確保 Java 已正確安裝，並且 JAVA_HOME 環境變數已設置。可以通過以下命令驗證 Java 安裝：

```bash
java -version
echo $JAVA_HOME
```

網路是 Hadoop 叢集正常工作的基礎。叢集中的所有節點應該能夠透過網路相互通訊，建議使用高速網路（如千兆乙太網路）。節點之間需要配置 SSH 無密碼登入，以便 Hadoop 可以在節點之間分發任務和傳輸資料。此外，需要在 /etc/hosts 檔案中配置所有節點的主機名和 IP 位址映射。

## 安裝步驟

### 單機模式安裝

單機模式是 Hadoop 最簡單的安裝方式，所有元件都運行在同一個 JVM 程序中。這種模式主要用於開發和測試，不涉及分散式儲存或計算。首先下載 Hadoop 的二進制發行版，可以從 Apache Hadoop 官方網站獲取：

```bash
# 下載 Hadoop（替換為最新版本）
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

# 解壓到指定目錄
tar -xzf hadoop-3.3.6.tar.gz -C /usr/local/

# 重新命名目錄（可選）
mv /usr/local/hadoop-3.3.6 /usr/local/hadoop
```

接下來配置環境變數。編輯 ~/.bashrc 或 /etc/profile 檔案，新增以下內容：

```bash
export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64  # 根據實際 Java 路徑調整
```

使配置生效：

```bash
source ~/.bashrc
```

驗證安裝：

```bash
hadoop version
```

### 偽分散式模式安裝

偽分散式模式在單機上模擬分散式環境，NameNode、DataNode、ResourceManager、NodeManager 等元件都運行在同一台機器上。這種模式適合開發和測試分散式功能。

首先配置 Hadoop 的核心檔案。進入 Hadoop 配置目錄：

```bash
cd $HADOOP_HOME/etc/hadoop
```

配置 core-site.xml，指定 HDFS 的預設檔案系統：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

配置 hdfs-site.xml，指定 HDFS 的副本數量（偽分散式模式下為 1）：

```xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///usr/local/hadoop/data/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///usr/local/hadoop/data/datanode</value>
    </property>
</configuration>
```

配置 mapred-site.xml，指定 MapReduce 使用的資源調度框架：

```xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_CLASSPATH</value>
    </property>
</configuration>
```

配置 yarn-site.xml，配置 YARN 相關的屬性：

```xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
        <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>
</configuration>
```

### SSH 無密碼登入配置

Hadoop 叢集中的節點需要配置 SSH 無密碼登入，以便 master 節點可以免密碼存取 slave 節點：

```bash
# 產生 SSH 密鑰對
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# 將公鑰添加到 authorized_keys
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# 設置正確的權限
chmod 0600 ~/.ssh/authorized_keys

# 測試 SSH 登入
ssh localhost
```

如果使用多節點叢集，需要將公鑰複製到所有節點：

```bash
ssh-copy-id user@slave1
ssh-copy-id user@slave2
```

### 格式化與啟動 HDFS

首次使用 HDFS 之前，需要格式化 NameNode：

```bash
hdfs namenode -format
```

啟動 HDFS 和 YARN：

```bash
start-dfs.sh
start-yarn.sh
```

或者使用 start-all.sh 一次性啟動所有服務：

```bash
start-all.sh
```

驗證服務啟動狀態：

```bash
# 查看 Java 程序
jps

# 預期的程序包括：
# NameNode
# DataNode
# SecondaryNameNode
# ResourceManager
# NodeManager
```

可以透過 Web UI 存取 Hadoop 服務：NameNode 的 Web UI 位址是 http://localhost:9870，ResourceManager 的 Web UI 位址是 http://localhost:8088。

## 完全分散式叢集安裝

### 叢集規劃

完全分散式模式需要多台實體機或虛擬機器組成叢集。在安裝之前，需要進行合理的叢集規劃。確定叢集的規模（包含多少個節點），分配各節點的角色（NameNode、Secondary NameNode、DataNode、ResourceManager、NodeManager）。對於小型叢集，可以將 NameNode 和 ResourceManager 部署在同一台機器上；對於大型叢集，建議分開部署以提高性能。

網路規劃也很重要。建議所有節點在同一內網中，延遲低、頻寬高。在 /etc/hosts 檔案中配置所有節點的主機名和 IP 位址：

```bash
192.168.1.1 master
192.168.1.2 slave1
192.168.1.3 slave2
192.168.1.4 slave3
```

### 設定檔案修改

在完全分散式模式下，需要修改更多設定檔。首先配置 workers 檔案（早期版本使用 slaves 檔案），列出所有 DataNode 和 NodeManager 節點：

```
slave1
slave2
slave3
```

core-site.xml 和 hdfs-site.xml 的配置與偽分散式類似，但需要新增更多屬性。對於 NameNode 高可用（HA）配置，需要配置兩個 NameNode：

```xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://mycluster</value>
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/hadoop/tmp</value>
    </property>
</configuration>
```

hdfs-site.xml 配置 NameNode HA：

```xml
<configuration>
    <property>
        <name>dfs.nameservices</name>
        <value>mycluster</value>
    </property>
    <property>
        <name>dfs.ha.namenodes.mycluster</name>
        <value>nn1,nn2</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn1</name>
        <value>master:9870</value>
    </property>
    <property>
        <name>dfs.namenode.http-address.mycluster.nn2</name>
        <value>slave1:9870</value>
    </property>
    <property>
        <name>dfs.namenode.shared.edits.dir</name>
        <value>qjournal://master:8485;slave1:8485/mycluster</value>
    </property>
    <property>
        <name>dfs.client.failover.proxy.provider.mycluster</name>
        <value>org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider</value>
    </property>
</configuration>
```

YARN 的配置類似，可以在 yarn-site.xml 中配置 ResourceManager HA。

### 同步設定檔案

將配置好的 Hadoop 安裝目錄同步到所有節點。可以使用 scp 命令逐個複製：

```bash
scp -r /usr/local/hadoop user@slave1:/usr/local/
scp -r /usr/local/hadoop user@slave2:/usr/local/
scp -r /usr/local/hadoop user@slave3:/usr/local/
```

或者使用分散式複製工具如 rsync：

```bash
rsync -avz /usr/local/hadoop slave1:/usr/local/
rsync -avz /usr/local/hadoop slave2:/usr/local/
rsync -avz /usr/local/hadoop slave3:/usr/local/
```

### 啟動叢集

在 NameNode 節點上格式化 NameNode（首次啟動時）：

```bash
hdfs namenode -format
```

啟動 HDFS：

```bash
start-dfs.sh
```

啟動 YARN：

```bash
start-yarn.sh
```

檢查所有節點的程序狀態，確保服務正常運行：

```bash
# 在 NameNode 上
jps

# 在 DataNode 上
jps
```

## 驗證與測試

### HDFS 測試

建立目錄並上傳檔案測試 HDFS：

```bash
# 建立目錄
hdfs dfs -mkdir -p /user/test

# 上傳本地檔案到 HDFS
hdfs dfs -put localfile.txt /user/test/

# 查看檔案
hdfs dfs -ls /user/test

# 讀取檔案
hdfs dfs -cat /user/test/localfile.txt

# 刪除檔案
hdfs dfs -rm /user/test/localfile.txt
```

### MapReduce 測試

執行 MapReduce 範例程式驗證叢集功能：

```bash
# 執行 WordCount 範例
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.3.6.jar wordcount /input /output

# 查看結果
hdfs dfs -cat /output/part-r-00000
```

### 叢集健康檢查

使用 Hadoop 提供的命令檢查叢集狀態：

```bash
# 查看 HDFS 狀態
hdfs dfsadmin -report

# 查看 YARN 節點狀態
yarn node -list -all

# 查看應用程式狀態
yarn application -list
```

## 常見問題

### Java 版本不相容

如果遇到 Java 版本相關的錯誤，確保安裝的 Java 版本與 Hadoop 版本相容。Hadoop 3.x 需要 Java 8 或更高版本。可以通過以下命令切換 Java 版本：

```bash
update-alternatives --config java
```

### 磁碟空間不足

DataNode 可能會因為磁碟空間不足而無法正常工作。定期監控磁碟使用情況，確保 DataNode 有足夠的可用空間。可以在 hdfs-site.xml 中配置 dfs.datanode.fsdataset.volume.choosing.policy 為 AvailableSpaceVolumeChoosingPolicy，讓 DataNode 優先使用可用空間更多的磁碟。

### 記憶體不足

如果叢集節點記憶體不足，YARN 可能無法啟動容器。可以透過 yarn-site.xml 調整記憶體配置：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>16384</value>
</property>
<property>
    <name>yarn.scheduler.minimum-allocation-mb</name>
    <value>1024</value>
</property>
```
