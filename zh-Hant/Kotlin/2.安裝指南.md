# Kotlin 安裝指南

## 環境準備

在開始安裝 Kotlin 之前，需要確保您的電腦滿足基本的系統要求。Kotlin 可以 在多種作業系統上執行，包括 Windows、macOS、Linux 以及其他類 Unix 系統。無論您使用哪種作業系統，都需要確保有足夠的磁碟空間來安裝開發工具和執行編譯後的程式。

對於 Windows 使用者，建議使用 Windows 10 或更高版本以獲得最佳體驗。macOS 使用者需要 macOS 10.14 或更高版本。Linux 使用者可以使用任何主流發行版，如 Ubuntu、Fedora、Debian 等。

## 透過 IDE 安裝

### IntelliJ IDEA

IntelliJ IDEA 是 JetBrains 出品的旗艦級整合開發環境，對 Kotlin 提供了最完善的支持。如果您的 主要目標是 Kotlin 開發，IntelliJ IDEA 是最佳選擇。

IntelliJ IDEA 分為兩個版本：社區版（Community）和旗艦版（Ultimate）。社區版免費開源，功能已經足夠支持 Kotlin 開發。旗艦版面向企業使用者，提供了更多高級特性，如高級資料庫工具、企業級安全功能等。

安裝步驟如下：首先，訪問 JetBrains 官方網站（https://www.jetbrains.com/idea/download/），根據您的作業系統選擇對應的安裝包。下載完成後，執行安裝程式，按照精靈指示完成安裝。在安裝過程中，可以選擇安裝路徑和建立桌面捷徑等選項。

安裝完成後，首次啟動 IntelliJ IDEA 時，IDE 會自動檢測並提示安裝 Kotlin 插件。如果系統沒有自動檢測，您也可以手動在插件市場中搜尋 Kotlin 並安裝。安裝完成後，重新啟動 IDE 即可開始 Kotlin 開發。

### Android Studio

Android Studio 是 Google 官方的 Android 開發 IDE，內置了 Kotlin 支持。如果您主要進行 Android 開發，Android Studio 是理想的選擇。

訪問 Android 開發者官網（https://developer.android.com/studio）下載 Android Studio。下載完成後，執行安裝程式完成安裝。Android Studio 預設包含了 Kotlin 插件，無需額外配置即可開始開發。

Android Studio 每年會發布多個更新版本，每個版本都包含最新的 Kotlin 支持和 Android 開發工具。建議保持版本更新以獲得最佳開發體驗和最新的語言特性。

## 透過命令列安裝

除了使用 IDE，您也可以選擇安裝 Kotlin 命令列編譯器。這種方式適合喜歡輕量級開發環境或需要在伺服器上進行 Kotlin 開發的使用者。

### 手動安裝

首先，訪問 Kotlin GitHub Releases 頁面（https://github.com/JetBrains/kotlin/releases）下載最新版本的編譯器。下載完成後是一個 zip 壓縮包，將其解壓到指定目錄即可。

解壓後，將 kotlinc/bin 目錄添加到系統的環境變數中，這樣就可以在命令列中直接使用 kotlinc 命令。在 Windows 上，可以透過系統屬性中的環境變數設定來新增路徑。在 macOS 和 Linux 上，可以修改 ~/.bashrc 或 ~/.zshrc 檔案，新增類似的行：

export PATH="$PATH:/path/to/kotlinc/bin"

新增完成後，開啟一個新的終端機視窗，輸入 kotlinc -version 命令，如果看到版本資訊，說明安裝成功。

### 使用 SDKMAN! 安裝

SDKMAN! 是一個用於管理 Java 開發工具版本的工具，在類 Unix 系統上使用非常方便。如果您使用的是 macOS、Linux、Cygwin、FreeBSD 或 Solaris，SDKMAN! 是安裝 Kotlin 的推薦方式。

首先，需要安裝 SDKMAN!。在終端機中執行以下命令：

curl -s "https://get.sdkman.io" | bash

安裝完成後，需要重新載入環境設定：

source ~/.sdkman/bin/sdkman-init.sh

然後就可以使用 sdk 命令安裝 Kotlin 了：

sdk install kotlin

安裝過程中會提示確認版本資訊，輸入確認後即可完成安裝。安裝完成後，同樣可以使用 kotlinc -version 驗證安裝。

### 使用 Homebrew 安裝

在 macOS 上，還可以使用 Homebrew 套件管理員來安裝 Kotlin。首先確保已經安裝了 Homebrew，然後執行以下命令：

brew update
brew install kotlin

Homebrew 會自動下載並安裝最新版本的 Kotlin 編譯器。安裝完成後，同樣可以使用 kotlinc -version 驗證安裝。

### 使用 Snap 安裝

在 Ubuntu 16.04 及更高版本上，可以使用 Snap 套件管理員安裝 Kotlin。執行以下命令：

sudo snap install --classic kotlin

Snap 安裝方式會將 Kotlin 安裝為系統級應用，可以在任何位置透過命令列呼叫。

## 建立第一個 Kotlin 程式

無論您選擇哪種安裝方式，都可以透過以下步驟建立和執行第一個 Kotlin 程式。

首先，建立一個名為 hello.kt 的檔案，內容如下：

fun main() {
    println("Hello, World!")
}

這是一個最簡單的 Kotlin 程式，定義了一個 main 函數，這是 Kotlin 程式的入口點。println 函數用於向控制台輸出文字。

如果使用命令列編譯器，編譯執行這個程式需要兩步。首先是編譯：

kotlinc hello.kt -include-runtime -d hello.jar

這裡的 -include-runtime 參數會將 Kotlin 執行時庫打包到生成的 jar 檔案中，使 jar 檔案可以獨立執行。-d 參數指定輸出檔案的路徑。

編譯完成後，執行程式：

java -jar hello.jar

您應該能在控制台看到 "Hello, World!" 的輸出。

如果您不想生成獨立的 jar 檔案，也可以直接使用 kotlinc 的互動式執行功能：

kotlinc -script hello.kt

或者使用 kotlin 命令直接執行編譯後的 class 檔案：

kotlin -classpath hello.jar HelloKt

其中 HelloKt 是 Kotlin 編譯器為 hello.kt 檔案生成的主類別名稱。

## 使用 REPL

Kotlin 提供了一個互動式的 REPL（Read-Eval-Print Loop）環境，您可以輸入 Kotlin 程式碼並立即看到執行結果。這對於學習語言特性或快速測試程式碼片段非常有用。

要啟動 REPL，只需在命令列中輸入：

kotlinc

不帶任何參數執行 kotlinc 會進入 REPL 模式。在這個模式下，您可以輸入任何有效的 Kotlin 程式碼，按 Enter 後即可看到執行結果。例如：

>>> val x = 10
>>> val y = 20
>>> x + y
30

輸入 :quit 或按 Ctrl+C 可以退出 REPL。

## Kotlin 腳本

Kotlin 還可以作為腳本語言使用。Kotlin 腳本檔案以 .kts 為副檔名，檔案中的頂層程式碼會被視為可執行程式碼。

例如，建立一個名為 list_folders.kts 的腳本：

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

執行這個腳本：

kotlinc -script list_folders.kts -- -d /path/to/folder

## 配置建置工具

在實際專案中，通常會使用 Gradle 或 Maven 作為建置工具。這兩個工具都原生支援 Kotlin 專案。

### Gradle

Gradle 是目前 Kotlin 開發中最流行的建置工具。Kotlin 團隊也提供了專門的 Kotlin DSL，讓建置腳本可以使用 Kotlin 語法編寫。

要建立一個基本的 Kotlin Gradle 專案，首先需要在專案中新增 Kotlin 插件。專案的 build.gradle 檔案需要包含以下內容：

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

或者使用 Kotlin DSL：

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

然後建立原始碼目錄結構，通常是 src/main/kotlin。在其中建立 Kotlin 原始檔案，使用 gradle build 命令即可編譯專案。

### Maven

Maven 是另一個流行的 Java 專案建置工具，同樣支援 Kotlin 專案。在 pom.xml 檔案中新增 Kotlin 相關依賴即可。

## 驗證安裝

安裝完成後，可以透過幾個簡單的命令來驗證一切是否正常運作。

首先驗證 Kotlin 編譯器：

kotlinc -version

應該看到類似 "Kotlin compiler version 2.3.10" 的輸出。

然後驗證 Kotlin 標準庫：

kotlin -e 'println("Hello Kotlin!")'

如果一切正常，說明安裝成功，可以開始您的 Kotlin 之旅了。

## 下一步

安裝完成 Kotlin 後，建議按照以下路徑繼續學習：首先閱讀 Kotlin 官方 Tour，了解語言的基礎特性；然後選擇一個您感興趣的開發方向深入學習，如 Android 開發、伺服器端開發或跨平台開發；最後透過實際專案練手，將所學知識應用到實踐中。

Kotlin 官方提供了豐富的學習資源，包括官方文檔、線上教學、視訊課程等。您可以在 Kotlin 官網上找到這些資源的連結。
