# MongoDB Häufig gestellte Fragen

## Grundkonzepte

### Was ist MongoDB?

MongoDB ist eine dokumentenorientierte Datenbank, die entwickelt wurde, um Entwicklern den schnelleren Aufbau moderner Anwendungen zu ermöglichen. Sie speichert Daten in flexiblen, JSON-ähnlichen Dokumenten.

### Ist MongoDB eine relationale Datenbank?

Nein. MongoDB ist eine dokumentenorientierte NoSQL-Datenbank, die das BSON-Format verwendet.

### Was ist ein Dokument in MongoDB?

Ein Dokument ist die Grundeinheit von Daten in MongoDB, bestehend aus Feld-Wert-Paaren. Jedes Dokument hat ein eindeutiges _id-Feld als Primärschlüssel.

### Was ist der Unterschied zwischen MongoDB und MySQL?

| Merkmal | MongoDB | MySQL |
|---------|---------|-------|
| Datenmodell | Dokument | Relational |
| Schema | Flexibel | Fest |
| Skalierung | Horizontal | Vertikal |
| Abfragesprache | MongoDB Query Language | SQL |
| Transaktionen | Multi-Dokument-ACID | Multi-Tabellen-ACID |

### Was ist BSON?

BSON bedeutet Binary JSON, die binäre Darstellung von JSON-Dokumenten mit zusätzlichen Datentypen.

### Was ist die maximale Dokumentgröße in MongoDB?

Die maximale Größe beträgt 16 MB. Für größere Dateien bietet MongoDB die GridFS-API.

## Installationsprobleme

### Wie viel Speicher benötigt MongoDB?

MongoDB verwendet verfügbare Speicher als Cache. Für Entwicklungsumgebungen werden mindestens 4 GB empfohlen.

### Unterstützt MongoDB 32-Bit-Systeme?

Nein. MongoDB erfordert ein 64-Bit-Betriebssystem.

### Wie ändere ich den Standardport von MongoDB?

Der Standardport ist 27017. Ändern Sie `net.port` in der Konfigurationsdatei oder verwenden Sie `--port`.

## Nutzungsfragen

### Wie sichere ich MongoDB-Daten?

```bash
mongodump --db=meinedatenbank --out=/sicherung
```

Wiederherstellen:

```bash
mongorestore --db=meinedatenbank /sicherung/meinedatenbank
```

### Wie erreicht MongoDB Hochverfügbarkeit?

Durch Replica Sets. Ein Replica Set besteht aus einem Primärknoten und mehreren Sekundärknoten.

### Was ist Sharding?

Sharding ist MongoDBs Technik zur horizontalen Skalierung, die Daten auf mehrere Server verteilt.

### Unterstützt MongoDB Transaktionen?

Ja. MongoDB 4.0+ unterstützt Multi-Dokument-Transaktionen.

### Wie optimiere ich Abfrageleistung?

1. Geeignete Indizes erstellen
2. Projektionen verwenden
3. Vollständige Collection-Scans vermeiden
4. Aggregations-Pipelines nutzen

### Wie schütze ich Daten in MongoDB?

1. Authentifizierung aktivieren
2. Rollenbasierte Zugriffskontrolle (RBAC) verwenden
3. TLS/SSL aktivieren
4. Speicherverschlüsselung nutzen
5. Regelmäßige Backups durchführen

### Was ist der Unterschied zwischen Community und Enterprise Edition?

Die Enterprise Edition enthält:
- LDAP-Authentifizierung
- Kerberos-Authentifizierung
- Audit
- Verschlüsselter Speichermotor
- Technischer Support

### Für welche Datentypen ist MongoDB geeignet?

- Content-Management-Systeme
- Benutzerkonfigurationen
- IoT-Sensordaten
- Echtzeitanalysen
- Mobile Anwendungen
- E-Commerce-Produktkataloge

## Fehlerbehebung

### Verbindung abgelehnt?

1. Prüfen, ob MongoDB-Dienst läuft
2. Port überprüfen
3. Firewall-Einstellungen prüfen

### Langsame Abfragen?

1. explain() zur Analyse verwenden
2. Geeignete Indizes erstellen
3. Langsame Abfragen prüfen

### Replica-Set-Wahl fehlgeschlagen?

1. Netzwerkkonnektivität prüfen
2. Knotenkonfiguration prüfen
3. Sicherstellen, dass大多数 Knoten verfügbar sind

- Python
- Java
- C#
- Go
- Ruby
- PHP
- Scala
- Kotlin
- Swift

### Wie verwende ich MongoDB in Anwendungen?

1. Treiber für Ihre Sprache installieren
2. Verbindung herstellen
3. CRUD-Operationen ausführen

Beispiel in Node.js:

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('meineDB');
  const collection = database.collection('benutzer');
  
  await collection.insertOne({ name: 'Max', alter: 30 });
  const benutzer = await collection.findOne({ name: 'Max' });
  
  await client.close();
}
```

### Was ist MongoDB Atlas?

MongoDB Atlas ist ein von MongoDB bereitgestellter Cloud-Datenbankdienst mit vollständig verwalteten Bereitstellungen.

### Wie modelliere ich Daten in MongoDB?

Grundsätze für MongoDB-Datenmodellierung:
1. Dokumentstruktur der Normalisierung vorziehen
2. Eingebettete Dokumente für "enthält"-Beziehungen nutzen
3. Referenzen für "viele-zu-viele"-Beziehungen nutzen
4. Collections und Indizes nach Abfragemustern entwerfen
5. Wachstum und Skalierbarkeit berücksichtigen
