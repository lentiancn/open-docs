# Node.js 사용 매뉴얼

## 시작하기

### 기본 개념

Node.js는 Chrome의 V8 JavaScript 엔진을 기반으로 구축되었으며, 데이터 집약적인 실시간 애플리케이션에 가볍고 효율적인 비동기 I/O 모델을 제공합니다.

#### 이벤트 루프
Node.js 이벤트 루프는 동시성 모델의 핵심입니다. 기존의 스레드 기반 모델과 달리 Node.js는 콜백, 프로미스 또는 async/await를 통해 비동기 작업을 처리하는 단일 스레드 이벤트 루프를 사용합니다.

#### 비차단 I/O
Node.js의 대부분의 작업은 비차단 방식으로, I/O 작업 완료를 기다리는 동안 프로그램 실행을 중단하지 않습니다. 대신 결과가 준비되면 콜백을 사용하여 결과를 처리합니다.

### Hello World 애플리케이션

이 간단한 HTTP 서버로 첫 번째 Node.js 애플리케이션을 생성합니다:

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

다음 명령어로 실행: `node server.js`

## 코어 모듈

Node.js는 외부 종속성 없이 필수 기능을 제공하는 풍부한 내장 모듈 세트와 함께 제공됩니다.

### 파일 시스템(fs)
파일 작업을 동기적 또는 비동기적으로 처리합니다:

```javascript
const fs = require('fs');

// 비동기 읽기
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// 동기 읽기(실행 차단)
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// 파일 쓰기
fs.writeFile('/path/to/output.txt', 'Hello Node.js!', (err) => {
  if (err) throw err;
  console.log('File written successfully');
});
```

### HTTP 서버
웹 서버를 생성하고 HTTP 요청을 수행합니다:

```javascript
const http = require('http');
const https = require('https');

// HTTP 서버 생성
const server = http.createServer((req, res) => {
  // 요청 처리
});

// HTTP 요청 수행
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });
});
req.end();
```

### 경로 유틸리티

```javascript
const path = require('path');

// 경로 결합
const fullPath = path.join('/home', 'user', 'documents', 'file.txt');

// 디렉토리 이름 가져오기

// 파일 확장자 가져오기
const ext = path.extname('file.txt'); // '.txt'

// 절대 경로 해결
const absPath = path.resolve('./relative/path');
```

### 이벤트
EventEmitter 클래스를 사용하여 사용자 정의 이벤트를 처리합니다:

```javascript
const EventEmitter = require('events');

myEmitter.on('event', () => {
  console.log('An event occurred!');

```

## npm을 통한 패키지 관리

npm(Node Package Manager)은 Node.js의 기본 패키지 관리자이며 세계 최대의 소프트웨어 레지스트리입니다.

### 기본 명령어

```bash
# 새 프로젝트 초기화
npm init

# 패키지 설치
npm install express           # 종속성에 추가
npm install --save-dev jest   # devDependencies에 추가
npm install -g nodemon        # 전역 설치

# package.json에 정의된 스크립트 실행
npm start
npm test
npm run build

# 패키지 업데이트
npm update
npm outdated

# 패키지 제거
npm uninstall express
```

## 오류 처리

비동기 특성으로 인해 Node.js 애플리케이션에서 적절한 오류 처리가 중요합니다.

### 콜백 기반 오류 처리
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Error reading file:', err);
    return;
  }
  console.log('File content:', data);
});
```

### 프로미스 기반 오류 처리
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('File content:', data);
  })
  .catch(err => {
    console.error('Error reading file:', err);
  });
```

### Async/Await 오류 처리
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('File content:', data);
  } catch (err) {
    console.error('Error reading file:', err);
  }
}
```

## 성능 최적화

### 클러스터링
클러스터 모듈을 사용하여 여러 CPU 코어를 활용합니다:

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);
  
  // 워커 생성
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died`);
  });
} else {
  // 워커는 TCP 연결 공유
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`Worker ${process.pid} started`);
}
```

### 메모리 관리
- 이벤트 리스너를 적절히 정리하여 메모리 누수 방지
- 큰 파일 처리 시 전체를 메모리에 로드하는 대신 스트림 사용
- `process.memoryUsage()`로 메모리 사용량 모니터링

### 캐싱
성능 향상을 위한 캐싱 전략 구현:

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```

```javascript
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));
}
```

### 속도 제한
남용을 방지하기 위해 속도 제한을 구현합니다:

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15분
  max: 100 // 각 IP를 windowMs당 100개 요청으로 제한
});

app.use(limiter);
```
