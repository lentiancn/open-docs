# Flutter ユーザーマニュアル

## 基本概念

### ウィジェット紹介

ウィジェットはFlutter UIの基本的な構成要素です。Flutterでは、レイアウト、padding、alignなど、すべてがウィジェットです。ウィジェットは、特定の設定と状態でビューがどのように見えるかを記述します。

Flutterのウィジェットは不変（immutable）です。UIを変更する必要がある場合、ウィジェットを再作成するか、StatefulWidgetと状態管理を使用する必要があります。ウィジェットをネストしてウィジェットツリーを形成でき、Flutterは古いウィジェットツリーと新しいウィジェットツリーを効率的に比較し、実際に変更があった部分だけをレンダリングします。

一般的なウィジェットには、Text（テキスト）、Image（画像）、Icon（アイコン）、Container（コンテナ）、Row/Column（水平/垂直レイアウト）、Stack（スタックレイアウト）、ListView（リスト）などが含まれます。

### StatelessWidgetとStatefulWidget

StatelessWidgetは状態のないウィジェットで、テキストや画像などの静的なUI展示に適しています。StatelessWidgetを定義するには、buildメソッドを実装し、レンダリングするウィジェットツリーを返すだけです。

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidgetは状態を持つウィジェットで、カounterやフォームなどの動的にUIを更新するシナリオに適しています。StatefulWidgetはStateと組み合わせて使用し、状態が変化した時にsetStateメソッドを呼び出してFlutterに再レンダリングを通知します。

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## レイアウト基礎

### リニアレイアウト

RowとColumnは最も一般的に使用されるリニアレイアウトウィジェットです。Rowは子を水平に配置し、Columnは子を垂直に配置します。MainAxisAlignmentとCrossAxisAlignmentを使用して配置方法を制御できます。

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### スタックレイアウト

Stackはスタックレイアウトを作成するために使用され、別のウィジェットの 위에ウィジェットを配置できます。PositionedはStack内の子を配置するために使用します。

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### リストレイアウト

ListViewは最も一般的に使用されるリストウィジェットで、長リストに適しています。ListView.builderは大量データのリストに適しており、レイジーローディングを実装できます。ListView.separatedで区切り線を追加できます。

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## 状態管理

### setState

setStateは最も基本的な状態管理方法で、ローカル状態に適しています。StatefulWidgetでは、setStateを呼び出すとウィジェットの再構築がトリガーされます。

```dart
setState(() {
  _counter++;
});
```

### Provider

ProviderはFlutter公式推奨の状態管理ソリューションの一つです。Providerを通じて、ウィジェットツリーで状態を共有でき、层层传递を避けられます。

```dart
// ChangeNotifierを定義
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Providerを使用
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## ネットワークリクエスト

### HTTPパッケージ

httpパッケージでシンプルなHTTPリクエストを実行できます：

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

### Dio

Dioは強力なHTTPクライアントで、インクターター、リクエストキャンセル、ファイルアップロードなどをサポートしています：

```dart
import 'package:dio/dio.dart';

final dio = Dio();
final response = await dio.get('https://api.example.com/data');
```

## ローカルストレージ

### SharedPreferences

シンプルなキーバリューデータの保存に使用：

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

### SQLite

構造化データにはsqflaketパッケージを使用：

```dart
import 'package:sqflite/sqflite.dart';

final db = await openDatabase('my.db');
await db.insert('users', {'name': 'Flutter'});
```

## ナビゲーションとルーティング

FlutterはNavigatorを使用してページナビゲーションを行います基本的な操作には、Navigator.push（新しいページをプッシュ）、Navigator.pop（戻る）、Navigator.pushReplacement（現在ページを置き換える）などが含まれます。

名前付きルーティングおすすめのルーティング管理方法です：

```dart
// ルーティングを定義
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// ナビゲーション
Navigator.pushNamed(context, '/detail');
```

## ビルドと公開

### Android公開

keytoolで署名キーを生成：

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

android/app/build.gradleで署名を構成し、その後プロジェクトルートで以下を実行：

```bash
flutter build apk --release
```

### iOS公開

Xcodeで署名証明書とプロビジョニングプロファイルを構成し、その後以下を実行：

```bash
flutter build ios --release
```

iOS公開にはApple開発者アカウントと соответствующие証明書が必要です。
