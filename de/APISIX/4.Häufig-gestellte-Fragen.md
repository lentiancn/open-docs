# Häufig gestellte Fragen zu Apache APISIX

## Grundlagen

### 1. Was ist Apache APISIX?

Apache APISIX ist ein Cloud-nativer API-Gateway mit dynamischem, Echtzeit- und Hochleistungs-Verkehrsmanagement.

### 2. APISIX vs. Nginx

APISIX ist ein Gateway auf Nginx-Basis mit Unterstützung für Hot-Update ohne Neustart.

### 3. Abhängigkeiten

- etcd: Konfigurationsspeicher
- Lua: Plugin-Logik
- Nginx: HTTP-Verarbeitung

### 4. Leistung

18.000 QPS pro Kern, <0.2ms Latenz

## Installation

### 5. Quickstart-Fehler

- Docker prüfen
- Ports prüfen
- Netzwerk prüfen

### 6. Ports ändern

In docker-compose.yml oder config.yaml

### 7. Upgrade

```bash
docker pull apache/apisix:neue-version
```

## Konfiguration

### 8. Admin-API-Schlüssel ändern

In config.yaml

### 9. HTTPS aktivieren

Mit acme-Plugin oder manuellen Zertifikaten

### 10. Mehrere Domains

"Fhosts"-Feld bei Routen verwenden

## Funktionen

### 11. Canary Release

traffic-split-Plugin mit Gewichtung

### 12. Circuit Breaker

Health-Checks passiv konfigurieren

### 13. CORS

cors-Plugin aktivieren

### 14. Ratenbegrenzung

limit-count, limit-req, limit-conn Plugins

### 15. Authentifizierung

Consumer erstellen + Plugin auf Route aktivieren

## Monitoring

### 16. Prometheus

prometheus-Plugin aktivieren

### 17. Logs

http-logger, kafka-logger Plugins

## Sicherheit

### 18. Empfehlungen

- Standard-Schlüssel ändern
- Admin-Zugriff beschränken
- HTTPS verwenden
- Regelmäßig aktualisieren

## Fehlerbehebung

### 19. 404-Fehler

- Route existiert?
- URI stimmt?
- Backend erreichbar?

### 20. 503-Fehler

- Backend läuft?
- Netzwerk OK?
- Health-Checks zu streng?

### 21. Logs anzeigen

```bash
docker logs apisix
```
