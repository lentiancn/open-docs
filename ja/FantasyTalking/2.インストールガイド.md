# FantasyTalking インストールガイド

## 動作環境

FantasyTalkingをインストールする前に、開発環境が以下の要件を満たしていることを確認してください。OSについては、プロジェクトはWindows、macOS、Linuxなどの主流OSをサポートし、最適なパフォーマンスを得るために64ビットシステムの使用をお勧めします。Node.jsのバージョンは14.0以上、Pythonのバージョンは3.7以上が必要です。Pythonバックエンドを使用する場合は、FFmpegオーディオ処理ツールもインストールする必要があります。

ハードウェアについては、スムーズな音声合成体验を得るために、システムには少なくとも4GBのRAMとデュアルコアプロセッサが必要です。リアルタイム音声合成を必要とするアプリケーションでは、I/O遅延を減らすためにSSDストレージの使用をお勧めします。モデルトレーニングやカスタム音声開発を計画している場合は、CUDA対応のグラフィックスカードが必要です。

## インストール方法

FantasyTalkingは複数のインストール方法をサポートしています。開発者はニーズとテクノロジース택に基づいて最も適切な方法を選択できます。

### npmを使用したインストール（Node.js）

```bash
# コアライブラリをインストール
npm install fantasy-talking

# オプションコンポーネントをインストール
npm install fantasy-talking-ui  # Web UIコンポーネント
npm install fantasy-talking-node  # Node.jsバックエンドサポート
```

### pipを使用したインストール（Python）

```bash
# コアライブラリをインストール
pip install fantasy-talking

# オプションコンポーネントをインストール
pip install fantasy-talking[ui]  # Web UIコンポーネント
pip install fantasy-talking[all]  # すべてのオプションコンポーネント
```

### ソースからのビルド

```bash
# リポジトリをクローン
git clone https://github.com/fantasytalking/fantasy-talking.git
cd fantasy-talking

# Node.js依存関係をインストール
npm install

# Python依存関係をインストール（オプション）
pip install -r requirements.txt

# プロジェクトをビルド
npm run build
```

## クイックスタート

### JavaScript/TypeScript使用例

```javascript
const FantasyTalking = require('fantasy-talking');

// クライアントを初期化
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  // オプション：声のスタイルを指定
  voice: {
    style: 'neutral',
    language: 'ja-JP',
    speed: 1.0,
    pitch: 1.0
  }
});

// 音声を合成
async function synthesize() {
  const audio = await client.speak('こんにちは、これはFantasyTalkingのテスト音声です');
  console.log('音声合成成功');
  
  // ファイルに保存
  const fs = require('fs');
  fs.writeFileSync('output.wav', audio);
  
  // または直接再生
  client.play(audio);
}

synthesize();
```

### Python使用例

```python
from fantasy_talking import FantasyTalking

# クライアントを初期化
client = FantasyTalking(
    api_key='your-api-key',
    voice={
        'style': 'neutral',
        'language': 'ja-JP',
        'speed': 1.0,
        'pitch': 1.0
    }
)

# 音声を合成
audio = client.speak('こんにちは、これはFantasyTalkingのテスト音声です')

# ファイルに保存
with open('output.wav', 'wb') as f:
    f.write(audio)

print('音声合成成功')
```

## 設定オプション

FantasyTalkingは、ニーズに応じて調整できる豊富な設定オプションを提供します。

### 基本設定

| パラメータ | タイプ | デフォルト | 説明 |
|-----------|--------|-----------|------|
| apiKey | string | - | APIキー必須 |
| apiUrl | string | デフォルト | APIサーバーアドレス |
| timeout | number | 30000 | タイムアウト（ミリ秒）|
| retries | number | 3 | リトライ回数 |

### 音声設定

| パラメータ | タイプ | デフォルト | 説明 |
|-----------|--------|-----------|------|
| voice.style | string | neutral | 声のスタイル: neutral, happy, sad, angry, excited |
| voice.language | string | ja-JP | 言語コード |
| voice.speed | number | 1.0 | 話速、範囲 0.5-2.0 |
| voice.pitch | number | 1.0 | ピッチ、範囲 0.5-2.0 |
| voice.volume | number | 1.0 | 音量、範囲 0-1.0 |

### 上級設定

上級ユーザーは、より精细な制御のために追加のパラメータを設定できます：

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  
  // オーディオ出力フォーマット
  audio: {
    format: 'wav',  // wav, mp3, ogg
    sampleRate: 16000,  // 8000, 16000, 24000, 48000
    bitDepth: 16  // 8, 16, 24
  },
  
  // 対話設定
  dialogue: {
    enable: true,
    contextWindow: 5,  // 保持する対話ターン数
    systemPrompt: 'あなたはriendlyな音声アシスタントです'
  },
  
  // ストリーミング出力
  stream: {
    enable: true,
    chunkSize: 1024  // ストリーミング出力のチャンクサイズ
  }
});
```

## サービスデプロイ

### Dockerデプロイ

プロジェクトは簡単なデプロイのためのDockerイメージを提供します：

```bash
# イメージをプル
docker pull fantasytalking/server:latest

# コンテナを実行
docker run -d \
  --name fantasy-talking \
  -p 8080:8080 \
  -e API_KEY=your-api-key \
  -e PORT=8080 \
  fantasytalking/server:latest
```

### セルフホストサービス

サービスを完全に制御する必要がある場合は、独自のバックエンドサービスをデプロイできます：

```bash
# バックエンドコードをクローン
git clone https://github.com/fantasytalking/server.git
cd server

# 環境変数を設定
cp .env.example .env
# .envファイルを編集し、APIキーとその他の設定を入力

# サービスを開始
npm run start
```

サービスの開始後、RESTful APIを通じて音声合成リクエストを行うことができます。詳細なAPIドキュメントは公式ドキュメントを参照してください。

## インストール確認

インストール後、FantasyTalkingが正常に動作していることを確認できます：

```bash
# コマンドラインツールでテスト
fantasy-talking --text "テスト音声" --output test.wav

# またはサンプルを実行
npm run example
```

すべてが正常であれば、音声ファイルが生成されるはずです。一般的なインストール問題と解決策はFAQページを参照してください。
