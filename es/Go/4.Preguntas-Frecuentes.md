# Preguntas Frecuentes de Go

> Respuestas a preguntas comunes sobre Go

---

## Instalación

### Q: ¿Comando no encontrado después de la instalación?

**A**: 
- Verificar que Go esté agregado a la variable PATH
- Windows se instala por defecto en C:\Go, necesita agregarse a PATH
- Linux/macOS generalmente necesitan agregar manualmente a ~/.bashrc o ~/.zshrc

### Q: ¿Cómo verificar si Go está instalado?

**A**: 
```bash
go version
go env GOPATH
```

### Q: ¿Cómo actualizar Go?

**A**: 
- Windows: Descargar y reinstallar nueva versión
- macOS: `brew upgrade go`
- Linux: Descargar nueva versión para reemplazar o usar gestor de paquetes

## Sintaxis

### Q: ¿Go tiene mecanismo de excepciones?

**A**: Go no tiene excepciones tradicionales. Los errores se pasan a través de valores de retorno:
```go
resultado, err := hacerAlgo()
if err != nil {
    // Manejar error
}
```

### Q: ¿Por qué no se pueden mezclar := y var?

**A**: 
- `:=` es declaración corta de variable, usado dentro de funciones
- `var` es declaración explícita, puede usarse fuera de funciones
- No pueden usarse juntos para la misma variable

### Q: ¿Go soporta genéricos?

**A**: Go 1.18+ soporta genéricos:
```go
func Imprimir[T any](s []T) {
    for _, v := range s {
        fmt.Println(v)
    }
}
```

## Concurrencia

### Q: ¿Cuál es la diferencia entre goroutines y hilos?

**A**: 
- Las goroutines son administradas por el runtime de Go, más ligeras que los hilos
- Crear miles de goroutines no agotará recursos
- Comunicación a través de canales

### Q: ¿Deben cerrarse los canales?

**A**: 
- Generalmente el emisor cierra el canal
- El receptor puede verificar si el canal está cerrado usando `v, ok := <-ch`
- No enviar a canales cerrados

### Q: ¿Cómo esperar elegantemente a que terminen las goroutines?

**A**: Usar sync.WaitGroup:
```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(i int) {
        defer wg.Done()
        fmt.Println(i)
    }(i)
}

wg.Wait()
```

## Gestión de Paquetes

### Q: ¿Cuál es la diferencia entre GOPATH y Go Modules?

**A**: 
- GOPATH: Método antiguo de gestión de dependencias, dependencias en $GOPATH/src
- Go Module (go.mod): Enfoque moderno, mejor control de versiones

### Q: ¿Dependencias descargan lentamente?

**A**: 
```bash
# Usar espejo de China
go env -w GOPROXY=https://goproxy.cn,direct

# También puede configurar
go env -w GOPROXY=https://proxy.golang.org,direct
```

### Q: ¿Cómo ver relaciones de dependencias?

**A**: 
```bash
go mod graph
go list -m all
```

## Rendimiento

### Q: ¿Cómo realizar análisis de rendimiento?

**A**: 
```bash
# Perfilado de CPU
go test -cpuprofile=cpu.out

# Perfilado de memoria
go test -memprofile=mem.out

# Usar pprof para analizar
go tool pprof cpu.out
```

### Q: ¿Cómo optimizar programas Go?

**A**: 
- Usar strings.Builder para concatenación de cadenas
- Evitar asignaciones de memoria innecesarias
- Usar pools (sync.Pool) para reutilización de objetos
- Usar canales eficientemente

## Herramientas

### Q: ¿Cuál es la diferencia entre go fmt y go vet?

**A**: 
- go fmt: Formatear estilo de código
- go vet: Análisis estático, verificar errores potenciales

### Q: ¿Cómo generar documentación?

**A**: 
```bash
go doc <nombre-paquete>
go doc <nombre-función>

# Servidor de documentación local
go doc -http=:6060
```

## Errores

### Q: ¿Cuáles son errores de compilación comunes?

**A**: 
- `undefined: xxx` - Variable no definida
- `mismatched types` - Tipos no coinciden
- `invalid character` - Problema de codificación
- `package xxx is not in GOROOT` - Error de ruta de paquete

### Q: ¿Qué hacer con panic en tiempo de ejecución?

**A**: 
- Usar defer y recover:
```go
func llamadaSegura() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recuperado:", r)
        }
    }()
    // Código que puede generar panic
}
```
