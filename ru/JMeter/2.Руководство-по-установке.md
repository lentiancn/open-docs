# Руководство по установке JMeter

Это руководство содержит подробные инструкции по установке и настройке Apache JMeter в различных операционных системах.

## Системные требования

- **Среда выполнения Java**: JMeter требует Java 8 или выше
- **Операционная система**: Поддерживает любую ОС, совместимую с Java, включая Linux, Windows и Mac OSX
- **Оперативная память**: Рекомендуется не менее 4 ГБ свободной памяти (может потребоваться больше в зависимости от масштаба тестирования)
- **Место на диске**: Рекомендуется не менее 1 ГБ свободного места

## Загрузка JMeter

1. Посетите официальную страницу загрузки JMeter: https://jmeter.apache.org/download_jmeter.cgi
2. Загрузите последнюю бинарную версию JMeter (Binaries)
3. Рекомендуется также загрузить файлы подписей PGP или SHA-512 для проверки целостности загрузки

## Установка в Windows

### Распаковка

1. Найдите загруженный архив Apache JMeter (apache-jmeter-x.x.zip)
2. Выберите подходящий каталог установки (избегайте путей с пробелами и специальными символами)
3. Распакуйте архив в выбранный каталог
4. JMeter установлен

### Проверка установки

1. Перейдите в папку bin в каталоге установки JMeter
2. Дважды щёлкните jmeter.bat (пакетный файл Windows)
3. Если всё в порядке, запустится графический интерфейс JMeter

## Установка в Linux и Mac OSX

### Распаковка

1. Найдите загруженный архив Apache JMeter (apache-jmeter-x.x.tar.gz)
2. Распакуйте с помощью следующей команды:
   ```bash
   tar -xf apache-jmeter-x.x.tar.gz
   ```
3. JMeter установлен

### Настройка переменных среды (Необязательно)

1. Откройте файл конфигурации терминала (~/.bashrc или ~/.profile)
2. Добавьте следующие переменные среды:
   ```bash
   export JMETER_HOME=/path/to/apache-jmeter-x.x
   export PATH=$PATH:$JMETER_HOME/bin:$JMETER_HOME/lib
   ```
3. Сохраните и обновите конфигурацию:
   ```bash
   source ~/.bashrc
   ```

### Проверка установки

1. Выполните в терминале:
   ```bash
   jmeter --version
   ```
2. Если отображается информация о версии, установка прошла успешно

## Запуск JMeter

### Графический режим

- **Windows**: Дважды щёлкните jmeter.bat в каталоге bin
- **Linux/Mac**: Выполните команду jmeter в терминале

### Режим командной строки

Для нагрузочного тестирования рекомендуется режим командной строки для снижения потребления ресурсов:

```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/output
```

Параметры:
- -n: Запустить в режиме без GUI
- -t: Указать файл плана тестирования
- -l: Указать файл результатов
- -e: Создать отчёт после завершения теста
- -o: Указать каталог вывода отчёта

## Структура каталогов

- **bin**: Содержит исполняемые файлы и сценарии запуска
- **lib**: Содержит библиотеки зависимостей JMeter
- **docs**: Содержит официальную документацию JMeter
- **extras**: Содержит дополнительные вспомогательные инструменты
- **printable_docs**: Содержит руководство пользователя в печатном формате

## Управление зависимостями

Каталог lib JMeter содержит основные зависимости. Чтобы добавить дополнительные плагины или библиотеки:

1. Скопируйте JAR-файлы в каталог lib
2. Некоторые плагины необходимо поместить в каталог lib/ext
3. Перезапустите JMeter, чтобы новые плагины вступили в силу

## Распространённые проблемы при установке

- **Java не найдена**: Убедитесь, что Java правильно установлена и переменная среды JAVA_HOME настроена
- **Недостаточно памяти**: Измените параметры размера кучи в bin/jmeter.sh или jmeter.bat
- **Ошибка запуска**: Проверьте, установлена ли правильная версия Java (Java 8 или выше)
