# Node.js 설치 가이드

## 공식 설치 방법

Node.js는 다양한 운영체제와 사용자 선호도에 맞춰 여러 가지 공식 설치 방법을 제공합니다. 모든 공식 방법은 Node.js 프로젝트에서 직접 제공하는 인증된 수정되지 않은 바이너리를 보장합니다.

### 공식 웹사이트에서 다운로드

Node.js를 설치하는 가장 간단한 방법은 [Node.js 공식 웹사이트](https://nodejs.org/ko/download)에서 적절한 설치 프로그램을 다운로드하는 것입니다.

**사용 가능한 버전:**
- **LTS (장기 지원)**: 대부분의 사용자에게 권장됩니다. 안정성과 장기 지원을 제공합니다.
- **Current**: 최신 기능을 포함하며 지원 주기가 짧습니다. 개발 및 테스트에 적합합니다.

### 패키지 관리자

#### macOS (Homebrew)
```bash
# Homebrew가 아직 설치되지 않은 경우
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# LTS 버전 설치
brew install node@lts

# 또는 최신 Current 버전 설치
brew install node
```

#### Linux (다양한 배포판)

**Ubuntu/Debian:**
```bash
# NodeSource 저장소 사용 (권장)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**CentOS/RHEL/Fedora:**
```bash
# NodeSource 저장소 사용
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs
# 또는 최신 버전의 경우: sudo dnf install nodejs
```

#### Windows
- 공식 웹사이트에서 Windows 설치 프로그램(.msi) 다운로드
- 설치 프로그램 실행 및 설정 마법사 따라 진행
- Node.js 및 npm이 자동으로 PATH에 추가됩니다

### 검증

설치 후 Node.js 및 npm이 올바르게 설치되었는지 확인합니다:

```bash
# Node.js 버전 확인
node --version

# npm 버전 확인  
npm --version

# 기본 기능 테스트
node -e "console.log('Hello Node.js!')"
```

### 여러 Node.js 버전 관리

다른 프로젝트에서 다른 Node.js 버전을 사용해야 하는 개발자의 경우, 버전 관리자가 훌륭한 솔루션을 제공합니다.

#### nvm (Node Version Manager) - macOS/Linux
```bash
# nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# LTS 버전 설치 및 사용
nvm install --lts
nvm use --lts

# 특정 버전 설치
nvm install 20.14.0
nvm use 20.14.0

# 사용 가능한 버전 나열
nvm list-remote
```

#### nvm-windows - Windows
- [nvm-windows GitHub 릴리스](https://github.com/coreybutler/nvm-windows/releases)에서 다운로드
- 설치 지침 따르기
- Unix 기반 nvm과 유사한 명령어 사용

### 환경 변수

Node.js 및 npm 실행 파일 경로가 시스템 PATH 환경 변수에 추가되었는지 확인합니다.

- **Windows**: 일반적으로 자동으로 PATH에 추가됨
- **macOS/Linux**: 패키지 관리자를 사용하는 경우 일반적으로 자동으로 구성됨

명령어를 찾을 수 없는 경우 수동으로 PATH에 추가합니다:
```bash
# 일반적으로 /usr/local/bin 또는 ~/.nvm/versions/node/vXX.XX.XX/bin에 위치
export PATH=$PATH:/usr/local/bin
```

### 문제 해결

#### 권한 문제
Linux/macOS에서 npm 전역 패키지 설치 시 권한 문제가 발생하는 경우 다음을 권장합니다:
- nvm을 사용하여 Node.js 관리
- 또는 npm 전역 디렉토리 구성:
  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  export PATH=~/.npm-global/bin:$PATH
  ```

#### 네트워크 문제
다운로드 속도가 느리거나 연결할 수 없는 경우 npm을 로컬 미러를 사용하도록 구성할 수 있습니다:
```bash
# 타오바오 미러 설정 (중국)
npm config set registry https://registry.npmmirror.com

# 또는 임시로 사용
npm install --registry https://registry.npmmirror.com
```
