# Guide d'installation

Jira propose plusieurs options d'installation pour répondre aux besoins des équipes de différentes tailles et capacités techniques. Ce guide fournit les étapes détaillées pour le déploiement cloud et Data Center sur site.

## Installation de la version cloud

La version cloud est le moyen le plus rapide de commencer avec Jira, sans nécessité de gestion d'infrastructure.

### Inscription

Premièrement, visitez le site web de Jira et cliquez sur le bouton d'inscription gratuite. Entrez votre adresse e-mail professionnelle pour créer un compte. Atlassian identifira automatiquement les membres de l'équipe et facilitera la collaboration.

### Sélection du plan

La version cloud propose quatre niveaux de tarifs :

**Plan Free** est gratuit définitivement pour jusqu'à 10 utilisateurs. Il inclut des objectifs et projets illimités, des listes de tâches et des vues de tableau, 100 règles d'automatisation par mois et 2 Go de stockage.

**Plan Standard** coûte 7,91 $ par utilisateur par mois. Il inclut les rôles et permissions utilisateur, la collaboration externe, la résidence des données multirégion, 1 700 règles d'automatisation par mois et 250 Go de stockage.

**Plan Premium** coûte 14,54 $ par utilisateur par mois. Il inclut la planification inter-équipes et la gestion des dépendances, des processus d'approbation personnalisables, un stockage illimité, un support 24/7 et un SLA de disponibilité de 99,9%.

**Plan Enterprise** est conçu pour les grandes entreprises. Il inclut l'analytique avancée et la sécurité de niveau entreprise, le support de plusieurs sites, des règles d'automatisation illimitées et un SLA de disponibilité de 99,95%.

### Création du site

Après l'inscription, le système vous guide pour créer votre premier site Jira. Vous pouvez créer un site pour votre organisation ou usage personnel, et sélectionner des modèles de projet adaptés aux besoins de votre équipe.

### Ajout d'utilisateurs

Après avoir créé un site, vous pouvez ajouter des membres de l'équipe via le panneau d'administration. Les utilisateurs peuvent être gratuits ou payants selon le plan choisi.

## Déploiement Data Center sur site

Les entreprises nécessitant un contrôle strict des données ou ayant des exigences de sécurité spéciales peuvent choisir la version Data Center sur site.

### Exigences système

La version Data Center nécessite ce qui suit :

Le support du système d'exploitation inclut Linux (Red Hat Enterprise Linux, Ubuntu Server, CentOS, etc.) et Windows Server.

Le support de base de données inclut PostgreSQL, Oracle, Microsoft SQL Server et d'autres bases de données entreprise.

Mémoire : au moins 8 Go de RAM recommandés, 16 Go ou plus pour les environnements de production.

Espace disque : dépend du volume de données, au moins 100 Go d'espace disponible recommandés.

### Étapes d'installation

Premièrement, téléchargez le programme d'installation Data Center depuis le site web d'Atlassian. Exécutez l'installeur et suivez l'assistant pour sélectionner le répertoire d'installation et configurer les connexions à la base de données. Après l'installation, accédez à l'interface d'administration via un navigateur pour la configuration initiale.

### Migration des données

Si vous migrez depuis une version antérieure, utilisez les outils d'importation et d'exportation de Jira pour migrer les données. Il est recommandé de sauvegarder toutes les données avant la migration et de vérifier les résultats de la migration dans un environnement de test.

## Applications mobiles

Jira propose des applications mobiles iOS et Android, vous permettant de voir et de mettre à jour les problèmes n'importe où. Les applications mobiles supportent la visualisation des problèmes, la création de commentaires, les mises à jour de statut et plus, adaptées aux membres de l'équipe qui doivent travailler en déplacement.

## Configuration initiale

Quelle que soit l'option de déploiement, la configuration initiale comprend :

Configurer la structure du projet, créer des projets et tableaux adaptés à votre équipe. Configurer les flux de travail, définir les statuts de problème et les règles de transition. Ajouter des membres d'équipe et définir les permissions. Intégrer d'autres outils comme Slack et GitHub. Établir des règles d'automatisation pour réduire les opérations manuelles.
