# Vue.js 使用手冊

## 應用實例

每個 Vue 應用都是通過 `createApp` 函數創建一個新的應用實例：

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* 根組件選項 */
})
```

### 根組件
傳入 `createApp` 的對象實際上是一個組件，每個應用都需要一個「根組件」，其他組件將作為其子組件。

### 掛載應用
應用實例必須在調用了 `.mount()` 方法後才會渲染出來：

```javascript
app.mount('#app')
```

### 應用配置
應用實例會暴露一個 `.config` 對象允許配置應用級選項：

```javascript
// 應用級錯誤處理器
app.config.errorHandler = (err) => {
  /* 處理錯誤 */
}

// 註冊全局組件
app.component('TodoDeleteButton', TodoDeleteButton)
```

## 模板語法

Vue 使用基於 HTML 的模板語法，使我們能夠聲明式地將組件實例的數據綁定到呈現的 DOM 上。

### 文本插值
最基本的數據綁定形式是文本插值，使用「Mustache」語法（雙大括號）：

```html
<span>Message: {{ msg }}</span>
```

### 原始 HTML
使用 `v-html` 指令插入 HTML：

```html
<p v-html="rawHtml"></p>
```

**安全警告**：動態渲染任意 HTML 非常危險，容易造成 XSS 漏洞。僅在內容安全可信時使用 `v-html`。

### Attribute 綁定
使用 `v-bind` 指令響應式地綁定 attribute：

```html
<!-- 完整語法 -->
<div v-bind:id="dynamicId"></div>

<!-- 簡寫語法 -->
<div :id="dynamicId"></div>

<!-- 同名簡寫（Vue 3.4+） -->
<div :id></div>
```

### 布爾型 Attribute
布爾型 attribute 依據 true/false 值決定是否存在於元素上：

```html
<button :disabled="isButtonDisabled">Button</button>
```

### 動態綁定多個值
通過不帶參數的 `v-bind` 綁定對象中的多個 attribute：

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript 表達式

Vue 在所有數據綁定中都支持完整的 JavaScript 表達式：

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### 調用函數
可以在綁定表達式中使用組件暴露的方法：

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### 受限的全局訪問
模板表達式僅能訪問有限的全局對象列表（如 Math、Date），其他全局對象需通過 `app.config.globalProperties` 顯式添加。

## 指令 (Directives)

指令是帶有 `v-` 前綴的特殊 attribute。

### 參數 (Arguments)
某些指令需要參數，通過冒號隔開：

```html
<!-- v-bind 參數 -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on 參數 -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### 動態參數
在指令參數上使用 JavaScript 表達式，包含在方括號內：

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

### 修飾符 (Modifiers)
以點開頭的特殊後綴，表明指令需要特殊處理：

```html
<form @submit.prevent="onSubmit">...</form>
```

## 響應式基礎

### 聲明響應式狀態
使用 `ref()` 創建響應式引用：

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### 響應式對象
使用 `reactive()` 創建響應式對象：

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### 計算屬性
使用 `computed()` 創建計算屬性：

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### 偵聽器
使用 `watch()` 偵聽響應式數據的變化：

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// 偵聽單個 ref
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## 組件基礎

### 定義組件
```javascript
// 定義組件
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### 傳遞 Props
```javascript
// 子組件
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// 父組件
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### 發送事件
```javascript
// 子組件
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### 插槽 (Slots)
```html
<!-- 子組件 -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- 父組件 -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
