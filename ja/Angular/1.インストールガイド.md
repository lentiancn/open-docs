## 1. 前提条件

### 1.1 Node.jsの要件

AngularにはNode.js環境が必要です。AngularのバージョンによってNode.jsの要件が異なります：

| Angularバージョン | 最小Node.jsバージョン | 推奨Node.jsバージョン |
|-------------------|----------------------|----------------------|
| Angular 17+ | Node.js 18.13.0+ | Node.js 20 LTS |
| Angular 16 | Node.js 16.14.0+ | Node.js 18 LTS |
| Angular 15 | Node.js 14.20.0+ | Node.js 16 LTS |
| Angular 14 | Node.js 14.20.0+ | Node.js 16 LTS |

> 💡 **ヒント**：最高の安定性を得るには、Node.js LTS（長期サポート）バージョンの使用をお勧めします。

### 1.2 npmの要件

Angular CLIはnpm（Node Package Manager）で配布されます。npmはNode.jsと一緒にインストールされます。

- 最小要件：npm 6.14.0+
- 推奨：npm 8+（Node.js 20に付属）

### 1.3 オプション：yarnまたはpnpm

packageマネージャーとしてyarnやpnpmを使用する場合：

```bash
# yarnのインストール（オプション）
npm install -g yarn

# pnpmのインストール（オプション）
npm install -g pnpm
```

## 2. Windowsへのインストール

### 2.1 方法1：npmを使用（推奨）

これは最もシンプルで一般的なインストール方法です。

#### ステップ1：Node.jsのインストール

1. Node.js公式ウェブサイトにアクセス：https://nodejs.org/
2. **LTS（長期サポート）**バージョンをダウンロード（Node.js 20 LTSを推奨）
3. インストーラーを実行
4. インストールウィザードで：
   - 「Add to PATH」にチェックを入れる（環境変数に追加）
   - 他のオプションはデフォルトのまま

#### ステップ2：Node.jsとnpmの確認

**コマンドプロンプト**（cmd）または**PowerShell**を開き、以下のコマンドを実行：

```cmd
node --version
npm --version
```

バージョン番号が表示されれば、インストール成功です。

#### ステップ3：Angular CLIのインストール

コマンドプロンプトまたはPowerShellで実行：

```cmd
npm install -g @angular/cli
```

> ⚠️ **注意**：`-g`はグローバルインストールを示します。これにより、どのディレクトリでも`ng`コマンドを使用できます。

#### ステップ4：Angular CLIの確認

```cmd
ng version
```

成功すれば、Angular CLIのバージョン情報が表示されます。

---

### 2.2 方法2：管理者権限でのインストール

権限エラーが発生した場合は管理者として実行：

1. 「コマンドプロンプト」または「PowerShell」を右クリック
2. 「管理者として実行」を選択
3. インストールコマンドを実行：

```cmd
npm install -g @angular/cli
```

---

### 2.3 方法3：Windows Package Manager（winget）の使用

Windows 10/11を使用している場合：

```cmd
winget install OpenJS.NodeJS.LTS
winget install Angular.AngularCLI
```

---

### 2.4 方法4：Chocolateyの使用

Chocolateyパッケージマネージャーがインストールされている場合：

```cmd
choco install nodejs-lts
choco install angular-cli
```

## 3. macOSへのインストール

### 3.1 方法1：npmを使用（推奨）

#### ステップ1：Node.jsのインストール

**方法A：Homebrewを使用（推奨）**

```bash
# Homebrewのインストール（まだない場合）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Node.js LTSのインストール
brew install node@20
```

**方法B：直接ダウンロード**

1. https://nodejs.org/にアクセス
2. macOS Installer (.pkg)をダウンロード
3. インストーラーを実行

#### ステップ2：インストールの確認

```bash
node --version
npm --version
```

#### ステップ3：Angular CLIのインストール

```bash
sudo npm install -g @angular/cli
```

> 💡 **ヒント**：macOSでは管理者権限（sudo）が必要な場合があります。

#### ステップ4：インストールの確認

```bash
ng version
```

---

### 3.2 方法2：nvmを使用した複数バージョン管理

複数のNode.jsバージョンを同時に使用する必要がある場合は、nvmを使用：

```bash
# nvmのインストール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# nvmでNode.jsをインストール
nvm install 20
nvm use 20

# 次にAngular CLIをインストール
npm install -g @angular/cli
```

---

### 3.3 方法3：yarnの使用

```bash
# yarnのインストール
brew install yarn

# yarnでAngular CLIをインストール
yarn global add @angular/cli
```

## 4. Linuxへのインストール

### 4.1 Ubuntu / Debian

#### ステップ1：Node.jsのインストール

```bash
# パッケージソースの更新
sudo apt update

# Node.js 20 LTSのインストール
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

#### ステップ2：インストールの確認

```bash
node --version
npm --version
```

#### ステップ3：Angular CLIのインストール

```bash
sudo npm install -g @angular/cli
```

---

### 4.2 CentOS / RHEL / Fedora

#### ステップ1：Node.jsのインストール

```bash
# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install nodejs

# Fedora
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf install nodejs
```

#### ステップ2：Angular CLIのインストール

```bash
sudo npm install -g @angular/cli
```

---

### 4.3 Arch Linux

```bash
sudo pacman -S nodejs npm
sudo npm install -g @angular/cli
```

---

### 4.4 openSUSE

```bash
sudo zypper install nodejs npm
sudo npm install -g @angular/cli
```

---

### 4.5 nvmの使用（すべてのLinuxディストリビューション向け）

```bash
# nvmのインストール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# シェル設定の再読み込み
source ~/.bashrc

# Node.js 20のインストールと使用
nvm install 20
nvm use 20

# Angular CLIのインストール
npm install -g @angular/cli
```

## 6. 最初のプロジェクトの作成

インストールが完了したら、最初のAngularプロジェクトを作成できます：

### 6.1 デフォルト設定でプロジェクトを作成

```bash
ng new my-first-app
```

これにより：
- `my-first-app`という名前のディレクトリが作成されます
- プロジェクトの依存関係がインストールされます
- Angular環境が設定されます

### 6.2 インストールをスキップ（プロジェクトファイルのみ作成）

```bash
ng new my-first-app --skip-install
```

### 6.3 特定のオプションでプロジェクトを作成

```bash
# Gitなしでプロジェクトを作成
ng new my-first-app --skip-git

# ユニットテストなしでプロジェクトを作成
ng new my-first-app --skip-tests

# CSSの代わりにSCSSを使用
ng new my-first-app --style=scss

# インラインスタイルとテンプレートを使用（学習時に便利）
ng new my-first-app --inline-template --inline-style
```

## 7. Angular CLIのアンインストール

Angular CLIをアンインストールする必要がある場合：

### 7.1 Angular CLIのアンインストール

```bash
npm uninstall -g @angular/cli
```

### 7.2 キャッシュのクリア

```bash
npm cache clean --force
```

### 7.3 アンインストールの確認

```bash
ng version
```

「command not found」または「'ng' は、内部コマンドまたは外部コマンド、操作可能なプログラムまたはバッチ ファイルとして認識されていません」と表示されるはずです。

## 8. よくある質問

### Q1: インストール中に権限エラーが発生した場合は？

**Windows:**
- 管理者としてコマンドプロンプトまたはPowerShellを実行
- Node.jsインストールディレクトリの権限を確認

**macOS/Linux:**
- `sudo npm install -g @angular/cli`を使用
- またはnpmグローバルディレクトリの権限を変更

---

### Q2: Angular CLIを最新バージョンに更新する方法は？

```bash
npm install -g @angular/cli@latest
```

---

### Q3: 現在のAngularバージョン確認する方法は？

プロジェクトディレクトリで実行：

```bash
ng version
```

または`package.json`で確認：

```json
{
  "dependencies": {
    "@angular/core": "^17.0.0"
  }
}
```

---

### Q4: 「'ng' は、内部コマンドまたは外部コマンド」として認識されない場合は？

**Windows:**
1. Node.jsが正しくインストールされているか確認：`node --version`
2. npmグローバルモジュールパスがPATHにあるか確認
3. システム環境変数に手動でパスを追加

**macOS/Linux:**
1. npmグローバルモジュールパスを確認
2. npmグローバルbinディレクトリをPATHに追加する必要があるかもしれません：
   ```bash
   export PATH="$PATH:$(npm global bin)"
   ```

---

### Q5: インストールが遅い場合は？

国内ミラーを使用できます：

```bash
# npmミラーの設定
npm config set registry https://registry.npmmirror.com

# Angular CLIのインストール
npm install -g @angular/cli
```

---

### Q6: 異なるプロジェクトで異なるバージョンのAngular CLIを使用する場合は？

nvmを使用して複数のNode.jsバージョンを管理するか、各プロジェクトにローカルにAngular CLIをインストール：

```bash
# プロジェクトディレクトリで
npm install @angular/cli --save-dev
```

次にを使用：

```bash
npx ng version
```

---

### Q7: Node.jsバージョンが互換性しない場合は？

nvm（Node Version Manager）を使用してNode.jsバージョンを切り替え：

```bash
# 使用可能なバージョンを表示
nvm list

# 特定バージョンをインストール
nvm install 18

# 特定バージョンを使用
nvm use 18
```

---

### Q8: macOS M1/M2チップのインストール問題

Apple Silicon (M1/M2/M3)チップを使用する場合：

```bash
# arm64バージョンのNode.jsを使用していることを確認
arch -arm64 node --version

# 問題が発生した場合は、x64互換バージョンをインストール
nvm install 20 --arm64
```

---

### Q9: Angular CLIのインストールパスを確認する方法は？

```bash
npm list -g @angular/cli
```

---

### Q10: 企業ネットワークがプロキシを必要とする場合は？

```bash
# プロキシの設定
npm config set proxy http://proxy.company.com:8080
npm config set https-proxy http://proxy.company.com:8080

# プロキシの削除
npm config delete proxy
npm config delete https-proxy
```
