# Vite Häufig Gestellte Fragen

## Allgemeine Fragen

### Was ist Vite?
Vite ist ein Frontend-Build-Tool der nächsten Generation, das von Evan You entwickelt wurde. Es bietet extrem schnellen HMR und optimierte Produktions-Builds mit nativen ES Modules.

### Ist Vite nur für Vue?
Nein, Vite ist framework-agnostisch und funktioniert mit Vue, React, Svelte, Preact und Vanilla JavaScript.

### Muss ich einen bestimmten Package Manager verwenden?
Vite funktioniert mit npm, yarn und pnpm. Verwenden Sie, was Sie bevorzugen.

## Installationsfragen

### Wie erstelle ich ein neues Vite-Projekt?
```bash
# npm
npm create vite@latest my-project -- --template vue

# yarn
yarn create vite my-project -- --template vue

# pnpm
pnpm create vite my-project -- --template vue
```

### Wie installiere ich Vite in einem bestehenden Projekt?
```bash
npm install -D vite
```

### Was sind die erforderlichen Abhängigkeiten?
Für Vue-Projekte:
```bash
npm install vue
npm install -D @vitejs/plugin-vue
```

Für React-Projekte:
```bash
npm install react react-dom
npm install -D @vitejs/plugin-react
```

## Nutzungsfragen

### Wie starte ich den Entwicklungsserver?
```bash
npm run dev
```

### Wie baue ich für Produktion?
```bash
npm run build
```

### Wie teste ich den Produktions-Build?
```bash
npm run preview
```

### Wie konfiguriere ich Vite?
Erstellen Sie `vite.config.js`:

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8000'
    }
  },
  build: {
    outDir: 'dist',
    sourcemap: true
  }
})
```

## Fehlerbehebung

### Modul nicht gefunden
- Überprüfen Sie, ob alle Abhängigkeiten installiert sind: `npm install`
- Verifizieren Sie Import-Pfade
- Stellen Sie sicher, dass package.json type: "module" hat

### CSS wird nicht angewendet
- Importieren Sie CSS in Ihrer Haupt-JavaScript-Datei
- Prüfen Sie, ob PostCSS-Plugins erforderlich sind
- Verifizieren Sie CSS-Dateipfad

### Port bereits belegt
Ändern Sie den Port in vite.config.js:
```javascript
server: {
  port: 3001
}
```

### Langsamer erster Load
- Prüfen Sie Netzwerk-Tab für große Abhängigkeiten
- Optimieren Sie Bilder und Assets
- Aktivieren Sie Lazy Loading für Routen


## Fortgeschrittene Fragen

### Wie funktioniert Vites HMR?
Vite verwendet native ES Modules und etabliert eine WebSocket-Verbindung für HMR. Wenn sich eine Datei ändert, aktualisiert Vite nur dieses Modul ohne die Seite neu zu laden.

### Kann ich Webpack Loader mit Vite verwenden?
Verwenden Sie `vite-plugin-webpack` oder konvertieren Sie Loader zu Vite-Plugins.

### Wie füge ich eigene Plugins hinzu?
```javascript
function myPlugin() {
  return {
    name: 'my-plugin',
    transform(code, id) {
      // Code transformieren
      return code
    }
  }
}

export default defineConfig({
  plugins: [myPlugin()]
})
```

## Best Practices

1. Verwenden Sie relative Imports für besseres Caching
2. Konfigurieren Sie Resolve Aliases
3. Verwenden Sie Umgebungsvariablen für Konfiguration
4. Aktivieren Sie Production Source Maps sorgfältig
5. Verwenden Sie Lazy Loading für Routen
6. Optimieren Sie Bilder vor dem Build
