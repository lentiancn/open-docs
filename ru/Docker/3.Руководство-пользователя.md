# Руководство пользователя Docker

Это руководство поможет вам быстро начать работу с Docker и освоить_common операции для повседневного использования.

## Предварительные требования

Перед началом убедитесь, что у вас есть:

- Установленный Docker (Docker Desktop или Docker Engine)
- Доступ к командной строке
- Базовые знания работы с командной строкой

## Обзор основных концепций

### Образы и контейнеры

- **Образ**: шаблон только для чтения, используемый для создания контейнеров
- **Контейнер**: запущенный экземпляр образа

## Ваш первый контейнер

### Запуск Hello World

Простейший тест — запустить образ hello-world:

```bash
docker run hello-world
```

Эта команда выполнит:
1. Проверит, существует ли образ hello-world локально
2. Если нет, загрузит из Docker Hub
3. Создаст и запустит контейнер
4. Отобразит приветственное сообщение и завершится

### Запуск интерактивного контейнера

Запустите интерактивный контейнер Ubuntu:

```bash
docker run -it ubuntu bash
```

Пояснение параметров:
- `-i`: интерактивный режим
- `-t`: выделить псевдотерминал

Внутри контейнера вы можете работать как в обычной системе Linux. Введите `exit` для выхода из контейнера.

### Запуск контейнеров в фоновом режиме

Запустите контейнер в фоновом режиме:

```bash
docker run -d nginx
```

Пояснение параметров:
- `-d`: запуск в фоновом режиме (detached mode)

Просмотр запущенных контейнеров:

```bash
docker ps
```

Остановка фонового контейнера:

```bash
docker stop <ID или имя контейнера>
```

## Управление контейнерами

### Просмотр контейнеров

Просмотр запущенных контейнеров:

```bash
docker ps
```

Просмотр всех контейнеров (включая остановленные):

```bash
docker ps -a
```

Просмотр деталей контейнера:

```bash
docker inspect <имя контейнера>
```

### Запуск и остановка контейнеров

Запуск остановленного контейнера:

```bash
docker start <имя контейнера>
```

Остановка запущенного контейнера:

```bash
docker stop <имя контейнера>
```

Перезапуск контейнера:

```bash
docker restart <имя контейнера>
```

### Удаление контейнеров

Удаление остановленного контейнера:

```bash
docker rm <имя контейнера>
```

Принудительное удаление запущенного контейнера:

```bash
docker rm -f <имя контейнера>
```

Удаление всех остановленных контейнеров:

```bash
docker container prune
```

## Управление образами

### Просмотр образов

Список локальных образов:

```bash
docker images
```

Или:

```bash
docker image ls
```

Просмотр деталей образа:

```bash
docker image inspect <имя образа>
```

### Загрузка образов

Загрузка образа из Docker Hub:

```bash
docker pull ubuntu:latest
```

Загрузка конкретной версии:

```bash
docker pull nginx:1.25
```

### Удаление образов

Удаление локального образа:

```bash
docker rmi <имя образа>
```

Или:

```bash
docker image rm <имя образа>
```

Удаление неиспользуемых образов:

```bash
docker image prune
```

## Сопоставление портов

Контейнеры по умолчанию изолированы. Вам нужно сопоставить порты для доступа к сервисам внутри контейнеров.

### Сопоставление портов

Сопоставьте порт хоста 8080 с портом контейнера 80:

```bash
docker run -d -p 8080:80 nginx
```

Теперь при посещении http://localhost:8080 отобразится приветственная страница Nginx.

### Форматы сопоставления портов

- `-p 8080:80`: сопоставить хостовый 8080 с контейнерным 80
- `-p 127.0.0.1:8080:80`: только локальный доступ на хосте
- `-p 8080-8090:80-90`: сопоставление диапазона портов

## Тома

Тома используются для сохранения данных, генерируемых контейнерами, или для предоставления доступа к файлам хоста из контейнеров.

### Создание тома

```bash
docker volume create mydata
```

### Использование тома

```bash
docker run -d -v mydata:/app/data nginx
```

### Монтирование директорий хоста

```bash
docker run -d -v /home/user/data:/app/data nginx
```

### Просмотр томов

```bash
docker volume ls
docker volume inspect mydata
```

### Удаление томов

```bash
docker volume rm mydata
docker volume prune  # Удалить неиспользуемые тома
```

## Сборка образов

### Основы Dockerfile

Создайте простой Dockerfile:

```dockerfile
# Укажите базовый образ
FROM node:18-alpine

# Установите рабочую директорию
WORKDIR /app

# Скопируйте файлы
COPY package*.json ./

# Установите зависимости
RUN npm install

# Скопируйте исходный код
COPY . .

# Откройте порт
EXPOSE 3000

# Команда запуска
CMD ["node", "index.js"]
```

### Сборка образа

```bash
docker build -t myapp:latest .
```

Пояснение параметра:
- `-t`: указать имя и тег образа

### Использование .dockerignore

Создайте файл .dockerignore для исключения файлов:

```
node_modules
npm-debug.log
.git
README.md
```

## Docker Compose

Docker Compose используется для определения и запуска многоконтейнерных приложений.

### Пример docker-compose.yml

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: postgres:15
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=secret

volumes:
  db-data:
```

### Общие команды

Запустить все сервисы:

```bash
docker-compose up
```

Запуск в фоновом режиме:

```bash
docker-compose up -d
```

Остановить все сервисы:

```bash
docker-compose down
```

Просмотр логов:

```bash
docker-compose logs -f
```

Пересобрать и запустить:

```bash
docker-compose up --build
```

## Сети

### Просмотр сетей

```bash
docker network ls
```

### Создание сети

```bash
docker network create mynetwork
```

### Подключение контейнера к сети

Подключите контейнер к сети:

```bash
docker network connect mynetwork mycontainer
```

### Общение между контейнерами

При использовании Docker Compose контейнеры могут общаться друг с другом через имена сервисов. Например:

```yaml
services:
  web:
    build: .
    depends_on:
      - db
  db:
    image: postgres:15
```

В контейнере web вы можете получить доступ к базе данных через имя хоста `db`.

## Общие техники

### Вход в запущенный контейнер

```bash
docker exec -it <имя контейнера> bash
```

### Просмотр логов контейнера

```bash
docker logs -f <имя контейнера>
```

### Копирование файлов

Из контейнера на хост:

```bash
docker cp container:/path/in/container /local/path
```

С хоста в контейнер:

```bash
docker cp /local/path container:/path/in/container
```

### Просмотр использования ресурсов

```bash
docker stats
```

### Просмотр процессов контейнера

```bash
docker top <имя контейнера>
```

### Очистка

Очистите остановленные контейнеры, неиспользуемые образы и кэш сборки:

```bash
docker system prune
```

Полная очистка:

```bash
docker system prune -a
```

## Рекомендации по использованию

### Оптимизация сборки образов

1. **Уменьшите количество слоёв образа**: объедините связанные команды
2. **Используйте многоэтапную сборку**: уменьшите размер финального образа
3. **Правильно используйте .dockerignore**: исключите ненужные файлы
4. **Используйте конкретные теги версий**: избегайте использования тега latest

### Рекомендации по безопасности

1. **Не запускайте контейнеры от root**
2. **Используйте официальные проверенные образы**
3. **Регулярно обновляйте базовые образы**
4. **Используйте Docker secrets для конфиденциальной информации** (режим Swarm) или secrets Kubernetes

### Ограничения ресурсов

Установите ограничения ресурсов для контейнеров:

```bash
docker run -d --memory="512m" --cpus="1.5" nginx
```

## Распространённые проблемы

### Проблемы с правами Docker

Если возникают ошибки прав доступа, добавьте текущего пользователя в группу docker:

```bash
sudo usermod -aG docker $USER
```

Затем выйдите из системы и войдите снова.

### Контейнеры автоматически завершаются

Контейнеры завершатся, если нет непрерывно выполняющегося процесса. Чтобы сохранить работу:

```bash
docker run -d nginx tail -f /dev/null
```

Или используйте менеджер процессов, например supervisor.

### Конфликты портов

Если порт уже используется, измените сопоставленный порт:

```bash
docker run -d -p 8081:80 nginx
```
