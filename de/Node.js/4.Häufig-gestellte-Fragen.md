# Häufig gestellte Fragen zu Node.js (FAQ)

## Allgemeine Fragen

### Was ist Node.js?
Node.js ist eine Open-Source, plattformübergreifende JavaScript-Laufzeitumgebung, die JavaScript-Code außerhalb eines Webbrowsers ausführt. Sie ermöglicht Entwicklern, JavaScript für Kommandozeilentools und serverseitiges Scripting zu verwenden, um dynamische Webseiteninhalte zu generieren, bevor die Seite an den Webbrowser des Benutzers gesendet wird.

### Warum wurde Node.js erstellt?
Node.js wurde 2009 von Ryan Dahl erstellt, um die Einschränkungen traditioneller Webserver anzugehen, die ein threadbasiertes Modell für die Handhabung gleichzeitiger Verbindungen verwendeten. Node.js führte ein ereignisgesteuertes, nicht-blockierendes I/O-Modell ein, das es leicht und effizient für den Aufbau skalierbarer Netzwerkanwendungen macht.

### Ist Node.js Single-Threaded?
Node.js-Anwendungen laufen auf einem einzigen Thread für die JavaScript-Ausführung, können aber mehrere gleichzeitige Operationen durch asynchrone E/A und die Ereignisschleife verarbeiten. Zusätzlich bietet Node.js APIs wie `worker_threads` und `child_process` an, um bei Bedarf mehrere CPU-Kerne zu nutzen.

### Was ist npm?
npm (Node Package Manager) ist der Standard-Paketmanager für Node.js und die größte Software-Registry der Welt. Es ermöglicht Entwicklern, Code-Pakete zu teilen und wiederzuverwenden, Abhängigkeiten zu verwalten und in package.json-Dateien definierte Skripte auszuführen.

## Installation und Einrichtung

### Welche Version sollte ich installieren: LTS oder Current?
- **LTS (Langfristiger Support)**: Empfohlen für Produktionsanwendungen. Bietet Stabilität, Sicherheitsupdates und langfristigen Support (insgesamt 30 Monate).
- **Current**: Enthält die neuesten Funktionen, hat aber einen kürzeren Support-Zyklus. Am besten für Entwicklung, Tests neuer Funktionen oder Lernen.

### Wie wechsle ich zwischen Node.js-Versionen?
Verwenden Sie einen Node-Version-Manager:
- **nvm** (macOS/Linux): `nvm install 18 && nvm use 18`
- **nvm-windows** (Windows): Ähnliche Befehle wie Unix-basiertes nvm
- **fnm** (Fast Node Manager): Plattformübergreifende Alternative mit besserer Leistung

### Warum erhalte ich Berechtigungsfehler bei der Installation globaler Pakete?
Dies tritt typischerweise auf, wenn npm versucht, in Systemverzeichnisse zu schreiben, die erhöhte Berechtigungen erfordern. Lösungen:
1. Konfigurieren Sie npm, um ein anderes Verzeichnis zu verwenden: `npm config set prefix ~/.npm-global`
2. Fügen Sie `~/.npm-global/bin` zu Ihrem PATH hinzu
3. Verwenden Sie einen Node-Version-Manager, der Berechtigungen ordnungsgemäß handhabt

### Wie überprüfe ich meine Node.js-Installation?
Führen Sie diese Befehle in Ihrem Terminal aus:
```bash
node --version    # Zeigt Node.js-Version an
npm --version     # Zeigt npm-Version an
node -e "console.log('Funktioniert!')"  # Testet grundlegende Funktionalität
```

## Entwicklungsfragen

### Was ist der Unterschied zwischen CommonJS und ES Modules?
- **CommonJS** (`require()`/`module.exports`): Das ursprüngliche Node.js-Modulsystem, synchron, weit verbreitet
- **ES Modules** (`import`/`export`): Moderner JavaScript-Standard, asynchron, erfordert `.mjs`-Erweiterung oder `"type": "module"` in package.json

Beispiel CommonJS:
```javascript
// math.js
exports.add = (a, b) => a + b;

// app.js
const { add } = require('./math');
```

Beispiel ES Modules:
```javascript
// math.mjs
export const add = (a, b) => a + b;

// app.mjs
import { add } from './math.mjs';
```

### Wie handle ich asynchrone Operationen?
Drei Hauptansätze:

1. **Callbacks** (traditionell, kann aber zu "Callback-Hölle" führen):
```javascript
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});
```

2. **Promises** (bessere Fehlerbehandlung und Verkettung):
```javascript
fs.promises.readFile('file.txt', 'utf8')
  .then(data => console.log(data))
  .catch(err => console.error(err));
```

3. **Async/Await** (am lesbarsten, sieht synchron aus):
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
```

### Was ist die Ereignisschleife und wie funktioniert sie?
Die Ereignisschleife ist der Mechanismus von Node.js zur Handhabung asynchroner Operationen ohne Blockierung des Hauptthreads. Sie überprüft kontinuierlich ausstehende Callbacks und führt sie aus, wenn ihre zugehörigen Operationen abgeschlossen sind.

Phasen der Ereignisschleife:
1. **Timer**: Führt setTimeout() und setInterval() Callbacks aus
2. **Ausstehende Callbacks**: Führt I/O-Callbacks aus, die auf die nächste Schleifeniteration verschoben wurden
3. **Leerlauf, Vorbereitung**: Nur für internen Gebrauch
4. **Poll**: Ruft neue I/O-Ereignisse ab; führt I/O-bezogene Callbacks aus
5. **Check**: Führt setImmediate() Callbacks aus
6. **Schließ-Callbacks**: Führt Schließ-Ereignis-Callbacks aus (z.B. socket.on('close'))

### Wie debugge ich Node.js-Anwendungen?
Mehrere Debugging-Optionen:

1. **Integrierter Inspector**: `node --inspect app.js`
2. **Chrome DevTools**: Verbinden Sie sich mit `chrome://inspect`
3. **VS Code Debugger**: Integrierter Node.js-Debugging-Support
4. **Konsolen-Logging**: Einfach aber effektiv für grundlegendes Debugging
5. **Dedizierte Debugging-Tools**: Wie Node.js Inspector Manager (NiM)

## Leistung und Skalierung

### Wie skaliere ich Node.js-Anwendungen?
Mehrere Strategien:

1. **Clustering**: Verwenden Sie das integrierte `cluster`-Modul, um Kindprozesse zu erstellen, die Server-Ports teilen
2. **Lastverteilung**: Verwenden Sie Reverse-Proxies wie NGINX, um den Datenverkehr auf mehrere Node.js-Instanzen zu verteilen
3. **Microservices**: Teilen Sie große Anwendungen in kleinere, unabhängige Dienste auf
4. **Caching**: Implementieren Sie Caching-Ebenen (Redis, Memcached), um die Datenbanklast zu reduzieren
5. **Datenbankoptimierung**: Verwenden Sie Verbindungspooling und Abfrageoptimierung

### Was verursacht Speicherlecks in Node.js?
Häufige Ursachen:
- **Globale Variablen**: Ansammlung von Daten im globalen Bereich
- **Nicht geschlossene Event-Listener**: Entfernen von Event-Listenern nicht, wenn Objekte zerstört werden
- **Closures**: Halten von Referenzen auf große Objekte länger als nötig
- **Timer**: Nicht löschen von Intervallen und Timeouts
- **Puffer**: Nicht ordnungsgemäße Verwaltung großer Binärdaten

Erkennungswerkzeuge:
- `process.memoryUsage()` für grundlegende Überwachung
- Heap-Snapshots mit Chrome DevTools
- Speicherprofilierungswerkzeuge wie Clinic.js

### Wie handle ich CPU-intensive Aufgaben?
Da Node.js für die JavaScript-Ausführung Single-Threaded ist, können CPU-intensive Aufgaben die Ereignisschleife blockieren. Lösungen:

1. **Worker Threads**: Verwenden Sie das `worker_threads`-Modul für parallele Verarbeitung
```javascript
const { Worker } = require('worker_threads');
const worker = new Worker('./cpu-intensive-task.js');
```

2. **Kindprozesse**: Starten Sie separate Prozesse mit `child_process`
```javascript
const { spawn } = require('child_process');
const child = spawn('python', ['script.py']);
```

3. **Auslagern an externe Dienste**: Verwenden Sie Nachrichtenwarteschlangen (RabbitMQ, Kafka), um schwere Aufgaben asynchron zu verarbeiten
4. **Algorithmus-Optimierung**: Profilieren und optimieren Sie den eigentlichen Code, der Leistungsprobleme verursacht

## Sicherheit

### Wie sichere ich meine Node.js-Anwendung?
Wichtige Sicherheitspraktiken:

1. **Eingabevalidierung**: Validieren und bereinigen Sie immer Benutzereingaben
2. **Abhängigkeitsverwaltung**: Aktualisieren Sie regelmäßig Abhängigkeiten und führen Sie `npm audit` aus
3. **Umgebungsvariablen**: Speichern Sie Geheimnisse in Umgebungsvariablen, niemals im Code
4. **Ratenbegrenzung**: Verhindern Sie Brute-Force-Angriffe mit Ratenbegrenzungs-Middleware
5. **Helmet.js**: Verwenden Sie Sicherheits-Header-Middleware
6. **CORS**: Konfigurieren Sie Cross-Origin Resource Sharing ordnungsgemäß
7. **HTTPS**: Verwenden Sie immer HTTPS in der Produktion
8. **Authentifizierung**: Implementieren Sie ordnungsgemäße Authentifizierung und Autorisierung

### Wie verhindere ich häufige Schwachstellen?
- **SQL-Injection**: Verwenden Sie parametrisierte Abfragen oder ORMs
- **XSS (Cross-Site Scripting)**: Escapen Sie Benutzereingaben vor dem Rendern
- **CSRF (Cross-Site Request Forgery)**: Verwenden Sie CSRF-Tokens
- **Prototyp-Verschmutzung**: Vermeiden Sie unsichere Objekt-Zusammenführungs-Funktionen
- **Pfad-Traversierung**: Validieren Sie Dateipfade und verwenden Sie `path.resolve()` mit Vorsicht

## Bereitstellung und Produktion

### Welchen Prozessmanager sollte ich verwenden?
Beliebte Optionen:

1. **PM2**: Am beliebtesten, funktionsreich, unterstützt Clustering und Überwachung
2. **Forever**: Einfacher Prozessmanager, gut für grundlegende Anwendungsfälle
3. **Systemd**: Nativer Linux-Dienstmanager, integriert sich gut mit dem Betriebssystem
4. **Docker**: Containerisierungsansatz, hervorragend für Microservices

### Wie handle ich umgebungsspezifische Konfiguration?
Verwenden Sie Umgebungsvariablen und Konfigurationsdateien:

```javascript
// config.js
const config = {
  development: {
    port: 3000,
    dbUrl: 'mongodb://localhost:27017/myapp'
  },
  production: {
    port: process.env.PORT || 8080,
    dbUrl: process.env.DATABASE_URL
  }
};

module.exports = config[process.env.NODE_ENV || 'development'];
```

### Welche Überwachungswerkzeuge sollte ich verwenden?
- **Application Performance Monitoring (APM)**: New Relic, Datadog, AppSignal
- **Protokollierung**: Winston, Bunyan, mit zentralisierter Protokollierung (ELK-Stack, Splunk)
- **Gesundheitsprüfungen**: Benutzerdefinierte Endpunkte, die die Anwendungsgesundheit überprüfen
- **Metriken**: Prometheus mit Grafana zur Visualisierung
- **Fehlerverfolgung**: Sentry, Rollbar für Ausnahmeüberwachung

### Wie implementiere ich ein sanftes Herunterfahren?
Behandeln Sie Prozesssignale, um Ressourcen zu bereinigen:

```javascript
const server = http.createServer(app);

const shutdown = () => {
  server.close(() => {
    console.log('Server geschlossen');
    process.exit(0);
  });
  
  // Erzwungenes Herunterfahren nach 10 Sekunden
  setTimeout(() => {
    console.error('Konnte Verbindungen nicht rechtzeitig schließen, erzwinge Herunterfahren');
    process.exit(1);
  }, 10000);
};

process.on('SIGTERM', shutdown);
process.on('SIGINT', shutdown);
```
