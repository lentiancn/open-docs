# Claude Code Benutzerhandbuch

Dieses Handbuch führt Sie schnell in Claude Code ein und vermittelt die gängigen Operationen für die tägliche Entwicklung.

## Vorbereitung

Stellen Sie sicher, dass Sie haben:

- Ein offenes Terminal oder eine Eingabeaufforderung
- Ein Codeprojekt zum Arbeiten
- Ein Claude-Abonnement (Pro, Max, Teams oder Enterprise), ein Claude Console-Konto oder Zugriff über einen unterstützten Cloud-Anbieter

## Sitzung starten

Öffnen Sie ein Terminal in Ihrem Projektverzeichnis und starten Sie Claude Code:

```bash
cd /path/to/your/project
claude
```

Beim ersten Mal sehen Sie einen Willkommensbildschirm mit Sitzungsinformationen, kürzlichen Gesprächen und Updates. Geben Sie `/help` für verfügbare Befehle ein oder `/resume`, um ein vorheriges Gespräch fortzusetzen.

## Grundlegende Operationen

### Codebasis verstehen

Lassen Sie Claude Ihr Projekt analysieren:

```
Was macht dieses Projekt?
```

```
Welche Technologien verwendet dieses Projekt?
```

```
Wo ist der Haupteinstiegspunkt?
```

```
Erklären Sie die Ordnerstruktur
```

Sie können Claude auch nach seinen Fähigkeiten fragen:

```
Was kann Claude Code?
```

```
Wie erstelle ich benutzerdefinierte Fähigkeiten in Claude Code?
```

**Hinweis: Claude Code liest bei Bedarf Projektdateien. Sie müssen keinen Kontext manuell hinzufügen.**

### Ihre erste Codeänderung

Lassen Sie Claude Code tatsächlich codieren. Versuchen Sie eine einfache Aufgabe:

```
Fügen Sie eine Hello-World-Funktion zur Hauptdatei hinzu
```

Claude Code wird:
1. Die passende Datei finden
2. Die vorgeschlagenen Änderungen anzeigen
3. Ihre Genehmigung einholen
4. Die Bearbeitung durchführen

**Hinweis: Claude Code fragt immer um Erlaubnis, bevor Dateien geändert werden.**

### Git-Operationen

Claude Code macht Git-Operationen konversationell:

```
Welche Dateien habe ich geändert?
```

```
Committen Sie meine Änderungen mit einer beschreibenden Nachricht
```

```
Erstellen Sie einen neuen Branch namens feature/quickstart
```

```
Zeigen Sie mir die letzten 5 Commits
```

```
Helfen Sie mir, Merge-Konflikte zu lösen
```

### Fehler beheben und Funktionen hinzufügen

Claude ist geschickt im Debugging und der Implementierung von Funktionen.

Beschreiben Sie, was Sie in natürlicher Sprache wollen:

```
Fügen Sie eine Eingabevalidierung zum Benutzerregistrierungsformular hinzu
```

Oder beheben Sie bestehende Probleme:

```
Es gibt einen Fehler, bei dem Benutzer leere Formulare senden können - beheben Sie ihn
```

Claude Code wird:
- Den relevanten Code finden
- Den Kontext verstehen
- Eine Lösung implementieren
- Tests ausführen (falls verfügbar)

## Wesentliche Befehle

Hier sind die wichtigsten Befehle für den täglichen Gebrauch:

| Befehl | Funktion | Beispiel |
|--------|----------|----------|
| `claude` | Interaktiven Modus starten | `claude` |
| `claude "Aufgabe"` | Einmalige Aufgabe ausführen | `claude "fix the build error"` |
| `claude -p "Abfrage"` | Einmalige Abfrage, dann beenden | `claude -p "explain this function"` |
| `claude -c` | Letztes Gespräch fortsetzen | `claude -c` |
| `claude -r` | Vorheriges Gespräch fortsetzen | `claude -r` |
| `claude commit` | Git-Commit erstellen | `claude commit` |
| `/clear` | Gesprächsverlauf löschen | `/clear` |
| `/help` | Verfügbare Befehle anzeigen | `/help` |
| `exit` oder Strg+C | Claude Code beenden | `exit` |

## Tipps für Fortgeschrittene

### CLAUDE.md verwenden

Erstellen Sie eine `CLAUDE.md`-Datei im Projektverzeichnis für persistenten Projektkontext.

### Benutzerdefinierte Fähigkeiten

Erstellen Sie benutzerdefinierte Befehle wie `/review-pr` oder `/deploy-staging`.

### Multi-Agenten-Zusammenarbeit

Für komplexe Aufgaben können Sie mehrere Claude Code-Agenten gleichzeitig starten.

## Best Practices

### Seien Sie spezifisch

❌ Nicht sagen: "Behebe den Fehler"

✅ Besser sagen: "Behebe den Login-Fehler, bei dem Benutzer nach falschen Anmeldedaten einen leeren Bildschirm sehen"

### Lassen Sie Claude zuerst erkunden

Bevor Sie Änderungen vornehmen, lassen Sie Claude Ihren Code verstehen:

```
Analysieren Sie das Datenbankschema
```
