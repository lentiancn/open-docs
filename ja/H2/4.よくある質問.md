# H2 データベースよくある質問

## インストール

### Q: H2データベースをダウンロード方法は？

A: https://www.h2database.com/html/download.htmlにアクセスして、プラットフォームに適したバージョンをダウンロードしてください。WindowsインストーラーとクロスプラットフォームZIPの2つのオプションがあります。

### Q: H2に必要なJavaバージョンは？

A: H2はJava 11以上が必要です。

### Q: MavenプロジェクトでH2を使用する方法は？

A: pom.xmlに依存関係を追加します：
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### Q: ZIPを展開した後、H2を実行する方法は？

A: コマンドラインで以下を実行します：
```bash
java -jar h2.jar
```
これによりH2 Webコンソールが起動します。

## 接続

### Q: H2データベースに接続方法は？

A: JDBC接続を使用します：
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### Q: H2のデフォルトのユーザー名とパスワードは？

A: デフォルトのユーザー名は「sa」、パスワードは空です。

### Q: ファイルデータベースに接続方法は？

A:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Q: H2 TCPサーバーを起動する方法は？

A:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### Q: ポートが使用中の場合は？

A: 接続URLでポートを指定できます：
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## データの問題

### Q: メモリデータベースのデータは失われますか？

A: はい、メモリデータベース（jdbc:h2:mem:）はすべての接続が閉じるとデータが失われます。永続化が必要な場合は、ファイルデータベースモードを使用してください。

### Q: データベースファイルの場所を確認する方法は？

A: 接続URLでパスを指定します：
```java
jdbc:h2:file:/path/to/database
```

### Q: データベースファイルが破損した場合は？

A: 回復モードを試みます：
```java
jdbc:h2:file:./testdb;LOG=0
```
またはバックアップから復元します。

### Q: データベースを暗号化する方法は？

A:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## パフォーマンス

### Q: バッチ挿入のパフォーマンスを改善する方法は？

A: バッチ操作を使用し、自動コミットを無効にします：
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Q: クエリが遅い場合は？

A: 
1. 一般的なクエリ条件にインデックスを作成
2. EXPLAINを使用してクエリプランを分析
3. SELECT *を避け、必要な列のみをクエリ

### Q: メモリ使用量が多い場合は？

A: JVMヒープサイズを調整し、接続プールを適切に使用します。

## SQL互換性

### Q: H2はどのデータベースモードをサポートしていますか？

A: H2は複数のデータベースをシミュレートできます：
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

使用方法：
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### Q: 自動インクリメントIDを処理する方法は？

A: H2は複数の方法をサポートしています：
```sql
-- 方法1：IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- 方法2：SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## トランザクション

### Q: トランザクションがコミットされていない場合は？

A: コミットされていないトランザクションは、接続が閉じると自動的にロールバックされます。

### Q: デッドロックを処理する方法は？

A: H2は自動的にデッドロックを検出し、いずれかのトランザクションをロールバックします。ロックを長時間保持しないよう、トランザクションの範囲を短縮することをお勧めします。

## セキュリティ

### Q: H2データベースは安全ですか？

A: H2は開発およびテスト環境に適しています。本番環境では以下をお勧めします：
1. 暗号化されたデータベースを使用
2. 強力なパスワードを使用
3. ネットワークアクセスを制限
4. SSL接続を使用

### Q: SQLインジェクションを防ぐ方法は？

A: 文字列連結の代わりにPreparedStatementを使用します：
```java
// 正しい
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// 間違い
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## ツール

### Q: H2コンソールを使用する方法は？

A: `java -jar h2.jar -web`を実行してからhttp://localhost:8082にアクセスします

### Q: H2 Shellを使用する方法は？

A: `java -jar h2.jar -shell`または`java -jar h2.jar -tool`を実行します

### Q: H2バージョンを確認する方法は？

A:
```bash
java -jar h2.jar -version
```
またはSQLで：`SELECT H2VERSION();`

## 移行

### Q: MySQLからH2に移行する方法は？

A:
1. H2互換モードを設定：`jdbc:h2:file:db;MODE=MySQL`
2. MySQLデータをSQLスクリプトとしてエクスポート
3. H2でスクリプトを実行

### Q: H2バージョンをアップグレードするときの注意点は？

A: H2 1.xから2.xへのアップグレードには特別な注意が必要です。公式移行ガイドを参照してください：
https://www.h2database.com/html/migration-to-v2.html
