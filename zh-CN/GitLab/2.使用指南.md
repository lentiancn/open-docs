# GitLab 使用指南

GitLab 基本操作指南。

---

## 快速开始

### 克隆仓库

```bash
# HTTPS 克隆
git clone https://gitlab.example.com/用户名/项目.git

# SSH 克隆
git clone git@gitlab.example.com:用户名/项目.git
```

### 配置 Git

```bash
git config --global user.name "你的名字"
git config --global user.email "your@email.com"
```

---

## 项目操作

### 基本命令

```bash
# 创建分支
git checkout -b feature/新功能

# 提交更改
git add .
git commit -m "添加新功能"

# 推送分支
git push origin feature/新功能
```

### 创建 Merge Request

1. 推送分支后点击 "Create merge request"
2. 填写标题和描述
3. 添加审核人
4. 提交

---

## CI/CD

### 基本 .gitlab-ci.yml

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build

test:
  stage: test
  script:
    - npm test

deploy:
  stage: deploy
  script:
    - npm run deploy
  only:
    - main
```

## 用户权限

| 角色 | 描述 |
|------|------|
| Guest | 查看项目 |
| Reporter | 克隆和拉取 |
| Developer | 推送分支，创建 MR |
| Maintainer | 管理分支和设置 |
| Owner | 完全访问 |
