# Nginx 常見問題

## 安裝問題

### Nginx 安裝失敗怎麼辦？

1. 檢查系統依賴是否完整
2. 查看錯誤日誌了解具體失敗原因
3. 確保有足夠的磁碟空間
4. 檢查是否有權限問題

### 如何查看 Nginx 版本？

```bash
nginx -v
nginx -V  # 詳細資訊，包括編譯參數
```

### Nginx 支援哪些作業系統？

Nginx 支援 Linux、FreeBSD、Solaris、AIX、HP-UX、macOS 和 Windows。

## 設定問題

### 如何檢查 Nginx 設定是否正確？

```bash
nginx -t
```

### 設定檔在哪裡？

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- 編譯安裝: `/usr/local/nginx/conf/nginx.conf`

### 修改設定後需要重啟嗎？

不需要重啟，使用 `nginx -s reload` 即可重新載入設定。

## 效能問題

### 如何優化 Nginx 效能？

1. **調整 worker 程序數**

```nginx
worker_processes auto;
```

2. **增加 worker 連線數**

```nginx
events {
    worker_connections 4096;
}
```

3. **啟用 gzip 壓縮**

```bash
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **啟用瀏覽器快取**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **使用連線池**

```nginx
upstream backend {
    server backend1.example.com;
    keepalive 32;
}
```

### Nginx 佔用記憶體過高怎麼辦？

1. 減少 worker_connections 數量
2. 限制請求體大小
3. 優化快取設定
4. 檢查是否有記憶體洩漏

### 如何處理高並發？

1. 使用負載均衡
2. 啟用快取
3. 調整核心參數
4. 使用 CDN

## 安全問題

### 如何防止 DDoS 攻擊？

1. **限制連線數**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **限制請求速率**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **關閉慢速連線**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### 如何設定 SSL 憑證？

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### 如何強制使用 HTTPS？

```nginx
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

## 代理問題

### 反向代理失敗怎麼辦？

1. 檢查後端服務是否執行
2. 檢查防火牆設定
3. 驗證 proxy_pass 設定正確
4. 查看錯誤日誌

### 如何代理 WebSocket？

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### 代理後如何取得真實 IP？

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## 日誌問題

### 日誌檔案太大怎麼辦？

使用 logrotate 進行日誌輪換：

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### 如何關閉存取日誌？

```nginx
access_log off;
```

### 日誌格式怎麼設定？

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## 負載均衡問題

### 負載均衡如何選擇後端？

Nginx 支援多種負載均衡演算法：

- **輪詢（預設）**：按順序分配請求
- **加權輪詢**：按權重分配請求
- **IP 雜湊**：同一 IP 的請求分配到同一後端
- **最少連線**：分配給連線數最少的後端

### 後端伺服器當機怎麼辦？

Nginx 會自動跳過故障節點，將請求轉發到健康節點。

## 快取問題

### 如何設定代理快取？

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### 快取不生效怎麼辦？

1. 檢查快取路徑權限
2. 確認回應標頭允許快取
3. 驗證快取鍵設定正確

### 如何清除快取？

```bash
rm -rf /var/cache/nginx/*
```

## 連接埠問題

### 連接埠被佔用怎麼辦？

```bash
# 查看連接埠佔用
netstat -tulpn | grep :80

# 改用其他連接埠
listen 8080;
```

### 如何監聽多個連接埠？

```nginx
server {
    listen 80;
    listen 8080;
    server_name example.com;
}
```

## 常見錯誤代碼

### 400 Bad Request

- 請求頭過大：增加 `client_header_buffer_size`
- URL 過長：檢查請求路徑

### 403 Forbidden

- 索引檔案不存在
- 權限不足：檢查檔案和目錄權限

### 404 Not Found

- 檔案路徑錯誤
- SELinux 或 AppArmor 阻止

### 502 Bad Gateway

- 後端服務未執行
- FastCGI/PHP-FPM 設定錯誤

### 504 Gateway Timeout

- 後端回應逾時
- 增加 `proxy_read_timeout`

## 調試技巧

### 如何查看詳細錯誤資訊？

1. 啟用偵錯日誌：

```nginx
error_log /var/log/nginx/error.log debug;
```

2. 檢查錯誤日誌：

```bash
tail -f /var/log/nginx/error.log
```

### 如何測試設定是否生效？

```bash
# 測試設定
nginx -t

# 顯示完整設定
nginx -T
```

### 如何偵錯正規表示式？

使用 `location` 測試：

```nginx
location /test {
    # 新增日誌查看匹配情況
    access_log /var/log/nginx/test.log;
}
```

## 升級問題

### 如何升級 Nginx？

1. 備份設定檔
2. 停止 Nginx
3. 安裝新版本
4. 啟動 Nginx
5. 測試功能

### 如何平滑升級？

```bash
# 編譯新版本
make

# 傳送 USR2 信號升級
kill -s USR2 $(cat /var/run/nginx.pid)

# 優雅關閉舊工作程序
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
