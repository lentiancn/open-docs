# FantasyTalking 安装指南

## 环境要求

在安装 FantasyTalking 之前，需要确保开发环境满足以下要求。操作系统方面，项目支持 Windows、macOS 和 Linux 等主流操作系统，推荐使用 64 位系统以获得最佳性能。Node.js 版本需要 14.0 或更高版本，Python 版本需要 3.7 或更高版本。如果使用 Python 后端，还需要安装 FFmpeg 音频处理工具。

硬件方面，为了获得流畅的语音合成体验，建议系统至少具有 4GB 内存和双核处理器。对于需要实时语音合成的应用场景，建议使用 SSD 存储以降低 I/O 延迟。如果计划进行模型训练或自定义声音开发，需要配备支持 CUDA 的显卡。

## 安装方式

FantasyTalking 支持多种安装方式，开发者可以根据自己的需求和技术栈选择最合适的方法。

### 使用 npm 安装（Node.js）

```bash
# 安装核心库
npm install fantasy-talking

# 安装可选组件
npm install fantasy-talking-ui  # Web UI 组件
npm install fantasy-talking-node  # Node.js 后端支持
```

### 使用 pip 安装（Python）

```bash
# 安装核心库
pip install fantasy-talking

# 安装可选组件
pip install fantasy-talking[ui]  # Web UI 组件
pip install fantasy-talking[all]  # 所有可选组件
```

### 从源码编译

```bash
# 克隆仓库
git clone https://github.com/fantasytalking/fantasy-talking.git
cd fantasy-talking

# 安装 Node.js 依赖
npm install

# 安装 Python 依赖（可选）
pip install -r requirements.txt

# 构建项目
npm run build
```

## 快速开始

### JavaScript/TypeScript 使用示例

```javascript
const FantasyTalking = require('fantasy-talking');

// 初始化客户端
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  // 可选：指定语音风格
  voice: {
    style: 'neutral',
    language: 'zh-CN',
    speed: 1.0,
    pitch: 1.0
  }
});

// 合成语音
async function synthesize() {
  const audio = await client.speak('你好，这是 FantasyTalking 的测试语音');
  console.log('语音合成成功');
  
  // 保存到文件
  const fs = require('fs');
  fs.writeFileSync('output.wav', audio);
  
  // 或直接播放
  client.play(audio);
}

synthesize();
```

### Python 使用示例

```python
from fantasy_talking import FantasyTalking

# 初始化客户端
client = FantasyTalking(
    api_key='your-api-key',
    voice={
        'style': 'neutral',
        'language': 'zh-CN',
        'speed': 1.0,
        'pitch': 1.0
    }
)

# 合成语音
audio = client.speak('你好，这是 FantasyTalking 的测试语音')

# 保存到文件
with open('output.wav', 'wb') as f:
    f.write(audio)

print('语音合成成功')
```

## 配置选项

FantasyTalking 提供了丰富的配置选项，可以根据具体需求进行调整。

### 基础配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| apiKey | string | - | API 密钥，必填 |
| apiUrl | string | 默认地址 | API 服务器地址 |
| timeout | number | 30000 | 请求超时时间（毫秒）|
| retries | number | 3 | 请求失败重试次数 |

### 语音配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| voice.style | string | neutral | 语音风格：neutral, happy, sad, angry, excited |
| voice.language | string | zh-CN | 语言代码 |
| voice.speed | number | 1.0 | 语速，范围 0.5-2.0 |
| voice.pitch | number | 1.0 | 音调，范围 0.5-2.0 |
| voice.volume | number | 1.0 | 音量，范围 0-1.0 |

### 高级配置

对于高级用户，可以配置更多的参数来实现更精细的控制：

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  
  // 音频输出格式
  audio: {
    format: 'wav',  // wav, mp3, ogg
    sampleRate: 16000,  // 8000, 16000, 24000, 48000
    bitDepth: 16  // 8, 16, 24
  },
  
  // 对话配置
  dialogue: {
    enable: true,
    contextWindow: 5,  // 保留的对话上下文轮数
    systemPrompt: '你是一个友好的语音助手'
  },
  
  // 流式输出
  stream: {
    enable: true,
    chunkSize: 1024  // 流式输出的块大小
  }
});
```

## 服务部署

### Docker 部署

项目提供了 Docker 镜像，方便快速部署：

```bash
# 拉取镜像
docker pull fantasytalking/server:latest

# 运行容器
docker run -d \
  --name fantasy-talking \
  -p 8080:8080 \
  -e API_KEY=your-api-key \
  -e PORT=8080 \
  fantasytalking/server:latest
```

### 自建服务

如果需要完全控制服务，可以自行部署后端服务：

```bash
# 克隆后端代码
git clone https://github.com/fantasytalking/server.git
cd server

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 API 密钥等配置

# 启动服务
npm run start
```

服务启动后，可以通过 RESTful API 进行语音合成请求。详细的 API 文档请参考官方文档。

## 验证安装

安装完成后，可以通过以下方式验证 FantasyTalking 是否正常工作：

```bash
# 使用命令行工具测试
fantasy-talking --text "测试语音" --output test.wav

# 或者运行示例代码
npm run example
```

如果一切正常，应该能看到语音文件生成成功。常见的安装问题及解决方案请参考常见问题页面。
