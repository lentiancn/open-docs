# OpenClaw 설치 가이드

OpenClaw는 여러 설치 방법을 지원합니다. 필요에 가장 적합한 방법을 선택하세요.

## 요구 사항

- **Node.js**: 22.x 이상
- **운영 체제**: macOS, Linux, Windows (WSL2 경유)
- **선택 사항**: Tailscale (원격 액세스용)

## 빠른 설치

### macOS / Linux

설치 스크립트를 사용하는 것이 가장 쉬운 방법입니다:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

또는 npm/pnpm 사용:

```bash
npm install -g openclaw@latest
# 또는
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## 초기 설정

설치 후 온보딩 마법사를 실행하세요:

```bash
openclaw onboard --install-daemon
```

마법사가 다음 단계를 안내합니다:

1. **인증 설정**: API 키 또는 OAuth 설정
2. **Gateway 설정**: 게이트웨이 포트 및 서비스 구성
3. **채널 연결**: 메시징 플랫폼 연결
4. **스킬 설치**: 필요한 스킬 선택 및 설치

## 시스템 서비스로 설치

`--install-daemon`을 사용하여 OpenClaw를 백그라운드 서비스로 설치하세요:

```bash
openclaw onboard --install-daemon
```

이렇게 하면 사용자가 생성한 서비스(launchd 또는 systemd)가 생성되어 OpenClaw가 항상 실행됩니다.

## Gateway 수동 실행

서비스로 설치하지 않으려면:

```bash
openclaw gateway --port 18789 --verbose
```

## Docker 설치

Docker를 선호하시면 공식 Docker 배포 문서를 참조하세요.

## 소스에서 빌드

개발에 참여하거나 OpenClaw를 사용자 정의하려면:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

참고: pnpm은 빌드에 사용됩니다. TypeScript는 tsx를 통해 직접 실행할 수 있습니다.

## 설치 확인

설치 후 다음을 수행할 수 있습니다:

1. **Gateway 상태 확인**:
   ```bash
   openclaw gateway status
   ```

2. **Control UI 열기**:
   ```bash
   openclaw dashboard
   ```
   또는 브라우저에서 `http://127.0.0.1:18789/` 방문

3. **테스트 메시지 보내기** (채널 설정 필요):
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## OpenClaw 업데이트

OpenClaw에는 세 가지 릴리스 채널이 있습니다:

- **stable**: 공식 릴리스 (권장)
- **beta**: 사전 릴리스 버전
- **dev**: 개발 빌드

채널 전환:

```bash
openclaw update --channel stable|beta|dev
```

업데이트 후 다음을 실행하는 것이 좋습니다:

```bash
openclaw doctor
```

## 일반적인 명령

| 명령 | 설명 |
|------|------|
| `openclaw onboard` | 초기 설정 마법사 실행 |
| `openclaw gateway start` | Gateway 시작 |
| `openclaw gateway status` | Gateway 상태 확인 |
| `openclaw dashboard` | Control UI 열기 |
| `openclaw agent --message "..."` | 어시스턴트에 메시지 보내기 |
| `openclaw message send --to xxx --message "xxx"` | 메시지 보내기 |
| `openclaw doctor` | 문제 진단 및 수정 |
| `openclaw update` | OpenClaw 업데이트 |

## 환경 변수

사용자 정의 구성 위치의 경우:

- `OPENCLAW_HOME`: 홈 디렉토리 설정
- `OPENCLAW_STATE_DIR`: 상태 디렉토리 재정의
- `OPENCLAW_CONFIG_PATH`: 구성 파일 경로 재정의

## 문제 해결

**설치 실패?**

- Node.js가 >= 22인지 확인
- 네트워크 연결 확인
- 진단을 위해 `openclaw doctor` 실행

**언인스톨 방법은?**

- Gateway 서비스 중지
- 설치 디렉토리 삭제
- 구성 파일 정리
