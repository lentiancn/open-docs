# Nginx 簡介

## 什麼是 Nginx

Nginx（發音為 "engine x"）是一個 HTTP Web 伺服器，反向代理、快取、負載均衡器、TCP/UDP 代理伺服器和郵件代理伺服器。

Nginx 最初由 Igor Sysoev 編寫，並根據 2 條款 BSD 許可證分發。

## Nginx 特點

Nginx 以其靈活性和高效能、低資源消耗而聞名：

- **全球最受歡迎的 Web 伺服器**
- **最受歡迎的 Docker 映像檔之一**
- **為 Kubernetes 多個 Ingress Controllers 提供支持**，包括官方的 kubernetes-ingress

### 基本 HTTP 伺服器功能

- 提供靜態檔案和索引檔案服務
- 自動索引功能
- 開啟檔案描述元快取
- 帶快取的反向代理加速
- 負載均衡和故障轉移
- 支援 FastCGI、uwsgi、SCGI 和 memcached 伺服器的快取加速
- 模組化架構
- 過濾器包括：gzip 壓縮、位元組範圍、分塊回應、XSLT、SSI 和圖像轉換
- SSL 和 TLS SNI 支援
- 支援 HTTP/2（基於權重和依賴的優先級）
- 支援 HTTP/3

### 其他 HTTP 伺服器功能

- 基於名稱和 IP 的虛擬伺服器
- Keep-alive 和管線連線支援
- 存取日誌格式、緩衝日誌寫入、快速日誌輪換和系統日誌
- 3xx-5xx 錯誤代碼重新導向
- 重寫模組：使用正規表示式更改 URI
- 根據用戶端位址執行不同功能
- 基於用戶端 IP 位址、密碼（HTTP 基本認證）和子請求結果的存取控制
- HTTP referer 驗證
- PUT、DELETE、MKCOL、COPY 和 MOVE 方法
- FLV 和 MP4 串流
- 回應速率限制
- 限制來自一個位址的並發連線數或請求數
- 基於 IP 的地理位置
- A/B 測試
- 請求鏡像
- 內嵌 Perl
- njs 腳本語言

### 郵件代理伺服器功能

- 用戶重新導向到 IMAP 或 POP3 伺服器，使用外部 HTTP 認證伺服器
- 用戶認證使用外部 HTTP 認證伺服器，並重新導向到內部 SMTP 伺服器
- 支援的認證方法：POP3（USER/PASS、APOP、AUTH LOGIN/PLAIN/CRAM-MD5）、IMAP（LOGIN、AUTH LOGIN/PLAIN/CRAM-MD5）、SMTP（AUTH LOGIN/PLAIN/CRAM-MD5）
- SSL 支援
- STARTTLS 和 STLS 支援

### TCP/UDP 代理伺服器功能

- TCP 和 UDP 通用代理
- TCP 的 SSL 和 TLS SNI 支援
- 負載均衡和故障轉移
- 基於用戶端位址的存取控制
- 根據用戶端位址執行不同功能
- 限制來自一個位址的並發連線數
- 存取日誌格式、緩衝日誌寫入、快速日誌輪換和系統日誌
- 基於 IP 的地理位置
- A/B 測試
- njs 腳本語言

### 架構和可擴展性

- 一個主程序和多個工作程序
- 工作程序以非特權用戶身份執行
- 靈活的設定
- 重新設定和升級可執行檔而不會中斷用戶端服務
- 支援 kqueue（FreeBSD 4.1+）、epoll（Linux 2.6+）、/dev/poll（Solaris 7 11/99+）、event ports（Solaris 10）、select 和 poll
- 支援各種 kqueue 特性
- 支援各種 epoll 特性
- 支援 sendfile、sendfile64 和 sendfilev
- 支援 File AIO
- 支援 DIRECTIO
- 支援 Accept-filters 和 TCP_DEFER_ACCEPT
- 10,000 個非活躍 HTTP keep-alive 連線僅佔用約 2.5M 記憶體
- 最小化資料複製操作

## Nginx 優勢

### 高效能

Nginx 採用事件驅動的非同步非阻塞模型，能夠處理高並發連線，同時保持低記憶體佔用。

### 低資源消耗

相比其他 Web 伺服器，Nginx 在同等並發下消耗更少的記憶體和 CPU。

### 高穩定性

Nginx 的穩定性經過全球數百萬網站的驗證，极少出現崩潰。

### 簡單設定

Nginx 的設定檔簡潔明瞭，易於理解和維護。

### 擴展性強

支援豐富的模組，可以根據需要啟用或停用相應功能。

### 熱部署

可以在不中斷服務的情況下重載設定和升級版本。

## 應用場景

Nginx 廣泛應用於以下場景：

- **Web 伺服器**：作為靜態檔案的 HTTP 伺服器
- **反向代理**：作為後端伺服器的反向代理
- **負載均衡**：作為多台後端伺服器的負載均衡器
- **快取伺服器**：作為 CDN 或內部快取伺服器
- **API 閘道**：作為微服務的入口閘道
- **郵件代理**：作為 POP3、IMAP、SMTP 代理
- **TCP/UDP 代理**：作為 TCP 和 UDP 協定的代理
