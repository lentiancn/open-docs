# OpenClaw インストールガイド

OpenClawは複数のインストール方法をサポートしています。ニーズに最適な方法を選択してください。

## 動作環境

- **Node.js**：22.x 以上
- **オペレーティングシステム**：macOS、Linux、Windows（WSL2経由）
- **オプション**：Tailscale（リモートアクセス用）

## クイックインストール

### macOS / Linux

インストールスクリプトを使用するのが最も簡単な方法です：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

またはnpm/pnpmを使用：

```bash
npm install -g openclaw@latest
# または
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## 初期設定

インストールが完了したら、オンボーディングウィザードを実行します：

```bash
openclaw onboard --install-daemon
```

ウィザードでは以下を設定できます：

1. **認証設定**：APIキーまたはOAuthを設定
2. **Gateway設定**：ゲートウェイポートとサービスを設定
3. **チャネル接続**：使用するコミュニケーションプラットフォームを接続
4. **スキルインストール**：必要なスキルを選択・インストール

## システムサービスとしてのインストール

`--install-daemon`パラメータを使用して、OpenClawをバックグラウンドサービスとしてインストールできます：

```bash
openclaw onboard --install-daemon
```

これにより、ユーザーレベルサービス（launchdまたはsystemd）が作成され、OpenClawが常に実行されていることが保証されます。

## Gatewayの手動実行

サービスとしてインストールしたくない場合は、Gatewayを手動で起動できます：

```bash
openclaw gateway --port 18789 --verbose
```

## Dockerインストール

Dockerを好む場合は、公式のDockerデプロイドキュメントを参照してください。

## ソースからのビルド

開発に参加하거나OpenClawをカスタマイズしたい場合は：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

注意：pnpmはビルドに使用されます。TypeScriptはtsx経由で直接実行できます。

## インストールの確認

インストール後は以下を実行できます：

1. **Gatewayの状態を確認**：
   ```bash
   openclaw gateway status
   ```

2. **Control UIを開く**：
   ```bash
   openclaw dashboard
   ```
   またはブラウザで`http://127.0.0.1:18789/`にアクセス

3. **テストメッセージを送信**（チャネル設定が必要）：
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## OpenClawの更新

OpenClawには3つのリリースチャンネルがあります：

- **stable**： móvのリリース（推奨）
- **beta**：プレリリースバージョン
- **dev**：開発ビルド

チャンネルを切り替える：

```bash
openclaw update --channel stable|beta|dev
```

更新後は以下を実行することをお勧めします：

```bash
openclaw doctor
```

## 一般的なコマンド

| コマンド | 説明 |
|----------|------|
| `openclaw onboard` | 初期設定ウィザードを実行 |
| `openclaw gateway start` | Gatewayを起動 |
| `openclaw gateway status` | Gatewayの状態を確認 |
| `openclaw dashboard` | Control UIを開く |
| `openclaw agent --message "..."` | アシスタントに直接メッセージを送信 |
| `openclaw message send --to xxx --message "xxx"` | メッセージを送信 |
| `openclaw doctor` | 問題を診断して修正 |
| `openclaw update` | OpenClawを更新 |

## 環境変数

カスタム設定場所が必要な場合：

- `OPENCLAW_HOME`：ホームディレクトリを設定
- `OPENCLAW_STATE_DIR`：状態ディレクトリをオーバーライド
- `OPENCLAW_CONFIG_PATH`：設定ファイルパスをオーバーライド

## トラブルシューティング

**インストールに失敗した場合は？**

- Node.jsバージョンが >= 22であることを確認
- ネットワーク接続を確認
- `openclaw doctor`を実行して診断

**アンインストール方法は？**

- Gatewayサービスを停止
- インストールディレクトリを削除
- 設定ファイルをクリーンアップ
