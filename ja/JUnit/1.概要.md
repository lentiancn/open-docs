# JUnit 概要

JUnitはJavaプラットフォームで最も人気のあるユニットテストフレームワークであり、再可能なテストを作成するために使用されます。高品質なJavaコード開発の基盤として、世界中の数百万の開発者に広く使用されています。

## JUnit 4とJUnit 5の違い

### バージョンの概要

- **JUnit 4**：2010年にリリースされた、長年Javaコミュニティで使用されてきたテストフレームワークの最終メジャーバージョン
- **JUnit 5**：2017年にリリースされた、JUnitフレームワークの全新規格版。3つのサブプロジェクトで構成：JUnit Platform、JUnit Jupiter、JUnit Vintage

### アーキテクチャの違い

JUnit 4はモノリシックアーキテクチャを採用し、すべての機能が単一のライブラリに統合されています。JUnit 5はモジュール式アーキテクチャを採用し、3つの独立したプロジェクトで構成されます：

- **JUnit Platform**：テストフレームワークの基盤。テストエンジンAPI、コマンドラインランチャーを提供
- **JUnit Jupiter**：新しいプログラミングモデルと拡張モデル。テストと拡張の作成用
- **JUnit Vintage**：互換エンジン。JUnit 5プラットフォームでJUnit 3とJUnit 4テストを実行可能

### 主な違い

| 機能 | JUnit 4 | JUnit 5 |
|------|---------|---------|
| 最低Javaバージョン | Java 5以上 | Java 8以上 |
| アノテーションパッケージ | org.junit | org.junit.jupiter.api |
| テストクラスの継承不要 | いいえ | はい |
| アサーションスタイル | Assert静的メソッド | Assertions静的メソッド |
| パラメータ化テスト | @Parameters | @ParameterizedTest |
| ネストされたテスト | 未対応 | 対応 |
| 動的テスト | 未対応 | 対応 |
| 並列実行 | 限定的対応 | ネイティブ対応 |
| 拡張モデル | @RunWith | @ExtendWith |

### バージョンの選択

- **新規プロジェクト**：JUnit 5推奨。最新の機能と優れた拡張性
- **既存プロジェクト**：JUnit 4を引き続き使用、またはJUnit Vintageで段階的移行
- **長期メンテナンス**：段階的にJUnit 5へ移行することを推奨
