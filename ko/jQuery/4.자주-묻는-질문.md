# jQuery 자주 묻는 질문

## 기본 질문

### jQuery와 JavaScript의 차이점은?

jQuery는 JavaScript 위에 구축된 라이브러리이며 대체품이 아닙니다. jQuery는 일반적인 JavaScript 작업을 단순화하지만 기본 코드는 여전히 JavaScript입니다. 예를 들어, `$("#myId")`는 `document.getElementById("myId")`와 같습니다.

### 왜 jQuery를 사용해야 하나요?

- **간결한 구문**: 더 적은 코드로 더 많은 작업 수행
- **크로스 브라우저 호환성**: 브라우저 간 차이 해결
- **풍부한 기능**: DOM 조작, 이벤트 처리, 애니메이션, Ajax - 모두 하나에서
- **큰 커뮤니티**: 풍부한 플러그인 및 튜토리얼 리소스
- **완만한 학습 곡선**: 초보자에게 친철함

### jQuery 4.0의 새로운 기능은?

jQuery 4.0에는 여러 가지 개선이 포함되어 있습니다:
- 더 나은 성능
- 개선된 TypeScript 지원
- 사용되지 않는 API 제거
- 개선된 애니메이션 엔진

### jQuery를 계속 사용하는 가치는 있나요?

프로젝트에 따라 다릅니다:
- **jQuery 적합**: 간단한 프로젝트, 이전 브라우저 지원 필요, 빠른 프로토타이핑
- **jQuery 필요 없을 수 있음**: 새 프로젝트, modern 프레임워크 사용 (React, Vue 등), 높은 성능 요구사항

## 설치 문제

### jQuery 로드 실패 시怎么做?

1. CDN 링크가 정확한지 확인
2. 네트워크가 CDN에 접근할 수 있는지 확인
3. 콘솔에 오류 메시지 있는지 확인
4. 로컬 다운로드 시도

```html
<!-- CDN 가용성 확인 -->
<script src="https://code.jquery.com/jquery-4.0.0.min.js"></script>
<script>
    if (typeof jQuery === 'undefined') {
        alert('jQuery 로드 실패');
    } else {
        alert('jQuery 버전: ' + jQuery.fn.jquery);
    }
</script>
```

### 여러 jQuery 버전 충돌 시怎么做?

`jQuery.noConflict()`를 사용하여 `$` 변수를 해제합니다:

```javascript
var $j = jQuery.noConflict();
$j("#myId").hide();  // $ 대신 $j 사용

// $를 복원해야 하는 경우
jQuery.noConflict(true);  // $와 jQuery 복원
```

### CDN 대 로컬 파일 - 어느 것을 선택?

| 방법 | 장점 | 단점 |
|------|------|------|
| CDN | 다운로드 불필요, 글로벌 CDN, 캐시됨 | 네트워크 의존, 지연 가능 |
| 로컬 | 오프라인 사용 가능, 완전 제어 | 수동 업데이트, 대역폭 사용 |

## 사용 문제

### $ 기호 충돌 시怎么做?

여러 라이브러리가 `$` 기호를 사용할 수 있습니다:

```javascript
// 방법 1: $ 대신 jQuery 사용
jQuery("#myId").hide();

// 방법 2: noConflict 사용
var $jq = jQuery.noConflict();

// 방법 3: IIFE 내 사용
(function($) {
    $("#myId").hide();  // $는 여기서 jQuery를 가리킴
})(jQuery);
```

### 선택자가 요소를 찾지 못하면怎么做?

1. DOM이 로드되었는지 확인
2. 선택자 구문 확인
3. 요소가 페이지에 실제로 존재하는지 확인

```javascript
$(document).ready(function() {
    // DOM이 로드된 후에 실행
    var element = $("#myId");
    console.log("요소 수:", element.length);
    
    if (element.length === 0) {
        console.log("요소를 찾을 수 없음");
    }
});
```

### 이벤트가 작동하지 않으면怎么做?

1. 이벤트 바인딩 전에 요소가 존재하는지 확인
2. 동적으로 추가된 요소의 경우 이벤트 위임 사용

```javascript
// 잘못됨: 요소가 아직 존재하지 않을 수 있음
$(".dynamic-item").click(handler);

// 올바름: 이벤트 위임
$("#container").on("click", ".dynamic-item", handler);
```

### 애니메이션이 효과 없으면怎么做?

1. CSS 충돌 확인
2. 요소가 보이는지 확인
3. 다른 애니메이션이 실행 중인지 확인

```javascript
// stop을 사용하여 이전 애니메이션 지우기
$("#myElement").stop().animate({ left: "100px" });
```

## Ajax 문제

### Ajax 요청 실패 시怎么做?

```javascript
$.ajax({
    url: "/api/data",
    error: function(xhr, status, error) {
        console.log("상태:", status);
        console.log("오류:", error);
        
        if (status === "timeout") {
            alert("요청 시간 초과");
        } else if (status === "parsererror") {
            alert("데이터 구문 분석 오류");
        } else {
            alert("요청 실패");
        }
    }
});
```

### 크로스 오리진 요청怎么处理?

jQuery Ajax는 기본적으로 크로스 오리진을 지원하지 않습니다. 백엔드에서 CORS 헤더를 설정하거나 JSONP를 사용하세요:

```javascript
$.ajax({
    url: "https://other-domain.com/api",
    dataType: "jsonp",
    success: function(data) {
        console.log(data);
    }
});
```

### Ajax 요청 취소 방법?

```javascript
var xhr = $.ajax({
    url: "/api/slow-request",
    success: function(data) {
        console.log(data);
    }
});

// 요청 취소
xhr.abort();
```

## 성능 문제

### jQuery 성능을 최적화하는 방법은?

1. **선택 결과 캐싱**: 동일한 요소를 반복적으로 쿼리하지 않기

```javascript
// 나쁨
$("#myId").addClass("a");
$("#myId").addClass("b");
$("#myId").addClass("c");

// 좋음
var $myId = $("#myId");
$myId.addClass("a b c");
```

2. **적절한 선택자 사용**

```javascript
// 가장 빠름: ID 선택자
$("#myId")

// 빠름: 클래스 선택자
$(".myClass")

// 느림: 의사 선택자
$("div:first")
```

3. **일괄 작업**: 여러 요소를 한 번에 수정

```javascript
// 객체를 사용하여 일괄 설정
$(".items").css({
    color: "red",
    background: "blue"
});
```

4. **이벤트 위임**: 이벤트 바인딩 수 줄이기

```javascript
// 나쁨: 각 요소에 별도로 바인딩
$(".item").on("click", handler);

// 좋음: 부모에 위임
$("#container").on("click", ".item", handler);
```

### jQuery 대 네이티브 JS - 어느 것이 더 빠름?

대부분의 경우 네이티브 JavaScript가 더 빠릅니다. jQuery는 편리함을 제공하지만 어느 정도의 성능 오버헤드가 있습니다. 성능이 중요한 애플리케이션의 경우 네이티브 JavaScript를 사용하는 것이 좋습니다.

## 호환성 문제

### 이전 브라우저를 지원하려면 어떻게 해야 하나요?

1. jQuery 3.x 이상 사용, IE 8 이하 더 이상 지원하지 않음
2. IE 8 지원이 필요한 경우 jQuery 1.x 사용
3. 호환성 문제에 대해 jQuery Migrate 플러그인 사용

### 업그레이드 후 코드가 호환되지 않으면 어떻게 해야 하나요?

1. 공식 업그레이드 가이드 읽기
2. jQuery Migrate 플러그인을 사용하여 경고 보기
3. 각 문제를 단계별로 수정
4. 전체 테스트 실행

```javascript
// Migrate를 포함한 후 콘솔에 호환되지 않는 API 경고가 표시됨
// 예: $(".selector").live(...)가 제거됨
```

### jQuery Mobile과 jQuery UI의 차이점은?

|-----------|------|
| jQuery | 코어 라이브러리, 기본 DOM 작업, 이벤트, 애니메이션, Ajax |
| jQuery UI | UI 컴포넌트 라이브러리, 대화상자, 드래그 앤 드롭, 날짜 선택기 등 |
| jQuery Mobile | 반응형 모바일 앱용 모바일 프레임워크 (폐기됨) |
