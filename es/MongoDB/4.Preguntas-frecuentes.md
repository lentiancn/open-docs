# MongoDB Preguntas Frecuentes

## Conceptos Básicos

### ¿Qué es MongoDB?

MongoDB es una base de datos documental diseñada para ayudar a los desarrolladores a crear aplicaciones modernas más rápido. Almacena datos en documentos flexibles similares a JSON.

### ¿MongoDB es una base de datos relacional?

No. MongoDB es una base de datos NoSQL orientada a documentos que usa formato BSON similar a JSON.

### ¿Qué es un documento en MongoDB?

Un documento es la unidad básica de datos en MongoDB, consiste en pares de campo y valor. Cada documento tiene un campo _id único como clave primaria.

### ¿Cuál es la diferencia entre MongoDB y MySQL?

| Característica | MongoDB | MySQL |
|----------------|---------|-------|
| Modelo de Datos | Documento | Relacional |
| Esquema | Flexible | Fijo |
| Escalamiento | Horizontal | Vertical |
| Lenguaje de Consulta | MongoDB Query Language | SQL |
| Transacciones | ACID multi-documento | ACID multi-tabla |

### ¿Qué es BSON?

BSON significa Binary JSON, es la representación binaria de documentos JSON con tipos de datos adicionales.

### ¿Cuál es el tamaño máximo de documento en MongoDB?

El tamaño máximo es 16 MB. Para archivos mayores, MongoDB proporciona la API GridFS.

## Problemas de Instalación

### ¿Cuánta memoria necesita MongoDB?

MongoDB usa la memoria disponible como caché. Se recomienda al menos 4GB para entornos de desarrollo.

### ¿MongoDB soporta sistemas de 32 bits?

No. MongoDB requiere un sistema operativo de 64 bits.

### ¿Cómo cambiar el puerto predeterminado de MongoDB?

El puerto predeterminado es 27017. Puede cambiarse modificando `net.port` en el archivo de configuración o usando `--port`.

## Problemas de Uso

### ¿Cómo hacer backup de MongoDB?

```bash
mongodump --db=midatabase --out=/respaldo
```

Restaurar:

```bash
mongorestore --db=midatabase /respaldo/midatabase
```

### ¿Cómo logra MongoDB alta disponibilidad?

A través de Replica Sets. Un replica set consiste en un nodo primario y múltiples nodos secundarios.

### ¿Qué es sharding?

Sharding es la técnica de MongoDB para escalar horizontalmente, distribuyendo datos en múltiples servidores.

### ¿MongoDB soporta transacciones?

Sí. MongoDB 4.0+ soporta transacciones de múltiples documentos.

### ¿Cómo optimizar el rendimiento de consultas?

1. Crear índices apropiados
2. Usar proyecciones para devolver solo campos necesarios
3. Evitar escaneos de colección completos
4. Usar pipelines de agregación

### ¿Cómo asegurar datos en MongoDB?

1. Habilitar autenticación
2. Usar control de acceso basado en roles (RBAC)
3. Habilitar TLS/SSL
4. Usar cifrado de almacenamiento
5. Hacer backups regulares

### ¿Cuál es la diferencia entre Edición Comunidad y Empresarial?

La Edición Empresarial incluye:
- Autenticación LDAP
- Autenticación Kerberos
- Auditoría
- Motor de almacenamiento cifrado
- Soporte técnico

### ¿Qué tipo de datos es adecuado para MongoDB?

- Sistemas de gestión de contenidos
- Configuraciones de usuario
- Datos de IoT
- Análisis en tiempo real
- Aplicaciones móviles
- Catálogos de comercio electrónico

## Solución de Problemas

### ¿Conexión rechazada?

1. Verificar si el servicio MongoDB está ejecutándose
2. Verificar el puerto
3. Verificar configuración de firewall

### ¿Consulta lenta?

1. Usar explain() para analizar el plan de consulta
2. Crear índices apropiados
3. Verificar queries lentas

### ¿Error en elección de replica set?

1. Verificar conectividad de red
2. Verificar configuración de nodos
3. Asegurar que la mayoría de nodos estén disponibles

## Desarrollo

### ¿Qué lenguajes de programación soporta MongoDB?

MongoDB proporciona drivers para casi todos los lenguajes principales:
- JavaScript/Node.js
- Python
- Java
- C#
- Go
- Ruby
- PHP
- Scala
- Kotlin
- Swift

### ¿Cómo usar MongoDB en aplicaciones?

1. Instalar el driver para su lenguaje
2. Establecer conexión
3. Ejecutar operaciones CRUD

Ejemplo en Node.js:

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('miDB');
  const collection = database.collection('usuarios');
  
  await collection.insertOne({ nombre: 'Juan', edad: 30 });
  const usuario = await collection.findOne({ nombre: 'Juan' });
  
  await client.close();
}
```

### ¿Qué es MongoDB Atlas?

MongoDB Atlas es un servicio de base de datos en la nube proporcionado por MongoDB, que ofrece implementaciones completamente administradas.

### ¿Cómo modelar datos en MongoDB?

Principios de modelado de datos MongoDB:
1. Preferir estructura de documentos sobre normalización
2. Usar documentos integrados para relaciones "contiene"
3. Usar referencias para relaciones "muchos a muchos"
4. Diseñar colecciones e índices según patrones de consulta
5. Considerar crecimiento y escalabilidad
