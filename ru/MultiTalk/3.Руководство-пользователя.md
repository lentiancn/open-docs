# Rukovodstvo pol'zovatelya

Eto rukovodstvo predostavlyayet podrobnyye instruktsii po ispol'zovaniyu MultiTalk, vklyuchaya osnovnyye operatsii, nastroyku parametrov i prodvinutyye tekhniki.

## Osnovnoye ispol'zovaniye

### Podgotovka faylov vvoda

MultiTalk trebuyet podgotovki konkretnykh formatov vvoda. Proekt ispol'zuyet JSON-fayly dlya opredeleniya parametrov generatsii, vklyuchaya audi puti, puti k izobrazheniyam i tekstovyye podskazki.

Dlya generatsii odnogo cheloveka format vvkhodnogo JSON trebuyet ukazaniya puti k audiofaylu, puti k referennomu izobrazheniyu, imeni soyedineniya video i drugikh parametrov. Fayly primerov nakhodyatsya v kataloge examples, naprimer single_example_1.json.

Dlya generatsii mnogolyudnogo razgovora vvkhodnyy JSON dolzhen soderzhat' neskol'ko potokov audio i opredeleniya oblastey sootvetstvuyushchikh lyudey. Fayly primerov nakhodyatsya v kataloge examples, naprimer multitalk_example_2.json.

### Zпуска osnovnoy generatsii

Posle ustanovki i nastroyki voydite v katalog proekta i vypolnite sleduyushchuyu komandu dlya generatsii video:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### Opisaniye parametrov

--ckpt_dir ukazyvayet put' k katalogu vesov modeli. --wav2vec_dir ukazyvayet put' k katalogu audio-enkodera. --input_json ukazyvayet put' k vvkhodnomu faylu parametrov JSON. --sample_steps ukazyvayet kolichestvo shagov otborki; bol'shiye znacheniya dayut luchsheye kachestvo, no men'shuyu skorost'. --mode ukazyvayet rezhim generatsii: streaming dlya dlinnykh video, clip dlya korotkikh video. --use_teacache vklyuchayet uskoreniye TeaCache. --save_file ukazyvayet imya fayla dlya soyedineniya vyvoda.

## Prodvinutyye tekhniki ispol'zovaniya

### Nastrojka kachestva generatsii

Shagi otborki - klyuchevyye parametry, vliyayushchiye na kachestvo generatsii. Po umolchaniyu 40 shagov dayut khoroshiye rezul'taty. Dlya boleye bystroy generatsii mozhno umen'shit' kolichestvo shagov do 10, chto minimal'no vliyayet na sinkhronizatsiyu gub, no vliyayet na plyvkost' dvizheniya i vizual'noye kachestvo. Uvelicheniye shagov uluchshayet proizvedeniye detaley.

Sila sinkhronizatsii gub upravlyayetsya koeffitsiyentom audio-navedeniya. Znacheniya Audio CFG ot 3 do 5 rabotayut luchshe vsego. Uvelicheniye etogo znacheniya dayet luchshuyu sinkhronizatsiyu gub.

### Generatsiya dlinnykh video

MultiTalk podderzhivayet generatsiyu video dlinoy do 15 sekund. Ispol'zuyte rezhim streaming dlya generatsii boleye dlinnykh klipov:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

Pri generatsii ochen' dlinnykh video mogut byt' razlichiya v ottenkakh mezhdu segmentami. Ustanovka Audio CFG na 3 pomogayet sokhranit' postoyanstvo ottenka.

### Rabota s nizkim VRAM

Esli VRAM ogranichen, mozhno ispol'zovat' sleduyushchiye parametry dlya umen'sheniya ispol'zovaniya pamyati:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

Dobavleniye parametra --num_persistent_param_in_dit 0 pozvolyayet generirovat' video 480P na odnom RTX 4090.

### Vklyucheniye uskoreniya

Proekt podderzhivayet neskol'ko metodov uskoreniya dlya uluchsheniya skorosti generatsii.

Uskoreniye TeaCache mozhet znachitel'no uluchshit' skorost', sokhranyaya kachestvo. Rekomenduyemyye znacheniya --teacache_thresh nakhodyatsya v diapazone ot 0.2 do 0.5:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### Ispol'zovaniye uskoreniya LoRA

Ispol'zovaniye LoRA mozhet znachitel'no umen'shit' kolichestvo shagov otborki. FusioniX LoRA trebuyet tol'ko 8 shagov, a lightx2v LoRA - vsego 4 shaga:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Ispol'zovaniye interfeisa Gradio

Proekt predostavlyayet web-interfeys Gradio dlya interaktivnogo ispol'zovaniya. Voydite v katalog proekta i vypolnite:

```bash
python app.py```

Ili ispol'zuyte versiyu s nizkim VRAM:

```bash
python app.py --num_persistent_param_in_dit 0
```

## Integratsiya sinteza rechi TTS

MultiTalk podderzhivayet pryamoy tekstovyy vvod. S pomoshch'yu integrirovannoy tekhnologii TTS on generiruyet rech' iz teksta, a zatem sozdayet video:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## Konfiguratsiya neskol'kikh GPU

Generatsiya video vysokogo razresheniya 720P trebuyet neskol'kikh GPU. Ispol'zuyte sleduyushchuyu komandu dlya konfiguratsii neskol'kikh GPU:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```
