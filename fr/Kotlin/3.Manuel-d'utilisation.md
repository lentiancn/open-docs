# Manuel d'utilisation de Kotlin

## Variables et types de base

Kotlin utilise deux mots-clés pour la déclaration de variables : var et val. var déclare une variable mutable, similaire à une variable régulière en Java. val déclare une variable immuable, similaire à une variable finale en Java. En Kotlin, préférer val est une bonne pratique qui rend le code plus sûr et plus facile à comprendre.

Lors de la déclaration de variables, vous pouvez spécifier explicitement le type ou laisser le compilateur inférer le type :

val name: String = "Kotlin"
val age = 10 // Le compilateur infère le type Int
var count = 0 // Variable mutable

Les types de base de Kotlin comprennent : les entiers (Int, Long, Short, Byte), les nombres à virgule flottante (Double, Float), les caractères (Char), les booléens (Boolean) et les chaînes (String). Contrairement à Java, tous les types de base en Kotlin sont des objets avec des méthodes membres correspondantes.

Les entiers en Kotlin sont de type Int par défaut. Pour représenter des nombres plus grands, utilisez le suffixe Long :

val longNumber = 100L // Type Long

Les chaînes en Kotlin sont puissantes et supportent les modèles de chaînes, permettant d'incorporer directement des variables ou des expressions dans les chaînes :

val name = "World"
println("Hello, $name!") // Affiche : Hello, World!
println("Length: ${name.length}") // Affiche : Length: 5

## Collections

La bibliothèque standard Kotlin fournit des types de collections riches, notamment List (liste), Set (ensemble) et Map (carte). Chaque type de collection a des versions immuables et mutables.

Les collections immuables ne peuvent pas être ajoutées, supprimées ou modifiées après leur création :

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

Les collections mutables permettent la modification des éléments :

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Les collections Kotlin fournissent des opérations fonctionnelles riches comme map, filter et reduce :

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## Contrôle de flux

Le contrôle de flux en Kotlin comprend les expressions conditionnelles, les boucles et les expressions de saut.

L'expression if en Kotlin retourne une valeur et peut être directement assignée à une variable :

val max = if (a > b) a else b

L'expression when est l'outil puissant de Kotlin pour les correspondances multi-branches qui peut correspondre à diverses conditions :

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Les boucles en Kotlin comprennent les boucles for et while. Les boucles for peuvent itérer sur les ranges, les tableaux et les collections :

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## Fonctions

Kotlin utilise le mot-clé fun pour déclarer des fonctions. Les fonctions peuvent spécifier les types de paramètres et les types de retour :

fun greet(name: String): String {
    return "Hello, $name!"
}

Si le corps de la fonction n'a qu'une seule expression, vous pouvez utiliser la syntaxe d'expression unique :

fun greet(name: String) = "Hello, $name!"

Kotlin supporte les arguments par défaut et les arguments nommés, ce qui réduit considérablement le besoin de surcharge de fonctions :

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin supporte également les paramètres variadiques en utilisant le mot-clé vararg :

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## Classes et objets

Les déclarations de classes en Kotlin sont plus concises qu'en Java. Par défaut, les classes en Kotlin sont finales et ne peuvent pas être héritées. Pour rendre une classe héritable, utilisez le mot-clé open :

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Les constructeurs de Kotlin comprennent les constructeurs primaires et secondaires. Le constructeur primaire est déclaré directement après le nom de la classe :

class Person(val name: String, val age: Int)

Si vous avez besoin d'exécuter une logique dans le constructeur, utilisez un bloc init :

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Les classes de données (data class) de Kotlin génèrent automatiquement les méthodes equals, hashCode, toString et copy, ce qui les rend parfaites pour les classes qui stockent des données :

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Sécurité nulle

La sécurité nulle est l'une des fonctionnalités les plus importantes de Kotlin. En Kotlin, le système de types distingue les types nullables et non nullables, ce qui réduit fondamentalement les NullPointerExceptions.

Par défaut, les types ne sont pas nullables :

val name: String = "Kotlin" // Ne peut pas être null
val name: String = null // Erreur de compilation

Pour permettre à une variable d'être null, utilisez les types nullables (?) :

val name: String? = null // Peut être null

Lors de l'accès aux propriétés ou méthodes de types nullables, utilisez l'opérateur d'appel sécurisé (?.) ou l'assertion non nulle (!!) :

val length = name?.length // Retourne null si name est null
val length = name!!.length // Lève une exception si name est null

L'opérateur Elvis (?:) offre un moyen concis de gérer les valeurs par défaut :

val length = name?.length ?: 0 // Retourne 0 si name est null

## Fonctions d'extension et propriétés

Kotlin permet d'ajouter de nouvelles fonctions et propriétés aux classes existantes sans hériter ou modifier la classe originale. Ce sont les fonctions d'extension et les propriétés d'extension.

Les fonctions d'extension sont déclarées en préfixant le nom de la fonction avec le nom de la classe à étendre :

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

Les propriétés d'extension peuvent également ajouter de nouvelles propriétés aux classes existantes :

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## Fonctions d'ordre supérieur et Lambdas

Les fonctions d'ordre supérieur sont des fonctions qui prennent des fonctions comme paramètres ou retournent des fonctions. Les expressions lambda sont le moyen principal d'implémenter cette fonctionnalité.

La syntaxe d'expression lambda utilise des accolades :

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

Si un lambda est le dernier paramètre d'une fonction, il peut être déplacé à l'extérieur des parenthèses :

list.filter { it > 0 }

Dans les expressions lambda, it représente le nom implicite pour un seul paramètre. Lorsqu'il n'y a qu'un seul paramètre, vous pouvez omettre la déclaration du paramètre.

## Coroutines

Les coroutines sont la fonctionnalité centrale de la programmation asynchrone en Kotlin. Par rapport aux threads, les coroutines sont beaucoup plus légères et peuvent exécuter plusieurs coroutines dans un seul thread.

launch est une façon courante de démarrer des coroutines, qui démarre une nouvelle coroutine en arrière-plan :

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

Les fonctions suspend ne peuvent être appelées que dans des coroutines :

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        // Effectuer une opération longue
    }
}

Les coroutines Kotlin fournissent plusieurs dispatchers pour contrôler sur quel thread les coroutines s'exécutent : Dispatchers.Main pour les opérations UI, Dispatchers.IO pour les opérations I/O, et Dispatchers.Default pour les tâches intensives en CPU.

## Classes scellées et énumérations

Les classes scellées sont des hiérarchies de classes restreintes qui limitent le nombre de sous-classes qui peuvent en hériter. Toutes les sous-classes doivent être déclarées dans le même fichier. Ceci est très utile pour représenter des états finis ou des types de résultats :

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

L'expression when peut pleinement utiliser les fonctionnalités des classes scellées, le compilateur vérifiant si tous les cas possibles sont gérés :

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

Les énumérations représentent un ensemble fini de valeurs constantes :

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

Les énumérations peuvent avoir des propriétés et des méthodes :

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## Interfaces et classes abstraites

Les interfaces en Kotlin peuvent contenir des implémentations par défaut, les rendant plus flexibles :

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
}

Les classes peuvent implémenter plusieurs interfaces :

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## Expressions et déclarations d'objets

Les expressions d'objets créent des instances de classes anonymes, similaires aux classes internes anonymes de Java :

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // Gérer l'événement de clic
    }
})

Les déclarations d'objets créent des singletons :

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## Objets compagnes

Chaque classe peut contenir un objet compagnon, similaire aux membres statiques en Java :

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

Les membres de l'objet compagnon peuvent être accédés directement via le nom de la classe :

MyClass.TAG
MyClass.create()

## Vérifications et conversions de types

L'opérateur is est utilisé pour la vérification de types, similaire à instanceof de Java :

if (obj is String) {
    // Dans cette branche, obj est automatiquement casté en String
    println(obj.length)
)

Les casts intelligents convertissent automatiquement les types qui ont été vérifiés. Si le compilateur peut déduire le type de la variable dans une branche spécifique, il effectue automatiquement la conversion.

Si les casts intelligents ne s'appliquent pas, vous pouvez utiliser une conversion explicite :

val str = obj as String

La conversion de type sécurisée utilise as?, qui retourne null lorsque la conversion échoue :

val str = obj as? String // Retourne null si la conversion échoue
