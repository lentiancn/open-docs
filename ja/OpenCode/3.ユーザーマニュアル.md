# OpenCode ユーザーマニュアル

OpenCode の基本的な使用方法とテクニック。

## クイックスタート

### 質問する

OpenCode にコードベースを説明を依頼：

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### 機能追加

機能を追加する前に、まず計画を作成させることが推奨：

1. **Tab** を押して Plan モードに切り替え
2. 希望する機能を描述
3. 計画を表示后、フィードバックや詳細を追加
4. 满意したら、**Tab** を押して Build モードに戻って実行

### 直接変更

より直接的な変更を依頼：

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### 変更を元に戻す

変更に满意できない場合は、元に戻すコマンドを使用：

```bash
/undo
```

OpenCode は変更を元に戻し元のメッセージを再度表示します。プロンプトを調整して再試行できます。

### やり直し

`/redo` で変更をやり直しできます。

## 会話を共有

```bash
/share
```

現在の会話へのリンクを作成しクリップボードにコピーします。

## よく使うコマンド

| コマンド | 機能 |
|----------|------|
| /init | プロジェクト初期化 |
| /connect | LLM プロバイダに接続 |
| /undo | 変更を元に戻す |
| /redo | 変更をやり直し |
| /share | 会話を共有 |

## ベストプラクティス

1. ジュニア開発者との会話のように OpenCode に話しかける
2. 十分なコンテキストと詳細を提供
3. まず計画を作成させ、それから実行
4. 满意できない場合は /undo を使用
5. AGENTS.md を定期的に確認・更新
