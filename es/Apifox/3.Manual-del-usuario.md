# Manual del Usuario

Este manual describe en detalle las funciones principales y métodos de uso de Apifox, incluyendo gestión de proyectos, diseño de interfaces, depuración, Mock, pruebas automatizadas, entre otras.

## Gestión de Proyectos

### Crear un Proyecto

En Apifox, el proyecto es la unidad principal para organizar APIs.

```bash
# Pasos para crear un nuevo proyecto
1. Haz clic en el "+" junto a la lista de "Proyectos" en el panel izquierdo
2. Ingresa el nombre del proyecto
3. Selecciona el tipo de proyecto
4. Selecciona la plantilla base (opcional)
5. Haz clic en "Crear"
```

### Configuración del Proyecto

Cada proyecto tiene opciones de configuración independientes:

- **Información básica**: Nombre, descripción e icono del proyecto
- **Variables de entorno**: Define variables para diferentes entornos como desarrollo, pruebas, producción, etc.
- **Autenticación**: Configura el método de autenticación global
- **Exportar/Importar**: Importación y exportación de datos del proyecto

### Gestión de Entornos

Apifox soporta configuración de múltiples entornos:

```bash
# Uso de variables de entorno
En las solicitudes, usa {{nombreVariable}} para hacer referencia a las variables de entorno
Ejemplo: {{baseUrl}}/api/users

Cambiar de entorno: haz clic en el selector de entorno en la parte superior para elegir un entorno diferente
```

## Diseño de Interfaces

### Crear una Interfaz

Una interfaz es la definición específica de una API:

```bash
# Pasos para crear una interfaz
1. En el proyecto, haz clic en "+" y selecciona "Nueva Interfaz"
2. Completa la información básica:
   - Nombre de la interfaz: Lista de usuarios
   - Ruta de solicitud: /api/users
   - Método de solicitud: GET
3. Define los parámetros de solicitud
4. Define la estructura de respuesta
5. Guarda
```

### Parámetros de Solicitud

Soporta los siguientes tipos de parámetros de solicitud:

- **Parámetros de ruta**: Como el id en /api/users/:id
- **Parámetros de consulta**: En la URL como ?name=test
- **Encabezados de solicitud**: Como Authorization
- **Cuerpo de solicitud**: Formatos como JSON, formulario, XML, etc.

### Definición de Respuestas

Define respuestas exitosas y fallidas para cada interfaz:

```json
{
  "Respuesta exitosa 200": {
    "id": 1,
    "name": "Zhang San",
    "email": "zhangsan@example.com"
  },
  "Respuesta de error 404": {
    "code": 404,
    "message": "El usuario no existe"
  }
}
```

### Gestión de la Lista de Interfaces

Soporta grouped management de interfaces, que se pueden clasificar por módulo o función:

- Arrastrar para reordenar
- Operaciones por lotes (eliminación masiva, exportación masiva, etc.)
- Búsqueda y filtrado

## Depuración de Interfaces

### Enviar Solicitudes

En la página de detalles de la interfaz, haz clic en el botón "Enviar" para enviar la solicitud:

```bash
# Pasos básicos de depuración
1. Abre la interfaz
2. Selecciona el entorno
3. Establece los valores de los parámetros
4. Haz clic en "Enviar"
5. Verifica la respuesta
```

### Configuración de Autenticación

Soporta múltiples métodos de autenticación:

- **Ninguna**: Sin autenticación
- **Bearer Token**: Autenticación JWT
- **Basic Auth**: Autenticación básica
- **OAuth 2.0**: Autenticación OAuth2
- **Autenticación personalizada**: Encabezados de autenticación personalizados según tus necesidades

### Preprocesamiento de Solicitudes

Puedes configurar scripts que se ejecutan antes de enviar la solicitud:

```javascript
// Ejemplo: Agregar marca de tiempo
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// Ejemplo: Generar ID aleatorio
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### Posprocesamiento de Respuestas

Puedes configurar scripts que se ejecutan después de recibir la respuesta:

```javascript
// Ejemplo: Extraer datos de la respuesta
const response = pm.response.json();
console.log('Datos de respuesta:', response);

// Ejemplo: Establecer variable de entorno
pm.environment.set('access_token', response.data.token);
```

## API Mock

### Mock Inteligente

Una de las funciones más potentes de Apifox. Solo necesitas definir la estructura de datos y el sistema generará datos Mock automáticamente:

```bash
# Activar Mock Inteligente
1. Abre la interfaz
2. Haz clic en la pestaña "Mock"
3. Activa "Mock Inteligente"
4. Guarda
```

### Reglas de Mock

Soporta personalización de reglas de Mock:

- `@string`: Generar cadena aleatoria
- `@integer`: Generar número entero aleatorio
- `@float`: Generar número decimal aleatorio
- `@boolean`: Generar valor booleano aleatorio
- `@date`: Generar fecha aleatoria
- `@email`: Generar correo electrónico aleatorio
- `@url`: Generar URL aleatoria
- `@guid`: Generar UUID

### Ejemplo

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

Datos Mock generados:

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Servidor Mock

Apifox proporciona un servidor Mock incorporado:

```bash
# Obtener la dirección Mock
En la página de detalles de la interfaz, copia la URL Mock
Ejemplo: https://mock.apifox.com/api/users
```

## Pruebas Automatizadas

### Crear Casos de Prueba

```bash
# Pasos para crear pruebas
1. Abre la interfaz
2. Haz clic en la pestaña "Pruebas"
3. Haz clic en "Crear Prueba"
4. Establece el nombre del caso de prueba
5. Agrega aserciones
6. Guarda
```

### Tipos de Aserciones

Soporta múltiples tipos de aserciones:

- **Aserción de código de estado**: Verifica el código de estado de la respuesta
- **Aserción JSON**: Verifica los datos JSON de la respuesta
- **Aserción de tiempo de respuesta**: Verifica el tiempo de respuesta
- **Aserción de encabezados**: Verifica los encabezados de respuesta

### Ejemplo

```javascript
// Aserción de código de estado
pm.test("El código de estado es 200", function() {
    pm.response.to.have.status(200);
});

// Aserción JSON
pm.test("Devuelve lista de usuarios", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// Aserción de tiempo de respuesta
pm.test("El tiempo de respuesta es menor a 500ms", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### Conjuntos de Pruebas

Puedes organizar los casos de prueba de múltiples interfaces en conjuntos de pruebas:

1. Crear un conjunto de pruebas
2. Agregar casos de prueba
3. Establecer el orden de ejecución
4. Ejecutar el conjunto

### Generar Informes de Prueba

Después de ejecutar las pruebas, puedes generar informes detallados:

- Estadísticas de tasa de aprobación
- Análisis de casos fallidos
- Estadísticas de tiempo de respuesta

## Colaboración en Equipo

### Invitar Miembros

```bash
# Invitar miembros del equipo
1. Entra en la configuración del proyecto
2. Haz clic en "Gestión de Miembros"
3. Haz clic en "Invitar Miembro"
4. Ingresa el correo electrónico del miembro
5. Configura los permisos del rol
6. Envía la invitación
```

### Roles y Permisos

- **Administrador**: Acceso completo al proyecto
- **Editor**: Puede editar interfaces y ejecutar pruebas
- **Visor**: Solo puede ver, no puede editar

### Sincronización en Tiempo Real

Después de que los miembros del equipo inicien sesión, los datos del proyecto se sincronizan automáticamente en tiempo real:

- Los cambios de interfaces se sincronizan automáticamente
- Los resultados de pruebas se sincronizan automáticamente
- Los miembros del equipo pueden ver el contenido más reciente

### Función de Comentarios

Puedes agregar comentarios en las interfaces:

1. Abre la interfaz
2. Haz clic en "Comentarios"
3. Ingresa el contenido del comentario
4. Usa @ para mencionar a miembros del equipo

Atajos más utilizados:

- `Ctrl/Cmd + S`: Guardar
- `Ctrl/Cmd + Enter`: Enviar solicitud
- `Ctrl/Cmd + B`: Formatear JSON
- `Ctrl/Cmd + /`: Comentar/Descomentar

## Mejores Prácticas

1. **Nomenclatura clara**: Los nombres de las interfaces deben describir claramente su función
2. **Comentarios completos**: Agrega说明 para cada parámetro
3. **Agrupación合理**: Organiza las interfaces por módulos
4. **Usar entornos**: Distingue entre entornos de desarrollo, pruebas y producción
5. **Escribir pruebas**: Crea pruebas automatizadas para interfaces importantes
6. **Sincronización regular**: Mantén los datos locales y en la nube sincronizados
