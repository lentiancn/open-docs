# Руководство по установке Kotlin

## Предварительные требования

Перед установкой Kotlin убедитесь, что ваш компьютер соответствует базовым системным требованиям. Kotlin может работать под управлением различных операционных систем, включая Windows, macOS, Linux и другие Unix-подобные системы. Независимо от используемой операционной системы, убедитесь, что у вас достаточно свободного места на диске для установки инструментов разработки и запуска скомпилированных программ.

Для пользователей Windows рекомендуется Windows 10 или более поздняя версия для лучшей работы. Пользователям macOS требуется macOS 10.14 или более поздняя версия. Пользователи Linux могут использовать любой основной дистрибутив, например Ubuntu, Fedora или Debian.

## Установка через IDE

### IntelliJ IDEA

IntelliJ IDEA — это флагманская среда разработки от JetBrains, предоставляющая наиболее полную поддержку Kotlin. Если ваша основная цель — разработка на Kotlin, IntelliJ IDEA — лучший выбор.

IntelliJ IDEA выпускается в двух версиях: Community (бесплатная с открытым исходным кодом) и Ultimate (для корпоративных пользователей с расширенными функциями). Версия Community бесплатна и предоставляет достаточные функции для разработки на Kotlin. Версия Ultimate предлагает дополнительные корпоративные функции, такие как расширенные инструменты для работы с базами данных и функции безопасности.

Для установки посетите официальный сайт JetBrains (https://www.jetbrains.com/idea/download/) и загрузите установщик для вашей операционной системы. После загрузки запустите установщик и следуйте инструкциям для завершения установки. Во время установки можно выбрать путь установки и создать ярлыки на рабочем столе.

При первом запуске IntelliJ IDEA среда разработки автоматически определит и предложит установить плагин Kotlin. Если автоматическое определение не сработало, можно вручную найти Kotlin в магазине плагинов и установить его. После установки перезапустите IDE, чтобы начать разработку на Kotlin.

### Android Studio

Android Studio — это официальная среда разработки Android от Google со встроенной поддержкой Kotlin. Если вы в основном разрабатываете под Android, Android Studio — идеальный выбор.

Посетите сайт разработчиков Android (https://developer.android.com/studio), чтобы загрузить Android Studio. После загрузки запустите установщик для завершения установки. Android Studio по умолчанию включает плагин Kotlin, поэтому дополнительная настройка не требуется для начала разработки.

Android Studio выпускает несколько обновлённых версий каждый год, каждая из которых содержит новейшую поддержку Kotlin и инструменты разработки под Android. Рекомендуется поддерживать версию в актуальном состоянии для получения лучшего опыта разработки и новейших языковых функций.

## Установка через командную строку

Помимо использования IDE, вы также можете установить компилятор Kotlin для командной строки. Этот подход подходит для разработчиков, которые предпочитают лёгкую среду разработки или которым нужен Kotlin для серверной разработки.

### Ручная установка

Сначала посетите страницу релизов Kotlin на GitHub (https://github.com/JetBrains/kotlin/releases), чтобы загрузить последнюю версию компилятора. Загрузка представляет собой zip-архив; распакуйте его в нужный каталог.

После распаковки добавьте каталог kotlinc/bin в переменную среды PATH вашей системы, чтобы использовать команду kotlinc из командной строки. В Windows добавьте путь через Свойства системы > Переменные среды. В macOS и Linux отредактируйте файл ~/.bashrc или ~/.zshrc и добавьте строку:

export PATH="$PATH:/path/to/kotlinc/bin"

После добавления пути откройте новое окно терминала и введите kotlinc -version. Если вы увидите информацию о версии, установка прошла успешно.

### Использование SDKMAN!

SDKMAN! — это инструмент для управления версиями инструментов разработки Java, который удобно использовать в Unix-подобных системах. Если вы используете macOS, Linux, Cygwin, FreeBSD или Solaris, SDKMAN! — рекомендуемый способ установки Kotlin.

Сначала установите SDKMAN!, выполнив эту команду в терминале:

curl -s "https://get.sdkman.io" | bash

После установки перезагрузите конфигурацию среды:

source ~/.sdkman/bin/sdkman-init.sh

Теперь вы можете использовать команду sdk для установки Kotlin:

sdk install kotlin

Во время установки вас попросят подтвердить версию. После подтверждения установка завершена. Вы можете проверить установку с помощью kotlinc -version.

### Использование Homebrew

В macOS вы также можете использовать менеджер пакетов Homebrew для установки Kotlin. Сначала убедитесь, что Homebrew установлен, затем выполните:

brew update
brew install kotlin

Homebrew автоматически загрузит и установит последнюю версию компилятора Kotlin. Проверьте установку с помощью kotlinc -version.

### Использование Snap

В Ubuntu 16.04 и более поздних версиях можно использовать Snap для установки Kotlin:

sudo snap install --classic kotlin

Установка через Snap устанавливает Kotlin как системное приложение, доступное из любого места через командную строку.

## Создание вашей первой программы на Kotlin

Независимо от выбранного способа установки, вы можете создать и запустить свою первую программу на Kotlin, выполнив следующие действия.

Сначала создайте файл hello.kt со следующим содержимым:

fun main() {
    println("Hello, World!")
}

Это простейшая программа на Kotlin, определяющая функцию main, которая является точкой входа в программу на Kotlin. Функция println выводит текст в консоль.

При использовании компилятора командной строки компиляция и запуск этой программы требуют двух шагов. Сначала компилируем:

kotlinc hello.kt -include-runtime -d hello.jar

Параметр -include-runtime упаковывает библиотеку времени выполнения Kotlin в созданный файл jar, делая его самодостаточным. Параметр -d указывает путь к выходному файлу.

После компиляции запустите программу:

java -jar hello.jar

Вы должны увидеть "Hello, World!" в консоли.

Если вы не хотите создавать автономный файл jar, вы также можете использовать интерактивный режим запуска kotlinc:

kotlinc -script hello.kt

Или используйте команду kotlin для непосредственного запуска скомпилированных файлов классов:

kotlin -classpath hello.jar HelloKt

Здесь HelloKt — это имя основного класса, сгенерированное компилятором Kotlin для файла hello.kt.

## Использование REPL

Kotlin предоставляет интерактивную среду REPL (Read-Eval-Print Loop), где вы можете вводить код Kotlin и немедленно видеть результаты. Это полезно для изучения языковых функций или быстрого тестирования фрагментов кода.

Для запуска REPL просто выполните kotlinc без параметров:

kotlinc

Это переводит в режим REPL. Вы можете ввести любой допустимый код Kotlin и нажать Enter, чтобы увидеть результат:

>>> val x = 10
>>> val y = 20
>>> x + y
30

Введите :quit или нажмите Ctrl+C для выхода из REPL.

## Скрипты Kotlin

Kotlin также может использоваться как скриптовый язык. Файлы скриптов Kotlin имеют расширение .kts, а код верхнего уровня в файле обрабатывается как исполняемый код.

Например, создайте скрипт с именем list_folders.kts:

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

Запустите скрипт:

kotlinc -script list_folders.kts -- -d /path/to/folder

## Настройка инструментов сборки

В реальных проектах обычно используются Gradle или Maven в качестве инструментов сборки. Оба поддерживают проекты Kotlin нативно.

### Gradle

Gradle — самый популярный инструмент сборки для разработки на Kotlin. Команда Kotlin также предоставляет Kotlin DSL, позволяющий писать скрипты сборки на синтаксисе Kotlin.

Чтобы создать базовый проект Gradle на Kotlin, добавьте плагин Kotlin в ваш проект. Ваш файл build.gradle должен содержать:

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

Или используя Kotlin DSL:

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

Затем создайте структуру каталогов с исходным кодом, обычно src/main/kotlin. Добавьте исходные файлы Kotlin и используйте gradle build для компиляции проекта.

### Maven

Maven — ещё один популярный инструмент сборки для Java-проектов, который также поддерживает проекты Kotlin. Добавьте зависимости Kotlin в ваш файл pom.xml.

## Проверка установки

После установки вы можете проверить, что всё работает правильно.

Сначала проверьте компилятор Kotlin:

kotlinc -version

Вы должны увидеть что-то вроде "Kotlin compiler version 2.3.10".

Затем проверьте стандартную библиотеку Kotlin:

kotlin -e 'println("Hello Kotlin!")'

Если всё работает, установка прошла успешно, и вы можете начать своё знакомство с Kotlin.

## Следующие шаги

После установки Kotlin рекомендуется продолжить обучение: сначала прочитайте официальный тур по Kotlin, чтобы понять основные языковые функции; затем выберите направление разработки, которое вас интересует — разработка под Android, серверная разработка или кроссплатформенная разработка; наконец, практикуйтесь на реальных проектах, чтобы применить полученные знания.

Kotlin предоставляет обширные учебные ресурсы, включая официальную документацию, онлайн-уроки и видеокурсы. Ссылки на эти ресурсы можно найти на веб-сайте Kotlin.
