# Руководство по установке Kubernetes

## Подготовка

Перед установкой Kubernetes необходимо понять основные требования:

### Системные требования
- Linux — рекомендуемая операционная система для запуска плоскости управления Kubernetes
- В кластере можно запускать приложения на Linux или других операционных системах, включая Windows
- Требуется операционная система с поддержкой среды выполнения контейнеров

### Выбор типа установки
При установке Kubernetes выберите тип установки на основе:
- Простота обслуживания
- Безопасность
- Контроль
- Доступные ресурсы
- Необходимая экспертиза для эксплуатации и управления кластером

## Способы установки

### Обучающая среда (рекомендуется для начинающих)

Если вы изучаете Kubernetes, используйте инструменты, поддерживаемые сообществом Kubernetes, или инструменты экосистемы для настройки кластера на локальном компьютере:

#### Minikube
Minikube — это облегчённая реализация Kubernetes, которая создаёт однодноузловой кластер на локальном компьютере. Идеально подходит для обучения и тестирования.

Шаги установки:
1. Установить Minikube
2. Установить kubectl (инструмент командной строки Kubernetes)
3. Запустить кластер Minikube

#### Kind
Kind (Kubernetes in Docker) — ещё один инструмент для создания локальных кластеров Kubernetes, использующий контейнеры Docker для имитации узлов.

### Производственная среда

Для производственных сред рассмотрите следующие варианты:

#### kubeadm
Официально поддерживаемый инструмент для развёртывания Kubernetes. Создаёт безопасные кластеры в соответствии с лучшими практиками.

Основные шаги:
1. Подготовить серверы (установить ОС, настроить сеть)
2. Установить среду выполнения контейнеров (containerd или Docker)
3. Установить kubeadm, kubelet и kubectl
4. Инициализировать кластер с помощью kubeadm
5. Настроить сетевой плагин (Calico, Cilium и т. д.)
6. Добавить рабочие узлы

#### Управляемые сервисы
Если вы не хотите управлять кластером Kubernetes самостоятельно, рассмотрите управляемые сервисы:
- Amazon EKS (Elastic Kubernetes Service)
- Google GKE (Google Kubernetes Engine)
- Azure AKS (Azure Kubernetes Service)
- Alibaba Cloud ACK
- Tencent Cloud TKE

#### Другие решения
Существуют также стандартизированные и пользовательские решения для различных облачных и физических сред.

## Установка инструментов

### Установка kubectl

kubectl — инструмент командной строки для взаимодействия с кластерами Kubernetes.

Установка в Linux:
```bash
# Скачать последнюю версию
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# Добавить права на выполнение
chmod +x kubectl

# Переместить в системный путь
sudo mv kubectl /usr/local/bin/
```

### Установка kubeadm

```bash
# Обновить индекс пакетов apt
sudo apt-get update

# Установить зависимости
sudo apt-get install -y apt-transport-https ca-certificates curl

# Добавить репозиторий Kubernetes APT
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Обновить и установить
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## Инициализация кластера

### Создание кластера с помощью kubeadm

1. Инициализировать узел плоскости управления:
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. Настроить kubectl:
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. Установить сетевой плагин (Flannel в качестве примера):
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. Добавить рабочие узлы:
```bash
# На каждом рабочем узле выполнить команду из вывода инициализации
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## Проверка установки

Проверить состояние кластера:
```bash
# Просмотреть узлы
kubectl get nodes

# Просмотреть поды
kubectl get pods --all-namespaces
```

Если все компоненты находятся в состоянии Running, установка прошла успешно.

## Удаление

Если необходимо удалить Kubernetes:

```bash
# Остановить все сервисы Kubernetes
sudo kubeadm reset

# Удалить файлы конфигурации
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# Удалить пакеты
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## Важные замечания

1. Выбор среды выполнения контейнеров: выберите подходящую среду для нового кластера
2. Планирование ресурсов: разумно выделяйте ресурсы процессора и памяти в соответствии с рабочими нагрузками
3. Планирование сети: заранее спланируйте CIDR сети подов и сервисов
4. Усиление безопасности: настройте аутентификацию, авторизацию и сетевые политики для производства
5. Высокая доступность: настройте кластеры с несколькими плоскостями управления для производственных сред
