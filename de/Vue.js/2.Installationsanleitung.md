# Vue.js Installationsanleitung

## Vue online ausprobieren

- **Spielwiese**: Probieren Sie es direkt in der [Vue Spielwiese](https://play.vuejs.org) aus
- **JSFiddle**: Verwenden Sie diese [JSFiddle-Vorlage](https://jsfiddle.net/yyx990803/2ke1ab0z/) für einfaches HTML-Setup
- **StackBlitz**: Probieren Sie ein komplettes Build-Setup direkt in Ihrem Browser auf [StackBlitz](https://vite.new/vue) aus
- **Interaktives Tutorial**: Sehen Sie sich das interaktive [Scrimba-Tutorial](http://scrimba.com/links/vue-quickstart) an, das Ihnen zeigt, wie Sie Ihre erste Vue-App ausführen, bearbeiten und bereitstellen

## Erstellen einer Vue-Anwendung (empfohlen)

### Voraussetzungen
- Vertrautheit mit der Kommandozeile
- Installieren Sie Node.js Version ^20.19.0 || >=22.12.0

### Verwendung des offiziellen Gerüstbau-Tools
Führen Sie einen der folgenden Befehle in Ihrem Terminal aus:

```bash
# npm
npm create vue@latest

# pnpm  
pnpm create vue@latest

# yarn (v1+)
yarn create vue

# yarn (Modern v2+)
yarn create vue@latest

# yarn (^v4.11)
yarn dlx create-vue@latest

# bun
bun create vue@latest
```

### Projektkonfigurationsoptionen
Sie werden nach mehreren optionalen Funktionen gefragt:
- Project name: Name Ihres Projekts
- Add TypeScript?: Ob TypeScript-Unterstützung hinzugefügt werden soll
- Add JSX Support?: Ob JSX-Unterstützung hinzugefügt werden soll
- Add Vue Router: Für die Entwicklung von Single-Page-Anwendungen
- Add Pinia: Für Zustandsverwaltung
- Add Vitest: Für Unit-Tests
- Add End-to-End Testing: Cypress/Nightwatch/Playwright
- Add ESLint: Für Code-Qualität
- Add Prettier: Für Code-Formatierung
- Add Vue DevTools 7: Für Debugging (experimentell)

### Starten des Entwicklungsservers
```bash
cd <your-project-name>
npm install
npm run dev
```

### Produktionsbuild
```bash
npm run build
```

## Verwendung von Vue über CDN

### Globale Build-Version
```html
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<div id="app">{{ message }}</div>

<script>
const { createApp } = Vue

createApp({
  data() {
    return {
      message: 'Hello Vue!'
    }
  }
}).mount('#app')
</script>
```

### ES-Modul-Build-Version
```html
<div id="app">{{ message }}</div>

<script type="module">
import { createApp } from 'https://unpkg.com/vue@3/dist/vue.esm-browser.js'

createApp({
  data() {
    return {
      message: 'Hello Vue!'
    }
  }
}).mount('#app')
</script>
```

### Aktivierung von Import-Maps
```html
<script type="importmap">
{
  "imports": {
    "vue": "https://unpkg.com/vue@3/dist/vue.esm-browser.js"
  }
}
</script>

<div id="app">{{ message }}</div>

<script type="module">
import { createApp } from 'vue'

createApp({
  data() {
    return {
      message: 'Hello Vue!'
    }
  }
}).mount('#app')
</script>
```

## Empfehlungen für die Entwicklungsumgebung

- **IDE**: Visual Studio Code + [Vue - Offizielle Erweiterung](https://marketplace.visualstudio.com/items?itemName=Vue.volar)
- **Build-Tool**: Vite (im offiziellen Gerüstbau enthalten)
- **Lokaler Server**: Für ES-Module verwenden Sie `npx serve`, um einen lokalen HTTP-Server zu starten
- **Alternative für einfache Fälle**: Erwägen Sie [petite-vue](https://github.com/vuejs/petite-vue) für leichtgewichtige Szenarien
