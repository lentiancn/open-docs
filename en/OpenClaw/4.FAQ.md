# OpenClaw FAQ

This page lists common questions and solutions when using OpenClaw.

## Installation

### What are the requirements for OpenClaw?

- Node.js 22.x or higher
- macOS, Linux, or Windows (WSL2)
- At least 2GB RAM recommended

### Installation fails. What should I do?

1. Check Node.js version: `node --version` (must be >= 22)
2. Check network connection
3. Try different package managers: npm, pnpm, or bun
4. Run `openclaw doctor` to diagnose

### How do I update OpenClaw?

```bash
# Switch to stable channel
openclaw update --channel stable

# Then restart Gateway
openclaw gateway restart
```

## Gateway

### Gateway won't start

- Check if port 18789 is occupied
- Check logs: `openclaw gateway --verbose`
- Try killing any existing processes

### How do I keep Gateway running constantly?

Use `--install-daemon` to install as system service:

```bash
openclaw onboard --install-daemon
```

### How do I access Gateway remotely?

1. **Tailscale**: Use `gateway.tailscale.mode: "serve"` or `"funnel"`
2. **SSH tunnel**: Establish SSH tunnel connection

## Channels

### WhatsApp login fails

- Ensure you've paired your device with `openclaw channels login`
- Check if network can reach WhatsApp servers

### Telegram Bot not working

- Check Bot Token is correct
- Verify you've started a chat with the Bot
- Verify `allowFrom` configuration

### Discord messages not responding

- Check if Bot has joined the server
- Confirm correct permissions are configured
- Check if channel is in allowlist

## Security

### Can strangers send me messages?

By default, OpenClaw requires pairing verification for unknown senders. Use `dmPolicy: "open"` to change to open mode, but not recommended.

### How do I manage pairing?

```bash
# View pairing list
openclaw pairing list

# Approve pairing
openclaw pairing approve

# Revoke pairing
openclaw pairing revoke
```

### Running `openclaw doctor` shows warnings?

Follow the instructions to fix. Common issues:
- DM policy too open
- Missing required auth config
- Port exposure risks

## Skills

### How do I install new skills?

```bash
# Install via ClawHub
openclaw skill install skill-name

# Or manually create skill directory and SKILL.md
```

### Skills not working?

- Check skill configuration is correct
- Check Gateway logs
- Confirm skill dependencies are installed

## Voice

### Voice Wake not working?

- macOS: Check System Preferences for Accessibility permissions
- iOS: Confirm microphone permission is enabled
- Check if wake word is set correctly

### Voice response too slow?

- Check network latency
- Consider using local TTS
- Adjust voice model settings

## Data

### Where is data stored?

- Config: `~/.openclaw/`
- Workspace: `~/.openclaw/workspace`
- Credentials: `~/.openclaw/credentials`

### How do I back up data?

```bash
# Backup entire config directory
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### How do I reset all settings?

1. Stop Gateway
2. Delete `~/.openclaw` directory
3. Run `openclaw onboard` again

## Other

### How do I view logs?

```bash
# View Gateway logs
openclaw gateway --verbose

# Or check specific log files
openclaw logs
```

### How do I get help?

- Documentation: https://docs.openclaw.ai
- Join Discord community
- Submit Issue on GitHub

### What's the difference between OpenClaw and other AI assistants?

- **Fully local**: Your data stays local, not uploaded to cloud
- **Open source**: Code is completely open, you can review it yourself
- **Highly customizable**: You can modify and extend any feature
- **Multi-channel unified**: One assistant manages all communication channels

### Can I run multiple OpenClaw instances simultaneously?

Yes, each instance needs:
- Different port
- Independent workspace
- Different configuration
