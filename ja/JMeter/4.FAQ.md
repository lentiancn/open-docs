# JMeter FAQ

## 一般質問

### Apache JMeterとは何ですか？
Apache JMeterは、負荷テストとパフォーマンス測定のために設計されたオープンソースソフトウェアです。機能動作をテストし、パフォーマンスを測定できる100％Pure Javaアプリケーションです。

### JMeterはどのプロトコルをサポートしていますか？
JMeterは複数のプロトサ PORTをサポート:
- HTTP/HTTPS
- SOAP/REST Webサービス
- FTP
- JDBC（データベース）
- LDAP
- JMS（メッセージ指向ミドルウェア）
- SMTP/POP3/IMAP（メール）
- TCP
- Javaオブジェクト

### JMeterはブラウザですか？
いいえ、JMeterはブラウザではありません。プロトコルレベルで動作し、実際のブラウザのようにJavaScriptを実行したりHTMLページをレンダリングしたりしません。

## インストール質問

### システム要件は何ですか？
- Java 8以上
- 少なくとも4GBのRAMを推奨
- 対応OS: Windows、Linux、macOS

### JMeterをインストールするには？
1. https://jmeter.apache.org/からダウンロード
2. アーカイブを展開
3. jmeter.sh（Linux/Mac）またはjmeter.bat（Windows）を実行

### JMeterが起動しないのはなぜですか？
- Javaがインストールされているか確認: `java -version`
- JAVA_HOMEが設定されているか確認
- ポートの競合を確認

### JMeterのメモリを増やすには？
jmeterスタートアップスクリプトを編集してHEAP変数を変更:
```bash
HEAP="-Xms1g -Xmx4g"
```

## 使用質問

### 基本的なテストplanを作成するには？
1. スレッドグループを追加
2. HTTPリクエストサンプラーを追加
3. リスナー（結果ツリー、概要レポート）を追加
4. テストを実行

### スレッドグループとは何ですか？
スレッドグループは、同じテストを実行する仮想ユーザーグループを定義します。 以下を制御:
- スレッド数（ユーザー）
- ランプアップ期間
- ループ回数

### レスポンスからデータを抽出するには？
エクストラクターを使用:
- JSONレスポンスにはJSONエクストラクター
- XML/HTMLにはXPathエクストラクター
- 任意のテキストには正規表現エクストラクター

### テストをパラメータ化するには？
変数と関数を使用:
- `${variable}` 変数を参照
- `${__Random(1,100)}` ランダム値
- `${__time()}`

### JMeterは認証を処理できますか？
はい、JMeterは以下をサポート:
- -basic認証
- NTLM認証
- Cookieマネージャー
- 認可マネージャー

## パフォーマンス質問

### JMeterは何人のユーザーをシミュレートできますか？
数はハードウェアに依存します。適切なチュ:
- GUIモード: 100-300ユーザー
- CLIモード: 1000+ユーザー
- 分散テスト: 10,000+ユーザー

### 分散テストとは何ですか？
分散テストは、複数のマシン Moreを使用して負荷を 生成します。マスターが複数のスレーブマシンを調整します。

### JMeterをヘッドレスモードで実行するには？
```bash
jmeter -n -t testplan.jmx -l results.jtl
```

## トラブルシューティング

### テスト中のOutOfMemoryError
- ヒープサイズを増加
- スレッド数を削減
- GUIではなくCLIモードを使用
- 不要なリスナーを無効化

### 結果が表示されない
- テストが実際に実行されているか確認
- リスナーが正しい要素に接続されているか確認
- ログファイルでエラーを確認

### レスポンス時間が高い
- ネットワークを確認
- サーバー Perkinsを確認
- サーバーが圧倒されている場合は負荷を削減

### テストplanをデバッグするには？
1. 結果ツーリーナーを使用
2. デバッグロギングを有効化
3. 単一スレッドから開始
4. アサーションを確認

## ベストプラクティス

1. 負荷テストには常にCLIモードを使用
2. 低負荷から開始し、段階的に増加
3. レスポンスを検証するためにアサーションを使用
4. テスト後にクリーンアップ（tearDown）
5. テストplanをバージョン管理下に置く
6. システムリソースを監視
