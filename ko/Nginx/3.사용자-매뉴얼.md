# Nginx 사용자 매뉴얼

## 기본 작업

### Nginx 시작

```bash
# Nginx 시작
nginx

# 또는 systemctl 사용 (systemd의 경우)
systemctl start nginx
```

### Nginx 중지

```bash
# 빠른 중지 (즉시 종료)
nginx -s stop

#优雅停止 (요청 완료 대기)
nginx -s quit
```

### 구성 다시 로드

```bash
# 구성 파일 다시 로드
nginx -s reload
```

### 로그 다시 열기

```bash
# 로그 파일 다시 열기
nginx -s reopen
```

### 구성 확인

```bash
# 구성 구문 테스트
nginx -t

# 전체 구성 표시
nginx -T
```

## 구성 파일 구조

### 기본 구성 구조

```
nginx.conf
├── main (전역 블록)
│   ├── user
│   ├── worker_processes
│   ├── error_log
│   └── pid
│
├── events (events 블록)
│   └── worker_connections
│
└── http (http 블록)
    ├── include
    ├── default_type
    ├── log_format
    ├── access_log
    ├── sendfile
    ├── keepalive_timeout
    ├── gzip
    │
    └── server (server 블록)
        ├── listen
        ├── server_name
        ├── location
        │   ├── root
        │   ├── index
        │   └── proxy_pass
        │
        └── error_page
```

### 일반적인 지시문

#### main 블록

```nginx
# Nginx 실행 사용자
user nginx;

# 워커 프로세스 수 (auto는 CPU 자동 감지)
worker_processes auto;

# 오류 로그 경로
error_log /var/log/nginx/error.log warn;

# PID 파일 경로
pid /var/run/nginx.pid;
```

#### events 블록

```nginx
events {
    # 워커당 최대 연결 수
    worker_connections 1024;
    
    # epoll 모델 사용 (Linux)
    use epoll;
    
    # 여러 연결을 동시에 수락
    multi_accept on;
}
```

#### http 블록

```nginx
http {
    # MIME 유형 포함
    include /etc/nginx/mime.types;
    
    # 기본 파일 유형
    default_type application/octet-stream;
    
    # 액세스 로그
    access_log /var/log/nginx/access.log;
    
    # 파일 전송
    sendfile on;
    
    # TCP_NOPUSH 및 TCP_NODELAY
    tcp_nopush on;
    tcp_nodelay on;
    
    # keep-alive 타임아웃
    keepalive_timeout 65;
    
    # gzip 압축 활성화
    gzip on;
}
```

## 정적 파일 제공

### 기본 구성

```nginx
server {
    listen 80;
    server_name example.co.kr www.example.co.kr;
    
    # 루트 디렉토리
    root /var/www/html;
    
    # 기본 인덱스 파일
    index index.html index.htm;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    # 액세스 로그
    access_log /var/log/nginx/example.co.kr.access.log;
    error_log /var/log/nginx/example.co.kr.error.log;
}
```

### 디렉토리 목록

```nginx
location / {
    # 디렉토리 목록 활성화
    autoindex on;
    
    # 정확한 파일 크기 표시
    autoindex_exact_size on;
    
    # 현지 시간 표시
    autoindex_localtime on;
}
```

### 정적 리소스 캐싱

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2|ttf|svg)$ {
    expires 30d;
    add_header Cache-Control "public, no-transform";
}
```

## 리버스 프록시

### 기본 리버스 프록시

```nginx
server {
    listen 80;
    server_name example.co.kr;
    
    location / {
        proxy_pass http://backend_server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 캐싱이 있는 리버스 프록시

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m use_temp_path=off;

server {
    listen 80;
    server_name example.co.kr;
    
    location / {
        proxy_cache my_cache;
        proxy_cache_valid 200 60m;
        proxy_cache_use_stale error timeout http_500 http_502 http_503 http_504;
        proxy_cache_lock on;
        
        proxy_pass http://backend_server;
        proxy_set_header Host $host;
    }
}
```

### WebSocket 프록시

```nginx
location /ws/ {
    proxy_pass http://backend_server;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_read_timeout 86400;
}
```

## 로드 밸런싱

### Round-Robin 로드 밸런싱

```nginx
upstream backend {
    server backend1.example.co.kr;
    server backend2.example.co.kr;
    server backend3.example.co.kr;
}

server {
    listen 80;
    server_name example.co.kr;
    
    location / {
        proxy_pass http://backend;
    }
}
```

### 가중 로드 밸런싱

```nginx
upstream backend {
    server backend1.example.co.kr weight=5;
    server backend2.example.co.kr weight=2;
    server backend3.example.co.kr;
}
```

### IP 해시 로드 밸런싱

```nginx
upstream backend {
    ip_hash;
    server backend1.example.co.kr;
    server backend2.example.co.kr;
    server backend3.example.co.kr down;
}
```

### Least Connections

```nginx
upstream backend {
    least_conn;
    server backend1.example.co.kr;
    server backend2.example.co.kr;
}
```

## SSL/TLS 구성

### 기본 HTTPS 구성

```nginx
server {
    listen 443 ssl http2;
    server_name example.co.kr;
    
    ssl_certificate /etc/nginx/ssl/example.co.kr.crt;
    ssl_certificate_key /etc/nginx/ssl/example.co.kr.key;
    
    # SSL 프로토콜 버전
    ssl_protocols TLSv1.2 TLSv1.3;
    
    # SSL 암호화 스위트
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;
    
    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;
    
    location / {
        root /var/www/html;
        index index.html;
    }
}
```

### HTTP에서 HTTPS로 리디렉션

```nginx
server {
    listen 80;
    server_name example.co.kr;
    return 301 https://$server_name$request_uri;
}
```

## 일반적인 구성 예제

### PHP-FPM 구성

```nginx
server {
    listen 80;
    server_name example.co.kr;
    root /var/www/html;
    index index.php index.html;
    
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
    
    location ~ /\.ht {
        deny all;
    }
}
```

### Node.js 프록시

```nginx
server {
    listen 80;
    server_name example.co.kr;
    
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 속도 제한 구성

```nginx
# 연결 기반 제한
limit_conn_zone $binary_remote_addr zone=addr:10m;

server {
    location / {
        limit_conn addr 10;
    }
}

# 요청 속도 기반 제한
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;

server {
    location / {
        limit_req zone=one burst=20 nodelay;
    }
}
```

```nginx
    if ($invalid_referer) {

    }
}
```

## 로그 관리

### 사용자 정의 로그 형식

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent" "$http_x_forwarded_for"';

access_log /var/log/nginx/access.log main;
```

### 조건부 로깅

```nginx
map $request_uri $loggable {
    ~^/health-check 0;
    default 1;
}

access_log /var/log/nginx/access.log combined if=$loggable;
```

### 로그 순환

logrotate 사용:

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data adm
    sharedscripts
    postrotate
        [ -f /var/run/nginx.pid ] && kill -USR1 `cat /var/run/nginx.pid`
    endscript
}
```
