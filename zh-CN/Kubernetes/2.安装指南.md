# Kubernetes 安装指南

## 准备工作

在安装 Kubernetes 之前，需要了解以下基本要求：

### 系统要求
- Linux 是运行 Kubernetes 控制平面的首选操作系统
- 在集群中可以在 Linux 或其他操作系统（包括 Windows）上运行应用程序
- 需要支持容器运行的操作系统

### 组件选择
安装 Kubernetes 时，请根据以下条件选择安装类型：
- 易于维护
- 安全性
- 可控制性
- 可用资源
- 操作和管理 Kubernetes 集群所需的专业知识

## 安装方式

### 学习环境（推荐入门）

如果正打算学习 Kubernetes，建议使用 Kubernetes 社区支持或生态系统中的工具在本地计算机上设置集群：

#### Minikube
Minikube 是一个轻量级的 Kubernetes 实现，可以在本地机器上创建单节点集群。适合学习和小规模测试。

安装步骤：
1. 安装 Minikube
2. 安装 kubectl（Kubernetes 命令行工具）
3. 启动 Minikube 集群

#### Kind
Kind 是另一个用于创建本地 Kubernetes 集群的工具，使用 Docker 容器模拟节点。

### 生产环境

对于生产环境，需要考虑以下选项：

#### kubeadm
官方支持的用于部署 Kubernetes 的工具。可以创建符合最佳实践的安全集群。

基本步骤：
1. 准备服务器（安装操作系统、配置网络）
2. 安装容器运行时（如 containerd 或 Docker）
3. 安装 kubeadm、kubelet 和 kubectl
4. 使用 kubeadm 初始化集群
5. 配置网络插件（如 Calico、Cilium）
6. 添加工作节点

#### 托管服务
如果不想自己管理 Kubernetes 集群，可以选择托管服务：
- Amazon EKS（Elastic Kubernetes Service）
- Google GKE（Google Kubernetes Engine）
- Azure AKS（Azure Kubernetes Service）
- 阿里云 ACK
- 腾讯云 TKE

#### 其他方案
在各种云和裸机环境中，还有其他标准化和定制的解决方案可供选择。

## 工具安装

### kubectl 安装

kubectl 是与 Kubernetes 集群交互的命令行工具。

Linux 安装：
```bash
# 下载最新版本
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 添加执行权限
chmod +x kubectl

# 移动到系统路径
sudo mv kubectl /usr/local/bin/
```

### kubeadm 安装

```bash
# 更新 apt 包索引
sudo apt-get update

# 安装依赖包
sudo apt-get install -y apt-transport-https ca-certificates curl

# 添加 Kubernetes APT 源
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# 更新并安装
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## 集群初始化

### 使用 kubeadm 创建集群

1. 初始化主节点：
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. 配置 kubectl：
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. 安装网络插件（以 Flannel 为例）：
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. 加入工作节点：
```bash
# 在每个工作节点上运行初始化输出中的命令
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## 验证安装

检查集群状态：
```bash
# 查看节点
kubectl get nodes

# 查看 Pod
kubectl get pods --all-namespaces
```

如果所有组件都处于 Running 状态，说明集群安装成功。

## 卸载

如果需要卸载 Kubernetes：

```bash
# 停止所有 Kubernetes 服务
sudo kubeadm reset

# 删除配置文件
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# 卸载软件包
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## 注意事项

1. 容器运行时选择：确保为新集群选择合适的容器运行时
2. 资源配置：根据工作负载合理分配 CPU 和内存资源
3. 网络规划：提前规划 Pod 和 Service 的网络 CIDR
4. 安全加固：生产环境需要配置认证、授权和网络策略
5. 高可用：生产环境建议配置多主节点集群
