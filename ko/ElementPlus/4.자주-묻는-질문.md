# Element Plus 자주 묻는 질문

이 페이지는 Element Plus 사용 시 일반적인 문제와 해결 방법을 나열합니다.

## 설치 문제

### 설치 실패

**증상**: npm install 또는 yarn add 실패

**해결 방법**:

1. Node.js 버전 확인, 14.0.0 이상인지
2. npm 캐시 지우기:
   ```bash
   npm cache clean --force
   ```
3. node_modules와 package-lock.json을 삭제하고 다시 설치

### TypeScript 타입 오류

**증상**: 컴포넌트 가져올 때 타입 오류

**해결 방법**:

1. Volar 플러그인이 설치되어 있는지 확인
2. tsconfig.json에 추가:
   ```json
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }
   ```

## 사용 문제

### 컴포넌트 스타일이 적용되지 않음

**증상**: 컴포넌트 스타일이 예상과 일치하지 않음

**해결 방법**:

1. CSS가 올바르게 가져와졌는지 확인:
   ```javascript
   import 'element-plus/dist/index.css'
   ```
2. 전역 스타일 재정의 확인

### 아이콘이 표시되지 않음

**증상**: 아이콘 컴포넌트가 비어 있거나 사각형으로 표시

**해결 방법**:

1. 아이콘 라이브러리 설치:
   ```bash
   npm install @element-plus/icons-vue
   ```

2. 아이콘 전역 등록:
   ```javascript
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }
   ```

### 필요 시 가져오기가 작동하지 않음

**증상**: 번들 크기가 여전히 큼

**해결 방법**:

1. 플러그인 구성이 올바른지 확인
2. 올바른 resolver가 사용되고 있는지 확인

### 국제화가 작동하지 않음

**증상**: 언어 전환 후 컴포넌트 텍스트가 변경되지 않음

**해결 방법**:

1. 언어 팩을 올바르게 가져오기:
   ```javascript
   import en from 'element-plus/es/locale/lang/en'
   ```
2. 로케일 구성:
   ```javascript
   app.use(ElementPlus, { locale: en })
   ```

## 컴포넌트 문제

### 테이블 문제

#### 테이블 정렬이 작동하지 않음

prop 속성이 설정되어 있는지 확인:

```vue
<el-table-column prop="date" sortable />
```

#### 테이블 고정 열이 맞지 않음

테이블에 고정 높이가 있는지 확인:

```vue
<el-table :data="data" height="300" />
```

### 폼 문제

#### 폼 유효성 검사가 트리거되지 않음

el-form-item에 prop 속성이 있는지 확인:

```vue
<el-form-item label="이름" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>
```

### 대화상자 문제

#### 대화상자 스크롤 문제

대화상자 콘텐츠가 너무 긴 경우 스크롤을 추가해야 합니다:

```vue
<el-dialog>
  <div style="max-height: 60vh; overflow-y: auto;">
    <!-- 콘텐츠 -->
  </div>
</el-dialog>
```

## 스타일 문제

### 스타일 충돌

**증상**: 사용자 정의 스타일이 재정의됨

**해결 방법**:

1. 더 구체적인 선택자 사용
2. ::v-deep 또는 :deep() 사용

### 테마 색상이 적용되지 않음

CSS 변수가 올바르게 재정의되었는지 확인:

```css
:root {
  --el-color-primary: #409eff;
}
```

element-plus 스타일 이후에 로드되었는지 확인하세요.

## 성능 문제

### 페이지 버벅임

대용량 데이터셋의 경우:

1. el-table-v2(가상 테이블) 사용
2. 페이지네이션으로 데이터 로드
3. 테이블에서 복잡한 컴포넌트 피하기

### 번들 크기가 너무 큼

1. 필요 시 가져오기 사용
2. 사용하지 않는 플러그인 제거

## Vue 버전 문제

### Vue 2와 호환되지 않음

Element Plus는 Vue 3만 지원합니다. Vue 2가 필요한 경우 Element UI를 사용하세요.

### Composition API가 지원되지 않음

더 나은 Composition API 지원을 위해 Vue 3.2+를 사용해야 합니다.
