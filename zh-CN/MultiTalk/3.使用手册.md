# MultiTalk 使用手册

本手册详细介绍 MultiTalk 的使用方法，包括基本操作、参数配置和进阶技巧。

## 基础使用方法

### 准备输入文件

MultiTalk 需要准备特定的输入格式。项目使用 JSON 文件定义生成参数，包括音频路径、图像路径和文本提示等信息。

单人物生成的输入 JSON 格式如下：需要指定音频文件路径、参考图像路径、视频保存名称等参数。示例文件位于 examples 目录下的 single_example_1.json。

多人对话生成的输入 JSON 需要包含多个音频流和对应的人物区域定义。示例文件位于 examples 目录下的 multitalk_example_2.json。

### 运行基本生成

安装配置完成后，进入项目目录，运行以下命令进行视频生成：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### 参数说明

--ckpt_dir 指定模型权重目录路径。--wav2vec_dir 指定音频编码器目录路径。--input_json 指定输入参数 JSON 文件路径。--sample_steps 指定采样步数，值越大生成质量越高但速度越慢。--mode 指定生成模式，streaming 为长视频模式，clip 为短视频模式。--use_teacache 是否启用 TeaCache 加速。--save_file 指定输出文件保存名称。

## 进阶使用技巧

### 调整生成质量

采样步数是影响生成质量的关键参数。默认 40 步可获得良好效果。如需快速生成，可将步数降至 10 步，这对口型同步影响较小，但会影响动作流畅度和视觉质量。增加步数可提升细节表现。

口型同步强度通过音频引导系数控制。Audio CFG 值设置在 3 到 5 之间效果最佳。增加该值可获得更好的口型同步效果。

### 长视频生成

MultiTalk 支持生成长达 15 秒的视频。使用 streaming 模式可以生成较长的视频片段：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

生成超长视频时，不同片段之间可能存在色调差异。将 Audio CFG 设置为 3 可以缓解这种色调变化问题。

### 低显存运行

如果显存有限，可以使用以下参数降低显存占用：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

添加 --num_persistent_param_in_dit 0 参数可以在单张 RTX 4090 上生成 480P 视频。

### 启用加速

项目支持多种加速方式来提升生成速度。

TeaCache 加速可在保持质量的同时显著提升速度。推荐将 --teacache_thresh 设置在 0.2 到 0.5 之间：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### 使用 LoRA 加速

使用 LoRA 可以大幅减少采样步数。使用 FusioniX LoRA 只需 8 步，使用 lightx2v LoRA 只需 4 步：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## 使用 Gradio 界面

项目提供了 Gradio 网页界面，方便交互式使用。进入项目目录，运行：

```bash
python app.py
```

或者使用低显存版本：

```bash
python app.py --num_persistent_param_in_dit 0
```

## 集成 TTS 语音合成

MultiTalk 支持直接输入文本，通过集成 TTS 技术生成语音后再生成视频：

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## 多 GPU 配置

生成 720P 高分辨率视频需要多 GPU 支持。使用以下命令配置多 GPU：

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## 使用 ComfyUI

项目还支持集成到 ComfyUI 工作流中使用。需要安装 ComfyUI-WanVideoWrapper 扩展包，然后加载 MultiTalk 模型进行工作流编排。
