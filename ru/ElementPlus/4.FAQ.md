# Element Plus FAQ

Часто задаваемые вопросы и ответы.

## Проблемы с установкой

### Q: Ошибка установки?

**Решения:**
1. Проверить версию Node.js
```bash
node --version  # Требуется 18.0+
```
2. Очистить кэш npm
```bash
npm cache clean --force
```
3. Удалить node_modules и package-lock.json, переустановить
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: Модуль не найден?

**Решения:**
1. Подтвердить, что пакет установлен
```bash
npm list element-plus
```
2. Проверить настройки npm/yarn/pnpm
3. Попробовать удалить файл блокировки и переустановить

### Q: Ошибки типов TypeScript?

**Решения:**
1. Установить определения типов
```bash
npm install -D @types/node
```
2. Добавить в tsconfig.json
```json
{
  "compilerOptions": {
    "types": ["element-plus/global"]
  }
}
```

## Проблемы с использованием

### Q: Компонент не отображается?

**Решения:**
1. Подтвердить правильный импорт
2. Проверить сообщения об ошибках в консоли
3. Подтвердить версию Vue (требуется 3.2+)

### Q: Стили не применяются?

**Решения:**
1. Подтвердить, что CSS импортирован
```typescript
import 'element-plus/dist/index.css'
```
2. Проверить приоритет CSS, использовать `:deep()` или `!important`
3. Проверить переопределения стилей другими правилами

### Q: Иконки не отображаются?

**Решения:**
1. Установить пакет иконок
```bash
npm install @element-plus/icons-vue
```
2. Зарегистрировать глобально
```typescript
import * as icons from '@element-plus/icons-vue'
for (const [key, component] of Object.entries(icons)) {
  app.component(key, component)
}
```
3. Или импортировать по требованию

### Q: Таблица не отображается?

**Решения:**
1. Подтвердить, что свойство data правильно привязано
2. Проверить правильность props
3. Подтвердить, что таблица находится вне el-table-column

### Q: Валидация формы не работает?

**Решения:**
1. Убедиться, что prop соответствует v-model
2. Убедиться, что есть ссылка ref
3. Правильно вызвать метод validate
```typescript
const formRef = ref()
await formRef.value.validate()
```

### Q: Диалог не отображается?

**Решения:**
1. Проверить привязку v-model
2. Подтвердить, что visible = true
3. Проверить конфликты z-index

## Проблемы с темой

### Q: Как изменить цвет темы?

**Решения:**
Использовать CSS-переменные:
```css
:root {
  --el-color-primary: #409eff;
}
```

### Q: Как использовать тёмную тему?

**Решения:**
Импортировать тёмный CSS:
```typescript
import 'element-plus/theme-chalk/dark/css-vars.css'
```

### Q: Как переопределить стили компонентов?

**Решения:**
Использовать CSS-селекторы:
```css
.el-button--primary {
  background-color: #ваш-цвет;
}
```

## Проблемы с интернационализацией

### Q: Как переключить язык?

**Решения:**
```typescript
import ru from 'element-plus/es/locale/lang/ru'
import en from 'element-plus/es/locale/lang/en'

app.use(ElementPlus, { locale: ru })
```

### Q: Настроить текст?

**Решения:**
```typescript
import {ElMessage} from 'element-plus'
ElMessage('Пользовательское сообщение')
```

## Проблемы с производительностью

### Q: Медленная начальная загрузка?

**Решения:**
1. Использовать импорт по требованию
2. Использовать плагин автоматического импорта
3. Настроить предварительную сборку Vite

### Q: Слишком большой размер бандла?

**Решения:**
1. Убедиться, что используется импорт по требованию
2. Использовать tree shaking
3. Проверить, не импортируется ли полный пакет

## Проблемы совместимости

### Q: Браузер IE не поддерживается?

Element Plus не поддерживает браузер IE. Использовать:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### Q: Конфликты с другими библиотеками?

**Решения:**
1. Проверить уровни z-index
2. Использовать CSS Modules для предотвращения конфликтов стилей
3. Проверить совместимость версии Vue
