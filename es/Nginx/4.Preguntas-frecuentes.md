# Preguntas frecuentes sobre Nginx

## Problemas de instalación

### ¿Qué debo hacer si la instalación de Nginx falla?

1. Verificar si las dependencias del sistema están completas
2. Revisar los logs de error para entender la causa específica del fallo
3. Asegurar que hay suficiente espacio en disco
4. Verificar si hay problemas de permisos

### ¿Cómo verificar la versión de Nginx?

```bash
nginx -v
nginx -V  # Información detallada incluyendo parámetros de compilación
```

### ¿Qué sistemas operativos soporta Nginx?

Nginx soporta Linux, FreeBSD, Solaris, AIX, HP-UX, macOS y Windows.

## Problemas de configuración

### ¿Cómo verificar si la configuración de Nginx es correcta?

```bash
nginx -t
```

### ¿Dónde está el archivo de configuración?

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- Instalación compilada: `/usr/local/nginx/conf/nginx.conf`

### ¿Necesito reiniciar después de modificar la configuración?

No es necesario reiniciar. Use `nginx -s reload` para recargar la configuración.

## Problemas de rendimiento

### ¿Cómo optimizar el rendimiento de Nginx?

1. **Ajustar el número de procesos worker**

```nginx
worker_processes auto;
```

2. **Aumentar conexiones de worker**

```nginx
events {
    worker_connections 4096;
}
```

3. **Habilitar compresión gzip**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **Habilitar caché del navegador**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **Usar pool de conexiones**

```nginx
upstream backend {
    server backend1.example.com;
    keepalive 32;
}
```

### ¿Qué hacer si Nginx consume demasiada memoria?

1. Reducir el número de worker_connections
2. Limitar el tamaño del cuerpo de solicitudes
3. Optimizar la configuración de caché
4. Verificar si hay fugas de memoria

### ¿Cómo manejar alta concurrencia?

1. Usar balanceo de carga
2. Habilitar caché
3. Ajustar parámetros del kernel
4. Usar CDN

## Problemas de seguridad

### ¿Cómo prevenir ataques DDoS?

1. **Limitar conexiones**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **Limitar tasa de solicitudes**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **Cerrar conexiones lentas**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### ¿Cómo configurar certificado SSL?

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### ¿Cómo forzar HTTPS?

```nginx
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

## Problemas de proxy

### ¿Qué hacer si el proxy inverso falla?

1. Verificar si el servicio backend está ejecutándose
2. Configuración del firewall de verificación
3. Verificar que la configuración de proxy_pass sea correcta
4. Revisar logs de error

### ¿Cómo hacer proxy de WebSocket?

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### ¿Cómo obtener IP real después del proxy?

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## Problemas de logs

### ¿Qué hacer si los archivos de log son muy grandes?

Usar logrotate para rotación de logs:

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### ¿Cómo desactivar el log de acceso?

```nginx
access_log off;
```

### ¿Cómo configurar formato de log?

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## Problemas de balanceo de carga

### ¿Cómo selecciona el backend el balanceo de carga?

Nginx soporta múltiples algoritmos de balanceo de carga:

- **Round-robin (predeterminado)**: Distribuye solicitudes secuencialmente
- **Round-robin ponderado**: Distribuye basado en peso
- **IP hash**: Ruta solicitudes de la misma IP al mismo backend
- **Least connections**: Ruta al backend con menos conexiones

### ¿Qué pasa si el servidor backend falla?

Nginx saltará automáticamente los nodos fallidos y reenviará solicitudes a nodos saludables.

## Problemas de caché

### ¿Cómo configurar proxy de caché?

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### ¿Qué hacer si la caché no funciona?

1. Verificar permisos de ruta de caché
2. Confirmar que las cabeceras de respuesta permiten caché
3. Verificar que la configuración de clave de caché sea correcta

### ¿Cómo limpiar la caché?

```bash
rm -rf /var/cache/nginx/*
```

## Problemas de puertos

### ¿Qué pasa si el puerto ya está en uso?

```bash
# Ver uso de puerto
netstat -tulpn | grep :80

# Usar otro puerto
listen 8080;
```

### ¿Cómo escuchar en múltiples puertos?

```nginx
server {
    listen 80;
    listen 8080;
    server_name example.com;
}
```

## Códigos de error comunes

### 400 Bad Request

- Cabecera de solicitud muy grande: Aumentar `client_header_buffer_size`
- URL muy larga: Verificar ruta de solicitud

### 403 Forbidden

- Archivo de índice no existe
- Permisos insuficientes: Verificar permisos de archivos y directorios

### 404 Not Found

- Error de ruta de archivo
- Bloqueado por SELinux o AppArmor

### 502 Bad Gateway

- Servicio backend no está ejecutándose
- Error de configuración FastCGI/PHP-FPM

### 504 Gateway Timeout

- Tiempo de espera de respuesta del backend
- Aumentar `proxy_read_timeout`

## Consejos de depuración

### ¿Cómo ver información detallada de errores?

1. Habilitar logs de depuración:

```nginx
error_log /var/log/nginx/error.log debug;
```

2. Revisar logs de error:

```bash
tail -f /var/log/nginx/error.log
```

### ¿Cómo probar si la configuración toma efecto?

```bash
# Probar configuración
nginx -t

# Mostrar configuración completa
nginx -T
```

### ¿Cómo depurar expresiones regulares?

Usar prueba de `location`:

```nginx
location /test {
    access_log /var/log/nginx/test.log;
}
```

## Problemas de actualización

### ¿Cómo actualizar Nginx?

1. Respaldar archivos de configuración
2. Detener Nginx
3. Instalar nueva versión
4. Iniciar Nginx
5. Probar funcionalidad

### ¿Cómo realizar actualización sin interrupción?

```bash
# Compilar nueva versión
make

# Enviar señal USR2 para actualizar
kill -s USR2 $(cat /var/run/nginx.pid)

# Apagar elegantemente procesos worker antiguos
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
