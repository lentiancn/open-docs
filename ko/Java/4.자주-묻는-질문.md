# Java 자주 묻는 질문

## 기본 문제

### 문제 1: java 및 javac 명령을 찾을 수 없음

문제: 명령줄에 java -version 또는 javac -version을 입력하면 명령을 찾을 수 없음이 표시됩니다.

해결 방법:
1. JDK가 올바르게 설치되었는지 확인
2. JAVA_HOME 환경 변수가 구성되어 있는지 확인
3. Path 환경 변수에 %JAVA_HOME%\bin(Windows) 또는 $JAVA_HOME/bin(Linux/macOS)이 포함되어 있는지 확인
4. 터미널을 다시 열어 환경 변수 변경 적용

### 문제 2: main 클래스를 찾을 수 없거나 로드할 수 없음

문제: java 명령을 실행하면 "Could not find or load main class"가 표시됩니다.

해결 방법:
1. 올바른 디렉토리에서 명령을 실행하고 있는지 확인
2. 클래스 이름이 파일 이름과 정확히 일치하는지 확인
3. 클래스가 패키지에 있으면 전체 패키지 이름으로 실행
4. classpath가 올바르게 설정되어 있는지 확인

### 문제 3: 인코딩 문제로 한글이 깨짐

문제: 프로그램 출력의 한글이 깨진 텍스트로 나타납니다.

해결 방법:
1. 컴파일 시 인코딩 지정: javac -encoding UTF-8 MyClass.java
2. 실행 시 인코딩 지정: java -Dfile.encoding=UTF-8 MyClass
3. 소스 파일이 UTF-8로 저장되어 있는지 확인
4. IDE에서 올바른 파일 인코딩 설정

## 환경 구성 문제

### 문제 4: 여러 JDK 버전 충돌

문제: 시스템에 여러 JDK 버전이 있으며 java 명령이 잘못된 버전을 사용합니다.

해결 방법:
1. JAVA_HOME이 어떤 JDK 버전을 가리키는지 확인
2. Windows에서 where java를 사용하여 모든 java 명령 위치 확인
3. Linux/macOS에서 which java를 사용하여 경로 확인
4. PATH 환경 변수에서 JDK bin 디렉토리 순서 조정
5. update-alternatives(Linux)를 사용하여 여러 버전 관리

### 문제 5: 포트가 이미 사용 중

문제: Java 웹 애플리케이션 시작 시 포트가 이미 사용 중임을 표시합니다.

해결 방법:
1. Windows: netstat -ano | findstr 8080으로 포트 사용 프로세스 찾기
2. Linux: lsof -i:8080 또는 netstat -tulpn | grep 8080
3. 포트를 사용하는 프로세스를 종료하거나 애플리케이션의 포트 구성 변경

### 문제 6: 메모리 부족 오류

문제: 실행 중 Program throws OutOfMemoryError.

해결 방법:
1. JVM 힙 크기 증가: java -Xmx2048m MyClass
2. 코드에서 메모리 누수 확인
3. 메모리 프로파일링 도구(VisualVM, JProfiler 등)를 사용하여 문제 식별
4. 데이터 구조를 최적화하여 메모리 사용량 줄이기

## 일반적인 개발 문제

### 문제 7: NullPointerException

문제: NullPointerException은 가장 일반적인 런타임 예외입니다.

해결 방법:
1. 사용하기 전에 객체가 null인지 확인
2. Optional 클래스를 사용하여 null 포인터 방지
3. Objects.requireNonNull()을 사용하여 매개변수 검증
4. 필요한 모든 객체 초기화

### 문제 8: ConcurrentModificationException

문제: 반복 중에 컬렉션을 수정할 때 예외가 발생합니다.

해결 방법:
1. 반복자의 remove() 메서드를 사용하여 요소 삭제
2. ArrayList 대신 CopyOnWriteArrayList 사용
3. 먼저 삭제할 요소를 기록한 후 반복 후 삭제
4. Stream API의 filter를 사용하여 요소 필터링

### 문제 9: ClassCastException

문제: ClassCastException이 발생합니다.

해결 방법:
1. instanceof를 사용하여 객체 타입 확인
2. 구체적인 구현보다 인터페이스 타입 선호
3. 캐스팅 전에 타입 확인

### 문제 10: 숫자 정밀도 문제

문제: 부동소수점 계산이 정확하지 않습니다.

해결 방법:
1. 정밀한 계산을 위해 double 대신 BigDecimal 사용
2. 부동소수점 정밀도 제한 이해
3. 부동소수점 비교 피하기, BigDecimal의 compareTo 사용

## 성능 문제

### 문제 11: 프로그램 실행

문제: Java 프로그램이 느리게 실행됩니다.

해결 방법:
1. 성능 프로파일링 도구를 사용하여 병목 지점 찾기
2. 루프에서 반복 계산 피하기
3. 문자열 연결 대신 StringBuilder 사용
4. 적절한 컬렉션 타입 선택
5. JIT 컴파일러 최적화 활성화

### 문제 12: 데드락

문제: 프로그램이 데드락되어 실행을 계속할 수 없습니다.

해결 방법:
1. 중첩된 잠금 피하기
2. 잠금 획득 순서 통일
3. tryLock() 타임아웃 메커니즘 사용
4. synchronized 블록 범위 줄이기

## 빌드 도구 문제

### 문제 13: Maven/Gradle 종속성 충돌

문제: 빌드 중 종속성 버전 충돌이 발생합니다.

해결 방법:
1. mvn dependency:tree를 사용하여 종속성 트리 보기
2. pom/gradle에서 종속성 버전 명시적으로 지정
3. 이행적 종속성 제외
4. Maven Central의 Bom을 사용하여 버전 관리

### 문제 14: 빌드 실패

문제: Maven 또는 Gradle 빌드가 실패합니다.

해결 방법:
1. 오류 메시지를 보고 문제 위치 파악
2. 캐시 정리: mvn clean 또는 gradle clean
3. 네트워크 연결 확인
4. JDK 버전이 프로젝트 요구사항 충족하는지 확인
5. 문법 오류 확인

## 프레임워크 문제

### 문제 15: Spring Boot 시작 실패

문제: Spring Boot 애플리케이션이 시작할 수 없습니다.

해결 방법:
1. 시작 로그의 오류 메시지 확인
2. application.properties 또는 application.yml 구성 확인
3. 데이터베이스 연결 구성이 올바른지 확인
4. 종속성이 완전한지 확인
5. 포트가 사용 중인지 확인

### 문제 16: Hibernate LazyInitializationException

문제: 세션이 닫힌 후 지연 로딩된 객체에 접근할 때 발생합니다.

해결 방법:
1. eager 로딩에 @Fetch(FetchMode.JOIN) 사용
2. 세션이 닫히 전에 필요한 객체 초기화
3. 수동 로딩에 Hibernate.initialize() 사용
4. spring.jpa.open-in-view=true 구성

## 버전 선택 문제

### 문제 17: 어떤 Java 버전을 학습해야 하나요?

권장 사항:
- 초급: 현대적인 문법과 풍부한 리소스로 Java 21 또는 Java 17 권장
- 기업 프로젝트: 프로젝트 종속성에 따라 일반적으로 Java 8, 11 또는 17
- 새 프로젝트: Java 21 이상을 강력 권장

### 문제 18: JDK와 JRE의 차이

답변:
- JDK(Java Development Kit): 개발 키트로, JRE 및 컴파일/디버깅 도구 포함
- JRE(Java Runtime Environment): 런타임 환경으로, Java 프로그램 실행에 필요한 클래스 라이브러리 및 JVM만 포함

Java 프로그램을 개발하려면 JDK를 설치합니다. 컴파일된 Java 프로그램만 실행하려면 JRE로 충분합니다.

## 기타 일반적인 문제

### 문제 19: Java를 어떻게 학습하나요?

권장 사항:
1. Java 기본 문법 및 객체지향 개념 마스터
2. Java Collections Framework 및 일반 유틸리티 클래스 숙지
3. JDBC 데이터베이스 작업 학습
4. 최소 하나의 웹 프레임워크(Spring/Spring Boot) 마스터
5. 디자인 패턴 및 코드 리팩토링 이해
6. 더 많이 연습하고 프로젝트 작성

### 문제 20: Java가 얼마나 오랫동안 Relevants할까요?

분석: 기업 개발의 주류 언어인 Java는 성숙한 생태계와 큰 사용자 기반을 가지고 있습니다. TIOBE 순위에 따르면 Java는 Consistently 상위 3위에 랭크됩니다. Java는 계속 업데이트되며, 새 버전은 6개월마다, LTS 버전은 2년마다 출시됩니다. 따라서 Java는 예측 가능한 미래에 계속 가치 있는 기술이 될 것입니다.
