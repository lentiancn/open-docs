# FantasyTalking 常見問題

## 安裝問題

### 安裝失敗怎麼辦？

安裝過程中可能會遇到各種問題，首先請確認環境滿足要求。檢查 Node.js 版本是否在 14.0 以上，可以使用 `node --version` 命令查看。如果使用的是 Python 版本，確保 Python 版本在 3.7 以上。

如果安裝仍然失敗，可以嘗試以下方法：清除 npm 快取 `npm cache clean --force`；使用國內鏡像源 `npm config set registry https://registry.npmmirror.com`；或者從原始碼編譯安裝。

### FFmpeg 安裝問題

在某些系統上，FFmpeg 可能沒有正確安裝或配置。可以從 FFmpeg 官網下載預編譯的二進制檔案，或者使用套件管理器安裝：macOS 上使用 `brew install ffmpeg`，Linux 上使用 `apt-get install ffmpeg` 或 `yum install ffmpeg`，Windows 上可以從官網下載或使用 Chocolatey 安裝。

## 使用問題

### 語音合成速度慢怎麼辦？

語音合成的速度取決於多個因素。首先檢查網絡連接，如果使用雲端服務，網絡延遲會影響回應時間。其次可以考慮自建服務來減少網絡開銷。對於大批量合成任務，建議使用批量合成介面，可以顯著提高效率。還可以調整音訊參數，降低採樣率可以加快合成速度。

### 如何選擇合適的語音風格？

FantasyTalking 提供了多種語音風格，包括 neutral（中性）、happy（開心）、sad（悲傷）、angry（憤怒）、excited（興奮）等。選擇時需要考慮應用場景：客服場景推薦使用 neutral 或 professional；教育應用可以根據內容選擇不同風格；娛樂應用可以選擇更具表現力的風格。可以通過 A/B 測試來選擇最適合的風格。

### 如何實現英文語音合成？

預設配置下需要指定語言為英文：

```javascript
const audio = await client.speak('Hello, world!', {
  voice: {
    language: 'en-US',  // 或 en-GB
    style: 'neutral'
  }
});
```

支援的語言代碼包括 en-US、en-GB、en-AU 等。不同的語言代碼會使用對應地區的發音模型。

### SSML 標記不生效怎麼辦？

確保在呼叫 speak 函數時傳入 `ssml: true` 參數。檢查 SSML 標記是否正確，標記名稱應該是小寫。某些特殊字元需要進行 XML 轉義，例如小於號需要改為 `<`。如果問題仍然存在，可能是當前語言不支援某些 SSML 特性。

## 對話功能

### 對話上下文不連貫怎麼辦？

確保使用的是同一個會話實例，每次呼叫 `session.chat()` 都會自動包含之前的對話歷史。如果需要新的對話，建立新的會話實例。檢查是否達到了上下文窗口限制，預設保留 5 輪對話。可以通過修改配置來調整上下文窗口大小。

### 如何保存和恢復對話？

使用 `session.getHistory()` 取得對話歷史，返回的是 JSON 格式的數據，可以保存到檔案。要恢復對話，使用 `client.loadSession(history)` 方法載入歷史記錄。需要注意的是，載入的會話會從保存的狀態繼續對話。

## 音訊問題

### 生成的音訊有雜訊怎麼辦？

雜訊問題通常與音訊配置或網絡傳輸有關。檢查音訊格式和採樣率是否匹配目標播放設備。嘗試使用降噪功能處理音訊 `client.process(audio, { denoise: true })`。如果是網絡傳輸問題，考慮使用本地部署的服務。

### 如何調整語速和音調？

在語音配置中調整相關參數：

```javascript
const audio = await client.speak('文字', {
  voice: {
    speed: 1.5,  // 1.5 倍速，範圍 0.5-2.0
    pitch: 0.8  // 降低音調，範圍 0.5-2.0
  }
});
```

需要注意的是，過度調整參數可能會影響語音的自然度。

## 帳號和計費

### 如何取得 API 密鑰？

訪問 FantasyTalking 官網註冊帳號，進入控制台建立 API 密鑰。新用戶通常會獲得一定的免費額度用於測試。API 密鑰應該妥善保管，不要在客戶端代碼中硬編碼，建議使用環境變數存儲。

### 如何查看使用量？

登入控制台後，可以在儀表板查看 API 呼叫次數和消耗的配額。也可以使用 SDK 提供的方法查詢：

```javascript
const usage = await client.getUsage();
console.log(`本月使用次數: ${usage.count}`);
console.log(`剩餘免費次數: ${usage.remaining}`);
console.log(`當前計費週期: ${usage.period}`);
```

## 部署問題

### Docker 部署失敗怎麼辦？

首先確認 Docker 已正確安裝並執行。檢查連接埠 8080 是否被佔用，可以更改映射連接埠。如果使用雲端服務，確保安全組已開放相應連接埠。查看容器日誌 `docker logs fantasy-talking` 取得詳細錯誤資訊。

### 如何實現高可用部署？

對於生產環境，建議使用負載均衡器配合多個 FantasyTalking 實例。會話數據需要使用外部存儲（如 Redis）來實現多實例共享。可以使用 Docker Swarm 或 Kubernetes 進行容器編排，實現自動擴縮容和故障恢復。

## 開發和調試

### 如何開啟調試模式？

在初始化客戶端時設定 debug 選項：

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  debug: true  // 開啟調試模式
});
```

調試模式下會輸出詳細的請求和回應資訊，便於排查問題。

### 如何取得詳細的錯誤資訊？

所有 SDK 方法都會在失敗時拋出包含詳細資訊的錯誤物件：

```javascript
try {
  await client.speak('text');
} catch (error) {
  console.log('錯誤代碼:', error.code);
  console.log('錯誤訊息:', error.message);
  console.log('HTTP 狀態:', error.status);
  console.log('請求 ID:', error.requestId);  // 用於向技術支援回饋
}
```

## 其他問題

### FantasyTalking 支援哪些平台？

FantasyTalking 支援主流的平台和框架：Web 端支援所有現代瀏覽器，行動端支援 iOS 和 Android，桌面端支援 Windows、macOS 和 Linux，服務端支援 Node.js 和 Python。
