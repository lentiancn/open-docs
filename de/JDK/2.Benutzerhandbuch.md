# JDK-Benutzerhandbuch

Diese Anleitung behandelt die Verwendung von JDK für die Java-Entwicklung.

## Schnellstart

### Kompilieren und Ausführen

```bash
# Kompilieren
javac HelloWorld.java

# Ausführen
java HelloWorld
```

### Einzeldatenausführung (JDK 11+)

```bash
# Direkt ausführen
java HelloWorld.java
```

## Grundbefehle

### javac - Java-Compiler

```bash
# Einzelne Datei kompilieren
javac MyClass.java

# Mehrere Dateien kompilieren
javac *.java

# Mit bestimmter Kodierung kompilieren
javac -encoding UTF-8 MyClass.java

# In bestimmtes Verzeichnis kompilieren
javac -d bin MyClass.java

# Alle Warnungen anzeigen
javac -Xlint MyClass.java
```

### java - Java-Launcher

```bash
# Klasse ausführen
java MyClass

# Mit Argumenten ausführen
java MyClass arg1 arg2

# Classpath setzen
java -classpath "lib/*:." MyClass

# Speicher setzen
java -Xmx2g -Xms512m MyClass
```

### jar - Java-Archiv

```bash
# JAR erstellen
jar cf myapp.jar -C bin .

# JAR mit Manifest erstellen
jar cfm myapp.jar manifest.txt -C bin .

# JAR entpacken
jar xf myapp.jar

# JAR-Inhalt auflisten
jar tf myapp.jar
```

### javadoc - Dokumentationsgenerator

```bash
# Dokumentation generieren
javadoc -d docs -sourcepath src *.java

# Dokumentation mit benutzerdefiniertem Titel generieren
javadoc -d docs -header "My App" *.java
```

### jdb - Java-Debugger

```bash
# Debugger starten
jdb MyClass

# Haltepunkte setzen
stop at MyClass:10

# Ausführen
run

# Durchschreiten
next
step
```

## JVM-Optionen

### Speicheroptionen

```bash
# Heap-Größe
-Xmx2g        # Maximaler Heap
-Xms512m      # Initialer Heap
-Xss256k      # Stack-Größe

# Garbage Collection
-XX:+UseG1GC  # G1 Garbage Collector
-XX:+UseParallelGC # Parallel GC
```

### Leistungsoptionen

```bash
# JIT-Compiler
-XX:+TieredCompilation

# Klassendatenfreigabe
-Xshare:on

# String-Deduplizierung
-XX:+UseStringDeduplication
```

## IDE-Integration

### IntelliJ IDEA

1. File → Project Structure → SDKs
2. JDK-Installationspfad hinzufügen
3. Project SDK festlegen

### VS Code

1. "Extension Pack for Java" installieren
2. JAVA_HOME in Einstellungen konfigurieren

### Eclipse

1. Window → Preferences → Java → Installed JREs
2. Standard-VM hinzufügen
3. Als Standard festlegen

## Build-Tools

### Maven

```xml
    <maven.compiler.source>21</maven.compiler.source>
    <maven.compiler.target>21</maven.compiler.target>
</properties>
```

### Gradle

```groovy
sourceCompatibility = '21'
targetCompatibility = '21'
```

## Module (Java 9+)

### Module auflisten

```bash
java --list-modules
```

### Moduloptionen

```bash
# Paket exportieren
--export-module java.base/java.lang

# Modul öffnen
--open-module java.base
```

## Sicherheit

### Keytool

```bash
# Schlüsselpaar generieren
keytool -genkeypair -alias mykey -keyalg RSA -keystore keystore.jks

# Zertifikate auflisten
keytool -list -keystore keystore.jks

# Zertifikat exportieren
keytool -export -alias mykey -file cert.crt -keystore keystore.jks
```

## Profiling

### jcmd

```bash
# Laufende Java-Prozesse auflisten
jcmd

# Prozessinfo abrufen
jcmd <pid> VM.flags

# GC auslösen
jcmd <pid> GC.run
```

### jvisualvm

```bash
# Visuelle VM starten
jvisualvm
```

## Best Practices

### 1. LTS für Produktion verwenden

```bash
# Empfohlen: JDK 17 oder 21 für Produktion
```

### 2. Angemessenen Speicher setzen

```bash
# Für Web-Anwendungen
java -Xmx4g -Xms2g -XX:+UseG1GC MyApp
```

### 3. GC-Protokollierung aktivieren

```bash
# GC-Protokollierung aktivieren
-Xlog:gc*:file=gc.log:time,uptime,level,tags
```
