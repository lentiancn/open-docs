# Hadoop 使用手冊

## HDFS 操作指南

### 基本檔案系統操作

HDFS 提供了類似於傳統檔案系統的命令列介面，使用者可以透過 hdfs dfs 命令進行各種檔案操作。這些命令與 Linux 的 shell 命令非常相似，熟悉 Linux 系統的使用者可以快速上手。理解這些基本操作是使用 Hadoop 的基礎，下面詳細介紹常用的 HDFS 操作命令。

建立目錄使用 mkdir 命令，可以建立單層或多層目錄：

```bash
# 建立單層目錄
hdfs dfs -mkdir /user/data

# 建立多層目錄
hdfs dfs -mkdir -p /user/data/input

# 查看目錄列表
hdfs dfs -ls /user/data

# 遞迴查看目錄列表
hdfs dfs -ls -R /user/data
```

檔案上傳和下載是常用的操作：

```bash
# 上傳本地檔案到 HDFS
hdfs dfs -put localfile.txt /user/data/

# 上傳並重新命名
hdfs dfs -put localfile.txt /user/data/newname.txt

# 下載 HDFS 檔案到本地
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# 複製 HDFS 內部檔案
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# 移動 HDFS 內部檔案
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

檔案查看和刪除：

```bash
# 查看檔案內容
hdfs dfs -cat /user/data/file.txt

# 查看檔案最後幾行
hdfs dfs -tail /user/data/file.txt

# 查看檔案大小
hdfs dfs -du /user/data/file.txt

# 刪除檔案
hdfs dfs -rm /user/data/file.txt

# 遞迴刪除目錄
hdfs dfs -rm -r /user/data/dir
```

### 權限管理

HDFS 的權限模型與 Unix/Linux 類似，基於使用者、使用者組和其他人的三層權限體系。每個檔案和目錄都有讀（r）、寫（w）、執行（x）三種權限，分別對應所有者、使用者組和其他使用者。理解權限管理對於保護資料安全至關重要。

```bash
# 修改檔案所有者
hdfs dfs -chown user:group /user/data/file.txt

# 修改檔案權限
hdfs dfs -chmod 755 /user/data/file.txt

# 修改所屬組
hdfs dfs -chgrp groupname /user/data/file.txt
```

### 資料複製與均衡

HDFS 預設會將每個資料塊複製三份，儲存在不同的 DataNode 上。可以透過 dfs.replication 屬性修改預設副本數：

```bash
# 設定特定檔案的副本數
hdfs dfs -setrep -w 2 /user/data/file.txt

# 查看檔案的塊位置資訊
hdfs fsck /user/data/file.txt -files -blocks -locations
```

當叢集新增節點或節點下線時，資料可能分布不均，此時可以使用 rebalance 命令進行資料均衡：

```bash
start-balancer.sh -threshold 10
```

threshold 參數表示節點磁碟使用率與叢集平均水準的最大差異百分比。均衡過程會佔用一定的網路頻寬，建議在業務低峰期執行。

### HDFS Web UI

HDFS 提供了功能豐富的 Web 介面，可以通過瀏覽器存取 NameNode 的 9870 連接埠。在 Web UI 上可以瀏覽檔案系統目錄、查看檔案內容、監控 DataNode 狀態、查看資料塊資訊等。Web UI 是運維人員日常監控和排查問題的重要工具。

Web UI 的主要功能包括：瀏覽 HDFS 目錄樹，查看檔案列表和檔案屬性；查看 DataNode 列表和每個 DataNode 的儲存使用情況；查看資料塊的分布和複製狀態；查看 NameNode 的執行時資訊和日誌；進行簡單的檔案操作如下載和刪除。

## MapReduce 程式設計

### WordCount 範例

WordCount 是 MapReduce 程式設計的入門範例，統計文字中每個單詞出現的次數。理解這個範例有助於掌握 MapReduce 程式設計模型。程式分為 Map 階段和 Reduce 階段，Map 階段將文字拆分單詞，Reduce 階段對相同單詞的計數進行匯總。

Map 階段程式碼範例：

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduce 階段程式碼範例：

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### 提交作業

將 MapReduce 打包成 JAR 檔案後，使用 hadoop jar 命令提交作業：

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

作業提交後，可以在 YARN 的 Web UI（連接埠 8088）上查看作業進度和結果。作業完成時，會輸出作業的統計資訊，包括作業ID、任務數量、執行時間等。

### 作業配置

可以透過 Configuration 物件配置作業的各種參數：

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

常用的配置參數包括：mapreduce.job.reduces 設定 Reduce 任務數量；mapreduce.map.memory.mb 和 mapreduce.reduce.memory.mb 設定容器記憶體；mapreduce.map.cpu.vcores 和 mapreduce.reduce.cpu.vcores 設定容器 CPU 核心數。

## YARN 操作

### 查看叢集資源

YARN 提供了豐富的命令列工具來管理叢集資源：

```bash
# 查看所有節點狀態
yarn node -list -all

# 查看應用程式列表
yarn application -list

# 查看特定應用程式狀態
yarn application -status application_1234567890123_0001

# 查看應用程式日誌
yarn logs -applicationId application_1234567890123_0001
```

### 佇列管理

YARN 使用佇列來管理資源分配。可以透過 Capacity Scheduler 或 Fair Scheduler 配置佇列：

```bash
# 查看佇列資訊
yarn queue -status default

# 查看佇列容量
yarn resourcemanager -refreshQueues
```

### 資源調度配置

Capacity Scheduler 是預設的調度器，支援多租戶資源共享。可以在 yarn-site.xml 中進行配置：

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## 效能優化

### MapReduce 調優

MapReduce 作業的效能調優涉及多個方面。首先是合理設定 Map 和 Reduce 任務數量。Map 任務數量通常由輸入資料的分片數量決定，可以透過 mapreduce.input.fileinputformat.split.maxsize 參數調整。Reduce 任務數量建議設定為叢集 Reduce 槽位總數的 0.9 到 1.2 倍。

```bash
# 設定 Map 任務數量
-D mapreduce.job.maps=10

# 設定 Reduce 任務數量
-D mapreduce.job.reduces=5
```

Shuffle 階段的優化對整體效能影響很大。可以調整以下參數：

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFS 調優

HDFS 的效能優化主要關注資料讀寫效率。對於大型檔案，可以調整塊大小來平衡資料本地性和故障恢復：

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

啟用短路讀取可以讓客戶端直接讀取本地 DataNode 的資料，減少網路傳輸：

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARN 調優

YARN 的資源分配對叢集利用率有重要影響。合理設定 NodeManager 的資源容量：

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

設定虛擬 CPU 核心數以更好地利用多核處理器：

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## 資料匯入匯出

### 從資料庫匯入資料

使用 Sqoop 可以方便地在關係型資料庫和 HDFS 之間遷移資料：

```bash
# 匯入整個表格
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# 匯入並建立 Hive 表格
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### 匯出資料到資料庫

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### 資料格式選擇

不同的資料格式對效能有不同的影響。常用的格式包括：TextFile（通用性好，但效能一般）；SequenceFile（適合 MapReduce，效能好）；Parquet（列式儲存，適合分析查詢）；ORC（Hive 優化列式儲存）。對於分析型工作負載，建議使用列式儲存格式如 Parquet 或 ORC。
