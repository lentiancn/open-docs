# Nacos 簡介

## 什麼是 Nacos

Nacos /nɑ:kəʊs/ 是 Dynamic Naming and Configuration Service 的首字母簡稱，一個易於構建雲端原生應用的動態服務發現、配置管理和平臺。

Nacos 致力於幫助您發現、配置和管理微服務及 AI 智慧代理應用。Nacos 提供了一組簡單易用的特性集，幫助您快速實現動態服務發現、服務配置、服務元資料、流量管理。

Nacos 説明您更敏捷和容易地構建、交付和管理微服務平臺。Nacos 是構建以「服務」為中心的現代應用架構（例如微服務範式、雲端原生範式）的服務基礎設施。

Nacos 支援幾乎所有主流類型的「服務」的發現、配置和管理：

- Kubernetes Service
- gRPC
- Dubbo RPC Service
- Spring Cloud RESTful Service
- Model Context Protocol

## 核心功能

### 服務發現和健康監測

Nacos 支援基於 DNS 和基於 RPC 的服務發現。服務提供者使用原生 SDK、OpenAPI 或獨立的 Agent 註冊 Service 後，服務消費者可以使用 DNS 或 HTTP API 查找和發現服務。

Nacos 提供對服務的即時健康檢查，阻止向不健康的主機或服務實體發送請求。Nacos 支援傳輸層（PING 或 TCP）和應用層（如 HTTP、MySQL、使用者自訂）的健康檢查。對於複雜的雲端環境和網路拓撲環境中（如 VPC、邊緣網路等）服務的健康檢查，Nacos 提供了 agent 上報模式和服務端主動檢測兩種健康檢查模式。Nacos 還提供了統一的健康檢查儀表板，説明您根據健康狀態管理服務的可用性及流量。

### 動態配置服務

動態配置服務可以讓您以集中化、外部化和動態化的方式管理所有環境的應用配置和服務配置。動態配置消除了配置變更時重新部署應用和服務的需要，讓配置管理變得更加高效和敏捷。

集中化管理讓實現無狀態服務變得更簡單，讓服務按需彈性擴展變得更容易。Nacos 提供了一個簡潔易用的 UI 説明您管理所有的服務和應用的配置。Nacos 還提供包括配置版本追蹤、金絲雀發布、一鍵回滾配置以及用戶端配置更新狀態追蹤在內的一系列開箱即用的配置管理特性，説明您更安全地在生產環境中管理配置變更和降低配置變更帶來的風險。

### 動態 DNS 服務

動態 DNS 服務支援權重路由，讓您更容易地實現中層負載均衡、更靈活的路由策略、流量控制以及資料中心內網的簡單 DNS 解析服務。動態 DNS 服務還能讓您更容易地實現以 DNS 協定為基礎的服務發現，以幫助您消除耦合到廠商私有服務發現 API 上的風險。

### 服務及其元資料管理

Nacos 能讓您從微服務平臺建設的視角管理資料中心的所有服務及元資料，包括管理服務的描述、生命週期服務的靜態依賴分析、服務的健康狀態、服務的流量管理、路由及安全策略、服務的 SLA 以及最首要的 metrics 統計資料。

### MCP 服務管理和轉化

Nacos 能讓您快速的將存量的微服務 API，透過簡單的聲明方式，快速實現零程式碼改動转化成 MCP 服務。另外也能使用 Spring AI Alibaba、nacos-mcp-wrapper-python 及 Nacos-MCP-Router 進行 MCP 服務的自動註冊及發現，就像使用微服務一樣使用 MCP 服務。同時能夠從 MCP 服務平臺的視角，進行 MCP 服務相關描述、工具、Prompt 等內容的管理，並對 MCP 服務的版本進行管理。

## 產品優勢

### 易於使用

Nacos 經過數萬人使用回饋優化，提供統一的服務發現和配置管理功能，透過直觀的 Web 介面和簡潔的 API，為開發和運維人員在雲端原生環境中帶來了便捷的服務註冊、發現和配置更新操作。面向 AI、雲端原生應用的動態配置管理、服務發現的一站式解決方案，二十多種開箱即用的以 AI 服務為中心的架構特性，輕量級易用管控介面，快速構建 AI 應用。

### 特性豐富

Nacos 提供了包括服務發現、配置管理、動態 DNS 服務、服務元資料管理、流量管理、服務監控、服務治理等在內的一系列特性，説明您在雲端原生時代更輕鬆地構建、交付和管理微服務。支援 Prompt、MCP、A2A 等 AI Registry 場景。直接支援或稍作擴展即可支援大量有用的互聯網、AI 應用場景流量調度和服務治理，支援限流、大促銷預案和異地多活。

### 極致效能

Nacos 經過阿里雙十一超快伸縮場景的錘煉，提供高效能的服務註冊和發現能力，以及低延遲的配置更新回應，確保在大規模分散式系統中的效率和穩定運行。

### 超大容量

Nacos 誕生自阿里的百萬實例規模，造就支援海量服務和配置的管理，能夠滿足大型分散式系統對高並發和高可用性的需求。

### 穩定可用

脱胎於歷經阿里巴巴十年生產驗證的內部產品，支援具有數百萬服務的大規模場景，具備企業級 SLA 的開源產品，上千萬級別使用者使用。Nacos 透過自研的同步協定，配合生態中應用廣泛的 Raft 協定，確保了服務的高可用性和資料的穩定性，保證阿里雙十一系統的高可用穩定運行。

### 開放生態

Nacos 擁有活躍的開源社區廣泛的生態整合和持續的創新發展，不僅大量相容了 Spring Cloud、Dubbo 等大受歡迎的開源框架、還提供了豐富的插件化能力，説明使用者在雲端原生時代提供可定制滿足自身特殊需求的獨有雲端原生微服務系統。無縫支援 Kubernetes、Spring Cloud、Spring AI、Dify，在主流公共雲上更容易部署和運行（例如阿里雲、華為雲、騰訊雲、Azure 和 AWS），支援多租戶和多環境支援。

## 設計理念

Nacos 致力於説明使用者在 AI 時代，在私有雲、混合雲或者公有雲等所有雲環境中更好的構建、交付、管理自己的 AI 管理平臺，更快的復用和組合業務服務，更快的交付商業創新的價值。Nacos 的設計理念被定位為：

### 易於使用

透過提供使用者友好的 Web 介面和簡潔的 API 來簡化服務的註冊、發現和配置管理過程。開發者可以輕鬆整合 Nacos 到他們的應用中，無需投入大量時間在複雜的設定和學習上。

### 面向標準

遵循雲端原生應用開發的最佳實踐和標準協定，以確保其服務發現和配置管理功能與廣泛的技術棧和平臺無縫對接。

### 高可用

實現了叢集模式，確保在節點發生故障時，服務的發現和配置管理功能不會受影響。叢集模式也意味著 Nacos 可以透過增加節點來水準擴展，提升系統的整體性能和承載能力。

### 方便擴展

採用模組化的設計使得各個元件都可以獨立地進行擴展或替換。這也為社區貢獻者提供了方便，使他們能夠針對特定的需求開發新的功能或者改善現有功能，進一步推動 Nacos 的生態發展。

透過上述設計理念的實現，Nacos 為使用者提供了一個強大而靈活的平臺，以支援不斷變化的業務需求，加速業務創新和數位轉型，最終幫助使用者在競爭激烈的市場中佔據有利地位。

## 部署模式

### 單機模式

單機模式又稱單例模式，擁有所有 Nacos 的功能及特性，具有極易部署、快速啟動等優點。但無法與其他節點組成叢集，無法在節點或網路故障時提供高可用能力。單機模式同樣可以使用內嵌 Derby 資料庫（預設）和外置資料庫進行存儲。單機模式主要適合工程師於本地搭建或於測試環境中搭建 Nacos 環境，主要用於開發調試及測試使用；也能夠兼顧部分對穩定性和可用性要求不高的業務場景。

### 叢集模式

叢集模式透過自研一致性協定 Distro 以及 Raft 協定，將多個 Nacos 節點構建成了高可用的 Nacos 叢集。資料將在叢集中各個節點進行同步，保證資料的一致性。叢集模式具有高可用、高擴展、高並發等優點，確保在故障發生時不影響業務的運行。叢集模式預設採用外置資料庫進行存儲，但也可以透過內嵌資料庫進行存儲。該模式主要適合於生產環境，也是社區最為推薦的部署模式。

## 適用場景

Nacos 適用於多種應用場景：

- 微服務架構中的服務註冊與發現
- 分散式系統中的配置管理
- 多環境配置管理（開發、測試、生產）
- 服務流量管理與路由
- 雲端原生應用的基礎設施
- AI 應用的模型服務管理（MCP）
