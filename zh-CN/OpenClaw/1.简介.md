# OpenClaw 简介

## 什么是 OpenClaw

OpenClaw 是一个**自托管的网关**，它将你喜欢的聊天应用——WhatsApp、Telegram、Discord、iMessage 等——连接到 AI 编程助手（如 Pi）。你只需要在自己的电脑或服务器上运行一个 Gateway 进程，它就会成为你的聊天应用和随时可用的 AI 助手之间的桥梁。

简单来说，OpenClaw 可以让你通过微信、WhatsApp、Telegram、Discord 等常用聊天软件，直接和一个 AI 助手对话，就像发消息一样简单。

## 核心定位

- **个人 AI 助手网关**：让你随时随地通过聊天软件联系 AI
- **多通道支持**：一个 Gateway 同时支持多个聊天平台
- **自托管**：数据存在你自己的电脑上，安全可控
- **开源免费**：MIT 许可证，社区驱动

## 主要特性

### 1. 多通道支持

OpenClaw 支持同时连接多个聊天平台：

- **WhatsApp**：最常用的即时通讯软件
- **Telegram**：功能强大的开源聊天软件
- **Discord**：游戏社区和团队协作的首选
- **iMessage**：苹果生态用户的即时通讯
- **插件通道**：通过扩展包支持 Mattermost 等更多平台

### 2. 强大的管理界面

OpenClaw 提供了可视化的 Web 控制台：

- **控制台 UI**：浏览器访问 http://127.0.0.1:18789/
- **聊天功能**：直接在浏览器中与 AI 对话
- **配置管理**：可视化配置通道、路由等
- **会话管理**：查看和管理活跃的对话会话
- **节点管理**：管理配对的移动设备节点

### 3. 多代理路由

- **隔离会话**：每个代理、工作区或发送者都有独立的会话
- **多代理支持**：可以同时运行多个 AI 代理
- **智能路由**：根据条件自动选择合适的代理处理请求

### 4. 媒体支持

- **图片收发**：发送和接收图片
- **音频消息**：支持语音消息
- **文件传输**：可以发送和接收各类文件
- **移动节点**：支持 iOS 和 Android 设备配对

### 5. 安全控制

- **Token 管理**：配置 API 密钥访问权限
- **白名单**：可以限制只有特定用户可以访问
- **群组控制**：支持群聊中的 @提及规则
- **安全模式**：多种安全策略可选

## 工作原理

```
聊天应用 → Gateway → AI 代理/CLI/Web控制台/移动节点
```

Gateway 是会话、路由和通道连接的单一数据源。

## 系统要求

- **Node.js**：22 或更高版本
- **API 密钥**：推荐使用 Anthropic（Claude）的 API 密钥
- **时间**：5 分钟即可完成基础配置

## 适用人群

- **开发者**：想要一个可以随时随地编程辅助的 AI 助手
- **技术爱好者**：喜欢自托管，追求数据控制权
- **团队协作**：需要团队共享 AI 编程助手
- **移动办公**：想要在手机上使用 AI 助手

## 快速开始

只需三条命令即可开始使用：

```bash
# 1. 安装 OpenClaw
npm install -g openclaw@latest

# 2. 初始化并安装服务
openclaw onboard --install-daemon

# 3. 启动并使用
openclaw gateway --port 18789
```

然后打开浏览器访问 http://127.0.0.1:18789/ 即可开始使用。
