# Introduction

Apache APISIX is a top-level project under the Apache Software Foundation, developed and donated by API7.ai. It is a cloud-native API gateway known for its dynamic, real-time, and high-performance characteristics.

APISIX serves as the entry point for all business traffic, providing rich features including dynamic routing, dynamic upstreams, dynamic certificates, A/B testing, gray release (canary release), blue-green deployment, rate limiting, attack protection, metrics collection, monitoring alerts, observability, and service governance.

## Core Features

### High Performance

Built on Nginx, APISIX inherits Nginx's excellent performance. Through its plugin mechanism, business logic is decoupled from core functionality, making extensions more flexible while maintaining efficient processing capabilities.

### Fully Dynamic

Unlike traditional Nginx requiring configuration reloads, APISIX supports fully dynamic configuration management. All routes, upstreams, and plugins can be updated in real-time through the Admin API without service restarts.

### Cloud-Native Design

APISIX was designed with cloud-native scenarios from the start, supporting multiple deployment methods including Docker, Kubernetes, and Helm Chart. It also provides an Ingress Controller that can be used directly as a Kubernetes ingress gateway.

### Plugin Ecosystem

APISIX offers a rich plugin system covering identity authentication, rate limiting, request rewriting, response processing, monitoring, and logging. Users can easily extend gateway functionality through the plugin mechanism.

### Multi-Protocol Support

Beyond traditional HTTP/HTTPS protocols, APISIX also supports gRPC, Dubbo, WebSocket, MQTT, and other protocols, meeting various scenario requirements in microservices architectures.

## Architecture Overview

APISIX's architecture is divided into two parts: data plane and control plane.

The **data plane** handles actual request forwarding. The APISIX Core, based on modified Nginx, implements a high-performance request processing pipeline.

The **control plane** manages all configurations through RESTful APIs, supporting multiple configuration persistence methods including etcd and Consul. APISIX also provides a Dashboard for visual configuration management.

## Use Cases

- Unified entry point for microservices APIs
- Kubernetes Ingress Controller
- API traffic management and monitoring
- Multi-tenant API management
- Service mesh sidecar proxy
- AI application gateway (LLM and AI workloads)

As a top-level project of the Apache Software Foundation, APISIX has an active community and continuous version updates, providing a reliable choice for enterprise applications.
