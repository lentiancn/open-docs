# Mockito Installationsanleitung

Anleitung zur Installation und Konfiguration von Mockito.

## Voraussetzungen

- Java Development Kit (JDK) 8 oder höher
- Maven oder Gradle (empfohlen) oder einfache JAR-Datei

## Installation

### Maven

Fügen Sie die folgende Abhängigkeit in Ihre `pom.xml` ein:

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>5.8.0</version>
    <scope>test</scope>
</dependency>
```

### Gradle

Fügen Sie die Abhängigkeit in Ihre `build.gradle` ein:

```groovy
testImplementation 'org.mockito:mockito-core:5.8.0'
```

### JAR-Datei

Laden Sie die neueste JAR-Datei von Maven Central herunter:
https://repo1.maven.org/maven2/org/mockito/mockito-core/

## IntelliJ IDEA Integration

Mockito funktioniert out-of-the-box mit IntelliJ IDEA:

1. Öffnen Sie Ihr Projekt
2. Fügen Sie die Maven-Abhängigkeit hinzu
3. Mockito wird automatisch erkannt

## Eclipse Integration

1. Klicken Sie mit der rechten Maustaste auf Ihr Projekt
2. Wählen Sie "Build Path" > "Configure Build Path"
3. Fügen Sie die JAR-Datei unter "Libraries" > "Add External JARs" hinzu

## Konfiguration

### Mockito-Extension für JUnit 5

Für JUnit 5 (Jupiter) gibt es eine integrierte Unterstützung:

```java
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class)
class MyTest {
    // Ihre Tests hier
}
```

### Mockito-Annotationen aktivieren

Um Annotationen wie `@Mock`, `@Spy` zu verwenden:

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @BeforeEach
    void init() {
        MockitoAnnotations.openMocks(this);
    }
}
```

## Fehlerbehebung

### Problem: NoClassDefFoundError

**Lösung:** Stellen Sie sicher, dass alle Abhängigkeiten (ByteBuddy, Objenesis) enthalten sind.

### Problem: Mock funktioniert nicht

**Lösung:** 
- Prüfen Sie, ob die Klasse final ist
- Stellen Sie sicher, dass Sie Schnittstellen mocken
- Verwenden Sie `mockito-inline` für finale Klassen

### Problem: Verifizierung schlägt fehl

**Lösung:** 
- Überprüfen Sie die Aufrufreihenfolge
- Verwenden Sie `times(1)` für explizite Überprüfung

## Upgrade von Mockito 4.x

1. Aktualisieren Sie die Version in Ihrer Build-Datei
2. Prüfen Sie auf veraltete APIs
3. Führen Sie Ihre Tests aus
