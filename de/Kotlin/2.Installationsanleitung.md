# Kotlin-Installationsanleitung

## Voraussetzungen

Bevor Sie Kotlin installieren, stellen Sie sicher, dass Ihr Computer die grundlegenden Systemanforderungen erfüllt. Kotlin kann auf mehreren Betriebssystemen ausgeführt werden, darunter Windows, macOS, Linux und andere Unix-ähnliche Systeme. Unabhängig davon, welches Betriebssystem Sie verwenden, stellen Sie sicher, dass Sie genügend Speicherplatz für die Installation von Entwicklungstools und die Ausführung kompilierter Programme haben.

Für Windows-Benutzer wird Windows 10 oder höher für das beste Erlebnis empfohlen. macOS-Benutzer benötigen macOS 10.14 oder höher. Linux-Benutzer können jede wichtige Distribution wie Ubuntu, Fedora oder Debian verwenden.

## Installation über IDE

### IntelliJ IDEA

IntelliJ IDEA ist die Flaggschiff-IDE von JetBrains und bietet die umfassendste Unterstützung für Kotlin. Wenn Ihr Hauptziel die Kotlin-Entwicklung ist, ist IntelliJ IDEA die beste Wahl.

IntelliJ IDEA ist in zwei Versionen erhältlich: Community (kostenlos und Open Source) und Ultimate (für Unternehmensbenutzer mit erweiterten Funktionen). Die Community-Edition ist kostenlos und bietet ausreichend Funktionen für die Kotlin-Entwicklung. Die Ultimate-Edition bietet zusätzliche Unternehmensfunktionen wie erweiterte Datenbanktools und Unternehmenssicherheitsfunktionen.

Um zu installieren, besuchen Sie die offizielle JetBrains-Website (https://www.jetbrains.com/idea/download/) und laden Sie das entsprechende Installationsprogramm für Ihr Betriebssystem herunter. Führen Sie nach dem Herunterladen das Installationsprogramm aus und folgen Sie dem Assistenten, um die Installation abzuschließen. Während der Installation können Sie den Installationspfad auswählen und Desktop-Verknüpfungen erstellen.

Wenn Sie IntelliJ IDEA zum ersten Mal starten, erkennt die IDE automatisch und fordert Sie zur Installation des Kotlin-Plugins auf. Falls das System nicht automatisch erkennt, können Sie im Plugin-Marktplatz manuell nach Kotlin suchen und es installieren. Starten Sie die IDE nach der Installation neu, um mit der Kotlin-Entwicklung zu beginnen.

### Android Studio

Android Studio ist die offizielle Android-Entwicklungs-IDE von Google mit integrierter Kotlin-Unterstützung. Wenn Sie hauptsächlich Android entwickeln, ist Android Studio die ideale Wahl.

Besuchen Sie die Android-Entwickler-Website (https://developer.android.com/studio), um Android Studio herunterzuladen. Führen Sie nach dem Herunterladen das Installationsprogramm aus, um die Installation abzuschließen. Android Studio enthält standardmäßig das Kotlin-Plugin, sodass keine zusätzliche Konfiguration erforderlich ist, um mit der Entwicklung zu beginnen.

Android Studio veröffentlicht jedes Jahr mehrere Aktualisierungsversionen, die jeweils die neueste Kotlin-Unterstützung und Android-Entwicklungstools enthalten. Es wird empfohlen, Ihre Version aktuell zu halten, um das beste Entwicklungserlebnis und die neuesten Sprachfunktionen zu erhalten.

## Installation über Befehlszeile

Zusätzlich zur Verwendung einer IDE können Sie auch den Kotlin-Befehlszeilen-Compiler installieren. Dieser Ansatz eignet sich für Entwickler, die eine leichtgewichtige Entwicklungsumgebung bevorzugen oder Kotlin für die serverseitige Entwicklung benötigen.

### Manuelle Installation

Besuchen Sie zunächst die Kotlin-GitHub-Releases-Seite (https://github.com/JetBrains/kotlin/releases), um die neueste Version des Compilers herunterzuladen. Der Download ist ein Zip-Archiv; entpacken Sie es in ein gewünschtes Verzeichnis.

Fügen Sie nach dem Entpacken das Verzeichnis kotlinc/bin zur PATH-Umgebungsvariable Ihres Systems hinzu, damit Sie den Befehl kotlinc von der Befehlszeile aus verwenden können. Unter Windows fügen Sie den Pfad über Systemeigenschaften > Umgebungsvariablen hinzu. Unter macOS und Linux bearbeiten Sie Ihre ~/.bashrc- oder ~/.zshrc-Datei und fügen eine Zeile hinzu:

export PATH="$PATH:/path/to/kotlinc/bin"

Öffnen Sie nach dem Hinzufügen des Pfads ein neues Terminalfenster und geben Sie kotlinc -version ein. Wenn Versionsinformationen angezeigt werden, war die Installation erfolgreich.

### Verwendung von SDKMAN!

SDKMAN! ist ein Tool zur Verwaltung von Versionen von Java-Entwicklungstools und funktioniert bequem auf Unix-ähnlichen Systemen. Wenn Sie macOS, Linux, Cygwin, FreeBSD oder Solaris verwenden, ist SDKMAN! die empfohlene Methode zur Installation von Kotlin.

Installieren Sie zunächst SDKMAN!, indem Sie diesen Befehl in Ihrem Terminal ausführen:

curl -s "https://get.sdkman.io" | bash

Laden Sie nach der Installation die Umgebungskonfiguration neu:

source ~/.sdkman/bin/sdkman-init.sh

Jetzt können Sie den sdk-Befehl verwenden, um Kotlin zu installieren:

sdk install kotlin

Während der Installation werden Sie aufgefordert, die Version zu bestätigen. Nach der Bestätigung ist die Installation abgeschlossen. Sie können die Installation mit kotlinc -version überprüfen.

### Verwendung von Homebrew

Unter macOS können Sie auch den Homebrew-Paketmanager verwenden, um Kotlin zu installieren. Stellen Sie zunächst sicher, dass Homebrew installiert ist, und führen Sie dann aus:

brew update
brew install kotlin

Homebrew lädt automatisch den neuesten Kotlin-Compiler herunter und installiert ihn. Überprüfen Sie die Installation mit kotlinc -version.

### Verwendung von Snap

Unter Ubuntu 16.04 und höher können Sie Snap verwenden, um Kotlin zu installieren:

sudo snap install --classic kotlin

Die Snap-Installationsmethode installiert Kotlin als systemweite Anwendung, die von überall über die Befehlszeile aufgerufen werden kann.

## Erstellen Ihres ersten Kotlin-Programms

Unabhängig von der gewählten Installationsmethode können Sie Ihr erstes Kotlin-Programm mit diesen Schritten erstellen und ausführen.

Erstellen Sie zunächst eine Datei namens hello.kt mit folgendem Inhalt:

fun main() {
    println("Hello, World!")
}

Dies ist das einfachste Kotlin-Programm, das eine main-Funktion definiert, die der Einstiegspunkt eines Kotlin-Programms ist. Die Funktion println gibt Text auf der Konsole aus.

Wenn Sie den Befehlszeilen-Compiler verwenden, erfordert das Kompilieren und Ausführen dieses Programms zwei Schritte. Zuerst kompilieren:

kotlinc hello.kt -include-runtime -d hello.jar

Der Parameter -include-runtime packt die Kotlin-Laufzeitbibliothek in die generierte Jar-Datei und macht sie eigenständig. Der Parameter -d gibt den Pfad für die Ausgabedatei an.

Führen Sie nach dem Kompilieren das Programm aus:

java -jar hello.jar

Sie sollten "Hello, World!" auf der Konsole sehen.

Wenn Sie keine eigenständige Jar-Datei generieren möchten, können Sie auch die interaktive Ausführungsfunktion von kotlinc verwenden:

kotlinc -script hello.kt

Oder verwenden Sie den kotlin-Befehl, um kompilierte Klassendateien direkt auszuführen:

kotlin -classpath hello.jar HelloKt

Hier ist HelloKt der Name der Hauptklasse, die der Kotlin-Compiler für die Datei hello.kt generiert hat.

## Verwenden des REPL

Kotlin bietet eine interaktive REPL-Umgebung (Read-Eval-Print Loop), in der Sie Kotlin-Code eingeben und sofort Ergebnisse sehen können. Dies ist nützlich zum Erlernen von Sprachfunktionen oder zum schnellen Testen von Code-Schnipseln.

Um REPL zu starten, führen Sie einfach kotlinc ohne Parameter aus:

kotlinc

Dies wechselt in den REPL-Modus. Sie können beliebigen gültigen Kotlin-Code eingeben und Enter drücken, um das Ergebnis zu sehen:

>>> val x = 10
>>> val y = 20
>>> x + y
30

Geben Sie :quit ein oder drücken Sie Strg+C, um REPL zu beenden.

## Kotlin-Skripte

Kotlin kann auch als Skriptsprache verwendet werden. Kotlin-Skriptdateien haben die Erweiterung .kts, und Code auf oberster Ebene in der Datei wird als ausführbarer Code behandelt.

Erstellen Sie beispielsweise ein Skript namens list_folders.kts:

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

Führen Sie das Skript aus:

kotlinc -script list_folders.kts -- -d /path/to/folder

## Konfigurieren von Build-Tools

In realen Projekten werden typischerweise Gradle oder Maven als Build-Tools verwendet. Beide unterstützen Kotlin-Projekte nativ.

### Gradle

Gradle ist das beliebteste Build-Tool für die Kotlin-Entwicklung. Das Kotlin-Team bietet auch Kotlin DSL, mit dem Build-Skripte in Kotlin-Syntax geschrieben werden können.

Um ein grundlegendes Kotlin-Gradle-Projekt zu erstellen, fügen Sie zunächst das Kotlin-Plugin zu Ihrem Projekt hinzu. Ihre build.gradle-Datei muss diese Inhalte enthalten:

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

Oder verwenden Sie Kotlin DSL:

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

Erstellen Sie dann die Quellcodeverzeichnisstruktur, typischerweise src/main/kotlin. Fügen Sie Kotlin-Quelldateien hinzu und verwenden Sie gradle build, um das Projekt zu kompilieren.

### Maven

Maven ist ein weiteres beliebtes Build-Tool für Java-Projekte, das auch Kotlin-Projekte unterstützt. Fügen Sie Kotlin-Abhängigkeiten in Ihrer pom.xml-Datei hinzu.

## Überprüfen der Installation

Nach der Installation können Sie überprüfen, ob alles korrekt funktioniert.

Überprüfen Sie zunächst den Kotlin-Compiler:

kotlinc -version

Sie sollten eine Ausgabe wie "Kotlin compiler version 2.3.10" sehen.

Überprüfen Sie dann die Kotlin-Standardbibliothek:

kotlin -e 'println("Hello Kotlin!")'

Wenn alles funktioniert, war die Installation erfolgreich und Sie können Ihre Kotlin-Reise beginnen.

## Nächste Schritte

Nach der Installation von Kotlin wird empfohlen, das Lernen fortzusetzen, indem Sie zunächst die offizielle Kotlin-Tour lesen, um die grundlegenden Sprachmerkmale zu verstehen; dann eine Entwicklungsrichtung wählen, die Sie interessiert, wie Android-Entwicklung, serverseitige Entwicklung oder plattformübergreifende Entwicklung; und schließlich mit realen Projekten üben, um das Gelernte anzuwenden.

Kotlin bietet umfangreiche Lernressourcen, darunter offizielle Dokumentation, Online-Tutorials und Videokurse. Sie können Links zu diesen Ressourcen auf der Kotlin-Website finden.
