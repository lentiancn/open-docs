# JUnit Häufig gestellte Fragen

## Sollte ich JUnit 4 oder JUnit 5 verwenden?

Für neue Projekte wird JUnit 5 stark empfohlen. Es bietet reichhaltigere Funktionen, einschließlich verschachtelter Tests, parametrisierter Tests, dynamischer Tests und besserer Erweiterbarkeit. Für bestehende JUnit 4-Projekte können Sie JUnit Vintage verwenden, um Tests auf der JUnit 5-Plattform auszuführen und schrittweise zu migrieren.

## Wie führe ich JUnit 4-Tests in JUnit 5 aus?

Fügen Sie die JUnit Vintage Engine-Abhängigkeit hinzu:

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## Warum werden meine Tests nicht ausgeführt?

Häufige Gründe: Testklasse hat keine @Test-Annotation, Testmethoden sind nicht public, maven-surefire-plugin fehlt oder useJUnitPlatform() nicht konfiguriert, Java-Version erfüllt die Anforderungen nicht.

## Was ist die Mindest-Java-Version für JUnit 5?

JUnit 5 erfordert Java 8 oder höher. JUnit 4 erfordert Java 5 oder höher.

## Wie überspringe ich bestimmte Tests?

Verwenden Sie die @Disabled-Annotation, um einzelne Tests oder ganze Testklassen zu überspringen:

```java
@Disabled("Vorübergehend deaktiviert")
@Test
void testSkipped() {
    // Wird nicht ausgeführt
}
```

## Wie teste ich private Methoden?

Unit-Tests sollten nur public-Methoden testen. Wenn Sie private Methoden testen müssen, refaktorieren Sie den Code, um die zu testende Logik in separate public-Methoden zu verschieben, oder verwenden Sie Reflection (nicht empfohlen).

## Wie erhalte ich detaillierte Informationen bei fehlgeschlagenen Assertions?

Verwenden Sie den dritten Parameter für benutzerdefinierte Fehlermeldungen:

```java
assertEquals(expected, actual, "Berechnungsergebnis ist falsch");
```

## Wie teste ich Exceptions?

Verwenden Sie die assertThrows-Methode:

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("Fehlermeldung");
    });
}
```

## Wie führe ich Tests parallel aus?

Konfigurieren Sie die parallele Ausführung in JUnit 5:

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // Tests werden parallel ausgeführt
}
```

## Was ist der Unterschied zwischen @Before und @BeforeEach?

@Before ist eine JUnit 4-Annotation, @BeforeEach ist eine JUnit 5-Annotation. Beide erfüllen denselben Zweck: Sie werden vor jedem Testmethodenaufruf ausgeführt. JUnit 5 unterstützt auch @BeforeAll und @AfterAll für Setup-/Teardown-Operationen, die einmal für die gesamte Testklasse ausgeführt werden.

## Wie erstelle ich parametrisierte Tests?

Verwenden Sie die @ParameterizedTest-Annotation mit @ValueSource:

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## Was tun, wenn Tests timeout haben?

Verwenden Sie die @Timeout-Annotation, um ein Timeout festzulegen:

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // Schlägt fehl, wenn 100 Millisekunden überschritten werden
}
```

## Wie wiederhole ich Tests?

Verwenden Sie die @RepeatedTest-Annotation:

```java
@RepeatedTest(10)
void testRepeated() {
    // Wird 10 Mal wiederholt
}
```

## Was ist das JUnit 5-Erweiterungsmodell?

JUnit 5 verwendet die @ExtendWith-Annotation, um einen Erweiterungsmechanismus bereitzustellen, der eine flexiblere Testanpassung ermöglicht:

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // Verwendet benutzerdefinierte Erweiterung
}
```
