# OpenClaw 사용자 매뉴얼

이 매뉴얼은 OpenClaw를 효과적으로 사용하는 방법을 설명합니다.

## 기본 사용법

### OpenClaw와 대화

설치 후 다음과 같은 방식으로 OpenClaw와 상호작용할 수 있습니다:

1. **Control UI**: 브라우저에서 `http://127.0.0.1:18789/` 방문
2. **CLI 명령**: `openclaw agent --message "메시지"`
3. **구성 채널**: WhatsApp, Telegram, Discord 등을 통해 메시지 전송

### 메시지 보내기

CLI를 사용하여 메시지 보내기:

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## 핵심 개념

### Gateway

Gateway는 모든 세션, 채널, 도구 및 이벤트를 관리하는 OpenClaw의 핵심 제어 평면입니다. 기본 리스닝: `ws://127.0.0.1:18789`.

### 세션

OpenClaw는 대화를 관리하기 위해 세션 모델을 사용합니다:

- **메인 세션**: 직접 채팅
- **그룹 격리**: 그룹 메시지는 별도로 처리
- **활성화 모드**: 응답 시점 제어
- **응답 전략**: 응답 방식 결정

### 워크스페이스

워크스페이스는 OpenClaw가 구성과 데이터를 저장하는 곳입니다:

- 기본 위치: `~/.openclaw/workspace`
- 포함: AGENTS.md, SOUL.md, TOOLS.md
- 스킬 디렉토리: `~/.openclaw/workspace/skills/`

## 채널 구성

### 지원 채널

OpenClaw는 다음과 같은 메시징 플랫폼을 지원합니다:

- 인스턴트 메시징: WhatsApp, Telegram, Signal, iMessage
- 소셜 플랫폼: Discord, Slack
- 엔터프라이즈 도구: Microsoft Teams, Google Chat, Feishu
- 기타: IRC, Matrix, LINE, Mattermost, Nostr 등

### 구성 예시

**Telegram**:

```json
{
  "channels": {
    "telegram": {
      "botToken": "당신의_BOT_토큰"
    }
  }
}
```

**Discord**:

```json
{
  "channels": {
    "discord": {
      "token": "당신의_BOT_토큰"
    }
  }
}
```

**WhatsApp**:

```bash
openclaw channels login
```

## 스킬 시스템

스킬은 OpenClaw가 기능을 확장하는 방식입니다. 다음과 같이 할 수 있습니다:

### 스킬 설치

ClawHub에서 커뮤니티 스킬을 설치하거나 직접 만드세요.

### 스킬 구조

각 스킬은 SKILL.md 파일이 있는 디렉토리입니다:

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHub는 스킬 레지스트리입니다. 활성화되면 OpenClaw는 필요한 스킬을 자동으로 검색하고 설치합니다.

## 자동화

### 크론 작업

크론을 사용하여 예약 작업 설정:

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### 웹훅

자동화를 트리거하는 웹훅 구성:

```bash
openclaw webhook add --url https://your-endpoint.com/hook
```

### Gmail 통합

새 이메일을 수신하기 위해 Gmail Pub/Sub을 설정합니다.

## 고급 기능

### 브라우저 제어

OpenClaw는 전용 브라우저를 제어할 수 있습니다:

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvas 시각화

Canvas는 A2UI 푸시/리셋, 평가 및 스냅샷을 지원하는 OpenClaw의 비주얼 워크스페이스입니다.

### 음성 기능

- **Voice Wake**: macOS/iOS에서 깨우기 단어로 활성화
- **Talk Mode**: 지속적인 음성 대화 (Android 지원)

### 기기 노드

OpenClaw는 다양한 기기에 연결할 수 있습니다:

- **macOS 노드**: 시스템 실행, 알림, Canvas, 카메라
- **iOS 노드**: Canvas, 음성, 카메라, 화면 녹화
- **Android 노드**: 연결, 채팅, 음성, Canvas, 카메라, 위치 등

## 보안 설정

### DM 페어링

기본적으로 알 수 없는 발신자는 페어링 검증이 필요합니다. `dmPolicy: "open"`으로 공개 모드로 변경 가능 (권장 안 함).

### 페어링 관리:

```bash
# 페어링 목록 보기
openclaw pairing list

# 페어링 승인
openclaw pairing approve

# 페어링 취소
openclaw pairing revoke
```

### 진단 실행

보안을 위해 정기적으로 진단 실행:

```bash
openclaw doctor
```

## 명령 참조

| 명령 | 설명 |
|------|------|
| `/status` | 세션 상태 보기 |
| `/new` 또는 `/reset` | 세션 재설정 |
| `/compact` | 세션 컨텍스트 압축 |
| `/think` | 사고 수준 설정 |
| `/verbose` | 상세 출력 전환 |
| `/usage` | 사용 통계 보기 |
| `/restart` | Gateway 재시작 |

## 모범 사례

1. **정기적으로 doctor 실행**: 시스템 유지
2. **DM 주의**: 보안 설정 확인
3. **강력한 모델 사용**: 최상의 경험
4. **정기적으로 백업**: 중요한 구성 저장 |

## 문제 해결

일반적인 문제:

- **Gateway 시작 안 됨**: 포트 18789 확인
- **채널 연결 실패**: 자격 증명 확인
- **메시지 전송 실패**: 로그 확인
