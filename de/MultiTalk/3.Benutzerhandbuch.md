# Benutzerhandbuch

Dieses Handbuch bietet detaillierte Anweisungen zur Verwendung von MultiTalk, einschliesslich grundlegender Operationen, Parameterkonfiguration und fortgeschrittener Techniken.

## Grundlegende Verwendung

### Vorbereiten von Eingabedateien

MultiTalk erfordert die Vorbereitung spezifischer Eingabeformate. Das Projekt verwendet JSON-Dateien zur Definition von Generierungsparametern, einschliesslich Audio-Pfaden, Bildpfaden und Textprompts.

Fuer die Einzelpersonen-Generierung erfordert das Eingabe-JSON-Format die Angabe des Audio-Dateipfads, des Referenzbildpfads, des Video-Speichernamens und anderer Parameter. Beispieldateien befinden sich im examples-Verzeichnis, z.B. single_example_1.json.

Fuer die Mehrpersonen-Gesprachs-Generierung muss das Eingabe-JSON mehrere Audiostreams und entsprechende Personenbereichsdefinitionen enthalten. Beispieldateien befinden sich im examples-Verzeichnis, z.B. multitalk_example_2.json.

### Ausfuehren der grundlegenden Generierung

Nach Abschluss der Installation und Konfiguration wechseln Sie in das Projektverzeichnis und fuehren Sie den folgenden Befehl zur Videogenerierung aus:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### Parameterbeschreibung

--ckpt_dir gibt den Modellgewichte-Verzeichnispfad an. --wav2vec_dir gibt den Audio-Encoder-Verzeichnispfad an. --input_json gibt den Eingabeparameter-JSON-Dateipfad an. --sample_steps gibt die Anzahl der Sampling-Schritte an; hohere Werte erzeugen bessere Qualitat, aber langsamere Geschwindigkeit. --mode gibt den Generierungsmodus an, streaming fuer lange Videos, clip fuer kurze Videos. --use_teacache aktiviert TeaCache-Beschleunigung. --save_file gibt den Ausgabedatei-Speichernamen an.

## Fortgeschrittene Verwendungstechniken

### Anpassen der Generierungsqualitaet

Sampling-Schritte sind wichtige Parameter, die die Generierungsqualitaet beeinflussen. Die Standard-40 Schritte liefern gute Ergebnisse. Fuer schnellere Generierung koennen die Schritte auf 10 reduziert werden, was minimale Auswirkungen auf die Lippensynchronisation hat, aber die Bewegungsfluessigkeit und visuelle Qualitaet beeintraechtigt. Mehr Schritte verbessern die Detaildarstellung.

Die Lippensynchronisationsstaerke wird durch den Audio-Fuehrungskoeffizienten gesteuert. Audio CFG-Werte zwischen 3 und 5 funktionieren am besten. Erhohen dieses Wertes erreicht bessere Lippensynchronisation.

### Langvideo-Generierung

MultiTalk unterstuetzt die Generierung von Videos bis zu 15 Sekunden. Verwenden Sie den Streaming-Modus, um laengere Videoclips zu generieren:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

Bei der Generierung sehr langer Videos koennen Farbtonunterschiede zwischen Segmenten auftreten. Das Setzen von Audio CFG auf 3 hilft, die Farbtonkonsistenz aufrechtzuerhalten.

### Betrieb mit geringem VRAM

Wenn der VRAM begrenzt ist, koennen Sie die folgenden Parameter verwenden, um die Speichernutzung zu reduzieren:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

Das Hinzufuegen von --num_persistent_param_in_dit 0 ermoeglicht 480P-Videogenerierung auf einer einzelnen RTX 4090.

### Aktivierung der Beschleunigung

Das Projekt unterstuetzt mehrere Beschleunigungsmethoden zur Verbesserung der Generierungsgeschwindigkeit.

TeaCache-Beschleunigung kann die Geschwindigkeit erheblich verbessern, wahrend die Qualitaet erhalten bleibt. Empfohlene --teacache_thresh-Werte liegen zwischen 0.2 und 0.5:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### Verwendung von LoRA-Beschleunigung

Die Verwendung von LoRA kann die Sampling-Schritte erheblich reduzieren. FusioniX LoRA erfordert nur 8 Schritte, lightx2v LoRA nur 4 Schritte:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Verwendung der Gradio-Oberflaeche

Das Projekt bietet eine Gradio-Weboberflaeche fuer interaktive Verwendung. Wechseln Sie in das Projektverzeichnis und fuehren Sie aus:

```bash
python app.py
```

Oder verwenden Sie die Version mit geringem VRAM:

```bash
python app.py --num_persistent_param_in_dit 0
```

## Integration von TTS-Sprachsynthese

MultiTalk unterstuetzt direkte Texteingabe. Durch integrierte TTS-Technologie generiert es Sprache aus Text und erstellt dann Videos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## Multi-GPU-Konfiguration

Die Generierung von 720P-Hochaufloesungsvideos erfordert mehrere GPUs. Verwenden Sie den folgenden Befehl zur Multi-GPU-Konfiguration:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```
