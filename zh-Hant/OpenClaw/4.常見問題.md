# OpenClaw 常見問題

本頁面匯總了 OpenClaw 使用過程中最常見的問題和解答。如果你在這裡沒有找到答案，建議查看官方文檔或加入社群尋求幫助。

---

## 基礎問題

### 什麼是 OpenClaw？

OpenClaw 是一個自托管的 AI 助手閘道，它可以連接 WhatsApp、Telegram、Discord、iMessage 等多種聊天平台到一個 AI 編程助手（如 Pi）。你可以將它運行在自己的電腦上，完全掌控自己的數據。

### OpenClaw 是免費的嗎？

OpenClaw 本身是開源且免費的（MIT 許可證）。但使用 AI 模型（如 Anthropic Claude 或 OpenAI GPT）可能需要付費。你只需要支付 AI API 的使用費用。

### 我需要什麼樣的設備來運行 OpenClaw？

最低配置要求：
- Node.js 22+
- 2GB RAM
- 500MB 磁盤空間

推薦配置：
- 4GB+ RAM
- 穩定的網絡連接
- 長期開機的電腦或伺服器

## 安裝問題

### 安裝失敗怎麼辦？

1. 檢查 Node.js 版本：
   ```bash
   node --version
   ```
   確保版本 ≥ 22.0.0

2. 使用官方安裝腳本：
   ```bash
   curl -fsSL https://openclaw.ai/install.sh | bash
   ```

3. 查看詳細錯誤信息：
   ```bash
   npm install -g openclaw@latest --verbose
   ```

4. 運行診斷：
   ```bash
   openclaw doctor
   ```

### Windows 上安裝有問題？

**強烈推薦使用 WSL2**。OpenClaw 在 Linux 環境下運行最佳。

安裝 WSL2：
```powershell
wsl --install
```

然後在 WSL2 中安裝 OpenClaw。

### 如何升級 OpenClaw？

```bash
npm update -g openclaw@latest
```

或使用官方更新命令：
```bash
openclaw update
```

升級後運行診斷：
```bash
openclaw doctor
```

## 配置問題

### 配置文件在哪裡？

默認位置：`~/.openclaw/openclaw.json`

你也可以指定自定義路徑：
```bash
openclaw gateway --config /path/to/config.json
```

### 如何配置多個渠道？

在配置文件中添加多個渠道配置：

```json5
{
  channels: {
    whatsapp: { enabled: true },
    telegram: { enabled: true, botToken: "xxx" },
    discord: { enabled: true, token: "xxx" },
  },
}
```

### 配置不生效？

1. 確認配置文件格式正確（JSON5）
2. 檢查 Gateway 是否在運行
3. 嘗試重啟 Gateway：
   ```bash
   openclaw gateway restart
   ```

## 連接問題

### Gateway 無法啟動？

可能的原因：
- 端口 18789 被佔用
- 配置文件格式錯誤
- 權限不足

解決方法：
```bash
# 查看端口佔用
lsof -i :18789

# 檢查配置
openclaw doctor

# 以管理員權限運行
sudo openclaw gateway
```

### 渠道連接失敗？

1. 檢查憑證是否有效
2. 查看渠道狀態：
   ```bash
   openclaw channels status --probe
   ```
3. 查看日誌：
   ```bash
   openclaw logs --follow
   ```

### 無法訪問控制界面？

1. 確認 Gateway 正在運行
2. 檢查 URL 是否正確：http://127.0.0.1:18789/
3. 檢查防火牆設置
4. 嘗試重新啟動：
   ```bash
   openclaw gateway restart
   ```

## 使用問題

### AI 不回覆訊息？

檢查以下幾點：
1. 渠道是否已連接
2. 是否在白名單中
3. 群組中是否需要 @ 提及
4. 查看日誌中的錯誤信息

### 如何重置會話？

使用 `/new` 或 `/reset` 命令，或在 CLI 中：
```bash
openclaw session reset
```

### 如何更改 AI 模型？

在配置文件中設置：

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",
      },
    },
  },
}
```

## 安全問題

### OpenClaw 安全嗎？

OpenClaw 注重安全性：
- 所有數據存儲在本地
- 憑證加密存儲
- 支援白名單和配對機制
- 可啟用沙盒隔離

建議：
- 使用白名單控制訪問
- 定期更新版本
- 運行 `openclaw doctor` 檢查安全

### 如何設置白名單？

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123", "+15555550124"],
    },
  },
}
```

### 什麼是配對（Pairing）？

配對是一種安全機制，未知用戶首次發送訊息時會被攔截，需要你批准後才能使用 AI。

批准用戶：
```bash
openclaw pairing approve --channel whatsapp --sender +15555550123
```

## 功能問題

### 如何使用語音功能？

1. 配置 Talk Mode：
   ```json5
   {
     talk: {
       enabled: true,
       voice: "elevenlabs",
     },
   }
   ```

2. 在支援的設備上啟用 Voice Wake

### 定時任務不執行？

1. 檢查 Cron 是否啟用：
   ```bash
   openclaw cron status
   ```

2. 查看任務列表：
   ```bash
   openclaw cron list
   ```

3. 查看運行日誌：
   ```bash
   openclaw cron runs --id <jobId>
   ```

### 移動端節點無法連接？

1. 確保手機和 Gateway 在同一網絡
2. 檢查節點狀態：
   ```bash
   openclaw nodes status
   ```

3. 重新配對節點：
   ```bash
   openclaw nodes pair
   ```

## 性能問題

### 運行緩慢怎麼辦？

1. 檢查系統資源：
   ```bash
   openclaw status
   ```

2. 壓縮會話：
   ```bash
   /compact
   ```

3. 清理舊日誌：
   ```bash
   openclaw logs clean
   ```

### 如何限制資源使用？

```json5
{
  agents: {
    defaults: {
      maxTokens: 100000,
    },
  },
  session: {
    maxAgeHours: 24,
  },
}
```
