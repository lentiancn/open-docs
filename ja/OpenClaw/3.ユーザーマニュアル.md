# OpenClaw ユーザーマニュアル

このマニュアルでは、OpenClawを効果的に使用する方法を説明します。

## 基本的な使用法

### OpenClawとの会話

インストール後は、複数の方法でOpenClawと対話できます：

1. **Control UI**：ブラウザで`http://127.0.0.1:18789/`にアクセス
2. **CLIコマンド**：`openclaw agent --message "メッセージ"`
3. **設定済みチャネル**：WhatsApp、Telegram、Discordなどでメッセージを送信

### メッセージの送信

CLIを使用してメッセージを送信：

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## コアコンセプト

### Gateway

GatewayはOpenClawのコアコントロールプレーンで、すべてのセッション、チャネル、ツール、イベントを管理します。デフォルトListening: `ws://127.0.0.1:18789`。

### セッション

OpenClawは会話管理にセッションモデを使用します：

- **メインセッション**：ダイレクトチャット
- **グループ分離**：グループメッセージは別途処理
- **アクティベーションモード**：応答するタイミングを制御
- **返信戦略**：どのように返信するかを決定

### ワークスペース

ワークスペースはOpenClawが設定とデータを保存する場所です：

- デフォルトの場所：`~/.openclaw/workspace`
- 含まれるファイル：AGENTS.md、SOUL.md、TOOLS.md
- スキルディレクトリ：`~/.openclaw/workspace/skills/`

## チャネル設定

### サポートされているチャネル

OpenClawはこれらのメッセージングプラットフォームをサポートしています：

- インスタントメッセージング：WhatsApp、Telegram、Signal、iMessage
- ソーシャルプラットフォーム：Discord、Slack
- エンタープライズツール：Microsoft Teams、Google Chat、Feishu
- その他：IRC、Matrix、LINE、Mattermost、Nostrなど

### 設定例

**Telegram**：

```json
{
  "channels": {
    "telegram": {
      "botToken": "あなたのBotトークン"
    }
  }
}
```

**Discord**：

```json
{
  "channels": {
    "discord": {
      "token": "あなたのBotトークン"
    }
  }
}
```

**WhatsApp**：

```bash
openclaw channels login
```

## スキルシステム

スキルはOpenClawが機能を拡張する方法です。以下のことができます：

### スキルのインストール

ClawHubからコミュニティスキルをインストールするか、独自のスキルを作成します。

### スキル構造

各スキルはSKILL.mdファイルを含むディレクトリです：

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHubはスキルレジストリです。有効にすると、OpenClawは必要なスキルを自動的に検索してインストールできます。

## 自動化

### Cronジョブ

Cronを使用してスケジュールタスクを設定：

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhook

自動化をトリガーするWebhookを設定：

```bash
openclaw webhook add --url https://your-endpoint.com/hook
```

### Gmail統合

新しいメールをリistenするためのGmail Pub/Subを設定します。

## 上級機能

### ブラウザ制御

OpenClawは専用ブラウザを制御できます：

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Canvasビジュアル化

CanvasはOpenClawのビジュアルワークスペースで、A2UIプッシュ/リセット、評価、スナップショットをサポートします。

### 音声機能

- **Voice Wake**：macOS/iOSでウェイクワードでアクティブ化
- **Talk Mode**：連続音声会話（Androidサポート）

### デバイスノード

OpenClawは様々なデバイスに接続できます：

- **macOSノード**：システム実行、通知、Canvas、カメラ
- **iOSノード**：Canvas、音声、カメラ、画面録画
- **Androidノード**：接続、チャット、音声、Canvas、カメラ、位置情報など

## セキュリティ設定

### DMペアリング

デフォルトでは、未知の送信者はペアリング検証が必要です。`dmPolicy: "open"`でオープンモードに変更できますが、お勧めしません。

### ペアリングの管理：

```bash
# ペアリングリストを表示
openclaw pairing list

# ペアリングを承認
openclaw pairing approve

# ペアリングを取り消し
openclaw pairing revoke
```

### 診断の実行

セキュリティのために定期的に診断を実行：

```bash
openclaw doctor
```

## コマンドリファレンス

| コマンド | 説明 |
|----------|------|
| `/status` | セッション状態を表示 |
| `/new` または `/reset` | セッションをリセット |
| `/compact` | セッションコンテキストを圧縮 |
| `/think` | 思考レベルを設定 |
| `/verbose` | 詳細出力を切り替え |
| `/usage` | 使用統計を表示 |
| `/restart` | Gatewayを再起動 |

## ベストプラクティス

1. **定期的にdoctorを実行**：システムを健全に保つ
2. **DMに注意**：セキュリティ設定を確認
3. **強力なモデルを使用**：最適な体験を 얻る
4. **定期的にバックアップ**：重要な設定を保存

## トラブルシューティング

一般的な問題の解決：

- **Gatewayが起動しない**：ポート18789を確認
- **チャネル接続に失敗**：認証情報を確認
- **メッセージ送信に失敗**：ログを確認
