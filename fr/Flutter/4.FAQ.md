# FAQ Flutter

## Problèmes d'Installation

### Que faire si Flutter doctor affiche des erreurs ?

La commande flutter doctor vérifie l'état de l'environnement de développement Flutter. Erreurs courantes et solutions :

- Android SDK non trouvé : Installez Android Studio ou configurez manuellement la variable d'environnement ANDROID_SDK_ROOT
- Version Xcode trop ancienne : Mettez à jour Xcode vers la dernière version
- Licences non acceptées : Exécutez flutter doctor --android-licences pour accepter les licences

### Que faire si le téléchargement de Flutter échoue ?

Si le téléchargement est lent ou échoue, essayiez d'utiliser un miroir. Les utilisateurs chinois peuvent définir les variables d'environnement PUB_HOSTED_URL et FLUTTER_STORAGE_BASE_URL pour utiliser des miroirs nationaux.

### Que faire si le clone Git échoue ?

Assurez-vous que le réseau peut accéder à GitHub. Essayiez d'utiliser un proxy ou un miroir national. Vous pouvez également télécharger l'archive précompilée depuis le site Web Flutter et l'installer manuellement.

## Problèmes de Développement

### Le rechargement à chaud ne fonctionne pas ?

Lorsque le rechargement à chaud échoue, essayiez : Cliquer sur Stop puis exécuter à nouveau ; Exécuter flutter clean pour nettoyer le cache de construction ; Vérifier s'il y a des erreurs de syntaxe dans le code.

### L'application est lente ?

Vérifiez si des opérations chronophages sont effectuées dans la méthode build. Les calculs complexes doivent utiliser Isolate ou un traitement asynchrone. Évitez les rebuilds inutiles de widgets, utilisez des constructeurs const et des clés appropriées.

### L'état est perdu ?

Le rechargement à chaud préserve l'état StatefulWidget, mais l'état est réinitialisé lors de la reconstruction de l'application ou lors d'un redémarrage à chaud complet. Pour persister l'état, utilisez SharedPreferences, SQLite ou d'autres solutions de stockage.

### Conflits de dépendances de packages ?

Exécutez flutter pub get pour voir l'arbre des dépendances. Utilisez flutter pub deps pour vérifier les dépendances des packages. Essayiez de mettre à jour les versions des packages ou utilisez dependency_overrides pour résoudre les conflits.

## Problèmes de Plateforme

### La construction Android échoue ?

Causes courantes : Version SDK non conforme, problèmes de configuration Gradle, erreurs de configuration de signature. Essayiez d'exécuter flutter clean puis reconstruisez. Vérifiez les configurations dans android/build.gradle et android/app/build.gradle.

### Le simulateur iOS ne démarre pas ?

Assurez-vous qu'Xcode est installé et correctement configuré. Exécutez xcrun simctl list devices dans Terminal pour voir les simulateurs disponibles. Essayiez de réinitialiser le simulateur ou d'en créer un nouveau.

### Erreur de construction Web ?

Vérifiez si les dépendances Web dans pubspec.yaml sont correctes. Utilisez flutter clean pour nettoyer puis reconstruisez. Assurez-vous que le navigateur Chrome est installé.

## Problèmes de Performance

### Taille de l'application trop grande ?

Utilisez flutter build apk --split-per-abi pour générer des APK pour différentes architectures CPU. Vérifiez et supprimez les images et ressources inutilisées. Utilisez l'obfuscation et la compression.

### Temps de démarrage trop long ?

Réduisez les opérations d'initialisation dans la fonction main. Chargez de manière paresseuse les composants non essentiels. Utilisez Splash Screen pour améliorer l'expérience utilisateur.

### Utilisation de mémoire trop élevée ?

Vérifiez s'il y a des fuites de mémoire. Utilisez les outils Profile pour analyser l'utilisation de la mémoire. Libérez les ressources et grands objets qui ne sont plus utilisés.

## Problèmes d'Apprentissage

### Comment choisir une solution de gestion d'état ?

Utilisez setState pour les applications simples. Provider est recommandé pour les applications de complexité moyenne. Riverpod, GetX ou BLoC peuvent être utilisés pour les applications complexes.

### Migration depuis d'autres frameworks ?

Le site Web Flutter propose des guides de migration pour les développeurs Android, iOS et React Native. Les concepts de base sont similaires, concentrez-vous sur l'apprentissage des widgets et du système de mise en page.

### Comment améliorer l'efficacité du développement ?

Maîtrisez les plugins Flutter pour VS Code ou Android Studio. Apprenez les raccourcis et les modèles de code. Utilisez le rechargement à chaud et les outils de débogage. Consultez la documentation officielle et les ressources communautaires.
