# Angular FAQ

Common questions and answers.

## Installation

### Q: Angular CLI installation failed?

**Solutions:**
1. Check Node.js version
```bash
node --version  # Requires 18+ or 20+
```
2. Clean npm cache
```bash
npm cache clean --force
```
3. Use administrator (Windows)
4. Check network connection

### Q: ng command not found?

**Solutions:**
1. Confirm Angular CLI is installed
```bash
npm list -g @angular/cli
```
2. Check PATH environment variable
3. Reinstall
```bash
npm uninstall -g @angular/cli
npm install -g @angular/cli@latest
```

### Q: Project creation failed?

**Solutions:**
1. Check disk space
2. Check npm permissions
3. Use --skip-install
```bash
ng new my-app --skip-install
cd my-app
npm install
```

## Development

### Q: Component not displaying?

**Solutions:**
1. Confirm component is imported correctly
2. Confirm selector is used correctly
3. Check template syntax
4. Check console for errors

### Q: Data not updating?

**Solutions:**
1. For Signals, use set/update methods
```typescript
count.set(1);           // Set new value
count.update(c => c + 1); // Update based on old value
```
2. For RxJS, ensure proper subscription
3. Use ChangeDetectorRef for manual detection
```typescript
constructor(private cdr: ChangeDetectorRef) {}

refresh() {
  this.cdr.detectChanges();
}
```

### Q: Routing not working?

**Solutions:**
1. Confirm route config is correct
2. Confirm RouterOutlet is placed
3. Check route order
```typescript
// Specific routes before wildcard
{ path: 'user/:id', ... },
{ path: '**', component: NotFoundComponent }
```

### Q: Form validation not working?

**Solutions:**
1. Import ReactiveFormsModule or FormsModule
2. Confirm FormControl names are correct
3. Use validators correctly
```typescript
this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', [Validators.required, Validators.minLength(6)]]
});
```

## Performance

### Q: Slow initial load?

**Solutions:**
1. Use lazy loading routes
```typescript
{
  path: 'admin',
  loadComponent: () => import('./admin/admin.component')
    .then(m => m.AdminComponent)
}
```
2. Enable production mode
```bash
ng build --configuration=production
```
3. AOT compilation is enabled by default

### Q: Memory leaks?

**Solutions:**
1. Unsubscribe properly
```typescript
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

@Component({...})
export class MyComponent implements OnDestroy {
  private destroy$ = new Subject<void>();

  ngOnInit() {
    this.http.get('/api').pipe(
      takeUntil(this.destroy$)
    ).subscribe();
  }

  ngOnDestroy() {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
```
2. Use async pipe for automatic subscribe/unsubscribe

### Q: Bundle size too large?

**Solutions:**
1. Check dependency sizes
```bash
npm run build -- --stats-json
npx webpack-bundle-analyzer dist/stats.json
```
2. Use lazy loading
3. Remove unused imports

## Testing

### Q: How to write unit tests?

**Solutions:**
```typescript
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { UserCardComponent } from './user-card.component';

describe('UserCardComponent', () => {
  let component: UserCardComponent;
  let fixture: ComponentFixture<UserCardComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [UserCardComponent]
    }).compileComponents();

    fixture = TestBed.createComponent(UserCardComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display user name', () => {
    component.user = { name: 'John', email: 'test@example.com' };
    fixture.detectChanges();
    const compiled = fixture.nativeElement as HTMLElement;
    expect(compiled.querySelector('h3')?.textContent).toContain('John');
  });
});
```

### Q: Test module not found?

**Solutions:**
```typescript
TestBed.configureTestingModule({
  imports: [
    CommonModule,
    HttpClientModule,
    // Other required modules
  ]
});
```

## Build

### Q: Build failed?

**Solutions:**
1. Check TypeScript errors
```bash
npx tsc --noEmit
```
2. Clean cache
```bash
rm -rf node_modules
npm install
```
3. Update Angular version
```bash
ng update @angular/core @angular/cli
```

### Q: Environment variables not working?

**Solutions:**
1. Confirm environment file names are correct
2. Configure environment in angular.json
```json
"configurations": {
  "production": {
    "fileReplacements": [
      {
        "replace": "src/environments/environment.ts",
        "with": "src/environments/environment.prod.ts"
      }
    ]
  }
}
```

## Security

### Q: How to prevent XSS attacks?

**Solutions:**
1. Angular auto-escapes by default
2. Be careful with innerHTML
```typescript
import { DomSanitizer } from '@angular/platform-browser';

constructor(private sanitizer: DomSanitizer) {}

getHtml() {
  // Only for trusted content
  return this.sanitizer.bypassSecurityTrustHtml(this.trustedContent);
}
```

### Q: CSRF protection?

**Solutions:**
```typescript
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
    HttpClientXsrfModule.withOptions({
      cookieName: 'XSRF-TOKEN',
      headerName: 'X-XSRF-TOKEN'
    })
  ]
})
export class AppModule {}
```

## Upgrade

### Q: How to upgrade Angular version?

**Solutions:**
1. Use ng update
```bash
ng update @angular/core@17 @angular/cli@17
```
2. Check migration guide
3. Update dependency versions
4. Run tests to ensure compatibility

### Q: Code incompatible after upgrade?

**Solutions:**
1. Check official upgrade guide
2. Use ng update for automatic migration
3. Upgrade incrementally, not jumping versions
