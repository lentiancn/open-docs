# OpenCode Manual de usuario

Uso básico y técnicas de OpenCode.

## Inicio rápido

### Hacer preguntas

Puedes pedirle a OpenCode que explique el código base:

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### Agregar funciones

Antes de agregar funciones, se recomienda que OpenCode cree un plan primero:

1. Presiona **Tab** para cambiar al modo Plan
2. Describe la función que quieres
3. Después de ver el plan, proporciona retroalimentación o agrega más detalles
4. Cuando estés satisfecho, presiona **Tab** para volver al modo Build

### Hacer cambios directos

Para cambios más directos, puedes pedirle a OpenCode directamente:

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### Deshacer cambios

Si no estás satisfecho con los cambios, usa el comando deshacer:

```bash
/undo
```

OpenCode revertirá los cambios y mostrará tu mensaje original de nuevo. Puedes ajustar tu mensaje y pedirle que intente de nuevo.

### Rehacer

Puedes usar `/redo` para rehacer los cambios.

## Compartir sesiones

```bash
/share
```

Esto crea un enlace a la sesión actual y lo copia al portapapeles.

## Comandos comunes

| Comando | Función |
|---------|----------|
| /init | Inicializar proyecto |
| /connect | Conectar proveedor LLM |
| /undo | Deshacer cambios |
| /redo | Rehacer cambios |
| /share | Compartir sesión |

## Mejores prácticas

1. Habla con OpenCode como si estuvieras hablando con un desarrollador junior
2. Proporciona suficiente contexto y detalles
3. Pide que OpenCode cree un plan primero, luego ejecuta
4. Usa /undo cuando no estés satisfecho
5. Revisa y actualiza AGENTS.md regularmente
