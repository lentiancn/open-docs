# Yarn 常見問題

## 安裝相關問題

### Q: 如何選擇 Yarn 版本？
**A**: 
- **新專案**: 推薦使用 Yarn v4+，享受現代特性和更好的效能
- **現有專案**: 如果已在使用 Yarn v1，可以繼續使用或考慮遷移到現代版本
- **企業環境**: Yarn v1 提供更穩定的長期支援，而 v4+ 提供更多創新功能

### Q: Corepack 是什麼？為什麼需要它？
**A**: Corepack 是 Node.js 附帶的中介工具，允許您基於每個專案配置套件管理器版本。它確保團隊成員使用相同的 Yarn 版本，避免版本不一致問題。

### Q: 安裝 Yarn 時遇到權限錯誤怎麼辦？
**A**: 
- 使用 `npm install --global yarn` 時新增 `--unsafe-perm` 標誌
- 或者使用 nvm 管理 Node.js 版本
- 在 CI/CD 環境中，通常不需要全域安裝

## 使用相關問題

### Q: Yarn 和 npm 有什麼區別？
**A**: 
- **速度**: Yarn 使用平行安裝和離線快取，通常更快
- **確定性**: Yarn 保證安裝的一致性（yarn.lock）
- **功能**: Yarn 提供工作區、PnP、外掛系統等進階功能
- **命令**: Yarn 重新實作了所有命令，提供更好的開發者體驗

### Q: 什麼是 PnP (Plug'n'Play)？
**A**: PnP 是 Yarn v2+ 的一項功能，它消除了 node_modules 目錄，直接從快取中解析相依性。這節省了磁碟空間並提高了安裝速度，但可能與某些不支援 PnP 的工具不相容。

### Q: 如何在專案中使用工作區？
**A**: 
1. 在根目錄的 `package.json` 中新增 `"workspaces"` 欄位
2. 建立子套件目錄（如 `packages/`）
3. 在子套件中建立各自的 `package.json`
4. 使用 `yarn workspace <name> <command>` 在特定工作區中執行命令

## 版本遷移問題

### Q: 如何從 Yarn v1 遷移到 v4+？
**A**: 
1. 確保專案使用 Git 進行版本控制
2. 執行 `yarn set version stable`（對於 v4+）
3. 按照遷移指南逐步更新配置
4. 測試所有功能確保相容性

### Q: 遷移後遇到相容性問題怎麼辦？
**A**: 
- 檢查是否有工具不支援 PnP，可以停用 PnP：`yarn config set nodeLinker node-modules`
- 更新相依性到相容版本
- 查看官方遷移文件獲取具體解決方案

## 故障排除

### Q: `yarn install` 失敗怎麼辦？
**A**: 
- 清除快取：`yarn cache clean`
- 刪除 `yarn.lock` 並重新安裝
- 檢查網路連線和註冊表配置
- 對於 Yarn v4+，檢查 `.yarnrc.yml` 配置

### Q: 如何查看 Yarn 版本？
**A**: 
- 全域版本：`yarn --version`
- 專案版本：查看 `.yarn/releases/` 目錄或 `.yarnrc.yml` 檔案

### Q: Yarn 命令執行很慢怎麼辦？
**A**: 
- 確保啟用了離線快取
- 檢查網路連線
- 對於大型專案，考慮使用工作區分割
- 更新到最新版本以獲得效能改進
