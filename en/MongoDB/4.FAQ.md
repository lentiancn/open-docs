# MongoDB Frequently Asked Questions

## Basic Concepts

### What is MongoDB?

MongoDB is a document database designed to help developers build modern applications faster. It stores data in flexible, JSON-like documents, making it easy to model data the same way your application code uses it. MongoDB provides a powerful query engine, horizontal scaling, and built-in high availability.

### Is MongoDB a Relational Database?

No. MongoDB is a document-oriented NoSQL database that stores data using BSON format similar to JSON. It is fundamentally different from traditional relational databases and does not use the concepts of tables, rows, and columns. Instead, it uses collections and documents.

### What is a Document in MongoDB?

A document is the basic data unit in MongoDB, consisting of field and value pairs. Documents can contain other documents, arrays, and nested objects, with a very flexible structure. Each document has a unique _id field as its primary key.

### What is the Difference Between MongoDB and MySQL?

| Feature | MongoDB | MySQL |
|---------|---------|-------|
| Data Model | Document (JSON-like) | Relational (rows and columns) |
| Schema | Flexible (dynamic) | Fixed (static) |
| Scaling | Horizontal (sharding) | Vertical |
| Query Language | MongoDB Query Language | SQL |
| Transactions | Multi-document ACID | Multi-table ACID |
| Relationships | Embedded documents/arrays | JOIN operations |
| Indexing | Multiple index types | Indexes supported |

### What is BSON?

BSON stands for Binary JSON, which is the binary representation of JSON documents with additional data type support (such as ObjectId, Date, Binary, etc.).

### What is the Maximum Document Size in MongoDB?

The maximum BSON document size is 16 MB. For files exceeding this limit, MongoDB provides the GridFS API.

### What Data Types Does MongoDB Support?

MongoDB supports many BSON data types, including: Double, String, Object, Array, Binary Data, ObjectId, Boolean, Date, Null, Regular Expression, JavaScript, JavaScript with Scope, 32-bit Integer, Timestamp, 64-bit Integer, Decimal128, Min Key, Max Key, and more.

## Installation Questions

### How Much Memory Does MongoDB Need?

MongoDB tries to use available memory as cache. The official recommendation is at least 4GB of memory for development environments. Production environments should have more memory based on data volume and workload.

### Does MongoDB Support 32-bit Systems?

No. MongoDB stopped supporting 32-bit platforms in a previous version. Please use a 64-bit operating system.

### How to Change MongoDB Default Port?

The default MongoDB port is 27017. You can change it by:

1. Modifying `net.port` in the configuration file
2. Specifying the `--port` parameter at startup

### Can Multiple MongoDB Instances Run on the Same Machine?

Yes, but you need to configure different:

- Port numbers
- Data directories (`--dbpath`)
- Log file paths
- Configuration files

### Where Does MongoDB Store Data?

By default:

- Linux/macOS: `/data/db`
- Windows: `C:\data\db`

You can specify a different path using `storage.dbPath` in the configuration file.

### Should I Install MongoDB 8.0 or 7.0?

- New projects: Use the latest stable version MongoDB 8.2
- Production environments: Choose the latest stable version or long-term support version MongoDB 7.0

MongoDB 6.0 reached end of life on July 31, 2025 and no longer receives security updates.

## Usage Questions

### How to Backup MongoDB Data?

Use the mongodump tool:

```bash
# Export entire database
mongodump --db=mydatabase --out=/backup

# Export specific collection
mongodump --db=mydatabase --collection=users --out=/backup
```

Restore using mongorestore:

```bash
mongorestore --db=mydatabase /backup/mydatabase
```

### How Does MongoDB Achieve High Availability?

Through Replica Sets. A replica set consists of a primary node and multiple secondary nodes:

- The primary node handles all write operations
- Secondary nodes replicate the primary's data
- When the primary fails, a new primary is automatically elected

### What is Sharding?

Sharding is MongoDB's approach to horizontal scaling, distributing data across multiple servers. Each shard contains a portion of the data, and together they form the complete data set. Sharding allows you to build very large datasets and high-throughput operations.

### What is the Difference Between Replica Sets and Sharded Clusters?

- **Replica Set**: Multiple MongoDB instances maintain the same data copy, providing high availability and read scaling
- **Sharded Cluster**: Data is distributed across multiple servers, providing horizontal scaling and supporting large-scale datasets

### How to Optimize Query Performance?

1. Create appropriate indexes: Use `createIndex()` for commonly queried fields
2. Use projections: Return only necessary fields to reduce network transfer
3. Avoid full collection scans: Ensure queries use indexes
4. Use aggregation pipelines: For complex data processing
5. Use covered indexes appropriately: Index contains all queried fields
6. Limit results: Use `limit()` and `skip()` for pagination

### Does MongoDB Support Transactions?

Yes. MongoDB 4.0+ introduced multi-document transaction support, which works across replica sets and sharded clusters. Transactions follow ACID semantics:

- Atomicity: All operations in a transaction either succeed or fail completely
- Consistency: Database state remains consistent before and after the transaction
- Isolation: Concurrent transactions are isolated from each other
- Durability: Committed data is persistently stored

### What are Change Streams?

Change streams are a feature of MongoDB that allow applications to listen to real-time data change events on collections, databases, or entire clusters. Applications can subscribe to change events to implement real-time synchronization, audit logs, and other features.

### How Does MongoDB Ensure Data Security?

1. **Authentication**: Enable username/password authentication or LDAP/Kerberos (Enterprise)
2. **Access Control**: Use Role-Based Access Control (RBAC)
3. **Encryption**: Enable TLS/SSL encryption for client-server communication
4. **Storage Encryption**: Use MongoDB Enterprise's encrypted storage engine
5. **Field-Level Encryption**: Encrypt sensitive fields
6. **Audit Logging**: Log database operations (Enterprise)
7. **Regular Backups**: Backup data regularly

### What is the Difference Between MongoDB Enterprise and Community Editions?

Enterprise edition includes more advanced features:

- LDAP authentication
- Kerberos authentication
- Audit
- Encrypted storage engine
- Flattened analytics
- Technical support

Community edition features are sufficient for most use cases.

### How to Choose the Right Version?

- New projects: Use the latest stable version (MongoDB 8.2)
- Production environments: Choose the latest stable version or long-term support version (MongoDB 7.0)
- Specific requirements: Choose based on application compatibility and feature requirements

### What is GridFS?

GridFS is a specification for storing large files in MongoDB. It splits large files into multiple chunks (default 255KB each), storing them in the `fs.chunks` collection, with file metadata stored in the `fs.files` collection.

### What Type of Data is MongoDB Suitable For?

- Content management system data (articles, comments, user-generated content)
- User configuration and profile data
- Log and event data
- IoT sensor data
- Real-time analytics data
- Mobile application backend data
- E-commerce product catalogs
- Chat messages and real-time communication data

### What is the Replica Set Oplog?

The oplog (operation log) is a special collection in replica sets that records all write operations on the primary node. Secondary nodes maintain data synchronization with the primary by reading and applying operations from the oplog.

### How to Monitor MongoDB Performance?

1. Use `db.serverStatus()` to view server status
2. Use `db.stats()` to view database status
3. Use Profiler to analyze slow queries: `db.setProfilingLevel(1, 100)`
4. Use `db.currentOp()` to view current operations
5. Integrate Prometheus and Grafana for visual monitoring
6. Use MongoDB Atlas built-in monitoring (if using cloud service)

### What is Replication Lag?

Replication lag is the delay between an operation on the primary and the application of that operation to the secondary. A small amount of lag is acceptable, but significant lag causes serious problems. Monitor with:

```bash
rs.status()
```

Check the difference between `optimes.lastCommittedOpTime` and the primary's operation time.

### What is Read Preference?

Read preference determines which node in a replica set the client reads from:

- **primary**: Read from primary (default)
- **primaryPreferred**: Prefer primary, fallback to secondary if needed
- **secondary**: Read from secondary only
- **secondaryPreferred**: Prefer secondary, fallback to primary if needed
- **nearest**: Read from the node with lowest network latency

### What is Write Concern?

Write concern specifies the acknowledgment level for write operations:

- **w: 1**: Wait for primary acknowledgment (default)
- **w: majority**: Wait for majority of replica set members
- **w: "all"**: Wait for all replica set members
- **j: true**: Wait for write to disk journal
- **wtimeout**: Timeout value

### What is a Capped Collection?

A capped collection is a fixed-size collection, similar to a circular queue. When the size limit is reached, old documents are automatically replaced by new ones. Commonly used for logs, caching, and other scenarios.

### What Index Types Does MongoDB Support?

- Single field index
- Compound index
- Multikey index (array field)
- Text index
- Geospatial index (2dsphere, 2d)
- Hashed index
- Unique index
- Partial index
- Sparse index
- TTL index

### What is a TTL Index?

TTL (Time-To-Live) indexes allow documents to automatically expire and be deleted after a specified time. Commonly used for sessions, caching, temporary data, and other scenarios.

## Troubleshooting

### Connection Refused, What to Do?

1. Check if MongoDB service is running: `sudo systemctl status mongod`
2. Check if port is correct: default is 27017
3. Check firewall settings
4. Check bind IP configuration

### What to Do When Disk Space is Low?

1. Clean up old data
2. Delete unneeded collections
3. Use `db.repairDatabase()` to reclaim space
4. Expand disk space

### What to Do When Queries are Slow?

1. Use `explain()` to analyze query plan
2. Create appropriate indexes
3. Check for slow queries: `db.getProfilingLevel()`
4. Consider using projections to return only necessary fields

### What to Do When Replica Set Election Fails?

1. Check network connectivity
2. Check node configuration
3. Ensure majority of nodes are available
4. Check logs: `rs.status()`

### What to Do When Data is Inconsistent?

1. Sync secondary nodes
2. Reinitialize replica set
3. Restore data from backup

### How to Use MongoDB in Applications?

1. Install the driver for your language
2. Establish a connection
3. Perform CRUD operations

Example in Node.js:

```javascript
const { MongoClient } = require('mongodb');
const client = new MongoClient('mongodb://localhost:27017');

async function main() {
  await client.connect();
  const database = client.db('mydb');
  const collection = database.collection('users');
  
  // Insert document
  await collection.insertOne({ name: 'John', age: 30 });
  
  // Query document
  const user = await collection.findOne({ name: 'John' });
  
  await client.close();
}
```

### How to Perform Data Modeling?

MongoDB data modeling principles:

1. Prefer document structure over normalization
2. Use embedded documents to represent "contains" relationships
3. Use references for "many-to-many" relationships
4. Design collections and indexes based on query patterns
5. Consider data growth and scalability
