# Node.js 安装指南

## 下载 Node.js

Node.js 提供了适用于各种操作系统的官方安装包。访问 [Node.js 官方网站](https://nodejs.org/zh-cn/download) 下载最新版本。

### 版本选择

- **LTS (长期支持) 版本**: 推荐用于生产环境，提供稳定的 API 和长期支持
- **Current (当前) 版本**: 包含最新功能，适合开发和测试环境

## Windows 安装

### 方法一：使用官方安装包

1. 访问 [Node.js 下载页面](https://nodejs.org/zh-cn/download)
2. 下载 Windows 安装程序 (.msi)
3. 运行安装程序并按照向导完成安装
4. 验证安装：
   ```bash
   node --version
   npm --version
   ```

### 方法二：使用 Windows Package Manager (winget)

```bash
# 安装 LTS 版本
winget install OpenJS.NodeJS.LTS

# 安装最新版本
winget install OpenJS.NodeJS
```

### 方法三：使用 Chocolatey

```bash
# 安装 LTS 版本
choco install nodejs-lts

# 安装最新版本
choco install nodejs
```

## macOS 安装

### 方法一：使用官方安装包

1. 下载 macOS 安装包 (.pkg)
2. 双击运行安装程序
3. 按照向导完成安装

### 方法二：使用 Homebrew

```bash
# 安装 LTS 版本
brew install node@lts

# 或安装最新版本
brew install node

# 如果需要将命令添加到 PATH
echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 方法三：使用 MacPorts

```bash
# 安装 LTS 版本
sudo port install nodejs-lts

# 安装最新版本
sudo port install nodejs
```

## Linux 安装

### Ubuntu/Debian

#### 方法一：使用包管理器（可能不是最新版本）

```bash
sudo apt update
sudo apt install nodejs npm
```

#### 方法二：使用 NodeSource 仓库获取最新版本

```bash
# 对于 LTS 版本
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# 对于最新版本
curl -fsSL https://deb.nodesource.com/setup_current.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### CentOS/RHEL/Fedora

#### 方法一：使用包管理器

```bash
# Fedora
sudo dnf install nodejs npm

# CentOS/RHEL 8+
sudo dnf module enable nodejs:18
sudo dnf install nodejs
```

#### 方法二：使用 NodeSource 仓库

```bash
# 对于 LTS 版本
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install nodejs

# 对于最新版本
curl -fsSL https://rpm.nodesource.com/setup_current.x | sudo bash -
sudo yum install nodejs
```

### 其他 Linux 发行版

大多数 Linux 发行版都可以通过各自的包管理器安装 Node.js：

- **Arch Linux**: `sudo pacman -S nodejs npm`
- **openSUSE**: `sudo zypper install nodejs npm`
- **Alpine Linux**: `apk add nodejs npm`

## 验证安装

安装完成后，验证 Node.js 和 npm 是否正确安装：

```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本  
npm --version

# 运行简单的测试
node -e "console.log('Hello Node.js!')"

# 检查全局安装目录
npm config get prefix
```

## 管理多个 Node.js 版本

在开发中可能需要在不同项目中使用不同的 Node.js 版本。推荐使用以下工具：

### nvm (Node Version Manager) - macOS/Linux

```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 或使用 wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载 shell 配置
source ~/.bashrc  # 或 ~/.zshrc
```

常用命令：
```bash
# 列出所有可用版本
nvm list-remote

# 列出已安装版本
nvm list

# 安装特定版本
nvm install 18.17.0

# 安装 LTS 版本
nvm install --lts

# 切换版本
nvm use 18.17.0

# 设置默认版本
nvm alias default 18.17.0

# 卸载版本
nvm uninstall 16.20.0
```

### nvm-windows - Windows

1. 下载 [nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. 运行安装程序
3. 使用命令行管理版本：

```cmd
:: 列出可用版本
nvm list available

:: 安装版本
nvm install 18.17.0

:: 切换版本
nvm use 18.17.0

:: 查看已安装版本
nvm list
```

### fnm (Fast Node Manager) - 跨平台

```bash
# 安装 fnm
curl -fsSL https://fnm.vercel.app/install | bash

# 常用命令
fnm install 18.17.0
fnm use 18.17.0
fnm default 18.17.0
```

## 环境变量配置

确保 Node.js 和 npm 的可执行文件路径已添加到系统 PATH 环境变量中。

### Windows

- 通常自动添加到 PATH
- 如果未找到命令，检查系统环境变量中的 PATH

### macOS/Linux

如果使用包管理器安装，通常会自动配置。如果手动安装或使用版本管理器：

```bash
# 对于 nvm 用户，通常自动配置
# 对于手动安装，添加到 ~/.bashrc 或 ~/.zshrc
export PATH=$PATH:/usr/local/bin
export PATH=$PATH:$HOME/.local/bin
```

## 故障排除

### 权限问题

在 Linux/macOS 上，如果遇到 npm 全局安装的权限问题，建议：

#### 解决方案 1：使用版本管理器
- 使用 nvm 或 fnm 管理 Node.js（推荐）

#### 解决方案 2：配置 npm 全局目录
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

#### 解决方案 3：使用 sudo（不推荐）
```bash
sudo npm install -g package-name
```

### 网络问题

如果下载速度慢或无法连接，可以配置 npm 使用国内镜像：

```bash
# 设置淘宝镜像（推荐）
npm config set registry https://registry.npmmirror.com

# 或临时使用
npm install --registry https://registry.npmmirror.com

# 验证设置
npm config get registry

# 恢复官方源
npm config delete registry
```

### 版本冲突

如果系统中存在多个 Node.js 版本导致冲突：

1. **卸载所有版本**：
   - Windows: 通过控制面板卸载
   - macOS: `brew uninstall node` 或删除安装目录
   - Linux: `sudo apt remove nodejs npm`

2. **使用版本管理器重新安装**：
   - 安装 nvm/fnm
   - 通过版本管理器安装所需版本

### 常见错误信息

#### "command not found: node"
- **原因**: PATH 未正确配置
- **解决**: 重新安装或手动添加到 PATH

#### "npm ERR! code EACCES"
- **原因**: 权限不足
- **解决**: 使用版本管理器或配置 npm 全局目录

#### "Error: Cannot find module"
- **原因**: 依赖未安装或路径错误
- **解决**: 运行 `npm install` 或检查模块路径

通过以上步骤，您应该能够成功安装和配置 Node.js 环境。如果遇到其他问题，请参考官方文档或社区支持资源。
