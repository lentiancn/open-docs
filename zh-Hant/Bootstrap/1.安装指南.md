## Bootstrap 簡介

Bootstrap 是最流行的 HTML、CSS 和 JS 框架，用於開發響應式佈局、行動裝置優先的網站和 Web 應用程式。

### Bootstrap 的特點

- **響應式設計**：內建響應式柵格系統
- **預先定義樣式**：豐富的元件和樣式類
- **JavaScript 元件**：互動式元件（彈出視窗、輪播等）
- **自訂化**：透過 Sass 變數自訂主題
- **瀏覽器相容**：支援所有現代瀏覽器

### Bootstrap 版本歷史

| 版本 | 發布時間 | 主要特性 |
|------|----------|----------|
| Bootstrap 5 | 2021年5月 | 移除了 jQuery 依賴、改進的表單元件、新的 Utilities API |
| Bootstrap 4 | 2018年1月 | Flexbox 支援、Sass 替代 Less、新的元件 |
| Bootstrap 3 | 2013年8月 | 行動裝置優先設計 |
| Bootstrap 2 | 2012年1月 | 響應式設計改進 |
| Bootstrap 1 | 2011年8月 | 初始版本 |

> **注意**：Bootstrap 5 是目前主要版本，建議新專案使用。Bootstrap 4 仍有維護更新，但新功能開發已停止。

## CDN 安裝

### CSS only

將以下代碼添加到 HTML 的 `<head>` 部分：

```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" crossorigin="anonymous">
```

### CSS + JavaScript

添加 CSS 後，在 `</body>` 前添加 JS：

```html
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js" integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz" crossorigin="anonymous"></script>
```

> **注意**：Bootstrap 5 使用 `bootstrap.bundle.min.js`（包含 Popper），不再需要單獨引入 Popper。

### 使用國內 CDN（推薦國內用戶）

```html
<!-- CSS -->
<link href="https://cdn.staticfile.org/twitter-bootstrap/5.3.3/css/bootstrap.min.css" rel="stylesheet">

<!-- JS -->
<script src="https://cdn.staticfile.org/twitter-bootstrap/5.3.3/js/bootstrap.bundle.min.js"></script>
```

## npm 安裝

### 安裝 Bootstrap

```bash
# 使用 npm 安裝
npm install bootstrap

# 安裝指定版本
npm install bootstrap@5.3.3

# 安裝 Bootstrap 4
npm install bootstrap@4.6.2
```

### 在專案中引用

**CSS**：
```javascript
// webpack 或 bundler
import 'bootstrap/dist/css/bootstrap.min.css';
```

**JavaScript**：
```javascript
import * as bootstrap from 'bootstrap';

// 或只導入需要的模組
import { Modal } from 'bootstrap';
```

## 使用套件管理器安裝

### yarn

```bash
yarn add bootstrap
```

### pnpm

```bash
pnpm add bootstrap
```

### Ruby on Rails

在 Gemfile 中添加：

```ruby
gem 'bootstrap', '~> 5.3'
```

然後運行 `bundle install`。

### Python/Django

```bash
pip install bootstrap5
```

或在 requirements.txt 中添加：

```
django-bootstrap5
```

## 下載安裝

### 下載預先編譯版本

1. 訪問 [Bootstrap 官網](https://getbootstrap.com/)
2. 點擊「Download」按鈕
3. 下載預先編譯的 CSS 和 JS 檔案

### 下載包含原始碼的版本

下載包含 Sass 原始碼的版本：

1. 訪問 [Bootstrap 原始碼](https://github.com/twbs/bootstrap/releases)
2. 下載原始碼壓縮包
3. 解壓後包含：
   - `dist/css/` - 編譯後的 CSS
   - `dist/js/` - 編譯後的 JS
   - `scss/` - Sass 原始碼

### 檔案結構

```
bootstrap/
├── css/
│   ├── bootstrap.css
│   ├── bootstrap.css.map
│   ├── bootstrap.min.css
│   ├── bootstrap.min.css.map
│   ├── bootstrap.rtl.css
│   └── bootstrap.rtl.min.css
├── js/
│   ├── bootstrap.bundle.js
│   ├── bootstrap.bundle.min.js
│   ├── bootstrap.esm.js
│   └── bootstrap.esm.min.js
└── scss/
    └── ...
```

## 驗證安裝

### 基本 HTML 範本

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Bootstrap 測試</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
  <div class="container mt-5">
    <h1>Hello, Bootstrap!</h1>
    <button class="btn btn-primary">主要按鈕</button>
  </div>
  
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
```

### 驗證步驟

1. 將上述代碼儲存為 `.html` 檔案
2. 在瀏覽器中開啟
3. 如果看到具有 Bootstrap 樣式的頁面，表示安裝成功

## 快速開始

### 使用柵格系統

```html
<div class="container">
  <div class="row">
    <div class="col-md-4">列 1</div>
    <div class="col-md-4">列 2</div>
    <div class="col-md-4">列 3</div>
  </div>
</div>
```

### 使用元件

**按鈕**：
```html
<button class="btn btn-primary">主要</button>
<button class="btn btn-secondary">次要</button>
<button class="btn btn-success">成功</button>
<button class="btn btn-danger">危險</button>
<button class="btn btn-warning">警告</button>
<button class="btn btn-info">資訊</button>
```

**卡片**：
```html
<div class="card" style="width: 18rem;">
  <img src="..." class="card-img-top" alt="...">
  <div class="card-body">
    <h5 class="card-title">卡片標題</h5>
    <p class="card-text">卡片內容</p>
    <a href="#" class="btn btn-primary">跳轉連結</a>
  </div>
</div>
```

**彈出視窗**：
```html
<!-- 觸發按鈕 -->
<button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#exampleModal">
  開啟彈出視窗
</button>

<!-- 彈出視窗 -->
<div class="modal fade" id="exampleModal" tabindex="-1">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title">彈出視窗標題</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal"></button>
      </div>
      <div class="modal-body">
        彈出視窗內容
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">關閉</button>
        <button type="button" class="btn btn-primary">儲存</button>
      </div>
    </div>
  </div>
</div>
```

## 常見問題

### Q1: Bootstrap 5 和 Bootstrap 4 有什麼區別？

主要區別：
- Bootstrap 5 移除了 jQuery 依賴
- 使用原生 JavaScript
- 改進的表單元件
- 新的 Utilities API
- 不再支援 IE 瀏覽器

### Q2: 如何自訂 Bootstrap 主題？

**方式一：使用 Sass**
```scss
// 自訂變數
$primary: #ff6600;
$secondary: #cccccc;

// 導入 Bootstrap
@import "bootstrap";
```

**方式二：使用 Bootstrap 自訂工具**
訪問 https://getbootstrap.com/docs/5.3/customize/options/

### Q3: Bootstrap 是否響應式？

是的，Bootstrap 是行動裝置優先的設計：
- `col-` 極小螢幕（<576px）
- `col-sm-` 小螢幕（≥576px）
- `col-md-` 中等螢幕（≥768px）
- `col-lg-` 大螢幕（≥992px）
- `col-xl-` 超大螢幕（≥1200px）
- `col-xxl-` 特大螢幕（≥1400px）

### Q4: 如何使用 Bootstrap 圖示？

```bash
npm install bootstrap-icons
```

或使用 CDN：

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css">
```

使用：
```html
<i class="bi bi-github"></i>
```

### Q5: Bootstrap 5 是否支援 IE11？

不支援。Bootstrap 5 需要現代瀏覽器。如果需要 IE11 支援，請使用 Bootstrap 4.6。

### Q6: 如何調試 Bootstrap 樣式？

1. 使用瀏覽器開發者工具（F12）
2. 檢查元素的類別名稱
3. 查看 Computed 面板中的樣式來源

### Q7: Bootstrap 與其他框架衝突怎麼辦？

- 使用特定的選擇器
- 利用 Bootstrap 的命名空間
- 使用 Bootstrap 提供的 API

### Q8: 如何升級 Bootstrap 版本？

1. 備份目前專案
2. 更新 CDN 連結或 npm 套件版本
3. 檢查破壞性更改
4. 測試所有功能
