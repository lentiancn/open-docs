# Mockito

## Обзор

Mockito — самый популярный фреймворк для мокирования при юнит-тестировании в экосистеме Java, разработанный для тестирования, управляемого тестами (TDD), и разработки, управляемой поведением (BDD). Он позволяет разработчикам создавать макетные объекты, которые имитируют поведение реальных объектов контролируемым образом, обеспечивая независимое и надёжное юнит-тестирование.

## Ключевые функции

Mockito отличается чистым и простым дизайном API, что делает обучение и использование понятным. Мощные функциональные возможности поддерживают проверку вызовов методов, заглушки возвращаемых значений, выброс исключений, сопоставление аргументов и многое другое. Отличная читаемость делает тестовый код интуитивно понятным и лёгким для понимания, с чёткими сообщениями об ошибках при сбое проверок. Активное сообщество обеспечивает постоянную поддержку фреймворка и полную документацию.

## Быстрый старт

### Создание макетного объекта

```java
List<String> mockedList = mock(List.class);
```

### Определение поведения

```java
when(mockedList.get(0)).thenReturn("first");
```

### Проверка взаимодействий

```java
verify(mockedList).get(0);
```

## Требования

Mockito 5.x требует JDK 11 или выше. Mockito 4.x поддерживает JDK 8. Проекты могут использовать Maven или Gradle для управления зависимостями.
