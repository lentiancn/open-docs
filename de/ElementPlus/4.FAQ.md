# Element Plus FAQ

Häufig gestellte Fragen und Antworten.

## Installation

### Q: Installation fehlgeschlagen?

**Lösungen:**
1. Node.js-Version prüfen
```bash
node --version  # Erfordert 18.0+
```
2. npm-Cache leeren
```bash
npm cache clean --force
```
3. node_modules und package-lock.json entfernen, neu installieren
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: Modul nicht gefunden?

**Lösungen:**
1. Prüfen, ob Paket installiert ist
```bash
npm list element-plus
```
2. npm/yarn/pnpm-Konfiguration prüfen
3. Lock-Datei entfernen und neu installieren versuchen

### Q: TypeScript-Typfehler?

**Lösungen:**
1. Typdefinitionen installieren
```bash
npm install -D @types/node
```
2. In tsconfig.json hinzufügen
```json
{
  "compilerOptions": {
    "types": ["element-plus/global"]
  }
}
```

## Verwendung

### Q: Komponente wird nicht angezeigt?

**Lösungen:**
1. korrekten Import prüfen
2. Fehlerkonsole prüfen
3. Vue-Version prüfen (erfordert 3.2+)

### Q: Stile werden nicht angewendet?

**Lösungen:**
1. Prüfen, ob CSS importiert ist
```typescript
import 'element-plus/dist/index.css'
```
2. CSS-Priorität prüfen, `:deep()` oder `!important` verwenden
3. Auf Überschreibungen durch andere Stile prüfen

### Q: Icons werden nicht angezeigt?

**Lösungen:**
1. Icon-Paket installieren
```bash
npm install @element-plus/icons-vue
```
2. Global registrieren
```typescript
import * as icons from '@element-plus/icons-vue'
for (const [key, component] of Object.entries(icons)) {
  app.component(key, component)
}
```
3. Oder bei Bedarf importieren

### Q: Table wird nicht gerendert?

**Lösungen:**
1. data-Eigenschaft korrekt gebunden
2. prop ist korrekt
3. Table ist außerhalb von el-table-column

### Q: Formularvalidierung funktioniert nicht?

**Lösungen:**
1. prop entspricht v-model
2. ref-Referenz vorhanden
3. validate-Methode korrekt aufrufen
```typescript
const formRef = ref()
await formRef.value.validate()
```

### Q: Dialog wird nicht angezeigt?

**Lösungen:**
1. v-model-Bindung prüfen
2. visible ist true
3. Auf z-index-Konflikte prüfen

## Theme

### Q: Theme-Farbe ändern?

**Lösungen:**
CSS-Variablen verwenden:
```css
:root {
  --el-color-primary: #409eff;
}
```

### Q: Dunkles Theme verwenden?

**Lösungen:**
Dunkles CSS importieren:
```typescript
import 'element-plus/theme-chalk/dark/css-vars.css'
```

### Q: Komponentenstile überschreiben?

**Lösungen:**
CSS-Selektoren verwenden:
```css
.el-button--primary {
  background-color: #Ihre-Farbe;
}
```

## Internationalisierung

### Q: Sprache wechseln?

**Lösungen:**
```typescript
import de from 'element-plus/es/locale/lang/de'
import en from 'element-plus/es/locale/lang/en'

app.use(ElementPlus, { locale: de })
```

### Q: Benutzerdefinierter Text?

**Lösungen:**
```typescript
import {ElMessage} from 'element-plus'
ElMessage('Benutzerdefinierte Meldung')
```

## Leistung

### Q: Langsames erstes Laden?

**Lösungen:**
1. Import bei Bedarf verwenden
2. Auto-Import-Plugin verwenden
3. Vite-Pre-Build konfigurieren

### Q: Bundle zu groß?

**Lösungen:**
1. Import bei Bedarf verwenden
2. Tree Shaking verwenden
3. Prüfen, ob vollständiges Paket importiert wird

## Kompatibilität

### Q: IE-Browser nicht unterstützt?

Element Plus unterstützt keinen IE-Browser. Verwenden Sie:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### Q: Konflikte mit anderen Bibliotheken?

**Lösungen:**
1. z-index-Ebenen prüfen
2. CSS Modules verwenden, um Style-Konflikte zu vermeiden
3. Vue-Versionskompatibilität prüfen
