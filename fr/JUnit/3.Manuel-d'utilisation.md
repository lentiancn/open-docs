# JUnit Manuel d'utilisation

Ce manuel couvre l'utilisation de JUnit 4 et JUnit 5, y compris l'écriture de tests, les assertions, les suites de tests et autres fonctionnalités principales.

## Utilisation de base de JUnit 5

### Écrire des classes de test

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {
    
    @Test
    void testAddition() {
        int result = 2 + 3;
        assertEquals(5, result);
    }
}
```

### Assertions courantes

```java
import static org.junit.jupiter.api.Assertions.*;

// Assertions d'égalité
assertEquals(expected, actual);
assertEquals(expected, actual, "Message d'erreur personnalisé");

// Assertions booléennes
assertTrue(condition);
assertFalse(condition);

// Assertions de null
assertNull(object);
assertNotNull(object);

// Assertions de tableaux
assertArrayEquals(expectedArray, actualArray);

// Assertions d'exceptions
assertThrows(Exception.class, () -> {
    // Code qui peut lever une exception
});

// Assertions combinées
assertAll("Propriétés de la personne",
    () -> assertEquals("John", person.getFirstName()),
    () -> assertEquals("Doe", person.getLastName())
);
```

### Opérations Before et After

```java
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;

class MyTest {
    
    @BeforeEach
    void setUp() {
        // S'exécute avant chaque méthode de test
    }
    
    @AfterEach
    void tearDown() {
        // S'exécute après chaque méthode de test
    }
}
```

### Tests imbriqués

```java
import org.junit.jupiter.api.Nested;

@Nested
class NestedTests {
    
    @Test
    void testInNestedClass() {
        // Tests dans la classe de test imbriquée
    }
}
```

### Tests paramétrés

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void testWithMultipleValues(int number) {
    assertTrue(number > 0);
}
```

### Tests dynamiques

```java
import org.junit.jupiter.api.TestFactory;
import org.junit.jupiter.api.DynamicTest;
import java.util.Arrays;

@TestFactory
Collection<DynamicTest> dynamicTestsFromIterator() {
    return Arrays.asList(
        DynamicTest.dynamicTest("Premier test dynamique", () -> assertTrue(true)),
        DynamicTest.dynamicTest("Deuxième test dynamique", () -> assertEquals(4, 2 + 2))
    );
}
```

## Utilisation de base de JUnit 4

### Écrire des classes de test

```java
import org.junit.Test;
import static org.junit.Assert.*;

public class CalculatorTest {
    
    @Test
    public void testAddition() {
        assertEquals(5, 2 + 3);
    }
}
```

### Assertions courantes

```java
// Assertions d'égalité
assertEquals(expected, actual);
assertEquals("message", expected, actual);

// Assertions booléennes
assertTrue(condition);
assertFalse(condition);

// Assertions de null
assertNull(object);
assertNotNull(object);

// Assertions d'exceptions
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}
```

### Before et After

```java
import org.junit.Before;
import org.junit.After;

public class MyTest {
    
    @Before
    public void setUp() {
        // S'exécute avant chaque méthode de test
    }
    
    @After
    public void tearDown() {
        // S'exécute après chaque méthode de test
    }
    
    @BeforeClass
    public static void setUpClass() {
        // S'exécute une fois avant tous les tests
    }
    
    @AfterClass
    public static void tearDownClass() {
        // S'exécute une fois après tous les tests
    }
}
```

### Suites de tests

```java
import org.junit.runner.RunWith;
import org.junit.runners.Suite;
import org.junit.runners.Suite.SuiteClasses;

@RunWith(Suite.class)
@SuiteClasses({Test1.class, Test2.class, Test3.class})
public class AllTests {
    // Exécute tous les tests dans Test1, Test2 et Test3
}
```

## Migration de JUnit 4 vers JUnit 5

### Changements d'annotations

| JUnit 4 | JUnit 5 |
|---------|---------|
| @Test | @Test |
| @Before | @BeforeEach |
| @After | @AfterEach |
| @BeforeClass | @BeforeAll |
| @AfterClass | @AfterAll |
| @Ignore | @Disabled |
| @SuiteClasses | @SelectClasses |

### Changements d'assertions

```java
// JUnit 4
Assert.assertEquals(expected, actual);

// JUnit 5
Assertions.assertEquals(expected, actual);
```

### Changements pour les tests d'exceptions

```java
// JUnit 4
@Test(expected = NullPointerException.class)
public void testException() {
    Object obj = null;
    obj.toString();
}

// JUnit 5
@Test
public void testException() {
    assertThrows(NullPointerException.class, () -> {
        Object obj = null;
        obj.toString();
    });
}
```

## Exécuter les tests

### Ligne de commande

```bash
# Maven
mvn test

# Gradle
gradle test

# Console autonome
java -jar junit-platform-console-standalone-5.14.3.jar -c MyTestClass
```

### Exécution dans l'IDE

- IntelliJ IDEA : Clic droit sur la classe ou méthode de test, sélectionner Run
- Eclipse : Clic droit sur la classe ou méthode de test, sélectionner Run As → JUnit Test
- VS Code : Utiliser l'extension JUnit pour exécuter les tests
