# Docker 常见问题

本页面列出 Docker 安装和使用过程中最常见的问题及解决方案。

## 安装问题

### Docker Desktop 启动失败

**症状**：Docker Desktop 无法启动，或启动后立即崩溃

**解决方案**：

1. 确保系统满足硬件要求：
   - Windows：启用 Hyper-V 或安装 WSL 2
   - macOS：确保未使用过时的 Mac 型号

2. 重置 Docker Desktop：
   - Windows：右键点击 Docker 图标 → Troubleshoot → Reset to factory defaults
   - Mac：Docker → Troubleshoot → Reset to factory defaults

3. 重新安装 Docker Desktop

### WSL2 安装失败

**症状**：Windows 上提示 WSL2 未正确安装

**解决方案**：

1. 启用 Windows 功能：
   - 打开"Windows 功能"
   - 启用"虚拟机平台"和"适用于 Linux 的 Windows 子系统"

2. 安装 WSL2 内核更新包：
   - 下载并安装 WSL2 Linux 内核更新包

3. 将 WSL2 设为默认版本：
   ```bash
   wsl --set-default-version 2
   ```

### Linux 上权限被拒绝

**症状**：运行 docker 命令时提示 "permission denied"

**解决方案**：

1. 将当前用户添加到 docker 组：
   ```bash
   sudo usermod -aG docker $USER
   ```

2. 重新登录或重启系统

3. 验证：
   ```bash
   docker ps
   ```

### 虚拟机问题

**症状**：在虚拟机中运行 Docker 时性能差或无法运行

**解决方案**：

1. 确保虚拟化已启用（在 BIOS 中）
2. 为虚拟机分配足够的 CPU 和内存
3. 使用 Docker Desktop 而非 Docker Engine

## 镜像问题

### 拉取镜像失败

**症状**：`docker pull` 命令超时或失败

**解决方案**：

1. 检查网络连接
2. 使用国内镜像源（配置 Docker 镜像加速器）：
   ```bash
   sudo mkdir -p /etc/docker
   sudo tee /etc/docker/daemon.json <<EOF
   {
     "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
   }
   EOF
   sudo systemctl daemon-reload
   sudo systemctl restart docker
   ```

3. 检查是否有防火墙或代理阻止连接

### 镜像太大

**解决方案**：

1. 使用多阶段构建减小镜像体积
2. 选择更小的基础镜像（如 alpine 版本）
3. 清理构建缓存：
   ```bash
   docker builder prune
   ```

### 清理未使用的镜像

```bash
# 删除悬空镜像（无标签）
docker image prune

# 删除所有未使用的镜像
docker image prune -a

# 删除构建缓存
docker builder prune
```

## 容器问题

### 容器无法启动

**症状**：容器创建后立即停止

**解决方案**：

1. 查看容器日志：
   ```bash
   docker logs <容器名称>
   ```

2. 检查 Dockerfile 中的 CMD 或 ENTRYPOINT 是否正确

3. 确保前台有进程在运行：
   ```bash
   docker run -d nginx tail -f /dev/null
   ```

### 端口冲突

**症状**：启动容器时报错 "port is already allocated"

**解决方案**：

1. 查找占用端口的进程：
   ```bash
   # Linux
   sudo lsof -i :8080
   
   # Windows
   netstat -ano | findstr :8080
   ```

2. 停止占用端口的进程，或使用其他端口

3. 修改容器端口映射：
   ```bash
   docker run -d -p 8081:80 nginx
   ```

### 容器间无法通信

**解决方案**：

1. 确保容器在同一网络：
   ```bash
   docker network ls
   docker network inspect <网络名称>
   ```

2. 使用服务名进行通信（在 Docker Compose 或同一网络中）

3. 检查容器防火墙设置

### 数据丢失

**症状**：容器删除后数据丢失

**解决方案**：

1. 使用数据卷持久化数据：
   ```bash
   docker run -d -v mydata:/app/data myimage
   ```

2. 定期备份数据卷：
   ```bash
   docker run --rm -v mydata:/data -v $(pwd):/backup alpine tar czf /backup/backup.tar.gz /data
   ```

### 容器中文乱码

**解决方案**：

在 Dockerfile 中设置 locale：
```dockerfile
ENV LANG=C.UTF-8 LC_ALL=C.UTF-8
```

或使用包含中文支持的基础镜像。

## 网络问题

### 无法访问外网

**解决方案**：

1. 检查 Docker 网络配置：
   ```bash
   docker network inspect bridge
   ```

2. 检查 DNS 设置：
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

3. 检查防火墙或代理设置

### 代理问题

如果需要通过代理访问外网，配置代理：

```bash
# 创建 Docker 配置目录
sudo mkdir -p /etc/systemd/system/docker.service.d

# 创建代理配置文件
sudo tee /etc/systemd/system/docker.service.d/http-proxy.conf <<EOF
[Service]
Environment="HTTP_PROXY=http://proxy.example.com:80"
Environment="HTTPS_PROXY=http://proxy.example.com:443"
Environment="NO_PROXY=localhost,127.0.0.1"
EOF

# 重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## 性能问题

### Docker 运行缓慢

**解决方案**：

1. 分配更多资源给 Docker Desktop：
   - Docker Desktop → Settings → Resources

2. 移动 Docker 镜像到更大磁盘空间

3. 清理未使用的容器、镜像和卷：
   ```bash
   docker system prune -a
   ```

### 磁盘空间不足

**解决方案**：

1. 查看磁盘使用情况：
   ```bash
   docker system df
   ```

2. 清理构建缓存：
   ```bash
   docker builder prune
   ```

3. 清理未使用的卷：
   ```bash
   docker volume prune
   ```

4. 移动 Docker 数据目录到其他磁盘（需要重新配置）

## Docker Compose 问题

### 服务无法启动

**解决方案**：

1. 查看详细日志：
   ```bash
   docker-compose logs -f <服务名>
   ```

2. 检查 docker-compose.yml 语法

3. 确保所有依赖服务已启动

### 环境变量不生效

**解决方案**：

1. 在 docker-compose.yml 中正确设置 environment：
   ```yaml
   environment:
     - NODE_ENV=production
   # 或
   environment:
     NODE_ENV: production
   ```

2. 检查 .env 文件是否存在

### 端口冲突

和单个容器一样，docker-compose 也可能遇到端口冲突。修改 docker-compose.yml 中的端口映射。

## 卸载问题

### 完全卸载 Docker

**Ubuntu/Debian：**

```bash
# 停止 Docker 服务
sudo systemctl stop docker

# 卸载 Docker 包
sudo apt-get purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 删除数据和配置
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd

# 删除用户组
sudo groupdel docker

# 删除配置文件
sudo rm -rf /etc/docker
```

**macOS：**

```bash
# 停止 Docker
osascript -e 'quit app "Docker"'

# 删除应用
rm -rf /Applications/Docker

# 删除配置和数据
rm -rf ~/Library/Containers/com.docker.docker
rm -rf ~/Library/Application\ Support/Docker
rm -rf ~/Library/Group\ Containers/group.com.docker
```

**Windows：**

通过"设置"→"应用"卸载 Docker Desktop，或使用控制面板卸载。

## 其他问题

### 如何查看 Docker 版本

```bash
docker --version
docker-compose --version
docker-machine --version
```

### 如何获取帮助

- Docker 官方文档：https://docs.docker.com/
- Docker Hub：https://hub.docker.com/
- Docker Community：https://www.docker.com/community/
- 查看命令帮助：`docker --help`

### Docker 命令找不到

确保 Docker 已正确安装并添加到 PATH。检查：

```bash
which docker
echo $PATH
```

如果不在 PATH 中，添加到 shell 配置文件。

### 容器自动重启

使用 `--restart` 选项：

```bash
docker run -d --restart=always nginx
```

重启策略包括：
- `no`：不自动重启
- `on-failure`：失败时重启
- `always`：总是重启
- `unless-stopped`：除非手动停止，否则总是重启

### 限制容器资源

```bash
# 限制内存
docker run -d --memory="512m" nginx

# 限制 CPU
docker run -d --cpus="1.5" nginx

# 限制内存和 CPU
docker run -d --memory="512m" --cpus="1.5" nginx
```
