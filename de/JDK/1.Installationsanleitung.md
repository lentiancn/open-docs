# JDK Installationsanleitung

Diese Anleitung beschreibt, wie Sie JDK (Java Development Kit) auf Windows, macOS und Linux installieren.

## Versionsübersicht

| Version | Typ | Veröffentlichung | Support bis |
|---------|------|--------------|---------------|
| JDK 25 | Neueste | 2026 | 6 Monate |
| JDK 21 | LTS | September 2023 | September 2029 |
| JDK 17 | LTS | September 2021 | September 2029 |
| JDK 11 | LTS | September 2018 | September 2031 |
| JDK 8 | Legacy | März 2014 | Dezember 2030 |

**Empfehlung:** Verwenden Sie JDK 21 (LTS) oder JDK 17 (LTS) für die Produktion. JDK 25 für neueste Funktionen.

## Installation unter Windows

### Option 1: Installer (.exe)

1. JDK von Oracle oder Adoptium herunterladen
2. Installer ausführen
3. "Weiter" durch den Assistenten klicken
4. Installationspfad notieren (Standard: `C:\Program Files\Java\jdk-21`)

### Option 2: ZIP-Archiv

1. ZIP von Oracle oder Adoptium herunterladen
2. An gewünschtem Ort entpacken (z.B. `C:\Java\jdk-21`)
3. Keine Installation erforderlich

### Umgebungsvariablen festlegen

```cmd
# Systemeigenschaften → Erweitert → Umgebungsvariablen

# Systemvariablen → Neu
JAVA_HOME = C:\Program Files\Java\jdk-21

# Path-Variable bearbeiten, hinzufügen:
%JAVA_HOME%\bin
```

### Installation überprüfen

```cmd
java -version
javac -version
```

## Installation unter macOS

### Option 1: DMG-Installer

1. DMG von Oracle oder Adoptium herunterladen
2. DMG-Datei öffnen
3. Paketinstaller doppelklicken
4. Assistent folgen

### Option 2: Homebrew

```bash
# Homebrew installieren, falls nicht vorhanden
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# JDK 21 installieren
brew install openjdk@21

# Mit System-Java verknüpfen
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk

# Zu PATH hinzufügen
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 21)' >> ~/.zshrc
source ~/.zshrc
```

### Option 3: SDKMAN

```bash
# SDKMAN installieren
curl -s "https://get.sdkman.io" | bash
source ~/.sdkman/bin/sdkman-init.sh

# JDK 21 installieren
sdk install java 21.0.5-tem
```

### Installation überprüfen

```bash
java -version
javac -version
```

## Installation unter Linux

### Ubuntu/Debian

```bash
# Paketliste aktualisieren
sudo apt update

# OpenJDK 21 installieren
sudo apt install openjdk-21-jdk

# JAVA_HOME setzen
echo 'export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64' >> ~/.bashrc
source ~/.bashrc
```

### RHEL/CentOS/Fedora

```bash
# JDK 21 installieren
sudo dnf install java-21-openjdk-devel

# JAVA_HOME setzen
echo 'export JAVA_HOME=/usr/lib/jvm/java-21-openjdk' >> ~/.bashrc
source ~/.bashrc
```

### Manuelle Installation (alle Linux)

```bash
# JDK herunterladen
wget https://download.oracle.com/java/21/latest/jdk-21_linux-x64_bin.tar.gz

# Entpacken
sudo tar -xzf jdk-21_linux-x64_bin.tar.gz -C /usr/local/

# JAVA_HOME setzen
echo 'export JAVA_HOME=/usr/local/jdk-21' >> ~/.bashrc
source ~/.bashrc
```

### Installation überprüfen

```bash
java -version
javac -version
```

## Mehrere JDK-Versionen

### JDK unter macOS wechseln

```bash
# Installierte JDKs auflisten
/usr/libexec/java_home -V

# Zu bestimmter Version wechseln
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
```

### JDK unter Linux wechseln

```bash
# update-alternatives verwenden (Ubuntu/Debian)
sudo update-alternatives --config java

# Manuell wechseln
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
```

## IDE-Konfiguration

### IntelliJ IDEA

1. File → Project Structure → Project
2. SDK auswählen: Add JDK → installiertes JDK wählen
3. Apply → OK

### VS Code

1. "Extension Pack for Java" installieren
2. Strg+Umschalt+P → Java: Configure Java Runtime
3. Installiertes JDK auswählen

### Eclipse

1. Window → Preferences → Java → Installed JREs
2. Add → Standard VM
3. Zum JDK-Installationsordner browsen
4. Als Standard festlegen

## Häufige Probleme

### "java" nicht erkannt

- Überprüfen, ob JAVA_HOME korrekt gesetzt ist
- Prüfen, ob PATH `%JAVA_HOME%\bin` enthält
- Terminal/Eingabeaufforderung neu starten

### Port 8080 bereits belegt

```bash
# Windows
netstat -ano | findstr :8080

# Linux/macOS
lsof -i :8080
```

### JAVA_HOME zeigt auf JRE statt JDK

- Sicherstellen, dass JAVA_HOME auf JDK zeigt (enthält `bin/javac`)
- Nicht JRE (enthält nur `bin/java`)
