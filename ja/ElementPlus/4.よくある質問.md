# Element Plus よくある質問

この頁では、Element Plusの使用中に発生する一般的な問題とその解決策を説明します。

## インストール問題

### インストール失敗

**症状**：npm installまたはyarn addが失敗する

**解決策**：

1. Node.jsバージョンが14.0.0以上であることを確認してください
2. npmキャッシュをクリアします：
   ```bash
   npm cache clean --force
   ```
3. node_modulesとpackage-lock.jsonを削除して再インストールします

### TypeScriptタイプエラー

**症状**：コンポーネントをインポートときにタイプエラーが発生する

**解決策**：

1. Volarプラグインがインストールされていることを確認します
2. tsconfig.jsonに追加します：
   ```json
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }
   ```

## 使用問題

### コンポーネントスタイルが機能しない

**症状**：コンポーネントスタイルが期待と一致しない

**解決策**：

1. CSSが正しくインポートされていることを確認します：
   ```javascript
   import 'element-plus/dist-index.css'
   ```
2. グローバルスタイルの上書きを確認します

### アイコンが表示されない

**症状**：アイコンコンポーネントが空白または四角形で表示される

**解決策**：

1. アイコンライブラリをインストールします：
   ```bash
   npm install @element-plus/icons-vue
   ```

2. アイコンをグローバルに登録します：
   ```javascript
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }
   ```

### オンデマンドインポートが機能しない

**症状**：バンドルサイズが大きいまま

**解決策**：

1. プラグイン設定が正しいかどうか確認します
2. 正しいresolverが使用されていることを確認します

### 国際化が機能しない

**症状**：言語を切り替えた後、コンポーネントのテキストが変化しない

**解決策**：

1. 言語パックを正しくインポートします：
   ```javascript
   import zhCn from 'element-plus/es/locale/lang/zh-cn'
   ```
2. localeを設定します：
   ```javascript
   app.use(ElementPlus, { locale: zhCn })
   ```

## コンポーネントの問題

### テーブル問題

#### テーブルソートが機能しない

prop属性が設定されていることを確認します：

```vue
<el-table-column prop="date" sortable />
```

#### テーブルの固定列がずれる

テーブルに固定の高さがあることを確認します：

```vue
<el-table :data="data" height="300" />
```

### フォーム問題

#### フォーム検証がトリガーされない

el-form-itemにprop属性があることを確認します：

```vue
<el-form-item label="名前" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>
```

### ダイアログ問題

#### ダイアログのスクロール問題

ダイアログの内容が多すぎる場合、スクロールを追加するようにします：

```vue
<el-dialog>
  <div style="max-height: 60vh; overflow-y: auto;">
    <!-- 内容 -->
  </div>
</el-dialog>
```

## スタイルの問題

### スタイルの競合

**症状**：カスタムスタイルが上書きされる

**解決策**：

1. より具体的なセレクターを使用します
2. ::v-deepまたは:deep()を使用します

### テーマの色が機能しない

CSS変数が正しく上書きされているか確認します：

```css
:root {
  --el-color-primary: #409eff;
}
```

element-plusのスタイルの後に読み込まれることを確認します。

## パフォーマンスの問題

### ページの遅延

大数据量の場合：

1. el-table-v2（仮想テーブル）を使用します
2. ページネーションを使用してデータを読み込みます
3. テーブルで複雑なコンポーネントを使用しないでください

### バンドルサイズが大きすぎる

1. オンデマンドインポートを使用します
2. 未使用のプラグインを削除します

## Vueバージョン問題

### Vue 2との互換性なし

Element PlusはVue 3のみをサポートします。Vue 2が必要な場合は、Element UIを使用してください。

### Composition APIがサポートされていない

より良いComposition APIサポートを得るために、Vue 3.2+を使用するようにしてください。
