# OpenClaw インストールガイド

OpenClawはmacOS、Linux、Windowsなど、複数のインストール方法をサポートしています。このガイドでは、各インストール方法とその前提条件を詳しく説明します。

---

## システム要件

インストールする前に、システムが以下の要件を満たしていることを確認してください：

- **Node.js**：22.0.0以上
- **オペレーティングシステム**：
  - macOS 10.15 (Catalina) 以上
  - Linux（Ubuntu 20.04+、Debian 11+、Fedora 36+など）
  - Windows 10/11（WSL2推奨）
- **ネットワーク**：npmレジストリとOpenClawサービスにアクセス可能
- **ストレージ**：少なくとも500MBの空き容量

Node.jsバージョンは以下のコマンドで確認できます：

```bash
node --version
```

バージョンが22.0.0未満の場合は、Node.jsをアップグレードしてください。nvm（Node Version Manager）を使用して複数のNode.jsバージョンを管理することをお勧めします。

## インストール方法

### 方法1：公式インストールスクリプト（推奨）

これが最もシンプルで 빠른インストール方法です。公式インストールスクリプトはオペレーティングシステムを自動検出します。

**macOS / Linux：**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

インストールスクリプトは以下の操作を行います：
1. システム環境を検出
2. OpenClawグローバルパッケージをインストール/アップグレード
3. 必要な設定ディレクトリを作成
4. 必要な権限を設定

**Windows（PowerShell）：**

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 方法2：npmグローバルインストール

すでにNode.jsとnpmをインストールしている場合は、npmでインストールできます：

```bash
npm install -g openclaw@latest
```

特定のバージョンを指定することもできます：

```bash
npm install -g openclaw@v2026.2.28
```

利用可能なバージョント）：
- `latest`：最新安定版
- `beta`：ベータ版
- `dev`：開発版

### 方法3：pnpmインストール

OpenClawはpnpmの使用を推奨しています：

```bash
pnpm add -g openclaw@latest
```

### 方法4：Bunインストール

BunをJavaScriptランタイムとして使用している場合：

```bash
bun add -g openclaw@latest
```

### 方法5：ソースからビルド

最新開発版を使用したい、またはプロジェクトに貢献したい場合は、GitHubからクローンしてビルド：

```bash
# リポジトリをクローン
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 依存関係をインストール
pnpm install

# プロジェクトをビルド
pnpm ui:build
pnpm build

# 初期化してサービスをインストール
pnpm openclaw onboard --install-daemon

# 開発モード（自動リロード）
pnpm gateway:watch
```

**注意**：ソースから実行するにはまずビルドが必要です。`pnpm openclaw ...`はTypeScriptを直接実行します（tsx経由）。ビルド後は`node dist/`で実行できます。

## インストール後の設定

### オンボーディングウィザード

インストール完了後、基本設定を行うためにウィザードを実行：

```bash
openclaw onboard --install-daemon
```

ウィザードでは以下をガイドします：
1. APIキーの設定（Anthropic推奨）
2. ワークスペースディレクトリの設定
3. チャネル接続の設定（WhatsApp、Telegram、Discordなど）
4. Gatewayバックグラウンドサービスのインストール

### 手動設定

手動設定のみを行いたい場合は：

```bash
openclaw configure
```

### インストール確認

インストール完了後、以下のコマンドで正常にインストールされたか確認：

```bash
# バージョンを確認
openclaw --version

# Gatewayの状態を確認
openclaw gateway status

# コントロールUIを開く
openclaw dashboard
```

問題がなければ、Gatewayが実行中で、ブラウザでコントロールUIを開けるはずです。

## プラットフォーム別の注意

### macOS

- HomebrewでNode.jsをインストール推奨：`brew install node`
- インストール後、シェル環境を再読み込みする必要がある場合あり
- キーチェーンアクセスを有効にして安全な認証情報保存を推奨

### Linux

- ほとんどのディストリビューションではパッケージマネージャーでNode.js 22+をインストール可能
- Ubuntu/Debian：NodeSourceリポジトリを使用
- build-essentialがインストールされていることを確認（ネイティブモジュールのコンパイル用）

### Windows

- **WSL2を強く推奨**（Windows Subsystem for Linux）
- Windows上で直接実行すると互換性问题が発生する可能性あり
- WSL2を使用すると最佳のエクスペリエンスを得られる

## よくあるインストール問題

### 問題1：npm権限エラー

`EACCES`権限エラーが発生した場合、sudoでnpmパッケージをインストールしないでください。nvmを使用してください：

```bash
# nvmをインストール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# nvmでNode.jsをインストール
nvm install 22
nvm use 22

# OpenClawをインストール
npm install -g openclaw@latest
```

### 問題2：Gatewayが起動しない

ポートが使用中か確認：

```bash
lsof -i :18789
```

ポートが使用中の場合は、設定でポートを変更するか、そのポートを使用しているプロセスを停止してください。

### 問題3：バージョンが古い

インストールしたバージョンが古すぎる場合は、最新バージョンに更新：

```bash
npm update -g openclaw@latest
```

または再インストール：

```bash
npm uninstall -g openclaw
npm install -g openclaw@latest
```

## Dockerインストール（オプション）

Dockerを使用したい場合は、OpenClawは公式Dockerイメージを提供：

```bash
# イメージをプル
docker pull openclaw/openclaw:latest

# コンテナを実行
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  openclaw/openclaw:latest
```

Dockerはクイックテストに適していますが、完全な機能にはホストに直接インストールすることを推奨。

## OpenClawのアップグレード

最新バージョンにアップグレード：

```bash
# 現在のバージョンを確認
openclaw --version

# アップグレード
npm update -g openclaw@latest

# または公式アップグレードスクリプトを使用
openclaw update
```

アップグレード後、診断を実行することを推奨：

```bash
openclaw doctor
```

---

## OpenClawのアンインストール

アンインストールが必要な場合：

```bash
# サービスを停止
openclaw gateway stop

# グローバルパッケージをアンインストール
npm uninstall -g openclaw

# 設定ディレクトリを削除（オプション）
rm -rf ~/.openclaw
```
