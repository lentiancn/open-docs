# Manual de Usuario de Kotlin

## Variables y Tipos Básicos

Kotlin usa dos palabras clave para la declaración de variables: var y val. var declara una variable mutable, similar a una variable regular en Java. val declara una variable inmutable, similar a una variable final en Java. En Kotlin, preferir val es una práctica recomendada que hace el código más seguro y fácil de entender.

Al declarar variables, puedes especificar explícitamente el tipo o dejar que el compilador infiera el tipo:

val name: String = "Kotlin"
val age = 10 // El compilador infiere tipo Int
var count = 0 // Variable mutable

Los tipos básicos de Kotlin incluyen: enteros (Int, Long, Short, Byte), números de punto flotante (Double, Float), caracteres (Char), booleanos (Boolean) y cadenas (String). A diferencia de Java, todos los tipos básicos en Kotlin son objetos con métodos de miembro correspondientes.

Los enteros en Kotlin por defecto son de tipo Int. Para representar números más grandes, usa el sufijo Long:

val longNumber = 100L // Tipo Long

Las cadenas en Kotlin son potentes y soportan plantillas de cadenas, permitiendo que variables o expresiones se incrusten directamente en cadenas:

val name = "World"
println("Hello, $name!") // Produce: Hello, World!
println("Length: ${name.length}") // Produce: Length: 5

## Colecciones

La biblioteca estándar de Kotlin proporciona tipos de colecciones ricos incluyendo Lista (List), Conjunto (Set) y Mapa (Map). Cada tipo de colección tiene versiones mutables e inmutables.

Las colecciones inmutables no pueden agregarse, eliminarse o modificarse después de su creación:

val list = listOf(1, 2, 3, 4, 5)
val set = setOf("a", "b", "c")
val map = mapOf("key1" to 1, "key2" to 2)

Las colecciones mutables permiten modificaciones de elementos:

val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4)
mutableList.removeAt(0)

Las colecciones de Kotlin proporcionan operaciones funcionales ricas como map, filter y reduce:

val numbers = listOf(1, 2, 3, 4, 5)
val doubled = numbers.map { it * 2 } // [2, 4, 6, 8, 10]
val evens = numbers.filter { it % 2 == 0 } // [2, 4]
val sum = numbers.reduce { acc, i -> acc + i } // 15

## Control de Flujo

El control de flujo en Kotlin incluye expresiones condicionales, bucles y expresiones de salto.

La expresión if en Kotlin devuelve un valor y puede asignarse directamente a una variable:

val max = if (a > b) a else b

La expresión when es la herramienta poderosa de Kotlin para coincidencias de múltiples ramas que puede coincidir con varias condiciones:

when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    in 3..10 -> println("x is between 3 and 10")
    else -> println("x is something else")
}

Los bucles en Kotlin incluyen bucles for y while. Los bucles for pueden iterar sobre rangos, arrays y colecciones:

for (i in 1..10) {
    println(i)
}

for (item in list) {
    println(item)
}

for (i in list.indices) {
    println("Index: $i, Value: ${list[i]}")
}

## Funciones

Kotlin usa la palabra clave fun para declarar funciones. Las funciones pueden especificar tipos de parámetros y tipos de retorno:

fun greet(name: String): String {
    return "Hello, $name!"
}

Si el cuerpo de la función tiene solo una expresión, puedes usar la sintaxis de expresión única:

fun greet(name: String) = "Hello, $name!"

Kotlin soporta argumentos por defecto y argumentos nombrados, lo que reduce greatly la necesidad de sobrecarga de funciones:

fun greet(name: String, greeting: String = "Hello") = "$greeting, $name!"

greet("World") // Hello, World!
greet("World", "Hi") // Hi, World!
greet(name = "World", greeting = "Hi") // Hi, World!

Kotlin también soporta parámetros variádicos usando la palabra clave vararg:

fun sum(vararg numbers: Int): Int {
    return numbers.sum()
}

sum(1, 2, 3, 4, 5) // 15

## Clases y Objetos

Las declaraciones de clases en Kotlin son más concisas que en Java. Por defecto, las clases en Kotlin son final y no pueden heredarse. Para hacer una clase heredable, usa la palabra clave open:

open class Animal(val name: String) {
    open fun speak() = println("$name makes a sound")
}

class Dog(name: String) : Animal(name) {
    override fun speak() = println("$name barks")
}

Los constructores de Kotlin incluyen constructores primarios y secundarios. El constructor primario se declara directamente después del nombre de la clase:

class Person(val name: String, val age: Int)

Si necesitas ejecutar lógica en el constructor, usa un bloque init:

class Person(name: String, age: Int) {
    val name: String
    val age: Int

    init {
        this.name = name
        this.age = age
        require(age >= 0) { "Age cannot be negative" }
    }
}

Las clases de datos (data class) de Kotlin generan automáticamente métodos equals, hashCode, toString y copy, lo que las hace perfectas para clases que almacenan datos:

data class User(val name: String, val email: String)

val user1 = User("Alice", "alice@example.com")
val user2 = user1.copy(email = "newemail@example.com")

## Seguridad Nula

La seguridad nula es una de las características más importantes de Kotlin. En Kotlin, el sistema de tipos distingue entre tipos anulables y no anulables, lo que reduce fundamentalmente las NullPointerExceptions.

Por defecto, los tipos no son anulables:

val name: String = "Kotlin" // No puede ser null
val name: String = null // Error de compilación

Para permitir que una variable sea null, usa tipos anulables (?):

val name: String? = null // Puede ser null

Al acceder a propiedades o métodos de tipos anulables, usa el operador de llamada segura (?.) o la aserción no nula (!!):

val length = name?.length // Retorna null si name es null
val length = name!!.length // Lanza excepción si name es null

El operador Elvis (?:) proporciona una forma concisa de manejar valores por defecto:

val length = name?.length ?: 0 // Retorna 0 si name es null

## Funciones de Extensión y Propiedades

Kotlin permite agregar nuevas funciones y propiedades a clases existentes sin heredar o modificar la clase original. Estas se llaman funciones de extensión y propiedades de extensión.

Las funciones de extensión se declaran prefijando el nombre de la función con el nombre de la clase que se extiende:

fun String.addExclamation() = this + "!"

"Hello".addExclamation() // "Hello!"

Las propiedades de extensión también pueden agregar nuevas propiedades a clases existentes:

val String.wordCount: Int
    get() = this.split(" ").size

"Hello World".wordCount // 2

## Funciones de Orden Superior y Lambdas

Las funciones de orden superior son funciones que toman funciones como parámetros o retornan funciones. Las expresiones lambda son la forma principal de implementar esta característica.

La sintaxis de expresión lambda usa llaves:

val sum = { a: Int, b: Int -> a + b }
sum(1, 2) // 3

Si una lambda es el último parámetro de una función, se puede mover fuera de los paréntesis:

list.filter { it > 0 }

En expresiones lambda, it representa el nombre implícito para un solo parámetro. Cuando hay solo un parámetro, puedes omitir la declaración del parámetro.

## Corrutinas

Las corrutinas son la característica central de la programación asíncrona en Kotlin. Comparadas con los hilos, las corrutinas son mucho más ligeras y pueden ejecutar múltiples corrutinas en un solo hilo.

launch es una forma común de iniciar corrutinas, que inicia una nueva corrutina en segundo plano:

GlobalScope.launch {
    val data = async { fetchData() }
    updateUI(data.await())
}

Las funciones suspend solo pueden llamarse dentro de corrutinas:

suspend fun fetchData(): Data {
    return withContext(Dispatchers.IO) {
        // Realizar operación que toma tiempo
    }
}

Las corrutinas de Kotlin proporcionan múltiples dispatchers para controlar en qué hilo se ejecutan las corrutinas: Dispatchers.Main para operaciones de UI, Dispatchers.IO para operaciones de I/O, y Dispatchers.Default para tareas intensivas de CPU.

## Clases Selladas y Enums

Las clases selladas son jerarquías de clases restringidas que limitan el número de subclases que pueden heredar de ella. Todas las subclases deben declararse en el mismo archivo. Esto es muy útil al representar estados finitos o tipos de resultados:

sealed class Result<out T> {
    data class Success<out T>(val data: T) : Result<T>()
    data class Error(val message: String) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

La expresión when puede aprovechar completamente las características de las clases selladas, con el compilador verificando si todos los casos posibles se manejan:

fun handleResult(result: Result<Any>) = when (result) {
    is Result.Success -> println(result.data)
    is Result.Error -> println(result.message)
    is Result.Loading -> println("Loading...")
}

Los enums representan un conjunto finito de valores constantes:

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

Los enums pueden tener propiedades y métodos:

enum class Color(val rgb: Int) {
    RED(0xFF0000),
    GREEN(0x00FF00),
    BLUE(0x0000FF);

    fun containsRed() = (rgb and 0xFF0000) != 0
}

## Interfaces y Clases Abstractas

Las interfaces en Kotlin pueden contener implementaciones por defecto, haciéndolas más flexibles:

interface Clickable {
    fun click()
    fun showOff() = println("I'm clickable!")
}

Las clases pueden implementar múltiples interfaces:

class Button : Clickable, Focusable {
    override fun click() = println("Button clicked")
    override fun showOff() {
        super<Clickable>.showOff()
        println("I'm focusable too!")
    }
}

## Expresiones y Declaraciones de Objetos

Las expresiones de objetos crean instancias de clases anónimas, similares a las clases internas anónimas de Java:

window.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        // Manejar evento de clic
    }
})

Las declaraciones de objetos crean singletons:

object Config {
    val apiUrl = "https://api.example.com"
    val timeout = 30
}

## Objetos Complementarios

Cada clase puede contener un objeto complementario, similar a los miembros estáticos en Java:

class MyClass {
    companion object {
        const val TAG = "MyClass"
        fun create() = MyClass()
    }
}

Los miembros del objeto complementario se pueden acceder directamente a través del nombre de la clase:

MyClass.TAG
MyClass.create()

## Verificaciones y Conversiones de Tipos

El operador is se usa para verificación de tipos, similar a instanceof de Java:

if (obj is String) {
    // En esta rama, obj se convierte automáticamente a String
    println(obj.length)
)

Los casts inteligentes convierten automáticamente tipos que han sido verificados. Si el compilador puede inferir el tipo de la variable en una rama específica, realiza la conversión automáticamente.

Si los casts inteligentes no aplican, puedes usar conversión explícita:

val str = obj as String

La conversión de tipos segura usa as?, que retorna null cuando la conversión falla:

val str = obj as? String // Retorna null si la conversión falla
