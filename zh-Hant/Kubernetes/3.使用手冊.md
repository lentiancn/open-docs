# Kubernetes 使用手冊

## 基本概念

### Pod
Pod 是 Kubernetes 中可部署的最小計算物件，代表叢集中執行的一個或多個容器。通常一個 Pod 包含一個容器，但也可以包含多個緊密耦合的容器。

### Deployment
Deployment 用於管理無狀態應用程式，提供了宣告式更新能力，可以輕鬆實現應用程式的部署、擴展和更新。

### Service
Service 為一組 Pod 提供穩定的網路端點，即使 Pod 的 IP 位址發生變化，Service 也能保持穩定。

### Namespace
Namespace 用於將叢集資源劃分為多個邏輯分區，適合多團隊或多專案共享同一叢集的場景。

## 常用指令

### 叢集管理

```bash
# 查看叢集資訊
kubectl cluster-info

# 查看節點
kubectl get nodes

# 查看所有 Pod
kubectl get pods --all-namespaces

# 查看叢集資源使用情況
kubectl top nodes
kubectl top pods
```

### 應用程式部署

```bash
# 建立 Deployment
kubectl create deployment nginx --image=nginx

# 查看 Deployment
kubectl get deployments

# 擴展/縮減
kubectl scale deployment nginx --replicas=3

# 更新映像檔
kubectl set image deployment/nginx nginx=nginx:1.21

# 回滾
kubectl rollout undo deployment/nginx

# 查看滾動更新狀態
kubectl rollout status deployment/nginx
```

### 服務管理

```bash
# 暴露服務
kubectl expose deployment nginx --port=80 --type=LoadBalancer

# 查看服務
kubectl get services

# 刪除服務
kubectl delete service <service-name>
```

### 偵錯

```bash
# 查看 Pod 日誌
kubectl logs <pod-name>

# 進入容器
kubectl exec -it <pod-name> -- /bin/bash

# 描述資源詳細資訊
kubectl describe pod <pod-name>

# 連接埠轉發用於偵錯
kubectl port-forward <pod-name> 8080:80
```

## 資源清單

Kubernetes 支援使用 YAML 檔案宣告式管理資源。以下是常見的資源定義範例：

### Deployment 範例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
```

### Service 範例

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: ClusterIP
```

### ConfigMap 範例

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  database_url: "postgres://db:5432/myapp"
  cache_enabled: "true"
```

### Secret 範例

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: changeme
```

## 應用程式生命週期管理

### 部署應用程式

```bash
# 使用 YAML 檔案部署
kubectl apply -f deployment.yaml

# 查看部署狀態
kubectl get pods -l app=my-app
```

### 更新應用程式

```bash
# 更新映像檔版本
kubectl set image deployment/my-app my-app=my-app:v2

# 查看更新進度
kubectl rollout status deployment/my-app
```

### 回滾

```bash
# 回滾到上一個版本
kubectl rollout undo deployment/my-app

# 回滾到指定版本
kubectl rollout undo deployment/my-app --to-revision=2

# 查看歷史版本
kubectl rollout history deployment/my-app
```

### 擴展/縮減

```bash
# 手動擴展/縮減
kubectl scale deployment my-app --replicas=5

# 自動擴展/縮減（HPA）
kubectl autoscale deployment my-app --min=2 --max=10 --cpu-percent=80
```

## 儲存管理

### 建立 PersistentVolume

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  storageClassName: standard
  hostPath:
    path: /data/my-app
```

### 建立 PersistentVolumeClaim

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

### 在 Pod 中使用 PVC

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: my-app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

## 配置管理

### 使用 ConfigMap

```bash
# 從檔案建立 ConfigMap
kubectl create configmap my-config --from-file=config.properties

# 從環境變數檔案建立
kubectl create configmap my-config --from-env-file=env.properties

# 在 Pod 中使用
kubectl apply -f pod-with-configmap.yaml
```

### 使用 Secret

```bash
# 從檔案建立 Secret
kubectl create secret generic my-secret --from-file=credentials.json

# 在 Pod 中使用
kubectl apply -f pod-with-secret.yaml
```

## 資源配額和限制

### ResourceQuota

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: my-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    pods: "20"
```

### LimitRange

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: my-limits
spec:
  limits:
  - max:
      cpu: "2"
      memory: 1Gi
    min:
      cpu: 100m
      memory: 128Mi
    default:
      cpu: 500m
      memory: 512Mi
    defaultRequest:
      cpu: 200m
      memory: 256Mi
    type: Container
```

## 監控和日誌

### 監控

```bash
# 使用 kubectl top 查看資源使用
kubectl top nodes
kubectl top pods

# 使用 kubectl describe 查看詳細狀態
kubectl describe node <node-name>
kubectl describe pod <pod-name>
```

### 日誌

```bash
# 查看 Pod 日誌
kubectl logs my-app-pod

# 即時查看日誌
kubectl logs -f my-app-pod

# 查看上一個容器執行個體的日誌
kubectl logs --previous my-app-pod
```
