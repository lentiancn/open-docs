# OpenClaw 常見問題

本頁面解答關於 OpenClaw 的常見問題。

## 基礎問題

### 什麼是 OpenClaw？

OpenClaw 是一個自託管閘道器，將你的聊天應用（WhatsApp、Telegram、Discord 等）連接到 AI 編碼助手。它讓你可以從任何地方透過訊息應用與 AI 助手交流，同時完全掌控自己的資料。

### OpenClaw 免費嗎？

是的，OpenClaw 是開源專案，採用 MIT 授權。但你需要自行承擔：
- 執行所需的伺服器或電腦
- AI 服務的 API 金鑰費用

### 我需要 AI 服務商訂閱嗎？

OpenClaw 支援多種模型服務商，包括：
- Anthropic (Claude)
- OpenAI (GPT)
- Google Gemini
- 以及其他相容 OpenAI API 的服務

你可以選擇按量付費的 API 金鑰，或者使用訂閱認證。

### OpenClaw 支援哪些平台？

- **桌面**：macOS、Linux、Windows
- **行動端**：iOS、Android（作為節點）
- **聊天應用**：WhatsApp、Telegram、Discord、iMessage、Signal、Slack 等 20+ 個平台

## 安裝問題

### 安裝失敗怎麼辦？

1. 檢查 Node.js 版本：`node --version`（需要 22 或更高）
2. 執行診斷：`openclaw doctor`
3. 查看日誌：`openclaw logs`

### 可以在樹莓派上執行嗎？

可以，OpenClaw 可以在樹莓派上執行。建議使用樹莓派 4 或更新型號，並使用 Raspberry Pi OS 或 Ubuntu。

### 需要買 Mac 電腦嗎？

不需要。OpenClaw 可以在任何支援 Node.js 的系統上執行。只有 iMessage 通道需要 Mac 電腦（因為 Apple 的限制）。

## 配置問題

### 配置檔案在哪裡？

預設位置：`~/.openclaw/openclaw.json`

### 如何限制誰可以傳訊息？

透過 `dmPolicy` 配置：

- `pairing`：未知傳送者需要配對碼
- `allowlist`：僅允許清單中的用戶
- `open`：允許所有私人訊息
- `disabled`：停用私人訊息

### 如何設定群組聊天？

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

## 使用問題

### 如何新增新的聊天通道？

```bash
openclaw channels login whatsapp
```

### 可以同時使用多個通道嗎？

可以，一個 Gateway 可以同時連接多個通道。

### 如何讓 AI 記住之前的對話？

OpenClaw 內建記憶功能，會自動儲存對話上下文。你可以透過配置調整對話保留策略。

### 可以執行多個代理嗎？

可以，配置多個代理並設定路由規則：

```json5
{
  agents: {
    list: [
      { id: "home", default: true },
      { id: "work" }
    ]
  }
}
```

## 安全問題

### 我的資料安全嗎？

是的。OpenClaw 是自託管的，所有資料都儲存在你的裝置上，不會傳送到第三方伺服器（除了 AI 服務商的 API）。

### 如何保護我的 Gateway？

1. 設定 `allowFrom` 限制訪問來源
2. 使用強密碼和 API 金鑰
3. 定期更新 OpenClaw 版本
4. 使用防火牆限制連接埠訪問

## 故障排查

### Gateway 啟動失敗

1. 檢查連接埠是否被佔用：`openclaw gateway status`
2. 查看錯誤日誌：`openclaw logs`
3. 執行診斷：`openclaw doctor`

### AI 不回覆訊息

1. 檢查 API 金鑰是否正確配置
2. 確認模型服務商帳戶有足夠餘額
3. 查看日誌中的錯誤訊息

### 訊息傳送失敗

1. 檢查通道是否已登入
2. 確認目標用戶或群組是否存在
3. 查看詳細的錯誤日誌
