# IBM Informix ユーザーマニュアル

## 基本的な管理コマンド

### サーバーの起動と停止

| コマンド | 説明 |
|---------|------|
| `oninit -ivy` | 完全初期化で起動 |
| `oninit` | 通常起動 |
| `onmode -ky` | 即時停止 |
| `onmode -sk` | セーフ停止（チェックポイント待機） |

### 状態確認コマンド

```bash
onstat -                 # サーバー全体の状態
onstat -g seg            # 共有メモリセグメント確認
onstat -g buf            # バッファプール確認
onstat -u                # ユーザーセッション一覧
onstat -d                # チャンク情報
onstat -l                # ロジカルログ情報
onstat -k                # ロック情報
onstat -x                # トランザクション情報
```

## SQL 基本操作

### データベースの作成

```sql
-- 単純なデータベースの作成
CREATE DATABASE testdb;

-- バッファ付きログを持つデータベース
CREATE DATABASE testdb WITH BUFFERED LOG;

-- ログを持つデータベース
CREATE DATABASE testdb WITH LOG;

-- ANSI準拠のデータベース
CREATE DATABASE testdb WITH LOG MODE ANSI;
```

### テーブルの作成

```sql
-- 基本的なテーブル作成
CREATE TABLE employee (
    emp_id INTEGER PRIMARY KEY,
    emp_name VARCHAR(100) NOT NULL,
    dept_id INTEGER,
    salary DECIMAL(10,2),
    hire_date DATE
);

-- 外部キー制約付きテーブル
CREATE TABLE department (
    dept_id INTEGER PRIMARY KEY,
    dept_name VARCHAR(50),
    location VARCHAR(100)
);

ALTER TABLE employee
    ADD CONSTRAINT fk_dept FOREIGN KEY (dept_id)
    REFERENCES department(dept_id);
```

### データの操作

**SELECT（データ取得）**

```sql
-- 全件取得
SELECT * FROM employee;

-- 条件付き取得
SELECT emp_name, salary FROM employee WHERE dept_id = 10;

-- ソート
SELECT * FROM employee ORDER BY salary DESC;

-- 集計関数
SELECT dept_id, COUNT(*) as cnt, AVG(salary) as avg_sal
FROM employee GROUP BY dept_id;

-- 結合
SELECT e.emp_name, d.dept_name
FROM employee e
JOIN department d ON e.dept_id = d.dept_id;

-- サブクエリ
SELECT * FROM employee
WHERE salary > (SELECT AVG(salary) FROM employee);
```

**INSERT（データ挿入）**

```sql
-- 単一行挿入
INSERT INTO employee (emp_id, emp_name, dept_id, salary, hire_date)
VALUES (1, '山田太郎', 10, 500000, '2020-04-01');

-- 複数行挿入
INSERT INTO employee (emp_id, emp_name, dept_id, salary)
VALUES 
    (2, '鈴木一郎', 10, 450000),
    (3, '佐藤花子', 20, 480000);
```

**UPDATE（データ更新）**

```sql
-- 条件付き更新
UPDATE employee
SET salary = salary * 1.05
WHERE dept_id = 10;
```

**DELETE（データ削除）**

```sql
-- 条件付き削除
DELETE FROM employee WHERE emp_id = 1;

-- 全件削除
DELETE FROM employee;
```

## インデックス操作

```sql
-- インデックスの作成
CREATE INDEX idx_emp_name ON employee(emp_name);

-- 複合インデックス
CREATE INDEX idx_emp_dept_sal ON employee(dept_id, salary DESC);

-- インデックス削除
DROP INDEX idx_emp_name;
```

## ビュー操作

```sql
-- ビューの作成
CREATE VIEW emp_view AS
SELECT e.emp_id, e.emp_name, d.dept_name, e.salary
FROM employee e
JOIN department d ON e.dept_id = d.dept_id
WHERE e.salary >= 400000;

-- ビューからの選択
SELECT * FROM emp_view;

-- ビュー削除
DROP VIEW emp_view;
```

## トランザクション管理

```sql
BEGIN WORK;

INSERT INTO employee VALUES (100, 'テスト', 10, 300000);
UPDATE employee SET salary = 350000 WHERE emp_id = 100;

COMMIT WORK;
-- ROLLBACK WORK;
```

## JDBC 接続（Javaアプリケーション）

```java
import java.sql.*;

public class InformixTest {
    public static void main(String[] args) {
        try {
            Class.forName("com.informix.jdbc.IfxDriver");
            
            String url = "jdbc:informix-sqli://localhost:9088/testdb:INFORMIXSERVER=ol_informix1210";
            Connection conn = DriverManager.getConnection(url, "informix", "password");
            
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT * FROM employee");
            
            while (rs.next()) {
                System.out.println(rs.getInt("emp_id") + ": " + rs.getString("emp_name"));
            }
            
            rs.close();
            stmt.close();
            conn.close();
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

## 設定ファイル

### 主要設定ファイル

| ファイル | 説明 |
|---------|------|
| `onconfig` | メインサーバー設定ファイル |
| `sqlhosts` | ネットワーク接続設定 |
| `$INFORMIXDIR/etc/onconfig.std` | 設定ファイルテンプレート |

### 主な設定パラメータ

```
ROOTNAME        rootdbs
ROOTPATH        /opt/informix/storage/rootdbs
ROOTSIZE        200000
DBSERVERNAME    ol_informix
```
