## Einführung in Docker

Docker ist eine Open-Source-Plattform für die Entwicklung, Bereitstellung und Ausführung von Anwendungen. Mit Docker können Sie Anwendungen und ihre Abhängigkeiten in leichte, portable Container verpacken und so "einmal bauen, überall ausführen" erreichen.

### Unterschied zwischen Docker Desktop und Docker Engine

| Funktion | Docker Desktop | Docker Engine |
|------|----------------|---------------|
| Unterstützte Plattformen | Windows, macOS, Linux | Linux |
| GUI | Ja | Nein |
| Enthaltene Komponenten | Docker Engine, Docker CLI, Docker Compose, Kubernetes usw. | Docker Engine, Docker CLI |
| Kommerzielle Nutzung | Erfordert kostenpflichtiges Abonnement für >250 Mitarbeiter oder >$10M Umsatz | Kostenlos (Open Source) |

> **Wichtiger Hinweis**: Die kommerzielle Nutzung von Docker Desktop in größeren Unternehmen (mehr als 250 Mitarbeiter ODER mehr als 10 Millionen US-Dollar Jahresumsatz) erfordert ein kostenpflichtiges Abonnement. Persönliche Nutzung, kleine Unternehmen (weniger als 250 Mitarbeiter UND weniger als 10 Millionen USD Jahresumsatz), Bildung und nicht-kommerzielle Open-Source-Projekte sind kostenlos.

## Versionsinformationen

Docker bietet zwei Update-Kanäle:

- **Stabil**: Bietet die neuesten allgemein verfügbaren Versionen für allgemeine Benutzer
- **Test**: Bietet Vorabversionen zum Testen neuer Funktionen (mit Vorsicht verwenden)

## Installation unter Windows

### Systemanforderungen

#### WSL 2 Backend (Empfohlen)

- Windows 10 64-Bit: Enterprise, Pro oder Education Version 22H2 (Build 19045)
- Windows 11 64-Bit: Enterprise, Pro oder Education Version 23H2 (Build 22631) oder höher
- WSL Version 2.1.5 oder höher
- 64-Bit-Prozessor mit Second Level Address Translation (SLAT)
- 4 GB System-RAM
- Hardware-Virtualisierung im BIOS/UEFI aktiviert

#### Hyper-V Backend

- Windows 10 64-Bit: Enterprise, Pro oder Education Version 22H2 (Build 19045)
- Windows 11 64-Bit: Enterprise, Pro oder Education Version 23H2 (Build 22631) oder höher
- Hyper-V und Container-Windows-Features aktiviert
- 64-Bit-Prozessor mit Second Level Address Translation (SLAT)
- 4 GB System-RAM
- Hardware-Virtualisierung im BIOS/UEFI aktiviert

> **Hinweis**: Docker Desktop wird nicht auf Windows Server (wie Windows Server 2019 oder Windows Server 2022) unterstützt. Um Container auf Windows Server auszuführen, lesen Sie die offizielle Microsoft-Dokumentation.

> **Wichtiger Hinweis**: Das Ausführen von Windows-Containern erfordert Windows 10 oder Windows 11 Professional oder Enterprise. Windows Home oder Education können nur Linux-Container ausführen.

### Installationsschritte

1. **Docker Desktop Installer herunterladen**:
   - [Docker Desktop für Windows - x86_64](https://desktop.docker.com/win/main/amd64/Docker%20Desktop%20Installer.exe)

2. **Doppelklicken Sie auf den Installer, um ihn auszuführen**. Der Standard-Installationspfad ist `C:\Program Files\Docker\Docker`.

3. **Wählen Sie auf der Konfigurationsseite WSL 2 anstelle von Hyper-V verwenden** (oder wählen Sie bei Bedarf Hyper-V).

4. **Befolgen Sie die Anweisungen des Installationsassistenten, um den Installer zu autorisieren und fortzufahren**.

5. **Wenn die Installation erfolgreich ist, wählen Sie Schließen, um den Vorgang abzuschließen**.

6. **Starten Sie Docker Desktop**.

### Administratorrechte

Wenn Ihr Benutzerkonto vom Administratorkonto abweicht, müssen Sie den Benutzer zur Gruppe **docker-users** hinzufügen, um auf Funktionen zuzugreifen, die höhere Privilegien erfordern:

1. Führen Sie **Computerverwaltung** als **Administrator** aus
2. Navigieren Sie zu **Lokale Benutzer und Gruppen** > **Gruppen** > **docker-users**
3. Klicken Sie mit der rechten Maustaste, um den Benutzer zur Gruppe hinzuzufügen
4. Melden Sie sich ab und erneut an, damit die Änderungen wirksam werden

## Installation unter macOS

### Systemanforderungen

#### Apple Silicon (M-Serie Chips)

- macOS 11.0 (Big Sur) oder höher
- Mindestens 4 GB RAM

#### Intel Chips

- macOS 10.15 (Catalina) oder höher
- Mindestens 4 GB RAM

### Installationsschritte

1. **Docker Desktop für Mac herunterladen**:
   - [Apple Silicon (M-Serie)](https://desktop.docker.com/mac/main/arm64/Docker.dmg)
   - [Intel Chips](https://desktop.docker.com/mac/main/amd64/Docker.dmg)

2. **Doppelklicken Sie auf Docker.dmg, um den Installer zu öffnen**.

3. **Ziehen Sie die Docker-App in den Ordner Programme**.

4. **Starten Sie Docker Desktop aus Programme oder Launchpad**.

5. **Akzeptieren Sie den Docker-Abonnementdienstvertrag**.

6. **Warten Sie, bis Docker Desktop initialisiert ist**.

## Installation unter Linux

### Unterstützte Distributionen

| Distribution | x86_64/amd64 | arm64/aarch64 |
|--------------|:------------:|:-------------:|
| Ubuntu | ✅ | ✅ |
| Debian | ✅ | ✅ |
| Fedora | ✅ | ✅ |
| CentOS | ✅ | ✅ |
| RHEL | ✅ | ✅ |

### Ubuntu Installation

1. **Alte Versionen deinstallieren**:
```bash
sudo apt-get remove docker docker-engine docker.io containerd runc
```

2. **apt-Paketindex aktualisieren**:
```bash
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release
```

3. **Offiziellen GPG-Schlüssel von Docker hinzufügen**:
```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
```

4. **Stabiles Repository einrichten**:
```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

5. **Docker Engine installieren**:
```bash
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### Debian Installation

1. **Alte Versionen deinstallieren**:
```bash
sudo apt-get remove docker docker-engine docker.io containerd runc
```

2. **Docker Engine installieren**:
```bash
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### Fedora Installation

```bash
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### CentOS Installation

```bash
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## Docker Engine Installation (Nur CLI)

Statische Binärinstallation für jede Linux-Distribution:

1. **Docker-Binärdateien herunterladen**:
```bash
curl -OL https://download.docker.com/linux/static/stable/x86_64/docker-24.0.7.tgz
```

2. **Entpacken**:
```bash
tar xzf docker-24.0.7.tgz
```

3. **Binärdateien nach PATH verschieben**:
```bash
sudo cp docker/* /usr/local/bin/
sudo rm -rf docker
```

4. **Docker-Daemon starten**:
```bash
sudo dockerd &
```

## Installation überprüfen

Unabhängig vom Betriebssystem überprüfen Sie, ob Docker funktioniert:

```bash
docker --version          # Docker-Version anzeigen
docker-compose --version  # Docker Compose-Version anzeigen
docker run hello-world    # Testcontainer ausführen
```

Wenn Sie die Meldung "Hello from Docker!" sehen, war die Installation erfolgreich.

## Docker deinstallieren

### Windows

1. Windows Einstellungen > Apps öffnen
2. Docker Desktop finden
3. Auf Deinstallieren klicken

### macOS

1. Docker-App aus dem Programme-Ordner entfernen
2. Führen Sie Folgendes aus, um Rückstände zu bereinigen:
```bash
rm -rf ~/Library/Containers/com.docker.docker
rm -rf ~/Library/Application\ Support/Docker
```

### Linux

```bash
sudo apt-get remove docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
```

## Häufig gestellte Fragen

### Q1: Benötigt Docker Desktop eine Zahlung?

Docker Desktop ist kostenlos für:
- Kleine Unternehmen (weniger als 250 Mitarbeiter UND weniger als 10 Millionen USD Jahresumsatz)
- Persönliche Nutzung
- Bildung
- Nicht-kommerzielle Open-Source-Projekte

Kommerzielle Nutzung (mehr als 250 Mitarbeiter ODER mehr als 10 Millionen USD Jahresumsatz) erfordert ein kostenpflichtiges Abonnement.

### Q2: Wie wechselt man zwischen Windows- und Linux-Containern in Docker Desktop?

Aus dem Docker Desktop-Menü auswählen:
- **Switch to Windows containers** - Windows-Container verwenden
- **Switch to Linux containers** - Linux-Container verwenden (Standard)

### Q3: Welche Windows-Versionen unterstützt Docker Desktop?

Docker Desktop unterstützt nur Windows-Versionen, die noch im Microsoft-Servicing-Zyklus sind:
- Windows 10 Enterprise, Pro oder Education 22H2
- Windows 11 Enterprise, Pro oder Education 23H2

### Q4: Wie konfiguriert man den Proxy für Docker Desktop?

Proxy über Docker Desktop Einstellungen > Ressourcen > Proxy konfigurieren.
