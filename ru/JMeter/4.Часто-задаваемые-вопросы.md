# JMeter FAQ

## Общие вопросы

### Что такое Apache JMeter?
Apache JMeter — программное обеспечение с открытым исходным кодом для нагрузочного тестирования и измерения производительности. Это 100% чистое Java-приложение, которое может тестировать функциональное поведение и измерять производительность.

### Какие протоколы поддерживает JMeter?
JMeter поддерживает множество протоколов:
- HTTP/HTTPS
- Веб-службы SOAP/REST
- FTP
- JDBC (базы данных)
- LDAP
- JMS (межплатформенное ПО, ориентированное на сообщения)
- SMTP/POP3/IMAP (почта)
- TCP
- Java-объекты

### JMeter — это браузер?
Нет, JMeter не браузер. Он работает на уровне протокола и не выполняет JavaScript и не отображает HTML-страницы как настоящий браузер.

## Вопросы об установке

### Каковы системные требования?
- Java 8 или выше
- Рекомендуется не менее ОЗУ
- 4 ГБ Совместимая ОС: Windows, Linux, macOS

### Как установить JMeter?
1. Скачайте с https://jmeter.apache.org/
2. Извлеките архивпустите j
3. Заmeter.sh (Linux/Mac) или jmeter.bat (Windows)

### Почему JMeter не запускается?
- Проверьте, установлена ли Java: `java -version`
- Проверьте настройку JAVA_HOME
- Проверьте конфликты портов

### Как увеличить память для JMeter?
Отредактируйте скрипт запуска JMeter и измените переменную HEAP:
```bash
HEAP="-Xms1g -Xmx4g"
```

## Вопросы об использовании

### Как создать базовый тестовый план?
1. Добавьте группу потоков
2. Добавьте HTTP-запрос сэмплер
3. Добавьте слушатели (дерево результатов, сводный отчёт)
4. Запустите тест

### Что такое группа потоков?
Группа потоков определяет группу виртуальных пользователей, которые будут выполнять один и тот же тест. Она контролирует:
- Количество потоков (пользователей)
- Период разгона
- Количество циклов

### Как извлекать данные из ответов?
Используйте экстракторы:
- JSON-экстрактор для JSON-ответов
- XPath-экстрактор для XML/HTML
- Экстрактор регулярных выражений для любого текста

### Как параметризовать тесты?
Используйте переменные и функции:
- `${variable}` для ссылок на переменные
- `${__Random(1,100)}` для случайных значений
- `${__time()}`

### Может ли JMeter обрабатывать аутентификацию?
Да, JMeter поддерживает:
- Базовую аутентификацию
- NTLM-аутентификацию
- Менеджер cookie
- Менеджер авторизации

## Вопросы о производительности

### Сколько пользователей может симулировать JMeter?
Количество зависит от вашего оборудования. При правильной настройке:
- Режим GUI: 100-300 пользователей
- Режим CLI: 1000+ пользователей
- Распределённое тестирование: 10 000+ пользователей

### Что такое распределённое тестирование?
Распределённое тестирование использует несколько машин для создания большей нагрузки. Один главный узел координирует несколько подчинённых машин.

### Как запустить JMeter в режиме headless?
```bash
jmeter -n -t testplan.jmx -l results.jtl
```

## Устранение неполадок

### OutOfMemoryError во время теста
- Увеличить размер кучи
- Уменьшить количество потоков
- Использовать режим CLI вместо GUI
- Отключить ненужные слушатели

### Результаты не отображаются
- Проверить, запущен ли тест
- Проверить, прикреплён ли слушатель к правильному элементу
- Проверить файлы журнала на наличие ошибок

### Высокие времена отклика
- Проверить сеть
- Проверить производительность сервера
- Уменьшить нагрузку, если сервер перегружен

### Как отладить тестовый план?
1. Использовать слушатель дерева результатов
2. Включить отладочное логирование
3. Начать с одного потока
4. Проверить утверждения

## Рекомендации

1. Всегда использовать режим CLI для нагрузочных тестов
2. Начинать с низкой нагрузки и постепенно увеличивать
3. Использовать утверждения для проверки ответов
4. Очищать после тестов (tearDown)
5. Хранить тестовые планы в системе контроля версий
6. Отслеживать системные ресурсы
