# MongoDB 소개

## MongoDB란?

MongoDB는 개발자가 최신 애플리케이션을 더 빨리 구축할 수 있도록 설계된 문서 지향 데이터베이스입니다. MongoDB는 애플리케이션 코드가 데이터를 사용하는 방식처럼 데이터를 모델링하기 쉽게 하는 유연한 JSON 유사 문서에 데이터를 저장합니다. 유연한 스키마를 사용하면 가동 중지 시간 없이 데이터 모델을 발전시키고, 빠르게 반복하며, 균일하지 않은 데이터를 쉽게 처리할 수 있습니다.

MongoDB는 빠른 프로토타이핑에서 대규모 중요 작업 부하까지 모든 것을 지원할 수 있는 강력한 쿼리 엔진, 수평 확장 및 내장된 고가용성을 제공합니다.

MongoDB는 다음을 포함한 다양한 작업 부하 유형을 지원하는  트랜잭션 운영 데이터베이스입니다:

- 문서 기반 구조화 검색 (OLTP)
- 데이터 집계
- 전체 텍스트 검색
- 벡터 검색
- 지리 공간 검색
- 시계열 데이터

## 문서 데이터 모델

MongoDB의 레코드는 문서이며, 필드-값 쌍으로 구성된 데이터 구조입니다. MongoDB 문서는 JSON 객체와 유사합니다. 필드 값에는 다른 문서, 배열 및 문서 배열이 포함될 수 있습니다.

MongoDB의 유연한 객체 지향 데이터 모델을 사용하면 코드베이스의 객체 모델을 반영하는 방식으로 데이터를 구조화할 수 있습니다. 이를 통해 관계형 데이터베이스를 사용할 때 필요한 복잡한 객체-관계 매핑이 필요하지 않습니다.

다음은 MongoDB 문서의 예입니다:

```
{
  _id: ObjectId("507f1f77bcf86cd799439011"),
  name: "Alice",
  birthdate: ISODate("1990-01-01T00:00:00Z"),
  address: {
    street: "123 Main St",
    city: "Springfield",
    state: "IL"
  },
  hobbies: ["reading", "hiking", "coding"]
}
```

## MongoDB 핵심 아키텍처 기능

### 문서 데이터베이스

유연한 문서 데이터 모델을 사용하면 데이터를 애플리케이션 요구 사항에 매핑할 수 있습니다. 기존의 엄격한 행-열 형식을 가진 관계형 데이터베이스와 달리, MongoDB 문서는 중첩 구조를 포함할 수 있어 복잡한 데이터 관계 처리에 이상적입니다.

### 트랜잭션

MongoDB는 여러 읽기 및 쓰기 작업을 단일 전부 또는 전무 이벤트로 실행할 수 있는 다중 문서 트랜잭션을 지원합니다.

### 고가용성

MongoDB의 내장 복제 메커니즘은 자동 장애 조치, 데이터 중복 및 읽기 용량 증가를 제공합니다.

### 수평 확장

MongoDB는 Sharding이라는 기술을 통해 기본적으로 수평 확장을 지원합니다.

## MongoDB 에디션

MongoDB는 두 가지 주요 에디션으로 제공됩니다:

- **Community Edition**: 오픈 소스 및 무료, 개별 개발자 및 소규모 프로젝트에 적합
- **Enterprise Edition**: 고급 보안 및 관리 기능이 포함된 상용 버전

현재 최신 안정 버전은 MongoDB 8.2이며, 장기 지원 버전은 MongoDB 7.0입니다.

## 지원 플랫폼

MongoDB는 다양한 플랫폼과 아키텍처를 지원합니다.

## 사용 사례

MongoDB는 다양한 애플리케이션 시나리오에 적합합니다:

- 콘텐츠 관리 시스템
- 모바일 애플리케이션
- 실시간 분석
- 사물 인터넷
- 전자상거래
- 인공 지능 애플리케이션

## MongoDB와 관계형 데이터베이스 비교

| 기능 | MongoDB | 관계형 데이터베이스 |
|------|---------|-------------------|
| 데이터 모델 | 문서 | 행과 열 |
| 스키마 | 유연 | 고정 |
| 확장 | 수평 | 수직 |
| 트랜잭션 | 다중 문서 ACID | 다중 테이블 ACID |
