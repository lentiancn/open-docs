# Guide d'Installation d'OpenClaw

OpenClaw prend en charge plusieurs méthodes d'installation. Vous pouvez choisir celle qui correspond le mieux à vos besoins.

## Prérequis

- **Node.js** : 22.x ou supérieur
- **Système d'exploitation** : macOS, Linux, Windows (via WSL2)
- **Optionnel** : Tailscale (pour l'accès à distance)

## Installation Rapide

### macOS / Linux

La méthode la plus simple est d'utiliser le script d'installation :

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Ou en utilisant npm/pnpm :

```bash
npm install -g openclaw@latest
# ou
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## Configuration Initiale

Après l'installation, exécutez l'assistant d'onboarding :

```bash
openclaw onboard --install-daemon
```

L'assistant vous guidera à travers :

1. **Configuration d'authentification** : Configurer les clés API ou OAuth
2. **Paramètres Gateway** : Configurer le port et le service gateway
3. **Connexion aux canaux** : Connecter les plateformes de messagerie
4. **Installation des compétences** : Sélectionner et installer les compétences nécessaires

## Installer en Tant que Service Système

Utilisez `--install-daemon` pour installer OpenClaw en tant que service en arrière-plan :

```bash
openclaw onboard --install-daemon
```

Cela crée un service au niveau utilisateur (launchd ou systemd) qui assure qu'OpenClaw s'exécute toujours.

## Exécuter Gateway Manuellement

Si vous préférez ne pas installer en tant que service :

```bash
openclaw gateway --port 18789 --verbose
```

## Installation Docker

Si vous préférez Docker, consultez la documentation de déploiement Docker officielle.

## Compiler depuis la Source

Si vous souhaitez contribuer ou personnaliser OpenClaw :

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

Note : pnpm est utilisé pour la compilation. TypeScript peut s'exécuter directement via tsx.

## Vérifier l'Installation

Après l'installation, vous pouvez :

1. **Vérifier le statut du Gateway** :
   ```bash
   openclaw gateway status
   ```

2. **Ouvrir Control UI** :
   ```bash
   openclaw dashboard
   ```
   Ou visitez `http://127.0.0.1:18789/` dans votre navigateur

3. **Envoyer un message de test** (nécessite la configuration d'un canal) :
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## Mettre à Jour OpenClaw

OpenClaw dispose de trois canaux de publication :

- **stable** : Versions officielles (recommandé)
- **beta** : Versions préliminaires
- **dev** : Builds de développement

Changer de canal :

```bash
openclaw update --channel stable|beta|dev
```

Après la mise à jour, il est recommandé d'exécuter :

```bash
openclaw doctor
```

## Commandes Courantes

| Commande | Description |
|----------|-------------|
| `openclaw onboard` | Exécuter l'assistant de configuration initiale |
| `openclaw gateway start` | Démarrer Gateway |
| `openclaw gateway status` | Vérifier le statut du Gateway |
| `openclaw dashboard` | Ouvrir Control UI |
| `openclaw agent --message "..."` | Envoyer un message à l'assistant |
| `openclaw message send --to xxx --message "xxx"` | Envoyer un message |
| `openclaw doctor` | Diagnostiquer et résoudre les problèmes |
| `openclaw update` | Mettre à jour OpenClaw |

## Variables d'Environnement

Pour des emplacements de configuration personnalisés :

- `OPENCLAW_HOME` : Définit le répertoire principal
- `OPENCLAW_STATE_DIR` : Remplace le répertoire d'état
- `OPENCLAW_CONFIG_PATH` : Remplace le chemin du fichier de configuration

## Dépannage

**L'installation échoue ?**

- Assurez-vous que Node.js >= 22
- Vérifiez la connexion réseau
- Exécutez `openclaw doctor` pour le diagnostic

**Comment désinstaller ?**

- Arrêtez le service Gateway
- Supprimez le répertoire d'installation
- Nettoyez les fichiers de configuration
