# Go 설치 가이드

## 요구 사항

Go는 Linux, macOS 및 Windows를 포함한 여러 운영 체제에서 실행할 수 있습니다. 다른 운영 체제는 다른 설치 방법이 있지만 핵심 요구 사항은 유사합니다. Go 언어 개발 환경을 성공적으로 설치하고 사용하려면 시스템이 다음 기본 요구 사항을 충족하는지 확인하세요.

운영 체제의 경우 Go는 광범위한 플랫폼을 지원합니다. Linux의 경우 Ubuntu, Debian, Fedora, CentOS, RHEL 등 주요 배포판에서 Go를 실행할 수 있습니다. macOS는 macOS 10.13(High Sierra) 이상이 필요합니다. Windows는 Windows 10 이상이 필요하며 Windows 7과 8.1도 지원됩니다(추가 구성이 필요할 수 있음).

하드웨어의 경우 Go의 요구 사항은 상대적으로 낮습니다. Go 설치에는 약 100MB의 디스크 공간이 필요하지만 실제 프로젝트 개발은 프로젝트 크기에 따라 더 많은 공간이 필요할 수 있습니다. 메모리의 경우 2GB RAM이 일상적인 개발에 충분하지만 대규모 컴파일이나 대규모 테스트 실행에는 더 많을 필요할 수 있습니다.

Go 컴파일러 자체에는 특별한 CPU 요구 사항이 없으며 x86-64 및 ARM64와 같은 최신 아키텍처가 잘 작동합니다. Go는 크로스 컴파일을 지원하여 한 플랫폼에서 다른 플랫폼용 바이너리를 컴파일할 수 있습니다.

## 설치 단계

### Linux에 설치

Linux 시스템에서는 공식 제공 tarball을 사용하는 것이 가장 깨끗하고 유연한 방법으로 권장됩니다. 먼저 Go 공식 웹사이트에서 최신 버전 tarball을 다운로드하세요:

```bash
# Go 다운로드 (최신 버전으로 교체)
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# 이전 Go 설치 제거 (있는 경우)
sudo rm -rf /usr/local/go

# /usr/local에 압축 해제
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

다음으로 Go 바이너리 디렉토리를 PATH 환경 변수에 추가하세요. ~/.bashrc 또는 ~/.profile 파일을 편집하고 다음 줄을 추가합니다:

```bash
export PATH=$PATH:/usr/local/go/bin
```

저장 후 구성을 적용하려면 다음 명령을 실행하세요:

```bash
source ~/.bashrc
```

또는 macOS 또는 일부 Linux 배포판의 경우:

```bash
source ~/.profile
```

### macOS에 설치

macOS 시스템에는 두 가지 주요 설치 방법이 있습니다. 첫 번째는 Homebrew 패키지 관리자를 사용하는 것으로, 이미 Homebrew를 구성한 사용자에게 적합합니다:

```bash
# Homebrew로 설치
brew install go

# 설치 확인
go version
```

두 번째 방법은 공식 macOS 설치 프로그램을 다운로드하는 것입니다. Go 공식 웹사이트에서 .pkg 설치 프로그램을 다운로드하고 더블 클릭하여 실행한 후 설치 마법사를 따르세요. 설치 프로그램이 자동으로 환경 변수를 구성합니다.

설치 후 터미널을 열고 설치를 확인하세요:

```bash
go version
```

"go1.21.5 darwin/amd64"와 같은 출력이 표시되어야 합니다.

### Windows에 설치

Windows 시스템에서는 MSI 설치 프로그램을 사용하는 것이 권장됩니다. Go 공식 웹사이트에서 Windows용 MSI 설치 프로그램을 다운로드하고 더블 클릭하여 실행한 후 설치 마법사를 따르세요. 설치 프로그램이 PATH를 포함한 환경 변수를 자동으로 구성합니다.

설치 후 명령 프롬프트 또는 PowerShell를 열고 설치를 확인하세요:

```bash
go version
```

Windows 사용자는 Chocolatey 패키지 관리자를 사용할 수도 있습니다:

```bash
choco install golang
```

### 버전 관리자 사용 (권장)

여러 Go 버전을 동시에 관리해야 하는 개발자에게는 버전 관리자 사용이 권장됩니다. 일반적인 버전 관리자에는 gvm(Go Version Manager)과 asdf-go가 있습니다:

gvm 설치 및 사용:

```bash
# gvm 설치
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# Go 버전 설치
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdf는 Go를 지원하는 다중 언어 버전 관리자입니다:

```bash
# asdf 플러그인 설치
asdf plugin add golang

# Go 버전 설치
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## 설치 확인

설치 후 Go가 올바르게 설치되었는지 확인해야 합니다. 터미널 또는 명령 프롬프트를 열고 다음 명령을 실행하세요:

```bash
# Go 버전 확인
go version

# Go 환경 정보 보기
go env
```

go env 명령은 GOPATH, GOROOT, GOOS, GOARCH 등을 포함한 여러 환경 변수를 출력합니다. 이 정보는 Go의 작동 방식과 환경 구성을 이해하는 데 중요합니다.

전체 개발 환경이 올바르게 작동하는지 확인하기 위해 간단한 프로그램을 실행하세요:

```bash
# 작업 디렉토리 생성
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# 모듈 초기화 (Go 1.16+에서는 go mod 사용 권장)
go mod init hello

# 테스트 프로그램 생성
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# 프로그램 실행
go run main.go
```

"Hello, Go!"가 출력되면 Go 개발 환경이 올바르게 구성되었습니다.

## 환경 구성

### GOPATH 및 모듈

Go 1.11 이전에는 Go가 종속성을 관리하기 위해 GOPATH 모드를 사용했습니다. Go 1.11부터 Go는 모듈 지원을 도입했으며 go mod를 사용하여 프로젝트 종속성을 관리하는 것이 권장됩니다. 모듈을 사용할 때 GOPATH를 설정할 필요가 없으며 어디서나 프로젝트를 만들 수 있습니다.

기업 인트라넷 환경 또는 개인 저장소를 사용할 때에서는 GOPRIVATE 환경 변수를 구성해야 할 수 있습니다:

```bash
export GOPRIVATE=github.com/company/*
```

### 프록시 구성

 등의 지역에서는 Go 공식 모듈 저장소에 대한 직접 액세스가 느리거나 불가능할 수 있습니다. Go 모듈 프록시를 구성할 수 있습니다:

```bash
# 알리바바 클라우드 프록시 사용
go env -w GOPROXY=https://marries.aliyun.com/goproxy/,direct

# 또는 치누 클라우드 프록시 사용
go env -w GOPROXY=https://goproxy.cn,direct
```

### IDE 구성

Go 개발은 일반적으로 VS Code, GoLand 또는 Sublime Text와 같은 편집기를 사용합니다. VS Code는 코드 완성, 디버깅, 서식 지정을 포함한 좋은 개발 경험을 얻기 위해 Go 확장 프로그램을 설치해야 합니다.

GoLand는 JetBrains에서 개발한 전문 Go IDE로, 고급 디버깅, 리팩토링, 테스트를 포함한 가장 완전한 Go 개발 지원을 제공합니다.

## 업그레이드 및 제거

### Go 버전 업그레이드

Go 업그레이드는 간단합니다. 새 버전 설치 프로그램을 다운로드하여 위의 설치 단계에 따라 다시 설치하세요. 새 설치 프로그램이 이전 버전을 덮어씁니다.

버전 관리자 사용자는 다양한 버전 간에 쉽게 전환할 수 있습니다:

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### Go 제거

Linux 또는 macOS에서 Go 제거는 설치 디렉토리만 삭제하면 됩니다:

```bash
# Go 설치 디렉토리 제거
sudo rm -rf /usr/local/go

# PATH에서 제거
# ~/.bashrc 또는 ~/.profile을 편집하고 관련 줄을 제거
```

Windows에서는 제어판의 프로그램 및 기능으로 제거하거나 Chocolatey를 사용하세요:

```bash
choco uninstall golang
```
