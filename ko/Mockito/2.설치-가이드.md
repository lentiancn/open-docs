# Mockito 설치 가이드

## 전제 조건

Mockito를 설치하기 전에 개발 환경이 다음 요구 사항을 충족하는지 확인하세요.

Mockito 5.x는 JDK 11 이상이 필요합니다. 프로젝트에서 JDK 8 지원이 필요한 경우 Mockito 4.x 시리즈를 사용할 수 있습니다. 최신 기능과 보안 수정 프로그램을 얻으려면 최신 Mockito를 사용하는 것이 좋습니다.

## Maven 설치

### 의존성 추가

Maven 프로젝트의 pom.xml 파일에 mockito-core 의존성을 추가하세요. 버전을 관리하기 위해 속성을 사용하여 Mockito 버전을 정의하는 것이 좋습니다.

```xml
<properties>
    <mockito.version>5.14.0</mockito.version>
</properties>

<dependencies>
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <version>${mockito.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

최종 클래스 및 메서드를 모킹하기 위해 Mockito의 인라인 기능을 사용해야 하는 경우 mockito-inline 의존성도 추가해야 합니다.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-inline</artifactId>
    <version>${mockito.version}</version>
    <scope>test</scope>
</dependency>
```

### 스냅샷 버전

Mockito는 모든 코드 변경 후 Sonatype 저장소에 스냅샷 버전을 게시합니다. 최신 개발 기능을 사용해야 하는 경우 pom.xml에 스냅샷 저장소를 추가할 수 있습니다.

```xml
<repositories>
    <repository>
        <id>sonatype-nexus-snapshots</id>
        <url>https://oss.sonatype.org/content/repositories/snapshots</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

## Gradle 설치

### 의존성 추가

Gradle 프로젝트의 build.gradle 파일에 의존성 선언을 추가하세요.

```groovy
dependencies {
    testImplementation 'org.mockito:mockito-core:5.14.0'
}
```

Gradle의 Kotlin DSL 파일(build.gradle.kts)을 사용하는 경우 구문이 약간 다릅니다.

```kotlin
dependencies {
    testImplementation("org.mockito:mockito-core:5.14.0")
}
```

### BOM을 사용한 버전 관리

Spring Dependency Management 플러그인 또는 Gradle의 BOM 기능을 사용하면 버전 관리를 단순화할 수 있습니다.

```groovy
dependencyManagement {
    imports {
        mavenBom 'org.mockito:mockito-bom:5.14.0'
    }
}

dependencies {
    testImplementation 'org.mockito:mockito-core'
}
```

## 수동 다운로드

Maven 또는 Gradle을 사용하지 않는 프로젝트의 경우 Maven Central 저장소에서 JAR 파일을 직접 다운로드할 수 있습니다. Maven Central를 방문하여 org.mockito를 검색하여 mockito-core 및 mockito-all(사용 중단됨)의 모든 버전을 찾으세요.

다운로드 후 JAR 파일을 프로젝트의 클래스패스에 추가하세요. Byte Buddy 및 Objenesis와 같은 종속 라이브러리도 다운로드해야 합니다.

## Spring Boot 통합

Spring Boot의 spring-boot-starter-test에는 이미 Mockito 의존성이 포함되어 있습니다. Spring Boot를 사용하는 프로젝트에서는 일반적으로 Mockito 의존성을 별도로 추가할 필요가 없습니다.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

Spring 환경에서 더 편리하게 Mocks를 사용해야 하는 경우 @MockBean 어노테이션을 사용하여 모 객체를 Spring 컨테이너에 자동으로 주입할 수 있습니다.

## JUnit 5 통합

### MockitoExtension 사용

JUnit 5는 확장 모델을 도입했으며 Mockito는 JUnit 5를 지원하기 위해 MockitoExtension을 제공합니다. mockito-core에 필요한 지원이 이미 포함되어 있으므로 추가 의존성 구성이 필요하지 않습니다.

```java
@ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    private List<String> mockedList;
    
    @Test
    void testSomething() {
        when(mockedList.get(0)).thenReturn("test");
        assertEquals("test", mockedList.get(0));
    }
}
```

### MockitoJUnitRunner 사용

JUnit 4의 경우 MockitoJUnitRunner를 사용하여 모 객체를 자동으로 초기화할 수 있습니다.

```java
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    private MyService myService;
    
    @Test
    public void testSomething() {
        when(myService.getData()).thenReturn("mocked data");
        assertEquals("mocked data", myService.getData());
    }
}
```

## 설치 확인

설치 후 간단한 테스트 클래스를 생성하여 Mockito가 올바르게 구성되었는지 확인할 수 있습니다. 테스트를 실행한 후 통과하면 Mockito가 올바르게 설치된 것입니다.

```java
import static org.mockito.Mockito.*;

public class VerifyInstallation {
    public static void main(String[] args) {
        List<String> mock = mock(List.class);
        when(mock.get(0)).thenReturn("Hello");
        System.out.println(mock.get(0));
        verify(mock).get(0);
        System.out.println("Mockito installed successfully!");
    }
}
```

## 버전 선택 권장 사항

새 프로젝트의 경우 최신 Mockito 5.x 버전을 사용하는 것이 좋습니다. 프로젝트에서 JDK 8 호환성이 필요한 경우 Mockito 4.x 시리즈를 사용할 수 있습니다. 더 나은 성능과 보안을 위해 항상 최신 안정 버전을 사용하세요.
