# JMeter 사용자 매뉴얼

이 매뉴얼은 Apache JMeter를 사용하여 부하 테스트를 생성하고 실행하는 방법에 대한 자세한 지침을 제공합니다.

## 테스트 플랜 구조

JMeter 테스트 플랜은 다음 핵심 구성로 구성됩니다:

### 1. 테스트 플랜

테스트 플랜은 모든 테스트의 루트 노드로, 전체 테스트에 대한 구성 정보를 포함합니다. 다음을 포함할 수 있습니다:
- 사용자 정의 변수
- 전역 속성 설정
- 스레드 그룹 구성

### 2. 스레드 그룹

스레드 그룹은 테스트의 실행 단위로, 다음을 정의합니다:
- **스레드 수**: 시뮬레이션된 사용자 수
- ** ramp-up 시간**: 모든 스레드를 시작하는 데 필요한 시간
- **반복 횟수**: 각 스레드가 테스트를 실행하는 횟수
- **스케줄러**: 테스트 시작 시간 및 지속 시간 설정

### 3. 샘플러

샘플러는 서버에 요청을 전송합니다. 일반적인 샘플러:

- **HTTP 요청**: HTTP/HTTPS 요청 전송
- **FTP 요청**: FTP 서버 테스트
- **JDBC 요청**: 데이터베이스 연결 테스트
- **LDAP 요청**: LDAP 디렉토리 테스트
- **JMS 메시지**: 메시지 큐 테스트
- **TCP 샘플러**: TCP 서비스 테스트

### 4. 로직 컨트롤러

로직 컨트롤러는 요청의 실행 순서를 제어합니다:

- **심플 컨트롤러**: 요청을 그룹화
- **루프 컨트롤러**: 요청 실행 반복
- **원스 온리 컨트롤러**: 테스트 중 한 번만 실행
- **인터리브 컨트롤러**: 요청 사이를 번갈아가며
- **랜덤 컨트롤러**: 실행할 요청을 무작위로 선택
- **스루풋 컨트롤러**: 요청 실행 빈도 제어

### 5. 리스너

리스너는 테스트 결과를 보고 분석합니다:

- **결과 트리 보기**: 각 요청의 상세 응답 보기
- **집계 보고서**: 집계 성능 통계 보기
- **그래프 결과**: 테스트 데이터를 그래픽으로 표시
- **응답 시간 그래프**: 응답 시간 추세 표시

### 6. 구성 요소

구성 요소는 기본값 및 공유 데이터를 설정합니다:

- **HTTP 요청 기본값**: HTTP 요청의 기본 매개변수 설정
- **사용자 정의 변수**: 재사용 가능한 변수 정의
- **CSV 데이터 세트 구성**: CSV 파일에서 테스트 데이터 읽기
- **HTTP 쿠키 관리자**: 쿠키 관리
- **HTTP 헤더 관리자**: 요청 헤더 정보 설정

### 7. 타이머

타이머는 요청 사이의 지연을 설정합니다:

- **고정 타이머**: 고정 지연 설정
- **가우시안 랜덤 타이머**: 랜덤 지연 설정
- **동기화 타이머**: 동시 요청 시뮬레이션
- **스루풋 타이머**: 요청 처리량 제어

### 8. 전처리기 및 후처리기

- **전처리기**: 요청 전송 전에 작업 실행
- **후처리기**: 응답 수신 후 데이터 추출

## 첫 번째 테스트 플랜 만들기

### 1단계: 테스트 플랜 생성

1. JMeter 실행
2. "파일" -> "새로 만들기"를 선택하여 새 테스트 플랜 생성
3. 테스트 플랜 이름 지정

### 2단계: 스레드 그룹 추가

1. 테스트 플랜에서 마우스 오른쪽 버튼 클릭
2. "추가" -> "스레드 (사용자)" -> "스레드 그룹" 선택
3. 스레드 수(예: 10), ramp-up 시간(예: 5초), 반복 횟수(예: 1) 설정

### 3단계: HTTP 요청 추가

1. 스레드 그룹에서 마우스 오른쪽 버튼 클릭
2. "추가" -> "샘플러" -> "HTTP 요청" 선택
3. 서버 이름 또는 IP, 포트 번호, 요청 경로 등 설정

### 4단계: 리스너 추가

1. 스레드 그룹 또는 HTTP 요청에서 마우스 오른쪽 버튼 클릭
2. "추가" -> "리스너" -> "결과 트리 보기" 선택
3. 성능 통계를 보려면 "집계 보고서" 추가

### 5단계: 테스트 실행

1. 도구 모음에서 "시작" 버튼 클릭
2. 리스너에 표시된 결과 데이터 관찰

## 매개변수화 및 데이터 기반 테스트

### CSV 데이터 파일 사용

1. "CSV 데이터 세트 구성" 요소 추가
2. 파일 이름 및 변수 이름 설정
3. 요청에서 ${variableName}으로 데이터 참조

### 사용자 정의 변수 사용

1. "사용자 정의 변수" 요소 추가
2. 변수 이름 및 값 정의
3. 요청에서 ${variableName} 사용

## 상관관계 처리

### 정규식 추출기 사용

요청에 Session ID와 같은 동적 데이터가 필요한 경우:

1. "정규식 추출기" 추가
2. 일치시킬 정규식 패턴 설정
3. 후속 요청에서 추출된 변수 사용

### JSON 추출기 사용

JSON 응답의 경우:

1. "JSON 추출기" 추가
2. JSON 경로 표현식 설정
3. 필요한 JSON 필드 값 추출

## 분산 테스트

단일 시스템에서 충분한 부하를 생성할 수 없는 경우 분산 테스트를 사용할 수 있습니다:

### 마스터 구성

1. 마스터 bin 디렉토리에서 jmeter.properties 편집
2. remote_hosts를 슬레이브 주소로 설정
3. JMeter 시작

### 슬레이브 구성

1. 각 슬레이브 시스템에서 jmeter-server 실행
2. 지정된 포트에서 통신을 허용하도록 방화벽 확인

### 분산 테스트 시작

1. 마스터에서 "원격 시작" 선택
2. 시작할 슬레이브 선택 또는 모두 시작

## 성능 최적화 팁

- GUI 리소스 소비를 줄이기 위해 명령줄 모드 사용
- 즉각적인 높은 부하를 피하기 위해 적절한 스레드 수 및 ramp-up 시간 설정
- 불필요한 리스너 끄기, 필요할 때만 결과 보기
- 응답 내용을 확인하기 위해 어설션 사용
- 사용하지 않는 테스트 플랜 정기적으로 정리
- 대규모 테스트의 경우 분산 테스트 고려
