# Введение

Apache APISIX — это проект верхнего уровня Apache Software Foundation, разработанный и переданный сообществу компанией API7.ai. Это облачный API-шлюз с динамическим, реальновременным и высокопроизводительным ядром.

APISIX выступает точкой входа для трафика всех ваших сервисов и предоставляет массу полезных фишек: динамическую маршрутизацию, динамические upstream-серверы, динамические сертификаты, A/B-тестирование, поэтапные релизы (canary), blue-green деплой, ограничение скорости, защиту от атак, сбор метрик, мониторинг с оповещениями, observability и управление сервисами.

## Ключевые возможности

### Высокая производительность

APISIX построен на базе Nginx и унаследовал от него отличную производительность. Благодаря плагинной архитектуре бизнес-логика отделена от базового функционала — это даёт гибкость в расширении, но при этом не снижает скорость обработки запросов.

### Полная динамика

В отличие от традиционного Nginx, которому для применения настроек нужна перезагрузка, APISIX поддерживает полностью динамическое управление конфигурацией. Любые маршруты, upstream-серверы, плагины можно обновлять через управляющий API на лету — без перезапуска сервиса.

### Облачная архитектура

APISIX с самого начала проектировался для облачных сценариев и поддерживает разные способы деплоя: Docker, Kubernetes, Helm Chart. Есть встроенный Ingress Controller, так что его можно использовать напрямую как входной шлюз в Kubernetes.

### Экосистема плагинов

В APISIX есть богатая система плагинов на все случаи жизни: аутентификация, ограничение скорости, переписывание запросов, обработка ответов, мониторинг и логирование. Через плагины можно легко расширять функциональность шлюза.

### Поддержка множества протоколов

Помимо классических HTTP/HTTPS, APISIX работает с gRPC, Dubbo, WebSocket, MQTT и другими протоколами. Это позволяет закрыть самые разные задачи в микросервисной архитектуре.

## Архитектура

Архитектура APISIX состоит из двух частей:

**Data Plane (плоскость данных)** отвечает за реальную маршрутизацию запросов. APISIX Core, построенный на основе Nginx, обеспечивает высокоскоростную обработку запросов.

**Control Plane (плоскость управления)** управляет всей конфигурацией через RESTful API и поддерживает разные способы её хранения — etcd, Consul и другие. Также есть удобный Dashboard с веб-интерфейсом для визуального управления.

## Когда это нужно

- Единая точка входа для API микросервисов
- Kubernetes Ingress Controller
- Управление и мониторинг API-трафика
- Мультитенантное API-управление
- Sidecar-прокси в service mesh
- API-шлюз для AI-приложений (LLM и AI-рабочие нагрузки)

Будучи проектом верхнего уровня Apache Software Foundation, APISIX имеет активное сообщество и регулярные обновления — это надёжный выбор для enterprise-приложений.
