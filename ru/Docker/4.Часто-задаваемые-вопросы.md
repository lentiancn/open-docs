# Часто задаваемые вопросы о Docker

Эта страница содержит распространённые проблемы и решения для установки и использования Docker.

## Проблемы с установкой

### Docker Desktop не запускается

**Симптомы**: Docker Desktop не запускается или аварийно завершается сразу после запуска

**Решения**:

1. Убедитесь, что система соответствует аппаратным требованиям:
   - Windows: включите Hyper-V или установите WSL 2
   - Mac: убедитесь, что не используете устаревшие модели Mac

2. Сбросьте Docker Desktop:
   - Windows: щёлкните правой кнопкой мыши значок Docker → Troubleshooting → Reset to factory defaults
   - Mac: Docker → Troubleshoot → Reset to factory defaults

3. Переустановите Docker Desktop

### Ошибка установки WSL2

**Симптомы**: В Windows появляется сообщение о том, что WSL2 установлен неправильно

**Решения**:

1. Включите функции Windows:
   - Откройте «Компоненты Windows»
   - Включите «Платформа виртуальной машины» и «Подсистема Windows для Linux»

2. Установите пакет обновления ядра WSL2:
   - Скачайте и установите пакет обновления ядра WSL2 для Linux

3. Установите WSL2 как версию по умолчанию:
   ```bash
   wsl --set-default-version 2
   ```

### Отказано в доступе на Linux

**Симптомы**: «permission denied» при выполнении команд docker

**Решения**:

1. Добавьте текущего пользователя в группу docker:
   ```bash
   sudo usermod -aG docker $USER
   ```

2. Выйдите из системы и войдите снова или перезагрузите систему

3. Проверьте:
   ```bash
   docker ps
   ```

### Проблемы с виртуальной машиной

**Симптомы**: Низкая производительность или невозможность запустить Docker в виртуальных машинах

**Решения**:

1. Убедитесь, что виртуализация включена (в BIOS)
2. Выделите достаточное количество CPU и памяти для виртуальной машины
3. Используйте Docker Desktop вместо Docker Engine

## Проблемы с образами

### Ошибка загрузки образов

**Симптомы**: команда `docker pull` превышает время ожидания или завершается ошибкой

**Решения**:

1. Проверьте сетевое подключение
2. Используйте зеркальные источники (настройте зеркала реестра Docker):
   ```bash
   sudo mkdir -p /etc/docker
   sudo tee /etc/docker/daemon.json <<EOF
   {
     "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
   }
   EOF
   sudo systemctl daemon-reload
   sudo systemctl restart docker
   ```

3. Проверьте, не блокирует ли брандмауэр или прокси соединение

### Образ слишком большой

**Решения**:

1. Используйте многоэтапную сборку для уменьшения размера образа
2. Выбирайте меньшие базовые образы (например, версии alpine)
3. Очистите кэш сборки:
   ```bash
   docker builder prune
   ```

### Очистка неиспользуемых образов

```bash
# Удалить висячие образы (без тегов)
docker image prune

# Удалить все неиспользуемые образы
docker image prune -a

# Удалить кэш сборки
docker builder prune
```

## Проблемы с контейнерами

### Контейнер не запускается

**Симптомы**: Контейнер останавливается сразу после создания

**Решения**:

1. Просмотрите логи контейнера:
   ```bash
   docker logs <имя контейнера>
   ```

2. Проверьте, правильные ли CMD или ENTRYPOINT в Dockerfile

3. Убедитесь, что есть запущенный процесс на переднем плане:
   ```bash
   docker run -d nginx tail -f /dev/null
   ```

### Конфликты портов

**Симптомы**: Ошибка «port is already allocated» при запуске контейнера

**Решения**:

1. Найдите процесс, использующий порт:
   ```bash
   # Linux
   sudo lsof -i :8080
   
   # Windows
   netstat -ano | findstr :8080
   ```

2. Остановите процесс, использующий порт, или используйте другой порт

3. Измените сопоставление портов контейнера:
   ```bash
   docker run -d -p 8081:80 nginx
   ```

### Контейнеры не могут общаться

**Решения**:

1. Убедитесь, что контейнеры в одной сети:
   ```bash
   docker network ls
   docker network inspect <имя сети>
   ```

2. Используйте имена сервисов для общения (в Docker Compose или одной сети)

3. Проверьте настройки брандмауэра контейнера

### Потеря данных

**Симптомы**: Данные теряются после удаления контейнера

**Решения**:

1. Используйте тома для сохранения данных:
   ```bash
   docker run -d -v mydata:/app/data myimage
   ```

2. Регулярно создавайте резервные копии томов:
   ```bash
   docker run --rm -v mydata:/data -v $(pwd):/backup alpine tar czf /backup/backup.tar.gz /data
   ```

### Проблемы с кодировкой в контейнере

**Решения**:

Установите локаль в Dockerfile:
```dockerfile
ENV LANG=C.UTF-8 LC_ALL=C.UTF-8
```

Или используйте базовые образы с поддержкой китайского языка.

## Проблемы с сетью

### Нет доступа к внешней сети

**Решения**:

1. Проверьте конфигурацию сети Docker:
   ```bash
   docker network inspect bridge
   ```

2. Проверьте настройки DNS:
   ```bash
   docker run --dns 8.8.8.8 alpine ping -c 3 google.com
   ```

3. Проверьте настройки брандмауэра или прокси

### Проблемы с прокси

Если вам нужен прокси для доступа к внешним сетям, настройте прокси:

```bash
# Создайте директорию конфигурации Docker
sudo mkdir -p /etc/systemd/system/docker.service.d

# Создайте файл конфигурации прокси
sudo tee /etc/systemd/system/docker.service.d/http-proxy.conf <<EOF
[Service]
Environment="HTTP_PROXY=http://proxy.example.com:80"
Environment="HTTPS_PROXY=http://proxy.example.com:443"
Environment="NO_PROXY=localhost,127.0.0.1"
EOF

# Перезапустите Docker
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## Проблемы с производительностью

### Docker работает медленно

**Решения**:

1. Выделите больше ресурсов для Docker Desktop:
   - Docker Desktop → Settings → Resources

2. Переместите образы Docker на диск с большим пространством

3. Очистите неиспользуемые контейнеры, образы и тома:
   ```bash
   docker system prune -a
   ```

### Недостаточно места на диске

**Решения**:

1. Проверьте использование диска:
   ```bash
   docker system df
   ```

2. Очистите кэш сборки:
   ```bash
   docker builder prune
   ```

3. Очистите неиспользуемые тома:
   ```bash
   docker volume prune
   ```

4. Переместите директорию данных Docker на другой диск (требует перенастройки)

## Проблемы с Docker Compose

### Сервисы не запускаются

**Решения**:

1. Просмотрите подробные логи:
   ```bash
   docker-compose logs -f <имя сервиса>
   ```

2. Проверьте синтаксис docker-compose.yml

3. Убедитесь, что все зависимые сервисы запущены

### Переменные среды не применяются

**Решения**:

1. Правильно установите переменные среды в docker-compose.yml:
   ```yaml
   environment:
     - NODE_ENV=production
   # или
   environment:
     NODE_ENV: production
   ```

2. Проверьте, существует ли файл .env

### Конфликты портов

Как и для одиночных контейнеров, docker-compose также может столкнуться с конфликтами портов. Измените сопоставление портов в docker-compose.yml.

## Проблемы с удалением

### Полное удаление Docker

**Ubuntu/Debian:**

```bash
# Остановите службу Docker
sudo systemctl stop docker

# Удалите пакеты Docker
sudo apt-get purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Удалите данные и конфигурацию
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd

# Удалите группу пользователей
sudo groupdel docker

# Удалите файлы конфигурации
sudo rm -rf /etc/docker
```

**macOS:**

```bash
# Остановите Docker
osascript -e 'quit app "Docker"'

# Удалите приложение
rm -rf /Applications/Docker

# Удалите конфигурацию и данные
rm -rf ~/Library/Containers/com.docker.docker
rm -rf ~/Library/Application\ Support/Docker
rm -rf ~/Library/Group\ Containers/group.com.docker
```

**Windows:**

Удалите Docker Desktop через Settings → Apps или используйте Панель управления.

## Другие вопросы

### Как проверить версию Docker

```bash
docker --version
docker-compose --version
docker-machine --version
```

### Как получить помощь

- Официальная документация Docker: https://docs.docker.com/
- Docker Hub: https://hub.docker.com/
- Сообщество Docker: https://www.docker.com/community/
- Просмотр справки по командам: `docker --help`

### Команда Docker не найдена

Убедитесь, что Docker правильно установлен и добавлен в PATH. Проверьте:

```bash
which docker
echo $PATH
```

Если нет в PATH, добавьте в файл конфигурации оболочки.

### Автоматический перезапуск контейнеров

Используйте параметр `--restart`:

```bash
docker run -d --restart=always nginx
```

Политики перезапуска включают:
- `no`: без автоматического перезапуска
- `on-failure`: перезапуск при сбое
- `always`: всегда перезапускать
- `unless-stopped`: всегда перезапускать, если не остановлен вручную

### Ограничение ресурсов контейнера

```bash
# Ограничить память
docker run -d --memory="512m" nginx

# Ограничить CPU
docker run -d --cpus="1.5" nginx

# Ограничить память и CPU
docker run -d --memory="512m" --cpus="1.5" nginx
```
