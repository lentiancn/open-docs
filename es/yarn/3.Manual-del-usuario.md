# Manual de Usuario de Yarn

## Comandos Básicos

### Comandos Generales

- `yarn add` - Agregar dependencias al proyecto
- `yarn install` - Instalar las dependencias del proyecto
- `yarn remove` - Eliminar dependencias del proyecto
- `yarn run` - Ejecutar un script definido en package.json
- `yarn init` - Crear un nuevo paquete
- `yarn info` - Ver información relacionada con paquetes
- `yarn why` - Mostrar la razón por la que se necesita un paquete

### Comandos Avanzados

- `yarn dlx` - Ejecutar un paquete en un entorno temporal
- `yarn dedupe` - Eliminar duplicados de dependencias con rangos superpuestos
- `yarn up` - Actualizar dependencias en todo el proyecto
- `yarn pack` - Generar un tarball desde el workspace activo
- `yarn rebuild` - Reconstruir los paquetes nativos del proyecto

### Comandos Relacionados con NPM

- `yarn npm audit` - Realizar una auditoría de vulnerabilidades contra los paquetes instalados
- `yarn npm publish` - Publicar el workspace activo en el registro npm
- `yarn npm login` - Almacenar nueva información de inicio de sesión para acceder al registro npm
- `yarn npm logout` - Cerrar sesión en el registro npm
- `yarn npm whoami` - Mostrar el nombre del usuario autenticado

### Gestión de Plugins

- `yarn plugin list` - Listar los plugins oficiales disponibles
- `yarn plugin import` - Descargar un plugin
- `yarn plugin remove` - Eliminar un plugin
- `yarn plugin runtime` - Listar los plugins activos

## Workspaces

Yarn fue el primer gestor de paquetes construido específicamente alrededor de workspaces, permitiéndote dividir tu proyecto en subcomponentes.

### Configuración de Workspaces

Configura workspaces en tu `package.json` raíz:

```json
{
  "workspaces": [
    "packages/*"
  ]
}
```

### Comandos de Workspace

- `yarn workspaces list` - Listar todos los workspaces
- `yarn workspace <name> <command>` - Ejecutar un comando en un workspace específico

### Referencias Cruzadas

Los paquetes de monorepos a menudo necesitan depender unos de otros. Yarn lo hace muy fácil gracias al protocolo especial `workspace:`:

```json
{
  "dependencies": {
    "@my-org/utils": "workspace:^"
  }
}
```

El protocolo `workspace:` acepta ya sea un rango semver regular, o los tokens especiales `^` / `~` / `*`.

### Instalaciones Enfocadas

Una preocupación común al descubrir workspaces es cómo necesitas instalar todas sus dependencias cada vez que deseas trabajar en uno solo de ellos. Yarn proporciona una solución mediante `yarn workspaces focus`.

Este comando toma una lista de workspaces, extiende la lista para incluir dependencias transitivas, y excluye todo lo demás de la instalación:

```bash
yarn workspaces focus @my-org/app
```

### Ejecución Paralela

Los scripts de múltiples workspaces se pueden ejecutar en paralelo si comparten el mismo nombre, usando `yarn workspaces foreach`:

```bash
yarn workspaces foreach --all -pt npm publish
```

## Archivos de Configuración

### .yarnrc.yml

Yarn v2+ usa el archivo `.yarnrc.yml` para configuración:

```yaml
nodeLinker: pnp
yarnPath: .yarn/releases/yarn-4.0.0.cjs
```

### .yarnrc (Yarn v1)

Yarn v1 usa el archivo `.yarnrc`:

```
--registry https://registry.npmjs.org
--strict-semver true
```

## Diferencias entre Versiones

### Yarn v4+ vs Yarn v1

| Característica | Yarn v4+ | Yarn v1 |
|----------------|----------|---------|
| Arquitectura | Basada en plugins (Berry) | Tradicional |
| Velocidad de Instalación | Más rápida (instalaciones paralelas) | Velocidad estándar |
| Espacio en Disco | Más eficiente (PnP) | node_modules estándar |
| Workspaces | Soporte nativo | Requiere configuración adicional |
| Sistema de Plugins | Soporte integrado | No soportado |
| Configuración | .yarnrc.yml | .yarnrc |

### Consideraciones de Migración

- Yarn v4+ usa PnP por defecto, lo que puede ser incompatible con algunas herramientas
- Las configuraciones de CI/CD necesitan actualizarse para adaptarse al nuevo flujo de instalación
- El sistema de plugins proporciona más opciones de personalización
- La funcionalidad de workspaces es más potente y flexible
