# Руководство по установке GitLab

## Требования

Перед установкой GitLab убедитесь, что ваш сервер соответствует следующим требованиям к аппаратному и программному обеспечению. Для небольших и средних команд рекомендуется минимум 4 ядра ЦП и 8 ГБ оперативной памяти, что может поддерживать сотни репозиториев и десятки одновременных пользователей. Более крупные команды или сценарии с высокой нагрузкой требуют больше ресурсов, предпочтительно 8+ ядер ЦП и 16 ГБ+ оперативной памяти.

Что касается операционных систем, GitLab поддерживает различные дистрибутивы Linux, включая Ubuntu, Debian, CentOS, Red Hat Enterprise Linux и Fedora. Официально рекомендуется Ubuntu или Debian, так как управление пакетами в них наиболее удобно. GitLab требует минимум 20 ГБ дискового пространства для установки, рекомендуется 50 ГБ+ для базовых конфигураций, а для производственных сред часто требуется несколько сотен ГБ.

Программные зависимости включают OpenSSH Server для поддержки протокола SSH, базу данных PostgreSQL (GitLab включает встроенную версию) и SMTP-сервер для почтовых уведомений. Также необходимо настроить правильный часовой пояс и имя хоста, а также убедиться, что сервер имеет нормальный доступ в Интернет для загрузки пакетов зависимостей.

## Шаги установки

### Установка в Ubuntu/Debian

В системах Ubuntu или Debian рекомендуется установка через официальный репозиторий apt, так как это обеспечивает получение последних стабильных версий и автоматических обновлений безопасности. Сначала установите необходимые пакеты зависимостей:

```bash
sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
```

Добавьте репозиторий APT GitLab и установите GitLab CE (Community Edition):

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" apt-get install gitlab-ce
```

Замените EXTERNAL_URL на фактический домен или IP-адрес для доступа к GitLab. Процесс установки автоматически настроит PostgreSQL, Redis и сам GitLab.

### Установка в CentOS/RHEL

В CentOS или Red Hat Enterprise Linux можно установить GitLab с помощью менеджера пакетов yum или dnf. Сначала установите необходимые зависимости:

```bash
sudo yum install -y curl policycoreutils openssh-server openssh-clients
sudo systemctl enable sshd
sudo systemctl start sshd
```

Добавьте репозиторий YUM GitLab и установите:

```bash
curl -fsSL https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo EXTERNAL_URL="https://gitlab.example.com" yum install -y gitlab-ce
```

## Начальная настройка

После установки необходимо настроить основные параметры GitLab. Отредактируйте файл /etc/gitlab/gitlab.rb, который является основным файлом конфигурации GitLab. Наиболее важным параметром конфигурации является external_url — внешний адрес доступа к сервису GitLab:

```ruby
external_url 'https://gitlab.example.com'
```

Если необходимо использовать пользовательский порт SSH, настройте параметр gitlab_rails['gitlab_shell_ssh_port']. Настройка почтового сервера также важна для отправки уведомлений и писем для сброса пароля:

```ruby
gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.example.com"
gitlab_rails['smtp_port'] = 587
gitlab_rails['smtp_user_name'] = "user@example.com"
gitlab_rails['smtp_password'] = "yourpassword"
gitlab_rails['smtp_domain'] = "example.com"
```

После сохранения конфигурации выполните следующую команду для применения изменений:

```bash
sudo gitlab-ctl reconfigure
```

Эта команда перенастроит все компоненты GitLab, включая миграции базы данных и создание начального пользователя. Весь процесс может занять несколько минут.

## Проверка установки

После завершения настройки вы можете получить доступ к адресу, указанному в external_url, через браузер. Если всё правильно, вы должны увидеть страницу входа в GitLab. При первом доступе вы увидите страницу входа в учётную запись администратора. Имя пользователя администратора по умолчанию — root, при первом входе нужно будет установить пароль.

GitLab предоставляет инструмент командной строки gitlab-ctl для управления состоянием сервиса:

```bash
# Проверить состояние сервиса
sudo gitlab-ctl status

# Просмотреть журналы
sudo gitlab-ctl tail

# Перезапустить сервис
sudo gitlab-ctl restart
```

Также можно использовать команду gitlab-rake для проверки работоспособности:

```bash
sudo gitlab-rake gitlab:env:info
sudo gitlab-rake gitlab:check
```

## Установка через Docker

Для быстрого тестирования или небольших развёртываний можно запустить GitLab с помощью Docker. Сначала установите Docker Engine, затем выполните следующую команду на хосте:

```bash
docker run --detach \
  --hostname gitlab.example.com \
  --publish 443:443 --publish 80:80 --publish 22:22 \
  --name gitlab \
  --restart always \
  --volume /srv/gitlab/config:/etc/gitlab \
  --volume /srv/gitlab/logs:/var/log/gitlab \
  --volume /srv/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest
```

При установке через Docker монтируются три тома для сохранения конфигурации, журналов и данных. Первый запуск контейнера Docker требует времени на инициализацию. После терпеливого ожидания можно получить доступ через браузер.

## Обновление и обслуживание

Регулярное обновление GitLab до последней версии обеспечивает новые функции и исправления безопасности. Обновление в Ubuntu/Debian:

```bash
sudo apt-get update
sudo apt-get install gitlab-ce
```

Рекомендуется создавать резервную копию данных перед обновлением. GitLab предоставляет встроенный инструмент резервного копирования:

```bash
sudo gitlab-backup create
```

Файлы резервных копий по умолчанию сохраняются в каталоге /var/opt/gitlab/backups, можно настроить политики хранения резервных копий и расположения удалённого хранилища.
