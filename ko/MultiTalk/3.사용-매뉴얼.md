# MultiTalk 사용 매뉴얼

이 매뉴얼에는 기본 작업, 매개변수 구성 및 고급 기술을 포함한 MultiTalk 사용 방법에 대한 자세한 지침이 포함되어 있습니다.

## 기본 사용법

### 입력 파일 준비

MultiTalk는 특정 입력 형식을 준비해야 합니다. 프로젝트는 오디오 경로, 이미지 경로 및 텍스트 프롬프트를 포함한 생성 매개변수를 정의하기 위해 JSON 파일을 사용합니다.

단일 인물 생성을 위한 입력 JSON 형식에는 오디오 파일 경로, 참조 이미지 경로, 비디오 저장 이름 및 기타 매개변수 지정이 필요합니다. 예제 파일은 examples 디렉토리에 있으며 single_example_1.json과 같습니다.

다인물 대화 생성을 위한 입력 JSON에는 여러 오디오 스트림과 해당 인물 영역 정의가 필요합니다. 예제 파일은 examples 디렉토리에 있으며 multitalk_example_2.json과 같습니다.

### 기본 생성 실행

설치 및 구성이 완료되면 프로젝트 디렉토리로 이동하여 비디오 생성을 위해 다음 명령을 실행합니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### 매개변수 설명

--ckpt_dir는 모델 가중치 디렉토리 경로를 지정합니다. --wav2vec_dir는 오디오 인코더 디렉토리 경로를 지정합니다. --input_json은 입력 매개변수 JSON 파일 경로를 지정합니다. --sample_steps는 샘플링 단계 수를 지정합니다. 값이 클수록 품질이 향상되지만 속도가 느려집니다. --mode는 생성 모드를 지정하며, streaming은 긴 비디오용, clip은 짧은 비디오용입니다. --use_teacache는 TeaCache 가속을 활성화합니다. --save_file은 출력 파일 저장 이름을 지정합니다.

## 고급 사용 기술

### 생성 품질 조정

샘플링 단계는 생성 품질에 영향을 주는 핵심 매개변수입니다. 기본값 40단계는 좋은 결과를 제공합니다. 더 빠른 생성을 위해 단계를 10까지 줄일 수 있으며, 이는 입술 동기화에 미미한 영향을 주지만 움직임의성과 시각적 품질에 영향을 줍니다. 단계를 늘리면 세부 사항 표현이 향상됩니다.

입술 동기화 강도는 오디오 안내 계에 의해 제어됩니다. Audio CFG 값은 3~5 사이에서 가장 잘 작동합니다. 이 값을 높이면 더 나은 입술 동기화 효과를 얻을 수 있습니다.

### 긴 비디오 생성

MultiTalk는 최대 15초까지 비디오 생성을 지원합니다. 더 긴 비디오 클립을 생성하려면 streaming 모드를 사용합니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

매우 긴 비디오를 생성할 때 세그먼트 간에 색조 차이가 있을 수 있습니다. Audio CFG를 3으로 설정하면 색조 일관성을 유지하는 데 도움이 됩니다.

### 저 VRAM 작동

VRAM이 제한된 경우 다음 매개변수를 사용하여 메모리 사용량을 줄일 수 있습니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

--num_persistent_param_in_dit 0 매개변수를 추가하면 단일 RTX 4090에서 480P 비디오 생성이 가능해집니다.

### 가속 활성화

프로젝트는 생성 속도를 개선하기 위한 여러 가속 방법을 지원합니다.

TeaCache 가속은 품질을 유지하면서 속도를 크게 향상시킬 수 있습니다. --teacache_thresh의 권장 값은 0.2~0.5 사이입니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### LoRA 가속 사용

LoRA를 사용하면 샘플링 단계를 크게 줄일 수 있습니다. FusioniX LoRA는 8단계만 필요하며, lightx2v LoRA는 4단계만 필요합니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Gradio 인터페이스 사용

프로젝트는 대화형 사용을 위한 Gradio 웹 인터페이스를 제공합니다. 프로젝트 디렉토리로 이동하여 다음을 실행합니다:

```bash
python app.py
```

또는 저 VRAM 버전을 사용합니다:

```bash
python app.py --num_persistent_param_in_dit 0
```

## TTS 음성 합성 통합

MultiTalk는 직접 텍스트 입력을 지원합니다. 통합된 TTS 기술을 통해 텍스트에서 음성을 생성한 다음 영상을 생성합니다:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## 다중 GPU 구성

720P 고해상도 영상 생성에는 여러 GPU가 필요합니다. 다중 GPU를 구성하려면 다음 명령을 사용합니다:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## ComfyUI 사용

프로젝트는 ComfyUI 워크플로우로의 통합도 지원합니다. ComfyUI-WanVideoRunner 확장을 설치한 다음 워크플로우 오케스트레이션을 위해 MultiTalk 모델을 로드해야 합니다.
