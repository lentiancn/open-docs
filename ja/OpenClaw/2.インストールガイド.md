# OpenClaw インストールガイド

このガイドでは、不同なOSにOpenClawをインストールする手順を説明します。

## 動作環境

- **Node.js**: バージョン22以上
- **OS**: macOS、Linux、Windows
- **オプション**: Docker（サンドボックス機能を使用する場合）

## インストール方法

### macOS と Linux（おすすめ）

公式インストールスクリプトを使うのが一番簡単です：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

このスクリプトは自動でシステム環境を検出し、必要な依存関係をインストールしてOpenClawを設定します。

### Windows

PowerShellで以下を実行：

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### npmで直接インストール

すでにNode.jsがセットアップされている場合：

```bash
npm install -g openclaw@latest
```

### ソースからインストール

最新の開発バージョンを使用する場合：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## インストール後の設定

### セットアップウィザードを実行

インストール後、基本設定を行うためにウィザードを実行することをお勧めします：

```bash
openclaw onboard --install-daemon
```

ウィザードで行うこと：
- APIキーの設定
- サーポート番号の設定
- チャンネルのログイン（WhatsApp、Telegram、Discordなど）
- セキュリティとアクセス制御の設定

### サービスの状態を確認

上記の手順でインストールした場合、サービスはすでに起動しているはずです：

```bash
openclaw gateway status
```

### フォアグラウンドで実行

ターミナルでログを直接確認したい場合（デバッグに便利）：

```bash
openclaw gateway --port 18789
```

### 管理画面を開く

```bash
openclaw dashboard
```

またはブラウザで直接アクセス：`http://127.0.0.1:18789/`

## 環境変数

データの保存場所をカスタマイズしたい場合：

- `OPENCLAW_HOME`: OpenClawのホームディレクトリを指定
- `OPENCLAW_STATE_DIR`: ステートデータの保存ディレクトリをオーバーライド
- `OPENCLAW_CONFIG_PATH`: 設定ファイルのパスを指定

## インストール確認

インストール後、以下を実行して確認できます：

1. `openclaw --version` でバージョンを確認
2. `openclaw doctor` でシステムの健常性をチェック
3. 管理画面を開いてテストメッセージを送信

## よくある問題

### Node.jsバージョンの確認

```bash
node --version
```

バージョンが22以上であることを確認してください。低い場合はNode.jsをアップグレードしてください。

### WindowsでGitが見つからない

GitがインストールされPATHに追加されていることを確認してください。git-scm.comからダウンロードできます。

### 権限エラー

権限エラーが発生した場合は、以下を試してください：

```bash
sudo npm install -g openclaw@latest
```
