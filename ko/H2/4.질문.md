# H2 데이터베이스 자주 묻는 질문

## 설치

### Q: H2 데이터베이스는 어떻게 다운로드합니까?

A: https://www.h2database.com/html/download.html에서 플랫폼에 맞는 버전을 다운로드하세요. 두 가지 옵션이 있습니다: Windows 설치 프로그램과 크로스플랫폼 ZIP 패키지.

### Q: H2에 필요한 Java 버전은 무엇입니까?

A: H2는 Java 11 이상이 필요합니다.

### Q: Maven 프로젝트에서 H2는 어떻게 사용합니까?

A: pom.xml에 종속성을 추가하세요:
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### Q: ZIP을 추출한 후 H2는 어떻게 실행합니까?

A: 명령줄에서 실행하세요:
```bash
java -jar h2.jar
```
이렇게 하면 H2 웹 콘솔이 시작됩니다.

## 연결

### Q: H2 데이터베이스에는 어떻게 연결합니까?

A: JDBC 연결을 사용하세요:
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### Q: H2의 기본 사용자 이름과 비밀번호는 무엇입니까?

A: 기본 사용자 이름은 "sa"이고 비밀번호는 비어 있습니다.

### Q: 파일 데이터베이스에는 어떻게 연결합니까?

A:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### Q: H2 TCP 서버는 어떻게 시작합니까?

A:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### Q: 포트가 이미 사용 중이면 어떻게 합니까?

A: 연결 URL에서 포트를 지정할 수 있습니다:
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## 데이터 문제

### Q: 인메모리 데이터베이스 데이터가 손실되나요?

A: 네, 인메모리 데이터베이스(jdbc:h2:mem:)의 데이터는 모든 연결이 닫히면 손실됩니다. 지속성을 위해 파일 데이터베이스 모드를 사용하세요.

### Q: 데이터베이스 파일 위치는 어떻게 확인합니까?

A: 연결 URL에서 경로를 지정하세요:
```java
jdbc:h2:file:/path/to/database
```

### Q: 데이터베이스 파일이 손상되면 어떻게 해야 합니까?

A: 복구 모드를 시도하세요:
```java
jdbc:h2:file:./testdb;LOG=0
```
또는 백업에서 복원하세요.

### Q: 데이터베이스는 어떻게 암호화합니까?

A:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## 성능

### Q: 일괄 삽입 성능은 어떻게 개선합니까?

A: 일괄 작업을 사용하고 자동 커밋을 비활성화하세요:
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (int i = 0; i < 1000; i++) {
    pstmt.setString(1, "User" + i);
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### Q: 쿼리가 느리면 어떻게 합니까?

A:
1. 일반적인 쿼리 조건에 인덱스 생성
2. EXPLAIN을 사용하여 쿼리 계획 분석
3. SELECT *를 피하고 필요한 열만 쿼리

### Q: 높은 메모리 사용량은 어떻게 처리합니까?

A: JVM 힙 크기를 조정하고 연결 풀을 적절히 사용하세요.

## SQL 호환성

### Q: H2는 어떤 데이터베이스 모드를 지원합니까?

A: H2는 여러 데이터베이스를 시뮬레이션할 수 있습니다:
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

사용법:
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### Q: 자동 증가 ID는 어떻게 처리합니까?

A: H2는 여러 방법을 지원합니다:
```sql
-- 방법 1: IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- 방법 2: SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## 트랜잭션

### Q: 트랜잭션이 커밋되지 않으면 어떻게 됩니까?

A: 커밋되지 않은 트랜잭션은 연결이 닫힐 때 자동으로 롤백됩니다.

### Q: 데드락은 어떻게 처리합니까?

A: H2는 데드락을 자동으로 감지하고 하나의 트랜잭션을 롤백합니다.locks를 오래 유지하지 않도록 트랜잭션 범위를 줄이는 것이 좋습니다.

## 보안

### Q: H2 데이터베이스는 안전합니까?

A: H2는 개발 및 테스트 환경에 적합합니다. 프로덕션의 경우:
1. 암호화된 데이터베이스 사용
2. 강력한 비밀번호 사용
3. 네트워크 액세스 제한
4. SSL 연결 사용

### Q: SQL 주입은 어떻게 방지합니까?

A: 문자열 연결 대신 PreparedStatement를 사용하세요:
```java
// 올바른 방법
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// 잘못된 방법
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## 도구

### Q: H2 콘솔은 어떻게 사용합니까?

A: `java -jar h2.jar -web`를 실행한 후 http://localhost:8082를 방문하세요

### Q: H2 셸은 어떻게 사용합니까?

A: `java -jar h2.jar -shell` 또는 `java -jar h2.jar -tool`을 실행하세요

### Q: H2 버전은 어떻게 확인합니까?

A:
```bash
java -jar h2.jar -version
```
또는 SQL에서: `SELECT H2VERSION();`

## 마이그레이션

### Q: MySQL에서 H2로 마이그레이션은 어떻게 합니까?

A:
1. H2 호환성 모드 설정: `jdbc:h2:file:db;MODE=MySQL`
2. MySQL 데이터를 SQL 스크립트로 내보내기
3. H2에서 스크립트 실행

### Q: H2 버전을 업그레이드할 때 무엇을 알아야 합니까?

A: H2 1.x에서 2.x로 업그레이드하려면 특별한 주의가 필요합니다. 공식 마이그레이션 가이드를 참조하세요:
https://www.h2database.com/html/migration-to-v2.html
