# OpenClaw Installationsanleitung

Diese Anleitung bietet detaillierte Installationsschritte für OpenClaw auf verschiedenen Betriebssystemen.

## Anforderungen

- **Node.js**: Version 22 oder höher
- **Betriebssystem**: macOS, Linux oder Windows
- **Optional**: Docker (für Sandbox-Funktionalität)

## Installationsmethoden

### macOS / Linux (Empfohlen)

Der einfachste Weg, OpenClaw zu installieren, ist das Installationsskript:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Dieses Skript erkennt automatisch deine Systemumgebung, installiert notwendige Abhängigkeiten und konfiguriert OpenClaw.

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Direkte npm-Installation

Wenn du Node.js bereits installiert hast, kannst du auch npm direkt verwenden:

```bash
npm install -g openclaw@latest
```

### Installation aus dem Quellcode

Wenn du die neueste Entwicklungsversion möchtest oder eine benutzerdefinierte Installation benötigst:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Nach der Installation

### Onboarding-Assistent ausführen

Nach der Installation führe den Assistenten aus, um Authentifizierung, Gateway-Einstellungen und optionale Kanäle zu konfigurieren:

```bash
openclaw onboard --install-daemon
```

Der Assistent führt dich durch:
- API-Schlüssel-Konfiguration
- Gateway-Port-Einstellungen
- Kanalverbindungen (WhatsApp, Telegram, Discord, etc.)
- Sicherheits- und Berechtigungseinstellungen

### Gateway-Status prüfen

Wenn du den Dienst wie oben gezeigt installiert hast, sollte das Gateway bereits laufen:

```bash
openclaw gateway status
```

### Gateway im Vordergrund starten

Wenn du das Gateway im Vordergrund ausführen möchtest (nützlich für Tests oder Fehlerbehebung):

```bash
openclaw gateway --port 18789
```

### Control UI öffnen

```bash
openclaw dashboard
```

Oder öffne direkt im Browser: `http://127.0.0.1:18789/`

## Umgebungsvariablen

Wenn du den Konfigurations- oder Statusverzeichnisort anpassen musst, kannst du diese Umgebungsvariablen verwenden:

- `OPENCLAW_HOME`: Legt das Home-Verzeichnis für interne Pfadauflösung fest
- `OPENCLAW_STATE_DIR`: Überschreibt das Statusverzeichnis
- `OPENCLAW_CONFIG_PATH`: Überschreibt den Konfigurationsdateipfad

## Installation verifizieren

Nach der Installation kannst du:

1. `openclaw --version` ausführen, um die Version zu bestätigen
2. `openclaw doctor` ausführen, um die Systemgesundheit zu prüfen
3. Das Control UI öffnen, um eine Testnachricht zu senden

## Häufige Installationsprobleme

### Node.js-Version prüfen

```bash
node --version
```

Stelle sicher, dass die Version 22 oder höher ist. Falls nicht, aktualisiere Node.js zuerst.

### Git auf Windows nicht gefunden

Stelle sicher, dass Git installiert und zu deinem System-PATH hinzugefügt ist. Du kannst es von [git-scm.com](https://git-scm.com) herunterladen.

### Berechtigungsfehler

Wenn du während der Installation auf Berechtigungsfehler stößt, versuche:

```bash
sudo npm install -g openclaw@latest
```
