# Руководство пользователя OpenClaw

Это руководство поможет вам эффективно использовать OpenClaw.

## Основное использование

### Общение с OpenClaw

После установки вы можете взаимодействовать с OpenClaw несколькими способами:

1. **Control UI**: Откройте `http://127.0.0.1:18789/` в браузере
2. **CLI-команда**: `openclaw agent --message "Ваше сообщение"`
3. **Настроенные каналы**: Отправляйте сообщения через WhatsApp, Telegram, Discord и т.д.

### Отправка сообщений

Отправляйте сообщения через CLI:

```bash
openclaw message send --to +1234567890 --message "Hello from OpenClaw"
```

## Основные концепции

### Gateway

Gateway — это основная панель управления OpenClaw, управляющая всеми сеансами, каналами, инструментами и событиями. По умолчанию слушает: `ws://127.0.0.1:18789`.

### Сеанс

OpenClaw использует модель сеансов для управления разговорами:

- **Основной сеанс**: Прямой чат
- **Изоляция групп**: Групповые сообщения обрабатываются отдельно
- **Режим активации**: Контролирует, когда отвечать
- **Стратегия ответа**: Определяет, как отвечать

### Рабочее пространство

Рабочее пространство — это место, где OpenClaw хранит конфигурацию и данные:

- Расположение по умолчанию: `~/.openclaw/workspace`
- Содержит: AGENTS.md, SOUL.md, TOOLS.md
- Каталог навыков: `~/.openclaw/workspace/skills/`

## Настройка каналов

### Поддерживаемые каналы

OpenClaw поддерживает эти мессенджеры:

- Мгновенные сообщения: WhatsApp, Telegram, Signal, iMessage
- Социальные платформы: Discord, Slack
- Корпоративные инструменты: Microsoft Teams, Google Chat, Feishu
- Другие: IRC, Matrix, LINE, Mattermost, Nostr и т.д.

### Примеры настройки

**Telegram**:

```json
{
  "channels": {
    "telegram": {
      "botToken": "ВАШ_BOT_TOKEN"
    }
  }
}
```

**Discord**:

```json
{
  "channels": {
    "discord": {
      "token": "ВАШ_BOT_TOKEN"
    }
  }
}
```

**WhatsApp**:

```bash
openclaw channels login
```

## Система навыков

Навыки — это способ расширения функциональности OpenClaw. Вы можете:

### Установить навыки

Установите навыки сообщества из ClawHub или создайте свои собственные.

### Структура навыков

Каждый навык — это каталог с файлом SKILL.md:

```
skills/
  └── my-skill/
      └── SKILL.md
```

### ClawHub

ClawHub — это реестр навыков. При включении OpenClaw может автоматически находить и устанавливать необходимые навыки.

## Автоматизация

### Cron-задачи

Настройте запланированные задачи с помощью Cron:

```bash
openclaw cron add "0 9 * * *" "Morning briefing"
```

### Webhooks

Настройте webhook для запуска автоматизации:

```bash
openclaw webhook add --url https://ваш-endpoint.com/hook
```

### Интеграция с Gmail

Настройте Gmail Pub/Sub для получения новых писем.

## Расширенные функции

### Управление браузером

OpenClaw может управлять выделенным браузером:

```json
{
  "browser": {
    "enabled": true,
    "color": "#FF4500"
  }
}
```

### Визуализация Canvas

Canvas — это визуальное рабочее пространство OpenClaw с поддержкой A2UI push/reset, оценки и снимков.

### Голосовые функции

- **Voice Wake**: Активация голосовыми командами на macOS/iOS
- **Talk Mode**: Непрерывный голосовой разговор (поддержка Android)

### Узлы устройств

OpenClaw может подключаться к различным устройствам:

- **macOS-узел**: Системные команды, уведомления, Canvas, камера
- **iOS-узел**: Canvas, голос, камера, запись экрана
- **Android-узел**: Подключение, чат, голос, Canvas, камера, местоположение и т.д.

## Настройки безопасности

### DM-спаривание

По умолчанию неизвестные отправители требуют проверки спаривания. Измените на открытый режим с помощью `dmPolicy: "open"` (не рекомендуется).

### Управление спариванием:

```bash
# Просмотр списка спариваний
openclaw pairing list

# Одобрение спаривания
openclaw pairing approve

# Отзыв спаривания
openclaw pairing revoke
```

### Запуск диагностики

Регулярно запускайте диагностику для безопасности:

```bash
openclaw doctor
```

## Справка по командам

| Команда | Описание |
|---------|----------|
| `/status` | Просмотр статуса сеанса |
| `/new` или `/reset` | Сброс сеанса |
| `/compact` | Сжатие контекста сеанса |
| `/think` | Установка уровня мышления |
| `/verbose` | Переключение подробного вывода |
| `/usage` | Просмотр статистики использования |
| `/restart` | Перезапуск Gateway |

## Рекомендации

1. **Регулярно запускайте doctor**: Поддерживайте систему здоровой
2. **Будьте осторожны с DM**: Проверяйте настройки безопасности
3. **Используйте мощные модели**: Получайте лучший опыт
4. **Регулярно делайте резервные копии**: Сохраняйте важные настройки |

## Устранение неполадок

Распространённые проблемы:

- **Gateway не запускается**: Проверьте порт 18789
- **Ошибка подключения канала**: Проверьте учётные данные
- **Ошибка отправки сообщения**: Проверьте логи
