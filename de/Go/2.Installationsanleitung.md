# Go Installationsanleitung

## Anforderungen

Go kann auf mehreren Betriebssystemen ausgeführt werden, einschließlich Linux, macOS und Windows. Verschiedene Betriebssysteme haben unterschiedliche Installationsmethoden, aber die Kernanforderungen sind ähnlich. Stellen Sie sicher, dass Ihr System die folgenden grundlegenden Anforderungen erfüllt, um die Go-Entwicklungsumgebung erfolgreich zu installieren und zu nutzen.

In Bezug auf Betriebssysteme unterstützt Go eine breite Palette von Plattformen. Für Linux kann Go auf wichtigen Distributionen wie Ubuntu, Debian, Fedora, CentOS und RHEL ausgeführt werden. macOS erfordert macOS 10.13 (High Sierra) oder höher. Windows erfordert Windows 10 oder höher, während auch Windows 7 und 8.1 unterstützt werden (obwohl möglicherweise zusätzliche Konfiguration erforderlich ist).

In Bezug auf Hardware sind die Anforderungen von Go relativ niedrig. Die Installation von Go erfordert nur etwa 100 MB Festplattenspeicher, obwohl die tatsächliche Projektentwicklung je nach Projektgröße mehr Speicher erfordern kann. Für Speicher sind 2 GB RAM für die tägliche Entwicklung ausreichend, obwohl möglicherweise mehr für große Kompilierungen oder umfangreiche Tests erforderlich sein kann.

Der Go-Compiler selbst hat keine besonderen CPU-Anforderungen, und moderne Architekturen wie x86-64 und ARM64 funktionieren gut. Go unterstützt Cross-Kompilierung, sodass Binärdateien für andere Plattformen von einer Plattform aus kompiliert werden können.

## Installationsschritte

### Installation unter Linux

Unter Linux-Systemen wird die Verwendung des offiziellen Tarballs empfohlen, da diese Methode am saubersten und flexibelsten ist. Laden Sie zuerst den Tarball der neuesten Version von der offiziellen Go-Website herunter:

```bash
# Go herunterladen (durch neueste Version ersetzen)
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz

# Alte Go-Installation entfernen (falls vorhanden)
sudo rm -rf /usr/local/go

# Nach /usr/local entpacken
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

Als Nächstes fügen Sie das Go-Binärverzeichnis zur PATH-Umgebungsvariable hinzu. Bearbeiten Sie die Datei ~/.bashrc oder ~/.profile und fügen Sie die folgende Zeile hinzu:

```bash
export PATH=$PATH:/usr/local/go/bin
```

Speichern Sie nach dem Speichern den folgenden Befehl, um die Konfiguration anzuwenden:

```bash
source ~/.bashrc
```

Oder unter macOS oder einigen Linux-Distributionen:

```bash
source ~/.profile
```

### Installation unter macOS

Unter macOS-Systemen gibt es zwei Hauptinstallationsmethoden. Die erste ist die Verwendung des Homebrew-Paketmanagers, geeignet für Benutzer, die Homebrew bereits konfiguriert haben:

```bash
# Mit Homebrew installieren
brew install go

# Installation überprüfen
go version
```

Die zweite Methode ist das Herunterladen des offiziellen macOS-Installationspakets. Besuchen Sie die offizielle Go-Website, um das .pkg-Installationsprogramm herunterzuladen, doppelklicken Sie zum Ausführen und folgen Sie dem Installationsassistenten. Das Installationsprogramm konfiguriert automatisch Umgebungsvariablen.

Öffnen Sie nach der Installation Terminal und überprüfen Sie die Installation:

```bash
go version
```

Sie sollten eine Ausgabe wie "go1.21.5 darwin/amd64" sehen.

### Installation unter Windows

Unter Windows-Systemen wird die Verwendung des MSI-Installationsprogramms empfohlen. Besuchen Sie die offizielle Go-Website, um das Windows-MSI-Installationsprogramm herunterzuladen, doppelklicken Sie zum Ausführen und folgen Sie dem Installationsassistenten. Das Installationsprogramm konfiguriert automatisch Umgebungsvariablen einschließlich PATH.

Öffnen Sie nach der Installation die Eingabeaufforderung oder PowerShell und überprüfen Sie die Installation:

```bash
go version
```

Windows-Benutzer können auch den Chocolatey-Paketmanager verwenden:

```bash
choco install golang
```

### Verwendung von Versionsmanagern (empfohlen)

Für Entwickler, die mehrere Go-Versionen gleichzeitig verwalten müssen, wird die Verwendung eines Versionsmanagers empfohlen. Häufige Versionsmanager sind gvm (Go Version Manager) und asdf-go:

Installation und Verwendung von gvm:

```bash
# gvm installieren
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

# Go-Version installieren
gvm install go1.21.5
gvm use go1.21.5 --default
```

asdf ist ein Multi-Sprachen-Versionsmanager, der Go unterstützt:

```bash
# asdf-Plugin installieren
asdf plugin add golang

# Go-Version installieren
asdf install golang 1.21.5
asdf global golang 1.21.5
```

## Installation überprüfen

Nach der Installation müssen Sie überprüfen, ob Go korrekt installiert ist. Öffnen Sie Terminal oder Eingabeaufforderung und führen Sie die folgenden Befehle aus:

```bash
# Go-Version überprüfen
go version

# Go-Umgebungsinformationen anzeigen
go env
```

Der Befehl go env gibt mehrere Umgebungsvariablen aus, darunter GOPATH, GOROOT, GOOS, GOARCH usw. Diese Informationen sind wichtig für das Verständnis der Funktionsweise und Umgebungskonfiguration von Go.

Führen Sie ein einfaches Programm aus, um zu überprüfen, ob die gesamte Entwicklungsumgebung korrekt funktioniert:

```bash
# Arbeitsverzeichnis erstellen
mkdir -p $HOME/go/src/hello
cd $HOME/go/src/hello

# Modul initialisieren (Go 1.16+ empfiehlt go mod)
go mod init hello

# Testprogramm erstellen
cat > main.go << 'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
EOF

# Programm ausführen
go run main.go
```

Wenn "Hello, Go!" ausgegeben wird, ist die Go-Entwicklungsumgebung korrekt konfiguriert.

## Umgebungskonfiguration

### GOPATH und Module

Vor Go 1.11 verwendete Go den GOPATH-Modus zur Verwaltung von Abhängigkeiten. Seit Go 1.11 führte Go die Unterstützung für Module ein, und die Verwendung von go mod zur Verwaltung von Projektabhängigkeiten wird empfohlen. Bei Verwendung von Modulen muss GOPATH nicht festgelegt werden, und Projekte können in einem beliebigen Verzeichnis erstellt werden.

Für Unternehmens-Intranetumgebungen oder bei Verwendung privater Repositorys kann es erforderlich sein, die GOPRIVATE-Umgebungsvariable zu konfigurieren:

```bash
export GOPRIVATE=github.com/company/*
```

### Proxy-Konfiguration

In Regionen wie Festlandchina kann der direkte Zugriff auf das offizielle Go-Modulrepository langsam oder nicht verfügbar sein. Sie können einen Go-Modulproxy konfigurieren:

```bash
# Alibaba Cloud-Proxy verwenden
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct

# Oder Qiniu Cloud-Proxy verwenden
go env -w GOPROXY=https://goproxy.cn,direct
```

### IDE-Konfiguration

Die Go-Entwicklung erfolgt normalerweise mit Editoren wie VS Code, GoLand oder Sublime Text. VS Code erfordert die Installation der Go-Erweiterung für eine gute Entwicklungserfahrung, einschließlich Codevervollständigung, Debugging, Formatierung und mehr.

GoLand ist eine professionelle Go-IDE von JetBrains, die die vollständigste Go-Entwicklungsunterstützung bietet, einschließlich erweitertem Debugging, Refactoring und Testing.

## Upgrade und Deinstallation

### Go-Version aktualisieren

Das Aktualisieren von Go ist einfach. Laden Sie das Installationsprogramm der neuen Version herunter und installieren Sie es erneut gemäß den oben genannten Schritten. Das neue Installationsprogramm überschreibt die alte Version.

Benutzer von Versionsmanagern können problemlos zwischen verschiedenen Versionen wechseln:

```bash
# gvm
gvm install go1.22.0
gvm use go1.22.0

# asdf
asdf install golang 1.22.0
asdf global golang 1.22.0
```

### Go deinstallieren

Unter Linux oder macOS erfordert das Deinstallieren von Go nur das Entfernen des Installationsverzeichnisses:

```bash
# Go-Installationsverzeichnis entfernen
sudo rm -rf /usr/local/go

# Aus PATH entfernen
# Bearbeiten Sie ~/.bashrc oder ~/.profile und entfernen Sie die entsprechenden Zeilen
```

Unter Windows deinstallieren Sie über die Systemsteuerung Programme und Features oder verwenden Sie Chocolatey:

```bash
choco uninstall golang
```
