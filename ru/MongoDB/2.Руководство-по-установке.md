# MongoDB Руководство по установке

## Выбор версии и требования к платформе

### Выбор версии

- **MongoDB 8.2**: последняя стабильная версия, рекомендуется для новых проектов
- **MongoDB 7.0**: версия с долгосрочной поддержкой

### Требования к платформе

MongoDB не поддерживает 32-битные платформы. Убедитесь, что ваша ОС 64-битная.

## Установка на Ubuntu

### Шаг 1: Импорт открытого ключа

```bash
curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor
```

### Шаг 2: Создание файла репозитория

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] http://repo.mongodb.org/apt/debian focal/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
```

### Шаг 3: Обновление и установка

```bash
sudo apt update
sudo apt install -y mongodb-org
```

### Шаг 4: Запуск MongoDB

```bash
sudo systemctl start mongod
sudo systemctl enable mongod
```

## Установка на RHEL/CentOS

### Шаг 1: Создание файла репозитория

```bash
[mongodb-org-8.0]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/yum/redhat/$releasever/mongodb-org/8.0/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-8.0.asc
```

### Шаг 2: Установка

```bash
sudo dnf install -y mongodb-org
```

## Установка на macOS

```bash
brew update
brew install mongodb-community
brew services start mongodb-community
```

## Установка на Windows

1. Скачайте MSI-установщик с сайта MongoDB
2. Запустите установщик и следуйте инструкциям

## Установка с Docker

```bash
docker pull mongodb/mongodb-community-server
docker run --name mongodb -d -p 27017:27017 mongodb/mongodb-community-server
```

## Проверка установки

### Подключение к MongoDB

```bash
mongosh
```

### Проверка версии

```bash
db.version()
```

## После установки

### Создание администратора

```bash
use admin
db.createUser({
  user: "admin",
  pwd: "ваш_пароль",
  roles: [
    { role: "userAdminAnyDatabase", db: "admin" },
    { role: "readWriteAnyDatabase", db: "admin" }
  ]
})
```

### Включение аутентификации

1. Остановите MongoDB
2. Измените `/etc/mongod.conf`:

```yaml
security:
  authorization: enabled
```

3. Перезапустите:

```bash
sudo systemctl restart mongod
```

## Часто задаваемые вопросы

### Какой порт по умолчанию у MongoDB?

По умолчанию используется порт 27017.

### Где MongoDB хранит данные?

По умолчанию:
- Linux/macOS: `/data/db`
- Windows: `C:\data\db`
