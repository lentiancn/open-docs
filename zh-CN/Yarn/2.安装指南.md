# Yarn Installation Guide

## Yarn v4+ (Modern Version) Installation

### Recommended Method: Using Corepack

Corepack is an intermediary tool that will let you configure your package manager version on a per-project basis.

```bash
# 1. Install Corepack
npm install -g corepack

# 2. Initialize a new project
yarn init -2
```

### Updating Yarn

Any time you'll want to update Yarn to the latest version, just run:

```bash
yarn set version stable
yarn install
```

Yarn will then configure your project to use the most recent stable binary.

### Using Canary Builds

Yarn also frequently ships Release Candidate builds. Use the following command should you need a feature not released on the stable channel yet:

```bash
yarn set version canary
```

Those builds are very stable, the only difference with the regular channel being a more staggered migration between major versions as we implement new breaking changes.

### Installing the Latest Build from Master

You may want to test a version of Yarn so recent it hasn't been released in a Release Candidate yet, or even not merged. The following command will clone, build, and install Yarn in your project, straight from our repository:

```bash
yarn set version from sources
```

It accepts a `--branch` flag which you can use to test specific PRs:

```bash
yarn set version from sources --branch 1211
```

> Warning: Unlike the stable and canary channels, the `yarn set version from sources` command can't leverage Corepack and will need to store the Yarn binary inside the `.yarn/releases` folder and reference it from your project's `.yarnrc.yml` file.

## Yarn v1 (Classic) Installation

### Via npm (Recommended)

It is recommended to install Yarn through the [npm package manager](http://npmjs.org/), which comes bundled with [Node.js](https://nodejs.org/) when you install it on your system.

Once you have npm installed you can run the following both to install and upgrade Yarn:

```bash
npm install --global yarn
```

### Check Installation

Check that Yarn is installed by running:

```bash
yarn --version
```

## Version Selection Recommendations

- **New Projects**: Recommended to use Yarn v4+ to enjoy modern features and better performance
- **Existing Projects**: If already using Yarn v1, you can continue using it or consider migrating to the modern version
- **Enterprise Environments**: Yarn v1 provides more stable long-term support, while v4+ offers more innovative features
