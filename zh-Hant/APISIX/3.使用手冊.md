# APISIX 使用手冊

本文檔介紹 APISIX 的核心功能和使用方法。

## 基本概念

### Route（路由）

路由是 APISIX 最核心的概念，用於匹配客戶端請求並將請求轉發到上游服務。

```bash
# 建立路由
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "my-route",
  "uri": "/api/v1/*",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend1.example.com:8080": 1,
      "backend2.example.com:8080": 1
    }
  }
}'
```

### Upstream（上游）

上游是一組提供相同服務的節點，APISIX 支援多種負載均衡策略：

```bash
# 建立上游
curl -i "http://127.0.0.1:9180/apisix/admin/upstreams" -X PUT -d '
{
  "id": "my-upstream",
  "type": "roundrobin",
  "nodes": {
    "host1:8080": 1,
    "host2:8080": 1
  }
}'
```

### Plugin（插件）

APISIX 提供了豐富的插件系統，支援在請求處理過程中執行各種功能。

## 路由匹配規則

### 1. 精確匹配

```json
{
  "uri": "/api/users"
}
```

### 2. 參數匹配

```json
{
  "uri": "/api/users/:id"
}
```

### 3. 正則匹配

```json
{
  "uri": "/api/*",
  "vars": [["uri", "==", "/api/users"]]
}
```

### 4. 條件匹配

```json
{
  "uri": "/api/*",
  "vars": [
    ["http_x_api_version", "==", "v2"],
    ["arg_page", ">", 1]
  ]
}
```

## 負載均衡策略

### 1. 輪詢（Round Robin）

```json
{
  "type": "roundrobin"
}
```

### 2. 雜湊（Hash）

```json
{
  "type": "hash",
  "key": "remote_addr"
}
```

### 3. 一致性雜湊

```json
{
  "type": "chash",
  "key": "remote_addr"
}
```

## 常用插件

### 1. 限流（limit-req）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "rate-limit-route",
  "uri": "/api/*",
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 2. 密鑰認證（key-auth）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "auth-route",
  "uri": "/api/*",
  "plugins": {
    "key-auth": {
      "header": "X-API-Key"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 3. JWT 認證（jwt-auth）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "jwt-route",
  "uri": "/api/*",
  "plugins": {
    "jwt-auth": {}
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 4. CORS（cross-origin）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "cors-route",
  "uri": "/api/*",
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "Content-Type"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 5. 請求重寫（proxy-rewrite）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "rewrite-route",
  "uri": "/api/*",
  "plugins": {
    "proxy-rewrite": {
      "uri": "/mock/$uri",
      "headers": {
        "X-Forwarded-By": "APISIX"
      }
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### 6. 響應快取（proxy-cache）

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "cache-route",
  "uri": "/api/*",
  "plugins": {
    "proxy-cache": {
      "cache_key": [
        "uri",
        "args"
      ],
      "cache_http_status": [200],
      "cache_ttl": 60
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 服務發現

APISIX 支援多種服務發現機制：

### 1. Consul

```yaml
discovery:
  consul:
    servers:
      - "http://127.0.0.1:8500"
```

### 2. Nacos

```yaml
discovery:
  nacos:
    host:
      - "http://127.0.0.1:8848"
    prefix: /nacos/v1
```

### 3. Eureka

```yaml
discovery:
  eureka:
    host:
      - "http://127.0.0.1:8761"
    port: 8761
```

## 健康檢查

APISIX 支援主動和被動健康檢查：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/upstreams" -X PUT -d '
{
  "id": "health-check-upstream",
  "type": "roundrobin",
  "nodes": {
    "backend1:8080": 1,
    "backend2:8080": 1
  },
  "checks": {
    "active": {
      "type": "http",
      "http_path": "/health",
      "interval": 5,
      "timeout": 2
    },
    "passive": {
      "type": "http",
      "healthy": {
        "http_statuses": [200, 201, 202, 203, 204],
        "successes": 3
      },
      "unhealthy": {
        "http_statuses": [500, 502, 503, 504],
        "http_failures": 3,
        "timeouts": 3
      }
    }
  }
}'
```

## 最佳實踐

1. **使用上游**：將上游資訊獨立設定，便於管理和復用
2. **合理使用插件**：按需啟用插件，避免不必要的效能開銷
3. **設定健康檢查**：保證服務的高可用性
4. **使用服務發現**：在動態環境中自動感知服務變化
5. **監控和日誌**：整合 Prometheus 和日誌系統，便於問題排查
