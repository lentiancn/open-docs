# Java 常见问题

## 基础问题

### 问题一：java 和 javac 命令不可用

**问题描述**：在命令行输入 java -version 或 javac -version 提示命令不存在。

**解决方法**：
1. 确认 JDK 已正确安装
2. 检查 JAVA_HOME 环境变量是否配置
3. 检查 Path 环境变量是否包含 %JAVA_HOME%\bin（Windows）或 $JAVA_HOME/bin（Linux/macOS）
4. 重新打开终端使环境变量生效

### 问题二：找不到或无法加载主类

**问题描述**：运行 java 命令时提示 "Could not find or load main class"。

**解决方法**：
1. 确保在正确的目录下运行命令
2. 检查类名是否与文件名完全一致
3. 如果类在包中，需要使用完整包名运行
4. 检查 classpath 是否正确设置

### 问题三：编码问题导致中文乱码

**问题描述**：程序中的中文输出显示为乱码。

**解决方法**：
1. 编译时指定编码：javac -encoding UTF-8 MyClass.java
2. 运行指定编码：java -Dfile.encoding=UTF-8 MyClass
3. 确保源文件保存为 UTF-8 编码
4. 集成开发环境中设置正确的文件编码

## 环境配置问题

### 问题四：多个 JDK 版本冲突

**问题描述**：系统中有多个 JDK 版本，java 命令使用了错误的版本。

**解决方法**：
1. 检查 JAVA_HOME 指向的 JDK 版本
2. Windows 上使用 where java 查看所有 java 命令位置
3. Linux/macOS 上使用 which java 查看路径
4. 调整 PATH 环境变量中 JDK bin 目录的顺序
5. 使用 update-alternatives（Linux）管理多版本

### 问题五：端口被占用

**问题描述**：启动 Java Web 应用时提示端口被占用。

**解决方法**：
1. Windows：netstat -ano | findstr 8080 查找占用端口的进程
2. Linux：lsof -i:8080 或 netstat -tulpn | grep 8080
3. 结束占用端口的进程，或修改应用程序的端口配置

### 问题六：内存溢出

**问题描述**：程序运行时出现 OutOfMemoryError。

**解决方法**：
1. 增加 JVM 堆内存大小：java -Xmx2048m MyClass
2. 检查代码是否存在内存泄漏
3. 使用内存分析工具（如 VisualVM、JProfiler）排查问题
4. 优化数据结构，减少内存占用

## 开发常见问题

### 问题七：NullPointerException

**问题描述**：空指针异常是最常见的运行时异常。

**解决方法**：
1. 在使用对象前检查是否为 null
2. 使用 Optional 类避免空指针
3. 使用 Objects.requireNonNull() 校验参数
4. 初始化所有必要的对象

### 问题八：ConcurrentModificationException

**问题描述**：遍历集合时修改集合内容导致异常。

**解决方法**：
1. 使用迭代器的 remove() 方法删除元素
2. 使用 CopyOnWriteArrayList 替代 ArrayList
3. 先记录要删除的元素，遍历后再删除
4. 使用 Stream API 的 filter 过滤元素

### 问题九：类转换异常

**问题描述**：出现 ClassCastException。

**解决方法**：
1. 使用 instanceof 检查对象类型
2. 尽量使用接口类型而非具体实现
3. 在需要转换前进行类型检查

### 问题十：数字精度问题

**问题描述**：浮点数计算结果不准确。

**解决方法**：
1. 使用 BigDecimal 替代 double 进行精确计算
2. 了解浮点数的精度限制
3. 避免使用浮点数比较，使用 BigDecimal 的 compareTo

## 性能问题

### 问题十一：程序运行缓慢

**问题描述**：Java 程序运行速度慢。

**解决方法**：
1. 使用性能分析工具定位瓶颈
2. 避免在循环中进行重复计算
3. 使用 StringBuilder 替代字符串拼接
4. 选择合适的集合类型
5. 启用 JIT 编译器的优化

### 问题十二：死锁

**问题描述**：程序出现死锁，无法继续执行。

**解决方法**：
1. 避免嵌套锁
2. 统一加锁顺序
3. 使用 tryLock() 超时机制
4. 减少同步块的范围

## 构建工具问题

### 问题十三：Maven/Gradle 依赖冲突

**问题描述**：构建时出现依赖版本冲突。

**解决方法**：
1. 使用 mvn dependency:tree 查看依赖树
2. 在 pom.gradle 中明确指定依赖版本
3. 排除传递性依赖
4. 使用 Maven Central 的 Bom 管理版本

### 问题十四：构建失败

**问题描述**：Maven 或 Gradle 构建失败。

**解决方法**：
1. 查看错误信息定位问题
2. 清理缓存：mvn clean 或 gradle clean
3. 检查网络连接
4. 确认 JDK 版本符合项目要求
5. 查看是否有语法错误

## 框架问题

### 问题十五：Spring Boot 启动失败

**问题描述**：Spring Boot 应用无法启动。

**解决方法**：
1. 查看启动日志中的错误信息
2. 检查 application.properties 或 application.yml 配置
3. 确认数据库连接配置正确
4. 检查依赖是否完整
5. 查看端口是否被占用

### 问题十六：Hibernate 懒加载异常

**问题描述**：Session 关闭后访问延迟加载的对象。

**解决方法**：
1. 使用 @Fetch(FetchMode.JOIN) 立即加载
2. 在 Session 关闭前初始化需要的对象
3. 使用 Hibernate.initialize() 手动加载
4. 配置 spring.jpa.open-in-view=true

## 版本选择问题

### 问题十七：应该学习哪个 Java 版本

**建议**：
- 初学者：推荐 Java 21 或 Java 17，语法现代，资料丰富
- 企业项目：根据项目依赖选择，通常是 Java 8、11 或 17
- 新项目：强烈推荐 Java 21 或更高版本

### 问题十八：JDK 和 JRE 的区别

**解答**：
- JDK（Java Development Kit）：开发工具包，包含 JRE 和编译、调试等开发工具
- JRE（Java Runtime Environment）：Java 运行时环境，仅包含运行 Java 程序所需的类库和 JVM

如果需要开发 Java 程序，安装 JDK；如果只需要运行已编译的 Java 程序，安装 JRE 即可。

## 其他常见问题

### 问题十九：如何学习 Java

**建议**：
1. 掌握 Java 基础语法和面向对象概念
2. 熟悉 Java 集合框架和常用工具类
3. 学习 JDBC 数据库操作
4. 掌握至少一种 Web 框架（Spring/Spring Boot）
5. 了解设计模式和代码重构
6. 多实践，动手编写项目

### 问题二十：Java 还能火多久

**分析**：
Java 作为企业级开发的主流语言，拥有成熟的生态和庞大的用户基础。根据 TIOBE 排行榜，Java 长期排名前三。Java 仍在持续更新，每半年发布一个新版本，每两年发布一个 LTS 版本。因此，在可预见的未来，Java 仍将是非常有价值的技能。
