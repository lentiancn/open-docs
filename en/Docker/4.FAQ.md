## Installation

### Q: Docker Desktop fails to start on Windows?

**Solutions**:
1. Enable virtualization in BIOS
2. Install/update WSL 2:
   ```powershell
   wsl --install
   wsl --update
   ```
3. Restart computer
4. Check for conflicting software (VirtualBox, VMware)

---

### Q: "docker: permission denied" error?

**Solutions**:
```bash
# Add user to docker group
sudo usermod -aG docker $USER

# Log out and log back in
# Or run:
newgrp docker
```

## Images

### Q: How to reduce Docker image size?

**Best Practices**:
1. Use alpine-based images
2. Use multi-stage builds
3. Minimize layers
4. Clean up in the same layer

```dockerfile
# Good example
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
```

## Containers

### Q: How to keep container running?

```bash
# Method 1: Tail a log
docker run -d nginx tail -f /dev/null

# Method 2: Use sleep
docker run -d nginx sleep infinity
```

---

### Q: Container exits immediately after starting?

**Reasons**:
1. No foreground process
2. Process completed

**Solutions**:
```bash
# Run interactively to see errors
docker run -it image_name /bin/bash

# Keep container running
docker run -d image_name tail -f /dev/null
```

---

### Q: How to enter a running container?

```bash
docker exec -it container_id /bin/bash
```

---

### Q: How to copy files to/from container?

```bash
# Copy from container to host
docker cp container_id:/path/file.txt ./

# Copy from host to container
docker cp ./file.txt container_id:/path/
```

## Networking

### Q: How to access container from host?

Use port mapping:
```bash
docker run -p 8080:80 nginx
# Access via http://localhost:8080
```

---

### Q: How to connect containers?

Create a network:
```bash
docker network create my-network
docker run --network my-network -d nginx
docker run --network my-network -d my-app
```

Containers on the same network can communicate by container name.

---

### Q: How to use custom DNS?

```bash
docker run --dns 8.8.8.8 nginx
```

## Volumes

### Q: How to persist data?

Use volumes:
```bash
# Named volume
docker volume create my-data
docker run -v my-data:/data nginx

# Bind mount
docker run -v /host/path:/container/path nginx
```

---

### Q: How to share data between containers?

```bash
# Create shared volume
docker volume create shared-data

# Use in multiple containers
docker run -v shared-data:/data1 nginx
docker run -v shared-data:/data2 alpine
```

## Performance

### Q: Docker is slow / consuming too much disk?

**Solutions**:

1. Clean up:
```bash
docker system prune -a
```

2. Check disk usage:
```bash
docker system df
```

3. Remove unused volumes:
```bash
docker volume prune
```

---

### Q: How to limit container resources?

```bash
# Memory limit
docker run -m 512m nginx

# CPU limit
docker run --cpus=0.5 nginx

# In docker-compose:
services:
  app:
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '0.5'
```

### Q: Should I run as root?

No, create a non-root user:

```dockerfile
FROM node:18-alpine
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -D appuser
USER appuser
```

## Docker vs VM

### Q: What's the difference between Docker and VM?

| Aspect | Docker | VM |
|--------|--------|-----|
| OS | Shares host kernel | Separate OS |
| Size | MBs | GBs |
| Start | Seconds | Minutes |
| Isolation | Containers | Full isolation |
