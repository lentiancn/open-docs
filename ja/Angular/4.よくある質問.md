# よくある質問

このドキュメントでは、Angularを使っているときにありがちな技術的な疑問とその答えをまとめています。

## 基本的な質問

### Angularとは何ですか？

Angularは、Googleが開発・メンテナンスを行っているオープンソースのWebアプリケーションフレームワークです。現代的なWebアプリケーションを構築するために使われます。TypeScriptを主要な開発言語として採用しており、コンポーネント化アーキテクチャ、双方向データバインディング、依存性注入などの特徴があります。

### AngularとAngularJSの違いは何ですか？

AngularJSはAngular 1.xの名前で、Angularの前身です。Angularは全面的に書き直されたバージョンで、TypeScriptを使っています。アーキテクチャも全く異なり、AngularJSとの互換性はありません。

### Angularは難し나요？

Angularには確かに学習曲線があります。特に依存性注入やRxJSといった概念は、時間がかかるもあります。でも、ドキュメントも充実していますしコミュニティも活発なので、一度慣れてしまえば開発効率は本当に高いですよ。

## インストールに関する問題

### npm installが失敗した場合は？

1. npmキャッシュをクリア：`npm cache clean --force`
2. node_modulesとpackage-lock.jsonを削除
3. 国内ミラーを使用する
4. Node.jsのバージョンが対応しているか確認

### Angular CLIのコマンドが見つからない？

グローバルインストールされているか確認：
```bash
npm install -g @angular/cli
```

それでもダメなら、npxを使ってみる：
```bash
npx @angular/cli new my-app
```

### Node.jsのバージョンが対応していない？

nvmを使ってNode.jsのバージョンを管理：
```bash
nvm install 18
nvm use 18
```

## 開発に関する問題

### コンポーネントのスタイルが反映されない？

1. スタイルファイルのパスが正しいか確認
2. styles.scssに構文エラーがないか確認
3. ViewEncapsulationの設定を確認

### ルーティング移動が失敗する？

1. ルーターの設定が正しいか確認
2. RouterModuleが正しくインポートされているか確認
3. ルートガードが移動をブロックしていないか確認

### HTTPリクエストが失敗する？

1. バックエンドサービスが起動しているか確認
2. CORSの設定を確認
3. HttpInterceptorを使ってエラー処理を追加

### データが更新されない？

1. 変更検出戦略を確認
2. OnPushを使っている場合は、手動で検出をトリガー
3. データの参照が変化しているか確認

## パフォーマンスの問題

### 初期表示が遅い？

1. ルーティングの遅延読み込みを有効化
2. プリロード戦略を使用
3. 画像リソースを最適化
4. 本番モードでビルド

### メモリリーク？

1. ngOnDestroyでObservableの購読を解除
2. takeUntilやasync pipeを使用
3. タイマーとイベントリスナーをクリーンアップ

### 変更検出が高頻度で走る？

1. OnPush戦略を使用
2. 変化するプロパティと変化しないプロパティを分離
3. *ngForでtrackByを使って最適化

## テストの問題

### ユニットテストが失敗する？

1. TestBedの設定を確認
2. 非同期処理が正しく処理されているか確認
3. 非同期コードのテストにはfakeAsyncとtickを使用

### モジュールが見つからない？

tsconfig.jsonでpathsが正しく設定されているか確認。

## ビルドの問題

### ビルドが失敗する？

1. エラーメッセージ的具体内容を確認
2. TypeScriptのコンパイルオプションを確認
3. キャッシュをクリアして再ビルド

### ビルドファイルが大きすぎる？

1. 本番モードを有効化
2. 遅延読み込みを使用
3. 使ってない依存がないか確認

## よくあるエラー

### ExpressionChangedAfterItHasBeenCheckedError

検出の間に変更してはいけない値をテンプレートで変更しています。ngOnInitに直接ビュー関連のコードを書いていないか確認してください。

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

上記と同じエラーで、データバインディングのタイミングを確認します。

### Cannot read property of undefined

データが初期化される前にアクセスしています。可選チェーン（?.）または*ngIfで保護してください。

### CORSエラー

バックエンドでCORSが設定されていないか、フロントエンドのリクエストヘッダーに問題があります。

### 401/403エラー

認証失敗で、tokenと権限の設定を確認してください。

## バージョンアップ

### Angularのバージョンアップ方法は？

1. CLIでアップグレード：`ng update @angular/core`
2. 公式のアップグレードガイドを参照
3. 依存関係を1つずつアップグレード
4. テストを実行して問題がないか確認

### バージョンアップに失敗したら？

1. コードをバックアップ
2. package.jsonのバージョンを手動で更新
3. node_modulesを削除して再インストール
