# JMeter よくある質問

この章では、Apache JMeterの使用中に発生する一般的な問題と解決策をまとめています。

## インストール関連の問題

### Q1: JMeterを実行すると「java」は内部または外部コマンドとして認識されません

**A**: Javaが正しくインストールされ、設定されていることを確認してください。コマンドラインで以下を実行：
```bash
java -version
```
インストールされていない場合は、Java JDK 8以上をダウンロードしてインストールしてください。Windowsの場合、JAVA_HOME環境変数を設定する必要もあります。

### Q2: JMeter起動後にインターフェースが文字化けする

**A**: フォントエンコーディングの問題 때문입니다。以下の方法で解決できます：
1. bin/jmeter.propertiesファイルを開く
2. 以下を追加または変更：
   ```
   sampleresult.default.encoding=UTF-8
   ```
3. JMeterを再起動

### Q3: JMeterのメモリ不足でクラッシュする

**A**: デフォルトのヒープメモリが不十分な可能性があります。調整できます：
- Windows：bin/jmeter.batファイルを編集し、set HEAPパラメータを増量
- Linux/Mac：bin/jmeterファイルを編集し、HEAP変数を調整
2GB以上を推奨：
```bash
HEAP="-Xms2g -Xmx2g"
```

## テスト実行の問題

### Q4: 複数の異なるユーザーログインをシミュレートするには？

**A**: パラメータ化を使用します：
1. ユーザー名とパスワードを含むCSVファイルを作成
2. 「CSVデータファイル設定」要素を追加
3. HTTPリクエスト内で${username}と${password}変数を使用

### Q5: JMeterがSessionまたはCookieを抽出できない

**A**: 適切な要素が追加されていることを確認：
1. スレッドグループに「HTTP Cookieマネージャー」を追加
2. ログインが必要なテストの場合、「Clear cookies each iteration」オプションが正しく設定されていることを確認
3. リクエストが正しくCookieを携带しているか確認

### Q6: レスポンスデータ中文が文字化けする

**A**: 以下の方法で解決できます：
1. HTTPリクエストで「コンテンツエンコーディング」をUTF-8に設定
2. 「HTTPリクエストデフォルト」でデフォルトエンコーディングを設定
3. ポストプロセッサで「正規表現抽出」を使用し、エンコーディングを指定

### Q7: HTTPS証明書の問題を処理するには？

**A**: テスト環境の場合：
1. 信頼証明書をJava cacertsストアにインポート
2. またはHTTPリクエストで「実装」をJavaに設定
3. 自己署名証明書のの場合、JMeterのtruststoreをインポートする必要がある場合あり

### Q8: 長時間実行するとメモリが増え続ける

**A**: 推奨される対策：
1. リスナーが収集するデータ量を削減
2. コマンドラインモードでテストを実行
3. jmeter.propertiesでゴミ回収頻度を増加：
   ```
   mode=Stripped
   ```
4. 結果ファイルを定期的にクリーンアップ

## パフォーマンスの問題

### Q9: JMeterが十分な負荷を生成できない

**A**: 次の方法を試してください：
1. 分散テストを使用（リモートテスト）
2. スレッド数とループ回数を調整
3. 不要なリスナーを削減
4. GUIモードではなくコマンドラインモードを使用
5. ネットワーク帯域幅がボトルネックになっているか確認

### Q10: テスト結果の応答時間が大きく変動する

**A**: 考えられる原因と解決策：
1. サーバー側のGCまたはキャッシングの問題
2. ネットワーク不安定
3. テストデータに依存関係がある
4. 同期タイマーを使用して并发リクエストを保証
5. ウォームアップ時間を増加させ、サーバーが安定してから統計を開始

### Q11: 適切な并发数を設定するには？

**A**: 推奨されるアプローチ：
1. 较小的并发数から開始（10-50）
2. 徐々に并发数を增加させ、システム応答を観察
3. パフォーマンスの転換点を見つける（応答時間が顕著に增加し始める）
4. 最大安全并发数を決定
5. ストレステストを実行してシステムの限界を検証

## レポートと結果分析

### Q12: HTMLレポートを生成するには？

**A**: コマンドラインを使用：
```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o /path/to/report
```
または既存の結果ファイルから生成：
```bash
jmeter -g results.jtl -o /path/to/report
```

### Q13: 聚合レポートの指標の意味は？

- **サンプル数（Samples）**：リクエスト総数
- **平均値（Average）**：平均応答時間（ミリ秒）
- **中央値（Median）**：応答時間の中央値
- **90%/95%/99%パーセンタイル**：90%/95%/99%のリクエストがこの値より速く応答
- **最小/最大**：最小と最大の応答時間
- **エラー率（Error %）**：失敗リクエストの割合
- **スループット（Throughput）**：毎秒処理されるリクエスト数

### Q14: 結果ファイルが大きすぎる

**A**: 対策：
1. リスナーでCSV形式で保存を設定
2. 必要なフィールドのみ保存するように設定
3. 「分散テスト」を使用して保存を分散
4. 結果ファイルを定期的にローテーション

## スクリプトの問題

### Q15: 動的に変化するパラメータを処理するには？

**A**: 一般的な方法：
1. 「正規表現抽出」を使用してレスポンスから抽出
2. 「JSON抽出」を使用してJSONデータを抽出
3. 「CSS/JQuery抽出」を使用してHTML要素を抽出
4. 「XPath抽出」を使用してXMLデータを抽出

### Q16: BeanShellスクリプトが実行できない

**A**: 確認事项：
1. JDKが正しくインストールされている（JREではない）
2. BeanShellライブラリファイルがlibディレクトリにある
3. スクリプト構文が正しいか確認

### Q17: 複雑なビジネスロジックを実装するには？

**A**: 以下を使用できます：
1. ロジックコントローラーの組み合わせ
2. BeanShellまたはJSR223スクリプト
3. プレ/ポストプロセッサ
4. 条件付き実行のためのスループットコントローラー

## その他の問題

### Q18: JMeterとブラウザの違いは？

JMeterはプロトコルレベルで動作するブラウザではありません：
- JavaScriptを実行しない
- HTMLページをレンダリングしない
- CSSを処理しない
- ブラウザのすべての動作をシミュレートできない

### Q19: JMeterを学ぶには？

推奨される学習パス：
1. JMeter公式ユーザーマニュアルを読む
2. 簡単なHTTPリクエストから練習開始
3. パラメータ化と相関処理をを学ぶ
4. 分散テストをマスター
5. カスタムスクリプト開発を学ぶ

### Q20: JMeterはどのオペレーティングシステムをサポート？

JMeterはJavaがインストールされた任意のOSをサポートする純粋なJavaアプリケーションで、Windows、Linux、Mac OSX、Solarisなどを含みます。
