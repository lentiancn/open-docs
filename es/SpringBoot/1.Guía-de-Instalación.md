# Guía de Instalación de Spring Boot

Esta guía cubre cómo instalar y configurar el entorno de desarrollo de Spring Boot en Windows, macOS y Linux.

## Requisitos del Sistema

### Requisitos de Versión de Java

| Versión de Spring Boot | Versión Mínima de Java |
|---------------------|---------------------|
| 3.x                 | Java 17+            |
| 2.7.x               | Java 8+             |
| 2.5.x               | Java 8+             |

**Nota:** Spring Boot 3.x requiere Java 17 o posterior. Se recomienda usar Java 21 (LTS) para mejor compatibilidad.

## Instalar Java

### Windows

**Opción 1: Instalar desde Oracle/OpenJDK**
1. Descarga JDK desde [Oracle](https://www.oracle.com/java/technologies/downloads/) o [Adoptium](https://adoptium.net/)
2. Ejecuta el instalador
3. Configura las variables de entorno:
   ```
   JAVA_HOME=C:\Program Files\Java\jdk-21
   PATH=%JAVA_HOME%\bin;%PATH%
   ```

**Opción 2: Usando Chocolatey**
```powershell
choco install openjdk21
```

**Opción 3: Usando Winget**
```powershell
winget install Oracle.JDK.21
```

Verificar:
```cmd
java -version
```

### macOS

**Opción 1: Usando Homebrew**
```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 21)' >> ~/.zshrc
source ~/.zshrc
```

**Opción 2: Usando SDKMAN**
```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.2-tem
```

Verificar:
```bash
java -version
```

### Linux

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install openjdk-21-jdk
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
```

**RHEL/CentOS/Fedora:**
```bash
sudo dnf install java-21-openjdk-devel
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH=$JAVA_HOME/bin:$PATH
```

**Usando SDKMAN:**
```bash
curl -s "https://get.sdkman.io" | bash
source ~/.sdkman/bin/sdkman-init.sh
sdk install java 21.0.2-tem
```

Verificar:
```bash
java -version
```

### Windows

**Opción 1: Instalación Manual**
1. Descarga desde [Apache Maven](https://maven.apache.org/download.cgi)
2. Extrae en `C:\apache-maven`

   PATH=%MAVEN_HOME%\bin;%PATH%

## Configuración del IDE

### IntelliJ IDEA (Recomendado)

1. Descarga desde [JetBrains](https://www.jetbrains.com/idea/download/)
2. Instala y abre
3. Configura JDK: `File` → `Project Structure` → `Project SDK`
4. Instala el plugin Spring Boot Assistant (opcional)

### VS Code

1. Instala [VS Code](https://code.visualstudio.com/)
2. Instala extensiones:
   - Extension Pack for Java
   - Spring Boot Extension Pack
3. Configura Java: `Ctrl+Shift+P` → `Java: Configure Java Runtime`

### Eclipse

1. Descarga [Spring Tools 4 (STS)](https://spring.io/tools)
2. Extrae y ejecuta
3. File → New → Spring Starter Project

## Inicio Rápido

### Usando Spring Initializr

1. Visita [start.spring.io](https://start.spring.io/)
2. Selecciona opciones:
   - Project: Maven / Gradle
   - Language: Java / Kotlin / Groovy
   - Spring Boot Version: 3.2.x (latest)
   - Group: com.example
   - Artifact: myapp
   - Packaging: Jar / War
   - Java Version: 17 / 21
3. Añade dependencias:
   - Spring Web
   - Spring Data JPA
   - H2 Database (desarrollo)
   - Spring Boot DevTools
4. Haz clic en "Generate" para descargar

### Usando Línea de Comandos

```bash
# Crear proyecto Maven
curl https://start.spring.io/starter.zip \
  -d type=maven-project \
  -d language=java \
  -d bootVersion=3.2.5 \
  -d baseDir=myapp \
  -d dependencies=web,jpa,h2,devtools \
  -o myapp.zip

unzip myapp.zip
cd myapp
./mvnw spring-boot:run
```

## Verificar Instalación

```bash
# Verificar Java
java -version

# Verificar Maven
mvn -version

# Verificar Gradle (si está instalado)
gradle -version

# Verificar Spring Boot CLI
spring --version

# Ejecutar proyecto de demostración
curl https://start.spring.io/starter.zip -o demo.zip
unzip demo.zip -d demo
cd demo
./mvnw spring-boot:run
```

## Problemas Comunes

### Java No Encontrado
- Asegúrate de que JAVA_HOME esté configurado correctamente
- Reinicia la terminal después de configurar las variables de entorno
- Verifica con `echo $JAVA_HOME` (Linux/macOS) o `echo %JAVA_HOME%` (Windows)

### Puerto 8080 Ya en Uso
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <pid> /F

# Linux/macOS
lsof -i :8080
kill -9 <pid>
```

### Fallo de Construcción Maven
- Limpiar caché Maven: `mvn clean`
- Actualizar dependencias: `mvn dependency:resolve`
- Usar `./mvnw` (Maven Wrapper) en lugar de Maven del sistema
