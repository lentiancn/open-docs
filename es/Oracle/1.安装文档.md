# Guía de Instalación de Oracle Database

Oracle Database es el sistema de gestión de bases de datos relacionales líder en la industria, desarrollado por Oracle. Esta guía cubre los métodos de instalación en diversos entornos.

## Información de Versión

| Versión | Estado | Descripción |
|------|------|------|
| Oracle 21c | ✅ Actual | Última versión |
| Oracle 19c | ✅ LTS | Versión de soporte a largo plazo |
| Oracle 18c | ❌ Obsoleta | Soporte terminado |

> **Importante**: Esta guía está dirigida principalmente a **Oracle 19c/21c**.

## Requisitos del Sistema

### Requisitos de Hardware

| Requisito | Mínimo | Recomendado |
|------|----------|----------|
| CPU | 2 núcleos | 4+ núcleos |
| Memoria | 2 GB | 8+ GB |
| Disco | 40 GB | 100+ GB |

### Sistemas Operativos Soportados

- **Linux**: Oracle Linux, RHEL, CentOS, SLES, Ubuntu
- **Windows**: Windows Server 2016+

## Métodos de Instalación

### Método 1: Instalación en Linux

#### 1. Preparar el Sistema

```bash
# Crear grupos de usuarios
groupadd oinstall
groupadd dba
groupadd oper
groupadd backupdba
groupadd dgdba
groupadd kmdba
groupadd racdba
groupadd asmdba
groupadd asmoper
groupadd asmadmin

# Crear usuario Oracle
useradd -g oinstall -G dba,oper,backupdba,dgdba,kmdba,racdba,asmdba,asmoper,asmadmin oracle

# Establecer contraseña
passwd oracle

# Crear directorios
mkdir -p /u01/app/oracle/product/19.3.0/dbhome_1
mkdir -p /u01/app/oraInventory
mkdir -p /u01/app/oracle

# Establecer permisos
chown -R oracle:oinstall /u01
chmod -R 775 /u01
```

#### 2. Configurar Parámetros del Kernel

```bash
# /etc/sysctl.conf
cat >> /etc/sysctl.conf << EOF
kernel.shmmax = 68719476736
kernel.shmmni = 4096
kernel.shmall = 2097152
kernel.sem = 250 32000 100 128
fs.file-max = 6815744
net.ipv4.ip_local_port_range = 9000 65500
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 1048576
fs.aio-max-nr = 1048576
EOF

# Aplicar
sysctl -p

# /etc/security/limits.conf
cat >> /etc/security/limits.conf << EOF
oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536
oracle soft stack 10240
oracle hard stack 32768
EOF
```

#### 3. Instalar Dependencias

```bash
# Oracle Linux
yum install -y libaio libaio-devel libXi libXtst libXext

# Ubuntu
apt-get install -y libaio1 libaio-dev libxi6 libxtst6 libxext6
```

#### 4. Ejecutar el Instalador

```bash
# Extraer archivos de instalación
unzip -q LINUX.X64_193000_db_home.zip -d /u01/app/oracle/product/19.3.0/dbhome_1/

# Ejecutar como usuario oracle
cd /u01/app/oracle/product/19.3.0/dbhome_1
./runInstaller
```

### Método 2: Instalación en Windows

1. Descargar el instalador de Oracle
2. Ejecutar setup.exe
3. Seguir el asistente de instalación:
   - Seleccionar "Crear y configurar una base de datos de instancia única"
   - Seleccionar "Clase de escritorio" o "Clase de servidor"
   - Establecer contraseñas para sys y system
   - Completar la instalación

### Método 3: Usar Docker

```bash
# Descargar imagen
docker pull container-registry.oracle.com/database/express:21.3.0-xe

# Ejecutar contenedor
docker run -d \
  --name oracle \
  -p 1521:1521 \
  -e ORACLE_PWD=MyPassword123 \
  -v oracle_data:/opt/oracle/oradata \
  container-registry.oracle.com/database/express:21.3.0-xe
```

## Configuración Post-Instalación

### Variables de Entorno

```bash
# ~/.bash_profile
export ORACLE_HOME=/u01/app/oracle/product/19.3.0/dbhome_1
export ORACLE_SID=orcl
export PATH=$ORACLE_HOME/bin:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:$LD_LIBRARY_PATH
```

### Iniciar la Base de Datos

```bash
# Iniciar listener
lsnrctl start

# Iniciar base de datos
sqlplus / as sysdba
STARTUP;
```

## Problemas Comunes

### ¿Memoria Insuficiente?

```bash
# Verificar memoria
grep MemTotal /proc/meminfo

# Ajustar swap
```

### ¿Error de Instalación?

Revisar logs:
```bash
cat /u01/app/oraInventory/logs/installActions*.log
```
