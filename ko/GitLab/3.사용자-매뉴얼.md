# GitLab 사용자 매뉴얼

> 개발 및 운영을 위한 GitLab 사용 종합 가이드

---

## 시작하기

### 계정 만들기

1. GitLab 인스턴스로 이동
2. "등록"을 클릭하여 계정 생성
3. 이메일 주소 확인
4. 사용자 프로필 완료

### 프로젝트 만들기

1. "새 프로젝트" 버튼 클릭
2. 프로젝트 템플릿 또는 빈 프로젝트 선택
3 설명 설정
4. 공개 수준 구성 (비공개, 내부, 공개)
. 프로젝트 이름 및5. "프로젝트 만들기" 클릭

---

## Git 작업

### 저장소 클론

```bash
# HTTPS로 클론
git clone https://gitlab.example.com/username/project.git

# SSH로 클론
git clone git@gitlab.example.com:username/project.git
```

### 기본 Git 워크플로우

```bash
# 새 브랜치 만들기
git checkout -b feature/new-feature

# 변경하고 스테이징
git add .
git commit -m "새 기능 추가"

# 원격에 푸시
git push origin feature/new-feature
```

## 이슈 추적

### 이슈 만들기

1. 프로젝트로 이동
2. 이슈 > 새 이슈로 이동
3. 제목 및 설명 추가
4. 레이블, 마일스톤, 담당자 할당
5. 가중치 및 기한 설정
6. 이슈 제출

---

## 병합 요청

### 병합 요청 만들기

1. 기능 브랜치를 GitLab에 푸시
2. "병합 요청 만들기" 버튼 클릭
3. 제목 및 설명 작성
4. 소스 및 대상 브랜치 선택
5. 레이블 및 검토자 추가
6. 병합 요청 제출

## CI/CD 파이프라인

### 기본 .gitlab-ci.yml

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build

test:
  stage: test
  script:
    - npm test

deploy:
  stage: deploy
  script:
    - npm run deploy
  only:
    - main
```
