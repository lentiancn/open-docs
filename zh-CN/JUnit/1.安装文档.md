# JUnit 安装指南

JUnit 是 Java 最流行的单元测试框架，也是 Spring Boot 的默认测试框架。

## 版本概述

JUnit 有两个主要版本：

| 版本 | 名称 | Java 版本 | 发布日期 | 状态 |
|------|------|----------|----------|------|
| JUnit 5 | Jupiter | Java 8+ | 2016 | 当前推荐 |
| JUnit 4 | - | Java 5+ | 2005 | 旧版本 |

**建议：** 所有新项目都使用 JUnit 5。JUnit 4 仅为向后兼容而维护。

### 版本差异

| 特性 | JUnit 4 | JUnit 5 |
|------|----------|----------|
| 注解 | @Test, @Before, @After | @Test, @BeforeEach, @AfterEach |
| 测试类 | 只能是 public 类 | 任意类 |
| 断言 | Assert 方法 | 带 lambda 支持的 assertions 类 |
| 参数化测试 | @Parameters | @ParameterizedTest |
| 嵌套测试 | @RunWith(Nested.class) | @Nested |

## 前置条件

### Java 要求

| JUnit 版本 | 最低 Java 版本 |
|------------|-------------|
| JUnit 5.11.x | Java 8+ |
| JUnit 4.13.x | Java 5+ |

- **Maven**: 3.6+

## 安装依赖

### Maven (JUnit 5 - 推荐)

在 `pom.xml` 中添加：

```xml
<project>
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-project</artifactId>
    <version>1.0-SNAPSHOT</version>
    
    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <junit.version>5.11.0</junit.version>
    </properties>

    <dependencies>
        <!-- JUnit 5 -->
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter</artifactId>
            <version>${junit.version}</version>
            <scope>test</scope>
        </dependency>
        
        <!-- JUnit Platform Launcher (IDE 支持) -->
        <dependency>
            <groupId>org.junit.platform</groupId>
            <artifactId>junit-platform-launcher</artifactId>
            <version>1.11.0</version>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>3.2.5</version>
            </plugin>
        </plugins>
    </build>
</project>
```

### Gradle (JUnit 5 - 推荐)

在 `build.gradle` 中添加：

```groovy
plugins {
    id 'java'
    id 'application'
}

repositories {
    mavenCentral()
}

dependencies {
    // JUnit 5 (Jupiter)
    testImplementation 'org.junit.jupiter:junit-jupiter:5.11.0'
    
    // JUnit Platform IDE 支持
    testRuntimeOnly 'org.junit.platform:junit-platform-launcher:1.11.0'
}

test {
    useJUnitPlatform()  // 启用 JUnit 5
}
```

### JUnit 4 (旧版 - 不推荐)

仅用于无法迁移的现有项目：

```xml
<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.13.2</version>
    <scope>test</scope>
</dependency>
```

## Spring Boot 集成

### Spring Boot 2.2+ (默认 JUnit 5)

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.3.0</version>
</parent>

<dependencies>
    <!-- 包含 JUnit 5 + Mockito + AssertJ -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

此启动器包含：
- **JUnit Jupiter** - JUnit 5 框架
- **Mockito** - Mock 框架
- **AssertJ** - 流式断言
- **Spring Test** - Spring 测试支持

### Spring Boot 2.1 及更早版本

需要手动添加 JUnit 5：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>

<!-- 添加 JUnit 5 -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <scope>test</scope>
</dependency>
```

## 从 JUnit 4 迁移到 JUnit 5

### 注解变化

| JUnit 4 | JUnit 5 |
|----------|-----------|
| @Before | @BeforeEach |
| @After | @AfterEach |
| @BeforeClass | @BeforeAll |
| @AfterClass | @AfterAll |
| @Ignore | @Disabled |
| @RunWith | @ExtendWith |

### 导入变化

```java
// JUnit 4
import org.junit.Test;
import org.junit.Before;
import org.junit.After;

// JUnit 5
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;
```

### 使用 JUnit Vintage 进行渐进式迁移

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <scope>test</scope>
</dependency>
```

这允许在 JUnit 5 平台上运行 JUnit 4 测试。

## 故障排除

### 测试不运行

**问题：** IDE 中测试不运行

**解决方案：**
1. 确保 JUnit 5 依赖正确
2. 检查 Maven/Gradle 能解析依赖
3. 验证 IDE 支持 JUnit 5
4. 清除 IDE 缓存并重建

### 找不到测试

**问题：** 显示"找不到测试"

**解决方案：**
1. 确保测试方法有 @Test 注解
2. 确保测试类名以"Test"结尾
3. 检查包结构
4. 验证测试在 src/test/java

## 最佳实践

1. **使用 JUnit 5** - 始终优先选择 JUnit 5
2. **有意义的测试名称** - 命名要描述场景
3. **每个测试一个断言** - 或使用 assertAll
4. **测试行为，而非实现** - 关注是什么，而非如何
5. **谨慎使用 @BeforeEach** - 保持设置最小化
