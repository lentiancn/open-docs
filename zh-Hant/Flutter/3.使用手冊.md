# Flutter 使用手冊

## 基礎概念

### Widget 介紹

Widget 是 Flutter UI 的基本建構區塊。在 Flutter 中，所有東西都是 Widget，包括佈局、padding、align 等都是 Widget。Widget 描述了它們的視圖在給定目前配置和狀態時應該是什麼樣子。

Flutter 的 Widget 是不可變的（immutable），當需要改變 UI 時，需要重新建立 Widget 或使用 StatefulWidget 和狀態管理。Widget 可以巢狀形成 Widget 樹，Flutter 會高效地比較新舊 Widget 樹，只渲染實際變化的部分。

常見的 Widget 包括：Text（文字）、Image（圖片）、Icon（圖示）、Container（容器）、Row/Column（水準/垂直佈局）、Stack（層疊佈局）、ListView（列表）等。

### StatelessWidget 和 StatefulWidget

StatelessWidget 是無狀態的 Widget，適合用於靜態的 UI 展示，如文字、圖片等。定義一個 StatelessWidget 只需實現 build 方法，返回需要渲染的 Widget 樹。

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget 是有狀態的 Widget，適合用於需要動態更新 UI 的場景，如計數器、表單等。StatefulWidget 需要配合 State 使用，狀態變化時呼叫 setState 方法通知 Flutter 重新渲染。

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## 佈局基礎

### 線性佈局

Row 和 Column 是最常用的線性佈局 Widget。Row 用於水準排列子 Widget，Column 用於垂直排列子 Widget。可以使用 MainAxisAlignment 和 CrossAxisAlignment 控制對齊方式。

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### 層疊佈局

Stack 用於建立層疊佈局，可以在另一個 Widget 上面放置 Widget。Positioned 用於定位 Stack 中的子 Widget。

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### 列表佈局

ListView 是最常用的列表 Widget，適用於長列表。ListView.builder 適合大量資料的列表，可以實現延遲載入。ListView.separated 可以添加分隔線。

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## 狀態管理

### setState

setState 是最基本的狀態管理方式，適用於簡單的局部狀態。在 StatefulWidget 中，呼叫 setState 會觸發 Widget 重新建立。

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider 是 Flutter 官方推薦的狀態管理方案之一。通過 Provider，可以在 Widget 樹中共享狀態，避免層層傳遞。

```dart
// 定義 ChangeNotifier
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// 使用 Provider
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

### 其他狀態管理方案

對於複雜應用，還有 Riverpod、GetX、BLoC 等狀態管理方案。Riverpod 是 Provider 的升級版，提供了更好的效能和類型安全。GetX 整合了狀態管理、路由管理和依賴注入。BLoC 使用事件流的方式管理狀態，適合大型複雜應用。

## 網路請求

### HTTP 封包

使用 http 封包可以發起簡單的 HTTP 請求：

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

### Dio

Dio 是一個強大的 HTTP 客戶端，支援攔截器、請求取消、檔案上傳等功能：

```dart
import 'package:dio/dio.dart';

final dio = Dio();
final response = await dio.get('https://api.example.com/data');
```

## 本地儲存

### SharedPreferences

用於儲存簡單的鍵值對資料：

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

### SQLite

對於結構化資料，可以使用 sqflite 封包：

```dart
import 'package:sqflite/sqflite.dart';

final db = await openDatabase('my.db');
await db.insert('users', {'name': 'Flutter'});
```

## 導航和路由

Flutter 使用 Navigator 進行頁面導航。基本操作包括 Navigator.push（推入新頁面）、Navigator.pop（返回）、Navigator.pushReplacement（替換目前頁面）等。

命名路由是推薦的路由管理方式：

```dart
// 定義路由
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// 導航
Navigator.pushNamed(context, '/detail');
```

## 打包和發布

### Android 發布

使用 keytool 產生簽章金鑰：

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

在 android/app/build.gradle 中配置簽章，然後在專案根目錄執行：

```bash
flutter build apk --release
```

### iOS 發布

在 Xcode 中配置簽章憑證和描述檔案，然後執行：

```bash
flutter build ios --release
```

需要注意的是，iOS 發布需要 Apple 開發者帳號和相應的憑證。
