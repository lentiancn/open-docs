# 사용 매뉴얼

이 매뉴얼에서는 라우터 구성, 업스트림 관리, 부하 분산, 속도 제한, 인증 등 Apache APISIX의 핵심 기능과 사용 방법을 설명합니다.

## 기본 개념

### Route (라우터)

라우터는 업스트림 서비스에 접근하는 경로입니다. APISIX에서 라우터는 먼저 사전 정의된 규칙을 사용하여 클라이언트 요청을 일치시킨 후, 해당 플러그인을 로드하고 실행하며, 마지막으로 요청을 특정 업스트림 서비스로转发합니다.

가장 간단한 라우터는 일치하는 경로와 업스트림 주소 두 가지 정보로만 구성됩니다.

### Upstream (업스트림)

업스트림은 동일한 기능을 가진 노드 집합으로, 가상 호스트에 대한 추상화입니다. 업스트림은 사전 정의된 규칙을 사용하여 여러 서비스 노드에 대한 부하 분산을 수행할 수 있습니다.

### Consumer (컨슈머)

컨슈머는 API를 사용하는 애플리케이션 또는 개발자입니다. APISIX에서 컨슈머는 전역적으로 고유한 이름을 필요로 하며 인증 플러그인에 바인딩되어야 합니다.

### Plugin (플러그인)

APISIX는 게이트웨이 기능을 확장하기 위한 다양한 플러그인 시스템을 제공합니다. 플러그인은 라우터, 컨슈머 또는 전역 수준에서 구성할 수 있습니다.

## 라우터 구성

관리 API를 통해 라우터를 쉽게 생성할 수 있습니다. 다음 예제는 httpbin.org로 요청을転送하는 라우터를 생성합니다:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "getting-started-ip",
  "uri": "/ip",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

생성 성공 시 `HTTP/1.1 201 Created`가 반환됩니다.

### 라우터 확인

```bash
curl "http://127.0.0.1:9080/ip"
```

다음과 같은 결과가 반환됩니다:

```json
{
  "origin": "183.94.122.205"
}
```

### 라우터 일치 규칙

APISIX는 다양한 라우터 일치 방식을 지원합니다:

**URI 일치:**

```json
{
  "uri": "/api/v1/*",
  "uri": ["/api/v1/*", "/api/v2/*"]
}
```

**호스트 일치:**

```json
{
  "host": "example.com",
  "host": ["example.com", "*.example.com"]
}
```

**메서드 일치:**

```json
{
  "methods": ["GET", "POST"]
}
```

**헤더 일치:**

```json
{
  "headers": {
    "Content-Type": "application/json",
    "X-API-Key": "*"
  }
}
```

## 부하 분산

APISIX는 라운드로빈, 가중 라운드로빈, 최소 연결 등 다양한 부하 분산 알고리즘을 지원합니다.

### 가중 부하 분산 구성

여러 업스트림 서비스를 가진 라우터를 생성합니다:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "load-balance-route",
  "uri": "/headers",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:443": 1,
      "mock.api7.ai:443": 1
    },
    "pass_host": "node",
    "scheme": "https"
  }
}'
```

**매개변수 설명:**

- `type`: 부하 분산 알고리즘 유형 - `roundrobin`, `weighted_roundrobin`, 또는 `least_conn`
- `nodes`: 업스트림 서비스 노드 형식 `{"host:port": weight}`
- `pass_host`: `node`로 설정하면 요청 헤더를 업스트림에 전달
- `scheme`: 업스트림 프로토콜, `http` 또는 `https`

## 속도 제한

APISIX는 `limit-count` 플러그인을 통해 유연한 속도 제한 기능을 제공합니다.

### 속도 제한 활성화

기존 라우터에 limit-count 플러그인을 추가합니다:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "count": 2,
      "time_window": 10,
      "rejected_code": 503
    }
  }
}'
```

**매개변수 설명:**

- `count`: 시간 창 내에서 허용되는 최대 요청 수
- `time_window`: 시간 창 크기(초)
- `rejected_code`: 제한 초과 시 반환되는 HTTP 상태 코드

이 구성은 들어오는 트래픽을 10초당 최대 2개의 요청으로 제한합니다.

### 속도 제한 비활성화

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "limit-count": {
      "_meta": {
        "disable": true
      }
    }
  }
}'
```

## 인증

APISIX는 Key Auth, Basic Auth, JWT, OAuth2 등 다양한 인증 방법을 지원합니다.

### 키 인증

#### 컨슈머 생성

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/consumers" -X PUT -d '
{
  "username": "tom",
  "plugins": {
    "key-auth": {
      "key": "secret-key"
    }
  }
}'
```

> ⚠️ 프로덕션에서는 복잡한 키를 사용하세요.

#### 라우터에 키 인증 활성화

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes/getting-started-ip" -X PATCH -d '
{
  "plugins": {
    "key-auth": {}
  }
}'
```

### 기타 인증 방법

**JWT 인증:**

```json
{
  "plugins": {
    "jwt-auth": {
      "key": "user-key",
      "secret": "my-secret-key"
    }
  }
}
```

**Basic 인증:**

```json
{
  "plugins": {
    "basic-auth": {
      "hide_credentials": false
    }
  }
}
```

## 서비스 거버넌스

### 그레이드 배포

APISIX는 가중치 기반 그레이드 배포 및 카나리 배포를 지원합니다:

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "canary-route",
  "uri": "/api/*",
  "upstream": {
    "type": "weighted_roundrobin",
    "nodes": [
      {"host": "v1.example.com", "port": 80, "weight": 90},
      {"host": "v2.example.com", "port": 80, "weight": 10}
    ]
  }
}'
```

### 요청 재작성

`proxy-rewrite` 플러그인을 사용하여 요청을 재작성합니다:

```json
{
  "plugins": {
    "proxy-rewrite": {
      "uri": "/new-path/$uri",
      "headers": {
        "X-New-Header": "value"
      }
    }
  }
}
```

### 응답 캐싱

`proxy-cache` 플러그인을 사용하여 업스트림 응답을 캐시합니다:

```json
{
  "plugins": {
    "proxy-cache": {
      "cache_key": ["uri", "query_string"],
      "cache_http_status": [200, 301, 404],
      "cache_ttl": 60
    }
  }
}
```

## 모니터링 및 가시성

### Prometheus 통합

```json
{
  "plugins": {
    "prometheus": {
      "prefer_name": true
    }
  }
}
```

지표 접근:

```bash
curl "http://127.0.0.1:9091/prometheus"
```

## 모범 사례

1. **의미 있는 ID 사용**: 라우터, 업스트림 등에 설명적인 ID 사용
2. **적절한超时 설정**: 업스트림 응답 시간에 기반한 합리적인超时 매개변수 구성
3. **상태 검사 활성화**: 중요한 서비스에 능동 및 수동 상태 검사 구성
4. **적절한 속도 제한 구성**: 비즈니스 요구 및 서버 용량에 따른 속도 제한 규칙 설정
5. **인증 활성화**: 프로덕션 환경에서는 반드시 인증 활성화
6. **TLS 사용**: 외부에 노출된 인터페이스는 HTTPS 사용
7. **关键 지표 모니터링**: Prometheus 모니터링 구성 및 경고 설정
