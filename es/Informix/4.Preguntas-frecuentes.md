# Preguntas Frecuentes de IBM Informix

## Preguntas Generales

### P: ¿Cuál es la diferencia entre IBM Informix y HCL Informix?

R: IBM Informix es vendido y comercializado por IBM, mientras que HCL Informix es desarrollado y soportado por HCLSoftware. Ambos comparten la misma base de código y son funcionalmente equivalentes para implementaciones locales. IBM maneja ventas y licencias, mientras HCL maneja desarrollo y soporte técnico.

### P: ¿Qué edición debo elegir para desarrollo?

R: Use **Developer Edition** (gratis) o **Innovator-C Edition** (gratis) para desarrollo y pruebas. Ambas tienen limitaciones de recursos pero incluyen funcionalidad completa:
- Developer Edition: 1 CPU VP/Core, 1GB RAM, 8GB almacenamiento
- Innovator-C Edition: 2 CPU Cores, 8GB RAM, 50GB almacenamiento

### P: ¿Es Informix adecuado para aplicaciones IoT?

R: Sí, Informix es excelente para IoT debido a su pequeña huella (tan pequeña como 100 MB), bajos requisitos de administración, soporte integrado de Series Temporales y capacidad de ejecutarse en dispositivos perimetrales incluyendo Raspberry Pi.

### P: ¿Qué lenguajes de programación son soportados?

R: Informix soporta múltiples interfaces de programación:
- SQL (nativo)
- ODBC
- JDBC
- ESQL/C
- PHP
- Python
- Node.js
- Compatibilidad MongoDB (JSON/BSON)

## Problemas de Instalación

### P: La instalación falla con error "Sin permiso"

R: Asegúrese de tener privilegios root/sudo. Cree el usuario y grupo informix antes de la instalación:

```bash
groupadd informix
useradd -g informix -d /home/informix informix
```

### P: El servidor no inicia después de la instalación

R: Verifique lo siguiente:
- Las variables de entorno están configuradas correctamente (`INFORMIXDIR`, `INFORMIXSERVER`, `ONCONFIG`)
- Los archivos `onconfig` y `sqlhosts` son correctos
- Revise los registros de error en `$INFORMIXDIR/tmp/`
- Asegúrese de tener suficiente espacio en disco y permisos

### P: ¿Cómo resolver problemas de clave de licencia?

R: Para versiones de evaluación, asegúrese de haber descargado desde IBM Passport Advantage con los derechos correctos. Para producción, aplique la clave de licencia usando IBM License Key Center.

### P: ¿Qué puertos usa Informix?

R: El puerto predeterminado es 9088. Puede verificar en el archivo `sqlhosts`. Asegúrese de que el firewall permita este puerto para conexiones remotas.

## Problemas Operativos Comunes

### P: ¿Cómo respaldar y restaurar Informix?

R: Use las utilidades `ontape` o `onbar`:

```bash
# Respaldo completo con ontape
ontape -s -L 0

# Restaurar con ontape
ontape -r

# Respaldo con onbar
onbar -b -L 0

# Restaurar con onbar
onbar -r
```

### P: ¿Cómo verificar el estado del servidor?

R: Use el comando `onstat`:

```bash
onstat -        # Estado general
onstat -d       # Uso de espacio en disco
onstat -l       # Logs lógicos
onstat -u       # Usuarios activos
onstat -k       # Información de bloqueos
onstat -g <cmd> # Grupo específico (memoria, bloqueos, etc.)
```

### P: ¿Cómo conectar a Informix desde aplicaciones externas?

R: Configure controladores ODBC, JDBC o clientes nativos. Actualice el archivo `sqlhosts` para conectividad de red. Ejemplo de cadena de conexión JDBC:

```
jdbc:informix-sqli://hostname:9088/dbname:INFORMIXSERVER=nombredelservidor
```

### P: ¿Qué causa el error "Logical Log Full"?

R: Esto ocurre cuando todos los logs lógicos están llenos y no han sido respaldados. Soluciones:
- Respaldar logs lógicos: `ontape -s -L 0`
- Agregar más archivos de log lógico: `onparams -a -d dbspace -s tamaño`
- Verificar transacciones de larga duración

### P: ¿Cómo manejar conflictos de bloqueos?

R: Use `onstat -k` para ver bloqueos y `onstat -u` para identificar sesiones que bloquean. Soluciones:
- Aumentar tiempo de espera de bloqueo
- Usar nivel de aislamiento más bajo (dirty read)
- Terminar sesión que bloquea: `onmode -z id_sesión`

## Rendimiento y Ajuste

### P: ¿Cómo mejorar el rendimiento de consultas?

R:
- Crear índices apropiados para columnas frecuentemente consultadas
- Usar `EXPLAIN` para analizar planes de consulta
- Configurar memoria compartida apropiadamente (`SHMVIRTSIZE`, `BUFFERPOOL`)
- Actualizar estadísticas regularmente: `UPDATE STATISTICS FOR TABLE tabla`
- Considerar usar Informix Warehouse Accelerator para consultas de BI

### P: ¿Cuáles son los parámetros clave de configuración de memoria?

R:
- `SHMVIRTSIZE` - Tamaño de memoria compartida virtual (asignación inicial)
- `SHMADD` - Segmentos adicionales de memoria compartida
- `BUFFERPOOL` - Configuración de caché de datos para diferentes tamaños de página

### P: ¿Cómo monitorear el rendimiento?

R:
- Usar comandos `onstat -g` para varias estadísticas
- Monitorear con IBM Informix Performance Manager (incluido en ediciones avanzadas)
- Verificar planes de consulta con `SET EXPLAIN ON`
