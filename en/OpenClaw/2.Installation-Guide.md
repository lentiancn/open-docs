# OpenClaw Installation Guide

This guide walks through installing OpenClaw on different operating systems.

## Requirements

- **Node.js**: version 22 or higher
- **Operating system**: macOS, Linux, or Windows
- **Optional**: Docker (for sandbox isolation features)

## Installation Methods

### macOS and Linux (Recommended)

The easiest approach is using the official install script:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

This script detects your system environment, installs dependencies, and handles basic configuration.

### Windows

Run this in PowerShell:

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### Direct npm Installation

If you already have Node.js set up:

```bash
npm install -g openclaw@latest
```

### From Source

For the latest development version or custom setups:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## Post-Install Setup

### Run the Setup Wizard

After installation, it's recommended to run the wizard for basic configuration:

```bash
openclaw onboard --install-daemon
```

The wizard walks you through:
- API key configuration
- Service port settings
- Channel login (WhatsApp, Telegram, Discord, etc.)
- Security and access control settings

### Check Service Status

If you installed as shown above, the service should be running:

```bash
openclaw gateway status
```

### Run in Foreground

To see logs directly in your terminal (useful for debugging):

```bash
openclaw gateway --port 18789
```

### Open Web Interface

```bash
openclaw dashboard
```

Or just open in your browser: `http://127.0.0.1:18789/`

## Environment Variables

Sometimes you need to customize data storage locations:

- `OPENCLAW_HOME`: Sets the home directory for OpenClaw
- `OPENCLAW_STATE_DIR`: Overrides the state data directory
- `OPENCLAW_CONFIG_PATH`: Specifies a custom config file path

## Verify Installation

After installing, you can:

1. Run `openclaw --version` to confirm it worked
2. Run `openclaw doctor` to check system health
3. Open the web interface and send a test message

## Common Issues

### Check Node.js Version

```bash
node --version
```

Make sure it's 22 or higher. Upgrade Node.js if needed.

### Git Not Found on Windows

Ensure Git is installed and in your system PATH. Download from git-scm.com.

### Permission Errors

If you hit permission issues, try:

```bash
sudo npm install -g openclaw@latest
```
