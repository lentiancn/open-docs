# Häufig gestellte Fragen

Dieser Abschnitt fasst häufige Fragen und Antworten während der Jira-Nutzung zusammen und hilft Ihnen, schnell auftretende Schwierigkeiten zu lösen.

## Konto und Berechtigungen

### Passwort zurücksetzen

Wenn Sie Ihr Passwort vergessen haben, klicken Sie auf der Anmeldeseite auf den Link "Passwort vergessen". Das System sendet einen Zurücksetzungslink an Ihre registrierte E-Mail-Adresse. Wenn Sie die E-Mail nicht erhalten können, überprüfen Sie Ihren Spam-Ordner oder wenden Sie sich an Ihren Administrator.

### Benutzer hinzufügen oder entfernen

Nur Administratoren können Benutzer hinzufügen oder entfernen. Melden Sie sich bei Jira an, gehen Sie zu den Administrationseinstellungen und finden Sie die Option "Benutzerverwaltung". Hier können Sie neue Benutzer hinzufügen, vorhandene Benutzer löschen oder Benutzerberechtigungen verwalten. Um Benutzer hinzuzufügen, müssen Sie ihre E-Mail-Adresse eingeben, und das System sendet eine Einladungs-E-Mail.

### Gibt es ein Benutzerlimit für die kostenlose Version

Ja, der Free-Plan unterstützt maximal 10 Benutzer. Wenn Sie mehr Benutzer hinzufügen müssen, müssen Sie auf Standard, Premium oder Enterprise upgraden. Wenn das Benutzerlimit überschritten wird, werden Sie vom System zum Upgrade aufgefordert.

## Projekte und Workflows

### Benutzerdefinierte Workflows erstellen

Administratoren können Workflows für Projekte erstellen und ändern. Gehen Sie zu den Projekteinstellungen und finden Sie die Option "Workflows". Sie können neue Workflow-Schemata hinzufügen und verschiedene Status und Übergangsregeln definieren. Jeder Übergang kann Bedingungen, Trigger und Nachfunktionen festlegen.

### Können Probleme zwischen Projekten verschoben werden

Ja. Sie können Probleme von einem Projekt in ein anderes verschieben. Beim Verschieben von Problemen können Sie wählen, ob Sie Verlauf, Kommentare und Anhänge behalten möchten. Einige benutzerdefinierte Felder sind möglicherweise nicht zwischen verschiedenen Projekten verschiebbar.

### Probleme stapelweise aktualisieren

Jira unterstützt die Stapelbearbeitung. Nachdem Sie mehrere Probleme ausgewählt haben, können Sie zugewiesene Personen, Prioritäten, Tags und andere Felder stapelweise ändern. Markieren Sie in der Listenansicht die zu bearbeitenden Probleme und verwenden Sie die Stapelverarbeitung für Updates.

## Integrationen und Automatisierung

### Welche Drittanbieter-Integrationen werden unterstützt

Jira unterstützt die Integration mit einer großen Anzahl von Drittanbieter-Tools. Hauptkategorien umfassen: Code-Management-Tools wie GitHub, GitLab, Bitbucket; Kommunikationstools wie Slack, Microsoft Teams; Design-Tools wie Figma, Adobe XD; Continuous-Integration-Tools wie Jenkins, CircleCI. Die vollständige Integrationsliste finden Sie im Atlassian Marketplace.

### Automatisierungsregeln einrichten

Finden Sie "Automatisierung" in den Projekteinstellungen und klicken Sie auf "Regel erstellen". Sie können Trigger (wann ausgelöst werden soll), Bedingungen (welche Bedingungen erfüllt sein müssen) und Aktionen (was getan werden soll) festlegen. Beispielsweise können Sie eine Regel einrichten, die automatisch eine Benachrichtigung an den Problem-Ersteller sendet, wenn ein Problem als "gelöst" markiert wird.

### Webhooks konfigurieren

Webhooks ermöglichen Jira die Echtzeitkommunikation mit externen Systemen. Finden Sie Webhooks in den Administrationseinstellungen unter "Entwicklertools" und fügen Sie neue Webhooks mit Triggerbedingungen und Ziel-URLs hinzu. Wenn Triggerbedingungen erfüllt sind, sendet Jira POST-Anfragen an die angegebene URL.

## Abrechnung und Abonnements

### Pläne upgraden oder downgraden

Finden Sie "Abrechnung" oder "Abonnement" in den Administrationseinstellungen, um Ihren aktuellen Plan anzuzeigen und ein Upgrade oder Downgrade zu wählen. Upgrades werden sofort wirksam, Downgrades werden am Ende des aktuellen Abrechnungszyklus wirksam. Einige Funktionen erfordern möglicherweise eine manuelle Konfiguration, um sich an die Grenzen des neuen Plans anzupassen.

### Kann ich monatlich zahlen

Ja, Jira unterstützt monatliche Abonnements. Monatliche Abonnements ermöglichen eine flexiblere Anpassung der Benutzeranzahl, sind aber normalerweise teurer als Jahresabonnements. Die Auswahl von Jahresabonnements kann bis zu 17% an Rabatten sparen.

### Abonnement kündigen

Sie können Ihr Jira-Abonnement jederzeit kündigen. Nach der Kündigung verlieren Sie den Zugang zu kostenpflichtigen Funktionen, können aber weiterhin den Free-Plan nutzen oder Daten exportieren. Die Kündigung erfolgt im Abschnitt "Abrechnung" der Administrationseinstellungen.

## Daten und Sicherheit

### Daten exportieren

Jira ermöglicht Administratoren den Export von Projektdaten. Finden Sie "Import und Export" in den Projekteinstellungen, um als CSV oder XML zu exportieren. Exportierte Daten umfassen Probleme, Kommentare, Anhänge und mehr.

### Wo werden Daten gespeichert

Daten der Cloud-Version werden auf Atlassian-Cloud-Servern gespeichert, wobei der geografische Standort von Ihrer gewählten Region abhängt. Beim Kauf von Enterprise-Plänen können Sie bestimmte Datenresidenzregionen auswählen, um Compliance-Anforderungen zu erfüllen.

### Wie wird Datensicherheit gewährleistet

Jira bietet mehrschichtige Sicherheit: Verschlüsselung bei Übertragung und im Ruhezustand; SSO-Single-Sign-On und SCIM-Benutzerverwaltungsunterstützung; IP-Zulassungslisten zur Einschränkung von Zugriffsquellen; vollständige Audit-Protokolle, die alle kritischen Operationen aufzeichnen. Enterprise-Pläne enthalten auch erweiterte Sicherheitsfunktionen von Atlassian Guard.

## Leistung und Nutzung

### Was tun, wenn Probleme langsam laden

Wenn Jira langsam lädt, versuchen Sie Folgendes: Browser-Cache und Cookies löschen; Netzwerkverbindung auf Stabilität prüfen; unnötige Browser-Erweiterungen schließen; anderen Browser versuchen. Wenn das Problem weiterhin besteht, hat das Projekt möglicherweise zu viele Daten. Erwägen Sie, alte Daten zu archivieren oder Ansichtsfilters zu optimieren.

### Wie man nach bestimmten Problemen sucht

Jira bietet leistungsstarke Suchfunktionen. Sie können JQL (Jira Query Language) für erweiterte Suchen verwenden, z.B. Probleme nach Zugewiesenem, Status oder Erstellungsdatum suchen. Einfache Suchen können das Schnellsuchfeld verwenden und Problem-Schlüsselwörter eingeben.
