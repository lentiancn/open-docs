# LivePortrait 使用手册

本文档详细介绍LivePortrait的各种使用方法，包括基础推理、Gradio界面、动物模式、视频编辑等功能。

## 基础推理

### 图像驱动

最基本的用法是使用静态图像作为源，驱动视频作为动画参考：

```bash
python inference.py
```

默认情况下，脚本会使用assets/examples/source目录下的示例图像和assets/examples/driving目录下的示例驱动视频。运行成功后，结果将保存在animations目录中，输出文件命名为“源文件--驱动文件_concat.mp4”。

输出视频包含三个部分：驱动视频、源图像/视频、以及生成的动画结果，按顺序拼接在一起。

### 自定义输入

您可以通过命令行参数指定自己的源文件和驱动文件：

```bash
# 源文件为图像
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# 源文件为视频
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

其中-s参数指定源图像或源视频路径，-d参数指定驱动视频路径。

### 查看更多选项

查看所有可用的命令行参数：

```bash
python inference.py -h
```

参数包括输出路径、图像尺寸、动画速度、拼接方式等详细配置。

## 使用Gradio Web界面

对于不熟悉命令行的用户，LivePortrait提供了基于Gradio的图形化界面。

### 启动人类模式界面

```bash
python app.py
```

启动后，打开浏览器访问显示的URL（通常是http://localhost:7860）。在界面中上传源图像或视频，然后选择驱动视频，点击生成按钮即可获得动画结果。

可以通过--server_port参数指定端口，--share参数生成公开链接，--server_name参数指定服务器地址。

### 启动动物模式界面

在Linux系统上使用NVIDIA GPU，可以启动动物模式界面：

```bash
python app_animals.py
```

## 驱动视频规范

为了获得最佳效果，驱动视频需要满足以下要求：

**宽高比**：建议裁剪为1:1正方形（如512x512或256x256像素）。也可以启用自动裁剪功能（--flag_crop_driving_video）。

**内容要求**：驱动视频应聚焦于头部区域，类似于示例视频。应尽量减少肩部动作。

**首帧要求**：驱动视频的第一帧应为正面面部表情自然的中性面孔。这是确保动画效果准确的关键。

### 自动裁剪

如果驱动视频不满足上述要求，可以使用自动裁剪功能：

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

如果自动裁剪效果不理想，可以调整--scale_crop_driving_video和--vy_ratio_crop_driving_video参数来微调缩放和偏移。

## 使用运动模板

为了加速推理并保护隐私，LivePortrait支持使用预生成的运动模板文件（.pkl格式）。这些模板文件记录了驱动视频中的运动信息，可以重复使用而无需每次都处理原始视频。

### 使用方法

```bash
# 肖像动画
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# 肖像视频编辑
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

模板文件不仅加速推理，还可以保护隐私——使用模板时无需上传原始驱动视频。

## 肖像视频编辑（Video-to-Video）

LivePortrait支持对现有视频中的人物表情和头部动作进行编辑修改。

### 基本用法

```bash
# 源为图像，驱动为视频
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### 调整动画参数

**表情强度**：通过--driving_multiplier参数调整表情变化的强度。值大于1会增强表情变化，小于1会减弱。

**拼接控制**：使用--flag_stitching参数控制是否启用拼接功能。关闭拼接可能会改变面部区域的呈现方式。

## 推理加速

### Torch Compile加速

LivePortrait支持使用PyTorch的torch.compile功能加速推理。首次推理时会触发优化过程（约一分钟），之后的推理速度将提升20%-30%。加速效果可能因CUDA版本不同而有所差异。

```bash
python app.py --flag_do_torch_compile
```

注意：torch.compile不支持Windows和macOS系统。

### 速度测试

项目提供了推理速度测试脚本：

```bash
python speed.py
```

测试结果会显示各个模块的处理耗时，帮助您了解性能瓶颈。

## 动物模式

动物模式支持对猫和狗等宠物的肖像进行动画处理。

### 使用条件

动物模式仅在以下环境中经过测试：Linux系统搭配NVIDIA GPU、Windows系统搭配NVIDIA GPU。

### 安装要求

在使用动物模式之前，需要按照安装指南中的说明安装X-Pose依赖，并下载动物模型权重。

### 运行动物模式推理

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### 动物模式界面

```bash
python app_animals.py
```

## 区域控制

LivePortrait支持对图像的特定区域进行动画控制，实现更精细的效果调整。

### 启用区域控制

在Gradio界面中可以找到区域控制选项，选择要驱动的面部区域（如眼睛、嘴巴、头部姿态等）。

## 肖像编辑

在Gradio界面中，用户可以精确编辑源肖像的各个属性，包括表情、眼神方向、头部角度等。

## 隐私保护

LivePortrait提供了多种隐私保护功能：

**运动模板**：使用.pkl模板文件代替原始驱动视频，避免上传敏感内容

**自动裁剪**：驱动视频自动裁剪聚焦头部，减少背景信息

**视频保护**：提供模板制作功能，将视频转换为运动模板后再使用

## 输出格式

默认输出格式为MP4视频文件，包含三个部分：驱动视频、源内容、生成结果。可以使用--no_flag_concat参数只输出生成的动画部分。

## 社区工具集成

LivePortrait被多个社区项目集成使用：

**ComfyUI**：多个ComfyUI节点支持LivePortrait，提供实时预览功能

**Stable Diffusion WebUI**：sd-webui-live-portrait扩展将LivePortrait集成到SD WebUI中

**FaceFusion**：3.0版本集成了LivePortrait作为表情恢复和面部编辑处理器

**HuggingFace Spaces**：在线体验LivePortrait，无需本地安装

## 最佳实践

为了获得最佳动画效果，建议遵循以下最佳实践：

选择高质量的源图像：清晰的正面肖像效果最好

使用专业的驱动视频：正面、表情自然的视频驱动效果更佳

控制驱动视频长度：过长的驱动视频会增加处理时间并可能导致效果衰减

合理设置参数：根据实际效果调整表情强度和其他参数

注意光照一致：源图像和驱动视频的光照条件尽量保持一致
