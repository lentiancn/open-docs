# Yarn ユーザーマニュアル

## 基本コマンド

### 一般的なコマンド

- `yarn add` - プロジェクトに依存関係を追加
- `yarn install` - プロジェクトの依存関係をインストール
- `yarn remove` - プロジェクトから依存関係を削除
- `yarn run` - package.json で定義されたスクリプトを実行
- `yarn init` - 新しいパッケージを作成
- `yarn info` - パッケージに関する情報を表示
- `yarn why` - パッケージが必要な理由を表示

### 高度なコマンド

- `yarn dlx` - 一時的な環境でパッケージを実行
- `yarn dedupe` - 重複する範囲を持つ依存関係をデデュープ
- `yarn up` - プロジェクト全体で依存関係をアップグレード
- `yarn pack` - アクティブなワークスペースから tarball を生成
- `yarn rebuild` - プロジェクトのネイティブパッケージを再ビルド

### NPM 関連コマンド

- `yarn npm audit` - インストール済みパッケージに対して脆弱性監査を実行
- `yarn npm publish` - アクティブなワークスペースを npm レジストリに公開
- `yarn npm login` - npm レジストリへのアクセス用の新しいログイン情報を保存
- `yarn npm logout` - npm レジストリからログアウト
- `yarn npm whoami` - 認証済みユーザーの名前を表示

### プラグイン管理

- `yarn plugin list` - 利用可能な公式プラグインを一覧表示
- `yarn plugin import` - プラグインをダウンロード
- `yarn plugin remove` - プラグインを削除
- `yarn plugin runtime` - アクティブなプラグインを一覧表示

## ワークスペース

Yarn はワークスペースを中心に構築された最初のパッケージマネージャーであり、プロジェクトをサブコンポーネントに分割できます。

### ワークスペース設定

ルートの `package.json` でワークスペースを設定:

```json
{
  "workspaces": [
    "packages/*"
  ]
}
```

### ワークスペースコマンド

- `yarn workspaces list` - すべてのワークスペースを一覧表示
- `yarn workspace <name> <command>` - 特定のワークスペースでコマンドを実行

### クロス参照

モノレポのパッケージは互いに依存する必要があります。Yarn は特別な `workspace:` プロトコルのおかげでこれを非常に簡単にします:

```json
{
  "dependencies": {
    "@my-org/utils": "workspace:^"
  }
}
```

`workspace:` プロトコルは通常の semver 範囲または特別な `^` / `~` / `*` トークンを受け入れます。

### フォーカスインストール

ワークスペースを発見する際によくある懸念は、単一のワークスペースで作業したい場合でもすべての依存関係をインストールする必要があることです。Yarn は `yarn workspaces focus` でこの問題を解決します。

このコマンドはワークスペースのリストを取り、推移的依存関係を含めるようにリストを拡張し、それ以外のすべてをインストールから除外します:

```bash
yarn workspaces focus @my-org/app
```

### 並列実行

複数のワークスペースのスクリプトは、同じ名前を持つ場合、`yarn workspaces foreach` を使用して並列で実行できます:

```bash
yarn workspaces foreach --all -pt npm publish
```

## 設定ファイル

### .yarnrc.yml

Yarn v2+ は設定に `.yarnrc.yml` ファイルを使用:

```yaml
nodeLinker: pnp
yarnPath: .yarn/releases/yarn-4.0.0.cjs
```

### .yarnrc (Yarn v1)

Yarn v1 は `.yarnrc` ファイルを使用:

```
--registry https://registry.npmjs.org
--strict-semver true
```

## バージョンの違い

### Yarn v4+ vs Yarn v1

| 機能 | Yarn v4+ | Yarn v1 |
|------|----------|---------|
| アーキテクチャ | プラグインベース (Berry) | 従来 |
| インストール速度 | 高速 (並列インストール) | 標準速度 |
| ディスク容量 | 効率的 (PnP) | 標準 node_modules |
| ワークスペース | ネイティブサポート | 追加設定が必要 |
| プラグインシステム | 組み込みサポート | サポートなし |
| 設定 | .yarnrc.yml | .yarnrc |

### 移行時の考慮事項

- Yarn v4+ はデフォルトで PnP を使用し、PnP をサポートしていないツールと互換性がない可能性があります
- CI/CD 設定を新しいインストールフローに対応するように更新する必要があります
- プラグインシステムにより、より多くのカスタマイズオプションが提供されます
- ワークスペース機能はより強力で柔軟です
