# APISIX Руководство пользователя

Основные функции и советы по использованию APISIX.

## Основные понятия

### Терминология

- **Route**: Определяет правила сопоставления запросов
- **Upstream**: Фактические бэкенд-сервисы
- **Service**: Коллекция маршрутов
- **Plugin**: Логика обработки запросов
- **Consumer**: Потребитель API

### Поток запросов

1. Клиент отправляет запрос в APISIX
2. APISIX сопоставляет с правилами маршрутизации
3. Выполняет логику плагинов
4. Перенаправляет на upstream-сервис
5. Возвращает ответ клиенту

## Управление маршрутами

### Создание маршрута

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "id": "1",
    "uri": "/api/users/*",
    "name": "user-api",
    "upstream": {
      "type": "roundrobin",
      "nodes": {"backend1.example.com:8080": 1}
    }
  }'
```

## Управление upstream

### Создание upstream

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/upstreams/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "type": "roundrobin",
    "nodes": {"host1:8080": 1, "host2:8080": 1}
  }'
```

## Плагины

### Частые плагины

**Контроль трафика:**
- limit-req: Ограничение частоты
- limit-conn: Ограничение подключений

**Аутентификация:**
- jwt-auth: JWT аутентификация
- key-auth: Аутентификация по ключу
- basic-auth: Базовая аутентификация

### Использование плагина limit-req

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{
    "uri": "/api/*",
    "plugins": {
      "limit-req": {
        "rate": 100,
        "burst": 50,
        "rejected_code": 503
      }
    }
  }'
```

### Использование JWT

**1. Создать Consumer:**

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/consumers \
  -H "X-API-KEY: 123456" \
  -d '{
    "username": "john",
    "plugins": {
      "jwt-auth": {"key": "user-key", "secret": "my-secret"}
    }
  }'
```

**2. Включить на маршруте:**

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{"uri":"/api/*","plugins":{"jwt-auth":{}}}'
```
