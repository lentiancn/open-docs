# OpenCode Руководство пользователя

Основное использование и техники для OpenCode.

## Быстрый старт

### Задавайте вопросы

Вы можете попросить OpenCode объяснить кодовую базу:

```
How is authentication handled in @packages/functions/src/api/index.ts
```

### Добавление функций

Перед добавлением функций рекомендуется сначала создать план:

1. Нажмите **Tab** для перехода в режим Plan
2. Опишите функцию, которую хотите
3. После просмотра плана дайте обратную связь или добавьте детали
4. Когда довольны, нажмите **Tab** для перехода в режим Build

### Прямые изменения

Для более простых изменений можно напрямую попросить OpenCode:

```
We need to add authentication to the /settings route. Take a look at how this is handled in the /notes route and implement the same logic.
```

### Отмена изменений

Если изменения не устраивают, используйте команду отмены:

```bash
/undo
```

OpenCode откатит изменения и снова покажет ваше исходное сообщение. Вы можете изменить запрос и повторить.

### Повтор

Используйте `/redo` для повтора изменений.

## Обмен сессиями

```bash
/share
```

Это создаст ссылку на текущую сессию и скопирует в буфер обмена.

## Частые команды

| Команда | Функция |
|---------|----------|
| /init | Инициализировать проект |
| /connect | Подключить LLM-провайдера |
| /undo | Отменить изменения |
| /redo | Повторить изменения |
| /share | Поделиться сессией |

## Лучшие практики

1. Общайтесь с OpenCode как с младшим разработчиком
2. Предоставляйте достаточно контекста и деталей
3. Сначала попросите создать план, затем выполните
4. Используйте /undo если недовольны
5. Регулярно проверяйте и обновляйте AGENTS.md
