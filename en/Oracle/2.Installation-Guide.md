# Oracle Database Installation Guide

## Prerequisites

- **Operating System**: Oracle Linux, RHEL, Solaris, Windows Server, AIX
- **Memory**: Minimum 2GB RAM (recommended 8GB+)
- **Disk Space**: Minimum 8GB for software, additional for data
- **CPU**: 2+ cores recommended

---

## Installation on Linux

### 1. System Requirements Check

```bash
# Check memory
grep MemTotal /proc/meminfo

# Check disk space
df -h

# Check OS version
cat /etc/oracle-release
```

### 2. Create Oracle User

```bash
# Create oinstall group
groupadd -g 54321 oinstall

# Create dba group
groupadd -g 54322 dba

# Create oracle user
useradd -u 54321 -g oinstall -G dba oracle

# Set password
passwd oracle
```

### 3. Configure Kernel Parameters

Edit `/etc/sysctl.conf`:

```bash
kernel.shmmax = 4294967296
kernel.shmall = 2097152
kernel.shmmni = 4096
kernel.sem = 250 32000 100 128
fs.file-max = 6815744
net.ipv4.ip_local_port_range = 9000 65500
```

Apply changes:
```bash
sysctl -p
```

### 4. Run Installer

```bash
# Navigate to Oracle software location
cd /path/to/database/software

# Run installer
./runInstaller
```

Follow the graphical installer wizard:
1. Configure Security Updates
2. Installation Option (Create and configure database)
3. System Class (Server Class)
4. Grid Installation Options (Single instance)
5. Product Languages
6. Database Edition (Enterprise Edition)
7. Installation Location
8. Create Inventory
9. Privileged Operating System Groups
10. Database Configuration (Memory, Character set)
11. Create Database (Sample schemas optional)

## Installation on Windows

### 1. Download Installer

Download from Oracle website: https://www.oracle.com/database/technologies/

### 2. Run Setup

1. Extract downloaded ZIP
2. Run `setup.exe` as Administrator
3. Follow installer wizard
4. Configure Windows Firewall (allow access)
5. Create database using Database Configuration Assistant (DBCA)

---

## Post-Installation Verification

```sql
-- Connect to database
sqlplus / as sysdba

-- Check database status
SELECT name, open_mode FROM v$database;

-- Check version
SELECT * FROM v$version;
```

## Common Issues

### Insufficient Memory

Increase swap space or reduce SGA/PGA memory allocation.

### Disk Space Errors

Ensure adequate disk space in Oracle home directory.

### ORA-12541: TNS:no listener

Start Oracle listener:
```bash
lsnrctl start
```
