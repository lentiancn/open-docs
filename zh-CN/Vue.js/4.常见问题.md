# Vue.js 常见问题

## Vue 2 和 Vue 3 的区别

Vue 3 是当前最新主版本，包含 Vue 2 中没有的新特性（如 Teleport、Suspense、多根元素模板）以及一些非兼容性变更。

**主要优势：**
- 更小的包体积
- 更好的性能  
- 更好的可扩展性
- 更好的 TypeScript/IDE 支持

**选择建议：**
- **新项目推荐使用 Vue 3**
- **需要支持 IE11 时选择 Vue 2**（Vue 3 不支持 IE11）

**迁移信息：**
- 组合式 API 已兼容至 Vue 2.7
- 详细迁移指南：[Vue 3 迁移指南](https://v3-migration.vuejs.org/zh/)

## Vue 2 维护状态

- Vue 2 在 2022 年 6 月发布最后版本 2.7
- **2023 年 12 月 31 日已结束官方维护**
- 需要延长支持的团队可参考 [Vue 2 延长 LTS](https://v2.vuejs.org/lts/)

## 浏览器支持

- **Vue 3.x**：仅支持原生支持 ES2016 的浏览器（不包括 IE11）
- **Vue 2.x**：支持 IE11 及更广泛的浏览器

## 性能表现

- Vue 3 是性能最强的主流前端框架之一
- 在 [js-framework-benchmark](https://krausest.github.io/js-framework-benchmark/current.html) 中表现优异
- 几乎不需要手动优化即可处理大多数应用场景
- 文档站点在 Moto G4 + 低速 4G 网络下仍获得 100 分性能得分

## 包体积

- **最小应用**（构建工具 + tree-shaking）：约 16kb（brotli 压缩）
- **完整功能应用**：约 27kb
- **无构建工具**（包含编译器）：约 41kb
- **渐进式增强场景**：可考虑 [petite-vue](https://github.com/vuejs/petite-vue)（仅 6kb）

## 大规模应用支持

Vue 完全能够胜任大规模应用场景：

- **单文件组件**：提供模块化开发模型
- **组合式 API**：一流的 TypeScript 集成，便于组织复杂逻辑
- **全面工具链**：支持应用持续增长
- **低入门门槛**：降低团队培训成本

## API 风格选择

### 选项式 API vs 组合式 API

**选项式 API 适合：**
- 初学者或面向对象背景的开发者
- 无需构建工具的简单场景
- 渐进式增强应用

**组合式 API 适合：**
- 完整单页应用开发
- 需要更好的逻辑组织和重用
- TypeScript 项目

## JavaScript vs TypeScript

- Vue 本身用 TypeScript 实现，提供一流 TS 支持
- **不强制要求使用 TypeScript**
- 即使使用 JavaScript，也能受益于 TS 友好的 API 设计
- 选择取决于团队背景和项目规模需求

## Vue 与 Web Components

- Vue 设计受 Web Components 启发（如插槽）
- Web Components 更底层（以自定义元素为中心）
- Vue 解决更多上层问题（渲染、状态管理、路由、SSR 等）
- Vue 完全支持在组件中使用原生自定义元素
- 也支持将 Vue 组件导出为原生自定义元素
