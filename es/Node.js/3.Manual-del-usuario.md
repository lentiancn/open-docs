# Manual del Usuario de Node.js

## Primeros Pasos

### Conceptos Básicos

Node.js se construye sobre el motor JavaScript V8 de Chrome y proporciona un modelo de E/S orientado a eventos y no bloqueante que lo hace ligero y eficiente para aplicaciones en tiempo real intensivas en datos.

#### Bucle de Eventos
El bucle de eventos de Node.js es el núcleo de su modelo de concurrencia. A diferencia de los modelos tradicionales basados en hilos, Node.js utiliza un bucle de eventos de un solo hilo que maneja operaciones asíncronas a través de callbacks, promesas o async/await.

#### E/S No Bloqueante
La mayoría de las operaciones de Node.js son no bloqueantes, lo que significa que no detienen la ejecución del programa mientras esperan que las operaciones de E/S se completen. En cambio, utilizan callbacks para manejar los resultados cuando están listos.

### Aplicación Hola Mundo

Crea tu primera aplicación Node.js con este servidor HTTP simple:

```javascript
// server.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

Ejecútalo con: `node server.js`

## Módulos Principales

Node.js viene con un rico conjunto de módulos integrados que proporcionan funcionalidad esencial sin requerir dependencias externas.

### Sistema de Archivos (fs)
Maneja operaciones de archivos de forma síncrona o asíncrona:

```javascript
const fs = require('fs');

// Lectura asíncrona
fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Lectura síncrona (bloquea la ejecución)
try {
  const data = fs.readFileSync('/path/to/file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Escribir archivo
fs.writeFile('/path/to/output.txt', '¡Hola Node.js!', (err) => {
  if (err) throw err;
  console.log('Archivo escrito exitosamente');
});
```

### Servidor HTTP
Crea servidores web y realiza solicitudes HTTP:

```javascript
const http = require('http');
const https = require('https');

// Crear servidor HTTP
const server = http.createServer((req, res) => {
  // Manejar solicitud
});

// Realizar solicitud HTTP
const req = http.request('http://example.com', (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log(data);
  });
});
req.end();
```

## Manejo de Errores

El manejo adecuado de errores es crucial en aplicaciones Node.js debido a su naturaleza asíncrona.

### Manejo de Errores Basado en Callbacks
```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error('Error al leer el archivo:', err);
    return;
  }
  console.log('Contenido del archivo:', data);
});
```

### Manejo de Errores Basado en Promesas
```javascript
fs.promises.readFile('file.txt')
  .then(data => {
    console.log('Contenido del archivo:', data);
  })
  .catch(err => {
    console.error('Error al leer el archivo:', err);
  });
```

### Manejo de Errores con Async/Await
```javascript
async function readFile() {
  try {
    const data = await fs.promises.readFile('file.txt');
    console.log('Contenido del archivo:', data);
  } catch (err) {
    console.error('Error al leer el archivo:', err);
  }
}
```

## Optimización de Rendimiento

### Clustering
Utiliza múltiples núcleos de CPU con el módulo cluster:

```javascript
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`Maestro ${process.pid} está ejecutándose`);
  
  // Crear trabajadores
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Trabajador ${worker.process.pid} murió`);
  });
} else {
  // Los trabajadores comparten conexión TCP
  const http = require('http');
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('hello world\n');
  }).listen(8000);
  
  console.log(`Trabajador ${process.pid} iniciado`);
}
```

### Gestión de Memoria
- Evita fugas de memoria limpiando correctamente los listeners de eventos
- Usa streams para procesar archivos grandes en lugar de cargar todo en memoria
- Monitorea el uso de memoria con `process.memoryUsage()`

### Caching
Implementa estrategias de caching para mejorar el rendimiento:

```javascript
const cache = new Map();

function expensiveOperation(key) {
  if (cache.has(key)) {
    return cache.get(key);
  }
  
  const result = performExpensiveCalculation(key);
  cache.set(key, result);
  return result;
}
```

### Registro de Logs
Implementa registro de logs adecuado para aplicaciones de producción:

```javascript
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));
}
```

```javascript
const { PerformanceObserver, performance } = require('perf_hooks');

obs.observe({ entryTypes: ['measure'] });

performance.mark('start');
// Tu código aquí
performance.mark('end');
performance.measure('Mi Operación', 'start', 'end');
```

### Limitación de Tasa
Implementa limitación de tasa para prevenir abusos:

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutos
  max: 100 // limitar cada IP a 100 solicitudes por windowMs
});

app.use(limiter);
```

Usa diferentes configuraciones para diferentes entornos:

```javascript

const config = {

  }

```
