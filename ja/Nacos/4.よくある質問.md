# Nacos よくある質問

## クイックスタートに関する質問

### Nacosはどのオペレーティングシステムをサポートしていますか？

NacosはLinux、Unix、Mac、Windowsオペレーティングシステムをサポートしています。本番環境ではLinux/Unix/Macを推奨します。

### Nacosに必要なJavaバージョンは？

NacosにはJDK 17以上が必要です。JAVA_HOME環境変数が正しく設定されていることを確認してください。

### Nacosが起動しない場合は？

1. Java環境を確認：java -versionを実行して正しいJDKバージョン인지확인
2. ポート使用状況を確認：8848、9848ポートが占用されていないことを確認
3. ログを確認：logs/start.outまたはlogs/nacos.logのエラーメッセージを確認
4. 認証設定を確認：Nacos 3.0+では認証関連のパラメータ設定が必要です

### Nacosのデフォルトポートを変更するには？

conf/application.propertiesファイルの以下の設定を変更します：

```properties
server.port=8848
```

### Nacosコンソールのデフォルトのユーザー名とパスワードは？

- デフォルトのユーザー名：nacos
- デフォルトのパスワード：nacos@demo

本番環境ではデフォルトのパスワードを変更してください。

## サービスの発見に関する質問

### サービスの登録に成功した後，为什么客户端无法发现服务？

1. サービスの健康状態を確認：インスタンスの状態が健康であることを確認
2. ネットワーク接続を確認：クライアントがNacosサーバーにアクセスできることを確認
3. サービス名を確認：查询的服务名与注册时一致（包括名前空間とグループ）
4. クライアント設定を確認：nacos.server-addr設定が正しいことを確認

### ロードバランシングを実装するには？

Nacosは複数のロードバランシング戦略を提供します：

1. クライアントサイドロードバランシング：Nacos SDKのselectOneHealthyInstanceメソッドを使用
2. 重み付けロードバランシング：インスタンス的重み设定で実装
3. クラスターロードバランシング：クラスター環境で自動的に実装

### サービスインスタンスが下线した後，为什么客户端仍然调用已下线的实例？

1. 健康チェック設定を確認：健康チェックメカニズムが正常に動作していることを確認
2. クライアントキャッシュを確認：クライアントはサービスインスタンスリストをキャッシュし、若干の遅延があります
3. 同期間隔を調整：クライアントのリフレッシュ間隔を変更

## 構成管理に関する質問

### 構成を公開した後，客户端没有收到更新通知？

1. リスナーを確認：クライアントが正しく構成リスナーを追加していることを確認
2. ネットワークを確認：ネットワーク接続が正常であることを確認
3. ログを確認：クライアントログのエラーメッセージを確認
4. dataIdとgroupを確認：完全に一致していることを確認

### マルチ環境構成管理を実装するには？

1. 名前空間（Namespace）の使用：異なる環境では異なる名前空間を使用して完全に分離
2. グループ（Group）の使用：同じ名前空間下で異なるグループを使用して構成を管理
3. DataIdの使用：異なるdataIdを使用して異なる環境の構成を区別

### 構成の履歴バージョンを表示するには？

Nacosコンソールは構成変更履歴機能を提供し、以下を表示できます：

- 構成の変更記録
- 各変更の内容比較
- ワンクickで履歴バージョンにロールバック

### 構成のカナリアリリースを実装するには？

Nacosは構成のカナリアリリース機能をサポートしています：

1. コンソールでカナリア構成を作成
2. カナリアクライアントの範囲を指定
3. カナリア構成効果を検証
4. 完全公開またはロールバック

## クラスター展開に関する質問

### Nacosクラスター展開にはいくつのノードが必要ですか？

本番環境では高可用性を確保するために少なくとも3ノードを展開することを推奨します。

### クラスターモードではどのようにデータストレージを選択しますか？

- テスト/開発環境：内蔵Derbyデータベースを使用可能
- 本番環境：データ信頼性を確保するために外部データベース（MySQL）を使用する必要があります

### クラスターノード間でデータ整合性をどのように保証しますか？

Nacosは自己開発のDistroプロトコルとRaftプロトコルを使用してクラスターデータ整合性を保証します：

- Distroプロトコル：データ同期を担当
- Raftプロトコル：リーダー選出とログレプリケーションを担当

### クラスターノードが故障した場合は？

Nacosクラスターには自动故障转移機能があります：

1. 健康チェックが自動的に故障ノードを検出
2. トラフィックが自動的に健全なノードに切り替え
3. 故障ノードが回復した後、自動的にデータを同期

## 認証とセキュリティに関する質問

### Nacos 3.0の認証を設定するには？

Nacos 3.0以降では认证がデフォルトで有効になっています。起動時に以下を設定する必要があります：

```bash
# トークンシークレットキー
nacos.core.auth.plugin.nacos.token.secret.key=<your-base64-secret-key>

# サービスID鍵
nacos.core.auth.server.identity.key=<your-identity-key>
nacos.core.auth.server.identity.value=<your-identity-value>
```

### Nacos認証を無効にするには？

conf/application.propertiesを変更：

```properties
nacos.core.auth.enabled=false
```

注意：本番環境では認証を無効にすることはお勧めしません。

### 異なるユーザーに権限を割り当てるには？

Nacosは役割ベースのアクセス制御をサポートしています：

1. コンソールでユーザーを作成
2. ユーザーに役割を割り当て
3. 役割の権限を設定

## MCPに関する質問

### MCPサーバーを起動時に互換性问题是怎么办？

自動登録されたMCPサーバーデータとNacos上の对应バージョンのデータの互換性を確認：

- MCPサーバープロトコル（stdio、sse、streamable）が一貫しているか
- サービス参照が一貫しているか
- Toolsデータが一貫しているか（Tools名、Tools数、パラメータタイプ、オプションパラメータ）

注意：Toolsの説明とパラメータの説明は一貫している必要はありません。

### MCPサーバーが登録后发现不了？

1. MCPサーバー状態を確認：サービスが正しく登録されていることを確認
2. 名前空間を確認：クライアントとサービスが同じ名前空間にあることを確認
3. 認証情報を確認：ユーザー名とパスワードが正しいことを確認

### MCPサーバーツール設定を動的に変更するには？

MCPサーバーがNacosに自動登録された後、コンソールでMCPサーバーツールの説明と开关を動的に変更できます。関連する変更はサービスを再起動せずにランタイムでMCPサーバーに動的に適用されます。

## パフォーマンスと最適化に関する質問

### Nacosは 대규모サービス登録をどのように処理しますか？

NacosはAlibabaの百万インスタンススケールから生まれ、以下の機能を備えています：

- 高性能なサービス登録と発見
- 低レイテンシーの構成更新応答
- 大規模なサービスと構成管理

### Nacosのパフォーマンスを最適化するには？

1. 健康チェック間隔を適切に設定
2. 適切なデータベース接続プール設定を使用
3. クライアントキャッシュを有効にしてリクエスト数を減らす
4. 名前空間とグループを適切に計画

### Nacosのメモリ設定の推奨は？

サービス規模に基づいて：

- 小規模（サービス数 < 1000）：2C4G
- 中規模（サービス数 1000-5000）：4C8G
- 大規模（サービス数 > 5000）：8C16G以上

## エコシステム統合に関する質問

### NacosとSpring Cloudを統合するには？

pom.xmlに以下を追加：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2023.0.3.2</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2023.0.3.2</version>
</dependency>
```

### NacosとKubernetesを統合するには？

NacosはKubernetes環境への展開をサポートしています：

- Helm Chart展開
- Operator展開
- カスタムYAML展開

### Nacosはどの言語のSDKをサポートしていますか？

Nacosは次の言語のSDKを提供しています：

- Java（公式）
- Python
- Go
- Node.js
- C/C++
- Ruby
- PHP

## トラブルシューティング

### サービス登録の失敗を調査するには？

1. Nacosサーバーが正常に動作していることを確認
2. ネットワーク接続を確認
3. クライアントログを確認
4. 登録パラメータが正しいことを確認

### 構成の取得の問題を調査するには？

1. 構成が正常に公開されていることを確認
2. dataIdとgroupが一致していることを確認
3. リスナーが正しく追加されていることを確認
4. 具体的なエラーのためにクライアントログを確認

### Nacosログはどこにありますか？

Nacosログファイルは以下にあります：

- メインログ：logs/nacos.log
- 起動ログ：logs/start.out
- アクセスメログ：logs/access.log

## アップグレードと移行

### Nacos 1.xから2.xまたは3.xにアップグレードするには？

1. データをバックアップ：データベースまたはスナップショットファイルをバックアップ
2. アップグレードガイドを参照：对应バージョンのアップグレードドキュメントを確認
3.段階的アップグレード：まずテスト環境で検証
4. 構成を移行：認証などの構成変更に注意

### Nacosデータを移行するには？

1. 元環境の構成データをエクスポート
2. 新環境データベースにインポート
3. データ整合性を検証
4. トラフィックを新環境に切り替え
