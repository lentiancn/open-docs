# Nginx 常见问题

## 安装问题

### Nginx 安装失败怎么办？

1. 检查系统依赖是否完整
2. 查看错误日志了解具体失败原因
3. 确保有足够的磁盘空间
4. 检查是否有权限问题

### 如何查看 Nginx 版本？

```bash
nginx -v
nginx -V  # 详细信息，包括编译参数
```

### Nginx 支持哪些操作系统？

Nginx 支持 Linux、FreeBSD、Solaris、AIX、HP-UX、macOS 和 Windows。

## 配置问题

### 如何检查 Nginx 配置是否正确？

```bash
nginx -t
```

### 配置文件在哪里？

- Debian/Ubuntu: `/etc/nginx/nginx.conf`
- CentOS/RHEL: `/etc/nginx/nginx.conf`
- 编译安装: `/usr/local/nginx/conf/nginx.conf`

### 如何重载配置？

```bash
nginx -s reload
```

### 修改配置后需要重启吗？

不需要重启，使用 `nginx -s reload` 即可重新加载配置。

## 性能问题

### 如何优化 Nginx 性能？

1. **调整 worker 进程数**

```nginx
worker_processes auto;
```

2. **增加 worker 连接数**

```nginx
events {
    worker_connections 4096;
}
```

3. **启用 gzip 压缩**

```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
```

4. **启用浏览器缓存**

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
}
```

5. **使用连接复用**

```nginx
upstream backend {
    server backend1.example.com;
    keepalive 32;  # 连接池大小
}
```

### Nginx 占用内存过高怎么办？

1. 减少 worker_connections 数量
2. 限制请求体大小
3. 优化缓存配置
4. 检查是否有内存泄漏

### 如何处理高并发？

1. 使用负载均衡
2. 启用缓存
3. 调整内核参数
4. 使用 CDN

## 安全问题

### 如何防止 DDoS 攻击？

1. **限制连接数**

```nginx
limit_conn_zone $binary_remote_addr zone=addr:10m;
```

2. **限制请求速率**

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
```

3. **关闭慢速连接**

```nginx
client_body_timeout 5s;
client_header_timeout 5s;
```

### 如何配置 SSL 证书？

```nginx
server {
    listen 443 ssl http2;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/cert.key;
    ssl_protocols TLSv1.2 TLSv1.3;
}
```

### 如何强制使用 HTTPS？

```nginx
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

## 代理问题

### 反向代理失败怎么办？

1. 检查后端服务是否运行
2. 检查防火墙设置
3. 验证 proxy_pass 配置正确
4. 查看错误日志

### 如何代理 WebSocket？

```nginx
location /ws/ {
    proxy_pass http://backend;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### 代理后获取真实 IP？

```nginx
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

## 日志问题

### 日志文件太大怎么办？

使用 logrotate 进行日志轮换：

```bash
# /etc/logrotate.d/nginx
/var/log/nginx/*.log {
    daily
    rotate 14
    compress
    notifempty
}
```

### 如何关闭访问日志？

```nginx
access_log off;
```

### 日志格式怎么配置？

```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent"';
```

## 负载均衡问题

### 负载均衡如何选择后端？

Nacos 支持多种负载均衡算法：

- **轮询（默认）**：按顺序分配请求
- **加权轮询**：按权重分配请求
- **IP 哈希**：同一 IP 的请求分配到同一后端
- **最少连接**：分配给连接数最少的后端

### 后端服务器宕机怎么办？

Nginx 会自动跳过故障节点，将请求转发到健康节点。

## 缓存问题

### 如何配置代理缓存？

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m;

location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 60m;
    proxy_pass http://backend;
}
```

### 缓存不生效怎么办？

1. 检查缓存路径权限
2. 确认响应头允许缓存
3. 验证缓存键配置正确

### 如何清除缓存？

```bash
# 删除缓存目录
rm -rf /var/cache/nginx/*
```

## 端口问题

### 端口被占用怎么办？

```bash
# 查看端口占用
netstat -tulpn | grep :80

# 改用其他端口
listen 8080;
```

```nginx

    listen 80;
    listen 8080;
    server_name example.com;
}
```

## 常见错误代码

### 400 Bad Request

- 请求头过大：增加 `client_header_buffer_size`
- URL 过长：检查请求路径

### 403 Forbidden

- 索引文件不存在
- 权限不足：检查文件和目录权限

### 404 Not Found

- 文件路径错误
- SELinux 或 AppArmor 阻止

### 502 Bad Gateway

- 后端服务未运行
- FastCGI/PHP-FPM 配置错误

### 504 Gateway Timeout

- 后端响应超时
- 增加 `proxy_read_timeout`

## 调试技巧

### 如何查看详细错误信息？

1. 启用调试日志：

```nginx
error_log /var/log/nginx/error.log debug;
```

2. 检查错误日志：

```bash
tail -f /var/log/nginx/error.log
```

### 如何测试配置是否生效？

```bash
# 测试配置
nginx -t

# 显示完整配置
nginx -T
```

### 如何调试正则表达式？

使用 `location` 测试：

```nginx
location /test {
    # 添加日志查看匹配情况
    access_log /var/log/nginx/test.log;
}
```

## 升级问题

### 如何升级 Nginx？

1. 备份配置文件
2. 停止 Nginx
3. 安装新版本
4. 启动 Nginx
5. 测试功能

### 如何平滑升级？

```bash
# 编译新版本
make

# 发送 USR2 信号升级
kill -s USR2 $(cat /var/run/nginx.pid)

# 优雅关闭旧工作进程
kill -s WINCH $(cat /var/run/nginx.pid.oldbin)
```
