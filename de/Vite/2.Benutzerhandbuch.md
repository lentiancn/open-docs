# Vite Benutzerhandbuch

Dieses Handbuch bietet eine umfassende Anleitung zur Verwendung von Vite für Ihre Frontend-Entwicklung.

## Grundlegende Konzepte

### Native ES Modules

Vite nutzt native ES Modules im Browser, ohne während der Entwicklung zu bündeln. Das bedeutet:
- Kein Warten auf Bundling beim Start
- Schnelle HMR-Updates
- On-Demand-Kompilierung

### Dependency Pre-bundling

Vite verwendet esbuild, um Abhängigkeiten vorzubündeln:
- Konvertiert CommonJS zu ESM
- Reduziert die Anzahl der Anfragen
- Beschleunigt den Start

## Dateiverwaltung

### HTML-Einstiegspunkt

Vite verwendet `index.html` als Einstiegspunkt:

```html
<!DOCTYPE html>
<html lang="de">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Meine App</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.js"></script>
  </body>
</html>
```

### JavaScript-Import

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import './style.css'

createApp(App).mount('#app')
```

### CSS-Import

```javascript
import './style.css'
```

oder mit CSS Modules:

```javascript
import styles from './style.module.css'
```

### Statische Assets

Assets im `public` Ordner werden unverändert serviert:

```
public/
└── logo.png
```

Zugriff im Code:

```javascript
const logo = '/logo.png'
```

## Features

###Globale Substitution

Importieren Sie Module mit Aliases:

```javascript
import '@' from '@/components/Header'
```

Konfiguration:

```javascript
export default defineConfig({
  resolve: {
    alias: {
      '@': '/src'
    }
  }
})
```

### Lazy Loading

Route-basiertes Code-Splitting:

```javascript
const Home = () => import('./views/Home.vue')
const About = () => import('./views/About.vue')
```

### JSON-Import

```javascript
import data from './data.json'
```

## Entwicklung

### Hot Module Replacement

Vite bietet sofortige HMR:
- Behält Anwendungszustand bei Updates
- Aktualisiert nur geänderte Module
- Funktioniert mit allen Frameworks

### Fehlerbehandlung

Fehler werden im Browser angezeigt:

```
Error: Cannot find module '@/components/Header'
```

### Entwicklungsserver

```bash
# Standard-Port (5173)
npm run dev

# Bestimmter Port
npm run dev -- --port 3000

# Öffnet nicht automatisch den Browser
npm run dev -- --open false

# Bestimmte Host
npm run dev -- --host 0.0.0.0
```

## Produktion

### Build-Optimierung

Vite optimiert automatisch:
- Tree Shaking
- Code Splitting
- Minifizierung

### Benutzerdefinierte Chunking-Strategie

```javascript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          utils: ['lodash']
        }
      }
    }
  }
})
```

### Asset-Handling

```javascript
export default defineConfig({
  build: {
    assetsInlineLimit: 4096, // bytes
    cssCodeSplit: true,
    sourcemap: false
  }
})
```

## Best Practices

1. **Relative Imports**: Verwenden Sie relative Imports für besseres Caching
2. **Resolve Aliases**: Konfigurieren Sie Aliase für kürzere Importpfade
3. **Lazy Loading**: Verwenden Sie Lazy Loading für Routen
4. **Umgebungsvariablen**: Nutzen Sie Umgebungsvariablen für Konfiguration
5. **Optimierung**: Konfigurieren Sie Build-Optionen für Produktion

## Troubleshooting

### Lange Ladezeiten

- Prüfen Sie Netzwerk-Tab für große Abhängigkeiten
- Optimieren Sie Bilder und Assets
- Aktivieren Sie Lazy Loading

### Modul nicht gefunden

- Überprüfen Sie alle Abhängigkeiten: `npm install`
- Verifizieren Sie Import-Pfade
- Stellen Sie sicher, dass package.json type: "module" hat

### Port bereits belegt

Ändern Sie den Port in vite.config.js:

```javascript
server: {
  port: 3001
}
```
