# Manual del usuario

Este manual proporciona instrucciones detalladas sobre el uso de MultiTalk, incluyendo operaciones basicas, configuracion de parametros y tecnicas avanzadas.

## Uso basico

### Preparacion de archivos de entrada

MultiTalk requiere preparar formatos de entrada especificos. El proyecto usa archivos JSON para definir parametros de generacion, incluyendo rutas de audio, rutas de imagenes y prompts de texto.

Para la generacion de una sola persona, el formato JSON de entrada requiere especificar la ruta del archivo de audio, la ruta de la imagen de referencia, el nombre de guardado del video y otros parametros. Los archivos de ejemplo se encuentran en el directorio examples, como single_example_1.json.

Para la generacion conversacional multipersonal, el JSON de entrada necesita contener multiples flujos de audio y definiciones de region de persona correspondientes. Los archivos de ejemplo se encuentran en el directorio examples, como multitalk_example_2.json.

### Ejecucion de generacion basica

Despues de completar la instalacion y configuracion, ingrese al directorio del proyecto y ejecute el siguiente comando para la generacion de video:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file single_exp
```

### Descripcion de parametros

--ckpt_dir especifica la ruta del directorio de pesos del modelo. --wav2vec_dir especifica la ruta del directorio del codificador de audio. --input_json especifica la ruta del archivo JSON de parametros de entrada. --sample_steps especifica el numero de pasos de muestreo; valores mas altos producen mejor calidad pero velocidad mas lenta. --mode especifica el modo de generacion, streaming para videos largos, clip para videos cortos. --use_teacache habilita la aceleracion TeaCache. --save_file especifica el nombre de guardado del archivo de salida.

## Tecnicas de uso avanzado

### Ajuste de calidad de generacion

Los pasos de muestreo son parametros clave que afectan la calidad de generacion. Los 40 pasos predeterminados proporcionan buenos resultados. Para una generacion mas rapida, los pasos se pueden reducir a 10, lo cual tiene un impacto minimo en la sincronizacion de labios pero afecta la fluidez del movimiento y la calidad visual. Aumentar los pasos mejora la representacion de detalles.

La fuerza de sincronizacion de labios se controla mediante el coeficiente de guia de audio. Los valores de Audio CFG entre 3 y 5 funcionan mejor. Aumentar este valor logra mejor sincronizacion de labios.

### Generacion de videos largos

MultiTalk soporta la generacion de videos de hasta 15 segundos. Use el modo streaming para generar clips de video mas largos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --save_file single_long_exp
```

Al generar videos muy largos, puede haber diferencias de tono de color entre segmentos. Establecer Audio CFG en 3 ayuda a mantener la consistencia del color.

### Operacion con VRAM bajo

Si el VRAM es limitado, puede usar los siguientes parametros para reducir el uso de memoria:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --num_persistent_param_in_dit 0 \
 --use_teacache \
 --save_file lowvram_exp
```

Agregar --num_persistent_param_in_dit 0 permite la generacion de video 480P en una sola RTX 4090.

### Habilitacion de aceleracion

El proyecto soporta multiples metodos de aceleracion para mejorar la velocidad de generacion.

La aceleracion TeaCache puede mejorar significativamente la velocidad mientras mantiene la calidad. Los valores recomendados de --teacache_thresh estan entre 0.2 y 0.5:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --use_teacache \
 --teacache_thresh 0.3 \
 --save_file teacache_exp
```

### Uso de aceleracion LoRA

Usar LoRA puede reducir significativamente los pasos de muestreo. FusioniX LoRA requiere solo 8 pasos, mientras que lightx2v LoRA requiere solo 4 pasos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_1.json \
 --lora_dir weights/Wan2.1_I2V_14B_FusionX_LoRA.safetensors \
 --lora_scale 1.0 \
 --sample_text_guide_scale 1.0 \
 --sample_audio_guide_scale 2.0 \
 --sample_steps 8 \
 --mode streaming \
 --save_file fusionx_exp
```

## Uso de la interfaz Gradio

El proyecto proporciona una interfaz web Gradio para uso interactivo. Ingrese al directorio del proyecto y ejecute:

```bash
python app.py
```

O use la version de bajo VRAM:

```bash
python app.py --num_persistent_param_in_dit 0
```

## Integracion de sintesis de voz TTS

MultiTalk soporta entrada de texto directa. A traves de tecnologia TTS integrada, genera voz a partir de texto y luego crea videos:

```bash
python generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --input_json examples/single_example_tts_1.json \
 --sample_steps 40 \
 --mode streaming \
 --audio_mode tts \
 --save_file tts_exp
```

## Configuracion de multiples GPUs

La generacion de videos de alta resolucion 720P requiere multiples GPUs. Use el siguiente comando para configurar multiples GPUs:

```bash
GPU_NUM=8
torchrun --nproc_per_node=$GPU_NUM --standalone generate_multitalk.py \
 --ckpt_dir weights/Wan2.1-I2V-14B-480P \
 --wav2vec_dir weights/chinese-wav2vec2-base \
 --dit_fsdp --t5_fsdp \
 --ulysses_size=$GPU_NUM \
 --input_json examples/single_example_1.json \
 --sample_steps 40 \
 --mode streaming \
 --use_teacache \
 --save_file multi_gpu_exp
```

## Uso de ComfyUI

El proyecto tambien soporta integracion en flujos de trabajo de ComfyUI. Necesita instalar la extension ComfyUI-WanVideoWrapper, luego cargar el modelo MultiTalk para la orquestacion del flujo de trabajo.
