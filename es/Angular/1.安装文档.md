# Guía de Instalación de Angular

Angular es un marco de trabajo moderno para aplicaciones web mantenido por Google. Esta guía cubre los métodos de instalación en todos los entornos.

## Información de Versiones

| Versión | Estado | Descripción |
|---------|--------|-------------|
| Angular 17+ | ✅ Actual | Última versión con todas las funciones nuevas |
| Angular 16 | ✅ Soportado | Versión estable |
| Angular 14-15 | ⚠️ Mantenimiento | Actualizaciones de seguridad |
| Angular 2-13 | ❌ Legacy | No recomendado para nuevos proyectos |

> **Importante**: Esta guía está dirigida principalmente a **Angular 17+**. Se recomienda usar la última versión para nuevos proyectos para obtener el mejor rendimiento y las últimas funciones.

## Métodos de Instalación

### Método 1: Usar Angular CLI (Recomendado)

Angular CLI es la forma oficial recomendada para crear y administrar proyectos Angular.

```bash
# Instalar Angular CLI
npm install -g @angular/cli

# Crear un nuevo proyecto
ng new my-app

# Entrar al directorio del proyecto
cd my-app

# Iniciar el servidor de desarrollo
ng serve
```

#### Opciones de Creación de Proyectos

```bash
# Crear proyecto con enrutamiento
ng new my-app --routing

# Crear proyecto con SCSS
ng new my-app --style=scss

# Crear proyecto con SSR
ng new my-app --ssr

# Omitir instalación de dependencias
ng new my-app --skip-install
```

### Método 2: Crear Proyecto Manualmente

```bash
# Inicializar proyecto npm
npm init -y

# Instalar dependencias principales de Angular
npm install @angular/core@17 @angular/common@17 @angular/compiler@17

# Instalar plataformas de Angular
npm install @angular/platform-browser@17 @angular/platform-browser-dynamic@17

# Instalar compilador de Angular
npm install @angular/compiler-cli@17 typescript@~5.2.0 zone.js@~0.14.0
```

### Método 3: Usar Vite

```bash
# Crear proyecto Angular con Vite
npm create vite@latest my-angular-app -- --template angular

# Entrar al directorio del proyecto
cd my-angular-app

# Instalar dependencias
npm install
```

### Método 4: Usar Nx Workspace

Nx es una poderosa herramienta de monorepo adecuada para proyectos empresariales grandes.

```bash
# Instalar Nx
npm install -g nx

# Crear aplicación Angular
npx create-nx-workspace my-org --preset=angular

# Añadir aplicación Angular
nx g @nx/angular:app my-app
```

## Comandos de Desarrollo

### Comandos Comunes

```bash
# Iniciar servidor de desarrollo
ng serve

# Construir para producción
ng build

# Ejecutar pruebas
ng test

# Ejecutar pruebas end-to-end
ng e2e

# Generar componente
ng g c my-component

# Generar servicio
ng g s my-service

# Generar módulo
ng g m my-module

# Generar enrutamiento
ng g m my-module --routing
```

### Opciones del Servidor de Desarrollo

```bash
# Especificar puerto
ng serve --port 4200

# Abrir navegador
ng serve --open

# Reemplazo de módulo en caliente
ng serve --hmr

# Construcción para producción
ng serve --configuration production
```

## Gestión de Dependencias

### Instalar Dependencias

```bash
# Instalar dependencias de tiempo de ejecución
npm install @angular/animations @angular/common @angular/compiler @angular/core @angular/forms @angular/platform-browser @angular/platform-browser-dynamic @angular/router

# Instalar dependencias de desarrollo
npm install --save-dev @angular-devkit/build-angular @angular/cli @angular/compiler-cli @types/jasmine @types/node typescript jasmine-core jasmine-spec-reporter karma karma-chrome-launcher karma-coverage karma-jasmine karma-jasmine-html-reporter zone.js
```

### Usar yarn

```bash
# Instalar con yarn
yarn install

# Añadir dependencia
yarn add @angular/core@17

# Añadir dependencia de desarrollo
yarn add --dev @angular/cli@17
```

## Preguntas Frecuentes

### ¿Los comandos de Angular CLI no funcionan?

Asegúrate de tener Angular CLI instalado globalmente:

```bash
npm install -g @angular/cli
```

### ¿La versión de Node.js no coincide?

Usa nvm para gestionar las versiones de Node.js:

```bash
# Instalar nvm (Windows)
winget install CoreyButler.NVMforWindows

# Usar una versión específica
nvm install 18
nvm use 18
```

### ¿La construcción falla?

Limpia la caché y reinstala:

```bash
rm -rf node_modules dist
npm install
```
