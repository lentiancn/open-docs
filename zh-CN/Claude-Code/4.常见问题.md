# Claude Code 常见问题

本页面列出 Claude Code 安装和使用过程中最常见的问题及解决方案。

## 安装问题

### 命令找不到

安装完成后遇到 `command not found: claude` 或 `'claude' is not recognized` 错误，说明安装目录不在系统 PATH 中。

**解决方案：** 检查并配置 PATH 环境变量。

macOS/Linux 添加到 .zshrc 或 .bashrc：
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

Windows PowerShell：
```powershell
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
```

然后重启终端并验证：
```bash
claude --version
```

### 安装脚本返回 HTML

运行安装命令时看到以下错误：
- `bash: line 1: syntax error near unexpected token '<'`
- `bash: line 1: '<!DOCTYPE html>'`
- PowerShell 上显示：`Invoke-Expression: Missing argument in parameter list`

这表示安装 URL 返回了 HTML 页面而不是安装脚本。

**解决方案：**

1. 使用替代安装方法：
   - macOS/Linux：使用 Homebrew
     ```bash
     brew install --cask claude-code
     ```
   - Windows：使用 WinGet
     ```powershell
     winget install Anthropic.ClaudeCode
     ```

2. 如果 HTML 页面显示 "App unavailable in region"，表示 Claude Code 在你所在的国家/地区不可用。请查看 [支持的国家和地区列表](https://www.anthropic.com/supported-countries)。

3. 等待几分钟后重试，问题通常是暂时的。

### curl 下载失败

错误信息：`curl: (56) Failure writing output to destination`

这表示连接在脚本下载完成前中断了。

**解决方案：**

1. 检查网络稳定性：
   ```bash
   curl -fsSL https://storage.googleapis.com -o /dev/null
   ```
   如果命令成功完成，说明连接正常，重试安装命令即可。

2. 使用替代安装方法（见上文）。

### TLS/SSL 连接错误

错误信息：`curl: (35) TLS connect error`、`schannel: next InitializeSecurityContext failed` 或 `Could not establish trust relationship for the SSL/TLS secure channel`

**解决方案：**

1. 更新系统 CA 证书：
   - Ubuntu/Debian：
     ```bash
     sudo apt-get update && sudo apt-get install ca-certificates
     ```
   - macOS（通过 Homebrew）：
     ```bash
     brew install ca-certificates
     ```

2. 在 Windows PowerShell 中启用 TLS 1.2：
   ```powershell
   [Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
   irm https://claude.ai/install.ps1 | iex
   ```

3. 检查代理或防火墙干扰，配置 `NODE_EXTRA_CA_CERTS` 环境变量。

### 无法获取版本信息

错误信息：`Failed to fetch version from storage.googleapis.com`

这表示 `storage.googleapis.com` 在你的网络被阻止。

**解决方案：**

1. 测试直接连接：
   ```bash
   curl -sI https://storage.googleapis.com
   ```

2. 如果在代理后面，设置 `HTTPS_PROXY` 环境变量。

### Windows 需要 Git Bash

错误信息：`Claude Code on Windows requires git-bash`

**解决方案：** 安装 [Git for Windows](https://git-scm.com/downloads/win)，然后重新运行安装命令。

如果已安装 Git 但仍然报错，在设置文件中配置 Git Bash 路径：
```json
{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}
```

### Linux 上非法指令

错误信息：`Illegal instruction`

这通常是架构不匹配导致的。请确认你的 CPU 架构与安装的二进制兼容。

### macOS 上二进制无法加载

错误信息：`dyld: cannot load` 或 `Abort trap`

这表示二进制与你的系统不兼容。请确认你下载了正确版本的 Claude Code（Intel 或 Apple Silicon）。

### 低内存服务器安装被杀

错误信息：在 Linux 服务器上安装时显示 `Killed`

**解决方案：** 为低内存服务器添加 swap 空间。

## 认证问题

### OAuth 错误或 403 Forbidden

**解决方案：**

1. 检查你的账户是否有 Claude Code 访问权限（Pro、Max、Teams、Enterprise 或 Console）
2. 清除缓存的凭据并重新登录：
   ```bash
   rm -rf ~/.claude
   claude
   ```
3. 检查你的地区是否在 [支持列表](https://www.anthropic.com/supported-countries) 中

### 区域不可用

错误信息：`App unavailable in region`

Claude Code 在你的国家/地区不可用。请查看 [Anthropic 支持的国家和地区](https://www.anthropic.com/supported-countries)。

## 网络配置

### 检查网络连接

安装程序从 `storage.googleapis.com` 下载文件。验证你能否访问：

```bash
curl -sI https://storage.googleapis.com
```

如果失败，可能是以下原因：
- 公司防火墙或代理阻止了 Google Cloud Storage
- 地区网络限制：尝试使用 VPN 或其他网络
- TLS/SSL 问题：更新系统 CA 证书，或检查是否配置了 `HTTPS_PROXY`

### 代理配置

如果在公司代理后面，在安装前设置代理环境变量：

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

## 检查冲突安装

多个 Claude Code 安装可能导致版本不匹配或意外行为。

检查已安装的内容：

**macOS/Linux：**
```bash
which -a claude
ls -la ~/.local/bin/claude
npm -g ls @anthropic-ai/claude-code 2>/dev/null
```

**Windows：**
```powershell
where.exe claude
Test-Path "$env:LOCALAPPDATA\Claude Code\claude.exe"
```

如果发现多个安装，只保留一个（推荐原生安装 `~/.local/bin/claude`）。

移除额外的安装：
- npm 全局安装：`npm uninstall -g @anthropic-ai/claude-code`
- Homebrew 安装：`brew uninstall --cask claude-code`

## 检查目录权限

安装程序需要写入 `~/.local/bin/` 和 `~/.claude/` 的权限。检查这些目录是否可写：

```bash
test -w ~/.local/bin && echo "writable" || echo "not writable"
test -w ~/.claude && echo "writable" || echo "not writable"
```

如果目录不可写：

```bash
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```

## 验证二进制文件

如果 `claude` 已安装但启动时崩溃或挂起：

确认二进制存在且可执行：
```bash
ls -la $(which claude)
```

在 Linux 上检查缺失的共享库：
```bash
ldd $(which claude) | grep "not found"
```

运行基本检查：
```bash
claude --version
```

## 诊断工具

使用 `claude doctor` 获取更详细的安装和配置信息：

```bash
claude doctor
```
