# Foire aux questions

## Questions de base

### Qu'est-ce que Jenkins ?

Jenkins est un serveur d'automatisation open source utilisé principalement pour l'Intégration Continue (CI) et la Livraison Continue (CD). Il aide les équipes de développement à automatiser le processus de construction, de test et de déploiement de logiciels, améliorant l'efficacité et la qualité du développement.

### Jenkins est-il gratuit ?

Oui, Jenkins est complètement gratuit et open source. Il utilise la licence MIT, permettant à quiconque de l'utiliser, le modifier et le distribuer librement.

### Quel environnement d'exécution Jenkins nécessite-t-il ?

Jenkins nécessite Java 8 ou supérieur. Il peut fonctionner sur plusieurs systèmes d'exploitation, notamment Windows, Linux et macOS. Un minimum de 1 Go de mémoire et 10 Go d'espace disque est recommandé.

### Quelle est la différence entre Jenkins et Hudson ?

Jenkins est un fork de Hudson. En 2011, après un conflit de marque avec Oracle, l'équipe de développement principale de Hudson a forké Hudson et créé Jenkins. Jenkins est maintenant le projet actif tandis que Hudson a été abandonné.

## Questions d'installation

### Impossible d'accéder après le premier démarrage. Que faire ?

1. Vérifier si Jenkins a démarré correctement en consultant les journaux
2. Confirmer que le port n'est pas utilisé (port par défaut 8080)
3. Vérifier les paramètres du pare-feu pour assurer l'accès au port
4. Consulter les messages d'erreur dans les journaux de démarrage

### Où est le mot de passe de déverrouillage ?

Lors de la première installation, le mot de passe de déverrouillage s'affiche dans les journaux de la console Jenkins. Si vous utilisez Docker, utilisez `docker logs jenkins-blueocean` pour le voir. Le fichier de mot de passe se trouve généralement dans `/var/jenkins_home/secrets/initialAdminPassword`.

### L'installation du plugin a échoué. Que faire ?

1. Vérifier la connexion réseau : Jenkins doit accéder au centre de mise à jour
2. Installer manuellement : Essayer de télécharger le fichier hpi du plugin depuis l'onglet Avancé
3. Utiliser le miroir Jenkins : Configurer une source de mise à jour nationale
4. Consulter les journaux : Comprendre la raison spécifique de l'échec

### Comment modifier le port de Jenkins ?

- Méthode Docker : Modifier le paramètre -p
- Méthode WAR : Utiliser `--httpPort`
- Package Linux : Modifier HTTP_PORT dans /etc/default/jenkins

## Questions sur les identifiants

### Les identifiants ne fonctionnent pas. Que faire ?

1. Confirmer que l'ID d'identifiant est correct
2. Vérifier que le type d'identifiant correspond
3. Valider que les identifiants n'ont pas expiré
4. Vérifier les permissions utilisateur

### Comment stocker les mots de passe en toute sécurité ?

Utiliser la fonction de stockage d'identifiants de Jenkins :

1. Aller dans Manage Credentials
2. Ajouter des identifiants, choisir le type
3. Jenkins chiffre le stockage, ne s'affiche pas en texte brut

### L'authentification par clé SSH a échoué. Que faire ?

1. Confirmer que la clé publique a été ajoutée au serveur cible
2. Vérifier les permissions de clé : le fichier de clé privée doit être 600
3. Valider que le format de clé est correct
4. Essayer de régénérer la paire de clés

## Questions sur le pipeline

### Comment résoudre les erreurs de syntaxe Jenkinsfile ?

1. Utiliser le validateur de pipeline déclaratif
2. Vérifier les messages d'erreur de la console Jenkins
3. Utiliser un simulateur Jenkins localement
4. Consulter la documentation officielle de syntaxe

### L'exécution du pipeline a échoué. Que faire ?

1. Consulter les journaux de build pour des informations d'erreur détaillées
2. Vérifier le statut de retour de chaque étape
3. Confirmer que les outils requis sont disponibles
4. Valider les variables d'environnement et les identifiants

### Comment implémenter l'exécution parallèle ?

Le pipeline déclaratif supporte la syntaxe parallèle :

```groovy
pipeline {
    agent any
    stages {
        stage('Build parallèle') {
            parallel {
                stage('Module A') {
                    steps {
                        sh './build-a.sh'
                    }
                }
                stage('Module B') {
                    steps {
                        sh './build-b.sh'
                    }
                }
            }
        }
    }
}
```

### Comment attendre une confirmation manuelle ?

Utiliser le pas input :

```groovy
stage('Déployer en production') {
    steps {
        input message: 'Confirmer le déploiement en production ?', ok: 'Déployer'
    }
}
```

## Questions sur le build

### Le build est très lent. Que faire ?

1. Utiliser les builds incrémentaux
2. Configurer le cache de build
3. Utiliser les étapes parallèles
4. Ajouter plus de nœuds de build
5. Optimiser les téléchargements de dépendances

### Les téléchargements de dépendances Maven sont lents. Que faire ?

1. Configurer un miroir Maven
2. Utiliser un gestionnaire de dépôt privé
3. Configurer et persister le chemin du dépôt local

### Comment configurer le timeout du build ?

```groovy
pipeline {
    agent any
    stages {
        stage('Test timeout') {
            options {
                timeout(time: 30, unit: 'MINUTES')
            }
            steps {
                // étapes de build
            }
        }
    }
}
```

### Comment conserver les artifacts de build ?

Utiliser le pas archive :

```groovy
post {
    always {
        archiveArtifacts artifacts: 'target/*.jar', fingerprint: true
    }
}
```

## Questions sur les builds distribués

### La connexion au nœud a échoué. Que faire ?

1. Vérifier la connectivité réseau
2. Confirmer la compatibilité de la version Java
3. Valider la configuration de la clé du nœud
4. Consulter les journaux du nœud

### Comment configurer les nœuds Docker ?

1. Installer le plugin Docker
2. Configurer le cloud Docker
3. Spécifier l'agent docker dans le pipeline

### Comment configurer l'équilibrage de charge ?

Configurer la stratégie de sélection d'agent dans le pipeline :

```groovy
pipeline {
    agent none
    stages {
        stage('Build') {
            agent { label 'docker' }
            steps {
                // build
            }
        }
    }
}
```

## Sauvegarde et restauration

### Comment sauvegarder Jenkins ?

Il faut sauvegarder :

1. Répertoire JENKINS_HOME (contient toutes les configurations et enregistrements de build)
2. Répertoire des plugins
3. Fichiers de configuration d'environnement

L'utilisation du plugin Configuration as Code est recommandée pour gérer les configurations.

### Comment migrer Jenkins ?

1. Arrêter Jenkins source
2. Copier JENKINS_HOME vers le nouveau serveur
3. Installer la même version de Jenkins
4. Assurer la compatibilité de la version Java
5. Démarrer le nouveau Jenkins

## Questions de sécurité

### Comment activer l'enregistrement des utilisateurs ?

1. Aller dans Configure Global Security
2. Activer le domaine de sécurité (base de données utilisateur propre à Jenkins)
3. Configurer la stratégie d'autorisation (les utilisateurs connectés peuvent tout faire)

### Comment restreindre l'accès aux projets ?

Utiliser la stratégie d'autorisation matricielle :

1. Configurer la sécurité globale
2. Choisir Matrix Authorization Strategy
3. Configurer différentes autorisations pour différents utilisateurs

### Comment gérer les vulnérabilités de sécurité ?

1. Mettre à jour la version du cœur Jenkins en temps opportun
2. Mettre à jour les plugins régulièrement
3. Faire attention aux avis de sécurité Jenkins
4. Limiter l'accès anonyme

## Questions de performance

### Que faire en cas de mémoire insuffisante ?

1. Ajuster les paramètres JVM : ajouter `-Xmx` dans le script de démarrage
2. Réduire le nombre de builds simultanés
3. Nettoyer régulièrement les anciens enregistrements de build
4. Augmenter la mémoire système

### Les enregistrements de build occupent beaucoup d'espace ?

1. Configurer la politique de rétention des builds
2. Nettoyer régulièrement les anciens builds
3. Utiliser l'archivage au lieu de conserver les enregistrements de build complets

## Questions d'intégration

### L'intégration GitHub a échoué ?

1. Vérifier les permissions de l'API GitHub
2. Valider la configuration du webhook
3. Confirmer que les identifiants sont corrects

### L'intégration Docker a échoué ?

1. Vérifier la configuration de l'API Docker
2. Confirmer les permissions du socket Docker
3. Valider la compatibilité de la version Docker

### L'intégration Kubernetes a échoué ?

1. Vérifier la configuration kubeconfig
2. Valider les permissions de l'API Kubernetes
3. Confirmer que le conteneur Jenkins a suffisamment de permissions

## Questions de mise à niveau

### Comment mettre à niveau Jenkins ?

- Méthode Docker : Mettre à jour la version de l'image
- Méthode WAR : Remplacer le fichier WAR
- Méthode package : Utiliser le gestionnaire de packages

### Plugins incompatibles après la mise à niveau ?

1. Consulter la liste de compatibilité
2. Mettre à jour les plugins vers des versions compatibles
3. Revenir à la version précédente de Jenkins si nécessaire
