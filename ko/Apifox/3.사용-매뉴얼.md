# 사용 매뉴얼

이 매뉴얼은 Apifox의 핵심 기능과 사용법에 대한 상세한 가이드입니다. 프로젝트 관리, 인터페이스 설계, 디버깅, 모의(mock) 데이터 생성, 자동화 테스트 등을 포함합니다.

## 프로젝트 관리

### 프로젝트 생성

Apifox에서 프로젝트는 API를 구성하는 최상위 단위입니다.

```bash
# 새 프로젝트 생성 단계
1. 왼쪽의 "프로젝트" 목록 옆에 있는 "+"를 클릭합니다
2. 프로젝트 이름을 입력합니다
3. 프로젝트 유형을 선택합니다
4. 기본 템플릿을 선택합니다 (선택사항)
5. "생성"을 클릭합니다
```

### 프로젝트 설정

각 프로젝트는 자체적인 독립적인 설정을 가집니다:

- **기본 정보**: 프로젝트 이름, 설명, 아이콘
- **환경 변수**: 개발, 테스트, 프로덕션 등 다양한 환경에 대한 변수 정의
- **인증**: 전역 인증 방법 설정
- **내보내기/가져오기**: 프로젝트 데이터 가져오기 및 내보내기

### 환경 관리

Apifox는 다중 환경 구성을 지원합니다:

```bash
# 환경 변수 사용
요청에서 {{variableName}}으로 환경 변수 참조
예시: {{baseUrl}}/api/users

환경 전환: 상단의 환경 선택기를 클릭하여 다른 환경 선택
```

## 인터페이스 설계

### 인터페이스 생성

인터페이스는 특정 API 정의입니다:

```bash
# 인터페이스 생성 단계
1. 프로젝트에서 "+"를 클릭하고 "새 인터페이스"를 선택합니다
2. 기본 정보를 입력합니다:
   - 인터페이스 이름: 사용자 목록
   - 요청 경로: /api/users
   - 요청 메서드: GET
3. 요청 매개변수를 정의합니다
4. 응답 구조를 정의합니다
5. 저장합니다
```

### 요청 매개변수

이러한 유형의 요청 매개변수가 지원됩니다:

- **경로 매개변수**: /api/users/:id의 id와 같은
- **쿼리 매개변수**: ?name=test와 같은 URL의
- **요청 헤더**: Authorization과 같은
- **요청 본문**: JSON, form, XML 및 기타 형식

### 응답 정의

각 인터페이스에 대한 성공 및 실패 응답을 정의합니다:

```json
{
  "성공 응답 200": {
    "id": 1,
    "name": "김철수",
    "email": "chulsoo@example.com"
  },
  "오류 응답 404": {
    "code": 404,
    "message": "사용자를 찾을 수 없습니다"
  }
}
```

### 인터페이스 목록 관리

그룹 관리가 지원됩니다 — 모듈이나 기능별로 인터페이스를 구성할 수 있습니다:

- 드래그 앤 드롭으로 순서 변경
- 일괄 작업 (일괄 삭제, 일괄 내보내기 등)
- 검색 및 필터링

## 인터페이스 디버깅

### 요청 보내기

인터페이스 상세 페이지에서 "전송"을 클릭하여 요청을 보냅니다:

```bash
# 기본 디버깅 단계
1. 인터페이스를 엽니다
2. 환경을 선택합니다
3. 매개변수 값을 설정합니다
4. "전송"을 클릭합니다
5. 응답을 확인합니다
```

### 인증 설정

다양한 인증 방법이 지원됩니다:

- **없음**: 인증 없음
- **Bearer Token**: JWT 인증
- **Basic Auth**: 기본 인증
- **OAuth 2.0**: OAuth2 인증
- **사용자 정의 인증**: 필요에 따라 사용자 정의 인증 헤더

### 사전 요청 스크립트

요청을 보내기 전에 실행되는 스크립트를 설정할 수 있습니다:

```javascript
// 예시: 타임스탬프 추가
const timestamp = Date.now();
pm.request.headers.append('X-Timestamp', timestamp);

// 예시: 랜덤 ID 생성
const uuid = 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, function(c) {
    var r = Math.random() * 16 | 0, v = c == 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
});
pm.request.headers.append('X-Request-Id', uuid);
```

### 사후 응답 스크립트

응답을 받은 후 실행되는 스크립트를 설정할 수 있습니다:

```javascript
// 예시: 응답 데이터 추출
const response = pm.response.json();
console.log('응답 데이터:', response);

// 예시: 환경 변수 설정
pm.environment.set('access_token', response.data.token);
```

## API 모의(mock) 데이터 생성

### 스마트 Mock

Apifox의 가장 강력한 기능 중 하나입니다 — 데이터 구조만 정의하면 시스템이 자동으로 모의 데이터를 생성합니다:

```bash
# 스마트 Mock 활성화
1. 인터페이스를 엽니다
2. "Mock" 탭을 클릭합니다
3. "스마트 Mock"을 활성화합니다
4. 저장합니다
```

### Mock 규칙

사용자 정의 Mock 규칙이 지원됩니다:

- `@string`: 랜덤 문자열 생성
- `@integer`: 랜덤 정수 생성
- `@float`: 랜덤 부동소수점 생성
- `@boolean`: 랜덤 부울 생성
- `@date`: 랜덤 날짜 생성
- `@email`: 랜덤 이메일 생성
- `@url`: 랜덤 URL 생성
- `@guid`: UUID 생성

### 예시

```json
{
  "id": "@integer",
  "name": "@string",
  "email": "@email",
  "age": "@integer(18, 60)",
  "createdAt": "@date"
}
```

생성된 모의 데이터:

```json
{
  "id": 12345,
  "name": "kF9s2m",
  "email": "user@example.com",
  "age": 35,
  "createdAt": "2024-03-15"
}
```

### Mock 서버

Apifox는 내장 Mock 서버를 제공합니다:

```bash
# Mock URL 가져오기
인터페이스 상세 페이지에서 Mock URL을 복사합니다
예시: https://mock.apifox.com/api/users
```

## 자동화 테스트

### 테스트 케이스 생성

```bash
# 테스트 생성 단계
1. 인터페이스를 엽니다
2. "테스트" 탭을 클릭합니다
3. "테스트 생성"을 클릭합니다
4. 테스트 케이스 이름을 설정합니다
5. 어설션을 추가합니다
6. 저장합니다
```

### 어설션 유형

다양한 어설션이 지원됩니다:

- **상태 코드 어설션**: 응답 상태 코드 확인
- **JSON 어설션**: 응답 JSON 데이터 확인
- **응답 시간 어설션**: 응답 시간 확인
- **응답 헤더 어설션**: 응답 헤더 확인

### 예시

```javascript
// 상태 코드 어설션
pm.test("상태 코드가 200임", function() {
    pm.response.to.have.status(200);
});

// JSON 어설션
pm.test("사용자 목록 반환", function() {
    const jsonData = pm.response.json();
    pm.expect(jsonData.code).to.eql(0);
    pm.expect(jsonData.data).to.be.an('array');
});

// 응답 시간 어설션
pm.test("응답 시간이 500ms 미만임", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});
```

### 테스트 스위트

여러 인터페이스의 테스트 케이스를 테스트 스위트로 구성할 수 있습니다:

1. 테스트 스위트 생성
2. 테스트 케이스 추가
3. 실행 순서 설정
4. 스위트 실행

### 테스트 보고서 생성

테스트 실행 후 상세 테스트 보고서를 생성할 수 있습니다:

- 통과율 통계
- 실패 케이스 분석
- 응답 시간 통계

## 팀 협업

### 팀원 초대

```bash
# 팀원 초대
1. 프로젝트 설정으로 이동합니다
2. "멤버 관리"를 클릭합니다
3. "팀원 초대"를 클릭합니다
4. 팀원의 이메일을 입력합니다
5. 역할 권한을 설정합니다
6. 초대를 보냅니다
```

### 역할 및 권한

- **관리자**: 전체 프로젝트 권한
- **편집자**: 인터페이스 편집 및 테스트 실행 가능
- **뷰어**: 보기만 가능, 편집 불가

### 실시간 동기화

팀원이 로그인하면 프로젝트 데이터가 자동으로 실시간으로 동기화됩니다:

- 인터페이스 변경 사항이 자동으로 동기화
- 테스트 결과가 자동으로 동기화
- 팀원이 항상 최신 콘텐츠를 확인

### 댓글

인터페이스에 댓글을 추가할 수 있습니다:

1. 인터페이스를 엽니다
2. "댓글"을 클릭합니다
3. 댓글 내용을 입력합니다
4. 팀원을 @멘션합니다
