# Часто задаваемые вопросы

В этом разделе собраны ответы на наиболее распространённые вопросы о MultiTalk.

## Общие вопросы

### Что такое MultiTalk?

MultiTalk — это фреймворк для генерации видео с многопользовательскими разговорами, управляемый аудио. Он позволяет создавать реалистичные видео с одним или несколькими персонажами на основе аудиофайлов, референсных изображений и текстовых подсказок.

### Какие основные функции MultiTalk?

- Генерация реалистичных разговорных видео с одним или несколькими людьми
- Поддержка многопотокового аудиоввода
- Высокая точность синхронизации губ
- Генерация видео с разрешением 480P и 720P
- Поддержка видеоклипов длиной до 15 секунд
- Интерактивное управление персонажами через текстовые подсказки

### Где я могу найти официальные ресурсы MultiTalk?

- Официальный сайт: https://meigen-ai.github.io/multi-talk/
- GitHub: https://github.com/MeiGen-AI/MultiTalk
- Hugging Face: https://huggingface.co/MeiGen-AI/MeiGen-MultiTalk
- Технический отчёт: https://arxiv.org/abs/2505.22647

## Требования и установка

### Какие системные требования для запуска MultiTalk?

Минимальные требования:
- GPU с 8 ГБ видеопамяти (например, RTX 3060)
- Python 3.10
- CUDA 12.1 или выше

Рекомендуемые требования:
- GPU с 16-24 ГБ видеопамяти (например, RTX 4090, A100)

### Можно ли запустить MultiTalk на системе с низким объёмом видеопамяти?

Да, MultiTalk поддерживает запуск на GPU с низким объёмом памяти. Используйте параметр `--num_persistent_param_in_dit 0` для снижения потребления памяти. Это позволяет генерировать 480P видео на одной RTX 4090.

### Какие операционные системы поддерживаются?

MultiTalk разработан для Linux. Для Windows и macOS рекомендуется использовать Docker или WSL2.

## Использование

### Как подготовить входные данные?

Подготовьте JSON файл с следующей структурой:
- Аудиофайл (или несколько аудиофайлов для многопользовательского режима)
- Референсное изображение
- Текстовая подсказка, описывающая сцену

### Какой формат аудио поддерживается?

MultiTalk поддерживает различные аудиоформаты. Для многопользовательской генерации необходимо подготовить отдельные аудиодорожки для каждого персонажа.

### Какой формат изображения нужен для референса?

Рекомендуется использовать изображения высокого качества с хорошим освещением. Изображение должно содержать того персонажа (или персонажей), которые будут анимированы.

### Какой максимальный размер видео можно сгенерировать?

Максимальная длина видеоклипа составляет 15 секунд. При этом модель оптимально работает с клипами длиной около 3-4 секунд (81 кадр при 25 FPS).

## Проблемы и решения

### Видео имеет плохую синхронизацию губ

Попробуйте увеличить значение параметра `--sample_audio_guide_scale`. Рекомендуемое значение находится в диапазоне 3-5.

### Генерация занимает слишком много времени

- Используйте TeaCache ускорение: `--use_teacache`
- Уменьшите количество шагов семплирования (до 10-15)
- Используйте LoRA (FusioniX или lightx2v) для ускорения до 4-8 шагов
- Включите квантование: `--quant int8`

### Недостаточно видеопамяти

- Используйте параметр `--num_persistent_param_in_dit 0`
- Включите квантование INT8
- Уменьшите размер батча
- Используйте более низкое разрешение (480P вместо 720P)

### Цвет тона несовместим между сегментами видео

Установите Audio CFG значение равным 3 для устранения вариаций тона при генерации длинных видео.

### Низкое качество сговора персонажей

- Увеличьте количество шагов семплирования
- Используйте более детальные текстовые подсказки
- Убедитесь, что референсные изображения высокого качества

## Расширенные функции

### Поддерживается ли TTS (Text-to-Speech)?

Да, MultiTalk поддерживает интеграцию с TTS. Используйте параметр `--audio_mode tts` при генерации.

### Можно ли использовать LoRA для ускорения?

Да, поддерживаются LoRA модели FusioniX и lightx2v. Это позволяет сократить количество шагов семплирования до 4-8.

### Поддерживается ли многопользовательская генерация на нескольких GPU?

Да, используйте `torchrun` с параметром `--nproc_per_node` для указания количества GPU.

### Есть ли веб-интерфейс?

Да, MultiTalk поставляется с Gradio демо. Запустите `python app.py` для запуска веб-интерфейса.

## Лицензия и использование

### Какую лицензию использует MultiTalk?

MultiTalk распространяется под лицензией Apache 2.0, что разрешает академическое и коммерческое использование.

### Есть ли ограничения на коммерческое использование?

Согласно лицензии Apache 2.0, вы можете использовать MultiTalk в коммерческих проектах. Обратите внимание, что некоторые материалы и видео могут быть основаны на реальных видео с ограничениями на коммерческое использование.

## Технические детали

### Какая архитектура модели используется?

MultiTalk основан на модели Wan2.1 I2V-14B и использует инновационный метод Label Rotary Position Embedding (L-RoPE) для решения проблемы связывания аудио и персонажей.

### Какой точности синхронизации губ достигает MultiTalk?

MultiTalk достигает лидирующего уровня точности синхронизации губ в индустрии на бенчмарках talking head, talking body и многопользовательских наборах данных.

### Поддерживается ли русский язык в TTS?

TTS поддерживает несколько языков. Для русского языка может потребоваться использование соответствующей модели TTS или предварительная обработка аудио.
