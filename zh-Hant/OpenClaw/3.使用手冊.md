# OpenClaw 使用手冊

本手冊介紹 OpenClaw 的日常使用方法和核心功能。

## 基本概念

### Gateway

Gateway 是 OpenClaw 的核心元件，負責對話管理、訊息路由和通道連接。你只需要運行一個 Gateway 程式，它會自動處理所有通道的接入請求。

### 通道（Channels）

通道是指你用來與 AI 助手交流的聊天應用。OpenClaw 支援同時連接多個通道，包括 WhatsApp、Telegram、Discord、iMessage 等。

### 代理（Agents）

代理是 AI 助手的實例。每個代理有獨立的工作區和對話。你可以配置多個代理，分別處理不同的工作任務。

## 日常操作

### 啟動和停止 Gateway

```bash
# 啟動 Gateway（背景服務）
openclaw gateway start

# 停止 Gateway
openclaw gateway stop

# 重新啟動 Gateway
openclaw gateway restart

# 查看狀態
openclaw gateway status

# 前景執行（用於除錯）
openclaw gateway --port 18789
```

### 傳送訊息

```bash
# 傳送私人訊息
openclaw message send --target +15555550123 --message "你好"

# 傳送群組訊息
openclaw message send --target "群組名稱" --message "大家好"
```

### 管理配置

```bash
# 查看當前配置
openclaw config get

# 設定配置項
openclaw config set agents.defaults.model "anthropic/claude-sonnet-4-5"

# 刪除配置項
openclaw config unset channels.whatsapp.enabled
```

## 配置管理

### 配置檔案位置

配置檔案位於 `~/.openclaw/openclaw.json`。

### 熱重載

OpenClaw 支援配置熱重載，修改配置檔案後無需手動重啟 Gateway。大多數配置更改會自動生效。

### 常用配置範例

### 設定允許的訊息來源

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  }
}
```

### 設定群組聊天提及規則

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true }
      }
    }
  }
}
```

### 配置多個代理

```json5
{
  agents: {
    list: [
      { id: "home", default: true, workspace: "~/.openclaw/workspace-home" },
      { id: "work", workspace: "~/.openclaw/workspace-work" }
    ]
  }
}
```

## 通道管理

### 新增新通道

```bash
# 登入 WhatsApp
openclaw channels login whatsapp

# 登入 Telegram
openclaw channels login telegram
```

### 查看已連接的通道

```bash
openclaw channels list
```

## 進階功能

### 定時任務（Cron Jobs）

配置定時執行的任務：

```json5
{
  cron: {
    enabled: true,
    maxConcurrentRuns: 2
  }
}
```

### 心跳檢測（Heartbeat）

設定定期檢查：

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last"
      }
    }
  }
}
```

### 沙箱模式

在隔離的 Docker 容器中執行代理：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "agent"
      }
    }
  }
}
```

### Webhooks

設定 HTTP Webhook 端點：

```json5
{
  hooks: {
    enabled: true,
    token: "your-secret-token",
    path: "/hooks"
  }
}
```

## 故障排查

### 查看日誌

```bash
# 查看 Gateway 日誌
openclaw logs

# 查看即時日誌
openclaw logs -f
```

### 健康檢查

```bash
openclaw health
```

### 診斷問題

```bash
openclaw doctor
```

## 控制介面

OpenClaw 提供網頁控制介面，你可以：

- 傳送和接收訊息
- 管理配置
- 查看對話歷史
- 管理行動節點
- 監控 Gateway 狀態

訪問位址：`http://127.0.0.1:18789/`
