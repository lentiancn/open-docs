# Guía de Instalación de Nacos

## Requisitos del Entorno

Nacos requiere un entorno de ejecución Java. Asegúrate de usar las siguientes versiones:

- Sistema operativo de 64 bits: Linux/Unix/Mac/Windows soportado, Linux/Unix/Mac recomendado
- JDK de 64 bits 17+ (JDK 17 o superior recomendado)
- Descarga del JDK: https://www.oracle.com/java/technologies/downloads/#java17
- Configuración del entorno JDK: https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## Configuración Recomendada

Se recomienda ejecutar Nacos con al menos la siguiente configuración de máquina:

- CPU: 2 núcleos
- Memoria: 4 GB
- Disco: 60 GB

## Métodos de Instalación

Nacos soporta múltiples métodos de instalación. Puedes elegir el método apropiado según tus necesidades.

### Método 1: Uso de Implementación con un Clic de nacos-setup (Recomendado)

nacos-setup proporciona implementación automatizada con un clic, manejando automáticamente descarga, configuración de autenticación, detección de puerto y verificación del entorno Java.

#### Instalar nacos-setup

Linux/macOS:

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows (PowerShell):

```powershell
# Ejecutar usando PowerShell
```

#### Iniciar Servicio Nacos

Implementar localmente la última versión de Nacos autónomo con un clic:

```bash
nacos-setup
```

Implementar localmente una versión específica de Nacos autónomo:

```bash
nacos-setup -v 3.1.1
```

Implementar localmente la última versión de Nacos en clúster:

```bash
nacos-setup -c cluster
```

Configurar almacenamiento externo globalmente, ingresar dirección de base de datos, puerto, usuario y contraseña, luego ejecutar nacos-setup nuevamente:

```bash
nacos-setup --datasource-conf
```

Para más uso, consulta el manual de usuario de nacos-setup.

### Método 2: Descarga Manual e Implementación

#### Descargar Paquete de Instalación

Puedes obtener el paquete de distribución de Nacos de dos maneras:

**Método 1: Descargar desde el Sitio Web Oficial de Nacos**

Visita la página de descarga de versiones del sitio web oficial de Nacos, selecciona una versión estable, luego haz clic para descargar el archivo zip en la columna de paquete binario.

**Método 2: Descargar desde GitHub**

Visita la página de Releases de Nacos en GitHub y descarga el paquete de distribución para la versión correspondiente.

Nota: Cuando muchos usuarios descargan simultáneamente, puedes encontrar fallas de limitación de descarga. Si esto sucede, espera y reintenta, o descarga desde GitHub.

#### Extraer Paquete de Distribución Nacos

Linux/Unix/Mac:

```bash
unzip nacos-server-$version.zip
# o
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows:

Usa herramientas de extracción para descomprimir nacos-server-$version.zip directamente.

#### Iniciar Servidor

Comando de inicio Linux/Unix/Mac (standalone significa modo no clúster):

```bash
sh startup.sh -m standalone
```

Si estás usando Ubuntu, o encuentras mensajes de error sobre símbolos no encontrados, prueba:

```bash
bash startup.sh -m standalone
```

El programa de inicio te pedirá que ingreses tres configuraciones relacionadas con autenticación (a partir de Nacos 3.0.0, la autenticación de consola está habilitada por defecto, por lo que las siguientes tres configuraciones de autenticación deben completarse):

```
nacos.core.auth.plugin.nacos.token.secret.key falta, por favor establece con cadena Base64: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key falta, por favor establece: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value falta, por favor establece: ${your_input_server_identity_key}
```

Comando de inicio Windows:

```bash
cmd startup.cmd
```

O haz doble clic en startup.cmd para ejecutar.

### Método 3: Instalación con Docker

#### Extraer Imagen Nacos

```bash
docker pull nacos/nacos-server
```

#### Ejecutar Modo Autónomo

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### Ejecutar Modo Clúster

Requiere implementación en clúster con Docker Compose o Kubernetes.

### Método 4: Instalación en Kubernetes

Nacos soporta implementación en entornos Kubernetes y puede instalarse usando Helm Chart u Operator. Para pasos específicos, consulta la documentación de implementación de Nacos en Kubernetes.

## Verificar Inicio del Servicio Nacos

Navega al directorio de logs de Nacos y usa `tail -f start.out` para ver los logs. Si ves el siguiente mensaje, el servicio ha iniciado correctamente:

```
Nacos started successfully in stand alone mode. use embedded storage
```

## Verificación Rápida

### Consola Nacos

Abre cualquier navegador e ingresa: http://127.0.0.1:8080/index.html para acceder a la consola Nacos.

Nota: La primera vez que la abras, necesitarás inicializar la contraseña del administrador para el usuario nacos. Usuario/contraseña por defecto: nacos/nacos@demo.

### Registro de Servicio

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### Descubrimiento de Servicio

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### Publicar Configuración

Primero, inicia sesión para obtener token de acceso:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

Usa el token de acceso para crear configuración:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### Obtener Configuración

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## Apagar Servidor

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

O haz doble clic en shutdown.cmd para ejecutar.

## Selección de Versión

La versión estable recomendada actualmente para producción es 3.1.1. Puedes conocer las funciones compatibles con cada versión desde:

- Página de descarga de versiones de Nacos
- Notas de lanzamiento de Nacos
- Anuncios de lanzamiento de Nacos

## Notas

1. A partir de Nacos 3.0.0, la autenticación de consola está habilitada por defecto. Los parámetros relacionados con autenticación deben configurarse en el primer inicio.
2. La implementación en modo clúster requiere base de datos externa (como MySQL) para almacenamiento de datos.
3. Los entornos de producción deben usar modo clúster para garantizar alta disponibilidad.
4. Asegúrate de que los puertos necesarios estén abiertos en el firewall (por defecto 8848, 9848).
5. Se recomienda JDK 17 o superior para un rendimiento óptimo.
