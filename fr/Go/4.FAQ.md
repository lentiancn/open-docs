# FAQ Go

## Problèmes d'Installation

### Que faire si l'installation de Go échoue?

L'échec de l'installation est généralement causé par quelques raisons courantes. Vérifiez d'abord si le système répond aux exigences minimales: Linux nécessite une version de noyau relativement récente, et Windows nécessite un système 64 bits. Si vous utilisez l'installation par tarball, assurez-vous d'avoir téléchargé la bonne version (amd64, arm64, etc.).

Si vous rencontrez des erreurs de permission, utilisez sudo sur Linux ou macOS pour élever les privilèges. Assurez-vous que le répertoire cible d'installation existe et est inscriptible:

```bash
# Créer le répertoire d'installation
sudo mkdir -p /usr/local
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
```

Si les problèmes persistent, essayez de télécharger à nouveau le programme d'installation; le fichier peut avoir été corrompu pendant le téléchargement. Vérifiez la somme de contrôle SHA256:

```bash
sha256sum go1.21.5.linux-amd64.tar.gz
```

### Comment vérifier que Go est correctement installé?

Exécutez les commandes suivantes pour vérifier si Go est correctement installé:

```bash
go version
go env GOROOT
go env GOPATH
```

Si la sortie affiche quelque chose comme "go1.21.5 linux/amd64", l'installation a réussi. Si elle indique commande non trouvée, vérifiez si la variable d'environnement PATH contient le répertoire binaire de Go.

## Problèmes de Développement

### Comment organiser la structure du projet?

Les modules (introduits dans Go 1.11) sont l'approche recommandée pour la gestion des dépendances. Créez un nouveau projet:

```bash
mkdir myproject
cd myproject
go mod init github.com/username/myproject
```

Structure de projet recommandée:

```
myproject/
├── go.mod
├── go.sum
├── main.go
├── internal/
│   ├── handler/
│   └── service/
└── pkg/
    └── utils/
```

- main.go: Point d'entrée du programme
- internal: Code privé qui ne peut pas être importé de l'extérieur
- pkg: Bibliothèques publiques qui peuvent être utilisées de l'extérieur

### Quelle est la différence entre GOPATH et Go Modules?

GOPATH est la méthode de gestion des dépendances utilisée avant Go 1.11, nécessitant que le code soit placé dans le répertoire $GOPATH/src. Go Modules, introduit dans Go 1.11, est une approche moderne qui permet de créer des projets à n'importe quel emplacement avec gestion des dépendances via les fichiers go.mod.

Go Modules est recommandé car il résout de nombreuses limitations de GOPATH, y compris la gestion des versions et le développement hors ligne. Depuis Go 1.16, Go Modules est devenu le mode par défaut.

### Comment gérer les importations circulaires?

Go n'autorise pas les importations circulaires entre les packages, ce qui provoque des erreurs de compilation. La solution est de refactorer le code pour éliminer les dépendances circulaires:

- Déplacer les types partagés vers un package séparé
- Utiliser des interfaces pour découpler les dépendances
- Fusionner le code dans un package

Vérifier les importations circulaires:

```bash
go mod tidy
```

### Comment détecter les fuites de goroutines?

Les fuites de goroutines se produisent généralement lorsque les canaux ne sont pas correctement fermés ou que personne ne reçoit des canaux envoyés. Utilisez l'outil pprof pour détecter:

```bash
# Activer pprof
import _ "net/http/pprof"

# Exporter les données d'analyse
go tool pprof http://localhost:6060/debug/pprof/goroutine
```

Ou utilisez runtime.NumGoroutine() pour imprimir périodiquement le nombre de goroutines et localiser où la fuite se produit.

### Comment déboguer?

Les utilisateurs de VS Code peuvent installer l'extension Go et appuyer sur F5 pour démarrer le débogage. GoLand a un support de débogage puissant intégré. Le débogage en ligne de commande utilise delve:

```bash
# Installer delve
go install github.com/go-delve/delve/cmd/dlv@latest

# Démarrer le débogage
dlv debug main.go
```

## Problèmes de Performance

### Comment analyser les performances?

Go dispose de l'outil d'analyse de performance pprof intégré. Activez d'abord le point de terminaison d'analyse des performances dans le code:

```go
import (
    "net/http"
    _ "net/http/pprof"
)

func main() {
    go func() {
        http.ListenAndServe("localhost:6060", nil)
    }()
    // Logique du programme principal
}
```

Utilisez ensuite pprof pour analyser:

```bash
go tool pprof http://localhost:6060/debug/pprof/profile  # Analyse CPU
go tool pprof http://localhost:6060/debug/pprof/heap     # Analyse mémoire
```

### Comment optimiser l'utilisation de la mémoire?

Utilisez sync.Pool pour réduire les allocations mémoire:

```go
var bufferPool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

func process() {
    buf := bufferPool.Get().([]byte)
    defer bufferPool.Put(buf)
    // Utiliser buf
}
```

Utilisez -ldflags pour réduire la taille du binaire:

```bash
go build -ldflags="-s -w" main.go
```

### Comment écrire du code haute performance?

- Évitez de créer des objets dans les boucles; réutilisez plutôt
- Utilisez defer pour le nettoyage des ressources, mais notez que defer a une surcharge
- Utilisez strconv au lieu de fmt pour la conversion de type
- Utilisez strings.Builder pour la concaténation de chaînes
- Préallouez la capacité des slices: make([]int, 0, 100)

## Problèmes de Déploiement

### Comment compiler pour différentes plateformes?

Utilisez les variables d'environnement GOOS et GOARCH pour la compilation croisée:

```bash
# Linux
GOOS=linux GOARCH=amd64 go build -o myapp-linux

# macOS
GOOS=darwin GOARCH=amd64 go build -o myapp-macos

# Windows
GOOS=windows GOARCH=amd64 go build -o myapp.exe
```

### Comment réduire la taille du fichier binaire?

La compilation statique réduit les dépendances:

```bash
CGO_ENABLED=0 go build -ldflags="-s -w" main.go
```

Vous pouvez également utiliser upx pour la compression:

```bash
upx -9 myapp
```

### Docker Build Multi-Étapes

Utilisez les builds multi-étapes pour réduire la taille de l'image:

```dockerfile
# Étape de construction
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o main .

# Étape d'exécution
FROM alpine
WORKDIR /app
COPY --from=builder /app/main .
CMD ["./main"]
```

## Problèmes d'Apprentissage

### Comment apprendre Go?

Parcours d'apprentissage recommandé:

1. Lisez la documentation officielle de Go: https://go.dev/doc/
2. Complétez le didacticiel interactif A Tour of Go
3. Lisez "The Go Programming Language"
4. Pratiquez: Essayez d'écrire un petit projet en Go
5. Lisez le code source d'excellents projets open source

### Pour quels types d'applications Go est-il adapté?

Go est particulièrement adapté pour développer: microservices, outils en ligne de commande, serveurs réseau, systèmes distribués, plateformes de calcul cloud, outils d'orchestration de conteneurs (comme Kubernetes, Docker), applications blockchain, etc.

Go n'est pas adapté aux applications nécessitant des fonctionnalités orientées objet complexes, ou aux scénarios avec des exigences de vitesse de démarrage extrêmement élevées (considérez Rust).

### Quels sont les projets Go recommandés?

- Kubernetes: Plateforme d'orchestration de conteneurs
- Docker: Runtime de conteneurs
- Prometheus: Système de surveillance
- etcd: Stockage clé-valeur distribué
- Hugo: Générateur de sites statiques
- Gin: Framework web
- Cobra: Framework d'outils en ligne de commande
