# Vue.js ユーザーマニュアル

## アプリケーションインスタンス

すべての Vue アプリケーションは、`createApp` 関数を使用して新しいアプリケーションインスタンスを作成することから始まります：

```javascript
import { createApp } from 'vue'

const app = createApp({
  /* ルートコンポーネントオプション */
})
```

### ルートコンポーネント
`createApp` に渡すオブジェクトは実際にはコンポーネントです。すべてのアプリには「ルートコンポーネント」が必要で、他のコンポーネントはその子として含まれます。

### アプリのマウント
アプリケーションインスタンスは `.mount()` メソッドが呼び出されるまで何もレンダリングしません：

```javascript
app.mount('#app')
```

### アプリ設定
アプリケーションインスタンスは `.config` オブジェクトを公開し、アプリレベルの設定が可能です：

```javascript
// アプリレベルのエラーハンドラー
app.config.errorHandler = (err) => {
  /* エラー処理 */
}

// グローバルコンポーネントの登録
app.component('TodoDeleteButton', TodoDeleteButton)
```

### 複数のアプリケーションインスタンス
同じページ上で複数の Vue アプリケーションを共存させることができます：

```javascript
const app1 = createApp({ /* ... */ })
app1.mount('#container-1')

const app2 = createApp({ /* ... */ })
app2.mount('#container-2')
```

## テンプレート構文

Vue は HTML ベースのテンプレート構文を使用し、レンダリングされた DOM をコンポーネントインスタンスのデータに宣言的にバインドできます。

### テキスト補間
最も基本的なデータバインディング形式は、「Mustache」構文（二重波括弧）を使用したテキスト補間です：

```html
<span>Message: {{ msg }}</span>
```

### 生の HTML
`v-html` ディレクティブを使用して実際の HTML を出力します：

```html
<p v-html="rawHtml"></p>
```

**セキュリティ警告**: 任意の HTML を動的にレンダリングすると XSS 脆弱性につながる可能性があります。`v-html` は信頼できるコンテンツでのみ使用し、ユーザー提供のコンテンツでは絶対に使用しないでください。

### 属性バインディング
HTML 属性内で Mustache は使用できません。代わりに `v-bind` ディレクティブを使用します：

```html
<!-- 完全構文 -->
<div v-bind:id="dynamicId"></div>

<!-- ショートハンド -->
<div :id="dynamicId"></div>

<!-- 同名ショートハンド（Vue 3.4+） -->
<div :id></div>
```

### ブール属性
ブール属性は、バインドされた値が真の場合に要素に含まれます：

```html
<button :disabled="isButtonDisabled">Button</button>
```

### 複数属性の動的バインディング
引数なしで `v-bind` を使用して、オブジェクトから複数の属性をバインドできます：

```javascript
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

## JavaScript 式

Vue はすべてのデータバインディング内で完全な JavaScript 式をサポートしています：

```html
{{ number + 1 }}
{{ ok ? 'YES' : 'NO' }}
{{ message.split('').reverse().join('') }}
<div :id="`list-${id}`"></div>
```

### 関数の呼び出し
バインディング式内でコンポーネント公開メソッドを呼び出すことができます：

```html
<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}
</time>
```

### 制限付きグローバルアクセス
テンプレート式は制限されたグローバルリスト（Math、Date など）にのみアクセスできます。追加のグローバルは `app.config.globalProperties` を介して明示的に定義できます。

## ディレクティブ

ディレクティブは `v-` プレフィックスを持つ特殊な属性です。

### 引数
一部のディレクティブはコロンで区切られた「引数」を取ります：

```html
<!-- v-bind 引数 -->
<a v-bind:href="url"> ... </a>
<a :href="url"> ... </a>

<!-- v-on 引数 -->
<a v-on:click="doSomething"> ... </a>
<a @click="doSomething"> ... </a>
```

### 動的引数
角括弧で囲んだ JavaScript 式をディレクティブ引数で使用できます：

```html
<a :[attributeName]="url"> ... </a>
<a @[eventName]="doSomething"> ... </a>
```

**制約事項**:
- 動的引数は文字列または null に評価される必要があります
- 複雑な式は避けて、代わりに算出プロパティを使用してください
- DOM テンプレートでは、動的キーに大文字を使用しないでください

### 修飾子
修飾子はドットで示される特殊な接尾辞で、ディレクティブを特別な方法でバインドすることを示します：

```html
<form @submit.prevent="onSubmit">...</form>
```

## リアクティビティの基礎

### リアクティブ状態の宣言
`ref()` を使用してリアクティブ参照を作成します：

```javascript
import { ref } from 'vue'

const count = ref(0)
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

### リアクティブオブジェクト
`reactive()` を使用してリアクティブオブジェクトを作成します：

```javascript
import { reactive } from 'vue'

const state = reactive({
  count: 0
})
```

### 算出プロパティ
`computed()` を使用して算出プロパティを作成します：

```javascript
import { ref, computed } from 'vue'

const author = ref({
  name: 'John Doe',
  books: [
    'Vue 2 Guide',
    'Vue 3 Guide',
    'Vue Mastery'
  ]
})

const publishedBooksMessage = computed(() => {
  return author.value.books.length > 0
    ? 'Yes'
    : 'No'
})
```

### ウォッチャー
`watch()` を使用してリアクティブデータの変化を監視します：

```javascript
import { ref, watch } from 'vue'

const question = ref('')
const answer = ref('Questions usually contain a question mark. ;-)')

// 単一の ref を監視
watch(question, async (newQuestion, oldQuestion) => {
  if (newQuestion.indexOf('?') > -1) {
    answer.value = 'Thinking...'
    try {
      const res = await fetch('https://yesno.wtf/api')
      answer.value = (await res.json()).answer
    } catch (error) {
      answer.value = 'Error! Could not reach the API. ' + error
    }
  }
})
```

## コンポーネントの基礎

### コンポーネントの定義
```javascript
// コンポーネントの定義
const Component = {
  data() {
    return {
      count: 0
    }
  },
  template: `<button @click="count++">Count is: {{ count }}</button>`
}
```

### Props の受け渡し
```javascript
// 子コンポーネント
const Child = {
  props: ['msg'],
  template: `<div>{{ msg }}</div>`
}

// 親コンポーネント
const Parent = {
  components: { Child },
  template: `<Child msg="Hello!"></Child>`
}
```

### イベントの発行
```javascript
// 子コンポーネント
const Child = {
  emits: ['response'],
  methods: {
    respond() {
      this.$emit('response', 'Hello from child!')
    }
  },
  template: `<button @click="respond">Respond</button>`
}
```

### スロット
```html
<!-- 子コンポーネント -->
<template>
  <div>
    <slot></slot>
    <slot name="footer"></slot>
  </div>
</template>

<!-- 親コンポーネント -->
<Child>
  <p>This is default slot content</p>
  <template #footer>
    <p>This is footer slot content</p>
  </template>
</Child>
```
