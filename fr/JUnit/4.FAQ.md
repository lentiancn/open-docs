# JUnit FAQ

## Dois-je utiliser JUnit 4 ou JUnit 5 ?

Pour les nouveaux projets, JUnit 5 est fortement recommandé. Il offre des fonctionnalités plus riches, notamment les tests imbriqués, les tests paramétrés, les tests dynamiques et une meilleure extensibilité. Pour les projets existants avec JUnit 4, vous pouvez utiliser JUnit Vintage pour exécuter les tests sur la plateforme JUnit 5 et migrer progressivement.

## Comment exécuter des tests JUnit 4 dans JUnit 5 ?

Ajoutez la dépendance du moteur JUnit Vintage :

```xml
<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.14.3</version>
    <scope>test</scope>
</dependency>
```

## Pourquoi mes tests ne s'exécutent-ils pas ?

Raisons courantes : classe de test sans annotation @Test, méthodes de test non publiques, maven-surefire-plugin manquant ou useJUnitPlatform() non configuré, version Java non conforme aux exigences.

## Quelle est la version Java minimale pour JUnit 5 ?

JUnit 5 nécessite Java 8 ou supérieur. JUnit 4 nécessite Java 5 ou supérieur.

## Comment ignorer certains tests ?

Utilisez l'annotation @Disabled pour ignorer des tests individuels ou des classes entières :

```java
@Disabled("Temporairement désactivé")
@Test
void testSkipped() {
    // Ne s'exécutera pas
}
```

## Comment tester des méthodes privées ?

Les tests unitaires ne doivent tester que les méthodes publiques. Si vous devez tester des méthodes privées, refactorez le code pour déplacer la logique à tester dans des méthodes publiques séparées, ou utilisez la réflexion (non recommandé).

## Comment obtenir des informations détaillées lors d'un échec d'assertion ?

Utilisez le troisième paramètre pour les messages d'erreur personnalisés :

```java
assertEquals(expected, actual, "Le résultat du calcul est incorrect");
```

## Comment tester les exceptions ?

Utilisez la méthode assertThrows :

```java
@Test
void testException() {
    assertThrows(IllegalArgumentException.class, () -> {
        throw new IllegalArgumentException("message d'erreur");
    });
}
```

## Comment exécuter les tests en parallèle ?

Configurez l'exécution parallèle dans JUnit 5 :

```java
import org.junit.jupiter.api.parallel.Execution;
import org.junit.jupiter.api.parallel.ExecutionMode;

@Execution(ExecutionMode.CONCURRENT)
class ConcurrentTests {
    // Les tests s'exécuteront en parallèle
}
```

## Quelle est la différence entre @Before et @BeforeEach ?

@Before est une annotation JUnit 4, @BeforeEach est une annotation JUnit 5. Elles servent toutes deux le même objectif : s'exécuter avant chaque méthode de test. JUnit 5 supporte également @BeforeAll et @AfterAll pour les opérations de setup/teardown qui s'exécutent une fois pour toute la classe de test.

## Comment créer des tests paramétrés ?

Utilisez l'annotation @ParameterizedTest avec @ValueSource :

```java
@ParameterizedTest
@ValueSource(strings = {"a", "b", "c"})
void testWithParams(String param) {
    assertNotNull(param);
}
```

## Que faire si les tests dépassent le timeout ?

Utilisez l'annotation @Timeout pour définir un délai :

```java
@Test
@Timeout(value = 100, unit = TimeUnit.MILLISECONDS)
void testWithTimeout() {
    // Échouera si dépasse 100 millisecondes
}
```

## Comment répéter des tests ?

Utilisez l'annotation @RepeatedTest :

```java
@RepeatedTest(10)
void testRepeated() {
    // Se répète 10 fois
}
```

## Qu'est-ce que le modèle d'extension JUnit 5 ?

JUnit 5 utilise l'annotation @ExtendWith pour fournir un mécanisme d'extension permettant une personnalisation plus flexible des tests :

```java
@ExtendWith(MyExtension.class)
@Test
void testWithExtension() {
    // Utilise l'extension personnalisée
}
```
