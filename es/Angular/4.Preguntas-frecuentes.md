# Preguntas Frecuentes

Este documento recopila las preguntas técnicas más frecuentes que surgen al trabajar con Angular y sus respuestas.

## Preguntas Básicas

### ¿Qué es Angular?

Angular es un framework de código abierto desarrollado y mantido por Google para crear aplicaciones web modernas. Utiliza TypeScript como lenguaje principal de desarrollo y ofrece características como arquitectura basada en componentes, enlace de datos bidireccional e inyección de dependencias.

### ¿Cuál es la diferencia entre Angular y AngularJS?

AngularJS es el nombre de Angular 1.x, el predecesor de Angular. Angular es una versión completamente reescrita, utiliza TypeScript y tiene una arquitectura totalmente diferente. No es compatible con AngularJS.

### ¿Es difícil aprender Angular?

Angular tiene una curva de aprendizaje moderada, especialmente conceptos como inyección de dependencias y RxJS que requieren tiempo para entender. Sin embargo, tiene documentación excelente y una comunidad activa, lo que hace que una vez dominado, el desarrollo sea muy eficiente.

## Problemas de Instalación

### ¿Qué hacer si npm install falla?

1. Limpiar la caché de npm: `npm cache clean --force`
2. Eliminar node_modules y package-lock.json
3. Usar un repositorio mirror de tu región
4. Verificar que la versión de Node.js sea compatible

### ¿El comando de Angular CLI no se encuentra?

Asegúrate de haberlo instalado globalmente:

```bash
npm install -g @angular/cli```

Si aún no funciona, intenta usar npx:

```bash
npx @angular/cli new my-app
```

### ¿La versión de Node.js no es compatible?

Usa nvm para gestionar las versiones de Node.js:

```bash
nvm install 18
nvm use 18
```

## Problemas de Desarrollo

### ¿Los estilos del componente no se aplican?

1. Verificar que la ruta del archivo de estilos sea correcta
2. Confirmar que no haya errores de sintaxis en styles.scss
3. Revisar la configuración de ViewEncapsulation

### ¿La navegación de rutas falla?

1. Verificar que la configuración de rutas sea correcta
2. Confirmar que RouterModule esté importado correctamente
3. Comprobar si hay algún guard de ruta bloqueando la navegación

### ¿Las solicitudes HTTP fallan?

1. Verificar que el servicio backend esté iniciado
2. Confirmar la configuración de CORS
3. Usar HttpInterceptor para añadir manejo de errores

### ¿Los datos no se actualizan?

1. Revisar la estrategia de detección de cambios
2. Si usas OnPush, necesitas dispara la detección manualmente
3. Asegurarte de que las referencias de datos hayan cambiado

## Problemas de Rendimiento

### ¿La carga inicial es lenta?

1. Habilitar carga diferida de rutas
2. Usar estrategias de precarga
3. Optimizar recursos de imágenes
4. Habilitar construcción en modo producción

### ¿Fugas de memoria?

1. Cancelar la suscripción de Observables en ngOnDestroy
2. Usar takeUntil o el pipe async
3. Limpiar temporizadores y escuchadores de eventos

### ¿Detección de cambios demasiado frecuente?

1. Usar estrategia OnPush
2. Separar las propiedades que cambian de las que no
3. Usar trackBy para optimizar *ngFor

## Problemas de Pruebas

### ¿Las pruebas unitarias fallan?

1. Revisar la configuración de TestBed
2. Confirmar que las operaciones asíncronas estén correctamente manejadas
3. Usar fakeAsync y tick para probar código asíncrono

### ¿No encuentra módulos?

Asegúrate de que las rutas estén correctamente configuradas en tsconfig.json.

## Problemas de Construcción

### ¿La construcción falla?

1. Ver el mensaje de error específico
2. Revisar las opciones de compilación de TypeScript
3. Limpiar la caché y reconstruir

### ¿Los archivos de construcción son muy grandes?

1. Habilitar modo producción
2. Usar carga diferida
3. Verificar si hay dependencias sin usar

## Errores Comunes

### ExpressionChangedAfterItHasBeenCheckedError

Se modificó un valor durante la verificación que no debería cambiar. Revisa si hay código en ngOnInit que modifique directamente algo relacionado con la vista.

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

Es el mismo error anterior, revisa la temporización de los enlaces de datos.

### Cannot read property of undefined

Se accede a los datos antes de que estén inicializados. Usa el operador opcional (?.) o *ngIf como protección.

### Error de CORS

El backend no tiene configurado CORS, o hay problemas con las cabeceras de la solicitud del frontend.

### Error 401/403

Error de autenticación, verifica la configuración del token y los permisos.

## Actualización de Versiones

### ¿Cómo actualizar Angular?

1. Usar CLI para actualizar: `ng update @angular/core`
2. Consultar la guía oficial de actualización
3. Actualizar las dependencias una por una
4. Ejecutar las pruebas para asegurar que todo funcione

### ¿Qué hacer si la actualización falla?

1. Hacer una copia de seguridad del código
2. Actualizar manualmente las versiones en package.json
3. Eliminar node_modules y reinstallar
