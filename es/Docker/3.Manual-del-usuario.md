# Manual del Usuario de Docker

Este manual te ayudará a comenzar rápidamente con Docker y dominar las operaciones comunes para el uso diario.

## Prerrequisitos

Antes de comenzar, asegúrate de tener:

- Docker instalado (Docker Desktop o Docker Engine)
- Acceso a la línea de comandos
- Conocimientos básicos de operaciones de línea de comandos

## Conceptos Básicos

### Imágenes y Contenedores

- **Imagen**: Una plantilla de solo lectura usada para crear contenedores
- **Contenedor**: Una instancia en ejecución de una imagen

### Comandos Comunes

```bash
# Ejecutar un contenedor
docker run hello-world

# Listar todos los contenedores
docker ps -a

# Listar todas las imágenes
docker images

# Detener un contenedor
docker stop id_contenedor

# Eliminar un contenedor
docker rm id_contenedor

# Eliminar una imagen
docker rmi id_imagen
```

## Trabajando con Contenedores

### Ejecutar un Contenedor

El comando básico para ejecutar un contenedor es:

```bash
docker run [opciones] nombre_imagen [comando]
```

Ejemplos:

```bash
# Iniciar un contenedor interactivo
docker run -it ubuntu bash

# Ejecutar un contenedor en segundo plano
docker run -d nginx

# Ejecutar un contenedor con redirección de puertos
docker run -p 8080:80 nginx
```

### Gestionar Contenedores

```bash
# Mostrar todos los contenedores en ejecución
docker ps

# Mostrar todos los contenedores (incluidos los detenidos)
docker ps -a

# Iniciar un contenedor
docker start nombre_contenedor

# Detener un contenedor
docker stop nombre_contenedor

# Reiniciar un contenedor
docker restart nombre_contenedor

# Eliminar un contenedor
docker rm nombre_contenedor
```

### Ver Logs de Contenedores

```bash
# Mostrar logs de un contenedor
docker logs nombre_contenedor

# Seguir logs en tiempo real
docker logs -f nombre_contenedor

# Mostrar las últimas 100 líneas
docker logs --tail 100 nombre_contenedor
```

## Trabajando con Imágenes

### Buscar y Descargar Imágenes

```bash
# Buscar imágenes en Docker Hub
docker search nginx

# Descargar una imagen
docker pull nginx

# Descargar una imagen con etiqueta
docker pull nginx:latest
```

### Crear Imágenes

Puedes crear tus propias imágenes con un Dockerfile:

```dockerfile
FROM ubuntu:20.04
RUN apt-get update
RUN apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Luego crea la imagen:

```bash
docker build -t mi-imagen-nginx .
```

### Gestionar Imágenes

```bash
# Listar todas las imágenes
docker images

# Eliminar una imagen
docker rmi nombre_imagen

# Eliminar imágenes sin usar
docker image prune
```

## Usar Volúmenes

Los volúmenes se usan para almacenar datos fuera de los contenedores.

```bash
# Crear un volumen
docker volume create mi_volumen

# Montar un volumen
docker run -v mi_volumen:/datos nginx

# Listar todos los volúmenes
docker volume ls

# Eliminar un volumen
docker volume rm mi_volumen
```

## Redes

Docker ofrece funciones de red para comunicación entre contenedores.

```bash
# Crear una red
docker network create mi_red

# Conectar un contenedor a una red
docker run --network mi_red nginx

# Listar todas las redes
docker network ls
```

## Docker Compose

Docker Compose permite definir y ejecutar aplicaciones de múltiples contenedores.

### Ejemplo docker-compose.yml

```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: contraseña
```

### Comandos

```bash
# Iniciar todos los contenedores
docker-compose up

# Iniciar en segundo plano
docker-compose up -d

# Detener todos los contenedores
docker-compose down
```

## Consejos y Mejores Prácticas

### Seguridad

- Usa imágenes oficiales siempre que sea posible
- No ejecutes contenedores como root
- Mantén las imágenes actualizadas
- Escanea imágenes en busca de vulnerabilidades

### Rendimiento

- Usa archivos .dockerignore
- Utiliza compilaciones de múltiples etapas
- Minimiza el número de capas en las imágenes
- Usa etiquetas de imagen específicas

### Mantenibilidad

- Usa nombres de contenedores significativos
- Documenta tus Dockerfiles
- Usa volúmenes para datos persistentes
- Registra la actividad de los contenedores
