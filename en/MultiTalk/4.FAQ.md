# Frequently Asked Questions

This page compiles common questions and answers from MultiTalk users.

## Basic Questions

### What is MultiTalk?

MultiTalk is an audio-driven multi-person conversational video generation framework developed by the MeiGen-AI team. It can generate videos containing multi-person interactions based on multi-stream audio input, reference images, and text prompts, supporting lip synchronization with audio.

### What features does MultiTalk support?

MultiTalk supports various features: single-person and multi-person conversational video generation, cartoon character video generation, singing video generation, and instruction-following video generation. It supports output at 480P and 720P resolutions, with maximum video clips up to 15 seconds.

### Is MultiTalk free?

MultiTalk is an open-source project and can be used for free. The project uses the Apache 2.0 license, allowing academic and commercial use.

## Installation Issues

### What to do if CUDA version mismatch occurs during installation?

MultiTalk requires CUDA 12.1 or higher. Please check the system CUDA version first: nvcc --version. If the version is too low, you need to upgrade the CUDA driver. You can also try using the corresponding PyTorch CUDA version.

### How to handle insufficient VRAM?

If VRAM is insufficient, you can try the following methods: Use --num_persistent_param_in_dit 0 parameter to reduce memory usage; Use TeaCache acceleration to reduce memory usage; Use LoRA acceleration to reduce steps; Lower output resolution.

### What to do if flash_attn installation fails?

flash_attn installation requires a specific compilation environment. Make sure ninja and the correct CUDA toolkit are installed. If it continues to fail, try installing a precompiled version or skip this dependency.

### What to do if model download is too slow?

You can use mirrors or other download tools to pre-download model files and then move them to the specified directory. You can also try using domestic mirror sources.

## Usage Issues

### What to do if generated video lips are not synchronized?

You can try increasing the Audio CFG parameter value, recommended settings between 3 and 5. You can also increase sampling steps to improve synchronization.

### What to do if video generation is too slow?

You can enable the following acceleration methods: Use TeaCache acceleration, which can improve speed by 2 to 3 times; Use LoRA acceleration, FusioniX requires only 8 steps, lightx2v requires only 4 steps; Use quantization version to reduce computation.

### How to generate longer conversational videos?

Use streaming mode to generate long videos. Ensure Audio CFG is set to 3 to maintain color consistency. Generated videos can be up to 15 seconds long.

### What to do if person binding is wrong in multi-person videos?

This is a common issue in multi-person video generation. MultiTalk solves this through L-RoPE technology. Ensure the input JSON correctly specifies the coordinates of each person region.

### How to use TTS text-to-speech functionality?

Prepare an input JSON file containing text, set --audio_mode tts parameter to automatically convert text to speech and generate video.

### Can cartoon character videos be generated?

Yes, MultiTalk supports cartoon character generation. Use cartoon character images in the reference image with corresponding descriptive text.

## Technical Issues

### What input audio formats are supported?

Common audio formats are supported, such as WAV and MP3. Audio should contain clear human speech.

### How to choose video resolution?

The current version supports 480P resolution by default. 720P requires multiple GPU configurations. 480P is sufficient for most usage scenarios.

### How to run on low VRAM devices?

Using --num_persistent_param_in_dit 0 parameter can significantly reduce VRAM requirements, allowing operation on a single RTX 4090.

### How to integrate with other platforms?

MultiTalk supports multiple usage methods: Can be called directly through Replicate cloud platform; Can be integrated into ComfyUI workflows; Can also use the Gradio interface provided by the project.
