# JIRA FAQ

> Frequently Asked Questions about JIRA

---

## General

### What is JIRA?

JIRA is a project management and issue tracking tool developed by Atlassian. It's widely used in software development for managing projects, tracking issues, and agile workflows.

### What types of issues can I create?

- Bug
- Story
- Task
- Epic
- Subtask
- Custom issue types

---

## Usage

### How to create an issue?

1. Click "Create" button in top navigation
2. Select project
3. Choose issue type
4. Fill in summary and description
5. Assign to team member
6. Set priority and labels
7. Submit

### How to create a sprint board?

1. Go to your project
2. Click "Boards" > "Create board"
3. Select "Scrum board"
4. Choose the project
5. Configure backlog and sprint settings

### How to configure workflows?

1. Go to Project Settings > Workflows
2. Click "Add workflow"
3. Design workflow steps
4. Configure transitions
5. Associate with issue types

---

## Administration

### How to add users?

1. Go to Site Administration
2. Click "Users"
3. "Invite users" or add manually
4. Assign appropriate licenses

### How to create custom fields?

1. Go to Project Settings > Custom fields
2. Click "Add custom field"
3. Select field type
4. Configure field options
5. Associate with screens

---

## Integrations

### How to integrate with GitHub?

1. Go to Project Settings > Integrations
2. Find GitHub
3. Click "Connect"
4. Authorize access
5. Configure repositories

### How to integrate with Slack?

1. Install JIRA app in Slack
2. Connect your workspace
3. Configure notifications

---

## Troubleshooting

### Can't create issues?

**Solutions**:
1. Check permissions
2. Verify project access
3. Ensure issue type is available

### Notifications not working?

**Solutions**:
1. Check notification scheme
2. Verify user email settings
3. Check spam folder
