# Kotlin 安装指南

## 环境准备

在开始安装 Kotlin 之前，需要确保你的计算机满足基本的系统要求。Kotlin 可以在多种操作系统上运行，包括 Windows、macOS、Linux 以及其他类 Unix 系统。无论你使用哪种操作系统，都需要确保有足够的磁盘空间来安装开发工具和运行编译后的程序。

对于 Windows 用户，建议使用 Windows 10 或更高版本以获得最佳体验。macOS 用户需要 macOS 10.14 或更高版本。Linux 用户可以使用任何主流发行版，如 Ubuntu、Fedora、Debian 等。

## 通过 IDE 安装

### IntelliJ IDEA

IntelliJ IDEA 是 JetBrains 出品的旗舰级集成开发环境，对 Kotlin 提供了最完善的支持。如果你的主要目标是 Kotlin 开发，IntelliJ IDEA 是最佳选择。

IntelliJ IDEA 分为两个版本：社区版（Community）和旗舰版（Ultimate）。社区版免费开源，功能已经足够支持 Kotlin 开发。旗舰版面向企业用户，提供了更多高级特性，如高级数据库工具、企业级安全功能等。

安装步骤如下：首先，访问 JetBrains 官方网站（https://www.jetbrains.com/idea/download/），根据你的操作系统选择对应的安装包。下载完成后，运行安装程序，按照向导指示完成安装。在安装过程中，可以选择安装路径和创建桌面快捷方式等选项。

安装完成后，首次启动 IntelliJ IDEA 时，IDE 会自动检测并提示安装 Kotlin 插件。如果系统没有自动检测，你也可以手动在插件市场中搜索 Kotlin 并安装。安装完成后，重启 IDE 即可开始 Kotlin 开发。

### Android Studio

Android Studio 是 Google 官方的 Android 开发 IDE，内置了 Kotlin 支持。如果你主要进行 Android 开发，Android Studio 是理想的选择。

访问 Android 开发者官网（https://developer.android.com/studio）下载 Android Studio。下载完成后，运行安装程序完成安装。Android Studio 默认包含了 Kotlin 插件，无需额外配置即可开始开发。

Android Studio 每年会发布多个更新版本，每个版本都包含最新的 Kotlin 支持和 Android 开发工具。建议保持版本更新以获得最佳开发体验和最新的语言特性。

## 通过命令行安装

除了使用 IDE，你也可以选择安装 Kotlin 命令行编译器。这种方式适合喜欢轻量级开发环境或需要在服务器上进行 Kotlin 开发的用户。

### 手动安装

首先，访问 Kotlin GitHub Releases 页面（https://github.com/JetBrains/kotlin/releases）下载最新版本的编译器。下载完成后是一个 zip 压缩包，将其解压到指定目录即可。

解压后，将 kotlinc/bin 目录添加到系统的环境变量中，这样就可以在命令行中直接使用 kotlinc 命令。在 Windows 上，可以通过系统属性中的环境变量设置来添加路径。在 macOS 和 Linux 上，可以修改 ~/.bashrc 或 ~/.zshrc 文件，添加类似以下的行：

export PATH="$PATH:/path/to/kotlinc/bin"

添加完成后，打开一个新的终端窗口，输入 kotlinc -version 命令，如果看到版本信息，说明安装成功。

### 使用 SDKMAN! 安装

SDKMAN! 是一个用于管理 Java 开发工具版本的工具，在类 Unix 系统上使用非常方便。如果你使用的是 macOS、Linux、Cygwin、FreeBSD 或 Solaris，SDKMAN! 是安装 Kotlin 的推荐方式。

首先，需要安装 SDKMAN!。在终端中执行以下命令：

curl -s "https://get.sdkman.io" | bash

安装完成后，需要重新加载环境配置：

source ~/.sdkman/bin/sdkman-init.sh

然后就可以使用 sdk 命令安装 Kotlin 了：

sdk install kotlin

安装过程中会提示确认版本信息，输入确认后即可完成安装。安装完成后，同样可以使用 kotlinc -version 验证安装。

### 使用 Homebrew 安装

在 macOS 上，还可以使用 Homebrew 包管理器来安装 Kotlin。首先确保已经安装了 Homebrew，然后执行以下命令：

brew update
brew install kotlin

Homebrew 会自动下载并安装最新版本的 Kotlin 编译器。安装完成后，同样可以使用 kotlinc -version 验证安装。

### 使用 Snap 安装

在 Ubuntu 16.04 及更高版本上，可以使用 Snap 包管理器安装 Kotlin。执行以下命令：

sudo snap install --classic kotlin

Snap 安装方式会将 Kotlin 安装为系统级应用，可以在任何位置通过命令行调用。

## 创建第一个 Kotlin 程序

无论你选择哪种安装方式，都可以通过以下步骤创建和运行第一个 Kotlin 程序。

首先，创建一个名为 hello.kt 的文件，内容如下：

fun main() {
    println("Hello, World!")
}

这是一个最简单的 Kotlin 程序，定义了一个 main 函数，这是 Kotlin 程序的入口点。println 函数用于向控制台输出文本。

如果使用命令行编译器，编译运行这个程序需要两步。首先是编译：

kotlinc hello.kt -include-runtime -d hello.jar

这里的 -include-runtime 参数会将 Kotlin 运行时库打包到生成的 jar 文件中，使 jar 文件可以独立运行。-d 参数指定输出文件的路径。

编译完成后，运行程序：

java -jar hello.jar

你应该能在控制台看到 "Hello, World!" 的输出。

如果不想生成独立的 jar 文件，也可以直接使用 kotlinc 的交互式运行功能：

kotlinc -script hello.kt

或者使用 kotlin 命令直接运行编译后的 class 文件：

kotlin -classpath hello.jar HelloKt

其中 HelloKt 是 Kotlin 编译器为 hello.kt 文件生成的主类名。

## 使用 REPL

Kotlin 提供了一个交互式的 REPL（Read-Eval-Print Loop）环境，你可以输入 Kotlin 代码并立即看到执行结果。这对于学习语言特性或快速测试代码片段非常有用。

要启动 REPL，只需在命令行中输入：

kotlinc

不带任何参数运行 kotlinc 会进入 REPL 模式。在这个模式下，你可以输入任何有效的 Kotlin 代码，按回车后即可看到执行结果。例如：

>>> val x = 10
>>> val y = 20
>>> x + y
30

输入 :quit 或按 Ctrl+C 可以退出 REPL。

## Kotlin 脚本

Kotlin 还可以作为脚本语言使用。Kotlin 脚本文件以 .kts 为扩展名，文件中的顶层代码会被视为可执行代码。

例如，创建一个名为 list_folders.kts 的脚本：

import java.io.File

val path = if (args.contains("-d")) args[1 + args.indexOf("-d")]
 else "."

val folders = File(path).listFiles { file -> file.isDirectory() }
folders?.forEach { folder -> println(folder) }

运行这个脚本：

kotlinc -script list_folders.kts -- -d /path/to/folder

## 配置构建工具

在实际项目中，通常会使用 Gradle 或 Maven 作为构建工具。这两个工具都原生支持 Kotlin 项目。

### Gradle

Gradle 是目前 Kotlin 开发中最流行的构建工具。Kotlin 团队也提供了专门的 Kotlin DSL，让构建脚本可以使用 Kotlin 语法编写。

要创建一个基本的 Kotlin Gradle 项目，首先需要在项目中添加 Kotlin 插件。项目的 build.gradle 文件需要包含以下内容：

plugins {
    id 'org.jetbrains.kotlin.jvm' version '1.9.0'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation kotlin-stdlib
}

plugins {
    kotlin("jvm") version "1.9.0"
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(kotlin("stdlib"))
}

然后创建源代码目录结构，通常是 src/main/kotlin。在其中创建 Kotlin 源文件，使用 gradle build 命令即可编译项目。

### Maven

Maven 是另一个流行的 Java 项目构建工具，同样支持 Kotlin 项目。在 pom.xml 文件中添加 Kotlin 相关依赖即可。

## 验证安装

安装完成后，可以通过几个简单的命令来验证一切是否正常工作。

首先验证 Kotlin 编译器：

kotlinc -version

应该看到类似 "Kotlin compiler version 2.3.10" 的输出。

然后验证 Kotlin 标准库：

kotlin -e 'println("Hello Kotlin!")'

如果一切正常，说明安装成功，可以开始你的 Kotlin 之旅了。
