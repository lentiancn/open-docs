# Element Plus Usage Guide

This guide covers how to use Element Plus components effectively in your Vue 3 application.

## Basic Usage

### Hello World Example

```vue
<template>
  <div>
    <el-button type="primary">Hello Element Plus</el-button>
  </div>
</template>

<script setup>
import { ElButton } from 'element-plus'
</script>
```

With auto-import (recommended):

```vue
<template>
  <div>
    <el-button type="primary">Click Me</el-button>
    <el-input v-model="input" placeholder="Enter text"></el-input>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const input = ref('')
</script>
```

## Component Categories

Element Plus provides these component categories:

| Category | Components |
|----------|------------|
| Basic | Button, Link, Typography, Divider |
| Form | Input, Select, Radio, Checkbox, Switch, Slider, TimePicker, DatePicker |
| Data | Table, Tree, Pagination, Tag |
| Notice | Alert, Message, Notification, Progress |
| Navigation | Menu, Tabs, Breadcrumb, Steps |
| Others | Dialog, Drawer, Card, Collapse |

### Form

```vue
<template>
  <el-form
    ref="ruleFormRef"
    :model="form"
    :rules="rules"
    label-width="120px"
  >
    <el-form-item label="Name" prop="name">
      <el-input v-model="form.name"></el-input>
    </el-form-item>
    
    <el-form-item label="Email" prop="email">
      <el-input v-model="form.email"></el-input>
    </el-form-item>
    
    <el-form-item label="Gender" prop="gender">
      <el-radio-group v-model="form.gender">
        <el-radio label="male">Male</el-radio>
        <el-radio label="female">Female</el-radio>
      </el-radio-group>
    </el-form-item>
    
    <el-form-item label="Hobby" prop="hobby">
      <el-checkbox-group v-model="form.hobby">
        <el-checkbox label="Reading"></el-checkbox>
        <el-checkbox label="Sports"></el-checkbox>
        <el-checkbox label="Music"></el-checkbox>
      </el-checkbox-group>
    </el-form-item>
    
    <el-form-item label="Country" prop="country">
      <el-select v-model="form.country" placeholder="Select country">
        <el-option label="China" value="china"></el-option>
        <el-option label="USA" value="usa"></el-option>
        <el-option label="UK" value="uk"></el-option>
      </el-select>
    </el-form-item>
    
    <el-form-item>
      <el-button type="primary" @click="submitForm(ruleFormRef)">
        Submit
      </el-button>
      <el-button @click="resetForm(ruleFormRef)">Reset</el-button>
    </el-form-item>
  </el-form>
</template>

<script setup>
import { reactive, ref } from 'vue'

const ruleFormRef = ref()

const form = reactive({
  name: '',
  email: '',
  gender: '',
  hobby: [],
  country: '',
})

const rules = reactive({
  name: [
    { required: true, message: 'Please enter name', trigger: 'blur' },
  ],
  email: [
    { required: true, message: 'Please enter email', trigger: 'blur' },
    { type: 'email', message: 'Invalid email format', trigger: 'blur' },
  ],
  gender: [
    { required: true, message: 'Please select gender', trigger: 'change' },
  ],
  country: [
    { required: true, message: 'Please select country', trigger: 'change' },
  ],
})

const submitForm = async (formEl) => {
  if (!formEl) return
  await formEl.validate((valid) => {
    if (valid) {
      console.log('Submit:', form)
    }
  })
}

const resetForm = (formEl) => {
  if (!formEl) return
  formEl.resetFields()
}
</script>
```

## Slots

Element Plus components support Vue slots:

```vue
<template>
  <!-- Default slot -->
  <el-button>Button Text</el-button>
  
  <!-- Named slot -->
  <el-dialog>
    <template #header>
      <h3>Custom Header</h3>
    </template>
    <p>Dialog content</p>
    <template #footer>
      <el-button>Cancel</el-button>
      <el-button type="primary">OK</el-button>
    </template>
  </el-dialog>
  
  <!-- Empty slot -->
  <el-table-column>
    <template #empty>
      <div>No Data Available</div>
    </template>
  </el-table-column>
</template>
```

## Events

```vue
<template>
  <el-button @click="handleClick">Click Me</el-button>
  
  <el-input
    v-model="input"
    @focus="handleFocus"
    @blur="handleBlur"
    @change="handleChange"
    @input="handleInput"
  ></el-input>
  
  <el-table
    :data="tableData"
    @select="handleSelect"
    @selection-change="handleSelectionChange"
    @row-click="handleRowClick"
  ></el-table>
</template>

<script setup>
const handleClick = () => {
  console.log('Button clicked')
}

const handleFocus = () => {
  console.log('Input focused')
}

const handleBlur = () => {
  console.log('Input blurred')
}

const handleChange = (value) => {
  console.log('Value changed:', value)
}

const handleInput = (value) => {
  console.log('Input:', value)
}

const handleSelect = (selection, row) => {
  console.log('Selected:', selection, row)
}

const handleSelectionChange = (selection) => {
  console.log('Selection changed:', selection)
}

const handleRowClick = (row) => {
  console.log('Row clicked:', row)
}
</script>
```

## v-model

Element Plus supports v-model for two-way binding:

```vue
<template>
  <!-- Input -->
  <el-input v-model="inputValue"></el-input>
  
  <!-- Checkbox -->
  <el-checkbox v-model="checked">Label</el-checkbox>
  <el-checkbox-group v-model="checkboxGroup">
    <el-checkbox label="A">A</el-checkbox>
    <el-checkbox label="B">B</el-checkbox>
  </el-checkbox-group>
  
  <!-- Radio -->
  <el-radio-group v-model="radio">
    <el-radio label="A">A</el-radio>
    <el-radio label="B">B</el-radio>
  </el-radio-group>
  
  <!-- Switch -->
  <el-switch v-model="switchValue"></el-switch>
  
  <!-- Select -->
  <el-select v-model="selectValue">
    <el-option label="Option 1" value="1"></el-option>
    <el-option label="Option 2" value="2"></el-option>
  </el-select>
  
  <!-- Slider -->
  <el-slider v-model="sliderValue"></el-slider>
</template>

<script setup>
import { ref } from 'vue'

const inputValue = ref('')
const checked = ref(true)
const checkboxGroup = ref(['A'])
const radio = ref('A')
const switchValue = ref(false)
const selectValue = ref('')
const sliderValue = ref(50)
</script>
```

    Input: Input,
    Table: Table,

## Best Practices

### 1. Use On-Demand Import

```typescript
// Bad - Full import
import ElementPlus from 'element-plus'

// Good - On-demand import via auto-import plugin
// Components auto-imported when used
```

### 2. Use Composition API

```vue
<script setup>
// Recommended
import { ref, computed, onMounted } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)

onMounted(() => {
  console.log('Mounted')
})
</script>
```

### 3. Properly Handle Forms

```vue
<script setup>
const formRef = ref()
const form = reactive({ ... })
const rules = reactive({ ... })

const handleSubmit = async () => {
  if (!formRef.value) return
  await formRef.value.validate((valid) => {
    if (valid) {
      // Submit
    }
  })
}
</script>
```

### 4. Use Slots for Customization

```vue
<el-table-column>
  <template #default="{ row }">
    <span>{{ row.name }}</span>
  </template>
</el-table-column>
```

### 5. Handle Loading States

```vue
<template>
  <el-button :loading="loading" @click="handleSubmit">
    {{ loading ? 'Loading...' : 'Submit' }}
  </el-button>
  
  <el-table v-loading="tableLoading" :data="tableData">
    <!-- columns -->
  </el-table>
</template>
```

## Accessibility

Element Plus components include ARIA attributes:

```vue
<!-- Automatic -->
<el-button type="primary">Click Me</el-button>

<!-- With aria-label -->
<el-button aria-label="Submit form">Submit</el-button>
```

## Performance Tips

1. **Use on-demand import** - Reduces bundle size
2. **Lazy load dialogs** - For large dialogs
3. **Virtual scrolling** - For large tables
4. **Cache computed values** - Avoid redundant calculations

## Common Issues

### Component Not Registered

```typescript
// Solution: Check auto-import configuration
// vite.config.ts
Components({
  resolvers: [ElementPlusResolver()],
})
```

### Style Conflicts

```css
/* Solution: Use more specific selectors */
.el-button.my-button {
  /* custom styles */
}
```

### Event Not Triggering

```vue
<!-- Solution: Use .native modifier for native events -->
<el-input @keyup.enter.native="handleEnter"></el-input>

<!-- Or use @key event in Vue 3 -->
<el-input @keyup.enter="handleEnter"></el-input>
```
