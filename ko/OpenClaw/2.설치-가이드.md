# OpenClaw 설치 가이드

이 가이드는 다양한 운영 체제에 OpenClaw를 설치하는 단계를 설명합니다.

## 요구사항

- **Node.js**: 버전 22 이상
- **운영 체제**: macOS, Linux 또는 Windows
- **선택**: Docker (샌드박스 기능 사용시)

## 설치 방법

### macOS 및 Linux (권장)

가장 쉬운 방법은 공식 설치 스크립트를 사용하는 것입니다:

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

이 스크립트는 시스템 환경을 자동으로 감지하고 필요한 종속성을 설치한 후 OpenClaw를 설정합니다.

### Windows

PowerShell에서 다음을 실행합니다:

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### npm으로 직접 설치

Node.js가 이미 설정된 경우:

```bash
npm install -g openclaw@latest
```

### 소스에서 설치

최신 개발 버전을 사용하려면:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm link
```

## 설치 후 설정

### 설정 마법사 실행

설치 후 기본 설정을 위해 마법사를 실행하는 것이 좋습니다:

```bash
openclaw onboard --install-daemon
```

마법사 진행 사항:
- API 키 설정
- 서비스 포트 설정
- 채널 로그인 (WhatsApp, Telegram, Discord 등)
- 보안 및 액세스 제어 설정

### 서비스 상태 확인

위와 같이 설치했다면 서비스가 이미 실행 중이어야 합니다:

```bash
openclaw gateway status
```

### 포그라운드에서 실행

终端에서 로그를 직접 확인하려면 (디버깅에 유용):

```bash
openclaw gateway --port 18789
```

### 웹 인터페이스 열기

```bash
openclaw dashboard
```

또는 브라우저에서 직접 열기: `http://127.0.0.1:18789/`

## 환경 변수

데이터 저장 위치를 사용자 정의해야 하는 경우:

- `OPENCLAW_HOME`: OpenClaw의 홈 디렉토리 설정
- `OPENCLAW_STATE_DIR`: 상태 데이터 디렉토리 재정의
- `OPENCLAW_CONFIG_PATH`: 구성 파일 경로 지정

## 설치 확인

설치 후 다음을 수행할 수 있습니다:

1. `openclaw --version` 실행하여 버전 확인
2. `openclaw doctor` 실행하여 시스템 상태 확인
3. 웹 인터페이스를 열어 테스트 메시지 전송

## 일반적인 문제

### Node.js 버전 확인

```bash
node --version
```

버전이 22 이상인지 확인하세요. 낮으면 Node.js를 업그레이드하세요.

### Windows에서 Git을 찾을 수 없음

Git이 설치되어 PATH에 추가되어 있는지 확인하세요. git-scm.com에서 다운로드할 수 있습니다.

### 권한 오류

권한 문제가 발생하면 다음을 시도하세요:

```bash
sudo npm install -g openclaw@latest
```
