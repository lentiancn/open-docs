# Guía de Instalación

Este documento describe las diferentes formas de instalar Apache APISIX para que puedas configurar tu entorno de gateway de API rápidamente.

## Requisitos del Sistema

Antes de instalar APISIX, asegúrate de cumplir con los siguientes requisitos:

- **Sistema operativo**: Linux (se recomienda Ubuntu 20.04+, CentOS 7+, Debian 11+)
- **Docker** (para despliegue con Docker): Tener Docker y Docker Compose instalados
- **etcd**: APISIX usa etcd como almacenamiento de configuración, se requiere etcd 3.4.0+
- **Perl**: Algunas dependencias necesitan Perl 5.5.32+
- **curl**: Para verificar la instalación

## Instalación Rápida con Docker (Recomendada)

Esta es la forma más rápida de instalar, ideal para entornos de desarrollo y pruebas.

### Paso 1: Crear el Directorio Quick Start

```bash
mkdir apisix-quickstart && cd apisix-quickstart
```

### Paso 2: Ejecutar el Script Quick Start

```bash
curl -sL https://run.api7.ai/apisix/quickstart | sh
```

Este comando inici automáticamente dos contenedores: apisix-quickstart y etcd. APISIX usa etcd para guardar y sincronizar la configuración. Los contenedores utilizan el modo de red host de Docker, por lo que puedes acceder directamente desde tu máquina local.

### Paso 3: Verificar la Instalación

Cuando la instalación sea exitosa, verás el siguiente mensaje en la terminal:

```
✔ APISIX is ready!
```

Puedes verificar el estado de APISIX con este comando:

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

Debería devolver algo como:

```
Server: APISIX/Versión
```

Donde Versión es la versión de APISIX instalada, por ejemplo APISIX/3.15.0.

### Paso 4: Acceder al Dashboard

APISIX incluye un Dashboard integrado que puedes acceder en:

```
http://127.0.0.1:9180/ui
```

> ⚠️ Nota: Para ofrecer una mejor experiencia, la API de administración no requiere autenticación por defecto. Por favor, activa la autenticación en entornos de producción.

## Instalación con Docker Compose

Si necesitas personalizar la configuración, puedes usar Docker Compose.

### Paso 1: Crear el Archivo de Configuración

Crea el archivo docker-compose.yaml:

```yaml
version: '3.8'

services:
  apisix:
    image: apache/apisix:${APISIX_VERSION:-3.15.0}
    container_name: apisix
    restart: always
    ports:
      - "9080:9080"
      - "9180:9180"
      - "9443:9443"
    environment:
      - APISIX_ADMIN_API_KEY=tu-clave-admin
    volumes:
      - ./apisix_config/apisix.yml:/usr/local/apisix/conf/apisix.yml
    networks:
      - apisix-network

  etcd:
    image: quay.io/coreos/etcd:v3.5.0
    container_name: etcd
    restart: always
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
    volumes:
      - ./etcd_data:/etcd
    networks:
      - apisix-network

networks:
  apisix-network:
    driver: bridge
```

### Paso 2: Iniciar los Servicios

```bash
docker-compose up -d
```

### Paso 3: Verificar los Servicios

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

## Instalación en Kubernetes

APISIX proporciona un Ingress Controller oficial que puedes desplegar directamente en Kubernetes.

### Instalar con Helm Chart

```bash
# Añadir el repositorio de APISIX Helm
helm repo add apisix https://charts.apiseven.com
helm repo update

# Instalar APISIX
helm install apisix apisix/apisix \
  --set gateway.type=LoadBalancer \
  --set etcd.persistence.size=10Gi
```

### Instalar con kubectl

```bash
kubectl apply -f https://raw.githubusercontent.com/apache/apisix-ingress-controller/master/deploy/manifests/deploy.yaml
```

## Instalación desde Código Fuente

Adecuada para entornos de producción que requieren una compilación personalizada.

### Paso 1: Instalar Dependencias

**Ubuntu/Debian:**

```bash
sudo apt-get update
sudo apt-get install -y perl build-essential git curl
```

**CentOS/RHEL:**

```bash
sudo yum install -y perl gcc git curl
```

### Paso 2: Instalar OpenResty

```bash
# Añadir el repositorio de OpenResty
wget https://openresty.org/package/pubkey.gpg
sudo apt-key add pubkey.gpg
echo "deb http://openresty.org/package/ubuntu $(lsb_release -sc) openresty" | sudo tee /etc/apt/sources.list.d/openresty.list

# Instalar OpenResty
sudo apt-get update
sudo apt-get install -y openresty openresty-opm
```

### Paso 3: Instalar APISIX

```bash
# Clonar el código fuente
git clone https://github.com/apache/apisix.git
cd apisix

# Instalar dependencias
make deps

# Inicializar configuración
make init

# Iniciar APISIX
make run
```

## Verificar la Instalación

Independientemente del método de instalación usado, puedes verificar de las siguientes formas:

1. **Verificar el Estado del Proceso**

```bash
curl "http://127.0.0.1:9180/apisix/admin/notifications" \
  -H "X-API-KEY: tu-clave-admin"
```

2. **Crear una Ruta de Prueba**

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "ruta-prueba",
  "uri": "/test",
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "httpbin.org:80": 1
    }
  }
}'
```

3. **Probar la Ruta**

```bash
curl "http://127.0.0.1:9080/test"
```

## Desinstalar

**Con Docker:**

```bash
# Detener los contenedores
docker-compose down

# Eliminar volúmenes (si es necesario)
docker-compose down -v
```

**Con Kubernetes:**

```bash
helm uninstall apisix
```
