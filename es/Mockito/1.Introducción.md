# Mockito Introduccion

## Que es Mockito

Mockito es un potente framework de mocking para pruebas unitarias de Java, disenado para el Desarrollo Guiado por Pruebas (TDD) y el Desarrollo Guiado por Comportamiento (BDD). Permite a los desarrolladores crear objetos simulados que emulan el comportamiento de objetos reales de manera controlada, logrando pruebas unitarias independientes y confiables.

## Por que elegir Mockito

### API limpia y simple

Mockito ofrece una API intuitiva y facil de usar, con una curva de aprendizaje suave. Los desarrolladores pueden comenzar rapidamente y escribir casos de prueba claros con codigo simplicado. En comparacion con otros frameworks de mocking complejos, la sintaxis de Mockito es mas natural, y leer codigo de prueba es como leer codigo de negocio normal.

### Funciones potentes

Mockito soporta varias caracteristicas avanzadas, incluyendo verificacion de llamadas a metodos, configuracion de valores de retorno, simulacion de excepciones, coincidencia de argumentos, verificacion de orden de llamadas, y mas. Estas capacidades satisfacen various escenarios de prueba complejos y ayudan a los desarrolladores a cubrir completamente la logica del codigo.

### Excelente legibilidad

El codigo de prueba escrito con Mockito tiene una excelente legibilidad. Cuando una prueba falla, Mockito puede proporcionar mensajes de error de verificacion claros, ayudando a los desarrolladores a localizar rapidamente el problema. Esta presentacion de errores amigable mejora enormemente la eficiencia de depuracion.

### Comunidad activa

Mockito es el framework de mocking mas popular en el ecosistema Java, con una gran base de usuarios y una comunidad activa. El framework se mantiene y actualiza continuamente, con documentacion completa y rica. Cuando surgen problemas, es muy facil encontrar soluciones. En Stack Overflow, Mockito ha recibido la mayor cantidad de votos, siendo reconocido como el mejor framework de mocking para Java.

## Conceptos basicos

### Objetos Mock

Un objeto mock es un sustituto del objeto real durante la prueba. Al crear objetos mock, se pueden aislar las dependencias externas, asegurando que las pruebas se enfoquen unicamente en la logica de la unidad bajo prueba, sin verse afectadas por sistemas externos.

### Stubbing

El stubbing se usa para configurar el comportamiento de un objeto mock cuando se llaman metodos especificos. Con la sintaxis when().thenReturn(), se puede controlar con precision las respuestas de los metodos simulados, permitiendo que las pruebas cubran diversos escenarios.

### Verificacion

La verificacion se usa para confirmar que la interaccion entre el codigo bajo prueba y los objetos mock ocurre segun lo esperado. El metodo verify() puede verificar si un metodo fue llamado, cuantas veces fue llamado, con que argumentos, asegurando que el codigo bajo prueba utilize correctamente sus dependencias.

### Spy

Spy permite el mocking parcial de objetos reales. A diferencia de los objetos completamente simulados, los Spy mantienen la mayor parte del comportamiento del objeto real, simulando solo metodos especificos. Esto es util cuando se necesita probar la mayor parte de la logica real mientras se simulan solo algunos metodos.

## Casos de uso tipicos

### Aislamiento de dependencias externas

Cuando el codigo bajo prueba depende de sistemas externos como bases de datos, servicios web o sistemas de archivos, Mockito puede simular estas dependencias externas, permitiendo que las pruebas se ejecuten sin necesidad de sistemas externos reales. Esto no solo acelera la ejecucion de pruebas, sino que juga elimina el riesgo de fallas de prueba causadas por sistemas externos inestables.

### Simulacion de modulos no terminados

En proyectos grandes, diferentes modulos pueden estar en diferentes etapas de desarrollo. Mockito puede simular modulos que aun no estan terminados, permitiendo que los modulos ya completados se prueben con anticipacion, evitando bloqueos en el desarrollo.

### Simulacion de objetos complejos

 Algunos objetos de negocio son muy complejos de construir, con multiples niveles de dependencias. Mockito facilita la creacion de versiones simuladas de estos objetos complejos, enfocandose en probar la logica del negocio en si.

## Integracion con JUnit

Mockito generalmente se usa junto con el framework de pruebas JUnit. Usando @RunWith(MockitoJUnitRunner.class) o @ExtendWith(MockitoExtension.class) de JUnit, los campos anotados con @Mock se inicializan automaticamente, simplificando la escritura del codigo de prueba. Esta combinacion representa la mejor practica para pruebas unitarias en Java.

## Compatibilidad de versiones

Mockito 5.x requiere JDK 11 o superior, soportando caracteristicas nuevas de Java 8 como expresiones lambda. La serie Mockito 4.x todavia soporta JDK 8. Los desarrolladores pueden elegir la version apropiada segun los requisitos del proyecto.

## Resumen

Mockito es una herramienta de prueba esencial para desarrolladores de Java. Resuelve elegantemente el problema del aislamiento de dependencias en pruebas unitarias, permitiendo a los desarrolladores escribir codigo de prueba confiable y mantenible. Dominar Mockito mejorara significativamente la calidad del codigo y la eficiencia del desarrollo, siendo una habilidad que todo ingeniero Java debe poseer.
