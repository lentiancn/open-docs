# OpenClaw よくある質問

このページでは、OpenClawの使用中に最もよくある質問とその回答をまとめています。ここで見つからない場合は、公式ドキュメントを確認するかコミュニティに参加してヘルプを求めてください。

---

## 基本的な質問

### OpenClawとは何ですか？

OpenClawは、WhatsApp、Telegram、Discord、iMessageなどの複数のチャットプラットフォームをAIコーディングアシスタント（Piなど）に接続する自己ホスト型AIアシスタントゲートウェイです。自分で実行でき、データを完全にコントロールできます。

### OpenClawは免费ですか？

OpenClaw自体はオープンソースで免费です（MITライセンス）。ただし、AIモデル（Anthropic ClaudeやOpenAI GPTなど）の使用には料金がかかることがあります。AI APIの使用料だけを支払う必要があります。

### OpenClawを実行するにはどんなデバイスが必要ですか？

最小要件：
- Node.js 22+
- 2GB RAM
- 500MBディスク容量

推奨：
- 4GB+ RAM
- 安定したネットワーク接続
- 常に電源が入っているパソコンまたはサーバー

## インストールに関する質問

### インストールに失敗しました。どうすべきですか？

1. Node.jsバージョンを確認：
   ```bash
   node --version
   ```
   22.0.0以上であることを確認

2. 公式インストールスクリプトを使用：
   ```bash
   curl -fsSL https://openclaw.ai/install.sh | bash
   ```

3.詳細なエラーメッセージを表示：
   ```bash
   npm install -g openclaw@latest --verbose
   ```

4. 診断を実行：
   ```bash
   openclaw doctor
   ```

### Windowsでのインストールに問題があります

**WSL2を強く推奨**します。OpenClawはLinux環境で最も適切に動作します。

WSL2をインストール：
```powershell
wsl --install
```

その後、WSL2にOpenClawをインストールします。

### OpenClawをアップグレードするには？

```bash
npm update -g openclaw@latest
```

または公式更新コマンドを使用：
```bash
openclaw update
```

アップグレード後、診断を実行：
```bash
openclaw doctor
```

## 設定に関する質問

### 設定ファイルはどこにありますか？

デフォルトの場所：`~/.openclaw/openclaw.json`

カスタムパスを指定することも可能：
```bash
openclaw gateway --config /path/to/config.json
```

### 複数のチャルを設定するには？

設定ファイルに複数のチャネル設定を追加：

```json5
{
  channels: {
    whatsapp: { enabled: true },
    telegram: { enabled: true, botToken: "xxx" },
    discord: { enabled: true, token: "xxx" },
  },
}
```

### 設定が反映されません

1. 設定ファイルの形式が正しいことを確認（JSON5）
2. Gatewayが実行中か確認
3. Gatewayを再起動してみてください：
   ```bash
   openclaw gateway restart
   ```

## 接続に関する質問

### Gatewayが起動しません

考えられる理由：
- ポート18789が使用中
- 設定ファイルの形式エラー
- 権限不足

解決方法：
```bash
# ポートの使用状況を確認
lsof -i :18789

# 設定を確認
openclaw doctor

# 管理者として実行
sudo openclaw gateway
```

### チャネルの接続に失敗しました

1. 認証情報が有効か確認
2. チャネルの状態を確認：
   ```bash
   openclaw channels status --probe
   ```
3. ログを確認：
   ```bash
   openclaw logs --follow
   ```

### コントロールUIにアクセスできません

1. Gatewayが実行中か確認
2. URLが正しいか確認：http://127.0.0.1:18789/
3. ファイアウォール設定を確認
4. 再起動を試み：
   ```bash
   openclaw gateway restart
   ```

## 使用に関する質問

### AIがメッセージに返信しません

以下を確認：
1. チャネルが接続されているか
2. ホワイトリストにあるか
3. グループで@メンションが必要か
4. ログのエラーメッセージを確認

### セッションをリセットするには？

`/new`または`/reset`コマンドを使用、またはCLIで：
```bash
openclaw session reset
```

### AIモデルを変更するには？

設定ファイルで設定：

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",
      },
    },
  },
}
```

## セキュリティに関する質問

### OpenClawは安全ですか？

OpenClawはセキュリティを重視：
- すべてのデータはローカルに保存
- 認証情報は暗号化して保存
- ホワイトリストとペアリングメカニズムをサポート
- サンドボックス分離を有効にすることが可能

推奨事項：
- ホワイトリストを使用してアクセスを制御
- 定期的にバージョンを更新
- `openclaw doctor`を実行してセキュリティを確認

### ホワイトリストを設定するには？

```json5
{
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123", "+15555550124"],
    },
  },
}
```

### ペアリング（Pairing）とは？

ペアリングはセキュリティメカニズムで、不明なユーザーが初めてメッセージを送信的时候会阻塞你需要批准后才能使用AI。

ユーザーを承認：
```bash
openclaw pairing approve --channel whatsapp --sender +15555550123
```

## 機能に関する質問

### 音声機能を使用するには？

1. Talk Modeを設定：
   ```json5
   {
     talk: {
       enabled: true,
       voice: "elevenlabs",
     },
   }
   ```

2. サポートされているデバイスでVoice Wakeを有効化

### スケジュールタスクが実行されません

1. Cronが有効か確認：
   ```bash
   openclaw cron status
   ```

2. タスクリストを表示：
   ```bash
   openclaw cron list
   ```

3. 実行ログを表示：
   ```bash
   openclaw cron runs --id <jobId>
   ```

### モバイルノードが接続できません

1. 電話とGatewayが同じネットワーク上にあるか確認
2. ノードの状態を確認：
   ```bash
   openclaw nodes status
   ```

3. ノードを再ペアリング：
   ```bash
   openclaw nodes pair
   ```

 ## パフォーマンスに関する質問

### 動作が遅いです

1. システムリソースを確認：
   ```bash
   openclaw status
   ```

2. セッションを圧縮：
   ```bash
   /compact
   ```

3. 古いログをクリーンアップ：
   ```bash
   openclaw logs clean
   ```

### リソース使用を制限するには？

```json5
{
  agents: {
    defaults: {
      maxTokens: 100000,
    },
  },
  session: {
    maxAgeHours: 24,
  },
}
```
