# Guía de Instalación de JIRA

JIRA es una herramienta de seguimiento de problemas y gestión de proyectos desarrollada por Atlassian. Esta guía cubre los métodos de instalación en diversos entornos.

## Información de Versión

| Versión | Estado | Descripción |
|---------|--------|-------------|
| JIRA 9.x | ✅ Actual | Última versión |
| JIRA 8.x | ✅ Soportado | Versión estable |

## Requisitos del Sistema

### Requisitos de Hardware

| Requisito | Mínimo | Recomendado |
|-----------|--------|-------------|
| CPU | 4 núcleos | 8+ núcleos |
| Memoria | 4 GB | 16+ GB |
| Disco | 10 GB | 50+ GB |

### Requisitos de Software

| Software | Versión |
|----------|---------|
| Java | JDK 11 o 17 |
| Sistema Operativo | Linux/Windows/macOS |
| Base de Datos | PostgreSQL/MySQL/Oracle |

## Métodos de Instalación

### Método 1: Instalación en Windows

#### 1. Descargar el Instalador

1. Visite https://www.atlassian.com/software/jira/download
2. Descargue JIRA Software
3. Ejecute el instalador

#### 2. Ejecutar la Instalación

Complete la instalación siguiendo el asistente:
- Seleccione el tipo de instalación
- Configure el puerto
- Configure la base de datos
- Cree la cuenta de administrador

### Método 2: Instalación en Linux

#### 1. Descargar el Paquete de Instalación

```bash
# Descargar JIRA
wget https://product-downloads.atlassian.com/software/jira/downloads/atlassian-jira-software-X.X.X-x64.bin
```

#### 2. Instalar

```bash
# Añadir permisos de ejecución
chmod +x atlassian-jira-software-X.X.X-x64.bin

# Ejecutar la instalación
sudo ./atlassian-jira-software-X.X.X-x64.bin
```

#### 3. Configurar

```bash
# Iniciar JIRA
sudo /opt/atlassian/jira/bin/start-jira.sh

# Detener JIRA
sudo /opt/atlassian/jira/bin/stop-jira.sh
```

### Método 3: Instalación con Docker

```bash
# Extraer la imagen
docker pull atlassian/jira-software

# Ejecutar el contenedor
docker run -v jira-data:/var/atlassian/application-data/jira --name="jira" -d -p 8080:8080 atlassian/jira-software
```

### Método 4: Usar AWS

1. Busque JIRA en AWS Marketplace
2. Seleccione el tipo de instancia
3. Configure la base de datos
4. Inicie la instancia

## Configuración

### Configuración de Base de Datos

```properties
# dbconfig.xml
<?xml version="1.0" encoding="UTF-8"?>
<jira-database-config>
  <name>defaultDS</name>
  <connection-type>default</connection-type>
  <url>jdbc:postgresql://localhost:5432/jira</url>
  <username>jira</username>
  <password>password</password>
  <pool-size>20</pool-size>
</jira-database-config>
```

### Configuración de Proxy Inverso

#### Nginx

```nginx
server {
    listen 80;
    server_name jira.example.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Preguntas Frecuentes

### ¿No puede iniciar?

Verifique la versión de Java:

```bash
java -version
```

### ¿Error de conexión a la base de datos?

Verifique la configuración de la base de datos y la información de conexión.

### ¿Memoria insuficiente?

Modifique la configuración de memoria:

```bash
# setenv.sh
export JVM_MINIMUM_MEMORY="2048m"
export JVM_MAXIMUM_MEMORY="4096m"
```
