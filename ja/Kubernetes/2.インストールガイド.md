# Kubernetes インストールガイド

## 準備

Kubernetesをインストールする前に、基本要件を理解してください：

### システム要件
- LinuxはKubernetesコントロールプレーンを実行するための推奨OSです
- クラスター内では、LinuxまたはWindowsを含むその他のオペレーティングシステムでアプリケーションを実行できます
- コンテナランタイムをサポートするOSが必要です

### インストールタイプの選択
Kubernetesをインストールタイプは次の条件に基づいて選択してください：
- メンテナンスの容易性
- セキュリティ
- コントロール
- 利用可能なリソース
- クラスターを操作・管理するために必要な専門知識

## インストール方法

### 学習環境（初心者向け）

Kubernetesを学習している場合コミュニティツールまたはエコシステムツールを使用してローカルマシングループを設定します：

#### Minikube
Minikubeはローカルマシンにシングルノードクラスターを作成できる軽量Kubernetes実装です。学習や小規模テストに最適です。

インストール手順：
1. Minikubeをインストール
2. kubectl（Kubernetesコマンドラインツール）をインストール
3. Minikubeクラスターを起動

#### Kind
Kind（Kubernetes in Docker）は、Dockerコンテナを使用してノードをシミュレートし、ローカルKubernetesクラスターを作成するためのツールです。

### 本番環境

本番環境では次のオプションを検討してください：

#### kubeadm
Kubernetesをデプロイするための公式サポートツールです。ベストプラクティスに従って安全なクラスターを作成できます。

基本的な手順：
1. サーバーの準備（OSインストール、ネットワーク設定）
2. コンテナランタイムのインストール（containerdまたはDocker）
3. kubeadm、kubelet、kubectlのインストール
4. kubeadmを使用してクラスターを初期化
5. ネットワークプラグインの設定（Calico、Ciliumなど）
6. ワーカーノードの参加

#### マネジメントサービス
Kubernetesクラスターを管理したくない場合は、マネジメントサービスの利用を検討してください：
- Amazon EKS（Elastic Kubernetes Service）
- Google GKE（Google Kubernetes Engine）
- Azure AKS（Azure Kubernetes Service）
- アリババクラウドACK
- テンセントクラウドTKE

#### その他のソリューション
クラウドやベアメタル環境には、標準化およびカスタムソリューションもあります。

## ツールのインストール

### kubectlのインストール

kubectlはKubernetesクラスターと対話するためのコマンドラインツールです。

Linuxへのインストール：
```bash
# 最新バージョンをダウンロード
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 実行権限を追加
chmod +x kubectl

# システムパスに移動
sudo mv kubectl /usr/local/bin/
```

### kubeadmのインストール

```bash
# aptパッケージインデックスを更新
sudo apt-get update

# 依存関係をインストール
sudo apt-get install -y apt-transport-https ca-certificates curl

# Kubernetes APTリポジトリを追加
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

# 更新してインストール
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```

## クラスターの初期化

### kubeadmでクラスターを作成

1. コントロールプレーンノードを初期化：
```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```

2. kubectlを設定：
```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

3. ネットワークプラグインをインストール（Flannelの例）：
```bash
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
```

4. ワーカーノードに参加：
```bash
# 各ワーカーノードで初期化出力のコマンドを実行
kubeadm join <master-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## インストールの確認

クラスター状態を確認：
```bash
# ノードを表示
kubectl get nodes

# Podを表示
kubectl get pods --all-namespaces
```

すべてのコンポーネントがRunning状態であれば、インストール成功です。

## アンインストール

Kubernetesをアンインストールする必要がある場合：

```bash
# すべてのKubernetesサービスを停止
sudo kubeadm reset

# 設定ファイルを削除
sudo rm -rf ~/.kube /etc/kubernetes /var/lib/etcd

# パッケージをアンインストール
sudo apt-get purge kubeadm kubectl kubelet kubernetes-cni etcd
```

## 注意事项

1. コンテナランタイムの選択：新クラスターに適切なコンテナランタイムを選択してください
2. リソース計画：ワークロードに基づいてCPUとメモリのリソースを適切に割り当て
3. ネットワーク計画：PodとServiceのネットワークCIDRを事前に計画
4. セキュリティ強化：本番環境では認証、認可、ネットワークポリシーを設定
5. 高可用性：本番環境ではマルチコントロールプレーンノードクラスターの設定を推奨
