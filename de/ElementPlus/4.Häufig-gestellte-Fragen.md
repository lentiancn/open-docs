# Element Plus FAQ

Diese Seite listet häufige Probleme und Lösungen bei der Verwendung von Element Plus auf.

## Installationsprobleme

### Installation fehlgeschlagen

Symptome: npm install oder yarn add schlägt fehl

Lösungen:

1. Überprüfen Sie die Node.js-Version, sie sollte 14.0.0 oder höher sein
2. npm-Cache leeren:
   npm cache clean --force
3. node_modules und package-lock.json löschen, dann neu installieren

### TypeScript-Fehler

Symptome: Typfehler beim Importieren von Komponenten

Lösungen:

1. Stellen Sie sicher, dass das Volar-Plugin installiert ist
2. In tsconfig.json hinzufügen:
   {
     "compilerOptions": {
       "types": ["element-plus/global"]
     }
   }

## Nutzungsprobleme

### Komponentenstile werden nicht angewendet

Symptome: Komponentenstile entsprechen nicht den Erwartungen

Lösungen:

1. Stellen Sie sicher, dass CSS korrekt importiert ist:
   import 'element-plus/dist/index.css'
2. Überprüfen Sie, ob globale Stile überschrieben werden

### Symbole werden nicht angezeigt

Symptome: Symbolkomponenten werden leer oder als Quadrate angezeigt

Lösungen:

1. Symbolbibliothek installieren:
   npm install @element-plus/icons-vue

2. Symbole global registrieren:
   import * as icons from '@element-plus/icons-vue'
   for (const [key, component] of Object.entries(icons)) {
     app.component(key, component)
   }

### Import bei Bedarf funktioniert nicht

Symptome: Bundle-Größe bleibt groß

Lösungen:

1. Überprüfen Sie, ob die Plugin-Konfiguration korrekt ist
2. Stellen Sie sicher, dass der richtige Resolver verwendet wird

### Internationalisierung funktioniert nicht

Symptome: Komponententext ändert sich nach Sprachwechsel nicht

Lösungen:

1. Sprachpaket korrekt importieren:
   import de from 'element-plus/es/locale/lang/de'
2. Locale konfigurieren:
   app.use(ElementPlus, { locale: de })

## Komponentenprobleme

### Tabellenprobleme

#### Tabellensortierung funktioniert nicht

Stellen Sie sicher, dass das prop-Attribut gesetzt ist:

<el-table-column prop="date" sortable />

#### Tabellenspalten falsch ausgerichtet

Stellen Sie sicher, dass die Tabelle eine feste Höhe hat:

<el-table :data="data" height="300" />

### Formularprobleme

#### Formularvalidierung wird nicht ausgelöst

Stellen Sie sicher, dass el-form-item das prop-Attribut hat:

<el-form-item label="Name" prop="name">
  <el-input v-model="ruleForm.name" />
</el-form-item>

### Dialogprobleme

#### Dialog-Scroll-Probleme

Wenn der Dialoginhalt zu lang ist, stellen Sie sicher, dass Scrollen hinzugefügt wird:

<el-dialog>
  <div style="max-height: 60vh; overflow-y: auto;">
    Inhalt
  </div>
</el-dialog>

## Stilprobleme

### Stilkonflikte

Symptome: Benutzerdefinierte Stile werden überschrieben

Lösungen:

1. Verwenden Sie spezifischere Selektoren
2. Verwenden Sie ::v-deep oder :deep()

### Theme-Farbe wird nicht angewendet

Überprüfen Sie, ob CSS-Variablen korrekt überschrieben werden:

:root {
  --el-color-primary: #409eff;
}

Stellen Sie sicher, dass es nach den-plus-Stilen geladen wird.

## Leistungspro elementbleme

### Seitenverzögerung

Bei großen Datenmengen:

1. Verwenden Sie el-table-v2 (virtuelle Tabelle)
2. Verwenden Sie Paginierung zum Laden von Daten
3. Vermeiden Sie komplexe Komponenten in Tabellen

### Bundle-Größe zu groß

1. Verwenden Sie Import bei Bedarf
2. Entfernen Sie ungenutzte Plugins

## Vue-Versionsprobleme

### Nicht kompatibel mit Vue 2

Element Plus unterstützt nur Vue 3. Wenn Sie Vue 2 benötigen, verwenden Sie Element UI.

### Composition API nicht unterstützt

Stellen Sie sicher, dass Sie Vue 3.2 oder höher für bessere Composition API-Unterstützung verwenden.
