# Element Plus Guide d'installation

> Element Plus est une bibliothèque de composants UI de bureau Vue 3 développée par l'équipe Element
> 
> Date de mise à jour : Février 2026
> 
> Version actuelle : 2.x (Stable)

---

## 1. Présentation du projet

Element Plus est une bibliothèque de composants UI de bureau basée sur Vue 3, développée et maintenue par l'équipe Element. En tant que version upgrade de Element UI pour Vue 3, Element Plus est entièrement écrit dans le style Composition API de Vue 3 et complètement réécrit en TypeScript, offrant une expérience de développement de composants avec sécurité des types.

La première version stable d'Element Plus a été publiée le 7 février 2022. Après des années d'itération, l'API est très stable. Actuellement, elle supporte le chinois simplifié, l'anglais et d'autres langues, et dispose d'un écosystème communautaire actif.

## 2. Exigences environnement

### 2.1 Exigences environnement de développement

| Environnement | Version minimum | Version recommandée |
|------|----------|----------|
| Node.js | 16.x | 18.x ou 20.x |
| npm | 8.x | 9.x ou 10.x |
| Vue | 3.2.0 | 3.4.x |
| Vite | 4.x | 5.x |

### 2.2 Compatibilité navigateurs

Element Plus supporte tous les navigateurs modernes, y compris :

- Chrome >= 80
- Firefox >= 78
- Safari >= 14
- Edge >= 80

Internet Explorer 11 n'est pas supporté.

### 2.3 Exigences pile technologique

| Technologie | Description |
|------|------|
| Vue 3 | Doit utiliser Vue 3.2.0 ou supérieur |
| TypeScript | Version 4.5 ou supérieure recommandée |
| Outil de build | Vite (recommandé) ou Webpack |

## 3. Étapes d'installation

### 3.1 Utiliser npm (Recommandé)

npm est le gestionnaire de packages de Node.js et c'est la méthode officiellement recommandée pour installer Element Plus.

```bash
# Accédez au répertoire de votre projet Vue 3
cd votre-projet-vue

# Installer Element Plus
npm install element-plus
```

### 3.2 Utiliser yarn

```bash
yarn add element-plus
```

### 3.3 Utiliser pnpm

```bash
pnpm add element-plus
```

## 4. Importer dans le projet Vue 3

### 4.1 Import complet (Adapté aux petits projets)

L'import complet inclura tous les composants, la configuration est simple mais la taille du bundle est plus grande.

```typescript
// main.ts ou main.js
import { createApp } from 'vue'
import App from './App.vue'

// Importer Element Plus et ses styles
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// Utiliser Element Plus
app.use(ElementPlus)

app.mount('#app')
```

### 4.2 Import à la demande (Recommandé pour la production)

L'import à la demande peut réduire considérablement la taille du bundle en n'important que les composants réellement utilisés.

#### 4.2.1 Installer le plugin d'auto-import

D'abord, installez unplugin-vue-components et unplugin-auto-import :

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

#### 4.2.2 Configurer Vite

Ajoutez la configuration dans vite.config.ts :

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

Avec cette configuration, les composants peuvent être importés automatiquement sans importation manuelle. Par exemple :

```vue
<template>
  <el-button type="primary">Bonjour</el-button>
</template>
```

## 5. Importer via CDN (Sans outil de build)

Si vous ne voulez pas utiliser npm ou les outils de build, vous pouvez importer Element Plus via CDN.

### 5.1 Importer via unpkg CDN

```html
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Exemple Element Plus CDN</title>
  <!-- Importer Vue 3 -->
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <!-- Importer Element Plus -->
  <link rel="stylesheet" href="https://unpkg.com/element-plus/dist/index.css">
  <script src="https://unpkg.com/element-plus"></script>
</head>
<body>
  <div id="app">
    <el-button type="primary">Bonjour Element Plus</el-button>
  </div>

  <script>
    const { createApp, ref } = Vue
    
    const App = {
      setup() {
        const visible = ref(false)
        return { visible }
      },
      template: `
        <el-button @click="visible = true">Ouvrir le dialogue</el-button>
        <el-dialog v-model="visible" title="Bonjour">Ceci est Element Plus</el-dialog>
      `
    }
    
    const app = createApp(App)
    app.use(ElementPlus)
    app.mount('#app')
  </script>
</body>
</html>
```

## 6. Configuration internationalisation

Element Plus utilise l'anglais par défaut. Pour utiliser français, vous devez configurer l'internationalisation.

### 6.1 Configuration française globale

```typescript
// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import fr from 'element-plus/es/locale/lang/fr'
import 'element-plus/dist/index.css'

const app = createApp(App)

app.use(ElementPlus, {
  locale: fr,
})

app.mount('#app')
```

## 7. Personnalisation du thème

### 7.1 Variables SCSS

Element Plus est écrit en SCSS, vous pouvez personnaliser le thème en écrasant les variables.

```scss
// styles/element-variables.scss
// Couleur primaire
$--color-primary: #409EFF;

// Écrraser les variables des composants
@use "element-plus/theme-chalk/src/common/var.scss" as *;
```

### 7.2 Mode sombre

Element Plus a un support intégré pour le mode sombre :

```typescript
# Activer le mode sombre
import 'element-plus/theme-chalk/dark/css-vars.css'
```

Ajoutez la classe à la balise HTML :

```html
<html class="dark">
  <!-- Votre application -->
</html>
```

## 8. Problèmes courants

### 8.1 Échec d'installation

**Problème** : npm install échoue

**Solutions** :

1. Vider le cache npm : `npm cache clean --force`
2. Vérifier le réseau, vous pouvez utiliser des miroirs nationaux :

```bash
# Configurer le miroir npm
npm config set registry https://registry.npmmirror.com
```

### 8.2 Styles non appliqués

**Problème** : Les styles des composants s'affichent anormalement

**Solutions** :

1. Assurez-vous d'avoir importé le fichier de styles : `import 'element-plus/dist/index.css'`
2. Vérifier si la configuration de build traite correctement le CSS
3. Vider le cache du navigateur et reconstruire
