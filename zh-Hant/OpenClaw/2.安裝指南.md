# OpenClaw 安裝指南

OpenClaw 支援多種安裝方式，您可以根據自己的需求選擇最合適的方法。

## 環境要求

- **Node.js**：22.x 或更高版本
- **作業系統**：macOS、Linux、Windows（透過 WSL2）
- **可選**：Tailscale（用於遠端存取）

## 快速安裝

### macOS / Linux

使用安裝腳本是最簡單的方式：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

或者使用 npm/pnpm：

```bash
npm install -g openclaw@latest
# 或
pnpm add -g openclaw@latest
```

### Windows (PowerShell)

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

## 初始化配置

安裝完成後，執行引導向導進行初始配置：

```bash
openclaw onboard --install-daemon
```

向導會引導您完成以下設定：

1. **驗證配置**：設定 API 密鑰或 OAuth
2. **Gateway 設定**：設定閘道連接埠和服務
3. **渠道連接**：連接您想要使用的通訊平台
4. **技能安裝**：選擇和安裝需要的技能

## 安裝為系統服務

使用 `--install-daemon` 參數可以將 OpenClaw 安裝為背景服務：

```bash
openclaw onboard --install-daemon
```

這會在您的系統上建立一個使用者級服務（launchd 或 systemd），確保 OpenClaw 始終運行。

## 手動執行 Gateway

如果您不想安裝服務，也可以手動啟動 Gateway：

```bash
openclaw gateway --port 18789 --verbose
```

## Docker 安裝

如果您更傾向於使用 Docker，可以參考官方 Docker 部署檔案。

## 從原始碼建構

如果您想參與開發或自訂 OpenClaw：

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
pnpm openclaw onboard --install-daemon
```

注意：pnpm 用於建構。TypeScript 可以直接透過 tsx 執行。

## 驗證安裝

安裝完成後，您可以：

1. **檢查 Gateway 狀態**：
   ```bash
   openclaw gateway status
   ```

2. **開啟 Control UI**：
   ```bash
   openclaw dashboard
   ```
   或在瀏覽器中存取 `http://127.0.0.1:18789/`

3. **發送測試訊息**（需要先配置渠道）：
   ```bash
   openclaw message send --target +15555550123 --message "Hello from OpenClaw"
   ```

## 更新 OpenClaw

OpenClaw 有三個發布渠道：

- **stable**：正式版發布（推薦）
- **beta**：測試版
- **dev**：開發版

切換渠道：

```bash
openclaw update --channel stable|beta|dev
```

更新後建議執行：

```bash
openclaw doctor
```

## 常用命令

| 命令 | 說明 |
|------|------|
| `openclaw onboard` | 執行初始設定向導 |
| `openclaw gateway start` | 啟動 Gateway |
| `openclaw gateway status` | 查看 Gateway 狀態 |
| `openclaw dashboard` | 開啟 Control UI |
| `openclaw agent --message "..."` | 直接發送訊息給助手 |
| `openclaw message send --to xxx --message "xxx"` | 發送訊息 |
| `openclaw doctor` | 診斷和修復問題 |
| `openclaw update` | 更新 OpenClaw |

## 環境變數

如果需要自訂配置位置：

- `OPENCLAW_HOME`：設定主目錄
- `OPENCLAW_STATE_DIR`：覆寫狀態目錄
- `OPENCLAW_CONFIG_PATH`：覆寫配置檔案路徑

## 常見問題

**安裝失敗怎麼辦？**

- 確認 Node.js 版本 >= 22
- 檢查網路連線
- 執行 `openclaw doctor` 進行診斷

**如何解除安裝？**

- 停止 Gateway 服務
- 刪除安裝目錄
- 清理配置檔案
