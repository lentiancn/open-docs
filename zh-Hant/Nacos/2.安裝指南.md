# Nacos 安裝指南

## 環境要求

Nacos 依賴 Java 環境來運行，請確保在以下版本環境中安裝使用：

- 64 位元作業系統，支援 Linux/Unix/Mac/Windows，推薦選用 Linux/Unix/Mac
- 64 位元 JDK 17+（推薦使用 JDK 17 或更高版本）
- JDK 下載位址：https://www.oracle.com/java/technologies/downloads/#java17
- JDK 環境配置說明：https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## 推薦配置

Nacos 的運行建議至少在以下機器配置下運行：

- CPU：2 核心
- 記憶體：4 GB
- 磁碟：60 GB

## 安裝方式

Nacos 支援多種安裝方式，您可以根據實際需求選擇適合的方式。

### 方式一：使用 nacos-setup 一鍵部署（推薦）

nacos-setup 提供一鍵自動化部署，自動處理下載、鑒權配置、埠檢測和 Java 環境驗證等操作。

#### 安裝 nacos-setup

Linux/macOS：

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows（PowerShell）：

```powershell
# 使用 PowerShell 執行
```

#### 啟動 Nacos 服務

本地一鍵部署一個最新版本的單機 Nacos：

```bash
nacos-setup
```

本地一鍵部署指定版本的單機 Nacos：

```bash
nacos-setup -v 3.1.1
```

本地一鍵部署一個最新版本的叢集 Nacos：

```bash
nacos-setup -c cluster
```

全域配置使用外置存儲，輸入資料庫位址埠帳號密碼，配置完成後重新運行 nacos-setup：

```bash
nacos-setup --datasource-conf
```

更多用法請參考 nacos-setup 使用手冊。

### 方式二：手動下載發行包部署

#### 下載安裝包

您可以透過以下兩種方式取得 Nacos 發行包：

**方式一：透過 Nacos 官網下載**

進入 Nacos 官網版本下載頁面，選擇穩定版本，然後點擊二進位包下載列中的 zip 檔案進行下載。

**方式二：透過 GitHub 下載**

進入 Nacos GitHub Releases 頁面，下載對應版本的發行包。

注意：有時大量使用者同時進行下載時，可能會遇到下載限流失敗的情況，若出現下載限流失敗，請稍等後重試，或採用從 GitHub 下載方式。

#### 解壓縮 Nacos 發行包

Linux/Unix/Mac：

```bash
unzip nacos-server-$version.zip
# 或者
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows：

直接使用解壓工具解壓 nacos-server-$version.zip

#### 啟動伺服器

Linux/Unix/Mac 啟動命令（standalone 代表著單機模式運行，非叢集模式）：

```bash
sh startup.sh -m standalone
```

如果您使用的是 Ubuntu 系統，或者運行腳本报错提示符號找不到，可嘗試如下運行：

```bash
bash startup.sh -m standalone
```

啟動程式會提示您輸入三個鑒權相關配置（Nacos 從 3.0.0 版本開始預設啟用控制台鑒權功能，因此如下三個鑒權相關配置必須填寫）：

```
nacos.core.auth.plugin.nacos.token.secret.key is missing, please set with Base64 string: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key is missing, please set: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value is missing, please set: ${your_input_server_identity_key}
```

Windows 啟動命令：

```bash
cmd startup.cmd
```

或者雙擊 startup.cmd 運行檔案。

### 方式三：使用 Docker 安裝

#### 拉取 Nacos 映像檔

```bash
docker pull nacos/nacos-server
```

#### 運行單機模式

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### 運行叢集模式

需要配合 Docker Compose 或 Kubernetes 進行叢集部署。

### 方式四：使用 Kubernetes 安裝

Nacos 支援在 Kubernetes 環境中部署，可以使用 Helm Chart 或 Operator 進行安裝。具體步驟請參考 Nacos Kubernetes 部署文檔。

## 驗證 Nacos 服務是否啟動成功

進入 Nacos 日誌目錄下，使用 tail -f start.out 查看日誌，如果看到如下日誌，說明服務啟動成功：

```
Nacos started successfully in stand alone mode. use embedded storage
```

## 快速驗證功能

### Nacos 控制台頁面

打開任意瀏覽器，輸入位址：http://127.0.0.1:8080/index.html，即可進入 Nacos 控制台頁面。

注意：首次打開會要求初始化管理員使用者 nacos 的密碼。預設使用者名稱/密碼為 nacos/nacos@demo。

### 服務註冊

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### 服務發現

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### 發布配置

首先登入獲取 access token：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

使用 access token 創建配置：

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### 獲取配置

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## 關閉伺服器

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

或者雙擊 shutdown.cmd 運行檔案。

## 版本選擇

生產環境目前推薦的穩定版本為 3.1.1。您可以透過以下途徑瞭解每個版本支援的功能：

- Nacos 版本下載頁面
- Nacos Release Notes
- Nacos 發布聲明

## 注意事項

1. Nacos 3.0.0 版本開始預設啟用控制台鑒權功能，首次啟動需要配置鑒權相關參數
2. 叢集模式部署需要使用外置資料庫（如 MySQL）進行資料存儲
3. 生產環境建議使用叢集模式以保證高可用性
4. 確保防火牆開放必要埠（預設 8848、9848）
5. 建議使用 JDK 17 或更高版本以獲得最佳效能
