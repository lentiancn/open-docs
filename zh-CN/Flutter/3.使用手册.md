# Flutter 使用手册

## 基础概念

### Widget 介绍

Widget 是 Flutter UI 的基本构建块。在 Flutter 中，所有东西都是 Widget，包括布局、padding、align 等都是 Widget。Widget 描述了他们的视图在给定当前配置和状态时应该是什么样子。

Flutter 的 Widget 是 immutable（不可变的），当需要改变 UI 时，需要重新创建 Widget 或者使用 StatefulWidget 和状态管理。Widget 可以嵌套形成 Widget 树，Flutter 会高效地比较新旧 Widget 树，只渲染实际变化的部分。

常见的 Widget 包括：Text（文本）、Image（图片）、Icon（图标）、Container（容器）、Row/Column（水平/垂直布局）、Stack（层叠布局）、ListView（列表）等。

### StatelessWidget 和 StatefulWidget

StatelessWidget 是无状态的 Widget，适合用于静态的 UI 展示，如文本、图片等。定义一个 StatelessWidget 只需要实现 build 方法，返回需要渲染的 Widget 树。

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget 是有状态的 Widget，适合用于需要动态更新 UI 的场景，如计数器、表单等。StatefulWidget 需要配合 State 使用，状态变化时调用 setState 方法通知 Flutter 重新渲染。

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## 布局基础

### 线性布局

Row 和 Column 是最常用的线性布局 Widget。Row 用于水平排列子 Widget，Column 用于垂直排列子 Widget。可以使用 MainAxisAlignment 和 CrossAxisAlignment 控制对齐方式。

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### 层叠布局

Stack 用于创建层叠布局，可以在另一个 Widget 上面放置 Widget。Positioned 用于定位 Stack 中的子 Widget。

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### 列表布局

ListView 是最常用的列表 Widget，适用于长列表。ListView.builder 适合大量数据的列表，可以实现懒加载。ListView.separated 可以添加分割线。

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## 状态管理

### setState

setState 是最基本的状态管理方式，适用于简单的局部状态。在 StatefulWidget 中，调用 setState 会触发 Widget 重新构建。

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider 是 Flutter 官方推荐的状态管理方案之一。通过 Provider，可以在 Widget 树中共享状态，避免层层传递。

```dart
// 定义 ChangeNotifier
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// 使用 Provider
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

### 其他状态管理方案

对于复杂应用，还有 Riverpod、X、BLoC 等状态Get管理方案。Riverpod 是 Provider 的升级版，提供了更好的性能和类型安全。GetX 集成了状态管理、路由管理和依赖注入。BLoC 使用事件流的方式管理状态，适合大型复杂应用。

## 网络请求

### HTTP 包

使用 http 包可以发起简单的 HTTP 请求：

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

### Dio

Dio 是一个强大的 HTTP 客户端，支持拦截器、请求取消、文件上传等功能：

```dart
import 'package:dio/dio.dart';

final dio = Dio();
final response = await dio.get('https://api.example.com/data');
```

## 本地存储

### SharedPreferences

用于存储简单的键值对数据：

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

### SQLite

对于结构化数据，可以使用 sqflite 包：

```dart
import 'package:sqflite/sqflite.dart';

final db = await openDatabase('my.db');
await db.insert('users', {'name': 'Flutter'});
```

## 导航和路由

Flutter 使用 Navigator 进行页面导航。基本操作包括 Navigator.push（推入新页面）、Navigator.pop（返回）、Navigator.pushReplacement（替换当前页面）等。

命名路由是推荐的路由管理方式：

```dart
// 定义路由
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// 导航
Navigator.pushNamed(context, '/detail');
```

## 打包和发布

### Android 发布

使用 keytool 生成签名密钥：

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

在 android/app/build.gradle 中配置签名，然后在项目根目录运行：

```bash
flutter build apk --release
```

### iOS 发布

在 Xcode 中配置签名证书和描述文件，然后运行：

```bash
flutter build ios --release
```

需要注意的是，iOS 发布需要 Apple 开发者账号和相应的证书。
