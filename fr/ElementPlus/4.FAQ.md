# Element Plus FAQ

Questions et réponses fréquentes.

## Problèmes d'installation

### Q: L'installation échoue?

**Solutions:**
1. Vérifier la version de Node.js
```bash
node --version  # Nécessite 18.0+
```
2. Vider le cache npm
```bash
npm cache clean --force
```
3. Supprimer node_modules et package-lock.json, réinstaller
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: Module introuvable?

**Solutions:**
1. Confirmer que le package est installé
```bash
npm list element-plus
```
2. Vérifier la configuration npm/yarn/pnpm
3. Essayer de supprimer le fichier lock et réinstaller

### Q: Erreurs de type TypeScript?

**Solutions:**
1. Installer les définitions de types
```bash
npm install -D @types/node
```
2. Ajouter dans tsconfig.json
```json
{
  "compilerOptions": {
    "types": ["element-plus/global"]
  }
}
```

## Problèmes d'utilisation

### Q: Le composant ne s'affiche pas?

**Solutions:**
1. Confirmer l'import correct
2. Vérifier les messages d'erreur de la console
3. Confirmer la version de Vue (3.2+ requise)

### Q: Les styles ne s'appliquent pas?

**Solutions:**
1. Confirmer que le CSS est importé
```typescript
import 'element-plus/dist/index.css'
```
2. Vérifier la priorité CSS, utiliser `:deep()` ou `!important`
3. Vérifier les surcharges de styles par d'autres règles

### Q: Les icônes ne s'affichent pas?

**Solutions:**
1. Installer le package d'icônes
```bash
npm install @element-plus/icons-vue
```
2. Enregistrer globalement
```typescript
import * as icons from '@element-plus/icons-vue'
for (const [key, component] of Object.entries(icons)) {
  app.component(key, component)
}
```
3. Ou importer à la demande

### Q: Le tableau ne s'affiche pas?

**Solutions:**
1. Confirmer que la propriété data est correctement liée
2. Vérifier que les props sont corrects
3. Confirmer que le tableau est à l'extérieur de el-table-column

### Q: La validation du formulaire ne fonctionne pas?

**Solutions:**
1. S'assurer que la prop correspond à v-model
2. S'assurer qu'il y a une référence ref
3. Appeler correctement la méthode validate
```typescript
const formRef = ref()
await formRef.value.validate()
```

### Q: Le dialogue ne s'affiche pas?

**Solutions:**
1. Vérifier la liaison v-model
2. Confirmer que visible est true
3. Vérifier les conflits de z-index

## Problèmes de thème

### Q: Comment modifier la couleur du thème?

**Solutions:**
Utiliser les variables CSS:
```css
:root {
  --el-color-primary: #409eff;
}
```

### Q: Comment utiliser le thème sombre?

**Solutions:**
Importer le CSS sombre:
```typescript
import 'element-plus/theme-chalk/dark/css-vars.css'
```

### Q: Comment surcharger les styles des composants?

**Solutions:**
Utiliser les sélecteurs CSS:
```css
.el-button--primary {
  background-color: #votre-couleur;
}
```

## Problèmes d'internationalisation

### Q: Comment changer de langue?

**Solutions:**
```typescript
import fr from 'element-plus/es/locale/lang/fr'
import en from 'element-plus/es/locale/lang/en'

app.use(ElementPlus, { locale: fr })
```

### Q: Personnaliser les textes?

**Solutions:**
```typescript
import {ElMessage} from 'element-plus'
ElMessage('Message personnalisé')
```

## Problèmes de performance

### Q: Le chargement initial est lent?

**Solutions:**
1. Utiliser l'import à la demande
2. Utiliser le plugin d'import automatique
3. Configurer le pré-build Vite

### Q: La taille du bundle est trop grande?

**Solutions:**
1. S'assurer que l'import à la demande est utilisé
2. Utiliser le tree shaking
3. Vérifier si le package complet est importé

## Problèmes de compatibilité

### Q: Le navigateur IE n'est pas supporté?

Element Plus ne supporte pas le navigateur IE. Utiliser:
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### Q: Conflits avec d'autres bibliothèques?

**Solutions:**
1. Vérifier les niveaux de z-index
2. Utiliser CSS Modules pour éviter les conflits de styles
3. Vérifier la compatibilité de la version Vue
