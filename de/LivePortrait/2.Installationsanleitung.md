# LivePortrait Installationsanleitung

Dieses Dokument bietet detaillierte Installationsanweisungen für LivePortrait auf verschiedenen Betriebssystemen und Umgebungen.

## Anforderungen

Stellen Sie vor der Installation sicher, dass Ihr System die folgenden grundlegenden Anforderungen erfüllt:

**Betriebssystem**: Linux (empfohlen), Windows 10/11, macOS (erfordert Apple Silicon)

**Python-Version**: Python 3.10

**Hardware-Anforderungen**: NVIDIA-GPU (RTX 4090 oder andere CUDA-fähige GPU empfohlen), mindestens 8GB VRAM

**Speicherplatz**: Mindestens 20GB für Modellgewichte und Laufzeitumgebung empfohlen

## Grundlegende Umgebungseinrichtung

### CUDA-Version überprüfen

Überprüfen Sie zunächst Ihre System-CUDA-Version, um die entsprechende PyTorch-Version zu installieren:

```bash
nvcc -V
```

Häufige CUDA-Versionen sind 11.1, 11.8 und 12.1. Wählen Sie basierend auf der angezeigten Versionsnummer den entsprechenden PyTorch-Installationsbefehl.

### PyTorch installieren

Wählen Sie den entsprechenden PyTorch-Installationsbefehl basierend auf Ihrer CUDA-Version:

**Für CUDA 11.1**:

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**Für CUDA 11.8**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**Für CUDA 12.1**:

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

Wenn Sie nur den Menschen-Modus ausführen müssen, können Sie die CUDA-Versionsprüfung überspringen und die neueste PyTorch-Version direkt installieren.

## Projektcode herunterladen

Klonen Sie das LivePortrait-Repository mit Git:

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Python-Umgebung erstellen

Es wird empfohlen, eine isolierte Python-Umgebung mit conda zu erstellen:

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

Oder verwenden Sie venv:

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## Abhängigkeiten installieren

Führen Sie Folgendes im Projektstammverzeichnis aus:

```bash
pip install -r requirements.txt
```

## Vortrainierte Modelle herunterladen

Vortrainierte Modelle können von HuggingFace heruntergeladen werden. Wenn Ihr Netzwerk auf HuggingFace zugreifen kann:

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Wenn Sie auf HuggingFace nicht zugreifen können, verwenden Sie eine Spiegelseite:

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Alternativ können Sie die vollständigen Modelldateien von Google Drive oder Baidu Netdisk herunterladen, entpacken und im Verzeichnis ./pretrained_weights ablegen.

## Installation überprüfen

Führen Sie nach der Installation das Testskript aus, um zu überprüfen, ob die Umgebung korrekt konfiguriert ist:

```bash
python inference.py
```

Wenn alles funktioniert, sehen Sie die generierte Ausgabedatei im animations-Verzeichnis.

## Zusätzliche Installationsschritte für Tiermodelle

Der Tiermodus wird nur auf Linux- und Windows-Systemen mit NVIDIA-GPU unterstützt.

### X-Pose-Abhängigkeit installieren

Der Tiermodus erfordert das X-Pose-Projekt als Keypoint-Erkennungsframework. Navigieren Sie zunächst zum X-Pose-Verzeichnis:

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### Tiermodelle von HuggingFace herunterladen

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Besondere Hinweise für Windows

Auf Windows-Systemen können bestimmte höhere CUDA-Versionen (wie 12.4, 12.6) Kompatibilitätsprobleme verursachen. Es wird empfohlen, CUDA auf Version 11.8 herabzustufen, um eine bessere Stabilität zu erreichen.

Windows-Benutzer können auch den Ein-Klick-Installer herunterladen und nach dem Entpacken durch Doppelklick auf run_windows.bat ausführen:

```bash
# Ein-Klick-Installer für Windows von HuggingFace herunterladen
# Adresse: https://huggingface.co/cleardusk/LivePortrait-Windows
```

## Besondere Hinweise für macOS

### Apple Silicon macOS

Für macOS-Benutzer mit Apple Silicon verwenden Sie die spezielle Abhängigkeitsdatei:

```bash
pip install -r requirements_macOS.txt
```

Stellen Sie beim Ausführen der Inferenz die Umgebungsvariable ein:

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

Hinweis: Das Ausführen der Inferenz auf Apple Silicon macOS kann etwa 20-mal langsamer sein als auf RTX 4090.

### Intel Mac

Intel-basierte Macs wurden nicht offiziell getestet und funktionieren möglicherweise nicht korrekt.

### Tiermodus-Einschränkung

Da die X-Pose-Abhängigkeit macOS nicht unterstützt, ist der Tiermodus auf macOS nicht verfügbar. Nur der Menschen-Modus kann verwendet werden.

## Verzeichnisstruktur überprüfen

Stellen Sie nach der Installation sicher, dass die Verzeichnisstruktur Folgendes erfüllt:

```
LivePortrait/
├── pretrained_weights/     # Verzeichnis für vortrainierte Modellgewichte
├── assets/                # Verzeichnis für Beispielmaterialien
│   ├── examples/
│   │   ├── source/        # Quellbild/Video
│   │   └── driving/       # Steuerungsvideo
├── src/                   # Quellcodeverzeichnis
├── inference.py          # Inferenz-Skript
├── app.py                # Gradio-Oberfläche
└── requirements.txt      # Abhängigkeitsliste
```

## Häufige Probleme

Wenn Sie während der Installation auf Probleme stoßen, überprüfen Sie Folgendes:

**CUDA nicht verfügbar**: Bestätigen Sie, dass NVIDIA-Treiber korrekt installiert sind; führen Sie nvidia-smi aus, um den Treiberstatus zu überprüfen

**Unzureichender VRAM**: Versuchen Sie, kleinere Eingabebilder zu verwenden oder die Stapelgröße zu reduzieren

**Modell-Download fehlgeschlagen**: Überprüfen Sie die Netzwerkverbindung oder verwenden Sie inländische Spiegelseiten

**Abhängigkeitskonflikte**: Es wird empfohlen, eine neue Python-Umgebung zu verwenden, um Konflikte mit anderen Projekten zu vermeiden
