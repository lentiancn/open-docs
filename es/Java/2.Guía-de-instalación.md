# Guía de Instalación de Java

## Obtener el Kit de Desarrollo de Java

Oracle proporciona descargas gratuitas del Java Development Kit (JDK). Las versiones LTS principales actuales incluyen JDK 25, JDK 21, JDK 17, JDK 11 y JDK 8.

Página oficial de descarga: https://www.oracle.com/java/technologies/downloads/

## Instalación en Windows

### Usando el Instalador

1. Visite la página oficial de descarga de Oracle y seleccione la versión de JDK para Windows
2. Descargue el instalador x64 (archivo exe) o el instalador MSI
3. Ejecute el instalador descargado
4. Siga las instrucciones del asistente de instalación para completar la instalación. La ruta predeterminada es C:\Program Files\Java\jdk-version
5. Después de la instalación, verifique que fue exitosa

### Usando el Archivo Comprimido

1. Descargue el archivo comprimido x64 (archivo zip)
2. Extraiga en un directorio designado, como C:\Java\jdk-25
3. Configure las variables de entorno

### Configuración de Variables de Entorno

1. Haga clic derecho en "Este PC" y seleccione "Propiedades"
2. Haga clic en "Configuración avanzada del sistema"
3. Haga clic en "Variables de entorno"
4. En Variables del sistema, haga clic en "Nueva"
   - Nombre de variable: JAVA_HOME
   - Valor de variable: Ruta de instalación del JDK (ej., C:\Program Files\Java\jdk-25)
5. Encuentre la variable Path y haga clic en "Editar"
6. Agregue nueva variable: %JAVA_HOME%\bin
7. Haga clic en Aceptar para guardar

### Verificar Instalación

Abra el símbolo del sistema y ejecute:

```cmd
java -version
javac -version
```

Si aparece información de la versión, la instalación fue exitosa.

## Instalación en Linux

### Instalar desde Archivo Comprimido

1. Descargue la versión de JDK para Linux (formato tar.gz)
2. Abra la terminal y extraiga en el directorio designado:

```bash
sudo mkdir -p /usr/local/java
sudo tar -zxf jdk-25_linux-x64_bin.tar.gz -C /usr/local/java
```

3. Configure las variables de entorno editando ~/.bashrc o /etc/profile:

```bash
export JAVA_HOME=/usr/local/java/jdk-25
export PATH=$JAVA_HOME/bin:$PATH
```

4. Aplique la configuración:

```bash
source ~/.bashrc
```

5. Verifique la instalación:

```bash
java -version
```

### Instalar con Gestor de Paquetes

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install openjdk-21-jdk
```

#### CentOS/RHEL

```bash
sudo yum install java-21-openjdk
```

#### Fedora

```bash
sudo dnf install java-21-openjdk
```

## Instalación en macOS

### Usando Instalador DMG

1. Descargue la versión de JDK para macOS (archivo dmg)
2. Abra el archivo dmg descargado
3. Arrastre JDK a la carpeta Applications
4. Abra la terminal y configure las variables de entorno:

```bash
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=$JAVA_HOME/bin:$PATH
```

5. Agregue lo anterior a ~/.zshrc o ~/.bash_profile
6. Verifique la instalación:

```bash
java -version
```

### Usando Homebrew

```bash
brew install openjdk@21
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

## Gestión de Múltiples Versiones de JDK

### Usando update-alternatives en Linux

```bash
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-21/bin/java" 1
sudo update-alternatives --install "/usr/bin/java" "java" "/usr/local/java/jdk-17/bin/java" 2
sudo update-alternatives --config java
```

### Cambiar Versiones en Windows

Modifique la variable de entorno JAVA_HOME para apuntar a diferentes directorios JDK.

### Usando SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.5-tem
sdk install java 17.0.5-tem
sdk install java 11.0.21-tem
```

## Verificar Entorno Java

### Verificar Versión de Java

```bash
java -version
```

Ejemplo de salida:

```
java version "25.0.1" 
Java(TM) SE Runtime Environment (build 25.0.1+12-29)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+12-29, mixed mode)
```

### Verificar Versión del Compilador

```bash
javac -version
```

Ejemplo de salida:

```
javac 25.0.1
```

### Verificar JAVA_HOME

```bash
echo $JAVA_HOME  # Linux/macOS
echo %JAVA_HOME%  # Windows
```

## Problemas Comunes de Instalación

### Problema 1: Comando java no encontrado

Solución: Verifique si la variable de entorno Path incluye el directorio bin del JDK.

### Problema 2: Permisos insuficientes

Solución: Use sudo para elevación en Linux/macOS, o verifique los permisos de archivos.

### Problema 3: Puerto en uso

Solución: Las aplicaciones Java usan el puerto 8080 por defecto. Asegúrese de que el puerto no esté usado por otros programas.

### Problema 4: Diferencia entre JDK y JRE

- JDK (Java Development Kit): Incluye herramientas de desarrollo y JRE
- JRE (Java Runtime Environment): Solo contiene componentes necesarios para ejecutar programas Java

Se requiere JDK para desarrollar programas Java. JRE es suficiente para ejecutar programas Java compilados.

## Recomendaciones Post-Instalación

1. Configure MAVEN_HOME o GRADLE_HOME si usa Maven o Grad
2. Configure M2_HOME para la ruta del repositorio local de Maven
3. Configure proxy HTTP si es necesario en entornos empresariales
4. Establezca la zona horaria para asegurar la hora correcta en aplicaciones Java

## Desinstalar Java

### Windows Panel de Control

1. Abra el Panel de Control
2. Seleccione "Programas y características"
3. Encuentre programas relacionados con Java
4. Haga clic derecho y seleccione "Desinstalar"

### Linux

```bash
sudo apt remove openjdk-*  # Debian/Ubuntu
sudo yum remove java-*    # CentOS/RHEL
```

### macOS

```bash
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-*
```
