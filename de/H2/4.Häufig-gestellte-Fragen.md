# Häufig gestellte Fragen zu H2

## Installationsprobleme

### F: Wie lade ich die H2-Datenbank herunter?

A: Besuchen Sie https://www.h2database.com/html/download.html, um die für Ihre Plattform geeignete Version herunterzuladen. Es gibt zwei Optionen: Windows-Installer und plattformübergreifendes ZIP-Paket.

### F: Welche Java-Version benötigt H2?

A: H2 benötigt Java 11 oder höher.

### F: Wie verwende ich H2 in einem Maven-Projekt?

A: Fügen Sie die Abhängigkeit in pom.xml hinzu:
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

### F: Wie führe ich H2 nach dem Entpacken des ZIP aus?

A: Führen Sie in der Befehlszeile aus:
```bash
java -jar h2.jar
```
Dadurch wird die H2-Webkonsole gestartet.

## Verbindungsprobleme

### F: Wie verbinde ich mich mit der H2-Datenbank?

A: Verwenden Sie JDBC-Verbindung:
```java
Class.forName("org.h2.Driver");
Connection conn = DriverManager.getConnection("jdbc:h2:mem:testdb", "sa", "");
```

### F: Was sind die Standard-Benutzerdaten von H2?

A: Der Standard-Benutzername ist "sa", mit leerem Passwort.

### F: Wie verbinde ich mich mit einer Dateidatenbank?

A:
```java
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### F: Wie starte ich den H2-TCP-Server?

A:
```java
import org.h2.tools.Server;
Server server = Server.createTcpServer().start();
```

### F: Was tun, wenn der Port belegt ist?

A: Sie können den Port in der Verbindungs-URL angeben:
```java
jdbc:h2:tcp://localhost:9093/testdb
```

## Datenprobleme

### F: Gehen In-Memory-Daten verloren?

A: Ja, die In-Memory-Datenbank (jdbc:h2:mem:) verliert Daten nach dem Schließen aller Verbindungen. Für Persistenz verwenden Sie den Dateidatenbankmodus.

### F: Wie sehe ich den Speicherort der Datenbankdatei?

A: Geben Sie den Pfad in der Verbindungs-URL an:
```java
jdbc:h2:file:/pfad/zur/datenbank
```

### F: Was tun, wenn die Datenbankdatei beschädigt ist?

A: Versuchen Sie den Wiederherstellungsmodus:
```java
jdbc:h2:file:./testdb;LOG=0
```
Oder stellen Sie aus einem Backup wieder her.

### F: Wie verschlüssele ich eine Datenbank?

A:
```java
jdbc:h2:file:./encrypted;CIPHER=AES
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

## Leistungsprobleme

### F: Wie verbessere ich die Leistung von Stapeleinfügungen?

A: Verwenden Sie Stapeloperationen und deaktivieren Sie Auto-Commit:
```java
conn.setAutoCommit(false);
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (...) {
    pstmt.setString(1, "...");
    pstmt.addBatch();
}
pstmt.executeBatch();
conn.commit();
```

### F: Was tun, wenn Abfragen langsam sind?

A: 
1. Erstellen Sie Indizes für häufige Abfragebedingungen
2. Verwenden Sie EXPLAIN zur Analyse des Abfrageplans
3. Vermeiden Sie SELECT *, fragen Sie nur benötigte Spalten ab

### F: Was tun bei zu hoher Speichernutzung?

A: Passen Sie die JVM-Heap-Größe an und verwenden Sie vernünftig Verbindugnspools.

## SQL-Kompatibilitätsprobleme

### F: Welche Datenbankmodi unterstützt H2?

A: H2 kann mehrere Datenbanken simulieren:
- MySQL
- PostgreSQL
- Oracle
- MS SQL Server
- DB2

Verwendung:
```java
jdbc:h2:mem:testdb;MODE=MySQL
```

### F: Wie behandle ich Auto-Inkrement-IDs?

A: H2 unterstützt mehrere Methoden:
```sql
-- Methode 1: IDENTITY
CREATE TABLE t1 (id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100));

-- Methode 2: SEQUENCE
CREATE SEQUENCE seq_id START WITH 1;
```

## Transaktionsprobleme

### F: Was passiert, wenn eine Transaktion nicht committed wird?

A: Nicht commitierte Transaktionen werden beim Schließen der Verbindung automatisch zurückgesetzt.

### F: Wie behandle ich Deadlocks?

A: H2 erkennt Deadlocks automatisch und setzt eine der Transaktionen zurück. Es wird empfohlen, den Transaktionsumfang zu verkürzen, um langfristige Sperren zu vermeiden.

## Sicherheitsprobleme

### F: Ist die H2-Datenbank sicher?

A: H2 eignet sich für Entwicklungs- und Testumgebungen. Für Produktionsumgebungen wird empfohlen:
1. Verschlüsselte Datenbank verwenden
2. Starke Passwörter verwenden
3. Netzwerkzugriff einschränken
4. SSL-Verbindungen verwenden

### F: Wie verhindere ich SQL-Injection?

A: Verwenden Sie PreparedStatement anstelle von String-Verkettung:
```java
// Richtig
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE name = ?");
pstmt.setString(1, name);

// Falsch
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE name = '" + name + "'");
```

## Tool-Probleme

### F: Wie verwende ich die H2-Konsole?

A: Führen Sie `java -jar h2.jar -web` aus und rufen Sie dann http://localhost:8082 auf

### F: Wie verwende ich die H2-Shell?

A: Führen Sie `java -jar h2.jar -shell` oder `java -jar h2.jar -tool` aus

### F: Wie sehe ich die H2-Version?

A:
```bash
java -jar h2.jar -version
```
Oder in SQL: `SELECT H2VERSION();`

## Migrationsprobleme

### F: Wie migriere ich von MySQL zu H2?

A:
1. H2-Kompatibilitätsmodus einstellen: `jdbc:h2:file:db;MODE=MySQL`
2. MySQL-Daten als SQL-Skript exportieren
3. Skript in H2 ausführen

### F: Was muss ich bei der Aktualisierung der H2-Version beachten?

A: Das Upgrade von H2 1.x auf 2.x erfordert besondere Überlegungen, bitte beachten Sie die offizielle Migrationsanleitung:
https://www.h2database.com/html/migration-to-v2.html
