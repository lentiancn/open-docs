# Docker 常见问题

常见问题解答。

## 安装问题

### Q: Docker 启动失败？

**解决方案**：
1. 启用 CPU 虚拟化
2. 重新安装 Docker Desktop
3. 检查系统要求

### Q: 权限被拒绝？

**解决方案**：
```bash
sudo usermod -aG docker $USER
# 重新登录
```

### Q: Docker 太慢？

**解决方案**：
1. 配置镜像加速
2. 使用国内镜像源

## 使用问题

### Q: 容器无法启动？

**解决方案**：
1. 检查日志
```bash
docker logs container_name
```
2. 检查端口占用
3. 检查配置文件

### Q: 镜像拉取失败？

**解决方案**：
1. 检查网络连接
2. 更换镜像源
3. 使用代理

### Q: 容器内无法联网？

**解决方案**：
1. 检查 DNS 配置
2. 检查网络模式
3. 重启 Docker

## 构建问题

### Q: 构建失败？

**解决方案**：
1. 检查 Dockerfile 语法
2. 检查网络连接
3. 清理缓存后重试

### Q: 镜像太大？

**解决方案**：
1. 使用多阶段构建
2. 使用 Alpine 基础镜像
3. 减少层数

### Q: .dockerignore 无效？

**解决方案**：
确保 .dockerignore 文件在正确位置。

## 数据问题

### Q: 容器删除后数据丢失？

**解决方案**：
1. 使用命名卷
2. 使用绑定挂载

### Q: 卷无法删除？

**解决方案**：
```bash
docker volume rm volume_name
```

## 网络问题

### Q: 容器间无法通信？

**解决方案**：
1. 使用同一网络
```bash
docker network create my-network
docker run --network my-network image1
docker run --network my-network image2
```
2. 检查网络配置

### Q: 端口映射失败？

**解决方案**：
1. 检查端口是否被占用
2. 使用不同端口

## 性能问题

### Q: Docker 占用太多内存？

**解决方案**：
1. 限制容器内存
2. 清理未使用的容器和镜像

### Q: 构建太慢？

**解决方案**：
1. 使用构建缓存
2. 优化 Dockerfile 层顺序

## 安全问题

### Q: 需要以 root 运行？

**解决方案**：
在 Dockerfile 中创建非 root 用户。
