# 쿠버네티스 사용자 매뉴얼

## 기본 개념

### 파드
파드는 Kubernetes에서 배포 가능한 가장 작은 단위로, 클러스터에서 실행되는 하나 이상의 컨테이너를 나타낸다. 일반적으로 파드는 단일 컨테이너를 포함하지만, 밀접하게 결합된 여러 컨테이너도 포함할 수 있다.

### 디플로이먼트
디플로이먼트는 상태 없는 애플리케이션 관리에 사용되며, 쉬운 배포, 확장을 위한 선언적 업데이트 기능을 제공한다.

### 서비스
서비스는 파드 세트에 안정적인 네트워크 엔드포인트를 제공하며, 파드의 IP 주소가 변경되어도 안정성을 유지한다.

### 네임스페이스
네임스페이스는 클러스터 리소스를 여러 논리적 파티션으로 나누며, 동일한 클러스터를 공유하는 여러 팀이나 프로젝트에 적합하다.

## 일반적인 명령어

### 클러스터 관리

```bash
# 클러스터 정보 보기
kubectl cluster-info

# 노드 보기
kubectl get nodes

# 모든 파드 보기
kubectl get pods --all-namespaces

# 클러스터 리소스 사용량 보기
kubectl top nodes
kubectl top pods
```

### 애플리케이션 배포

```bash
# 디플로이먼트 생성
kubectl create deployment nginx --image=nginx

# 디플로이먼트 보기
kubectl get deployments

# 스케일링
kubectl scale deployment nginx --replicas=3

# 이미지 업데이트
kubectl set image deployment/nginx nginx=nginx:1.21

# 롤백
kubectl rollout undo deployment/nginx

# 롤아웃 상태 보기
kubectl rollout status deployment/nginx
```

### 서비스 관리

```bash
# 서비스 노출
kubectl expose deployment nginx --port=80 --type=LoadBalancer

# 서비스 보기
kubectl get services

# 서비스 삭제
kubectl delete service <service-name>
```

### 디버깅

```bash
# 파드 로그 보기
kubectl logs <pod-name>

# 컨테이너에서 명령 실행
kubectl exec -it <pod-name> -- /bin/bash

# 리소스 세부 정보 설명
kubectl describe pod <pod-name>

# 디버깅을 위한 포트 포워딩
kubectl port-forward <pod-name> 8080:80
```

## 리소스 매니페스트

Kubernetes는 YAML 파일을 사용한 선언적 리소스 관리를 지원한다. 일반적인 리소스 정의 예시:

### 디플로이먼트 예시

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
```

### 서비스 예시

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: ClusterIP
```

### 컨피그맵 예시

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  database_url: "postgres://db:5432/myapp"
  cache_enabled: "true"
```

### 시크릿 예시

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: changeme
```

## 애플리케이션 수명 주기 관리

### 애플리케이션 배포

```bash
# YAML 파일로 배포
kubectl apply -f deployment.yaml

# 배포 상태 보기
kubectl get pods -l app=my-app
```

### 애플리케이션 업데이트

```bash
# 이미지 버전 업데이트
kubectl set image deployment/my-app my-app=my-app:v2

# 업데이트 진행 상황 보기
kubectl rollout status deployment/my-app
```

### 롤백

```bash
# 이전 버전으로 롤백
kubectl rollout undo deployment/my-app

# 특정 리비전으로 롤백
kubectl rollout undo deployment/my-app --to-revision=2

# 리비전 기록 보기
kubectl rollout history deployment/my-app
```

### 스케일링

```bash
# 수동 스케일링
kubectl scale deployment my-app --replicas=5

# 자동 스케일링 (HPA)
kubectl autoscale deployment my-app --min=2 --max=10 --cpu-percent=80
```

## 스토리지 관리

### 퍼시스턴트볼륨 생성

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  storageClassName: standard
  hostPath:
    path: /data/my-app
```

### 퍼시스턴트볼륨클레임 생성

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

### 파드에서 PVC 사용

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: my-app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

## 구성 관리

### 컨피그맵 사용

```bash
# 파일에서 컨피그맵 생성
kubectl create configmap my-config --from-file=config.properties

# 환경 변수 파일에서 생성
kubectl create configmap my-config --from-env-file=env.properties

# 파드에서 사용
kubectl apply -f pod-with-configmap.yaml
```

### 시크릿 사용

```bash
# 파일에서 시크릿 생성
kubectl create secret generic my-secret --from-file=credentials.json

# 파드에서 사용
kubectl apply -f pod-with-secret.yaml
```

## 리소스 쿼터 및 제한

### 리소스쿼터

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: my-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    pods: "20"
```

### 리밋레인지

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: my-limits
spec:
  limits:
  - max:
      cpu: "2"
      memory: 1Gi
    min:
      cpu: 100m
      memory: 128Mi
    default:
      cpu: 500m
      memory: 512Mi
    defaultRequest:
      cpu: 200m
      memory: 256Mi
    type: Container
```

## 모니터링 및 로그

### 모니터링

```bash
# kubectl top으로 리소스 사용량 보기
kubectl top nodes
kubectl top pods

# 세부 상태 보기
kubectl describe node <node-name>
kubectl describe pod <pod-name>
```

### 로그

```bash
# 파드 로그 보기
kubectl logs my-app-pod

# 실시간 로그 보기
kubectl logs -f my-app-pod

# 이전 컨테이너 인스턴스의 로그 보기
kubectl logs --previous my-app-pod
```
