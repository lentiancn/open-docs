# Flutter 자주 묻는 질문

## 설치 문제

### Flutter doctor가 오류를 표시하면 어떻게 해야 하나요?

flutter doctor 명령은 Flutter 개발 환경 상태를 검사합니다. 일반적인 오류 및 해결 방법:

- Android SDK를 찾을 수 없음: Android Studio를 설치하거나 ANDROID_SDK_ROOT 환경 변수를 수동으로 구성
- Xcode 버전이 너무 오래됨: Xcode를 최신 버전으로 업데이트
- 라이선스를 받지 않음: flutter doctor --android-licenses를 실행하여 라이선스 수용

### Flutter 다운로드가 실패하면 어떻게 해야 하나요?

다운로드가 느리거나 실패하면 미러를 사용해보세요. 중국 사용자는 PUB_HOSTED_URL 및 FLUTTER_STORAGE_BASE_URL 환경 변수를 설정하여 국내 미러를 사용할 수 있습니다.

### Git 클론이 실패하면 어떻게 해야 하나요?

네트워크가 GitHub에 액세스할 수 있는지 확인하세요. 프록시 또는 국내 미러를 사용해보세요. Flutter 웹사이트에서 사전 컴파일된 아카이브를 다운로드하여 수동으로 설치할 수도 있습니다.

## 개발 문제

### 핫 리로드가 작동하지 않으면 어떻게 해야 하나요?

핫 리로드가 실패하면 다음을 시도해보세요: 중지을 클릭한 다음 다시 실행; flutter clean을 실행하여 빌드 캐시 지우기; 코드에 구문 오류가 있는지 확인

### 애플리케이션이 느리면 어떻게 해야 하나요?

build 메서드에서 시간이 오래 걸리는 작업이 수행되고 있는지 확인하세요. 복잡한 계산을하려면 Isolate 또는 비동기 처리를 사용하세요. 불필요한 위젯 재구성을 피하고 const 생성자와 적절한 키를 사용하세요.

### 상태가 손실되면 어떻게 해야 하나요?

핫 리로드는 StatefulWidget 상태를 유지하지만 애플리케이션을 재구성하거나 완전한 핫 리스타트를 실행하면 상태가 재설정됩니다. 상태를 유지하려면 SharedPreferences, SQLite 또는 다른 저장소 솔루션을 사용하세요.

### 패키지 종속성 충돌이 발생하면 어떻게 해야 하나요?

flutter pub get을 실행하여 종속성 트리를 확인하세요. flutter pub deps를 사용하여 패키지 종속성을 확인하세요. 패키지 버전을 업데이트하거나 dependency_overrides를 사용하여 충돌을 해결해보세요.

## 플랫폼 문제

### Android 빌드가 실패하면 어떻게 해야 하나요?

일반적인 원인: SDK 버전 불일치, Gradle 구성 문제, 서명 구성 오류. flutter clean을 실행한 다음 다시 빌드해보세요. android/build.gradle 및 android/app/build.gradle 구성을 확인하세요.

### iOS 시뮬레이터가 시작되지 않으면 어떻게 해야 하나요?

Xcode가 설치되어 있고 올바르게 구성되어 있는지 확인하세요. 터미널에서 xcrun simctl list devices를 실행하여 사용 가능한 시뮬레이터를 확인하세요. 시뮬레이터를 재설정하거나 새 시뮬레이터를 만들어보세요.

### 웹 빌드 오류가 발생하면 어떻게 해야 하나요?

pubspec.yaml의 웹 종속성이 올바른지 확인하세요. flutter clean으로 정리한 다음 다시 빌드하세요. Chrome 브라우저가 설치되어 있는지 확인하세요.

## 성능 문제

### 앱 크기가 너무 크면 어떻게 해야 하나요?

flutter build apk --split-per-abi를 사용하여 다양한 CPU 아키텍처용 APK를 생성하세요. 사용되지 않는 이미지와 리소스를 확인하고 제거하세요. 난독화 및 압축을 사용하세요.

### 시작 시간이 너무 길면 어떻게 해야 하나요?

main 함수의 초기화 작업을 줄이세요. 필수적이지 않은 컴포넌트를 지연 로딩하세요. 스플래시 스크린을 사용하여 사용자 경험을 개선하세요.

### 메모리 사용량이 너무 높으면 어떻게 해야 하나요?

메모리 누수가 있는지 확인하세요. 프로파일 도구를 사용하여 메모리 사용량을 분석하세요. 더 이상 사용되지 않는 리소스와 큰 객체를 즉시 확보하세요.

## 학습 문제

### 상태 관리 솔루션을 어떻게 선택해야 하나요?

 간단한 앱에는 setState를 사용하세요. 중간 복잡도의 앱에는 Provider를 권장합니다. 복잡한 앱에는 Riverpod, GetX 또는 BLoC를 사용할 수 있습니다.

### 다른 프레임워크에서 마이그레이션怎么做?

Flutter 웹사이트에는 Android, iOS, React Native 개발자를 위한 마이그레이션 가이드가 있습니다. 핵심 개념은 유사하며 위젯과 레이아웃 시스템 학습에 집중하세요.

### 개발 효율성을 어떻게 개선할 수 있나요?

VS Code 또는 Android Studio Flutter 플러그인을 마스터하세요. 바로 가기와 코드 템플릿을 배우세요. 핫 리로드 및 디버깅 도구를 사용하세요. 공식 문서와 커뮤니티 리소스를 참조하세요.
