# Claude Code 使用手冊

本手冊將帶你快速上手 Claude Code，掌握日常開發中常用的操作。

## 準備工作

開始之前，請確保你具備以下條件：

- 已開啟終端或命令提示字元
- 有一個程式碼專案
- 擁有 Claude 訂閱（Pro、Max、Teams 或 Enterprise）、Claude Console 帳戶，或透過支援的雲端提供商存取

## 啟動會話

在專案目錄中開啟終端，啟動 Claude Code：

```bash
cd /path/to/your/project
claude
```

首次使用時會看到歡迎畫面，顯示會話資訊、最近的對話和最新更新。輸入 `/help` 查看可用命令，或輸入 `/resume` 繼續之前的對話。

## 基本操作

### 了解程式碼庫

讓 Claude 分析你的專案：

```
這個專案是做什麼的？
```

```
這個專案使用了哪些技術？
```

```
主入口檔案在哪里？
```

```
解釋一下資料夾結構
```

你也可以問 Claude 它的能力：

```
Claude Code 能做什麼？
```

```
如何在 Claude Code 中建立自訂技能？
```

**注意：Claude Code 會根據需要讀取專案檔案，不需要手動添加上下文。**

### 第一次程式碼修改

讓 Claude Code 完成實際的程式設計工作，嘗試一個簡單的任務：

```
在主檔案中添加一個 hello world 函數
```

Claude Code 會：
1. 找到合適的檔案
2. 展示擬定的更改
3. 請求你的批准
4. 執行編輯

**注意：Claude Code 始終會在修改檔案之前請求許可。你可以批准單個更改或為整個會話啟用"全部接受"模式。**

### Git 操作

Claude Code 讓 Git 操作變得簡單：

```
我修改了哪些檔案？
```

```
用描述性消息提交我的更改
```

```
建立一個名為 feature/quickstart 的新分支
```

```
顯示最近 5 次提交
```

```
幫我解決合併衝突
```

### 修復錯誤和新增功能

Claude 擅長偵錯和功能實現。

用自然語言描述你想要的内容：

```
為使用者註冊表單添加輸入驗證
```

或者修復現有問題：

```
有一個錯誤，使用者提交空表單後沒有提示 - 修復它
```

Claude Code 會：
- 定位相關程式碼
- 理解上下文
- 實現解決方案
- 執行測試（如果有）

### 常用工作流

**重構程式碼**

```
將認證模組重構為使用 async/await 而不是回調
```

**編寫測試**

```
為計算器函數編寫單元測試
```

**更新文件**

```
用安裝說明更新 README
```

**程式碼審查**

```
審查我的更改並提出改進建議
```

## 常用命令

以下是日常使用中最重要的命令：

| 命令 | 功能 | 範例 |
|------|------|------|
| `claude` | 啟動互動模式 | `claude` |
| `claude "任務"` | 執行一次性任務 | `claude "fix the build error"` |
| `claude -p "查詢"` | 執行一次性查詢後退出 | `claude -p "explain this function"` |
| `claude -c` | 在目前目錄繼續最近的對話 | `claude -c` |
| `claude -r` | 恢復之前的對話 | `claude -r` |
| `claude commit` | 建立 Git 提交 | `claude commit` |
| `/clear` | 清除對話歷史 | `/clear` |
| `/help` | 顯示可用命令 | `/help` |
| `exit` 或 Ctrl+C | 退出 Claude Code | `exit` |

## 進階技巧

### 使用 CLAUDE.md

在專案根目錄建立 `CLAUDE.md` 檔案，為 Claude Code 提供持久的專案上下文。這個檔案會在每次會話開始時自動載入。

CLAUDE.md 可以包含：
- 編碼標準和规范
- 架構決策
- 首選函式庫
- 審查清單
- 專案特定命令和腳本

### 自訂技能

建立自訂命令來封裝可重複的工作流，例如 `/review-pr` 或 `/deploy-staging`。

### 使用 Hooks

Hooks 允許你在 Claude Code 操作前後執行 shell 命令，例如：
- 每次編輯檔案後自動格式化
- 提交前執行 lint

### MCP 伺服器

透過 Model Context Protocol 將 Claude Code 連接到外部工具和資料來源：
- 讀取 Google Drive 中的設計文件
- 更新 Jira 工單
- 從 Slack 取得資料
- 使用自訂工具

### 多代理協作

對於複雜任務，可以同時啟動多個 Claude Code 代理並行工作。牵头代理協調工作、分配子任務並合併結果。

### 遠端控制

工作不限於單一裝置：
- 使用遠端控制從手機或其他裝置繼續本地會話
- 在 Web 或 iOS 應用上啟動長時間運行的任務，然後用 `/teleport` 拉回終端
- 使用 `/desktop` 將終端會話切換到桌面應用進行可視化 diff 審查

## 最佳實踐

### 請求要具體

❌ 不要說：「修復錯誤」

✅ 應該說：「修復登入錯誤，使用者輸入錯誤憑證後看到空白畫面」

### 使用逐步指令

將複雜任務分解為步驟：

```
1. 為使用者配置文件建立一個新的資料庫表格
2. 建立取得和更新使用者配置文件的 API 端點
3. 建構一個允許使用者查看和編輯資訊的網頁
```

### 讓 Claude 先探索

在做出更改之前，讓 Claude 理解你的程式碼：

```
分析資料庫結構
```

```
建構一個儀表板，顯示英國客戶退貨最多的產品
```

### 使用快捷鍵節省時間

- 按 `?` 查看所有可用的鍵盤快捷鍵
- 使用 Tab 補全命令
- 按 `↑` 查看命令歷史
- 輸入 `/` 查看所有命令和技能

## 常見問題

**Claude Code 讀取我的程式碼嗎？**

是的，Claude Code 會讀取專案檔案來理解程式碼庫。它不會將你的程式碼發送到外部伺服器（除非你使用第三方 API 提供商）。

**Claude Code 會自動修改我的檔案嗎？**

不會。Claude Code 始終會請求許可才會修改檔案。你可以批准單個更改或為整個會話啟用"全部接受"模式。

**我的程式碼安全嗎？**

Claude Code 設計注重安全性。你的程式碼不會被發送給 Anthropic 用於訓練模型（除非你明確選擇加入）。你可以在 [Anthropic 隱私權政策](https://www.anthropic.com/privacy)中了解更多。
