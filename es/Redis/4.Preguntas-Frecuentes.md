# Redis Preguntas Frecuentes

## Preguntas Generales

### ¿Qué es Redis?
Redis (Remote Dictionary Server) es un almacenamiento de estructuras de datos en memoria de código abierto que se puede utilizar como base de datos, caché y broker de mensajes.

### ¿Redis es gratuito?
Sí, Redis es de código abierto y gratuito. Se publica bajo la licencia BSD 3-Clause.

### ¿Qué lenguajes de programación puedo usar con Redis?
Redis proporciona bibliotecas clientes para casi todos los lenguajes de programación principales, incluyendo Python, JavaScript, Java, Go, C, C++, Ruby, PHP y muchos más.

## Preguntas de Instalación

### ¿Cómo instalo Redis en Windows?
Redis está diseñado principalmente para sistemas tipo Unix. Sin embargo, puedes usar:
- Windows Subsystem for Linux (WSL)
- Memurai (alternativa compatible con Redis)
- Contenedores Docker

### ¿Cómo instalo Redis en Linux/macOS?
```bash
# Ubuntu/Debian
sudo apt-get install redis-server

# macOS (usando Homebrew)
brew install redis
```

### ¿Cómo inicio el servidor Redis?
```bash
redis-server
```

Para conectar a Redis:
```bash
redis-cli
```

## Preguntas de Rendimiento

### ¿Qué tan rápido es Redis?
Redis puede realizar millones de operaciones por segundo con latencia sub-milisegundo.

### ¿Cuántos datos puede almacenar Redis?
Redis puede almacenar cientos de millones de claves. La principal limitación es la RAM disponible.

### ¿Redis soporta clustering?
Sí, Redis Cluster proporciona partición automática de datos en múltiples nodos con alta disponibilidad.

## Solución de Problemas

### ¿Por qué Redis consume demasiada memoria?
- Verificar claves o valores grandes
- Revisar uso de memoria con `INFO memory`
- Usar estructuras de datos apropiadas
- Habilitar políticas de evicción

### ¿Cómo persisto datos en Redis?
Configura persistencia en `redis.conf`:
- RDB: Snapshots periódicos
- AOF: Logging append-only file

### ¿Cómo protejo Redis?
- Vincular solo a localhost
- Usar autenticación con contraseña
- Habilitar encriptación TLS
- Establecer permisos de archivo apropiados
