# OpenClaw 설치 가이드

> OpenClaw 설정 및 설치에 대한 완벽한 가이드

---

## 시스템 요구사항

| 요구사항 | 사양 |
|---------|------|
| 운영 체제 | Windows 10+, macOS, Linux |
| Node.js | 버전 18 이상 |
| 메모리 | 최소 4GB RAM |
| 저장공간 | 약 500MB |

---

## 설치

### 전제 조건

Node.js가 설치되어 있는지 확인합니다:

```bash
node --version
npm --version
```

### 단계별 설치

#### 1. npm으로 설치

```bash
npm install -g openclaw
```

#### 2. Gateway 서비스 시작

Gateway 서비스는 OpenClaw의 핵심입니다:

```bash
# 상태 확인
openclaw gateway status

# Gateway 시작
openclaw gateway start

# Gateway 재시작
openclaw gateway restart

# Gateway 중지
openclaw gateway stop
```

#### 3. 초기 구성

설치 후 구성 파일을 설정합니다:

```bash
# 구성 디렉토리 생성
mkdir -p ~/.openclaw

# 기본 구성 복사
cp /path/to/openclaw/config/* ~/.openclaw/
```

## 구성

### 주요 구성 파일

| 파일 | 설명 |
|------|------|
| AGENTS.md | 작업 공간 구성 |
| SOUL.md | 성격 설정 |
| TOOLS.md | 도구 구성 |
| USER.md | 사용자 환경설정 |

### 환경 변수

```bash
# 선택 사항: 사용자 지정 작업 공간 디렉토리
OPENCLAW_WORKSPACE=/path/to/workspace

# 선택 사항: Gateway 포트
OPENCLAW_PORT=3000
```

## 확인

### 설치 테스트

```bash
# 버전 정보 표시
openclaw --version

# Gateway 상태 확인
openclaw gateway status

# 사용 가능한 명령 표시
openclaw --help
```

### 빠른 시작

```bash
# 대화형 세션 시작
openclaw chat

# 단일 작업 실행
openclaw exec "인사하기"
```

## 문제 해결

### Gateway가 시작되지 않음?

**해결책**:
1. Node.js가 올바르게 설치되어 있는지 확인
2. 권한 확인
3. 로그 검토: `openclaw gateway logs`

### 포트가 이미 사용 중?

**해결책**:
```bash
# 포트 변경
openclaw gateway start --port 3001
```

### 모듈을 찾을 수 없음?

**해결책**:
```bash
# 재설치
npm uninstall -g openclaw
npm install -g openclaw
```

## 제거

```bash
npm uninstall -g openclaw

# Gateway 서비스 제거
openclaw gateway stop
rm -rf ~/.openclaw
```
