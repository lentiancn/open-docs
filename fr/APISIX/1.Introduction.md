# Introduction

Apache APISIX est un projet de premier plan de la Apache Software Foundation, développé et offert par API7.ai. Il s'agit d'une passerelle API native pour le cloud, caractérisée par sa dynamique, sa réactivité en temps réel et ses performances élevées.

APISIX sert de point d'entrée pour tout le trafic métier et offre une richesse de fonctionnalités : routage dynamique, upstream dynamique, certificats dynamiques, tests A/B, déploiement progressif, déploiement canary, déploiement blue-green, limitation de débit, protection contre les attaques, collecte de métriques, alertes et surveillance, observabilité et gouvernance des services.

## Fonctionnalités principales

### Haute performance

APISIX est construit sur Nginx et hérite de ses excellentes performances. Grâce à son système de plugins, la logique métier est découplée des fonctionnalités core, ce qui rend les extensions plus souples tout en maintenant une capacité de traitement efficace.

### Entièrement dynamique

Contrairement à Nginx traditionnel qui nécessite un rechargement de la configuration, APISIX prend en charge une gestion de configuration entièrement dynamique. Tous les routages, upstream, plugins et autres éléments peuvent être mis à jour en temps réel via l'API d'administration, sans avoir besoin de redémarrer le service.

### Conception native pour le cloud

APISIX a été conçu dès le départ pour les environnements cloud natifs, avec prise en charge de plusieurs modes de déploiement : Docker, Kubernetes, Helm Chart, etc. Il fournit également un Ingress Controller qui peut être utilisé directement comme passerelle d'entrée pour Kubernetes.

### Écosystème de plugins

APISIX dispose d'un système de plugins riche couvrant l'authentification, la limitation de débit, la réécriture de requêtes, le traitement des réponses, la surveillance et la journalisation. Les utilisateurs peuvent étendre facilement les fonctionnalités de la passerelle via ce mécanisme de plugins.

### Support multi-protocoles

En plus des protocoles traditionnels HTTP/HTTPS, APISIX prend également en charge gRPC, Dubbo, WebSocket, MQTT et bien d'autres protocoles, répondant ainsi aux besoins des différentes architectures de microservices.

## Architecture

L'architecture d'APISIX se divise en deux parties :

**Le plan de données** assure le traitement réel de la transmission des requêtes. APISIX Core, basé sur une version modifiée de Nginx, implémente un pipeline de traitement de requêtes haute performance.

**Le plan de contrôle** gère toutes les configurations via une API RESTful et prend en charge plusieurs méthodes de persistance, notamment etcd et Consul. APISIX propose également un Dashboard pour une gestion visuelle des configurations via une interface web.

## Cas d'utilisation

- Point d'entrée unifié pour les API de microservices
- Ingress Controller pour Kubernetes
- Gestion et surveillance du trafic API
- Gestion API multi-locataire
- Proxy sidecar pour service mesh
- Passerelle pour applications IA (LLM et charges de travail IA)

En tant que projet de premier plan de la Apache Software Foundation, APISIX bénéficie d'une communauté active et de mises à jour régulières, offrant une solution fiable pour les applications d'entreprise.
