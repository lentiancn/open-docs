# GitLab Руководство пользователя

> Полное руководство по использованию GitLab

---

## Начало работы

### Создание аккаунта

1. Перейдите к вашему экземпляру GitLab
2. Нажмите "Зарегистрироваться"
3. Подтвердите email
4. Заполните профиль

### Создание проекта

1. Нажмите "Новый проект"
2. Выберите шаблон или пустой проект
3. Укажите название и описание
4. Настройте видимость
5. Создайте проект

---

## Git-операции

### Клонирование репозитория

```bash
# Клонировать через HTTPS
git clone https://gitlab.example.com/пользователь/проект.git

# Клонировать через SSH
git clone git@gitlab.example.com:пользователь/проект.git
```

### Базовый Git-рабочий процесс

```bash
# Создать ветку
git checkout -b feature/новая-функция

# Изменить и подготовить
git add .
git commit -m "Добавить новую функцию"

# Отправить на удаленный сервер
git push origin feature/новая-функция
```

## Отслеживание проблем

### Создание проблем

1. Перейдите в проект
2. Проблемы > Новая проблема
3. Добавьте заголовок и описание
4. Назначьте метки и вехи
5. Отправьте проблему

---

## Merge-запросы

### Создание merge-запроса

1. Отправьте ветку в GitLab
2. Нажмите "Создать merge-запрос"
3. Заполните заголовок и описание
4. Выберите ветки
5. Отправьте запрос

## CI/CD

### Базовый .gitlab-ci.yml

```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - npm install
    - npm run build

test:
  stage: test
  script:
    - npm test

deploy:
  stage: deploy
  script:
    - npm run deploy
  only:
    - main
```
