# FAQ OpenClaw

Cette page liste les questions fréquentes et les solutions lors de l'utilisation d'OpenClaw.

## Installation

### Quelles sont les exigences pour OpenClaw ?

- Node.js 22.x ou supérieur
- macOS, Linux ou Windows (WSL2)
- Au moins 2 Go de RAM recommandés

### L'installation échoue. Que dois-je faire ?

1. Vérifiez la version de Node.js : `node --version` (doit être >= 22)
2. Vérifiez la connexion réseau
3. Essayez différents gestionnaires de paquets : npm, pnpm ou bun
4. Exécutez `openclaw doctor` pour diagnostiquer

### Comment mettre à jour OpenClaw ?

```bash
# Passer au canal stable
openclaw update --channel stable

# Puis redémarrer Gateway
openclaw gateway restart
```

## Gateway

### Gateway ne démarre pas

- Vérifiez si le port 18789 est occupé
- Consultez les logs : `openclaw gateway --verbose`
- Essayez de tuer les processus existants

### Comment garder Gateway en fonctionnement constant ?

Utilisez `--install-daemon` pour installer en tant que service système :

```bash
openclaw onboard --install-daemon
```

### Comment accéder à Gateway à distance ?

1. **Tailscale** : Utilisez `gateway.tailscale.mode: "serve"` ou `"funnel"`
2. **Tunnel SSH** : Établir une connexion tunnel SSH

## Canaux

### Échec de connexion WhatsApp

- Assurez-vous d'avoir apparié votre appareil avec `openclaw channels login`
- Vérifiez si le réseau peut atteindre les serveurs WhatsApp

### Le Bot Telegram ne fonctionne pas

- Vérifiez que le Token Bot est correct
- Vérifiez que vous avez commencé une conversation avec le Bot
- Vérifiez la configuration `allowFrom`

### Les messages Discord ne répondent pas

- Vérifiez si le Bot a rejoint le serveur
- Confirmez que les bonnes permissions sont configurées
- Vérifiez si le canal est dans la liste d'autorisation

## Sécurité

### Les inconnus peuvent-ils m'envoyer des messages ?

Par défaut, OpenClaw exige une vérification de pairing pour les expéditeurs inconnus. Utilisez `dmPolicy: "open"` pour passer en mode ouvert, mais non recommandé.

### Comment gérer le pairing ?

```bash
# Voir la liste des pairings
openclaw pairing list

# Approuver le pairing
openclaw pairing approve

# Révoquer le pairing
openclaw pairing revoke
```

### L'exécution de `openclaw doctor` affiche des avertissements ?

Suivez les instructions pour corriger. Problèmes courants :
- Politique DM trop ouverte
- Configuration d'authentification requise manquante
- Risques d'exposition des ports

## Compétences

### Comment installer de nouvelles compétences ?

```bash
# Installer via ClawHub
openclaw skill install nom-de-la-competence

# Ou créer manuellement le répertoire de compétence et SKILL.md
```

### Les compétences ne fonctionnent pas ?

- Vérifiez que la configuration de la compétence est correcte
- Consultez les logs Gateway
- Confirmez que les dépendances de la compétence sont installées

## Voix

### Voice Wake ne fonctionne pas ?

- macOS : Vérifiez les permissions d'accessibilité dans les Préférences Système
- iOS : Confirmez que la permission microphone est activée
- Vérifiez si le mot d'activation est correctement défini

### La réponse vocale est trop lente ?

- Vérifiez la latence du réseau
- Envisagez d'utiliser TTS local
- Ajustez les paramètres du modèle vocal

## Performance

### OpenClaw utilise trop de mémoire ?

- Nettoyez régulièrement l'historique des sessions
- Ajustez les paramètres du modèle
- Utilisez la compression de session

### Les messages répondent lentement ?

- Vérifiez la connexion réseau
- Essayez de passer à un modèle plus rapide
- Vérifiez la charge du Gateway

## Données

### Où sont stockées les données ?

- Configuration : `~/.openclaw/`
- Espace de travail : `~/.openclaw/workspace`
- Identifiants : `~/.openclaw/credentials`

### Comment sauvegarder les données ?

```bash
# Sauvegarder le répertoire de configuration entier
cp -r ~/.openclaw ~/backup/openclaw-backup
```

### Comment réinitialiser tous les paramètres ?

1. Arrêtez Gateway
2. Supprimez le répertoire `~/.openclaw`
3. Exécutez à nouveau `openclaw onboard`

## Autres

### Comment voir les logs ?

```bash
# Voir les logs Gateway
openclaw gateway --verbose

# Ou consultez les fichiers de logs spécifiques
openclaw logs
```

### Comment obtenir de l'aide ?

- Documentation : https://docs.openclaw.ai
- Rejoignez la communauté Discord
- Soumettez une Issue sur GitHub

### Quelle est la différence entre OpenClaw et autres assistants IA ?

- **Entièrement local** : Vos données restent locales, non téléchargées sur le cloud
- **Open source** : Le code est complètement ouvert, vous pouvez le réviser vous-même
- **Highly customizable** : Vous pouvez modifier et étendre n'importe quelle fonctionnalité
- **Multi-canal unifié** : Un assistant gère tous les canaux de communication

### Puis-je exécuter plusieurs instances OpenClaw simultanément ?

Oui, chaque instance nécessite :
- Un port différent
- Un espace de travail indépendant
- Une configuration différente
