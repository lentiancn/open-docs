# Kubernetes よくある質問

## 基本コンセプト

### Kubernetesとは？

Kubernetes（K8s）は、デプロイやスケーリングを自動化したり、コンテナ化されたアプリケーションを管理したりするための、オープンソースのシステムです。アプリケーションを論理的な単位にグループ化し、管理と検出を容易にします。

### Podとコンテナの違いは？

PodはKubernetesで展開可能な最小単位です。Podは1つまたは複数のコンテナを含むことができます。通常、各Podで1つのコンテナのみを実行することが推奨されます。同一Pod内のコンテナはネットワークとストレージリソースを共有します。

### DeploymentとStatefulSetの違いは？

Deploymentはステートレスアプリケーションの管理に使用され、宣言的な更新、スケーリング、ロールバックをサポートします。StatefulSetはステートフルアプリケーションの管理に使用され、安定したネットワークIDと永続化ストレージを提供し、永続化を必要とするデータベースなどに適しています。

### Namespaceとは？

Namespaceはクラスターリソースを複数の論理パーティションに分割し、複数のチームやプロジェクト間でリソースを分離するのに役立ちます。一般的なデフォルトNamespaceにはdefault、kube-system、kube-publicがあります。

## インストールと設定

### MinikubeとKindの違いは？

Minikubeは完全なシングルノードKubernetesクラスターを作成し、学習と開発に適しています。Kind（Kubernetes in Docker）はDockerコンテナを使用してKubernetesノードをシミュレートし、より軽量で素早いテストに適しています。

### コンテナランタイムの選択方法は？

一般的なコンテナランタイムには、Docker、containerd、CRI-Oがあります。containerdはDockerのコアコンテナランタイムであり、現在広く推奨されています。

### kubeadm initが失敗した場合は？

一般的な原因：
- ポートが既に使用されている：6443、10250、10255などのポートを確認
- Swapが無効になっていない：`swapoff -a`を実行
- ファイアウォールルール：ノード間のネットワーク通信を確認
- SELinux/AppArmor：パーミッシブモードに設定するか無効化

### マルチコントロールプレーンノードクラスターの設定方法は？

kubeadmを使用して高可用性クラスターを作成：
1. 複数のサーバーノードを準備
2. kubeadm initで最初のコントロールプレーンノードを作成
3. 証明書を生成して他のノードにコピー
4. kubeadm joinを使用して追加のコントロールプレーンノードを追加

## ネットワークとストレージ

### Podが相互に通信できない場合は？

一般的な原因：
- ネットワークプラグインが正しくインストールされていない
- ファイアウォールがPodネットワークCIDRの通信をブロック
- CNIプラグイン設定エラー
- kube-proxyが正しく実行されているか確認

### Podに永続ストレージを設定する方法は？

PersistentVolume（PV）とPersistentVolumeClaim（PVC）を作成する必要があります。PVはローカルストレージ、ネットワークストレージ（NFS、iSCSI）、またはクラウドストレージ（AWS EBS、GCE PD、Azure Disk）にできます。

### Serviceの種類は？

- ClusterIP：クラスター内からアクセス可能な仮想IP
- NodePort：各ノードでポートを開く
- LoadBalancer：クラウドプロバイダーのロードバランサーを使用
- ExternalName：サービスを外部ドメイン名にマッピング

## リソース管理

### コンテナリソース制限を設定する方法は？

PodまたはDeploymentのspecにresourcesフィールドを追加：
```yaml
resources:
  limits:
    memory: "512Mi"
    cpu: "500m"
  requests:
    memory: "256Mi"
    cpu: "250m"
```

### ResourceQuotaとは？

ResourceQuotaはNamespace内のリソース总量を制限し、単一のチームやプロジェクトがリソースを消費しすぎるのを防ぎます。

### LimitRangeとは？

LimitRangeはNamespace内のPodとコンテナにデフォルトのリソース制限とリクエスト値を設定するために使用されます。

## デプロイと運用

### アプリケーションのローリングアップデートを実現する方法は？

Deploymentを使用すると、Kubernetesは自動的にローリングアップデートを実行：
```bash
kubectl set image deployment/my-app my-app=my-app:v2
kubectl rollout status deployment/my-app
```

### デプロイをロールバックする方法は？

```bash
# 前のバージョンにロールバック
kubectl rollout undo deployment/my-app

# 特定のリビジョンにロールバック
kubectl rollout undo deployment/my-app --to-revision=2
```

### HPAとは？

Horizontal Pod Autoscaler（HPA）はCPU、メモリなどのメトリクスに基づいてPodレプリカ数を自動的に調整します。Metrics Serverと組み合わせて使用​​する必要があります。

### ヘルスチェックを設定する方法は？

生存性と準備状態のプローブを設定：
```yaml
livenessProbe:
  httpGet:
    path: /healthz
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

## セキュリティ

### RBACとは？

Role-Based Access Control（RBAC）はKubernetesクラスター内の権限を管理するために使用されます。RoleとClusterRoleを作成し、ユーザーまたはグループにバインドできます。

### 機密情報を管理する方法は？

Secretを使用してパスワード、キー、証明書などの機密情報を保存します。kubectlコマンドラインで作成するか、YAMLファイルで定義できます。

### クラスターを保護する方法は？

- RBAC認証を有効化
- ネットワークポリシーを使用してPod間通信を制限
- Kubernetesバージョンを定期的に更新
- etcdデータを暗号化
- TLSを使用して通信を暗号化
- kubeletのアクセス権限を制限

## トラブルシューティング

### PodがPending状態のままの場合は？

可能性のある原因：
- リソース不足（CPUまたはメモリ）
- ノードセレクターまたはテイントが不一致
- PVCが正しくバインドされていない
- `kubectl describe pod <pod-name>`で詳細を確認

### PodがImagePullBackOff状態のままの場合は？

一般的な原因：
- イメージ名が正しくない
- イメージタグが存在しない
- プライベートレジストリの認証問題
- ネットワーク問題でイメージをプルできない

### PodがCrashLoopBackOff状態のままの場合は？

- コンテナログを確認：`kubectl logs --previous <pod-name>`
- アプリケーションが正しく起動しているか確認
- リソース設定が正しいか確認
- 依存サービスが利用可能か確認

### ネットワーク問題をデバッグする方法は？

- kube-proxyが正しく実行されているか確認
- CNIプラグインの状態を確認
- kubectl execを使用してコンテナに入りネットワーク接続性をテスト
- iptablesルールを確認
- ネットワーク診断ツールを使用（curl、ping、traceroute）

## パフォーマンス最適化

### クラスターパフォーマンスを向上させる方法は？

- 適切なリソースリクエストと制限を設定
- アンチAffinityを使用してPodを分散
- APIキャッシュを有効にしてリクエストを削減
- ローカルストレージを使用してネットワーク遅延を削減
- イメージサイズを最適化してプル時間を削減

### 大規模クラスターを処理する方法は？

- Federationを使用して複数のクラスターを管理
- 適切なetcdハードウェアを構成
- APIキャッシュを有効化
- ロードバランサーを使用してリクエストを分散
