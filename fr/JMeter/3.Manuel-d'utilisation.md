# Manuel d'utilisation de JMeter

Ce manuel fournit des instructions détaillées sur l'utilisation d'Apache JMeter pour créer et exécuter des tests de charge.

## Structure de base d'un plan de test

Un plan de test JMeter se compose des composants principaux suivants :

### 1. Plan de test

Le plan de test est le nœud racine de tous les tests, contenant les informations de configuration pour l'ensemble du test. Il peut inclure :
- Variables définies par l'utilisateur
- Paramètres des propriétés globales
- Configuration du groupe de threads

### 2. Groupe de threads

Les groupes de threads sont les unités d'exécution des tests, définissant :
- **Nombre de threads** : Nombre d'utilisateurs simulés
- **Temps de montée en puissance** : Temps requis pour démarrer tous les threads
- **Nombre de boucles** : Nombre de fois que chaque thread exécute le test
- **Planificateur** : Définir l'heure de début et la durée du test

### 3. Échantillonneurs

Les échantillonneurs envoient des requêtes aux serveurs. Les échantillonneurs courants incluent :

- **Requête HTTP** : Envoyer des requêtes HTTP/HTTPS
- **Requête FTP** : Tester les serveurs FTP
- **Requête JDBC** : Tester les connexions de base de données
- **Requête LDAP** : Tester les répertoires LDAP
- **Message JMS** : Tester les files d'attente de messages
- **Échantillonneur TCP** : Tester les services TCP

### 4. Contrôleurs logiques

Les contrôleurs logiques contrôlent l'ordre d'exécution des requêtes :

- **Contrôleur simple** : Regrouper les requêtes
- **Contrôleur de boucle** : Répéter l'exécution des requêtes
- **Contrôleur une seule fois** : Exécuter une seule fois pendant le test
- **Contrôleur alterné** : Alterner entre les requêtes
- **Contrôleur aléatoire** : Sélectionner aléatoirement des requêtes à exécuter
- **Contrôleur de débit** : Contrôler la fréquence d'exécution des requêtes

### 5. Écouteurs

Les écouteurs visualisent et analysent les résultats des tests :

- **Arborescence des résultats** : Visualiser la réponse détaillée de chaque requête
- **Rapport agrégé** : Visualiser les statistiques de performance agrégées
- **Résultats graphiques** : Afficher les données de test graphiquement
- **Graphique du temps de réponse** : Montrer les tendances du temps de réponse

### 6. Éléments de configuration

Les éléments de configuration définissent les valeurs par défaut et les données partagées :

- **Valeurs par défaut de requête HTTP** : Définir les paramètres par défaut pour les requêtes HTTP
- **Variables définies par l'utilisateur** : Définir des variables réutilisables
- **Configuration du jeu de données CSV** : Lire les données de test depuis des fichiers CSV
- **Gestionnaire de cookies HTTP** : Gérer les cookies
- **Gestionnaire d'en-têtes HTTP** : Définir les informations d'en-tête de requête

### 7. Minuteries

Les minuteurs définissent les délais entre les requêtes :

- **Minuterie fixe** : Définir un délai fixe
- **Minuterie aléatoire gaussienne** : Définir un délai aléatoire
- **Minuterie de synchronisation** : Simuler des requêtes simultanées
- **Minuterie de débit** : Contrôler le débit des requêtes

### 8. Pré-processeurs et post-processeurs

- **Pré-processeurs** : Exécuter des opérations avant l'envoi de la requête
- **Post-processeurs** : Extraire les données après réception de la réponse

## Créer votre premier plan de test

### Étape 1 : Créer un plan de test

1. Lancez JMeter
2. Sélectionnez "Fichier" -> "Nouveau" pour créer un nouveau plan de test
3. Nommez votre plan de test

### Étape 2 : Ajouter un groupe de threads

1. Faites un clic droit sur le plan de test
2. Sélectionnez "Ajouter" -> "Threads (Utilisateurs)" -> "Groupe de threads"
3. Définissez le nombre de threads (ex: 10), le temps de montée (ex: 5 secondes), le nombre de boucles (ex: 1)

### Étape 3 : Ajouter une requête HTTP

1. Faites un clic droit sur le groupe de threads
2. Sélectionnez "Ajouter" -> "Échantillonneur" -> "Requête HTTP"
3. Définissez le nom du serveur ou IP, le numéro de port, le chemin de requête, etc.

### Étape 4 : Ajouter un écouteur

1. Faites un clic droit sur le groupe de threads ou la requête HTTP
2. Sélectionnez "Ajouter" -> "Échantillonneur" -> "Vue результатов arbre"
3. Ajoutez "Rapport agrégé" pour voir les statistiques de performance

### Étape 5 : Exécuter le test

1. Cliquez sur le bouton "Démarrer" dans la barre d'outils
2. Observez les données de résultats affichées dans l'écouteur

## Paramétrage et tests pilotés par les données

### Utiliser des fichiers de données CSV

1. Ajoutez l'élément "Configuration du jeu de données CSV"
2. Définissez le nom de fichier et les noms de variables
3. Utilisez ${nomVariable} dans les requêtes pour référencer les données

### Utiliser des variables définies par l'utilisateur

1. Ajoutez l'élément "Variables définies par l'utilisateur"
2. Définissez les noms et valeurs des variables
3. Utilisez ${nomVariable} dans les requêtes

## Gestion de la corrélation

### Utiliser l'extracteur d'expressions régulières

Lorsque les requêtes nécessitent des données dynamiques (comme l'ID de session) :

1. Ajoutez "Extracteur d'expressions régulières"
2. Définissez le motif d'expression régulière à faire correspondre
3. Utilisez la variable extraite dans les requêtes suivantes

### Utiliser l'extracteur JSON

Pour les réponses JSON :

1. Ajoutez "Extracteur JSON"
2. Définissez l'expression JSON Path
3. Extrayez les valeurs de champ JSON requises

## Tests distribués

Lorsqu'une seule machine ne peut pas générer suffisamment de charge, des tests distribués peuvent être utilisés :

### Configuration du maître

1. Modifiez jmeter.properties dans le répertoire bin du maître
2. Définissez remote_hosts pour pointer vers les adresses des esclaves
3. Lancez JMeter

### Configuration de l'esclave

1. Exécutez jmeter-server sur chaque machine esclave
2. Assurez-vous que le pare-feu permet la communication sur les ports spécifiés

### Démarrer un test distribué

1. Sélectionnez "Démarrage distant" sur le maître
2. Choisissez quels esclaves démarrer ou démarrez tous

## Conseils d'optimisation des performances

- Utilisez le mode ligne de commande pour réduire la consommation de ressources GUI
- Définissez un nombre de threads et un temps de montée appropriés pour éviter une charge instantanée élevée
- Désactivez les écouteurs inutiles, visualisez les résultats uniquement si nécessaire
- Utilisez des assertions pour vérifier le contenu de la réponse
- Nettoyez régulièrement les plans de test non utilisés
- Pour les tests à grande échelle, envisagez des tests distribués
