# Kubernetes 使用手册

## 基本概念

### Pod
Pod 是 Kubernetes 中可部署的最小计算对象，代表集群中运行的一个或多个容器。通常一个 Pod 包含一个容器，但也可以包含多个紧密耦合的容器。

### Deployment
Deployment 用于管理无状态应用，提供了声明式更新能力，可以轻松实现应用的部署、扩缩和更新。

### Service
Service 为一组 Pod 提供稳定的网络端点，即使 Pod 的 IP 地址发生变化，Service 也能保持稳定。

### Namespace
Namespace 用于将集群资源划分为多个逻辑分区，适合多团队或多项目共享同一集群的场景。

## 常用命令

### 集群管理

```bash
# 查看集群信息
kubectl cluster-info

# 查看节点
kubectl get nodes

# 查看所有 Pod
kubectl get pods --all-namespaces

# 查看集群资源使用情况
kubectl top nodes
kubectl top pods
```

### 应用部署

```bash
# 创建 Deployment
kubectl create deployment nginx --image=nginx

# 查看 Deployment
kubectl get deployments

# 扩缩容
kubectl scale deployment nginx --replicas=3

# 更新镜像
kubectl set image deployment/nginx nginx=nginx:1.21

# 回滚
kubectl rollout undo deployment/nginx

# 查看滚动更新状态
kubectl rollout status deployment/nginx
```

### 服务管理

```bash
# 暴露服务
kubectl expose deployment nginx --port=80 --type=LoadBalancer

# 查看服务
kubectl get services

# 删除服务
kubectl delete service <service-name>
```

### 调试

```bash
# 查看 Pod 日志
kubectl logs <pod-name>

# 进入容器
kubectl exec -it <pod-name> -- /bin/bash

# 描述资源详细信息
kubectl describe pod <pod-name>

# 端口转发用于调试
kubectl port-forward <pod-name> 8080:80
```

## 资源清单

Kubernetes 支持使用 YAML 文件声明式管理资源。以下是常见的资源定义示例：

### Deployment 示例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "128Mi"
            cpu: "500m"
          requests:
            memory: "64Mi"
            cpu: "250m"
```

### Service 示例

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: ClusterIP
```

### ConfigMap 示例

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  database_url: "postgres://db:5432/myapp"
  cache_enabled: "true"
```

### Secret 示例

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: changeme
```

## 应用生命周期管理

### 部署应用

```bash
# 使用 YAML 文件部署
kubectl apply -f deployment.yaml

# 查看部署状态
kubectl get pods -l app=my-app
```

### 更新应用

```bash
# 更新镜像版本
kubectl set image deployment/my-app my-app=my-app:v2

# 查看更新进度
kubectl rollout status deployment/my-app
```

### 回滚

```bash
# 回滚到上一版本
kubectl rollout undo deployment/my-app

# 回滚到指定版本
kubectl rollout undo deployment/my-app --to-revision=2

# 查看历史版本
kubectl rollout history deployment/my-app
```

### 扩缩容

```bash
# 手动扩缩容
kubectl scale deployment my-app --replicas=5

# 自动扩缩容（HPA）
kubectl autoscale deployment my-app --min=2 --max=10 --cpu-percent=80
```

## 存储管理

### 创建 PersistentVolume

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  storageClassName: standard
  hostPath:
    path: /data/my-app
```

### 创建 PersistentVolumeClaim

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

### 在 Pod 中使用 PVC

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: my-app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

## 配置管理

### 使用 ConfigMap

```bash
# 从文件创建 ConfigMap
kubectl create configmap my-config --from-file=config.properties

# 从环境变量文件创建
kubectl create configmap my-config --from-env-file=env.properties

# 在 Pod 中使用
kubectl apply -f pod-with-configmap.yaml
```

### 使用 Secret

```bash
# 从文件创建 Secret
kubectl create secret generic my-secret --from-file=credentials.json

# 在 Pod 中使用
kubectl apply -f pod-with-secret.yaml
```

## 资源配额和限制

### ResourceQuota

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: my-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    pods: "20"
```

### LimitRange

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: my-limits
spec:
  limits:
  - max:
      cpu: "2"
      memory: 1Gi
    min:
      cpu: 100m
      memory: 128Mi
    default:
      cpu: 500m
      memory: 512Mi
    defaultRequest:
      cpu: 200m
      memory: 256Mi
    type: Container
```

## 监控和日志

### 监控

```bash

kubectl top nodes
kubectl top pods

# 使用 kubectl describe 查看详细状态
```

### 日志

```bash
# 查看 Pod 日志
kubectl logs my-app-pod

# 实时查看日志
kubectl logs -f my-app-pod

# 查看上一个容器实例的日志
kubectl logs --previous my-app-pod
```
