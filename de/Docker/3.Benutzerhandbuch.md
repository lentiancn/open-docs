# Docker Benutzerhandbuch

Dieses Handbuch hilft Ihnen, schnell mit Docker zu beginnen und gängige Operationen für die tägliche Nutzung zu meistern.

## Voraussetzungen

Bevor Sie beginnen, stellen Sie sicher, dass Sie haben:

- Docker installiert (Docker Desktop oder Docker Engine)
- Zugriff auf die Befehlszeile
- Grundkenntnisse über Befehlszeilenoperationen

## Grundkonzepte im Überblick

### Images und Container

- **Image**: Eine schreibgeschützte Vorlage zur Erstellung von Containern
- **Container**: Eine laufende Instanz eines Images

### Gängige Befehle

```bash
# Container ausführen
docker run hello-world

# Alle Container auflisten
docker ps -a

# Alle Images auflisten
docker images

# Einen Container stoppen
docker stop container_id

# Einen Container entfernen
docker rm container_id

# Ein Image entfernen
docker rmi image_id
```

## Mit Containern arbeiten

### Einen Container ausführen

Der grundlegende Befehl zum Ausführen eines Containers lautet:

```bash
docker run [Optionen] image_name [Befehl]
```

Beispiele:

```bash
# Einen interaktiven Container starten
docker run -it ubuntu bash

# Einen Container im Hintergrund ausführen
docker run -d nginx

# Einen Container mit Portweiterleitung ausführen
docker run -p 8080:80 nginx
```

### Container verwalten

```bash
# Alle laufenden Container anzeigen
docker ps

# Alle Container (auch gestoppte) anzeigen
docker ps -a

# Einen Container starten
docker start container_name

# Einen Container stoppen
docker stop container_name

# Einen Container neu starten
docker restart container_name

# Einen Container entfernen
docker rm container_name
```

### Container-Logs anzeigen

```bash
# Logs eines Containers anzeigen
docker logs container_name

# Logs in Echtzeit verfolgen
docker logs -f container_name

# Letzte 100 Zeilen anzeigen
docker logs --tail 100 container_name
```

## Mit Images arbeiten

### Images suchen und herunterladen

```bash
# Images im Docker Hub suchen
docker search nginx

# Ein Image herunterladen
docker pull nginx

# Ein Image mit Tag herunterladen
docker pull nginx:latest
```

### Images erstellen

Sie können eigene Images mit einer Dockerfile erstellen:

```dockerfile
FROM ubuntu:20.04
RUN apt-get update
RUN apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Dann erstellen Sie das Image:

```bash
docker build -t mein-nginx-image .
```

### Images verwalten

```bash
# Alle Images auflisten
docker images

# Ein Image entfernen
docker rmi image_name

# Unbenannte Images entfernen
docker image prune
```

## Volumes verwenden

Volumes werden verwendet, um Daten außerhalb von Containern zu speichern.

```bash
# Ein Volume erstellen
docker volume create mein_volume

# Ein Volume einhängen
docker run -v mein_volume:/daten nginx

# Alle Volumes auflisten
docker volume ls

# Ein Volume entfernen
docker volume rm mein_volume
```

## Netzwerke

Docker bietet Netzwerkfunktionen für Container-Kommunikation.

```bash
# Ein Netzwerk erstellen
docker network create mein_netzwerk

# Container mit einem Netzwerk verbinden
docker run --network mein_netzwerk nginx

# Alle Netzwerke auflisten
docker network ls
```

## Docker Compose

Docker Compose ermöglicht die Definition und Ausführung von Multi-Container-Anwendungen.

### docker-compose.yml Beispiel

```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: passwort
```

### Befehle

```bash
# Alle Container starten
docker-compose up

# Im Hintergrund starten
docker-compose up -d

# Alle Container stoppen
docker-compose down
```

## Tipps und Best Practices

### Sicherheit

- Verwenden Sie offizielle Images wann immer möglich
- Führen Sie Container nicht als Root aus
- Halten Sie Images aktuell
- Scannen Sie Images auf Schwachstellen

### Leistung

- Verwenden Sie .dockerignore-Dateien
- Nutzen Sie Multi-Stage-Builds
- Minimieren Sie die Anzahl der Schichten in Images
- Verwenden Sie spezifische Image-Tags

### Wartbarkeit

- Verwenden Sie aussagekräftige Containernamen
- Dokumentieren Sie Ihre Dockerfiles
- Nutzen Sie Volumes für persistente Daten
- Protokollieren Sie Container-Aktivitäten
