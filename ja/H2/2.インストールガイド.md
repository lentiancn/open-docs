# H2 データベースインストールガイド

## 環境要件

- Java 11以上
- Javaをサポートする任意のオペレーティングシステム（Windows、Linux、macOSなど）

## インストール方法

### 方法1：ZIPパッケージのダウンロード

1. 公式サイトダウンロードページにアクセス：https://www.h2database.com/html/download.html
2. 「Platform-Independent Zip」バージョンをダウンロード
3. ダウンロードしたZIPファイルを任意のディレクトリに展開（例：C:\h2 または /opt/h2）

### 方法2：Maven依存関係

MavenまたはGradleのJavaプロジェクトを使用している場合、依存関係を直接追加できます：

**Maven:**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.2.224</version>
</dependency>
```

**Gradle:**
```groovy
implementation 'com.h2database:h2:2.2.224'
```

### 方法3：Windowsインストーラー

1. 「Windows Installer」バージョンをダウンロード
2. インストーラーを実行しウィザードに従ってインストールを完了

## 環境変数の設定（オプション）

ZIPパッケージをダウンロードし、コマンドラインで直接H2を使用したい場合：

**Windows:**
1. 「PC」を右クリック → 「プロパティ」
2. 「システムの詳細設定」をクリック
3. 「環境変数」をクリック
4. 「Path」を見つけてクリック
5. H2のbinフォルダのパスを追加

**Linux/macOS:**
```bash
export PATH=$PATH:/opt/h2/bin
```

## インストールの確認

### コマンドラインを使用

展開後、コマンドラインで以下を実行：

```bash
java -jar h2.jar
```

これによりH2コンソールのWebインターフェースが起動します。

### JDBC接続を使用

JavaコードでH2データベースに接続：

```java
// 組み込みモード
String url = "jdbc:h2:mem:testdb";
Connection conn = DriverManager.getConnection(url, "sa", "");

// ファイルデータベース
String url = "jdbc:h2:file:./data/testdb";
Connection conn = DriverManager.getConnection(url, "sa", "password");
```

### H2コンソールを使用

1. コマンドを実行：`java -jar h2.jar`
2. ブラウザで開く：http://localhost:8082
3. デフォルトユーザー名：sa、パスワード：空

## 接続URLフォーマット

H2は様々な接続URLフォーマットをサポートします：

| モード | URL例 | 説明 |
|------|------|------|
| メモリモード | jdbc:h2:mem:testdb | データはメモリに保存 |
| 組み込みファイル | jdbc:h2:file:./data/testdb | データはファイルに保存 |
| TCPサーバー | jdbc:h2:tcp://localhost:9092/testdb | クライアントサーバーモード |
| TCP暗号化 | jdbc:h2:ssl://localhost:9092/testdb | SSL暗号化接続 |
| PostgreSQL互換 | jdbc:h2:postgres://localhost:5432/testdb | PostgreSQLプロトコル互換 |

## よくある問題

### ポートが使用中
H2はデフォルトで9092ポート（TCPモード）と8082ポート（Webコンソール）を使用します。ポートが使用中の場合は、接続URLで他のポートを指定できます。

### Javaバージョン問題
Java 11以上がインストールされていることを確認してください。確認方法：

```bash
java -version
```

### ドライバーが見つからない
H2のJARファイルがclasspathにあることを確認してください。Mavenプロジェクトを使用している場合、依存関係が正しく追加されていることを確認してください。
