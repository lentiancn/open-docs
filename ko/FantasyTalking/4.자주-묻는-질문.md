# FantasyTalking 자주 묻는 질문

## 설치 관련

### 설치에 실패하면 어떻게 해야 하나요?

설치 중 다양한 문제가 발생할 수 있습니다. 먼저 환경이 요구 사항을 충족하는지 확인하세요. `node --version`으로 Node.js 버전을 확인합니다 (14.0 이상 필요). Python을 사용하는 경우 3.7 이상인지 확인하세요.

설치가 계속 실패하면 다음을 시도해 보세요: npm 캐시 지우기 `npm cache clean --force`; 국내 미러 사용; 또는 소스에서 설치.

### FFmpeg 설치 문제

일부 시스템에서 FFmpeg가 올바르게 설치되지 않았을 수 있습니다. FFmpeg 공식 웹사이트에서 사전 컴파일된 바이너리를 다운로드하거나 패키지 관리자를 사용하여 설치할 수 있습니다.

## 사용 관련

### 음성 합성이 느리면 어떻게 해야 하나요?

음성 합성 속도는 여러 요인에 따라 달라집니다. 먼저 네트워크 연결을 확인하세요. 클라우드 서비스를 사용하는 경우 네트워크 지연이 응답 시간에 영향을 합니다. 오버헤드를 줄이기 위해 자체 호스팅 서비스 사용을 고려하세요. 대량 합성 작업에는 배치 합성 인터페이스를 사용하세요.

### 적절한 음성 스타일을 선택하는 방법은?

FantasyTalking은 neutral, happy, sad, angry, excited 등 다양한 음성 스타일을 제공합니다. 적용 시나리오에 따라 선택하세요: 고객 서비스에는 neutral 또는 professional 사용.

### 영어 음성 합성을 구현하려면?

기본 설정에서 영어를 지정해야 합니다:

```javascript
const audio = await client.speak('Hello, world!', {
  voice: { language: 'en-US', style: 'neutral' }
});
```

지원되는 언어 코드에는 en-US, en-GB, en-AU 등이 포함됩니다.

### SSML 마크업이 작동하지 않으면?

speak 함수 호출 시 `ssml: true` 매개변수를 전달해야 합니다. SSML 마크업이 올바른지 확인하세요.

## 대화 기능

### 대화이 일관되지 않으면?

동일한 세션 인스턴스를 사용하고 있는지 확인하세요. 각 `session.chat()` 호출은 자동으로 이전 대화 기록을 포함합니다.

### 대화를 저장하고 복원하려면?

`session.getHistory()`를 사용하여 대화 기록을 가져오고, `client.loadSession(history)`를 사용하여 복원하세요.

## 오디오 문제

### 생성된 오디오에 노이즈가 있으면?

노이즈 문제는 일반적으로 오디오 구성 또는 네트워크 전송과 관련이 있습니다. 오디오 형식과 샘플 레이트가 대상 재생 장치와 일치하는지 확인하세요.

### 속도와 피치를 조정하려면?

```javascript
const audio = await client.speak('텍스트', {
  voice: { speed: 1.5, pitch: 0.8 }
});
```

## 계정 및 요금

### API 키를 얻으려면?

FantasyTalking 공식 웹사이트에서등록하고 콘솔에서 API 키를 만드세요.

### 사용량을 확인하려면?

콘솔에 로그인 후 대시보드에서 API 호출 횟수와 확인할 수 있습니다.

## 배포 관련

### Docker 배포가 실패하면?

먼저 Docker가 올바르게 설치되어 실행 중인지 확인하세요. 포트 8080이 사용 중인지 확인하세요.

## 개발 및 디버깅

### 디버그 모드를 활성화하려면?

```javascript
const client = new FantasyTalking({
  apiKey: 'your-api-key',
  debug: true
});
```
