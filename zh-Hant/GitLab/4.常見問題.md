# GitLab 常見問題

## 安裝問題

### GitLab 安裝失敗怎麼辦？

安裝失敗通常由幾個常見原因導致。首先檢查系統是否滿足最低要求，包括作業系統版本、記憶體和磁碟空間。如果使用自托管腳本安裝失敗，可以嘗試手動安裝依賴包，然後分步驟安裝。

查看安裝日誌可以取得詳細錯誤資訊：

```bash
# Ubuntu/Debian
sudo apt-get install -y gitlab-ce 2>&1 | tee /tmp/gitlab-install.log

# 查看日誌
sudo tail -f /var/log/gitlab/install.log
```

常見錯誤包括：埠衝突（80、443、22 埠被佔用）、記憶體不足、SSL 憑證問題等。如果是埠衝突，需要先停止佔用埠的服務，或者修改 GitLab 設定使用其他埠。

### 如何修改 GitLab 存取位址？

安裝後如果需要修改 GitLab 的存取位址，編輯 /etc/gitlab/gitlab.rb 檔案中的 external_url 參數，然後重新設定：

```bash
sudo nano /etc/gitlab/gitlab.rb
# 修改 external_url
sudo gitlab-ctl reconfigure
```

修改後需要確保 DNS 解析或本地 hosts 檔案正確設定，以便新位址可以正常存取。如果從 HTTP 改為 HTTPS，還需要設定 SSL 憑證。

## 使用問題

### SSH 金鑰無法連線怎麼辦？

SSH 金鑰連線失敗可能由多種原因導致。首先確認公鑰已正確新增到 GitLab 帳戶的 SSH 金鑰設定中：

```bash
# 查看本地公鑰
cat ~/.ssh/id_ed25519.pub

# 測試連線
ssh -T git@gitlab.example.com
```

如果測試連線失敗，檢查 SSH 用戶端設定：

```bash
# 編輯 SSH 設定
nano ~/.ssh/config

# 新增以下內容
Host gitlab.example.com
  HostName gitlab.example.com
  User git
  IdentityFile ~/.ssh/id_ed25519
  IdentitiesOnly yes
```

確保遠端倉庫 URL 使用 SSH 格式（git@gitlab.example.com:group/project.git），而不是 HTTPS 格式。

### Merge Request 無法合併怎麼辦？

合併請求可能因為以下原因無法合併：存在衝突、CI/CD 測試失敗、未滿足保護分支規則、缺少審批等。解決方法是逐一排查：

- 合併衝突：在合併請求頁面點擊「更新分支」，將目標分支合併到來源分支，解決衝突後重新推送
- CI 失敗：查看失敗的作業日誌，解決問題後重新執行流水線
- 分支保護：聯繫專案管理員修改保護分支設定，或由有權限的成員合併

### CI/CD 流水線執行失敗如何排查？

流水線失敗時，首先在 CI/CD → 流水線頁面查看失敗作業的日誌。日誌會顯示具體的錯誤資訊和失敗原因。常見問題包括：依賴下載失敗、腳本命令錯誤、測試案例失敗等。

如果問題難以定位，可以在流水線設定中加入偵錯步驟，例如輸出環境變數、安裝前後檢查檔案狀態等。還可以使用互動式偵錯功能，在流水線作業中啟動互動式 Shell 進行排查。

## 效能問題

### GitLab 存取速度很慢怎麼辦？

存取速度慢可能由伺服器資源不足、網路延遲或設定不當導致。首先檢查伺服器負載：

```bash
sudo gitlab-ctl status
top -bn1 | head -20
```

如果 CPU 或記憶體使用率很高，可能需要升級伺服器設定或優化 GitLab 設定。還可以檢查資料庫效能：

```bash
sudo gitlab-rake gitlab:git:fsck  # 檢查 Git 倉庫完整性
sudo gitlab-rake db:index_cache_daily  # 優化資料庫快取
```

對於網路問題，可以考慮使用 CDN 加速靜態資源，或在更接近使用者的網路位置部署 GitLab。

### 如何優化 GitLab 效能？

優化 GitLab 效能可以從多個方面入手。資料庫優化是重要的一環，定期維護可以顯著提升效能：

```bash
# 清理資料庫
sudo gitlab-rake gitlab:db:analyze
sudo gitlab-rake gitlab:db:optimize

# 清理未使用的檔案
sudo gitlab-rake gitlab:cleanup:builds
```

對於大型倉庫，可以啟用 Git 物件池減少磁碟佔用，或者設定 Git LFS 儲存大型檔案。合理設定 CI/CD 快取可以加快建置速度。

## 安全問題

### 如何設定 SSL 憑證？

GitLab 支援 Let's Encrypt 自動設定 SSL，也可以使用自訂憑證。首先編輯設定檔：

```ruby
# Let's Encrypt 自動設定
letsencrypt['enable'] = true
external_url 'https://gitlab.example.com'

# 自訂憑證
nginx['ssl_certificate'] = "/etc/gitlab/ssl/server.crt"
nginx['ssl_certificate_key'] = "/etc/gitlab/ssl/server.key"
```

設定後執行 sudo gitlab-ctl reconfigure 使設定生效。如果使用自簽名憑證，用戶端需要信任該憑證才能透過 HTTPS 存取。

### 如何備份和復原 GitLab？

定期備份是防止資料遺失的重要措施。GitLab 提供內建備份工具：

```bash
# 建立備份
sudo gitlab-backup create

# 備份檔案位置
ls -la /var/opt/gitlab/backups/
```

備份檔案包含資料庫、上傳檔案和 Git 倉庫。復原時，停止 GitLab 服務，複製備份檔案到預設位置，然後執行復原命令：

```bash
sudo gitlab-ctl stop
sudo gitlab-backup restore BACKUP=timestamp
sudo gitlab-ctl reconfigure
sudo gitlab-ctl start
```

建議將備份檔案自動同步到遠端儲存，如物件儲存服務或另一台伺服器。

## 升級問題

### 升級後出現錯誤怎麼辦？

升級後可能出現相容性錯誤或功能異常。首先檢查 GitLab 狀態：

```bash
sudo gitlab-ctl status
sudo gitlab-rake gitlab:check
```

查看升級日誌了解具體錯誤：

```bash
sudo tail -100 /var/log/gitlab/gitlab-rails/production.log
```

常見問題包括：資料庫遷移失敗、設定檔語法錯誤、依賴缺失等。如果是資料庫遷移問題，可能需要手動修復：

```bash
sudo gitlab-rake db:migrate
```

建議在升級前備份資料，並仔細閱讀官方升級文件，了解版本間的重大變更。

### 如何在不同版本間安全升級？

升級 GitLab 應遵循版本跳躍限制，每次升級跳躍不超過一個主版本。例如從 15.x 升級到 16.x，而不是直接從 15.x 升級到 17.x。先升級到中間版本，確認正常運行後再繼續升級。

升級前執行以下步驟：備份資料、檢查相容性問題、閱讀發布說明。升級過程中保持電源和網路穩定，升級完成後驗證所有功能正常。
