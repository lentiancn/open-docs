# Jenkins 설치 가이드

## 시스템 요구사항

Jenkins를 설치하기 전에 시스템이 다음 최소 요구사항을 충족하는지 확인하세요:

### 최소 권장 구성

- 256MB 사용 가능 메모리
- 1GB 사용 가능 디스크 공간(Docker 컨테이너로 실행 시 10GB 권장)

### 소규모 팀 권장 구성

- 1GB+ 사용 가능 메모리
- 50GB+ 사용 가능 디스크 공간

### 소프트웨어 요구사항

- Java 8 이상(JRE 또는 JDK 모두 가능)
- 참고: Docker 컨테이너로 Jenkins를 실행하는 경우 호스트에 Java를 설치할 필요가 없습니다

## 설치 플랫폼

Jenkins는 여러 설치 방법을 지원합니다. 이 가이드에서는 가장 일반적인 접근 방식을 설명합니다.

## Docker 설치

Docker는 격리된 환경을 제공하고 구성 프로세스를 단순화하므로 Jenkins를 설치하는 권장 방법입니다.

### Docker 설치

먼저 시스템에 Docker를 설치해야 합니다. Docker 공식 웹사이트를 방문하여 운영 체제에 맞는 적절한 Docker Community Edition 버전을 선택하여 설치합니다.

### Jenkins 다운로드 및 실행

macOS 및 Linux에서 터미널을 열고 다음 명령을 실행합니다:

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

Windows에서 명령 프롬프트를 열고 유사한 명령을 실행합니다:

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

매개변수 설명:

- `-u root`: 루트 사용자로 컨테이너 실행
- `--rm`: 중지 시 컨테이너 자동 삭제
- `-d`: 백그라운드에서 실행(분리 모드)
- `-p 8080:8080`: 컨테이너 포트 8080을 호스트 포트 8080에 매핑
- `-p 50000:50000`: 컨테이너 포트 50000을 호스트 포트 50000에 매핑(JNLP 에이전트 통신용)
- `-v jenkins-data:/var/jenkins_home`: Jenkins 데이터 영구 저장
- `-v /var/run/docker.sock:/var/run/docker.sock`: 파이프라인에서 Docker 명령 실행 허용
- `jenkinsci/blueocean`: Blue Ocean 인터페이스가 포함된 공식 Jenkins 이미지

### Jenkins 컨테이너 접근

Jenkins 컨테이너 내부에 접근하려면 다음을 사용합니다:

```bash
docker exec -it jenkins-blueocean bash
```

### Jenkins 로그 보기

Jenkins 콘솔 로그를 보려면:

```bash
docker logs jenkins-blueocean
```

## WAR 파일 설치

Jenkins WAR 파일 버전은 Java를 지원하는 모든 운영 체제에 설치할 수 있습니다.

### 설치 단계

1. 공식 웹사이트에서 최신 안정版 Jenkins WAR 패키지를 다운로드
2. 다운로드 디렉토리에서 터미널 열기
3. 명령 실행: `java -jar jenkins.war`
4. http://localhost:8080으로 이동하여 Jenkins 잠금 해제 페이지가 나타날 때까지 기다리기

### 참고 사항

- WAR 파일 설치는 Blue Ocean을 자동으로 설치하지 않습니다. 플러그인 관리를 통해 별도로 설치해야 합니다
- `--httpPort`를 사용하여 포트를 변경할 수 있습니다(예: `java -jar jenkins.war --httpPort=9090`)

## macOS 설치

### 패키지 사용

1. macOS용 Jenkins 패키지 다운로드
2. 더블 클릭하여 설치 마법사 따르기

### Homebrew 사용

최신 버전 설치:

```bash
brew install jenkins
```

LTS 버전 설치:

```bash
brew install jenkins-lts
```

## Linux 설치(Debian/Ubuntu)

Debian 기반 배포판에서는 apt를 통해 Jenkins를 설치할 수 있습니다:

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

설치 후 Jenkins는 다음을 수행합니다:

- 부팅 시 데몬으로 시작하도록 설정
- 서비스를 실행할 jenkins 사용자 생성
- 로그를 /var/log/jenkins/jenkins.log에 출력
- 기본적으로 포트 8080에서 수신

## 설치 후 설정 마법사

Jenkins를 처음 시작하면 설정 마법사가 시작됩니다.

### Jenkins 잠금 해제

1. http://localhost:8080으로 이동
2. Jenkins 잠금 해제 페이지가 나타나면 콘솔 로그에서 자동으로 생성된 비밀번호를 복사
3. 비밀번호를 관리자 비밀번호 필드에 붙여넣고 계속 클릭

### Jenkins 플러그인 사용자 지정

플러그인 설치 페이지에서 다음을 선택할 수 있습니다:

- 권장 플러그인 설치: 일반적인 사용 사례를 기반으로 권장되는 플러그인 세트 설치
- 설치할 플러그인 선택: 설치할 플러그인을 수동으로 선택

### 첫 번째 관리자 사용자 만들기

다음과 같이 관리자 계정을 만듭니다:

- 사용자 이름
- 비밀번호
- 비밀번호 확인
- 전체 이름
- 이메일 주소

저장 및 완료를 클릭한 다음 Jenkins 사용 시작을 클릭하여 주 인터페이스에 액세스합니다.

## 설치 확인

다음을 통해 설치를 확인합니다:

1. http://localhost/8080으로 이동
2. 만든 관리자 계정으로 로그인
3. 시스템 상태가 정상인지 확인

## 일반적인 문제

### 포트가 이미 사용 중

포트 8080이 다른 프로그램에서 사용 중인 경우:

- Docker 방법: -p 매개변수 수정
- WAR 방법: --httpPort 매개변수 사용

### 메모리 부족

시스템 메모리가 제한적인 경우:

- 컨테이너 리소스 제한과 함께 Docker 사용
- Jenkins 시작 매개변수에서 JVM 힙 크기 조정

### 첫 번째 시작이 느림

첫 번째 시작은 플러그인을 다운로드하고 설치하므로 시간이 걸릴 수 있습니다.辛抱強くお待ちください。
