# Hadoop User Manual

## HDFS Operations Guide

### Basic File System Operations

HDFS provides a command-line interface similar to traditional file systems, allowing users to perform various file operations through the hdfs dfs command. These commands are very similar to Linux shell commands, so users familiar with Linux systems can quickly get started. Understanding these basic operations is fundamental to using Hadoop; let's detail commonly used HDFS commands below.

Creating directories uses the mkdir command, which can create single or multiple levels of directories:

```bash
# Create single-level directory
hdfs dfs -mkdir /user/data

# Create multi-level directory
hdfs dfs -mkdir -p /user/data/input

# List directory contents
hdfs dfs -ls /user/data

# Recursively list directory contents
hdfs dfs -ls -R /user/data
```

File upload and download are common operations:

```bash
# Upload local file to HDFS
hdfs dfs -put localfile.txt /user/data/

# Upload and rename
hdfs dfs -put localfile.txt /user/data/newname.txt

# Download HDFS file to local
hdfs dfs -get /user/data/hdfsfile.txt ./localdir/

# Copy HDFS internal files
hdfs dfs -cp /user/data/file1.txt /user/data/file2.txt

# Move HDFS internal files
hdfs dfs -mv /user/data/file1.txt /user/data/file2.txt
```

File viewing and deletion:

```bash
# View file contents
hdfs dfs -cat /user/data/file.txt

# View last few lines of file
hdfs dfs -tail /user/data/file.txt

# View file size
hdfs dfs -du /user/data/file.txt

# Delete file
hdfs dfs -rm /user/data/file.txt

# Recursively delete directory
hdfs dfs -rm -r /user/data/dir
```

### Permission Management

HDFS's permission model is similar to Unix/Linux, based on a three-tier system of user, group, and others. Each file and directory has three permissions—read (r), write (w), and execute (x)—corresponding to owner, group, and others. Understanding permission management is crucial for protecting data security.

```bash
# Change file owner
hdfs dfs -chown user:group /user/data/file.txt

# Change file permissions
hdfs dfs -chmod 755 /user/data/file.txt

# Change group ownership
hdfs dfs -chgrp groupname /user/data/file.txt
```

### Data Replication and Balancing

HDFS by default replicates each data block three times, storing them on different DataNodes. The default replication factor can be modified via the dfs.replication property:

```bash
# Set replication factor for specific file
hdfs dfs -setrep -w 2 /user/data/file.txt

# View block location information for file
hdfs fsck /user/data/file.txt -files -blocks -locations
```

When nodes are added to or removed from the cluster, data may become unbalanced. Use the rebalance command to balance data:

```bash
start-balancer.sh -threshold 10
```

The threshold parameter represents the maximum percentage difference between node disk usage and the cluster average. The balancing process consumes network bandwidth; it's recommended to run during low-traffic periods.

### HDFS Web UI

HDFS provides a feature-rich web interface accessible through the browser on port 9870 of the NameNode. The web UI can browse file system directories, view file contents, monitor DataNode status, view block information, and more. The web UI is an important tool for operations personnel in daily monitoring and troubleshooting.

Main features of the web UI include: browsing the HDFS directory tree, viewing file lists and file properties; viewing the DataNode list and storage usage for each DataNode; viewing block distribution and replication status; viewing NameNode runtime information and logs; performing simple file operations like download and deletion.

## MapReduce Programming

### WordCount Example

WordCount is the entry-level example for MapReduce programming, counting occurrences of each word in text. Understanding this example helps master the MapReduce programming model. The program is divided into Map and Reduce phases—the Map phase splits text into words, and the Reduce phase aggregates counts for the same words.

Map phase code example:

```java
public static class TokenizerMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
        throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}
```

Reduce phase code example:

```java
public static class IntSumReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
        Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### Submitting Jobs

After packaging MapReduce into a JAR file, submit the job using the hadoop jar command:

```bash
hadoop jar myapp.jar com.example.WordCount /input /output
```

After job submission, you can view job progress and results on YARN's Web UI (port 8088). When the job completes, it outputs job statistics including job ID, number of tasks, execution time, etc.

### Job Configuration

Various job parameters can be configured through the Configuration object:

```java
Configuration conf = new Configuration();
Job job = Job.getInstance(conf, "word count");
job.setJarByClass(WordCount.class);
job.setMapperClass(TokenizerMapper.class);
job.setReducerClass(IntSumReducer.class);
job.setOutputKeyClass(Text.class);
job.setOutputValueClass(IntWritable.class);
FileInputFormat.addInputPath(job, new Path(args[0]));
FileOutputFormat.setOutputPath(job, new Path(args[1]));
```

Commonly used configuration parameters include: mapreduce.job.reduces sets the number of Reduce tasks; mapreduce.map.memory.mb and mapreduce.reduce.memory.mb set container memory; mapreduce.map.cpu.vcores and mapreduce.reduce.cpu.vcores set container CPU cores.

## YARN Operations

### Viewing Cluster Resources

YARN provides rich command-line tools for managing cluster resources:

```bash
# View all node status
yarn node -list -all

# View application list
yarn application -list

# View specific application status
yarn application -status application_1234567890123_0001

# View application logs
yarn logs -applicationId application_1234567890123_0001
```

### Queue Management

YARN uses queues to manage resource allocation. Queues can be configured through Capacity Scheduler or Fair Scheduler:

```bash
# View queue information
yarn queue -status default

# View queue capacity
yarn resourcemanager -refreshQueues
```

### Resource Scheduler Configuration

Capacity Scheduler is the default scheduler, supporting multi-tenant resource sharing. Configure in yarn-site.xml:

```xml
<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>30</value>
</property>
<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>30</value>
</property>
```

## Performance Optimization

### MapReduce Tuning

MapReduce job performance tuning involves multiple aspects. First, reasonably set the number of Map and Reduce tasks. Map task count is usually determined by the number of input data splits, which can be adjusted via the mapreduce.input.fileinputformat.split.maxsize parameter. Reduce task count is recommended to be 0.9 to 1.2 times the total number of cluster Reduce slots.

```bash
# Set Map task count
-D mapreduce.job.maps=10

# Set Reduce task count
-D mapreduce.job.reduces=5
```

Shuffle phase optimization significantly impacts overall performance. Adjust these parameters:

```xml
<property>
    <name>mapreduce.reduce.shuffle.parallelcopies</name>
    <value>10</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
    <value>0.7</value>
</property>
<property>
    <name>mapreduce.reduce.shuffle.merge.percent</name>
    <value>0.66</value>
</property>
```

### HDFS Tuning

HDFS performance optimization mainly focuses on data read/write efficiency. For large files, adjust block size to balance data locality and fault recovery:

```xml
<property>
    <name>dfs.blocksize</name>
    <value>268435456</value>  <!-- 256MB -->
</property>
```

Enable short-circuit reads to allow clients to directly read data from local DataNodes, reducing network transfer:

```xml
<property>
    <name>dfs.client.read.shortcircuit</name>
    <value>true</value>
</property>
```

### YARN Tuning

YARN's resource allocation significantly impacts cluster utilization. Reasonably set NodeManager resource capacity:

```xml
<property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>32768</value>  <!-- 32GB -->
</property>
<property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>16</value>
</property>
```

Set virtual CPU cores to better utilize multi-core processors:

```xml
<property>
    <name>yarn.nodemanager.resource.detect-hardware-capabilities</name>
    <value>true</value>
</property>
```

## Data Import and Export

### Importing Data from Databases

Use Sqoop to conveniently migrate data between relational databases and HDFS:

```bash
# Import entire table
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --target-dir /user/hive/warehouse/employees

# Import and create Hive table
sqoop import \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --hive-table employees \
    --hive-import
```

### Exporting Data to Databases

```bash
sqoop export \
    --connect jdbc:mysql://localhost:3306/mydb \
    --username root \
    --password password \
    --table employees \
    --export-dir /user/hive/warehouse/employees \
    --input-fields-terminated-by '\t'
```

### Choosing Data Formats

Different data formats have different performance impacts. Commonly used formats include: TextFile (good generality, but average performance); SequenceFile (good for MapReduce); Parquet (columnar storage, good for analytical queries); ORC (Hive-optimized columnar storage). For analytical workloads, use columnar storage formats like Parquet or ORC.
