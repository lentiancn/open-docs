# 常見問題

本文檔匯總了 Angular 使用過程中常見的技術問題和解答。

## 基礎問題

### 什麼是 Angular？

Angular 是由 Google 開發維護的開源 Web 應用框架，用於建構現代 Web 應用程式。它採用 TypeScript 作為主要開發語言，提供元件化架構、雙向資料綁定、依賴注入等特性。

### Angular 和 AngularJS 有什麼區別？

AngularJS 是 Angular 1.x 的名稱，是 Angular 的前身。Angular 是完全重寫的版本，使用 TypeScript，架構完全不同，不相容 AngularJS。

### Angular 難學嗎？

Angular 有一定的學習曲線，特別是依賴注入、RxJS 等概念需要時間理解。但它的文件完善，社群活躍，上手後開發效率很高。

## 安裝問題

### npm install 失敗怎麼辦？

1. 清除 npm 快取：`npm cache clean --force`
2. 刪除 node_modules 和 package-lock.json
3. 使用國內鏡像
4. 檢查 Node.js 版本是否相容

### Angular CLI 命令找不到？

確保已全域安裝：
```bash
npm install -g @angular/cli
```

如果還是不行，嘗試使用 npx：
```bash
npx @angular/cli new my-app
```

### Node.js 版本不相容？

使用 nvm 管理 Node.js 版本：
```bash
nvm install 18
nvm use 18
```

## 開發問題

### 元件樣式不生效？

1. 檢查樣式檔案路徑是否正確
2. 確認 styles.scss 中沒有語法錯誤
3. 檢查 ViewEncapsulation 設定

### 路由跳轉失敗？

1. 檢查路由配置是否正確
2. 確認 RouterModule 已正確匯入
3. 檢查是否有路由守衛阻止跳轉

### HTTP 請求失敗？

1. 檢查後端服務是否啟動
2. 確認 CORS 配置
3. 使用 HttpInterceptor 新增錯誤處理

### 資料不更新？

1. 檢查變更檢測策略
2. 如果使用 OnPush，需要手動觸發檢測
3. 確保資料參考發生變化

## 效能問題

### 首屏載入慢？

1. 啟用路由懶載入
2. 使用預載入策略
3. 優化圖片資源
4. 啟用生產模式建構

### 記憶體洩漏？

1. 在 ngOnDestroy 中取消訂閱 Observable
2. 使用 takeUntil 或 async 管道
3. 清理計時器和事件監聽器

### 變更檢測頻繁？

1. 使用 OnPush 策略
2. 分離變化和不變化的屬性
3. 使用 trackBy 優化 *ngFor

## 測試問題

### 單元測試失敗？

1. 檢查 TestBed 配置
2. 確認非同步操作已正確處理
3. 使用 fakeAsync 和 tick 測試非同步程式碼

### 找不到模組？

確保在 tsconfig.json 中正確配置了 paths。

## 建構問題

### 建構失敗？

1. 查看具體錯誤訊息
2. 檢查 TypeScript 編譯選項
3. 清除快取重新建構

### 建構檔案太大？

1. 啟用 production 模式
2. 使用懶載入
3. 檢查是否有未使用的相依套件

## 常見錯誤

### ExpressionChangedAfterItHasBeenCheckedError

模板中修改了檢測期間不應改變的值。檢查 ngOnInit 中是否有直接修改視圖相關的程式碼。

### NG0100: ExpressionChangedAfterItHasBeenCheckedError

同上個錯誤，檢查資料綁定的時序。

### Cannot read property of undefined

資料未初始化就存取。使用可選鏈（?.）或 *ngIf 保護。

### CORS 錯誤

後端未配置 CORS，或前端請求頭有問題。

### 401/403 錯誤

身份驗證失敗，檢查 token 和權限配置。

## 版本升級

### 如何升級 Angular？

1. 使用 CLI 升級：`ng update @angular/core`
2. 查閱官方升級指南
3. 逐個升級相依套件
4. 執行測試確保正常

### 升級失敗怎麼辦？

1. 備份程式碼
2. 手動更新 package.json 中的版本
3. 刪除 node_modules 重新安裝
