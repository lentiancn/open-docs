# Nacos Installationsanleitung

## Umgebungsanforderungen

Nacos erfordert eine Java-Laufzeitumgebung. Bitte stellen Sie sicher, dass Sie die folgenden Versionen verwenden:

- 64-Bit-Betriebssystem: Linux/Unix/Mac/Windows unterstützt, Linux/Unix/Mac empfohlen
- 64-Bit JDK 17+ (JDK 17 oder höher empfohlen)
- JDK-Download: https://www.oracle.com/java/technologies/downloads/#java17
- JDK-Umgebungskonfiguration: https://docs.oracle.com/cd/E19182-01/820-7851/inst_cli_jdk_javahome_t/

## Empfohlene Konfiguration

Es wird empfohlen, Nacos mindestens mit folgender Maschinenkonfiguration auszuführen:

- CPU: 2 Kerne
- Speicher: 4 GB
- Festplatte: 60 GB

## Installationsmethoden

Nacos unterstützt mehrere Installationsmethoden. Sie können die geeignete Methode basierend auf Ihren Anforderungen auswählen.

### Methode 1: Ein-Klick-Bereitstellung mit nacos-setup (Empfohlen)

nacos-setup bietet eine automatisierte Ein-Klick-Bereitstellung und übernimmt automatisch Download, Authentifizierungskonfiguration, Porterkennung und Java-Umgebungsverifizierung.

#### nacos-setup installieren

Linux/macOS:

```bash
curl -fsSL https://nacos.io/nacos-installer.sh | sudo bash
```

Windows (PowerShell):

```powershell
# Ausführen mit PowerShell
```

#### Nacos-Dienst starten

Die neueste Version von Standalone-Nacos lokal mit einem Klick bereitstellen:

```bash
nacos-setup
```

Eine bestimmte Version von Standalone-Nacos lokal bereitstellen:

```bash
nacos-setup -v 3.1.1
```

Die neueste Version von Cluster-Nacos lokal bereitstellen:

```bash
nacos-setup -c cluster
```

Externen Speicher global konfigurieren, Datenbankadresse, Port, Benutzername und Passwort eingeben, dann nacos-setup erneut ausführen:

```bash
nacos-setup --datasource-conf
```

Weitere Verwendung finden Sie im nacos-setup-Benutzerhandbuch.

### Methode 2: Manuelle Download- und Bereitstellung

#### Installationspaket herunterladen

Sie können das Nacos-Distributionspaket auf zwei Arten erhalten:

**Methode 1: Von der Nacos-Website herunterladen**

Besuchen Sie die Nacos-Versions-Downloadseite, wählen Sie eine stabile Version und klicken Sie dann auf die ZIP-Datei in der Binärpaket-Spalte zum Herunterladen.

**Methode 2: Von GitHub herunterladen**

Besuchen Sie die Nacos GitHub Releases-Seite und laden Sie das Distributionspaket für die entsprechende Version herunter.

Hinweis: Wenn viele Benutzer gleichzeitig herunterladen, können Download-Drosselungsfehler auftreten. Warten Sie in diesem Fall und versuchen Sie es erneut oder laden Sie von GitHub herunter.

#### Nacos-Distributionspaket entpacken

Linux/Unix/Mac:

```bash
unzip nacos-server-$version.zip
# oder
tar -xvf nacos-server-$version.tar.gz
cd nacos/bin
```

Windows:

Verwenden Sie Extraktionswerkzeuge, um nacos-server-$version.zip direkt zu entpacken.

#### Server starten

Linux/Unix/Mac-Startbefehl (standalone bedeutet Nicht-Cluster-Modus):

```bash
sh startup.sh -m standalone
```

Wenn Sie Ubuntu verwenden oder Fehlermeldungen über fehlende Symbole erhalten, versuchen Sie:

```bash
bash startup.sh -m standalone
```

Das Startprogramm fordert Sie auf, drei authentifizierungsbezogene Konfigurationen einzugeben (ab Nacos 3.0.0 ist die Konsolenauthentifizierung standardmäßig aktiviert, daher müssen die folgenden drei Authentifizierungskonfigurationen ausgefüllt werden):

```
nacos.core.auth.plugin.nacos.token.secret.key fehlt, bitte mit Base64-String setzen: ${your_input_token_secret_key}
nacos.core.auth.server.identity.key fehlt, bitte setzen: ${your_input_server_identity_key}
nacos.core.auth.server.identity.value fehlt, bitte setzen: ${your_input_server_identity_key}
```

Windows-Startbefehl:

```bash
cmd startup.cmd
```

Oder doppelklicken Sie auf startup.cmd zum Ausführen.

### Methode 3: Docker-Installation

#### Nacos-Image abrufen

```bash
docker pull nacos/nacos-server
```

#### Standalone-Modus ausführen

```bash
docker run --name nacos -e MODE=standalone -p 8848:8848 -p 9848:9848 nacos/nacos-server
```

#### Cluster-Modus ausführen

Erfordert Cluster-Bereitstellung mit Docker Compose oder Kubernetes.

### Methode 4: Kubernetes-Installation

Nacos unterstützt die Bereitstellung in Kubernetes-Umgebungen und kann mit Helm Chart oder Operator installiert werden. Spezifische Schritte finden Sie in der Nacos Kubernetes-Bereitstellungsdokumentation.

## Nacos-Dienststart überprüfen

Navigieren Sie zum Nacos-Logverzeichnis und verwenden Sie `tail -f start.out` um Logs anzuzeigen. Wenn Sie die folgende Nachricht sehen, wurde der Dienst erfolgreich gestartet:

```
Nacos started successfully in stand alone mode. use embedded storage
```

## Schnellüberprüfung

### Nacos-Konsole

Öffnen Sie einen beliebigen Browser und geben Sie ein: http://127.0.0.1:8080/index.html um auf die Nacos-Konsole zuzugreifen.

Hinweis: Beim ersten Öffnen müssen Sie das Administratorkennwort für den Benutzer nacos initialisieren. Standard-Benutzername/-passwort: nacos/nacos@demo.

### Service-Registrierung

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/client/ns/instance?serviceName=quickstart.test.service&ip=127.0.0.1&port=8080'
```

### Service-Entdeckung

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/ns/instance/list?serviceName=quickstart.test.service'
```

### Konfiguration veröffentlichen

Zuerst anmelden, um Access-Token zu erhalten:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/auth/user/login' -d 'username=nacos' -d 'password=${your_password}'
```

Access-Token verwenden, um Konfiguration zu erstellen:

```bash
curl -X POST 'http://127.0.0.1:8848/nacos/v3/admin/cs/config?dataId=quickstart.test.config&groupName=test&content=HelloWorld' -H "accessToken:${your_access_token}"
```

### Konfiguration abrufen

```bash
curl -X GET 'http://127.0.0.1:8848/nacos/v3/client/cs/config?dataId=quickstart.test.config&groupName=test'
```

## Server herunterfahren

### Linux/Unix/Mac

```bash
sh shutdown.sh
```

### Windows

```bash
shutdown.cmd
```

Oder doppelklicken Sie auf shutdown.cmd zum Ausführen.

## Versionsauswahl

Die aktuell empfohlene stabile Version für die Produktion ist 3.1.1. Sie können sich über die von jeder Version unterstützten Funktionen informieren unter:

- Nacos-Versions-Downloadseite
- Nacos Release Notes
- Nacos-Veröffentlichungsankündigungen

## Hinweise

1. Ab Nacos 3.0.0 ist die Konsolenauthentifizierung standardmäßig aktiviert. Authentifizierungsbezogene Parameter müssen beim ersten Start konfiguriert werden.
2. Cluster-Modus-Bereitstellung erfordert externe Datenbank (wie MySQL) zur Datenspeicherung.
3. Produktionsumgebungen sollten den Cluster-Modus verwenden, um Hochverfügbarkeit zu gewährleisten.
4. Stellen Sie sicher, dass erforderliche Ports in der Firewall geöffnet sind (standardmäßig 8848, 9848).
5. JDK 17 oder höher wird für optimale Leistung empfohlen.
