# 常见问题

本文档汇总了 Apache APISIX 使用过程中常见的技术问题和解答。

## 基础问题

### 什么是 Apache APISIX？

Apache APISIX 是 Apache 软件基金会下的顶级项目，是一个具有动态、实时、高性能等特点的云原生 API 网关。它提供了动态路由、动态上游、动态证书、A/B 测试、灰度发布、限速、防攻击、监控报警等功能。

### APISIX 和 Nginx 有什么区别？

Nginx 是一个传统的 Web 服务器和反向代理，而 APISIX 是基于 Nginx 构建的 API 网关。APISIX 在 Nginx 基础上增加了完全动态的配置管理、丰富的插件系统、精细的流量管理和内置的可观测性功能。

### APISIX 的性能如何？

APISIX 继承了 Nginx 的高性能特性，同时通过优化和插件机制保持了高效的处理能力。具体性能取决于硬件配置和业务复杂度，一般情况下可以处理每秒数万甚至数十万请求。

## 安装问题

### Docker 启动失败怎么办？

1. 检查 Docker 是否正常运行：`docker ps`
2. 查看容器日志：`docker-compose logs apisix`
3. 常见问题：
   - 端口冲突：检查 9080、9180、9443 端口是否被占用
   - etcd 连接失败：确认 etcd 容器是否正常启动
   - 权限问题：确保有足够的权限运行 Docker

### 如何查看 APISIX 版本？

```bash
curl "http://127.0.0.1:9080" --head | grep Server
```

返回格式：`Server: APISIX/Version`

### 安装后无法访问 Dashboard？

1. 确认 Dashboard 端口是否正确映射（默认 9180）
2. 检查防火墙设置
3. 确认容器网络配置是否正确

## 配置问题

### 如何修改管理 API 端口？

编辑 `apisix.yml` 配置文件：

```yaml
deployment:
  admin:
    listen:
      host: 0.0.0.0
      port: 9180
```

### 如何启用 HTTPS？

1. 配置 SSL 证书：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/ssl" -X PUT -d '
{
  "cert": "证书内容",
  "key": "私钥内容",
  "snis": ["example.com"]
}'
```

2. 在路由中启用 HTTPS：

```json
{
  "plugins": {
    "proxy-rewrite": {
      "scheme": "https"
    }
  }
}
```

### 路由配置不生效？

1. 检查路由配置是否正确
2. 确认 etcd 配置是否正确同步
3. 检查插件是否有语法错误
4. 查看 APISIX 错误日志

### 如何实现跨域请求？

使用 `cors` 插件：

```json
{
  "plugins": {
    "cors": {
      "allow_origins": "*",
      "allow_methods": "GET,POST,PUT,DELETE",
      "allow_headers": "*"
    }
  }
}
```

## 性能问题

### 如何提高吞吐量？

1. **增加工作进程数：**`apisix.worker_processes: auto`
2. **启用 HTTP/2：**在 listen 配置中设置 `http2: true`
3. **优化上游连接池：**配置 `keepalive_pool.size`
4. **使用缓存插件：**启用 `proxy-cache` 插件

### 内存占用过高怎么办？

1. 减少不必要的插件加载
2. 限制上游连接数
3. 调整日志缓冲区大小
4. 使用流式响应处理大文件

## 功能问题

### 如何实现 A/B 测试？

使用 `traffic-split` 插件：

```json
{
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 30
        },
        {
          "weight": 70,
          "upstream_id": "upstream-v2"
        }
      ]
    }
  }
}
```

### 如何实现请求限流？

使用 `limit-req` 插件：

```json
{
  "plugins": {
    "limit-req": {
      "rate": 100,
      "burst": 50,
      "key": "remote_addr"
    }
  }
}
```

## 集成问题

### 如何与 Kubernetes 集成？

使用 APISIX Ingress Controller：

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
    - name: example
      match:
        paths:
          - /api/*
        methods:
          - GET
      backends:
        - serviceName: example-service
          servicePort: 8080
```

### 如何与 Prometheus 集成？

1. 启用 Prometheus 插件
2. 配置 Prometheus 抓取配置

## 安全问题

### 如何防止 APISIX 被滥用？

1. **启用身份验证**：在所有路由上启用 key-auth 或其他认证插件
2. **限制管理 API 访问**：只允许受信任的 IP 访问管理 API
3. **使用 HTTPS**：启用 TLS 加密
4. **配置限速**：防止恶意请求
5. **启用访问日志**：记录所有请求以便审计

### 管理 API 如何开启授权？

```yaml
deployment:
  admin:
    admin_key_required: true
```

## 运维问题

### 如何升级 APISIX？

**Docker 方式：**

```bash
docker-compose pull
docker-compose up -d
```

**Kubernetes 方式：**

```bash
helm upgrade apisix apisix/apisix
```

### 常见错误代码

- **401 Unauthorized**：身份验证失败
- **403 Forbidden**：没有访问权限
- **404 Not Found**：路由不存在
- **429 Too Many Requests**：请求超过限速
- **500 Internal Server Error**：上游服务错误
- **502 Bad Gateway**：上游服务不可用
- **503 Service Unavailable**：服务不可用
