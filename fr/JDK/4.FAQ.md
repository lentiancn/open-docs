# JDK Foire aux questions

## Questions de base

### Qu'est-ce que JDK

JDK (Java Development Kit) est le kit de développement logiciel pour le langage de programmation Java. C'est le package logiciel nécessaire pour développer des applications Java. JDK comprend le compilateur Java (javac), la machine virtuelle Java (JVM), les bibliothèques de classes Java de base et divers outils de développement. Sans JDK, vous ne pouvez pas développer ni exécuter des programmes Java.

La différence entre JDK et JRE est : JRE (Java Runtime Environment) contient uniquement l'environnement nécessaire pour exécuter des programmes Java, sans outils de développement. Si vous avez seulement besoin d'exécuter des programmes Java développés par d'autres, installez JRE ; si vous devez développer vous-même des programmes Java, vous devez installer JDK.

### Qu'est-ce que JVM

JVM (Java Virtual Machine) est le cœur de l'exécution des programmes Java. La JVM est responsable de l'interprétation ou de la compilation du bytecode Java (.class fichiers) pour l'exécution de code natif. L'existence de JVM permet aux programmes Java d'atteindre la capacité multiplateforme « Écrire une fois, exécuter partout ».

La JVM n'exécute pas seulement des programmes Java, mais gère également la gestion de la mémoire (collecte des ordures), la gestion des threads, la gestion de la sécurité, etc. Comprendre le fonctionnement de la JVM est essentiel pour écrire des applications Java haute performance.

### Quelle est la différence entre JDK et JRE

JDK (Java Development Kit) est le kit de développement complet, contenant JRE et outils de développement. JRE (Java Runtime Environment) est l'environnement d'exécution Java, ne contenant que les composants nécessaires pour exécuter des programmes Java.

En termes simples : si vous voulez seulement exécuter des programmes Java faits par d'autres, installer JRE suffit ; si vous voulez développer vous-même des programmes Java, vous devez installer JDK. JDK contient JRE complet, donc après avoir installé JDK, vous n'avez pas besoin d'installer JRE séparément.

### Quelle est la meilleure version de JDK

Il n'y a pas de version « meilleure » absolue ; le choix dépend des besoins spécifiques. Pour les nouveaux projets, JDK 21 (la dernière version LTS) ou JDK 25 (la dernière version) est recommandée. Pour les projets d'entreprise nécessitant une maintenance à long terme, les versions LTS comme JDK 21 ou JDK 17 sont recommandées.

JDK 21 est la dernière version LTS, recevra au moins 8 ans de support. JDK 17 est encore largement utilisé et gratuit. JDK 8, bien que ancien, est encore utilisé dans de nombreux systèmes hérités.

## Questions d'installation

### Comment vérifier si JDK est installé correctement

Exécutez java -version dans la ligne de commande. Si les informations de version s'affichent, JDK est installé correctement. Exécutez javac -version pour vérifier si le compilateur est disponible. Exécutez echo %JAVA_HOME% (Windows) ou echo $JAVA_HOME (Linux/macOS) pour vérifier si les variables d'environnement sont correctement configurées.

Si ces commandes montrent des erreurs dans la ligne de commande, vérifiez d'abord si JDK est installé correctement, puis vérifiez si les variables d'environnement sont configurées correctement.

### Qu'est-ce que JAVA_HOME

JAVA_HOME est une variable d'environnement pointant vers le répertoire d'installation de JDK. De nombreux outils Java et scripts de construction (comme Maven, Gradle) s'appuient sur JAVA_HOME pour localiser JDK. Configurer correctement JAVA_HOME garantit que ces outils peuvent fonctionner correctement.

Sous Windows, JAVA_HOME est généralement défini sur des chemins comme C:\Program Files\Java\jdk-21. Sous Linux/macOS, il est généralement défini sur des chemins comme /usr/lib/jvm/java-21-openjdk.

### Comment installer plusieurs versions de JDK sous Windows

Vous pouvez installer plusieurs versions de JDK sur le même ordinateur. Après avoir installé plusieurs versions, changez la version par défaut en modifiant l'ordre des répertoires bin de JDK dans la variable d'environnement PATH, ou en modifiant la variable d'environnement JAVA_HOME pour pointer vers différents répertoires d'installation de JDK.

Vous pouvez également utiliser des outils tiers pour gérer plusieurs versions de JDK, comme la version Windows de SDKMAN! ou JetBrains Toolbox. Ces outils facilitent la commutation entre versions.

### Quelle est la différence entre Oracle JDK et OpenJDK

Oracle JDK et OpenJDK sont presque identiques au niveau du code, les principales différences étant les accords de licence et les services de support. Oracle JDK offre un support commercial et des fonctionnalités supplémentaires de niveau entreprise, mais certaines versions peuvent nécessiter un paiement pour une utilisation commerciale. OpenJDK est entièrement open source et gratuit, maintenu par la communauté.

Pour la plupart des cas d'utilisation, OpenJDK est parfaitement adéquat. De nombreuses organisations choisissent d'utiliser OpenJDK pour éviter les problèmes de licence, tout en obtenant des performances et une fonctionnalité presque identiques à Oracle JDK.

## Questions d'utilisation

### Qu'est-ce que classpath

Classpath est le chemin que la JVM utilise pour trouver les fichiers de classe. Lorsque vous exécutez java MyClass, la JVM recherchera My.class dans les répertoires et fichiers JAR spécifiés par classpath.

Lors de la compilation et de l'exécution de programmes Java, classpath peut être spécifié en utilisant les options -classpath ou -cp. Si non spécifié, la JVM utilise le répertoire actuel comme classpath.

### Qu'est-ce que le bytecode

Le bytecode est une forme intermédiaire de code compilé par le compilateur Java à partir du code source. Le bytecode n'est pas du code natif et ne peut pas s'exécuter directement sur le CPU ; il ne peut être interprété par la JVM ou compilé JIT pour l'exécution.

L'existence du bytecode donne aux programmes Java la capacité multiplateforme. Le même bytecode peut s'exécuter sur tout système avec JVM installé, que ce soit Windows, Linux ou macOS.

### Qu'est-ce qu'un fichier JAR

JAR (Java Archive) est un format pour emballer plusieurs fichiers de classe Java dans un seul fichier. Les fichiers JAR sont en fait des fichiers compressés au format ZIP contenant des fichiers .class compilés et des métadonnées.

L'outil jar peut être utilisé pour créer, voir et extraire des fichiers JAR. Les fichiers JAR exécutables contiennent un fichier MANIFEST.MF spécifiant la classe principale, et peuvent être exécutés directement avec java -jar.

### Qu'est-ce que le système de modules

Le système de modules Java (Project Jigsaw) a été introduit dans JDK 9, divisant JDK et applications en modules. Les modules offrent une meilleure encapsulation et déclarent clairement les dépendances, aidant à construire des applications plus robustes et maintenables.

Les modules sont déclarés via des fichiers module-info.java, définissant le nom du module, les dépendances d'autres modules, les packages exportés, etc. Le système de modules aide à résoudre les conflits de classpath et offre une meilleure gestion des versions.

## Questions de performances

### Comment optimiser les performances de JVM

L'optimisation des performances de JVM repose principalement sur : configurer correctement la taille de la mémoire du tas, éviter les paramètres trop grands ou trop petits ; choisir le nettoyeur de mémoire approprié, G1 est un bon choix pour la plupart des cas ; pour les applications cherchant une faible latence, envisagez d'utiliser ZGC ou Shenandoah ; activer le partage des données de classe (-Xshare:classes) pour accélérer le démarrage.

Utilisez des outils de surveillance des performances (comme VisualVM, JConsole) pour analyser les performances de l'application, trouver les goulots d'étranglement et effectuer une optimisation ciblée.

### Qu'est-ce que la collecte des ordures

La collecte des ordures (GC) est le mécanisme par lequel la JVM gère automatiquement la mémoire. Pendant l'exécution du programme, des objets sont créés et consomment de la mémoire. Lorsque les objets ne sont plus référencés, le nettoyeur de mémoire identifie et récupère automatiquement la mémoire occupée par ces objets.

La collecte des ordures de Java simplifie considérablement le travail de gestion de mémoire ; les programmeurs n'ont pas besoin de libérer explicitement la mémoire. Cependant, comprendre les principes de la collecte des ordures aide à écrire du code plus efficace, évitant les fuites de mémoire et les GC fréquentes.

### Qu'est-ce qu'une fuite de mémoire

Une fuite de mémoire fait référence à la mémoire qui ne peut pas être récupérée pendant l'exécution du programme, provoquant une diminution progressive de la mémoire disponible. En Java, les fuites de mémoire sont généralement causées par des collections (comme les Map, List statiques) qui ajoutent continuellement des objets sans les nettoyer à temps.

Pour enquêter sur les fuites de mémoire, vous pouvez utiliser jmap pour générer des fichiers de vidage de tas, puis utiliser des outils comme MAT (Memory Analyzer Tool) pour analyser et trouver les objets causant la fuite.

### Qu'est-ce que le compilateur JIT

Le compilateur JIT (Just-In-Time) fait partie de la JVM, compilant le bytecode chaud en code natif pour améliorer les performances d'exécution. Pendant l'exécution, la JVM analyse l'exécution du code ; pour le code fréquemment exécuté (points chauds), le compilateur JIT le compile en code machine natif, améliorant considérablement les performances.

La compilation JIT se produit au moment de l'exécution sans étapes de compilation supplémentaires. Le compilateur C1 compile rapidement mais avec moins d'optimisation, le compilateur C2 compile lentement mais avec plus d'optimisation. La compilation par niveaux combine les avantages des deux.

## Questions de versions

### Quelle version de Java dois-je utiliser

Pour les nouveaux projets, la dernière version LTS JDK 21 est recommandée. Elle offre de bonnes performances, des fonctionnalités riches et un support à long terme. Pour les projets nécessitant une compatibilité avec Java 8/11, ces versions peuvent être utilisées.

Il n'est pas recommandé d'utiliser des versions dont le support a été abandonné, comme JDK 8 (sauf si vous avez un abonnement payant). Mettre à niveau vers les nouvelles versions de JDK en temps opportun offre des mises à jour de sécurité et des améliorations de performances.

### Que signifie LTS dans les numéros de version Java

LTS (Long-Term Support) désigne les versions à support à long terme. Les versions LTS reçoivent au moins 8 ans de support technique et de mises à jour de correctifs de la part d'Oracle. Les versions LTS actuelles incluent JDK 8, JDK 11, JDK 17 et JDK 21.

Les versions non LTS (comme JDK 25) ont des cycles de support plus courts, généralement seulement 6 mois. Ces versions conviennent aux développeurs souhaitant les dernières fonctionnalités, mais les versions LTS sont recommandées pour les environnements de production.

### Puis-je encore utiliser Java 17

Java 17 est une version LTS mature et peut encore être utilisée normalement. Oracle a changé la licence de JDK 17 en octobre 2024. Cependant, selon la déclaration d'Oracle, JDK 17 peut encore être utilisé gratuitement pour le développement, l'utilisation personnelle et l'exécution de produits sous licence Oracle.

De nombreuses entreprises utilisent encore JDK 17 car c'est une version très mature et stable avec de nombreuses ressources de support communautaire. Si le projet utilise déjà JDK 17, il peut continuer à l'utiliser ; pour un nouveau projet, JDK 21 est plus recommandé.

## Questions de développement

### Comment compiler des fichiers Java

Utilisez la commande javac pour compiler des fichiers Java. Utilisation de base : javac MyClass.java. Après la compilation, un fichier de bytecode MyClass.class est généré.

Pour les projets, des outils de construction comme Maven ou Gradle sont généralement utilisés pour gérer le processus de compilation. Ces outils gèrent automatiquement les dépendances et les options de compilation.

### Comment exécuter des programmes Java

Utilisez la commande java pour exécuter des programmes Java compilés. Utilisation de base : java MyClass. Notez que l'extension .class n'est pas nécessaire ici.

Si le programme est packagé dans un fichier JAR, utilisez java -jar MyApp.jar pour exécuter.

### Comment configurer classpath

Spécifiez classpath à l'exécution en utilisant -classpath ou -cp : java -cp lib/*:out MyClass. Sous Windows, le séparateur est le point-virgule (;) ; sous Linux/macOS, c'est le deux-points (:).

Vous pouvez également configurer via la variable d'environnement CLASSPATH, mais la spécification en ligne de commande est plus flexible.

### Que sont Maven et Gradle

Maven et Gradle sont des outils de construction pour les projets Java. Ils peuvent automatiquement télécharger des dépendances, compiler du code, exécuter des tests, emballer pour la publication, simplifiant considérablement le processus de construction de projets.

Maven utilise XML (pom.xml) pour définir la configuration du projet, tandis que Gradle utilise des scripts Groovy ou Kotlin (build.gradle) pour définir la logique de construction. Ce sont tous deux des outils de construction largement utilisés dans les projets Java.
