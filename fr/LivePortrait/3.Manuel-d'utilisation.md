# Manuel d'utilisation de LivePortrait

Ce document fournit des instructions d'utilisation détaillées pour LivePortrait, incluant l'inférence de base, l'interface Gradio, le mode Animal, l'édition vidéo et d'autres fonctionnalités.

## Inférence de Base

### Pilotage par Image

L'utilisation la plus simple consiste à utiliser une image statique comme source et une vidéo de pilotage comme référence d'animation :

```bash
python inference.py
```

Par défaut, le script utilise des images d'exemple du répertoire assets/examples/source et des vidéos de pilotage d'exemple du répertoire assets/examples/driving. En cas d'exécution réussie, les résultats sont enregistrés dans le répertoire animations avec le nom "fichier_source--fichier_pilotage_concat.mp4".

La vidéo de sortie contient trois parties : la vidéo de pilotage, l'image/vidéo source et le résultat d'animation généré, concaténés en séquence.

### Entrées Personnalisées

Vous pouvez spécifier vos propres fichiers source et de pilotage en utilisant des paramètres de ligne de commande :

```bash
# Le fichier source est une image
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4

# Le fichier source est une vidéo
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d0.mp4
```

Le paramètre -s spécifie le chemin de l'image ou vidéo source, tandis que le paramètre -d spécifie le chemin de la vidéo de pilotage.

### Voir Plus d'Options

Affichez tous les paramètres de ligne de commande disponibles :

```bash
python inference.py -h
```

Les paramètres incluent le chemin de sortie, la taille de l'image, la vitesse d'animation, la méthode d'assemblage et d'autres configurations détaillées.

## Utilisation de l'Interface Web Gradio

Pour les utilisateurs non familiarisés avec les interfaces en ligne de commande, LivePortrait offre une interface graphique basée sur Gradio.

### Lancer l'Interface en Mode Humain

```bash
python app.py
```

Après le lancement, ouvrez votre navigateur à l'URL affichée (généralement http://localhost:7860). Dans l'interface, téléchargez une image ou vidéo source, puis sélectionnez une vidéo de pilotage et cliquez sur le bouton générer pour obtenir le résultat d'animation.

Vous pouvez spécifier le port en utilisant le paramètre --server_port, générer un lien public en utilisant --share, et spécifier l'adresse du serveur en utilisant --server_name.

### Lancer l'Interface en Mode Animal

Sur les systèmes Linux avec GPU NVIDIA, vous pouvez lancer l'interface en mode Animal :

```bash
python app_animals.py
```

## Spécifications des Vidéos de Pilotage

Pour des résultats optimaux, les vidéos de pilotage doivent répondre aux exigences suivantes :

**Rapport d'Aspect** : Il est recommandé de recadrer en carré 1:1 (comme 512x512 ou 256x256 pixels). Vous pouvez également activer le recadrage automatique (--flag_crop_driving_video).

**Exigences de Contenu** : La vidéo de pilotage doit se concentrer sur la zone de la tête, similaire aux vidéos d'exemple. Les mouvements d'épaules doivent être minimisés.

**Exigence de la Première Image** : La première image de la vidéo de pilotage doit être un visage de face avec une expression neutre. Ceci est crucial pour assurer des effets d'animation précis.

### Recadrage Automatique

Si votre vidéo de pilotage ne répond pas aux exigences ci-dessus, vous pouvez utiliser la fonction de recadrage automatique :

```bash
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d13.mp4 --flag_crop_driving_video
```

Si les résultats du recadrage automatique ne sont pas satisfaisants, vous pouvez ajuster les paramètres --scale_crop_driving_video et --vy_ratio_crop_driving_video pour affiner l'échelle et le décalage.

## Utilisation des Modèles de Mouvement

Pour accélérer l'inférence et protéger la vie privée, LivePortrait prend en charge l'utilisation de fichiers de modèle de mouvement pré-générés (.pkl). Ces fichiers de modèle enregistrent les informations de mouvement des vidéos de pilotage et peuvent être réutilisés sans traiter la vidéo originale à chaque fois.

### Utilisation

```bash
# Animation de portrait
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d5.pkl

# Édition de vidéo de portrait
python inference.py -s assets/examples/source/s13.mp4 -d assets/examples/driving/d5.pkl
```

Les fichiers de modèle accélèrent non seulement l'inférence mais protègent également la vie privée — l'utilisation de modèles élimine le besoin de télécharger des vidéos de pilotage originales.

## Édition de Vidéo de Portrait (Video-to-Video)

LivePortrait prend en charge l'édition et la modification des expressions et des mouvements de tête des sujets dans les vidéos existantes.

### Utilisation de Base

```bash
# La source est une image, le pilotage est une vidéo
python inference.py -s assets/examples/source/s9.jpg -d assets/examples/driving/d0.mp4
```

### Ajustement des Paramètres d'Animation

**Intensité d'Expression** : Ajustez la force des changements d'expression en utilisant le paramètre --driving_multiplier. Les valeurs supérieures à 1 améliorent les changements d'expression, tandis que les valeurs inférieures à 1 les réduisent.

**Contrôle de l'Assemblage** : Utilisez le paramètre --flag_stitching pour contrôler si l'assemblage est activé. Désactiver l'assemblage peut altérer la présentation des régions faciales.

## Accélération de l'Inférence

### Accélération Torch Compile

LivePortrait prend en charge l'utilisation de la fonction torch.compile de PyTorch pour accélérer l'inférence. La première inférence déclenche un processus d'optimisation (environ une minute), après quoi la vitesse d'inférence s'améliore de 20 à 30 %. Les effets d'accélération peuvent varier selon la version de CUDA.

```bash
python app.py --flag_do_torch_compile
```

Note : torch.compile n'est pas pris en charge sur Windows et macOS.

### Test de Vitesse

Le projet fournit un script de test de vitesse d'inférence :

```bash
python speed.py
```

Les résultats du test affichent le temps de traitement de chaque module, vous aidant à identifier les goulots d'étranglement.

## Mode Animal

Le mode Animal prend en charge l'animation de portraits d'animaux de compagnie comme les chats et les chiens.

### Conditions d'Utilisation

Le mode Animal a été testé uniquement dans les environnements suivants : Système Linux avec GPU NVIDIA, Système Windows avec GPU NVIDIA.

### Exigences d'Installation

Avant d'utiliser le mode Animal, installez la dépendance X-Pose comme décrit dans le guide d'installation et téléchargez les poids du modèle Animal.

### Exécuter l'Inférence en Mode Animal

```bash
python inference_animals.py -s assets/examples/source/s39.jpg -d assets/examples/driving/wink.pkl --driving_multiplier 1.75 --no_flag_stitching
```

### Interface du Mode Animal

```bash
python app_animals.py
```

## Contrôle Régional

LivePortrait prend en charge le contrôle de l'animation pour des régions spécifiques d'images, permettant des ajustements d'effets plus raffinés.

### Activer le Contrôle Régional

Dans l'interface Gradio, vous pouvez trouver des options de contrôle régional pour sélectionner quelles régions faciales animer (comme les yeux, la bouche, la pose de la tête, etc.).

## Édition de Portrait

Dans l'interface Gradio, les utilisateurs peuvent éditer précisément divers attributs du portrait source, y compris l'expression, la direction du regard, l'angle de la tête, et plus encore.

## Protection de la Vie Privée

LivePortrait fournit plusieurs fonctionnalités de protection de la vie privée :

**Modèles de Mouvement** : Utilisez des fichiers de modèle .pkl au lieu de vidéos de pilotage originales pour éviter de télécharger du contenu sensible

**Recadrage Automatique** : Recadrez automatiquement les vidéos de pilotage pour se concentrer sur la tête, réduisant les informations d'arrière-plan

**Protection Vidéo** : La fonction de création de modèle convertit les vidéos en modèles de mouvement avant utilisation

## Format de Sortie

Le format de sortie par défaut est un fichier vidéo MP4 contenant trois parties : vidéo de pilotage, contenu source et résultat généré. Utilisez le paramètre --no_flag_concat pour ne sorties que la portion d'animation générée.

## Intégration des Outils Communautaires

LivePortrait est intégré par plusieurs projets communautaires :

**ComfyUI** : Plusieurs nœuds ComfyUI supportent LivePortrait avec des fonctionnalités d'aperçu en temps réel

**Stable Diffusion WebUI** : L'extension sd-webui-live-portrait intègre LivePortrait dans SD WebUI

**FaceFusion** : La version 3.0 intègre LivePortrait en tant que restaurateur d'expression et processeur d'éditeur de visage

**HuggingFace Spaces** : Expérimentez LivePortrait en ligne sans installation locale

## Meilleures Pratiques

Pour des effets d'animation optimaux, suivez ces meilleures pratiques :

Sélectionnez des images source de haute qualité : Les portraits de face nets produisent les meilleurs résultats

Utilisez des vidéos de pilotage professionnelles : Les vidéos avec des expressions de face et naturelles conduisent à de meilleurs effets

Contrôlez la longueur des vidéos de pilotage : Des vidéos de pilotage excessivement longues augmentent le temps de traitement et peuvent causer une dégradation de l'effet

Établissez des paramètres de manière raisonnable : Ajustez l'intensité de l'expression et d'autres paramètres en fonction des résultats réels

Maintenez un éclairage cohérent : Essayez de garder les conditions d'éclairage cohérentes entre les images source et les vidéos de pilotage
