# JIRA Benutzerhandbuch

Diese Anleitung beschreibt, wie Sie JIRA für Projektmanagement und Problemverfolgung verwenden.

## Schnellstart

### Projekt erstellen

1. Klicken Sie auf "Projekt erstellen"
2. Wählen Sie eine Projektvorlage
3. Geben Sie den Projektnamen und Schlüssel ein
4. Klicken Sie auf "Erstellen"

## Projekttypen

### Agile Projekte

```markdown
- Scrum-Projekte
- Kanban-Projekte
- Skaliertes Agile
```

### Klassische Projekte

```markdown
- Geschäftsprojekte
- Projektmanagement
- Aufgabenverfolgung
```

## Problemtypen

### Problem erstellen

```markdown
1. Klicken Sie auf "Problem erstellen"
2. Wählen Sie das Projekt
3. Wählen Sie den Problemtyp
4. Füllen Sie Titel und Beschreibung aus
5. Legen Sie Priorität und Komponenten fest
6. Weisen Sie einem Benutzer zu
```

### Problemtypen

- **Bug** - Softwarefehler
- **Story** - Benutzergeschichte
- **Task** - Aufgabe
- **Epic** - Epics
- **Sub-task** - Unteraufgabe

## Workflows

### Standard-Workflow

```
Erstellen → Zuweisen → Lösen → Schließen
```

### Agile Workflow

```
Backlog → In Bearbeitung → Fertig → Überprüft
```

## Agile Funktionen

### Kanban

1. Kanban erstellen
2. Spalten hinzufügen
3. Problemkarten ziehen
4. WIP-Limits festlegen

### Scrum

```markdown
- Sprint-Planung
- Daily Standup
- Sprint-Review
- Sprint-Retrospektive
```

## Felder

### Benutzerdefinierte Felder

1. Gehen Sie zu Administration → Felder
2. Benutzerdefiniertes Feld hinzufügen
3. Feldoptionen konfigurieren

### Feldkonfiguration

```markdown
- Pflichtfelder
- Standardwerte
- Feldrechte
```

## Berechtigungen

### Projektberechtigungen

- Projekt durchsuchen
- Probleme erstellen
- Probleme bearbeiten
- Probleme löschen
- Probleme zuweisen

### Problemsicherheitsstufen

- Öffentlich
- Projektmitglieder
- Bestimmte Benutzer
- Bestimmte Gruppen

## Automatisierung

### Regel erstellen

1. Gehen Sie zu Projekteinstellungen → Automatisierung
2. Regel erstellen
3. Trigger konfigurieren
4. Aktionen festlegen

### Automatisierungsbeispiel

```
Trigger: Problemerstellung
Bedingung: Typ = Bug
Aktion: Zuweisen an = Verantwortlicher
```
