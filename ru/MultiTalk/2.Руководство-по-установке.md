# Rukovodstvo po ustanovke

Protsess ustanovki MultiTalk trebuyet nakroyki sredy Python i zagruzki vesov modeley. Podrobnyye instruktsii po ustanovke predstavleny nizhe.

## Trebovaniya k srede

### Apparatnyye trebovaniya

MultiTalk - eto bol'shaya model' generatsii video s vysokimi apparatnymi trebovaniyami. Rekomenduyetsya ispol'zovat' GPU NVIDIA. Trebovaniya k VRAM varyiruyutsya v zavisimosti ot razresheniya vyvoda. Generatsiya video 480P trebuyet ne men'she 16GB VRAM, a generatsiya video 720P trebuyet konfiguratsii s neskol'kimi GPU. Proekt takzhe podderzhivayet rabotu v sredakh s nizkim VRAM, chto pozvolyayet generirovat' video 480P na odnom RTX 4090 blagodarya spetsificheskoy optimizatsii.

### Programmnyye trebovaniya

Podderzhivayetsya operatsionnaya sistema Linux. Trebuyetsya Python versiya 3.10 ili vyshe. Takzhe trebuyetsya podderzhka CUDA 12.1 ili vyshe. Proekt zavisit ot instrumentov glubinnogo obucheniya, takikh kak PyTorch 2.4.1 i xformers.

## Sozdaniye virtual'noy sredy

Snachala sozdayte novuyu virtual'nuyu sredu Python dlya izolyatsii zavisimostey proekta. Otkroyte terminal i vypolnite sleduyushchiye komandy:

```bash
conda create -n multitalk python=3.10
conda activate multitalk
```

## Ustanovka PyTorch i xformers

Ustanovite PyTorch i svyazannyye pakety v aktivnoy srede. Vypolnite sleduyushchuyu komandu dlya ustanovki PyTorch 2.4.1:

```bash
pip install torch==2.4.1 torchvision==0.19.1 torchaudio==2.4.1 --index-url https://download.pytorch.org/whl/cu121
```

Zatem ustanovite xformers:

```bash
pip install -U xformers==0.0.28 --index-url https://download.pytorch.org/whl/cu121
```

## Ustanovka zavisimostey proekta

Ustanovite drugiye neobkhodimyye pakety zavisimostey:

```bash
pip install misaki[en]
pip install ninja
pip install psutil
pip install packaging
pip install flash_attn==2.7.4.post1
```

## Ustanovka bibliotek obrabotki audio

Proekt trebuyet bibliotek, svyazannykh s obrabotkoy audio. Vypolnite sleduyushchiye komandy dlya ustanovki:

```bash
conda install -c conda-forge librosa
conda install -c conda-forge ffmpeg
```

Dlya sistem CentOS ili RHEL ispol'zuyte:

```bash
sudo yum install ffmpeg ffmpeg-devel
```

## Zagruzka vesov modeley

MultiTalk trebuyet zagruzki neskol'kikh faylov modeley dlya pravil'noy raboty. Osnovnyye modeli vklyuchayut:

Wan2.1-I2V-14B-480P - bazovaya model diffuzii, zagruzhaemaya s Hugging Face. chinese-wav2vec2-base - audioenkoder dlya obrabotki vvodnogo audio. Kokoro-82M - vesy TTS dlya funktsii tekst-v-rech'. MeiGen-MultiTalk - vesy usloviya audio, predostavlayemyye proektom.

Zagruzite modeli s pomoshch'yu huggingface-cli:

```bash
huggingface-cli download Wan-AI/Wan2.1-I2V-14B-480P --local-dir ./weights/Wan2.1-I2V-14B-480P
huggingface-cli download TencentGameMate/chinese-wav2vec2-base --local-dir ./weights/chinese-wav2vec2-base
huggingface-cli download hexgrad/Kokoro-82M --local-dir ./weights/Kokoro-82M
huggingface-cli download MeiGen-AI/MeiGen-MultiTalk --local-dir ./weights/MeiGen-MultiTalk
```

## Nakroyka ssylok na modeli

Posle zagruzki modeley nuzhno nakroit' ssylki. Yest' dva varianta.

Pervyy variant - ispol'zovaniye simvolicheskikh ssylok. Pereydite v katalog weights i vypolnite:

```bash
cd weights
mv Wan2.1-I2V-14B-480P/diffusion_pytorch_model.safetensors.index.json Wan2.1-I2V-14B-480P/diffusion_pytorch_model.safetensors.index.json_old
ln -s /path/to/weights/MeiGen-MultiTalk/diffusion_pytorch_model.safetensors.index.json Wan2.1-I2V-14B-480P/
ln -s /path/to/weights/MeiGen-MultiTalk/multitalk.safetensors Wan2.1-I2V-14B-480P/
```

Vtoroy variant - pryamoye kopirovaniye faylov:

```bash
cd weights
mv Wan2.1-I2V-14B-480P/diffusion_pytorch_model.safetensors.index.json Wan2.1-I2V-14B-480P/diffusion_pytorch_model.safetensors.index.json_old
cp MeiGen-MultiTalk/diffusion_pytorch_model.safetensors_index.json Wan2.1-I2V-14B-480P/
cp MeiGen-MultiTalk/multitalk.safetensors Wan2.1-I2V-14B-480P/
```
