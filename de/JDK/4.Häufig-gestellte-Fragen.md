# JDK Häufig gestellte Fragen

## Grundlegende Fragen

### Was ist JDK?

JDK (Java Development Kit) ist das Software Development Kit für die Java-Programmiersprache. Es ist das Softwarepaket, das für die Entwicklung von Java-Anwendungen erforderlich ist. JDK enthält den Java-Compiler (javac), die Java Virtual Machine (JVM), Java-Kernklassenbibliotheken und verschiedene Entwicklungstools. Ohne JDK können Sie keine Java-Programme entwickeln oder ausführen.

Der Unterschied zwischen JDK und JRE ist: JRE (Java Runtime Environment) enthält nur die Umgebung, die zum Ausführen von Java-Programmen erforderlich ist, ohne Entwicklungstools. Wenn Sie nur Java-Programme ausführen möchten, die andere entwickelt haben, installieren Sie JRE; wenn Sie selbst Java-Programme entwickeln möchten, müssen Sie JDK installieren.

### Was ist JVM?

JVM (Java Virtual Machine) ist der Kern der Java-Programmausführung. Die JVM ist dafür verantwortlich, Java-Bytecode (.class-Dateien) zu interpretieren oder für die Ausführung in nativen Code zu kompilieren. Das Vorhandensein der JVM ermöglicht es Java-Programmen, die Eigenschaft "Einmal schreiben, überall ausführen" zu erreichen.

Die JVM führt nicht nur Java-Programme aus, sondern übernimmt auch Speicherverwaltung (Garbage Collection), Thread-Verwaltung, Sicherheitsmanagement usw. Das Verständnis der Funktionsweise der JVM ist für das Schreiben leistungsstarker Java-Anwendungen unerlässlich.

### Was ist der Unterschied zwischen JDK und JRE?

JDK (Java Development Kit) ist das komplette Entwicklungspaket, das JRE und Entwicklungstools enthält. JRE (Java Runtime Environment) ist die Java-Laufzeitumgebung, die nur die Komponenten enthält, die zum Ausführen von Java-Programmen erforderlich sind.

Einfach ausgedrückt: Wenn Sie nur Java-Programme ausführen möchten, die andere entwickelt haben, reicht die Installation von JRE aus; wenn Sie selbst Java-Programme entwickeln möchten, müssen Sie JDK installieren. JDK enthält das vollständige JRE, sodass Sie nach der Installation von JDK JRE nicht separat installieren müssen.

### Welche JDK-Version ist am besten?

Es gibt keine absolut "beste" Version; die Wahl hängt von den spezifischen Anforderungen ab. Für neue Projekte werden JDK 21 (die neueste LTS-Version) oder JDK 25 (die neueste Version) empfohlen. Für Unternehmensprojekte, die langfristige Wartung erfordern, werden LTS-Versionen wie JDK 21 oder JDK 17 empfohlen.

JDK 21 ist die neueste LTS-Version und erhält mindestens 8 Jahre Support. JDK 17 wird immer noch weit verbreitet verwendet und ist kostenlos. JDK 8 wird, obwohl veraltet, immer noch in vielen Legacy-Systemen verwendet.

## Installationsfragen

### Wie überprüfe ich, ob JDK erfolgreich installiert ist?

Führen Sie java -version in der Befehlszeile aus. Wenn Versionsinformationen angezeigt werden, ist JDK korrekt installiert. Führen Sie javac -version aus, um zu überprüfen, ob der Compiler verfügbar ist. Führen Sie echo %JAVA_HOME% (Windows) oder echo $JAVA_HOME (Linux/macOS) aus, um zu überprüfen, ob die Umgebungsvariablen korrekt eingestellt sind.

Wenn diese Befehle Fehler in der Befehlszeile anzeigen, überprüfen Sie zuerst, ob JDK korrekt installiert ist, und dann, ob die Umgebungsvariablen korrekt konfiguriert sind.

### Was ist JAVA_HOME?

JAVA_HOME ist eine Umgebungsvariable, die auf das JDK-Installationsverzeichnis zeigt. Viele Java-Tools und Build-Skripte (wie Maven, Gradle) verlassen sich auf JAVA_HOME, um JDK zu lokalisieren. Das korrekte Einstellen von JAVA_HOME stellt sicher, dass diese Tools ordnungsgemäß funktionieren können.

Unter Windows wird JAVA_HOME typischerweise auf Pfade wie C:\Programme\Java\jdk-21 gesetzt. Unter Linux/macOS wird es typischerweise auf Pfade wie /usr/lib/jvm/java-21-openjdk gesetzt.

### Wie installiere ich mehrere JDK-Versionen unter Windows?

Sie können mehrere JDK-Versionen auf demselben Computer installieren. Nach der Installation mehrerer Versionen wechseln Sie die Standardversion, indem Sie die Reihenfolge der JDK-bin-Verzeichnisse in der PATH-Umgebungsvariable ändern oder die JAVA_HOME-Umgebungsvariable ändern, um auf verschiedene JDK-Installationsverzeichnisse zu zeigen.

Sie können auch Drittanbieter-Tools zur Verwaltung mehrerer JDK-Versionen verwenden, wie die Windows-Version von SDKMAN! oder JetBrains Toolbox. Diese Tools erleichtern das Wechseln zwischen Versionen.

### Was ist der Unterschied zwischen Oracle JDK und OpenJDK?

Oracle JDK und OpenJDK sind auf Codeebene fast identisch, mit Hauptunterschieden in Lizenzvereinbarungen und Supportservices. Oracle JDK bietet kommerziellen Support und zusätzliche Funktionen auf Unternehmensebene, aber einige Versionen können für die kommerzielle Nutzung gebührenpflichtig sein. OpenJDK ist vollständig Open-Source und kostenlos und wird von der Community gepflegt.

Für die meisten Benutzungszenarien ist OpenJDK völlig ausreichend. Viele Organisationen wählen OpenJDK, um Lizenzprobleme zu vermeiden und gleichzeitig fast identische Leistung und Funktionalität wie Oracle JDK zu erhalten.

## Benutzungsfragen

### Was ist Classpath?

Classpath ist der Pfad, den die JVM verwendet, um Klassendateien zu finden. Wenn Sie java MyClass ausführen, sucht die JVM nach My.class in den Verzeichnissen und JAR-Dateien, die durch Classpath angegeben sind.

Beim Kompilieren und Ausführen von Java-Programmen kann Classpath mit den Optionen -classpath oder -cp angegeben werden. Wenn nicht angegeben, verwendet die JVM das aktuelle Verzeichnis als Classpath.

### Was ist Bytecode?

Bytecode ist eine Zwischenform von Code, die der Java-Compiler aus Quellcode kompiliert. Bytecode ist kein nativer Code und kann nicht direkt auf der CPU ausgeführt werden; er kann nur von der JVM interpretiert oder JIT-kompiliert zur Ausführung gebracht werden.

Das Vorhandensein von Bytecode gibt Java-Programmen plattformunabhängige Fähigkeiten. Derselbe Bytecode kann auf jedem System mit installierter JVM ausgeführt werden, ob Windows, Linux oder macOS.

### Was ist eine JAR-Datei?

JAR (Java Archive) ist ein Format zum Verpacken mehrerer Java-Klassendateien in eine einzelne Datei. JAR-Dateien sind eigentlich ZIP-formatierte komprimierte Pakete, die kompilierte .class-Dateien und Metadaten enthalten.

Das jar-Werkzeug kann zum Erstellen, Anzeigen und Extrahieren von JAR-Dateien verwendet werden. Ausführbare JAR-Dateien enthalten eine MANIFEST.MF-Datei, die die Hauptklasse angibt, und können direkt mit java -jar ausgeführt werden.

### Was ist das Modulsystem?

Das Java-Modulsystem (Project Jigsaw) wurde in JDK 9 eingeführt und teilt JDK und Anwendungen in Module auf. Module bieten bessere Kapselung und deklarieren klar Abhängigkeiten, was beim Aufbau robusterer und wartbarer Anwendungen hilft.

Module werden über module-info.java-Dateien deklariert, die Modulnamen, Abhängigkeiten von anderen Modulen, exportierte Pakete usw. definieren. Das Modulsystem hilft bei der Lösung von Classpath-Konflikten und bietet bessere Versionskontrolle.

## Leistungsfragen

### Wie optimiere ich die JVM-Leistung?

Die JVM-Leistungsoptimierung basiert hauptsächlich auf: angemessener Einstellung der Heap-Speichergröße, Vermeidung von zu großen oder zu kleinen Einstellungen; Auswahl des geeigneten Garbage Collectors, G1 ist für die meisten Fälle eine gute Wahl; für Anwendungen, die niedrige Latenz anstreben, sollten Sie ZGC oder Shenandoah in Betracht ziehen; Aktivieren der Klassendatenfreigabe (-Xshare:classes) zur Beschleunigung des Starts.

Verwenden Sie Leistungsüberwachungstools (wie VisualVM, JConsole), um die Anwendungsleistung zu analysieren, Engpässe zu finden und gezielte Optimierung durchzuführen.

### Was ist Garbage Collection?

Garbage Collection (GC) ist der Mechanismus, durch den die JVM automatisch Speicher verwaltet. Während der Programmausführung werden Objekte erstellt, die Speicher verbrauchen. Wenn Objekte nicht mehr referenziert werden, identifiziert der Garbage Collector automatisch und gibt den von diesen Objekten belegten Speicher frei.

Die Garbage Collection von Java vereinfacht die Speicherverwaltungsarbeit erheblich; Programmierer müssen Speicher nicht explizit freigeben. Das Verständnis der Prinzipien der Garbage Collection hilft beim Schreiben effizienteren Codes und vermeidet Speicherlecks und häufige GC.

### Was ist ein Speicherleck?

Ein Speicherleck bezieht sich auf Speicher, der während der Programmausführung nicht wiederhergestellt werden kann, was dazu führt, dass der verfügbare Speicher allmählich abnimmt. In Java werden Speicherlecks normalerweise durch Sammlungen (wie statische Maps, Listen) verursacht, die kontinuierlich Objekte hinzufügen, ohne diese rechtzeitig zu bereinigen.

Um Speicherlecks zu untersuchen, können Sie jmap verwenden, um Heap-Dump-Dateien zu generieren, und dann Tools wie MAT (Memory Analyzer Tool) verwenden, um analysieren und die verursachenden Objekte zu finden.

### Was ist ein JIT-Compiler?

Der JIT (Just-In-Time)-Compiler ist Teil der JVM und kompiliert heißen Bytecode in nativen Code, um die Ausführungsleistung zu verbessern. Während der Laufzeit analysiert die JVM die Codeausführung; für häufig ausgeführten Code (heiße Punkte) kompiliert der JIT-Compiler diesen in nativen Maschinencode, was die Leistung erheblich verbessert.

Die JIT-Kompilierung erfolgt zur Laufzeit ohne zusätzliche Kompilierungsschritte. Der C1-Compiler kompiliert schnell, aber mit weniger Optimierung; der C2-Compiler kompiliert langsam, aber mit mehr Optimierung. Mehrstufige Kompilierung kombiniert die Vorteile beider.

## Versionsfragen

### Welche Java-Version sollte ich verwenden?

Für neue Projekte wird die neueste LTS-Version JDK 21 empfohlen. Sie bietet gute Leistung, reichhaltige Funktionen und langfristigen Support. Für Projekte, die Kompatibilität mit Java 8/11 benötigen, können diese Versionen verwendet werden.

Die Verwendung von Versionen, die den Support eingestellt haben, wie JDK 8 (es sei denn, Sie haben ein kostenpflichtiges Abonnement), wird nicht empfohlen. Das rechtzeitige Upgrade auf neue JDK-Versionen bietet Sicherheitsupdates und Leistungsverbesserungen.

### Was bedeutet LTS in Java-Versionsnummern?

LTS (Long-Term Support) steht für Versionen mit Langzeit-Support. LTS-Versionen erhalten mindestens 8 Jahre technischen Support und Patch-Updates von Oracle. Aktuelle LTS-Versionen umfassen JDK 8, JDK 11, JDK 17 und JDK 21.

Nicht-LTS-Versionen (wie JDK 25) haben kürzere Supportzyklen, normalerweise nur 6 Monate. Diese Versionen eignen sich für Entwickler, die die neuesten Funktionen wünschen, aber für Produktionsumgebungen werden LTS-Versionen empfohlen.

### Kann ich Java 17 noch verwenden?

Java 17 ist eine reife LTS-Version und kann normal verwendet werden. Oracle änderte die JDK 17-Lizenz im Oktober 2024. Laut Oracles Aussage kann JDK 17 jedoch immer noch kostenlos für Entwicklung, persönliche Nutzung und das Ausführen Oracle-lizenzierter Produkte verwendet werden.

Viele Unternehmen verwenden immer noch JDK 17, da es eine sehr reife und stabile Version mit vielen Community-Support-Ressourcen ist. Wenn das Projekt bereits JDK 17 verwendet, kann es weiter verwendet werden; für neue Projekte wird JDK 21 bevorzugt.

## Entwicklungsfragen

### Wie kompiliere ich Java-Dateien?

Verwenden Sie den javac-Befehl zum Kompilieren von Java-Dateien. Grundlegende Verwendung: javac MyClass.java. Nach dem Kompilieren wird eine MyClass.class-Bytecodedatei generiert.

Für Projekte werden normalerweise Build-Tools wie Maven oder Gradle verwendet, um den Kompilierungsprozess zu verwalten. Diese Tools behandeln automatisch Abhängigkeiten und Kompilierungsoptionen.

### Wie führe ich Java-Programme aus?

Verwenden Sie den java-Befehl zum Ausführen kompilierter Java-Programme. Grundlegende Verwendung: java MyClass. Beachten Sie, dass die .class-Erweiterung hier nicht erforderlich ist.

Wenn das Programm in einer JAR-Datei verpackt ist, verwenden Sie java -jar MyApp.jar zum Ausführen.

### Wie richte ich Classpath ein?

Geben Sie Classpath zur Laufzeit mit -classpath oder -cp an: java -cp lib/*:out MyClass. Unter Windows ist das Trennzeichen ein Semikolon (;); unter Linux/macOS ist es ein Doppelpunkt (:).

Sie können auch über die CLASSPATH-Umgebungsvariable konfigurieren, aber die Befehlszeilenspezifikation ist flexibler.

### Was sind Maven und Gradle?

Maven und Gradle sind Build-Tools für Java-Projekte. Sie können automatisch Abhängigkeiten herunterladen, Code kompilieren, Tests ausführen und für die Veröffentlichung verpacken, was den Projekt-Build-Prozess erheblich vereinfacht.

Maven verwendet XML (pom.xml), um die Projektkonfiguration zu definieren, während Gradle Groovy- oder Kotlin-Skripte (build.gradle) verwendet, um die Build-Logik zu definieren. Beides sind weit verbreitete Build-Tools in Java-Projekten.
