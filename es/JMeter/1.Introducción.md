# Introducción a Apache JMeter

Apache JMeter es un software de código abierto de la Apache Software Foundation, una aplicación 100% pura Java diseñada para pruebas de carga y medición de rendimiento. JMeter fue diseñado originalmente para probar aplicaciones web pero desde entonces se ha expandido a otras funciones de prueba.

## ¿Qué puede hacer JMeter?

JMeter puede usarse para probar el rendimiento de recursos estáticos y dinámicos, incluyendo aplicaciones web. Puede simular una carga pesada en un servidor, grupo de servidores, red u objeto para probar su resistencia o analizar el rendimiento general bajo diferentes tipos de carga.

Las principales características incluyen:

- **Soporte multi-protocolo**: Capacidad para probar rendimiento de carga de muchos tipos diferentes de aplicaciones, servidores y protocolos, incluyendo:
  - Web - HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET, etc.)
  - Servicios web SOAP / REST
  - FTP
  - Bases de datos mediante JDBC
  - LDAP
  - Middleware orientado a mensajes (MOM) mediante JMS
  - Correo - SMTP(S), POP3(S) e IMAP(S)
  - Comandos nativos o scripts de shell
  - TCP
  - Objetos Java

- **IDE de prueba completo**: Permite la grabación rápida de planes de prueba (desde navegadores o aplicaciones nativas), construcción y depuración

- **Modo CLI**: Modo de línea de comandos (antes llamado modo Non GUI o modo headless) para pruebas de carga desde cualquier sistema operativo compatible con Java (Linux, Windows, Mac OSX, etc.)

- **Informes HTML dinámicos**: Generación completa y lista para presentar informes HTML dinámicos

- **Extracción de datos**: Capacidad fácil de correlacionar extrayendo datos de formatos populares como HTML, JSON, XML o cualquier formato de texto

- **Portabilidad 100% Java**: Completamente portable e implementación pura en Java

- **Framework multi-hilo**: Muestreo concurrente por muchos hilos y muestreo simultáneo de diferentes funciones por grupos de hilos separados

- **Altamente extensible**: Samplers conectables, Samplers scripting (lenguajes compatibles con JSR223 como Groovy y BeanShell)

- **Integración continua**: Soporte para integración con Maven, Gradle y Jenkins

## JMeter no es un navegador

JMeter trabaja a nivel de protocolo. En lo que respecta a servicios web y servicios remotos, JMeter parece un navegador (o más bien, múltiples navegadores); sin embargo, JMeter no realiza todas las acciones soportadas por los navegadores. En particular, JMeter no ejecuta JavaScript encontrado en páginas HTML, ni renderiza las páginas HTML como lo hace un navegador.

## Escenarios de aplicación de JMeter

- **Pruebas de carga**: Simular grandes cantidades de usuarios concurrentes para probar el rendimiento del sistema bajo alta carga
- **Pruebas de rendimiento**: Medir el tiempo de respuesta y throughput del sistema bajo varias condiciones de carga
- **Pruebas funcionales**: Verificar el comportamiento funcional de la aplicación
- **Pruebas de estrés**: Probar la estabilidad y tolerancia a fallas del sistema bajo alta presión
- **Pruebas distribuidas**: Soporte para pruebas de carga coordinadas en múltiples máquinas
