# APISIX FAQ

Häufig gestellte Fragen und Antworten.

## Installation

### Q: Docker-Container startet nicht?

**Lösungen:**
1. Bestätigen, dass Docker installiert und ausgeführt wird
```bash
docker --version
docker ps
```
2. Überprüfen, ob Ports belegt sind
```bash
netstat -tlnp | grep 9080
```
3. Container-Logs anzeigen
```bash
docker logs apisix
docker logs etcd
```

### Q: etcd-Verbindungsfehler?

**Lösungen:**
1. Bestätigen, dass etcd läuft
```bash
docker ps | grep etcd
```
2. Überprüfen, ob etcd auf dem richtigen Port lauscht
3. etcd-Adresse in der Konfiguration prüfen

### Q: Port bereits belegt?

**Lösungen:**
1. Prozess identifizieren, der den Port verwendet
```bash
lsof -i:9080
```
2. Prozess beenden oder APISIX-Port ändern
```yaml
apisix:
  node_listen: 9081
```

## Verwendung

### Q: Route funktioniert nicht?

**Lösungen:**
1. Routenkonfigurationsformat prüfen
2. Bestätigen, dass Route korrekt erstellt wurde
3. Fehlerlogs prüfen
```bash
tail -f logs/error.log
```

### Q: Plugin funktioniert nicht?

**Lösungen:**
1. Bestätigen, dass Plugin aktiviert ist
2. Plugin-Konfigurationsformat prüfen
3. Plugin-Abhängigkeiten prüfen

### Q: Upstream-Dienst nicht erreichbar?

**Lösungen:**
1. Bestätigen, dass Upstream-Dienst läuft
2. Netzwerkkonnektivität prüfen
3. Firewall-Regeln prüfen

## Konfiguration

### Q: SSL/TLS-Zertifikate konfigurieren?

**Lösungen:**
1. SSL-Zertifikat hochladen
```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/ssls/1 \
  -H "X-API-KEY: 123456" \
  -d '{"cert":"Zertifikatsinhalt","key":"privater Schlüssel","sni":"example.com"}'
```

### Q: CORS konfigurieren?

**Lösungen:**
cors-Plugin verwenden:

```bash
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: 123456" \
  -d '{"uri":"/api/*","plugins":{"cors":{}}}'
```

## Leistung

### Q: Speichernutzung steigt ständig?

**Lösungen:**
1. Auf Speicherlecks in Plugins prüfen
2. Puffergröße begrenzen

### Q: Hohe CPU-Auslastung?

**Lösungen:**
1. Anfragevolumen prüfen
2. Regex-Matching-Regeln optimieren
3. Plugin-Anzahl reduzieren

## Sicherheit

### Q: Admin-API schützen?

**Lösungen:**
1. Starke Schlüssel verwenden
2. Zugriffs-IPs beschränken
3. HTTPS aktivieren

### Q: Angriffe verhindern?

**Lösungen:**
1. Rate-Limiting-Plugin aktivieren
2. IP-Blacklist aktivieren
3. Firewall aktivieren
4. Anormale Anfragen überwachen
