# Apache JMeter 簡介

Apache JMeter 是 Apache 軟體基金會旗下的開源軟體，是一個 100% 純 Java 編寫的應用程式，專為負載測試功能設計和效能測量而開發。JMeter 最初是為測試 Web 應用程式而設計的，後來已擴展到其他測試功能領域。

## JMeter 能做什麼

JMeter 可用於對靜態資源和動態資源進行效能測試，包括 Web 應用程式。它可以模擬伺服器、伺服器組、網路或物件的高負載情況，測試其強度或在不同負載類型下分析整體效能。

JMeter 的主要功能特性包括：

- **多協定支援**：支援測試多種應用程式、伺服器和協定類型，包括 Web（HTTP/HTTPS）、SOAP/REST Web 服務、FTP、資料庫（JDBC）、LDAP、JMS 訊息中介軟體、郵件協定（SMTP/POP3/IMAP）、TCP、Java 物件等
- **完整的測試 IDE**：提供功能完整的測試 IDE，支援從瀏覽器或原生應用程式快速錄製測試計畫、構建和偵錯
- **命令列模式**：支援 CLI 模式（命令列模式/無頭模式），可在任何 Java 相容作業系統（Linux、Windows、Mac OSX 等）上進行負載測試
- **動態 HTML 報告**：內建完整且易於展示的動態 HTML 報告生成功能
- **資料擷取能力**：支援從多種常見回應格式中擷取資料，包括 HTML、JSON、XML 或任何文字格式
- **純 Java 可攜性**：完全可攜帶，100% 純 Java 實現
- **多執行緒框架**：完整的並發取樣能力，支援多執行緒同時取樣和不同功能組獨立測試
- **高度可擴展**：支援可插拔的取樣器、計時器，以及 JSR223 相容指令碼語言（如 Groovy 和 BeanShell）
- **持續整合支援**：提供與 Maven、Gradle 和 Jenkins 整合的第三方開源庫

## JMeter 不是瀏覽器

JMeter 工作在協定級別，它不是瀏覽器。對於 Web 服務和遠端服務，JMeter 表现得像一個瀏覽器（或多個瀏覽器）；但 JMeter 不執行瀏覽器支援的所有操作，特別是不會執行 HTML 頁面中的 JavaScript，也不會像瀏覽器那樣渲染 HTML 頁面。

## JMeter 的應用場景

- 負載測試：模擬大量並發使用者，測試系統在高負載下的表現
- 效能測試：測量系統在不同負載條件下的回應時間和吞吐量
- 功能測試：驗證應用程式的功能行為
- 壓力測試：測試系統在高壓力下的穩定性和容錯能力
- 分散式測試：支援多台機器協同進行負載測試
