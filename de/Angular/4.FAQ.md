# Angular FAQ

Häufig gestellte Fragen und Antworten.

## Installation

### Q: Angular CLI-Installation fehlgeschlagen?

**Lösungen:**
1. Node.js-Version überprüfen
```bash
node --version  # Benötigt 18+ oder 20+
```
2. npm-Cache leeren
```bash
npm cache clean --force
```
3. Administrator verwenden (Windows)
4. Netzwerkverbindung überprüfen

### Q: ng-Befehl nicht gefunden?

**Lösungen:**
1. Bestätigen, dass Angular CLI installiert ist
```bash
npm list -g @angular/cli
```
2. PATH-Umgebungsvariable überprüfen
3. Neu installieren
```bash
npm uninstall -g @angular/cli
npm install -g @angular/cli@latest
```

### Q: Projekterstellung fehlgeschlagen?

**Lösungen:**
1. Festplattenplatz überprüfen
2. npm-Berechtigungen überprüfen
3. --skip-install verwenden
```bash
ng new my-app --skip-install
cd my-app
npm install
```

## Entwicklung

### Q: Komponente wird nicht angezeigt?

**Lösungen:**
1. Bestätigen, dass Komponente korrekt importiert ist
2. Bestätigen, dass Selektor korrekt verwendet wird
3. Template-Syntax überprüfen
4. Konsole auf Fehler prüfen

### Q: Daten werden nicht aktualisiert?

**Lösungen:**
1. Für Signale set/update-Methoden verwenden
```typescript
count.set(1);           // Neuen Wert setzen
count.update(c => c + 1); // Basierend auf altem Wert aktualisieren
```
2. Für RxJS korrektes Abonnieren sicherstellen
3. ChangeDetectorRef für manuelle Erkennung verwenden
```typescript
constructor(private cdr: ChangeDetectorRef) {}

refresh() {
  this.cdr.detectChanges();
}
```

### Q: Routing funktioniert nicht?

**Lösungen:**
1. Bestätigen, dass Routenkonfiguration korrekt ist
2. Bestätigen, dass RouterOutlet platziert ist
3. Routenreihenfolge überprüfen
```typescript
// Spezifische Routen vor Wildcard
{ path: 'user/:id', ... },
{ path: '**', component: NotFoundComponent }
```

### Q: Formularvalidierung funktioniert nicht?

**Lösungen:**
1. ReactiveFormsModule oder FormsModule importieren
2. Bestätigen, dass FormControl-Namen korrekt sind
3. Validatore korrekt verwenden
```typescript
this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', [Validators.required, Validators.minLength(6)]]
});
```

## Leistung

### Q: Langsamer erster Ladevorgang?

**Lösungen:**
1. Lazy Loading für Routen verwenden
```typescript
{
  path: 'admin',
  loadComponent: () => import('./admin/admin.component')
    .then(m => m.AdminComponent)
}
```
2. Produktionsmodus aktivieren
```bash
ng build --configuration=production
```
3. AOT-Kompilierung ist standardmäßig aktiviert

### Q: Speicherlecks?

**Lösungen:**
1. Korrekt abbestellen
```typescript
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

@Component({...})
export class MyComponent implements OnDestroy {
  private destroy$ = new Subject<void>();

  ngOnInit() {
    this.http.get('/api').pipe(
      takeUntil(this.destroy$)
    ).subscribe();
  }

  ngOnDestroy() {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
```
2. Async Pipe für automatisches Abonnieren/Abbestellen verwenden

### Q: Bundle zu groß?

**Lösungen:**
1. Abhängigkeitsgrößen prüfen
```bash
npm run build -- --stats-json
npx webpack-bundle-analyzer dist/stats.json
```
2. Lazy Loading verwenden
3. Unbenutzte Importe entfernen

## Testing

### Q: Wie schreibe ich Unit-Tests?

**Lösungen:**
```typescript
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { UserCardComponent } from './user-card.component';

describe('UserCardComponent', () => {
  let component: UserCardComponent;
  let fixture: ComponentFixture<UserCardComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [UserCardComponent]
    }).compileComponents();

    fixture = TestBed.createComponent(UserCardComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display user name', () => {
    component.user = { name: 'Max', email: 'test@example.com' };
    fixture.detectChanges();
    const compiled = fixture.nativeElement as HTMLElement;
    expect(compiled.querySelector('h3')?.textContent).toContain('Max');
  });
});
```

### Q: Testmodul nicht gefunden?

**Lösungen:**
```typescript
TestBed.configureTestingModule({
  imports: [
    CommonModule,
    HttpClientModule,
    // Andere benötigte Module
  ]
});
```

## Build

### Q: Build fehlgeschlagen?

**Lösungen:**
1. TypeScript-Fehler prüfen
```bash
npx tsc --noEmit
```
2. Cache leeren
```bash
rm -rf node_modules
npm install
```
3. Angular-Version aktualisieren
```bash
ng update @angular/core @angular/cli
```

### Q: Umgebungsvariablen funktionieren nicht?

**Lösungen:**
1. Bestätigen, dass Umgebungsdateinamen korrekt sind
2. Umgebung in angular.json konfigurieren
```json
"configurations": {
  "production": {
    "fileReplacements": [
      {
        "replace": "src/environments/environment.ts",
        "with": "src/environments/environment.prod.ts"
      }
    ]
  }
}
```

## Sicherheit

### Q: Wie verhindere ich XSS-Angriffe?

**Lösungen:**
1. Angular escapt standardmäßig automatisch
2. Vorsichtig mit innerHTML
```typescript
import { DomSanitizer } from '@angular/platform-browser';

constructor(private sanitizer: DomSanitizer) {}

getHtml() {
  // Nur für vertrauenswürdige Inhalte verwenden
  return this.sanitizer.bypassSecurityTrustHtml(this.trustedContent);
}
```

### Q: CSRF-Schutz?

**Lösungen:**
```typescript
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
    HttpClientXsrfModule.withOptions({
      cookieName: 'XSRF-TOKEN',
      headerName: 'X-XSRF-TOKEN'
    })
  ]
})
export class AppModule {}
```

## Upgrade

### Q: Wie aktualisiere ich die Angular-Version?

**Lösungen:**
1. ng update verwenden
```bash
ng update @angular/core@17 @angular/cli@17
```
2. Migrationsleitfaden prüfen
3. Abhängigkeitsversionen aktualisieren
4. Tests ausführen, um Kompatibilität sicherzustellen

### Q: Code nach Upgrade nicht kompatibel?

**Lösungen:**
1. Offiziellen Upgrade-Leitfaden prüfen
2. ng update für automatische Migration verwenden
3. Inkrementell aktualisieren, nicht Versionssprünge machen
