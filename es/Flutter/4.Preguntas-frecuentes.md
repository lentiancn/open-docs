# Preguntas Frecuentes de Flutter

## Problemas de Instalación

### ¿Qué hacer si Flutter doctor muestra errores?

El comando flutter doctor verifica el estado del entorno de desarrollo de Flutter. Errores comunes y soluciones:

- Android SDK no encontrado: Necesitas instalar Android Studio o configurar manualmente la variable de entorno ANDROID_SDK_ROOT
- Versión de Xcode muy antigua: Actualiza Xcode a la última versión
- Licencias no aceptadas: Ejecuta flutter doctor --android-licenses para aceptar las licencias

### ¿Qué hacer si la descarga de Flutter falla?

Si la descarga es lenta o falla, intenta usar un espejo. Los usuarios chinos pueden configurar las variables de entorno PUB_HOSTED_URL y FLUTTER_STORAGE_BASE_URL para usar espejos nacionales.

### ¿Qué hacer si el clone de Git falla?

Asegúrate de que la red pueda acceder a GitHub. Intenta usar un proxy o espejo nacional. También puedes descargar el archivo precompilado del sitio web de Flutter e instalarlo manualmente.

## Problemas de Desarrollo

### ¿La recarga en caliente no funciona?

Cuando la recarga en caliente falla, intenta: Hacer clic en Stop y luego ejecutar nuevamente; Ejecutar flutter clean para limpiar la caché de compilación; Verificar si hay errores de sintaxis en el código.

### ¿La aplicación está lenta?

Verifica si se están realizando operaciones que consumen mucho tiempo en el método build. Los cálculos complejos deben usar Isolate o procesamiento asíncrono. Evita reconstrucciones innecesarias de widgets, usa constructores const y claves apropiadas.

### ¿El estado se pierde?

La recarga en caliente preserva el estado de StatefulWidget, pero el estado se restablece al reconstruir la aplicación o realizar un reinicio completo en caliente. Para persistir el estado, usa SharedPreferences, SQLite u otras soluciones de almacenamiento.

### ¿Conflictos de dependencias de paquetes?

Ejecuta flutter pub get para ver el árbol de dependencias. Usa flutter pub deps para verificar las dependencias del paquete. Intenta actualizar las versiones de los paquetes o usa dependency_overrides para resolver conflictos.

## Problemas de Plataforma

### ¿La compilación de Android falla?

Causas comunes incluyen: versión de SDK no coincide, problemas de configuración de Gradle, errores de configuración de firma. Intenta ejecutar flutter clean y luego recompilar. Verifica las configuraciones en android/build.gradle y android/app/build.gradle.

### ¿El simulador de iOS no inicia?

Asegúrate de que Xcode esté instalado y configurado correctamente. Ejecuta xcrun simctl list devices en Terminal para ver los simuladores disponibles. Intenta reiniciar el simulador o crear uno nuevo.

### ¿Error al compilar Web?

Verifica si las dependencias web en pubspec.yaml son correctas. Usa flutter clean para limpiar y luego recompilar. Asegúrate de que el navegador Chrome esté instalado.

## Problemas de Rendimiento

### ¿El tamaño de la aplicación es demasiado grande?

Usa flutter build apk --split-per-abi para generar APKs para diferentes arquitecturas de CPU. Verifica y elimina imágenes y recursos no utilizados. Usa ofuscación y compresión.

### ¿El tiempo de inicio es demasiado largo?

Reduce las operaciones de inicialización en la función main. Carga componentes no esenciales de forma diferida. Usa Splash Screen para mejorar la experiencia del usuario.

### ¿El uso de memoria es demasiado alto?

Verifica si hay fugas de memoria. Usa herramientas de Profile para analizar el uso de memoria. Libera recursos y objetos grandes que ya no se necesiten.

## Problemas de Aprendizaje

### ¿Cómo elegir una solución de gestión de estado?

Usa setState para aplicaciones simples. Provider se recomienda para aplicaciones de complejidad media. Riverpod, GetX o BLoC se pueden usar para aplicaciones complejas.

### ¿Migrando desde otros marcos?

El sitio web de Flutter tiene guías de migración para desarrolladores de Android, iOS, React Native. Los conceptos principales son similares, enfócate en aprender widgets y el sistema de diseño.

### ¿Cómo mejorar la eficiencia del desarrollo?

Domina los plugins de Flutter para VS Code o Android Studio. Aprende atajos y plantillas de código. Usa herramientas de recarga en caliente y depuración. Consulta la documentación oficial y recursos de la comunidad.
