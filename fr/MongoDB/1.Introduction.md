# MongoDB Introduction

## Qu'est-ce que MongoDB ?

MongoDB est une base de données orientée documents conçue pour aider les développeurs à construire plus rapidement des applications modernes. Il stocke les données dans des documents flexibles de type JSON, ce qui facilite la modélisation des données comme le code de l'application les utilise. Le schéma flexible vous permet de faire évoluer votre modèle de données sans temps d'arrêt, d'itérer rapidement et de gérer facilement les données non uniformes.

MongoDB fournit un moteur de requête puissant, une mise à l'échelle horizontale et une haute disponibilité intégrée pour tout prendre en charge, du prototypage rapide aux charges de travail critiques de grande échelle.

MongoDB est une base de données opérationnelle entièrement transactionnelle qui prend en charge une large gamme de types de charges de travail, notamment :

- Recherche structurée basée sur les documents (OLTP)
- Agrégation de données
- Recherche en texte intégral
- Recherche vectorielle
- Recherche géospatiale
- Données de séries temporelles

## Modèle de données de document

Un enregistrement dans MongoDB est un document, qui est une structure de données composée de paires champ-valeur. Les documents MongoDB sont similaires aux objets JSON. Les valeurs des champs peuvent inclure d'autres documents, tableaux et tableaux de documents.

Le modèle de données orienté objet flexible de MongoDB vous permet de structurer les données d'une manière qui reflète les modèles d'objets dans votre code. Cela élimine la nécessité d'un mappage objet-relationnel complexe requis lors de l'utilisation de bases de données relationnelles.

Voici un exemple de document MongoDB montrant comment les documents peuvent exprimer des données complexes et hiérarchiques dans un format qui reflète le modèle d'objets de votre application :

```
{
  _id: ObjectId("507f1f77bcf86cd799439011"),
  name: "Alice",
  birthdate: ISODate("1990-01-01T00:00:00Z"),
  address: {
    street: "123 Main St",
    city: "Springfield",
    state: "IL"
  },
  hobbies: ["reading", "hiking", "coding"]
}
```

Les avantages supplémentaires du modèle de données de document sont :

- Les documents correspondent aux types de données natifs dans de nombreux langages de programmation
- Les documents et tableaux intégrés réduisent le besoin de jointures coûteuses qui peuvent ralentir les performances
- Les schémas dynamiques prennent en charge le polymorphisme, permettant des structures variées entre les documents de la même collection

## Composants principaux de l'architecture MongoDB

### Base de données de documents

Le modèle de données de document flexible vous permet de mapper vos données aux besoins de votre application. Contrairement aux bases de données relationnelles traditionnelles avec des formats rigides de lignes et colonnes, les documents MongoDB peuvent contenir des structures imbriquées, ce qui les rend idéaux pour gérer des relations de données complexes.

### Transactions

MongoDB prend en charge les transactions multi-documents, vous permettant d'exécuter plusieurs opérations de lecture et d'écriture comme un seul événement tout ou rien. Les fonctionnalités principales des transactions incluent :

- Garanties ACID : Les transactions fournissent l'atomicité, la cohérence, l'isolation et la durabilité sur plusieurs opérations
- Opérations multi-documents : Vous pouvez exécuter des opérations complexes couvrant plusieurs documents tout en maintenant la cohérence des données
- Transactions distribuées : Vous pouvez coordonner des transactions sur des clusters fragmentés avec les mêmes garanties ACID

### Haute disponibilité

Le mécanisme de réplication intégré de MongoDB fournit un basculement automatique, une redondance des données et une capacité de lecture accrue. Avec le basculement automatique, si le serveur primaire devient indisponible, le cluster sélectionne automatiquement un nouveau primaire, ce qui garantit que les écritures restent disponibles. De plus, plusieurs copies de vos données sont stockées sur différents serveurs pour améliorer la durabilité des données.

### Mise à l'échelle horizontale

MongoDB prend en charge la mise à l'échelle horizontale via une technique appelée sharding. Les fonctionnalités principales du sharding incluent :

- Distribution automatique des données : MongoDB partitionne automatiquement les données en fonction d'une clé de fragment et les distribue sur un cluster de machines
- Sharding par zones : Vous pouvez définir des zones géographiques pour contrôler le placement des documents en fonction des plages de clés de fragmentation
- Raffinement de la clé de fragmentation : Vous pouvez affiner votre clé de fragmentation pour améliorer les performances à mesure que votre application évolue

## Éditions MongoDB

MongoDB est disponible en deux éditions principales :

- **Édition Communauté (Community Edition)** : Open source et gratuit, adapté aux développeurs individuels et aux petits projets
- **Édition Entreprise (Enterprise Edition)** : Version commerciale avec des fonctions avancées de sécurité et de gestion

La dernière version stable actuelle est MongoDB 8.2, avec MongoDB 7.0 comme version de support à long terme. MongoDB 6.0 a atteint sa fin de vie le 31 juillet 2025.

## Plateformes non prises en charge

MongoDB ne prend pas en charge les plateformes x86 32 bits. Assurez-vous que votre système d'exploitation est 64 bits.

## Cas d'utilisation

MongoDB convient à une grande variété de scénarios d'application :

- **Systèmes de gestion de contenu** : Stocker des articles, des commentaires, du contenu généré par les utilisateurs et autres données non structurées ou semi-structurées
- **Applications mobiles** : Stocker les configurations utilisateur, les états d'application et autres structures de données flexibles
- **Analyse en temps réel** : Les capacités d'écriture haute vitesse et d'agrégation en temps réel conviennent à l'analyse des journaux et aux statistiques en temps réel
- **Internet des objets** : Stocker de grandes quantités de données de séries temporelles provenant de capteurs
- **Commerce électronique** : Catalogues de produits, gestion des commandes, profils utilisateurs et plus
- **Applications d'intelligence artificielle** : Les capacités de recherche vectorielle prennent en charge les exigences de recherche sémantique pour les applications IA

## MongoDB vs Bases de données relationnelles

| Fonctionnalité | MongoDB | Bases de données relationnelles |
|----------------|---------|--------------------------------|
| Modèle de données | Document (type JSON) | Lignes et colonnes |
| Schéma | Flexible (dynamique) | Fixe (statique) |
| Mise à l'échelle | Horizontale (sharding) | Verticale |
| Transactions | ACID multi-documents | ACID multi-tables |
| Relations | Documents/tableaux intégrés | Opérations JOIN |
| Indexation | Plusieurs types d'index pris en charge | Index pris en charge |

## Résumé

MongoDB, en tant que base de données NoSQL leader, est devenu un choix populaire pour le développement d'applications modernes grâce à son modèle de documents flexible, ses puissantes capacités de requête, sa haute performance et sa haute disponibilité. Que ce soit pour les startups ou les grandes entreprises, MongoDB propose des options de déploiement qui peuvent être adaptées aux besoins spécifiques. La philosophie de conception de MongoDB permet aux développeurs de construire des applications modernes plus rapidement, avec des modèles de documents flexibles, des langages de requête intuitifs et de puissantes capacités de mise à l'échelle horizontale qui aident les équipes à itérer et à mettre à l'échelle rapidement.
