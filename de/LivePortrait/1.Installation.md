# LivePortrait Installationsanleitung

LivePortrait ist ein effizientes Porträt-Animationsframework, entwickelt von Kwai VGI. Es animiert statische Porträts mit Bewegungen aus Steuerungsvideos und unterstützt Stitching- und Retargeting-Kontrolle.

## Versionsübersicht

| Version | Status | Beschreibung |
|---------|--------|--------------|
| LivePortrait 1.x | Aktuell | Hauptversion |

## Systemanforderungen

### Hardwareanforderungen

| Komponente | Minimum | Empfohlen |
|------------|---------|-----------|
| GPU | NVIDIA 8GB VRAM | NVIDIA 16GB VRAM |
| RAM | 8GB | 16GB |
| Speicher | 30GB | 50GB |
| CUDA | 11.8+ | 12.1+ |

### Softwareanforderungen

| Software | Version | Hinweise |
|----------|---------|----------|
| Python | 3.8 - 3.11 | Empfohlen 3.10 |
| CUDA | 11.8+ | GPU-Beschleunigung erforderlich |
| cuDNN | 8.5+ | Erforderlich für CUDA |
| ffmpeg | Neueste | Videoverarbeitung erforderlich |

## Linux/Ubuntu Installation

### Schritt 1: Systemabhängigkeiten installieren

```bash
# System aktualisieren
sudo apt update && sudo apt upgrade -y

# Python und Entwicklungstools installieren
sudo apt install python3.10 python3-pip python3-venv git wget curl

# Systembibliotheken installieren
sudo apt install libgl1-mesa-glx libglib2.0-0 libsm6 libxext6 libxrender-dev libgomp1

# ffmpeg installieren
sudo apt install ffmpeg
```

### Schritt 2: CUDA installieren

```bash
# CUDA 12.1 herunterladen und installieren
wget https://developer.download.nvidia.com/compute/cuda/12.1.0/local_installers/cuda_12.1.0_520.53_linux.run
sudo sh cuda_12.1.0_520.53_linux.run
```

### Schritt 3: Projekt klonen

```bash
git clone https://github.com/KwaiVGI/LivePortrait.git
cd LivePortrait
```

### Schritt 4: Virtuelle Umgebung erstellen

```bash
python3.10 -m venv venv
source venv/bin/activate

# Python-Version überprüfen
python --version
```

### Schritt 5: PyTorch installieren

```bash
# CUDA 12.1 Version
pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu121

# Installation überprüfen
python -c "import torch; print(torch.cuda.is_available())"
```

### Schritt 6: Abhängigkeiten installieren

```bash
# Basisanforderungen installieren
pip install -r requirements.txt

# Zusätzliche Abhängigkeiten installieren
pip install numpy opencv-python librosa soundfile
```

### Schritt 7: Modelle herunterladen

```bash
# Modellverzeichnis erstellen
mkdir -p pretrained_models

# Von der Release-Seite herunterladen oder Download-Skript verwenden
bash scripts/download_weights.sh
```

## Windows Installation

### Schritt 1: Python installieren

1. Python 3.10 von python.org herunterladen
2. "Add Python to PATH" während der Installation aktivieren
3. Überprüfen: `python --version`

### Schritt 2: CUDA installieren

1. CUDA 12.1 von der NVIDIA-Website herunterladen
2. Mit Standardeinstellungen installieren
3. cuDNN 8.9 herunterladen und installieren

### Schritt 3: Klonen und Einrichten

```powershell
git clone https://github.com/KwaiVGI/LivePortrait.git
cd LivePortrait

python -m venv venv
venv\Scripts\activate
```

### Schritt 4: Abhängigkeiten installieren

```powershell
# PyTorch installieren (mit CUDA-Unterstützung)
pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu121

pip install -r requirements.txt
```

### Schritt 5: Modelle herunterladen

Modelle von GitHub releases herunterladen und in das `pretrained_models/` Verzeichnis legen.

## macOS Installation

### Schritt 1: Abhängigkeiten installieren

```bash
# Homebrew installieren (falls nicht vorhanden)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Python und ffmpeg installieren
brew install python@3.10 ffmpeg git
```

### Schritt 2: Einrichten

```bash
git clone https://github.com/KwaiVGI/LivePortrait.git
cd LivePortrait

python3.10 -m venv venv
source venv/bin/activate

# PyTorch installieren (macOS nur CPU)
pip install torch==2.1.0 torchvision==0.16.0

pip install -r requirements.txt
```

Hinweis: macOS unterstützt nur den CPU-Modus, GPU-Beschleunigung ist nicht möglich.

## Docker Installation

### Docker verwenden

```bash
# Image herunterladen
docker pull liveportrait:latest

# Ausführen (GPU erforderlich)
docker run --gpus all -v $(pwd):/workspace -p 7860:7860 liveportrait:latest
```

### Docker Compose verwenden

```yaml
version: '3.8'
services:
  liveportrait:
    image: liveportrait:latest
    volumes:
      - ./workspace:/workspace
    ports:
      - "7860:7860"
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```

## Häufige Probleme

### CUDA Out of Memory

**Lösungen:**
- Batchgröße reduzieren: `--batch_size 1`
- Niedrigere Auflösung verwenden
- Model-Offloading aktivieren

### Fehlende Bibliotheken (Windows)

**Lösungen:**
- Visual C++ Redistributable 2015-2022 installieren
- CUDA zum System-PATH hinzufügen

### Modell-Download fehlgeschlagen

**Lösungen:**
- VPN oder Proxy verwenden
- Manuell von GitHub releases herunterladen
- Netzwerkverbindung überprüfen

### ffmpeg nicht gefunden

**Lösungen:**
- Linux: `sudo apt install ffmpeg`
- Windows: ffmpeg zum PATH hinzufügen
- macOS: `brew install ffmpeg`
