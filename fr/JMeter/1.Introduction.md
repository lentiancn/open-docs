# Introduction à Apache JMeter

Apache JMeter est un logiciel open source de la Apache Software Foundation, une application 100% pure Java conçue pour les tests de charge et la mesure de performance. JMeter a été initialement conçu pour tester les applications Web mais s'est depuis étendu à d'autres fonctions de test.

## Que peut faire JMeter ?

JMeter peut être utilisé pour tester les performances des ressources statiques et dynamiques, y compris les applications Web. Il peut simuler une charge lourde sur un serveur, un groupe de serveurs, un réseau ou un objet pour tester sa résistance ou analyser les performances globales sous différents types de charge.

Les principales caractéristiques incluent :

- **Support multi-protocoles** : Capacité de test de charge pour de nombreux types d'applications, serveurs et protocoles différents :
  - Web - HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET, etc.)
  - Services Web SOAP / REST
  - FTP
  - Base de données via JDBC
  - LDAP
  - Middleware orienté message (MOM) via JMS
  - Messagerie - SMTP(S), POP3(S) et IMAP(S)
  - Commandes natives ou scripts shell
  - TCP
  - Objets Java

- **IDE de test complet** : Permet l'enregistrement rapide de plans de test (depuis les navigateurs ou applications natives), la construction et le débogage

- **Mode CLI** : Mode ligne de commande (anciennement appelé mode Non GUI ou mode headless) pour les tests de charge depuis n'importe quel OS compatible Java (Linux, Windows, Mac OSX, etc.)

- **Rapports HTML dynamiques** : Génération de rapports HTML dynamiques complets et prêts à présenter

- **Extraction de données** : Corrélation facile grâce à la capacité d'extraire des données des formats populaires comme HTML, JSON, XML ou tout format texte

- **Portabilité 100% Java** : Entièrement portable et implémentation 100% pure Java

- **Framework multi-threads** : Échantillonnage concurrent par de nombreux threads et échantillonnage simultané de différentes fonctions par des groupes de threads séparés

- **Hautement extensible** : Échantillonneurs branchables, Échantillonneurs scriptables (langages compatibles JSR223 comme Groovy et BeanShell)

- **Intégration continue** : Support pour l'intégration avec Maven, Gradle et Jenkins

## JMeter n'est pas un navigateur

JMeter fonctionne au niveau du protocole. En ce qui concerne les services Web et les services distants, JMeter ressemble à un navigateur (ou plutôt à plusieurs navigateurs) ; cependant, JMeter n'effectue pas toutes les actions prises en charge par les navigateurs. En particulier, JMeter n'exécute pas le JavaScript trouvé dans les pages HTML, ni ne rend les pages HTML comme le fait un navigateur.

## Scénarios d'application de JMeter

- **Tests de charge** : Simuler un grand nombre d'utilisateurs simultanés pour tester les performances du système sous charge élevée
- **Tests de performance** : Mesurer le temps de réponse et le débit du système sous diverses conditions de charge
- **Tests fonctionnels** : Vérifier le comportement fonctionnel de l'application
- **Tests de contrainte** : Tester la stabilité et la tolérance aux pannes du système sous haute pression
- **Tests distribués** : Support pour les tests de charge coordonnés sur plusieurs machines
