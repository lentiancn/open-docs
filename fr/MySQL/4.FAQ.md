# FAQ MySQL

## Problèmes d'installation

### Q : Que faire si le service MySQL ne démarre pas ?

A : Vérifiez les points suivants :
1. Confirmez que MySQL est correctement installé
2. Vérifiez les journaux d'erreurs : `mysqld --console`
3. Vérifiez que le port n'est pas utilisé : `netstat -an | grep 3306`
4. Assurez-vous que les permissions du répertoire de données sont correctes
5. Essayez de réinitialiser le répertoire de données

### Q : Comment réinitialiser le mot de passe root si je l'ai oublié ?

A : 
1. Arrêtez le service MySQL
2. Démarrez en mode sans tables de privilèges : `mysqld --skip-grant-tables`
3. Connectez-vous à MySQL
4. Réinitialisez le mot de passe : `UPDATE mysql.user SET Password=PASSWORD('nouveau_mot_de_passe') WHERE User='root';`
5. Actualisez les privilèges : `FLUSH PRIVILEGES;`
6. Redémarrez le service MySQL

### Q : Comment changer le numéro de port MySQL ?

A : Modifiez le fichier de configuration my.cnf ou my.ini :
```ini
[mysqld]
port=3307
```
Redémarrez ensuite le service MySQL.

## Problèmes de connexion

### Q : Que faire si la connexion distante est refusée ?

A : 
1. Confirmez que MySQL autorise les connexions distantes (vérifiez l'hôte de l'utilisateur dans la table des utilisateurs)
2. Vérifiez que le pare-feu autorise le port
3. Assurez-vous que bind-address n'est pas 127.0.0.1 dans la configuration MySQL
4. Vérifiez les paramètres SELinux ou AppArmor

### Q : Comment gérer les délais de connexion ?

A : 
1. Vérifiez la connexion réseau
2. Augmentez la configuration wait_timeout
3. Utilisez des connexions persistantes ou des pools de connexions
4. Vérifiez la charge du serveur

## Problèmes d'exploitation de base de données

### Q : Comment gérer les problèmes de jeu de caractères ?

A : 
1. Spécifiez le jeu de caractères lors de la création de la base : `CREATE DATABASE db CHARACTER SET utf8mb4;`
2. Spécifiez le jeu de caractères lors de la connexion : `SET NAMES utf8mb4;`
3. Définissez le jeu de caractères par défaut dans la configuration :
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

### Q : Comment gérer l'import/export avec des problèmes de caractères ?

A : 
1. Spécifiez le jeu de caractères lors de l'export : `mysqldump --default-character-set=utf8mb4`
2. Spécifiez le jeu de caractères lors de l'import : `mysql --default-character-set=utf8mb4`

### Q : Est-il normal que les clés primaires auto-incrémentées ne soient pas séquentielles ?

A : Oui, c'est normal. Les raisons incluent :
- Les insertions ratées consomment des IDs
- Les rollbacks de transactions ne récupèrent pas les IDs utilisés
- Les insertions par lots peuvent causer des sauts
- La suppression d'enregistrements n'affecte pas le compteur auto-incrémentiel

## Problèmes de performances

### Q : Que faire si les requêtes sont lentes ?

A : 
1. Utilisez EXPLAIN pour analyser le plan d'exécution des requêtes
2. Créez des index pour les colonnes utilisées dans WHERE, JOIN, ORDER BY
3. Évitez les scans de tables complets
4. Optimisez les requêtes, évitez SELECT *
5. Envisagez l'utilisation du cache de requêtes (supprimé dans MySQL 8.0)
6. Divisez les tables ou les bases de données pour réduire la pression

### Q : Comment voir les requêtes lentes ?

A : 
1. Activez la journalisation des requêtes lentes :
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```
2. Affichez l'emplacement du journal des requêtes lentes :
```sql
SHOW VARIABLES LIKE 'slow_query_log_file';
```

### Q : Comment optimiser une utilisation élevée de la mémoire ?

A : 
1. Ajustez innodb_buffer_pool_size (70% de la mémoire disponible recommandé)
2. Optimisez les requêtes pour réduire les scans inutiles
3. Limitez le nombre de connexions
4. Nettoyez les caches inutiles

## Problèmes de sauvegarde et restauration

### Q : Que faire si le fichier de sauvegarde est trop volumineux ?

A : 
1. Utilisez une sauvegarde compressée : `mysqldump | gzip > backup.sql.gz`
2. Sauvegardez les bases/tables séparément
3. Utilisez xtrabackup pour les sauvegardes physiques
4. Nettoyez régulièrement les données historiques

### Q : Comment implémenter une sauvegarde incrémentale ?

A : 
1. Activez la journalisation binaire : `log-bin=mysql-bin`
2. Sauvegardez régulièrement les journaux binaires
3. Utilisez l'outil mysqlbinlog pour restaurer les données incrémentales

## Problèmes de réplication

### Q : Que faire si le décalage de réplication maître-esclave est trop élevé ?

A : 
1. Vérifiez la latence réseau
2. Ajustez slave_parallel_workers pour augmenter la réplication parallèle
3. Assurez-vous que le matériel de l'esclave est suffisant
4. Optimisez la vitesse d'écriture du maître
5. Vérifiez les règles de filtre de réplication

### Q : Comment gérer l'incohérence des données maître-esclave ?

A : 
1. Resynchronisez l'esclave
2. Utilisez pt-table-checksum pour vérifier l'incohérence
3. Utilisez pt-table-sync pour réparer les données
4. Reconfigurez la réplication maître-esclave

## Problèmes de sécurité

### Q : Comment renforcer la sécurité MySQL ?

A : 
1. Définissez des politiques de mot de passe fortes
2. Limitez la connexion distante pour l'utilisateur root
3. Supprimez les comptes anonymes
4. Supprimez la base de données de test
5. Mettez régulièrement à jour la version MySQL
6. Activez les connexions chiffrées SSL/TLS
7. Configurez les règles du pare-feu
8. Suivez le principe du moindre privilège

### Q : Comment prévenir l'injection SQL ?

A : 
1. Utilisez des instructions préparées (Prepared Statements)
2. Validez et filtrez les entrées utilisateur
3. Limitez les privilèges des utilisateurs de base de données
4. Évitez la concaténation dynamique de SQL
5. Utilisez des procédures stockées
