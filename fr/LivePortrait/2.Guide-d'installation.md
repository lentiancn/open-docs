# Guide d'installation de LivePortrait

Ce document fournit des instructions d'installation détaillées pour LivePortrait sur différents systèmes d'exploitation et environnements.

## Configuration Requise

Avant l'installation, assurez-vous que votre système répond aux exigences suivantes :

**Système d'Exploitation** : Linux (recommandé), Windows 10/11, macOS (nécessite Apple Silicon)

**Version Python** : Python 3.10

**Configuration Matérielle** : GPU NVIDIA (RTX 4090 ou autre GPU compatible CUDA recommandé), minimum 8Go de VRAM

**Espace de Stockage** : Minimum 20Go recommandés pour les poids des modèles et l'environnement d'exécution

## Configuration de l'Environnement de Base

### Vérifier la Version CUDA

Tout d'abord, confirmez la version CUDA de votre système pour installer la version appropriée de PyTorch :

```bash
nvcc -V
```

Les versions CUDA courantes incluent 11.1, 11.8 et 12.1. En fonction du numéro de version affiché, sélectionnez la commande d'installation PyTorch correspondante.

### Installer PyTorch

Choisissez la commande d'installation PyTorch appropriée en fonction de votre version CUDA :

**Pour CUDA 11.1** :

```bash
pip install torch==1.10.1+cu111 torchvision==0.11.2 torchaudio==0.10.1 -f https://download.pytorch.org/whl/cu111/torch_stable.html
```

**Pour CUDA 11.8** :

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu118
```

**Pour CUDA 12.1** :

```bash
pip install torch==2.3.0 torchvision==0.18.0 torchaudio==2.3.0 --index-url https://download.pytorch.org/whl/cu121
```

Si vous avez uniquement besoin d'exécuter le mode Humain, vous pouvez ignorer la vérification de version CUDA et installer directement la dernière version de PyTorch.

## Télécharger le Code du Projet

Clonez le dépôt LivePortrait en utilisant Git :

```bash
git clone https://github.com/KlingTeam/LivePortrait
cd LivePortrait
```

## Créer un Environnement Python

Il est recommandé de créer un environnement Python isolé en utilisant conda :

```bash
conda create -n LivePortrait python=3.10
conda activate LivePortrait
```

Ou en utilisant venv :

```bash
python -m venv LivePortrait
# Linux/macOS
source LivePortrait/bin/activate
# Windows
LivePortrait\Scripts\activate
```

## Installer les Dépendances

Exécutez ce qui suit dans le répertoire racine du projet :

```bash
pip install -r requirements.txt
```

## Télécharger les Modèles Pré-entraînés

Les modèles pré-entraînés peuvent être téléchargés depuis HuggingFace. Si votre réseau peut accéder à HuggingFace :

```bash
pip install -U "huggingface_hub[cli]"
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Si vous ne pouvez pas accéder à HuggingFace, utilisez un site miroir :

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download KlingTeam/LivePortrait --local-dir pretrained_weights --exclude "*.git*" "README.md" "docs"
```

Alternativement, vous pouvez télécharger tous les fichiers de modèle depuis Google Drive ou Baidu Netdisk, les extraire et les placer dans le répertoire ./pretrained_weights.

## Vérifier l'Installation

Après l'installation, exécutez le script de test pour vérifier que l'environnement est correctement configuré :

```bash
python inference.py
```

Si tout fonctionne correctement, vous verrez le fichier de sortie généré dans le répertoire animations.

## Étapes d'Installation Supplémentaires pour les Modèles Animaux

Le mode Animal n'est pris en charge que sur les systèmes Linux et Windows avec GPU NVIDIA.

### Installer la Dépendance X-Pose

Le mode Animal nécessite le projet X-Pose comme framework de détection de points clés. Premièrement, accédez au répertoire X-Pose :

```bash
cd src/utils/dependencies/XPose/models/UniPose/ops
python setup.py build install
cd -
```

### Télécharger les Modèles Animaux depuis HuggingFace

```bash
huggingface-cli download KlingTeam/LivePortrait --include "animal*" --local-dir pretrained_weights
```

## Notes Spéciales pour Windows

Sur les systèmes Windows, certaines versions plus récentes de CUDA (comme 12.4, 12.6) peuvent causer des problèmes de compatibilité. Il est recommandé de rétrograder CUDA à la version 11.8 pour une meilleure stabilité.

Les utilisateurs Windows peuvent également choisir de télécharger le programme d'installation en un clic et l'exécuter en double-cliquant sur run_windows.bat après extraction :

```bash
# Télécharger le programme d'installation en un clic pour Windows depuis HuggingFace
# Adresse : https://huggingface.co/cleardusk/LivePortrait-Windows
```

## Notes Spéciales pour macOS

### macOS avec Apple Silicon

Pour les utilisateurs macOS avec Apple Silicon, utilisez le fichier de dépendances spécial :

```bash
pip install -r requirements_macOS.txt
```

Lors de l'exécution de l'inférence, définissez la variable d'environnement :

```bash
PYTORCH_ENABLE_MPS_FALLBACK=1 python inference.py
```

Note : L'exécution de l'inférence sur macOS avec Apple Silicon peut être environ 20 fois plus lente que sur RTX 4090.

### Mac Intel

Les Mac basés sur Intel n'ont pas été testés officiellement et peuvent ne pas fonctionner correctement.

### Limitation du Mode Animal

Comme la dépendance X-Pose ne prend pas en charge macOS, le mode Animal n'est pas disponible sur macOS. Seul le mode Humain peut être utilisé.

## Vérifier la Structure des Répertoires

Après l'installation, assurez-vous que la structure des répertoires est conforme à ce qui suit :

```
LivePortrait/
├── pretrained_weights/     # Répertoire des poids des modèles pré-entraînés
├── assets/                # Répertoire des exemples de matériaux
│   ├── examples/
│   │   ├── source/        # Image/vidéo source
│   │   └── driving/       # Vidéo de pilotage
├── src/                   # Répertoire du code source
├── inference.py          # Script d'inférence
├── app.py                # Interface Gradio
└── requirements.txt      # Liste des dépendances
```

## Problèmes Courants

Si vous rencontrez des problèmes pendant l'installation, vérifiez les points suivants :

**CUDA non disponible** : Confirmez que les pilotes NVIDIA sont correctement installés ; exécutez nvidia-smi pour vérifier l'état du pilote

**VRAM insuffisante** : Essayez d'utiliser des images d'entrée plus petites ou de réduire la taille du lot

**Échec du téléchargement du modèle** : Vérifiez la connexion réseau ou utilisez des sites miroirs nationaux

**Conflits de dépendances** : Il est recommandé d'utiliser un nouvel environnement Python pour éviter les conflits avec d'autres projets
