# APISIX 常见问题

## 基础问题

### Q1: 为什么要使用 APISIX？

随着组织向云原生微服务架构演进，需要一个高性能、灵活、安全且可扩展的 API 网关。APISIX 在这些指标上优于其他 API 网关，同时支持多协议、细粒度路由和多语言插件。

### Q2: APISIX 与其他 API 网关有何不同？

- **配置存储**：使用 etcd 而非 PostgreSQL 或 MySQL，实时配置同步更高效
- **完全动态**：无需重启即可修改配置
- **热加载**：支持插件热加载

### Q3: APISIX 性能如何？

APISIX 单核 QPS 达 18,000，平均延迟仅 0.2ms，是性能最优的 API 网关。

### Q4: APISIX 支持哪些平台？

APISIX 平台无关，避免供应商锁定。可运行在裸机、Kubernetes、Apple Silicon 等环境。

## 安装问题

### Q5: Docker 容器启动失败

**解决**：
1. 检查 Docker 日志：`docker logs apisix`
2. 确认 etcd 是否正常运行
3. 检查端口是否被占用

### Q6: LuaRocks 安装超时

**解决**：使用国内源

```bash
make deps ENV_LUAROCKS_SERVER=https://luarocks.cn
```

### Q7: etcd 连接失败

**解决**：
1. 确认 etcd 服务正在运行
2. 检查 etcd 端口（默认 2379）是否可访问
3. 验证配置中的 endpoints 地址

## 配置问题

### Q8: 如何启用 Admin API 鉴权？

在 `config.yaml` 中配置：

```yaml
deployment:
  admin:
    admin_key:
      - name: admin
        key: your-secret-key
        role: admin
```

使用密钥访问：

```bash
curl -H "X-API-Key: your-secret-key" \
  http://127.0.0.1:9180/apisix/admin/routes
```

### Q9: 如何配置多端口监听？

```yaml
apisix:
  node_listen:
    - 9080
    - 9081
    - 9082
```

### Q10: 如何配置 HTTPS？

1. 获取 SSL 证书
2. 配置 SSL 路由：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "https-route",
  "uri": "/api/*",
  "plugins": {
    "ssl": {
      "cert": "/path/to/cert.pem",
      "key": "/path/to/key.pem"
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

## 使用问题

### Q11: 如何实现 HTTP 到 HTTPS 重定向？

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "redirect-route",
  "uri": "/hello",
  "plugins": {
    "redirect": {
      "http_to_https": true
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "backend:8080": 1
    }
  }
}'
```

### Q12: 如何实现灰度发布？

使用 traffic-split 插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/routes" -X PUT -d '
{
  "id": "canary",
  "uri": "/api/*",
  "plugins": {
    "traffic-split": {
      "rules": [
        {
          "weight": 10
        }
      ]
    }
  },
  "upstream": {
    "type": "roundrobin",
    "nodes": {
      "new-backend:8080": 1
    }
  }
}'
```

### Q13: 如何实现 A/B 测试？

使用 request.Request header 或 cookie 进行路由：

```json
{
  "uri": "/api/*",
  "vars": [
    ["http_x_ab_version", "==", "v2"]
  ],
  "upstream": {
    "nodes": {
      "v2-backend:8080": 1
    }
  }
}
```

### Q14: 插件不生效

**解决**：
1. 确认插件名称正确
2. 检查插件配置格式
3. 查看错误日志

### Q15: 路由匹配失败

**解决**：
1. 检查 uri 配置是否正确
2. 确认上游节点可用
3. 使用 `curl -v` 查看详细请求信息

### Q18: 如何编写 Lua 插件？

```lua
local plugin_name = "my-plugin"

local schema = {
    type = "object",
    properties = {
        value = {type = "string"}
    },
    required = {"value"},
}

local _M = {
    version = 0.1,
    priority = 1000,
    name = plugin_name,
    schema = schema,
}

function _M.access(conf, ctx)
    ngx.log(ngx.INFO, "Plugin executed")
end

return _M
```

## 性能问题

### Q19: 性能不达预期

**解决**：
1. 调整 worker 进程数：`nginx.worker_processes`
2. 增加连接数：`nginx.worker_connections`
3. 启用 HTTP/2
4. 禁用不必要的插件

### Q20: 内存占用过高

**解决**：
1. 检查是否有内存泄漏的插件
2. 限制缓存大小
3. 定期清理日志

## 日志问题

### Q21: 如何修改日志级别？

```yaml
nginx_config:
  error_log_level: "info"
```

### Q22: 如何配置访问日志？

```yaml
nginx:
  access_log: /var/log/apisix/access.log
```

## 集成问题

### Q23: 如何与 Kubernetes 集成？

使用 APISIX Ingress Controller：

```bash
helm install apisix-ingress apisix/apisix-ingress \
  --set ingressController.enabled=true
```

### Q24: 如何与 Prometheus 集成？

启用 Prometheus 插件：

```bash
curl -i "http://127.0.0.1:9180/apisix/admin/plugins" -X PUT -d '
{
  "prometheus": {}
}'
```

Prometheus 指标端点：`http://127.0.0.1:9091/apisix/prometheus/metrics`

## 常见错误码

| 错误码 | 说明 | 解决方法 |
|--------|------|----------|
| 400 | 请求参数错误 | 检查请求格式和参数 |
| 401 | 未授权 | 配置并使用 API 密钥 |
| 403 | 禁止访问 | 检查权限配置 |
| 404 | 路由不存在 | 确认路由配置 |
| 429 | 超过限流 | 调整限流配置 |
| 500 | 服务器内部错误 | 检查日志 |
| 502 | 网关错误 | 检查上游服务 |
| 503 | 服务不可用 | 检查上游健康状态 |
