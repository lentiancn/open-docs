# Claude Code 使用手册

本手册将带你快速上手 Claude Code，掌握日常开发中常用的操作。

## 准备工作

开始之前，请确保你具备以下条件：

- 已打开终端或命令提示符
- 有一个代码项目
- 拥有 Claude 订阅（Pro、Max、Teams 或 Enterprise）、Claude Console 账户，或通过支持的云提供商访问

## 启动会话

在项目目录中打开终端，启动 Claude Code：

```bash
cd /path/to/your/project
claude
```

首次使用时会看到欢迎屏幕，显示会话信息、最近的对话和最新更新。输入 `/help` 查看可用命令，或输入 `/resume` 继续之前的对话。

## 基本操作

### 了解代码库

让 Claude 分析你的项目：

```text
这个项目是做什么的？
```

```text
这个项目使用了哪些技术？
```

```text
主入口文件在哪里？
```

```text
解释一下文件夹结构
```

你也可以问 Claude 它的能力：

```text
Claude Code 能做什么？
```

```text
如何在 Claude Code 中创建自定义技能？
```

**注意：Claude Code 会根据需要读取项目文件，不需要手动添加上下文。**

### 第一次代码修改

让 Claude Code 完成实际的编码工作，尝试一个简单的任务：

```text
在主文件中添加一个 hello world 函数
```

Claude Code 会：
1. 找到合适的文件
2. 展示拟定的更改
3. 请求你的批准
4. 执行编辑

**注意：Claude Code 始终会在修改文件之前请求许可。你可以批准单个更改或为会话启用"全部接受"模式。**

### Git 操作

Claude Code 让 Git 操作变得简单：

```text
我修改了哪些文件？
```

```text
用描述性消息提交我的更改
```

```text
创建一个名为 feature/quickstart 的新分支
```

```text
显示最近 5 次提交
```

```text
帮我解决合并冲突
```

### 修复 Bug 和添加功能

Claude 擅长调试和功能实现。

用自然语言描述你想要的内容：

```text
为用户注册表单添加输入验证
```

或者修复现有问题：

```text
有一个 bug，用户提交空表单后没有提示 - 修复它
```

Claude Code 会：
- 定位相关代码
- 理解上下文
- 实现解决方案
- 运行测试（如果有）

### 常用工作流

**重构代码**

```text
将认证模块重构为使用 async/await 而不是回调
```

**编写测试**

```text
为计算器函数编写单元测试
```

**更新文档**

```text
用安装说明更新 README
```

**代码审查**

```text
审查我的更改并提出改进建议
```

## 常用命令

以下是日常使用中最重要的命令：

| 命令 | 功能 | 示例 |
|------|------|------|
| `claude` | 启动交互模式 | `claude` |
| `claude "任务"` | 运行一次性任务 | `claude "fix the build error"` |
| `claude -p "查询"` | 运行一次性查询后退出 | `claude -p "explain this function"` |
| `claude -c` | 在当前目录继续最近的对话 | `claude -c` |
| `claude -r` | 恢复之前的对话 | `claude -r` |
| `claude commit` | 创建 Git 提交 | `claude commit` |
| `/clear` | 清除对话历史 | `/clear` |
| `/help` | 显示可用命令 | `/help` |
| `exit` 或 Ctrl+C | 退出 Claude Code | `exit` |

## 进阶技巧

### 使用 CLAUDE.md

在项目根目录创建 `CLAUDE.md` 文件，为 Claude Code 提供持久的项目上下文。这个文件会在每次会话开始时自动加载。

CLAUDE.md 可以包含：
- 代码标准和规范
- 架构决策
- 首选库
- 审查清单
- 项目特定的命令和脚本

### 自定义技能

创建自定义命令来封装可重复的工作流，例如 `/review-pr` 或 `/deploy-staging`。

### 使用 Hooks

Hooks 允许你在 Claude Code 操作前后运行 shell 命令，例如：
- 每次编辑文件后自动格式化
- 提交前运行 lint

### MCP 服务器

通过 Model Context Protocol 将 Claude Code 连接到外部工具和数据源：
- 读取 Google Drive 中的设计文档
- 更新 Jira 工单
- 从 Slack 获取数据
- 使用自定义工具

### 多代理协作

对于复杂任务，可以同时启动多个 Claude Code 代理并行工作。牵头代理协调工作、分配子任务并合并结果。

### 远程控制

工作不局限于单一设备：
- 使用远程控制从手机或其他设备继续本地会话
- 在 Web 或 iOS 应用上启动长时间运行的任务，然后用 `/teleport` 拉回到终端
- 使用 `/desktop` 将终端会话切换到桌面应用进行可视化 diff 审查

## 最佳实践

### 请求要具体

❌ 不要说："修复 bug"

✅ 应该说："修复登录 bug，用户输入错误凭据后看到空白屏幕"

### 使用逐步指令

将复杂任务分解为步骤：

```text
1. 为用户配置文件创建一个新的数据库表
2. 创建获取和更新用户配置文件的 API 端点
3. 构建一个允许用户查看和编辑信息的网页
```

### 让 Claude 先探索

在做出更改之前，让 Claude 理解你的代码：

```text
分析数据库模式
```

```text
构建一个仪表板，显示英国客户退货最多的产品
```

### 使用快捷键节省时间

- 按 `?` 查看所有可用的键盘快捷键
- 使用 Tab 补全命令
- 按 `↑` 查看命令历史
- 输入 `/` 查看所有命令和技能

## 常见问题

**Claude Code 读取我的代码吗？**

是的，Claude Code 会读取项目文件来理解代码库。它不会将你的代码发送到外部服务器（除非你使用第三方 API 提供商）。

**Claude Code 会自动修改我的文件吗？**

不会。Claude Code 始终会请求许可才会修改文件。你可以批准单个更改或为整个会话启用"全部接受"模式。

**我的代码安全吗？**

Claude Code 设计注重安全性。你的代码不会被发送给 Anthropic 用于训练模型（除非你明确选择加入）。你可以在 [Anthropic 隐私政策](https://www.anthropic.com/privacy)中了解更多。
