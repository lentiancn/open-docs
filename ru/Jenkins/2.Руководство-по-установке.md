# Руководство по установке

## Системные требования

Перед установкой Jenkins убедитесь, что ваша система соответствует следующим минимальным требованиям:

### Минимальная рекомендуемая конфигурация

- 256 МБ свободной оперативной памяти
- 1 ГБ свободного дискового пространства (10 ГБ рекомендуется при запуске в Docker-контейнере)

### Рекомендуемая конфигурация для небольших команд

- 1 ГБ+ свободной оперативной памяти
- 50 ГБ+ свободного дискового пространства

### Программные требования

- Java 8 или выше (подходит как JRE, так и JDK)
- Примечание: при запуске Jenkins в Docker-контейнере устанавливать Java на хосте не требуется

## Платформы установки

Jenkins поддерживает несколько способов установки. В этом руководстве описаны наиболее распространенные подходы.

## Установка через Docker

Docker — рекомендуемый способ установки Jenkins, поскольку он предоставляет изолированную среду и упрощает процесс настройки.

### Установка Docker

Сначала установите Docker в своей системе. Посетите официальный веб-сайт Docker и выберите подходящую версию Docker Community Edition для вашей операционной системы.

### Загрузка и запуск Jenkins

В macOS и Linux откройте терминал и выполните:

```bash
docker run \
  -u root \
  --rm \
  -d \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkinsci/blueocean
```

В Windows откройте командную строку и выполните аналогичную команду:

```
docker run ^
  -u root ^
  --rm ^
  -d ^
  -p 8080:8080 ^
  -p 50000:50000 ^
  -v jenkins-data:/var/jenkins_home ^
  -v /var/run/docker.sock:/var/run/docker.sock ^
  jenkinsci/blueocean
```

Описание параметров:

- `-u root`: запуск контейнера от имени пользователя root
- `--rm`: автоматическое удаление контейнера при остановке
- `-d`: запуск в фоновом режиме (отсоединенный режим)
- `-p 8080:8080`: сопоставление порта контейнера 8080 с портом хоста 8080
- `-p 50000:50000`: сопоставление порта контейнера 50000 с портом хоста 50000 (для связи агентов JNLP)
- `-v jenkins-data:/var/jenkins_home`: сохранение данных Jenkins
- `-v /var/run/docker.sock:/var/run/docker.sock`: разрешение Jenkins выполнять команды Docker в конвейерах
- `jenkinsci/blueocean`: официальный образ Jenkins с интерфейсом Blue Ocean

### Доступ к контейнеру Jenkins

Для доступа внутрь контейнера Jenkins используйте:

```bash
docker exec -it jenkins-blueocean bash
```

### Просмотр журналов Jenkins

Просмотр журналов консоли Jenkins:

```bash
docker logs jenkins-blueocean
```

## Установка через WAR-файл

Версия Jenkins в виде WAR-файла может быть установлена на любой операционной системе с поддержкой Java.

### Шаги установки

1. Скачайте последний стабильный пакет WAR Jenkins с официального сайта
2. Откройте терминал в каталоге загрузки
3. Выполните команду: `java -jar jenkins.war`
4. Перейдите по адресу http://localhost:8080 и дождитесь появления страницы разблокировки Jenkins

### Примечания

- При установке через WAR-файл Blue Ocean не устанавливается автоматически; его нужно установить отдельно через управление плагинами
- Изменить порт можно с помощью параметра `--httpPort`, например: `java -jar jenkins.war --httpPort=9090`

## Установка в macOS

### Использование пакета

1. Скачайте пакет Jenkins для macOS
2. Дважды щелкните, чтобы открыть, и следуйте инструкциям установщика

### Использование Homebrew

Установить последнюю версию:

```bash
brew install jenkins
```

Установить LTS-версию:

```bash
brew install jenkins-lts
```

## Установка в Linux (Debian/Ubuntu)

В дистрибутивах на основе Debian Jenkins можно установить через apt:

```bash
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt-get update
sudo apt-get install jenkins
```

После установки Jenkins:

- Будет настроен на запуск в качестве демона при загрузке
- Создаст пользователя jenkins для запуска службы
- Выведет журналы в /var/log/jenkins/jenkins.log
- По умолчанию будет прослушивать порт 8080

## Мастер настройки после установки

При первом запуске Jenkins начинается мастер настройки.

### Разблокировка Jenkins

1. Перейдите по адресу http://localhost:8080
2. Когда появится страница разблокировки Jenkins, скопируйте автоматически сгенерированный пароль из журналов консоли
3. Вставьте пароль в поле пароля администратора и нажмите «Продолжить»

### Настройка плагинов Jenkins

На странице установки плагинов можно выбрать:

- Установить рекомендуемые плагины: устанавливает набор плагинов на основе распространенных сценариев использования
- Выбрать плагины для установки: вручную выбрать плагины для установки

### Создание первого администратора

Создайте учетную запись администратора, указав:

- Имя пользователя
- Пароль
- Подтверждение пароля
- Полное имя
- Адрес электронной почты

Нажмите «Сохранить и завершить», затем нажмите «Начать использовать Jenkins» для входа в главный интерфейс.

## Проверка установки

Проверьте установку:

1. Перейдите по адресу http://localhost:8080
2. Войдите с созданной учетной записью администратора
3. Проверьте, что статус системы нормальный

## Распространенные проблемы

### Порт уже занят

Если порт 8080 используется другой программой:

- Способ Docker: изменить параметр -p
- Способ WAR: использовать параметр --httpPort

### Недостаточно памяти

Если системной памяти мало:

- Использовать Docker с ограничениями ресурсов контейнера
- Настроить размер кучи JVM в параметрах запуска Jenkins

### Медленный первый запуск

При первом запуске загружаются и устанавливаются плагины, что может занять время. Пожалуйста, будьте терпеливы.

## Следующие шаги

После установки рекомендуется изучить:

- Создание первой задачи конвейера
- Настройка исходного кода (например, Git)
- Настройка триггеров сборки
- Интеграция инструментов тестирования и развертывания
