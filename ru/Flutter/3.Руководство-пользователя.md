# Руководство пользователя Flutter

## Основные концепции

### Введение в виджеты

Виджет — это основной строительный блок пользовательского интерфейса Flutter. B Flutter все является виджетом, включая макеты, padding, align и т.д. Виджеты описывают, как должен выглядеть их вид с учетом текущей конфигурации и состояния.

Виджеты Flutter являются неизменяемыми. Когда вам нужно изменить пользовательский интерфейс, вам нужно пересоздать виджет или использовать StatefulWidget с управлением состоянием. Виджеты могут быть вложены для формирования дерева виджетов, и Flutter эффективно сравнивает старое и новое дерево виджетов, отрисовывая только фактически изменившиеся части.

Распространенные виджеты включают: Text (текст), Image (изображение), Icon (иконка), Container, Row/Column (горизонтальный/вертикальный макет), Stack (стековый макет), ListView (список) и т.д.

### StatelessWidget и StatefulWidget

StatelessWidget — это виджет без состояния, подходящий для статических отображений пользовательского интерфейса, таких как текст и изображения. Определение StatelessWidget требует только реализации метода build, возвращающего дерево виджетов для отрисовки.

```dart
class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Hello Flutter');
  }
}
```

StatefulWidget — это виджет с состоянием, подходящий для сценариев, требующих динамического обновления пользовательского интерфейса, таких как счетчики и формы. StatefulWidget должен использоваться с State, вызывая метод setState для уведомления Flutter о необходимости перерисовать при изменении состояния.

```dart
class Counter extends StatefulWidget {
  @override
  State<Counter> createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _count = 0;
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $_count'),
        ElevatedButton(
          onPressed: () => setState(() => _count++),
          child: Text('Increment'),
        ),
      ],
    );
  }
}
```

## Основы макетов

### Линейный макет

Row и Column — наиболее часто используемые виджеты линейного макета. Row организует дочерние виджеты горизонтально, Column — вертикально. Вы можете использовать MainAxisAlignment и CrossAxisAlignment для управления выравниванием.

```dart
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.home),
    Text('Home'),
    Icon(Icons.settings),
  ],
)
```

### Стековый макет

Stack используется для создания стековых макетов, размещая виджеты поверх других виджетов. Positioned используется для позиционирования дочерних элементов в Stack.

```dart
Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('Overlay'),
    ),
  ],
)
```

### Списочный макет

ListView — наиболее часто используемый виджет списка, подходящий для длинных списков. ListView.builder подходит для больших наборов данных с ленивой загрузкой. ListView.separated может добавлять разделители.

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) => ListTile(
    title: Text(items[index]),
  ),
)
```

## Управление состоянием

### setState

setState — наиболее базовый метод управления состоянием, подходящий для простого локального состояния. B StatefulWidget вызов setState триггерит пересборку виджета.

```dart
setState(() {
  _counter++;
});
```

### Provider

Provider — одно из рекомендуемых Flutter решений для управления состоянием. Через Provider можно передавать состояние в дереве виджетов, избегая передачи через каждый уровень.

```dart
// Определить ChangeNotifier
class CounterModel extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Использовать Provider
ChangeNotifierProvider(
  create: (_) => CounterModel(),
  child: Consumer<CounterModel>(
    builder: (context, model, _) => Text('${model.count}'),
  ),
)
```

## Сетевые запросы

### Пакет HTTP

Используйте пакет http для простых HTTP-запросов:

```dart
import 'package:http/http.dart' as http;

final response = await http.get(Uri.parse('https://api.example.com/data'));
```

## Локальное хранилище

### SharedPreferences

Используется для хранения простых данных ключ-значение:

```dart
import 'package:shared_preferences/shared_preferences.dart';

final prefs = await SharedPreferences.getInstance();
await prefs.setString('name', 'Flutter');
final name = prefs.getString('name');
```

## Навигация и маршрутизация

Flutter использует Navigator для навигации между страницами. Основные операции включают Navigator.push (добавить новую страницу), Navigator.pop (назад), Navigator.pushReplacement (заменить текущую страницу) и т.д.

Именованная маршрутизация — рекомендуемый способ управления маршрутами:

```dart
// Определить маршруты
MaterialApp(
  routes: {
    '/home': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

// Переход
Navigator.pushNamed(context, '/detail');
```

## Сборка и публикация

### Публикация Android

Используйте keytool для создания ключа подписи:

```bash
keytool -genkey -v -keystore my-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
```

Настройте подпись в android/app/build.gradle, затем выполните в корне проекта:

```bash
flutter build apk --release
```

### Публикация iOS

Настройте сертификат подписи и профиль обеспечения в Xcode, затем выполните:

```bash
flutter build ios --release
```

Обратите внимание, что для публикации iOS требуется учетная запись разработчика Apple и соответствующие сертификаты.
